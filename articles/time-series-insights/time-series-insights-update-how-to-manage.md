# <a name="provision-and-manage-azure-time-series-insights-preview"></a>Подготовка к работе службы "Аналитика временных рядов Azure" (предварительная версия) и управление ею

В этой статье описывается создание среды "Аналитика временных рядов Azure" (предварительная версия) и управление ею с помощью [портала Azure](https://portal.azure.com/).

## <a name="overview"></a>Обзор

Среды Azure Time Series Insights Previews — это среды с оплатой по *мере вху-гоу* (PAYG).

При создании среды просмотра azure Time Series Insights создается следующие ресурсы Azure:

* Среда "Аналитика временных рядов Azure (предварительная версия)"  
* Учетная запись службы хранилища Azure общего назначения версии 1
* Дополнительный теплый магазин для более быстрых и неограниченных запросов

> [!TIP]
> * [Узнайте, как спланировать среду.](./time-series-insights-update-plan.md)
> * Узнайте о том, как [добавить источник концентратора событий](./time-series-insights-how-to-add-an-event-source-eventhub.md) или как [добавить источник концентратора IoT.](./time-series-insights-how-to-add-an-event-source-iothub.md)

Вы научитесь:

1. **(Необязательно)** Связать каждую среду Azure Time Series Insights с исходным моментом. Вы также предоставите свойство Timestamp ID и уникальную группу потребителей, чтобы гарантировать, что среда имеет доступ к соответствующим событиям.

   > [!NOTE]
   > Предыдущий шаг является необязательным при подготовке среды. Если вы пропустите этот шаг, необходимо прикрепить источник событий к среде позже, чтобы к данным можно было получить доступ в среде.

1. Когда процесс подготовки будет завершен, вы можете изменять политику доступа и другие атрибуты среды в соответствии со своими бизнес-требованиями.

## <a name="create-the-environment"></a>Создание среды

Для создания среды предварительного просмотра серии времени Azure:

1. Выберите **PAYG** в качестве **уровня**. Предоставьте имя среды и выберите для использования группу подписки и группу ресурсов. Затем выберите поддерживаемое место для размещения среды.

   [![Создайте экземпляр службы "Аналитика временных рядов Azure".](media/v2-update-manage/create-and-manage-configuration.png)](media/v2-update-manage/create-and-manage-configuration.png#lightbox)

1. Введите идентификатор временного ряда.

    > [!NOTE]
    > * Идентификатор серии времени является чувствительным к *случаям* и *неизменяемым.* (Его нельзя изменить после установки.)
    > * Иди имитирует time Series до *трех* ключей.
    > * Подробнее о [том, как выбрать идентификатор серии времени](time-series-insights-update-how-to-id.md)

1. Создайте учетную запись Хранилища Azure, выбрав имя учетной записи хранилища и назначав выбор репликации. При этом будет автоматически создана учетная запись хранения Azure общего назначения версии 1. Учетная запись создается в том же регионе, что и среда просмотра azure Time Series Insights, выбранная ранее.

    [![Конфигурация холодного хранения](media/v2-update-manage/create-and-manage-cold-store.png)](media/v2-update-manage/create-and-manage-cold-store.png#lightbox)

1. **(Необязательно)** Включите теплое хранилище для вашей среды, если вы хотите быстрее и неограниченные запросы по самым последним данным в вашей среде. Вы также можете создать или удалить теплый магазин через опцию **конфигурации хранения** в левом навигационном стеле, после создания среды просмотра Time Series Insights.

    [![Теплая конфигурация хранения](media/v2-update-manage/create-and-manage-warm-storage.png)](media/v2-update-manage/create-and-manage-warm-storage.png#lightbox)

1. **(Необязательно)** Теперь можно добавить источник событий. Вы также можете подождать до тех пор, пока экземпляр не будет подготовлен.

   * Time Series Insights поддерживает [концентраторы Azure IoT](./time-series-insights-how-to-add-an-event-source-iothub.md) и [Azure Event hubs](./time-series-insights-how-to-add-an-event-source-eventhub.md) в качестве источников событий. Хотя при создании среды можно добавить только один источник событий, позже можно добавить другой источник событий. 
   
     Вы можете выбрать существующую группу потребителей или создать новую группу потребителей при добавлении источника событий. Лучше всего создать уникальную группу потребителей, чтобы убедиться, что все события будут видны вашей среде Azure Time Series Insights Preview.

   * Выберите соответствующее свойство Timestamp. По умолчанию Azure Time Series Insights использует время, освоенное сообщениями, для каждого источника событий.

     > [!TIP]
     > Время, установленное в сообщениях, может быть не самым оптимальной настройкой для использования в сценариях пакетных событий или сценариях загрузки исторических данных. В таких случаях убедитесь, что вы можете проверить свое решение об использовании или не использовании свойства Timestamp.

     [![Вкладка конфигурации Исхода событий](media/v2-update-manage/create-and-manage-event-source.png)](media/v2-update-manage/create-and-manage-event-source.png#lightbox)

1. Подтвердите, что ваша среда была подготовлена и настроена так, как вы хотите.

    [![Вкладка "Просмотр и создание"](media/v2-update-manage/create-and-manage-review-and-confirm.png)](media/v2-update-manage/create-and-manage-review-and-confirm.png#lightbox)

## <a name="manage-the-environment"></a>Управление средой

Вы можете управлять своей средой службы "Аналитика временных рядов Azure" (предварительная версия) с помощью портала Azure. Существует несколько ключевых различий между средой PAYG Azure Time Series Insights Previews и общедоступной средой S1 или S2, чтобы иметь в виду при управлении средой через портал Azure:

* На портале Azure Preview **Обзор** лезвия имеет следующие изменения:

  * Емкость удаляется, поскольку она не применяется к средам PAYG.
  * Добавлено свойство **идентификатора серии времени.** Оно определяет способ секционирования данных.
  * Удалены наборы эталонных данных.
  * Отображаемый URL-адрес указывает на [проводник службы "Аналитика временных рядов Azure (предварительная версия)"](./time-series-insights-update-explorer.md).
  * Указано имя вашей учетной записи хранения Azure.

* Лезвие **конфигурации** портала Azure удаляется в Azure Time Series Insights Preview, поскольку среды PAYG не настраиваются. Тем не менее, можно использовать **конфигурацию хранения** для настройки недавно введенного теплого магазина.

* Лезвие **справочных данных** портала Azure удаляется в Azure Time Series Insights Preview, поскольку справочные данные не являются частью среды PAYG.

[![Среда "Аналитика временных рядов" (предварительная версия) на портале Azure](media/v2-update-manage/create-and-manage-overview-confirm.png)](media/v2-update-manage/create-and-manage-overview-confirm.png#lightbox)

# <a name="visualize-data-in-the-explorer-preview"></a>Визуализация данных в обозревателе (предварительная версия)

В этой статье описаны функции и возможности, доступные в [веб-приложении обозревателя](https://insights.timeseries.azure.com/preview/samples) службы "Аналитика временных рядов Azure" (предварительная версия).

## <a name="prerequisites"></a>Технические условия

Прежде чем использовать обозреватель службы "Аналитика временных рядов Azure" (предварительная версия), необходимо сделать следующее:

* Настроить среду службы "Аналитика временных рядов". Дополнительные сведения см. в статье [Руководство. Настройка среды "Аналитика временных рядов Azure" (предварительная версия)](./time-series-insights-update-create-environment.md).
* Предоставить доступ к данным среде службы "Аналитика временных рядов", созданной вами для учетной записи. Вы можете предоставить доступ как другим пользователям, так и себе.
* Добавить источник события в среду службы "Аналитика временных рядов" для отправки данных в эту среду.

## <a name="learn-about-the-azure-time-series-insights-preview-explorer"></a>Сведения об обозревателе службы "Аналитика временных рядов Azure" (предварительная версия)

  ![explorer-one][1]

Обозреватель службы "Аналитика временных рядов Azure" (предварительная версия) состоит из следующих элементов:

* **Панель навигации**. Позволяет переключаться между страницами аналитики и модели.
* **Дерево иерархии**. Позволяет выбрать определенные элементы данных для диаграммы.
* **Источник временных рядов**. Отображает текущие выбранные элементы данных.
* **Панель диаграммы**. Отображает текущую рабочую диаграмму.
* **Временная шкала**. Позволяет изменить рабочий интервал времени.
* **Панель приложения**. Содержит параметры управления пользователями, такие как текущий клиент, а также позволяет изменять параметры темы и языка.

## <a name="time-series-insights-preview-environment-panel"></a>Панель среды службы "Аналитика временных рядов" (предварительная версия)

На панели среды отображаются все среды службы "Аналитика временных рядов", к которым у вас есть доступ. В этот список входят среды с оплатой по мере использования (предварительная версия) и среды S1/S2 (общедоступная версия). Просто щелкните среду службы "Аналитика временных рядов", которую следует использовать.

  ![explorer-two][2]

## <a name="time-series-insights-preview-navigation-menu"></a>Меню навигации службы "Аналитика временных рядов" (предварительная версия)

  ![explorer-three][3]

С помощью меню навигации можно переключаться между приложениями службы "Аналитика временных рядов":

* **Анализ**. Позволяет строить диаграммы и выполнять расширенную аналитику по смоделированным или несмоделированным данным временных рядов.

* **Модель**. Позволяет отправлять новые типы, иерархии и экземпляры из службы "Аналитика временных рядов" (предварительная версия) в вашу модель аналитики временных рядов.

## <a name="time-series-insights-preview-model-authoring"></a>Создание модели в службе "Аналитика временных рядов" (предварительная версия)

С помощью этого приложения можно выполнять операции создания, чтения, обновления и удаления (CRUD) над моделью временных рядов.  

* **Тип модели временных рядов**. Типы аналитики временных рядов позволяют определять переменные или формулы для выполнения вычислений. Они связываются с заданным экземпляром службы "Аналитика временных рядов". Один тип может содержать одну или несколько переменных.
* **Иерархия модели временных рядов**. Иерархии представляют собой систематическую организацию данных. Иерархии изображают связи между разными сущностями данных аналитики временных рядов.
* **Экземпляр модели временных рядов**. Экземпляры — это собственно временные ряды. В большинстве случаев они представляют собой DeviceID или AssetID — уникальный идентификатор ресурса в среде.

Дополнительные сведения о модели временных рядов см. в [этой статье](./time-series-insights-update-tsm.md).

## <a name="time-series-insights-preview-model-search-panel"></a>Панель поиска модели в службе "Аналитика временных рядов" (предварительная версия)

Панель поиска модели позволяет легко осуществлять поиск и навигацию в иерархии модели временных рядов, находя определенные экземпляры временных рядов для отображения в диаграмме. При выборе экземпляров они добавляются как в текущую диаграмму, так и в источник данных.

  ![explorer-four][4]

## <a name="time-series-insights-preview-well"></a>Источник в службе "Аналитика временных рядов" (предварительная версия)

В источнике отображаются поля экземпляров и другие метаданные, связанные с выбранными экземплярами временных рядов. Флажки с правой стороны позволяют скрывать или отображать определенные экземпляры в текущей диаграмме. Также можно удалить определенные элементы данных из текущего источника данных, щелкнув элемент управления в виде красного значка "x" справа от соответствующего элемента.

  ![explorer-five][5]

Также можно выдвинуть панель телеметрии для улучшения вертикального представления элементов в источнике данных.

  ![explorer-six][6]

> [!NOTE]
> Если отображается показанное ниже сообщение, это значит, что экземпляр не содержит данных за выбранный интервал времени. Чтобы устранить проблему, можно увеличить интервал времени или убедиться, что экземпляр передает данные.

## <a name="time-series-insights-preview-chart"></a>Диаграмма в службе "Аналитика временных рядов" (предварительная версия)

С помощью диаграммы можно отобразить экземпляры временных рядов в виде линий. Чтобы увеличить диаграмму, можно свернуть панель среды, модель данных и панель управления интервалом времени, щелкнув соответствующие веб-элементы управления.

  ![explorer-eight][8]

1. **Выбранный диапазон дат**. Регулирует, какие элементы данных доступны для визуализации.

1. **Ползунок внутреннего диапазона дат**. Используйте два элемента управления конечными точками, перетаскивая их по нужному интервалу времени.

1. **Элемент управления для сворачивания интервала времени**. Сворачивает и разворачивает редактор панели интервала времени.

1. **Элемент управления форматом оси Y**. Выполняет циклический переход по доступным вариантам отображения оси Y.

    * `Default`: У каждой линии есть отдельная ось Y.
    * `Stacked`: Позволяет одновременно отобразить несколько линий на одной оси Y, данные которой изменяются в зависимости от выбранной линии.
    * `Shared`: Все данные оси Y отображаются вместе.

1. **Текущий элемент данных**. Текущий выбранный элемент данных и связанные с ним сведения.

Можно дополнительно проанализировать определенный срез данных, щелкнув левой кнопкой мыши точку данных на текущем графике, а затем перетащив выделенную область в конечную точку по своему усмотрению. Щелкните правой кнопкой мыши окрашенную серым цветом выделенную область и выберите команду масштабирования, как показано на следующем рисунке:

  ![explorer-nine][9]

После выполнения действия масштабирования вы увидите выбранный набор данных. Щелкните элемент управления форматом оси Y для циклического перебора трех представлений данных аналитики временных рядов на оси Y.

  ![explorer-ten][10]

Здесь показан пример общих осей Y:

  ![explorer-eleven][11]

## <a name="time-series-insights-preview-time-editor-panel"></a>Панель редактора времени в службе "Аналитика временных рядов" (предварительная версия)

При работе со службой "Аналитика временных рядов" (предварительная версия) сначала следует выбрать интервал времени. Выбранный интервал времени определяет набор данных, доступный для операций с помощью мини-приложений службы "Аналитика временных рядов" (предварительная версия). В службе "Аналитика временных рядов" (предварительная версия) для выбора рабочего интервала времени доступны следующие веб-элементы управления.

  ![explorer-twelve][12]

1. **Ползунок внутреннего диапазона дат**. Используйте два элемента управления конечными точками, перетаскивая их по нужному интервалу времени. Этот внутренний диапазон дат ограничивается ползунком внешнего диапазона дат.

1. **Кнопки увеличения и уменьшения диапазона дат**. Интервал времени можно увеличить или уменьшить, выбрав одну из кнопок для требуемого интервала.

1. **Элемент управления для сворачивания интервала времени**. Этот веб-элемент управления позволяет скрыть все элементы управления, за исключением ползунка внутреннего диапазона дат.

1. **Ползунок внешнего диапазона дат**. С помощью элементов управления конечными точками выберите внешний диапазон дат, который будет доступен для элемента управления внутренним диапазоном дат.

1. **Раскрывающийся список быстрого выбора диапазона времени**. Позволяет быстро переключаться между заранее установленными интервалами времени, такими как последние 30 минут, последние 12 часов или настраиваемый диапазон. С изменением этого значения изменяются также доступные диапазоны интервалов, описанные в параграфе о ползунке размера интервала.

1. **Ползунок размера интервала**. Позволяет увеличивать и уменьшать размер интервалов в одном и том же промежутке времени. Это действие позволяет точнее управлять перемещением между большими срезами времени. Оно отображает плавные тенденции вплоть до таких мелких срезов, как миллисекунда, позволяя просматривать детализированные срезы данных с высоким разрешением. Начальная точка ползунка по умолчанию устанавливается как наиболее оптимальный вид данных из вашего выбора, в котором разрешение, скорость запросов и степень детализации сбалансированы.

1. **Веб-элемент управления началом и окончанием диапазона дат**. С помощью этого веб-элемента управления можно легко выбрать щелчком нужные диапазоны дат и времени. Также с его помощью можно переключаться между разными часовыми поясами. Чтобы применить внесенные изменения к текущей рабочей области, выберите **Сохранить**.

   ![explorer-thirteen][13]

## <a name="s1s2-environment-controls"></a>Элементы управления средой S1/S2

### <a name="time-series-insights-preview-terms-panel"></a>Панель условий службы "Аналитика временных рядов" (предварительная версия)

Этот раздел относится только к существующим средам S1/S2, которые пытаются использовать обозреватель в обновленном пользовательском интерфейсе. Возможно, вам потребуется использовать сочетание общедоступной и предварительной версий продукта. Мы добавили некоторые функциональные возможности из существующего пользовательского интерфейса в обновленный обозреватель, но вы можете получить доступ ко всем возможностям пользовательского интерфейса среды S1/S2 среды в существующем обозревателе службы "Аналитика временных рядов".  

Вместо иерархии вы увидите панель условий службы "Аналитика временных рядов", которая служит для определения запросов в вашей среде. Она позволяет фильтровать данные на основе предиката.

  ![explorer-eighteen][18]

Панель редактора условий службы "Аналитика временных рядов" (предварительная версия) принимает следующие параметры:

**Where**. Предложение where позволяет быстро фильтровать события, используя набор операндов, перечисленных в следующей таблице. При выполнении поиска путем выбора операнда предикат автоматически обновляется на основе поиска. Поддерживаемые типы операнд:

| Операция | Поддерживаемые типы   | Примечания |
| --- | --- | --- |
| `<`, `>`, `<=`, `>=` | Double, DateTime, TimeSpan | |
| `=`, `!=`, `<>` | String, Bool, Double, DateTime, TimeSpan, NULL |
| `IN` | String, Bool, Double, DateTime, TimeSpan, NULL | Все операнды должны быть одного типа или являться константой NULL. |
| `HAS` | Строка | Справа разрешены только константы строковых литералов. Пустая строка и значение NULL недопустимы. |

Дополнительные сведения о поддерживаемых операциях запросов и типах данных см. в статье [Time Series Expression (TSX)](https://docs.microsoft.com/rest/api/time-series-insights/preview-tsx) (Язык выражений временных рядов (TSX)).

### <a name="examples-of-where-clauses"></a>Примеры предложений where

  ![explorer-nineteen][19]

**Мера**. В этом раскрывающемся списке отображаются все числовые столбцы (типа **Double**), которые можно использовать в качестве элементов для текущей диаграммы.

**Разделять по**. В этом раскрывающемся списке отображаются все доступные категориальные столбцы (типа String) в модели, по которым можно группировать данные. Можно добавить до пяти просматриваемых условий на одной оси X. Введите требуемые параметры, а затем выберите **Добавить**, чтобы добавить новое условие.

  ![explorer-twenty][20]

На панели диаграммы можно отображать и скрывать элементы, выбрав значок видимости, показанный на следующем изображении. Можно полностью удалить запросы, щелкнув красный значок **x**.

  ![explorer-twenty-one][21]

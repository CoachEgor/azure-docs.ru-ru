# <a name="visualize-data-in-the-explorer-preview"></a>Визуализация данных в обозревателе (предварительная версия)

В этом документе описывается пользовательский интерфейс и возможности для пользователя и интерфейса, предварительная версия Azure время Series Insights [Демонстрация веб-приложения](https://insights.timeseries.azure.com/preview/demo). В частности здесь рассматриваются макета образец, параметры настройки интерфейса и навигации, предоставленный демонстраций.

## <a name="prerequisites"></a>Технические условия

Чтобы начать работу с помощью обозревателя Azure время Series Insights Preview, необходимо сделать следующее:

* Настроить среду службы "Аналитика временных рядов". Дополнительные сведения о подготовке экземпляра, попробуйте [предварительного просмотра аналитики Azure во время ряда](./time-series-insights-update-create-environment.md) руководства.
* [Предоставлять доступ к данным](./time-series-insights-data-access.md) в среду Time Series Insights, созданный для учетной записи. Вы можете предоставить доступ как другим пользователям, так и себе.
* Добавьте источник события для среды Time Series Insights для отправки данных в среде:
  * Узнайте, [способ подключения к концентратору событий](./time-series-insights-how-to-add-an-event-source-eventhub.md).
  * Узнайте, [способ подключения к центру Интернета вещей](./time-series-insights-how-to-add-an-event-source-iothub.md).

## <a name="hierarchy-tree"></a>дерево иерархии

Элемент дерева иерархии отображает выбранные данные элементы, которые включают моделей, определенных устройств и датчиков на устройствах.

### <a name="model-search-panel"></a>Панель поиска модели

На панели поиска модели можно использовать поиск и перехода иерархии модели временных рядов, чтобы найти экземпляры серии определенное время, которые нужно отобразить на диаграмме. После выбора экземпляров, они добавляются в текущую диаграмму и какие данные хорошо.

  [![На панели поиска модели](media/v2-update-explorer/explorer-four.png)](media/v2-update-explorer/explorer-four.png#lightbox)

### <a name="model-authoring"></a>Создание модели

Полный предварительного просмотра Insights рядов во время Azure поддерживает создание, чтение, обновление и удаления (CRUD) в вашей модели временных рядов.

* **Тип модели временных рядов**. Типы аналитики временных рядов можно использовать для определения переменных или формулы для выполнения вычислений. Они связаны с данным экземпляром Time Series Insights. Один тип может содержать одну или несколько переменных.
* **Иерархия модели временных рядов**. Иерархии представляют собой систематическую организацию данных. Иерархии изображают связи между разными сущностями данных аналитики временных рядов.
* **Экземпляр модели временных рядов**. Экземпляры — это собственно временные ряды. В большинстве случаев они **DeviceID** или **AssetID**, который является уникальным идентификатором ресурса в среде.

Дополнительные сведения о модели временных рядов см. в [этой статье](./time-series-insights-update-tsm.md).

## <a name="preview-chart"></a>Предварительный просмотр диаграммы

С диаграммой можно отображать экземпляры Time Series Insights в строках. Панель среды, модели данных и элементе управления span панель времени можно свернуть, выбрав веб-элементы управления, чтобы увеличить диаграмму.

- **Выбрать диапазон дат**: Регулирует, какие элементы данных доступны для визуализации.

- **Ползунком диапазона даты внутреннее**: Используйте эти два элемента управления конечной точки, перетаскивая их за нужный период времени.

    * `Default`: Каждая строка содержит отдельные оси y.
    * `Stacked`: Используйте его для стека несколько строк на одной оси y, с помощью данных оси y, изменение на основе выбранной строки.
    * `Shared`: Все данные по оси y, отображаемые вместе.

- **Текущий элемент данных**. Текущий выбранный элемент данных и связанные с ним сведения.

## <a name="time-editor-panel"></a>Панель редактора времени

При работе со службой "Аналитика временных рядов" (предварительная версия) сначала следует выбрать интервал времени. Выбранный период времени управляет набором данных, доступный для манипуляции с мини-приложений, предварительного просмотра во время Series Insights. Следующие веб-элементы управления доступны в предварительного просмотра Insights рядов во время выбора ваш рабочий интервал времени:

  [![Панель выбора времени](media/v2-update-explorer/explorer-twelve.png)](media/v2-update-explorer/explorer-twelve.png#lightbox)

1. **Ползунком диапазона даты внутреннее**: Используйте эти два элемента управления конечной точки, перетаскивая их за нужный период времени. Этот диапазон дат внутреннее ограничивается управления "ползунок" диапазон внешнего даты.

1. **Кнопки увеличения и уменьшения диапазона дат**. Интервал времени можно увеличить или уменьшить, выбрав одну из кнопок для требуемого интервала.

1. **Элемент управления для сворачивания интервала времени**. Этот веб-элемент управления позволяет скрывать все элементы управления, за исключением ползунком диапазона внутреннее даты.

1. **Управления "ползунок" диапазон внешнего даты**: Выберите диапазон дат внешнее, которое будет доступно для элемента управления диапазона внутреннего даты с помощью элементов управления конечной точки.

1. **Краткое время диапазон дат с раскрывающимся списком**: Позволяет быстро переключаться между span выбранные значения времени, таких как последний **30 минут**, **последние 12 часов**, или **пользовательский диапазон**. С изменением этого значения изменяются также доступные диапазоны интервалов, описанные в параграфе о ползунке размера интервала.

1. **Ползунок размера интервала**. Используйте его для изменения масштаба и из него интервалов в одном промежутке времени. Это действие позволяет точнее управлять перемещением между большими срезами времени. В нем отображаются тенденции вниз, чтобы фрагменты, размером миллисекунды. Его можно использовать для просмотра разделенные, с высоким разрешением порезов данных. Начальная точка ползунка по умолчанию устанавливается как наиболее оптимальный вид данных из вашего выбора, в котором разрешение, скорость запросов и степень детализации сбалансированы.

1. **Дата диапазона в и из веб-управления**: С этого веб-элемента управления можно легко выбрать необходимые диапазоны даты и времени. Также с его помощью можно переключаться между разными часовыми поясами. После внесения изменений для применения в текущую рабочую область, выберите **Сохранить**.

   [![В и из панели выбора](media/v2-update-explorer/explorer-thirteen.png)](media/v2-update-explorer/explorer-thirteen.png#lightbox)

## <a name="s1s2-environment-controls"></a>Элементы управления средой S1/S2

### <a name="preview-terms-panel"></a>Панель предварительного просмотра условий поиска

Этот раздел относится только к существующим средам S1/S2, которые пытаются использовать обозреватель в обновленном пользовательском интерфейсе. Можно использовать в сочетании продукта общедоступная и предварительная версия. Мы добавили некоторые функциональные возможности из существующего пользовательского интерфейса в обновленный обозреватель, но вы можете получить доступ ко всем возможностям пользовательского интерфейса среды S1/S2 среды в существующем обозревателе службы "Аналитика временных рядов". 

Вместо иерархии вы увидите панель условий поиска Time Series Insights, где определение запросов в вашей среде. Используйте для фильтрации данных на основе предиката.

  [![Поступают запросы панели](media/v2-update-explorer/explorer-eighteen.png)](media/v2-update-explorer/explorer-eighteen.png#lightbox)

Панель редактора условий службы "Аналитика временных рядов" (предварительная версия) принимает следующие параметры:

**Where**. Используйте аргументы where предложение, чтобы быстро фильтровать события с помощью набора операнд, перечисленных в следующей таблице. При выполнении поиска путем выбора операнда предикат автоматически обновляется на основе поиска. Поддерживаемые типы операнд:

| Операция | Поддерживаемые типы   | Примечания |
| --- | --- | --- |
| `<`, `>`, `<=`, `>=` | Double, DateTime, TimeSpan | |
| `=`, `!=`, `<>` | String, Bool, Double, DateTime, TimeSpan, NULL |
| `IN` | String, Bool, Double, DateTime, TimeSpan, NULL | Все операнды должны быть одного типа или являться константой NULL. |
| `HAS` | String | Справа разрешены только константы строковых литералов. Пустая строка и значение NULL не допускаются. |

Дополнительные сведения о поддерживаемых операциях запросов и типах данных см. в разделе [выражение ряда времени (TSX)](https://docs.microsoft.com/rest/api/time-series-insights/preview-tsx).

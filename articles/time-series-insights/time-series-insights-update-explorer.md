# <a name="azure-time-series-insights-preview-explorer"></a>Azure Время Серии Исследования Обзор исследователь

В этой статье описаны различные функции и опции, доступные в [демо-приложении](https://insights.timeseries.azure.com/preview/demo)Azure Time Series Insights Preview.

## <a name="prerequisites"></a>Предварительные требования

Чтобы начать работу с исследователем Azure Time Series Insights Preview, необходимо:

* Подгоните среду Time Series Insights. Подробнее о подготовке экземпляра можно прочитать учебник [Azure Time Series Insights Preview.](./time-series-insights-update-create-environment.md)
* [Предоставьте доступ к данным](./time-series-insights-data-access.md) к среде Time Series Insights, созданной для учетной записи. Вы можете предоставить доступ как другим пользователям, так и себе.
* Добавьте источник событий в среду Time Series Insights, чтобы довести данные до среды:
  * [Узнайте, как подключиться к концентратору событий](./time-series-insights-how-to-add-an-event-source-eventhub.md) 
  * [Узнайте, как подключиться к концентратору IoT](./time-series-insights-how-to-add-an-event-source-iothub.md)

### <a name="model-authoring"></a>Автор модели

Обзор Серии времени Azure Insights поддерживает полное создание, чтение, обновление и удаление (CRUD) операций на модели временных рядов.

[![Панель поиска модели](media/v2-update-explorer/model-search-panel.png)](media/v2-update-explorer/model-search-panel.png#lightbox)

* **Тип модели временных рядов:** Вы можете использовать типы Time Series Insights для определения переменных или формул для выполнения вычислений. Они связаны с данным экземпляром Time Series Insights. Один тип может содержать одну или несколько переменных.
* **Иерархия модели временных рядов:** Иерархии являются систематическими организациями ваших данных. Иерархии изображают связи между разными сущностями данных аналитики временных рядов.
* **Пример модели временных рядов**: Инстанции сами серии времени. В большинстве случаев это **DeviceID** или **AssetID,** который является уникальным идентификатором актива в среде.

Чтобы узнать больше о модели серии времени, прочитайте [модели серии Времен](./time-series-insights-update-tsm.md).

## <a name="5-chart-panel"></a>5. Панель диаграмм

Диаграмма позволяет отображать экземпляры временных рядов в виде строк. Чтобы увеличить диаграмму, можно свернуть панель среды, модель данных и панель управления интервалом времени, щелкнув соответствующие веб-элементы управления. 

  [![Предварительный обзор диаграммы](media/v2-update-explorer/tsi-preview-chart-overview.png)](media/v2-update-explorer/tsi-preview-chart-overview.png#lightbox)

1. **Тип диаграммы**: Элементы управления, какие элементы данных доступны для визуализации.

1. **Размер интервала:** инструмент ползунка размера Интервала позволяет увеличить и выйти из интервалов в течение того же периода времени. Это обеспечивает более точный контроль движения между большими ломтиками времени, которые показывают плавные тенденции вплоть до ломтиков, как маленькие, как миллисекунды, что позволяет просматривать гранулированные, с высоким разрешением сокращений ваших данных. Начальная точка ползунка по умолчанию устанавливается как наиболее оптимальный вид данных из вашего выбора (разрешение, скорость запросов и степень детализации сбалансированы).

1. **Увеличьте масштаб ирование и панорамирование:** Выберите этот элемент управления, чтобы увеличить и панорамировать диаграмму.

1. **Управление Y-оси**: Циклы через доступные параметры представления y-оси:

    * `Stacked`: Каждая линия имеет индивидуальную оси Y.
    * `Overlap`: Используйте его для укладки нескольких строк на той же оси Y, с Y-оси данных изменения на основе строки выбранной.
    * `Shared`: Все данные Y-оси отображаются вместе.

1. **Элемент маркера**: Выбранный в настоящее время элемент данных и связанные с ним детали.

Вы можете продолжить бурение в определенный срез данных, **нажав** на точку данных на текущем графике, удерживая мышь, а затем перетащив выбранную область к конечной точке по вашему выбору. **Нажмите правой кнопкой** синий, выбранной области, а затем выберите **Увеличить,** как показано ниже. Вы также можете просматривать и загружать события телеметрии в выбранном временном поясе.

  [![Масштаб диаграммы предварительного просмотра](media/v2-update-explorer/preview-chart-zoom.png)](media/v2-update-explorer/preview-chart-zoom.png#lightbox)

После выполнения действия **«Увеличить»** выбранный набор данных будет отображаться. Выберите элемент управления форматом для цикла через три представления y-оси данных Time Series Insights.

  [![Предварительный график y-оси](media/v2-update-explorer/tsi-preview-explorer-standard-chart.png)](media/v2-update-explorer/tsi-preview-explorer-standard-chart.png#lightbox)

Здесь приводится пример **перекрывающейся диаграммы:**

Кнопка **"Больше действий"** расширяется для отображения **загрузки в качестве CSV,** **Подключите к Power BI,** **отображайте данные диаграммы в виде таблицы**и **исследуйте параметры необработанных событий.**

## <a name="6-time-editor-panel"></a>6. Панель редактора времени

При работе с Time Series Insights вы сначала выберете промежуток времени. Выбранный промежуток времени будет управлять набором данных, который доступен для манипуляций с виджетами Time Series Insights.

> [!TIP]
> Часть временной шкалы выделена в янтарном или оранжевом цвете, чтобы указать диапазон данных, которые доступны в теплом магазине.

Следующие веб-элементы управления доступны в обновлении Time Series Insights для выбора рабочего времени. 

  [![Управление разведочной скважиной](media/v2-update-explorer/exploration-well-control.png)](media/v2-update-explorer/exploration-well-control.png#lightbox)

1. **Внутренний диапазон даты ползунка управления:** Используйте два элемента управления конечными точками, перетащив их на промежуток времени вы хотите. Этот внутренний диапазон дат ограничен внешним управлением ползунок диапазона дат.

1. **Кнопки увеличения и уменьшения диапазона дат**: Увеличьте или уменьшите продолжительность времени, выбрав одну из нужных кнопок.

1. **Управление ползунок диапазона времени**: Используйте его для быстрого переключения между заданными выборами промежутка времени, такими как последние **30 минут,** **последние 12 часов**или **пользовательский диапазон.** С изменением этого значения изменяются также доступные диапазоны интервалов, описанные в параграфе о ползунке размера интервала.

## <a name="s1s2-environment-controls"></a>Элементы управления средой S1/S2

### <a name="preview-terms-panel"></a>Панель условий предварительного просмотра

Этот раздел относится только к существующим средам S1/S2, которые пытаются использовать обозреватель в обновленном пользовательском интерфейсе. Вы можете использовать общедоступный продукт и Предварительный просмотр в сочетании. Мы добавили некоторые функциональные возможности из существующего пользовательского интерфейса в обновленный обозреватель, но вы можете получить доступ ко всем возможностям пользовательского интерфейса среды S1/S2 среды в существующем обозревателе службы "Аналитика временных рядов". 

Вместо иерархии отображается панель терминов Time Series Insights. Панель терминов позволяет определить запросы в вашей среде. Используйте его для фильтрации данных на основе предиката.

  [![Где панель запросов](media/v2-update-explorer/s1-s2-preview-query.png)](media/v2-update-explorer/s1-s2-preview-query.png#lightbox)

Панель редактора условий службы "Аналитика временных рядов" (предварительная версия) принимает следующие параметры:

**Где**: Используйте положение, где быстро фильтровать события, используя набор operands, перечисленных в следующей таблице. При выполнении поиска путем выбора операнда предикат автоматически обновляется на основе поиска. Поддерживаемые типы операнд включают в себя следующие:

| Операция | Поддерживаемые типы   | Примечания |
| --- | --- | --- |
| `<`, `>`, `<=`, `>=` | Double, DateTime, TimeSpan | |
| `=`, `!=`, `<>` | String, Bool, Double, DateTime, TimeSpan, NULL |
| `IN` | String, Bool, Double, DateTime, TimeSpan, NULL | Все операнды должны быть одного типа или являться константой NULL. |
| `HAS` | Строка | Только постоянные строки буквальные допускается на правой стороне. Пустая строка и NULL не допускаются. |

Чтобы узнать больше об поддерживаемых операциях запросов и типах данных, прочитайте [выражение временных рядов (TSX).](https://docs.microsoft.com/rest/api/time-series-insights/preview#time-series-expression-and-syntax)

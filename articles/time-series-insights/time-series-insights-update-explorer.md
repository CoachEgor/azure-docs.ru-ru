# <a name="visualize-data-in-the-explorer-preview"></a>Визуализация данных в обозревателе (предварительная версия)

Этот документ описывает возможности пользовательского интерфейса и интерфейса, предварительная версия Azure время Series Insights [Демонстрация веб-приложения](https://insights.timeseries.azure.com/preview/demo). В частности здесь рассматриваются макета образец, параметры настройки интерфейса и навигации, предоставленный демонстраций.

## <a name="prerequisites"></a>Технические условия

Чтобы начать работу с помощью обозревателя Azure время Series Insights Preview, необходимо сделать следующее:

* Настроить среду службы "Аналитика временных рядов". Дополнительные сведения о подготовке экземпляра попробуйте наши [предварительного просмотра аналитики Azure во время ряда](./time-series-insights-update-create-environment.md) руководства.
* [Предоставлять доступ к данным](./time-series-insights-data-access.md) в среду Time Series Insights, созданный для учетной записи. Вы можете предоставить доступ как другим пользователям, так и себе.
* Добавьте источник события для среды Time Series Insights для отправки данных в среде:
  * Узнайте, [способ подключения к концентратору событий](./time-series-insights-how-to-add-an-event-source-eventhub.md)
  * Узнайте, [способ подключения к центру Интернета вещей](./time-series-insights-how-to-add-an-event-source-iothub.md)

## <a name="navigation-menu"></a>Меню навигации

  [![В меню навигации](media/v2-update-explorer/explorer-three.png)](media/v2-update-explorer/explorer-three.png#lightbox)

В меню навигации можно выбрать между двумя представлениями:

* **Анализ**. Позволяет строить диаграммы и выполнять расширенную аналитику по смоделированным или несмоделированным данным временных рядов.
* **Модель**. Позволяет отправлять новые типы, иерархии и экземпляры из службы "Аналитика временных рядов" (предварительная версия) в вашу модель аналитики временных рядов.

## <a name="hierarchy-tree"></a>дерево иерархии

В дереве иерархии отображаются выбранные данные элементов, включая модели, определенных устройств и датчиков на устройствах.

### <a name="model-search-panel"></a>Панель поиска модели

Панель поиска модели позволяет легко осуществлять поиск и навигацию в иерархии модели временных рядов, находя определенные экземпляры временных рядов для отображения в диаграмме. При выборе экземпляров они добавляются как в текущую диаграмму, так и в источник данных.

  [![На панели поиска модели](media/v2-update-explorer/explorer-four.png)](media/v2-update-explorer/explorer-four.png#lightbox)

### <a name="model-authoring"></a>Создание модели

Предварительная версия Insights серии время Azure поддерживает полный операций создания, чтения, обновления и удаления (CRUD) в вашей модели временных рядов.  

* **Тип модели временных рядов**. Типы аналитики временных рядов позволяют определять переменные или формулы для выполнения вычислений. Они связываются с заданным экземпляром службы "Аналитика временных рядов". Один тип может содержать одну или несколько переменных.
* **Иерархия модели временных рядов**. Иерархии представляют собой систематическую организацию данных. Иерархии изображают связи между разными сущностями данных аналитики временных рядов.
* **Экземпляр модели временных рядов**. Экземпляры — это собственно временные ряды. В большинстве случаев они являются **DeviceID** или **AssetID**, который является уникальным идентификатором ресурса в среде.

Дополнительные сведения о модели временных рядов см. в [этой статье](./time-series-insights-update-tsm.md).

## <a name="preview-chart"></a>Предварительный просмотр диаграммы

С диаграммой можно отображать экземпляры TSI как строки. Чтобы увеличить диаграмму, можно свернуть панель среды, модель данных и панель управления интервалом времени, щелкнув соответствующие веб-элементы управления.

1. **Ползунок внутреннего диапазона дат**. Используйте два элемента управления конечными точками, перетаскивая их по нужному интервалу времени.

1. **Элемент управления для сворачивания интервала времени**. Сворачивает и разворачивает редактор панели интервала времени.

1. **Текущий элемент данных**. Текущий выбранный элемент данных и связанные с ним сведения.

## <a name="time-editor-panel"></a>Панель редактора времени

При работе со службой "Аналитика временных рядов" (предварительная версия) сначала следует выбрать интервал времени. Выбранный интервал времени определяет набор данных, доступный для операций с помощью мини-приложений службы "Аналитика временных рядов" (предварительная версия). В службе "Аналитика временных рядов" (предварительная версия) для выбора рабочего интервала времени доступны следующие веб-элементы управления.

  [![Панель выбора времени](media/v2-update-explorer/explorer-twelve.png)](media/v2-update-explorer/explorer-twelve.png#lightbox)

1. **Ползунок внутреннего диапазона дат**. Используйте два элемента управления конечными точками, перетаскивая их по нужному интервалу времени. Этот внутренний диапазон дат ограничивается ползунком внешнего диапазона дат.

1. **Кнопки увеличения и уменьшения диапазона дат**. Интервал времени можно увеличить или уменьшить, выбрав одну из кнопок для требуемого интервала.

1. **Элемент управления для сворачивания интервала времени**. Этот веб-элемент управления позволяет скрыть все элементы управления, за исключением ползунка внутреннего диапазона дат.

1. **Ползунок внешнего диапазона дат**. С помощью элементов управления конечными точками выберите внешний диапазон дат, который будет доступен для элемента управления внутренним диапазоном дат.

1. **Выберите диапазон дат краткое время в раскрывающемся**: Позволяет быстро переключаться между span выбранные значения времени, таких как последний **30 минут**, **последние 12 часов**, или **пользовательский диапазон**. С изменением этого значения изменяются также доступные диапазоны интервалов, описанные в параграфе о ползунке размера интервала.

1. **Ползунок размера интервала**. Позволяет увеличивать и уменьшать размер интервалов в одном и том же промежутке времени. Это действие позволяет точнее управлять перемещением между большими срезами времени. Оно отображает плавные тенденции вплоть до таких мелких срезов, как миллисекунда, позволяя просматривать детализированные срезы данных с высоким разрешением. Начальная точка ползунка по умолчанию устанавливается как наиболее оптимальный вид данных из вашего выбора, в котором разрешение, скорость запросов и степень детализации сбалансированы.

1. **Веб-элемент управления началом и окончанием диапазона дат**. С этого веб-элемента управления можно легко щелкните и выберите нужные дату и временных интервалов. Также с его помощью можно переключаться между разными часовыми поясами. Чтобы применить внесенные изменения к текущей рабочей области, выберите **Сохранить**.

   [![В и из панели выбора](media/v2-update-explorer/explorer-thirteen.png)](media/v2-update-explorer/explorer-thirteen.png#lightbox)

## <a name="s1s2-environment-controls"></a>Элементы управления средой S1/S2

### <a name="preview-terms-panel"></a>Панель предварительного просмотра условий поиска

Этот раздел относится только к существующим средам S1/S2, которые пытаются использовать обозреватель в обновленном пользовательском интерфейсе. Возможно, вам потребуется использовать сочетание общедоступной и предварительной версий продукта. Мы добавили некоторые функциональные возможности из существующего пользовательского интерфейса в обновленный обозреватель, но вы можете получить доступ ко всем возможностям пользовательского интерфейса среды S1/S2 среды в существующем обозревателе службы "Аналитика временных рядов".  

Вместо иерархии вы увидите панель условий службы "Аналитика временных рядов", которая служит для определения запросов в вашей среде. Она позволяет фильтровать данные на основе предиката.

  [![Поступают запросы панели](media/v2-update-explorer/explorer-eighteen.png)](media/v2-update-explorer/explorer-eighteen.png#lightbox)

Панель редактора условий службы "Аналитика временных рядов" (предварительная версия) принимает следующие параметры:

**Where**. Предложение where позволяет быстро фильтровать события, используя набор операндов, перечисленных в следующей таблице. При выполнении поиска путем выбора операнда предикат автоматически обновляется на основе поиска. Поддерживаемые типы операнд:

| Операция | Поддерживаемые типы   | Примечания |
| --- | --- | --- |
| `<`, `>`, `<=`, `>=` | Double, DateTime, TimeSpan | |
| `=`, `!=`, `<>` | String, Bool, Double, DateTime, TimeSpan, NULL |
| `IN` | String, Bool, Double, DateTime, TimeSpan, NULL | Все операнды должны быть одного типа или являться константой NULL. |
| `HAS` | Строка | Справа разрешены только константы строковых литералов. Пустая строка и значение NULL недопустимы. |

Дополнительные сведения о поддерживаемых операциях запросов и типах данных см. в статье [Time Series Expression (TSX)](https://docs.microsoft.com/rest/api/time-series-insights/preview-tsx) (Язык выражений временных рядов (TSX)).

### <a name="examples-of-where-clauses"></a>Примеры предложений where

  [![Где примеры предложения](media/v2-update-explorer/explorer-nineteen.png)](media/v2-update-explorer/explorer-nineteen.png#lightbox)

**Мера**. Раскрывающийся список, отображающий все числовые столбцы (**дублеров**) можно использовать в качестве элементов для текущей диаграммы.

**Разделять по**. В этом раскрывающемся списке отображаются все доступные категориальные столбцы (типа String) в модели, по которым можно группировать данные. Можно добавить до пяти просматриваемых условий на одной оси X. Введите требуемые параметры, а затем выберите **Добавить**, чтобы добавить новое условие.

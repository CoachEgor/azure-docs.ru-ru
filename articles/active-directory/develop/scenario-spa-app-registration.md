# <a name="single-page-application---app-registration"></a>Одностраничное приложение — Регистрация приложения

Этой странице рассматриваются особенности регистрации приложения для одностраничных приложений (SPA).

Выполните действия, чтобы [зарегистрировать новое приложение с платформой Microsoft identity](quickstart-register-app.md)и выберите поддерживаемые учетные записи для вашего приложения. В сценарии SPA может поддерживать проверку подлинности с учетными записями в вашей организации или любой организации и личных учетных записей Майкрософт.

Узнайте, специфических аспектах регистрации приложения, которые применяются к одностраничных приложениях.

## <a name="register-a-redirect-uri"></a>Регистрация URI перенаправления

Неявный поток отправляет токены при перенаправлении одностраничного приложения, работающего в веб-браузере. Таким образом он является важным требованием для регистрации URI, где приложение может получать маркеры перенаправления. Убедитесь, что перенаправления, который соответствует URI точно совпадает с URI для вашего приложения.

В [портала Azure](https://go.microsoft.com/fwlink/?linkid=2083908), перейти к зарегистрированного приложения на **проверки подлинности** страницы приложения, установите флажок **Web** платформы и введите значение URI перенаправления для приложения в **URI перенаправления** поля.

## <a name="enable-the-implicit-flow"></a>Подключите этот неявный поток

На том же **проверки подлинности** раздела **Дополнительные параметры**, необходимо также включить **неявное предоставление**. Если приложение только выполняет вход пользователей и получение маркеров Идентификации, этого достаточно включить **маркеры Идентификации** флажок.

Если приложению требуется также для получения маркеров доступа для вызова API, не забудьте включить **маркеры доступа** также флажок. Дополнительные сведения см. в разделе [маркеры Идентификации](./id-tokens.md) и [маркеры доступа](./access-tokens.md).

## <a name="api-permissions"></a>Разрешения API

Интерфейсы API можно вызвать в одностраничных приложениях от имени выполнившего вход пользователя. Они должны запрашивать делегированные разрешения. Дополнительные сведения см. в разделе [добавить разрешения для доступа к веб-API](quickstart-configure-app-access-web-apis.md#add-permissions-to-access-web-apis)

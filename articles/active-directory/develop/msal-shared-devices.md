# <a name="overview-of-shared-device-mode"></a>Обзор режима общего устройства

Режим совместного устройства — это функция Active Directory Azure, которая позволяет создавать приложения, поддерживающие firstline Workers и включающие общий режим устройства на развернутых на них устройствах.

> [!NOTE]
> Эта функция предоставляется в общедоступной предварительной версии.
> Эта предварительная версия предоставляется без соглашения об уровне обслуживания и не рекомендована для использования рабочей среде. Некоторые функции могут не поддерживаться или их возможности могут быть ограничены.
> Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

## <a name="what-are-firstline-workers"></a>Что такое работники первой линии?

Firstline Workers — это розничные работники, агенты по техническому обслуживанию и полевым работам, медицинский персонал и другие пользователи, которые не сидят перед компьютером и не используют корпоративную электронную почту для совместной работы. В следующих разделах представлены аспекты и проблемы поддержки сотрудников Firstline, а затем представлены функции, предоставляемые корпорацией Майкрософт, которые позволяют использовать ваше приложение сотрудникам первой линии организации.

### <a name="challenges-of-supporting-firstline-workers"></a>Проблемы поддержки работников первой линии

Включение рабочих процессов Firstline Включает проблемы, обычно не представляемые типичными информационными работниками. Такие проблемы могут включать высокую текучесть кадров и меньшее знакомство с основными инструментами производительности организации. Для расширения прав и возможностей своих работников Первой линии, организации принимают различные стратегии. Некоторые из них принимают принести свой собственный-устройство (BYOD) стратегии, в которой их сотрудники используют бизнес-приложений на своем личном телефоне, в то время как другие предоставляют своим сотрудникам общие устройства, такие как iPads или Android таблетки.

### <a name="supporting-multiple-users-on-devices-designed-for-one-user"></a>Поддержка нескольких пользователей на устройствах, предназначенных для одного пользователя

Поскольку мобильные устройства под управлением iOS или Android были разработаны для одиноких пользователей, большинство приложений оптимизируют их опыт использования одним пользователем. Часть этого оптимизированного опыта означает возможность одного включения в приложения и сохранение пользователей в ввод на свое устройство. Когда пользователь удаляет свою учетную запись из приложения, приложение обычно не считает его событием, связанным с безопасностью. Многие приложения даже сохранить учетные данные пользователя вокруг для быстрого вхрома. Возможно, вы даже испытали это сами, когда удалили приложение с вашего мобильного устройства, а затем переустановили его, только чтобы обнаружить, что вы все еще подписаны.

### <a name="global-sign-in-and-sign-out-sso"></a>Глобальный входить в систему и входить в систему (SSO)

Чтобы позволить сотрудникам организации использовать свои приложения в пуле устройств, которыми пользуются эти сотрудники, разработчикам необходимо обеспечить обратный опыт. Сотрудники должны иметь возможность выбрать устройство из пула и выполнить один жест, чтобы "сделать его своим" в течение всего срока их смены. В конце смены, они должны быть в состоянии выполнить еще один жест, чтобы выйти глобально на устройстве, со всей их личной и компании информации удалены, чтобы они могли вернуть его в пул устройства. Кроме того, если сотрудник забывает выйти из системы, устройство должно быть автоматически подписано в конце смены и/или после периода бездействия.

Активный каталог Azure позволяет использовать эти сценарии с функцией, называемой **режимом общего устройства.**

## <a name="introducing-shared-device-mode"></a>Введение режима общего устройства

Как уже упоминалось, режим совместного устройства является особенностью Active Directory Azure, которая позволяет:

* Создание приложений, поддерживающих первых работников
* Развертывание устройств для сотрудников Firstline и включаните общий режим устройства

### <a name="build-applications-that-support-firstline-workers"></a>Создание приложений, поддерживающих первых работников

Вы можете поддерживать firstline Workers в приложениях с помощью библиотеки подлинности Майкрософт (MSAL) и [приложения Microsoft Authenticator](../user-help/user-help-auth-app-overview.md) для включения состояния устройства, называемого *режимом общего устройства.* Когда устройство находится в режиме общего устройства, корпорация Майкрософт предоставляет вашему приложению информацию, позволяющую ему изменять свое поведение в зависимости от состояния пользователя на устройстве, защищая данные пользователей.

Поддерживаемые функции:

* **Войти в пользовательское устройство через** любое поддерживаемое приложение.
* **Вырегистрируйте пользовательское устройство через** любое поддерживаемое приложение.
* **Запрос состояния устройства,** чтобы определить, находится ли ваше приложение на устройстве, которое находится в режиме общего устройства.
* **Запрос состояния устройства пользователя** на устройстве, чтобы определить, если что-то изменилось с момента последнего использования приложения.

Поддержка режима общего устройства должна рассматриваться как повышение безопасности и обновление функций для вашего приложения, и может помочь увеличить его принятие в строго регулируемых средах, таких как здравоохранение и финансы.

Ваши пользователи зависят от вас, чтобы убедиться, что их данные не просочились другому пользователю. Share Device Mode предоставляет полезным сигналам, указывающим на ваше приложение, что произошло изменение, которым вы должны управлять. Ваше приложение отвечает за проверку состояния пользователя на устройстве при каждом использовании приложения, очищая данные предыдущего пользователя. Это включает в себя, если он перезагружается из фона в многозадачности. При изменении пользователя следует обеспечить удаление данных предыдущего пользователя и удаление любых кэшированных данных, отображаемых в приложении. Мы рекомендуем вам всегда выполнять тщательный процесс проверки безопасности после добавления общих возможностей режима устройства в ваше приложение.

Подробнее о том, как изменить приложения для поддержки режима общего устройства, можно узнать в разделе [«Следующие шаги»](#next-steps) в конце этой статьи.

### <a name="deploy-devices-to-firstline-workers-and-turn-on-shared-device-mode"></a>Развертывание устройств для сотрудников Firstline и включаните общий режим устройства

После того, как приложения поддержат общий режим устройства и включат необходимые изменения в данные и безопасность, вы сможете рекламировать их как пригодные для использования работниками Firstline.

Администраторы устройств организации могут развертывать свои устройства и приложения в своих магазинах и на рабочих местах с помощью решения для управления мобильными устройствами (MDM), например Microsoft Intune. Частью процесса подготовки является маркировка устройства как *совместного устройства.* Администраторы настраивают общий режим устройства путем развертывания [приложения Microsoft Authenticator](../user-help/user-help-auth-app-overview.md) и настройки общего режима устройства через параметры конфигурации. После выполнения этих задач все приложения, поддерживающие режим общего пользования, будут использовать приложение Microsoft Authenticator для управления состоянием пользователя и предоставления функций безопасности для устройства и организации.

# <a name="validation-differences-by-supported-account-types-signinaudience"></a>Различия в проверке по поддерживаемым типам учетных записей (signInAudience)

При регистрации приложения на платформе идентификации Майкрософт для разработчиков предлагается выбрать типы учетных записей, которые поддерживает ваше приложение. В объекте приложения и манифесте это свойство . `signInAudience`

Возможны следующие варианты:

- *AzureADMyOrg*: Только учетные записи в каталоге организации, где приложение зарегистрировано (один-арендатор)
- *AzureADMultipleOrgs*: Учетные записи в любом каталоге организации (мультиарендатор)
- *AzureADandPersonalMicrosoftAccount*: Учетные записи в любом каталоге организаций (многофункциональные) и личные учетные записи Майкрософт (например, Skype, Xbox и Outlook.com)

Для зарегистрированных приложений можно найти значение для типов поддерживаемых учетных записей в разделе **Аутентификация** приложения. Вы также можете найти `signInAudience` его под собственностью в **манифесте**.

Значение, выбранное для этого свойства, имеет последствия для других свойств объектов приложения. В результате, если вы измените это свойство, вам может потребоваться сначала изменить другие свойства.

# <a name="azure-portal-registration-fields-for-custom-developed-apps"></a>Поля регистрации порталов Azure для пользовательских приложений

В этой статье дается краткое описание доступных полей в форме регистрации приложения на [портале Azure](https://portal.azure.com).

## <a name="register-a-new-application"></a>Регистрация нового приложения

-   Чтобы зарегистрировать новое приложение, перейдите на [портал Azure](https://portal.azure.com).

-   С левой навигационной панели щелкните **Active Directory Azure.**

-   Выберите **Регистрация приложений** и щелкните **Добавить**.

-   Откроется форма регистрации приложения.

## <a name="fields-in-the-application-registration-form"></a>Поля в форме регистрации приложения

| Поле            | Описание                                                                              |
|------------------|------------------------------------------------------------------------------------------|
| Имя             | Имя приложения. Оно должно содержать не менее четырех знаков.                |
| Поддерживаемые типы учетных записей| Выберите учетные записи, которые вы хотели бы, чтобы ваше приложение поддерживало: учетные записи только в этом каталоге организации, учетные записи в любом каталоге организации или учетные записи в любом каталоге организации и личных учетных записей Майкрософт.  |
| Перенаправить URI (необязательно) | Выберите тип приложения, которое вы строите, **веб-клиент** **(мобильный & рабочий стол),** а затем введите перенаправление URI (или URL ответа) для вашего приложения. Для веб-приложений укажите основной URL-адрес приложения. Например, http://localhost:31544 может быть URL-адресом веб-приложения, выполняемого на локальном компьютере. Пользователи будут использовать этот URL-адрес для входа в приложение веб-клиента. Для общедоступных клиентских приложений укажите универсальный код ресурса (URI), который использует AAD для возвращения ответов маркера. Введите значение, специфичное для приложения, например myapp://auth. Чтобы увидеть конкретные примеры для веб-приложений или нативных приложений, ознакомьтесь с нашими [quickstarts](https://docs.microsoft.com/azure/active-directory/develop).|

После заполнения вышеуказанных полей приложение регистрируется на портале Azure и перенаправляется на страницу обзора приложения. Страницы настроек в левом стеле под **Управлением** имеют больше полей для настройки приложения. В приведенных ниже таблицах описаны все поля. Вы увидите только подмножество этих полей, в зависимости от того, создали ли вы веб-приложение или приложение для публичных клиентов.

### <a name="overview"></a>Обзор

| Поле           | Описание        |
|-----------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Идентификатор приложения  | При регистрации приложения Azure AD назначает ему идентификатор приложения. Его можно использовать для уникальной идентификации приложения в запросах проверки подлинности в Azure AD, а также для доступа к ресурсам, таким как API Graph.                                                          |
| URI идентификатора приложения      | Это должно быть уникальное URI, обычно формы **https://&lt;имя\_&gt;/&lt;\_&gt;клиента.** Это используется во время потока гранта авторизации, как уникальный идентификатор для указания ресурса, для которого должен быть выдан токен. Он также становится утверждением aud в выданном токене. |

### <a name="branding"></a>Фирменная символика

| Поле           | Описание        |
|-----------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Отправка нового логотипа | С помощью этого элемента можно передать логотип для вашего приложения. Логотип должен быть в формате .bmp, .jpg или .png, а размер файла должен быть меньше 100 кБ. Изображение должно быть размером 215 x 215 пикселей с размером центрального изображения 94 x 94 пикселя.|
| URL-адрес домашней страницы   | Это URL-адрес входа, указанный при регистрации приложения.|

### <a name="authentication"></a>Аутентификация

| Поле           | Описание        |
|-----------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| URL-адрес выхода.      | Это единственный URL-адрес выхода из системы вывески. Azure AD отправляет запрос на выход по этому URL-адресу, когда пользователь очищает сеанс с Azure AD с помощью любого другого зарегистрированного приложения.|
| Поддерживаемые типы учетных записей  | Этот переключатель позволяет указать, может ли приложение использоваться несколькими клиентами. Обычно это означает, что внешние организации могут использовать ваше приложение, зарегистрировав его в своем клиенте и предоставив доступ к своим корпоративным данным.|
| URL-адреса перенаправления      | Url-адреса перенаправления или ответа — это конечные точки, где Azure AD возвращает токены, которые запрашивает приложение. Для собственных приложений пользователь отправляется сюда после успешной авторизации. Azure AD проверяет, что перенаправление URI, поставляются приложением в запросе OAuth 2.0, соответствует одному из зарегистрированных значений на портале.|

### <a name="certificates-and-secrets"></a>Сертификаты и секреты

| Поле           | Описание        |
|-----------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Секреты клиента            | Вы можете создавать секреты клиентов или ключи для программного доступа к web-aIS, защищенным Azure AD, без какого-либо взаимодействия с пользователем. С **секретной** страницы Нового клиента введите описание ключа и срок действия и сэкономьте для создания ключа. Сохраните ключ в надежном месте, так как вы не сможете получить к нему доступ позже.             |

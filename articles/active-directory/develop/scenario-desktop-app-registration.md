# <a name="desktop-app-that-calls-web-apis-app-registration"></a>Настольное приложение, которое вызывает веб-AIS: Регистрация приложений

Эта статья охватывает специфику регистрации приложения для настольного приложения.

## <a name="supported-account-types"></a>Поддерживаемые типы учетных записей

Типы учетных записей, поддерживаемые в настольном приложении, зависят от того, какой опыт вы хотите зажечь. Из-за этой взаимосвязи поддерживаемые типы учетных записей зависят от потоков, которые вы хотите использовать.

### <a name="audience-for-interactive-token-acquisition"></a>Аудитория для интерактивного приобретения токенов

Если ваше настольное приложение использует интерактивную аутентификацию, вы можете войти в число пользователей из любого [типа учетной записи.](quickstart-register-app.md#register-a-new-application-using-the-azure-portal)

### <a name="audience-for-desktop-app-silent-flows"></a>Аудитория для тихих потоков настольного приложения

- Для использования интегрированной аутентификации Windows или имени пользователя и пароля, ваше приложение должно войти в пользователей в вашем собственном арендаторе, например, если вы являетесь разработчиком линии бизнеса (LOB). Или, в организациях Active Directory Azure, приложение должно войти в число пользователей в собственном арендаторе, если это сценарий ISV. Эти потоки аутентификации не поддерживаются для личных учетных записей Майкрософт.
- Если вы хотите использовать поток кода устройства, вы не можете войти в пользователей с их личными учетными записями Майкрософт.
- Если вы регистрируете пользователей с социальными удостоверениями, которые передают полномочия и политику между бизнесом и интернетом (B2C), вы можете использовать только интерактивную проверку подлинности и идентификации имени пользователя.

## <a name="redirect-uris"></a>URI перенаправления

Перенаправление URI для использования в настольном приложении зависит от потока, который вы хотите использовать.

- Если вы используете интерактивную аутентификацию или поток кода устройства, используйте `https://login.microsoftonline.com/common/oauth2/nativeclient`. Для достижения этой конфигурации выберите соответствующий URL-адрес в разделе **«Аутентификация»** для приложения.
  
  > [!IMPORTANT]
  > Сегодня MSAL.NET использует другой перенаправить URI по умолчанию`urn:ietf:wg:oauth:2.0:oob`в настольных приложениях, которые работают на Windows ( ). В будущем мы хотим изменить этот дефолт, поэтому мы `https://login.microsoftonline.com/common/oauth2/nativeclient`рекомендуем вам использовать .

- Если вы создаете нативное приложение Objective-C или Swift для macOS, зарегистрируйте перенаправление URI на основе идентификатора пакета приложения в следующем формате: msauth.<your.app.bundle.id>://auth. Заменить <your.app.bundle.id> идентификатором пакета приложения.
- Если ваше приложение использует только интегрированную аутентификацию Windows или имя пользователя и пароль, вам не нужно регистрировать перенаправление URI для вашего приложения. Эти потоки делают поездку туда и обратно на платформу microsoft v2.0 endpoint. Ваше заявление не будет отозвано на каких-либо конкретных URI.
- Чтобы различать поток кода устройства, интегрированную аутентификацию Windows, а также имя пользователя и пароль от конфиденциального потока клиентского приложения, который также не имеет перенаправления URIs (поток учетных данных клиента, используемый в приложениях daemon), необходимо выразить это Ваша заявка является публичным клиентским приложением. Для достижения этой конфигурации перейдите в раздел **Аутентификация** для вашего приложения. В подразделе **Расширенные настройки** в пункте **типа клиента по умолчанию** выберите приложение **Yes** для лечения в **качестве публичного клиента.**

  ![Разрешить публичному клиенту](media/scenarios/default-client-type.png)

## <a name="api-permissions"></a>Разрешения API

Настольные приложения вызывают AA для пользователя, вписаваемого в нее. Они должны запрашивать делегированные разрешения. Они не могут запрашивать разрешения на применение, которые обрабатываются только в [приложениях daemon.](scenario-daemon-overview.md)

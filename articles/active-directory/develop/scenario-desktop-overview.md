## <a name="overview"></a>Обзор

Вы пишете настольное приложение, и вы хотите, чтобы войти в пользователей в ваше приложение и вызова веб-API, таких как Microsoft Graph, другие API Microsoft, или ваш собственный веб-API. У вас есть несколько возможностей:

- Интерактивный токен можно приобрететь:

  - Если приложение для рабочего стола поддерживает графические элементы управления, например, если это приложение Windows.Form, приложение WPF или нативное приложение macOS.
  - Или, если это приложение .NET Core, и вы согласны, что взаимодействие аутентификации с Azure Active Directory (Azure AD) происходит в системе браузера.

- Для приложений, размещенных с Windows, также возможно, чтобы приложения, работающие на компьютерах, объединенных с доменом Windows или Azure AD, присоединились, чтобы получить токен молча, используя интегрированную аутентификацию Windows.
- Наконец, и хотя это не рекомендуется, вы можете использовать имя пользователя и пароль в общедоступных клиентских приложениях. Это все еще необходимо в некоторых сценариях, как DevOps. Использование его накладывает ограничения на ваше приложение. Например, он не может войти в пользователя, которому необходимо выполнить многофакторную аутентификацию (условный доступ). Кроме того, ваша заявка не будет пользоваться одним ввозам (SSO).

  Это также противоречит принципам современной проверки подлинности и предоставляется только для прежних версий.

  ![Настольное приложение](media/scenarios/desktop-app.svg)

- Если вы пишете портативный инструмент командной строки, вероятно, приложение .NET Core, которое работает на Linux или Mac, и если вы согласитесь с тем, что аутентификация будет делегирована в системный браузер, вы можете использовать интерактивную аутентификацию. .NET Core не предоставляет [веб-браузер,](https://aka.ms/msal-net-uses-web-browser)поэтому аутентификация происходит в системном браузере. В противном случае, лучшим вариантом в этом случае является использование потока кода устройства. Этот поток также используется для приложений без браузера, таких как IoT-приложения.

  ![Приложение Browserless](media/scenarios/device-code-flow-app.svg)

## <a name="specifics"></a>Специфики

Настольные приложения имеют ряд особенностей. Они зависят главным образом от того, использует ли ваше приложение интерактивную аутентификацию или нет.

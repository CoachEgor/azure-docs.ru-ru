# <a name="public-client-and-confidential-client-applications"></a>Открытый клиент и конфиденциального клиента приложения
Библиотеки аутентификации Майкрософт (MSAL) определяет два типа клиентов: открытый клиентов и конфиденциальных клиентов. Типы два клиента различаются по их возможность безопасного доступа к серверу авторизации и сохранять конфиденциальность своих учетных данных клиента. Напротив, библиотеки аутентификации Azure AD (ADAL) использует то, что называется *контекст проверки подлинности* (то есть подключение к Azure AD).

- **Конфиденциального клиента приложения** приложения, которые выполняют на серверах (веб-приложений, приложений веб-API или даже приложения службы или управляющей программы). Считаются сложно, для доступа и по этой причине может сохранить секрет приложения. Конфиденциальные клиенты могут хранить секреты во время настройки. Каждый экземпляр клиента имеет отдельные конфигурации (включая идентификатор клиента и секрет клиента). Эти значения являются сложной для конечных пользователей для извлечения. Веб-приложения является наиболее распространенных конфиденциального клиента. Идентификатор клиента предоставляется с помощью веб-браузере, но передается только в обратного канала и предоставляются непосредственно никогда не секрет.

    Конфиденциального клиентского приложения: <BR>
    ![Веб-приложения](media/msal-client-applications/web-app.png) ![веб-API](media/msal-client-applications/web-api.png) ![управляющей программы или службы](media/msal-client-applications/daemon-service.png)

- **Открытый клиентских приложений** приложения, которые выполняют на устройствах или настольных компьютерах или в веб-браузере. Они не являются доверенными для безопасно хранить секреты приложения, поэтому они только получить доступ к веб-API от имени пользователя. (Они поддерживают только потоки общедоступного клиента). Общедоступные клиенты не может содержать секретные данные во время настройки, чтобы им не пришлось секреты клиента.

    Общедоступного клиентского приложения: <BR>
    ![Классическое приложение](media/msal-client-applications/desktop-app.png) ![Browserless API](media/msal-client-applications/browserless-app.png) ![мобильного приложения](media/msal-client-applications/mobile-app.png)

> [!NOTE]
> В MSAL.js не отделяется общедоступный "и" конфиденциальный клиентских приложений.  MSAL.js представляет клиентских приложений как приложения на основе агента пользователя, общедоступные клиенты, в которых код клиента выполняется в агент пользователя, таких как веб-браузер. Эти клиенты не храните секретные данные, так как контекст браузера открыто доступно.

## <a name="comparing-the-client-types"></a>Сравнение типов клиента
Ниже приведены некоторые сходства и различия между общедоступным клиентом и конфиденциального клиента приложения.

- Оба вида приложения поддержки кэша маркера пользователя и может получить токен автоматически (если токен уже находится в кэше маркеров). Конфиденциального клиентского приложения также имеют кэш маркеров приложения для маркеров, которые предназначены для самого приложения.
- Оба типа приложения управления учетными записями пользователей и можно получить учетную запись из кэша маркеров пользователей, получить учетную запись по его идентификатору или удалить учетную запись.
- Открытый клиент приложения имеют четыре способа для получения маркера (четыре потока аутентификации). Конфиденциального клиента приложения имеют три способа получения маркера (и конечной точки авторизации один из способов вычисления URL-адрес поставщика удостоверений). Дополнительные сведения см. в разделе [приобретения токенов](msal-acquire-cache-tokens.md).

Если вы использовали ADAL, следует отметить, что, в отличие от контекст проверки подлинности ADAL, MSAL идентификатор клиента (также называется *идентификатор приложения* или *идентификатор приложения*) передается один раз во время создания приложения. Он не должен передаваться еще раз, когда приложение получает маркер. Это справедливо как для общедоступного и конфиденциальной клиентских приложений для. Конструкторы конфиденциального клиента приложения, также передаются учетные данные клиента: секрет, они совместно используют с поставщиком удостоверений.

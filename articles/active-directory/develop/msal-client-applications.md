# <a name="public-client-and-confidential-client-applications"></a>Публичные клиентские и конфиденциальные клиентские приложения
Библиотека аутентификации Майкрософт (MSAL) определяет два типа клиентов: публичных и конфиденциальных клиентов. Два типа клиентов отличаются своей способностью надежно обеспечивать подлинность с сервером авторизации и сохранять конфиденциальность своих учетных данных клиента. В отличие от этого, Библиотека аутентификации Azure AD (ADAL) использует так называемый *контекст аутентификации* (который является соединением с Azure AD).

- **Конфиденциальные клиентские приложения** — это приложения, которые работают на серверах (веб-приложения, веб-приложения API или даже приложения для обслуживания/деймона). Они считаются труднодоступными, и по этой причине способны держать приложение в тайне. Конфиденциальные клиенты могут хранить секреты времени конфигурации. Каждый экземпляр клиента имеет определенную конфигурацию (включая идентификатор клиента и секрет клиента). Эти значения трудно извлечь конечным пользователям. Веб-приложение является наиболее распространенным конфиденциальным клиентом. Идентификатор клиента разоблачается через веб-браузер, но секрет передается только в обратном канале и никогда непосредственно не подвергается.

- **Публичные клиентские приложения** — это приложения, работающие на устройствах или настольных компьютерах или в веб-браузере. Им не доверяют безопасно хранить секреты приложений, поэтому они получают доступ к Web AA только от имени пользователя. (Они поддерживают только потоки государственных клиентов.) Публичные клиенты не могут хранить секреты времени конфигурации, поэтому у них нет секретов клиента.

    Приложения для публичных клиентов: <BR>

> [!NOTE]
> В MSAL.js нет разделения общедоступных и конфиденциальных клиентских приложений.  MSAL.js представляет клиентские приложения как приложения на основе агента пользователя, публичные клиенты, в которых клиентский код выполняется в пользовательском агенте, как веб-браузер. Эти клиенты не хранят секреты, потому что контекст браузера открыт.

## <a name="comparing-the-client-types"></a>Сравнение типов клиентов
Вот некоторые сходства и различия между публичными клиентскими и конфиденциальными клиентскими приложениями:

- Оба типа приложения поддерживают кэш маркеров пользователя и могут приобретать маркер молча (когда маркер уже находится в кэше маркеров). Конфиденциальные клиентские приложения также имеют кэш маркеров приложения для токенов, которые для самого приложения.
- Оба типа приложения управляют учетными записями пользователей и могут получить учетную запись из кэша маркеров пользователя, получить учетную запись из идентификатора или удалить учетную запись.
- Приложения для публичных клиентов имеют четыре способа приобретения токена (четыре потока аутентификации). Конфиденциальные клиентские приложения имеют три способа приобретения токена (и один способ вычислить URL поставщика идентификационных данных, авторизующего конечную точку). Для получения дополнительной [информации см.](msal-acquire-cache-tokens.md)

Если вы использовали ADAL, вы можете заметить, что, в отличие от контекста аутентификации ADAL, в MSAL идентификатор клиента (также называемый *идентификатор приложения* или *идентификатор приложения)* передается один раз при построении приложения. Он не должен быть принят снова, когда приложение приобретает токен. Это относится как к публичным, так и к конфиденциальным клиентским приложениям. Конструкторы конфиденциальных клиентских приложений также передаются учетные данные клиентов: секрет, которым они делятся с поставщиком идентификационных данных.

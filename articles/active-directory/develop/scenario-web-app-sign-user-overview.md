## <a name="overview"></a>Обзор

Таким образом, чтобы он входа пользователей с веб-приложения, вы добавляете проверку подлинности. Добавление проверки подлинности позволяет веб-приложения, доступ к сведения профиля, а, например фирменную для пользователей. Веб-приложения проверки подлинности пользователя через веб-браузер. В этом сценарии веб-приложение направляет браузер пользователя для выполнения входа в Azure AD. Azure AD возвращает ответ входа через браузер пользователя, который содержит утверждения о пользователе в маркере безопасности. Вход в систему пользователи использовать [Open ID Connect](./v2-protocols-oidc.md) стандартного протокола, сам упрощены благодаря использованию по промежуточного слоя [библиотеки](scenario-web-app-sign-user-app-configuration.md#libraries-used-to-protect-web-apps).

![Веб-приложение выполняет вход пользователей](./media/scenario-webapp/scenario-webapp-signs-in-users.svg)

Как второй этап можно также включить приложение для вызова веб-API от имени выполнившего вход пользователя. Это следующий этап — это другой сценарий, который находится в [веб-приложение вызывает веб-API](scenario-web-app-call-api-overview.md)

> [!NOTE]
> Добавление входа в веб-приложение является о защите веб-приложения и проверка маркер пользователя, который является то, что **по промежуточного слоя** сделать библиотеки. В этом сценарии не требуется, хотя интерфейсы API, защищенные Майкрософт проверки подлинности библиотеки (MSAL), которые являются о получении токен для вызова. Библиотеки проверки подлинности будут вводиться только в случае обработки результатов при веб-приложение должно вызывать веб-API.

## <a name="specifics"></a>Особенности

- Во время регистрации приложения вам потребуется предоставить один или несколько (Если развертывание приложения в нескольких местах) ответ идентификаторы URI. В некоторых случаях (ASP.NET/ASP.NET ядра) необходимо включить IDToken. Наконец необходимо настроить выхода URI, чтобы приложение реагирует на пользователей горизонтального подписи.
- В коде приложения необходимо предоставить полномочия, к которому вы веб-делегаты вход в приложение. Может потребоваться настройка маркера проверки (в частности, в сценариях независимого поставщика программного обеспечения).
- Веб-приложений поддерживает все типы учетной записи. Дополнительные сведения см. в разделе [поддерживаемые типы учетных записей](v2-supported-account-types.md).

# <a name="prompt-behavior-in-msaljs-interactive-requests"></a>Оперативное поведение в интерактивных запросах MSAL.js

Когда пользователь создал активный сеанс Azure AD с несколькими учетными записями пользователей, знак Azure AD на странице по умолчанию подскажет пользователю выбрать учетную запись, прежде чем приступить к входе. Пользователи не увидят опыт выбора учетной записи, если с Azure AD есть только один сеанс аутентификации.

Библиотека MSAL.js (начиная с v0.2.4) не отправляет оперативный`loginRedirect`параметр во время интерактивных запросов (, `loginPopup` `acquireTokenRedirect` и `acquireTokenPopup`), и, таким образом, не обеспечивает каких-либо оперативных поведения. Для запросов бесшумных токенов с помощью `acquireTokenSilent` метода `none`MSAL.js передает набор оперативного параметра.

Основываясь на сценарии приложения, можно управлять оперативным поведением интерактивных запросов, задавая параметр запроса в параметрах запроса, передаваемых методам. Например, если вы хотите вызвать опыт выбора учетной записи:

```javascript
var request = {
    scopes: ["user.read"],
    prompt: 'select_account',
}

userAgentApplication.loginRedirect(request);
```


Следующие оперативные значения могут быть пройдены при проверке подлинности с помощью Azure AD:

**Войти:** Это значение заставит пользователя ввести учетные данные в запросе на проверку подлинности.

**select_account:** Это значение предоставит пользователю возможность выбора учетной записи, перечисляя все учетные записи в сеансе.

**согласие:** Это значение будет вызывать диалог о согласии OAuth, который позволяет пользователям предоставлять разрешения приложению.

**нет:** Это значение гарантирует, что пользователь не видит интерактивный запрос. Рекомендуется не передавать это значение интерактивным методам в MSAL.js, так как оно может иметь неожиданное поведение. Вместо этого `acquireTokenSilent` используйте метод для достижения бесшумных вызовов.

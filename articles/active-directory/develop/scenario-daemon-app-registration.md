# <a name="daemon-app-that-calls-web-apis---app-registration"></a>Приложение управляющей программы, вызовы веб-интерфейсы API — Регистрация приложения

Для управляющей программой Вот что вам необходимо знать при регистрации приложения.

## <a name="supported-account-types"></a>Поддерживаемые типы учетных записей

Учитывая, что использование приложений управляющей программы смысл только в клиенте Azure AD, при создании приложения необходимо будет выбрать:

- либо **учетных записей в организации каталога только**. Этот выбор имеет наиболее распространенный случай, как управляющая программа приложений обычно создаются разработчиками line-of-business (LOB).
- или **учетных записей в любой организации directory**. Сделать этот выбор будет, если вы являетесь независимым поставщиком по предоставляет средство служебная программа с заказчиками. Вам потребуется Администраторы клиентов заказчика требуется утвердить его.

## <a name="authentication---no-reply-uri-needed"></a>Проверка подлинности - URI ответа не требуется

В случае, где приложение конфиденциальный клиент использует **только** поток учетных данных клиента, URL-адрес ответа не должны быть зарегистрированы. Она не нужна для конфигурации/построение приложения. Поток клиентских учетных данных не использует его.

## <a name="api-permissions---app-permissions-and-admin-consent"></a>Разрешения API - приложению разрешения и согласие администратора

Управляющее приложение может запросить только разрешений приложения для API-интерфейсы (не делегированных разрешений). В **API-разрешение** страницы для регистрации приложения, после выбора **добавить разрешение** и выбрано семейство API, выберите **разрешения приложения**, а затем выберите разрешение

![Приложению разрешения и согласие администратора](media/scenario-daemon-app/app-permissions-and-admin-consent.png)

Управляющая программа приложениям у администратора клиента предварительно согласие на приложение, вызывающее веб-API. Это согласие предоставляется в том же **API-разрешение** страницу, выбрав клиента администратора **предоставлять согласие администратора *нашей организации***

Если вы являетесь независимым разработчиком, Создавшим построения многопользовательского приложения, следует проверить [развертывание – вариант приложений управляющей программы с несколькими клиентами](scenario-daemon-production.md#deployment---case-of-multi-tenant-daemon-apps) абзаца.

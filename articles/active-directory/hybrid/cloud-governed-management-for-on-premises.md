# <a name="how-azure-ad-delivers-cloud-governed-management-for-on-premises-workloads"></a>Как Azure AD обеспечивает Cloud регулируемого управления для локальных рабочих нагрузок

Azure Active Directory (Azure AD) — это комплексная идентификации как услуги (IDaaS) решение, используемого миллионами организаций, которые охватывают все аспекты идентификации, управления доступом и безопасности. Azure AD содержит более миллиарда пользователей и помогает пользователям вход и безопасный доступ к оба:

* Внешние ресурсы, такие как Microsoft Office 365, портал Azure и тысячи других приложений программного обеспечения как услуга (SaaS).
* Внутренним ресурсам, например приложений в корпоративной сети организации и интрасети, вместе с любой облачных приложений, разработанных этой организации.

Организации могут использовать Azure AD, если они есть «облачных», а также для гибридного развертывания, при наличии локальных рабочих нагрузок. Гибридное развертывание Azure AD может быть частью стратегии организации для миграции в облако ИТ-активами или продолжить интегрировать существующие в локальную инфраструктуру вместе с новых облачных служб.

Исторически сложилось так, что «гибридные» в организациях наблюдается Azure AD как расширение существующей локальной инфраструктурой. В этих случаях развертывания администрирования управления удостоверений в локальной, Windows Server Active Directory или другими системами внутреннего каталога являются контрольные точки, и пользователи и группы синхронизируются из этих систем с облачным каталогом, например Azure AD. Если эти удостоверения в облаке, они могут доступны в Office 365, Azure и других приложений.

![Жизненный цикл удостоверений](media/cloud-governed-management-for-on-premises//image1.png)

Как организации переносят часть ИТ-инфраструктуры приложений в облако, многие ищут улучшенную безопасность и упрощенное управление возможности управления удостоверениями как услуга. Обеспечиваемая облаком функций IDaaS в Azure AD Ускорьте переход в облако, предоставляя решения и возможности, которые позволяют организациям быстро внедрять и переместить их управления удостоверениями из традиционных локальных управляемых управления системы в Azure AD, продолжая поддерживать существующие, а также новые приложения.

В этом документе описаны стратегии корпорации Майкрософт для гибридных IDaaS и описывает, как организации могут использовать Azure AD для их приложений.

## <a name="the-azure-ad-approach-to-cloud-governed-identity-management"></a>Подход Azure AD для управления идентификацией, управляемое облако

Что организаций переход в облако они должны гарантии, что они имеют элементы управления над своей средой полный - более безопасным и дополнительные представления о действиях, поддерживаемых автоматизации и упреждающее insights. "**Cloud регулируемого управления**» описывается, как организации управлять и управляющих их пользователей, приложений, групп и устройствами из облака.

В современном мире технологий организациям необходимо иметь возможность эффективного управления в масштабе, из-за распространение приложений SaaS и все большую роль совместной работы и внешние удостоверения. Новый риск альбомная облака означает, что организации должны быть повысить скорость отклика — вредоносного субъекта, который представляет угрозу для пользователя cloud может повлиять на облачных и локальных приложений.

В частности, гибридные, организациям необходимо иметь возможность делегировать и автоматизировать задачи, который Исторически ИТ было вручную. Чтобы автоматизировать задачи, они должны API-интерфейсов и процессы, которые организуют жизненного цикла различные ресурсы с удостоверением (пользователей, групп, приложений, устройств), поэтому они могут делегировать повседневное управление этими ресурсами для нескольких сотрудников за пределами Основные ИТ-специалистов. Azure AD отвечает этим требованиям, через управление учетными записями пользователей и собственную проверку подлинности для пользователей, не требуя локальной инфраструктуры удостоверений. Не расширения локальной инфраструктуры могут воспользоваться организации, имеющие новый сообщества пользователей, например деловых партнеров, который не был изначально передан в их в локальном каталоге, но, управления доступом является критически важным для достижения бизнес-показатели.

Кроме того управление не завершена без управления---и система управления в этом новом мире является неотъемлемой частью системы идентификации, а не надстройку. Управление удостоверениями позволяет организациям для управления удостоверениями и доступом жизненный цикл сотрудников, партнеров и поставщиков и служб и приложений.

Включение функции управления удостоверениями облегчает позволяют организации к переходу в облако, управляемое управления, позволяет ИТ-специалистам увеличить, новые трудности, связанные с гостей и получить более глубокие сведения и автоматизации, чем клиентов испытывали с в локальной инфраструктуре. Система управления в этом новом мире означает возможность организации прозрачности, отслеживать и соответствующие элементы управления доступом к ресурсам в пределах организации. С помощью Azure AD, операции безопасности и аудита команды, имеют видимость у кого есть---и который должен иметь - доступ к каким ресурсам в организации (на какие устройства), что эти пользователи делают с такой доступ и того, имеет ли организация и использует соответствующие элементы управления для удаления или ограничить доступ в соответствии с компании или законодательным требованиям.

Новая модель управления преимущества организациям SaaS и бизнес-приложений (LOB), так как они более легко смогут управлять и защищать доступ к этим приложениям. Благодаря интеграции приложений с Azure AD, организации смогут использовать и управлять доступом к облачных, так и в локальной порождено удостоверения, постоянно. Управление жизненным циклом приложений становится более автоматизировано и Azure AD предоставляет подробные сведения о использования приложений, не было сложно достичь в локальной identity management. Через Azure AD, групп Office 365 и группами самообслуживания, организации могут легко создавать группы для управления доступом и совместной работы и добавить или удалить пользователей в облаке, чтобы включить совместной работы и управления требований доступа.

Выбор возможностей справа Azure AD для облака, управляемое управления зависит от приложения, которые должны использоваться и как эти приложения будут интегрироваться с Azure AD. В следующих разделах описаны подходы для приложения, интегрированные AD и приложения, использующие протоколы федерации (например, SAML, OAuth или OpenID Connect).

## <a name="cloud-governed-management-for-ad-integrated-applications"></a>Управление облачной регулируемого для приложения, интегрированные AD

Azure AD повышает управления для организации с локальной интегрированный с Active Directory приложений с помощью условного доступа и безопасного удаленного доступа к этим приложениям. Кроме того Azure AD предоставляет управление жизненным циклом учетной записи и управления учетными данными пользователя существующих учетных записей AD, включая:

* **Условный доступ для локальных приложений и безопасного удаленного доступа**

Для многих организаций, первым шагом в управлении доступом в облаке для Интернета AD интегрировать локальных и удаленных приложений, основанных на рабочей является развертывание [прокси приложения](https://docs.microsoft.com/azure/active-directory/manage-apps/application-proxy) перед эти приложения для обеспечения безопасности удаленный доступ.

Выполнив единый вход в AAD, пользователи получают доступ к облачным и локальным приложениям через внешний URL-адрес или внутренний портал приложений. Например прокси приложения предоставляет удаленный доступ и единый вход для удаленного рабочего стола, SharePoint, а также приложения, такие как Tableau, Qlik и бизнес-приложений. Кроме того, политики условного доступа можно включить отображение [условия использования](https://docs.microsoft.com/azure/active-directory/governance/active-directory-tou) и [обеспечение пользователь согласился им](https://docs.microsoft.com/azure/active-directory/conditional-access/require-tou) прежде чем он сможет получить доступ к приложению.

* **Автоматическое управление жизненным циклом учетных записей Active Directory**

Функции управления удостоверениями помогает организациям достичь баланса между *производительность* ---насколько быстро пользователь может иметь доступ к ресурсам, необходимым, например при их присоединения к организации? ---и *безопасности* ---следует их доступ изменение за время, например при изменении его состояние занятости? Управление жизненным циклом удостоверений — базовый процесс службы Identity Governance. Чтобы обеспечить эффективность управления в масштабе всей организации, необходимо модернизировать инфраструктуру управления жизненным циклом удостоверений для приложений.

Для организаций, используя в качестве своей системы HCM Workday, Azure AD можно обеспечить, учетных записей пользователей в AD [автоматически подготовлены и отмене подготовки для работников в Workday](https://docs.microsoft.com/azure/active-directory/saas-apps/workday-inbound-tutorial). Поэтому приводит к улучшение производительности пользователей посредством автоматизации birthright учетные записи и управляет риском, гарантируя приложения доступ автоматически обновляется при изменении роли пользователя, или покидает организацию. Подготовки пользователей Workday [плана развертывания](https://aka.ms/WorkdayDeploymentPlan) является пошаговым руководством, в рамках этого руководства организации за счет лучшие методики реализации Workday для подготовки пользователей Active Directory решение в пять этапов.

Azure AD Premium также включает в себя Microsoft Identity Manager, который можно импортировать записи из других локальных систем HCM, включая SAP, Oracle eBusiness, Oracle к PeopleSoft или наоборот.

Бизнес сотрудничество все чаще требуется предоставление доступа к пользователям за пределами вашей организации. [Azure AD B2B](https://docs.microsoft.com/azure/active-directory/b2b/) совместной работы, организации могут безопасно обмениваться своих приложений и служб с гостевым пользователям и внешним партнерам, сохраняя контроль над корпоративными данными.

Azure AD можно [автоматически создавать учетные записи в AD для гостевых пользователей](https://docs.microsoft.com/azure/active-directory/b2b/hybrid-cloud-to-on-premises) при необходимости, реализация бизнес-Гости, для доступа к локальных приложений, интегрированных в AD без необходимости другой пароль. Можно настроить организаций [политики многофакторной проверки подлинности (MFA) для гостевого пользователя](https://docs.microsoft.com/azure/active-directory/b2b/conditional-access)s, поэтому многофакторной проверки Подлинности проверяет, выполняются во время проверки подлинности прокси-сервера приложения. Кроме того, любой [проверок доступа](https://docs.microsoft.com/azure/active-directory/governance/manage-guest-access-with-access-reviews) , выполняются в облаке B2B пользователи применять к локальным пользователям. Например если с помощью политик управления жизненным циклом удаляет облачного пользователя, локальный пользователь также удаляется.

**Учетные данные управления для учетных записей Active Directory** Azure AD, самостоятельного сброса пароля позволяет пользователям, вы забыли пароль для повторной проверки подлинности и сбрасывать свои пароли с измененным паролям [записи в локальной Active Directory](https://docs.microsoft.com/azure/active-directory/authentication/concept-sspr-writeback). Процесс сброса пароля можно также использовать политики паролей в локальной Active Directory: Когда пользователь сбрасывает свой пароль, он проверяется на соответствие политике локальной Active Directory перед его фиксацией в соответствующем каталоге. Самостоятельный сброс пароля [плана развертывания](https://aka.ms/deploymentplans/sspr) описаны рекомендуемые способы развертывания самостоятельного сброса пароля для пользователей с помощью веб-приложения и интерфейсы, интегрированной в Windows.

## <a name="ongoing-access-management-across-cloud-and-on-premises-applications"></a>Непрерывное управление доступом в облачных и локальных приложений

Организациям нужна процесс для управления доступом является масштабируемой. Пользователи по-прежнему накапливаются права доступа и в итоге за то, что изначально была подготовлена для них. Кроме того предприятиям необходимо поддерживать возможность эффективно разрабатывать и применять политики доступа и элементы управления на постоянной основе.

Обычно ИТ-подразделения передают решения по утверждению доступа ответственным лицам в организации. Также в этом процессе могут участвовать сами пользователи. Например, пользователям, которые работают с конфиденциальными данными клиентов в маркетинговом приложении для Европы, нужна информация о соответствующих корпоративных политиках. Гостевые пользователи также не ожидая обработки требований для данных в организации, к которому они пригласили.

Организации можно автоматизировать процесс жизненного цикла доступа за счет использования технологий, таких как [динамические группы](https://docs.microsoft.com/azure/active-directory/users-groups-roles/groups-dynamic-membership), а также для подготовки пользователей [приложений SaaS](https://docs.microsoft.com/azure/active-directory/saas-apps/tutorial-list), или [приложений интегрированы с помощью системы для управления удостоверениями между доменами (SCIM](https://docs.microsoft.com/azure/active-directory/manage-apps/use-scim-to-provision-users-and-groups)) standard. Организации также можно выбирать, какие [гостевые пользователи имеют доступ к локальным приложениям](https://docs.microsoft.com/azure/active-directory/b2b/hybrid-cloud-to-on-premises). Также эти права доступа можно регулярно проверять с помощью [проверок доступа Azure Active Directory](https://docs.microsoft.com/azure/active-directory/governance/access-reviews-overview).

## <a name="future-directions"></a>Дальнейшие направления развития

В гибридных средах, стратегия корпорации Майкрософт является возможность развертывания где **облако — плоскости управления для идентификации**и локальными каталогами, так и другими системами удостоверений, например Active Directory и других локальных приложения, являющиеся целевыми для подготовки пользователей с доступом. Такая стратегия будет продолжать обеспечения права, удостоверений и доступа в этих приложениях и рабочих нагрузок, которые зависят от их. В этом конечного состояния организации смогут диска конечных пользователей полностью из облака.

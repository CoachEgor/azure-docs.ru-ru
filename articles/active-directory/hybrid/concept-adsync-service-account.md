# <a name="adsync-service-account"></a>Учетная запись службы ADSync
Azure AD Connect устанавливает предпосылок, которая организует синхронизацию между Active Directory и Azure Active Directory.  Служба синхронизации Microsoft Azure AD Sync (ADSync) работает на сервере в вашей предварительной среде.  Учетные данные службы устанавливаются по умолчанию в установках Express, но могут быть настроены в соответствии с вашими организационными требованиями безопасности.  Эти учетные данные не используются для подключения к вашим вонючим лесам или Active Directory Azure.

Выбор учетной записи службы ADSync является важным решением планирования, который необходимо принять до установки Azure AD Connect.  Любая попытка изменить учетные данные после установки приведет к тому, что служба не запустится, потеряет доступ к базе данных синхронизации и не даст аутентификацию в подключенных каталогах (Azure и AD DS).  Синхронизация не происходит до тех пор, пока исходные учетные данные не будут восстановлены.

## <a name="the-default-adsync-service-account"></a>Учетная запись службы ADSync по умолчанию

При запуске на сервере-участнике служба AdSync работает в контексте виртуального сервисного счета (VSA).  Из-за ограничения продукта при установке на контроллер домена создается учетная запись пользовательского сервиса.  Если учетная запись службы настроек Express не соответствует вашим требованиям безопасности организации, разключите Azure AD Connect, выбрав опцию настройки.  Затем выберите опцию учетной записи службы, которая отвечает требованиям организации.

>[!NOTE]
>Учетная запись службы по умолчанию при установке на контроллер домена имеет форму домена AAD_InstallationIdentifier.  Пароль для этой учетной записи генерируется случайным образом и создает значительные проблемы для восстановления и вращения паролей.  Корпорация Майкрософт рекомендует настроить учетную запись службы во время первоначальной установки на контроллер домена, чтобы использовать либо автономный или групповой управляемый сервисный счет (sMSA / gMSA)

|Местоположение Azure AD Connect|Создана учетная запись сервиса|
|-----|-----|
|Сервер участников|NT СЕРВИС-Адсинк|
|Контроллер домена|доменAAD_74dc30c01e80 (см. примечание)|

## <a name="custom-adsync-service-accounts"></a>Учетные записи пользовательских служб ADSync
Корпорация Майкрософт рекомендует запускать службу ADSync в контексте либо виртуальной учетной записи службы, либо отдельной или групповой учетной записи управляемой службы.  Администратор домена может также создать учетную запись службы, подготовленную для удовлетворения ваших конкретных организационных требований безопасности.   Чтобы настроить учетную запись службы, используемую во время установки, выберите опцию настройки на странице «Настройки экспресса» ниже.   Доступны следующие варианты:

- учетная запись по умолчанию - Azure AD Connect предоставит учетную запись службы, как описано выше
- управляемая учетная запись обслуживания - используйте автономный или групповой MSA, подготовленный администратором
- domain account - используйте учетную запись доменного обслуживания, подготовленную администратором

## <a name="diagnosing-adsync-service-account-changes"></a>Диагностика изменений в учетной записи службы ADSync
Изменение учетных данных службы ADSync после установки приведет к тому, что служба не запустится, потеряет доступ к базе данных синхронизации и не будет проверена на подключенные каталоги (Azure и AD DS).  Предоставление доступа к базе данных к новой учетной записи службы ADSync недостаточно для восстановления этой проблемы. Синхронизация не происходит до тех пор, пока исходные учетные данные не будут восстановлены.

Служба ADSync выдаст сообщение уровня ошибки журналу событий, когда оно не может быть завершено.  Содержание сообщения будет варьироваться в зависимости от того, используется ли встроенная база данных (localdb) или полный S'L.  Ниже приведены примеры записей журнала событий, которые могут присутствовать.

### <a name="example-1"></a>Пример 1

Ключи шифрования службы AdSync не могут быть найдены и воссозданы.  Синхронизация не произойдет до тех пор, пока эта проблема не будет исправлена.

Устранение проблемы ключи шифрования Microsoft Azure AD Sync станут недоступными, если учетные данные службы AdSync будут изменены.  Если учетные данные были изменены, используйте приложение Службы, чтобы изменить учетную запись Log On обратно в его первоначально настроенное значение (напротив. NT SERVICE-AdSync) и перезапустите сервис.  Это позволит немедленно восстановить правильную работу службы AdSync.

Дополнительную информацию можно найти в следующей [статье.](https://go.microsoft.com/fwlink/?linkid=2086764)

### <a name="example-2"></a>Пример 2

Служба не смогла начать сярприза из-за того, что не удалось установить подключение к локальной базе данных (localdb).

Устранение проблемы служба синхронизации Microsoft Azure AD Sync потеряет разрешение на доступ к локальному поставщику баз данных в случае изменения учетных данных службы AdSync.  Если учетные данные были изменены, используйте приложение Службы, чтобы изменить учетную запись Log On на его первоначально настроенное значение (напротив. NT SERVICE-AdSync) и перезапустите сервис.  Это позволит немедленно восстановить правильную работу службы AdSync.

Дополнительную информацию можно найти в следующей [статье.](https://go.microsoft.com/fwlink/?linkid=2086764)

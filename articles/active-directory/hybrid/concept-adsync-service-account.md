# <a name="adsync-service-account"></a>Учетная запись службы ADSync
Azure AD Connect устанавливает локальной службы, который координирует синхронизации между Active Directory и Azure Active Directory.  Службы синхронизации Microsoft Azure AD Sync (ADSync) выполняется на сервере в вашей локальной среде.  Учетные данные для службы заданы по умолчанию в установках Express, но можно настроить в соответствии с требованиями безопасности организации.  Эти учетные данные не используются для подключения локальных лесов или Azure Active Directory.

Выбор ADSync учетной записи службы — важное решение планирования чтобы сделать перед установкой Azure AD Connect.  Любая попытка изменить учетные данные, после установки приведет к службе, не удается запустить, потери доступа к базе данных синхронизации, а не пройдет проверку подлинности в подключенных каталогах (Azure и AD DS).  Синхронизация не будет выполняться до восстановления исходных учетных данных.

## <a name="the-default-adsync-service-account"></a>Учетная запись службы ADSync по умолчанию

При запуске на рядовом сервере, службы AdSync выполняется в контексте виртуальной учетной записи службы (VSA).  Из-за ограничений продукта пользовательскую учетную запись службы создается при установке на контроллере домена.  Если стандартные параметры учетной записи службы не отвечают требованиям организации, развертывание Azure AD Connect, выбрав параметр "Настройка".  Выберите вариант учетной записи службы, который соответствует требованиям организации.

>[!NOTE]
>Учетная запись службы по умолчанию при установке на контроллере домена имеет форму Domain\AAD_InstallationIdentifier.  Пароль для этой учетной записи формируется случайным образом и является значительным препятствием для поворота восстановления и пароль.  Корпорация Майкрософт рекомендует Настройка учетной записи службы во время первоначальной установки на контроллере домена для использования в отдельном или групповую управляемую учетную запись службы (sMSA / групповой управляемой учетной записи)

|Расположение Azure AD Connect|Учетная запись службы|
|-----|-----|
|Рядовой сервер|NT SERVICE\ADSync|
|Контроллер домена|Domain\AAD_74dc30c01e80 (см. Примечание)|

## <a name="custom-adsync-service-accounts"></a>Пользовательские учетные записи службы ADSync
Корпорация Microsoft рекомендует под управлением ADSync службы в контексте учетной записи виртуальной службы или автономный или групповую управляемую учетную запись службы.  Администратор домена также можно создать учетную запись службы, подготовленных в соответствии с требованиями безопасности организации.   Для настройки учетной записи службы во время установки, выберите параметр "Настройка" на странице "стандартные параметры" ниже.   Доступны следующие варианты:

- Учетная запись по умолчанию — Azure AD Connect предоставит учетной записи службы, как описано выше
- управляемые учетной записи службы — использовать автономный или групповую управляемую учетную запись службы, подготовленных с помощью к администратору
- Учетная запись домена — использование учетной записи, подготовленных с помощью к администратору службы домена

## <a name="diagnosing-adsync-service-account-changes"></a>Диагностика изменения учетной записи службы ADSync
Изменение учетных данных для службы ADSync, после установки приведет к службе, не удается запустить, потери доступа к базе данных синхронизации, а не пройдет проверку подлинности в подключенных каталогах (Azure и AD DS).  Предоставление разрешений базы данных для новой учетной записи службы ADSync не позволяет устранить эту проблему. Синхронизация не будет выполняться до восстановления исходных учетных данных.

Служба ADSync выдаст уровня сообщение об ошибке в журнал событий, когда не удается запустить.  Содержимое сообщения будет зависеть от ли встроенная база данных (localdb) или полную версию SQL Server уже используется.  Ниже приведены примеры записей журнала событий, которые могут присутствовать.

### <a name="example-1"></a>Пример 1

Не удалось найти ключи шифрования для службы AdSync и созданы заново.  Синхронизация не будет выполняться, пока эта проблема не будет устранена.

Устранение этой проблемы Microsoft Azure AD Sync ключи шифрования станет недоступным, если изменяются AdSync вход в систему учетных данных службы.  Если учетные данные были изменены, выполните приложение «службы», чтобы изменить учетную запись входа в первоначально настроенной значение (например) NT SERVICE\AdSync) и перезапустите службу.  Это позволит немедленно восстановить правильной работы службы AdSync.

См. следующие [статье](https://go.microsoft.com/fwlink/?linkid=2086764) для получения дополнительных сведений.

### <a name="example-2"></a>Пример 2

Службе не удалось запуститься, поскольку не удалось установить подключение к локальной базе данных (localdb).

Устранение этой проблемы Microsoft Azure AD Sync службы потеряют разрешение на доступ к поставщику локальной базы данных, если изменяются AdSync вход в систему учетных данных службы.  Если учетные данные были изменены выполните приложение «службы», чтобы изменить учетную запись входа в первоначально настроенной значение (например) NT SERVICE\AdSync) и перезапустите службу.  Это позволит немедленно восстановить правильной работы службы AdSync.

См. следующие [статье](https://go.microsoft.com/fwlink/?linkid=2086764) для получения дополнительных сведений.

Дополнительные сведения, которые поставщик возвратил следующие сведения об ошибке:
 

``` 
OriginalError=0x80004005 OLEDB Provider error(s): 
Description  = 'Login timeout expired'

Minor Number = 0 
Description  = 'A network-related or instance-specific error has occurred while establishing a connection to SQL Server. Server is not found or not accessible. Check if instance name is correct and if SQL Server is configured to allow remote connections. For more information see SQL Server Books Online.'
```

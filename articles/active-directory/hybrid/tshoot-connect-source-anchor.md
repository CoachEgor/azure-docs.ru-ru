# <a name="troubleshooting-source-anchor-issues-during-installation"></a>Устранение неполадок источника привязки во время установки
В этой статье описывается различные проблемы, которые могут возникнуть во время установки и предлагает способы устранения этих проблем, связанные с привязки к источнику.

## <a name="invalid-source-anchor-in-azure-active-directory"></a>Недопустимый источник привязки в Azure Active Directory

### <a name="custom-installation"></a>Выборочная установка

При выборочной установке Azure AD Connect считывает политика источника привязки из Azure Active Directory. Если политика существует в Azure Active Directory, Azure AD Connect применяет ту же политику, если оно не переопределено пользователем. Мастер сообщит о том, какой атрибут был считан. Кроме того мастер отобразит предупреждение при попытке переопределить политику источника привязки.

Во время этой операцией чтения вполне возможно, что политика источника привязки в Azure Active Directory является непредвиденное. В этом случае Azure AD Connect не знает, какие привязки к источнику для использования и требуется ручное переопределение.</br>
![Непредвиденная](media/tshoot-connect-source-anchor/source1.png)

Чтобы устранить эту проблему, вручную можно переопределить, выбрав определенный атрибут привязки к источнику. Продолжить этот параметр только в том случае, если вы уверены, какой атрибута для выбора. Если нет уверенности, обратитесь в службу [поддержки Майкрософт](https://support.microsoft.com/contactus/) рекомендации. При изменении политики привязки источника, он может разорвать связь между пользователями в локальной и связанным ресурсам Azure.</br>
![Непредвиденная](media/tshoot-connect-source-anchor/source2.png)

### <a name="express-installation"></a>Экспресс-установка
Во время экспресс-установки Azure AD Connect считывает политика источника привязки из Azure Active Directory. Если политика существует в Azure Active Directory, Azure AD Connect применяет ту же политику. Нет возможности делать вручную.

Во время этой операцией чтения вполне возможно, что политика источника привязки в Azure Active Directory является непредвиденное. В этом случае Azure AD Connect не знает, какой должна быть привязки к источнику.</br>
![Непредвиденная](media/tshoot-connect-source-anchor/source3.png)

Чтобы устранить эту проблему, необходимо повторно установить с помощью пользовательского режима и переопределить, выбрав определенный атрибут привязки к источнику. Продолжить этот параметр только в том случае, если вы уверены, какой атрибута для выбора. Если нет уверенности, обратитесь в службу [поддержки Майкрософт](https://support.microsoft.com/contactus/) рекомендации. При изменении политики привязки источника, он может разорвать связь между пользователями в локальной и связанным ресурсам Azure.

### <a name="invalid-source-anchor-in-sync-engine"></a>Недопустимый источник привязки в модуле синхронизации
Во время установки, это возможно Azure AD Connect попытки переустановить конфигурацию модуль синхронизации, с помощью привязки Недопустимый исходный. Эта операция, скорее всего проблеме с продуктом и установка Azure AD Connect не сможет. Обратитесь к [поддержки Майкрософт](https://support.microsoft.com/contactus/) при выполнении в этой проблемы.</br>
![Непредвиденная](media/tshoot-connect-source-anchor/source4.png)

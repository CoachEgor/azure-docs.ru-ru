# <a name="health-service-data-is-not-up-to-date-alert"></a>Оповещение "Данные службы работоспособности неактуальны"

## <a name="overview"></a>Обзор

Агенты на предприимжениях, которые периодически загружают мониторинг Azure AD Connect Health, периодически загружают данные в службу здравоохранения Azure AD Connect. Если служба не получает данные от агента, информация, представленная порталом, будет устаревшей. Чтобы подчеркнуть проблему, служба поднимет **данные службы здравоохранения не обновляются в актуальном состоянии.** Это оповещение генерируется, когда служба не получила полных данных за последние два часа.  

- Предупреждение **о состоянии предупреждения о** состоянии предупреждения запускается, если Служба работоспособности получила только **частичные** типы данных, отправленные с сервера за последние два часа. Предупреждение о состоянии предупреждения не вызывает уведомления по электронной почте для настроенных получателей. 
- Предупреждение о состоянии **ошибки** запускается, если Служба работоспособности не получила типы данных с сервера за последние два часа. Предупреждение о состоянии ошибки запускает уведомления электронной почты для настроенных получателей.

Служба получает данные от агентов, работающих на банкоматах, в зависимости от типа обслуживания. В следующей таблице перечислены агенты, выполняющие сядёт в машине, что они делают, и типы данных, генерируемые службой. В некоторых случаях, Есть несколько услуг, участвующих в этом процессе, так что любой из них может быть виновником. 

## <a name="understanding-the-alert"></a>Понимание оповещения

Лезвие **"Детали оповещения"** показывает, когда произошло оповещение и было обнаружено в последний раз. Фоновый процесс, который выполняется каждые два часа, генерирует и переоценивает оповещение. В следующем примере первоначальное предупреждение произошло в 03/10 в 9:59. Предупреждение по-прежнему существовало на 03/12 в 10:00, когда оповещение было оценено снова. Лезвие также подробно описывает время, когда Служба здравоохранения в последний раз получала определенный тип данных.

## <a name="troubleshooting-steps"></a>Действия по устранению неполадок 

Ниже приведены шаги, необходимые для диагностики проблемы. Во-первых, это набор основных проверок, которые являются общими для всех типов служб. 

> [!IMPORTANT] 
> Для этого оповещения соблюдается [политика хранения данных](reference-connect-health-user-privacy.md#data-retention-policy) Connect Health

* Убедитесь, что последние версии агентов установлены. Просмотр [истории выпуска](reference-connect-health-version-history.md). 
* Убедитесь, что **службы** Azure AD Connect Подключаются к службе безопасности на машине. Например, Connect Health для AD FS должен состоять из трех служб.
  ![Проверка Azure AD Connect Health](./media/how-to-connect-health-agent-install/install5.png)

* Откройте [раздел требований](how-to-connect-health-agent-install.md#requirements) и убедитесь, что все они соблюдены.
* Используйте [средство тестирования подключения](how-to-connect-health-agent-install.md#test-connectivity-to-azure-ad-connect-health-service) для обнаружения возможных проблем с подключением.
* Если вы используете прокси-сервер HTTP, следуйте этим [этапам конфигурации](how-to-connect-health-agent-install.md#configure-azure-ad-connect-health-agents-to-use-http-proxy).

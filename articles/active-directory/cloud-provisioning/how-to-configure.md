# <a name="create-a-new-configuration-for-azure-ad-connect-cloud-based-provisioning"></a>Создание новой конфигурации для облачного подключения Azure AD

После установки агента необходимо войти на портал Azure и настроить настройку облачного каталога Azure Active Directory (Azure AD) Connect. Выполните следующие действия, чтобы включить агента.

## <a name="configure-provisioning"></a>Настройка подготовки
Чтобы настроить подготовку, выполните следующие шаги.

1.  На портале Azure выберите **Active Directory Azure.**
1.  Выберите **Azure AD Connect**.
1.  Выберите **Управление подготовкой (Предварительный просмотр)**.

    ![Управление подготовкой (Предварительный просмотр)](media/how-to-configure/manage1.png)

1.  Выберите **новую конфигурацию.**
1.  На экране конфигурации домысел предварительно заселен доселем.
1.  Введите **электронную почту Уведомления**. Это письмо будет уведомляться, когда подготовка не является здоровой.
1.  Переместите селектор для **включения**и выберите **Сохранить**.

    ![Подготовка Azure AD (Предварительный просмотр)](media/tutorial-single-forest/configure2.png)

## <a name="scope-provisioning-to-specific-users-and-groups"></a>Подготовка к охвату для конкретных пользователей и групп
Можно использовать область действия агента для синхронизации конкретных пользователей и групп с помощью групп active Directory или организационных подразделений. Вы не можете настроить группы и организационные единицы в конфигурации. 

1.  На портале Azure выберите **Active Directory Azure.**
1.  Выберите **Azure AD Connect**.
1.  Выберите **Управление подготовкой (Предварительный просмотр)**.
1.  В **соответствии с конфигурацией**выберите конфигурацию.

    ![Раздел конфигурации](media/how-to-configure/scope1.png)

1.  В разделе **Настройка** выберите **Все пользователи**, чтобы изменить область действия правила конфигурации.

    ![Все пользователи опции](media/how-to-configure/scope2.png)

1. Справа можно изменить область действия, включив в нее только группы безопасности. Введите выдающееся имя группы и **выберите Добавить**.

    ![Выбранный вариант групп безопасности](media/how-to-configure/scope3.png)

1.  Или можно изменить область действия, включив в него только определенные организационные единицы. Выберите **готовое** и **сохраните**.  
2.  После изменения сферы действия необходимо [перезапустить подготовку,](#restart-provisioning) чтобы немедленно синхронизировать изменения.

    ![Выбранный вариант организационных единиц](media/how-to-configure/scope4.png)


## <a name="restart-provisioning"></a>Перезапуск подготовки 
Если вы не хотите ждать следующего запланированного запуска, запустите запуск подготовки с помощью **кнопки подготовки restart.** 
1.  На портале Azure выберите **Active Directory Azure.**
1.  Выберите **Azure AD Connect**.
1.  Выберите **Управление подготовкой (Предварительный просмотр)**.
1.  В **соответствии с конфигурацией**выберите конфигурацию.

    ![Выбор конфигурации для перезапуска подготовки](media/how-to-configure/scope1.png)

1.  В верхней части выберите **подготовку restart.**

## <a name="remove-a-configuration"></a>Удалить конфигурацию
Чтобы удалить конфигурацию, выполните следующие действия.

1.  На портале Azure выберите **Active Directory Azure.**
1.  Выберите **Azure AD Connect**.
1.  Выберите **Управление подготовкой (Предварительный просмотр)**.
1.  В **соответствии с конфигурацией**выберите конфигурацию.

    ![Выбор конфигурации для удаления конфигурации](media/how-to-configure/scope1.png)

1.  В верхней части экрана конфигурации выберите **Удалить**.

    ![Кнопка "Удалить"](media/how-to-configure/remove1.png)

>[!IMPORTANT]
>До удаляния конфигурации подтверждения нет. Убедитесь, что это действие, необходимое для выбора **удаления.**

# <a name="azure-active-directory-connect-faq"></a>Часто задаваемые вопросы по Azure Active Directory Connect

Ознакомьтесь с часто задаваемыми вопросами о подготовке облака Azure Active Directory (Azure AD) Connect.

## <a name="general-installation"></a>Общая установка

**Вопрос. Как часто запускается подготовка облака?**

Запуск подготовки облака запланирован каждые 2 минуты. Каждые 2 минуты все изменения хэша пользователя, группы и пароля будут подготавливаться в Azure AD.

**Вопрос. При первом запуске отображаются ошибки синхронизации хэша паролей. Почему?**

Это ожидаемое поведение. Ошибки возникают из-за того, что объект пользователя отсутствует в Azure AD. После подготовки пользователя в Azure AD необходимо подготовить хэши паролей в рамках последующего запуска. После нескольких запусков убедитесь, что при синхронизации хэша паролей больше не возникает ошибок.

**Вопрос. В чем разница между синхронизацией Azure AD Connect и подготовкой облака?**

При синхронизации Azure AD Connect подготовка выполняется на локальном сервере синхронизации. Конфигурация хранится на локальном сервере синхронизации. При подготовке облака Azure AD Connect конфигурация подготовки хранится и выполняется в облаке как часть службы подготовки Azure AD. 

**Вопрос. Можно ли использовать подготовку облака для синхронизации нескольких лесов Active Directory?**

Да. Подготовку облака можно использовать для синхронизации нескольких лесов Active Directory. В среде с несколькими лесами все ссылки (например, на диспетчер) должны находиться в пределах домена.  

**Вопрос. Как обновляется агент?**

Агенты обновляются корпорацией Майкрософт автоматически. Это снижает нагрузку на ИТ-ресурсы в контексте тестирования и проверки новых версий агента. 

**Вопрос. Можно ли отключить автоматическое обновление?**

Поддерживаемый способ отключения автоматического обновления отсутствует.

**Вопрос. Можно ли изменить привязку к источнику для подготовки облака?**

По умолчанию при подготовке облака используется атрибут ms-ds-consistency-GUID с откатом к ObjectGUID в качестве привязки к источнику. Поддерживаемый способ изменения привязки к источнику отсутствует.

**Вопрос. При использовании подготовки облака отображаются новые субъекты-службы с доменными именами AD. Так и нужно?**

Да. В процессе подготовки облака для настройки создается субъект-служба, в качестве имени которого используется имя домена. Не вносите изменения в конфигурацию субъекта-службы.

**Вопрос. Что происходит, когда от синхронизируемого пользователя требуется смена пароля при следующем входе в систему?**

Если при подготовке облака включена синхронизация хэша паролей, а синхронизируемый пользователь должен сменить пароль при следующем входе в локальную службу AD, при подготовке облака не подготавливается хэш пароля для изменения в Azure AD. После изменения пароля он подготавливается из AD в Azure AD.

**Вопрос. Поддерживает ли подготовка облака обратную запись MS-DS-consistencyGUID для любого объекта?**

Нет, подготовка облака не поддерживает обратную запись MS-DS-consistencyGUID для любого объекта (включая объекты пользователя). 

**Вопрос. Я подготавливаю пользователей с помощью подготовки облака. Выполняю удаление конфигурации. Почему в Azure AD по-прежнему отображаются старые синхронизированные объекты?** 

При удалении конфигурации подготовка облака не очищает синхронизированные объекты в Azure AD. Чтобы гарантировать отсутствие старых объектов, измените область конфигурации на пустую группу или подразделения. После выполнения подготовки и очистки объектов отключите и удалите конфигурацию.

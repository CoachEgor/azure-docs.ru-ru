# <a name="azure-active-directory-connect-faq"></a>Часто задаваемые вопросы по Azure Active Directory Connect

Ознакомьтесь с часто задаваемыми вопросами о облачной подготовке Azure Active Directory (Azure AD) Connect.

## <a name="general-installation"></a>Общая установка

**Вопрос. Как часто запускается облачная подготовка?**

Запуск облачной подготовки запланирован каждые 2 минуты. Каждые 2 минуты все изменения хэша пользователя, группы и пароля будут подготавливаться в Azure AD.

**Вопрос. При первом запуске отображаются ошибки синхронизации хэша паролей. Почему?**

Это ожидаемое поведение. Ошибки возникают из-за того, что объект пользователя отсутствует в Azure AD. После подготовки пользователя в Azure AD необходимо подготовить хэши паролей в рамках последующего запуска. После нескольких запусков убедитесь, что при синхронизации хэша паролей больше не возникает ошибок.

**Вопрос. В чем разница между синхронизацией Azure AD Connect и облачной подготовкой?**
При синхронизации Azure AD Connect подготовка выполняется на локальном сервере синхронизации. Конфигурация хранится на локальном сервере синхронизации. При облачной подготовке Azure AD Connect конфигурация подготовки хранится и выполняется в облаке как часть службы подготовки Azure AD. 

**Вопрос. Можно ли использовать облачную подготовку для синхронизации нескольких лесов Active Directory?**

Да. Oблачную одготовку можно использовать для синхронизации нескольких лесов Active Directory. В среде с несколькими лесами все ссылки (например, на диспетчер) должны находиться в пределах домена.  

**Вопрос. Как обновляется агент?**

Агенты обновляются корпорацией Майкрософт автоматически. Это снижает нагрузку на ИТ-отдел в контексте тестирования и проверки новых версий агента. 

**Вопрос. Можно ли отключить автоматическое обновление?**

Поддерживаемый способ отключения автоматического обновления отсутствует.

**Вопрос. Можно ли изменить привязку к источнику для облачной подготовки?**

По умолчанию при облачной подготовке используется атрибут ms-ds-consistency-GUID с откатом к ObjectGUID в качестве привязки к источнику. Поддерживаемый способ изменения привязки к источнику отсутствует.

**Вопрос. При использовании облачной подготовки отображаются новые субъекты-службы с доменными именами AD. Так и нужно?**

Да. В процессе облачной подготовки для настройки создается субъект-служба, в качестве имени которого используется имя домена. Не вносите изменения в конфигурацию субъекта-службы.

**Вопрос. Что происходит, когда от синхронизируемого пользователя требуется смена пароля при следующем входе в систему?**

Если при облачной подготовке включена синхронизация хэша паролей, а синхронизируемый пользователь должен сменить пароль при следующем входе в локальную службу AD. При облачной подготовке не подготавливается хэш пароля для изменения в Azure AD. После изменения пароля он подготавливается из AD в Azure AD.

**Вопрос. Поддерживает ли подготовка облака обратную запись MS-DS-consistencyGUID для любого объекта?**

Нет, подготовка облака не поддерживает обратную запись MS-DS-consistencyGUID для любого объекта (включая объекты пользователя). 

**Вопрос. Я подготавливаю пользователей с помощью подготовки облака. Выполняю удаление конфигурации. Почему в Azure AD по-прежнему отображаются старые синхронизированные объекты?** 

При удалении конфигурации подготовка облака не очищает синхронизированные объекты в Azure AD. Чтобы гарантировать отсутствие старых объектов, измените область конфигурации на пустую группу или подразделения. После выполнения подготовки и очистки объектов отключите и удалите конфигурацию.

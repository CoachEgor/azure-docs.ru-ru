# <a name="azure-active-directory-identity-protection-notifications"></a>Уведомления защиты идентификации Azure Active Directory

Azure AD Identity Protection отправляет два типа автоматизированных уведомлений, чтобы помочь вам управлять рисками и рисками пользователей:

- сообщение электронной почты об обнаружении пользователей в группе риска;
- сообщение электронной почты еженедельного дайджеста.

В этой статье рассматриваются оба уведомления по электронной почте.

## <a name="users-at-risk-detected-email"></a>сообщение электронной почты об обнаружении пользователей в группе риска;

При обнаружении учетной записи в группе риска служба "Защита идентификации Azure Active Directory" создает уведомление по электронной почте с темой **Users at risk detected** (Обнаружены пользователи в группе риска). В таком электронном сообщении содержится ссылка на отчет **[Пользователи, находящиеся в группе риска](../reports-monitoring/concept-user-at-risk.md)**. Мы советуем немедленно проверить пользователей в группе риска.

Конфигурация для этого оповещения позволяет указать, на каком уровне риска пользователя вы хотите, чтобы оповещение было создано. Электронная почта будет создана, когда уровень риска пользователя достигнет указанного вами; однако, вы не будете получать новых пользователей, подверженных риску обнаруженных оповещений по электронной почте для этого пользователя после того, как они перейдут на этот уровень риска пользователя. Например, если вы установите политику для оповещения о среднем риске пользователя и пользователь Джон переходит к среднему риску, вы получите пользователей, подвергающихся риску обнаруженной электронной почты для Джона. Тем не менее, вы не получите второго пользователя, подверженного риску обнаруженного оповещения, если Джон переходит к высокому риску или имеет дополнительные обнаружения риска.

![сообщение электронной почты об обнаружении пользователей в группе риска;](./media/howto-identity-protection-configure-notifications/01.png)

### <a name="configure-users-at-risk-detected-alerts"></a>Настройка пользователей, подвергающихся риску обнаруженных предупреждений

Как администратор вы можете настроить:

- **Уровень риска пользователя, который запускает генерацию этого письма** - по умолчанию уровень риска настроен на "высокий" риск.
- **Получателей.** По умолчанию это письмо отправляется всем глобальным администраторам. Глобальные администраторы также могут добавить в список получателей других глобальных администраторов, администраторов безопасности и читателей сведений о безопасности.
   - Дополнительно вы можете **добавить дополнительные электронные письма для получения уведомлений,** что эта функция является предварительным просмотром, и пользователи должны иметь соответствующие разрешения для просмотра связанных отчетов на портале Azure.

Настройка пользователей, подвергающихся риску, на **портале Azure** под управлением **пользователя Azure Active Directory** > **Security** > **Identity Protection,** > **подверженного риску обнаружения множеству.**

## <a name="weekly-digest-email"></a>сообщение электронной почты еженедельного дайджеста.

Еженедельный дайджест электронной почты содержит резюме новых обнаружений риска.  
Сюда входят:

- Пользователи, подверженные риску
- подозрительные действия;
- обнаруженные уязвимости;
- ссылки на связанные отчеты в службе защиты идентификации.

![сообщение электронной почты еженедельного дайджеста.](./media/howto-identity-protection-configure-notifications/400.png)

По умолчанию получатели включают все глобальные админы. Глобальные администраторы также могут добавить в список получателей других глобальных администраторов, администраторов безопасности и читателей сведений о безопасности.

### <a name="configure-weekly-digest-email"></a>Настройка еженедельнодайд дайджест электронной почты

Как администратор, вы можете переключить отправку еженедельного дайджеста электронной почты или выключается и выбрать пользователей, назначенных для получения электронной почты.

Нанастройка еженедельной электронной почты дайджеста на **портале Azure** под**еженедельным дайджетом** **Azure Active Directory** > **Security** > **Identity Identity.** >

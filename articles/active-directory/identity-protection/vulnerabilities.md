# <a name="vulnerabilities-detected-by-azure-active-directory-identity-protection"></a>Уязвимости, обнаруживаемые защитой идентификации Azure Active Directory

Уязвимости — это слабые стороны в среде, которая может быть использована злоумышленником. Мы рекомендуем администраторам устранить эти уязвимости, чтобы повысить уровень защищенности своей организации.

![Уязвимости, о которых сообщает защита идентификации](./media/vulnerabilities/identity-protection-vulnerabilities.png)

В следующих разделах представлен обзор уязвимостей, о которых сообщает защита идентификации.

## <a name="multi-factor-authentication-registration-not-configured"></a>Регистрация многофакторной проверки подлинности, не настроена

Эта уязвимость способствует оценки развертывания многофакторной идентификации Azure в вашей организации.

Для просмотра счетчик для пользователей, которые не зарегистрированы для многофакторной проверки Подлинности, нажмите кнопку на уязвимости и вы будете перенаправлены Статистика в оценке безопасности удостоверений.

Многофакторная Идентификация Azure предоставляет дополнительный уровень безопасности для проверки подлинности пользователя. Она помогает защитить доступ к данным и приложениям, при этом не усложняя процесс входа пользователя в систему. Многофакторная Идентификация Azure предоставляет удобные проверки вариантах, например:

* Телефонный звонок
* Текстовое сообщение
* Уведомление от мобильного приложения
* Код проверки OTP

Мы рекомендуем настроить обязательную проверку с помощью Многофакторной идентификации Azure при входе пользователей. Multi-Factor Authentication играет основную роль в основанных на рисках политиках условного доступа в защите идентификации.

Дополнительные сведения см. в статье [Что такое Многофакторная идентификация Azure?](../authentication/multi-factor-authentication.md)

## <a name="unmanaged-cloud-apps"></a>Неуправляемые облачные приложения

Благодаря этой уязвимости можно выявить неуправляемые облачные приложения в организации.

ИТ-специалисты часто не знают о всех облачных приложений в своей организации. Легко понять, почему администраторов беспокоят несанкционированный доступ к корпоративным данным, возможная утечка данных и другие угрозы безопасности.

Мы рекомендуем развернуть Cloud Discovery обнаруживать неуправляемые облачные приложения и управлять ими с помощью Azure Active Directory.

Дополнительные сведения см. в статье [Cloud Discovery](/cloud-app-security/set-up-cloud-discovery).

## <a name="security-alerts-from-privileged-identity-management"></a>Оповещения системы безопасности в рамках управления привилегированными пользователями

Эта уязвимость способствует обнаружению и устранению оповещений о привилегированных удостоверениях в организации.  

Чтобы пользователи могли выполнять привилегированные операции, организациям необходимо предоставлять им временный или постоянный привилегированный доступ к Azure AD, ресурсам Azure или Office 365, а также другим приложениям SaaS. С каждым из привилегированных пользователей повышается вероятность атаки организации. Эта уязвимость позволяет выявить пользователей, которым не нужен привилегированный доступ, и предпринять соответствующие действия по уменьшению или исключению представляемого ими риска.

Мы рекомендуем организациям использовать Azure AD Privileged Identity Management для управления, контролировать и отслеживать привилегированных пользователей в Azure AD, а также другие Microsoft online services, например Office 365 или Microsoft Intune.

Дополнительные сведения см. в статье [об Azure Active Directory Privileged Identity Management](../privileged-identity-management/pim-configure.md).

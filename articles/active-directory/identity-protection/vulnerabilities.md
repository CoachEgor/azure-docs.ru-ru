# <a name="vulnerabilities-detected-by-azure-active-directory-identity-protection"></a>Уязвимости, обнаруживаемые защитой идентификации Azure Active Directory

Уязвимости представляют собой слабые места в среде, которые могут использовать злоумышленник. Мы рекомендуем администраторам устранить эти уязвимости, чтобы повысить уровень безопасности своей организации.

![Уязвимости, о которых сообщила Защита идентификации](./media/vulnerabilities/identity-protection-vulnerabilities.png)

В следующих разделах представлен обзор уязвимостей, о которых сообщает защита идентификации.

## <a name="multi-factor-authentication-registration-not-configured"></a>Регистрация многофакторной проверки подлинности не настроена

Эта уязвимость помогает оценить развертывание службы многофакторной идентификации Azure в Организации.

Многофакторная идентификация Azure обеспечивает второй уровень безопасности для проверки подлинности пользователей. Она помогает защитить доступ к данным и приложениям, при этом не усложняя процесс входа пользователя в систему. Многофакторная идентификация Azure позволяет легко использовать такие параметры проверки, как:

* Телефонный звонок
* SMS
* Уведомление мобильного приложения
* Код проверки OTP

Мы рекомендуем настроить обязательную проверку с помощью Многофакторной идентификации Azure при входе пользователей. Многофакторная идентификация играет ключевую роль в политиках условного доступа на основе рисков, доступных через защиту идентификации.

Дополнительные сведения см. в статье [Что такое Многофакторная идентификация Azure?](../authentication/multi-factor-authentication.md)

## <a name="unmanaged-cloud-apps"></a>Неуправляемые облачные приложения

Благодаря этой уязвимости можно выявить неуправляемые облачные приложения в организации.

ИТ-специалисты часто не знают обо всех облачных приложениях в Организации. Легко понять, почему администраторов беспокоят несанкционированный доступ к корпоративным данным, возможная утечка данных и другие угрозы безопасности.

Рекомендуется развернуть Cloud Discovery, чтобы обнаружить неуправляемые облачные приложения и управлять этими приложениями с помощью Azure Active Directory.

Дополнительные сведения см. в статье [Cloud Discovery](/cloud-app-security/set-up-cloud-discovery).

## <a name="security-alerts-from-privileged-identity-management"></a>Предупреждения системы безопасности из службы управления привилегированными пользователями

Эта уязвимость способствует обнаружению и устранению оповещений о привилегированных удостоверениях в организации.  

Чтобы пользователи могли выполнять привилегированные операции, организациям необходимо предоставлять им временный или постоянный привилегированный доступ к Azure AD, ресурсам Azure или Office 365, а также другим приложениям SaaS. С каждым из привилегированных пользователей повышается вероятность атаки организации. Эта уязвимость позволяет выявить пользователей, которым не нужен привилегированный доступ, и предпринять соответствующие действия по уменьшению или исключению представляемого ими риска.

Мы рекомендуем организациям использовать Azure AD Privileged Identity Management для управления и мониторинга привилегированных удостоверений в Azure AD, а также для других веб-службы Майкрософт, таких как Office 365 или Microsoft Intune.

Дополнительные сведения см. в статье [об Azure Active Directory Privileged Identity Management](../privileged-identity-management/pim-configure.md).

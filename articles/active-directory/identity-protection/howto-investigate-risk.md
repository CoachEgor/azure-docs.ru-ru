# <a name="how-to-investigate-risk"></a>Как Исследование рисков

С помощью отчетов о рисках входа, рискованных пользователей и обнаружении рисков можно исследовать и анализировать риски в вашей среде. Возможность фильтрации и сортировки рискованных входов в систему, пользователей и обнаружений позволяет лучше понять потенциальное вторжение в Организации. 

## <a name="risky-users-report"></a>Отчет о пользователях, выполняющих рискованные действия

Информация, доступная в отчете о пользователях, выполняющих рискованные действия, поможет вам ответить на следующие вопросы:

- Какие пользователи связаны с повышенным риском?
- Состояние каких пользователей можно исправить?

Первым шагом к получению этого отчета является раздел **Исследовать** на странице безопасности.

![Отчет о пользователях, выполняющих рискованные действия](./media/howto-investigate-risky-users-signins/01.png)

Отчеты о пользователях, выполняющих рискованные действия, отображают по умолчанию.

- Название
- Состояние риска
- Уровень риска
- Сведения о риске
- Последнее обновление риска
- Тип
- Status

![Отчет о пользователях, выполняющих рискованные действия](./media/howto-investigate-risky-users-signins/03.png)

Представление списка можно настроить, щелкнув **Столбцы** на панели инструментов.

![Отчет о пользователях, выполняющих рискованные действия](./media/howto-investigate-risky-users-signins/04.png)

В диалоговом окне столбцов вы можете добавить или удалить отображаемые поля.

Щелкнув элемент в представлении списка, вы получите дополнительные сведения о нем в горизонтальном представлении.

![Отчет о пользователях, выполняющих рискованные действия](./media/howto-investigate-risky-users-signins/05.png)

В представлении показаны такие сведения:

- Общие сведения
- Недавние рискованные входы в систему
- Обнаружения рисков, не связанных с входом
- Журнал рисков

Кроме того, вы можете:

![Отчет о пользователях, выполняющих рискованные действия](./media/howto-investigate-risky-users-signins/08.png)

- использовать ярлык "Показать все события входа", чтобы увидеть отчет о событиях входа для этого пользователя;
- использовать "Показать все рискованные входы в систему", чтобы увидеть все события входа для этого пользователя, которые были отмечены как рискованные;
- сбросить пароль пользователя, если вы считаете, что удостоверение пользователя было скомпрометировано;
- Отклонить риск для пользователя, если вы думаете, что активные определения рисков пользователя являются ложными срабатываниями. Дополнительные сведения см. в статье [обратная связь по обнаружению рисков в Защита идентификации Azure AD](howto-provide-risk-event-feedback.md).

### <a name="filter-risky-users"></a>Фильтрация пользователей, выполняющих рискованные действия

Для сужения результатов до нужного уровня вы можете отфильтровать данные о пользователях, выполняющих рискованные действия, используя следующие поля по умолчанию:

- Название
- Имя пользователя
- Состояние риска
- Уровень риска
- Тип
- Status

![Отчет о пользователях, выполняющих рискованные действия](./media/howto-investigate-risky-users-signins/06.png)

Фильтр **Имя** позволяет указать интересующее вас имя пользователя или имя участника-пользователя (UPN).

В фильтре **Состояние риска** можете выбрать:

- В группе риска
- Исправленные
- Отклонено

В фильтре **Уровень риска** можете выбрать:

- Высокий
- Среднее
- Низкий

В фильтре **Тип** можете выбрать:

- Гость
- Член

В фильтре **Состояние** можете выбрать:

- Удален
- Активен

Детальную информацию о пользователях, выполняющих рискованные действия, вы можете получить в ящике "Сведения".

- пользователя
- Приложение
- состояние входа.
- Состояние риска
- Уровень риска (агрегированный)
- Уровень риска (реальное время)

![Отчет о входах, представляющих риск](./media/howto-investigate-risky-users-signins/14.png)

В фильтре по **состоянию входа** можно выбрать:

- Все
- Success
- Сбой

В фильтре **Состояние риска** можете выбрать:

- В группе риска
- Подтвержденные скомпрометированные

- Отклонено

- Высокий
- Среднее
- Низкий

- Высокий
- Среднее
- Низкий

Фильтр **условного доступа** позволяет выбрать:

- Все
- Неприменимо

Фильтр **дат** позволяет определить интервал времени для возвращаемых данных.

## <a name="risk-detections-report"></a>Отчет об обнаружении рисков

Отчет об обнаружении рисков позволяет получить представление о каждом обнаружении риска для защиты идентификации в вашем клиенте.


Отчет об обнаружении рисков имеет представление по умолчанию, которое показывает:

- Дата

- пользователя

- IP-адрес

- Местоположение

- Определяемый тип

- Состояние риска

- Уровень риска

- Идентификатор запроса
 

Представление списка можно настроить, щелкнув **Столбцы** на панели инструментов. В диалоговом окне столбцов вы можете добавить или удалить отображаемые поля.

Щелкнув элемент в представлении списка, вы получите дополнительные сведения о нем в горизонтальном представлении.


В подробном представлении отображаются дополнительные сведения об обнаружении рисков.

Подробное представление также содержит ссылки на

- Просмотр отчета о рисках для пользователя
- Просмотр входов пользователей
- Просмотр рискованных входов пользователя
- Просмотреть связанный рискованный вход
- Просмотр обнаружений рисков пользователя


### <a name="filter-risk-detections"></a>Фильтрация обнаружений рисков

Чтобы сократить количество сообщаемых данных до уровня, который подходит для вас, можно отфильтровать данные обнаружения рисков, используя следующие поля по умолчанию:

- Время обнаружения
- Определяемый тип
- Состояние риска
- Уровень риска
- пользователя
- Имя пользователя
- IP-адрес
- Местоположение
- Время обнаружения
- Действия 
- Source
- Тип издателя токена
- Идентификатор запроса
- Идентификатор корреляции


Фильтр **времени обнаружения** позволяет определить временные рамки для возвращаемых данных. Этот фильтр позволяет выбрать:
- Последние 90 дней
- За последние 30 дней
- За последние 7 дней
- За последние 24 часа
- Настраиваемый интервал

Фильтр **типов обнаружения** позволяет указать тип обнаружения рисков (например, незнакомые свойства входа).

В фильтре **Состояние риска** можете выбрать:

- В группе риска
- Подтвержденные скомпрометированные
- Подтвержденные безопасные
- Отклонено
- Исправленные

В фильтре **Уровень риска** можете выбрать:

- Высокий
- Среднее
- Низкий

Фильтр **пользователей** позволяет указать имя или идентификатор объекта нужного пользователя.

Фильтр **username (имя пользователя** ) позволяет указать имя участника-пользователя (UPN) пользователя.

Фильтр **времени обнаружения** позволяет указать, было ли обнаружение установлено в режиме реального времени или вне сети. Примечание. Вы можете выполнять вход в систему с помощью политик риска входа в реальном времени. 

Фильтр **действий** позволяет указать, было ли обнаружение связано с входом (например, АНОНИМНЫМ IP-адресом) или пользователем (например, с утечкой учетных данных).

Фильтр **источников** позволяет указать источник обнаружения рисков (например, Azure AD или Microsoft Cloud App Security). 

Фильтр **типа издателя маркера** позволяет указать, где был выдан маркер (например, Azure AD или службы федерации AD).


### <a name="download-risk-detections-data"></a>Загрузка данных обнаружения рисков

Данные обнаружения рисков можно скачать, если вы хотите работать с ним за пределами портал Azure. При нажатии кнопки скачать создается CSV-файл с самыми последними записями 5 000.

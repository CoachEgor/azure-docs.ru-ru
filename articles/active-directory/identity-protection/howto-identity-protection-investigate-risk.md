# <a name="how-to-investigate-risk"></a>Как: Исследовать риск

Identity Protection предоставляет организациям три отчета, которые они могут использовать для расследования рисков идентификации в своей среде. Эти отчеты являются **рискованными пользователями,** **рискованными ввозами**и **обнаружением рисков.** Исследование событий является ключом к лучшему пониманию и выявлению любых слабых мест в вашей стратегии безопасности.

Все три отчета позволяют загружать события в . Формат CSV для дальнейшего анализа за пределами портала Azure. Рискованные пользователи и рискованные отчеты о входах позволяют загрузить последние 2500 записей, в то время как отчет об обнаружении рисков позволяет загрузить последние 5000 записей.

Организации могут воспользоваться интеграцией Microsoft Graph API для агрегирования данных с другими источниками, к которых они могут иметь доступ как к организации.

Эти три отчета можно найти на **портале** > Azure**Active Directory** > **Security.**

## <a name="navigating-the-reports"></a>Навигация по отчетам

Каждый отчет запускается со списком всех обнаружений за период, показанный в верхней части отчета. Каждый отчет позволяет добавить или удалить столбцы на основе предпочтений администратора. Администраторы могут загрузить данные в . CSV или . Формат JSON. Отчеты могут быть отфильтрошены с помощью фильтров в верхней части отчета.

Выбор отдельных записей может включать дополнительные записи в верхней части отчета, такие как возможность подтвердить вход как скомпрометированный или безопасный, подтвердить пользователя как скомпрометированного или уволить риск пользователя.

Выбор отдельных записей расширяет окно деталей под обнаружением. Представление данных позволяет администраторам исследовать и выполнять действия по каждому обнаружению.

## <a name="risky-users"></a>Пользователи, выполняющие рискованные действия

С информацией, представленной отчетом о рискованных пользователях, администраторы могут найти:

- Какие пользователи подвергаются риску, были исправлены или были уволены?
- Подробная информация об обнаружении
- История всех рискованных всхвай
- Журнал рисков
 
Администраторы могут принять решение об этих событиях. Администраторы могут выбрать:

- Сбросить пароль пользователя
- Подтверждение компромата пользователей
- Параметр "Закрыть уведомление о риске для пользователя"
- Блокировка пользователя от входиного
- Исследуйте дальнейшее использование Azure ATP

## <a name="risky-sign-ins"></a>Вход, представляющий риск

Отчет о рискованных входе содержит фильтруемые данные за последние 30 дней (1 месяц).

С информацией, представленной в отчете о рискованных входе, администраторы могут найти:

- Какие ввои классифицируются как подверженные риску, подтвержденные скомпрометированы, подтверждены безопасными, уволенными или исправленными.
- Уровни риска в реальном времени и совокупные риски, связанные с попытками входной связи.
- Типы обнаружения срабатывают
- Применяются политики условного доступа
- Подробности МИД
- Сведения об устройстве
- Сведения о приложении
- сведения о расположении;

Администраторы могут принять решение об этих событиях. Администраторы могут выбрать:

- Подтверждение компромисса
- Подтвердить вхасав в сейф

## <a name="risk-detections"></a>Обнаружение рисков

Отчет об обнаружении рисков содержит фильтруемые данные за последние 90 дней (3 месяца).

С помощью информации, представленной отчетом об обнаружении рисков, администраторы могут найти:

- Информация о каждом обнаружении риска, включая тип.
- Другие риски, вызванные в то же время
- Местоположение попытки регистрации
- Ссылка на более подробную информацию из Microsoft Cloud App Security (MCAS).

Администраторы могут затем вернуться к отчету о риске пользователя или входе в систему, чтобы принять меры на основе собранной информации.

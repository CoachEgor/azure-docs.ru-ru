# <a name="azure-active-directory-identity-protection-risk-detections-reference"></a>Справочник по обнаружению рисков Защита идентификации Azure Active Directory

Подавляющее большинство нарушений безопасности — это случаи, когда злоумышленники получают доступ к среде за счет кражи удостоверения пользователя. Обнаружение скомпрометированных удостоверений — непростая задача. Azure Active Directory использует адаптивные алгоритмы машинного обучения и эвристические методы, чтобы обнаруживать подозрительные действия, связанные с вашими учетными записями. Каждое обнаруженное подозрительные действие хранится в записи, называемой обнаружением рисков.

## <a name="anonymous-ip-address"></a>Анонимный IP-адрес

**Тип обнаружения**. В режиме реального времени  
**Старое имя**. Входы в систему с анонимного IP-адреса

Этот тип обнаружения рисков указывает на входы с анонимного IP-адреса (например, Tor Browser, анонимизер VPN).
Эти IP-адреса обычно используются субъектами, которые хотят скрыть свои данные телеметрии входа (IP-адрес, местоположение, устройство и другое) для потенциально злого намерения.

## <a name="atypical-travel"></a>Необычный переход

**Тип обнаружения**. Не в сети  
**Старое имя**. Невозможно переместиться в нетипичные расположения

Этот тип обнаружения рисков определяет два входа, исходящие из географически нераспределенных расположений, где по крайней мере одно из расположений может быть нетипичным для пользователя с учетом предыдущего поведения. Помимо других факторов, этот алгоритм машинного обучения учитывает промежуток времени между двумя входами, и время перехода пользователя из одного расположения в другое. Это значит, что другой пользователь использует те же учетные данные.

Этот алгоритм игнорирует очевидные ложные срабатывания, такие как VPN и расположения, которые постоянно используют другие пользователи в организации, и позволяет получить результаты, соответствующие невозможным условиям перемещения. В системе предусмотрен первоначальный период обучения — 14 дней или 10 входов в систему, в течение которых она изучает поведение нового пользователя при входе.

## <a name="leaked-credentials"></a>Утечка учетных данных

**Тип обнаружения**. Не в сети  
**Старое имя**. Пользователи с краденными учетными данными

Этот тип обнаружения рисков указывает, что допустимые учетные данные пользователя были утечки.
Киберпреступники зачастую предоставляют общий доступ к украденным ими действительным учетным данным законных пользователей. Обычно они публикуют их в теневом Интернете или сайтах для киберпреступников, либо продают их на теневом рынке. Служба Майкрософт, занимающаяся отслеживанием утечек учетных данных, узнает такие пары имен пользователя и паролей, обращаясь к таким общедоступным и теневым веб-сайтам, а также сотрудничая со следующими сторонами:

- исследователи;
- правоохранительные органы;
- группы безопасности в корпорации Майкрософт;
- другие доверенные источники.

Когда служба получает учетные данные пользователя из темного Интернета, вставляет сайты или указанные выше источники, они проверяются по текущим действительным учетным данным пользователей Azure AD, чтобы найти действительные совпадения.

## <a name="malware-linked-ip-address"></a>IP-адрес, который помечен как вредоносный

**Тип обнаружения**. Не в сети  
**Старое имя**. Входы в систему с зараженных устройств

Этот тип обнаружения рисков указывает на попытки входа с IP-адресов, зараженных вредоносными программами, которые активно взаимодействуют с сервером-Bot. Определение этих событий осуществляется путем сопоставления IP-адресов устройства пользователя с IP-адресами, которые использовались для связи с таким сервером, в то время как он был включен.

## <a name="unfamiliar-sign-in-properties"></a>Неизвестные свойства входа

**Тип обнаружения**. В режиме реального времени  
**Старое имя**. Входы в систему из необычного расположения

Этот тип обнаружения рисков учитывает историю входа в систему (IP, Широта, Долгота и ASN) для поиска аномальных входов в систему. Система хранит сведения о предыдущих расположениях входа пользователя и учитывает их в качестве знакомых. Обнаружение рисков активируется, когда вход выполняется из расположения, которое еще не находится в списке привычных расположений. Недавно созданные пользователи будут находиться в режиме обучения в течение определенного периода времени, в течение которого незнакомые свойства входа в систему будут отключены, а наши алгоритмы узнают о поведении пользователя. Длительность режима обучения является динамической и зависит от того, сколько времени использует алгоритм для сбора достаточной информации о пользовательских шаблонах входа. Минимальная длительность составляет пять дней. Пользователь может вернуться в режим обучения после длительного периода бездействия. Система также игнорирует вход с помощью знакомых устройств и из расположений, географически близких к знакомому. 

Мы также запускаем это обнаружение для базовой аутентификации (или устаревших протоколов). Поскольку эти протоколы не имеют современных свойств, например идентификатора клиента, существует ограниченная телеметрия для сокращения количества ложноположительных результатов. Мы рекомендуем нашим клиентам перейти на современные способы проверки подлинности.

## <a name="azure-ad-threat-intelligence"></a>Аналитика угроз Azure AD

**Тип обнаружения**. Не в сети <br>
**Старое имя**. Это обнаружение будет отображаться в устаревших отчетах защита идентификации Azure AD (пользователи, помеченные для риска, обнаружения рисков) как "Пользователи с потерянными учетными данными"

Этот тип обнаружения рисков указывает на активность пользователей, необычных для данного пользователя, или согласуется с известными шаблонами атак, основанными на внутренних и внешних источниках аналитики угроз Майкрософт.

## <a name="admin-confirmed-user-compromised"></a>Подтвержденная администратором компрометация пользователя

**Тип обнаружения**. Не в сети <br>
Это обнаружение означает, что администратор выбрал параметр "подтвердить безопасность пользователя" в пользовательском интерфейсе рискованных пользователей или с помощью API Рискюсерс. Чтобы узнать, какое из администраторов подтвердило безопасность этого пользователя, проверьте журнал рисков пользователя (через пользовательский интерфейс или API).

## <a name="malicious-ip-address"></a>Вредоносный IP-адрес

**Тип обнаружения**. Не в сети <br>
Это обнаружение означает вход с вредоносного IP-адреса. IP-адрес считается вредоносным, исходя из следующих:
-   Высокие показатели сбоев (из-за недопустимых учетных данных, полученных с IP-адреса)
-   Другие источники репутации IP-адресов

## <a name="additional-risk-detected"></a>Обнаружен дополнительный риск

**Тип обнаружения**. В режиме реального времени или вне сети <br>
Это обнаружение означает, что обнаружено одно из приведенных выше обновлений уровня "Премиум". Поскольку обнаружения уровня "Премиум" видны только Azure AD Premiumным клиентам P2, они называются "обнаружены дополнительные риски" для клиентов, не относящихся к P2.

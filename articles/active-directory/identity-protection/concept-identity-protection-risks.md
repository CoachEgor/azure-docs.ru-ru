# <a name="what-is-risk"></a>Что такое риск?

Обнаружение рисков в Azure AD Identity Protection включает любые выявленные подозрительные действия, связанные с учетными записями пользователей в каталоге.

Identity Protection предоставляет организациям доступ к мощным ресурсам, чтобы быстро видеть и реагировать на эти подозрительные действия. 

![Обзор безопасности, показывающий рискованных пользователей и вхындерктворы](./media/concept-identity-protection-risks/identity-protection-security-overview.png)

## <a name="risk-types-and-detection"></a>Типы рисков и обнаружение

Существует два типа риска **пользователя** и **ввоза и** два типа обнаружения или расчета в **режиме реального времени** и в **автономном режиме.**

### <a name="user-risk"></a>Риск пользователя

Риск пользователя представляет вероятность того, что данная личность или учетная запись скомпрометирована. 

Эти риски рассчитываются в автономном режиме с использованием внутренних и внешних источников разведки угроз корпорации Майкрософт, включая исследователей безопасности, сотрудников правоохранительных органов, групп безопасности корпорации Майкрософт и других надежных источников.

| Обнаружение рисков | Описание |
| --- | --- |
| Утерянные учетные данные | Этот тип обнаружения рисков указывает на утечку действительных учетных данных пользователя. Киберпреступники зачастую предоставляют общий доступ к украденным ими действительным учетным данным законных пользователей. Этот обмен, как правило, делается путем размещения публично на темной паутине, вставить сайты, или путем торговли и продажи учетных данных на черном рынке. Когда служба утечки учетных данных Майкрософт приобретает учетные данные пользователей из темных веб-сайтов, сайтов пасты или других источников, они проверяются на соответствие текущим действительным учетным данным пользователей Azure AD для поиска действительных совпадений. |
| Аналитика угроз Azure AD | Этот тип обнаружения рисков указывает на активность пользователя, которая необычна для данного пользователя или согласуется с известными шаблонами атак на основе внутренних и внешних источников разведки угроз корпорации Майкрософт. |

### <a name="sign-in-risk"></a>Риск при входе

Риск ввхавания представляет вероятность того, что данный запрос на проверку подлинности не авторизован владельцем удостоверения личности. 

Эти риски могут быть рассчитаны в режиме реального времени или рассчитаны в автономном режиме с использованием внутренних и внешних источников разведки угроз корпорации Майкрософт, включая исследователей безопасности, сотрудников правоохранительных органов, групп безопасности корпорации Майкрософт и других надежных источников.

| Обнаружение рисков | Тип обнаружения | Описание |
| --- | --- | --- |
| Анонимный IP-адрес | В режиме реального времени | Этот тип обнаружения рисков указывает на встыки с анонимного IP-адреса (например, браузер Tor или анонимный VPN). Эти IP-адреса обычно используются субъектами, которые хотят скрыть свои данные телеметрии входа (IP-адрес, местоположение, устройство и другое) для потенциально злого намерения. |
| Необычный переход | Вне сети | Этот тип обнаружения рисков идентифицирует два всходного, происходящих из географически удаленных мест, где по крайней мере одно из мест также может быть нетипичным для пользователя, учитывая прошлое поведение. Помимо других факторов, этот алгоритм машинного обучения учитывает промежуток времени между двумя входами, и время перехода пользователя из одного расположения в другое. Это значит, что другой пользователь использует те же учетные данные. <br><br> Этот алгоритм игнорирует очевидные ложные срабатывания, такие как VPN и расположения, которые постоянно используют другие пользователи в организации, и позволяет получить результаты, соответствующие невозможным условиям перемещения. Система имеет начальный период обучения из самых ранних из 14 дней или 10 логинов, в течение которых он узнает поведение входа нового пользователя. |
| IP-адрес, который помечен как вредоносный | Вне сети | Этот тип обнаружения риска указывает на встыковые с IP-адресов, зараженных вредоносными программами, которые, как известно, активно общаются с сервером ботов. Это обнаружение определяется путем сопоставления IP-адресов устройства пользователя с IP-адресами, которые были в контакте с сервером бота в то время как бот-сервер был активен. |
| Неизвестные свойства входа | В режиме реального времени | Этот тип обнаружения риска рассматривает прошлые вписывания в историю (IP, Широта / Longitude и ASN) для поиска аномальных врезов. Система хранит информацию о предыдущих местоположениях, используемых пользователем, и учитывает эти "знакомые" места. Обнаружение риска срабатывает при входе в систему из места, которого еще нет в списке знакомых местоположений. Вновь созданные пользователи будут находиться в "режиме обучения" в течение определенного периода времени, в течение которого незнакомые обнаружения рисков вхданных свойств будут отключены, пока наши алгоритмы узнают поведение пользователя. Длительность режима обучения является динамической и зависит от того, сколько времени требуется алгоритму для сбора достаточного количества информации о шаблонах вхождения пользователя. Минимальная продолжительность составляет пять дней. Пользователь может вернуться в режим обучения после длительного периода бездействия. Система также игнорирует вход с помощью знакомых устройств и из расположений, географически близких к знакомому. <br><br> Мы также запускаем это обнаружение для базовой аутентификации (или устаревших протоколов). Поскольку эти протоколы не имеют современных свойств, например идентификатора клиента, существует ограниченная телеметрия для сокращения количества ложноположительных результатов. Мы рекомендуем нашим клиентам перейти на современные способы проверки подлинности. |
| Админ подтвердил, что пользователь скомпрометирован | Вне сети | Это обнаружение указывает на то, что админ выбрал "Подтвердить, что пользователь скомпрометирован" в пользовательском интерфейсе рискованных пользователей или с помощью рискованного API пользователей. Чтобы узнать, какой админ подтвердил, что этот пользователь скомпрометирован, проверьте историю риска пользователя (через пользовательский интерфейс или API). |
| Вредоносный IP-адрес | Вне сети | Это обнаружение указывает на ввоза с вредоносного IP-адреса. IP-адрес считается вредоносным на основе высоких показателей сбоев из-за недействительных учетных данных, полученных от IP-адреса или других источников репутации IP. |
| Подозрительные правила для папки "Входящие" | Вне сети | Это обнаружение обнаружено [Microsoft Cloud App Security (MCAS).](/cloud-app-security/anomaly-detection-policy#suspicious-inbox-manipulation-rules) Эта политика профилирует среду и активирует оповещения при обнаружении подозрительных правил, которые удаляют или перемещают сообщения либо папки в папке "Входящие" пользователя. Это может означать, что учетная запись пользователя скомпрометирована, что сообщения намеренно скрыты, и что почтовый ящик используется для распространения спама или вредоносных программ в вашей организации. |
| невозможное перемещение. | Вне сети | Это обнаружение обнаружено [Microsoft Cloud App Security (MCAS).](/cloud-app-security/anomaly-detection-policy#impossible-travel) Это обнаружение идентифицирует два действия пользователя (в одном или нескольких сеансах), выполняемых в географически отдаленных расположениях в течение периода времени, недостаточного для того, чтобы переместиться из первого расположения во второе. Оно указывает на то, что другой пользователь использует те же учетные данные. |

### <a name="other-risk-detections"></a>Другие обнаружения рисков

| Обнаружение рисков | Тип обнаружения | Описание |
| --- | --- | --- |
| Обнаружен дополнительный риск | В режиме реального времени или в автономном режиме | Это обнаружение указывает на то, что один из вышеперечисленных обнаружений премиум-класса был обнаружен. Поскольку обнаружение премий отображается только для клиентов Azure AD Premium P2, они называются «обнаруженным дополнительным риском» для клиентов без лицензий Azure AD Premium P2. |

# <a name="identity-protection-policies"></a>Политики Защиты идентификации

Защита идентификационных данных Active Directory Azure включает три политики по умолчанию, которые администраторы могут выбрать для включения. Эти политики включают ограниченную настройку, но применимы к большинству организаций. Все политики позволяют исключать пользователей, таких как учетные [записи администратора экстренного доступа или взломщика.](../users-groups-roles/directory-emergency-access.md)

![Политики Защиты идентификации](./media/concept-identity-protection-policies/identity-protection-policies.png)

## <a name="azure-mfa-registration-policy"></a>Политика регистрации Мидуза Azure

Защита идентификации может помочь организациям внедрить многофакторную аутентификацию Azure (MFA) с помощью политики условного доступа, требующей регистрации при вхвошне. Включение этой политики является отличным способом для обеспечения новых пользователей в вашей организации зарегистрировались для МИД в первый день. Многофакторная аутентификация является одним из методов самовосстановления событий риска в рамках Identity Protection. Самовосстановление позволяет пользователям самостоятельно принимать меры по сокращению объема вызовов службы поддержки.

Более подробную информацию о многофакторной аутентификации Azure можно найти в статье [«Как это работает: Многофакторная аутентификация Azure](../authentication/concept-mfa-howitworks.md)».

## <a name="sign-in-risk-policy"></a>Политика риска входа

Identity Protection анализирует сигналы от каждого вхвнутри, как в режиме реального времени, так и в автономном режиме, и вычисляет оценку риска на основе вероятности того, что входе не было выполнено пользователем. Администраторы могут принять решение, основываясь на этом сигнале оценки риска для обеспечения соблюдения организационных требований. Администраторы могут блокировать доступ, разрешить доступ или разрешить доступ, но требуют многофакторной аутентификации.

При обнаружении риска пользователи могут выполнять многофакторную аутентификацию для самостоятельного устранения и закрытия рискованного события, связанного с ввозами, чтобы предотвратить ненужный шум для администраторов.

> [!NOTE] 
> Пользователи должны предварительно зарегистрироваться для azure Multi-Factor Authentication, прежде чем запустить политику риска ввне.

### <a name="custom-conditional-access-policy"></a>Пользовательская политика условного доступа

Администраторы также могут создать пользовательскую политику условного доступа, включая риск вхластного в качестве условия назначения. Более подробную информацию об условном доступе можно найти в статье ["Что такое условный доступ?](../conditional-access/overview.md)

![Пользовательская политика риска входиного доступа](./media/concept-identity-protection-policies/identity-protection-custom-sign-in-policy.png)

## <a name="user-risk-policy"></a>Политика риска пользователя

Identity Protection может вычислить то, что он считает нормальным для поведения пользователя, и использовать его для базирования решений для их риска. Риск пользователя — это расчет вероятности того, что идентификация была скомпрометирована. Администраторы могут принять решение, основываясь на этом сигнале оценки риска для обеспечения соблюдения организационных требований. Администраторы могут заблокировать доступ, разрешить доступ или разрешить доступ, но требуют изменения пароля с помощью [сбросить пароля самообслуживания Azure AD.](../authentication/howto-sspr-deployment.md)

При обнаружении риска пользователи могут выполнить сбросить пароли самообслуживания для самостоятельного устранения и закрытия события риска пользователя, чтобы предотвратить ненужный шум для администраторов.

> [!NOTE] 
> Пользователи должны предварительно зарегистрироваться для сбросить пароля самообслуживания перед запуском политики риска пользователя.

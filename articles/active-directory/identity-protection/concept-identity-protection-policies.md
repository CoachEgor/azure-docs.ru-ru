# <a name="identity-protection-policies"></a>Политики Защиты идентификации

Защита идентификации Azure Active Directory включает три политики по умолчанию, которые администраторы могут выбрать для включения. Эти политики включают ограниченную настройку, но применимы для большинства организаций. Все политики позволяют исключить пользователей, например [учетные записи администратора для аварийного доступа или прерывания](../users-groups-roles/directory-emergency-access.md).

![Политики Защиты идентификации](./media/concept-identity-protection-policies/identity-protection-policies.png)

## <a name="azure-mfa-registration-policy"></a>Политика регистрации Azure MFA

Защита идентификации может помочь организациям развернуть многофакторную идентификацию Azure (MFA) с помощью политики условного доступа, которая требует регистрации при входе в систему. Включение этой политики — отличный способ убедиться, что новые пользователи в вашей организации зарегистрировались для MFA в первый день. Многофакторная идентификация является одним из методов самостоятельного исправления событий риска в рамках защиты идентификации. Самостоятельное исправление позволяет пользователям самостоятельно предпринимать меры для сокращения объема вызовов службы технической поддержки.

Дополнительные сведения о многофакторной идентификации Azure см. в статье [как это работает: многофакторная идентификация Azure](../authentication/concept-mfa-howitworks.md).

## <a name="sign-in-risk-policy"></a>Политика риска входа

Защита идентификации анализирует сигналы каждого входа, как в режиме реального времени, так и вне сети, и вычисляет оценку риска на основе вероятности, когда вход не выполнялся пользователем. Администраторы могут принять решение на основе этого сигнала оценки риска, чтобы обеспечить соблюдение требований Организации. Администраторы могут заблокировать доступ, разрешить доступ или разрешить доступ, но требовать многофакторную проверку подлинности.

При обнаружении риска пользователи могут выполнять многофакторную идентификацию для самостоятельного устранения неполадок и закрывать рискованные события входа, чтобы предотвратить ненужные помехи для администраторов.

> [!NOTE] 
> Прежде чем активировать политику риска входа, пользователи должны были зарегистрироваться ранее для использования многофакторной идентификации Azure.

### <a name="custom-conditional-access-policy"></a>Настраиваемая политика условного доступа

Администраторы также могут создать настраиваемую политику условного доступа, включая риск входа в качестве условия назначения. Дополнительные сведения об условном доступе можно найти в статье [что такое условный доступ?](../conditional-access/overview.md)

![Политика риска входа настраиваемого условного доступа](./media/concept-identity-protection-policies/identity-protection-custom-sign-in-policy.png)

## <a name="user-risk-policy"></a>Политика риска пользователя

С помощью защиты идентификации можно вычислить, что это нормально для поведения пользователя, и использовать его для принятия решений о своих рисках. Риск для пользователей — это вычисление вероятности компрометации удостоверения. Администраторы могут принять решение на основе этого сигнала оценки риска, чтобы обеспечить соблюдение требований Организации. Администраторы могут заблокировать доступ, разрешить доступ или разрешить доступ, но запрашивать изменение пароля с помощью [самостоятельного сброса пароля в Azure AD](../authentication/howto-sspr-deployment.md).

При обнаружении риска пользователи могут самостоятельно выполнять самостоятельное сброс пароля и закрывать событие риска пользователя, чтобы предотвратить ненужные помехи для администраторов.

> [!NOTE] 
> Прежде чем активировать политику риска пользователя, пользователи должны были зарегистрироваться ранее для самостоятельного сброса пароля.

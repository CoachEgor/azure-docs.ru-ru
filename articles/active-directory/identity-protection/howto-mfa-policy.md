# <a name="how-to-configure-the-azure-multi-factor-authentication-registration-policy"></a>Как Настройка политики регистрации многофакторной идентификации Azure

Защита идентификации Azure AD помогает управлять развертыванием регистрации многофакторной проверки подлинности (MFA), настроив политику условного доступа требуется регистрация в MFA независимо от того, какие приложения, вы входите. Эта статья объясняет, что политика может использоваться для и способах его настройки.

## <a name="what-is-the-azure-multi-factor-authentication-registration-policy"></a>Что такое политика регистрации многофакторной идентификации Azure?

Многофакторная Идентификация Azure предоставляет средства проверки, который вы используете больше, чем только имя пользователя и пароль. Он обеспечивает второй уровень безопасности для входа в систему. Чтобы пользователи могли отвечать на запросы многофакторной проверки Подлинности они должны зарегистрироваться для многофакторной идентификации Azure.

Рекомендуется, чтобы потребовать многофакторной идентификации Azure для входа в систему, так как он:

- обеспечивает строгую аутентификацию с помощью ряда простых параметров проверки;
- Играет ключевую роль в подготовке организации для защиты и восстановления из событий риска в службе защиты идентификации

Дополнительные сведения о многофакторной проверки Подлинности, см. в разделе [что такое многофакторная Идентификация Azure?](../authentication/howto-mfa-getstarted.md)

## <a name="how-do-i-access-the-registration-policy"></a>Как получить доступ к политике регистрации?

Политика регистрации с многофакторной проверкой подлинности указывается в разделе **Настройка** страницы [Azure AD Identity Protection](https://portal.azure.com/#blade/Microsoft_AAD_ProtectionCenter/IdentitySecurityDashboardMenuBlade/SignInPolicy).

![Политика MFA](./media/howto-mfa-policy/1014.png)

## <a name="policy-settings"></a>Параметры политики

При настройке политики регистрации многофакторной проверки Подлинности, необходимо внести следующие изменения конфигурации:

- Пользователи и группы, к которому применяется политика. Не забудьте исключить организации [учетные записи аварийного доступа](../users-groups-roles/directory-emergency-access.md).

    ![Пользователи и группы](./media/howto-mfa-policy/11.png)

- Элемент управления, который вы хотите применить - **регистрации требования MFA Azure**

    ![Доступ](./media/howto-mfa-policy/12.png)

- Принудительное применение политики должно быть присвоено **на**.

    ![Принудительное применение политики](./media/howto-mfa-policy/14.png)

- **Сохранить** политики

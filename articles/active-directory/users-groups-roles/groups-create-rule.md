# <a name="create-or-update-a-dynamic-group-in-azure-active-directory"></a>Создание или обновление динамической группы в Active Directory Azure

В Active Directory Azure (Azure AD) можно использовать правила для определения членства в группе на основе свойств пользователя или устройства. В этой статье рассказывается о том, как настроить правило для динамической группы на портале Azure.
Динамическое членство поддерживается для групп безопасности или групп Office 365. При применении правила членства в группе атрибуты пользователя и устройства оцениваются на соответствие правилу членства. При изменении атрибута для пользователя или устройства все динамические правила группы в организации обрабатываются для изменения членства. Пользователи и устройства добавляются или удаляются, если они отвечают условиям группы. Группы безопасности могут использоваться для устройств или пользователей, но группы Office 365 могут быть только группами пользователей.

## <a name="rule-builder-in-the-azure-portal"></a>Строитель правил на портале Azure

Azure AD обеспечивает создание правил для более быстрого создания и обновления важных правил. Строитель правила поддерживает конструкцию до 5 выражений. Строитель правил упрощает формирование правила с помощью нескольких простых выражений, однако его нельзя использовать для воспроизведения каждого правила. Если создатель правил не поддерживает правило, которое вы хотите создать, можно использовать текстовое поле.

Вот несколько примеров передовых правил или синтаксиса, для которых мы рекомендуем построить с помощью текстового поля:

- Правило с более чем пятью выражениями
- Правило Прямые отчеты
- Установка [приоритета оператора](groups-dynamic-membership.md#operator-precedence)
- [Правила со сложными выражениями](groups-dynamic-membership.md#rules-with-complex-expressions); Например`(user.proxyAddresses -any (_ -contains "contoso"))`

> [!NOTE]
> Строитель правил может быть не в состоянии отображать некоторые правила, построенные в текстовом поле. Вы можете увидеть сообщение, когда строитель правил не может отобразить правило. Строитель правил никоим образом не изменяет поддерживаемый синтаксис, проверку или обработку динамических правил группы.

![Добавление правила членства для динамической группы](./media/groups-create-rule/update-dynamic-group-rule.png)

На примерах синтаксиса, поддерживаемых свойств, операторов и значений для правила членства см. [Динамические правила членства для групп в Active Directory Azure.](groups-dynamic-membership.md)

## <a name="to-create-a-group-membership-rule"></a>Создание правила членства в группе

1. Вопием в [центре администратора АС](https://aad.portal.azure.com) ВС С узлом учетной записи, накоторуйтесь в глобальном администраторе, администраторе Intune или роли администратора пользователя в арендаторе.
1. Поиск и выбор **групп.**
1. Выберите **Все группы**, а затем — **Новая группа**.

   ![Выберите команду для добавления новой группы](./media/groups-create-rule/create-new-group-azure-active-directory.png)

1. На странице **Группы** введите имя и описание новой группы. Выберите **тип членства** для пользователей или устройств, а затем выберите **Добавление динамического запроса.** Строитель правил поддерживает до пяти выражений. Чтобы добавить более пяти выражений, необходимо использовать текстовое поле.

   ![Добавление правила членства для динамической группы](./media/groups-create-rule/add-dynamic-group-rule.png)

1. Чтобы увидеть пользовательские свойства расширения, доступные для вашего запроса на членство:
   1. Выберите **Пользовательские свойства расширения**
   1. Введите идентификатор приложения, а затем выберите **свойства обновления.**
1. После создания правила выберите **Сохранить**.
1. Выберите **Создать** на новой странице **группы** для создания группы.

Если введенное правило не является действительным, в уведомлении Azure на портале отображается объяснение того, почему правило не может быть обработано. Внимательно прочитайте его, чтобы понять, как исправить правило.

## <a name="to-update-an-existing-rule"></a>Обновление существующего правила

1. Вопием в [центре администратора АС](https://aad.portal.azure.com) ВС С узлом учетной записи, которая находится в глобальном администраторе, администраторе группы, администраторе Intune или роли администратора пользователя в арендаторе.
1. Выберите **группы** > **Все группы**.
1. Выберите группу, чтобы открыть свой профиль.
1. На странице профиля группы выберите **правила динамического членства.** Строитель правил поддерживает до пяти выражений. Чтобы добавить более пяти выражений, необходимо использовать текстовое поле.

   ![Добавление правила членства для динамической группы](./media/groups-create-rule/update-dynamic-group-rule.png)

1. Чтобы увидеть пользовательские свойства расширения, доступные для правила членства:
   1. Выберите **Пользовательские свойства расширения**
   1. Введите идентификатор приложения, а затем выберите **свойства обновления.**
1. После обновления правила выберите **Сохранить**.

## <a name="turn-on-or-off-welcome-email"></a>Отключение отправки приветственного сообщения электронной почты

При создании новой группы Office 365 отправляется приветственное уведомление по электронной почте пользователям, которые добавляются в группу. Позже, если какие-либо атрибуты пользователя или устройства изменяются, все динамические правила группы в организации обрабатываются для изменения членства. После этого добавляемым пользователям также отправляются приветственные уведомления. Вы можете отключить это поведение в [Exchange PowerShell.](https://docs.microsoft.com/powershell/module/exchange/users-and-groups/Set-UnifiedGroup?view=exchange-ps)

## <a name="check-processing-status-for-a-rule"></a>Проверка состояния обработки для правила

Вы можете просматривать состояние обработки членства и дату последнего обновления на странице **Обзор** для группы.
  
  ![отображение динамического статуса группы](./media/groups-create-rule/group-status.png)

Для состояния **Обработка членства** могут отображаться следующие сообщения о состоянии:

- **Оценка** — изменение группы получено и оценивается.
- **Обработка** — обновления обрабатываются.
- **Обновление выполнено** — обработка завершена, все применимые обновления внесены.
- **Ошибка обработки**: Обработка не может быть завершена из-за ошибки, оценивающей правило членства.
- **Обновление приостановлено** — обновление правила динамического членства приостановлено администратором. MembershipRuleProcessingState имеет статус "Приостановлено".

Для состояния **Последнее обновление членства** могут отображаться следующие сообщения о состоянии:

- &lt;**Дата и время**&gt;: Последний раз, когда членство было обновлено.
- **Выполняется** — обновления выполняются.
- **Неизвестный**: Последнее время обновления не может быть извлечено. Группа может быть новой.

Если произошла ошибка при обработке правила членства для конкретной группы, отображается предупреждение в верхней части **страницы "Обзор"** для группы. Если динамические обновления членства не могут быть обработаны для всех групп в составе арендатора более 24 часов, предупреждение отображается в верхней части **всех групп.**

![обработка предупреждений об ошибках](./media/groups-create-rule/processing-error.png)

# <a name="create-a-dynamic-group-and-check-status"></a>Создание динамической группы и проверка состояния

В Azure Active Directory (Azure AD) можно использовать правила для определения членства в группах на основе свойств пользователя или устройства. В этой статье описывается настройка правила для динамической группы на портале Azure.
Динамическое членство поддерживается для групп безопасности или группы Office 365. При применении правила членства в группе атрибутов пользователей и устройств, проверяются на соответствия с помощью правила членства. При изменении атрибута для пользователя или устройство для изменения членства обрабатываются все правила динамических групп в организации. Пользователей и устройств добавляются или удаляются, если они удовлетворяют условиям для группы.

Примеры синтаксиса, поддерживаемых свойств, операторы и значения для правила членства, см. в разделе [правила динамического членства для групп в Azure Active Directory](groups-dynamic-membership.md).

## <a name="to-create-a-group-membership-rule"></a>Создание правила членства в группе

1. Войдите в [Центр администрирования Azure AD](https://aad.portal.azure.com) под учетной записью, глобальный администратор, администратор Intune или роль администратора пользователя в клиенте.
2. Выберите **Группы**.
3. Выберите **Все группы**, а затем — **Новая группа**.

   ![Выберите команду, чтобы добавить новую группу](./media/groups-create-rule/new-group-creation.png)

4. На **группы** введите имя и описание для новой группы. Выберите **тип членства** для пользователей или устройств, а затем выберите **добавить динамический запрос**. Чтобы создать простое правило, можно использовать конструктор правил или [самостоятельно создать правило членства](groups-dynamic-membership.md).

   ![Добавление правила членства для динамической группы](./media/groups-create-rule/add-dynamic-group-rule.png)

5. Чтобы увидеть расширение пользовательских свойств, доступных для запроса членства
   1. Выберите **получить свойства пользовательского расширения**
   2. Введите идентификатор приложения, а затем выберите **обновить свойства**. 
6. После создания правила выберите **Добавить запрос** в нижней части колонки.
7. Щелкните **Создать** в колонке **Группа**, чтобы создать группу.

Если правило, которое вы ввели не подходит, объяснение, почему не удалось обработать правило отображается в правом верхнем углу портала. Прочитайте его осторожно, чтобы понять, как исправить правило.

## <a name="turn-on-or-off-welcome-email"></a>Отключение отправки приветственного сообщения электронной почты

При создании новой группы Office 365, приветствия уведомление отправляется пользователей, которые добавляются в группу. Позже при изменении любых атрибутов пользователя или устройства, для изменения членства обрабатываются все правила динамических групп в организации. После этого добавляемым пользователям также отправляются приветственные уведомления. Вы можете отключить это поведение в [Exchange PowerShell](https://docs.microsoft.com/powershell/module/exchange/users-and-groups/Set-UnifiedGroup?view=exchange-ps). 

## <a name="check-processing-status-for-a-rule"></a>Проверьте состояние обработки для правила

Вы можете просматривать состояние обработки членства и дату последнего обновления на странице **Обзор** для группы.
  
  ![Отображение состояния динамической группы](./media/groups-create-rule/group-status.png)

Для состояния **Обработка членства** могут отображаться следующие сообщения о состоянии:

* **Оценка**:  изменение группы получено и оценивается.
* **Обработка**: обновления обрабатываются.
* **Обновление выполнено**: обработка завершена, все применимые обновления внесены.
* **Ошибка обработки**:  Не удалось завершить обработку из-за ошибка при оценке правила членства.
* **Обновление приостановлено**: обновление правила динамического членства приостановлено администратором. MembershipRuleProcessingState имеет статус "Приостановлено".

Для состояния **Последнее обновление членства** могут отображаться следующие сообщения о состоянии:

* &lt;**Дата и время**&gt;: время последнего обновления членства.
* **Выполняется**. Обновления выполняются.
* **Неизвестно**: Не удалось получить время последнего обновления. Эта группа может быть новый.

Если произошла ошибка при обработке правила членства для конкретной группы, отображается предупреждение в верхней части **страницы "Обзор"** для группы. Если более 24 часов невозможно было обработать никакие ожидающие обработки изменения динамического членства для всех групп в клиенте, отображается предупреждение в верхней части страницы **Все группы**.

![Обработка оповещения сообщения об ошибках](./media/groups-create-rule/processing-error.png)

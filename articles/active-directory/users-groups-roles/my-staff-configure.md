# <a name="manage-your-users-with-my-staff-preview"></a>Управление пользователями с помощью My Staff (предварительный просмотр)

Мой персонал позволяет вам делегировать на должность руководителя магазина или руководителя группы разрешения, чтобы обеспечить доступ своих сотрудников к своим учетным записям Azure AD. Вместо того, чтобы полагаться на центральную службу поддержки, организации могут делегировать общие задачи, такие как сброс паролей или изменение телефонных номеров менеджеру группы. С my Staff пользователь, который не может получить доступ к своей учетной записи, может восстановить доступ всего за пару кликов, без необходимости в справочной службе или ИТ-персонале.

Перед настройкой My Staff для вашей организации мы рекомендуем вам просмотреть эту документацию, а также [пользовательскую документацию,](../user-help/my-staff-team-manager.md) чтобы убедиться, что вы понимаете функциональность и влияние этой функции на пользователей. Вы можете использовать пользовательскую документацию для обучения и подготовки пользователей к новому опыту и помочь обеспечить успешное развертывание.

## <a name="how-my-staff-works"></a>Как работает мой персонал

Мой персонал основан на административных единицах (AUs), которые представляют собой контейнер ресурсов, которые могут быть использованы для ограничения сферы административного контроля за назначением ролей. В моем персонале AUs используются для определения подмножества пользователей организации, таких как магазин или отдел. Затем, например, менеджеру группы может быть назначена роль, область действия которой является одним или более AUs. В приведенном ниже примере пользователю была предоставлена административная роль аутентификации, и три AUs являются областью роли. Более подробную информацию об административных подразделениях можно узнать [в каталоге Azure Active Directory.](directory-administrative-units.md)

## <a name="how-to-enable-my-staff"></a>Как включить мой персонал

После настройки AUs можно применить эту область к пользователям, которые получают доступ к Моему Персоналу. Доступ к Моему персоналу могут получить только пользователи, которым назначена административная роль. Чтобы включить мой персонал, выполните следующие шаги:

1. Войти на портал Azure в качестве администратора пользователя.
2. Просмотр настроек **azure Active Directory** > **Пользовательские функции** > **Пользователь функция предварительного просмотра** > **Управления настройками функции пользователя.**
3. Администраторы **могут получить доступ к Моему Персоналу,** вы можете выбрать для включения для всех пользователей, выбранных пользователей или без доступа к пользователю.

> [!Note]
> Доступ к моему персоналу могут получить только пользователи, которым была назначена роль админ. Если вы включите Мой Персонал для пользователя, которому не назначена роль админ, он не сможет получить доступ к моему персоналу.

## <a name="using-my-staff"></a>Использование моего персонала

Когда пользователь идет в Мой Персонал, ему показывают имена [административных единиц,](directory-administrative-units.md) на которые у него есть административные разрешения. В [пользовательской документации My Staff](../user-help/my-staff-team-manager.md)мы используем термин "местоположение" для обозначения административных единиц. Если разрешения администратора не имеют сферы действия AU, разрешения применяются в организации. После включения моего персонала пользователи, которым включена и назначена административная роль, могут получить к нему доступ. [https://mystaff.microsoft.com](https://mystaff.microsoft.com) Они могут выбрать AU для просмотра пользователей в этом АС, и выбрать пользователя, чтобы открыть свой профиль.

## <a name="licenses"></a>Лицензии

Каждый пользователь, включенный в Мой Персонал, должен иметь лицензию, даже если он не использует портал My Staff. Каждый пользователь с включенным пользователем должен иметь одну из следующих лицензий Azure AD или Microsoft 365:

- Azure AD Premium (P1 или P2)
- Microsoft 365 F1 или F3

## <a name="reset-a-users-password"></a>Сброс пароля пользователя

Следующие роли имеют разрешение на сбросить пароль пользователя:

- [Администратор аутентификации](directory-assign-admin-roles.md#authentication-administrator)
- [Привилегированный администратор аутентификации](directory-assign-admin-roles.md#privileged-authentication-administrator)
- [Глобальный администратор](directory-assign-admin-roles.md#global-administrator--company-administrator)
- [администратор службы технической поддержки;](directory-assign-admin-roles.md#helpdesk-administrator)
- [Администратор пользователей](directory-assign-admin-roles.md#user-administrator)
- [Администратор паролей](directory-assign-admin-roles.md#password-administrator)

От **моего персонала**, открыть профиль пользователя. Выберите **Сбросить пароль**.

- Если пользователь только облако, вы можете увидеть временный пароль, который вы можете дать пользователю.
- Если пользователь синхронизирован с предприимчивым Active Directory, вы можете ввести пароль, который соответствует вашим правилам AD. Затем вы можете дать этот пароль пользователю.

    ![Индикатор сбросить пароль и уведомление об успехе](media/my-staff-configure/reset-password.png)

Пользователь обязан изменить свой пароль при следующем входе.

## <a name="manage-a-phone-number"></a>Управление номером телефона

От **моего персонала**, открыть профиль пользователя.

- Выберите Добавить раздел **номера телефона,** чтобы добавить номер телефона для пользователя
- Выберите **номер телефона для отсеивании** номера телефона
- **Выберите Удалить номер телефона,** чтобы удалить номер телефона пользователя

В зависимости от настроек пользователь может использовать настроенный номер телефона для регистрации с помощью SMS, выполнять многофакторную аутентификацию и выполнять сброс пароля самообслуживания.

Чтобы управлять номером телефона пользователя, необходимо назначить одну из следующих ролей:

- [Администратор аутентификации](directory-assign-admin-roles.md#authentication-administrator)
- [Привилегированный администратор аутентификации](directory-assign-admin-roles.md#privileged-authentication-administrator)
- [Глобальный администратор](directory-assign-admin-roles.md#global-administrator--company-administrator)

## <a name="search"></a>Поиск

Вы можете искать AUs и пользователей в вашей организации с помощью панели поиска в My Staff. Вы можете искать во всех AUs и пользователей в вашей организации, но вы можете вносить изменения только для пользователей, которые находятся в Au, над которым и получили разрешения админа.

Вы также можете искать пользователя в ас-УК. Для этого используйте панель поиска в верхней части списка пользователей.

## <a name="audit-logs"></a>Журналы аудита

Вы можете просмотреть журналы аудита для действий, выполненных в my Staff на портале Active Directory Azure. Если журнал аудита был создан в результате действий, принятых в My Staff, вы увидите это, указанное в мероприятии AUDIT DETAILS в событии аудита.

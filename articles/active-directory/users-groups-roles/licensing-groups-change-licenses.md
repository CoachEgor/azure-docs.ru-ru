# <a name="change-license-assignments-for-a-user-or-group-in-azure-active-directory"></a>Изменение назначений лицензии для пользователя или группы в active-каталоге Azure

В этой статье описывается, как перемещать пользователей и группы между планами лицензий служб в Active Directory Azure (Azure AD). Целью подхода Azure AD является обеспечение потери сервиса или данных во время изменения лицензии. Пользователи должны беспрепятственно переключаться между службами. Шаги назначения лицензионного плана в этой статье описывают изменение пользователя или группы на Office 365 E1 на Office 365 E3, но эти шаги применяются ко всем лицензионным планам. При обновлении лицензионных заданий для пользователя или группы одновременно производятся удаления назначений лицензий и новые назначения, чтобы пользователи не теряли доступ к своим службам во время изменений лицензии или не видели конфликты лицензий между планами.

## <a name="before-you-begin"></a>Перед началом

Перед обновлением назначений лицензий важно проверить некоторые предположения, верные для всех пользователей или групп, которые должны быть обновлены. Если предположения не верны для всех пользователей в группе, миграция может не удается для некоторых. В результате некоторые пользователи могут потерять доступ к данным или службам. Убедитесь в следующем:

- Пользователи имеют текущий план лицензии (в данном случае Office 365 E1), который назначен группе и наследуется пользователем и не назначается напрямую.

- У вас достаточно доступных лицензий для назначенного вами лицензионного плана. Если у вас недостаточно лицензий, некоторым пользователям может не быть назначен новый план лицензии. Вы можете проверить количество доступных лицензий.

- У пользователей нет других назначенных лицензий службы, которые могут встать в противоречие с желаемой лицензией или предотвратить удаление текущей лицензии. Например, лицензия от службы, такой как Workplace Analytics или Project Online, которая зависит от других служб.

- Если вы управляете группами на месте и синхронизируете их в Azure AD через Azure AD Connect, то вы добавляете или удаляете пользователей с помощью своей системы. Синхронизация изменений с Azure AD может занять некоторое время при лицензировании группы.

- Если вы используете членство в динамической группе Azure AD, вы добавляете или удаляете пользователей, изменяя их атрибуты, но процесс обновления для назначений лицензий остается прежним.

## <a name="change-user-license-assignments"></a>Изменение назначений лицензии пользователя

На странице **назначений лицензий обновления,** если вы видите, что некоторые флажки недоступны, это указывает на службы, которые не могут быть изменены, потому что они унаследованы от групповой лицензии.

1. Войти на [портал Azure](https://portal.azure.com/) с помощью учетной записи администратора лицензии в организации Azure AD.
1. Выберите > **пользователей** **Active Directory Azure,** а затем откройте страницу **профиля** для пользователя.
1. Выберите **Лицензии**.
1. Выберите **назначения** для отсеивательного назначения лицензии для пользователя или группы. Страница **«Назначения»** — это место, где можно разрешить конфликты по назначению лицензий.
1. Выберите флажок office 366 E3 и убедитесь, что как минимум будут выбраны все службы E1, назначенные пользователю.
1. Очистите чек-бокс для Office 365 E1.

    ![страница лицензионных назначений для пользователя, показывающего Office 365 E1 очищены и Office 365 E3 выбран](media/licensing-groups-change-licenses/update-user-license-assignments.png)

1. Нажмите кнопку **Сохранить**.

Azure AD применяет новые лицензии и удаляет старые лицензии одновременно, чтобы обеспечить непрерывность обслуживания.

## <a name="change-group-license-assignments"></a>Изменение групповых назначений лицензий

1. Войти на [портал Azure](https://portal.azure.com/) с помощью учетной записи администратора лицензии в организации Azure AD.
1. Выберите > **группы** **активных каталогов Azure,** а затем откройте страницу **«Обзор»** для группы.
1. Выберите **Лицензии**.
1. Выберите команду **«Назначения»** для отсеивательного назначения лицензии для пользователя или группы.
1. Выберите чек-бокс для Office 366 E3. Чтобы сохранить непрерывность обслуживания, убедитесь, что вы выберите все службы E1, которые уже назначены пользователю.
1. Очистите чек-бокс для Office 365 E1.

    ![Выберите команду «Назначения» на странице лицензий пользователя или группы](media/licensing-groups-change-licenses/update-group-license-assignments.png)

1. Нажмите кнопку **Сохранить**.

Чтобы обеспечить непрерывность обслуживания, Azure AD применяет новые лицензии и удаляет старые лицензии одновременно для всех пользователей в группе.

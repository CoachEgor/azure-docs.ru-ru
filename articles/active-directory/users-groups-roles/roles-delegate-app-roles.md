# <a name="delegate-app-administrator-roles-in-azure-active-directory"></a>Делегирование ролей администратора приложения в Azure Active Directory

 Azure AD позволяет делегировать управление доступом приложения к набору встроенных ролей администратора. Помимо снижения издержек глобального администратора делегирование специализированных привилегий для управления заданиями доступа к приложению может улучшить состояние безопасности и уменьшить количество потенциально несанкционированных попыток доступа. Вопросы делегирования и общие рекомендации рассматриваются в статье [Делегирование администрирования в Azure Active Directory](roles-concept-delegation.md).

## <a name="delegate-app-administration"></a>Делегирование прав на администрирование приложений

Следующие роли предоставляют разрешения на управление регистрациями приложения, параметрами единого входа, пользовательскими и групповыми назначениями, а также позволяют дать согласие на использование делегированных разрешений и разрешений приложений (кроме Microsoft Graph и Azure AD Graph). Единственная разница в том, что роль администратора приложения также предоставляет разрешение на управление параметрами прокси приложения. Ни одна из ролей не предоставляет возможность управления параметрами условного доступа.
> [!IMPORTANT]
> Пользователи, которым назначена эта роль, могут добавить учетные данные в приложение и использовать их для олицетворения удостоверения приложения. Это олицетворение удостоверения приложения может представлять собой повышение привилегий пользователя по сравнению с другими назначенным ему ролями в Azure AD. Присвоенный этой роли пользователь может потенциально создавать или обновлять пользователей или другие объекты при олицетворении приложения.

Чтобы предоставить возможность управлять доступом приложения на портале Azure, сделайте следующее:

1. Войдите в [клиент Azure AD](https://portal.azure.com/#blade/Microsoft_AAD_IAM/ActiveDirectoryMenuBlade/Overview) с помощью учетной записи, которая подходит для роли глобального администратора в клиенте.
2. При наличии достаточных разрешений откройте страницу [роли и администраторы](https://portal.azure.com/#blade/Microsoft_AAD_IAM/ActiveDirectoryMenuBlade/RolesAndAdministrators).
3. Откройте одну из следующих ролей, чтобы просмотреть назначения ее элементов:
   * **администратор приложения**;
   * **администратор облачных приложений**.
4. На странице **Члены** роли выберите **Добавить участника**.
5. Выберите одного или нескольких участников, которые будут добавлены в роль. <!--Members can be users or groups.-->

Вы можете просмотреть описание этих двух ролей в разделе [Доступные роли](directory-assign-admin-roles.md#available-roles).

## <a name="delegate-app-registration"></a>Делегирование прав на регистрацию приложений

По умолчанию все пользователи могут создавать регистрации приложений, но вы можете выборочно предоставлять разрешение на создание регистрации приложения или разрешения на согласие, чтобы авторизовать приложение.

1. Войдите в [клиент Azure AD](https://portal.azure.com/#blade/Microsoft_AAD_IAM/ActiveDirectoryMenuBlade/Overview) с помощью учетной записи, которая подходит для роли глобального администратора в клиенте.
2. Когда вы получите достаточные разрешения, установите одно из следующих значений или оба:
   * На [странице пользовательских настроек для клиента](https://portal.azure.com/#blade/Microsoft_AAD_IAM/ActiveDirectoryMenuBlade/UserSettings) задайте для параметра **Пользователи могут регистрировать приложение** значение "Нет".
   * В [параметрах пользователя для корпоративных приложений](https://portal.azure.com/#blade/Microsoft_AAD_IAM/StartboardApplicationsMenuBlade/UserSettings/menuId/) задайте для параметра **Users can consent to applications accessing company data on their behalf** (Пользователи могут разрешать приложениям доступ к корпоративным данным от своего имени) значение "Нет".
3. Затем назначьте пользователей, которым требуется это разрешение, членами роли разработчика приложения, если это необходимо.

Когда пользователь регистрирует приложение, он автоматически добавляется в качестве первого владельца приложения.

## <a name="delegate-app-ownership"></a>Делегирование прав владения приложениями

Владельцам приложений и владельцев приложений регистрации можно каждого управлять только приложения и регистрации приложений, которыми они владеют. Например, при добавлении владельца в приложение Salesforce он может управлять доступом к приложению Salesforce и его настройкой, но не к другим приложениям. У приложения может быть много владельцев. Пользователь может владеть многими приложениями.

Владелец приложения может:

* Изменять свойства приложения, такие как имя и разрешения запросов приложения.
* Управление учетными данными
* Настройка единого входа
* Назначать доступ пользователей.
* Добавлять и удалять других владельцев.
* Изменять манифест приложения.
* Опубликовать приложение в коллекции приложений.

> [!IMPORTANT]
> Пользователи, которым назначена эта роль, могут добавить учетные данные в приложение и использовать их для олицетворения удостоверения приложения. Это олицетворение удостоверения приложения может представлять собой повышение привилегий пользователя по сравнению с другими назначенным ему ролями в Azure AD. Присвоенный этой роли пользователь может потенциально создавать или обновлять пользователей или другие объекты при олицетворении приложения.

Владелец регистрации приложения может просматривать и изменять ее.

### <a name="to-assign-an-owner-to-an-application"></a>Назначение владельца в приложение

1. Войдите в [клиент Azure AD](https://portal.azure.com/#blade/Microsoft_AAD_IAM/ActiveDirectoryMenuBlade/Overview) с помощью учетной записи, разрешенной для администратора приложения или облачного приложения для клиента.
2. На [странице регистраций приложений](https://portal.azure.com/#blade/Microsoft_AAD_IAM/StartboardApplicationsMenuBlade/AllApps/menuId/) для клиента выберите приложение, чтобы открыть страницу **Обзор** приложения.
3. Выберите **Владельцы** для просмотра списка владельцев в приложении.
4. Выберите **Добавить** для выбора одного или нескольких владельцев для добавления в приложение.

### <a name="to-assign-an-owner-to-an-application-registration"></a>Назначение владельца регистрации приложения

1. Войдите в [клиент Azure AD](https://portal.azure.com/#blade/Microsoft_AAD_IAM/ActiveDirectoryMenuBlade/Overview) с помощью учетной записи, разрешенной для роли администратора приложения или облачного приложения в клиенте.
2. При наличии необходимых разрешений на [странице корпоративных приложений](https://portal.azure.com/#blade/Microsoft_AAD_IAM/StartboardApplicationsMenuBlade/AllApps/menuId/) для клиента выберите регистрацию приложения, чтобы открыть ее.
3. Выберите элемент **Параметры**.
4. Выберите **Владельцы** на странице **Параметры** для просмотра списка владельцев приложения.
5. Выберите **Добавить владельца**, чтобы выбрать одного или нескольких владельцев для добавления в приложение.

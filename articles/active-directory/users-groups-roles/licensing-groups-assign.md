# <a name="assign-licenses-to-users-by-group-membership-in-azure-active-directory"></a>Назначение лицензий пользователям в соответствии с членством в группах в Azure Active Directory

В этой статье вы можете присвоить лицензии на продукт группе пользователей и проверить, что они лицензированы правильно в Active Directory Azure (Azure AD).

В этом примере клиент содержит группу безопасности с именем **Отдел кадров**. Эта группа включает в себя всех сотрудников отдела кадров (около 1000 пользователей). Вы хотите назначить лицензии Office 365 корпоративный E3 всему отделу. Службу Yammer корпоративный, включенную в продукт, необходимо временно отключить, пока отдел не будет готов ее использовать. Вы также планируете развернуть лицензии на Enterprise Mobility + Security в той же группе пользователей.

> [!NOTE]
> Некоторые службы Майкрософт недоступны во всех расположениях. Прежде чем назначать лицензию пользователю, администратор должен указать для пользователя свойство "Место использования".
>
> Если лицензии назначаются группам, все пользователи, для которых не указано расположение, наследуют расположение каталога. Если пользователи находятся в разных расположениях, мы рекомендуем всегда указывать расположение использования при создании пользователей в Azure AD (например, при настройке AAD Connect). Это гарантирует, что результат назначения лицензий будет всегда правильным, и пользователи не получат службы в недопустимых расположениях.

## <a name="step-1-assign-the-required-licenses"></a>Шаг 1. Назначение необходимых лицензий

1. Войти на [**сайт администратора Ad Azure**](https://aad.portal.azure.com) с учетной записью администратора лицензии. Для управления лицензиями учетная запись должна быть администратором лицензий, администратором пользователя или глобальным администратором.

1. Выберите **Лицензии,** чтобы открыть страницу, где вы можете видеть и управлять всеми лицензируемыми продуктами в арендаторе.

1. В рамках **всех продуктов**выберите Office 365 Enterprise E5 и Enterprise Mobility s Security E3, выбрав названия продуктов. Чтобы начать назначение, выберите **Назначайте** в верхней части страницы.

   ![Выбор продуктов для присвоения лицензий](./media/licensing-groups-assign/licenses-all-products-assign.png)
  
1. На странице **лицензии Assign** выберите **пользователей и группы,** чтобы открыть список пользователей и групп.

1. Выберите пользователя или группу, а затем используйте кнопку **«Выберите»** в нижней части страницы для подтверждения выбора.

1. На странице **лицензии Assign** щелкните **параметры назначения,** которые отображают все планы обслуживания, включенные в два продукта, которые мы выбрали ранее. Найдите **Yammer Enterprise** и **выключите** его, чтобы отключить эту услугу от лицензии продукта. Подтвердите, нажав **OK** в нижней части **вариантов лицензии**.

   ![выбрать планы обслуживания лицензий](./media/licensing-groups-assign/assignment-options.png)
  
1. Для выполнения задания на странице **лицензии Assign** нажмите **«Назначить»** в нижней части страницы.

1. В правом верхнем углу появится уведомление, показывающее состояние и результат процесса. Если не удалось назначить лицензии группе (например, из-за лицензий, уже существующих в группе), щелкните уведомление, чтобы просмотреть сведения об ошибке.

При присвоении лицензий группе Azure AD обрабатывает все существующие члены этой группы. Этот процесс может занять некоторое время, варьируясь в зависимости от размера группы. В следующем шаге объясняется, как проверить, завершен ли процесс, и узнать, требуются ли дополнительные действия для решения проблем.

## <a name="step-2-verify-that-the-initial-assignment-has-finished"></a>Шаг 2. Проверка завершения первоначального назначения

1. Перейдите к > **группам** **активных каталогов Azure**. Выберите группу, к которым были назначены лицензии.

1. На странице группы выберите **Лицензии.** Это позволяет быстро проверить назначение лицензий пользователям и наличие ошибок, которые нужно рассмотреть. Доступны следующие сведения:

   - Лицензии на обслуживание, которые в настоящее время назначены группе. Выберите запись, чтобы просмотреть конкретные службы, которые включены в лицензию, и внести изменения.

   - Обновления статуса последних изменений лицензии, которые доступны, если изменения обрабатываются или если обработка завершена для всех пользователей.

   - Информация об уступках лицензий пользователя, наданных в состоянии ошибки.

   ![ошибки лицензирования и статус лицензии](./media/licensing-groups-assign/assignment-errors.png)

1. Более подробная информация об обработке лицензий в**журналах** **Azure Active Directory** > **и групповых** > *групп.* >  Проверьте следующие действия:

   - Деятельность: `Start applying group based license to users`. Это действие регистрируется, когда система принимает изменение назначения лицензии на группу и начинает применять его ко всем пользователям. Оно содержит сведения о внесенных изменениях.

   - Деятельность: `Finish applying group based license to users`. Это действие регистрируется, когда система завершает обработку всех пользователей в группе. Оно содержит сводку с числом успешно обработанных пользователей и пользователей, которым не удалось назначить лицензии группы.

   Дополнительные сведения об использовании журналов аудита для анализа изменений, произведенных путем группового лицензирования, см. в [этом разделе](licensing-group-advanced.md#use-audit-logs-to-monitor-group-based-licensing-activity).

## <a name="step-3-check-for-license-problems-and-resolve-them"></a>Шаг 3. Проверка наличия проблем лицензирования и их устранение

1. Перейдите в > **группы** **активных каталогов Azure**и найдите группу, к которым были назначены лицензии.
1. На странице группы выберите **Лицензии.** Уведомление поверх страницы показывает, что существует 10 пользователей, которым лицензии не могут быть назначены. Откройте его, чтобы увидеть список всех пользователей в состоянии ошибки лицензирования для этой группы.
1. Столбец **Failed assignments**(Сбой назначений) указывает, что не удалось назначить пользователям лицензии обоих продуктов. В столбец **Основная причина сбоев** указывается причина сбоя. В данном случае это **Конфликтующие планы обслуживания**.

   ![лицензии, которые не могут быть назначены](./media/licensing-groups-assign/failed-assignments.png)

1. Выберите пользователя, чтобы открыть страницу **Лицензий** пользователя. На этой странице показаны все лицензии, которые в настоящее время назначены пользователю. В этом примере пользователь унаследовал лицензию Office 365 корпоративный E1 от группы **Пользователи киоска**. Эта лицензия конфликтует с лицензией E3 группы **Отдел кадров**, которую система пыталась применить. Поэтому пользователю не назначена ни одна лицензия.

   ![Просмотр всех лицензионных конфликтов для пользователя](./media/licensing-groups-assign/user-licence-conflicting-service-plans.png)

1. Чтобы устранить этот конфликт, удалите пользователя из группы **Пользователи киоска**. После того как служба Azure AD обработает изменение, лицензии группы **Отдел кадров** будут назначены правильно.

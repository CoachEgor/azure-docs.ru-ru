# <a name="home-realm-discovery-for-azure-active-directory-sign-in-pages"></a>Обнаружение домашней области для страницы входа Azure Active Directory

Мы меняем поведение входа Azure Active Directory (Azure AD), чтобы добавить новые методы аутентификации и повысить удобство использования. Во время входа Azure AD определяет, нужно ли пользователю пройти аутентификацию. Azure AD принимает обоснованные решения, считывая параметры организации и пользователя для имени пользователя, введенного на странице входа. Это шаг к будущему отказу от использования паролей, который позволит применять дополнительные учетные данные, например FIDO 2.0.

## <a name="home-realm-discovery-behavior"></a>Поведение обнаружения домашней области

Исторически сложилось так, что обнаружение домашней области была регулируется доменом, предоставляемый при входе в систему, или с помощью политики обнаружения домашней области для некоторых устаревших приложений. Например в нашей поведение обнаружения пользователя Azure Active Directory может допустить свое имя пользователя, но по-прежнему будет поступать в экран коллекции учетных данных организации. Это происходит, когда пользователь правильно предоставляет имя домена организации «contoso.com». Такое поведение не предусматривает детализацию для настройки входа отдельных пользователей.

Поведение подстановки имени пользователя Azure Active Directory во время процесса входа в систему для поддержки более широкого круга учетные данные и повысить удобство использования, теперь обновляется. Новое поведение делает обоснованные решения, считывая клиента и пользователя уровня параметры в зависимости от имени пользователя, введенное на странице входа в систему. Для этого Azure Active Directory будет проверять, существует в их указанный домен имя пользователя, введенное на странице входа в систему и перенаправляет пользователя для предоставления учетных данных.

Дополнительным преимуществом часть этой работы является улучшенная ошибки обмена сообщениями. Вот несколько примеров, улучшенный ошибки обмена сообщениями при входе в приложение, которое поддерживает только для пользователей Azure Active Directory.

- Неверном вводе имени пользователя или имя пользователя не был синхронизирован с Azure AD:
  
    ![имя пользователя неправильно ввел или не найден](./media/signin-realm-discovery/typo-username.png)
  
- Неверном вводе имя домена:
  
    ![имя домена неправильно ввел или не найден](./media/signin-realm-discovery/typo-domain.png)
  
- Пользователь пытается выполнить вход с использованием доменом известных клиента:
  
    ![вход в систему с доменом известных клиента](./media/signin-realm-discovery/consumer-domain.png)
  
- Является неправильно ввел пароль, но точное имя пользователя:  
  
    ![неверном вводе пароля с хорошим именем пользователя](./media/signin-realm-discovery/incorrect-password.png)
  
> [!IMPORTANT]
> Эта функция может повлиять на федеративных доменов, полагаясь на старый режим домена обнаружения домашней области для принудительного федерации. Для обновления на при будет добавлена поддержка федеративного домена, см. в разделе [обнаружения домашней области при входе в систему для служб Microsoft 365](https://azure.microsoft.com/updates/signin-hrd/). В то же время в некоторых организациях обучения своих сотрудников, выполнить вход с использованием имени пользователя, который не существует в Azure Active Directory, но содержит правильное имя домена, так как имена доменов направляет пользователей в настоящее время к конечной точке домена своей организации. Новое поведение входа в систему не позволяет это сделать. Пользователь будет уведомлен о необходимости исправьте имя пользователя, и они не может выполнить вход с использованием имени пользователя, который не существует в Azure Active Directory.
>
> У вас или вашей организации и рекомендации, которые зависят от прежнего поведения, очень важно для администраторов организации для обновления документации по входа в систему и проверки подлинности сотрудников и для обучения сотрудников, чтобы использовать свое имя пользователя Azure Active Directory для входа.

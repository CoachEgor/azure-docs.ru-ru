# <a name="home-realm-discovery-for-azure-active-directory-sign-in-pages"></a>Открытие домашнего царства для страниц регистрации Active Directory Azure

Мы меняем поведение ввне в систему Azure Active Directory (Azure AD), чтобы освободить место для новых методов проверки подлинности и улучшить удобство использования. Во время входе в систему Azure AD определяет, где пользователю необходима аутентификация. Azure AD принимает интеллектуальные решения, читая организацию и настройки пользователя для имени пользователя, ввесенного на странице ввода. Это шаг к будущему без паролем, которое позволяет получить дополнительные учетные данные, такие как FIDO 2.0.

## <a name="home-realm-discovery-behavior"></a>Домашнее поведение открытия realm

Исторически, открытие домашнего realm было управлено доменом который обеспечен на входе-входе или домашней политикой открытия realm для некоторых устаренных применений. Например, в нашем поведении обнаружения пользователь Azure Active Directory может неправильно ввести свое имя пользователя, но все равно будет прибывать на экран сбора учетных данных своей организации. Это происходит, когда пользователь правильно предоставляет доменное имя организации "contoso.com". Такое поведение не позволяет детализации настроить опыт для отдельного пользователя.

Для поддержки более широкого диапазона учетных данных и повышения удобства использования теперь обновляется поведение поиска имени пользователя Azure Active Directory во время процесса вхинга. Новое поведение принимает интеллектуальные решения, читая настройки уровня пользователя на основе имени пользователя, введенного на странице ввода. Чтобы сделать это возможным, Azure Active Directory проверит, существует ли имя пользователя, внесенное на страницу ввода в систему, в указанном домене или перенаправляет пользователя на предоставление своих учетных данных.

Дополнительным преимуществом этой работы является улучшение обмена ошибками. Вот несколько примеров улучшения обмена сообщениями об ошибках при входе в приложение, которое поддерживает только пользователей Active Directory Azure.

- Имя пользователя неправильно введено или имя пользователя еще не синхронизировано с Azure AD:
  
    ![имя пользователя неправильно введено или не найдено](./media/signin-realm-discovery/typo-username.png)
  
- Доменное имя неправильно введено:
  
    ![доменное имя неправильно введено или не найдено](./media/signin-realm-discovery/typo-domain.png)
  
- Пользователь пытается войти в известный потребительский домен:
  
    ![войти в систему с известным потребительским доменом](./media/signin-realm-discovery/consumer-domain.png)
  
- Пароль неправильно, но имя пользователя точно:  
  
    ![пароль неправильно вводится с хорошим именем пользователя](./media/signin-realm-discovery/incorrect-password.png)
  
> [!IMPORTANT]
> Эта функция может оказать влияние на федеративные домены, опирающиеся на старый домен уровня Home Realm Discovery, чтобы заставить федерацию. Для получения обновлений о том, когда будет добавлена поддержка федеративного домена, [см.](https://azure.microsoft.com/updates/signin-hrd/) В то же время некоторые организации обучили своих сотрудников входить в систему с именем пользователя, которое не существует в Active Directory Azure, но содержит правильное доменное имя, потому что доменные имена направляют пользователей в настоящее время в конечную точку домена их организации. Новое поведение ввески не позволяет этого. Пользователю уведомляются об исправлении имени пользователя, и ему не разрешается входить в систему с именем пользователя, которое не существует в Active Directory Azure.
>
> Если у вас или вашей организации есть методы, зависят от старого поведения, администраторам организации важно обновить документацию о входе сотрудника и аутентификации и обучить сотрудников использовать имя пользователя Azure Active Directory для входного входного запроса.
  
Если у вас есть проблемы с новым поведением, оставьте свои замечания в разделе **Отзывы** этой статьи.

# <a name="home-realm-discovery-for-azure-active-directory-sign-in-pages"></a>Обнаружение домашней области для страниц входа Azure Active Directory

Мы изменим поведение входа в Azure Active Directory (Azure AD), чтобы освободить место для новых методов проверки подлинности и повысить удобство использования. Во время входа Azure AD определяет, где пользователь должен пройти проверку подлинности. Azure AD принимает интеллектуальные решения, считывая параметры организации и пользователя для имени пользователя, указанного на странице входа. Это шаг к будущему без пароля, который включает дополнительные учетные данные, такие как FIDO 2,0.

## <a name="home-realm-discovery-behavior"></a>Поведение обнаружения домашней области

Исторически обнаружение домашней области регулируется доменом, предоставленным при входе или политикой обнаружения домашней области для некоторых устаревших приложений. Например, в нашем способе обнаружения пользователь Azure Active Directory мог ввести имя пользователя неправильно, но по-прежнему появится на экране сбора учетных данных Организации. Это происходит, когда пользователь правильно предоставляет доменное имя организации "contoso.com". Такое поведение не позволяет настраивать возможности для отдельного пользователя.

Для поддержки более широкого спектра учетных данных и повышения удобства использования Azure Active Directory Поиск имени пользователя во время входа в систему теперь обновляется. Новое поведение принимает интеллектуальные решения, считывая параметры уровня клиента и пользователя на основе имени пользователя, введенного на странице входа. Чтобы сделать это возможным, Azure Active Directory проверит, существует ли имя пользователя, введенное на странице входа в указанном домене, или перенаправит пользователя для предоставления учетных данных.

Дополнительное преимущество этой работы — Улучшенная система обмена сообщениями об ошибках. Ниже приведены некоторые примеры улучшенных сообщений об ошибках при входе в приложение, которое поддерживает только Azure Active Directory пользователей.

- Введено неправильное имя пользователя, или имя пользователя еще не синхронизировано с Azure AD:
  
    ![имя пользователя введено неправильно или не найдено](./media/signin-realm-discovery/typo-username.png)
  
- Имя домена введено с недопустимым именем:
  
    ![доменное имя неправильно введено или не найдено](./media/signin-realm-discovery/typo-domain.png)
  
- Пользователь пытается войти с помощью известного домена потребителя:
  
    ![Вход в систему с известным доменом потребителя](./media/signin-realm-discovery/consumer-domain.png)
  
- Пароль введен неправильно, но имя пользователя является точным:  
  
    ![Неверный ввод пароля с хорошим именем пользователя](./media/signin-realm-discovery/incorrect-password.png)
  
> [!IMPORTANT]
> Эта функция может оказать влияние на федеративные домены, использующие старое обнаружение домашней области на уровне домена для принудительной интеграции. Сведения об обновлениях при добавлении поддержки федеративного домена см. [в разделе обнаружение домашней области во время входа для Microsoft 365 Services](https://azure.microsoft.com/updates/signin-hrd/). В то же время некоторые организации обучили свои сотрудники для входа с помощью имени пользователя, которое не существует в Azure Active Directory, но содержит правильное доменное имя, так как имена доменов перенаправляют пользователей в конечную точку домена своей организации. Новое поведение при входе не допускает этого. Пользователь получает извещение об исправлении имени пользователя, и ему запрещено выполнять вход с помощью имени пользователя, отсутствующего в Azure Active Directory.
>
> Если у вас или вашей организации есть рекомендации, зависящие от старого поведения, важно, чтобы администраторы организации могли обновлять документацию по входу и проверке подлинности сотрудников и обучать сотрудникам использовать Azure Active Directory имя пользователя для входа.
  
Если у вас есть проблемы с новым поведением, оставьте свои замечания в разделе " **Отзывы** " этой статьи.

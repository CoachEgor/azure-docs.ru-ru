# <a name="delete-a-directory-in-azure-active-directory"></a>Удаление каталога в Azure Active Directory

При удалении каталога Azure AD все содержащиеся в нем ресурсы также удаляются. Подготовка Организации путем минимизации связанных с ней ресурсов перед удалением. Только глобальный администратор Azure Active Directory (Azure AD) может удалить каталог Azure AD с портала.

## <a name="prepare-the-directory"></a>Подготовка каталога

Вы не можете удалить каталог в Azure AD, пока не будет пройдено несколько проверок. Эти проверки снижают риск того, что удаление каталога Azure AD отрицательно влияет на доступ пользователей, например на возможность входа в Office 365 или доступ к ресурсам в Azure. Например, если каталог, связанный с подпиской, непреднамеренно удален, пользователи не смогут получить доступ к ресурсам Azure для этой подписки. Проверяются следующие условия.

* В каталоге не может быть пользователей, кроме одного глобального администратора, который может удалить каталог. Всех других пользователей следует удалить перед удалением каталога. Если пользователи синхронизируются из локальной среды, сначала необходимо отключить синхронизацию, а пользователи должны быть удалены из облачного каталога с помощью командлетов портал Azure или Azure PowerShell.
* В каталоге не должно быть ни одного приложения. Перед удалением каталога необходимо удалить все приложения.
* К каталогу не могут быть привязаны поставщики многофакторной проверки подлинности.
* С каталогом не должны быть связаны подписки на любые службы Microsoft Online Services, такие как Microsoft Azure, Office 365 или Azure AD Premium. Например, если каталог по умолчанию создан в Azure, его невозможно удалить, если ваша подписка Azure по-прежнему использует его для проверки подлинности. Точно так же нельзя удалить каталог, если у другого пользователя есть связанная с ним подписка.

## <a name="delete-the-directory"></a>Удаление каталога

1. Войдите в [центр администрирования Azure AD](https://aad.portal.azure.com) с помощью учетной записи глобального администратора вашей организации.

2. Выберите **Azure Active Directory**.

3. Перейдите в каталог, который необходимо удалить.
  
   ![Подтвердить организацию перед удалением](./media/directory-delete-howto/delete-directory-command.png)

4. Нажмите **Удалить каталог**.
  
   ![выберите команду для удаления Организации](./media/directory-delete-howto/delete-directory-list.png)

5. Если каталог не проходит одну или несколько проверок, вы можете получить ссылку на дополнительные сведения о том, как передать. Когда клиент пройдет все проверки, нажмите **Удалить** для завершения процесса.

## <a name="if-you-cant-delete-the-directory"></a>Если не удается удалить каталог

При настройке каталога Azure AD вы также могли активировать подписки на основе лицензий для вашей организации, например Azure AD Premium P2, Office 365 Business Premium или Enterprise Mobility + Security. Чтобы избежать случайной потери данных, нельзя удалить каталог, пока подписки не будут полностью удалены. Чтобы разрешить удаление каталога, подписки должны находиться **в состоянии "** отозвано". Подписка с истекшим или **отмененным** **действием** переходит в **отключенное** состояние, а последний этап **— в состояние** отозвано.

В следующей таблице описано, что происходит, когда истекает срок действия пробной подписки Office 365 (не включая платную партнерскую подписку/CSP, соглашение Enterprise или корпоративные лицензии). Дополнительные сведения о жизненном цикле хранения данных и подписок в Office 365 см. в разделе [Что происходит с моими данными и доступом при завершении подписки Office 365 для бизнеса?](https://support.office.com/article/what-happens-to-my-data-and-access-when-my-office-365-for-business-subscription-ends-4436582f-211a-45ec-b72e-33647f97d8a3) 

Состояние подписки | Данные | Доступ к данным
----- | ----- | -----
Активно (пробная версия — 30 дней) | Данные доступны для всех | Пользователи имеют обычный доступ к файлам Office 365 или приложениям<br>Администраторы имеют стандартный доступ к Microsoft 365 центру администрирования и ресурсам. 
Просрочено (30 дней) | Данные доступны для всех| Пользователи имеют обычный доступ к файлам Office 365 или приложениям<br>Администраторы имеют стандартный доступ к Microsoft 365 центру администрирования и ресурсам.
Отключено (30 дней) | Данные доступны только для администратора | Пользователи не имеют доступ к файлам Office 365 или приложениям<br>Администраторы могут получать доступ к центру администрирования Microsoft 365, но не могут назначать лицензии или обновлять пользователей.

## <a name="delete-a-subscription"></a>Удаление подписки

Вы можете добавить подписку в состояние "отозвано **", чтобы** удалить ее в течение трех дней с помощью центра администрирования Microsoft 365.

1. Войдите в [центр администрирования Microsoft 365](https://admin.microsoft.com) с помощью учетной записи глобального администратора в вашей организации. Если вы пытаетесь удалить каталог contoso с начальным доменом по умолчанию contoso.onmicrosoft.com, выполните вход с помощью имени участника-пользователя, например @no__t – 0.

2. Выполните предварительный просмотр нового центра администрирования Microsoft 365, убедившись в том, что включен **новый переключатель центр администрирования** .

   ![Предварительный просмотр нового интерфейса центра администрирования M365](./media/directory-delete-howto/preview-toggle.png)

3. После включения нового центра администрирования необходимо отменить подписку, прежде чем ее можно будет удалить. Выберите **выставление счетов** и выберите **продукты & Services**, а затем выберите **Отменить подписку** для подписки, которую нужно отменить. Откроется страница обратной связи.

   ![Выберите подписку для отмены](./media/directory-delete-howto/cancel-choose-subscription.png)

4. Заполните форму обратной связи и выберите **Отменить подписку** , чтобы отменить подписку.

   ![Команда "Отмена" в предварительной версии подписки](./media/directory-delete-howto/cancel-command.png)

5. Теперь можно удалить подписку. Выберите **Удалить** для подписки, которую нужно удалить. Если вы не можете найти подписку на странице **products & Services** , убедитесь, что для параметра **состояние подписки** задано значение **все**.

   ![Удаление связи для удаления подписки](./media/directory-delete-howto/delete-command.png)

6. Выберите **удалить подписку** , чтобы удалить подписку и принять условия. Все данные будут безвозвратно удалены в течение трех дней. Вы можете [повторно активировать подписку](https://docs.microsoft.com/office365/admin/subscriptions-and-billing/reactivate-your-subscription?view=o365-worldwide) в течение трех дней, если вы передумали.
  
   ![внимательно читайте условия](./media/directory-delete-howto/delete-terms.png)

7. Теперь состояние подписки изменилось, и подписка помечена для удаления. Подписка входит в состояние **Отозвано** через 72 часа.

8. После удаления подписки в каталоге и 72 часов вы можете снова войти в центр администрирования Azure AD и не обязательно предпринимать никаких действий и подписки не будут блокировать Удаление каталога. Вы можете успешно удалить каталог Azure AD.
  
   ![пройденная проверка подписки на экране удаления](./media/directory-delete-howto/delete-checks-passed.png)

## <a name="i-have-a-trial-subscription-that-blocks-deletion"></a>У меня есть пробная подписка, которая блокирует удаление

Существуют такие [продукты самостоятельной регистрации](https://docs.microsoft.com/office365/admin/misc/self-service-sign-up?view=o365-worldwide) , как Microsoft Power BI, Rights Management Services, Microsoft Power Apps или Dynamics 365, отдельные пользователи могут зарегистрироваться через Office 365, что также создает гостевого пользователя для проверки подлинности в каталоге Azure AD. . Эти продукты самообслуживания блокируют удаление каталогов, пока они не будут полностью удалены из каталога, чтобы избежать потери данных. Они могут быть удалены только администратором Azure AD, независимо от того, зарегистрирован ли пользователь по отдельности или назначен продукт.

Существует два типа продуктов для самостоятельной регистрации в порядке их назначения: 

* Назначение на уровне Организации. Администратор Azure AD назначает продукт всей Организации, и пользователь может активно использовать службу с этим назначением на уровне Организации, даже если они не лицензированы по отдельности.
* Назначение уровня пользователя. индивидуальный пользователь во время самостоятельной регистрации самостоятельно назначает продукт самому себе без прав администратора. После того как Организация будет управляться администратором (см. раздел [Administrator перенаправление (администратор неуправляемого каталога](domains-admin-takeover.md)), администратор может напрямую назначить продукт пользователям без самостоятельной регистрации.  

Когда вы начинаете удалять продукт самостоятельной регистрации, это действие окончательно удаляет данные и удаляет всех пользователей, обращающихся к службе. Любой пользователь, которому было назначено предложение по отдельности или на уровне Организации, блокирует вход в систему или доступ к существующим данным. Если вы хотите предотвратить потери данных с помощью продукта самостоятельной регистрации, например [Microsoft Power BI панели мониторинга](https://docs.microsoft.com/power-bi/service-export-to-pbix) или [конфигурации политики Rights Management Services](https://docs.microsoft.com/azure/information-protection/configure-policy#how-to-configure-the-azure-information-protection-policy), убедитесь, что данные архивируются и сохраняются в других местах.

Дополнительные сведения о доступных для самостоятельной регистрации продуктах и службах см. в разделе [Доступные программы самообслуживания](https://docs.microsoft.com/office365/admin/misc/self-service-sign-up?view=o365-worldwide#available-self-service-programs).

В следующей таблице описано, что происходит, когда истекает срок действия пробной подписки Office 365 (не включая платную партнерскую подписку/CSP, соглашение Enterprise или корпоративные лицензии). Дополнительные сведения о сроке хранения данных в Office 365 и жизненном цикле подписки см. в статье [что происходит с моими данными и доступ к ним при завершении подписки office 365 для бизнеса?](https://docs.microsoft.com/office365/admin/subscriptions-and-billing/what-if-my-subscription-expires?view=o365-worldwide).

Состояние продукта | Данные | Доступ к данным
------------- | ---- | --------------
Активно (пробная версия — 30 дней) | Данные доступны для всех | Пользователи имеют стандартный доступ к продукту самостоятельной регистрации, файлам или приложениям.<br>Администраторы имеют стандартный доступ к Microsoft 365 центру администрирования и ресурсам.
Удалена | Данные удалены | Пользователи не могут получить доступ к продукту, файлам или приложениям самостоятельной регистрации.<br>Администраторы могут получить доступ к центру администрирования Microsoft 365 для приобретения других подписок и управления ими.

## <a name="how-can-i-delete-a-self-service-sign-up-product-in-the-azure-portal"></a>Как удалить продукт самостоятельной регистрации в портал Azure?

Вы можете добавить продукт самостоятельной регистрации, например Microsoft Power BI или Azure Rights Management Services, в состояние **удаления** , чтобы немедленно удалить его на портале Azure AD.

1. Войдите в [центр администрирования Azure AD](https://aad.portal.azure.com/#blade/Microsoft_AAD_IAM/ActiveDirectoryMenuBlade/Overview)@no__t — 1with учетную запись, которая является глобальным администратором в Организации. Если вы пытаетесь удалить каталог contoso с начальным доменом по умолчанию contoso.onmicrosoft.com, выполните вход с помощью имени участника-пользователя, например @no__t – 0.

2. Выберите **лицензии**, а затем **самостоятельная регистрация продуктов**. Вы можете просматривать все продукты самостоятельной регистрации отдельно от подписок на основе места. Выберите продукт, который вы хотите удалить без возможности восстановления. Ниже приведен пример в Microsoft Power BI.

    ![имя пользователя введено неправильно или не найдено](./media/directory-delete-howto/licenses-page.png)

3. Выберите **удалить** to удалить продукт и принять условия, когда данные удаляются немедленно и твердо. Это действие удаления приведет к удалению всех пользователей и удалению доступа Организации к продукту. Нажмите кнопку Да, чтобы перейти к удалению.  

    ![имя пользователя введено неправильно или не найдено](./media/directory-delete-howto/delete-product.png)

4. При нажатии **кнопки Да**будет инициировано удаление продукта самообслуживания. Появится уведомление о том, что удаление выполняется.  

    ![имя пользователя введено неправильно или не найдено](./media/directory-delete-howto/progress-message.png)

5. Теперь состояние продукта самостоятельной регистрации изменено на **удалено**. При обновлении страницы продукт следует удалить со страницы **продукты для самостоятельной регистрации** .  

    ![имя пользователя введено неправильно или не найдено](./media/directory-delete-howto/product-deleted.png)

6. После удаления всех продуктов вы можете снова войти в центр администрирования Azure AD и не обязательно предпринимать никаких действий, и ни один из продуктов не будет блокировать Удаление каталога. Вы можете успешно удалить каталог Azure AD.

    ![имя пользователя введено неправильно или не найдено](./media/directory-delete-howto/delete-organization.png)

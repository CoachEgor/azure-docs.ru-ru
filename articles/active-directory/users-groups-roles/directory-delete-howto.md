# <a name="delete-a-directory-in-azure-active-directory"></a>Удаление каталога в active-каталоге Azure

При удалении каталога Azure AD все ресурсы, содержащиеся в каталоге, также удаляются. Подготовьте организацию, снизив с нее связанные с ней ресурсы, прежде чем удалить ее. Только глобальный администратор Azure Active Directory (Azure AD) может удалить из портала каталог Azure AD.

## <a name="prepare-the-directory"></a>Подготовка каталога

Вы не можете удалить каталог в Azure AD до тех пор, пока он не пройдет несколько проверок. Эти проверки снижают риск того, что исключение каталога Azure AD негативно влияет на доступ пользователей, например возможность входить в Office 365 или доступ к ресурсам Azure. Например, если каталог, связанный с подпиской, непреднамеренно удален, пользователи не могут получить доступ к ресурсам Azure для этой подписки. Проверяются следующие условия.

* В каталоге не может быть пользователей, кроме одного глобального администратора, который должен удалить каталог. Всех других пользователей следует удалить перед удалением каталога. Если пользователи синхронизированы с помещений, синхронизация должна быть сначала отключена, и пользователи должны быть удалены в облачном каталоге с помощью портала Azure или cmdlets Azure PowerShell.
* В каталоге не должно быть ни одного приложения. Любые приложения должны быть удалены до того, как каталог может быть удален.
* Не может быть многофакторных поставщиков аутентификации, связанных с каталогом.
* С каталогом не должны быть связаны подписки на любые службы Microsoft Online Services, такие как Microsoft Azure, Office 365 или Azure AD Premium. Например, если каталог по умолчанию создан в Azure, его невозможно удалить, если ваша подписка Azure по-прежнему использует его для проверки подлинности. Точно так же нельзя удалить каталог, если у другого пользователя есть связанная с ним подписка.

## <a name="delete-the-directory"></a>Удалить каталог

1. Войти в [центр администратора Ad Azure](https://aad.portal.azure.com) с учетной записью, которая является Глобальным администратором вашей организации.

2. Выберите **активный каталог Azure**.

3. Переключитесь на каталог, который вы хотите удалить.
  
   ![Подтвердить организацию перед удалянием](./media/directory-delete-howto/delete-directory-command.png)

4. Нажмите **Удалить каталог**.
  
   ![выберите команду для удаления организации](./media/directory-delete-howto/delete-directory-list.png)

5. Если ваш каталог не проходит одну или несколько проверок, вам предоставляется ссылка на более подробную информацию о том, как пройти. Когда клиент пройдет все проверки, нажмите **Удалить** для завершения процесса.

## <a name="if-you-cant-delete-the-directory"></a>Если вы не можете удалить каталог

При настройке каталога Azure AD вы, возможно, также активировали лицензионные подписки для вашей организации, такие как Azure AD Premium P2, Office 365 Business Premium или Enterprise Mobility и Security E5. Чтобы избежать случайной потери данных, нельзя удалить каталог до тех пор, пока подписки не будут полностью удалены. Подписки должны находиться в **состоянии, задеваемом,** чтобы позволить удаление каталога. **Просроченная** или **отмененная** подписка перемещается в состояние **инвалидов,** а заключительным этапом является **состояние Деprovisioned.**

В следующей таблице описано, что происходит, когда истекает срок действия пробной подписки Office 365 (не включая платную партнерскую подписку/CSP, соглашение Enterprise или корпоративные лицензии). Дополнительные сведения о жизненном цикле хранения данных и подписок в Office 365 см. в разделе [Что происходит с моими данными и доступом при завершении подписки Office 365 для бизнеса?](https://support.office.com/article/what-happens-to-my-data-and-access-when-my-office-365-for-business-subscription-ends-4436582f-211a-45ec-b72e-33647f97d8a3)

----- | ----- | -----
Активно (пробная версия — 30 дней) | Данные доступны для всех | Пользователи имеют обычный доступ к файлам Office 365 или приложениям<br>У админов есть нормальный доступ к центру и ресурсам Microsoft 365

## <a name="delete-a-subscription"></a>Удаление подписки

Подписку можно поместить в состояние **Деprovisioned,** чтобы удалить в течение трех дней с помощью центра админирования Microsoft 365.

1. Вопийте в [центре администратора Microsoft 365](https://admin.microsoft.com) с учетной записью, которая является глобальным администратором в вашей организации. Если вы пытаетесь удалить каталог "Contoso", который имеет первоначальный домен по admin@contoso.onmicrosoft.comумолчанию contoso.onmicrosoft.com, восвайтесь с UPN, например.

2. Предварительный просмотр нового центра microsoft 365, убедившись, что **новый центр админ** переключается включен.

   ![Предварительный просмотр нового опыта M365 админ-центра](./media/directory-delete-howto/preview-toggle.png)

3. После включения нового центра админ-центра необходимо отменить подписку, прежде чем вы сможете удалить ее. Выберите **Биллинг** и выберите **Продукты & услуги,** а затем выберите **Отмена подписки** для подписки, которая вы хотите отменить. Вы будете приведены к странице обратной связи.

   ![Выберите подписку для отмены](./media/directory-delete-howto/cancel-choose-subscription.png)

4. Заполните форму обратной связи и выберите **Подписку Cancel,** чтобы отменить подписку.

   ![Отмена команды в предварительном просмотре подписки](./media/directory-delete-howto/cancel-command.png)

5. Теперь вы можете удалить подписку. Выберите **Удалить** для подписки, которая требуется удалить. Если вы не можете найти подписку на странице **Услуги & Продуктов,** убедитесь, что у вас есть **статус подписки,** установленный для **всех.**

   ![Удаление связи для удаления подписки](./media/directory-delete-howto/delete-command.png)

6. Выберите **подписку Удалить подписку** и принять условия. Все данные будут безвозвратно удалены в течение трех дней. Вы можете [активировать подписку](https://docs.microsoft.com/office365/admin/subscriptions-and-billing/reactivate-your-subscription?view=o365-worldwide) в течение трех дней, если передумаете.
  
   ![тщательно читать условия](./media/directory-delete-howto/delete-terms.png)

7. Теперь состояние подписки изменилось, и подписка помечена для удаления. Подписка входит в состояние **Отозвано** через 72 часа.

8. После удаления подписки в каталоге и 72 часа, вы можете снова войти в центр админа Azure Ad, и не должно быть никаких необходимых действий и никаких подписок, блокирующих удаление каталога. Вы должны иметь возможность успешно удалить каталог Azure AD.
  
   ![пройденная проверка подписки на экране удаления](./media/directory-delete-howto/delete-checks-passed.png)

## <a name="i-have-a-trial-subscription-that-blocks-deletion"></a>У меня есть пробная подписка, которая блокирует удаление

Есть [продукты самообслуживания регистрации,](https://docs.microsoft.com/office365/admin/misc/self-service-sign-up?view=o365-worldwide) такие как Microsoft Power BI, Службы управления правами, Microsoft Power Apps, или Dynamics 365, отдельные пользователи могут зарегистрироваться через Office 365, который также создает гостевого пользователя для проверки подлинности в каталоге Azure AD. Эти продукты самообслуживания блокируют удаления каталогов до тех пор, пока они не будут полностью удалены из каталога, чтобы избежать потери данных. Они могут быть удалены только админом Azure AD независимо от того, зарегистрировался ли пользователь индивидуально или был назначен продукт.

Есть два типа самообслуживания регистрации продуктов в том, как они назначаются: 

* Назначение на уровне org-level: Админ Azure AD назначает продукт всей организации, и пользователь может активно использовать службу с этим назначением орг-уровня, даже если он не лицензирован по отдельности.
* Назначение уровня пользователя: Индивидуальный пользователь во время регистрации самообслуживания по существу назначает продукт себе без админа. Как только организация становится управляемой администратором (см. [Администратор поглощения неуправляемого каталога,](domains-admin-takeover.md)то администратор может непосредственно назначить продукт пользователям без регистрации самообслуживания.  

Когда вы начинаете удаление продукта регистрации самообслуживания, действие навсегда удаляет данные и удаляет весь пользовательский доступ к службе. Любой пользователь, который был назначен предложение индивидуально или на уровне организации, затем блокируется от входе или доступа к любым существующим данным. Если вы хотите предотвратить потерю данных с помощью продукта регистрации самообслуживания, такого как [панели мониторинга Microsoft Power BI](https://docs.microsoft.com/power-bi/service-export-to-pbix) или [конфигурация политики управления правами,](https://docs.microsoft.com/azure/information-protection/configure-policy#how-to-configure-the-azure-information-protection-policy)убедитесь, что данные будут резервироваться и сохраняться в другом месте.

Для получения дополнительной информации о доступных в настоящее время продуктов самообслуживания регистрации и услуг, [см.](https://docs.microsoft.com/office365/admin/misc/self-service-sign-up?view=o365-worldwide#available-self-service-programs)

В следующей таблице описано, что происходит, когда истекает срок действия пробной подписки Office 365 (не включая платную партнерскую подписку/CSP, соглашение Enterprise или корпоративные лицензии). Дополнительные сведения о жизненном цикле хранения данных и подписок в Office 365 см. в разделе [Что происходит с моими данными и доступом при завершении подписки Office 365 для бизнеса?](https://docs.microsoft.com/office365/admin/subscriptions-and-billing/what-if-my-subscription-expires?view=o365-worldwide)

Состояние продукта | Данные | Доступ к данным
------------- | ---- | --------------
Активно (пробная версия — 30 дней) | Данные доступны для всех | Пользователи имеют нормальный доступ к продукту регистрации самообслуживания, файлам или приложениям<br>У админов есть нормальный доступ к центру и ресурсам Microsoft 365
Deleted | Данные удалены | Пользователи не могут получить доступ к продукту регистрации самообслуживания, файлам или приложениям<br>Админы могут получить доступ к центру microsoft 365 для покупки и управления другими подписками

## <a name="how-can-i-delete-a-self-service-sign-up-product-in-the-azure-portal"></a>Как удалить продукт регистрации самообслуживания на портале Azure?

Вы можете поместить продукт регистрации самообслуживания, такой как Microsoft Power BI или Azure Rights Management Services, в состояние **удаления,** чтобы быть немедленно удаленным на портале Azure AD.

1. Войти на [администратора Azure Ad](https://aad.portal.azure.com/#blade/Microsoft_AAD_IAM/ActiveDirectoryMenuBlade/Overview) с учетной записью, которая является глобальным администратором в организации. Если вы пытаетесь удалить каталог "Contoso", который имеет первоначальный домен по admin@contoso.onmicrosoft.comумолчанию contoso.onmicrosoft.com, зарегистрируйтесь с UPN, например.

2. Выберите **Лицензии,** а затем выберите **продукты регистрации самообслуживания.** Вы можете увидеть все продукты самообслуживания регистрации отдельно от подписки на место. Выберите продукт, который вы хотите удалить навсегда. Вот пример в Microsoft Power BI:

    ![имя пользователя неправильно введено или не найдено](./media/directory-delete-howto/licenses-page.png)

3. Выберите **Удалить,** чтобы удалить продукт и принять условия, что данные удаляются немедленно и безвозвратно. Это действие удаления удалит всех пользователей и удалит доступ организации к продукту. Нажмите "Да", чтобы двигаться вперед с удалением.  

    ![имя пользователя неправильно введено или не найдено](./media/directory-delete-howto/delete-product.png)

4. При выборе **Да**будет инициировано удаление продукта самообслуживания. Существует уведомление, которое сообщит вам об удалении в процессе.  

    ![имя пользователя неправильно введено или не найдено](./media/directory-delete-howto/progress-message.png)

5. Теперь состояние продукта регистрации самообслуживания изменено на **Удаленное.** При обновлении страницы продукт должен быть удален со **страницы продуктов регистрации самообслуживания.**  

    ![имя пользователя неправильно введено или не найдено](./media/directory-delete-howto/product-deleted.png)

6. После удаления всех продуктов вы можете снова войти в центр админа Azure AD, и не должно быть никаких необходимых действий и никаких продуктов, блокирующих удаление каталога. Вы должны иметь возможность успешно удалить каталог Azure AD.

    ![имя пользователя неправильно введено или не найдено](./media/directory-delete-howto/delete-organization.png)

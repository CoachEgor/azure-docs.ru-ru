# <a name="set-up-self-service-group-management-in-azure-active-directory"></a>Настройка самостоятельного управления группами в Azure Active Directory 

Можно разрешить пользователям создавать и управлять ими, свои собственные группы безопасности или группы Office 365 в Azure Active Directory (Azure AD). Владелец группы можно утверждать или отклонять запросы на членство и можно делегировать управление членством. Функции управления группами самообслуживания недоступны для групп безопасности с включенной почтой или списки рассылки.

## <a name="self-service-group-membership-defaults"></a>Значения по умолчанию членства группы самообслуживания

Когда создаются группы безопасности на портале Azure или с помощью Azure AD PowerShell, только владельцы группы могут обновлять членство. Группы безопасности, созданные в [панели доступа](https://account.activedirectory.windowsazure.com/r#/joinGroups) и доступны всем группам Office 365 для соединения для всех пользователей, Утверждено владельцем или утверждено автоматически. На панели доступа можно изменить параметры членства при создании группы.

## <a name="self-service-group-management-scenarios"></a>Сценарии управления группами самообслуживания

* **Делегированное управление группами**. В качестве примера можно привести администратора, который управляет доступом к приложению SaaS, которым пользуется компания. Управление этими правами доступа становится трудоемким, поэтому администратор запрашивает у владельца компании разрешение на создание новой группы. Администратор предоставляет доступ к приложению новой группе и помещает в эту группу всех пользователей, которые имеют доступ к приложению. Впоследствии владелец компании может добавить несколько пользователей, и им автоматически будет предоставлен доступ к приложению. Владельцу компании не нужно ждать, пока администратор выполнит свою работу. Если администратор предоставляет те же разрешения для помощника в другой бизнес-группы, то этот пользователь также может управлять доступом для собственных членов группы. Ни владелец компании, ни manager можно просматривать и управлять друг друга в группах. Администратор может просматривать всех пользователей, имеющих доступ к приложению, и блокировать права доступа при необходимости.
* **Самостоятельное управление группами**. В качестве примера можно привести двух пользователей, у каждого из которых есть свой сайт SharePoint Online. Каждый пользователь настраивает свой сайт самостоятельно. Они хотели бы предоставить группам взаимный доступ к своим сайтам. Для этого они создают одну группу в Azure AD, а затем в SharePoint Online каждый из них выбирает одну и ту же группу для предоставления доступа к сайтам. Когда кому-нибудь требуется доступ, он запрашивает его на панели доступа, и после утверждения он автоматически получает доступ к обоим сайтам SharePoint Online. Позже один из них решает, что все пользователи сайта также должны получить доступ к определенному приложению SaaS. Администратор приложения SaaS может добавить права доступа к сайту SharePoint Online для приложения. Теперь любые утвержденные им запросы предоставят доступ к двум сайтам SharePoint Online и приложению SaaS.

## <a name="make-a-group-available-for-user-self-service"></a>Включение функции самообслуживания для пользователей группы

1. Войдите в [центр администрирования Azure AD](https://aad.portal.azure.com) с помощью учетной записи глобального администратора каталога.
2. Последовательно выберите элементы **Пользователи и группы** и **Настройки группы**.
3. Задайте для параметра **Управление группами самообслуживания включено** значение **Да**.
4. Затем задайте для параметра **Пользователи могут создавать группы безопасности** или **Пользователи могут создавать группы Office 365** значение **Да**.
   * Если эти параметры включены, всем пользователям в каталоге разрешено создавать группы безопасности и добавлять в них пользователей. Эти новые группы также будут отображаться на панели доступа для всех остальных пользователей. Пользователи смогут создавать запросы на присоединение к таким группам, если это разрешено параметром политики для группы. 
   * Если эти параметры отключены, пользователи не могут создавать новые группы и изменять существующие группы, владельцами которых они являются. Однако они могут управлять членством в этих группах и утверждать запросы на присоединение к группам от других пользователей.

С помощью параметров **Пользователи, которые могут управлять группами безопасности** и **Пользователи, которые могут управлять группами Office 365** можно также более точно управлять доступом благодаря возможностям самостоятельного управления группами, предусмотренными для пользователей. Если параметр **Users can create groups** (Пользователи могут создавать группы) включен, всем пользователям в клиенте разрешено создавать группы и добавлять в них участников. Нельзя указать отдельных пользователей, которые могут создавать собственные группы. Можно указать отдельных пользователей только для внесения владельцем группы другой участник группы.

Установив **пользователей, которые могут использовать Самостоятельное управление группами безопасности** и **пользователей, которые могут управлять группами Office 365** для **Да**, вы разрешите всем пользователям в клиенте для создания новых группы.

Кроме того, вы можете использовать параметры **Группа, которая может управлять группами безопасности** и **Группа, которая может управлять группами Office 365**, чтобы указать отдельную группу, участники которой могут использовать самостоятельное управление.

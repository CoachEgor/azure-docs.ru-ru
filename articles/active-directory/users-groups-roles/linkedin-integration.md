# <a name="integrate-linkedin-account-connections-in-azure-active-directory"></a>Интеграция подключений учетных записей LinkedIn в Azure Active Directory

Вы можете разрешить пользователям в организации получать доступ к своим подключениям LinkedIn в некоторых приложениях Майкрософт. Никакие данные не предоставляются, пока пользователи не будут иметь согласие на подключение своих учетных записей. Вы можете интегрировать свою организацию в [центр администрирования](https://aad.portal.azure.com)Azure Active Directory (Azure AD).

> [!IMPORTANT]
> В настоящее время параметр подключения учетной записи LinkedIn выполняется в организациях Azure AD. После развертывания в Организации она включается по умолчанию.
> 
> Исключения:
> * Этот параметр недоступен для клиентов, использующих Microsoft Cloud for US Government, Microsoft Cloud Germany, а также служб Azure и Office 365, обслуживаемых 21Vianet в Китае.
> * Этот параметр по умолчанию отключен для клиентов, подготовленных в Германии. Обратите внимание, что этот параметр недоступен для клиентов, использующих Microsoft Cloud Germany.
> * Этот параметр по умолчанию отключен для клиентов, подготовленных во Франции.
>
> После включения подключений к учетной записи LinkedIn в Организации подключения учетной записи работают после согласия пользователей с доступом к данным компании от их имени. Сведения о параметре согласия пользователя см. [в разделе Удаление доступа пользователя к приложению](https://docs.microsoft.com/azure/active-directory/application-access-assignment-how-to-remove-assignment).

## <a name="enable-linkedin-account-connections-in-the-azure-portal"></a>Включение подключений к учетной записи LinkedIn в портал Azure

Подключения к учетной записи LinkedIn можно включить только для тех пользователей, которые должны иметь доступ, от всей Организации до выбранных пользователей в Организации.

1. Войдите в [центр администрирования Azure AD](https://aad.portal.azure.com/) с помощью учетной записи глобального администратора организации Azure AD.
1. Выберите **Пользователи**.
1. В колонке **Пользователи** выберите **Параметры пользователей**.
1. В разделе **подключения к учетной записи LinkedIn**разрешите пользователям подключать свои учетные записи для доступа к своим подключениям LinkedIn в некоторых приложениях Майкрософт. Никакие данные не предоставляются, пока пользователи не будут иметь согласие на подключение своих учетных записей.

    * Выберите **"Да"** , чтобы включить службу для всех пользователей в Организации
    * Выберите **выбранную группу** , чтобы включить службу только для группы выбранных пользователей в Организации
    * Выберите **нет** , чтобы снять согласие со всех пользователей в Организации

    ![Интеграция подключений к учетной записи LinkedIn в Организации](./media/linkedin-integration/linkedin-integration.png)

1. Когда все будет готово, нажмите кнопку **сохранить** , чтобы сохранить параметры.

> [!Important]
> Интеграция с LinkedIn не включена для пользователей, пока не будет предоставлено согласие на подключение своих учетных записей. При включении подключений учетной записи для пользователей доступ к данным не предоставляется.

### <a name="assign-selected-users-with-a-group"></a>Назначить выбранных пользователей группой
Мы заменили параметр "Selected", указывающий список пользователей с возможностью выбора группы пользователей, чтобы можно было включить возможность подключения учетных записей LinkedIn и Майкрософт для одной группы, а не для нескольких отдельных пользователей. Если вы не включили подключения к учетной записи LinkedIn для выбранных отдельных пользователей, ничего делать не нужно. Если вы ранее включили подключения к учетной записи LinkedIn для выбранных отдельных пользователей, необходимо:

1. Получение текущего списка отдельных пользователей
1. Перемещение всех включенных в данный момент отдельных пользователей в группу
1. Используйте группу из предыдущей в качестве выбранной группы в параметре подключения учетной записи LinkedIn в центре администрирования Azure AD.

> [!NOTE]
> Даже если вы не перемещаете выбранных отдельных пользователей в группу, они по-прежнему могут просматривать сведения LinkedIn в приложениях Майкрософт.

### <a name="get-the-current-list-of-selected-users"></a>Получение текущего списка выбранных пользователей

1. Войдите в Microsoft 365 с помощью учетной записи администратора.
1. Перейдите на сайт https://linkedinselectedusermigration.azurewebsites.net/. Вы увидите список пользователей, которые выбраны для подключений к учетной записи LinkedIn.
1. Экспортируйте список в CSV-файл.

### <a name="move-the-currently-selected-individual-users-to-a-group"></a>Переместить выбранных пользователей в группу

1. Запуск PowerShell
1. Установите модуль Azure AD, выполнив `Install-Module AzureAD`
1. Выполните следующий скрипт:

  ``` PowerShell
  $groupId = "GUID of the target group"
  
  $users = Get-Content 
  Path to the CSV file
  
  $i = 1
  foreach($user in $users} { Add-AzureADGroupMember -ObjectId $groupId -RefObjectId $user ; Write-Host $i Added $user ; $i++ ; Start-Sleep -Milliseconds 10 }
  ```

Чтобы использовать группу из шага 2 в качестве выбранной группы в параметре подключения учетной записи LinkedIn в центре администрирования Azure AD, см. раздел [Включение подключений к учетной записи LinkedIn в портал Azure](#enable-linkedin-account-connections-in-the-azure-portal).

## <a name="use-group-policy-to-enable-linkedin-account-connections"></a>Использование групповая политика для включения подключений к учетной записи LinkedIn

1. Скачайте [файлы административных шаблонов Office 2016 (ADMX/ADML)](https://www.microsoft.com/download/details.aspx?id=49030).
1. Извлеките файлы **ADMX** и скопируйте их в центральное хранилище.
1. Откройте средство управления групповыми политиками.
1. Создайте объект групповой политики со следующими параметрами: **Конфигурация пользователя** > **Административные шаблоны** > **Microsoft Office 2016** > **Разное** > **Show LinkedIn features in Office applications** (Показывать функции LinkedIn в приложениях Office).
1. Выберите **Включено** или **Отключено**.
  
   Состояние | Результат
   ------ | ------
   **Включено** | Параметр **Show LinkedIn features in Office applications** (Показывать функции LinkedIn в приложениях Office) в Office 2016 включен. Пользователи в вашей организации могут использовать функции LinkedIn в приложениях Office 2016.
   **отключено**. | Параметр **Show LinkedIn features in Office applications** (Показывать функции LinkedIn в приложениях Office) в Office 2016 отключен, и пользователи не могут изменить это. Пользователи в вашей организации не могут использовать функции LinkedIn в приложениях Office 2016.

Эта групповая политика касается только приложений Office 2016 для локальных компьютеров. Если пользователи отключают LinkedIn в приложениях Office 2016, они по-прежнему смогут просматривать функции LinkedIn в Office 365.

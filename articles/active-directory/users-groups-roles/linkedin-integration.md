# <a name="integrate-linkedin-account-connections-in-azure-active-directory"></a>Интеграция подключений к учетным записям LinkedIn в Active Directory Azure

Вы можете разрешить пользователям вашей организации получать доступ к своим соединениям LinkedIn в некоторых приложениях Майкрософт. Данные не передаются до тех пор, пока пользователи не согласятся на подключение своих учетных записей. Вы можете интегрировать организацию в [центр ещёвого](https://aad.portal.azure.com)элемента Azure Active Directory (Azure AD).

> [!IMPORTANT]
> Настройка подключения к учетной записи LinkedIn в настоящее время внедряется в организации Azure AD. Когда он развернут в организации, он включен по умолчанию.
> 
> Исключения:
> * Этот параметр недоступен для клиентов, использующих Microsoft Cloud for US Government, Microsoft Cloud Germany, а также служб Azure и Office 365, обслуживаемых 21Vianet в Китае.
> * Этот параметр по умолчанию отключен для клиентов, подготовленных в Германии. Обратите внимание, что этот параметр недоступен для клиентов, использующих Microsoft Cloud Germany.
> * Этот параметр по умолчанию отключен для клиентов, подготовленных во Франции.
>
> После включения подключений к учетной записи LinkedIn для вашей организации соединения с учетной записью работают после того, как пользователи дают согласие на доступ к данным компании от их имени. Для получения информации о настройках согласия пользователя см. [Как удалить доступ пользователя к приложению.](https://docs.microsoft.com/azure/active-directory/application-access-assignment-how-to-remove-assignment)

## <a name="enable-linkedin-account-connections-in-the-azure-portal"></a>Включить подключение учетной записи LinkedIn на портале Azure

Вы можете включить подключение учетной записи LinkedIn только для пользователей, к которых вы хотите иметь доступ, от всей организации до только выбранных пользователей в вашей организации.

1. Войти на [админ-центр Azure AD](https://aad.portal.azure.com/) с учетной записью, которая является глобальным админом для организации Azure AD.
1. Выберите **пользователей**.
1. В колонке **Пользователи** выберите **Параметры пользователей**.
1. Под **подключением к учетным записям LinkedIn**позволяет пользователям подключать свои учетные записи для доступа к своим соединениям LinkedIn в некоторых приложениях Майкрософт. Данные не передаются до тех пор, пока пользователи не согласятся на подключение своих учетных записей.

    * Выберите **«Да»,** чтобы включить службу для всех пользователей вашей организации
    * Выберите **выбранную группу,** чтобы включить службу только для группы выбранных пользователей в вашей организации
    * Выберите **Нет** для отзыва согласия от всех пользователей вашей организации

    ![Интеграция подключений к учетным записям LinkedIn в организацию](./media/linkedin-integration/linkedin-integration.png)

1. Когда вы закончите, выберите **Сохранить,** чтобы сохранить настройки.

> [!Important]
> Интеграция LinkedIn не полностью включена для пользователей до тех пор, пока они не согласятся на подключение своих учетных записей. При включании подключения к учетной записи для пользователей данные не передаются.

### <a name="assign-selected-users-with-a-group"></a>Назначить выбранных пользователей группой
Мы заменили опцию "Выбранный", который определяет список пользователей с возможностью выбора группы пользователей, так что вы можете включить возможность подключения LinkedIn и Microsoft учетных записей для одной группы, а не многих отдельных пользователей. Если у вас нет подключений к учетной записи LinkedIn для отдельных пользователей, вам не нужно ничего делать. Если ранее вы включили соединения учетных записей LinkedIn для отдельных пользователей, следует:

1. Получить текущий список отдельных пользователей
1. Перемещение включенных в настоящее время пользователей в группу
1. Используйте группу из предыдущего в качестве выбранной группы в настройках подключения linkedIn в центре админ-аминота Azure Ad.

> [!NOTE]
> Даже если вы не переместите выбранных в настоящее время отдельных пользователей в группу, они все равно могут видеть информацию LinkedIn в приложениях Майкрософт.

### <a name="get-the-current-list-of-selected-users"></a>Получить текущий список выбранных пользователей

1. Войти в Microsoft 365 с учетной записью администратора.
1. Перейдите к https://linkedinselectedusermigration.azurewebsites.net/. Вы увидите список пользователей, выбранных для подключения к учетной записи LinkedIn.
1. Экспорт ировать список в файл CSV.

### <a name="move-the-currently-selected-individual-users-to-a-group"></a>Перемещение выбранных в настоящее время отдельных пользователей в группу

1. Запуск PowerShell
1. Установка модуля Azure AD при запуске`Install-Module AzureAD`
1. Выполните следующий скрипт:

  ``` PowerShell
  $groupId = "GUID of the target group"
  
  $users = Get-Content 
  Path to the CSV file
  
  $i = 1
  foreach($user in $users} { Add-AzureADGroupMember -ObjectId $groupId -RefObjectId $user ; Write-Host $i Added $user ; $i++ ; Start-Sleep -Milliseconds 10 }
  ```

Чтобы использовать группу со вон два в качестве выбранной группы в настройках подключения к учетной записи LinkedIn в центре админации Azure, [см.](#enable-linkedin-account-connections-in-the-azure-portal)

## <a name="use-group-policy-to-enable-linkedin-account-connections"></a>Используйте групповую политику для включения подключений к учетной записи LinkedIn

1. Скачайте [файлы административных шаблонов Office 2016 (ADMX/ADML)](https://www.microsoft.com/download/details.aspx?id=49030).
1. Извлеките файлы **ADMX** и скопируйте их в центральное хранилище.
1. Откройте средство управления групповыми политиками.
1. Создайте объект групповой политики со следующим параметром: **Пользовательские настройки** > **административных шаблонов** > Microsoft Office**2016** > **Разные** > **функции отображают LinkedIn в приложениях Office.**
1. Выберите **Включено** или **Отключено**.
  
   Штат | Действие
   ------ | ------
   **Включен** | Параметр **Show LinkedIn features in Office applications** (Показывать функции LinkedIn в приложениях Office) в Office 2016 включен. Пользователи вашей организации могут использовать функции LinkedIn в своих приложениях Office 2016.
   **Отключен** | Параметр **Show LinkedIn features in Office applications** (Показывать функции LinkedIn в приложениях Office) в Office 2016 отключен, и пользователи не могут изменить это. Пользователи в вашей организации не могут использовать функции LinkedIn в приложениях Office 2016.

Эта групповая политика касается только приложений Office 2016 для локальных компьютеров. Если пользователи отдают LinkedIn в приложениях Office 2016, они все равно могут видеть функции LinkedIn в Office 365.

# <a name="bulk-create-users-in-azure-active-directory"></a>Массовые создания пользователей в Active Directory Azure

Активный каталог Azure (Azure AD) поддерживает массовые операции пользователей, массовое приглашение гостей и загрузку списков пользователей, групп и членов группы.

## <a name="required-permissions"></a>Необходимые разрешения

Для того, чтобы навалом создавать пользователей на портале администрирования, вы должны быть подписаны в качестве глобального администратора или администратора пользователя.

## <a name="to-create-users-in-bulk"></a>Для создания пользователей оптом

1. [Восвай в организацию Azure AD](https://aad.portal.azure.com) с учетной записью, которая является администратором пользователя в организации.
1. В Azure AD выберите **пользователей** > **Bulk create.**
1. На странице **массового создания пользователя** выберите **Download,** чтобы получить допустимый файл значений, разделенных запятой (CSV), свойств пользователя, а затем добавьте пользователей, которые вы хотите создать.

   ![Выберите локальный файл CSV, в котором вы перечисляете пользователей, которых вы хотите добавить](./media/users-bulk-add/upload-button.png)

1. Откройте файл CSV и добавьте строку для каждого пользователя, который вы хотите создать. Единственными требуемыми значениями являются **имя,** **основное имя пользователя,** **первоначальный пароль** и **блок войти (Да/Нет).** Затем сохраните файл.

   ![Файл CSV содержит имена и идентивы пользователей для создания](./media/users-bulk-add/add-csv-file.png)

1. На странице **массового создания пользователя,** под загрузить файл CSV, просматривать файл. При выборе файла и нажмите **Отправить,** проверка файла CSV начинается.
1. После проверки содержимого файла вы увидите **успешно загруженный файл.** Если будут обнаружены ошибки, их нужно исправить для отправки задания.
1. Когда файл проходит проверку, выберите **отправку,** чтобы начать операцию навалом Azure, которая импортирует новых пользователей.
1. Когда операция импорта завершится, вы увидите уведомление о состоянии задания основной операции.

Если есть ошибки, вы можете загрузить и просмотреть файл результатов на странице **результатов операции Bulk.** Файл содержит причину каждой ошибки.

## <a name="check-status"></a>Проверка состояния

Вы можете увидеть состояние всех ожидающих натковых запросов на странице **результатов операции Bulk.**

   [![](media/users-bulk-add/bulk-center.png "Check create status in the Bulk Operations Results page")](media/users-bulk-add/bulk-center.png#lightbox)

Далее можно проверить, что созданные вами пользователи существуют в организации Azure AD либо на портале Azure, либо с помощью PowerShell.

## <a name="verify-users-in-the-azure-portal"></a>Проверка пользователей на портале Azure

1. [Войти на администратора Azure Ad](https://aad.portal.azure.com) с учетной записью, которая является администратором пользователя в организации.
1. В области навигации выберите **Azure Active Directory**.
1. В разделе **Управление** выберите **Пользователи**.
1. Под **показать**, выберите **все пользователи** и убедитесь, что пользователи, которые вы создали перечислены.

### <a name="verify-users-with-powershell"></a>Проверка пользователей с помощью PowerShell

Выполните следующую команду:

``` PowerShell
Get-AzureADUser -Filter "UserType eq 'Member'"
```

Вы должны видеть, что пользователи, которые вы создали, перечислены.

## <a name="bulk-import-service-limits"></a>Ограничения массовых импортных услуг

Каждое массовое действие для создания пользователей может работать до одного часа. Это позволяет навалом создать не менее 50 000 пользователей.

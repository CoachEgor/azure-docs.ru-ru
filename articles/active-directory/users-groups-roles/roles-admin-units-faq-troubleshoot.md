# <a name="troubleshooting-and-faq-for-administrative-units-in-azure-active-directory"></a>Устранение проблем и часто задаваемых вопросов для административных подразделений в Active Directory Azure

Для более детального административного управления в Active Directory Azure (Azure AD) можно назначить пользователям роль Azure AD с областью, ограниченной одним или несколькими административными единицами (AUs). Вы можете найти пример ы powerShell https://docs.microsoft.com/powershell/azure/active-directory/working-with-administrative-units?view=azureadps-2.0скрипты для общих задач на .

## <a name="frequently-asked-questions"></a>Часто задаваемые вопросы

**В: Я не могу создать административную единицу**

**A:** Только Глобальный администратор или привилегированный администратор ролей может создать административную единицу в Azure AD. Убедитесь, что пользователю, пытаясь создать административное подразделение, присваивается либо роль Глобального администратора, либо привилегировая роль администратора ролей.

**В: Я добавил группу в административное подразделение, но члены группы до сих пор не явились в административное подразделение**

**A:** При добавлении группы в административное подразделение это не приводит к добавлению всех членов группы в административное подразделение. Пользователи должны быть непосредственно назначены административной единице.

**В: Я только что добавил / удалил члена административного подразделения, и он все еще появляется в uI**

**A:** Иногда обработка добавления или удаления одного или нескольких членов административного подразделения может занять несколько минут, чтобы отразить на странице **административных единиц.** Вы можете подождать несколько минут, чтобы отразить его под административными единицами. Кроме того, можно перейти непосредственно к свойствам связанного ресурса и посмотреть, завершено ли действие. Для получения дополнительной информации о пользователях и группах [List administrative units for a group](roles-admin-units-add-manage-groups.md)в AUs [см.](roles-admin-units-add-manage-users.md)

**В: Как делегированный администратор паролей в административном блоке, я не могу сбросить пароль конкретного пользователя**

**A:** Администратор, назначенный над административным подразделением, может сбросить пароль только для пользователей, назначенных в ваше административное подразделение. Убедитесь, что пользователь, для которого сбросить пароль не удается принадлежит к административным единицам, над которыми вы были назначены роли. Если пользователь принадлежит к той же административной единице и вы все еще не можете сбросить пароль пользователя, проверьте роли, которые назначены пользователю. Чтобы предотвратить повышение привилегий, администратор с охватом административного блока не может сбросить пароль пользователя, назначенного роли с общеорганизационным охватом.

**В: Зачем нужны административные единицы? Разве мы не могли использовать группы безопасности как способ определения области?**

**A:** Группы безопасности имеют существующую модель назначения и авторизации. Например, администратор пользователей может управлять членством всех групп безопасности в организации Azure AD, например, использовать группы для управления доступом к приложениям, таким как Salesforce. Администратор пользователя не должен иметь возможность управлять самой моделью делегирования, что было бы результатом, если группы безопасности были расширены для поддержки сценариев «группирования ресурсов». Административные единицы, такие как организационные единицы в Active Directory Windows Server, предназначены для обеспечения способа администрирования сферы действия широкого спектра объектов каталога. Сами группы безопасности могут быть членами областей ресурсов. Использование групп безопасности для определения набора групп безопасности, которыми может управлять администратор, может ввести в заблуждение.

**В: Что значит добавить группу в административную единицу?**

**A:** Добавление группы в административное подразделение приводит саму группу в область управления любого администратора пользователя, который также относится к этому блоку администратора. Администрация административного подразделения может управлять именем и членством самой группы. Он не предоставляет адм-менеджеру пользователя для административной единицы никакого разрешения на управление пользователями группы (например, сбросить их пароли). Чтобы предоставить Администратору Пользователя возможность управлять пользователями, пользователи должны быть прямыми членами административного блока.

**В: Может ли ресурс (пользователь или группа) быть членом более чем одной административной единицы?**

**A:** Да, ресурс может быть членом более чем одной административной единицы. Ресурсом могут управлять все общеорганизационные и административные аммины, имеющие разрешения на ресурс.

**В: Доступны ли административные единицы в организациях B2C?**

**A:** Нет, административные единицы недоступны для организаций B2C.

**В: Поддерживаются ли вложенные административные единицы?**

**A:** Nested административные единицы не поддерживаются.

**В: Поддерживаются ли административные единицы в PowerShell и Graph API?**

**A:** Да. Поддержка административных единиц существует в [документации PowerShell cmdlet](https://docs.microsoft.com/powershell/module/Azuread/?view=azureadps-2.0-preview) и [образцах скриптов,](https://docs.microsoft.com/powershell/azure/active-directory/working-with-administrative-units?view=azureadps-2.0-preview)а поддержка — в графике Microsoft для [типа ресурсов административного блока](https://developer.microsoft.com/graph/docs/api-reference/beta/resources/administrativeunit)Unit.

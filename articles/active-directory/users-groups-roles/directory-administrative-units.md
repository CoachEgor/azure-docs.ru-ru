# <a name="administrative-units-management-in-azure-active-directory-preview"></a>Управление административными подразделениями в Active Directory Azure (предварительный просмотр)

В этой статье описаны административные единицы в Active Directory Azure (Azure AD). Административная единица — это ресурс Azure AD, который может быть контейнером для других ресурсов Azure AD. В этом выпуске предварительного просмотра административное подразделение может содержать только пользователей и группы.

Административные единицы позволяют выдавать разрешения админова, которые ограничены отделом, регионом или другим сегментом организации, который вы определяете. Можно использовать административные единицы для делегирования разрешений региональным администраторам или для настройки политики на гранулированном уровне. Например, администратор учетной записи пользователя может обновлять информацию о профиле, сбросить пароли и назначать лицензии пользователям только в их административном подразделении.

 Например, делегирование региональным специалистам по поддержке роли [Администратора службы поддержки](directory-assign-admin-roles.md#helpdesk-administrator) ограничивается управлением только пользователями в поддерживаемом ими регионе.

## <a name="deployment-scenario"></a>Сценарий развертывания

Ограничение административной сферы с использованием административных единиц может быть полезным в организациях, которые состоят из независимых подразделений любого рода. Рассмотрим пример крупного университета, состоящего из многих автономных школ (Школа бизнеса, Инженерная школа и т.д.), в каждом из которых есть команда ИТ-администрирования, которые контролируют доступ, управляют пользователями и устанавливают политики для своей школы. Центральный администратор может:

- Создание роли с административными разрешениями только для пользователей Azure AD в административном подразделении бизнес-школы
- Создание административного подразделения для Школы бизнеса
- Заполнить подразделение админ только с учащимися бизнес-школы и персоналом
- Добавьте ИТ-команду бизнес-школы к роли с их областью

### <a name="planning-your-administrative-units"></a>Планирование административных подразделений

Административные единицы могут использоваться для логический группы ресурсов Azure AD. Например, для организации, ИТ-отдел которой разбросан по всему миру, возможно, имеет смысл создать административные единицы, определяющие эти географические границы. В другом сценарии, когда многонациональная организация имеет различные "суборганизации", которые являются полуавтономными в операциях, каждая подорганизация может быть представлена административной единицей.

Критерии, по которым создаются административные единицы, будут руководствоваться уникальными требованиями организации. Административные единицы являются распространенным способом определения структуры в службах M365. Мы рекомендуем вам подготовить административные единицы с их использованием в службах M365 в виду. Вы можете получить максимальную стоимость из административных единиц, когда вы можете связать общие ресурсы через M365 под административным блоком.

Можно ожидать, что создание административных подразделений в организации будет проходить следующие этапы:

1. Первоначальное утверждение: Ваша организация начнет создавать административные единицы на основе первоначальных критериев, и количество административных единиц будет увеличиваться по мере уточнения критериев.
1. Обрезка: После того, как критерии будут четко определены, административные единицы, которые больше не требуются, будут удалены.
1. Стабилизация: Ваша организационная структура четко определена, и количество административных единиц не изменится существенно за короткие сроки.

## <a name="currently-supported-scenarios"></a>Поддерживаемые в настоящее время сценарии

Глобальные администраторы или привилегированные администраторы ролей могут использовать портал Azure AD для создания административных единиц, добавлять пользователей в состав административных подразделений, а затем назначать ИТ-персоналу административные функции администратора. Затем административные административные адм-армины могут использовать портал Office 365 для базового управления пользователями в своих административных подразделениях.

Кроме того, группы могут быть добавлены в качестве членов административного подразделения, а администратор группы, обрамленный администратором подразделения, может управлять ими с помощью PowerShell, Microsoft Graph и портала Azure AD.

Административные единицы применяют сферу применения только к разрешениям управления. Они не запрещают пользователям или администраторам использовать [свои пользовательские разрешения по умолчанию](../fundamentals/users-default-permissions.md) для просмотра других пользователей, групп или ресурсов за пределами административного блока. На портале Office 365 отфильтровываются пользователи, не входящие в административные подразделения, но можно просматривать других пользователей на портале Azure AD, PowerShell и других службах Майкрософт.

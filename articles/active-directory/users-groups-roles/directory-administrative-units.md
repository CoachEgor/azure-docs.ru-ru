# <a name="administrative-units-management-in-azure-active-directory-preview"></a>Управление административными единицами в Azure Active Directory (Предварительная версия)

В этой статье описываются административные единицы в Azure Active Directory (Azure AD). Административная единица — это ресурс Azure AD, который может быть контейнером для других ресурсов Azure AD. В этой предварительной версии административная единица может содержать только пользователей и группы.

Административные единицы позволяют предоставлять разрешения администратора, которые ограничены подразделением, регионом или другим определенным вами сегментом вашей организации. Административные единицы можно использовать для делегирования разрешений региональным администраторам или для установки политики на детализированном уровне. Например, администратор учетной записи пользователя может обновлять данные профиля, сбрасывать пароли и назначать лицензии только пользователям в административной единице.

 Например, делегирование специалистам по поддержке региональных стандартов ролью [администратора службы поддержки](directory-assign-admin-roles.md#helpdesk-administrator) ограничена доступом пользователей в поддерживаемом регионе.

## <a name="deployment-scenario"></a>Сценарий развертывания

Ограничение административной области с использованием административных единиц может быть полезно в организациях, состоящие из независимых подразделений любого типа. Рассмотрим пример большого университета, сопоставленного с множеством автономных школ («предприятие», «учебное заведение» и т. д.), у каждого из которых есть группа ИТ-администраторов, контролирующих доступ, Управление пользователями и Установка политик для своего учебного заведения. Администратор центра может:

- Создание роли с разрешениями администратора только для пользователей Azure AD в административной единице "бизнес-школьие"
- Создание административной единицы для вашего учебного заведения
- Заполните административную единицу только студентами и сотрудниками отрасли
- Добавление рабочей группы бизнес-учебного заведения в роль с их областью действия

## <a name="license-requirements"></a>Требования лицензий

Для использования административных единиц требуется лицензия на Azure Active Directory Premium для каждого административного администратора и Azure Active Directory Free лицензий для членов административной единицы. Дополнительные сведения см. в статье [Приступая к работе с Azure AD Premium](../fundamentals/active-directory-get-started-premium.md).

В этой предварительной версии можно управлять административными единицами с помощью портал Azure, командлетов PowerShell и сценариев, а также Microsoft Graph. Дополнительные сведения см. в документации:

- [Создание, удаление, заполнение и Добавление ролей в административные единицы](roles-admin-units-manage.md): завершение процедур
- [Работа с единицами администрирования](https://docs.microsoft.com/powershell/azure/active-directory/working-with-administrative-units?view=azureadps-2.0): работа с административными единицами с помощью PowerShell
- [Поддержка административного модуля](https://docs.microsoft.com/graph/api/resources/administrativeunit?view=graph-rest-beta). Подробная документация по Microsoft Graph для административных единиц.

### <a name="planning-your-administrative-units"></a>Планирование административных единиц

Административные единицы могут использоваться для логической группировки ресурсов Azure AD. Например, для Организации, в которой ИТ-отдел разбросана глобально, может иметь смысл создать административные единицы, определяющие эти географические границы. В другом сценарии, в котором многонациональная организация имеет разные "подорганизации", которые находятся частично автономно в операциях, каждая подорганизация может быть представлена административной единицей.

Критерии, по которым создаются административные единицы, будут посвящены уникальным требованиям Организации. Административные единицы — это распространенный способ определения структуры в M365 Services. Рекомендуется подготавливать административные единицы с учетом их использования в службах M365 Services. Вы можете получить максимальное значение из административных единиц, если вы можете связать общие ресурсы в M365 с административной единицей.

Вы можете дождаться создания административных единиц в Организации, чтобы пройти следующие этапы.

1. Первоначальное внедрение. Ваша организация начнет создавать административные единицы на основе начальных критериев, а количество административных единиц увеличится по мере уточнения условий.
1. Очистка. После того как критерии определены правильно, административные единицы, которые больше не требуются, будут удалены.
1. Стабилизации: структура Организации хорошо определена, и количество административных единиц не будет значительно меняться за короткие длительности.

## <a name="currently-supported-scenarios"></a>Поддерживаемые в настоящее время сценарии

Глобальные администраторы или Администраторы привилегированных ролей могут использовать портал Azure AD для создания административных единиц, добавления пользователей в качестве членов административных единиц, а затем назначать ИТ-сотрудникам административные роли администраторов с областью действия. Административные администраторы с областью действия "административный" могут затем использовать портал Office 365 для базового управления пользователями в своих административных единицах.

Кроме того, группы можно добавлять в качестве членов административной единицы, а администратор группы с областью действия администратора может управлять ими с помощью PowerShell, Microsoft Graph и портала Azure AD.

В приведенной ниже таблице описывается текущая поддержка сценариев административной единицы.

Добавление и удаление элементов административной единицы с помощью CSV-файла   |    Не поддерживается     |  Поддерживается   |    Нет планов для поддержки

Динамическое добавление и удаление элементов AU на основе атрибутов | Не поддерживается | Не поддерживается | Не поддерживается

> [!NOTE]
>
> Администраторы с областью административной единицы не могут управлять правилами членства в динамических группах.

Административные единицы применяют область только к разрешениям управления. Они не запрещают членам или администраторам использовать [разрешения пользователей по умолчанию](../fundamentals/users-default-permissions.md) для просмотра других пользователей, групп или ресурсов за пределами административной единицы. На портале Office 365 пользователи за пределами административных единиц с заданной областью отфильтровываются, но вы можете просматривать других пользователей на портале Azure AD, PowerShell и других службах Майкрософт.

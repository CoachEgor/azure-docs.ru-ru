# <a name="data-residency-and-customer-data-for-azure-multi-factor-authentication"></a>Резидентство данных и данные клиентов для многофакторной аутентификации Azure

Данные о клиентах хранятся Azure AD в географическом местоположении, основанном на адресе, предоставленном вашей организацией при подписке на службу Microsoft Online, такую как Office 365 и Azure. Для получения информации о том, где хранятся данные ваших клиентов, вы можете использовать раздел [Where is your data?](https://www.microsoft.com/trustcenter/privacy/where-your-data-is-located)

Облачный процесс многофакторной аутентификации Azure и многофакторного сервера аутентификации Azure и хранения определенного объема персональных данных и организационных данных. В этой статье излагается, что и где хранятся данные.

Следующие многофакторные действия аутентификации в настоящее время происходят из центров обработки данных США, за исключением отмеченных:

* Двухфакторная аутентификация с помощью телефонных звонков или SMS обычно поступает из центров обработки данных в США и направляется глобальными поставщиками.
    * Запросы на аутентификацию пользователей общего назначения из других регионов, таких как Европа или Австралия, в настоящее время обрабатываются центрами обработки данных в этом регионе. Другие события, такие как сбросы паролей самообслуживания, события Azure B2C или гибридные сценарии с использованием NPS Extension или адаптера AD FS, в настоящее время обрабатываются центрами обработки данных в США.
* Push-уведомления с помощью приложения Microsoft Authenticator поступают из центров обработки данных в США. Кроме того, услуги, предоставляемые конкретным поставщикам устройств, могут также вступать в игру из разных регионов.
* Коды OATH, как правило, в настоящее время проверяются в США.
    * Опять же, события аутентификации пользователей общего назначения, происходящие в других регионах, таких как Европа или Австралия, обрабатываются центрами обработки данных в этом регионе. Дополнительные события в настоящее время обрабатываются центрами обработки данных в США.

## <a name="personal-data-stored-by-azure-multi-factor-authentication"></a>Личные данные, хранящиеся в Azure Multi-Factor Authentication

Личные данные — это информация на уровне пользователя, связанная с конкретным человеком. Следующие хранилища данных содержат личную информацию:

* Заблокированные пользователи
* Обойдя пользователей
* Запросы на изменение маркера устройства Microsoft Authenticator
* Многофакторная аутентификация отчетов о деятельности
* Активации microsoft Authenticator

Эта информация сохраняется в течение 90 дней.

Multi-Factor Authentication не регистрирует персональные данные, такие как имя пользователя, номер телефона или IP-адрес, но есть *UserObjectId,* который определяет многофакторную аутентификацию попыток пользователей. Данные журнала хранятся в течение 30 дней.

## <a name="organizational-data-stored-by-azure-multi-factor-authentication"></a>Организационные данные, хранящиеся в Azure Multi-Factor Authentication

Организационные данные — это информация уровня клиента, которая может предоставить настройку конфигурации или среды. Настройки клиента со следующих страниц Azure Portal Multi-Factor Authentication могут хранить организационные данные, такие как пороги блокировки или сведения идентификатора вызываемого абонента для входящих запросов подлинности телефона:

* Блокировка учетной записи
* Предупреждение о мошенничестве
* Уведомления
* Параметры телефонного звонка

А для azure Multi-Factor Authentication Server следующие страницы портала Azure могут содержать организационные данные:

* Параметры сервера
* Разовый обход
* Правила кэширования
* Статус многофакторного сервера аутентификации

## <a name="log-data-location"></a>Местоположение данных журнала

Где хранится информация о журнале, зависит от региона, в котором они обрабатываются. В большинстве географических регионов имеются возможности многофакторной аутентификации Azure Multi-Factor, поэтому данные журнала хранятся в том же регионе, что и запрос на многофакторную аутентификацию. В географических регионах без поддержки мультифакторной аутентификации Azure они обслуживаются в сша или Европе, а данные журналов хранятся в том же регионе, который обрабатывает запрос на многофакторную аутентификацию.

Некоторые основные данные журнала аутентификации хранятся только в США. Microsoft Azure Германия и Microsoft Azure, работающие 21Vianet, всегда хранятся в соответствующем облаке. Данные журнала Microsoft Government Cloud всегда хранятся в США.

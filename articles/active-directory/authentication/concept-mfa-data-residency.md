# <a name="data-residency-and-customer-data-for-azure-multi-factor-authentication"></a>Местонахождение данных и клиентские данные для многофакторной идентификации Azure

Данные клиентов хранятся в службе Azure AD в географическом расположении на основе адреса, предоставленного вашей организацией при подписке на Microsoft Online Service, например Office 365 и Azure. Сведения о том, где хранятся данные клиентов, можно найти в разделе [где находятся ваши данные?](https://www.microsoft.com/trustcenter/privacy/where-your-data-is-located) центра управления безопасностью Майкрософт.

Облачная многофакторная идентификация Azure и Azure сервер Многофакторной идентификации процесс, а затем хранят некоторые персональные данные и данные Организации. В этой статье описывается, что и где хранятся данные.

Следующие действия многофакторной проверки подлинности в настоящее время исходят из центров обработки данных США, за исключением случаев, когда отмечено:

* Двухфакторная проверка подлинности с помощью телефонных звонков или SMS обычно происходит из центров обработки данных США и направляется глобальными поставщиками.
    * Запросы проверки подлинности пользователей общего назначения из других регионов, таких как Европа или Австралия, в настоящее время обрабатываются центрами обработки данных в этом регионе. Другие события, такие как самостоятельный сброс пароля, события B2C Azure или гибридные сценарии с использованием расширения NPS или адаптера AD FS, обрабатываются в настоящее время в центрах обработки данных США.
* Push-уведомления, использующие приложение Microsoft Authenticator, берутся из центров обработки данных США. Кроме того, службы, зависящие от поставщика устройств, также могут играть из разных регионов.
* Коды OATH обычно проверяются в США
    * Опять же, события проверки подлинности пользователей общего назначения, происходящие в других регионах, например в Европе или Австралии, обрабатываются центрами обработки данных в этом регионе. Дополнительные события в настоящее время обрабатываются центрами обработки данных США.

## <a name="personal-data-stored-by-azure-multi-factor-authentication"></a>Персональные данные, хранящиеся в службе многофакторной идентификации Azure

Личные данные — это сведения на уровне пользователя, связанные с конкретным пользователем. Следующие хранилища данных содержат персональные данные:

* Заблокированные пользователи
* Пропущенные пользователи
* Microsoft Authenticator запросов на изменение маркера устройства
* Отчеты по действиям многофакторной проверки подлинности
* Активация Microsoft Authenticator

Эти сведения сохраниться в течение 90 дней.

Служба многофакторной идентификации Azure не регистрирует личные данные, такие как имя пользователя, номер телефона или IP-адрес, но существует *значение userobjectid* , определяющий попытки многофакторной проверки подлинности для пользователей. Данные журнала хранятся в течение 30 дней.

|--------------------------------------|-----------------|
| OATH-токен                           | В журналах многофакторной проверки подлинности     |

> [!NOTE]

## <a name="organizational-data-stored-by-azure-multi-factor-authentication"></a>Данные организации, хранящиеся в службе многофакторной идентификации Azure

Организационные данные — это сведения на уровне клиента, которые могут предоставлять конфигурацию или настройку среды. Параметры клиента из следующих портал Azure страниц многофакторной проверки подлинности могут хранить организационные данные, такие как пороговые значения блокировки или сведения об ИДЕНТИФИКАТОРе вызывающего объекта для входящих запросов проверки подлинности телефона:

* Блокировка учетной записи
* Предупреждение о мошенничестве
* Уведомления
* Параметры телефонного звонка

А для сервер Многофакторной идентификации Azure следующие страницы портал Azure могут содержать данные организации:

* Параметры сервера
* Разовый обход
* Правила кэширования
* Состояние сервер Многофакторной идентификации

## <a name="log-data-location"></a>Расположение данных журнала

Место хранения сведений журнала зависит от того, в каком регионе они обрабатывались. В большинстве географических регионов есть собственные возможности многофакторной идентификации Azure, поэтому данные журнала хранятся в том же регионе, который обрабатывает запрос многофакторной проверки подлинности. В географических регионах без собственной поддержки многофакторной проверки подлинности Azure они обслуживаются либо географическими США, либо Европа, либо данные журнала хранятся в том же регионе, который обрабатывает запрос многофакторной проверки подлинности.

Некоторые основные данные журнала проверки подлинности хранятся только в США. Microsoft Azure — Германия и Microsoft Azure, которыми управляет 21Vianet, всегда хранятся в соответствующем облаке. Данные журналов облака Microsoft для государственных организаций всегда хранятся в США.

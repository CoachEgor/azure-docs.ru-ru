# <a name="customize-the-user-experience-for-azure-active-directory-self-service-password-reset"></a>Настройка пользовательского интерфейса для сна паролей самообслуживания Azure Active Directory

Сбросить паролем самообслуживания (SSPR) дает пользователям в Azure Active Directory (Azure AD) возможность изменять или сбросить пароль без участия администратора или службы поддержки. Если учетная запись пользователя заблокирована или пользователь забыл пароль, выполнив несложные инструкции, он сможет выполнить разблокировку и вернуться к работе. Эта возможность снижает количество обращений в службу технической поддержки и минимизирует время простоя из-за невозможности войти на устройство или в приложение.

Чтобы улучшить работу SSPR для пользователей, вы можете настроить внешний вид страницы сбросить пароль, уведомления по электронной почте или страницы регистрации. Эти параметры настройки позволяют дать понять пользователю, что они находятся в нужном месте, и дают им уверенность в том, что они получают доступ к ресурсам компании.
    
В этой статье показано, как настроить ссылку на электронную почту SSPR для пользователей, брендинг компании и ссылку на страницу AD FS.

## <a name="customize-the-contact-your-administrator-link"></a>Настройка ссылки "Обратитесь к администратору"

Чтобы помочь пользователям обратиться за помощью в сбрасе пароля самообслуживания, на портале сбросить пароль для снава паролей отображается ссылка "Связаться с администратором". Если пользователь выбирает эту ссылку, он делает одну из двух вещей:

* Если эта контактная ссылка остается в состоянии по умолчанию, администраторам отправляется электронное письмо с просьбой оказать помощь в изменении пароля пользователя. Следующий образец электронной почты показывает это сообщение электронной почты по умолчанию:

    ![Пример запроса для сбросить электронную почту, отправленную администратору](./media/howto-sspr-customization/sspr-contact-admin.png)

* Если настроить, отправляет пользователя на веб-страницу или адрес электронной почты, указанный администратором для помощи.
    * Если вы подгонетесь этим, мы рекомендуем установить это на то, что пользователи уже знакомы с для поддержки.

    > [!WARNING]
    > Если настроить эту настройку с адресом электронной почты и учетной записью, которая нуждается в сбросе пароля, пользователь может не обратиться за помощью.

### <a name="default-email-behavior"></a>Поведение электронной почты по умолчанию

Электронная почта контакта по умолчанию отправляется получателям в следующем порядке:

1. Если назначена роль *администратора службы поддержки* или роль *администратора паролей,* администраторы с этими ролями уведомляются.
1. Если администратор службы поддержки или администратор паролей не назначены, то администраторы с ролью *администратора пользователя* уведомляются.
1. Если ни одна из предыдущих ролей не *назначена, то глобальные администраторы* уведомляются.

Во всех случаях уведомление будет отправлено не более чем 100 получателям.

Дополнительные сведения о различных ролях администраторов и их назначении см. в статье [Назначение ролей администратора в Azure Active Directory](../users-groups-roles/directory-assign-admin-roles.md).

### <a name="disable-contact-your-administrator-emails"></a>Отключение отправки электронных сообщений при выборе ссылки "Обратитесь к администратору"

Если организация не хочет уведомлять администраторов о запросах на сбросить пароль, можно использовать следующие параметры конфигурации:

* Настройте ссылку на службу технической поддержки для предоставления URL-адреса или адреса mailto, который пользователи могут использовать для получения помощи. Эта опция находится под **Password Сбросить** > **пользовательские** > **справочная служба электронной почты или URL**.
* Включить сбросить пароли самообслуживания для всех пользователей. Эта опция находится под**свойствами** **сбросить** > пароль . Если вы не хотите, чтобы пользователи могли сбрасывать пароли, можно ограничить доступ, назначив пустую группу. *Мы не рекомендуем этот вариант.*

## <a name="customize-the-sign-in-page-and-access-panel"></a>Настройка страницы ввне и панели доступа

Вы можете настроить страницу ввоза, например, добавить логотип, который появляется вместе с изображением, которое соответствует брендингу вашей компании. Для получения дополнительной информации о том, как настроить брендинг компании, [см. Добавить брендинг компании на страницу ввне в Azure AD](../fundamentals/customize-branding.md).

Выбранные вами изображения отображаются в следующих случаях:

* После того, как пользователь вводит свое имя пользователя.
* Когда пользователь обращается к настраиваемому URL-адресу:
   * Передавая `whr` параметр на страницу сбросить пароль, например,`https://login.microsoftonline.com/?whr=contoso.com`
   * Передавая `username` параметр на страницу сбросить пароль, например,`https://login.microsoftonline.com/?username=admin@contoso.com`

### <a name="directory-name"></a>Имя каталога

Чтобы сделать вещи более удобными для пользователей, вы можете изменить название организации на портале и в автоматизированной связи. Чтобы изменить атрибут иимения каталога на портале Azure, просмотрите в **Azure Active Directory** > **Properties.** Эта дружелюбная опция названия организации наиболее заметна в автоматизированных электронных письмах, как и в следующих примерах:

* Дружелюбное имя в электронной почте, например ,*Microsoft от имени демо CONTOSO*"
* Тема в электронной почте, например: код*проверки электронной почты демо-аккаунта CONTOSO*"

## <a name="customize-the-ad-fs-sign-in-page"></a>Настройка страницы регистрации AD FS

Если вы используете службы Федерации активных каталогов (AD FS) для событий, входящих в систему пользователя, вы можете добавить ссылку на страницу регистрации, используя руководство в статье для [добавления описания страницы.](/windows-server/identity/ad-fs/operations/add-sign-in-page-description)

Предоставьте пользователям ссылку на страницу для входа в рабочий *https://passwordreset.microsoftonline.com*процесс SSPR, например. Чтобы добавить ссылку на страницу ввеком AD FS, используйте следующую команду на сервере AD FS:

``` powershell
Set-ADFSGlobalWebContent -SigninPageDescriptionText "<p><a href='https://passwordreset.microsoftonline.com' target='_blank'>Can't access your account?</a></p>"
```

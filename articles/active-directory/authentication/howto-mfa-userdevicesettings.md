# <a name="manage-user-settings-for-azure-multi-factor-authentication"></a>Управление настройками пользователей для многофакторной аутентификации Azure

Чтобы помочь пользователям Azure Multi-Factor Authentication, можно потребовать от пользователей сбросить пароль, перерегистрироваться на МИД или отменить существующие сеансы МИД. Для пользователей, которые определили пароли приложений, вы также можете удалить эти пароли, в результате чего устаревшая аутентификация сбой в этих приложениях. Эти действия могут быть необходимы, если вам необходимо оказать помощь пользователю или сбросить его состояние безопасности.

## <a name="manage-user-authentication-options"></a>Управление опциями проверки подлинности пользователей

Если вам назначена роль *администратора аутентификации,* вы можете потребовать от пользователей сбросить пароль, перерегистрироваться на МИД или отозвать существующие сеансы MFA с объекта пользователя. Чтобы управлять настройками пользователя, выполните следующие действия:

1. Войдите на [портал Azure](https://portal.azure.com).
1. Слева выберите**пользователей** >  **Azure Active Directory** > **Все пользователи.**
1. Выберите пользователя, на который вы хотите выполнить действие, и выберите **методы аутентификации.** В верхней части окна выберите один из следующих вариантов для пользователя:
   - **Сброс пароль** сбросит пароль пользователя и назначает временный пароль, который должен быть изменен при следующем входе.
   - **Требование перерегистрации МИД** делает это так, что, когда пользователь подписывает в следующий раз, они просили создать новый метод проверки подлинности МИД.
   - **Отмена сессии МИД** очищает запоминаемых сессий МИД пользователя и требует, чтобы он выполнил МИД в следующий раз, когда это требуется по полису на устройстве.

   ![Управление методами проверки подлинности с портала Azure](./media/howto-mfa-userdevicesettings/manage-authentication-methods-in-azure.png)

## <a name="delete-users-existing-app-passwords"></a>Удалять существующие пароли приложений пользователей

При необходимости можно удалить все созданные пользователем пароли приложения. Работа не использующих браузер приложений, связанных с этими паролями, приостанавливается, пока не будут созданы новые пароли. Для выполнения этого действия требуются глобальные разрешения *администратора.*

Чтобы удалить пароли приложения пользователя, выполните следующие шаги:

1. Войдите на [портал Azure](https://portal.azure.com).
1. С левой стороны выберите**пользователей** >  **Azure Active Directory** > **Все пользователи.**
1. Выберите **многофакторную аутентификацию**. Возможно, вам придется прокрутить вправо, чтобы увидеть этот вариант меню. Выберите скриншот приведенного ниже примера, чтобы увидеть полное окно портала Azure и расположение меню:[![](media/howto-mfa-userstates/selectmfa-cropped.png "Выберите многофакторную аутентификацию из окна пользователей в Azure AD")](media/howto-mfa-userstates/selectmfa.png#lightbox)
1. Установите флажок рядом с именем пользователя или пользователей, которыми требуется управлять. Справа отображается список вариантов быстрого шага.
1. Выберите **Управление настройками пользователя,** а затем проверить поле для **удаления всех существующих паролей приложения, генерируемых выбранными пользователями,** как показано в следующем примере: ![Удалите все существующие пароли приложения](./media/howto-mfa-userdevicesettings/deleteapppasswords.png)
1. Выберите **сохранить,** а затем **закрыть**.

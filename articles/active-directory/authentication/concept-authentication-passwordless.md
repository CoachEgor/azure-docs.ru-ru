# <a name="passwordless-authentication-options-for-azure-active-directory"></a>Параметры проверки подлинности без паролей для active Directory Azure

Многофакторная аутентификация (MFA) — это отличный способ обезопасить вашу организацию, но пользователи часто расстраиваются из-за дополнительного уровня безопасности, что не позволяет запомнить свои пароли. Методы проверки подлинности без паролей более удобны, потому что пароль удаляется и заменяется чем-то, что у вас есть, плюс то, что вы или что-то вы знаете.

|   | Что-то у вас есть | То, что вы или знаете |
| --- | --- | --- |
| Вход без пароля | Устройство Windows 10, телефон или ключ безопасности | Биометрический или PIN-код |

Каждая организация имеет различные потребности, когда дело доходит до проверки подлинности. Корпорация Майкрософт предлагает следующие три варианта проверки подлинности без паролей, которые интегрируются с Active Directory Azure (Azure AD):

- Windows Hello для бизнеса
- Приложение Microsoft Authenticator
- Ключи безопасности FIDO2

![Аутентификация: безопасность против удобства](./media/concept-authentication-passwordless/passwordless-convenience-security.png)

## <a name="windows-hello-for-business"></a>Windows Hello для бизнеса

Windows Hello for Business идеально подходит для информационных работников, у которых есть собственный назначенный Windows PC. Биометрические и PIN-код ы напрямую привязаны к компьютеру пользователя, что препятствует доступу к кому-либо, кроме владельца. Благодаря интеграции инфраструктуры для общего доступа (PKI) и встроенной поддержке единого всхотвляемого (SSO) Windows Hello for Business предоставляет удобный метод для беспрепятственного доступа к корпоративным ресурсам на местах и в облаке.

![Пример вху пользователя в систему windows Hello for Business](./media/concept-authentication-passwordless/windows-hellow-sign-in.jpeg)

Следующие шаги показывают, как процесс вхинга работает с Active Directory Azure.

![Диаграмма, в которой излагаются шаги, связанные с входе пользователя в систему windows Hello for Business](./media/concept-authentication-passwordless/windows-hello-flow.png)

1. Пользователь входит в Windows с помощью биометрического или ПИН-жеста. Этот жест открывает закрытый ключ Windows Hello for Business и отправляется поставщику поддержки безопасности Облачной аутентификации, называемому *поставщиком Облачных А.П..*
1. Поставщик Облачных А.П. запрашивает nonce от Azure AD.
1. Azure AD возвращает nonce, который действителен в течение 5 минут.
1. Поставщик Cloud AP подписывает nonce с помощью закрытого ключа пользователя и возвращает подписанный nonce в Azure AD.
1. Azure AD проверяет подписанный nonce, используя надежно зарегистрированный общедоступный ключ пользователя против подписи nonce. После проверки подписи Azure AD проверяет возвращенный подписанный nonce. При проверке Nonce Azure AD создает основной маркер обновления (PRT) с ключом сеанса, который зашифрован на источник переноса устройства и возвращает его поставщику Cloud AP.
1. Поставщик Cloud AP получает зашифрованный PRT с ключом сеанса. Используя личный транспортный ключ устройства, поставщик Cloud AP расшифровывает ключ сеанса и защищает ключ сеанса с помощью модуля trusted Platform (TPM).
1. Поставщик Cloud AP возвращает вам успешный ответ на аутентификацию windows. Затем пользователь может получить доступ к Windows, а также облачным и корпоративным приложениям без необходимости повторной проверки подлинности (SSO).

[Руководство windows](https://docs.microsoft.com/windows/security/identity-protection/hello-for-business/hello-planning-guide) Hello for Business planning может быть использовано для принятия решений о типе развертывания Windows Hello для бизнеса и вариантах, которые необходимо учитывать.

## <a name="microsoft-authenticator-app"></a>Приложение Microsoft Authenticator

Позвольте телефону вашего сотрудника стать методом проверки подлинности без паролей. Возможно, вы уже используете приложение Microsoft Authenticator в качестве удобной многофакторной проверки подлинности в дополнение к паролю. Вы также можете использовать приложение Authenticator в качестве безпачного варианта.

![Войти на Microsoft Edge с приложением Microsoft Authenticator](./media/concept-authentication-passwordless/concept-web-sign-in-microsoft-authenticator-app.png)

Приложение Authenticator превращает любой iOS или Android телефон в надежный, без паролей учетных данных. Пользователи могут войти в любую платформу или браузер, получив уведомление на свой телефон, сопоставив номер, отображаемый на экране, с номером на телефоне, а затем используя свои биометрические (прикосновение или лицо) или PIN-код для подтверждения. Пожалуйста, обратитесь к [Download и установите приложение Microsoft Authenticator](https://docs.microsoft.com/azure/active-directory/user-help/user-help-auth-app-download-install) для получения сведений об установке.

Безпачная аутентификация с помощью приложения Authenticator следует той же основной схеме, что и Windows Hello for Business. Это немного сложнее, так как пользователь должен быть идентифицирован, чтобы Azure AD мог найти используемую версию приложения Microsoft Authenticator:

![Диаграмма, в которой излагаются шаги, связанные с входе пользователя в приложение Microsoft Authenticator](./media/concept-authentication-passwordless/authenticator-app-flow.png)

1. Пользователь вводит свое имя пользователя.
1. Azure AD обнаруживает, что пользователь имеет сильную учетную запись, и запускает поток Strong Credential.
1. Уведомление отправляется в приложение через службу уведомлений Apple Push (APNS) на устройствах iOS или через Firebase Cloud Messaging (FCM) на устройствах Android.
1. Пользователь получает push-уведомление и открывает приложение.
1. Приложение вызывает Azure AD и получает вызов подтверждения присутствия и nonce.
1. Пользователь завершает задачу, введя свой биометрический или PIN-код для разблокировки частного ключа.
1. Nonce подписывается с закрытым ключом и отправляется обратно в Azure AD.
1. Azure AD выполняет проверку ключей общего/частного значения и возвращает токен.

## <a name="fido2-security-keys"></a>Ключи безопасности FIDO2

Ключи безопасности FIDO2 — это нефишируемый метод проверки подлинности без паролей, основанный на стандартах, который может прийти в любом форм-факторе. Fast Identity Online (FIDO) является открытым стандартом для проверки подлинности без паролей. FIDO позволяет пользователям и организациям использовать стандарт для входной системы в свои ресурсы без имени пользователя или пароля, используя внешний ключ безопасности или ключ платформы, встроенный в устройство.

Для публичного предварительного просмотра сотрудники могут использовать ключи безопасности для входной связи в свои приложения Azure AD или гибридные устройства Azure AD, подключаемые к Windows 10, и получить одноразовые знаки на своих облачных и местных ресурсах. Пользователи также могут войти в поддерживаемые браузеры. Ключи безопасности FIDO2 являются отличным вариантом для предприятий, которые очень чувствительны к безопасности или имеют сценарии или сотрудников, которые не желают или не могут использовать свой телефон в качестве второго фактора.

![Войти на microsoft Edge с ключом безопасности](./media/concept-authentication-passwordless/concept-web-sign-in-security-key.png)

Следующий процесс используется при входе пользователя с ключом безопасности FIDO2:

![Диаграмма, в которой излагаются шаги, связанные с входе пользователя с ключом безопасности FIDO2](./media/concept-authentication-passwordless/fido2-security-key-flow.png)

1. Пользователь подключает ключ безопасности FIDO2 к своему компьютеру.
2. Windows обнаруживает ключ безопасности FIDO2.
3. Windows отправляет запрос на аутентификацию.
4. Azure AD отправляет обратно nonce.
5. Пользователь завершает свой жест, чтобы разблокировать закрытый ключ, хранящийся в безопасном анклаве ключа безопасности FIDO2.
6. Ключевой ключ безопасности FIDO2 подписывает nonce с закрытым ключом.
7. Основной запрос маркера обновления (PRT) с подписанным nonce отправляется в Azure AD.
8. Azure AD проверяет подписанный nonce с помощью общедоступного ключа FIDO2.
9. Azure AD возвращает PRT, чтобы обеспечить доступ к ресурсам.

Хотя есть много ключей, сертифицированных FIDO2 Альянсом FIDO, Microsoft требует некоторых дополнительных расширений спецификации ПРОТОКОЛа FIDO2 клиента к аутентификатору (CTAP), которые должны быть реализованы поставщиком для обеспечения максимальной безопасности и наилучшего опыта.

Ключ безопасности **должен** реализовать следующие функции и расширения из протокола FIDO2 CTAP, чтобы быть совместимым с Корпорацией Майкрософт:

| # | Функция / Расширение доверия | Почему требуется эта функция или расширение? |
| --- | --- | --- |
| 1 | Ключ-резидент | Эта функция позволяет быть портативным ключом безопасности, где ваши учетные данные хранятся на ключе безопасности. |
| 2 | Клиентский штифт | Эта функция позволяет защитить учетные данные с помощью второго фактора и применяется к ключам безопасности, которые не имеют пользовательского интерфейса. |
| 3 | hmac-секрет | Это расширение гарантирует, что вы можете войти в устройство, когда оно отключено или в режиме самолета. |
| 4 | Несколько учетных записей на RP | Эта функция гарантирует, что вы можете использовать один и тот же ключ безопасности в нескольких службах, таких как Учетная запись Майкрософт и Активный каталог Azure. |

## <a name="what-scenarios-work-with-the-preview"></a>Какие сценарии работают с предварительным просмотром?

- Администраторы могут включить методы проверки подлинности без паролей для своего клиента
- Администраторы могут настроить таргетинг на всех пользователей или выбрать пользователей/групп в пределах своего клиента для каждого метода
- Конечные пользователи могут зарегистрировать ся и управлять этими методами проверки подлинности без паролей на портале учетных записей
- Конечные пользователи могут войти в систему с помощью этих методов проверки подлинности без паролей
   - Microsoft Authenticator App: Работает в сценариях, где azure AD аутентификация используется, в том числе во всех браузерах, во время установки Windows 10 Out Of Box (OOBE) и с интегрированными мобильными приложениями на любой операционной системе.
   - Ключи безопасности: Работа на экране блокировки для Windows 10 и в Интернете в поддерживаемых браузерах, таких как Microsoft Edge (как наследие, так и новый Edge).

# <a name="what-is-passwordless"></a>Что такое без пароля?

Многофакторная Идентификация (MFA) — отличный способ Защитите свою организацию, но пользователи получают дополнительный уровень поверх необходимости запоминать пароли со связью. Методы проверки подлинности без пароля более удобны, так как пароль будет удален и заменен то, что у вас есть, а также то, что вы являетесь или то, что вы знаете.

|   | То, что у вас есть | Что-либо, или знаете |
| --- | --- | --- |
| Вход без пароля | Телефон или безопасности ключа | Биометрическая или ПИН-код |

Мы понимаем, что каждая организация может иметь иные потребности, когда дело доходит до проверки подлинности. Корпорация Майкрософт в настоящее время предлагает Windows Hello, опыт premier без пароля для ПК Windows. Мы добавляем новые учетные данные к семейству без пароля. Приложение Microsoft Authenticator и FIDO2 ключей безопасности.

## <a name="microsoft-authenticator-app"></a>Приложение Microsoft Authenticator

Разрешение телефона сотрудника в качестве метода проверки подлинности без пароля. Вы можете уже использовать приложение Microsoft Authenticator в случае удобный многофакторной проверки подлинности только пароль. Но теперь она доступна как средство без пароля.

Становится блочным любой iOS или Android phone строгий, без пароля учетных данных, позволяя пользователям входить любой платформы или браузера, получив уведомление на свой телефон, сопоставления номер, показанный на экране, чтобы на свой телефон и затем с помощью их биометрические ( Touch или сталкиваются) или ПИН-код для подтверждения.

## <a name="fido2-security-keys"></a>Ключи безопасности FIDO2

Ключи безопасности FIDO2 являются unphishable стандартизированное без пароля метод проверки подлинности, могут поступать в форм-факторе. Online быстрый удостоверений (FIDO) — это открытый стандарт для проверки подлинности без пароля. Она позволяет пользователям и организациям использовать стандартный вход к своим ресурсам без имени пользователя или пароль, используя внешний ключ или ключ платформы, встроенный в устройство.

В общедоступной предварительной версии сотрудников можно использовать ключи безопасности для входа на своих компьютерах Azure Active Directory присоединенных к Windows 10 (версия 1809 или более поздней версии) и получить возможность единого входа в систему на своих облачных ресурсов. Их можно также войти в поддерживаемых браузерах.

Хотя существует много ключей, которые являются FIDO2 сертифицированы FIDO Alliance, корпорация Майкрософт требует некоторые необязательные расширения спецификации FIDO2 CTAP для реализации с целью добиться максимальной безопасности и получения наилучших результатов.

Ключ безопасности **необходимо** реализовать следующие функции и расширения протокола FIDO2 CTAP совместимыми с Microsoft:

| # | Функция / расширения доверять | Почему это требуется? |
| --- | --- | --- |
| 1 | Резидентных ключ | Эта функция позволяет ключ безопасности, которую можно переносить, где ваши учетные данные хранятся на ключ безопасности. |
| 2 | ПИН-код клиента | Эта функция позволяет защитить учетные данные с помощью второй фактор и применяется к ключам, у которых нет пользовательского интерфейса. |
| 3 | HMAC-secret | Это расширение гарантирует, что вы можете войти устройство при автономном режиме или в режиме "в самолете". |
| 4 | Несколько учетных записей на RP | Эта функция гарантирует, что можно использовать один и тот же ключ безопасности между несколькими службами, например учетной записи Майкрософт и Azure Active Directory. |

## <a name="what-scenarios-work-with-the-preview"></a>Для каких сценариев работы с предварительной версией?

1. Администраторы могут включить методы проверки подлинности без пароля для своего клиента
1. Администраторы могут охватить всех пользователей, и выберите пользователей или групп в клиенте для каждого метода
1. Конечным пользователям можно зарегистрировать и управлять эти методы проверки подлинности без пароля на портале учетной записи
1. Пользователи могли входить в систему с помощью этих методов проверки подлинности без пароля
   1. Приложение Microsoft Authenticator: Будет работы в каждом сценарии, где используется проверка подлинности Azure AD, включая со всеми обозревателями, во время Windows 10 Out установки (OOBE) и с интегрирована мобильных приложений в любой операционной системе.
   1. Ключи безопасности: Будет работать на экране блокировки для Windows 10 1809 или более поздней версии и в Интернете в поддерживаемых браузеров, таких как Microsoft Edge.

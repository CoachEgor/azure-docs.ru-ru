# <a name="authentication-methods-usage--insights-preview"></a>Использование методов проверки подлинности & insights (Предварительная версия)

Использование & insights позволяет понять, как при работе методов проверки подлинности для функции, такие как многофакторная Идентификация Azure и самостоятельного сброса пароля в вашей организации. Этой возможности создания отчетов предоставляет вашей организации с помощью средства, чтобы понять, какие методы регистрируются и как они используются.

## <a name="permissions-and-licenses"></a>Разрешения и лицензии

Об использовании и, доступны следующие роли:

- глобального администратора;
- Читатель сведений о безопасности
- Администратор безопасности
- Читатель отчетов

Без дополнительных затрат на лицензии необходим для использования доступа и аналитики. Многофакторная Идентификация Azure и сведения о лицензировании для Самостоятельного сброса пароля можно найти на [сайт с ценами на Azure Active Directory](https://azure.microsoft.com/pricing/details/active-directory/).

## <a name="how-it-works"></a>Принцип работы

Для доступа к об использовании метода проверки подлинности и:

1. Перейдите на [портал Azure](https://portal.azure.com).
1. Перейдите к **Azure Active Directory** > **сброса пароля** > **использования & insights**.
1. Из **регистрации** или **использования** обзоры, вы можете открыть предварительно отфильтрованных отчетов для фильтрации в соответствии с потребностями.

![Общие сведения об использовании & insights](./media/howto-authentication-methods-usage-insights/usage-insights-overview.png)

Прямой доступ к использованию & insights, перейдите к [ https://portal.azure.com/#blade/Microsoft_AAD_IAM/AuthMethodsOverviewBlade ](https://portal.azure.com/#blade/Microsoft_AAD_IAM/AuthMethodsOverviewBlade). Эту ссылку, вы попадете на общие сведения о регистрации.

Пользователи, зарегистрированные, пользователям разрешен и поддержкой плиток пользователей отображать следующие данные регистрации для пользователей:

- Регистрации: Пользователь считается зарегистрированным, если они (или администратор), зарегистрировали достаточное количество методов проверки подлинности в соответствии с политикой вашей организации SSPR или многофакторная проверка подлинности.
- Включено: Пользователь считается включенным, если они находятся в области SSPR политики. Если SSPR включен для группы, то пользователь считается включенным, если они находятся в этой группе. Если SSPR включен для всех пользователей, то все пользователи в клиенте (за исключением гостей), считаются включена.
- Поддерживает: Пользователь считается может, если они и являются функциональными зарегистрирован. Это состояние означает, что они могут выполнять SSPR в любое время, при необходимости.

Щелкнув любое из этих плиток или сведений аналитики в них, чтобы перейти к предварительно отфильтрованный список сведения о регистрации.

**Регистраций** диаграммы на **регистрации** вкладке отображается число успешных и неудачных проверки подлинности метод регистрации с помощью метода проверки подлинности. **Сбрасывает** диаграммы на **использования** вкладке отображается число успешных и неудачных проверок подлинности во время его сброс потока методом проверки подлинности.

Щелкнув любую из диаграмм вы хотели предварительно отфильтрованный список регистрации или сброса событий.

Использование элемента управления в верхнем правом углу окна, можно изменить диапазон дат для данных аудита, показанные в диаграммах регистрации и сброса, до 24 часов, 7 дней или 30 дней.

### <a name="registration-details"></a>Сведения о регистрации

Щелкнув **пользователей, зарегистрированных**, **пользователям разрешен**, или **пользователей может** плитки или аналитические сводки, чтобы перейти к сведения о регистрации.

Отчет о регистрации отображается следующая информация для каждого пользователя:

- ИМЯ
- Имя пользователя
- Состояние регистрации (все, зарегистрированные, не зарегистрирован)
- Включенное состояние (все, включены, не включена)
- Поддерживает состояние (все, возможно, не может)
- Методы (уведомление от приложения, код приложения, телефонный звонок, SMS, электронной почты, вопросы безопасности)

Использование элементов управления в верхней части списка, можно найти пользователя и фильтровать список пользователей, основанные на столбцах показано.

### <a name="reset-details"></a>Сброс сведений

При выборе команды на регистрацию или сбрасывает диаграммы выводится к сведениям о сброса.

Сброс подробный отчет показывает события регистрации и сброса паролей за последние 30 дней, в том числе:

- ИМЯ
- Имя пользователя
- Функция (All, регистрации, Reset)
- Метод проверки подлинности (уведомление от приложения, код приложения, телефонного звонка, звонок на рабочий, SMS, электронной почты, вопросы безопасности)
- Состояние (все, успех, ошибка)

Использование элементов управления в верхней части списка, можно найти пользователя и фильтровать список пользователей, основанные на столбцах показано.

## <a name="limitations"></a>Ограничения

Данные, отображаемые в этих отчетах будет задержан до 60 минут. Поле «Последнее обновление» существует на портале Azure, чтобы определить, как последние данные будут.

Данные об использовании и не является заменой для отчетов об активности многофакторной идентификации Azure или сведения, содержащиеся в отчете входы в систему Azure AD.

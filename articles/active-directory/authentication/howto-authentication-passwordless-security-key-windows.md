# <a name="enable-passwordless-security-key-sign-in-to-windows-10-devices-with-azure-active-directory-preview"></a>Включить ключ безопасности без паролей, вкак исключаемый на устройства Windows 10 с помощью Active Directory Azure (предварительный просмотр)

Этот документ фокусируется на включении ключа безопасности FIDO2 на основе безпадочной аутентификации с устройствами Windows 10. В конце этой статьи вы сможете войти как в Azure AD, так и на гибридные устройства Azure AD с windows 10 с учетной записью Azure AD с помощью ключа безопасности FIDO2.

### <a name="unsupported-scenarios"></a>Неподдерживаемые сценарии

Следующие сценарии не поддерживаются:

- Развертывание доменных служб с активным каталогом Windows Server Active Directory Domain Services (AD DS) (на непредподанных только устройствах).
- Сценарии RDP, VDI и Citrix с использованием ключа безопасности.
- S/MIME с помощью ключа безопасности.
- "Беги как" с помощью ключа безопасности.
- Вход на сервер с помощью ключа безопасности.
- Если вы не использовали ключ безопасности для вху-сина на устройство в режиме онлайн, вы не можете использовать его для вху-на-айму или разблокировки в автономном режиме.
- Регистрация или разблокировка устройства Windows 10 с ключом безопасности, содержащим несколько учетных записей Azure AD. В этом сценарии используется последняя учетная запись, добавленная в ключ безопасности. WebAuthN позволяет пользователям выбирать учетную запись, которой они хотят пользоваться.
- Разблокировать устройство под управлением Windows 10 версия 1809. Для лучшего опыта используйте Windows 10 версию 1903 или выше.

## <a name="prepare-devices-for-preview"></a>Подготовка устройств к предварительному просмотру

Azure AD объединила устройства, которые вы пилотируете во время просмотра функции с должны запустить Windows 10 версии 1809 или выше. Лучшее впечатление на Windows 10 версия 1903 или выше.

Гибридные устройства, объединенные ВС УАЗ, должны работать с Windows 10 Insider Build 18945 или новее.

## <a name="enable-security-keys-for-windows-sign-in"></a>Включить ключи безопасности для входиного окна

Организации могут использовать один или несколько следующих методов, позволяющих использовать ключи безопасности для входе в Windows в соответствии с требованиями их организации:

- [Включить с intune](#enable-with-intune)
- [Целевое развертывание Intune](#targeted-intune-deployment)
- [Включить пакет подготовки](#enable-with-a-provisioning-package)
- [Включить с помощью групповой политики (только гибридные устройства Azure AD)](#enable-with-group-policy)

> [!IMPORTANT]
> Организации с **гибридными устройствами, объединенными Azure AD,** **также** должны выполнить этапы в [статье« Включить аутентификацию FIDO2 для внутренних ресурсов,](howto-authentication-passwordless-security-key-on-premises.md) прежде чем работает ключ проверки безопасности Windows 10 FIDO2.
>
> Организации с **устройствами, объединенными Azure AD,** должны сделать это до того, как их устройства смогут проверить подлинность на внутренних ресурсах с ключами безопасности FIDO2.

### <a name="enable-with-intune"></a>Включить с intune

Чтобы обеспечить использование ключей безопасности с помощью Intune, выполните следующие действия:

1. Войдите на [портал Azure](https://portal.azure.com).
1. Просмотр **корпорации Майкрософт Intune** > **Device регистрации** > **Windows регистрации** > **Windows Привет для бизнес-свойств** > **Properties**.
1. В **настройках**установите **используйте ключи безопасности для ввоза в** **enabled.**

Конфигурация ключей безопасности для вхинга не зависит от настройки Windows Hello for Business.

### <a name="targeted-intune-deployment"></a>Целевое развертывание Intune

Для целевой определенной группы устройств для включения поставщика учетных данных используйте следующие пользовательские настройки через Intune:

1. Войдите на [портал Azure](https://portal.azure.com).
1. Просмотр**профилей конфигурации** > **Profiles** > Microsoft **Intune** > Device**Создание профиля.**
1. Настроили новый профиль со следующими настройками:
   - Название: Ключи безопасности для Входиные окна
   - Описание: Позволяет использовать ключи безопасности FIDO во время вхводом Windows
   - Платформа: Windows 10 и более поздние
   - Тип профиля: Пользовательский
   - Пользовательские настройки OMA-URI:
      - Имя: Включите ключи безопасности FIDO для входиных Windows
      - OMA-URI: ./Device/Vendor/MSFT/PassportForWork/SecurityKey/UseSecurityKeyForSignin
      - Тип данных: Integer
      - Значение: 1
1. Эта политика может быть назначена определенным пользователям, устройствам или группам. Для получения дополнительной информации смотрите [профили пользователей и устройств В Корпорации Майкрософт Intune.](https://docs.microsoft.com/intune/device-profile-assign)

![Создание политики конфигурации пользовательских устройств](./media/howto-authentication-passwordless-security-key/intune-custom-profile.png)

### <a name="enable-with-a-provisioning-package"></a>Включить пакет подготовки

Для устройств, не управляемых Intune, может быть установлен пакет подготовки для включения функциональности. Приложение Для настройки конфигурации Windows может быть установлено в [Microsoft Store.](https://www.microsoft.com/p/windows-configuration-designer/9nblggh4tx22) Выполнить следующие шаги для создания пакета подготовки:

1. Запуск дизайнера конфигурации Windows.
1. Выберите **файл** > **Новый проект**.
1. Назовите проект и обратите внимание на путь создания проекта, а затем выберите **Next.**
1. Оставьте *пакет подготовки,* выбранный в качестве **рабочего процесса выбранного проекта,** и выберите **Следующий.**
1. Выберите *все настольные издания Windows* под **выберите настройки для просмотра и настройки,** а затем выберите **Следующий.**
1. Нажмите кнопку **Готово**.
1. В недавно созданном проекте просматривайте **настройки** > Runtime**WindowsHelloForBusiness** > **SecurityKeys** > **UseSecurityKeyForSignIn.**
1. Установите **UseSecurityKeyForSignIn** для *включенного.*
1. Выберите пакет **экспортного** > **обеспечения**
1. Оставьте по умолчанию в окне **сборки** под **описаниепакета подготовки,** а затем выберите **Следующий.**
1. Оставьте по умолчанию в окне **сборки** под **сведениями безопасности Select для пакета подготовки** и выберите **Следующий.**
1. Обратите внимание или измените путь в окнах **сборки** под **Select, где сохранить пакет подготовки** и выбрать **Следующий.**
1. Выберите **«Сборку** на странице **подготовки пакета».**
1. Сохранить два файла, созданные *(ppkg* и *кошка*) в месте, где вы можете применить их к машинам позже.
1. Чтобы применить созданный пакет подготовки, [см.](https://docs.microsoft.com/windows/configuration/provisioning-packages/provisioning-apply-package)

> [!NOTE]
> Устройства под управлением Windows 10 Version 1809 также должны включать общий режим ПК *(EnableSharedPCMode*). Для получения дополнительной информации о включении этой функциональности, смотрите [Насете общий или гостевой ПК с Windows 10](https://docs.microsoft.com/windows/configuration/set-up-shared-or-guest-pc).

### <a name="enable-with-group-policy"></a>Включить с групповой политикой

Для **гибридных устройств, объединенных Azure AD,** организации могут настроить следующие настройки групповой политики для включения ключа безопасности FIDO. Настройка может быть найдена в соответствии с **компьютерной конфигурации** > **административных шаблонов системы** > **System** > **Logon** > **Включите безопасности ключ входа в:**

- Установка этой политики **в Enabled** позволяет пользователям входить в систему с ключами безопасности.
- Установка этой политики **на отключенную** или **не настроенную** не позволяет пользователям входить в систему с ключами безопасности.

Эта настройка групповой политики `credentialprovider.admx` требует обновленной версии шаблона групповой политики. Этот новый шаблон доступен со следующей версией Windows Server и с Windows 10 20H1. Управлять этой настройкой можно с помощью устройства, работая с одной из этих новых версий Windows или централизованно, следуя рекомендациям в теме [поддержки, Как создать и управлять Центральным магазином для административных шаблонов групповой политики в Windows.](https://support.microsoft.com/help/3087759/how-to-create-and-manage-the-central-store-for-group-policy-administra)

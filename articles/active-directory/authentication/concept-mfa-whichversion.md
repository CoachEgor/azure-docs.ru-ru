# <a name="which-version-of-azure-mfa-is-right-for-my-organization"></a>Какая версия Многофакторной идентификации Microsoft Azure подходит организации?

Прежде чем решить, где и как развернуть Многофакторную идентификацию Microsoft Azure, необходимо ответить на три основных вопроса.

* [Что я пытаюсь защитить?](#what-am-i-trying-to-secure)
* [Где находятся пользователи?](#where-are-the-users-located)
* [Какие функции мне нужны?](#what-features-do-i-need)

Каждый из следующих разделов содержит сведения, помогающие ответить на эти вопросы.

## <a name="what-am-i-trying-to-secure"></a>Что я пытаюсь защитить

Чтобы выяснить, какой из вариантов двухфакторной проверки подлинности вам нужен, определите, что вы пытаетесь защитить с помощью дополнительного фактора проверки подлинности. Это приложение в службе Azure? Или в системе удаленного доступа? Определив объект, который необходимо защитить, вы будете знать, где именно нужно активировать Многофакторную идентификацию.

| Что вы пытаетесь защитить | MFA в облаке | Сервер MFA |
| --- |:---:|:---:|
| Приложения Майкрософт, получающие данные напрямую из источника |● |● |
| Приложения SaaS в коллекции приложений |● |  |
| Веб-приложения, опубликованные через прокси приложения Azure AD |● |  |
| Приложения IIS, опубликованные не через прокси приложения Azure AD | |● |
| Удаленный доступ, например VPN или RDG, с помощью расширения NPS или существующий сервер политики сети | ● | ● |

## <a name="where-are-the-users-located"></a>Где находятся пользователи?

Далее в зависимости от местонахождения пользователей организации можно определить, какое решение нужно для использования сервера MFA — облачное или локальное.

| Местонахождение пользователей | MFA в облаке | Сервер MFA |
| --- |:---:|:---:|
| Azure Active Directory |● | |
| Azure AD и локальная служба AD с использованием федерации в AD FS |● |● |
| Azure AD и локальная служба AD с использованием Azure AD Connect без синхронизации хэшей паролей или сквозной аутентификации |● |● |
| Azure AD и локальная служба AD с использованием Azure AD Connect с синхронизацией хэшей паролей или сквозной аутентификацией |● | |
| Локальная служба Active Directory | |● |

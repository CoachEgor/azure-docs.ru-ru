# <a name="enable-combined-security-information-registration-in-azure-active-directory"></a>Включить комбинированную регистрацию информации о безопасности в Active Directory Azure

Перед комбинированной регистрацией пользователи отдельно зарегистрировали методы аутентификации для Azure Multi-Factor Authentication и сбросить паролем самообслуживания (SSPR). Люди были смущены тем, что аналогичные методы были использованы для Multi-Фактор аутентификации и SSPR, но они должны были зарегистрироваться для обеих функций. Теперь, с комбинированной регистрацией, пользователи могут зарегистрироваться один раз и получить преимущества как Multi-фактор аутентификации и SSPR.

Прежде чем включить новый опыт, просмотрите статью [Комбинированная регистрация информации о безопасности,](concept-registration-mfa-sspr-combined.md) чтобы убедиться, что вы понимаете функциональность и последствия этой функции.

![Комбинированная регистрация информации по безопасности расширенный опыт](media/howto-registration-mfa-sspr-combined/combined-security-info-more-required.png)

## <a name="enable-combined-registration"></a>Включение объединенной регистрации

Выполнить следующие шаги, чтобы включить комбинированную регистрацию:

1. Вопиюсь на портал Azure в качестве администратора пользователя или глобального администратора.
2. Перейдите к**настройкам** >  **пользователя Azure Active Directory** > **Manage настройки предварительного просмотра функций пользователя.**
3. В соответствии с **функциями предварительного просмотра для регистрации и управления информацией о безопасности,** выбрать для включения для **выбранной** группы пользователей или для **всех** пользователей.

   ![Включить комбинированный опыт предварительного просмотра информации о безопасности для всех пользователей](media/howto-registration-mfa-sspr-combined/enable-the-combined-security-info-preview.png)

> [!NOTE]
> После включения комбинированной регистрации пользователи, которые регистрируют или подтверждают свой номер телефона или мобильное приложение через новый опыт, могут использовать их для мультифакторной аутентификации и SSPR, если эти методы включены в политиках Multi-Factor Authentication и SSPR. Если вы откажете сьюет, пользователи, которые перейдут `https://aka.ms/ssprsetup` на предыдущую страницу регистрации SSPR, должны будут выполнить многофакторную аутентификацию, прежде чем они смогут получить доступ к странице.

Если вы настроили список назначения зоны в Internet Explorer, следующие сайты должны находиться в одной зоне:

* [https://login.microsoftonline.com](https://login.microsoftonline.com)
* [https://mysignins.microsoft.com](https://mysignins.microsoft.com)
* [https://account.activedirectory.windowsazure.com](https://account.activedirectory.windowsazure.com)

## <a name="conditional-access-policies-for-combined-registration"></a>Политики условного доступа для комбинированной регистрации

Обеспечение безопасности, когда и как пользователи регистрируются для сбросить паролем Azure Multi-Factor и сбросить пароли самообслуживания, теперь возможно с помощью действий пользователя в политике условного доступа. Эта функция доступна организациям, которые включили [комбинированную функцию регистрации.](../authentication/concept-registration-mfa-sspr-combined.md) Эта функциональность может быть включена в организациях, где они хотят, чтобы пользователи зарегистрировались для Azure Multi-Factor Authentication и SSPR из центрального местоположения, например, доверенное местоположение сети во время включения в систему HR. Для получения дополнительной информации о создании надежных местоположений в Условиях Доступа см. [What is the location condition in Azure Active Directory Conditional Access?](../conditional-access/location-condition.md#named-locations)

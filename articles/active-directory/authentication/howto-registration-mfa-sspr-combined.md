# <a name="enable-combined-security-information-registration-preview"></a>Включить комбинированную регистрацию информации безопасности (предварительный просмотр)

Прежде чем включить новый опыт, просмотрите статью [Комбинированная регистрация информации безопасности (предварительный просмотр),](concept-registration-mfa-sspr-combined.md) чтобы убедиться, что вы понимаете функциональность и последствия этой функции.

![Комбинированная регистрация информации по безопасности расширенный опыт](media/howto-registration-mfa-sspr-combined/combined-security-info-more-required.png)

|     |
| --- |
| Комбинированная регистрация информации о безопасности для Azure Multi-Factor Authentication и Azure Active Directory (Azure AD) сбросить пароля самообслуживания — это общедоступной функцией предварительного просмотра Azure AD. Для получения дополнительной информации о предварительных просмотрах [см.](https://azure.microsoft.com/support/legal/preview-supplemental-terms/)|
|     |

> [!NOTE]
> Организации, которые включили предыдущий предварительный просмотр для регистрации и управления информацией о безопасности, должны выполнить приведенные ниже шаги, чтобы обеспечить расширенный опыт предварительного просмотра. Для организаций, которые не делают переключатель, 8 октября 2019 года, Microsoft будет переключать пользователей предыдущего предварительного просмотра для регистрации и управления информацией о безопасности на расширенный опыт. 
> 
> Если вы не включили ни одной версии предварительного просмотра, ваша организация не будет затронута.

## <a name="enable-combined-registration"></a>Включение объединенной регистрации

Выполнить следующие шаги, чтобы включить комбинированную регистрацию:

1. Вопиюсь на портал Azure в качестве администратора пользователя или глобального администратора.
2. Перейдите к**настройкам** >  **пользователя Azure Active Directory** > **Manage настройки предварительного просмотра функций пользователя.**
3. В соответствии с **функциями предварительного просмотра для регистрации и управления информацией о безопасности,** выбрать для включения для **выбранной** группы пользователей или для **всех** пользователей.

   ![Включить комбинированный опыт предварительного просмотра информации о безопасности для всех пользователей](media/howto-registration-mfa-sspr-combined/enable-the-combined-security-info-preview.png)

> [!IMPORTANT]
> Начиная с марта 2019 года, параметры телефонного звонка не будут доступны для пользователей Multi-Factor Authentication и SSPR в бесплатных/пробных aD-клиентах. Это изменение не влияет на SMS-сообщения. Параметры телефонного звонка по-прежнему будут доступны пользователям в платных ad-арендаторах Azure.

> [!NOTE]
> После включения комбинированной регистрации пользователи, которые регистрируют или подтверждают свой номер телефона или мобильное приложение через новый опыт, могут использовать их для мультифакторной аутентификации и SSPR, если эти методы включены в политиках Multi-Factor Authentication и SSPR. Если вы откажете сьюет, пользователи, которые перейдут `https://aka.ms/ssprsetup` на предыдущую страницу регистрации SSPR, должны будут выполнить многофакторную аутентификацию, прежде чем они смогут получить доступ к странице.

Если вы настроили список назначения зоны в Internet Explorer, следующие сайты должны находиться в одной зоне:

* [https://login.microsoftonline.com](https://login.microsoftonline.com)
* [https://mysignins.microsoft.com](https://mysignins.microsoft.com)
* [https://account.activedirectory.windowsazure.com](https://account.activedirectory.windowsazure.com)

## <a name="conditional-access-policies-for-combined-registration"></a>Политики условного доступа для комбинированной регистрации

Обеспечение безопасности, когда и как пользователи регистрируются для сбросить паролем Azure Multi-Factor и сбросить пароли самообслуживания, теперь возможно с помощью действий пользователя в политике условного доступа. Эта функция предварительного просмотра доступна организациям, которые включили [комбинированный предварительный просмотр регистрации.](../authentication/concept-registration-mfa-sspr-combined.md) Эта функциональность может быть включена в организациях, где они хотят, чтобы пользователи зарегистрировались для Azure Multi-Factor Authentication и SSPR из центрального местоположения, например, доверенное местоположение сети во время включения в систему HR. Для получения дополнительной информации о создании надежных местоположений в Условиях Доступа см. [What is the location condition in Azure Active Directory Conditional Access?](../conditional-access/location-condition.md#named-locations)

### <a name="create-a-policy-to-require-registration-from-a-trusted-location"></a>Создание политики, требуя регистрации из доверенного места

Следующая политика применяется ко всем выбранным пользователям, которые пытаются зарегистрироваться, используя комбинированный опыт регистрации, и блокирует доступ, если они не подключаются к местоположению, помеченному как доверенная сеть.

![Создание политики CA для контроля за регистрацией информации о безопасности](media/howto-registration-mfa-sspr-combined/require-registration-from-trusted-location.png)

1. На **портале Azure**просмотрите доступ к > **условную безопасность** **безопасности** **Azure Active Directory** > 
1. Выберите **Новая политика**.
1. Во имя введите имя для этой политики. Например, **комбинированная регистрация информации о безопасности в надежных сетях**
1. В **соответствии**с назначениями щелкните **пользователей и групп**и выберите пользователей и группы, к которых вы хотите, чтобы эта политика была применена к

   > [!WARNING]
   > Пользователи должны быть включены для [комбинированного предварительного просмотра регистрации.](../authentication/howto-registration-mfa-sspr-combined.md)

1. В **приложениях или действиях облака**выберите **действия пользователя,** проверьте **информацию о безопасности регистра (предварительный просмотр)**
1. В **условиях** > **локации**

   1. Затем нажмите **Выберите**

1. Затем нажмите **Создать**

# <a name="configure-and-enable-users-for-sms-based-authentication-using-azure-active-directory-preview"></a>Настройка и включение пользователей для проверки подлинности на основе SMS с помощью Active Directory Azure (предварительный просмотр)

Чтобы уменьшить сложность и риски безопасности для пользователей, входящих в приложения и службы, Azure Active Directory (Azure AD) предоставляет несколько вариантов аутентификации. Проверка подлинности на основе SMS, которая в настоящее время находится в предварительном просмотре, позволяет пользователям входить в систему без необходимости предоставлять или даже знать их имя пользователя и пароль. После того, как их учетная запись создана администратором идентификации, они могут ввести свой номер телефона в запросе входа и предоставить код аутентификации, который отправляется им с помощью текстового сообщения. Этот метод проверки подлинности упрощает доступ к приложениям и услугам, особенно для работников, работающих на передовой.

В этой статье показано, как включить проверку подлинности на основе SMS для избранных пользователей или групп в Azure AD.

## <a name="before-you-begin"></a>Подготовка к работе

Для завершения этой статьи необходимы следующие ресурсы и привилегии:

* Активная подписка Azure.
    * Если у вас еще нет подписки Azure, [создайте учетную запись](https://azure.microsoft.com/free/?WT.mc_id=A261C142F).
* Арендатор Active Directory Azure, связанный с подпиской.
    * Если потребуется, [создайте клиент Azure Active Directory][create-azure-ad-tenant] или [свяжите подписку Azure со своей учетной записью][associate-azure-ad-tenant].
* Для включения проверки подлинности на основе SMS-авт.на основе SMS необходимы глобальные привилегии *администратора* в вашем адентичном АТАКЖЕ.
* Каждый пользователь, включенный в политику метода проверки подлинности текстового сообщения, должен быть лицензирован, даже если он не используется. Каждый пользователь с включенным пользователем должен иметь одну из следующих лицензий Azure AD или Microsoft 365:
    * [Azure AD Premium (P1 или P2)][azuread-licensing]
    * [Microsoft 365 (M365) F1 или F3][m365-firstline-workers-licensing]
    * [Корпоративная мобильность - Безопасность (EMS) E3 или E5][ems-licensing] или [Microsoft 365 (M365) E3 или E5][m365-licensing]

## <a name="limitations"></a>Ограничения

Во время публичного предварительного просмотра sMS-аутентификации применяются следующие ограничения:

* Проверка подлинности на основе SMS в настоящее время не совместима с azure Multi-Factor Authentication.
* За исключением команд, проверка подлинности на основе SMS в настоящее время не совместима с приложениями native Office.
* Проверка подлинности на основе SMS не рекомендуется для учетных записей B2B.
* Федеративные пользователи не будут аутентифицировать в доме арендатора. Они только аутентифицировать в облаке.

## <a name="enable-the-sms-based-authentication-method"></a>Включить метод проверки подлинности на основе SMS

Существует три основных шага для включения и использования sMS-аутентификации в вашей организации:

* Включить политику метода проверки подлинности.
* Выберите пользователей или группы, которые могут использовать метод проверки подлинности на основе SMS.
* Назначай номер телефона для каждой учетной записи пользователя.
    * Этот номер телефона может быть присвоен на портале Azure (который показан в этой статье) и в *My Staff* или *My Profile*.

Во-первых, давайте включим проверку подлинности на основе SMS для вашего клиента Azure AD.

1. Вопиюсь на [порталAzв][azure-portal] в качестве *глобального администратора.*
1. Найдите и выберите **Azure Active Directory**.
1. Из меню навигации на левой стороне окна Active Directory Azure выберите **методы проверки безопасности > аутентификации > политики метода проверки подлинности (предварительный просмотр)**.

    [![](media/howto-authentication-sms-signin/authentication-method-policy-cropped.png "Browse to and select the Authentication method policy (preview) window in the Azure portal")](media/howto-authentication-sms-signin/authentication-method-policy.png#lightbox)

1. Из списка доступных методов аутентификации выберите **текстовое сообщение.**
1. Установить **включить** *да*.

    ![Включить проверку подлинности текста в окне политики метода проверки подлинности](./media/howto-authentication-sms-signin/enable-text-authentication-method.png)

    Вы можете включить SMS-проверку подлинности для *всех пользователей* или пользователей и *групп.* В следующем разделе вы включите sMS-проверку подлинности для тестируемого пользователя.

## <a name="assign-the-authentication-method-to-users-and-groups"></a>Назначить метод проверки подлинности пользователям и группам

С помощью sMS-проверки подлинности в вашем атакжесе Azure AD теперь выберите некоторых пользователей или групп, которым будет разрешено использовать этот метод проверки подлинности.

1. В окне политики проверки подлинности текстового сообщения установите **целевой** набор *для выбора пользователей.*
1. Выберите **добавить пользователей или групп,** а затем выбрать тест пользователя или группы, такие как *Contoso пользователей* или *Contoso SMS пользователей.*

    [![](media/howto-authentication-sms-signin/add-users-or-groups-cropped.png "Choose users or groups to enable for SMS-based authentication in the Azure portal")](media/howto-authentication-sms-signin/add-users-or-groups.png#lightbox)

1. Когда вы выберете пользователей или группы, выберите **Select,** а затем **сохраните** обновленную политику метода проверки подлинности.

Каждый пользователь, включенный в политику метода проверки подлинности текстового сообщения, должен быть лицензирован, даже если он не используется. Убедитесь, что у вас есть соответствующие лицензии для пользователей, которые вы включите в политике метода проверки подлинности, особенно когда вы включите функцию для больших групп пользователей.

## <a name="set-a-phone-number-for-user-accounts"></a>Установить номер телефона для учетных записей пользователей

Пользователи теперь включены для проверки подлинности на основе SMS, но их номер телефона должен быть связан с профилем пользователя в Azure AD, прежде чем они смогут войти в систему. Пользователь может [установить этот номер телефона самостоятельно](../user-help/sms-sign-in-explainer.md) в моем *профиле,* или вы можете назначить номер телефона с помощью портала Azure. Номера телефонов могут быть *установлены глобальными админами,* *админами аутентификации*или *привилегированными админами аутентификации.*

Когда номер телефона установлен для SMS-знака, он также доступен для использования с [Azure Multi-фактораутации][tutorial-azure-mfa] и [самообслуживания сбросить пароль.][tutorial-sspr]

1. Найдите и выберите **Azure Active Directory**.
1. Из меню навигации на левой стороне окна Active Directory Azure выберите **пользователей.**
1. Выберите пользователя, включенного для проверки подлинности на основе SMS в предыдущем разделе, например *Contoso User,* а затем выберите **методы аутентификации.**
1. Введите номер телефона пользователя, в том числе код страны, например, *1 xxxxxxxxx.* Портал Azure проверяет номер телефона в правильном формате.

    ![Установите номер телефона для пользователя на портале Azure для использования с помощью проверки подлинности на основе SMS](./media/howto-authentication-sms-signin/set-user-phone-number.png)

    Номер телефона должен быть уникальным в вашем арендатора. Если вы попытаетесь использовать один и тот же номер телефона для нескольких пользователей, отображается сообщение об ошибке.

1. Чтобы применить номер телефона к учетной записи пользователя, выберите **Сохранить**.

При успешной подготовке для *включения SMS-ввоза*появляется контрольный знак.

## <a name="test-sms-based-sign-in"></a>Тест SMS-вхливый

Чтобы протестировать учетную запись пользователя, которая теперь включена для входиной системы на основе SMS, выполните следующие шаги:

1. Откройте новое окно веб-браузера InPrivate или Incognito[https://www.office.com][office]
1. В правом верхнем углу выберите **войти в**.
1. При входе в запрос введите номер телефона, связанный с пользователем в предыдущем разделе, затем выберите **Next**.

    ![Введите номер телефона в запросе входа для пользователя теста](./media/howto-authentication-sms-signin/sign-in-with-phone-number.png)

1. Текстовое сообщение отправляется на предоставленный номер телефона. Чтобы завершить процесс входа, введите 6-значный код, представленный в текстовом сообщении, в запросе входа.

    ![Введите код подтверждения, отправленный с помощью текстового сообщения на номер телефона пользователя](./media/howto-authentication-sms-signin/sign-in-with-phone-number-confirmation-code.png)

1. Пользователь теперь всасывается без необходимости увеять имя пользователя или пароль.

## <a name="troubleshoot-sms-based-sign-in"></a>Неполадка SMS-вхол

Следующие сценарии и шаги устранения неполадок могут использоваться, если у вас есть проблемы с включением и использованием SMS-ввоза.

### <a name="phone-number-already-set-for-a-user-account"></a>Номер телефона, уже установленный для учетной записи пользователя

Если пользователь уже зарегистрировался для azure Multi-Фактор аутентификации и / или самообслуживания сбросить пароль (SSPR), у него уже есть номер телефона, связанный с их учетной записью. Этот номер телефона не доступен автоматически для использования с SMS-вводом.

Пользователю с уже установленным номером телефона для учетной записи отображается кнопка *для включения SMS в* систему на странице **"Мой профиль".** Выберите эту кнопку, и учетная запись включена для использования с SMS-ввоза и предыдущей регистрации многофакторной аутентификации Azure или SSPR.

Для получения дополнительной информации об опыте работы с конечным пользователем, [см.](../user-help/sms-sign-in-explainer.md)

### <a name="error-when-trying-to-set-a-phone-number-on-a-users-account"></a>Ошибка при попытке установить номер телефона в учетной записи пользователя

Если вы получили ошибку при попытке установить номер телефона для учетной записи пользователя на портале Azure, просмотрите следующие шаги устранения неполадок:

1. Убедитесь, что вы включены для предварительного просмотра на основе SMS-ввоза.
1. Подтвердите, что учетная запись пользователя включена в политике проверки подлинности *текстового сообщения.*
1. Убедитесь, что вы установите номер телефона с надлежащей форматирования, как это подтверждено на портале Azure (например, *No 1 4251234567*).
1. Убедитесь, что номер телефона не используется в другом месте вашего арендатора.
1. Проверьте, что на счету нет голосового номера. Если голосовой номер установлен, удалите и повторите попытку позвонить по номеру телефона.

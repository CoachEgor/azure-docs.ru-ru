# <a name="configure-and-enable-users-for-sms-based-authentication-using-azure-active-directory-preview"></a>Настройка и включение пользователей для проверки подлинности на основе SMS с помощью Azure Active Directory (Предварительная версия)

Чтобы снизить сложность и риски безопасности пользователей при входе в приложения и службы, Azure Active Directory (Azure AD) предоставляет несколько вариантов проверки подлинности. Проверка подлинности на основе SMS, в настоящее время на этапе предварительной версии, позволяет пользователям входить без необходимости предоставлять или даже знать имя пользователя и пароль. После создания учетной записи администратором удостоверений они могут ввести свой номер телефона в приглашении на вход и предоставить код проверки подлинности, отправляемый им через текстовое сообщение. Этот метод проверки подлинности упрощает доступ к приложениям и службам, особенно для интерфейсных рабочих ролей.

В этой статье показано, как включить проверку подлинности на основе SMS для выбора пользователей или групп в Azure AD.

## <a name="before-you-begin"></a>Перед началом

Для работы с этой статьей необходимы следующие ресурсы и привилегии:

* Активная подписка Azure.
    * Если у вас еще нет подписки Azure, [создайте учетную запись](https://azure.microsoft.com/free/?WT.mc_id=A261C142F).
* Клиент Azure Active Directory, связанный с подпиской.
    * Если потребуется, [создайте клиент Azure Active Directory][create-azure-ad-tenant] или [свяжите подписку Azure со своей учетной записью][associate-azure-ad-tenant].
* Для включения проверки подлинности на основе SMS требуются права *глобального администратора* в клиенте Azure AD.
* Каждый пользователь, включенный в политике метода проверки подлинности текстовых сообщений, должен иметь лицензию, даже если они не используют ее. Каждый включенный пользователь должен иметь одну из следующих лицензий Azure AD, Office 365 или Microsoft 365:
    * [Azure AD Premium (P1 или P2)][azuread-licensing]
    * [Microsoft 365 (M365) F1 или F3][m365-firstline-workers-licensing] или [Office 365 F1][o365-f1] или [F3][o365-f3]
    * [Enterprise Mobility + Security (EMS) E3][ems-licensing] или в [Microsoft 365 (M365) E3 или][m365-licensing] в.

## <a name="limitations"></a>Ограничения

Во время общедоступной предварительной версии проверки подлинности на основе SMS действуют следующие ограничения.

* В настоящее время проверка подлинности на основе SMS не совместима с многофакторной идентификацией Azure.
* За исключением групп, проверка подлинности на основе SMS в настоящее время не совместима с собственными приложениями Office.
* Для учетных записей B2B не рекомендуется использовать проверку подлинности на основе SMS.
* Федеративные пользователи не проходят проверку подлинности в домашнем клиенте. Они только проходят проверку подлинности в облаке.

## <a name="enable-the-sms-based-authentication-method"></a>Включение метода проверки подлинности на основе SMS

Для включения и использования проверки подлинности на основе SMS в Организации необходимо выполнить три основных шага.

* Включите политику метода проверки подлинности.
* Выберите пользователей или группы, которые могут использовать метод проверки подлинности на основе SMS.
* Назначьте номер телефона для каждой учетной записи пользователя.
    * Этот номер телефона можно назначить в портал Azure (который показан в этой статье), а также в *моем штате* или в моем *профиле*.

Во-первых, давайте разберем проверку подлинности на основе SMS для вашего клиента Azure AD.

1. Войдите в [портал Azure][azure-portal] в качестве *глобального администратора*.
1. Найдите и выберите **Azure Active Directory**.
1. В меню навигации в левой части окна Azure Active Directory выберите **безопасность > методы проверки Подлинности > политика метода проверки подлинности (Предварительная версия)**.

    [![](media/howto-authentication-sms-signin/authentication-method-policy-cropped.png "Browse to and select the Authentication method policy (preview) window in the Azure portal")](media/howto-authentication-sms-signin/authentication-method-policy.png#lightbox)

1. В списке доступных методов проверки подлинности выберите **текстовое сообщение**.
1. Задайте для параметра **включить** значение *Да*.

    ![Включение проверки подлинности текста в окне политики метода проверки подлинности](./media/howto-authentication-sms-signin/enable-text-authentication-method.png)

    Вы можете включить проверку подлинности на основе SMS для *всех пользователей* или *выбрать пользователей* и группы. В следующем разделе описано, как включить проверку подлинности на основе SMS для тестового пользователя.

## <a name="assign-the-authentication-method-to-users-and-groups"></a>Назначение метода проверки подлинности пользователям и группам

После включения проверки подлинности на основе SMS в клиенте Azure AD теперь выберите некоторых пользователей или группы, которым будет разрешено использовать этот метод проверки подлинности.

1. В окне Политика проверки подлинности текстовых сообщений задайте для параметра **Цель** значение *выбрать пользователей*.
1. Выберите **Добавление пользователей или групп**, затем выберите тестового пользователя или группу, например пользователей *contoso* или *contoso SMS*.

    [![](media/howto-authentication-sms-signin/add-users-or-groups-cropped.png "Choose users or groups to enable for SMS-based authentication in the Azure portal")](media/howto-authentication-sms-signin/add-users-or-groups.png#lightbox)

1. Выбрав пользователей или группы, нажмите кнопку **выбрать**, а затем **Сохраните** обновленную политику метода проверки подлинности.

Каждый пользователь, включенный в политике метода проверки подлинности текстовых сообщений, должен иметь лицензию, даже если они не используют ее. Убедитесь, что у вас есть соответствующие лицензии для пользователей, включенных в политике метода проверки подлинности, особенно при включении этой функции для больших групп пользователей.

## <a name="set-a-phone-number-for-user-accounts"></a>Задание номера телефона для учетных записей пользователей

Теперь пользователи включены для проверки подлинности на основе SMS, но их номер телефона должен быть связан с профилем пользователя в Azure AD, прежде чем они смогут войти в систему. Пользователь может [задать этот номер телефона](../user-help/sms-sign-in-explainer.md) в *моем профиле*или назначить номер телефона с помощью портал Azure. Номера телефонов могут устанавливаться *глобальными администраторами*, *администраторами проверки подлинности*или *администраторами привилегированной проверки подлинности*.

Если для входа в SMS задан номер телефона, он также доступен для использования со службой [многофакторной идентификации Azure][tutorial-azure-mfa] и [самостоятельного сброса пароля][tutorial-sspr].

1. Найдите и выберите **Azure Active Directory**.
1. В меню навигации в левой части окна Azure Active Directory выберите **Пользователи**.
1. Выберите пользователя, для которого вы включили проверку подлинности на основе SMS, в предыдущем разделе, например *пользователя Contoso*, а затем выберите **методы проверки подлинности**.
1. Введите номер телефона пользователя, включая код страны, например *+ 1 XXXXXXXXX*. Портал Azure проверяет правильность формата номера телефона.

    ![Задание номера телефона пользователя в портал Azure для использования с проверкой подлинности на основе SMS](./media/howto-authentication-sms-signin/set-user-phone-number.png)

    Номер телефона должен быть уникальным в клиенте. При попытке использовать один и тот же номер телефона для нескольких пользователей отображается сообщение об ошибке.

1. Чтобы применить номер телефона к учетной записи пользователя, нажмите кнопку **сохранить**.

После успешной подготовки появится галочка, чтобы *включить вход в SMS*.

## <a name="test-sms-based-sign-in"></a>Тестирование входа на основе SMS

Чтобы проверить учетную запись пользователя, которая теперь включена для входа на основе SMS, выполните следующие действия.

1. Открытие нового закрытого или режиме инкогнито окна веб-браузера для[https://www.office.com][office]
1. В правом верхнем углу выберите **Вход**.
1. В строке входа введите номер телефона, связанный с пользователем, в предыдущем разделе, а затем нажмите кнопку **Далее**.

    ![Введите номер телефона в приглашении на вход для тестового пользователя](./media/howto-authentication-sms-signin/sign-in-with-phone-number.png)

1. В указанный номер телефона отправляется текстовое сообщение. Чтобы завершить процесс входа, введите 6-значный код, указанный в текстовом сообщении, в приглашении на вход.

    ![Введите код подтверждения, отправленный с помощью текстового сообщения, на номер телефона пользователя.](./media/howto-authentication-sms-signin/sign-in-with-phone-number-confirmation-code.png)

1. Теперь пользователь вошел в учетную запись без указания имени пользователя или пароля.

## <a name="troubleshoot-sms-based-sign-in"></a>Устранение неполадок при входе на сервер SMS

Следующие сценарии и действия по устранению неполадок могут использоваться при возникновении проблем с включением и использованием входа на основе SMS.

### <a name="phone-number-already-set-for-a-user-account"></a>Номер телефона уже задан для учетной записи пользователя

Если пользователь уже зарегистрирован для многофакторной идентификации Azure и (или) самостоятельного сброса пароля (SSPR), у него уже есть номер телефона, связанный с учетной записью. Этот номер телефона не будет автоматически доступен для использования при входе на сервер SMS.

Пользователь с номером телефона, уже установленным для своей учетной записи, выводит кнопку, чтобы *включить вход в SMS* на странице " **Мой профиль** ". Нажмите эту кнопку, и учетная запись будет доступна для использования при входе на основе SMS и предыдущей многофакторной идентификации Azure или регистрации SSPR.

Дополнительные сведения о пользовательском интерфейсе см. в статье [взаимодействие с пользователем при входе в SMS для получения номера телефона (Предварительная версия)](../user-help/sms-sign-in-explainer.md).

### <a name="error-when-trying-to-set-a-phone-number-on-a-users-account"></a>Ошибка при попытке задать номер телефона для учетной записи пользователя

Если при попытке задать номер телефона для учетной записи пользователя в портал Azure появляется сообщение об ошибке, ознакомьтесь со следующими действиями по устранению неполадок.

1. Убедитесь, что вы включили предварительную версию входа на основе SMS.
1. Убедитесь, что учетная запись пользователя включена в политике метода проверки подлинности *текстовых сообщений* .
1. Убедитесь, что для номера телефона задано правильное форматирование, как проверено в портал Azure (например, *+ 1 4251234567*).
1. Убедитесь, что номер телефона не используется в других местах вашего клиента.
1. Установите флажок для учетной записи не задан номер голоса. Если задан номер голоса, удалите его и повторите попытку с номером телефона.

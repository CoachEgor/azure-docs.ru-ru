# <a name="features-and-licenses-for-azure-multi-factor-authentication"></a>Функции и лицензии для многофакторной аутентификации Azure

Для защиты учетных записей пользователей в организации следует использовать многофакторную аутентификацию. Эта функция особенно важна для учетных записей, которые имеют привилегированный доступ к ресурсам. Основные функции многофакторной аутентификации доступны администраторам Office 365 и Azure Active Directory (Azure AD) без дополнительной платы. Если вы хотите обновить функции для ваших админинов или распространить многофакторную аутентификацию на остальных пользователей, вы можете приобрести Многофакторную аутентификацию Azure несколькими способами.

> [!IMPORTANT]
> В этой статье подробно описаны различные способы лицензирования и использования Azure Multi-Factor Authentication. Для получения более подробной информации о ценах и выставлении счетов можно ознакомиться на [странице ценообразования azure Multi-Factor Authentication.](https://azure.microsoft.com/pricing/details/multi-factor-authentication/)

## <a name="available-versions-of-azure-multi-factor-authentication"></a>Доступные версии службы Многофакторной идентификации Azure

Многофакторная аутентификация Azure может использоваться и лицензироваться несколькими способами в зависимости от потребностей организации. Вы уже можете иметь право использовать многофакторную аутентификацию Azure в зависимости от лицензии Azure AD, Office 365, EMS или лицензии Microsoft 365, которая у вас есть в настоящее время. В следующей таблице подробно описаны различные способы получения многофакторной аутентификации Azure и некоторые функции и случаи использования для каждого из них.

| Если вы являетесь пользователем | Возможности и случаи использования |
| --- | --- |
| EMS или Microsoft 365 E3 и E5 | EMS E3 или Microsoft 365 E3 (включает EMS и Office 365) включает azure AD Premium P1. EMS E5 или Microsoft 365 E5 включает в себя Azure AD Premium P2. Вы можете использовать те же функции условного доступа, отмеченные в следующих разделах, чтобы обеспечить многофакторную аутентификацию для пользователей. |
| Azure AD Premium P1 | Условный [доступ Azure AD](../conditional-access/overview.md) можно использовать для того, чтобы подсказывать пользователям многофакторную аутентификацию во время определенных сценариев или событий в соответствии с вашими бизнес-требованиями. |
| Azure AD Premium P2 | Обеспечивает наиболее сильное положение безопасности и улучшенный пользовательский опыт. Добавляет [риск-ориентированный условный доступ](../conditional-access/howto-conditional-access-policy-risk.md) к функциям Azure AD Premium P1, которые адаптируются к шаблонам пользователя и минимизируют многофакторные запросы аутентификации. |
| Office 365 Business Premium, E3 или E5 | Multi-факторная аутентификация Azure включена или отключена для всех пользователей для всех событий, вскакиваемых в систему. Не существует возможности включить многофакторную аутентификацию только для подмножества пользователей или только при определенных сценариях. Управление ведется через портал Office 365. Для улучшения пользовательского опыта продикируйтесь до Azure AD Premium P1 или P2 и используйте Условный доступ. Для получения дополнительной информации [см.](https://support.office.com/article/Set-up-multi-factor-authentication-for-Office-365-users-8f0454b2-f51a-4d9c-bcde-2c48e41621c6) |
| Azure AD уровня "Бесплатный" | Можно использовать [по умолчанию для](../fundamentals/concept-fundamentals-security-defaults.md) включения многофакторной аутентификации для всех пользователей каждый раз, когда делается запрос на проверку подлинности. У вас нет детального управления включенными пользователями или сценариями, но он предоставляет дополнительный шаг безопасности.<br /> Даже если по умолчанию безопасности не используются для включения многофакторной аутентификации для всех, пользователи, назначенные *ролью глобального администратора Azure AD,* могут быть настроены для использования многофакторной аутентификации. Эта функция свободного уровня гарантирует, что критически важные учетные записи администратора защищены многофакторной аутентификацией. |

## <a name="purchase-and-enable-azure-multi-factor-authentication"></a>Покупка и включение многофакторной аутентификации Azure

Чтобы использовать многофакторную аутентификацию Azure, зарегистрируйтесь или купите подходящий уровень Azure AD. Azure AD поставляется в четырех изданиях : Free, Office 365 apps edition (для клиентов Office 365 Business Premium E3, или E5), Premium P1 и Premium P2.

Бесплатное издание включено в подписку Azure. Ниже [приведена информация](#azure-ad-free-tier) о том, как использовать по умолчанию безопасности или защищать учетные записи с ролью *глобального администратора Azure AD.*

Издания Azure AD Premium доступны через представителя корпорации Майкрософт, [программу лицензирования открытого объема](https://www.microsoft.com/licensing/licensing-programs/open-license.aspx)и [программу поставщиков облачных решений.](https://go.microsoft.com/fwlink/?LinkId=614968&clcid=0x409) Подписчики Azure и Office 365 могут приобрести Active Directory Premium через Интернет. [Войти на](https://portal.office.com/Commerce/Catalog.aspx) покупку.

> [!IMPORTANT]
> Лицензирование на основе потребления больше не доступно для новых клиентов с 1 сентября 2018 года. Существующие клиенты, использующие модель, основанную на потреблении, могут продолжать использовать либо на одного пользователя, либо на выставление счетов за аутентификацию.

После приобретения необходимого уровня Azure AD [спланируйте и разместите многофакторную аутентификацию Azure.](howto-mfa-getstarted.md)

### <a name="azure-ad-free-tier"></a>Лазурный AD Бесплатный ярус

Все пользователи в ат-аттепользователе Azure AD Могут использовать аутентификацию Azure Multi-Factor с помощью по умолчанию безопасности. Эти по умолчанию позволяют Azure Multi-Factor аутентификации для всех пользователей, каждый раз, когда они вписываются. Мобильное приложение аутентификации является единственным методом, который может быть использован для многофакторной аутентификации Azure при использовании по умолчанию безопасности Azure AD Free.

* [Подробнее о по умолчанию безопасности Azure AD](../fundamentals/concept-fundamentals-security-defaults.md)
* [Включить по умолчанию для пользователей в Azure AD Free](../fundamentals/concept-fundamentals-security-defaults.md#enabling-security-defaults)

Если вы не хотите включить многофакторную аутентификацию Azure для всех пользователей и каждое событие вхинга, вы можете выбрать только защиту учетных записей пользователей с ролью *глобального администратора Azure AD.* Этот подход предоставляет дополнительные запросы аутентификации для критически важных учетных записей администратора. Вы позволяет включить многофакторную аутентификацию Azure одним из следующих способов, в зависимости от типа учетной записи, которая используется:

* Если вы используете учетную запись Майкрософт, [зарегистрируйтесь для многофакторной аутентификации.](https://support.microsoft.com/help/12408/microsoft-account-about-two-step-verification)
* Если вы не используете учетную запись Майкрософт, [включите многофакторную аутентификацию для пользователя или группы в Azure AD.](howto-mfa-userstates.md)

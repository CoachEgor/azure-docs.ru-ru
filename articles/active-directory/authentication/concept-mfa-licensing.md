# <a name="features-and-licenses-for-azure-multi-factor-authentication"></a>Функции и лицензии для службы многофакторной идентификации Azure

Чтобы защитить учетные записи пользователей в Организации, следует использовать многофакторную проверку подлинности. Эта функция особенно важна для учетных записей, которые имеют привилегированный доступ к ресурсам. Основные функции многофакторной проверки подлинности доступны администраторам Office 365 и Azure Active Directory (Azure AD) без дополнительных затрат. Если вы хотите обновить функции для администраторов или расширить многофакторную проверку подлинности для остальных пользователей, вы можете приобрести многофакторную идентификацию Azure несколькими способами.

> [!IMPORTANT]
> В этой статье описаны различные способы лицензирования и использования многофакторной идентификации Azure. Конкретные сведения о ценах и выставлении счетов см. на [странице с ценами на многофакторную идентификацию Azure](https://azure.microsoft.com/pricing/details/multi-factor-authentication/).

## <a name="available-versions-of-azure-multi-factor-authentication"></a>Доступные версии службы Многофакторной идентификации Azure

Многофакторную идентификацию Azure можно использовать и лицензировать с помощью нескольких различных способов в зависимости от потребностей вашей организации. Возможно, вы уже имеете право использовать многофакторную идентификацию Azure в зависимости от имеющейся лицензии Azure AD, Office 365, EMS или Microsoft 365. В следующей таблице описаны различные способы получения многофакторной идентификации Azure и некоторые функции и варианты использования для каждого из них.

| Если вы являетесь пользователем | Возможности и варианты использования |
| --- | --- |
| EMS или Microsoft 365 E3 и в. | EMS E3 или Microsoft 365 E3 (включая EMS и Office 365) включает Azure AD Premium P1. EMS, или Microsoft 365, включает Azure AD Premium P2. Вы можете использовать те же функции условного доступа, что и в следующих разделах, чтобы предоставить пользователям многофакторную проверку подлинности. |
| Azure AD Premium P1 | Вы можете использовать [Условный доступ Azure AD](../conditional-access/overview.md) , чтобы запрашивать у пользователей многофакторную проверку подлинности в определенных сценариях или событиях в соответствии с бизнес-требованиями. |
| Azure AD Premium P2 | Обеспечивает наивысшую степень безопасности и улучшает взаимодействие с пользователем. Добавляет [Условный доступ на основе рисков](../conditional-access/howto-conditional-access-policy-risk.md) к функциям Azure AD Premium P1, которые адаптируются к шаблонам пользователей и сокращают запросы многофакторной проверки подлинности. |
| Office 365 Business Premium, E3 или «е» | Многофакторная идентификация Azure может быть включена или отключена для всех пользователей для всех событий входа. Невозможно включить многофакторную проверку подлинности только для подмножества пользователей или только в определенных сценариях. Управление осуществляется через портал Office 365. Для повышения удобства работы пользователей выполните обновление до Azure AD Premium P1 или P2 и используйте условный доступ. Дополнительные сведения см. в статье [Защита ресурсов Office 365 с помощью многофакторной проверки подлинности](https://support.office.com/article/Set-up-multi-factor-authentication-for-Office-365-users-8f0454b2-f51a-4d9c-bcde-2c48e41621c6). |
| Azure AD уровня "Бесплатный" | [Параметры безопасности по умолчанию](../fundamentals/concept-fundamentals-security-defaults.md) можно использовать для включения многофакторной проверки подлинности для всех пользователей при каждом запросе проверки подлинности. У вас нет детального контроля над включенными пользователями или сценариями, но он обеспечивает дополнительный этап безопасности.<br /> Даже если параметры безопасности по умолчанию не используются для включения многофакторной проверки подлинности для всех, пользователям, которым назначена роль *глобального администратора Azure AD* , можно настроить использование многофакторной проверки подлинности. Эта функция уровня Free обеспечивает защиту критических учетных записей администраторов с помощью многофакторной проверки подлинности. |

## <a name="purchase-and-enable-azure-multi-factor-authentication"></a>Приобретение и включение многофакторной идентификации Azure

Чтобы использовать многофакторную идентификацию Azure, зарегистрируйте или приобретите подходящего уровня Azure AD. Azure AD поставляется в четырех выпусках — бесплатном выпуске Office 365 Apps (для клиентов Office 365 Business Premium или "Категория 2"), "Премиум P1" и "Премиум P2".

Бесплатный выпуск включен в подписку Azure. Сведения о том, как использовать параметры безопасности по умолчанию или защитить учетные записи с помощью роли *глобального администратора Azure AD* , см. в [разделе ниже](#azure-ad-free-tier) .

Выпуски Azure AD Premium доступны через представителя корпорации Майкрософт, [программы корпоративного лицензирования Open](https://www.microsoft.com/licensing/licensing-programs/open-license.aspx)и [программы поставщиков облачных решений](https://go.microsoft.com/fwlink/?LinkId=614968&clcid=0x409). Подписчики Azure и Office 365 также могут купить Azure Active Directory Premium P1 и P2 в сети. [Войдите](https://portal.office.com/Commerce/Catalog.aspx) в систему, чтобы приобрести.

> [!IMPORTANT]
> Лицензирование на основе потребления больше не доступно для новых клиентов, действующих с 1 сентября 2018 г. Существующие клиенты, использующие модель на основе потребления, могут по-прежнему использовать для каждого включенного пользователя или выставления счетов за проверку подлинности.

После приобретения требуемого уровня Azure AD выполните [планирование и развертывание многофакторной идентификации Azure](howto-mfa-getstarted.md).

### <a name="azure-ad-free-tier"></a>Уровень Azure AD Free

Все пользователи в клиенте Azure AD Free могут использовать многофакторную идентификацию Azure с помощью параметров безопасности по умолчанию. Эти настройки безопасности позволяют включить многофакторную идентификацию Azure для всех пользователей при каждом входе. Приложение для проверки подлинности мобильных устройств — единственный метод, который можно использовать для многофакторной идентификации Azure при использовании Azure AD Free по умолчанию.

* [Дополнительные сведения о параметрах безопасности Azure AD по умолчанию](../fundamentals/concept-fundamentals-security-defaults.md)
* [Включить параметры безопасности по умолчанию для пользователей в Azure AD Free](../fundamentals/concept-fundamentals-security-defaults.md#enabling-security-defaults)

Если вы не хотите включать многофакторную идентификацию Azure для всех пользователей и событий входа, вы можете использовать только защиту учетных записей пользователей с ролью *глобального администратора Azure AD* . Этот подход предоставляет дополнительные запросы проверки подлинности для критически важных учетных записей администраторов. Включить многофакторную идентификацию Azure можно одним из следующих способов в зависимости от типа используемой учетной записи.

* Если вы используете учетную запись Майкрософт, [Зарегистрируйтесь для использования многофакторной проверки подлинности](https://support.microsoft.com/help/12408/microsoft-account-about-two-step-verification).
* Если вы не используете учетную запись Майкрософт, [включите многофакторную проверку подлинности для пользователя или группы в Azure AD](howto-mfa-userstates.md).

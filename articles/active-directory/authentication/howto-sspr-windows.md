# <a name="how-to-enable-password-reset-from-the-windows-login-screen"></a>Как: Включить сбросить пароль с экрана входа в Windows

Для машин с Windows 7, 8, 8.1 и 10 вы можете позволить пользователям сбросить свой пароль на экране входа windows. Пользователям больше не нужно искать устройство с веб-браузером для доступа к [порталу SSPR.](https://aka.ms/sspr)

![Пример Windows 7 и 10 экранов входа с SSPR ссылка показано](./media/howto-sspr-windows/windows-reset-password.png)

## <a name="general-limitations"></a>Общие ограничения

- Сбросить пароль в настоящее время не поддерживается с помощью удаленного рабочего стола или от hyper-V расширенных сессий.
- Известно, что некоторые поставщики учетных данных сторон ы на сторону не могут вызвать проблемы с этой функцией.
- Отключение UAC через модификацию [ключа реестра EnableLUA,](https://docs.microsoft.com/openspecs/windows_protocols/ms-gpsb/958053ae-5397-4f96-977f-b7700ee461ec) как известно, вызывает проблемы.
- Эта функция не работает для сетей с 802.1x сетевой аутентификации развернуты и опция "Выполнить непосредственно перед пользователем logon". Чтобы включить эту функцию для сетей с развернутыми решением аутентификации 802.1X, используйте аутентификацию компьютера.
- Гибридные машины Azure AD должны иметь линию видимости сетевого подключения к контроллеру домена, чтобы использовать новый пароль и обновлять кэшированные учетные данные.
- При использовании изображения, перед запуском sysprep убедитесь, что веб-кэш очищается для встроенного администратора до выполнения шага CopyProfile. Более подробную информацию об этом шаге можно найти в статье поддержки [Performance poor при использовании пользовательского профиля пользователя по умолчанию.](https://support.microsoft.com/help/4056823/performance-issue-with-custom-default-user-profile)
- Следующие настройки, как известно, мешают возможности использования и сброса паролей на устройствах Windows 10
    - Если согласно политике версий Windows 10 до 1809 нужно нажать CTRL+ALT+DEL, **Сброс пароля** не будет работать.
    - Если уведомления на экране блокировки отключены, **Сброс пароля** не будет работать.
    - HideFastUserSwitching со значением Enabled (Вкл.) или 1;
    - DontDisplayLastUserName со значением Enabled (Вкл.) или 1;
    - NoLockScreen со значением Enabled (Вкл.) или 1;
    - EnableLostMode, установленный на устройстве;
    - файл Explorer.exe, замененный на файл пользовательской оболочки.
- Сочетание следующих трех параметров может привести к тому, что эта функция не сработает.
    - Интерактивный логон: Не требуйте CTRL-ALT-DEL
    - ОтключитьУведомления LockScreenApp - 1 или включенный
    - IsContentDeliveryPolicyEnforced No 1 или True

## <a name="windows-10-password-reset"></a>Сбросить пароли Windows 10

### <a name="windows-10-prerequisites"></a>Необходимые компоненты для Windows 10

- Администратор должен включить сбросить пароль самообслуживания Azure AD с портала Azure.
- **Пользователи должны зарегистрироваться для SSPR перед использованием этой функции**
- Требования к прокси-серверам сети
   - Устройства под управлением Windows 10 
       - Порт 443 `passwordreset.microsoftonline.com` в и`ajax.aspnetcdn.com`
       - Устройства Windows 10 поддерживают только конфигурацию прокси-сервера машинного уровня
- Запуск по крайней мере Windows 10, версия Апрель 2018 Обновление (v1803), и устройства должны быть либо:
    - присоединение к Azure AD;
    - присоединение к Azure AD (гибридные устройства).

### <a name="enable-for-windows-10-using-intune"></a>Включить для Windows 10 с помощью Intune

Самый гибкий метод — это развертывание изменений конфигурации для включения сброса пароля в окне входа в систему с помощью Intune. Intune позволяет развертывать изменения конфигурации для определенной группы компьютеров. Этот метод требует регистрации устройства в Intune.

#### <a name="create-a-device-configuration-policy-in-intune"></a>Создание политики конфигурации устройства в Intune

1. Войдите на [портал Azure](https://portal.azure.com) и щелкните **Intune**.
1. Создание нового профиля конфигурации устройства, перейдя в > **профиль конфигурации** >  **устройства,****создать профиль**
   - Введите понятное имя для профиля.
   - При необходимости введите содержательное описание профиля.
   - Выберите платформу **Windows 10 и более поздних версий**.
   - Для типа профиля укажите **Пользовательский**.
1. Настройка **настроек**
   - В разделе "Параметры OMA-URI" **добавьте** следующий параметр, чтобы включить отображение ссылки сброса пароля.
      - Введите понятное имя, которое указывает на действия параметра.
      - При необходимости введите содержательное описание параметра.
      - Для параметра **OMA-URI** укажите значение `./Vendor/MSFT/Policy/Config/Authentication/AllowAadPasswordReset`.
      - Для параметра **Тип данных** задайте значение **Целое число**.
      - Для параметра **Значение** установите значение **1**.
      - Нажмите кнопку **ОК**
   - Нажмите кнопку **ОК**
1. Нажмите кнопку **Создать**.
1. Эта политика может быть назначена определенным пользователям, устройствам или группам. Более подробную информацию можно найти в статье [Назначаем пользователей и профилей устройств в Microsoft Intune](https://docs.microsoft.com/intune/device-profile-assign).

### <a name="enable-for-windows-10-using-the-registry"></a>Включить Windows 10 с помощью реестра

1. Войдите в Windows PC, используя административные учетные данные.
1. Запустите **regedit** от имени администратора.
1. Настройте следующий раздел реестра:
   - `HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\AzureADAccount`
      - `"AllowPasswordReset"=dword:00000001`

#### <a name="troubleshooting-windows-10-password-reset"></a>Устранение проблем Windows 10 сбросить пароль

Журнал аудита Azure AD включают в себя сведения об IP-адресе и типе клиента (ClientType), на котором произошел сброс пароля.

![Пример сбросить пароля Windows 7 в журнале аудита Azure AD](media/howto-sspr-windows/windows-7-sspr-azure-ad-audit-log.png)

Когда пользователи сбросили свой пароль с экрана входа устройства Windows 10, `defaultuser1` создается временная учетная запись с низкой привилегией. Эта учетная запись используется для обеспечения безопасности процесса сброса пароля. Сама учетная запись имеет случайно сгенерированный пароль, не отображается для входе устройства в систему и автоматически удаляется после того, как пользователь сбросит свой пароль. Несколько `defaultuser` профилей могут существовать, но могут быть безопасно проигнорированы.

## <a name="windows-7-8-and-81-password-reset"></a>Сбросить пароля Windows 7, 8 и 8.1

### <a name="windows-7-8-and-81-prerequisites"></a>Windows 7, 8 и 8.1 предпосылки

- Администратор должен включить сбросить пароль самообслуживания Azure AD с портала Azure.
- **Пользователи должны зарегистрироваться для SSPR перед использованием этой функции**
- Требования к прокси-серверам сети
   - Устройства Windows 7, 8 и 8.1
       - Порт 443 к`passwordreset.microsoftonline.com`
- В операционной системе Windows 7 или Windows 8.1 должны быть установлены исправления.
- Должен быть включен протокол TLS 1.2 (см. руководство по [настройке параметров реестра для протокола TLS](https://docs.microsoft.com/windows-server/security/tls/tls-registry-settings#tls-12)).
- Если на вашей машине включено более одного поставщика учетных данных третьей стороны, пользователи увидят на экране входа более одного профиля пользователя.

> [!WARNING]
> TLS 1.2 должен быть включен, а не просто установить на автоматическое согласование

### <a name="install"></a>Установка

1. Скачайте соответствующий установщик для версии Windows, в которой следует включить этот протокол.
   - Программное обеспечение доступно в центре загрузки Майкрософт по адресу:[https://aka.ms/sspraddin](https://aka.ms/sspraddin)
1. Войдите на компьютер, где следует выполнить установку, и запустите программу установки.
1. После установки настоятельно рекомендуется выполнить перезагрузку.
1. После перезагрузки на экране входа выберите пользователя и нажмите кнопку "Забыли пароль?" чтобы инициировать процесс сброса пароля.
1. Завершите процесс, следуя указаниям на экране для сброса пароля.

![Пример для Windows 7 со ссылкой "Забыли пароль?", Поток SSPR](media/howto-sspr-windows/windows-7-sspr.png)

#### <a name="silent-installation"></a>Автоматическая установка

- Для бесшумной установки используйте команду "msiexec/i SsprWindowsLogon.PROD.msi/qn"
- Для бесшумной удаления используйте команду "msiexec /x SsprWindowsLogon.PROD.msi/qn"

#### <a name="troubleshooting-windows-7-8-and-81-password-reset"></a>Сбросить пароля Windows 7, 8, 8 и 8.1

События регистрируются в журнале как на компьютере, так и в Azure AD. События Azure AD включают в себя сведения об IP-адресе и типе клиента (ClientType), на котором произошел сброс пароля.

![Пример сбросить пароля Windows 7 в журнале аудита Azure AD](media/howto-sspr-windows/windows-7-sspr-azure-ad-audit-log.png)

При необходимости для включения подробного ведения журнала можно изменить соответствующий раздел реестра на компьютере. Подробное ведение журнала следует включать только для устранения неполадок.

`HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Authentication\Credential Providers\{86D2F0AC-2171-46CF-9998-4E33B3D7FD4F}`

- Для обеспечения многословной `REG_DWORD: "EnableLogging"`регистрации создайте и установите его на 1.
- Чтобы отключить многословную лесозаготовки, измените `REG_DWORD: "EnableLogging"` на 0.

## <a name="what-do-users-see"></a>Что видят пользователи?

Теперь, когда вы настроили сбросить пароли для устройств Windows, какие изменения для пользователя? Как они узнают, что можно сбросить пароль на экране входа в систему?

![Пример Windows 7 и 10 экранов входа с SSPR ссылка показано](./media/howto-sspr-windows/windows-reset-password.png)

Когда пользователи пытаются войти в систему, они теперь видят **пароль снаважа** или забыли ссылку **на пароль,** которая открывает возможность сбросить пароль самообслуживания на экране входа. С помощью этой функции пользователи могут сбросить пароль, не используя другое устройство для получения доступа к браузеру.

Инструкции по использованию этой функции пользователи могут найти в статье [Я не помню свой пароль Azure AD](../user-help/active-directory-passwords-update-your-own-password.md).

# <a name="assign-azure-ad-roles-in-privileged-identity-management"></a>Назначать функции Azure AD в привилегированном управлении идентификацией

С помощью Active Directory Azure (Azure AD) глобальный администратор может делать **постоянные** назначения ролей администратора Azure Ad. Эти назначения ролей можно создать с помощью [портала Azure](../users-groups-roles/directory-assign-admin-roles.md) или [команд PowerShell](/powershell/module/azuread#directory_roles).

Служба управления привилегированными удостоверениями данных Azure AD (PIM) также позволяет администраторам привилегированных ролей делать постоянные назначения ролей администратора. Кроме того, привилегированные администраторы ролей могут дать пользователям **право на** функции администратора Ad Azure. Такой разрешенный администратор может активировать роль при необходимости. Срок действия его разрешений истекает, когда роль больше не требуется ему для работы.

## <a name="determine-your-version-of-pim"></a>Определите свою версию PIM

Начиная с ноября 2019 года, часть ролей Azure AD в Привилегированном управлении идентификацией обновляется в новую версию, которая соответствует опыту для ролей ресурсов Azure. Это создает дополнительные функции, а также [изменения в существующем API.](azure-ad-roles-features.md#api-changes) В то время как новая версия в настоящее время развернута, какие процедуры, которые вы следуете в этой статье зависит от версии Привилегированного управления идентификацией у вас в настоящее время. Выполните действия в этом разделе, чтобы определить, какая версия привилегированного управления идентификацией у вас есть. После того, как вы знаете свою версию Privileged Identity Management, вы можете выбрать процедуры в этой статье, которые соответствуют этой версии.

1. Вопием на [портал Azure](https://portal.azure.com/) с пользователем, который занимается ролью [администратора привилегированных.](../users-groups-roles/directory-assign-admin-roles.md#privileged-role-administrator)
1. Открытое **управление привилегированным удостоверением личности Azure AD.** Если у вас есть баннер в верхней части страницы обзора, следуйте инструкциям в **новой версии** вкладке этой статьи. В противном случае следуйте инструкциям во вкладке **Предыдущая версия.**

## <a name="assign-a-role"></a>Назначение роли

Выполните следующие действия, чтобы пользователь имел право на роль админа Ad Azure.

1. Войти на [портал Azure](https://portal.azure.com/) с пользователем, который является участником роли [администратора привилегированных](../users-groups-roles/directory-assign-admin-roles.md#privileged-role-administrator) ролей.

    Для получения информации о том, как предоставить другому администратору доступ к управлению привилегированными удостоверениями, можно [получить доступ к другим администраторам для управления привилегированным управлением идентификацией.](pim-how-to-give-access-to-pim.md)

1. Открытое **управление привилегированным удостоверением личности Azure AD.**

1. Выберите **роли AD Azure**.

1. Выберите **роли,** чтобы просмотреть список ролей для разрешений Azure AD.

    ![Роли Azure AD](./media/pim-how-to-add-role-to-user/roles-list.png)

1. Выберите **участника Добавить,** чтобы открыть страницу **нового назначения.**

1. Выберите **роль,** чтобы открыть страницу «Выберите роль».

    ![Панель нового назначения](./media/pim-how-to-add-role-to-user/select-role.png)

1. Выберите роль, которую вы намерены назначить, и щелкните **Выбрать**.

1. Выберите участника, которому вы хотите назначить роль, а затем **выберите.**

1. В списке **типов назначений** в **панели настроек параметров членства** выберите **подходящий** или **активный**вариант .

    - **Соответствующие** назначения требуют, чтобы участник роли выполнял действие для использования роли. Поддерживаются такие действия, как прохождение многофакторной проверки подлинности (MFA), предоставление коммерческого обоснования или запрос утверждения от назначенных утверждающих.

    - **Активные** назначения не требуют от участника выполнения каких-либо действий для использования роли. Члены, назначенные в качестве активных, постоянно имеют все полномочия, присвоенные роли.

1. Если назначение должно быть постоянным (постоянно подходящим или постоянно назначенным), выберите **постоянно** флажок.

    В зависимости от параметров роли флажок может не отображаться или быть неизменяемым.

1. Чтобы указать конкретную продолжительность назначения, снимите флажок и измените поля даты и времени начала и окончания. После завершения, выберите **Готово**.

    ![Параметры членства — дата и время](./media/pim-how-to-add-role-to-user/start-and-end-dates.png)

1. Чтобы создать новое назначение ролей, выберите **Добавить**. Отображается уведомление о состоянии.

    ![Новое назначение — уведомление](./media/pim-how-to-add-role-to-user/assignment-notification.png)

## <a name="update-or-remove-an-existing-role-assignment"></a>Обновление или удаление существующего назначения роли

Выполните следующие действия, чтобы обновить или удалить существующее назначение роли.

1. Открытое **управление привилегированным удостоверением личности Azure AD.**

1. Выберите **роли AD Azure**.

1. Выберите **роли,** чтобы просмотреть список ролей для Azure AD.

1. Щелкните роль, которую нужно обновить или удалить.

1. Найдите назначение роли на вкладках **Доступные роли** или **Активные роли**.

    ![Обновление или удаление назначения роли](./media/pim-how-to-add-role-to-user/remove-update-assignments.png)

1. Нажмите кнопку **Обновить** или **Удалить**, чтобы обновить или удалить назначение роли.

## <a name="make-a-user-eligible-for-a-role"></a>Предоставление пользователю разрешения на роль

Выполните следующие действия, чтобы пользователь имел право на роль админа Ad Azure.

1. Выберите **роли** или **участников.**

    ![Роли Azure AD](./media/pim-how-to-add-role-to-user/pim-directory-roles.png)

1. Выберите **добавить участника,** чтобы открыть **управляемые участники Add.**

1. Выберите **роль,** выберите роль, которую вы хотите управлять, а затем **выберите.**

    ![Выбор роли](./media/pim-how-to-add-role-to-user/pim-select-a-role.png)

1. Выберите **участников,** выберите пользователей, которых вы хотите назначить роли, а затем **выберите.**

    ![Выбор роли](./media/pim-how-to-add-role-to-user/pim-select-members.png)

1. В **добавлении управляемых членов,** выберите **OK,** чтобы добавить пользователя в роль.

1. В списке ролей выберите только что назначенную роль, чтобы увидеть список участников.

     Когда роль назначена, выбранный вами пользователь будет отображаться в списке участников как **подходящий** для роли.

    ![Пользователь, имеющий разрешение на роль](./media/pim-how-to-add-role-to-user/pim-directory-role-eligible.png)

1. Теперь, когда пользователь имеет право на роль, дайте ему знать, что он может активировать ее в соответствии с инструкциями в [функционативе ролей Azure AD в Privileged Identity Management.](pim-how-to-activate-role.md)

    Во время активации разрешенным администраторам будет предложено зарегистрироваться в службе "Многофакторная идентификация Azure" (MFA). Если пользователь не может зарегистрироваться на МИД или использует @outlook.comучетную запись Майкрософт (например), вам необходимо сделать его постоянным во всех своих ролях.

## <a name="make-a-role-assignment-permanent"></a>Настройка постоянного назначения роли

По умолчанию новые пользователи *имеют право* только на роль админа Ad Azure. Если вы хотите сделать назначение роли постоянным, выполните следующие действия.

1. Открытое **управление привилегированным удостоверением личности Azure AD.**

1. Выберите **роли AD Azure**.

1. Выберите **Элементы**.

    ![Список участников](./media/pim-how-to-add-role-to-user/pim-directory-role-list-members.png)

1. Выберите **подходяую** роль, которую вы хотите сделать постоянной.

1. Выберите **больше,** а затем выберите **Сделать Пермь**.

    ![Настройка постоянного назначения роли](./media/pim-how-to-add-role-to-user/pim-make-perm.png)

    Теперь роль отображается как **постоянная**.

    ![Список участников с постоянными ролями](./media/pim-how-to-add-role-to-user/pim-directory-role-list-members-permanent.png)

## <a name="remove-a-user-from-a-role"></a>Удаление пользователя из роли

Вы можете удалить пользователей из ролевых заданий, но убедитесь, что всегда есть хотя бы один пользователь, который является постоянным глобальным администратором. Если вам неизвестно, каким пользователям по-прежнему требуются назначенные им роли, можно [запустить проверку доступа для роли](pim-how-to-start-security-review.md).

Выполните следующие действия, чтобы удалить определенного пользователя из роли админа Ad Azure.

1. Открытое **управление привилегированным удостоверением личности Azure AD.**

1. Выберите **роли AD Azure**.

1. Выберите **Элементы**.

    ![Список участников](./media/pim-how-to-add-role-to-user/pim-directory-role-list-members.png)

1. Выберите назначение роли, которое необходимо удалить.

1. Выберите **больше,** а затем выберите **Удалить**.

    ![Удаление роли](./media/pim-how-to-add-role-to-user/pim-remove-role.png)

1. В сообщении, которое просит вас подтвердить, выберите **Да**.

    ![Удаление роли](./media/pim-how-to-add-role-to-user/pim-remove-role-confirm.png)

    Назначение роли удалено.

## <a name="authorization-error-when-assigning-roles"></a>Ошибка авторизации при назначении ролей

Если недавно вы включили Privileged Identity Management для подписки и получили ошибку авторизации при попытке сделать пользователя, имеющего право на роль админ-аминатора Azure, это может быть связано с тем, что у директора службы MS-PIM пока нет соответствующих разрешений. Директор службы MS-PIM должен иметь роль [администратора пользовательского доступа](../../role-based-access-control/built-in-roles.md#user-access-administrator) для присвоения ролей другим пользователям. Чтобы не ждать, пока субъекту-службе MS-PIM будет назначена роль "Администратор доступа пользователей", вы можете назначить ее вручную.

Выполните следующие действия, чтобы назначить роль "Администратор доступа пользователей" субъекту-службе MS-PIM для подписки.

1. Войдите на портал Azure как глобальный администратор.

1. Выберите **Все службы** и щелкните **Подписки**.

1. Выберите свою подписку.

1. Выберите **Управление доступом (IAM)**.

1. Выберите **Назначения ролей**, чтобы просмотреть текущий список назначений ролей в области действия подписки.

   ![Колонка "Управление доступом (IAM)" для подписки](./media/pim-how-to-add-role-to-user/ms-pim-access-control.png)

1. Проверьте, назначена ли субъекту-службе **MS-PIM** роль **Администратор доступа пользователей**.

1. Если она не назначена, выберите **Добавить назначения ролей**, чтобы открыть панель **Добавление назначения ролей**.

1. Из раскрывающегося списка **Роли** выберите роль **Администратор доступа пользователей**.

1. В списке **Выбор** найдите и выберите субъект-службу **MS-PIM**.

   ![Добавление панели назначения ролей - Добавление разрешений для основного обслуживания MS-PIM](./media/pim-how-to-add-role-to-user/ms-pim-add-permissions.png)

1. Чтобы присвоить роль, выберите **Сохранить**.

   Через несколько секунд субъекту-службе MS-PIM будет назначена роль "Администратор доступа пользователей" на уровне подписки.

   ![Страница управления доступом, показывающая назначение роли админ-аммин-интерфейса пользователя для основного обслуживания MS-PIM](./media/pim-how-to-add-role-to-user/ms-pim-user-access-administrator.png)

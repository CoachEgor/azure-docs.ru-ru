# <a name="view-audit-history-for-azure-ad-roles-in-privileged-identity-management"></a>Просмотр истории аудита для ролей Azure AD в привилегированном управлении идентификацией

Вы можете использовать историю аудита Privileged Identity Management (PIM), чтобы просмотреть все назначения и активации ролей в течение последних 30 дней для всех привилегированных ролей. Если вы хотите увидеть полную историю деятельности аудита в организации Azure Active Directory (Azure AD), включая действия администратора, конечного пользователя и синхронизации, вы можете использовать [отчеты безопасности и активности Azure Active Directory.](../reports-monitoring/overview-reports.md)

## <a name="determine-your-version-of-pim"></a>Определите свою версию PIM

Начиная с ноября 2019 года, часть ролей Azure AD в Привилегированном управлении идентификацией обновляется в новую версию, которая соответствует опыту для ролей ресурсов Azure. Это создает дополнительные функции, а также [изменения в существующем API.](azure-ad-roles-features.md#api-changes) В то время как новая версия в настоящее время развернута, какие процедуры, которые вы следуете в этой статье зависит от версии Привилегированного управления идентификацией у вас в настоящее время. Выполните действия в этом разделе, чтобы определить, какая версия привилегированного управления идентификацией у вас есть. После того, как вы знаете свою версию Privileged Identity Management, вы можете выбрать процедуры в этой статье, которые соответствуют этой версии.

1. Вопием на [портал Azure](https://portal.azure.com/) с пользователем, который занимается ролью [администратора привилегированных.](../users-groups-roles/directory-assign-admin-roles.md#privileged-role-administrator)
1. Открытое **управление привилегированным удостоверением личности Azure AD.** Если у вас есть баннер в верхней части страницы обзора, следуйте инструкциям в **новой версии** вкладке этой статьи. В противном случае следуйте инструкциям во вкладке **Предыдущая версия.**

    [![Лазурный AD роми новой версии](media/pim-how-to-use-audit-log/directory-roles-audit-history.png "Выберите вкладку для версии")](media/pim-how-to-use-audit-log/directory-roles-audit-history.png)

Выполните следующие действия, чтобы просмотреть историю аудита для ролей Azure AD.

## <a name="view-resource-audit-history"></a>Просмотр журнала аудита ресурсов

Аудит ресурсов дает представление обо всех действиях, связанных с ролями Azure AD.

1. Открытое **управление привилегированным удостоверением личности Azure AD.**

1. Выберите **роли AD Azure**.

1. Выберите **аудит ресурсов**.

1. Отфильтруйте журнал по предопределенной дате или настраиваемому диапазону.

    ![Список аудита ресурсов с фильтрами](media/azure-pim-resource-rbac/rbac-resource-audit.png)

## <a name="view-my-audit"></a>Просмотр раздела "Мой аудит"

В разделе "Мой аудит" можно просматривать действия, выполняемые в личной роли.

1. Открытое **управление привилегированным удостоверением личности Azure AD.**

1. Выберите **роли AD Azure**.

1. Выберите ресурс, для просмотра истории аудита.

1. Выберите **мой аудит**.

1. Отфильтруйте журнал по предопределенной дате или настраиваемому диапазону.

    ![Список аудита для текущего пользователя](media/azure-pim-resource-rbac/my-audit-time.png)

## <a name="view-audit-history"></a>Просмотр журнала аудита

Выполните следующие действия, чтобы просмотреть историю аудита для ролей Azure AD.

1. Войдите на [портал Azure](https://portal.azure.com/) с учетными данными пользователя, который является участником роли [Администратор привилегированных ролей](../users-groups-roles/directory-assign-admin-roles.md#privileged-role-administrator).

1. Открытое **управление привилегированным удостоверением личности Azure AD.**

1. Выберите **роли AD Azure**.

1. Выберите **историю аудита ролей каталога**.

    В зависимости от истории аудита отображается диаграмма столбца вместе с общими активациями, максимальными активациями в день и средними активациями в день.

    [![Лазурный AD роми новой версии](media/pim-how-to-use-audit-log/directory-roles-audit-history.png "Просмотр истории аудита ролей каталога")](media/pim-how-to-use-audit-log/directory-roles-audit-history.png)

    В нижней части страницы отображается таблица с информацией о каждом действии в доступной истории аудита. Столбцы имеют следующие значения:

    | Столбец | Описание |
    | --- | --- |
    | Time | Когда произошло действие. |
    | Requestor | Пользователь, запросивший активацию или изменение роли. Если значение — **система Azure,** проверьте историю аудита Azure для получения дополнительной информации. |
    | Действие | Действия, принятые запрашивателем. Действия могут включать в себя назначать, не назначать, активировать, деактивировать или добавленыOutsidePIM. |
    | Участник | Пользователь, который активирует или назначается на роль. |
    | Роль | Роль, назначенная или активированная пользователем. |
    | Рассуждения | Текст, который был введен в поле причины во время активации. |
    | Окончание срока действия | Когда активация роли истекает. Применяется только к подходящим ролевым назначениям. |

1. Чтобы сортировать историю аудита, нажмите кнопки **Time,** **Action**и **Role.**

## <a name="filter-audit-history"></a>Фильтрация журнала аудита

1. В верхней части страницы истории аудита щелкните кнопку **«Фильтр».**

    Отображается **панель параметров диаграммы обновления.**

1. В **диапазоне времени**выберите временной диапазон.

1. В **ролях**выберите флажки, чтобы указать роли, которые вы хотите просмотреть.

    ![Панель параметров диаграммы обновления](media/pim-how-to-use-audit-log/update-chart-parameters.png)

1. Выберите **Готово** для просмотра отфильтроченной истории аудита.

## <a name="get-reason-approver-and-ticket-number-for-approval-events"></a>Получить разум, утверждение и номер билета для проведения мероприятий по утверждению

1. Вопийте на [портале Azure](https://aad.portal.azure.com) с разрешениями администратора привилегированных ролей и открывайте AD Azure.
1. Выберите **журналы аудита**.
1. Используйте фильтр **Службы** для отображения только событий аудита для службы privileged identity Management. На странице **журналов аудита** можно:

    - См, что произошло для события аудита, в столбце **причины состояния.**
    - См. утверждение в колонке **«Инициированный (актером)** для события «добавить участника в утвержденный запрос роли».

    [![Лазурный AD роми новой версии](media/pim-how-to-use-audit-log/filter-audit-logs.png "Фильтр журнала аудита для службы PIM")](media/pim-how-to-use-audit-log/filter-audit-logs.png)

1. Выберите событие журнала аудита, чтобы увидеть номер билета на вкладке **Activity** панели **«Детали».**
  
    [![Лазурный AD роми новой версии](media/pim-how-to-use-audit-log/audit-event-ticket-number.png "Проверьте номер билета на аудиторское мероприятие")](media/pim-how-to-use-audit-log/audit-event-ticket-number.png)

1. Вы можете просмотреть запрашивающий (лицо, активизировавшее роль) на вкладке **«Цели»** панели **«Детали»** для события аудита. Для ролей Azure AD существует два целевых типа:

    - Роль **(Тип** и роль)
    - Запрашиваете:**Тип** пользователя

Как правило, событие журнала аудита непосредственно над событием утверждения является событием для "Добавить участника к завершенной роли", где **Инициатором (актером)** является запрашивателем. В большинстве случаев вам не нужно будет находить запрашивающего в запросе утверждения с точки зрения аудита.

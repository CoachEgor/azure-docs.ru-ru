# <a name="assign-azure-resource-roles-in-privileged-identity-management"></a>Назначать роли ресурсов Azure в привилегированном управлении identity

Функциональное управление иждивестью Azure (Azure AD) Привилегированное управление identity (PIM) может управлять встроенными ролями ресурсов Azure, а также пользовательскими ролями, втомим (но не ограничивающимися):

- Владелец
- Администратор доступа пользователей
- Участник
- администратор безопасности;
- Диспетчер безопасности

> [!NOTE]
> Пользователи или члены группы, назначенные функциям подписки "Владелец" или Администратору пользовательского доступа, а также администраторы Azure AD, которые позволяют управлению подпиской в Azure AD, по умолчанию имеют разрешения администратора ресурсов. Эти администраторы могут назначать роли, настраивать параметры ролей и просматривать доступ с помощью Privileged Identity Management для ресурсов Azure. Пользователь не может управлять привилегированным управлением идентификацией ресурсов без разрешения администратора ресурсов. Просмотрите список [встроенных ролей ресурсов Azure](../../role-based-access-control/built-in-roles.md).

## <a name="assign-a-role"></a>Назначение роли

Выполните следующие действия, чтобы предоставить пользователю разрешение для роли ресурса Azure AD.

1. Войти на [портал Azure](https://portal.azure.com/) с пользователем, который является участником роли [администратора привилегированных](../users-groups-roles/directory-assign-admin-roles.md#privileged-role-administrator) ролей.

    Для получения информации о том, как предоставить другому администратору доступ к управлению привилегированными удостоверениями, можно [получить доступ к другим администраторам для управления привилегированным управлением идентификацией.](pim-how-to-give-access-to-pim.md)

1. Открытое **управление привилегированным удостоверением личности Azure AD.**

1. Щелкните **Ресурсы Azure**.

1. Используйте **Фильтр ресурсов**, чтобы отфильтровать список управляемых ресурсов.

    ![Список ресурсов Azure для управления](./media/pim-resource-roles-assign-roles/resources-list.png)

1. Выберите ресурс, которым вы хотите управлять, например группу по подписке или управлению.

1. В «Управлении» выберите **роли,** чтобы просмотреть список ролей для ресурсов Azure.

    ![Роли ресурсов Azure](./media/pim-resource-roles-assign-roles/resources-roles.png)

1. Выберите **участник добавить,** чтобы открыть новое стекло назначения.

1. Выберите **роль,** чтобы открыть панель ролей.

    ![Панель нового назначения](./media/pim-resource-roles-assign-roles/resources-select-role.png)

1. Выберите роль, которую вы намерены назначить, и щелкните **Выбрать**.

    Откроется панель выбора участника или группы.

1. Выберите участника или группу, которую вы хотите назначить роли, а затем нажмите **Select.**

    ![Панель выбора участника или группы](./media/pim-resource-roles-assign-roles/resources-select-member-or-group.png)

    Откроется панель параметров членства.

1. В списке **Тип назначения** выберите **Допустимое** или **Активное**.

    ![Панель параметров членства](./media/pim-resource-roles-assign-roles/resources-membership-settings-type.png)

    Привилегированное управление идентификацией для ресурсов Azure предоставляет два отдельных типа назначений:

    - **Соответствующие** назначения требуют, чтобы участник роли выполнял действие для использования роли. Поддерживаются такие действия, как прохождение многофакторной проверки подлинности (MFA), предоставление коммерческого обоснования или запрос утверждения от назначенных утверждающих.

    - **Активные** назначения не требуют от участника выполнения каких-либо действий для использования роли. Члены, назначенные в качестве активных, постоянно имеют все полномочия, присвоенные роли.

1. Если назначение должно быть постоянным (постоянно подходящим или постоянно назначенным), выберите **постоянно** флажок.

    В зависимости от параметров роли флажок может не отображаться или быть неизменяемым.

1. Чтобы указать конкретную продолжительность назначения, снимите флажок и измените поля даты и времени начала и окончания.

    ![Параметры членства — дата и время](./media/pim-resource-roles-assign-roles/resources-membership-settings-date.png)

1. После завершения, выберите **Готово**.

    ![Новое назначение — добавление](./media/pim-resource-roles-assign-roles/resources-new-assignment-add.png)

1. Чтобы создать новое назначение ролей, выберите **Добавить**. Отображается уведомление о состоянии.

    ![Новое назначение — уведомление](./media/pim-resource-roles-assign-roles/resources-new-assignment-notification.png)

## <a name="update-or-remove-an-existing-role-assignment"></a>Обновление или удаление существующего назначения роли

Выполните следующие действия, чтобы обновить или удалить существующее назначение роли.

1. Открытое **управление привилегированным удостоверением личности Azure AD.**

1. Щелкните **Ресурсы Azure**.

1. Выберите ресурс, которым вы хотите управлять, например группу по подписке или управлению.

1. В «Управлении» выберите **роли,** чтобы просмотреть список ролей для ресурсов Azure.

    ![Роли ресурсов Azure — выбор роли](./media/pim-resource-roles-assign-roles/resources-update-select-role.png)

1. Щелкните роль, которую нужно обновить или удалить.

1. Найдите назначение роли на вкладках **Доступные роли** или **Активные роли**.

    ![Обновление или удаление назначения роли](./media/pim-resource-roles-assign-roles/resources-update-remove.png)

1. Нажмите кнопку **Обновить** или **Удалить**, чтобы обновить или удалить назначение роли.

    Для получения информации о продлении назначения ролей [см.](pim-resource-roles-renew-extend.md)

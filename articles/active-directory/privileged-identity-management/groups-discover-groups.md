# <a name="discover-privileged-access-groups-preview-to-manage-in-privileged-identity-management"></a>Обнаружение привилегированных групп доступа (Предварительная версия) для управления в управление привилегированными пользователями

В Azure Active Directory (Azure AD) можно назначить встроенные роли Azure AD для облачных групп, чтобы упростить управление назначениями ролей. Теперь вы можете использовать управление привилегированными пользователями (PIM), чтобы назначить права на членство или владение этими группами, чтобы защитить роли Azure AD и безопасный доступ. Прежде чем можно будет управлять назначаемой группой ролей Azure AD в качестве группы привилегированного доступа в управление привилегированными пользователями, необходимо обнаружить ее и перенести в систему управления PIM.

## <a name="discover-resources"></a>Обнаружение ресурсов

1. [Войдите в Azure AD](https://aad.portal.azure.com) с правами администратора привилегированных ролей.
1. Создание назначаемой роли группы в Azure AD. Для обнаружения и управления с помощью управление привилегированными пользователями необходимо быть владельцем группы.
1. Откройте **Управление привилегированными пользователями**.
1. Выберите **привилегированный доступ (Предварительная версия)**.

    ![Команда "обнаружить группы" при первом запуске](./media/groups-discover-groups/groups-discover-groups.png)

1. Выберите **обнаружение групп**.
1. Поиск по имени группы.
1. Выберите свою группу и щелкните **Управление группами** , чтобы перенести ее в раздел Управление PIM.

    ![Обнаружение групп без ресурсов, перечисленных при первом запуске](./media/groups-discover-groups/groups-bring-under-management.png)

    > [!NOTE]
    > После того как группа привилегированного доступа будет управляема, ее нельзя будет использовать для управления. Это предотвращает удаление параметров управление привилегированными пользователями другим администратором ресурсов.

1. Если появится сообщение с подтверждением адаптации выбранного ресурса к управлению, выберите **Да**.

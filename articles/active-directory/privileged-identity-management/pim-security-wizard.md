# <a name="azure-ad-roles-security-wizard-in-privileged-identity-management"></a>Мастер безопасности ролей Azure AD в управление привилегированными пользователями

Если вы первый пользователь, использующий управление привилегированными пользователями (PIM) в вашей организации Azure Active Directory (Azure AD), вы получаете мастер для начала работы. Мастер помогает понять риски безопасности привилегированных удостоверений и использование управление привилегированными пользователями для снижения этих рисков. Нет необходимости вносить изменения в существующие назначения ролей в мастере, это можно сделать позже.

## <a name="wizard-overview"></a>Общие сведения о мастере

Прежде чем Организация начнет использовать управление привилегированными пользователями, все назначения ролей будут постоянными: пользователи всегда находятся в этих ролях, даже если им не нужны права доступа. На первом шаге мастера выводится список ролей с высоким уровнем привилегий и текущее количество пользователей с этими ролями. Можно просмотреть сведения об определенной роли для получения дополнительных сведений о пользователях, если один или несколько из них вам неизвестны.

Второй шаг мастера предоставляет возможность изменить назначения ролей администраторов.  

> [!WARNING]
> Важно иметь по крайней мере один глобальный администратор и более одного администратора привилегированных ролей с учетной записью организации (а не учетная запись Майкрософт). Если имеется только один администратор привилегированных ролей, Организация не может управлять управление привилегированными пользователями, если эта учетная запись удалена.
> Кроме того, если у пользователя есть учетная запись Майкрософт (иными словами, учетная запись, используемая для входа в службы Майкрософт, такие как Skype и Outlook.com), следует учитывать постоянное назначение ролей. Если для активации этой роли планируется требовать многофакторную проверку подлинности, этот пользователь будет заблокирован.

## <a name="run-the-wizard"></a>Запуск мастера

1. Войдите на [портале Azure](https://portal.azure.com/).

1. Откройте страницу **Azure AD Privileged Identity Management**.

1. Выберите **роли Azure AD** , а затем выберите **Мастер**.

    ![Роли Azure AD. страница мастера, на которой показаны 3 шага для запуска мастера](./media/pim-security-wizard/wizard-start.png)

1. Выберите **1 Обнаружение привилегированных ролей**.

1. Просмотрите список привилегированных ролей, чтобы узнать, какие пользователи имеют постоянные, а какие — временные роли.

    ![Обнаружение привилегированных ролей — область ролей с постоянными и допустимыми членами](./media/pim-security-wizard/discover-privileged-roles-users.png)

1. Нажмите кнопку **Далее** , чтобы выбрать пользователей или группы, которые нужно сделать доступными.

    ![Преобразовать элементы в подходящую страницу с параметрами для выбора элементов, которые нужно сделать доступными для ролей](./media/pim-security-wizard/convert-members-eligible.png)

1. Выбрав пользователей или группы, нажмите кнопку **Далее**.

    ![Страница "Просмотр изменений" с элементами с постоянными назначениями ролей, которые будут преобразованы](./media/pim-security-wizard/review-changes.png)

1. Нажмите кнопку **ОК** , чтобы преобразовать постоянные назначения в подходящие.

    После успешного преобразования вы получите уведомление.

    ![Уведомление, показывающее состояние преобразования](./media/pim-security-wizard/notification-completion.png)

Если вам нужно преобразовать другие назначения привилегированных ролей во временные, запустите мастер еще раз. Если вместо мастера требуется использовать интерфейс управление привилегированными пользователями, см. раздел [назначение ролей Azure AD в Управление привилегированными пользователями](pim-how-to-add-role-to-user.md).

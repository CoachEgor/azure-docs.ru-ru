# <a name="azure-ad-roles-security-wizard-in-privileged-identity-management"></a>Функции AD Azure AD в привилегированном управлении идентификацией

Если вы первый человек, который использует Privileged Identity Management (PIM) в организации Azure Active Directory (Azure AD), вам будет представлен мастер, который начнет работу. Мастер помогает понять риски безопасности привилегированных идентификационных данных и как использовать Privileged Identity Management для уменьшения этих рисков. Нет необходимости вносить изменения в существующие назначения ролей в мастере, это можно сделать позже.

> [!Important]
> Мастер безопасности временно недоступен. Благодарим за терпение.

## <a name="wizard-overview"></a>Общие сведения о мастере

До того, как организация начинает использовать Privileged Identity Management, все назначения ролей являются постоянными: пользователи всегда находятся в этих ролях, даже если они в настоящее время не нуждаются в своих привилегиях. На первом шаге мастера выводится список ролей с высоким уровнем привилегий и текущее количество пользователей с этими ролями. Можно просмотреть сведения об определенной роли для получения дополнительных сведений о пользователях, если один или несколько из них вам неизвестны.

Второй шаг мастера предоставляет возможность изменить назначения ролей администраторов.  

> [!WARNING]
> Важно, чтобы у вас был хотя бы один глобальный администратор и несколько привилегированных ролевых администраторов с рабочей или школьной учетной записью (не учетная запись Майкрософт). Если есть только один администратор привилегированных ролей, организация не может управлять привилегированным управлением идентификацией, если эта учетная запись удалена.
> Кроме того, сохраняйте постоянные назначения ролей, если у пользователя есть учетная запись Майкрософт (другими словами, учетная запись, которую он использует для вхвнутри в службы Майкрософт, такие как Skype и Outlook.com). Если для активации этой роли требуется многофакторная аутентификация, этот пользователь будет заблокирован.

## <a name="run-the-wizard"></a>Запустите мастер.

1. Войдите на [портал Azure](https://portal.azure.com/).

1. Открытое **управление привилегированным удостоверением личности Azure AD.**

1. Выберите **роли Azure AD,** а затем выберите **Мастера.**

    ![Роли Azure AD - Страница мастера, показывающая 3 шага для запуска мастера](./media/pim-security-wizard/wizard-start.png)

1. Выберите **1 Открывайте привилегированные роли.**

1. Просмотрите список привилегированных ролей, чтобы узнать, какие пользователи имеют постоянные, а какие — временные роли.

    ![Откройте для себя привилегированные роли - Панель ролей, показывающая постоянных и подходящих членов](./media/pim-security-wizard/discover-privileged-roles-users.png)

1. Выберите **далее,** чтобы выбрать пользователей или группы, которые вы хотите сделать право.

    ![Преобразование участников на подходяую страницу с вариантами выбора участников, которые вы хотите получить право на роли](./media/pim-security-wizard/convert-members-eligible.png)

1. После того как вы выбрали пользователей или группы, выберите **Следующий**.

    ![Просмотр изменений страницы, показывающие членов с постоянными назначениями ролей, которые будут преобразованы](./media/pim-security-wizard/review-changes.png)

1. Выберите **OK** для преобразования постоянных назначений в подходящие.

    После успешного преобразования вы получите уведомление.

    ![Уведомление о состоянии конверсии](./media/pim-security-wizard/notification-completion.png)

Если вам нужно преобразовать другие назначения привилегированных ролей во временные, запустите мастер еще раз. Если вместо мастера используется интерфейс Privileged Identity Management, обратитесь к [ролим Azure AD в Privileged Identity Management.](pim-how-to-add-role-to-user.md)

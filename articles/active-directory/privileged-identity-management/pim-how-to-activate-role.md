# <a name="activate-my-azure-ad-roles-in-pim"></a>Активация ролей Azure AD в PIM

Компонент Azure Active Directory (Azure AD) Privileged Identity Management (PIM) упрощает управление привилегированным доступом пользователей к ресурсам в Azure AD и других веб-службах Майкрософт, например Office 365 или Microsoft Intune.  

Если вам назначена роль временного администратора, то это значит, что вы можете активировать эту роль, когда вам потребуется выполнить привилегированные задачи. Например, если вы управляете компонентами Office 365 нерегулярно, корпоративные администраторы привилегированных ролей могут не назначить вам роль постоянного глобального администратора, так как эта роль влияет и на другие службы. Вместо этого они могут назначить вам более подходящие для работы с Azure AD роли, например роль администратора Exchange Online. При необходимости вы можете запросить активацию роли, чтобы получить административный доступ на определенный период времени.

Эта статья предназначена для администраторов, которым необходимо активировать свою роль Azure AD в Привилегированном управлении идентификацией.

## <a name="determine-your-version-of-pim"></a>Определите свою версию PIM

Начиная с ноября 2019 года, часть ролей Azure AD в Привилегированном управлении идентификацией обновляется в новую версию, которая соответствует опыту для ролей ресурсов Azure. Это создает дополнительные функции, а также [изменения в существующем API.](azure-ad-roles-features.md#api-changes) В то время как новая версия в настоящее время развернута, какие процедуры, которые вы следуете в этой статье зависит от версии Привилегированного управления идентификацией у вас в настоящее время. Выполните действия в этом разделе, чтобы определить, какая версия привилегированного управления идентификацией у вас есть. После того, как вы знаете свою версию Privileged Identity Management, вы можете выбрать процедуры в этой статье, которые соответствуют этой версии.

1. Вопием на [порталaz](https://portal.azure.com/) с ролью [администратора привилегированных ролей.](../users-groups-roles/directory-assign-admin-roles.md#privileged-role-administrator)
1. Открытое **управление привилегированным удостоверением личности Azure AD.** Если у вас есть баннер в верхней части страницы обзора, следуйте инструкциям в **новой версии** вкладке этой статьи. В противном случае следуйте инструкциям во вкладке **Предыдущая версия.**

    [![](media/pim-how-to-add-role-to-user/pim-new-version.png "Select Azure AD > Privileged Identity Management")](media/pim-how-to-add-role-to-user/pim-new-version.png#lightbox)

## <a name="activate-a-role"></a>Активация роли

При необходимости выполнения роли Azure AD можно запросить активацию, используя навигационную опцию **«Мои роли»** в Privileged Identity Management.

1. Войдите на [портал Azure](https://portal.azure.com/).

1. Открытое **управление привилегированным удостоверением личности Azure AD.** Для получения информации о том, как добавить плитку Privileged Identity Management в панель мониторинга, [см.](pim-getting-started.md)

1. Выберите **мои роли,** а затем выберите **роли Azure AD,** чтобы просмотреть список подходящих ролей Azure AD.

    ![Моя страница ролей, отображая роли, которые вы можете активировать](./media/pim-how-to-activate-role/my-roles.png)

1. В списке **ролей Azure AD** найдите роль, которую вы хотите активировать.

    ![Роли AD Azure - Список подходящих ролей](./media/pim-how-to-activate-role/activate-link.png)

1. Выберите **активировать,** чтобы открыть панель активации.

    ![Роли Azure AD - страница активации содержит продолжительность и область действия](./media/pim-how-to-activate-role/activate-page.png)

1. Если эта роль требует многофакторной проверки подлинности, щелкните **Чтобы продолжить, подтвердите свою личность**. Проверка подлинности выполняется один раз за сеанс.

    ![Проверьте мою личность с МИД перед активацией роли](./media/pim-resource-roles-activate-your-roles/resources-my-roles-mfa.png)

1. Выберите **Проверить мою личность** и следовать инструкциям, чтобы обеспечить дополнительную проверку безопасности.

    ![Экран для проверки безопасности, такой как PIN-код](./media/pim-resource-roles-activate-your-roles/resources-mfa-enter-code.png)

1. Если требуется указать уменьшенную область, выберите **Область,** чтобы открыть панель фильтра. На панели фильтров можно указать ресурсы Azure AD, к которым необходим доступ. Лучше всего запрашивать доступ только к необходимым ресурсам.

1. При необходимости укажите время начала настраиваемой активации. Роль Azure AD будет активирована после выбранного времени.

1. В поле **Причина** введите основание для запроса активации.

1. Выберите **Активировать**.

    Если роль не требует утверждения, она будет активирована и добавлена в список активных ролей. Если вы хотите использовать роль, выполните следующие действия в следующем разделе.

    ![Завершено активация панели с областью, время запуска, продолжительность и причина](./media/pim-how-to-activate-role/azure-ad-activation-status.png)

    Если [роль требует утверждения](pim-resource-roles-approval-workflow.md) для активации, в правом верхнем углу браузера появится уведомление о том, что запрос ожидает утверждения.

    ![Запрос активации находится на рассмотрении уведомления об утверждении](./media/pim-resource-roles-activate-your-roles/resources-my-roles-activate-notification.png)

## <a name="use-a-role-immediately-after-activation"></a>Использование роли сразу после активации

В случае задержки после активации выполните следующие действия после активации, чтобы немедленно использовать роли Azure AD.

1. Откройте страницу Azure AD Privileged Identity Management.

1. Выберите **мои роли,** чтобы просмотреть список подходящих ролей Azure AD и функций ресурсов Azure.

1. Выберите **роли AD Azure**.

1. Выберите вкладку **Активные роли.**

1. Как только роль активна, вывемите из портала и вопием обратно.

    Роль теперь должна быть доступна для использования.

## <a name="view-the-status-of-your-requests"></a>Просмотр состояния запросов

Вы можете просмотреть состояние запросов, ожидающих активации.

1. Откройте страницу Azure AD Privileged Identity Management.

1. Выберите **мои запросы,** чтобы просмотреть список ваших ролей Azure AD и запросов на роль ресурса Azure.

    ![Мои запросы - Страница Azure AD, показывающая ваши незавершенные запросы](./media/pim-how-to-activate-role/my-requests-page.png)

1. Прокрутите вправо, чтобы увидеть столбец **Состояние запроса**.

## <a name="cancel-a-pending-request"></a>Отмена ожидающего запроса

Если вы не активируете роль, требующую утверждения, вы можете в любой момент отменить запрос, ожидающий утверждения.

1. Откройте страницу Azure AD Privileged Identity Management.

1. Выберите **Мои запросы**.

1. Для роли, которую вы хотите отменить, выберите ссылку **«Отмена».**

    При выборе Отмена запрос будет отменен. Чтобы активировать роль снова, необходимо будет отправить новый запрос на активацию.

   ![Мой список запросов с выделено действием Cancel](./media/pim-resource-roles-activate-your-roles/resources-my-requests-cancel.png)

## <a name="troubleshoot"></a>Устранение неполадок

### <a name="permissions-are-not-granted-after-activating-a-role"></a>Разрешения не предоставляются после активации роли

При активации роли в Привилегированном управлении идентификацией активация может не мгновенно распространяться на все порталы, требующие привилегирования. Иногда, даже если изменение распространилось, из-за веб-кэширования на портале оно может начать действовать не сразу. Если активация задерживается, вот что вы должны сделать.

1. Закройте портал Azure, а затем войдите на него снова.

    Когда вы активируете роль Azure AD, вы увидите этапы активации. По завершении всех этапов появится ссылка **Выйти**. Вы можете использовать эту ссылку, чтобы выйти. Это позволит решить большинство случаев задержки активации.

1. В Privileged Identity Management убедитесь, что вы указаны в качестве участника роли.

## <a name="activate-a-role"></a>Активация роли

При необходимости выполнения роли Azure AD можно запросить активацию, используя навигационную опцию **«Мои роли»** в Privileged Identity Management.

1. Войдите на [портал Azure](https://portal.azure.com/).

1. Открытое **управление привилегированным удостоверением личности Azure AD.** Для получения информации о том, как добавить плитку Privileged Identity Management в панель мониторинга, [см.](pim-getting-started.md)

1. Выберите **Роли Azure AD**.

1. Нажмите **на мои роли,** чтобы просмотреть список подходящих ролей Azure AD.

    ![Роли Azure AD - Мои роли, показывающие подходящий или активный список ролей](./media/pim-how-to-activate-role/directory-roles-my-roles.png)

1. Найдите роль, которую необходимо активировать.

    ![Роли AD Azure - Мой список подходящих ролей, показывающий ссылку Activate](./media/pim-how-to-activate-role/directory-roles-my-roles-activate.png)

1. Щелкните **Активировать**, чтобы открыть область сведений об активации роли.

1. Если ваша роль требует многофакторной проверки подлинности (MFA), щелкните **Чтобы продолжить, подтвердите свою личность**. Проверка подлинности выполняется один раз за сеанс.

    ![Проверьте мое удостоверение с МИД перед активацией роли](./media/pim-how-to-activate-role/directory-roles-my-roles-mfa.png)

1. Щелкните **Подтверждение вашей личности** и следуйте инструкциям для настройки дополнительной проверки безопасности.

    ![Дополнительная страница проверки безопасности с вопросом, как связаться с вами](./media/pim-how-to-activate-role/additional-security-verification.png)

1. Щелкните **Активировать**, чтобы открыть область активации.

    ![Панель активации для указания времени начала, продолжительности, билета и причины](./media/pim-how-to-activate-role/directory-roles-activate.png)

1. При необходимости укажите время начала настраиваемой активации.

1. Укажите длительность активации.

1. В поле **Причина активации** введите основание для запроса активации. Для некоторых ролей требуется предоставить номер запроса на устранение неисправности.

    ![Завершено активация панели с пользовательским время начала, продолжительность, билет, и причина](./media/pim-how-to-activate-role/directory-roles-activation-pane.png)

1. Щелкните **Активировать**.

    Если роль не требует утверждения, появляется панель **состояния активации,** отображая состояние активации.

    ![Страница состояния активации, показывающая три этапа активации](./media/pim-how-to-activate-role/activation-status.png)

    Как только все этапы завершены, щелкните ссылку **Знака,** чтобы выйти из портала Azure. Когда вы восстанете на портал, теперь вы можете использовать эту роль.

    Если для активации [роли требуется утверждение,](./azure-ad-pim-approval-workflow.md) в правом верхнем углу браузера появится уведомление Azure, информирующее вас о том, что запрос находится на стадии утверждения.

## <a name="view-the-status-of-your-requests"></a>Просмотр состояния запросов

1. Выберите **Роли Azure AD**.

1. Щелкните **Мои запросы**, чтобы просмотреть список ваших запросов.

## <a name="deactivate-a-role"></a>Деактивация роли

Активированная роль автоматически деактивируется через определенный период времени (соответствующую длительность).

Если вы завершили задачи администратора раньше, роль также можно отключить вручную в Azure AD Privileged Identity Management.

1. Откройте страницу Azure AD Privileged Identity Management.

1. Выберите **Роли Azure AD**.

1. Щелкните **Мои роли**.

1. Щелкните **Активные роли**, чтобы просмотреть список активных ролей.

1. Найдите роль, которая вам больше не нужна, а затем щелкните **Деактивировать**.

## <a name="cancel-a-pending-request"></a>Отмена ожидающего запроса

Если вы не активируете роль, требующую утверждения, вы можете в любой момент отменить запрос, ожидающий утверждения.

1. Откройте страницу Azure AD Privileged Identity Management.

1. Выберите **Роли Azure AD**.

1. Щелкните **Мои запросы**.

1. Нажмите кнопку **Отмена** для роли, которую хотите отменить.

    При нажатии Кнопка отмены запроса будет отменена. Чтобы активировать роль снова, необходимо будет отправить новый запрос на активацию.

   ![Список запросов с выделенной кнопкой «Отмена»](./media/pim-how-to-activate-role/directory-role-cancel.png)

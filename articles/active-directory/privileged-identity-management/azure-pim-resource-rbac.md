# <a name="view-activity-and-audit-history-for-azure-resource-roles-in-privileged-identity-management"></a>Просмотр истории активности и аудита для ролей ресурсов Azure в Привилегированном управлении identity

С помощью Privileged Identity Management (PIM) Azure Active Directory (Azure AD) вы можете просмотреть журнал действий, активации и аудита для ролей ресурсов Azure в своей организации. Включая подписки, группы ресурсов и даже виртуальные машины. Любой ресурс в портале Azure, использующий функцию управления доступом на основе ролей Azure, может воспользоваться возможностями управления безопасностью и жизненным циклом в привилегированном управлении идентификацией.

> [!NOTE]
> Если организация передала функции управления на внешний подряд поставщику услуг, который использует [управление делегированными ресурсами Azure,](../../lighthouse/concepts/azure-delegated-resource-management.md)назначения ролей, авторизованные этим поставщиком услуг, не будут отображаться здесь.

## <a name="view-activity-and-activations"></a>Просмотр действий и активаций

Чтобы выяснить, какие действия выполняет конкретный пользователь с разными ресурсами, вы можете просмотреть активность ресурсов Azure, связанную с данным периодом активации.

1. Открытое **управление привилегированным удостоверением личности Azure AD.**

1. Щелкните **Ресурсы Azure**.

1. Выберите ресурс, для просмотра деятельности и активации.

1. Выберите **роли** или **участников.**

1. Выберите пользователя.

    В ресурсах Azure по дате вы видите сводку действий пользователя. В нем также представлены недавние активации роли за тот же период времени.

    ![Сведения о пользователях с резюме активности ресурсов и активациями ролей](media/azure-pim-resource-rbac/rbac-user-details.png)

1. Выберите конкретную активацию ролей, чтобы увидеть детали и соответствующие действия ресурсов Azure, которые произошли во время активации этого пользователя.

    [![Выбранная активация ролей и детали активности](media/azure-pim-resource-rbac/export-membership.png "Выбранная активация ролей и детали активности")](media/azure-pim-resource-rbac/export-membership.png)

## <a name="export-role-assignments-with-children"></a>Экспорт назначений ролей с дочерними элементами

Для соответствия требованиям вам может потребоваться указать полный список назначений ролей для аудиторов. Привилегированое управление идентификацией позволяет запросить назначения ролей на определенном ресурсе, который включает в себя назначения ролей для всех ресурсов ребенка. Ранее у администраторов возникали трудности при получении полного списка назначений ролей для подписки, и им нужно было экспортировать назначения ролей для каждого конкретного ресурса. Используя Privileged Identity Management, можно запросить все активные и подходящие ролевые назначения в подписке, включая ролевые назначения для всех групп ресурсов и ресурсов.

1. Открытое **управление привилегированным удостоверением личности Azure AD.**

1. Щелкните **Ресурсы Azure**.

1. Выберите ресурс, для которого необходимо экспортировать назначения ролей, например подписку.

1. Выберите **Элементы**.

1. Выберите **Экспорт,** чтобы открыть панель членства экспорта.

    [![Экспорт членства панели для экспорта всех членов](media/azure-pim-resource-rbac/export-membership.png "Страница членства экспорта для экспорта всех членов")](media/azure-pim-resource-rbac/export-membership.png)

1. Выберите **Экспорт всех участников,** чтобы экспортировать все назначения ролей в файле CSV.

    ![Экспортированные ролевые назначения в файле CSV в качестве дисплея в Excel](media/azure-pim-resource-rbac/export-csv.png)

## <a name="view-resource-audit-history"></a>Просмотр журнала аудита ресурсов

Аудит ресурсов дает вам представление о всей активности роли для этого ресурса.

1. Открытое **управление привилегированным удостоверением личности Azure AD.**

1. Щелкните **Ресурсы Azure**.

1. Выберите ресурс, для просмотра истории аудита.

1. Выберите **аудит ресурсов**.

1. Отфильтруйте журнал по предопределенной дате или настраиваемому диапазону.

    [![Список аудита ресурсов с фильтрами](media/azure-pim-resource-rbac/rbac-resource-audit.png "Список аудита ресурсов с фильтрами")](media/azure-pim-resource-rbac/rbac-resource-audit.png)

1. В раскрывающемся списке **Тип аудита** выберите **Activate (Assigned + Activated)** (Активировать (назначено + активировано)).

    [![Список аудита ресурсов, отфильтрованный](media/azure-pim-resource-rbac/rbac-audit-activity.png "Список аудита ресурсов, отфильтрованный Activate")](media/azure-pim-resource-rbac/rbac-audit-activity.png) ![по списку аудита типа «Активировать», который фильтруется по типу аудита Activate](media/azure-pim-resource-rbac/rbac-audit-activity.png)

1. В разделе **Действие** щелкните **(действие)**, чтобы пользователь мог просмотреть сведения о своих действиях в ресурсах Azure.

    ![Сведения о действиях пользователя для конкретного действия](media/azure-pim-resource-rbac/rbac-audit-activity-details.png)

## <a name="view-my-audit"></a>Просмотр раздела "Мой аудит"

В разделе "Мой аудит" можно просматривать действия, выполняемые в личной роли.

1. Открытое **управление привилегированным удостоверением личности Azure AD.**

1. Щелкните **Ресурсы Azure**.

1. Выберите ресурс, для просмотра истории аудита.

1. Выберите **мой аудит**.

1. Отфильтруйте журнал по предопределенной дате или настраиваемому диапазону.

    [![Список аудита для текущего пользователя](media/azure-pim-resource-rbac/my-audit-time.png "Список аудита для текущего пользователя")](media/azure-pim-resource-rbac/my-audit-time.png)

> [!NOTE]
> Доступ к истории аудита требует роли Глобального Администратора или Привилегированном администратора ролей.

## <a name="get-reason-approver-and-ticket-number-for-approval-events"></a>Получить разум, утверждение и номер билета для проведения мероприятий по утверждению

1. Вопийте на [портале Azure](https://aad.portal.azure.com) с разрешениями администратора привилегированных ролей и открывайте AD Azure.
1. Выберите **журналы аудита**.
1. Используйте фильтр **Службы** для отображения только событий аудита для службы privileged identity Management. На странице **журналов аудита** можно:

    - См, что произошло для события аудита, в столбце **причины состояния.**
    - См. утверждение в колонке **«Инициированный (актером)** для события «добавить участника в утвержденный запрос роли».

    [![Фильтр журнала аудита для службы PIM](media/azure-pim-resource-rbac/filter-audit-logs.png "Фильтр журнала аудита для службы PIM")](media/azure-pim-resource-rbac/filter-audit-logs.png)

1. Выберите событие журнала аудита, чтобы увидеть номер билета на вкладке **Activity** панели **«Детали».**
  
    [![Проверьте номер билета на аудиторское мероприятие](media/azure-pim-resource-rbac/audit-event-ticket-number.png "Проверьте номер билета на аудиторское мероприятие")](media/azure-pim-resource-rbac/audit-event-ticket-number.png)

1. Вы можете просмотреть запрашивающий (лицо, активизировавшее роль) на вкладке **«Цели»** панели **«Детали»** для события аудита. Для ролей ресурсов Azure существует три целевых типа:

    - Роль **(Тип** и роль)
    - Запрашиваемый запрос (**Тип** и другие)
    - Утверждение **(Тип** и пользователь)

    [![Проверка целевого типа](media/azure-pim-resource-rbac/audit-event-target-type.png "Проверка целевого типа")](media/azure-pim-resource-rbac/audit-event-target-type.png)

Как правило, событие журнала непосредственно над событием утверждения является событием для "Добавить участника к завершенной роли", где **Инициированный (актер)** является запрашивателем. В большинстве случаев вам не нужно будет находить запрашивающего в запросе утверждения с точки зрения аудита.

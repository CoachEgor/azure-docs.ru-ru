# <a name="multi-factor-authentication-and-privileged-identity-management"></a>Многофакторная аутентификация и привилегированном управлении идентификацией

Рекомендуется настроить обязательную многофакторную проверку подлинности (MFA) для всех администраторов. Это позволяет снизить риск атак, связанных с компрометацией паролей.

Вы можете потребовать, чтобы пользователи выполнили многофакторную задачу проверки подлинности при вхвошне. Вы также можете потребовать, чтобы пользователи выполнили многофакторную задачу проверки подлинности при активации роли в привилегированном управлении идентификацией Azure Active Directory (Azure AD) Privileged Identity Management (PIM). Таким образом, если пользователь не выполнил многофакторную задачу проверки подлинности при входе в систему, ему будет предложено сделать это привилегированным управлением идентификацией.

> [!IMPORTANT]
> Сейчас Azure Multi-Factor Authentication работает только с рабочими или школьными учетными записями, а не с личными учетными записями Майкрософт (обычно это личная учетная запись, которая используется для вхинга в службы Майкрософт, такие как Skype, Xbox или Outlook.com). Из-за этого любой человек, использующий личную учетную запись, не может быть подходящим администратором, поскольку он не может использовать многофакторную аутентификацию для активации своих ролей. В качестве временного решения, если этим пользователям необходимо продолжать управлять рабочими нагрузками с помощью учетных записей Майкрософт, повысьте их статус до постоянных администраторов.

## <a name="how-pim-validates-mfa"></a>Как PIM проверяет MFA

Существует два варианта проверки многофакторной аутентификации при активации роли пользователем.

Самый простой вариант – полагаться на многофакторную аутентификацию Azure для пользователей, которые активируют привилегированное значение. Для этого сначала проверьте, что эти пользователи имеют лицензию, если это необходимо, и зарегистрировались для Azure Multi-Фактор аутентификации. Для получения дополнительной информации о том, как развернуть многофакторную аутентификацию Azure, можно узнать об [облачную azure Multi-Factor Authentication.](../authentication/howto-mfa-getstarted.md) Рекомендуется, но не обязательно настраивать Azure AD для обеспечения многофакторной аутентификации для этих пользователей при вхотворении. Это связано с тем, что многофакторная проверка аутентификации будет осуществляться самим privileged Identity Management.

Кроме того, если пользователи удостоверяем подлинность, вы можете вознести ответственность за многофакторную аутентификацию. Например, если службы федерации AD требуют проверку подлинности на основе смарт-карты для доступа к Azure AD, можете воспользоваться инструкциями по настройке AD FS для отправки утверждений в Azure AD, приведенными в разделе [Защита облачных ресурсов с помощью Многофакторной идентификации Azure и AD FS](../authentication/howto-mfa-adfs.md). Когда пользователь пытается активировать роль, Привилегированном управлении идентификацией будет признано, что многофакторная аутентификация уже была проверена для пользователя, как только он получает соответствующие требования.

# <a name="approve-or-deny-requests-for-azure-ad-roles-in-privileged-identity-management"></a>Утверждение или отказ в предоставлении функций Azure AD в управлении привилегированными удостоверениями

С помощью Azure Active Directory (Azure AD) Privileged Identity Management (PIM) можно настроить роли, требующие утверждения для активации, и выбрать одного или нескольких пользователей или групп в качестве делегированных одобрителей. Делегированные утверждающие лица имеют 24 часа на утверждение запросов. Если запрос не был утвержден в течение 24 часов, пользователю, наделенному соответствующим правом, необходимо повторно отправить запрос. Продолжительность 24-часового периода для утверждения нельзя изменить.

## <a name="determine-your-version-of-pim"></a>Определите свою версию PIM

Начиная с ноября 2019 года, часть ролей Azure AD в Привилегированном управлении идентификацией обновляется в новую версию, которая соответствует опыту для ролей Azure. Это создает дополнительные функции, а также [изменения в существующем API.](azure-ad-roles-features.md#api-changes) В то время как новая версия в настоящее время развернута, какие процедуры, которые вы следуете в этой статье зависит от версии Привилегированного управления идентификацией у вас в настоящее время. Выполните действия в этом разделе, чтобы определить, какая версия привилегированного управления идентификацией у вас есть. После того, как вы знаете свою версию Privileged Identity Management, вы можете выбрать процедуры в этой статье, которые соответствуют этой версии.

1. Вопием на [портал Azure](https://portal.azure.com/) с пользователем, который занимается ролью [администратора привилегированных.](../users-groups-roles/directory-assign-admin-roles.md#privileged-role-administrator)
1. Открытое **управление привилегированным удостоверением личности Azure AD.** Если у вас есть баннер в верхней части страницы обзора, следуйте инструкциям в **новой версии** вкладке этой статьи. В противном случае следуйте инструкциям во вкладке **Предыдущая версия.**

    [![](media/pim-how-to-add-role-to-user/pim-new-version.png "Select Azure AD > Privileged Identity Management")](media/pim-how-to-add-role-to-user/pim-new-version.png#lightbox)

Выполните действия в этой статье, чтобы утвердить или отклонить запросы на роли Azure AD.

# <a name="new-version"></a>[Новая версия](#tab/new)

## <a name="view-pending-requests"></a>Просмотр ожидающих запросов

Как делегированный утверждатель, вы получите уведомление по электронной почте, когда запрос роли AD Azure AD ожидает вашего утверждения. Вы можете просмотреть эти незавершенные запросы в Privileged Identity Management.

1. Войдите на [портал Azure](https://portal.azure.com/).

1. Открытое **управление привилегированным удостоверением личности Azure AD.**

1. Выберите **Запросы утверждения**.

    ![Утверждение запросов - страница, показывающая запрос на просмотр ролей Azure AD](./media/azure-ad-pim-approval-workflow/resources-approve-pane.png)

    В разделе **Запросы на активацию ролей** вы увидите список запросов, ожидающих вашего утверждения.

## <a name="approve-requests"></a>Утверждение запросов

1. Найдите и выберите запрос, который вы хотите одобрить. Отображается страница «Утверждение или отказ».

    ![Утвердить запросы - утвердить или отказать в панели с подробной информацией и Обоснование поле](./media/azure-ad-pim-approval-workflow/resources-approve-pane.png)

1. В поле **Justification** введите бизнес-обоснование.

1. Выберите **Утвердить**. Вы получите уведомление Azure о вашем одобрении.

    ![Утверждено уведомление с указанием запроса](./media/pim-resource-roles-approval-workflow/resources-approve-pane.png))

## <a name="deny-requests"></a>Отклонение запросов

1. Найдите и выберите запрос, который вы хотите отклонить. Отображается страница «Утверждение или отказ».

    ![Утвердить запросы - утвердить или отказать в панели с подробной информацией и Обоснование поле](./media/pim-resource-roles-approval-workflow/resources-approve-pane.png)

1. В поле **Justification** введите бизнес-обоснование.

1. Выберите **Отрицание**. Появится уведомление о вашем отказе.

## <a name="workflow-notifications"></a>Уведомления в рамках рабочего процесса

Ниже приведена информация об уведомлениях рабочего процесса.

- Утвердители уведомляются по электронной почте, когда запрос на роль находится на рассмотрении. В этих уведомлениях содержится прямая ссылка на запрос, который утверждающий может подтвердить или отклонить.
- Запросы решаются первым утверждением, который утверждает или отрицает.
- Когда одобритель отвечает на запрос, все одобрители уведомляются об действии.
- Глобальные админы и привилегированные роли админантов уведомляются, когда одобренный пользователь становится активным в их роли.

>[!NOTE]
>Глобальный админ или привилегированный реч-ан-он, который считает, что одобренный пользователь не должен быть активным, может удалить назначение активной роли в Привилегированном управлении идентификацией. Хотя администраторы не уведомляются о ожидающих запросах, если они не являются одобренными, они могут просматривать и отменять любые ожидающие запросы для всех пользователей, просматривая ожидающие запросы в Privileged Identity Management.

# <a name="previous-version"></a>[Предыдущая версия](#tab/previous)

## <a name="view-pending-requests"></a>Просмотр ожидающих запросов

Как делегированный утверждатель, вы получите уведомление по электронной почте, когда запрос роли AD Azure AD ожидает вашего утверждения. Вы можете просмотреть эти незавершенные запросы в Privileged Identity Management.

1. Войдите на [портал Azure](https://portal.azure.com/).

1. Открытое **управление привилегированным удостоверением личности Azure AD.**

1. Выберите **Роли Azure AD**.

1. Выберите **Утверждение запросов**.

    ![Роли Azure AD - Утверждение запросов](./media/azure-ad-pim-approval-workflow/approve-requests.png)

    Отобразится список запросов, ожидающих утверждения.

## <a name="approve-requests"></a>Утверждение запросов

1. Выберите запросы, которые требуется утвердить, а затем выберите **Утвердить**, чтобы открыть область "Утверждение выбранных запросов".

    ![Утверждение списка запросов с выделенной опцией «Утвердить»](./media/azure-ad-pim-approval-workflow/pim-approve-requests-list.png)

1. В области **Причина утверждения** введите причину.

    ![Утверждение выбранного панели запросов с помощью утвердиваемого основания](./media/azure-ad-pim-approval-workflow/pim-approve-selected-requests.png)

1. Нажмите **Утвердить**.

    Символ состояния будет обновлен с вашим разрешением.

    ![Утвердить выбранное панель запросов после нажатия кнопки «Утверждение»](./media/azure-ad-pim-approval-workflow/pim-approve-status.png)

## <a name="deny-requests"></a>Отклонение запросов

1. Выберите запросы, которые требуется отклонить, а затем выберите **Запретить**, чтобы открыть область "Отклонение выбранных запросов".

    ![Утверждение списка запросов с выделенной опцией Deny](./media/azure-ad-pim-approval-workflow/pim-deny-requests-list.png)

1. В области **Причина отклонения** введите причину.

    ![Отказать выбранным запросам панели с отказом причине](./media/azure-ad-pim-approval-workflow/pim-deny-selected-requests.png)

1. Выберите **Отрицание**.

    Символ состояния будет обновлен с вашим отказом.

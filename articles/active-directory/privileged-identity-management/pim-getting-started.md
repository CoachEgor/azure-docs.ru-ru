# <a name="start-using-privileged-identity-management"></a>Начало работы с управлением привилегированными пользователями

С помощью управление привилегированными пользователями (PIM) вы можете управлять доступом, контролировать и отслеживать их в вашей организации Azure Active Directory (Azure AD). Эта область включает доступ к ресурсам Azure, Azure AD и другим веб-службы Майкрософт, таким как Office 365 или Microsoft Intune.

В этой статье описывается, как включить и начать работу с управление привилегированными пользователями.

## <a name="prerequisites"></a>Технические условия

Чтобы использовать управление привилегированными пользователями, необходимо иметь одну из следующих лицензий:

- Azure AD Premium P2
- Enterprise Mobility + Security (EMS) E5.

Дополнительные сведения см. [в статье требования к лицензиям для использования Управление привилегированными пользователями](subscription-requirements.md).

## <a name="first-person-to-use-pim"></a>Первый пользователь, использующий PIM

Если вы первый пользователь управление привилегированными пользователями в каталоге, вы автоматически назначаете роли [администратора безопасности](../users-groups-roles/directory-assign-admin-roles.md#security-administrator) и [администратора привилегированных ролей](../users-groups-roles/directory-assign-admin-roles.md#privileged-role-administrator) в каталоге. Только администраторы привилегированных ролей могут управлять назначениями ролей Azure AD. Кроме того, вы можете запустить [мастер безопасности](pim-security-wizard.md), который поможет вам выполнить начальное обнаружение и назначение.

## <a name="enable-pim"></a>Включение PIM

Чтобы начать использовать управление привилегированными пользователями в каталоге, необходимо сначала включить управление привилегированными пользователями.

1. Войдите на [портал Azure](https://portal.azure.com/) как глобальный администратор каталога.

    Для включения управление привилегированными пользователями для каталога необходимо быть глобальным администратором с учетной записью организации (например, @yourdomain.com), а не учетная запись Майкрософт (например, @outlook.com).

1. Выберите **Все службы** и найдите службу **Azure AD Privileged Identity Management**.

    ![Azure AD Privileged Identity Management в разделе "Все службы"](./media/pim-getting-started/pim-all-services-find.png)

1. Щелкните, чтобы открыть краткое руководство по управление привилегированными пользователями.

1. В списке щелкните **Согласие для PIM**.

    ![Согласие на управление привилегированными пользователями для включения управление привилегированными пользователями](./media/pim-getting-started/consent-pim.png)

1. Выберите **Подтверждение вашей личности**, чтобы подтвердить свою личность с помощью Azure MFA. Вам будет предложено выбрать учетную запись.

    ![Выберите окно учетной записи для подтверждения личности](./media/pim-getting-started/pick-account.png)

1. Если для проверки требуются дополнительные сведения, вы получите пошаговые инструкции по их добавлению. Дополнительные сведения см. в статье [Получение справки по двухфакторной проверке подлинности](https://go.microsoft.com/fwlink/p/?LinkId=708614).

    ![Дополнительные сведения о окне требуется, если вашей организации требуются дополнительные сведения](./media/pim-getting-started/more-information-required.png)

    Например, может потребоваться проверка по телефону.

    ![Страница «дополнительная проверка безопасности» с вопросом о том, как связаться с вами](./media/pim-getting-started/additional-security-verification.png)

1. После завершения процесса проверки нажмите кнопку **Согласие**.

1. В появившемся сообщении нажмите кнопку **Да** , чтобы подтвердить согласие на службу Управление привилегированными пользователями.

    ![Согласие на управление привилегированными пользователями сообщение для завершения процесса разрешения](./media/pim-getting-started/consent-pim-message.png)

## <a name="sign-up-pim-for-azure-ad-roles"></a>Регистрация PIM для ролей Azure AD

После включения управление привилегированными пользователями для каталога необходимо зарегистрироваться управление привилегированными пользователями для управления ролями Azure AD.

1. Откройте страницу **Azure AD Privileged Identity Management**.

1. Выберите **Роли Azure AD**.

    ![Регистрация управление привилегированными пользователями для ролей Azure AD](./media/pim-getting-started/sign-up-pim-azure-ad-roles.png)

1. Щелкните **Регистрация**.

1. В появившемся сообщении нажмите кнопку **Да** , чтобы зарегистрироваться управление привилегированными пользователями управления РОЛЯМИ Azure AD.

    ![Регистрация управление привилегированными пользователями для сообщения ролей Azure AD](./media/pim-getting-started/sign-up-pim-message.png)

    После завершения регистрации станут доступны параметры Azure AD. Вам может потребоваться обновить портал.

    Сведения о том, как найти и выбрать ресурсы Azure для защиты с помощью управление привилегированными пользователями, см. [в разделе Обнаружение ресурсов Azure для управления в Управление привилегированными пользователями](pim-resource-roles-discover-resources.md).

## <a name="navigate-to-your-tasks"></a>Переход к задачам

После настройки управление привилегированными пользователями можно приступить к задачам управления удостоверениями.

![Окно навигации в управление привилегированными пользователями отображение задач и управление параметрами](./media/pim-getting-started/pim-quickstart-tasks.png)

| Задача и управление | Описание |
| --- | --- |
| **Мои роли**  | Отображается список назначенных вам действительных и активных ролей. Здесь можно активировать все действительные назначенные роли. |
| **Мои запросы** | Отображаются все запросы, ожидающие активации назначения действительных ролей. |
| **Утверждение запросов** | Отображается список запросов на активацию действительных ролей в вашем каталоге для пользователей, которых вы должны подтвердить. |
| **Проверка доступа** | Приведены активные проверки доступа, которые вы должны выполнить (для себя или для кого-то другого). |
| **Роли Azure AD** | Отображает панель мониторинга и параметры для администраторов привилегированных ролей для управления назначениями ролей Azure AD. Эта панель мониторинга отключена для всех, кто не является администратором привилегированных ролей. Эти пользователи имеют доступ к специальной панели мониторинга My view (Мое представление). На панели мониторинга My view (Мое представление) отображаются только сведения о пользователе, который получает доступ к панели мониторинга, а не о клиенте. |
| **Ресурсы Azure** | Отображаются панель мониторинга и параметры для администраторов привилегированных ролей, чтобы они могли управлять назначениями ролей ресурса Azure. Эта панель мониторинга отключена для всех, кто не является администратором привилегированных ролей. Эти пользователи имеют доступ к специальной панели мониторинга My view (Мое представление). На панели мониторинга My view (Мое представление) отображаются только сведения о пользователе, который получает доступ к панели мониторинга, а не о клиенте. |

## <a name="add-a-pim-tile-to-the-dashboard"></a>Добавление плитки PIM на панель мониторинга

Чтобы упростить открытие управление привилегированными пользователями, необходимо добавить управление привилегированными пользователями плитку на панель мониторинга портал Azure.

1. Войдите на [портале Azure](https://portal.azure.com/).

1. Выберите **Все службы** и найдите службу **Azure AD Privileged Identity Management**.

    ![Azure AD Privileged Identity Management в разделе "Все службы"](./media/pim-getting-started/pim-all-services-find.png)

1. Щелкните, чтобы открыть краткое руководство по управление привилегированными пользователями.

1. Установите флажок **закрепить колонку на панели мониторинга** , чтобы закрепить колонку краткого руководства Управление привилегированными пользователями на панели мониторинга.

    ![Значок канцелярской кнопки для закрепления колонки управление привилегированными пользователями на панели мониторинга](./media/pim-getting-started/pim-quickstart-pin-to-dashboard.png)

    На панели мониторинга Azure вы увидите такие плитки:

    ![Плитка быстрого запуска управление привилегированными пользователями на панели мониторинга](./media/pim-getting-started/pim-quickstart-dashboard-tile.png)

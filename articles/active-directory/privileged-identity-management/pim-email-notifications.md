# <a name="email-notifications-in-pim"></a>Уведомления по электронной почте в PIM

Привилегированное управление идентификацией (PIM) позволяет узнать, когда происходят важные события в организации Azure Active Directory (Azure AD), например, когда назначена или активирована роль. Привилегированный identity Management держит вас в курсе, отправляя вам и другим участникам уведомления по электронной почте. В этих сообщениях электронной почты также могут содержаться ссылки на соответствующие задачи, такие как активация или обновление роли. В этой статье описывается, как выглядят такие сообщения электронной почты, когда они отправляются и кто получает их.

## <a name="sender-email-address-and-subject-line"></a>Адрес электронной почты и строка темы сообщения отправителя

Электронные письма, отправленные из Privileged Identity Management для ролей ресурсов Azure AD и Azure, имеют следующий адрес электронной почты отправителя:

- Адрес электронной почты: **\@azure-noreply microsoft.com**
- Отображаемое имя: Microsoft Azure

Эти адреса электронной почты включают префикс **PIM** в строке темы. Ниже приведен пример:

- PIM: Ален Харон был постоянно назначен резервного читателя роль

## <a name="notifications-for-azure-ad-roles"></a>Уведомления для ролей Azure AD

Привилегированное управление идентификацией отправляет электронные письма, когда происходят следующие события для ролей Azure AD:

- если активация привилегированной роли ожидает утверждения;
- если запрос на активацию привилегированной роли выполнен;
- При включении управления конфиденциальностью идентификационных данных Azure AD

Получатель этих сообщений электронной почты для ролей Azure AD зависит от вашей настройки роли, события и уведомления:

| Пользователь | Активация роли ожидает утверждения | Выполнен запрос на активацию роли | Включен PIM |
| --- | --- | --- | --- |
| Администратор привилегированных ролей</br>(активирован или разрешен) | Да</br>(только если не указаны явные утверждающие) | Да* | Да |
| Администратор безопасности</br>(активирован или разрешен) | нет | Да* | Да |
| глобального администратора;</br>(активирован или разрешен) | нет | Да* | Да |

\* Если для параметра [**Уведомления**](pim-how-to-change-default-settings.md#notifications) задано значение **Включить**.

Ниже приведен пример сообщения электронной почты, отправляемого при активации пользователем роли Azure AD для вымышленной организации Contoso.

![Новая электронная почта Privileged Identity Management для ролей Azure AD](./media/pim-email-notifications/email-directory-new.png)

### <a name="weekly-privileged-identity-management-digest-email-for-azure-ad-roles"></a>Еженедельный привилегированный управление идентификацией переваривает электронную почту для ролей Azure AD

Еженедельное сводной электронной почте Privileged Identity Management для ролей Azure AD отправляется привилегированным администраторам ролей, администраторам безопасности и глобальным администраторам, которые позволили привилегированному управлению идентификацией. В этом еженедельном письме содержится снимок мероприятий privileged Identity Management за неделю, а также привилегированные назначения ролей. Оно доступно только для клиентов в общедоступном облаке. Ниже приведен пример.

![Еженедельный привилегированный управление идентификацией переваривает электронную почту для ролей Azure AD](./media/pim-email-notifications/email-directory-weekly.png)

Электронное сообщение содержит четыре плитки:

| Плитка | Описание |
| --- | --- |
| **Активированные пользователи** | Число активаций допустимых ролей пользователями внутри клиента. |
| **Постоянные пользователи** | Количество пользователей с допустимым назначением, которые стали постоянными. |
| **Назначения ролей в привилегированном управлении идентификацией** | Количество раз, когда пользователям присваивается право на роль в Привилегированном управлении идентификацией. |
| **Назначения ролей за пределами PIM** | Количество раз, когда пользователям присваивается постоянная роль за пределами Privileged Identity Management (внутри Azure AD). |

В разделе **Overview of your top roles** (Обзор лучших ролей) перечислены пять лучших ролей в вашем клиенте по общему числу постоянных и допустимых администраторов для каждой роли. Перейдя по ссылке **Выполнить действие**, вы откроете [мастер PIM](pim-security-wizard.md), где можете массово преобразовать постоянных администраторов в допустимых.

## <a name="pim-emails-for-azure-resource-roles"></a>Сообщения электронной почты PIM для ролей ресурсов Azure

Привилегированное управление идентификацией отправляет электронные письма владельцам и администраторам пользовательского доступа при следующих событиях для ролей ресурсов Azure:

- назначение ролей ожидает утверждения;
- при назначении роли;
- приближение даты истечения срока действия роли;
- когда роль подходит для расширения;
- обновление роли пользователем;
- выполнение запроса на активацию роли.

Привилегированное управление идентификацией отправляет электронные письма конечным пользователям при следующих событиях для ролей ресурсов Azure:

- при назначении роли пользователю;
- если истек срок действия роли пользователя;
- если срок действия роли пользователя продлен;
- выполнение запроса пользователя на активацию роли.

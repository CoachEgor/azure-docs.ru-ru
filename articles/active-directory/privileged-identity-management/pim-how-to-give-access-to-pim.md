# <a name="grant-access-to-other-administrators-to-manage-privileged-identity-management"></a>Предоставить доступ к другим администраторам для управления привилегированным управлением идентификацией

Глобальный администратор, который позволяет Привилегированному управлению идентификацией (PIM) для организации, автоматически получать ролевые назначения и доступ к privileged Identity Management. Никто другой в организации Azure Active Directory (Azure AD) не получает доступа к записи по умолчанию, включая других глобальных администраторов. Другие глобальные администраторы, администраторы безопасности и читатели security имеют доступ только к privileged Identity Management. Чтобы предоставить доступ к привилегированному управлению идентификацией, первый пользователь может назначить других на роль **администратора привилегированных ролей.**

> [!NOTE]
> Управление привилегированным управлением идентификацией требует многофакторной аутентификации Azure. Поскольку учетные записи Майкрософт не могут зарегистрироваться для многофакторной аутентификации Azure, пользователь, вскакивающий в учетную запись Майкрософт, не может получить доступ к privileged Identity Management.

Убедитесь, что в роли администратора привилегированных ролей всегда есть по крайней мере два пользователя, на случай, если один пользователь будет заблокирован или его учетная запись будет удалена.

## <a name="grant-access-to-manage-pim"></a>Предоставление доступа для управления PIM

1. Войдите на [портал Azure](https://portal.azure.com/).

1. В Azure AD откройте **привилегированное управление идентификацией.**

1. Выберите **роли AD Azure**.

1. Выберите **Роли**.

    ![Привилегированные роли управления идентификационными ресурсами Azure AD - Роли](./media/pim-how-to-give-access-to-pim/pim-directory-roles-roles.png)

1. Выберите роль **администратора привилегированных ролей,** чтобы открыть страницу участников.

    ![Администратор привилегированных ролей. Участники](./media/pim-how-to-give-access-to-pim/pim-pra-members.png)

1. Выберите **добавить участника,** чтобы открыть панель управляемых участников Add.

1. Выберите **избранных участников,** чтобы открыть панель Избранных участников.

    ![Администратор привилегированных ролей. Выбор участников](./media/pim-how-to-give-access-to-pim/pim-pra-select-members.png)

1. Выберите участника и нажмите кнопку **Выбрать**.

1. Выберите **OK,** чтобы член имел право на роль **администратора привилегированных ролей.**

    При присвоении новой роли кому-либо из привилегированных управлений идентификацией они автоматически настраиваются как **подходящие** для активации роли.

1. Чтобы сделать участника постоянным, выберите пользователя в списке участников Privileged Role Administrator.

1. Выберите **больше,** а затем **сделать постоянным,** чтобы сделать назначение постоянным.

    ![Администратор привилегированных ролей. Постоянное назначение](./media/pim-how-to-give-access-to-pim/pim-pra-make-permanent.png)

1. Отправить пользователю ссылку [на начало использования Привилегированного управления идентификацией.](pim-getting-started.md)

## <a name="remove-access-to-manage-pim"></a>Удаление доступа для управления PIM

Прежде чем удалить кого-либо из роли администратора привилегированных ролей, всегда проверяйте, остается ли хотя бы два пользователя, которым назначена эта роль.

1. Войдите на [портал Azure](https://portal.azure.com/).

1. Открытое **управление привилегированным удостоверением личности Azure AD.**

1. Выберите **роли AD Azure**.

1. Выберите **Роли**.

1. Выберите роль **администратора привилегированных ролей,** чтобы открыть страницу участников.

1. Выберите флажок рядом с пользователем, который вы хотите удалить, а затем выберите **Удалить участника.**

    ![Администратор привилегированных ролей. Удаление участника](./media/pim-how-to-give-access-to-pim/pim-pra-remove-member.png)

1. Когда вас попросят подтвердить, что вы хотите удалить участника из роли, выберите **«Да».**

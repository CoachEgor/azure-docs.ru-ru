# <a name="complete-an-access-review-of-azure-ad-roles-in-privileged-identity-management"></a>Завершите обзор ролей Azure AD в привилегированном управлении идентификацией

После [запуска проверки доступа](pim-how-to-start-security-review.md) администраторы привилегированных ролей могут проверить привилегированный доступ.  Привилегированное управление идентификацией (PIM) автоматически отправляет электронное письмо пользователям в организации Azure Active Directory (Azure AD), что предлагает им проверить свой доступ. Если пользователь не получил электронное письмо, ему можно отправить [инструкции по выполнению проверки доступа](pim-how-to-perform-security-review.md).

После окончания периода проверки доступа или после завершения всеми пользователями самостоятельной проверки выполните шаги, описанные в этой статье, для управления проверкой и просмотра результатов.

## <a name="manage-access-reviews"></a>Управление проверками доступа

1. Перейдите на [портал Azure](https://portal.azure.com/) и выберите службу **управления привилегированными удостоверениями личности Azure AD** на панели мониторинга.
1. Выберите раздел **Проверки доступа** на панели мониторинга.
1. Выберите необходимую проверку доступа.

На лезвии детального обзора доступа существует несколько вариантов управления этим обзором.

![Кнопки обзора конфиденциального управления идентификацией - скриншот](./media/pim-how-to-complete-review/review-buttons.png)

### <a name="remind"></a>Напомнить

Если проверка доступа настроена таким образом, что пользователи проверяют сами себя, то кнопка **Напомнить** отправляет уведомление.

### <a name="stop"></a>Остановить

Все проверки доступа имеют дату окончания, но можно воспользоваться кнопкой **Остановить** , чтобы завершить их выполнение раньше. Если к этому моменту еще не все пользователи были проверены, то они уже не смогут пройти проверку после ее остановки. Нельзя перезапустить проверку после ее остановки.

### <a name="apply"></a>Применить

После того, как проверка доступа завершается из-за достижения даты окончания или из-за остановки вручную, реализовать результат проверки можно с помощью кнопки **Применить** . Если в процессе проверки пользователю было отказано в доступе, то этот шаг приведет к удалению назначенной пользователю роли.  

### <a name="export"></a>Экспорт

Если вы хотите применить результаты проверки доступа вручную, можно экспортировать проверку. При нажатии кнопки **Экспорт** будет скачан CSV-файл. Открыть этот файл можно в Excel или в других программах, поддерживающих CSV-файлы.

### <a name="delete"></a>DELETE

Если проверка вам больше не нужна, удалите ее. Кнопка **«Удалить»** удаляет отзыв из службы privileged Identity Management.

> [!IMPORTANT]
> Вам не потребуется подтвердить это разрушительное изменение, поэтому убедитесь, что вы хотите удалить этот отзыв.

# <a name="management-capabilities-for-azure-ad-roles-in-privileged-identity-management"></a>Возможности управления для ролей Azure AD в привилегированном управлении идентификацией

Опыт управления для ролей Azure AD в Privileged Identity Management был обновлен, чтобы унифицировать управление ролями Azure AD и ресурсами Azure. Ранее в привилегированном управлении идентификацией для ресурсов Azure было несколько ключевых функций, которые не были доступны для ролей Azure AD.

В настоящее время, когда обновление развертывается, мы объединяем их в единый опыт управления, и в нем вы получаете ту же функциональность для ролей Azure AD, что и для ролей ресурсов Azure. Эта статья информирует вас об обновленных функциях и любых требованиях.


## <a name="time-bound-assignments"></a>Назначения, связанные с графиком

Ранее в привилегированных функциях управления идентификацией для ролей Azure AD вы были знакомы с рожными назначениями с двумя возможными состояниями : *подходящим* и *постоянным.* Теперь вы можете установить время начала и окончания для каждого типа заданий. Это дополнение дает четыре возможных состояния, в которых вы можете разместить задание:

- Подходит для постоянной основе
- Активный постоянный
- Подходит, с указанными датами начала/конца для назначения
- Активный, с указанными датами начала/конца для назначения

Во многих случаях, даже если вы не хотите, чтобы пользователи имели право на назначение и активировали роли каждый раз, вы все равно можете защитить свою организацию Azure AD, установив время истечения срока действия заданий. Например, если у вас есть временные пользователи, которые имеют право, подумайте об установлении истечения срока действия, чтобы удалить их автоматически из назначения роли, когда их работа завершена.

## <a name="new-role-settings"></a>Новые настройки ролей

Мы также добавляем новые настройки для ролей Azure AD. Ранее можно было настроить настройки активации только на основе одной роли. То есть, параметры активации, такие как многофакторные требования аутентификации и требования к билетам на инцидент/запрос, применялись ко всем пользователям, имеющим право на определенную роль. Теперь можно настроить, нужно ли отдельному пользователю выполнять многофакторную аутентификацию, прежде чем он сможет активировать роль. Кроме того, вы можете иметь расширенный контроль над электронной почтой Privileged Identity Management, связанными с определенными ролями.

## <a name="extend-and-renew-assignments"></a>Продление и возобновление заданий

Как только вы вычисляете назначение запределяемых по времени, первый вопрос, который вы можете задать, это то, что произойдет, если срок действия роли истек? В этой новой версии мы предоставляем два варианта для этого сценария:

- Extend - Когда назначение роли приближается к истечению срока действия, пользователь может использовать Привилегированном управлении идентификацией, чтобы запросить продление для этого назначения роли
- Обновление : По истечении срока действия назначения ролей пользователь может использовать Privileged Identity Management для запроса продления для этого назначения ролей

Оба действия, инициированные пользователем, требуют одобрения глобального администратора или администратора привилегированных ролей. Администрированию больше не нужно будет находиться в бизнесе по управлению этими сроками годности. Им просто нужно дождаться запросов на продление или продление и утвердить их, если запрос действителен.

## <a name="api-changes"></a>Изменения в API

Когда обновленная версия будет развернута в организации Azure AD, существующий API-график перестанет работать. Необходимо перейти к использованию [API-извне для ресурсов Azure.](https://docs.microsoft.com/graph/api/resources/privilegedidentitymanagement-resources?view=graph-rest-beta) Чтобы управлять ролями Azure AD `/azureResources` с `/aadroles` помощью этого API, подменяйте с подписью и используйте идентификатор каталога для `resourceId`.

Мы постарались обратиться ко всем клиентам, которые используют предыдущий API, чтобы сообщить им об этом изменении заранее. Если организация Azure AD перешла на новую версию и вы по-прежнему зависите от старого API, обратитесь к команде в центре pim_preview@microsoft.comвнимания.

## <a name="powershell-change"></a>Смена PowerShell

Для клиентов, использующих модуль Privileged Identity Management PowerShell для ролей Azure AD, PowerShell прекратит работу с обновлением. Вместо предыдущих cmdlets необходимо использовать cmdlets Privileged Identity Management внутри модуля Azure AD Preview PowerShell. Установите модуль Azure AD PowerShell из [галереи PowerShell.](https://www.powershellgallery.com/packages/AzureADPreview/2.0.0.17) Теперь вы можете [прочитать документацию и образцы для операций PIM в этом модуле PowerShell.](powershell-for-azure-ad-roles.md)

# <a name="configure-security-alerts-for-azure-ad-roles-in-privileged-identity-management"></a>Настройка предупреждений безопасности для ролей Azure AD в привилегированном управлении идентификацией

Привилегированное управление идентификацией (PIM) генерирует оповещения при подозрительной или небезопасной деятельности в организации Azure Active Directory (Azure AD). При срабатывании оповещения оно отображается на панели мониторинга Privileged Identity Management. Выберите оповещение, чтобы просмотреть список пользователей или ролей, активировавших предупреждение.

## <a name="determine-your-version-of-pim"></a>Определите свою версию PIM

Начиная с ноября 2019 года, часть ролей Azure AD в Привилегированном управлении идентификацией обновляется в новую версию, которая соответствует опыту для ролей ресурсов Azure. Это создает дополнительные функции, а также [изменения в существующем API.](azure-ad-roles-features.md#api-changes) В то время как новая версия в настоящее время развернута, какие процедуры, которые вы следуете в этой статье зависит от версии Привилегированного управления идентификацией у вас в настоящее время. Выполните действия в этом разделе, чтобы определить, какая версия привилегированного управления идентификацией у вас есть. После того, как вы знаете свою версию Privileged Identity Management, вы можете выбрать процедуры в этой статье, которые соответствуют этой версии.

1. Вопием на [портал Azure](https://portal.azure.com/) с пользователем, который занимается ролью [администратора привилегированных.](../users-groups-roles/directory-assign-admin-roles.md#privileged-role-administrator)
1. Открытое **управление привилегированным удостоверением личности Azure AD.** Если у вас есть баннер в верхней части страницы обзора, следуйте инструкциям в **новой версии** вкладке этой статьи. В противном случае следуйте инструкциям во вкладке **Предыдущая версия.**

  [![](media/pim-how-to-add-role-to-user/pim-new-version.png "Select Azure AD > Privileged Identity Management")](media/pim-how-to-add-role-to-user/pim-new-version.png#lightbox)

Выполните действия в этой статье, чтобы исследовать оповещения безопасности для ролей Azure AD.

## <a name="security-alerts"></a>Оповещения безопасности

В этом разделе перечислены все оповещения безопасности для ролей Azure AD, а также способы исправления и предотвращения. Серьезность означает следующее:

- **Высокий**: требуется немедленное действие из-за нарушения политики.
- **Средний**: немедленное действие не требуется, но обнаружено потенциальное нарушение политики.
- **Низкий**: Немедленное действие не требуется, но предлагается рекомендуемое изменение политики.

### <a name="administrators-arent-using-their-privileged-roles"></a>Администраторы не используют свои привилегированные роли

| | |
| --- | --- |
| **Серьезность** | Низкий |
| **Почему появилось это оповещение?** | Если пользователям назначены привилегированные роли, которые им не нужны, это увеличивает вероятность атаки. Это также позволяет злоумышленникам оставаться незамеченными в учетными записях, которые не используются активно. |
| **Как исправить?** | Просмотрите пользователей в списке и удалите их из привилегированных ролей, которые им не нужны. |
| **Предотвращения** | Назначать привилегированные роли только пользователям, имеющим бизнес-обоснование. </br>Запланируйте регулярные [проверки доступа](pim-how-to-start-security-review.md), чтобы убедиться, что пользователям по прежнему требуется доступ. |
| **Действие по устранению рисков на портале** | Удаляет учетную запись из привилегированной роли. |
| **Триггер** | Срабатывает, если пользователь переходит определенное количество дней без активации роли. |
| **Число дней** | Эта настройка определяет максимальное количество дней, от 0 до 100, что пользователь может пойти без активации роли.|

### <a name="roles-dont-require-multi-factor-authentication-for-activation"></a>Для активации ролям не нужна многофакторная проверка подлинности

| | |
| --- | --- |
| **Серьезность** | Низкий |
| **Почему появилось это оповещение?** | Без многофакторной аутентификации скомпрометированные пользователи могут активировать привилегированные роли. |
| **Как исправить?** | Просмотрите список ролей и [требуйте многофакторной аутентификации](pim-how-to-change-default-settings.md) для каждой роли. |
| **Предотвращения** | [Настройте принудительное выполнение MFA](pim-how-to-change-default-settings.md) для каждой роли.  |
| **Действие по устранению рисков на портале** | Делает многофакторную аутентификацию, необходимую для активации привилегирования. |

### <a name="the-organization-doesnt-have-azure-ad-premium-p2"></a>Организация не имеет Azure AD Premium P2

| | |
| --- | --- |
| **Серьезность** | Низкий |
| **Почему появилось это оповещение?** | Текущая организация Azure AD не имеет Azure AD Premium P2. |
| **Как исправить?** | Просмотрите сведения о [выпусках Azure AD](../fundamentals/active-directory-whatis.md). Обновите до Azure AD Premium P2. |

### <a name="potential-stale-accounts-in-a-privileged-role"></a>Потенциальные устаревшие учетные записи с привилегированной ролью

| | |
| --- | --- |
| **Серьезность** | Средний |
| **Почему появилось это оповещение?** | Учетные записи в привилегиротом роли не изменили свой пароль в течение последних 90 дней. Это могут быть неиспользуемые учетные записи служб или общие учетные записи, которые уязвимы к атакам злоумышленников. |
| **Как исправить?** | Просмотрите учетные записи в списке. Если им больше не нужен доступ, удалите их из привилегированных ролей. |
| **Предотвращения** | Убедитесь, что при изменении пользователей, которым известен пароль, для общих учетных записей сменяются надежные пароли. </br>Регулярно просматривайте учетные записи с привилегированными ролями с применением [проверок доступа](pim-how-to-start-security-review.md) и удаляйте ненужные назначения ролей. |
| **Действие по устранению рисков на портале** | Удаляет учетную запись из привилегированной роли. |
| **Рекомендации** | Учетные записи общего доступа, службы общего пользования и экстренного доступа, которые удостоверяются с помощью пароля и присваиваются привилегированным административным функциям, таким как Глобальный администратор или администратор безопасности, должны вращаться в следующих случаях:<ul><li>после проблемы с безопасностью, в том числе несанкционированного использования или компрометации прав администратора для получения доступа;</li><li>после изменения каких-либо разрешений пользователя, чтобы отозвать права администратора (например, если сотрудник, которому была назначена роль администратора покидает ИТ-отдел или организацию);</li><li>регулярно (например, ежеквартально или ежегодно), даже если не обнаружено брешей в системе безопасности или состав ИТ-отдела не изменился.</li></ul>Так как доступ к этим учетным данным учетных записей есть у нескольких пользователей, нужно сменить учетные данные, чтобы у пользователей с отозванными ролями не было доступа к этим учетным записям. [Подробнее об обеспечении безопасности учетных записей](../users-groups-roles/directory-admin-roles-secure.md) |

### <a name="roles-are-being-assigned-outside-of-privileged-identity-management"></a>Роли назначаются за пределами привилегированного управления идентификацией

| | |
| --- | --- |
| **Серьезность** | Высокий |
| **Почему появилось это оповещение?** | Привилегированные назначения ролей, сделанные за пределами Privileged Identity Management, не контролируются должным образом и могут указывать на активную атаку. |
| **Как исправить?** | Просмотрите пользователей в списке и удалите их из привилегированных ролей, назначенных за пределами Привилегированного управления идентификацией. |
| **Предотвращения** | Исследуйте, где пользователям присваиваются привилегированные роли за пределами Привилегированного управления идентификацией, и запретите будущие назначения оттуда. |
| **Действие по устранению рисков на портале** | Удаляет пользователя из привилегировой роли. |

### <a name="there-are-too-many-global-administrators"></a>Слишком много глобальных администраторов

| | |
| --- | --- |
| **Серьезность** | Низкий |
| **Почему появилось это оповещение?** | Глобальный администратор является наивысшей привилегиротой ролью. Если Глобальный администратор скомпрометирован, злоумышленник получает доступ ко всем своим разрешениям, что ставит под угрозу всю систему. |
| **Как исправить?** | Просмотрите пользователей в списке и удалите все, что не обязательно нуждается в роли глобального администратора. </br>Вместо этого назначаем этим пользователям более низкие привилегированные роли. |
| **Предотвращения** | Назначайте пользователям привилегированные роли с минимально возможными правами. |
| **Действие по устранению рисков на портале** | Удаляет учетную запись из привилегированной роли. |
| **Триггер** | Активируется, если выполняются два условия, каждое из которых можно настроить. Во-первых, необходимо достичь определенного порога глобальных администраторов. Во-вторых, определенный процент от общего числа назначений ролей должен быть глобальными администраторами. Если выполняется только одно из условий, оповещение не отображается. |
| **Минимальное число глобальных администраторов** | В этом параметре указывается количество глобальных администраторов, от 2 до 100, которых, по вашему усмотрению, слишком мало для вашей организации Azure AD. |
| **Процентная доля глобальных администраторов** | В этом параметре указывается минимальный процент администраторов, которые являются глобальными администраторами, от 0% до 100%, ниже которого вы не хотите, чтобы ваша организация Azure AD окуналась. |

### <a name="roles-are-being-activated-too-frequently"></a>Роли активируются слишком часто

| | |
| --- | --- |
| **Серьезность** | Низкий |
| **Почему появилось это оповещение?** | Множественные активации одной привилегированной роли тем же пользователем являются признаком атаки. |
| **Как исправить?** | Просмотрите пользователей в списке и убедитесь, что для привилегированной роли задана достаточная [длительность активации](pim-how-to-change-default-settings.md) для выполнения задач пользователя. |
| **Предотвращения** | Убедитесь, что для привилегированной роли задана достаточная [длительность активации](pim-how-to-change-default-settings.md) для выполнения задач пользователя.</br>[Требуется многофакторная аутентификация](pim-how-to-change-default-settings.md) для привилегированных ролей, учетных записей которых разделяют несколько администраторов. |
| **Действие по устранению рисков на портале** | Недоступно |
| **Триггер** | Активируется, если пользователь активирует одну и ту же привилегированную роль несколько раз в течение указанного периода времени. Можно настроить как период времени, так и количество активаций. |
| **Временной интервал обновления активации** | Этот параметр указывает период времени в днях, часах, минутах и секундах. Он будет использоваться для отслеживания подозрительных обновлений. |
| **Число обновлений активации** | В этой настройке указывается количество активаций, от 2 до 100, в которых вы хотели бы быть уведомлены, в течение выбранного срока. Чтобы изменить значение, переместите ползунок или введите число в текстовое поле. |

| **Рекомендации** | Учетные записи общего доступа, службы общего пользования и экстренного доступа, которые удостоверяются с помощью пароля и присваиваются привилегированным административным функциям, таким как Глобальный администратор или администратор безопасности, должны вращаться в следующих случаях:<ul><li>после проблемы с безопасностью, в том числе несанкционированного использования или компрометации прав администратора для получения доступа;</li><li>после изменения каких-либо разрешений пользователя, чтобы отозвать права администратора (например, если сотрудник, которому была назначена роль администратора покидает ИТ-отдел или организацию);</li><li>регулярно (например, ежеквартально или ежегодно), даже если не обнаружено брешей в системе безопасности или состав ИТ-отдела не изменился.</li></ul>Так как доступ к этим учетным данным учетных записей есть у нескольких пользователей, нужно сменить учетные данные, чтобы у пользователей с отозванными ролями не было доступа к этим учетным записям. [Дополнительные сведения](https://aka.ms/breakglass) |

### <a name="roles-are-being-assigned-outside-of-privileged-identity-management"></a>Роли назначаются за пределами привилегированного управления идентификацией

| | |

| **Серьезность** | Высокий |
| **Почему появилось это оповещение?** | Привилегированные назначения ролей, сделанные за пределами Privileged Identity Management, не контролируются должным образом и могут указывать на активную атаку. |
| **Как исправить?** | Просмотрите пользователей в списке и удалите их из привилегированных ролей, назначенных за пределами Привилегированного управления идентификацией. |
| **Предотвращения** | Исследуйте, где пользователям присваиваются привилегированные роли за пределами Привилегированного управления идентификацией, и запретите будущие назначения оттуда. |
| **Действие по устранению рисков на портале** | Удаляет пользователя из привилегировой роли. |

### <a name="there-are-too-many-global-administrators"></a>Слишком много глобальных администраторов

| | |
| --- | --- |
| **Серьезность** | Низкий |
| **Почему появилось это оповещение?** | Глобальный администратор является наивысшей привилегиротой ролью. Если Глобальный администратор скомпрометирован, злоумышленник получает доступ ко всем своим разрешениям, что ставит под угрозу всю систему. |
| **Как исправить?** | Просмотрите пользователей в списке и удалите все, что не обязательно нуждается в роли глобального администратора. </br>Вместо этого назначаем этим пользователям более низкие привилегированные роли. |
| **Предотвращения** | Назначайте пользователям привилегированные роли с минимально возможными правами. |
| **Действие по устранению рисков на портале** | Удаляет учетную запись из привилегированной роли. |
| **Триггер** | Активируется, если выполняются два условия, каждое из которых можно настроить. Во-первых, необходимо достичь определенного порога глобальных администраторов. Во-вторых, определенный процент от общего числа назначений ролей должен быть глобальными администраторами. Если выполняется только одно из условий, оповещение не отображается. |
| **Минимальное число глобальных администраторов** | В этом параметре указывается количество глобальных администраторов, от 2 до 100, которых, по вашему усмотрению, слишком мало для вашей организации Azure AD. |
| **Процентная доля глобальных администраторов** | В этом параметре указывается минимальный процент администраторов, которые являются глобальными администраторами, от 0% до 100%, ниже которого вы не хотите, чтобы ваша организация Azure AD окуналась. |

### <a name="roles-are-being-activated-too-frequently"></a>Роли активируются слишком часто

| | |
| --- | --- |
| **Серьезность** | Низкий |
| **Почему появилось это оповещение?** | Множественные активации одной привилегированной роли тем же пользователем являются признаком атаки. |
| **Как исправить?** | Просмотрите пользователей в списке и убедитесь, что для привилегированной роли задана достаточная [длительность активации](pim-how-to-change-default-settings.md) для выполнения задач пользователя. |
| **Предотвращения** | Убедитесь, что для привилегированной роли задана достаточная [длительность активации](pim-how-to-change-default-settings.md) для выполнения задач пользователя.</br>[Требуется многофакторная аутентификация](pim-how-to-change-default-settings.md) для привилегированных ролей, учетных записей которых разделяют несколько администраторов. |
| **Действие по устранению рисков на портале** | Недоступно |
| **Триггер** | Активируется, если пользователь активирует одну и ту же привилегированную роль несколько раз в течение указанного периода времени. Можно настроить как период времени, так и количество активаций. |

| **Число обновлений активации** | В этой настройке указывается количество активаций, от 2 до 100, в которых вы хотели бы быть уведомлены, в течение выбранного срока. Чтобы изменить значение, переместите ползунок или введите число в текстовое поле. |

## <a name="configure-security-alert-settings"></a>Настройка параметров оповещений системы безопасности

Вы можете настроить некоторые оповещения безопасности в Privileged Identity Management для работы с потребностями и целями безопасности вашей организации. Чтобы открыть параметры оповещений системы безопасности, сделайте следующее:

1. Открытое **привилегированное управление идентификацией** в Azure AD.

1. Выберите **роли AD Azure**.

1. Выберите **настройки,** а затем **оповещения.**

    ![Роли Azure AD - Настройки с выбранными оповещениями](./media/pim-how-to-configure-security-alerts/settings-alerts.png)

1. Выберите имя оповещения для настройки настройки для этого оповещения.

    ![Для выбранного оповещения панель настроек настроек оповещения](./media/pim-how-to-configure-security-alerts/security-alert-settings.png)

# <a name="approve-or-deny-requests-for-azure-resource-roles-in-pim"></a>Подтверждение или отклонение запросов для ролей ресурсов Azure в PIM

С помощью Azure Active Directory (Azure AD) Privileged Identity Management (PIM), можно настроить роли, чтобы требовать утверждения для активации и выберите один или несколько пользователей или группы в качестве делегированных утверждающих лиц. Делегированные утверждающие лица имеют 24 часа на утверждение запросов. Если запрос не был утвержден в течение 24 часов, пользователю, наделенному соответствующим правом, необходимо повторно отправить запрос. Продолжительность 24-часового периода для утверждения нельзя изменить.

Выполните действия, описанные в этой статье, чтобы утверждать или отклонять запросы для ролей ресурсов Azure.

## <a name="view-pending-requests"></a>Просмотр ожидающих запросов

При наличии запроса роли ресурса Azure, ожидающего вашего утверждения, вы, как делегированное утверждающее лицо, получите уведомление по электронной почте. Эти ожидающие утверждения запросы можно просмотреть в PIM.

1. Войдите на [портале Azure](https://portal.azure.com/).

1. Откройте страницу **Azure AD Privileged Identity Management**.

1. Выберите **Утверждение запросов**.

    ![Ресурсы Azure. Утверждение запросов](./media/pim-resource-roles-approval-workflow/resources-approve-requests.png)

    В разделе **Запросы на активацию ролей** вы увидите список запросов, ожидающих вашего утверждения.

## <a name="approve-requests"></a>Утверждение запросов

1. Найдите и щелкните запрос, который требуется утвердить. Откроется область утверждения.

    ![Область "Утверждение запросов"](./media/pim-resource-roles-approval-workflow/resources-approve-pane.png)

1. В поле **Обоснование** введите причину.

1. Нажмите кнопку **Утвердить**.

    Появится уведомление о вашем утверждении.

    ![Уведомление об утверждении](./media/pim-resource-roles-approval-workflow/resources-approve-notification.png)

## <a name="deny-requests"></a>Отклонение запросов

1. Найдите и щелкните запрос, который требуется отклонить. Откроется область утверждения.

    ![Область "Утверждение запросов"](./media/pim-resource-roles-approval-workflow/resources-approve-pane.png)

1. В поле **Обоснование** введите причину.

1. Выберите **Отклонить**.

    Появится уведомление о вашем отказе.

## <a name="workflow-notifications"></a>Уведомления в рамках рабочего процесса

Ниже приведена информация об уведомлениях рабочего процесса.

- Все участники в списке утверждающих получают уведомления по электронной почте, когда требуется рассмотреть запрос на утверждение роли. В этих уведомлениях содержится прямая ссылка на запрос, который утверждающий может подтвердить или отклонить.
- Запросы разрешаются первым участником в списке. Он утверждает или отклоняет запрос.
- Когда утверждающий отвечает на запрос, все участники в списке утверждающих получают уведомление об этом действии.
- Администраторы ресурсов получают уведомление, когда роль участника, запрос которого утвержден, становится активной.

>[!Note]
>Если администратор ресурсов считает, что не нужно активировать роль для участника, запрос которого был утвержден, он может удалить назначение активной роли в PIM. Хотя администраторы ресурсов не получают уведомления об ожидающих запросах (если только они не являются участниками в списке утверждающих лиц), они могут просмотреть и отменить запросы в режиме ожидания для всех пользователей, просмотрев эти запросы в PIM.

# <a name="approve-or-deny-requests-for-azure-resource-roles-in-privileged-identity-management"></a>Утверждение или отказ в предоставлении функций ресурсов Azure в привилегированном управлении identity

С помощью Privileged Identity Management (PIM) в Active Directory Azure (Azure AD) можно настроить роли, требующие утверждения для активации, и выбрать пользователей или групп из организации Azure AD в качестве делегированных утверждений. Мы рекомендуем выбрать двух или более одобрителей для каждой роли, чтобы уменьшить рабочую нагрузку для администратора привилегированных ролей. Делегированные утверждающие лица имеют 24 часа на утверждение запросов. Если запрос не был утвержден в течение 24 часов, пользователю, наделенному соответствующим правом, необходимо повторно отправить запрос. Продолжительность 24-часового периода для утверждения нельзя изменить.

Выполните действия, описанные в этой статье, чтобы утверждать или отклонять запросы для ролей ресурсов Azure.

## <a name="view-pending-requests"></a>Просмотр ожидающих запросов

При наличии запроса роли ресурса Azure, ожидающего вашего утверждения, вы, как делегированное утверждающее лицо, получите уведомление по электронной почте. Вы можете просмотреть эти незавершенные запросы в Privileged Identity Management.

1. Войдите на [портал Azure](https://portal.azure.com/).

1. Открытое **управление привилегированным удостоверением личности Azure AD.**

1. Выберите **Запросы утверждения**.

    ![Утверждение запросов - страница ресурсов Azure, показывающая запрос на рецензию](./media/pim-resource-roles-approval-workflow/resources-approve-requests.png)

    В разделе **Запросы на активацию ролей** вы увидите список запросов, ожидающих вашего утверждения.

## <a name="approve-requests"></a>Утверждение запросов

1. Найдите и выберите запрос, который вы хотите одобрить. Отображается страница «Утверждение или отказ».

    ![Утвердить запросы - утвердить или отказать в панели с подробной информацией и Обоснование поле](./media/pim-resource-roles-approval-workflow/resources-approve-pane.png)

1. В поле **Justification** введите бизнес-обоснование.

1. Выберите **Утвердить**. Вы получите уведомление Azure о вашем одобрении.

    ![Утверждено уведомление с указанием запроса](./media/pim-resource-roles-approval-workflow/resources-approve-notification.png)

## <a name="deny-requests"></a>Отклонение запросов

1. Найдите и выберите запрос, который вы хотите отклонить. Отображается страница «Утверждение или отказ».

    ![Утвердить запросы - утвердить или отказать в панели с подробной информацией и Обоснование поле](./media/pim-resource-roles-approval-workflow/resources-approve-pane.png)

1. В поле **Justification** введите бизнес-обоснование.

1. Выберите **Отрицание**. Появится уведомление о вашем отказе.

## <a name="workflow-notifications"></a>Уведомления в рамках рабочего процесса

Ниже приведена информация об уведомлениях рабочего процесса.

- Утвердители уведомляются по электронной почте, когда запрос на роль находится на рассмотрении. В этих уведомлениях содержится прямая ссылка на запрос, который утверждающий может подтвердить или отклонить.
- Запросы решаются первым утверждением, который утверждает или отрицает.
- Когда одобритель отвечает на запрос, все одобрители уведомляются об действии.
- Администраторы ресурсов уведомляются, когда утвержденный пользователь становится активным в своей роли.

>[!Note]
>Администратор ресурсов, который считает, что утвержденный пользователь не должен быть активным, может удалить назначение активной роли в Privileged Identity Management. Хотя администраторы ресурсов не уведомляются о ожидающих запросах, если они не являются одобренными, они могут просматривать и отменять ожидающие запросы для всех пользователей, просматривая ожидающие запросы в Privileged Identity Management.

# <a name="license-requirements-to-use-privileged-identity-management"></a>License requirements to use Privileged Identity Management (Требования к лицензии для использования PIM)

Чтобы использовать управление привилегированными пользователями (PIM) в Azure Active Directory (Azure AD), у каталога должна быть допустимая лицензия. Кроме того, лицензии должны быть назначены администраторам и соответствующим пользователям. В этой статье описаны лицензионные требования к использованию Privileged Identity Management.

## <a name="valid-licenses"></a>Действительные лицензии

## <a name="how-many-licenses-must-you-have"></a>Сколько лицензий должно быть у вас?

Убедитесь, что в каталоге есть не менее столько лицензий Azure AD Premium P2, сколько у вас есть сотрудники, которые будут выполнять следующие задачи:

- Пользователи, назначенные как имеющие право на роли Azure AD, управляемые с помощью PIM
- Пользователи могут утверждать или отклонять запросы на активацию в PIM
- Пользователи, которым назначена роль ресурса Azure с использованием JIT-назначений или прямых назначений (на основе времени).  
- Пользователи, которые назначены для проверки доступа.
- Пользователи, которые выполняют проверки доступа.

Лицензии Azure AD Premium P2 **не** требуются для следующих задач:

- Для пользователей с функциями Глобального администратора или привилегированном администратора ролей, которые настраивают PIM, настраивают политики, получают оповещения и настраивают обзоры доступа, не требуются лицензии.

Для получения дополнительной информации о лицензиях см. [Назначьте или удалите лицензии с помощью портала Active Directory Azure.](../fundamentals/license-users-groups.md)

## <a name="what-happens-when-a-license-expires"></a>Что происходит, когда срок действия лицензии истекает?

Если срок действия Azure AD Premium P2, EMS E5 или пробной лицензии истекает, функции Privileged Identity Management больше не будут доступны в вашем каталоге:

- назначенные постоянные роли Azure AD сохраняются без изменений;
- Служба privileged Identity Management на портале Azure, а также интерфейсы Graph API cmdlets и интерфейсы PowerShell privileged Identity Management больше не будут доступны пользователям для активации привилегированных ролей, управления привилегированным доступом или выполнения интерфейсов PowerShell обзоры привилегированных ролей.
- удаляются назначения для ролей Azure AD, так как пользователи уже не могут активировать привилегированные роли;
- Любые текущие проверки ролей Azure AD закроются, а настройки конфигурации Privileged Identity Management будут удалены.
- Привилегированном управлении идентификацией больше не будет отправлять электронные письма по изменениям назначения ролей.

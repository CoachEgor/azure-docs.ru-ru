# <a name="tutorial-configure-smartsheet-for-automatic-user-provisioning"></a>Учебник: Настройка Smartsheet для автоматического подготовки пользователей

Цель этого руководства состоит в том, чтобы продемонстрировать шаги, которые будут выполнены в Smartsheet и Azure Active Directory (Azure AD) для настройки Azure AD для автоматического предоставления и де-предоставления пользователям и/или группам Smartsheet.

> [!NOTE]
> В этом руководстве рассматривается соединитель, созданный на базе службы подготовки пользователей Azure AD. Подробные сведения о том, что делает эта служба, как она работает, и часто задаваемые вопросы см. в статье [Автоматическая подготовка пользователей и ее отзыв для приложений SaaS в Azure Active Directory](../app-provisioning/user-provisioning.md).
>
> Сейчас этот соединитель предоставляется в общедоступной предварительной версии. Дополнительные сведения об общих условиях использования продуктов в предварительной версии см. в документе [Дополнительные условия использования Предварительных версий Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

## <a name="prerequisites"></a>Предварительные требования

В сценарии, описанном в этом руководстве, предполагается, что у вас уже имеется:

* клиент Azure AD;
* [Арендатор Smartsheet](https://www.smartsheet.com/pricing)
* Учетная запись пользователя в плане Smartsheet Enterprise или Enterprise Premier с разрешениями системного администратора.

## <a name="assign-users-to-smartsheet"></a>Назначить пользователей Smartsheet

Активный каталог Azure использует концепцию, называемую *назначениями,* чтобы определить, какие пользователи должны получить доступ к выбранным приложениям. В контексте автоматической подготовки пользователей синхронизируются только пользователи и/или группы, назначенные приложению azure AD.

Прежде чем настроить и включить автоматическую подготовку пользователей, необходимо решить, какие пользователи и/или группы в Azure AD нуждаются в доступе к Smartsheet. После решения вы можете назначить этих пользователей и/или групп Smartsheet, следуя инструкциям здесь:

* [Назначение корпоративному приложению пользователя или группы](../manage-apps/assign-user-or-group-access-portal.md)

### <a name="important-tips-for-assigning-users-to-smartsheet"></a>Важные советы по назначению пользователей в Smartsheet

* Для тестирования автоматической конфигурации подготовки пользователей рекомендуется назначить одного пользователя Azure AD. Дополнительные пользователи и/или группы можно назначить позднее.

* При назначении пользователя Smartsheet необходимо выбрать любую действительную роль приложения (если она имеется) в диалоге заданий. Пользователи с ролью **доступа по умолчанию** исключены из подготовки.

* Для обеспечения паритета в назначениях ролей пользователя между Smartsheet и Azure AD рекомендуется использовать те же назначения ролей, заполненные в полном списке пользователей Smartsheet. Чтобы получить этот список пользователей из Smartsheet, перейдите к администрации пользователя > > **больше действий > Загрузить список пользователей (csv).**

* Чтобы получить доступ к определенным функциям в приложении, Smartsheet требует, чтобы пользователь должен иметь несколько ролей. Чтобы узнать больше о типах пользователей и разрешениях в Smartsheet, перейдите на [типы и разрешения пользователей.](https://help.smartsheet.com/learning-track/shared-users/user-types-and-permissions)

*  Если пользователь имеет несколько ролей, назначенных в Smartsheet, **необходимо** убедиться, что эти назначения ролей реплицируются в Azure AD, чтобы избежать сценария, при котором пользователи могут потерять доступ к объектам Smartsheet навсегда. Каждая уникальная роль в Smartsheet **должна** быть отнесена к другой группе в Azure AD. Пользователь **должен** быть добавлен к каждой из групп, соответствующих желаемым ролям. 

## <a name="set-up-smartsheet-for-provisioning"></a>Настройка Smartsheet для подготовки

Прежде чем настроить Smartsheet для автоматического предоставления пользовательского запроса с помощью Azure AD, необходимо включить SCIM-подготовку на Smartsheet.

1. Войдите в систему в качестве **SysAdmin** на **[портале Smartsheet](https://app.smartsheet.com/b/home)** и перейдите к **учетной записи Admin**.

    ![Админ учетной записи Smartsheet](media/smartsheet-provisioning-tutorial/smartsheet-accountadmin.png)

2. Перейти к **безопасности управления > пользователь автоматического обеспечения > отсылать**.

    ![Средства управления безопасностью смарт-листов](media/smartsheet-provisioning-tutorial/smartsheet-securitycontrols.png)

3. Добавляйте и проверяйте домены электронной почты для пользователей, которые вы планируете предоставлять от Azure AD к Smartsheet. Выберите **не включенные,** чтобы убедиться, что все действия подготовки происходят только из Azure AD, а также убедиться, что список пользователей Smartsheet синхронизирован с назначениями Azure AD.

    ![Подготовка пользователей смарт-листов](media/smartsheet-provisioning-tutorial/smartsheet-userprovisioning.png)

4. Как только проверка будет завершена, вам придется активировать домен. 

    ![Интеллектуальный домен](media/smartsheet-provisioning-tutorial/smartsheet-activatedomain.png)

5. Создайте **секретный токен,** необходимый для настройки автоматической подготовки пользователей с помощью Azure AD, перемещаясь по **приложениям и интеграциям.**

    ![Установка смарт-листов](media/smartsheet-provisioning-tutorial/Smartsheet05.png)

6. Выберите **доступ к API**. Нажмите **Создать новый токен доступа**.

    ![Установка смарт-листов](media/smartsheet-provisioning-tutorial/Smartsheet06.png)

7. Определите название токена доступа API. Нажмите кнопку **ОК**.

    ![Установка смарт-листов](media/smartsheet-provisioning-tutorial/Smartsheet07.png)

8. Копируйте токен доступа API и сохраните его, так как это будет единственный раз, когда вы сможете его просмотреть. Это необходимо в поле **Secret Token** в Azure AD.

    ![Токен смарт-листов](media/smartsheet-provisioning-tutorial/Smartsheet08.png)

## <a name="add-smartsheet-from-the-gallery"></a>Добавить Smartsheet из галереи

Чтобы настроить Smartsheet для автоматического оформления пользовательского интерфейса с помощью Azure AD, необходимо добавить Smartsheet из галереи приложений Azure AD в список управляемых приложений SaaS.

1. На **[портале Azure](https://portal.azure.com)** на левой навигационной панели выберите **Active Directory Azure.**

    ![Кнопка Azure Active Directory](common/select-azuread.png)

2. Перейдите к **корпоративным приложениям,** а затем выберите **все приложения.**

    ![Колонка "Корпоративные приложения"](common/enterprise-applications.png)

3. Чтобы добавить новое приложение, выберите кнопку **Нового приложения** в верхней части панели.

    ![Кнопка "Создать приложение"](common/add-new-app.png)

4. В поле поиска введите **Smartsheet,** выберите **Smartsheet** в панели результатов. 

    ![Смарт-лист в списке результатов](common/search-new-app.png)

5. Выберите кнопку **Регистрация для Smartsheet,** которая перенаправит вас на страницу входа Smartsheet. 

    ![Smartsheet OIDC Добавить](media/smartsheet-provisioning-tutorial/smartsheet-OIDC-add.png)

6. Поскольку Smartsheet является приложением OpenIDConnect, выберите вход в Smartsheet с помощью рабочей учетной записи Майкрософт.

    ![Логин Smartsheet OIDC](media/smartsheet-provisioning-tutorial/smartsheet-OIDC-login.png)

7. После успешной проверки подлинности примите запрос согласия на страницу согласия. Приложение будет автоматически добавлено к вашему арендатору, и вы будете перенаправлены на ваш аккаунт Smartsheet.

    ![Smartsheet OIDc Согласие](media/smartsheet-provisioning-tutorial/smartsheet-OIDC-consent.png)

## <a name="configure-automatic-user-provisioning-to-smartsheet"></a>Настройка автоматического предоставления пользователям smartsheet 

В этом разделе вы проведет шаги по настройке службы предоставления AD Azure AD для создания, обновления и отработки пользователей и/или групп в Smartsheet на основе пользовательских и/или групповых заданий в Azure AD.

### <a name="to-configure-automatic-user-provisioning-for-smartsheet-in-azure-ad"></a>Для настройки автоматического подготовки пользователя для Smartsheet в Azure AD:

1. Войдите на [портал Azure](https://portal.azure.com). Выберите **корпоративные приложения,** затем выберите **все приложения.**

    ![Колонка "Корпоративные приложения"](common/enterprise-applications.png)

2. В списке приложений выберите **Smartsheet**.

    ![Ссылка на смарт-лист в списке приложений](common/all-applications.png)

3. Выберите вкладку **«Подготовка».**

    ![Вкладка подготовки](common/provisioning.png)

4. Установите **режим обеспечения** **автоматического.**

    ![Вкладка подготовки](common/provisioning-automatic.png)

5. В разделе **Admin Credentials** `https://scim.smartsheet.com/v2/` ввод **URL-адреса клиента**. Ввейте значение, которое вы извлекли и сохранено ранее из Smartsheet в **Secret Token.** Нажмите **на тестовое соединение,** чтобы обеспечить подключение Azure AD к Smartsheet. Если соединение не удается, убедитесь, что ваша учетная запись Smartsheet имеет разрешения SysAdmin и повторите попытку.

    ![Токен](common/provisioning-testconnection-tenanturltoken.png)

6. В поле **Почтовое уведомление** введите адрес электронной почты пользователя или группы, которые должны получать уведомления об ошибках подготовки, а также установите флажок **Send an email notification when a failure occurs** (Отправить уведомление по электронной почте при сбое).

    ![Почтовое уведомление](common/provisioning-notification-email.png)

7. Нажмите **Сохранить**.

8. В разделе **Картпинги** выберите **синхронизацию активных пользователей каталога Azure в Smartsheet.**

    ![Картирование пользователей смарт-листов](media/smartsheet-provisioning-tutorial/smartsheet-user-mappings.png)

9. Просмотрите атрибуты пользователя, синхронизированные с Azure AD на Smartsheet в разделе **Атрибуты Mapping.** Атрибуты, выбранные в качестве свойств **соответствия,** используются для сопоставления учетных записей пользователей в Smartsheet для операций обновления. Нажмите кнопку **Сохранить**, чтобы зафиксировать все изменения.

    ![Атрибуты пользователя Смарт-листа](media/smartsheet-provisioning-tutorial/smartsheet-user-attributes.png)

10. Чтобы настроить фильтры области, ознакомьтесь со следующими инструкциями, предоставленными в [руководстве по фильтрам области](../app-provisioning/define-conditional-rules-for-provisioning-user-accounts.md).

11. Чтобы включить службу предоставления AD Azure AD для Smartsheet, измените **статус подготовки** в разделе **"Настройки".** **On**

    ![Состояние подготовки "Включено"](common/provisioning-toggle-on.png)

12. Определите пользователей и/или группы, которые вы хотели бы предоставить Smartsheet, выбрав желаемые значения в **разделе Область** в разделе **Параметры.**

    ![Область действия подготовки](common/provisioning-scope.png)

13. Когда будете готовы выполнить подготовку, нажмите кнопку **Сохранить**.

    ![Сохранение конфигурации подготовки](common/provisioning-configuration-save.png)

После этого начнется начальная синхронизация пользователей и (или) групп, определенных в поле **Область** раздела **Параметры**. Начальная синхронизация занимает больше времени, чем последующие операции синхронизации. Если служба запущена, они выполняются примерно каждые 40 минут. Раздел **«Детали синхронизации»** можно использовать для мониторинга прогресса и отслеживания ссылок на отчет о деятельности, в котором описаны все действия, выполняемые службой предоставления AD Azure AD на Smartsheet.

Дополнительные сведения о чтении журналов подготовки Azure AD см. в руководстве по [отчетам об автоматической подготовке учетных записей](../app-provisioning/check-status-user-account-provisioning.md).

## <a name="connector-limitations"></a>Ограничения соединителя

* Smartsheet не поддерживает мягкие удаления. Когда **активный** атрибут пользователя настроен на False, Smartsheet удаляет пользователя навсегда.

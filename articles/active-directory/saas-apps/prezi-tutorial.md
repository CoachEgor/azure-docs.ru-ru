# <a name="tutorial-azure-active-directory-single-sign-on-sso-integration-with-prezi"></a>Руководство по Интеграция единого входа Azure Active Directory с Prezi

В этом руководстве описано, как интегрировать Prezi с Azure Active Directory (Azure AD). Интеграция Prezi с Azure AD обеспечивает следующие возможности:

* Контроль доступа к Prezi с помощью Azure AD.
* Автоматический вход пользователей в Prezi с помощью учетных записей Azure AD.
* Централизованное управление учетными записями через портал Azure.

Чтобы узнать больше об интеграции приложений SaaS с Azure AD, прочитайте статью [Что такое доступ к приложениям и единый вход с помощью Azure Active Directory?](https://docs.microsoft.com/azure/active-directory/manage-apps/what-is-single-sign-on)

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее.

* Подписка Azure AD. Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Подписка Prezi с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* Prezi поддерживает вход, инициированный **поставщиком услуг или поставщиком удостоверений**.
* Prezi поддерживает **JIT**-подготовку пользователей.
* После настройки Prezi вы можете применить функцию управления сеансом, которая защищает от хищения конфиденциальных данных вашей организации и несанкционированного доступа к ним в реальном времени. Управление сеансом является расширением функции условного доступа. [Узнайте, как применять управление сеансами с помощью Microsoft Cloud App Security](https://docs.microsoft.com/cloud-app-security/proxy-deployment-any-app).

## <a name="adding-prezi-from-the-gallery"></a>Добавление Prezi из коллекции

Чтобы настроить интеграцию Prezi с Azure AD, необходимо добавить Prezi из коллекции в список управляемых приложений SaaS.

1. Войдите на [портал Azure](https://portal.azure.com) с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите **Prezi**.
1. Выберите **Prezi** в области результатов и добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.

## <a name="configure-and-test-azure-ad-single-sign-on-for-prezi"></a>Настройка и проверка единого входа Azure AD для Prezi

Настройте и проверьте единый вход Azure AD в Prezi с помощью тестового пользователя **B.Simon**. Для обеспечения работы единого входа необходимо установить связь между пользователем Azure AD и соответствующим пользователем в Prezi.

Чтобы настроить и проверить единый вход Azure AD в Prezi, выполните действия в следующих стандартных блоках.

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
    1. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD с помощью пользователя B.Simon.
    1. **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить пользователю B.Simon использовать единый вход Azure AD.
1. **[Настройка единого входа в Prezi](#configure-prezi-sso)** необходима, чтобы настроить параметры единого входа на стороне приложения.
    1. **[Создание тестового пользователя в Prezi](#create-prezi-test-user)** нужно для того, чтобы в Prezi существовал пользователь B.Simon, связанный с одноименным пользователем в Azure AD.
1. **[Проверка единого входа](#test-sso)** позволяет убедиться в правильности конфигурации.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На [портале Azure](https://portal.azure.com/) на странице интеграции с приложением **Prezi** найдите раздел **Управление** и выберите **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок "Изменить" (значок пера), чтобы открыть диалоговое окно **Базовая конфигурация SAML** и изменить параметры.

   ![Изменение базовой конфигурации SAML](common/edit-urls.png)

1. В разделе **Базовая конфигурация SAML** не нужно выполнять никаких действий, так как приложение уже предварительно интегрировано с Azure.

1. Чтобы настроить приложение для работы в режиме, инициируемом **поставщиком услуг**, щелкните **Задать дополнительные URL-адреса** и выполните следующие действия.

    В текстовом поле **URL-адрес входа** введите URL-адрес: `https://prezi.com/login/sso/`.

1. Выберите команду **Сохранить**.

1. Приложение Prezi ожидает проверочные утверждения SAML в определенном формате, поэтому добавьте настраиваемые сопоставления атрибутов в конфигурацию атрибутов токена SAML. На следующем снимке экрана показан список атрибутов по умолчанию.

    ![Изображение](common/default-attributes.png)

1. В дополнение к описанному выше приложение Prezi ожидает несколько дополнительных атрибутов в ответе SAML, как показано ниже. Эти атрибуты также заранее заполнены, но вы можете изменить их в соответствии со своими требованиями.
    
    | Имя | Исходный атрибут|
    | ---------------| --------------- |
    | given_name | user.givenname |
    | family_name | user.surname |

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** найдите пункт **Сертификат (Base64)** и щелкните **Скачать**, чтобы скачать сертификат. Сохраните этот сертификат на компьютере.

    ![Ссылка для скачивания сертификата](common/certificatebase64.png)

1. Требуемые URL-адреса можно скопировать в разделе **Настройка Prezi**.

    ![Копирование URL-адресов настройки](common/copy-configuration-urls.png)

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory**, **Пользователи**, а затем — **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите `B.Simon`.  
   1. В поле **Имя пользователя** введите username@companydomain.extension. Например, `B.Simon@contoso.com`.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **Создать**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как включить единый вход в Azure для пользователя B.Simon, предоставив этому пользователю доступ к Prezi.

1. На портале Azure выберите **Корпоративные приложения**, а затем — **Все приложения**.
1. В списке приложений выберите **Prezi**.
1. На странице "Обзор" приложения найдите раздел **Управление** и выберите **Пользователи и группы**.

   ![Ссылка "Пользователи и группы"](common/users-groups-blade.png)

1. Выберите **Добавить пользователя**, а в диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.

    ![Ссылка "Добавить пользователя"](common/add-assign-user.png)

1. В диалоговом окне **Пользователи и группы** выберите **B.Simon** в списке пользователей, а затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если ожидается, что в утверждении SAML будет получено какое-либо значение роли, то в диалоговом окне **Выбор роли** нужно выбрать соответствующую роль для пользователя из списка и затем нажать кнопку **Выбрать**, расположенную в нижней части экрана.
1. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить**.

## <a name="configure-prezi-sso"></a>Настройка единого входа Prezi

1. В другом окне веб-браузера войдите в Prezi с помощью учетной записи команды и перейдите в **[консоль администрирования](https://prezi.com/organizations/manage)** .

1. В **консоли администрирования**, перейдите на вкладку **Settings** (Параметры).

    ![Конфигурация Prezi](./media/prezi-tutorial/settings-image.png)

1. Перейдите к разделу **Single Sign-On(SSO)** (Единый вход) и используйте переключатель, чтобы включить единый вход.
    
    ![Конфигурация Prezi](./media/prezi-tutorial/single-signon.png)

1. В разделе **Single sign-on(SSO)** (Единый вход) выполните следующие действия.

    ![Конфигурация Prezi](./media/prezi-tutorial/configuration.png)

    а. В текстовое поле **Identifier or Issuer URL** (Идентификатор или URL-адрес издателя) вставьте значение **идентификатора Azure AD**, скопированное на портале Azure.

    b. В текстовое поле **SAML 2.0 Endpoint(HTTP)** (Конечная точка SAML 2.0 (HTTP)) вставьте значение поля **URL-адрес входа**, скопированное на портале Azure.

    c. Откройте в Блокноте скачанный с портала Azure файл **сертификата (Base64)** , скопируйте содержимое сертификата и вставьте его в текстовое поле **Certificate (X.509)** (Сертификат (X.509)).

    d. Выберите команду **Сохранить**.

### <a name="create-prezi-test-user"></a>Создание тестового пользователя Prezi

В этом разделе показано, как создать в Prezi пользователя с именем Britta Simon. Приложение Prezi поддерживает JIT-подготовку пользователей, которая включена по умолчанию. В этом разделе никакие действия с вашей стороны не требуются. Если пользователь еще не существует в Prezi, он создается после проверки подлинности.

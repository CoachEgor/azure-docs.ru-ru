# <a name="tutorial-configure-merchlogix-for-automatic-user-provisioning"></a>Учебник: Настройка MerchLogix для автоматического подготовки пользователей

Цель этого руководства состоит в том, чтобы продемонстрировать шаги, которые будут выполнены в MerchLogix и Azure Active Directory (Azure AD) для настройки Azure AD для автоматического предоставления и де-предоставления пользователям и/или группам МерчЛогикс.

> [!NOTE]
> В этом руководстве рассматривается соединитель, созданный на базе службы подготовки пользователей Azure AD. Подробные сведения о том, что делает эта служба, как она работает, и часто задаваемые вопросы см. в статье [Автоматическая подготовка пользователей и ее отзыв для приложений SaaS в Azure Active Directory](../app-provisioning/user-provisioning.md).

## <a name="prerequisites"></a>Предварительные требования

В сценарии, описанном в этом руководстве, предполагается, что у вас уже имеется:

* клиент Azure AD;
* Арендатор MerchLogix
* Технический контакт в MerchLogix, который может предоставить URL-адрес endpoint SCIM и секретный токен, необходимый для подготовки пользователей

## <a name="adding-merchlogix-from-the-gallery"></a>Добавление MerchLogix из галереи

Прежде чем настроить MerchLogix для автоматической подготовки пользователей с Azure AD, необходимо добавить MerchLogix из галереи приложений Azure AD в список управляемых приложений SaaS.

**Чтобы добавить MerchLogix из галереи приложений Azure AD, выполните следующие действия:**

1. На **[портале Azure](https://portal.azure.com)** в области навигации слева щелкните значок **Azure Active Directory**. 

    ![Кнопка Azure Active Directory][1]

2. Перейдите к **корпоративным приложениям** > **Все приложения**.

    ![Раздел "Корпоративные приложения"][2]

3. Чтобы добавить MerchLogix, нажмите кнопку **Новое приложение** в верхней части диалога.

    ![Кнопка "Создать приложение"][3]

4. В поле поиска введите **MerchLogix**.

5. В панели результатов выберите **MerchLogix,** а затем нажмите кнопку **Добавить,** чтобы добавить MerchLogix в список приложений SaaS.

    ![МерчЛогс Конъюнты][4]

## <a name="assigning-users-to-merchlogix"></a>Назначение пользователей MerchLogix

В Azure Active Directory для определения того, какие пользователи должны получать доступ к выбранным приложениям, используется концепция, называемая "назначение". В контексте автоматической подготовки синхронизированы будут только те пользователи и группы, которые были "назначены" приложению в Azure AD. 

Прежде чем настроить и включить автоматическую подготовку пользователей, необходимо решить, какие пользователи и/или группы в Azure AD нуждаются в доступе к MerchLogix. После решения, вы можете назначить этих пользователей и / или групп MerchLogix, следуя инструкциям здесь:

* [Назначение корпоративному приложению пользователя или группы](../manage-apps/assign-user-or-group-access-portal.md)

### <a name="important-tips-for-assigning-users-to-merchlogix"></a>Важные советы по назначению пользователей MerchLogix

* Рекомендуется, чтобы один пользователь Azure AD был назначен MerchLogix для тестирования начальной автоматической конфигурации подготовки пользователей. Дополнительные пользователи и (или) группы можно будет назначить позже, после успешного прохождения тестов.

* При назначении пользователя MerchLogix необходимо выбрать любую действительную роль приложения (если она имеется) в диалоге о назначении. Пользователи с ролью **доступа по умолчанию** исключены из подготовки.

## <a name="configuring-automatic-user-provisioning-to-merchlogix"></a>Настройка автоматического обеспечения пользователя к MerchLogix 

В этом разделе вы проведет шаги по настройке службы предоставления AD Azure AD для создания, обновления и отработки пользователей и/или групп в MerchLogix на основе пользовательских и/или групповых заданий в Azure AD.

> [!TIP]
> Вы также можете включить SAML основе одного входа на MerchLogix, следуя инструкциям, представленным в [MerchLogix одного входа на учебник](merchlogix-tutorial.md). Единый вход можно настроить независимо от автоматической подготовки пользователей, хотя эти две возможности дополняют друг друга.

### <a name="to-configure-automatic-user-provisioning-for-merchlogix-in-azure-ad"></a>Для настройки автоматической подготовки пользователей для MerchLogix в Azure AD:

1. Войдите на [портал Azure](https://portal.azure.com) и перейдите к **Azure Active Directory > Корпоративные приложения > Все приложения**.

2. Выберите MerchLogix из списка приложений SaaS.

3. Выберите вкладку **«Подготовка».**

4. Установите **режим обеспечения** **автоматического.**

    ![МерчЛогс Конъюнты](./media/merchlogix-provisioning-tutorial/Merchlogix1.png)

5. В разделе **Полномочия админ:**

    * В поле **URL-адреса Tenant** введите URL-адрес endpoint SCIM, предоставленный техническим контактом MerchLogix.

    * В поле **Secret Token** введите секретный токен, предоставленный вашим техническим контактом MerchLogix.

6. После заполнения полей, показанных в шаге 5, щелкните **тестовое соединение,** чтобы обеспечить подключение Azure AD к MerchLogix. Если соединение выходит из строя, убедитесь, что ваша учетная запись MerchLogix имеет разрешения Admin и повторите попытку.

7. В поле **Почтовое уведомление** введите адрес электронной почты пользователя или группы, которые должны получать уведомления об ошибках подготовки, а также установите флажок **Send an email notification when a failure occurs** (Отправить уведомление по электронной почте при сбое).

8. Нажмите **Сохранить**.

9. В разделе **Картпинги** выберите **синхронизацию активных пользователей каталога Azure в MerchLogix.**

10. Просмотрите атрибуты пользователя, синхронизированные с Azure AD с MerchLogix в разделе **Атрибуткарт.** Атрибуты, выбранные в качестве свойств **соответствия,** используются для сопоставления учетных записей пользователей в MerchLogix для операций обновления. Нажмите кнопку **Сохранить**, чтобы зафиксировать все изменения.

11. В разделе **Картпинги** выберите **синхронизацию групп активных каталогов Azure для MerchLogix.**

12. Просмотрите атрибуты группы, синхронизированные с Azure AD с MerchLogix в разделе **Атрибуткарт.** Атрибуты, выбранные в качестве свойств **соответствия,** используются для соответствия группам в MerchLogix для операций обновления. Нажмите кнопку **Сохранить**, чтобы зафиксировать все изменения.

13. Чтобы включить службу предоставления AD Azure AD для MerchLogix, измените **статус подготовки** в разделе **«Настройки».** **On**

14. Когда будете готовы выполнить подготовку, нажмите кнопку **Сохранить**.

После этого начнется начальная синхронизация пользователей и (или) групп, определенных в поле **Область** раздела **Параметры**. Начальная синхронизация занимает больше времени, чем последующие операции синхронизации. Если служба запущена, они выполняются примерно каждые 40 минут. Раздел **«Детали синхронизации»** можно использовать для мониторинга прогресса и отслеживания ссылок на отчет о деятельности, в котором описаны все действия, выполняемые службой предоставления AD Azure AD на MerchLogix.

Дополнительные сведения о чтении журналов подготовки Azure AD см. в руководстве по [отчетам об автоматической подготовке учетных записей](../app-provisioning/check-status-user-account-provisioning.md).

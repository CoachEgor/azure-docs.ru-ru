# <a name="tutorial-configure-elium-for-automatic-user-provisioning"></a>Учебник. Настройка Elium для автоматической подготовки пользователей

В этом руководстве показано, как настроить Elium и Azure Active Directory (Azure AD) для автоматической инициализации и отзыва пользователей или групп в Elium.

> [!NOTE]
> В этом руководстве описывается соединитель, созданный на основе службы подготовки пользователей Azure AD. Важные сведения о том, что делает эта служба и как она работает, а также о часто задаваемых вопросах см. [в статье Автоматизация подготовки пользователей и ее отмены в приложениях SaaS с помощью Azure Active Directory](../app-provisioning/user-provisioning.md).
>
> Сейчас этот соединитель доступен в режиме предварительной версии. Общие условия использования функций Azure в предварительной версии см. в разделе Дополнительные [условия использования для предварительного просмотра Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

## <a name="prerequisites"></a>Предварительные требования

В этом учебнике предполагается, что у вас уже есть следующие необходимые компоненты.

* клиент Azure AD;
* [Клиент Elium](https://www.elium.com/pricing/)
* Учетная запись пользователя в Elium с разрешениями администратора

## <a name="assigning-users-to-elium"></a>Назначение пользователей в Elium

Azure AD использует концепцию, называемую *назначениями* , чтобы определить, какие пользователи получают доступ к выбранным приложениям. В контексте автоматической подготовки учетных записей пользователей синхронизируются только те пользователи и группы, которые были назначены приложению в Azure AD.

Перед настройкой и включением автоматической подготовки пользователей решите, каким пользователям и группам в Azure AD нужен доступ к Elium. Затем назначьте этих пользователей и группы Elium, выполнив действия, описанные в разделе [Назначение пользователя или группы корпоративному приложению](../manage-apps/assign-user-or-group-access-portal.md).

## <a name="important-tips-for-assigning-users-to-elium"></a>Важные советы по назначению пользователей в Elium 

Мы рекомендуем назначить одного пользователя Azure AD для Elium, чтобы протестировать автоматическую настройку подготовки пользователей. Другие пользователи и группы могут быть назначены позже.

При назначении пользователя в Elium необходимо выбрать допустимую роль конкретного приложения (если она доступна) в диалоговом окне Назначение. Пользователи, которым назначена роль **доступа по умолчанию** , исключаются из подготовки.

## <a name="set-up-elium-for-provisioning"></a>Настройка Elium для подготовки

Перед настройкой Elium для автоматической подготовки пользователей с помощью Azure AD необходимо включить систему для подготовки между доменами (SCIM) в Elium. Выполните следующие действия.

1. Войдите в Elium и перейдите к **My Profile**  >  **параметрам** профиля.

    ![Пункт меню "Параметры" в Elium](media/Elium-provisioning-tutorial/setting.png)

1. В левом нижнем углу в разделе **Дополнительно** выберите **Безопасность**.

    ![Ссылка на безопасность в Elium](media/Elium-provisioning-tutorial/security.png)

1. Скопируйте **URL-адрес клиента** и значения **токена секрета** . Эти значения будут использоваться позже в соответствующих полях на вкладке **Подготовка** приложения Elium в портал Azure.

    ![URL-адрес клиента и поля токена секрета в Elium](media/Elium-provisioning-tutorial/token.png)

## <a name="add-elium-from-the-gallery"></a>Добавление Elium из коллекции

Чтобы настроить Elium для автоматической подготовки пользователей с помощью Azure AD, необходимо также добавить Elium из коллекции приложений Azure AD в список управляемых приложений программного обеспечения как услуги (SaaS). Выполните следующие действия.

1. В [портал Azure](https://portal.azure.com)на панели навигации слева выберите **Azure Active Directory**.

    ![Пункт меню Azure Active Directory](common/select-azuread.png)

1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.

     ![Колонка корпоративных приложений Azure AD](common/enterprise-applications.png)

1. Чтобы добавить новое приложение, выберите **новое приложение** в верхней части панели.

    ![Ссылка на новое приложение](common/add-new-app.png)

1. В поле поиска введите **Elium** , выберите **Elium** в списке результатов и нажмите кнопку **добавить** , чтобы добавить приложение.

    ![Поле поиска в галерее](common/search-new-app.png)

## <a name="configure-automatic-user-provisioning-to-elium"></a>Настройка автоматической подготовки пользователей в Elium

В этом разделе описано, как настроить службу подготовки Azure AD для создания, обновления и отключения пользователей и групп в Elium на основе назначений пользователей и групп в Azure AD.

> [!TIP]
> Вы также можете включить единый вход для Elium на основе язык разметки зявлений системы безопасности (SAML) (SAML), следуя инструкциям в [руководстве по единому входу Elium](Elium-tutorial.md). Вы можете настроить единый вход независимо от автоматической подготовки пользователей, хотя две функции дополняют друг друга.

Чтобы настроить автоматическую подготовку учетных записей пользователей для Elium в Azure AD, выполните следующие действия.

1. Войдите в [портал Azure](https://portal.azure.com), выберите **корпоративные приложения** , а затем выберите **все приложения**.

    ![Колонка корпоративных приложений Azure AD](common/enterprise-applications.png)

1. В списке приложений выберите **Elium**.

    ![Список приложений в колонке "корпоративные приложения"](common/all-applications.png)

1. Выберите вкладку **Подготовка**.

    ![Вкладка "подготовка" в колонке "корпоративные приложения"](common/provisioning.png)

1. Для параметра **Режим подготовки к работе** выберите значение **Automatic** (Автоматически).

    ![Автоматический параметр для режима подготовки](common/provisioning-automatic.png)

1. В разделе **учетные данные администратора** введите **\<tenantURL\> /scim/v2** в поле **URL-адрес клиента** . ( **Тенантурл** — это значение, полученное ранее из консоли администрирования Elium.) Также введите значение **маркера секрета** Elium в поле **секретный токен** . Наконец, выберите **проверить подключение** , чтобы убедиться, что Azure AD может подключиться к Elium. В случае сбоя подключения убедитесь, что у учетной записи Elium есть разрешения администратора, и повторите попытку.

    ![Поля URL-адреса клиента и секретного токена в учетных данных администратора](common/provisioning-testconnection-tenanturltoken.png)

1. В поле e-mail **Notification (уведомление по электронной почте** ) введите адрес электронной почты пользователя или группы, которые будут получать уведомления об ошибках подготовки. Затем установите флажок **Отправить уведомление по электронной почте при возникновении сбоя** .

    ![Почтовое уведомление](common/provisioning-notification-email.png)

1. Выберите команду **Сохранить**.

1. В разделе **сопоставления** выберите **синхронизировать Azure Active Directory пользователей с Elium**.

    ![Синхронизация ссылки для сопоставления пользователей Azure AD с Elium](media/Elium-provisioning-tutorial/usermapping.png)

1. Проверьте атрибуты пользователя, которые синхронизированы из Azure AD в Elium в разделе **сопоставления атрибутов** . Атрибуты, выбранные как свойства **Matching** , используются для сопоставления учетных записей пользователей в Elium для операций обновления. Чтобы зафиксировать изменения, щелкните **Сохранить**.

    ![Сопоставления атрибутов между Azure AD и Elium](media/Elium-provisioning-tutorial/userattribute.png)

1. Чтобы настроить фильтры области, следуйте инструкциям в [руководстве по фильтрации областей](../app-provisioning/define-conditional-rules-for-provisioning-user-accounts.md).

1. Чтобы включить службу подготовки Azure AD для Elium, измените значение параметра **состояние подготовки** на **включено** в разделе **Параметры** .

    ![Для параметра "состояние подготовки" задано значение Вкл.](common/provisioning-toggle-on.png)

1. Определите пользователей и группы, которые вы хотите подготавливать к Elium, выбрав нужные значения в раскрывающемся списке **область** в разделе **Параметры** .

    ![Список области подготовки](common/provisioning-scope.png)

1. Когда будете готовы выполнить подготовку, нажмите кнопку **Сохранить**.

    ![Кнопка "Сохранить" для настройки подготовки](common/provisioning-configuration-save.png)

Эта операция запускает начальную синхронизацию всех пользователей и групп, определенных в **области** , в разделе **параметров** . Этот процесс начальной синхронизации занимает больше времени, чем последующие операции синхронизации. Дополнительные сведения о времени, требуемом для подготовки, см. в статье [как долго займет подготовка пользователей?](../app-provisioning/application-provisioning-when-will-provisioning-finish-specific-user.md#how-long-will-it-take-to-provision-users).

Используйте раздел **Текущее состояние** для отслеживания хода выполнения и перехода по ссылкам к отчету о действиях по подготовке. В отчете о действиях по подготовке описаны все действия, выполняемые службой подготовки Azure AD в Elium. Дополнительные сведения см. в статье [Проверка состояния подготовки пользователей](../app-provisioning/application-provisioning-when-will-provisioning-finish-specific-user.md). Чтобы прочитать журналы подготовки Azure AD, см. раздел [Создание отчетов об автоматической подготовке учетных записей пользователей](../app-provisioning/check-status-user-account-provisioning.md).

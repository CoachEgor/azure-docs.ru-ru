# <a name="tutorial-configure-elium-for-automatic-user-provisioning"></a>Учебник: Настройка Elium для автоматического подготовки пользователей

В этом уроке показано, как настроить активный каталог Elium и Azure (Azure AD) для автоматического предоставления и де-предоставления пользователям или группам ВИУ.

> [!NOTE]
> В этом уроке описывается разъем, построенный поверх службы обеспечения пользователей Azure AD. Для получения важных сведений о том, что эта служба делает и как она работает, а также для часто задаваемых вопросов, [см. Автоматизировать подготовку пользователей и деобеспечения приложений SaaS с Azure Active Directory.](../app-provisioning/user-provisioning.md)
>
> Сейчас этот соединитель доступен в режиме предварительной версии. Общие термины использования функций Azure в предварительном просмотре [см.](https://azure.microsoft.com/support/legal/preview-supplemental-terms/)

## <a name="prerequisites"></a>Предварительные требования

Этот учебник предполагает, что у вас уже есть следующие предпосылки:

* клиент Azure AD;
* [Арендатор Elium](https://www.elium.com/pricing/)
* Учетная запись пользователя в Elium с разрешениями админа

## <a name="assigning-users-to-elium"></a>Назначение пользователей Elium

Azure AD использует концепцию, называемую *назначениями,* чтобы определить, какие пользователи получают доступ к выбранным приложениям. В контексте автоматической подготовки пользователей синхронизируются только пользователи и группы, назначенные приложению Azure AD.

Прежде чем настроить и включить автоматическую подготовку пользователей, решите, какие пользователи и группы в Azure AD нуждаются в доступе к Elium. Затем назначаем этим пользователям и группам Elium, выполняя следующие действия в [присваивании пользователю или группе корпоративному приложению.](../manage-apps/assign-user-or-group-access-portal.md)

## <a name="important-tips-for-assigning-users-to-elium"></a>Важные советы по назначению пользователей Elium 

Мы рекомендуем назначить одного пользователя Azure AD Elium для тестирования автоматической конфигурации предоставления пользовательского обеспечения. Больше пользователей и групп могут быть назначены позже.

При назначении пользователя Elium необходимо выбрать действительную роль, специфичную для приложения (если такая есть имеется) в поле диалога назначения. Пользователи, имеющие роль **доступа по умолчанию,** исключены из подготовки.

## <a name="set-up-elium-for-provisioning"></a>Настройка Elium для подготовки

Прежде чем настроить Elium для автоматической подготовки пользователей с Azure AD, необходимо включить систему управления кросс-доменами identity Management (SCIM) на Elium. Выполните следующие действия.

1. Войдите в Elium и перейдите к**моим настройкам** **профиля.** > 

    ![Элемент меню настроек в Elium](media/Elium-provisioning-tutorial/setting.png)

1. В левом нижнем углу, под **ADVANCED,** выберите **безопасность.**

    ![Связь безопасности в Elium](media/Elium-provisioning-tutorial/security.png)

1. Копирование **URL-адреса арендатора** и значения **маркеров secret.** Эти значения будут использоваться позже в соответствующих полях в вкладке **«Подготовка»** приложения Elium на портале Azure.

    ![URL-адрес арендатора и секретные токеновы в Elium](media/Elium-provisioning-tutorial/token.png)

## <a name="add-elium-from-the-gallery"></a>Добавить Элиум из галереи

Чтобы настроить Elium для автоматической подготовки пользователей с помощью Azure AD, необходимо также добавить Elium из галереи приложений Azure AD в список управляемых приложений для программного обеспечения как услуги (SaaS). Выполните следующие действия.

1. На [портале Azure](https://portal.azure.com)на левой навигационной панели выберите **Active Directory Azure.**

    ![Элемент меню активного каталога Azure](common/select-azuread.png)

1. Перейдите к **корпоративным приложениям,** а затем выберите **все приложения.**

     ![Лезвие приложений Azure AD Enterprise](common/enterprise-applications.png)

1. Чтобы добавить новое приложение, выберите **новое приложение** в верхней части панели.

    ![Новая ссылка приложения](common/add-new-app.png)

1. В поле поиска, введите **Elium**, выберите **Elium** в списке результатов, а затем выберите **Добавить,** чтобы добавить приложение.

    ![Поле поиска галереи](common/search-new-app.png)

## <a name="configure-automatic-user-provisioning-to-elium"></a>Настройка автоматического предоставления пользователям Elium

В этом разделе вы проведет шаги по настройке службы предоставления AD Azure AD для создания, обновления и отработки пользователей и групп в Elium на основе пользовательских и групповых заданий в Azure AD.

> [!TIP]
> Вы также можете включить одиночный знак для Elium на основе языка безопасности Утверждение Markup Language (SAML), следуя инструкциям в [Elium один знак на учебник](Elium-tutorial.md). Вы можете настроить один знак независимо от автоматического подготовки пользователей, хотя эти две функции дополняют друг друга.

Для настройки автоматического пользовательского обеспечения Elium в Azure AD выполните следующие действия:

1. Вопийте на [портале Azure,](https://portal.azure.com)выберите **приложения Enterprise,** а затем выберите **все приложения.**

    ![Лезвие приложений Azure AD Enterprise](common/enterprise-applications.png)

1. В списке приложений выберите **Elium**.

    ![Список приложений в лезвии приложений предприятия](common/all-applications.png)

1. Выберите вкладку **«Подготовка».**

    ![Вкладка подготовки в лезвии приложений «Предприятие»](common/provisioning.png)

1. Установите **режим обеспечения** **автоматического.**

    ![Автоматическая настройка для режима обеспечения](common/provisioning-automatic.png)

1. В разделе **Admin Credentials** введите ** \<tenantURL\>/scim/v2** в поле **URL-адреса арендатора.** **(TenantURL** — это значение, извлеченное ранее из консоли admin Elium.) Также введите значение **маркера** Elium Secret в поле **Secret Token.** Наконец, выберите **тестовое соединение,** чтобы убедиться, что Azure AD может подключиться к Elium. Если соединение не удается, убедитесь, что ваша учетная запись Elium имеет разрешения на админ и повторите попытку.

    ![URL-адрес клиента и секретные поля токенов в учетных данных Админ](common/provisioning-testconnection-tenanturltoken.png)

1. В поле **уведомления email** введите адрес электронной почты человека или группы, которые получат уведомления об ошибке. Затем выберите **уведомление отправить сообщение по электронной почте, когда происходит сбой** флажка.

    ![Почтовое уведомление](common/provisioning-notification-email.png)

1. Нажмите **Сохранить**.

1. В разделе **Картпинги** выберите **синхронизацию активных пользователей каталога Azure в Elium.**

    ![Синхронизация ссылки для отображения пользователей Azure AD с Elium](media/Elium-provisioning-tutorial/usermapping.png)

1. Просмотрите атрибуты пользователя, синхронизированные с Azure AD на Elium в разделе **Атрибут Картпинги.** Атрибуты, выбранные в качестве свойств **соответствия,** используются для сопоставления учетных записей пользователей в Elium для операций обновления. Чтобы зафиксировать изменения, щелкните **Сохранить**.

    ![Отображение атрибутов между Azure AD и Elium](media/Elium-provisioning-tutorial/userattribute.png)

1. Для настройки фильтров для скопирования следуйте инструкциям в [учебнике scoping filter.](../app-provisioning/define-conditional-rules-for-provisioning-user-accounts.md)

1. Чтобы включить службу предоставления Azure AD для Elium, измените **статус подготовки** в разделе **«Настройки».** **On**

    ![Положение о предоставлении установлено на](common/provisioning-toggle-on.png)

1. Определите пользователей и группы, которые вы хотели бы предоставить Elium, выбрав значения, которые вы хотите в поле списка **выпадающих** скопов в разделе **Настройки.**

    ![Коробка списка области подготовки](common/provisioning-scope.png)

1. Когда вы будете готовы к предоставлению, выберите **Сохранить**.

    ![Сохранить кнопку для настройки подготовки](common/provisioning-configuration-save.png)

Эта операция запускает начальную синхронизацию всех пользователей и групп, определенных в **разделе Область** **в разделе Настройки.** Этот первоначальный процесс синхронизации занимает больше времени, чем более поздние синхронизации. Для получения дополнительной информации о времени, необходимом для подготовки, см., [сколько времени это займет, чтобы обеспечить пользователей?](../app-provisioning/application-provisioning-when-will-provisioning-finish-specific-user.md#how-long-will-it-take-to-provision-users).

Используйте раздел **Текущее состояние** для мониторинга прогресса и отслеживания ссылок на отчет о деятельности подготовки. Отчет о деятельности подготовки описывает все действия, выполняемые службой предоставления AD Azure AD на Elium. Для получения дополнительной [информации, см.](../app-provisioning/application-provisioning-when-will-provisioning-finish-specific-user.md) Чтобы прочитать журналы подготовки Azure AD, [см.](../app-provisioning/check-status-user-account-provisioning.md)

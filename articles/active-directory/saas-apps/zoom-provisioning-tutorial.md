# <a name="tutorial-configure-zoom-for-automatic-user-provisioning"></a>Руководство по Настройка масштаба для автоматической подготовки пользователей

Цель данного руководства является описаны шаги, которые необходимо выполнить в масштаб и Azure Active Directory (Azure AD), чтобы настроить Azure AD для автоматической подготовки и отмену подготовки пользователей и/или группы в Zoom.

> [!NOTE]
> В этом руководстве рассматривается соединитель, созданный на базе службы подготовки пользователей Azure AD. Подробные сведения о том, что делает эта служба, как она работает, и часто задаваемые вопросы см. в статье [Автоматическая подготовка пользователей и ее отзыв для приложений SaaS в Azure Active Directory](../manage-apps/user-provisioning.md).
>
> Сейчас этот соединитель предоставляется в общедоступной предварительной версии. Дополнительные сведения о общие Microsoft Azure условия использования предварительных версий функций, см. в разделе [дополнительным условиям использования предварительных версий Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

## <a name="prerequisites"></a>Технические условия

В сценарии, описанном в этом руководстве, предполагается, что у вас уже имеется:

* клиент Azure AD;
* [Клиент Zoom](https://zoom.us/pricing)
* Учетной записи пользователя в Zoom с разрешениями администратора.

## <a name="add-zoom-from-the-gallery"></a>Добавить Zoom из коллекции

Прежде чем настроить масштаб для автоматической подготовки пользователей с Azure AD, необходимо добавить Zoom из коллекции приложений Azure AD в список управляемых приложений SaaS.

**Чтобы добавить Zoom из коллекции приложений Azure AD, выполните следующие действия.**

1. В  **[портала Azure](https://portal.azure.com)** , на панели навигации слева выберите **Azure Active Directory**.

    ![Кнопка Azure Active Directory](common/select-azuread.png)

2. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.

    ![Колонка "Корпоративные приложения"](common/enterprise-applications.png)

3. Чтобы добавить новое приложение, выберите **новое приложение** кнопку в верхней части области.

    ![Кнопка "Создать приложение"](common/add-new-app.png)

4. В поле поиска введите **масштаб**выберите **масштаб** в панели результатов и нажмите кнопку **добавить** кнопку, чтобы добавить это приложение.

    ![Zoom в списке результатов](common/search-new-app.png)

## <a name="assign-users-to-zoom"></a>Назначить пользователей Zoom

Azure Active Directory используется концепция, называемая *назначения* для определения, какие пользователи должны получать доступ к выбранным приложениям. В контексте автоматической подготовки пользователей синхронизируются только те пользователи и группы, которые были назначены приложению в Azure AD.

Перед настройкой и включением автоматической подготовки пользователей, следует решить, какие пользователи или группы в Azure AD требуется доступ к Zoom. Сделав это, можно назначить этих пользователей и групп в Zoom, следуя инструкциям ниже:

* [Назначение корпоративному приложению пользователя или группы](../manage-apps/assign-user-or-group-access-portal.md)

### <a name="important-tips-for-assigning-users-to-zoom"></a>Важные рекомендации по назначению пользователей в Zoom

* Рекомендуется одного пользователя Azure AD назначается масштаб для тестирования конфигурации автоматической подготовки пользователей. Дополнительные пользователи и/или группы можно назначить позднее.

* При назначении пользователя в Zoom, вам необходимо выбрать действительную роль для конкретного приложения (если доступно) в диалоговом окне назначения. Пользователи с ролью **Доступ по умолчанию** исключаются из подготовки.

## <a name="configure-automatic-user-provisioning-to-zoom"></a>Настройка автоматической подготовки пользователей в Zoom 

В этом разделе поможет выполнить действия, чтобы настроить службу подготовки Azure AD для создания, изменения и отключения пользователей или групп в масштабирования на основе назначений в Azure AD.

> [!TIP]
> Также можно включить на основе SAML единого входа для Zoom, следуйте инструкциям, указанным на [масштаб входа в руководстве по настройке единого](zoom-tutorial.md). Единый вход можно настроить независимо от автоматической подготовки пользователей, хотя эти две возможности дополняют друг друга.

### <a name="configure-automatic-user-provisioning-for-zoom-in-azure-ad"></a>Настройка автоматической подготовки пользователей для масштабирования в Azure AD

1. Войдите на [портале Azure](https://portal.azure.com). Выберите **корпоративные приложения**, а затем выберите **все приложения**.

    ![Колонка "Корпоративные приложения"](common/enterprise-applications.png)

2. В списке приложений выберите **Zoom**.

    ![Ссылка на Zoom в списке приложений](common/all-applications.png)

3. Выберите вкладку **Подготовка**.

    ![Инициализация](common/provisioning.png)

4. Для параметра **Режим подготовки к работе** выберите значение **Automatic** (Автоматически).

    ![Инициализация](common/provisioning-automatic.png)

5. В разделе **учетные данные администратора** введите `https://api.zoom.us/scim` в **URL-адрес клиента**. Для получения **секретный токен** масштабирования вашей учетной записи, следуйте указаниям пошагового руководства, как описано на шаге 6.

6. Войдите в ваш [масштаб консоли администрирования](https://zoom.us/signin). Перейдите к **Дополнительно > Масштаб для разработчиков** в левой области навигации.

    ![Масштаб интеграции](media/zoom-provisioning-tutorial/zoom01.png)

    Перейдите к **управление** в правом верхнем углу страницы. 

    ![Установить масштаб](media/zoom-provisioning-tutorial/zoom02.png)

    Перейдите в созданный приложения Azure AD. 
    
    ![Масштаб приложения](media/zoom-provisioning-tutorial/zoom03.png)

    Выберите **учетных данных приложения** в левой области навигации.

    ![Масштаб приложения](media/zoom-provisioning-tutorial/zoom04.png)

    Получить значение маркера JWT, показано ниже и введите этот код в **секретный токен** в Azure AD. Если вам требуется новый маркер с неограниченным сроком, необходимо будет перенастроить истечения срока действия времени, которая будет автоматически создать новый токен. 

    ![Установить масштаб](media/zoom-provisioning-tutorial/zoom05.png)

7. После заполнения полей, указанных в шаге 5, щелкните **проверить подключение** и убедиться, что Azure AD может подключиться к Zoom. Если подключение отсутствует, убедитесь, что учетная запись масштаба имеет разрешения администратора и повторите попытку.

    ![по маркеру](common/provisioning-testconnection-tenanturltoken.png)

8. В поле **Почтовое уведомление** введите адрес электронной почты пользователя или группы, которые должны получать уведомления об ошибках подготовки, а также установите флажок **Send an email notification when a failure occurs** (Отправить уведомление по электронной почте при сбое).

    ![Уведомление по электронной почте](common/provisioning-notification-email.png)

9. Выберите команду **Сохранить**.

10. В разделе **сопоставления** выберите **синхронизировать пользователей Azure Active Directory с Zoom**.

    ![Масштаб сопоставления пользователей](media/zoom-provisioning-tutorial/zoom-user-mapping.png)

11. Просмотрите пользовательские атрибуты, которые синхронизированы из Azure AD в Zoom в **сопоставление атрибутов** раздел. Атрибуты, выбранные как **Matching** свойства используются для сопоставления учетных записей пользователей в Zoom для операций обновления. Нажмите кнопку **Сохранить**, чтобы зафиксировать все изменения.
    
     ![Масштаб сопоставления пользователей](media/zoom-provisioning-tutorial/zoom-user-attributes.png)

12. Чтобы настроить фильтры области, ознакомьтесь со следующими инструкциями, предоставленными в [руководстве по фильтрам области](../manage-apps/define-conditional-rules-for-provisioning-user-accounts.md).

13. Чтобы включить службу для Zoom подготовки Azure AD, измените **состояние подготовки** для **на** в **параметры** раздел.
    
    ![Состояние подготовки на противоположное](common/provisioning-toggle-on.png)

14. Определите пользователей или группы, которые вы хотите для подготовки в Zoom, выбрав нужные значения в **область** в **параметры** раздел.

    ![Инициализация области](common/provisioning-scope.png)

15. Когда будете готовы выполнить подготовку, нажмите кнопку **Сохранить**.

    ![Идет сохранение конфигурации подготовки](common/provisioning-configuration-save.png)

После этого начнется начальная синхронизация пользователей и (или) групп, определенных в поле **Область** раздела **Параметры**. Начальная синхронизация занимает больше времени, чем последующие операции синхронизации. Если служба запущена, они выполняются примерно каждые 40 минут. Можно использовать **сведения о синхронизации** раздел, чтобы отслеживать ход выполнения и переходить по ссылкам для просмотра отчетов, который описывает все действия, выполняемые с помощью Azure AD, служба при увеличении масштаба подготовки о подготовке.

Дополнительные сведения о чтении журналов подготовки Azure AD см. в руководстве по [отчетам об автоматической подготовке учетных записей](../manage-apps/check-status-user-account-provisioning.md).

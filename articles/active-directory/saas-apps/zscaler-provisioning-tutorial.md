# <a name="tutorial-configure-zscaler-for-automatic-user-provisioning"></a>Учебник: Настройка «Скразлер» для автоматического подготовки пользователей

Цель этого руководства состоит в том, чтобы продемонстрировать шаги, которые должны быть выполнены в Active Directory (Azure AD) для настройки Azure AD для автоматического предоставления и де-предоставления пользователям и/или группам в «Scaler».

> [!NOTE]
> В этом руководстве рассматривается соединитель, созданный на базе службы подготовки пользователей Azure AD. Подробные сведения о том, что делает эта служба, как она работает, и часто задаваемые вопросы см. в статье [Автоматическая подготовка пользователей и ее отзыв для приложений SaaS в Azure Active Directory](../active-directory-saas-app-provisioning.md).
>

## <a name="prerequisites"></a>Предварительные требования

Сценарий, описанный в этом руководстве, предполагает, что у вас уже имеется:

* Клиент Azure AD.
* Арендатор из Осклер.
* Учетная запись пользователя в Sscaler с разрешениями Админа.

> [!NOTE]
> Интеграция Azure AD основана на API SCIM, который доступен разработчикам sscaler для учетных записей с пакетом Enterprise.

## <a name="adding-zscaler-from-the-gallery"></a>Добавление Zscaler из коллекции

Перед настройкой sscaler для автоматического предоставления пользовательского запроса с помощью Azure AD необходимо добавить в список управляемых приложений SaaS sscaler из галереи приложений Azure AD.

**Чтобы добавить sscaler из галереи приложений Azure AD, выполните следующие действия:**

1. На **[портале Azure](https://portal.azure.com)** в области навигации слева щелкните значок **Azure Active Directory**.

    ![Кнопка Azure Active Directory](common/select-azuread.png)

2. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.

    ![Колонка "Корпоративные приложения"](common/enterprise-applications.png)

3. Чтобы добавить новое приложение, в верхней части диалогового окна нажмите кнопку **Создать приложение**.

    ![Кнопка "Создать приложение"](common/add-new-app.png)

4. В поле поиска введите **Zscaler**, выберите **Zscaler** на панели результатов и нажмите кнопку **Добавить**, чтобы добавить это приложение.

    ![Zscaler в списке результатов](common/search-new-app.png)

## <a name="assigning-users-to-zscaler"></a>Назначение пользователей на «Шкалер»

В Azure Active Directory для определения того, какие пользователи должны получать доступ к выбранным приложениям, используется концепция, называемая "назначение". В контексте автоматической подготовки синхронизированы будут только те пользователи и группы, которые были "назначены" приложению в Azure AD.

Прежде чем настроить и включить автоматическую подготовку пользователей, необходимо решить, какие пользователи и/или группы в Azure AD нуждаются в доступе к Sscaler. После того, как решение будет принято, вы можете назначить этих пользователей и/или групп ыскейлу, следуя инструкциям здесь:

* [Назначение корпоративному приложению пользователя или группы](https://docs.microsoft.com/azure/active-directory/active-directory-coreapps-assign-user-azure-portal)

### <a name="important-tips-for-assigning-users-to-zscaler"></a>Важные советы по назначению пользователей в «Масштаб»

* Для тестирования автоматической конфигурации подготовки пользователей рекомендуется назначить одного пользователя Azure AD. Дополнительные пользователи и/или группы можно назначить позднее.

* При назначении пользователя в журнал «Scaler» необходимо выбрать любую действительную роль приложения (если она имеется) в диалоге о назначении. Пользователи с ролью **доступа по умолчанию** исключены из подготовки.

## <a name="configuring-automatic-user-provisioning-to-zscaler"></a>Настройка автоматического обеспечения пользователя к «Масштабе»

В этом разделе вы проведет шаги по настройке службы предоставления AD Azure AD для создания, обновления и отработки пользователей и/или групп в группе в компании «Масштаб» на основе пользовательских и/или групповых заданий в Azure AD.

> [!TIP]
> Вы также можете включить SAML на основе одного знака для Sscaler, следуя инструкциям, представленным в [одном учебнике по регистрации.](zscaler-tutorial.md) Единый вход можно настроить независимо от автоматической подготовки пользователей, хотя эти две возможности дополняют друг друга.

### <a name="to-configure-automatic-user-provisioning-for-zscaler-in-azure-ad"></a>Для настройки автоматического подготовки пользователя для sscaler в Azure AD:

1. Вопийте на [портале Azure](https://portal.azure.com) и выберите **корпоративные приложения,** выберите **все приложения,** а затем выберите **sscaler.**

    ![Колонка "Корпоративные приложения"](common/enterprise-applications.png)

2. Из списка приложений выберите **Zscaler**.

    ![Ссылка на Zscaler в списке приложений](common/all-applications.png)

3. Выберите вкладку **«Подготовка».**

    ![Зскалер Подготовка](./media/zscaler-provisioning-tutorial/provisioning-tab.png)

4. Установите **режим обеспечения** **автоматического.**

    ![Зскалер Подготовка](./media/zscaler-provisioning-tutorial/provisioning-credentials.png)

5. В разделе **Admin Credentials** ввейди **URL-адрес арендатора** и **секретный маркер** вашей учетной записи , как описано в шаге 6.

6. Чтобы получить **URL-адрес арендатора** и **секретный токен,** перейдите в **администрация > настройки аутентификации** в пользовательском интерфейсе портала «Склер» и нажмите на **SAML** под **типом аутентификации.**

    ![Зскалер Подготовка](./media/zscaler-provisioning-tutorial/secret-token-1.png)

    Нажмите на **настройку SAML,** чтобы открыть параметры **конфигурации SAML.**

    ![Зскалер Подготовка](./media/zscaler-provisioning-tutorial/secret-token-2.png)

    Выберите **Включение SCIM-на основе обеспечения** для получения **базового URL** и **маркера bearer,** а затем сохранить настройки. Копируйте **URL-адрес базы** на **URL-адрес арендатора**и **токен Bearer** на **портале** Secret Token на портале Azure.

7. После заполнения полей, показанных в шаге 5, щелкните **тестовое соединение,** чтобы обеспечить подключение Azure AD к Sscaler. Если соединение не удается, убедитесь, что ваша учетная запись Sscaler имеет разрешения Admin и повторите попытку.

    ![Зскалер Подготовка](./media/zscaler-provisioning-tutorial/test-connection.png)

8. В поле **Почтовое уведомление** введите адрес электронной почты пользователя или группы, которые должны получать уведомления об ошибках подготовки, а также установите флажок **Send an email notification when a failure occurs** (Отправить уведомление по электронной почте при сбое).

    ![Зскалер Подготовка](./media/zscaler-provisioning-tutorial/notification.png)

9. Нажмите **Сохранить**.

10. В разделе **Картпинги** выберите **синхронизацию активных пользователей каталога Azure в разделе «Шкалер».**

    ![Зскалер Подготовка](./media/zscaler-provisioning-tutorial/user-mappings.png)

11. Просмотрите атрибуты пользователя, синхронизированные с Azure AD на «масштабатор» в разделе **«Отображение атрибутов».** Атрибуты, выбранные в качестве свойств **соответствия,** используются для сопоставления учетных записей пользователей в sscaler для операций обновления. Нажмите кнопку **Сохранить**, чтобы зафиксировать все изменения.

    ![Зскалер Подготовка](./media/zscaler-provisioning-tutorial/user-attribute-mappings.png)

12. В разделе **Картпинги** выберите **синхронизацию активных групп каталогов Azure в разделе «Шкалер».**

    ![Зскалер Подготовка](./media/zscaler-provisioning-tutorial/group-mappings.png)

13. Просмотрите атрибуты группы, синхронизированные с Azure AD с «масштабатором» в разделе **«Отображение атрибутов».** Атрибуты, выбранные в качестве свойств **соответствия,** используются для соответствия группам в sscaler для операций обновления. Нажмите кнопку **Сохранить**, чтобы зафиксировать все изменения.

    ![Зскалер Подготовка](./media/zscaler-provisioning-tutorial/group-attribute-mappings.png)

14. Чтобы настроить фильтры области, ознакомьтесь со следующими инструкциями, предоставленными в [руководстве по фильтрам области](./../active-directory-saas-scoping-filters.md).

15. Чтобы включить службу предоставления AD Azure AD для Sscaler, измените **статус подготовки** **в** разделе **«Настройки».**

    ![Зскалер Подготовка](./media/zscaler-provisioning-tutorial/provisioning-status.png)

16. Определите пользователей и/или группы, которые вы хотели бы предоставить «Масштабе», выбрав желаемые значения в **разделе Область** в разделе **Параметры.**

    ![Зскалер Подготовка](./media/zscaler-provisioning-tutorial/scoping.png)

17. Когда будете готовы выполнить подготовку, нажмите кнопку **Сохранить**.

    ![Зскалер Подготовка](./media/zscaler-provisioning-tutorial/save-provisioning.png)

После этого начнется начальная синхронизация пользователей и (или) групп, определенных в поле **Область** раздела **Параметры**. Начальная синхронизация занимает больше времени, чем последующие операции синхронизации. Если служба запущена, они выполняются примерно каждые 40 минут. Раздел **«Детали синхронизации»** можно использовать для мониторинга прогресса и отслеживания ссылок на отчет о деятельности, в котором описаны все действия, выполняемые службой предоставления AD Azure AD на sscaler.

Дополнительные сведения о чтении журналов подготовки Azure AD см. в руководстве по [отчетам об автоматической подготовке учетных записей](../active-directory-saas-provisioning-reporting.md).

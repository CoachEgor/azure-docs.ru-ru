# <a name="tutorial-configure-zendesk-for-automatic-user-provisioning"></a>Руководство по настройке Zendesk для автоматической подготовки пользователей

В этом уроке показаны шаги, которые необходимо выполнить в active-каталогах zendesk и Azure (Azure AD) для настройки Azure AD для автоматического предоставления и деобеспечения пользователей и групп в «Зендеск».

> [!NOTE]
> В этом уроке описывается разъем, построенный поверх службы предоставления пользовательского обеспечения Azure AD. Для получения информации о том, что эта служба делает, как она работает, и часто задаваемые вопросы, см [Автоматизировать пользовательские подготовки и деобеспечения программного обеспечения как службы (SaaS) приложений с Azure Active Directory](../app-provisioning/user-provisioning.md).

## <a name="prerequisites"></a>Предварительные требования

Сценарий, изложенный в этом учебнике предполагает, что у вас есть:

* Клиент Azure AD.
* Арендатор Зендекс с профессиональным планом или лучше включен.
* Учетная запись пользователя в Зендексе с разрешениями администратора.

## <a name="add-zendesk-from-the-azure-marketplace"></a>Добавить Зендеск с рынка Azure

Перед настройкой зендеска для автоматического предоставления пользовательской подготовки с помощью Azure AD добавьте в список управляемых приложений SaaS список управляемых приложений SaaS.

Чтобы добавить Зендеск с маркетплейса, выполните следующие действия.

1. На [портале Azure](https://portal.azure.com)в навигационном стеле слева выберите **Active Directory Azure.**

    ![Значок активного каталога Azure](common/select-azuread.png)

2. Перейдите к **корпоративным приложениям,** а затем выберите **все приложения.**

    ![Колонка "Корпоративные приложения"](common/enterprise-applications.png)

3. Чтобы добавить новое приложение, в верхней части диалогового окна выберите **Новое приложение**.

    ![Кнопка "Создать приложение"](common/add-new-app.png)

4. В поле поиска введите **Зендеск** и выберите **Зендеск** из панели результатов. Чтобы добавить приложение, **выберите Добавить**.

    ![Zendesk в списке результатов](common/search-new-app.png)

## <a name="assign-users-to-zendesk"></a>Присвоить пользователям зендеск

Активный каталог Azure использует концепцию, называемую *назначениями,* чтобы определить, какие пользователи должны получить доступ к выбранным приложениям. В контексте автоматической подготовки пользователей синхронизируются только пользователи или группы, назначенные приложению Azure AD.

Прежде чем настроить и включить автоматическую подготовку пользователей, решите, каким пользователям или группам в Azure AD нужен доступ к zendesk. Чтобы назначить этих пользователей или групп в «Зендеск», следуйте инструкциям в [присваивать пользователю или группе корпоративное приложение.](../manage-apps/assign-user-or-group-access-portal.md)

### <a name="important-tips-for-assigning-users-to-zendesk"></a>Важные советы по назначению пользователей в Zendesk

* Сегодня роли Зендеск автоматически и динамически заполняются на uI портала Azure. Перед тем, как назначить пользователям роли зендек, убедитесь, что начальная синхронизация завершена в отношении Зендеск, чтобы получить последние роли в вашем арендаторе зендек.

* Мы рекомендуем назначить одного пользователя Azure AD компании zendesk для тестирования начальной автоматической конфигурации подготовки пользователей. Вы можете назначить дополнительных пользователей или группы позже после успешного проведения тестов.

* При назначении пользователя в журнал zendesk выберите любую действительную роль, конкретную для приложения, если она имеется, в поле диалога назначения. Пользователи с ролью **доступа по умолчанию** исключены из подготовки.

## <a name="configure-automatic-user-provisioning-to-zendesk"></a>Настройка автоматического предоставления пользовательского запроса на Зендеск 

В этом разделе вы проведете шаги по настройке службы обеспечения AD Azure AD. Используйте его для создания, обновления и отработки пользователей или групп в компании zendesk на основе пользовательских или групповых заданий в Azure AD.

> [!TIP]
> Вы также можете включить SAML на основе одного знака для Зендеск. Следуйте инструкциям в [одиночном учебнике по регистрации .](zendesk-tutorial.md) Единый набор может быть настроен независимо от автоматического подготовки пользователей, хотя эти две функции дополняют друг друга.

### <a name="configure-automatic-user-provisioning-for-zendesk-in-azure-ad"></a>Настройка автоматического подготовки пользователя для Зендеск в Azure AD

1. Войдите на [портал Azure](https://portal.azure.com). Выберите **приложения** > Предприятия**Все приложения** > **Зендеск**.

    ![Колонка "Корпоративные приложения"](common/enterprise-applications.png)

2. В списке приложений выберите **Zendesk**.

    ![Ссылка на Зендеск в списке заявок](common/all-applications.png)

3. Выберите вкладку **«Подготовка».**

    ![Подготовка Zendesk](./media/zendesk-provisioning-tutorial/ZenDesk16.png)

4. Установите **режим обеспечения** **автоматического.**

    ![Режим подготовки Зендеск](./media/zendesk-provisioning-tutorial/ZenDesk1.png)

5. В разделе **Admin Credentials** ввейди имя пользователя администратора, секретный токен и домен вашей учетной записи zendesk. Примеры этих значений:

   * В поле **Admin Username** заполните имя пользователя учетной записи администратора на вашем арендаторе. Например, admin@contoso.com.

   * В коробке **Secret Token** заполните секретный маркер, как описано в Шаге 6.

   * В поле **домена** заполните поддомен вашего арендатора Зендеск. Например, для учетной записи `https://my-tenant.zendesk.com`с URL-адресом арендатора ваш поддомен является **моим арендатором.**

6. Секретный токен для вашей учетной записи zendesk находится в**настройках** > **API** >  **Admin.** Убедитесь, что **доступ к токену** настроен на **включенное.**

    ![Настройки администратора Зендек](./media/zendesk-provisioning-tutorial/ZenDesk4.png)

    ![Секретный токен Зендеск](./media/zendesk-provisioning-tutorial/ZenDesk2.png)

7. После заполнения коробок, показанных в шаге 5, выберите **тестовое соединение,** чтобы убедиться, что Azure AD может подключиться к зендеску. Если соединение не удается, убедитесь, что ваша учетная запись зендеск имеет разрешения администратора и повторите попытку.

    ![Соединение испытания Зендекс](./media/zendesk-provisioning-tutorial/ZenDesk19.png)

8. В поле **электронной почты уведомления** введите адрес электронной почты человека или группы для получения уведомлений об ошибке. Выберите **уведомление отправить сообщение по электронной почте, когда происходит сбой** флажка.

    ![Уведомление Зендеск По электронной почте](./media/zendesk-provisioning-tutorial/ZenDesk9.png)

9. Нажмите кнопку **Сохранить**.

10. В разделе **Сопоставления** выберите **Синхронизировать пользователей Azure Active Directory с Zendesk**.

    ![Синхронизация пользователей Зендеск](./media/zendesk-provisioning-tutorial/ZenDesk10.png)

11. Просмотрите атрибуты пользователя, синхронизированные с Azure AD с zendesk в разделе **Атрибут Картпинги.** Атрибуты, выбранные как свойства **Matching**, используются для сопоставления учетных записей пользователей Zendesk для операций обновления. Чтобы сохранить все изменения, выберите **Сохранить**.

    ![Соответствие атрибутов пользователя Зендеск](./media/zendesk-provisioning-tutorial/ZenDesk11.png)

12. В разделе **Картпинги** выберите **синхронизацию активных групп каталогов Azure в зендеск.**

    ![Синхронизация группы Зендеск](./media/zendesk-provisioning-tutorial/ZenDesk12.png)

13. Просмотрите атрибуты группы, синхронизированные с Azure AD с zendesk в разделе **Атрибут Картпинги.** Атрибуты, выбранные как свойства **сопоставления**, используются для сопоставления групп в Zendesk для операций обновления. Чтобы сохранить все изменения, выберите **Сохранить**.

    ![Соответствующие атрибуты группы Зендек](./media/zendesk-provisioning-tutorial/ZenDesk13.png)

14. Для настройки фильтров для обсажания следуйте инструкциям в [учебнике по отслеживанию фильтров.](../app-provisioning/define-conditional-rules-for-provisioning-user-accounts.md)

15. Для обеспечения сервиса предоставления Azure AD для zendesk в разделе **«Настройки»** измените **статус «Обеспечение»** на **On**.

    ![Статус обеспечения Зендеск](./media/zendesk-provisioning-tutorial/ZenDesk14.png)

16. Определите пользователей или группы, которые вы хотите предоставить в Зендеск. В разделе **Настройки** выберите значения, которые вы хотите в **области области.**

    ![Сфера охвата Зендеск](./media/zendesk-provisioning-tutorial/ZenDesk15.png)

17. Когда вы будете готовы к предоставлению, выберите **Сохранить**.

    ![Зендеск Сохранить](./media/zendesk-provisioning-tutorial/ZenDesk18.png)

Эта операция запускает начальную синхронизацию всех пользователей или групп, определенных в **разделе Область** в разделе **Настройки.** Начальная синхронизация занимает больше времени, чем более поздняя синхронизация. Они возникают примерно каждые 40 минут при запуске службы обеспечения AD Azure AD. 

Раздел **«Детали синхронизации»** можно использовать для мониторинга прогресса и отслеживания ссылок на отчет о деятельности подготовки. В отчете описаны все действия, выполняемые службой предоставления AD Azure AD на зендек.

Сведения о журналах подготовки Azure AD см. в руководстве [по отчетам об автоматической подготовке учетных записей пользователей](../app-provisioning/check-status-user-account-provisioning.md).

## <a name="connector-limitations"></a>Ограничения соединителя

* Зендеск поддерживает использование групп только для пользователей с ролями **агента.** Для получения более подробной информации, [см.](https://support.zendesk.com/hc/en-us/articles/203661966-Creating-managing-and-using-groups)

* Когда пользовательская роль назначается пользователю или группе, служба автоматического предоставления пользовательского обеспечения Azure AD также назначает **агенту**роли по умолчанию. Только агентам можно назначать пользовательскую роль. Для получения дополнительной информации ознакомьтесь с [документацией API-аИС зендекса](https://developer.zendesk.com/rest_api/docs/support/users#json-format-for-agent-or-admin-requests).

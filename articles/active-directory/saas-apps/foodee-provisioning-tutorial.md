# <a name="tutorial-configure-foodee-for-automatic-user-provisioning"></a>Учебник: Настройка Foodee для автоматического подготовки пользователей

В этой статье показано, как настроить Активный каталог Azure (Azure AD) в Foodee и Azure AD для автоматического предоставления или деобеспечения пользователей или групп Foodee.

> [!NOTE]
> В статье описывается разъем, построенный поверх службы обеспечения пользователей Azure AD. Чтобы узнать, что делает эта служба и как она работает, а также получить ответы на часто задаваемые вопросы, [см.](../app-provisioning/user-provisioning.md)
>
> Сейчас этот соединитель доступен в режиме предварительной версии. Для получения дополнительной информации о функции Azure терминов использования для просмотра функций, перейдите на [дополнительные условия использования для Microsoft Azure Previews.](https://azure.microsoft.com/support/legal/preview-supplemental-terms/)

## <a name="prerequisites"></a>Предварительные требования

Этот учебник предполагает, что вы выполнили следующие предпосылки:

* клиент Azure AD;
* [Арендатор foodee](https://www.food.ee/about/)
* Учетная запись пользователя в Foodee с разрешениями админа

## <a name="assign-users-to-foodee"></a>Назначить пользователей Foodee 

Azure AD использует концепцию, называемую *назначениями,* чтобы определить, какие пользователи должны получить доступ к выбранным приложениям. В контексте автоматической подготовки пользователей синхронизируются только пользователи или группы, назначенные приложению Azure AD.

Прежде чем настроить и включить автоматическую подготовку пользователей, необходимо решить, какие пользователи или группы в Azure AD нуждаются в доступе к Foodee. После этого определения вы можете назначить этих пользователей или групп Foodee, следуя инструкциям в [присваивать пользователю или группе корпоративное приложение.](../manage-apps/assign-user-or-group-access-portal.md)

## <a name="important-tips-for-assigning-users-to-foodee"></a>Важные советы по назначению пользователей Foodee 

При назначении пользователей имейте в виду следующие советы:

* Мы рекомендуем назначить Foodee только одного пользователя Azure AD для проверки конфигурации автоматической подготовки пользователей. Вы можете назначить дополнительных пользователей или групп ы позже.

* При назначении пользователя Foodee выберите любую действительную роль, конкретную для приложения, если она доступна, в панели **назначения.** Пользователи, имеющие роль *доступа по умолчанию,* исключены из подготовки.

## <a name="set-up-foodee-for-provisioning"></a>Настройка Foodee для подготовки

Прежде чем настроить Foodee для автоматической подготовки пользователей с помощью Azure AD, необходимо включить систему управления кросс-доменами identity Management (SCIM) в Foodee.

1. Вопийте в [Foodee,](https://www.food.ee/login/)а затем выберите идентификатор арендатора.

    ![Foodee](media/Foodee-provisioning-tutorial/tenant.png)

1. Под **порталом Предприятия**, выберите **Единый знак на**.

    ![Foodee Enterprise Portal левое меню](media/Foodee-provisioning-tutorial/scim.png)

1. Копирование значения в поле **API Token** для последующего использования. Вы введете его в поле **Secret Token** в вкладке **«Подготовка»** приложения Foodee на портале Azure.

    ![Foodee](media/Foodee-provisioning-tutorial/token.png)

## <a name="add-foodee-from-the-gallery"></a>Добавить Foodee из галереи

Для настройки Foodee для автоматической подготовки пользователей с помощью Azure AD необходимо добавить Foodee из галереи приложений Azure AD в список управляемых приложений SaaS.

Чтобы добавить Foodee из галереи приложений Azure AD, сделайте следующее:

1. На [портале Azure](https://portal.azure.com) в области слева щелкните **Azure Active Directory**.

    ![Команда активного каталога Azure](common/select-azuread.png)

1. Выберите **приложения** > Enterprise**Все приложения.**

    ![Область "Корпоративные приложения"](common/enterprise-applications.png)

1. Чтобы добавить новое приложение, выберите **новое приложение** в верхней части панели.

    ![Кнопка "Создать приложение"](common/add-new-app.png)

1. В поле поиска, введите **Foodee**, выберите **Foodee** в панели результатов, а затем выберите **Добавить,** чтобы добавить приложение.

    ![Foodee в списке результатов](common/search-new-app.png)

## <a name="configure-automatic-user-provisioning-to-foodee"></a>Настройка автоматического предоставления пользовательу Foodee 

В этом разделе вы настраиваете службу предоставления Azure AD для создания, обновления и отработки пользователей или групп в Foodee на основе пользовательских или групповых заданий в Azure AD.

> [!TIP]
> Вы также можете включить SAML основе одного знака для Foodee, следуя инструкциям в [Foodee одного знака на учебник](Foodee-tutorial.md). Вы можете настроить один знак независимо от автоматического подготовки пользователей, хотя эти две функции дополняют друг друга.

Нанастройка автоматического пользовательского обеспечения для Foodee в Azure AD, сделав следующее:

1. На [портале Azure](https://portal.azure.com)выберите **корпоративные приложения** > **Все приложения.**

    ![Область "Корпоративные приложения"](common/enterprise-applications.png)

1. В списке **приложений** выберите **Foodee**.

    ![Ссылка Foodee в списке приложений](common/all-applications.png)

1. Выберите вкладку **«Подготовка».**

    ![Вкладка подготовки](common/provisioning.png)

1. В **списке** отбрасывания режима подготовки выберите **Автоматический**.

    ![Вкладка подготовки](common/provisioning-automatic.png)

1. В соответствии с **полномочиями админа,** сделать следующее:

   а. В поле **URL-адреса Tenant** введите **\/https: /concierge.food.ee/scim/v2** значение, которое вы получили ранее.

   b. В поле **Secret Token** введите значение **API Token,** которое было получено ранее.
   
   c. Чтобы обеспечить подключение Azure AD к Foodee, выберите **Test Connection.** Если соединение выходит из строя, убедитесь, что ваша учетная запись Foodee имеет разрешения администратора, а затем повторите попытку.

    ![Ссылка на подключение к тесту](common/provisioning-testconnection-tenanturltoken.png)

1. В поле **электронной почты уведомления** введите адрес электронной почты человека или группы, которые должны получить уведомления об ошибке, а затем выберите **уведомление отправить сообщение по электронной почте, когда происходит сбой.**

    ![Текстовое окно по электронной почте](common/provisioning-notification-email.png)

1. Нажмите кнопку **Сохранить**.

1. Под **картографами**выберите **синхронизацию активных пользователей каталога Azure foodee.**

    ![Отображение пользователей Foodee](media/Foodee-provisioning-tutorial/usermapping.png)

1. В соответствии с **атрибутами Mappings**просмотрите атрибуты пользователя, синхронизированные с Azure AD с Foodee. Атрибуты, выбранные в качестве свойств **соответствия,** используются для сопоставления *учетных записей пользователей* в Foodee для операций обновления. 

    ![Отображение пользователей Foodee](media/Foodee-provisioning-tutorial/userattribute.png)

1. Чтобы зафиксировать изменения, выберите **Сохранить**.
1. Под **картографами**выберите **синхронизацию групп активных каталогов Azure для Foodee.**

    ![Отображение пользователей Foodee](media/Foodee-provisioning-tutorial/groupmapping.png)

1. В соответствии с **атрибутами Mappings**просмотрите атрибуты пользователя, синхронизированные с Azure AD с Foodee. Атрибуты, выбранные в качестве свойств **соответствия,** используются для *сопоставления учетных записей группы* в Foodee для операций обновления.

    ![Отображение пользователей Foodee](media/Foodee-provisioning-tutorial/groupattribute.png)

1. Чтобы зафиксировать изменения, выберите **Сохранить**.
1. Нанастройка фильтров для скопирования. Чтобы узнать, как, обратитесь к инструкциям в [scoping фильтр учебник](../app-provisioning/define-conditional-rules-for-provisioning-user-accounts.md).

1. Чтобы обеспечить предоставление услуг Azure AD для Foodee в разделе **«Настройки»,** измените **статус подготовки** к **On**.

    ![Переключатель статуса обеспечения](common/provisioning-toggle-on.png)

1. В **настройках**в списке drop-down **Scope** определите пользователей или группы, которые вы хотите предоставить Foodee.

    ![Список выпадающих скопов для подготовки](common/provisioning-scope.png)

1. Когда вы будете готовы к предоставлению, выберите **Сохранить**.

    ![Кнопка "Сохранение конфигурации настройки"](common/provisioning-configuration-save.png)

Предыдущая операция запускает начальную синхронизацию пользователей или групп, определенных в списке выпадающих **данных.** Начальная синхронизация занимает больше времени, чем последующие синхронизации. Для получения дополнительной информации, см. [Сколько времени это займет, чтобы обеспечить пользователей?](../app-provisioning/application-provisioning-when-will-provisioning-finish-specific-user.md#how-long-will-it-take-to-provision-users).

Вы можете использовать раздел **Текущее состояние** для мониторинга прогресса и отслеживания ссылок на отчет о деятельности. В отчете описаны все действия, выполняемые службой предоставления AD Azure aD на Foodee. Для получения дополнительной [информации, см.](../app-provisioning/application-provisioning-when-will-provisioning-finish-specific-user.md) Чтобы прочитать журналы подготовки Azure AD, [см.](../app-provisioning/check-status-user-account-provisioning.md)

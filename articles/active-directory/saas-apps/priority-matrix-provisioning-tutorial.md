# <a name="tutorial-configure-priority-matrix-for-automatic-user-provisioning"></a>Учебник: Настройка ПриоритетНая Матрица для автоматического подготовки пользователей

Цель этого руководства состоит в том, чтобы продемонстрировать шаги, которые будут выполнены в Priority Matrix и Azure Active Directory (Azure AD) для настройки Azure AD для автоматического предоставления и де-предоставления пользователям и/или группам в Priority Matrix.

> [!NOTE]
> В этом руководстве рассматривается соединитель, созданный на базе службы подготовки пользователей Azure AD. Подробные сведения о том, что делает эта служба, как она работает, и часто задаваемые вопросы см. в статье [Автоматическая подготовка пользователей и ее отзыв для приложений SaaS в Azure Active Directory](../app-provisioning/user-provisioning.md).
>
> Сейчас этот соединитель предоставляется в общедоступной предварительной версии. Дополнительные сведения об общих условиях использования продуктов в предварительной версии см. в документе [Дополнительные условия использования Предварительных версий Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

## <a name="prerequisites"></a>Предварительные требования

В сценарии, описанном в этом руководстве, предполагается, что у вас уже имеется:

* клиент Azure AD;
* [Арендатор Приоритетной матрицы](https://appfluence.com/pricing/)
* Учетная запись пользователя на матрице приоритета с разрешениями Админа.

## <a name="assign-users-to-priority-matrix"></a>Назначить пользователей В приоритетной матрице

Активный каталог Azure использует концепцию, называемую назначениями, чтобы определить, какие пользователи должны получить доступ к выбранным приложениям. В контексте автоматической подготовки пользователей синхронизируются только пользователи и/или группы, назначенные приложению azure AD.

Прежде чем настроить и включить автоматическую подготовку пользователей, необходимо решить, какие пользователи и/или группы в Azure AD нуждаются в доступе к Priority Matrix. После решения вы можете назначить этих пользователей и/или групп в Priority Matrix, следуя инструкциям здесь:

* [Назначение корпоративному приложению пользователя или группы](../manage-apps/assign-user-or-group-access-portal.md)

### <a name="important-tips-for-assigning-users-to-priority-matrix"></a>Важные советы по назначению пользователей в Priority Matrix

* Для тестирования автоматической конфигурации подготовки пользователей рекомендуется назначить одного пользователя Azure AD в Priority Matrix. Дополнительные пользователи и/или группы можно назначить позднее.

* При назначении пользователя в Priority Matrix необходимо выбрать любую действительную роль приложения (если она имеется) в диалоге заданий. Пользователи с ролью **доступа по умолчанию** исключены из подготовки.

## <a name="set-up-priority-matrix-for-provisioning"></a>Настройка Приоритетной Матрицы для подготовки

Перед настройкой Priority Matrix для автоматического предоставления пользовательской подготовки с Azure AD необходимо получить некоторую информацию о подготовке из Priority Matrix.

1. Вопийте на [консоль Приоритетной Матрицы Admin.](https://sync.appfluence.com/accounts/login/?next=/accounts/provisioning)

3. Нажмите **тоин входа Oauth** для приоритетной матрицы

    ![Приоритет Матрица Добавить SCIM](media/priority-matrix-provisioning-tutorial/oauthlogin.png)

4. Нажмите кнопку **GET NEW TOKEN.** Копирование **токен строки**. Это значение будет введено в поле **Secret Token** в вкладке «Обеспечение приоритетной матрицы» на портале Azure. 

    ![Приоритетная матрица Создать токен](media/priority-matrix-provisioning-tutorial/token.png)

## <a name="add-priority-matrix-from-the-gallery"></a>Добавление Матрицы Приоритета из галереи

Чтобы настроить Priority Matrix для автоматической подготовки пользователей с помощью Azure AD, необходимо добавить Матрицу Приоритетов из галереи приложений Azure AD в список управляемых приложений SaaS.

1. На **[портале Azure](https://portal.azure.com)** на левой навигационной панели выберите **Active Directory Azure.**

    ![Кнопка Azure Active Directory](common/select-azuread.png)

2. Перейдите к **корпоративным приложениям,** а затем выберите **все приложения.**

    ![Колонка "Корпоративные приложения"](common/enterprise-applications.png)

3. Чтобы добавить новое приложение, выберите кнопку **Нового приложения** в верхней части панели.

    ![Кнопка "Создать приложение"](common/add-new-app.png)

4. В поле поиска введите **Priority Matrix,** выберите **Матрицу Приоритета** в панели результатов. 

    ![Приоритетная матрица в списке результатов](common/search-new-app.png)

5. Выберите кнопку **«Регистрация на кнопку «Приоритетная матрица»,** которая перенаправит вас на страницу входа Priority Matrix. 

    ![Приоритет Матрица OIDC Добавить](media/priority-matrix-provisioning-tutorial/signup.png)

6. Поскольку Priority Matrix — это приложение OpenIDConnect, выберите вход в Priority Matrix с помощью рабочей учетной записи Майкрософт.

    ![Приоритет Наятрима входа oidC](media/priority-matrix-provisioning-tutorial/msftsignin.png)

7. После успешной проверки подлинности примите запрос согласия на страницу согласия. Приложение будет автоматически добавлено вашему арендатору, и вы будете перенаправлены на вашу учетную запись Priority Matrix.

    ![ПриоритетНая матрица OIDc Согласие](media/priority-matrix-provisioning-tutorial/consent.png)

## <a name="configure-automatic-user-provisioning-to-priority-matrix"></a>Настройка автоматического подготовки пользователя к Матрице Приоритета 

В этом разделе вы проведет шаги по настройке службы предоставления AD Azure AD для создания, обновления и отработки пользователей и/или групп в priority Matrix на основе пользовательских и/или групповых заданий в Azure AD.

> [!NOTE]
> Чтобы узнать больше о приоритетной матрице SCIM в endpoint, обратитесь к [подготовке пользователей и приоритет нойматрицы.](https://appfluence.com/help/article/user-provisioning/)

### <a name="to-configure-automatic-user-provisioning-for-priority-matrix-in-azure-ad"></a>Для настройки автоматической подготовки пользователя для приоритетной матрицы в Azure AD:

1. Войдите на [портал Azure](https://portal.azure.com). Выберите **корпоративные приложения,** затем выберите **все приложения.**

    ![Колонка "Корпоративные приложения"](common/enterprise-applications.png)

2. В списке приложений выберите **Priority Matrix.**

    ![Ссылка «Приоритетная матрица» в списке приложений](common/all-applications.png)

3. Выберите вкладку **«Подготовка».**

    ![Вкладка подготовки](common/provisioning.png)

4. Установите **режим обеспечения** **автоматического.**

    ![Вкладка подготовки](common/provisioning-automatic.png)

5. В разделе **Admin Credentials** `https://sync.appfluence.com/scim/v2/` ввод **URL-адреса клиента**. Ввейте значение, которое вы извлекли и сохранено ранее из priority Matrix в **Secret Token.** Нажмите **на тестовое соединение,** чтобы обеспечить подключение Azure AD к приоритетной матрице. Если соединение выходит из строя, убедитесь, что учетная запись Priority Matrix имеет разрешения Admin и повторите попытку.

    ![URL-адрес клиента + токен](common/provisioning-testconnection-tenanturltoken.png)

6. В поле **Почтовое уведомление** введите адрес электронной почты пользователя или группы, которые должны получать уведомления об ошибках подготовки, а также установите флажок **Send an email notification when a failure occurs** (Отправить уведомление по электронной почте при сбое).

    ![Почтовое уведомление](common/provisioning-notification-email.png)

7. Нажмите **Сохранить**.

8. В разделе **Картпинги** выберите **синхронизацию активных пользователей каталога Azure в разделе Приоритетная матрица.**

    ![Приоритетные Матрицы Пользовательские Карты](media/priority-matrix-provisioning-tutorial/usermappings.png)

9. Просмотрите атрибуты пользователя, синхронизированные с Azure AD с матрицой приоритетов в разделе **Атрибуты.** Атрибуты, выбранные в качестве свойств **соответствия,** используются для сопоставления учетных записей пользователей в Priority Matrix для операций обновления. Нажмите кнопку **Сохранить**, чтобы зафиксировать все изменения.

    ![Атрибуты пользователя priority Matrix](media/priority-matrix-provisioning-tutorial/userattributes.png)

10. Чтобы настроить фильтры области, ознакомьтесь со следующими инструкциями, предоставленными в [руководстве по фильтрам области](../app-provisioning/define-conditional-rules-for-provisioning-user-accounts.md).

11. Чтобы включить службу предоставления Azure AD для priority Matrix, измените **статус подготовки** в разделе **«Настройки».** **On**

    ![Состояние подготовки "Включено"](common/provisioning-toggle-on.png)

12. Определите пользователей и/или группы, которые вы хотели бы предоставить Ватрии Приоритета, выбрав желаемые значения в **разделе Область** в разделе **Параметры.**

    ![Область действия подготовки](common/provisioning-scope.png)

13. Когда будете готовы выполнить подготовку, нажмите кнопку **Сохранить**.

    ![Сохранение конфигурации подготовки](common/provisioning-configuration-save.png)

После этого начнется начальная синхронизация пользователей и (или) групп, определенных в поле **Область** раздела **Параметры**. Начальная синхронизация занимает больше времени, чем последующие операции синхронизации. Если служба запущена, они выполняются примерно каждые 40 минут. Раздел **«Детали синхронизации»** можно использовать для мониторинга прогресса и отслеживания ссылок на отчет о деятельности, в котором описаны все действия, выполняемые службой предоставления AD Azure AD на Priority Matrix.

Дополнительные сведения о чтении журналов подготовки Azure AD см. в руководстве по [отчетам об автоматической подготовке учетных записей](../app-provisioning/check-status-user-account-provisioning.md).

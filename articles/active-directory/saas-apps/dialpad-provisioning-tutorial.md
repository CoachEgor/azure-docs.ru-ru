# <a name="tutorial-configure-dialpad-for-automatic-user-provisioning"></a>Учебник: Настройка Dialpad для автоматического подготовки пользователей

Цель этого руководства состоит в том, чтобы продемонстрировать шаги, которые будут выполнены в Dialpad и Azure Active Directory (Azure AD) для настройки Azure AD для автоматического предоставления и де-предоставления пользователям и/или группам Dialpad.

> [!NOTE]
>  В этом руководстве рассматривается соединитель, созданный на базе службы подготовки пользователей Azure AD. Подробные сведения о том, что делает эта служба, как она работает, и часто задаваемые вопросы см. в статье [Автоматическая подготовка пользователей и ее отзыв для приложений SaaS в Azure Active Directory](../app-provisioning/user-provisioning.md).

> Сейчас этот соединитель доступен в режиме предварительной версии. Дополнительные сведения об общих условиях использования продуктов в предварительной версии см. в документе [Дополнительные условия использования Предварительных версий Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

## <a name="prerequisites"></a>Предварительные требования

В сценарии, описанном в этом руководстве, предполагается, что у вас уже имеется:

* Клиент Azure AD.
* [Арендатор Dialpad](https://www.dialpad.com/pricing/).
* Учетная запись пользователя в Dialpad с разрешениями Admin.

## <a name="assign-users-to-dialpad"></a>Назначить пользователей Dialpad
Активный каталог Azure использует концепцию, называемую назначениями, чтобы определить, какие пользователи должны получить доступ к выбранным приложениям. В контексте автоматической подготовки пользователей синхронизируются только пользователи и/или группы, назначенные приложению azure AD.

Прежде чем настроить и включить автоматическую подготовку пользователей, необходимо решить, какие пользователи и/или группы в Azure AD нуждаются в доступе к Dialpad. После того, как решение, вы можете назначить этих пользователей и / или групп Dialpad, следуя инструкциям здесь:
 
* [Назначение корпоративному приложению пользователя или группы](../manage-apps/assign-user-or-group-access-portal.md) 

 ## <a name="important-tips-for-assigning-users-to-dialpad"></a>Важные советы по назначению пользователей Dialpad

 * Для тестирования автоматической конфигурации подготовки пользователей рекомендуется назначить одного пользователя Azure AD Dialpad. Дополнительные пользователи и/или группы можно назначить позднее.

* При назначении пользователя Dialpad необходимо выбрать любую действительную роль приложения (если она имеется) в диалоге о назначении. Пользователи с ролью доступа по умолчанию исключены из подготовки.

## <a name="setup-dialpad-for-provisioning"></a>Настройка Dialpad для подготовки

Перед настройкой Dialpad для автоматического предоставления пользовательского запроса с помощью Azure AD необходимо получить некоторую информацию о подготовке к dialpad.

1. Вопийте на консоль [Dialpad Admin](https://dialpadbeta.com/login) и выберите **настройки Admin.** Убедитесь, что **моя компания** выбрана из выпадения. Перейдите к **аутентификации > API Keys.**

    ![Dialpad Добавить SCIM](media/dialpad-provisioning-tutorial/dialpad01.png)

2. Создайте новый ключ, нажав **Добавить ключ** и настройки свойств вашего секретного маркера.

    ![Dialpad Добавить SCIM](media/dialpad-provisioning-tutorial/dialpad02.png)

    ![Dialpad Добавить SCIM](media/dialpad-provisioning-tutorial/dialpad03.png)

3. Нажмите кнопку **«Нажмите», чтобы показать** значение ключа API и скопировать отображаемый значение. Это значение будет введено в поле **Secret Token** в вкладке «Подготовка» приложения Dialpad на портале Azure. 

    ![Dialpad Создать токен](media/dialpad-provisioning-tutorial/dialpad04.png)

## <a name="add-dialpad-from-the-gallery"></a>Добавить Dialpad из галереи

Для настройки Dialpad для автоматического предоставления пользовательского запроса с помощью Azure AD необходимо добавить Dialpad из галереи приложений Azure AD в список управляемых приложений SaaS.

**Чтобы добавить Dialpad из галереи приложений Azure AD, выполните следующие действия:**

1. На **[портале Azure](https://portal.azure.com)** на левой навигационной панели выберите **Active Directory Azure.**

    ![Кнопка Azure Active Directory](common/select-azuread.png)

2. Перейдите к **корпоративным приложениям,** а затем выберите **все приложения.**

    ![Колонка "Корпоративные приложения"](common/enterprise-applications.png)

3. Чтобы добавить новое приложение, выберите кнопку **Нового приложения** в верхней части панели.

    ![Кнопка "Создать приложение"](common/add-new-app.png)

4. В поле поиска введите **Dialpad**, выберите **Dialpad** в панели результатов.
    ![Dialpad в списке результатов](common/search-new-app.png)

5. Перейдите к **URL** выделены ниже в отдельном браузере. 

    ![Dialpad Добавить SCIM](media/dialpad-provisioning-tutorial/dialpad05.png)

6. В правом верхнем углу выберите **Log In > используйте Dialpad онлайн.**

    ![Dialpad Добавить SCIM](media/dialpad-provisioning-tutorial/dialpad06.png)

7. Поскольку Dialpad является приложением OpenIDConnect, выберите вход в Dialpad с помощью рабочей учетной записи Майкрософт.

    ![Dialpad Добавить SCIM](media/dialpad-provisioning-tutorial/loginpage.png)

8. После успешной проверки подлинности примите запрос согласия на страницу согласия. Приложение будет автоматически добавлено к вашему арендатору, и вы будете перенаправлены на ваш аккаунт Dialpad.

    ![Dialpad Добавить SCIM](media/dialpad-provisioning-tutorial/redirect.png)

 ## <a name="configure-automatic-user-provisioning-to-dialpad"></a>Настройка автоматического пользовательского обеспечения к Dialpad

В этом разделе вы проведет шаги по настройке службы предоставления AD Azure AD для создания, обновления и отработки пользователей и/или групп в Dialpad на основе пользовательских и/или групповых заданий в Azure AD.

### <a name="to-configure-automatic-user-provisioning-for-dialpad-in-azure-ad"></a>Для настройки автоматического пользовательского обеспечения Dialpad в Azure AD:

1. Войдите на [портал Azure](https://portal.azure.com). Выберите **корпоративные приложения,** затем выберите **все приложения.**

    ![Колонка "Корпоративные приложения"](common/enterprise-applications.png)

2. В списке приложений выберите **Dialpad**.

    ![Ссылка Найлпад в списке приложений](common/all-applications.png)

3. Выберите вкладку **«Подготовка».**

    ![Вкладка подготовки](common/provisioning.png)

4. Установите **режим обеспечения** **автоматического.**

    ![Вкладка подготовки](common/provisioning-automatic.png)

5. В разделе **Admin Credentials** `https://dialpad.com/scim` ввод **URL-адреса клиента**. Ввейте значение, которое вы извлекли и сохранили ранее из Dialpad в **Secret Token.** Нажмите **на тестовое соединение,** чтобы обеспечить подключение Azure AD к Dialpad. Если соединение не удается, убедитесь, что ваша учетная запись Dialpad имеет разрешения admin и повторите попытку.

    ![URL-адрес клиента + токен](common/provisioning-testconnection-tenanturltoken.png)

6. В поле **Почтовое уведомление** введите адрес электронной почты пользователя или группы, которые должны получать уведомления об ошибках подготовки, а также установите флажок **Send an email notification when a failure occurs** (Отправить уведомление по электронной почте при сбое).

    ![Почтовое уведомление](common/provisioning-notification-email.png)

7. Нажмите **Сохранить**.

8. В разделе **Картпинги** выберите **синхронизацию активных пользователей каталога Azure в Dialpad.**

    ![Картирование пользователей Dialpad](media/dialpad-provisioning-tutorial/dialpad-user-mappings-new.png)

9. Просмотрите атрибуты пользователя, синхронизированные с Azure AD с Dialpad в разделе **Атрибуты.** Атрибуты, выбранные в качестве свойств **соответствия,** используются для сопоставления учетных записей пользователей в Dialpad для операций обновления. Нажмите кнопку **Сохранить**, чтобы зафиксировать все изменения.

    ![Атрибуты пользователя Dialpad](media/dialpad-provisioning-tutorial/dialpad07.png)

10. Чтобы настроить фильтры области, ознакомьтесь со следующими инструкциями, предоставленными в [руководстве по фильтрам области](../app-provisioning/define-conditional-rules-for-provisioning-user-accounts.md).

11. Чтобы включить службу предоставления Azure AD для Dialpad, измените **статус подготовки** **в** разделе **«Настройки».**

    ![Состояние подготовки "Включено"](common/provisioning-toggle-on.png)

12. Определите пользователей и/или группы, которые вы хотели бы предоставить Dialpad, выбрав желаемые значения в **разделе Область** в разделе **Настройки.**

    ![Область действия подготовки](common/provisioning-scope.png)

13. Когда будете готовы выполнить подготовку, нажмите кнопку **Сохранить**.

    ![Сохранение конфигурации подготовки](common/provisioning-configuration-save.png)

После этого начнется начальная синхронизация пользователей и (или) групп, определенных в поле **Область** раздела **Параметры**. Начальная синхронизация занимает больше времени, чем последующие операции синхронизации. Если служба запущена, они выполняются примерно каждые 40 минут. Раздел **«Детали синхронизации»** можно использовать для мониторинга прогресса и отслеживания ссылок на отчет о деятельности, в котором описаны все действия, выполняемые службой предоставления AD Azure AD на Dialpad.

Для получения дополнительной информации о том, как читать журналы подготовки Azure AD, [см.](../app-provisioning/check-status-user-account-provisioning.md)
##  <a name="connector-limitations"></a>Ограничения соединителя
* Dialpad не поддерживает группу переименований сегодня. Это означает, что любые изменения в **displayName** группы в Azure AD не будут обновляться и отражаться в Dialpad.

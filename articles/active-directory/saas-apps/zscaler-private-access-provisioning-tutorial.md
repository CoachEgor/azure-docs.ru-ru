# <a name="tutorial-configure-zscaler-private-access-zpa-for-automatic-user-provisioning"></a>Учебник: Наконфигурируйте частный доступ (Зсклер Частный доступ) для автоматического подготовки пользователей

Цель этого руководства состоит в том, чтобы продемонстрировать шаги, которые должны быть выполнены в «Масштабном частном доступе» (ЗАП) и Active Directory Azure (Azure AD) для настройки Azure AD для автоматического предоставления и де-предоставления пользователям и/или группам для частного доступа (ЗСКЛЕР).

> [!NOTE]
> В этом руководстве рассматривается соединитель, созданный на базе службы подготовки пользователей Azure AD. Подробные сведения о том, что делает эта служба, как она работает, и часто задаваемые вопросы см. в статье [Автоматическая подготовка пользователей и ее отзыв для приложений SaaS в Azure Active Directory](../app-provisioning/user-provisioning.md).
>
> Сейчас этот соединитель предоставляется в общедоступной предварительной версии. Дополнительные сведения об общих условиях использования продуктов в предварительной версии см. в документе [Дополнительные условия использования Предварительных версий Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

## <a name="prerequisites"></a>Предварительные требования

В сценарии, описанном в этом руководстве, предполагается, что у вас уже имеется:

* клиент Azure AD;
* [Арендатор частного доступа «Скшкал»](https://www.zscaler.com/pricing-and-plans#contact-us)
* Учетная запись пользователя в частном доступе (ЗСклер) с разрешениями админа.

## <a name="assigning-users-to-zscaler-private-access-zpa"></a>Назначение пользователей на «Кшкал» Частный доступ (ЗПА)

Активный каталог Azure использует концепцию, называемую *назначениями,* чтобы определить, какие пользователи должны получить доступ к выбранным приложениям. В контексте автоматической подготовки пользователей синхронизируются только пользователи и/или группы, назначенные приложению azure AD.

Прежде чем настроить и включить автоматическую подготовку пользователей, необходимо решить, какие пользователи и/или группы в Azure AD нуждаются в доступе к частному доступу к sscaler Private Access. После того, как решение будет принято, вы можете назначить этих пользователей и/или групп ысклерным частным доступом , следуя инструкциям здесь:
* [Назначение корпоративному приложению пользователя или группы](../manage-apps/assign-user-or-group-access-portal.md)

## <a name="important-tips-for-assigning-users-to-zscaler-private-access-zpa"></a>Важные советы по назначению пользователей в «Кшкал» Частный доступ (ЗПА)

* Для тестирования автоматической конфигурации подготовки пользователей рекомендуется назначить одного пользователя Azure AD для тестирования конфигурации автоматического обеспечения пользователя. Дополнительные пользователи и/или группы можно назначить позднее.

* При назначении пользователя частному доступу (ЗСклер Частный доступ) необходимо выбрать любую действительную роль приложения (если она имеется) в диалоге о назначении. Пользователи с ролью **доступа по умолчанию** исключены из подготовки.

## <a name="set-up-zscaler-private-access-zpa-for-provisioning"></a>Настройка частного доступа «Сккалер» (ЗПА) для подготовки

1. Вопийте на вашу [админ-консоль «Скшкалр Частный доступ» (ЗПА)](https://admin.private.zscaler.com/). Перейдите к **конфигурации администрирования > IdP.**

    ![Админ-консоль «Сккер» (Заклер Приватный Доступ)](media/zscaler-private-access-provisioning-tutorial/idpconfig.png)

2.  Проверить, чтобы убедиться, что IdP для **одного входного** настроен. Если idP не настроен, затем добавьте один, нажав значок плюс в правом верхнем углу экрана.

    ![«Сккер Частный доступ» (ЗПА) Добавить SCIM](media/zscaler-private-access-provisioning-tutorial/plusicon.png)

3. Следуйте через мастер **конфигурации AddId,** чтобы добавить IdP. Оставьте для **пользователя** **единое поле для записи.** Предоставьте **имя** и выберите **домены** из списка drop down. Нажмите на **далее,** чтобы перейти к следующему окну.

    ![«Скшкал» Частный доступ (ЗПА) Добавить IdP](media/zscaler-private-access-provisioning-tutorial/addidp.png)

4. Скачать **сертификат поставщика услуг**. Нажмите на **далее,** чтобы перейти к следующему окну.

    ![Сертификат СП «Сккер»](media/zscaler-private-access-provisioning-tutorial/spcertificate.png)

5. В следующем окне загрузите **сертификат поставщика услуг,** загруженный ранее.

    ![Сертификат загрузки сертификата «Сккер Частный доступ»](media/zscaler-private-access-provisioning-tutorial/uploadfile.png)

6.  Прокрутите вниз, чтобы предоставить **единый идентификатор url-адресов и** **id IdP Entity.**

    ![Идентификатор IdP id](media/zscaler-private-access-provisioning-tutorial/idpid.png)

7.  Прокрутите вниз, чтобы **включить синхронизацию SCIM.** Нажмите на кнопку **Generate New Token.** Копировать **токен bearer**. Это значение будет введено в поле Secret Token в вкладке «Обеспечение частного доступа» на портале Azure.

    ![«Скшкал частный доступ» (ЗПА) Создать токен](media/zscaler-private-access-provisioning-tutorial/token.png)

8.  Чтобы найти **URL-адрес арендатора,** перейдите на **конфигурацию администрирования > IdP.** Нажмите на имя недавно добавленной конфигурации IdP, указанной на странице.

    ![«Сккер Частный доступ» (Запп) Имя Идп](media/zscaler-private-access-provisioning-tutorial/idpname.png)

9.  Прокрутите вниз, чтобы просмотреть **конечную точку поставщика услуг SCIM** в конце страницы. Копирование **конечная точка поставщика услуг SCIM**. Это значение будет введено в поле URL-адреса арендатора в вкладке «Обеспечение частного доступа» на портале Azure.

    ![URL-адрес SCIM (КККЕР Частный доступ)](media/zscaler-private-access-provisioning-tutorial/tenanturl.png)


## <a name="add-zscaler-private-access-zpa-from-the-gallery"></a>Добавьте из галереи частный доступ к компании «Сккер» (ЗСклер)

Перед настройкой частного доступа (ЗСклер) для автоматического предоставления пользователям с помощью Azure AD необходимо добавить из галереи приложений Azure AD список управляемых приложений SaaS.

**Чтобы добавить частный доступ к компании «Скразлер» из галереи приложений Azure AD, выполните следующие действия:**

1. На **[портале Azure](https://portal.azure.com)** на левой навигационной панели выберите **Active Directory Azure.**

    ![Кнопка Azure Active Directory](common/select-azuread.png)

2. Перейдите к **корпоративным приложениям,** а затем выберите **все приложения.**

    ![Колонка "Корпоративные приложения"](common/enterprise-applications.png)

3. Чтобы добавить новое приложение, выберите кнопку **Нового приложения** в верхней части панели.

    ![Кнопка "Создать приложение"](common/add-new-app.png)

4. В поле поиска введите **кнопку «Частный доступ» (Зсклер Частный доступ)**, выберите **«Sscaler Private Access»** в панели результатов, а затем нажмите кнопку **«Добавить»,** чтобы добавить приложение.

    ![Zscaler Private Access (ZPA) в списке результатов](common/search-new-app.png)

## <a name="configuring-automatic-user-provisioning-to-zscaler-private-access-zpa"></a>Настройка автоматической подготовки пользователя к частному доступу «Scaler» (ЗПА) 

В этом разделе вы проведет шаги по настройке службы предоставления AD Azure AD для создания, обновления и отработки пользователей и/или групп в разделе «Частный доступ» (ЗПА) на основе пользовательских и/или групповых заданий в Azure AD.

> [!TIP]
> Вы также можете включить SAML на основе одного знака для частного доступа (ЗСклер [Zscaler Private Access (ZPA) Single sign-on tutorial](https://docs.microsoft.com/azure/active-directory/saas-apps/zscalerprivateaccess-tutorial)Частный доступ) ( Единый набор может быть настроен независимо от автоматического подготовки пользователей, хотя эти две функции дополняют друг друга.

> [!NOTE]
> Чтобы узнать больше о сигойой SCIM компании Sscaler Private Access, обратитесь к [этому.](https://www.zscaler.com/partners/microsoft)

### <a name="to-configure-automatic-user-provisioning-for-zscaler-private-access-zpa-in-azure-ad"></a>Для настройки автоматической подготовки пользователей для «Масштабного частного доступа» в Azure AD:

1. Войдите на [портал Azure](https://portal.azure.com). Выберите **корпоративные приложения,** затем выберите **все приложения.**

    ![Колонка "Корпоративные приложения"](common/enterprise-applications.png)

2. Из списка приложений выберите **Zscaler Private Access (ZPA)**.

    ![Ссылка на Zscaler Private Access (ZPA) в списке "Приложения"](common/all-applications.png)

3. Выберите вкладку **«Подготовка».**

    ![Вкладка подготовки](common/provisioning.png)

4. Установите **режим обеспечения** **автоматического.**

    ![Вкладка подготовки](common/provisioning-automatic.png)

5. В разделе **"Учетные данные админ-веричек"** ввешай значение **конечных точек поставщика услуг SCIM,** извлеченное ранее в **URL-адресе Tenant.** Ввейте значение **token Bearer,** полученное ранее в **Secret Token.** Нажмите **на тестовое соединение,** чтобы обеспечить подключение Azure AD к частному доступу (ЗСКЛЕР). Если подключение не удается, убедитесь, что ваша учетная запись «Частный доступ» имеет разрешения для админов и повторите попытку.

    ![URL-адрес клиента + токен](common/provisioning-testconnection-tenanturltoken.png)

6. В поле **Почтовое уведомление** введите адрес электронной почты пользователя или группы, которые должны получать уведомления об ошибках подготовки, а также установите флажок **Send an email notification when a failure occurs** (Отправить уведомление по электронной почте при сбое).

    ![Почтовое уведомление](common/provisioning-notification-email.png)

7. Нажмите **Сохранить**.

8. В разделе **Картпинги** выберите **синхронизацию активных пользователей каталога Azure в разделе «Частный доступ» (ЗСКЛЕР).**

    ![Картирование пользователей (КСПА)](media/zscaler-private-access-provisioning-tutorial/usermappings.png)

9. Просмотрите атрибуты пользователя, синхронизированные с Azure AD с разделом «Частный доступ» (ЗПА) в разделе **«Отображение атрибутов».** Атрибуты, выбранные в качестве свойств **соответствия,** используются для сопоставления учетных записей пользователей в «Масштабном частном доступе» для операций обновления. Нажмите кнопку **Сохранить**, чтобы зафиксировать все изменения.

    ![Атрибуты пользователя «Scaler Private Access»](media/zscaler-private-access-provisioning-tutorial/userattributes.png)

10. В разделе **Картпинги** выберите **синхронизацию групп активных каталогов Azure для «Масштабного частного доступа».**

    ![Картирование группы «Сккармер»](media/zscaler-private-access-provisioning-tutorial/groupmappings.png)

11. Просмотрите атрибуты группы, синхронизированные с Azure AD с разделом «Частный доступ» (ЗПА) в разделе **«Отображение атрибутов».** Атрибуты, выбранные в качестве свойств **соответствия,** используются для соответствия группам в «Масштабном частном доступе» для операций обновления. Нажмите кнопку **Сохранить**, чтобы зафиксировать все изменения.

    ![Групповые атрибуты «Сккер»](media/zscaler-private-access-provisioning-tutorial/groupattributes.png)

12. Чтобы настроить фильтры области, ознакомьтесь со следующими инструкциями, предоставленными в [руководстве по фильтрам области](../app-provisioning/define-conditional-rules-for-provisioning-user-accounts.md).

13. Для обеспечения сервиса предоставления Azure AD для частного доступа **Provisioning Status** в формат **On** «Scaler» ( **Settings**

    ![Состояние подготовки "Включено"](common/provisioning-toggle-on.png)

14. Определите пользователей и/или группы, которые вы хотели бы предоставить «Частному доступу» ( зПА), выбрав желаемые значения в **разделе Область** в разделе **«Настройки».**

    ![Область действия подготовки](common/provisioning-scope.png)

15. Когда будете готовы выполнить подготовку, нажмите кнопку **Сохранить**.

    ![Сохранение конфигурации подготовки](common/provisioning-configuration-save.png)

После этого начнется начальная синхронизация пользователей и (или) групп, определенных в поле **Область** раздела **Параметры**. Начальная синхронизация занимает больше времени, чем последующие операции синхронизации. Если служба запущена, они выполняются примерно каждые 40 минут. Раздел **«Детали синхронизации»** можно использовать для мониторинга прогресса и отслеживания ссылок на отчет о деятельности, в котором описаны все действия, выполняемые службой предоставления AD Azure AD на «Масштабном частном доступе».

Дополнительные сведения о чтении журналов подготовки Azure AD см. в руководстве по [отчетам об автоматической подготовке учетных записей](../app-provisioning/check-status-user-account-provisioning.md).

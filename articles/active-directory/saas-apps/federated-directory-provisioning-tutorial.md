# <a name="tutorial-configure-federated-directory-for-automatic-user-provisioning"></a>Учебник: Настройка Федеративного Каталога для автоматического подготовки пользователей

Цель этого руководства состоит в том, чтобы продемонстрировать шаги, которые будут выполнены в Федеративном каталоге и активном каталоге Azure (Azure AD) для настройки Azure AD для автоматического предоставления и де-предоставления пользователям и/или группам В Federated Directory.

> [!NOTE]
>  В этом руководстве рассматривается соединитель, созданный на базе службы подготовки пользователей Azure AD. Подробные сведения о том, что делает эта служба, как она работает, и часто задаваемые вопросы см. в статье [Автоматическая подготовка пользователей и ее отзыв для приложений SaaS в Azure Active Directory](../app-provisioning/user-provisioning.md).
>
> Сейчас этот соединитель предоставляется в общедоступной предварительной версии. Дополнительные сведения об общих условиях использования продуктов в предварительной версии см. в документе [Дополнительные условия использования Предварительных версий Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

## <a name="prerequisites"></a>Предварительные требования

В сценарии, описанном в этом руководстве, предполагается, что у вас уже имеется:

* Клиент Azure AD.
* [Федеративный каталог](https://www.federated.directory/pricing).
* Учетная запись пользователя в Federated Directory с разрешениями Admin.

## <a name="assign-users-to-federated-directory"></a>Назначить пользователей в Федеративный каталог
Активный каталог Azure использует концепцию, называемую назначениями, чтобы определить, какие пользователи должны получить доступ к выбранным приложениям. В контексте автоматической подготовки пользователей синхронизируются только пользователи и/или группы, назначенные приложению azure AD.

Прежде чем настроить и включить автоматическую подготовку пользователей, необходимо решить, какие пользователи и/или группы в Azure AD нуждаются в доступе к Федерированию. После решения вы можете назначить этих пользователей и/или групп в Federated Directory, следуя инструкциям здесь:

 * [Назначение корпоративному приложению пользователя или группы](../manage-apps/assign-user-or-group-access-portal.md) 
 
 ## <a name="important-tips-for-assigning-users-to-federated-directory"></a>Важные советы по назначению пользователей в Federated Directory
 * Для тестирования автоматической конфигурации подготовки пользователей рекомендуется назначить одного пользователя Azure AD в Federated Directory. Дополнительные пользователи и/или группы можно назначить позднее.

* При назначении пользователя в Federated Directory необходимо выбрать любую действительную роль приложения (если она имеется) в диалоге о назначении. Пользователи с ролью доступа по умолчанию исключены из подготовки.
    
 ## <a name="set-up-federated-directory-for-provisioning"></a>Настройка Федеративного каталога для подготовки

Прежде чем настроить Federated Directory для автоматического предоставления пользовательских данных с Azure AD, необходимо включить sciM-подготовку по Федерированию.

1. Войти на [консоль для адденов-дидеированных директоров Federated](https://federated.directory/of)

    ![Учебник по федеративному каталогу](media/federated-directory-provisioning-tutorial/companyname.png)

2. Перейдите в **каталоги > пользовательские каталоги** и выберите арендатора. 

    ![федеративный каталог](media/federated-directory-provisioning-tutorial/ad-user-directories.png)

3.  Чтобы создать токен постоянного носителя, перейдите на **клавиши каталога > создать новый ключ.** 

    ![федеративный каталог](media/federated-directory-provisioning-tutorial/federated01.png)

4. Создайте ключ каталога. 

    ![федеративный каталог](media/federated-directory-provisioning-tutorial/federated02.png)
    

5. Скопируйте значение параметра **Маркер доступа**. Это значение будет введено в поле **Secret Token** в вкладке «Обеспечение» приложения Federated Directory на портале Azure. 

    ![федеративный каталог](media/federated-directory-provisioning-tutorial/federated03.png)
    
## <a name="add-federated-directory-from-the-gallery"></a>Добавить Федеративный каталог из галереи

Чтобы настроить Federated Directory для автоматического предоставления пользовательских данных с помощью Azure AD, необходимо добавить Federated Directory из галереи приложений Azure AD в список управляемых приложений SaaS.

**Чтобы добавить Federated Directory из галереи приложений Azure AD, выполните следующие действия:**

1. На **[портале Azure](https://portal.azure.com)** на левой навигационной панели выберите **Active Directory Azure.**

    ![Кнопка Azure Active Directory](common/select-azuread.png)

2. Перейдите к **корпоративным приложениям,** а затем выберите **все приложения.**

    ![Колонка "Корпоративные приложения"](common/enterprise-applications.png)

3. Чтобы добавить новое приложение, выберите кнопку **Нового приложения** в верхней части панели.

    ![Кнопка "Создать приложение"](common/add-new-app.png)

4. В поле поиска введите **Федеративный каталог**, выберите **Федеративный каталог** в панели результатов.

    ![Федеративный каталог в списке результатов](common/search-new-app.png)

5. Перейдите к **URL** выделены ниже в отдельном браузере. 

    ![федеративный каталог](media/federated-directory-provisioning-tutorial/loginpage1.png)

6. Нажмите **войти в систему**.

    ![федеративный каталог](media/federated-directory-provisioning-tutorial/federated04.png)

7.  Поскольку Federated Directory — это приложение OpenIDConnect, выберите вход в Federated Directory с помощью рабочей учетной записи Майкрософт.
    
    ![федеративный каталог](media/federated-directory-provisioning-tutorial/loginpage3.png)
 
8. После успешной проверки подлинности примите запрос согласия на страницу согласия. Приложение будет автоматически добавлено вашему арендатору, и вы будете перенаправлены на ваш счет Federated Directory.

    ![федеративный каталог Добавить SCIM](media/federated-directory-provisioning-tutorial/premission.png)



## <a name="configuring-automatic-user-provisioning-to-federated-directory"></a>Настройка автоматического обеспечения пользователя к Федеративному каталогу 

В этом разделе вы проведет шаги по настройке службы предоставления AD Azure AD для создания, обновления и отработки пользователей и/или групп в Федеративном каталоге на основе пользовательских и/или групповых заданий в Azure AD.

### <a name="to-configure-automatic-user-provisioning-for-federated-directory-in-azure-ad"></a>Для настройки автоматического подготовки пользователей для Федеративного каталога в Azure AD:

1. Войдите на [портал Azure](https://portal.azure.com). Выберите **корпоративные приложения,** затем выберите **все приложения.**

    ![Колонка "Корпоративные приложения"](common/enterprise-applications.png)

2. В списке заявок выберите **Федеративный каталог.**

    ![Ссылка На Федеративную справочник в списке приложений](common/all-applications.png)

3. Выберите вкладку **«Подготовка».**

    ![Вкладка подготовки](common/provisioning.png)

4. Установите **режим обеспечения** **автоматического.**

    ![Вкладка подготовки](common/provisioning-automatic.png)

5. В разделе **Admin Credentials** `https://api.federated.directory/v2/` ввод url-адреса клиента. Ввейте значение, которое вы извлекли и сохранили ранее из Federated Directory в **Secret Token.** Нажмите **на тестовое соединение,** чтобы обеспечить подключение Azure AD к Федеративному каталогу. Если соединение не удается, убедитесь, что ваша учетная запись Federated Directory имеет разрешения Admin и повторите попытку.

    ![URL-адрес клиента + токен](common/provisioning-testconnection-tenanturltoken.png)

8. В поле **Почтовое уведомление** введите адрес электронной почты пользователя или группы, которые должны получать уведомления об ошибках подготовки, а также установите флажок **Send an email notification when a failure occurs** (Отправить уведомление по электронной почте при сбое).

    ![Почтовое уведомление](common/provisioning-notification-email.png)

9. Нажмите **Сохранить**.

10. В разделе **Картпинги** выберите **синхронизацию активных пользователей каталога Azure в Федеративном каталоге.**

    ![Учебник по федеративному каталогу](media/federated-directory-provisioning-tutorial/user-mappings.png)
    
    
11. Просмотрите атрибуты пользователя, синхронизированные с Azure AD с Федеративным каталогом в разделе **Атрибуткарт.** Атрибуты, выбранные в качестве свойств **соответствия,** используются для сопоставления учетных записей пользователей в Федеративном каталоге для операций обновления. Нажмите кнопку **Сохранить**, чтобы зафиксировать все изменения.

    ![Учебник по федеративному каталогу](media/federated-directory-provisioning-tutorial/user-attributes.png)
    

12. Чтобы настроить фильтры области, ознакомьтесь со следующими инструкциями, предоставленными в [руководстве по фильтрам области](../app-provisioning/define-conditional-rules-for-provisioning-user-accounts.md).

13. Чтобы включить службу предоставления услуг Azure AD для Федеративного каталога, измените **статус подготовки** в разделе **«В** **настройках».**

    ![Состояние подготовки "Включено"](common/provisioning-toggle-on.png)

14. Определите пользователей и/или группы, которые вы хотели бы предоставить в Federated Directory, выбрав желаемые значения в **разделе Область** в разделе **Параметры.**

    ![Область действия подготовки](common/provisioning-scope.png)

15. Когда будете готовы выполнить подготовку, нажмите кнопку **Сохранить**.

    ![Сохранение конфигурации подготовки](common/provisioning-configuration-save.png)

После этого начнется начальная синхронизация пользователей и (или) групп, определенных в поле **Область** раздела **Параметры**. Начальная синхронизация занимает больше времени, чем последующие операции синхронизации. Если служба запущена, они выполняются примерно каждые 40 минут. Раздел **«Детали синхронизации»** можно использовать для мониторинга прогресса и отслеживания ссылок на отчет о деятельности, в котором описаны все действия, выполняемые службой предоставления AD Azure AD в Федеративном каталоге.

Для получения дополнительной информации о том, как читать журналы подготовки Azure AD, [см.](../app-provisioning/check-status-user-account-provisioning.md)

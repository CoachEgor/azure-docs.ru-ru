# <a name="tutorial-configure-sap-cloud-platform-identity-authentication-for-automatic-user-provisioning"></a>Учебник: Налаживание подлинности идентификации облачной платформы SAP для автоматического подготовки пользователей

Цель этого руководства состоит в том, чтобы продемонстрировать шаги, которые должны быть выполнены в SAP Облачная платформа аутентификации подлинности и Azure Active Directory (Azure AD) для настройки Azure AD для автоматического предоставления и де-обеспечения пользователей и / или групп для SAP Cloud Platform Identity Authentication.

> [!NOTE]
> В этом руководстве рассматривается соединитель, созданный на базе службы подготовки пользователей Azure AD. Подробные сведения о том, что делает эта служба, как она работает, и часто задаваемые вопросы см. в статье [Автоматическая подготовка пользователей и ее отзыв для приложений SaaS в Azure Active Directory](../app-provisioning/user-provisioning.md).
>
> Сейчас этот соединитель предоставляется в общедоступной предварительной версии. Дополнительные сведения об общих условиях использования продуктов в предварительной версии см. в документе [Дополнительные условия использования Предварительных версий Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

## <a name="prerequisites"></a>Предварительные требования

В сценарии, описанном в этом руководстве, предполагается, что у вас уже имеется:

* клиент Azure AD;
* [Арендатор аутентификации подлинности облачной платформы SAP](https://cloudplatform.sap.com/pricing.html)
* Учетная запись пользователя в идентификационном запросе облачной платформы SAP с разрешениями админа.

## <a name="assigning-users-to-sap-cloud-platform-identity-authentication"></a>Назначение пользователей на проверку подлинности идентификации облачной платформы SAP

Активный каталог Azure использует концепцию, называемую *назначениями,* чтобы определить, какие пользователи должны получить доступ к выбранным приложениям. В контексте автоматической подготовки пользователей синхронизируются только пользователи и/или группы, назначенные приложению azure AD.

Прежде чем настроить и включить автоматическую подготовку пользователей, необходимо решить, каким пользователям и/или группам в Azure AD необходим доступ к аутентификации идентификации облачной платформы SAP. После решения вы можете назначить этих пользователей и/или групп в систему проверки подлинности подлинности облачной платформы SAP, следуя инструкциям здесь:
* [Назначение корпоративному приложению пользователя или группы](../manage-apps/assign-user-or-group-access-portal.md)

## <a name="important-tips-for-assigning-users-to-sap-cloud-platform-identity-authentication"></a>Важные советы по назначению пользователей в систему проверки подлинности идентификации облачной платформы SAP

* Для проверки автоматической конфигурации обеспечения пользователей рекомендуется назначить одного пользователя Azure AD для проверки автоматической конфигурации обеспечения пользователей. Дополнительные пользователи и/или группы можно назначить позднее.

* При назначении пользователя подлинной аутентификации идентификации облачная платформа SAP необходимо выбрать любую действительную роль приложения (если она доступна) в диалоге о назначении. Пользователи с ролью **доступа по умолчанию** исключены из подготовки.

## <a name="setup-sap-cloud-platform-identity-authentication-for-provisioning"></a>Настройка sAP Облачная платформа аутентификация личности для подготовки

1. Вопийте на консоль [admin-аутентификации подлинности облачной платформы SAP.](https://sapmsftintegration.accounts.ondemand.com/admin) Перейдите к **пользователям & авторизаций > администраторов**.

    ![SAP Облачная платформа аутентификации админ консоль](media/sap-cloud-platform-identity-authentication-provisioning-tutorial/adminconsole.png)

2.  Создайте пользователя админ и выберите пользователя.  

3.  При настройке авторизаций включите кнопку переключения против **пользователей управления** и **групп управления.**

    ![Проверка подлинности идентификации облачной платформы SAP добавить SCIM](media/sap-cloud-platform-identity-authentication-provisioning-tutorial/configurationauth.png)

4. Вы получите электронное письмо, чтобы активировать свою учетную запись и установить пароль для **службы аутентификации подлинности облачной платформы SAP.**

4.  Копирование **идентификатора пользователя** и **пароля**. Эти значения будут вводиться в поля имени пользователя и пароля Admin соответственно в вкладке «Обеспечение подлинности идентификации облачной платформы SAP» на портале Azure.

## <a name="add-sap-cloud-platform-identity-authentication-from-the-gallery"></a>Добавление приложения SAP Cloud Platform Identity Authentication из коллекции

Прежде чем настроить проверку подлинности подлинности облачной платформы SAP для автоматической подготовки пользователей с помощью Azure AD, необходимо добавить аутентификацию подлинности подлинности облачной платформы SAP из галереи приложений Azure AD в список управляемых приложений SaaS.

**Чтобы добавить проверку подлинности подлинности облачной платформы SAP из галереи приложений Azure AD, выполните следующие действия:**

1. На **[портале Azure](https://portal.azure.com)** на левой навигационной панели выберите **Active Directory Azure.**

    ![Кнопка Azure Active Directory](common/select-azuread.png)

2. Перейдите к **корпоративным приложениям,** а затем выберите **все приложения.**

    ![Колонка "Корпоративные приложения"](common/enterprise-applications.png)

3. Чтобы добавить новое приложение, выберите кнопку **Нового приложения** в верхней части панели.

    ![Кнопка "Создать приложение"](common/add-new-app.png)

4. В поле поиска введите **проверку подлинности идентификации облачной платформы SAP,** выберите **проверку подлинности подлинности облачной платформы SAP** в панели результатов, а затем нажмите кнопку **«Добавить»,** чтобы добавить приложение.

    ![Приложения SAP Cloud Platform Identity Authentication в списке результатов](common/search-new-app.png)

## <a name="configuring-automatic-user-provisioning-to-sap-cloud-platform-identity-authentication"></a>Настройка автоматической подготовки пользователя к аутентификации идентификации облачной платформы SAP 

В этом разделе вы проведет шаги по настройке службы предоставления AD Azure AD для создания, обновления и отработки пользователей и/или групп в sAP Cloud Platform Identity Authentication на основе пользовательских и/или групповых заданий в Azure AD.

> [!TIP]
> Вы также можете включить SAML на основе одного знака для SAP Cloud Platform Identity Authentication , следуя инструкциям, представленным в [SAP Cloud Platform Identityation Единый подсобный знак.](https://docs.microsoft.com/azure/active-directory/saas-apps/sap-hana-cloud-platform-identity-authentication-tutorial) Единый подгон может быть настроен независимо от автоматического подготовки пользователей, хотя эти две функции дополняют друг друга

### <a name="to-configure-automatic-user-provisioning-for-sap-cloud-platform-identity-authentication-in-azure-ad"></a>Для настройки автоматической подготовки пользователей для проверки подлинности идентификации облачной платформы SAP в Azure AD:

1. Войдите на [портал Azure](https://portal.azure.com). Выберите **корпоративные приложения,** затем выберите **все приложения.**

    ![Колонка "Корпоративные приложения"](common/enterprise-applications.png)

2. В списке приложений выберите **SAP Cloud Platform Identity Authentication**.

    ![Ссылка на SAP Cloud Platform Identity Authentication в списке приложений](common/all-applications.png)

3. Выберите вкладку **«Подготовка».**

    ![Вкладка подготовки](common/provisioning.png)

4. Установите **режим обеспечения** **автоматического.**

    ![Вкладка подготовки](common/provisioning-automatic.png)

5. В разделе **Admin Credentials** `https://<tenantID>.accounts.ondemand.com/service/scim ` ввод **URL-адреса клиента**. Ввешай значения **идентификатора пользователя** и **пароля,** полученные ранее в **Admin Username** и **Admin Password** соответственно. Нажмите **на тестовое соединение,** чтобы обеспечить подключение Azure AD к аутентификации подлинности облачной платформы SAP. Если подключение не удается, убедитесь, что ваша учетная запись подлинности идентификации обкома SAP Cloud Platform имеет разрешения для админовцев и повторите попытку.

    ![URL-адрес клиента + токен](media/sap-cloud-platform-identity-authentication-provisioning-tutorial/testconnection.png)

6. В поле **Почтовое уведомление** введите адрес электронной почты пользователя или группы, которые должны получать уведомления об ошибках подготовки, а также установите флажок **Send an email notification when a failure occurs** (Отправить уведомление по электронной почте при сбое).

    ![Почтовое уведомление](common/provisioning-notification-email.png)

7. Нажмите **Сохранить**.

8. В разделе **Картпинги** выберите **синхронизацию активных пользователей каталога Azure для проверки подлинности идентификации облачной платформы SAP.**

    ![SAP Облачная платформа Аутентификация пользователь Картирование](media/sap-cloud-platform-identity-authentication-provisioning-tutorial/mapping.png)

9. Просмотрите атрибуты пользователя, синхронизированные с Azure AD с проверкой подлинности идентификации облачной платформы SAP в разделе **«Отображение атрибутов».** Атрибуты, выбранные в качестве свойств **соответствия,** используются для сопоставления учетных записей пользователей в аутентификации идентификации облачной платформы SAP для операций обновления. Нажмите кнопку **Сохранить**, чтобы зафиксировать все изменения.

    ![Атрибуты проверки подлинности подлинности облачной платформы](media/sap-cloud-platform-identity-authentication-provisioning-tutorial/userattributes.png)

10. Чтобы настроить фильтры области, ознакомьтесь со следующими инструкциями, предоставленными в [руководстве по фильтрам области](../app-provisioning/define-conditional-rules-for-provisioning-user-accounts.md).

11. Чтобы включить службу обеспечения azure AD для проверки подлинности идентификации облачной платформы SAP, измените **статус подготовки** в разделе **«В** **настройках».**

    ![Состояние подготовки "Включено"](common/provisioning-toggle-on.png)

12. Определите пользователей и/или группы, которые вы хотели бы предоставить для проверки подлинности идентификации облачной платформы SAP, выбрав желаемые значения в **разделе Область** в разделе **Параметры.**

    ![Область действия подготовки](common/provisioning-scope.png)

13. Когда будете готовы выполнить подготовку, нажмите кнопку **Сохранить**.

    ![Сохранение конфигурации подготовки](common/provisioning-configuration-save.png)

После этого начнется начальная синхронизация пользователей и (или) групп, определенных в поле **Область** раздела **Параметры**. Начальная синхронизация занимает больше времени, чем последующие операции синхронизации. Если служба запущена, они выполняются примерно каждые 40 минут. Раздел **«Детали синхронизации»** можно использовать для мониторинга прогресса и отслеживания ссылок на отчет о деятельности, в котором описаны все действия, выполняемые службой предоставления AD Azure AD на подлинности идентификации подлинности облачной платформы SAP.

Дополнительные сведения о чтении журналов подготовки Azure AD см. в руководстве по [отчетам об автоматической подготовке учетных записей](../app-provisioning/check-status-user-account-provisioning.md).

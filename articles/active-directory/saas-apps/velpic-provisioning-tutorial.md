# <a name="tutorial-configuring-velpic-for-automatic-user-provisioning"></a>Учебник: Настройка Velpic для автоматического обеспечения пользователя

Цель этого руководства состоит в том, чтобы показать вам шаги, необходимые для выполнения в Velpic и Azure AD, чтобы автоматически предоставлять и де-предоставлять учетные записи пользователей от Azure AD до Velpic.

> [!NOTE]
> В этом руководстве рассматривается соединитель, созданный на базе службы подготовки пользователей Azure AD. Подробные сведения о том, что делает эта служба, как она работает, и часто задаваемые вопросы см. в статье [Автоматическая подготовка пользователей и ее отзыв для приложений SaaS в Azure Active Directory](../app-provisioning/user-provisioning.md).

## <a name="prerequisites"></a>Предварительные требования

Сценарий, описанный в этом учебнике, предполагает, что у вас уже имеется:

* Клиент Azure Active Directory.
* Вельпиковый арендатор с [планом Enterprise](https://www.velpic.com/pricing.html) или лучше включенный
* Учетная запись пользователя в Velpic с разрешениями Админа

## <a name="assigning-users-to-velpic"></a>Назначение пользователей Velpic

В Azure Active Directory для определения того, какие пользователи должны получать доступ к выбранным приложениям, используется концепция, называемая "назначение". В контексте автоматической подготовки учетной записи синхронизированы будут только те пользователи и группы, которые были "назначены" приложению в Azure AD. 

Прежде чем настроить и включить службу обеспечения, необходимо решить, какие пользователи и/или группы в Azure AD представляют пользователей, которым нужен доступ к вашему приложению Velpic. После того, как решение, вы можете назначить этих пользователей в вашем приложении Velpic, следуя инструкциям здесь:

[Назначение корпоративному приложению пользователя или группы](../manage-apps/assign-user-or-group-access-portal.md)

### <a name="important-tips-for-assigning-users-to-velpic"></a>Важные советы по назначению пользователей Velpic

* Для тестирования конфигурации подготовки рекомендуется назначить одного пользователя Azure AD Velpic. Дополнительные пользователи и/или группы можно назначить позднее.

* При назначении пользователя Velpic необходимо выбрать либо роль **пользователя,** либо другую действительную роль приложения (если она имеется) в диалоге о назначении. Обратите внимание, что роль **доступа по умолчанию** не работает для подготовки, и эти пользователи будут пропущены.

## <a name="configuring-user-provisioning-to-velpic"></a>Настройка пользовательского подготовки к Velpic

В этом разделе вам будет подключено подключение Azure AD к учетной записи пользователя Velpic, предоставляющее API, и настройка службы обеспечения для создания, обновления и отключения назначенных учетных записей пользователей в Velpic на основе назначения пользователя и группы в Azure AD.

> [!TIP]
> Вы также можете выбрать включенный SAML-однозначный знак для Velpic, следуя инструкциям, предоставленным на [портале Azure](https://portal.azure.com). Единый вход можно настроить независимо от автоматической подготовки, хотя эти две возможности дополняют друг друга.

### <a name="to-configure-automatic-user-account-provisioning-to-velpic-in-azure-ad"></a>Для настройки автоматической подготовки учетной записи пользователя к Velpic в Azure AD:

1. На [портале Azure](https://portal.azure.com) перейдите в раздел **Azure Active Directory > Корпоративные приложения > Все приложения**.

2. Если вы уже настроили Velpic для одного входного, ищите экземпляр Velpic с помощью поля поиска. В противном случае выберите **Добавить** и искать **Velpic** в галерее приложений. Выберите Velpic из результатов поиска и добавьте его в список приложений.

3. Выберите экземпляр Velpic, а затем выберите вкладку **«Подготовка».**

4. Установите **режим обеспечения** **автоматического.**

    ![Вельпич-Подготовка](./media/velpic-provisioning-tutorial/Velpic1.png)

5. В разделе **Admin Credentials** ввейди **URL-адрес арендатора&Secret Token** of Velpic. (Вы можете найти эти значения под вашей учетной записи Velpic: **Управление** > **интеграцией** > **Plugin** > **SCIM**)

    ![Значения авторизации](./media/velpic-provisioning-tutorial/Velpic2.png)

6. На портале Azure щелкните **тестовое соединение,** чтобы обеспечить подключение Azure AD к приложению Velpic. Если соединение не удается, убедитесь, что ваша учетная запись Velpic имеет разрешения admin и попробуйте шаг 5 снова.

7. В поле **Почтовое уведомление** введите адрес электронной почты пользователя или группы, которые должны получать уведомления об ошибках подготовки, а также установите флажок ниже.

8. Нажмите **Сохранить**.

9. В разделе Картпинги выберите **синхронизацию активных пользователей каталога Azure в Velpic.**

10. В разделе **Атрибуты Картирования** просмотрите атрибуты пользователя, которые будут синхронизированы с Azure AD на Velpic. Обратите внимание, что атрибуты, выбранные в качестве свойств **соответствия,** будут использоваться для сопоставления учетных записей пользователей в Velpic для операций обновления. Нажмите кнопку "Сохранить", чтобы подтвердить все изменения.

11. Чтобы обеспечить обслуживание Azure AD для Velpic, измените **статус подготовки** в разделе **«Настройки»** **On**

12. Нажмите **Сохранить**.

Это позволит начать начальную синхронизацию любых пользователей и/или групп, назначенных Velpic в разделе «Пользователи и группы». Обратите внимание, что начальная синхронизация будет занимать больше времени, чем последующие операции синхронизации. Если служба запущена, они выполняются примерно каждые 40 минут. В разделе **Сведения о синхронизации** можно отслеживать ход выполнения и переходить по ссылкам для просмотра отчетов о подготовке, в которых зафиксированы все действия, выполняемые службой подготовки.

Дополнительные сведения о чтении журналов подготовки Azure AD см. в руководстве по [отчетам об автоматической подготовке учетных записей](../app-provisioning/check-status-user-account-provisioning.md).

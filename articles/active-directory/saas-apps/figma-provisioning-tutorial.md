# <a name="tutorial-configure-figma-for-automatic-user-provisioning"></a>Учебник: Налаживайте Figma для автоматического подготовки пользователей

Цель этого руководства – продемонстрировать шаги, которые будут выполняться в Figma и Azure Active Directory (Azure AD) для настройки Azure AD для автоматического предоставления и де-предоставления пользователям и/или группам ВФигму.

> [!NOTE]
> В этом руководстве рассматривается соединитель, созданный на базе службы подготовки пользователей Azure AD. Подробные сведения о том, что делает эта служба, как она работает, и часто задаваемые вопросы см. в статье [Автоматическая подготовка пользователей и ее отзыв для приложений SaaS в Azure Active Directory](../app-provisioning/user-provisioning.md).
>
> Сейчас этот соединитель предоставляется в общедоступной предварительной версии. Дополнительные сведения об общих условиях использования продуктов в предварительной версии см. в документе [Дополнительные условия использования Предварительных версий Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

## <a name="prerequisites"></a>Предварительные требования

В сценарии, описанном в этом руководстве, предполагается, что у вас уже имеется:

* Клиент Azure AD.
* [Арендатор Figma](https://www.figma.com/pricing/).
* Учетная запись пользователя в Figma с разрешениями Admin.

## <a name="assign-users-to-figma"></a>Назначаем пользователей Figma.
Активный каталог Azure использует концепцию, называемую назначениями, чтобы определить, какие пользователи должны получить доступ к выбранным приложениям. В контексте автоматической подготовки пользователей синхронизируются только пользователи и/или группы, назначенные приложению azure AD.

Прежде чем настроить и включить автоматическую подготовку пользователей, необходимо решить, какие пользователи и/или группы в Azure AD нуждаются в доступе к Figma. После того, как решение, вы можете назначить этих пользователей и / или групп Figma, следуя инструкциям здесь:
 
* [Назначение корпоративному приложению пользователя или группы](../manage-apps/assign-user-or-group-access-portal.md)
## <a name="important-tips-for-assigning-users-to-figma"></a>Важные советы по назначению пользователей Figma

 * Для тестирования автоматической конфигурации подготовки пользователей figma рекомендуется назначить одного пользователя Azure AD. Дополнительные пользователи и/или группы можно назначить позднее.

* При назначении пользователя Figma необходимо выбрать любую действительную роль приложения (если она имеется) в диалоге о назначении. Пользователи с ролью доступа по умолчанию исключены из подготовки.

## <a name="set-up-figma-for-provisioning"></a>Настройка Figma для подготовки

Перед настройкой Figma для автоматического предоставления пользовательской подготовки с Azure AD необходимо получить некоторую информацию о подготовке от Figma.

1. Войти на [консоль Figma Admin.](https://www.Figma.com/) Нажмите на значок передачряды рядом с арендатором.

    ![FigmaFigma-сотрудник-обеспечение](media/Figma-provisioning-tutorial/image0.png)

2. Перейдите к **общим > обновления входа в настройки**.

    ![FigmaFigma-сотрудник-обеспечение](media/Figma-provisioning-tutorial/figma03.png)

3. Копирование **идентификатора арендатора**. Это значение будет использоваться для построения URL-адреса конечной точки SCIM, который будет введен в поле **URL-адреса арендатора** в вкладке «Обеспечение вашего приложения Figma» на портале Azure.

    ![Фигма Создать токен](media/Figma-provisioning-tutorial/figma-tenantid.png)

4. Прокрутите вниз и нажмите на **токен Generate API.**

    ![Фигма Создать токен](media/Figma-provisioning-tutorial/token.png)

5. Копирование значения **токенов API.** Это значение будет введено в поле **Secret Token** в вкладке «Подготовка» приложения Figma на портале Azure. 

    ![Фигма Создать токен](media/Figma-provisioning-tutorial/figma04.png)

## <a name="add-figma-from-the-gallery"></a>Добавить Фигму из галереи

Чтобы настроить Figma для автоматической подготовки пользователей с помощью Azure AD, необходимо добавить Figma из галереи приложений Azure AD в список управляемых приложений SaaS.

1. На **[портале Azure](https://portal.azure.com)** на левой навигационной панели выберите **Active Directory Azure.**

    ![Кнопка Azure Active Directory](common/select-azuread.png)

2. Перейдите к **корпоративным приложениям,** а затем выберите **все приложения.**

    ![Колонка "Корпоративные приложения"](common/enterprise-applications.png)

3. Чтобы добавить новое приложение, выберите кнопку **Нового приложения** в верхней части панели.

    ![Кнопка "Создать приложение"](common/add-new-app.png)

4. В поле поиска введите **Figma**, выберите **Figma** в панели результатов, а затем нажмите кнопку **Добавить,** чтобы добавить приложение.

    ![Фигма в списке результатов](common/search-new-app.png)

## <a name="configuring-automatic-user-provisioning-to-figma"></a>Настройка автоматического предоставления пользователям Figma 

В этом разделе вы проведет шаги по настройке службы предоставления AD Azure AD для создания, обновления и отработки пользователей и/или групп в Figma на основе пользовательских и/или групповых заданий в Azure AD.

> [!TIP]
> Вы также можете включить SAML основе одного знака для Figma , следуя инструкциям, представленным в [Figma Единый знак на учебник](figma-tutorial.md). Единый вход можно настроить независимо от автоматической подготовки пользователей, хотя эти две возможности дополняют друг друга.

### <a name="to-configure-automatic-user-provisioning-for-figma--in-azure-ad"></a>Для настройки автоматического подготовки пользователей для Figma в Azure AD:

1. Войдите на [портал Azure](https://portal.azure.com). Выберите **корпоративные приложения,** затем выберите **все приложения.**

    ![Колонка "Корпоративные приложения"](common/enterprise-applications.png)

2. Из списка приложений выберите **Figma**.

    ![Ссылка Figma в списке приложений](common/all-applications.png)

3. Выберите вкладку **«Подготовка».**

    ![Вкладка подготовки](common/provisioning.png)

4. Установите **режим обеспечения** **автоматического.**

    ![Вкладка подготовки](common/provisioning-automatic.png)

5. В разделе **Admin Credentials** `https://www.figma.com/scim/v2/<TenantID>` вввод в **URL-адрес Tenant,** где **TenantID** является значением, которое вы извлекли из Figma ранее. Ввейте значение **токенов API** в **Secret Token.** Нажмите **на тестовое соединение,** чтобы обеспечить подключение Azure AD к Figma. Если соединение не удается, убедитесь, что ваша учетная запись Figma имеет разрешения Admin и повторите попытку.

    ![URL-адрес клиента + токен](common/provisioning-testconnection-tenanturltoken.png)

8. В поле **Почтовое уведомление** введите адрес электронной почты пользователя или группы, которые должны получать уведомления об ошибках подготовки, а также установите флажок **Send an email notification when a failure occurs** (Отправить уведомление по электронной почте при сбое).

    ![Почтовое уведомление](common/provisioning-notification-email.png)

9. Нажмите **Сохранить**.

10. В разделе **Картпинги** выберите **синхронизацию активных пользователей каталога Azure в Figma.**

    ![Картирование пользователей Figma](media/Figma-provisioning-tutorial/figma05.png)

11. Просмотрите атрибуты пользователя, синхронизированные с Azure AD на Figma в разделе **Атрибуты Mapping.** Атрибуты, выбранные в качестве свойств **соответствия,** используются для сопоставления учетных записей пользователей в Figma для операций обновления. Нажмите кнопку **Сохранить**, чтобы зафиксировать все изменения.

    ![Атрибуты пользователя Figma](media/Figma-provisioning-tutorial/figma06.png)

12. Чтобы настроить фильтры области, ознакомьтесь со следующими инструкциями, предоставленными в [руководстве по фильтрам области](../app-provisioning/define-conditional-rules-for-provisioning-user-accounts.md).

13. Чтобы включить службу предоставления Услуг Azure AD для Figma, измените **статус подготовки** **в** разделе **«Настройки».**

    ![Состояние подготовки "Включено"](common/provisioning-toggle-on.png)

14. Определите пользователей и/или группы, которые вы хотели бы предоставить Figma, выбрав желаемые значения в **разделе Область** в разделе **Параметры.**

    ![Область действия подготовки](common/provisioning-scope.png)

15. Когда будете готовы выполнить подготовку, нажмите кнопку **Сохранить**.

    ![Сохранение конфигурации подготовки](common/provisioning-configuration-save.png)

После этого начнется начальная синхронизация пользователей и (или) групп, определенных в поле **Область** раздела **Параметры**. Начальная синхронизация занимает больше времени, чем последующие операции синхронизации. Если служба запущена, они выполняются примерно каждые 40 минут. Раздел **«Детали синхронизации»** можно использовать для мониторинга прогресса и отслеживания ссылок на отчет о деятельности, в котором описаны все действия, выполняемые службой предоставления AD Azure AD на Figma.

Дополнительные сведения о чтении журналов подготовки Azure AD см. в руководстве по [отчетам об автоматической подготовке учетных записей](../app-provisioning/check-status-user-account-provisioning.md).

# <a name="tutorial-azure-active-directory-single-sign-on-sso-integration-with-cloud-academy---sso"></a>Руководство по интеграции единого входа Azure Active Directory с приложением "Единый вход для Cloud Academy"

В этом руководстве описано, как интегрировать приложение "Единый вход для Cloud Academy" с Azure Active Directory (Azure AD). Интеграция приложения "Единый вход для Cloud Academy" с Azure AD обеспечивает приведенные ниже возможности.

* Контроль доступа к приложению "Единый вход для Cloud Academy" с помощью Azure AD.
* Включение автоматического входа пользователей в приложение "Единый вход для Cloud Academy" с помощью учетных записей Azure AD.
* Централизованное управление учетными записями через портал Azure.

Чтобы узнать больше об интеграции приложений SaaS с Azure AD, прочитайте статью [Что такое доступ к приложениям и единый вход с помощью Azure Active Directory?](https://docs.microsoft.com/azure/active-directory/manage-apps/what-is-single-sign-on)

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее.

* Подписка Azure AD. Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Подписка на приложение "Единый вход для Cloud Academy" с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* Приложение "Единый вход для Cloud Academy" поддерживает единый вход, инициированный **поставщиком услуг**.

* После настройки приложения "Единый вход для Cloud Academy" можно применять элемент управления сеансами, который защищает от хищения и несанкционированного доступа к конфиденциальным данным вашей организации в режиме реального времени. Управление сеансом является расширением функции условного доступа. [Узнайте, как применять управление сеансами с помощью Microsoft Cloud App Security](https://docs.microsoft.com/cloud-app-security/proxy-deployment-any-app).

## <a name="adding-cloud-academy---sso-from-the-gallery"></a>Добавление приложения "Единый вход для Cloud Academy" из коллекции

Чтобы настроить интеграцию приложения "Единый вход для Cloud Academy" с Azure AD, необходимо добавить это приложение из коллекции в список управляемых приложений SaaS.

1. Войдите на [портал Azure](https://portal.azure.com) с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите **Единый вход для Cloud Academy**.
1. Выберите **Единый вход для Cloud Academy** в области результатов и добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.


## <a name="configure-and-test-azure-ad-sso-for-cloud-academy---sso"></a>Настройка и проверка единого входа Azure AD для приложения "Единый вход для Cloud Academy"

Настройте и проверьте единый вход Azure AD в приложение "Единый вход для Cloud Academy" с помощью тестового пользователя **B.Simon**. Чтобы обеспечить работу единого входа, необходимо установить связь между пользователем Azure AD и соответствующим пользователем в приложении "Единый вход для Cloud Academy".

Чтобы настроить и проверить единый вход Azure AD в приложение "Единый вход для Cloud Academy", выполните действия в следующих стандартных блоках:

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
    1. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD с помощью пользователя B.Simon.
    1. **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить пользователю B.Simon использовать единый вход Azure AD.
1. **[Настройка единого входа в приложение "Единый вход для Cloud Academy"](#configure-cloud-academy-sso-sso)** необходима, чтобы настроить параметры единого входа на стороне приложения.
    1. **[Создание тестового пользователя приложения "Единый вход для Cloud Academy"](#create-cloud-academy-sso-test-user)** требуется для того, чтобы в приложении "Единый вход для Cloud Academy" существовал пользователь B.Simon, связанный с одноименным пользователем в Azure AD.
1. **[Проверка единого входа](#test-sso)** позволяет убедиться в правильности конфигурации.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На [портале Azure](https://portal.azure.com/) на странице интеграции с приложением **Единый вход для Cloud Academy** найдите раздел **Управление** и выберите **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок "Изменить" (значок пера), чтобы открыть диалоговое окно **Базовая конфигурация SAML** и изменить параметры.

   ![Изменение базовой конфигурации SAML](common/edit-urls.png)

1. На странице **Базовая конфигурация SAML** введите значения следующих полей.

    В текстовом поле **URL-адрес входа** введите URL-адрес: `https://cloudacademy.com/login/enterprise/`.

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** нажмите кнопку "Копировать", чтобы скопировать **URL-адрес метаданных федерации приложений** и сохранить его на компьютере.

    ![Ссылка для скачивания сертификата](common/copy-metadataurl.png)

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory**, **Пользователи**, а затем — **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите `B.Simon`.  
   1. В поле **Имя пользователя** введите username@companydomain.extension. Например, `B.Simon@contoso.com`.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **Создать**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как включить единый вход Azure для пользователя B.Simon, предоставив этому пользователю доступ к приложению "Единый вход для Cloud Academy".

1. На портале Azure выберите **Корпоративные приложения**, а затем — **Все приложения**.
1. Из списка приложений выберите **Единый вход для Cloud Academy**.
1. На странице "Обзор" приложения найдите раздел **Управление** и выберите **Пользователи и группы**.

   ![Ссылка "Пользователи и группы"](common/users-groups-blade.png)

1. Выберите **Добавить пользователя**, а в диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.

    ![Ссылка "Добавить пользователя"](common/add-assign-user.png)

1. В диалоговом окне **Пользователи и группы** выберите **B.Simon** в списке пользователей, а затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если ожидается, что в утверждении SAML будет получено какое-либо значение роли, то в диалоговом окне **Выбор роли** нужно выбрать соответствующую роль для пользователя из списка и затем нажать кнопку **Выбрать**, расположенную в нижней части экрана.
1. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить**.

## <a name="configure-cloud-academy-sso-sso"></a>Настройка единого входа для приложения "Единый вход для Cloud Academy"

1. В другом окне браузера войдите на корпоративный веб-сайт приложения "Единый вход для Cloud Academy" с правами администратора.

1. Щелкните название компании и выберите в меню пункт **Settings & Integrations** (Параметры и интеграции).

    ![Конфигурация ](./media/cloud-academy-sso-tutorial/config-1.PNG)

1. На странице **Settings & Integrations** (Параметры и интеграции) перейдите на вкладку **Интеграции** и щелкните карту **Единый вход**.

    ![Конфигурация ](./media/cloud-academy-sso-tutorial/config-2.PNG)

1. На следующей странице выполните приведенные ниже шаги.

    ![Конфигурация ](./media/cloud-academy-sso-tutorial/config-3.PNG)

    а. В текстовое поле **Entity ID URL** (URL-адрес идентификатора сущности) вставьте значение **идентификатора сущности**, скопированное на портале Azure.

    b. В текстовое поле **SSO URL** (URL-адрес единого входа) вставьте значение **URL-адрес входа**, скопированное на портале Azure.

    c. Откройте в блокноте скачанный с портала Azure файл **сертификата (Base64)** и вставьте его содержимое в текстовое поле **Certificate** (Сертификат).

    d. В текстовом поле **Name ID Format** (Формат идентификатора имени) — значение по умолчанию `urn:oasis:names:tc:SAML:1.1:nameid-format:unspecified`

1. Нажмите кнопку **Сохранить**.

    > [!NOTE]
    > Дополнительные сведения о настройке приложения "Единый вход в Cloud Academy" см. в [статье службы поддержки](https://support.cloudacademy.com/hc/articles/360043908452-Setting-Up-Single-Sign-On).

### <a name="create-cloud-academy-sso-test-user"></a>Создание тестового пользователя приложения "Единый вход в Cloud Academy"

1. Войдите в **приложение "Единый вход в Cloud Academy"** .

1. Щелкните название компании и выберите в меню пункт **Участники**.

    ![ Создание тестового пользователя ](./media/cloud-academy-sso-tutorial/create-user.PNG)

1. Щелкните **Пригласить участников** и выберите **Invite a Single Member** (Пригласить одного участника).

    ![ Создание тестового пользователя ](./media/cloud-academy-sso-tutorial/create-user-1.PNG)

1. Введите данные в обязательные поля и щелкните **Пригласить**.

    ![ Создание тестового пользователя ](./media/cloud-academy-sso-tutorial/create-user-2.PNG)

# <a name="tutorial-azure-active-directory-single-sign-on-integration-with-cloud-academy---sso"></a>Руководство по интеграции единого входа Azure Active Directory с приложением "Единый вход для Cloud Academy"

В этом руководстве описано, как интегрировать приложение "Единый вход для Cloud Academy" с Azure Active Directory (Azure AD). Интеграция приложения "Единый вход для Cloud Academy" с Azure AD обеспечивает приведенные ниже возможности.

* Управление доступом к приложению "Единый вход для Cloud Academy" с помощью Azure AD.
* Автоматический вход пользователей в приложение "Единый вход для Cloud Academy" с помощью учетных записей Azure AD.
* Централизованное управление учетными записями через портал Azure.

Дополнительные сведения об интеграции приложений SaaS с Azure AD см. в статье [Что собой представляет единый вход?](https://docs.microsoft.com/azure/active-directory/manage-apps/what-is-single-sign-on)

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее.

* Подписка Azure AD. Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Подписка на приложение "Единый вход для Cloud Academy" с поддержкой единого входа.

## <a name="tutorial-description"></a>Описание учебника

В рамках этого учебника вы настроите и проверите единый вход Azure AD в тестовой среде.

Приложение "Единый вход для Cloud Academy" поддерживает единый вход, инициированный поставщиком услуг.

После настройки приложения "Единый вход для Cloud Academy" вы можете применить функцию управления сеансом, которая в реальном времени защищает конфиденциальные данные вашей организации от хищения и несанкционированного доступа. Элементы управления сеансом являются расширением функции условного доступа. [Узнайте, как применять управление сеансами с помощью Microsoft Cloud App Security](https://docs.microsoft.com/cloud-app-security/proxy-deployment-any-app).

## <a name="add-cloud-academy---sso-from-the-gallery"></a>Добавление приложения "Единый вход для Cloud Academy" из коллекции

Чтобы настроить интеграцию приложения "Единый вход для Cloud Academy" с Azure AD, необходимо добавить это приложение из коллекции в список управляемых приложений SaaS:

1. Войдите на [портал Azure](https://portal.azure.com) с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области слева выберите **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите **Единый вход для Cloud Academy**.
1. Выберите **Единый вход для Cloud Academy** в области результатов и добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.


## <a name="configure-and-test-azure-ad-sso-for-cloud-academy---sso"></a>Настройка и проверка единого входа Azure AD для приложения "Единый вход для Cloud Academy"

Настройте и проверьте единый вход Azure AD в приложение "Единый вход для Cloud Academy" с помощью тестового пользователя **B. Simon**. Чтобы обеспечить работу единого входа, необходимо установить связь между пользователем Azure AD и соответствующим пользователем в приложении "Единый вход для Cloud Academy".

Чтобы настроить и проверить единый вход Azure AD в приложение "Единый вход для Cloud Academy", вам потребуется выполнить приведенные ниже пошаговые действия.

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
    1. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD.
    1. **[Предоставление тестовому пользователю доступа](#grant-access-to-the-test-user)** необходимо, чтобы разрешить пользователю использовать единый вход Azure AD.
1. **[Настройка единого входа для приложения "Единый вход для Cloud Academy"](#configure-single-sign-on-for-cloud-academy)** на стороне приложения.
    1. **[Создание тестового пользователя приложения "Единый вход для Cloud Academy"](#create-a-cloud-academy-test-user)** , связанного с одноименным пользователем в Azure AD.
1. **[Проверка единого входа](#test-sso)** необходима, чтобы убедиться в корректной работе конфигурации.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure:

1. На [портале Azure](https://portal.azure.com/) на странице интеграции с приложением **Единый вход для Cloud Academy** найдите раздел **Управление** и выберите **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок карандаша, чтобы открыть диалоговое окно **Базовая конфигурация SAML** и изменить эти параметры:

   ![Снимок экрана, на котором показана кнопка карандаша для изменения базовой конфигурации SAML.](common/edit-urls.png)

1. В разделе **Базовая конфигурация SAML** в поле **URL-адрес для входа** введите `https://cloudacademy.com/login/enterprise/`.

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** нажмите кнопку "Копировать", чтобы скопировать **URL-адрес метаданных федерации приложений**. Сохраните URL-адрес —

    ![Снимок экрана, на котором показана кнопка копирования для URL-адреса метаданных федерации приложений.](common/copy-metadataurl.png)

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как создать на портале Azure тестового пользователя с именем B.Simon.

1. На портале Azure в области слева щелкните **Azure Active Directory**. Щелкните **Пользователи**, а затем **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. В свойствах **пользователя** выполните следующие действия:
   1. В поле **Имя** введите **B.Simon**.  
   1. В поле **Имя пользователя** введите \<username>@\<companydomain>.\<extension> Например, `B.Simon@contoso.com`.
   1. Выберите **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **создания**.

### <a name="grant-access-to-the-test-user"></a>Предоставление доступа тестовому пользователю

В этом разделе описано, как включить единый вход Azure для пользователя B. Simon, предоставив этому пользователю доступ к приложению "Единый вход для Cloud Academy".

1. На портале Azure выберите **Корпоративные приложения**, а затем — **Все приложения**.
1. Из списка приложений выберите **Единый вход для Cloud Academy**.
1. В разделе **Управление** на странице сводных данных о приложении выберите **Пользователи и группы**:

   ![Снимок экрана с параметром "Пользователи и группы".](common/users-groups-blade.png)

1. Выберите **Добавить пользователя**, а в диалоговом окне **Добавление назначения** — **Пользователи и группы**:

    ![Снимок экрана с кнопкой "Добавить пользователя".](common/add-assign-user.png)

1. В диалоговом окне **Пользователи и группы** выберите **B. Simon** в списке **Пользователи**, а затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если вы хотите получить значение роли в утверждении SAML, выберите в диалоговом окне **Выбор роли** подходящую роль для пользователя в предложенном списке. В нижней части экрана нажмите кнопку **Выбрать**.
1. В диалоговом окне **Добавление назначения** выберите **Назначить**.

## <a name="configure-single-sign-on-for-cloud-academy"></a>Настройка единого входа для Cloud Academy

1. В другом окне браузера войдите на корпоративный веб-сайт приложения "Единый вход для Cloud Academy" с правами администратора.

1. Выберите название своей компании, а затем в появившемся меню выберите **Settings & Integrations** (Параметры и интеграции):

    ![Снимок экрана, на котором показан параметр Settings & Integrations (Параметры и интеграции).](./media/cloud-academy-sso-tutorial/config-1.PNG)

1. На странице **Settings & Integrations** (Параметры и интеграции) перейдите на вкладку **Integrations** (Интеграции) и щелкните карту **SSO** (Единый вход):

    ![Снимок экрана, на котором показана карта единого входа на вкладке "Интеграции".](./media/cloud-academy-sso-tutorial/config-2.PNG)

1. На этой странице выполните указанные ниже действия.

    ![Снимок экрана, на котором показана страница интеграции и единого входа.](./media/cloud-academy-sso-tutorial/config-3.PNG)

    a. В поле **Entity ID URL** (URL-адрес идентификатора сущности) вставьте значение идентификатора сущности, скопированное на портале Azure.

    b. В поле **SSO URL** (URL-адрес единого входа) вставьте значение URL-адрес входа, скопированное на портале Azure.

    c. Откройте в Блокноте скачанный с портала Azure сертификат Base64. Вставьте его содержимое в поле **Certificate** (Сертификат).

    d. В поле **Name ID Format** (Формат идентификатора имени) сохраните значение по умолчанию: `urn:oasis:names:tc:SAML:1.1:nameid-format:unspecified`.

1. Щелкните **Сохранить**.

    > [!NOTE]
    > Дополнительные сведения о настройке приложения "Единый вход для Cloud Academy" см. в [этой статье](https://support.cloudacademy.com/hc/articles/360043908452-Setting-Up-Single-Sign-On).

### <a name="create-a-cloud-academy-test-user"></a>Создание тестового пользователя приложения "Единый вход для Cloud Academy"

1. Войдите в приложение "Единый вход для Cloud Academy".

1. Выберите название своей компании, а затем в появившемся меню выберите **Members** (Участники):

    ![Снимок экрана, на котором показан параметр Members (Участники).](./media/cloud-academy-sso-tutorial/create-user.PNG)

1. Щелкните **Invite Members** (Пригласить участников) и выберите **Invite a Single Member** (Пригласить одного участника):

    ![Снимок экрана с параметром Invite a Single Member (Пригласить одного участника).](./media/cloud-academy-sso-tutorial/create-user-1.PNG)

1. Введите значения в обязательные поля, а затем выберите **Invite** (Пригласить):

    ![Снимок экрана с диалоговым окном приглашения участника.](./media/cloud-academy-sso-tutorial/create-user-2.PNG)

## <a name="test-sso"></a>Проверка единого входа 

Затем выполните проверку конфигурации единого входа Azure AD с помощью Панели доступа.

Щелкнув плитку приложения "Единый вход для Cloud Academy" на Панели доступа, вы автоматически войдете в экземпляр приложения "Единый вход для Cloud Academy", для которого настроили единый вход. Дополнительные сведения о панели доступа см. в [этой статье](https://docs.microsoft.com/azure/active-directory/active-directory-saas-access-panel-introduction).

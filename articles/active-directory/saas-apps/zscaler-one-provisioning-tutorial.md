# <a name="tutorial-configure-zscaler-one-for-automatic-user-provisioning"></a>Учебник: Настройка «Scaler One» для автоматического подготовки пользователей

В этом уроке показаны шаги, которые необходимо выполнить в active-каталогах «Scaler One» и Azure Active Directory (Azure AD) для настройки Azure AD для автоматического предоставления и деформатива пользователей и групп на «Scaler One».

> [!NOTE]
> В этом уроке описывается разъем, построенный поверх службы предоставления пользовательского обеспечения Azure AD. Для получения информации о том, что эта служба делает, как она работает, и часто задаваемые вопросы, см [Автоматизировать пользовательские подготовки и деобеспечения программного обеспечения как службы (SaaS) приложений с Azure Active Directory](../active-directory-saas-app-provisioning.md).


## <a name="prerequisites"></a>Предварительные требования

Сценарий, изложенный в этом учебнике предполагает, что у вас есть:

* Клиент Azure AD.
* Арендатор "Скразлер один" .
* Учетная запись пользователя в «Scaler One» с разрешениями админа.

> [!NOTE]
> Интеграция Azure AD опирается на API sciM sscaler One. Этот API доступен разработчикам sscaler One для учетных записей с пакетом Enterprise.

## <a name="add-zscaler-one-from-the-azure-marketplace"></a>Добавьте один из них в Azure Marketplace

Перед настройкой «scaler One» для автоматического предоставления пользовательской подготовки с помощью Azure AD добавьте в список управляемых приложений SaaS список управляемых приложений SaaS.

Чтобы добавить один из маркетплейса, выполните следующие действия.

1. На [портале Azure](https://portal.azure.com)в навигационном стеле слева выберите **Active Directory Azure.**

    ![Значок активного каталога Azure](common/select-azuread.png)

2. Перейдите к **корпоративным приложениям,** а затем выберите **все приложения.**

    ![Колонка "Корпоративные приложения"](common/enterprise-applications.png)

3. Чтобы добавить новое приложение, в верхней части диалогового окна выберите **Новое приложение**.

    ![Кнопка "Создать приложение"](common/add-new-app.png)

4. В поле поиска введите **«Scaler One»** и выберите **«Scaler One»** из панели результатов. Чтобы добавить приложение, **выберите Добавить**.

    ![Zscaler One в списке результатов](common/search-new-app.png)

## <a name="assign-users-to-zscaler-one"></a>Присвоить пользователям «Scaler One»

Активный каталог Azure использует концепцию, называемую *назначениями,* чтобы определить, какие пользователи должны получить доступ к выбранным приложениям. В контексте автоматической подготовки пользователей синхронизируются только пользователи или группы, назначенные приложению Azure AD.

Прежде чем настроить и включить автоматическую подготовку пользователей, решите, каким пользователям или группам в Azure AD нужен доступ к «scaler One». Чтобы присвоить этим пользователям или группам «Scaler One», следуйте инструкциям в [присваивать пользователю или группе корпоративное приложение.](https://docs.microsoft.com/azure/active-directory/active-directory-coreapps-assign-user-azure-portal)

### <a name="important-tips-for-assigning-users-to-zscaler-one"></a>Важные советы по назначению пользователей на «Scaler One»

* Мы рекомендуем назначить одного пользователя Azure AD для тестирования автоматической конфигурации подготовки пользователей. Вы можете назначить дополнительных пользователей или групп ы позже.

* При назначении пользователя в журнал «Scaler One» выберите любую действительную роль, конкретную для приложения, если она имеется, в поле диалога назначения. Пользователи с ролью **доступа по умолчанию** исключены из подготовки.

## <a name="configure-automatic-user-provisioning-to-zscaler-one"></a>Настройка автоматического предоставления пользовательу для «Scaler One»

В этом разделе вы проведете шаги по настройке службы обеспечения AD Azure AD. Используйте его для создания, обновления и отработки пользователей или групп в группе в «Scaler One» на основе пользовательских или групповых заданий в Azure AD.

> [!TIP]
> Вы также можете включить SAML на основе одного знака для Zscaler One. Следуйте инструкциям в [учебнике по единому знаку .](zscaler-One-tutorial.md) Единый набор может быть настроен независимо от автоматического подготовки пользователей, хотя эти две функции дополняют друг друга.

### <a name="configure-automatic-user-provisioning-for-zscaler-one-in-azure-ad"></a>Настройка автоматического получения пользовательского резерва для «Scaler One» в Azure AD

1. Войдите на [портал Azure](https://portal.azure.com). Выберите **Корпоративные приложения** > **Все приложения** > **« Scaler One**.

    ![Колонка "Корпоративные приложения"](common/enterprise-applications.png)

2. Из списка приложений выберите **Zscaler One**.

    ![Ссылка на одну ссылку в списке приложений](common/all-applications.png)

3. Выберите вкладку **«Подготовка».**

    ![«Скалер Одно Обеспечение](./media/zscaler-one-provisioning-tutorial/provisioning-tab.png)

4. Установите **режим обеспечения** **автоматического.**

    ![Режим обеспечения одного из них](./media/zscaler-one-provisioning-tutorial/provisioning-credentials.png)

5. В разделе **Admin Credentials** заполните **URL-адрес URL-адрес ы клиента** и коробки **Secret Token** с настройками для вашей учетной записи «Scaler One», как описано в шаге 6.

6. Чтобы получить URL-адрес клиента и секретный токен, перейдите в**параметры аутентификации** **администрации** > в uI портала «Scaler One». В разделе **Authentication Type** (Тип проверки подлинности) выберите **SAML**.

    ![Настройки один сертификат подлинности](./media/zscaler-one-provisioning-tutorial/secret-token-1.png)

    а. Выберите **настройку SAML,** чтобы открыть параметры **Настройки SAML.**

    ![«Скразлер Одна настройка SAML](./media/zscaler-one-provisioning-tutorial/secret-token-2.png)

    b. Выберите **Включение SCIM-на основе обеспечения,** чтобы получить настройки в **базовом URL** и **маркера предъявителя.** Затем сохраните настройки. Копируйте настройки **URL-адреса базы** на **URL-адресе Tenant** на портале Azure. Копируйте настройку **маркера Bearer** на портале **Secret Token** на портале Azure.

7. После заполнения коробок, показанных в шаге 5, выберите **тестовое соединение,** чтобы убедиться, что Azure AD может подключиться к «Scaler One». Если соединение не удается, убедитесь, что ваша учетная запись «Scaler One» имеет разрешения для админоватора, и повторите попытку.

    ![Одно тест-соединение «Сккер»](./media/zscaler-one-provisioning-tutorial/test-connection.png)

8. В поле **электронной почты уведомления** введите адрес электронной почты человека или группы для получения уведомлений об ошибке. Выберите **уведомление отправить сообщение по электронной почте, когда происходит сбой** флажка.

    ![«Scaler Одно из юлийша электронной почты](./media/zscaler-one-provisioning-tutorial/notification.png)

9. Нажмите кнопку **Сохранить**.

10. В разделе **Картпинги** выберите **синхронизацию активных пользователей каталога Azure на «Scaler One».**

    ![Синхронизация пользователя «Scaler One»](./media/zscaler-one-provisioning-tutorial/user-mappings.png)

11. Просмотрите атрибуты пользователя, синхронизированные с Azure AD на «Scaler One» в разделе **Атрибут карт.** Атрибуты, выбранные в качестве свойств **соответствия,** используются для сопоставления учетных записей пользователей в «Scaler One» для операций обновления. Чтобы сохранить все изменения, выберите **Сохранить**.

    ![«Scaler Один из соответствующих атрибутов пользователя](./media/zscaler-one-provisioning-tutorial/user-attribute-mappings.png)

12. В разделе **Картпинги** выберите **синхронизацию групп активных каталогов Azure на «Scaler One».**

    ![Синхронизация группы «Сккер Одна»](./media/zscaler-one-provisioning-tutorial/group-mappings.png)

13. Просмотрите атрибуты группы, синхронизированные с Azure AD с «Scaler One» в разделе **Атрибут Картпинги.** Атрибуты, выбранные в качестве свойств **соответствия,** используются для сопоставления групп в «Scaler One» для операций обновления. Чтобы сохранить все изменения, выберите **Сохранить**.

    ![Соответствующие групповые атрибуты «Scaler One»](./media/zscaler-one-provisioning-tutorial/group-attribute-mappings.png)

14. Для настройки фильтров для обсажания следуйте инструкциям в [учебнике по отслеживанию фильтров.](./../active-directory-saas-scoping-filters.md)

15. Для обеспечения сервиса предоставления Azure AD для «scaler One» в разделе **«Настройки»** измените **статус «Обеспечение»** на **On**.

    ![Статус «Скаллер Один Обеспечение»](./media/zscaler-one-provisioning-tutorial/provisioning-status.png)

16. Определите пользователей или группы, которые вы хотите предоставить в «Scaler One». В разделе **Настройки** выберите значения, которые вы хотите в **области области.**

    ![«Скразлер Одна область](./media/zscaler-one-provisioning-tutorial/scoping.png)

17. Когда вы будете готовы к предоставлению, выберите **Сохранить**.

    ![«Скалер Один Сохранить](./media/zscaler-one-provisioning-tutorial/save-provisioning.png)

Эта операция запускает начальную синхронизацию всех пользователей или групп, определенных в **разделе Область** в разделе **Настройки.** Начальная синхронизация занимает больше времени, чем более поздняя синхронизация. Они возникают примерно каждые 40 минут при запуске службы обеспечения AD Azure AD. 

Раздел **«Детали синхронизации»** можно использовать для мониторинга прогресса и отслеживания ссылок на отчет о деятельности подготовки. В отчете описаны все действия, выполняемые службой предоставления AD Azure AD на sscaler One.

Сведения о журналах подготовки Azure AD см. в руководстве [по отчетам об автоматической подготовке учетных записей пользователей](../active-directory-saas-provisioning-reporting.md).

# <a name="tutorial-configure-smartfile-for-automatic-user-provisioning"></a>Учебник: Налаживание SmartFile для автоматического подготовки пользователей

Цель этого руководства состоит в том, чтобы продемонстрировать шаги, которые будут выполнены в SmartFile и Azure Active Directory (Azure AD) для настройки Azure AD для автоматического предоставления и де-предоставления пользователям и/или группам SmartFile.

> [!NOTE]
> В этом руководстве рассматривается соединитель, созданный на базе службы подготовки пользователей Azure AD. Подробные сведения о том, что делает эта служба, как она работает, и часто задаваемые вопросы см. в статье [Автоматическая подготовка пользователей и ее отзыв для приложений SaaS в Azure Active Directory](../app-provisioning/user-provisioning.md).
>
> Сейчас этот соединитель предоставляется в общедоступной предварительной версии. Дополнительные сведения об общих условиях использования продуктов в предварительной версии см. в документе [Дополнительные условия использования Предварительных версий Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

## <a name="prerequisites"></a>Предварительные требования

В сценарии, описанном в этом руководстве, предполагается, что у вас уже имеется:

* Клиент Azure AD.
* [Арендатор SmartFile](https://www.SmartFile.com/pricing/).
* Учетная запись пользователя в SmartFile с разрешениями Admin.

## <a name="assigning-users-to-smartfile"></a>Назначение пользователей SmartFile

Активный каталог Azure использует концепцию, называемую *назначениями,* чтобы определить, какие пользователи должны получить доступ к выбранным приложениям. В контексте автоматической подготовки пользователей синхронизируются только пользователи и/или группы, назначенные приложению azure AD.

Прежде чем настроить и включить автоматическую подготовку пользователей, необходимо решить, какие пользователи и/или группы в Azure AD нуждаются в доступе к SmartFile. После решения, вы можете назначить этих пользователей и / или групп SmartFile, следуя инструкциям здесь:
* [Назначение корпоративному приложению пользователя или группы](../manage-apps/assign-user-or-group-access-portal.md)

## <a name="important-tips-for-assigning-users-to-smartfile"></a>Важные советы по назначению пользователей SmartFile

* Для тестирования автоматической конфигурации подготовки пользователей рекомендуется назначить одного пользователя Azure AD SmartFile. Дополнительные пользователи и/или группы можно назначить позднее.

* При назначении пользователя ВС-Файлу необходимо выбрать любую действительную роль приложения (если она имеется) в диалоге о назначении. Пользователи с ролью **доступа по умолчанию** исключены из подготовки.

## <a name="setup-smartfile-for-provisioning"></a>Настройка SmartFile для подготовки

Прежде чем настроить SmartFile для автоматического предоставления пользовательского запроса с Azure AD, необходимо включить SCIM-подготовку на SmartFile и собрать дополнительные детали, необходимые.

1. Вопийте на консоль SmartFile Admin. Перейдите в верхний правый угол консоли SmartFile Admin. Выберите **ключ продукта**.

    ![Консоль SmartFile Admin](media/smartfile-provisioning-tutorial/login.png)

2. Чтобы создать маркер носителя, скопируйте **ключ продукта** и **пароль продукта.** Вставьте их в блокнот с толстой кишки между ними.
    
     ![SmartFile Добавить SCIM](media/smartfile-provisioning-tutorial/auth.png)

    ![SmartFile Добавить SCIM](media/smartfile-provisioning-tutorial/key.png)

## <a name="add-smartfile-from-the-gallery"></a>Добавить SmartFile из галереи

Чтобы настроить SmartFile для автоматической подготовки пользователей с помощью Azure AD, необходимо добавить SmartFile из галереи приложений Azure AD в список управляемых приложений SaaS.

**Чтобы добавить SmartFile из галереи приложений Azure AD, выполните следующие действия:**

1. На **[портале Azure](https://portal.azure.com)** на левой навигационной панели выберите **Active Directory Azure.**

    ![Кнопка Azure Active Directory](common/select-azuread.png)

2. Перейдите к **корпоративным приложениям,** а затем выберите **все приложения.**

    ![Колонка "Корпоративные приложения"](common/enterprise-applications.png)

3. Чтобы добавить новое приложение, выберите кнопку **Нового приложения** в верхней части панели.

    ![Кнопка "Создать приложение"](common/add-new-app.png)

4. В поле поиска введите **SmartFile,** выберите **SmartFile** в панели результатов, а затем нажмите кнопку **Добавить,** чтобы добавить приложение.

    ![SmartFile в списке результатов](common/search-new-app.png)

## <a name="configuring-automatic-user-provisioning-to-smartfile"></a>Настройка автоматического предоставления пользователям SmartFile 

В этом разделе вы проведет шаги по настройке службы предоставления AD Azure AD для создания, обновления и отработки пользователей и/или групп в SmartFile на основе пользовательских и/или групповых заданий в Azure AD.

> [!TIP]
> Вы также можете включить SAML на основе одного знака для SmartFile , следуя инструкциям, представленным в [SmartFile Единый набор.](SmartFile-tutorial.md) Единый подгон может быть настроен независимо от автоматического подготовки пользователей, хотя эти две функции дополняют друг друга

### <a name="to-configure-automatic-user-provisioning-for-smartfile-in-azure-ad"></a>Для настройки автоматического пользовательского обеспечения SmartFile в Azure AD:

1. Войдите на [портал Azure](https://portal.azure.com). Выберите **корпоративные приложения,** затем выберите **все приложения.**

    ![Колонка "Корпоративные приложения"](common/enterprise-applications.png)

2. Из списка приложений выберите **SmartFile**.

    ![Ссылка на SmartFile в списке "Приложения"](common/all-applications.png)

3. Выберите вкладку **«Подготовка».**

    ![Вкладка подготовки](common/provisioning.png)

4. Установите **режим обеспечения** **автоматического.**

    ![Вкладка подготовки](common/provisioning-automatic.png)

5.  В разделе **Admin Credentials** `https://<SmartFile sitename>.smartfile.com/ftp/scim` ввод **URL-адреса клиента**. Пример будет выглядеть `https://demo1test.smartfile.com/ftp/scim`как . Введите значение **маркера Bearer** (ProductKey:ProductPassword), которое вы получили ранее в **Secret Token.** Нажмите **на тестовое соединение,** чтобы обеспечить подключение Azure AD к SmartFile. Если соединение не удается, убедитесь, что ваша учетная запись SmartFile имеет разрешения admin и повторите попытку.

    ![URL-адрес клиента + токен](common/provisioning-testconnection-tenanturltoken.png)

6. В поле **Почтовое уведомление** введите адрес электронной почты пользователя или группы, которые должны получать уведомления об ошибках подготовки, а также установите флажок **Send an email notification when a failure occurs** (Отправить уведомление по электронной почте при сбое).

    ![Почтовое уведомление](common/provisioning-notification-email.png)

7. Нажмите **Сохранить**.

8. В разделе **Картпинги** выберите **синхронизацию активных пользователей каталога Azure в SmartFile.**

    ![Картирование пользователей SmartFile](media/smartfile-provisioning-tutorial/usermapping.png)

9. Просмотрите атрибуты пользователя, синхронизированные с Azure AD на SmartFile в разделе **«Отображение атрибутов».** Атрибуты, выбранные в качестве свойств **соответствия,** используются для сопоставления учетных записей пользователей в SmartFile для операций обновления. Нажмите кнопку **Сохранить**, чтобы зафиксировать все изменения.

    ![Атрибуты пользователя SmartFile](media/smartfile-provisioning-tutorial/userattribute.png)

10. В разделе **Картпинги** выберите **синхронизацию групп активных каталогов Azure в SmartFile.**

    ![Картирование группsmartFile](media/smartfile-provisioning-tutorial/groupmapping.png)

11. Просмотрите атрибуты группы, синхронизированные с Azure AD с SmartFile в разделе **«Отображение атрибутов».** Атрибуты, выбранные в качестве свойств **соответствия,** используются для соответствия группам в SmartFile для операций обновления. Нажмите кнопку **Сохранить**, чтобы зафиксировать все изменения.

    ![Атрибуты группы SmartFile](media/smartfile-provisioning-tutorial/groupattribute.png)

12. Чтобы настроить фильтры области, ознакомьтесь со следующими инструкциями, предоставленными в [руководстве по фильтрам области](../app-provisioning/define-conditional-rules-for-provisioning-user-accounts.md).

13. Чтобы включить службу предоставления AD Azure AD для SmartFile, измените **статус подготовки** **в** разделе **«Настройки».**

    ![Состояние подготовки "Включено"](common/provisioning-toggle-on.png)

14. Определите пользователей и/или группы, которые вы хотели бы предоставить SmartFile, выбрав желаемые значения в **разделе Область** в разделе **Параметры.**

    ![Область действия подготовки](common/provisioning-scope.png)

15. Когда будете готовы выполнить подготовку, нажмите кнопку **Сохранить**.

    ![Сохранение конфигурации подготовки](common/provisioning-configuration-save.png)

    После этого начнется начальная синхронизация пользователей и (или) групп, определенных в поле **Область** раздела **Параметры**. Начальная синхронизация занимает больше времени, чем последующие операции синхронизации. Если служба запущена, они выполняются примерно каждые 40 минут. Раздел **«Детали синхронизации»** можно использовать для мониторинга прогресса и отслеживания ссылок на отчет о деятельности, в котором описаны все действия, выполняемые службой предоставления AD Azure AD на SmartFile.

    Для получения дополнительной информации о том, как читать журналы подготовки Azure AD, [см.](../app-provisioning/check-status-user-account-provisioning.md)

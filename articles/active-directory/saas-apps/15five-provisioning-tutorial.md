# <a name="tutorial-configure-15five-for-automatic-user-provisioning"></a>Учебник: Настройка 15Five для автоматического подготовки пользователей

Цель этого руководства состоит в том, чтобы продемонстрировать шаги, которые будут выполнены в 15Five и Azure Active Directory (Azure AD) для настройки Azure AD для автоматического предоставления и де-предоставления пользователей и/или групп до 15Five.

> [!NOTE]
> В этом руководстве рассматривается соединитель, созданный на базе службы подготовки пользователей Azure AD. Подробные сведения о том, что делает эта служба, как она работает, и часто задаваемые вопросы см. в статье [Автоматическая подготовка пользователей и ее отзыв для приложений SaaS в Azure Active Directory](../app-provisioning/user-provisioning.md).
>
> Сейчас этот соединитель предоставляется в общедоступной предварительной версии. Дополнительные сведения об общих условиях использования продуктов в предварительной версии см. в документе [Дополнительные условия использования Предварительных версий Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

## <a name="prerequisites"></a>Предварительные требования

В сценарии, описанном в этом руководстве, предполагается, что у вас уже имеется:

* Клиент Azure AD.
* [15Пять арендаторов](https://www.15five.com/pricing/).
* Учетная запись пользователя в 15Five с разрешениями админ.

## <a name="assigning-users-to-15five"></a>Назначение пользователей 15Five

Активный каталог Azure использует концепцию, называемую *назначениями,* чтобы определить, какие пользователи должны получить доступ к выбранным приложениям. В контексте автоматической подготовки пользователей синхронизируются только пользователи и/или группы, назначенные приложению azure AD.

Прежде чем настроить и включить автоматическую подготовку пользователей, необходимо решить, каким пользователям и/или группам в Azure AD нужен доступ к 15Five. После того, как решение, вы можете назначить этих пользователей и / или групп до 15Five, следуя инструкциям здесь:
* [Назначение корпоративному приложению пользователя или группы](../manage-apps/assign-user-or-group-access-portal.md)

## <a name="important-tips-for-assigning-users-to-15five"></a>Важные советы по назначению пользователей на 15Five

* Для тестирования автоматической конфигурации подготовки пользователей рекомендуется назначить одному пользователю Azure AD 15Five. Дополнительные пользователи и/или группы можно назначить позднее.

* При назначении пользователя 15Five необходимо выбрать любую действительную роль приложения (если она имеется) в диалоге о назначении. Пользователи с ролью **доступа по умолчанию** исключены из подготовки.

## <a name="setup-15five-for-provisioning"></a>Настройка 15Пять для подготовки

Прежде чем настроить 15Five для автоматического предоставления пользовательского запроса с Azure AD, необходимо включить SCIM-подготовку на 15Five.

1. Войти на [консоль 15Five Admin.](https://my.15five.com/) Перейдите к **функциям > интеграции.**

    ![15Пять Админ консоль](media/15five-provisioning-tutorial/integration.png)

2.  Нажмите на **SCIM 2.0**.

    ![15Пять Админ консоль](media/15five-provisioning-tutorial/image00.png)

3.  Перейдите к **интеграции SCIM > generate OAuth токен.**

    ![15Пять Добавить SCIM](media/15five-provisioning-tutorial/image02.png)

4.  Копируйте значения для **базового URL-адреса SCIM 2.0** и **маркера доступа.** Это значение будет введено в поле **URL-адреса арендатора** и **Secret Token** в вкладке «Обеспечение вашего приложения 15Five» на портале Azure.
    
    ![15Пять Добавить SCIM](media/15five-provisioning-tutorial/image03.png)

## <a name="add-15five-from-the-gallery"></a>Добавить 15Five из галереи

Чтобы настроить 15Five для автоматического оформления пользовательского интерфейса с помощью Azure AD, необходимо добавить 15Five из галереи приложений Azure AD в список управляемых приложений SaaS.

**Чтобы добавить 15Five из галереи приложений Azure AD, выполните следующие действия:**

1. На **[портале Azure](https://portal.azure.com)** на левой навигационной панели выберите **Active Directory Azure.**

    ![Кнопка Azure Active Directory](common/select-azuread.png)

2. Перейдите к **корпоративным приложениям,** а затем выберите **все приложения.**

    ![Колонка "Корпоративные приложения"](common/enterprise-applications.png)

3. Чтобы добавить новое приложение, выберите кнопку **Нового приложения** в верхней части панели.

    ![Кнопка "Создать приложение"](common/add-new-app.png)

4. В поле поиска, введите **15Five**, выберите **15Five** в панели результатов, а затем нажмите кнопку **Добавить,** чтобы добавить приложение.

    ![15Five в списке результатов](common/search-new-app.png)

## <a name="configuring-automatic-user-provisioning-to-15five"></a>Настройка автоматического обеспечения пользователя до 15Five 

В этом разделе вы проведет шаги по настройке службы предоставления AD Azure AD для создания, обновления и отработки пользователей и/или групп в 15Five на основе пользовательских и/или групповых заданий в Azure AD.

> [!TIP]
> Вы также можете включить SAML основе одного знака на 15Five , следуя инструкциям, представленным в [15Five Один очный учебник](15five-tutorial.md). Единый вход можно настроить независимо от автоматической подготовки пользователей, хотя эти две возможности дополняют друг друга.

### <a name="to-configure-automatic-user-provisioning-for-15five-in-azure-ad"></a>Для настройки автоматического обеспечения пользователя для 15Five в Azure AD:

1. Войдите на [портал Azure](https://portal.azure.com). Выберите **корпоративные приложения,** затем выберите **все приложения.**

    ![Колонка "Корпоративные приложения"](common/enterprise-applications.png)

2. В списке приложений выберите **15Five**.

    ![Ссылка на 15Five в списке приложений](common/all-applications.png)

3. Выберите вкладку **«Подготовка».**

    ![Вкладка подготовки](common/provisioning.png)

4. Установите **режим обеспечения** **автоматического.**

    ![Вкладка подготовки](common/provisioning-automatic.png)

5.  В разделе "Учетные данные админ" введомые **значения базы SCIM 2.0 и маркеры доступа,** полученные ранее в **URL-адресе арендатора** и **секретном токене** соответственно. Нажмите **на тестовое соединение,** чтобы обеспечить подключение Azure AD к 15Five. Если соединение не удается, убедитесь, что ваша учетная запись 15Five имеет разрешения admin и повторите попытку.

    ![URL-адрес клиента + токен](common/provisioning-testconnection-tenanturltoken.png)

6. В поле **Почтовое уведомление** введите адрес электронной почты пользователя или группы, которые должны получать уведомления об ошибках подготовки, а также установите флажок **Send an email notification when a failure occurs** (Отправить уведомление по электронной почте при сбое).

    ![Почтовое уведомление](common/provisioning-notification-email.png)

7. Нажмите **Сохранить**.

8. В разделе **Картпинги** выберите **синхронизацию активных пользователей каталога Azure до 15Five**.

    ![15Пять карт пользователей](media/15five-provisioning-tutorial/usermapping.png)

9. Просмотрите атрибуты пользователя, синхронизированные с Azure AD, с 15Пятью в разделе **«Отображение атрибутов».** Атрибуты, выбранные в качестве свойств **соответствия,** используются для сопоставления учетных записей пользователей в 15Five для операций обновления. Нажмите кнопку **Сохранить**, чтобы зафиксировать все изменения.

    ![15Пять атрибутов пользователя](media/15five-provisioning-tutorial/userattribute.png)

10. В разделе **Картпинги** выберите **синхронизацию групп активных каталогов Azure до 15Five**.

    ![15Пять карт групп](media/15five-provisioning-tutorial/groupmapping.png)

11. Просмотрите атрибуты группы, синхронизированные с Azure AD, с 15Пятью в разделе **Атрибуты Mapping.** Атрибуты, выбранные в качестве свойств **соответствия,** используются для сопоставления групп в 15Five для операций обновления. Нажмите кнопку **Сохранить**, чтобы зафиксировать все изменения.

    ![15Пять групповых атрибутов](media/15five-provisioning-tutorial/groupattribute.png)

12. Чтобы настроить фильтры области, ознакомьтесь со следующими инструкциями, предоставленными в [руководстве по фильтрам области](../app-provisioning/define-conditional-rules-for-provisioning-user-accounts.md).

13. Чтобы включить службу предоставления Azure AD для 15Five, измените **статус подготовки** в разделе **"Настройки".** **On**

    ![Состояние подготовки "Включено"](common/provisioning-toggle-on.png)

14. Определите пользователей и/или группы, которые вы хотели бы предоставить 15Five, выбрав желаемые значения в **разделе Область** в разделе **Параметры.**

    ![Область действия подготовки](common/provisioning-scope.png)

15. Когда будете готовы выполнить подготовку, нажмите кнопку **Сохранить**.

    ![Сохранение конфигурации подготовки](common/provisioning-configuration-save.png)

    После этого начнется начальная синхронизация пользователей и (или) групп, определенных в поле **Область** раздела **Параметры**. Начальная синхронизация занимает больше времени, чем последующие операции синхронизации. Если служба запущена, они выполняются примерно каждые 40 минут. Раздел **«Детали синхронизации»** можно использовать для мониторинга прогресса и отслеживания ссылок на отчет о деятельности, в котором описаны все действия, выполняемые службой предоставления AD Azure aD на 15Five.

    Для получения дополнительной информации о том, как читать журналы подготовки Azure AD, [см.](../app-provisioning/check-status-user-account-provisioning.md)

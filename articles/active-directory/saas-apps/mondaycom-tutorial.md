# <a name="tutorial-azure-active-directory-integration-with-mondaycom"></a>Руководство по Интеграция Azure Active Directory с monday.com

В этом руководстве описано, как интегрировать monday.com с Azure Active Directory (Azure AD).

Интеграция monday.com с Azure AD обеспечивает следующие преимущества.

* С помощью Azure AD вы можете контролировать доступ к monday.com.
* Пользователи могут автоматически входить в monday.com с помощью учетных записей Azure AD (единый вход).
* Вы можете управлять учетными записями централизованно, через портал Azure.

Дополнительные сведения об интеграции приложений SaaS с Azure AD см. в статье [Единый вход в приложениях в Azure Active Directory](https://docs.microsoft.com/azure/active-directory/active-directory-appssoaccess-whatis).

## <a name="prerequisites"></a>Предварительные требования

Чтобы настроить интеграцию Azure AD с monday.com, вам потребуется:

* подписка Azure AD Если у вас еще нет подписки Azure AD, [создайте бесплатную учетную запись](https://azure.microsoft.com/free/), прежде чем приступить к работе.
* подписка monday.com с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде и интеграцию monday.com с Azure AD.

monday.com поддерживает следующие функции:

* **единый вход, инициируемый поставщиком услуг**.
* **единый вход, инициированный поставщиком удостоверений**;
* **JIT-подготовку пользователей**.

## <a name="add-mondaycom-in-the-azure-portal"></a>Добавление monday.com на портале Azure

Чтобы интегрировать monday.com с Azure AD, необходимо добавить monday.com в список управляемых приложений SaaS.

1. Войдите на [портале Azure](https://portal.azure.com).

1. В меню слева выберите **Azure Active Directory**.

    ![Пункт Azure Active Directory](common/select-azuread.png)

1. Щелкните **Корпоративные приложения** > **Все приложения**.

    ![Область "Корпоративные приложения"](common/enterprise-applications.png)

1. Чтобы добавить приложение, выберите **Новое приложение**.

    ![Элемент "Новое приложение"](common/add-new-app.png)

1. В поле поиска введите **monday.com**. В результатах поиска выберите **monday.com**, а затем щелкните **Добавить**.

    ![monday.com в списке результатов](common/search-new-app.png)

Чтобы настроить и проверить единый вход Azure AD в monday.com, потребуется выполнить действия в следующих стандартных блоках.

| Задача | ОПИСАНИЕ |

### <a name="configure-azure-ad-single-sign-on"></a>Настройка единого входа Azure AD

В этом разделе описывается настройка единого входа Azure AD в monday.com на портале Azure.

1. На [портале Azure](https://portal.azure.com/) в области интеграции с приложением **monday.com** выберите **Единый вход**.

    ![Параметр для настройки единого входа](common/select-sso.png)

1. В области **Выбрать метод единого входа** выберите режим **SAML** или **SAML/WS-FED**, чтобы включить единый вход.

    ![Режим выбора единого входа](common/select-saml-option.png)

1. В области **Настройка единого входа с помощью SAML** щелкните **Изменить** (значок карандаша), чтобы открыть область **Базовая конфигурация SAML**.

    ![Правка базовой конфигурации SAML](common/edit-urls.png)

1. Если вам нужно настроить единый вход, *инициируемый поставщиком удостоверений*, и у вас есть файл метаданных поставщика услуг, выполните следующие действия в разделе **Базовая конфигурация SAML**.

    1. Щелкните **Отправить файл метаданных**.

       ![Параметр "Отправить файл метаданных"](common/upload-metadata.png)

    1. Чтобы выбрать файл метаданных, выберите значок папки, а затем выберите **Отправить**.

       ![Выберите файл метаданных, а затем нажмите кнопку "Отправить"](common/browse-upload-metadata.png)

    1. После успешной передачи файла метаданных значения параметров **Идентификатор** и **URL-адрес ответа** в области **Базовая конфигурация SAML** будут заполнены автоматически.

       ![Значения параметров поставщика удостоверений в области "Базовая конфигурация SAML"](common/idp-intiated.png)

       > [!Note]
       > Если значения **Идентификатор** и **URL-адрес ответа** не заполнены, введите их вручную.

1. Чтобы настроить приложение в режиме, *инициируемом поставщиком служб*, выполните следующие действия.

    1. Щелкните **Задать дополнительные URL-адреса**.
    
    1. В поле **URL-адрес для входа** введите URL-адрес в следующем формате: https:\//\<ваш_домен>.monday.com. Чтобы получить URL-адрес для входа, обратитесь к [группе поддержки клиентов monday.com](mailto:support@monday.com).

        ![Параметр "Задать дополнительные URL-адреса"](common/metadata-upload-additional-signon.png)

1. Приложение monday.com ожидает утверждения SAML в определенном формате. Настройте следующие утверждения для этого приложения. Чтобы изменить значения этих атрибутов, в области **Настройка единого входа с помощью SAML** выберите **Изменить**. Откроется область **Атрибуты пользователя**.

    ![Область "Атрибуты пользователя"](common/edit-attribute.png)

1. В разделе **Утверждения пользователя** выберите **Изменить**, чтобы изменить утверждения. Чтобы добавить утверждение, выберите **Добавить новое утверждение**, а затем настройте атрибут токена SAML, как показано на предыдущем рисунке. Затем сделайте следующее: 

    1. Выберите **Добавить новое утверждение**.

        ![Параметр "Добавить новое утверждение" в области "Утверждения пользователя"](common/new-save-attribute.png)

    1. В области **Управление утверждениями пользователя** настройте следующие значения.
        
       1. В поле **Имя** введите имя атрибута, отображаемое для строки утверждения пользователя.

       1. Поле **Пространство имен** оставьте пустым.

       1. В поле **Источник** выберите **Атрибут**.

       1. Из списка **Атрибут источника** выберите значение атрибута, отображаемое для строки утверждения пользователя.

       1. Нажмите кнопку **ОК**, а затем нажмите кнопку **Сохранить**.

       ![Область "Управление утверждениями пользователя"](common/new-attribute-details.png)

1. В области **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** нажмите кнопку **Скачать**, расположенную рядом с полем **Сертификат (Base64)** . Выберите параметр для скачивания в соответствии с вашими требованиями. Сохраните сертификат на своем компьютере.

    ![Параметр для скачивания сертификата в кодировке Base64](common/certificatebase64.png)

1. Требуемые URL-адреса можно скопировать из раздела **Настройка monday.com**.

    * URL-адрес входа.
    * Идентификатор Azure AD
    * URL-адрес выхода.

    ![Копирование URL-адресов настройки](common/copy-configuration-urls.png)

### <a name="configure-mondaycom-single-sign-on"></a>Настройка единого входа в monday.com

Чтобы настроить единый вход на стороне monday.com, нужно отправить скачанный файл сертификата (Base64) и соответствующие URL-адреса, скопированные на портале Azure, [группе поддержки monday.com](mailto:support@monday.com). Группа поддержки monday.com на основе полученной информации выполнит все необходимое для правильной настройки подключения для единого входа SAML с обеих сторон.

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как создать на портале Azure тестового пользователя с именем Britta Simon.

1. На портале Azure выберите **Azure Active Directory** > **Пользователи** > **Все пользователи**.

    ![Параметры "Пользователи" и "Все пользователи"](common/users.png)

1. Выберите **Новый пользователь**.

    ![Параметр "Новый пользователь"](common/new-user.png)

1. В области **Пользователь** выполните приведенные ниже действия.

    1. В поле **Имя** введите **BrittaSimon**.
  
    1. В поле **Имя пользователя** введите **brittasimon\@\<домен_вашей_компании>.\<доменная_зона>** . Например, **brittasimon\@contoso.com**.

    1. Установите флажок **Показать пароль**. Запишите значение, которое отображается в поле **Пароль**.

    1. Нажмите кнопку **Создать**.

    ![Панель "Пользователь"](common/user-properties.png)

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе вы предоставите Britta Simon доступ к monday.com, что позволит этому пользователю использовать единый вход Azure.

1. На портале Azure выберите **Корпоративные приложения** > **Все приложения** > **monday.com**.

    ![Область "Корпоративные приложения"](common/enterprise-applications.png)

1. В списке приложений выберите **monday.com**.

    ![monday.com в списке приложений](common/all-applications.png)

1. В меню выберите **Пользователи и группы**.

    ![Параметр "Пользователи и группы"](common/users-groups-blade.png)

1. Выберите **Добавить пользователя**. Затем в области **Добавление назначения** щелкните **Пользователи и группы**.

    ![Область "Добавление назначения"](common/add-assign-user.png)

1. В области **Пользователи и группы** из списка пользователей выберите **Britta Simon**. Щелкните **Выбрать**.

1. Если вы планируете получать значение роли в утверждении SAML, выберите в области **Выбор роли** подходящую роль для пользователя из предложенного списка. Щелкните **Выбрать**.

1. В области **Добавление назначения** выберите **Назначить**.

### <a name="create-a-mondaycom-test-user"></a>Создание тестового пользователя monday.com

В этом разделе вы создадите в приложении monday.com пользователя Britta Simon. Приложение monday.com поддерживает JIT-подготовку пользователей, которая включена по умолчанию. В этом разделе никакие действия с вашей стороны не требуются. Если пользователь еще не существует в monday.com, он создается после проверки подлинности.

### <a name="test-single-sign-on"></a>Проверка единого входа

После настройки единого входа вы сможете автоматически войти в приложение monday.com, щелкнув **monday.com** на портале "Мои приложения". Дополнительные сведения о портале "Мои приложения" см. в разделе [Доступ и использование приложений на портале "Мои приложения"](../user-help/my-apps-portal-end-user-access.md).

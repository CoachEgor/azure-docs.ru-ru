#  <a name="adopting-azure-ad-conditional-access"></a>Внедрение условный доступ Azure AD

В мире мобильных и облачных пользователей к ресурсам организации из любого места с различных типов устройств и приложений. Таким образом просто указать, кто имеет доступ к ресурсу больше не достаточно. Вы можете управлять, кто имеет доступ и определить, где пользователь является и какое устройство используется и многое другое.

Для такого рода контроль обеспечивается **условного доступа Azure Active Directory (AD)** можно указать условия, любой пользователь должен соответствовать для доступа к приложению, например многофакторную идентификацию (MFA). С помощью политик условного доступа управляет авторизованных пользователей (пользователей, которые предоставлен доступ к облачному приложению) доступ к облачным приложениям при определенных условиях. Ссылаться на [что такое условный доступ в Azure Active Directory](overview.md#conditional-access-policies) Дополнительные сведения.

## <a name="key-benefits"></a>Основные преимущества

Ниже перечислены основные преимущества использования условного доступа Azure AD.

* **Повышение производительности:** Политики условного доступа (ЦС) позволяют ориентироваться точки, по которому пользователям предлагается для использования многофакторной проверки Подлинности, имеют доступ к заблокирован или требуется использовать доверенное устройство. Например можно задать политики, например тому, что пользователям многофакторной проверки Подлинности в приложение при работе вне корпоративной сети. Уменьшение запросы многофакторной проверки Подлинности позволяет пользователям более производительны, чем после MFA каждый раз при входе. Кроме того условного доступа Azure AD можно указать политики каждого пользователя отдельно, а также создает политики для конкретного приложения.
* **Управление рисками:** Включение политик условного доступа вы найдете защиты идентификации масштабируемой в облаке, возможности управления доступом на основе рисков и поддержки собственную многофакторную проверку подлинности. Взаимозависимость условного доступа с помощью защиты идентификации можно задать при блокировке или с условным доступом к приложению.
* **Их нормативных адрес:** Аудит запросов на доступ и утверждения для приложения и основные сведения об использовании приложения в целом проще с Azure AD, так как он поддерживает журналы аудита в машинном коде для каждого запроса доступа приложения выполняется. Аудит включает в себя удостоверения запрашивающей стороны, запрошенный день, бизнес-обоснование, состояние утверждения и удостоверения утверждающего. Эти данные можно загрузить с помощью API, позволяющие импорта этих данных в инцидент безопасности и система наблюдения за событиями (SIEM).
* **Управление затратами на:** Перемещение политики доступа в Azure AD сокращает зависимость от пользовательских или локальных решений например служб федерации Active Directory (ADFS) для условного доступа, уменьшая расходы на ИТ-инфраструктуры.

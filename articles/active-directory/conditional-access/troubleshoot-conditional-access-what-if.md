# <a name="troubleshooting-conditional-access-using-the-what-if-tool"></a>Устранение проблем условный доступ с помощью инструмента «Что делать если»

Инструмент [What If](what-if-tool.md) в условиях условного доступа является мощным при попытке понять, почему политика была или не применялась к пользователю в определенных обстоятельствах или если политика будет применяться в известном состоянии.

Инструмент What If находится на **портале** > Azure**Active Directory** > **Conditional Access** > **What If.**

![Условный доступ Что делать инструмент в состоянии по умолчанию](./media/troubleshoot-conditional-access-what-if/conditional-access-what-if-tool.png)

> [!NOTE]
> В настоящее время инструмент What If не оценивает политики в режиме только для отчетов.

## <a name="gathering-information"></a>Сбор информации

Что делать, если инструмент требует только **пользователя,** чтобы начать работу. 

Следующая дополнительная информация является факультативной, но поможет сузить сферу для конкретных случаев.

* Облачные приложения или действия
* IP-адрес 
* Country
* Платформа устройства
* Клиентские приложения (предварительный просмотр)
* Состояние устройства (предварительный просмотр) 
* Риск при входе

Эта информация может быть собрана от пользователя, его устройства или входа в журнал Azure AD.

## <a name="generating-results"></a>Генерация результатов

Ввейте критерии, собранные в предыдущем разделе, и выберите **Что делать,** если для создания списка результатов. 

В любой момент можно выбрать **сбросить,** чтобы очистить ввод любых критериев и вернуться в состояние по умолчанию.

## <a name="evaluating-results"></a>Оценка результатов

### <a name="policies-that-will-apply"></a>Политики, которые будут применяться

В этом списке будет утвянено, какие политики условного доступа будут применяться с учетом условий. Список будет включать в себя как грант и сессии управления, которые применяются. Примеры включают требование многофакторной аутентификации для доступа к определенному приложению.

### <a name="policies-that-will-not-apply"></a>Политики, которые не будут применяться

В этом списке будут отображаться политики условного доступа, которые не будут применяться, если применяются условия. Список будет включать в себя любые политики и причину, по которой они не применяются. Примеры включают пользователей и группы, которые могут быть исключены из политики.

## <a name="use-case"></a>Вариант использования

Многие организации создают политики, основанные на местоположениях сетей, разрешая доверенные места и блокируя места, где не должно быть доступа.

Чтобы проверить, что конфигурация была сделана надлежащим образом, администратор может использовать инструмент What If для имитации доступа, из места, которое должно быть разрешено, и из места, которое должно быть отказано.

![Что делать, если инструмент показывает результаты с доступом к блоку](./media/troubleshoot-conditional-access-what-if/conditional-access-what-if-results.png)

В этом случае пользователю будет заблокирован доступ к любому облачному приложению во время поездки в Северную Корею, поскольку Contoso заблокировал доступ из этого места.

Этот тест можно было бы расширить, чтобы включить в них другие точки данных, чтобы сузить сферу охвата.

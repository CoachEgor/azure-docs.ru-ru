# <a name="configure-a-conditional-access-policy-in-report-only-mode-preview"></a>Настройка политики условного доступа в режиме только для отчетов (Preview)

Для настройки политики условного доступа в режиме только для отчетов:

> [!IMPORTANT]
> Если организация еще не сделала этого, [навяжьте интеграцию Azure Monitor с Azure AD.](#set-up-azure-monitor-integration-with-azure-ad) Этот процесс должен быть проведен до того, как данные будут доступны для проверки.

1. Войти на **портал Azure** в качестве администратора условного доступа, администратора безопасности или глобального администратора.
1. Просмотрите доступ к > **условную безопасность****безопасности** **Azure Active Directory.** > 
1. Выберите **новую политику**.
1. Настройка условий политики и необходимый контроль за предоставлением грантов по мере необходимости.
1. В **соответствии** с политикой включить установить переключатель в **режимтолько для отчетов.**
1. Выберите **Сохранить**

> [!TIP]
> Можно отсечь состояние **политики включить** существующую политику от **On** к **отчету только,** но это отключит правоприменение политики. 

Просмотр результатов только для отчетов в журналах входа в журналы входа Azure AD.

Для просмотра результатов политики только для конкретного входиного регистра:

1. Войти на **портал Azure** в качестве читателя отчетов, считывателя безопасности, администратора безопасности или глобального администратора.
1. Просмотрите > **для вхывана в**Active **Directory Каталог.**
1. Выберите входить в систему или добавьте фильтры для узких результатов.
1. В ящике **«Подробности»** выберите вкладку **«Только для отчетов» (Preview)** для просмотра политик, оцениваемых во время входини.

> [!NOTE]
> При загрузке журналов входов выберите формат JSON, чтобы включить данные о результатах отчета «Условный доступ».

1. [Подпишитесь на подписку Azure Monitor и создайте рабочее пространство.](/azure/azure-monitor/learn/quick-create-workspace)

Более подробную информацию о ценах Azure Monitor можно найти на [странице ценообразования Azure Monitor.](https://azure.microsoft.com/pricing/details/monitor/) Ресурсы для оценки затрат, установления дневного лимита или настройки периода хранения данных можно найти в статье [«Управление использованием и затратами с помощью журналов Azure Monitor.](../../azure-monitor/platform/manage-cost-storage.md#estimating-the-costs-to-manage-your-environment)

## <a name="view-conditional-access-insights-workbook"></a>Просмотр рабочей книги «Условная доступность»

После интеграции журналов Azure AD с Azure Monitor можно отслеживать влияние политик условного доступа, используя новые рабочие книги «Условный доступ».

1. Войти на **портал Azure** в качестве администратора безопасности или глобального администратора.
1. Просмотрите > **рабочие книги**об **активных каталогах Azure.**
1. Выберите **Условную информацию о доступе (Предварительный просмотр)**.
1. Выберите одну или несколько политик из зоны **условного** доступа. Все включенные политики выбираются по умолчанию.
1. Выберите временной диапазон (если временной диапазон превышает имеющийся набор данных, в отчете будут отображаться все доступные данные). После установки параметров **политики условного доступа** и **диапазона времени** отчет загружается.
   1. Дополнительно, поиск отдельных **пользователей** или **приложений,** чтобы сузить область охвата отчета.
1. Выберите между просмотром данных в временном диапазоне по количеству пользователей или количеству всхваев.
1. В зависимости от **представления данных,** **Резюме воздействия** отображает количество пользователей или вводов в объем выбранных параметров, сгруппированных по общему числу, **Успех**, **Отказ**, Пользователь **действий требуется,** и **не применяется**. Выберите плитку для изучения ввода в определенный тип результата. 
   1. Если вы изменили параметры трудовой книжки, вы можете сохранить копию для использования в будущем. Выберите значок сохранения в верхней части отчета и предоставьте имя и местоположение для сохранения.
1. Прокрутите вниз, чтобы просмотреть разбивку всасывания для каждого состояния.
1. Просмотр **сведений о входе в** нижней части отчета для изучения отдельных событий, отфильтрованных выше.

> [!TIP] 
> Нужно сверлить на конкретном запросе или экспортировать детали регистрации? Выберите кнопку справа от любого запроса, чтобы открыть запрос в журнале Analytics. Выберите Экспорт для экспорта в CSV или Power BI.

## <a name="common-problems-and-solutions"></a>Распространенные проблемы и способы их решения

### <a name="why-are-the-queries-in-the-workbook-failing"></a>Почему запросы в трудовой книжке не удавят?

Клиенты заметили, что запросы иногда завершаются неудачей, если неправильные или несколько рабочих областей связаны с рабочей книгой. Чтобы решить эту проблему, нажмите **"Отредкируй"** в верхней части рабочей книги, а затем в передаче "Настройки". Выберите, а затем удалите рабочие области, не связанные с рабочей книгой. С каждой рабочей книгой должно быть связано только одно рабочее пространство.

### <a name="why-doesnt-the-conditional-access-policies-dropdown-parameter-contain-my-policies"></a>Почему параметр отсеивания политик условного доступа не содержит моих политик?

Выпадение политик условного доступа заполняется запросом последних входов в течение 4 часов. Если у арендатора нет входе в систему за последние 4 часа, возможно, что выпадение будет пустым. Если эта задержка является постоянной проблемой, например, в небольших клиентах с редкими входами, админы могут отсеивать запрос для отсечения политик и условлять время запроса на время более 4 часов.

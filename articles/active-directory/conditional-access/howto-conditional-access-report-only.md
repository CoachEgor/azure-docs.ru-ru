# <a name="configure-a-conditional-access-policy-in-report-only-mode"></a>Настройка политики условного доступа в режиме «только отчет»

Настройка политики условного доступа в режиме «только отчет».

> [!IMPORTANT]
> Если ваша организация еще не [настроена, Настройте интеграцию Azure Monitor с Azure AD](#set-up-azure-monitor-integration-with-azure-ad). Этот процесс должен выполняться до того, как данные будут доступны для просмотра.

1. Войдите в **портал Azure** в качестве администратора условного доступа, администратора безопасности или глобального администратора.
1. Выберите **Azure Active Directory** > **Безопасность** > **Условный доступ**.
1. Выберите **Новая политика**.
1. Настройте условия политики и требуемые разрешения на предоставление элементов управления.
1. В разделе **включить политику** установите переключатель в режим **только для отчетов** .
1. Нажмите кнопку **Сохранить**.

> [!TIP]
> Состояние **включения политики** для существующей политики можно изменить с **на** **только для отчета** , но при этом будет отключено применение политики. 

Просмотр результатов только отчетов в журналах входа Azure AD.

Чтобы просмотреть результат политики «только отчет» для конкретного входа, выполните следующие действия.

1. Войдите в **портал Azure** как средство чтения отчетов, читатель безопасности, администратор безопасности или глобальный администратор.
1. Перейдите к **Azure Active Directory**  >  **входов**.
1. Выберите вход или добавьте фильтры для узких результатов.
1. В контейнере **сведений** выберите вкладку **только отчет** , чтобы просмотреть политики, вычисленные во время входа в систему.

> [!NOTE]
> При скачивании журналов входа выберите формат JSON, чтобы включить в отчет условный доступ только результирующие данные.

## <a name="set-up-azure-monitor-integration-with-azure-ad"></a>Настройка интеграции Azure Monitor с Azure AD

Чтобы просмотреть статистическое влияние политик условного доступа с помощью новой книги по условному доступу, необходимо интегрировать Azure Monitor с Azure AD и экспортировать журналы входа. Для настройки этой интеграции необходимо выполнить два действия. 

1. [Подпишитесь на Azure Monitorную подписку и создайте рабочую область](../../azure-monitor/learn/quick-create-workspace.md).
1. [Экспортируйте журналы входа из Azure AD в Azure Monitor](../reports-monitoring/howto-integrate-activity-logs-with-log-analytics.md).

Дополнительные сведения о ценах на Azure Monitor можно найти на [странице цен на Azure Monitor](https://azure.microsoft.com/pricing/details/monitor/). Ресурсы для оценки затрат, установки ежедневного ограничения или настройки срока хранения данных можно найти в статье [Управление использованием и затратами с помощью журналов Azure Monitor](../../azure-monitor/platform/manage-cost-storage.md#estimating-the-costs-to-manage-your-environment).

## <a name="view-conditional-access-insights-workbook"></a>Просмотр книги "условный доступ к аналитике"

После интеграции журналов Azure AD с Azure Monitor можно отслеживать влияние политик условного доступа с помощью новых книг по условному доступу.

1. Войдите в **портал Azure** в качестве администратора безопасности или глобального администратора.
1. Перейдите к **Azure Active Directory**  >  **книгам**.
1. Выберите **аналитика условного доступа**.
1. Выберите одну или несколько политик из раскрывающегося списка **Политика условного доступа** . По умолчанию выбраны все включенные политики.
1. Выберите диапазон времени (если диапазон времени превышает доступный набор данных, в отчете будут показаны все доступные данные). После задания параметров **политики условного доступа** и **диапазона времени** отчет будет загружен.
   1. При необходимости выполните поиск отдельных **пользователей** или **приложений** , чтобы ограничить область отчета.
1. Выберите между просмотром данных в диапазоне времени числом пользователей или числом входов.
1. В зависимости от **представления данных**в **сводке влияния** отображается количество пользователей или входов в области выбранных параметров, сгруппированных по общему числу, **успеху**, **неудаче**, **требуемому действию пользователя**и **не применению**. Выберите плитку, чтобы проверить входы определенного типа результатов. 
   1. Если параметры книги были изменены, можно сохранить копию для использования в будущем. Щелкните значок сохранить в верхней части отчета и укажите имя и расположение для сохранения.
1. Прокрутите вниз, чтобы просмотреть разбивку данных на события входа для каждого условия.
1. Просмотрите **сведения о входе** в нижней части отчета, чтобы исследовать отдельные события входа, отфильтрованные по выбранным выше параметрам.

> [!TIP] 
> Необходимо выполнить детализацию определенного запроса или экспортировать сведения о входе? Нажмите кнопку справа от любого запроса, чтобы открыть запрос в Log Analytics. Выберите экспорт для экспорта в CSV-файл или Power BI.

## <a name="common-problems-and-solutions"></a>Распространенные проблемы и способы их решения

### <a name="why-are-the-queries-in-the-workbook-failing"></a>Почему происходит сбой запросов в книге?

Клиенты заметили, что запросы иногда завершаются ошибкой, если с книгой связано неправильное или несколько рабочих областей. Чтобы устранить эту проблему, щелкните **изменить** в верхней части книги, а затем выберите параметры. Выберите и удалите рабочие области, не связанные с книгой. С каждой книгой должна быть связана только одна рабочая область.

### <a name="why-doesnt-the-conditional-access-policies-dropdown-parameter-contain-my-policies"></a>Почему параметр раскрывающегося списка политик условного доступа содержит мои политики?

Раскрывающийся список политики условного доступа заполняется с помощью запроса последних входов в течение 4 часов. Если у клиента нет входов за последние 4 часа, возможно, раскрывающийся список будет пустым. Если эта задержка является постоянной, например, в небольших клиентах с редкими входами, администраторы могут изменить запрос в раскрывающемся списке политики условного доступа и продлить время выполнения запроса дольше 4 часов.

# <a name="what-is-conditional-access-report-only-mode"></a>Что такое режим отчета только для условного доступа?

Условный доступ широко используется нашими клиентами для обеспечения безопасности, применяя правильный контроль доступа в правильных обстоятельствах. Однако одной из проблем, связанных с развертыванием политики условного доступа в организации, является определение влияния на конечных пользователей. Может быть трудно предвидеть количество и имена пользователей, на которых влияют общие инициативы по развертыванию, такие как блокирование устаревшей аутентификации, требующая многофакторной аутентификации для группы пользователей или реализация политик и ных священцев. 

Режим только для отчетов — это новое состояние политики условного доступа, которое позволяет администраторам оценивать влияние политик «Условный доступ», прежде чем включить их в среду.  С выпуском режима только для отчетов:

- Политики условного доступа могут быть включены в режиме только для отчетов.
- Во время вхождения в систему политики в режиме только для отчетов оцениваются, но не применяются.
- Результаты регистрируются во вкладке **"Условный доступ"** и **"Предварительный просмотр"** данных журнала "Входе".
- Клиенты с подпиской Azure Monitor могут отслеживать влияние своих политик условного доступа, используя рабочую книгу «Условный доступ».

> [!WARNING]
> Политики в режиме только для отчетов, требующие совместимых устройств, могут побудить пользователей на Mac, iOS и Android выбрать сертификат устройства во время оценки политики, даже если соответствие требованиям устройства не обеспечивается. Эти запросы могут повторяться до тех пор, пока устройство не будет совместимо. Чтобы конечные пользователи не получали запросы во время вхлечения, исключите платформы устройств Mac, iOS и Android из политики, выполняющей только отчетные правила, выполняющие проверку соответствия устройств.

## <a name="policy-results"></a>Результаты политики

При оценке политики в режиме только для отчета для вхождения в систему, существует четыре новых возможных значения результата:

| Результат | Описание |
| --- | --- |
| Отчет только: Успех | Все настроенные условия политики, необходимые неинтерактивные средства управления грантами и контроль сеансов были удовлетворены. Например, требование о многофакторной аутентификации удовлетворяется претензией МИД, уже присутствующем в токене, или соответствующей политике устройства, выполняющей проверку устройства на совместимом устройстве. |
| Отчет только: Отказ | Все настроенные условия политики были удовлетворены, но не все необходимые неинтерактивные средства управления грантами или контроль сеанса были удовлетворены. Например, политика применяется к пользователю, где настроен элемент управления блоком или устройство не выполняет соответствующую политику устройства. |
| Отчет только: Требуется действия пользователя | Все настраиваемые условия политики были удовлетворены, но для удовлетворения требуемого контроля гранта или управления сеансом потребуются действия пользователя. В режиме только для отчетов пользователю не предлагается удовлетворить требуемые элементы управления. Например, пользователям не предлагается утилизать многофакторную аутентификацию или условия использования.   |
| Отчет только: не применяется | Не все настроенные условия политики были удовлетворены. Например, пользователь исключается из политики или политика применяется только к определенным доверенным именитым местоположениям. |

## <a name="conditional-access-insights-workbook"></a>Условная книга Insights

Администраторы имеют возможность создавать несколько политик в режиме только для отчетов, поэтому необходимо понимать как индивидуальное воздействие каждой политики, так и совокупное влияние нескольких политик, оцениваемых вместе. Новая трудовая книжка Conditional Access Insights позволяет администраторам визуализировать запросы условного доступа и отслеживать влияние политики на данный диапазон времени, набор приложений и пользователей.

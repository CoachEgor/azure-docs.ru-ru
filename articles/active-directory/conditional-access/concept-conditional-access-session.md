# <a name="conditional-access-session"></a>Условный доступ: сессия

В рамках политики условного доступа администратор может использовать элементы управления сеансами для обеспечения ограниченного взаимодействия в определенных облачных приложениях.

![Политика условного доступа с контролем грантов, требующим многофакторной аутентификации](./media/concept-conditional-access-session/conditional-access-session.png)

## <a name="application-enforced-restrictions"></a>Принудительные ограничения применения

Организации могут использовать этот элемент управления, чтобы потребовать от Azure AD передавать информацию об устройстве выбранным облачным приложениям. По этим сведениям об устройстве облачные приложения могут определить, инициируется ли подключение с соответствующего требованиям устройства или устройства, присоединенного к домену. Этот элемент управления поддерживает только SharePoint Online и Exchange Online в качестве выбранных облачных приложений. Выбранное облачное приложение использует сведения об устройствах для предоставления пользователям ограниченных или полных возможностей в зависимости от состояния устройства.

Для получения дополнительной информации об использовании и конфигурации принудительного ограничения приложения см.

- [Включение ограниченного доступа с помощью SharePoint Online](/sharepoint/control-access-from-unmanaged-devices)
- [Включение ограниченного доступа с помощью Exchange Online](https://aka.ms/owalimitedaccess)

## <a name="conditional-access-application-control"></a>Управление приложениям условного доступа

Условный доступ App Control использует обратную архитектуру прокси и однозначно интегрирован с Azure AD Условный доступ. Условный доступ Azure AD позволяет принудительно применять политики управления доступом к приложениям в организации на основе определенных условий. Условия позволяют определить, к кому (пользователи или группы пользователей), к чему (облачные приложения) и где (расположения и сети) применяется политика условного доступа. После определения условий можно направлять пользователей в [Microsoft Cloud App Security,](/cloud-app-security/what-is-cloud-app-security) где можно защитить данные с помощью управления приложениями условного доступа, применяя элементы управления доступом и сеансами.

Функция управления настройками условного доступа для приложений позволяет в режиме реального времени отслеживать и контролировать доступ пользователей к приложениям и сеансы с помощью политик доступа и сеансов. Политики доступа и сеансов Cloud App Security используются на портале для дополнительного уточнения фильтров и определения действий, которые могут предприниматься в отношении пользователя. Возможности политик доступа и сеансов

- Предотвращение эксфильтрации данных: Вы можете блокировать загрузку, вырезать, копировать и печатать конфиденциальные документы, например, на неуправляемых устройствах.
- Защита от загрузки: Вместо того, чтобы блокировать загрузку конфиденциальных документов, вы можете потребовать, чтобы документы были помечены и защищены защитой информации Azure. Это действие гарантирует защиту документа и ограничение доступа пользователей в потенциально рискованной сессии.
- Предотвращение загрузки немаркированных файлов: Прежде чем конфиденциальный файл будет загружен, распространен и использован другими пользователями, важно убедиться, что файл имеет правильную метку и защиту. Можно гарантировать, что немаркированные файлы с конфиденциальным контентом будут заблокированы до тех пор, пока пользователь не классифицирует содержимое.
- Мониторинг пользовательских сессий на соответствие требованиям: Рискованные пользователи отслеживаются при входе в приложения, а их действия регистрируются в рамках сеанса. Благодаря этому вы можете исследовать и анализировать действия пользователя, чтобы понять, где и при каких условиях необходимо применять политики сеансов в будущем.
- Блокирование доступа: Вы можете детально блокировать доступ для определенных приложений и пользователей в зависимости от нескольких факторов риска. Например, их можно заблокировать, если они используют сертификаты клиентов в качестве одной из форм управления устройствами.
- Блокировать пользовательские действия: Некоторые приложения имеют уникальные сценарии, которые несут риск, например, отправка сообщений с конфиденциальным контентом в приложениях, таких как Microsoft Teams или Slack. В таких сценариях можно сканировать сообщения для конфиденциального содержимого и блокировать их в режиме реального времени.

Для получения дополнительной информации смотрите статью [Развертывание условный доступ App Control для признакам приложений](/cloud-app-security/proxy-deployment-aad).

## <a name="sign-in-frequency-preview"></a>Частота ввоза (Предварительный просмотр)

Частота ввоза определяет период времени, когда пользователю будет предложено войти в систему снова при попытке получить доступ к ресурсу.

Настройка частоты регистрации работает с приложениями, которые внедрили протоколы OAUTH2 или OIDC в соответствии со стандартами. Большинство нацприложений Microsoft для Windows, Mac и Mobile, включая следующие веб-приложения, соответствуют настройкам.

- Слово, Excel, PowerPoint Online
- OneNote онлайн
- Office.com
- Портал O365 Админ
- Exchange Online
- SharePoint и OneDrive
- Команды веб-клиента
- Dynamics CRM Online
- Портал Azure

Для получения дополнительной информации см. [Configure authentication session management with Conditional Access](howto-conditional-access-session-lifetime.md#user-sign-in-frequency)

## <a name="persistent-browser-session-preview"></a>Стойкая сессия браузера (Предварительный просмотр)

Постоянный сеанс браузера позволяет пользователям оставаться в сети после закрытия и повторного открытия окна браузера.

# <a name="what-are-service-dependencies-in-azure-active-directory-conditional-access"></a>Что такое зависимость от службы в условном доступе к каталогу Azure Active Directory? 

С помощью политик «Условный доступ» можно указать требования к доступу к веб-сайтам и службам. Например, требования к доступу могут включать требование многофакторной аутентификации (MFA) или [управляемых устройств.](require-managed-devices.md) 

При непосредственном доступе к сайту или службе влияние соответствующей политики обычно легко оценить. Например, если у вас есть политика, требующая настройки MFA для SharePoint Online, МИД применяется для каждого входе на веб-портал SharePoint. Однако оценивать влияние политики не всегда можно, поскольку существуют облачные приложения с зависимостями от других облачных приложений. Например, команды Майкрософт могут предоставить доступ к ресурсам в SharePoint Online. Таким образом, при доступе к командам Майкрософт в нашем текущем сценарии на вас также распространяется политика SharePoint MFA.   

## <a name="policy-enforcement"></a>Принудительное применение политики 

Если у вас настроена зависимость службы, политика может быть применена с помощью раннего или позднего принудительного исполнения. 

- **Правоприменение политики, находящиеся на раннем этапе,** означает, что пользователь должен удовлетворить зависимую политику службы, прежде чем получить доступ к вызывающей приложению. Например, пользователь должен удовлетворить политику SharePoint перед подписанием в команды MS. 
- **Принуждение к политике с поздним перептомом** происходит после того, как пользователь входит в приложение вызова. Правоприменение откладывается при вызове запросов приложений, токен для службы ниже по течению. Примеры включают MS Команды доступа планировщик и Office.com доступ к SharePoint. 

На приведенной ниже диаграмме показаны зависимости службы MS Teams. Твердые стрелки указывают на раннее выполнение, на что стрелка для Planner указывает на поздний привязка. 

![Зависимости от службы MS Teams](./media/service-dependencies/01.png)

В качестве наилучшей практики следует, когда это возможно, устанавливать общие политики в соответствующих приложениях и службах. Наличие последовательной позы безопасности обеспечивает вам лучший пользовательский опыт. Например, установление общей политики в Exchange Online, SharePoint Online, Microsoft Teams и Skype для бизнеса значительно сокращает неожиданные запросы, которые могут возникнуть в результате различных политик, применяемых к службам ниже по течению.

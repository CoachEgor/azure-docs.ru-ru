# <a name="what-are-service-dependencies-in-azure-active-directory-conditional-access"></a>Каковы зависимости служб в условном доступе Azure Active Directory? 


С помощью политик условного доступа можно указать требований доступа к веб-сайтов и служб. Например, требования к доступу можно включить требование многофакторной идентификации (MFA) или [управляемые устройства](require-managed-devices.md). 


При доступе к сайт или служба, непосредственно, обычно легко оценить влияние соответствующей политики. Например при наличии политику, которая требует многофакторной проверки Подлинности для SharePoint Online настроена многофакторной проверки Подлинности является обязательной для каждого входа на веб-портал SharePoint. Тем не менее не всегда очевидно чтобы оценить влияние политики, так как существуют облачных приложений с зависимостями с другими облачными приложениями. Например Microsoft Teams использует SharePoint online. Таким образом при доступе к Microsoft Teams в данного сценария, вы также подлежат политики многофакторной проверки Подлинности SharePoint.   


## <a name="policy-enforcement"></a>Принудительное применение политики 

Если у вас есть зависимость службы настроены, политика может применяться с помощью принудительного применения с ранним связыванием или с поздним связыванием. 

**Применение политики с ранним связыванием** означает, что пользователь должен выполнить политики зависимые службы, прежде чем обращаться к вызывающего приложения. Например пользователь должен выполнить политика SharePoint, прежде чем войти в Microsoft Teams. 

**Применение политики с поздним связыванием** происходит после входом пользователя в вызывающее приложение. Принудительное применение перекладывается на при вызове запросов приложения, маркер для подчиненной службе. Примеры Microsoft Teams, планировщик и Office.com, доступ к SharePoint. 

На следующей схеме показаны зависимости служб Microsoft Teams. Сплошные стрелки указывают принудительного применения с ранним связыванием пунктирные стрелки для планировщика указывает принудительного применения с поздним связыванием. 



![Зависимости служб Microsoft Teams](./media/service-dependencies/01.png)



  

Рекомендуется необходимо задать общие политики в связанных приложениях и службах, когда это возможно. Наличие согласованного защищенности предоставляет удобства работы пользователей. Например параметр общую политику в Exchange Online, SharePoint Online, Microsoft Teams и Скайп для бизнеса значительно снижает непредвиденные запросы, которые могут быть вызваны разные политики, применяемых к подчиненным службам.

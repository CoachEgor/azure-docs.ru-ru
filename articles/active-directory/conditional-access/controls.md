# <a name="custom-controls-preview"></a>Настраиваемые элементы управления (предварительная версия)

Пользовательские элементы управления — это возможность предварительного просмотра каталога Azure Active. При использовании пользовательских элементов управления пользователи перенаправляются в совместимую службу для удовлетворения требований аутентификации за пределами Active Directory Azure. Чтобы удовлетворить этот элемент управления, браузер пользователя перенаправляется во внешний сервис, выполняет любую необходимую аутентификацию, а затем перенаправляется обратно в Active Directory Azure. Активный каталог Azure проверяет ответ, и если пользователь был успешно проверен или проверен, пользователь продолжает находиться в потоке условного доступа.

> [!NOTE]
> Для получения дополнительной информации об изменениях, которые мы планируем сделать в отношении возможности пользовательского контроля, см. [What's new update](../fundamentals/whats-new.md#upcoming-changes-to-custom-controls)

## <a name="creating-custom-controls"></a>Создание пользовательских элементов управления

Пользовательский контроль работает с ограниченным набором утвержденных поставщиков аутентификации. Чтобы создать пользовательский элемент управления, сначала обратитесь к поставщику, которым вы хотите воспользоваться. Каждый поставщик, не являющиеся корпорацией Майкрософт, имеет свой собственный процесс и требования для регистрации, подписки или иным образом стать частью службы, а также указать, что вы хотите интегрироваться с Условным Доступом. После выполнения этих требований поставщик предоставит вам блок данных в формате JSON. Эти данные позволяют поставщику и условного доступа работать вместе для вашего клиента, создают новый элемент управления и определяют, как условный доступ может определить, если ваши пользователи успешно выполнили проверку с поставщиком.

Скопируйте данные JSON и вставьте их в соответствующее текстовое поле. Не внося никаких изменений в JSON, если вы не понимаете изменения, которые вы вносите. Изменение данных JSON может привести к разрыву связи между поставщиком и корпорацией Майкрософт и к блокировке учетных записей ваших пользователей.

Возможность создания пользовательского элемента управления находится в разделе **Управление** **страницем условного доступа.**

![Control](./media/controls/82.png)

Щелкните **Создать пользовательский элемент управления**. Откроется колонка с текстовым полем, в котором нужно указать данные JSON элемента управления.  

![Control](./media/controls/81.png)

## <a name="deleting-custom-controls"></a>Удаление пользовательских элементов управления

Чтобы удалить пользовательский элемент управления, необходимо сначала убедиться, что он не используется в какой-либо политике условного доступа. После завершения:

1. Перейдите к списку пользовательских элементов управления
1. Нажмите кнопку ...  
1. Выберите **Удалить**.

## <a name="editing-custom-controls"></a>Изменение пользовательских элементов управления

Чтобы изменить пользовательский элемент управления, необходимо удалить текущий элемент управления и создать новый элемент управления с обновленными данными.

## <a name="known-limitations"></a>Известные ограничения

Пользовательские элементы управления не могут использоваться с автоматизацией Identity Protection, требующей многофакторной аутентификации Azure, сбросить пароля самообслуживания Azure AD (SSPR), удовлетворяющих многофакторным требованиям требований к требованию проверки подлинности, или для повышения роли в Привилегированном Менеджер идентификации (PIM).

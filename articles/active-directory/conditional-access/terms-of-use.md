# <a name="azure-active-directory-terms-of-use"></a>Условия использования Active Directory Azure

Условия использования Azure AD предоставляют простой метод, который организации могут использовать для представления информации конечным пользователям. Благодаря этой презентации пользователи видят соответствующие заявления об отказе согласно юридическим требованиям и для соответствия стандартам. В этой статье описывается, как начать работу с условиями использования.

## <a name="what-can-i-do-with-terms-of-use"></a>Что я могу сделать с условиями использования?

Условия использования Azure AD могут использовать следующие возможности:

- Требуйте от сотрудников или гостей принять ваши условия использования, прежде чем получить доступ.
- Требуйте от сотрудников или гостей принять ваши условия использования на каждом устройстве, прежде чем получить доступ.
- Требуйте от сотрудников или гостей принять ваши условия использования по постоянному графику.
- Требуйте от сотрудников или гостей принять ваши условия использования перед регистрацией информации о безопасности в Azure Multi-Factor Authentication (MFA).
- Требуйте от сотрудников принять ваши условия использования перед регистрацией информации о безопасности в сбое паролей самообслуживания Azure AD (SSPR).
- Представить общие условия использования для всех пользователей в вашей организации.
- Представляем конкретные условия использования на основе атрибутов пользователя (напротив. для медсестер и врачей или местных и международных сотрудников, что выполняется с помощью [динамических групп](../users-groups-roles/groups-dynamic-membership.md));
- Представляем конкретные условия использования при доступе к приложениям с высоким влиянием на бизнес, таким как Salesforce.
- Современные условия использования на разных языках.
- Перечислите, кто принимал или не принимал в соответствии с вашими условиями использования.
- помогать в соблюдении законов о конфиденциальности;
- Отобразите журнал условий деятельности по использованию для соответствия и аудита.
- Создание и управление условиями использования с помощью [AIS Microsoft Graph](https://developer.microsoft.com/graph/docs/api-reference/beta/resources/agreement) (в настоящее время в предварительном просмотре).

## <a name="prerequisites"></a>Предварительные требования

Чтобы использовать и настроить условия использования Azure AD, необходимо:

- Подписка Azure AD Premium P1, P2, EMS E3 или EMS E5.
   - Если у вас нет одной из этих подписок, вы можете [получить подписку Azure AD Premium](../fundamentals/active-directory-get-started-premium.md) или [активировать пробную версию Azure AD Premium](https://azure.microsoft.com/trial/get-started-active-directory/).
- Одна из следующих учетных записей администратора для каталога, который нужно настроить.
   - глобального администратора;
   - Администратор безопасности
   - Администратор условного доступа

## <a name="terms-of-use-document"></a>Документ "Условия использования"

Условия использования Azure AD используют формат PDF для представления содержимого. Это может быть любое содержимое, например существующая контрактная документация, позволяющее собрать пользовательские соглашения во время входа пользователя. Для поддержки пользователей на мобильных устройствах в PDF-файлах рекомендуется использовать 24-й размер шрифта.

## <a name="add-terms-of-use"></a>Добавление условий использования

После завершения оформления документа о сроках использования используйте следующую процедуру, чтобы добавить его.

1. Войдите на портал Azure от имени глобального администратора, администратора безопасности или администратора условного доступа.
1. Перейдите к документу **Условия использования** в разделе [https://aka.ms/catou](https://aka.ms/catou).

   ![Условный доступ - Условия использования лезвия](./media/terms-of-use/tou-blade.png)

1. Щелкните **Добавить условия**.

   ![Новый термин использования панели для указания параметров условий использования](./media/terms-of-use/new-tou.png)

1. В поле **«Имя»** введите имя для терминов использования, которые будут использоваться на портале Azure.
1. В поле **Отображаемое имя** введите название, которое будет отображаться для пользователей при входе.
1. Для **документа «Условия использования»** просмотрите завершенные условия использования PDF и выберите его.
1. Выберите язык для документа, в котором используются условия использования. Настроив параметр языка, вы сможете передать несколько экземпляров условий использования на разных языках. Версия условий использования, отображаемая для пользователя, будет зависеть от настроек браузера.
1. Чтобы обязать конечных пользователей просматривать условия использования до их принятия, **установите Требование о том, чтобы пользователи расширили условия использования** до **On**.
1. Чтобы требовать от конечных пользователей принять ваши условия использования на каждом устройстве, с которого они имеют доступ, **установите Требование к пользователям дать согласие на каждое устройство** **на On.** Пользователям может потребоваться установить дополнительные приложения, если эта опция включена. Для получения дополнительной [информации см.](#per-device-terms-of-use)
1. Если вы хотите истечения срока действия соглашений об использовании в расписании, установите, что **expire дает согласие** на **On**. Когда этот параметр включен, отображаются два дополнительных параметра расписания.

   ![Настройки согласия истекают для установления даты начала, частоты и продолжительности](./media/terms-of-use/expire-consents.png)

1. Используйте настройки **expire, начиная** с и **частоты,** чтобы указать расписание сроков истечения срока действия использования. В следующей таблице показан результат нескольких примеров настроек.

   | Начало истечения срока действия | Частота | Результат |
   | --- | --- | --- |
   | Текущая дата  | Ежемесячная | Начиная с сегодняшнего дня, пользователи должны принимать условия использования, а затем повторно принимать каждый месяц. |
   | Дата в будущем  | Ежемесячная | Начиная с сегодняшнего дня, пользователи должны принять условия использования. Когда наступит указанная дата, срок действия согласия истечет и пользователи должны будут повторно принимать условия каждый месяц.  |

   Например, если задать срок действия с **1 января** и частоту **Ежемесячно**, то вот как может истекать срок действия принятия для двух пользователей.

   | Пользователь | Дата первого принятия | Дата первого истечения срока действия | Дата второго истечения срока действия | Дата третьего истечения срока действия |
   | --- | --- | --- | --- | --- |
   | Алиса | 1 января | 1 февраля | 1 марта | 1 апреля |
   | Владимир | 15 января | 1 февраля | 1 марта | 1 апреля |

1. Используйте **Продолжительность до повторного принятия требует (дней)** настройки, чтобы указать количество дней, прежде чем пользователь должен повторно принять условия использования. Это позволяет пользователям следовать собственному расписанию. Например, если установить период в **30** дней, то вот как может истекать срок действия принятия для двух пользователей.

   | Пользователь | Дата первого принятия | Дата первого истечения срока действия | Дата второго истечения срока действия | Дата третьего истечения срока действия |
   | --- | --- | --- | --- | --- |
   | Алиса | 1 января | 31 января | 2 марта | 1 апреля |
   | Владимир | 15 января | 14 февраля | 16 марта | 15 апреля |

   Можно использовать параметры **Истечение срока действия согласия** и **Duration before re-acceptance requires (days)** (Период до обязательного повторного принятия (дн)) вместе, но обычно используется только один из них.

1. При **условиях условного доступа**используйте список **шаблонов шаблонов шаблонов «Принуждение с условным доступом»,** чтобы выбрать шаблон для обеспечения соблюдения условий использования.

   ![Условный список выпадающих доступа для выбора шаблона политики](./media/terms-of-use/conditional-access-templates.png)

   | Шаблон | Описание |
   | --- | --- |
   | **Доступ к облачным приложениям для всех гостей** | Для всех гостей и всех облачных приложений будет создана политика условного доступа. Эта политика влияет на портал Azure. После ее создания может потребоваться выйти из системы, а затем снова войти. |
   | **Доступ к облачным приложениям для всех пользователей** | Для всех пользователей и всех облачных приложений будет создана политика условного доступа. Эта политика влияет на портал Azure. После ее создания потребуется выйти из системы, а затем снова войти. |
   | **Настраиваемая политика** | Выберите пользователей, группы и приложения, к которым будут применены эти условия использования. |
   | **Создание политики условного доступа позже** | Эти условия использования будут отображаться в списке контроля грантов при создании политики условного доступа. |

   >[!IMPORTANT]
   >Контроль политики условного доступа (включая условия использования) не поддерживает правоприменение в учетных записях служб. Рекомендуем исключить все учетные записи служб из политики условного доступа.

    Пользовательские политики условного доступа позволяют использовать детальные условия использования, вплоть до конкретного облачного приложения или группы пользователей. Для получения дополнительной информации [см.](require-tou.md)

1. Нажмите кнопку **Создать**.

   Если вы выбрали пользовательский шаблон условного доступа, появится новый экран, который позволяет создать пользовательскую политику условного доступа.

   ![Новая панель условного доступа, если вы выбрали пользовательский шаблон политики условного доступа](./media/terms-of-use/custom-policy.png)

   Теперь вы должны увидеть ваши новые условия использования.

   ![Новые условия использования, перечисленные в терминах использования лезвия](./media/terms-of-use/create-tou.png)

## <a name="view-report-of-who-has-accepted-and-declined"></a>Просмотр отчета о тех, кто принял или отклонил условия

В колонке "Условия использования" показано количество пользователей, которые приняли или отклонили условия использования. Эти подсчеты и те, кто принял / отказался хранятся на срок службы условий использования.

1. Войдите в Azure и перейдите к документу **Условия использования** в разделе [https://aka.ms/catou](https://aka.ms/catou).

   ![Условия использования лезвия с указанием числа пользователей шоу приняли и снизились](./media/terms-of-use/view-tou.png)

1. Для условий использования щелкните номера под **«Принято** или **отклонено»,** чтобы просмотреть текущее состояние для пользователей.

   ![Условия использования согласия панели список пользователей, которые приняли](./media/terms-of-use/accepted-tou.png)

1. Чтобы просмотреть журнал для отдельного пользователя, щелкните многоточие (**…**) и выберите **Просмотреть журнал**.

   ![Просмотр контекстного меню «История» для пользователя](./media/terms-of-use/view-history-menu.png)

   В области "Просмотр журнала" отобразится журнал всех принятий, отклонений и сроков действия.

   ![Панель просмотра истории перечисляет историю, которая принимается, отклоняется и истекает для пользователя](./media/terms-of-use/view-history-pane.png)

## <a name="view-azure-ad-audit-logs"></a>Просмотр журналов аудита Azure AD

Если требуется просмотреть дополнительные действия, условия использования Azure AD включают журналы аудита. Принятие условий каждым пользователем инициирует событие, которое регистрируется в журналах аудита и хранится в течение **30 дней**. Эти журналы можно просмотреть на портале или загрузить в виде CSV-файла.

Чтобы начать работу с журналами аудита Azure AD, выполните следующие действия.

1. Войдите в Azure и перейдите к документу **Условия использования** в разделе [https://aka.ms/catou](https://aka.ms/catou).
1. Выберите условия использования.
1. Нажмите кнопку **Просмотреть журналы аудита**

   ![Условия использования лезвия с выделенным вариантом журналов аудита View](./media/terms-of-use/audit-tou.png)

1. На экране "Журналы аудита" Azure AD можно фильтровать информацию с помощью списков, чтобы получить информацию журнала аудита для конкретного целевого объекта.

   Вы также можете скачать данные в CSV-файл для использования на локальном компьютере, щелкнув **Загрузить**.

   ![Аудит аудита Azure AD журналы даты перечисления экрана, целевая политика, инициированная и деятельность](./media/terms-of-use/audit-logs-tou.png)

   Если щелкнуть журнал, отобразится область с дополнительными сведениями о действиях.

   ![Сведения о действиях для журнала, показывающий активность, состояние действия, инициированное целевой политикой](./media/terms-of-use/audit-log-activity-details.png)

   ![Сайт MyApps с открытым панелью пользователя](./media/terms-of-use/tou14.png)

1. На странице профиля щелкните **Ознакомьтесь с условиями использования**.

   ![Страница профиля для пользователя, показывающая условия использования ссылки На обзор](./media/terms-of-use/tou13a.png)

1. Здесь можно просмотреть принятые вами условия использования.

## <a name="edit-terms-of-use-details"></a>Отсечение деталей использования

Вы можете отсеить некоторые сведения об условиях использования, но не можете изменить существующий документ. Ниже описано, как изменить эти сведения.

1. Войдите в Azure и перейдите к документу **Условия использования** в разделе [https://aka.ms/catou](https://aka.ms/catou).
1. Выберите условия использования, которые вы хотите отменить.
1. Щелкните **Изменить условия**.
1. В области "Изменение условий использования" измените значения параметров "Имя", "Отображаемое имя" или "Требовать, чтобы пользователи развернули условия использования".

   Если есть другие параметры, которые вы хотели бы изменить, такие как pdf-документ, требуют, чтобы пользователи давали согласие на каждое устройство, истекали согласия, продолжительность до повторного принятия или политика условного доступа, необходимо создать новые условия использования.

   ![Отсечение условий использования панели с указанием имени и расширение опций](./media/terms-of-use/edit-tou.png)

1. Нажмите кнопку **Сохранить**, чтобы сохранить изменения.

   Когда вы сохраните изменения, пользователям не потребуется принимать их.

## <a name="add-a-terms-of-use-language"></a>Добавление языка использования

Следующая процедура описывает, как добавить термины использования языка.

1. Войдите в Azure и перейдите к документу **Условия использования** в разделе [https://aka.ms/catou](https://aka.ms/catou).
1. Выберите условия использования, которые вы хотите отменить.
1. В области сведений щелкните вкладку **Языки**.

   ![Условия использования выбраны и отображаются вкладка Языки в деталях панели](./media/terms-of-use/languages-tou.png)

1. Щелкните **Добавить язык**.
1. В области "Добавление языка условий использования" передайте локализованный PDF-файл и выберите язык.

## <a name="per-device-terms-of-use"></a>Условия использования на устройстве

**Требование к пользователям дать согласие на каждую** настройку устройства позволяет требовать от конечных пользователей принимать ваши условия использования на каждом устройстве, с которого они имеют доступ. Конечный пользователь должен будет зарегистрировать свое устройство в Azure AD. Когда устройство зарегистрировано, идентификатор устройства используется для обеспечения соблюдения условий использования на каждом устройстве.

Ниже приведен список поддерживаемых платформ и программного обеспечения.

> [!div class="mx-tableFixed"]
> |  | iOS | Android | Windows 10 | Другие |
> | --- | --- | --- | --- | --- |
> | **Собственное приложение** | Да | Да | Да |  |
> | **Microsoft Edge** | Да | Да | Да |  |
> | **Internet Explorer** | Да | Да | Да |  |
> | **Chrome (с расширением)** | Да | Да | Да |  |

Условия использования на устройстве имеет следующие ограничения:

- устройство может быть присоединено только к одному клиенту;
- у пользователя должны быть разрешения на присоединение устройства;
- приложение для регистрации Intune не поддерживается.
- Пользователи Azure AD B2B не поддерживаются.

Если устройство пользователя не присоединено, он получит сообщение о том, что требуется присоединить устройство. Возможности взаимодействия будут зависеть от платформы и программного обеспечения.

### <a name="join-a-windows-10-device"></a>Присоединение устройства Windows 10

Если пользователь использует Windows 10 и Microsoft Edge, он получит сообщение, аналогичное приведенному ниже, предлагающее [присоединить устройство](../user-help/user-help-join-device-on-network.md#to-join-an-already-configured-windows-10-device).

![Windows 10 и Microsoft Edge - Сообщение с указанием вашего устройства должно быть зарегистрировано](./media/terms-of-use/per-device-win10-edge.png)

Если используется Chrome, то пользователю будет предложено установить [расширение для учетных записей Windows 10](https://chrome.google.com/webstore/detail/windows-10-accounts/ppnbnpeolgkicgegkbkbjmhlideopiji).

### <a name="register-an-ios-device"></a>Регистрация устройства iOS

Если пользователь использует устройство iOS, им будет предложено установить [приложение Microsoft Authenticator.](https://apps.apple.com/us/app/microsoft-authenticator/id983156458)

### <a name="register-an-android-device"></a>Регистрация Android-устройства

Если пользователь использует android-устройство, им будет предложено установить [приложение Microsoft Authenticator.](https://play.google.com/store/apps/details?id=com.azure.authenticator)

### <a name="browsers"></a>Браузеры

Если пользователь использует браузер, который не поддерживается, ему будет предложено использовать другой браузер.

## <a name="delete-terms-of-use"></a>Удаление условий использования

Вы можете удалить старые условия использования, используя следующую процедуру.

1. Войдите в Azure и перейдите к документу **Условия использования** в разделе [https://aka.ms/catou](https://aka.ms/catou).
1. Выберите условия использования, которые вы хотите удалить.
1. Нажмите кнопку **Удалить условия**.
1. В сообщении, которое появится на экране с вопросом о том, нужно ли продолжить, нажмите кнопку **Да**.

   ![Сообщение с просьбой о подтверждении для удаления условий использования](./media/terms-of-use/delete-tou.png)

   Вы больше не должны видеть ваши условия использования.

## <a name="deleted-users-and-active-terms-of-use"></a>Удаленные пользователи и активные условия использования

По умолчанию удаленный пользователь будет оставаться удаленным в Azure AD в течение 30 дней, на протяжении которых его при необходимости может восстановить администратор. По истечении 30 дней пользователь будет окончательно удален. Кроме того, с помощью портала Azure Active Directory глобальный администратор может явно [навсегда удалить недавно удаленного пользователя](../fundamentals/active-directory-users-restore.md) до истечения этого периода времени. Один пользователь был навсегда удален, последующие данные об этом пользователе будут удалены из активных условий использования. Сведения аудита об удаленных пользователях остаются в журнале аудита.

## <a name="policy-changes"></a>Изменения политики

Политики условного доступа всменяются немедленно. В этом случае администратор начнет видеть "грустные облака" или сообщение типа "Проблемы с токеном Azure AD". Администратору необходимо выйти и войти снова для удовлетворения требований новой политики.

> [!IMPORTANT]
> Охватываемым пользователям необходимо выйти и снова войти для удовлетворения требований новой политики, если:
>
> - политика условного доступа включена на условиях использования
> - созданы другие условия использования.

## <a name="b2b-guests-preview"></a>Гости B2B (предварительная версия)

Большинство организаций имеют процесс в месте для своих сотрудников, чтобы дать согласие на условия использования их организации и конфиденциальности заявления. Но как требовать аналогичное принятие условий от гостей B2B (бизнес-бизнес) в Azure AD, когда они добавляются с помощью SharePoint или Teams? Используя условный доступ и условия использования, вы можете применять политику непосредственно к гостевым пользователям B2B. Во время потока выкупа приглашений пользователю представлены условия использования. Сейчас эта возможность доступна в предварительной версии.

Условия использования отображаются только в том случае, если у пользователя есть гостевая учетная запись в Azure AD. SharePoint Online в настоящее время имеет [специальный внешний опыт получателя обмена](/sharepoint/what-s-new-in-sharing-in-targeted-release) для обмена документом или папкой, которая не требует от пользователя иметь учетную запись гостя. В этом случае условия использования не отображаются.

## <a name="frequently-asked-questions"></a>Часто задаваемые вопросы

**В: Как я могу видеть, когда/если пользователь принял условия использования?**<br />
Ответ. В колонке "Условия использования" щелкните число под надписью **Принято**. Вы также можете просматривать или выполнять поиск в журналах аудита Azure AD. Дополнительные сведения см. в разделах "Просмотр тех, кто принял или отклонил Условия использования" и [Просмотр журналов аудита Azure AD](#view-azure-ad-audit-logs).

**В. Как долго хранится информация?**<br />
О: Пользователь рассчитывает в терминах отчета об использовании и кто принял/отказался, которые хранятся в течение всего срока условий использования. Журналы аудита Azure AD хранятся в течение 30 дней.

**В: Почему я вижу различное количество согласований в терминах отчета об использовании по сравнению с журналами аудита Azure AD?**<br />
О: Отчет о сроках использования сохраняется в течение всего срока действия этого срока использования, в то время как журналы аудита Azure AD хранятся в течение 30 дней. Кроме того, отчет о сроках использования отображает только текущее состояние согласия пользователей. Например, если пользователь отклоняется, а затем принимает, условия отчета об использовании будут показывать только то, что пользователь принимает. Если нужно просмотреть историю, используйте журналы аудита Azure AD.

**В: Если я отдаю подробную информацию для условий использования, требуется ли пользователям принять их снова?**<br />
О: Нет, если администратор отменяет детали для терминов использования (имя, имя дисплея, требует, чтобы пользователи расширяли или добавляли язык), он не требует, чтобы пользователи повторно приняли новые термины.

**В: Могу ли я обновить существующий документ об использовании?**<br />
О: В настоящее время нельзя обновлять существующие условия использования документа. Чтобы изменить термины документа об использовании, необходимо создать новый экземпляр использования.

**В: Если гиперссылки находятся в терминах использования PDF-документа, смогут ли конечные пользователи нажать на них?**<br />
О: Да, конечные пользователи могут выбирать гиперссылки на дополнительные страницы, но ссылки на разделы в документе не поддерживаются.

**В: Может ли термин ы использования поддерживать несколько языков?**<br />
Ответ. Да. В настоящее время существует 108 различных языков, которые администратор может настроить для одного термина использования. Администратор может передать несколько PDF-файлов и пометить эти документы соответствующими тегами языков (до 108). Когда пользователь выполняет вход, мы определяем языковые параметры его браузера и отображаем соответствующий документ. Если совпадений нет, отображается документ по умолчанию, которым является первый переданный документ.

**В: Когда срабатывают условия использования?**<br />
О. Условия использования активируются, когда пользователь входит в систему.

**В. Для каких приложений можно применять условия использования?**<br />
О: С помощью современной аутентификации можно создать политику условного доступа для корпоративных приложений. Дополнительные сведения см. в статье [Просмотр всех корпоративных приложений, которыми вы можете управлять в Azure Active Directory](./../manage-apps/view-applications-portal.md).

**В. Можно ли добавить несколько экземпляров условий использования для определенного пользователя или приложения?**<br />
О: Да, путем создания нескольких политик условного доступа, ориентированных на эти группы или приложения. Если пользователь попадает в сферу действия нескольких терминов использования, он принимает по одному термину использования одновременно.

**В. Что произойдет, если пользователь отклонит условия использования?**<br />
О. Пользователь не сможет получить доступ к приложению. Чтобы получить доступ, ему необходимо будет повторно войти в систему и принять условия.

**Вопрос: Можно ли не принимать условия использования, которые были приняты ранее?**<br />
О: Вы можете [просмотреть ранее принятые условия использования,](#how-users-can-review-their-terms-of-use)но в настоящее время нет способа неприемлемости.

**В. Что произойдет, если использовать и условия Intune?**<br />
О: Если вы настроили как условия использования Azure AD, так и [условия Intune,](/intune/terms-and-conditions-create)пользователь должен будет принять и то, и другое. Дополнительные сведения см. в записи блога о [выборе правильного решения условий использования для организации](https://go.microsoft.com/fwlink/?linkid=2010506&clcid=0x409).

**В: Какие конечные точки используются для проверки подлинности?**<br />
О: Условия использования используют следующие конечные https://tokenprovider.termsofuse.identitygovernance.azure.com точки https://account.activedirectory.windowsazure.comдля проверки подлинности: и . Если в вашей организации есть список url-адресов для регистрации, необходимо добавить эти конечные точки в список разрешений, а также конечные точки Azure AD для входиный ввод.

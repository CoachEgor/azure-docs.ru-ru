# <a name="conditional-access-insights-and-reporting"></a>Аналитика и отчеты условного доступа

Книга условного доступа и отчет «отчеты» позволяют понять влияние политик условного доступа в Организации с течением времени. Во время входа в систему могут применяться одна или несколько политик условного доступа, которые предоставляют доступ, если определенные элементы управления Grant удовлетворяют или запрещают доступ иным образом. Поскольку во время каждого входа могут оцениваться несколько политик условного доступа, книга Insights и Reporting позволяет проверить влияние отдельной политики или подмножества всех политик.  

## <a name="prerequisites"></a>Предварительные условия

Чтобы включить аналитическую информацию и книгу отчетов, у клиента должна быть Рабочая область Log Analytics, чтобы хранить данные журналов входа. Для использования условного доступа пользователи должны иметь лицензии Azure AD Premium P1 или P2.

Следующие роли могут получать доступ к аналитическим данным и отчетам:  

- Администратор условного доступа 
- Читатель сведений о безопасности 
- Администратор безопасности. 
- Глобальный читатель 
- Глобальный администратор. 

Пользователям также требуется одна из следующих Log Analytics ролей рабочей области:  

- Читатель 
- Читатель мониторинга 
- Модуль чтения Log Analytics 
- Участник  
- Владелец 

### <a name="stream-sign-in-logs-from-azure-ad-to-azure-monitor-logs"></a>Потоковая передача журналов входа из Azure AD в журналы Azure Monitor 

Если журналы Azure AD не интегрированы с журналами Azure Monitor, необходимо выполнить следующие действия, прежде чем книга будет загружена:  

1. [Создайте рабочую область log Analytics в Azure Monitor](../../azure-monitor/learn/quick-create-workspace.md).
1. [Интегрируйте журналы Azure AD с журналами Azure Monitor](../reports-monitoring/howto-integrate-activity-logs-with-log-analytics.md).

## <a name="how-it-works"></a>Принцип работы 

Чтобы получить доступ к книге Insights и Reporting, сделайте следующее:  

1. Войдите на **портал Azure**.
1. Перейдите в **Azure Active Directory** > **Безопасность** > **условного доступа** > **и отчеты**.

### <a name="get-started-select-parameters"></a>Начало работы: выбор параметров 

Панель мониторинга "аналитика и отчетность" позволяет увидеть влияние одной или нескольких политик условного доступа за указанный период. Начните с установки каждого из параметров в верхней части книги. 

![Панель мониторинга условного доступа и отчетов в портал Azure](./media/howto-conditional-access-insights-reporting/conditional-access-insights-and-reporting-dashboard.png)

**Политика условного доступа**. Выберите одну или несколько политик условного доступа для просмотра их комбинированных последствий. Политики разделяются на две группы: включенные и политики только для отчетов. По умолчанию выбираются все включенные политики. Эти политики включения являются политиками, которые в настоящее время применяются в клиенте.  

**Диапазон времени**. Выберите диапазон времени от 4 часов до 90 дней. Если вы выберете диапазон времени позже, чем при интеграции журналов Azure AD с Azure Monitor, будут отображаться только входы после времени интеграции.  

**Пользователь**. по умолчанию на панели мониторинга отображается влияние выбранных политик для всех пользователей. Чтобы выполнить фильтрацию по отдельному пользователю, введите имя пользователя в текстовое поле. Чтобы выполнить фильтрацию по всем пользователям, введите "все пользователи" в текстовое поле или оставьте параметр пустым. 

**Приложение**. по умолчанию на панели мониторинга отображается влияние выбранных политик для всех приложений. Чтобы выполнить фильтрацию по отдельному приложению, введите имя приложения в текстовое поле. Чтобы выполнить фильтрацию по всем приложениям, введите "все приложения" в текстовое поле или оставьте параметр пустым. 

**Представление данных**: выберите, следует ли отображать результаты на панели мониторинга в соответствии с числом пользователей или количеством входов. У отдельного пользователя могут быть сотни входов во множество приложений с множеством различных результатов в течение заданного диапазона времени. Если выбрано представление данных для пользователей, то пользователь может входить как в счетчики успеха, так и из числа сбоев (например, если имеется 10 пользователей, 8 из них могли получить результат успеха за последние 30 дней и 9 из них могли получить результат сбоя за последние 30 дней).

## <a name="impact-summary"></a>Сводка по влиянию 

После установки параметров сводка по влиянию загружается. В сводке показано, сколько пользователей или входов в течение интервала времени привело к "успеху", "сбой", "требуется действие пользователя" или "не применено" при оценке выбранных политик.  

![Сводка по влиянию в книге условного доступа](./media/howto-conditional-access-insights-reporting/workbook-impact-summary.png)

**Всего**: количество пользователей или входов в систему за период времени, в течение которого была оценена хотя бы одна из выбранных политик.

**Успешно**. число пользователей или входов в систему за период времени, в течение которого Объединенный результат выбранных политик имел значение "успешно" или "только отчет: успешно".

**Сбой**. число пользователей или входов в систему за период времени, в течение которого по крайней мере одна из выбранных политик имеет значение "сбой" или "только отчет: сбой".

**Требуемое действие пользователя**: число пользователей или операций входа за период времени, в течение которого Объединенный результат выбранных политик был "только отчет: требуется вмешательство пользователя". Действие пользователя требуется, если для политики условного доступа только для отчетов требуется интерактивный контроль предоставления, например многофакторная идентификация. Так как интерактивные элементы управления предоставлением не применяются политиками только для отчетов, успех или сбой не могут быть определены.  

**Не применяется**: количество пользователей или входов в систему за период времени, в течение которого ни одна из выбранных политик не применена.

### <a name="understanding-the-impact"></a>Основные сведения о влиянии 

![Разбиение книги по условию и состоянию](./media/howto-conditional-access-insights-reporting/workbook-breakdown-condition-and-status.png)

Просмотр распределения пользователей или входов для каждого из условий. Вы можете отфильтровать операции входа определенного результата (например, успех или неудача), выбрав элемент сводки в верхней части книги. Вы можете увидеть разделение входов для каждого из условий условного доступа: состояние устройства, платформа устройства, клиентское приложение, расположение, приложение и риск входа.

## <a name="troubleshooting"></a>Устранение неполадок

### <a name="why-is-the-workbook-taking-a-long-time-to-load"></a>Почему книга занимает много времени?  

В зависимости от выбранного диапазона времени и размера клиента книга может оценить чрезвычайно большое количество событий входа. Для больших клиентов объем входов в систему может превысить емкость запросов Log Analytics. Попробуйте сократить диапазон времени до 4 часов, чтобы узнать, загружается ли книга.  

### <a name="after-loading-for-a-few-minutes-why-is-the-workbook-returning-zero-results"></a>Почему книга возвращает нулевые результаты после загрузки в течение нескольких минут? 

Если объем входов в систему превышает емкость запроса Log Analytics, книга вернет нулевые результаты. Попробуйте сократить диапазон времени до 4 часов, чтобы узнать, загружается ли книга.  

### <a name="can-i-save-my-parameter-selections"></a>Можно ли сохранить настройки параметров?  

Вы можете сохранить настройки параметров в верхней части книги, перейдя в **Azure Active Directory** > **книги** > **Условный доступ и отчеты**. Здесь можно найти шаблон книги, в котором можно изменить книгу и сохранить копию в рабочей области, включая параметры, выделенные в разделе **«Мои отчеты** » или « **Общие отчеты**». 

### <a name="can-i-edit-and-customize-the-workbook-with-additional-queries"></a>Можно ли редактировать и настраивать книгу с помощью дополнительных запросов? 

Вы можете изменить и настроить книгу, перейдя в **Azure Active Directory** > **книги** > **Условный доступ и отчеты**. Здесь можно найти шаблон книги, в котором можно изменить книгу и сохранить копию в рабочей области, включая параметры, выделенные в разделе **«Мои отчеты** » или « **Общие отчеты**». Чтобы начать редактирование запросов, нажмите кнопку **изменить** в верхней части книги.

# <a name="sign-in-activity-reports-in-the-azure-active-directory-portal"></a>Отчеты о действиях входа на портале Azure Active Directory

Архитектура создания отчетов в Azure Active Directory (Azure AD) состоит из следующих компонентов:

- **Действие** 
    - **События входа**. Информация об использовании управляемых приложений и действиях входа пользователей.
    - **Журналы аудита** - [Аудит](concept-audit-logs.md) предоставляют системную информацию о деятельности пользователей и управлении группами, управляемых приложениях и действиях каталогов.
- **Безопасность** 
    - **Рискованные входе** - [рискованный входе](concept-risky-sign-ins.md) является индикатором попытки входной связи кем-то, кто не является законным владельцем учетной записи пользователя.
    - **Пользователи, помеченные для риска** - [рискованный пользователь](concept-user-at-risk.md) является индикатором для учетной записи пользователя, которая может быть скомпрометирована.

В этой статье содержится обзор отчета о входе в систему.

## <a name="prerequisites"></a>Предварительные требования

### <a name="who-can-access-the-data"></a>Кто может получить доступ к данным?

* Пользователи в роли администратора безопасности, безопасности Reader, Global Reader и роли читателя отчетов
* Глобальные администраторы.
* Любой пользователь (не администратор) может получить доступ к своим данным о действиях входа. 

### <a name="what-azure-ad-license-do-you-need-to-access-sign-in-activity"></a>Какие лицензии Azure AD требуются для доступа к действию входа?

- Отчет о активности в входе доступен во [всех изданиях Azure AD](reference-reports-data-retention.md#how-long-does-azure-ad-store-the-data).

- Если вы хотите получить доступ к данным, всданным с помощью API, у арендатора должна быть связана лицензия [Azure Active Directory Premium.](../fundamentals/active-directory-get-started-premium.md)



## <a name="sign-ins-report"></a>Отчет о входе

Отчет о событиях входа пользователя содержит ответы на следующие вопросы:

* Что такое шаблон входа пользователя?
* Сколько пользователей входили в течение недели?
* Каков статус их входа?

В меню [портала Azure](https://portal.azure.com) выберите **Active Directory Azure**или ищите и выберите **Active Directory Azure** на любой странице.

![Выберите Azure Active Directory.](./media/concept-sign-ins/select-azure-active-directory.png "Azure Active Directory")

Под **мониторингом**выберите **входе,** чтобы открыть [отчет о входе.](https://portal.azure.com/#blade/Microsoft_AAD_IAM/ActiveDirectoryMenuBlade/SignIns)

![Активность регистрации](./media/concept-sign-ins/monitoring-sign-ins-in-azure-active-directory.png "Действия при входе")

Для показа некоторых записей о входе в систему может потребоваться до двух часов.

> [!IMPORTANT]
> В отчете о событиях входа отображаются только **интерактивные** события входа, то есть когда пользователь выполняет вход вручную, указывая имя пользователя и пароль. В этом отчете не отображаются неинтерактивные события входа, такие как аутентификация между службами. 

Журнал событий входа содержит представление списка по умолчанию, в котором отображаются:

- дата входа;
- пользователь;
- Приложение, в которое подписался пользователь
- состояние входа;
- состояние обнаружения риска;
- состояние требования многофакторной проверки подлинности (MFA).

![Активность регистрации](./media/concept-sign-ins/sign-in-activity.png "Действия при входе")

Вы можете настроить представление списка, нажав **столбцы** в панели инструментов.

![Активность регистрации](./media/concept-sign-ins/19.png "Действия при входе")

Диалог **столбцов** дает вам доступ к выбранным атрибутам. В отчете о всколых нельзя иметь поля, которые имеют более одного значения для данного запроса в в рамках входной системы в качестве столбца. Это, например, верно для данных аутентификации, данных условного доступа и местоположения сети.   

![Активность регистрации](./media/concept-sign-ins/columns.png "Действия при входе")

Чтобы получить более подробную информацию, выберите элемент в списке.

![Активность регистрации](./media/concept-sign-ins/basic-sign-in.png "Действия при входе")

> [!NOTE]
> Теперь клиенты могут устранить неполадки политики условного доступа через все отчеты о входе. Нажав на вкладку **«Условный доступ»** для записи записи, клиенты могут просмотреть статус условного доступа и погрузиться в детали политики, применяемых к входе в систему, и результат для каждой политики.
> Дополнительные сведения см. в разделе [Часто задаваемые вопросы об отчетах Azure Active Directory](reports-faq.md#conditional-access).



## <a name="filter-sign-in-activities"></a>Фильтрация действий входа

Во-первых, сужение данных до уровня, который работает для вас. Во-вторых, фильтруй данные о встыках, используя поле даты в качестве фильтра по умолчанию. Azure AD предоставляет широкий спектр дополнительных фильтров, которые вы можете установить:

![Активность регистрации](./media/concept-sign-ins/04.png "Действия при входе")

**Идентификатор запроса** - Идентификатор запроса, который вам небезразлимы.

**Пользователь** - Имя или основное имя пользователя (UPN) пользователя, который вам небезразлиявен.

**Приложение** - Название целевого приложения.
 
**Статус** - Статус ввески, о вас небезразлияно:

- Успешно

- Failure

- Прервана


**IP-адрес** - IP-адрес устройства, используемого для подключения к арендатору.

**Расположение** - Местоположение, где соединение было инициировано из:

- Город

- Государство / Провинция

- Страна/регион


**Ресурс** - Название службы, используемой для ввоза.


**Идентификатор ресурса** - Идентификатор службы, используемой для входе в систему.

- **Не применяется**: Политика не применяется к пользователю и приложению при входе в систему.

- **Успех**: Одна или несколько условных политик доступа, применяемых к пользователю и приложению (но не обязательно другие условия) во время входного ввена. 

- **Неудача**: Одна или несколько условных политик доступа применяется и не была удовлетворена во время входного ввена.

## <a name="sign-ins-data-shortcuts"></a>Точки для получения данных о входе

Azure AD и портал Azure предоставляют вам дополнительные точки входа для входа в систему:

- Обзор защиты удостоверений
- Пользователи
- Группы
- корпоративные приложения.

### <a name="users-sign-ins-data-in-identity-security-protection"></a>Данные событий входа пользователей в разделе защиты удостоверений

На графике входной связи пользователя на странице **обзора защиты безопасности identity** показывает еженедельные скопления всхываек. По умолчанию для периода времени составляет 30 дней.

![Активность регистрации](./media/concept-sign-ins/06.png "Действия при входе")

Щелкнув день на графике входов, вы увидите обзор действий входа за этот день.

Каждая строка в списке действий входа содержит такие сведения:

* Кто выполнил вход?
* Какое приложение было целью входа?
* Каково состояние входа в систему?
* Каково состояние MFA входа в систему?

Щелкнув элемент, вы получите более подробную информацию об операции входа:

- Идентификатор пользователя.
- Пользователь
- Имя пользователя
- Идентификатор приложения
- Приложение
- "Клиент";
- Расположение
- IP-адрес
- Дата
- Требуется многофакторная идентификация
- состояние входа.

> [!NOTE]
> IP-адреса выдаются таким образом, что определенная связь между IP-адресом и физическим расположением компьютера с этим адресом отсутствует. Сопоставление IP-адресов осложняется тем, что поставщики мобильной связи и виртуальные частные сети выдают IP-адреса из центральных пулов, которые нередко находятся очень далеко от места, где фактически используется клиентское устройство. В настоящее время в отчетах Azure AD преобразование IP-адреса в физическое расположение не гарантируется и осуществляется на основе трассировок, данных реестра, обратных просмотров и других сведений.

На странице **Пользователи** можно получить полный обзор всех входов пользователей, щелкнув **Вход в систему** в разделе **Действия**.

![Активность регистрации](./media/concept-sign-ins/08.png "Действия при входе")

## <a name="usage-of-managed-applications"></a>Использование управляемых приложений

Представление данных входа, ориентированное на приложения, позволяет ответить на такие вопросы:

* Кто использует мои приложения?
* Каковы три основных приложения в вашей организации?
* Как делается мое новейшее приложение?

Точка входа в эти данные — три лучших приложения в организации. Данные содержатся в отчете за последние 30 дней в разделе **Обзор** **приложений Enterprise.**

![Активность регистрации](./media/concept-sign-ins/10.png "Действия при входе")

Использование приложений графики еженедельных агрегаций входе в систему для вашего топ-три приложений в данный период времени. который по умолчанию составляет 30 дней.

![Активность регистрации](./media/concept-sign-ins/graph-chart.png "Действия при входе")

Если нужно, вы можете переместить фокус на определенное приложение.

![Отчетности](./media/concept-sign-ins/single-app-usage-graph.png "Отчеты")

Щелкнув день на графике использования приложений, вы увидите подробный список действий входа.

С помощью параметра **Входов** можно полностью отобразить все события входа в ваши приложения.

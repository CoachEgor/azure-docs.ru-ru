# <a name="usage-and-insights-report-in-the-azure-active-directory-portal"></a>Отчет об использовании и на портале Azure Active Directory

С помощью отчета об использовании и можно получить представление данных входа в систему на уровне приложения. Чтобы получить ответы на следующие вопросы:

*   Каковы верхней используемых приложений в моей организации?
*   Какие приложения есть количеством неудачных входов в систему? 
*   Каковы самые распространенные ошибки входа в систему для каждого приложения?

## <a name="prerequisites"></a>Технические условия 

Чтобы доступ к данным об использовании и отчет, вам потребуется:

* клиент Azure AD;
* Лицензия Azure AD premium (P1 и P2), чтобы просмотреть данные входа в систему
* Пользователь в глобального администратора, администратора безопасности, читатель безопасности или роли читателей отчета. Кроме того любой пользователь (не администраторам) можно получить доступ к свои собственные входы в систему. 

## <a name="access-the-usage-and-insights-report"></a>Доступ к отчету об использовании и

1. Перейдите на [портал Azure](https://portal.azure.com).
2. Выберите правильный каталог, а затем **Azure Active Directory** и выберите **корпоративные приложения**.
3. Из **действия** выберите **использования & insights** для открытия отчета. 

![Отчет об использовании и](./media/concept-usage-insights-report/main-menu.png)
                                     

## <a name="use-the-report"></a>С помощью отчета

Отчет об использовании и отображает список приложений с помощью одного или нескольких входа на попытки и позволяет сортировать по числу успешных входов, неудачных входов и Доля успешных попыток.

Нажав кнопку load более в нижней части списка позволяет просматривать дополнительные приложения на странице. Можно выбрать диапазон дат, чтобы просмотреть все приложения, которые использовались в пределах диапазона.

Можно также установить фокус на определенное приложение. Выберите **просмотреть действия при входе в** для см. в разделе входа в действии со временем приложения, а также самые распространенные ошибки.  

Выбрать день на графике использования приложений, появится подробный список действий входа для приложения.

# <a name="audit-activity-reports-in-the-azure-active-directory-portal"></a>Отчеты о действиях аудита на портале Azure Active Directory 

Функция отчетов в Azure Active Directory (Azure AD) позволяет получать всю необходимую информацию о работе среды.

Архитектура отчета состоит из следующих компонентов:

- **Действие** 
    - **Входы**. [Отчет о входах](concept-sign-ins.md) содержит информацию об использовании управляемых приложений и входах пользователей.
    - **Журналы аудита** — возможность отслеживания с помощью журналов всех изменений, внесенных при использовании разных функций в Azure AD. Примеры журналов аудита включают изменения, внесенные в такие ресурсы в Azure AD, как добавление или удаление пользователей, приложений, групп, ролей и политик.
- **Безопасность** 
    - **Рискованные входе** - [рискованный входе](concept-risky-sign-ins.md) является индикатором попытки входной связи, которая могла быть выполнена кем-то, кто не является законным владельцем учетной записи пользователя. 
    - **Пользователи, помеченные для риска** - [рискованный пользователь](concept-user-at-risk.md) является индикатором для учетной записи пользователя, которая может быть скомпрометирована.

В этом разделе содержатся общие сведения об отчете аудита.
 
## <a name="who-can-access-the-data"></a>Кто может получить доступ к данным?

* Пользователи в **функции администратора безопасности,** **reader security,** **Reader Report** Reader, **Глобальный читатель** или роли **глобального администратора**

## <a name="audit-logs"></a>Журналы аудита

Журналы аудита Azure AD содержат записи о действиях системы (необходимые для соответствия требованиям). Чтобы получить доступ к отчету о аудите, выберите **журналы аудита** в разделе **Мониторинг** **Active Directory.** Обратите внимание, что журналы аудита могут иметь задержку до часа, поэтому данные аудиторской деятельности могут занять много времени после того, как вы выполнили задачу.



Журнал аудита содержит представление списка по умолчанию, в котором отображаются:

- дата и время выполнения действия;
- служба, которая зарегистрировала возникновение
- категории и названия деятельности *(что*) 
- статус деятельности (успех или неудача)
- целевой объект.
- инициатор или субъект (кто?) действия;

![Журналы аудита](./media/concept-audit-logs/listview.png "Журналы аудита")

Вы можете настроить представление списка, нажав **столбцы** в панели инструментов.

![Журналы аудита](./media/concept-audit-logs/columns.png "Журналы аудита")

Здесь вы можете добавить или удалить отображаемые поля.

![Журналы аудита](./media/concept-audit-logs/columnselect.png "Журналы аудита")

Чтобы получить более подробную информацию, выберите элемент в списке.

![Журналы аудита](./media/concept-audit-logs/details.png "Журналы аудита")


## <a name="filtering-audit-logs"></a>Фильтрация журналов аудита

Данные аудита можно отфильтровать в следующих полях:

- Служба
- Категория
- Действие
- Состояние
- Назначение
- инициатор (субъект);
- Диапазон даты

![Журналы аудита](./media/concept-audit-logs/filter.png "Журналы аудита")

Фильтр **службы** позволяет выбрать из списка выпадающих ниже следующих услуг:

- All
- AAD Менеджмент UX
- Проверки доступа
- "Account Provisioning" (Подготовка учетных записей).
- Прокси приложения
- Способы проверки подлинности
- B2C
- Условный доступ
- "Core Directory" (Основной каталог);
- Управление правами
- Гибридная аутентификация
- Защита идентификации
- Invited Users (Приглашаемые пользователи)
- MIM Service (Служба MIM)
- MyApps
- PIM
- "Self-service Group Management" (Самостоятельное управление группами);
- "Self-service Password Management" (Самостоятельное управление паролями);
- Условия использования

Фильтр **категории** позволяет выбрать один из следующих фильтров:

- All
- AdministrativeUnit
- ApplicationManagement
- Проверка подлинности
- Авторизация
- Контакт
- Устройство
- DeviceConfiguration
- Управление каталогом
- Управление правом
- Групповое управление
- КерберосДойм
- KeyManagement
- Метка
- Другие
- PermissionGrantPolicy
- Политика
- Ресурсоуправление
- Ролевой менеджмент
- Управление пользователями

Фильтр **активности** основан на выделении категории и типа ресурсов деятельности. Вы можете выбрать определенное действие или просмотреть все. 

Вы можете получить список всех аудиторских действий с помощью API Графика:`https://graph.windows.net/<tenantdomain>/activities/auditActivityTypesV2?api-version=beta`

Фильтр **«Статус»** позволяет фильтровать в зависимости от состояния операции аудита. Статус может быть одним из следующих:

- All
- Успешно
- Failure

**Целевой** фильтр позволяет искать конкретную цель с момента начала имени или главного имени пользователя (UPN). Целевое имя и UPN чувствительны к случаям. 

**Инициированный фильтром** позволяет определить, с чего начинается имя актера или универсальное главное имя (UPN). Имя и UPN чувствительны к делу.

Фильтр **диапазона Дат** позволяет определить временные рамки для возвращенных данных.  
Возможны следующие значения:

- 7 дней
- 24 часа
- Другой

При выборе пользовательского интервала времени можно настроить время начала и окончания.

Вы также можете загрузить отфильтрованные данные, до 250 000 записей, выбрав кнопку **Загрузка.** Вы можете скачать журналы в формате CSV или JSON. На число записей, которые можно скачать, влияют особенности [политики хранения отчетов Azure Active Directory](reference-reports-data-retention.md).

![Журналы аудита](./media/concept-audit-logs/download.png "Журналы аудита")

## <a name="audit-logs-shortcuts"></a>Ярлыки журналов аудита

Помимо **Azure Active Directory** портал Azure предоставляет две дополнительные точки входа для данных аудита:

- Пользователи и группы
- корпоративные приложения.

### <a name="users-and-groups-audit-logs"></a>Журналы аудита пользователей и групп

Отчеты аудита, касающиеся пользователей и групп, дают возможность ответить на такие вопросы:

- Обновления каких типов были применены к пользователям?

- Сколько пользователей было изменено?

- Сколько паролей было изменено?

- Что делал администратор в каталоге?

- Какие группы были добавлены?

- В каких группах произошли изменения членства?

- Изменены ли владельцы групп?

- Какие лицензии были назначены пользователю или группе?

Если вы хотите просмотреть только аудиторские данные, связанные с пользователями, вы можете найти отфильтрованное представление в **журналах аудита** в разделе **Мониторинг** вкладки **пользователей.** Эта точка входа имеет **UserManagement** как заранее выбранную категорию.

![Журналы аудита](./media/concept-audit-logs/users.png "Журналы аудита")

Если вы хотите просмотреть только аудиторские данные, связанные с группами, можно найти отфильтрованное представление в **журналах аудита** в разделе **«Мониторинг»** вкладки **Группы.** Эта точка входа имеет **GroupManagement** как заранее выбранную категорию.

![Журналы аудита](./media/concept-audit-logs/groups.png "Журналы аудита")

### <a name="enterprise-applications-audit-logs"></a>Журналы аудита корпоративных приложений

Отчеты аудита, касающиеся приложений, дают возможность ответить на такие вопросы:

* Какие приложения были добавлены или обновлены?
* Какие приложения были удалены?
* Изменилcя ли субъект-служба для приложения?
* Изменены ли имена приложений?
* Кто дал согласие на использование приложения?

Если нужно просмотреть данные аудита, связанные с приложениями, отфильтрованное представление вы можете найти, щелкнув элемент **Журналы аудита** в разделе **Действие** колонки **Корпоративные приложения**. Эта точка входа имеет **корпоративные приложения,** предварительно выбранные в качестве **типа приложения.**

![Журналы аудита](./media/concept-audit-logs/enterpriseapplications.png "Журналы аудита")

## <a name="office-365-activity-logs"></a>Журналы действий Office 365

Вы можете просмотреть журналы активности Office 365 из [центра обработки данных Microsoft 365.](https://docs.microsoft.com/office365/admin/admin-overview/about-the-admin-center) Несмотря на то, что деятельность Office 365 и журналы активности Azure AD имеют много ресурсов каталога, только центр админ-ресурсов Microsoft 365 предоставляет полное представление журналов деятельности Office 365. 

Вы также можете получить доступ к журналу активности Office 365 программно, используя [AIS Office 365 Management.](https://docs.microsoft.com/office/office-365-management-api/office-365-management-apis-overview)

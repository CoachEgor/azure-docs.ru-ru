# <a name="how-to-manage-inactive-user-accounts-in-azure-ad"></a>Руководство. управление неактивными учетными записями пользователей в Azure AD

В больших средах учетные записи пользователей не всегда удаляются, когда сотрудники оставляют организацию. ИТ-администратору нужно обнаружить и обработать эти устаревшие учетные записи пользователей, так как они представляют угрозу безопасности.

В этой статье описывается метод для управления устаревшими учетными записями пользователей в Azure AD. 

## <a name="what-are-inactive-user-accounts"></a>Что такое неактивные учетные записи пользователей?

Неактивные учетные записи — это учетные записи пользователей, которые больше не требуются членам Организации для получения доступа к ресурсам. Один идентификатор ключа для неактивных учетных записей заключается в том, что они не использовались в *течение* длительного входа в среду. Так как неактивные учетные записи привязаны к действию входа, можно использовать отметку времени последнего входа, который успешно обнаружил их. 

Задача этого метода заключается в том, чтобы определить *,* что следует делать в случае использования среды. Например, пользователи могут не входить в среду в *течение определенного*времени, так как они находятся в отпуске. При определении изменений для неактивных учетных записей пользователей необходимо учитывать все возможные причины отсутствия входа в среду. Во многих организациях разница между неактивными учетными записями пользователей составляет от 90 до 180 дней. 

Последний успешный вход в систему позволяет получить ценные сведения о возможном доступе пользователя к ресурсам.  Он поможет определить, требуется ли по-прежнему членство в группе или доступ к приложению, или удалить его. Для управления внешними пользователями можно определить, активен ли внешний пользователь в клиенте или он должен быть очищен. 

    
## <a name="how-to-detect-inactive-user-accounts"></a>Как обнаружить неактивные учетные записи пользователей

Определить неактивные учетные записи можно, выполнив оценку свойства **ластсигниндатетиме** , предоставленного типом ресурса **сигнинактивити** API **Microsoft Graph** . С помощью этого свойства можно реализовать решение для следующих сценариев:

- **Пользователи по имени**. в этом сценарии вы ищете конкретного пользователя по имени, который позволяет оценить ластсигниндате:`https://graph.microsoft.com/beta/users?$filter=startswith(displayName,'markvi')&$select=displayName,signInActivity`

- **Пользователи по дате**: в этом сценарии вы запрашиваете список пользователей с ластсигниндатетиме до указанной даты:`https://graph.microsoft.com/beta/users?filter=signInActivity/lastSignInDateTime le 2019-06-01T00:00:00Z`






## <a name="what-you-need-to-know"></a>Что необходимо знать

В этом разделе перечислены необходимые сведения о свойстве Ластсигниндатетиме.

### <a name="how-can-i-access-this-property"></a>Как получить доступ к этому свойству?

Свойство **ластсигниндатетиме** предоставляется [типом ресурса сигнинактивити](https://docs.microsoft.com/graph/api/resources/signinactivity?view=graph-rest-beta) [REST API Microsoft Graph](https://docs.microsoft.com/graph/overview?view=graph-rest-beta#whats-in-microsoft-graph).   

### <a name="is-the-lastsignindatetime-property-available-through-the-get-azureaduser-cmdlet"></a>Доступно ли свойство Ластсигниндатетиме в командлете Get-AzureAdUser?

Нет.

### <a name="what-edition-of-azure-ad-do-i-need-to-access-the-property"></a>Какой выпуск Azure AD необходим для доступа к свойству?

Доступ к этому свойству можно получить во всех выпусках Azure AD.

### <a name="what-permission-do-i-need-to-read-the-property"></a>Какое разрешение необходимо для чтения свойства?

Чтобы прочитать это свойство, необходимо предоставить следующие права: 

- AuditLogs. Read. ALL
- Организация. чтение. все  


### <a name="when-does-azure-ad-update-the-property"></a>Когда Azure AD обновляет свойство?

Каждый интерактивный вход, который прошел успешно, приводит к обновлению базового хранилища данных. Как правило, успешные попытки входа отображаются в соответствующем отчете на входе в течение 10 минут.
 

### <a name="what-does-a-blank-property-value-mean"></a>Что означает пустое значение свойства?

Для создания метки времени Ластсигниндатетиме требуется успешный вход. Поскольку свойство Ластсигниндатетиме является новой функцией, значение свойства Ластсигниндатетиме может быть пустым, если:

- Последний успешный вход пользователя выполнялся до выпуска этой функции (1 декабря 2019 г.).
- Затронутая учетная запись пользователя никогда не использовалась для успешного входа.

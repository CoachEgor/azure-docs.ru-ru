# <a name="how-to-use-azure-monitor-workbooks-for-azure-active-directory-reports"></a>Как использовать Azure Monitor книг отчетов Azure Active Directory

Вы хотите:

- Эффект вашей [политики условного доступа](../conditional-access/overview.md) на пользователей вход в систему?

- Устранении неполадок входа в систему для получения более полное представление о работоспособности входа вашей организации и быстрое устранение проблем?

- Узнать, кто использует устаревший проверок подлинности для входа в вашей среде? (По [блокировки проверки подлинности](../conditional-access/block-legacy-authentication.md), можно улучшить защиту вашего клиента.)

Чтобы помочь пользователям решить эти задачи, Active Directory предоставляет книг для мониторинга. [Azure Monitor книг](https://docs.microsoft.com/azure/azure-monitor/app/usage-workbooks) объединить текст, запросы аналитики, метрик и параметров в подробные интерактивные отчеты. 

В этой статье:

- Предполагается, что вы знакомы с работой [Создавайте интерактивные отчеты с помощью монитора книг](https://docs.microsoft.com/azure/azure-monitor/app/usage-workbooks).

- Объясняется, как использовать монитор книги, чтобы понять влияние политики условного доступа, устранение неполадок при сбоях входа в систему и для определения устаревших проверок подлинности.
 


## <a name="prerequisites"></a>Технические условия

Чтобы использовать книги, монитор, вам потребуется:

- Клиент Active Directory с лицензией premium (P1 или P2). Узнайте, как [получить лицензию уровня "премиум"](https://docs.microsoft.com/azure/active-directory/fundamentals/active-directory-get-started-premium).

- Объект [рабочей области Log Analytics](https://docs.microsoft.com/azure/azure-monitor/learn/quick-create-workspace).

## <a name="workbook-access"></a>Доступа к книге 

Для доступа к книгам:

1. Войдите на [портале Azure](https://portal.azure.com).

2. В области навигации слева выберите **Azure Active Directory**.

3. В **мониторинг** выберите **Insights**. 

    ![Выберите Insights](./media/howto-use-azure-monitor-workbooks/41.png)

4. Выберите отчет или шаблон, или на панели инструментов выберите **откройте**. 

    ![Выберите открыть](./media/howto-use-azure-monitor-workbooks/42.png)


## <a name="sign-in-analysis"></a>Анализ входа в систему

Для доступа к книге анализа входа в систему, в **использования** выберите **входы в систему**. 

Эта книга отображаются следующие тенденции входа в систему:

- Все входы в систему

- Успешно

- Действие ожидающих пользователей

- Сбой

Вы можете отфильтровать каждого тренда по следующим категориям:

- Диапазон времени

- Приложения

- Пользователи

![Анализ входа в систему](./media/howto-use-azure-monitor-workbooks/43.png)


Для каждого тенденции получить разбивку по следующим категориям:

- Расположение

    ![Входы в систему по расположению](./media/howto-use-azure-monitor-workbooks/45.png)

- Устройство

    ![Входы в систему каждого устройства](./media/howto-use-azure-monitor-workbooks/46.png)

Эта книга отображаются следующие тенденции входа в систему:

- Все входы в систему

- Успешно


Вы можете отфильтровать каждого тренда по следующим категориям:

- Диапазон времени

- Приложения

- Пользователи

- Протоколы

![Входы в систему с помощью проверки подлинности прежних версий](./media/howto-use-azure-monitor-workbooks/47.png)


Для каждого тенденции вы получаете декомпозиция приложения и протокола.

![Проверка подлинности для прежних версий входы в систему, приложения и протокола](./media/howto-use-azure-monitor-workbooks/48.png)



## <a name="sign-ins-by-conditional-access"></a>Входы в систему с помощью условного доступа 


Для доступа к книге для входа в систему с [политики условного доступа](../conditional-access/overview.md)в **условного доступа** выберите **входы в систему с помощью условного доступа**. 

Эта книга отображаются тенденции для отключенного входы в систему. Вы можете отфильтровать каждого тренда по следующим категориям:

- Диапазон времени

- Приложения

- Пользователи

![Операции входа в систему с использованием условного доступа](./media/howto-use-azure-monitor-workbooks/49.png)


Для отключенного входы в систему вы получаете декомпозиция по состоянию условного доступа.

![Состояние условного доступа](./media/howto-use-azure-monitor-workbooks/conditional-access-status.png)

Эта книга отображаются следующие отключенные вход тенденции:

- Требовать заявление о конфиденциальности

- Другие


Вы можете отфильтровать каждого тренда по следующим категориям:

- Диапазон времени

- Приложения

- Пользователи

![Входы в систему, элементов управления предоставлением прав](./media/howto-use-azure-monitor-workbooks/50.png)

![Декомпозиция последние входы в систему](./media/howto-use-azure-monitor-workbooks/51.png)




## <a name="sign-ins-failure-analysis"></a>Анализ сбоя входы в систему

Используйте **Анализ сбоя входы в систему** книги для устранения неполадок на следующий:

- Вход в систему
- Политики условного доступа
- Устаревшие способы аутентификации 


Для доступа к входами условного доступа к данным, в **Устранение** выберите **входы в систему с использованием проверки подлинности для прежних версий**. 

Эта книга отображаются следующие тенденции входа в систему:

- Все входы в систему

- Успешно

- Незавершенное действие

- Сбой


Вы можете отфильтровать каждого тренда по следующим категориям:

- Диапазон времени

- Приложения

- Пользователи

![Устранение неполадок входы в систему](./media/howto-use-azure-monitor-workbooks/52.png)


Для устранения входы в систему, Azure Monitor позволяет выбрать распределение по следующим категориям:

- Самые распространенные ошибки

    ![Сводка по самые распространенные ошибки](./media/howto-use-azure-monitor-workbooks/53.png)

- Входы в систему ожидает действий пользователя

    ![Сводка по входы в систему ожидает действий пользователя](./media/howto-use-azure-monitor-workbooks/54.png)

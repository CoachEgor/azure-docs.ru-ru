# <a name="how-to-use-azure-monitor-workbooks-for-azure-active-directory-reports"></a>Использование книг Azure Monitor для Azure Active Directory отчетов

Вы хотите:

- Понимаете последствия [политик условного доступа](../conditional-access/overview.md) для входа пользователей в систему?

- Устраните ошибки входа, чтобы получить лучшее представление о работоспособности входа в Организации и быстро устранить проблемы?

- Узнайте, кто использует устаревшие проверки подлинности для входа в вашу среду. ( [Блокировка устаревшей проверки](../conditional-access/block-legacy-authentication.md)подлинности позволяет улучшить защиту клиента.)

Чтобы помочь вам решить эти вопросы, Active Directory предоставляет книги для мониторинга. [Azure Monitor книгах](https://docs.microsoft.com/azure/azure-monitor/app/usage-workbooks) объединяет текст, аналитические запросы, метрики и параметры в расширенные Интерактивные отчеты. 

В этой статье:

- Предполагается, что вы знакомы с [созданием интерактивных отчетов с помощью Monitoring книги](https://docs.microsoft.com/azure/azure-monitor/app/usage-workbooks).

- В этой статье объясняется, как использовать рабочие книги для отслеживания влияния политик условного доступа, устранения неполадок при входе и определения устаревших проверок подлинности.
 


## <a name="prerequisites"></a>предварительные требования

Для использования рабочих книг монитора необходимо следующее:

- Клиент Active Directory с лицензией Premium (P1 или P2). Узнайте, как [получить лицензию Premium](https://docs.microsoft.com/azure/active-directory/fundamentals/active-directory-get-started-premium).

- [Рабочая область log Analytics](https://docs.microsoft.com/azure/azure-monitor/learn/quick-create-workspace).

## <a name="workbook-access"></a>Доступ к книге 

Для доступа к книгам:

1. Войдите на [портале Azure](https://portal.azure.com).

2. В области навигации слева выберите **Azure Active Directory**.

3. В разделе **мониторинг** выберите пункт **книги**. 

    ![Выбор аналитических сведений](./media/howto-use-azure-monitor-workbooks/41.png)

4. Выберите отчет или шаблон или нажмите кнопку **Открыть**на панели инструментов. 

    ![Выберите Открыть.](./media/howto-use-azure-monitor-workbooks/42.png)


## <a name="sign-in-analysis"></a>Анализ входов

Чтобы получить доступ к книге анализ входов, в разделе **Использование** выберите входы. 

В этой книге показаны следующие тенденции входа:

- Все входы в систему

- Success

- Ожидается действие пользователя

- Сбой

Можно отфильтровать каждую тенденцию по следующим категориям:

- Диапазон времени

- Приложения

- Пользователи

![Анализ входов](./media/howto-use-azure-monitor-workbooks/43.png)


Для каждого тренда вы получаете декомпозицию по следующим категориям:

- Местоположение

    ![Входы в систему по расположению](./media/howto-use-azure-monitor-workbooks/45.png)

- Устройство

    ![Входы по устройствам](./media/howto-use-azure-monitor-workbooks/46.png)

В этой книге показаны следующие тенденции входа:

- Все входы в систему

- Success


Можно отфильтровать каждую тенденцию по следующим категориям:

- Диапазон времени

- Приложения

- Пользователи

- Протоколы

![Входы с помощью устаревшей проверки подлинности](./media/howto-use-azure-monitor-workbooks/47.png)


Для каждого тренда вы получаете декомпозицию по приложениям и протоколу.

![Устаревшие входы с проверкой подлинности по приложениям и протоколу](./media/howto-use-azure-monitor-workbooks/48.png)



## <a name="sign-ins-by-conditional-access"></a>Входы в систему по условному доступу 


Чтобы получить доступ к книге для входа с помощью [политик условного](../conditional-access/overview.md)доступа, в разделе **Условный доступ** выберите **Входы по условному доступу**. 

Эта книга показывает тенденции отключенных входов в систему. Можно отфильтровать каждую тенденцию по следующим категориям:

- Диапазон времени

- Приложения

- Пользователи

![Операции входа в систему с использованием условного доступа](./media/howto-use-azure-monitor-workbooks/49.png)


Для отключенных входов вы получаете подразделение по состоянию условного доступа.

![Состояние условного доступа](./media/howto-use-azure-monitor-workbooks/conditional-access-status.png)








## <a name="sign-ins-by-grant-controls"></a>Входы с помощью предоставления элементов управления

Чтобы получить доступ к книге для входа с помощью [предоставления элементов управления](../conditional-access/controls.md), в разделе **Условный доступ** выберите **входы с помощью элементов управления Grant**. 

В этой книге показаны следующие отключенные тенденции входа:

- Требовать многофакторную идентификацию
 
- Обязательное применение условий использования

- Требовать заявление о конфиденциальности

- Другой


Можно отфильтровать каждую тенденцию по следующим категориям:

- Диапазон времени

- Приложения

- Пользователи

![Входы с помощью предоставления элементов управления](./media/howto-use-azure-monitor-workbooks/50.png)


Для каждого тренда вы получаете декомпозицию по приложениям и протоколу.

![Разбивка последних входов в систему](./media/howto-use-azure-monitor-workbooks/51.png)




## <a name="sign-ins-failure-analysis"></a>Анализ сбоев входа

Используйте книгу **анализ** неудачных погрешностей при входе, чтобы устранить ошибки следующим образом.

- Входов
- Политики условного доступа
- Устаревший механизм проверки подлинности 


Чтобы получить доступ к входам с помощью данных условного доступа, в разделе " **Устранение неполадок** " выберите **входы с использованием устаревшей проверки**подлинности. 

В этой книге показаны следующие тенденции входа:

- Все входы в систему

- Success

- Незавершенное действие

- Сбой


Можно отфильтровать каждую тенденцию по следующим категориям:

- Диапазон времени

- Приложения

- Пользователи

![Устранение неполадок при входе](./media/howto-use-azure-monitor-workbooks/52.png)


Чтобы помочь в устранении неполадок при входе, Azure Monitor предоставляет подразделение по следующим категориям:

- Основные ошибки

    ![Сводка основных ошибок](./media/howto-use-azure-monitor-workbooks/53.png)

- Входы в систему в ожидании действия пользователя

    ![Сводка по входу в систему в ожидании действия пользователя](./media/howto-use-azure-monitor-workbooks/54.png)

# <a name="how-to-use-azure-monitor-workbooks-for-azure-active-directory-reports"></a>Как использовать рабочие книги Azure Monitor для отчетов об активных каталогах Azure

> [!IMPORTANT]
> Для оптимизации основных запросов в этой рабочей книге, пожалуйста, нажмите на кнопку "Edit", нажмите на значок "Настройки" и выберите рабочее пространство, где вы хотите запустить эти запросы. В учебниках по умолчанию будут выбраны все рабочие области, где вы будете перекрывать журналы Azure AD. 

Вы хотите:

- Понять влияние [политики условного доступа](../conditional-access/overview.md) на опыт вхождения пользователей?

- Сбои в регистрации, чтобы получить лучшее представление о состоянии вхламых в вашей организации и быстро устранить проблемы?

- Узнайте, кто использует устаревшие аутентификации для входиный ввашжий вашего окружения? [(Заблокировав устаревшую аутентификацию,](../conditional-access/block-legacy-authentication.md)вы можете улучшить защиту арендатора.)

- Нужно ли понимать влияние политик «Условный доступ» на клиента?

- Хотите, чтобы возможность рецензирования: запросы входа в журнал, рабочая книга сообщает, сколько пользователей было предоставлено или отказано в доступе, а также сколько пользователей обходят политики условного доступа при доступе к ресурсам?

- Заинтересованы в разработке более глубокого понимания: подробная информация о рабочей книге в состоянии, так что влияние политики может быть контекстуализированы в состоянии, в том числе платформы устройства, состояние устройства, клиент скоеприложение, риск входе, местоположение и приложение?

- Получите более глубокое представление о запросах входа в журнал, в трудовой книжке сообщается, сколько пользователей было предоставлено или им было отказано в доступе, а также сколько пользователей обходят политики условного доступа при доступе к ресурсам.

- Чтобы помочь вам решить эти вопросы, Active Directory предоставляет трудовые книжки для мониторинга. [Рабочие книги Azure Monitor](https://docs.microsoft.com/azure/azure-monitor/app/usage-workbooks) объединяют текст, запросы аналитики, метрики и параметры в богатые интерактивные отчеты.



В этой статье:

- Предполагается, что вы знакомы с тем, как [создавать интерактивные отчеты с помощью рабочих книг Monitor.](https://docs.microsoft.com/azure/azure-monitor/app/usage-workbooks)

- Объясняет, как использовать трудовые книжки Monitor для понимания влияния политик условного доступа, устранения сбоев в системе регистрации и выявления устаревших аутентификаций.
 


## <a name="prerequisites"></a>Предварительные требования

Для использования трудовых книжек Monitor необходимо:

- Активный арендатор каталога с премиальной (P1 или P2) лицензией. Узнайте, как [получить премиальную лицензию.](https://docs.microsoft.com/azure/active-directory/fundamentals/active-directory-get-started-premium)

- [Рабочее пространство аналитики журнала](https://docs.microsoft.com/azure/azure-monitor/learn/quick-create-workspace).

- [Доступ к](https://docs.microsoft.com/azure/azure-monitor/platform/manage-access#manage-access-using-workspace-permissions) рабочей области аналитики журналов
- Следующие роли в Active Directory Azure (если вы получаете доступ к аналитике журналов через портал Active Directory Azure)
    - администратор безопасности;
    - Читатель сведений о безопасности
    - Читатель отчетов
    - Глобальный администратор.

## <a name="roles"></a>Роли
Вы должны быть в одной из следующих ролей, а также иметь [доступ к основной рабочей](https://docs.microsoft.com/azure/azure-monitor/platform/manage-access#manage-access-using-azure-permissions) области журнала Analytics для управления трудовыми книжками:
-   Глобальный администратор.
-   администратор безопасности;
-   Читатель сведений о безопасности
-   Читатель отчетов
-   администратор приложений;

## <a name="workbook-access"></a>Доступ к трудовой книжке 

Для доступа к трудовым книжки:

1. Войдите на [портал Azure](https://portal.azure.com).

1. Перейдите к**рабочим книгам****мониторинга** >  **активных каталогов** > Azure. 

1. Выберите отчет или шаблон или на панели инструментов выберите **Открытый.**

## <a name="conditional-access-insights"></a>Сведения об условном доступе

### <a name="overview"></a>Обзор

В учебниках содержатся запросы входа в журнал, которые могут помочь ИТ-администраторам отслеживать влияние политик условного доступа на клиента. У вас есть возможность сообщить о том, сколько пользователей было бы предоставлено или отказано в доступе. В трудовой книжке содержится информация о том, сколько пользователей обойдя политики условного доступа на основе атрибутов этих пользователей во время входиний. Он содержит сведения о состоянии, так что влияние политики может быть контекстуализировано в состоянии, включая платформу устройства, состояние устройства, клиентское приложение, риск входного ввождения, местоположение и приложение.

### <a name="instructions"></a>Instructions 
Чтобы получить доступ к трудовой книжке для условного доступа, выберите рабочую книгу **Conditional Access Insights** в разделе Условный доступ. В этой рабочей книге показано ожидаемое влияние каждой политики условного доступа на клиента. Выберите одну или несколько политик условного доступа из списка выпадающих и сузите область действия рабочей книги, применяя следующие фильтры: 

- **Диапазон времени**

- **Пользователь**

- **Приложения**

- **Представление данных**

![Статус условного доступа](./media/howto-use-azure-monitor-workbooks/access-insights.png)


Резюме воздействия показывает количество пользователей или вводов, для которых выбранные политики имели определенный результат. Итого — это количество пользователей или входе, для которых выбранные политики оценивались в выбранном Диапазоне Времени. Нажмите на плитку, чтобы отфильтровать данные в трудовой книжке по этому типу результата. 

![Статус условного доступа](./media/howto-use-azure-monitor-workbooks/impact-summary.png)

В этой трудовой книжке также показано влияние выбранных политик, разбиваемых каждым из шести условий: 
- **Состояние устройства**
- **Платформа устройства**
- **Клиентские приложения**
- **Риск вххастывать в**
- **Расположение**
- **Приложения**

![Статус условного доступа](./media/howto-use-azure-monitor-workbooks/device-platform.png)

Вы также можете исследовать отдельные вводы, отфильтрованные по параметрам, выбранным в трудовой книжке. Поиск отдельных пользователей, отсортированных по частоте ввоза, и просмотр соответствующих событий ввне. 

![Статус условного доступа](./media/howto-use-azure-monitor-workbooks/filtered.png)

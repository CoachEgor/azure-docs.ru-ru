# <a name="what-is-the-identity-secure-score-in-azure-active-directory"></a>Общие сведения об оценке безопасности удостоверений в Azure Active Directory

Насколько хорошо защищен ваш клиент AAD? Если вы не знаете, как ответить на этот вопрос, в этой статье объясняется, как оценка безопасности удостоверений помогает отслеживать и повысить уровень безопасности удостоверений.

## <a name="what-is-an-identity-secure-score"></a>Что собой представляет оценка безопасности удостоверений

Оценка безопасности удостоверения является числом от 1 до 223, которое работает в качестве индикатора для степени, вы не с Майкрософт практические рекомендации для обеспечения безопасности. Каждое действие улучшения в Оценка безопасности удостоверений предназначена для конкретной конфигурации.  

![Оценка безопасности](./media/identity-secure-score/identity-secure-score-overview.png)

Оценка поможет вам:

- объективно определить уровень безопасности удостоверений;
- спланировать улучшения безопасности;
- оценить успешность внесенных улучшений.

Показатель оценки и дополнительные сведения о ней можно просмотреть на панели мониторинга "Оценка безопасности удостоверений". Здесь представлены следующие данные:

- Оценка безопасности вашего удостоверения
- Сравнение диаграмма, показывающая сравнение результатов безопасного удостоверения для других клиентов в одной отрасли и одного размера
- Тренд диаграмму, показывающую, как оценка безопасности вашего удостоверения изменялась во времени
- Список возможных улучшений

Выполнив эти рекомендации, вы сможете:

- Повысить уровень безопасности и тогда оценка
- Использовать преимущества функций, доступных для вашей организации, как часть ваших инвестиций удостоверений

## <a name="how-do-i-get-my-secure-score"></a>Как выполнить оценку безопасности

Оценка безопасности удостоверений доступна во всех выпусках Azure AD. Чтобы узнать свой показатель оценки, откройте [панель мониторинга "Обзор" в Azure AD](https://portal.azure.com/#blade/Microsoft_AAD_IAM/ActiveDirectoryMenuBlade/IdentitySecureScore).

## <a name="how-does-it-work"></a>Как это работает?

Каждые 48 часов Azure проверяет конфигурацию безопасности и сравнивает ее параметры с актуальными рекомендациями. В зависимости от результата этой оценки новые показатели вычисляются для вашего каталога. Вполне возможно, что конфигурацию безопасности не полностью соответствует рекомендаций и улучшения действия выполняются лишь частично. В таких случаях вы будет проводиться часть max оценки, доступной для элемента управления.

Каждая рекомендация оценивается по текущей конфигурации AAD. Если вы используете сторонние продукты для включения практическая рекомендация, можно указать этой конфигурации в параметрах действия улучшения. Также имеется возможность указать, рекомендации, которые учитываются в том случае, если они не относятся к вашей среде. Игнорируемые рекомендации не учитываются в расчете оценки.

![Пропустить или пометить действие как покрытый третьей стороной](./media/identity-secure-score/identity-secure-score-ignore-or-third-party-reccomendations.png)

## <a name="how-does-it-help-me"></a>Возможности оценки безопасности

Оценка безопасности позволит вам:

- объективно определить уровень безопасности удостоверений;
- спланировать улучшения безопасности;
- оценить успешность внесенных улучшений.

## <a name="what-you-should-know"></a>Необходимая информация

### <a name="who-can-use-the-identity-secure-score"></a>Кто может использовать оценку безопасности удостоверений

Оценка безопасности удостоверений доступна для следующих ролей:

- глобальный администратор;
- администратор безопасности;
- читатели сведений о безопасности.

### <a name="how-are-controls-scored"></a>Как вычисляются элементы управления?

Элементы управления могут быть рассчитаны двумя способами. Некоторые оцениваются в двоичном режиме – вы получаете 100% оценки, если у вас есть функция или параметр, настроенный на основе нашей рекомендаций. Другие показатели вычисляется как процент от общего конфигурации. Например, если рекомендации по улучшению указывается, если вы получите 30 точек защитить всех пользователей с многофакторной проверкой Подлинности и имеется только 5 100 общее количество пользователей, которые защищены, вам будет предоставлена частичная оценка около 2 пункта (защищенные 5 / всего 100 * 30 пт max = 2 счет частичного pts) .

### <a name="what-does-not-scored-mean"></a>Что означает отметка [Не оценивается]?

Отметку [Не оценивается] получают действия, которые могут выполняться для вашей организации, но пока не включены в оценку, так как (пока что) не поддерживаются этим средством. Это означает, что они позволят повысить безопасность, но пока не улучшат ваш показатель оценки.

### <a name="how-often-is-my-score-updated"></a>Как часто обновляется оценка

Оценка выполняется один раз в день (примерно в 1:00 по тихоокеанскому времени). Если вы внесете изменения в оцениваемые действия, оценка автоматически обновится на следующий день. Для учета изменений в оценке может потребоваться до 48 часов.

### <a name="my-score-changed-how-do-i-figure-out-why"></a>Мой показатель оценки изменился. Как узнать причину?

Перейдите по адресу [центр безопасности Microsoft 365](https://security.microsoft.com/), где вы найдете полный Оценка безопасности Microsoft. Было видно все изменения безопасности очков, просмотрев подробные изменения на вкладке "Журнал".

### <a name="does-the-secure-score-measure-my-risk-of-getting-breached"></a>Оценка безопасности измерения моей риск получения нарушения?

Если отвечать коротко, то нет. Оценка безопасности не предоставляется абсолютный мер как скорее всего, вы являетесь для получения нарушения. Показатель оценки выражает лишь то, в какой мере вы используете возможности, позволяющие снизить риск взлома. Служба не может гарантировать вы не быть превышены, что оценка безопасности не следует рассматривать как гарантии любым способом.

### <a name="how-should-i-interpret-my-score"></a>Как интерпретировать показатель оценки

Вы получаете баллы за использование в конфигурации рекомендованных функций безопасности и (или) выполнение других задач, связанных с безопасностью (например, за просмотр отчетов). Некоторые действия оцениваются даже за частичное выполнение, например включение многофакторной идентификации (MFA) для пользователей. Оценка вашей безопасности непосредственно является представителем Microsoft служб безопасности, которые можно использовать. Помните, что безопасность должна быть сбалансирована с удобством использования. Все элементы управления безопасностью так или иначе влияют на пользователей. Элементы управления с низким уровнем влияния практически не изменяют повседневную работу пользователей.

Чтобы просмотреть историю оценка, перейдите к [центр безопасности Microsoft 365](https://security.microsoft.com/) и просмотрите Общая оценка безопасности Microsoft. Можно просмотреть изменения, чтобы общего Оценка безопасности нажимать на просмотр журнала. Выберите любую дату, чтобы просмотреть включенные в этот день элементы управления и количество начисленных баллов за каждый из них.

### <a name="how-does-the-identity-secure-score-relate-to-the-office-365-secure-score"></a>Как оценка безопасности удостоверений согласуется с Оценкой безопасности Office 365?

[Майкрософт для безопасного Оценка](https://docs.microsoft.com/office365/securitycompliance/microsoft-secure-score) содержит пять различных элементов управления и оценка категории:

- Удостоверение
- Данные
- Устройства
- Инфраструктура
- Приложения

Оценка безопасности удостоверения представляет удостоверение частью Оценка безопасности Microsoft. Такое перекрытие означает, что оценка защитить ваши рекомендации для удостоверения и оценка удостоверений в Microsoft одинаковы.

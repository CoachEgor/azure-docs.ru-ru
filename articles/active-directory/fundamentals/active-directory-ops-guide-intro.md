# <a name="azure-active-directory-operations-reference-guide"></a>Справочный справочник по операциям активных каталогов Azure

В этом справочном руководстве по операциям описаны проверки и действия, которые следует предпринять для обеспечения безопасности и поддержания следующих областей:

- **[Управление идентификацией и доступом](active-directory-ops-guide-iam.md)** - способность управлять жизненным циклом удостоверений личности и их правами.
- **[Управление аутентификацией](active-directory-ops-guide-auth.md)** - способность управлять учетным данными, определять опыт проверки подлинности, делегировать уступки, измерять использование и определять политики доступа на основе позиции безопасности предприятия.
- **[Управление](active-directory-ops-guide-govern.md)** - способность оценивать и засвидетельствовать доступ, предоставленный непривилегированным и привилегированным удостоверениям, аудит и контроль изменений в окружающей среде.
- **[Операции](active-directory-ops-guide-ops.md)** - оптимизация операций Azure Active Directory (Azure AD).

Некоторые рекомендации могут быть применимы не ко всем средам клиентов, например, рекомендации AD FS могут не применяться, если организация использует синхронизацию хэша паролей.

> [!NOTE]
> Эти рекомендации актуальны на дату публикации, но могут меняться с течением времени. Организации должны постоянно оценивать свою практику идентификации по мере развития продуктов и служб Майкрософт с течением времени. Рекомендации могут меняться, когда организации подписываются на другую лицензию Azure AD Premium. Например, Azure AD Premium P2 будет включать в себя больше рекомендаций по управлению.

## <a name="stakeholders"></a>Заинтересованных сторон

Каждый раздел в этом справочном руководстве рекомендует направлять заинтересованные стороны для успешного планирования и реализации ключевых задач. В следующей таблице излагается список всех заинтересованных сторон в этом руководстве:

| Заинтересованное лицо | Описание |
| :- | :- |
| Оперативная группа IAM | Эта группа занимается управлением повседневной работой системы identity и Access Management |
| Команда производительности | Эта группа владеет и управляет приложениями производительности, такими как электронная почта, обмен файлами и совместная работа, обмен мгновенными сообщениями и конференц-связь. |
| Application Owner; | Эта группа владеет конкретным приложением с бизнесом и, как правило, технической точки зрения в организации. |
| Архитектурная команда InfoSec | Эта группа планирует и разрабатывает методы информационной безопасности организации. |
| Оперативная группа InfoSec | Эта группа выполняет и отслеживает реализованные методы информационной безопасности команды InfoSec Architecture. |

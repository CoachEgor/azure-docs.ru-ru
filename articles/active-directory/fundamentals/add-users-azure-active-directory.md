# <a name="add-or-delete-users-using-azure-active-directory"></a>Добавление или удаление пользователей с помощью Azure Active Directory

Добавляйте новых пользователей или удаляйте существующих пользователей из организации Azure Active Directory (Azure AD). Чтобы добавить или удалить пользователей, вы должны быть администратором пользователя или глобальным администратором.

## <a name="add-a-new-user"></a>Добавление нового пользователя

Вы можете создать пользователя с помощью портала Azure Active Directory.

Чтобы добавить нового пользователя, выполните следующие действия:

1. Вопиюсь на [портал Azure](https://portal.azure.com/) в качестве администратора пользователя для организации.

1. Поиск и выберите *Активный каталог Azure* с любой страницы.

1. Выберите **пользователей,** а затем выберите **нового пользователя**.

    ![Добавление пользователя через пользователей - Все пользователи в Azure AD](media/add-users-azure-active-directory/add-user-in-users-all-users.png)

1. На странице **Пользователя** введите информацию для этого пользователя:

   - **Имя**. Обязательный элемент. Имя и фамилия нового пользователя. Например, *Мэри Паркер*.

   - **Имя пользователя**. Обязательный элемент. Имя нового пользователя. Например, `mary@contoso.com`.

     Домен часть имени пользователя должна использовать либо первоначальное доменное имя по умолчанию, * \<yourdomainname>.onmicrosoft.com*, или пользовательские доменное имя, такие как *contoso.com*. Для получения дополнительной информации о том, как создать пользовательское доменное имя, [см.](add-custom-domain.md)

   - **Группы**. При необходимости можно добавить пользователя к одной или нескольким существующим группам. Вы также можете добавить пользователя в группы позже. Для получения дополнительной информации о добавлении пользователей в группы [см. Создать базовую группу и добавить участников с помощью Active Directory Azure.](active-directory-groups-create-azure-portal.md)

   - **Роль каталога:** Если пользователю требуются административные разрешения Azure AD, их можно добавить в роль Azure AD. Вы можете назначить пользователя глобальным администратором или одной или нескольким из ограниченных ролей администратора в Azure AD. Дополнительные сведения о назначении ролей см. в разделе [Назначение ролей пользователям](active-directory-users-assign-role-azure-portal.md).

   - **Информация о вакансиях**: Вы можете добавить больше информации о пользователе здесь, или сделать это позже. Дополнительные сведения о добавлении информации о пользователе см. в разделе [How to add or change user profile information](active-directory-users-profile-azure-portal.md) (Как добавлять и изменять сведения в профиле пользователя).

1. Копирование автогенерируемого пароля, предоставленного в поле **Password.** Вам нужно будет дать этот пароль пользователю, чтобы войти в первый раз.

1. Выберите **Создать**.

Пользователь создается и добавляется в организацию Azure AD.

## <a name="add-a-new-guest-user"></a>Добавление нового гостевого пользователя

Вы также можете пригласить нового гостевого пользователя для совместной работы с вашей организацией, выбрав **пользователя Invite** со страницы **Нового пользователя.** Если внешние настройки совместной работы вашей организации настроены таким образом, что вам будет разрешено приглашать гостей, пользователю будет отправлено приглашение, которое он должен принять, чтобы начать сотрудничество. Для получения дополнительной информации о приглашении пользователей Совместной работы B2B можно в вести [в Active Directory](../b2b/add-users-administrator.md)

## <a name="add-a-consumer-user"></a>Добавление потребителя

Могут быть сценарии, в которых необходимо вручную создавать учетные записи потребителей в каталоге Azure Active Directory B2C (Azure AD B2C). Для получения дополнительной информации о создании учетных записей потребителей [см.](../../active-directory-b2c/manage-users-portal.md)

## <a name="add-a-new-user-within-a-hybrid-environment"></a>Добавление нового пользователя в гибридной среде

Если у вас есть среда с Azure Active Directory (облако) и Windows Server Active Directory (локально), можно добавить новых пользователей путем синхронизации существующих данных учетной записи пользователя. Дополнительные сведения о гибридных средах и пользователях см. в статье [Интеграция локальных каталогов с Azure Active Directory](../hybrid/whatis-hybrid-identity.md).

## <a name="delete-a-user"></a>Удаление пользователя

Вы можете удалить существующего пользователя с помощью портала Azure Active Directory.

Чтобы удалить пользователя, выполните следующие действия:

1. Войти на [портал Azure](https://portal.azure.com/) с помощью учетной записи администратора пользователя для организации.

1. Поиск и выберите *Активный каталог Azure* с любой страницы.

1. Поиск и выберите пользователя, который вы хотите удалить из вашего ad-наниматели Azure. Например, _Мэри Паркер_.

1. Выберите **Удалить пользователя**.

    ![Страница "Пользователи — Все пользователи" с выделенным вариантом "Удалить пользователя"](media/add-users-azure-active-directory/delete-user-all-users-blade.png)

Пользователь удален и больше не отображается на странице **Пользователи — Все пользователи**. Пользователь может отображаться на странице **Удаленные пользователи** в течение 30 дней. В этот период его можно восстановить. Для получения дополнительной информации о восстановлении пользователя [см. Восстановление или удаление недавно удаленного пользователя с помощью Active Directory Azure.](active-directory-users-restore.md)

При удалении пользователя любые лицензии, потребляемые пользователем, становятся доступными для других пользователей.

>[!Note]
>Необходимо использовать Active Directory Windows Server для обновления идентификационных данных, контактной информации или информации о работе пользователей, источником полномочий которых является Active Directory Windows Server. После завершения обновления изменения будут отображаться после следующего цикла синхронизации.

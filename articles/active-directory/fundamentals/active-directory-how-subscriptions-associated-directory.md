# <a name="associate-or-add-an-azure-subscription-to-your-azure-active-directory-tenant"></a>Связывание или добавление подписки Azure в клиент Azure Active Directory

Подписка Azure имеет доверительные отношения с активным каталогом Azure (Azure AD). Подписка доверяет Azure AD для проверки подлинности пользователей, служб и устройств.

Несколько подписок могут доверять тому же каталогу Azure AD. Каждая подписка может доверять только одному каталогу.

Если срок действия подписки истекает, доступ к другим ресурсам, связанным с этой подпиской, также прекращается. Однако каталог Azure AD остается в Azure. Вы можете связать и управлять каталогом с помощью другой подписки Azure.

Все пользователи имеют единый *домашний* каталог для проверки подлинности. Ваши пользователи также могут быть гостями в других каталогах. Домашний и гостевые каталоги для каждого пользователя отображаются в Azure AD.

> [!Important]
> При связывании подписки с другим каталогом пользователи, которые имеют роли, назначенные с помощью [элемента управления доступом на основе ролей (RBAC),](../../role-based-access-control/role-assignments-portal.md) теряют доступ. Администраторы классической подписки, включая администратора служб и соадминистраторов, также теряют доступ.
>
> Назначения политик также удаляются из подписки, если подписка связана с другим каталогом.
>
> Перемещение кластера Azure Kubernetes Service (AKS) в другую подписку или перемещение подписки, владеющей кластером, новому арендатору приводит к потере функциональности кластера из-за утраченных назначений ролей и прав директора службы. Для получения дополнительной информации о AKS, см [Azure Kubernetes службы (AKS)](https://docs.microsoft.com/azure/aks/).


## <a name="before-you-begin"></a>Перед началом

Прежде чем вы сможете связать или добавить подписку, выполняйте следующие задачи:

- Просмотрите следующий список изменений и то, как вы можете быть затронуты:

  - Пользователи, которым были назначены роли с помощью RBAC, потеряют доступ
  - Администратор и соадминистраторы службы потеряют доступ
  - Если у вас есть какие-либо ключевые хранилища, они будут недоступны, и вам придется исправить их после ассоциации
  - Если у вас есть управляемые идентификаторы для ресурсов, таких как виртуальные машины или логические приложения, вы должны повторно включить или воссоздать их после объединения
  - Если у вас есть зарегистрированный Azure Stack, вам придется перерегистрировать его после объединения

- Войдите, используя учетную запись, которая:

  - Имеет назначение роли [владельца](../../role-based-access-control/built-in-roles.md#owner) для подписки. Для получения информации о том, как назначить роль владельца, можно получить информацию о том, как назначить роль владельца, [см. Управление доступом к ресурсам Azure с помощью RBAC и портала Azure.](../../role-based-access-control/role-assignments-portal.md)
  - Существует как в текущем каталоге, так и в новом каталоге. Текущий каталог связан с подпиской. Новый каталог будет ассоциироваться с подпиской. Для получения дополнительной информации о получении доступа к другому каталогу на портале Azure можно [добавить пользователей совместной работы Azure Active Directory B2B.](../b2b/add-users-administrator.md)

- Убедитесь, что не используете следующие подписки: поставщики облачных служб Azure (CSP) (MS-AZR-0145P, MS-AZR-0146P, MS-AZR-159P), внутренняя подписка Майкрософт (MS-AZR-0015P) или Microsoft Imagine (MS-AZR-0144P).

## <a name="associate-a-subscription-to-a-directory"></a>Ассоциировать подписку на каталог<a name="to-associate-an-existing-subscription-to-your-azure-ad-directory"></a>

Чтобы связать существующую подписку с каталогом Azure AD, выполните следующие действия:

1. Войдите и выберите нужную подписку на [странице подписок на портале Azure](https://portal.azure.com/#blade/Microsoft_Azure_Billing/SubscriptionsBlade).

1. Выберите **Изменить каталог**.

    ![Страница подписок с выделенным параметром "Изменить каталог"](media/active-directory-how-subscriptions-associated-directory/change-directory-in-azure-subscriptions.png)

1. Просмотрите все предупреждения и нажмите **Изменить**.

    ![Страница "Изменить каталог" с новым каталогом](media/active-directory-how-subscriptions-associated-directory/edit-directory-ui.png)

    Каталог для подписки изменен, вы получаете сообщение об успешном выполнении.

    ![Сообщение об успехе об изменении каталога](media/active-directory-how-subscriptions-associated-directory/edit-directory-success.png)

Используйте **каталог switch** для перехода в новый каталог. Это может занять несколько часов для всего, чтобы показать должным образом. Если это, кажется, занимает слишком много времени, проверьте **глобальный фильтр подписки.** Убедитесь, что перемещенная подписка не скрыта. Возможно, вам придется выйти из портала Azure и войти обратно, чтобы увидеть новый каталог.

![Страница переключателя каталога, с информацией образца](media/active-directory-how-subscriptions-associated-directory/directory-switcher.png)

Изменение каталога подписки происходит на уровне службы и не влияет на выставление счетов для подписки. Администратор учетной записи по-прежнему может изменить администратора службы в [Центре управления учетной записью](https://account.azure.com/subscriptions). Чтобы удалить исходный каталог, необходимо передать право собственности на выставление счетов по подписке новому аминину учетной записи. Чтобы узнать больше о передаче права собственности на выставление счетов, [см.](../../cost-management-billing/manage/billing-subscription-transfer.md)

## <a name="post-association-steps"></a>Постассоциационные шаги

После ассоциации подписки с другим каталогом может потребоваться выполнение следующих задач для возобновления работы:

- Если у вас есть хранилища ключей, необходимо изменить идентификатор клиента хранилища ключей. Для получения дополнительной информации [см.](../../key-vault/key-vault-subscription-move-fix.md)

- Если для ресурсов использовались системные управляемые идентификаторы, необходимо повторно включить эти идентификаторы. Если вы использовали управляемые идентификаторы, назначенные пользователем, необходимо воссоздать эти идентификаторы. После повторного включения или воссоздания управляемых идентификационных данных необходимо восстановить разрешения, назначенные этим идентификаторам. Для получения дополнительной информации [см. Что такое управляемые идентификаторы для ресурсов Azure?.](../managed-identities-azure-resources/overview.md)

- Если вы зарегистрировали Azure Stack с помощью этой подписки, необходимо перерегистрироваться. Для получения дополнительной [Register Azure Stack with Azure](/azure-stack/operator/azure-stack-registration)информации см.

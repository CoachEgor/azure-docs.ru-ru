# <a name="add-your-custom-domain-name-using-the-azure-active-directory-portal"></a>Добавление имени личного домена с помощью портала Azure Active Directory

Каждый новый арендатор Azure AD поставляется с первоначальным доменным именем, * \<доменным именем>.onmicrosoft.com*. Вы не можете изменить или удалить первоначальное доменное имя, но вы можете добавить имена вашей организации. Добавление пользовательских доменных имен поможет вам создать имена пользователей, знакомые пользователям, такие как *\@alain contoso.com.*

## <a name="before-you-begin"></a>Перед началом

Прежде чем вы сможете добавить пользовательское доменное имя, создайте доменное имя с регистратором домена. Аккредитованные регистраторы перечислены [на этой странице](https://www.icann.org/registrar-reports/accredited-list.html).

## <a name="create-your-directory-in-azure-ad"></a>Создание каталога в AAD

Получив имя домена, создайте первый каталог AAD. Войти на портал Azure для каталога, используя учетную запись с ролью **владельца** для подписки.

Создайте новый каталог, выполнив действия из [этого раздела](active-directory-access-create-new-tenant.md#create-a-new-tenant-for-your-organization).

>[!IMPORTANT]
>Создатель клиента автоматически становится глобальным администратором для этого клиента. Глобальный администратор может добавить дополнительных администраторов для клиента.

Для получения дополнительной информации [Azure RBAC roles](../../role-based-access-control/rbac-and-directory-admin-roles.md#azure-rbac-roles)о роли подписки см.

>[!TIP]
> Если вы планируете подавать планы по федеративному устройству Windows Server AD с помощью Azure AD, то вам нужно выбрать, что **я планирую настроить этот домен для единого входного с моим локальным Active Directory** при запуске инструмента Azure AD Connect для синхронизации каталогов.
>
> Необходимо также зарегистрировать то же имя домена, выбранное для федерации с локальным каталогом на шаге **Домен Azure AD** мастера. Чтобы узнать, как выглядит эта настройка, [см.](../hybrid/how-to-connect-install-custom.md#verify-the-azure-ad-domain-selected-for-federation) Если у вас нет инструмента Azure AD Connect, вы можете [скачать его здесь.](https://go.microsoft.com/fwlink/?LinkId=615771)

## <a name="add-your-custom-domain-name-to-azure-ad"></a>Добавление имени личного домена в Azure Active Directory

После создания каталога вы можете добавить в него имя личного домена.

1. Войдите на [портал Azure](https://portal.azure.com/) с учетной записью глобального администратора каталога.

1. Поиск и выберите *Активный каталог Azure* с любой страницы. Затем выберите **пользовательские доменные имена** > **Добавить пользовательский домен**.

    ![Пользовательские доменные имена страницы, с добавлением пользовательский домен показано](media/add-custom-domain/add-custom-domain.png)

1. В **пользовательском доменном имени**введите новое имя организации, в этом примере *contoso.com*. Выберите **Add domain** (Добавить предметную область).

    ![Пользовательские доменные имена страницы, с Добавлением пользовательской страницы домена](media/add-custom-domain/add-custom-domain-blade.png)

    >[!IMPORTANT]
    >Вы должны включить *.com*, *.net*, или любое другое расширение верхнего уровня для этого, чтобы работать должным образом.

    Добавлен непроверенный домен. На **contoso.com** странице отображается ваша информация DNS. Сохраните эти сведения. Позже для настройки DNS необходимо создать запись TXT.

    ![Страница Contoso со сведениями о записях DNS](media/add-custom-domain/contoso-blade-with-dns-info.png)

## <a name="add-your-dns-information-to-the-domain-registrar"></a>Настройка сведений о DNS в системе регистратора доменных имен

После добавления имени личного домена в Azure AD вам следует вернуться на сайт регистратора домена и настроить сведения о DNS, которые вы скопировали из AAD в TXT-файл. Создание этой записи TXT для вашего домена проверяет право собственности на ваше доменное имя.

Вернитесь к регистратору доменов и создайте новую запись TXT для вашего домена на основе скопированных DNS-информации. Установите время для жизни (TTL) до 3600 секунд (60 минут), а затем сохранить запись.

>[!IMPORTANT]
>Вы можете зарегистрировать любое количество доменных имен. Но для каждого домена AAD создает собственную запись типа TXT. Будьте осторожны при вводе файла TXT в регистраторе доменов. Если вы введете неправильную или дублировать информацию по ошибке, вам придется подождать, пока TTL раз (60 минут), прежде чем вы можете попробовать еще раз.

## <a name="verify-your-custom-domain-name"></a>Проверка имени личного домена

После регистрации пользовательского доменного имени убедитесь, что оно действительно в Azure AD. Распространение от регистратора домена до Azure AD может быть мгновенным или может занять несколько дней, в зависимости от регистратора домена.

Чтобы проверить пользовательское доменное имя, выполните следующие действия:

1. Войдите на [портал Azure](https://portal.azure.com/) с учетной записью глобального администратора каталога.

1. Поиск и выберите *Активный каталог Azure* с любой страницы, а затем выберите **пользовательские доменные имена.**

1. В **пользовательских доменных имен**, выберите пользовательские доменное имя. В этом примере выберите **contoso.com**.

    ![Страница имен личных доменов Fabrikam с выделенным именем Contoso](media/add-custom-domain/custom-blade-with-contoso-highlighted.png)

1. На **странице contoso.com** выберите **Verify,** чтобы убедиться, что пользовательский домен зарегистрирован должным образом и действителен для Azure AD.

    ![Страница Contoso со сведениями о записях DNS и кнопкой "Проверить"](media/add-custom-domain/contoso-blade-with-dns-info-verify.png)

После проверки пользовательского доменного имени можно удалить файл tXT или MX.

## <a name="common-verification-issues"></a>Распространенные проблемы при проверке

Если Azure AD не сможет подтвердить имя личного домена, воспользуйтесь приведенными ниже рекомендациями.

- **Подождите не менее часа и повторите попытку**. Azure AD сможет проверить домен только после распространения DNS-записей. Для этого может понадобиться один или несколько часов.

- **Проверьте правильность записи DNS.** Вернитесь на сайт регистратора доменных имен. Убедитесь, что запись есть, и что она соответствует информации о входе DNS, предоставляемой Azure AD.

  Если вы не можете обновить запись на сайте регистратора, поделитесь записью с кем-то, у кого есть разрешения на добавление записи и проверку ее правильности.

- **Убедитесь, что введенное доменное имя не используется в другом каталоге.** Доменное имя может быть проверено только в одном каталоге. Если ваше доменное имя в настоящее время проверено в другом каталоге, оно также не может быть проверено в новом каталоге. Чтобы устранить эту проблему дублирования, удалите доменное имя из старого каталога. Дополнительные сведения об удалении доменных имен см. в статье об [управлении именами личных доменов](../users-groups-roles/domains-manage.md).

- **Убедитесь, что у вас нет неуправляемых клиентов Power BI.** Если пользователи активировали Power BI через функцию самостоятельной регистрации и создали неуправляемый клиент для организации, необходимо взять на себя управление в качестве внутреннего или внешнего администратора с помощью PowerShell. Дополнительные сведения см. в [Смена неуправляемого каталога от имени администратора в Azure Active Directory](../users-groups-roles/domains-admin-takeover.md).

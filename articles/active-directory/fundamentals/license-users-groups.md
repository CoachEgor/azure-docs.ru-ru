# <a name="assign-or-remove-licenses-in-the-azure-active-directory-portal"></a>Назначать или удалять лицензии на портале Active Directory Azure

Многие службы Active Directory (Azure AD) требуют лицензировать каждого из пользователей или групп (и связанных с ней членов) для этой службы. Только пользователи с активными лицензиями смогут получить доступ к лицензированным службам Azure AD и использовать их, для которых это верно. Лицензии применяются на одного арендатора и не передаются другим арендаторам. 

## <a name="available-license-plans"></a>Доступные лицензионные планы

Существует несколько лицензионных планов для службы Azure AD, в том числе:

- Azure AD уровня "Бесплатный"

- Azure AD Premium P1

- Azure AD Premium P2

Для получения конкретной информации о каждом лицензионном плане и связанных с ним лицензионных данных см., [Какая лицензия мне нужна?](https://azure.microsoft.com/pricing/details/active-directory/)

Некоторые службы Майкрософт доступны не во всех регионах. Прежде чем назначать лицензию группе, вы должны указать **Место использования** для всех участников. Это значение можно задать в разделе **Azure Active Directory &gt; Пользователи &gt; Профиль &gt; Параметры** в Azure AD. Любой пользователь, местоположение использования которого не указано, наследует местоположение организации Azure AD.

## <a name="view-license-plans-and-plan-details"></a>Просмотр лицензионных планов и сведений о планах

Вы можете просматривать доступные планы обслуживания, включая отдельные лицензии, проверять ожидающие истечения срока действия и просматривать количество доступных назначений.

### <a name="to-find-your-service-plan-and-plan-details"></a>Чтобы найти свой план обслуживания и детали плана

1. Войти на [портал Azure](https://portal.azure.com/) с помощью учетной записи администратора лицензии в организации Azure AD.

1. Выберите **Azure Active Directory**, а затем щелкните **Лицензии**.

    ![Страница лицензий с количеством приобретенных услуг и назначенных лицензий](media/license-users-groups/license-details-blade.png)

1. Выберите **приобретенную** ссылку, чтобы просмотреть **страницу Продуктов** и просмотреть **назначенные,** **доступные**и **срок действия скоро** номера для ваших лицензионных планов.

    ![страница услуг, с планами лицензии службы и связанной с ними информацией о лицензии](media/license-users-groups/license-products-blade-with-products.png)

1. Выберите имя плана, чтобы увидеть его лицензированных пользователей и групп.

## <a name="assign-licenses-to-users-or-groups"></a>Назначение лицензий пользователям или группам

Убедитесь, что все пользователи, которым необходимо использовать лицензированные службы Azure AD, имеют соответствующую лицензию. Вы можете добавить лицензионные права для пользователей или для всей группы.

### <a name="to-assign-a-license-to-a-user"></a>Присвоение лицензии пользователю

1. На странице **Продуктов** выберите название плана лицензии, который вы хотите назначить пользователю.

    ![страница услуг с выделенным планом лицензии службы](media/license-users-groups/license-products-blade-with-product-highlight.png)

1. На странице обзора плана лицензии выберите **Назначайте**.

    ![страница услуг с выделенным вариантом Назначать](media/license-users-groups/license-products-blade-with-assign-option-highlight.png)

1. На странице **Назначить** выберите **Пользователи и группы**, а затем найдите и выберите пользователя, которому вы хотите назначить лицензию.

    ![Страница "Назначить лицензию" с выделенными параметрами "Поиск" и "Выбрать"](media/license-users-groups/assign-license-blade-with-highlight.png)

1. Выберите **Варианты назначения**, включите нужные варианты лицензии и нажмите **OK**.

    ![Страница опций лицензии со всеми опциями, доступными в плане лицензии](media/license-users-groups/license-option-blade-assignments.png)

    Страница **Назначить лицензию** обновится, и вы увидите выбранного пользователя и настроенные назначения.

    > [!NOTE]
    > Некоторые службы Майкрософт доступны не во всех регионах. Прежде чем назначать лицензию, вы должны указать для пользователя **Место использования**. Это значение можно задать в разделе **Azure Active Directory &gt; Пользователи &gt; Профиль &gt; Параметры** в Azure AD. Любой пользователь, местоположение использования которого не указано, наследует местоположение организации Azure AD.

1. Выберите **Назначить**.

    Пользователь добавляется в список лицензированных пользователей и получает доступ к входящим в лицензию службам Azure AD.
    > [!NOTE]
    > Лицензии также могут быть назначены непосредственно пользователю со **страницы Лицензий** пользователя. Если у пользователя есть лицензия, назначенная через членство в группе, и вы хотите назначить ту же лицензию пользователю напрямую, это может быть сделано только со **страницы Продуктов,** указанной только в шаге 1.

### <a name="to-assign-a-license-to-a-group"></a>Присвоение лицензии группе

1. На странице **Продуктов** выберите название плана лицензии, который вы хотите назначить пользователю.

    ![Лезвие продукции с выделенным планом лицензирования продукта](media/license-users-groups/license-products-blade-with-product-highlight.png)

1. На странице **Azure Active Directory Premium (план 2)** выберите **Назначить**.

    ![Страница "Продукты" с выделенным вариантом "Назначить"](media/license-users-groups/license-products-blade-with-assign-option-highlight.png)

1. На странице **Назначить** выберите **Пользователи и группы**, а затем найдите и выберите группу, которой вы хотите назначить лицензию.

    ![Страница "Назначить лицензию" с выделенными параметрами "Поиск" и "Выбрать"](media/license-users-groups/assign-group-license-blade-with-highlight.png)

1. Выберите **Варианты назначения**, включите нужные варианты лицензии и нажмите **OK**.

    ![Страница опций лицензии со всеми опциями, доступными в плане лицензии](media/license-users-groups/license-option-blade-group-assignments.png)

    Страница **Назначить лицензию** обновится, и вы увидите выбранного пользователя и настроенные назначения.

1. Выберите **Назначить**.

    Группа будет добавлена в список лицензированных групп, и все ее участники получат доступ к входящим в лицензию службам Azure AD.

## <a name="remove-a-license"></a>Удаление лицензии

Можно удалить лицензию со страницы пользователя Azure AD, со страницы обзора группы для группового задания или начать со **страницы лицензий** Azure AD, чтобы увидеть пользователей и группы для получения лицензии.

### <a name="to-remove-a-license-from-a-user"></a>Удаление лицензии у пользователя

1. На странице **Лицензированных пользователей** для плана службы выберите пользователя, у которого больше не должно быть лицензии. Например, _Alain Charon_.

1. Выберите **Удалить лицензию**.

    ![Страница "Лицензированные пользователи" с выделенным параметром "Удалить лицензию"](media/license-users-groups/license-products-user-blade-with-remove-option-highlight.png)

> [!IMPORTANT]
> Лицензии, наследуемые пользователем из группы, не могут быть удалены напрямую. Вместо этого нужно удалить пользователя из группы, от которой наследуется эта лицензия.

### <a name="to-remove-a-license-from-a-group"></a>Удаление лицензии для группы

1. На странице **лицензированных групп** для лицензионного плана выберите группу, которая больше не должна иметь лицензию.

1. Выберите **Удалить лицензию**.

    ![Страница "Лицензированные группы" с выделенным параметром "Удалить лицензию"](media/license-users-groups/license-products-group-blade-with-remove-option-highlight.png)
    
    > [!NOTE]
    > Когда локта пользователя, синхронизированная с Azure AD, выходит за рамки синхронизации или когда синхронизация удалена, пользователь удаляется в Azure AD. Когда это происходит, лицензии, назначенные пользователю непосредственно или через групповое лицензирование, будут помечены как **приостановленные,** а не **удаленные.**

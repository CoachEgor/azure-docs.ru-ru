# <a name="the-elements-of-the-b2b-collaboration-invitation-email---azure-active-directory"></a>Элементы сообщения с приглашением службы совместной работы B2B Azure Active Directory

Сообщение с приглашением является важнейшим компонентом, необходимым для подключения партнеров в качестве пользователей службы совместной работы B2B в Azure AD. Хотя это [не требуется, что вы отправляете по электронной почте, чтобы пригласить кого-то с помощью B2B сотрудничества](add-user-without-invite.md), это дает пользователю всю информацию, необходимую для принятия решения о том, следует ли принять приглашение. Это также дает им ссылку, на которой они всегда могут ссылаться в будущем, когда им нужно вернуться к вашим ресурсам.

![Снимок экрана с приглашением](media/invitation-email-elements/invitation-email.png)

> [!NOTE]
> Этот новый шаблон электронной почты все еще развертывается для всех арендаторов, так что некоторые арендаторы все еще используют старый дизайн. К концу мая 2020 года приглашения от всех арендаторов будут использовать этот шаблон.

## <a name="explaining-the-email"></a>Из чего состоит сообщение

Давайте рассмотрим основные элементы сообщения, чтобы понять, как лучше использовать их возможности.

### <a name="subject"></a>Тема

Тема письма следует этому шаблону:

&lt;имя&gt; пользователя предложило вам получить доступ к приложениям в рамках их организации.

### <a name="from-address"></a>Адрес отправителя

Для адреса отправителя используется формат, аналогичный сети LinkedIn. Этот шаблон должен дать понять, invites@microsoft.comчто, хотя письмо исходит от , приглашение от другой организации. Формат: приглашения Майкрософт <invites@microsoft.com> или приглашения Майкрософт &lt;от&gt; <invites@microsoft.com>имени имен арендаторов. 

### <a name="reply-to"></a>Ответить

В поле обратного адреса указывается адрес электронной почты приглашающего (если доступен). Таким образом, ответ на полученное сообщение будет отправлен приглашающему клиенту.

### <a name="phishing-warning"></a>Предупреждение о фишинге

Письмо начинается с краткого предупреждения пользователю о фишинге, предупреждая его о том, что они должны принимать только приглашения, которые они ожидают. Это хорошая практика, чтобы убедиться, что партнеры, которых вы приглашаете, не будут удивлены вашим приглашением, упомянув его им заранее.

![Изображение фишингового предупреждения в электронной почте](media/invitation-email-elements/phishing-warning.png)

### <a name="inviters-information"></a>Сведения о приглашающем

Электронная почта содержит информацию о приглашенном и организации, от которой они отправляют приглашение. Это включает в себя имя отправителя и адрес электронной почты, а также имя и основной домен, связанный с организацией. Вся эта информация должна помочь приглашению принять обоснованное решение о принятии приглашения.

![Изображение информации приглашенного в электронной почте](media/invitation-email-elements/inviters-information.png)

### <a name="invitation-message"></a>Сообщение о приглашении

Если приглашенный включает сообщение в качестве части приглашения, когда он приглашает приглашенного пользователя в [каталог, группу или приложение](add-users-administrator.md) или когда он использует [API приглашения,](customize-invitation-api.md)сообщение выделяется в основном разделе электронной почты. Также включены имя и изображение профиля приглашенного, если они установили один. Само сообщение является текстовой областью, поэтому по соображениям безопасности оно не обрабатывает HTML-теги.

![Изображение пригласительных сообщений в электронной почте](media/invitation-email-elements/invitation-message.png)

### <a name="accept-button-and-redirect-url"></a>Принять кнопку и перенаправить URL

Следующий раздел письма содержит информацию о том, где приглашенный будет доставлен после того, как они примут приглашение, а также кнопку, чтобы сделать это.  В будущем приглашенный всегда может использовать эту ссылку, чтобы вернуться к вашим ресурсам напрямую.

![Изображение кнопки «Принять» и перенаправить URL-адрес электронной почты](media/invitation-email-elements/accept-button.png)

### <a name="footer-section"></a>Нижний колонтитул

Колонтитул содержит больше информации о приглашении, отправляемом. Приглашать приглашать всегда есть возможность блокировать будущие приглашения. Если организация [установила заявление о конфиденциальности,](https://docs.microsoft.com/azure/active-directory/fundamentals/active-directory-properties-area)ссылка на выписку отображается здесь.  В противном случае в заметке указывается, что организация не установила заявление о конфиденциальности.

![Изображение раздела колонтитула в электронной почте](media/invitation-email-elements/footer-section.png)
 
## <a name="how-the-language-is-determined"></a>Определение языка

Язык, представляемый гостю в электронном письме с приглашением, определяется следующими параметрами. Эти параметры перечислены в порядке приоритета. Если параметр не настроен, язык определит следующий в списке параметр.

- Свойство **messageLanguage** объекта [InviteUserMessageInfo](https://docs.microsoft.com/graph/api/resources/invitedusermessageinfo?view=graph-rest-1.0), если используется API создания приглашений.
-   Свойство **preferredLanguage**, указанное в [объекте user](https://docs.microsoft.com/graph/api/resources/user?view=graph-rest-1.0) гостя.
-   **Язык уведомлений** устанавливается в свойствах домашнего клиента гостевого пользователя (только для клиентов Azure AD).
-   **Язык уведомлений** задается в свойствах ресурса клиента.

Если ни один из этих параметров не настроен, по умолчанию выбирается английский язык (США).

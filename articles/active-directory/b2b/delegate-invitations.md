# <a name="enable-b2b-external-collaboration-and-manage-who-can-invite-guests"></a>Включить внешнее сотрудничество B2B и управлять тем, кто может приглашать гостей

В этой статье описывается, как включить совместное взаимодействие СУ Active Directory (Azure AD) B2B и определить, кто может приглашать гостей. По умолчанию все пользователи и гости в вашем каталоге могут приглашать гостей, даже если они не назначены на роль админ. Внешние настройки совместной работы позволяют включить или выключить приглашения для различных типов пользователей в организации. Вы также можете делегировать приглашения отдельным пользователям, назначая роли, которые позволяют им приглашать гостей.

## <a name="configure-b2b-external-collaboration-settings"></a>Настройка внешних настроек совместной работы B2B

Используя службу совместной работы Azure AD B2B, администратор клиентов может устанавливать следующие политики приглашения:

- отключение приглашений;
- приглашать могут только администраторы и пользователи с ролью Guest Inviter;
- приглашать могут администраторы и участники роли Guest Inviter;
- приглашать могут все пользователи, включая гостей.

По умолчанию все пользователи, включая гостей, могут приглашать приглашенных пользователей.

### <a name="to-configure-external-collaboration-settings"></a>Настройка внешних настроек совместной работы:

1. Вопиюсь на [портал Azure](https://portal.azure.com) в качестве администратора-арендатора.
2. Выберите > настройки > **пользователей****активных каталогов** **Azure.**
3. Под **внешними пользователями**выберите **Параметры внешнего взаимодействия.**
   > [!NOTE]
   > **Параметры внешнего взаимодействия** также доступны на странице **Организационные связи**. В Azure Active Directory в разделе **Управление** в меню выберите **Организационные связи** > **Параметры**.
4. На странице **внешних настроек выберите** политики, которые вы хотите включить.

   ![Параметры внешнего взаимодействия](./media/delegate-invitations/control-who-to-invite.png)

  - **Разрешения гостевых пользователей ограничены:** Эта политика определяет разрешения для гостей в вашем каталоге. Выберите **«Да»,** чтобы блокировать гостей от определенных задач каталога, таких как перечисление пользователей, групп или других ресурсов каталога. Выберите **Нет,** чтобы предоставить гостям такой же доступ к данным каталога, как и обычным пользователям в вашем каталоге.
   - **Админы и пользователи в роли приглашенного гостя могут предложить:** Разрешить админорам и пользователям в роли "Guest Inviter" приглашать гостей, установить эту политику **да**.
   - **Участники могут предложить:** Чтобы не-амин-члены вашего каталога приглашали гостей, установить эту политику **в Да**.
   - **Гости могут пригласить**: Чтобы гости могли пригласить других гостей, установить эту политику **в Да**.
   - **Включите одноразовый пароль электронной почты для гостей (Preview)**: Для получения дополнительной информации о одноразовом коде доступа [см.](one-time-passcode.md)
   - **Ограничения на совместное использование:** Для получения дополнительной информации о разрешении или блокировке приглашений в определенные домены см. [Разрешить или заблокировать приглашения для пользователей B2B от конкретных организаций.](allow-deny-list.md)

## <a name="assign-the-guest-inviter-role-to-a-user"></a>Назначить роль приглашенного гостя пользователю

С ролью Guest Inviter вы можете предоставить отдельным пользователям возможность приглашать гостей, не назначая им глобального администратора или другую роль администратора. Присвоить роль приглашенного гостевого лица. Затем убедитесь, что вы **установите Admins и пользователи в роли приглашенного гостя могут пригласить** **на Да**.

Ниже приведен пример, показывающий, как использовать PowerShell, чтобы добавить пользователя в роль Guest Inviter:

```
Add-MsolRoleMember -RoleObjectId 95e79109-95c0-4d8e-aee3-d01accf2d47b -RoleMemberEmailAddress <RoleMemberEmailAddress>
```

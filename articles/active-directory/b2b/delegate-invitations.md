# <a name="enable-b2b-external-collaboration-and-manage-who-can-invite-guests"></a>Включить внешние службы совместной работы B2B и управлять ими, могут приглашать гостей

В этой статье описывается, как включить службы совместной работы B2B Azure Active Directory (Azure AD) и определить, кто может приглашать гостей. По умолчанию всех пользователей и гостей в каталоге могут приглашать гостей даже если они не назначена роль администратора. Параметры внешнего взаимодействия позволяют преобразовывать приглашениями гостей или отключить для различных типов пользователей в вашей организации. Вы также можете делегировать приглашения для отдельных пользователей путем назначения ролей, которые позволяют им приглашать гостей.

## <a name="configure-b2b-external-collaboration-settings"></a>Настройте параметры внешнего взаимодействия B2B

Используя службу совместной работы Azure AD B2B, администратор клиентов может устанавливать следующие политики приглашения:

- отключение приглашений;
- приглашать могут только администраторы и пользователи с ролью Guest Inviter;
- приглашать могут администраторы и участники роли Guest Inviter;
- приглашать могут все пользователи, включая гостей.

По умолчанию все пользователи, включая гостей приглашать гостевых пользователей.

### <a name="to-configure-external-collaboration-settings"></a>Чтобы настроить параметры внешнего взаимодействия:

1. Войдите в [портала Azure](https://portal.azure.com) администратор клиента.
2. Выберите **Azure Active Directory** > **Пользователи** > **Параметры пользователей**.
3. В разделе **Внешние пользователи** выберите **Manage external collaboration settings** (Управление параметрами внешних партнеров).
   > [!NOTE]
   > **Параметры внешнего взаимодействия** также доступны на странице **Организационные связи**. В Azure Active Directory в разделе **Управление** в меню выберите **Организационные связи** > **Параметры**.
4. На **параметры внешнего взаимодействия** выберите политики, необходимо включить.

   ![Параметры внешнего взаимодействия](./media/delegate-invitations/control-who-to-invite.png)

  - **Разрешения для гостей ограничены**: Эта политика определяет разрешения для гостей в каталоге. Выберите **Да** блок гостевым виртуальным машинам из некоторые задачи каталога, например перечисление пользователей, групп или других ресурсов каталога. Выберите **нет** для предоставления гостей, тот же уровень доступа к данным каталога как для обычных пользователей в каталоге.
   - **Приглашать могут администраторы и пользователи в роль guest inviter**: Чтобы разрешить Администраторы и пользователи с ролью «Приглашающий гостей» приглашать гостей, задать эту политику **Да**.
   - **Участники могут приглашать**: Чтобы разрешить членам без прав администратора каталога приглашать гостей, задать эту политику **Да**.
   - **Гости могут приглашать**: Чтобы разрешить гостей приглашать других гостей, задать эту политику **Да**.
   - **Включить секретный код одноразового электронной почты для гостевых систем (Предварительная версия)** : Дополнительные сведения о функции одноразовый секретный код, см. в разделе [проверкой подлинности с одноразовым паролем электронной почты (Предварительная версия)](one-time-passcode.md).
   - **Ограничения взаимодействия**: Дополнительные сведения о разрешить или блокировать приглашения к определенным доменам, см. в разделе [разрешить или блокировать приглашений пользователям B2B из отдельных организаций](allow-deny-list.md).

## <a name="assign-the-guest-inviter-role-to-a-user"></a>Назначить пользователю роль Guest Inviter

Имея в роль Guest Inviter отдельным пользователям можно предоставить возможность приглашать гостей, не назначая им права глобального администратора или другие роли администратора. Назначьте роль Guest inviter для частных лиц. Убедитесь, что выбрано **приглашать могут администраторы и пользователи в роль guest inviter** для **Да**.

Ниже приведен пример, показывающий, как использовать PowerShell, чтобы добавить пользователя в роль Guest Inviter:

```
Add-MsolRoleMember -RoleObjectId 95e79109-95c0-4d8e-aee3-d01accf2d47b -RoleMemberEmailAddress <RoleMemberEmailAddress>
```

# <a name="limitations-of-azure-ad-b2b-collaboration"></a>Ограничения службы совместной работы Azure Active Directory B2B
К службе совместной работы Azure Active Directory B2B применяются ограничения, описанные в этой статье.

## <a name="possible-double-multi-factor-authentication"></a>Возможное дублирование Многофакторной идентификации
С помощью Azure AD B2B вы можете принудительно применять многофакторную проверку подлинности в ресурсной организации (приглашающей организации). Причины такого подхода подробно описаны в [«Условном доступе» для пользователей совместной работы B2B.](conditional-access.md) Если ваш партнер уже настроил многофакторную проверку подлинности и принудительно применяет ее, возможно, его пользователи должны будут выполнять эту проверку один раз в своей организации, а затем еще раз — в вашей организации.

## <a name="instant-on"></a>Мгновенное включение
В рабочих потоках службы совместной работы B2B пользователи добавляются в каталог и динамически обновляются в процессе активации приглашений, при назначении приложений и т. д. Операции обновления и записи обычно выполняются в одном экземпляре каталога и должны реплицироваться по всем экземплярам. Репликация будет завершена после обновления всех экземпляров. Иногда, когда объект записывается или обновляется в одном экземпляре, а вызов для извлечения этого объекта попадает на другой экземпляр, это вызывает задержку репликации. В таком случае выполните обновление или повторите попытку. Если вы пишете приложение с помощью нашего API, то повторы с некоторой задержкой являются хорошей защитной практикой для облегчения этой проблемы.

## <a name="azure-ad-directories"></a>Каталоги Azure AD
Для Azure AD B2B действуют ограничения каталогов служб Azure AD. Дополнительные сведения о количество каталогов, которые может создать пользователь, и количестве каталогов, с которыми может быть связан обычный или гостевой пользователь, см. в разделе [Ограничения службы Azure AD](https://docs.microsoft.com/azure/active-directory/users-groups-roles/directory-service-limits-restrictions).

## <a name="national-clouds"></a>Национальные облака
[Национальные облака](https://docs.microsoft.com/azure/active-directory/develop/authentication-national-cloud) являются физически изолированными экземплярами Azure. Сотрудничество B2B не поддерживается в национальных границах облачных технологий. Например, если ваш арендатор Azure находится в общедоступном глобальном облаке, вы не можете пригласить пользователя, учетная запись которого находится в национальном облаке. Чтобы сотрудничать с пользователем, попросите у них другой адрес электронной почты или создайте учетную запись пользователя-члена для них в вашем каталоге.

## <a name="azure-us-government-clouds"></a>Лазурные облака правительства США
В облаке Правительства Сша ВС США сотрудничество B2B в настоящее время поддерживается только между арендаторами, которые находятся в облаке правительства Azure, и поддерживают сотрудничество B2B. Если вы приглашаете пользователя в составе клиента, который не является частью облака правительства Azure В США или еще не поддерживает совместное взаимодействие B2B, приглашение не будет передано, или пользователь не сможет выкупить приглашение. Подробнее о других ограничениях [можно](https://docs.microsoft.com/azure/azure-government/documentation-government-services-securityandidentity#azure-active-directory-premium-p1-and-p2)узнать о других ограничениях, см.

### <a name="how-can-i-tell-if-b2b-collaboration-is-available-in-my-azure-us-government-tenant"></a>Как укажите, доступно ли совместное сотрудничество B2B в моего клиента Правительства Сша По области?
Чтобы узнать, поддерживает ли ваш правительственный облак Azure в США совместное сотрудничество B2B, сделайте следующее:

1. В браузере перейдите на следующий URL, заменив имя арендатора на * &lt;имя арендатора:&gt;*

   `https://login.microsoftonline.com/<tenantname>/v2.0/.well-known/openid-configuration`

2. Найти `"tenant_region_scope"` в ответе JSON:

   - Если `"tenant_region_scope":"USGOV”` отображается, B2B поддерживается.
   - Если `"tenant_region_scope":"USG"` отображается, B2B не поддерживается.

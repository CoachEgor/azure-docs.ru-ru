# <a name="building-secure-workstations"></a>Создание безопасных рабочих станций

Защищенное изолированных рабочих станций являются критически важными для безопасности конфиденциальных ролей, таких как администраторы, разработчики и операторы критически важных служб. Многие другие средства управления безопасностью и гарантии будет ошибкой или не действовать, если базовые средства обеспечения безопасности рабочей станции клиента был скомпрометирован.

В этом документе объясняется, что требуется для создания безопасного клиентской рабочей станции с подробными инструкциями шаг за шагом, включая способы начать средства управления безопасностью. Этот тип рабочих станций, иногда называется станций с привилегированным доступом (PAW), которой этот справочник, используется и построенных на основе. Тем не менее рекомендации выглядит на облачные технологии для управления службой и включает некоторые функции безопасности, появившиеся начиная с Windows 10RS5, ATP в Защитнике Microsoft, Azure Active Directory и Intune.

## <a name="why-securing-workstation-access-is-important"></a>Почему важна защита доступа к рабочей станции

Быстрое внедрение облачных служб и возможность работы из любого места, создала новый метод для использования. Злоумышленники связанным с использованием элементов управления должную безопасность на устройствах, где администраторы работают и могут получить доступ к привилегированным ресурсам.

Как описано в статье [отчетов Verizon угроз](https://enterprise.verizon.com/resources/reports/dbir/), и [отчета службы безопасности](https://aka.ms/sir) неправильного использования привилегий, и атак цепочки поставок являются пять механизмы, используемые для уведомлений о нарушении организаций и Во-вторых наиболее часто обнаруживаемых тактика в событиях, в 2018 г.

Большинство злоумышленников к контуру ниже:

* Начните с разведывательной атаки, часто относящиеся к отрасли, чтобы найти расположение в
* Анализ собранной информации для идентификации оптимальным средством для получения доступа (несанкционированного доступа), низкая значимость рабочей станции
* Сохраняемость и посмотрите, означает, что для перемещения [режиме бокового смещения](https://en.wikipedia.org/wiki/Network_Lateral_Movement)
* Вести Эксфильтрацию конфиденциальных данных

Злоумышленники часто проникновения устройств, которые кажутся низким уровнем риска или недооцененными для разведывательной атаки. Затем эти уязвимых устройств используются для поиска возможность перемещения по сети, найти пользователей с правами администратора и устройства и идентификации высокой ценных данных, чтобы успешно вести эксфильтрацию сведения после получения этих ролей привилегированного пользователя.

![Типичный компрометации шаблон](./media/concept-azure-managed-workstation/typical-timeline.png)

Этот документ представляет собой решение для Защита вычислительных устройств, изолируя управления и служб для защиты от бокового или атаки с менее ценным производительности устройств. Проектирование помогает снизить возможность успешного выполнения нарушения безопасности, разбив цепочке до несанкционированного доступа устройства, используемого для доступа к конфиденциальным облачных ресурсов. В описании решения будет использовать собственные службы Azure, которые являются частью стека Microsoft 365 Enterprise том числе:

* Intune для управления устройствами, включая приложения и URL-адрес в список разрешений
* AutoPilot для настройки устройства и развертывания и обновления 
* Azure AD для управления пользователями, условного доступа и многофакторной проверки подлинности
* Windows 10 (версии) для устройства работоспособности аттестации и взаимодействия с пользователем
* Microsoft Defender Advanced Threat Protection (ATP) для защиты конечной точки, обнаружения и устранения с помощью управления облачными клиентами
* Azure AD PIM для управления авторизации, включая только что In Time (JIT) привилегированный доступ к ресурсам

## <a name="who-benefit-from-using-a-secure-workstation"></a>Кто преимущества безопасной рабочей станции

Все пользователи и операторы с выгодой использовать безопасной рабочей станции. Злоумышленник, нарушающую ПК или устройства можно выполнить несколько задач олицетворять все кэшированные учетные записи и использовать учетные данные и маркеры, используемые на этом устройстве, при входе в систему. Делает этот риск, обеспечение безопасности устройства, используемые для любой привилегированные роли, включая так важен, так как устройства, где используется привилегированной учетной записью, целевые объекты для бокового смещения и атаки повышения привилегий администратора. Эти учетные записи может использоваться для различных средств, например:

* Администраторы локальных и облачных систем
* Рабочие станции разработчиков для критически важных систем
* Администратор учетных записей социальных с возможностью высокий уровень доступа
* Конфиденциальных рабочих станций, такие как терминалы SWIFT оплаты
* Обработка тайны рабочих станций

Корпорация Майкрософт рекомендует реализации средства управления безопасностью с повышенными правами для рабочих станций с привилегированным, где используются эти учетные записи для снижения риска. Дополнительные инструкции можно найти в [руководство по развертыванию компонентов Azure Active Directory](https://docs.microsoft.com/azure/active-directory/fundamentals/active-directory-deployment-checklist-p2), [план Office 365](https://aka.ms/o365secroadmap), и [стратегии обеспечения безопасности привилегированного доступа](https://aka.ms/sparoadmap)).

## <a name="why-dedicated-workstations"></a>Почему выделенных рабочих станций

Хотя можно добавить в существующее устройство безопасности, лучше начать с защищенной платформой. Начиная с хорошо известным устройством и набор помещает элементы управления безопасности вашей организации для поддержания, наилучшим образом увеличить уровень безопасности. С постоянно растущим числом атак, допускаемых случайного электронной почты и веб-страниц все более сложно убедитесь, что устройство можно доверять. Это руководство работает в предположении, выделенная рабочая станция отделены от стандартных производительность, просмотр, и выполнения задач по электронной почте. Удаление продуктивность, просмотра веб-страниц и электронной почты на устройстве может иметь негативное влияние на производительность, но эта мера безопасности обычно приемлемо для сценариев, где задачи задания не требуются явно и велик риск нарушения безопасности.

> [!NOTE]
> Просмотр веб-страниц здесь ссылается на общий доступ к произвольной веб-сайтов, который большой риск заметно отличаются от использование веб-браузер для доступа к небольшое количество известных веб-сайты администрирования для служб, таких как Azure, Office 365, других поставщиков облачных служб и SaaS приложения.

Стратегии сдерживания предоставления гарантий повышения безопасности путем увеличения количества и типа элементов управления, которые злоумышленник имеет преодолеть, чтобы получить доступ к конфиденциальным ресурсам. Модели, разработанной здесь предоставляет вложения административных привилегий для определенных устройств, с помощью другого уровня прав доступа модели.

## <a name="supply-chain-management"></a>Управление логистическими цепочками

Необходимо для безопасной станции — решение для управления цепочкой поставок где рабочей станции, использовании доверенных, «корневого доверия». Это решение будет адрес корне доверие с помощью [Microsoft Autopilot](https://docs.microsoft.com/windows/deployment/windows-autopilot/windows-autopilot) технологии. Для защищенной рабочей станции Microsoft Autopilot предоставляет возможность использования устройств Microsoft Windows 10, оптимизированный для OEM, которые предоставляют известного рабочего состояния от производителя. Вместо повторного создания образа устройства, которое не установлено доверие, Autopilot Майкрософт можно преобразовать устройства Windows в состояние «готовность», применяя параметры и политики, установка приложений, и даже изменение выпуска Windows 10 используется (например, с Windows 10 Pro до Windows 10 Корпоративная, для поддержки расширенных функций).

## <a name="device-roles-and-profiles"></a>Устройство ролей и профилей

На протяжении всего руководства несколько профилей безопасности и роли будут учтены для достижения более безопасное решение для пользователей, разработчиков и ИТ-персоналу. Эти профили были выровнены для поддержки обычные пользователи в организации, которые могут использовать преимущества расширенной или безопасной рабочей станции, соблюдая разумный баланс между удобством использования и риск. Рекомендации предоставит настройку параметров на основе стандартов промышленным стандартам. В этом руководстве используется в качестве наглядного метода в ограниченный режим работы Windows 10 и снижение рисков, связанных с компрометация устройства или пользователя, с помощью политики и технологии для управления функции безопасности и риски.
![Уровни безопасной рабочей станции](./media/concept-azure-managed-workstation/seccon-levels.png)

* **Низкий безопасности** — управляемый стандартной рабочей станции предоставляет хорошей отправной точкой для большинства бизнес-пользователей домашних и малых. Эти устройства, зарегистрированные в Azure AD, управляемого Intune. Профиль позволяет пользователям запускать любые приложения и просмотреть любой веб-сайт. Это решение для защиты от вредоносных программ, например [Защитника Microsoft](https://www.microsoft.com/windows/comprehensive-security) должен быть включен.
* **Усиленная безопасность** — — это решение начального уровня защищенных, действует для пользователей домашних компьютеров, представителями малого бизнеса, а также общие разработчиков.
   * Расширенный рабочей станции предоставляет политику на основе средства для повышения уровня безопасности от профиля безопасности низким. Этот профиль позволяет защищенные средства для работы с данными о клиентах и иметь возможность использовать средства повышения производительности, таких как проверка электронной почты и просмотра веб-страниц. На рабочей станции расширенный может использоваться для аудита использования профиля рабочей станции и поведением пользователей путем включения политики аудита и ведения журнала в Intune. В этом профиле рабочей станции позволит средства управления безопасностью и политики, описанным в содержимое, а также развернуты в улучшенной рабочей станции - Windows 10 (1809) скрипт. Развертывание также используются преимущества сложных вредоносных программ защита с использованием [Advanced Threat Protection (ATP)](https://docs.microsoft.com/office365/securitycompliance/office-365-atp)
* **Высокий уровень безопасности** — наиболее эффективный способ уменьшения поверхности атаки рабочей станции заключается в удалении возможность самостоятельно администрировать рабочей станции. Удаление права локального администратора является одним из этапов, повышающий безопасность и может повлиять на производительность, если неправильно реализовано. Профиль высокий уровень безопасности основан на профиль безопасности с одно значительное изменение, удаление локального администратора. Этот профиль предназначен для пользователей, которым может быть высокий профиль пользователя, например руководителя или пользователей, которые, возможно, обратитесь в службу с конфиденциальными данными, таких как расчет заработной платы или утверждения служб и процессов.
   * Профиль пользователя высокого уровня безопасности требует более управляемой среде сохранением возможности для выполнения их производительность действия, такие как почта и веб-страниц при сохранении простой в использовании интерфейс. Пользователи ожидают, что функции, такие как файлы cookie, Избранное и другие сочетания клавиш для работы устройства. Тем не менее эти пользователи могут не требовать возможность изменения или отладка свое устройство и не нужно устанавливать драйверы. Высокий уровень безопасности профиля развертывается с помощью высокий уровень безопасности - скрипт Windows 10 (1809).
* **Специализированные** — разработчики и ИТ-администраторов являются привлекательной целью для злоумышленников, как эти роли могут изменять систем интерес для злоумышленников. Специализированные рабочей станции требует усилий, развернутых в рабочей станции высокого уровня безопасности и дальнейшего emphases систему безопасности, управление локальными приложениями, ограничивая веб-узлов и ограничивая возможности повышения производительности, которые могут быть очень высокими риска, такие как ActiveX, Java, подключаемый модуль браузера и несколько других элементов управления известных высоким уровнем риска на устройстве Windows. В этом профиле рабочей станции позволит средства управления безопасностью и политики, описанным в содержимое, а также развернуты в DeviceConfiguration_NCSC - Windows 10 (1803) SecurityBaseline скрипта.
* **Защищенный** — злоумышленник может нарушить безопасность учетной записи администратора обычно повреждают сложной бизнес-логикой, кражи данных, изменения данных или перебоев в работе службы. В этом состоянии жесткой рабочей станции, обеспечивает все средства управления безопасностью и политики, ограничивающие непосредственным управлением локального приложения управления и средств повышения производительности, удаляются. В результате ущерба устройство становится более сложным, как почта и социальных блокируются, которые отражают наиболее распространенным способом добиться успеха фишинг-атаках.  Защищенной рабочей станции можно развернуть с помощью защиты рабочей станции - скрипт SecurityBaseline Windows 10 (1809).

   ![Защищенной рабочей станции](./media/concept-azure-managed-workstation/secure-workstation.png)

   Безопасной рабочей станции предоставляет администратору Усиление защиты рабочей станции, приложений управления и службы application guard. Рабочая станция будет использовать учетные данные, устройства и exploit guard для защиты от вредоносного поведения узла. Кроме того, все локальные диски шифруются с помощью шифрования Bitlocker.

* **Изолированное** — это пользовательский сценарий автономного режима представляет extreme конце спектра (сценарии установки не указаны для этого случая). Организации может потребоваться управлять изолированной бизнеса критические функции например высокое значение производственной линии или систем поддержки жизненного, требующей не поддерживается или без исправлений устаревших операционных систем. Так как безопасность очень важна, и облачные службы недоступны, организации могут вручную управлять или обновить эти компьютеры или использовать изолированная архитектура леса Active Directory (например усиленной безопасности администратора среды (ESAE)) для управления ими. В этих обстоятельствах, удаление доступа, за исключением базовых Intune и ATP должен считаться проверки работоспособности.
   * [Требование к связи сети Intune](https://docs.microsoft.com/intune/network-bandwidth-use)
   * [Требование communications сеть ATP](https://docs.microsoft.com/azure-advanced-threat-protection/configure-proxy)

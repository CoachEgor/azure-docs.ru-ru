# <a name="understand-secure-azure-managed-workstations"></a>Понять безопасные, управляемые Azure рабочих станций

Защищенный и изолированных рабочих станций являются критически важными для безопасности конфиденциальных ролей, таких как администраторы, разработчики и операторы критические службы. Угроза безопасности рабочей станции клиента, многие средства управления безопасностью и гарантии может завершиться ошибкой или не сработает.

В этом документе объясняется, что необходимо для создания безопасной рабочей станции, часто это называют станций с привилегированным доступом (PAW). Эта статья также содержит подробные инструкции по настройке средства управления безопасностью начальной. В этом руководстве описано, как облачные технологии могут управлять службой. Он полагается на возможности обеспечения безопасности, которые были введены в 10RS5 Windows, Microsoft Advanced Threat Protection (ATP в Защитнике), Azure Active Directory и Intune.

> [!NOTE]
> В этой статье объясняется понятие безопасной рабочей станции и рассказывает о роли. Если вы уже знакомы с концепцией и хотите пропустить Развертывание, посетите [развертывания рабочей станции Secure](https://docs.microsoft.com/azure/active-directory/devices/howto-azure-managed-workstation).

## <a name="why-secure-workstation-access-is-important"></a>Почему важен доступ безопасной рабочей станции

Быстрое внедрение облачных служб и возможность работы из любого места, создала новый метод использования. Используя элементы управления должную безопасность на устройствах, где администраторы работают, злоумышленники могут получить доступ к привилегированным ресурсам.

Неправильного использования привилегий и атак цепочки поставок являются верхней пять методов, которые злоумышленники используют для нарушения безопасности организации. Они также второй наиболее часто обнаруживаемых тактики в событиях, в 2018 г. согласно [отчетов Verizon угроз](https://enterprise.verizon.com/resources/reports/dbir/)и [отчета службы безопасности](https://aka.ms/sir).

Большинство злоумышленников выполните следующие действия.

1. Разведка в поиске способа, часто относящиеся к отрасли.
1. Анализ для сбора информации и определить лучший способ проникновения рабочая станция, воспринимается как низкое значение.
1. Сохраняемость искать средства для перемещения [элементам](https://en.wikipedia.org/wiki/Network_Lateral_Movement).
1. Утечка конфиденциальных данных.

Во время разведывательной атаки злоумышленники часто проникновения устройств, которые кажутся низким уровнем риска или недооцененными. Чтобы найти возможности для перемещения по сети и для поиска пользователей и устройств, они используют эти уязвимых устройств. После получения доступа к роли привилегированных пользователей, злоумышленники определите важных данных и успешно вести эксфильтрацию данных.

![Типичный компрометации шаблон](./media/concept-azure-managed-workstation/typical-timeline.png)

В этом документе описывается решение, которое помогает защитить ваши вычислительные устройства от таких атак бокового смещения. Решение изолирует управления и служб от менее ценным устройств повышения производительности, нарушения цепочки, прежде чем можно infiltrated устройство, которое имеет доступ к конфиденциальным облачных ресурсов. В решении используются собственные службы Azure, которые являются частью стека Microsoft 365 Enterprise:

* Intune для управления устройствами и список надежных приложений и URL-адреса
* AutoPilot для настройки устройства, развертывания и обновления
* Azure AD для управления пользователями, условного доступа и многофакторной проверки подлинности
* Windows 10 (версии) для устройства работоспособности аттестации и взаимодействия с пользователем
* ATP в Защитнике для защиты облачной конечной точкой, обнаружения и реагирования
* Azure AD PIM для управления авторизации и just-in-time (JIT) привилегированный доступ к ресурсам

## <a name="who-benefits-from-a-secure-workstation"></a>Кто использует преимущества безопасной рабочей станции?

Преимущества при использовании безопасной рабочей станции для всех пользователей и операторы. Представляет угрозу для любой компьютер или устройство, злоумышленник может олицетворять все кэшированные учетные записи. При входе на устройство, они также могут использовать учетные данные и токены. Этот риск в результате важно обеспечивать безопасность устройств, которые используются для привилегированных ролей, включая права администратора. Устройства с привилегированными учетными записями, целевые объекты для бокового смещения и атаки повышения привилегий. Эти учетные записи можно использовать для различных средств, таких как:

* Администратор для локальных и облачных систем
* Рабочей станции разработчика для критически важных систем
* Администраторам учетных записей социальных высокой уязвимости
* Рабочей станции, строго конфиденциальными, например SWIFT терминал оплаты
* Рабочая станция обработки тайны

Чтобы снизить риск, следует реализовать средства управления безопасностью с повышенными правами для рабочих станций с привилегированным, упрощающих использование этих учетных записей. Дополнительные сведения см. в разделе [руководство по развертыванию компонентов Azure Active Directory](https://docs.microsoft.com/azure/active-directory/fundamentals/active-directory-deployment-checklist-p2), [план Office 365](https://aka.ms/o365secroadmap), и [стратегии обеспечения безопасности привилегированного доступа](https://aka.ms/sparoadmap)).

## <a name="why-use-dedicated-workstations"></a>Зачем использовать выделенных рабочих станций?

Хотя можно добавить в существующее устройство безопасности, лучше начать с защищенной платформой. Для помещения вашей организации, наилучшим образом, чтобы поддерживать высокий уровень безопасности, начните с устройства, вы знаете, безопасность и реализовать набор элементов безопасности.

Все большее число атак через сообщения электронной почты и просмотра веб-страниц все более затрудняет убедитесь, что устройство можно доверять. В этом руководстве предполагается, что выделенная рабочая станция изолирован от стандартный продуктивность, просмотра и электронной почты. Удаление продуктивность, просмотра веб-страниц и электронной почты на устройстве может иметь негативное влияние на производительность. Тем не менее эта мера безопасности обычно приемлемо для сценариев, где задачи задания не требуются явно и велик риск нарушения безопасности.

> [!NOTE]
> Просмотр веб-страниц здесь ссылается на общий доступ к произвольной веб-сайтов, который может быть действием высоким уровнем риска. Просмотр таких заметно отличаются от использование веб-браузер для доступа к небольшое количество известных веб-сайты администрирования для служб, таких как Azure, Office 365, других поставщиков облачных служб и приложений SaaS.

Стратегии сдерживания усиления безопасности путем увеличения количества и типа элементов управления, для предотвращения злоумышленник проникновение конфиденциальные ресурсы. Модель, описанную в этой статье используется многоуровневый привилегий дизайн и ограничивает права администратора для определенных устройств.

## <a name="supply-chain-management"></a>Управление логистическими цепочками

Необходимо для безопасной станции используется решение для управления цепочкой поставок где надежных рабочих станций, называется «корне доверия». Для этого решения использует корне доверия [Microsoft Autopilot](https://docs.microsoft.com/windows/deployment/windows-autopilot/windows-autopilot) технологии. Для защиты рабочей станции, Autopilot позволяет использовать устройства Microsoft Windows 10, оптимизированный для OEM. Эти устройства производителем в известное работоспособное состояние. Вместо повторного создания образа потенциально небезопасной устройства, Autopilot можно преобразовать в состояние «готовность» на устройстве Windows. Он применяет параметры и политики, устанавливает приложения и даже изменяет выпуск Windows 10. Например Autopilot должны измениться установки на устройстве с Windows из Windows 10 Pro до Windows 10 Корпоративная, чтобы он мог использовать расширенные возможности.

## <a name="device-roles-and-profiles"></a>Устройство ролей и профилей

В этом руководстве ссылается на несколько профилей безопасности и ролей, которые помогут вам создавать более надежные решения для пользователей, разработчиков и ИТ-специалистов. Эти профили сбалансировать удобство использования и риски для обычных пользователей, которые могут использовать преимущества расширенной или безопасной рабочей станции. Параметры конфигурации, представленные в этом разделе основаны на стандартах промышленным стандартам. В этом руководстве показано, как для повышения безопасности Windows 10 и уменьшить риски, связанные с компрометация устройства или пользователя. Это делается с помощью политики и технологии для управления функции безопасности и риски.
![Уровни безопасной рабочей станции](./media/concept-azure-managed-workstation/seccon-levels.png)

* **Низкий безопасности** — управляемый, Стандартная рабочая станция предоставляет хорошей отправной точкой для большинства бизнес-пользователей домашних и малых. Эти устройства регистрируются в Azure AD и управляемых с помощью Intune. Этот профиль позволяет пользователям запускать любые приложения и просмотреть любой веб-сайт. Это решение для защиты от вредоносных программ, например [Защитника Microsoft](https://www.microsoft.com/windows/comprehensive-security) должен быть включен.

* **Усиленная безопасность** — это решение начального уровня, защищенных хорошо подходит для пользователей домашних компьютеров, представителями малого бизнеса и общие разработчиков.

   Улучшенные рабочей станции способ на основе политик для повышения безопасности профиля с низким уровнем безопасности. Он предоставляет безопасные средства для работы с данными клиента, а также с использованием средств повышения производительности, как электронная почта и просмотра веб-страниц. Политики аудита и Intune можно использовать для мониторинга использования поведение и профиль пользователя улучшенные рабочей станции. Вы развертываете профиль улучшенные рабочей станции с помощью сценария Windows 10 (1809), и он использует преимущества сложных вредоносных программ защита с использованием [Advanced Threat Protection (ATP)](https://docs.microsoft.com/office365/securitycompliance/office-365-atp).

* **Высокий уровень безопасности** — наиболее эффективный способ уменьшения поверхности атаки рабочей станции заключается в удалении возможность самостоятельно администрировать рабочей станции. Удаление права локального администратора — это этап, который повышает безопасность, но он может повлиять на производительность, если неверной реализации. Профиль высокий уровень безопасности основан на профиль безопасности с одно значительное изменение: удаление локального администратора. Этот профиль предназначен для пользователей высокий профиль: руководители, расчет заработной платы и конфиденциальных данных для пользователей, утверждающих лиц для служб и процессов.

   Высокого уровня безопасности пользователь требует более контролируемую среду, сохраняя при этом возможность выполнять действия, такие как электронная почта и веб-страниц в более простой в использовании возможности. Пользователи ожидают, что функции, такие как файлы cookie "Избранное" и другие сочетания клавиш для работы. Тем не менее эти пользователи не требуется возможность изменять или отладка свое устройство. Они также не нужно устанавливать драйверы. Высокий уровень безопасности профиля развертывается с помощью высокий уровень безопасности - скрипт Windows 10 (1809).

* **Специализированные** — злоумышленники совершают разработчиков и ИТ-администраторов, так как они могут изменять систем интерес для злоумышленников. Специализированные рабочей станции расширяет политики высокого уровня безопасности рабочей станции управления локальными приложениями и ограничив веб-сайтов. Он также ограничивает возможности повышения производительности с высоким риском, такие как ActiveX, Java, подключаемые модули браузера и других элементов управления Windows. Разверните этот профиль с DeviceConfiguration_NCSC - скрипт SecurityBaseline Windows 10 (1803).

* **Защищенный** — представляет угрозу для учетной записи администратора, злоумышленник может нанести ущерб сложной бизнес-логикой кражи данных, изменения данных или перебоев в работе службы. В этом состоянии жесткой рабочей станции включает все средства управления безопасностью и политики, ограничивающие непосредственным управлением локального приложения управления. Защищенной рабочей станции имеет нет средств повышения производительности, поэтому устройство, более сложным, нарушения. Он блокируется наиболее распространенных вектора для фишинг-атаках: адрес электронной почты и социальные сети.  Защищенной рабочей станции можно развернуть с помощью защиты рабочей станции - скрипт SecurityBaseline Windows 10 (1809).

   ![Защищенной рабочей станции](./media/concept-azure-managed-workstation/secure-workstation.png)

   Безопасной рабочей станции предоставляет администратору Усиление защиты рабочей станции управления приложений и службы application guard. Рабочая станция использует компонент credential guard, device guard и exploit guard для защиты от вредоносного поведения узла. Кроме того, все локальные диски шифруются по технологии BitLocker.

* **Изолированное** — этот сценарий пользовательские «вне сети» представляет extreme конце спектра. Скрипты установки не предоставляются для этого случая. Может потребоваться для управления критически важных бизнес-функции, требующей прежних версий операционной системы не поддерживается или незащищенной. Например высокое значение производственной линии или в жизнь – системах. Так как безопасность очень важна, и облачные службы недоступны, можно управлять и обновлять эти компьютеры вручную или с изолированной архитектуры леса Active Directory, например усиленной безопасности администратора среды (ESAE). В этих случаях рекомендуется удалить доступа, за исключением основные проверки работоспособности Intune и ATP.

  * [Требование к связи сети Intune](https://docs.microsoft.com/intune/network-bandwidth-use)
  * [Требование communications сеть ATP](https://docs.microsoft.com/azure-advanced-threat-protection/configure-proxy)

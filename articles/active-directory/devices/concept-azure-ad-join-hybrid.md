# <a name="hybrid-azure-ad-joined-devices"></a>Гибридные устройства, присоединенные к Azure AD

На протяжении более десяти лет многие организации использовали присоединение к доменам в своей локальной среде Active Directory для предоставления возможностей:

- ИТ-отделам для управления корпоративными устройствами центрального расположения;
- пользователям для входа на устройства с помощью рабочих или учебных учетных записей Active Directory.

Как правило, организации с присутствием на месте полагаются на методы визуализации для предоставления устройств, и они часто используют **configuration Manager** или групповую **политику (GP)** для управления ими.

Если в вашей среде действует локальная служба AD и вы также хотите получить выгоду от возможностей, предоставляемых Azure Active Directory, можно использовать гибридные устройства, присоединенные к Azure AD. Эти устройства, являются устройствами, которые соединены с вашим наемным Active Directory и зарегистрированы в вашем active Directory Azure.

|   | Гибридное присоединение к Azure AD |
| --- | --- |
| **Определение** | Присоединился к on-premises AD и Azure AD, требующим, чтобы организационный учет вошел в устройство |
| **Основная аудитория** | Подходит для гибридных организаций с существующей инфраструктурой AD |
|   | Применяется ко всем пользователям в организации |
| **Владение устройствами** | План |
| **Операционные системы** | Windows 10, 8.1 и 7 |
|   | Windows Server 2008/R2, 2012/R2, 2016 и 2019 |
| **Провизионирование** | Windows 10, Windows Server 2016/2019 |
|   | Домен присоединяется к ИТ и автоматически соединяется через Azure AD Connect или aDFS config |
|   | Домен присоединиться к Windows Autopilot и автосоединение через Azure AD Connect или ADFS config |
|   | Windows 8.1, Windows 7, Windows Server 2012 R2, Windows Server 2012 и Windows Server 2008 R2 - Требуйте MSI |
| **Регистрация устройства в опциях** | Организационные счета с использованием: |
|   | Пароль |
|   | Windows Привет для бизнеса для Win10 |
| **Управление устройствами** | Групповая политика |
|   | Настройка менеджера автономного или совместного управления с Microsoft Intune |
| **Ключевые возможности** | SSO для облачных и локтевых ресурсов |
|   | Условный доступ через домен присоединиться или через Intune, если совместно ею |
|   | Самообслуживание Сбросить паролей и Windows Hello PIN сбросить на экране блокировки |
|   | Корпоративное государственное роуминг через устройства |

## <a name="scenarios"></a>Сценарии

Используйте гибридные устройства Azure AD, если:

- На этих устройствах развернуты приложения Win32, которые полагаются на аутентификацию компьютера с помощью Active Directory.
- Необходимо продолжать использовать групповую политику для управления конфигурацией устройств.
- Необходимо продолжать использовать существующие решения для развертывания и настройки устройств.
- Вы должны поддерживать устройства Windows 7 и 8.1 в дополнение к Windows 10

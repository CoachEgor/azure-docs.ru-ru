# <a name="hybrid-azure-ad-joined-devices"></a>Гибридные устройства, присоединенные к Azure AD

На протяжении более десяти лет многие организации использовали присоединение к доменам в своей локальной среде Active Directory для предоставления возможностей:

- ИТ-отделам для управления корпоративными устройствами центрального расположения;
- пользователям для входа на устройства с помощью рабочих или учебных учетных записей Active Directory.

Как правило, организации с локальной службой создают для подготовки устройств системные образы, а для управления устройствами часто применяют **System Center Configuration Manager (SCCM)** или **групповые политики**.

Если в вашей среде действует локальная служба AD и вы также хотите получить выгоду от возможностей, предоставляемых Azure Active Directory, можно использовать гибридные устройства, присоединенные к Azure AD. Эти устройства, устройств, присоединенных к вашей локальной Active Directory и зарегистрировано в Azure Active Directory.

|   | Гибридное присоединение к Azure AD |
| --- | --- |
| **Определение** | Объединить с локальным AD и Azure AD требуется учетная запись организации для входа в устройство |
| **Основная аудитория** | Подходит для гибридных организаций с существующей локальной инфраструктуры AD |
|   | Ко всем пользователям в организации |
| **Владение устройством** | План |
| **Операционные системы** | Windows 10, 8.1 и 7 |
|   | Windows Server 2008/R2, 2012 или R2, 2016 и 2019 г. |
| **Подготовка** | Windows 10, Windows Server 2016/2019 г. |
|   | Присоединение к домену по ИТ и объединение с помощью Azure AD Connect или служб федерации Active Directory config |
|   | Присоединение к домену Windows Autopilot и объединение с помощью конфигурации Azure AD Connect или служб федерации Active Directory |
|   | Windows 8.1, Windows 7, Windows Server 2012 R2, Windows Server 2012 и Windows Server 2008 R2 — требовать MSI |
| **Варианты входа устройства** | Учетные записи в организации с помощью: |
|   | Пароль |
|   | Windows Hello для бизнеса для Windows 10 |
| **Управление устройствами** | Групповая политика |
|   | System Center Configuration Manager отдельно или совместного управления с помощью Microsoft Intune |
| **Ключевые возможности** | Единый вход для облачных и локальных ресурсов |
|   | Условный доступ посредством присоединения к домену или с помощью Intune, если управляемые |
|   | Самостоятельного сброса пароля и Windows Hello ПИН-код сброса на экране блокировки |
|   | Enterprise State Roaming для устройств |

## <a name="scenarios"></a>Сценарии

Гибридного использования Azure AD устройств, присоединенных к если:

- На этих устройствах развернуты приложения Win32, которые полагаются на аутентификацию компьютера с помощью Active Directory.
- Вы хотите продолжать использовать групповую политику для управления конфигурацией устройства.
- Вы хотите продолжать использовать существующие решения для работы с образами для развертывания и настройки устройств.
- Требуется поддержка ранних версий Windows 7 и устройства с версией 8.1 в дополнение к Windows 10

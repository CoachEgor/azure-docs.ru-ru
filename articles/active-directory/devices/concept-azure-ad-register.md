# <a name="azure-ad-registered-devices"></a>Устройства, зарегистрированные в Azure AD

Целью зарегистрированных устройств Azure AD является предоставление пользователям поддержки сценариев использования собственных устройств (BYOD) или мобильных устройств. В этих сценариях пользователь может получить доступ к контролируемым ресурсам Организации Azure Active Directory с помощью личного устройства.

|   | Зарегистрировано в Azure AD |
| --- | --- |
| **Определение** | Зарегистрировано в Azure AD, не требуя от учетной записи организации входа на устройство |
| **Основная аудитория** | Применимо для всех пользователей со следующими условиями: |
|   | Принеси свое устройство (BYOD) |
|   | Мобильные устройства |
| **Владение устройствами** | Пользователь или организация |
| **Операционные системы** | Windows 10, iOS, Android и MacOS |
| **Провизионирование** | Windows 10 — параметры |
|   | iOS/Android — корпоративный портал или Microsoft Authenticator приложение |
|   | MacOS — корпоративный портал |
| **Параметры входа устройства** | Локальные учетные данные конечного пользователя |
|   | Пароль |
|   | Windows Hello |
|   | Пин-код |
|   | Биометрия или шаблон для других устройств |
| **Управление устройствами** | Управление мобильными устройствами (пример: Microsoft Intune) |
|   | управление мобильными приложениями |
| **Ключевые возможности** | Единый вход для облачных ресурсов |
|   | Условный доступ при регистрации в Intune |
|   | Условный доступ через политику защиты приложений |
|   | Включает вход с помощью телефона в приложении Microsoft Authenticator |

![Устройства, зарегистрированные в Azure AD](./media/concept-azure-ad-register/azure-ad-registered-device.png)

Зарегистрированные устройства Azure AD входят в локальную учетную запись, например на учетная запись Майкрософт на устройстве с Windows 10, но для доступа к ресурсам Организации также требуется учетная запись Azure AD. Доступ к ресурсам в Организации может быть более ограничен в зависимости от того, что учетная запись Azure AD и политики условного доступа применяются к удостоверению устройства.

Администраторы могут защищать и контролировать эти устройства, зарегистрированные Azure AD, с помощью таких средств управления мобильными устройствами (MDM), как Microsoft Intune. MDM предоставляет средства для применения необходимых для Организации конфигураций, таких как требование шифрования хранилища, сложность пароля и обновление программного обеспечения безопасности. 

Регистрация Azure AD может быть выполнена при первом обращении к рабочему приложению или вручную с помощью меню параметров Windows 10. 

## <a name="scenarios"></a>Сценарии

Пользователь в Организации хочет получить доступ к средствам электронной почты, отправке отчетов по времени и преимуществам при регистрации с домашнего компьютера. В вашей организации имеются эти средства для политики условного доступа, которым требуется доступ с устройства, совместимого с Intune. Пользователь добавляет свою учетную запись организации и регистрирует свой домашний ПК в Azure AD, а необходимые политики Intune применяются для предоставления пользователю доступа к своим ресурсам.

Другой пользователь хочет получить доступ к электронной почте Организации на своем личном телефоне Android, который был корневым. Для вашей компании требуется соответствующее устройство и создана политика соответствия Intune для блокировки всех устройств с корнем. Сотрудник прекратил доступ к ресурсам Организации на этом устройстве.

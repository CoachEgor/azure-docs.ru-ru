# <a name="azure-ad-registered-devices"></a>Устройства, зарегистрированные в Azure AD

Устройства, зарегистрированные в Azure AD призван предоставить пользователям для поддержки принеси свое устройство (BYOD) и сценарии мобильными устройствами. В этих сценариях доступа пользователя к ресурсам вашей организации управлять Azure Active Directory, с помощью личного устройства.

|   | Зарегистрированные в Azure AD |
| --- | --- |
| **Определение** | Зарегистрирован в Azure AD без необходимости организации запись для входа на устройство |
| **Основная аудитория** | Применимо ко всем пользователям, включающую следующие условия: |
|   | Принеси свое устройство (BYOD) |
|   | Мобильные устройства |
| **Владение устройством** | Пользователя или организации |
| **Операционные системы** | Windows 10, iOS, Android и MacOS |
| **Подготовка** | Windows 10 — параметры |
|   | iOS и Android — корпоративного портала или Microsoft приложения проверки подлинности |
|   | Корпоративный портал MacOS |
| **Варианты входа устройства** | Локальные учетные данные конечного пользователя |
|   | Пароль |
|   | Windows Hello |
|   | PIN |
|   | Биометрические данные или шаблон для других устройств |
| **Управление устройствами** | Управление мобильными устройствами (пример: Microsoft Intune) |
|   | управление мобильными приложениями |
| **Ключевые возможности** | Единый вход для облачных ресурсов |
|   | Условный доступ при регистрации в Intune |
|   | Условный доступ с помощью политики защиты приложений |
|   | Позволяет Phone входа через приложение Microsoft Authenticator |

![Устройства, зарегистрированные в Azure AD](./media/concept-azure-ad-register/azure-ad-registered-device.png)

Устройства Azure AD регистрации входа в систему с помощью локальной учетной записи, например учетную запись Майкрософт на устройстве Windows 10, но кроме того, у учетной записи Azure AD для доступа к ресурсам организации. Доступ к ресурсам в организации может быть дальнейших limited, на основе этой учетной записи Azure AD и применяться к удостоверению устройства, политики условного доступа.

Администраторы могут защитить и последующего управления этих устройств зарегистрированные в Azure AD, с помощью средств управления мобильными устройствами (MDM), как Microsoft Intune. MDM предоставляет средства для принудительного применения конфигурации, необходимые организации, например хранилища должны быть зашифрованы, сложность паролей и программного обеспечения безопасности, обновления. 

Регистрация в Azure AD может быть выполнено при доступе к рабочему приложению в первый раз или вручную с помощью меню "Параметры Windows 10". 

## <a name="scenarios"></a>Сценарии

Пользователь в вашей организации хочет получить доступ к средствам для электронной почты, reporting свободного времени и преимущества регистрации из своих домашних ПК. Вашей организации есть эти средства за политику условного доступа, которому требуется доступ с совместимые устройства Intune. Пользователь добавляет свою учетную запись организации и регистрирует их домашних ПК в Azure AD, и применяются необходимые политики Intune, предоставляя пользователю доступ к своим ресурсам.

Другой пользователь хочет получить доступ к электронной почте организации на их личных телефоне Android, который получен административный доступ. Компании требуется соответствующее устройство и создал политику соответствия Intune, чтобы блокировать все устройства с административным доступом. Сотрудник остановлена, доступ к ресурсам организации на этом устройстве.

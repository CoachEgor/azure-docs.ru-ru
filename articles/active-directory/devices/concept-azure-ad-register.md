# <a name="azure-ad-registered-devices"></a>Устройства, зарегистрированные в Azure AD

Цельзарегистрированных устройств Azure AD состоит в том, чтобы предоставить пользователям поддержку сценариев Bring Your Own Device (BYOD) или мобильных устройств. В этих сценариях пользователь может получить доступ к ресурсам, управляемым Active Directory в вашей организации, с помощью личного устройства.

|   | Зарегистрировано aD Azure |
| --- | --- |
| **Определение** | Зарегистрирован в Azure AD, не требуя, чтобы организационный учет входе в устройство |
| **Основная аудитория** | Применяется ко всем пользователям со следующими критериями: |
|   | Принеси свое устройство (BYOD) |
|   | Мобильные устройства |
| **Владение устройствами** | Пользователь или организация |
| **Операционные системы** | Windows 10, iOS, Android и MacOS |
| **Провизионирование** | Windows 10 - Настройки |
|   | iOS/Android - Приложение компании "Портал" или приложение Microsoft Authenticator |
|   | MacOS - Портал компании |
| **Регистрация устройства в опциях** | Локальные учетные данные конечных пользователей |
|   | Пароль |
|   | Windows Hello |
|   | Пин-код |
|   | Биометрия или шаблон для других устройств |
| **Управление устройствами** | Управление мобильными устройствами (пример: Microsoft Intune) |
|   | управление мобильными приложениями |
| **Ключевые возможности** | SSO для облачных ресурсов |
|   | Условный доступ при зачислении в Intune |
|   | Условный доступ через политику защиты приложений |
|   | Позволяет войти в телефон с приложением Microsoft Authenticator |

![Устройства, зарегистрированные в Azure AD](./media/concept-azure-ad-register/azure-ad-registered-device.png)

Зарегистрированные устройства Azure AD подписываются на использование локальной учетной записи, такой как учетная запись Майкрософт на устройстве Windows 10, но дополнительно есть учетная запись Azure AD, прикрепленная для доступа к организационным ресурсам. Доступ к ресурсам в организации может быть еще более ограничен в зависимости от того, что учетная запись Azure AD и политики условного доступа, применяемые к идентификатору устройства.

Администраторы могут обеспечить безопасность и дальнейшее управление этими устройствами Azure AD, используя инструменты управления мобильными устройствами (MDM), такие как Microsoft Intune. MDM предоставляет средства для обеспечения соблюдения требуемых организацией конфигураций, таких как требование шифрования хранилища, сложность паролей и обновление программного обеспечения безопасности. 

Регистрация Azure AD может быть осуществлена при первом доступе к рабочему приложению или вручную с помощью меню настроек Windows 10. 

## <a name="scenarios"></a>Сценарии

Пользователь в вашей организации хочет получить доступ к инструментам для электронной почты, отчетности отойти, а также преимущества регистрации с своего домашнего компьютера. В вашей организации есть эти инструменты, лежащие в основе политики условного доступа, которая требует доступа с устройства, совместимого с Intune. Пользователь добавляет свою учетную запись организации и регистрирует свой домашний компьютер с Azure AD, и необходимые политики Intune применяются, предоставляя пользователю доступ к их ресурсам.

Другой пользователь хочет получить доступ к своей организационной электронной почты на своем личном телефоне Android, который был укоренен. Ваша компания требует совместимого устройства и создала политику соответствия Intune для блокировки любых корневых устройств. Сотруднику не будет доступ к организационным ресурсам на этом устройстве.

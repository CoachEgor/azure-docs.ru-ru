# <a name="manage-device-identities-using-the-azure-portal"></a>Управление идентификаторами устройств с помощью портала Azure

С управлением идентификацией устройств в Каталоге Active Azure (Azure AD) можно гарантировать, что пользователи получают доступ к вашим ресурсам с устройств, отвечающих вашим стандартам безопасности и соответствия требованиям.

В этой статье:

- Предполагается, что вы знакомы с [введением в управление идентификацией устройств в Active Directory Azure](overview.md)
- Предоставляет информацию об управлении идентификаторами устройства с помощью портала Azure AD

## <a name="manage-device-identities"></a>Управление удостоверениями устройств

Портал Azure AD предоставляет вам центральное место для управления идентификаторами устройств. Вы можете добраться до этого места, используя [прямую ссылку](https://portal.azure.com/#blade/Microsoft_AAD_IAM/DevicesMenuBlade/Devices) или:

1. Войдите на [портал Azure](https://portal.azure.com).
1. Просмотрите доступ к > **активным устройствам** **каталога Azure.**

На странице **Устройства** можно:

- Настройка параметров устройства
- Поиск устройств
- Выполнение задач управления идентификацией устройств
- Просмотр журналов аудита, связанных с устройствами  
  
## <a name="configure-device-settings"></a>Настройка параметров устройства

Для управления идентификаторами устройств с помощью портала Azure AD устройства должны быть [зарегистрированы или соединены с](overview.md) Azure AD. Администратор может точно настроить процесс регистрации и присоединения устройств, задав параметры устройства.

Страница настроек устройства позволяет настроить настройки, связанные с идентификаторами устройства:

![Управление устройством Intune](./media/device-management-azure-portal/21.png)

- **Пользователи могут привязываться к устройствам azure AD** - Эта настройка позволяет выбрать пользователей, которые могут зарегистрировать свои устройства в качестве устройств, связанных с Azure AD. Значение по умолчанию — **All**.

> [!NOTE]
> **Пользователи могут присоединяться к устройствам к** настройкам Azure AD, применимо только к присоединению Azure AD к Windows 10.

- **Дополнительные локальные администраторы на устройствах, присоединенных к Azure AD**. Этот параметр позволяет выбрать пользователей, которым предоставляются права локального администратора на устройстве. Добавленные здесь пользователи добавляются в роль *Администраторы устройств* в Azure AD. По умолчанию права локального администратора предоставляются глобальным администраторам в Azure AD и владельцам устройств. Этот параметр относится к возможностям выпуска Premium и доступен в таких продуктах, как Azure AD Premium или Enterprise Mobility Suite (EMS).
- **Пользователи могут зарегистрировать свои устройства с помощью Azure AD** - Вам необходимо настроить эту настройку, чтобы позволить Windows 10 персональные, iOS, Android и macOs устройства, которые будут зарегистрированы с Azure AD. При выборе **Нет,** устройства не могут зарегистрироваться в Azure AD. Для регистрации в Microsoft Intune или службе управления мобильными устройствами (MDM) для Office 365 требуется регистрация. Если вы настроили одну из этих служб, **все** выбраны, и **нет.**
- **Требуйте присоединения К устройствам Multi-Factor Auth** - Вы можете выбрать, должны ли пользователи предоставить дополнительный фактор аутентификации для присоединения своего устройства к Azure AD. Значение по умолчанию — **No**. Рекомендуется использовать Многофакторную идентификацию при регистрации устройства. Перед включением Многофакторной идентификации для этой службы необходимо убедиться, что Многофакторная идентификация настроена для пользователей, которые регистрируют свои устройства. Дополнительные сведения о различных службах Многофакторной идентификации в Azure см. в статье [Приступая к работе со службой "Многофакторная идентификация Microsoft Azure" в облаке](../authentication/concept-mfa-whichversion.md). 

> [!NOTE]
> **Требование Multi-Factor Auth присоединиться к устройствам** применяется к устройствам, которые либо присоединились Azure AD, либо зарегистрированы Azure AD. Эта настройка не применяется к гибридным устройствам Azure AD.

- **Максимальное количество устройств** - Эта настройка позволяет выбрать максимальное количество зарегистрированных устройств Azure AD или Azure AD, которые пользователь может иметь в Azure AD. По достижении этой квоты пользователь больше не сможет добавлять дополнительные устройства до тех пор, пока не будет удалены одно или несколько существующих устройств. По умолчанию используется значение **20**.

> [!NOTE]
> **Максимальное количество устройств,** насчитывающих настройки, применяется к устройствам, которые либо присоединились к Azure AD, либо зарегистрированы Azure AD. Эта настройка не применяется к гибридным устройствам Azure AD.

- **Пользователи могут выполнять синхронизацию параметров и данных разных устройств**. По умолчанию этот параметр имеет значение **Нет**. Выбор конкретных пользователей, групп или значения "Все" позволяет синхронизировать параметры и данные приложений пользователя на его устройствах Windows 10. Узнайте больше о том, как работает синхронизация в Windows 10.
Этот параметр относится к возможностям Premium и доступен в таких продуктах, как Azure AD Premium или Enterprise Mobility Suite (EMS).

## <a name="locate-devices"></a>Поиск устройств

Есть два варианта обнаружения зарегистрированных и присоединенных к домену устройств:

- **Все устройства** в разделе **Управление** на странице **Устройства**;  
- **Устройства** в разделе **Управление** на странице **Пользователь**.

С помощью любого из этих средств можно получить представление, которое:

- Позволяет осуществлять поиск устройств, использующих имя дисплея или идентификатор устройства в качестве фильтра.
- предоставляет подробный обзор зарегистрированных и присоединенных к домену устройств;
- позволяет выполнять общие задачи управления устройствами.

![Все устройства](./media/device-management-azure-portal/51.png)

>[!TIP]
>
>* Если вы видите устройство, которое является "Hybrid Azure AD, соединено" с состоянием "В ожидании" под столбцом REGISTERED, это означает, что устройство было синхронизировано с подключением Azure AD и ожидает завершения регистрации от клиента. Узнайте больше о том, как [спланировать внедрение гибридного Azure AD.](hybrid-azuread-join-plan.md) Дополнительную информацию можно найти в статье, [Устройства часто задают вопросы](faq.md).
>
>   ![Устройства, ожидающие ожидания](./media/device-management-azure-portal/75.png)
>
>* Для некоторых устройств iOS, имена которых содержат апострофы, потенциально можно использовать другие знаки, которые выглядят как апострофы. Так что поиск таких устройств немного сложнее - если вы не видите результаты поиска правильно, убедитесь, что строка поиска содержит соответствующий символ апострофа.

## <a name="device-identity-management-tasks"></a>Задачи управления идентификацией устройств

Как глобальный администратор или администратор облачных устройств, вы можете управлять зарегистрированными или объединенными устройствами. Администраторы службы Intune могут:

- обновлять устройства — например, при ежедневных операциях, таких как включение и отключение устройств;
- удалять устройства — когда устройство списывается и должно быть удалено из Azure AD.

В этом разделе содержится информация об общих задачах управления идентификацией устройств.

### <a name="manage-an-intune-device"></a>Управление устройством Intune

Если вы являетесь администратором Intune, то вы можете управлять устройствами, которые отмечены как **Microsoft Intune**. Если устройство не зарегистрировано в Microsoft Intune, опция "Управление" будет серой.

![Управление устройством Intune](./media/device-management-azure-portal/31.png)

### <a name="enable--disable-an-azure-ad-device"></a>Включение или отключение устройства Azure AD

Включить или отключить устройство можно с помощью:

- меню задач ("...") на странице **Все устройства**;

   ![Управление устройством Intune](./media/device-management-azure-portal/71.png)

- панели инструментов на странице **Устройства**.

   ![Управление устройством Intune](./media/device-management-azure-portal/32.png)

**Примечания:**

- Для включения/отключаемого устройства необходимо быть глобальным администратором или администратором облачных устройств. 
- Отключение устройства предотвращает успешную аутентификацию устройства с помощью Azure AD, тем самым не позволяя устройству получить доступ к ресурсам Azure AD, которые охраняются с помощью CA-устройства или используют учетные данные WH4B.
- Отключение устройства отменяет как токен Primary Refresh Token (PRT), так и любые токены Refresh (RT) на устройстве.

### <a name="delete-an-azure-ad-device"></a>Удаление устройства Azure AD

Удалить отключить устройство можно с помощью:

- меню задач ("...") на странице **Все устройства**;

   ![Управление устройством Intune](./media/device-management-azure-portal/72.png)

- панели инструментов на странице **Устройства**.

   ![Удаление устройства](./media/device-management-azure-portal/34.png)

**Примечания:**

- Для удаления устройства требуются права глобального администратора или администратора Intune в Azure AD.
- Удаление устройства:
   - оно больше не сможет обращаться к вашим ресурсам Azure AD;
   - будут удалены все данные, связанные с устройством, например ключи BitLocker для устройств Windows;  
   - вы не сможете отменить это действие (не рекомендуется выполнять без необходимости).

Если устройство управляется другим органом управления (например, Microsoft Intune), убедитесь, что устройство было уничтожено/ удалено перед удалянием устройства в Azure AD. Просмотрите, как [управлять устаревшими устройствами](device-management-azure-portal.md) перед удалять любые устройства.

### <a name="view-or-copy-device-id"></a>Просмотр и копирование идентификатора устройства

С помощью идентификатора устройства можно проверять соответствующие сведения об устройстве или устранять неполадки, используя PowerShell. Чтобы перейти к функции копирования, щелкните устройство.

![Просмотр идентификатора устройства](./media/device-management-azure-portal/35.png)
  
### <a name="view-or-copy-bitlocker-keys"></a>Просмотр и копирование ключей BitLocker

Вы можете просмотреть и скопировать ключи BitLocker, чтобы помочь пользователям восстановить свои зашифрованные диски. Эти ключи доступны только для устройств Windows, которые были зашифрованы и ключи которых были сохранены в Azure AD. Эти ключи можно скопировать, отобразив сведения об устройстве.

![Просмотр ключей BitLocker](./media/device-management-azure-portal/36.png)

Чтобы просмотреть или копировать ключи BitLocker, необходимо быть владельцем устройства или пользователем, которому назначена по крайней мере одна из следующих ролей:

- Администратор облачного устройства
- глобального администратора;
- Администратор службы технической поддержки
- Администратор службы Intune
- Администратор безопасности
- Читатель сведений о безопасности

> [!NOTE]
> Устройства Windows 10, присоединенные к гибридной среде Azure AD, не имеют владельца. Таким образом, если вы ищете устройство по владельцу и оно не обнаруживается, выполните поиск по идентификатору устройства.

## <a name="audit-logs"></a>Журналы аудита

Действия устройств можно просмотреть в журналах действий. Эти журналы включают действия, инициированные службой регистрации устройств и пользователями:

- создание устройства и добавление владельцев или пользователей устройства;
- изменение параметров устройства;
- операции с устройством, например удаление или обновление устройства.

Анализ данных аудита следует начать с **журналов аудита** в разделе **Действие** на странице **Устройства**.

В журнале аудита есть представление списка по умолчанию, показуевакоторое:

- дата и время выполнения действия;
- целевые объекты;
- инициатор или субъект действия;
- действие.

![Журналы аудита](./media/device-management-azure-portal/63.png)

Вы можете настроить представление списка, нажав **столбцы** в панели инструментов.

![Журналы аудита](./media/device-management-azure-portal/64.png)

Для сужения результатов до подходящего уровня вы можете отфильтровать данные аудита, используя следующие поля:

- Категория
- Тип ресурса действия
- Действие
- Диапазон даты
- Назначение
- "Кем инициировано (субъект)".

Помимо применения фильтров можно выполнить поиск конкретных записей.

![Журналы аудита](./media/device-management-azure-portal/65.png)

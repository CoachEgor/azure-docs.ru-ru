# <a name="azure-ad-joined-devices"></a>Устройства, присоединенные к Azure AD

Присоединение к Azure AD предназначено для организаций, которые хотят быть облачным или только для облака. Любой организации можно развернуть присоединенные к Azure AD устройств, независимо от размера и отрасли. Присоединение к Azure AD работает даже в гибридной среде, чтобы предоставить доступ к локальным и облачным приложениям и ресурсам.

|   | Присоединение к Azure AD |
| --- | --- |
| **Определение** | Объединить только с Azure AD, требуется учетная запись организации, войдите на устройство |
| **Основная аудитория** | Подходит для обоих только для облачных и гибридных организаций. |
|   | Ко всем пользователям в организации |
| **Владение устройством** | План |
| **Операционные системы** | Все устройства Windows 10 |
| **Подготовка** | Самообслуживания: Windows OOBE или параметры |
|   | Массовая регистрация |
|   | Windows Autopilot |
| **Варианты входа устройства** | Учетные записи в организации с помощью: |
|   | Пароль |
|   | Windows Hello для бизнеса |
|   | Ключи безопасности FIDO2.0 (Предварительная версия) |
| **Управление устройствами** | Управление мобильными устройствами (пример: Microsoft Intune) |
|   | Совместное управление с помощью Microsoft Intune и System Center Configuration Manager |
| **Ключевые возможности** | Единый вход для облачных и локальных ресурсов |
|   | Условный доступ посредством регистрации MDM и оценка соответствия MDM |
|   | Самостоятельного сброса пароля и Windows Hello ПИН-код сброса на экране блокировки |
|   | Enterprise State Roaming для устройств |

Устройства, присоединенные к AD Azure входа в систему с помощью организации учетной записи Azure AD. Доступ к ресурсам в организации можно разделить limited на основании этой учетной записи Azure AD и [политики условного доступа](../conditional-access/overview.md) применяется к удостоверению устройства.

Администраторы могут защитить и дальнейшего управления Azure AD устройств, присоединенных к с помощью средств управления мобильными устройствами (MDM), например Microsoft Intune или в сценариях совместного управления, с помощью System Center Configuration Manager. Эти средства позволяют принудительное применение конфигураций, необходимые организации, как требование хранилища должны быть зашифрованы, сложность пароля, установки программного обеспечения и обновлений программного обеспечения. Администраторы можно сделать доступными для устройств на присоединенных к Azure AD, с помощью приложений организации [System Center Configuration Manager и Microsoft Store для бизнеса](https://docs.microsoft.com/sccm/apps/deploy-use/manage-apps-from-the-windows-store-for-business).

Присоединение к Azure AD можно выполнить при помощи самостоятельного параметры как Out of Box Приветствия, массовой регистрации или [Windows Autopilot](https://docs.microsoft.com/intune/enrollment-autopilot).

Устройства, присоединенные к AD Azure по-прежнему могут поддерживать единого входа для доступа к локальным ресурсам, когда они находятся в сети организации. Устройства, которые присоединены к Azure AD могут проходить проверку подлинности на локальных серверах, как файл, печать и других приложений.

## <a name="scenarios"></a>Сценарии

Несмотря на то что присоединение к Azure AD в первую очередь предназначено для организаций у которых нет локальной инфраструктуры Windows Server Active Directory, его, безусловно, можно использовать в следующих сценариях:

- Вы хотите перейти на облачную инфраструктуру, используя Azure AD и приложение управления основными данными, такое как Intune.
- вы не можете использовать присоединение к домену в локальной среде, например если требуется управлять мобильными устройствами, такими как планшеты и телефоны;
- вашим пользователям в основном нужен доступ к Office 365 или другим приложения SaaS, интегрированным с Azure AD;
- вы хотите управлять группой пользователей в Azure AD, а не в Active Directory. Этот сценарий можно применить, например, для сезонных работников, подрядчиков или учащихся.
- вы хотите, чтобы работники удаленных филиалов с ограниченной локальной инфраструктурой могли подключаться к основной сети;

Можно настроить устройства, присоединенные к Azure AD, для устройств Windows 10.

Устройства, присоединенные к Azure AD, упрощают выполнение следующих задач:

- развертывание Windows на корпоративных устройствах;
- доступ к корпоративным приложениям и ресурсам с любого устройства Windows.
- Возможность управлять корпоративными устройствами в облаке
- Пользователи должны входить устройства с помощью их Azure AD или синхронизированных из Active Directory, рабочей или учебной учетных записей.

![Устройства, присоединенные к Azure AD](./media/concept-azure-ad-join/azure-ad-joined-device.png)

Присоединение к Azure AD могут развертываться с помощью любого из следующих методов:

- [Windows Autopilot](https://docs.microsoft.com/windows/deployment/windows-autopilot/windows-10-autopilot);
- [массовое развертывание](https://docs.microsoft.com/intune/windows-bulk-enroll);
- [самообслуживание пользователей](azuread-joined-devices-frx.md).

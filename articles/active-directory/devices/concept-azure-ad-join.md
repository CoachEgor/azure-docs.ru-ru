# <a name="azure-ad-joined-devices"></a>Устройства, присоединенные к Azure AD

Присоединение к Azure AD предназначено для организаций, которые должны быть только облачными или облачными. Любая организация может развертывать устройства, присоединенные к Azure AD, независимо от размера или отрасли. Присоединение к Azure AD работает даже в гибридной среде, обеспечивая доступ к облачным и локальным приложениям и ресурсам.

|   | Присоединение к Azure AD |
| --- | --- |
| **Определение** | Присоединено только к Azure AD, для входа на устройство требуется учетная запись Организации |
| **Основная аудитория** | Подходит для облачных и гибридных организаций. |
|   | Применимо ко всем пользователям в Организации |
| **Владение устройствами** | План |
| **Операционные системы** | Все устройства с Windows 10 |
| **Провизионирование** | Self-Service: Windows OOBE или параметры |
|   | Массовая регистрация |
|   | Windows Autopilot |
| **Параметры входа устройства** | Учетные записи организации с помощью: |
|   | Пароль |
|   | Windows Hello для бизнеса |
|   | Ключи безопасности FIDO 2.0 (Предварительная версия) |
| **Управление устройствами** | Управление мобильными устройствами (пример: Microsoft Intune) |
|   | Совместное управление с помощью Microsoft Intune и Configuration Manager конечных точек Майкрософт |
| **Ключевые возможности** | Единый вход для облачных и локальных ресурсов |
|   | Условный доступ с использованием регистрации MDM и оценки соответствия MDM |
|   | Самостоятельный сброс пароля и сброс ПИН-кода Windows Hello на экране блокировки |
|   | Enterprise State Roaming на разных устройствах |

Устройства, присоединенные к Azure AD, вошли в учетную запись Azure AD Организации. Доступ к ресурсам в Организации может быть более ограничен в зависимости от того, что учетная запись Azure AD и [политики условного доступа](../conditional-access/overview.md) применяются к удостоверению устройства.

Администраторы могут защищать устройства, присоединенные к Azure AD, и управлять ими с помощью таких средств управления мобильными устройствами (MDM), как Microsoft Intune или в сценариях совместного управления с помощью Configuration Manager конечных точек Майкрософт. Эти инструменты предоставляют средства для применения необходимых для Организации конфигураций, таких как требование шифрования хранилища, сложность пароля, установка программного обеспечения и обновление программного обеспечения. Администраторы могут сделать приложения Организации доступными для устройств, присоединенных к Azure AD, с помощью Configuration Manager для [управления приложениями из Microsoft Store для бизнеса и образовательных учреждений](/configmgr/apps/deploy-use/manage-apps-from-the-windows-store-for-business).

Присоединение к Azure AD можно выполнить с помощью параметров самообслуживания, таких как "готовая среда" (OOBE), "многофакторная регистрация" или " [автопилот Windows](/intune/enrollment-autopilot)".

Устройства, присоединенные к Azure AD, могут по-прежнему поддерживать единый вход в локальные ресурсы, если они находятся в сети Организации. Устройства, присоединенные к Azure AD, по-прежнему могут проходить проверку подлинности на локальных серверах, таких как файлы, печать и другие приложения.

## <a name="scenarios"></a>Сценарии

Несмотря на то что присоединение к Azure AD в первую очередь предназначено для организаций у которых нет локальной инфраструктуры Windows Server Active Directory, его, безусловно, можно использовать в следующих сценариях:

- Вы хотите перейти на облачную инфраструктуру, используя Azure AD и приложение управления основными данными, такое как Intune.
- вы не можете использовать присоединение к домену в локальной среде, например если требуется управлять мобильными устройствами, такими как планшеты и телефоны;
- вашим пользователям в основном нужен доступ к Office 365 или другим приложения SaaS, интегрированным с Azure AD;
- вы хотите управлять группой пользователей в Azure AD, а не в Active Directory. Этот сценарий может применяться, например, к сезонным работникам, подрядчикам или учащимся.
- вы хотите, чтобы работники удаленных филиалов с ограниченной локальной инфраструктурой могли подключаться к основной сети;

Можно настроить устройства, присоединенные к Azure AD, для устройств Windows 10.

Устройства, присоединенные к Azure AD, упрощают выполнение следующих задач:

- развертывание Windows на корпоративных устройствах;
- доступ к корпоративным приложениям и ресурсам с любого устройства Windows.
- Возможность управлять корпоративными устройствами в облаке
- Пользователи могут входить на свои устройства с помощью Azure AD или синхронизированных Active Directory рабочих или учебных учетных записей.

![Устройства, присоединенные к Azure AD](./media/concept-azure-ad-join/azure-ad-joined-device.png)

Присоединение к Azure AD могут развертываться с помощью любого из следующих методов:

- [Windows Autopilot](/windows/deployment/windows-autopilot/windows-10-autopilot)
- [массовое развертывание](/intune/windows-bulk-enroll);
- [самообслуживание пользователей](azuread-joined-devices-frx.md).

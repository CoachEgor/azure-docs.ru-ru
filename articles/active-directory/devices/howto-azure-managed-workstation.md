## <a name="azure-active-directory-configuration"></a>Настройка Azure Active Directory

Azure Active Directory (Azure AD) управляет пользователей, групп и устройств для рабочие станции администратора. Необходимо включить службы удостоверений и компоненты с [учетной записи администратора](https://docs.microsoft.com/azure/active-directory/users-groups-roles/directory-assign-admin-roles).

При создании учетной записи администратора защищенной рабочей станции, либо предоставить учетной записи для вашей текущей рабочей станции. Убедитесь, что используется известных безопасном устройство для выполнения этой начальной конфигурации и все глобальную конфигурацию. Чтобы снизить вероятность атак для первого опыта, рассмотрим следующие [инструкции по предотвращению заражения вредоносными программами](https://docs.microsoft.com/windows/security/threat-protection/intelligence/prevent-malware-infection).

Вы также должны требовать многофакторную проверку подлинности, по крайней мере для администраторов. См. в разделе [развертывание MFA в облаке](https://docs.microsoft.com/azure/active-directory/authentication/howto-mfa-getstarted) для руководство по реализации.

### <a name="azure-ad-users-and-groups"></a>Azure AD пользователей и групп

1. На портале Azure перейдите к **Azure Active Directory** > **пользователей** > **нового пользователя**.
1. Создание администратора устройства, выполнив действия, описанные в [руководства пользователя по созданию](https://docs.microsoft.com/Intune/quickstart-create-user).
1. Введите:
   * **Имя** — защита рабочей станции администратора
   * **Имя пользователя** - `secure-ws-admin@identityitpro.com`
   * **Роль каталога** - **администратор с ограниченными правами** и выберите **администратор Intune** роли.
1. Нажмите кнопку **Создать**.

Создайте две группы: пользователи рабочих станций и устройств рабочей станции.

На портале Azure перейдите к **Azure Active Directory** > **группы** > **новую группу**.

1. Для группы пользователей рабочей станции, может потребоваться настроить [группового лицензирования](https://docs.microsoft.com/azure/active-directory/users-groups-roles/licensing-groups-assign) для автоматизации подготовки лицензий для пользователей.
1. Для группы пользователей рабочей станции введите следующую команду:
   * **Тип группы** -безопасности
   * **Имя группы** -Secure пользователи рабочей станции
   * **Тип членства** - назначенного
1. Добавьте пользователя-администратора безопасной рабочей станции: `secure-ws-admin@identityitpro.com`
1. Можно добавить других пользователей, которые будут управлять безопасных рабочих станций.
1. Нажмите кнопку **Создать**.

1. Для группы устройств рабочей станции введите следующую команду:
   * **Тип группы** -безопасности
   * **Имя группы** -Secure рабочих станций
   * **Тип членства** - назначенного
1. Нажмите кнопку **Создать**.

### <a name="azure-ad-device-configuration"></a>Настройка устройства Azure AD

#### <a name="specify-who-can-join-devices-to-azure-ad"></a>Укажите, могут присоединять устройства к Azure AD

Настройте параметр в Active Directory, чтобы разрешить вашей группы безопасности администрирования присоединить устройства к домену устройства. Чтобы настроить этот параметр на портале Azure:

1. Перейдите к **Azure Active Directory** > **устройств** > **параметры устройства**.
1. Выберите **выбранные** под **пользователи могут присоединять устройства к Azure AD**, а затем выберите группу «Пользователи рабочей станции Secure».

#### <a name="removal-of-local-admin-rights"></a>Удаление прав локального администратора

Этот метод требуется виртуальный IP-адрес, DevOps и рабочих станциях уровня защиты нет прав администратора на своих компьютерах. Чтобы настроить этот параметр на портале Azure:

1. Перейдите к **Azure Active Directory** > **устройств** > **параметры устройства**.
1. Выберите **None** под **присоединение дополнительных локальных администраторов в Azure AD устройств, присоединенных к**.

#### <a name="require-multi-factor-authentication-to-join-devices"></a>Требовать многофакторную проверку подлинности, чтобы присоединить устройства

Для дальнейшего усиления процесс присоединения устройства к Azure AD:

1. Перейдите к **Azure Active Directory** > **устройств** > **параметры устройства**.
1. Выберите **Да** под **требует многофакторной проверки подлинности для присоединения устройств**.
1. Щелкните **Сохранить**.

#### <a name="configure-mdm"></a>Настройка управления мобильными Устройствами

На портале Azure выполните следующие действия.

1. Перейдите к **Azure Active Directory** > **мобильность (MDM и MAM)**  > **Microsoft Intune**.
1. Изменение **область пользователя MDM** присвоить **все**.
1. Щелкните **Сохранить**.

Эти действия позволяют управлять любым устройством с помощью Intune. Дополнительные сведения см. в разделе [Intune краткое руководство: Настройка автоматической регистрации для устройств Windows 10](https://docs.microsoft.com/Intune/quickstart-setup-auto-enrollment). Создание политики конфигурации и соответствия требованиям Intune на следующем шаге.

#### <a name="azure-ad-conditional-access"></a>Условный доступ Azure AD

Azure AD условный доступ может помочь ограничить привилегированных административные задачи для соответствующих требованиям устройств. Предопределенные членами **защиты рабочей станции пользователей** группы необходимы для выполнения многофакторной проверки подлинности при входе в систему облачных приложений. Рекомендуется исключить учетные записи аварийного доступа из политики. Дополнительные сведения см. в разделе [управление учетные записи аварийного доступа в Azure AD](https://docs.microsoft.com/azure/active-directory/users-groups-roles/directory-emergency-access).

Чтобы настроить условный доступ на портале Azure:

1. Перейдите к **Azure Active Directory** > **условного доступа** > **новая политика**.
1. Введите:
   * **Имя** -политика обязательности устройство обеспечения безопасности
   * Назначения
     * **Пользователи и группы**
       * Включить - **пользователей и групп** — выберите **защиты рабочей станции пользователей** ранее созданной группе.
       * Исключить - **пользователей и групп** -выберите учетные записи аварийного доступа вашей организации.
     * **Облачные приложения** -включают **все облачные приложения**.
    * Элементы управления доступом
      * **Предоставление** — выберите **предоставить доступ** "переключатель".
        * **Требовать многофакторную проверку подлинности**.
        * **Требовать, чтобы было отмечено как соответствующее устройство**.
        * Для нескольких элементов управления - **требовать все выбранные элементы управления**.
    * Включить политику - **на**.

Вы можете создавать политики, которые блокируют странах, где бы не доступ к ресурсам компании. Дополнительные сведения о политиках условного доступа на основе расположения IP-адрес, см. в разделе [условия расположения в Azure Active Directory условного доступа](https://docs.microsoft.com/azure/active-directory/conditional-access/location-condition).

## <a name="intune-configuration"></a>Настройка Intune

### <a name="configure-enrollment-status"></a>Настройка состояния регистрации

Очень важно убедитесь, что ваш безопасной рабочей станции доверенное Очистить устройство. При покупке новых устройств, можно предполагается, что они заводская установка [Windows 10 Pro, в режиме S](https://docs.microsoft.com/Windows/deployment/Windows-10-pro-in-s-mode), который ограничивает возможность раскрытия уязвимостей во время управления цепочкой поставок. Когда устройства получают от поставщика, можно использовать Autopilot переключиться из режима S. Далее представлены сведения о применении процесса преобразования.

Чтобы убедиться, что перед использованием полностью настройки устройств, Intune предоставляет средства для **заблокировать использование устройств, пока не будут установлены все приложения и профили**.

На **портале Azure**
1. Перейдите к **Microsoft Intune** > **регистрации устройств** > **регистрации Windows** > **состояние регистрации Страница** > **по умолчанию** > **параметры**.
1. Задайте **отображаться ход выполнения установки профиля приложения** для **Да**.
1. Задайте **заблокировать использование устройств, пока не будут установлены все приложения и профили** для **Да**.

### <a name="create-an-autopilot-deployment-profile"></a>Создание профиля развертывания Autopilot

После создания группу устройств, необходимо создать профиль развертывания для настройки устройств Autopilot.

В Intune на портале Azure:

1. Выберите **регистрации устройств** > **регистрации Windows** > **профили развертывания** > **Создание профиля** .
1. Введите:
   * Имя — **Secure профиля развертывания рабочей станции**.
   * Описание - **развертывания безопасных рабочих станций**.
   * Задайте **преобразовать все целевые устройства Autopilot** для **Да**. Этот параметр гарантирует, что все устройства в списке, зарегистрируйтесь в службе развертывания Autopilot. Разрешить 48 часов для регистрации для обработки.
1. Щелкните **Далее**.
   * Для **режим развертывания**, выберите **самостоятельного развертывания (Предварительная версия)** . Устройства с этим профилем связаны с пользователем, который регистрирует устройство. Для регистрации устройства необходимы учетные данные пользователя.
   * **Присоединение к Azure AD в качестве** поле должно отображаться **присоединенные к Azure AD** и быть неактивен.
   * Выберите Langugage (регион), тип учетной записи пользователя **стандартный**. 
1. Щелкните **Далее**.
   * Выберите тег области, если содержат готовые один.
1. Щелкните **Далее**.
1. Выберите **назначения** > **назначить** > **выбранные группы**. В **выбрать группы для включения**, выберите **защиты рабочей станции пользователей**.
1. Щелкните **Далее**.
1. Нажмите кнопку **Создать**, чтобы создать профиль. Теперь профиль развертывания Autopilot доступна для назначения на устройствах.

### <a name="configure-windows-update"></a>Настройка центра обновления Windows

Обновление Windows 10 в актуальном состоянии — один из самых важных факторов, которые можно выполнить. Чтобы обеспечить поддержку Windows в безопасное состояние, развертывание круг обновления для управления скоростью, обновления применяются на рабочих станциях. 

В этом руководстве, то рекомендуется создать новое кольцо обновления и измените следующие параметры по умолчанию:

На портале Azure выполните следующие действия.

1. Перейдите к **Microsoft Intune** > **обновления программного обеспечения** > **круги обновлений Windows 10**.
1. Введите:
   * Имя — **обновления рабочей станции под управлением Azure**
   * Канал - обслуживания **программа предварительной оценки Windows — Fast**
   * Отсрочка для обновления (в днях) - **3**
   * Период отсрочки обновления компонентов (в днях) - **3**
   * Режим автоматического обновления - **автоматическая установка и перезагрузка без управления конечным пользователем**
   * Запретить пользователю Приостановка обновления Windows — **блока**
   * Требовать утверждения пользователя для перезапуска вне рабочих часов - **требуется**
   * Разрешить пользователю перезагрузить (занято) - **требуется**
   * Переход пользователей заинтересованности перезапуска после автоматическую перезагрузку (в днях) - **3**
   * Отложить заинтересованности перезапуска напоминания (в днях) - **3**
   * Установить крайний срок для ожидающие перезагрузки (в днях) - **3**

1. Нажмите кнопку **Создать**.
1. На **назначения** вкладку, добавьте **защиты рабочих станций** группы.

Дополнительные сведения о политики центра обновления Windows, см. в разделе [политика CSP - обновление](https://docs.microsoft.com/windows/client-management/mdm/policy-csp-update).

### <a name="windows-defender-atp-intune-integration"></a>Интеграция с ATP в Защитнике Windows Intune

ATP в Защитнике Windows и Microsoft Intune совместно для предотвращения нарушений безопасности. Они также можно ограничить влияние брешей. ATP возможности предоставления в реальном времени от угроз, а также включить расширенную аудит и ведение журнала конечными устройствами.

Чтобы настроить интеграцию ATP в Защитнике Windows и Intune, перейдите на портал Azure.

1. Перейдите к **Microsoft Intune** > **соответствие устройства политике** > **ATP в Защитнике Windows**.
1. На шаге 1 в разделе **Настройка ATP в Защитнике Windows**выберите **подключения ATP в Защитнике Windows в Microsoft Intune в центре безопасности Защитника Windows**.
1. В центре безопасности Защитника Windows:
   1. Выберите **параметры** > **дополнительные функции**.
   1. Для **подключения Microsoft Intune**, выберите **на**.
   1. Выберите **сохранить параметры**.
1. После подключения, вернитесь в Intune и выберите **обновить** вверху.
1. Задайте **подключения Windows версии устройств 10.0.15063 и более поздние версии ATP в Защитнике Windows** для **на**.
1. Щелкните **Сохранить**.

Дополнительные сведения см. в разделе [Windows Defender Advanced Threat Protection](https://docs.microsoft.com/Windows/security/threat-protection/windows-defender-atp/windows-defender-advanced-threat-protection).

* Ниже приведен, где можно найти устройств Intune политики соответствия, созданные с помощью сценариев. **Портал Azure** > **Microsoft Intune** > **соответствие устройства политике** > **политики**.

Чтобы просмотреть изменения, внесенные с помощью сценариев, вы можете экспортировать профили. Таким образом можно определить дополнительное Усиление защиты, которые могут потребоваться, как описано в документации по SECCON.

Выполнить сценарий экспорта данных Intune `DeviceConfiguration_Export.ps1` из [GiuHub конфигурация устройства репозитория](https://github.com/microsoftgraph/powershell-intune-samples/tree/master/DeviceConfiguration) для экспорта всех текущего профилей Intune.

## <a name="additional-configurations-and-hardening-to-consider"></a>Дополнительные конфигурации и усиление безопасности, которые следует учитывать

Следуя инструкциям ниже, вы развернули безопасной рабочей станции. Тем не менее также следует дополнительные элементы управления. Пример:

* ограничить доступ к альтернативный браузеры
* Разрешить исходящий HTTP
* Выберите веб-сайты
* Установка разрешений для запуска пользовательских сценариев PowerShell

### <a name="set-rules-in-the-firewall-configuration-service-provider-csp"></a>Задать правила в брандмауэр поставщике служб конфигурации (CSP)

Можно внести дополнительные изменения в управлении правила входящего и исходящего трафика, при необходимости для разрешенных и заблокированных конечных точек. По мере усиления защиты безопасной рабочей станции, может ослабить ограничения, который запрещает весь входящий и исходящий трафик. Можно добавить разрешенные исходящие узлы для включения распространенных и надежных веб-сайтов. Дополнительные сведения см. в разделе [брандмауэра службы настройки](https://docs.microsoft.com/Windows/client-management/mdm/firewall-csp).

Ниже приведены рекомендации по умолчанию ограничены.

* Запретить весь входящий трафик
* Запретить весь исходящий трафик

Проекта Spamhaus [списка блок домена (Двойной)](https://www.spamhaus.org/dbl/): хороший ресурс для использования в качестве отправной точки для блокирования.

### <a name="manage-local-applications"></a>Управление локальными приложениями

Безопасной рабочей станции переходит в состояние по-настоящему усиленной защитой в том случае, когда удаляются локальные приложения, включая производительность приложения. Здесь добавьте Chrome браузером по умолчанию и ограничить возможности пользователя изменять браузер и его подключаемых модулей с помощью Intune.

#### <a name="deploy-applications-using-intune"></a>Развертывание приложений с помощью Intune

В некоторых ситуациях приложений, таких как браузер Google Chrome, нужно на защищенной рабочей станции. Ниже приведены инструкции по установке Chrome для устройств в группе безопасности **защиты рабочих станций**.

1. Загрузите автономный установщик [пакет Chrome для Windows 64‑bit](https://cloud.google.com/chrome-enterprise/browser/download/).
1. Извлеките файлы и запишите расположение `GoogleChromeStandaloneEnterprise64.msi` файл.
1. В **портала Azure** перейдите к **Microsoft Intune** > **клиентских приложений** > **приложений**  >  **Добавить**.
1. В разделе **тип приложения**, выберите **бизнес-** .
1. В разделе **файл пакета приложения**выберите `GoogleChromeStandaloneEnterprise64.msi` файла из нового расположения и выберите **ОК**.
1. В разделе **сведения о приложении**, укажите в описании и издателя. Нажмите кнопку **ОК**.
1. Выберите **Добавить**.
1. На **назначения** выберите **доступно для зарегистрированных устройств** под **тип назначения**.
1. В разделе **включенные группы**, добавьте **защиты рабочих станций** группы.
1. Нажмите кнопку **ОК**, а затем нажмите кнопку **Сохранить**.

Дополнительные сведения о настройке параметров Chrome см. в разделе [управления браузера Chrome с помощью Microsoft Intune](https://support.google.com/chrome/a/answer/9102677).

#### <a name="configuring-the-company-portal-for-custom-apps"></a>Настройка корпоративного портала для пользовательских приложений

В безопасном режиме Установка приложения будет ограничен корпоративного портала Intune. Тем не менее при установке портала требуется доступ к Microsoft Store. В решении защищенной корпоративного портала можно сделать доступными на всех устройствах с помощью автономный режим.

Под управлением Intune копию [корпоративного портала](https://docs.microsoft.com/Intune/store-apps-company-portal-app) предоставляет доступ по запросу к дополнительные средства, которые можно отправить пользователям защищенных рабочих станций.

Может потребоваться установить Windows 32-разрядных приложений или другие приложения, в которых развертывания требуют специальных подготовительные действия. В таких случаях [Microsoft win32 содержимого prep tool](https://github.com/Microsoft/Microsoft-Win32-Content-Prep-Tool) может предоставить готовых к использованию `.intunewin` файла форматирования для установки.

### <a name="use-powershell-to-create-custom-settings"></a>Создайте настраиваемые параметры с помощью PowerShell

Можно также использовать PowerShell для расширения возможностей управления узла. Этот пример сценария настраивает фон по умолчанию на узле. Это возможность, которая также доступна через портал Azure и профили. В примере сценария служит только для иллюстрации функциональные возможности PowerShell.

Может потребоваться настроить некоторые пользовательские элементы управления и параметры на рабочих станциях безопасной. В этом примере изменяется фон рабочей станции с помощью способности Powershell легко идентифицировать устройство как готовые к использованию, безопасной рабочей станции.

[SetDesktopBackground.ps1](https://gallery.technet.microsoft.com/scriptcenter/Set-Desktop-Image-using-5430c9fb/) сценарий из центра сценариев Майкрософт позволяет Windows для загрузки это [бесплатный, универсальный фоновое изображение](https://i.imgur.com/OAJ28zO.png) на запуск.

1. Скачайте скрипт на локальное устройство.
1. Обновите customerXXXX и загрузки расположение фонового изображения. В нашем примере мы заменим customerXXXX на фоне элементов.  
1. Перейдите к **портала Azure** > **Microsoft Intune** > **конфигурации устройства** > **PowerShell сценарии** > **добавить**.
1. Укажите **имя** для скрипта и указать **расположение скрипта**.
1. Нажмите кнопку **Настроить**.
   1. Задайте **запустите этот скрипт с использованием данных вошедшего** для **Да**.
   1. Нажмите кнопку **ОК**.
1. Нажмите кнопку **Создать**.
1. Выберите **назначения** > **выберите группы**.
   1. Добавьте группу безопасности **защиты рабочих станций**.
   1. Щелкните **Сохранить**.

## <a name="enroll-and-validate-your-first-device"></a>Зарегистрировать и проверить первого устройства

1. Чтобы зарегистрировать устройство, вам потребуется следующее:
   * **Серийный номер** - найден в корпусе устройства.
   * **Идентификатор продукта Windows** - найден в разделе **системы** > **о** из меню "Параметры Windows".
   * Можно запустить [Get-WindowsAutoPilotInfo](https://aka.ms/Autopilotshell) для получения хэша CSV-файл со всеми необходимые сведения для регистрации устройств.
   
     Запустите `Get-WindowsAutoPilotInfo – outputfile device1.csv` для вывода данных в CSV-файл, который можно импортировать в Intune.

     > [!NOTE]
     > Сценарий требует повышенных прав. Он выполняется в качестве удаленных подписи. `Set-ExecutionPolicy -ExecutionPolicy RemoteSigned` Команда позволяет правильно выполнение сценария.

   * Эти данные можно собирать при входе в Windows 10 версии 1809 или более поздней версии устройства. К поставщику оборудования, также способны предоставить эти данные.
1. В **портала Azure**, перейдите в меню **Microsoft Intune** > **регистрации устройств** > **регистрации Windows**  >  **Устройства — управление Windows Autopilot**.
1. Выберите **импорта** и выберите CSV-файл.
1. Добавьте устройство, чтобы **защиты рабочих станций** группы безопасности.
1. На устройстве Windows 10, вы хотите настроить, перейдите в каталог **параметры Windows** > **обновление и безопасность** > **восстановления**.
   1. Выберите **приступить к работе** под **сбросить этот компьютер**.
   1. Следуйте инструкциям на экране для сброса и изменить конфигурацию устройства с помощью профиля и соответствие требованиям политики, настроенные.

После настройки устройства, выполните проверку и проверьте конфигурацию. Проверьте правильность настройки первого устройства для продолжения развертывания.

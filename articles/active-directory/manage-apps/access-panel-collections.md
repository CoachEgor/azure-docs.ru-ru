# <a name="create-collections-on-the-my-apps-portal"></a>Создание коллекций на портале My Apps

Пользователи могут использовать портал My Apps для просмотра и запуска облачных приложений, к которых они имеют доступ. По умолчанию все приложения, к которым пользователь может получить доступ, перечислены вместе на одной странице. Чтобы лучше организовать эту страницу для пользователей, если у вас есть лицензия Azure AD Premium P1 или P2, вы можете настроить коллекции. С помощью коллекции можно сгруппировать связанные приложения (например, по роли задания, задаче или проекту) и отобразить их на отдельной вкладке. Коллекция по существу применяет фильтр к приложениям, к которым пользователь уже может получить доступ, поэтому пользователь видит только те приложения в коллекции, которые были назначены им.

> [!NOTE]
> В этой статье рассказывается о том, как админ может создавать коллекции и создавать их. Для получения информации для конечных пользователей о том, как использовать портал и коллекции My Apps, [см.](https://docs.microsoft.com/azure/active-directory/user-help/my-applications-portal-workspaces)

## <a name="enable-the-latest-my-apps-features"></a>Включить последние функции My Apps

1. Откройте [**портал Azure**](https://portal.azure.com/) и вопием в качестве администратора пользователя или глобального администратора.

2. Перейдите к > **настройкам пользователя**Active **Directory Azure.**

3. В **превью функции пользователя**выберите **настройки предварительного просмотра функций функции пользователя.**

4. В **соответствии с функциями предварительного просмотра для моих приложений,** выбрать один из следующих вариантов:
   * **Выбранный** - Включает в себя функции для определенной группы. Используйте **параметр «Выберите группу»,** чтобы выбрать группу, для которой вы хотите включить объекты.  
   * **Все** - Включает в себя функции для всех пользователей.

> [!NOTE]
> Чтобы открыть портал My Apps, пользователи могут использовать ссылку `https://myapps.microsoft.com` или `https://myapps.microsoft.com/contoso.com`настраиваемую ссылку для вашей организации, например. После включения нового опыта My Apps **обновленный опыт My Applications будет отображаться** в верхней части страницы My Apps, и пользователи могут выбрать **его** для просмотра нового опыта. Чтобы отказаться от использования нового опыта, пользователи могут выбрать **«Да»** из **баннера «Оставьте новый опыт»** в верхней части страницы.

## <a name="create-a-collection"></a>Создание коллекции

Для создания коллекции необходимо иметь лицензию Azure AD Premium P1 или P2.

1. Откройте [**портал Azure**](https://portal.azure.com/) и вопийте в качестве админа с лицензией Azure AD Premium P1 или P2.

2. Перейдите к корпоративным**приложениям** **Azure Active Directory.** > 

3. Под **управлением**, выберите **Коллекции**.

4. Выберите **новую коллекцию**. На странице **Новой коллекции** введите **название** для коллекции (рекомендуем не использовать "сбор" в названии. Затем введите **Описание**.

   ![Новая страница коллекции](media/acces-panel-collections/new-collection.png)

5. Выберите вкладку **Приложения.** Выберите **приложение,** а затем на странице **приложения Добавить,** выберите все приложения, которые вы хотите добавить в коллекцию, или используйте поле **поиска** для поиска приложений.

   ![Добавить приложение в коллекцию](media/acces-panel-collections/add-applications.png)

6. Когда вы закончите добавлять приложения, выберите **Добавить.** Отображается список выбранных приложений. Вы можете использовать вверх стрелки, чтобы изменить порядок приложений в списке. Чтобы переместить приложение вниз или удалить его из коллекции, выберите меню **More** **(...**).

7. Выберите вкладку **«Владельцы.** **Выберите» и добавите пользователей и группы,** а затем на странице **Добавления,** выберите пользователей или группы, которые вы хотите назначить право собственности. Когда вы закончите выбор пользователей и групп, выберите **выберите**.

9. Выберите вкладку **пользователей и групп.** Выберите **и добавить пользователей и групп,** а затем в **добавить пользователей и групп** страницы, выберите пользователей или групп, которые вы хотите назначить коллекцию. Или используйте поле **поиска,** чтобы найти пользователей или группы. Когда вы закончите выбор пользователей и групп, выберите **выберите**.

   ![Добавление пользователей и групп](media/acces-panel-collections/add-users-and-groups.png)

11. Выберите **Обзор и Создайте**. Появляются свойства новой коллекции.


## <a name="view-audit-logs"></a>Просмотр журналов аудита

В журналах аудита записывается операции коллекций Моих Приложений, включая действия конечных пользователей создания коллекции. Следующие события генерируются из моих приложений:

* Создание коллекции
* Редактировать коллекцию
* Удаление коллекции
* Запуск приложения (конечный пользователь)
* Добавление приложения самообслуживания (конечный пользователь)
* Удаление приложения самообслуживания (конечный пользователь)

Вы можете получить доступ к журналам аудита на [портале Azure,](https://portal.azure.com) выбрав > **журналы аудита** приложений **azure Active Directory** > **Enterprise Applications**В разделе Activity. Для **обслуживания**, выберите **Мои приложения**.

## <a name="get-support-for-my-account-pages"></a>Получите поддержку на страницах "Моей учетной записи"

На странице My Apps пользователь может выбрать Мой аккаунт View my **account,** > **View my account** чтобы открыть настройки своей учетной записи. На странице Azure AD **My Account** пользователи могут управлять информацией о безопасности, устройствами, паролями и многое другое. Они также могут получить доступ к настройкам учетной записи Office.

В случае необходимости отправки запроса на непоправимый запрос на страницу учетной записи Azure AD или на странице учетной записи Office выполните следующие действия, чтобы ваш запрос был надлежащим образом направлен: 

* Для проблем со страницей **Azure AD "Моя учетная запись"** откройте запрос на поддержку на портале Azure. Перейдите на **портал** > Azure**Active Directory** > **New request.**

* Для проблем со страницей **Office "Моя учетная запись"** откройте запрос на поддержку в центре microsoft 365. Перейдите на > **службу в**центр **поддержки Microsoft 365.**

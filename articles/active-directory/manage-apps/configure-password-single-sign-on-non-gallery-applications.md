# <a name="configure-password-single-sign-on"></a>Настройка пароля единого входа

При [добавлении приложения галереи](add-gallery-app.md) или [веб-приложения, не являющегося галереей,](add-non-gallery-app.md) в приложения Azure AD Enterprise Applications, одним из доступных для вас опций— является [одиночный знак на основе паролей.](what-is-single-sign-on.md#password-based-sso) Эта опция доступна для любого веб-страницы с HTML-ввоза. Единый вход с помощью пароля (хранение пароля в хранилище) позволяет управлять доступом и паролями для веб-приложений, которые не поддерживают федерацию удостоверений. Это также полезно для сценариев, где нескольким пользователям необходимо совместно поделиться одной учетной записью, например, с учетными записями приложений в социальных сетях вашей организации. 

SSO на основе паролей — это отличный способ быстро начать интеграцию приложений в Azure AD и:

-   Использование **единого входа для пользователей** с помощью безопасного хранения и воспроизведения имен пользователей и паролей для приложений, интегрированных с Azure AD.

-   **Поддержка приложений, использующих несколько полей ввода учетных данных** помимо обычных имени пользователя и пароля.

-   **Настройка меток** для полей ввода имени пользователя и пароля, которые будут отображаться при вводе учетных данных на [панели доступа к приложениям](https://docs.microsoft.com/azure/active-directory/active-directory-saas-access-panel-introduction).

-   **Пользователи** могут использовать имена пользователей и пароли для любых существующих учетных записей при вводе вручную в [панели доступа к приложениям](https://docs.microsoft.com/azure/active-directory/active-directory-saas-access-panel-introduction).

-   **Члены бизнес-группы** могут назначать пользователям имена для входа и пароли, используя функцию [самостоятельной настройки доступа к приложениям](https://docs.microsoft.com/azure/active-directory/active-directory-self-service-application-access).

-   Разрешить **администратору** указать имя пользователя и пароль, которые будут использоваться отдельными лицами или группами при входе в приложение, используя функцию «Новые учетные данные» 

## <a name="before-you-begin"></a>Перед началом

Если приложение еще не добавлено в клиент Azure AD, ознакомьтесь со статьей о [добавлении приложения из коллекции](add-gallery-app.md) или [добавлении приложения не из коллекции](add-non-gallery-app.md).

## <a name="open-the-app-and-select-password-single-sign-on"></a>Откройте приложение и выберите пароль одиночный входе

1. Войдите на [портал Azure](https://portal.azure.com) в качестве администратора облачных приложений или администратора приложения для клиента Azure AD.

2. Перейдите к приложениям **Azure Active Directory** > **Enterprise.** Вы увидите случайную выборку приложений, размещенных в арендаторе Azure AD. 

3. В меню **Тип приложения** выберите **Все приложения** и щелкните **Применить**.

4. В поле поиска введите имя приложения, а затем выберите нужное приложение на панели результатов.

5. В разделе **Управление** щелкните **Единый вход**. 

6. Выберите **пароль на основе**.

7. Введите URL-адрес веб-страницы входа приложения. Эта строка должна быть страницей, которая включает в себя поле ввода имени пользователя.

   ![Единый вход на основе пароля](./media/configure-single-sign-on-non-gallery-applications/password-based-sso.png)

8. Нажмите кнопку **Сохранить**. Azure AD пытается разобрать страницу ввода в вода химаху пользователя и ввод паролей. Если попытка увенчается успехом, вы закончите. 
 
> [!NOTE]
> Следующим шагом является [присвоение приложения пользователям или группам.](methods-for-assigning-users-and-groups.md) После того, как вы назначили пользователей и группы, вы можете предоставить учетные данные, которые будут использоваться от имени пользователя, когда они воспридают в приложение. Выберите **пользователей и группы,** выберите флажок для строки пользователя или группы, а затем нажмите **«Новые учетные данные».** Затем введите имя пользователя и пароль, которые будут использоваться от имени пользователя или группы. В противном случае пользователям будет предложено ввести учетные данные сами после запуска.
 

## <a name="manual-configuration"></a>Настройка вручную

Если попытка разбора Azure AD не удается, можно настроить регистрацию вручную.

1. Под ** \<названием приложения> конфигурации**выберите **название \<приложения> Настройки одного пароля** для отображения страницы **настройки знака.** 

2. Выберите **вручную обнаружить поля ввески.** Появляются дополнительные инструкции, описывающие ручное обнаружение полей ввески.

   ![Ручная конфигурация одного указателя на основе паролей](./media/configure-password-single-sign-on/password-configure-sign-on.png)
3. Выберите **поля регистрации захвата.** Страница состояния захвата открывается в новой вкладке, показывающая, что **в настоящее время происходит захват метаданных**сообщения.

4. Если необходимое поле для **расширения панели доступа** отображается в новой вкладке, выберите **«Установить сейчас»,** чтобы установить расширение браузера **My Apps Secure Sign-in.** (Расширение браузера требует Microsoft Edge, Chrome или Firefox.) Затем установите, запустите и включите расширение и обновите страницу состояния захвата.

   Затем расширение браузера открывает другую вкладку, отображая введенный URL.
5. Во вкладке с введенным URL пройдите процесс ввода. Заполните поля имени пользователя и пароля и попытайтесь войти в систему. (Вы не должны предоставить правильный пароль.)

   Запрос просит вас сохранить захваченные поля ввечек.
6. Нажмите кнопку **ОК**. Расширение браузера обновляет страницу состояния захвата с сообщением **Metadata был обновлен для приложения.** Вкладка браузера закрывается.

7. На странице Azure AD **Configure sign-on,** выберите **Ok, я смог войти в приложение успешно.**

8. Нажмите кнопку **ОК**.

После захвата страницы ввместь, вы можете назначить пользователей и групп, и вы можете настроить политики учетных данных так же, как обычные [приложения SSO паролей.](what-is-single-sign-on.md)

> [!NOTE]
> Вы можете загрузить логотип, который будет отображаться на плитке приложения. Для этого нажмите кнопку **Загрузить логотип** на вкладке **Настройка** для этого приложения.

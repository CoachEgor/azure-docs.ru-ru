# <a name="problems-configuring-password-single-sign-on-for-a-non-gallery-application"></a>Проблемы с настройкой одного входного входного знака пароля для приложения, не являющегося галереей

В этой статье описаны общие проблемы, которые могут возникнуть при настройке *одного входного знака* (SSO) для приложения, не являющийся галереей.

## <a name="capture-sign-in-fields-for-an-app"></a>Захват полей ввески для приложения

Захват поля ввески поддерживается только для страниц, включенных в систему HTML. Он не поддерживается для нестандартных страниц ввески, как те, которые используют Adobe Flash или другие технологии, не связанные с HTML.

Существует два способа захвата полей вхасов для пользовательских приложений:

- **Автоматический захват поля хорошо** работает с большинством страниц с поддержкой HTML, *если они используют хорошо известные DIV ID* для имен пользователя и полей паролей. HTML на странице Царапины, чтобы найти DIV ИДенс, которые соответствуют определенным критериям. Эти метаданные сохраняются, чтобы они могли быть воспроизведены в приложении позже.

- **Ручной захват поля ввода** используется, если поставщик приложения *не маркирует входные поля.* Ручной захват также используется, если поставщик *отображает несколько полей, которые не могут быть автоматически обнаружены.* Активный каталог Azure (Azure AD) может хранить данные для столько полей, сколько есть на странице ввне, если вы сообщите ему, где эти поля находятся на странице.

Как правило, если автоматический захват поля не работает, попробуйте ручной вариант.

### <a name="automatically-capture-sign-in-fields-for-an-app"></a>Автоматически захват полей вхащиков для приложения

Для настройки SSO на основе паролей с помощью автоматического захвата поля ввешании выполните следующие действия:

1. Откройте [портал Azure](https://portal.azure.com/). Войти в качестве глобального администратора или совместного администратора.

2. В навигационном стежке с левой стороны выберите **все службы,** чтобы открыть расширение Azure AD.

3. Введите **active-каталог Azure** в поле поиска фильтров, а затем выберите **Active Directory Azure.**

4. Выберите **корпоративные приложения** в навигационном панели Azure AD.

5. Выберите **все приложения** для просмотра списка ваших приложений.

   > [!NOTE]
   > Если вы не видите нужное приложение, используйте элемент управления **фильтром** в верхней части списка **всех приложений.** Установите опцию **Show** на "Все приложения".

6. Выберите приложение, которое вы хотите настроить для SSO.

7. После загрузки приложения выберите **единый входе** в навигационное стекло на левой стороне.

8. Выберите режим **регистрации на основе пароля.**

9. Введите **URL-адрес входа**в систему, который является URL-адресом страницы, на которой пользователи вводят свое имя пользователя и пароль для входа. *Убедитесь, что поля ввне будут видны на странице для URL-адреса, который вы предоставляете,*—

10. Нажмите кнопку **Сохранить**.

    Страница автоматически соскребается для имени пользователя и ввода пароля. Теперь вы можете использовать Azure AD для безопасной передачи паролей к этому приложению с помощью расширения браузера Access Panel.

### <a name="manually-capture-sign-in-fields-for-an-app"></a>Ручнозахват полей ввещиков для приложения

Чтобы вручную зафиксировать поля ввески, необходимо установить расширение браузера Панели доступа. Кроме того, ваш браузер не может работать в *inPrivate,* *incognito*или *в частном* режиме.

Для установки расширения смотрите раздел [расширения браузера панели доступа](#install-the-access-panel-browser-extension) в этой статье.

Для настройки SSO на основе паролей для приложения с помощью ручного захвата поля для вхинга выполните следующие действия:

1. Откройте [портал Azure](https://portal.azure.com/). Войти в качестве глобального администратора или совместного администратора.

2. В навигационном стежке с левой стороны выберите **все службы,** чтобы открыть расширение Azure AD.

3. Введите **active-каталог Azure** в поле поиска фильтров, а затем выберите **Active Directory Azure.**

4. Выберите **корпоративные приложения** в навигационном панели Azure AD.

5. Выберите **все приложения** для просмотра списка ваших приложений.

   > [!NOTE] 
   > Если вы не видите нужное приложение, используйте элемент управления **фильтром** в верхней части списка **всех приложений.** Установите опцию **Show** на "Все приложения".

6. Выберите приложение, которое вы хотите настроить для SSO.

7. После загрузки приложения выберите **единый входе** в навигационное стекло на левой стороне.

8. Выберите режим **регистрации на основе пароля.**

9. Введите **URL-адрес входа**в систему, на котором пользователи вводят свое имя пользователя и пароль для входа. *Убедитесь, что поля ввне будут видны на странице для URL-адреса, который вы предоставляете,*—

10. **Выберите * &lt;настройку&gt; приложения* Пароль Одиночный Подгон Настройки.**

11. Выберите **вручную обнаружить поля ввески.**

14. Выберите **Ok**.

15. Нажмите кнопку **Сохранить**.

16. Следуйте инструкциям по использованию панели доступа.

## <a name="troubleshoot-problems"></a>Устранение неполадок

### <a name="i-get-a-we-couldnt-find-any-sign-in-fields-at-that-url-error"></a>Я получаю "Мы не могли найти каких-либо введомая поля на этом URL" ошибка

Вы получаете это сообщение об ошибке, когда автоматическое обнаружение полей введанных не удается. Чтобы решить проблему, попробуйте ручное обнаружение полей. Смотрите [поля для вхданных вручную для раздела приложения](#manually-capture-sign-in-fields-for-an-app) этой статьи.

### <a name="i-get-an-unable-to-save-single-sign-on-configuration-error"></a>Я получаю ошибку "Не удается сохранить одну конфигурацию входного знака"

Редко обновление конфигурации SSO завершается неудачей. Чтобы решить эту проблему, попробуйте сохранить конфигурацию снова.

Если вы продолжаете получать ошибку, откройте дело поддержки. Включите информацию, описанную в [уведомлении портала View,](#view-portal-notification-details) и [отправьте сведения об уведомлении инженеру службы поддержки, чтобы получить](#send-notification-details-to-a-support-engineer-to-get-help) разделы справки в этой статье.

### <a name="i-cant-manually-detect-sign-in-fields-for-my-app"></a>Я не могу вручную обнаружить поля ввески для моего приложения

Вы можете наблюдать следующие действия, когда ручное обнаружение не работает:

- Процесс ручного захвата, как представляется, работает, но захваченные поля не являются правильными.

- Правильные поля не подсвечивается при запуске процесса захвата.

- Процесс захвата приведет вас к странице вступления приложения, как ожидалось, но ничего не происходит.

- Ручной захват, кажется, работает, но SSO не происходит, когда пользователи перемещаются в приложение из панели доступа.

Если вы испытываете какие-либо из этих проблем, сделать следующие вещи:

- Убедитесь, что у вас есть последняя версия расширения браузера Access Panel *установление и включено.* Смотрите раздел [расширения браузера панели доступа](#install-the-access-panel-browser-extension) в этой статье.

- Убедитесь, что ваш браузер не в *инкогнито,* *inPrivate*, или *частный* режим во время процесса захвата. Расширение панели доступа не поддерживается в этих режимах.

- Убедитесь, что пользователи не пытаются войти в приложение из панели доступа в то время как в *инкогнито,* *inPrivate*, или *частный режим.*

- Повторите попытку ручного захвата. Убедитесь, что красные маркеры находятся над правильными полями.

- Если процесс ручного захвата, кажется, перестает отвечать или страница ввне не отвечает, попробуйте процесс захвата вручную еще раз. Но на этот раз, после завершения процесса, нажмите клавишу F12, чтобы открыть консоль разработчика вашего браузера. Выберите **консоль** нуючку. Введите **window.location",*&lt;URL-адрес&gt;входа, указанный при настройке приложения*",** а затем нажмите Enter. Это заставляет перенаправить страницу, которая завершает процесс захвата и хранит захваченные поля.

### <a name="contact-support"></a>Обращение в службу поддержки

Если у вас все еще есть проблемы, откройте дело в службе поддержки Майкрософт. Опишите, что вы пытались. Включите сведения, описанные в [уведомлении портала View,](#view-portal-notification-details) и [отправьте сведения об уведомлении инженеру службы поддержки, чтобы получить](#send-notification-details-to-a-support-engineer-to-get-help) разделы справки в этой статье (если это применимо).

## <a name="install-the-access-panel-browser-extension"></a>Установка расширения браузера панели доступа

Выполните следующие действия.

1. Панель открытого [доступа](https://myapps.microsoft.com) в поддерживаемом браузере. Вопиюсь в Azure AD в качестве *пользователя.*

2. Выберите **приложение для пароля SSO** в панели доступа.

3. Когда вам будет предложено установить программное обеспечение, выберите **Install Now.**

4. Вы будете направлены на страницу загрузки для вашего браузера. Выберите **дополнение.**

5. Если вам подается запрос, выберите **Включить** или **разрешить**.

6. После установки перезапустите браузер.

7. Войти на панель доступа. Проверьте, можете ли вы открыть приложения с поддержкой паролей с поддержкой SSO.

Вы также можете непосредственно скачать расширение браузера для Chrome и Firefox через эти ссылки:

-   [Расширение панели доступа Chrome](https://chrome.google.com/webstore/detail/access-panel-extension/ggjhpefgjjfobnfoldnjipclpcfbgbhl)

-   [Расширение панели доступа Firefox](https://addons.mozilla.org/firefox/addon/access-panel-extension/)

## <a name="view-portal-notification-details"></a>Просмотр уведомлений на портале

Чтобы увидеть подробную информацию о любом уведомлении портала, выполните следующие действия:

1. Выберите значок **уведомлений** (колокол) в правом верхнем углу портала Azure.

2. Выберите любое уведомление, отображаевое состояние *ошибки.* (У них есть красное "!".)

   > [!NOTE]
   > Нельзя выбирать уведомления, находимые в состоянии *«Успешный»* или *«В прогрессе».*

3. Откроется панель **Сведения об уведомлении**. Ознакомьте с информацией, чтобы узнать о проблеме.

5. Если вам все еще нужна помощь, поделитесь информацией с инженером службы поддержки или группой продуктов. Выберите значок **копии** справа от окна **ошибки Copy,** чтобы скопировать сведения об уведомлении, чтобы поделиться.

## <a name="send-notification-details-to-a-support-engineer-to-get-help"></a>Отправка уведомлений инженеру службы поддержки для получения справки

Важно, чтобы вы делились *всеми* деталями, перечисленными в этом разделе, с поддержкой, чтобы они могли помочь вам быстро. Чтобы записать его, вы можете сделать скриншот или выбрать **ошибку копирования.**

Ниже приводится информация о том, что означает каждый элемент уведомления, и приводятся примеры.

### <a name="essential-notification-items"></a>Основные элементы уведомления

- **Название**: описательное название уведомления.

   Пример: *Настройки прокси-сервера приложений*

- **Описание**: то, что произошло в результате операции.

   Пример: *Внутренний URL-адрес, введенный, уже используется другим приложением.*

- **Идентификатор уведомления**: уникальный идентификатор уведомления.

    Пример: *clientNotification-2adbfc06-2073-4678-a69f-7eb78d96b068*

- **Идентификатор запроса клиента:** идентификатор конкретного запроса, сделанный вашим браузером.

    Пример: *302fd775-3329-4670-a9f3-bea37004f0bc*

- **Time Stamp UTC**: отметка времени момента, когда произошло уведомление, в UTC.

    Пример: *2017-03-23T19:50:43.7583681*

- **Внутренний идентификатор транзакций:** внутренний идентификатор, который используется для поиска ошибки в наших системах.

    Пример: **71a2f329-ca29-402f-aa72-bc00a7aca603**

- **UPN**: Пользователь, который запустил операцию.

    Пример: *tperkins\@f128.info*

- **Идентификатор клиента:** уникальный идентификатор клиента, членом которого является пользователь, управлявший операцией.

    Пример: *7918d4b5-0442-4a97-be2d-36f99962ece*

- **Идентификатор объекта пользователя**: Уникальный идентификатор пользователя, который запустил операцию.

    Пример: *17f84be4-51f8-483a-b533-383791227a99*

### <a name="detailed-notification-items"></a>Подробные элементы уведомления

- **Название дисплея**: (может быть пустым) более подробное название отображения для ошибки.

    Пример: *Настройки прокси-сервера приложений*

- **Статус**: конкретный статус уведомления.

    Пример: *Не удалось*

- **Идентификатор объекта:**(может быть пустым) идентификатор объекта, против которого была запущена операция.

   Пример: *8e08161d-f2fd-40ad-a34a-a9632d6bb599*

- **Подробности**: подробное описание того, что произошло в результате операции.

    Пример: *Внутренний<https://bing.com/>URL ' является недействительным, поскольку он уже используется.*

- **Ошибка копирования**: Позволяет выбрать **значок копии** справа от текстового ящика **ошибки Copy** для копирования деталей уведомлений, чтобы помочь с поддержкой.

    Примере:```{"errorCode":"InternalUrl\_Duplicate","localizedErrorDetails":{"errorDetail":"Internal url 'https://google.com/' is invalid since it is already in use"},"operationResults":\[{"objectId":null,"displayName":null,"status":0,"details":"Internal url 'https://bing.com/' is invalid since it is already in use"}\],"timeStampUtc":"2017-03-23T19:50:26.465743Z","clientRequestId":"302fd775-3329-4670-a9f3-bea37004f0bb","internalTransactionId":"ea5b5475-03b9-4f08-8e95-bbb11289ab65","upn":"tperkins@f128.info","tenantId":"7918d4b5-0442-4a97-be2d-36f9f9962ece","userObjectId":"17f84be4-51f8-483a-b533-383791227a99"}```

# <a name="problems-configuring-password-single-sign-on-for-a-non-gallery-application"></a>Проблемы при настройке пароля единого входа для приложения не из коллекции

В этой статье описаны распространенные проблемы, которые могут возникнуть при настройке *единый вход с паролем* (SSO) для приложения не из коллекции.

## <a name="capture-sign-in-fields-for-an-app"></a>Полей входа в систему для приложения

Полей входа в систему поддерживается только для с поддержкой HTML страницы входа. Он имеет не поддерживается для нестандартных страниц входа, наподобие тех, в которых используется, Adobe Flash или других технологий, не поддерживающими HTML.

Существует два способа для полей входа в систему для пользовательских приложений:

- **Автоматическое полей входа в систему** хорошо работает с большинством с поддержкой HTML страницы входа, *при работе с хорошо известные идентификаторы DIV* для полей имени и пароля пользователя. На странице HTML является полученное идентификаторы DIV, которые соответствуют определенным критериям. Эти метаданные сохраняются таким образом, они могут быть воспроизведены в приложение более поздней версии.

- **Вручную полей входа в систему** используется, если приложение поставщика *не метки поля ввода вход*. Определения полей вручную используется также в том случае, если поставщик *предусмотрел несколько полей, которые не могут быть автоматически обнаруженных*. Azure Active Directory (Azure AD) может хранить данные для любого количества полей, так как на странице входа в систему, если вы сообщаете, где находятся эти поля на странице.

Как правило если захват полей автоматического входа не подходит, попробуйте вручную.

### <a name="automatically-capture-sign-in-fields-for-an-app"></a>Автоматически полей входа в систему для приложения

Чтобы настроить единый вход по паролю с помощью полей автоматического входа в систему, выполните следующие действия.

1. Откройте [портал Azure](https://portal.azure.com/). Войдите в качестве глобального администратора или соадминистратора.

2. В области навигации слева выберите **все службы** открыть расширения Azure AD.

3. Тип **Azure Active Directory** в поле фильтра поиска, а затем выберите **Azure Active Directory**.

4. Выберите **корпоративные приложения** в области навигации Azure AD.

5. Выберите **все приложения** для просмотра списка приложений.

   > [!NOTE]
   > Если вы не видите приложение, для которого необходимо, используйте **фильтра** элемента управления в верхней части **все приложения** списка. Задайте **Показать** возможность «Все приложения».

6. Выберите приложение, которое вы хотите настроить единый вход.

7. После загрузки приложения выберите **единого входа** в области навигации слева.

8. Выберите **пароль входа на основе** режим.

9. Введите **URL-адрес входа**, который является URL-адрес страницы, где пользователи вводят имя пользователя и пароль для входа. *Убедитесь, что поля входа в систему доступны на странице URL-адреса, введенного*.

10. Щелкните **Сохранить**.

    Страницы будет автоматически использоваться для пользователя имя и пароль полей ввода. Теперь можно использовать Azure AD для безопасной передачи паролей в приложение с помощью расширения браузера для панели доступа.

### <a name="manually-capture-sign-in-fields-for-an-app"></a>Вручную полей входа в систему для приложения

Полей вручную входа в систему, необходимо установить расширение обозревателя панели доступа. Кроме того, в браузере не может выполняться в *inPrivate*, *инкогнито*, или *частного* режим.

Чтобы установить расширение, см. в разделе [установить расширение панели доступа для браузера](#install-the-access-panel-browser-extension) этой статьи.

Чтобы настроить единый вход по паролю для приложения с помощью ручной полей входа в систему, выполните следующие действия.

1. Откройте [портал Azure](https://portal.azure.com/). Войдите в качестве глобального администратора или соадминистратора.

2. В области навигации слева выберите **все службы** открыть расширения Azure AD.

3. Тип **Azure Active Directory** в поле фильтра поиска, а затем выберите **Azure Active Directory**.

4. Выберите **корпоративные приложения** в области навигации Azure AD.

5. Выберите **все приложения** для просмотра списка приложений.

   > [!NOTE] 
   > Если вы не видите приложение, для которого необходимо, используйте **фильтра** элемента управления в верхней части **все приложения** списка. Задайте **Показать** возможность «Все приложения».

6. Выберите приложение, которое вы хотите настроить единый вход.

7. После загрузки приложения выберите **единого входа** в области навигации слева.

8. Выберите **пароль входа на основе** режим.

9. Введите **URL-адрес входа**, это страница, где пользователи вводят имя пользователя и пароль для входа. *Убедитесь, что поля входа в систему доступны на странице URL-адреса, введенного*.

10. Выберите **Настройка *&lt;appname&gt;* параметры пароля единого входа**.

11. Выберите **вручную обнаруживать поля для входа в систему**.

14. Выберите **ОК**.

15. Щелкните **Сохранить**.

16. Следуйте инструкциям, чтобы использовать панель доступа.

## <a name="troubleshoot-problems"></a>Устранение неполадок

### <a name="i-get-a-we-couldnt-find-any-sign-in-fields-at-that-url-error"></a>Появляется сообщение об ошибке «Не удалось найти все поля входа в этот URL-адрес»

Вы получаете это сообщение об ошибке при сбое автоматическое определение полей входа в систему. Чтобы устранить эту проблему, попробуйте определение полей вручную входа в систему. См. в разделе [вручную полей входа в систему для приложения](#manually-capture-sign-in-fields-for-an-app) этой статьи.

### <a name="i-get-an-unable-to-save-single-sign-on-configuration-error"></a>Я получаю «Не удалось сохранить конфигурацию единого входа» ошибка

В редких случаях обновление конфигурации единого входа сбоя. Чтобы устранить эту проблему, попробуйте еще раз сохранить конфигурацию.

Если появляется ошибка, откройте обращение в службу поддержки. Включает информацию, которая описана в [просмотреть сведения об уведомлении портала](#view-portal-notification-details) и [отправлять сведения об уведомлении инженеру службы поддержки для получения справки](#send-notification-details-to-a-support-engineer-to-get-help) разделах этой статьи.

### <a name="i-cant-manually-detect-sign-in-fields-for-my-app"></a>Не удается вручную определить поля входа для приложения

После входа в систему вручную не работает, можно заметить следующие особенности:

- По-видимому, процесс определения полей вручную работает, но записанного поля были указаны неправильно.

- Нужные поля не получить остается выделенным, когда запускается процесс отслеживания.

- Процесс отслеживания осуществляется переход к странице входа приложения должным образом, но ничего не происходит.

- По-видимому, определения полей вручную работает, но единого входа не выполняется, когда пользователи переходят к приложению из панели доступа.

При возникновении любой из этих проблем, выполните указанные ниже действия:

- Убедитесь, что у вас есть последнюю версию расширение обозревателя панели доступа *установлено и включено*. См. в разделе [Установка расширения браузера для панели доступа](#install-the-access-panel-browser-extension) этой статьи.

- Убедитесь, что браузер не находится в *инкогнито*, *inPrivate*, или *частного* режим во время записи. Расширение панели доступа не поддерживается в этих режимах.

- Убедитесь, что пользователи не пытаются войти в приложение из панели доступа во время в *инкогнито*, *inPrivate*, или *режиме защищенного просмотра*.

- Повторите процесс определения полей вручную. Убедитесь, что нужные поля помечены красными отметками.

- Если кажется, что процесс определения полей вручную перестает отвечать на запросы или страницы входа не отвечает, повторите процесс определения полей вручную. Но на этот раз по завершении процесса нажмите клавишу F12, чтобы открыть консоль разработчика в браузере. Выберите **консоли** вкладки. Тип **window.location=» *&lt;входа URL-адрес, указанный при настройке приложения&gt;* "** , и нажмите клавишу ВВОД. В результате перенаправления на странице, процесс определения завершится и найденные поля, которые были записаны.

## <a name="install-the-access-panel-browser-extension"></a>Установка расширения браузера для панели доступа

Выполните следующие действия.

1. Откройте [панели доступа](https://myapps.microsoft.com) в поддерживаемом браузере. Войдите в Azure AD в качестве *пользователя*.

2. Выберите **password-SSO application** панели доступа.

3. Когда появится запрос на его установку, выберите **установить сейчас**.

4. Вы будете перенаправлены на страницу загрузки для веб-браузера. Решили **добавить** расширения.

5. Если появится запрос, выберите **включить** или **Разрешить**.

6. После завершения установки перезапустите браузер.

7. Войдите в панель доступа. См. в разделе, если его можно открыть свои приложения с поддержкой единого входа. пароль.

Можно также напрямую скачивание расширения браузера для Chrome и Firefox через следующие ссылки:

-   [Расширение панели доступа для Chrome](https://chrome.google.com/webstore/detail/access-panel-extension/ggjhpefgjjfobnfoldnjipclpcfbgbhl)

-   [Расширение панели доступа для Firefox](https://addons.mozilla.org/firefox/addon/access-panel-extension/)

## <a name="view-portal-notification-details"></a>Просмотр сведений о уведомление на портале

Чтобы просмотреть сведения о уведомления с портала, выполните следующие действия:

1. Выберите **уведомления** значок (колокольчик) в правом верхнем углу портала Azure.

2. Выберите уведомление, показывающий *ошибка* состояния. (Они имеют красным значком «!».)

   > [!NOTE]
   > Невозможно выбрать уведомления, которые находятся в *успех* или *In Progress* состояния.

3. Откроется панель **Сведения об уведомлении**. Ознакомьтесь с информацией, чтобы узнать о проблеме.

5. Если вы по-прежнему нужна помощь, делиться информацией с инженером службы поддержки или сотруднику группы продуктов. Выберите **копирования** значок справа от **Копировать ошибку** поле, чтобы скопировать сведения об уведомлении для совместного использования.

## <a name="send-notification-details-to-a-support-engineer-to-get-help"></a>Отправить сведения об уведомлении инженеру службы поддержки для получения справки

Очень важно, что используют *все* сведения, приведенные в этом разделе с поддержкой таким образом, они могут помочь быстро. Чтобы записать его, можно сделать снимок экрана или выбрать **Копировать ошибку**.

Ниже приведено описание, что каждый элемент уведомления означает, что и приводит примеры.

### <a name="essential-notification-items"></a>Основные элементы уведомления

- **Заголовок**: описательное название уведомления.

   Пример: *Параметры прокси приложения*

- **Описание**: что именно произошло в результате выполнения операции.

   Пример: *Введенный внутренний URL-адрес уже используется другим приложением.*

- **Идентификатор уведомления**: Уникальный идентификатор уведомления.

    Пример: *clientNotification-2adbfc06-2073-4678-a69f-7eb78d96b068*

- **Идентификатор запроса клиента**: идентификатор конкретного запроса, внесенные в браузере.

    Пример: *302fd775-3329-4670-a9f3-bea37004f0bc*

- **Время UTC штамп**: метка времени возникновения уведомления, в формате UTC.

    Пример: *2017-03-23T19:50:43.7583681Z*

- **ИД внутренней транзакции**: внутренний идентификатор, используемый для поиска ошибки в наших системах.

    Пример: **71a2f329-ca29-402f-aa72-bc00a7aca603**

- **ИМЯ УЧАСТНИКА-ПОЛЬЗОВАТЕЛЯ**: Пользователь, выполнивший операцию.

    Пример: *tperkins\@f128.info*

- **Идентификатор клиента**: Уникальный идентификатор клиента, который входит пользователь, выполнивший операцию.

    Пример: *7918d4b5-0442-4a97-be2d-36f9f9962ece*

- **Идентификатор объекта пользователя**: Уникальный идентификатор пользователя, выполнившего операцию.

    Пример: *17f84be4-51f8-483a-b533-383791227a99*

### <a name="detailed-notification-items"></a>Элементы уведомления с подробными сведениями

- **Отображаемое имя**: (может быть пустым) более подробные отображаемые имена для ошибки.

    Пример: *Параметры прокси приложения*

- **Состояние**: конкретное состояние уведомления.

    Пример: *Сбой*

- **Идентификатор объекта**: (может быть пустым) идентификатор объекта, для которого была выполнена операция.

   Пример: *8e08161d-f2fd-40ad-a34a-a9632d6bb599*

- **Сведения о**: подробное описание результата операции.

    Пример: *Внутренний URL-адрес "<https://bing.com/>" является недопустимым, так как он уже используется.*

- **Скопировать ошибку**: Позволяет выбрать **значок копирования** справа от **Копировать ошибку** текстовое поле, чтобы скопировать подробности уведомления, чтобы помочь с поддержкой.

    Пример:   ```{"errorCode":"InternalUrl\_Duplicate","localizedErrorDetails":{"errorDetail":"Internal url 'https://google.com/' is invalid since it is already in use"},"operationResults":\[{"objectId":null,"displayName":null,"status":0,"details":"Internal url 'https://bing.com/' is invalid since it is already in use"}\],"timeStampUtc":"2017-03-23T19:50:26.465743Z","clientRequestId":"302fd775-3329-4670-a9f3-bea37004f0bb","internalTransactionId":"ea5b5475-03b9-4f08-8e95-bbb11289ab65","upn":"tperkins@f128.info","tenantId":"7918d4b5-0442-4a97-be2d-36f9f9962ece","userObjectId":"17f84be4-51f8-483a-b533-383791227a99"}```

# <a name="configure-password-single-sign-on-for-an-azure-ad-gallery-application"></a>Настройка единого входа по паролю для приложения из коллекции Azure AD

При добавлении приложения из [коллекции приложений Azure Active Directory (Azure AD)](https://docs.microsoft.com/azure/active-directory/active-directory-appssoaccess-whatis)можно выбрать способ входа пользователей в это приложение. Этот вариант можно настроить в любое время, выбрав **единый вход** в корпоративном приложении на [портал Azure](https://portal.azure.com/).

Одним из доступных вариантов единого входа является [единый вход на основе пароля](https://docs.microsoft.com/azure/active-directory/active-directory-appssoaccess-whatis). Это отличный способ начать быструю интеграцию приложений в Azure AD. Он обеспечивает следующие действия:

-   Безопасно сохраняет и воспроизводит имена пользователей и пароли для приложения, интегрированного с Azure AD.

-   Обеспечивает поддержку для приложений, для которых требуется несколько полей входа, помимо полей username и Password.

-   Позволяет настроить метки полей имени пользователя и пароля, которые пользователи видят на [панели доступа к приложениям](https://docs.microsoft.com/azure/active-directory/active-directory-saas-access-panel-introduction) при вводе учетных данных.

-   Позволяет пользователям предоставлять собственные имена пользователей и пароли для всех существующих учетных записей приложений, которые они вводят вручную на [панели доступа к приложениям](https://docs.microsoft.com/azure/active-directory/active-directory-saas-access-panel-introduction) .

-   Позволяет члену бизнес-группы использовать функцию самостоятельного [доступа к приложениям](https://docs.microsoft.com/azure/active-directory/active-directory-self-service-application-access) для указания имен пользователей и паролей, назначенных пользователю.

-   Позволяет администратору указать имена пользователей и пароли, назначенные пользователю, с помощью функции обновления учетных данных при [назначении пользователя приложению](#assign-a-user-to-an-application-directly) .

-   Позволяет администратору использовать функцию обновления учетных данных, чтобы указать общее имя пользователя или пароль для группы людей, [назначенных приложению](#assign-an-application-to-a-group-directly) .

В следующем разделе описано, как включить [единый вход на основе пароля](https://docs.microsoft.com/azure/active-directory/active-directory-appssoaccess-whatis) в приложение, которое уже находится в [коллекции приложений Azure AD](https://docs.microsoft.com/azure/active-directory/active-directory-appssoaccess-whatis).

## <a name="overview-of-required-steps"></a>Общие сведения о необходимых шагах

-   [Добавление приложения из коллекции Azure AD](#add-an-application-from-the-azure-ad-gallery)

-   [настройте приложение для единого входа по паролю](#configure-the-application-for-password-single-sign-on);

-   Назначьте приложение пользователю или группе:

    -   [Назначьте приложение группе напрямую](#assign-an-application-to-a-group-directly).

## <a name="add-an-application-from-the-azure-ad-gallery"></a>Добавление приложения из коллекции Azure AD

Чтобы добавить приложение из коллекции Azure AD, выполните следующие действия.

1.  Откройте [портал Azure](https://portal.azure.com)и войдите в систему как **глобальный администратор** или Соадминистратор.

2.  Откройте **расширение Azure Active Directory** , выбрав **все службы** в верхней части меню слева.

3.  В поле поиска введите **Azure Active Directory** , а затем выберите элемент **Azure Active Directory** .

4.  Выберите **корпоративные приложения** в меню Azure AD слева.

5.  Нажмите кнопку **Добавить** в правом верхнем углу области **корпоративные приложения** .

6.  В поле **введите имя** в разделе **Добавить из коллекции** введите имя приложения.

7.  Выберите приложение, для которого требуется настроить единый вход.

8.  Перед добавлением приложения его имя можно изменить в поле **имя** .

9.  Нажмите кнопку **Добавить** , чтобы добавить приложение.

Через некоторое время появится область настройки приложения.

## <a name="configure-the-application-for-password-single-sign-on"></a>Настройка приложения для единого входа на основе пароля

Чтобы настроить для приложения единый вход, выполните следующие инструкции:

1. Откройте [портал Azure](https://portal.azure.com/) и войдите в систему как **глобальный администратор** или Соадминистратор.

2. Откройте **расширение Azure Active Directory** , выбрав **все службы** в верхней части меню слева.

3. В поле поиска введите **Azure Active Directory** , а затем выберите элемент **Azure Active Directory** .

4. Выберите **корпоративные приложения** в меню Azure Active Directory слева.

5. Щелкните **Все приложения**, чтобы просмотреть полный список приложений.

   Если нужное приложение не отображается, используйте элемент управления **Фильтр** в верхней части **списка все приложения**и задайте для параметра **Показать** значение **все приложения**.

6. Выберите приложение, для которого требуется настроить единый вход.

7. После загрузки приложения выберите пункт **единый вход** в меню приложения слева.

8. Выберите режим **входа на основе пароля** .

9. [Назначьте пользователей приложению](#assign-a-user-to-an-application-directly).

10. Вы также можете указать учетные данные от имени пользователей, выбрав строку пользователя, щелкнув **обновить учетные данные**, а затем введя имя пользователя и пароль. В противном случае пользователям будет предложено ввести учетные данные при запуске приложения.

## <a name="assign-a-user-to-an-application-directly"></a>Назначение приложения пользователю напрямую

Чтобы напрямую назначить одного или нескольких пользователей для приложения, выполните следующие инструкции:

1. Откройте [портал Azure](https://portal.azure.com/) и войдите в систему как **глобальный администратор**.

2. Откройте **расширение Azure Active Directory** , выбрав **все службы** в верхней части меню слева.

3. В поле поиска введите **Azure Active Directory** , а затем выберите элемент **Azure Active Directory** .

4. Выберите **корпоративные приложения** в меню Azure Active Directory слева.

5. Щелкните **Все приложения**, чтобы просмотреть полный список приложений.

   Если нужное приложение не отображается, используйте элемент управления **Фильтр** в верхней части **списка все приложения**и задайте для параметра **Показать** значение **все приложения**.

6. Выберите приложение, которому нужно назначить пользователя.

7. После загрузки приложения выберите **Пользователи и группы** в меню приложения слева.

8. Нажмите кнопку **Добавить** в верхней части списка **Пользователи и группы** , чтобы открыть область **Добавление назначения** .

9. На панели **Добавление назначения** выберите **Пользователи и группы** .

10. Введите полное имя или адрес электронной почты пользователя в поле **Поиск по имени или адресу электронной почты** .

11. Наведите указатель мыши на пользователя в списке, а затем установите флажок рядом с фотографией или логотипом профиля пользователя, чтобы добавить их в **выбранный** список.

12. Дополнительно Если вы хотите добавить несколько пользователей, введите другое имя или адрес электронной почты в поле **Поиск по имени или адресу электронной почты** и установите флажок для этого пользователя, чтобы добавить его в **выбранный** список.

13. Завершив выбор пользователей, нажмите кнопку **выбрать** , чтобы добавить их в список пользователей и групп, которые будут назначены приложению.

14. Дополнительно Используйте команду **выбрать роль** на панели **Добавление назначения** , чтобы выбрать роль для назначения выбранным пользователям.

15. Выберите **назначить** , чтобы назначить приложение выбранным пользователям.

## <a name="assign-an-application-to-a-group-directly"></a>Назначение приложения группе напрямую

Чтобы напрямую назначить приложение одной или нескольким группам, выполните следующие инструкции:

1. Откройте [портал Azure](https://portal.azure.com/) и войдите в систему как **глобальный администратор**.

2. Откройте **расширение Azure Active Directory** , выбрав **все службы** в верхней части меню слева.

3. В поле поиска введите **Azure Active Directory** , а затем выберите элемент **Azure Active Directory** .

4. Выберите **корпоративные приложения** в меню Azure AD слева.

5. Щелкните **Все приложения**, чтобы просмотреть полный список приложений.

   Если нужное приложение не отображается, используйте элемент управления **Фильтр** в верхней части **списка все приложения** и задайте для параметра **Показать** значение **все приложения**.

6. Выберите приложение, которому нужно назначить пользователя.

7. После загрузки приложения выберите **Пользователи и группы** в меню приложения слева.

8. Нажмите кнопку **Добавить** в верхней части списка **Пользователи и группы** , чтобы открыть область **Добавление назначения** .

9. На панели **Добавление назначения** выберите **Пользователи и группы** .

10. Введите полное имя группы, которую вы хотите назначить, в поле поиска **по имени или адресу электронной почты** .

11. Наведите указатель мыши на **группу** в списке, а затем установите флажок рядом с фотографией или эмблемой профиля группы, чтобы добавить группу в **выбранный** список.

12. Дополнительно Если вы хотите добавить несколько групп, введите другое имя группы в поле Поиск **по имени или адресу электронной почты** , а затем установите соответствующий флажок, чтобы добавить эту группу в **выбранный** список.

13. После завершения выбора групп нажмите кнопку **выбрать** , чтобы добавить их в список пользователей и групп, которые будут назначены приложению.

14. Дополнительно Используйте команду **выбрать роль** на панели **Добавление назначения** , чтобы выбрать роль для назначения выбранным группам.

15. Выберите **назначить** , чтобы назначить приложение выбранным группам.

Через некоторое время выбранные пользователи должны иметь возможность запускать эти приложения с панели доступа.

# <a name="integrate-with-azure-ad-application-proxy-on-a-network-device-enrollment-service-ndes-server"></a>Интеграция с прокси-сервером Azure AD на сервере службы регистрации сетевых устройств (NDES)

Прокси-сервер приложений Azure Active Directory (AD) позволяет публиковать приложения внутри сети. Эти приложения являются такими, как сайты SharePoint, Microsoft Outlook Web App и другие веб-приложения. Он также обеспечивает безопасный доступ к пользователям за пределами вашей сети через Azure.

Если вы новичок в приложении Azure AD Proxy и хотите узнать больше, [см.](application-proxy.md)

Прокси-приложение Azure AD построено на Azure. Это дает вам огромное количество пропускной способности сети и серверной инфраструктуры для лучшей защиты от распределенных атак типа «отказ в обслуживании» (DDOS) и превосходной доступности. Кроме того, нет необходимости открывать внешние порты брандмауэра для вашей преднациональной сети, и не требуется сервер ДМЗ. Весь трафик происходит входящих. Полный список исходящих портов можно получить в [справочнике: Добавить приложение для удаленного доступа через приложение Proxy в Active Directory Azure Active.](https://docs.microsoft.com/azure/active-directory/manage-apps/application-proxy-add-on-premises-application#prepare-your-on-premises-environment)

> Прокси-приложение Azure AD — это функция, которая доступна только в том случае, если вы используете Премиум или Базовые издания Active Directory Azure. Дополнительные сведения см. в разделе [Цены на Azure Active Directory](https://azure.microsoft.com/pricing/details/active-directory/). 
> Если у вас есть лицензии Enterprise Mobility Suite (EMS), вы имеете право использовать это решение.
> Разъем Azure AD Application Proxy устанавливается только на Windows Server 2012 R2 или позже. Это также является требованием сервера NDES.

## <a name="install-and-register-the-connector-on-the-ndes-server"></a>Установка и регистрация разъема на сервере NDES

1. Войдите на [портал Azure](https://portal.azure.com/) как администратор приложения каталога, который использует прокси приложения. Например, если домен клиента contoso.com, администратором должен быть пользователь admin@contoso.com или другой псевдоним администратора в этом домене.
1. В правом верхнем углу выберите свое имя пользователя. Убедитесь, что вы вошли в каталог, который использует Application Proxy. Если необходимо изменить каталоги, щелкните **Переключение каталога** и выберите каталог, который использует Application Proxy.
1. В области навигации слева выберите **Azure Active Directory**.
1. В разделе **Управление** выберите **Прокси приложения**.
1. Выберите **Скачать службу соединителя**.

    ![Скачайте службу соединителя, чтобы просмотреть условия предоставления услуг.](./media/active-directory-app-proxy-protect-ndes/application-proxy-download-connector-service.png)

1. Ознакомьтесь с условиями предоставления услуг. Когда вы будете готовы, выберите **Accept terms & Download** (Принять условия и скачать).
1. Копируйте файл настройки подключения azure AD Для подключения к серверу NDES. 
   > Вы можете установить разъем на любом сервере в корпоративной сети с доступом к NDES. Вам не нужно устанавливать его на самом сервере NDES.
1. Выполнить файл настройки, например *AADApplicationProxyConnectorInstaller.exe*. Примите условия лицензионного соглашения.
1. Во время установки вам предлагается зарегистрировать разъем с помощью прокси-сервера приложения в каталоге Azure AD.
   * Предоставьте учетные данные для глобального администратора или администратора приложений в каталоге Azure AD. Глобальные учетные данные Администратора Azure AD или администратора приложений могут отличаться от учетных данных Azure на портале.

        > [!NOTE]
        > Глобальная учетная запись администратора приложений, используемая для регистрации разъема, должна принадлежать к тому же каталогу, где вы включите службу Application Proxy.
        >
        > Например, если домен Azure AD *находится contoso.com,* `admin@contoso.com` администратор глобального/приложения должен быть или другим действительным псевдонимом этого домена.

   * Если internet Explorer Улучшенная конфигурация безопасности включена для сервера, где вы устанавливаете разъем, экран регистрации может быть заблокирован. Чтобы обеспечить доступ, следуйте инструкциям в сообщении об ошибке или выключите Internet Explorer Enhanced Security во время процесса установки.
   * Если регистрация разъема не удается, [см.](application-proxy-troubleshoot.md)
1. В конце установки отображается примечание для сред с исходящим прокси. Чтобы настроить разъем прокси-сервер Azure AD Application для работы через `C:\Program Files\Microsoft AAD App Proxy connector\ConfigureOutBoundProxy.ps1`исходящий прокси, запустите предоставленный скрипт, например
1. На странице прокси Приложения на портале Azure новый разъем указан со статусом *Active,* как показано в следующем примере:

    ![Новый разъем прокси-сервер Azure AD, работающий на портале Azure](./media/active-directory-app-proxy-protect-ndes/connected-app-proxy.png)

    > [!NOTE]
    > Чтобы обеспечить высокую доступность для приложений, удостоверяющих подлинность через прокси-приложение Azure AD, можно установить разъемы на нескольких всах. Повторите те же шаги, перечисленные в предыдущем разделе, чтобы установить разъем на других серверах, объединенных с доменом Azure AD DS.

1. После успешной установки вернитесь на портал Azure.

1. Выберите **Корпоративные приложения**.

   ![убедитесь, что вы привлекаете нужных заинтересованных сторон](./media/active-directory-app-proxy-protect-ndes/azure-active-directory-enterprise-applications.png)

1. Выберите **новое приложение,** а затем выберите **приложение On-premises**. 

1. На **добавлении собственного приложения,** настроить следующие поля:

   * **Имя**: Введите имя приложения.
   * **Внутренний Url**: Введите внутренний URL/F-DN вашего сервера NDES, на котором вы установили разъем.
   * **Предварительная аутентификация**: Выберите **прохождение**. Невозможно использовать любую форму предварительной аутентификации. Протокол, используемый для запросов сертификатов (SCEP), не предоставляет такой опции.
   * Копируйте предоставленный **внешний URL** на буфер обмена.

1. Выберите **«Добавить,** чтобы сохранить приложение.

1. Проверьте, можно ли получить доступ к серверу NDES через прокси-сервер Azure AD Application, вставив ссылку, которую вы скопировали в шаге 10, в браузер. Вы должны увидеть страницу приветствия IIS по умолчанию.

1. В качестве заключительного теста добавьте путь *mscep.dll* к существующему URL-адресу, который вы вставили на предыдущем этапе:

   https://scep-test93635307549127448334.msappproxy.net/certsrv/mscep/mscep.dll

1. Вы должны увидеть **HTTP Ошибка 403 - Запретный** ответ.

1. Изменение URL-адреса NDES при условии (через Microsoft Intune) на устройства, это изменение может быть либо в Microsoft Endpoint Configuration Center или в Intune Cloud.

   * Для центра конфигурации перейдите в пункт регистрации сертификатов (CRP) и отрегулируйте URL-адрес. Этот URL-адрес является то, что устройства призывают и представить свою задачу.
   * Для Intune Cloud Only, также известного как Intune Standalone, либо отображайте, либо создайте новую политику SCEP и добавьте новый URL-адрес.

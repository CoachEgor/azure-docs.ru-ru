# <a name="debug-application-proxy-application-issues"></a>Устранить проблемы приложения прокси приложения 

Эта статья поможет вам устранить неполадки с приложениями прокси приложения Azure Active Directory (Azure AD). Если вы используете службу прокси приложения для удаленного доступа к веб-приложение в локальной, но возникли проблемы с подключением к приложению, используйте этот блок-схема для отладки проблем приложений. 

## <a name="before-you-begin"></a>Перед началом работы

При устранении неполадок прокси приложения, мы рекомендуем начать с соединителями. Во-первых, следуйте процедуре по устранению неполадок в [отладка соединитель прокси приложения выдает](application-proxy-debug-connectors.md) чтобы убедиться в правильной настройке соединителей прокси приложения. Если вас по-прежнему возникают проблемы, вернитесь к этой статье для устранения неполадок приложения.  

Дополнительные сведения о прокси приложения см. в разделе:

- [Удаленный доступ к локальным приложениям через прокси приложения](application-proxy.md)
- [Соединители прокси приложения](application-proxy-connectors.md)
- [Установка и регистрация соединителя](application-proxy-add-on-premises-application.md)
- [Устранение неполадок и сообщения об ошибках прокси приложения](application-proxy-troubleshoot.md)

## <a name="flowchart-for-application-issues"></a>Блок-схема проблем приложений

Этот блок-схема поможет выполнить действия для отладки, некоторые из более распространенных проблем с подключением к приложению. Дополнительные сведения о каждом шаге см. в разделе таблице на блок-схеме.

![Блок-схема, показывающая шаги для отладки приложения](media/application-proxy-debug-apps/application-proxy-apps-debugging-flowchart.png)

|  | Действие | Описание | 
|---------|---------|---------|
|1 | Откройте браузер, доступа к приложению и введите свои учетные данные | Попробуйте использовать свои учетные данные для входа в приложении, а затем проверьте наличие ошибок, относящиеся к пользователю, например [не может получить доступ к этому корпоративному приложению](application-proxy-sign-in-bad-gateway-timeout-error.md). |
|2 | Проверить назначения пользователя в приложение | Убедитесь, что учетная запись пользователя имеет разрешения на доступ к приложению из корпоративной сети, а затем проверить вход приложение, выполнив действия, описанные в [тестирование приложения](application-proxy-add-on-premises-application.md#test-the-application). Если проблемы входа в систему не удается устранить, см. в разделе [Устранение ошибок входа в систему](https://docs.microsoft.com/azure/active-directory/reports-monitoring/howto-troubleshoot-sign-in-errors).  |
|3 | Откройте браузер и попробуйте получить доступ к приложению | Если сразу же отображается сообщение об ошибке, проверьте, правильно ли настроен прокси приложения. Дополнительные сведения о специальных сообщений об ошибках, см. в разделе [Устранение неполадок и прокси приложения сообщения об ошибках](application-proxy-troubleshoot.md).  |
|4. | Проверьте настройку личного домена или устранения ошибки | Если страница не отображается вообще, убедитесь, что ваш пользовательский домен настроен правильно, просмотрев [работа с пользовательскими доменами](application-proxy-configure-custom-domain.md).<br></br>Если страница не загружается, и появится сообщение об ошибке, устраните ошибки путем обращения к [Устранение неполадок и прокси приложения сообщения об ошибках](application-proxy-troubleshoot.md). <br></br>Если он длится более 20 секунд для сообщение об ошибке отображается, может быть проблема с подключением. Перейдите к [соединители прокси приложения Отладка](application-proxy-debug-connectors.md) статьи об устранении неполадок.  |
|5 | Если проблемы не удается устранить, перейдите к отладке соединителя | Может быть проблема с подключением между прокси и соединителем или между соединителем и серверной части. Перейдите к [соединители прокси приложения Отладка](application-proxy-debug-connectors.md) статьи об устранении неполадок. |
|6 | Опубликовать все ресурсы, проверьте средства разработчика браузера и исправление ссылок | Убедитесь, что путь публикации включает в себя все необходимые изображения, скрипты и таблицы стилей для вашего приложения. Дополнительные сведения см. в разделе [Добавление локального приложения в Azure AD](application-proxy-add-on-premises-application.md#add-an-on-premises-app-to-azure-ad). <br></br>Использовать средства разработчика (Инструменты F12 в Internet Explorer или Microsoft Edge) и проверьте браузера для публикации проблемы, как описано в разделе [Неправильное отображение страницы приложения](application-proxy-page-appearance-broken-problem.md). <br></br>Просмотрите параметры разрешения неработающие ссылки в [ссылки на страницы не работают](application-proxy-page-links-broken-problem.md). |
|7 | Проверка задержки в сети | Если страница загружается медленно, узнайте о способах свести к минимуму задержки в сети [рекомендации по уменьшению задержки](application-proxy-network-topology.md#considerations-for-reducing-latency). | 
|8 | Дополнительную помощь в устранении неполадок см. в разделе | Если проблемы не удается устранить, найти дополнительные статьи по устранению неполадок в [прокси приложения, документацию по устранению неполадок](application-proxy-page-appearance-broken-problem.md). |

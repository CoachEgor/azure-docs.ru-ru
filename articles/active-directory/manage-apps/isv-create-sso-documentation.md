# <a name="create-and-publish-single-sign-on-documentation-for-your-application"></a>Создание и публикация единой документации для приложения   

## <a name="documentation-on-your-site"></a>Документация на вашем сайте

Простота принятия является важным фактором в принятии корпоративных программных решений. Четкая простая в последующей документации поддерживает ваших клиентов в их принятии путешествие и снижает расходы на поддержку. Работая с тысячами поставщиков программного обеспечения, корпорация Майкрософт видела, что работает.

Мы рекомендуем, чтобы ваша документация на вашем сайте, как минимум, включала следующие элементы.

* Введение в функциональность SSO

  * Поддерживаемые протоколы

  * Версия и СКУ

  * Список поставщиков поддерживаемых идентификационных данных со ссылками на документацию

* Информация о лицензировании вашего приложения

* Элемент управления доступом на основе ролей для настройки SSO

* Шаги конфигурации SSO

  * Элементы конфигурации uI для SAML с ожидаемыми значениями от поставщика

  * Информация поставщика услуг, которая будет передана поставщикам удостоверений личности

* Если OIDC/OAuth

  * Список разрешений, необходимых для согласия с бизнес-обоснованиями

* Этапы тестирования для пилотных пользователей

* Информация о устранении неполадок, включая коды ошибок и сообщения

* Механизмы поддержки клиентов

## <a name="documentation-on-the-microsoft-site"></a>Документация на сайте Майкрософт

При рассылке приложения в галерее приложений Active Directory Gallery, которая также публикует приложение в Azure Marketplace, корпорация Майкрософт будет создавать документацию для наших общих клиентов, объясняющих поэтапный процесс. Вы можете увидеть пример [здесь](https://aka.ms/appstutorial). Эта документация создается на основе отправки в галерею, и вы можете легко обновить ее, если внести изменения в приложение с помощью учетной записи GitHub.

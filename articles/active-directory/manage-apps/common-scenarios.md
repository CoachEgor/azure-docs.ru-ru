# <a name="centralize-application-management-with-azure-ad"></a>Централизация управления приложениями с помощью Azure AD

Пароли, как ИТ кошмар и боль для сотрудников по всему миру. Именно поэтому все больше компаний обращаются к сервису Azure Active Directory, решению корпорации Майкрософт по управлению идентификацией и доступом для облака и всех других ресурсов. Перейти от приложения к приложению без необходимости вводить пароль для каждого из них. Перейти от Outlook, к workday, к ADP так быстро, как вы можете открыть их, быстро и безопасно. Затем сотрудничайте с партнерами и даже другими лицами, не входящими в организацию, без вызова ИТ. Более того, Azure AD помогает управлять рисками, обеспечивая безопасность приложений, которые вы используете с помощью многофакторной аутентификации для проверки того, кто вы, используя постоянно адаптивное машинное обучение и интеллект безопасности для обнаружения подозрительных входов, дающих вам безопасный доступ к приложениям, где бы вы ни находились. Это отлично подходит не только для пользователей, но и для ИТ. Благодаря проверкам доступа и полномасштабному набору управления Azure AD, которые выполняют сяодардисту и обеспечивают соблюдение политик. И получить это, вы даже можете автоматизировать предоставление учетных записей пользователей, что делает управление доступом ветер. ознакомьтесь с некоторыми из распространенных сценариев, для которых клиент использует возможности управления приложениями Azure Active Directory.

**Общие сценарии**


> [!div class="checklist"]
> * SSO для всех ваших приложений
> * Автоматизация обеспечения и деобеспечения 
> * Защищайте приложения
> * Управлять доступом к вашим приложениям
> * Гибридный безопасный доступ

## <a name="scenario-1-set-up-sso-for-all-your-applications"></a>Сценарий 1: Настройка SSO для всех ваших приложений

Нет больше управления паролем. Безопасный доступ ко всем ресурсам, необходимым с вашими корпоративными учетными данными. 

|Компонент  | Описание | Рекомендация |
|---------|---------|---------|
|Единый вход|Стандартная федеративная SSO с использованием надежных отраслевых стандартов.|Всегда используйте [SAML / OIDC](https://docs.microsoft.com/azure/active-directory/manage-apps/isv-choose-multi-tenant-federation) для включения SSO, когда ваше приложение поддерживает его.|
|Панель доступа|Предоставляет пользователям простой центр для обнаружения приложений и доступа к ним. Обесключайте их более продуктивными с возможностями самообслуживания, например запросом доступа к приложениям и группам или управлением доступом к ресурсам от имени других.| Развертывание [панели доступа](https://docs.microsoft.com/azure/active-directory/manage-apps/access-panel-deployment-plan) в организации после интеграции приложений с Azure AD для SSO.|

## <a name="scenario-2-automate-provisioning-and-deprovisioning"></a>Сценарий 2: Автоматизация подготовки и деобеспечения 


Большинство приложений требуют, чтобы пользователь был подготовлен в приложение, прежде чем получить доступ к ресурсам, которые им нужны. Использование файлов CSV или сложных скриптов может быть дорогостоящим и трудным в управлении. Кроме того, клиенты должны убедиться, что учетные записи удаляются, когда кто-то не должен иметь доступ больше. Используйте приведенные ниже инструменты для автоматизации подготовки и декручения. 


|Компонент  |Описание|Рекомендация |
|---------|---------|---------|
|Предоставление SCIM|[SCIM](https://aka.ms/SICMOverview) является передовой практикой для автоматизации подготовки пользователей. Любое приложение, соответствующее ТРЕБОВАНИЯМ SCIM, может быть интегрировано с Azure AD. Автоматически создавайте, обновляйте и удаляйте учетные записи пользователей без необходимости поддерживать файлы CSV, пользовательские скрипты или решения на prem.|Ознакомьтесь с растущим списком [предварительно интегрированных](https://docs.microsoft.com/azure/active-directory/saas-apps/tutorial-list) приложений в галерее приложений Azure AD|
|Microsoft Graph|Используйте дыхание и глубину данных, которые Azure AD должен обогатить ваше приложение данными, которые ему нужны.|Используйте [график Майкрософт](https://developer.microsoft.com/graph/) для получения данных из экосистемы Майкрософт. |


## <a name="scenario-3-secure-your-applications"></a>Сценарий 3: Безопасность приложений
Идентичность является стержнем для обеспечения безопасности. Если личность скомпрометирована, это невероятно трудно остановить эффект домино, пока не стало слишком поздно. В среднем проходит более 100 дней, прежде чем организации обнаружат, что был компромисс. Используйте инструменты, предоставляемые Azure AD, чтобы улучшить безопасность приложений. 

|Компонент  |Описание| Рекомендация |
|---------|---------| ---------|
|Многофакторная идентификация Azure|Многофакторная идентификация (MFA) Azure — решение Майкрософт для двухэтапной проверки. Используя одобренные амином методы аутентификации, Azure MFA помогает защитить доступ к вашим данным и приложениям при удовлетворении спроса на простой процесс вхинга.| [Включить МИД](https://techcommunity.microsoft.com/t5/azure-active-directory-identity/all-your-creds-are-belong-to-us/ba-p/855124) для пользователей.  |
|Условный доступ|С помощью Условия доступа можно внедрять автоматизированные решения по управлению доступом для тех, кто может получить доступ к облачным приложениям, в зависимости от условий.| Просмотрите [по умолчанию безопасности](https://docs.microsoft.com/azure/active-directory/fundamentals/concept-fundamentals-security-defaults) и [общие политики,](https://docs.microsoft.com/azure/active-directory/conditional-access/concept-conditional-access-policy-common) которые используют клиенты. | 
|Защита идентификации|Защита идентификации использует сведения, полученные корпорацией Майкрософт в ходе выполнения обязанностей, в организациях с Azure AD, пространстве пользователей с учетными записями Майкрософт, а также в играх с Xbox, чтобы защитить пользователей. Чтобы обнаружить угрозы и защитить клиентов от таких угроз, корпорация Майкрософт анализирует 6,5 триллионов сигналов в день.|Включить [политики защиты идентификационных данных по умолчанию, предоставляемые](https://docs.microsoft.com/azure/active-directory/identity-protection/concept-identity-protection-policies) нашим сервисом. | 

## <a name="scenario-4-govern-access-to-your-applications"></a>Сценарий 4: Регулируйте доступ к вашим приложениям
Управление идентификацией помогает организациям достичь баланса между производительностью - Как быстро человек может получить доступ к приложениям, в которых он нуждается, например, при вступлении в мою организацию? и безопасности (динамика изменения уровня доступа со временем, например при изменении роли пользователя в организации). 

|Компонент  |Описание|Рекомендация |
|---------|---------| ---------|
|Elm|Управление правами Azure AD может помочь пользователям как внутри организации, так и за ее пределами более эффективно управлять доступом к своим приложениям.| Разрешить не-аммингов управлять доступом к своим приложениям с [пакетами доступа.](https://docs.microsoft.com/azure/active-directory/governance/entitlement-management-access-package-first)|
|Проверки доступа|Доступ пользователей к приложениям может быть рассмотрен на регулярной основе, чтобы убедиться, что только нужные люди имеют постоянный доступ.| [Просмотрите доступ](https://docs.microsoft.com/azure/active-directory/governance/access-reviews-overview) к наиболее чувствительным приложениям. |
|Log Analytics|Создание отчетов о том, кто получает доступ к каким приложениям, и храните их в выборе инструмента SIEM для соотношения данных между источниками данных и с течением времени.| Включите [аналитику журналов](https://docs.microsoft.com/azure/active-directory/reports-monitoring/howto-analyze-activity-logs-log-analytics) и настроите оповещения для критических событий, связанных с вашими приложениями. |


## <a name="scenario-5-hybrid-secure-access"></a>Сценарий 5: Гибридный безопасный доступ
Identity может быть вашей плоскостью управления только в том случае, если она может подключить все через облачные и корпоративные приложения. Используйте инструменты, предоставляемые Azure AD и ее партнерами, чтобы обеспечить доступ к устаревшим приложениям.

|Компонент  |Описание|Рекомендация |
|---------|---------|---------|
|Прокси приложения|Сегодня сотрудникам нужно продуктивно работать без привязки к расположению, времени и оборудованию. Они должны получить доступ к приложениям SaaS в облачных и корпоративных приложениях на месте. Прокси-сервер Azure AD Application обеспечивает такой надежный доступ без дорогостоящих и сложных виртуальных частных сетей (VPN) или демилитаризованных зон (ДМЗ).|Настройка [удаленного доступа](https://docs.microsoft.com/azure/active-directory/manage-apps/application-proxy) для приложений на прем. |
|F5, Акамай, Оскалр|Используя имеющийся контроллер доставки и сети, вы можете легко обеспечить безопасность важных устаревших приложений, которые вы раньше не могли защитить с помощью Azure AD. Теперь вы имеете все необходимое для защиты этих приложений.| Используете Akamai, Citrix, F5 или Sscaler? Ознакомьтесь с нашими [заранее построенными решениями.](https://docs.microsoft.com/azure/active-directory/manage-apps/secure-hybrid-access) |

# <a name="one-click-sso-feature-for-azure-ad-gallery-applications"></a>Возможность "Единый вход одним щелчком" для приложений из коллекции Azure AD

 Из этого учебника вы узнаете, как выполнить Единый вход одним щелчком для всех приложений SAML, которые предоставляют пользовательский интерфейс для настройки единого входа.

## <a name="introduction-to-one-click-sso"></a>Общие сведения о Едином входе одним щелчком

Возможность "Единый вход одним щелчком" введена для настройки единого входа для приложений из коллекции Azure AD, которые поддерживают протокол SAML. Мы предоставили этот параметр на странице настройки единого входа Azure AD, чтобы наши клиенты могли автоматически настроить метаданные Azure AD на стороне приложения. Цель — помочь клиентам быстро настроить единый вход, выполняя минимальное количество действий вручную. 

## <a name="advantages-of-the-one-click-sso"></a>Преимущества Единого входа одним щелчком

- Быстрая настройка единого входа в приложениях коллекции, где клиентам необходимо выполнить установку вручную на стороне приложения.
- Более эффективный и точный способ конфигурации.
- Для настройки не нужна связь с партнерами или их поддержка, так как приложение предоставляет пользовательский интерфейс для настройки SAML.

## <a name="prerequisites"></a>Предварительные требования

- Активная подписка приложения с учетными данными администратора, для которой необходимо настроить Единый вход одним щелчком.
- **Расширение браузера "Безопасный вход в мои приложения"** от корпорации Майкрософт, установленное в браузере. Если вы хотите узнать больше об этом расширении, перейдите по этой [ссылке](https://docs.microsoft.com/azure/active-directory/user-help/my-apps-portal-end-user-access).

## <a name="one-click-sso-feature-step-by-step-details"></a>Пошаговые инструкции по настройке возможности "Единый вход одним щелчком"

1. Добавьте приложение из коллекции Azure AD App.

2. Щелкните "Единый вход".

3. Щелкните "Включить единый вход".

4. Введите значения обязательной конфигурации в разделе "Базовая конфигурация SAML".

    > [!NOTE] 
    > Если приложению требуется конфигурация настраиваемых утверждений, настройте их перед выполнением Единого входа одним щелчком.

5. Если возможность "Единый вход одним щелчком" реализована для любого приложения из коллекции, появится следующий экран. Если **расширение браузера "Безопасный вход в мои приложения"** еще не установлено, необходимо щелкнуть параметр **Установить расширение**.

    ![Установка расширения браузера "Безопасный вход в мои приложения"](./media/one-click-sso-tutorial/install-myappssecure-extension.png)

6. После добавления расширения в браузер щелкните **Setup Application Name** (Настроить имя приложения), после чего вы будете перенаправлены на портал администрирования приложения. Чтобы войти в приложение, необходимо выполнить вход от имени администратора.

    ![Настройка имени приложения](./media/one-click-sso-tutorial/setup-sso.png)

7. Теперь расширение браузера автоматически настроит приложение. Сначала оно запрашивает подтверждение того, хотите ли вы продолжить. Щелкните **Да**.

    ![Сохранение автоматически заполненных данных](./media/one-click-sso-tutorial/save-autopopulate.png)

    > [!NOTE]
    > Если в любом приложении требуется дополнительная навигация или действия, вы увидите соответствующие сообщения, запрашивающие выполнение этих действий. 

8. После завершения конфигурации нажмите кнопку **ОК**, чтобы сохранить изменения.

    ![Сохранение автоматически заполненных данных](./media/one-click-sso-tutorial/save-data.png)

9. Отобразится всплывающее сообщение с подтверждением успешного выполнения. Параметры единого входа успешно настроены. Теперь вы можете протестировать приложение.

    ![Единый вход настроен](./media/one-click-sso-tutorial/sso-configured.png)

10. После успешной настройки приложение выйдет из системы и вы вернетесь обратно на портал Azure.

11. Чтобы протестировать единый вход, можно нажать кнопку "Проверка".

# <a name="enable-single-sign-on-for-your-multi-tenant-application"></a>Включить единый вход для мультитенантного приложения  

Когда вы предлагать свое приложение для использования с другими компаниями в покупки или подписки, можно освободить приложение для клиентов в их собственными клиентами Azure. Это называется созданием мультитенантное приложение. Общие сведения об этой концепции, см. в разделе [Мультитенантного приложения в Azure](https://docs.microsoft.com/azure/dotnet-develop-multitenant-applications) и [приложения в Azure Active Directory](../develop/single-and-multi-tenant-apps.md).

## <a name="what-is-single-sign-on"></a>Что такое единый вход

Единый вход (SSO) добавляет безопасность и удобство при входе пользователей приложения с помощью Azure Active Directory и другие удостоверения. При разработке приложения с поддержкой единого входа, пользователям не нужно вводить отдельные учетные данные для доступа к этому приложению. Полное описание единого входа. [См. в разделе единого входа в приложения в Azure Active Directory](what-is-single-sign-on.md).

## <a name="why-enable-single-sign-on-in-your-application"></a>Почему Включить единый вход в приложении?

Есть много преимуществ для включения единого входа в ваше мультитенантное приложение. При включении единого входа для приложения:

* Приложения могут быть перечислены в Azure Marketplace, где приложение будет обнаруживаться средой миллионов организаций, с помощью Azure Active Directory.
  * Позволяет пользователям быстро настроить приложение в Azure AD.

* Приложение может быть видимым в галерее приложений Office 365, в средстве запуска приложений Office 365 и в Microsoft Search на сайте Office.com

* Приложение может использовать Microsoft Graph REST API для доступа к данным, влияющим на производительность пользователя, которая доступна в Microsoft Graph.

* Снизить затраты на поддержку, облегчая для своих клиентов.
  * С группой Azure AD облегчает переход общим клиентам coproduced документацию для конкретного приложения.
  * Если будет включен единый вход одним щелчком, ИТ-администраторам ваших клиентов нет необходимости сведения о настройке приложения для использования в своей организации.

* Предоставить пользователям возможность полностью управлять проверкой подлинности и авторизации их сотрудников и гостевой удостоверений.

  * Поместив все учетной записи управления и соответствие ответственности с владельцем этих идентификаторов клиента.

  * Обеспечивает возможность включить или отключить единый вход для определенных поставщиков удостоверений, групп или пользователей, в соответствии с потребностями бизнеса.

* Можно повысить конкурентоспособность и adoptability. Многие Крупные организации требуют (или цель) своих сотрудников не заметят единого входа для всех приложений. Важно, что упрощает единый вход.

* Можно снизить сопротивление конечных пользователей, что позволяет увеличить использование конечных пользователей и повысить свои доходы.

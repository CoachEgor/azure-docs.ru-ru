# <a name="enable-single-sign-on-for-your-multi-tenant-application"></a>Включить единый вданный для приложения с несколькими тенантами  

Когда вы предлагаете свое приложение для использования другими компаниями через покупку или подписку, вы делаете ваше приложение доступным для клиентов в рамках своих собственных арендаторов Azure. Это называется созданием мультитенантного приложения. Для просмотра этой концепции [см.](https://docs.microsoft.com/azure/dotnet-develop-multitenant-applications) [Tenancy in Azure Active Directory](../develop/single-and-multi-tenant-apps.md)

## <a name="what-is-single-sign-on"></a>Что такое единый знак-on

Единая регистрация (SSO) повышает безопасность и удобство при подписании пользователями приложений с помощью Active Directory Azure и других идентификационных данных. Когда приложение включено SSO, пользователям не нужно вводить отдельные учетные данные для доступа к этому приложению. Для полного объяснения Единого входного. [Смотрите единый подгон приложений в active-каталоге Azure.](what-is-single-sign-on.md)

## <a name="why-enable-single-sign-on-in-your-application"></a>Почему включить единый всвойла в приложении?

Включение SSO в мультитенантное приложение имеет много преимуществ. При включании SSO для приложения:

* Ваше приложение может быть перечислено в Azure Marketplace, где ваше приложение может быть обнаружено миллионами организаций, использующих Active Directory Azure.
  * Позволяет клиентам быстро настроить приложение с помощью Azure AD.

* Ваше приложение можно обнаружить в галерее Office 365 App Gallery, Office 365 App Launcher и в рамках поиска в Microsoft Office.com

* Приложение может использовать API Microsoft Graph REST для доступа к данным, которые повысят производительность пользователей, доступные в Microsoft Graph.

* Вы сокращаете расходы на поддержку, облегчая работу для ваших клиентов.
  * Документация по конкретному применению, сопроизведенная совместно с командой Azure AD для наших общих клиентов, облегчает принятие.
  * Если включена SSO одним щелчком мыши, ИТ-администраторам ваших клиентов не нужно учиться настраивать приложение для использования в своей организации.

* Вы предоставляете своим клиентам возможность полностью управлять аутентификацией и авторизацией идентификационных данных сотрудников и гостей.

  * Размещение всех обязанностей по управлению учетной записью и соответствие требованиям с владельцем этих идентификационных данных.

  * Предоставление возможности включить или отключить SSO для конкретных поставщиков идентификационных данных, групп или пользователей для удовлетворения своих бизнес-потребностей.

* Вы увеличиваете свою товарность и доступность. Многие крупные организации требуют, чтобы (или стремились) их сотрудники имели бесшовный опыт SSO во всех приложениях. Важно упрощать SSO.

* Уменьшается трение конечных пользователей, что может увеличить использование конечных пользователей и увеличить доход.

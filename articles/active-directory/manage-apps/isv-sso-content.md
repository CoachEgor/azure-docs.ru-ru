# <a name="enable-single-sign-on-for-your-multi-tenant-application"></a>Включение единого входа для приложения с несколькими клиентами  

Когда вы предлагаете ваше приложение для использования другими компаниями через покупку или подписку, вы сделаете приложение доступным для клиентов в своих клиентах Azure. Это называется созданием приложения с несколькими клиентами. Общие сведения об этой концепции см. в статье [приложения для клиентов в Azure](https://docs.microsoft.com/azure/dotnet-develop-multitenant-applications) и [в Azure Active Directory](../develop/single-and-multi-tenant-apps.md).

## <a name="what-is-single-sign-on"></a>Что такое единый вход

Единый вход (SSO) обеспечивает безопасность и удобство при входе пользователей в приложения с помощью Azure Active Directory и других удостоверений. Если для приложения включен единый вход, пользователям не нужно вводить отдельные учетные данные для доступа к этому приложению. Полное описание единого входа. [См. раздел единый вход в приложения в Azure Active Directory](what-is-single-sign-on.md).

## <a name="why-enable-single-sign-on-in-your-application"></a>Зачем использовать единый вход в приложении?

Существует множество преимуществ включения единого входа в приложении с несколькими клиентами. При включении единого входа для приложения:

* Приложение может быть указано в Azure Marketplace, где ваше приложение обнаруживается миллионами организаций, использующих Azure Active Directory.
  * Позволяет клиентам быстро настроить приложение с помощью Azure AD.

* Приложение может быть обнаружено в коллекции приложений Office 365, в средстве запуска приложений Office 365 и в Microsoft Search на Office.com

* Приложение может использовать Microsoft Graph REST API для доступа к данным, которые позволяют повысить производительность пользователей, доступную в Microsoft Graph.

* Вы сокращаете затраты на поддержку, делая их более удобными для ваших клиентов.
  * Документация, относящаяся к конкретному приложению, сопоставленная группе Azure AD для наших взаимосвязанных клиентов, упрощает внедрение.
  * Если включен единый вход по одному щелчку, ИТ-администраторам пользователей не нужно научиться настраивать приложение для использования в Организации.

* Вы предоставляете своим клиентам возможность полностью управлять проверкой подлинности и авторизацией своих сотрудников и гостевых удостоверений.

  * Помещая все обязанности по управлению и соответствию учетных записей у владельца этих удостоверений.

  * Предоставление возможности включать или отключать единый вход для конкретных поставщиков удостоверений, групп или пользователей в соответствии с потребностями бизнеса.

* Вы повышаете эффективность рынка и помогаете внедрять свои технологии. Многие крупные организации нуждаются в том, что (или стремитесь) их сотрудники имеют простой единый вход во всех приложениях. Очень важно сделать единый вход.

* Вы сокращаете трение конечного пользователя, что может увеличить использование конечных пользователей и увеличить доходы.

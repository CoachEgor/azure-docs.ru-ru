# <a name="sign-in-problems-with-an-azure-ad-gallery-app-configured-for-sso"></a>Проблемы с ввозами в галерею Azure AD, настроенную для SSO

Панель доступа — это веб-портал. Он позволяет пользователям с помощью рабочих или школьных учетных записей Azure Active Directory (Azure AD) получить доступ к облачным приложениям, на которые у них есть разрешения. Пользователи, имеющие выпуски Azure AD, также могут использовать возможности группы самообслуживания и управления приложениями через панель доступа.

Панель доступа отделена от портала Azure. Пользователям не нужна подписка Azure на использование панели доступа.

Для использования одного знака для вхоложного знака (SSO) в панели доступа необходимо установить расширение панели доступа в вашем браузере. Расширение загружается автоматически при выборе приложения, настроенного для SSO на основе паролей.

## <a name="browser-requirements-for-access-panel"></a>Требования к браузеру для панели доступа

Панель доступа требует браузера, который поддерживает JavaScript и имеет CSS включен.

Следующие браузеры поддерживают SSO на основе паролей:

- Internet Explorer 8, 9, 10 и 11 на Windows 7 или позже

- Хром на Windows 7 или позже или на MacOS X или позже

- Firefox 26.0 или позже на Windows XP SP2 или позже или на Mac OS X 10.6 или позже

>[!NOTE]
>Расширение SSO на основе паролей стало доступным для Microsoft Edge в Windows 10, когда поддержка расширений браузера была добавлена в Microsoft Edge.

## <a name="install-the-access-panel-browser-extension"></a>Установка расширения браузера панели доступа

Выполните следующие действия.

1. Панель открытого [доступа](https://myapps.microsoft.com) в поддерживаемом браузере и вход в систему в качестве пользователя в Azure AD.

2. Выберите приложение с поддержкой паролей с поддержкой SSO в панели доступа.

3. Когда вы попросите, выберите **Установить сейчас**.

4. Вы будете направлены на ссылку на загрузку на основе вашего браузера. Выберите **Добавить** для установки расширения браузера.

5. Если вам подается запрос, выберите **Включить** или **разрешить**.

6. После установки перезапустите браузер.

7.  Вопийте в панели доступа и посмотрите, сможете ли вы запустить приложения с поддержкой паролей sSO.

Вы также можете непосредственно скачать расширения для Chrome и Firefox через эти ссылки:

-   [Расширение панели доступа Chrome](https://chrome.google.com/webstore/detail/access-panel-extension/ggjhpefgjjfobnfoldnjipclpcfbgbhl)

-   [Расширение панели доступа Firefox](https://addons.mozilla.org/firefox/addon/access-panel-extension/)

## <a name="set-up-a-group-policy-for-internet-explorer"></a>Настройка групповой политики для Internet Explorer

Можно настроить групповую политику, которая позволяет удаленно устанавливать расширение панели доступа для Internet Explorer на машинах пользователей.

Вот предпосылки:

-   Необходимо настроить [службы домена Active Directory,](https://msdn.microsoft.com/library/aa362244%28v=vs.85%29.aspx) а также соединить компьютеры пользователей с вашим доменом.

-   У вас есть разрешение на отославшие сяре "Настройки отсечения" на отодевать объект групповой политики (GPO). По умолчанию такое разрешение имеют члены следующих групп безопасности: «Администраторы домена», «Администраторы предприятия» и «Владельцы-создатели групповой политики». Ознакомьтесь с [дополнительными сведениями](https://technet.microsoft.com/library/cc781991%28v=ws.10%29.aspx).

Чтобы настроить политику группы и развернуть ее для пользователей, см. [Как развернуть расширение панели доступа для Internet Explorer с помощью групповой политики.](https://docs.microsoft.com/azure/active-directory/active-directory-saas-ie-group-policy)

## <a name="configure-password-sso-for-an-azure-ad-gallery-app"></a>Настройка пароля SSO для приложения галереи Azure AD

Чтобы настроить приложение из галереи Azure AD, необходимо сделать следующие вещи:

-   Добавить приложение из галереи Azure AD
-   [Настройка приложения для одного входного знака пароля](#configure-the-app-for-password-sso)
-   [Назначение пользователей приложению](#assign-users-to-the-app)

### <a name="add-the-app-from-the-azure-ad-gallery"></a>Добавить приложение из галереи Azure AD

Выполните следующие действия.

1. Откройте [портал Azure](https://portal.azure.com) и вопийте в качестве глобального администратора или совместного администратора.

2. Выберите **все службы** в верхней части навигационного стекла на левой стороне, чтобы открыть расширение Azure AD.

3. Введите **active-каталог Azure** в поле поиска фильтров, а затем выберите **Active Directory Azure.**

4. Выберите **корпоративные приложения** из навигационного панели Azure AD.

5. Выберите **Добавить** в правом верхнем углу панели **корпоративных приложений.**

6. В **разделе Добавить из раздела галереи** введите имя приложения в поле **имени.**

7. Выберите приложение, которое вы хотите настроить для SSO.

8. *Необязательно:* Перед добавлением приложения можно изменить его название в поле **Name.**

9. Нажмите кнопку **Добавить**, чтобы добавить приложение.

   После короткой задержки вы сможете увидеть панель конфигурации приложения.

### <a name="configure-the-app-for-password-sso"></a>Настройка приложения для пароля SSO

Выполните следующие действия.

1. Откройте [портал Azure](https://portal.azure.com/) и вопийте в качестве глобального администратора или совместного администратора.

2. Выберите **все службы** в верхней части навигационного стекла на левой стороне, чтобы открыть расширение Azure AD.

3. Введите **active-каталог Azure** в поле поиска фильтров, а затем выберите **Active Directory Azure.**

4. Выберите **корпоративные приложения** в навигационном панели Azure AD.

5. Выберите **все приложения** для просмотра списка ваших приложений.

   > [!NOTE]
   > Если вы не видите нужное приложение, используйте элемент управления **фильтром** в верхней части **списка всех приложений.** Установите опцию **Show** на "Все приложения".

6. Выберите приложение, которое вы хотите настроить для SSO.

7. После загрузки приложения выберите **единый входе** в панель на левой стороне приложения.

8. Выберите режим **регистрации на основе пароля.**

9. Назначаем пользователям приложение.

10. Вы также можете предоставить учетные данные для пользователей. (В противном случае пользователям будет предложено ввести учетные данные при запуске приложения.) Для этого выберите строки пользователей. Затем выберите **учетные данные обновления** и введите их имена пользователей и пароли.

### <a name="assign-users-to-the-app"></a>Назначение пользователей приложению

Чтобы назначить пользователей в приложение напрямую, выполните следующие действия:

1. Откройте [портал Azure](https://portal.azure.com/) и вопием в качестве глобального админа.

2. Выберите **все службы** в навигационной боли на левой стороне, чтобы открыть расширение Azure AD.

3. Введите **active-каталог Azure** в поле поиска фильтров, а затем выберите **Active Directory Azure.**

4. Выберите **корпоративные приложения** в навигационном панели Azure AD.

5. Выберите **все приложения** для просмотра списка приложений.

   > [!NOTE]
   > Если вы не видите нужное приложение, используйте элемент управления **фильтром** в верхней части **списка всех приложений.** Установите опцию **Show** на "Все приложения".

6. Из списка выберите приложение, которое вы хотите назначить пользователю.

7. После загрузки приложения выберите **пользователей и группы** из навигационного панели приложения на левой стороне.

8. Выберите **Добавить** в верхней части списка **пользователей и групп,** чтобы открыть панель **добавления назначения.**

9. Выберите **пользователей и группы** в панели **добавления назначения.**

10. В **поле поиска по имени или адресу электронной почты** введите полное имя или адрес электронной почты пользователя, который вы хотите назначить.

11. Нависните над пользователем в списке. Выберите флажок рядом с фотографией или логотипом профиля пользователя, чтобы добавить этого пользователя в **выбранный** список.

12. *Необязательно:* Чтобы добавить другого пользователя, введите другое имя или адрес электронной почты в поле **поиска по имени или адресу электронной почты,** а затем выберите флажок, чтобы добавить этого пользователя в **выбранный** список.

13. Когда вы закончите выбор пользователей, нажмите **Select,** чтобы добавить их в список пользователей и групп, назначенных в приложение.

14. *Необязательно:* Нажмите **Выберите роль** в панели **добавления назначения,** чтобы выбрать роль для назначения выбранным пользователям.

15. Выберите **назначить** приложение выбранным пользователям.

    После короткой задержки пользователи смогут получить доступ к этим приложениям из панели доступа.

## <a name="request-support"></a>Создание запроса 
Если вы получаете сообщение об ошибке при настройке SSO и назначаете пользователей, откройте билет поддержки. Включите как можно больше следующей информации:

-   идентификатор ошибки корреляции;
-   имя участника-пользователя (адрес электронной почты пользователя);
-   идентификатор клиента;
-   тип браузера;
-   Часовой пояс и временные рамки, когда произошла ошибка
-   трассировки Fiddler.

# <a name="sign-in-problems-with-an-azure-ad-gallery-app-configured-for-sso"></a>Проблемы входа в систему с помощью приложения Azure AD коллекции, настроенные для единого входа

Панель доступа представляет собой веб-портал. Он позволяет пользователям, которым Azure Active Directory (Azure AD) рабочих или учебных учетных записей для доступа к облачных приложений, которые у них есть разрешения для. Пользователи, имеющие выпуски Azure AD можно также использовать группами самообслуживания и возможности управления приложениями с помощью панели доступа.

Панель доступа отделена от портала Azure. Пользователям не требуется подписка Azure, чтобы использовать панель доступа.

Чтобы использовать паролю единого входа (SSO) в панели доступа, расширение панели доступа должны устанавливаться в браузере. Расширение автоматически загружается при выборе приложения, настроенного для единого входа по паролю.

## <a name="browser-requirements-for-access-panel"></a>Требования к браузеру для панели доступа

С панелью доступа требуется браузер, поддерживающий JavaScript и CSS включения.

Следующие браузеры с поддержкой Единого входа с паролем:

- Internet Explorer 8, 9, 10 и 11 в Windows 7 или более поздней версии

- Chrome в Windows 7 или более поздней версии или в MacOS X или более поздней версии

- Firefox 26.0 или более поздней версии, в Windows XP SP2 или более поздней версии или в Mac OS X 10.6 или более поздней версии

>[!NOTE]
>Расширение для единого входа с использованием пароля будет доступно для Microsoft Edge в Windows 10 при поддержка расширения обозревателя была добавлена в Microsoft Edge.

## <a name="install-the-access-panel-browser-extension"></a>Установите расширение обозревателя панели доступа

Выполните следующие действия.

1. Откройте [панели доступа](https://myapps.microsoft.com) в поддерживаемом браузере и войдите в систему как пользователь в Azure AD.

2. Выберите это приложение с поддержкой единого входа. пароль панели доступа.

3. Когда появится запрос, выберите **установить сейчас**.

4. Вы будете перенаправлены на ссылку для скачивания в некоторых браузерах. Выберите **добавить** установить расширение браузера.

5. Если появится запрос, выберите **включить** или **Разрешить**.

6. После завершения установки перезапустите браузер.

7.  Войдите в панель доступа и см. в разделе, если можно запустить свои приложения с поддержкой единого входа. пароль.

Можно также напрямую скачать расширения для Chrome и Firefox по ссылкам:

-   [Расширение панели доступа для Chrome](https://chrome.google.com/webstore/detail/access-panel-extension/ggjhpefgjjfobnfoldnjipclpcfbgbhl)

-   [Расширение панели доступа для Firefox](https://addons.mozilla.org/firefox/addon/access-panel-extension/)

## <a name="set-up-a-group-policy-for-internet-explorer"></a>Настройка групповой политики для Internet Explorer

Можно настроить групповую политику, которая позволяет удаленно устанавливать расширение панели доступа для Internet Explorer на компьютерах пользователей.

Ниже приведены предварительные требования.

-   [Доменные службы Active Directory](https://msdn.microsoft.com/library/aa362244%28v=vs.85%29.aspx) необходимо настроить, и компьютеры пользователей должны быть присоединены к домену.

-   Имеется «Изменить параметры» разрешения на изменение объекта групповой политики (GPO). По умолчанию это разрешение имеют члены следующих групп безопасности: "Администраторы домена", "Администраторы предприятия" и "Владельцы-создатели групповой политики". [Узнайте больше](https://technet.microsoft.com/library/cc781991%28v=ws.10%29.aspx).

Чтобы настроить групповую политику и развернуть его для пользователей, см. в разделе [развертывание расширения панели доступа для Internet Explorer с помощью групповой политики](https://docs.microsoft.com/azure/active-directory/active-directory-saas-ie-group-policy).

## <a name="troubleshoot-access-panel-in-internet-explorer"></a>Устранение неполадок панели доступа в Internet Explorer

Чтобы получить доступ к средству диагностики и инструкции по настройке расширения, см. в разделе [Устранение неполадок расширения панели доступа для Internet Explorer](https://docs.microsoft.com/azure/active-directory/active-directory-saas-ie-troubleshooting).

## <a name="configure-password-sso-for-an-azure-ad-gallery-app"></a>Настройка единого входа по паролю для приложения Azure AD коллекции

Чтобы настроить приложение из коллекции Azure AD, необходимо выполнить следующие действия:

-   Добавить приложение из коллекции Azure AD
-   [Настройка приложения для пароля единого входа](#configure-the-app-for-password-sso)
-   [Назначить пользователей для приложения](#assign-users-to-the-app)

### <a name="add-the-app-from-the-azure-ad-gallery"></a>Добавить приложение из коллекции Azure AD

Выполните следующие действия.

1. Откройте [портала Azure](https://portal.azure.com) и войдите в систему качестве глобального администратора или соадминистратора.

2. Выберите **все службы** в верхней части области навигации с левой стороны, чтобы открыть расширения Azure AD.

3. Тип **Azure Active Directory** в поле фильтра поиска, а затем выберите **Azure Active Directory**.

4. Выберите **корпоративные приложения** в области навигации Azure AD.

5. Выберите **добавить** в правом верхнем углу **корпоративные приложения** области.

6. В **добавить из коллекции** введите имя приложения в **введите имя** поле.

7. Выберите приложение, которое вы хотите настроить единый вход.

8. *Необязательно*. Прежде чем добавить приложение, можно изменить ее имя в **имя** поле.

9. Нажмите кнопку **добавить** Чтобы добавить приложение.

   После небольшой задержки вы сможете область настройки приложения.

### <a name="configure-the-app-for-password-sso"></a>Настройка приложения для единого входа по паролю

Выполните следующие действия.

1. Откройте [портала Azure](https://portal.azure.com/) и войдите в систему качестве глобального администратора или соадминистратора.

2. Выберите **все службы** в верхней части области навигации с левой стороны, чтобы открыть расширения Azure AD.

3. Тип **Azure Active Directory** в поле фильтра поиска, а затем выберите **Azure Active Directory**.

4. Выберите **корпоративные приложения** в области навигации Azure AD.

5. Выберите **все приложения** для просмотра списка приложений.

   > [!NOTE]
   > Если вы не видите приложение, для которого необходимо, используйте **фильтра** элемента управления в верхней части **список всех приложений**. Задайте **Показать** возможность «Все приложения».

6. Выберите приложение, которое вы хотите настроить единый вход.

7. После загрузки приложения выберите **единого входа** в области в левой части приложения.

8. Выберите **пароль входа на основе** режим.

9. Назначьте пользователей для приложения.

10. Можно также предоставить учетные данные для пользователей. (В противном случае пользователям будет предложено ввести учетные данные при запуске приложения.) Чтобы сделать это, выберите строки пользователей. Затем выберите **обновить учетные данные** и введите их имена пользователей и пароли.

### <a name="assign-users-to-the-app"></a>Назначение пользователей приложению

Чтобы назначить пользователей непосредственно в приложение, выполните следующие действия:

1. Откройте [портала Azure](https://portal.azure.com/) и войдите в систему качестве глобального администратора.

2. Выберите **все службы** в трудностей навигации с левой стороны, чтобы открыть расширения Azure AD.

3. Тип **Azure Active Directory** в поле фильтра поиска, а затем выберите **Azure Active Directory**.

4. Выберите **корпоративные приложения** в области навигации Azure AD.

5. Выберите **все приложения** для просмотра списка приложений.

   > [!NOTE]
   > Если вы не видите приложение, для которого необходимо, используйте **фильтра** элемента управления в верхней части **список всех приложений**. Задайте **Показать** возможность «Все приложения».

6. В списке выберите приложение, которое вы хотите назначить пользователю.

7. После загрузки приложения выберите **пользователей и групп** в области навигации приложения слева.

8. Выберите **добавить** в верхней части **пользователей и групп** списка, чтобы открыть **Добавление назначения** области.

9. Выберите **пользователей и групп** в **Добавление назначения** области.

10. В **поиск по имени или адресу электронной почты** введите полное имя или адрес электронной почты пользователя, который вы хотите назначить.

11. Наведите указатель мыши пользователя в списке. Установите флажок рядом с пользователя фотографией или логотипом профиля добавьте этого пользователя к **выбранные** списка.

12. *Необязательно*. Чтобы добавить другого пользователя, введите другое имя или адрес электронной почты адрес в **поиск по имени или адресу электронной почты** поле, затем установите флажок, чтобы добавить его к **выбранные** списка.

13. После завершения выбора пользователей, нажмите кнопку **выберите** Чтобы добавить их в список пользователей и групп, назначенных для приложения.

14. *Необязательно*. Нажмите кнопку **Выбор роли** в **Добавление назначения** панели, чтобы выбрать роль для назначения пользователям, которые вы выбрали.

15. Выберите **назначить** Чтобы назначить приложение выбранным пользователям.

    После небольшой задержки пользователи смогут получить доступ к этим приложениям из панели доступа.

## <a name="request-support"></a>Создание запроса 
Если появится сообщение об ошибке при настройке единого входа и назначить пользователей, откройте запрос в службу поддержки. Включить как можно большую часть максимально следующие сведения:

-   идентификатор ошибки корреляции;
-   имя участника-пользователя (адрес электронной почты пользователя);
-   идентификатор клиента;
-   тип браузера;
-   Часовой пояс и время или время кадра при возникновении ошибки
-   трассировки Fiddler.

# <a name="grant-tenant-wide-admin-consent-to-an-application"></a>Грант ан-нанимателю согласия на заявку

Узнайте, как упростить взаимодействие с пользователем, предоставив запрос на приложение общеаниманию. Эта статья дает различные способы достижения этой цели. Эти методы применяются для всех пользователей в клиенте Azure Active Directory (AAD).

Дополнительные сведения о предоставлении согласия для приложений см. в статье [Платформа предоставления согласия Azure Active Directory](../develop/consent-framework.md).

## <a name="prerequisites"></a>Предварительные требования

Предоставление согласия администратора администратора по всему ан-наэтом требует, чтобы вы встыли в качестве [глобального администратора,](../users-groups-roles/directory-assign-admin-roles.md#global-administrator--company-administrator) [администратора приложений](../users-groups-roles/directory-assign-admin-roles.md#application-administrator)или [администратора облачных приложений.](../users-groups-roles/directory-assign-admin-roles.md#cloud-application-administrator)

> [!IMPORTANT]
> Когда приложению будет предоставлено согласие админора по всему ан-начуза, все пользователи смогут войти в приложение, если оно не настроено для запроса назначения пользователя. Чтобы ограничить, какие пользователи могут войти в приложение, необходимо назначение пользователя, а затем назначить пользователей или групп в приложение. Подробнее см. статью [Методы назначения пользователей и групп](methods-for-assigning-users-and-groups.md).

> [!WARNING]
> Предоставление согласия админового приложения для всего клиента предоставит приложению и издателю приложения доступ к данным организации. Тщательно просмотрите разрешения, запрашиваемые приложением, прежде чем дать согласие.

## <a name="grant-admin-consent-from-the-azure-portal"></a>Грант админ согласие от портала Azure

### <a name="grant-admin-consent-in-enterprise-apps"></a>Грант админ-согласие в приложениях Enterprise

Вы можете предоставить согласие админа с помощью *приложений Enterprise,* если приложение уже подготовлено в вашем арендаторе. Например, приложение может быть подготовлено в вашем арендаторе, если хотя бы один пользователь уже дал согласие на приложение. Для получения дополнительной информации узнайте, [как и почему приложения добавляются в Active Directory Azure.](../develop/active-directory-how-applications-are-added.md)

Чтобы предоставить ан-начн- аминому для приложения, указанного в **приложениях Enterprise:**

1. Вопиюсь на [портал Azure](https://portal.azure.com) в качестве [глобального администратора,](../users-groups-roles/directory-assign-admin-roles.md#global-administrator--company-administrator) [администратора приложений](../users-groups-roles/directory-assign-admin-roles.md#application-administrator)или [администратора облачных приложений.](../users-groups-roles/directory-assign-admin-roles.md#cloud-application-administrator)
2. Выберите **Активный каталог Azure,** а затем **корпоративные приложения.**
3. Выберите приложение, на которое вы хотите предоставить согласие админа ширины клиента.
4. Выберите **Разрешения,** а затем нажмите **Грант админ согласия**.
5. Тщательно просмотрите разрешения, необходимые приложению.
6. Если вы согласны с разрешениями, которые требуется для приложения, дайте согласие. Если нет, нажмите **Отменить** или закрыть окно.

### <a name="grant-admin-consent-in-app-registrations"></a>Грант админ согласия в App регистрации

Для приложений, разработанных организацией или зарегистрированных непосредственно в вашем арендаторе Azure AD, вы также можете предоставить согласие админа на общеклиентов от **регистрации приложений** на портале Azure.

Чтобы предоставить консультанту общеанимание согласия от **регистрации приложений:**

1. Вопиюсь на [портал Azure](https://portal.azure.com) в качестве [глобального администратора,](../users-groups-roles/directory-assign-admin-roles.md#global-administrator--company-administrator) [администратора приложений](../users-groups-roles/directory-assign-admin-roles.md#application-administrator)или [администратора облачных приложений.](../users-groups-roles/directory-assign-admin-roles.md#cloud-application-administrator)
2. Выберите **Активный каталог Azure,** а затем **регистрацию приложений.**
3. Выберите приложение, на которое вы хотите предоставить согласие админа ширины клиента.
4. Выберите **разрешения API,** а затем нажмите **согласие на участие в гранте.**
5. Тщательно просмотрите разрешения, необходимые приложению.
6. Если вы согласны с разрешениями, которые требуется для приложения, дайте согласие. Если нет, нажмите **Отменить** или закрыть окно.

## <a name="construct-the-url-for-granting-tenant-wide-admin-consent"></a>Построить URL-адрес для предоставления согласия админа широкого клиента

При предоставлении запроса на запрос в масштабах всего клиента с использованием любого описанного выше метода с портала Azure открывается окно, чтобы заставить его дать согласие на запрос об аминте. Если вы знаете идентификатор клиента (также известный как идентификатор приложения) приложения, можно создать тот же URL для предоставления согласия админа для всего клиента.

URL-адрес согласия всего клиента выполняет следующий формат:

    https://login.microsoftonline.com/{tenant-id}/adminconsent?client_id={client-id}

где:

* `{client-id}`это идентификатор клиента приложения (также известный как идентификатор приложения).
* `{tenant-id}`является идентификатором арендатора вашей организации или любым проверенным доменным именем.

Как всегда, внимательно изучите разрешения, которые запрашивает приложение, прежде чем дать согласие.

# <a name="managing-user-account-provisioning-for-enterprise-apps-in-the-azure-portal"></a>Управление подготовкой учетных записей пользователей для корпоративных приложений на портале Azure

В этой статье описывается использование [портала Azure](https://portal.azure.com) для управления автоматической подготовкой и удалением учетных записей пользователей в приложениях, которые поддерживают эту функциональность. Дополнительные сведения об автоматической подготовке учетных записей пользователей см. в статье [Автоматическая подготовка пользователей и ее отзыв для приложений SaaS в Azure Active Directory](user-provisioning.md).

## <a name="finding-your-apps-in-the-portal"></a>Поиск приложений на портале

На портале Azure Active Directory позволяет просматривать и администрировать все приложения, настроенные для единого входа в каталоге. Корпоративные приложения — это приложения, развертываемые и используемые в вашей организации. Выполните следующие действия для просмотра и управления корпоративными приложениями.

1. Откройте [портал Azure Active Directory](https://aad.portal.azure.com).
1. Выберите **корпоративные приложения** в левой области. Отображается список всех настроенных приложений, включая приложения, которые были добавлены из коллекции.
1. Выберите любое приложение для загрузки ресурсов области, где можно просматривать отчеты и управление параметрами приложения.
1. Выберите **Подготовка** для управления параметрами для выбранного приложения подготовки учетной записи пользователя.

   ![Подготовка экрана для управления параметрами подготовки учетной записи пользователя](./media/configure-automatic-user-provisioning-portal/enterprise-apps-provisioning.png)

## <a name="provisioning-modes"></a>Режимы подготовки

**Подготовка** панели начинается с **режим** меню, в котором показаны режимы подготовки, поддерживаемые приложением enterprise позволяет настраивать их. Доступны следующие параметры.

* **Автоматическое** -этот параметр отображается, если Azure AD поддерживает автоматическую на основе API подготовку или отмену подготовки учетных записей пользователей для этого приложения. Выберите этот режим, чтобы отобразить интерфейс, который помогает администраторам:

  * Настройка Azure AD для подключения к Интерфейсу управления пользователями приложения
  * Создание сопоставления учетных записей и рабочие процессы, которые определяют направление данных учетных записей пользователей между Azure AD и приложения
  * Управление службой подготовки Azure AD

* **Вручную** -этот параметр отображается, если Azure AD не поддерживает автоматическую подготовку учетных записей пользователей для этого приложения. В этом случае учетной записи, сохраняются в приложение должно осуществляться с использованием внешнего процесса, на основе пользователя возможностей управления и подготовки, предоставляемых соответствующим приложением (который может включать JIT-Подготовка SAML).

## <a name="configuring-automatic-user-account-provisioning"></a>Настройка автоматической подготовки учетных записей пользователей

Выберите **автоматического** позволяет задать параметры для учетных данных администратора, сопоставления, запуска и остановки и синхронизации.

### <a name="admin-credentials"></a>Учетные данные администратора

Разверните **учетные данные администратора** вводить учетные данные, необходимые для Azure AD для подключения к Интерфейсу управления пользователями приложения. Требуемые входные данные зависят от приложения. Дополнительные сведения о типах учетных данных и требованиях для конкретных приложений см. в [руководстве по конфигурации для конкретного приложения](user-provisioning.md).

Выберите **проверить подключение** для проверки работы учетных данных за счет Azure AD, попытка подключения к приложению для подготовки приложения, с помощью указанных учетных данных.

### <a name="mappings"></a>Сопоставления

Разверните **сопоставления** просматривать и изменять пользовательские атрибуты, которые проходят между Azure AD и целевом приложении при подготовке или обновлении учетных записей пользователей.

Есть набор предварительно настроенных сопоставлений между объектами пользователей Azure AD и каждого приложения SaaS. Некоторые приложения управляют другими типами объектов, такими как группы или контакты. Выберите сопоставление в таблице, чтобы открыть редактор сопоставлений справа, где можно просматривать и настраивать их.

![Показан экран сопоставления атрибутов](./media/configure-automatic-user-provisioning-portal/enterprise-apps-provisioning-mapping.png)

Поддерживаемые настройки

* Включение и отключение сопоставлений для определенных объектов, например между объектом пользователя Azure AD и приложения SaaS.
* Изменение атрибутов, поступающих из объекта пользователя Azure AD к объекту пользователя приложения. Дополнительные сведения о сопоставлении атрибутов см. в разделе [Основные сведения о типах сопоставления атрибутов](customize-application-attributes.md#understanding-attribute-mapping-types).
* Фильтрация действий подготовки, Azure AD выполняется по целевому приложению. Вместо того чтобы полностью синхронизировать объекты Azure AD, можно ограничить действия запуска.

  Например, выбрать только **обновления** и Azure AD только обновит существующего пользователя, учетные записи в приложении, но не создавать новые. Выбрать только **создать** и Azure создает новые учетные записи пользователей, но не обновит имеющиеся. Эта функция позволяет администраторам создавать различные сопоставления для создания учетной записи и рабочие процессы обновления.

* Добавление нового сопоставления атрибута. Выберите **добавить новый сопоставление** в нижней части **сопоставление атрибутов** области. Заполните **изменение атрибута** форму и выберите **ОК** для добавления в список новое сопоставление.

### <a name="settings"></a>Параметры

Можно запускать и останавливать подготовки службы для выбранного приложения в Azure AD **параметры** область **Подготовка** экрана. Также можно очищать кэш подготовки и перезапустите службу.

Если подготовка включается для приложения впервые, то включите службу, переместив переключатель **Состояние подготовки** в положение **Вкл**. Это изменение вызовет службу подготовки Azure AD для выполнения начальной синхронизации. Он считывает пользователи, назначенные в **пользователей и групп** раздел, запрашивается целевое приложение для них, а затем запускает действия по подготовке, определенные в Azure AD **сопоставления** раздел. В ходе этого процесса служба подготовки сохраняет кэшированные данные об управляемых учетных записях, управляемых им, чтобы неуправляемые учетные записи в целевых приложениях, которые планировалось для назначения, не будут затронуты операций отмены подготовки. После первоначальной синхронизации служба обеспечения автоматически синхронизирует пользователя и групповые объекты с десятиминутным интервалом.

Изменение **состояние подготовки** для **Off** приостановить службу подготовки. В этом состоянии Azure не создать, обновить или удалить все объекты пользователя или группы в приложении. Верните состояние **на** и служба принимает места остановки.

Выберите **очистить текущее состояние и перезапустить синхронизацию** флажок и выберите **Сохранить** для:

* Остановка службы подготовки
* Дамп кэшированных данных об управляемых учетных записях, управление Azure AD
* Перезапустите службы и запустить исходную синхронизацию еще раз

Этот параметр позволяет администраторам начать процесс подготовки развертывания снова и снова.

### <a name="synchronization-details"></a>Сведения о синхронизации

Дополнительные сведения о работе службы подготовки, включая первый и последний раз выполняли службы подготовки для приложения и сколько объектов пользователей и групп он управляет.

Отобразится ссылка для **просмотра отчетов о подготовке**, который предоставляет журнал всех пользователей и групп созданных, обновленных и удаленных между Azure AD и целевом приложении. Также предоставляется ссылка на **отчета об ошибках Провизионирования**, который предоставляет более подробные сообщения об ошибках для объектов пользователей и групп, которые не удалось прочитать, создать, обновить или удалить.

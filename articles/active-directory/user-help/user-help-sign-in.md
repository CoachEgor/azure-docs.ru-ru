# <a name="sign-in-using-two-step-verification-or-security-info"></a>Вход с помощью двухфакторной проверки подлинности или сведений для защиты

После настройки двухэтапной информации о проверке или безопасности вы сможете войти в свою учетную запись с помощью указанного метода проверки.

> [!Note]
> Если вы все еще используете двухэтапный опыт проверки, вам нужно настроить методы проверки, следуя инструкциям в [настройке моей учетной записи для двухэтапной](multi-factor-authentication-end-user-first-time.md) статьи проверки.
>
> Если администратор включил информацию о безопасности, необходимо настроить методы проверки, используя эти пошаговые статьи:<ul><li>[Настройка использования приложения проверки подлинности в сведениях для защиты (предварительная версия)](security-info-setup-auth-app.md)</li><li>[Настройка использования обмена текстовыми сообщениями в сведениях для защиты (предварительная версия)](security-info-setup-text-msg.md)</li><li>[Настройка использования телефонного звонка в сведениях для защиты (предварительная версия)](security-info-setup-phone-number.md)</li><li>[Настройка информации о безопасности для использования ключа безопасности](security-info-setup-security-key.md)</li></ul>

## <a name="sign-in-using-an-authenticator-app-notification-on-your-mobile-device"></a>Вход с помощью уведомления от приложения проверки подлинности на мобильном устройстве

1. Войдите в учетную запись, используя имя пользователя и пароль.

2. В уведомлении, которое отправилось на мобильное устройство, выберите **Утвердить**.

## <a name="sign-in-using-an-authenticator-app-code-on-your-mobile-device"></a>Вход с помощью кода в приложении проверки подлинности на мобильном устройстве

1. Войдите в учетную запись, используя имя пользователя и пароль.

2. Откройте приложение проверки подлинности и введите случайным образом сформированный код для своей учетной записи в поле **Введите код**.

## <a name="sign-in-using-your-phone-number"></a>Вход с использованием номера телефона

1. Войдите в учетную запись, используя имя пользователя и пароль.

2. Ответьте на вызов, а затем следуйте инструкциям.

## <a name="sign-in-using-a-text-message"></a>Вход с использованием текстового сообщения

1. Войдите в учетную запись, используя имя пользователя и пароль.

2. Откройте сообщение и введите код из текста в поле **Введите код**.

## <a name="sign-in-using-a-security-key-at-the-lock-screen"></a>Вопиюте в использовании ключа безопасности на экране блокировки

1. После регистрации ключа безопасности выберите изображение ключа безопасности на экране блокировки Windows 10.

2. Вставьте ключ безопасности в USB-порт устройства и вопийте в Windows с помощью PIN-кода ключа безопасности.

    ![Входе ключа безопасности на экране блокировки Windows 10](./media/security-info/security-info-windows-10-lock-screen-security-key.png)

## <a name="sign-in-using-a-security-key-and-the-microsoft-edge-browser"></a>Вопиюте в использовании ключа безопасности и браузера Microsoft Edge

1. После регистрации ключа безопасности откройте браузер Microsoft Edge.

2. При вхотворетесь в систему, вставьте ключ безопасности в USB-порт устройства и вопийте в Windows с помощью PIN-кода ключа безопасности.

    ![Входе в ключ безопасности с помощью браузера Microsoft Edge](./media/security-info/security-info-edge-security-key.png)

    >[!NOTE]
    >Для получения информации о входе в приложение Microsoft Authenticator смотрите статью, [вопийте в своих учетных записях с помощью приложения Microsoft Authenticator.](user-help-auth-app-sign-in.md)

## <a name="sign-in-using-another-verification-method"></a>Выполнение входа с использованием другого способа проверки

Если по какой-либо причине вы не можете использовать основной метод регистрации, можно использовать другой ранее настроенный метод проверки.

1. Войдите в учетную запись в обычном режиме, а затем выберите **Знак по-другому** на странице **проверки на двухэтапной странице проверки.**

    ![Изменение метода проверки при входе](media/security-info/two-factor-auth-signin-another-way.png)

    >[!Note]
    >Если вы не видите **Знак по-другому,** это означает, что вы не создали никаких других методов проверки и что вам придется связаться с администратором для подписания в свой аккаунт. После того, как администратор поможет вам войти в систему, убедитесь, что вы добавляете дополнительные методы проверки. Дополнительные сведения о добавлении методов проверки см. в статье [Управление параметрами двухфакторной проверки подлинности](multi-factor-authentication-end-user-manage-settings.md).
    >
    >Если вы видите **Знак в другой** ссылке, но все еще не видите других методов проверки, вам придется обратиться к администратору для входной связи в свой аккаунт.

2. Выберите альтернативный метод проверки подлинности и продолжайте процесс двухфакторной проверки подлинности.

3. При необходимости можно обновить свои методы проверки, как только вы вернетесь в свою учетную запись. Дополнительные сведения о добавлении или изменении методов см. в статье [Управление параметрами двухфакторной проверки подлинности](multi-factor-authentication-end-user-manage-settings.md).

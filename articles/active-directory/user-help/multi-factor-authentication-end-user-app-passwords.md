# <a name="manage-app-passwords-for-two-step-verification"></a>Управление паролями приложения для двухфакторной проверки подлинности

>[!Important]
>Администратор может запретить использование паролей приложений. Если вы не видите параметр **Пароли приложений**, значит он недоступен для вашей организации.

При использовании паролей приложений необходимо помнить следующее:

- Пароли приложений создаются автоматически и должны быть созданы и введены один раз для каждого приложения.

- Максимальное количество паролей на пользователя — 40. Если вы попытаетесь создать пароль после достижения этого предела, вам будет предложено удалить один из существующих паролей, чтобы создать новый.

    >[!Note]
    >Все клиенты Office 2013 (включая Outlook) поддерживают новые протоколы идентификации и двухфакторную проверку подлинности. Это означает, что после включения двухфакторной проверки подлинности отпадает потребность в использовании паролей приложений для клиентов Office 2013. Дополнительные сведения см. в статье [Современная проверка подлинности в клиентских приложениях Office 2013 и Office 2016](https://support.office.com/article/how-modern-authentication-works-for-office-2013-and-office-2016-client-apps-e4c45989-4b1a-462e-a81b-2a13191cf517).

## <a name="create-new-app-passwords"></a>Создание паролей приложений

Во время начальной регистрации двухфакторной проверки вы можете указать один пароль приложения. Если требуется более одного, необходимо создать их самостоятельно. Вы можете создавать пароли приложений из нескольких областей в зависимости от того, как в организации настроена двухфакторная проверка подлинности. Дополнительные сведения о регистрации для использования двухфакторной проверки с рабочей или учебной учетной записью см. в статье [Обзор двухфакторной проверки и рабочей или учебной учетной записи](multi-factor-authentication-end-user-first-time.md) и связанных с ней статей.

### <a name="where-to-create-and-delete-your-app-passwords"></a>Создание и удаление паролей приложений

Вы можете создавать и удалять пароли приложений в зависимости от того, как используется двухфакторная проверка подлинности:

- **Ваша организация использует двухфакторную проверку подлинности и дополнительную страницу проверки безопасности.** Если вы используете рабочую или учебную учетную запись (например, alain@contoso.com) с двухфакторной проверкой в вашей организации, вы можете управлять паролями приложений на [странице дополнительной проверки безопасности](https://account.activedirectory.windowsazure.com/Proofup.aspx). Подробные инструкции см. на [странице Создание и удаление паролей приложений с помощью дополнительной проверки безопасности](#create-and-delete-app-passwords-from-the-additional-security-verification-page) в этой статье.

- **Ваша организация использует двухфакторную проверку подлинности и портал Office 365.** Если вы используете рабочую или учебную учетную запись (например, alain@contoso.com), двухфакторную проверку подлинности и приложения Office 365 в Организации, вы можете управлять паролями приложений на [странице портала Office 365](https://www.office.com). Подробные инструкции см. в статье [Создание и удаление паролей приложений с помощью портала Office 365](#create-and-delete-app-passwords-using-the-office-365-portal) .

- **Вы используете двухфакторную проверку подлинности с персональным учетная запись Майкрософт.** Если вы используете личную учетная запись Майкрософт (например, alain@outlook.com) с двухфакторной проверкой, вы можете управлять паролями приложения на странице " [основы безопасности](https://account.microsoft.com/security/)". Подробные инструкции см. в статье [использование паролей приложений с приложениями, которые не поддерживают двухфакторную проверку подлинности](https://support.microsoft.com/help/12409/microsoft-account-app-passwords-and-two-step-verification).

## <a name="create-and-delete-app-passwords-from-the-additional-security-verification-page"></a>Создание и удаление паролей приложений на странице "дополнительная проверка безопасности"

Вы можете создавать и удалять пароли приложений на странице " **Дополнительная проверка безопасности** " для вашей рабочей или учебной учетной записи.

1. Войдите на страницу " [Дополнительная проверка безопасности](https://account.activedirectory.windowsazure.com/Proofup.aspx)", а затем выберите " **пароли приложений**".

    ![Страница "пароли приложений" с выделенной вкладкой "пароли приложений"](media/multi-factor-authentication-end-user-app-passwords/mfa-app-passwords-page.png)

2. Выберите **создать**, введите имя приложения, которому требуется пароль приложения, а затем нажмите кнопку **Далее**.

    ![Страница создания паролей приложений с именем приложения, которому требуется пароль](media/multi-factor-authentication-end-user-app-passwords/mfa-create-app-password-page.png)

3. Скопируйте пароль на странице **пароля приложения** и нажмите кнопку **Закрыть**.

    ![Страница пароля приложения с паролем для указанного приложения](media/multi-factor-authentication-end-user-app-passwords/mfa-your-app-password-page.png)

4. На странице **пароли приложений** убедитесь, что приложение указано в списке.

     ![Страница "пароли приложений" с новым приложением, отображаемым в списке](media/multi-factor-authentication-end-user-app-passwords/mfa-app-passwords-page-with-new-password.png)  

5. Откройте приложение, для которого вы создали пароль приложения (например, Outlook 2010), а затем вставьте пароль приложения при появлении запроса. Это необходимо сделать только один раз для каждого приложения.

### <a name="to-delete-an-app-password-using-the-app-passwords-page"></a>Удаление пароля приложения с помощью страницы "пароли приложений"

1. На странице **пароли приложений** выберите **Удалить** рядом с паролем приложения, которое нужно удалить.

   ![Удаление пароля приложения](media/multi-factor-authentication-end-user-app-passwords/mfa-app-passwords-page-delete.png)

2. Щелкните **Да**, чтобы подтвердить удаление пароля, а затем выберите **Закрыть**.

    Пароль приложения успешно удален.

## <a name="create-and-delete-app-passwords-using-the-office-365-portal"></a>Создание и удаление паролей приложений с помощью портала Office 365

При использовании двухфакторной проверки подлинности для рабочей или учебной учетной записи и приложений Office 365 вы можете создавать и удалять пароли приложений на портале Office 365.

### <a name="to-create-app-passwords-using-the-office-365-portal"></a>Создание паролей приложений с помощью портала Office 365

1. Войдите в Office 365, а затем перейдите на [страницу Моя учетная запись](https://portal.office.com), выберите **Безопасность & конфиденциальность**, а затем разверните узел **Дополнительная проверка безопасности**.

    ![Развернутая область дополнительной проверки безопасности на портале Office](media/multi-factor-authentication-end-user-app-passwords/mfa-app-passwords-o365-my-account-page.png)

2. Выберите текст с текстом, **Создайте пароли приложений и управляйте ими** , чтобы открыть страницу **пароли приложений** .

    ![Страница "пароли приложений" с выделенной вкладкой "пароли приложений"](media/multi-factor-authentication-end-user-app-passwords/mfa-app-passwords-page.png)

3. Выберите **создать**, введите имя приложения, которому требуется пароль приложения, а затем нажмите кнопку **Далее**.

    ![Страница создания паролей приложений с именем приложения, которому требуется пароль](media/multi-factor-authentication-end-user-app-passwords/mfa-create-app-password-page.png)

4. Скопируйте пароль на странице **пароля приложения** и нажмите кнопку **Закрыть**.

    ![Страница пароля приложения с паролем для указанного приложения](media/multi-factor-authentication-end-user-app-passwords/mfa-your-app-password-page.png)

5. На странице **пароли приложений** убедитесь, что приложение указано в списке.

     ![Страница "пароли приложений" с новым приложением, отображаемым в списке](media/multi-factor-authentication-end-user-app-passwords/mfa-app-passwords-page-with-new-password.png)  

6. Откройте приложение, для которого вы создали пароль приложения (например, Outlook 2010), а затем вставьте пароль приложения при появлении запроса. Это необходимо сделать только один раз для каждого приложения.

### <a name="to-delete-app-passwords-using-the-app-passwords-page"></a>Удаление паролей приложений с помощью страницы "пароли приложений"

1. На странице **пароли приложений** выберите **Удалить** рядом с паролем приложения, которое нужно удалить.

   ![Удаление пароля приложения](media/multi-factor-authentication-end-user-app-passwords/mfa-app-passwords-page-delete.png)

2. В поле подтверждения выберите **Да** , а затем нажмите кнопку **Закрыть**.

    Пароль приложения успешно удален.

## <a name="if-your-app-passwords-arent-working-properly"></a>Действия в случае неправильного функционирования паролей приложений

Убедитесь, что пароль был правильно введен. Если вы уверены, что правильно ввели свой пароль, попробуйте снова войти в систему и создать новый пароль приложения. Если ни один из этих вариантов не устранит проблему, обратитесь в службу технической поддержки Организации, чтобы они могли удалять существующие пароли приложений, позволяя создавать новые.

# <a name="manage-app-passwords-for-two-step-verification"></a>Управление паролями приложения для двухфакторной проверки подлинности

>[!Important]
>Администратор может запретить использование паролей приложений. Если вы не видите параметр **Пароли приложений**, значит он недоступен для вашей организации.

При использовании паролей приложений необходимо помнить следующее:

- Пароли приложений автоматически генерируются и должны создаваться и вводиться один раз в приложении.

- Максимальное количество паролей на пользователя — 40. Если вы попытаетесь создать пароль после достижения этого предела, вам будет предложено удалить один из существующих паролей, чтобы создать новый.

    >[!Note]
    >Все клиенты Office 2013 (включая Outlook) поддерживают новые протоколы идентификации и двухфакторную проверку подлинности. Это означает, что после включения двухфакторной проверки подлинности отпадает потребность в использовании паролей приложений для клиентов Office 2013. Дополнительные сведения см. в статье [Современная проверка подлинности в клиентских приложениях Office 2013 и Office 2016](https://support.office.com/article/how-modern-authentication-works-for-office-2013-and-office-2016-client-apps-e4c45989-4b1a-462e-a81b-2a13191cf517).

## <a name="create-new-app-passwords"></a>Создание паролей приложений

Во время первоначального двухфакторного процесса регистрации проверки вам предоставляется единый пароль приложения. Если вам требуется несколько, вам придется создать их самостоятельно. Вы можете создавать пароли приложений из нескольких областей, в зависимости от того, как в вашей организации настраивается двухфакторная проверка. Для получения дополнительной информации о регистрации для использования двухфакторной проверки с вашей работы или школы счета, см [Обзор для двухфакторной проверки и вашей работы или школы счета](multi-factor-authentication-end-user-first-time.md) и связанных с ними статей.

### <a name="where-to-create-and-delete-your-app-passwords"></a>Создание и удаление паролей приложений

Вы можете создавать и удалять пароли приложений, основываясь на том, как вы используете двухфакторную проверку:

- **Ваша организация использует двухфакторную проверку и страницу дополнительной проверки безопасности.** Если вы используете свою рабочую или alain@contoso.comшкольную учетную запись (например, с двухфакторной проверкой в организации, вы можете управлять паролями приложения со [страницы дополнительной проверки безопасности.](https://account.activedirectory.windowsazure.com/Proofup.aspx) Для получения подробных инструкций смотрите [Создать и удалить пароли приложений, используя страницу дополнительной проверки безопасности](#create-and-delete-app-passwords-from-the-additional-security-verification-page) в этой статье.

- **Ваша организация использует двухфакторную проверку и портал Office 365.** Если вы используете свою рабочую или alain@contoso.comшкольную учетную запись (например, двухфакторную проверку и приложения Office 365 в вашей организации, вы можете управлять паролями приложения со [страницы портала Office 365.](https://www.office.com) Для получения подробных инструкций смотрите [Создать и удалить пароли приложений с помощью портала Office 365](#create-and-delete-app-passwords-using-the-office-365-portal) в этой статье.

- **Вы используете двухфакторную проверку с помощью личной учетной записи Майкрософт.** Если вы используете личную учетную alain@outlook.comзапись Майкрософт (например, с двухфакторной проверкой, вы можете управлять паролями приложения со [страницы основ безопасности.](https://account.microsoft.com/security/) Для получения подробных инструкций [см.](https://support.microsoft.com/help/12409/microsoft-account-app-passwords-and-two-step-verification)

## <a name="create-and-delete-app-passwords-from-the-additional-security-verification-page"></a>Создание и удаление паролей приложений со страницы дополнительной проверки безопасности

Вы можете создавать и удалять пароли приложений со страницы **дополнительной проверки безопасности** для вашей учетной записи или школы.

1. Вопийте на [странице дополнительной проверки безопасности,](https://account.activedirectory.windowsazure.com/Proofup.aspx)а затем выберите **пароли App.**

    ![Страница паролей приложений с выделенной вкладкой паролей приложения](media/multi-factor-authentication-end-user-app-passwords/mfa-app-passwords-page.png)

2. Выберите **Создать,** введите имя приложения, которое требует пароль приложения, а затем выберите **Следующий.**

    ![Создание страницы паролей приложения с именем приложения, которое нуждается в пароле](media/multi-factor-authentication-end-user-app-passwords/mfa-create-app-password-page.png)

3. Копируйте пароль со страницы **пароля Вашего приложения,** а затем выберите **Close**.

    ![Страница паролей приложения с паролем для указанного приложения](media/multi-factor-authentication-end-user-app-passwords/mfa-your-app-password-page.png)

4. Со **страницы паролей приложения** убедитесь, что ваше приложение включено в список.

     ![Страница паролей приложений, с новым приложением, показанным в списке](media/multi-factor-authentication-end-user-app-passwords/mfa-app-passwords-page-with-new-password.png)  

5. Откройте приложение, для создания пароля приложения (например, Outlook 2010), а затем вставьте пароль приложения, когда его попросят. Вы должны сделать это только один раз в приложении.

### <a name="to-delete-an-app-password-using-the-app-passwords-page"></a>Удаление пароля приложения с помощью страницы паролей Приложения

1. На странице **паролей приложения** выберите **Удалить** рядом с паролем приложения, который вы хотите удалить.

   ![Удаление пароля приложения](media/multi-factor-authentication-end-user-app-passwords/mfa-app-passwords-page-delete.png)

2. Щелкните **Да**, чтобы подтвердить удаление пароля, а затем выберите **Закрыть**.

    Пароль приложения успешно удален.

## <a name="create-and-delete-app-passwords-using-the-office-365-portal"></a>Создание и удаление паролей приложений с помощью портала Office 365

При использовании двухфакторной проверки подлинности для рабочей или учебной учетной записи и приложений Office 365 вы можете создавать и удалять пароли приложений на портале Office 365.

### <a name="to-create-app-passwords-using-the-office-365-portal"></a>Создание паролей приложений с помощью портала Office 365

1. Войдите в Office 365, а затем перейдите на [страницу Мой аккаунт](https://portal.office.com), выберите **безопасности & конфиденциальности**, а затем расширить **Дополнительную проверку безопасности**.

    ![Развернутая область дополнительной проверки безопасности на портале Office](media/multi-factor-authentication-end-user-app-passwords/mfa-app-passwords-o365-my-account-page.png)

2. Выберите текст, в который сказано, **создайте и управляйте паролями приложений,** чтобы открыть страницу **паролей приложения.**

    ![Страница паролей приложений с выделенной вкладкой паролей приложения](media/multi-factor-authentication-end-user-app-passwords/mfa-app-passwords-page.png)

3. Выберите **Создать,** введите имя приложения, которое требует пароль приложения, а затем выберите **Следующий.**

    ![Создание страницы паролей приложения с именем приложения, которое нуждается в пароле](media/multi-factor-authentication-end-user-app-passwords/mfa-create-app-password-page.png)

4. Копируйте пароль со страницы **пароля Вашего приложения,** а затем выберите **Close**.

    ![Страница паролей приложения с паролем для указанного приложения](media/multi-factor-authentication-end-user-app-passwords/mfa-your-app-password-page.png)

5. Со **страницы паролей приложения** убедитесь, что ваше приложение включено в список.

     ![Страница паролей приложений, с новым приложением, показанным в списке](media/multi-factor-authentication-end-user-app-passwords/mfa-app-passwords-page-with-new-password.png)  

6. Откройте приложение, для создания пароля приложения (например, Outlook 2010), а затем вставьте пароль приложения, когда его попросят. Вы должны сделать это только один раз в приложении.

### <a name="to-delete-app-passwords-using-the-app-passwords-page"></a>Удаление паролей приложений с помощью страницы паролей Приложения

1. На странице **паролей приложения** выберите **Удалить** рядом с паролем приложения, который вы хотите удалить.

   ![Удаление пароля приложения](media/multi-factor-authentication-end-user-app-passwords/mfa-app-passwords-page-delete.png)

2. Выберите **Да** в коробке подтверждения, а затем выберите **Закрыть**.

    Пароль приложения успешно удален.

## <a name="if-your-app-passwords-arent-working-properly"></a>Действия в случае неправильного функционирования паролей приложений

Убедитесь, что пароль был правильно введен. Если вы уверены, что правильно ввели свой пароль, попробуйте снова войти в систему и создать новый пароль приложения. Если ни один из этих вариантов не исправит вашу проблему, обратитесь в службу поддержки организации, чтобы они могли удалить существующие пароли приложений, что позволит вам создавать совершенно новые.

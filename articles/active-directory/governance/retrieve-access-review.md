# <a name="retrieve-access-review-results-for-groups-or-applications-in-azure-ad-access-reviews"></a>Получить проверку доступа проверок доступа результаты для групп или приложений в Azure AD

Администраторы могут использовать Azure Active Directory (Azure AD), чтобы [создать проверку доступа](create-access-review.md) для участников группы или пользователей, назначенных для приложения.  Пользователь, являющийся в **глобального администратора**, **администратор пользователей**, **администратор безопасности** или **читатель безопасности** роли можно также Ознакомьтесь с результатами проверки доступа.  Чтобы назначить пользователей для одной из этих ролей, администратор привилегированных ролей можно использовать Azure AD PIM, чтобы разрешить пользователю активировать роль или глобальный администратор может окончательно [назначение пользователю роли](../fundamentals/active-directory-users-assign-role-azure-portal.md).

[!INCLUDE [GDPR-related guidance](../../../includes/gdpr-intro-sentence.md)]

## <a name="locating-an-access-review"></a>Поиск проверки доступа

Если вы знаете, какая программа содержит проверку доступа, перейдите к странице проверок доступа, а затем на вкладку **Программы** и выберите программу, содержащую элемент управления проверки доступа.  Затем щелкните **Элементы управления** и выберите элемент управления проверки доступа. Если в программе существует множество элементов управления, можно отфильтровать их по определенному типу и отсортировать по состоянию. Также можно найти имя элемента управления проверки доступа или отображаемое имя владельца, который его создал. 

Если вы не знаете, какая программа содержит проверку доступа, перейдите на страницу проверки доступа и выберите **Элементы управления**.  Вы можете отфильтровать для отображения элементов определенного типа, сортировать по их состоянию, а также искать по имени элемента управления проверки доступа или отображаемого имени владельца, который его создал. 

## <a name="retrieving-the-results-for-a-one-time-access-review"></a>Получение результатов для одноразовой проверки доступа

Если тип повторения проверки одноразовый, то ход выполнения активной проверки доступа и результаты завершенной проверки доступа можно получить в разделе **Результаты**.  Вы можете ввести отображаемое имя или имя участника-пользователя, чей доступ проверяется, чтобы просмотреть сведения для пользователя.  Чтобы проверить все результаты выполненной проверки доступа, нажмите кнопку **Скачать**.

## <a name="retrieving-the-results-for-multiple-instances-of-a-recurring-access-review"></a>Получение результатов для нескольких экземпляров повторяющейся проверки доступа

Чтобы просмотреть ход выполнения активной проверки доступа, которая повторяется, щелкните раздел **Результаты**.  Вы можете ввести отображаемое имя или имя участника-пользователя, чей доступ проверяется.

Чтобы просмотреть результаты завершенного экземпляра проверки доступа, который повторяется, выберите **Журнал проверок**, а затем выберите конкретный экземпляр из списка завершенных экземпляров проверки доступа на основе даты начала и окончания экземпляра.   Результаты этого экземпляра можно получить в разделе **Результаты**.  Вы можете ввести отображаемое имя или имя участника-пользователя, чей доступ проверяется, чтобы просмотреть сведения для пользователя.  Чтобы проверить все результаты выполненного экземпляра повторяющейся проверки доступа, нажмите кнопку **Скачать**.


## <a name="removing-users-from-an-access-review"></a>Удаление пользователей из проверки доступа

По умолчанию удаленный пользователь будет оставаться удаленным в Azure AD в течение 30 дней, на протяжении которых его при необходимости может восстановить администратор.  По истечении 30 дней пользователь будет окончательно удален.  Кроме того, с помощью портала Azure Active Directory глобальный администратор может явно [навсегда удалить недавно удаленного пользователя](../fundamentals/active-directory-users-restore.md) до истечения этого периода времени.  Если пользователь удален навсегда, впоследствии данные об этом пользователе будут удалены из активных проверок доступа.  Сведения аудита об удаленных пользователях остаются в журнале аудита.

# <a name="view-reports-and-logs-in-azure-ad-entitlement-management"></a>Просмотр отчетов и журналов в управлении правами Azure AD

Отчеты об управлении правами Azure AD и журнал аудита Azure AD предоставляют дополнительные сведения о том, к каким ресурсам пользователи имеют доступ. Как администратор, вы можете просматривать пакеты доступа и назначения ресурсов для журнала запросов пользователя и просматривать журналы запросов для целей аудита или для определения статуса запроса пользователя. В этой статье описывается, как использовать отчеты об управлении правами и журналы аудита Azure AD.

Посмотрите следующее видео, чтобы узнать, как просматривать ресурсы, к каким ресурсам пользователи имеют доступ при управлении правами:

>[!VIDEO https://www.youtube.com/embed/omtNJ7ySjS0]

## <a name="view-access-packages-for-a-user"></a>Просмотр пакетов доступа для пользователя

Этот отчет позволяет перечислить все пакеты доступа, которые пользователь может запросить, и пакеты доступа, которые в настоящее время назначены пользователю.

**Роль предпосылки:** Глобальный администратор или администратор пользователя

1. Щелкните **Azure Active Directory** и выберите **Управление удостоверениями**.

1. В левом меню нажмите **Отчеты**.

1. Нажмите **пакеты доступа для пользователя.**

1. Нажмите **Выберите пользователей,** чтобы открыть панель пользователей Select.

1. Найти пользователя в списке, а затем нажмите **Выберите**.

    Вкладка **Can Request** отображает список пакетов доступа, которые пользователь может запросить. Этот список определяется [политиками запроса, определенными](entitlement-management-access-package-request-policy.md#for-users-in-your-directory) для пакетов доступа. 

    ![Пакеты доступа для пользователя](./media/entitlement-management-reports/access-packages-report.png)

1. Если для пакета доступа имеется несколько ролей или политик ресурсов, нажмите на ввод ролей ресурсов или политик, чтобы увидеть детали выбора.

1. Нажмите **на вкладку Назначенные,** чтобы увидеть список пакетов доступа, назначенных пользователю. Когда пакет доступа назначается пользователю, это означает, что пользователь имеет доступ ко всем ролям ресурса в пакете доступа.

## <a name="view-resource-assignments-for-a-user"></a>Просмотр заданий ресурсов для пользователя

Этот отчет позволяет перечислить ресурсы, в настоящее время назначенные пользователю в управлении правами. Обратите внимание, что данный отчет предназначен для ресурсов, управляемых с помощью управления правами. Пользователь может иметь доступ к другим ресурсам в вашем каталоге за пределами управления правами.

**Роль предпосылки:** Глобальный администратор или администратор пользователя

1. Щелкните **Azure Active Directory** и выберите **Управление удостоверениями**.

1. В левом меню нажмите **Отчеты**.

1. Нажмите **на запросы ресурсов для пользователя.**

1. Нажмите **Выберите пользователей,** чтобы открыть панель пользователей Select.

1. Найти пользователя в списке, а затем нажмите **Выберите**.

    Отображается список ресурсов, назначенных пользователю в настоящее время. В списке также показан пакет доступа и политика, из которой они получили роль ресурса, а также дата начала и окончания доступа.
    
    Если пользователь получил доступ к одному и тому же ресурсу в двух или более пакетах, вы можете нажать на стрелку, чтобы увидеть каждый пакет и политику.

    ![Назначения ресурсов для пользователя](./media/entitlement-management-reports/resource-assignments-report.png)

## <a name="determine-the-status-of-a-users-request"></a>Определение состояния запроса пользователя

Чтобы получить дополнительные сведения о том, как пользователь запрашивал и получил доступ к пакету доступа, можно использовать журнал аудита Azure AD. В частности, можно использовать записи журналов в `EntitlementManagement` категориях, `UserManagement` чтобы получить дополнительные сведения о этапах обработки для каждого запроса.  

1. Нажмите **Active Directory Azure,** а затем нажмите **журналы аудита**.

1. В верхней части, изменить `EntitlementManagement` `UserManagement` **категорию** либо или , в зависимости от аудиторской записи вы ищете.  

1. Щелкните **Применить**.

1. Чтобы загрузить журналы, нажмите **Скачать**.

Когда Azure AD получает новый запрос, он записывает `EntitlementManagement` запись аудита, `User requests access package assignment`в которой **категория** и **активность** обычно .  В случае прямого назначения, созданного на портале Azure, поле `Administrator directly assigns user to access package` **активности** аудиторской записи является, и пользователь, выполняющий назначение, идентифицируется **ActorUserPrincipalName**.

Azure AD будет записывать дополнительные аудиторские отчеты во время выполнения запроса, в том числе:

| Категория | Действие | Request status (Состояние запроса) |
| :---- | :------------ | :------------ |
| `EntitlementManagement` | `Auto approve access package assignment request` | Запрос не требует одобрения |
| `UserManagement` | `Create request approval` | Запрос требует одобрения |
| `UserManagement` | `Add approver to request approval` | Запрос требует одобрения |
| `EntitlementManagement` | `Approve access package assignment request` | Запрос одобрен |
| `EntitlementManagement` | `Ready to fulfill access package assignment request` |Запрос одобрен или не требует одобрения |

При присвоении пользователю доступа Azure AD записывает запись аудита для `EntitlementManagement` категории с **activity** `Fulfill access package assignment`.  Пользователь, получивший доступ, идентифицируется полем **ActorUserPrincipalName.**

Если доступ не был назначен, то Azure AD `EntitlementManagement` записывает `Deny access package assignment request`запись аудита для категории с `Access package assignment request timed out (no approver action taken)` **Activity,** если запрос был отклонен одобрителя, или , если запрос приурочен, прежде чем одобритель может утвердить.

По истечении срока действия угона пакета доступа пользователя, отмены пользователем или удалены администратором, azure AD записывает аудиторскую запись для `EntitlementManagement` категории с **Activity** of `Remove access package assignment`.

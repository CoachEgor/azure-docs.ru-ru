# <a name="troubleshoot-azure-ad-entitlement-management-preview"></a>Устранение неполадок управления прав Azure AD (Предварительная версия)

> [!IMPORTANT]
> Управление правами Azure Active Directory (Azure AD) в настоящее время находится в общедоступной предварительной версии.
> Эта предварительная версия предоставляется без соглашения об уровне обслуживания и не рекомендована для использования рабочей среде. Некоторые функции могут не поддерживаться или их возможности могут быть ограничены.
> Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

В этой статье описаны некоторые элементы, необходимо проверить, поможет вам справиться с управление правами Azure Active Directory (Azure AD).

## <a name="checklist-for-entitlement-management-administration"></a>Контрольный список для администрирования прав управления

* Если вы являетесь глобальным администратором можно получить сообщение об отказе при настройке управление правами доступа, убедитесь, что каталог имеет [лицензии Azure AD Premium P2 (или EMS E5)](entitlement-management-overview.md#prerequisites).  
* Если вы являетесь членом группы создателя каталога можно получить отказ в сообщение при создании или просмотре пакетов доступ, необходимо создать каталог, прежде чем создать свой первый пакет доступа.

## <a name="checklist-for-adding-a-resource"></a>Контрольный список для добавления ресурса

* При наличии пользователей, которым уже назначены к ресурсу, который вы хотите управлять с помощью пакета доступа, убедитесь, что пользователи имеют разрешение для пакета доступ с помощью соответствующую политику. Например может потребоваться включить группу доступа, в пакет уже содержит пользователей в группе. Если таких пользователей в группы требуется постоянный доступ, они должны иметь соответствующую политику для пакетов доступ таким образом, чтобы они не потеряли доступ к группе. Можно назначить доступ пакета, либо задав пользователям запрашивать доступ пакет, содержащий этот ресурс или путем непосредственного назначения их к пакету доступ. Дополнительные сведения см. в разделе [изменение и управление ею существующий пакет доступа](entitlement-management-access-package-edit.md).

## <a name="checklist-for-request-issues"></a>Контрольный список для запроса проблемы

* Когда пользователь хочет запросить доступ к пакету доступ, убедитесь, что они используют **ссылка портале мой доступ** для пакета доступ. Дополнительные сведения см. в разделе [ссылка портале мой доступ копирования](entitlement-management-access-package-edit.md#copy-my-access-portal-link).

* При входе пользователя на портал мой доступ для запроса пакета доступа, убедитесь, что они проверку подлинности с использованием учетной записи организации. Учетная запись организации может быть либо учетную запись в каталог ресурсов, или в каталоге, который включен в одну из политик доступа пакета. Если учетная запись пользователя не учетной записи организации или каталог не включается в политике, пользователь не увидит доступ пакет. Дополнительные сведения см. в разделе [запросить доступ к пакету доступ](entitlement-management-request-access.md).

* Если пользователь не сможет войти каталог ресурсов, они не смогут запрашивать доступ на портале мой доступ. Прежде чем пользователь может запросить доступ, необходимо удалить блок входа в систему из профиля пользователя. Чтобы снять блокировку входа в систему, на портале Azure, щелкните **Azure Active Directory**, нажмите кнопку **пользователей**, выберите пользователя и нажмите кнопку **профиль**. Изменить **параметры** раздел и измените **заблокировать вход** для **нет**. Дополнительные сведения см. в разделе [добавить или обновить сведения о профиле пользователя, с помощью Azure Active Directory](../fundamentals/active-directory-users-profile-azure-portal.md)

* На портале мой доступ, если пользователь является запрашивающая сторона и утверждающего, он не увидит их запроса для пакета доступ на **утверждения** страницы. Такое поведение является преднамеренным - пользователь не может утверждать свои собственные запроса. Убедитесь, что пакет доступ, они запрашивают у других утверждающих лиц, настроить в политике. Дополнительные сведения см. в разделе [изменить существующую политику](entitlement-management-access-package-edit.md#edit-an-existing-policy).

* Если нового внешнего пользователя, которые не подписаны ранее в каталоге, получает пакет доступа, включая сайт SharePoint Online, их доступ пакет будет отображаться в конфигурации не полностью, до инициализации учетной записи в SharePoint Online.

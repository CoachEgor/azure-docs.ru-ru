# <a name="view-and-manage-requests-for-an-access-package-in-azure-ad-entitlement-management-preview"></a>Просмотр и управление запросами для пакета Access в управлении назначениями Azure AD (Предварительная версия)

> [!IMPORTANT]
> Управление правами Azure Active Directory (Azure AD) сейчас предоставляется в общедоступной предварительной версии.
> Эта предварительная версия предоставляется без соглашения об уровне обслуживания и не рекомендована для использования рабочей среде. Некоторые функции могут не поддерживаться или их возможности могут быть ограничены.
> Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

В разделе "Управление назначениями Azure AD" можно увидеть, кто запросил пакеты доступа, их политику и состояние. В этой статье описывается просмотр, повторная обработка и Отмена запросов пакетов доступа.

## <a name="view-requests"></a>Просмотр запросов

**Предварительная роль:** Глобальный администратор, администратор пользователей, владелец каталога или диспетчер пакетов Access

1. На портале Azure щелкните **Azure Active Directory** и выберите **Управление удостоверениями**.

1. В меню слева щелкните **доступ к пакетам** и откройте пакет Access.

1. Щелкните **запросы**.

1. Щелкните конкретный запрос, чтобы просмотреть дополнительные сведения.

    ![Список запросов для пакета Access](./media/entitlement-management-access-package-requests/requests-list.png)

## <a name="view-a-requests-delivery-errors"></a>Просмотр ошибок доставки запроса

**Предварительная роль:** Глобальный администратор, администратор пользователей, владелец каталога или диспетчер пакетов Access

1. На портале Azure щелкните **Azure Active Directory** и выберите **Управление удостоверениями**.

1. В меню слева щелкните **доступ к пакетам** и откройте пакет Access.

1. Щелкните **запросы**.

1. Выберите запрос, который требуется просмотреть.

    Если запрос содержит ошибки доставки, состояние запроса будет **не доставлено или** **частично доставлено**.

    При наличии ошибок доставки в области сведений о запросе отображается число ошибок доставки.

1. Щелкните число, чтобы просмотреть все ошибки доставки запроса.

## <a name="reprocess-a-request"></a>Повторная обработка запроса

Если запрос встречает ошибку, можно повторно обработать запрос, чтобы повторить попытку. Можно выполнить только повторную обработку запроса, состояние **доставки которого не выполнено** или **частично доставлено** , а Дата завершения — менее одной недели.

**Предварительная роль:** Глобальный администратор, администратор пользователей, владелец каталога или диспетчер пакетов Access

1. На портале Azure щелкните **Azure Active Directory** и выберите **Управление удостоверениями**.

1. В меню слева щелкните **доступ к пакетам** и откройте пакет Access.

1. Щелкните **запросы**.

1. Щелкните запрос, который требуется повторно обработать.

1. В области сведения о запросе щелкните **повторно обработать запрос**.

    ![Повторная обработка невыполненного запроса](./media/entitlement-management-access-package-requests/reprocess-request.png)

## <a name="cancel-a-pending-request"></a>Отмена ожидающего запроса

Можно отменять только ожидающий запрос, который еще не был доставлен или с которым произошел сбой доставки.

**Предварительная роль:** Глобальный администратор, администратор пользователей, владелец каталога или диспетчер пакетов Access

1. На портале Azure щелкните **Azure Active Directory** и выберите **Управление удостоверениями**.

1. В меню слева щелкните **доступ к пакетам** и откройте пакет Access.

1. Щелкните **запросы**.

1. Щелкните запрос, который нужно отменить.

1. В области сведения о запросе щелкните **отменить запрос**.

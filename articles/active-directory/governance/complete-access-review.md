# <a name="complete-an-access-review-of-groups-or-applications-in-azure-ad-access-reviews"></a>Завершение проверки доступа групп или приложений в Azure AD проверок доступа

Будучи администратором вы [Создание проверки доступа групп или приложений](create-access-review.md) и рецензенты [выполнить проверку доступа](perform-access-review.md). В этой статье описывается, как просмотреть результаты проверки доступа и применить результаты.

[!INCLUDE [GDPR-related guidance](../../../includes/gdpr-intro-sentence.md)]

## <a name="prerequisites"></a>Технические условия

- Azure AD Premium P2
- Глобальный администратор, администратор пользователей, администратора безопасности или читателя безопасности

Дополнительные сведения см. в разделе [какие пользователи должны иметь лицензии?](access-reviews-overview.md#which-users-must-have-licenses).

## <a name="view-an-access-review"></a>Просмотр проверки доступа

Можно отслеживать ход выполнения проверяющими их проверок.

1. Войдите на портале Azure и откройте [страница управления удостоверениями](https://portal.azure.com/#blade/Microsoft_AAD_ERM/DashboardBlade/).

1. В меню слева щелкните **проверок доступа**.

1. В списке выберите проверку доступа.

    Чтобы просмотреть серию доступа обзоры, перейдите на проверку доступа, и вы найдете предстоящих вхождений в запланированные проверки.

    На **Обзор** страницы, вы увидите ход выполнения. Нет прав доступа, изменяются в каталоге до завершения проверки.

    ![Ход выполнения проверки доступа](./media/complete-access-review/overview-progress.png)

1. Если вы хотите остановить проверку доступа до достижения запланированная дата окончания, нажмите кнопку **остановить** кнопки.

    Если остановка проверки, рецензенты больше не смогут оставлять замечания. Невозможно перезапустить проверку после ее остановки.

1. Если вы не заинтересованы в проверку доступа, его можно удалить, щелкнув **удалить** кнопки.

## <a name="apply-the-changes"></a>Применение изменений

Если **автоматически применять результаты к ресурсу** был включен и с учетом выбранных в **после заполнения параметров**, auto-применить будет выполняться после окончания проверки или когда вручную остановите проверку.

Если **автоматически применять результаты к ресурсу** не была включена для проверки, нажмите кнопку **применить** вручную применить изменения. Если пользователя отказ в доступе во время проверки, если щелкнуть **применить**, Azure AD удаляет их членство или назначение в приложении.

![Применить изменения проверки доступа](./media/complete-access-review/apply-changes.png)

Состояние проверки изменится с **завершено** через несколько промежуточных этапов, такие как **применение** и, наконец состояние **результат применен**. Все пользователи, отклоненные по результатам проверки, будут лишены членства в группах или назначений в приложениях в течение нескольких минут.

Автоматическое или запущенное вручную (действием **Применить**) применение изменений не оказывает влияния на группы, созданные в локальном каталоге, и на динамические группы. Если вы хотите изменить группу, которая создана в локальной среде, скачайте результаты и примените изменения к представлению группы в этом каталоге.

## <a name="retrieve-the-results"></a>Получение результатов

Чтобы просмотреть результаты для одноразовых доступе, щелкните **результаты** страницы. Чтобы просмотреть только что доступа пользователя, в поле поиска, введите отображаемое имя или имя участника-пользователя пользователя права доступа которых прошел проверку.

![Получить результаты для проверки доступа](./media/complete-access-review/retrieve-results.png)

Чтобы просмотреть ход выполнения проверки доступа active, повторяется, щелкните **результаты** страницы.

Чтобы просмотреть результаты завершенного экземпляра проверку доступа, который повторяется, щелкните **Просмотр журнала**, а затем выберите конкретный экземпляр из списка экземпляров проверок доступа завершенных, на основе начала данного экземпляра и Дата окончания. Результаты данного экземпляра можно получить из **результаты** страницы.

Чтобы получить все результаты проверки доступа, щелкните **загрузить** кнопки. Полученный CSV-файл можно просмотреть в Excel или в других программах, которые поддерживают CSV-файлы в кодировке UTF-8.

## <a name="remove-users-from-an-access-review"></a>Удалять пользователей из проверки доступа

 По умолчанию удаленный пользователь будет оставаться удаленным в Azure AD в течение 30 дней, на протяжении которых его при необходимости может восстановить администратор.  По истечении 30 дней пользователь будет окончательно удален.  Кроме того, с помощью портала Azure Active Directory глобальный администратор может явно [навсегда удалить недавно удаленного пользователя](../fundamentals/active-directory-users-restore.md) до истечения этого периода времени.  Если пользователь удален навсегда, впоследствии данные об этом пользователе будут удалены из активных проверок доступа.  Сведения аудита об удаленных пользователях остаются в журнале аудита.

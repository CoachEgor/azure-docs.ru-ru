# <a name="delegate-tasks-in-azure-ad-entitlement-management-preview"></a>Делегирование задач в управление правами Azure AD (Предварительная версия)

> [!IMPORTANT]
> Управление правами Azure Active Directory (Azure AD) сейчас предоставляется в общедоступной предварительной версии.
> Эта предварительная версия предоставляется без соглашения об уровне обслуживания и не рекомендована для использования рабочей среде. Некоторые функции могут не поддерживаться или их возможности могут быть ограничены.
> Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

По умолчанию глобальные Администраторы и Администраторы пользователей можно создать и контролировать все аспекты управление правами Azure AD. Тем не менее пользователи в эти роли могут не знать всех сценариев, где необходимы пакеты. Обычно это пользователи в отделах умеющие тех, кому требуется совместная работа.

Вместо предоставления неограниченный разрешений пользователям без прав администратора, можно предоставить пользователям минимальные разрешения, необходимые для выполнения их работы и избежать создания конфликтующих или ненадлежащие права доступа. В этой статье описываются роли, которые можно назначить делегировать различные задачи в управление правами.

## <a name="delegate-example-for-departmental-adoption"></a>Пример делегата для отделов внедрения

Чтобы понять, как может делегировать задачи в управление правами, полезно рассмотрим пример. Предположим, что в вашей организации есть пять следующих пользователей:

| пользователя | Department | Примечания |
| --- | --- | --- |
| Алиса | IT | Глобальный администратор. |
| Владимир | Исследование | Боб также является владельцем группы исследований |
| Carole | Исследование |  |
| Дейв | Маркетинг |  |
| Элиза | Маркетинг | Элиза также является владельцем приложения маркетинг |

Справочные материалы и отдел маркетинга требуется используется управление правами пользователей. Алиса еще не готов для других отделов использовать управление правами. Вот один из способов, что Алиса может делегировать задачи исследования и маркетинга.

1. Алиса создает новую безопасности Azure AD группы для создателей каталог и добавляет Боб, Carol, Дэйв и Элиза как члены этой группы.

1. Алиса использует параметры управления прав для добавить эту группу в роли creators каталога.

1. Создает Carol **Research** каталога, и добавляет Боба как владелец этого каталога. Боб добавляет группу, справочные материалы, в которую он занимается в каталог в качестве ресурса, так, что он может использоваться в пакете доступ для совместной работы для исследований.

1. Дейв создает **маркетинга** каталога, и добавляет Элиза как владелец этого каталога. Элиза добавляет маркетингового приложения, которые она настроила в каталог в качестве ресурса, чтобы он может использоваться в пакете доступа для участников маркетинговых совместной работы.

Теперь исследований и маркетинговых отделы могут использовать управление правами. Боб, Carol, Дэйв, Элиза создавать и управлять пакетами доступ в свои соответствующие каталоги.

## <a name="entitlement-management-roles"></a>Назначение ролей управления

Управление правами имеет следующие роли, относящиеся к управление правами.

| Role | Описание |
| --- | --- |
| Создатель каталога | Создание каталогов и управление ими. Обычно ИТ-администратор не глобальный администратор или владелец ресурса для коллекции ресурсов. Лица, которое автоматически создается каталог становится первый каталог владелец каталога, а также можно добавить владельцев дополнительных каталога. |
| Владелец каталога | Изменение и управление ею существующих каталогов. Обычно ИТ-администратор или владельцы ресурса либо пользователь обозначил владельца каталога. |
| Диспетчер пакетов доступ | Изменение и управление ею всех существующих пакетов доступ в каталоге. |

Кроме того, назначенного утверждающего лица и запрашивающий пакета доступа также имеют права, несмотря на то, что они не ролей.
 
* Утверждающий: Разрешение с помощью политики, чтобы утверждать или отклонять запросы для доступа к пакетам, хотя они не могут изменять определения пакетов доступ.
* Запрашивающая сторона: Разрешено политикой доступа пакета запрашивать доступ пакет.

## <a name="add-a-catalog-creator"></a>Добавление создатель каталога

Если вы хотите делегировать создание каталога, добавлении пользователей в роль "Автор" каталога.  Вы можете добавить отдельных пользователей или для удобства можно добавить группы, члены которой могут затем для создания каталогов. Выполните следующие действия, чтобы назначить пользователя в роль "Автор" каталога.

**Требуемая роль:** глобальный администратор или администратор пользователей.

1. На портале Azure щелкните **Azure Active Directory** и выберите **Управление удостоверениями**.

1. В меню слева в **управление правами** щелкните **параметры**.

1. Нажмите кнопку **Изменить**.

1. В **делегировать управление правами** щелкните **добавить каталог creators** выбрать пользователей или группы, которые будут членами этой роли права управления.

1. Нажмите кнопку **Выбрать**.

1. Нажмите кнопку **Сохранить**.

## <a name="add-a-catalog-owner-or-an-access-package-manager"></a>Добавление владельца каталога или диспетчером пакетов с доступом

Если вы хотите делегировать управление каталога или доступа пакетов в каталоге, можно назначить пользователям владельца каталога или доступа пакета диспетчера ролей. Кто создается каталог становится первый владельца каталога. Выполните следующие действия, чтобы назначить пользователя для владельца каталога или роль диспетчера пакета доступа.

**Требуемая роль:** Администратор или владелец каталога

1. На портале Azure щелкните **Azure Active Directory** и выберите **Управление удостоверениями**.

1. В меню слева щелкните **каталоги** , а затем откройте каталог, который вы хотите добавить администраторам.

1. В меню слева щелкните **роли и Администраторы**.

1. Нажмите кнопку **добавить владельцев** или **добавить диспетчеры пакетов доступ** выбрать элементы для этих ролей.

1. Нажмите кнопку **выберите** Чтобы добавить эти элементы.

# <a name="delegate-tasks-in-azure-ad-entitlement-management-preview"></a>Делегирование задач в управлении назначениями Azure AD (Предварительная версия)

> [!IMPORTANT]
> Управление правами Azure Active Directory (Azure AD) сейчас предоставляется в общедоступной предварительной версии.
> Эта предварительная версия предоставляется без соглашения об уровне обслуживания и не рекомендована для использования рабочей среде. Некоторые функции могут не поддерживаться или их возможности могут быть ограничены.
> Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

По умолчанию глобальные администраторы и администраторы пользователей могут создавать и администрировать все аспекты управления назначением Azure AD. Однако пользователи этих ролей могут не учитывать все сценарии, в которых требуются пакеты доступа. Обычно это пользователи внутри отделов, которые узнают, кому нужно сотрудничать. 

Вместо того чтобы предоставлять неограниченные разрешения для пользователей, не являющихся администраторами, можно предоставить пользователям минимальные разрешения, необходимые для выполнения их работы, и избежать создания конфликтующих или нежелательных прав доступа. В этой статье описываются роли, которые можно назначить для делегирования различных задач в управлении назначением. 

## <a name="delegate-example-for-departmental-adoption"></a>Пример делегата для внедрения в отдел

Чтобы понять, как можно делегировать задачи в управлении назначением, полезно рассмотреть пример. 

Предположим, что у вашей организации пять пользователей:

| пользователя | Отдел | Примечания |
| --- | --- | --- |
| Алиса | IT | Глобальный администратор |
| Владимир | Исследование | Боб также является владельцем группы исследования |
| кароле | Исследование |  |
| Дейв | Маркетинг |  |
| Элиза | Маркетинг | Элиза также является владельцем маркетингового приложения |

Отделы исследования и маркетинга хотят использовать управление назначением для своих пользователей. Алиса еще не готова к использованию управления назначением другими отделами. Вот один из способов, которым Алиса может делегировать задачи в отделы исследования и маркетинга.

1. Алиса создает новую группу безопасности Azure AD для создателей каталога и добавляет в нее в качестве членов группы Bob, Дейв и Элиза.

1. Алиса использует параметры управления назначением для добавления этой группы к роли создателей каталога.

1. В ходе создания каталога **исследования** и добавления Боба в качестве совладельца этого каталога. Боб добавляет справочную группу, которой он владеет, в каталог как ресурс, чтобы его можно было использовать в пакете Access для совместной работы с исследованиями.

1. Дейв создает **маркетинговый** каталог и добавляет Элиза в качестве совладельца этого каталога. Элиза добавляет приложение маркетинга, которое она владеет, в каталог как ресурс, чтобы его можно было использовать в пакете Access для совместной работы в маркетинговых целях.

Теперь отделы исследования и маркетинга могут использовать управление назначением. Боб, Дейв и Элиза могут создавать пакеты доступа и управлять ими в соответствующих каталогах.

## <a name="entitlement-management-roles"></a>Роли управления назначениями

Управление назначением имеет следующие роли, относящиеся к управлению назначением.

| Role | Описание |
| --- | --- |
| Создатель каталога | Создание каталогов и управление ими. Обычно это ИТ-администратор, не являющийся глобальным администратором, или владелец ресурса для коллекции ресурсов. Пользователь, который создает каталог, автоматически станет первым владельцем каталога каталога и может добавить дополнительных владельцев каталога. Создатель каталога не может управлять каталогами, которыми они не владеют, и не может добавлять ресурсы, не принадлежащие каталогу. Если создателю каталога необходимо управлять другим каталогом или добавить ресурсы, которыми они не владеют, они могут запросить совладельца этого каталога или ресурса. |
| Владелец каталога | Изменение существующих каталогов и управление ими. Обычно это ИТ-администратор или владельцы ресурсов, или пользователь, которому назначен владелец каталога. |
| Доступ к диспетчеру пакетов | Изменение и управление всеми существующими пакетами доступа в каталоге. |

Кроме того, назначенный утверждающий и запрашивающий пакет Access также имеют права, хотя они не являются ролями.
 
* Утверждающего Авторизация политикой для утверждения или запрета запросов на доступ к пакетам, однако они не могут изменять определения пакетов доступа.
* Запрашивающей стороны Авторизация с помощью политики пакета доступа для запроса этого пакета доступа.

## <a name="add-a-catalog-creator"></a>Добавление создателя каталога

Если вы хотите делегировать создание каталога, вы добавляете пользователей в роль создателя каталога.  Можно добавить отдельных пользователей или добавить группу, члены которой могут создавать каталоги. Выполните следующие действия, чтобы назначить пользователя роли создателя каталога.

**Требуемая роль:** глобальный администратор или администратор пользователей.

1. На портале Azure щелкните **Azure Active Directory** и выберите **Управление удостоверениями**.

1. В меню слева в разделе " **Управление** назначением" щелкните " **Параметры**".

1. Нажмите кнопку **Изменить**.

1. В разделе **Делегирование управления полномочиями** нажмите кнопку **Добавить создатели каталога** , чтобы выбрать пользователей или группы, которые будут членами этой роли управления назначением.

1. Нажмите кнопку **Выбрать**.

1. Нажмите кнопку **Сохранить**.

## <a name="add-a-catalog-owner-or-an-access-package-manager"></a>Добавление владельца каталога или диспетчера пакетов Access

Для делегирования управления каталогом или доступом к пакетам в каталоге необходимо добавить пользователей в роли владельца каталога или получить доступ к ролям диспетчера пакетов. Кто-то создает каталог, который станет первым владельцем каталога. 

Назначенный владелец каталога или диспетчер пакетов доступа должен быть знаком с проектом. Автор каталога должен создать пакет Access, если он участвует в операциях с днем по дням проекта, и они узнают следующую информацию:
- необходимые ресурсы
- кому потребуется доступ
- кому необходимо утвердить доступ
- время, в течение которого проект будет последним

Создатель каталога должен делегировать задачу руководителю проекта, который будет создавать пакет Access и управлять им, если он не участвует в повседневных операциях проекта. Выполните следующие действия, чтобы назначить пользователю роль "владелец каталога" или "Диспетчер пакетов доступа":

**Требуемая роль:** Глобальный администратор, администратор пользователей или владелец каталога

1. На портале Azure щелкните **Azure Active Directory** и выберите **Управление удостоверениями**.

1. В меню слева щелкните **каталоги** , а затем откройте каталог, в который нужно добавить администраторов.

1. В меню слева щелкните **роли и администраторы**.

1. Щелкните **Добавить владельцев** или **добавить диспетчер пакетов доступа** , чтобы выбрать элементы для этих ролей.

1. Нажмите кнопку **выбрать** , чтобы добавить эти члены.

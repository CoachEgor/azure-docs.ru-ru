# <a name="tutorial-create-your-first-access-package-in-azure-ad-entitlement-management-preview"></a>Руководство по созданию пакета для доступа в управлении правами Azure AD (предварительная версия)

> [!IMPORTANT]
> Управление правами Azure Active Directory (Azure AD) сейчас предоставляется в общедоступной предварительной версии.
> Эта предварительная версия предоставляется без соглашения об уровне обслуживания и не рекомендована для использования рабочей среде. Некоторые функции могут не поддерживаться или их возможности могут быть ограничены.
> Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

Управление доступом ко всем ресурсам, которые требуются сотрудникам, таким как группы, приложения и сайты, является важной функцией для организаций. Вам нужна возможность предоставлять сотрудникам нужный уровень доступа, который требуется им для продуктивной работы, и удалять права доступа, когда они больше не нужны.

В этом руководстве предполагается, что вы работаете ИТ-администратором в банке Woodgrove Bank. Вас нужно создать пакет ресурсов для веб-проекта, который внутренние пользователи смогут запрашивать самостоятельно. Запросы требуют утверждения, а права доступа пользователя истекают через 30 дней. В этом руководстве для ресурсов веб-проекта предполагается только участие в одной группе, но это также может быть коллекция групп, приложений или сайтов SharePoint Online.

![Обзор сценария](./media/entitlement-management-access-package-first/elm-scenario-overview.png)

Из этого руководства вы узнаете, как выполнять следующие задачи:

> [!div class="checklist"]
> * Создавать пакет для доступа с группой в качестве ресурса.
> * Назначать утверждающего.
> * Показывать, как внутренний пользователь может запрашивать пакет для доступа.
> * Утверждать запрос на доступ.

Если у вас нет лицензии Azure AD Premium P2 или Enterprise Mobility + Security E5, создайте бесплатную [пробную версию Enterprise Mobility + Security E5](https://signup.microsoft.com/Signup?OfferId=87dd2714-d452-48a0-a809-d2f58c4f68b7&ali=1).

## <a name="prerequisites"></a>Предварительные требования

Чтобы использовать управление правами Azure AD (предварительная версия), необходимо иметь одну из следующих лицензий:

- Azure AD Premium P2
- лицензия Enterprise Mobility + Security (EMS) E5.

## <a name="step-1-set-up-users-and-group"></a>Шаг 1. Настройка колонки пользователей и групп

Каталог ресурсов имеет один или несколько ресурсов для совместного использования. На этом шаге создайте группу с именем **Группа разработки** в каталоге банка Woodgrove Bank, которая будет целевым ресурсом для управления правами. Можно также настроить внутреннюю запрашивающую сторону.

**Требуемая роль:** глобальный администратор или администратор пользователей.

![Создание пользователей и групп](./media/entitlement-management-access-package-first/elm-users-groups.png)

1. Войдите на [портал Azure](https://portal.azure.com) как глобальный администратор или администратор пользователей.  

1. В области навигации слева щелкните **Azure Active Directory**.

1. Создайте или настройте следующих двух пользователей. Можно использовать указанные или другие имена. **Admin1** может быть пользователем, под именем которого вы вошли сейчас.

    | ИМЯ | Роль каталога | ОПИСАНИЕ |
    | --- | --- | --- |
    | **Admin1** | Глобальный администратор.<br/>-или-<br/>Администратор с ограниченными правами (администратор пользователей) | Администратор и утверждающий |
    | **Requestor1** | Пользователь | Внутренняя запросившая сторона |

    В этом руководстве администратор и утверждающий — это один и тот же пользователь. Но, как правило, в качестве утверждающих можно назначить и нескольких пользователей.

1. Создайте группу безопасности Azure AD с именем **Группа разработки** и типом участия **Назначено**.

    Эта группа будет целевым ресурсом для управления правами. Для начала работы группа должна быть пустой.

## <a name="step-2-create-an-access-package"></a>Шаг 2. Создание пакета для доступа

*Пакет для доступа* — это набор всех ресурсов, которые нужны пользователю для работы над проектом или для выполнения своих задач. Пакеты для доступа добавляются в контейнеры, называемые *каталогами*. На этом шаге создайте пакет с именем **Пакет для доступа к веб-проекту** в каталоге **Общий**.

**Требуемая роль:** глобальный администратор или администратор пользователей.

![Создание пакета для доступа](./media/entitlement-management-access-package-first/elm-access-package.png)

1. На портале Azure в области навигации слева щелкните **Azure Active Directory**.

1. В меню слева щелкните **Управление удостоверениями**.

1. В меню слева щелкните **Пакеты для доступа**.  Если вы видите сообщение **Отказано в доступе**, убедитесь, что для этого каталога используется лицензия Azure AD Premium P2.

1. Щелкните **Создать пакет для доступа**.

    ![Управление правами на портале Azure](./media/entitlement-management-access-package-first/access-packages-list.png)

1. На вкладке **Основные сведения** введите имя **Пакет для доступа к веб-проекту** и описание **Проект для доступа к веб-проекту группы разработки**.

1. В раскрывающемся списке **Каталог** оставьте значение **Общий**.

    ![Создание пакета для доступа на вкладке "Общие сведения"](./media/entitlement-management-access-package-first/basics.png)

1. Щелкните **Далее**, чтобы открыть вкладку **Роли ресурсов**.

    На этой вкладке выберите разрешения для включения в пакет для доступа.

1. Щелкните **Группы**.

1. На панели "Выбор групп" найдите и выберите группу **Группа разработки**, созданную ранее.

    По умолчанию вы увидите группы, входящие и не входящие в каталог **Общий**. Если выбрать группу, не входящую в каталог **Общий**, она добавится в каталог **Общий**.

    ![Создание пакета для доступа на вкладке "Роли ресурсов"](./media/entitlement-management-access-package-first/resource-roles-select-groups.png)

1. Чтобы добавить группу в список, нажмите кнопку **Выбрать**.

1. В раскрывающемся списке **Роль** выберите роль **Участник**.

    ![Создание пакета для доступа на вкладке "Роли ресурсов"](./media/entitlement-management-access-package-first/resource-roles.png)

1. Щелкните **Далее**, чтобы открыть вкладку **Политика**.

1. Задайте для переключателя **Создать первую политику** значение **Позже**.

    В следующем разделе вы создадите эту политику.

    ![Создание пакета для доступа на вкладке "Политика"](./media/entitlement-management-access-package-first/policy.png)

1. Нажмите кнопку **Далее**, чтобы открыть вкладку **Проверка и создание**.

    ![Создание пакета для доступа на вкладке "Проверка и создание"](./media/entitlement-management-access-package-first/review-create.png)

1. Проверьте параметры пакета для доступа и щелкните **Создать**.

    Может появиться сообщение о том, что пакет для доступа не будет виден пользователям, так как каталог еще не включен.

    ![Создание пакета для доступа с сообщением о том, что пакет не отображается](./media/entitlement-management-access-package-first/not-visible.png)

1. Последовательно выберите **ОК**.

    Через несколько секунд вы увидите уведомление, что пакет для доступа успешно создан.

## <a name="step-3-create-a-policy"></a>Шаг 3. Создание политики

*Политика* определяет правила или ограничения доступа к пакету для доступа. На этом шаге вы создадите политику, которая позволяет конкретному пользователю в каталоге ресурсов запросить пакет для доступа. Можно также указать, что запросы должны быть утверждены и назначить утверждающего.

![Создание политики пакета для доступа](./media/entitlement-management-access-package-first/elm-access-package-policy.png)

**Требуемая роль:** глобальный администратор или администратор пользователей.

1. В разделе **Пакет для доступа к веб-проекту** в меню слева щелкните **Политики**.

    ![Список политик пакета для доступа](./media/entitlement-management-access-package-first/policies-list.png)

1. Щелкните **Добавить политику**, чтобы открыть раздел "Создание политики".

1. Введите имя **Политика внутренней запрашивающей стороны** и описание **Разрешить пользователям в этом каталоге запрашивать доступ к ресурсам веб-проекта**.

1. В разделе **Пользователи, которые могут запрашивать доступ** щелкните **Для пользователей в каталоге**.

    ![Создание политики](./media/entitlement-management-access-package-first/policy-create.png)

1. Прокрутите вниз до раздела **Выбор пользователей и групп** и щелкните **Добавить пользователей и группы**.

1. На панели "Выбор пользователей и групп" выберите пользователя **Requestor1**, созданного ранее, а затем щелкните **Выбрать**.

1. В разделе **Запрос** задайте для параметра **Требуется утверждение** значение **Да**.

1. В разделе **Выбор утверждающих** щелкните **Добавить утверждающих**.

1. На панели "Выбор утверждающих" выберите пользователя **Admin1**, созданного ранее, а затем щелкните **Выбрать**.

    В этом руководстве администратор и утверждающий — это один и тот же пользователь, но в качестве утверждающего можно назначить и другого пользователя.

1. В разделе **Окончание срока действия** задайте для параметра **Срок действия пакета для доступа** значение **Количество дней**.

1. Для параметра **Срок доступа истекает через** задайте значение **30** дней.

1. Для параметра **Включить политику** выберите значение **Да**.

    ![Создание настроек политики](./media/entitlement-management-access-package-first/policy-create-settings.png)

1. Щелкните **Создать**, чтобы создать **политику внутренней запросившей стороны**.

1. В меню слева проекта для доступа веб-пакета щелкните **Обзор**.

1. Скопируйте ссылку на портал **Мой доступ**.

    Эта ссылка потребуется вам на следующем шаге.

    ![Обзор пакета для доступа со ссылкой на портал "Мой доступ"](./media/entitlement-management-shared/my-access-portal-link.png)

## <a name="step-4-request-access"></a>Шаг 4. Запрос доступа

На этом шаге вы выполните действия в качестве **внутренней запросившей стороны** и запросите доступ к пакету для доступа. Запросившие стороны отправляют свои запросы с помощью портала "Мой доступ". Портал "Мой доступ" позволяет запросившей стороне отправлять запросы на получение пакетов для доступа и просматривать пакеты, к которым у них уже есть доступ, а также просматривать историю запросов.

**Требуемая роль:** внутренняя запросившая сторона

1. Выйдите с портала Azure.

1. В новом окне браузера перейдите по ссылке на портал "Мой доступ", которую вы скопировали на предыдущем шаге.

1. Войдите на портал "Мой доступ" в качестве **Requestor1**.

    Вы увидите пакет с именем **Пакет для доступа к веб-проекту**.

1. При необходимости в столбце **Описание** щелкните стрелку, чтобы просмотреть сведения о пакете для доступа.

    ![Портал "Мой доступ" с пакетами для доступа](./media/entitlement-management-shared/my-access-access-packages.png)

1. Установите флажок рядом с выбранным пакетом.

1. Щелкните **Запросить доступ**, чтобы открыть панель "Запрос доступа".

1. В поле **Коммерческое обоснование** введите текст **Работа над веб-проектом**.

1. Для переключателя **Запрос на определенный период** задайте значение **Да**.

1. Для параметра **Дата начала** задайте сегодняшнюю дату, а для параметра **Дата окончания** — завтрашнюю.

    ![Портал "Мой доступ" с запросом на доступ](./media/entitlement-management-shared/my-access-request-access.png)

1. Нажмите кнопку **Submit**(Отправить).

1. В меню слева щелкните **История запросов**, чтобы убедиться, что ваш запрос был отправлен.

## <a name="step-5-approve-access-request"></a>Шаг 5. Утверждение запроса на доступ

На этом шаге войдите как **утверждающий** пользователь и утвердите запрос на доступ для внутренней запросившей стороны. Утверждающие заходят на тот же портал "Мой доступ", который используют запросившие стороны для отправки запросов. С помощью портала "Мой доступ" утверждающие могут просмотреть запросы, ожидающие утверждения, и утвердить или отклонить их.

**Требуемая роль:** Утверждающий

1. Выйдите из портала "Мой доступ".

1. Войдите на [портал Мой доступ](https://myaccess.microsoft.com) в качестве **Admin1**.

1. В меню слева щелкните **Утверждения**.

1. На вкладке **Ожидающие** найдите **Requestor1**.

    Если вы не видите запрос от Requestor1, подождите несколько минут и повторите попытку.

1. Щелкните ссылку **Просмотреть**, чтобы открыть панель "Запрос доступа".

1. Нажмите кнопку **Утвердить**.

1. В поле **Причина** введите причину **Утвержденный доступ к веб-проекту**.

    ![Портал "Мой доступ" с запросом доступа](./media/entitlement-management-shared/my-access-approve-request.png)

1. Щелкните **Отправить**, чтобы отправить решение.

    Вы увидите сообщение, что запрос был утвержден.

## <a name="step-6-validate-that-access-has-been-assigned"></a>Шаг 6. Проверка назначения доступа

Теперь, когда вы утвердили запрос на доступ, на этом шаге убедитесь, что **внутренней запросившей стороне** был назначен пакет для доступа и что эта сторона теперь является участником группы с именем **Группа разработки**.

**Требуемая роль:** глобальный администратор или администратор пользователей.

1. Выйдите из портала "Мой доступ".

1. Войдите на [портал Azure](https://portal.azure.com) в качестве **Admin1**.

1. Щелкните **Azure Active Directory** и выберите **Управление удостоверениями**.

1. В меню слева щелкните **Пакеты для доступа**.

1. Найдите и щелкните **Пакет для доступа к веб-проекту**.

1. В меню слева щелкните **Запросы**.

    Вы увидите Requestor1 и политику внутренней запросившей стороны с состоянием **Доставлено**.

1. Выберите запрос, чтобы просмотреть сведения о нем.

    ![Пакет для доступа со сведениями о запросе](./media/entitlement-management-access-package-first/request-details.png)

1. В области навигации слева щелкните **Azure Active Directory**.

1. Щелкните **Группы** и откройте группу **Группа разработки**.

1. Нажмите кнопку **Участники**.

    Вы увидите **Requestor1** в качестве участника.

    ![Участники группы разработки](./media/entitlement-management-access-package-first/group-members.png)

## <a name="step-7-clean-up-resources"></a>Шаг 7. Очистка ресурсов

На этом шаге вы удалите внесенные изменения и пакет с именем **Пакет для доступа к веб-проекту**.

**Требуемая роль:**  глобальный администратор или администратор пользователей.

1. На портале Azure щелкните **Azure Active Directory** и выберите **Управление удостоверениями**.

1. Откройте **пакет для доступа веб-проекта**.

1. Нажмите **Назначения**.

1. Рядом с пунктом **Requestor1** нажмите кнопку с многоточием ( **...** ), а затем щелкните **Удалить доступ**.

    Состояние изменится с "Доставлено" на "Срок действия истек".

1. Щелкните **Политики**.

1. Рядом с пунктом **Политики внутренней запросившей стороны** нажмите кнопку с многоточием ( **...** ), а затем щелкните **Удалить**.

1. Щелкните **Роли ресурсов**.

1. Рядом с пунктом **Группа разработки** нажмите кнопку с многоточием ( **...** ), а затем щелкните **Удалить роль ресурса**.

1. Откройте список пакетов для доступа.

1. Рядом с пунктом **Пакет для доступа веб-проекта** нажмите кнопку с многоточием ( **...** ), а затем щелкните **Удалить**.

1. В Azure Active Directory удалите всех созданных пользователей, а именно **Requestor1** и **Admin1**.

1. Удалите группу **Группа разработки**.

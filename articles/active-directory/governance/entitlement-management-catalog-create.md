# <a name="create-and-manage-a-catalog-of-resources-in-azure-ad-entitlement-management"></a>Создание и управление каталогом ресурсов в управлении правами Azure AD

## <a name="create-a-catalog"></a>Создание каталога

Каталог представляет собой контейнер ресурсов и пакетов доступа. Вы создаете каталог, когда вы хотите группировать связанные ресурсы и пакеты доступа. Тот, кто создает каталог, становится первым владельцем каталога. Владелец каталога может добавить дополнительных владельцев каталогов.

**Роль предпосылки:** Глобальный администратор, администратор пользователей или создатель каталога

1. На портале Azure щелкните **Azure Active Directory** и выберите **Управление удостоверениями**.

1. В левом меню нажмите **Каталоги**.

    ![Каталоги управления правами на портале Azure](./media/entitlement-management-catalog-create/catalogs.png)

1. Нажмите **Новый каталог**.

1. Введите уникальное название каталога и предоставьте описание.

    Пользователи увидят эту информацию в деталях пакета доступа.

1. Если вы хотите, чтобы пакеты доступа в этом каталоге были доступны пользователям, чтобы запросить, как только они будут созданы, набор **Включенный** **в "Да"**

1. Если вы хотите разрешить пользователям в отдельных внешних каталогах, чтобы иметь возможность запрашивать пакеты доступа в этом каталоге, набор **Включено для внешних пользователей** **да**.

    ![Новая панель каталога](./media/entitlement-management-shared/new-catalog.png)

1. Нажмите **Создать** для создания каталога.

### <a name="creating-a-catalog-programmatically"></a>Создание каталога программно

Вы также можете создать каталог с помощью Microsoft Graph.  Пользователь в соответствующей роли с приложением `EntitlementManagement.ReadWrite.All` с делегированным разрешением может вызвать API для [создания accessPackageCatalog.](https://docs.microsoft.com/graph/api/accesspackagecatalog-post?view=graph-rest-beta)

## <a name="add-resources-to-a-catalog"></a>Добавление ресурсов в каталог

Для включения ресурсов в пакет доступа ресурсы должны существовать в каталоге. Типы ресурсов, которые можно добавить, — это группы, приложения и сайты SharePoint Online. Группы могут быть созданными облаком группами Office 365 или созданными облаком группами безопасности Azure AD. Приложения могут быть корпоративными приложениями Azure AD, включая как приложения SaaS, так и собственные приложения, отключаемые Azure AD. Сайты могут быть SharePoint Интернет сайтов или SharePoint Интернет коллекции сайтов.

**Роль предпосылки:** Просмотреть [Обязательные роли для добавления ресурсов в каталог](entitlement-management-delegate.md#required-roles-to-add-resources-to-a-catalog)

1. На портале Azure щелкните **Azure Active Directory** и выберите **Управление удостоверениями**.

1. В левом меню щелкните **Каталоги,** а затем откройте каталог, в который вы хотите добавить ресурсы.

1. В левом меню нажмите **Ресурсы**.

1. Нажмите **Добавить ресурсы**.

1. Нажмите на тип ресурса: **группы и команды,** **приложения**или **сайты SharePoint.**

    Если вы не видите ресурс, который вы хотите добавить или не можете добавить ресурс, убедитесь, что у вас есть необходимая роль каталога Azure AD и роль управления правами. Возможно, вам потребуется, чтобы кто-то с требуемыми ролями добавить ресурс в каталог. Для получения дополнительной информации [см. Обязательные роли для добавления ресурсов в каталог.](entitlement-management-delegate.md#required-roles-to-add-resources-to-a-catalog)

1. Выберите один или несколько ресурсов типа, которые вы хотели бы добавить в каталог.

    ![Добавление ресурсов в каталог](./media/entitlement-management-catalog-create/catalog-add-resources.png)

1. После завершения, нажмите **Добавить**.

    Теперь эти ресурсы могут быть включены в пакеты доступа в каталоге.

### <a name="adding-a-resource-to-a-catalog-programmatically"></a>Добавление ресурса в каталог программно

Вы также можете добавить ресурс в каталог с помощью Microsoft Graph.  Пользователь в соответствующей роли, или каталог и владелец ресурса, `EntitlementManagement.ReadWrite.All` с приложением, которое имеет делегированное разрешение, может вызвать API для [создания accessPackageResourceRequest.](https://docs.microsoft.com/graph/api/accesspackageresourcerequest-post?view=graph-rest-beta)

## <a name="remove-resources-from-a-catalog"></a>Удаление ресурсов из каталога

Вы можете удалить ресурсы из каталога. Ресурс может быть удален из каталога только в том случае, если он не используется ни в одном из пакетов доступа каталога.

**Роль предпосылки:** Просмотреть [Обязательные роли для добавления ресурсов в каталог](entitlement-management-delegate.md#required-roles-to-add-resources-to-a-catalog)

1. На портале Azure щелкните **Azure Active Directory** и выберите **Управление удостоверениями**.

1. В левом меню нажмите **Каталоги,** а затем откройте каталог, из который вы хотите удалить ресурсы.

1. В левом меню нажмите **Ресурсы**.

1. Выберите ресурсы, которые вы хотите удалить.

1. Нажмите **Удалить** (или нажмите эллипсис **(...**) и нажмите **Удалить ресурс).**

## <a name="add-additional-catalog-owners"></a>Добавить дополнительных владельцев каталогов

Пользователь, создававсвойй каталог, становится первым владельцем каталога. Чтобы делегировать управление каталогом, вы добавляете пользователей к роли владельца каталога. Это помогает разделить обязанности по управлению каталогом. 

Выполните следующие действия, чтобы назначить пользователю роль владельца каталога:

**Роль предпосылки:** Глобальный администратор, администратор пользователей или владелец каталога

1. На портале Azure щелкните **Azure Active Directory** и выберите **Управление удостоверениями**.

1. В левом меню щелкните **Каталоги,** а затем откройте каталог, в который вы хотите добавить администраторов.

1. В левом меню щелкните **роли и администраторы**.

    ![Каталоги ролей и администраторов](./media/entitlement-management-shared/catalog-roles-administrators.png)

1. Нажмите **Добавить владельцев,** чтобы выбрать членов для этих ролей.

1. Нажмите **Выберите,** чтобы добавить этих членов.

## <a name="edit-a-catalog"></a>Отодвить каталог

Вы можете отсвазать имя и описание для каталога. Пользователи видят эту информацию в деталях пакета доступа.

**Роль предпосылки:** Глобальный администратор, администратор пользователей или владелец каталога

1. На портале Azure щелкните **Azure Active Directory** и выберите **Управление удостоверениями**.

1. В левом меню щелкните **Каталоги,** а затем откройте каталог, который вы хотите отсеить.

1. На странице **обзор** каталога нажмите **«Отметуйте».**

1. Изнайте название каталога, описание или настройки включенного.

    ![Настройки каталога отсечения](./media/entitlement-management-shared/catalog-edit.png)

1. Нажмите **Сохранить**.

## <a name="delete-a-catalog"></a>Удалить каталог

Вы можете удалить каталог, но только если он не имеет каких-либо пакетов доступа.

**Роль предпосылки:** Глобальный администратор, администратор пользователей или владелец каталога

1. На портале Azure щелкните **Azure Active Directory** и выберите **Управление удостоверениями**.

1. В левом меню щелкните **Каталоги,** а затем откройте каталог, который вы хотите удалить.

1. На **обзоре**каталога нажмите **Удалить**.

1. Щелкните **Да** в появившемся диалоговом окне.

### <a name="deleting-a-catalog-programmatically"></a>Программное составление каталога

Вы также можете удалить каталог с помощью Microsoft Graph.  Пользователь в соответствующей роли с приложением, `EntitlementManagement.ReadWrite.All` которое имеет делегированное разрешение, может позвонить в API для [удаления accessPackageCatalog.](https://docs.microsoft.com/graph/api/accesspackagecatalog-delete?view=graph-rest-beta)

# <a name="view-add-and-remove-assignments-for-an-access-package-in-azure-ad-entitlement-management"></a>Просмотр, добавление и удаление назначений для пакета доступа в управлении правами Azure AD

В управлении правами Azure AD можно увидеть, кому было поручено получить доступ к пакетам, их политике и статусу. Если пакет доступа имеет соответствующую политику, вы также можете непосредственно назначить пользователя пакету доступа. В этой статье описывается, как просматривать, добавлять и удалять назначения для пакетов доступа.

## <a name="prerequisites"></a>Предварительные требования

Чтобы использовать управление правами Azure AD и назначить пользователям доступ к пакетам, необходимо иметь одну из следующих лицензий:

- Azure AD Premium P2
- лицензия Enterprise Mobility + Security (EMS) E5.

## <a name="view-who-has-an-assignment"></a>Просмотр того, у кого есть назначение

**Роль предпосылки:** Глобальный администратор, администратор пользователей, владелец каталога или менеджер пакетов доступа

1. На портале Azure щелкните **Azure Active Directory** и выберите **Управление удостоверениями**.

1. В левом меню щелкните **пакеты Access,** а затем откройте пакет доступа.

1. Нажмите **«Назначения»,** чтобы увидеть список активных заданий.

    ![Список назначений в пакет доступа](./media/entitlement-management-access-package-assignments/assignments-list.png)

1. Нажмите на конкретное задание, чтобы увидеть дополнительные сведения.

1. Чтобы увидеть список назначений, которые не имеют все роли ресурса должным образом подготовлены, нажмите статус фильтра и выберите **Доставка**.

    Дополнительные сведения об ошибках доставки можно увидеть, обнаружив соответствующий запрос пользователя на странице **Запросов.**

1. Чтобы просмотреть просроченные назначения, щелкните статус фильтра и выберите **Expired**.

1. Чтобы загрузить файл CSV фильтрованного списка, нажмите **Скачать**.

### <a name="viewing-assignments-programmatically"></a>Просмотр заданий программно

Вы также можете получить назначения в пакете доступа с помощью Microsoft Graph.  Пользователь в соответствующей роли с приложением `EntitlementManagement.ReadWrite.All` с делегированным разрешением может вызвать API для [списка accessPackageAssignments.](https://docs.microsoft.com/graph/api/accesspackageassignment-list?view=graph-rest-beta)

## <a name="directly-assign-a-user"></a>Непосредственно назначить пользователя

В некоторых случаях может потребоваться непосредственно назначить определенных пользователей пакету доступа, чтобы пользователям не пришлось проходить процесс запроса пакета доступа. Для прямого назначения пользователей пакет доступа должен иметь политику, позволяющую администратору выполнять прямые назначения.

**Роль предпосылки:** Глобальный администратор, администратор пользователей, владелец каталога или менеджер пакетов доступа

1. На портале Azure щелкните **Azure Active Directory** и выберите **Управление удостоверениями**.

1. В левом меню щелкните **пакеты Access,** а затем откройте пакет доступа.

1. В левом меню нажмите **«Назначения».**

1. Нажмите **Новое назначение,** чтобы открыть Добавить пользователя для доступа к пакету.

    ![Назначения - Добавление пользователя к пакету доступа](./media/entitlement-management-access-package-assignments/assignments-add-user.png)

1. Нажмите **Добавить пользователей,** чтобы выбрать пользователей, которые вы хотите назначить этот пакет доступа.

1. В списке **политики Select** выберите политику, по которым будущие запросы и жизненный цикл пользователей будут регулироваться и отслеживаться. Если вы хотите, чтобы выбранные пользователи имели различные параметры политики, можно нажать **«Создать новую политику»,** чтобы добавить новую политику.

1. Установите дату и время, когда вы хотите, чтобы назначение выбранных пользователей началось и закончилось. Если дата окончания не указана, будут использоваться параметры жизненного цикла политики.

1. Дополнительно предоставьте обоснование вашего прямого назначения для ведения записей.

1. Нажмите **Добавить,** чтобы непосредственно назначить выбранных пользователей в пакет доступа.

    Через несколько мгновений нажмите **«Обновить»,** чтобы увидеть пользователей в списке назначений.

### <a name="directly-assigning-users-programmatically"></a>Непосредственное назначение пользователей программно

Вы также можете непосредственно назначить пользователю пакет доступа с помощью Microsoft Graph.  Пользователь в соответствующей роли с приложением `EntitlementManagement.ReadWrite.All` с делегированным разрешением может вызвать API для [создания accessPackageAssignmentRequest.](https://docs.microsoft.com/graph/api/accesspackageassignmentrequest-post?view=graph-rest-beta)

## <a name="remove-an-assignment"></a>Удалить назначение

**Роль предпосылки:** Глобальный администратор, администратор пользователей, владелец каталога или менеджер пакетов доступа

1. На портале Azure щелкните **Azure Active Directory** и выберите **Управление удостоверениями**.

1. В левом меню щелкните **пакеты Access,** а затем откройте пакет доступа.

1. В левом меню нажмите **«Назначения».**
 
1. Нажмите на флажок рядом с пользователем, чье задание вы хотите удалить из пакета доступа. 

1. Нажмите кнопку **Удалить** в верхней части левой панели. 
 
    ![Назначения - Удалить пользователя из пакета доступа](./media/entitlement-management-access-package-assignments/remove-assignment-select-remove-assignment.png)

    Появится уведомление, информирующее вас об удалении назначения.

# <a name="request-process-and-email-notifications-in-azure-ad-entitlement-management-preview"></a>Запрос уведомления процесса и электронной почты в управление правами Azure AD (Предварительная версия)

> [!IMPORTANT]
> Управление правами Azure Active Directory (Azure AD) в настоящее время находится в общедоступной предварительной версии.
> Эта предварительная версия предоставляется без соглашения об уровне обслуживания и не рекомендована для использования рабочей среде. Некоторые функции могут не поддерживаться или их возможности могут быть ограничены.
> Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

Когда пользователь отправляет запрос к пакету доступ, запуске процесса для доставки этого запроса. Управление правами Azure AD также отправляет уведомления по электронной почте утверждающих лиц и запрашивающими сторонами ключа событиями во время процесса.

В этой статье описывается процесс запроса, а также уведомления по электронной почте, отправляемых.

## <a name="request-process"></a>Процесс запроса

Пользователь, которому требуется доступ к пакету доступ можно отправить запрос доступа. В зависимости от конфигурации политики запрос может потребоваться утверждение. При запрос утвержден назначить пользователю доступ для каждого ресурса в пакете доступа начинает процесс. В примере ниже показан обзор процесса и различных состояний.

![Схема процесса утверждения](./media/entitlement-management-process/request-process.png)

| Состояние | ОПИСАНИЕ |
| --- | --- |
| Отправлено | Пользователь отправляет запрос. |
| Ожидающие утверждения | Если политика доступа пакета требует утверждения, перемещает запрос ожидает утверждения. |
| Срок действия истек | Если утверждающие не просмотрите запрос до истечения времени ожидания запроса утверждения, истечения срока действия запроса. Чтобы повторить попытку, пользователю будет нужно повторно отправьте свой запрос. |
| Запрещен | Утверждающий отклоняет запрос. |
| Approved | Утверждающее лицо утверждает запрос. |
| Доставка содержимого | Пользователь имеет **не** был назначен доступ ко всем ресурсам в пакете доступа. Если это внешнего пользователя, пользователь еще не доступ к каталог ресурсов и принял запрос разрешения. |
| Доставлено | Пользователь был присвоен доступ ко всем ресурсам в пакете доступа. |
| Доступ к расширенным | Если в политике разрешены расширения, пользователь расширена назначения. |
| Истек срок действия доступа | Доступ пользователя к пакету доступа истек. Чтобы получить доступ, он будет иметь для отправки запроса. |

## <a name="email-notifications"></a>Уведомления по почте

Если вы являетесь утверждающим лицом, когда нужно утвердить запрос доступа, и после завершения запрос доступа отправляются уведомления по электронной почте. Если вы являетесь запрашивающая сторона, будут отправляться уведомления по электронной почте, которые указывают состояние запроса. На следующей схеме показано, когда эти уведомления по электронной почте отправляются.

![Процесс по электронной почте права управления](./media/entitlement-management-process/email-notifications.png)

Ниже приведены более подробные сведения о каждой из этих уведомлений по электронной почте.

| # | Тема сообщения | При отправке | Отправлено |
| --- | --- | --- | --- |
| 1 | Требуется действие: Запрос на проверку доступа из *[запрашивающей стороне]* для *[доступа пакета]* по *[date]* | Когда запрашивающая сторона отправляет запрос для пакета доступ. | Все утверждающие |
| 2 | Требуется действие: Запрос на проверку доступа из *[запрашивающей стороне]* для *[доступа пакета]* по *[date]* | Время ожидания запроса X дней перед утверждением | Все утверждающие |
| 3 | Уведомление о состоянии: *[запрашивающей стороне]* в запрос на доступ к *[доступа пакета]* истек | Когда утверждающих не утверждать или отклонять запрос доступа в течение срока действия запроса | Источник запроса |
| 4. | Уведомление о состоянии: *[запрашивающей стороне]* запрос на доступ к *[доступа пакета]* завершения | При первом утверждающее лицо утверждает или отклоняет запрос доступа | Все утверждающие |
| 5 | Вы не имеете доступа к *[доступа пакета]* | Если запрашивающий запрещен доступ к пакету доступ | Источник запроса |
| 6 | Теперь у вас есть доступ к *[доступа пакета]*  | Если запрашивающий был предоставлен доступ для каждого ресурса в пакете доступа | Источник запроса |
| 7 | Доступ к *[доступа пакета]* срок действия истекает через X дней | X дней перед запрашивающей доступ к пакету доступ истечения срока действия | Источник запроса |
| 8 | Доступ к *[доступа пакета]* истек | Когда истекает срок действия запрашивающего объекта доступа к пакету доступ | Источник запроса |

### <a name="review-access-request-emails"></a>Отправляет сообщения запроса проверки доступа

Когда запрашивающая сторона отправляет запрос доступа для пакета доступ, который настроен для использования утверждения, все утверждающие, настроенные в политике получают уведомление по электронной почте с сведения о запросе. Сведения включают имя запрашивающего объекта, организации, доступа к дату начала и окончания, если указано, коммерческое обоснование, когда был отправлен запрос и когда запроса истечет. Адрес электронной почты со ссылкой где утверждающие могут утверждать или отклонять запрос на доступ. Ниже приведен пример уведомления по электронной почте, который отправляется на утверждение, когда запрашивающий отправляет запрос доступа.

![Письмо с запросом проверки доступа](./media/entitlement-management-shared/email-approve-request.png)

### <a name="approved-or-denied-emails"></a>Утвержденных или отклоненных сообщений электронной почты

Запрашивающие стороны уведомляются при их запрос доступа утвержденные, так и для доступа или при их доступе отказывается. Когда утверждающий просматривает запрос доступа, отправленных запрашивающая сторона, они могут утверждать или отклонять запрос на доступ. Утверждающий должен добавить коммерческое обоснование для их решения.

После утверждения запроса на доступ, управление правами запускает процесс предоставления запрашивающий сможет получать доступ к каждому из ресурсов в пакете доступа. После запрашивающей стороне был предоставлен доступ к каждому ресурсу пакета доступ, уведомление по электронной почте отправляется запрашивающей стороне, что их запрос доступа был утвержден и теперь имеют доступ к пакету доступ. Ниже приведен пример уведомления по электронной почте, отправляемое запрашивающая сторона, когда пользователи получают доступ к пакету доступ.

Когда запрос на доступ запрещен, уведомление по электронной почте отправляется запрашивающей стороне. Ниже приведен пример уведомления по электронной почте, отправляемое запрашивающая сторона, когда запрос на доступ запрещен.

### <a name="expired-access-request-emails"></a>Истек срок действия сообщений электронной почты для запроса доступа

Запрашивающие стороны получите уведомление, когда истек срок действия их запрос на доступ. Когда запрашивающая сторона отправляет запрос доступа, запрос содержит длительность запроса, после чего истечения срока его действия. Если нет, утверждающие отправить решение об одобрения/отклонения, запрос продолжает оставаться в состоянии ожидает утверждения. Когда запрос достигает его настроенных срок, запрос истекает и можно нужно больше не утвердить или отклонить утверждающими. В этом случае запрос переходит в состояние, истекшим сроком действия. Больше не нужно утвердить или отклонить запрос с истекшим сроком действия. Уведомление по электронной почте отправляется запрашивающей стороне с истекшим сроком действия запрос на доступ и что им нужно повторно отправить запрос на доступ. Ниже приведен пример уведомления электронной почты, отправляемые запрашивающая сторона при истечении срока действия их запрос на доступ.

![Истек срок действия запроса доступ к электронной почте](./media/entitlement-management-process/email-expired-access-request.png)

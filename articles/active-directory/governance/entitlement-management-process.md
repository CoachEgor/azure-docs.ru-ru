# <a name="request-process-and-email-notifications-in-azure-ad-entitlement-management-preview"></a>Обработка запросов и уведомлений по электронной почте в управлении назначениями Azure AD (Предварительная версия)

> [!IMPORTANT]
> Управление правами Azure Active Directory (Azure AD) сейчас предоставляется в общедоступной предварительной версии.
> Эта предварительная версия предоставляется без соглашения об уровне обслуживания и не рекомендована для использования рабочей среде. Некоторые функции могут не поддерживаться или их возможности могут быть ограничены.
> Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

Когда пользователь отправляет запрос в пакет Access, запускается процесс для доставки этого запроса. Управление назначением Azure AD также отправляет уведомления по электронной почте утверждающим и запрашивающим лицам при возникновении ключевых событий во время процесса.

В этой статье описывается процесс запроса и отправляемые уведомления по электронной почте.

## <a name="request-process"></a>Процесс запроса

Пользователь, которому необходим доступ к пакету Access, может отправить запрос на доступ. В зависимости от конфигурации политики запрос может потребовать утверждения. При утверждении запроса процесс начинает назначать пользователю доступ к каждому ресурсу в пакете доступа. На следующей диаграмме показан обзор процесса и различные состояния.

![Схема процесса утверждения](./media/entitlement-management-process/request-process.png)

| Область | Описание |
| --- | --- |
| Отправлено | Пользователь отправляет запрос. |
| Ожидающие утверждения | Если политика для пакета доступа требует утверждения, запрос переходит к ожидающему утверждению. |
| Истек срок | Если ни один из утверждающих не утвердит запрос в течение времени ожидания запроса на утверждение, срок действия запроса истечет. Чтобы повторить попытку, пользователю придется повторно отправить свой запрос. |
| Запрещен | Утверждающий отклоняет запрос. |
| Одобрен | Утверждающий утверждает запрос. |
| Доставка | Пользователю **не** был назначен доступ ко всем ресурсам в пакете доступа. Если это внешний пользователь, пользователь еще не получил доступ к каталогу ресурсов и принял запрос на разрешение. |
| Доставлено | Пользователю был назначен доступ ко всем ресурсам в пакете доступа. |
| Расширенный доступ | Если в политике разрешены расширения, пользователь расширил назначение. |
| Срок действия доступа истек | Срок действия доступа пользователя к пакету доступа истек. Чтобы снова получить доступ, пользователю придется отправить запрос. |

## <a name="email-notifications"></a>Уведомления по электронной почте

Если вы являетесь утверждающим, вы отправляете уведомления по электронной почте, когда вам нужно утвердить запрос на доступ и после завершения запроса на доступ. Если вы являетесь инициатором запроса, вы отправляете уведомления по электронной почте, которые указывают на состояние вашего запроса. На следующей схеме показано, когда отправляются уведомления по электронной почте.

![Процесс электронной почты управления назначениями](./media/entitlement-management-process/email-notifications.png)

В следующей таблице приведены более подробные сведения о каждом из этих уведомлений по электронной почте.

| # | Тема электронного письма | При отправке | Кому отправлено |
| --- | --- | --- | --- |
| 1 | Требуется действие: Проверьте запрос доступа от *[запрашивающей стороны]* к *[пакету доступа]* на *[DATE]* . | Когда запрашивающий отправляет запрос на пакет Access | Все утверждающие |
| 2 | Требуется действие: Проверьте запрос доступа от *[запрашивающей стороны]* к *[пакету доступа]* на *[DATE]* . | X дней до истечения времени ожидания запроса на утверждение | Все утверждающие |
| 3 | Уведомление о состоянии: *[запрашивающий]* срок действия запроса на доступ к *[пакету Access]* истек | Когда утверждающие не утверждают или не отклоняют запрос на доступ в течение срока запроса | Источник запроса |
| 4 | Уведомление о состоянии: *[запрашивающий]* запрос на доступ к *[пакету Access]* завершен | Когда первый утверждающий утверждает или отклоняет запрос на доступ | Все утверждающие |
| 5 | Вам отказано в доступе к *[пакету Access]* | Когда запрашивающей стороны отказано в доступе к пакету Access | Источник запроса |
| 6 | Теперь у вас есть доступ к *[пакету Access]*  | Когда запрашивающему пользователю предоставлен доступ к каждому ресурсу в пакете доступа | Источник запроса |
| 7 | Срок действия доступа к *[пакету Access]* истекает через X дн. | Через X дней до истечения срока действия запрашивающего пакета доступа | Источник запроса |
| 8 | Срок действия доступа к *[пакету Access]* истек | Когда истекает срок доступа запрашивающей стороны к пакету Access | Источник запроса |

### <a name="access-request-emails"></a>Сообщения с запросами на доступ

Когда запрашивающий отправляет запрос на доступ к пакету Access, настроенному для обязательного утверждения, все утверждающие, настроенные в политике, получают по электронной почте уведомление с подробными сведениями о запросе. Сведения включают имя запрашивающей стороны, организацию, дату начала и окончания действия, если предоставлено, Деловое обоснование, когда запрос был отправлен, а также время истечения срока запроса. Сообщение электронной почты содержит ссылку, где утверждающие могут утвердить или отклонить запрос на доступ. Ниже приведен пример уведомления по электронной почте, отправляемого утверждающему лицу при отправке запрашивающей стороны запроса на доступ.

![Проверка адреса электронной почты для запроса доступа](./media/entitlement-management-shared/email-approve-request.png)

### <a name="approved-or-denied-emails"></a>Утвержденные или отклоненные сообщения электронной почты

Запрашивающие стороны уведомляются, когда их запрос на доступ утверждается и доступен для доступа, или когда их запрос на доступ отклоняется. Когда утверждающий получает запрос на доступ, отправленный запрашивающей стороны, он может утвердить или отклонить запрос на доступ. Утверждающему лицу необходимо добавить Деловое обоснование для принятия решения.

При утверждении запроса на доступ управление назначением начинает процесс предоставления запрашивающей стороны доступа к каждому из ресурсов в пакете доступа. Когда запрашивающему пользователю предоставляется доступ к каждому ресурсу в пакете доступа, ему отправляется уведомление по электронной почте о том, что запрос на доступ был утвержден и теперь имеет доступ к пакету Access. Ниже приведен пример уведомления по электронной почте, которое отправляется запрашивающему, когда ему предоставляется доступ к пакету Access.

Когда запрос на доступ отклоняется, запрашивающему получателю отправляется уведомление по электронной почте. Ниже приведен пример уведомления по электронной почте, которое отправляется запрашивающему, когда запрос на доступ отклоняется.

### <a name="expired-access-request-emails"></a>Сообщения электронной почты для запроса на доступ с истекшим сроком

Запрашивающие стороны уведомляются по истечении срока их запроса на доступ. Когда запрашивающий объект отправляет запрос на доступ, запрос получает длительность запроса, по истечении которого он истечет. Если нет утверждающих, которые отправляют решение утвердить или отклонить, запрос остается в состоянии ожидания утверждения. По истечении заданного срока действия запрос истекает, и он больше не может утверждаться или отклоняться утверждающими. В этом случае запрос переходит в состояние с истекшим сроком действия. Запрос с истекшим сроком действия больше не может быть утвержден или запрещен. Уведомление по электронной почте отправляется запрашивающему пользователю о том, что срок действия его запроса на доступ истек, а также на необходимость повторной отправки запроса на доступ. Ниже приведен пример уведомления по электронной почте, отправляемого запрашивающему инициатору при истечении срока действия запроса на доступ.

![Адрес электронной почты для запроса на доступ с истекшим сроком](./media/entitlement-management-process/email-expired-access-request.png)

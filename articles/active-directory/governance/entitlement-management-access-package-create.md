# <a name="create-a-new-access-package-in-azure-ad-entitlement-management-preview"></a>Создание нового пакета доступ в управление правами Azure AD (Предварительная версия)

> [!IMPORTANT]
> Управление правами Azure Active Directory (Azure AD) сейчас предоставляется в общедоступной предварительной версии.
> Эта предварительная версия предоставляется без соглашения об уровне обслуживания и не рекомендована для использования рабочей среде. Некоторые функции могут не поддерживаться или их возможности могут быть ограничены.
> Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

Пакет доступа позволяет выполнять однократная настройка ресурсов и политики, которая автоматически управляет доступ в течение жизненного цикла пакета доступа. В этой статье описывается создание нового пакета доступа.

## <a name="overview"></a>Обзор

Все пакеты должны быть размещены в контейнере с именем каталога. Каталог определяет, какие ресурсы, можно добавить к пакету доступ. Если не указать каталог, доступ пакет будет отправляться в общий каталог. В настоящее время невозможно переместить существующий пакет доступа в другой каталог.

Все пакеты доступа должен иметь по крайней мере одна политика. Политики определяют, кто может запросить доступ пакета, а также параметры истечения срока действия и утверждения. При создании нового пакета доступ, можно создать политику начальной для пользователей в каталоге, пользователи не в каталоге, для администратора только прямые назначения или вы можете создать политику позже.

В примере ниже показан общий процесс создания нового пакета доступа.

![Создание пакета процесс доступа](./media/entitlement-management-access-package-create/access-package-process.png)

## <a name="start-new-access-package"></a>Запуск нового пакета доступа

**Требуемая роль:** Администратор или владелец каталога

1. Войдите на [портале Azure](https://portal.azure.com).

1. Щелкните **Azure Active Directory** и выберите **Управление удостоверениями**.

1. В меню слева щелкните **Пакеты для доступа**.

    ![Управление правами на портале Azure](./media/entitlement-management-shared/elm-access-packages.png)

1. Щелкните **Создать пакет для доступа**.

## <a name="basics"></a>Основы

На **основы** вкладке доступа пакета имя а также указать для создания пакета доступа в каком каталоге.

1. Введите отображаемое имя и описание для пакета доступ. Пользователи будут видеть эти сведения при отправке запроса для пакета доступ.

1. В **каталога** стрелку раскрывающегося списка, выберите каталога, вы хотите создать доступ упаковать в. Например возможно, владельца каталога, который управляет все маркетинговые ресурсы, которые могут запрашиваться. В этом случае можно выбрать маркетинга каталога.

    Вы увидите только каталоги у вас есть разрешение на создание пакетов доступ. Чтобы создать пакет доступ в существующий каталог, необходимо по крайней мере пользователя администратора, владельца каталога или доступа диспетчера пакетов.

    ![Доступ к пакету — основы](./media/entitlement-management-access-package-create/basics.png)

    Если вы хотите создать пакет доступ в новый каталог, нажмите кнопку **создать**. Введите имя каталога и описание и нажмите кнопку **создать**.

    Создание пакета доступ и все ресурсы, включенные в него будут добавлены в новый каталог. Кроме того вы автоматически станете первый владелец каталога. Вы можете добавить владельцев дополнительных каталога.

    Чтобы создать новый каталог, необходимо быть по крайней мере пользователя администратора или создателя каталога.

1. Нажмите кнопку **Далее**.

## <a name="resource-roles"></a>Роли ресурсов

На **ролей ресурсов** вкладке выберите ресурсы для включения в пакет доступа.

1. Щелкните тип ресурса, который вы хотите добавить (**группы**, **приложений**, или **сайтов SharePoint**).

1. В открывшейся области выберите выберите один или несколько ресурсов из списка.

    ![Пакет доступа — роли ресурсов](./media/entitlement-management-access-package-create/resource-roles.png)

    При создании пакета доступа в каталоге Общие или новый каталог, можно выбрать любой ресурс из каталога, вы являетесь владельцем. Необходимо быть по крайней мере пользователь администратором или каталога создателя.

    Если вы создаете пакет доступ в существующий каталог, можно выбрать любой ресурс, который уже находится в каталоге не владея его.

    Если вы являетесь администратором пользователя или владельца каталога, имеется дополнительная возможность выбрать ресурсы, которыми вы владеете, которых еще нет в каталоге. Если выделить ресурсы в выбранный каталог в настоящее время не, эти ресурсы также будут добавляться в каталог для других администраторов каталога для построения пакетов доступ с помощью. Если вы хотите выбрать ресурсы, используемые в данный момент выбранный каталог, проверка **только см. в разделе** флажок в верхней части выберите pan.

1. После выбора ресурсов, в **роли** выберите роли, которые нужно назначить для ресурса.

    ![Пакет доступа — Выбор роли ресурса](./media/entitlement-management-access-package-create/resource-roles-role.png)

1. Нажмите кнопку **Далее**.

## <a name="policy"></a>Политика

На **политики** вкладку, создание первого политику, чтобы указать, кто может запросить доступ пакета, а также параметры истечения срока действия и утверждения. Позже можно создать дополнительные политики, позволяя пользователям запрашивать доступ пакет с собственные параметры истечения срока действия и утверждения дополнительные группы. Можно также создать политики позже.

1. Задайте **создайте первую политику** переключатель **теперь** или **позже**.

    ![Доступ к пакету - политики](./media/entitlement-management-access-package-create/policy.png)

1. При выборе **позже**, перейдите вниз к разделу [проверки + создать](#review--create) раздела, чтобы создать пакет доступа.

1. При выборе **теперь**, выполните действия в следующих разделах политики.

На **просмотрите + создать** вкладке можно просмотреть параметры и проверить наличие ошибок проверки.

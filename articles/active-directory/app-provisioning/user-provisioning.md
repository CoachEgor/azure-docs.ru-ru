# <a name="automate-user-provisioning-and-deprovisioning-to-applications-with-azure-active-directory"></a>Автоматизация подготовки и деобеспечения приложений с помощью Active Directory Azure

В Active Directory Azure (Azure AD) термин **«подготовка приложений»** означает автоматическое создание идентификаторов пользователей и ролей в приложениях облака[(SaaS),](https://azure.microsoft.com/overview/what-is-saas/)к которым пользователи нуждаются в доступе. Кроме создания удостоверений пользователей, автоматическая подготовка включает в себя обслуживание и удаление удостоверений пользователей по мере изменения их статуса или ролей. Типичные сценарии включают подготовку пользователя Azure AD к таким приложениям, как [Dropbox](../saas-apps/dropboxforbusiness-provisioning-tutorial.md), [Salesforce](../saas-apps/salesforce-provisioning-tutorial.md), [ServiceNow](../saas-apps/servicenow-provisioning-tutorial.md), и других.

![Диаграмма обзора подготовки](./media/user-provisioning/provisioning-overview.png)

Эта функция позволяет:

- **Автоматизация подготовки:** Автоматически создавайте новые учетные записи в правильных системах для новых людей, когда они присоединяются к вашей команде или организации.
- **Автоматизация дезавуирования:** Автоматизирует учетные записи в нужных системах, когда люди покидают команду или организацию.
- **Синхронизация данных между системами:** Убедитесь, что идентификаторы в приложениях и системах обновляются на основе изменений в каталоге или вашей системе людских ресурсов.
- **Группы обеспечения:** Предоставление групп приложениям, которые их поддерживают.
- **Управлять доступом:** Мониторинг и аудит, которые были подготовлены в ваших приложениях.
- **Бесшовно развернуть в коричневых полевых сценариях:** Сопойтесь существующим идентификаторам между системами и допускаем легкую интеграцию, даже если пользователи уже существуют в целевой системе.
- **Используйте богатую настройку:** Воспользуйтесь настраиваемыми отображениями атрибутов, которые определяют, какие пользовательские данные должны передаваться из исходной системы в целевую систему.
- **Получайте оповещения о критических событиях:** Служба обеспечения обеспечивает оповещения для критических событий и позволяет интегрировать Log Analytics, где можно определить пользовательские оповещения для набора ваших бизнес-потребностей.

## <a name="benefits-of-automatic-provisioning"></a>Преимущества автоматического обеспечения

По мере того, как число приложений, используемых в современных организациях, продолжает расти, ИТ-аналитикам поручено в масштабе управления доступом. Такие стандарты, как Язык markups Markup (SAML) или Open ID Connect (OIDC), позволяют адменорам быстро настроить единый вход (SSO), но доступ также требует, чтобы пользователи были подготовлены к приложению. Для многих админинов подготовка означает вручную создавать каждую учетную запись пользователя или загружать файлы CSV каждую неделю, но эти процессы являются трудоемкими, дорогостоящими и подверженными ошибкам. Для автоматизации подготовки были приняты такие решения, как SAML (JIT), но предприятиям также необходимо решение для деобеспечения пользователей, когда они покидают организацию или больше не требуют доступа к определенным приложениям на основе изменения роли.

Некоторые общие мотивы для использования автоматического подготовки включают в себя:

- Максимальное повышение эффективности и точности процессов подготовки.
- Экономия на расходах, связанных с хостингом и обслуживанием разработанных на заказ решений и скриптов.
- Обеспечение безопасности организации путем мгновенного удаления идентификационных данных пользователей из ключевых приложений SaaS при выходе из организации.
- Легко импортировать большое количество пользователей в конкретное приложение или систему SaaS.
- Наличие единого набора политик для определения того, кто подготовлен и кто может войти в приложение.

Подготовка пользователей Azure AD может помочь решить эти проблемы. Чтобы узнать больше о том, как клиенты использовали подготовку пользователей Azure AD, вы можете прочитать [пример ASOS](https://aka.ms/asoscasestudy). Видео ниже содержит обзор подготовки пользователей в Azure AD:

## <a name="what-is-system-for-cross-domain-identity-management-scim"></a>Что такое система управления кросс-доменом идентификации (SCIM)?

Чтобы автоматизировать подготовку и деобеспечение, приложения разоблачают несвободные пользовательские и групповые AIS. Однако каждый, кто пытался управлять пользователями в нескольких приложениях, скажет вам, что каждое приложение пытается выполнять одни и те же простые действия, такие как создание или обновление пользователей, добавление пользователей в группы или деирование пользователей. Тем не менее, все эти простые действия реализуются немного по-разному, используя различные пути конечных точек, различные методы для указания информации пользователя и различные схемы для представления каждого элемента информации.

Для решения этих проблем спецификация SCIM предоставляет общую схему пользователя, помогая пользователям перемещаться в приложения, из них и вокруг них. SCIM становится де-факто стандартом для подготовки и, при использовании в сочетании со стандартами федерации, как SAML или OpenID Connect, предоставляет администраторам сквозное стандартное решение для управления доступом.

Для получения подробных рекомендаций по разработке конечной точки SCIM для автоматизации подготовки и деобеспечения пользователей и групп приложения [см.](use-scim-to-provision-users-and-groups.md) Для предварительно интегрированных приложений в галерее (Slack, Azure Databricks, Snowflake и т.д.), вы можете пропустить документацию разработчика и использовать приведенные [здесь](../saas-apps/tutorial-list.md)учебники .

## <a name="manual-vs-automatic-provisioning"></a>Сравнение подготовки вручную и автоматической подготовки

Приложения в галерее Azure AD поддерживают один из двух режимов подготовки:

* **Ручная** подготовка означает, что автоматического разъема Azure AD для приложения пока нет. Учетные записи пользователей должны создаваться вручную, например, путем добавления пользователей непосредственно в административный портал приложения или загрузки электронной таблицы с подробной информацией учетной записи пользователя. Проконсультируйтесь с документацией, предоставленной приложением, или свяжитесь с разработчиком приложения, чтобы определить, какие механизмы доступны.

* **Автоматическая подготовка** означает, что для этого приложения был разработан соединитель для подготовки Azure AD. Вы должны следовать инструкции учебник, специфичный для настройки подготовки к приложению. Руководства по настройке для приложений можно найти в [Списке руководств по интеграции приложений SaaS в Azure Active Directory](../saas-apps/tutorial-list.md).

В галерее Azure AD приложения, поддерживающие автоматическое обеспечение, обозначаются иконой **«Подготовка».** Переключитесь на новый опыт просмотра галереи, чтобы увидеть эти значки (в баннере в верхней части **страницы приложения**, выберите ссылку, которая говорит **Нажмите здесь, чтобы опробовать новые и улучшенные галереи приложений).**

![Представление значка в галерее приложений](./media/user-provisioning/browse-gallery.png)

Режим подготовки, поддерживаемый приложением, также отображается на вкладке **«Подготовка»** после добавления приложения в **приложения Enterprise.**

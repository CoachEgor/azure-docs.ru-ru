# <a name="manage-ibm-informix-database-resources-by-using-azure-logic-apps"></a>Управление ресурсами баз данных IBM Informix с помощью приложений логики Azure

С [помощью приложений Azure Logic Apps](../logic-apps/logic-apps-overview.md) и [разъема Informix](/connectors/informix/)можно создавать автоматизированные задачи и рабочие процессы, которые управляют ресурсами в базе данных IBM Informix. Этот разъем включает в себя клиент Microsoft, который общается с удаленными серверными компьютерами Informix в сети TCP/IP, включая облачные базы данных, такие как IBM Informix для Windows, работающих в виртуализации Azure, и базы данных при использовании [внутреннего шлюза данных.](../logic-apps/logic-apps-gateway-connection.md) Вы можете подключиться к этим платформам и версиям Informix, если они настроены для поддержки распределенных реляционных архитектурных баз данных (DRDA) клиентских соединений:

* IBM Informix 12.1
* IBM Informix 11.7

В этой статье показано, как использовать соединитель в приложении логики для обработки операций базы данных.

## <a name="prerequisites"></a>Предварительные требования

* Подписка Azure. Если у вас еще нет подписки Azure, [зарегистрируйтесь для получения бесплатной учетной записи Azure](https://azure.microsoft.com/free/).

* Для локальных баз данных [загрузите и установите локальный шлюз данных](../logic-apps/logic-apps-gateway-install.md) на локальном компьютере, а затем [создайте ресурс шлюза данных Azure на портале Azure.](../logic-apps/logic-apps-gateway-connection.md)

* Логическое приложение, где вам нужен доступ к базе данных Informix. Этот разъем предоставляет только действия, поэтому приложение логики уже должно начаться с триггера, например, [триггера Recurrence.](../connectors/connectors-native-recurrence.md) 

## <a name="add-an-informix-action"></a>Добавить действие Informix

1. Откройте приложение логики в конструкторе приложений логики на [портале Azure](https://portal.azure.com), если оно еще не открыто.

1. Под шагом, где вы хотите добавить действие Informix, выберите **Новый шаг.**

   Чтобы добавить действие между существующими шагами, наведите указатель мыши на соединяющую стрелку. Выберите знак**+** плюс (), который появляется, а затем выберите **Добавить действие.**

1. В поле поиска введите `informix` в качестве фильтра. Из списка действий выберите нужное действие, например:

   ![Выберите действие Informix для запуска](./media/connectors-create-api-informix/select-informix-connector-action.png)

   Разъем предоставляет следующие действия, которые заражают соответствующие операции базы данных:

   * Получить таблицы - Список `CALL` таблиц базы данных с помощью оператора
   * Получить строки - Прочитайте все `SELECT *` строки с помощью оператора
   * Получить строку - Прочитайте `SELECT WHERE` строку с помощью оператора
   * Добавление строки с `INSERT` помощью оператора
   * Отсечение `UPDATE` строки с помощью оператора
   * Удалить строку с `DELETE` помощью оператора

1. Если вам предложено предоставить сведения о подключении к базе данных Informix, выполните [следующие шаги по созданию соединения,](#create-connection)а затем продолжайте делать следующий шаг.

1. Предоставьте информацию для выбранного вами действия:

   | Действие | Описание | Свойства и описания |
   |--------|-------------|-----------------------------|
   | **Получение таблиц** | Перечислите таблицы баз данных, запустив выписку Informix CALL. | None |
   | **Получение строк** | Извлеките все строки в заданную таблицу, запустив оператора Informix. `SELECT *` | **Название таблицы**: Название для захотевной таблицы Informix <p><p>Чтобы добавить другие свойства в это действие, выберите их из нового списка **параметров Добавить.** Для получения дополнительной информации смотрите [справочную тему разъема](/connectors/informix/). |
   | **Получение строки** | Извлеките строку из указанной таблицы, запустив оператора Informix. `SELECT WHERE` | - **Название таблицы**: Название для захотевной таблицы Informix <br>- **Идентификатор строки**: Уникальный идентификатор для строки, например,`9999` |
   | **Вставка строки** | Добавьте строку в указанную таблицу Informix, запустив оператора Informix. `INSERT` | - **Название таблицы**: Название для захотевной таблицы Informix <br>- **элемент**: строка с значениями для добавления |
   | **Обновление строки** | Измените строку в указанной таблице Informix, запустив оператора Informix. `UPDATE` | - **Название таблицы**: Название для захотевной таблицы Informix <br>- **Идентификатор строки**: Уникальный идентификатор для обновления строки, например,`9999` <br>- **Строка**: Строка с обновленными значениями, например,`102` |
   | **Удалить строку** | Удалите строку из указанной таблицы Informix, запустив оператора Informix. `DELETE` | - **Название таблицы**: Название для захотевной таблицы Informix <br>- **Идентификатор строки**: Уникальный идентификатор для строки для удаления, например,`9999` |
   ||||

1. Сохраните приложение логики. Теперь либо [проверьте приложение логики,](#test-logic-app) либо продолжайте строить приложение логики.

## <a name="connect-to-informix"></a>Подключение к Informix

1. Если приложение логики подключается к предварительной базе данных, выберите **Connect через шлюз данных.**

1. Предоставьте эту информацию о подключении, а затем выберите **Создать**.

   | Свойство | Свойство JSON | Обязательно | Пример значения | Описание |
   |----------|---------------|----------|---------------|-------------|
   | Имя соединения | `name` | Да | `informix-demo-connection` | Имя для подключения к базе данных Informix |
   | Сервер | `server` | Да | - Облако:`informixdemo.cloudapp.net:9089` <br>- На территории:`informixdemo:9089` | Адрес TCP/IP или псевдоним, который находится в формате IPv4 или IPv6, за которым следуют толстой кишки и номер порта TCP/IP |
   | База данных | `database` | Да | `nwind` | Имя реляционной базы данных DRDA (RDBNAM) или имя базы данных Informix (dbname). Informix принимает струну на 128 байт. |
   | Проверка подлинности | `authentication` | Только на территории | **Базовый** или **Windows** (kerberos) | Тип аутентификации, требуемый вашей базой данных Informix. Это свойство появляется только при выборе **Connect через шлюз данных.** |
   | Имя пользователя | `username` | нет | <*база данных-имя пользователя*> | Имя пользователя базы данных |
   | Пароль | `password` | нет | <*база данных-пароль*> | Пароль для базы данных |
   | Шлюз | `gateway` | Только на территории | - <*подписка на Azure*> <br>- <*Ресурс Azure-on-premises-data-gateway*> | Подписка Azure и название ресурса Azure для портативного шлюза данных, созданного на портале Azure. Свойство и субсвойства **шлюза** появляется только при выборе **Connect через шлюз данных.** |
   ||||||

   Пример:

   * **Облачная база данных**

     ![Информация о подключении к облачной базе данных](./media/connectors-create-api-informix/informix-cloud-connection.png)

   * **База данных на территории**

     ![Информация о подключении к базе данных](./media/connectors-create-api-informix/informix-on-premises-connection.png)

1. Сохраните приложение логики.

## <a name="test-your-logic-app"></a>Тестирование приложения логики

1. На панели инструментов Logic App Designer выберите **Run.** После запуска приложения логики можно просмотреть выводы из этого запуска.

1. Из меню приложения логики выберите **Обзор.** На панели обзора, под **историей резюме** > **выполнить**, выберите самый последний запуск.

1. Под **логикой приложение запустить**, выберите **Выполнить Подробная информация**.

1. Из списка действий выберите действие с выводами, которые вы хотите просмотреть, например, **Get_tables.**

   Если акция была успешной, их **свойство Status** помечено как **Успешное.**

1. Для просмотра входных данных под **ссылкой входов**выберите ссылку URL. Для просмотра выходов в ссылке **Outputs Link** выберите ссылку URL. Вот некоторые примеры выходов:

# <a name="create-schedule-and-run-recurring-tasks-and-workflows-with-the-sliding-window-trigger-in-azure-logic-apps"></a>Создавать, планировать и выполнять повторяющиеся задачи и рабочие процессы с триггером скользящего окна в Azure Logic Apps

Регулярно запускать задачи, процессы или задания, которые необходимо обрабатывать данные фрагментами, непрерывное, можно запустить рабочий процесс приложения логики с **скользящее окно - расписание** триггера. Можно задать дату и время, а также часовой пояс для запуска рабочего процесса и сведения о повторении для повторения этого рабочего процесса. Если для какой-либо причине не приводил повторения, этот триггер обрабатывает эти пропущенных повторения. Например при синхронизации данных между базой данных и хранилища резервных копий, используйте триггер скользящее окно, чтобы синхронизировать данные возвращает не неся пропуски. Дополнительные сведения о встроенных расписание триггеры и действия, см. в разделе [расписание и повторяющиеся автоматического запуска, задачи и рабочие процессы с помощью Azure Logic Apps](../logic-apps/concepts-schedule-automated-recurring-tasks-workflows.md).

Ниже приведены некоторые шаблоны, которые поддерживает этот триггер.

* Незамедлительное выполнение и повторение каждые *n* число секунд, минут или часов.

* Запустить в определенную дату и время, а затем запуск и повторение каждые *n* число секунд, минут или часов. С этим триггером можно указать время начала в прошлом, которая выполняет все предыдущие повторения.

* Задержка каждого повторения в течение определенного периода времени перед запуском.

Различия между этот триггер и триггер повторения или Дополнительные сведения о планировании повторяющихся рабочих процессов, см. в разделе [автоматической расписание и выполнения повторяющихся задач, процессы и рабочие процессы с помощью Azure Logic Apps](../logic-apps/concepts-schedule-automated-recurring-tasks-workflows.md).

> [!TIP]
> Если вы хотите вызвать приложение логики и запустить только один раз в будущем, см. раздел [выполнения заданий только один раз](../logic-apps/concepts-schedule-automated-recurring-tasks-workflows.md#run-once).

## <a name="add-sliding-window-trigger"></a>Добавление триггера скользящего окна

1. Войдите на [портале Azure](https://portal.azure.com). Создание пустого приложения логики.

1. После того как отобразится конструктор приложений логики, в поле поиска введите «скользящее окно» в качестве фильтра. В списке триггеров выберите этот триггер как первый шаг в рабочем процессе приложения логики: **Скользящее окно**

   ![Выберите триггер «Скользящее окно»](./media/connectors-native-sliding-window/add-sliding-window-trigger.png)

1. Задайте интервал повторения и единицу времени. В этом примере задайте запуск рабочего процесса каждый день.

   ![Установка интервала и частоты](./media/connectors-native-sliding-window/sliding-window-trigger-details.png)

   | Свойство | Обязательно для заполнения | Имя JSON | type | Описание |
   |----------|----------|-----------|------|-------------|
   | **Интервал** | Да | interval | Integer | Положительное целое число, которое описывает, как часто выполняется рабочий процесс с учетом заданной частоты. Ниже приведены минимальный и максимальный интервалы. <p>— Час: 1–12 000 часов. </br>— Минута: 1–72 000 минут. </br>— Секунда: 1–9 999 999 секунд.<p>Например если интервал равен 6, а значение частоты — «Hour», то Повтор будет происходить каждые 6 часов. |
   | **Frequency** | Да | frequency | String | Единица измерения времени, используемая для повторения: **Второй**, **минуту**, или **час** |

## <a name="workflow-definition---sliding-window"></a>Определение рабочего процесса — скользящего окна

В свое приложение логики базового определения рабочего процесса, который использует формат JSON, можно просмотреть определение триггера скользящее окно с параметрами, которые были выбраны. Чтобы просмотреть это определение, на панели инструментов конструктора, выберите **представление кода**. Чтобы вернуться в конструктор, выберите на панели инструментов конструктора **конструктор**.

В этом примере показано, как определение триггера скользящее окно может выглядеть в определении базового рабочего процесса где задержки для каждого повторения — 5 секунд для почасовой повторения:

``` json
"triggers": {
   "Recurrence": {
      "type": "SlidingWindow",
      "Sliding_Window": {
         "inputs": {
            "delay": "PT5S"
         },
         "recurrence": {
            "frequency": "Hour",
            "interval": 1,
            "startTime": "2019-05-13T14:00:00Z",
            "timeZone": "Pacific Standard Time"
         }
      }
   }
}
```

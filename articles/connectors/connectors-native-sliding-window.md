# <a name="schedule-and-run-tasks-for-contiguous-data-by-using-the-sliding-window-trigger-in-azure-logic-apps"></a>Расписание и выполнение задач для смежных данных с помощью триггера sliding Window в приложениях логики Azure

Для регулярного выполнения задач, процессов или заданий, которые должны обрабатывать данные в смежных фрагментах, можно запустить рабочий процесс приложения логики с триггером **Sliding Window.** Можно установить дату и время, а также часовой пояс для запуска рабочего процесса и повторение для повторения этого рабочего процесса. Если рецидивы пропущены по какой-либо причине, этот триггер обрабатывает эти пропущенные рецидивы. Например, при синхронизации данных между базой данных и резервным хранилищем используйте триггер Sliding Window, чтобы данные были синхронизированы без возникновения пробелов. Для получения дополнительной информации о встроенных триггерах и действиях Расписание [см. Расписание и запуск повторяющихся автоматизированных задач и рабочих процессов с приложениями Azure Logic Apps.](../logic-apps/concepts-schedule-automated-recurring-tasks-workflows.md)

Вот некоторые шаблоны, которые поддерживает этот триггер:

* Выполнить немедленно и повторить каждый *п* количество секунд, минут или часов.

* Начните с определенной даты и времени, затем запустите и повторите каждое *n* количество секунд, минут или часов. С помощью этого триггера можно указать время начала в прошлом, которое выполняет все прошлые рецидивы.

* Задержка каждого повторения в течение определенного периода времени перед запуском.

Для различий между этим триггером и триггером повторения или для получения дополнительной информации о планировании повторяющихся рабочих процессов [см. Расписание и запуск повторяющихся автоматизированных задач, процессов и рабочих процессов с azure Logic Apps.](../logic-apps/concepts-schedule-automated-recurring-tasks-workflows.md)

> [!TIP]
> Если вы хотите запустить приложение логики и запустить только один раз в будущем, [см.](../logic-apps/concepts-schedule-automated-recurring-tasks-workflows.md#run-once)

## <a name="add-sliding-window-trigger"></a>Добавить спусковой крючок сползания

1. Войдите на [портал Azure](https://portal.azure.com). Создание пустого приложения логики.

1. После того, как Logic App Designer появится в поле поиска, введите "раздвижное окно" в качестве фильтра. Из списка триггеров выберите этот триггер в качестве первого шага в рабочем процессе приложения **логики: Раздвижное окно**

   ![Выберите триггер "Скользящее окно"](./media/connectors-native-sliding-window/add-sliding-window-trigger.png)

1. Задайте интервал повторения и единицу времени. В этом примере задайте запуск рабочего процесса каждый день.

   ![Установка интервала и частоты](./media/connectors-native-sliding-window/sliding-window-trigger-details.png)

   | Свойство | Обязательно | Имя JSON | Тип | Описание |
   |----------|----------|-----------|------|-------------|
   | **Интервал** | Да | interval | Целое число | Положительное целое число, которое описывает, как часто выполняется рабочий процесс с учетом заданной частоты. Ниже приведены минимальный и максимальный интервалы. <p>— Час: 1–12 000 часов; </br>— Минута: 1–72 000 минут; </br>— Секунда: 1–9 999 999 секунд.<p>Например, если интервал 6, а частота "Час", то повторение происходит каждые 6 часов. |
   | **Частота** | Да | frequency | Строка | Единица времени для повторения: **Второй**, **Минута**, или **час** |

   Любые выбранные параметры отображаются на спусковом крючке после выбора.

   | Свойство | Обязательно | Имя JSON | Тип | Описание |
   |----------|----------|-----------|------|-------------|
   | **Задержки** | нет | delay | Строка | Продолжительность задержки каждого повторения с помощью [спецификации времени даты ISO 8601](https://en.wikipedia.org/wiki/ISO_8601#Durations) |

## <a name="workflow-definition---sliding-window"></a>Определение рабочего процесса - Раздвижное окно

В базовом определении рабочего процесса приложения логики, в котором используется JSON, можно просмотреть определение триггера Sliding Window с выбранными опциями. Чтобы просмотреть это определение на панели инструментов конструктора, выберите **представление кода.** Чтобы вернуться к дизайнеру, выберите на дизайнере панель инструментов, **Дизайнер**.

Этот пример показывает, как определение триггера sliding Window может выглядеть в базовом определении рабочего процесса, где задержка для каждого повторения составляет пять секунд для почасового повторения:

``` json
"triggers": {
   "Recurrence": {
      "type": "SlidingWindow",
      "Sliding_Window": {
         "inputs": {
            "delay": "PT5S"
         },
         "recurrence": {
            "frequency": "Hour",
            "interval": 1,
            "startTime": "2019-05-13T14:00:00Z",
            "timeZone": "Pacific Standard Time"
         }
      }
   }
}
```

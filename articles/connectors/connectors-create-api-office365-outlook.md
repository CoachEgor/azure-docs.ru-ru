# <a name="manage-email-contacts-and-calendars-in-office-365-outlook-by-using-azure-logic-apps"></a>Управление сообщениями электронной почты, контактами и календарями в Office 365 Outlook с помощью Azure Logic Apps

С [помощью приложений Azure Logic Apps](../logic-apps/logic-apps-overview.md) и [разъема Office 365 Outlook](/connectors/office365connector/)можно создавать автоматизированные задачи и рабочие процессы, которые управляют учетной записью Office 365 путем создания логических приложений. Например, вы автоматизировать следующие задачи:

* Получить, отправить и ответить на электронную почту. 
* Расписание встреч в календаре.
* Добавляйте и отсваивайте контакты. 

Можно использовать любой триггер для запуска рабочего процесса, например, при поступлении нового письма, при обновлении элемента календаря или при событии в службе разлительных данных, например В Salesforce. Можно использовать действия, которые реагируют на событие триггера, например, отправлять электронное письмо или создавать новое событие календаря. 

> [!NOTE]
> Для автоматизации задач @outlook.com @hotmail.com для учетной записи используйте [Outlook.com разъем.](../connectors/connectors-create-api-outlook.md)

## <a name="prerequisites"></a>Предварительные требования

* Офис [365 счет](https://www.office.com/)

* Подписка Azure. Если у вас еще нет подписки Azure, [зарегистрируйтесь для получения бесплатной учетной записи Azure](https://azure.microsoft.com/free/). 

* Логическое приложение, где вы хотите получить доступ к учетной записи Office 365 Outlook. Чтобы начать рабочий процесс с триггера Office 365 Outlook, необходимо иметь [пустое приложение логики.](../logic-apps/quickstart-create-first-logic-app-workflow.md) Чтобы добавить в рабочий процесс действие Office 365 Outlook, приложение логики должно уже иметь триггер.

## <a name="add-a-trigger"></a>Добавление триггера

[Триггер](../logic-apps/logic-apps-overview.md#logic-app-concepts) — это событие, которое запускает рабочий процесс в приложении логики. В этом примере логическое приложение использует триггер "опрос", который проверяет любое обновленное событие календаря в вашей учетной записи электронной почты, на основе указанного интервала и частоты.

1. На [портале Azure](https://portal.azure.com)откройте пустое логическое приложение в Logic App Designer.

1. В поле поиска введите `office 365 outlook` в качестве фильтра. Этот пример **выбирает, когда предстоящее событие начинается в ближайшее время.**
   
   ![Выберите триггер для запуска приложения логики](./media/connectors-create-api-office365-outlook/office365-trigger.png)

1. Если вам было предложено войти в систему, предоставьте учетные данные Office 365, чтобы приложение логики подключилось к вашей учетной записи. В противном случае, если соединение уже существует, предоставьте информацию о свойствах триггера.

   Этот пример выбирает календарь, который проверяет триггер, например:

   ![Настройка свойств триггера](./media/connectors-create-api-office365-outlook/select-calendar.png)

1. В триггере установите значения **частоты** и **интервала.** Чтобы добавить другие доступные триггерные свойства, такие как **часовой пояс,** выберите эти свойства из нового списка **параметров Добавить.**

   Например, если вы хотите, чтобы триггер проверял календарь каждые 15 `15`минут, установите **частоту** **на минуту**и **установите Interval.** 

   ![Установите частоту и интервал для триггера](./media/connectors-create-api-office365-outlook/calendar-settings.png)

1. На панели инструментов конструктора щелкните **Сохранить**.

Теперь добавьте действие, которое выполняется после срабатывания спускового крючка. Например, можно добавить действие Twilio **Send message,** которое отправляет текст, когда событие календаря начинается через 15 минут.

## <a name="add-an-action"></a>Добавление действия

[Действие](../logic-apps/logic-apps-overview.md#logic-app-concepts) — это операция, запущенная рабочим процессом в приложении логики. Этот пример логики приложение создает новый контакт в Office 365 Outlook. Для создания контакта можно использовать выход из другого триггера или действия. Например, предположим, что приложение логики использует триггер Dynamics 365 **при создании записи.** Вы можете добавить Office 365 Outlook **Создание контактных** действий и использовать выходы из триггера SalesForce для создания нового контакта.

1. На [портале Azure](https://portal.azure.com)откройте логическое приложение в Logic App Designer.

1. Чтобы добавить действие в качестве последнего шага в рабочем процессе, выберите **Новый шаг.** 

   Чтобы добавить действие между шагами, переместите указатель на стрелку между этими шагами. Выберите знак**+** плюс (), который появляется, а затем выберите **Добавить действие.**

1. В поле поиска введите `office 365 outlook` в качестве фильтра. Этот пример выбирает **Контакт Создать**.

   ![Выберите действие для выполнения в приложении логики](./media/connectors-create-api-office365-outlook/office365-actions.png) 

1. Если вам было предложено войти в систему, предоставьте учетные данные Office 365, чтобы приложение логики подключилось к вашей учетной записи. В противном случае, если ваше соединение уже существует, предоставьте информацию о свойствах действия.

   Этот пример выбирает папку контактов, в которой действие создает новый контакт, например:

   ![Настройка свойств действия](./media/connectors-create-api-office365-outlook/select-contacts-folder.png)

   Чтобы добавить другие доступные свойства действия, выберите эти свойства из нового списка **параметров Добавить.**

1. На панели инструментов конструктора щелкните **Сохранить**.

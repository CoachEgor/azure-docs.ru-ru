# <a name="call-http-or-https-endpoints-by-using-azure-logic-apps"></a>Вызов конечных точек HTTP или HTTPS с помощью Azure Logic Apps

С помощью [Azure Logic Apps](../logic-apps/logic-apps-overview.md) и встроенный соединитель HTTP, вы можете автоматизировать рабочие процессы, которые регулярно вызова любой конечной точки HTTP или HTTPS с помощью приложений логики. Например можно отслеживать конечную точку службы для веб-сайта, проверив эту конечную точку по указанному расписанию. Когда происходит определенное событие в этой конечной точке, например веб-сайта, выходят из строя, событие запускает рабочий процесс приложения логики и запускает указанные действия.

Чтобы проверить или *опроса* конечную точку по регулярному расписанию, можно использовать триггер HTTP в качестве первого шага в рабочем процессе. При каждой проверке триггер отправляет вызов или *запрос* к конечной точке. Ответ от конечной точки определяет, выполняется ли рабочий процесс приложения логики. Триггер передает любое содержимое из ответа действиям в приложении логики.

При необходимости действия HTTP можно использовать как любой другой этап в рабочем процессе для вызова конечной точки. Ответ от конечной точки определяет, как выполнить оставшиеся действия рабочего процесса.

На основе возможностей целевая конечная точка, соединитель HTTP поддерживает уровень безопасности транспорта (TLS) версии 1.0, 1.1 и 1.2. Служба Logic Apps согласовывает с конечной точкой по сравнению с использованием наивысшей поддерживаемой версии невозможно. Таким образом например, если конечная точка поддерживает 1.2, соединитель использует 1.2 сначала. В противном случае соединитель использует наивысшей поддерживаемой версии.

## <a name="prerequisites"></a>Технические условия

* Подписка Azure. Если у вас еще нет подписки Azure, [зарегистрируйтесь для получения бесплатной учетной записи Azure](https://azure.microsoft.com/free/).

* URL-адрес для целевой конечной точки, необходимо вызвать

* Базовые знания о [создании приложений логики](../logic-apps/quickstart-create-first-logic-app-workflow.md). Если вы не знакомы с приложениями логики, ознакомьтесь со статьей [Что такое Azure Logic Apps](../logic-apps/logic-apps-overview.md).

* Приложение логики, из которой необходимо вызвать целевой конечной точки. Чтобы начать с триггером HTTP, [создать пустое приложение логики](../logic-apps/quickstart-create-first-logic-app-workflow.md). Чтобы использовать действия HTTP, запустите свое приложение логики с любой триггер, который вы хотите. В этом примере триггер HTTP в качестве первого шага.

## <a name="add-an-http-trigger"></a>Добавление триггера HTTP

Этот триггер встроенные совершает вызов HTTP на указанный URL-адрес для конечной точки и возвращает ответ.

1. Войдите на [портале Azure](https://portal.azure.com). Откройте ваш пустое приложение логики в конструкторе приложений логики.

1. В конструкторе в поле поиска введите «http» в качестве фильтра. Из **триггеры** выберите **HTTP** триггера.

   ![Выбор триггера HTTP](./media/connectors-native-http/select-http-trigger.png)

   В этом примере переименовывает триггера «Триггер HTTP», чтобы этот шаг имеет более описательное имя. Кроме того в примере ниже добавляется действие HTTP, и оба имена должны быть уникальными.

1. Укажите значения для [параметры триггера HTTP](../logic-apps/logic-apps-workflow-actions-triggers.md##http-trigger) , необходимо включить в вызове на целевой конечной точке. Настройте для интервала триггера повторения для проверки целевой конечной точки.

   ![Ввод параметров триггера HTTP](./media/connectors-native-http/http-trigger-parameters.png)

   Дополнительные сведения о доступных типах проверки подлинности для HTTP, см. в разделе [проверки подлинности HTTP триггеры и действия](../logic-apps/logic-apps-workflow-actions-triggers.md#connector-authentication).

1. Чтобы добавить другие доступные параметры, откройте **добавьте параметр** и выберите нужные параметры.

1. Продолжите создание рабочего процесса приложения логики с действиями, выполняемыми при срабатывании триггера.

1. После завершения этого сделать, не забудьте сохранить приложение логики. На панели инструментов конструктора выберите **Сохранить**.

## <a name="add-an-http-action"></a>Добавление действия HTTP

Это действие встроенные совершает вызов HTTP на указанный URL-адрес для конечной точки и возвращает ответ.

1. Войдите на [портале Azure](https://portal.azure.com). Создание приложения логики в Конструкторе приложений логики.

   В этом примере триггер HTTP в качестве первого шага.

1. Установите шаг, где вы хотите добавить действия HTTP, **новый шаг**.

   Чтобы добавить действие между шагами, переместите указатель на стрелку между шагами. Выберите знак «плюс» ( **+** ), появится, а затем выберите **добавить действие**.

1. В конструкторе в поле поиска введите «http» в качестве фильтра. Из **действия** выберите **HTTP** действие.

   ![Выбор действия HTTP](./media/connectors-native-http/select-http-action.png)

   В этом примере переименовывает действие «Действия"HTTP», чтобы этот шаг имеет более описательное имя.

1. Укажите значения для [параметров действия HTTP](../logic-apps/logic-apps-workflow-actions-triggers.md##http-action) , необходимо включить в вызове на целевой конечной точке.

   ![Ввод параметров для действия HTTP](./media/connectors-native-http/http-action-parameters.png)

   Дополнительные сведения о доступных типах проверки подлинности для HTTP, см. в разделе [проверки подлинности HTTP триггеры и действия](../logic-apps/logic-apps-workflow-actions-triggers.md#connector-authentication).

1. Чтобы добавить другие доступные параметры, откройте **добавьте параметр** и выберите нужные параметры.

1. Когда закончите, сохраните приложение логики. На панели инструментов конструктора выберите **Сохранить**.

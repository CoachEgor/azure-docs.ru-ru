# <a name="delay-running-the-next-action-in-azure-logic-apps"></a>Задержка выполнения следующего действия в приложениях логики Azure

Чтобы ваше логическое приложение подождать много времени перед запуском следующего действия, можно добавить встроенное действие **Delay - Расписание** перед действием в рабочем процессе приложения логики. Или можно добавить встроенную **задержку до - Расписание** действий, чтобы дождаться определенной даты и времени, прежде чем запустить следующее действие. Для получения дополнительной информации о встроенных действиях и триггерах Расписание см. [Расписание и запуск повторяющихся автоматизированных задач и рабочих процессов с приложениями Azure Logic Apps.](../logic-apps/concepts-schedule-automated-recurring-tasks-workflows.md)

* **Задержка**: Подождите указанное количество единиц времени, например, секунды, минуты, часы, дни, недели или месяцы, до следующего действия.

* **Задержка до**: Подождите, пока не будет указана дата и время до следующего действия.

Вот несколько примеров использования этих действий:

* ожидание буднего дня, чтобы отправить обновление состояния по почте;

* Отложите рабочий процесс до завершения вызова HTTP перед возобновлением и извлечением данных.

## <a name="prerequisites"></a>Предварительные требования

* Подписка Azure. Если у вас нет подписки, вы можете [подписаться на бесплатную учетную запись Azure.](https://azure.microsoft.com/free/)

* Основные знания о [логических приложениях](../logic-apps/logic-apps-overview.md). Прежде чем использовать действие, приложение логики должно сначала начать с триггера. Вы можете использовать любой триггер, который вы хотите, и добавить другие действия, прежде чем добавить действие задержки. В этой теме используется триггер Office 365 Outlook. Если вы новичок в логических приложениях, узнайте, [как создать свое первое логическое приложение.](../logic-apps/quickstart-create-first-logic-app-workflow.md)

<a name="add-delay"></a>

## <a name="add-the-delay-action"></a>Добавить действие задержки

1. В Logic App Designer, под шагом, где вы хотите добавить действие задержки, выберите **новый шаг.**

   Чтобы добавить действие задержки между шагами, переместите указатель на стрелку, которая соединяет шаги. Выберите знак плюс (яп.), который появляется, а затем выберите **Добавить действие.**

1. В поле поиска введите "задержку" в качестве фильтра. Из списка действий выберите это действие: **Задержка**

   ![Добавить действие "Задержка"](./media/connectors-native-delay/add-delay-action.png)

1. Укажите время ожидания до запуска следующего действия.

   ![Установить время для задержки](./media/connectors-native-delay/delay-time-intervals.png)

   | Свойство | Имя JSON | Обязательно | Тип | Описание |
   |----------|-----------|----------|------|-------------|
   | Count | count | Да | Целое число | Число единиц времени для задержки |
   | Единицы | unit | Да | Строка | Единица времени, `Second`например: `Minute` `Hour`, `Day` `Week`, ,`Month` |
   ||||||

1. Добавьте все другие действия, которые вы хотите выполнить в рабочем процессе.

1. Сохраните приложение логики, когда закончите.

<a name="add-delay-until"></a>

## <a name="add-the-delay-until-action"></a>Добавить действие задержки до действия

1. В Logic App Designer, под шагом, где вы хотите добавить действие задержки, выберите **новый шаг.**

   Чтобы добавить действие задержки между шагами, переместите указатель на стрелку, которая соединяет шаги. Выберите знак плюс (яп.), который появляется, а затем выберите **Добавить действие.**

1. В поле поиска введите "задержку" в качестве фильтра. Из списка действий выберите это действие: **Задержка до**

   ![Добавление действия "Задержка до"](./media/connectors-native-delay/add-delay-until-action.png)

1. Укажите дату и время окончания рабочего процесса.

   ![Укажите отметку времени для того, когда прекратить задержку](./media/connectors-native-delay/delay-until-timestamp.png)

   | Свойство | Имя JSON | Обязательно | Тип | Описание |
   |----------|-----------|----------|------|-------------|
   | Отметка времени | TIMESTAMP | Да | Строка | Дата окончания и время возобновления рабочего процесса с помощью этого формата: <p>YYYY-MM-DDThh:mm:ss <p>Так, например, если вы хотите 18 сентября 2017 года в 14:00, укажите "2017-09-18T14:00:00". <p>**Примечание:** На этот раз формат должен следовать [спецификации времени даты ISO 8601](https://en.wikipedia.org/wiki/ISO_8601#Combined_date_and_time_representations) в [формате даты UTC,](https://en.wikipedia.org/wiki/Coordinated_Universal_Time)но без [смещения UTC.](https://en.wikipedia.org/wiki/UTC_offset) Без часового пояса, вы должны добавить букву "Я" в конце без каких-либо пробелов. Эта буква Z ссылается на соответствующее [судовое время](https://en.wikipedia.org/wiki/Nautical_time). |
   ||||||

1. Добавьте все другие действия, которые вы хотите выполнить в рабочем процессе.

1. Сохраните приложение логики, когда закончите.

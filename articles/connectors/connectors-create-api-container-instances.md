# <a name="deploy-and-manage-azure-container-instances-by-using-azure-logic-apps"></a>Развертывание и управление экземплярами контейнеров Azure с помощью приложений логики Azure

С помощью приложений Azure Logic Apps и разъема Azure Container Instance можно настроить автоматизированные задачи и рабочие процессы, которые развертывают и управляют [контейнерными группами.](../container-instances/container-instances-container-groups.md) Разъем контейнерной инстанции поддерживает следующие действия:

* Создание или удаление группы контейнеров
* Получить свойства контейнерной группы
* Получить список контейнерных групп
* Получить журналы экземпляра контейнера

Используйте эти действия в логических приложениях для таких задач, как выполнение рабочей нагрузки контейнера в ответ на триггер Logic Apps. Можно также использовать другие действия, применяя выход из действий Контейнерной инстанции. 

Этот разъем предоставляет только действия, поэтому для запуска приложения логики используйте отдельный триггер, например триггер **Recurrence** для выполнения рабочей нагрузки контейнера в обычном графике. Или может потребоваться запустить развертывание контейнерной группы после события, такого как прибытие сообщения электронной почты Outlook. 

Если вы новичок в логических приложениях, просмотрите [Что такое приложения логики Azure?](../logic-apps/logic-apps-overview.md)

## <a name="prerequisites"></a>Предварительные требования

* Подписка Azure. Если у вас еще нет подписки Azure, [зарегистрируйтесь для получения бесплатной учетной записи Azure](https://azure.microsoft.com/free/). 

* Базовые знания о [том, как создавать логические приложения](../logic-apps/quickstart-create-first-logic-app-workflow.md) и [как создавать и управлять экземплярами контейнеров](../container-instances/container-instances-quickstart.md)

* Приложение логики, где вы хотите получить доступ к экземплярам контейнера. Чтобы использовать действие, запустите приложение логики с другим триггером, например, триггером **Recurrence.**

## <a name="add-a-container-instance-action"></a>Добавление действия инстанции контейнера

[!INCLUDE [Create connection general intro](../../includes/connectors-create-connection-general-intro.md)]

1. Войдите на [портал Azure](https://portal.azure.com) и откройте свое приложение логики в конструкторе приложений логики, если оно еще не открыто.

1. Выберите путь: 

   * На последнем шаге, в котором необходимо добавить действие, выберите **Новый шаг**. 

     -или-

   * Между шагами, в которых вы хотите добавить действие, переместите указатель на стрелку между шагами. 
   Выберите знак**+** плюс (), который появляется, а затем выберите **Добавить действие.**

1. В поле поиска введите "контейнерный экземпляр" в качестве фильтра. В списке действий выберите захотечное действие разъема azure Container Instance.

1. Укажите имя подключения. 

1. Укажите необходимые сведения для выбранного действия и продолжайте создание рабочего процесса приложения логики.

  Например, выберите **Создать группу контейнеров** и ввести свойства для группы контейнеров и одного или нескольких экземпляров контейнера в группе, как показано на следующем изображении (частичная деталь):

  ![Создание группы контейнеров](./media/connectors-create-api-container-instances/logic-apps-aci-connector.png)

# <a name="manage-email-calendars-and-contacts-in-outlookcom-by-using-azure-logic-apps"></a>Управление электронной почтой, календарями и контактами в Outlook.com с помощью приложений логики Azure

С [помощью приложений Azure Logic Apps](../logic-apps/logic-apps-overview.md) и Outlook.com [разъема](/connectors/outlook/)можно создавать автоматизированные задачи и рабочие процессы, которые управляют вашей @outlook.com учетной записью путем @hotmail.com создания логических приложений. Например, вы автоматизировать следующие задачи:

* Получить, отправить и ответить на электронную почту.
* Расписание встреч в календаре.
* Добавляйте и отсваивайте контакты.

Можно использовать любой триггер для запуска рабочего процесса, например, при поступлении нового письма, при обновлении элемента календаря или при возникновении события в службе различий. Можно использовать действия, которые реагируют на событие триггера, например, отправлять электронное письмо или создавать новое событие календаря.

> [!NOTE]
> Для автоматизации задач для рабочей @fabrikam.onmicrosoft.comучетной записи Майкрософт, таких как, используйте [разъем Office 365 Outlook.](../connectors/connectors-create-api-office365-outlook.md)

## <a name="prerequisites"></a>Предварительные требования

* Учетная [запись Outlook.com](https://outlook.live.com/owa/)

* Подписка Azure. Если у вас еще нет подписки Azure, [зарегистрируйтесь для получения бесплатной учетной записи Azure](https://azure.microsoft.com/free/). 

* Приложение логики, из которого необходимо получить доступ к учетной записи Outlook.com. Чтобы начать рабочий процесс с Outlook.com триггера, необходимо иметь [пустое приложение логики.](../logic-apps/quickstart-create-first-logic-app-workflow.md) Чтобы добавить Outlook.com действие в рабочий процесс, приложение логики должно уже иметь триггер.

## <a name="add-a-trigger"></a>Добавление триггера

[Триггер](../logic-apps/logic-apps-overview.md#logic-app-concepts) — это событие, которое запускает рабочий процесс в приложении логики. В этом примере логическое приложение использует триггер "опрос", который проверяет любую новую электронную почту в вашей учетной записи электронной почты, на основе указанного интервала и частоты.

1. На [портале Azure](https://portal.azure.com)откройте пустое логическое приложение в Logic App Designer.

1. В поле поиска введите outlook.com в качестве фильтра. В этом примере выберите **при поступления нового письма.**

1. Если вам было предложено войти в систему, предоставьте свои Outlook.com учетные данные, чтобы ваше приложение логики могли подключиться к вашей учетной записи. В противном случае, если соединение уже существует, предоставьте информацию для свойств триггера:

1. В триггере установите значения **частоты** и **интервала.**

   Например, если вы хотите, чтобы триггер опрашивал каждые 15 минут, установите **частоту** на **минуту**и установите **интервал** до **15.**

1. На панели инструментов дизайнера выберите **Сохранить**, который сохраняет вашу логику приложение.

Чтобы ответить на спусковой крючок, добавьте еще одно действие. Например, можно добавить действие Twilio **Send message,** которое отправляет текст по прибытии письма.

## <a name="add-an-action"></a>Добавление действия

[Действие](../logic-apps/logic-apps-overview.md#logic-app-concepts) — это операция, запущенная рабочим процессом в приложении логики. Этот пример логики приложение отправляет письмо с вашей учетной записи Outlook.com. Для заполнения действия можно использовать вывод из другого триггера. Например, предположим, что приложение логики использует триггер SalesForce **When an object is created** trigger. Вы можете добавить Outlook.com Отправить действие **по электронной почте** и использовать выходы из триггера SalesForce в электронной почте.

1. На [портале Azure](https://portal.azure.com)откройте логическое приложение в Logic App Designer.

1. Чтобы добавить действие в качестве последнего шага в рабочем процессе, выберите **Новый шаг.** 

   Чтобы добавить действие между шагами, переместите указатель на стрелку между этими шагами. Выберите знак**+** плюс (), который появляется, а затем выберите **Добавить действие.**

1. В поле поиска введите "outlook.com" в качестве фильтра. В этом примере выберите **Отправить электронное письмо.** 

1. Если вам было предложено войти в систему, предоставьте свои Outlook.com учетные данные, чтобы ваше приложение логики могли подключиться к вашей учетной записи. В противном случае, если подключение уже существует, предоставьте информацию для свойств действия.

1. На панели инструментов дизайнера выберите **Сохранить**, который сохраняет вашу логику приложение.

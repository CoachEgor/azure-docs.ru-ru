# <a name="user-flow-versions-in-azure-active-directory-b2c"></a>Версии потоков пользователей в Azure Active Directory B2C

Потоки пользователей в Azure Active Directory B2C (Azure AD B2C) помогают настроить общие [политики,](user-flow-overview.md) полностью описывающие взаимодействие с клиентами. Эти возможности взаимодействия включают в себя вход в систему, регистрацию, сброс пароля и изменение профиля. В Azure AD B2C можно выбирать из коллекции как рекомендуемых потоков пользователей, так и потоков пользователей предварительной версии.

Новые потоки пользователей добавляются как новые версии. Когда потоки пользователей стабилизируются, они становятся рекомендуемыми к использованию. Потоки пользователей помечаются как **рекомендуемые**, если они были тщательно протестированы. До того времени они будут считаться потоками в предварительной версии. Используйте рекомендуемые потоки пользователей для любого рабочего приложения и выбирайте другие версии при тестировании новой доступной возможности. Не следует использовать более старые версии рекомендуемых потоков пользователей.

>[!IMPORTANT]
> Если поток пользователя не идентифицирован как **Рекомендуемый,** он считается *в предварительном просмотре*. Для производственных приложений следует использовать только рекомендуемые потоки пользователей.

| Вопиюте в использовании ROPC | нет | Позволяет пользователю с локальной учетной записью напрямую входить в собственные приложения (браузер не требуется). | Вход | нет | Позволяет пользователю входить в свою учетную запись. Используя этот поток пользователя, можно настроить следующее: <ul><li>[Многофакторная аутентификация](custom-policy-multi-factor-authentication.md)</li><li>[Срок службы токенов](tokens-overview.md)</li><li>Параметры маркеров совместимости</li><li>Режим работы сеанса</li><li>Блокирование входа</li><li>Принудительный сброс пароля</li><li>Возможность оставаться в системе</ul><br>С помощью этого потока пользователя нельзя настроить пользовательский интерфейс. |

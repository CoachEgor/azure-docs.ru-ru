# <a name="billing-model-for-azure-active-directory-b2c"></a>Модель выставления счетов для Azure Active Directory B2C

Использование Azure Active Directory B2C (Azure AD B2C) выставлено на счет связанной подписки Azure и использует ежемесячную модель выставления счетов для активных пользователей (MAU). Узнайте, как связать ресурс Azure AD B2C с подпиской и как работает модель выставления счетов MAU в следующих разделах.

> [!IMPORTANT]
> Данная статья не содержит информации о ценах. Для получения последней информации об использовании биллинга и ценообразования, см [Azure Активный каталог B2C ценообразования](https://azure.microsoft.com/pricing/details/active-directory-b2c/).

## <a name="monthly-active-users-mau-billing"></a>Ежемесячные активные пользователи (MAU) биллинг

Счет azure AD B2C учитывается на счету уникальных пользователей с активностью аутентификации в течение календарного месяца, известного как ежемесячный биллинг активных пользователей (MAU).

С **01 ноября 2019**года все вновь созданные арендаторы Azure AD B2C выставляют счета за ежемесячно активных пользователей (MAU). Существующие арендаторы, [которые связаны с подпиской](#link-an-azure-ad-b2c-tenant-to-a-subscription) на или после 01 ноября 2019 года, будут выставляться на ежемесячной основе активным пользователям (MAU).

Если у вас есть существующий клиент Azure AD B2C, который был подключен к подписке до 01 ноября 2019 года, вы можете сделать один из следующих:

* Обновление до ежемесячной модели выставления счетов активным пользователям (MAU) или
* Оставайтесь на модели выставления счетов за аутентификацию

### <a name="upgrade-to-monthly-active-users-billing-model"></a>Обновление до ежемесячной модели выставления счетов активным пользователям

Владельцы подписок Azure, которые имеют административный доступ к ресурсу Azure AD B2C, могут перейти на модель биллинга MAU. Параметры выставления счетов настроены в ресурсе Azure AD B2C.

Переход на ежемесячные активные пользователи (MAU) биллинга **является необратимым.** После преобразования ресурса Azure AD B2C в модель выставления счетов на основе МАУ, вы не сможете вернуть этот ресурс в модель выставления счетов за проверку подлинности.

Вот как сделать переход на mau биллинг для существующего ресурса Azure AD B2C:

1. Вопиюсь на [портал Azure](https://portal.azure.com) в качестве владельца подписки.
1. Выберите фильтр **подписки каталога в** верхнем меню, а затем выберите каталог Azure AD B2C, который вы хотите обновить до выставления счетов МАУ.<br/>
    ![Фильтр каталога и подписки на портале Azure](./media/billing/portal-mau-01-select-b2c-directory.png)
1. В меню слева выберите **Azure AD B2C**. Либо щелкните **Все службы**, а затем найдите и выберите **Azure AD B2C**
1. На странице **Обзор** клиента Azure AD B2C выберите ссылку под **именем Ресурса**. Вы направляетесь на ресурс Azure AD B2C в вашем арендаторе Azure AD.<br/>
    ![Ссылка ресурсов Azure AD B2C выделена на портале Azure](./media/billing/portal-mau-02-b2c-resource-link.png)
1. На странице **Обзор** ресурса Azure AD B2C под **платежными единицами**выберите ссылку **Per Authentication (Изменение на MAU).**<br/>
    ![Изменение ссылки MAU выделено на портале Azure](./media/billing/portal-mau-03-change-to-mau-link.png)
1. Выберите **Подтверждение** для завершения обновления до mau биллинга.<br/>
    ![Диалог подтверждения биллинга на основе MAU на портале Azure](./media/billing/portal-mau-04-confirm-change-to-mau.png)

### <a name="what-to-expect-when-you-transition-to-mau-billing-from-per-authentication-billing"></a>Чего ожидать при переходе на mau биллинга от выставления счетов за аутентификацию

Измерение на основе MAU включено, как только вы, владелец подписки/ресурса, подтверждаете изменение. Ваш ежемесячный счет будет отражать единицы аутентификации, выставленные счета до изменения, и новые единицы МАУ, начиная с изменения.

Пользователи не учитываются дважды в течение переходного месяца. Уникальные активные пользователи, которые аутентифицируют подлинность до изменения, взимаются за процент проверки подлинности в календарном месяце. Те же пользователи не включены в расчет MAU на оставшуюся часть цикла выставления счетов подписки. Пример:

* Арендатор Contoso B2C имеет 1000 пользователей. 250 пользователей активны в любой данный месяц. Администратор подписки меняется от аутентификации к ежемесячным активным пользователям (MAU) 10 числа месяца.
* Счет за 1-10-й счет выставляется с использованием модели проверки подлинности.
  * Если 100 пользователей входят в систему в течение этого периода (1-10-й), эти пользователи помечены как *оплаченные за месяц.*
* Счет с 10-го (эффективное время перехода) выставляется по ставке МАУ.
  * Если в этот период заходят еще 150 пользователей (10-30-е место), выставляется только 150 дополнительных.
  * Продолжающаяся активность первых 100 пользователей не влияет на выставление счетов на оставшуюся часть календарного месяца.

В период выставления счетов переходного периода владелец подписки, скорее всего, увидит записи для обоих методов (за аутентификацию и per-MAU), которые отображаются в выписке по выставлению счетов по подписке Azure:

* Запись для использования до даты/времени изменения, отражающего проверку подлинности.
* Запись для использования после изменения, которое отражает ежемесячных активных пользователей (MAU).

Для получения последней информации об использовании биллинга и цены на Azure AD B2C, см [Azure Active Directory B2C ценообразования](https://azure.microsoft.com/pricing/details/active-directory-b2c/).

## <a name="link-an-azure-ad-b2c-tenant-to-a-subscription"></a>Связь с арендатором Azure AD B2C с подпиской

Плата за использование для Azure Active Directory B2C (Azure AD B2C) выставлена на подписку Azure. Администратор созданного клиента Azure AD B2C должен явным образом связать созданный клиент Azure AD B2C с подпиской Azure.

Ссылка на подписку достигается путем создания *ресурса* Azure AD B2C в рамках целевой подписки Azure. Несколько ресурсов Azure AD B2C могут быть созданы в одной подписке Azure, а также в других ресурсах Azure, таких как виртуальные машины, учетные записи хранения данных и логические приложения. Все ресурсы в подписке можно увидеть, перейдя в наниматель Azure Active Directory (Azure AD), с которым связана подписка.

Подписка, связанная с арендатором Azure AD B2C, может использоваться для выставления счетов за использование Azure AD B2C или других ресурсов Azure, включая дополнительные ресурсы Azure AD B2C. Она не позволяет добавлять другие лицензируемые службы Azure или лицензии Office 365 в клиенте Azure AD B2C.

### <a name="prerequisites"></a>Предварительные требования

* [Подписка Azure.](https://azure.microsoft.com/free/)
* [Арендатор Azure AD B2C,](tutorial-create-tenant.md) который требуется связать с подпиской
  * Вы должны быть администратором-арендатором
  * Арендатор не должен быть уже связан с подпиской

### <a name="create-the-link"></a>Создание ссылки

1. Войдите на [портал Azure](https://portal.azure.com).
1. Выберите фильтр **подписки каталога** в верхнем меню, а затем выберите каталог, содержащий подписку Azure, которую вы хотели бы использовать *(а не* каталог, содержащий арендатор Azure AD B2C).
1. Выберите **Создать ресурс,** введите `Active Directory B2C` поле **поиска Marketplace,** а затем выберите **Azure Active Directory B2C.**
1. Выберите **Создать**
1. Выберите **ссылку на существующую подписку Azure AD B2C на мою подписку Azure.**
1. Выберите **арендатора Azure AD B2C** из выпадения. Отображаются только арендаторы, для которых вы являетесь глобальным администратором и которые еще не связаны с подпиской. Поле **имени ресурса Azure AD B2C** населено доменным именем выбранного вами арендатора Azure AD B2C.
1. Выберите активную **подписку** Azure, администратором которой вы являетесь.
1. В **группе ресурсов**выберите **Создать новое,** а затем указать **местоположение группы ресурсов.** Настройки группы ресурсов не влияют на местоположение, производительность или состояние выставления счетов на место расположение, производительность или платежное положение группы Ресурсов AD B2C.
1. Выберите **Создать**.
    ![Страница создания ресурсов Azure AD B2C на портале Azure](./media/billing/portal-01-create-b2c-resource-page.png)

После завершения этих шагов для арендатора Azure AD B2C ваша подписка Azure выставляется в соответствии с вашими сведениями о прямом или корпоративном соглашении Azure Direct или Enterprise, если это применимо.

### <a name="manage-your-azure-ad-b2c-tenant-resources"></a>Управление ресурсами клиента Azure AD B2C

После создания ресурса Azure AD B2C в подписке Azure следует увидеть новый ресурс типа "Арендатор B2C" с другими ресурсами Azure.

Этот ресурс можно использовать в следующих целях:

* Перейдите к подписке для просмотра информации о выставлении счетов
* Получите идентификатор арендатора Azure AD B2C в формате GUID
* для перехода к клиенту Azure AD B2C;
* Отправить запрос на техническую поддержку.
* Переместите ресурс-арендатор Azure AD B2C в другую группу подписки или ресурса Azure

![Страница настроек ресурсов B2C на портале Azure](./media/billing/portal-02-b2c-resource-overview.png)

Если в подписке установлены региональные ограничения для создания ресурсов Azure, это ограничение может помешать вам создать ресурс Azure AD B2C.

Чтобы смягчить эту проблему, ослабьте региональные ограничения.

## <a name="azure-cloud-solution-providers-csp-subscriptions"></a>Подписки поставщиков облачных решений Azure (CSP)

Подписки поставщиков облачных решений (CSP) Azure поддерживаются в Azure AD B2C. Эта функция доступна при использовании API или портала Azure для Azure AD B2C и для всех ресурсов Azure. Администраторы подписки CSP могут связывать, перемещать и удалять отношения с Azure AD B2C, как это делается с другими ресурсами Azure.

Администрирование Azure AD B2C с помощью управления доступом на основе ролей не зависит от связи между клиентом Azure AD B2C и подпиской Azure CSP. Контроль доступа на основе ролей достигается с помощью ролей, основанных на клиентах, а не ролей, основанных на подписке.

## <a name="change-the-azure-ad-b2c-tenant-billing-subscription"></a>Измените подписку на выставление счетов-арендаторов Azure AD B2C

Арендаторы Azure AD B2C могут быть переведены на другую подписку, если подписка на источник и пункт назначения существует в пределах того же арендатора Active Directory Azure.

Чтобы узнать, как переместить ресурсы Azure, такие как ваш клиент Azure AD B2C, в другую подписку, [см. Перемещение ресурсов в новую группу ресурсов или подписку.](../azure-resource-manager/management/move-resource-group-and-subscription.md)

Прежде чем инициировать переезд, обязательно прочтите всю статью, чтобы полностью понять ограничения и требования к такому шагу. В дополнение к инструкциям по перемещению ресурсов, она включает в себя важную информацию, как предварительный контрольный список перемещения и как проверить операцию перемещения.

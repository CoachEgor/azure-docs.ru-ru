# <a name="tutorial-create-user-flows-in-azure-active-directory-b2c"></a>Руководство по Создание потоков пользователей в Azure Active Directory B2C

В приложениях может быть [маршруты пользователей](active-directory-b2c-reference-policies.md) , благодаря которым пользователи регистрации, входа или управлению профилем. В своем клиенте Azure Active Directory (Azure AD) B2C вы можете создать несколько потоков пользователей различных типов и при необходимости использовать их в приложениях. Потоки пользователей могут повторно использоваться в разных приложениях.

В этой статье раскрываются следующие темы:

> [!div class="checklist"]
> * Создание потока пользователя для регистрации и входа в систему
> * Создание потока пользователя изменения профиля
> * Создание потока пользователя сброса пароля

В этом руководстве показано, как создать несколько рекомендуемых потоков пользователей с помощью портала Azure. Если вы ищете информацию о том, как настроить учетные данные пароля владельца ресурса (ROPC) потока в приложении, см. в разделе [настройка потока учетные данные пароля владельца ресурса в Azure AD B2C](configure-ropc.md).

Если у вас еще нет подписки Azure, [создайте бесплатную учетную запись Azure](https://azure.microsoft.com/free/?WT.mc_id=A261C142F), прежде чем начинать работу.

## <a name="prerequisites"></a>Технические условия

[Зарегистрируйте приложения](tutorial-register-applications.md), являющиеся частью потоков пользователей, которые вы хотите создать.

## <a name="create-a-sign-up-and-sign-in-user-flow"></a>Создание потока пользователя для регистрации и входа в систему

Потоки пользователей регистрации и входа регулируют процедуры регистрации и входа в систему в рамках одной конфигурации. Пользователи вашего приложения проходят через соответствующую процедуру в зависимости от контекста.

1. Войдите на [портале Azure](https://portal.azure.com).
1. Убедитесь, что используете каталог, содержащий клиент Azure AD B2C, щелкнув **Фильтр каталога и подписки** в верхнем меню и выбрав каталог, содержащий ваш клиент.

    ![Переключение на каталог подписки](./media/tutorial-create-user-flows/switch-directories.PNG)

1. Выберите **Все службы** в левом верхнем углу окна портала Azure, а затем найдите и выберите **Azure AD B2C**.
1. В меню слева в разделе **политики**выберите **маршруты пользователей (политики)** , а затем выберите **новый маршрут**.

    ![Выберите "Новый поток пользователя"](./media/tutorial-create-user-flows/signup-signin-user-flow.png)

1. На **рекомендуется** выберите **вход и вход** поток пользователя.

    ![Выбор потока пользователя регистрации и входа](./media/tutorial-create-user-flows/signup-signin-type.png)

1. Введите **имя** потока пользователя. Например, *signupsignin1*.
1. В разделе **Поставщики удостоверений** выберите **Регистрация по электронной почте**.

    ![Установка свойств потока](./media/tutorial-create-user-flows/signup-signin-properties.png)

1. В разделе **Атрибуты пользователя и утверждения** выберите утверждения и атрибуты, которые вы хотите собирать и отправлять от пользователя во время регистрации. К примеру, выберите **Показать больше**, а затем выберите атрибуты и утверждения для **страны или региона**, **отображаемое имя**, и **почтовый**. Последовательно выберите **ОК**.

    ![Выбор атрибутов и утверждений](./media/tutorial-create-user-flows/signup-signin-attributes.png)

1. Для добавления потока пользователя щелкните **Создать**. Префикс *B2C_1* автоматически добавляется к имени.

### <a name="test-the-user-flow"></a>Тестирование потока пользователя

1. Выберите поток пользователя, который вы создали, чтобы открыть страницу обзора и выберите команду **выполнения потока пользователя**.
1. В разделе **Приложение** выберите зарегистрированное ранее веб-приложение с именем *webapp1*. В поле **URL-адрес ответа** должно содержаться значение `https://jwt.ms`.
1. Щелкните **Выполнить поток пользователя**, а затем выберите **Зарегистрироваться сейчас**.

    ![Запуск потока пользователя](./media/tutorial-create-user-flows/signup-signin-run-now.PNG)

1. Введите допустимый адрес электронной почты, нажмите кнопку **отправить проверочный код**, введите код проверки, который можно получить, а затем выберите **проверки кода**.
1. Введите новый пароль и подтвердите его.
1. Выберите вашу страну и регион, введите имя, которое будет отображаться, введите почтовый индекс и нажмите кнопку **создать**. Токен должен вернуться по адресу `https://jwt.ms` и отобразиться пользователю.
1. Теперь вы снова можете запустить поток пользователя и на этот раз войти в учетную запись, которую создали. Возвращаемый маркер содержит утверждения, которые вы выбрали страны или региона, имя и почтовый индекс.

## <a name="create-a-profile-editing-user-flow"></a>Создание потока пользователя изменения профиля

Чтобы добавить для пользователей приложения возможность изменять профиль, используйте поток пользователя изменения профиля.

1. В меню слева на странице Общие сведения о клиенте Azure AD B2C выберите **маршруты пользователей (политики)** , а затем выберите **новый маршрут**.
1. Выберите поток пользователя **Profile editing** (Изменение профиля) на вкладке "Рекомендуется".
1. Введите **имя** потока пользователя. Например, *profileediting1*.
1. Щелкните **Поставщики удостоверений** и выберите **Local Account SignIn** (Вход с локальной учетной записью).
1. В разделе **Атрибуты пользователя** выберите атрибуты, которые клиенты будут изменять в своих профилях. Например, выберите **Показать больше**, а затем выберите атрибуты и утверждения для **отображаемое имя** и **должность**. Последовательно выберите **ОК**.
1. Для добавления потока пользователя щелкните **Создать**. Префикс *B2C_1* автоматически добавляется к имени.

### <a name="test-the-user-flow"></a>Тестирование потока пользователя

1. Выберите поток пользователя, который вы создали, чтобы открыть страницу обзора и выберите команду **выполнения потока пользователя**.
1. В разделе **Приложение** выберите зарегистрированное ранее веб-приложение с именем *webapp1*. В поле **URL-адрес ответа** должно содержаться значение `https://jwt.ms`.
1. Щелкните **Выполнить поток пользователя** и выполните вход в учетную запись, созданную ранее.
1. Теперь вы как пользователь можете изменять отображаемое имя и должность. Нажмите кнопку **Продолжить**. Токен должен вернуться по адресу `https://jwt.ms` и отобразиться пользователю.

## <a name="create-a-password-reset-user-flow"></a>Создание потока пользователя сброса пароля

Чтобы предоставить пользователям приложения для сброса пароля, используйте потока пользователя сброса пароля.

1. В меню слева выберите **маршруты пользователей (политики)** , а затем выберите **новый маршрут**.
1. Выберите поток пользователя **Password reset** (Сброс пароля) на вкладке "Рекомендуется".
1. Введите **имя** потока пользователя. Например, *passwordreset1*.
1. В разделе **Поставщики удостоверений** включите **Reset password using email address** (Сброс пароля с использованием адреса электронной почты).
1. В разделе "Утверждения приложений" щелкните **Показать больше** и выберите утверждения, которые необходимо вернуть в токенах проверки подлинности, отправляемых приложению. Например, выберите **ИД объекта пользователя**.
1. Последовательно выберите **ОК**.
1. Для добавления потока пользователя щелкните **Создать**. Префикс *B2C_1* автоматически добавляется к имени.

### <a name="test-the-user-flow"></a>Тестирование потока пользователя

1. Выберите поток пользователя, который вы создали, чтобы открыть страницу обзора и выберите команду **выполнения потока пользователя**.
1. В разделе **Приложение** выберите зарегистрированное ранее веб-приложение с именем *webapp1*. В поле **URL-адрес ответа** должно содержаться значение `https://jwt.ms`.
1. Нажмите кнопку **выполнения потока пользователя**, проверьте адрес электронной почты учетной записи, который вы ранее создали и выберите **Продолжить**.
1. Теперь вы как пользователь сможете изменять пароль. Сменить пароль и выберите **Продолжить**. Токен должен вернуться по адресу `https://jwt.ms` и отобразиться пользователю.

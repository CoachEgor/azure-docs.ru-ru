# <a name="tutorial-create-user-flows-in-azure-active-directory-b2c"></a>Учебник: Создание потоков пользователей в Azure Active Directory B2C

В приложениях могут быть [потоки пользователей,](user-flow-overview.md) которые позволяют пользователям зарегистрироваться, войти в систему или управлять своим профилем. Можно создать несколько пользовательских потоков различных типов в вашем наемщике Azure Active Directory B2C (Azure AD B2C) и использовать их в приложениях по мере необходимости. Потоки пользователей могут повторно использоваться в разных приложениях.

Вы узнаете, как выполнять следующие задачи:

> [!div class="checklist"]
> * Создание потока пользователя для регистрации и входа в систему
> * Создание потока пользователя изменения профиля
> * Создание потока пользователя сброса пароля

В этом руководстве показано, как создать несколько рекомендуемых потоков пользователей с помощью портала Azure. Если вы ищете информацию о том, как настроить поток учетных данных паролей владельца ресурса (ROPC) в вашем приложении, [см.](configure-ropc.md)

Если у вас еще нет подписки Azure, [создайте бесплатную учетную запись](https://azure.microsoft.com/free/?WT.mc_id=A261C142F), прежде чем начинать работу.

## <a name="prerequisites"></a>Предварительные требования

[Зарегистрируйте приложения](tutorial-register-applications.md), являющиеся частью потоков пользователей, которые вы хотите создать.

## <a name="create-a-sign-up-and-sign-in-user-flow"></a>Создание потока пользователя для регистрации и входа в систему

Потоки пользователей регистрации и входа регулируют процедуры регистрации и входа в систему в рамках одной конфигурации. Пользователи вашего приложения проходят через соответствующую процедуру в зависимости от контекста.

1. Войдите на [портал Azure](https://portal.azure.com).
1. Выберите **значок каталога и подписки** в панели инструментов портала, а затем выберите каталог, содержащий ваш клиент Azure AD B2C.

    ![Арендатор B2C, каталог и панель подписки, портал Azure](./media/tutorial-create-user-flows/directory-subscription-pane.png)

1. На портале Azure ищите **Azure AD B2C.**
1. В соответствии с **политиками**выберите **потоки пользователей (политики),** а затем выберите **новый поток пользователя.**

    ![Страница потоков пользователей на портале с выделенной кнопкой «Новый поток»](./media/tutorial-create-user-flows/signup-signin-user-flow.png)

1. На **вкладке «Рекомендуемый»** выберите **регистрацию и вопийте в** потоке пользователя.

    ![Выберите страницу потока пользователя с подрегистривай и впишитесь в выделенный поток](./media/tutorial-create-user-flows/signup-signin-type.png)

1. Введите **имя** для потока пользователя. Например, *signupsignin1*.
1. В разделе **Поставщики удостоверений** выберите **Регистрация по электронной почте**.

    ![Создание страницы потока пользователей на портале Azure с выделенными свойствами](./media/tutorial-create-user-flows/signup-signin-properties.png)

1. В разделе **Атрибуты пользователя и утверждения** выберите утверждения и атрибуты, которые вы хотите собирать и отправлять от пользователя во время регистрации. Например, выберите **Показать больше,** а затем выбрать атрибуты и претензии для **страны / региона,** **Название дисплея,** и **почтовый индекс.** Нажмите кнопку **ОК**.

    ![Страница выбора атрибутов и претензий с тремя выбранными претензиями](./media/tutorial-create-user-flows/signup-signin-attributes.png)

1. Для добавления потока пользователя щелкните **Создать**. Префикс *B2C_1* автоматически добавляется к имени.

### <a name="test-the-user-flow"></a>Тестирование потока пользователя

1. Выберите созданный поток пользователя, чтобы открыть страницу обзора, а затем выберите **поток пользователя Run.**
1. В разделе **Приложение** выберите зарегистрированное ранее веб-приложение с именем *webapp1*. В поле **URL-адрес ответа** должно содержаться значение `https://jwt.ms`.
1. Щелкните **Выполнить поток пользователя**, а затем выберите **Зарегистрироваться сейчас**.

    ![Выполнить страницу потока пользователя на портале с выделенной кнопкой Run user flow](./media/tutorial-create-user-flows/signup-signin-run-now.PNG)

1. Введите действительный адрес электронной почты, нажмите **Отправить код проверки,** введите код проверки, который вы получаете, а затем выберите **Код проверки**.
1. Введите новый пароль и подтвердите его.
1. Выберите свою страну и регион, введите имя, которое вы хотите отобразить, введите почтовый индекс, а затем нажмите **Создать**. Токен должен вернуться по адресу `https://jwt.ms` и отобразиться пользователю.
1. Теперь вы снова можете запустить поток пользователя и на этот раз войти в учетную запись, которую создали. Возвратный токен включает в себя утверждения, выбранные страной/регионом, именем и почтовым индексом.

## <a name="create-a-profile-editing-user-flow"></a>Создание потока пользователя изменения профиля

Чтобы добавить для пользователей приложения возможность изменять профиль, используйте поток пользователя изменения профиля.

1. В меню страницы обзора арендатора Azure AD B2C выберите **потоки пользователей (политики),** а затем выберите **новый поток пользователя.**
1. Выберите поток **пользователя для редактирования профиля** на вкладке **«Рекомендуемый».**
1. Введите **имя** для потока пользователя. Например, *profileediting1*.
1. Щелкните **Поставщики удостоверений** и выберите **Local Account SignIn** (Вход с локальной учетной записью).
1. В разделе **Атрибуты пользователя** выберите атрибуты, которые клиенты будут изменять в своих профилях. Например, выберите **Показать больше,** а затем выбрать как атрибуты и претензии для **названия дисплея** и **название работы.** Нажмите кнопку **ОК**.
1. Для добавления потока пользователя щелкните **Создать**. Префикс *B2C_1* автоматически добавляется к имени.

### <a name="test-the-user-flow"></a>Тестирование потока пользователя

1. Выберите созданный поток пользователя, чтобы открыть страницу обзора, а затем выберите **поток пользователя Run.**
1. В разделе **Приложение** выберите зарегистрированное ранее веб-приложение с именем *webapp1*. В поле **URL-адрес ответа** должно содержаться значение `https://jwt.ms`.
1. Щелкните **Выполнить поток пользователя** и выполните вход в учетную запись, созданную ранее.
1. Теперь вы как пользователь можете изменять отображаемое имя и должность. Нажмите кнопку **Продолжить**. Токен должен вернуться по адресу `https://jwt.ms` и отобразиться пользователю.

## <a name="create-a-password-reset-user-flow"></a>Создание потока пользователя сброса пароля

Чтобы пользователи приложения могли сбросить свой пароль, вы используете поток пользователя сготовона пароля.

1. В меню обзора для арендаторов Azure AD B2C выберите **потоки пользователей (политики),** а затем выберите **новый поток пользователя.**
1. Выберите поток пользователя **сбросить пароль** на **рекомендуемой** вкладке.
1. Введите **имя** для потока пользователя. Например, *passwordreset1*.
1. В разделе **Поставщики удостоверений** включите **Reset password using email address** (Сброс пароля с использованием адреса электронной почты).
1. В разделе "Утверждения приложений" щелкните **Показать больше** и выберите утверждения, которые необходимо вернуть в токенах проверки подлинности, отправляемых приложению. Например, выберите **ИД объекта пользователя**.
1. Нажмите кнопку **ОК**.
1. Для добавления потока пользователя щелкните **Создать**. Префикс *B2C_1* автоматически добавляется к имени.

### <a name="test-the-user-flow"></a>Тестирование потока пользователя

1. Выберите созданный поток пользователя, чтобы открыть страницу обзора, а затем выберите **поток пользователя Run.**
1. В разделе **Приложение** выберите зарегистрированное ранее веб-приложение с именем *webapp1*. В поле **URL-адрес ответа** должно содержаться значение `https://jwt.ms`.
1. Нажмите **Выполнить поток пользователей,** проверить адрес электронной почты учетной записи, которую вы ранее создали, и выберите **Продолжить**.
1. Теперь вы как пользователь сможете изменять пароль. Измените пароль и выберите **Продолжить**. Токен должен вернуться по адресу `https://jwt.ms` и отобразиться пользователю.

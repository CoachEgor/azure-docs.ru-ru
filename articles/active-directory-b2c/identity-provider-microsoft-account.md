# <a name="set-up-sign-up-and-sign-in-with-a-microsoft-account-using-azure-active-directory-b2c"></a>Настройка регистрации и входа с учетной записью Microsoft через Azure Active Directory B2C

## <a name="create-a-microsoft-account-application"></a>Создание приложения для учетной записи Майкрософт

Чтобы использовать учетную запись Майкрософт в качестве [поставщика идентификационных данных](openid-connect.md) в Azure Active Directory B2C (Azure AD B2C), необходимо создать приложение в аниматоре Azure AD. Клиент Azure AD отличается от клиента Azure AD B2C. Если у вас еще нет учетной записи [https://www.live.com/](https://www.live.com/)Майкрософт, вы можете получить ее по адресу .

1. Войдите на [портал Azure](https://portal.azure.com).
1. Убедитесь, что вы используете каталог, содержащий ваш клиент Azure AD, выбрав фильтр **подписки каталога в** верхнем меню и выбрав каталог, содержащий арендатор Azure AD.
1. Выберите **Все службы** в левом верхнем углу окна портала Azure, а затем найдите и выберите **Регистрация приложений**.
1. Выберите **Новая регистрация**.
1. Введите значение **Name** (Имя) для приложения. Например, *MSAapp1*.
1. Под **типами поддерживаемых учетных записей**выберите **учетные записи в любом каталоге организаций и личных учетных записей Майкрософт (например, Skype, Xbox, Outlook.com).** Эта опция предназначена для самого широкого набора идентификаторов Майкрософт.

   Для получения дополнительной информации о различных выборах типа учетной записи [см.](../active-directory/develop/quickstart-register-app.md)
1. Под **перенаправлениеuri (необязательно)**, `https://your-tenant-name.b2clogin.com/your-tenant-name.onmicrosoft.com/oauth2/authresp` выберите **веб** и введите в текстовое поле. Замените `your-tenant-name` имя арендатора Azure AD B2C.
1. Выберите **Регистр**
1. Запись **идентификатора приложения (клиента),** показанного на странице Обзор приложения. Это необходимо при настройке поставщика идентификационных данных в следующем разделе.
1. Выберите **сертификаты & секреты**
1. Нажмите **Новый секрет клиента**
1. Введите **описание** секрета, например *пароль приложения 1,* а затем нажмите **Добавить**.
1. Запись пароля приложения, отображаемого в столбце **Значение.** Это необходимо при настройке поставщика идентификационных данных в следующем разделе.

## <a name="configure-a-microsoft-account-as-an-identity-provider"></a>Настройка учетной записи Microsoft в качестве поставщика удостоверений

1. Вопийте на [портале Azure](https://portal.azure.com/) в качестве глобального администратора вашего арендатора Azure AD B2C.
1. Убедитесь, что используете каталог с клиентом Azure AD B2C, выбрав фильтр **Каталог и подписка** в меню вверху и каталог с вашим клиентом.
1. Выберите **Все службы** в левом верхнем углу окна портала Azure, найдите службу **Azure AD B2C** и выберите ее.
1. Выберите **поставщиков идентификационных данных,** а затем выберите **учетную запись Майкрософт.**
1. Введите **имя**. Например, *MSA*.
1. Для **идентификатора клиента**введите идентификатор приложения (клиента) приложения Azure AD, созданного ранее.
1. Для **секрета клиента,** введите секрет клиента, который вы записали.
1. Нажмите кнопку **Сохранить**.

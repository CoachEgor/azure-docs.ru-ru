# <a name="use-the-azure-portal-to-create-and-delete-consumer-users-in-azure-ad-b2c"></a>Используйте портал Azure для создания и удаления потребительских пользователей в Azure AD B2C

Могут быть сценарии, в которых необходимо вручную создавать учетные записи потребителей в каталоге Azure Active Directory B2C (Azure AD B2C). Хотя учетные записи потребителей в каталоге Azure AD B2C чаще всего создаются, когда пользователи регистрируются, чтобы использовать одно из ваших приложений, их можно создавать программно и с помощью портала Azure. В этой статье основное внимание уделяется методу создания и удаления пользователей портала Azure.

Чтобы добавить или удалить пользователей, вашей учетной записи должна быть назначена роль *администратора пользователя* или *Глобальная роль администратора.*

## <a name="types-of-user-accounts"></a>Типы учетных записей пользователей

Как описано в [Обзоре учетных записей пользователей в Azure AD B2C](user-overview.md), есть три типа учетных записей пользователей, которые могут быть созданы в каталоге Azure AD B2C:

* Work
* Гость
* Группы клиентов

В этой статье основное внимание уделяется работе с **учетными записями потребителей** на портале Azure. Для получения информации о создании и удалении учетных записей «Работа» и «Гостевые» [см.](../active-directory/fundamentals/add-users-azure-active-directory.md)

## <a name="create-a-consumer-user"></a>Создание потребителя

1. Войдите на [портал Azure](https://portal.azure.com).
1. Выберите фильтр **Каталог и подписка** в верхнем меню, а затем выберите каталог, содержащий клиент Azure AD B2C.
1. В меню слева выберите **Azure AD B2C**. Либо щелкните **Все службы**, а затем найдите и выберите **Azure AD B2C**
1. В разделе **Управление** выберите **Пользователи**.
1. Выберите **нового пользователя**.
1. Выберите **создать пользователя Azure AD B2C.**
1. Выберите **метод входа и** введите адрес **электронной почты** или имя **пользователя** для нового пользователя. Вывешной знак в выбранном здесь методе должен соответствовать заданной настройке для локального поставщика *идентификационных* данных клиента Azure AD B2C (см. **Поставщики** > **идентификационных данных** ВАшего Клиента AD B2C).
1. Введите **имя** пользователя. Обычно это полное имя (данное и фамилия) пользователя.
1. (Необязательно) Вы можете **заблокировать входе,** если вы хотите отложить возможность для пользователя, чтобы войти в систему. Вы можете включить в систему позже, редактируя **профиль** пользователя на портале Azure.
1. Выберите **пароль автоматического генерации** или **позвольте мне создать пароль.**
1. Укажите **имя** и **фамилию**пользователя.
1. Выберите **Создать**.

Если вы не выбрали **входе в Block,** пользователь теперь может войти в систему, используя указанный вами знак (электронная почта или имя пользователя).

## <a name="delete-a-consumer-user"></a>Удалить потребителя

1. В каталоге Azure AD B2C выберите **пользователей,** а затем выберите пользователя, которого вы хотите удалить.
1. Выберите **Удалить,** а затем **Да,** чтобы подтвердить удаление.

Для получения подробной информации о восстановлении пользователя в течение первых 30 дней после удаления или для постоянного удаления пользователя [см. Восстановление или удаление недавно удаленного пользователя с помощью Azure Active Directory.](../active-directory/fundamentals/active-directory-users-restore.md)

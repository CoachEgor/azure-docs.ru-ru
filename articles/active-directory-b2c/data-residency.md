# <a name="azure-active-directory-b2c-region-availability--data-residency"></a>Azure Active Directory B2C: доступность в регионах и местонахождение данных

Доступность в регионах и местонахождение данных — это два разных понятия, которые применяются к Azure AD B2C не так, как к другим службам Azure. В этой статье разъясняются различия между этими двумя концепциями и сравниваются их применение к Azure против Azure AD B2C.

Azure AD B2C, **как правило, доступен во всем мире** с возможностью **проживания данных** в **США, Европе или Азиатско-Тихоокеанском регионе.**

[Доступность в регионах](#region-availability) определяет то, где служба доступна для использования.

[Местонахождение данных](#data-residency) определяет то, где хранятся данные пользователей.

## <a name="region-availability"></a>Доступность по регионам

Служба Azure AD B2C доступна во всем мире посредством общедоступного облака Azure.

Это отличается от модели, за которой следуют большинство других служб Azure, которые обычно *соединяли доступность* с *резидентством данных.* Примеры этого можно увидеть на странице [Доступность продуктов по регионам](https://azure.microsoft.com/regions/services/) и в [калькуляторе цен Azure Active Directory B2C](https://azure.microsoft.com/pricing/details/active-directory-b2c/).

## <a name="data-residency"></a>Местонахождение данных

Azure AD B2C хранит данные пользователей в США, Европе или Азиатско-Тихоокеанском регионе.

Местопроживания данных определяется по выбранной вами стране/региону при [создании арендатора Azure AD B2C:](tutorial-create-tenant.md)

![Скриншот арендатора предварительного просмотра](./media/data-residency/data-residency-b2c-tenant.png)

Данные находятся в **Европе** по следующим странам/регионам:

> Алжир, Австрия, Азербайджан, Бахрейн, Беларусь, Бельгия, Болгария, Хорватия, Кипр, Чехия, Дания, Египет, Эстония, Финляндия, Франция, Германия, Греция, Венгрия, Исландия, Ирландия, Израиль, Италия, Иордания, Казахстан, Кения, Кувейт, Латвия, Ливан, Лихтенштейн, Литва, Люксембург, Северная Македония, Мальта, Черногория, Марокко, Нидерланды, Нигерия, Норвегия, Оман, Пакистан, Польша, Португалия, Катар, Румыния, Россия, Саудовская Аравия, Сербия, Словакия, Словения, Южная Африка, Испания, Швеция, Швейцария, Тунис, Турция, Украина, Объединенные Арабские Эмираты и Великобритания.

Данные находятся в **Азиатско-Тихоокеанском регионе** по следующим странам/регионам:

Следующие страны/регионы находятся в процессе добавления в список. Пока же вы можете использовать Azure AD B2C, выбрав любую из стран или регионов, приведенных выше.

> Аргентина, Австралия, Бразилия, Чили, Колумбия, Эквадор, Ирак, Новая Зеландия, Парагвай, Перу, Уругвай и Венесуэла.

## <a name="preview-tenant"></a>Предварительная версия клиента

Если вы создали клиента B2C во время предварительного просмотра Azure AD B2C, вполне вероятно, что **тип клиента** говорит, что **арендатор.**

В этом случае необходимо использовать только для разработки и тестирования только для арендатора. НЕ используйте клиента предварительного просмотра для производственных приложений.

**Путь миграции** от клиента Предварительного просмотра B2C до арендатора B2C производственного масштаба. Для производственных приложений необходимо создать нового арендатора B2C.

Известны проблемы, когда вы удаляете клиента предварительного просмотра B2C и создаете арендатора B2C производственного масштаба с тем же доменным именем. *Необходимо создать арендатора B2C производственного масштаба с другим доменным именем.*

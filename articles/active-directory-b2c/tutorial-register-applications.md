# <a name="tutorial-register-an-application-in-azure-active-directory-b2c"></a>Учебник: Зарегистрируйте приложение в Azure Active Directory B2C

Прежде чем [приложения](application-types.md) смогут взаимодействовать с Azure Active Directory B2C (Azure AD B2C), они должны быть зарегистрированы в управляющем клиентом. В этом руководстве показано, как зарегистрировать веб-приложение с помощью портала Azure.

Вы узнаете, как выполнять следующие задачи:

> [!div class="checklist"]
> * Регистрация веб-приложения
> * Создание секрета клиента

Если у вас еще нет подписки Azure, [создайте бесплатную учетную запись](https://azure.microsoft.com/free/?WT.mc_id=A261C142F), прежде чем начинать работу.

## <a name="prerequisites"></a>Предварительные требования

Если вы еще не создали свой собственный [Azure AD B2C Tenant,](tutorial-create-tenant.md)создайте его сейчас. Вы можете использовать имеющийся клиент Azure Active Directory B2C.

## <a name="register-a-web-application"></a>Регистрация веб-приложения

Чтобы зарегистрировать приложение в клиенте Azure AD B2C, можно использовать текущий интерфейс **приложений** или новый объединенный интерфейс **Регистрация приложений (предварительная версия)**. [См. дополнительные сведения о новом интерфейсе](https://aka.ms/b2cappregintro).

#### <a name="applications"></a>[Приложения](#tab/applications/)

1. Войдите на [портал Azure](https://portal.azure.com).
1. Выберите **значок каталога и подписки** в панели инструментов портала, а затем выберите каталог, содержащий ваш клиент Azure AD B2C.
1. На портале Azure ищите **Azure AD B2C.**
1. Выберите **приложения,** а затем выберите **Добавить**.
1. Введите имя приложения. Например, *webapp1*.
1. Для поля **Включить веб-приложение или веб-API** и **Разрешить неявный поток** выберите **Да**.
1. Для **URL-адреса ответа** введите конечные точки, куда Azure AD B2C возвращает все маркеры, запрашиваемые вашим приложением. Например, можно настроить его для `https://localhost:44316`локального прослушивания. Если вы еще не знаете номер порта, можно ввести значение заполнителя и изменить его позже.

    Для целей тестирования, как этот `https://jwt.ms` учебник вы можете установить его, к которому отображает содержимое маркера для проверки. Для этого учебника установите `https://jwt.ms` **URL-адрес ответа.**

    Следующие ограничения применяются для ответа URL-адресов:

    * URL-адрес ответа должен `https`начинаться со схемы.
    * URL-адрес ответа является чувствительным к случаям. Его случай должен соответствовать случаю пути URL-адреса вашего запущенного приложения. Например, если приложение включает в `.../abc/response-oidc`себя как `.../ABC/response-oidc` часть своего пути, не укажите в URL-адресе ответа. Поскольку веб-браузер рассматривает пути как чувствительные к `.../abc/response-oidc` случаям, файлы cookie, связанные с ними, могут быть исключены, если они перенаправлены на несовпадаемый `.../ABC/response-oidc` URL-адрес.

1. Выберите **Создать** для завершения регистрации приложения.

#### <a name="app-registrations-preview"></a>[Регистрация приложений (предварительная версия)](#tab/app-reg-preview/)

1. Войдите на [портал Azure](https://portal.azure.com).
1. Выберите **значок каталога и подписки** в панели инструментов портала, а затем выберите каталог, содержащий ваш клиент Azure AD B2C.
1. На портале Azure ищите **Azure AD B2C.**
1. Щелкните **Регистрация приложений (предварительная версия)** и выберите **Новая регистрация**.
1. Введите **имя** приложения. Например, *webapp1*.
1. Выберите **учетные записи в любом каталоге организации или любом поставщике идентификационных данных.**
1. Под **перенаправлениеuri**, выберите `https://jwt.ms` **веб**, а затем введите в url текстовое поле.

    Перенаправление URI — это конечная точка, на которую пользователь отправляется сервером авторизации (Azure AD B2C, в данном случае) после завершения взаимодействия с пользователем и в которую при успешной авторизации отправляется код токена доступа или авторизации. В производственном приложении это, как правило, общедоступная конечная точка, где работает ваше приложение, например. `https://contoso.com/auth-response` Для целей тестирования, таких как `https://jwt.ms`этот учебник, вы можете настроить его на веб-приложение, принадлежащее корпорации Майкрософт, которое отображает декодированное содержимое маркера (содержимое маркера никогда не покидает ваш браузер). Во время разработки приложения можно добавить конечную точку, где приложение слушает локально, например. `https://localhost:5000` Вы можете добавлять и изменять перенаправление URIs в зарегистрированных приложениях в любое время.

    Следующие ограничения применяются к перенаправлению УРИ:

    * URL-адрес ответа должен `https`начинаться со схемы.
    * URL-адрес ответа является чувствительным к случаям. Его случай должен соответствовать случаю пути URL-адреса вашего запущенного приложения. Например, если приложение включает в `.../abc/response-oidc`себя как `.../ABC/response-oidc` часть своего пути, не укажите в URL-адресе ответа. Поскольку веб-браузер рассматривает пути как чувствительные к `.../abc/response-oidc` случаям, файлы cookie, связанные с ними, могут быть исключены, если они перенаправлены на несовпадаемый `.../ABC/response-oidc` URL-адрес.

1. В разделе **Разрешения** установите флажок *Предоставьте согласие администратора для разрешений openid и offline_access*.
1. Выберите **Зарегистрировать**.

После завершения регистрации заявки включите неявный поток грантов:

1. В разделе **Управление** выберите **Проверка подлинности**.
1. Выберите **Попробовать новый пользовательский интерфейс** (если этот параметр показан).
1. В соответствии с **неявной субсидией,** выберите как **маркеры доступа,** так и флажки **маркеров ID.**
1. Нажмите кнопку **Сохранить**.

## <a name="create-a-client-secret"></a>Создание секрета клиента

Если приложение обменивается кодом авторизации для токена доступа, необходимо создать секрет приложения.

#### <a name="applications"></a>[Приложения](#tab/applications/)

1. На странице **Azure AD B2C - Приложения** выберите созданное приложение, например *webapp1.*
1. Выберите **ключи,** а затем выберите **ключ для создания.**
1. Чтобы представить ключ, нажмите **Сохранить**. Запишите значение параметра **Ключ приложения**. Это значение используется в качестве секрета приложения в коде приложения.

#### <a name="app-registrations-preview"></a>[Регистрация приложений (предварительная версия)](#tab/app-reg-preview/)

1. На странице **Azure AD B2C - Регистрация приложений (Preview)** выберите созданное приложение, например *webapp1.*
1. В разделе **Управление** выберите **Сертификаты и секреты**.
1. Выберите **New client secret** (Создать секрет клиента).
1. Введите описание секрета клиента в поле **Описание**. Например, *clientsecret1*.
1. В разделе **Истекает** выберите срок действия секрета, а затем выберите **Добавить**.
1. Запись **значения**секрета . Это значение используется в качестве секрета приложения в коде приложения.

# <a name="troubleshoot-an-azure-ad-authentication-vpn-client"></a>Устранение проблем VPN-клиента Azure AD

Эта статья поможет вам устранить проблемы VPN-клиента для подключения к виртуальной сети с помощью VPN-сайта и проверки подлинности Active Directory.

## <a name="view-status-log"></a><a name="status"></a>Посмотреть журнал статуса

Просмотр журнала состояния для сообщений об ошибках.

![журналы](./media/troubleshoot-ad-vpn-client/1.png)

1. Нажмите значок стрелок в правом нижнем углу окна клиента, чтобы показать **журналы статуса.**
2. Проверьте журналы на наличие ошибок, которые могут указывать на проблему.
3. Сообщения об ошибках отображаются красным цветом.

## <a name="clear-sign-in-information"></a><a name="clear"></a>Четкая информация о вхвохе

Очистите информацию о входе.

![войти](./media/troubleshoot-ad-vpn-client/2.png)

1. Выберите ... рядом с профилем, который вы хотите устранить. Выберите **настройку -> очистить сохраненную учетную запись.**
2. Нажмите кнопку **Сохранить**.
3. Попытайтесь подключиться.
4. Если соединение по-прежнему выходит из строя, перейти к следующему разделу.

## <a name="run-diagnostics"></a><a name="diagnostics"></a>Запуск диагностики

Запустите диагностику на клиенте VPN.

![диагностика](./media/troubleshoot-ad-vpn-client/3.png)

1. Нажмите **на ...** рядом с профилем, на который вы хотите выполнить диагностику. Выберите **диагностику - > диагностика выполнения**.
2. Клиент проведет серию тестов и отобразит результаты теста

   * Доступ в Интернет - Проверка, есть ли у клиента подключение к Интернету
   * Учетные данные клиентов - Проверьте, достижима ли конечная точка проверки подлинности Active Directoration Azure
   * Server Resolvable - Контакты dNS-сервера для решения IP-адреса настроенного VPN-сервера
   * Server Reachable - Проверяет, отвечает ли VPN-сервер или нет
3. Если какой-либо из тестов не удается, обратитесь к сетевому администратору, чтобы решить проблему.
4. В следующем разделе показан, как собирать журналы, если это необходимо.

## <a name="collect-client-log-files"></a><a name="logfiles"></a>Сбор файлов журналов клиентов

Соберите файлы журнала VPN клиента. Файлы журнала могут быть отправлены в службу поддержки/администратора по вашему выбору. Например, электронная почта.

1. Нажмите кнопку "..." рядом с профилем, на который вы хотите выполнить диагностику. Выберите **Каталог журналов Диагностика -> Показать журналы.**

   ![показать журналы](./media/troubleshoot-ad-vpn-client/4.png)
2. Windows Explorer открывается в папку, содержащую файлы журнала.

   ![файл просмотра](./media/troubleshoot-ad-vpn-client/5.png)

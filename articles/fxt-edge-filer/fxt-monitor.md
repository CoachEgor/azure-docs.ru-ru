# <a name="monitor-azure-fxt-edge-filer-hardware-status"></a>Мониторинг состояния оборудования Azure FXT Edge Filer

Azure FXT Edge Filer гибридного кэша система хранения должна нескольких индикаторов состояния, встроенной в корпус, чтобы помочь администраторам понять, каким образом оборудование работает.

## <a name="system-health-status"></a>Состояние работоспособности системы

Для наблюдения за операциями кэша на более высоком уровне, использование программного обеспечения панели управления **панели мониторинга** странице, как описано в разделе [руководство по панелям мониторинга в панели управления](https://azure.github.io/Avere/legacy/dashboard/4_7/html/ops_dashboard_index.html)

## <a name="hardware-status-leds"></a>Индикаторы состояния оборудования

В этом разделе объясняется различные индикаторы состояния, встроенной в Azure FXT Edge Filer оборудования.

### <a name="hard-drive-status-leds"></a>Индикаторы состояния жесткого диска

![изображение жесткого диска на передней панели, по горизонтали с метками Выноска 2 (левом верхнем углу), 1 (в нижнем левом углу) и 3 (справа)](media/fxt-monitor/fxt-drive-callouts.png)

Каждого дискового модуля имеет два состояния индикатора: Индикатор активности (1) и Признак состояния (2). 

* Действие (1) индикаторы при использовании диска.  
* (2) Индикатор состояния указывает состояние диска с помощью кодов в таблице ниже.

| Состояние Индикатора состояния накопителя              | Значение  |
|-------------------------------------|----------------------------------------------------------|
| Начнет мигать зеленый два раза в секунду      | Определение диска *или* <br> Подготовка диска для удаления  |
| OFF (Неосвещенные)                         | Система еще не завершил запуска *или* <br>Диск готов для удаления |
| Начнет мигать зеленый, желтый и отключен       | Прогнозируется отказ диска   |
| Оранжевый начнет мигать четыре раза в секунду | Ошибка диска   |
| Сплошной зеленой                         | Диск находится в сети |

В правой части диска (3) помечается емкость диска и другие сведения.

Номера дисков нанесены на область между дисками. В Azure FXT Edge Filer диск 0 — это верхний диск слева, а диск 1 находится непосредственно под ним. Нумерация продолжится в этот шаблон. 

![фотография одного отсека для жесткого диска в корпус FXT, на которой видны номера дисков и метки емкости](media/fxt-drives-photo.png)

### <a name="system-health-status-indicator"></a>Индикатор состояния работоспособности системы

Освещенного большую кнопку в правой части панели управления слева указывает общее состояние системы, а также используется в качестве единицы локатора светлый режим идентификатор системы.

Нажмите идентификатор кнопку для переключения между режимами идентификатор системы и работоспособности системы и работоспособности системы.

|Состояние работоспособности системы состояния | Условие |
|-------------------------------------------|-----------------------------------------------|
| Сплошная синяя | Нормальной работе: система будет включен, двигатель нормально работает, и не используется режим идентификатор системы. <br/>Нажмите клавишу работоспособности системы и идентификатор кнопки, чтобы переключиться в режим идентификатор системы. |
| Мигает синий | Активен режим идентификатор системы. Нажмите клавишу работоспособности системы и кнопку идентификатор системы, чтобы переключиться в режим работоспособности системы. |
| Оранжевым | Система находится в безопасном режиме. Если проблема сохранится, [обратитесь в службу поддержки Майкрософт поддержки](fxt-support-ticket.md). |
| Мигает желтый | Сбой системы. Проверьте журнал событий системы для сообщения об ошибках. Сведения о событиях и ошибках сообщения, создаваемые встроенного по и агентов наблюдения за системными компонентами см. на странице "Поиск кода ошибки" в qrl.dell.com. |

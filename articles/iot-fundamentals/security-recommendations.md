# <a name="security-recommendations-for-azure-internet-of-things-iot-deployment"></a>Рекомендации по безопасности для развертывания Azure Internet of Things (IoT)

Эта статья содержит рекомендации по безопасности для IoT. Реализация этих рекомендаций поможет вам выполнить свои обязательства по обеспечению безопасности, как это описано в нашей модели общей ответственности. Для получения дополнительной информации о том, что корпорация Майкрософт делает для выполнения обязанностей поставщика услуг, прочитайте [Общие обязанности по облачным вычислениям.](https://gallery.technet.microsoft.com/Shared-Responsibilities-81d0ff91)

Некоторые рекомендации, включенные в эту статью, могут быть автоматически проверены Центром безопасности Azure. Центр безопасности Azure — это первая линия защиты ресурсов в Azure. Он периодически анализирует состояние безопасности ресурсов Azure для выявления потенциальных уязвимостей безопасности. Затем он предоставляет вам рекомендации о том, как их решать.

- Более подробную информацию о рекомендациях Центра безопасности Azure можно узнать [в Центре безопасности Azure.](../security-center/security-center-recommendations.md)
- Для получения [информации](../security-center/security-center-intro.md) о Центре безопасности Azure см.

| Рекомендация | Комментарии | При поддержке ASC |
|-|----|--|

| Сохранить ключи проверки подлинности в безопасности | Сохраняйте идентификации устройств и ключи аутентификации физически безопасными после развертывания. Это позволит избежать вредоносного устройства маскарад в качестве зарегистрированного устройства. | - |
| При возможности использовать SDK устройства | Устройства SDK реализуют различные функции безопасности, такие как шифрование, аутентификация и так далее, чтобы помочь вам в разработке надежного и безопасного приложения устройства. Для получения дополнительной информации смотрите [и используйте концентратор Azure IoT.](https://docs.microsoft.com/azure/iot-hub/iot-hub-devguide-sdks) | - |

## <a name="identity-and-access-management"></a>Управление удостоверениями и доступом 

| Рекомендация | Комментарии | При поддержке ASC |
|-|----|--|
| Определение управления доступом для концентратора | [Понять и определить тип доступа](iot-security-deployment.md#securing-the-cloud) каждый компонент будет иметь в вашем решении IoT концентратор, на основе функциональности. Разрешенные разрешения *— это регистрационные чтения,* *реестраReadWrite,* *ServiceConnect*и *DeviceConnect.* Политики общего доступа по умолчанию [в концентраторе IoT](https://docs.microsoft.com/azure/iot-hub/iot-hub-devguide-security#access-control-and-permissions) также могут помочь определить разрешения для каждого компонента на основе его роли. | - |
| Определение управления доступом для бэкэнд-сервисов | Данные, потребляемые вашим решением IoT Hub, могут быть использованы другими службами Azure, такими как [Cosmos DB,](https://docs.microsoft.com/azure/cosmos-db/) [Stream Analytics,](https://docs.microsoft.com/azure/stream-analytics/) [App Service,](https://docs.microsoft.com/azure/app-service/) [Logic Apps](https://docs.microsoft.com/azure/logic-apps/)и [Blob.](https://docs.microsoft.com/azure/storage/blobs/storage-blobs-introduction) Убедитесь в том, чтобы понять и разрешить соответствующие разрешения доступа, как это задокументировано для этих служб. | - |

## <a name="data-protection"></a>Защита данных

| Рекомендация | Комментарии | При поддержке ASC |
|-|----|--|
| Безопасная аутентификация устройства | Обеспечьте безопасную связь между устройствами и концентратором IoT, используя либо [уникальный ключ идентификации, либо маркер безопасности,](iot-security-deployment.md#iot-hub-security-tokens)либо [сертификат X.509 на устройстве](iot-security-deployment.md#x509-certificate-based-device-authentication) для каждого устройства. Используйте соответствующий метод [для использования маркеров безопасности на основе выбранного протокола (МЗТТ, АМЗП или HTTPS).](https://docs.microsoft.com/azure/iot-hub/iot-hub-devguide-security) | - |
| Безопасная связь с устройством | IoT Концентратор обеспечивает подключение к устройствам, использующим стандарт Transport Layer Security (TLS), поддерживающий версии 1.2 и 1.0. Используйте [TLS 1.2](https://tools.ietf.org/html/rfc5246) для обеспечения максимальной безопасности. | - |
| Защита взаимодействия со службой | IoT Hub предоставляет конечные точки для подключения к бэкэнд-сервисам, таким как [Azure Storage](/azure/storage/) или [Event Hubs,](/azure/event-hubs) используя только протокол TLS, и ни одна конечная точка не разоблачается на незашифрованном канале. Как только эти данные понижатся за эти бэкэнд-сервисы для хранения или анализа, не забудьте использовать соответствующие методы безопасности и шифрования для этой службы и защитите конфиденциальную информацию в бэкэнде. | - |

## <a name="networking"></a>Сеть

| Рекомендация | Комментарии | При поддержке ASC |
|-|----|--|
| Защита доступа к устройствам | Держите аппаратные порты в устройствах к минимуму, чтобы избежать нежелательного доступа. Кроме того, создайте механизмы для предотвращения или обнаружения физического подделки устройства. Ознакомьтесь с [рекомендациями по безопасности IoT](iot-security-best-practices.md) для получения подробной информации. | - |
| Создание защищенного оборудования | Включите функции безопасности, такие как зашифрованное хранилище или модуль trusted Platform Module (TPM), чтобы обеспечить более безопасную безопасность устройств и инфраструктуры. Держите операционную систему устройства и драйверы обновлены до последних версий, и если позволяет пространство, установить антивирусные и антивредоносные возможности. Прочитайте [архитектуру безопасности IoT,](iot-security-architecture.md) чтобы понять, как это может помочь смягчить несколько угроз безопасности. | - |

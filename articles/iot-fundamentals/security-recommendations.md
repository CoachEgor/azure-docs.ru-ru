# <a name="security-recommendations-for-azure-internet-of-things-iot-deployment"></a>Рекомендации по безопасности для развертывания Azure "Интернет вещей" (IoT)

Эта статья содержит рекомендации по безопасности для Интернета вещей. Реализация этих рекомендаций поможет вам удовлетворить обязательства по обеспечению безопасности, как описано в нашей общей модели ответственности. Дополнительные сведения о том, что делает корпорация Майкрософт для выполнения обязанностей поставщиков услуг, см. в статье [Общие обязанности для облачных вычислений](https://gallery.technet.microsoft.com/Shared-Responsibilities-81d0ff91/file/225237/1/Shared%20Responsibilities%20for%20Cloud%20Computing%20(2017-04-03).pdf).

Некоторые из рекомендаций, описанных в этой статье, можно автоматически отслеживать с помощью центра безопасности Azure. Центр безопасности Azure — это первая строка защиты ресурсов в Azure. Он периодически анализирует состояние безопасности ресурсов Azure и выявляет потенциальные уязвимости системы безопасности. Затем он предоставляет рекомендации по их устранению.

- Дополнительные сведения о рекомендациях центра безопасности Azure см. [в статье рекомендации по безопасности в центре безопасности Azure](../security-center/security-center-recommendations.md).
- Сведения о центре безопасности Azure см. в статье [что такое центр безопасности Azure?](../security-center/security-center-intro.md)

## <a name="recommendations"></a>Рекомендации

| Категория | Рекомендации | Комментарии | Поддерживается по УБЫВАНИЮ |
|-|-|----|--|
| Общие сведения | Оставайтесь в курсе последних обновлений | Используйте последние версии поддерживаемых платформ, языков программирования, протоколов и платформ. | - |
| Общие сведения | Обеспечение безопасности ключей проверки подлинности | Обеспечьте физическую безопасность идентификаторов устройств и их ключей проверки подлинности после развертывания. Это позволит избежать имитации вредоносного устройства в качестве зарегистрированного устройства. | - |
| Общие сведения | Используйте пакеты SDK для устройств, когда это возможно | Пакеты SDK для устройств реализуют различные функции безопасности, такие как шифрование, проверка подлинности и т. д., которые помогут вам в разработке надежного и безопасного приложения для устройств. Дополнительные сведения см. в статье [изучение и использование пакетов SDK для центра Интернета вещей Azure](https://docs.microsoft.com/azure/iot-hub/iot-hub-devguide-sdks) . | - |
| Управление идентификационной информацией и доступом | Определение контроля доступа для концентратора | [Изучите и определите тип доступа](iot-security-deployment.md#securing-the-cloud) каждого компонента в решении центра Интернета вещей на основе функциональных возможностей. Допустимые разрешения: *Чтение реестра*, *RegistryReadWrite*, *ServiceConnect*и *DeviceConnect*. [Политики общего доступа по умолчанию в центре Интернета вещей](https://docs.microsoft.com/azure/iot-hub/iot-hub-devguide-security#access-control-and-permissions) также могут помочь определить разрешения для каждого компонента в зависимости от его роли. | - |
| Управление идентификационной информацией и доступом | Определение контроля доступа для серверных служб | Данные, принимаемые решением центра Интернета вещей, можно использовать в других службах Azure, таких как [Cosmos DB](https://docs.microsoft.com/azure/cosmos-db/), [Stream Analytics](https://docs.microsoft.com/azure/stream-analytics/), [служба приложений](https://docs.microsoft.com/azure/app-service/), [Logic Apps](https://docs.microsoft.com/azure/logic-apps/)и [хранилище BLOB-объектов](https://docs.microsoft.com/azure/storage/blobs/storage-blobs-introduction). Убедитесь, что вы понимаете и разрешите соответствующие разрешения на доступ, как описано для этих служб. | - |
| Защита данных | Безопасная проверка подлинности устройства | Обеспечьте безопасную связь между устройствами и центром Интернета вещей с помощью [уникального ключа удостоверения или токена безопасности](iot-security-deployment.md#iot-hub-security-tokens)или [сертификата X. 509 на устройстве](iot-security-deployment.md#x509-certificate-based-device-authentication) для каждого устройства. Используйте соответствующий метод для [использования маркеров безопасности на основе выбранного протокола (MQTT, AMQP или HTTPS)](https://docs.microsoft.com/azure/iot-hub/iot-hub-devguide-security). | - |
| Защита данных | Безопасное взаимодействие устройств | Центр Интернета вещей обеспечивает безопасность подключения к устройствам по стандарту TLS, поддерживающим версии 1,2 и 1,0. Используйте [TLS 1,2](https://tools.ietf.org/html/rfc5246) для обеспечения максимальной безопасности. | - |
| Защита данных | Защита взаимодействия со службой | Центр Интернета вещей предоставляет конечные точки для подключения к серверным службам, таким как служба [хранилища Azure](/azure/storage/) или [концентраторы событий](/azure/event-hubs) , используя только протокол TLS, а конечная точка не предоставляется в незашифрованном канале. Когда эти данные поступает в эти серверные службы для хранения или анализа, необходимо применить соответствующие методы обеспечения безопасности и шифрования для этой службы, а также защитить конфиденциальные данные в серверной части. | - |
| Работа в сети | Защита доступа к устройствам | Чтобы избежать нежелательного доступа, обеспечьте минимальное число устройств для аппаратных портов на своих устройствах. Кроме того, механизмы сборки для предотвращения или обнаружения физического изменения устройства. Дополнительные сведения см. в статье [рекомендации по обеспечению безопасности IOT](iot-security-best-practices.md) . | - |
| Работа в сети | Создание безопасного оборудования | Реализуйте такие функции безопасности, как зашифрованное хранилище или доверенный платформенный модуль (TPM) (доверенный платформенный модуль), чтобы обеспечить безопасность устройств и инфраструктуры. Обеспечьте обновление операционной системы и драйверов устройства до последних версий, а также, если место позволяет, установить антивирусные и Антивредоносные функции. Ознакомьтесь с [архитектурой безопасности IOT](iot-security-architecture.md) , чтобы понять, как это может помочь устранить несколько угроз безопасности. | - |
| Мониторинг | Отслеживание несанкционированного доступа к устройствам |  Используйте функцию ведения журнала операционной системы устройства для отслеживания нарушений безопасности или физического изменения устройства или его портов. | - |
| Мониторинг | Мониторинг решения Интернета вещей в облаке | Отслеживайте общую работоспособность решения центра Интернета вещей с помощью [метрик в Azure Monitor](https://docs.microsoft.com/azure/iot-hub/iot-hub-metrics). | - |
| Мониторинг | Настройка диагностики | Внимательно наблюдайте за операциями, записывая в решение события, а затем отправляя журналы диагностики в Azure Monitor, чтобы получить представление о производительности. Дополнительные сведения см. [в статье мониторинг и диагностика проблем в центре Интернета вещей](https://docs.microsoft.com/azure/iot-hub/iot-hub-monitor-resource-health) . | - |

# <a name="frequently-asked-questions-about-networking-in-azure-functions"></a>Часто задаваемые вопросы о сети в Azure Functions

В этой статье часто перечисляются часто задаваемые вопросы о сети в Azure Functions. Для более полного обзора [см.](functions-networking-options.md)

## <a name="how-do-i-set-a-static-ip-in-functions"></a>Как установить статический IP в функциях?

Развертывание функции в среде службы приложений в настоящее время является единственным способом иметь статический входящий и исходящий IP для вашей функции. Для получения подробной информации об использовании среды службы приложений начните со статьи [Создайте и используйте внутренний балансер нагрузки с средой службы приложений.](../app-service/environment/create-ilb-ase.md)

## <a name="how-do-i-restrict-internet-access-to-my-function"></a>Как ограничить доступ в Интернет к моей функции?

Вы можете ограничить доступ в Интернет несколькими способами:

* [Ограничения Ip- ограничений](../app-service/app-service-ip-restrictions.md): Ограничить входящий трафик в приложение функции по диапазону IP.
    * Под ограничениями IP вы также можете настроить [конечные точки службы,](../virtual-network/virtual-network-service-endpoints-overview.md)которые ограничивают функцию только приемом входящего трафика из определенной виртуальной сети.
* Удаление всех триггеров HTTP. Для некоторых приложений достаточно просто избежать триггеров HTTP и использовать любой другой источник событий для запуска функции.

Имейте в виду, что редактор портала Azure требует прямого доступа к работающей функции. Любые изменения кода через портал Azure потребуют, чтобы устройство, которое вы используете для просмотра портала, внесло свой IP в белый список. Но вы все еще можете использовать что-либо под платформой функции вкладке с сетевыми ограничениями на месте.

## <a name="how-do-i-restrict-my-function-app-to-a-virtual-network"></a>Как ограничить приложение функции виртуальной сетью?

Вы можете ограничить **входящий** трафик для функционального приложения виртуальной сетью с помощью [Служебных endpoints.](./functions-networking-options.md#private-site-access) Эта конфигурация по-прежнему позволяет приложению функции совершать исходящие звонки в Интернет.

Единственный способ полностью ограничить функцию таким образом, чтобы весь трафик течет через виртуальную сеть, это использовать внутренне сбалансированную среду службы приложений. Эта опция развертывает ваш сайт на выделенной инфраструктуре внутри виртуальной сети и отправляет все триггеры и трафик через виртуальную сеть. 

Для получения подробной информации об использовании среды службы приложений начните со статьи [Создайте и используйте внутренний балансер нагрузки с средой службы приложений.](../app-service/environment/create-ilb-ase.md)

## <a name="how-can-i-access-resources-in-a-virtual-network-from-a-function-app"></a>Как получить доступ к ресурсам в виртуальной сети из функционального приложения?

Вы можете получить доступ к ресурсам в виртуальной сети с помощью функции запуска с помощью виртуальной сетевой интеграции. Для получения дополнительной информации [см.](functions-networking-options.md#virtual-network-integration)

## <a name="how-do-i-access-resources-protected-by-service-endpoints"></a>Как получить доступ к ресурсам, защищенным конечными точками службы?

С помощью виртуальной сетевой интеграции вы можете получить доступ к ресурсам, защищенным от службы, защищенным конечными ресурсами, из работая функции. Для получения дополнительной информации [см.](functions-networking-options.md#virtual-network-integration)

## <a name="how-can-i-trigger-a-function-from-a-resource-in-a-virtual-network"></a>Как запустить функцию с ресурса в виртуальной сети?

Вы можете разрешить вызов триггеров HTTP из виртуальной сети с помощью [конечных точек службы.](./functions-networking-options.md#private-site-access) 

Вы также можете запустить функцию из всех других ресурсов в виртуальной сети, развернув приложение функции в премиум-план, план службы приложений или среду службы приложений. Для получения дополнительной информации посмотреть [триггеры виртуальной сети non-HTTP](./functions-networking-options.md#virtual-network-triggers-non-http)

## <a name="how-can-i-deploy-my-function-app-in-a-virtual-network"></a>Как развернуть приложение функции в виртуальной сети?

Развертывание в среде службы приложений является единственным способом создания функционального приложения, которое полностью находится внутри виртуальной сети. Для получения подробной информации об использовании внутреннего баланса нагрузки с средой службы приложений начните со статьи [Создайте и используйте внутренний балансер нагрузки с средой службы приложений.](https://docs.microsoft.com/azure/app-service/environment/create-ilb-ase)

Для сценариев, где вам нужен только односторонний доступ к виртуальным сетевым ресурсам или менее полная сетевая [изоляция, см.](functions-networking-options.md)

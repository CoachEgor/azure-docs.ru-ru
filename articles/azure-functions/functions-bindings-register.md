# <a name="register-azure-functions-binding-extensions"></a>Регистрация связывающих расширений функций Azure

В версии Azure Functions 2.x [привязки](./functions-triggers-bindings.md) доступны как отдельные пакеты от времени выполнения функций. В то время как функции привязки доступа .NET через пакеты NuGet, пакеты расширения позволяют другим функциям получить доступ ко всем привязки через настройки конфигурации.

Рассмотрим следующие элементы, связанные с обязательными расширениями:

- Связывающие расширения явно не зарегистрированы в Функциях 1.x, за исключением [создания библиотеки класса C' с помощью Visual Studio.](#local-csharp)

- Триггеры HTTP и таймер поддерживаются по умолчанию и не требуют расширения.

В следующей таблице указывается, когда и как вы регистрируете привязки.

| Среда разработки |Регистрация<br/> в службе "Функции" версии 1.x  |Регистрация<br/> в службе "Функции" версии 2.x  |
|-------------------------|------------------------------------|------------------------------------|
|Портал Azure|Автоматически|Автоматически|
|Non-.NET языки или локальная разработка основных инструментов Azure|Автоматически|[Используйте основные инструменты функций Azure и пакеты расширений](#extension-bundles)|
|Библиотека класса C' с помощью Visual Studio|[С помощью средств NuGet](#vs)|[С помощью средств NuGet](#vs)|
|Библиотека классов C# с использованием Visual Studio Code|Недоступно|[С помощью .NET Core CLI](#vs-code)|

## <a name="extension-bundles-for-local-development"></a><a name="extension-bundles"></a>Пакеты расширения для местного развития

Пакеты расширения — это технология развертывания, которая позволяет добавлять совместимый набор связывающих расширений функций в приложение функции. При создании приложения добавляется предопределенный набор расширений. Пакеты расширения, определенные в пакете, совместимы друг с другом, что позволяет избежать конфликтов между пакетами. Вы включите пакеты расширения в файле приложения host.json.  

Вы можете использовать пакеты расширения с версией 2.x и более поздними версиями времени выполнения функций. При локальной разработке убедитесь, что вы используете последнюю версию [основных функций Azure.](functions-run-local.md#v2)

Используйте пакеты расширений для локальной разработки с помощью основных инструментов Azure Functions, Visual Studio Code и при удаленной сборке.

Если вы не используете пакеты расширений, необходимо установить .NET Core 2.x SDK на локальном компьютере перед установкой каких-либо связывающих расширений. Пакеты расширения устраняет это требование для локальной разработки. 

Чтобы использовать пакеты расширения, обновите файл *host.json,* чтобы включить следующую запись для: `extensionBundle`

## <a name="c-class-library-with-visual-studio"></a><a name="vs"></a>Библиотека\# C класса с Визуальной студией

В **Visual Studio**можно установить пакеты из консоли Package Manager с помощью команды [Install-Package,](https://docs.microsoft.com/nuget/tools/ps-ref-install-package) как показано в следующем примере:

```powershell
Install-Package Microsoft.Azure.WebJobs.Extensions.ServiceBus -Version <TARGET_VERSION>
```

Название пакета, используемого для данной привязки, приводится в справочной статье для этой привязки. Например, вы можете ознакомиться с [разделом о пакетах в справочной статье о привязках Служебной шины](functions-bindings-service-bus.md#functions-1x).

Замените `<TARGET_VERSION>` в этом примере определенной версией пакета, например `3.0.0-beta5`. Действительные версии перечислены на страницах отдельных пакетов в [NuGet.org.](https://nuget.org) Основные версии, соответствующие времени выполнения функций 1.x или 2.x, указаны в справочной статье для привязки.

Если вы `Install-Package` ссылаетесь на привязку, вам не нужно использовать [пакеты расширения.](#extension-bundles) Этот подход специфичен для библиотек классов, построенных в Visual Studio.

## <a name="c-class-library-with-visual-studio-code"></a><a name="vs-code"></a>Библиотека класса C с кодом Visual Studio

В **Visual Studio Code**установите пакеты для проекта библиотеки класса C' из запроса команды, используя команду [пакетов dotnet](https://docs.microsoft.com/dotnet/core/tools/dotnet-add-package) в .NET Core CLI. Следующий пример показывает, как вы добавляете привязку:

```terminal
dotnet add package Microsoft.Azure.WebJobs.Extensions.<BINDING_TYPE_NAME> --version <TARGET_VERSION>
```

Интерфейс .NET Core CLI можно использовать только при разработке в среде "Функции Azure" версии 2.х.

Замените `<BINDING_TYPE_NAME>` название пакета, содержащего необходимый привязки. Вы можете найти желаемую связывающую справочную статью в [списке поддерживаемых привязок.](./functions-triggers-bindings.md#supported-bindings)

Замените `<TARGET_VERSION>` в этом примере определенной версией пакета, например `3.0.0-beta5`. Действительные версии перечислены на страницах отдельных пакетов в [NuGet.org.](https://nuget.org) Основные версии, соответствующие времени выполнения функций 1.x или 2.x, указаны в справочной статье для привязки.

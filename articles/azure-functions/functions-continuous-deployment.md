# <a name="continuous-deployment-for-azure-functions"></a>Непрерывное развертывание для Функций Azure

Функции Azure можно постоянно развертывать с помощью [интеграции исходного управления.](functions-deployment-technologies.md#source-control) Интеграция управления исходным кодом позволяет работать, в котором обновление кода запускает развертывание в Azure. Если вы новичок в функциях Azure, начинайте обзор [обзора функций Azure.](functions-overview.md)

Непрерывное развертывание является хорошим вариантом для проектов, где интегрируется несколько и частых вкладов. При постоянном развертывании необходимо поддерживать единый источник правды для кода, который позволяет группам легко работать. Можно настроить непрерывное развертывание в Azure Functions из следующих местоположений исходного кода:

* [Лазурный РЕПО](https://azure.microsoft.com/services/devops/repos/)
* [GitHub](https://github.com)
* [Bitbucket](https://bitbucket.org/)

Подразделение развертывания для функций в Azure — это функциональное приложение. Все функции в приложении функции развертываются одновременно. После непрерывного развертывания доступ к функциональному коду на портале Azure настраивается *как считываемый,* поскольку источник истины установлен в другом месте.

## <a name="set-up-continuous-deployment"></a><a name="credentials"></a>Непрерывное развертывание с использованием GIT в службе приложений Azure

Чтобы настроить непрерывное развертывание для существующего приложения функции, выполните эти шаги. Эти шаги демонстрируют интеграцию с репозиторием GitHub, но аналогичные шаги применяются для Репоза Azure или других репозиторий исходного кода.

1. В приложении функции на [портале Azure](https://portal.azure.com)выберите**Центр развертывания функций** >  **платформы.**

    ![Открытый центр развертывания](./media/functions-continuous-deployment/platform-features.png)

2. В **Центре развертывания**выберите **GitHub,** а затем выберите **Authorize.** Если вы уже авторизировали GitHub, выберите **Продолжить**. 

    ![Центр развертывания службы приложений Azure](./media/functions-continuous-deployment/github.png)

3. В GitHub выберите кнопку **«Авторизация AzureAppService».** 

    ![Авторизация службы приложений Azure](./media/functions-continuous-deployment/authorize.png)
    
    В **Центре развертывания** на портале Azure выберите **Продолжить**.

4. Выберите одного из следующих поставщиков сборки:

    * **Служба сборки службы приложений:** лучше всего, когда вам не нужна сборка или если вам нужна общая сборка.
    * **Лазурные трубопроводы (Preview)**: Лучше всего, когда вам нужно больше контроля над сборкой. Этот поставщик в настоящее время находится в предварительном просмотре.

    ![Выберите поставщика сборки](./media/functions-continuous-deployment/build.png)

5. Налажить информацию, специфичную для указанного вами варианта управления исходным ресурсом. Для GitHub необходимо ввести или выбрать значения для **Организации, Репозитория**и **ветви.** **Organization** Значения основаны на местоположении кода. Затем выберите **Продолжить**.

    ![Настройка GitHub](./media/functions-continuous-deployment/github-specifics.png)

6. Просмотрите все детали, а затем выберите **Завершение** для завершения конфигурации развертывания.

    ![Сводка](./media/functions-continuous-deployment/summary.png)

Когда процесс завершен, весь код из указанного источника развертывается в приложении. В этот момент изменения источника развертывания вызывают развертывание этих изменений в приложении функции в Azure.

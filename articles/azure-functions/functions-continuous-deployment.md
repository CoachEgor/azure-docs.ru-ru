# <a name="continuous-deployment-for-azure-functions"></a>Непрерывное развертывание для Функций Azure

Функции Azure позволяет развернуть свой код, который постоянно до [интеграция системы управления версиями](functions-deployment-technologies.md#source-control). Это позволяет рабочего процесса, где код обновляет триггера развертывания в Azure. Если вы работали с функциями Azure, начало работы с [Обзор функций Azure](functions-overview.md).

Непрерывное развертывание очень удобно для проектов, где для интеграции нескольких и часто вклад. Это также позволяет поддерживать единый источник истины для кода функции. Непрерывное развертывание можно настроить в функциях Azure из следующих расположений источника кода:

* [Azure репозиториев](https://azure.microsoft.com/services/devops/repos/)
* [GitHub](https://github.com)
* [Bitbucket;](https://bitbucket.org/)

Единицей развертывания для функций Azure является приложение-функция. Это означает, что все функции в приложении-функции развертываются в то же время. После включения непрерывного развертывания доступ к коду функции на портале Azure работает в режиме *только для чтения*, так как источник истины должна быть в другом месте.

## <a name="credentials"></a>Настройка непрерывного развертывания

Используйте эту процедуру, чтобы настроить непрерывное развертывание для имеющегося приложения-функции. Здесь показана интеграция с репозиторием GitHub, но аналогичная процедура подходит для репозиториев Azure или другие репозитории исходного кода.

1. В приложении-функции в [портала Azure](https://portal.azure.com)выберите **функции платформы** > **центр развертывания**.

    ![Откройте центр развертывания](./media/functions-continuous-deployment/platform-features.png)

2. В **центр развертывания**выберите **GitHub**, а затем выберите **Authorize**. Или, если вы уже задали GitHub, выберите **Продолжить**. 

    ![Центр развертывания](./media/functions-continuous-deployment/github.png)

3. В GitHub, выберите **авторизовать AzureAppService**. 

    ![Авторизация](./media/functions-continuous-deployment/authorize.png)
    
    На портале Azure **центр развертывания**выберите **Продолжить**.

4. Выберите один из следующих поставщиков:

    * **Служба сборок службы приложений** — рекомендации, когда нет необходимости сборки или при необходимости универсальной сборки.
    * **Конвейеры Azure (Предварительная версия)** - рекомендации, когда требуется больший контроль над сборки. Этот поставщик в настоящее время доступна Предварительная версия.

    ![Выбрав поставщик построения](./media/functions-continuous-deployment/build.png)

5. Настройте сведения, относящиеся к указанный параметр управления источника. Для GitHub, необходимо предоставить **организации**, **репозитория**, и **ветви** там, где находятся ваш код. Затем выберите **Продолжить**.

    ![Настройка GitHub](./media/functions-continuous-deployment/github-specifics.png)

6. Наконец, просмотрите все сведения и выберите **Готово** для завершения конфигурации развертывания.

    ![Сводка](./media/functions-continuous-deployment/summary.png)

После завершения процесса, весь код из указанного источника развертывается приложение. После этого изменения в источнике развертывания активировать развертывание этих изменений для приложения-функции в Azure.

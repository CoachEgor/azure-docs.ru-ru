# <a name="azure-functions-diagnostics-overview"></a>Обзор диагностики функций Azure

Когда вы работаете с приложением функции, вы хотите быть готовым к любым проблемам, которые могут возникнуть, от ошибок 4xx, чтобы вызвать сбои. Диагностика Azure Functions — это интеллектуальный и интерактивный опыт, который поможет вам устранить неполадки в приложении без каких-либо конфигураций или дополнительных затрат. Когда вы столкнетесь с проблемами с приложением функции, диагностика Azure Functions указывает на то, что не так, чтобы направлять вас к нужной информации, чтобы более легко и быстро устранить неполадки и решить проблему. В этой статье показаны основы использования диагностики функций Azure для более быстрой диагностики и решения общих проблем приложений функций.

## <a name="start-azure-functions-diagnostics"></a>Запуск диагностики функций Azure

Чтобы получить доступ к диагностике функций Azure:

1. Перейдите к приложению функции на [портале Azure.](https://portal.azure.com)
2. Выберите вкладку **«Платформа».**
3. Выберите **диагностику и решение проблем** под **управлением ресурсов**, который открывает azure Функции диагностики.
4. Выберите категорию, которая наилучшим образом описывает проблему приложения функции, используя ключевые слова в плитке главной страницы. Вы также можете ввести ключевое слово, которое лучше всего описывает вашу проблему в панели поиска. Например, можно `execution` ввести список диагностических отчетов, связанных с выполнением приложения функции, и открыть их непосредственно с главной страницы.

![Домашняя страница](./media/functions-diagnostics/homepage.png)

## <a name="use-the-interactive-interface"></a>Использование интерактивного интерфейса

Как только вы выберете категорию главной страницы, которая наилучшим образом соответствует проблеме приложения функции, интерактивный интерфейс диагностики Azure Functions, Genie, может помочь вам с помощью диагностики и решения проблемы вашего приложения. Вы можете использовать ярлыки плитки, предоставляемые Genie, для просмотра полного диагностического отчета проблемной категории, которая вас интересует. Ярлыки плитки предоставляют вам прямой доступ к диагностическим показателям.

![Джинн](./media/functions-diagnostics/genie.png)

После выбора плитки можно увидеть список тем, связанных с проблемой, описанной в плитке. Эти темы предоставляют фрагменты заметной информации из полного доклада. Вы можете выбрать любую из этих тем для дальнейшего изучения проблем. Кроме того, можно выбрать **просмотр полного отчета** для изучения всех тем на одной странице.

![Предварительный просмотр диагностического отчета](./media/functions-diagnostics/preview-of-diagnostic-report.png)

## <a name="view-a-diagnostic-report"></a>Просмотр диагностического отчета

После выбора темы можно просмотреть диагностический отчет, специфичный для приложения функции. Диагностические отчеты используют значки статуса, чтобы указать, есть ли какие-либо конкретные проблемы с вашим приложением. Вы видите подробное описание проблемы, рекомендуемые действия, связанные метрики и полезные документы. Индивидуальные диагностические отчеты генерируются из серии проверок, проводимых в приложении функции. Диагностические отчеты могут быть полезным инструментом для выявления проблем в приложении функции и направляя вас к решению проблемы.

## <a name="find-the-problem-code"></a>Найти проблемный код

Для функций, основанных на сценариях, можно использовать **выполнение функций и ошибки** в **функции App Down или Reporting Errors,** чтобы сузить строку кода, вызывая исключения или ошибки. Эта функция может быть полезным инструментом для получения к первопричине и устранения проблем из определенной строки кода. Эта опция недоступна для предкомпилированных функций C и Java.

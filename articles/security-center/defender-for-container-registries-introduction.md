# <a name="introduction-to-azure-defender-for-container-registries"></a>Общие сведения о защитнике Azure для реестров контейнеров

Реестр контейнеров Azure (запись контроля доступа) — это управляемая частная служба реестра DOCKER, которая хранит образы контейнеров для развертываний Azure и управляет ими в центральном реестре. Он основан на реестре DOCKER с открытым кодом 2,0.

Чтобы защитить все Azure Resource Manager на основе реестров в подписке, включите **защитник Azure для реестров контейнеров** на уровне подписки. После этого центр безопасности проверит образы, которые были отправлены в реестр, импортированы в реестр или какие бы то ни было образы, полученные за последние 30 дней. За использование этого компонента берется отдельное изображение.

## <a name="what-are-the-benefits-of-azure-defender-for-container-registries"></a>Каковы преимущества защитника Azure для реестров контейнеров?

Центр безопасности определяет Azure Resource Manager реестры контроля доступа на основе записей в подписке и позволяет легко выполнять оценку и управление уязвимыми местами для образов реестра в Azure.

**Защитник Azure для реестров контейнеров** содержит средство проверки уязвимостей для сканирования образов в реестре контейнеров Azure на основе Azure Resource Manager и предоставления более подробного представления об уязвимостях образов. Встроенный сканер питается от Qualys, ведущим в отрасли поставщиком для сканирования уязвимостей.

При обнаружении проблем — по Qualys или центру безопасности — вы получите уведомления на панели мониторинга центра безопасности. Для каждой уязвимости центр безопасности предоставляет практические рекомендации, а также классификацию серьезности и рекомендации по исправлению проблемы. Сведения о рекомендациях центра безопасности для контейнеров см. в [справочном списке рекомендаций](recommendations-reference.md#recs-containers).

Центр безопасности фильтрует и классифицирует результаты проверки. Если образ работоспособен, центр безопасности помечает его как таковой. Центр безопасности создает рекомендации по безопасности только для образов с проблемами, которые могут быть разрешены. Центр безопасности предоставляет подробные сведения о каждой обнаруженной уязвимости и классификации серьезности. Кроме того, он предоставляет рекомендации по исправлению конкретных уязвимостей, обнаруженных на каждом образе.

Только уведомляя о наличии проблем, центр безопасности сокращает вероятность нежелательных информационных оповещений.


## <a name="when-are-images-scanned"></a>Когда проверяются изображения?

Для сканирования изображений существует три триггера:

- При **принудительной отправке** образа в реестр центр безопасности автоматически проверяет этот образ. Чтобы запустить сканирование образа, отправьте его в репозиторий.

- **Недавно извлеченные** — поскольку новые уязвимости обнаруживаются каждый день, **защитник Azure для реестров контейнеров** также проверяет все образы, которые были получены в течение последних 30 дней. Дополнительная плата за повторное сканирование не взимается; Как упоминалось выше, счет выставляется один раз для каждого образа.

- **При импорте** . Реестр контейнеров Azure содержит средства импорта для переноса образов в реестр из центра DOCKER, реестра контейнеров Майкрософт или другого реестра контейнеров Azure. **Защитник Azure для реестров контейнеров** просматривает все поддерживаемые импортированные образы. Дополнительные сведения см. в статье [Импорт образов контейнеров в реестр контейнеров](../container-registry/container-registry-import-images.md).
 
Сканирование обычно выполняется в течение 2 минут, но может занять до 15 минут. Результаты доступны в виде таких рекомендаций центра безопасности, как:

## <a name="faq-for-azure-container-registry-image-scanning"></a>Вопросы и ответы по сканированию образа реестра контейнеров Azure

### <a name="how-does-security-center-scan-an-image"></a>Как центр безопасности сканирует образ?
Образ извлекается из реестра. Затем он выполняется в изолированной песочнице с помощью сканера Qualys, который извлекает список известных уязвимостей.

Центр безопасности фильтрует и классифицирует результаты проверки. Если образ работоспособен, центр безопасности помечает его как таковой. Центр безопасности создает рекомендации по безопасности только для образов с проблемами, которые могут быть разрешены. Только уведомляя о наличии проблем, центр безопасности сокращает вероятность нежелательных информационных оповещений.

### <a name="how-often-does-security-center-scan-my-images"></a>Как часто центр безопасности проверяет мои образы?
Сканирование изображений активируется при каждой операции отправки или импорта, а также в том случае, если образ был извлечен в течение последних 30 дней. 

### <a name="can-i-get-the-scan-results-via-rest-api"></a>Можно ли получить результаты проверки с помощью REST API?
Да. Результаты находятся в [подсистеме API подоценки](/rest/api/securitycenter/subassessments/list/). Кроме того, вы можете использовать граф ресурсов Azure (ARG), Kusto API для всех ресурсов: запрос может получить определенное сканирование.
 
### <a name="what-registry-types-are-scanned-what-types-are-billed"></a>Какие типы реестра проверяются? На какие типы взимается плата?
В разделе Доступность перечислены типы реестров контейнеров, поддерживаемые защитником Azure для реестров контейнеров. 

Если вы подключаете неподдерживаемые реестры к подписке Azure, они не будут проверены и плата за них не будет взиматься.

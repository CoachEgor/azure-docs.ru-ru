# <a name="adaptive-network-hardening-in-azure-security-center"></a>Адаптивное усиление защиты сети в центре безопасности Azure
Узнайте, как настроить Адаптивное усиление защиты сети в центре безопасности Azure.

## <a name="what-is-adaptive-network-hardening"></a>Что такое Адаптивное усиление защиты сети?
Применение [групп безопасности сети (NSG)](https://docs.microsoft.com/azure/virtual-network/security-overview) для фильтрации трафика в ресурсы и из ресурсов повышает уровень безопасности сети. Тем не менее в некоторых случаях фактический трафик, передаваемый через NSG, может быть подмножеством определенных правил NSG. В таких случаях дальнейшее улучшение безопасности может быть достигнуто за счет усиления правил NSG на основе реальных шаблонов трафика.

Адаптивное усиление защиты сети предоставляет рекомендации для дальнейшей защиты правил NSG. Он использует алгоритм машинного обучения, который учитывает фактический трафик, известную надежную конфигурацию, аналитику угроз и другие признаки компрометации, а затем предоставляет рекомендации по разрешению трафика только из конкретных кортежей IP/Port.

Например, пусть существующее правило NSG разрешать трафик от 140.20.30.10/24 через порт 22. Адаптивная рекомендация по усилению защиты сети, основанная на анализе, заключается в том, чтобы сократить диапазон и разрешить трафик от 140.23.30.10/29, который является более узким диапазоном IP-адресов, и запретить весь остальной трафик к этому порту.

>[!TIP]
> Адаптивные рекомендации по усилению защиты сети поддерживаются только на конкретных портах. Полный список [поддерживаемых портов](#which-ports-are-supported) см. ниже. 


![Режим усиления защиты сети](./media/security-center-adaptive-network-hardening/traffic-hardening.png)


## <a name="view-adaptive-network-hardening-alerts-and-rules"></a>Просмотр адаптивных предупреждений и правил усиления защиты сети

1. В центре безопасности выберите **сетевые** -> **Адаптивное усиление защиты сети**. Сетевые виртуальные машины перечислены в трех отдельных вкладках:
   * **Неработоспособные ресурсы**: виртуальные машины, которые в настоящее время имеют рекомендации и оповещения, запущенные с помощью адаптивного алгоритма усиления защиты сети. 
   * **Работоспособные ресурсы**: виртуальные машины без оповещений и рекомендаций.
   * **Непроверенные ресурсы**. виртуальные машины, на которых не удается запустить адаптивный алгоритм усиления защиты сети, по одной из следующих причин:
      * **Виртуальные машины являются классическими**: поддерживаются только Azure Resource Manager виртуальные машины.
      * **Недостаточно данных**: для создания точных рекомендаций по усилению защиты трафика центр безопасности требует не менее 30 дней передачи данных трафика.
      * **Виртуальная машина не защищена с помощью ASC Standard**: для этой функции подходят только виртуальные машины, для которых задана ценовая категория центра безопасности "Стандартный".

     ![неработоспособные ресурсы](./media/security-center-adaptive-network-hardening/unhealthy-resources.png)

2. На вкладке **неработоспособные ресурсы** выберите виртуальную машину, чтобы просмотреть ее оповещения и Рекомендуемые правила усиления защиты, которые необходимо применить.

    ![предупреждения усиления защиты](./media/security-center-adaptive-network-hardening/anh-recommendation-rules.png)


## <a name="review-and-apply-adaptive-network-hardening-recommended-rules"></a>Проверка и применение рекомендуемых правил адаптивной защиты сети

1. На вкладке **неработоспособные ресурсы** выберите виртуальную машину. Будут перечислены предупреждения и Рекомендуемые правила усиления защиты.

     ![правила усиления защиты](./media/security-center-adaptive-network-hardening/hardening-alerts.png)

   > [!NOTE]
   > На вкладке **правила** перечислены правила, которые рекомендуется добавить с помощью адаптивной защиты сети. На вкладке **оповещения** перечислены оповещения, которые были созданы из-за трафика, передаются в ресурс, который не входит в диапазон IP-адресов, допустимый в рекомендуемых правилах.

2. Если необходимо изменить некоторые параметры правила, его можно изменить, как описано в описании раздела [Изменение правила](#modify-rule).
   > [!NOTE]
   > Можно также [Удалить](#delete-rule) или [Добавить](#add-rule) правило.

3. Выберите правила, которые необходимо применить к NSG, и нажмите кнопку **Применить**.

      > [!NOTE]
      > Принудительные правила добавляются в NSG (ы), защищающие виртуальную машину. (Виртуальная машина может быть защищена NSG, связанным с сетевой картой, или подсетью, в которой находится виртуальная машина, или и то, и другое).

    ![принудительное применение правил](./media/security-center-adaptive-network-hardening/enforce-hard-rule2.png)


### Изменение правила <a name ="modify-rule"> </a>

Может потребоваться изменить параметры правила, которое было рекомендовано. Например, может потребоваться изменить Рекомендуемые диапазоны IP-адресов.

Некоторые важные рекомендации по изменению адаптивного правила усиления защиты сети.

* Вы можете изменить параметры только для правил "разрешить". 
* Нельзя изменить правила "разрешить", чтобы они стали "запрещенными" правилами. 

  > [!NOTE]
  > Создание и изменение правил "deny" выполняется непосредственно в NSG. Дополнительные сведения см. [в статье Создание, изменение или удаление группы безопасности сети](https://docs.microsoft.com/azure/virtual-network/manage-network-security-group).

* Правило " **запретить все трафик** " является единственным типом правила "запретить", которое будет указано здесь и не может быть изменено. Однако его можно удалить (см. раздел [Удаление правила](#delete-rule)).
  > [!NOTE]
  > Правило **запрета всего трафика** рекомендуется, если в результате выполнения алгоритма центр безопасности не определяет трафик, который должен быть разрешен, на основе существующей конфигурации NSG. Поэтому рекомендуемым правилом является запрет всего трафика на указанный порт. Имя этого типа правила отображается как "*система сформирована*". После применения этого правила его фактическое имя в NSG будет представлять собой строку, состоящую из протокола, направления трафика, "DENY" и случайного числа.

*Чтобы изменить Адаптивное правило усиления защиты сети, выполните следующие действия.*

1. Чтобы изменить некоторые параметры правила, на вкладке **правила** щелкните три точки (...) в конце строки правила и нажмите кнопку **изменить**.

   ![изменить правило](./media/security-center-adaptive-network-hardening/edit-hard-rule.png)

1. В окне **Изменение правила** обновите сведения, которые необходимо изменить, и нажмите кнопку **сохранить**.

   > [!NOTE]
   > После нажатия кнопки " **сохранить**" вы успешно изменили правило. *Однако вы не применили его к NSG.* Чтобы применить его, необходимо выбрать правило в списке и нажать кнопку **Применить** (как описано на следующем шаге).

   ![изменить правило](./media/security-center-adaptive-network-hardening/edit-hard-rule3.png)

3. Чтобы применить обновленное правило, выберите обновленное правило в списке и нажмите кнопку **Применить**.

    ![применить правило](./media/security-center-adaptive-network-hardening/enforce-hard-rule.png)

### Добавить новое правило <a name ="add-rule"> </a>

Можно добавить правило "разрешить", которое не рекомендуется центром безопасности.

> [!NOTE]
> Здесь можно добавить только правила "разрешить". Если вы хотите добавить правила "запретить", это можно сделать непосредственно в NSG. Дополнительные сведения см. [в статье Создание, изменение или удаление группы безопасности сети](https://docs.microsoft.com/azure/virtual-network/manage-network-security-group).

*Чтобы добавить Адаптивное правило усиления защиты сети, выполните следующие действия.*

1. Щелкните **Добавить правило** (находится в левом верхнем углу).

   ![Добавить правило](./media/security-center-adaptive-network-hardening/add-hard-rule.png)

1. В окне **новое правило** введите сведения и нажмите кнопку **Добавить**.

   > [!NOTE]
   > После нажатия кнопки " **Добавить**" правило будет добавлено и указано в списке других рекомендуемых правил. Однако вы не применили его к NSG. Чтобы активировать его, необходимо выбрать правило в списке и нажать кнопку **Применить** (как описано на следующем шаге).

3. Чтобы применить новое правило, выберите новое правило из списка и нажмите кнопку **Применить**.

    ![применить правило](./media/security-center-adaptive-network-hardening/enforce-hard-rule.png)


### Удаление правила <a name ="delete-rule"> </a>

При необходимости можно удалить рекомендуемое правило для текущего сеанса. Например, можно определить, что применение предлагаемого правила может привести к блокировке легального трафика.

*Чтобы удалить Адаптивное правило усиления защиты сети для текущего сеанса, выполните следующие действия.*

1. На вкладке **правила** щелкните три точки (...) в конце строки правила и нажмите кнопку **Удалить**.  

    ![правила усиления защиты](./media/security-center-adaptive-network-hardening/delete-hard-rule.png)

# <a name="adaptive-network-hardening-in-azure-security-center"></a>Укрепление адаптивной сети в Центре безопасности Azure
Узнайте, как настроить закаливание адаптивной сети в Центре безопасности Azure.

## <a name="what-is-adaptive-network-hardening"></a>Что такое упрочнение адаптивной сети?
Применение [групп сетевой безопасности (NSG)](https://docs.microsoft.com/azure/virtual-network/security-overview) для фильтрации трафика на ресурсы и из нее улучшает вашу безопасность сети. Тем не менее, все еще могут быть некоторые случаи, когда фактический трафик, протекающий через NSG, является подмножеством определенных правил NSG. В этих случаях дальнейшее улучшение положения в области безопасности может быть достигнуто за счет ужесточения правил НСГ, основанных на фактических схемах движения.

Адаптивное укрепление сети дает рекомендации по дальнейшему ужесточению правил NSG. Он использует алгоритм машинного обучения, который влияет на фактический трафик, известную доверенную конфигурацию, интеллект угроз и другие индикаторы компромисса, а затем предоставляет рекомендации, позволяющие трафик только с определенных ip/port tuples.

Например, предположим, что существующее правило NSG позволяет трафик от 140.20.30.10/24 на порту 22. Рекомендация Адаптивной сети Hardening, основанная на анализе, будет заключаться в том, чтобы сузить диапазон и разрешить трафик от 140.23.30.10/29 - который является более узким диапазоном ИС, и отрицать все другие движения в этот порт.

>[!TIP]
> Рекомендации по укреплению адаптивной сети поддерживаются только в следующих конкретных портах (как для UDP, так и для TCP): 13, 17, 19, 22, 23, 53, 69, 81, 111, 119, 123, 135, 137, 138, 139, 161, 162, 389, 445, 512, 514, 593, 636, 873, 1433, 1430, 230 , 2323, 2381, 3268, 3306, 3389, 4333, 5353, 5432, 5555, 5800, 5900, 5900, 5985, 5986, 6379, 6379, 7000, 7001, 7199, 8081, 8089, 8545, 9042, 9160, 9300, 11211, 16379, 26379, 27017, 37215


![Представление закаливания сети](./media/security-center-adaptive-network-hardening/traffic-hardening.png)


## <a name="view-adaptive-network-hardening-alerts-and-rules"></a>Просмотр оповещений и правил укрепления адаптивной сети

1. В Центре безопасности выберите -> **закаливание сетевой адаптивной сети.** **Networking** Сеть VM перечислены под тремя отдельными вкладками:
   * **Нездоровые ресурсы**: VMs, которые в настоящее время имеют рекомендации и оповещения, которые были вызваны запуском алгоритма затвердевания адаптивной сети. 
   * **Здоровые ресурсы**: VMs без предупреждений и рекомендаций.
   * **Неотсканированные ресурсы:** VMs, на котором не может работать алгоритм затвердевания адаптивной сети по одной из следующих причин:
      * **ВМ — это классические вм:** поддерживаются только управляющими ресурсами Azure.
      * **Недостаточно данных имеется**: Для получения точных рекомендаций по затвердеванию трафика Центр безопасности требует не менее 30 дней данных о трафике.
      * **VM не защищен стандартом ASC:** только VMs, которые установлены на стандартный ценовой уровень Центра безопасности, имеют право на эту функцию.

     ![нездоровые ресурсы](./media/security-center-adaptive-network-hardening/unhealthy-resources.png)

2. Из вкладки **Нездоровые ресурсы** выберите VM для просмотра своих оповещений и рекомендуемых правил ужесточения.

    ![закаливание оповещения](./media/security-center-adaptive-network-hardening/anh-recommendation-rules.png)


## <a name="review-and-apply-adaptive-network-hardening-recommended-rules"></a>Обзор и применение адаптивных правил укрепления сети

1. Из вкладки **Нездоровые ресурсы** выберите VM. Перечислены оповещения и рекомендуемые правила ужесточения.

     ![правила ужесточения](./media/security-center-adaptive-network-hardening/hardening-alerts.png)

   > [!NOTE]
   > Вкладка **Правила** перечисляет правила, которые адаптивная сеть затвердевания рекомендуется добавить. Вкладка **Alerts** перечисляет оповещения, которые были созданы из-за трафика, поступающей на ресурс, который не входит в диапазон IP, разрешенный в рекомендуемых правилах.

2. Если вы хотите изменить некоторые параметры правила, вы можете изменить его, как это объясняется в [правиле modify.](#modify-rule)
   > [!NOTE]
   > Вы также можете [удалить](#delete-rule) или [добавить](#add-rule) правило.

3. Выберите правила, которые вы хотите применить на NSG, и нажмите **Enforce**.

      > [!NOTE]
      > Принудительные правила добавляются в NSG (ы), защищающие VM. (VM может быть защищен NSG, который связан с его NIC, или подсеть, в которой проживает VM, или и то, и другое)

    ![соблюдение правил](./media/security-center-adaptive-network-hardening/enforce-hard-rule2.png)


### <a name="modify-a-rule"></a>Изменение правила <a name ="modify-rule"> </a>

Возможно, вы захотите изменить параметры рекомендуемого правила. Например, можно изменить рекомендуемые диапазоны IP.

Некоторые важные рекомендации по изменению правила укрепления адаптивной сети:

* Можно изменить параметры только правил "разрешить". 
* Вы не можете изменить правила "разрешить" стать "отрицать" правила. 

  > [!NOTE]
  > Создание и изменение правил "отрицания" осуществляется непосредственно на NSG. Для получения дополнительной информации [см. Создать, изменить или удалить группу сетевой безопасности.](https://docs.microsoft.com/azure/virtual-network/manage-network-security-group)

* **Отрицание всех** правил движения является единственным типом правила "отрицание", которое будет перечислено здесь, и оно не может быть изменено. Вы можете, однако, удалить его (см. [Удалить правило](#delete-rule)).
  > [!NOTE]
  > **Отрицание всех** правил трафика рекомендуется, когда в результате запуска алгоритма Центр безопасности не определяет трафик, который должен быть разрешен, на основе существующей конфигурации NSG. Таким образом, рекомендуемое правило заключается в том, чтобы отказать в перевозке в указанный порт. Название этого типа правила отображается как «*Система генерируется*». После применения этого правила, его фактическое название в NSG будет строка, состоящая из протокола, направления движения, "DENY", и случайное число.

*Чтобы изменить правило укрепления адаптивной сети:*

1. Чтобы изменить некоторые параметры правила, во вкладке **Правила** нажмите на три точки (...) в конце строки правила и нажмите **"Изменить".**

   ![правило отсвазание](./media/security-center-adaptive-network-hardening/edit-hard-rule.png)

1. В окне **правила «Изменить»** обновите детали, которые вы хотите изменить, и нажмите **«Сохранить».**

   > [!NOTE]
   > После нажатия **Сохранить,** вы успешно изменили правило. *Тем не менее, вы не применили его к NSG.* Чтобы применить его, необходимо выбрать правило в списке и нажать **Enforce** (как это объясняется на следующем этапе).

   ![правило отсвазание](./media/security-center-adaptive-network-hardening/edit-hard-rule3.png)

3. Чтобы применить обновленное правило, из списка выберите обновленное правило и нажмите **Enforce**.

    ![соблюдение правила](./media/security-center-adaptive-network-hardening/enforce-hard-rule.png)

### <a name="add-a-new-rule"></a>Добавление нового <a name ="add-rule"> </a> правила

Можно добавить правило "разрешить", которое не было рекомендовано Центром безопасности.

> [!NOTE]
> Здесь можно добавить только правила "разрешить". Если вы хотите добавить правила "отрицать", вы можете сделать это непосредственно на NSG. Для получения дополнительной информации [см. Создать, изменить или удалить группу сетевой безопасности.](https://docs.microsoft.com/azure/virtual-network/manage-network-security-group)

*Чтобы добавить правило укрепления адаптивной сети:*

1. Нажмите **Правило Добавить** (находится в верхнем левом углу).

   ![добавить правило](./media/security-center-adaptive-network-hardening/add-hard-rule.png)

1. В **окне нового правила** введите детали и нажмите **Добавить**.

   > [!NOTE]
   > После нажатия **Добавить,** вы успешно добавили правило, и он указан с другими рекомендуемыми правилами. Тем не менее, вы не применили его на NSG. Чтобы активировать его, необходимо выбрать правило в списке и нажать **Enforce** (как это объясняется на следующем этапе).

3. Чтобы применить новое правило, из списка, выберите новое правило и нажмите **Enforce**.

    ![соблюдение правила](./media/security-center-adaptive-network-hardening/enforce-hard-rule.png)


### <a name="delete-a-rule"></a>Удалить правило <a name ="delete-rule"> </a>

При необходимости можно удалить рекомендуемое правило для текущей сессии. Например, можно определить, что применение предлагаемого правила может заблокировать законный трафик.

*Чтобы удалить правило затвердевания адаптивной сети для текущей сессии:*

1. Во вкладке **Правила** нажмите на три точки (...) в конце строки правила и нажмите **«Удалить».**  

    ![правила ужесточения](./media/security-center-adaptive-network-hardening/delete-hard-rule.png)

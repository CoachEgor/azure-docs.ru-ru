# <a name="remediate-recommendations-in-azure-security-center"></a>Рекомендации по устранению проблем (Центр безопасности Azure)

Рекомендации дают вам предложения о том, как лучше защитить свои ресурсы. Вы выполняете рекомендацию, следуя мерам по исправлению положения, предусмотренным в рекомендации.

## <a name="remediation-steps"></a>Шаги восстановления<a name="remediation-steps"></a>

После рассмотрения всех рекомендаций, решить, какой из них для исправления в первую очередь. Мы рекомендуем использовать [эффект Secure Score,](security-center-recommendations.md#monitor-recommendations) чтобы помочь определить приоритеты, что делать в первую очередь.

1. Из списка щелкните рекомендацию.

1. Следуйте инструкциям в разделе **Шаги восстановления.** Каждая рекомендация имеет свой собственный набор инструкций. На следующем скриншоте показаны шаги восстановления для настройки приложений, чтобы разрешить трафик только по HTTPS.

    ![Сведения о рекомендации](./media/security-center-remediate-recommendations/security-center-remediate-recommendation.png)

1. После завершения работы появляется уведомление, информирующее вас о том, удалось ли восстановить исправление.

## <a name="quick-fix-remediation"></a>Быстрый Исправление исправление<a name="one-click"></a>

Быстрая исправление позволяет быстро исправлять рекомендации на нескольких ресурсах. Он доступен только для конкретных рекомендаций. Быстрый Fix упрощает исправление и позволяет быстро увеличить ваш Безопасный результат, повышая безопасность вашей среды.

Для реализации исправления быстрого исправления:

1. Из списка рекомендаций, которые имеют **Быстрый Fix!** этикетку, нажмите на рекомендацию.

    [![Выберите Быстрый Fix!](media/security-center-remediate-recommendations/security-center-one-click-fix-select.png)](media/security-center-remediate-recommendations/security-center-one-click-fix-select.png#lightbox)

1. Из вкладки **Нездоровые ресурсы** выберите ресурсы, на которых вы хотите реализовать рекомендацию, и нажмите **Remediate.**

    > [!NOTE]
    > Некоторые из перечисленных ресурсов могут быть отключены, так как у вас нет соответствующих разрешений на их изменение.

1. В коробке подтверждения прочитайте детали и последствия восстановления.

    ![Быстрое исправление](./media/security-center-remediate-recommendations/security-center-one-click-fix-view.png)

    > [!NOTE]
    > Последствия перечислены в сером поле в окне **ресурсов Remediate,** которое открывается после нажатия **Remediate.** Они перечисляют, какие изменения происходят при продолжении исправления Быстрого исправления.

1. Вставьте соответствующие параметры при необходимости и утвердите исправление.

    > [!NOTE]
    > Это может занять несколько минут после завершения исправления, чтобы увидеть ресурсы во вкладке **«Здоровые ресурсы».** Чтобы просмотреть действия по исправлению положения, проверьте [журнал действий.](#activity-log)

1. После завершения работы появляется уведомление, информирующее вас о том, удалось ли восстановить исправление.

## <a name="quick-fix-remediation-logging-in-the-activity-log"></a>Быстрая исправление регистрации в журнале действий<a name="activity-log"></a>

Операция по исправлению использует развертывание шаблонов или вызов REST PATCH API для применения конфигурации на ресурсе. Эти операции регистрируются в [журнале активности Azure.](../azure-resource-manager/management/view-activity-logs.md)

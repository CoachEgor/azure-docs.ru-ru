# <a name="threat-detection-for-data-services-in-azure-security-center"></a>Обнаружение угроз для служб данных в центре безопасности Azure

 Центр безопасности анализирует журналы служб хранилища данных и запускает оповещения при обнаружении угрозы для ресурсов данных. В этом разделе перечислены оповещения, создаваемые центром безопасности для следующих служб:

* [База данных SQL Azure и хранилище данных SQL](#data-sql)
* [Хранилище Azure](#azure-storage)

## База данных SQL Azure и хранилище данных SQL<a name="data-sql"></a>

Обнаружение угроз SQL обнаруживает аномальные действия, указывающие на необычные и потенциально опасные попытки доступа к базам данных или их использования. Центр безопасности анализирует журналы аудита SQL и выполняется в собственном режиме в ядре SQL.

|Предупреждение|Описание|
|---|---|
|**Уязвимость внедрения кода SQL**|Приложение создало ошибочную инструкцию SQL в базе данных. Это может указывать на возможную уязвимость атак путем внедрения кода SQL. Существует две причины создания инструкции с ошибкой: Либо ошибка в коде приложения создавала ошибочную инструкцию SQL. Или в коде приложения или хранимых процедурах не удалось очистить ввод пользователя при создании ошибочной инструкции SQL, которая может быть использована для внедрения кода SQL.|
|**Потенциальная атака путем внедрения кода SQL**|Обнаружена активная атака для идентифицированного приложения, уязвимого для внедрения кода SQL. Это означает, что злоумышленник пытается внедрить вредоносные инструкции SQL, используя уязвимый код приложения или хранимые процедуры.|
|**Доступ из необычного расположения**|Изменился шаблон доступа к SQL Server, где кто-то вошел в SQL Server из необычного географического расположения. Иногда предупреждение определяет допустимые действия (новое приложение, обслуживание разработчиком). В других случаях предупреждение обнаруживает угрозу (бывшего сотрудника, внешнего злоумышленника).|
|**Доступ от незнакомых участников**|Изменился шаблон доступа к SQL Server. кто-то вошел в SQL Server с помощью необычного участника (пользователя SQL). Иногда предупреждение определяет допустимые действия (новое приложение, обслуживание разработчиком). В других случаях предупреждение обнаруживает угрозу (бывшего сотрудника, внешнего злоумышленника).|
|**Доступ из потенциально опасного приложения**|Для доступа к базе данных использовалось потенциально опасное приложение. В некоторых случаях предупреждения обнаруживают выполнение теста на проникновение. В других случаях — атаку с помощью обычных средств.|
|**Учетные данные SQL для принудительного подбора**|Возникло слишком большое число неудачных попыток входа с разными учетными данными. В некоторых случаях предупреждения обнаруживают выполнение теста на проникновение. В других случаях — атаку методом подбора.|

Дополнительные сведения о предупреждениях об обнаружении угроз SQL см. в разделе[обнаружение угроз базы данных SQL Azure](https://docs.microsoft.com/azure/sql-database/sql-database-threat-detection-overview)и просмотрите раздел оповещения об обнаружении угроз. Также Узнайте [, как центр безопасности Azure позволяет обнаружить кибератаках](https://azure.microsoft.com/blog/how-azure-security-center-helps-reveal-a-cyberattack/) , чтобы увидеть пример того, как центр безопасности использовал вредоносное обнаружение действий SQL для обнаружения атаки.

## Служба хранилища Azure<a name="azure-storage"></a>

>[!NOTE]
> Расширенная защита от угроз для службы хранилища Azure в настоящее время доступна только для хранилища BLOB-объектов. 

Расширенная защита от угроз Azure для службы хранилища Azure предоставляет дополнительный уровень безопасности. Она обнаруживает необычные и потенциально опасные попытки доступа к вашим учетным записям хранения или их использования. Этот уровень защиты позволяет устранять угрозы без необходимости быть экспертом по безопасности и управлять системами мониторинга безопасности.

Центр безопасности анализирует журналы диагностики запросов на чтение, запись и удаление в хранилище BLOB-объектов для обнаружения угроз и запускает оповещения при возникновении аномалий в действии. Дополнительные сведения см. в статье [Настройка ведения журнала аналитика службы хранилища](https://docs.microsoft.com/azure/storage/common/storage-monitor-storage-account#configure-logging) для получения дополнительных сведений.

> [!div class="mx-tableFixed"]

|Предупреждение|Описание|
|---|---|
|**Нестандартные аномалии при доступе к расположению**|Анализ сетевого трафика с выборкой обнаружил аномальный исходящий трафик протокол удаленного рабочего стола (RDP), поступающий из ресурса в развертывании. Это действие считается ненормальным для этой среды и может означать, что ваш ресурс был скомпрометирован и теперь используется для подбора внешней конечной точки RDP. Обратите внимание, что этот тип активности может привести к тому, что ваш IP-адрес будет помечен как вредоносный внешними сущностями.|
|**Аномалия при доступе к приложению**|Указывает, что у необычного приложения была доступ к этой учетной записи хранения. Потенциальная причина заключается в том, что злоумышленник получил доступ к вашей учетной записи хранения с помощью нового приложения.|
|**Аномалий анонимного доступа**|Указывает на изменение шаблона доступа к учетной записи хранения. Например, доступ к учетной записи осуществляется анонимно (без какой-либо проверки подлинности), что является непредвиденным по сравнению с последним шаблоном доступа в этой учетной записи. Потенциальная причина заключается в том, что злоумышленник использует общий доступ на чтение к контейнеру, содержащему хранилище BLOB-объектов.|
|**Аномалия утечка данных**|Указывает, что в этом контейнере хранилища были извлечены редко большие объемы данных по сравнению с последними действиями. Потенциальной причиной является то, что злоумышленник извлекли большой объем данных из контейнера, хранящего хранилище BLOB-объектов.|
|**Непредвиденная аномалия при удалении**|Указывает, что в учетной записи хранения произошла одна или несколько непредвиденных операций удаления, по сравнению с последней операцией в этой учетной записи. Потенциальная причина заключается в том, что злоумышленник удалил данные из вашей учетной записи хранения.|
|**Отправка пакета облачной службы Azure**|Указывает, что пакет облачной службы Azure (CSPKG-файл) был передан в учетную запись хранения обычным образом по сравнению с последними действиями в этой учетной записи. Потенциальная причина заключается в том, что злоумышленник готовится к развертыванию вредоносного кода из вашей учетной записи хранения в облачную службу Azure.|
|**Аномалии доступа к разрешениям**|Указывает, что разрешения на доступ к этому контейнеру хранилища были изменены необычным образом. Потенциальная причина заключается в том, что злоумышленник изменил разрешения контейнера, чтобы ослабить уровень безопасности или получить сохраняемость.|
|**Проверка доступа к проверке**|Указывает, что разрешения на доступ к учетной записи хранения были проверены необычным образом по сравнению с последними действиями в этой учетной записи. Потенциальная причина заключается в том, что злоумышленник выполнил разведывательная атака для атаки в будущем.|
|**Аномалия исследования данных**|Указывает, что большие двоичные объекты или контейнеры в учетной записи хранения были перечислены в ненормальном виде по сравнению с последними действиями в этой учетной записи. Потенциальная причина заключается в том, что злоумышленник выполнил разведывательная атака для атаки в будущем.|

>[!NOTE]
>В настоящее время Расширенная защита от угроз для службы хранилища Azure недоступна в регионах Azure для государственных организаций и независимых.

Дополнительные сведения о предупреждениях для хранилища см. в статье [Advanced Threat Protection для службы хранилища Azure](https://docs.microsoft.com/azure/storage/common/storage-advanced-threat-protection) и просмотрите раздел оповещения о защите.

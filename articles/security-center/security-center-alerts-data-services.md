# <a name="threat-detection-for-data-services-in-azure-security-center"></a>Обнаружение угроз для служб данных в центре безопасности Azure

 Центр безопасности анализирует журналы служб хранения данных и активирует оповещения при обнаружении угрозы для ресурсов данных. В этом разделе перечислены предупреждения, создаваемые в центре безопасности для следующих служб:

* [База данных Azure SQL и хранилище данных SQL](#data-sql)
* [Хранилище Azure](#azure-storage)

## База данных Azure SQL и хранилище данных SQL <a name="data-sql"></a>

Угроз SQL обнаружения выявляет аномальные операции, указывающие на нестандартные и потенциально вредоносные пытается получить доступ к или воспользоваться их уязвимостями баз данных. Центр безопасности анализирует журналы аудита SQL и выполняется скомпилированных в собственном коде в ядре SQL.

|Предупреждение|Описание|
|---|---|
|**Уязвимость для атак SQL Injection**|Приложения создала инструкцию SQL с ошибкой в базе данных. Это может указывать на возможную уязвимость к атакам путем внедрения кода SQL. Существует две причины создания инструкции с ошибкой: Либо дефект в коде приложения создается ошибочной инструкции SQL. Кроме того, код приложения или хранимые процедуры не очищают ввод данных пользователя при создании ошибочной инструкции SQL, которая может быть использована для внедрения кода SQL.|
|**Потенциальная атака путем внедрения кода SQL**|Active эксплойт произошла для определенного приложения уязвимым для атак SQL injection. Это означает, что злоумышленник пытается внедрить вредоносные инструкции SQL, с помощью кода уязвимого приложения или хранимых процедур.|
|**Доступ из незнакомого расположения**|Произошел изменении шаблона доступа к SQL server, если пользователь вошел сервер SQL из незнакомого географического расположения. Иногда предупреждение определяет допустимые действия (новое приложение, обслуживание разработчиком). В других случаях предупреждение обнаруживает угрозу (бывшего сотрудника, внешнего злоумышленника).|
|**Доступ из незнакомого субъекта**|Произошли изменения шаблона доступа к SQL server — пользователь вошел на SQL server, с помощью нестандартного субъекта (пользователя SQL). Иногда предупреждение определяет допустимые действия (новое приложение, обслуживание разработчиком). В других случаях предупреждение обнаруживает угрозу (бывшего сотрудника, внешнего злоумышленника).|
|**Доступ из потенциально опасного приложения**|Потенциально опасного приложения использует для доступа к базе данных. В некоторых случаях предупреждения обнаруживают выполнение теста на проникновение. В других случаях — атаку с помощью обычных средств.|
|**Подбор учетных данных SQL**|Произошли Аномальное большое число неудачных попыток входа с другими учетными данными. В некоторых случаях предупреждения обнаруживают выполнение теста на проникновение. В других случаях — атаку методом подбора.|

Дополнительные сведения см. оповещения обнаружения угроз SQL[обнаружения угроз базы данных SQL Azure](https://docs.microsoft.com/azure/sql-database/sql-database-threat-detection-overview)и ознакомьтесь с разделом оповещения обнаружения угроз. Также см. в разделе [как центр безопасности Azure помогает выявить Cyberattack](https://azure.microsoft.com/blog/how-azure-security-center-helps-reveal-a-cyberattack/) пример использования центра безопасности обнаружения в вредоносных действиях SQL для обнаружения атаки.

## Служба хранилища Azure<a name="azure-storage"></a>

>[!NOTE]
> Advanced Threat Protection для службы хранилища Azure только в настоящее время доступны для хранилища BLOB-объектов. 

Расширенная защита от угроз Azure для службы хранилища Azure предоставляет дополнительный уровень безопасности. Она обнаруживает необычные и потенциально опасные попытки доступа к вашим учетным записям хранения или их использования. Этот уровень защиты обеспечивает угрозы без необходимости эксперт по безопасности и управления системами мониторинга безопасности.

Центр безопасности анализирует журналы диагностики, чтения, записи и запросов на удаление в хранилище BLOB-объектов для обнаружения угроз, и он активирует оповещения при возникновении ошибки в действии. Дополнительные сведения см. в подразделе [настроить ведение журнала аналитики хранилища](https://docs.microsoft.com/azure/storage/common/storage-monitor-storage-account#configure-logging) Дополнительные сведения.

> [!div class="mx-tableFixed"]

|Предупреждение|Описание|
|---|---|
|**Аномалий доступа незнакомого расположения**|Анализ выборки сетевого трафика выявил аномальную исходящую связь протокола удаленного рабочего стола (RDP), исходящий из ресурсов в развертывании. Это действие считается ненормальным для этой среды и может означать, что ресурс была скомпрометирована и теперь используется для перебора внешнюю конечную точку RDP. Обратите внимание, что этот тип активности может привести к тому, что ваш IP-адрес будет помечен как вредоносный внешними сущностями.|
|**Аномалий доступа приложения**|Указывает, что необычные приложения доступ к этой учетной записи хранения. Возможная причина-, что злоумышленник получил доступ к вашей учетной записи хранения с помощью нового приложения.|
|**Анонимный доступ аномалий**|Указывает, что при наличии изменений в схемы доступа к учетной записи хранения. Например учетной записи уже был осуществлен анонимно (без проверки подлинности), это непредвиденное, по сравнению с последней схему доступа к этой учетной записи. Возможная причина-в том, что злоумышленник злоумышленниками общий доступ на чтение к контейнеру, содержит BLOB-объектов хранилища.|
|**Аномалий утечка данных**|Указывает, что были извлечены слишком большой объем данных по сравнению с последних действий в этом контейнере хранилища. Возможная причина-, что злоумышленник извлечен большой объем данных из контейнера, содержит BLOB-объектов хранилища.|
|**Непредвиденная delete аномалий**|Указывает, что один или несколько операций удаления Непредвиденная произошло в учетной записи хранения по сравнению с последние действия в этой учетной записи. Возможная причина-, что он удален из вашей учетной записи хранения данных.|
|**Отправьте пакет облачной службы Azure**|Указывает, что пакет облачной службы (cspkg-файл) был загружен в учетную запись хранения необычным способом, по сравнению с последние действия в этой учетной записи. Возможная причина-, что злоумышленник Подготовка к развертыванию вредоносный код из вашей учетной записи хранения в Azure облачную службу.|
|**Разрешение доступа аномалий**|Указывает, что разрешения на доступ этого контейнера хранилища были изменены необычным способом. Возможная причина-, что он был изменен разрешения контейнера ослабление безопасности или для получения постоянного хранения.|
|**Проверки доступа аномалий**|Указывает, что разрешения на доступ учетной записи хранения уже было проверено необычным способом, по сравнению с последние действия в этой учетной записи. Возможная причина-, что злоумышленник выполнил разведка атак в будущем.|
|**Данные исследования аномалий**|Указывает, что перечисленные больших двоичных объектов или контейнеров в учетной записи хранения будет использоваться необычным способом, по сравнению с последние действия в этой учетной записи. Возможная причина-, что злоумышленник выполнил разведка атак в будущем.|

>[!NOTE]
>Advanced Threat Protection для службы хранилища Azure сейчас недоступна в Azure для государственных организаций и независимых облачных регионов.

Дополнительные сведения об оповещениях для хранения см. в разделе [Advanced Threat Protection для службы хранилища Azure](https://docs.microsoft.com/azure/storage/common/storage-advanced-threat-protection) статьи и ознакомьтесь с разделом предупреждения защиты.

# <a name="threat-detection-for-data-services-in-azure-security-center"></a>Обнаружение угроз для служб данных в центре безопасности Azure

 Центр безопасности Azure анализирует журналы служб хранилища данных и запускает оповещения при обнаружении угрозы для ресурсов данных. В этом разделе перечислены оповещения, создаваемые центром безопасности для следующих служб:

* [База данных SQL Azure и хранилище данных SQL](#data-sql)
* [Хранилище Azure](#azure-storage)
* [Azure Cosmos DB](#cosmos-db)

## База данных SQL и хранилище данных SQL<a name="data-sql"></a>

Обнаружение угроз SQL определяет аномальные действия, указывающие на необычные и потенциально опасные попытки доступа к базам данных или их использования. Центр безопасности анализирует журналы аудита SQL и выполняется в собственном режиме в ядре SQL.

|Оповещение|Описание|
|---|---|
|**Уязвимость внедрения кода SQL**|Приложение создало ошибочную инструкцию SQL в базе данных. Это может указывать на возможную уязвимость атак путем внедрения кода SQL. Существует две возможные причины неудачной инструкции. Ошибка в коде приложения может быть сконструирована с ошибкой инструкции SQL. Или код приложения или хранимые процедуры не обвернули данные, введенные пользователем при создании ошибочной инструкции SQL, которую можно использовать для внедрения кода SQL.|
|**Потенциальная атака путем внедрения кода SQL**|Обнаружена активная атака для идентифицированного приложения, уязвимого для внедрения кода SQL. Это означает, что злоумышленник пытается внедрить вредоносные инструкции SQL с помощью кода уязвимого приложения или хранимых процедур.|
|**Доступ из необычного расположения**|В шаблоне доступа внесены изменения в SQL Server, когда кто-то вошел на сервер из необычного географического расположения. Иногда предупреждение определяет допустимые действия (новое приложение, обслуживание разработчиком). В других случаях предупреждение обнаруживает вредоносное действие (бывший сотрудник или внешний злоумышленник).|
|**Доступ от незнакомых участников**|Изменился шаблон доступа для SQL Server. Кто-то вошел на сервер с помощью необычного участника (пользователя). Иногда предупреждение определяет допустимые действия (новое приложение, обслуживание разработчиком). В других случаях предупреждение обнаруживает вредоносное действие (бывший сотрудник или внешний злоумышленник).|
|**Доступ из потенциально опасного приложения**|Для доступа к базе данных использовалось потенциально опасное приложение. В некоторых случаях предупреждения обнаруживают выполнение теста на проникновение. В других случаях предупреждение обнаруживает атаку, использующую общие средства.|
|**Учетные данные SQL для принудительного подбора**|Возникло слишком большое число неудачных входов с разными учетными данными. В некоторых случаях предупреждения обнаруживают выполнение теста на проникновение. В других случаях — атаку методом подбора.|

Дополнительные сведения о предупреждениях об обнаружении угроз SQL см. в статье [обнаружение угроз для базы данных SQL Azure](https://docs.microsoft.com/azure/sql-database/sql-database-threat-detection-overview). В частности, ознакомьтесь с разделом оповещения об обнаружении угроз. Также Узнайте [, как центр безопасности Azure позволяет обнаружить кибератаках](https://azure.microsoft.com/blog/how-azure-security-center-helps-reveal-a-cyberattack/) , чтобы увидеть пример того, как центр безопасности использовал вредоносное обнаружение действий SQL для обнаружения атаки.

## Объема<a name="azure-storage"></a>

>[!NOTE]
> Расширенная защита от угроз для хранилища сейчас доступна только для хранилища BLOB-объектов.

Расширенная защита от угроз для хранилища предоставляет дополнительный уровень логики безопасности, который обнаруживает необычные и потенциально опасные попытки доступа или использования учетных записей хранения. Этот уровень защиты позволяет устранять угрозы без необходимости быть экспертом по безопасности и управлять системами мониторинга безопасности.

Центр безопасности анализирует журналы диагностики запросов на чтение, запись и удаление в хранилище BLOB-объектов для обнаружения угроз и запускает оповещения при возникновении аномалий в действии. Дополнительные сведения см. в разделе [Настройка ведения журнала аналитика службы хранилища](https://docs.microsoft.com/azure/storage/common/storage-monitor-storage-account#configure-logging).

> [!div class="mx-tableFixed"]

|Оповещение|Описание|
|---|---|
|**Нестандартные аномалии при доступе к расположению**|Анализ сетевого трафика с выборкой обнаружил аномальную исходящую связь протокол удаленного рабочего стола (RDP), которая исходит от ресурса в развертывании. Это действие считается ненормальным для этого окружения. Это может означать, что ваш ресурс был скомпрометирован, и теперь он используется для взлома внешней конечной точки RDP. Обратите внимание, что этот тип действия может привести к тому, что ваш IP-адрес будет помечен как вредоносный внешними сущностями.|
|**Аномалия при доступе к приложению**|Указывает, что у необычного приложения была доступ к этой учетной записи хранения. Потенциальная причина заключается в том, что злоумышленник получил доступ к вашей учетной записи хранения с помощью нового приложения.|
|**Аномалий анонимного доступа**|Указывает на изменение шаблона доступа к учетной записи хранения. Например, доступ к учетной записи осуществляется анонимно (без какой-либо проверки подлинности), что является непредвиденным по сравнению с последним шаблоном доступа в этой учетной записи. Потенциальной причиной является то, что злоумышленник использует общий доступ на чтение к контейнеру, содержащему хранилище BLOB-объектов.|
|**Tor аномалии**|Указывает, что доступ к этой учетной записи был успешно получен с IP-адреса, известного как активный узел выхода Tor (прокси-сервер анонимизирующие). Серьезность этого предупреждения рассматривает используемый тип проверки подлинности (если он есть) и является ли это первым вариантом такого доступа. Потенциальными причинами могут быть злоумышленники, которые обращаются к вашей учетной записи хранения с помощью Tor или авторизованного пользователя, который получил доступ к вашей учетной записи хранения с помощью Tor.|
|**Аномалия утечка данных**|Указывает, что в этом контейнере хранилища были извлечены редко большие объемы данных по сравнению с последними действиями. Потенциальной причиной является то, что злоумышленник извлекли большой объем данных из контейнера, содержащего хранилище BLOB-объектов.|
|**Непредвиденная аномалия при удалении**|Указывает, что в учетной записи хранения произошла одна или несколько непредвиденных операций удаления, по сравнению с последней операцией в этой учетной записи. Потенциальная причина заключается в том, что злоумышленник удалил данные из вашей учетной записи хранения.|
|**Отправка пакета облачных служб Azure**|Указывает, что пакет облачных служб Azure (CSPKG-файл) был передан в учетную запись хранения непривычным образом по сравнению с последними действиями в этой учетной записи. Потенциальная причина заключается в том, что злоумышленник готовится к развертыванию вредоносного кода из вашей учетной записи хранения в облачную службу Azure.|
|**Аномалии доступа к разрешениям**|Указывает, что разрешения на доступ к этому контейнеру хранилища были изменены необычным образом. Потенциальная причина заключается в том, что злоумышленник изменил разрешения контейнера, чтобы ослабить уровень безопасности или получить сохраняемость.|
|**Проверка доступа к проверке**|Указывает, что разрешения на доступ к учетной записи хранения были проверены необычным образом по сравнению с последними действиями в этой учетной записи. Потенциальная причина заключается в том, что злоумышленник выполнил разведывательная атака для атаки в будущем.|
|**Аномалия исследования данных**|Указывает, что большие двоичные объекты или контейнеры в учетной записи хранения были перечислены в ненормальном виде по сравнению с последними действиями в этой учетной записи. Потенциальная причина заключается в том, что злоумышленник выполнил разведывательная атака для атаки в будущем.|

>[!NOTE]
>В настоящее время Расширенная защита от угроз для хранилища недоступна в регионах Azure для государственных организаций и независимых.

Дополнительные сведения о предупреждениях для хранилища см. в статье [Advanced Threat Protection для службы хранилища Azure](../storage/common/storage-advanced-threat-protection.md). В частности, ознакомьтесь с разделом "оповещения о защите".

## Azure Cosmos DB<a name="cosmos-db"></a>

Следующие предупреждения генерируются необычными и потенциально опасными попытками доступа или использования Azure Cosmos DB учетных записей.

|Оповещение|Описание|
|---|---|
|**Доступ из необычного расположения**|Указывает, что в шаблоне доступа было внесено изменение в учетную запись Azure Cosmos DB. Кто-то получил доступ к этой учетной записи с незнакомого IP-адреса по сравнению с последним действием. Либо злоумышленник получил доступ к учетной записи, либо законный пользователь получил доступ к нему из нового и необычного географического расположения. Примером последнего является удаленное обслуживание нового приложения или разработчика.|
|**Необычные утечка данных**|Указывает на изменение шаблона извлечения данных из учетной записи Azure Cosmos DB. Пользователь извлекли необычное количество данных по сравнению с последними действиями. Злоумышленник может извлечь большой объем данных из Azure Cosmos DB базы данных (например, утечка данных или утечки или несанкционированную пересылку данных). Либо законный пользователь или приложение могли извлечь необычное количество данных из контейнера (например, для действия "обслуживание резервного копирования").|

Дополнительные сведения см. в разделе [Advanced Threat Protection для Azure Cosmos DB](../cosmos-db/cosmos-db-advanced-threat-protection.md).

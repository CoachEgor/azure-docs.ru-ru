# <a name="introduction-to-azure-defender"></a>Общие сведения о защитнике Azure

Функции центра безопасности Azure охватывают два основных аспекта безопасности в облаке:

- **Управление безопасностью в облаке (КСПМ)**
- **Защита облачных рабочих нагрузок (CWP)**

Функции КСПМ центра безопасности, такие как безопасность оценки, обнаружение недопустимых конфигураций безопасности на компьютерах Windows и Linux Azure, являются частью бесплатного интерфейса центра безопасности, доступного для всех пользователей Azure. Используйте эти КСПМ функции, чтобы усилить уровень и обеспечить соответствие нормативным требованиям.

**Защитник Azure** — это облачная платформа защиты рабочих нагрузок (квпп), интегрированная в центр безопасности, для расширенной и интеллектуальной защиты рабочих нагрузок Azure и гибридной рабочей нагрузки.

## <a name="what-resource-types-can-azure-defender-secure"></a>Какие типы ресурсов могут быть защищены защитником Azure?

Защитник Azure предоставляет оповещения системы безопасности и расширенную защиту от угроз для виртуальных машин, баз данных SQL, контейнеров, веб-приложений, сети и т. д.

При включении защитника Azure из области " **цены и параметры** " центра безопасности Azure все следующие планы защитника включены одновременно и обеспечивают комплексную защиту для уровней вычислений, данных и служб в вашей среде.

- [Azure Defender для серверов](defender-for-servers-introduction.md)
- [Azure Defender для Службы приложений](defender-for-app-service-introduction.md)
- [Azure Defender для службы хранилища](defender-for-storage-introduction.md)
- [Azure Defender для SQL](defender-for-sql-introduction.md)
- [Azure Defender для Интернета вещей](defender-for-iot-introduction.md)
- [Azure Defender для Kubernetes](defender-for-kubernetes-introduction.md)
- [Azure Defender для реестров контейнеров](defender-for-container-registries-introduction.md)
- [Azure Defender для Key Vault](defender-for-key-vault-introduction.md)

Каждый из этих планов описан отдельно в документации по центру безопасности.


## <a name="hybrid-cloud-protection"></a>Гибридная защита облака

Кроме защиты среды Azure, вы можете добавить возможности защитника Azure в гибридную облачную среду:

- Защитите свои серверы, не относящиеся к Azure
- Защитите виртуальные машины в других облаках (например, AWS и обеспечить).
- Защита устройств IoT

Вы получите настраиваемую аналитику угроз и приоритеты предупреждений в соответствии с конкретной средой, чтобы вы могли сосредоточиться на том, что самое важное

Разверните [Azure Arc](https://azure.microsoft.com/services/azure-arc/) и включите Защитник Azure, чтобы расширить защиту локальных и многооблачных виртуальных машин и баз данных SQL. Служба "Дуга Azure для серверов" является бесплатной службой, но службы, используемые на серверах с поддержкой Arc, например в защитнике Azure, будут оплачены согласно ценам для этой службы.

Дополнительные [сведения о службе "Дуга Azure](https://docs.microsoft.com/azure/azure-arc/overview)".


## <a name="azure-defender-alerts"></a>Оповещения в Azure Defender 

Когда защитник Azure обнаруживает угрозу в любой области вашей среды, он создает оповещение. В этих оповещениях содержатся сведения об оказавшихся под угрозой ресурсах и предлагаемые действия по исправлению. В некоторых случаях предоставляется также возможность запуска приложения логики в ответ на угрозу.

Можно ли экспортировать оповещение, созданное центром безопасности или полученное центром безопасности из интегрированного продукта безопасности. Чтобы экспортировать оповещения в Azure Sentinel, любое стороннее средство SIEM или другие внешние средства, следуйте инструкциям в статье [Экспорт оповещений о безопасности и рекомендаций](continuous-export.md).

> [!NOTE]
> Время, требуемое для отображения оповещений из разных источников, может отличаться. Например, для отображения оповещений, требующих анализа сетевого трафика, может потребоваться больше времени, чем для оповещений, связанных с выполнением подозрительных процессов на виртуальных машинах.


## <a name="azure-defender-advanced-protection-capabilities"></a>Возможности расширенной защиты в защитнике Azure

В защитнике Azure используется Расширенная аналитика для адаптированных рекомендаций, связанных с вашими ресурсами. 

Защита включает в себя защиту портов управления виртуальных машин с помощью JIT-доступа и адаптивных элементов управления приложениями, чтобы создать списки разрешений для приложений, которые не должны запускаться на ваших компьютерах. 

Используйте плитки расширенной защиты на панели мониторинга защитника Azure, чтобы отслеживать и настраивать все эти защиты. 

## <a name="vulnerability-assessment-and-management"></a>Оценка уязвимостей и управление ими

Защитник Azure включает поиск уязвимостей для виртуальных машин и реестров контейнеров без дополнительных затрат. Сканеры работают с Qualys, но не требуется лицензия Qualys или даже учетная запись Qualys — все, что полностью обрабатывалось в центре безопасности. 

Изучите результаты этих сканеров уязвимостей и ответьте на них в центре безопасности. Центр безопасности ближе всего к единой области прозрачности для обеспечения безопасности в облаке.

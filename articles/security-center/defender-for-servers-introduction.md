# <a name="introduction-to-azure-defender-for-servers"></a>Общие сведения о защитнике Azure для серверов

Защитник Azure для серверов добавляет средства обнаружения угроз и расширенную защиту для компьютеров под управлением Windows и Linux.

Для Windows служба "защитник Azure" интегрируется со службами Azure для мониторинга и защиты компьютеров под управлением Windows. Он предоставляет оповещения и рекомендации по исправлению изо всех этих служб в удобном для использования формате.

Для Linux защитник Azure собирает записи аудита с компьютеров Linux с помощью **аудита**, одной из наиболее распространенных платформ аудита Linux. Решение auditd размещено в магистральном ядре. 


## <a name="what-are-the-benefits-of-azure-defender-for-servers"></a>Каковы преимущества защитника Azure для серверов?

Ниже перечислены возможности обнаружения и защиты угроз, обеспечиваемые защитником Azure для серверов.

- **Сканирование оценки уязвимостей для виртуальных машин** . средство проверки уязвимостей, включенное в центр безопасности Azure, включено в Qualys. 

    Qualys "сканер — это одно из ведущих средств для идентификации уязвимостей на виртуальных машинах Azure в режиме реального времени. Вам не потребуется лицензия или учетная запись Qualys: вся обработка выполняется в Центре безопасности. [Подробнее](deploy-vulnerability-assessment-vm.md).

- **JIT-доступ к виртуальной машине**  . субъекты угроз активно прорегистрируют доступные компьютеры с открытыми портами управления, такими как RDP или SSH. Все ваши виртуальные машины являются потенциальными целями для атаки. Если виртуальная машина успешно скомпрометирована, она используется в качестве точки входа для атаки на дополнительные ресурсы в вашей среде.

    При включении защитника Azure для серверов можно использовать JIT-доступ к виртуальным машинам для блокировки входящего трафика к виртуальным машинам, уменьшая вероятность атак и предоставляя простой доступ к виртуальным машинам при необходимости. [Подробнее](just-in-time-explained.md).

- **Мониторинг целостности файлов (FIM)** — мониторинг целостности файлов (FIM), также называемый мониторингом изменений, проверяет файлы и реестры операционной системы, программного обеспечения и других приложений на предмет изменений, которые могут указывать на атаку. Метод сравнения позволяет определить, отличается ли текущее состояние файла от состояния на момент последней проверки. Такое сравнение позволяет узнать, вносились ли в файлы допустимые или подозрительные изменения.

    При включении защитника Azure для серверов можно использовать FIM для проверки целостности файлов Windows, реестров Windows и файлов Linux. [Подробнее](security-center-file-integrity-monitoring.md).

- **Адаптивные элементы управления приложениями (AAC)** . адаптивные элементы управления приложениями — это интеллектуальное и автоматизированное решение для определения списков разрешений для известных приложений на компьютерах.

    Если вы включили и настроили адаптивные элементы управления приложениями, вы получите оповещения системы безопасности, если какое бы другое приложение запускалось, кроме тех, которые определены как надежные. [Подробнее](security-center-adaptive-application.md).

- **Адаптивное усиление защиты сети (АНХ)** . применение групп безопасности сети (NSG) для фильтрации трафика между и из ресурсов повышает уровень безопасности сети. Тем не менее в некоторых случаях фактический трафик, передаваемый через NSG, может быть подмножеством определенных правил NSG. В таких случаях дальнейшее улучшение безопасности может быть достигнуто за счет усиления правил NSG на основе реальных шаблонов трафика.

    Адаптивное усиление защиты сети предоставляет рекомендации для дальнейшей защиты правил NSG. Он использует алгоритм машинного обучения, который учитывает фактический трафик, известную надежную конфигурацию, аналитику угроз и другие признаки компрометации, а затем предоставляет рекомендации по разрешению трафика только из конкретных кортежей IP/Port. [Подробнее](security-center-adaptive-network-hardening.md).

- **Интеграция с Microsoft защитника Advanced Threat protection (ATP) (только для Windows)** . Служба "защитник Azure" интегрируется с Windows Advanced Threat protection (ATP). Вместе они обеспечивают широкие возможности обнаружения конечных точек и реагирования. [Подробнее](security-center-wdatp.md).

    > [!IMPORTANT]
    > Датчик ATP в Microsoft Defender автоматически включается для серверов Windows, которые используют Центр безопасности.

    Когда средство ATP в Microsoft Defender обнаруживает угрозу, оно инициирует оповещение. Оповещение отображается в центре безопасности. В центре безопасности можно также выполнить сведение к консоли Microsoft Defender ATP и провести подробное исследование, чтобы выявить область атаки. Дополнительные сведения об ATP в Microsoft Defender см. в статье [Подключение серверов к службе ATP в Microsoft Defender](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/configure-server-endpoints).

- **Усиление защиты узла DOCKER** . Центр безопасности Azure определяет неуправляемые контейнеры, размещенные на виртуальных машинах IaaS Linux или на других компьютерах Linux с контейнерами DOCKER. Центр безопасности постоянно оценивает конфигурации этих контейнеров. Затем он сравнивает их с центром тестирования в центре безопасности Интернета (CIS) DOCKER. Центр безопасности включает весь набор правил тестового тестирования DOCKER и предупреждает вас, если контейнеры не соответствуют какому бы то ни было элементу управления. [Подробнее](harden-docker-hosts.md).

- **Обнаружение атак без файлов (только для Windows)** . атаки, не имеющие файлов, вставляют вредоносные полезные данные в память во избежание обнаружения с помощью методов сканирования на основе дисков. Затем полезные данные злоумышленника сохраняются в памяти скомпрометированных процессов и выполняют широкий спектр вредоносных действий.

  При обнаружении атак такого рода используются автоматизированные средства криминалистического анализа, которые позволяют выявить наборы средств, методики и поведение, применяемые для осуществления бесфайловых атак. Это решение периодически сканирует компьютер во время выполнения и извлекает аналитические сведения непосредственно из памяти процессов. Конкретные аналитические сведения включают в себя идентификацию: 

  - Хорошо известные наборы средств и программное обеспечение для интеллектуального анализа данных 

  - Обнаружение кода оболочки — это небольшой фрагмент кода, который обычно используется в качестве полезных данных при использовании уязвимости программного обеспечения.

  - Внедренный вредоносный исполняемый файл в памяти процесса

  Обнаружение атак без файлов создает подробные оповещения системы безопасности, содержащие описания с дополнительными метаданными процесса, такими как сетевая активность. Это ускоряет рассмотрение предупреждений, корреляцию и время отклика. Этот подход дополняет ЕДР решения на основе событий и обеспечивает повышенный объем обнаружения.

  Дополнительные сведения об оповещениях об обнаружении бесфайловых атак см. в [справочной таблице оповещений](alerts-reference.md#alerts-windows).

- Контрольные **оповещения Linux и интеграция агента log Analytics (только для Linux)** . аудит системы состоит из подсистемы уровня ядра, которая отвечает за наблюдение за системными вызовами. Она фильтрует их по указанному набору правил и записывает для них сообщения в сокет. Центр безопасности интегрирует функциональные возможности из пакета auditd с агентом Log Analytics. Эта интеграция обеспечивает сбор событий аудита во всех поддерживаемых дистрибутивах Linux без каких бы то ни было обязательных компонентов.

    Записи auditd собираются, обогащаются и объединяются в события с помощью агента Log Analytics для агента Linux. В Центр безопасности постоянно добавляется новая аналитика, которая использует сигналы Linux для обнаружения вредоносного поведения на облачных и локальных компьютерах с Linux. Как и в случае с Windows, эти аналитические данные охватывают все подозрительные процессы, сомнительные попытки входа, загрузку модуля ядра и другие действия. Эти действия могут указывать на то, что компьютер атакуют злоумышленники либо он взломан.  

    Список оповещений Linux см. в [справочной таблице оповещений](alerts-reference.md#alerts-linux).

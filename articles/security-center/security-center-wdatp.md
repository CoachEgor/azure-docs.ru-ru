# <a name="microsoft-defender-advanced-threat-protection-with-azure-security-center"></a>Усовершенствованная защита от угроз Microsoft Defender с помощью центра безопасности Azure

Центр безопасности Azure расширяет возможности облачных платформ по защите рабочих нагрузок благодаря интеграции с [Microsoft защитника Advanced Threat protection](https://www.microsoft.com/microsoft-365/windows/microsoft-defender-atp) (ATP).
Это изменение обеспечивает широкие возможности обнаружения конечных точек и реагирования на них. С помощью интеграции ATP в защитнике Майкрософт можно разявлять ненормальные выявление. Кроме того, можно обнаруживать расширенные атаки на конечные точки сервера, отслеживаемые центром безопасности Azure, и реагировать на них.

## <a name="microsoft-defender-atp-features-in-security-center"></a>Функции ATP в защитнике Майкрософт в центре безопасности

При использовании ATP в защитнике Майкрософт вы получаете:

- **Усовершенствованные датчики обнаружения после нарушения**: датчики ATP в защитнике Майкрософт для серверов Windows собираются большие массивы сигналов о поведении.

- **Обнаружение нарушений в облаке, основанное на аналитике**. ATP в защитнике Майкрософт быстро адаптируется к изменяющимся угрозам. Эта служба использует расширенную аналитику и большие данные. Служба ATP в защитнике Майкрософт распространяется на мощь Intelligent Security Graph с сигналами по Windows, Azure и Office для обнаружения неизвестных угроз. Таким образом, вы получаете действенные оповещения и можете быстро реагировать на угрозы.

- **Аналитика угроз**. ATP в защитнике Майкрософт создает оповещения, когда он определяет средства, методы и процедуры злоумышленников. В этой службе используются данные, создаваемые командами по поиску угроз и командами по безопасности Майкрософт, а также аналитические сведения, предоставляемые партнерами.

В центре безопасности Azure теперь доступны следующие возможности.

- **Автоматизированное**подключение. датчик ATP для защитника Майкрософт включен автоматически для серверов Windows, подключенных к центру безопасности Azure.

- **Одиночная область стекла**: в консоли центра безопасности Azure ОТОБРАЖАЮТСЯ оповещения ATP в защитнике Майкрософт.

- **Подробное исследование машин**. клиенты центра безопасности Azure могут воспользоваться консолью ATP в защитнике Майкрософт, чтобы провести подробное исследование, чтобы выявить область нарушения.

![Центр безопасности Azure, где отображается список оповещений и общая информация о каждом из них](media/security-center-wdatp/image1.png)

Для дальнейшего изучения используйте ATP в защитнике Майкрософт. Пакет ATP в защитнике Майкрософт предоставляет дополнительные сведения, такие как дерево процессов предупреждений и граф инцидентов. Вы также можете увидеть подробную временную шкалу компьютера, где показана каждая реакция на событие за исторический период до шести месяцев.

![Страница ATP в защитнике Майкрософт с подробными сведениями о предупреждении](media/security-center-wdatp/image3.png)

## <a name="platform-support"></a>Поддержка платформы

ATP в защитнике Майкрософт в центре безопасности поддерживает обнаружение в Windows Server 2016, 2012 R2 и 2008 R2 с пакетом обновления 1 (SP1) для виртуальных машин Azure, для которых требуется подписка уровня Standard и для виртуальных машин, отличных от Azure, необходим уровень Standard только на уровне рабочей области.

> [!NOTE]
> При использовании центра безопасности Azure для мониторинга серверов автоматически создается клиент ATP в защитнике Майкрософт, а данные ATP в защитнике Майкрософт по умолчанию хранятся в Европе. Если необходимо переместить данные в другое расположение, необходимо обратиться в служба поддержки Майкрософт для сброса клиента. Мониторинг конечных точек сервера с использованием этой интеграции отключен для клиентов Office 365 GCC.

## <a name="onboarding-servers-to-security-center"></a>Подключение серверов к Центру безопасности 

Чтобы подключить серверы к центру безопасности, щелкните **"переход к центру безопасности Azure** ", чтобы подключить серверы из адаптации сервера ATP в защитнике Майкрософт.

1. В области **адаптации** выберите или создайте рабочую область, в которой должны храниться данные. <br>
2. Если отображаются не все рабочие области, это может быть вызвано отсутствием разрешений. Убедитесь, что для вашей рабочей области выбран уровень "Стандартный" Центра безопасности Azure. Дополнительные сведения см. [в статье обновление до уровня "Стандартный" центра безопасности для повышения безопасности](security-center-pricing.md).
    
3. Выберите **Добавить серверы**, чтобы просмотреть инструкции по установке Microsoft Monitoring Agent. 

4. После подключения вы сможете выполнять мониторинг компьютеров в разделе **Вычисления и приложения**.

   ![Подключение компьютеров](media/security-center-wdatp/onboard-computers.png)

## <a name="enable-microsoft-defender-atp-integration"></a>Включить интеграцию ATP в защитнике Майкрософт

Чтобы просмотреть, включена ли интеграция Microsoft Defender ATP, выберите **Центр безопасности** > **цены & параметры** > щелкните подписку.
Здесь можно увидеть включенные в данный момент интеграции.

  ![Страница параметров обнаружения угроз в центре безопасности Azure с включенной интеграцией Microsoft Defender ATP](media/security-center-wdatp/enable-integrations.png)

- Если вы уже подключили серверы в центре безопасности Azure уровня "Стандартный", вам не требуется предпринимать никаких дополнительных действий. Центр безопасности Azure автоматически направит серверы в службу ATP в защитнике Майкрософт. Адаптации может занять до 24 часов.

- Если вы никогда не подключали серверы в центре безопасности Azure уровня "Стандартный", подключите их к нему обычным образом.

- Если серверы были подключены с помощью ATP в защитнике Майкрософт:
  - Обратитесь к руководству по [отключению серверных компьютеров](https://go.microsoft.com/fwlink/p/?linkid=852906).
  - Подключите эти серверы к центру безопасности Azure.

## <a name="access-to-the-microsoft-defender-atp-portal"></a>Доступ к порталу ATP в защитнике Майкрософт

Следуйте инструкциям в статье по [назначению пользовательского доступа к порталу](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/assign-portal-access).

## <a name="set-the-firewall-configuration"></a>Настройка конфигурации брандмауэра

Если у вас есть прокси-сервер или брандмауэр, блокирующий анонимный трафик, так как датчик ATP в защитнике Майкрософт подключается из контекста системы, убедитесь, что разрешен анонимный трафик. Следуйте инструкциям в разделе [Включение доступа к URL-адресам службы ATP в защитнике Майкрософт на прокси-сервере](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/configure-proxy-internet#enable-access-to-microsoft-defender-atp-service-urls-in-the-proxy-server).

## <a name="test-the-feature"></a>Тестирование компонента

Создание некритического тестового оповещения ATP в защитнике Майкрософт:

1. Создайте папку "К:\тест-мдатп-тест".

1. Используйте удаленный рабочий стол для доступа к виртуальной машине Windows Server 2012 R2 или Windows Server 2016. Откройте окно командной строки.

1. В командной строке скопируйте и выполните следующую команду: Окно командной строки закроется автоматически.

    ```
    powershell.exe -NoExit -ExecutionPolicy Bypass -WindowStyle Hidden (New-Object System.Net.WebClient).DownloadFile('http://127.0.0.1/1.exe', 'C:\\test-MDATP-test\\invoice.exe'); Start-Process 'C:\\test-MDATP-test\\invoice.exe'
    ```

   ![Окно командной строки с командой выше](media/security-center-wdatp/image4.jpeg)

3. Если команда выполнена успешно, вы увидите новое оповещение на панели мониторинга центра безопасности Azure и на портале ATP в защитнике Майкрософт. Это оповещение может отобразиться через несколько минут.

4. Чтобы просмотреть оповещение в центре безопасности, перейдите в раздел **Оповещения системы безопасности** > **Suspicious Powershell CommandLine** (Подозрительная командная строка PowerShell).

5. В окне расследования выберите ссылку для перехода на портал ATP в защитнике Майкрософт.

# <a name="microsoft-defender-advanced-threat-protection-with-azure-security-center"></a>Усовершенствованная защита от угроз Microsoft Defender с помощью центра безопасности Azure

## <a name="microsoft-defender-atp-features-in-security-center"></a>Функции ATP в защитнике Майкрософт в центре безопасности

Пакет ATP в защитнике Майкрософт предоставляет:

- **Усовершенствованные датчики обнаружения после нарушения**: датчики ATP в защитнике Майкрософт для серверов Windows собираются большие массивы сигналов о поведении.

- **Обнаружение нарушений в облаке, основанное на аналитике**. ATP в защитнике Майкрософт быстро адаптируется к изменяющимся угрозам. Эта служба использует расширенную аналитику и большие данные. Служба ATP в защитнике Майкрософт распространяется на мощь Intelligent Security Graph с сигналами по Windows, Azure и Office для обнаружения неизвестных угроз. Таким образом, вы получаете действенные оповещения и можете быстро реагировать на угрозы.

- **Аналитика угроз**. ATP в защитнике Майкрософт создает оповещения, когда он определяет средства, методы и процедуры злоумышленников. В этой службе используются данные, создаваемые командами по поиску угроз и командами по безопасности Майкрософт, а также аналитические сведения, предоставляемые партнерами.


Интеграция пакета ATP в защитник с центром безопасности Azure обеспечивает также следующие дополнительные возможности:

- **Автоматизированное**подключение. Интеграция автоматически включает датчик Microsoft Defender ATP для серверов Windows, отслеживаемых центром безопасности Azure (если не используется Windows Server 2019).

- **Одиночная область стекла**: в консоли центра безопасности Azure ОТОБРАЖАЮТСЯ оповещения ATP в защитнике Майкрософт. Для дальнейшего изучения используйте ATP в защитнике Майкрософт. Пакет ATP в защитнике Майкрософт предоставляет дополнительные сведения, такие как дерево процессов предупреждений и граф инцидентов. Вы также можете увидеть подробную временную шкалу компьютера, где показана каждая реакция на событие за исторический период до шести месяцев.

    ![Страница ATP в защитнике Майкрософт с подробными сведениями о предупреждении](media/security-center-wdatp/image3.png)

## <a name="platform-support"></a>Поддержка платформы

ATP в защитнике Майкрософт в центре безопасности поддерживает обнаружение в Windows Server 2016, 2012 R2 и 2008 R2 с пакетом обновления 1 (SP1). Для виртуальных машин Azure требуется подписка уровня Standard и для виртуальных машин, не относящихся к Azure, необходим уровень Standard только на уровне рабочей области.

Мониторинг конечных точек сервера с помощью этой интеграции отключен для клиентов Office 365 GCC.

## <a name="data-storage-location"></a>Место хранения данных

При использовании центра безопасности Azure для мониторинга серверов автоматически создается клиент ATP в защитнике Майкрософт. Данные, собираемые ATP в защитнике Майкрософт, хранятся в географическом расположении клиента, как определено во время подготовки. Данные клиента в форме анонимные также могут храниться в центральных хранилищах и системах обработки в США. 

Настроенное место хранения данных изменить невозможно. Если необходимо переместить данные в другое расположение, обратитесь в служба поддержки Майкрософт для сброса клиента.


## <a name="onboard-servers-to-security-center"></a>Подключение серверов к центру безопасности 

Чтобы подключить серверы к центру безопасности, щелкните **"переход к центру безопасности Azure** ", чтобы подключить серверы из адаптации сервера ATP в защитнике Майкрософт.

1. В области **адаптации** выберите или создайте рабочую область, в которой должны храниться данные.

2. Если вы не видите все рабочие области, возможно, это вызвано отсутствием разрешений, убедитесь, что для рабочей области задана ценовая категория "Стандартный" безопасности Azure. Дополнительные сведения см. [в статье обновление до уровня "Стандартный" центра безопасности для повышения безопасности](security-center-pricing.md).
    
3. Выберите **Добавить серверы** , чтобы просмотреть инструкции по установке агента log Analytics. 

4. После адаптации можно отслеживать компьютеры в разделе **COMPUTE and Apps**.

   ![Подключение компьютеров](media/security-center-wdatp/onboard-computers.png)

## <a name="enable-microsoft-defender-atp-integration"></a>Включить интеграцию ATP в защитнике Майкрософт

Чтобы просмотреть, включена ли интеграция Microsoft Defender ATP, выберите параметры цены **центра безопасности**  >  **& настройки** > щелкните подписку.
Здесь можно увидеть включенные в данный момент интеграции.

  ![Страница параметров обнаружения угроз в центре безопасности Azure с включенной интеграцией Microsoft Defender ATP](media/security-center-wdatp/enable-integrations.png)

- Если вы уже подключили серверы в центре безопасности Azure уровня "Стандартный", вам не требуется предпринимать никаких дополнительных действий. Центр безопасности Azure автоматически направит серверы в службу ATP в защитнике Майкрософт. Адаптации может занять до 24 часов.

- Если вы никогда не подключали серверы в центре безопасности Azure уровня "Стандартный", подключите их к нему обычным образом.

- Если серверы были подключены с помощью ATP в защитнике Майкрософт:
  - Обратитесь к руководству по [отключению серверных компьютеров](https://go.microsoft.com/fwlink/p/?linkid=852906).
  - Подключите эти серверы к центру безопасности Azure.

## <a name="access-to-the-microsoft-defender-atp-portal"></a>Доступ к порталу ATP в защитнике Майкрософт

1. Следуйте инструкциям в статье по [назначению пользовательского доступа к порталу](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/assign-portal-access).

1. Проверьте наличие прокси-сервера или брандмауэра, блокирующего анонимный трафик. Датчик ATP в защитнике подключается к системному контексту, поэтому необходимо разрешить анонимный трафик. Чтобы обеспечить неограниченный доступ к порталу ATP в защитнике Майкрософт, следуйте инструкциям в разделе [Включение доступа к URL-адресам службы ATP в защитнике Майкрософт на прокси-сервере](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/configure-proxy-internet#enable-access-to-microsoft-defender-atp-service-urls-in-the-proxy-server).

## <a name="test-the-feature"></a>Тестирование компонента

Создание некритического тестового оповещения ATP в защитнике Майкрософт:

1. Создайте папку "К:\тест-мдатп-тест".

1. Используйте удаленный рабочий стол для доступа к виртуальной машине Windows Server 2012 R2 или Windows Server 2016. Откройте окно командной строки.

1. В командной строке скопируйте и выполните следующую команду: Окно командной строки закроется автоматически.

    ```
    powershell.exe -NoExit -ExecutionPolicy Bypass -WindowStyle Hidden (New-Object System.Net.WebClient).DownloadFile('http://127.0.0.1/1.exe', 'C:\\test-MDATP-test\\invoice.exe'); Start-Process 'C:\\test-MDATP-test\\invoice.exe'
    ```

   ![Окно командной строки с командой выше](media/security-center-wdatp/image4.jpeg)

1. Если команда выполнена успешно, вы увидите новое оповещение на панели мониторинга центра безопасности Azure и на портале ATP в защитнике Майкрософт. Это оповещение может отобразиться через несколько минут.

1. Чтобы ознакомиться с оповещением в центре безопасности, перейдите в раздел **оповещения системы безопасности**  >  **подозрительные команды PowerShell**.

1. В окне расследования выберите ссылку для перехода на портал ATP в защитнике Майкрософт.

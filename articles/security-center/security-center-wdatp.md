# <a name="microsoft-defender-advanced-threat-protection-with-azure-security-center"></a>Microsoft Defender Расширенная защита от угроз с Azure центр безопасности

Центр безопасности Azure расширяет свои облачные платформы защиты рабочей нагрузки, интегрируясь с [Microsoft Defender Advanced Threat Protection](https://www.microsoft.com/microsoft-365/windows/microsoft-defender-atp) (ATP).
Это изменение обеспечивает широкие возможности обнаружения конечных точек и реагирования на них. С интеграцией Microsoft Defender ATP можно обнаружить аномалии. Кроме того, можно обнаруживать расширенные атаки на конечные точки сервера, отслеживаемые центром безопасности Azure, и реагировать на них.

## <a name="microsoft-defender-atp-features-in-security-center"></a>Microsoft Defender ATP функции в Центре безопасности

При использовании АТС Microsoft Defender вы получаете:

- **Расширенные датчики обнаружения после нарушения:** датчики АТС Microsoft Defender для серверов Windows собирают широкий спектр поведенческих сигналов.

- **Аналитическое обнаружение пост-нарушений на основе аналитики:** Microsoft Defender ATP быстро адаптируется к меняющимся угрозам. Эта служба использует расширенную аналитику и большие данные. Microsoft Defender ATP усиливается благодаря возможности интеллектуального графика безопасности с сигналами через Windows, Azure и Office для обнаружения неизвестных угроз. Таким образом, вы получаете действенные оповещения и можете быстро реагировать на угрозы.

- **Угроза интеллекта**: Microsoft Defender ATP генерирует оповещения, когда он определяет злоумышленник инструменты, методы и процедуры. В этой службе используются данные, создаваемые командами по поиску угроз и командами по безопасности Майкрософт, а также аналитические сведения, предоставляемые партнерами.

Следующие возможности теперь доступны в Центре безопасности Azure:

- **Автоматизированная посадка**: датчик АТС Microsoft Defender автоматически включен для серверов Windows, которые находятся на борту в Центр безопасности Azure.

- **Одноразовое стекло:** консоль Azure Security Center отображает оповещения Microsoft Defender ATP.

- **Детальное исследование машины:** Клиенты Центра безопасности Azure могут использовать консоль Microsoft Defender ATP для проведения детального расследования, чтобы выявить масштабы нарушения.

![Центр безопасности Azure, где отображается список оповещений и общая информация о каждом из них](media/security-center-wdatp/image1.png)

Для дальнейшего исследования используйте Microsoft Defender ATP. Microsoft Defender ATP предоставляет дополнительную информацию, такую как дерево процесса оповещения и график инцидента. Вы также можете увидеть подробную временную шкалу компьютера, где показана каждая реакция на событие за исторический период до шести месяцев.

![Страница Microsoft Defender ATP с подробной информацией о оповещении](media/security-center-wdatp/image3.png)

## <a name="platform-support"></a>Поддержка платформы

Microsoft Defender ATP в Центре безопасности поддерживает обнаружение на Windows Server 2016, 2012 R2 и 2008 R2 SP1, для Azure VMs нужна подписка стандартного уровня, а для вс-мисов, не относясь к Azure, вам нужен стандартный уровень только на уровне рабочего пространства.

> [!NOTE]
> При использовании Центра безопасности Azure для мониторинга серверов автоматически создается арендатор АТС Microsoft Defender и данные Microsoft Defender ATP хранятся в Европе по умолчанию. Если необходимо переместить данные в другое место, вам необходимо связаться с службой поддержки Майкрософт для сброса клиента. Мониторинг конечных точек сервера, используя эту интеграцию, был отключен для клиентов Office 365 GCC.

## <a name="onboarding-servers-to-security-center"></a>Подключение серверов к Центру безопасности 

Чтобы на бортовых серверах в Центр безопасности, нажмите Перейдите в **Центр безопасности Azure, чтобы на бортовых серверах** с сервера Microsoft Defender ATP.

1. В области **посадки** выберите или создайте рабочее пространство для хранения данных. <br>
2. Если отображаются не все рабочие области, это может быть вызвано отсутствием разрешений. Убедитесь, что для вашей рабочей области выбран уровень "Стандартный" Центра безопасности Azure. Для получения дополнительной информации смотрите [обновление до стандартного уровня Центра безопасности для повышения безопасности.](security-center-pricing.md)
    
3. Выберите **серверы добавления** для просмотра инструкций по установке агента мониторинга Майкрософт. 

4. После посадки, вы можете контролировать машины под **compute и приложений.**

   ![Подключение компьютеров](media/security-center-wdatp/onboard-computers.png)

## <a name="enable-microsoft-defender-atp-integration"></a>Включить интеграцию Microsoft Defender ATP

Чтобы просмотреть, включена ли интеграция АТФ Microsoft Defender, выберите настройки **центра** > **безопасности & настройки,** > нажмите на подписку.
Здесь можно увидеть включенные в данный момент интеграции.

  ![Страница настроек обнаружения угроз Центра безопасности Azure с включенной интеграцией Microsoft Defender ATP](media/security-center-wdatp/enable-integrations.png)

- Если вы уже подключили серверы в центре безопасности Azure уровня "Стандартный", вам не требуется предпринимать никаких дополнительных действий. Центр безопасности Azure автоматически будет находиться на борту серверов Microsoft Defender ATP. Посадка может занять до 24 часов.

- Если вы никогда не подключали серверы в центре безопасности Azure уровня "Стандартный", подключите их к нему обычным образом.

- Если вы на борту серверов через Microsoft Defender ATP:
  - Обратитесь к руководству по [отключению серверных компьютеров](https://go.microsoft.com/fwlink/p/?linkid=852906).
  - Подключите эти серверы к центру безопасности Azure.

## <a name="set-the-firewall-configuration"></a>Настройка конфигурации брандмауэра

Если у вас есть прокси или брандмауэр, который блокирует анонимный трафик, так как датчик Microsoft Defender ATP подключается к контексту системы, убедитесь, что анонимный трафик разрешен. Следуйте инструкциям в [Включить доступ к Microsoft Defender ATP URL-адресов службы на прокси-сервере](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/configure-proxy-internet#enable-access-to-microsoft-defender-atp-service-urls-in-the-proxy-server).

## <a name="test-the-feature"></a>Тестирование компонента

Для создания доброкачественного тестового оповещения Microsoft Defender ATP:

1. Создайте папку 'C:'test-MDATP-тест'.

1. Используйте удаленный рабочий стол для доступа к виртуальной машине Windows Server 2012 R2 или Windows Server 2016. Откройте окно командной строки.

1. В командной строке скопируйте и выполните следующую команду: Окно командной строки закроется автоматически.

    ```
    powershell.exe -NoExit -ExecutionPolicy Bypass -WindowStyle Hidden (New-Object System.Net.WebClient).DownloadFile('http://127.0.0.1/1.exe', 'C:\\test-MDATP-test\\invoice.exe'); Start-Process 'C:\\test-MDATP-test\\invoice.exe'
    ```

   ![Окно командной строки с командой выше](media/security-center-wdatp/image4.jpeg)

3. Если команда успешна, вы увидите новое оповещение на панели мониторинга Azure Security Center и на портале АТФ Microsoft Defender. Это оповещение может отобразиться через несколько минут.

4. Чтобы просмотреть оповещение в Центре безопасности, перейдите на **оповещения** > безопасности**Подозрительные PowerShell CommandLine.**

5. Из окна расследования выберите ссылку для перейдна на портал Microsoft Defender ATP.

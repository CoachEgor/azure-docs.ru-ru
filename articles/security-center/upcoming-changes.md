На этой странице содержатся сведения об изменениях, запланированных для Центра безопасности. Здесь описываются запланированные изменения в продукте, которые могут повлиять на такие элементы, как оценка безопасности или рабочие процессы.

## <a name="planned-changes"></a>Запланированные изменения

### <a name="system-updates-should-be-installed-on-your-machines-recommendation-getting-sub-recommendations"></a>Рекомендация "На компьютерах должны быть установлены обновления системы". Получение вложенных рекомендаций

#### <a name="summary"></a>Сводка

| Аспект | Сведения |
|---------|---------|
|Дата объявления | 9 ноября 2020 г.  |
|Дата для этого изменения  |  Середина-конец ноября 2020 г. |
|Влияние     | Во время замены текущей версии этой рекомендации оценка безопасности может измениться. |
|  |  |

Мы выпускаем улучшенную версию рекомендации **На компьютерах должны быть установлены обновления системы**. Новая версия *заменит* текущую в элементе управления безопасностью для применения обновлений системы и предоставит следующие улучшения:

- Вложенные рекомендации для каждого отсутствующего обновления.
- Переработанный интерфейс на страницах Центра безопасности Azure портала Azure.
- Обогащенные данные для рекомендации из Azure Resource Graph.

#### <a name="transition-period"></a>Переходной период

Переходной период будет длиться 36 часов (приблизительно). Чтобы минимизировать возможное нарушение работы, мы запланировали обновление в выходные дни. Переход может повлиять на ваши оценки безопасности.

#### <a name="redesigned-portal-experience"></a>Переработанный интерфейс портала

Страница сведений о рекомендации **На компьютерах должны быть установлены обновления системы** содержит список результатов, как показано ниже. Если выбрать один результат, откроется область сведений со ссылкой на информацию об исправлении и списком затрагиваемых ресурсов.

#### <a name="richer-data-from-azure-resource-graph"></a>Более подробные данные из Azure Resource Graph

Azure Resource Graph — это служба в Azure, которая предоставляет эффективную оценку ресурсов. Вы можете использовать ARG для выполнения масштабных запросов к заданному набору подписок, чтобы вы могли эффективно управлять своей средой. 

При работе с Центром безопасности Azure вы можете применять ARG с [языком запросов Kusto (KQL)](https://docs.microsoft.com/azure/data-explorer/kusto/query/), чтобы получить расширенный набор данных о состоянии безопасности.

Если вы запрашиваете текущую версию рекомендации **На компьютерах должны быть установлены обновления системы**, в ARG будут доступны лишь сведения о том, что рекомендацию необходимо исправить на компьютере. После выпуска обновленной версии следующий запрос возвратит все отсутствующие обновления системы, сгруппированные по компьютерам.

```kusto
securityresources
| where type =~ "microsoft.security/assessments/subassessments"
| where extract(@"(?i)providers/Microsoft.Security/assessments/([^/]*)", 1, id) == "4ab6e3c5-74dd-8b35-9ab9-f61b30875b27"
| where properties.status.code == "Unhealthy"
```

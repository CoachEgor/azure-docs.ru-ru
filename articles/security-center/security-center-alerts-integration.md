# <a name="security-center-integration-with-azure-security-products-in-asc"></a>Интеграция центра безопасности с продуктами безопасности Azure в ASC

Центр безопасности предлагает клиентам дополнительные лицензии Майкрософт, чтобы подключить свои результаты к центру безопасности и просмотреть их в консолидированной форме.

* [Azure WAF](#azure-waf)
* [Azure от атак DDoS](#azure-ddos)

## Azure WAF<a name="azure-waf"></a>

Шлюз приложений Azure предоставляет брандмауэр веб-приложения (WAF), обеспечивающий централизованную защиту веб-приложений от распространенных эксплойтов и уязвимостей.

Веб-приложения все чаще предназначены для атак злоумышленников, которые часто используют известные уязвимости. WAF шлюза приложений основан на наборе правил Core (CR) 3,0 или 2.2.9 из открытого проекта безопасности веб-приложений (OWASP). WAF обновляется автоматически для защиты от новых уязвимостей без необходимости дополнительной настройки. Предупреждения, созданные WAF, передаются в центр безопасности. Дополнительные сведения о предупреждениях, создаваемых WAF, см. в этой [статье](https://docs.microsoft.com/azure/application-gateway/application-gateway-crs-rulegroups-rules?tabs=owasp3#crs911).

## Azure от атак DDoS<a name="azure-ddos"></a>

Известно, что атаки типа "отказ в обслуживании" (от атак DDoS) являются простыми в выполнении. Поэтому они стали отличным вопросом безопасности для клиентов, которые перемещают свои приложения в облако. 

Целью DDoS-атаки является исчерпание ресурсов приложения, чтобы приложение стало недоступным для обычных пользователей. Атаки от атак DDoS могут ориентироваться на любые конечные точки, к которым можно получить доступ через Интернет.

Защита от атак DDoS Azure в сочетании с рекомендациями по проектированию приложений обеспечивает защиту от атак от атак DDoS. Azure от атак DDoS Protection предоставляет различные уровни служб. Дополнительные сведения см. в статье [Общие сведения о защите Azure от атак DDoS](https://docs.microsoft.com/azure/virtual-network/ddos-protection-overview).

С помощью службы "Защита от атак DDoS" уровня "Стандартный" можно устранить следующие типы атак.

> [!div class="mx-tableFixed"]

|Предупреждение|Описание|
|---|---|
|**Обнаружена атака объемные**|Цель этой атаки — перегрузить сетевой уровень с очень значительным объемом подлинного трафика. Такие атаки включают UDP-флуд, заполнение с усилением и другие типы заполнения с использованием поддельных пакетов. Служба "Защита от атак DDoS" уровня "Стандартный" устраняет влияние таких атак, объем которых может достигать многих гигабайтов, автоматически поглощая и нейтрализуя их на уровне глобальной сети Azure.|
|**Обнаружена атака по протоколу**|Эти атаки отображают целевой объект недоступным путем использования слабости в стеке протоколов уровня 3 и 4. Такие атаки включают SYN-флуд, атаки отражения и другие атаки протокола. Служба "Защита от атак DDoS" уровня "Стандартный" позволяет устранить эти атаки, разделяя допустимый и вредоносный трафик путем взаимодействия с клиентом и блокировки вредоносного трафика.|
|**Обнаружена атака на уровне ресурсов (приложений)**|Эти атаки нацелены на пакеты веб-приложений, чтобы нарушить передачу данных между узлами. Такие атаки включают в себя нарушение протокола HTTP, атаки путем внедрения кода SQL, межсайтовые сценарии и другие атаки уровня 7. Используйте брандмауэр веб-приложения шлюза приложений Azure со стандартом защиты от атак DDoS, чтобы защититься от этих атак. Существуют также предложения брандмауэра веб-приложений сторонних производителей, доступные в Azure Marketplace.|

# <a name="security-center-integration-with-azure-security-products-in-asc"></a>Интеграция центра безопасности с продуктами безопасности Azure в ASC

Центр безопасности предлагает пользователям дополнительные лицензии Майкрософт, чтобы подключить их результаты центра безопасности и просматривать их в виде консолидированные.

* [Azure WAF](#azure-waf)
* [От атак DDoS Azure](#azure-ddos)

## Azure WAF <a name="azure-waf"></a>

Шлюз приложений Azure предоставляет брандмауэр веб-приложения (WAF), обеспечивающий централизованную защиту веб-приложений от распространенных эксплойтов и уязвимостей.

Веб-приложения все чаще распространяется вредоносных атак, использующих известные уязвимости. WAF шлюза приложений основана на Core правила задайте (CRS) версии 3.0 или 2.2.9 из Open Web Application Security Project (OWASP). Брандмауэр веб-приложения обновляются автоматически для защиты от новых уязвимостей, которых не требуется дополнительная настройка. Предупреждения, созданные WAF будут отправлены в центр безопасности. Дополнительные сведения об оповещениях, созданных WAF см. в этом [статье](https://docs.microsoft.com/azure/application-gateway/application-gateway-crs-rulegroups-rules?tabs=owasp3#crs911).

## От атак DDoS Azure <a name="azure-ddos"></a>

Для упрощения выполнения известны распределенной отказ в обслуживании (DDoS). Таким образом они становятся проблемой безопасностью для клиентов, перемещающие свои приложения в облако. 

Целью DDoS-атаки является исчерпание ресурсов приложения, чтобы приложение стало недоступным для обычных пользователей. Атаки DDoS можно привязать к любой конечной точки, которая может быть доступна через Интернет.

Защита от атак DDoS Azure, вместе с рекомендациями по разработки приложений, дает защиту от атак DDoS. Защита от атак DDoS Azure предоставляет разных уровнях служб. Дополнительные сведения см. в разделе [Обзор защиты от атак DDoS Azure](https://docs.microsoft.com/azure/virtual-network/ddos-protection-overview).

С помощью службы "Защита от атак DDoS" уровня "Стандартный" можно устранить следующие типы атак.

> [!div class="mx-tableFixed"]

|Предупреждение|Описание|
|---|---|
|**Объемные обнаружена атака**|Эта атака цель — заполнить сетевой уровень значительным объемом сетевого трафика. Такие атаки включают UDP-флуд, заполнение с усилением и другие типы заполнения с использованием поддельных пакетов. Служба "Защита от атак DDoS" уровня "Стандартный" устраняет влияние таких атак, объем которых может достигать многих гигабайтов, автоматически поглощая и нейтрализуя их на уровне глобальной сети Azure.|
|**Обнаружена атака протокола**|Эти атаки ухудшить доступность целевого объекта, используя уязвимости в слое 3 и 4 протоколов уровня. Такие атаки включают SYN-флуд, атаки отражения и другие атаки протокола. Служба "Защита от атак DDoS" уровня "Стандартный" позволяет устранить эти атаки, разделяя допустимый и вредоносный трафик путем взаимодействия с клиентом и блокировки вредоносного трафика.|
|**Обнаружена атака уровня ресурсов (приложения)**|Эти атаки направлены пакеты веб-приложения и стремятся нарушить передачу данных между узлами. Такие атаки включают в себя нарушение протокола HTTP, атаки путем внедрения кода SQL, межсайтовые сценарии и другие атаки уровня 7. Используйте брандмауэр веб-приложения шлюза приложений Azure с помощью стандартного защиты от атак DDoS для защиты от таких атак. Доступны также брандмауэры сторонних веб-приложения в Azure Marketplace.|

# <a name="threat-detection-for-the-azure-service-layer-in-azure-security-center"></a>Обнаружение угроз для уровня служб Azure в центре безопасности Azure

В этом разделе представлены оповещения центра безопасности Azure, доступные при наблюдении за следующими уровнями служб Azure.

* [Сетевой уровень Azure](#network-layer)
* [Уровень управления Azure (Azure Resource Manager) (Предварительная версия)](#management-layer)

>[!NOTE]
>Следующая аналитика применима ко всем типам ресурсов. Они используют данные телеметрии, предоставляемые центром безопасности, путем касания внутренних веб-каналов Azure.

## Сетевой уровень Azure<a name="network-layer"></a>

Анализ уровня сети центра безопасности основан на примерах [данных ипфикс](https://en.wikipedia.org/wiki/IP_Flow_Information_Export), которые представляют собой заголовки пакетов, собираемые маршрутизаторами Azure Core. На основе этого веб-канала данных модели машинного обучения центра безопасности указывают и помечают действия вредоносного трафика. Для обогащения IP-адресов в центре безопасности используется база данных аналитики угроз (Майкрософт).

> [!div class="mx-tableFixed"]

|Оповещение|Описание|
|---|---|
|**Подозрительные исходящие действия сети RDP**|Анализ сетевого трафика с выборкой обнаружил аномальную исходящую связь протокол удаленного рабочего стола (RDP), которая исходит от ресурса в развертывании. Это действие считается ненормальным для этого окружения. Это может означать, что ваш ресурс был скомпрометирован и теперь используется для перенаправления на внешнюю конечную точку RDP. Обратите внимание, что этот тип действия может привести к тому, что ваш IP-адрес будет помечен как вредоносный внешними сущностями.|
|**Подозрительная исходящая активность сети RDP для нескольких назначений**|Анализ сетевого трафика с выборкой обнаружил аномальное исходящее взаимодействие по протоколу RDP, полученное от ресурса в развертывании на несколько назначений. Это действие считается ненормальным для этого окружения. Это может означать, что ваш ресурс был скомпрометирован и теперь используется для перенаправления внешних конечных точек RDP. Обратите внимание, что этот тип действия может привести к тому, что ваш IP-адрес будет помечен как вредоносный внешними сущностями.|
|**Подозрительная исходящая сетевая активность SSH**|Анализ сетевого трафика с выборкой обнаружил аномальное исходящее Secure Shell (SSH), полученное от ресурса в развертывании. Это действие считается ненормальным для этого окружения. Это может означать, что ваш ресурс был скомпрометирован, и теперь он используется для перенаправления на внешнюю конечную точку SSH. Обратите внимание, что этот тип действия может привести к тому, что ваш IP-адрес будет помечен как вредоносный внешними сущностями.|
|**Подозрительная исходящая сетевая активность SSH для нескольких назначений**|Анализ сетевого трафика с выборкой обнаружил аномальные исходящие подключения SSH, полученные от ресурса в развертывании на несколько назначений. Это действие считается ненормальным для этого окружения. Это может означать, что ваш ресурс был скомпрометирован, и теперь он используется для прямого перенаправления внешних конечных точек SSH. Обратите внимание, что этот тип действия может привести к тому, что ваш IP-адрес будет помечен как вредоносный внешними сущностями.|
|**Подозрительная входящая сетевая активность SSH из нескольких источников**|Анализ сетевого трафика с выборкой обнаружил аномальные входящие подключения SSH из нескольких источников к ресурсу в развертывании. Различные уникальные IP-адреса, подключающиеся к ресурсу, считаются ненормальными для этого окружения. Это действие может указывать на попытку подбора злоумышленником интерфейса SSH из нескольких узлов (ботнетом).|
|**Подозрительная входящая сетевая активность SSH**|Анализ сетевого трафика с выборкой обнаружил аномальный входящий обмен данными SSH с ресурсом в развертывании. Относительно большое количество входящих подключений к ресурсу считается ненормальным для этого окружения. Это действие может указывать на попытку перенаправления принудительной атаки на интерфейс SSH.
|**Подозрительная входящая активность сети RDP из нескольких источников**|Анализ сетевого трафика с выборкой обнаружил аномальные входящие подключения RDP из нескольких источников к ресурсу в развертывании. Различные уникальные IP-адреса, подключающиеся к ресурсу, считаются ненормальными для этого окружения. Это действие может указывать на попытку подбора атаки с помощью интерфейса RDP с нескольких узлов (ботнетом).|
|**Подозрительная входящая активность сети RDP**|Анализ сетевого трафика с выборкой обнаружил аномальные входящие подключения RDP к ресурсу в развертывании. Относительно большое количество входящих подключений к ресурсу считается ненормальным для этого окружения. Это действие может указывать на попытку перенаправления принудительной атаки на интерфейс SSH.|
|**Обнаружено сетевое взаимодействие с вредоносным адресом**|В ходе анализа сетевого трафика обнаружено взаимодействие, полученное от ресурса в развертывании, с помощью возможного сервера команды и элемента управления (C & C). Обратите внимание, что этот тип активности может привести к тому, что ваш IP-адрес будет помечен как вредоносный внешними сущностями.|

Чтобы понять, как центр безопасности может использовать сигналы, связанные с сетью, для применения защиты от угроз, см. раздел [эвристические обнаружения DNS в центре безопасности Azure](https://azure.microsoft.com/blog/heuristic-dns-detections-in-azure-security-center/).

>[!NOTE]
>Оповещения об обнаружении угроз в сетевом уровне Azure в центре безопасности Azure создаются только на виртуальных машинах, которым назначен один и тот же IP-адрес за весь час, в течение которого произошло подозрительное взаимодействие. Это относится к виртуальным машинам, а также к виртуальным машинам, созданным в подписке клиента как часть управляемой службы (например, AKS, кирпичы данных).

## Уровень управления Azure (Azure Resource Manager) (Предварительная версия)<a name ="management-layer"></a>

>[!NOTE]
>Уровень защиты центра безопасности на основе Azure Resource Manager в настоящее время находится на этапе предварительной версии.

Центр безопасности предлагает дополнительный уровень защиты с помощью Azure Resource Manager событий, которые считаются плоскостью управления для Azure. Анализируя записи Azure Resource Manager, центр безопасности обнаруживает необычные или потенциально опасные операции в среде подписки Azure.

> [!div class="mx-tableFixed"]

|Оповещение|Описание|
|---|---|
|**Запуск набора средств микроускорения**|В вашей среде обнаружен известный запуск набора средств разведывательная атака Cloud окружения. Микронагрузка [инструмента может](https://github.com/NetSPI/MicroBurst) использоваться злоумышленником (или тестировщиком уязвимости) для соответствия ресурсов подписок, обнаружения небезопасных конфигураций и утечки конфиденциальной информации.|
|**Запуск набора средств азурите**|В вашей среде обнаружен известный запуск набора средств разведывательная атака Cloud окружения. Средство [азурите](https://github.com/mwrlabs/Azurite) может использоваться злоумышленником (или тестировщиком уязвимости) для отображения ресурсов подписок и обнаружения небезопасных конфигураций.|
|**Подозрительный сеанс управления, использующий неактивную учетную запись**|При анализе журналов действий подписки обнаружено подозрительное поведение. Участник, который не используется в течение длительного периода времени, теперь выполняет действия, которые могут защитить сохраняемость злоумышленника.|
|**Подозрительный сеанс управления с помощью PowerShell**|При анализе журналов действий подписки обнаружено подозрительное поведение. Участник, который регулярно использует PowerShell для управления средой подписки, теперь использует PowerShell и выполняет действия, которые могут защитить сохраняемость злоумышленника.|
|**Использование расширенных методов сохраняемости в Azure**|При анализе журналов действий подписки обнаружено подозрительное поведение. Настроенным ролям были предоставлены легитимизед сущности удостоверений. Это может привести к тому, что злоумышленник сможет получить сохраняемость в среде клиента Azure.|
|**Действие из редко упоминаемой страны**|Действие из расположения, которое не было недавно или когда-либо посещенных каким-либо пользователем в Организации.<br/>Это обнаружение изучает расположения, где выполнялись предыдущие действия, чтобы определить новые редко упоминаемые расположения. Механизм обнаружения аномалий хранит информацию о предыдущих расположениях, используемых пользователями в организации. 
|**Действия, выполняемые с анонимных IP-адресов**|Обнаружена активность пользователей с IP-адреса, идентифицированного как IP-адрес анонимного прокси-сервера. <br/>Эти учетные записи-посредники используются людьми, которые хотят скрыть их, и могут использоваться для злонамеренных целей. Это обнаружение использует алгоритм машинного обучения, который сокращает ложные срабатывания, например IP-адреса с неправильными тегами, широко используемые пользователями организации.|
|**Обнаружено невозможное путешествие**|Произошло два действия пользователя (в одном или нескольких сеансах), полученные из географически удаленных расположений. Это происходит в течение периода времени, меньшего, чем время, которое пользователь перейдет из первого расположения во вторую. Это означает, что другой пользователь использует одни и те же учетные данные. <br/>Это обнаружение использует алгоритм машинного обучения, который игнорирует очевидные ложные срабатывания в условиях невозможности поездки, таких как VPN и расположения, которые регулярно используются другими пользователями в Организации. Для обнаружения предусмотрен начальный период обучения в семь дней, в течение которого он узнает о новом шаблоне действия пользователя.|

>[!NOTE]
> Некоторые из предыдущих аналитиков работают с Microsoft Cloud App Security. Чтобы воспользоваться этими аналитиками, необходимо активировать лицензию на Cloud App Security. Если у вас есть лицензия Cloud App Security, эти оповещения включены по умолчанию. Чтобы отключить их, сделайте следующее:
>
> 1. В колонке **Центр безопасности** выберите **Политика безопасности**. Для подписки, которую необходимо изменить, выберите **изменить параметры**.
> 2. Выберите **обнаружение угроз**.
> 3. В разделе **включить интеграции**снимите флажок **разрешить Microsoft Cloud App Security доступ к моим данным**и нажмите кнопку **сохранить**.

>[!NOTE]
>Центр безопасности хранит данные клиентов, связанные с безопасностью, в том же географическом виде, что и его ресурс. Если в географическом расположении ресурса Корпорация Майкрософт еще не развернула центр безопасности, данные сохраняются в США. Если Cloud App Security включен, эти сведения сохраняются в соответствии с правилами географического расположения Cloud App Security. Дополнительные сведения см. в статье [хранилище данных для служб, не являющихся региональными](https://azuredatacentermap.azurewebsites.net/).

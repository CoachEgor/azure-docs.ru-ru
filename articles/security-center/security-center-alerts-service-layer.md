# <a name="threat-detection-for-azure-service-layer-in-azure-security-center"></a>Обнаружение угроз для уровня служб Azure в центре безопасности Azure

В этом разделе представлены оповещения центра безопасности, доступные при наблюдении за следующими уровнями служб Azure.

* [Сетевой уровень Azure](#network-layer)
* [Уровень управления Azure (Azure Resource Manager) (Предварительная версия)](#management-layer)

>[!NOTE]
>Используя данные телеметрии, которые центр безопасности использует для касания на внутренние каналы Azure, приведенные ниже аналитики применимы ко всем типам ресурсов.

## Сетевой уровень Azure<a name="network-layer"></a>

Анализ уровня сети центра безопасности основан на примерах [данных ипфикс](https://en.wikipedia.org/wiki/IP_Flow_Information_Export), которые представляют собой заголовки пакетов, собираемые маршрутизаторами Azure Core. На основе этого веб-канала данных модели машинного обучения центра безопасности указывают и помечают действия вредоносного трафика. Для обогащения IP-адресов центр безопасности использует базу данных аналитики угроз Майкрософт.

> [!div class="mx-tableFixed"]

|Оповещение|Описание|
|---|---|
|**Подозрительные исходящие действия сети RDP**|Анализ сетевого трафика с выборкой обнаружил аномальный исходящий трафик протокол удаленного рабочего стола (RDP), поступающий из ресурса в развертывании. Это действие считается ненормальным для этой среды и может означать, что ваш ресурс был скомпрометирован и теперь используется для подбора внешней конечной точки RDP. Обратите внимание, что этот тип активности может привести к тому, что ваш IP-адрес будет помечен как вредоносный внешними сущностями.|
|**Подозрительная исходящая активность сети RDP для нескольких назначений**|Анализ сетевого трафика с выборкой обнаружил аномальный исходящий трафик протокол удаленного рабочего стола (RDP), поступающий из ресурса в развертывании в несколько назначений. Это действие считается ненормальным для этой среды и может означать, что ваш ресурс был скомпрометирован и теперь используется для подбора внешних конечных точек RDP. Обратите внимание, что этот тип активности может привести к тому, что ваш IP-адрес будет помечен как вредоносный внешними сущностями.|
|**Подозрительная исходящая сетевая активность SSH**|Анализ сетевого трафика с выборкой обнаружил аномальное исходящее Secure Shell (SSH), исходящий из ресурса в развертывании. Это действие считается ненормальным для этой среды и может означать, что ваш ресурс был скомпрометирован и теперь используется для подбора внешней конечной точки SSH. Обратите внимание, что этот тип активности может привести к тому, что ваш IP-адрес будет помечен как вредоносный внешними сущностями.|
|**Подозрительная исходящая сетевая активность SSH для нескольких назначений**|Анализ сетевого трафика с выборкой обнаружил аномальное исходящее Secure Shell (SSH), исходящий из ресурса в развертывании, в несколько назначений. Это действие считается ненормальным для этой среды и может означать, что ваш ресурс был скомпрометирован и теперь используется для подбора внешних конечных точек SSH. Обратите внимание, что этот тип активности может привести к тому, что ваш IP-адрес будет помечен как вредоносный внешними сущностями.|
|**Подозрительная входящая сетевая активность SSH из нескольких источников**|Анализ сетевого трафика с выборкой обнаружил аномальные входящие подключения SSH из нескольких источников к ресурсу в развертывании. Различные уникальные IP-адреса, подключающиеся к ресурсу, считаются ненормальными для этого окружения. Это действие может указывать на попытку подбора интерфейса SSH из нескольких узлов (ботнетом).|
|**Подозрительная входящая сетевая активность SSH**|Анализ сетевого трафика с выборкой обнаружил аномальный входящий обмен данными SSH с ресурсом в развертывании. Относительно большое число входящих подключений к ресурсу считается ненормальным для этого окружения. Это действие может указывать на попытку подбора интерфейса SSH.
|**Подозрительная входящая активность сети RDP из нескольких источников**|Анализ сетевого трафика с выборкой обнаружил аномальные входящие подключения RDP из нескольких источников к ресурсу в развертывании. Различные уникальные IP-адреса, подключающиеся к ресурсу, считаются ненормальными для этого окружения. Это действие может указывать на попытку подбора интерфейса RDP с нескольких узлов (ботнетом).|
|**Подозрительная входящая активность сети RDP**|Анализ сетевого трафика с выборкой обнаружил аномальные входящие подключения RDP к ресурсу в развертывании. Относительно большое число входящих подключений к ресурсу считается ненормальным для этого окружения. Это действие может указывать на попытку подбора интерфейса SSH.|
|**Обнаружено сетевое взаимодействие с вредоносным адресом**|В ходе анализа сетевого трафика обнаружено взаимодействие, полученное от ресурса в развертывании, с помощью возможного сервера команды и элемента управления (C & C). Обратите внимание, что этот тип активности может привести к тому, что ваш IP-адрес будет помечен как вредоносный внешними сущностями.|

Чтобы понять, как центр безопасности может использовать сигналы, связанные с сетью, для применения защиты от угроз, см. раздел [эвристические обнаружения DNS в центре безопасности Azure](https://azure.microsoft.com/blog/heuristic-dns-detections-in-azure-security-center/).

>[!NOTE]
>Оповещения об обнаружении угроз в сетевом уровне Azure в центре безопасности Azure не создаются в течение первого часа после создания виртуальной машины.

## Уровень управления Azure (Azure Resource Manager) (Предварительная версия)<a name ="management-layer"></a>

>[!NOTE]
>Уровень защиты центра безопасности на основе Azure Resource Manager в настоящее время находится на этапе предварительной версии.

Центр безопасности предлагает дополнительный уровень защиты, используя события Azure Resource Manager, которые считаются плоскостью управления для Azure. Анализируя записи Azure Resource Manager, центр безопасности обнаруживает необычные или потенциально опасные операции в среде подписки Azure.

> [!div class="mx-tableFixed"]

|Оповещение|Описание|
|---|---|
|**Запуск набора средств микроускорения**|В вашей среде обнаружен известный запуск набора средств разведывательная атака Cloud окружения. Средство "микронагрузка" (см https://github.com/NetSPI/MicroBurst) . статью может использовать злоумышленника или тестер уязвимости) для соответствия ресурсов подписок, обнаружения небезопасных конфигураций и утечки конфиденциальной информации.|
|**Запуск набора средств азурите**|В вашей среде обнаружен известный запуск набора средств разведывательная атака Cloud окружения. Средство "азурите" (см https://github.com/mwrlabs/Azurite) . статью может использовать злоумышленника или тестер уязвимости) для соответствия ресурсов подписок и обнаружения небезопасных конфигураций.|
|**Подозрительный сеанс управления, использующий неактивную учетную запись**|Анализ журналов действий подписки обнаружил подозрительное поведение. Участник, который не использовался в течение длительного периода времени, теперь выполняет действия, которые могут защитить сохраняемость злоумышленника.|
|**Подозрительный сеанс управления с помощью PowerShell**|Анализ журналов действий подписки обнаружил подозрительное поведение. Участник, который регулярно не использует PowerShell для управления средой подписки, теперь использует PowerShell и выполняет действия, которые могут защитить сохраняемость злоумышленника.|
|**Использование расширенных методов сохраняемости в Azure**|Анализ журналов действий подписки обнаружил подозрительное поведение. Настроенным ролям были предоставлены легитимизед сущности удостоверений. Это может привести к тому, что злоумышленник сможет получить сохраняемость в среде клиента Azure.|
|**Действие из редко упоминаемой страны**|Действие из расположения, которое не было недавно или никогда не было посещено каким-либо пользователем в Организации.<br/>Это обнаружение изучает расположения, где выполнялись предыдущие действия, чтобы определить новые редко упоминаемые расположения. Механизм обнаружения аномалий хранит информацию о предыдущих расположениях, используемых пользователями в организации. 
|**Действия, выполняемые с анонимных IP-адресов**|Обнаружена активность пользователей с IP-адреса, идентифицированного как IP-адрес анонимного прокси-сервера. <br/>Эти прокси-серверы используют пользователи, желающие скрыть IP-адреса своих устройств. Иногда их используют злоумышленники. Это обнаружение использует алгоритм машинного обучения, который уменьшает количество "ложных срабатываний" (например, ошибочно помеченных IP-адресов, которые часто используются корпоративными пользователями).|
|**Обнаружено невозможное путешествие**|Произошло два действия пользователя (это один или несколько сеансов), исходящие из географически удаленных расположений за период времени, меньший, чем момент, когда пользователь перейдет из первого расположения во вторую. Это означает, что другой пользователь использует одни и те же учетные данные. <br/>Такое обнаружение использует алгоритм машинного обучения, который игнорирует очевидные ложные срабатывания в условиях невозможности перемещения, таких как VPN и расположения, которые регулярно используются другими пользователями в Организации. В обнаружении предусмотрен первоначальный период обучения — 7 дней, в течение которых оно изучает последовательность действий нового пользователя.|

>[!NOTE]
> Некоторые из аналитических данных, приведенные выше, работают с Microsoft Cloud App Security (МКАС). Чтобы воспользоваться этими аналитиками, требуется Активированная лицензия МКАС. Если у вас есть лицензия МКАС, эти оповещения включены по умолчанию. Чтобы отключить их, сделайте следующее:
>
> 1. В колонке "Центр безопасности" выберите **Политика безопасности**. Для подписки, которую нужно изменить, выберите **Изменить параметры**.
> 2. Щелкните **Обнаружение угроз**.
> 3. В разделе **Enable integrations** (Включить интеграцию) снимите флажок **Allow Microsoft Cloud App Security to access my data** (Разрешить Microsoft Cloud App Security доступ к моим данным) и щелкните **Сохранить**.

>[!NOTE]
>Центр безопасности Azure хранит данные клиентов, связанные с безопасностью, в том же географическом виде, что и его ресурс. Если корпорация Майкрософт еще не развернула центр безопасности Azure в географическом ресурсе ресурса, то данные сохраняются в США. Если Microsoft Cloud App Security (МКАС) включен, эти сведения сохраняются в соответствии с правилами географического расположения МКАС. Дополнительные сведения см. в статье [хранилище данных для служб, не являющихся региональными стандартами](https://azuredatacentermap.azurewebsites.net/).

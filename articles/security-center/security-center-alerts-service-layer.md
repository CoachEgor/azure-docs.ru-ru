# <a name="threat-detection-for-azure-service-layer-in-azure-security-center"></a>Обнаружение угроз на уровне службы Azure в центре безопасности Azure

В этом разделе представлены доступные оповещения центра безопасности, при мониторинге со следующими уровнями службы Azure.

* [Уровень сети Azure](#network-layer)
* [Уровень управления Azure (Azure Resource Manager) (Предварительная версия)](#management-layer)

>[!NOTE]
>Используя данные телеметрии, Центр безопасности использует из освоение Azure внутренних каналов, аналитики, приведенные ниже применимы ко всем типам ресурсов.

## Уровень сети Azure<a name="network-layer"></a>

Центр безопасности сетевого уровня analytics основаны на образце [IPFIX данных](https://en.wikipedia.org/wiki/IP_Flow_Information_Export), которые являются заголовки пакетов, собранные Azure core маршрутизаторы. Исходя из этого веб-канала данных, Центр безопасности и моделей машинного обучения определить флаг действия вредоносного трафика. Чтобы расширить IP-адресов, Центр безопасности использует базу данных аналитики угроз корпорации Майкрософт.

> [!div class="mx-tableFixed"]

|Предупреждение|Описание|
|---|---|
|**Подозрительные исходящих сетевая активность через RDP**|Анализ выборки сетевого трафика выявил аномальную исходящую связь протокола удаленного рабочего стола (RDP), исходящий из ресурсов в развертывании. Это действие считается ненормальным для этой среды и может означать, что ресурс была скомпрометирована и теперь используется для перебора внешнюю конечную точку RDP. Обратите внимание, что этот тип активности может привести к тому, что ваш IP-адрес будет помечен как вредоносный внешними сущностями.|
|**Подозрительная Исходящая сетевая активность через RDP к нескольким местам назначения**|Анализ выборки сетевого трафика выявил аномальную исходящую связь протокола удаленного рабочего стола (RDP), исходящий из ресурсов в развертывании в несколько назначений. Это действие считается ненормальным для этой среды и может означать, что ресурс была скомпрометирована и теперь используется для атаки методом подбора внешних конечных точек протокола удаленного рабочего СТОЛА. Обратите внимание, что этот тип активности может привести к тому, что ваш IP-адрес будет помечен как вредоносный внешними сущностями.|
|**Подозрительная исходящих сетевая активность SSH**|Анализ выборки сетевого трафика выявил аномальную исходящую связь Secure Shell (SSH), исходящий из ресурсов в развертывании. Это действие считается ненормальным для этой среды и может означать, что ресурс была скомпрометирована и теперь используется для перебора внешнюю конечную точку SSH. Обратите внимание, что этот тип активности может привести к тому, что ваш IP-адрес будет помечен как вредоносный внешними сущностями.|
|**Подозрительная Исходящая сетевая активность SSH к нескольким местам назначения**|Анализ выборки сетевого трафика выявил аномальную исходящую связь Secure Shell (SSH), исходящий из ресурсов в развертывании в несколько назначений. Это действие считается ненормальным для этой среды и может означать, что ресурс была скомпрометирована и теперь используется для атаки методом подбора внешних конечных точек SSH. Обратите внимание, что этот тип активности может привести к тому, что ваш IP-адрес будет помечен как вредоносный внешними сущностями.|
|**Подозрительная входящая SSH сетевая активность из нескольких источников**|Анализ выборки сетевого трафика выявил аномальную входящие SSH данные из нескольких источников к ресурсу в развертывании. Различные уникальные IP-адреса, подключение к ресурсу, считается ненормальным для этой среды. Это действие может указывать на попытку подбора интерфейса SSH из нескольких узлов (Ботнет).|
|**Подозрительная входящая сетевая активность SSH**|Анализ выборки сетевого трафика выявил аномальную входящую связь по протоколу SSH к ресурсу в развертывании. Относительно большое число входящих подключений к ресурсу, считается ненормальным для этой среды. Это действие может указывать на попытку подбора интерфейса SSH.
|**Подозрительное входящее сетевая активность через RDP из нескольких источников**|Анализ выборки сетевого трафика выявил аномальных входящих связь по RDP из нескольких источников к ресурсу в развертывании. Различные уникальные IP-адреса, подключение к ресурсу, считается ненормальным для этой среды. Это действие может указывать на попытку подбора интерфейса RDP из нескольких узлов (Ботнет).|
|**Подозрительная входящая сетевая активность по протоколу RDP**|Анализ выборки сетевого трафика выявил аномальную входящую связь по протоколу RDP к ресурсу в развертывании. Относительно большое число входящих подключений к ресурсу, считается ненормальным для этой среды. Это действие может указывать на попытку подбора интерфейса SSH.|

Чтобы понять, как центр безопасности может использовать сети связанные сигналов, чтобы применить защиту от угроз, см. в разделе [DNS эвристического обнаружения в центре безопасности Azure](https://azure.microsoft.com/blog/heuristic-dns-detections-in-azure-security-center/).
## Уровень управления Azure (Azure Resource Manager) (Предварительная версия)<a name ="management-layer"></a>

>[!NOTE]
>Центр безопасности уровня защиты, на основе Azure Resource Manager доступна в предварительной версии.

Центр безопасности обеспечивает дополнительный уровень защиты за счет использования событий Azure Resource Manager, который считается плоскости управления для Azure. Анализируя записей Azure Resource Manager, Центр безопасности обнаруживает необычные или потенциально опасные операции в среде подписки Azure.

> [!div class="mx-tableFixed"]

|Предупреждение|Описание|
|---|---|
|**Набор средств microBurst запуска**|Это набор средств разведывательной атаки известных облачной среды выполнения обнаружила в вашей среде. Средство «MicroBurst» (см. в разделе https://github.com/NetSPI/MicroBurst) можно использовать, взломщику (или на проникновение) для сопоставления ресурсов подписок, определите небезопасных конфигурации, утечка конфиденциальной информации.|
|**Набор средств azurite запуска**|Это набор средств разведывательной атаки известных облачной среды выполнения обнаружила в вашей среде. Средство «Azurite» (см. в разделе https://github.com/mwrlabs/Azurite) можно использовать, взломщику (или на проникновение) для сопоставления ресурсов подписки и определения небезопасных конфигураций.|
|**Подозрительные управления сеанса, с помощью неактивной учетной записью**|Анализ журналов активности подписки обнаружила подозрительное поведение. Участник, для которого не используется в течение длительного периода времени, в данный момент выполняет действия, которые можно защитить сохраняемости для злоумышленников.|
|**Подозрительные управления сеанса с помощью PowerShell**|Анализ журналов активности подписки обнаружила подозрительное поведение. Субъект, который не использует регулярно PowerShell для управления подпиской, выполняется с помощью PowerShell и выполнения действий, которые могут защитить сохраняемости для злоумышленников.|
|**Использование методов расширенные Azure сохраняемости**|Анализ журналов активности подписки обнаружила подозрительное поведение. Настраиваемые роли выдается legitimized идентификаторов сущностей. Это может привести, злоумышленник может получить постоянного присутствия в среде Azure клиента.|
|**Действие из редко упоминаемой страны**|Из расположения, которое не недавно или никогда не посещался ни одним пользователем организации действия.<br/>Это обнаружение изучает расположения, где выполнялись предыдущие действия, чтобы определить новые редко упоминаемые расположения. Механизм обнаружения аномалий хранит информацию о предыдущих расположениях, используемых пользователями в организации. 
|**Действия, выполняемые с анонимных IP-адресов**|Действия пользователей с IP-адреса, который был определен как IP-адрес анонимного прокси-сервера обнаружено. <br/>Эти прокси-серверы используют пользователи, желающие скрыть IP-адреса своих устройств. Иногда их используют злоумышленники. Это обнаружение использует алгоритм машинного обучения, который уменьшает количество "ложных срабатываний" (например, ошибочно помеченных IP-адресов, которые часто используются корпоративными пользователями).|
|**Обнаружено неосуществимое перемещение**|Два действия пользователя (это один или несколько сеансов) возникшие, из географически отдаленных расположений в течение периода времени, короче, чем время его потребовалось бы пользователю на перемещение из одного расположения на второй. Это означает, что другой пользователь использует те же учетные данные. <br/>Функция использует алгоритм машинного обучения игнорирует очевидные «ложные срабатывания» к условиям Невозможное перемещение, таких как VPN и расположения, которые постоянно используют другие пользователи в организации. В обнаружении предусмотрен первоначальный период обучения — 7 дней, в течение которых оно изучает последовательность действий нового пользователя.|

>[!NOTE]
> Некоторые из выше аналитики работают на по безопасности приложения Microsoft Cloud (MCAS). Чтобы использовать преимущества этих analytics, лицензии на активированных MCAS является обязательным. Если у вас есть лицензия MCAS, эти оповещения включены по умолчанию. Чтобы отключить их:
>
> 1. В колонке "Центр безопасности" выберите **Политика безопасности**. Для подписки, которую нужно изменить, выберите **Изменить параметры**.
> 2. Щелкните **Обнаружение угроз**.
> 3. В разделе **Enable integrations** (Включить интеграцию) снимите флажок **Allow Microsoft Cloud App Security to access my data** (Разрешить Microsoft Cloud App Security доступ к моим данным) и щелкните **Сохранить**.

>[!NOTE]
>Центр безопасности Azure хранятся данные клиента, связанных с безопасностью в том же геообъекте, что его ресурсов. Если Майкрософт еще не развернут центр безопасности Azure в географически ресурса, затем она сохраняет данные в Соединенных Штатах. При включении безопасности приложения Microsoft Cloud (MCAS), эти сведения хранятся в соответствии с правилами географическое расположение MCAS. См. в разделе [хранение данных для служб без привязки к региону Подробнее](http://azuredatacentermap.azurewebsites.net/).

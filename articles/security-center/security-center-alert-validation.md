# <a name="alert-validation-eicar-test-file-in-azure-security-center"></a>Проверка оповещений (в тестовом файле EICAR) в центре безопасности Azure
Этот документ содержит информацию о том, как убедиться, что ваша система правильно настроена для оповещений центра безопасности Azure.

## <a name="what-are-security-alerts"></a>Что такое оповещения системы безопасности?
Оповещения — это уведомления, создаваемые центром безопасности при обнаружении угрозы для ресурсов. Он устанавливает приоритеты и список предупреждений, а также сведения, необходимые для быстрого анализа проблемы. Центр безопасности также приведены рекомендации по как можно устранить атаки.
Дополнительные сведения см. в разделе [оповещения системы безопасности в центре безопасности Azure](security-center-alerts-overview.md) и [реагирование на оповещения безопасности в центре безопасности Azure и управление ими](security-center-managing-and-responding-alerts.md)

## Проверка оповещений на виртуальной Машине Windows <a name="validate-windows"></a>

После центра безопасности, которые агент установлен на компьютере выполните следующие действия с компьютера, где необходимо предоставить права атакованный ресурс оповещения:

1. Скопируйте исполняемый файл (например **calc.exe**) на рабочем столе компьютера, или в другом каталоге вашего удобства и переименуйте его в качестве **ASC_AlertTest_662jfi039N.exe**.
1. Откройте командную строку и выполните этот файл с помощью аргумента (просто фиктивное имя аргумента), такие как: ```ASC_AlertTest_662jfi039N.exe -foo```
1. Подождите 5–10 минут и откройте оповещения центра безопасности. Оповещение, аналогичную [пример](#alert-validate) должен быть показанный ниже:

> [!NOTE]
> При просмотре этого оповещения теста для Windows, убедитесь, что поле **включен аудит аргументов** — **true**. Если это **false**, то необходимо включить аудит аргументов командной строки. Чтобы включить ее, используйте следующую команду:
>
>```reg add "HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\policies\system\Audit" /f /v "ProcessCreationIncludeCmdLine_Enabled"```

## Проверка оповещений на виртуальной Машине Linux <a name="validate-linux"></a>

После центра безопасности, которые агент установлен на компьютере выполните следующие действия с компьютера, где необходимо предоставить права атакованный ресурс оповещения:
1. Скопируйте исполняемый файл в удобном месте и переименуйте его в **. / asc_alerttest_662jfi039n**, например:

    ```cp /bin/echo ./asc_alerttest_662jfi039n```

1. Откройте командную строку и выполните этот файл.

    ```./asc_alerttest_662jfi039n testing eicar pipe```

1. Подождите 5–10 минут и откройте оповещения центра безопасности. Оповещение, аналогичную [пример](#alert-validate) должен быть показанный ниже:

# <a name="introduction-to-azure-defender-for-storage"></a>Общие сведения о защитнике Azure для хранилища

**Защитник Azure для хранилища** обнаруживает потенциально опасные действия в учетных записях хранения Azure. Данные могут быть защищены независимо от того, хранятся ли они в виде контейнеров больших двоичных объектов, файловых ресурсов или озера данных.

Этот уровень защиты позволяет устранять угрозы *без* необходимости быть экспертом по безопасности и помогает управлять системами мониторинга безопасности.

## <a name="what-kind-of-alerts-does-azure-defender-for-storage-provide"></a>Какие оповещения предоставляет защитник Azure для хранилища?

Оповещения системы безопасности активируются, если таковые имеются:

- **Подозрительные действия** . Например, доступ к учетной записи хранения был успешно получен с IP-адреса, известного как активный узел выхода Tor
- **Аномальное поведение** — например, изменения в шаблоне доступа к учетной записи хранения
- **Потенциальная загруженная вредоносная программа** — анализ репутации хэша указывает, что отправленный файл содержит вредоносные программы

Оповещения содержат сведения об инциденте, который их инициировал, а также рекомендации по исследованию и исправлению угроз.

> [!TIP]
> Вы можете имитировать оповещения хранилища, следуя инструкциям в [этой записи блога](https://techcommunity.microsoft.com/t5/azure-security-center/validating-atp-for-azure-storage-detections-in-azure-security/ba-p/1068131).


## <a name="what-is-hash-reputation-analysis-for-malware"></a>Что такое анализ репутации хэша для вредоносных программ?

Чтобы определить, является ли отправленный файл подозрительным, в защитнике Azure для хранилища используется анализ репутации хэша, поддерживаемый службой анализа [угроз Майкрософт](https://go.microsoft.com/fwlink/?linkid=2128684). Средства защиты от угроз не проверяют загруженные файлы, а проверяют журналы хранилища и сравнивают хэши недавно отправленных файлов с известными вирусами, троянами, шпионскими программами и программой-шантажистом. 

Когда файл может содержать вредоносные программы, центр безопасности отображает оповещение и при необходимости может отправить владельцу хранилища сообщение электронной почты на утверждение для удаления подозрительного файла. Чтобы настроить автоматическое удаление файлов, которые анализ репутации хэша указывает на наличие вредоносных программ, разверните [автоматизацию рабочего процесса, чтобы активировать оповещения, содержащие "потенциальные вредоносные программы, отправленные в учетную запись хранения"](https://techcommunity.microsoft.com/t5/azure-security-center/how-to-respond-to-potential-malware-uploaded-to-azure-storage/ba-p/1452005).

> [!NOTE]
> Чтобы включить возможности защиты от угроз для центра безопасности, необходимо включить защитник Azure в подписке, содержащей применимые рабочие нагрузки.
>
> Вы можете включить **защитник Azure для хранилища** на уровне подписки или на уровне ресурса.

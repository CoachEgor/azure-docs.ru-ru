# <a name="cloud-smart-alert-correlation-in-azure-security-center-incidents"></a>Корреляция облачных интеллектуальных оповещений в центре безопасности Azure (инциденты)

Центр безопасности Azure постоянно анализирует гибридные облачные рабочие нагрузки с помощью расширенной аналитики и анализа угроз для оповещения о вредоносных действиях.

Растет покрытие угроз. Потребность в обнаружении даже самого незначительного компромисса очень важна, и аналитикам безопасности может быть сложно рассмотреть различные оповещения и определить фактическую атаку. Центр безопасности помогает аналитикам справляться с этим оповещением выносливости. Она помогает диагностировать атаки по мере их возникновения, сопоставляя различные предупреждения и сигналы низкого качества в инцидентах безопасности.

Аналитика Fusion — это технология и серверная резервная копия, которая обеспечивает безопасность инцидентов центра безопасности, позволяя им одновременно сопоставлять различные предупреждения и контекстные сигналы. Fusion рассматривает различные сигналы, переданные по подписке для всех ресурсов. Fusion находит закономерности, которые позволяют выявить ход атаки или сигналы с общими контекстными сведениями, указывая, что для них следует использовать единую процедуру ответа.

Аналитика Fusion сочетает в себе знания о домене безопасности с помощью искусственного интеллекта для анализа оповещений, обнаружения новых шаблонов атак по мере их возникновения. 

Центр безопасности использует матрицу MITREных атак, чтобы связать предупреждения с воспринимаемой целью, помогая формализации знаний предметной области безопасности. Кроме того, с помощью сведений, собранных для каждого этапа атаки, центр безопасности может исключить действие, которое кажется пошаговым, но на самом деле это не так.

Поскольку атаки часто возникают в разных клиентах, центр безопасности может сочетать алгоритмы искусственного интеллекта для анализа последовательностей атак, о которых сообщается в каждой подписке. Этот метод определяет последовательности атак как распространенные шаблоны оповещений, а не просто связаны друг с другом по инциденту.

Во время расследования инцидента аналитикам часто требуется дополнительный контекст для достижения вердикт о характере угрозы и способах ее устранения. Например, даже при обнаружении аномалии сети, не зная о том, что еще происходит в сети или в отношении целевого ресурса, трудно понять, какие действия следует предпринять. Чтобы помочь, инцидент безопасности может включать артефакты, связанные события и сведения. Дополнительные сведения, доступные для инцидентов безопасности, зависят от типа обнаруженной угрозы и конфигурации среды.

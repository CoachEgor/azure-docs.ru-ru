# <a name="cloud-smart-alert-correlation-in-azure-security-center-incidents"></a>Корреляция облачных интеллектуальных оповещений в центре безопасности Azure (инциденты)

Центр безопасности непрерывно анализирует гибридные облачные рабочие нагрузки с помощью расширенной аналитики и анализа угроз для оповещения о вредоносных действиях.

По мере роста покрытия угроз и необходимости обнаружения даже самой незначительной информации о скомпрометированных возможностях, аналитикам безопасности может быть сложно рассмотреть различные предупреждения и определить фактическую атаку. Центр безопасности помогает аналитикам справляться с выносливостиами предупреждений и диагностикой атак по мере их возникновения, сопоставляя различные предупреждения и сигналы низкого качества в инцидентах безопасности.

Fusion — это технология и серверная резервная копия, которая обеспечивает безопасность инцидентов центра безопасности, позволяя им одновременно сопоставлять различные предупреждения и контекстные сигналы. Fusion работает путем просмотра различных сигналов, переданных по подписке на ресурсы, и поиска самых распространенных шаблонов, демонстрирующих ход атаки или сигналы с общими контекстными сведениями, которые указывают на то, что процедура унифицированного ответа должна быть для них.

Аналитика Fusion объединяет знания о домене безопасности с AI для анализа оповещений, обнаружения новых шаблонов атак по мере их возникновения. 

Центр безопасности использует матрицу MITREных атак, чтобы связать предупреждения с воспринимаемой целью, помогая формализации знаний предметной области безопасности. Кроме того, с помощью сведений, собранных для каждого этапа атаки, центр безопасности может полагаться на действия по истечению действия, которые появляются в качестве этапов атаки, но это не так.  

Так как атаки часто возникают в разных клиентах, центр безопасности может сочетать алгоритмы искусственного интеллекта для анализа последовательностей атак, которые выводятся в каждой подписке, чтобы их определяли как распространенные шаблоны оповещений, а не просто связаны с каждым из них. иной.

Во время расследования инцидента аналитикам часто требуется дополнительный контекст для достижения вердикт о характере угрозы и способах ее устранения. Например, даже при обнаружении аномалии сети, не зная о том, что еще происходит в сети или в отношении целевого ресурса, трудно понять, какие действия следует предпринять далее. Чтобы помочь, инцидент безопасности может включать артефакты, связанные события и сведения. Дополнительные сведения, доступные для инцидентов безопасности, зависят от типа обнаруженной угрозы и конфигурации среды.

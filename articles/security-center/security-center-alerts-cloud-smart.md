# <a name="cloud-smart-alert-correlation-in-azure-security-center-incidents"></a>Корреляции предупреждений смарт-облако в центре безопасности Azure (инцидентов)

Центр безопасности постоянно анализирует гибридных облачных рабочих нагрузок с помощью расширенной аналитики и аналитики угроз, чтобы оповещать вас о вредоносных действий.

По мере роста широту охвата угроз и необходимость обнаруживать даже малейших указание компрометации увеличивается, может оказаться сложной задачей для аналитиков по вопросам безопасности рассмотреть различные оповещения и определения фактической атакой. Центр безопасности помогает справиться с предупреждения усталости и диагностики атак, при их возникновении, аналитиков путем корреляции данных различные предупреждения и сигналов низкого качества в инциденты безопасности.

Fusion, — это технология и аналитической серверной части, степени инцидентов центра безопасности, включение его для корреляции с различными предупреждениями и контекстные сигналы вместе. Fusion работает по различным сигналам отчета подписки по ресурсам и поиск наиболее распространенных шаблонов, которые, показывает течение атаки или сигналы с общей контекстную информацию, указывает на процедуру единой ответа должно быть необходимое для них.

Fusion analytics объединить знания о предметной области безопасности с помощью искусственного Интеллекта, чтобы анализировать предупреждения, обнаружение новых шаблонов атаки, происходящие. 

Центр безопасности использует MITRE атаки матрицу, чтобы привязка предупреждений к заявили о намерении предполагаемых, помогая формализации знаний о предметной области безопасности. Кроме того используя сведения, собранные для каждого шага атаки, Центр безопасности можно исключить действие, которое выглядит действия атаки, но не является.  

Так как они осуществляются часто в разных клиентах, Центр безопасности можно объединить алгоритмов искусственного Интеллекта для анализа последовательностей атаки, выданные для каждой подписки идентифицировать их как наиболее распространенных предупреждений шаблонов вместо просто он между прочим связанных с каждым другие.

При расследовании происшествия аналитикам часто требуется дополнительный контекст, чтобы выяснить характер угрозы и способ ее предотвращения. Например даже в том случае, если обнаружена аномалия сети, не понимая, что еще происходит в сети или о целевом ресурсе трудно понять, какие действия нужно предпринять. Чтобы помочь, нарушения безопасности может включать артефакты, связанные события и сведения. Дополнительные сведения, доступные для инцидентов безопасности зависит от типа Обнаружена угроза и конфигурации вашей среды.

# <a name="cloud-smart-alert-correlation-in-azure-security-center-incidents"></a>Связь облачного смарт-оповещения в Центре безопасности Azure (инциденты)

Центр безопасности Azure непрерывно анализирует гибридные облачные рабочие нагрузки, используя расширенную аналитику и интеллект угроз, чтобы предупредить вас о вредоносных действиях.

Масштабы охвата угрозами растут. Необходимость обнаружения даже малейшего компромисса имеет важное значение, и аналитики по безопасности могут быть сложными для сортировки различных предупреждений и выявления фактической атаки. Центр безопасности помогает аналитикам справиться с этой усталостью оповещения. Это помогает диагностировать атаки по мере их возникновения, сопоставляя различные оповещения и сигналы низкой точности в инциденты безопасности.

Аналитика Fusion — это технология и аналитический задний конец, который позволяет инцидентов Центра безопасности, позволяя ему соотносить различные оповещения и контекстуальные сигналы вместе. Fusion рассматривает различные сигналы, сообщаемые в подписке на ресурсах. Fusion находит закономерности, которые показывают прогрессию атаки или сигналы с общей контекстуальной информацией, указывая, что вы должны использовать единую процедуру ответа для них.

Fusion Analytics сочетает в себе знания в области безопасности с ИИ для анализа оповещений, открывая новые шаблоны атак по мере их возникновения. 

Центр безопасности использует МАТРИЦу атаки MITRE, чтобы связать оповещения с их предполагаемыми намерениями, помогая формализовать знания домена безопасности. Кроме того, используя информацию, собранную для каждого шага атаки, Центр безопасности может исключить действия, которые, как представляется, являются шагами атаки, но на самом деле это не так.

Поскольку атаки часто происходят между разными арендаторами, Центр безопасности может комбинировать алгоритмы ИИ для анализа последовательностей атак, которые сообщаются по каждой подписке. Этот метод определяет последовательности атак как распространенные модели оповещения, вместо того, чтобы быть случайно связанными друг с другом.

Во время расследования инцидента аналитики часто нуждаются в дополнительном контексте для вынесения вердикта о характере угрозы и о том, как ее смягчить. Например, даже при обнаружении сетевой аномалии, не понимая, что еще происходит в сети или в отношении целевого ресурса, трудно понять, какие действия следует предпринять дальше. Для помощи инцидент безопасности может включать артефакты, связанные с ними события и информацию. Дополнительная информация, доступная для инцидентов безопасности, варьируется в зависимости от типа обнаруженной угрозы и конфигурации среды.

# <a name="retirement-of-security-center-features-july-2019"></a>Прекращении работы функции центра безопасности (июля 2019 г.)

Мы внесли несколько [улучшения](https://azure.microsoft.com/updates/?product=security-center) к центру безопасности Azure за последние 6 месяцев.  
С усовершенствованные возможности мы удаляем ряд избыточных компонентов, а также связанные интерфейсы API центра безопасности на 31 июля 2019 г.  

Большинство функций, выведенного из эксплуатации можно заменить на новые функции в центре безопасности Azure или Log Analytics; и некоторые функции могут быть реализованы с помощью [Azure Sentinel (Предварительная версия)](https://azure.microsoft.com/services/azure-sentinel/), недавно представленные.

Список функций, выводится из эксплуатации центра обеспечения безопасности включает в себя:

- [Информационная панель событий](#menu_events)
- [Запись меню поиска](#menu_search)
- [Представление классической удостоверения и доступ ссылку Идентификация и доступ (Предварительная версия)](#menu_classicidentity)
- [События безопасности сопоставить кнопки на карте оповещения безопасности (Предварительная версия)](#menu_securityeventsmap)
- [Настраиваемые правила генерации оповещений (Предварительная версия)](#menu_customalerts)
- [Исследовать кнопки в оповещениях системы безопасности для защиты от угроз](#menu_investigate)
- [Подмножество решений по обеспечению безопасности](#menu_solutions)
- [Изменение конфигураций безопасности для политик безопасности](#menu_securityconfigurations)
- [Безопасность и аудит панель мониторинга (изначально используется на портале OMS) для рабочих областей Log Analytics.](#menu_securityomsdashboard)

Ниже представлены подробные сведения для каждой функции выведенного из эксплуатации, а также действия, которые можно выполнить с помощью функций замены.

## Информационная панель событий<a name="menu_events"></a>
Центр безопасности использует агент Microsoft Monitoring Agent для сбора различных конфигураций, связанных с безопасностью, и событий из виртуальных машин, а затем сохраняет эти события в рабочей области. [Информационная панель событий](https://docs.microsoft.com/azure/security-center/security-center-events-dashboard) позволяет просмотреть эти данные и по существу предоставляет другой точкой входа в службу Log Analytics.

Забегая вперед, информационная панель событий прекращает свою работу:

![Экран выбора рабочей области события][1]

Информационная панель событий, которая появится, когда пользователь щелкает на рабочей области, будут также отключены:

![Информационная панель событий][2]

### <a name="events-dashboard---new-experience"></a>Панель мониторинга событий — новый интерфейс

Клиенты могут использовать встроенные возможности Log Analytics для просмотра важные события на их рабочих областях.
Если вы уже создали настраиваемых важных событий в центре обеспечения безопасности, это будут доступны через Log analytics -> выберите рабочую область "->" сохраненные условия поиска. Не данных будет потерян или изменены. Собственный важные события, также доступны на одном экране.

![Сохраненные поисковые запросы рабочей области][3]

По существу этот экран служит в качестве оболочки для Log Analytics "[поиска](https://docs.microsoft.com/azure/security-center/security-center-search)" страницы — что разрешать пользователям выполнять поисковые запросы на их выбранной рабочей области. Забегая вперед, это окно поиска прекращает свою работу:

## Схема событий безопасности<a name="menu_securityeventsmap"></a>
Центр безопасности предоставляет вам [карты](https://docs.microsoft.com/azure/security-center/security-center-threat-intel) , помогает выявлять угрозы безопасности среды. Кнопка «Перейти к схема событий безопасности» на этой карте приводит к панели мониторинга, которая позволяет просматривать события необработанной безопасности на выбранной рабочей области.
Кнопки, а также панели мониторинга в рабочей области, будут удалены после об устаревании.

![Карта — оповещение системы безопасности кнопку][10]

Сегодня при щелчке «Перейти в схема событий безопасности» открывается панель анализа угроз. Информационная панель анализа угроз будет прекращено.  

![Панель мониторинга для анализа угроз][11]

При выборе рабочую область, чтобы просмотреть информационную панель анализа угроз безопасности оповещает экрана map(Preview) *в Log Analytics* открыт. Этот экран будет прекращено.

![Схема безопасности оповещения в Log Analytics][12]

Существующие данные останутся доступными в решение log analytics безопасности и будет не быть изменены или удалены.
Исходный запросы Log Analytics, используемый для заполнения панели мониторинга аналитики угроз доступны в центре безопасности [репозиторий GitHub](https://github.com/Azure/Azure-Security-Center/tree/master/Legacy%20Log%20Analytics%20dashboards).

### <a name="security-events-map---new-experience"></a>Схема события безопасности — новый интерфейс
Мы рекомендуем клиентам использовать карты оповещения, встроенное в центр безопасности — «оповещения системы безопасности карта (Предварительная версия)». Это обеспечивает оптимизирована работы и работает во всех подписок и связанных рабочих областей, позволяя макрос просмотра в вашей среде и не сосредоточиться на одной рабочей области.

## Настраиваемые правила генерации оповещений (Предварительная версия)<a name="menu_customalerts"></a>
В работе настраиваемые оповещения [прекращено](https://docs.microsoft.com/azure/security-center/security-center-custom-alert) 30 июня 2019 г., из-за прекращения использования базовой инфраструктуры построен на. В период до устаревания пользователи будут иметь возможность редактировать существующие настраиваемые правила генерации оповещений, но не сможете добавить новые. Пользователям рекомендуется включить [Azure Sentinel](https://azure.microsoft.com/services/azure-sentinel/) с подключением одним щелчком, чтобы автоматически перенести свои существующие оповещения и создавать новые или также повторно создать свои оповещения с помощью Azure Monitor оповещения журнала.

Чтобы сохранить существующие оповещения и перенести их в Azure Sentinel, запустите Azure Sentinel. В качестве первого шага выберите рабочую область, где хранятся ваши пользовательские оповещения и выберите пункт меню «Аналитика», чтобы автоматически перенести ваши оповещения.

![Настраиваемые оповещения][13]

Клиенты, не заинтересованные в подключении к Azure Sentinel, рекомендуется для повторного создания оповещений с помощью оповещений Azure Monitor log. Инструкции см. в разделе: [создание, просмотр и управление с помощью Azure Monitor оповещения журнала](https://docs.microsoft.com/azure/azure-monitor/platform/alerts-log). Инструкции по созданию оповещений журнала см.: [Оповещения журнала в Azure Monitor](https://docs.microsoft.com/azure/azure-monitor/platform/alerts-unified-log).

Дополнительные сведения о пользовательских оповещений вывода из эксплуатации см. в разделе [оповещения центра безопасности пользовательских документации](https://docs.microsoft.com/azure/security-center/security-center-custom-alert).

## Исследование оповещений безопасности<a name="menu_investigate"></a>
[Функция изучения](https://docs.microsoft.com/azure/security-center/security-center-investigation) в центре безопасности позволяет рассматривать, понять область действия и отслеживать первопричину потенциальные угрозы безопасности. Этот компонент будет удален из центра безопасности, так как он был заменен более удобной работу в [Azure Sentinel](https://azure.microsoft.com/services/azure-sentinel/).

![Инцидент][14]

Если щелкнуть кнопку «Проверить», «исследование открывается панель мониторинга (Предварительная версия)» в Log Analytics. Панели мониторинга изучения прекращает свою работу.  
Существующие данные останутся доступными в Log Analytics решения для системы безопасности и будет не быть изменены или удалены.

![Исследовательская панель мониторинга в Log Analytics][15]

### <a name="investigation---new-experience"></a>Исследование — новый интерфейс

Мы рекомендуем нашим клиентам встроенного [Azure Sentinel](https://azure.microsoft.com/services/azure-sentinel/) для исследования широкие возможности взаимодействия на платформе охота функция оповещения. Azure Sentinel предоставляет мощный поиск работы средства поиска и запросов для слежения наличие угроз безопасности в источниках данных вашей организации.  

## Подмножество решений по обеспечению безопасности<a name="menu_solutions"></a>

Центр безопасности предусматривает возможность включения [интегрированных решений безопасности в Azure](https://docs.microsoft.com/azure/security-center/security-center-partner-integration). Указанные ниже решения партнеров бы быть удалены, а также [Azure Sentinel](https://azure.microsoft.com/services/azure-sentinel/), а также еще большему количеству источников данных.

- Следующее поколение брандмауэра и решений веб-приложений брандмауэра (Azure Sentinel [документации](https://docs.microsoft.com/azure/sentinel/connect-data-sources))
- Интеграция решений по обеспечению безопасности, которые поддерживают общий формат событий (CEF) (Azure Sentinel [документации](https://docs.microsoft.com/azure/sentinel/connect-common-event-format))
- Microsoft Advanced Threat Analytics (Azure Sentinel [документации](https://docs.microsoft.com/azure/sentinel/connect-azure-atp))
- Защита идентификации Azure AD (Azure Sentinel [документации](https://docs.microsoft.com/azure/sentinel/connect-azure-ad-identity-protection))

Следующие об устаревании пользователи не смогут добавить новые или изменять существующие подключенные решения типов, упомянутых выше, из пользовательского интерфейса и API.
Существующие подключенные решения рекомендуется переместить Azure Sentinel к концу этого периода.

![Безопасность центры решений][16]

## Изменение конфигураций безопасности для политик безопасности<a name="menu_securityconfigurations"></a>
Центр безопасности Azure отслеживает конфигурации безопасности на основе [более 150 рекомендуемых правил](https://gallery.technet.microsoft.com/Azure-Security-Center-a789e335) усиления защиты операционной системы, в том числе правил, связанных с брандмауэрами, аудитом, политиками паролей и многими другими компонентами. Если конфигурация компьютера уязвима к атакам, центр безопасности формирует рекомендации по безопасности. [Экран конфигурации безопасности редактирования](https://docs.microsoft.com/azure/security-center/security-center-customize-os-security-config) позволяет пользователям настроить конфигурацию безопасности операционной системы по умолчанию в центре безопасности.

Эта функция в предварительной версии, которая будет удалена.

![Изменение настроек системы безопасности][17]

### <a name="edit-security-configurations---new-experience"></a>Изменение конфигураций безопасности - новые возможности

Центр безопасности будет поддерживать [конфигурации гостевого агента](https://docs.microsoft.com/azure/governance/policy/concepts/guest-configuration) в ближайшем будущем, позволяя богаче набор функций - поддержку для дополнительных операционных систем и интеграция с гостевой конфигурации с помощью политики Azure (включая политики в гостевой системе). Кроме того, это позволит обеспечить возможность управления в масштабе и автоматически применять новые ресурсы.

## Безопасность и аудит панель мониторинга (изначально используется на портале OMS) для рабочих областей Log Analytics<a name="menu_securityomsdashboard"></a>

В рабочей области панели мониторинга "Безопасность" в Log analytics обзор событий безопасности и угроз, карта анализа угроз и удостоверения и доступ оценки событий безопасности, сохраненные в рабочей области. Панели мониторинга будут удалены в будущем. Как мы уже рекомендовано в панели мониторинга пользовательского интерфейса, наших пользователей рекомендуется использовать Центр безопасности Azure в дальнейшем.

![Панель мониторинга безопасности log analytics][18]

### <a name="security--audit-dashboard---new-experience"></a>Безопасность и аудит панель мониторинга — новый интерфейс
Клиентам рекомендуется использовать Центр безопасности Azure, который предоставляет же общие сведения о безопасности в нескольких подписках и рабочих областей, связанных с ними, а также широким набором функций.

Исходный запросов Log Analytics, используемых для заполнения безопасности и аудита панели мониторинга доступны в центре безопасности [репозиторий GitHub](https://github.com/Azure/Azure-Security-Center/tree/master/Legacy%20Log%20Analytics%20dashboards).

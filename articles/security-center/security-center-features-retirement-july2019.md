# <a name="retirement-of-security-center-features-july-2019"></a>Прекращении работы функции центра безопасности (июля 2019 г.)

Мы внесли несколько [улучшения](https://azure.microsoft.com/updates/?product=security-center) к центру безопасности Azure за последние шесть месяцев.
Эти улучшенные возможности мы удаляете некоторые компоненты, обеспечивающие избыточность и соответствующие интерфейсы API в центре обеспечения безопасности на 31 июля 2019 г.  

Большинство этих эксплуатации функций могут быть заменены на новые функциональные возможности центра безопасности Azure или Azure Log Analytics. Другие функции может быть реализован с помощью [Azure Sentinel (Предварительная версия)](https://azure.microsoft.com/services/azure-sentinel/).

Возможности центра безопасности будет выведена из эксплуатации:

- [Информационная панель событий](#menu_events)
- [Запись меню поиска](#menu_search)
- [Представление классической удостоверения и доступ ссылку Идентификация и доступ (Предварительная версия)](#menu_classicidentity)
- [События безопасности сопоставить кнопки на карте оповещения безопасности (Предварительная версия)](#menu_securityeventsmap)
- [Настраиваемые правила генерации оповещений (Предварительная версия)](#menu_customalerts)
- [Исследовать кнопки в оповещениях системы безопасности для защиты от угроз](#menu_investigate)
- [Подмножество решений по обеспечению безопасности](#menu_solutions)
- [Изменение конфигураций безопасности для политик безопасности](#menu_securityconfigurations)
- [Безопасность и аудит панель мониторинга (изначально используется на портале OMS) для рабочих областей Log Analytics](#menu_securityomsdashboard)

В этой статье содержится информация для каждой функции выведенного из эксплуатации, а также действия, которые можно предпринять для реализации возможностей замены.

## Информационная панель событий<a name="menu_events"></a>

Центр безопасности использует Microsoft Monitoring Agent для сбора различных конфигураций, связанных с безопасностью и событиями с машин. Она сохраняет эти события в рабочих областях. [Информационная панель событий](https://docs.microsoft.com/azure/security-center/security-center-events-dashboard) позволяет просматривать эти данные и предоставляет точку входа к Log Analytics.

Мы все снятия с учета, информационная панель событий, который отображается при выборе рабочей области:

![Информационная панель событий][2]

### <a name="events-dashboard---the-new-experience"></a>Информационная панель событий - новые возможности

Мы рекомендуем использовать встроенные возможности Azure Log Analytics, чтобы просмотреть важные события на рабочие области.

Если вы создали пользовательские важные события в центре безопасности, они будут доступны. В Log Analytics, перейдите в раздел **выберите рабочую область** > **сохраненные условия поиска**. Данные не будут потеряны или изменены. Собственный важные события, также доступны на одном экране в Log Analytics.

![Сохраненные поисковые запросы рабочей области][3]

## Запись меню поиска<a name="menu_search"></a>

Поиск в журналах Azure Monitor в настоящее время используется в центре безопасности Azure получает и анализирует данные безопасности. Этот экран служит в качестве окна на страницу поиска Log Analytics и позволяет пользователям выполнять поисковые запросы на их выбранной рабочей области. Дополнительные сведения см. в разделе [поиск в центре безопасности Azure](https://docs.microsoft.com/azure/security-center/security-center-search). Мы все снятия с учета, это окно поиска:

![Страница "Поиск"][4]

### <a name="search-menu-entry---the-new-experience"></a>Запись меню поиска: новые возможности

Мы рекомендуем использовать встроенные возможности Azure Log Analytics для выполнения запросов поиска с рабочими областями. Перейдите к Azure Log Analytics и выберите **журналы**.

![Log Analytics записывает страницы][5]

## Классический удостоверения и доступ (Предварительная версия)<a name="menu_classicidentity"></a>

Интерфейс классического удостоверения и доступ в центр безопасности сейчас отображается панель мониторинга идентификации и доступа к информации в Log Analytics. Для просмотра этой информационной панели:

1. Выберите **просмотреть классический интерфейс удостоверений и доступа**.

   ![Страница удостоверений][6]

1. Представление **удостоверения и доступ мониторинга**.

    ![Страница "identity" — Выбор рабочей области][7]

1. Выберите рабочую область, чтобы открыть **удостоверения и доступ** панели мониторинга в Log Analytics для просмотра удостоверений и доступа к данным в рабочей области.

   ![Страница "identity" — панель мониторинга][8]

Мы все снятия с учета все три экрана, показанный на предыдущем шаге. Ваши данные остаются доступными в Log Analytics решения для системы безопасности и не удалить или изменить.

### <a name="classic-identity--access-preview---the-new-experience"></a>Классический интерфейс удостоверений и доступа (Предварительная версия) — новый интерфейс

Панель мониторинга Log Analytics показало insights на одной рабочей области. Однако встроенные возможности центра безопасности обеспечивают видимость все подписки и всех рабочих областей, связанных с ними. Доступ к простой — использовать представление, которое позволяет сосредоточиться на самом важном рекомендации, которые ранжируются в соответствии с их Оценка безопасности.

Все функции **удостоверения и доступ** панели мониторинга в Log Analytics можно получить, выбрав **удостоверения и доступ (Предварительная версия)** в центре безопасности.

![Страница "удостоверение" — Классический возможности вывода из эксплуатации][9]

## Схема событий безопасности<a name="menu_securityeventsmap"></a>

Центр безопасности предоставляет вам [схема оповещения безопасности](https://docs.microsoft.com/azure/security-center/security-center-threat-intel) для выявления угроз безопасности. **Перейти к схеме событий безопасности** кнопка на этой карте открывает панель мониторинга, которая позволяет просматривать события необработанной безопасности в выбранной рабочей области.

Мы удаляете **перейти к схеме событий безопасности** кнопки и панели мониторинга в рабочей области.

![Карта — оповещение системы безопасности кнопку][10]

При выборе **перейти к схеме событий безопасности** кнопку, при открытии панели мониторинга аналитики угроз. Мы все снятия с учета, информационная панель анализа угроз.  

![Панель мониторинга аналитики угроз][11]

Если выбрать рабочую область, чтобы просмотреть информационную панель анализа угроз, открытии экрана карта (Предварительная версия) оповещений безопасности в Log Analytics. Мы все снятия с учета, этот экран.

![Схема безопасности оповещения в Log Analytics][12]

Существующие данные остаются доступными в Log Analytics решения для системы безопасности и не будет изменен ни удалить.

### <a name="security-events-map---the-new-experience"></a>Схема события безопасности — новый интерфейс

Мы рекомендуем использовать карты оповещения, встроенное в центре безопасности: **Схема безопасности оповещения (Предварительная версия)** . Эта функция предоставляет оптимизированные возможности и поддерживается всеми подписками и связанные рабочие области. Он предоставляет высокоуровневое представление в вашей среде и не уделяется одной рабочей области.

## Настраиваемые правила генерации оповещений (Предварительная версия)<a name="menu_customalerts"></a>

Мы [снятия с учета пользовательский интерфейсе оповещений](https://docs.microsoft.com/azure/security-center/security-center-custom-alert) на 30 июня 2019 г. Поскольку снятия с учета его базовой инфраструктурой. До этого можно изменить существующие настраиваемые правила генерации оповещений, но вы не сможете добавить новые. Мы рекомендуем включить [Azure Sentinel](https://azure.microsoft.com/services/azure-sentinel/) автоматически перенести ваши оповещения и создавать новые. Кроме того можно создать оповещения с помощью оповещений Azure Monitor log.

Чтобы сохранить существующие оповещения и перенести их в Azure Sentinel:

1. Откройте Azure Sentinel и выберите рабочую область, где хранятся ваши пользовательские оповещения.
1. Выберите **Analytics** меню, чтобы автоматически перенести ваши оповещения.

![Настраиваемые оповещения][13]

Если вы не заинтересованы в процессе перехода к Azure Sentinel, мы рекомендуем вам для создания оповещения с помощью оповещений Azure Monitor log. Инструкции см. в разделе [создание, просмотр и управление оповещений журнала с помощью Azure Monitor](https://docs.microsoft.com/azure/azure-monitor/platform/alerts-log) и [оповещениях журналов в Azure Monitor](https://docs.microsoft.com/azure/azure-monitor/platform/alerts-unified-log).

Дополнительные сведения о пользовательских оповещений вывода из эксплуатации см. в разделе [настраиваемые правила генерации оповещений в центре безопасности Azure (Предварительная версия)](https://docs.microsoft.com/azure/security-center/security-center-custom-alert).

## Исследование оповещений безопасности<a name="menu_investigate"></a>

[Функция изучения](https://docs.microsoft.com/azure/security-center/security-center-investigation) в центре безопасности рассмотрение потенциальные угрозы безопасности. Эта функция позволяет понять область действия инцидента и отслеживать их первопричину. Мы ходе мы устраняем этой функции в центре обеспечения безопасности, так как он будет заменен улучшенные функции в [Azure Sentinel](https://azure.microsoft.com/services/azure-sentinel/).

![Инцидент][14]

При выборе **исследовать** кнопки **инцидента безопасности** экрана, откройте панель мониторинга исследования (Предварительная версия) в Log Analytics. Мы прекращение использования панели мониторинга изучения.  

Существующие данные остаются доступными в Log Analytics решения для системы безопасности и не будет изменен ни удалить.

![Исследовательская панель мониторинга в Log Analytics][15]

### <a name="investigation---the-new-experience"></a>Исследование — новый интерфейс

Мы рекомендуем перейти к [Azure Sentinel](https://azure.microsoft.com/services/azure-sentinel/) для многофункциональных исследования. Azure Sentinel предоставляет мощные средства поиска и запросов для слежения наличие угроз безопасности в источниках данных вашей организации.  

## Подмножество решений по обеспечению безопасности<a name="menu_solutions"></a>

Центр безопасности можно включить [интегрированных решений безопасности в Azure](https://docs.microsoft.com/azure/security-center/security-center-partner-integration). Мы все снятия с учета указанные ниже решения партнеров в центре обеспечения безопасности. Эти решения, включенные в [Azure Sentinel](https://azure.microsoft.com/services/azure-sentinel/) реализован ряд дополнительных источников данных.

- [Решения нового поколения брандмауэра и веб-приложение брандмауэра](https://docs.microsoft.com/azure/sentinel/connect-data-sources)
- [Интеграция решений по обеспечению безопасности, которые поддерживают общий формат событий (CEF)](https://docs.microsoft.com/azure/sentinel/connect-common-event-format)
- [Microsoft Advanced Threat Analytics](https://docs.microsoft.com/azure/sentinel/connect-azure-atp)
- [Защита идентификации Azure AD](https://docs.microsoft.com/azure/sentinel/connect-azure-ad-identity-protection)

После снятия с учета нельзя будет добавлять или изменять любой из типов решения, упомянутые в предыдущем списке, либо из пользовательского интерфейса или API.

Если у вас есть существующие подключенные решения, мы рекомендуем вам переместиться Azure Sentinel.

![Безопасность центры решений][16]

## Изменение конфигураций безопасности для политик безопасности<a name="menu_securityconfigurations"></a>

Центр безопасности Azure отслеживает конфигурации безопасности, применяя набор [более 150 рекомендуемых правил](https://gallery.technet.microsoft.com/Azure-Security-Center-a789e335). для усиления защиты операционной системы. Эти правила относятся к брандмауэры, аудит, политики паролей и многое другое. Если конфигурация компьютера уязвима к атакам, центр безопасности формирует рекомендации по безопасности. [Экран конфигурации безопасности редактирования](https://docs.microsoft.com/azure/security-center/security-center-customize-os-security-config) позволяет пользователям настроить конфигурацию безопасности операционной системы по умолчанию в центре безопасности.

Мы все снятия с учета, эта функция предварительного просмотра. Если после даты прекращения использования, вы хотите сбросить настройки безопасности обратно к значениям по умолчанию, это можно сделать с помощью API или Powershell, используя [следуйте инструкциям](https://aka.ms/ascresetsecurityconfigurations)

![Изменение конфигураций безопасности][17]

### <a name="edit-security-configurations---the-new-experience"></a>Изменение конфигураций безопасности - новые возможности

Мы планируем включить Центр безопасности для поддержки [конфигурации гостевого агента](https://docs.microsoft.com/azure/governance/policy/concepts/guest-configuration). Такое обновление позволит богаче набор функций, включая поддержку для других операционных систем и интеграции Azure в гостевой системе политик для конфигурации гостевого. После включения этих изменений, имеется также возможность управлять конфигурациями в масштабе и автоматически применять их для новых ресурсов.

## Панель мониторинга безопасности и аудита для рабочих областей Log Analytics<a name="menu_securityomsdashboard"></a>

Панели мониторинга безопасности и аудита изначально использовался на портале OMS. В Log Analytics панели мониторинга содержит обзор рабочими областями события безопасности и угроз, карта анализа угроз и оценку удостоверения и доступ событий безопасности, сохраненные в рабочей области. Мы удаляется с панели мониторинга. Как мы уже рекомендовано в панели мониторинга пользовательского интерфейса, мы советуем вам перейти к центру безопасности Azure.

![Безопасность панель мониторинга log Analytics][18]

### <a name="security-and-audit-dashboard---the-new-experience"></a>Безопасность и аудит панель мониторинга — новый интерфейс

Мы рекомендуем переключиться к центру безопасности Azure. Он содержит же общие сведения о безопасности для нескольких подписок и рабочих областей, связанных с ними, а также задать больше возможностей.

Вы можете получить исходные запросы Log Analytics, которые заполняют панель мониторинга безопасности и аудита в [репозиторий GitHub](https://github.com/Azure/Azure-Security-Center/tree/master/Legacy%20Log%20Analytics%20dashboards) для центра безопасности.

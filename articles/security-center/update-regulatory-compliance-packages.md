# <a name="update-to-dynamic-compliance-packages-in-your-regulatory-compliance-dashboard-preview"></a>Обновление пакетов обеспечения совместимости на панели мониторинга соответствия нормативным требованиям (Предварительная версия)

Центр безопасности Azure постоянно сравнивает конфигурацию ресурсов с требованиями отраслевых стандартов, нормативных требований и тестов производительности. **Панель мониторинга соответствия нормативным** требованиям предоставляет аналитические сведения о соответствии вашим требованиям в зависимости от того, как вы отвечаете за соблюдение конкретных требований и контроля соответствия.

Один стандарт, для которого можно отвести свой уровень соответствия требованиям, — это [Azure CIS 1.1.0](https://www.cisecurity.org/benchmark/azure/) (более формально — "CI Microsoft Azure Foundation тест производительности версии 1.1.0"). 

Представление Azure CI, изначально отображаемое на панели мониторинга соответствия требованиям, полагается на статический набор правил, включенных в центр безопасности.

Благодаря функции **пакетов динамического соответствия (Предварительная версия)** центр безопасности автоматически улучшает свое покрытие отраслевыми стандартами с течением времени. Пакеты соответствия по сути являются инициативами, определенными в политике Azure. Их можно назначить выбранной области (подписка, Группа управления и т. д.). Чтобы просмотреть данные о соответствии, сопоставленные в качестве оценок на панели мониторинга, добавьте пакет соответствия в группу управления или подписку в рамках политики безопасности. Добавление пакета соответствия эффективно назначит инициативу по соответствию нормативным требованиям выбранной области. Таким образом, вы можете отслеживанию недавно опубликованных нормативных требований в виде стандартов на панели мониторинга. Когда корпорация Майкрософт выпускает новое содержимое для инициативы (новые политики, которые сопоставляются с дополнительными элементами управления в стандарте), дополнительное содержимое отображается на панели мониторинга автоматически.

Пакет динамического соответствия для теста производительности Azure CIS, **Azure CIS 1.1.0 (новый)** улучшает исходную *статическую* версию следующим образом.

* Включение дополнительных политик
* Автоматическое обновление с новым покрытием при его добавлении 

Обновите новый динамический пакет, как описано ниже.

## <a name="adding-a-dynamic-compliance-package"></a>Добавление пакета динамического соответствия

Ниже описано, как добавить динамический пакет для мониторинга соответствия требованиям в Azure CI 1.1.0 v.   

### <a name="update-to-the-azure-cis-110-new-dynamic-compliance-package"></a>Обновление пакета динамического соответствия Azure CIS 1.1.0 (новое) 

1. Откройте страницу **Политика безопасности** . На этой странице показано количество групп управления, подписок, рабочих областей и структуры группы управления.

1. Выберите подписку или группу управления, для которой требуется управлять соответствием нормативным требованиям. Рекомендуется выбрать максимальную область, для которой применяется стандартный, чтобы данные о соответствии были агрегированы и отслеживанию для всех вложенных ресурсов. 

1. В разделе "отраслевые & нормативные стандарты (Предварительная версия)" вы увидите, что Azure CIS 1.1.0 можно обновить для получения нового содержимого. Нажмите кнопку **Обновить сейчас**. 

1. При необходимости щелкните **добавить дополнительные стандарты** , чтобы открыть страницу **Добавление стандартов соответствия нормативным требованиям** . Там вы можете выполнить поиск вручную для **Azure CI 1.1.0 (новый)** и динамических пакетов для других стандартов соответствия, таких **как NIST SP 800-53 R4**, **SWIFT CSP КСКФ-V2020**, **УКО и Великобритания отделения**и **Канада пбмм**.
    
    ![Добавление нормативных пакетов на панель мониторинга соответствия нормативным требованиям центра безопасности Azure](./media/update-regulatory-compliance-packages/security-center-dynamic-regulatory-compliance-additional-standards.png)


1. На боковой панели центра безопасности выберите **соответствие нормативным требованиям** , чтобы открыть панель мониторинга соответствия нормативным требованиям. 
    * Azure CIS 1.1.0 (новые) теперь отображается в списке отраслевых & нормативных требований. 
    * Исходное *статическое* представление соответствия требованиям Azure CIS 1.1.0 также останется рядом с ним. Она может быть автоматически удалена в будущем.

    > [!NOTE]
    > Для отображения вновь добавленного стандарта на панели мониторинга соответствия может потребоваться несколько часов.

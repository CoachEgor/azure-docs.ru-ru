# <a name="update-to-dynamic-compliance-packages-in-your-regulatory-compliance-dashboard-preview"></a>Обновление динамических пакетов соответствия в панели мониторинга соответствия нормативным требованиям (Preview)

Центр безопасности Azure постоянно сравнивает конфигурацию ваших ресурсов с требованиями в отраслевых стандартах, правилах и тестах. Панель **мониторинга соответствия нормативным требованиям** позволяет получить представление о вашей позиции соответствия в зависимости от того, как вы выполняете конкретные элементы контроля и требования соответствия.

Одним из стандартов, по которому вы можете отслеживать свою позицию соответствия, является [Azure CIS 1.1.0](https://www.cisecurity.org/benchmark/azure/) (более формально, "CIS Microsoft Azure Foundations Benchmark version 1.1.0"). 

Представление Azure CIS, которое первоначально отображается в панели мониторинга соответствия, основано на статичном наборе правил, включенных в Центр безопасности.

Благодаря **функции динамических пакетов соответствия (предварительного просмотра)** Центр безопасности автоматически улучшает охват отраслевых стандартов с течением времени. Пакеты соответствия по существу являются инициативами, определенными в политике Azure. Они могут быть назначены выбранной вами области (подписка, группа управления и так далее). Чтобы увидеть данные о соответствии, отображаемые в качестве оценок в панели мониторинга, добавьте пакет соответствия группе управления или подписку в рамках Политики безопасности. Добавление пакета соответствия эффективно присваивает инициативу по соблюдению нормативных требований выбранной сфере. Таким образом, вы можете отслеживать недавно опубликованные нормативные инициативы в качестве стандартов соответствия в панели мониторинга. Когда корпорация Майкрософт выпускает новый контент для инициативы (новые политики, которые отображать больше элементов управления в стандарте), дополнительное содержимое автоматически отображается в панели мониторинга.

Динамический пакет соответствия для бенчмарка Azure CIS, **Azure CIS 1.1.0 (новый)**, улучшает оригинальную *статичную* версию:

* Включение дополнительных политик
* Автоматическое обновление с новым покрытием по мере его добавления 

Обновление нового динамического пакета, описанного ниже.

## <a name="adding-a-dynamic-compliance-package"></a>Добавление динамического пакета соответствия

Следующие шаги объясняют, как добавить динамический пакет для мониторинга соответствия требованиям с бенчмарком Azure CIS v1.1.0.   

### <a name="update-to-the-azure-cis-110-new-dynamic-compliance-package"></a>Обновление пакета динамических соответствий Azure CIS 1.1.0 (новый) 

1. Откройте страницу **политики безопасности.** На этой странице отображается количество групп управления, подписок, рабочих областей и структуры группы управления.

1. Выберите группу по подписке или управлению, для которой вы хотите управлять позицией соответствия нормативным требованиям. Мы рекомендуем выбрать наивысший объем, для которого применяется стандарт, чтобы данные о соответствии агрегировались и отслеживались для всех вложенных ресурсов. 

1. В разделе Отраслевые & нормативные стандарты (предварительный просмотр) можно увидеть, что Azure CIS 1.1.0 может быть обновлен для нового контента. Нажмите **Обновление сейчас**. 

1. Дополнительно нажмите **Кнопка Добавить дополнительные стандарты,** чтобы открыть страницу **стандартов соответствия нормативным требованиям Add.** Там вы можете искать вручную для **Azure CIS 1.1.0 (Новый)** и динамические пакеты для других стандартов соответствия, таких как **NIST SP 800-53 R4**, **SWIFT CSP CSCF-v2020**, **UKO и UK NHS**, и Канада **PBMM**.
    
    > [!TIP]
    > Только пользователи, являющиеся владельцем или участником политики, имеют необходимые разрешения на добавление стандартов соответствия. 

    ![Добавление нормативных пакетов в панель мониторинга соответствия требованиям Центра безопасности Azure](./media/update-regulatory-compliance-packages/security-center-dynamic-regulatory-compliance-additional-standards.png)


1. На боковой панели Центра безопасности выберите **соответствие нормативным** требованиям, чтобы открыть панель мониторинга соответствия нормативным требованиям. 
    * Azure CIS 1.1.0 (New) теперь отображается в списке отраслевых & нормативных стандартов. 
    * Исходное *статический* вид соответствия Azure CIS 1.1.0 также останется рядом с ним. Он может быть автоматически удален в будущем.

    > [!NOTE]
    > Появление нового добавленного стандарта в панели мониторинга соответствия может занять несколько часов.

# <a name="threat-detection-for-cloud-native-compute-in-azure-security-center"></a>Обнаружение угроз для вычислений в машинном облаке в центре безопасности Azure

Как поставщик облачных служб центр безопасности Azure использует уникальную видимость, которую он должен использовать для анализа внутренних журналов и определения методологии атак на нескольких целевых объектах. В этом разделе представлены оповещения, доступные для следующих служб Azure:

* [Служба приложений Azure](#app-services)
* [Служба контейнеров](#azure-containers) 

## Служба приложений Azure<a name="app-services"></a>

Центр безопасности использует масштаб облака для определения атак, направленных на приложения, работающие в службе приложений. Веб-приложения широко используются в современных сетях, и злоумышленники могут обнаружить эти недостатки и воспользоваться ею. Прежде чем перенаправляться в конкретные среды, запросы к приложениям, работающим в Azure, проходят через несколько шлюзов, которые проверяются и регистрируются в журнале. Эти данные затем используются для обнаружения эксплойтов и злоумышленников и для изучения новых шаблонов, которые будут использоваться позже.

Используя видимость, которую Azure использует в качестве поставщика облачных служб, центр безопасности анализирует внутренние журналы службы приложений, чтобы выявление методологии атак на нескольких целевых объектах. Например, методология включает широкое сканирование и распределенные атаки. Атаки такого типа обычно поступают из небольшого подмножества IP-адресов и представляют закономерности обхода в аналогичные конечные точки на нескольких узлах. Атаки осуществляют поиск уязвимой страницы или подключаемого модуля и не могут быть идентифицированы с точки зрения одного узла.

Центр безопасности также имеет доступ к базовым изолированным средам и виртуальным машинам. Вместе с судебными механизмами памяти инфраструктура может сообщить историю, от новой атаки, которая будет прерываться в случае нарушения безопасности клиентских компьютеров. Таким образом, центр безопасности может обнаружить атаки для веб-приложений длиннее, чем использовать их.

|Оповещение|Описание|
|---|---|
|**Обнаружен подозрительный вызов темы WordPress**|Журнал действий службы приложений указывает на возможное действие вставки кода в ресурсе службы приложений.<br/> Это подозрительное действие напоминает действие, которое управляет темой WordPress для поддержки выполнения кода на стороне сервера, за которым следует прямой веб-запрос на вызов файла управляемой темы. Этот тип деятельности может быть частью кампании атаки по сравнению с WordPress.|
|**Обнаружено подключение к веб-странице с аномальным IP-адресом**|Журнал действий службы приложений указывает на подключение к конфиденциальной веб-странице из исходного адреса, который ранее не был подключен к нему. Это может означать, что кто-то пытается выполнить атаку с методом подбора на страницы администрирования. Это также может быть результатом использования законного пользователя с новым IP-адресом.|
|**В аналитике угроз обнаружен IP-адрес, подключенный к интерфейсу FTP службы приложений Azure.**|При анализе журналов FTP службы приложений обнаружено подключение с исходного адреса, найденного в веб-канале аналитики угроз. Во время этого подключения пользователь обратился к указанным страницам.|
|**Обнаружены веб-отпечатки**|Журнал действий службы приложений указывает на возможные действия по отпечатку веб-страниц в ресурсе службы приложений. <br/>Это подозрительное действие связано с инструментом, который называется слепым слоном. Средство отпечатка веб-серверов и пытается обнаружить установленные приложения и их версии. Злоумышленники часто используют это средство для поиска уязвимостей в веб-приложениях.|
|**Обнаружен подозрительный доступ к потенциально уязвимой веб-странице**|Журнал действий службы приложений указывает, что был получен доступ к веб-странице, которая кажется конфиденциальной. <br/>Это подозрительное действие исходит от исходного адреса, шаблон доступа которого напоминает веб-сканер. Такое действие часто связано с попыткой злоумышленника проверить сеть, чтобы попытаться получить доступ к конфиденциальным или уязвимым веб-страницам.|
|**Файл PHP в папке отправки**|Журнал действий службы приложений указывает на то, что у вас есть доступ к подозрительным страницам PHP, расположенным в папке upload. <br/>Этот тип папки обычно не содержит файлы PHP. Существование этого типа файлов может указывать на использование преимуществ произвольных уязвимостей при передаче файлов.|
|**Попытка выполнить команды Linux в службе приложений Windows**|При анализе процессов службы приложений обнаружена попытка выполнить команду Linux в службе приложений Windows. Это действие выполнялось веб-приложением. Такое поведение часто встречается во время кампаний, использующих уязвимость в общем веб-приложении.|
|**Обнаружено подозрительные выполнение PHP**|Журналы компьютера указывают на то, что выполняется подозрительный процесс PHP. Действие включало попытку выполнить команды операционной системы или код PHP из командной строки с помощью процесса PHP. Хотя такое поведение может быть приемлемым, в веб-приложениях такое поведение может указывать на вредоносные действия, такие как попытки заражения веб-узлов оболочкой.|
|**Обработка выполнения из временной папки**|При анализе процессов службы приложений обнаружено выполнение процесса из временной папки приложения. Хотя такое поведение может быть приемлемым, в веб-приложениях такое поведение может указывать на вредоносные действия.|
|**Обнаружена попытка запуска команды с высоким уровнем привилегий**|При анализе процессов службы приложений обнаружена попытка выполнить команду, для которой требуются высокие привилегии. Команда выполнена в контексте веб-приложения. Хотя такое поведение может быть приемлемым, в веб-приложениях такое поведение может указывать на вредоносные действия.|

> [!NOTE]
> Обнаружение угроз в центре безопасности для службы приложений сейчас недоступно в регионах Azure для государственных организаций и независимых.

## Служба контейнеров Azure<a name="azure-containers"></a>

Центр безопасности обеспечивает обнаружение угроз в реальном времени для контейнеров на компьютерах Linux на основе инфраструктуры, для которой проведен аудит. Оповещения указывают несколько подозрительных действий DOCKER, таких как создание привилегированного контейнера на узле, указание сервера Secure Shell (SSH), работающего в контейнере DOCKER, или использование шифрования занимающихся анализом. 

Эти сведения можно использовать для быстрого устранения проблем безопасности и повышения безопасности контейнеров. Помимо обнаружений Linux, центр безопасности также предлагает аналитику, которая более специфична для развертываний контейнеров.

# <a name="threat-detection-for-cloud-native-compute-in-azure-security-center"></a>Обнаружение угроз для облачных собственного вычислительных ресурсов в центре безопасности Azure

Как поставщик облака центр безопасности использует уникальный видимость его для анализа журналов внутренней и идентификации методологии атаки на несколько целевых объектов. В этом разделе представлен оповещениях, доступных для следующих служб Azure:

* [Служба приложений Azure](#app-services)
* [Контейнеры](#azure-containers) 

## Служба приложений Azure <a name="app-services"></a>

Центр безопасности использует масштабирования облака для выявления атак, направленных клиенты, работающие по службе приложений Azure. С веб-приложениями, практически в любой современной сети злоумышленники тестирование с целью найти и использовать его слабые места. Перед направляется в определенных средах, запросы к приложениям, работающим в Azure проходят через несколько шлюзов, где они проверен и в журнал. Эти данные затем используются для определения уязвимостей, злоумышленники и Дополнительные новые шаблоны, которые будут использоваться позже.

За счет использования видимости для Azure как поставщик облака, Центр безопасности анализирует журналы, внутренней службы приложений, чтобы определить методологии атаки на несколько целевых объектов. Например, широко сканирования и распределенных атак. Этот тип атаки обычно поступают из небольшой набор IP-адресов и характеризуется шаблоны обхода аналогично конечные точки на нескольких узлах, поиск уязвимую страницу или подключаемого модуля. Это могут быть обнаружены в облаке, но не может быть определен с точки зрения одного узла.

Центр безопасности также имеет доступ к базовым "песочницы" и виртуальных машин. Вместе с экспертизы памяти инфраструктуры можно представить данные на, от новых атак, бесполезных на практике для компрометации на машинах клиента. Таким образом центр безопасности может выявить атаки, направленные на веб-приложений long после злоумышленниками.

## Контейнеры <a name="azure-containers"></a>

Центр безопасности предоставляет обнаружение угроз в реальном времени для контейнеров на компьютерах Linux, на основе auditd framework. Оповещения содержат несколько подозрительных действий Docker, таких как создание привилегированных контейнер на узле, значение, указывающее Secure Shell (SSH) сервера, работающего в контейнере Docker или использование шифрования пользователей. Эти сведения можно использовать для быстрого устранения проблем безопасности и повышения безопасности контейнеров. Помимо обнаружения Linux центр безопасности также предлагает аналитики, характерных для развертывания контейнеров.

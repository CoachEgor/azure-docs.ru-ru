# <a name="monitor-identity-and-access"></a>Мониторинг идентификации и доступа

> [!TIP]
> В марте 2020 рекомендации по удостоверениям и доступу центра безопасности Azure включены во все подписки в ценовой категории Free. Если у вас есть подписки на уровне "бесплатный", их оценка безопасности будет затронута, так как они не были ранее оценены для обеспечения безопасности идентификации и доступа. 

Когда центр безопасности выявляет потенциальные уязвимости в системе безопасности, он создает рекомендации по настройке необходимых элементов управления, чтобы укрепить защиту ресурсов.

Периметр безопасности прошел развитие от периметра сети до периметра на основе удостоверений. Защита сети в моделях безопасности отходит на второй план, уступая место защите данных, а также управлению безопасностью приложений и пользователей. В настоящее время, когда большинство данных и приложений перемещаются в облако, идентификация становится новым периметром.

Отслеживая действия по идентификации, можно выполнять упреждающие действия перед возникновением инцидента или реактивные действия, чтобы предотвратить попытки атаки. Например, центр безопасности может пометить устаревшие учетные записи (учетные записи, которые больше не нужны) и заблокировать вход с помощью Azure Active Directory) для удаления. 

Ниже приведены примеры рекомендаций, которые можно увидеть в разделе Безопасность ресурсов **удостоверений и доступ** в центре безопасности Azure.

- MFA должна быть включена для учетных записей с разрешениями владельца в вашей подписке.
- Подписке можно назначить не более 3 владельцев.
- Внешние учетные записи с разрешениями на чтение следует удалять из подписки.
- Устаревшие учетные записи следует удалять из подписки.

Дополнительные сведения об этих рекомендациях, а также полный список рекомендаций см. в разделе [рекомендации по удостоверениям и доступу](recommendations-reference.md#recs-identity).

> [!NOTE]
> Если подписка имеет более чем 600 учетных записей, Центр безопасности не сможет определить рекомендации по идентификации для вашей подписки. Рекомендации, которые не выполняются, перечислены ниже в разделе "недоступные оценки".
Центр безопасности не может определять рекомендации по идентификации для агентов администрирования поставщика облачных решений.
>


Все рекомендации по удостоверениям и доступу доступны в двух элементах управления безопасностью на странице **рекомендации** :

- Управление доступом и разрешениями 
- Включение MFA

## <a name="enable-multi-factor-authentication-mfa"></a>Включение многофакторной проверки подлинности (MFA)

Для включения MFA требуются [разрешения клиента Azure Active Directory (AD)](https://docs.microsoft.com/azure/active-directory/users-groups-roles/directory-assign-admin-roles). 

- Если у вас есть выпуск AD уровня "Премиум", включите MFA с помощью [условного доступа](https://docs.microsoft.com/azure/active-directory/conditional-access/overview).

- Пользователи бесплатного выпуска AD могут включить **Параметры безопасности по умолчанию** в Azure Active Directory, как описано в [документации по AD](https://docs.microsoft.com/azure/active-directory/fundamentals/concept-fundamentals-security-defaults) , но рекомендации центра безопасности по включению MFA будут по-прежнему отображаться.

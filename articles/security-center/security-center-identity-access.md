# <a name="monitor-identity-and-access"></a>Мониторинг идентификации и доступа

> [!TIP]
> С марта 2020 года рекомендации Центра безопасности Azure по идентификации и доступу включены во все подписки бесплатной ценовой категории. Если у вас есть подписки бесплатной ценовой категории, это повлияет на их рейтинг безопасности, так как ранее безопасность идентификации и доступа таких подписок не оценивалась. 

Когда центр безопасности выявляет потенциальные уязвимости в системе безопасности, он создает рекомендации по настройке необходимых элементов управления, чтобы укрепить защиту ресурсов.

Периметр безопасности прошел развитие от периметра сети до периметра на основе удостоверений. Защита сети в моделях безопасности отходит на второй план, уступая место защите данных, а также управлению безопасностью приложений и пользователей. В настоящее время, когда большинство данных и приложений перемещаются в облако, идентификация становится новым периметром.

Отслеживая действия по идентификации, можно принимать упреждающие меры до возникновения инцидента или ответные меры для предотвращения попыток атак. Например, Центр безопасности может пометить устаревшие учетные записи (учетные записи, которые больше не нужны и заблокированы Azure Active Directory для входа) для удаления. 

Примеры рекомендаций, которые можно увидеть в разделе Центра безопасности Azure, посвященном безопасности ресурсов (**Идентификация и доступ**), включают следующее:

- MFA должна быть включена для учетных записей с разрешениями владельца в вашей подписке.
- Подписке можно назначить не более 3 владельцев.
- Внешние учетные записи с разрешениями на чтение следует удалять из подписки.
- Устаревшие учетные записи следует удалять из подписки.

Дополнительные сведения об этих рекомендациях, а также полный список вы найдете в статье [Рекомендации по идентификации и доступу](recommendations-reference.md#recs-identity).

> [!NOTE]
> Если подписка имеет более чем 600 учетных записей, Центр безопасности не сможет определить рекомендации по идентификации для вашей подписки. Рекомендации, проверки на которые не выполняются, отображаются в категории "Оценки недоступны".
Центр безопасности не может определять рекомендации по идентификации для агентов администрирования поставщика облачных решений.
>


Все рекомендации по идентификации и доступу доступны в двух разделах с инструментами управления безопасностью на странице **Рекомендации**:

- Управление доступом и разрешения 
- Включение MFA

## <a name="enable-multi-factor-authentication-mfa"></a>Включение многофакторной проверки подлинности (MFA)

Для включения MFA требуются [разрешения клиента Azure Active Directory (AD)](https://docs.microsoft.com/azure/active-directory/users-groups-roles/directory-assign-admin-roles). 

- Если вы пользуетесь выпуском AD Premium, включите MFA, используя [Условный доступ](../active-directory/conditional-access/concept-conditional-access-policy-common.md).

- Пользователи бесплатного выпуска AD могут включить **параметры безопасности по умолчанию** в Azure Active Directory, как описано в [документации к AD](https://docs.microsoft.com/azure/active-directory/fundamentals/concept-fundamentals-security-defaults), однако рекомендация Центра безопасности о включении MFA будет по-прежнему отображаться.

# <a name="monitor-identity-and-access"></a>Мониторинг идентификации и доступа

> [!TIP]
> С марта 2020 года рекомендации центра идентификации и доступа Azure Security Center включены во все подписки на уровень бесплатного ценообразования. Если у вас есть подписки на бесплатный уровень, их Безопасный счет будет затронут, поскольку они не были ранее оценены для их идентификации и безопасности доступа. 

Когда центр безопасности выявляет потенциальные уязвимости в системе безопасности, он создает рекомендации по настройке необходимых элементов управления, чтобы укрепить защиту ресурсов.

Периметр безопасности прошел развитие от периметра сети до периметра на основе удостоверений. Защита сети в моделях безопасности отходит на второй план, уступая место защите данных, а также управлению безопасностью приложений и пользователей. В настоящее время, когда большинство данных и приложений перемещаются в облако, идентификация становится новым периметром.

Мониторинг действия идентификационных данных можно предпринять упреждающие действия до инцидента или реактивные действия, чтобы остановить попытку атаки. Например, Центр безопасности может отмечать депрепроизлагаемые учетные записи (учетные записи, которые больше не нужны, и блокировать для удаления ввод в систему Active Directory. 

Примеры рекомендаций, которые можно увидеть в разделе безопасности **ресурсов идентификации и доступа** центра безопасности Azure:

- MFA должна быть включена для учетных записей с разрешениями владельца в вашей подписке.
- Подписке можно назначить не более 3 владельцев.
- Внешние учетные записи с разрешениями на чтение следует удалять из подписки.
- Устаревшие учетные записи следует удалять из подписки.

Для получения дополнительной информации об этих рекомендациях, а также полный список рекомендаций, которые вы можете увидеть здесь, смотрите [рекомендации по идентификации и доступу](recommendations-reference.md#recs-identity).

> [!NOTE]
> Если подписка имеет более чем 600 учетных записей, Центр безопасности не сможет определить рекомендации по идентификации для вашей подписки. Рекомендации, которые не запущены, перечислены в "недоступных оценках" ниже.
Центр безопасности не может определять рекомендации по идентификации для агентов администрирования поставщика облачных решений.
>


Все рекомендации по идентификации и доступу доступны в рамках двух элементов управления безопасностью на странице **Рекомендации:**

- Управление доступом и разрешениями 
- Включение MFA

![Два контроля безопасности с рекомендациями, касающимися идентификации и доступа](media/security-center-identity-access/two-security-controls-for-identity-and-access.png)


## <a name="enable-multi-factor-authentication-mfa"></a>Включить многофакторную аутентификацию (MFA)

Включение MFA требует [разрешения на использование арендаторов Найзурного Активного каталога (AD).](https://docs.microsoft.com/azure/active-directory/users-groups-roles/directory-assign-admin-roles) 

- Если у вас есть премиум издание AD, включите МИД с помощью [условного доступа.](https://docs.microsoft.com/azure/active-directory/conditional-access/overview)

- Пользователи бесплатного издания AD могут включить **по умолчанию безопасности** в Azure Active Directory, как описано в [документации AD,](https://docs.microsoft.com/azure/active-directory/fundamentals/concept-fundamentals-security-defaults) но рекомендация Центра безопасности для включения MFA все равно будет отображаться.

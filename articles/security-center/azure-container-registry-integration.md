# <a name="azure-container-registry-integration-with-security-center"></a>Интеграция реестра контейнеров Azure с центром безопасности

Реестр контейнеров Azure (запись контроля доступа) — это управляемая частная служба реестра DOCKER, которая хранит образы контейнеров для развертываний Azure и управляет ими в центральном реестре. Он основан на реестре DOCKER с открытым кодом 2,0.

Если вы используете уровень "Стандартный" центра безопасности Azure, можно добавить пакет реестров контейнеров. Эта дополнительная функция позволяет более подробно ознакомиться с уязвимостями образов в реестров на основе ARM. Включите или отключите набор на уровне подписки, чтобы охватить все реестры в подписке. Плата за этот компонент насчитывается отдельно для каждого образа, как показано на [странице цен](security-center-pricing.md). При включении пакета реестров контейнеров обеспечивается готовность центра безопасности к сканированию образов, помещаемых в реестр. 


## <a name="availability"></a>Доступность

- Состояние выпуска: **Общая доступность**
- Обязательные роли: **читатель безопасности** и [роль читателя реестра контейнеров Azure](https://docs.microsoft.com/azure/container-registry/container-registry-roles)
- Поддерживаемые реестры:
    - ✔ Реестрами записей контроля доступа, размещенных в Linux, которые доступны из общедоступного Интернета и предоставляют доступ к оболочке.
    - ✘ Реестрами записей контроля доступа, размещенных в Windows.
    - ✘ "Private" реестра — центр безопасности требует, чтобы ваши реестры были доступны из общедоступного Интернета. Если вы ограничили доступ к вашим реестрам с помощью брандмауэра, конечной точки службы или частной конечной точки (например, частной связи Azure), центр безопасности не сможет подключиться к реестру или проверить его.
    - ✘Ные образы Super, такие как образы " [Рабочая](https://hub.docker.com/_/scratch/) область", или образы "дистролесс", которые содержат только приложение и его зависимости времени выполнения без диспетчера пакетов, оболочки или ОС.
- Облако. 
    - ✔ коммерческие облака;
    - Облако ✘ для государственных организаций США
    - ✘ Китая для государственных организаций, других govных облаков


## <a name="when-are-images-scanned"></a>Когда проверяются изображения?

При каждом отправке образа в реестр центр безопасности автоматически проверяет этот образ. Чтобы запустить сканирование образа, отправьте его в репозиторий.

По завершении проверки (обычно через 2 минуты, но может составлять до 15 минут) результаты доступны в виде рекомендаций центра безопасности следующим образом:

## <a name="benefits-of-integration"></a>Преимущества интеграции

Центр безопасности определяет реестры записей контроля доступа на базе ARM в подписке и позволяет легко предоставить следующие сведения:

* **Поиск уязвимостей** для всех отправленных образов Linux в Azure. Центр безопасности сканирует образ с помощью сканера, который является ведущим в отрасли для проверки уязвимостей поставщика, Qualys. Это собственное решение по умолчанию тесно интегрировано.

* **Рекомендации по безопасности** для образов Linux с известными уязвимостями. Центр безопасности предоставляет подробные сведения о каждой обнаруженной уязвимости и классификации серьезности. Кроме того, он предоставляет рекомендации по исправлению конкретных уязвимостей, обнаруженных в каждом образе, отправленном в реестр.

## <a name="acr-with-security-center-faq"></a>Вопросы и ответы по центру безопасности

### <a name="how-does-azure-security-center-scan-an-image"></a>Как центр безопасности Azure сканирует образ?
Образ извлекается из реестра. Затем он выполняется в изолированной песочнице с помощью сканера Qualys, который извлекает список известных уязвимостей.

Центр безопасности фильтрует и классифицирует результаты проверки. Если образ работоспособен, центр безопасности помечает его как таковой. Центр безопасности создает рекомендации по безопасности только для образов с проблемами, которые могут быть разрешены. Только уведомляя о наличии проблем, центр безопасности сокращает вероятность нежелательных информационных оповещений.

### <a name="how-often-does-azure-security-center-scan-my-images"></a>Как часто центр безопасности Azure проверяет мои образы?
Сканирование изображений активируется при каждой отправке.

### <a name="can-i-get-the-scan-results-via-rest-api"></a>Можно ли получить результаты проверки с помощью REST API?
Да. Результаты находятся в [подсистеме API подоценки](/rest/api/securitycenter/subassessments/list/). Кроме того, вы можете использовать граф ресурсов Azure (ARG), Kusto API для всех ресурсов: запрос может получить определенное сканирование.

# <a name="azure-container-registry-integration-with-security-center"></a>Интеграция реестра контейнеров Azure с Центром безопасности

Реестр контейнеров Azure (ACR) — это управляемая частная служба реестра докеров, которая хранит и управляет изображениями контейнеров для развертывания Azure в центральном реестре. Он основан на открытом исходном коде Docker Registry 2.0.

Если вы находитесь на стандартном уровне Центра безопасности Azure, можно добавить пакет контейнерных реестров. Эта дополнительная функция обеспечивает более глубокую видимость уязвимостей изображений в реестрах на основе ARM. Включить или отключить пакет на уровне подписки, чтобы охватить все реестры в подписке. Эта функция заряжается за изображение, как показано на [странице ценообразования.](security-center-pricing.md) Включение пакета контейнерных реестров гарантирует, что Центр безопасности готов к сканированию изображений, которые попадают в реестр. 

Всякий раз, когда изображение выталкивается в ваш реестр, Центр безопасности автоматически сканирует это изображение. Чтобы вызвать сканирование изображения, нажмите его на репозиторий.

Когда сканирование завершается (обычно примерно через 10 минут), выводы доступны в рекомендациях Центра безопасности:

## <a name="benefits-of-integration"></a>Преимущества интеграции

Центр безопасности определяет реестры ACR на основе ARM в вашей подписке и обеспечивает:

* **Сканирование уязвимости, нативной Azure** для всех изображений Linux. Центр безопасности сканирует изображение с помощью сканера от ведущего в отрасли поставщика сканирования уязвимостей, Квалис. Это родное решение легко интегрируется по умолчанию.

* **Рекомендации по безопасности** для изображений Linux с известными уязвимостями. Центр безопасности предоставляет подробную информацию о каждой зарегистрированной уязвимости и классификации серьезности. Кроме того, в нем содержатся рекомендации по устранению конкретных уязвимостей, обнаруженных на каждом изображении, отображаемом в реестре.

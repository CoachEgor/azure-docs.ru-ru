# <a name="azure-container-registry-image-scanning-by-security-center"></a>Сканирование образа реестра контейнеров Azure по центру безопасности

Реестр контейнеров Azure (запись контроля доступа) — это управляемая частная служба реестра DOCKER, которая хранит образы контейнеров для развертываний Azure и управляет ими в центральном реестре. Он основан на реестре DOCKER с открытым кодом 2,0.

Включите **защитник Azure для реестров контейнеров** , чтобы получить более подробные сведения об уязвимостях образов в реестре на основе Azure Resource Manager. Включите или отключите план на уровне подписки, чтобы охватить все реестры в подписке. Плата за этот компонент насчитывается отдельно для каждого образа, как показано на [странице цен](security-center-pricing.md). При включении защитника Azure обеспечивается готовность центра безопасности к сканированию образов, помещаемых в реестр. 


## <a name="when-are-images-scanned"></a>Когда проверяются изображения?

При каждом отправке образа в реестр центр безопасности автоматически проверяет этот образ. Чтобы запустить сканирование образа, отправьте его в репозиторий.

По завершении проверки (обычно через 2 минуты, но может составлять до 15 минут) результаты доступны в виде рекомендаций центра безопасности следующим образом:

## <a name="benefits-of-integration"></a>Преимущества интеграции

Центр безопасности определяет Azure Resource Manager реестры контроля доступа на основе записей в подписке и легко предоставляет:

* **Поиск уязвимостей** для всех отправленных образов Linux в Azure. Центр безопасности сканирует образ с помощью сканера, который является ведущим в отрасли для проверки уязвимостей поставщика, Qualys. Это собственное решение по умолчанию тесно интегрировано.

* **Рекомендации по безопасности** для образов Linux с известными уязвимостями. Центр безопасности предоставляет подробные сведения о каждой обнаруженной уязвимости и классификации серьезности. Кроме того, он предоставляет рекомендации по исправлению конкретных уязвимостей, обнаруженных в каждом образе, отправленном в реестр.

## <a name="faq-for-azure-container-registry-image-scanning"></a>Вопросы и ответы по сканированию образа реестра контейнеров Azure

### <a name="how-does-security-center-scan-an-image"></a>Как центр безопасности сканирует образ?
Образ извлекается из реестра. Затем он выполняется в изолированной песочнице с помощью сканера Qualys, который извлекает список известных уязвимостей.

Центр безопасности фильтрует и классифицирует результаты проверки. Если образ работоспособен, центр безопасности помечает его как таковой. Центр безопасности создает рекомендации по безопасности только для образов с проблемами, которые могут быть разрешены. Только уведомляя о наличии проблем, центр безопасности сокращает вероятность нежелательных информационных оповещений.

### <a name="how-often-does-security-center-scan-my-images"></a>Как часто центр безопасности проверяет мои образы?
Сканирование изображений активируется при каждой отправке.

### <a name="can-i-get-the-scan-results-via-rest-api"></a>Можно ли получить результаты проверки с помощью REST API?
Да. Результаты находятся в [подсистеме API подоценки](/rest/api/securitycenter/subassessments/list/). Кроме того, вы можете использовать граф ресурсов Azure (ARG), Kusto API для всех ресурсов: запрос может получить определенное сканирование.
 
### <a name="what-registry-types-are-scanned-what-types-are-billed"></a>Какие типы реестра проверяются? На какие типы взимается плата?
В разделе Доступность перечислены типы реестров контейнеров, поддерживаемые защитником Azure для реестров контейнеров. 

Если реестры, которые не поддерживаются, подключены к вашей подписке Azure, они не будут просканированы и плата за них не будет взиматься.

#  <a name="connect-your-gcp-accounts-to-azure-security-center"></a>Подключение учетных записей обеспечить к центру безопасности Azure

Облачные рабочие нагрузки, обычно охватывающие несколько облачных платформ, должны выполнять те же действия в облачной службе безопасности.

Центр безопасности Azure защищает рабочие нагрузки в Azure, Amazon Web Services (AWS) и Google Cloud Platform (обеспечить).

При адаптации учетной записи обеспечить к центру безопасности интегрируется команда безопасности обеспечить и центр безопасности Azure. Таким образом, центр безопасности обеспечивает видимость и защиту в обеих этих облачных средах для предоставления следующих средств:

- Обнаружение ненастроек безопасности
- Одно представление, в котором показаны рекомендации центра безопасности и результаты работы центра команд безопасности обеспечить
- Организация ресурсов обеспечить в расчеты безопасных оценок центра безопасности
- Интеграция рекомендаций центра безопасности обеспечить на основе CI Standard в панель мониторинга соответствия нормативным требованиям центра безопасности

На снимке экрана ниже можно увидеть проекты обеспечить, отображаемые в обзорной панели мониторинга центра безопасности.

:::image type="content" source="./media/quickstart-onboard-gcp/gcp-account-in-overview.png" alt-text="3 обеспечить проекты, перечисленные на информационной панели обзора центра безопасности" lightbox="./media/quickstart-onboard-gcp/gcp-account-in-overview.png":::

## <a name="connect-your-gcp-account"></a>Подключение к учетной записи GCP

### <a name="step-1-set-up-gcp-security-command-center-with-security-health-analytics"></a>Шаг 1. Настройка центра команд безопасности обеспечить с помощью анализа работоспособности системы безопасности

Для всех проектов обеспечить в Организации необходимо также:

1. Настройте **центр команд безопасности обеспечить** , используя [эти инструкции из документации по обеспечить](https://cloud.google.com/security-command-center/docs/quickstart-scc-setup).
1. Включите **Анализ работоспособности системы безопасности** с помощью [этих инструкций из документации по обеспечить](https://cloud.google.com/security-command-center/docs/how-to-use-security-health-analytics).
1. Убедитесь, что в центре команд безопасности есть данные.

Инструкции по подключению среды обеспечить к конфигурации безопасности приведены в разделе рекомендации Google по использованию рекомендаций по настройке безопасности. Интеграция использует центр команд Google Security и будет использовать дополнительные ресурсы, которые могут повлиять на выставление счетов.

При первом включении анализа работоспособности системы безопасности доступ к данным может занять несколько часов.


### <a name="step-2-enable-gcp-security-command-center-api"></a>Шаг 2. Включение API центра команд безопасности обеспечить

1. Из **библиотеки API облачной консоли**Google выберите проект, который вы хотите подключить к центру безопасности Azure.
1. В библиотеке API найдите и выберите **API центра команд безопасности**.
1. На странице API выберите **включить**.

Дополнительные сведения об [API центра команд безопасности](https://cloud.google.com/security-command-center/docs/reference/rest/).


### <a name="step-3-create-a-dedicated-service-account-for-the-security-configuration-integration"></a>Шаг 3. Создание выделенной учетной записи службы для интеграции настройки безопасности

1. В **консоли обеспечить**выберите проект, который нужно подключить к центру безопасности.
1. В **меню навигации**в разделе параметры **администратора & IAM** выберите **учетные записи служб**.
1. Выберите **создать учетную запись службы**.
1. Введите имя учетной записи и нажмите кнопку **создать**.
1. Укажите **роль** в качестве **средства просмотра администратора центра безопасности**и нажмите кнопку **продолжить**.
1. Раздел **предоставление пользователям доступа к этой учетной записи службы** является необязательным. Нажмите кнопку **Готово**.
1. Скопируйте **значение электронной почты** созданной учетной записи службы и сохраните его для последующего использования.
1. В **меню навигации**в разделе Параметры **администратора IAM &** выберите **IAM** .
    1. Переключитесь на уровень организации.
    1. Нажмите кнопку **Добавить**.
    1. В поле **New Members (новые члены** ) Вставьте **значение электронного письма** , скопированное ранее.
    1. Укажите роль в качестве **средства просмотра администратора центра безопасности** и нажмите кнопку Сохранить.
        :::image type="content" source="./media/quickstart-onboard-gcp/iam-settings-gcp-permissions-admin-viewer.png" alt-text="Установка соответствующих разрешений обеспечить":::


### <a name="step-4-create-a-private-key-for-the-dedicated-service-account"></a>Шаг 4. Создание закрытого ключа для выделенной учетной записи службы
1. Переключитесь на уровень проекта.
1. В **меню навигации**в разделе параметры **администратора & IAM** выберите **учетные записи служб**.
1. Откройте выделенную учетную запись службы и выберите Изменить.
1. В разделе **ключи** выберите **Добавить раздел** , а затем **создать новый раздел**.
1. На экране создание закрытого ключа выберите **JSON**и нажмите кнопку **создать**.
1. Сохраните этот JSON файл для последующего использования.


### <a name="step-5-connect-gcp-to-security-center"></a>Шаг 5. Подключение обеспечить к центру безопасности 
1. В меню центра безопасности выберите " **облачные соединители**".
1. Выберите Добавить учетную запись обеспечить.
1. На странице адаптации выполните следующие действия, а затем нажмите кнопку **Далее**.
    1. Проверьте выбранную подписку.
    1. В поле **Отображаемое имя** введите отображаемое имя соединителя.
    1. В поле **идентификатор организации** введите идентификатор своей организации. Если вы его не узнаете, см. статью [создание организаций и управление ими](https://cloud.google.com/resource-manager/docs/creating-managing-organization).
    1. В поле файл **закрытого ключа** перейдите к файлу JSON, скачанному на [шаге 4. Создайте закрытый ключ для выделенной учетной записи службы](#step-4-create-a-private-key-for-the-dedicated-service-account).


### <a name="step-6-confirmation"></a>Шаг 6. Подтверждение

Если соединитель успешно создан и обеспечить центр команд безопасности настроен правильно:

- Стандарт обеспечить CIS будет отображаться на панели мониторинга соответствия нормативным требованиям центра безопасности.
- Рекомендации по безопасности для ресурсов обеспечить будут отображаться на портале Центра безопасности и на панели мониторинга соответствия нормативам 5-10 минут после подключения: :::image type="content" source="./media/quickstart-onboard-gcp/gcp-resources-in-recommendations.png" alt-text="ресурсы и рекомендации на странице рекомендаций центра безопасности"::: .

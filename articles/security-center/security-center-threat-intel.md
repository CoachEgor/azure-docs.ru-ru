# <a name="security-alerts-map-and-threat-intelligence"></a>Карта оповещений системы безопасности и аналитика угроз
Эта статья поможет вам использовать основанные на событиях безопасности карту оповещений безопасности и карту анализа угроз центра безопасности Azure для устранения неполадок, связанных с безопасностью.

## <a name="how-the-security-alerts-map-works"></a>Как работает схема оповещений безопасности
Центр безопасности предоставляет карту, которая помогает выявлять угрозы безопасности для вашей среды. Например, можно определить, что конкретный компьютер является частью ботнета, и источник угрозы. Компьютеры могут стать узлами в ботнете, если злоумышленники незаконно установят вредоносную программу, которая тайно подключает этот компьютер к среде управления и контроля, управляющей ботнетом. 

Для построения карты центр безопасности использует данные, поступающие из нескольких источников в Майкрософт. Центр безопасности использует эти данные для составления карты потенциальных угроз в вашей среде. 

В рамках [реагирования на нарушение безопасности](https://docs.microsoft.com/azure/security-center/security-center-planning-and-operations-guide#incident-response) необходимо определить степень серьезности нарушения безопасности систем. На этом этапе следует выполнить следующие задачи:

- определить характер атаки;
- определить источник атаки;
- определить цель атаки (была ли атака направлена на вашу организацию для получения конкретных данных, или она случайна);
- определить системы, безопасность которых была нарушена;
- определить файлы, к которым был получен доступ, и степень конфиденциальности этих файлов.

Данные на карте оповещений безопасности угроз в центре безопасности позволяют выполнять эти задачи.

## <a name="access-the-security-alerts-map"></a>Доступ к карте оповещений безопасности
Для визуализации текущих угроз в рабочей среде откройте карту оповещений безопасности:

1. Откройте панель мониторинга **центра безопасности**.
2. В области слева в разделе **Защита от угроз** выберите пункт **Карта оповещений безопасности**. Откроется карта.
3. Чтобы получить дополнительные сведения об оповещении и устранении проблемы, щелкните красную точку оповещения на карте и следуйте инструкциям. 
 
Карта оповещений безопасности основана на оповещениях. Эти оповещения основаны на действиях, в ходе которых сетевая активность была связана с IP-адресом, который был успешно разрешен, независимо от того, является ли IP-адрес известным опасным IP-адресом (например, известным криптомайнером) или IP-адресом, который не распознавался ранее как рискованный. Карты содержат оповещения по всем подпискам, выбранным вами ранее в Azure. 

Оповещения на карте отображаются в соответствии с географическим расположением, откуда они исходят, и обозначаются цветами по уровню серьезности. 
    ![Сведения об анализе угроз](./media/security-center-threat-intel/security-center-alert-map.png)

## <a name="viewing-the-event-based-threat-intelligence-dashboard"></a>Просмотр панели мониторинга для аналитики угроз на основе событий
Чтобы просмотреть карту анализа угроз на основе необработанных событий безопасности, можно выполнить эту процедуру. Эта карта отображает только события, в которых участвуют IP-адреса, которые считаются рискованными, например IP-адрес известного ботнета.

1. Откройте панель мониторинга **центра безопасности**.

1. В области слева в разделе **Защита от угроз** выберите пункт **Карта оповещений безопасности**. Откроется карта.
2. В правом верхнем углу нажмите кнопку **Перейти к карте событий безопасности**.
3. Выберите рабочую область, для которой требуется просмотреть панели мониторинга.
4. В верхней части карты выберите **Просмотреть классическую аналитику угроз**. Откроется панель мониторинга **Анализ угроз**.

   > [!NOTE]
   > Если в столбце справа отображается **UPGRADE PLAN** (План обновления), рабочая область использует бесплатную подписку. Повысьте уровень подписки до категории "Стандартный", чтобы использовать эту функцию. Если в крайнем правом столбце отображается **требуется обновить**, обновить [журналы Azure Monitor](https://docs.microsoft.com/azure/log-analytics/log-analytics-overview) для использования этой функции. Дополнительные сведения о тарифном плане см. на странице с ценами на центр безопасности Azure.
   >
5. Если вам нужно изучить более одной рабочей области, вы можете расставить приоритеты в соответствии со столбцом **Вредоносные IP-адреса**. В нем отображается текущее число вредоносных IP-адресов в этой рабочей области. Щелкните рабочую область, которую необходимо использовать, и появится панель мониторинга **Аналитика угроз**.

    ![Сведения об анализе угроз](./media/security-center-threat-intel/security-center-threat-intel-fig5.png)

6. Эта панель мониторинга состоит из трех плиток:

    a.  **Типы угроз**. Здесь представлены типы угроз, которые были обнаружены в выбранной рабочей области.

    2.  **Страна или регион источника**. Здесь представлен суммарный объем трафика по его источнику.

    c.  **Расположение угроз**. Помогает определить текущие расположения, в которых происходит обмен данными с вашей средой, по всему миру. На приведенной карте оранжевые (входящие) и красные (исходящие) стрелки определяют направления трафика. При выборе одной из этих стрелок отображается тип угрозы и направление трафика.

    d.  **Сведения об угрозах**. Здесь представлены дополнительные сведения об угрозе, выбранной на карте.

Независимо от выбранной плитки панель мониторинга, которая появляется, основана на запросе поиска по журналам. Единственным различием является тип запроса и результат.

### <a name="threat-types"></a>Типы угроз
Выберите плитку **Типы угроз**, чтобы открыть панель мониторинга **Поиск по журналам**. Параметры фильтра отображаются слева, а результаты запроса — справа.

![Поиск по журналам](./media/security-center-threat-intel/security-center-threat-intel-fig3.png)

В результатах запроса угрозы сгруппированы по имени. В области слева можно выбрать атрибут для фильтрации. Например, чтобы просмотреть только угрозы, которые в данный момент подключены к машинам, выберите для **Состояние сеанса** значение **Подключено** > **Применить**.

![Состояние сеанса](./media/security-center-threat-intel/security-center-threat-intel-fig4.png)

Для виртуальных машин Azure на панели мониторинга **Аналитика угроз** будут отображаться только данные сети, которые проходят через агент. Кроме того, при анализе угроз используются следующие данные:

- Данные CEF (тип = CommonSecurityLog)
- WireData (тип = WireData)
- Журналы IIS (тип = W3CIISLog)
- Брандмауэр Windows (тип = WindowsFirewall)
- События DNS (тип = DnsEvents)

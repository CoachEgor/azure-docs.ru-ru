# <a name="custom-alert-rules-in-azure-security-center-preview"></a>Настраиваемые правила генерации оповещений в центре безопасности Azure (предварительная версия)
В этом документе описана процедура создания настраиваемых правил генерации оповещений в центре безопасности Azure.

> [!NOTE]
> Настраиваемые оповещения больше поддерживается в центре обеспечения безопасности. 

## <a name="retirement-of-custom-alert-rules-in-azure-security-center"></a>Правила вывода из эксплуатации настраиваемые оповещения в центре безопасности Azure

Пользовательские оповещения больше поддерживается на 30 июня 2019 г., из-за прекращения использования базовой инфраструктуры, он предполагавшим статическую компоновку. любые пользовательские предупреждения, определенные до retriement не вступают в силу и оповещения системы безопасности на основе этих правил настраиваемые оповещения не создаются. Чтобы повторно создать их в альтернативные варианты, описанные ниже можно просматривать запросы вашего настраиваемые правила генерации оповещений в центре безопасности:

Пользователям рекомендуется либо:

Чтобы сохранить существующие оповещения и создайте их повторно в Azure Sentinel, пожалуйста, [запуска Azure Sentinel](https://portal.azure.com/#create/Microsoft.ASI/preview). В качестве первого шага выберите рабочую область, где хранятся ваши пользовательские оповещения и затем выберите пункт меню «Аналитика» для настройки правил настраиваемые оповещения. Для повторного создания оповещения с помощью оповещений журнала Azure Monitor, см.: [Создавать, просматривать и управлять ими с помощью Azure Monitor оповещения журнала](https://docs.microsoft.com/azure/azure-monitor/platform/alerts-log) инструкции по созданию оповещений журнала. Общие сведения о оповещения журнала в Azure Monitor, щелкните [здесь](https://docs.microsoft.com/azure/azure-monitor/platform/alerts-unified-log).

## <a name="what-are-custom-alert-rules-in-security-center"></a>Что такое настраиваемые правила генерации оповещений в центре безопасности

Центр обеспечения безопасности имеет ряд предопределенных [оповещений безопасности](https://docs.microsoft.com/azure/security-center/security-center-managing-and-responding-alerts), которые активируются при возникновении угрозы или подозрительной активности. В некоторых сценариях может потребоваться создать настраиваемые оповещения для решения конкретных задач среды.

Настраиваемые правила генерации оповещений в центре безопасности позволяют определять новые оповещения безопасности на основе данных, собранных из среды. Можно создавать запросы, а результат этих запросов можно использовать в качестве критериев для настраиваемого правила. Как только критерий будет удовлетворен, это правило выполняется. Для создания настраиваемых запросов можно использовать события безопасности компьютеров, журналы решений безопасности партнеров или данные, полученные с помощью API-интерфейсов.

> [!NOTE]
> Настраиваемые оповещения не поддерживаются [функцией изучения](security-center-investigation.md) центра безопасности.
>
>

## <a name="how-to-create-a-custom-alert-rule-in-security-center"></a>Как создать настраиваемое правило генерации оповещений в центре безопасности

Откройте панель мониторинга **центра безопасности** и выполните следующие действия для создания настраиваемого правила:

1.  В левой области в разделе **Обнаружение** щелкните **Custom alert rules (Preview)** (Настраиваемые правила генерации оповещений (предварительная версия)).
2.  На странице **Центр безопасности. Настраиваемые правила генерации оповещений (предварительная версия)** щелкните **New custom alert rule** (Новое настраиваемое правило генерации оповещений).

    ![Настраиваемое оповещение](./media/security-center-custom-alert/security-center-custom-alert-fig1.png)

3.  Страница создания настраиваемого правила генерации оповещений отображается со следующими параметрами:

    ![Создание](./media/security-center-custom-alert/security-center-custom-alert-fig2.png)

4.  В поле **Имя** укажите имя настраиваемого правила.
5.  В поле **Описание** введите краткое описание, которое отражает суть этого правила.
6.  В поле **Серьезность** выберите уровень серьезности (высокий, средний или низкий) в соответствии с потребностями.
7.  В поле **Подписка** выберите подписку, в которой применяется это правило.
8.  В поле **Рабочая область** выберите рабочую область, которую требуется отслеживать с помощью этого правила, а в поле **Поисковый запрос** — запрос, который нужно использовать для получения результатов.

    > [!NOTE]
    > Для рабочей области, которую вы выбрали для хранения настраиваемых оповещений, требуются права на запись.
    >
    >

    Результат запроса активирует это оповещение. Обратите внимание, что при вводе допустимого запроса отображается зеленый флажок в правом углу этого поля:

    ![Запрос](./media/security-center-custom-alert/security-center-custom-alert-fig3.png)

10. В поле **Период** выберите интервал времени, в котором приведенный выше запрос будет выполняться. Обратите внимание, что результат поиска в нижней части этого поля изменится в соответствии с выбранным интервалом времени.

    ![Период](./media/security-center-custom-alert/security-center-custom-alert-fig4.png)

11. В поле **Проверка** выберите частоту, с которой следует проверять и выполнять это правило.
12. В поле **Количество результатов** выберите оператор (больше или меньше).
13. В поле **Пороговое значение** введите число, которое будет использоваться в качестве эталонного для оператора, который был выбран ранее.
14. Включите параметр **Отменить вывод оповещений**, если вы хотите установить значение времени ожидания, прежде чем центр безопасности отправит еще одно оповещение для этого правила.
15. Нажмите кнопку **ОК** для завершения.

Когда новое правило генерации оповещений будет создано, оно появится в соответствующем списке. Как только условия этого правила будут выполнены, будет активировано новое оповещение, которое вы сможете увидеть на панели мониторинга **Оповещения безопасности**.

![Предупреждение](./media/security-center-custom-alert/security-center-custom-alert-fig5.png)

Обратите внимание, что параметры (поисковый запрос, пороговое значение и т. д.), которые были установлены во время создания правила, доступны в оповещении для этого настраиваемого правила.

# <a name="investigate-incidents-and-alerts-in-azure-security-center-retired"></a>Исследование инцидентов и оповещений в центре безопасности Azure (прекращено)
Этот документ поможет вам использовать функцию расследования (Предварительная версия) в центре безопасности Azure для исследования инцидентов и предупреждений безопасности.

> [!NOTE]
> Процесс оповещений и исследований инцидентов (Предварительная версия) был снят с 31 июля 2019. Дополнительные сведения и альтернативные службы см. в разделе [выбытие средств центра безопасности (июль 2019)](security-center-features-retirement-july2019.md#menu_investigate).

## <a name="what-is-investigation-in-security-center"></a>Что такое изучение в центре безопасности
Функция изучения в центре безопасности позволяет рассматривать, определять область и отслеживать первопричину вероятного [инцидента](https://docs.microsoft.com/azure/security-center/security-center-incident).

Цель состоит в том, чтобы облегчить процесс изучения, связав все сущности ([предупреждения системы безопасности](https://docs.microsoft.com/azure/security-center/security-center-alerts-type), пользователей, компьютеры и инциденты), которые связаны с изучаемым инцидентом.  Центр безопасности может сделать это, сопоставляя релевантные данные с любыми задействованными сущностями и предоставляя эту корреляцию с использованием динамического графика, который позволяет перемещаться между объектами и визуализировать соответствующую информацию.


> [!NOTE]
> * [Настраиваемые оповещения](security-center-custom-alert.md) не поддерживаются функцией изучения центра безопасности.
> * Исследование поддерживается только для оповещений, основанных на данных, собранных с серверов Windows.


## <a name="how-investigation-works"></a>Как работает изучение
Изучение происходит при помощи графика, который занимает центральную область панели мониторинга изучения. График всегда ориентирован на конкретную сущность и представляет связанные с ней сущности. Сущность может быть предупреждением системы безопасности, пользователем, компьютером или инцидентом.

![Карта](./media/security-center-investigation/security-center-investigation-fig1.png)

Пользователь может перейти от одной сущности к другой, щелкнув ее на графике. График автоматически обновляется для выбранной сущности и связанных с ней сущностей. Сущности, которые больше не являются релевантными, могут быть удалены из графика.

### <a name="investigation-path"></a>Путь исследования
Пока пользователь перемещается между различными сущностями, путь изучения помогает отслеживать контекст исследования и выполнять быстрое перемещение. Инцидент, содержащий результаты исследования, всегда находится в крайнем левом инциденте в пути исследования.

![Path](./media/security-center-investigation/security-center-investigation-fig2.png)

### <a name="general-information"></a>Общая информация
Когда сущность представлена на графике, на вкладке отображаются дополнительные сведения об этой сущности. На вкладке **Сведения** представлены общие сведения о сущности из различных доступных источников.

![Общие сведения](./media/security-center-investigation/security-center-investigation-fig3.png)

На вкладке сведений отображается информация, относящаяся к инциденту, выбранному на карте. Инцидент является контейнером, который содержит результаты изучения. Каждое изучение происходит в контексте инцидента.

Инцидент создается только в том случае, когда пользователь нажимает кнопку **Start investigation** (Начать исследование) для указанного оповещения. Основная возможность, доступная исследователю, — это возможность отмечать сущности, такие как пользователь, компьютер или оповещение. Когда сущность помечена как связанная, предоставляется причина. С этого момента эта сущность появляется непосредственно под инцидентом на графике и в списке сущностей инцидента.

### <a name="entities"></a>Сущности

На вкладке **Сущности** отображаются все связанные сущности, сгруппированные по типу. Это полезно в двух случаях: когда слишком много сущностей присутствуют на графике и когда имена сущностей слишком длинные и их проще рассматривать в табличном виде.

![Сущности](./media/security-center-investigation/security-center-investigation-fig4.png)

### <a name="search"></a>Найти

На вкладке **Поиск** представлены все типы журналов, доступные для сущности. Для каждого типа журнала можно просмотреть количество доступных записей. Щелкнув каждый тип журнала, можно перейти на экран поиска. На экране поиска можно уточнить параметры поиска и использовать различные функции, такие как настройка оповещений. В текущем выпуске вкладка поиска доступна только для сущностей пользователей и компьютеров.

![Найти](./media/security-center-investigation/security-center-investigation-fig5.png)

### <a name="exploration"></a>Изучение

Вкладка **Exploration** (Исследование) позволяет исследователю изучать данные, относящиеся к различным проблемам, связанным с сущностью. Например, при изучении компьютера список выполняемых на ней процессов представлен на вкладке исследования. В некоторых случаях вкладка исследования представляет данные, которые дают основание подозревать о существовании проблемы. Исследователь может изучить данные на вкладке или открыть их на экране поиска, чтобы изучить большие наборы данных и использовать расширенные параметры поиска, такие как фильтрация и экспорт в Excel.

![Изучение](./media/security-center-investigation/security-center-investigation-fig6.png)

### <a name="timeline"></a>Временная шкала

Большинство данных, представленных на графике и различных вкладках, относятся к определенному периоду времени. Этот промежуток времени устанавливается с помощью селектора промежутка времени в верхней левой части графика. Исследователю доступны различные методы выбора промежутка времени.

![Временная шкала](./media/security-center-investigation/security-center-investigation-fig7.png)

Следующие элементы зависят от промежутка времени:

- Отношения "пользователь — компьютер" на графике представлены только пользователями, которые вошли на компьютер в этот период времени.
- При просмотре компьютеров и пользователей представлены только оповещения, которые возникли в этом промежутке времени.
- Вкладка сущностей следует той же логике, что и график.

Следующие элементы будут представлены независимо от выбранного промежутка времени:

- При появлении оповещения в нем всегда отображаются все содержащиеся в нем сущности (пользователи и компьютеры).
- Если инцидент содержит сущность, она будет представлена.

> [!NOTE]
> На вкладках **Поиск** и **Исследование** отображаются только записи, возникшие в рамках этого промежутка времени.

## <a name="how-to-perform-an-investigation"></a>Способы выполнения изучения

Вы можете начать свое изучение с инцидента или с оповещения. Выбранный вами вариант будет отличаться в зависимости от ваших потребностей. Следующие шаги используются для запуска расследования из предупреждения.

1.  Откройте панель мониторинга **Центр безопасности**.
2.  Щелкните **Оповещения системы безопасности** и выберите инцидент, который требуется изучить.
3.  На странице инцидента нажмите кнопку **Start Investigation** (Начать анализ), после чего появится панель мониторинга **Investigation** (Исследование).

    ![Оповещение](./media/security-center-investigation/security-center-investigation-fig8.png)

4. На этой панели вы можете выбрать сущность на карте, и соответствующие сведения о сущности появятся в правой части экрана.

    ![Панель мониторинга исследования](./media/security-center-investigation/security-center-investigation-fig9.png)

Теперь вы можете изучить сущности, которые были вовлечены в этот инцидент, а также изучить более подробные сведения о каждой из них.

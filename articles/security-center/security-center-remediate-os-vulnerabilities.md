# <a name="remediate-security-configurations-in-azure-security-center"></a>Исправление конфигурации системы безопасности в центре безопасности Azure
Центр безопасности Azure ежедневно анализирует операционную систему виртуальных машин и компьютеров, чтобы определить конфигурацию, повышающую риск атак на них. Центр безопасности рекомендует устранять уязвимости, если конфигурация операционной системы не соответствует правилам рекомендуемой конфигурации безопасности, и изменять конфигурацию, чтобы устранить эти уязвимости.

Дополнительные сведения о конкретных отслеживаемых конфигурациях см. в [списке рекомендуемых правил конфигурации](https://gallery.technet.microsoft.com/Azure-Security-Center-a789e335). Сведения о настройке оценок конфигурации безопасности см. в статье [Настройка конфигураций безопасности операционной системы в центре безопасности Azure (предварительная версия)](security-center-customize-os-security-config.md).

## <a name="implement-the-recommendation"></a>Выполнение рекомендаций
Исправление конфигураций системы безопасности представлено в центре безопасности в виде рекомендации. Рекомендация отображается в разделе **Рекомендации** > **Вычислительные ресурсы и приложения**.

В этом примере рассматривается рекомендация из раздела **Вычислительные ресурсы и приложения**, посвященная исправлению конфигураций системы безопасности.
1. В центре безопасности в области слева выберите **Вычислительные ресурсы и приложения**.  
   Откроется окно **Вычислительные ресурсы и приложения**.

   ![Исправление конфигураций системы безопасности][1]

2. Выберите **Remediate security configurations** (Исправление конфигураций системы безопасности).  
   Откроется окно **Конфигурации безопасности**.

   ![Окно "Конфигурации безопасности"][2]

   В верхней части панели мониторинга отображаются такие плитки:

   - **Правила со сбоями по степени серьезности**: общее количество правил, не выполненных в конфигурации ОС виртуальной машины и компьютера, разбитых по типу серьезности.
   - **Правила со сбоями по типу**: общее количество правил, не выполненных в конфигурации ОС виртуальной машины и компьютера, разбитых по типу.
   - **Не сработавшие правила Windows**: общее количество правил, не выполненных в конфигурации ОС Windows.
   - **Не сработавшие правила Linux**: общее количество правил, не выполненных в конфигурации ОС Linux.

   В нижней части панели мониторинга перечислены все невыполненные правила виртуальных машин и компьютеров, а также уровень серьезности отсутствующего обновления. В списке содержатся следующие элементы:

   - **CCEID**: уникальный идентификатор CCE правила. Центр безопасности назначает уникальные идентификаторы правилам конфигурации на основе перечисления общей конфигурации (CCE).
   - **Name**. имя невыполненного правила.
   - **Тип правила**: *Раздел реестра*, *Политика безопасности*, *Политика аудита* или *IIS*.
   - **Число виртуальных машин и компьютеров**: общее количество виртуальных машин и компьютеров, к которым применялось невыполненное правило.
   - **Важность правила**: значение CCE — *Критическая ошибка*, *Важное уведомление* или *Предупреждение*.
   - **Состояние**: текущее состояние рекомендации:

     - **Открыто**. рекомендация еще не выполнена.
     - **Выполняется**: Это означает, что в настоящее время к ресурсам применяется рекомендация. От вас не требуется никаких действий.
     - **Разрешено**: рекомендация была применена. Когда проблема устранена, запись становится недоступной.

3. Для просмотра сведений о невыполненном правиле выберите его в списке.

   ![Подробное представление невыполненного правила конфигурации][3]

   В подробном представлении отображается следующая информация:

   - **Name**. имя правила.
   - **CCIED**: уникальный идентификатор CCE правила.
   - **Версия ОС**: версия ОС виртуальной машины или компьютера.
   - **Важность правила**: значение CCE — *Критическая ошибка*, *Важное уведомление* или *Предупреждение*.
   - **Полное описание**: описание правила.
   - **Уязвимость**: описание уязвимости или риска в случае, если правило не применяется.
   - **Потенциальное влияние**: влияние на работу организации в случае применения правила.
   - **Мера противодействия**: шаги по устранению проблемы.
   - **Ожидаемое значение**: значение, ожидаемое при анализе Центром безопасности конфигурации ОС виртуальной машины посредством правила.
   - **Фактическое значение**: значение, возвращаемое после анализа конфигурации ОС виртуальной машины посредством правила.
   - **Операция правила**: операция правила, используемая Центром безопасности во время анализа конфигурации ОС виртуальной машины посредством правила.

4. В верхней части окна подробного представления выберите **Поиск**.  
   В окне поиска откроется список рабочих областей, к которым относятся виртуальные машины и компьютеры с выбранным несоответствием конфигурации системы безопасности. Выбор рабочей области доступен, только если выбранное правило применяется к нескольким виртуальным машинам, подключенным к разным рабочим областям.

   ![Указанные рабочие области][4]

5. Выберите рабочую область.  
   Из Azure Monitor журналов поискового запроса откроется отфильтрованный по рабочей области с несоответствием конфигурации системы безопасности.

   ![Рабочая область с уязвимостью операционной системы][5]

6. Выберите компьютер из списка.  
   Откроется новый результат поиска с данными только для этого компьютера.

   ![Подробные сведения о выбранном компьютере][6]

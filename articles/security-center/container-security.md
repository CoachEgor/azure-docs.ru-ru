# <a name="container-security-in-security-center"></a>Безопасность контейнеров в Центре безопасности

Центр безопасности Azure — это решение, нативное для безопасности контейнеров, нативное для Azure. Центр безопасности также является оптимальным одним стеклом для обеспечения безопасности рабочих нагрузок, вс-вс, серверов и контейнеров.

В этой статье описывается, как Центр безопасности помогает вам улучшить, контролировать и поддерживать безопасность контейнеров и их приложений. Вы узнаете, как Центр безопасности помогает с этими основными аспектами безопасности контейнеров:

* управление уязвимостями;
* Упрочнение среды контейнера
* Защита продолжительности выполнения

[![Вкладка безопасности контейнеров Центра безопасности Azure](media/container-security/container-security-tab.png)](media/container-security/container-security-tab.png#lightbox)

Для получения инструкций о том, как использовать эти функции, [см. Мониторинг безопасности контейнеров.](monitor-container-security.md)

## <a name="vulnerability-management---scanning-container-images"></a>Управление уязвимостями - сканирование изображений контейнеров
Чтобы следить за реестром контейнеров Azure на основе ARM, убедитесь, что вы находитесь на стандартном уровне Центра безопасности (см. [цены).](/azure/security-center/security-center-pricing) Затем включите дополнительный пакет контейнерных реестров. При нажатии нового изображения Центр безопасности сканирует изображение с помощью сканера от ведущего в отрасли поставщика сканирования уязвимостей, квалис.

Когда проблемы будут найдены - по Квалис или Центр безопасности - вы получите уведомление в панели мониторинга Центра безопасности. Для каждой уязвимости Центр безопасности предоставляет практические рекомендации, а также классификацию серьезности и рекомендации по устранению проблемы. Подробнее о рекомендациях Центра безопасности для контейнеров можно узнать в [справочном списке рекомендаций.](recommendations-reference.md#recs-containers)

Центр безопасности фильтрует и классифицирует выводы сканера. Когда изображение является здоровым, Центр безопасности отмечает его как таковое. Центр безопасности генерирует рекомендации по безопасности только для изображений, которые должны быть решены. Уведомляя только о возникновении проблем, Центр безопасности снижает вероятность нежелательных информационных оповещений.

## <a name="environment-hardening"></a>Закаливание окружающей среды

### <a name="continuous-monitoring-of-your-docker-configuration"></a>Непрерывный мониторинг конфигурации Docker
Центр безопасности Azure определяет неуправляемые контейнеры, размещенные на IaaS Linux VMs, или другие linux-машины, работающие с контейнерами Docker. Центр безопасности постоянно оценивает конфигурацию этих контейнеров. Затем он сравнивает их с [Центром интернет-безопасности (CIS) Docker Benchmark](https://www.cisecurity.org/benchmark/docker/).

Центр безопасности включает в себя весь набор правил CIS Docker Benchmark и предупреждает вас, если ваши контейнеры не удовлетворяют ни один из элементов управления. Когда он находит неправильные настройки, Центр безопасности генерирует рекомендации по безопасности. Используйте **страницу рекомендаций** для просмотра рекомендаций и устранения проблем. Вы также увидите рекомендации на вкладке **Контейнеры,** которая отображает все виртуальные машины, развернутые с Docker. 

Для получения подробной информации о соответствующих рекомендациях [container section](recommendations-reference.md#recs-containers) Центра безопасности, которые могут появиться для этой функции, см.

Когда вы изучаете проблемы безопасности VM, Центр безопасности предоставляет дополнительную информацию о контейнерах на машине. Такая информация включает версию Docker и количество изображений, работающих на хосте. 

>[!NOTE]
> Эти контрольные проверки CIS не будут проводиться на экземплярах, управляемых AKS, или управляемых Databricks VMs.

### <a name="continuous-monitoring-of-your-kubernetes-clusters"></a>Непрерывный мониторинг кластеров Kubernetes
Центр безопасности работает совместно с Azure Kubernetes Service (AKS), службой управляемой обработки контейнеров корпорации Майкрософт для разработки, развертывания и управления контейнерными приложениями.

AKS обеспечивает контроль безопасности и видимость в позе безопасности кластеров. Центр безопасности использует эти функции для:
* Постоянно следите за конфигурацией кластеров AKS
* Создание рекомендаций по безопасности, приведенных в соответствии с отраслевыми стандартами

Для получения подробной информации о соответствующих рекомендациях [container section](recommendations-reference.md#recs-containers) Центра безопасности, которые могут появиться для этой функции, см.

## <a name="run-time-protection---real-time-threat-detection"></a>Защита от времени выполнения - обнаружение угроз в режиме реального времени

Центр безопасности обеспечивает обнаружение угроз в режиме реального времени для контейнерных сред и генерирует оповещения о подозрительных действиях. Эти сведения можно использовать для быстрого устранения проблем безопасности и повышения безопасности контейнеров.

Мы обнаруживаем угрозы на уровне кластера хоста и AKS. Для получения подробной информации смотрите [обнаружение угроз для контейнеров Azure.](threat-protection.md#azure-containers)


## <a name="container-security-faq"></a>Часто задаваемые вопросы безопасности контейнеров

### <a name="what-types-of-images-can-azure-security-center-scan"></a>Какие типы изображений можно сканировать в Центре безопасности Azure?
Центр безопасности сканирует изображения на основе Linux OS, которые обеспечивают доступ к оболочке. 

Сканер «Квалис» не поддерживает супер минималистские изображения, такие как [изображения царапин Docker](https://hub.docker.com/_/scratch/) или изображения «Distroless», которые содержат только ваше приложение и его зависимости от времени выполнения без менеджера пакетов, оболочки или ОС.

### <a name="how-does-azure-security-center-scan-an-image"></a>Как центр безопасности Azure сканирует изображение?
Изображение извлекается из реестра. Затем он работает в изолированной песочнице со сканером Квалис, который извлекает список известных уязвимостей.

Центр безопасности фильтрует и классифицирует выводы сканера. Когда изображение является здоровым, Центр безопасности отмечает его как таковое. Центр безопасности генерирует рекомендации по безопасности только для изображений, которые должны быть решены. Уведомляя только о возникновении проблем, Центр безопасности снижает вероятность нежелательных информационных оповещений.

### <a name="how-often-does-azure-security-center-scan-my-images"></a>Как часто Центр безопасности Azure сканирует мои изображения?
Сканирование изображений срабатывает при каждом нажатии.

### <a name="can-i-get-the-scan-results-via-rest-api"></a>Могу ли я получить результаты сканирования через REST API?
Да. Результаты находятся под [Суб-Оценки Остальные API](/rest/api/securitycenter/subassessments/list/). Кроме того, для всех ресурсов можно использовать Azure Resource Graph (ARG), aPI, похожий на Kusto: запрос может получить определенное сканирование.

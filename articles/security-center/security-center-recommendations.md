# <a name="security-recommendations-in-azure-security-center"></a>Рекомендации по безопасности в Центре безопасности Azure 
В этой теме объясняется, как просматривать и понимать рекомендации в Центре безопасности Azure, чтобы помочь вам защитить ресурсы Azure.

> [!NOTE]
> В документе приводится обзор службы с помощью примера развертывания.  Этот документ не является пошаговым руководством.
>

## <a name="what-are-security-recommendations"></a>Что такое рекомендации по безопасности?

Рекомендации – это действия, которые вы должны предпринять для того, чтобы защитить свои ресурсы.

Центр безопасности периодически анализирует состояние безопасности ресурсов Azure для выявления потенциальных уязвимостей безопасности. Затем он предоставляет вам рекомендации о том, как удалить их.

Каждая рекомендация предоставляет вам:

- Краткое описание того, что рекомендуется.
- Восстановительные работы по выполнению этой рекомендации. <!-- In some cases, Quick Fix remediation is available. -->
- Какие ресурсы нуждаются в вас выполнения рекомендуемых действий на них.
- **Безопасный результат воздействия**, который является сумма, что ваш безопасный счет будет идти вверх, если вы реализуете эту рекомендацию.

## <a name="monitor-recommendations"></a>Рекомендации по мониторингу<a name="monitor-recommendations"></a>

Центр безопасности анализирует состояние безопасности ваших ресурсов для выявления потенциальных уязвимостей. Плитка **Рекомендаций** в **обзоре** показывает общее количество рекомендаций, определенных Центром безопасности.

![Обзор центра безопасности](./media/security-center-recommendations/asc-overview.png)

1. Выберите **плитку Рекомендации** под **обзором.** Открывается список **Рекомендаций.**

      ![Просмотр рекомендаций](./media/security-center-recommendations/view-recommendations.png)

    Вы можете отфильтровать рекомендации. Для этого выберите **Фильтр** в колонке **Рекомендации**. Откроется колонка **Фильтр** , где можно выбрать значения серьезности и состояния для просмотра.

   * **РЕКОМЕНДАЦИИ**: Рекомендация.
   * **SECURE SCORE IMPACT**: Оценка, генерируемая Центром безопасности с использованием рекомендаций по безопасности и применение множещихся алгоритмов для определения того, насколько важна каждая рекомендация. Для получения дополнительной информации [см.](security-center-secure-score.md#secure-score-calculation)
   * **Ресурс**— список ресурсов, к которым применяется эта рекомендация.
   * **СТАТУС БАРС**: Описывает серьезность этой конкретной рекомендации:
       * **High (Red)**: Уязвимость существует с значимым ресурсом (например, приложением, VM или группой сетевой безопасности) и требует внимания.
       * **Средний (оранжевый)**: Уязвимость существует и некритические или дополнительные шаги, необходимые для ее устранения или завершения процесса.
       * **Низкий (синий)**: Существует уязвимость, которая должна быть устранена, но не требует немедленного внимания. (По умолчанию низкий уровень не представлен, однако вы можете выполнить фильтрацию по этому значению, чтобы просмотреть соответствующие рекомендации). 
       * **Здоровый (зеленый)**:
       * **Недоступно (серый)**:

1. Чтобы просмотреть данные каждой рекомендации, нажмите на рекомендацию.

    ![Сведения о рекомендации](./media/security-center-recommendations/recommendation-details.png)

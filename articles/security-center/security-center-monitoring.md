# <a name="strengthen-your-security-posture-with-azure-security-center"></a>Укрепление системы безопасности с помощью центра безопасности Azure
Эта статья поможет вам укрепить систему безопасности. В этой статье описываются возможности мониторинга, доступные в центре безопасности Azure, которые позволяют обеспечить максимальную безопасность ресурсов и отслеживать соответствие политикам.

## <a name="how-do-you-strengthen-your-security-posture"></a>Как укрепить систему безопасности?
Наблюдение часто определяется как отслеживание и ожидание возникновения события для реагирования на какую-либо ситуацию. Укрепление системы безопасности означает использование упреждающей стратегии для аудита ресурсов с целью выявления систем, которые не соответствуют стандартам и рекомендациям, принятым в организации.

Когда вы включаете [политики безопасности](tutorial-security-policy.md) для ресурсов в рамках подписки, центр безопасности проверяет состояние защиты ресурсов, чтобы определить потенциальные уязвимости. Хотя сведения о конфигурации сети становятся доступными сразу же, Сбор информации о конфигурации виртуальных машин и компьютеров, включая сведения о состоянии обновления системы безопасности и конфигурации операционной системы, может длиться час или дольше. Время выполнения операции зависит от числа виртуальных машин и компьютеров, установленных на агенте. Можно просмотреть полный список проблем и способы повышения безопасности сети и немедленного уменьшения риска в плитке **Рекомендации**.

Сведения о состоянии безопасности ваших ресурсов, а также возникшие проблемы можно просмотреть для отдельных типов ресурсов.

- Для наблюдения за работоспособностью ресурсов компьютера и приложений и получения рекомендаций по повышению безопасности см. раздел [Защита ваших машин и приложений в центре безопасности Azure](security-center-virtual-machine-protection.md).
- Чтобы наблюдать за сетевыми ресурсами, такими как виртуальные машины, группы безопасности сети и конечные точки, и получать рекомендации по повышению безопасности, см. раздел [Защита сети в центре безопасности Azure](security-center-network-recommendations.md). 
- Для мониторинга ресурсов данных и хранилища, например серверов SQL Server и учетных записей хранения, и получения рекомендаций по повышению безопасности см. раздел [Защита службы SQL Azure и данных в центре безопасности Azure](security-center-sql-service-recommendations.md). 
- Чтобы отслеживать ресурсы удостоверений и доступа, включая многофакторную проверку подлинности и разрешения учетных записей, и получить рекомендации по повышению безопасности, см. раздел [Мониторинг удостоверений и доступа в центре безопасности Azure](security-center-identity-access.md). 
- Чтобы отслеживать JIT-доступ к ресурсам, см. раздел [Управление доступом к виртуальным машинам с помощью JIT](security-center-just-in-time.md). 


Дополнительные сведения о том, как применить рекомендации, см. в разделе [Внедрение рекомендаций по безопасности](security-center-recommendations.md).

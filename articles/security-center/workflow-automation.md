# <a name="workflow-automation"></a>Автоматизация рабочих процессов

Каждая программа безопасности включает в себя несколько рабочих процессов для реагирования на инциденты. Эти процессы могут включать уведомление соответствующих заинтересованных сторон, запуск процесса управления изменениями и применение конкретных шагов по исправлению положения. Эксперты по безопасности рекомендуют автоматизировать как можно больше шагов этих процедур. Автоматизация уменьшает накладные расходы. Это также может повысить вашу безопасность, гарантируя, что шаги процесса выполняются быстро, последовательно и в соответствии с вашими предопределенными требованиями.

В этой статье описывается функция автоматизации рабочего процесса Центра безопасности Azure. Эта функция может вызвать логические приложения на оповещения и рекомендации безопасности. Например, при возникновении предупреждения центр безопасности может заставить по электронной почте по электронной почте конкретному пользователю. Вы также узнаете, как создавать логические приложения с помощью [приложений Логики Azure.](https://docs.microsoft.com/azure/logic-apps/logic-apps-overview)

> [!NOTE]
> Если ранее вы использовали представление Playbooks (Preview) на боковой панели, вы найдете те же функции вместе с расширенной функциональностью на новой странице автоматизации рабочего процесса.


## <a name="requirements"></a>Требования

* Для работы с рабочими процессами Azure Logic Apps необходимо иметь следующие роли/разрешения логических приложений:

    * Требуются разрешения [оператора Logic App](https://docs.microsoft.com/azure/role-based-access-control/built-in-roles#logic-app-operator) или доступ к logic App read/trigger (эта роль не может создавать или редактировать логические приложения; *запускать* только существующие)

    * [Разрешения logic App Contributor](https://docs.microsoft.com/azure/role-based-access-control/built-in-roles#logic-app-contributor) необходимы для создания и модификации Logic App

* Если вы хотите использовать разъемы Logic App, вам могут понадобиться дополнительные учетные данные для входной связи в соответствующие службы (например, в экземплярах Outlook/Teams/Slack)


## <a name="create-a-logic-app-and-define-when-it-should-automatically-run"></a>Создайте приложение Логика и определите, когда оно должно работать автоматически 

1. Из боковой панели Центра безопасности выберите **автоматизацию рабочего процесса.**

    [![Список автоматизации рабочих процессов](media/workflow-automation/list-of-workflow-automations.png)](media/workflow-automation/list-of-workflow-automations.png#lightbox)

    На этой странице можно создать новые правила автоматизации, а также включить, отключить или удалить существующие.  
1. Чтобы определить новый рабочий процесс, нажмите **Добавить автоматизации рабочего процесса.** 

    Появляется панель с опциями для новой автоматизации. Здесь вы можете ввести:
    1. Имя и описание автоматизации.
    1. Триггеры, которые инициируют этот автоматический рабочий процесс. Например, при срабатывании оповещения о безопасности, содержащем s'L, может потребоваться запуск приложения Logic App.
    1. Logic App, который будет работать, когда ваши условия триггера будут выполнены. 

        [![Список автоматизации рабочих процессов](media/workflow-automation/add-workflow.png)](media/workflow-automation/add-workflow.png#lightbox)

1. Из раздела «Действия» нажмите **«Создайте новый,** чтобы начать процесс создания logic App».

    Вы будете приняты к Azure Логические приложения.

    [![Создание нового приложения логики](media/workflow-automation/logic-apps-create-new.png)](media/workflow-automation/logic-apps-create-new.png#lightbox)

1. Введите имя, группу ресурсов и местоположение и нажмите **Создать.**

1. В новом приложении Logic Можно выбрать встроенные, предопределенные шаблоны из категории безопасности. Или вы можете определить пользовательский поток событий, который происходит, когда этот процесс срабатывает.

    В logic App дизайнер следующие триггеры из разъемов Центра безопасности поддерживаются:

    * **При создании или срабатывании рекомендации Центра безопасности Azure**
    * **При создании или срабатывании оповещения Центра безопасности Azure** 
    
    > [!TIP]
    > Вы можете настроить триггер так, чтобы он относиться только к предупреждениям с уровнями серьезности, которые вас интересуют.
    
    > [!NOTE]
    > Если вы используете устаревший триггер "При срабатывании ответа на оповещение Центра безопасности Azure", ваши приложения логики не будут запущены функцией автоматизации рабочего процесса. Вместо этого используйте любой из вышеуказанных триггеров. 

    [![Пример логики App](media/workflow-automation/sample-logic-app.png)](media/workflow-automation/sample-logic-app.png#lightbox)

1. После определения приложения Logic App вернитесь к панели определения автоматизации рабочего процесса ("Добавить автоматизацию рабочего процесса"). Нажмите **Refresh,** чтобы убедиться, что ваше новое приложение Logic доступно для выбора.

    ![Обновить](media/workflow-automation/refresh-the-list-of-logic-apps.png)

1. Выберите logic App и сохраните автоматизацию. Обратите внимание, что логическое отключение App показывает только логические приложения с поддержкой разъемов Центра безопасности, упомянутых выше.


## <a name="manually-trigger-a-logic-app"></a>Вручную вызвать logic App

Вы также можете запускать логические приложения вручную при просмотре оповещения о безопасности или любой рекомендации, которая предлагает [быстрое исправление исправление.](https://docs.microsoft.com/azure/security-center/security-center-remediate-recommendations#quick-fix-remediation)

Чтобы вручную запустить Logic App, откройте оповещение или рекомендацию, которая поддерживает быстрое исправление и нажмите **Trigger Logic App:**

[![Вручную вызвать logic App](media/workflow-automation/manually-trigger-logic-app.png)](media/workflow-automation/manually-trigger-logic-app.png#lightbox)

Чтобы просмотреть необработанные схемы событий событий событий, передаваемые в экземпляр Logic App, посетите [схемы автоматизации типов данных Workflow.](https://aka.ms/ASCAutomationSchemas) Это может быть полезно в тех случаях, когда вы не используете встроенные разъемы Logic App, упомянутые выше, но вместо этого используете общий разъем HTTP logic App - вы можете использовать схему JSON, чтобы вручную разобрать его по мере того, как вы считаете нужным.

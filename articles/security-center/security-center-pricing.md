# <a name="pricing-of-azure-security-center"></a>Цены на центр безопасности Azure
Центр безопасности Azure предоставляет возможности унифицированного управления безопасностью и расширенной защиты от угроз для рабочих нагрузок в Azure, в локальной среде и в других облаках. Он повышает вашу информированность о гибридных облачных рабочих нагрузках, обеспечивает управление ими, а также предоставляет активные средства защиты, снижающие уровень риска угроз, и интеллектуальные средства обнаружения, которые позволяют быть в курсе быстро развивающихся кибератак.


## <a name="free-option-vs-azure-defender-enabled"></a>Бесплатный вариант и защитник Azure включен

Центр безопасности предлагается в двух режимах:

- **Защитник Azure отключен** (бесплатный) — центр безопасности без защитника Azure включен бесплатно для всех подписок Azure при первом посещении панели мониторинга центра безопасности azure в портал Azure или при включении программными средствами через API. В этом режиме предоставляется политика безопасности, непрерывная оценка безопасности и рекомендации по обеспечению безопасности, помогающие защитить ресурсы Azure.

- Служба "Защитник **Azure"** , включенная в защитник Azure, расширяет возможности бесплатного режима для рабочих нагрузок в частных и других общедоступных облаках, предоставляя единую систему управления безопасностью и защиту от угроз в гибридных облачных рабочих нагрузках. Некоторые основные функции защитника Azure:

    - **Гибридная защита.** Получите комплексный обзор безопасности всех локальных и облачных рабочих нагрузок. Применяйте политики безопасности и непрерывно проверяйте уровень безопасности гибридных облачных рабочих нагрузок, чтобы обеспечить соответствие стандартам безопасности. Собирайте, ищите и анализируйте данные безопасности из нескольких источников, включая брандмауэры и другие партнерские решения.
    - **Оповещения защиты от угроз** — Улучшенная аналитика поведения и Microsoft Intelligent Security Graph обеспечивают ребро по мере развития кибератак атак. Определяйте атаки и уязвимости нулевого дня с помощью встроенного анализа поведения и машинного обучения. Обнаруживайте входящие атаки и отслеживайте дальнейшую работу систем за счет мониторинга сетей, компьютеров и облачных служб. Ускорьте анализ с помощью интерактивных инструментов и контекстной аналитики угроз.
    - **Сканирование уязвимостей для виртуальных машин и реестров контейнеров** . простое развертывание сканера на всех виртуальных машинах, предоставляющая наиболее продвинутое решение для управления уязвимостью в отрасли. Просматривайте, изучайте и исправьте результаты непосредственно в центре безопасности. 
    - **Доступ и элементы управления приложениями** — блокируйте вредоносные программы и другие нежелательные приложения, применяя рекомендации на основе машинного обучения, адаптированные к конкретным рабочим нагрузкам для создания списков разрешений и запретов. Уменьшите зону сетевой атаки, используя JIT-управляемый доступ к портам управления на виртуальных машинах Azure. Это радикально снижает вероятность перебора и других сетевых атак.
    - **Функции безопасности контейнеров** — преимущества управления уязвимостью и защиты от угроз в реальном времени в контейнерных средах. При включении **защитника Azure для реестров контейнеров**может потребоваться до 12hrs, пока не будут включены все компоненты. Расходы основываются на количестве уникальных образов контейнеров, отправленных в подключенный реестр. После того, как изображение было просмотрено, вы не будете списывать его повторно, пока он не будет изменен и отправлен еще раз. 

## <a name="try-azure-defender-free-for-30-days"></a>Бесплатное использование Azure Defender в течение 30 дней

Служба "защитник Azure" бесплатная в течение первых 30 дней. По истечении 30 дней, если вы решили продолжить использование службы, мы автоматически будем заставлять оплату за использование.

## <a name="enable-azure-defender"></a>Включение Azure Defender

Вы можете защитить всю подписку Azure с помощью защитника Azure, и защита будет наследоваться всеми ресурсами в подписке.

Чтобы включить защитник Azure, сделайте следующее:

1. В главном меню центра безопасности выберите **параметры ценообразования &**.
1. Выберите подписку, которую требуется обновить.
1. Выберите **защитник Azure на on** для обновления.
1. Щелкните **Сохранить**.

Ниже приведена страница с ценами на пример подписки. Вы заметите, что каждый план в защитнике Azure оплачивается отдельно и может быть установлен по отдельности.

:::image type="content" source="./media/security-center-pricing/pricing-tier-page.png" alt-text="Страница цен центра безопасности на портале":::

> [!NOTE]
> Чтобы включить все функции центра безопасности, включая возможности защиты от угроз, необходимо включить защитник Azure в подписке, содержащей применимые рабочие нагрузки. Включение на уровне рабочей области не включает JIT-доступ к виртуальным машинам, адаптивные элементы управления приложениями и обнаружение сети для ресурсов Azure. 
>
> Вы можете включить **защитник Azure для учетных записей хранения** на уровне подписки или на уровне ресурса.
> Вы можете включить **защитник Azure для SQL** на уровне подписки или ресурса.
> Для **Базы данных Azure для MariaDB, MySQL и PostgreSQL** защиту от угроз можно включить только на уровне ресурсов.

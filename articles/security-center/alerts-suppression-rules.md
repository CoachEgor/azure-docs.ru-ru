# <a name="suppressing-alerts-from-azure-security-centers-threat-protection"></a>Подавление оповещений от защиты от угроз в центре безопасности Azure

На этой странице объясняется, как можно использовать правила подавления оповещений для подавления ложных срабатываний или других нежелательных оповещений системы безопасности в центре безопасности Azure.

## <a name="availability"></a>Доступность

- Состояние выпуска: **Предварительная версия**
- Обязательные роли: администратор безопасности и владелец могут создавать и удалять правила. Средства чтения и чтения безопасности могут просматривать правила.
- Облака: все (глобальные, Национальный, государственные и независимых)


## <a name="introduction-to-suppression-rules"></a>Общие сведения о правилах подавления

Компоненты защиты от угроз в центре безопасности Azure обнаруживают угрозы в любой области среды и создают оповещения системы безопасности.

Если одно оповещение не является интересным или релевантным, его можно закрыть вручную. Кроме того, можно использовать функцию правила подавления для автоматического закрытия подобных предупреждений в будущем. Как правило, правило подавления используется в следующих случаях:

- подавлять оповещения, которые были идентифицированы как ложные срабатывания

- подавить оповещения, которые вызываются слишком часто, чтобы быть полезными

Правила подавления определяют условия, по которым оповещения должны автоматически открываться.

> [!CAUTION]
> Подавление оповещений системы безопасности сокращает защиту от угроз в центре безопасности. Следует тщательно проверить возможное воздействие любого правила подавления и отслеживать его с течением времени.

[![Страница оповещений системы безопасности центра безопасности Azure с параметрами подавления оповещений](media/alerts-suppression-rules/alerts-screen-with-options.png)](media/alerts-suppression-rules/alerts-screen-with-options.png#lightbox)

## <a name="creating-a-suppression-rule"></a>Создание правила подавления

Существует несколько способов создания правил для подавления нежелательных оповещений системы безопасности.

- Чтобы отключить оповещения на уровне группы управления, используйте политику Azure.

- Чтобы отключить оповещения на уровне подписки, можно использовать портал Azure или REST API, как описано ниже.

Правила подавления могут отклонять только те оповещения, которые уже активированы в выбранных подписках.

Чтобы создать правило непосредственно в портал Azure:

1. На странице оповещений системы безопасности центра безопасности:

    - Выберите нужное оповещение, которое больше не нужно видеть, и в меню с многоточием (...) для оповещения щелкните **создать правило подавления**:

        [![* * Параметр "создать правило подавления" * *](media/alerts-suppression-rules/auto-dismiss-future-option.png)](media/alerts-suppression-rules/auto-dismiss-future-option.png#lightbox)

    - Или щелкните ссылку **правила подавления** в верхней части страницы, а на странице правила подавления выберите **создать новое правило подавления**:

        ![Кнопка "создать новое правило подавления" * *](media/alerts-suppression-rules/create-new-suppression-rule.png)

1. На панели новое правило подавления введите сведения о новом правиле.

    - Правило может отклонить оповещение для **всех ресурсов** , чтобы вы не могли получать такие предупреждения в будущем. 
    
    - Правило может отклонить оповещение **по конкретным критериям** — если оно связано с конкретным IP-адресом, именем процесса, учетной записью пользователя, ресурсом Azure или расположением.

    > [!TIP]
    > Если вы открыли новую страницу правила из определенного предупреждения, оповещение и подписка будут автоматически настроены в новом правиле. Если вы использовали ссылку **создать правило подавления** , выбранные подписки будут соответствовать текущему фильтру на портале.

    [![Панель создания правил подавления](media/alerts-suppression-rules/new-suppression-rule-pane.png)](media/alerts-suppression-rules/new-suppression-rule-pane.png#lightbox)

1. Введите сведения о правиле:

    - **Имя** — имя правила. Имена правил должны начинаться с буквы или цифры, иметь длину от 2 до 50 символов и не должны содержать символы, отличные от дефиса (-) или знаков подчеркивания (_). 
    - **Состояние** — включено или отключено.
    - **Причина** . Выберите одну из встроенных причин или "другое", если они не соответствуют вашим потребностям.
    - **Дата окончания срока действия** — конечная дата и время для правила. Правила могут выполняться в течение шести месяцев.

1. При необходимости протестируйте правило с помощью кнопки **имитировать** , чтобы узнать, сколько предупреждений было отклонено, если это правило было активно.

1. Сохраните правило. 

## <a name="editing-suppression-rules"></a>Изменение правил подавления

Чтобы изменить созданные правила, используйте страницу правила подавления.

1. На странице оповещения системы безопасности центра безопасности выберите ссылку **правила подавления** в верхней части страницы.

1. Откроется страница правила подавления, в которой перечислены все доступные правила в соответствии с выбранными подписками. 

    [![Список правил подавления](media/alerts-suppression-rules/suppression-rules-page.png)](media/alerts-suppression-rules/suppression-rules-page.png#lightbox)

1. Чтобы изменить одно правило, откройте меню с многоточием (...) для правила и выберите **изменить**.

1. Внесите необходимые изменения и нажмите кнопку **Применить**. 

## <a name="deleting-suppression-rules"></a>Удаление правил подавления

Чтобы удалить одно или несколько созданных правил, используйте страницу правила подавления.

1. На странице оповещения системы безопасности центра безопасности выберите ссылку **правила подавления** в верхней части страницы.

1. Откроется страница правила подавления, в которой перечислены все доступные правила в соответствии с выбранными подписками. 

1. Чтобы удалить одно правило, откройте меню с многоточием (...) для правила и выберите **Удалить**.

1. Чтобы удалить несколько правил, установите флажки для удаляемых правил и нажмите кнопку **Удалить**.

    ![Удаление одного или нескольких правил подавления](media/alerts-suppression-rules/delete-multiple-alerts.png)

## <a name="viewing-alerts-that-have-been-suppressed"></a>Просмотр подавленных предупреждений

Предупреждения, которые соответствуют включенным правилам подавления, по-прежнему будут создаваться, но их состояние будет установлено на **отклонено**. Вы можете увидеть состояние в портал Azure или при доступе к оповещениям безопасности центра безопасности. 

> [!TIP]
> [Метку Azure](https://azure.microsoft.com/services/azure-sentinel/) не создают инциденты для подавленных оповещений. Для других решения Siem можно отфильтровать подавленные предупреждения, используя состояние предупреждения ("закрыто").

Используйте фильтр центра безопасности для просмотра предупреждений, которые были отклонены правилами.

* На странице оповещения системы безопасности центра безопасности откройте параметры фильтра и выберите **отклоненные**.  

   [![Просмотр отклоненных предупреждений](media/alerts-suppression-rules/view-dismissed-alerts.png)](media/alerts-suppression-rules/view-dismissed-alerts.png#lightbox)


## <a name="using-the-api-to-create-and-manage-suppression-rules"></a>Использование API для создания правил подавления и управления ими

Вы можете создавать, просматривать или удалять правила подавления оповещений с помощью REST API центра безопасности. 

Соответствующие методы HTTP для правил подавления в REST API:

- **Размещение**: для создания или обновления правила подавления в указанной подписке.

- **Получить**:

    - Перечисление всех правил, настроенных для указанной подписки. Этот метод возвращает массив применимых правил.

    - Получение сведений о конкретном правиле для указанной подписки. Этот метод возвращает одно правило подавления.

    - Для моделирования влияния правила подавления на этапе проектирования. Этот вызов определяет, какие из существующих предупреждений были бы отклонены, если правило было активно.

- **Удалить**: удаляет существующее правило (но не изменяет состояние предупреждений, которые уже были отклонены).

Полные сведения и примеры использования см. в [документации по API](https://docs.microsoft.com/api/securitycenter/).

# <a name="integrate-azure-security-center-with-windows-admin-center"></a>Интеграция Центра безопасности Azure с админ-центром Windows

Admin Center Windows — это инструмент управления для серверов Windows. Это единое место для системных администраторов для доступа к большинству наиболее часто используемых инструментов администратора. Из центра windows Admin Center вы можете непосредственно находиться на борту серверов в Центре безопасности Azure. Затем можно просмотреть резюме рекомендаций по безопасности и оповещения непосредственно в интерфейсе Центра админ-центра Windows.

> [!NOTE]
> Подписка Azure и связанная с ней рабочая область Log Analytics должны включить стандартный уровень Центра безопасности, чтобы обеспечить интеграцию в Центр админ-аналитики Windows.
> Стандартный уровень является бесплатным в течение первых 30 дней, если вы ранее не использовали его в подписке и рабочем пространстве. Для получения дополнительной информации смотрите [страницу информации о ценах](security-center-pricing.md).
>

Если вы успешно на борту сервера от Центра администратора Windows до Центра безопасности Azure, вы можете:

* Просмотр предупреждений и рекомендаций безопасности внутри расширения Центра безопасности в Центре админ-центра Windows
* Просмотр позы безопасности и получение дополнительной подробной информации о управляемых серверах Windows Admin Center в Центре безопасности на портале Azure (или через API)

Объединив эти два инструмента, Центр безопасности становится вашим единым стеклом для просмотра всей вашей информации о безопасности, независимо от ресурса: защита вашего сервера Windows Admin Center, управляемых на прем-серверах, ваших вс-мисах и любых дополнительных рабочих нагрузок PaaS.

## <a name="onboarding-windows-admin-center-managed-servers-into-security-center"></a>На борту Windows Админ Центр управляемых серверов в Центр безопасности

1. Из центра windows Admin Center выберите один из серверов, а в панели **инструментов** — расширение Azure Security Center:

    ![Расширение Центра безопасности Azure в Центре Админ Windows](./media/windows-admin-center-integration/onboarding-from-wac.png)

    > [!NOTE]
    > Если сервер уже доставлен в Центр безопасности, окно настройки не отображается.

1. Нажмите **войти в Azure и настроить .**
    ![Расширение центра админ-центра Windows в Центр безопасности Azure](./media/windows-admin-center-integration/onboarding-from-wac-welcome.png)

1. Следуйте инструкциям по подключению сервера к Центру безопасности. После того, как вы ввели необходимые детали и подтвердили, Центр безопасности вносит необходимые изменения конфигурации, чтобы убедиться, что все следующие верны:
    * Зарегистрирован шлюз Azure.
    * Сервер имеет рабочее пространство для отчета и связанную подписку.
    * Стандартное решение Центра безопасности по анализу журналов включено в рабочее пространство. Это решение предоставляет функции стандартного уровня Центра безопасности для *всех* серверов и виртуальных машин, сообщающих об этом рабочем пространстве.
    * Стандартные цены уровня центра безопасности для виртуальной машины включены по подписке.
    * Агент мониторинга Майкрософт (MMA) устанавливается на сервер и настраивается для отчета в выбранное рабочее пространство. Если сервер уже отчитывается перед другим рабочим пространством, он настроен для отчета и в невыбранное рабочее пространство.

    > [!NOTE]
    > Это может занять некоторое время после посадки на борт для рекомендаций, чтобы появиться. На самом деле, в зависимости от активности сервера вы можете не получать *никаких* оповещений. Для создания тестовых оповещений для проверки оповещений работайте правильно, следуйте инструкциям в [процедуре проверки оповещения.](security-center-alert-validation.md)


## <a name="viewing-security-recommendations-and-alerts-in-windows-admin-center"></a>Просмотр рекомендаций по безопасности и предупреждений в Центре админ-миссии Windows

После того, как вы были на борту, вы можете просматривать оповещения и рекомендации непосредственно в зоне Azure Security Center ВАшего административного центра Windows. Нажмите рекомендацию или оповещение, чтобы просмотреть их на портале Azure. Там вы получите дополнительную информацию и узнаете, как устранять проблемы.

[![Рекомендации и оповещения Центра безопасности, как видно из Центра админов Windows](media/windows-admin-center-integration/asc-recommendations-and-alerts-in-wac.png)](media/windows-admin-center-integration/asc-recommendations-and-alerts-in-wac.png#lightbox)

## <a name="viewing-security-recommendations-and-alerts-for-windows-admin-center-managed-servers-in-security-center"></a>Просмотр рекомендаций по безопасности и предупреждений для серверов, управляемых Центром управления админ-центром Windows в Центре безопасности
Из Центра безопасности Azure:

* Чтобы просмотреть рекомендации по безопасности для всех серверов Windows Admin Center, откройте **вычислительные & приложения** и нажмите на вкладку **VMs и Computers.**

    [![Просмотр рекомендаций по безопасности для управляемых серверов Windows Admin Center](media/windows-admin-center-integration/viewing-recommendations-wac.png)](media/windows-admin-center-integration/viewing-recommendations-wac.png#lightbox)

* Для просмотра оповещений о безопасности для всех серверов Windows Admin Center откройте **оповещения о безопасности.** Нажмите **фильтр** и убедитесь, что **выбирается только** "Non-Azure":

    ![Предупреждения о безопасности фильтра для управляемых серверов Windows Admin Center](./media/windows-admin-center-integration/filtering-alerts-to-non-azure.png)

    [![Просмотр предупреждений безопасности для управляемых серверов Windows Admin Center](media/windows-admin-center-integration/viewing-alerts-wac.png)](media/windows-admin-center-integration/viewing-alerts-wac.png#lightbox)

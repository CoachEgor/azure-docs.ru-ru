# <a name="advanced-data-security-for-sql-servers-on-azure-virtual-machines-preview"></a>Расширенная безопасность данных для серверов СЗЛ на виртуальных машинах Azure (Предварительный просмотр)
Расширенная безопасность данных для серверов S'L на виртуальных машинах Azure — это единый пакет для передовых возможностей безопасности S'L. Эта функция предварительного просмотра включает в себя функциональность для выявления и уменьшения потенциальных уязвимостей базы данных и обнаружения аномальных действий, которые могут указывать на угрозы для вашей базы данных. 

Это предложение безопасности для серверов Azure VMs S'L основано на той же фундаментальной технологии, которая используется в [пакете расширенной безопасности данных Лазурного средства s'L.](https://docs.microsoft.com/azure/sql-database/sql-database-advanced-data-security)


## <a name="overview"></a>Обзор

Расширенная безопасность данных обеспечивает набор передовых возможностей безопасности СЗЛ, состоящий из оценки уязвимости и расширенной защиты от угроз.

* [Оценка уязвимости](https://docs.microsoft.com/azure/sql-database/sql-vulnerability-assessment) — это простая для настройки служба, которая может обнаружить, отслеживать и помогать вам устранять потенциальные уязвимости базы данных. Он обеспечивает видимость в вашем состоянии безопасности, и включает в себя шаги для решения проблем безопасности и улучшения укреплений базы данных.
* [Advanced Threat Protection](https://docs.microsoft.com/azure/sql-database/sql-database-threat-detection-overview) обнаруживает аномальные действия, указывающие на необычные и потенциально опасные попытки получить доступ к серверу или использовать его. Он постоянно отслеживает вашу базу данных для подозрительных действий и предоставляет ориентированные на действия оповещения безопасности на аномальных шаблонах доступа к базе данных. Эти предупреждения предоставляют подозрительные сведения о деятельности и рекомендуемые действия по расследованию и смягчению угрозы.

## <a name="get-started-with-advanced-data-security-for-sql-on-azure-vms"></a>Начатые с расширенной безопасности данных для S'L на Azure VMs

Следующие шаги позволяют начать работу с расширенной безопасности данных для S'L на открытом предварительном просмотре пазов Azure VMs.

### <a name="set-up-advanced-data-security-for-sql-on-azure-vms"></a>Настройка расширенной безопасности данных для S'L на MMs Azure

Включите расширенную безопасность данных для серверов S'L на виртуальных машинах на уровне подписки/рабочего пространства:
 
1. Со боковой панели Центра безопасности откройте страницу **настройки цен &.**

1. Выберите подписку или рабочее пространство, для которого вы хотите включить расширенную безопасность данных для S'L на Azure VMs.

1. Переключить опцию для **серверов S'L на VM (Preview)** для включения. 

    (Нажмите на скриншот, чтобы расширить)

    [![Рекомендации и оповещения Центра безопасности, как видно из Центра админов Windows](media/security-center-advanced-iaas-data/sql-servers-on-vms-in-pricing-small.png)](media/security-center-advanced-iaas-data/sql-servers-on-vms-in-pricing-large.png#lightbox)

    Расширенная безопасность данных для серверов S'L будет включена на всех серверах S'L, подключенных к выбранной рабочей области или рабочей области по умолчанию выбранной подписки.

    >[!NOTE]
    > Решение будет полностью активным после первого перезапуска сервера S'L. 

Чтобы создать новое рабочее пространство, следуйте инструкциям в [создании рабочего пространства log Analytics.](https://docs.microsoft.com/azure/azure-monitor/learn/quick-create-workspace)

Чтобы подключить хост сервера S'L к рабочему пространству, следуйте инструкциям в [Подключении компьютеров Windows к Azure Monitor.](https://docs.microsoft.com/azure/azure-monitor/platform/agent-windows)


## <a name="set-up-email-notification-for-security-alerts"></a>Настройка уведомления электронной почты для предупреждений безопасности 

Вы можете установить список получателей для получения уведомления по электронной почте при создании предупреждений Центра безопасности. Электронная почта содержит прямую ссылку на оповещение в Центре безопасности Azure со всеми соответствующими деталями. 

1. Перейдите в **Центр** > **безопасности Ценообразование & настройки** и нажмите на соответствующую подписку

    ![Параметры подписки](./media/security-center-advanced-iaas-data/subscription-settings.png)

1. Из меню «Настройки» нажмите **уведомления по электронной почте.** 
1. В текстовом окне **адреса электронной почты** введите адреса электронной почты для получения уведомлений. Вы можете ввести более одного адреса электронной почты, разделив адреса электронной почты с запятой (,.  admin1@mycompany.comНапример,admin2@mycompany.com,admin3@mycompany.com

    ![Параметры электронной почты](./media/security-center-advanced-iaas-data/email-settings.png)

1. В настройках **уведомления по электронной почте** установите следующие параметры:
  
    * **Отправить уведомление по электронной почте для предупреждений высокой степени тяжести:** Вместо отправки писем для всех предупреждений, отправить только для высокой серьезности оповещения.
    * **Также отправьте уведомления по электронной почте владельцам подписок:** Отправить уведомления владельцам подписок тоже.

1. В верхней части экрана **уведомлений по электронной почте** нажмите **Сохранить**.

  > [!NOTE]
  > Не забудьте нажать **Сохранить** перед закрытием окна, или новые настройки **уведомления электронной почты** не будут сохранены.

## <a name="advanced-threat-protection-for-sql-servers-on-azure-vms-alerts"></a>Расширенная защита угроз для серверов S'L на оповещениях Пазур VMs
Оповещения генерируются необычными и потенциально опасными попытками получить доступ к серверам S'L или использовать их. Эти события могут активировать создание следующих оповещений.

### <a name="anomalous-access-pattern-alerts-preview"></a>Аномальные оповещения шаблона доступа (Предварительный просмотр)

* **Доступ из необычного места:** Это оповещение срабатывает при изменении шаблона доступа к серверу S'L, где кто-то вошел в систему на сервер S'L из необычного географического положения. Возможные причины:
    * Злоумышленник или бывший злонамеренный сотрудник компании получил доступ к вашему серверу S'L Server.
    * Законный пользователь получил доступ к вашему серверу S'L с нового места.
* **Доступ из потенциально опасного приложения.** Это предупреждение активируется, когда доступ к базе данных получают потенциально опасные приложения. Возможные причины:
    * Злоумышленник, пытающийся взломать ваш S'L, используя общие инструменты атаки.
    * Законное тестирование на проникновение в действии.
* **Доступ из незнакомого субъекта.** Это предупреждение активируется при изменении шаблона доступа к серверу SQL, если пользователь вошел на сервер SQL с помощью нестандартного субъекта (пользователя SQL). Возможные причины:
    * Злоумышленник или бывший злонамеренный сотрудник компании получил доступ к вашему серверу S'L Server. 
    * Законный пользователь получил доступ к вашему серверу S'L с новым принципом.
* **Подбор учетных данных SQL.** Это предупреждение активируется при большом количестве неудачных попыток входа с разными учетными данными. Возможные причины:
    * Злоумышленник, пытающийся взломать ваш S'L с помощью грубой силы.
    * Законное тестирование на проникновение в действии.

### <a name="potential-sql-injection-attacks-supported-in-sql-server-2019"></a>Потенциальные атаки на впрыски СЗЛ (Поддерживается в сервере S'L 2019)

* **Уязвимость к впрыску S'L:** Это оповещение срабатывает, когда приложение генерирует неисправное заявление s'L в базе данных. Оно может указывать на возможную уязвимость к атакам путем внедрения кода SQL. Возможные причины:
    * Дефект в коде приложения, который создает инструкцию SQL с ошибкой.
    * Код приложения или хранимые процедуры не очищают ввод данных пользователя при создании ошибочной инструкции SQL, которая может быть использована для внедрения кода SQL.
* **Потенциальная атака путем внедрения кода SQL.** Это предупреждение запускается при активном эксплойте идентифицированной уязвимости приложения для внедрения кода SQL. Это означает, что злоумышленник пытается внедрить вредоносные инструкции SQL с помощью кода уязвимого приложения или хранимых процедур.


### <a name="unsafe-command-supported-in-sql-server-2019"></a>Небезопасная команда (Поддерживается в сервере S'L 2019)

* **Потенциально небезопасные действия**: Это предупреждение срабатывает при выполнении высокопривилегированной и потенциально небезопасной команды. Возможные причины:
    * Команда, которая рекомендовала быть отключена для улучшения положения безопасности включена.
    * Злоумышленник пытается использовать доступ к S'L или обострять привилегии.   


## <a name="explore-and-investigate-security-alerts"></a>Исследуйте и исследуйте оповещения о безопасности

Оповещения о безопасности данных доступны на странице оповещения Центра безопасности, вкладке безопасности ресурса или через прямую ссылку в письмах с оповещением.

1. Чтобы просмотреть оповещения, выполните указанные ниже действия.

    * В Центре безопасности - Нажмите **оповещения безопасности** с боковой панели и выберите оповещение.
    * В области ресурса - откройте соответствующую страницу ресурса, а из боковой панели нажмите **Security.** 

1. Оповещения предназначены для автономных, с подробными мерами по исправлению положения и информацией о расследовании в каждом из них. Вы можете продолжить исследование, используя другие возможности Azure Security Center и Azure Sentinel для более широкого представления:

    * Включите функцию аудита сервера S'L Server для дальнейших исследований. Если вы являетесь пользователем Azure Sentinel, вы можете загрузить журналы аудита с S'L из событий журнала безопасности Windows в Sentinel и наслаждаться богатым опытом исследования.
    * Чтобы улучшить свою безопасность, используйте рекомендации Центра безопасности для машины-хозяина, указанные в каждом предупреждении. Это снизит риски будущих атак.

# <a name="protect-your-azure-app-service-web-apps-and-apis"></a>Защита веб-приложений и API службы приложений Azure

Служба приложений Azure — это полностью управляемая платформа для создания и размещения веб-приложений и интерфейсов API, не беспокоясь о необходимости управлять инфраструктурой. Она обеспечивает управление, мониторинг и оперативную аналитику для удовлетворения требований к производительности, безопасности и соответствию корпоративного уровня. Дополнительные сведения см. в статье [служба приложений Azure](https://azure.microsoft.com/services/app-service/).

Чтобы включить расширенную защиту от угроз для плана службы приложений Azure, необходимо выполнить следующие действия.

* Подпишитесь на ценовую категорию центра безопасности Azure "Стандартный"
* Включите план службы приложений, как показано ниже. Центр безопасности изначально интегрирован со службой приложений, устраняя необходимость в развертывании и адаптации — интеграция прозрачна.
* План службы приложений связан с выделенными компьютерами. Поддерживаемые планы: "базовый", "Стандартный", "Премиум", "изолированный" или Linux. Центр безопасности не поддерживает планы бесплатных, общих ресурсов или потребления. Дополнительные сведения см. в статье [Планы службы приложений](https://azure.microsoft.com/pricing/details/app-service/plans/).

Если план службы приложений включен, центр безопасности оценивает ресурсы, охваченные планом службы приложений, и создает рекомендации по обеспечению безопасности в зависимости от полученных результатов. Центр безопасности защищает экземпляр виртуальной машины, в которой выполняется служба приложений, и интерфейс управления. Он также отслеживает запросы и ответы, отправляемые и принимаемые приложением, работающим в службе приложений.

Центр безопасности использует масштаб облака, а также видимость Azure в качестве поставщика облачных служб для отслеживания распространенных атак веб-приложений. Центр безопасности может обнаруживать атаки на ваши приложения и выявлять новые атаки, даже если злоумышленники находятся на этапе разведывательная атака, проверяя, чтобы выявить уязвимости в нескольких приложениях, размещенных в Azure. Как служба Azure Native, центр безопасности также находится в уникальной должности, чтобы обеспечить аналитику безопасности на основе узла, охватывающую базовые расчетные узлы для этой PaaS, что позволяет центру безопасности обнаруживать атаки на веб-приложения, которые уже использовались злоумышленниками. Дополнительные сведения см. в статье [Защита от угроз для службы приложений Azure](threat-protection.md#app-services).


## <a name="enabling-monitoring-and-protection-of-app-service"></a>Включение мониторинга и защиты службы приложений

1. В портал Azure выберите Центр безопасности.
2. Перейдите к разделу **цены & параметры** и выберите подписку.
3. В разделе **Ценовая категория** строки **Служба приложений** переведите план в состояние **Включено**.

    [![включение служб приложений в подписке уровня "Стандартный"](media/security-center-app-services/app-services-toggle.png)](media/security-center-app-services/app-services-toggle.png#lightbox)


>[!NOTE]
> Число экземпляров, перечисленных для **количества ресурсов** , представляет общее число экземпляров вычислений во всех планах службы приложений в этой подписке, которые выполняются в момент открытия колонки ценовая категория.
>
> Служба приложений Azure предлагает разнообразные планы. План службы приложений определяет набор ресурсов вычислений для запуска веб-приложения. Они эквивалентны фермам серверов в обычном веб-размещении. Вы можете настроить одно или несколько приложений для выполнения на одних вычислительных ресурсах (или в одном плане службы приложений).
>
>Чтобы проверить число планов службы приложений на портале Azure, перейдите на эту странице, где можно просмотреть число экземпляров вычислений, используемых каждым планом. 






Чтобы отключить мониторинг и рекомендации для службы приложений, повторите этот процесс и переключите план **службы приложений** в значение **Отключено**.

# <a name="protect-your-azure-app-service-web-apps-and-apis"></a>Защитите веб-приложения и AI службы app Azure

Azure App Service — это полностью управляемая платформа для создания и хостинга веб-приложений и AI, не беспокоясь о необходимости управлять инфраструктурой. Она обеспечивает управление, мониторинг и оперативную информацию в соответствии с требованиями производительности, безопасности и соответствия требованиям корпоративного уровня. Для получения дополнительной [Azure App Service](https://azure.microsoft.com/services/app-service/)информации см.

Для обеспечения расширенной защиты от угроз для вашего плана службы приложений Azure необходимо:

* Подпишитесь на стандартный ценовой уровень Центра безопасности Azure
* Включить план службы приложений, как показано ниже. Центр безопасности интегрируется с Службой безопасности, устраняя необходимость развертывания и посадки - интеграция прозрачна.
* Исесть план службы приложений, связанный с выделенными машинами. Поддерживаемые планы: Базовый, Стандартный, Премиум, Изолированный или Linux. Центр безопасности не поддерживает планы Free, Shared или Consumption. Для получения дополнительной [App Service Plans](https://azure.microsoft.com/pricing/details/app-service/plans/)информации см.

Благодаря включению плана службы приложений Центр безопасности оценивает ресурсы, охватываемые планом службы приложений, и генерирует рекомендации по безопасности на основе его выводов. Центр безопасности защищает экземпляр VM, в котором работает служба app, и интерфейс управления. Он также отслеживает запросы и ответы, отправляемые и принимаемые приложением, работающим в службе приложений.

Центр безопасности использует масштаб облака и видимость, которую Azure имеет в качестве поставщика облачных услуг для мониторинга общих атак веб-приложений. Центр безопасности может обнаруживать атаки на приложения и выявлять возникающие атаки - даже в то время как злоумышленники находятся в стадии разведки, сканируя для выявления уязвимостей в нескольких приложениях, размещенных в Azure. Как служба, основанная на Azure, Центр безопасности также имеет уникальную возможность предложить аналитику безопасности на базе узлов, охватывающих базовые вычислительные узлы для этого PaaS, что позволяет Центру безопасности обнаруживать атаки на веб-приложения, которые уже были использованы. Для получения более подробной информации [см.](threat-protection.md#app-services)


## <a name="enabling-monitoring-and-protection-of-app-service"></a>Включение мониторинга и защиты службы приложений

1. На портале Azure выберите Центр безопасности.
2. Перейдите к **настройкам ценообразования &** и выберите подписку.
3. В разделе **Ценовая категория** строки **Служба приложений** переведите план в состояние **Включено**.

    [![Включение служб приложений в подписку стандартного уровня](media/security-center-app-services/app-services-toggle.png)](media/security-center-app-services/app-services-toggle.png#lightbox)


>[!NOTE]
> Количество экземпляров, перечисленных для **вашего количества ресурсов,** представляет общее количество вычисляемых экземпляров во всех планах Службы App по этой подписке, запущенной в момент открытия лезвия уровня ценообразования.
>
> Служба приложений Azure предлагает различные планы. План Службы приложений определяет набор вычислительных ресурсов для запуска веб-приложения. Они эквивалентны серверным фермам в обычном веб-хостинге. Вы можете настроить одно или несколько приложений для выполнения на одних вычислительных ресурсах (или в одном плане службы приложений).
>
>Чтобы проверить подсчет, отправляйтесь в "Планы службы приложений" на портале Azure, где можно увидеть количество вычисляемых экземпляров, используемых каждым планом. 






Чтобы отключить мониторинг и рекомендации для службы приложений, повторите этот процесс и переключите план **службы приложений** в значение **Отключено**.

# <a name="security-alerts-in-azure-security-center"></a>Оповещения системы безопасности в центре безопасности Azure

В этой статье представлены различные типы оповещений системы безопасности в центре безопасности Azure (ASC). Существуют разнообразные оповещения для многих разных типов ресурсов. ASC создает оповещения для обоих ресурсов, развернутых в Azure, а ресурсы развернуты на локальных и гибридных облачных средах. 

## <a name="what-are-security-alerts"></a>Что такое оповещения системы безопасности?

Оповещения — это уведомления, создаваемые центром безопасности при обнаружении угрозы для ресурсов. Он устанавливает приоритеты и список предупреждений, а также сведения, необходимые для быстрого анализа проблемы. Центр безопасности также приведены рекомендации по как можно устранить атаки.

## <a name="how-does-security-center-detect-threats"></a>Как центр безопасности обнаруживает угрозы?

Для выявления реальных угроз и сокращения ложных срабатываний, Центр безопасности собирает, анализирует и объединяет данные журналов из ресурсов Azure, сети и подключенных партнерских решений, таких как брандмауэр и решения для защиты конечной точки. Центр безопасности анализирует эту информацию, часто сравнивая сведения из нескольких источников и определяя угрозы.

ASC отслеживает ресурсы развернутых в Azure и на других локальных и гибридных облачных средах. Дополнительные сведения об определении и реагирование на угрозы, см. в разделе [как центр безопасности обнаруживает угрозы и реагирует на](security-center-detection-capabilities.md#asc-detects).

## <a name="what-are-alert-incidents"></a>Что такое оповещения инцидентов

Инцидент безопасности — коллекция связанных оповещений, а не перечисляют каждого предупреждения по отдельности. Центр безопасности использует fusion, могут быть связаны различные предупреждения и сигналов низкого качества инцидентов безопасности.

Использование инцидентов, Центр безопасности предоставляет с общую картину кампании атак и все соответствующие оповещения. Это представление позволяет быстро понять, какие действия выполнил злоумышленник и какие ресурсы затронуты. Дополнительные сведения см. в разделе [смарт-корреляции предупреждений Cloud](security-center-alerts-cloud-smart.md).

## <a name="upgrade-to-standard-for-advanced-detections"></a>Обновление до Standard на расширенное обнаружение

Чтобы настроить расширенное обнаружение, обновите центр безопасности Azure до уровня "Стандартный". 

1. В меню центра безопасности выберите **политики безопасности**.
2. Для подписок, вы хотите переместить в уровень "стандартный", выберите **изменение параметров**. 
3. На странице "Параметры" выберите **Ценовая**. 
   Бесплатная пробная версия доступна в течение месяца. Дополнительные сведения см. Дополнительные [странице с ценами](https://azure.microsoft.com/pricing/details/security-center/).

# <a name="security-alerts-in-azure-security-center"></a>Оповещения системы безопасности в центре безопасности Azure

В этой статье представлены различные типы оповещений системы безопасности, доступных в центре безопасности Azure (ASC). Существует множество оповещений для различных типов ресурсов. ASC создает оповещения для ресурсов, развернутых в Azure, и ресурсов, развернутых в локальных и гибридных облачных средах. 

## <a name="what-are-security-alerts"></a>Что такое оповещения системы безопасности?

Оповещения — это уведомления, которые центр безопасности создает при обнаружении угроз для ресурсов. Он определяет приоритеты и отображает оповещения вместе с информацией, необходимой для быстрого изучения проблемы. Центр безопасности также предоставляет рекомендации по устранению атак.

## <a name="how-does-security-center-detect-threats"></a>Как центр безопасности обнаруживает угрозы?

Чтобы выявить реальные угрозы и сократить число ложных срабатываний, центр безопасности собирает, анализирует и интегрирует данные журналов из ресурсов Azure, сети и подключенных партнерских решений, таких как брандмауэры и решения для защиты конечных точек. Центр безопасности анализирует эту информацию, часто сопоставляя информацию из нескольких источников, чтобы выявление угроз.

ASC отслеживает ресурсы, развернутые в Azure или развернутые в других локальных и гибридных облачных средах. Дополнительные сведения об обнаружении угроз и реагировании на них см. в статье [как центр безопасности обнаруживает угрозы и](security-center-detection-capabilities.md#asc-detects)отвечает на них.

## <a name="what-are-alert-incidents"></a>Что такое инциденты предупреждений?

Инцидент безопасности представляет собой набор связанных предупреждений, вместо того чтобы перечислять каждое оповещение по отдельности. Центр безопасности использует Fusion для корреляции различных предупреждений и сигналов низкого качества в инцидентах безопасности.

С помощью инцидентов центр безопасности предоставляет единое представление кампании по атакам и всех связанных предупреждений. Это представление позволяет быстро понять, какие действия потребовались злоумышленнику и какие ресурсы были затронуты. Дополнительные сведения см. в статье [корреляция облачных смарт-предупреждений](security-center-alerts-cloud-smart.md).

## <a name="upgrade-to-standard-for-advanced-detections"></a>Обновление до уровня "Стандартный" для расширенных обнаружений

Чтобы настроить расширенное обнаружение, обновите центр безопасности Azure до уровня "Стандартный". 

1. В меню центр безопасности выберите пункт **Политика безопасности**.
2. Для подписок, которые вы хотите переместить на уровень Standard, щелкните **изменить параметры**. 
3. На странице Параметры выберите **ценовая**Категория. 
   Бесплатная пробная версия доступна в месяц. Дополнительные сведения см. на [странице с ценами](https://azure.microsoft.com/pricing/details/security-center/).

# <a name="provide-security-contact-details-in-azure-security-center"></a>Предоставление сведений о контактных лицах по вопросам безопасности в центре безопасности Azure
Центр безопасности Azure порекомендует вам предоставить контактные сведения по вопросам безопасности для подписки Azure, если это еще не сделано. Корпорация Майкрософт будет использовать эту информацию для связи с вами, если центр Microsoft Security Response Center (MSRC) обнаружит, что к вашим пользовательским данным был получен незаконный или несанкционированный доступ. Центр Microsoft Security Response Center (MSRC) осуществляет выборочный мониторинг безопасности сети и инфраструктуры Azure, а также получает аналитические данные об угрозах и жалобы о нарушениях от третьих лиц.

Уведомление по электронной почте отправляется для первого за день оповещения и только тогда, когда у него высокий уровень серьезности. Параметры электронной почты можно настроить только для политик подписки. Группы ресурсов в подписке унаследуют эти параметры. Оповещения доступны только на стандартном уровне центра безопасности Azure.

Особенности отправки уведомления об оповещениях по электронной почте:
- для одного получателя электронной почты на тип оповещения в день;  
- не более 3 сообщений электронной почты отправляются одному получателю за один день;
- каждое сообщение электронной почты содержит одно оповещение, а не совокупность оповещений.
- только оповещения высокой серьезности;

> [!TIP]
> Для оповещений с другими уровнями серьезности создайте [автоматизацию рабочего процесса](workflow-automation.md) для использования приложения логики, которое будет отсылать сообщения электронной почты соответствующему персоналу.
 
Например, если сообщение электронной почты уже было отправлено, чтобы предупредить вас об атаке RDP, вы не получите другое сообщение электронной почты об атаке RDP в тот же день, даже если активировалось другое оповещение. 

> [!IMPORTANT]
> В документе приводится обзор службы с помощью примера развертывания.  Он не является пошаговым руководством.

## <a name="set-up-email-notifications-for-alerts"></a>Настройка уведомлений по электронной почте для оповещений<a name="email"></a>

1. В качестве пользователя с администратором безопасности роли или владельцем подписки откройте страницу **уведомления по электронной почте** :

    - Для оповещений откройте **Параметры цен &**, выберите соответствующую подписку и щелкните **уведомления по электронной почте**.

    - Если вы реализуете рекомендацию, в разделе **рекомендации**выберите **сведения о контактах безопасности**, а затем выберите подписку Azure для предоставления контактных сведений. Откроются **Уведомления по электронной почте**.

   ![Предоставление сведений о контактном лице по вопросам безопасности][2]

1. Введите адрес или разделенные запятыми адреса электронной почты контактных лиц по вопросам безопасности. Количество адресов электронной почты, которые можно ввести, не ограничено.

1. Чтобы получать по электронной почте оповещения с высоким уровнем серьезности, включите параметр **Send me emails about alerts**(Отправлять мне электронные сообщения об оповещениях). Для других уровней серьезности используйте приложение логики, как описано в этой [модели автоматизации рабочих процессов](workflow-automation.md).

1. Вы можете отправлять уведомления по электронной почте владельцам подписки (классических администраторов служб и соадминистраторов, а также роль владельца RBAC в области действия подписки).

1. Чтобы применить контактную информацию безопасности к своей подписке, нажмите кнопку **сохранить**.

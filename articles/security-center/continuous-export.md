# <a name="export-security-alerts-and-recommendations"></a>Экспорт оповещений о безопасности и рекомендаций

Центр безопасности Azure генерирует подробные оповещения и рекомендации по безопасности. Вы можете просмотреть их на портале или с помощью программных инструментов. Возможно, вам также придется экспортировать эту информацию или отправлять ее другим средствам мониторинга в вашей среде. 

В этой статье описывается набор инструментов, которые позволяют экспортировать оповещения и рекомендации либо вручную, либо непрерывно, непрерывно.

С помощью этих инструментов вы можете:

* Непрерывный экспорт в рабочие области Log Analytics
* Непрерывный экспорт в центры событий Azure (для интеграции с сторонними SIEMs)
* Экспорт в CSV (один раз)


## <a name="setting-up-a-continuous-export"></a>Настройка непрерывного экспорта

Приведенные ниже шаги необходимы независимо от того, настраиваете ли вы непрерывный экспорт в рабочее пространство журнала Analytics или концентраторы событий Azure.

1. Из боковой панели Центра безопасности выберите **настройки цен &.**

1. Выберите конкретную подписку, для которой необходимо настроить экспорт данных.
    
1. Со боковой панели страницы настроек для этой подписки выберите **Непрерывный экспорт.**

    [Варианты экспорта в Центре безопасности Azure ![](media/continuous-export/continuous-export-options-page.png)](media/continuous-export/continuous-export-options-page.png#lightbox) Здесь вы видите варианты экспорта. Для каждой доступной цели экспорта есть вкладка. 

1. Выберите тип данных, который вы хотите экспортировать, и выберите из фильтров для каждого типа (например, экспортировать только предупреждения высокой степени тяжести).

1. Из области "Экспортная цель" выберите, где вы хотите сохранить данные. Данные могут быть сохранены в целевой области по другой подписке (например, в экземпляре Центрального центра событий или в центральном рабочем пространстве Log Analytics).

1. Нажмите **Сохранить**.



## <a name="configuring-siem-integration-via-azure-event-hubs"></a>Настройка интеграции SIEM через концентраторы событий Azure

Azure Event Hubs — отличное решение для программного потребления любых потоковых данных. Для предупреждений и рекомендаций Центра безопасности Azure — это предпочтительный способ интеграции со сторонним SIEM.

> [!NOTE]
> Наиболее эффективным методом потоковой передачи данных мониторинга на внешние инструменты в большинстве случаев является использование концентраторов Azure Event. [В этой статье](https://docs.microsoft.com/azure/azure-monitor/platform/stream-monitoring-data-event-hubs) приводится краткое описание того, как можно передавать данные мониторинга из разных источников в центр событий, а также ссылки на подробные рекомендации.

> [!NOTE]
> Если ранее вы экспортировали оповещения Центра безопасности в SIEM с помощью журнала Деятельности Azure, приведенная ниже процедура заменяет эту методологию.

Для просмотра схем событий, накоторыенных типов данных, посетите [схемы событий Event Hub.](https://aka.ms/ASCAutomationSchemas)


### <a name="to-integrate-with-a-siem"></a>Интеграция с SIEM 

После того как вы настроили непрерывный экспорт выбранных данных Центра безопасности в центр событий Azure, можно настроить соответствующий разъем для SIEM:

* **Azure Sentinel** - Используйте наинимый центр безопасности Azure, предупреждая [о ней.](https://docs.microsoft.com/azure/sentinel/connect-azure-security-center)
* **Splunk** - Используйте [надстройки Azure Monitor для Splunk](https://github.com/Microsoft/AzureMonitorAddonForSplunk/blob/master/README.md)
* **IBM (Радар)** - Используйте [ручной настроенный источник журнала](https://www.ibm.com/support/knowledgecenter/SS42VS_DSM/com.ibm.dsm.doc/t_dsm_guide_microsoft_azure_enable_event_hubs.html)
* **ArcSight** - Используйте [SmartConnector](https://community.microfocus.com/t5/ArcSight-Connectors/SmartConnector-for-Microsoft-Azure-Monitor-Event-Hub/ta-p/1671292)

Кроме того, если вы хотите автоматически перевести постоянно экспортированные данные из настроенного концентратора событий в Azure Data Explorer, используйте инструкции в [данных Ingest из концентратора событий в Azure Data Explorer.](https://docs.microsoft.com/azure/data-explorer/ingest-data-event-hub)



## <a name="continuous-export-to-a-log-analytics-workspace"></a>Непрерывный экспорт в рабочее пространство log Analytics

Если вы хотите проанализировать данные Центра безопасности Azure в рабочей области Log Analytics или использовать оповещения Azure вместе с Центром безопасности, установите непрерывный экспорт в рабочее пространство Log Analytics.

Для экспорта в рабочее пространство Log Analytics необходимо, чтобы в рабочем пространстве были включены решения Центра безопасности по анализу журналов. При использовании портала Azure бесплатное решение Центра безопасности автоматически включается при включении непрерывного экспорта. Однако, если вы настраиваете свои непрерывные настройки экспорта программно, вы должны вручную выбрать свободный или стандартный уровень ценообразования для требуемого рабочего пространства из **параметров & ценообразования.**  

### <a name="log-analytics-tables-and-schemas"></a>Таблицы аналитики журналов и схемы

Оповещения и рекомендации по безопасности хранятся в таблицах *SecurityAlert* и *SecurityRecommendations* соответственно. Название решения Log Analytics, содержащего эти таблицы, зависит от того, находитесь ли вы на свободном или стандартном уровне (см. [цены:](security-center-pricing.md)Безопасность («Безопасность и аудит») или SecurityCenterFree.

![Таблица «SecurityAlert» в журнале Analytics](./media/continuous-export/log-analytics-securityalert-solution.png)

Для просмотра схем событий, накоторыенных типов данных, посетите [таблицы таблицы Журнала Analytics.](https://aka.ms/ASCAutomationSchemas)

###  <a name="view-exported-security-alerts-and-recommendations-in-azure-monitor"></a>Просмотр экспортированных предупреждений и рекомендаций по безопасности в Azure Monitor

В некоторых случаях можно просмотреть экспортированные оповещения о безопасности и/или рекомендации в [Azure Monitor.](https://docs.microsoft.com/azure/azure-monitor/platform/alerts-overview) 

Azure Monitor предоставляет унифицированный опыт оповещения для различных предупреждений Azure, включая диагностические журналы, метрические оповещения и пользовательские оповещения на основе запросов рабочего пространства Log Analytics.

Чтобы просмотреть оповещения и рекомендации центра безопасности в Azure Monitor, настроите правило оповещения на основе запросов Log Analytics (Log Alert):

1. На странице **оповещения** Azure Monitor нажмите **«Новое правило оповещения».**

    ![Страница предупреждений Azure Monitor](./media/continuous-export/azure-monitor-alerts.png)

1. На странице правила создания наверсможно настроить новое правило (так же, как настроить [правило оповещения о журнале в Azure Monitor):](https://docs.microsoft.com/azure/azure-monitor/platform/alerts-unified-log)

    * Для **ресурса**выберите рабочее пространство Log Analytics, в которое вы экспортировали оповещения и рекомендации по безопасности.

    * Для **условия**, выберите **пользовательский поиск журнала**. На странице, которая отображается, настроить запрос, период возврата и период частоты. В поисковом запросе можно ввести *SecurityAlert* или *SecurityRecommendation* для запроса типов данных, которые Центр безопасности постоянно экспортирует, поскольку вы включите функцию непрерывного экспорта в log Analytics. 
    
    * Дополнительно настройте [группу действий,](https://docs.microsoft.com/azure/azure-monitor/platform/action-groups) которую вы хотите вызвать. Группы действий могут инициировать отправку электронной почты, itsM-билеты, WebHooks и многое другое.
    ![Правило оповещения Azure Monitor](./media/continuous-export/azure-monitor-alert-rule.png)

Теперь в оповещениях Azure Monitor monitor будут представлены новые оповещения или рекомендации Центра безопасности Azure (в зависимости от конфигурации) с автоматическим запуском группы действий (если это предусмотрено).

Чтобы загрузить отчет CSV для оповещений или рекомендаций, откройте **страницу предупреждений безопасности** или **рекомендаций** и нажмите кнопку **Отчета Скачать CSV.**

[![Загрузка оповещений данных как файл CSV](media/continuous-export/download-alerts-csv.png)](media/continuous-export/download-alerts-csv.png#lightbox)

> [!NOTE]

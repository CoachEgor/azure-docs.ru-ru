# <a name="protect-your-machines-and-applications"></a>Защитите свои компьютеры и приложения
Когда центр безопасности Azure выявляет потенциальные уязвимости системы безопасности, он создает рекомендации по настройке необходимых элементов управления для защиты ресурсов.

В этой статье описывается страница " **Вычисление и приложения** " в разделе "безопасность ресурсов" центра безопасности.

Полный список рекомендаций, которые могут отображаться на этой странице, см. в разделе [рекомендации по данным и хранению](recommendations-reference.md#recs-computeapp).


## <a name="view-the-security-of-your-compute-and-apps-resources"></a>Просмотр безопасности ресурсов вычислений и приложений

[панель мониторинга центра безопасности ![](./media/security-center-virtual-machine-recommendations/overview.png)](./media/security-center-virtual-machine-recommendations/overview.png#lightbox)

Чтобы просмотреть состояние ресурсов вычислений и приложений, выберите **compute & приложения** в разделе **ресурсы** на боковой панели центра безопасности. Доступны следующие вкладки:

* **Обзор**. перечислены рекомендации для всех ресурсов вычислений и приложений, а также их текущее состояние безопасности. 

* [**Виртуальные машины и компьютеры**](#vms-and-computers): список рекомендаций для виртуальных машин, компьютеров и текущего состояния безопасности каждого

* [**Масштабируемые наборы виртуальных машин**](#vmscale-sets): список рекомендаций для масштабируемых наборов. 

* [**Облачные службы**](#cloud-services): содержит рекомендации для веб-и рабочих ролей, отслеживаемых центром безопасности.

* [**Службы приложений**](#app-services): список рекомендаций для сред службы приложений и текущее состояние безопасности каждого

* **Контейнеры**: список рекомендаций для контейнеров и оценка безопасности их конфигураций.

* **Ресурсы для вычислений**: список рекомендаций для ресурсов, таких как кластеры Service Fabric и концентраторы событий.

### <a name="whats-in-each-tab"></a>Что есть на каждой вкладке?

Каждая вкладка содержит несколько разделов, и в каждом разделе можно выполнить детализацию, чтобы просмотреть дополнительные сведения об элементе.

На каждой вкладке также будут отображаться рекомендации для соответствующих ресурсов в наблюдаемой среде. В первом столбце приведена рекомендация, а во втором — общее количество затронутых ресурсов, а в третьем — серьезность проблемы.

Каждая рекомендация включает в себя набор действий, которые могут быть выполнены при ее выборе. Например, если выбрать рекомендацию **Отсутствуют обновления системы**, откроется количество виртуальных машин и компьютеров, для которых отсутствуют обновления. Также отобразятся сведения об уровне серьезности отсутствующих обновлений.

> [!NOTE]
> Рекомендации по безопасности те же, что и на странице **рекомендации** , но здесь они фильтруются по выбранному типу ресурса. Дополнительные сведения об устранении рекомендаций см. в статье [Реализация рекомендаций по безопасности в центре безопасности Azure](security-center-recommendations.md).
>

### <a name="vms-and-computers"></a>Виртуальные машины и компьютеры
Раздел виртуальные машины и компьютеры содержит обзор всех рекомендаций по безопасности для виртуальных машин и компьютеров. Включены четыре типа компьютеров:

![компьютер, который не относится к Azure;](./media/security-center-virtual-machine-recommendations/security-center-monitoring-icon1.png) Компьютер, который не относится к Azure.

![Виртуальная машина, развернутая с помощью модели Azure Resource Manager](./media/security-center-virtual-machine-recommendations/security-center-monitoring-icon2.png) Виртуальная машина, развернутая с помощью модели Azure Resource Manager.

![Виртуальная машина, развернутая с помощью классической модели Azure](./media/security-center-virtual-machine-recommendations/security-center-monitoring-icon3.png) Виртуальная машина, развернутая с помощью классической модели Azure.

![Виртуальные машины, которые идентифицируются из рабочей области](./media/security-center-virtual-machine-recommendations/security-center-monitoring-icon4.png) Виртуальные машины, которые идентифицируются только из рабочей области, которая является частью просматриваемой подписки. Сюда входят виртуальные машины из других подписок, которые отправляют отчеты в рабочую область в этой подписке, а также виртуальные машины, которые были установлены с Operations Manager Direct Agent и не имеют идентификатора ресурса.

Под каждым таким набором отображается значок. Он указывает на виртуальную машину или компьютер, на которые следует обратить внимание, и содержит тип рекомендации. Можно также использовать фильтры для поиска в списке по **типу ресурса** и **уровню серьезности**.

Для детализации рекомендаций по безопасности для каждой виртуальной машины щелкните виртуальную машину.
Здесь отображаются сведения о состоянии защиты виртуальной машины или компьютера. В нижней части экрана можно увидеть рекомендуемое действие и серьезность каждой проблемы.

[![облачных служб](./media/security-center-virtual-machine-recommendations/recommendation-list.png)](./media/security-center-virtual-machine-recommendations/recommendation-list.png#lightbox)

### <a name="cloud-services"></a>Облачные службы
Рекомендация создается для облачных служб, если версия операционной системы устарела.

![Облачные сервисы](./media/security-center-virtual-machine-recommendations/security-center-monitoring-fig1-new006-2017.png)

В сценарии, в котором имеется рекомендация, выполните действия, описанные в рекомендациях по обновлению операционной системы. Когда обновление станет доступным, вы получите предупреждение (красный или оранжевый в зависимости от серьезности проблемы). Чтобы получить полное описание этой рекомендации, щелкните **обновить версию ОС** в столбце **Описание** .

### <a name="app-services"></a>Службы приложений
Чтобы просмотреть сведения о службе приложений, необходимо быть ценовой категории "Стандартный" центра безопасности и включить службу приложений в подписке. Инструкции по включению этой функции см. в статье [Защита службы приложений с помощью центра безопасности Azure](security-center-app-services.md).


В разделе **Службы приложений** содержится список сред службы приложений и сводка работоспособности, основанная на оценке центра безопасности.

![Службы приложений](./media/security-center-virtual-machine-recommendations/app-services.png)

Показаны три типа служб приложений:

![Среда службы приложений](./media/security-center-virtual-machine-recommendations/ase.png) Среда службы приложений

![Веб-приложение](./media/security-center-virtual-machine-recommendations/web-app.png) Веб-приложение

![Приложение-функция](./media/security-center-virtual-machine-recommendations/function-app.png) Приложение-функция

Если выбрать веб-приложение, откроется представление сводки с тремя вкладками:

   - **Рекомендации**: на основе оценок, выполненных центром безопасности, с ошибкой.
   - **Пройденные проверки**. Список пройденных оценок, выполняемых центром безопасности.
   - **Оценки недоступны**. Список оценок, которые не были выполнены из-за ошибки или из-за того, что рекомендация не относится к конкретной службе приложения.

   В разделе **Рекомендации** есть список рекомендаций для выбранного веб-приложения и серьезность каждой рекомендации.

   ![Рекомендации по службам приложений](./media/security-center-virtual-machine-recommendations/app-services-rec.png)

Выберите рекомендацию, чтобы просмотреть ее описание, список работоспособных, неработоспособных и непроверенных ресурсов.

   - В столбце **переданные оценки** отображается список пройденных оценок. Уровень серьезности этих проверок всегда имеет зеленый цвет.

   - Выберите пройденную проверку из списка, чтобы просмотреть ее описание, список работоспособных, неработоспособных и непроверенных ресурсов. Есть вкладка для неработоспособных ресурсов, но список там пуст, если проверки прошли успешно.

### <a name="vmscale-sets"></a>Масштабируемые наборы виртуальных машин
Центр безопасности автоматически обнаруживает масштабируемые наборы и рекомендует установить на них Microsoft Monitoring Agent.

Установка Microsoft Monitoring Agent 

1. Выберите рекомендацию **Установка агента мониторинга на масштабируемом наборе виртуальных машин (предварительная версия)** . Вы получите список неотслеживаемых масштабируемых наборов.

1. Выберите неработоспособный масштабируемый набор. Следуйте инструкциям по установке агента мониторинга, используя существующую заполненную рабочую область или создав новую. Не забудьте задать для рабочей области [ценовую категорию](security-center-pricing.md), если она не задана.

   ![Установка MMS](./media/security-center-virtual-machine-recommendations/install-mms.png)

Чтобы задать новые масштабируемые наборы для автоматической установки Microsoft Monitoring Agent, выполните следующие действия.
1. Перейдите в раздел "Политика Azure" и щелкните **Определения**.

1. Выполните поиск политики **развертывание log Analytics агент для масштабируемых наборов виртуальных машин Windows** и щелкните его.

1. Щелкните **Назначить**.

1. Выберите **Область**, **Рабочую область Log Analytics** и нажмите кнопку **Назначить**.

Если вы хотите задать для всех существующих масштабируемых наборов установку Microsoft Monitoring Agent, в политике Azure перейдите на страницу **Исправление** и примените существующую политику к существующим наборам.

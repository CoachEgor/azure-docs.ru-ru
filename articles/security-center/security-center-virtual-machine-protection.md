# <a name="protect-your-machines-and-applications"></a>Защитите свои машины и приложения
Когда Центр безопасности Azure определяет потенциальные уязвимости безопасности, он создает рекомендации, которые помогут вам в процессе настройки необходимых элементов управления для затвердевания и защиты ресурсов.

В этой статье объясняется страница **«Вычисления и приложения»** раздела безопасности Центра безопасности.

Полный список рекомендаций, которые вы можете увидеть на этой странице, смотрите [рекомендации По вычислительной технике и приложениям.](recommendations-reference.md#recs-computeapp)


## <a name="view-the-security-of-your-compute-and-apps-resources"></a>Просмотр безопасности ресурсов вычислений и приложений

[![Панель мониторинга центра безопасности](./media/security-center-virtual-machine-recommendations/compute-and-apps-recs-overview.png)](./media/security-center-virtual-machine-recommendations/compute-and-apps-recs-overview.png#lightbox)

Для просмотра состояния ресурсов вычислений и приложений с левого стекла в Центре безопасности выберите **& приложения.** Доступны следующие вкладки:

* **Обзор**: перечислены рекомендации для всех ресурсов вычислений и приложений, а также их текущее состояние безопасности 

* [**VMs и серверы**](#vms-and-computers): перечисляет рекомендации для ваших vMs, компьютеров и текущего состояния безопасности каждого

* [**Наборы масштабов VM:**](#vmscale-sets)перечислены рекомендации для наборов масштабов, 

* [**Облачные службы**](#cloud-services): перечислены рекомендации для ваших веб-ролей и ролей сотрудников, контролируемых Центром безопасности

* [**Службы приложений:**](#app-services)перечислены рекомендации для сред службы приложений и текущее состояние безопасности каждого

* [**Контейнеры**](#containers): перечислены рекомендации для контейнеров и оценка безопасности их конфигураций

* **Вычислите ресурсы**: списки рекомендаций для вычислительных ресурсов, таких как кластеры Service Fabric и концентраторы событий

### <a name="whats-in-each-tab"></a>Что в каждой вкладке?

Каждая вкладка имеет несколько разделов, и в каждом разделе, вы можете просверлить вниз, чтобы увидеть дополнительные сведения о элементе показано.

В каждой вкладке вы также увидите рекомендации для соответствующих ресурсов в контролируемой среде. В первом столбце указана рекомендация, вторая — общее количество затронутых ресурсов, а в третьем — серьезность проблемы.

Каждая рекомендация включает в себя набор действий, которые могут быть выполнены при ее выборе. Например, если выбрать рекомендацию **Отсутствуют обновления системы**, откроется количество виртуальных машин и компьютеров, для которых отсутствуют обновления. Также отобразятся сведения об уровне серьезности отсутствующих обновлений.

> [!NOTE]
> Рекомендации по безопасности такие же, как и на странице **Рекомендации,** но здесь они отфильтроваются на выбранный тип ресурсов. Для получения дополнительной информации о том, как разрешить рекомендации, смотрите [Рекомендации по безопасности «Реализация» в Центре безопасности Azure.](security-center-recommendations.md)
>




### <a name="vms-and-servers"></a><a name="vms-and-computers"></a>ВМ и серверы
Раздел VMs и компьютеров дает вам обзор всех рекомендаций по безопасности для ваших vMs и компьютеров. Четыре типа машин включены:

![компьютер, который не относится к Azure;](./media/security-center-virtual-machine-recommendations/security-center-monitoring-icon1.png) Компьютер, который не относится к Azure.

![Виртуальная машина, развернутая с помощью модели Azure Resource Manager](./media/security-center-virtual-machine-recommendations/security-center-monitoring-icon2.png) Виртуальная машина, развернутая с помощью модели Azure Resource Manager.

![Виртуальная машина, развернутая с помощью классической модели Azure](./media/security-center-virtual-machine-recommendations/security-center-monitoring-icon3.png) Виртуальная машина, развернутая с помощью классической модели Azure.

![Виртуальные машины, которые идентифицируются из рабочей области](./media/security-center-virtual-machine-recommendations/security-center-monitoring-icon4.png) Виртуальные машины, которые идентифицируются только из рабочей области, которая является частью просматриваемой подписки. Это включает в себя включающие в себя ввожжа в ви-х атакжем из других подписок, которые отчитываются перед рабочим пространством в этой подписке, и vMs, которые были установлены с прямым агентом Operations Manager, и не имеют идентификатора ресурсов.

Под каждым таким набором отображается значок. Он указывает на виртуальную машину или компьютер, на которые следует обратить внимание, и содержит тип рекомендации. Можно также использовать фильтры для поиска в списке по **типу ресурса** и **уровню серьезности**.

Для детализации рекомендаций по безопасности для каждой виртуальной машины щелкните виртуальную машину.
Здесь отображаются сведения о состоянии защиты виртуальной машины или компьютера. Внизу можно увидеть рекомендуемое действие и серьезность каждой проблемы.

[![Облачные службы](./media/security-center-virtual-machine-recommendations/recommendation-list.png)](./media/security-center-virtual-machine-recommendations/recommendation-list.png#lightbox)




### <a name="virtual-machine-scale-sets"></a><a name="vmscale-sets"></a>Наборы виртуального масштаба машины
Центр безопасности автоматически обнаруживает, есть ли у вас наборы масштабов, и рекомендует установить на них агента мониторинга Майкрософт.

Установка Microsoft Monitoring Agent 

1. Выберите рекомендацию **Установка агента мониторинга на масштабируемом наборе виртуальных машин (предварительная версия)**. Вы получите список неотслеживаемых масштабируемых наборов.

1. Выберите неработоспособный масштабируемый набор. Следуйте инструкциям по установке агента мониторинга, используя существующую заполненную рабочую область или создав новую. Не забудьте задать для рабочей области [ценовую категорию](security-center-pricing.md), если она не задана.

   ![Установка MMS](./media/security-center-virtual-machine-recommendations/install-mms.png)

Для установки новых наборов масштабов для автоматической установки агента мониторинга Майкрософт:
1. Перейдите в раздел "Политика Azure" и щелкните **Определения**.

1. Поиск агента по **развертыванию журналов** для набора виртуальной машины Windows и нажмите на него.

1. Щелкните **Назначить**.

1. Выберите **Область**, **Рабочую область Log Analytics** и нажмите кнопку **Назначить**.

Если вы хотите задать для всех существующих масштабируемых наборов установку Microsoft Monitoring Agent, в политике Azure перейдите на страницу **Исправление** и примените существующую политику к существующим наборам.





### <a name="cloud-services"></a><a name="cloud-services"></a>Облачные службы
Рекомендация создается для облачных служб, если версия операционной системы устарела.

![Облачные службы](./media/security-center-virtual-machine-recommendations/security-center-monitoring-fig1-new006-2017.png)

В сценарии, где у вас есть рекомендации, следуйте шагам в рекомендации по обновлению операционной системы. Когда обновление доступно, у вас будет оповещение (красный или оранжевый - в зависимости от серьезности проблемы). Для полного объяснения этой рекомендации нажмите **Обновление ОС версии** под колонкой **ОПИСАНИЕ.**






### <a name="app-services"></a><a name="app-services"></a>Услуги приложений
Чтобы просмотреть информацию службы приложений, вы должны быть на стандартном уровне ценообразования Центра безопасности и включить службу app Service в подписке. Для получения инструкций по [Protect App Service with Azure Security Center](security-center-app-services.md)включению этой функции см.

В разделе **Службы приложений** содержится список сред службы приложений и сводка работоспособности, основанная на оценке центра безопасности.

![Службы приложений](./media/security-center-virtual-machine-recommendations/app-services.png)

Показано три типа прикладных услуг:

![Среда службы приложений](./media/security-center-virtual-machine-recommendations/ase.png) Среда службы приложений

![Веб-приложение](./media/security-center-virtual-machine-recommendations/web-app.png) Веб-приложение

![Приложение-функция](./media/security-center-virtual-machine-recommendations/function-app.png) Приложение-функция

При выборе веб-приложения сводный вид открывается тремя вкладками:

   - **Рекомендации**: на основе оценок, проведенных Центром безопасности, которые не удалось.
   - **Пройденные проверки**. Список пройденных оценок, выполняемых центром безопасности.
   - **Оценки недоступны**. Список оценок, которые не были выполнены из-за ошибки или из-за того, что рекомендация не относится к конкретной службе приложения.

   В разделе **Рекомендации** есть список рекомендаций для выбранного веб-приложения и серьезность каждой рекомендации.

   ![Рекомендации по службам приложений](./media/security-center-virtual-machine-recommendations/app-services-rec.png)

Выберите рекомендацию, чтобы просмотреть ее описание, список работоспособных, неработоспособных и непроверенных ресурсов.

   - В столбце **«Пройдемые оценки»** показан список пройденых оценок. Уровень серьезности этих проверок всегда имеет зеленый цвет.

   - Выберите пройденную проверку из списка, чтобы просмотреть ее описание, список работоспособных, неработоспособных и непроверенных ресурсов. Есть вкладка для неработоспособных ресурсов, но список там пуст, если проверки прошли успешно.

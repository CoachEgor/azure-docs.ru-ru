# <a name="security-recommendations---a-reference-guide"></a>Рекомендации по безопасности — справочное руководство

В этой статье перечислены рекомендации, которые могут отображаться в центре безопасности Azure. Рекомендации, показанные в вашей среде, зависят от защищаемых ресурсов и настроенной конфигурации.

Дополнительные сведения о том, как реагировать на эти рекомендации, см. [в статье исправление рекомендаций в центре безопасности Azure](security-center-remediate-recommendations.md).

Оценка безопасности зависит от количества выполненных рекомендаций центра безопасности. Чтобы решить, какие рекомендации следует решить первыми, просмотрите уровень серьезности каждого из них и его потенциальное влияние на оценку безопасности.

>[!TIP]
> Если в описании рекомендации указано "нет связанной политики", обычно это обусловлено тем, что эта рекомендация зависит от другой рекомендации и *ее* политики. Например, рекомендуется исправить рекомендацию «сбои работоспособности Endpoint Protection»...», полагается на рекомендацию, которая проверяет, *установлено* ли решение для защиты конечных точек («решение для защиты конечных точек должно быть установлено...»). Базовая *рекомендация имеет* политику. Ограничение политик только базовыми рекомендациями упрощает управление политиками.

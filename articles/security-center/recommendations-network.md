# <a name="network-recommendations---reference-guide"></a>Рекомендации по сети — Справочное руководство

В этой статье содержится полный список рекомендаций, которые могут отображаться в центре безопасности Azure, в отношении топологии сети и конечных точек с выходом в Интернет.

Описание того, как найти эти и способы их устранения, см. [здесь](security-center-network-recommendations.md).

## <a name="network-recommendations"></a>Рекомендации по сети

|Имя рекомендации|Описание|Серьезность|Оценка безопасности.|Тип ресурса|
|----|----|----|----|----|----|
|Должны быть включены группы безопасности сети на уровне подсети.|Включите группы безопасности сети для управления доступом к сети ресурсов, развернутых в подсетях.|Высокий/средний|30|Подсеть|
|Виртуальные машины должны быть связаны с группой безопасности сети.|Включение групп безопасности сети для управления доступом виртуальных машин к сети.|Высокий/средний|30|Виртуальная машина|
|Доступ должен быть ограничен для групп безопасности сети с виртуальными машинами с выходом в Интернет.|Заусиление групп безопасности сети для виртуальных машин с выходом в Интернет путем ограничения доступа существующих правил.|Высокая|20|Виртуальная машина|
|Правила для веб-приложений в IaaS группы безопасности сети должны быть зафиксированы|Заусиление группы безопасности сети (NSG) для виртуальных машин, на которых выполняются веб-приложения, с правилами NSG, которые имеют более строгие ограничения по отношению к портам веб-приложений.|Высокая|20|Виртуальная машина|
|Доступ к службам приложений должен быть ограничен|Ограничьте доступ к службам приложений, изменив конфигурацию сети, чтобы запретить входящий трафик из диапазонов, которые являются слишком широкими.|Высокая|10|Служба приложений|
|Порты управления должны быть закрыты на виртуальных машинах|Заусиление группы безопасности сети виртуальных машин, чтобы ограничить доступ к портам управления.|Высокая|10|Виртуальная машина|
Стандарт Защиты от атак DDoS должен быть включен.|Защитите виртуальные сети, содержащие приложения с общедоступными IP-адресами, включив стандартную службу защиты от атак DDoS. Защита от атак DDoS обеспечивает устранение сетевых объемные и атак по протоколу.|Высокая|10|Виртуальная сеть|
|IP-пересылка на виртуальной машине должна быть отключена|Отключите IP-пересылку. Если IP-пересылка включена на сетевом адаптере виртуальной машины, компьютер может получить трафик, адресованный другим получателям. IP-пересылка редко требуется (например, при использовании виртуальной машины в качестве сетевого виртуального модуля), поэтому она должна быть проверена командой безопасности сети.|Средние|10|Виртуальная машина|
|Веб-приложение должно быть доступно только по HTTPS|Разрешить доступ только по протоколу HTTPS для веб-приложений. Использование HTTPS обеспечивает проверку подлинности сервера и службы и защищает данные при передаче от атак с перехватом сетевого уровня.|Средние|20|Веб-приложение|
|Управление JIT-доступом к сети должно применяться к виртуальным машинам|Применение контроля доступа (ВИРТУАЛЬНОЙ машины) на основе JIT для постоянного блокирования доступа к выбранным портам и предоставления полномочным пользователям возможности их открытия с помощью JIT-компилятора в течение ограниченного времени.|Высокая|20|Виртуальная машина|
|Приложения функций должны быть доступны только по протоколу HTTPS|Включить доступ только по протоколу HTTPS для приложений функций. Использование HTTPS обеспечивает проверку подлинности сервера и службы и защищает данные при передаче от атак с перехватом сетевого уровня.|Средние|20|Приложение-функция|
|Должно выполняться безопасное перемещение в учетные записи хранения.|Включите безопасную пересылку учетных записей хранения. Безопасная передача данных — это параметр, при включении которого ваша учетная запись хранения принимает запросы только с безопасных подключений (HTTPS). Использование протокола HTTPS обеспечивает проверку подлинности между сервером и службой и защищает данные при передаче от атак из сетевого уровня, таких как "злоумышленник в середине", "перехваты" и "перехват сеансов".|Высокая|20|Учетная запись хранения|

# <a name="enhanced-secure-score-preview-in-azure-security-center"></a>Улучшенная оценка безопасности (Предварительная версия) в центре безопасности Azure

## <a name="introduction-to-secure-score"></a>Введение в безопасность оценки

Центр безопасности Azure имеет две основные цели: это поможет вам разобраться в текущей ситуации безопасности и эффективно и эффективно повысить безопасность. Центральным аспектом центра безопасности, который позволяет достичь этих целей, является безопасность оценки.

Центр безопасности постоянно оценивает ресурсы, подписки и Организации на предмет проблем безопасности. Затем он объединяет все выводы в одну оценку, чтобы вы могли сразу же понять текущую ситуацию безопасности: чем выше оценка, тем ниже определенная степень риска. Используйте оценку для контроля за усилиями и проектами в Организации. 

Страница оценка безопасности центра безопасности включает:

- **Оценка** — показатель безопасности отображается в виде процентного значения, но базовые значения также очищаются:

    [![Показатель безопасности, отображаемый как процентное значение с базовыми числами, очищается](media/secure-score-security-controls/secure-score-with-percentage.png)](media/secure-score-security-controls/secure-score-with-percentage.png#lightbox)

- **Элементы управления безопасностью** — каждый элемент управления является логической группой связанных рекомендаций по безопасности и отражает уязвимые области атак. Элемент управления — это набор рекомендаций по обеспечению безопасности с инструкциями, помогающими реализовать эти рекомендации. Оценка повысится только при исправлении *всех* рекомендаций для одного ресурса в элементе управления.

    Чтобы немедленно увидеть, насколько хорошо ваша организация обеспечивает безопасность каждой отдельной уязвимой зоны, просмотрите баллы за каждый контроль безопасности.

    Дополнительные сведения см. [в разделе как вычисляется показатель безопасности](secure-score-security-controls.md#how-the-secure-score-is-calculated) ниже. 


>[!TIP]
> Более ранние версии центра безопасности набрали баллы на уровне рекомендаций: при исправлении рекомендации для одного ресурса повышение уровня безопасности повысится. На сегодняшний день ваша оценка повысится только при исправлении *всех* рекомендаций для одного ресурса в элементе управления. Поэтому оценка повысится только при повышении безопасности ресурса.
> Хотя эта улучшенная версия по-прежнему доступна в предварительной версии, в качестве варианта на портале Azure доступен более ранняя оценка безопасности. 


## <a name="locating-your-secure-score"></a>Поиск оценки безопасности

Центр безопасности наглядно отображает рейтинг: это первое, что показано на странице Обзор. Если перейти на страницу выделенной оценки безопасности, вы увидите, что оценка разбивается по подписке. Щелкните одну подписку, чтобы просмотреть подробный список приоритетных рекомендаций и потенциальное влияние, устранения их на оценку подписки.

## <a name="how-the-secure-score-is-calculated"></a>Как вычисляется показатель безопасности 

Вклад каждого элемента управления безопасностью в общую оценку безопасности отображается на странице рекомендации.

[![Расширенный показатель безопасности предоставляет средства управления безопасностью](media/secure-score-security-controls/security-controls.png)](media/secure-score-security-controls/security-controls.png#lightbox)

Чтобы получить все возможные точки для управления безопасностью, все ресурсы должны соответствовать всем рекомендациям по безопасности в рамках управления безопасностью. Например, центр безопасности имеет несколько рекомендаций, касающихся защиты портов управления. В прошлом можно было исправить некоторые из этих связанных и взаимозависимых рекомендаций, в то же время оставляя другие, не разрешаемые, и ваш показатель безопасности улучшится. Когда вы просматриваете объектно, можно легко утверждать, что безопасность не улучшилась до тех пор, пока вы не решите их все. Теперь необходимо исправить все, чтобы отличить вашу оценку безопасности.

Например, элемент управления безопасности под названием "применить обновления системы" имеет максимальную оценку из шести точек, которые можно увидеть в подсказке к потенциальному значению увеличения элемента управления:

[![Управление безопасностью "применить обновления системы"](media/secure-score-security-controls/apply-system-updates-control.png)](media/secure-score-security-controls/apply-system-updates-control.png#lightbox)

На снимке экрана выше возможны проблемы с управлением безопасностью "применить обновления системы". Это означает, что если вы исправите все рекомендации в этом элементе управления, оценка будет увеличена на 2% (в данном случае одна точка). Для простоты значения в столбце "потенциальное увеличение" списка рекомендаций округляются до целых чисел. В подсказках отображаются точные значения:

* **Максимальная оценка** — максимальное количество точек, которые можно получить, выполнив все рекомендации в элементе управления. Максимальная оценка элемента управления указывает относительную значимость этого элемента управления. Используйте значения MAX Score, чтобы оценить, какие проблемы должны быть обработаны первыми. 
* **Потенциальное увеличение** — остальные доступные точки в элементе управления. Чтобы добавить эти точки в вашу оценку безопасности, исправьте все рекомендации по данному элементу управления. В приведенном выше примере одной из точек, показанных для элемента управления, является факт 0,96 точек.
* **Текущая оценка** — текущий показатель для этого элемента управления. Каждый элемент управления влияет на общую оценку. В этом примере элемент управления участвует в 5,04, что указывает на итог.

## <a name="improving-your-secure-score"></a>Улучшение оценки безопасности

Чтобы улучшить безопасность, исправьте рекомендации по безопасности из списка рекомендаций. Каждую рекомендацию можно исправить вручную для каждого ресурса или с помощью **быстрого исправления!** параметр (если доступен) для быстрого применения исправления к группе ресурсов. Дополнительные сведения см. в разделе [исправление рекомендаций](security-center-remediate-recommendations.md).

>[!IMPORTANT]
> Только встроенные рекомендации влияют на оценку безопасности.

## <a name="secure-score-faq"></a>Вопросы и ответы о безопасности

### <a name="why-has-my-secure-score-gone-down"></a>Почему потеряна Оценка безопасности?
Центр безопасности переключился на расширенный показатель безопасности (в настоящее время находится в состоянии предварительной версии), который включает изменения в способе вычисления оценки. Теперь необходимо разрешить все рекомендации для ресурса, чтобы получить баллы. Оценки также изменяются до шкалы 0-10.

### <a name="if-i-address-only-three-out-of-four-recommendations-in-a-security-control-will-my-secure-score-change"></a>Если я использую только три из четырех рекомендаций в контроле безопасности, будет ли изменена безопасность?
Нет. Он не изменится, пока не будут исправлены все рекомендации для отдельного ресурса. Чтобы получить максимальную оценку для элемента управления, необходимо исправить все рекомендации для всех ресурсов.

### <a name="is-the-previous-experience-of-the-secure-score-still-available"></a>Доступен ли предыдущий опыт оценки безопасности? 
Да. В течение этого время они будут работать параллельно, чтобы упростить переход. Предполагалось, что предыдущая модель была выйдет за пределы времени. 

### <a name="if-a-recommendation-isnt-applicable-to-me-and-i-disable-it-in-the-policy-will-my-security-control-be-fulfilled-and-my-secure-score-updated"></a>Если рекомендация неприменима для меня и я отключил ее в политике, будет ли элемент управления безопасностью выполнен и обновлена ли моя безопасность?
Да. Рекомендуется отключить рекомендации, если они неприменимы в вашей среде. Инструкции по отключению конкретной рекомендации см. в разделе [Отключение политик безопасности](https://docs.microsoft.com/azure/security-center/tutorial-security-policy#disable-security-policies).

### <a name="if-a-security-control-offers-me-zero-points-towards-my-secure-score-should-i-ignore-it"></a>Если элемент управления безопасностью предлагает мне нулевые баллы для оценки безопасности, следует ли его игнорировать?
В некоторых случаях вы увидите контрольный показатель Max больше нуля, но это значение равно нулю. Если инкрементная Оценка ресурсов незначительна, она округляется до нуля. Не игнорируйте эти рекомендации, так как они по-прежнему улучшают безопасность. Единственным исключением является элемент управления "дополнительная рекомендация". Устранения эти рекомендации не увеличивают оценку, но будут повышать общую безопасность.

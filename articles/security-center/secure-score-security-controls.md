# <a name="enhanced-secure-score-preview-in-azure-security-center"></a>Улучшенная безопасная оценка (предварительный просмотр) в Центре безопасности Azure

## <a name="introduction-to-secure-score"></a>Введение для обеспечения оценка

Центр безопасности Azure имеет две основные цели: помочь вам понять текущую ситуацию с безопасностью и помочь вам эффективно и эффективно улучшить вашу безопасность. Центральным аспектом Центра безопасности, который позволяет достичь этих целей, является безопасный счет.

Центр безопасности постоянно оценивает ваши ресурсы, подписки и организацию по вопросам безопасности. Затем он агрегирует все выводы в одну оценку, чтобы вы могли с первого взгляда сказать текущую ситуацию с безопасностью: чем выше оценка, тем ниже указанный уровень риска. Используйте счет для отслеживания усилий по обеспечению безопасности и проектов в вашей организации. 

Безопасная страница оценки Центра безопасности включает в себя:

- **Оценка** - Безопасный балл отображается как процентное значение, но основные значения также ясны:

    [![Безопасный балл, показанный в процентном соотношении с основными числами ясно слишком](media/secure-score-security-controls/secure-score-with-percentage.png)](media/secure-score-security-controls/secure-score-with-percentage.png#lightbox)

- **Элементы управления безопасностью** - Каждый элемент управления представляет собой логическую группу связанных с ними рекомендаций по безопасности и отражает ваши уязвимые поверхности атак. Контроль — это набор рекомендаций по безопасности, в которые содержатся инструкции, которые помогут вам выполнить эти рекомендации. Ваша оценка улучшается только при исправлении *всех* рекомендаций для одного ресурса в элементе управления.

    Чтобы сразу узнать, насколько хорошо ваша организация обеспечивает безопасность каждой отдельной поверхности атаки, просмотрите баллы для каждого элемента управления безопасностью.

    Для получения дополнительной информации смотрите [Как рассчитывается безопасный балл](secure-score-security-controls.md#how-the-secure-score-is-calculated) ниже. 


>[!TIP]
> Более ранние версии Центра безопасности начисляли баллы на рекомендательном уровне: при исправлении рекомендации для одного ресурса улучшился ваш безопасный балл. Сегодня ваш балл только улучшается, если вы исправите *все* рекомендации для одного ресурса в рамках контроля. Таким образом, ваш счет только улучшается, когда вы улучшили безопасность ресурса.
> Несмотря на то, что эта улучшенная версия все еще находится в предварительном просмотре, более ранний безопасный опыт оценки доступен в качестве опции из портала Azure. 


## <a name="locating-your-secure-score"></a>Поиск вашего безопасного счета

Центр безопасности отображает ваш счет на видном месте: это первое, что показано на странице Обзор. Если вы нажмете на специальную страницу безопасного счета, вы увидите счет, разбитый по подписке. Нажмите на одну подписку, чтобы увидеть подробный список приоритетных рекомендаций и потенциальное влияние, которое их исправления будут иметь на счет подписки.

## <a name="how-the-secure-score-is-calculated"></a>Как рассчитывается безопасный балл 

Вклад каждого контроля безопасности в общую безопасную оценку четко показан на странице рекомендаций.

[![Улучшенная безопасная оценка вводит элементы управления безопасностью](media/secure-score-security-controls/security-controls.png)](media/secure-score-security-controls/security-controls.png#lightbox)

Чтобы получить все возможные точки для контроля безопасности, все ваши ресурсы должны соответствовать всем рекомендациям безопасности в рамках контроля безопасности. Например, центр безопасности имеет несколько рекомендаций относительно того, как обезопасить порты управления. В прошлом, вы могли бы исправить некоторые из этих связанных и взаимозависимых рекомендаций, оставляя другие нерешенными, и ваш безопасный балл улучшится. Если смотреть объективно, то легко утверждать, что ваша безопасность не улучшилась, пока вы не решили их все. Теперь, вы должны исправления их всех, чтобы изменить ситуацию к вашей безопасной оценки.

Например, контроль безопасности под названием "Apply System updates" имеет максимальный балл в шесть баллов, который вы можете увидеть в инструментарии о потенциальном увеличении стоимости элемента:

[![Контроль безопасности "Приложить обновления системы"](media/secure-score-security-controls/apply-system-updates-control.png)](media/secure-score-security-controls/apply-system-updates-control.png#lightbox)

Потенциал для контроля безопасности "Применить обновления системы" в скриншоте выше показывает "2% (1 точка)". Это означает, что если вы исправите все рекомендации в этом контроле, ваш балл увеличится на 2% (в этом случае, один балл). Для простоты значения в столбце списка рекомендаций "Потенциальное увеличение" округлены на целые числа. Наконечники инструментов показывают точные значения:

* **Max score** - максимальное количество очков, которые вы можете набрать, выполнив все рекомендации в рамках контроля. Максимальный балл для элемента управления указывает на относительную значимость этого элемента управления. Используйте значения максимальной оценки для сортировки проблем, над которыми следует работать в первую очередь. 
* **Потенциальное увеличение** - остальные точки доступны для вас в пределах контроля. Чтобы получить эти очки, добавленные к вашему безопасному счету, исполните все рекомендации управления. В приведенном выше примере одна точка, показанная для управления, на самом деле составляет 0,96 балла.
* **Текущий счет** - текущий счет для этого контроля. Каждый элемент управления вносит свой вклад в общий балл. В этом примере элемент управления вносит 5,04 балла в общую сумму.

## <a name="improving-your-secure-score"></a>Улучшение вашего безопасного счета

Чтобы улучшить свой безопасный счет, исправьте рекомендации по безопасности из списка рекомендаций. Вы можете исправить каждую рекомендацию вручную для каждого ресурса, или с помощью **быстрого fix!** возможность (при наличии) быстро применить исправление рекомендации группе ресурсов. Для получения дополнительной информации, [см.](security-center-remediate-recommendations.md)

>[!IMPORTANT]
> Только встроенные рекомендации влияют на безопасный балл.

## <a name="secure-score-faq"></a>Безопасный балл часто задаваемые вопросы

### <a name="why-has-my-secure-score-gone-down"></a>Почему мой безопасный балл снизился?
Центр безопасности переключился на расширенный безопасный балл (в настоящее время находится в состоянии предварительного просмотра), который включает изменения в способе расчета оценки. Теперь необходимо решить все рекомендации для ресурса для получения очков. Оценки также изменились по шкале 0-10.

### <a name="if-i-address-only-three-out-of-four-recommendations-in-a-security-control-will-my-secure-score-change"></a>Если я обращаюсь только к трем из четырех рекомендаций в области контроля безопасности, изменится ли моя безопасная оценка?
Нет. Он не изменится, пока вы не исправите все рекомендации для одного ресурса. Чтобы получить максимальный балл за контроль, необходимо исправления всех рекомендаций, для всех ресурсов.

### <a name="is-the-previous-experience-of-the-secure-score-still-available"></a>Доступен ли предыдущий опыт безопасного балла? 
Да. Какое-то время они будут работать бок о бок, чтобы облегчить переход. Ожидайте, что предыдущая модель будет постепенно выведена из нее во времени. 

### <a name="if-a-recommendation-isnt-applicable-to-me-and-i-disable-it-in-the-policy-will-my-security-control-be-fulfilled-and-my-secure-score-updated"></a>Если рекомендация не применима ко мне, и я отменяю ее в политике, будет ли мой контроль безопасности выполнен и мой защищенный балл обновлен?
Да. Мы рекомендуем отключить рекомендации, когда они неприменимы в вашей среде. Для получения инструкций о том, как отключить конкретную рекомендацию, [см.](https://docs.microsoft.com/azure/security-center/tutorial-security-policy#disable-security-policies)

### <a name="if-a-security-control-offers-me-zero-points-towards-my-secure-score-should-i-ignore-it"></a>Если контроль безопасности предлагает мне ноль очков к моему безопасному счету, должен ли я игнорировать его?
В некоторых случаях вы увидите максимальный балл управления больше нуля, но влияние равно нулю. Когда дополнительный балл для фиксации ресурсов незначителен, он округляется до нуля. Не игнорируйте эти рекомендации, поскольку они по-прежнему приносят улучшения безопасности. Единственным исключением является контроль "Дополнительная лучшая практика". Исправление этих рекомендаций не увеличит ваш счет, но повысит общую безопасность.

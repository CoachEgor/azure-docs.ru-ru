# <a name="409002-linkcreationconflict"></a>409002 LinkCreationConflict

В этой статье описаны причины и решения ошибок **409002 LinkCreationConflict.**

## <a name="symptoms"></a>Симптомы

Вы видите ошибку **409002 LinkCreationConflict,** зарегистрированную в диагностических журналах вместе с отключением устройства или сбоем сообщения облака к устройству. 

<!-- When using AMQP? -->

## <a name="cause"></a>Причина:

Как правило, эта ошибка возникает, когда IoT Концентратор обнаруживает, что клиент имеет более одного соединения. Фактически, когда приходит новый запрос на подключение для устройства с существующим соединением, IoT Hub закрывает существующее соединение с этой ошибкой.

### <a name="cause-1"></a>Причина 1

В наиболее распространенном случае, отдельная проблема (например, [404104 DeviceConnectionClosedRemoteRemoteLy)](iot-hub-troubleshoot-error-404104-deviceconnectionclosedremotely.md)вызывает устройство отключиться. Устройство пытается восстановить соединение немедленно, но IoT Концентронпов по-прежнему считает устройство подключенным. Центр Интернета вещей закрывает предыдущее подключение и записывает эту ошибку.

### <a name="cause-2"></a>Причина 2

Неисправная логика на стороне устройства заставляет устройство устанавливать соединение, когда он уже открыт.

## <a name="solution"></a>Решение

Эта ошибка обычно возникает как побочный эффект другой временной проблемы, поэтому рекомендуем проверять журналы на наличие других ошибок для более эффективного устранения неполадок. В противном случае выполняйте новый запрос на подключение только при разрыве подключения.

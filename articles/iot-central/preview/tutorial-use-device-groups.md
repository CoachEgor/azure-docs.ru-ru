# <a name="tutorial-use-device-groups-to-analyze-device-telemetry-preview-features"></a>Учебник. Использование групп устройств для анализа телеметрии устройств (предварительные версии компонентов)

[!INCLUDE [iot-central-pnp-original](../../../includes/iot-central-pnp-original-note.md)]

В этой статье описывается, как оператор, чтобы использовать группы устройств для анализа телеметрии устройств в приложении IoT Central Azure.

Группа устройств — это список устройств, сгруппированных вместе, так как они соответствуют некоторым указанным критериям. Группы устройств помогают управлять, визуализировать и анализировать устройства в масштабе, объединяя устройства в логические группы. Например, можно создать группу устройств, чтобы получить список всех устройств кондиционирования воздуха в Сиэтле, чтобы дать специалисту возможность найти устройства, на которых они отвечают.

Из этого руководства вы узнаете, как выполнять следующие задачи:

> [!div class="checklist"]
> * Создание группы устройств
> * Использование группы устройств для анализа телеметрии устройства

## <a name="prerequisites"></a>Технические условия

Перед началом работы вам необходимо завершить два предыдущих кратких руководства по [созданию приложения Azure IoT Central](./quick-deploy-iot-central.md) и [добавлению имитированного устройства в приложение IoT Central](./quick-create-pnp-device.md), чтобы создать шаблон устройства **Датчик окружающей среды** и работать с ним.

## <a name="create-simulated-devices"></a>Создание виртуальных устройств

Перед созданием группы устройств добавьте по крайней мере пять виртуальных устройств из шаблона устройства **датчика среды** для использования в этом руководстве:

![Пять моделируемых устройств датчиков окружающей среды](./media/tutorial-use-device-groups/simulated-devices.png)

Для четырех устройств датчиков окружающей среды используйте представление " **Свойства датчика окружающей среды** ", чтобы задать имя клиента **contoso**:

![Задать имя клиента Contoso](./media/tutorial-use-device-groups/customer-name.png)

## <a name="create-a-device-group"></a>Создание группы устройств

Чтобы создать группу устройств, выполните следующие действия.

1. В области слева выберите **группы устройств** .

1. Выберите **+ Создать**.

    ![Создать группу устройств](media/tutorial-use-device-groups/image1.png)

1. Присвойте группе устройств имя, например **устройства Contoso**. Вы также можете добавить описание. Группа устройств может содержать только устройства из одного шаблона устройства. Выберите шаблон устройства " **датчик окружающей среды** ", который будет использоваться для этой группы.

1. Создайте запрос для обнаружения устройств, принадлежащих **компании Contoso** , для группы устройств, выбрав свойство **Customer Name** , оператор сравнения **Equals** и **contoso** в качестве значения. Вы можете добавить несколько запросов и устройств, соответствующих **всем** критериям, в группу устройств. Созданная группа устройств доступна всем, кто имеет доступ к приложению, чтобы любой пользователь мог просматривать, изменять или удалять группу устройств.

    ![Запрос группы устройств](media/tutorial-use-device-groups/image2.png)

    > [!NOTE]
    > Группа устройств является динамическим запросом. Каждый раз, когда вы просматриваете список устройств, в нем могут содержаться разные устройства. Список зависит от того, какие устройства в настоящее время соответствуют критериям запроса.

1. Нажмите **Сохранить**.

> [!NOTE]
> Для Azure IoT Edge устройств выберите шаблон Azure IoT Edge, чтобы создать группу устройств.

## <a name="analytics"></a>Analytics

**Аналитику** можно использовать с группой устройств для анализа данных телеметрии с устройств в группе. Например, можно построить график средней температуры, сообщаемой всеми датчиками окружающей среды Contoso.

Чтобы проанализировать данные телеметрии для группы устройств, выполните следующие действия.

1. На левой панели выберите **аналитика** .

1. Выберите созданную группу устройств **contoso Devices** . Затем добавьте типы телеметрии **температуры** и **влажности** :

    ![Создать аналитику](./media/tutorial-use-device-groups/create-analysis.png)

    Используйте значки шестеренки рядом с типами телеметрии, чтобы выбрать тип статистической обработки. Значение по умолчанию — **Average**. Чтобы изменить способ отображения статистических данных, используйте метод **Split By** . Например, если разделить по ИДЕНТИФИКАТОРу устройства, то при выборе **анализа**вы увидите график для каждого устройства.

1. Выберите **анализ** , чтобы просмотреть среднее значение телеметрии:

    ![Анализ представления](./media/tutorial-use-device-groups/view-analysis.png)

    Вы можете настроить представление, изменить показанный период времени и экспортировать данные.

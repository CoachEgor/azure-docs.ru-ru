# <a name="configure-the-application-dashboard"></a>Настройка панели мониторинга приложения

**Панель мониторинга** — это страница, которая загружается, когда пользователи, имеющие доступ к приложению, обращаются к URL-адресу приложения. Если приложение создано из одного из **шаблонов**, приложение будет иметь предварительно определенную панель мониторинга для запуска. Если вы создали приложение из шаблона приложения **прежних версий** , панель мониторинга будет пуста для запуска.

> [!NOTE]
> В дополнение к панели мониторинга приложения по умолчанию пользователи могут [создавать несколько панелей мониторинга](howto-create-personal-dashboards.md) . Эти панели мониторинга могут быть личными только для пользователя или общими для всех пользователей приложения. 

## <a name="add-tiles"></a>Добавление плиток

На следующем снимке экрана показана панель мониторинга в приложении, созданном из шаблона **пользовательского приложения** . Чтобы настроить панель мониторинга по умолчанию для приложения, выберите **изменить** в верхней левой части страницы.

> [!div class="mx-imgBorder"]
> ![панели мониторинга для приложений на основе шаблона "пример Contoso"](media/howto-add-tiles-to-your-dashboard/dashboard-sample-contoso.png)

При выборе команды **Правка** открывается панель Библиотека панели мониторинга. Библиотека содержит плитки и примитивы панели мониторинга, которые можно использовать для настройки панели мониторинга.

> [!div class="mx-imgBorder"]
> Библиотека ![ной панели мониторинга](media/howto-add-tiles-to-your-dashboard/dashboard-library.png)

Например, можно добавить плитку **телеметрии** для текущей температуры устройства. Для этого:
1. Выберите **шаблон устройства**
1. Выберите **экземпляр устройства** для устройства, которое вы хотите просмотреть на плитке панели мониторинга. Затем вы увидите список свойств устройства, которые можно использовать на плитке.
1. Чтобы создать плитку на панели мониторинга, щелкните **температуру** и перетащите ее в область панель мониторинга. Можно также щелкнуть флажок рядом с полем **температура** и выбрать команду **объединить**. На следующем снимке экрана показано, как выбрать шаблон устройства и экземпляр устройства, а затем создать плитку телеметрии температуры на панели мониторинга.
1. Выберите **сохранить** в верхнем левом углу, чтобы сохранить плитку на панели мониторинга.

> [!div class="mx-imgBorder"]
> !["Настройка сведений об устройстве" с подробными сведениями о параметрах и свойствах](media/howto-add-tiles-to-your-dashboard/device-details.png)

Теперь, когда оператор просматривает панель мониторинга приложения по умолчанию, он увидит новую плитку с **температурой** для устройства. У каждой плитки есть предварительно выбранный график, диаграмма и т. д., которые будут отображаться при создании плитки. Однако пользователи могут изменить и изменить эту визуализацию. 

> [!div class="mx-imgBorder"]
> ![вкладку "панель мониторинга" с отображаемыми параметрами и свойствами плитки](media/howto-add-tiles-to-your-dashboard/settings-and-properties.png)


## <a name="edit-tiles"></a>Изменить плитки

Чтобы изменить плитку на панели мониторинга, сначала щелкните **изменить** в верхней левой части страницы, чтобы открыть режим редактирования для панели мониторинга и всех ее плиток. 

> [!div class="mx-imgBorder"]
> экран панели мониторинга ![с режимом редактирования, активированным для выбранной плитки](media/howto-add-tiles-to-your-dashboard/edit-mode.png)

Затем щелкните значок **шестеренки** в правом верхнем углу плитки, которую вы хотите изменить. Здесь можно редактировать аспекты плитки, включая ее название, визуализацию, агрегирование и т. д.

> [!div class="mx-imgBorder"]
> раскрывающийся список ![для параметров агрегирования плиток](media/howto-add-tiles-to-your-dashboard/aggregation-settings.png)

Можно также изменить визуализацию диаграммы, щелкнув значок **линейки** на плитке.

> [!div class="mx-imgBorder"]
> раскрывающийся список ![для параметров визуализации плитки](media/howto-add-tiles-to-your-dashboard/visualization-settings.png)

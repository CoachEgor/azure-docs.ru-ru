# <a name="manage-users-and-roles-in-your-iot-central-application"></a>Управление пользователями и ролями в приложении IoT Central

В этой статье описывается, как, будучи администратором, можно добавлять, отсылать и удалять пользователей в приложении Azure IoT Central. В статье также описывается, как управлять ролями в приложении Azure IoT Central.

Чтобы получить доступ к разделу **Администрирование**, вам должна быть назначена роль **Администратор** приложения Azure IoT Central. При создании центрального приложения Azure IoT автоматически добавляется в роль **администратора** для этого приложения.

## <a name="add-users"></a>Добавление пользователей

Каждый пользователь должен иметь учетную запись, чтобы войти в систему и получить доступ к приложению Azure IoT Central. Учетные записи Майкрософт и учетные записи активных каталогов Azure поддерживаются в Azure IoT Central. Группы Azure Active Directory сейчас не поддерживаются в Azure IoT Central.

Дополнительные сведения см. на странице [справки по учетной записи Майкрософт](https://support.microsoft.com/products/microsoft-account?category=manage-account) и в статье [Краткое руководство по добавлению новых пользователей в Azure Active Directory](https://docs.microsoft.com/azure/active-directory/add-users-azure-active-directory).

1. Чтобы добавить пользователя в приложение IoT Central, в разделе **Администрирование** перейдите на страницу **Пользователи**.
    
    > [!div class="mx-imgBorder"]
    >![Управление пользователями](media/howto-manage-users-roles/manage-users-pnp.png)

1. На странице **Пользователи** выберите **+Добавить пользователя**.

1. Выберите роль в раскрывающемся списке **Роль**. Дополнительные сведения о ролях см. в разделе [Управление ролями](#manage-roles) этой статьи.

    > [!div class="mx-imgBorder"]
    >![Добавление пользователя и выбрать роль](media/howto-manage-users-roles/add-user-pnp.png)

    > [!NOTE]
    > Пользователь, который находится в пользовательской роли, которая предоставляет им разрешение на добавление других пользователей, может только добавить пользователей к роли с тем и тем же или меньшим количеством разрешений, чем их собственная роль.

Если идентификатор пользователя IoT Central будет удален из каталога Azure Active Directory, а затем восстановлен, пользователь не сможет войти в приложение IoT Central. Чтобы повторно включить доступ, центральный администратор IoT должен удалить и прочитать пользователя в приложении.

### <a name="edit-the-roles-that-are-assigned-to-users"></a>Изменение ролей, назначенных пользователям

Роли не могут быть изменены после их присвоения. Чтобы изменить роль, назначенную пользователю, удалите учетную запись и повторно добавьте пользователя с другой ролью.

> [!NOTE]
> Назначенные роли специфичны для центрального приложения IoT и не могут управляться с портала Azure.

## <a name="delete-users"></a>Удаление пользователей

Чтобы удалить пользователей, установите один или несколько флажков на странице **Пользователи**. Затем выберите **Удалить**.

## <a name="manage-roles"></a>Управление ролями

Роли позволяют контролировать, кому в вашей организации разрешено выполнять различные задачи в IoT Central. Есть три встроенные роли, которые можно назначить пользователям приложения. Вы также можете [создавать пользовательские роли,](#create-a-custom-role) если требуется более тонкий контроль.

> [!div class="mx-imgBorder"]
> ![Управление выбором ролей](media/howto-manage-users-roles/manage-roles-pnp.png)

### <a name="administrator"></a>Администратор

Пользователи в роли **Администратора** могут управлять и контролировать каждую часть приложения, включая выставление счетов.

Пользователю, создавшему приложение, автоматически назначается роль **администратора**. В роли **администратора** должен быть хотя бы один пользователь.

### <a name="builder"></a>Конструктор

Пользователи в роли **Builder** могут управлять каждой частью приложения, но не могут вносить изменения во вкладки «Администрация» или «Непрерывный экспорт данных».

### <a name="operator"></a>Оператор

Пользователи в роли **Оператора** могут следить за работоспособностью и состоянием устройства. Они не могут вносить изменения в шаблоны устройств или администрировать приложение. Операторы могут добавлять и удалять устройства, управлять наборами устройств, а также запускать аналитику и задания. 

## <a name="create-a-custom-role"></a>Создание настраиваемой роли

Если ваше решение требует тонкого управления доступом, можно создавать пользовательские роли с пользовательскими наборами разрешений. Чтобы создать пользовательскую роль, перейдите на страницу **ролей** в разделе **Администрация** приложения. Затем выберите **новую роль**и добавьте имя и описание для вашей роли. Выберите разрешения, необходимые для вашей роли, а затем выберите **Сохранить.**

Вы можете добавить пользователей к вашей пользовательской роли таким же образом, как вы добавляете пользователей в встроенную роль.

> [!div class="mx-imgBorder"]
> ![Создание пользовательской роли](media/howto-manage-users-roles/create-custom-role-pnp.png)

### <a name="custom-role-options"></a>Пользовательские параметры роли

При определении пользовательской роли вы выбираете набор разрешений, которые предоставляется пользователю, если он является участником роли. Некоторые разрешения зависят от других. Например, при добавлении разрешения на работу **панели мониторинга приложения Обновления** в роль автоматически добавляется разрешение на панели **мониторинга приложения View.** В следующих таблицах суммируются доступные разрешения и их зависимости, которые можно использовать при создании пользовательских ролей.

# <a name="stay-connected-with-azure-iot-central"></a>Оставайтесь на связи с Azure IoT Central

В этой статье приводятся общие сведения о том, как Azure IoT Central помогает в подключении и работоспособности устройств.

Благодаря любому решению IoT, разработанному для масштабирования, важно хорошо структурированный подход к управлению устройствами. Недостаточное только для того, чтобы устройства были подключены к, поэтому необходим подход к подключению и работоспособности устройств по мере развития, роста и возраста вашего решения. Azure IoT Central предоставляет необходимые возможности, чтобы гарантировать, что устройства в решении IoT хорошо таком в течение всего жизненного цикла приложения.

## <a name="dashboards"></a>Панели мониторинга 
Встроенные [панели мониторинга](howto-manage-devices.md#import-devices) предоставляют настраиваемую контактную зону для мониторинга работоспособности и телеметрии устройств. Начните с предварительно созданной панели мониторинга из [шаблона приложения](howto-use-app-templates-pnp.md) или создайте собственные панели мониторинга, адаптированные к потребностям приложения. Панели мониторинга могут совместно использоваться со всеми пользователями в приложении или оставаться закрытыми.

## <a name="rules-and-actions"></a>Правила и действия 
Быстро выявление устройств, требующих внимания, путем создания [настраиваемых правил](howto-create-event-rules.md) на основе состояния и телеметрии устройства. Настройка [действий](howto-create-event-rules.md#configure-actions) для уведомления правильных пользователей о том, что меры по исправлению происходят своевременно.

## <a name="jobs"></a>Задания 
[Задания](howto-run-a-job.md) позволяют выполнять единое или групповое обновление свойств, параметров и команд устройства. 

## <a name="user-roles-and-permissions"></a>Роли и разрешения пользователей
[Роли и разрешения](howto-manage-users-roles-pnp.md) дают возможность адаптировать работу каждого пользователя. Создавайте роли непосредственно через пользовательский интерфейс и легко назначайте соответствующие разрешения.

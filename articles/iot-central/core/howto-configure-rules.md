# <a name="configure-rules"></a>Настройка правил



*Эта статья предназначена для операторов, разработчиков и администраторов.*

Правила в IoT Central служат настраиваемым инструментом реагирования, который запускает активно контролируемые события с подключенных устройств. В следующих разделах описывается, как оцениваются правила.

## <a name="select-target-devices"></a>Выберите целевые устройства

Используйте раздел целевых устройств, чтобы выбрать, на каких устройствах будет применяться это правило. Фильтры позволяют дополнительно уточнить, какие устройства должны быть включены. Фильтры используют свойства на шаблоне устройства для фильтрации набора устройств. Фильтры сами по себе не вызывают действия. В следующем скриншоте, устройства, которые в настоящее время целевых имеют шаблон устройства типа **холодильника.** Фильтр гласит, что правило должно включать только **холодильники,** где **изготовленная государственная** собственность равна **Вашингтону.**

![Условия](media/howto-configure-rules/filters.png)

## <a name="use-multiple-conditions"></a>Использование нескольких условий

Условия - это то, что срабатывают правила. В настоящее время, когда вы добавляете несколько условий к правилу, они логически and'd вместе. Другими словами, необходимо выйти на все условия, чтобы правило оценивалось как истинное.  

На следующем скриншоте условия проверяют, когда температура превышает&deg; 70 F, а влажность менее 10. Когда оба эти утверждения верны, правило оценивается как истинное и запускает действие.

![Условия](media/howto-configure-rules/conditions.png)

## <a name="use-aggregate-windowing"></a>Использование агрегированного оконного окна

Правила оценивают агрегированные временные окна как акробатики окон. На скриншоте ниже, временное окно составляет пять минут. Каждые пять минут правило оценивает последние пять минут данных. Данные оцениваются только один раз в окне, к которому они соответствуют.

![Переворачивающиеся окна](media/howto-configure-rules/tumbling-window.png)

## <a name="use-rules-with-iot-edge-modules"></a>Использование правил с модулями IoT Edge

Ограничение распространяется на правила, применяемые к модулям IoT Edge. Правила телеметрии из разных модулей не оцениваются как действительные правила. Возьмем в качестве примера следующее. Первое условие правила - на температурной телеметрии из модуля А. Второе условие правила - телеметрия влажности на модуле В. Поскольку эти два условия относятся к разным модулям, это недействительный набор условий. Правило не является действительным и будет бросать ошибку при попытке сохранить правило.

# <a name="tutorial-use-device-groups-to-analyze-device-telemetry"></a>Учебник: Использование групп устройств для анализа телеметрии устройства

В этой статье описывается, как, будучи оператором, использовать группы устройств для анализа телеметрии устройств в приложении Azure IoT Central.

Группа устройств представляет собой список устройств, которые сгруппированы вместе, поскольку они соответствуют определенным критериям. Группы устройств помогают управлять, визуализировать и анализировать устройства в масштабе, группируя устройства на более мелкие логические группы. Например, можно создать группу устройств для включения всех устройств кондиционеров в Сиэтле, чтобы специалист мог найти устройства, за которые они отвечают.

В этом руководстве описано следующее:

> [!div class="checklist"]
> * Создание группы устройств
> * Используйте группу устройств для анализа телеметрии устройства

## <a name="prerequisites"></a>Предварительные требования

Перед началом работы вам необходимо пройти два предыдущих кратких руководства по [созданию приложения Azure IoT Central](./quick-deploy-iot-central.md) и [добавлению имитированного устройства в приложение IoT Central](./quick-create-pnp-device.md), чтобы создать шаблон устройства **MXChip IoT DevKit** и работать с ним.

## <a name="create-simulated-devices"></a>Создание смоделированных устройств

Прежде чем создать группу устройств, добавьте по крайней мере пять смоделированных устройств из шаблона устройства **MXChip IoT DevKit** для использования в этом учебнике:

![Пять смоделированных сенсорных устройств](./media/tutorial-use-device-groups/simulated-devices.png)

Для четырех смоделированных сенсорных устройств используйте представление **устройства Управления,** чтобы установить имя клиента *в Contoso:*

![Установите имя клиента в Contoso](./media/tutorial-use-device-groups/customer-name.png)

## <a name="create-a-device-group"></a>Создание группы устройств

Для создания группы устройств:

1. Выберите **группы устройств** на левом стеле.

1. Выберите: **+**

    ![Новая группа устройств](media/tutorial-use-device-groups/image1.png)

1. Назовите группу устройств *Contoso.* Вы также можете добавить описание. Группа устройств может содержать только устройства из одного шаблона устройства. Выберите шаблон устройства **MXChip IoT DevKit** для использования в этой группе.

1. Чтобы настроить группу устройств, чтобы включить только устройства, принадлежащие **Contoso,** выберите **фильтр.** Выберите свойство **имени клиента,** оператора сравнения **равных** и **Contoso** в качестве значения. Можно добавить несколько фильтров и устройств, отвечающих **всем** критериям фильтра, в группе устройств. Созданная вами группа устройств доступна любому, кто имеет доступ к приложению, поэтому любой желающий может просмотреть, изменить или удалить группу устройств:

    ![Групповый запрос устройства](media/tutorial-use-device-groups/image2.png)

    > [!TIP]
    > Группа устройств представляет собой динамический запрос. Каждый раз, когда вы просматриваете список устройств, в нем могут содержаться разные устройства. Список зависит от того, какие устройства в настоящее время соответствуют критериям запроса.

1. Нажмите **Сохранить**.

> [!NOTE]
> Для устройств Azure IoT Edge выберите шаблоны Azure IoT Edge для создания группы устройств.

## <a name="analytics"></a>Analytics

Для анализа телеметрии с устройств в группе можно использовать **Analytics** с группой устройств. Например, можно построить среднюю температуру, о которых сообщают все датчики окружающей среды Contoso.

Для анализа телеметрии для группы устройств:

1. Выберите **Аналитика** на левом стеле.

1. Выберите созданную **группу устройств Contoso.** Затем добавьте как типы телеметрии **температуры** и **влажности:**

    ![Создание аналитики](./media/tutorial-use-device-groups/create-analysis.png)

    Используйте значки колес рядом с типами телеметрии, чтобы выбрать тип агрегации. По умолчанию **средний**. Используйте **Split,** чтобы изменить способ отображаемого совокупного объема данных. Например, если вы разделились на идентификатор устройства, вы увидите участок для каждого устройства при выборе **анализа.**

1. Выберите **анализ** для просмотра средних значений телеметрии:

    ![Анализ просмотра](./media/tutorial-use-device-groups/view-analysis.png)

    Можно настроить представление, изменить отображаемый период времени и экспортировать данные.

# <a name="manage-devices-in-your-azure-iot-central-application"></a>Управление устройствами в приложении Azure IoT Central



В этой статье описывается, как оператор может управлять устройствами в приложении Microsoft Azure IoT Central. Как оператор вы можете:

- Используйте страницу **устройств** для просмотра, добавления и удаления устройств, подключенных к приложению Azure IoT Central.
- Поддерживать актуальность сведений об используемых устройствах.
- Поддерживайте обновляемое метаданные устройства, изменяя значения, хранящиеся в свойствах устройства, из ваших представлений.
- Контролируйте поведение устройств, обновляя настройки на определенном устройстве из ваших представлений.

## <a name="view-your-devices"></a>Просмотр устройств

Чтобы просмотреть отдельное устройство, сделайте следующее:

1. Выберите **устройства** на левом стеле. Здесь вы видите список всех устройств и шаблонов вашего устройства.

1. Выберите шаблон устройства.

1. На правой панели **страницы «Устройства»** вы видите список устройств, созданных из шаблона этого устройства. Выберите одно устройство, чтобы просмотреть страницу сведений о нем:

    ![Страница сведений об устройстве](./media/howto-manage-devices/devicelist.png)


## <a name="add-a-device"></a>Добавление устройства

Чтобы добавить устройство в приложение Azure IoT Central, сделайте следующее:

1. Выберите **устройства** на левом стеле.

1. Выберите шаблон устройства, на основе которого требуется создать устройство.

1. Выберите **+ Создать**.

1. Включите **имитацию** переключения **в Включить** или **выключить**. Под реальным устройством подразумевается физическое устройство, подключаемое к приложению Azure IoT Central. Имитированное устройство содержит образцы данных, созданные в Azure IoT Central.

1. Нажмите кнопку **Создать**.

1. Это устройство теперь отображается в списке устройств для этого шаблона. Выберите устройство, чтобы увидеть страницу данных устройства, содержащую все представления для устройства.

## <a name="import-devices"></a>Импорт устройств

Для подключения к приложению большого количества устройств вы можете выполнить массовый импорт устройств с помощью CSV-файла. CSV-файл должен содержать следующие столбцы и заголовки:

* **IOTC_DeviceID** — код устройства должен состоять только из знаков нижнего регистра.
* **IOTC_DeviceName** — этот столбец является необязательным.

Для массовой регистрации устройств в приложении сделайте следующее:

1. Выберите **устройства** на левом стеле.

1. На левой панели выберите шаблон устройства, для которого требуется выполнить массовое создание устройств.

    > [!NOTE]
    > Если у вас еще нет шаблона устройства, то вы можете импортировать устройства под **всеми устройствами** и зарегистрировать их без шаблона. После импорта устройств можно перенести их в шаблон.

1. Выберите **Импортировать**.

    ![Действие импорта](./media/howto-manage-devices/bulkimport1a.png)


1. Выберите CSV-файл, содержащий список идентификаторов устройств, которые нужно импортировать.

1. Импорт устройств начнется сразу после передачи файла. Можно отслеживать состояние импорта в панели операций с устройствами. Эта панель появляется автоматически после начала импорта или вы можете получить доступ к ней через значок колокола в правом верхнем углу.

1. После завершения импорта в панели операций устройств отображается сообщение об успешном показе.

    ![Успешное завершение импорта](./media/howto-manage-devices/bulkimport3a.png)


Если операция импорта устройства не сработала, на панели операций с устройством можно увидеть сообщение об ошибке. Создается файл журнала, содержащий все ошибки. Вы можете скачать этот файл.

**Миграция устройств в шаблон**

Если вы регистрируете устройства, запустив импорт под **всеми устройствами,** то устройства создаются без какой-либо ассоциации шаблонов устройства. Для просмотра данных и других сведений об устройстве устройства необходимо связать с шаблоном. Чтобы связать устройства с шаблоном, выполните следующие действия:

1. Выберите **устройства** на левом стеле.

1. На левой панели выберите **все устройства:**

    ![Несвязанные устройства](./media/howto-manage-devices/unassociateddevices1a.png)


1. Используйте фильтр на сетке, чтобы определить, является ли значение в столбце **шаблона устройства** «несвязанным» для любого из ваших устройств.

1. Выберите устройства, которые требуется связать с шаблоном.

1. Выберите **Мигрировать:**

    ![Связывание устройств](./media/howto-manage-devices/unassociateddevices2a.png)


1. Выберите шаблон из списка доступных шаблонов и выберите **Migrate.**

1. Необходимые устройства связаны с выбранным шаблоном устройства.


## <a name="export-devices"></a>Экспорт устройств

Чтобы подключить реальное устройство к IoT Central, необходимо наличие его строки подключения. Вы можете экспортировать данные об устройстве оптом, чтобы получить информацию, необходимую для создания строк соединения устройства. Процесс экспорта создает файл CSV с идентификатором устройства, именем устройства и ключами для всех выбранных устройств.

Чтобы выполнить массовый экспорт устройств из приложения, сделайте следующее:

1. Выберите **устройства** на левом стеле.

1. На левой панели выберите шаблон устройства, из которого вы хотите экспортировать устройства.

1. Выберите устройства, которые вы хотите экспортировать, а затем выберите действие **Экспорт.**

    ![Экспорт](./media/howto-manage-devices/export1a.png)


1. Начнется процесс экспорта. Вы можете отслеживать состояние с помощью панели операций устройства.

1. После завершения экспорта отобразится сообщение об успешном выполнении и ссылка для скачивания созданного файла.

1. Выберите ссылку **Скачать файл,** чтобы загрузить файл в локальную папку на диске.

    ![Успешное завершение экспорта](./media/howto-manage-devices/export2a.png)


1. Экспортированный CSV-файл содержит столбцы со сведениями о коде устройства, его имени, ключах устройства и отпечатках сертификатов X509:

    * IOTC_DEVICEID
    * IOTC_DEVICENAME
    * IOTC_SASKEY_PRIMARY
    * IOTC_SASKEY_SECONDARY
    * IOTC_X509THUMBPRINT_PRIMARY
    * IOTC_X509THUMBPRINT_SECONDARY

Для получения дополнительной информации о строках подключения и подключении реальных устройств к приложению IoT Central см. подключение [устройств в Azure IoT Central.](concepts-get-connected.md)

## <a name="delete-a-device"></a>Удаление устройства

Чтобы удалить реальное или имитированное устройство из приложения Azure IoT Central, сделайте следующее:

1. Выберите **устройства** на левом стеле.

1. Выберите шаблон устройства, которое требуется удалить.

1. Используйте фильтры для фильтрации и поиска устройств. Проверьте окно рядом с устройствами, чтобы удалить.

1. Щелкните **Удалить**. Вы можете отслеживать состояние этого удаления в панели операций устройств.

## <a name="change-a-property"></a>Изменение свойства

Облачные свойства — это метаданные устройства, связанные с устройством, такие как городской и серийный номер. Записываемые свойства контролируют поведение устройства. Другими словами, параметры позволяют предоставлять входные данные устройству.  Свойства устройства устанавливаются устройством и считываются только в IoT Central. Вы можете просматривать и обновлять свойства в **представлениях данных устройств** для вашего устройства.

1. Выберите **устройства** на левом стеле.

1. Выберите шаблон устройства, свойства которого вы хотите изменить, и выберите целевое устройство.

1. Выберите представление, содержащее свойства для вашего устройства, это представление позволяет ввести значения и выбрать **Сохранить** в верхней части страницы. Здесь вы видите свойства устройства и их текущие значения. Свойства облака и читаемые свойства имеют редитативные поля, в то время как свойства устройств только считываются. Для свойств, подаваемых для записи, можно увидеть их состояние синхронизации в нижней части поля. 

1. Измените свойства в нужные значения. Вы можете изменять несколько свойств одновременно и обновлять их одновременно.

1. Нажмите **Сохранить**. При сохранении свойств, пригодных для записи, значения отправляются на устройство. Когда устройство подтверждает изменение для свойств, подаваемое записью, статус возвращается обратно **в синхронизацию.** При сохранении облачного свойства значение обновляется.

# <a name="create-and-run-a-job-in-your-azure-iot-central-application"></a>Создание и запуск задания в приложении Azure IoT Central

Вы можете использовать Microsoft Azure IoT Central для управления подключенными устройствами в масштабе с использованием заданий. Задания позволяют выполнять оптовые обновления для свойств устройств и выполнять команды. Эта статья показывает вам, как начать использовать рабочие места в вашем собственном приложении.

## <a name="create-and-run-a-job"></a>Создание и запуск заданий

В этом разделе показано, как создать и запустить задание. Он показывает, как установить порог света для группы логистических устройств шлюза.

1. Перейдите к **вакансиям** из левого стекла.

2. Выберите **новое для** создания нового задания:

    ![Создание задания](./media/howto-run-a-job/createnewjob.png)

3. Введите имя и описание, чтобы определить работу, которую вы создаете.

4. Выберите группу целевого устройства, к ней вы хотите, чтобы ваша работа была применена. Вы можете увидеть, на сколько устройств в конфигурации задания применяется в разделе **Резюме.**

5. Далее выберите свойство **облако,** **свойство** или **команду** в качестве типа задания для настройки. Чтобы настроить конфигурацию задания **свойств,** выберите свойство и установите его новое значение. Чтобы настроить **команду,** выберите команду для выполнения. Задание свойства может установить несколько свойств:

    ![Настройка задания](./media/howto-run-a-job/configurejob.png)

6. После создания вашей работы, выберите **Выполнить** или **сохранить**. Задание теперь отображается на вашей главной странице **Вакансии.** На этой странице можно увидеть работу, работая в настоящее время, и историю любых ранее запущенных или сохраненных заданий. Сохраненное задание может быть открыто снова в любое время, чтобы продолжить редактирование или запустить ее:

    ![Просмотр задания](./media/howto-run-a-job/viewjob.png)

    > [!NOTE]
    > Вы можете просмотреть 30 дней истории для ранее запущенных заданий.

7. Чтобы получить обзор вашей работы, выберите задание для просмотра из списка. Этот обзор содержит данные о заданиях, устройствах и значениях состояния устройства. Из этого обзора вы также можете выбрать **скачать данные о работе,** чтобы загрузить файл CSV ваших рабочих данных, включая устройства и их значения статуса. Эта информация может быть полезна для устранения неполадок:

    ![Просмотр состояния устройства](./media/howto-run-a-job/downloaddetails.png)

### <a name="manage-a-job"></a>Управление заданием

Чтобы остановить одну из рабочих мест, откройте его и выберите **Stop.** Изменение статуса задания с учетом задания остановлено. Раздел **Резюме** показывает, какие устройства завершены, сбой или еще находятся на рассмотрении.

Чтобы выполнить задание, которое в настоящее время остановлено, выберите его, а затем выберите **Выполнить.** Изменение статуса задания, отражающее задание, теперь снова запущено. Раздел **Резюме** продолжает обновляться с последними достижениями.

![Управление заданием](./media/howto-run-a-job/managejob.png)

## <a name="copy-a-job"></a>Копирование задания

Чтобы скопировать одну из существующих заданий, выберите его на странице **Вакансии** и выберите **Copy**. Копия конфигурации задания открывается для отправления, и **копия** придатится к названию задания. Вы можете сохранить или запустить новую работу:

![Копирование задания](./media/howto-run-a-job/copyjob.png)

## <a name="view-the-job-status"></a>Просмотр состояния задания

После создания задания столбец **«Статус»** обновляется с последним сообщением о состоянии задания. В следующей таблице перечислены возможные значения статуса:

| Сообщение о состоянии       | Значение состояния                                          |
| -------------------- | ------------------------------------------------------- |
| Завершено            | Это задание было выполнено на всех устройствах.              |
| Ошибка               | Это задание завершилось сбоем и выполнено не полностью на устройствах.  |
| Ожидает              | Эта работа еще не началась выполнения на устройствах.         |
| Запущен              | Это задание в данный момент выполняется на устройствах.             |
| Остановлена              | Это задание вручную остановлено пользователем.           |

Сообщение о состоянии сопровождается обзором устройств в заданиях. В следующей таблице перечислены возможные значения состояния устройства:

| Сообщение о состоянии       | Значение состояния                                                     |
| -------------------- | ------------------------------------------------------------------ |
| Выполнено            | Количество устройств, на которые успешно выполнено задание.       |
| Ошибка               | Количество устройств, которые задание не выполнено.       |

### <a name="view-the-device-status"></a>Просмотр состояния устройства

Чтобы просмотреть состояние задания и всех затронутых устройств, откройте задание. Чтобы загрузить файл CSV, который включает в себя сведения о работе, включая список устройств и значения их статуса, выберите **сведения о работе Download.** Рядом с именем каждого устройства вы видите одно из следующих сообщений о состоянии:

| Сообщение о состоянии       | Значение состояния                                                                |
| -------------------- | ----------------------------------------------------------------------------- |
| Завершено            | Это задание было выполнено на этом устройстве.                                     |
| Ошибка               | Это задание завершилось сбоем на этом устройстве. Сообщение об ошибке показывает больше информации.  |
| Ожидает              | Задание еще не выполнено на этом устройстве.                                   |

> [!NOTE]
> Если устройство было удалено, вы не можете выбрать устройство. Он отображается как удаленный с идентификатором устройства.

# <a name="export-your-application"></a>Экспорт приложения



В этой статье описывается, как, как менеджер решений, экспортировать приложение IoT Central, чтобы иметь возможность повторно использовать его.

Имеются две возможности.

- Вы можете создать копию приложения, если вам просто нужно создать дубликат приложения.
- Вы можете создать шаблон приложения из приложения, если планируете создать несколько копий.

## <a name="copy-your-application"></a>Копирование приложения

Можно создать копию любого приложения без экземпляров устройства, журнала данных устройства и данных пользователя. Копия использует стандартный план ценообразования, который вам будет выставлен счет. Вы не можете создать приложение, которое использует план свободного ценообразования, копируя приложение.

Нажмите **Копировать**. В диалоговом окне введите детали для нового приложения. Затем выберите **Copy,** чтобы подтвердить, что вы хотите продолжить. Чтобы узнать больше о полях в форме, [см.](quick-deploy-iot-central.md)

![Страница "Параметры приложения"](media/howto-use-app-templates/appcopy2.png)

После того, как операция копирования приложения будет успешной, вы можете перейти к новому приложению, используя ссылку.

![Страница "Параметры приложения"](media/howto-use-app-templates/appcopy3a.png)

Копирование приложения также копирует определение правил и действия по электронной почте. Некоторые действия, такие как flow и Logic Apps, привязаны к определенным правилам через идентификатор правила. Когда правило скопировано в другое приложение, оно получает свой собственный идентификатор правил. В этом случае пользователям придется создать новое действие, а затем связать новое правило с ним. В общем, рекомендуется проверить правила и действия, чтобы убедиться, что они актуальны в новом приложении.

> [!WARNING]
> Если панель мониторинга включает плитки, отображающие информацию о конкретных устройствах, то эти плитки показывают, что **запрошенный ресурс не найден** в новом приложении. Необходимо перенастроить эти плитки для отображения информации об устройствах в новом приложении.

## <a name="create-an-application-template"></a>Создание шаблона приложения

При создании центрального приложения Azure IoT у вас есть выбор встроенных шаблонов образцов. Вы также можете создать свои собственные шаблоны приложений из существующих приложений IoT Central. При создании новых приложений можно использовать собственные шаблоны приложений.

При создании шаблона приложения он включает в себя следующие элементы из существующего приложения:

- Панель мониторинга приложения по умолчанию, включая макет панели мониторинга и все определенные плитки.
- Шаблоны устройств, включая измерения, настройки, свойства, команды и панель мониторинга.
- Правила. Все определения правил включены. Однако действия, за исключением действий электронной почты, не включены.
- Наборы устройств, включая их условия и панели мониторинга.

> [!WARNING]
> Если панель мониторинга включает плитки, отображающие информацию о конкретных устройствах, то эти плитки показывают, что **запрошенный ресурс не найден** в новом приложении. Необходимо перенастроить эти плитки для отображения информации об устройствах в новом приложении.

При создании шаблона приложения не содержится следующих элементов:

- Устройства
- Пользователи
- Определения вакансий
- Непрерывные определения экспорта данных

Добавляйте эти элементы вручную в любые приложения, созданные из шаблона приложения.

Для создания шаблона приложения из существующего приложения IoT Central:

1. Перейти в раздел **Администрации** в вашем приложении.
1. Выберите **экспорт шаблонов приложений**.
1. На странице **Экспорт шаблонов приложений** введите имя и описание шаблона.
1. Выберите кнопку **«Экспорт»** для создания шаблона приложения. Теперь вы можете скопировать **Shareable Link,** которая позволяет кому-то создать новое приложение из шаблона:

![Создание шаблона приложения](media/howto-use-app-templates/create-template.png)

### <a name="use-an-application-template"></a>Использование шаблона приложения

Для использования шаблона приложения для создания нового приложения IoT Central необходимо ранее созданное **Shareable Link.** Вставьте **shareable Link** в адресную строку вашего браузера. **Создать страницу приложения** с выбранным шаблоном пользовательского приложения:

![Создание приложения из шаблона](media/howto-use-app-templates/create-app.png)

Выберите план ценообразования и заполните другие поля на форме. Затем выберите **Создать** для создания нового приложения IoT Central из шаблона приложения.

### <a name="manage-application-templates"></a>Управление шаблонами приложений

На странице **Экспорт шаблонов приложений** можно удалить или обновить шаблон приложения.

При удалении шаблона приложения вы больше не сможете использовать ранее сгенерированную общую ссылку для создания новых приложений.

Чтобы обновить шаблон приложения, измените имя или описание шаблона на странице **Экспорт шаблонов приложений.** Затем выберите кнопку **Экспорт** еще раз. Это действие генерирует новую **ссылку Shareable** и аннулирует любой предыдущий **URL-адрес юаней.**

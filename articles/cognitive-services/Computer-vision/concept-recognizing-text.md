# <a name="recognize-printed-and-handwritten-text"></a>Распознавание печатного и рукописного текста

Компьютерное зрение предоставляет ряд служб, которые обнаруживают и извлекают печатный или рукописный текст, отображаемый в изображениях. Это полезно в различных сценариях, таких как создание примечаний, медицинские записи, безопасность и банковские операции. В следующих трех разделах подробно описаны три разных API распознавания текста, оптимизированные для различных вариантов использования.

## <a name="read-api"></a>API чтения

API чтения определяет текстовое содержимое изображения, используя наши последние модели распознавания, и преобразует обнаруженный текст в поток символов, доступный для машинного чтения. Он оптимизирован для больших объемов текста (например, документов, которые были просканированы в цифровом формате) и для изображений с большим количеством визуальных помех. Он определяет, какую модель распознавания следует использовать для каждой строки текста, и поддерживает изображения с печатным и рукописным текстом. API чтения выполняется асинхронно, так как больше документов может потребоваться несколько минут для возврата результата.

Операция чтения сохраняет исходные группы строк распознанных слов в выходных данных. Каждая строка поступает с координатами ограничивающего прямоугольника, а каждое слово в строке также имеет собственные координаты. Если слово было распознано с низкой достоверностью, эта информация также будет передана. Дополнительные сведения см. в [справочных документах по API чтения](https://westus.dev.cognitive.microsoft.com/docs/services/5adf991815e1060e6355ad44/operations/2afb498089f74080d7ef85eb) .

> [!NOTE]
> Эта функция доступна только для английского текста.

### <a name="image-requirements"></a>Требования к образам

API чтения работает с образами, которые соответствуют следующим требованиям:

- Образ должен быть представлен в формате JPEG, PNG, BMP, PDF или TIFF.
- Размеры изображения должны находиться в диапазоне от 50 x 50 до 10000 x 10000 пикселей. PDF-страницы должны иметь размер 17 x 17 дюймов или меньше.
- Размер файла образа должен быть меньше 20 мегабайт (МБ).

### <a name="limitations"></a>Ограничения

Если вы используете подписку уровня Free, API чтения будет обрабатывать только первые две страницы документа PDF или TIFF. Благодаря платной подписке она будет обрабатывать до 200 страниц. Также обратите внимание, что API определит не более 300 строк на странице.

## <a name="ocr-optical-character-recognition-api"></a>API OCR (оптическое распознавание символов)

API-интерфейс оптического распознавания символов (OCR) Компьютерное зрение аналогичен API чтения, но он выполняется синхронно и не оптимизирован для больших документов. Он использует более раннюю модель распознавания, но работает с другими языками. полный список поддерживаемых языков см. в разделе [Поддержка языков](language-support.md#text-recognition) .

При необходимости OCR исправляет поворот в градусах распознанного текста с помощью смещения относительно горизонтальной оси изображения. OCR также предоставляет координаты каждого слова, как показано на следующем рисунке.

![Поворот изображения и чтение его текста](./Images/vision-overview-ocr.png)

Дополнительные сведения см. в [справочных документах по оптическому распознаванию текста](https://westus.dev.cognitive.microsoft.com/docs/services/5adf991815e1060e6355ad44/operations/56f91f2e778daf14a499e1fc) .

### <a name="image-requirements"></a>Требования к образам

API OCR работает с образами, которые соответствуют следующим требованиям:

* Образ должен быть представлен в формате JPEG, PNG, GIF или BMP.
* Размер входного изображения должен составлять от 50 x 50 до 4200 x 4200 пикселей.
* Текст на изображении можно многократно поворачивать на 90 градусов, а также на небольшой угол в пределах 40 градусов.

### <a name="limitations"></a>Ограничения

На фотографиях, где преобладает текст, может быть ложное срабатывание частично распознанных слов. На некоторых фотографиях, особенно в фотографиях без текста, точность может зависеть от типа изображения.

## <a name="recognize-text-api"></a>Распознавание текста API

> [!NOTE]
> API-интерфейс Распознавание текста устарел в пользу API чтения. API чтения имеет аналогичные возможности и обновляется для работы с файлами PDF, TIFF и многостраничных файлов.

API Распознавание текста похож на OCR, но выполняется асинхронно и использует обновленные модели распознавания. Дополнительные сведения см. в [справочных документах по распознавание текста API](https://westus.dev.cognitive.microsoft.com/docs/services/5adf991815e1060e6355ad44/operations/587f2c6a154055056008f200) .

### <a name="image-requirements"></a>Требования к образам

Распознавание текста API работает с образами, которые соответствуют следующим требованиям:

- Образ должен быть представлен в формате JPEG, PNG или BMP.
- Размеры изображения должны находиться в диапазоне от 50 x 50 до 4200 x 4200 пикселей.
- Размер файла образа должен быть меньше 4 мегабайт (МБ).

## <a name="limitations"></a>Ограничения

Точность операций распознавания текста зависит от качества изображений. Следующие факторы могут привести к неточному чтению:

* Размытость изображения.
* Рукописный текст или курсив.
* Художественные стили шрифтов.
* Мелкий размер текста.
* Сложный фон, тени или отблеск поверх текста, или искажение перспективы.
* Слишком большой или отсутствующий прописные буквы в начале слов.
* Подстрочный, надстрочный или зачеркнутый текст.

# <a name="recognize-printed-and-handwritten-text"></a>Распознавание печатного и рукописного текста

Computer Vision предоставляет ряд услуг, которые обнаруживают и извлекают печатный или рукописный текст, который появляется в изображениях. Это полезно в различных сценариях, таких как заметок, медицинские записи, безопасность и банковское дело. В следующих трех разделах подробно описаны три различных AAP распознавания текста, каждый из которых оптимизирован для различных случаев использования.

## <a name="read-api"></a>Читать API

Read API обнаруживает текстовое содержимое на изображении с помощью наших последних моделей распознавания и преобразует идентифицированный текст в машиночитаемый поток символов. Он оптимизирован для текстовых изображений (таких как документы, которые были отсканированы в цифровом виде) и для изображений с большим количеством визуального шума. Он определит, какую модель распознавания использовать для каждой строки текста, поддерживая изображения как печатным, так и рукописным текстом. API Read выполняется асинхронно, так как большие документы могут занять несколько минут, чтобы вернуть результат.

Операция Read поддерживает исходные линейные группирования признанных слов в своем выходе. Каждая линия поставляется с координатами ящика, и каждое слово в пределах линии также имеет свои координаты. Если слово было признано с низкой уверенностью, эта информация передается также. Дополнительные данные read [API](https://westus.dev.cognitive.microsoft.com/docs/services/5adf991815e1060e6355ad44/operations/2afb498089f74080d7ef85eb) и справочные [документы Read API Preview,](https://go.microsoft.com/fwlink/?linkid=2118322) чтобы узнать больше.

> [!NOTE]
> Эта функция доступна только для текста на английском и испанском языках (предварительный просмотр).

### <a name="image-requirements"></a>Требования к образам

Read API работает с изображениями, которые отвечают следующим требованиям:

- Изображение должно быть представлено в формате JPEG, PNG, BMP, PDF или TIFF.
- Размеры изображения должны быть между 50 x 50 и 10000 x 10000 пикселей. PDF страниц ы должны быть 17 х 17 дюймов или меньше.
- Размер файла изображения должен быть менее 20 мегабайт (МБ).

### <a name="limitations"></a>Ограничения

Если вы используете подписку свободного уровня, API Read будет обрабатывать только первые две страницы документа PDF или TIFF. С платной подпиской, он будет обрабатывать до 200 страниц. Также обратите внимание, что API будет обнаруживать максимум 300 строк на страницу.

## <a name="ocr-optical-character-recognition-api"></a>API OCR (оптическое распознавание символов)

Оптическое распознавание символов Computer Vision (OCR) API аналогично API Read API, но выполняется синхронно и не оптимизировано для больших документов. Он использует более раннюю модель распознавания, но работает с большим количеством языков; [см. поддержку языка](language-support.md#text-recognition) для полного списка поддерживаемых языков.

При необходимости OCR исправляет поворот в градусах распознанного текста с помощью смещения относительно горизонтальной оси изображения. OCR также предоставляет координаты кадров каждого слова, как видно на следующей иллюстрации.

![Изображение, вращаемые и его текст читается и разграничено](./Images/vision-overview-ocr.png)

Подробнее о рекомендациях OCR читайте в [справке OCR.](https://westus.dev.cognitive.microsoft.com/docs/services/5adf991815e1060e6355ad44/operations/56f91f2e778daf14a499e1fc)

### <a name="image-requirements"></a>Требования к образам

API OCR работает над изображениями, которые отвечают следующим требованиям:

* Изображение должно быть представлено в формате JPEG, PNG, GIF или BMP.
* Размер входного изображения должен быть между 50 x 50 и 4200 x 4200 пикселей.
* Текст на изображении можно многократно поворачивать на 90 градусов, а также на небольшой угол в пределах 40 градусов.

### <a name="limitations"></a>Ограничения

На фотографиях, где преобладает текст, может быть ложное срабатывание частично распознанных слов. На некоторых фотографиях, особенно на фотографиях без текста, точность может варьироваться в зависимости от типа изображения.

## <a name="recognize-text-api"></a>Распознать API текста

> [!NOTE]
> API распознавания текста унижается в пользу Read API. API Read имеет аналогичные возможности и обновляется для обработки PDF, TIFF и многостраничных файлов.

API распознавания текста похож на OCR, но он выполняет асинхронно и использует обновленные модели распознавания. Дополнительные документы по [распознаванию API API,](https://westus.dev.cognitive.microsoft.com/docs/services/5adf991815e1060e6355ad44/operations/587f2c6a154055056008f200) чтобы узнать больше.

### <a name="image-requirements"></a>Требования к образам

API распознавания текста работает с изображениями, которые отвечают следующим требованиям:

- Изображение должно быть представлено в формате JPEG, PNG или BMP.
- Размеры изображения должны быть между 50 x 50 и 4200 x 4200 пикселей.
- Размер файла изображения должен быть менее 4 мегабайт (МБ).

## <a name="limitations"></a>Ограничения

Точность операций распознавания текста зависит от качества изображений. Следующие факторы могут привести к неточному чтению:

* Размытость изображения.
* Рукописный текст или курсив.
* Художественные стили шрифтов.
* Мелкий размер текста.
* Сложный фон, тени или отблеск поверх текста, или искажение перспективы.
* Негабаритные или отсутствующие заглавные буквы в начале слов.
* Подстрочный, надстрочный или зачеркнутый текст.

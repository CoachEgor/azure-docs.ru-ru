# <a name="optical-character-recognition-ocr"></a>Оптическое распознавание текста (OCR)

API компьютерного зрения Майкрософт включает возможности оптического распознавания символов (OCR), которые извлекают печатный или рукописный текст из изображений и PDF-документов. Интерфейсы API OCR извлекают текст из аналоговых документов (изображений, отсканированных документов) и документов с цифрами. Вы можете извлекать текст из изображений с подстановочными знаками, например фотографиями лицензионных форм или контейнеров с серийными номерами, а также из документов — счетов, ведомостей, финансовых отчетов, статей и т. д. Новый API OCR для чтения доступен как часть управляемой службы в облаке или локальной среде (контейнерах). Кроме того, она поддерживает виртуальные сети и частные конечные точки в соответствии с потребностями корпоративного уровня и требованиями к конфиденциальности.

## <a name="read-api"></a>API чтения 

[API чтения](https://westcentralus.dev.cognitive.microsoft.com/docs/services/computer-vision-v3-ga/operations/5d986960601faab4bf452005) компьютерное зрение — это новейшая технология OCR, которая извлекает печатный текст на нескольких языках, рукописный текст (только на английском языке), цифры и символы валют из изображений и многостраничных PDF-документов. Он оптимизирован для извлечения текста из готовых текстовых изображений и многостраничных PDF-документов с смешанными языками. Он поддерживает обнаружение печатного и рукописного текста (только на английском языке) в том же изображении или документе. См. полный список [поддерживаемых OCR страниц языков](https://docs.microsoft.com/azure/cognitive-services/computer-vision/language-support#optical-character-recognition-ocr) .

### <a name="how-ocr-works"></a>Как работает OCR

[API чтения](https://westcentralus.dev.cognitive.microsoft.com/docs/services/computer-vision-v3-ga/operations/5d986960601faab4bf452005) поддерживает документы с большим объемом текста до 2000 страниц, поэтому выполняется асинхронно. Первым шагом является вызов операции чтения. Операция чтения принимает изображение или PDF-документ в качестве входных данных и возвращает идентификатор операции. 

Второй шаг — вызов операции [получения результатов](https://westcentralus.dev.cognitive.microsoft.com/docs/services/computer-vision-v3-ga/operations/5d9869604be85dee480c8750) . Эта операция принимает идентификатор операции, созданный операцией чтения. Затем он возвращает извлеченное текстовое содержимое из изображения или документа в формате JSON. Ответ JSON поддерживает первоначальные группы строк распознанных слов. Он включает извлеченные текстовые строки и координаты ограничивающих прямоугольников. Каждая строка текста включает все извлеченные слова с их координатами и рейтингами достоверности.

При необходимости чтение исправляет поворот распознанной страницы, возвращая смещение поворота в градусах относительно горизонтальной оси изображения, как показано на следующем рисунке.

### <a name="input-requirements"></a>Требования к входным данным

API чтения принимает следующие входные данные:
* Поддерживаемые форматы файлов: JPEG, PNG, BMP, PDF и TIFF
* Для документов PDF и TIFF обрабатывается до 2000 страниц. Для подписчиков уровня Free обрабатываются только первые две страницы.
* Размер файла должен быть менее 50 МБ, а размер измерений не менее 50 x 50 пикселей и не более 10000 x 10000 пикселей.
* Размеры PDF-файла должны составлять не более 17 x 17 дюймов, соответствующих юридическим или стандартным размерам бумаги и меньшему размеру.

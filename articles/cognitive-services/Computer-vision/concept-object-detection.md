# <a name="detect-common-objects-in-images"></a>Обнаружение общих объектов на изображениях

Обнаружение объекта похоже на [добавление тегов](concept-tagging-images.md), но в этом случае API возвращает координаты (в пикселях) прямоугольника, ограничивающего каждый найденный объект. Например, если на изображении есть собака, кот и человек, операция определения отобразит список этих объектов с их координатами на изображении. Эту функциональность можно использовать, чтобы обрабатывать отношения между объектами на изображении. Это также позволяет определить, есть ли несколько экземпляров такой же тег изображения.

API обнаружения применяет теги на основе объектов или живых существ, найденных на изображении. В данный момент не формальной связи между таксономию тегов и таксономии обнаружения объекта. На концептуальном уровне API обнаружения находит только те объекты и живых организмов хотя API тега также могут содержать контекстные такие термины, как «внутренний», которые не могут быть локализованы с ограничивающих прямоугольников.

## <a name="limitations"></a>Ограничения

Очень важно Обратите внимание на ограничения обнаружения объекта, что позволяет избежать или устранить последствия ложных отрицательных результатов (пропущенные объекты) и ограниченные сведения.

* Объекты обычно не обнаруживаются, если они небольшой (менее 5% изображения).
* Объекты обычно не обнаруживаются, если идут вместе (стек сверху, например).
* Объекты не различаются по торговой марке или названию (например, различные виды соды на магазинной полке). Тем не менее можно получить сведения о торговых марках на изображении с помощью функции [обнаружения торговых марок](concept-brand-detection.md).

## <a name="use-the-api"></a>Использование API

Функция обнаружения объектов является частью API [анализа изображений](https://westcentralus.dev.cognitive.microsoft.com/docs/services/5adf991815e1060e6355ad44/operations/56f91f2e778daf14a499e1fa). Вы можете вызывать этот API с помощью собственного пакета SDK или с помощью вызовов REST. Включить `Objects` в **visualFeatures** параметр запроса. Затем, когда вы получаете полный ответ JSON, просто синтаксический анализ строки для содержимого `"objects"` раздел.

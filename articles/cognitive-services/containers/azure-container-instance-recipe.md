# <a name="deploy-and-run-container-on-azure-container-instance"></a>Развертывание образа контейнера в экземпляре контейнера

С помощью следующих шагов масштабируйте приложения Azure Cognitive Services в облаке с помощью [экземпляров контейнеров](https://docs.microsoft.com/azure/container-instances/)Azure. Контейнеризация помогает сосредоточиться на создании приложений, а не на управлении инфраструктурой. Для получения дополнительной информации об использовании контейнеров, см [особенности и преимущества](../cognitive-services-container-support.md#features-and-benefits).

## <a name="prerequisites"></a>Предварительные требования

Рецепт работает с любым контейнером Cognitive Services. Ресурс Когнитивной службы должен быть создан на портале Azure перед использованием рецепта. Каждая когнитивная служба, поддерживающая контейнеры, имеет документ «Как установить» специально для установки и настройки службы для контейнера. Некоторые службы требуют файла или набора файлов в качестве ввода для контейнера, важно, что вы понимаете и успешно использовали контейнер перед использованием этого решения.

* Ресурс Когнитивных сервисов, созданный на портале Azure.
* **URL-адрес когнитивной службы** - просмотрите данные службы "Как установить" для контейнера, чтобы узнать, где URL-адрес конечной точки находится внутри портала Azure и как выглядит правильный пример URL. Точный формат может измениться от службы к службе.
* **Ключ** когнитивного сервиса - ключи находятся на странице **ключей** для ресурса Azure. Вам потребуется только один из двух ключей. Ключ представляет собой строку из 32 альфа-числосимволов.
* Один контейнер когнитивных услуг на локальном хосте (ваш компьютер). Убедитесь, что вы можете:
  * Потяните `docker pull` вниз изображение с командой.
  * Успешно запустите локальный контейнер со всеми `docker run` необходимыми настройками конфигурации с командой.
  * Вызовите конечную точку контейнера, получив ответ HTTP 2xx и ответ JSON обратно.

Все переменные в угловых скобках, `<>`должны быть заменены на ваши собственные значения. Эта замена включает в себя угловые скобки.

## <a name="use-the-container-instance"></a>Использование контейнерного экземпляра

1. Выберите **обзор** и скопируйте IP-адрес. Это будет числовый IP-адрес, такой как `55.55.55.55`.
1. Откройте новую вкладку браузера и `http://<IP-address>:5000 (http://55.55.55.55:5000`используйте IP-адрес, например, ). Вы увидите домашнюю страницу контейнера, сообщив, что контейнер работает.

1. Выберите **описание API службы,** чтобы просмотреть страницу чванства для контейнера.

1. Выберите любой из **AA** POST и выберите **Попробуйте его.**  Параметры отображаются, включая вход. Заполните параметры.

1. Выберите **Выполнить,** чтобы отправить запрос в контейнер Instance.

    Вы успешно создали и использовали контейнеры Cognitive Services в Azure Container Instance.

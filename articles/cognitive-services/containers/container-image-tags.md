# <a name="azure-cognitive-services-container-image-tags-and-release-notes"></a>Теги образа контейнера Cognitive Services Azure и заметки о выпуске

Azure Cognitive Services предлагает множество образов контейнеров. Реестры контейнеров и соответствующие репозитории зависят от образов контейнеров. Каждое имя образа контейнера предлагает несколько тегов. Тег образа контейнера — это механизм управления версиями образа контейнера. Эта статья предназначена для использования в качестве исчерпывающего справочника по перечислению всех Cognitive Services образов контейнеров и их доступных тегов.

> [!TIP]
> При использовании [`docker pull`](https://docs.docker.com/engine/reference/commandline/pull/) следует уделять особое внимание Регистру реестра контейнеров, репозитория, имени образа контейнера и соответствующего тега, так как они **чувствительны к регистру**.

## <a name="anomaly-detector"></a>Детектор аномалий

Образ контейнера [детектора аномалий][ad-containers] можно найти в разделе " `mcr.microsoft.com` Открытие реестра контейнеров". Полное имя образа контейнера —, `mcr.microsoft.com/azure-cognitive-services/decision/anomaly-detector` .

Этот образ контейнера имеет следующие доступные теги. Полный список тегов можно также найти [на мкр](https://mcr.microsoft.com/v2/azure-cognitive-services/decision/anomaly-detector/tags/list).

# <a name="previous-versions"></a>[предыдущих версий](#tab/previous)

Заметки о выпуске `1.1.013050001-amd64-preview`

* Добавлена `ReadEngineConfig:ResultExpirationPeriod` Конфигурация инициализации контейнера, указывающая, когда система должна очищать результаты распознавания. 
    * Значение параметра — в часах, а по умолчанию — 48 часов.
    * Параметр может уменьшить использование памяти для хранения результатов, особенно если используется хранилище контейнера в памяти.
    * Пример 1. Реаденгинеконфиг: Ресултекспиратионпериод = 1, система очистит результаты распознавания 1 час после этого процесса.
    * Если для этой конфигурации установлено значение 0, система очистит результаты распознавания после получения результата.

* Исправлена внутренняя ошибка сервера 500, когда в систему передается недопустимый формат изображения. Теперь будет возвращена ошибка 400:

    ```json
    {
        "error": {
        "code": "InvalidImageSize",
        "message": "Image must be between 1024 and 209715200 bytes."
        }
    }
    ```

# <a name="build-a-training-data-set-for-a-custom-model"></a>Построение обучающий набор данных для пользовательской модели

При использовании пользовательской модели распознаватель формы, поэтому модель можно обучить в конкретной отрасли формы предоставляют обучающих данных. Вы можете обучить модель с пять заполненные формы или пустую форму (необходимо включить слово «empty» в имени файла) плюс две формы заполнены по умолчанию. Даже если у вас есть достаточно заполненные формы для обучения на основе, Добавление пустой формы для набора данных для обучения может улучшить точность модели.

## <a name="training-data-tips"></a>Полезные советы данных

Очень важно использовать набор данных, которая оптимизирована для обучения. Используйте приведенные ниже советы, чтобы получить наилучший результат из [Обучение модели](https://westus2.dev.cognitive.microsoft.com/docs/services/form-recognizer-api/operations/TrainCustomModel) операции:

* По возможности используйте текстовых документов PDF, а не документов на основе образа. Отсканированные документы PDF организованы в виде изображения.
* Используйте одну пустую форму и две формы заполнены по умолчанию, если они доступны.
* Для заполнения форм используйте примеры, которые имеют все свои поля заполнены.
* Используйте формы с разными значениями в каждом поле.
* Если форма образы имеют более низкое, используйте большего набора данных (например, изображений 10 – 15).

Убедитесь, что набор данных для обучения также соответствует требования к входным данным для всего содержимого формы распознаватель.

## <a name="upload-your-training-data"></a>Отправка обучающих данных

Если вы собрали набор документов формы, которые будут использоваться для обучения, необходимо передать его в контейнер хранилища BLOB-объектов Azure. Если вы не знаете, как создать учетную запись хранения с контейнером, следуя [краткое руководство по службе хранилища Azure для портала Azure](https://docs.microsoft.com/azure/storage/blobs/storage-quickstart-blobs-portal).

### <a name="organize-your-data-in-subfolders-optional"></a>Организовывать данные во вложенных папках (необязательно)

По умолчанию [Обучение модели](https://westus2.dev.cognitive.microsoft.com/docs/services/form-recognizer-api/operations/TrainCustomModel) API будет использовать только формы документов, которые расположены в корневом контейнере хранилища. Тем не менее вы можете обучить с данными во вложенных папках, если он указан в вызове API. Как правило, тело [Обучение модели](https://westus2.dev.cognitive.microsoft.com/docs/services/form-recognizer-api/operations/TrainCustomModel) вызов имеет следующую форму, где `<SAS URL>` является общий доступ подписанный URL-адрес контейнера:

```json
{
  "source":"<SAS URL>"
}
```

Если в тексте запроса, добавьте следующее содержимое, API будет обучения на основе документы, расположенные во вложенных папках. `"prefix"` Поле является необязательным и ограничит обучающего набора данных в файлы, пути которых начинаются с заданной строкой. Поэтому значение `"Test"`, например, вызовет API для просмотра только файлы или папки, которые начинаются со слова «Test».

```json
{
  "source": "<SAS URL>",
  "sourceFilter": {
    "prefix": "<prefix string>",
    "includeSubFolders": true
  }
}
```

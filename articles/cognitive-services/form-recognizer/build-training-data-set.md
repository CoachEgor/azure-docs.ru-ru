# <a name="build-a-training-data-set-for-a-custom-model"></a>Создание набора обучаемых данных для пользовательской модели

При использовании пользовательской модели Form Recognizer вы предоставляете свои собственные обучающие данные, чтобы модель мог тренироваться в ваших отраслевых формах. Вы можете обучить модель с пятью заполненными формами или пустой формой (вы должны включить слово «пустой» в имя файла) плюс две заполненные формы. Даже если у вас достаточно заполненных форм для обучения, добавление пустой формы в набор обучаемых данных может повысить точность модели.

Если вы хотите использовать обучающие данные с ручной маркировкой, следует начать с по крайней мере пяти форм одного и того же типа. В одном и том же наборе данных все еще можно использовать немаркированные формы и пустую форму.

## <a name="training-data-tips"></a>Советы по обучению данным

Важно использовать набор данных, оптимизированный для обучения. Используйте следующие советы, чтобы убедиться, что вы получите лучшие результаты от [работы модели Train Custom Model:](https://westus2.dev.cognitive.microsoft.com/docs/services/form-recognizer-api-v2-preview/operations/TrainCustomModelAsync)

* Если это возможно, используйте текстовые PDF-документы вместо документов, основанных на изображениях. Отсканированные PDF-файлы обрабатываются как изображения.
* Для заполненных форм используйте примеры, на которых заполнены все поля.
* Используйте формы с разными значениями в каждом поле.
* Если изображения формы имеют более низкое качество, используйте больший набор данных (например, 10-15 изображений).
* Общий размер набора обучаемых данных может составить до 500 страниц.

## <a name="upload-your-training-data"></a>Загрузка учебных данных

Когда вы собрали набор форм-документов, которые вы будете использовать для обучения, вам нужно загрузить его в контейнер для хранения капли Azure. Если вы не знаете, как создать учетную запись хранения Azure с помощью контейнера, после [быстрого запуска Azure Storage для портала Azure.](https://docs.microsoft.com/azure/storage/blobs/storage-quickstart-blobs-portal)

### <a name="organize-your-data-in-subfolders-optional"></a>Организация данных в субфалкерах (необязательно)

По умолчанию API [Train Custom Model](https://westus2.dev.cognitive.microsoft.com/docs/services/form-recognizer-api-v2-preview/operations/TrainCustomModelAsync) будет использовать только документы формы, которые находятся в корне контейнера для хранения. Тем не менее, вы можете тренироваться с данными в субфалкерах, если указать их в вызове API. Как правило, тело вызова [модели Train Custom Model](https://westus2.dev.cognitive.microsoft.com/docs/services/form-recognizer-api-v2-preview/operations/TrainCustomModelAsync) `<SAS URL>` имеет следующую форму, где находится URL-адрес подписи общего доступа вашего контейнера:

```json
{
  "source":"<SAS URL>"
}
```

Если вы добавите следующее содержимое в орган запроса, API будет тренироваться с документами, расположенными в субфайдерах. Поле `"prefix"` является необязательным и будет ограничивать набор обучаемых данных файлами, пути которых начинаются с данной строки. Таким `"Test"`образом, значение, например, приведет к тому, что API будет смотреть только на файлы или папки, которые начинаются со слова "Тест".

```json
{
  "source": "<SAS URL>",
  "sourceFilter": {
    "prefix": "<prefix string>",
    "includeSubFolders": true
  },
  "useLabelFile": false
}
```

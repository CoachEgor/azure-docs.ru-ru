# <a name="deploy-the-sample-labeling-tool"></a>Развертывание примера средства создания меток

Пример средства распознавания меток распознавателя форм — это приложение, которое выполняется в контейнере DOCKER. Он предоставляет полезный пользовательский интерфейс, который можно использовать для ручной маркировки документов форм в целях контролируемого обучения. В кратком руководстве [обучение с метками](./quickstarts/label-tool.md) показано, как запустить средство на локальном компьютере, которое является наиболее распространенным сценарием. 

В этом руководством описаны альтернативные способы развертывания и запуска примера средства создания меток. 

## <a name="deploy-with-azure-container-instances"></a>Развертывание с помощью службы "экземпляры контейнеров Azure"

Программу Label можно запустить в контейнере веб-приложения DOCKER. Сначала [Создайте новый ресурс веб-приложения](https://ms.portal.azure.com/#create/Microsoft.WebSite) на портал Azure. Заполните форму, указав сведения о подписке и группе ресурсов. Введите следующие сведения в обязательные поля:
* **Публикация**: контейнер DOCKER
* **Операционная** Система: Linux

На следующей странице заполните следующие поля для настройки контейнера docker:

* **Параметры**: один контейнер
* **Источник образа**: реестр контейнеров Azure
* **Тип доступа**: Открытый
* **Изображение и тег**: MCR.Microsoft.com/Azure-Cognitive-Services/Custom-Form/labeltool:Latest

Дальнейшие действия являются необязательными. После завершения развертывания приложения его можно запустить и получить доступ к средству меток в Интернете.

### <a name="connect-to-azure-ad-for-authorization"></a>Подключение к Azure AD для авторизации

Мы рекомендуем подключить веб-приложение к Azure Active Directory (AAD), чтобы только пользователь с вашими учетными данными мог войти в приложение и использовать его. Следуйте инструкциям в разделе [Настройка приложения службы приложений](https://docs.microsoft.com/azure/app-service/configure-authentication-provider-aad) для подключения к AAD.

В этой статье мы предоставим ссылки и инструкции, которые научат вас:

* [Развертывание инструмента маркировки образцов в экземпляре контейнеров Azure (ACI)](#deploy-with-azure-container-instances-aci)
* [Использование и внесение вклада в инструмент маркировки формы OCR с открытым исходным кодом](#open-source-on-github)

## <a name="run-the-sample-labeling-tool-locally"></a>Выполнить инструмент маркировки образцов локально

Самый быстрый способ начать маркировку данных — локально запустить инструмент маркировки образцов. Следующий quickstart использует Form Recognizer REST API и инструмент маркировки образцов для обучения пользовательской модели с вручную помеченными данными. 

* [Быстрый запуск: Форма этикетки, обучить модель и проанализировать форму с помощью инструмента маркировки образцов.](./quickstarts/label-tool.md)

## <a name="deploy-with-azure-container-instances-aci"></a>Развертывание с azure Контейнер Instances (ACI)

Прежде чем мы начнем, важно отметить, что существует два способа развертывания инструмента маркировки образцов в Azure Container Instance (ACI). Оба варианта используются для запуска инструмента маркировки образцов с помощью ACI: 

* [Использование портала Azure](#azure-portal)
* [Использование Azure CLI](#azure-cli)

### <a name="azure-portal"></a>Портал Azure

Выполните следующие действия, чтобы создать новый ресурс с помощью портала Azure: 

1. Войдите на [портал Azure](https://portal.azure.com/signin/index/).
2. Выберите **Создать ресурс**. 
3. Далее выберите **Web App**. 

   > [!div class="mx-imgBorder"]
   > ![Выбор веб-приложения](./media/quickstarts/formre-create-web-app.png)
   
4. Во-первых, убедитесь, что вкладка **Основы** выбрана. Теперь вам нужно предоставить некоторую информацию: 

   > [!div class="mx-imgBorder"]
   > ![Выберите Основы](./media/quickstarts/formre-select-basics.png)
   * Подписка - Выберите существующую подписку Azure
   * Группа ресурсов - Вы можете повторно использовать существующую группу ресурсов или создать новую для этого проекта. Рекомендуется создать новую группу ресурсов.
   * Имя - Дайте вашему веб-приложению имя. 
   * Публикация - Выберите **докерный контейнер**
   * Операционная система - Выберите **Linux**
   * Регион - Выберите регион, который имеет смысл для вас.
   * Linux Plan - Выберите уровень ценообразования/план для службы приложений. 

   > [!div class="mx-imgBorder"]
   > ![Наверстойнете ваше веб-приложение](./media/quickstarts/formre-select-docker-linux.png)

5. Затем выберите вкладку **Docker.** 

   > [!div class="mx-imgBorder"]
   > ![Выберите Докера](./media/quickstarts/formre-select-docker.png)

6. Теперь давайте навескажем контейнер Docker. Все поля необходимы, если не указано иное:

   * Варианты - Выберите **единый контейнер**
   * Источник изображения - Выберите **частный реестр** 
   * URL сервера - Установите это`https://mcr.microsoft.com`
   * Имя пользователя (необязательно) - Создайте имя пользователя. 
   * Пароль (необязательно) - Создайте безопасный пароль, который вы запомните.
   * Изображение и тег - Установите это`mcr.microsoft.com/azure-cognitive-services/custom-form/labeltool:latest`
   * Непрерывное развертывание - Установите это **на,** если вы хотите получать автоматические обновления, когда команда разработчиков вносит изменения в инструмент маркировки образцов.
   * Команда запуска - Установите это`./run.sh eula=accept`

   > [!div class="mx-imgBorder"]
   > ![Настройка Docker](./media/quickstarts/formre-configure-docker.png)

7. Вот и все. Далее выберите **Обзор и Создайте,** затем **создайте** для развертывания веб-приложения. После завершения, вы можете получить доступ к веб-приложению по URL, предоставленному в **Обзоре** для вашего ресурса.

> [!NOTE]
> При создании веб-приложения можно также настроить авторизацию/аутентификацию. Это не обязательно, чтобы начать работу. 

### <a name="azure-cli"></a>Azure CLI

В качестве альтернативы использованию портала Azure можно создать ресурс с помощью Azure CLI. Перед продолжением необходимо установить [CLI Azure.](https://docs.microsoft.com/cli/azure/install-azure-cli?view=azure-cli-latest) Вы можете пропустить этот шаг, если вы уже работаете с Azure CLI. 

Есть несколько вещей, которые вам нужно знать об этой команде:

* `DNS_NAME_LABEL=aci-demo-$RANDOM`генерирует случайное имя DNS. 
* Этот пример предполагает, что у вас есть группа ресурсов, которую можно использовать для создания ресурса. Замените `<resource_group_name>` действительную группу ресурсов, связанную с подпиской. 
* Вам нужно будет указать, где вы хотите создать ресурс. Замените желаемую `<region name>` область для веб-приложения. 
* Эта команда автоматически принимает EULA.

Из Azure CLI запустите эту команду для создания веб-ресурса приложения для инструмента маркировки образцов: 

```azurecli
DNS_NAME_LABEL=aci-demo-$RANDOM

az container create \
  --resource-group <resorunce_group_name> \
  --name <name> \
  --image mcr.microsoft.com/azure-cognitive-services/custom-form/labeltool \
  --ports 3000 \
  --dns-name-label $DNS_NAME_LABEL \
  --location <region name> \
  --cpu 2 \
  --memory 8
  --command-line "./run.sh eula=accept"
```

### <a name="connect-to-azure-ad-for-authorization"></a>Подключение к Azure AD для авторизации

Рекомендуется подключить веб-приложение к Active Directory Azure. Это гарантирует, что только пользователи с действительными учетными данными могут войти в систему и использовать ваше веб-приложение. Следуйте инструкциям в [приложении «Настройка»](https://docs.microsoft.com/azure/app-service/configure-authentication-provider-aad) для подключения к active-каталогу Azure.

# <a name="cognitive-services-and-machine-learning"></a>Cognitive Services и машинное обучение

Cognitive Services предоставляет возможности машинного обучения для решения общих проблем, таких как анализ текста для эмоционального настроения или анализа изображений для распознавания объектов или лиц. Для использования этих услуг не требуется специальное машинное обучение или знания о данных. 

[Cognitive Services](welcome.md) — это группа служб, каждая из которых поддерживает различные, обобщенные возможности прогнозирования. Услуги делятся на различные категории, чтобы помочь вам найти правильный сервис. 

|Категория обслуживания|Назначение|
|--|--|
|[Решение](https://azure.microsoft.com/services/cognitive-services/directory/decision/)|Создавайте приложения, которые предоставят рекомендации для информированного и эффективного принятия решения.|
|[Язык](https://azure.microsoft.com/services/cognitive-services/directory/lang/)|Реализуйте в приложениях обработку естественного языка на основе готовых скриптов, распознавание тональности и определение намерений пользователей.|
|[Поиск](https://azure.microsoft.com/services/cognitive-services/directory/search/)|Добавьте в свои приложения API Поиска Bing и пользуйтесь возможностью объединять множество веб-страниц, изображений, видео и новостей с помощью одного вызова API.|
|[Речь](https://azure.microsoft.com/services/cognitive-services/directory/speech/)|Преобразуйте речь в текст или текст в естественно звучащую речь. Переводите текст на другой язык, а также выполняйте проверку и распознавание говорящего.|
|[Зрение](https://azure.microsoft.com/services/cognitive-services/directory/vision/)|Распознавайте, идентифицируйте, подписывайте, индексируйте и модерируйте изображения, видео и содержимое рукописного ввода.|

Используйте когнитивные услуги, когда вы:

* Можно использовать обобщенный решение.
* Решение доступа из программирования REST API или SDK. 

Используйте другое решение для машинного обучения при:

* Нужно выбрать алгоритм и нужно тренироваться на очень специфических данных.

## <a name="what-is-machine-learning"></a>Что такое машинное обучение?

Машинное обучение – это концепция, в которой вы объединяете данные и алгоритм для решения конкретных потребностей. После обучения данных и алгоритма вывод представляет собой модель, которую можно использовать снова с помощью различных данных. Обученная модель предоставляет информацию, основанную на новых данных. 

Процесс построения системы машинного обучения требует определенных знаний о машинном обучении или науке о данных.

Машинное обучение обеспечивается с помощью [продуктов и услуг Azure Machine Learning (AML).](https://docs.microsoft.com/azure/architecture/data-guide/technology-choices/data-science-and-machine-learning?context=azure/machine-learning/studio/context/ml-context)

## <a name="what-is-a-cognitive-service"></a>Что такое когнитивная услуга?

Когнитивная служба предоставляет часть или все компоненты в решении для машинного обучения: данные, алгоритм и обученную модель. Эти услуги предназначены для того, чтобы требовать общих знаний о ваших данных без необходимости опыта работы с машинным обучением или наукой о данных. Эти услуги предоставляют как REST API (ы), так и языковые SDK. В результате, вам необходимо иметь знания языка программирования, чтобы использовать услуги.

## <a name="how-are-cognitive-services-and-azure-machine-learning-aml-similar"></a>Чем похожи когнитивные сервисы и машинное обучение Azure (AML)?

Оба имеют конечную цель применения искусственного интеллекта (ИИ) для повышения бизнес-операций, хотя, как каждый обеспечивает это в соответствующих предложений отличается. 

Как правило, аудитория разная:

* Cognitive Services для разработчиков, не имеющих опыта машинного обучения.
* Azure Machine Learning предназначен для ученых-данных. 

## <a name="how-is-a-cognitive-service-different-from-machine-learning"></a>Чем когнитивная служба отличается от машинного обучения?

Когнитивная служба предоставляет для вас обученную модель. Это объединяет данные и алгоритм, доступный из REST API (ы) или SDK. Эту услугу можно реализовать в течение нескольких минут, в зависимости от сценария.  Когнитивная служба предоставляет ответы на общие проблемы, такие как ключевые фразы в тексте или идентификация элементов в изображениях. 

Машинное обучение — это процесс, который, как правило, требует более длительного периода времени для успешной реализации. Это время тратится на сбор данных, очистку, трансформацию, выбор алгоритмов, обучение модели и развертывание, чтобы получить тот же уровень функциональности, предоставляемый Когнитивной службой. С помощью машинного обучения можно дать ответы на узкоспециализированные и/или конкретные проблемы. Проблемы машинного обучения требуют ознакомления с конкретным предметом и данными рассматриваемой проблемы, а также опыта в области науки о данных.

## <a name="what-kind-of-data-do-you-have"></a>Какие у вас данные?

Cognitive Services, как группа служб, может не требовать никаких, некоторых или всех пользовательских данных для обученной модели. 

### <a name="no-additional-training-data-required"></a>Дополнительные данные об обучении не требуются

Услуги, предоставляющие полностью обученную модель, можно рассматривать как _черный ящик._ Вам не нужно знать, как они работают или какие данные были использованы для их обучения. Вы приносите свои данные в полностью обученную модель, чтобы получить прогноз. 

### <a name="some-or-all-training-data-required"></a>Требуются некоторые или все обучающие данные

Некоторые услуги позволяют принести свои собственные данные, а затем обучить модель. Это позволяет расширить модель, используя данные службы и алгоритм с вашими собственными данными. Выход соответствует вашим потребностям. При внедравливую собственные данные, возможно, потребуется отметить данные определенным образом для службы. Например, если вы обучаете модель для идентификации цветов, вы можете предоставить каталог изображений цветов вместе с расположением цветка на каждом изображении для обучения модели. 

Служба может _позволить_ вам предоставлять данные для улучшения своих собственных данных. Служба может _потребовать_ от вас предоставить данные. 

### <a name="real-time-or-near-real-time-data-required"></a>Требуются данные в режиме реального времени или почти в режиме реального времени

Для создания эффективной модели службе могут потребоваться данные в реальном времени или почти в реальном времени. Эти службы обрабатывают значительные объемы типовых данных.

## <a name="where-can-you-use-cognitive-services"></a>Где можно использовать когнитивные сервисы?
 
Службы используются в любом приложении, которое может сделать REST API (ы) или SDK звонки. Примеры приложений включают веб-сайты, боты, виртуальную или смешанную реальность, настольные и мобильные приложения. 

## <a name="how-is-azure-cognitive-search-related-to-cognitive-services"></a>Как когнитивный поиск Azure связан с когнитивными сервисами?

[Azure Cognitive Search](../search/search-what-is-azure-search.md) — это отдельная служба поиска облаков, которая дополнительно использует Cognitive Services для добавления изображений и обработки естественного языка для индексации рабочих нагрузок. Когнитивные службы подвергаются воздействию в Azure Cognitive Search благодаря [встроенным навыкам,](../search/cognitive-search-predefined-skills.md) которые окутывает отдельные AA. Вы можете использовать бесплатный ресурс для пошагов, но план по созданию и присоединению [оплачиваемого ресурса](../search/cognitive-search-attach-cognitive-services.md) для больших объемов.

## <a name="how-can-you-use-cognitive-services"></a>Как вы можете использовать когнитивные услуги?

Каждая служба предоставляет информацию о ваших данных. Вы можете объединить службы для цепных решений, таких как преобразование речи (аудио) в текст, перевод текста на многие языки, а затем с помощью переведенных языков для получения ответов из базы знаний. Хотя Cognitive Services может использоваться для самостоятельного создания интеллектуальных решений, они также могут быть объединены с традиционными проектами машинного обучения в дополнение к моделям или ускорению процесса разработки.

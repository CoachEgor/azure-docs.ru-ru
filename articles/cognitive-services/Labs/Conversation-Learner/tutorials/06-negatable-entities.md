# <a name="how-to-use-negatable-entities-with-a-conversation-learner-model"></a>Использование отрицаемых сущностей в модели Conversation Learner

В этом руководстве описано свойство отрицаемости для сущностей.

## <a name="video"></a>Видео

[![Предварительная версия учебника по отрицаемым сущностям](https://aka.ms/cl_Tutorial_v3_NegatableEntities_Preview)](https://aka.ms/cl_Tutorial_v3_NegatableEntities)

## <a name="requirements"></a>Требования
Для работы с этим руководством требуется запущенный бот обучения общего назначения.

    npm run tutorial-general

## <a name="details"></a>Подробнее
Свойство "отрицание" сущности позволяет помечать как обычные (положительные), так и отрицательные экземпляры сущности, обучаться на основе положительных и отрицательных моделей и очищать значение существующей сущности. Сущности с установленным свойством отрицаемости называются отрицаемыми сущностями в Conversation Leaner.

## <a name="steps"></a>Шаги

Процедура начинается на домашней странице веб-интерфейса.

### <a name="create-the-model"></a>Создание модели

1. Выберите **создать модель**.
2. Введите **негатаблинтити** в качестве **имени**.
3. Нажмите кнопку **Создать**.

### <a name="entity-creation"></a>Создание сущности

1. На левой панели выберите **сущности** , а затем — **Новая сущность**.
2. Выберите **настраиваемое обучение** для **типа сущности**.
3. Введите **имя** для **имени сущности**.
4. Установите флажок с отрицанием, чтобы разрешить пользователям предоставлять значение сущности или сказать, что что-то *не* является значением сущности, тем самым удаляя совпадающее значение сущности.
5. Нажмите кнопку **Создать**.

![](../media/T06_entity_create.png)

### <a name="create-the-first-action"></a>Создание первого действия

1. Выберите **действия** на левой панели, а затем — **новое действие**.
2. Введите **я не зная ваше имя.** для **ответа Bot...**
3. Введите **имя** длянеквалифицированных прав.
4. Нажмите кнопку **Создать**.

![](../media/T06_action_create_1.png)

### <a name="create-the-second-action"></a>Создание второго действия

1. Выберите **действия** на левой панели, а затем — **новое действие**.
2. Введите **имя, которое я знал. Это $name.** для **ответа Bot...**
3. Нажмите кнопку **Создать**.

> [!NOTE]
> Сущность **Name** была автоматически добавлена в качестве **обязательных сущностей** по ссылке в ответном utterance.

Теперь у вас есть два действия.

![](../media/T06_action_create_2.png)

### <a name="train-the-model"></a>Обучение модели

1. Выберите пункт **обучение диалоговых окон** на левой панели, а затем **новое диалоговое окно "обучение**".
2. Введите **Hello** для utterance пользователя на левой панели разговора.
3. Выберите **действия оценки**.
4. Выберите **я не зная ваше имя.** из списка действий. Значение процентиля равно 100% в качестве единственного допустимого действия на основе ограничений.
5. Введите **мое имя Федор** для utterance пользователя на левой панели разговора.
6. Выделите **Федор** , а затем выберите **+ имя**. Сущности с отрицанием имеют два экземпляра: (+) плюс добавляет или перезаписывает значение; (-) минус удаляет значение.
7. Выберите **действия оценки**. Сущность **Name** теперь определена как **Федор** в памяти модели **, поэтому я знал ваше имя. Это действие доступно** $Name.
8. Выберите **я знал ваше имя. Это $name.** из списка действий.
9. Введите **мое имя, а не Федор.** для utterance пользователя на левой панели разговора.
10. Выделите **Федор** , а затем выберите **-Name** , чтобы очистить значение в поле **имя** сущности.
11. Выберите **действия оценки**.
12. Выберите **я не зная ваше имя.** из списка действий.
13. Введите **мое имя — Ирина.** для третьего utterance пользователя на левой панели разговора.
14. Выделите **Ирина** then **+ Name**

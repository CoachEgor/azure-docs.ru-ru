# <a name="how-to-use-negatable-entities-with-a-conversation-learner-model"></a>Использование отрицаемых сущностей в модели Conversation Learner

В этом руководстве описано свойство отрицаемости для сущностей.

## <a name="video"></a>Видео

[![Предварительная версия учебника по отрицаемым сущностям](https://aka.ms/cl_Tutorial_v3_NegatableEntities_Preview)](https://aka.ms/cl_Tutorial_v3_NegatableEntities)

## <a name="requirements"></a>Требования
Для работы с этим руководством требуется запущенный бот обучения общего назначения.

    npm run tutorial-general

## <a name="details"></a>Сведения
Свойство «Negatable» сущности позволяет пометить как обычный (положительное значение) и отрицательные экземпляры сущности, обучение, основанные на моделях положительные и отрицательные и очистите значение существующей сущности. Сущности с установленным свойством отрицаемости называются отрицаемыми сущностями в Conversation Leaner.

## <a name="steps"></a>Действия

Процедура начинается на домашней странице веб-интерфейса.

### <a name="create-the-model"></a>Создание модели

1. Выберите **новой модели**.
2. Введите **NegatableEntity** для **имя**.
3. Нажмите кнопку **Создать**.

### <a name="entity-creation"></a>Создание сущности

1. Выберите **сущностей** на левой панели, затем **новую сущность**.
2. Выберите **Custom обучена** для **тип сущности**.
3. Введите **имя** для **имя сущности**.
4. Проверьте **Negatable** чтобы пользователи могли предоставить значение сущности, или сказать что-то не *не* сущности значение, тем самым удаление совпадающее значение сущности.
5. Нажмите кнопку **Создать**.

![](../media/T06_entity_create.png)

### <a name="create-the-first-action"></a>Создание первого действия

1. Выберите **действия** на левой панели, затем **новое действие**.
2. Введите **я не знаю, ваше имя.** для **Bot ответа...** .
3. Введите **имя** для **дисквалификации дает право**.
4. Нажмите кнопку **Создать**.

![](../media/T06_action_create_1.png)

### <a name="create-the-second-action"></a>Создание второго действия

1. Выберите **действия** на левой панели, затем **новое действие**.
2. Введите **я знаю, ваше имя. Это $name.** для **Bot ответа...** .
3. Нажмите кнопку **Создать**.

> [!NOTE]
> **Имя** сущности был автоматически добавлен как **необходимые сущности** по ссылке в utterance ответа.

Теперь у вас есть два действия.

![](../media/T06_action_create_2.png)

### <a name="train-the-model"></a>Обучение модели

1. Выберите **Train диалоговые окна** на левой панели, затем **новое диалоговое окно Train**.
2. Введите **hello** для пользователя utterance чата на левой панели.
3. Выберите **оценка действия**.
4. Выберите **я не знаю, ваше имя.** из списка действий. Процентиль — 100% как единственное допустимое действие на основе ограничений.
5. Введите **меня зовут Михаил** для пользователя utterance чата на левой панели.
6. Выделите **Фрэнк** выберите **+ имя**. Операция логического отрицания сущности имеют два экземпляра: (+), а также добавляет или перезаписывает значение; (-) минус удаляет значение.
7. Выберите **оценка действия**. **Имя** сущность теперь определяется как **Фрэнк** в модели памяти, поэтому **я знаю, ваше имя. Это $name** доступно действие.
8. Выберите **я знаю, ваше имя. Это $name.** из списка действий.
9. Введите **меня зовут не Франк.** для пользователя utterance чата на левой панели.
10. Выделите **Фрэнк** выберите **-имя** очистки этого значения из **имя** сущности.
11. Выберите **оценка действия**.
12. Выберите **я не знаю, ваше имя.** из списка действий.
13. Введите **меня зовут Сергей.** для третьей utterance пользователя на панели слева чата.
14. Выделите **Сьюзан** затем **+ имя** 

![](../media/T06_training.png)

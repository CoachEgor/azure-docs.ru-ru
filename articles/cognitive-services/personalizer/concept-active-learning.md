# <a name="learning-policy-and-settings"></a>Политика и настройки обучения

Настройки обучения определяют *гиперпараметры* обучения модели. Две модели одних и тех же данных, которые обучаются на разных настройках обучения, в конечном итоге будут отличаться.

[Политика и настройки обучения](how-to-settings.md#configure-rewards-for-the-feedback-loop) установлены на ресурсе Personalizer на портале Azure.

## <a name="import-and-export-learning-policies"></a>Политика обучения в области импорта и экспорта

Вы можете импортировать и экспортировать файлы политики обучения с портала Azure. Используйте этот метод, чтобы сохранить существующие политики, протестировать их, заменить и архивировать их в элементах управления исходным кодом в качестве артефактов для будущих ссылок и аудита.

[Узнайте, как импортировать](how-to-manage-model.md#import-a-new-learning-policy) и экспортировать политику обучения на портале Azure для вашего ресурса Personalizer.

## <a name="understand-learning-policy-settings"></a>Понимание параметров политики обучения

Настройки в политике обучения не предназначены для изменения. Изменяйте настройки только в том случае, если вы понимаете, как они влияют на Personalizer. Без этих знаний, вы можете вызвать проблемы, в том числе недействительными модели Personalizer.

Personalizer использует [vowpalwabbit](https://github.com/VowpalWabbit) для обучения и оценки событий. Обратитесь к [документации vowpalwabbit](https://github.com/VowpalWabbit/vowpal_wabbit/wiki/Command-line-arguments) о том, как отсеять настройки обучения с помощью vowpalwabbit. Если у вас есть правильные аргументы командной строки, сохраните команду в файлсоименном со следующим форматом (замените значение свойства аргументов нужной командой) и загрузите файл для импорта настроек обучения в панели **настройки модели и обучения** в панели Azure для вашего ресурса Personalizer.

Ниже `.json` приводится пример политики обучения.

```json
{
  "name": "new learning settings",
  "arguments": " --cb_explore_adf --epsilon 0.2 --power_t 0 -l 0.001 --cb_type mtr -q ::"
}
```

## <a name="compare-learning-policies"></a>Сравнение политик обучения

Вы можете сравнить, как различные политики обучения выполняются против прошлых данных в журналах Personalizer, выполняя [оценки в автономном режиме.](concepts-offline-evaluation.md)

[Загрузите свои собственные политики обучения,](how-to-manage-model.md) чтобы сравнить их с текущей политикой обучения.

## <a name="optimize-learning-policies"></a>Оптимизация политики обучения

Personalizer может создать оптимизированную политику обучения в [автономной оценке.](how-to-offline-evaluation.md) Оптимизированная политика обучения, которая имеет лучшие награды в автономной оценке даст лучшие результаты, когда он используется в Интернете в Personalizer.

После оптимизации политики обучения можно применить ее непосредственно к Personalizer, чтобы она немедленно заменила текущую политику. Или вы можете сохранить оптимизированную политику для дальнейшей оценки, а затем решить, следует ли отказаться, сохранить или применить его.

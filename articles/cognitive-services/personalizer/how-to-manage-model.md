# <a name="how-to-manage-model-and-learning-settings"></a>Как управлять настройками моделей и обучения

Машинная модель и настройки обучения могут быть экспортированы для резервного копирования в вашей собственной системе управления исходным источником.

## <a name="export-the-personalizer-model"></a>Экспорт модели Персонализатора

Из раздела управления ресурсами для **параметров модели и обучения**просмотрите создание модели и последнюю обновленную дату и экспорттекущего текущей модели. Экспортировать файл модели для архивирования можно с помощью портала Azure или API Персонализатора.

![Экспорт текущей модели Персонализатора](media/settings/export-current-personalizer-model.png)

## <a name="clear-data-for-your-learning-loop"></a>Очистить данные для вашего цикла обучения

1. На портале Azure для ресурса Personalizer на странице **настройки модели и обучения** выберите **«Чистые данные».**
1. Для того, чтобы очистить все данные и сбросить цикл обучения в исходное состояние, выберите все 3 флажка.

    ![На портале Azure четкие данные с ресурса Personalizer.](./media/settings/clear-data-from-personalizer-resource.png)

    |Значение|Назначение|
    |--|--|
    |Регистрационная персонализация и данные о вознаграждении.|Эти данные журнала используются в автономных оценках. Очистите данные, если вы сбросите ресурс.|
    |Сбросить модель Personalizer.|Эта модель изменяется при каждой переподготовке. Эта частота обучения указана в **частоте загрузки модели** на странице **Конфигурации.** |
    |Установите политику обучения по умолчанию.|Если вы изменили политику обучения в автономном режиме, это сброс в исходную политику обучения.|

1. Выберите **чистые выбранные данные,** чтобы начать процесс клиринга. Статус сообщается в уведомлениях Azure, в навигации в правом верхнем правом направлении.

## <a name="import-a-new-learning-policy"></a>Импорт новой политики обучения

Параметры [политики обучения](concept-active-learning.md#understand-learning-policy-settings) определяют _гиперпараметры_ обучения модели. Выполните оценку в [автономном режиме,](how-to-offline-evaluation.md) чтобы найти новую политику обучения.

1. Откройте [портал Azure](https://portal.azure.com)и выберите ресурс Personalizer.
1. Выберите **настройки модели и обучения** в разделе **Управление ресурсами.**
1. Для **настроек обучения Import** выберите файл, созданный с указанным выше форматом JSON, а затем выберите кнопку **загрузки.**

    Дождитесь уведомления о том, что политика обучения была загружена успешно.

## <a name="export-a-learning-policy"></a>Экспорт политики обучения

1. Откройте [портал Azure](https://portal.azure.com)и выберите ресурс Personalizer.
1. Выберите **настройки модели и обучения** в разделе **Управление ресурсами.**
1. Для **настроек обучения импорта** выберите кнопку **настройки обучения Экспорта.** Это сохраняет `json` файл на локальном компьютере.

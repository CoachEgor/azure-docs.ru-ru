# <a name="terminology"></a>Терминология

Personalizer использует терминологию от обучения подкрепления. Эти термины используются на портале Azure и AA.

## <a name="conceptual-terminology"></a>Концептуальная терминология

* **Learning Loop**: Вы создаете ресурс Personalizer, называемый _циклом обучения,_ для каждой части приложения, которая может извлечь выгоду из персонализации. Если у вас есть несколько сред для персонализации, создайте цикл для каждой из них.

* **Модель**: Модель Personalizer фиксирует все данные, полученные о поведении пользователя, получение обучающих данных из комбинации аргументов, которые вы отправляете на вызовы rank and Reward, а также с обучающего поведения, определяемого политикой обучения.

## <a name="personalizer-configuration"></a>Конфигурация персонализатора

Personalizer настроен с [портала Azure.](https://portal.azure.com)

* **Вознаграждение:** настройка значений по умолчанию для времени ожидания вознаграждения, вознаграждения по умолчанию и политики агрегирования вознаграждения.

* **Исследование**: настройка процентного созыва ранга для использования для разведки

* **Частота обновления модели**: Как часто модель переобучается.

* **Хранение данных:** Сколько дней стоит данных для хранения. Это может повлиять на оценки в автономном режиме, которые используются для улучшения цикла обучения.

## <a name="use-rank-and-reward-apis"></a>Используйте AA-рейтинг и вознаграждение

* **Ранг**: Учитывая действия с функциями и функциями контекста, используйте исследовать или использовать, чтобы вернуть верхнее действие (элемент содержимого).

    * **Действия**: Действия — это элементы содержимого, такие как продукты или акции, из них можно выбрать. Personalizer выбирает верхнее действие (возвращенный идентификатор действий вознаграждения), чтобы показать пользователям через Rank API.

    * **Контекст**: Чтобы обеспечить более точный ранг, предоставьте информацию о контексте, например:
        * ваш пользователь;
        * используемое устройство;
        * Текущее время.
        * другие данные о текущей ситуации;
        * исторические данные о пользователе или контексте.

        Конкретное приложение может включать в себя другие контекстные сведения.

    * **[Характеристики](concepts-features.md)**: Единица информации о элементе содержимого или контексте пользователя. Убедитесь в том, чтобы использовать только функции, которые агрегируются. Не используйте в качестве функций определенное время, иные иудеи пользователей или другие неагрегированные данные.

        * _Функция действия_ — это метаданные о содержимом.
        * _Функция контекста_ — это метаданные о контексте, в котором представлено содержимое.

* **Исследование**: Служба Personalizer изучает, когда вместо того, чтобы вернуть лучшее действие, она выбирает другое действие для пользователя. Эта служба позволяет избежать смещения и застоя и может адаптироваться к текущему поведению пользователя путем изучения.

* **Эксплуатация**: Служба Personalizer использует текущую модель, чтобы определить наилучшие действия на основе прошлых данных.

* **Длительность эксперимента**: Количество времени, которое служба Personalizer ждет от вознаграждения, начиная с момента возникновения вызова ранга для этого события.

* **Неактивные события**: Неактивное событие — это событие, в котором вы вызвали rank, но вы не уверены, что пользователь когда-либо увидит результат из-за решений клиентских приложений. Неактивные события позволяют создавать и сохранять результаты персонализации, а решение об их отмене принимать позже, что не повлияет на модель машинного обучения.


* **Вознаграждение**: Мера того, как пользователь ответил на возвращенный идентификатор действий награды Rank API, как оценка между 0 к 1. Бизнес-логика задает значение от 0 до 1 в ​​зависимости от того, как этот выбор помог достичь ваших бизнес-целей персонализации. Цикл обучения не хранит эту награду как индивидуальную историю пользователей.

## <a name="offline-evaluations"></a>Автономная оценка

* **Оценка**: оценка в автономном режиме определяет наилучшую политику обучения для цикла на основе данных цикла.

* **Политика обучения**: Как Personalizer тренирует модель на каждом событии, будет определяться некоторыми параметрами, влияющими на работу алгоритма машинного обучения. Новый цикл обучения начинается с **политики обучения**по умолчанию, которая может дать умеренную производительность. При [запуске оценки,](concepts-offline-evaluation.md)Personalizer создает новые политики обучения, специально оптимизированные для использования случаев вашего цикла. Personalizer будет работать значительно лучше с политиками, оптимизированными для каждого конкретного цикла, генерируемых во время оценки. Политика обучения называется _настройками обучения_ на **модели и настройках обучения** для ресурса Personalizer на портале Azure.

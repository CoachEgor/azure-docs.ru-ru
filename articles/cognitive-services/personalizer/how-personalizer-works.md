# <a name="how-personalizer-works"></a>Принцип работы Персонализатора

Ресурс Personalizer, _ваш цикл обучения,_ использует машинное обучение для создания модели, которая предсказывает лучшее действие для вашего контента. Модель обучается исключительно на ваших данных, которые вы послали на него с **рангом** и **вознаграждением** звонки. Каждая петля полностью не зависит друг от друга.

## <a name="rank-and-reward-apis-impact-the-model"></a>AI-аПО ранга и вознаграждения влияют на модель

Вы _отправляете действия с функциями_ и _функциями контекста_ в API ранга. API **ранжирования** решает, какую из следующих моделей использовать.

* _Эксплуатация_: Текущая модель для принятия наилучших мер на основе прошлых данных.
* _Исследуйте:_ Выберите другое действие вместо верхнего действия. Вы [настраиваете этот процент](how-to-settings.md#configure-exploration-to-allow-the-learning-loop-to-adapt) для ресурса Personalizer на портале Azure.

Вы определяете балл вознаграждения и отправляете этот балл на API Вознаграждения. API **вознаграждения**:

* собирает данные для обучения модели, записывая характеристики и награды каждого вызова "Ранг";
* Использует эти данные для обновления модели на основе конфигурации, указанной в _политике обучения._

## <a name="your-system-calling-personalizer"></a>Ваша система вызова Personalizer

На следующем изображении показан архитектурный процесс вызовов "Ранг" и "Вознаграждение".

![альт текст](./media/how-personalizer-works/personalization-how-it-works.png "Как работает персонализация")

1. Вы _отправляете действия с функциями_ и _функциями контекста_ в API ранга.

    * Personalizer решает, следует ли использовать текущую модель или изучить новые варианты для модели.
    * Результат ранжирования отправляется в EventHub.
1. Высший ранг возвращается в вашу систему в качестве _идентификатора действий вознаграждения._
    Ваша система представляет этот контент и определяет вознаграждение баллов на основе ваших собственных бизнес-правил.
1. Ваша система возвращает награду балл в цикл обучения.
    * Когда Персонализатор получает вознаграждение, оно отправляется в EventHub.
    * "Ранг" и "Вознаграждение" связаны друг с другом.
    * Модель ИИ обновляется на основе результатов корреляции.
    * Механизм вывода обновляется с помощью новой модели.

## <a name="personalizer-retrains-your-model"></a>Personalizer перенапрягает вашу модель

Personalizer перенапрягает модель на основе настройки **обновления частоты модели** на ресурсе Personalizer на портале Azure.

Personalizer использует все сохраненные в настоящее время данные, основываясь на параметре **хранения данных** в течение нескольких дней на ресурсе Personalizer на портале Azure.

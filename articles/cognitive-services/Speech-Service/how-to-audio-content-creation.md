# <a name="improve-synthesis-with-audio-content-creation"></a>Улучшение синтеза с помощью создания звукового содержимого

[Создание звукового содержимого](https://aka.ms/audiocontentcreation) — это интерактивное средство, которое позволяет настраивать и тонко настраивать вывод текста в речь для приложений и продуктов Майкрософт. Это средство можно использовать для точной настройки открытых и пользовательских голосов для более точных естественных выражений, а также для управления выходными данными в облаке.

Средство создания аудио-содержимого основано на [языке разметки речи (SSML)](speech-synthesis-markup.md). Для упрощения настройки и настройки создание звукового содержимого позволяет визуально проверять выходные данные текста в речь в режиме реального времени.

## <a name="how-does-it-work"></a>Как это работает?

На этой схеме показаны действия, которые необходимо выполнить для настройки и экспорта настраиваемых выходов речи в текст. Используйте приведенные ниже ссылки для получения дополнительных сведений о каждом шаге.

![](media/audio-content-creation/audio-content-creation-diagram.jpg)

1. Первым шагом является [Создание учетной записи Azure, регистрация речевого ресурса и получение ключа подписки](#create-a-speech-resource). После получения ключа подписки его можно использовать для вызова службы речевого ввода и доступа к [созданию звукового содержимого](https://aka.ms/audiocontentcreation).
2. [Создайте файл настройки звука](#create-an-audio-tuning-file) с помощью обычного текста или SSML.
3. Выберите голосовое и язык, который вы хотите настроить. Для создания звукового содержимого используются все [голоса Microsoft Text-To-Speech](language-support.md#text-to-speech). Вы можете использовать стандартный, нейронный или собственный пользовательский Voice.
   >[!NOTE]
   > Для настраиваемых нейронных голосов доступен многоуровневый доступ, позволяющий создавать голоса высокой четкости, аналогичные голосовым средствам для естественного звука. Дополнительные сведения см. в разделе [ограничение процесса](https://aka.ms/ignite2019/speech/ethics).

4. Проверьте результат по умолчанию. Затем используйте средство настройки, чтобы настроить произношение, тон, частоту, интонатион, стиль голоса и многое другое. Полный список параметров см. в разделе [язык разметки речи](speech-synthesis-markup.md).
5. Сохраните и [экспортируйте настроенный звук](#export-tuned-audio). При сохранении в системе курса настройки можно продолжить работу и выполнить итерацию по выходным данным. Когда выходные данные будут удовлетворены, можно создать задачу создания звука с помощью функции экспорта. Вы можете наблюдать за состоянием задачи экспорта и скачивать выходные данные для использования с вашими приложениями и продуктами.
6. Последним шагом является использование настраиваемого настроенного голоса в приложениях и продуктах.

## <a name="create-a-speech-resource"></a>Создание речевого ресурса

Выполните следующие действия, чтобы создать речевой ресурс и подключить его с помощью Speech Studio.

1. Выполните эти инструкции, чтобы [зарегистрировать учетную запись Azure](get-started.md#new-resource) и [создать речевой ресурс](https://docs.microsoft.com/azure/cognitive-services/speech-service/get-started#create-the-resource). Убедитесь, что для ценовой категории задано значение **S0**. Если вы используете один из нейронных голосов, убедитесь, что вы создаете ресурс в [поддерживаемом регионе](regions.md#standard-and-neural-voices).
2. Войдите в службу [создания звукового содержимого](https://aka.ms/audiocontentcreation).
3. Выберите существующий проект или нажмите кнопку **создать**.
4. Вы можете изменить подписку в любое время с помощью параметра **Параметры** , расположенного в верхней панели навигации.

## <a name="create-an-audio-tuning-file"></a>Создание файла настройки звука

Существует два способа получить содержимое в средстве создания звукового содержимого.

**Вариант 1.**

1. После входа в службу " [Создание звукового содержимого](https://aka.ms/audiocontentcreation)" нажмите кнопку **Настройка звука** , чтобы создать новый файл настройки звука.
2. Когда появится окно редактирования, можно ввести до 10 000 символов.
3. Не забудьте сохранить.

**Вариант 2.**

1. После входа в службу [создания звукового содержимого](https://aka.ms/audiocontentcreation)нажмите кнопку **Отправить** , чтобы импортировать один или несколько текстовых файлов. Поддерживаются как обычный текст, так и SSML.
2. При отправке текстовых файлов убедитесь, что содержимое соответствует этим требованиям.

   | Свойство | Значение/примечания |
   |----------|---------------|
   | Формат файла | Обычный текст (TXT)<br/> Текст SSML (. txt)<br/> ZIP-файлы не поддерживаются |
   | Формат кодировки | UTF-8 |
   | Имя файла | Каждый файл должен иметь уникальное имя. Дубликаты не поддерживаются. |
   | Длина текста | Длина текстовых файлов не должна превышать 10 000 символов. |
   | Ограничения SSML | Каждый файл SSML может содержать только один элемент SSML. |

### <a name="plain-text-example"></a>Пример обычного текста

```txt
Welcome to use Audio Content Creation to customize audio output for your products.
```

### <a name="ssml-text-example"></a>Пример текста SSML

```xml
<speak xmlns="http://www.w3.org/2001/10/synthesis" xmlns:mstts="http://www.w3.org/2001/mstts" version="1.0" xml:lang="en-US">
    <voice name="Microsoft Server Speech Text to Speech Voice (en-US, JessaNeural)">
    Welcome to use Audio Content Creation <break time="10ms" />to customize audio output for your products.
    </voice>
</speak>
```

## <a name="export-tuned-audio"></a>Экспорт настроенного звука

После того как вы проверили выходные данные звука и удовлетворены настройками и корректировкой, вы можете экспортировать аудио.

1. В средстве [создания звукового содержимого](https://aka.ms/audiocontentcreation) нажмите кнопку **Экспорт** , чтобы создать задачу создания звука.
2. Выберите формат выходных данных для настроенного звука. Список поддерживаемых форматов и частот выборки см. ниже.
3. Состояние задачи можно просмотреть на вкладке **задача экспорта** . Если задача завершается неудачно, см. страницу подробных сведений для полного отчета.
4. После завершения задачи ваш звук будет доступен для загрузки на вкладке **Библиотека аудио** .
5. Щелкните элемент **Загрузить**. Теперь вы готовы использовать настраиваемый звуковой элемент в своих приложениях или продуктах.

### <a name="supported-audio-formats"></a>Поддерживаемые аудиоформаты

| Формат | Частота выборки 16 кГц | Частота выборки 24 кГц |
|--------|--------------------|--------------------|
| WAV | Metallica-16khz-16-разрядный-Mono-PCM | Metallica-24khz-16-разрядный-Mono-PCM |
| MP3 | Audio-16khz-128kbitrate-Mono-MP3 | Audio-24khz-160kbitrate-Mono-MP3 |

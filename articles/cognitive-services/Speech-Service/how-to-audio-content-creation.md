# <a name="improve-synthesis-with-audio-content-creation"></a>Улучшение синтеза с созданием аудиоконтента

[Audio Content Creation](https://aka.ms/audiocontentcreation) — это онлайн-инструмент, позволяющий настраивать и дорабатывать текстовое представление корпорации Майкрософт для ваших приложений и продуктов. Этот инструмент можно использовать для тонкой настройки общедоступных и пользовательских голосов для более точных естественных выражений и управления выходом в облаке.

Инструмент создания аудиоконтента основан на [языке речевого синтеза разметки (SSML).](speech-synthesis-markup.md) Чтобы упростить настройку и настройку, Audio Content Creation позволяет визуально проверять ваши текстовые выходы в речевые материалы в режиме реального времени.

## <a name="how-does-it-work"></a>Как это работает?

На этой диаграмме показаны шаги, которые необходимо предпринять для настройки и экспорта индивидуальных выходов от речи к тексту. Используйте ссылки ниже, чтобы узнать больше о каждом шаге.

![](media/audio-content-creation/audio-content-creation-diagram.jpg)

1. Первым шагом является [создание учетной записи Azure, регистрация ресурса Speech и ключ подписки.](#create-a-speech-resource) После того, как у вас есть ключ подписки, вы можете использовать его для вызова службы речи и доступа к [аудиоконтенту Creation.](https://aka.ms/audiocontentcreation)
2. [Создайте файл аудио-тюнинга](#create-an-audio-tuning-file) с помощью простого текста или SSML.
3. Выберите голос и язык, который вы хотели бы настроить. Аудио Контент Создание включает в себя все [текстовые голоса Microsoft в речи](language-support.md#text-to-speech). Вы можете использовать стандартный, нейронный или собственный пользовательский голос.
   >[!NOTE]
   > Закрытый доступ доступен для пользовательских нейронных голосов, которые позволяют создавать голоса высокой четкости, похожие на естественно звучащие речи. Для получения дополнительной информации [см.](https://aka.ms/ignite2019/speech/ethics)

4. Просмотрите результат по умолчанию. Затем используйте инструмент настройки для настройки произношения, шаг, скорость, интонация, стиль голоса, и многое другое. Полный список опций можно узнать на [языке разметки синтеза речи.](speech-synthesis-markup.md)
5. Сохранить и [экспортировать настроенный звук](#export-tuned-audio). При сохранении тюнинг-трека в системе можно продолжить работу и итерировать выход. Если вы удовлетворены выходом, вы можете создать задачу создания звука с функцией экспорта. Вы можете наблюдать за состоянием экспортной задачи и загружать выход для использования с помощью приложений и продуктов.
6. Последним шагом является использование пользовательского настроенного голоса в приложениях и продуктах.

## <a name="create-a-speech-resource"></a>Создание ресурса речи

Выполните следующие действия, чтобы создать ресурс Speech и связать его со студией речи.

1. Следуйте этим инструкциям, чтобы [подписаться на учетную запись Azure](get-started.md#new-resource) и [создать ресурс Speech.](https://docs.microsoft.com/azure/cognitive-services/speech-service/get-started#create-the-resource) Убедитесь, что ваш ценовой уровень установлен на **S0.** Если вы используете один из нейронных голосов, убедитесь, что вы создаете свой ресурс в [поддерживаемом регионе.](regions.md#standard-and-neural-voices)
2. Войти в [создание аудиоконтента](https://aka.ms/audiocontentcreation).
3. Выберите существующий проект или нажмите **«Создайте новый».**
4. Вы можете изменить подписку в любое время с опцией **Настройки,** расположенной в верхнем навигаторе.

## <a name="create-an-audio-tuning-file"></a>Создание файла аудио-тюнинга

Есть два способа, чтобы получить содержание в инструмент создания аудио контента.

**Вариант 1.**

1. После вступления в [аудио-контента Создание](https://aka.ms/audiocontentcreation), нажмите **Аудио тюнинг** для создания нового аудио настройки файла.
2. При появляются окна редактирования, вы можете ввести до 10 000 символов.
3. Не забудь спасти.

**Вариант 2.**

1. После вступления в [аудиоконтент Creation](https://aka.ms/audiocontentcreation)нажмите **«Загрузить»** для импорта одного или нескольких текстовых файлов. Поддерживаются как простой текст, так и SSML.
2. При загрузке текстовых файлов убедитесь, что содержимое соответствует этим требованиям.

   | Свойство | Значение / Заметки |
   |----------|---------------|
   | Формат файла | Обычный текст (TXT)<br/> Текст SSML (.txt)<br/> Файлы ципа не поддерживаются |
   | Формат кодирования | UTF-8 |
   | Имя файла | Каждый файл должен иметь уникальное имя. Дубликаты не поддерживаются. |
   | Длина текста | Текстовые файлы не должны превышать 10 000 символов. |
   | Ограничения SSML | Каждый файл SSML может содержать только одну часть SSML. |

### <a name="plain-text-example"></a>Обычный пример текста

```txt
Welcome to use Audio Content Creation to customize audio output for your products.
```

### <a name="ssml-text-example"></a>Пример текста SSML

```xml
<speak xmlns="http://www.w3.org/2001/10/synthesis" xmlns:mstts="http://www.w3.org/2001/mstts" version="1.0" xml:lang="en-US">
    <voice name="Microsoft Server Speech Text to Speech Voice (en-US, AriaNeural)">
    Welcome to use Audio Content Creation <break time="10ms" />to customize audio output for your products.
    </voice>
</speak>
```

## <a name="export-tuned-audio"></a>Экспорт настроенный звук

После того как вы просмотрели аудиовыход и удовлетворены настройкой и настройкой, вы можете экспортировать аудио.

1. Из инструмента [создания аудиоконтента](https://aka.ms/audiocontentcreation) нажмите **«Экспорт»,** чтобы создать задачу создания звука.
2. Выберите формат вывода для настроенного звука. Список поддерживаемых форматов и выборочных ставок приведен ниже.
3. Можно просмотреть состояние задачи на вкладке **«Экспортная задача».** Если задача не выполнена, смотрите подробную страницу информации для полного отчета.
4. Когда задача будет выполнена, звук будет доступен для скачивания на вкладке **Аудио Библиотеки.**
5. Нажмите кнопку **Загрузить**. Теперь вы готовы использовать настраиваемый звук на заказ в приложениях или продуктах.

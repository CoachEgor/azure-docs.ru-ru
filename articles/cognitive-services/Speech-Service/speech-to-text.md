# <a name="what-is-speech-to-text"></a>Что такое речь-текст?

Преобразование речи в текст из речевых служб Azure, также известных как преобразование речи в текст, позволяет подводить звуковые потоки в реальном времени к тексту, которые приложения, средства или устройства могут использовать, отображать и предпринимать действия в качестве входных данных команды. Эта служба работает с помощью той же технологии распознавания, которая используется корпорацией Майкрософт для Кортаны и продуктов Office. она легко работает с переводом и текстом в речь.  Полный список доступных языков для перевода речи в текст см. в разделе [Поддерживаемые языки](https://docs.microsoft.com/azure/cognitive-services/speech-service/language-support#speech-to-text).

По умолчанию в службе "речь-текст" используется универсальная языковая модель. Эта модель была обучена с использованием данных, принадлежащих корпорации Майкрософт, и развертывается в облаке. Это оптимальный вариант для сценариев взаимодействия и диктовки. Если вы используете преобразование речи в текст для распознавания и расшифровки в уникальной среде, вы можете создавать и обучать пользовательские акустические, языковые модели и модели произношения для учета окружающих помех или отраслевого словаря.

Вы можете легко записывать звук с микрофона, читать из потока или получать доступ к звуковым файлам из хранилища с помощью пакета SDK для распознавания речи и интерфейсов API. Пакет SDK для распознавания речи поддерживает 16-разрядный, 16 кГц/8 кГц, один канал для распознавания речи. Дополнительные звуковые форматы поддерживаются с помощью [конечной точки передачи речи в текст](https://docs.microsoft.com/azure/cognitive-services/speech-service/rest-apis) или [службы транскрипции пакетов](https://docs.microsoft.com/azure/cognitive-services/speech-service/batch-transcription#supported-formats).

## <a name="core-features"></a>Основные компоненты

Ниже приведены функции, доступные через пакет SDK для распознавания речи и интерфейсы API-интерфейсов.

| Вариант использования | SDK | REST |
|----------|-----|------|
| Транскрипция Short фразы продолжительностью (< 15 секунд). Поддерживает только окончательный результат транскрипции. | Да | Да |
| Непрерывная транскрипция длинных фразы продолжительностью и потокового звука (> 15 секунд). Поддерживает промежуточные и окончательные результаты транскрипции. | Да | Нет |
| Получение целей из результатов распознавания с помощью [Luis](https://docs.microsoft.com/azure/cognitive-services/luis/what-is-luis). | Да | Нет\* |
| Пакетная транскрипция звуковых файлов в асинхронном режиме. | Нет | Да\** |
| Создание и управление моделями речи. | Нет | Да\** |
| Создание и управление развертываниями пользовательских моделей. | Нет | Да\** |
| Создание проверок точности для измерения точности базовых моделей по сравнению с пользовательскими моделями. | Нет | Да\** |
| Управление подписками. | Нет | Да\** |

\* *Для получения сведений о намерениях и сущностях через LUIS потребуется отдельная подписка LUIS. С этой подпиской пакет SDK может вызывать LUIS для вас и предоставлять результаты для сущности и намерения. Вы можете самостоятельно вызывать LUIS с помощью REST API для получения сведений о намерениях и сущностях, используя подписку LUIS.*

\** *Эти службы доступны с помощью конечной точки cris.ai. См. [Справочник по Swagger](https://westus.cris.ai/swagger/ui/index).*

## <a name="customization"></a>Настройка

Помимо универсальной модели, используемой речевыми службами, вы можете создавать собственные модели акустических, языковых и произношения, относящиеся к вашему интерфейсу. Ниже приведен список параметров настройки.

| Модель | Описание |
|-------|-------------|
| [Акустическая модель](how-to-customize-acoustic-models.md) | Создание настраиваемой акустической модели полезно, если приложение, инструменты или устройства используются в определенной среде, например в автомобиле или фабрике с определенными условиями записи. В качестве примера можно привести речь с акцентом, наличие фонового шума или использование определенного микрофона для записи. |
| [Языковая модель](how-to-customize-language-model.md) | Создайте пользовательскую языковую модель для улучшения транскрибирования специализированных слов и грамматических конструкций, включая медицинскую терминологию и IT-жаргон. |
| [Модель произношения](how-to-customize-pronunciation.md) | Используя модель пользовательского произношения, вы можете определить фонетическую и отображаемую форму слова или термина. Это удобно для обработки настраиваемых терминов, например имен продуктов или аббревиатур. Все, что требуется для начала работы — это файл с записями произношения (простой TXT-файл). |

> [!NOTE]
> Параметры настройки зависят от языка и языкового стандарта (см. раздел [Поддерживаемые языки](supported-languages.md)).

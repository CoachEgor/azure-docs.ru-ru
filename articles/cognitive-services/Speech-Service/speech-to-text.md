# <a name="what-is-speech-to-text"></a>Что такое преобразование речи в текст?

Преобразование речи в текст из речевых служб Azure, также известных как преобразование речи в текст, позволяет подводить звуковые потоки в реальном времени к тексту, которые приложения, средства или устройства могут использовать, отображать и предпринимать действия в качестве входных данных команды. Эта служба работает с помощью той же технологии распознавания, которая используется корпорацией Майкрософт для Кортаны и продуктов Office. она легко работает с переводом и текстом в речь. Полный список доступных языков для перевода речи в текст см. в разделе [Поддерживаемые языки](https://docs.microsoft.com/azure/cognitive-services/speech-service/language-support#speech-to-text).

По умолчанию в службе "речь-текст" используется универсальная языковая модель. Эта модель была обучена с использованием данных, принадлежащих корпорации Майкрософт, и развертывается в облаке. Это оптимальный вариант для сценариев взаимодействия и диктовки. Если вы используете преобразование речи в текст для распознавания и расшифровки в уникальной среде, вы можете создавать и обучать пользовательские акустические, языковые модели и модели произношения для учета окружающих помех или отраслевого словаря.

Вы можете легко записывать звук с микрофона, читать из потока или получать доступ к звуковым файлам из хранилища с помощью пакета SDK для распознавания речи и интерфейсов API. Пакет SDK для служб "Речь" поддерживает распознавание речи для одноканальных 16-битных аудиофайлов в формате WAV/PCM с частотой 16 кГц/8 кГц. Дополнительные аудиоформаты поддерживаются при использовании [конечных точек REST для преобразования речи в текст](https://docs.microsoft.com/azure/cognitive-services/speech-service/rest-apis) или [службы пакетного транскрибирования](https://docs.microsoft.com/azure/cognitive-services/speech-service/batch-transcription#supported-formats).

## <a name="core-features"></a>Основные компоненты

Ниже приведены функции, доступные через пакет SDK для распознавания речи и интерфейсы API-интерфейсов.

| Вариант использования | SDK | REST |
|--------- | --- | ---- |
| Транскрипция Short фразы продолжительностью (< 15 секунд). Поддерживает только окончательный результат транскрипции. | Yes | Yes |
| Непрерывная транскрипция длинных фразы продолжительностью и потокового звука (> 15 секунд). Поддерживает промежуточные и окончательные результаты транскрипции. | Yes | Нет |
| Получение целей из результатов распознавания с помощью [Luis](https://docs.microsoft.com/azure/cognitive-services/luis/what-is-luis). | Yes | Нет\* |
| Пакетная транскрипция звуковых файлов в асинхронном режиме. | Нет  | Да\*\* |
| Создание и управление моделями речи. | Нет | Да\*\* |
| Создание и управление развертываниями пользовательских моделей. | Нет  | Да\*\* |
| Создание проверок точности для измерения точности базовых моделей по сравнению с пользовательскими моделями. | Нет  | Да\*\* |
| Управление подписками. | Нет  | Да\*\* |

\*_Luis и сущности могут быть производными с помощью отдельной подписки Luis. С этой подпиской пакет SDK может вызывать LUIS для вас и предоставлять результаты для сущности и намерения. С помощью REST API вы можете позвонить себе самостоятельно для получения целей и сущностей с помощью подписки LUIS._

\*\*_эти службы доступны с помощью конечной точки Cris.AI. См. [Справочник по Swagger](https://westus.cris.ai/swagger/ui/index)._

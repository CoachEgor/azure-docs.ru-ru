# <a name="about-custom-voice-first-virtual-assistants-preview"></a>О пользовательских предварительных выставляемых помощниках для предварительного просмотра

Настраиваемые виртуальные помощники, использующие Службы речи Azure, позволяют разработчикам создавать естественные и привычные для человека разговорные функции для своих приложений и интерфейсов. Канал "Речь Direct Line" в Bot Framework расширяет эти возможности, предоставляя согласованные и организованные точки входа для совместимых ботов, которые включают возможности входящего и исходящего голосового взаимодействия с низкой задержкой и высокой надежностью. Эти программы-роботы могут использовать Language Understanding Майкрософт (LUIS) для взаимодействия на естественном языке. Доступ к прямой текстовой речи осуществляется устройствами, использующими пакет средств разработки речевых программ (SDK).

   ![Концептуальная схема потока обслуживания прямого перевода речи](media/voice-first-virtual-assistants/overview.png "Поток речевого канала")


Прямая построчная речь и связанная с ней функциональность для пользовательских виртуальных помощников — это идеальное дополнение к [решению виртуального помощника и шаблону Enterprise](https://docs.microsoft.com/azure/bot-service/bot-builder-enterprise-template-overview). Несмотря на то, что прямое распознавание речи может работать с любым совместимым роботом, эти ресурсы предоставляют возможность повторного использования для высококачественных диалоговых возможностей, а также общие вспомогательные навыки и модели для быстрого начала работы.


## <a name="core-features"></a>Основные компоненты

| Категория | Компоненты |
|----------|----------|
|[Пользовательское пробуждение](speech-devices-sdk-create-kws.md) | Вы можете разрешить пользователям начинать диалоги с помощью программы-роботы, используя пользовательское ключевое слово, например "Эй Contoso". Эта задача выполняется с помощью пользовательского обработчика слов для пробуждения в пакете SDK для распознавания речи, который можно настроить с помощью настраиваемого текстового слова [, которое можно создать здесь](speech-devices-sdk-create-kws.md). Прямой канал распознавания речи включает проверку слова для пробуждения на стороне службы, что повышает точность активации пробуждения по словам и только для устройства.
|[Преобразование речи в текст](speech-to-text.md) | Канал перевода речи в реальном времени включает в себя расшифровку звука в распознанный текст с помощью преобразования [речи в текст](speech-to-text.md) из служб распознавания речи Azure. Этот текст доступен как для программы-робота, так и для клиентского приложения, как это расшифрованной.
|[Преобразование текста в речь](text-to-speech.md) | Текстовые ответы от Bot будут синтезированы с помощью преобразования [текста в речь](text-to-speech.md) из служб распознавания речи Azure. Затем этот синтез будет доступен вашему клиентскому приложению в качестве звукового потока. Корпорация Майкрософт предлагает возможность создать собственный высококачественный голосовой речевой TTS, который позволяет получить дополнительные сведения о вашей торговой марке, чтобы узнать больше, [свяжитесь с нами](mailto:mstts@microsoft.com).
|[Прямое речевое распознавание](https://docs.microsoft.com/azure/bot-service/bot-service-channel-connect-directlinespeech) | В качестве канала в Bot-среде Прямая речь обеспечивает гладкое и простое подключение между клиентским приложением, совместимым роботом и возможностями служб распознавания речи Azure. Дополнительные сведения о настройке программы Bot для использования канала голосовой речи см. на [странице документации по платформе Bot](https://docs.microsoft.com/azure/bot-service/bot-service-channel-connect-directlinespeech).

> Параметры настройки зависят от языка и языкового стандарта (см. раздел [Поддерживаемые языки](supported-languages.md)).

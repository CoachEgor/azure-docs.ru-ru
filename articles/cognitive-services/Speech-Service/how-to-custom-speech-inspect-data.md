# <a name="inspect-custom-speech-data"></a>Проверка пользовательских данных речи

> [!NOTE]
> Эта страница предполагает, что вы прочитали [данные теста Подготовка для пользовательской речи](how-to-custom-speech-test-data.md) и загрузили набор данных для проверки.

Custom Speech предоставляет инструменты, позволяющие визуально проверить качество распознавания модели, сравнивая аудиоданные с соответствующим результатом распознавания. С [портала Custom Speech](https://speech.microsoft.com/customspeech)вы можете воспроизвести загруженный звук и определить, является ли предоставленный результат распознавания правильным. Этот инструмент позволяет быстро проверить качество базовой модели речи к тексту от Microsoft или обученной пользовательской модели без необходимости расшифровывать аудиоданные.

В этом документе вы узнаете, как визуально проверить качество модели, используя обучающие данные, которые вы ранее загрузили.

На этой странице вы узнаете, как визуально проверить качество базовой модели речи к тексту корпорации Майкрософт и/или пользовательской модели, которую вы обучили. Вы будете использовать данные, загруженные на вкладку **Данные,** для тестирования.

## <a name="create-a-test"></a>Создание теста

Следуйте этим инструкциям, чтобы создать тест:

1. Войти на [портал Custom Speech](https://speech.microsoft.com/customspeech).
2. Перейдите к **речи к тексту > пользовательской речи > тестирования.**
3. Нажмите **Добавить тест**.
4. Выберите **качество проверки (только аудио данные)**. Дайте тесту имя, описание и выберите набор аудиоданных.
5. Выберите до двух моделей, которые вы хотите протестировать.
6. Нажмите кнопку **Создать**.

После успешного тестирования можно сравнить модели бок о бок.

## <a name="side-by-side-model-comparisons"></a>Сравнение моделей

При успешном _Succeeded_статусе теста нажмите на имя элемента теста, чтобы просмотреть детали теста. На этой странице подробной информации перечислены все высказывания в наборе данных, указывающие результаты распознавания двух моделей наряду с транскрипцией из представленного набора данных.

Чтобы проверить сравнение бок о бок, можно переключить различные типы ошибок, включая вставку, удаление и замену. Слушая аудио- и сравнивая результаты распознавания в каждой колонке (показывая транскрипцию с маркировкой человека и результаты двух моделей речи к тексту), вы можете решить, какая модель отвечает вашим потребностям и где необходимы улучшения.

Проверка качества тестирования полезна для проверки, достаточно ли качества конечной точки распознавания речи для приложения. Для объективной меры точности, требующей транскрибированного звука, следуйте инструкциям, найденным в [Оценке точности](how-to-custom-speech-evaluate-data.md).

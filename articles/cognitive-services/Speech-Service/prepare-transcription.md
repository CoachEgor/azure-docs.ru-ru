# <a name="transcription-guidelines-for-using-the-speech-service"></a>Рекомендации по транскрибированию для обучения службы распознавания речи

Чтобы настроить **преобразование речи в текст** или **преобразование текста в речь**, необходимо предоставить текст, наряду с речью. Каждая строка текста соответствует одному фрагменту речи. Текст должен как можно более точно соответствовать речи. Текст называется *расшифровкой*, и его необходимо создавать в определенном формате.

Службы речи нормализации входных данных для сохранения согласованного текста.

В этой статье описываются оба типа нормализации. Рекомендации немного различаются для разных языков.

## <a name="us-english-en-us"></a>Английский, США (en-US)

Текстовые данные должны содержать по одному высказыванию в строке и должны быть записаны в виде обычного текста только с использованием кодировки ASCII.

Избегайте использования расширенной кодировки (Latin-1) или знаков препинания Юникода. Эти символы могут быть включены случайно при подготовке данных в текстовом редакторе или импорте данных из веб-страниц. Замените эти знаки соответствующими знаками ASCII. Например: 

| Нерекомендуемые символы | Замена |
|----- | ----- |
| “Hello world” (открывающие и закрывающие двойные кавычки) | "Hello world" (двойные кавычки) |
| John’s day (правая одинарная кавычка) | John's day (апостроф) |
| it was good—no, it was great! (длинное тире) | it was good--no, it was great! (дефисы) |

### <a name="text-normalization-rules-for-english"></a>Правила нормализации текста для английского языка

Службы речи выполните следующие правила нормализации.

* использование строчных букв для всего текста;
* удаление всех знаков пунктуации, кроме апострофов внутри слов;
* расширение чисел в речевую форму, включая денежные суммы.

Ниже приведены некоторые примеры:

| Исходный текст. | После нормализации |
|----- | ----- |
| "Holy cow!" said Batman. | holy cow said batman |
| "What?" said Batman's sidekick, Robin. | what said batman's sidekick robin |
| Go get - em! | go get em |
| I'm double-jointed | I'm double jointed |
| 104 Elm Street | one oh four Elm street |
| Tune to 102.7 | tune to one oh two seven |
| Pi is about 3.14 | pi is about three point one four |
| It costs $3.14 | it costs three fourteen |

Примените следующую нормализацию к расшифровке текста:

* Сокращения должны записываться словами.
* Нестандартные числовые строки (например, некоторые даты или бухгалтерские формы) должны быть написаны словами.
* Слова с неалфавитными знаками или смешанными буквенно-цифровыми знаками должны записываться так, как произносятся.
* Аббревиатуры, которые произносятся как слова, остаются без изменений (например, radar, laser, RAM или NATO).
* Сокращения, которые произносятся как отдельные буквы, записываются буквами, разделенными пробелами (например, IBM, CPU, FBI, TBD или NaN).

Ниже приведены некоторые примеры:

| Исходный текст. | После нормализации |
|----- | ----- |
| 14 NE 3rd Dr. | fourteen northeast third drive |
| Dr. Bruce Banner | Doctor Bruce Banner |
| James Bond, 007 | James Bond, double oh seven |
| Ke$ha | Kesha |
| How long is the 2x4 | How long is the two by four |
| The meeting goes from 1-3pm | The meeting goes from one to three pm |
| my blood type is O+ | My blood type is O positive |
| water is H20 | water is H 2 O |
| play OU812 by Van Halen | play O U 8 1 2 by Van Halen |
| UTF-8 with BOM | U T F 8 with BOM |

## <a name="chinese-zh-cn"></a>Китайский (zh-CN)

Текстовые данные, который загружается в службы речи Custom следует использовать кодировку UTF-8 с отметкой порядка байтов. Высказывания должны записываться в файл по одному на строку.

Избегайте использования полуширинных знаков препинания. Эти символы могут быть включены случайно при подготовке данных в текстовом редакторе или импорте данных из веб-страниц. Замените их соответствующими знаками полной ширины. Например: 

| Нерекомендуемые символы | Замена |
|----- | ----- |
| "你好" (открывающие и закрывающие двойные кавычки) | "你好" (двойные кавычки) |
| 需要什么帮助? (вопросительный знак) | 需要什么帮助？ |

### <a name="text-normalization-rules-for-chinese"></a>Правила нормализации текста для китайского языка

Службы речи выполните следующие правила нормализации.

* удаление всех знаков препинания;
* расширение чисел до речевой формы;
* преобразование полноширинных букв в полуширинные буквы;
* Использование прописных букв для всех английских слов

Ниже приведены некоторые примеры:

| Исходный текст. | После нормализации |
|----- | ----- |
| 3.1415 | 三 点 一 四 一 五 |
| ￥3.5 | 三 元 五 角 |
| w f y z | W F Y Z |
| 1992年8月8日 | 一 九 九 二 年 八 月 八 日 |
| 你吃饭了吗? | 你 吃饭 了 吗 |
| 下午5:00的航班 | 下午 五点 的 航班 |
| 我今年21岁 | 我 今年 二十 一 岁 |

До импорта текста примените к нему следующую нормализацию:

* Сокращения должны записываться словами, которые отражают речевую форму.
* Записывайте числовые строки в речевой форме.

Ниже приведены некоторые примеры:

| Исходный текст. | После нормализации |
|----- | ----- |
| 我今年21 | 我今年二十一 |
| 3号楼504 | 三号 楼 五 零 四 |

## <a name="other-languages"></a>Остальные языки

Для текстовых данных, отправляемых в службу **преобразования речи в текст**, следует использовать кодировку UTF-8 с метками порядка следования байтов. Высказывания должны записываться в файл по одному на строку.

> [!NOTE]
> В следующих примерах используется немецкий язык. Однако эти правила применимы для всех языков, кроме английского или китайского.

### <a name="text-normalization-rules-for-german"></a>Правила нормализации текста для немецкого языка

Службы речи выполните следующие правила нормализации.

* использование строчных букв для всего текста;
* удаление всех знаков препинания, включая различные виды кавычек ("test", 'test', "test„ или «test»);
* исключение строк, содержащих специальные знаки из набора ¢ ¤ ¥ ¦ § © ª ¬ ® ° ± ² µ × ÿ Ø¬¬
* расширение чисел в речевую форму, включая денежные суммы;
* надстрочные знаки допустимы только для a, o, u; другие будут заменены "th" или исключены.

Ниже приведены некоторые примеры:

| Исходный текст. | После нормализации |
|----- | ----- |
| Frankfurter Ring | frankfurter ring |
| ¡Eine Frage! | eine frage |
| wir, haben | wir haben |

До импорта текста примените к нему следующую нормализацию:

* Десятичным разделителем должна быть запятая (","), а не точка (".").
* Разделителем между часами и минутами должно быть двоеточие (":"), а не точка (".") (например, 12:00 Uhr).
* Такие сокращения, как "ca.", не заменяются. Мы рекомендуем использовать полную форму.
* Удаляются четыре основные математические операторы (+, -, \*, /) Мы рекомендуем заменить их речевыми формами: plus, minus, mal, geteilt.
* То же правило применяется к операторам сравнения (=, < и >). Мы рекомендуем заменить их на "gleich", "kleiner als" и "grösser als".
* Используйте дроби, например 3/4, в словесной форме (например, "drei viertel" вместо ¾).
* Замените символ € словом "Euro".

Ниже приведены некоторые примеры:

| Исходный текст. | После нормализации пользователя | После нормализации системы |
|--------  | ----- | -------- |
| Es ist 12.23 Uhr | Es ist 12:23 Uhr | es ist zwölf uhr drei und zwanzig uhr |
| {12.45} | {12,45} | zwölf komma vier fünf |
| 2 + 3 - 4 | 2 plus 3 minus 4 | zwei plus drei minus vier|

# <a name="quickstart-create-a-custom-commands-application-with-parameters-preview"></a>Краткое руководство. Создание пользовательского командного приложения с параметрами (Предварительная версия)

В [предыдущей статье](./quickstart-custom-speech-commands-create-new.md)вы создали простое приложение настраиваемых команд без параметров.

В этой статье вы будете расширять это приложение, чтобы использовать параметры, чтобы он мог обрабатывать включение и отключение нескольких устройств.

## <a name="create-parameters"></a>Create Parameters

1. Откройте [ранее созданный](./quickstart-custom-speech-commands-create-new.md) проект.
1. Измените существующую команду, чтобы включить и отключить несколько устройств.
1. Так как команда теперь обрабатывается и будет отключена, переименуйте команду `TurnOnOff`в.
   - В левой области выберите `TurnOn` команду и щелкните `...` значок рядом с `+ New command` пунктом в верхней части панели.
   
   - Выберите `Rename` значок. Во всплывающем окне **команда переименования** измените **имя** на `TurOnOff`. Щелкните **Сохранить**.

1. Затем создается новый параметр, который указывает, нужно ли включать или отключать устройство.
   - В `+ Add` верхней части средней панели щелкните значок. В раскрывающемся списке выберите **параметр**.
   - В области справа можно увидеть раздел Конфигурация **параметров** .
   - Добавить значение для **имени**.
   - Установите флажок " **обязательный** ". В окне **Добавление ответа для обязательного параметра** выберите **простой редактор** и **первый вариант**, добавить
        ```
        On or Off?
        ```
   - Выберите **Обновить**.

## <a name="add-example-sentences"></a>Добавить примеры предложений

При использовании команд с параметрами рекомендуется добавить примеры предложений, охватывающие все возможные сочетания. Пример:

1. Полные сведения о параметрах —`turn {OnOff} the {SubjectDevice}`
1. Сведения о частичном параметре-`turn it {OnOff}`
1. Нет сведений о параметрах —`turn something`

Примеры предложений с различными сведениями позволяют приложению настраиваемых команд разрешать как однозначные разрешения, так и множественные разрешения с частичными сведениями.

Учитывая это, измените примеры предложений, чтобы использовать указанные ниже параметры.

```
turn {OnOff} the {SubjectDevice}
{SubjectDevice} {OnOff}
turn it {OnOff}
turn something {OnOff}
turn something
```
> [!TIP]
> В редакторе примеров предложений для ссылки на параметры используются фигурные скобки. - `turn {OnOff} the {SubjectDevice}`Используйте вкладку для автоматического завершения с помощью созданных ранее параметров.

## <a name="add-parameters-to-completion-rules"></a>Добавление параметров в правила завершения

Измените правило завершения, созданное в [предыдущем кратком руководстве](./quickstart-custom-speech-commands-create-new.md).

1. В разделе **условия** добавьте новое условие, выбрав **+ Добавить условие** .
1. В новом всплывающем окне **новое условие**выберите `Required parameters` из раскрывающегося списка **тип** . В списке ниже выберите `OnOff` и. `SubjectDevice`
1. Нажмите кнопку **Создать**.
1. В разделе **действия** измените существующее действие отправка речевого ответа, наведя указатель мыши на действие и щелкнув значок редактирования. На этот раз мы будем использовать только что созданные `OnOff` параметры и `SubjectDevice`

    ```
    Ok, turning {OnOff} the {SubjectDevice}
    ```

# <a name="quickstart-create-a-custom-command-with-parameters-preview"></a>Быстрый запуск: Создайте пользовательскую команду с параметрами (Предварительный просмотр)

В [предыдущей статье](./quickstart-custom-speech-commands-create-new.md)мы создали новый проект пользовательских команд для реагирования на команды без параметров.

В этой статье мы расширим это приложение параметрами, чтобы оно работало с включением и выключением нескольких устройств.

## <a name="create-parameters"></a>Create Parameters

1. Откройте проект, [который мы создали ранее](./quickstart-custom-speech-commands-create-new.md)
1. Поскольку Командование теперь будет обрабатывать и выключать, переименовать командование в "TurnOnOff"
   - Нависните над названием Команды и выберите значок изменения, чтобы изменить имя
1. Создайте новый параметр, чтобы представить, хочет ли пользователь включить или выключить устройство
   - Выберите `+` значок рядом с разделом Параметры

   > [!div class="mx-imgBorder"]
   > ![Создание параметра](media/custom-speech-commands/create-on-off-parameter.png)

   | Параметр            | Рекомендуемое значение     | Описание                                                                                               |
   | ------------------ | ------------------- | --------------------------------------------------------------------------------------------------------- |
   | name               | OnOff               | Описательное имя для параметра                                                                     |
   | Является глобальным          | unchecked           | Чекбокс, указывающий, применяется ли значение для этого параметра глобально ко всем командам в проекте |
   | Обязательно           | checked             | Чекбокс, указывающий, требуется ли значение для этого параметра перед завершением команды          |
   | Шаблон ответа  | "- В выключенное или выключенное?"      | Запрос для запроса значения этого параметра, когда он не известен                                       |
   | Тип               | Строка              | Тип параметра, например число, строка или время даты                                               |
   | Параметр Configuration      | Строка Список         | Для строк строк строки список ограничивает входные данные набором возможных значений                                      |
   | Значения списка строк | on, off             | Для параметра строки, набор возможных значений и их синонимы                                |

   - Затем выберите `+` значок еще раз, чтобы добавить второй параметр для представления имени устройств. В этом примере телевизор и вентилятор

   | Параметр            | Рекомендуемое значение       | Описание                                                                                               |
   | ------------------ | --------------------- | --------------------------------------------------------------------------------------------------------- |
   | name               | SubjectDevice         | Описательное имя для параметра                                                                     |
   | Является глобальным          | unchecked             | Чекбокс, указывающий, применяется ли значение для этого параметра глобально ко всем командам в проекте |
   | Обязательно           | checked               | Чекбокс, указывающий, требуется ли значение для этого параметра перед завершением команды          |
   | Шаблон ответа  | "- Какое устройство?"     | Запрос для запроса значения этого параметра, когда он не известен                                       |
   | Тип               | Строка                | Тип параметра, например число, строка или время даты                                               |
   | Параметр Configuration      | Строка Список           | Для строк строк строки список ограничивает входные данные набором возможных значений                                      |
   | Значения списка строк | телевизор, вентилятор               | Для параметра строки, набор возможных значений и их синонимы                                |
   | Синонимы (тв)      | телевидение, телевизор     | Дополнительные синонимы для каждого возможного значения параметра строки                                      |

## <a name="add-sample-sentences"></a>Добавление образца приговоров

С параметрами полезно добавить примеры предложений, которые охватывают все возможные комбинации. Пример:

1. Полная информация о параметрах -`"turn {OnOff} the {SubjectDevice}"`
1. Частичная информация о параметрах -`"turn it {OnOff}"`
1. Нет информации о параметрах -`"turn something"`

Примеры предложений с различными объемами информации позволяют приложению Custom Commands разрешать как одноразовые, так и многоразовые разрешения с частичной информацией.

Имея это в виду, отспособьте примеры предложений, чтобы использовать параметры, как показано ниже.

> [!TIP]
> В редакторе Sample Sentences используются фигурные скобки для обозначения ваших параметров. - `turn {OnOff} the {SubjectDevice}`Используйте завершение вкладок для обозначения ранее созданных параметров.

> [!div class="mx-imgBorder"]
> ![Примеры предложений с параметрами](media/custom-speech-commands/create-parameter-sentences.png)

```
turn {OnOff} the {SubjectDevice}
{SubjectDevice} {OnOff}
turn it {OnOff}
turn something {OnOff}
turn something
```

## <a name="add-parameters-to-completion-rule"></a>Добавление параметров в правило Завершения

Измените правило Завершения, [созданное в предыдущем быстром запуске:](./quickstart-custom-speech-commands-create-new.md)

1. Добавьте новое условие и выберите необходимый параметр. Выберите `OnOff` и`SubjectDevice`
1. Отспособите действие реакции речи для использования `OnOff` и: `SubjectDevice`

   ```
   - Ok, turning {OnOff} the {SubjectDevice}
   ```

## <a name="try-it-out"></a>Попробуйте продукт

Откройте панель чата Test и попробуйте несколько взаимодействий.

- Вход: выключите телевизор
- Выход: Ok, выключив телевизор

- Вход: выключить телевизор
- Выход: Ok, выключив телевизор

- Вход: выключите его
- Выход: Какое устройство?
- Вход: телевизор
- Выход: Ok, выключив телевизор

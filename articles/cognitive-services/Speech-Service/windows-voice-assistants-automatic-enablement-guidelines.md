# <a name="privacy-guidelines-for-voice-assistants-on-windows"></a>Рекомендации по конфиденциальности для речевых помощников в Windows

Важно, чтобы пользователи получили четкие сведения о том, как собираются и используются их голоса, и важно, что они получают контроль над тем, когда и как происходит эта коллекция. Эти основные аспекты конфиденциальности — *раскрытие* и *согласие* — особенно важны для помощников по голосовым сообщениям, интегрированных в Windows, с учетом того, что их использование всегда прослушивается.

Разработчики, создающие речевые помощники в Windows, должны включать в свои приложения четкие элементы пользовательского интерфейса, отражающие возможности помощника по прослушиванию.

> [!NOTE]
> Невозможность предоставить соответствующее раскрытие и согласие для приложения помощника, в том числе после обновления приложения, может привести к тому, что помощник станет недоступным для активации голоса, пока не будут устранены проблемы конфиденциальности. 

## <a name="minimum-requirements-for-feature-inclusion"></a>Минимальные требования для включения компонентов

Пользователи Windows могут видеть и контролировать доступность приложений помощника в **`Settings > Privacy > Voice activation`**.

 > [!div class="mx-imgBorder"]
 > [![конфиденциальность — список приложений](media/voice-assistants/windows_voice_assistant/privacy-app-listing.png "Запись настройки конфиденциальности для активации голоса Windows для приложения помощника")](media/voice-assistants/windows_voice_assistant/privacy-app-listing.png#lightbox)

Чтобы иметь право на включение в этот список, приложение должно выполнить следующие действия:

1. Выразительное уведомление пользователей о том, что он будет прослушивать ключевое слово, даже если приложение не выполняется и что ключевое слово
1. Включите описание того, как будут использоваться данные голоса пользователя, включая ссылку или ссылку на соответствующие политики конфиденциальности.
1. Сообщите пользователям, что в дополнение к любым параметрам в приложении пользователи могут просматривать и изменять параметры конфиденциальности в **`Settings > Privacy > Voice activation`**, при необходимости включив ссылку `ms-settings:privacy-voiceactivation` на протокол для прямого доступа.

После выполнения этих требований и получения утверждения от корпорации Майкрософт приложение Assistant появится в списке приложений для активации голоса после регистрации с помощью `Windows.ApplicationModel.ConversationalAgent` API, и пользователи смогут предоставить согласие на активацию для приложения при активации ключевого слова. По умолчанию оба этих параметра являются `Off` обязательными, и пользователь должен вручную посетить страницу параметров, чтобы включить.

> [!NOTE]
> Во всех случаях для разрешения активации голоса требуется разрешение Microphone. Если приложение помощника не имеет доступа к микрофону, оно не будет иметь права на активацию голоса и будет отображаться в параметрах конфиденциальности активации голоса в отключенном состоянии.

## <a name="additional-requirements-for-inclusion-in-microphone-consent"></a>Дополнительные требования к включению в согласие на микрофон

Авторы помощников, которые хотят сделать так, чтобы пользователи могли принять участие в активации голоса, могут сделать это, добавив к ним несколько дополнительных требований. После этого для приложения помощника по умолчанию будет выбран один раз (и только один раз) для `On` доступа к микрофону. Это избавляет от необходимости дополнительного пути к параметрам перед активацией помощником.

Дополнительные требования — приложение помощника должно:

1. **Перед** запросом согласия на микрофон (например, с помощью `AppCapability.RequestAccessAsync` API) Предоставьте пользователю представление о том, что приложение помощника должно прослушивать голосовое сообщение пользователя для получения ключевого слова, даже если приложение не работает и ему требуется согласие пользователя.
2. Включите все необходимые сведения о политиках использования и конфиденциальности данных, **прежде чем** запрашивать доступ к `Windows.ApplicationModel.ConversationalAgent` микрофону или использовать API
3. Избегайте какой-либо директивы или ведущих слов (например, "нажмите кнопку Да в следующем запросе") в потоке возможностей, чтобы закрыть звуковую запись и запросить разрешение.

После выполнения этих требований в списке приложений, подходящих для активации голоса в `enabled` состоянии после предоставления доступа к микрофону, появится соответствующее приложение.

> [!NOTE]
> Активация голоса выше блокировки не предусматривает автоматическое включение с использованием микрофона и по-прежнему требует от пользователя посетить страницу "конфиденциальность активации речи", чтобы включить доступ на основе блокировки для помощника.

# <a name="add-validations-to-a-command-parameter-in-a-custom-commands-preview-application"></a>Добавление проверок в параметр команды в предварительной версии приложения пользовательских команд

В этой статье вы узнаете, как добавлять проверки в параметры и запрашивать исправление.

Выполните действия, описанные в следующих статьях:

> [!div class="checklist"]
 
> * [Краткое руководство. создание предварительной версии приложения настраиваемых команд](./quickstart-custom-speech-commands-create-new.md)
> * [Краткое руководство. создание предварительной версии приложения настраиваемых команд с параметрами](./quickstart-custom-speech-commands-create-parameters.md)

## <a name="create-a-settemperature-command"></a>Создание команды SetTemperature

Чтобы продемонстрировать проверки, создайте новую команду, позволяющую пользователям устанавливать температуру.

1. В окне " [речь Studio](https://speech.microsoft.com/)" Откройте созданное приложение Custom Commands Preview.
1. Создайте новую команду **SetTemperature** .
1. Добавьте параметр температуры, имеющий следующую конфигурацию:

   | Конфигурация параметров           | Рекомендуемое значение    |Описание                 |                                    
   | ----------------- | ----------------------------------| -------------|
   | **имя**;              | **Первого**                       | Описательное имя параметра                                |
   | **Обязательное**          | Флажок установлен                           | Флажок, указывающий, требуется ли значение этого параметра перед выполнением команды |
   | **Ответ на обязательный параметр**     | **Простой редактор — >, какую температуру вам хотелось бы?**  | Запрос на запрос значения этого параметра, если он неизвестен |
   | **Тип**              | **Число**                            | Тип параметра, например число, строка, Дата и время или география   |

1. Добавьте проверку для параметра температуры.

    1. На странице Конфигурация **параметров** для параметра температура выберите **добавить проверку** в разделе **проверки** .

    1. Во всплывающем окне **Новая проверка** настройте проверку следующим образом.
  
       | Конфигурация параметров         | Рекомендуемое значение                                          | Описание                                                                        |
       | ----------------- | -------------------------------------------------------- | ------------------------------------------------------------------------------------------------ |
       | **Минимальное значение**        | **60**               | Для числовых параметров — минимальное значение, которое может предположить этот параметр. |
       | **Максимальное значение**        | **80**               | Для числовых параметров — максимальное значение, которое может предположить этот параметр. |
       | **Неудачный ответ — простой редактор**| **Первая разновидность, я могу установить только от 60 до 80 градусов.**      | Запрашивать новое значение, если проверка не пройдена                                       |

       > [!div class="mx-imgBorder"]
       > ![Добавление проверки диапазона](media/custom-speech-commands/validations-add-temperature.png)

1. Выберите **Создать**.

1. Добавьте несколько примеров предложений.

   ```
   set the temperature to {Temperature} degrees
   change the temperature to {Temperature}
   set the temperature
   change the temperature
   ```

1. Добавьте правило завершения со следующей конфигурацией. Это правило подтверждает результат.

   | Параметр    | Рекомендуемое значение                                           |Описание                                     |
   | ---------- | --------------------------------------------------------- |-----|
   | Имя       | Сообщение с подтверждением                                      |Имя, описывающее назначение правила. |
   | **Условия** | **Обязательные параметры — температура**                       |Условия, определяющие, когда может выполняться правило    |   
   | **Действия**    | **Отправка речевого ответа — ОК, установка температуры в {температура} градусов** | Действие, выполняемое, если условие правила имеет значение true |

> [!TIP]
> В этом примере для подтверждения результата используется речевой ответ. Примеры выполнения команды с помощью действия клиента см. в разделе [как выполнять команды на клиенте с помощью пакета SDK для распознавания речи](./how-to-custom-speech-commands-fulfill-sdk.md).

## <a name="try-it-out"></a>Попробуйте!

1. Щелкните **Обучить**.

1. По завершении обучения выберите **тест**и попробуйте выполнить следующие действия:

    - Входные данные: установите температуру в 72 градусов.
    - Выходные данные: ОК, установка температуры в 72 градусов
    - Входные данные: установите температуру в 45 градусов.
    - Выходные данные: приносим к сожалению, можно задать значение в диапазоне от 60 до 80 градусов
    - Входные данные: сделайте его 72 градусов
    - Выходные данные: ОК, установка температуры в 72 градусов

# <a name="migrate-from-the-translator-speech-api-to-the-speech-service"></a>Миграция из API речи переводчика в службу речи

Используйте эту статью для переноса приложений из API речи переводчика Майкрософт в [службу speech.](index.yml) В этом руководстве излагаются различия между API-иречи переводчика и службой речевого сообщения, а также предлагаются стратегии миграции приложений.

> [!NOTE]
> Ключ подписки На API Translator Не будет принят службой Speech. Вам нужно будет создать новую подписку службы речи.

## <a name="migration-strategies"></a>Стратегии миграции

Если у вас или вашей организации есть приложения в разработке или производстве, которые используют API речи переводчика, их следует обновить, чтобы использовать службу speech. Ознакомьтесь с документацией [службы речевого обслуживания](index.yml) для доступных SDK, образцов кода и учебников. При выполнении миграции учитывайте следующие факторы.

* Служба речевой речи не предлагает глобальную конечную точку. Необходимо определить, будет ли ваше приложение эффективно функционировать, используя одну региональную конечную точку для всего трафика. В противном случае используйте геолокацию, чтобы определить наиболее эффективную конечную точку.

* Если приложение использует длительные подключения и не поддерживает доступный пакет SDK, можно воспользоваться подключением WebSocket. Управляйте 10-минутным временем ожидания путем повторного подключения в нужное время.

* Если приложение использует API-код Translator Text и Translator Speech API для включения пользовательских моделей перевода, вы можете добавить идентификации категории непосредственно с помощью службы Speech.

* В отличие от API речи переводчика, служба речевой речи может выполнять переводы на несколько языков в одном запросе.

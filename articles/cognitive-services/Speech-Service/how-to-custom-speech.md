# <a name="what-is-custom-speech"></a>Что такое Custom речи?

[Настраиваемое преобразование речи](https://aka.ms/custom-speech) — это набор онлайн-средства, позволяющие оценить и повысить точность распознавания речи в текст корпорации Майкрософт для приложений, средств и продуктов. Все, что требуется, чтобы приступить к работе множество звуковых файлов теста. Перейдите по ссылкам ниже, чтобы приступить к созданию пользовательского взаимодействия речи в текст.

## <a name="whats-in-custom-speech"></a>Что такое настраиваемое преобразование речи?

Прежде чем речевой Custom вам потребуется учетная запись Azure и подписка на службы распознавания речи. После того, как учетную запись, вы можно Подготовка данных, обучение и проверка моделей, проверить качество распознавания, оценить точность и в конечном счете развернуть и использовать пользовательские модели речи в текст.

На этой схеме показаны компоненты, составляющие речи пользовательского портала. Используйте приведенные ниже ссылки для получения дополнительных сведений о каждом шаге.

![Описываются различные компоненты, составляющие речи пользовательского портала.](./media/custom-speech/custom-speech-overview.png)

1. [Подпишитесь и создайте проект](#set-up-your-azure-account) — создать учетную запись Azure и подписки служб речи. Эта подписка единой предоставляет доступ к речи в текст, текст в речь, перевод речи и портал распознавания речи. Затем через подписку служб речи, создайте свой первый проект Custom речи.

2. [Отправить тестовые данные](how-to-custom-speech-test-data.md) -Отправка проверочные данные (аудиофайлы) для оценки корпорации Майкрософт речи в текст предложения для приложений, средств и продуктов.

3. [Проверить качество распознавания](how-to-custom-speech-inspect-data.md) -воспроизвести отправленного аудио и проверить качество распознавания речи тестовых данных с помощью портала Custom речи. Количественных измерений, см. в разделе [проверки данных](how-to-custom-speech-inspect-data.md).

4. [Оценить точность](how-to-custom-speech-evaluate-data.md) -оценить точность модели речи в текст. Речи пользовательский портал будет предоставлять *частота ошибок Word*, который может использоваться для определения необходимости дополнительного обучения. Если вас устраивают точность, интерфейсы API службы распознавания речи можно использовать напрямую. Если вы хотите повысить точность благодаря относительный среднее из 5 – 20%, используйте **обучения** вкладка на портале, чтобы отправить дополнительными обучающими данными, например с меткой human табель и соответствующем текстовом.

5. [Обучение модели](how-to-custom-speech-train-model.md) — повысить точность модели речи в текст, предоставляя письменного табель (1000 10 часов) и связанные с текстового (< 200 МБ) вместе с вашей аудио тестовых данных. Эти данные помогают для обучения модели речи в текст. После обучения, повторное тестирование, и если результат вас устраивает, вы можете развернуть свою модель.

6. [Развертывание модели](how-to-custom-speech-deploy-model.md) - Создание пользовательской конечной точки для вашей модели речи в текст и использовать его в приложениях, средств или продуктов.

## <a name="set-up-your-azure-account"></a>Настройка учетной записи Azure

Прежде чем можно использовать портал распознавания речи для создания пользовательской модели требуется подписка службы распознавания речи. Выполните эти инструкции для создания стандартной подписки службы распознавания речи. [Создание подписки речи](https://docs.microsoft.com/en-us/azure/cognitive-services/speech-service/get-started#new-azure-account).

> [!NOTE]
> Убедитесь, что создание standard (S0) подписок, подписки на бесплатную пробную версию (F0) не поддерживаются.

После создания учетной записи Azure и подписка на службы распознавания речи, необходимо войти речи пользовательский портал и подключение вашей подписки.

1. Получите ключ подписки служб речи на портале Azure.
2. Для входа на [речи пользовательский портал](https://aka.ms/custom-speech).
3. Выберите подписку, которую нужно для работы на и создание структуры проекта речи.
4. Если вы хотите изменить свою подписку, используйте **шестеренки** значок, расположенный в верхней панели навигации.

## <a name="how-to-create-a-project"></a>Создание проекта

Содержимое как данных, моделей, тесты и конечные точки организованы в **проекты** речи пользовательского портала. Каждый проект относится к домену и страны и языка. Например можно создать проект для центры обработки вызовов, использующих английский в США.

Чтобы создать свой первый проект, выберите **речи речи в текст или настраиваемый**, нажмите кнопку **новый проект**. Следуйте инструкциям мастера для создания проекта. После создания проекта, вы увидите четыре вкладки: **Данные**, **тестирование**, **обучения**, и **развертывания**. По ссылкам ниже в [дальнейшие действия](#next-steps) Дополнительные способы использования каждой вкладки.

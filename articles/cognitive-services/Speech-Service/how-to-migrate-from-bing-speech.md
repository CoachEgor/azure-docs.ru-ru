# <a name="migrate-from-bing-speech-to-the-speech-service"></a>Миграция из Bing Speech в речевую службу

Используйте эту статью для переноса приложений из API речи Bing в службу speech.

В этой статье излагаются различия между AA, полученными в связи с речью Bing Speech, и предлагаются стратегии миграции приложений. Ключ подписки Bing Speech API не будет работать с службой speech; Вам понадобится новая подписка на услугу speech.

Ключ подписки одного сервиса Speech предоставляет доступ к следующим функциям. Каждый тарифицируется отдельно, поэтому плата взимается только за функции, которые вы используете.

* [Речево-текст](speech-to-text.md)
* [Настраиваемое преобразование речи в текст](https://cris.ai)
* [Текст к речи](text-to-speech.md)
* [Настраиваемое преобразование текста в речь](how-to-customize-voice-font.md)
* [Перевод речи](speech-translation.md) (не включает [перевод текста](../translator/translator-info-overview.md))

[Пакет SDK для службы "Речь"](speech-sdk.md) является функциональной заменой клиентских библиотек службы "Распознавание речи Bing", однако использует другой API.

## <a name="migration-strategies"></a>Стратегии миграции

Если у вас или вашей организации есть приложения в разработке или производстве, которые используют API Bing Speech, их следует обновить, чтобы как можно скорее использовать службу speech. Ознакомьтесь с [документацией службы речевого обслуживания](index.yml) для доступных SDK, образцов кода и учебников.

Служба [речевого обслуживания REST AIs](rest-apis.md) совместима с AA, вспениваемыми в речь Bing. Если в настоящее время вы используете APIs Bing Speech REST, вам нужно только изменить конечную точку REST и переключиться на ключ подписки службы речи.

Протоколы службы речевого обслуживания WebSockets также совместимы с протоколами, используемыми Bing Speech. Для разработки приложений мы рекомендуем использовать пакет SDK службы "Речь", а не протоколы WebSocket. Рекомендуется также перенести существующий код в этот пакет SDK. Однако, как и с REST API, существующий код, использующий службу "Распознавание речи Bing" через WebSocket, требует только изменения конечной точки и обновления ключа.

Если вы используете клиентскую библиотеку службы "Распознавание речи Bing" для определенного языка программирования, переход на [пакет SDK для службы "Речь"](speech-sdk.md) потребует внесения изменений в приложение из-за отличий в API. Пакет SDK службы "Речь" может упростить код, а также предоставить доступ к новым функциям. Speech SDK доступен на самых разных языках программирования. Интерфейсы API на всех платформах похожи, что упрощает разработку для нескольких платформ.

Служба речевой речи не предлагает глобальную конечную точку. Необходимо определить, будет ли ваше приложение эффективно функционировать, используя одну региональную конечную точку для всего трафика. В противном случае используйте геолокацию, чтобы определить наиболее эффективную конечную точку. Вам нужна отдельная подписка на услуги речевого обслуживания в каждом используемом регионе.

Если приложение использует длительные подключения и не поддерживает доступный пакет SDK, можно воспользоваться подключением WebSocket. Управляйте 10-минутным временем ожидания путем повторного подключения в нужное время.

Чтобы начать работу с пакетом SDK службы "Речь":

1. Загрузите [пакет SDK службы "Речь"](speech-sdk.md).
1. Работа через службу речи [quickstart руководства](~/articles/cognitive-services/Speech-Service/quickstarts/speech-to-text-from-microphone.md?pivots=programming-language-csharp&tabs=dotnet) и [учебники](how-to-recognize-intents-from-speech-csharp.md). Также ознакомьтесь с [примерами кода](samples.md), чтобы получить опыт работы с новыми интерфейсами API.
1. Обновите приложение, чтобы использовать службу Speech.

Кроме того, свяжитесь с нами, если для поддержки вам требуется [план технической поддержки](https://azure.microsoft.com/support/plans/).

Для службы речевой речи, поддержки SDK и API посетите [страницу поддержки](support.md)службы speech.

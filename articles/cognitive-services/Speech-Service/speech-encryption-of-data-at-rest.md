# <a name="speech-service-encryption-of-data-at-rest"></a>Речевой сервис шифрования данных в состоянии покоя

Служба речевых сообщений автоматически шифрует ваши данные, когда они сохраняются в облаке. Шифрование речевых служб защищает ваши данные и помогает вам выполнять свои организационные обязательства по обеспечению безопасности и соблюдению нормативных требований.

## <a name="about-cognitive-services-encryption"></a>О шифровании когнитивных служб

Данные шифруются и расшифровываются с помощью [256-битного](https://en.wikipedia.org/wiki/Advanced_Encryption_Standard) шифрования [AES FIPS 140-2.](https://en.wikipedia.org/wiki/FIPS_140-2) Шифрование и расшифровка прозрачны, то есть шифрование и доступ управляются для вас. Ваши данные защищены по умолчанию, и вам не нужно изменять код или приложения, чтобы воспользоваться преимуществами шифрования.

## <a name="about-encryption-key-management"></a>Об управлении ключами шифрования

При использовании пользовательской речи и пользовательского голоса служба speech может хранить следующие данные в облаке:  

* Данные следа речи - только если ваш поворот след дальше для вашей custom endpoint
* Загруженные данные об обучении и тестировании

По умолчанию ваши данные хранятся в хранилище Майкрософт, а в подписке используются ключи шифрования, управляемые корпорацией Майкрософт. У вас также есть возможность подготовить свой собственный счет хранения. Доступ к магазину управляется управляемой идентификацией, и служба speech не может напрямую получить доступ к вашим собственным данным, таким как данные речевого следа, данные обучения настройке и пользовательские модели.

Для получения дополнительной информации [What are managed identities](https://docs.microsoft.com/azure/active-directory/managed-identities-azure-resources/overview)об управляемой идентификации см.

## <a name="bring-your-own-storage-byos-for-customization-and-logging"></a>Принесите свой собственный хранения (BYOS) для настройки и регистрации

Чтобы запросить доступ, чтобы принести свой собственный хранения, заполнить и отправить [службу речи - принести свой собственный хранения (BYOS) форма запроса](https://aka.ms/cogsvc-cmk). После утверждения необходимо создать собственную учетную запись хранения данных, необходимых для настройки и ведения журнала. При добавлении учетной записи хранилища ресурс службы speech позволит системе присвоить управляемое удостоверение личности. После включения системы, назначенной управляемой идентификацией, этот ресурс будет зарегистрирован в Active Directory Azure (AAD). После регистрации управляемому удостоверению будет предоставлен доступ к учетной записи хранилища. Вы можете узнать больше о управляемых идентификационных данных здесь. Для получения дополнительной информации [What are managed identities](https://docs.microsoft.com/azure/active-directory/managed-identities-azure-resources/overview)об управляемой идентификации см.

> [!IMPORTANT]
> Если вы отстраните систему назначенных управляемых идентификаторов, доступ к учетной записи хранилища будет удален. Это приведет к тому, что части службы speech, требующие доступа к учетной записи хранилища, прекратят работу.  

## <a name="regional-availability"></a>Доступность по регионам

BYOS в настоящее время доступна в этих регионах:

* Центрально-южная часть США
* западная часть США 2
* Восточная часть США

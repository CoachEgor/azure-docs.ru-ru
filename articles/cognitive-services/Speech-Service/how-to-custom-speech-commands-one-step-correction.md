# <a name="add-a-one-step-correction-to-a-custom-command-in-a-custom-commands-preview-application"></a>Добавление одноэтапного исправления в пользовательскую команду в предварительной версии приложения

В этой статье вы узнаете, как добавить одношаговое подтверждение в команду в приложении предварительной версии пользовательских команд.

Одношаговое исправление используется для обновления только что выполненной команды. Когда вы добавляете одноэтапное исправление в команду будильника, вы можете изменить свое внимание и обновить время оповещения. Пример:

- Входные данные: задать оповещение для завтрашнего дня в полдень
- Выходные данные: ОК, оповещение установлено для 2020-05-02 12:00:00
- Ввод: нет, завтра по адресу 13:00-14:30
- Выходные данные: ОК

> [!NOTE]
> В реальном сценарии клиент выполняет действие в результате выполнения команды. В этой статье предполагается, что у вас есть механизм обновления оповещений в серверной части приложения.

## <a name="add-interaction-rules-for-one-step-correction"></a>Добавление правил взаимодействия для одношаговой корректировки

Чтобы продемонстрировать одношаговое исправление, расширьте команду **сеталарм** , созданную в статье [как добавить подтверждение в команду в предварительной версии пользовательских команд](./how-to-custom-speech-commands-confirmations.md).

1. Добавьте правило взаимодействия для обновления команды **сеталарм** .

    Это правило запрашивает у пользователя подтверждение даты и времени оповещения и ожидает подтверждения (да/нет) для следующей активации.

   | Параметр               | Рекомендуемое значение                                                  | Описание                                        |
   | --------------------- | ---------------------------------------------------------------- | -------------------------------------------------- |
   | **Имя правила**             | **Обновить предыдущий будильник**                                            | Имя, описывающее назначение правила.          |
   | **Условия**            | **Предыдущая команда должна быть обновлена & обязательный параметр-> DateTime**                | Условия, определяющие, когда может выполняться правило    |   
   | **Действия**               | **Отправка речевого ответа — > простой редактор > обновление предыдущего оповещения до {DateTime}**      | Действие, выполняемое, если условие правила имеет значение true |
   | **Состояние после выполнения** | **Команда выполнена**        | Состояние пользователя после включения                   |

1. На панели выберите только что созданное правило взаимодействия. Нажмите кнопку с многоточием (**...**) в левом верхнем углу панели. Затем используйте стрелку **вверх** , чтобы переместить правило в верхнюю часть списка **правил взаимодействия** .
   > [!div class="mx-imgBorder"]
   > ![Добавление проверки диапазона](media/custom-speech-commands/one-step-correction-rules.png)

    > [!NOTE]
    > В реальном приложении используйте раздел **действия** , чтобы отправить обратно клиенту действие или вызвать конечную точку HTTP для обновления сигнала в системе.

## <a name="try-it-out"></a>Попробуйте!

1. Щелкните **Обучить**.

1. По завершении обучения выберите **тест**и попробуйте выполнить следующие действия:

   - Входные данные: задать оповещение для завтрашнего дня в полдень
   - Выходные данные: вы действительно хотите установить оповещение для 2020-05-02 12:00:00
   - Входные данные: Да
   - Выходные данные: ОК, оповещение установлено для 2020-05-02 12:00:00
   - Ввод: нет, завтра по адресу 2pm
   - Выходные данные: обновление предыдущего оповещения до 2020-05-02 14:00:00

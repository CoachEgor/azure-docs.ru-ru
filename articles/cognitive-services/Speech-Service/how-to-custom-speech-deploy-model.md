# <a name="deploy-a-custom-model"></a>Развертывание пользовательской модели

После загрузки и проверки данных, оценки точности и обучения пользовательской модели можно развернуть пользовательскую конечную точку для использования в приложениях, инструментах и продуктах. В этом документе вы узнаете, как создать и развернуть конечную точку с помощью [портала custom Speech.](https://speech.microsoft.com/customspeech)

## <a name="create-a-custom-endpoint"></a>Создание пользовательской конечной точки

Чтобы создать новую конечную точку, вопийте на [портале Custom Speech](https://speech.microsoft.com/customspeech) и выберите **развертывание** из меню Custom Speech в верхней части страницы. Если это ваш первый запуск, вы заметите, что в таблице нет конечных точек. После создания конечной точки вы будете использовать эту страницу для отслеживания каждой развернутой конечной точки.

Затем выберите **добавить конечную точку** и ввести **имя** и **описание** для вашей пользовательской конечной точки. Затем выберите пользовательскую модель, которую вы хотели бы связать с этой конечной точкой. С этой страницы можно также включить журнал. Регистрация позволяет отслеживать трафик конечных точек. Если трафик отключен, трафик не будет храниться.

![Как развернуть модель](./media/custom-speech/custom-speech-deploy-model.png)

> [!NOTE]
> Не забудьте принять условия использования и детали ценообразования.

Далее выберите **Создать**. Это действие возвращает вас на страницу **развертывания.** Таблица теперь включает запись, которая соответствует вашей пользовательской конечной точке. Состояние конечной точки показывает ее текущее состояние. Это может занять до 30 минут, чтобы мгновенно создать новую конечную точку с помощью пользовательских моделей. При изменении состояния развертывания в **завершение**конечная точка готова к использованию.

После развертывания конечной точки имя конечных точек отображается в виде ссылки. Нажмите на ссылку, чтобы отобразить информацию, специфичную для вашей конечной точки, такую как ключ конечных точек, URL-адрес конечных точек и код образца.

## <a name="view-logging-data"></a>Просмотр данных журнала

Данные о регистрации доступны для скачивания в **рамках Endpoint > Подробности**.

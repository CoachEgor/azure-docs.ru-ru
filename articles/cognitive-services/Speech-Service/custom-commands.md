# <a name="custom-commands-preview"></a>Пользовательские команды (предварительная версия)

[Голосовые помощники](voice-assistants.md) прослушивают пользователей и принимают меры в ответе, которые часто говорят обратно. Они используют [речь в тексте](speech-to-text.md) для транскрипция речи пользователя, а затем принимают меры по естественному пониманию текста. Это действие часто включает в себя речевой вывод от помощника, созданного с помощью преобразования [текста в речь](text-to-speech.md). Устройства подключаются к помощникам с помощью объекта `DialogServiceConnector` SDK для распознавания речи.

**Пользовательские команды (Предварительная версия)** — это упрощенное решение для создания речевого помощника. Он предоставляет единый интерфейс разработки, модель автоматического размещения и относительно более низкую сложность по сравнению с другими вариантами создания помощника, такими как [прямое речевое распознавание](direct-line-speech.md). Однако это упрощение заключается в снижении гибкости. Таким образом, пользовательские команды (Предварительная версия) лучше всего подходят для выполнения задач или сценариев команд и элементов управления. Это особенно хорошо соответствует для "Интернет вещей" (IoT) и устройств без монитора.

Для сложных диалоговых действий и интеграции с другими решениями, такими как [решение виртуального помощника и шаблон предприятия](https://docs.microsoft.com/azure/bot-service/bot-builder-enterprise-template-overview) , рекомендуется использовать прямую голосовую речь.

Хорошие кандидаты для пользовательских команд (Предварительная версия) имеют фиксированный словарь с четко определенными наборами переменных. Например, главные задачи автоматизации, например Управление термостата, являются идеальными.

   ![Примеры сценариев завершения задач](media/voice-assistants/task-completion-examples.png "Примеры завершения задач")

## <a name="getting-started-with-custom-commands-preview"></a>Приступая к работе с пользовательскими командами (Предварительная версия)

Первый шаг для использования настраиваемых команд (Предварительная версия) для создания голосового помощника — [Получение ключа подписки на речь](get-started.md) и доступ к построителю настраиваемых команд (Предварительная версия) в среде [Speech Studio](https://speech.microsoft.com). После этого можно создать новое приложение настраиваемых команд (Предварительная версия) и опубликовать его, после чего приложение на устройстве может взаимодействовать с ним с помощью речевого пакета SDK.

## <a name="customization"></a>Настройка

Речевые помощники, созданные с помощью службы распознавания речи, могут использовать полный спектр вариантов настройки, доступных для преобразования [речи в текст](speech-to-text.md), преобразования [текста в речь](text-to-speech.md)и [выбора настраиваемых ключевых слов](speech-devices-sdk-create-kws.md).

> [!NOTE]
> Параметры настройки зависят от языка и языкового стандарта (см. раздел [Поддерживаемые языки](supported-languages.md)).

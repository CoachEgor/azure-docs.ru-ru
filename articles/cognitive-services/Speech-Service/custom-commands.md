# <a name="what-are-custom-commands-preview"></a>Что такое пользовательские команды (Предварительная версия)?

Голосовые приложения, такие как [голосовые помощники](voice-assistants.md) , прослушивают пользователей и принимают меры в ответе, часто говорят. Они используют [речь в тексте](speech-to-text.md) для транскрипция речи пользователя, а затем принимают меры по естественному пониманию текста. Это действие часто включает в себя речевой вывод от помощника, созданного с помощью преобразования [текста в речь](text-to-speech.md). Устройства подключаются к помощникам с помощью объекта `DialogServiceConnector` SDK для распознавания речи.

**Пользовательские команды (Предварительная версия)** — это упрощенное решение для создания речевых приложений. Она предоставляет единый интерфейс разработки, модель автоматического размещения и относительно более низкую сложность по сравнению с другими параметрами, такими как [прямое речевое распознавание](direct-line-speech.md). Однако это упрощение заключается в снижении гибкости. Таким образом, пользовательские команды (Предварительная версия) лучше всего подходят для выполнения задач или сценариев команд и элементов управления. Это особенно хорошо соответствует для "Интернет вещей" (IoT) и устройств без монитора.

Для сложных диалоговых действий и интеграции с другими решениями, такими как [решение виртуального помощника и шаблон предприятия](https://docs.microsoft.com/azure/bot-service/bot-builder-enterprise-template-overview) , рекомендуется использовать прямую голосовую речь.

Хорошие кандидаты для пользовательских команд (Предварительная версия) имеют фиксированный словарь с четко определенными наборами переменных. Например, главные задачи автоматизации, например Управление термостата, являются идеальными.

   ![Примеры сценариев завершения задач](media/voice-assistants/task-completion-examples.png "Примеры завершения задач")

## <a name="getting-started-with-custom-commands-preview"></a>Приступая к работе с пользовательскими командами (Предварительная версия)

Первым шагом при использовании настраиваемых команд (Предварительная версия) для создания голосового приложения — [Получение ключа подписки на речь](get-started.md) и доступ к построителю настраиваемых команд (Предварительная версия) в среде [Speech Studio](https://speech.microsoft.com). После этого можно создать новое приложение настраиваемых команд (Предварительная версия) и опубликовать его, после чего приложение на устройстве может взаимодействовать с ним с помощью речевого пакета SDK.

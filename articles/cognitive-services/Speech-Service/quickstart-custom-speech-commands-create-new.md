# <a name="quickstart-create-a-custom-command-preview"></a>Краткое руководство. Создание настраиваемой команды (Предварительная версия)

В этой статье вы узнаете, как создать и протестировать размещенное приложение пользовательских команд.
Приложение распознает utterance, например "включить телевизор" и ответит простым сообщением "ОК, включение телевизора".

## <a name="prerequisites"></a>Технические условия

- Подписка на речь. [Попробуйте службу распознавания речи бесплатно](~/articles/cognitive-services/speech-service/get-started.md).

  > [!NOTE]
  > На этапе предварительной версии для ключей подписки поддерживается только регион westus2.

## <a name="go-to-the-speech-studio-for-custom-commands"></a>Переход в речевое меню для пользовательских команд

1. Откройте веб-браузер и перейдите к [речевой студии](https://speech.microsoft.com/) .
1. Введите учетные данные для входа на портал

   - Представлением по умолчанию является список подписок на речь
     > [!NOTE]
     > Если страница Выбор подписки не отображается, можно перейти на нее, выбрав "ресурсы речи" в меню параметров на верхней панели.

1. Выберите свою подписку на речь, а затем щелкните **"переход к студии"** .
1. Выбор **настраиваемых команд (Предварительная версия)**

Представление по умолчанию — это список созданных приложений пользовательских команд.

## <a name="create-a-custom-commands-project"></a>Создание проекта настраиваемых команд

1. Выберите " **создать проект** ", чтобы создать новый проект

   > [!div class="mx-imgBorder"]
   > ![Создание проекта](media/custom-speech-commands/create-new-project.png)

1. Введите имя и язык проекта.
1. Выберите ресурс для разработки. Если у вас нет допустимых ресурсов для разработки, создайте его, выбрав **создать новый ресурс**.

   > [!div class="mx-imgBorder"]
   > ![Создание ресурса](media/custom-speech-commands/create-new-resource.png)

   1. Введите имя ресурса, группу, расположение и ценовую категорию.

         > [!NOTE]
         > Чтобы создать группы ресурсов, введите нужное имя группы ресурсов в поле "Группа ресурсов". Группа ресурсов будет создана при выборе **создать** .

1. Нажмите кнопку **создать** , чтобы создать проект.
1. После создания выберите проект.

Теперь представление должно быть обзором приложения пользовательских команд.

## <a name="create-a-new-command"></a>Создание новой команды

Теперь можно создать команду. Давайте рассмотрим пример, который будет принимать одно utterance, `turn on the tv`и реагировать на `Ok, turning on the TV`сообщения.

1. Создайте новую команду, щелкнув значок `+` рядом с командами и присвойте ему имя `TurnOn`
1. Нажмите кнопку **Сохранить**.

> [!div class="mx-imgBorder"]
> ![создать команду](media/custom-speech-commands/create-add-command.png)

Команда является набором:

| Группа            | Описание                                                                                                                 |
| ---------------- | --------------------------------------------------------------------------------------------------------------------------- |
| Примеры предложений | Пример фразы продолжительностью, который пользователь может сказать, чтобы активировать эту команду                                                                 |
| Параметры       | Сведения, необходимые для выполнения команды                                                                                |
| Правила завершения | Действия, выполняемые для выполнения команды. Например, для реагирования на пользователя или взаимодействия с другой веб-службой |
| Расширенные правила   | Дополнительные правила для работы с более конкретными или сложными ситуациями                                                              |

### <a name="add-a-sample-sentence"></a>Добавление образца предложения

Начнем с примеров предложений и предоставьте пример того, что может сказать пользователь:

```
turn on the tv
```

Сейчас у нас нет параметров, поэтому мы можем перейти к правилам завершения.

### <a name="add-a-completion-rule"></a>Добавление правила завершения

Теперь добавьте правило завершения для реагирования на пользователя, указывая, что выполняется действие.

1. Создайте новое правило завершения, щелкнув значок `+` рядом с правилами завершения.
1. Введите имя правила
1. Добавление действия
   1. Создайте новое действие речевого ответа, щелкнув значок `+` рядом с пунктом действия и выбрав пункт `SpeechResponse`
   1. Введите ответ

   > [!NOTE]
   > Обычный текст должен начинаться с тире. Дополнительные сведения см. [здесь](https://aka.ms/sc-lg-format) .

   > [!div class="mx-imgBorder"]
   > ![создать речевой ответ](media/custom-speech-commands/create-speech-response-action.png)

1. Нажмите кнопку **сохранить** , чтобы сохранить правило.

> [!div class="mx-imgBorder"]
> ![создать правило завершения](media/custom-speech-commands/create-basic-completion-response-rule.png)


| Параметр    | Рекомендуемое значение                        | Описание                                        |
| ---------- | -------------------------------------- | -------------------------------------------------- |
| Имя правила  | "Конфирматионреспонсе"                 | Имя, описывающее назначение правила.          |
| Условия | Нет                                   | Условия, определяющие, когда может выполняться правило    |
| Действия    | Спичреспонсе "-ОК, включение телевизора" | Действие, выполняемое, если условие правила имеет значение true |

# <a name="what-is-the-conversation-transcription-service"></a>Что такое службы расшифровка дикторского текста для обмена данными?

Расшифровка дикторского текста для службы обмена данными является дополнительным служб речи, сочетающий в себе распознавание речи в режиме реального времени, идентификация говорящего и diarization. Расшифровка дикторского текста для службы обмена данными идеально подходит для фотографировать личных встреч, возможность различать докладчиков, он позволяет узнать, кто сказал, что и когда, позволяя участникам сосредоточиться на собрание и быстро дальнейших действий дальнейшие действия. Эта функция также повышает доступность. Расшифровка дикторского текста вы можете активно обратиться участников с нарушениями слуха.   

Расшифровка дикторского текста для службы обмена данными обеспечивает точность распознавания с настраиваемые модели речи, которые можно адаптировать для понимания отрасли и словаря конкретной компании. Кроме того вы можете связать службы расшифровка дикторского текста для обмена данными с помощью пакета SDK устройства речи, чтобы оптимизировать работу нескольких микрофон устройства.

>[!NOTE]
> В настоящее время службы расшифровка дикторского текста для диалога рекомендуется для небольших собраний. Если вы хотите расширить службы расшифровка дикторского текста для обмена данными для больших собраний в масштабе, свяжитесь с нами.

Схема иллюстрирует оборудования, программного обеспечения и служб, которые работают совместно со службой расшифровка дикторского текста для диалога.

## <a name="get-started-with-conversation-transcription"></a>Начало работы с расшифровка дикторского текста для диалога

Из трех этапов, которые необходимо предпринять для начала работы со службой расшифровка дикторского текста для диалога.

1. Собирать образцы голосовой от пользователей.
2. Создание профилей пользователей, с помощью примеров голос пользователя
3. Используйте пакет SDK для распознавания речи для определения пользователей (динамики) и транскрипция речи

## <a name="collect-user-voice-samples"></a>Собирать образцы голос пользователя

Первый шаг — это собрать аудиозаписи с каждого пользователя. Речи пользователя должны быть записаны в тихом месте без фонового шума. Рекомендуемая длина для каждой аудиовыборки — от 30 секунд до двух минут. Более длинные примеры аудио приведет к повышенную точность при определении динамики. Звук должен быть одним каналом с 16 кГц частота выборки.

Помимо указанных выше рекомендаций, как аудио записываются и хранятся возлагается--рекомендуется защищенную базу данных. В следующем разделе мы рассмотрим, как этот звук используется для создания профилей пользователей, которые используются с пакетом SDK для распознавания речи для распознавания динамики.

## <a name="transcribe-and-identify-speakers"></a>Транскрипция и определение говорящих

Расшифровка дикторского текста для службы обмена данными ожидает в качестве входных данных для создания транскрипции и определение говорящих многоканальный аудиопотоков и профили пользователей. Аудио и данных профиля пользователя отправляются службы расшифровка дикторского текста для обмена данными с помощью пакета SDK устройства речи. Как упоминалось ранее для использования службы расшифровка дикторского текста для обмена данными требуются циклического массива семи микрофона и Speech SDK устройства.

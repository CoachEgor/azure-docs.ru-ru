# <a name="what-is-conversation-transcription"></a>Что такое запись разговора?

Запись разговора — это расширенная функция речевых служб, которая сочетает в себе распознавание речи в реальном времени, идентификацию докладчика и диаризатион. Транскрипция беседы идеально подходит для фотографироватьных совещаний с возможностью различения динамиков, она позволяет узнать, кто сказал, что и когда, позволяя участникам сосредоточиться на собрании и быстро перейти к дальнейшим действиям. Эта функция также улучшает доступность. С помощью транскрипции можно активно привлекать участников с нарушениями слуха.   

Запись разговора обеспечивает точное распознавание с помощью настраиваемых моделей речи, которые можно адаптировать для понимания отрасли и словаря, относящегося к конкретной компании. Кроме того, можно связать транскрипцию беседы с пакетом SDK для речевых устройств, чтобы оптимизировать интерфейс для устройств с несколькими микрофонами.

>[!NOTE]
> В настоящее время запись разговора рекомендуется для небольших совещаний. Если вы хотите продлить транскрипцию беседы для больших собраний в масштабе, свяжитесь с нами.

## <a name="get-started-with-conversation-transcription"></a>Приступая к работе с записью разговора

Чтобы начать работу с записью беседы, необходимо выполнить три шага.

1. Сбор речевых примеров от пользователей.
2. Создание профилей пользователей с помощью пользовательских примеров голоса
3. Использование речевого пакета SDK для распознавания пользователей (докладчиков) и транскрипция речи

## <a name="collect-user-voice-samples"></a>Сбор пользовательских примеров голоса

Первым шагом является получение звуковых записей от каждого пользователя. Речевой ввод пользователя следует записывать в тихом окружении без фонового шума. Рекомендуемая длина для каждого звукового образца составляет от 30 секунд до двух минут. Более длинные образцы звука приведут к повышению точности при идентификации докладчиков. Звук должен быть в виде канала Mono с частотой дискретизации 16 кГц.

Помимо упомянутых выше рекомендаций, способ записи и сохранения звука — это безопасная база данных. В следующем разделе мы рассмотрим, как этот звук используется для создания профилей пользователей, которые используются с речевым пакетом SDK для распознавания докладчиков.

## <a name="generate-user-profiles"></a>Создание профилей пользователей

Затем необходимо отправить звуковые записи, собранные в службу формирования подписей, чтобы проверить аудио и создать профили пользователей. [Служба формирования подписей](https://aka.ms/cts/signaturegenservice) — это набор интерфейсов API, позволяющих создавать и извлекать профили пользователей.

Чтобы создать профиль пользователя, необходимо использовать `GenerateVoiceSignature` API. Сведения о спецификации и пример кода доступны:

## <a name="transcribe-and-identify-speakers"></a>Транскрипция и выявление докладчиков

В процессе записи беседы в качестве входных данных для создания транскрипций и распознавания докладчиков требуются многоканальные звуковые потоки и профили пользователей. Данные профиля звука и пользователя отправляются в службу записи сообщений с помощью пакета SDK для речевых устройств. Как упоминалось ранее, для использования записи беседы требуется круглый семь массивов микрофонов и SDK речевых устройств.

>[!NOTE]
> Сведения о спецификации и проектировании см. в [статье микрофон речевых устройств Microsoft](https://aka.ms/cts/microphone). Дополнительные сведения о приобретении комплекта разработки см. в статье [Получение пакета SDK для устройств Microsoft Speech](https://aka.ms/cts/getsdk).

Сведения об использовании транскрипции беседы с пакетом SDK для речевых устройств см. в разделе [Использование записи бесед](https://aka.ms/cts/howto).

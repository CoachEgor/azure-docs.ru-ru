# <a name="what-is-conversation-transcription"></a>Что такое расшифровка дикторского текста для диалога

Расшифровка дикторского текста для диалога является дополнительным служб речи, сочетающий в себе распознавание речи в режиме реального времени, идентификация говорящего и diarization. Расшифровка дикторского текста для диалога идеально подходит для фотографировать личных встреч, возможность различать докладчиков, он позволяет узнать, кто сказал, что и когда, позволяя участникам сосредоточиться на собрание и быстро дальнейших действий дальнейшие действия. Эта функция также повышает доступность. Расшифровка дикторского текста вы можете активно обратиться участников с нарушениями слуха.   

Расшифровка дикторского текста для диалога обеспечивает точность распознавания с настраиваемые модели речи, которые можно адаптировать для понимания отрасли и словаря конкретной компании. Кроме того вы можете связать расшифровка дикторского текста сообщений с помощью пакета SDK устройства речи, чтобы оптимизировать работу нескольких микрофон устройства.

>[!NOTE]
> В настоящее время для небольших собраний рекомендуется расшифровка дикторского текста для диалога. Если вы хотите расширить транскрипции диалога для больших собраний в масштабе, свяжитесь с нами.

Схема иллюстрирует оборудования, программного обеспечения и служб, работающих вместе с расшифровка дикторского текста для диалога.

## <a name="get-started-with-conversation-transcription"></a>Начало работы с расшифровка дикторского текста для диалога

Из трех этапов, которые необходимо выполнить, чтобы приступить к работе с расшифровка дикторского текста для диалога.

1. Собирать образцы голосовой от пользователей.
2. Создание профилей пользователей, с помощью примеров голос пользователя
3. Используйте пакет SDK для распознавания речи для определения пользователей (динамики) и транскрипция речи

## <a name="collect-user-voice-samples"></a>Собирать образцы голос пользователя

Первый шаг — это собрать аудиозаписи с каждого пользователя. Речи пользователя должны быть записаны в тихом месте без фонового шума. Рекомендуемая длина для каждой аудиовыборки — от 30 секунд до двух минут. Более длинные примеры аудио приведет к повышенную точность при определении динамики. Звук должен быть одним каналом с 16 кГц частота выборки.

Помимо указанных выше рекомендаций, как аудио записываются и хранятся возлагается--рекомендуется защищенную базу данных. В следующем разделе мы рассмотрим, как этот звук используется для создания профилей пользователей, которые используются с пакетом SDK для распознавания речи для распознавания динамики.

## <a name="generate-user-profiles"></a>Создание профилей пользователей

## <a name="transcribe-and-identify-speakers"></a>Транскрипция и определение говорящих

Расшифровка дикторского текста для диалога ожидает в качестве входных данных для создания транскрипции и определение говорящих многоканальный аудиопотоков и профили пользователей. Аудио и данных профиля пользователя отправляются службе расшифровка дикторского текста для диалога, с помощью пакета SDK для устройств речи. Как упоминалось ранее, циклические семь микрофон и Speech SDK устройства должен использовать расшифровка дикторского текста для диалога.

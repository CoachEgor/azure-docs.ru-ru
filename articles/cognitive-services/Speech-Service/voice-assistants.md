# <a name="what-is-a-voice-assistant"></a>Что такое голосовой помощник?

Голосовые помощники, использующие службу Speech, позволяют разработчикам создавать естественные, похожие на человека разговорные интерфейсы для своих приложений и опыта.

Служба голосового помощника обеспечивает быстрое и надежное взаимодействие между устройством и реализацией помощника, которая использует либо (1) канал прямой линии Bot Framework или (2) интегрированную службу пользовательских команд (Preview) для завершения задачи.

Приложения подключаются к службе голосового помощника с набором разработки программного обеспечения для речи (SDK).

   ![Концептуальная диаграмма потока службы оркестровки голосового помощника](media/voice-assistants/overview.png "Поток голосового помощника")

## <a name="choosing-an-assistant-solution"></a>Выбор помощника решения

Первый шаг к созданию голосового помощника заключается в том, чтобы решить, что он должен делать. Служба речевого сообщения предоставляет несколько дополнительных решений для создания взаимодействия с помощником. Нужна ли вам гибкость и универсальность канала [Прямой линии](direct-line-speech.md) Bot Framework или простота [пользовательских команд (Preview)](custom-commands.md) для простых сценариев, выбор правильных инструментов поможет вам начать работу.

| Если вы хотите... | Тогда рассмотрим... | Например... |
|-------------------|------------------|----------------|
|Открытый разговор с надежной интеграцией навыков и полным управлением развертыванием | Канал [прямой линии](direct-line-speech.md) Bot Framework | <ul><li>"Мне нужно поехать в Сиэтл"</li><li>"Какую пиццу я могу заказать?"</li></ul>
|Командно-контрольный или ориентированный на задачи разговор с упрощенным автором и хостингом | [Пользовательские команды (Предварительный просмотр)](custom-commands.md) | <ul><li>"Поверните на накладной свет"</li><li>"Сделайте его на 5 градусов теплее"</ul>

Мы рекомендуем [Direct Line Speech](direct-line-speech.md) как лучший выбор по умолчанию, если вы еще не уверены, что вы хотите, чтобы ваш помощник для обработки. Он предлагает интеграцию с богатым набором инструментов и авторских средств, таких как [виртуальный помощник решения и шаблон аферы](https://docs.microsoft.com/azure/bot-service/bot-builder-enterprise-template-overview) и [сервис SnA Maker,](https://docs.microsoft.com/azure/cognitive-services/QnAMaker/Overview/overview) чтобы опираться на общие шаблоны и использовать существующие источники знаний.

[Пользовательские команды (Preview)](custom-commands.md) обеспечивают обтекаемый опыт авторизации и хостинга, специально разработанный для сценариев управления естественным языком.

   ![Сравнение помощников решений](media/voice-assistants/assistant-solution-comparison.png "Сравнение помощников решений")

## <a name="core-features"></a>Основные функции

Независимо от того, выбираете ли вы [Direct Line Speech](direct-line-speech.md) или Custom [Commands (Preview)](custom-commands.md) для создания взаимодействий помощника, вы можете использовать богатый набор функций настройки для настройки вашего помощника для вашего бренда, продукта и личности.

| Категория | Компоненты |
|----------|----------|
|[Custom - ключевое слово](speech-devices-sdk-create-kws.md) | Пользователи могут начать разговоры с помощниками с пользовательским ключевым словом, как "Эй Contoso". Приложение делает это с пользовательским движком ключевых слов в речи SDK, который может быть настроен с пользовательским ключевым [словом, которое вы можете создать здесь](speech-devices-sdk-create-kws.md). Голосовые помощники могут использовать проверку ключевых слов со стороны службы для повышения точности активации ключевых слов (по сравнению с только устройством).
|[Речь к тексту](speech-to-text.md) | Голосовые помощники преобразуют звук в реальном времени в распознаваемый текст с помощью [речевого текста](speech-to-text.md) из службы Speech. Этот текст доступен, как он транскрибируется, как для вашего помощника реализации и вашего приложения клиента.
|[Преобразование текста в речь](text-to-speech.md) | Текстовые ответы от вашего помощника синтезируются с помощью [text-to-speech](text-to-speech.md) от службы Speech. Этот синтез затем становится доступным для вашего клиентского приложения в качестве аудио потока. Microsoft предлагает возможность создавать свой собственный пользовательский, высококачественный голос Neural TTS, который дает голос вашему бренду. Чтобы узнать больше, [свяжитесь с нами](mailto:mstts@microsoft.com).

Монитор метрик — это служба для обнаружения, диагностики и анализа аномалий временных рядов. Это служба на основе ИИ, а значит она использует предоставленные данные для обучения используемой модели. Эта служба принимает таблицы агрегированных данных со следующими столбцами:

* **Measure** (Мера), обязательный: один или несколько столбцов, содержащих числовые значения.
* **Timestamp** (Метка времени), необязательный: не более одного столбца с типом `DateTime` или `String`. Если этот столбец не задан, меткой времени считается время начала каждого периода приема. Метка времени должна иметь такой формат: `yyyy-MM-ddTHH:mm:ssZ`. 
  * **Метка времени должна соответствовать степени детализации метрики. Например, для ежедневно собираемой метрики часы, минуты и секунды в метке времени должны быть обозначены как `00:00:00`** .
* **Dimension** (Измерение), необязательный: столбцы с любым типом данных. Соблюдайте осторожность при работе с большим количеством столбцов и значений, чтобы не выполнять обработку слишком большого числа измерений.

> [!Note]
> Для каждой метрики должна быть только одна метка времени, соответствующая одному сочетанию измерений. Выполните агрегирование данных до подключения или используйте запрос, чтобы указать данные для приема.

# <a name="project-acoustics-unity-bake-tutorial"></a>Руководство по моделированию Project Acoustics в Unity
В этом руководстве описано, как выполнять акустическое моделирование с использованием Project Acoustics в Unity.

Требования к программному обеспечению:
* [Unity 2018.2+](https://unity3d.com) для Windows или MacOS
* [Подключаемый модуль Project Acoustics, интегрированный в проект Unity](unity-integration.md), или [содержимое примера Project Acoustics Unity](unity-quickstart.md).
* *Необязательно*. [Учетная запись пакетной службы Azure](create-azure-account.md) для ускорения моделирования с использованием облачных вычислений.

## <a name="open-the-project-acoustics-bake-window"></a>Открытие окна моделирования Project Acoustics
В меню Unity выберите **Acoustics** (Акустика) в меню **Window** (Окно).

![Редактор Unity с выделенным параметром Acoustics (Акустика) в меню Window (Окно)](media/window-acoustics.png)

## <a name="create-a-navigation-mesh"></a>Создание навигационной сетки
Project Acoustics использует навигационную сетку, чтобы расположить точки зонда прослушивателя для моделирования. Вы можете использовать [рабочий процесс навигационной сетки](https://docs.unity3d.com/Manual/nav-BuildingNavMesh.html) Unity. Также вы можете создать собственную сетку с использованием любого другого пакета трехмерного моделирования.

## <a name="mark-acoustic-scene-objects"></a>Метка объектов акустической сцены
Для моделирования Project Acoustics использует два типа объектов сцены:
- объекты, которые отражают и перекрывают звук в модели;
- сетка навигации проигрывателя, которая ограничивает точки размещения слушателя в модели.

Оба типа объектов размещаются с помощью вкладки **Objects** (Объекты).

Так как маркировка объектов просто добавляет компоненты *AcousticsGeometry* или *AcousticsNavigation* к объекту, вы также можете использовать [стандартный рабочий процесс компонента Unity](https://docs.unity3d.com/Manual/UsingComponents.html), чтобы отметить или снять метки с объектов. Вы можете ставить метки только для отрисовщиков сетки и ландшафтов. Объекты всех остальных типов игнорируются. Флажки позволяют добавлять или снимать метки для всех допустимых объектов.

### <a name="mark-acoustic-occlusion-and-reflection-geometry"></a>Метка геометрии отражения и акустической окклюзии
Откройте вкладку **Objects** (Объекты) в окне **Acoustics**. Обязательно пометьте объекты как *Acoustics Geometry* (Геометрия акустики), если они должны перекрывать, отражать или поглощать звук. Геометрия акустики может включать такие объекты, как пол, стена, крыша, окна и стекла, ковры и крупная мебель. Для этих объектов вы можете использовать произвольный уровень сложности. Так как сцена перед моделированием вокселизируется, моделировать сетки с высокой степенью детализации, например, деревья с множеством листьев, не более затратно, чем упрощенные объекты.

Следите за тем, чтобы не отметить очевидно лишние объекты, например невидимые сетки конфликтов.

В результатах наложения фиксируется форма объекта, существовавшая на момент вычисления зонда (выполняется на вкладке **Probes** (Зонды)). Если после этого вы переместите в сцене любой из объектов с метками, расчет зонда и наложение придется выполнить заново.

### <a name="mark-the-navigation-mesh"></a>Метка навигационной сетки
Навигационные сетки, которые были созданы в рабочем процессе Unity, автоматически распознаются системой акустики. Чтобы применить собственные сетки, отметьте их на вкладке **Objects** (Объекты).

### <a name="for-reference-the-objects-tab-parts"></a>Справочная информация. Элементы вкладки Objects ("Объекты")
Страница вкладок (размещается после описаний) содержит следующие элементы:

1. Кнопки выбора вкладок (по умолчанию выбрана вкладка **Objects** (Объекты)). Эти кнопки позволяют переключаться между разными этапами наложения акустики (слева направо).
1. Краткое описание действий, выполняемых на этой вкладке.
1. Доступные фильтры для окна иерархии. Эти элементы позволяют отфильтровать список в окне иерархии по определенному типу объектов, чтобы их легче было пометить. Если вы еще ничего не отметили для акустики, при выборе двух последних элементов не отобразится ничего. Но вы можете искать уже отмеченные объекты.
1. Если выбранных объектов нет, в этом разделе отображается состояние всех объектов в сцене.
    * Всего: Общее число активных нескрытых объектов.
    * Ignored (Игнорируется). Число объектов, которые не являются отрисовщиками сетки или ландшафтами.
    * Mesh (Сетка). Число объектов отрисовщика сетки.
    * Terrain (Ландшафт). Число объектов ландшафта.
    * Geometry (Геометрия). Число объектов сетки и ландшафта, помеченных как **Acoustics Geometry** (Геометрия акустики).
    * Navigation (Навигация). Число объектов сетки и ландшафта, помеченных как **Acoustics Navigation** (Навигация акустики). Это число не включает элементы Unity NavMesh.
1. Общее число объектов в сцене, которые вы можете пометить, то есть отрисовщиков сетки и ландшафтов. С помощью флажков вы можете пометить объекты для геометрии или навигации акустики (то есть добавить соответствующий компонент).
1. Если ничего не выбрано, здесь отображается напоминание о необходимости выбрать помечаемые объекты. Вы также можете пометить все объекты на сцене, установив любой из флажков или оба сразу.
1. После выбора объектов в этом разделе отображается состояние только для выбранных объектов.
1. Общее число выбранных помечаемых объектов. Установка или снятие флажков применима только к выбранным объектам.

Объекты можно помечать одновременно для геометрии и навигации.

## <a name="select-acoustic-materials"></a>Выбор акустических материалов
Пометив объекты, нажмите кнопку **Materials** (Материалы). После этого назначьте акустические материалы. Система материалов в Project Acoustics связана с системой визуальных материалов Unity. Чтобы два объекта состояли из разных акустических материалов, им должны быть присвоены разные визуальные материалы.

Выбор акустических материалов определяет объем звуковой энергии, которая отражается от каждой поверхности. По умолчанию используется акустический материал с характеристиками поглощения, близкими к стали. Project Acoustics предлагает материалы, основанные на названии визуального материала. Вы можете назначить материалу акустический материал **Custom** (Пользовательский) и выбирать коэффициент поглощения, смещая ползунок.

Время реверберации для материала в помещении обратно пропорционально его коэффициенту поглощения. Большинство материалов имеют коэффициент поглощения в диапазоне от 0,01 до 0,20. Материалы с коэффициентами вне этого диапазона имеют очень высокий уровень поглощения.

![Диаграмма демонстрирует обратную зависимость между временем реверберации и коэффициентом поглощения.](media/reverb-time-graph.png)

### <a name="for-reference-parts-of-the-materials-tab"></a>Справочная информация. Элементы вкладки Materials (Материалы)
![Вкладка Acoustics Materials (Материалы Acoustics) в Unity](media/materials-tab-detail.png)
1. Эта вкладка открывается с помощью кнопки **Materials** (Материалы).
2. Краткое описание действий, выполняемых на этой вкладке.
3. Если установлен этот флажок, отображаются только те материалы, которые используются для объектов с пометкой **Acoustics Geometry** (Геометрия акустики). В противном случае перечисляются все материалы, используемые в сцене.
4. Эти параметры позволяют изменить порядок объектов меню в столбце **Acoustics** (Акустика) (№6). Сортировка акустических материалов по **имени** или **коэффициенту поглощения** в порядке возрастания.
5. Список материалов, используемых в сцене, отсортированный в алфавитном порядке. Если установлен флажок **Show Marked Only** (Показать только отмеченные) (№3), отображаются только те материалы, которые используются объектами с пометкой **Acoustics Geometry** (Геометрия акустики). Выберите материал, чтобы выбрать в сцене все объекты, для которых используется этот материал.
6. Акустический материал, которому назначен материал сцены. Выберите другой акустический материал, чтобы изменить это назначение для материала сцены. Чтобы изменить порядок сортировки этих меню, используйте параметры **Sort Acoustics By** (Сортировать акустику по) (№4).
7. Коэффициент акустического поглощения для материала, выбранного в столбце слева (№6). Значение 0 означает, что материал ничего не поглощает (полное отражение), а значение 1 означает полное поглощение (без отражения). Коэффициент поглощения можно изменить только для пользовательских материалов (**Custom**).
8. Если материал имеет тип **Custom** (Пользовательский), ползунок активируется. Вы можете переместить этот ползунок или ввести точное значение, чтобы изменить коэффициент поглощения.

## <a name="calculate-and-review-listener-probe-locations"></a>Вычисление и проверка расположений зонда прослушивателя
Завершив назначение материалов, переходите на вкладку **Probes** (Зонды). Щелкните **Calculate** (Вычислить), чтобы расположить точки зонда слушателя для моделирования.

### <a name="what-the-calculate-button-does"></a>Что делает кнопка Calculate (Вычислить)
Кнопка **Calculate** (Вычислить) использует выбранную геометрию акустической сцены, чтобы подготовить сцену к моделированию.

- Она извлекает сведения о геометрии из сеток сцены и вычисляет *объемное пространство*. Объемное пространство сцены состоит из маленьких кубических объемных пикселей. Размер объемных пикселей определяется частотой моделирования, которая зависит от значения параметра **Simulation Resolution** (Разрешение моделирования). Каждый объемный пиксель помечается как открытое пространство или элемент геометрии сцены. Если объемный пиксель содержит геометрию, ему присваивается тег со значением коэффициента поглощения, соответствующего материалу этого элемента геометрии.
- На основе навигационных сеток размещаются точки зондов слушателя. Алгоритм ищет компромисс между полнотой охвата пространства, временем моделирования и размером файла. Он старается выбрать такой режим, при котором все узкие коридоры и небольшие участки имеют хотя бы какое-то покрытие. Типичные значения точек зондов варьируются от 100 для небольших сцен до нескольких тысяч для больших.

В зависимости от размера сцены и скорости компьютера эти вычисления могут занять несколько минут.

### <a name="review-voxel-and-probe-placement"></a>Обзор размещения объемных пикселей и зондов
Просмотрите данные об объемных пикселях и точках размещения зондов, чтобы проверить готовность к моделированию. На этапе просмотра обычно хорошо заметны недостающие элементы навигационной сетки, а также лишние или отсутствующие элементы акустической геометрии. Размещение объемных пикселей и зондов можно включить или отключить с помощью меню **Gizmos** (Инструменты).

![Меню Gizmos (Инструменты) в Unity](media/gizmos-menu.png)

Объемные пиксели, содержащие акустическую геометрию, отображаются в виде зеленых кубов. Изучите сцену и убедитесь, что все элементы геометрии содержат объемные пиксели. Объемные пиксели отображаются только в том случае, если расстояние от камеры сцены до объекта не превышает 5 метров.

Сравнивая объемные пиксели, созданные в разрешениях грубого и точного моделирования, вы увидите, что в грубом разрешении создаются пиксели вдвое большего размера.

![Предварительный просмотр объемных пикселей в грубом разрешении в редакторе Unity](media/voxel-cubes-preview.png)

Результаты моделирования интерполируются между точками расположения зонда прослушивателя в среде выполнения. Проверьте, есть ли точки зонда в тех местах, где игрок будет перемещаться в этой сцене.

![Просмотр зондов в редакторе Unity](media/probes-preview.png)

### <a name="take-care-with-scene-renames"></a>Выполнение переименования сцен
Имя сцены используется для сопоставления сцены с файлами, в которых хранятся данные о точках размещения зондов и объемных пикселях. Если вы переименуете сцену после расчета точек расположения зондов, все данные о назначении материалов и размещении объектов будут утрачены и придется повторно выполнить вычисления.

### <a name="for-reference-parts-of-the-probes-tab"></a>Справочная информация. Элементы вкладки Probes (Зонды)
![Вкладка Acoustics Probes (Акустические зонды) в Unity](media/probes-tab-detail.png)

1. Эта вкладка открывается с помощью кнопки **Probes** (Зонды).
2. Краткое описание действий, выполняемых на этой вкладке.
3. Эти параметры позволяют выбрать грубое (Coarse) или точное (Fine) разрешение моделирования. Грубое разрешение реализуется быстрее, но в ущерб качеству. См. сведения о [разрешении моделирования](bake-resolution.md).
4. Указывает место размещения файлов с акустическими данными. Нажмите кнопку **...** , чтобы открыть средство выбора папки. По умолчанию используется папка *Assets/AcousticsData*. В этом расположении будет создана вложенная папка *Editor*. См. раздел [Файлы данных, которые добавляются в процессе моделирования](#Data-Files) далее в этой статье.
5. Указанный здесь префикс используется для именования файлов данных для этой сцены. По умолчанию используется формат Acoustics_ *[имя сцены]* .
6. После вычисления зондов описанные выше элементы управления отключаются. Нажмите кнопку **Clear** (Очистить), чтобы удалить все результаты вычислений и включить элементы управления, если вам нужно повторить расчеты с новыми параметрами.
7. Нажмите кнопку **Calculate** (Вычислить), чтобы выполнить расчет объемных пикселей и точек зондов. Вычисление выполняется на локальном компьютере. Это обязательный процесс перед акустическим моделированием.

В этой версии Project Acoustics зонды нельзя размещать вручную. Используйте автоматический процесс на вкладке **Probes** (Зонды).

См. [сравнение грубого и точного разрешений для моделирования](bake-resolution.md).

## <a name="bake-your-scene-by-using-azure-batch"></a>Моделирование сцены с помощью пакетной службы Azure
Акустическое моделирование для сцены можно выполнить в облачном вычислительном кластере, используя пакетную службу Azure. Подключаемый модуль Unity для Project Acoustics напрямую подключается к пакетной службе Azure чтобы создать экземпляр кластера пакетной службы Azure для каждого моделирования, управлять им и удалить после использования. На вкладке **Bake** (Моделирование) введите учетные данные Azure, выберите тип и размер компьютера для кластера, а затем щелкните **Bake** (Моделировать).

### <a name="for-reference-parts-of-the-bake-tab"></a>Справочная информация. элементы вкладки Bake ("Моделирование")
![Вкладка Acoustics Bake (Акустическое моделирование) в Unity](media/bake-tab-details.png)

1. Эта вкладка открывается с помощью кнопки **Bake** (Моделирование).
2. Краткое описание действий, выполняемых на этой странице.
3. Введите учетные данные Azure в эти поля, когда создадите учетную запись Azure. См. сведения о [создании учетной записи пакетной службы Azure](create-azure-account.md).
4. Поле для тега образа Docker, который предоставляется набору инструментов акустической обработки.
5. Открывает портал Azure, чтобы управлять подписками, отслеживать потребление, просматривать информацию о выставлении счетов и т. д.
6. Определяет тип вычислительного узла в Пакетной службе Azure, который используется для вычислений. Можно использовать только те типы узлов, которые поддерживаются в выбранном расположении центра обработки данных Azure. Если вы не уверены в выборе, сохраните значение по умолчанию **Standard_F8s_v2**.
7. Количество узлов, используемых для вычислений. Это значение влияет на время моделирования. Ваш выбор ограничен количеством выделенных ядер для пакетной службы Azure. Распределение по умолчанию допускает только два узла по 8 ядер или один узел с 16 ядрами. Эти значения можно увеличить. См. сведения об ограничениях для выделения ядер в руководстве по [созданию учетной записи Пакетной службы Azure](create-azure-account.md).
8. Установите этот флажок, чтобы настроить для пула вычислений [узлы с низким приоритетом](https://docs.microsoft.com/azure/batch/batch-low-pri-vms). Вычислительные узлы с низким приоритетом стоят намного дешевле. На эти узлы не всегда доступны и могут быть замещены в любое время.
9. Количество зондов в обрабатываемой сцене, которое было вычислено на вкладке **Probes** (Зонды). Количество зондов определяет количество процессов моделирования, которые будут выполняться в облаке. Количество узлов не может превышать количество зондов.
10. Оценка времени на выполнения задания в облаке, не считая времени для запуска узла. После начала выполнения задания в этом поле отображается ожидаемое время получения результатов.
11. Общая длительность вычислений для выполнения этого моделирования. Это значение определяет общую длительность работы вычислительных узлов, которые будут запущены в Azure. См. сведения об [оценке стоимости моделирования в Azure](#Estimating-bake-cost).
12. Это сообщение содержит информацию о том, где будут размещены результаты моделирования после завершения задания.
13. *(Только для опытных пользователей.)* Эта кнопка удаляет из Unity все сведения об отправленном моделировании. Например, если вы скачали результаты с другого компьютера, нажмите кнопку **Clear State** (Очистить состояние), чтобы удалить сведения о задании. В этом случае готовый файл результатов *не будет* скачиваться. *Это действие отличается от отмены задания. Само задание, если оно уже запущено, продолжит выполняться в облаке.*
14. Нажмите эту кнопку, чтобы передать задание моделирования в облако. Когда задание уже выполняется, на этой кнопке появляется надпись **Cancel Job** (Отменить задание).
15. Нажмите эту кнопку, чтобы подготовиться к [обработке акустической симуляции на локальном компьютере](#Local-bake).
16. В этой области отображается состояние процесса наложения. Когда моделирование завершится, отобразится состояние Downloaded (Скачано).

Полные сведения об активных заданиях, вычислительных пулах и хранилищах вы всегда можете получить на [портале Azure](https://portal.azure.com).

Пока выполняется задание, на кнопке **Bake** (Моделирование) появляется надпись **Cancel Job** (Отменить задание). Используйте эту кнопку, чтобы отменить выполняющееся задание. Вам будет предложено подтвердить действие. Отмена задания является необратимой. В случае отмены вы не получите никаких результатов, при этом вам будет выставлен счет в соответствии со временем, потраченным на вычисления в Azure.

Начав процесс моделирования, вы можете закрыть Unity. Вычисление наложения в облаке может занять несколько часов в зависимости от характеристик проекта, типа и количества узлов. Состояние задания наложения обновляется, когда вы при перезагружаете проект и открываете окно Acoustics. Когда задание завершается, сразу же скачивается выходной файл.

Из соображений безопасности учетные данные Azure сохраняются на локальном компьютере и связываются с редактором Unity. Они используются исключительно для создания безопасного подключения к Azure.

### <a name="estimate-azure-bake-cost"></a><a name="Estimating-bake-cost"></a> Оценка затрат на моделирование в Azure

Чтобы оценить стоимость акустического моделирования, используйте значение **Estimated Compute Cost** (Оценка стоимости вычислений), которое отображает длительность процесса. Умножьте его на почасовую стоимость в местной валюте, которая применяется для выбранного **типа узла виртуальной машины**. Не забывайте, что полученный результат не учитывает время работы, необходимое для включения и настройки узлов.

Предположим, что вы выбрали тип узла **Standard_F8s_v2**, использование которого стоит 0,40 долл. США в час. Если **расчетное время вычислений** составляет 3 часа 57 минут, то общую стоимость этого задания можно приближенно оценить так: 0,4 долл. США × 4 часа = ~1,60 долл. США. Фактическая стоимость, скорее всего, будет немного выше за счет времени на подготовку узлов к работе.

Почасовую стоимость узлов см. на [странице цен на пакетную службу Azure](https://azure.microsoft.com/pricing/details/virtual-machines/linux). (Выберите категорию **Оптимизированный для вычислений** или **Высокопроизводительный**.)

## <a name="bake-your-scene-on-your-pc"></a><a name="Local-bake"></a> Моделирование сцены на ПК
Вы можете смоделировать сцену на локальном компьютере. Этот удобно для проведения экспериментов с акустикой в малых сценах перед созданием учетной записи пакетной службы Azure. Но не забывайте, что в зависимости от размера сцены локальное моделирование акустики может занять много времени.

### <a name="minimum-hardware-requirements"></a>Минимальные требования к оборудованию
* 64 или 86-разрядный процессор с минимум 8 ядрами и 32 ГБ ОЗУ.
* [Hyper-V включен](https://docs.microsoft.com/virtualization/hyper-v-on-windows/quick-start/enable-hyper-v), чтобы запустить Docker

Например, на тестовом компьютере с 8 ядрами, процессором Intel Xeon E5-1660 с частотой 3 ГГц и 32 ГБ ОЗУ получены следующие результаты:
* моделирование небольшой сцены со 100 зондами с низким разрешением занимает примерно 2 часа, а с высоким разрешением — примерно 32 часа;
* моделирование средней сцены с 1,000 зондами с грубым разрешением занимает примерно 20 часов, а с точным разрешением — примерно 21 день.

### <a name="set-up-docker"></a>Настройка Docker
Установите и настройте Docker на том компьютере, где будет обрабатываться модель.
1. [Установка Docker Desktop](https://www.docker.com/products/docker-desktop).
2. Откройте параметры Docker, перейдите в раздел **Advanced** (Расширенный) и выделите по меньшей мере 8 ГБ ОЗУ. Чем больше ресурсов ЦП вы можете выделить для Docker, тем быстрее завершится моделирование.  
![Пример параметров Docker](media/docker-settings.png)
1. Перейдите в раздел **Shared Drives** (Общие диски) и включите общий доступ к диску, используемому для обработки.  
![Параметры общего диска Docker](media/docker-shared-drives.png)

### <a name="run-the-local-bake"></a>Запустите локальное моделирование
1. Нажмите кнопку **Prepare Local Bake** (Подготовить локальное моделирование) на вкладке **Bake** (Моделирование). Затем выберите расположение папки, в которой будут храниться входные файлы и скрипты выполнения. Теперь моделирование можно выполнить на любом компьютере, который соответствует минимальным требованиям к оборудованию и на котором установлен Docker. Для этого нужно скопировать папку на этот компьютер.
2. Запустите моделирование с помощью скрипта *runlocalbake.bat* в Windows или *runlocalbake.sh* в MacOS. Этот скрипт извлекает образ Docker для Project Acoustics, в котором настроен набор инструментов для моделирования, и запускает этот процесс.
3. После завершения работы скопируйте полученный *ACE-файл* обратно в проект Unity. Чтобы Unity распознал его как двоичный файл, добавьте расширение .bytes к имени файла (например, Scene1.ace.bytes). Подробные журналы моделирования хранятся в файле *AcousticsLog.txt*. Если у вас возникнут проблемы, изучите этот файл для диагностики.

## <a name="data-files-added-by-the-bake-process"></a><a name="Data-Files"></a> Добавление файлов данных в процессе моделирования

В процессе моделирования создаются четыре файла данных. Один содержит результаты моделирования и поставляется вместе с названием. Другие хранят данные, связанные с редактором Unity.

Результат имитации:
* *Assets/AcousticsData/Acoustics\_[имя_сцены].ace.bytes*. Этот файл содержит таблицу подстановки для среды выполнения. В ней хранятся результаты моделирования и элементы акустической сцены в формате объемных пикселей. Имя и расположение этого файла можно изменить на вкладке **Probes** (Зонды).

   *Соблюдайте осторожность, чтобы не удалить файл с результатами моделирования. Его невозможно восстановить без повторного моделирования сцены.*

Редактор файлов данных:
* *Assets/Editor/[имя_сцены]\_AcousticsParameters.asset*. В этом файле хранятся данные, которые вы ввели в поля пользовательского интерфейса Acoustics. Имя и расположение этого файла изменить невозможно.
* *Assets/AcousticsData/Editor/Acoustics_[имя_сцены].vox*. В этом файле хранится геометрия акустики в формате объемных пикселей и свойства материалов, которые вычисляются с помощью кнопки **Calculate** (Вычислить) на вкладке **Probes** (Зонды). Имя и расположение этого файла можно изменить на вкладке **Probes** (Зонды).
* *Assets/AcousticsData/Editor/Acoustics\_[имя_сцены]\_config.xml*. В этом файле хранятся параметры моделирования, которые вычисляются с помощью кнопки **Calculate** (Вычислить). Имя и расположение этого файла можно изменить на вкладке **Probes** (Зонды).

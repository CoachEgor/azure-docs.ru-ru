# <a name="unsupported-language-deployments"></a>Развертывания неподдерживаемых языков

<!--Custom Translator provides the highest-quality translations possible using the latest techniques in neural machine learning. While Microsoft intends to make neural training available in all languages, there are some limitations that prevent us from being able to offer neural machine translation in all language pairs.-->  

В предстоящем прекращении использования центра Microsoft Translator Microsoft undeploying все модели, развернутые через Центр. Многие из вас имеют моделей, развертываемых в центре которого языковых пар не поддерживаются в пользовательских Translator.  Нам не нужно пользователям в этой ситуации нет ресурсов для перевода их содержимого.

Теперь у нас есть процесс, который позволяет развернуть неподдерживаемые модели через пользовательский преобразователь.  Этот процесс позволяет продолжать преобразует содержимое с помощью последней версии 3 API.  Эти модели будут размещаться пока вы решили отменить развертывание их или языковой пары не станет доступным в пользовательский преобразователь.  В этой статье объясняется процесс для развертывания моделей с парами неподдерживаемый язык.

## <a name="prerequisites"></a>Технические условия

Для модели, чтобы быть кандидатами для развертывания, они должна удовлетворять следующим условиям:
* Необходимо, проект, содержащий модели были перенесены из концентратора Custom переводчику, с помощью средства миграции.  Процесс миграции проектов и рабочих областях можно найти [здесь](how-to-migrate.md).
* Модель должна быть в развернутом состоянии, когда происходит миграции.  
* Языковой пары модели должен быть пару язык не поддерживается в пользовательских Translator.  Языковые пары, в которых поддерживается язык, либо английский язык либо, но пары, сам не включает английский, являются кандидатами для развертываний неподдерживаемый язык.  Например модель концентратора для французского, немецкого языка пару считается пару неподдерживаемый язык пары даже хотя французском языке, на английском языке и на английском языке для немецкого являются поддерживаемого языка.

## <a name="process"></a>Process
После переноса моделей из вещей, подходят для развертывания, их можно найти, перейдя к **параметры** странице рабочей области и прокрутка в конец страницы, где вы увидите **не поддерживается Центр обучения Translator** раздел.  В этом разделе отображается только в том случае, если у вас есть проекты, соответствующих требованиям, упомянутых выше.

![Перенос из центра](media/unsupported-language-deployments/unsupported-translator-hub-trainings.jpg)

В рамках **неподдерживаемый Translator центр обучения** странице "Выбор" **незапрошенного обучающих курсов** вкладка содержит модели, которые подходят для развертывания.  Выберите модель, которую вы хотите развернуть и отправить запрос.   До 30 апреля крайнего срока развертывания можно выбрать столько моделей, сколько необходимо для развертывания.
 
![Перенос из центра](media/unsupported-language-deployments/unsupported-translator-hub-trainings-list.jpg)

После отправки модель больше недоступна на **незапрошенного обучающих курсов** вкладку и вместо него будет отображаться на **запрошенный обучающих курсов** вкладки.  Ваш запрошенного обучающих курсов можно просмотреть в любое время.

![Перенос из центра](media/unsupported-language-deployments/request-unsupported-trainings.jpg) 

## <a name="whats-next"></a>Что дальше?

Модель, которую вы выбрали для развертывания сохраняются после списания концентратора и отменить развертывание всех моделей.  Это можно сделать до 24 мая, для отправки запросов для развертывания моделей не поддерживается.  Развернем эти модели, 15 июня, после чего они будут доступны через API V3 перевода.  Кроме того они будут доступны через API версии 2 до 1 июля.  

Дополнительные сведения о важные даты в устаревании концентратору чека [здесь](https://www.microsoft.com/translator/business/hub/).
После развертывания, обычное размещения плата будет взиматься.  См. в разделе [цены](https://azure.microsoft.com/pricing/details/cognitive-services/translator-text-api/) подробные сведения.  

В отличие от стандартных моделей Custom Translator центр модели будут доступны только в одном регионе, поэтому в нескольких регионах размещения не взимается.  После развертывания, можно отменить развертывание модели концентратора в любое время через переноса проекта Custom Translator.

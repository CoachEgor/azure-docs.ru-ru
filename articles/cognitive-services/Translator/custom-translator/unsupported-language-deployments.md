# <a name="unsupported-language-deployments"></a>Развертывания неподдерживаемых языков

<!--Custom Translator provides the highest-quality translations possible using the latest techniques in neural machine learning. While Microsoft intends to make neural training available in all languages, there are some limitations that prevent us from being able to offer neural machine translation in all language pairs.-->  

С предстоящим выходом на пенсию Microsoft Translator Hub, Microsoft будет развертывать все модели, в настоящее время развернутычерез через концентратор. У многих из вас есть модели, развернутые в концентраторе, языковые пары которых не поддерживаются в Custom Translator.  Мы не хотим, чтобы пользователи в этой ситуации не прибегали к переводу своего контента.

Теперь у нас есть процесс, который позволяет развертывать неподдерживаемые модели через пользовательский переводчик.  Этот процесс позволяет продолжать переводить контент с помощью последнего API V3.  Эти модели будут размещаться до тех пор, пока вы не решите развернуть их или языковая пара станет доступна в Custom Translator.  В этой статье объясняется процесс развертывания моделей с неподдерживаемыми языковыми парами.

## <a name="prerequisites"></a>Предварительные требования

Для того, чтобы ваши модели были кандидатами для развертывания, они должны соответствовать следующим критериям:
* Проект, содержащий модель, должен быть перенесен из концентратора в пользовательский переводчик с помощью инструмента миграции.  Процесс миграционных проектов и рабочих пространств можно найти [здесь.](how-to-migrate.md)
* Модель должна находиться в развернутом состоянии, когда происходит миграция.  
* Языковая пара модели должна быть неподдерживаемой языковой парой в Custom Translator.  Языковые пары, в которых язык поддерживается на английском или из английского, но сама пара не включает английский язык, являются кандидатами для неподдерживаемых развертываний языка.  Например, модель Hub для пары из французского и немецкого языков считается неподдерживаемой языковой парой, хотя французская на английском и немецком языках поддерживается языковой парой.

## <a name="process"></a>Процесс
После того как вы перенесли модели из концентратора, которые являются кандидатами для развертывания, вы можете найти их, перейдя на страницу **"Настройки"** для рабочего пространства и прокрутив до конца страницы, где вы увидите раздел **Неподдерживаемых Учебных центров переводчиков.**  Этот раздел появляется только в том случае, если у вас есть проекты, которые отвечают предпосылок, упомянутых выше.

![Перенос из центра](media/unsupported-language-deployments/unsupported-translator-hub-trainings.jpg)

На странице выбора **неподдерживаемых учебных** центров translators **Unrequested** содержит модели, которые имеют право на развертывание.  Выберите модели, которые вы хотите развернуть, и отправьте запрос.   До 30 апреля можно выбрать столько моделей, сколько пожелает для развертывания.
 
![Перенос из центра](media/unsupported-language-deployments/unsupported-translator-hub-trainings-list.jpg)

После отправки модель больше не будет доступна на вкладке **Unrequested trainings** и вместо этого будет отображаться во вкладке **Запрошенные тренинги.**  Вы можете просматривать запрошенные тренинги в любое время.

![Перенос из центра](media/unsupported-language-deployments/request-unsupported-trainings.jpg) 

## <a name="whats-next"></a>Дальнейшие действия

Выбранные для развертывания модели сохраняются после того, как концентратор будет выведен из эксплуатации и все модели не развернуты.  У вас есть время до 24 мая, чтобы отправить запросы на развертывание неподдерживаемых моделей.  Мы разместим эти модели 15 июня, после чего они будут доступны через Translator V3 API.  Кроме того, они будут доступны через API V2 до 1 июля.  

Для получения дополнительной информации о важных датах в амортизацию концентратора проверить [здесь](https://www.microsoft.com/translator/business/hub/).
После развертывания будет применяться обычная плата за хостинг.  [Ознакомьтесь](https://azure.microsoft.com/pricing/details/cognitive-services/translator-text-api/) с ценами для получения подробной информации.  

В отличие от стандартных моделей Custom Translator, модели Hub будут доступны только в одном регионе, поэтому плата за размещение в нескольких регионах не будет применяться.  После развертывания вы сможете развернуть модель концентратора в любое время в рамках проекта мигрированного пользовательского переводчика.

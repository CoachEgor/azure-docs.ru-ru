# <a name="how-the-translator-counts-characters"></a>Подсчет символов в переводчике

Переводчик учитывает каждую кодовую точку Юникода входного текста как символ. Каждый перевод текста на язык считается отдельным переводом, даже если запрос был сделан в одном вызове API для перевода на несколько языков. Длина ответа не имеет значения.

Что учитывается при подсчете:

* Текст, переданный транслятору в тексте запроса
   * `Text`, если используются методы Translate, Transliterate и Dictionary Lookup;
   * `Text` и `Translation`, если используется метод Dictionary Examples.
* Все разметка: HTML, XML-теги и т. д. в текстовом поле текста запроса. нотация JSON, используемая для создания запроса (например, "Text:") не учитывается;
* отдельная буква.
* Пунктуация
* Пробел, табуляция, разметка и любой пробел.
* Каждая кодовая точка, определенная в Юникоде.
* Повторяющийся перевод, даже если один и тот же текст был переведен ранее.

Для сценариев, основанных на идеограмс, таких как китайские и японские иероглифы, Служба переводчиков по-прежнему подсчитывает количество кодовых позиций Юникода, по одному символу на идеограм. Исключение: суррогаты Юникода считаются двумя символами.

Количество запросов, слов, байтов или предложений не учитывается при подсчете количества знаков.

Вызовы методов Detect и BreakSentence не учитываются при подсчете использованных символов. Тем не менее предполагается, что вызовы методов Detect и BreakSentence логически соотносятся с использованием других функций, которые учитываются. Если число отправленных вызовов Detect или BreakSentence превышает число других сосчитанных методов в 100 раз, корпорация Майкрософт оставляет за собой право ограничить возможность использования методов Detect и BreakSentence.

Дополнительные сведения о количестве символов см. в [разделе вопросы и ответы](https://www.microsoft.com/en-us/translator/faq.aspx)о подсчетах переводчиков.

# <a name="add-entities-to-extract-data"></a>Добавление сущностей для извлечения данных 

Создание сущностей для извлечения ключевых данных из высказываний пользователей в приложениях Language Understanding (LUIS). Извлеченные данные сущности используются вашим клиентским приложением для полного использования запросов клиентов.

Сущность представляет подлежащее извлечению слово или фразу в высказывании. Сущности описывают информацию, относящуюся к намерению, и иногда они необходимы приложению для выполнения поставленной задачи. Можно создавать сущности, когда вы добавляете высказывание на пример к намерениям или отдельно от (до или после) добавляя высказывание примера к намерениям.

[!INCLUDE [Uses preview portal](includes/uses-portal-preview.md)]

## <a name="plan-entities-then-create-and-label"></a>Планируйте сущности, а затем создавайте и маркируйте

Машино-изученные реальности могут быть созданы на примере высказываний или созданы со страницы **Сущностей.** 

В общем, наилучшей практикой является тратить время на планирование объектов, прежде чем создавать машино-изученную сущность на портале. Затем создайте машино-изученную сущность на примере высказывания с таким количеством деталей в подкомпонентах и дескрипторах и ограничениях, как вы знаете в то время. [В учебнике по разлагаемым сущностям](tutorial-machine-learned-entity.md) показано, как использовать этот метод. 

В рамках планирования сущностей может быть известно, что вам нужны объекты, соответствующие тексту (например, предварительно построенные сущности, объекты регулярного выражения или объекты списка). Вы можете создать их со страницы **Entities,** прежде чем они будут помечены в примере высказываний. 

При маркировке можно либо маркировать отдельные объекты, а затем создать объект, изучаемый родительской машиной. Или вы можете начать с родительской машины узнал лица и разложить в сущности ребенка. 

> [!TIP] 
>Пометьте все слова, которые могут указывать на сущность, даже если слова не используются при извлечении в клиентском приложении. 

## <a name="creating-an-entity-before-or-with-labeling"></a>Создание сущности до или с маркировкой

Используйте следующую таблицу, чтобы понять, какие сущности можно создать или добавить в приложение. 

|Тип сущности|Где создать сущность на портале LUIS|
|--|--|
|Сущность, прошедшая машинное обучение|Сущности или детали намерения|
|Сущность списка|Сущности или детали намерения|
|Сущность регулярного выражения|Сущности|
|Сущность Pattern.any|Сущности|
|Предварительно созданная сущность|Сущности|
|Предварительно построенная сущность домена|Сущности|

Можно создать все сущности со страницы **Сущностей** или создать несколько сущностей в рамках маркировки сущности в примере высказывания на странице **подробной информации Intent.** Можно _пометить_ объект только в примере с высказывания со страницы **подробной информации Intent.**

## <a name="create-a-text-matching-entity"></a>Создание сущности, соответствующая тексту

Использование сущностей сопоставления текста предоставляет несколько способов извлечения данных:

|Объекты, соответствующие текстовому соответствию|Назначение|
|--|--|
|[Сущность списка](#add-list-entities-for-exact-matches)|список канонических имен наряду с синонимами в качестве альтернативных форм|
|Сущность регулярного выражения|совпадают текст с помощью обычной сущности выражения|
|[Предварительно построенная сущность](tutorial-machine-learned-entity.md#add-prebuilt-number-to-help-extract-data)|совпадают с общими типами данных, такими как номер, электронная почта, дата|
|Предварительно построенная сущность домена|матч с использованием выбранных доменов предмета|
|[Pattern.Any](#add-a-patternany-entity)| для сопоставления сущностей, которые могут быть легко спутаны с окружающим текстом|  

Предварительно построенные объекты работают без предоставления каких-либо пользовательских данных обучения. Другим объектам необходимо предоставить либо данные об обучении клиентов (например, элементы организации List), либо выражение (например, обычное выражение или шаблон.

### <a name="how-to-create-a-new-custom-entity"></a>Как создать новую пользовательскую сущность

1. На портале LUIS, goto **Управление** разделе, то **Entities** странице. 
1. Выберите **: Создайте,** затем выберите тип сущности. 
1. Продолжить настройку сущности, а затем выберите **Создать,** когда вы сделали. 

### <a name="add-list-entities-for-exact-matches"></a>Добавление объектов списка для точных совпадений

Сущности списка — это фиксированный закрытый набор связанных машинных слов. В то время как вы, как автор, можете изменить список, LUIS не будет расти или сокращать сяризне. Вы также можете импортировать существующую сущность списка с помощью формата «список сущность .json» (reference-entity-list.md-example-json-to-import-into-list-entity). 

В следующем списке показано каноническое название и синонимы. 

|Цвет - имя элемента списка|Цвет - синонимы|
|--|--|
|Красный|малиновый, кровь, яблоко, пожарный двигатель|
|Синий|небо, лазурный, кобальт|
|Зеленый|Келли, лайм|

Используйте процедуру для создания сущности списка. После создания сущности списка не нужно маркировать примервысказываний в намеренном. Элементы списка и синонимы сопоставляются с использованием точного текста. 

1. Из раздела **Сборка** выберите **Сущности** в левой панели, а затем выберите **: Создать**.

1. В поле диалога **типа «Создание»** введите имя объекта, например, `Colors` и выберите **Список.**
1. В **поле диалога сущности списка,** в **добавлении нового подсписка....,** `Green`введите имя элемента списка, например, , затем добавьте синонимы.

    > [!div class="mx-imgBorder"]
    > ![Создайте список цветов в качестве сущности списка на странице детализации Entity.](media/how-to-add-entities/create-list-entity-of-colors.png) 

1. Когда вы закончите добавлять элементы списка и синонимы, выберите **Создать.**

    Когда вы закончите с группой изменений в приложении, не забудьте **обучить** приложение. Не тренируйте приложение после одного изменения. 

    > [!NOTE]
    > Эта процедура демонстрирует создание и маркировку сущности списка из примера высказывания на странице **подробной информации Intent.** Можно также создать ту же сущность со страницы **Сущностей.**

## <a name="add-a-role-for-an-entity"></a>Добавление роли для сущности

Роль — это подтип имени сущности, основанный на контексте. 

### <a name="add-a-role-to-distinguish-different-contexts"></a>Добавление роли для различения различных контекстов

В следующем высказывании, Есть два места, и каждый определяется семантически слова вокруг него, такие как `to` и: `from` 

`Pick up the package from Seattle and deliver to New York City.`

В этой процедуре добавляйте `origin` и `destination` роли к заранее построенной сущности geographyV2.

1. В разделе **Сборка** на левой панели выберите **Сущности**.

1. Выберите **и добавьте заранее построенную сущность.** Выберите **географиюV2,** затем выберите **Done.** Это добавляет заранее построенную сущность в приложение.
    
    Если ваш шаблон, который включает сущность Pattern.any, извлекает сущности неправильно, используйте [явный список](reference-pattern-syntax.md#explicit-lists), чтобы решить эту проблему. 

1. Выберите недавно добавленную заранее построенную сущность geographyV2 из списка объектов **страницы Entities.** 
1. Чтобы добавить новую **+** роль, выберите рядом с **добавленные роли.**
1. В **роли типа...** текстовый ящик введите название роли, `Origin` а затем введите. Добавить имя второй `Destination` роли затем введите. 

    > [!div class="mx-imgBorder"]
    > ![Снимок экрана добавления роли отправления сущности Location](media/how-to-add-entities//add-role-to-prebuilt-geographyv2-entity.png)

    Роль добавляется к заранее построенной сущности, но не добавляется к каким-либо высказываниям с использованием этого объекта. 

### <a name="label-text-with-a-role-in-an-example-utterance"></a>Текст этикетки с ролью в примере высказывания

1. Перейдите на страницу сведения о намерениях, на которой есть примерные высказывания, в которых используется роль. 
1. Чтобы пометить роль, выберите метку сущности (твердая линия под текстом) в примере высказывания, а затем выберите **View in entity palette** из списка выпадающих. 

    > [!div class="mx-imgBorder"]
    > ![Скриншот выбора View в палитре сущности](media/how-to-add-entities/select-text-label-with-entity-palette-for-role.png)   

    Палитра сущности открывается справа. 

1. Выберите сущность, затем перейдите в нижней части палитры и выберите роль. 

    > [!div class="mx-imgBorder"]
    > ![Скриншот выбора View в палитре сущности](media/how-to-add-entities/select-role-from-entity-palette-entity-inspector.png)

## <a name="add-a-patternany-entity"></a>Добавление шаблона.любая сущность

[Pattern.any](luis-concept-entity-types.md) сущности действительны только в [шаблонах,](luis-how-to-model-intent-pattern.md)а не в примере намерений высказываний. Этот вид сущности помогает службе LUIS найти конец сущностей различной длины и выбора машинного слова. Так как эта сущность используется в шаблоне, служба LUIS знает, где конец сущности в шаблоне высказывания.

### <a name="steps-to-create-a-patternany-entity"></a>Шаги для создания шаблона.любая сущность

1. Из раздела **Сборка** выберите **Сущности** в левой панели, а затем выберите **: Создать**.

1. В поле диалога **типа «Выбрать объект»** введите имя сущности в поле **«Имя»** и выберите **Pattern.Any** по мере того, как **тип** затем выберите **Создать.**

    Как только вы [создаете высказывание шаблона](luis-how-to-model-intent-pattern.md) с помощью этой сущности, сущность извлекается с комбинированным алгоритмом сочетания машин и сопоставления текста. 

### <a name="create-a-pattern-template-utterance-to-use-patternany-entity"></a>Создание высказывания шаблона шаблона для использования pattern.any entity

Чтобы использовать сущность pattern.any, добавьте шаблон на странице **Шаблоны** в разделе **Повышение производительности приложения** с помощью правильных фигурных скобок, например `Where is **{HumanResourcesFormTitle}** on the server?`.

Если ваш шаблон, который включает сущность Pattern.any, извлекает сущности неправильно, используйте [явный список](reference-pattern-syntax.md#explicit-lists), чтобы решить эту проблему. 

## <a name="do-not-change-entity-type"></a>Не изменяйте тип сущности

Интеллектуальная служба распознавания речи не позволяет изменить тип сущности, поскольку неизвестно, что добавлять или удалять для создания этой сущности. Чтобы изменить тип, лучше создать новую сущность правильного типа с незначительно отличающимся именем. После создания сущности в каждом высказывании удалите старое отмеченное имя сущности и добавьте новое. После того как все высказывания будут повторно отмечены, удалите старую сущность.

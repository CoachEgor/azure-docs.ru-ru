# <a name="boundaries-for-your-luis-model-and-keys"></a>Границы модели LUIS и ключи
У LUIS есть несколько областей границ. Первая — [граница модели](#model-boundaries), которая управляет намерениями, сущностями и возможностями в LUIS. Вторая область — [пределы квот](#key-limits) на основе типа ключа. Третья область границ — [сочетания клавиш](#keyboard-controls) для управления веб-сайтом LUIS. Четвертая область — [сопоставление региона мира](luis-reference-regions.md) между веб-сайтом разработки LUIS и API-интерфейсами [конечной точки](luis-glossary.md#endpoint) LUIS.

## <a name="name-uniqueness"></a>Название уникальности

Используйте следующие правила уникальности именования.

Следующие должны быть уникальными в приложении LUIS:

* название версии
* intent
* сущность
* roles

Следующие должны быть уникальными в применяемой области:

* список фраз

## <a name="object-naming"></a>Именование объектов

Не используйте следующие символы в следующих именах.

|Объект|Исключить символы|
|--|--|
|Намерения, сущность и имена ролей|`:`<br>`$` <br> `&`|
|Имя версии|`\`<br> `/`<br> `:`<br> `?`<br> `&`<br> `=`<br> `*`<br> `+`<br> `(`<br> `)`<br> `%`<br> `@`<br> `$`<br> `~`<br> `!`<br> `#`|

## <a name="resource-usage-and-limits"></a>Использование ресурсов и ограничения

Language Understand имеет отдельные ресурсы, один тип для авторизации и один тип для запроса конечную точку прогнозирования. Дополнительные сведения о различиях между основными типами ключей см. в статье [Ключи разработки и запрашивания конечной точки прогнозирования в LUIS](luis-concept-keys.md).

### <a name="authoring-resource-limits"></a>Авторство лимитов ресурсов

Используйте _вид,_ `LUIS.Authoring`при фильтрации ресурсов на портале Azure. LUIS ограничивает 500 приложений для авторов ресурса Azure.

|Авторинг-ресурс|Авторство TPS|
|--|--|
|Ключ для начала работы|1 млн/месяц, 5/с|
|F0 - Бесплатный уровень |1 млн/месяц, 5/с|

* TPS - Транзакции в секунду

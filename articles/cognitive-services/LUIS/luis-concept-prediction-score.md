# <a name="prediction-scores-indicate-prediction-accuracy-for-intent-and-entities"></a>Оценки прогнозирования отражают точность прогнозирования намерения и сущностей

Оценку прогноза обозначает степень достоверности LUIS имеет для результатов прогноза, исходя из utterance пользователя.

Оценка прогнозирования находится в диапазоне от нуля (0) до единицы (1). Пример оценки прогнозирования LUIS с высокой степенью достоверности — 0,99. Пример оценки прогнозирования LUIS с низкой достоверностью — 0,01. 

|Значение оценки|Достоверность|
|--|--|
|1|Точное соответствие|
|0.99|Высокая степень достоверности|
|0,01|Низкая степень достоверности|
|0|Точное несоответствие|

Если высказывание обладает низкой степенью достоверности, LUIS подчеркивает, что на странице **Intent** веб-сайта [LUIS](luis-reference-regions.md) идентифицировано **помеченное намерение**, выделенное красным.

![Расхождение оценки](./media/luis-concept-score/score-discrepancy.png)

## <a name="top-scoring-intent"></a>Намерение с максимальной оценкой

Для каждого прогнозирования высказывание возвращает намерение с максимальной оценкой. Этот прогноз представляет собой числовое сравнение прогнозирующие оценки. Верхних оценок 2 может иметь очень небольшое различие между ними. LUIS не указано, что этот с учетом расположения, отличного от возврата верхней оценка.  

## <a name="return-prediction-score-for-all-intents"></a>Возврат оценки прогнозирования для всех намерений

Результат для теста или конечной точки может содержать все намерения. Такая конфигурация задается в [конечной точке](https://aka.ms/v1-endpoint-api-docs) с помощью пары "имя/значение" `verbose=true` в строке запроса.

## <a name="review-intents-with-similar-scores"></a>Просмотр намерений с похожими оценками

Просмотр оценки для всех намерений позволяет не только определить правильное намерение, но и сделать так, чтобы оценка каждого следующего определенного намерения была значительно ниже для всех высказываний.

Если у нескольких намерений есть близкие оценки прогнозирования, то на основе контекста высказывания LUIS может переключаться между намерениями. Чтобы устранить эту проблему, добавление высказываний в каждом намерения с более широкий диапазон контекстные различия по-прежнему или вы можете устанавливать приложение, например чат-бот, делать программный Выбор способ обработки 2 top намерений.

2 целей, которые оцениваются по слишком тесно, может обратить из-за недетерминистической обучения. Первая оценка может стать второй, а вторая может стать первой. Чтобы предотвратить такую ситуацию, добавление высказываний примере к каждому top двух способов для этого utterance word и богатство выбора контекст, который отличает 2 намерений. Для двух намерений нужно добавить приблизительно одинаковое число примеров фрагментов речи. Общее правило для разделения для предотвращения изменения порядка намерений из-за обучение заключается в том, что разница в оценках должна составлять 15 %.

Вы можете отключить недетерминированным обучения по [обучения со всеми данными](luis-how-to-train.md#train-with-all-data).

## <a name="differences-with-predictions-between-different-training-sessions"></a>Различия между сеансами разными обучающими прогнозами

При обучении той же модели, в другое приложение и оценки не совпадают, это различие связано недетерминированным обучения (элемент случайности). Во-вторых, любое перекрытие высказывания более чем на одно намерение означает, что намерение с максимальной оценкой для того же высказывания может измениться в зависимости от обучения.

Если вашей чатбот требуют более конкретных LUIS для указания более уверенно работать на объект intent, следует использовать оценку разницу между верхней двух целей. Это обеспечивает гибкость для различных вариантов обучения.

## <a name="e-exponent-notation"></a>Запись E (экспоненциальная запись)

Оценки прогнозирования можно записывать в форме экспоненциальной записи, *над* диапазоном 0–1, например `9.910309E-07`. Эта оценка свидетельствует об очень **маленьком** числе.

|Экспоненциальная запись оценки |Фактическая оценка|
|--|--|
|9.910309E-07|0,0000009910309|

## <a name="punctuation"></a>Пунктуация

В LUIS пунктуация является отдельным маркером. Utterance, который содержит точку в конце и utterance, который не содержит точку в конце являются два отдельных фразы и может получить два разных прогноза. Убедитесь, что модель обрабатывает пунктуацию согласно [примерам высказываний](luis-concept-utterance.md) (имеющих и не имеющих пунктуации) или как [pattern](luis-concept-patterns.md), в котором пунктуацию легче игнорировать с помощью специального синтаксиса: `I am applying for the {Job} position[.]`.

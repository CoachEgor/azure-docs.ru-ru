# <a name="machine-learned-features"></a>Машино-изученные характеристики 

В машинном обучении _функция_ является отличительной чертой или атрибутом данных, которые ваша система наблюдает &. В Language Understanding (LUIS) функция описывает и объясняет, что важно в ваших намерениях и сущностях.

На [портале preview LUIS](https://preview.luis.ai)функции являются _дескрипторами,_ поскольку они используются для _описания_ намерения или сущности.  

## <a name="features-_descriptors_-in-language-understanding"></a>Особенности _(дескрипторы)_ в понимании языка

Функции, также известные как дескрипторы, описывают подсказки, помогая языковому пониманию определить пример высказываний. Доступны следующие функции: 

* Список фраз как функция для намерений или сущностей
* Сущности как объекты для намерений или сущностей

Функции следует рассматривать как необходимую часть вашей схемы для разложения модели. 

## <a name="what-is-a-phrase-list"></a>Что такое список фраз

Список фраз — это список слов, фраз, чисел или других символов, которые помогают определить концепцию, которую вы пытаетесь определить. Список нечувствительный. 

## <a name="when-to-use-a-phrase-list"></a>Когда использовать список фраз

С помощью списка фраз LUIS учитывает контекст и обобщает для определения элементов, которые похожи на, но не точно совпадают с текстом. Если вам нужно приложение LUIS, чтобы иметь возможность обобщать и идентифицировать новые элементы, используйте список фраз. 

Если вы хотите иметь возможность распознавать новые экземпляры, такие как планировщик собраний, который должен распознавать имена новых контактов, или приложение инвентаризации, которое должно распознавать новые продукты, начните с объекта, изучаемого машиной. Затем создайте список фраз, который помогает LUIS найти слова с похожим значением. Этот список фраз направляет LUIS распознавать примеры, добавляя дополнительное значение стоимости этих слов. 

Списки фраз похожи на словари в предметной области, помогающие улучшить качество понимания намерений и сущностей. 

## <a name="considerations-when-using-a-phrase-list"></a>Рассмотрение при использовании списка фраз

По умолчанию список фраз применяется ко всем моделям в приложении. Это будет работать для списков фраз, которые могут пересекать все намерения и сущности. Для разлагаемости следует применять список фраз только к тем моделям, к которые он имеет отношение. 

Если вы создаете список фраз (созданный по умолчанию по умолчанию), затем примените его в качестве дескриптора (функции) к конкретной модели, он удаляется из других моделей. Это удаление добавляет релевантность списку фраз для модели, к которую оно применяется, помогая повысить точность, которую она обеспечивает в модели. 

Флаг `enabledForAllModels` управляет этой областью модели в API. 

<a name="how-to-use-phrase-lists"></a>

### <a name="how-to-use-a-phrase-list"></a>Как использовать список фраз

[Создайте список фраз,](luis-how-to-add-features.md) когда у вашего намерения или сущности есть важные слова или фразы, такие как:

* отраслевая терминология;
* сленг;
* сокращения;
* специальный для компании язык;
* язык, происходящий из другого языка, но часто используемый в вашем приложении;
* ключевые слова и фразы в примере фразы;

Не **добавляйте** все возможные слова или фразы. Вместо этого добавьте несколько слов или фраз за один раз, затем переучивайте и публикуйте. По мере того как список растет над временем, вы можете найти некоторые термины имеют много форм (синонимов). Разбейте их в другой список.

## <a name="when-to-use-an-entity-as-a-feature"></a>Когда использовать сущность в качестве функции 

Сущность может быть добавлена в качестве функции на уровне намерения или сущности. 

### <a name="entity-as-a-feature-to-an-intent"></a>Сущность как функция намерения

Добавьте сущность в качестве дескриптора (функции) к намерениям, когда обнаружение этой сущности имеет важное значение для намерения.

Например, если целью является бронирование рейса, а сущностью является информация о билетах (например, количество мест, происхождения и пункта назначения), то поиск организации, информирующей о билете, должен добавить вес предсказанию намерения по полету книги. 

### <a name="entity-as-a-feature-to-another-entity"></a>Сущность как функция для другого объекта

Сущность (A) должна быть добавлена в качестве объекта другому объекту (B), когда обнаружение этой сущности (A) имеет важное значение для прогнозирования сущности (B).

Например, если обнаружен адрес улицы (A), то поиск адреса улицы (A) добавляет вес к прогнозу для сущности адреса доставки (B).

# <a name="authoring-cycle-for-your-luis-app"></a>Цикл разработки приложения LUIS
LUIS лучше всего обучается на итеративных циклах изменения моделей, примерах высказываний, публикации содержимого и сборе данных из запросов к конечным точкам. 

![Цикл разработки](./media/luis-concept-app-iteration/iteration.png)

## <a name="building-a-luis-model"></a>Создание модели LUIS
Назначение модели — понять, что запрашивает пользователь (намерение или назначение) и какие части вопроса содержат сведения (сущности), позволяющие определить ответ. 

Чтобы определить связанные слова и фразы, а также стандартный порядок слов, модель должна относиться к конкретной предметной области приложения. 

Модель требует указания целей и _должна иметь_ сущности. 

## <a name="add-training-examples"></a>Добавление обучающих примеров
В намерениях LUIS требуются примеры высказываний. Чтобы определять цель высказывания, примеры должны содержать достаточно вариантов выбора слов и порядка слов. В каждом примере высказывания все необходимые данные должны быть помечены как сущности. 

Чтобы указать LUIS игнорировать высказывания, которые не относятся к предметной области приложения, следует назначить высказывание намерению **None**. Слова или фразы, которые не нужно извлекать из высказывания, помечать не требуется. Метка для игнорируемых слов или фраз, отсутствует. 

## <a name="train-and-publish-the-app"></a>Обучение и публикация приложения
После создания 15–30 различных фразы продолжительностью в каждом намерении с пометкой обязательных сущностей необходимо [обучить](luis-how-to-train.md) и [опубликовать](luis-how-to-publish-app.md). Получив уведомление об успешной публикации, перейдите по ссылке, чтобы получить свои конечные точки. Убедитесь, что вы создали и опубликовали свое приложение, чтобы оно было доступно в нужных [регионах конечных точек](luis-reference-regions.md) . 

## <a name="https-prediction-endpoint-testing"></a>Тестирование конечной точки с прогнозированием HTTPS
Вы можете протестировать приложение LUIS из конечной точки прогнозирования HTTPS. Тестирование из конечной точки прогнозирования позволяет LUIS выбирать любой фразы продолжительностью с низкой достоверностью для [проверки](luis-how-to-review-endpoint-utterances.md).  

## <a name="recycle"></a>Перезапуск

Завершив цикл разработки, можно начать процесс снова. Начните с [проверки конечной точки прогнозирования фразы продолжительностью](luis-how-to-review-endpoint-utterances.md) Luis, помеченной с низким уровнем достоверности. В этих высказываниях проверьте намерения и сущности. После работы с высказываниями список проверки должен быть пустым.  

Рассмотрите возможность [клонирования](luis-concept-version.md#clone-a-version) текущей версии в новую версию, а затем начните разработку изменений в новой версии. 

## <a name="batch-testing"></a>Пакетное тестирование

[Пакетное тестирование](luis-concept-batch-test.md) — это способ увидеть, сколько примеров фразы ПРОДОЛЖИТЕЛЬНОСТЬЮ оценивает Luis. Примеры должны быть новыми для LUIS, в них должны быть правильно помечены намерения и сущности, которые требуется найти LUIS. Из результатов теста можно узнать, насколько успешно LUIS будет работать с этим набором высказываний.

# <a name="how-to-use-the-dashboard-to-improve-your-app"></a>Как улучшить приложение с помощью панели мониторинга

Поиск и устранение проблем с намерениями обученной приложения при использовании пример фразы. На панели мониторинга отображаются общие сведения приложения, с выделенными намерений, которые должны быть исправлены. 

Просмотрите панель мониторинга, анализа является интерактивным процессом, повторяющиеся, как изменить и улучшить модель.

Эта страница не будет иметь соответствующие анализа для приложений, у которых нет любой пример фразы в намерения, известный как _только для шаблона_ приложений. 

## <a name="what-issues-can-be-fixed-from-dashboard"></a>Какие проблемы могут быть исправлены с панели мониторинга?

Ниже приведены три проблемы, решаемые на панели мониторинга.

|Проблема|Цвет диаграммы|Объяснение|
|--|--|--|
|Несбалансированность данных|-|Это происходит, когда существенно изменяется количество пример фразы. Все объекты Intent нам нужно _примерно_ столько же примере фразы — за исключением нет намерений. Он должен иметь только 10 – 15% от общего количества фразы в приложении.<br><br> Если данные, но намерений точность превышает определенный порог, этот дисбаланс не сообщается как ошибки.<br><br>**Начать с этой проблемы - он может стать основной причиной других проблем.**|
|Непонятно прогнозов|Оранжевый|Это происходит, когда верхний намерения и далее намерение оценки находятся достаточно близко для того, они могут отражение на Далее обучения, из-за [отрицательной выборке](luis-how-to-train.md#train-with-all-data) или дополнительные пример фразы, добавлены в намерения. |
|Неверные прогнозы|Красный|Это происходит, когда utterance пример не прогнозируемое для помеченных цель (цель, которой он находится).|

Правильно угаданных представлены синий цвет.

Панели мониторинга отображаются эти проблемы и рассказывается о том, какие объекты Intent затрагиваются и предполагает, что следует сделать для улучшения приложений. 

## <a name="before-app-is-trained"></a>До обучения приложения 

Прежде чем начинать обучение приложения, панели мониторинга не содержит какие-либо предложения для устранения проблемы. Обучение приложение, чтобы просмотреть эти предложения.

## <a name="review-training-evaluation"></a>Просмотрите оценки обучения

**Оценки обучения** карта содержит статистические сводки обо общая точность приложения по областям. Оценка указывает намерений качество. 

![Карта обучения оценки содержит первой области элемента сведения о общая точность вашего приложения.](./media/luis-how-to-use-dashboard/analytics-card-2-shows-app-overall-accuracy.png)

Диаграмма показывает, что правильно прогнозируемое намерения и проблемных областей с различными цветами. Как вы улучшите приложение с предложениями, этот показатель увеличивается. 

Предлагаемые исправления разделяются по типу проблемы и являются наиболее важными для вашего приложения. Если вы хотите использовать проверить и исправить проблемы в намерения, используйте **[намерения с ошибками](#intents-with-errors)** карточки в нижней части страницы. 

Каждая область проблема имеет намерений, которые должны быть исправлены. При выборе имени намерений **намерение** фильтр, применяемый к фразы при открытии страницы. Этот фильтр позволяет сосредоточиться на фразы, которые являются причиной проблемы.

### <a name="compare-changes-across-versions"></a>Сравните изменения всех версий

Создайте новую версию, прежде чем вносить изменения в приложение. В новой версии измените предложенные намерение пример фразы, а затем снова обучения. На странице панели мониторинга **оценки обучения** карте **Показать изменения из обученной версии** для сравнения изменения. 

![Сравните изменения всех версий](./media/luis-how-to-use-dashboard/compare-improvement-across-versions.png)

### <a name="fix-version-by-adding-or-editing-example-utterances-and-retraining"></a>Исправить версии путем добавления или редактирования пример фразы и повторного обучения

Чтобы добавить или изменить пример фразы и повторное обучение будет основной способ исправления приложения. Новые или измененные фразы должны следовать рекомендациям [разнообразными фразы](luis-concept-utterance.md).

Добавление высказываний пример следует выполнить кем-то кто:

* имеет высокую степень понимания того, что такое фразы в разными намерениями
* знает, как можно принять фразы в одной целью с другой целью.
* может решить, если два целей, которые часто путают друг с другом, следует объединить в одной цели и извлекать различные данные с сущностями

### <a name="patterns-and-phrase-lists"></a>Шаблоны и списки фразы

На странице аналитики не указывает, когда следует использовать [шаблоны](luis-concept-patterns.md) или [фразу списки](luis-concept-feature.md). Если добавить их, может помочь с неправильные или непонятный прогнозов, но не поможет с дисбаланс данных. 

### <a name="review-data-imbalance"></a>Несбалансированность проверки данных

Начать с этой проблемы - он может стать основной причиной других проблем.

**Дисбаланс данных** намерений перечислены способы, которым требуется более фразы для исправления данных дисбаланс. 

**Чтобы устранить эту проблему**:

* Добавление высказываний дополнительные к объекту intent, а затем снова обучения. 

Пока не добавлять фразы нет намерений, предлагаемое на панели мониторинга.

> [!Tip]
> Использовать третий раздел на странице **высказываний на намерение** с **фразы (number)** установку в качестве быстрого визуальную подсказку, из которых намерения необходимые дополнительные фразы.  
    ![Используйте «Фразы (число)» для поиска намерения с дисбаланс данных.](./media/luis-how-to-use-dashboard/predictions-per-intent-number-of-utterances.png)

### <a name="review-incorrect-predictions"></a>Просмотреть неверные прогнозы

**Неверный прогноз** намерений перечислены способы, которые имеют фразы, которые используются в качестве примеров для определенной цели, но по прогнозу для разных целей. 

**Чтобы устранить эту проблему**:

* Измените фразы точнее к цели, а также проводить обучение еще раз.
* Объединить объекты Intent, если фразы слишком тесно выравниваются и повторно обучить.

### <a name="review-unclear-predictions"></a>Просмотрите непонятно прогнозов

**Непонятно прогноза** намерений перечислены способы со фразы с прогнозирующие оценки, которые не являются достаточно далеко способом из их ближайшего конкурент, которое top цель для utterance может изменяться на Далее обучения, из-за [ отрицательной выборке](luis-how-to-train.md#train-with-all-data).

**Чтобы устранить эту проблему**;

* Измените фразы точнее к цели, а также проводить обучение еще раз.
* Объединить объекты Intent, если фразы слишком тесно выравниваются и повторно обучить.

### <a name="filter-intents-by-issue-and-percentage"></a>Фильтр намерений, проблемы и процент

В этом разделе карточки можно найти пример фразы, снижается за пределами вашей пороговое значение ошибки. В идеале нужно правильно угаданных как значительные. Процент от — на основе клиентов и бизнеса. 

Определите процентный порог, которые вы уже освоились с для вашего бизнеса. 

Фильтр позволяет находить объекты Intent с определенной проблемой:

|Фильтр|Предложенный процент|Назначение|
|--|--|--|
|Наиболее проблематичные намерений|-|**Начните здесь** -Устранение фразы в этого намерения улучшит приложения больше, чем другие исправления.|
|Ниже правильными прогнозами|60 %|Это процент фразы в выбранный метод, которые указаны правильно, но также имеют показатель достоверности ниже порогового значения. |
|Непонятно прогнозов выше|15 %|Это процент фразы в выбранный метод, которые являются путать с целью ближайшего-конкурента.|
|Неверные прогнозы выше|15 %|Это процент в выбранный метод фразы, которые прогнозируются неправильно. |

### <a name="correct-prediction-threshold"></a>Пороговое значение правильного прогноза

Что такое показатель достоверности уверены прогноза для вас? В начале разработки приложений, 60% может быть на целевом сервере. Используйте **исправьте прогнозов ниже** с процентным отношением 60%, чтобы найти любые фразы продолжительностью в выбранной пожеланий, должны быть исправлены.

### <a name="unclear-or-incorrect-prediction-threshold"></a>Прогноз непонятно или неправильное пороговое значение

Эти два фильтра позволяют найти фразы в выбранный метод за пределы порогового. Можно считать эти два процентов проценты ошибок. Если вы знакомы с частота ошибок на 10 – 15% для создания прогнозов, задайте пороговое значение фильтра на 15%, чтобы найти все фразы, при достижении этого значения.

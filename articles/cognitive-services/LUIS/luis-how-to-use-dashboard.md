# <a name="how-to-use-the-dashboard-to-improve-your-app"></a>Как использовать панель мониторинга для улучшения приложения

Найти и устранить проблемы с намерениями вашего обученного приложения, когда вы используете пример высказываний. Панель мониторинга отображает общую информацию о приложении, с основными моментами намерений, которые должны быть исправлены. 

Анализ панели мониторинга — это итеративный процесс, повторяемый при изменении и улучшении модели.

Эта страница не будет иметь соответствующего анализа для приложений, которые не имеют _каких-либо_ примеров высказываний в намерениях, известных как приложения только для шаблонов. 

## <a name="what-issues-can-be-fixed-from-dashboard"></a>Какие проблемы можно устранить с панели мониторинга?

Три проблемы, решенные в панели мониторинга:

|Проблема|Цвет диаграммы|Объяснение|
|--|--|--|
|Дисбаланс данных|-|Это происходит, когда количество высказываний на примере значительно варьируется. Все намерения должны иметь _примерно_ одинаковое количество примеров высказываний - за исключением намерения None. Он должен иметь только 10%-15% от общего количества высказываний в приложении.<br><br> Если данные несбалансированы, но точность намерения выше определенного порога, этот дисбаланс не сообщается как проблема.<br><br>**Начните с этой проблемы - это может быть основной причиной других проблем.**|
|Неясные прогнозы|Оранжевый|Это происходит, когда верхний намерения и оценки следующего намерения достаточно близко, что они могут перевернуть на следующей подготовки, из-за [отрицательной выборки](luis-how-to-train.md#train-with-all-data) или более пример высказывания добавил к намерениям. |
|Неправильные прогнозы|Красный|Это происходит, когда высказывание примера не прогнозируется для помеченного намерения (намерения, в нем он находится).|

Правильные прогнозы представлены с синим цветом.

Панель мониторинга показывает эти проблемы и сообщает, какие намерения затронуты, и предлагает, что вы должны сделать, чтобы улучшить приложение. 

## <a name="before-app-is-trained"></a>Перед обучением приложения 

Перед тем, как обучить приложение, панель мониторинга не содержит никаких предложений по исправлению. Обучите приложение, чтобы увидеть эти предложения.  

## <a name="check-your-publishing-status"></a>Проверьте свой статус публикации

Карта **состояния публикации** содержит информацию о последней публикации активной версии. 

Убедитесь, что активная версия — это версия, которую вы хотите исправить. 

![Панель мониторинга показывает внешние службы приложения, опубликованные регионы и агрегированные конечные точки.](./media/luis-how-to-use-dashboard/analytics-card-1-shows-app-summary-and-endpoint-hits.png)

Это также показывает любые внешние службы, опубликованные регионы и агрегированные конечные точки. 

## <a name="review-training-evaluation"></a>Обзор оценки обучения

Карта **оценки обучения** содержит агрегированную сводку общей точности приложения по регионам. Оценка указывает на качество намерения. 

![Карта оценки обучения содержит первую область информации об общей точности приложения.](./media/luis-how-to-use-dashboard/analytics-card-2-shows-app-overall-accuracy.png)

Диаграмма указывает правильно прогнозируемые намерения и проблемные области с разными цветами. По мере улучшения приложения с предложениями, этот балл увеличивается. 

Предлагаемые исправления разделены по типу проблеми и являются наиболее значительными для вашего приложения. Если вы предпочитаете рассматривать и исправлять проблемы на один умысел, используйте **[«Намерения» с](#intents-with-errors)** картой ошибок в нижней части страницы. 

Каждая проблемная область имеет намерения, которые должны быть исправлены. При выборе имени намерения страница **Intent** открывается фильтром, применяемым к высказываниям. Этот фильтр позволяет сосредоточиться на высказываниях, которые вызывают проблемы.

### <a name="compare-changes-across-versions"></a>Сравнение изменений в разных версиях

Создайте новую версию, прежде чем вносить изменения в приложение. В новой версии внести предлагаемые изменения в пример намерения, а затем снова тренироваться. Для сравнения изменений используйте **изменение show от подготовленной версии** на странице **«Учебная панель».** 

![Сравнение изменений в разных версиях](./media/luis-how-to-use-dashboard/compare-improvement-across-versions.png)

### <a name="fix-version-by-adding-or-editing-example-utterances-and-retraining"></a>Исправить версию, добавив или редактируя пример высказываний и переподготовки

Основным методом исправления приложения будет добавление или отодвилость от примера высказываний и переподготовка. Новые или измененные высказывания должны следовать руководящим принципам для [разнообразных высказываний.](luis-concept-utterance.md)

Добавление примера высказываний должно быть сделано кем-то, кто:

* имеет высокую степень понимания того, что высказывания в различных намерениях.
* знает, как высказывания в одном намерении могут быть спутаны с другим намерением.
* может решить, следует ли свернуть два намерения, которые часто путают друг с другом. В этом случае различные данные должны быть извлечены с помощью сущностей.

### <a name="patterns-and-phrase-lists"></a>Шаблоны и списки фраз

Страница аналитики не указывает, когда использовать [шаблоны](luis-concept-patterns.md) или [списки фраз.](luis-concept-feature.md) Если вы добавите их, это может помочь с неправильными или неясными прогнозами, но не поможет с дисбалансом данных. 

### <a name="review-data-imbalance"></a>Обзор диспропорции данных

Начните с этой проблемы - это может быть основной причиной других проблем.

Список **намерений по дисбалансу данных** показывает намерения, которые нуждаются в большем количестве высказываний для исправления дисбаланса данных. 

**Чтобы исправить эту проблему:**

* Добавить больше высказываний к намерениям, а затем поезд снова. 

Не добавляйте высказывания в намерения None, если это не предлагается на панели мониторинга.

> [!Tip]
> Используйте третий раздел на странице, **Высказывания в намерениях** с **настройками Utterances (номер),** в качестве быстрого визуального руководства, намерения которого нуждаются в большем количестве высказываний.  
    ![Используйте 'Utterances (число)' для поиска намерений с дисбалансом данных.](./media/luis-how-to-use-dashboard/predictions-per-intent-number-of-utterances.png)

### <a name="review-incorrect-predictions"></a>Просмотр неверных прогнозов

В списке **неверных** намерений прогнозирования отображаются намерения, которые имеют высказывания, которые используются в качестве примеров для конкретного намерения, но прогнозируются для различных намерений. 

**Чтобы исправить эту проблему:**

* Отоденять высказывания, чтобы быть более конкретными для намерения и тренироваться снова.
* Объедините намерения, если высказывания слишком тесно выровнены, и тренируйтесь снова.

### <a name="review-unclear-predictions"></a>Обзор неясных прогнозов

**Неясный** список намерений прогноза показывает намерения с высказываниями с оценками предсказаний, которые не достаточно далеко от своего ближайшего соперника, что верхняя цель для высказывания может измениться на следующей тренировке, из-за [отрицательной выборки.](luis-how-to-train.md#train-with-all-data)

**Чтобы исправить эту проблему**;

* Отоденять высказывания, чтобы быть более конкретными для намерения и тренироваться снова.
* Объедините намерения, если высказывания слишком тесно выровнены, и тренируйтесь снова.

### <a name="filter-intents-by-issue-and-percentage"></a>Намерения фильтра по вопросу и проценту

Этот раздел карты позволяет найти пример высказываний, которые выходят за пределы порога ошибки. В идеале вы хотите, чтобы правильные прогнозы были значительными. Этот процент является бизнес омичем и клиентом. 

Определите процентпорога, который вам удобно для вашего бизнеса. 

Фильтр позволяет находить намерения с конкретной проблемой:

|Filter|Рекомендуемый процент|Назначение|
|--|--|--|
|Наиболее проблемные намерения|-|**Начните здесь** - Исправление высказываний в этом намерении улучшит приложение больше, чем другие исправления.|
|Правильные прогнозы ниже|60%|Это процент высказываний в выбранном намерении, которые являются правильными, но имеют оценку доверия ниже порога. |
|Неясные прогнозы выше|15 %|Это процент высказываний в выбранном намерении, которые путают с ближайшим намерением соперника.|
|Неправильные прогнозы выше|15 %|Это процент высказываний в выбранном намерении, которые неправильно предсказаны. |

### <a name="correct-prediction-threshold"></a>Правильный порог прогнозирования

Что такое уверенный прогноз доверия оценка для вас? В начале разработки приложения 60% могут быть вашей целью. Используйте **правильные прогнозы ниже** с процентом 60%, чтобы найти какие-либо высказывания в выбранном намерении, которые должны быть исправлены.

### <a name="unclear-or-incorrect-prediction-threshold"></a>Неясный или неправильный порог прогнозирования

Эти два фильтра позволяют находить высказывания в выбранном намерении за пределами порога. Эти два процента можно считать процентами ошибок. Если вы знакомы с 10-15% частота ошибок для прогнозов, установите порог фильтра до 15%, чтобы найти все высказывания выше этого значения.

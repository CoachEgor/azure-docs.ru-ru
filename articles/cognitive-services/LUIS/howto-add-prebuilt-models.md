# <a name="add-prebuilt-models-for-common-usage-scenarios"></a>Добавление заранее построенных моделей для сценариев общего использования 

LUIS включает в себя набор заранее построенных моделей для быстрого добавления общих, разговорных сценариев пользователя. Это быстрый и простой способ добавить способности к разговорному клиенту приложения без разработки моделей для этих способностей. 

[!INCLUDE [Uses preview portal](includes/uses-portal-preview.md)]

## <a name="add-a-prebuilt-domain"></a>Добавление готовой предметной области

1. На странице **Мои приложения** выберите свое приложение. Откроется приложение на разделе **Сборка**. 

1. Выберите **предварительно построенные домены** из левой панели инструментов. 

1. Найдите домен, который вы хотите добавить в приложение, а затем выберите кнопку **«Добавить домен».**

    > [!div class="mx-imgBorder"]
    > ![Добавление предварительно созданного домена календаря](./media/luis-prebuilt-domains/add-prebuilt-domain.png)

## <a name="add-a-prebuilt-intent"></a>Добавление предварительно созданного намерения

1. На странице **Мои приложения** выберите свое приложение. Откроется приложение на разделе **Сборка**. 

1. На странице **Intents** выберите **Добавить заранее построенный домен из** панели инструментов выше списка намерений. 

1. Во всплывающем диалоговом окне выберите намерение **Utilities.Cancel**. 

    > [!div class="mx-imgBorder"]
    > ![Добавление предварительно созданного намерения](./media/luis-prebuilt-domains/add-prebuilt-domain-intents.png)

1. Нажмите кнопку **Готово**.

## <a name="add-a-prebuilt-entity"></a>Добавить предварительно созданную сущность

1. Откройте приложение, щелкнув его имя на странице **Мои приложения**, а затем щелкните **Сущности** в левом углу. 

1. На странице **Сущности** щелкните **Add prebuilt entity** (Добавить предварительно созданную сущность).

1. В **добавлении заранее построенного** диалогового окна, выберите заранее построенную сущность. 

    > [!div class="mx-imgBorder"]
    > ![Диалоговое окно добавления предварительно созданной сущности](./media/luis-prebuilt-domains/add-prebuilt-entity.png)

1. Выберите **готовое**. После добавления сущности не требуется проводить обучение приложения. 

## <a name="publish-to-view-prebuilt-model-from-prediction-endpoint"></a>Опубликовать для просмотра заранее построенной модели с точки зрения прогнозирования

Самый простой способ просмотра значения заранее построенной модели — это запрос с опубликованной конечной точки. 

## <a name="entities-containing-a-prebuilt-entity-token"></a>Сущности, содержащие токен предварительно построенного объекта
 
Если у вас есть объект, обученный машиной, который ограничен заранее построенной сущностью, добавьте подкомпонент к объекту, извлеченному из машины, затем добавьте ограничение заранее построенной сущности.

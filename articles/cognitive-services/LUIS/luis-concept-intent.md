# <a name="intents-in-your-luis-app"></a>Намерения в приложении LUIS

Намерение представляет задачу или действие, которое хочет выполнить пользователь. Это цель или цель, выраженная в [высказывании](luis-concept-utterance.md)пользователя.

Определите набор намерений, которые соответствуют действиям пользователей в вашем приложении. Например, в приложении для путешествий можно определить следующие намерения:

Намерения приложения для путешествий   |   Примеры высказываний   | 
------|------|
 BookFlight     |   "Book me a flight to Rio next week" ("Забронировать авиаперелет в Рио на следующей неделе") <br/> "Fly me to Rio on the 24th" ("Найти авиаперелет в Рио 24-го числа") <br/> "I need a plane ticket next Sunday to Rio de Janeiro" ("Мне нужен билет на самолет в Рио-де-Жанейро на следующее воскресенье")    |
 Greeting     |   "Hi" ("Привет") <br/>"Hello" <br/>"Good morning" ("Доброе утро")  |
 CheckWeather | "What's the weather like in Boston?" ("Какая сейчас погода в Бостоне?") <br/> "Show me the forecast for this weekend" ("Покажите мне прогноз на эту неделю") |
 None         | "Get me a cookie recipe" ("Рецепт печенья")<br>"Did the Lakers win?" ("Лейкерс победили?") |

Все приложения поставляются с предопределенным намерением, "[Нет](#none-intent)", который является замыслом резервного копирования. 

## <a name="prebuilt-domains-provide-intents"></a>Намерения от предварительно созданных доменов
В дополнение к намерениям, которые вы определяете, вы можете использовать заранее построенные намерения из одного из [заранее построенных доменов.](luis-how-to-use-prebuilt-domains.md) 

## <a name="return-all-intents-scores"></a>Получение оценок для всех намерений
Высказывание назначается одному намерению. Когда LUIS получает высказывание на конечную точку, по умолчанию, он возвращает главное намерение для этого высказывания. 

Если для произнесения нужны баллы для всех намерений, можно предоставить флаг строки запроса API прогнозирования. 

|Версия API прогнозирования|Флаг|
|--|--|
|V2|`verbose=true`|
|V3|`show-all-intents=true`|

## <a name="intent-compared-to-entity"></a>Сравнение намерения и сущности
Намерение представляет собой действие, которое приложение должно предпринять для пользователя, и основано на полном высказывании. Высказывание может иметь только одно намерение с максимальной оценкой и может иметь несколько сущностей. 

<a name="how-do-intents-relate-to-entities"></a>

 Создайте намерение в том случае, когда _намерение_ пользователя вызывает выполнения действия в вашем клиентском приложении, например вызов функции checkweather(). Затем создайте сущности для представления параметров, необходимых для выполнения действия. 

|Блокировка с намерением   | Сущность | Пример фразы   | 
|------------------|------------------------------|------------------------------|
| CheckWeather | { "type": "location", "entity": "seattle" }<br>{ "type": "builtin.datetimeV2.date","entity": "tomorrow","resolution":"2018-05-23" } | Какая погода? `Seattle` `tomorrow` |
| CheckWeather | { "type": "date_range", "entity": "this weekend" } | Show me the forecast for (Покажите мне прогноз на) `this weekend` | 
||||

## <a name="prebuilt-domain-intents"></a>Намерения в предварительно созданных доменах

[Заранее построенные домены](luis-how-to-use-prebuilt-domains.md) предоставляют намерения с высказываниями. 

## <a name="none-intent"></a>Намерение None

Намерение **None** создается и оставляется пустым намеренно. Намерение **None** является обязательным и не может быть удалено или переименовано. Заполните его высказываниями, которые не относятся к вашему домену.

**Намерение None** является запором, важным в каждом приложении, и должно иметь 10% от общего числа высказываний. Оно используется для обучения высказываний LUIS, которые не являются важными в домене (предметной области) приложения. Если вы не добавили ни одно высказывание для намерения **None**, LUIS принудительно поместит высказывание, находящееся за пределами домена, в одно из намерений домена. Это приведет к искажению оценок прогнозирования, так как LUIS будет обучен ложному намерению для высказывания. 

Когда высказывание предсказано как намерение None, клиентское приложение может задать больше вопросов или предоставить меню, чтобы направить пользователя к допустимым выборам. 

## <a name="negative-intentions"></a>Отрицательные намерения 
Если вы хотите определить положительные и отрицательные намерения, например, "Я **хочу** автомобиль" и "Я **не хочу** автомобиль", вы можете создать два намерения (одно положительное и одно отрицательное) и добавить соответствующие высказывания для каждого из них. Вы также можете создать одно намерение и пометить положительное и отрицательное понятия как сущности.  

## <a name="intents-and-patterns"></a>Намерения и шаблоны

Если у вас есть примерные высказывания, которые могут быть определены частично или целиком как обычное выражение, рассмотрите возможность использования [обычной сущности выражения](luis-concept-entity-types.md#regular-expression-entity) в паре с [шаблоном.](luis-concept-patterns.md) 

Использование обычной сущности выражения гарантирует извлечение данных таким образом, чтобы шаблон соответствовал. Соответствие шаблона гарантирует возврат точного намерения. 

## <a name="intent-balance"></a>Баланс намерений
Для каждого намерения домена приложения необходимо соблюдать баланс. Не создавайте одно намерение с 10 высказываниями и другое с 500 высказываниями. Это нарушает баланс. При наличии такой ситуации изучите намерение с 500 высказываниями и посмотрите, можно ли заключить большую часть этих высказываний в [шаблон](luis-concept-patterns.md). 

Необходимость поддержания баланса не относится к намерению **None**. Это намерение должно содержать 10 % от общего числа высказываний в приложении.

## <a name="intent-limits"></a>Ограничения намерений
Ознакомьтесь с [ограничениями](luis-boundaries.md#model-boundaries), чтобы понять, какое количество намерений можно добавить в модель. 

### <a name="if-you-need-more-than-the-maximum-number-of-intents"></a>Если требуемое количество намерений превышает максимально допустимое значение 
Во-первых, определите, не используется ли в вашей системе слишком много намерений. 

### <a name="can-multiple-intents-be-combined-into-single-intent-with-entities"></a>Затем определите, можно ли объединить несколько намерений в одно намерение с сущностями 
Если намерения слишком близки, то LUIS может испытывать трудности в том, чтобы различить их. Намерений должно быть достаточно для того, чтобы охватить основные задачи пользователя. При этом не стоит стараться охватить все направления вашего кода. Например, в приложении для путешествий BookFlight и FlightCustomerService могут быть отдельными намерениями, однако намерения BookInternationalFlight и BookDomesticFlight могут быть слишком похожими. Если в вашей системе необходимо их различать, используйте сущности или другую логику вместо намерений. 

### <a name="dispatcher-model"></a>Модель диспетчера
Ознакомьтесь с дополнительными сведениями об объединении приложений LUIS и QnA Maker с помощью [модели диспетчера](luis-concept-enterprise.md#when-you-need-to-combine-several-luis-and-qna-maker-apps). 

### <a name="request-help-for-apps-with-significant-number-of-intents"></a>Если намерений слишком много, обратитесь в службу поддержки
Если уменьшение числа намерений или разделение намерений на несколько приложений не помогает добиться цели, обратитесь в службу поддержки. Если ваша подписка Azure включает службы поддержки, обратитесь в службу [технической поддержки Azure](https://azure.microsoft.com/support/options/).

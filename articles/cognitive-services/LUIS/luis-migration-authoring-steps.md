# <a name="steps-to-migrate-to-the-azure-authoring-resource"></a>Шаги для перехода на ресурс авторов Azure

С портала Language Understanding (LUIS) переносите все приложения, которые у вас есть, для использования авторского ресурса Azure.

## <a name="prerequisites"></a>Предварительные требования

* **Дополнительно,** резервное копирование приложений из списка приложений портала LUIS, экспортируя каждое приложение или используйте [экспортный API.](https://westus.dev.cognitive.microsoft.com/docs/services/5890b47c39e2bb17b84a55ff/operations/5890b47c39e2bb052c5b9c40)
* **Дополнительно**сохраните список сотрудников каждого приложения. Все сотрудники могут быть отправлены по электронной почте в рамках процесса миграции.
* **Требуется,** вы должны иметь [подписку Azure](https://azure.microsoft.com/free/). Часть процесса подписки требует платежной информации. Тем не менее, вы можете использовать бесплатные (F0) уровни ценообразования при использовании LUIS. В конце концов вы можете обнаружить, что вам нужен платный уровень, так как ваше использование увеличивается.

Если у вас нет подписки На Azure, [зарегистрируйтесь.](https://azure.microsoft.com/free/)

## <a name="access-the-migration-process"></a>Доступ к процессу миграции

На еженедельной основе вам предлагается перенести свои приложения. Вы можете отменить это окно без миграции. Если требуется миграция до следующего запланированного периода, можно начать процесс миграции из значка **Azure** на верхней панели инструментов портала LUIS.

> [!div class="mx-imgBorder"]
> ![Значок миграции](./media/migrate-authoring-key/migration-button.png)

## <a name="app-owner-begins-the-migration-process"></a>Владелец приложения начинает процесс миграции

Процесс миграции доступен, если вы являетесь владельцем любых приложений LUIS.

1. Вопием на [портале LUIS](https://www.luis.ai) и соглашайтесь с условиями использования.
1. Всплывающее окно миграции позволяет продолжить миграцию или мигрировать позже. Выберите **Migrate сейчас**. Если вы решите перенестись позже, у вас есть 9 месяцев, чтобы перейти на новый ключ-автор в Azure.

    ![Первое всплывающее окно в процессе миграции, выберите Migrate сейчас.](./media/migrate-authoring-key/migrate-now.png)

1. В обязательном порядке, если в каком-либо из ваших приложений есть сотрудники, вам предлагается **отправить им электронное письмо,** в которых им будет узнать о миграции. Этот параметр является необязательным.

    После того, как вы перенесли свою учетную запись в Azure, ваши приложения больше не будут доступны для сотрудников.

    Для каждого сотрудника и приложения приложение по умолчанию открывается с помощью слегка отформатированных сообщений электронной почты. Вы можете отсвазать письмо перед отправкой.

    Шаблон электронной почты включает в себя точный идентификатор приложения и имя приложения.

    ```html
    Dear Sir/Madam,

    I will be migrating my LUIS account to Azure. Consequently, you will no longer have access to the following app:

    App Id: <app-ID-omitted>
    App name: Human Resources

    Thank you
    ```

1. Выберите для создания ресурса авторов LUIS, выбрав для использования существующего авторского ресурса или для создания нового авторского ресурса.

    > [!div class="mx-imgBorder"]
    > ![Создание ресурса для разработки](./media/migrate-authoring-key/choose-existing-authoring-resource.png)

1. В следующем окне введите ключевую информацию ресурса. После ввода информации выберите **ресурс Create.** Вы можете иметь 10 бесплатных авторских ресурсов в регионе, за подписку.

    ![Создание ресурса для разработки](./media/migrate-authoring-key/choose-authoring-resource-form.png)

    При **создании ресурса для разработки** укажите следующие сведения:

    * **Имя ресурса.** Выбранное пользователем имя, используемое в качестве части URL-адреса для запросов конечной точки разработки и прогнозирования.
    * **Клиент.** Клиент, с которым связана подписка Azure.
    * **Имя подписки.** Подписка, для которой будет выставлен счет за ресурс.
    * **Группа ресурсов.** Выбранное или созданное пользователем имя группы ресурсов. Группы ресурсов позволяют группировать ресурсы Azure для осуществления доступа и управления.
    * **Расположение.** Выбор расположения зависит от выбранной **группы ресурсов**.
    * **Ценовая категория.** Ценовая категория определяет максимальное количество транзакций в секунду и месяц.

1. Проверка авторингового ресурса и **мигрируйте сейчас.**

    ![Создание ресурса для разработки](./media/migrate-authoring-key/choose-authoring-resource-and-migrate.png)

1. При создании авторского ресурса отображается сообщение об успехе. Выберите **Близко,** чтобы закрыть всплывающее окно.

    ![Успешно создан ваш авторский ресурс.](./media/migrate-authoring-key/migration-success.png)

    В списке **«Мои приложения»** показаны приложения, перекочевавающие на новый авторский ресурс.

    Вам не нужно знать ключ авторского ресурса, чтобы продолжить редактирование приложений на портале LUIS. Если вы планируете программировать свои приложения, вам нужны ключевые значения. Эти значения отображаются на странице **ресурсов Управления-> Azure** на портале LUIS, а также доступны на портале Azure на странице **ключей** ресурса.

1. Прежде чем получить доступ к приложениям, выберите ресурс подписки и авторов LUIS, чтобы просмотреть приложения, которые вы можете авторов.

    ![Выберите ресурс для авторов подписки и АВТОРов LUIS, чтобы увидеть приложения, которые можно авторовать.](./media/migrate-authoring-key/app-list-by-subscription-and-resource.png)


## <a name="app-contributor-begins-the-migration-process"></a>Автор приложения начинает процесс миграции

Выполните те же шаги, что и владелец приложения для миграции. Процесс создает новый авторский `LUIS.Authoring`ресурс рода.

Необходимо перенести свою учетную запись, чтобы быть добавленным в качестве участника в мгновенные приложения, принадлежащие другим пользователям.

## <a name="troubleshooting-errors-with-the-migration-process"></a>Ошибки устранения неполадок в процессе миграции

Если вы `MissingSubscriptionRegistration` получили ошибку на портале LUIS с красной панелью уведомлений во время процесса миграции, создайте ресурс Когнитивных Служб на [портале Azure](luis-how-to-azure-subscription.md#create-resources-in-the-azure-portal) или [Azure CLI.](luis-how-to-azure-subscription.md#create-resources-in-azure-cli) Подробнее о [причинах этой ошибки.](../../azure-resource-manager/templates/error-register-resource-provider.md#cause)

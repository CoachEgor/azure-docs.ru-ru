# <a name="content-moderation-reviews-workflows-and-jobs"></a>Обзоры модерации содержимого, рабочие процессы и задания

Content Moderator объединяет модерацию с возможностями человека в цикл для создания оптимальной Модерация процесса для реальных сценариев. Это осуществляется с помощью облачной [средство проверки](https://contentmoderator.cognitive.microsoft.com). В этом руководстве вы узнаете об основных концепциях из средства проверки: проверки, рабочие процессы и задания.

## <a name="reviews"></a>Проверки

При проверке, содержимое загружается в средстве проверки и появится в разделе **просмотрите** вкладки. На этой странице пользователи можно изменить примените теги и применять собственные пользовательские теги, соответствующим образом. Когда пользователь отправляет отзыв, результаты отправляются в конечную точку заданный обратный вызов, и содержимое удаляется с сайта.

![Откройте веб-сайта средство проверки в браузере, на вкладке "Обзор"](./Review-Tool-user-Guide/images/image-workflow-review.png)

См. в разделе [руководство по средству проверки](./review-tool-user-guide/review-moderated-images.md) приступить к созданию обзоры, или см. в разделе [руководство по REST API](./try-review-api-review.md) для Узнайте, как сделать это программным способом.

## <a name="workflows"></a>Рабочие процессы

Рабочий процесс — облачных настраиваемый фильтр для содержимого. Рабочие процессы могут подключаться к различным службам для фильтрации содержимого по-разному и предпринять соответствующие действия. С соединителем Content Moderator рабочий процесс можно автоматически применять теги "умеренность" и создайте проверки с содержимое.

### <a name="view-workflows"></a>Просмотр процессов

Для просмотра с существующими рабочими процессами, перейдите к [средство проверки](https://contentmoderator.cognitive.microsoft.com/) и выберите **параметры** > **рабочие процессы**.

![Рабочий процесс по умолчанию](images/default-workflow-listed.PNG)

Рабочие процессы можно полностью описать как строк JSON, что делает их доступными для программным способом. При выборе **изменить** параметр для рабочего процесса, а затем выберите **JSON** вкладку, вы увидите выражении JSON следующим образом:

```json
{
    "Type": "Logic",
    "If": {
        "ConnectorName": "moderator",
        "OutputName": "isAdult",
        "Operator": "eq",
        "Value": "true",
        "Type": "Condition"
        },
    "Then": {
    "Perform": [
    {
        "Name": "createreview",
        "CallbackEndpoint": null,
        "Tags": []
    }
    ],
    "Type": "Actions"
    }
}
```

См. в разделе [руководство по средству проверки](./review-tool-user-guide/workflows.md) приступите к созданию и использование рабочих процессов, или см. в разделе [руководство по REST API](./try-review-api-workflow.md) для Узнайте, как сделать это программным способом.

## <a name="jobs"></a>Задания

Задание "умеренность" выступает в качестве оболочки для модерации содержимого, рабочие процессы и проверки функциональности. Задание сканирования содержимого с помощью Модерация изображений Content Moderator, API или API модерации текста и затем проверяет ее относительно указанного рабочего процесса. На основании результатов рабочего процесса он может или не может создать Обзор для содержимого в [средство проверки](./review-tool-user-guide/human-in-the-loop.md). Во время проверки и рабочие процессы можно создать и настроить с их соответствующими интерфейсами API, задание API позволяет получить подробный отчет о весь процесс (который можно отправить в конечную точку заданный обратный вызов).

См. в разделе [руководство по REST API](./try-review-api-job.md) приступить к использованию заданий.

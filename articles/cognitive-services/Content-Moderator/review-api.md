# <a name="content-moderation-reviews-workflows-and-jobs"></a>Обзоры модерации содержимого, рабочие процессы и рабочие места

Модератор содержимого сочетает в себе режимную помощь с помощью машины с возможностями «человека в петле» для создания оптимального процесса модерации для реальных сценариев. Он делает это с помощью облачного [инструмента обзора](https://contentmoderator.cognitive.microsoft.com). В этом руководстве вы узнаете об основных концепциях инструмента Обзора: обзоры, рабочие процессы и рабочие места.

## <a name="reviews"></a>Проверки

В обзоре содержимое загружается в инструмент «Обзор» и отображается под вкладкой **«Обзор».** Отсюда пользователи могут изменять прикладные теги и применять свои собственные пользовательские теги по мере необходимости. Когда пользователь отправляет отзыв, результаты отправляются в указанную конечную точку обратного вызова, и содержимое удаляется с сайта.

![Обзор инструмент веб-сайт открыт в браузере, на вкладке Обзор](./Review-Tool-user-Guide/images/image-workflow-review.png)

Ознакомьтесь с [руководством по инструменту «Обзор»,](./review-tool-user-guide/review-moderated-images.md) чтобы начать создавать отзывы, или посмотрите [руководство REST API,](./try-review-api-review.md) чтобы узнать, как это сделать программно.

## <a name="workflows"></a>Рабочие процессы

Рабочий процесс — это облачный настраиваемый фильтр для содержимого. Рабочие процессы могут подключаться к различным службам для фильтрации содержимого различными способами, а затем принять соответствующие меры. С разъемом модератора содержимого рабочий процесс может автоматически применять теги модерации и создавать отзывы с представленным контентом.

### <a name="view-workflows"></a>Просмотр рабочих процессов

Для просмотра существующих рабочих процессов перейдите в [инструмент «Обзор»](https://contentmoderator.cognitive.microsoft.com/) и выберите**рабочие процессы** **настроек.** > 

![Рабочий процесс по умолчанию](images/default-workflow-listed.PNG)

Рабочие процессы можно полностью охарактеризовать как строки JSON, что делает их доступными программно. Если вы выберете опцию **«Отобрачье»** для рабочего процесса, а затем выберите вкладку **JSON,** вы увидите выражение JSON следующее:

```json
{
    "Type": "Logic",
    "If": {
        "ConnectorName": "moderator",
        "OutputName": "isAdult",
        "Operator": "eq",
        "Value": "true",
        "Type": "Condition"
        },
    "Then": {
    "Perform": [
    {
        "Name": "createreview",
        "CallbackEndpoint": null,
        "Tags": []
    }
    ],
    "Type": "Actions"
    }
}
```

Ознакомьтесь с [руководством по использованию инструментов «Обзор»,](./review-tool-user-guide/workflows.md) чтобы начать создавать рабочие процессы, или смотрите [руководство REST API,](./try-review-api-workflow.md) чтобы узнать, как это сделать программно.

## <a name="jobs"></a>Задания

Задание модерации служит своего рода оберткой для функциональности модерации содержимого, рабочих процессов и обзоров. Задание сканирует содержимое с помощью API модерации изображения Модератора содержимого или API модерации текста, а затем проверяет его на указанном рабочем процессе. На основе результатов рабочего процесса он может или не может создать рецензию на содержимое [в инструменте «Обзор».](./review-tool-user-guide/human-in-the-loop.md) В то время как обзоры и рабочие процессы могут быть созданы и настроены с их соответствующими API, API задания позволяет получить подробный отчет всего процесса (который может быть отправлен в указанную конечную точку обратного вызова).

Смотрите [руководство REST API,](./try-review-api-job.md) чтобы начать использовать рабочие места.

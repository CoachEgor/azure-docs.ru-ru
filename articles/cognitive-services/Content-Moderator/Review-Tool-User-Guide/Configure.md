# <a name="configure-the-review-tool"></a>Настройка средства проверки

[Инструмент «Обзор»](https://contentmoderator.cognitive.microsoft.com) имеет несколько важных функций, к которым можно получить доступ через меню **«Настройки»** на панели мониторинга.

![Содержимое Модератор Обзор слишком настройки меню](images/settings-1.png)

## <a name="manage-team-and-subteams"></a>Управление командой и подгруппами

Вкладка **Team** позволяет управлять группами&mdash;и подгруппами пользователей, которые могут быть уведомлены при начале определенных [отзывов людей.](../review-api.md#reviews) Вы можете иметь только одну команду (которую вы создаете, когда вы зарегистрируетесь с инструментом обзор), но вы можете создать несколько подгрупп. Администратор группы может приглашать участников, устанавливать их разрешения и назначать их различным подгруппам.

![Просмотр настроек группы инструментов](images/settings-2-team.png)

Подкоманды удобно использовать для создания команд эскалации или отдельных команд для проверки содержимого определенных категорий. Например, можно отправить содержимое для взрослых в отдельную группу для дальнейшего рассмотрения.

В этом разделе объясняется, как создавать подгруппы и быстро назначать отзывы на лету. Тем не менее, можно использовать вкладку [Workflows](workflows.md) (Рабочие процессы) для назначения проверок на основе заданных критериев.

### <a name="create-a-subteam"></a>Создание подкоманды

Перейти к разделу **Subteams** и нажмите **Добавить подгруппы**. Введите имя подгруппы в диалоге и нажмите **Сохранить**.

![Имя подкоманды](images/1-Teams-2.PNG)

#### <a name="invite-teammates"></a>Пригласить товарищей по команде

Вы не можете назначить кого-либо в подгруппу, если он еще не является членом группы по умолчанию, поэтому сначала необходимо добавить рецензентов в команду по умолчанию. Нажмите **«Пригласить»** на вкладку **Команды.**

![Приглашение пользователей](images/invite-users.png)

#### <a name="assign-teammates-to-subteam"></a>Назначить товарищей по команде подкоманду

Нажмите кнопку **Добавить участника,** чтобы назначить членов из вашей команды по умолчанию в одну или несколько подкоманд. В подкоманду можно добавить только существующих пользователей. Чтобы добавить новых пользователей, не зарегистрированных в инструменте проверки, пригласите их с помощью кнопки "Invite" (Пригласить) на странице "Team Setting" (Параметры команды).

![Назначение участников подкоманды](images/1-Teams-3.PNG)

### <a name="assign-reviews-to-subteams"></a>Назначить отзывы подкомандам

После создания подгрупп и назначенных членов можно приступить к назначению [отзывов](../review-api.md#reviews) о содержимом этим подгруппам. Это делается из вкладки **Обзор** сайта.
Чтобы назначить содержимое подгруппе, щелкните эллипсис в правом верхнем углу, выберите **Move to**и выберите подгруппу.

![Назначение подкоманде проверки изображений](images/3-review-image-subteam-1.png)

### <a name="switch-between-subteams"></a>Переключение между подкомандами

Если вы являетесь членом нескольких подгрупп, вы можете переключаться между этими подгруппами, чтобы изменить, какие отзывы о содержимом отображаются для вас. Во вкладке **«Обзор»** выберите выпадающее меню с по **умолчанию** и выберите **подгруппу «Выбрать».** Вы можете просматривать отзывы о содержании для различных подгрупп, но только те из которых ваш являетесь членом.

![Переключение между подкомандами](images/3-review-image-subteam-2.png)

## <a name="tags"></a>Теги

Вкладка **Теги** позволяет определить пользовательские теги модерации в&mdash;дополнение к двум по умолчанию модерации теги**isadult** **(a)** и **isracy** **(r**). При создании пользовательского тега он становится доступен в обзорах вместе с тегами по умолчанию. Вы можете изменить, какие теги отображаются в обзорах, переключая настройки видимости.

![Представление тегов, включая флажки "Видимые"](images/tags-4-disable.png)

### <a name="create-custom-tags"></a>Создание пользовательских тегов

Чтобы создать новый тег, необходимо ввести короткий код, имя и описание в соответствующих полях.

- **Короткий код**: Введите двухбуквейный код для тега. Пример: **cb**
- **Имя**: Введите короткое и описательное имя тега в нижнем регистре без пробелов. Пример: **издевательство**.
- **Описание**: (необязательно) Введите описание содержимого, на которое нацелен ваш тег. Пример: **Изображения или случаи кибер-издевательств.**

Нажмите **Добавить,** чтобы добавить тег, и нажмите **Сохранить,** когда вы закончите создавать теги.

![Инструмент просмотра создаст новый диалог тегов](images/settings-3-tags.png)

### <a name="delete-tags"></a>Удаление тегов

Вы можете удалить пользовательские теги, выбрав значок мусора рядом с их записями в списке тегов, но вы не можете удалить теги по умолчанию.

## <a name="connectors"></a>Соединители

Вкладка **Connectors** позволяет управлять разъемами, которые являются плагинами, которые могут обрабатывать содержимое различными способами в рамках [рабочих процессов](../review-api.md#workflows)содержимого.

Разъем по умолчанию при создании рабочего процесса является разъем модератором содержимого, который может отмечать содержимое как **взрослый** или **колоритный,** найти ненормативную лексику, и так далее. Тем не менее, вы можете использовать другие разъемы, перечисленные здесь, до тех пор, пока у вас есть учетные данные для их соответствующих услуг (для использования разъема Face, например, вам нужно будет получить ключ подписки [Face).](https://docs.microsoft.com/azure/cognitive-services/face/overview)

[Инструмент Обзора](./human-in-the-loop.md) включает в себя следующие разъемы:

- API распознавания эмоций
- Распознавание лиц
- облачная служба PhotoDNA.
- Анализ текста

### <a name="add-a-connector"></a>Добавление соединителя

Чтобы добавить разъем (и сделать его доступным для использования в [рабочих процессах](../review-api.md#workflows)содержимого), выберите соответствующую **кнопку Connect.** В следующем диалоге введите ключ подписки для этой службы. Когда вы закончите, ваш новый разъем должен появиться в верхней части страницы.

![Параметры соединители Content Moderator](images/settings-4-connectors.png)

## <a name="workflows"></a>Рабочие процессы

Вкладка **рабочих потоков** позволяет управлять [рабочими процессами.](../review-api.md#workflows) Рабочие процессы являются облачными фильтрами для содержимого, и они работают с разъемами для различной сортировки содержимого и принятия соответствующих мер. Здесь можно определить, отсеить и протестировать рабочие процессы. [См. Определить и использовать рабочие процессы](Workflows.md) для руководства о том, как это сделать.

![Параметры рабочего процесса Content Moderator](images/settings-5-workflows.png)

## <a name="credentials"></a>Учетные данные

Вкладка **Credentials** обеспечивает быстрый доступ к ключу подписки Модератора Содержимого, который вам необходимо будет получить доступ к любой из служб модерации с помощью вызова REST или клиента SDK.

![Учетные данные Content Moderator](images/settings-6-credentials.png)

### <a name="use-external-credentials-for-workflows"></a>Использование внешних учетных данных для рабочих процессов

[Инструмент «Обзор»](https://contentmoderator.cognitive.microsoft.com) генерирует бесплатный пробный ключ для служб модератора содержимого Azure при регистрации, но также можно настроить его на использование существующего ключа из учетной записи Azure. Это рекомендуется для крупномасштабных сценариев, так как бесплатные пробные ключи имеют строгие ограничения использования[(Цены и ограничения).](https://azure.microsoft.com/pricing/details/cognitive-services/content-moderator/)

Если вы создали [ресурс модератора содержимого](https://ms.portal.azure.com/#create/Microsoft.CognitiveServicesContentModerator) в Azure, перейдите к нему на портале Azure и выберите лезвие **ключей.** Копируйте один из ключей.

![Ключи Content Moderator на портале Azure](images/credentials-azure-portal-keys.PNG)

Во вкладке **«Полномочия»** [инструмента «Обзор»](https://contentmoderator.cognitive.microsoft.com)перейдите на панель **настроек рабочего процесса,** выберите **«Исключите»** и вставьте ключ в поле **Ocp-Apim-Subscription-Key.** Теперь рабочие процессы, вызывающие AIS модерации, будут использовать учетные данные Azure.

> [!NOTE]
> Два других поля в панели **настроек рабочего процесса** предназначены для пользовательских списков терминов и изображений. Ознакомьтесь с пользовательскими [терминами](../try-terms-list-api.md) или пользовательскими [изображениями,](../try-image-list-api.md) чтобы узнать об этом.

### <a name="use-your-azure-account-with-the-review-apis"></a>Используйте свою учетную запись Azure с помощью AIS обзора

Чтобы использовать ключ Azure с aIs отзыва, необходимо получить идентификатор ресурса. Перейдите на ресурс Модератора содержимого на портале Azure и выберите лезвие **свойств.** Копируйте значение идентификатора ресурса и вставьте его в поле **Whitelisted Resource Id (ы)** **вкладки «Полномочия» инструмента «Обзор».**

![Идентификатор ресурса Content Moderator на портале Azure](images/credentials-azure-portal-resourceid.PNG)

Если вы ввели ключ подписки в обоих местах, пробный ключ, который поставляется с учетной записью инструмента Review, не будет использоваться, но останется доступным.

# <a name="define-and-use-moderation-workflows"></a>Определение и использование рабочих процессов модерации

В этом руководстве вы узнаете, как настроить и использовать [рабочие процессы](../review-api.md#workflows) на веб-сайте [инструмента Обзор.](https://contentmoderator.cognitive.microsoft.com) Рабочие процессы — это облачные индивидуальные фильтры, которые можно использовать для более эффективной обработки содержимого. Рабочие процессы могут подключаться к различным службам для фильтрации содержимого различными способами, а затем принять соответствующие меры. В этом руководстве показано, как использовать разъем модератора содержимого (который включен по умолчанию) для фильтрации содержимого и настройки отзывов людей в типичном сценарии модерации.

## <a name="create-a-new-workflow"></a>Создание нового рабочего процесса

Перейдите к [инструменту «Обзор модератора контента»](https://contentmoderator.cognitive.microsoft.com/) и войдите в систему. На вкладке **Settings** (Параметры) выберите **Workflows** (Рабочие процессы).

![Настройка рабочих процессов](images/2-workflows-0.png)

На следующем экране выберите **Добавить рабочий процесс.**

![Добавление рабочего процесса](images/2-workflows-1.png)

### <a name="assign-a-name-and-description"></a>Задание имени и описания

Назовите свой рабочий процесс, введите описание и выберите, будет ли рабочий процесс обрабатывать изображения или текст.

![Имя и описание рабочего процесса](images/image-workflow-create.PNG)

### <a name="define-evaluation-criteria"></a>Определение критериев оценки

На следующем экране перейдите в раздел **"Если".** В верхнем меню выпадения выберите **Состояние**. Это позволит настроить состояние, при котором рабочий процесс будет действовать. Если вы хотите использовать несколько условий, выберите **комбинацию** вместо этого. 

Затем выберите разъем. В этом примере используется **Content Moderator**. В зависимости от выбора разъема вы получите различные варианты вывода данных. Ознакомьтесь с [разделом «Соединитель»](./configure.md#connectors) в руководстве по настройкам инструментов «Обзор», чтобы узнать, как настроить другие разъемы.

![Выберите разъем рабочего процесса](images/image-workflow-connect-to.PNG)

Выберите нужный выход для использования и установите условия, чтобы проверить его против.

![Определение условия рабочего процесса](images/image-workflow-condition.PNG)

### <a name="define-the-action"></a>Определить действие

Перейдите в раздел **«Тогда»,** где вы выбираете действие. Следующий пример создает обзор изображений и назначает тег. Дополнительно можно добавить альтернативный (else) путь и установить для этого также действие.

![Определение действия рабочего процесса](images/image-workflow-action.PNG)

### <a name="save-the-workflow"></a>Сохранить рабочий процесс

Обратите внимание на имя рабочего процесса; для запуска задания модерации с помощью API рабочего процесса (см. ниже). Наконец, сохраните рабочий процесс с помощью кнопки **«Сохранить»** в верхней части страницы.

## <a name="test-the-workflow"></a>Проверка рабочего процесса

Теперь, когда вы определили пользовательский рабочий процесс, проверьте его с содержанием образца. Перейдите к **рабочим процессам** и выберите соответствующую кнопку **выполнения рабочего процесса.**

![Проверка рабочего процесса](images/image-workflow-execute.PNG)

Сохраните этот [образец изображения](https://moderatorsampleimages.blob.core.windows.net/samples/sample2.jpg) на локальном диске. Затем выберите **файл (ы)** и загрузите изображение в рабочий процесс.

![Бегун с цитатой, наложенной на изображение](images/sample-text.jpg)

### <a name="track-progress"></a>Отслеживание хода выполнения

Можно просмотреть ход работы в следующем всплывающем окне.

![Отслеживание выполнения рабочего процесса](images/image-workflow-job.PNG)

### <a name="verify-workflow-action"></a>Проверка действия рабочего процесса

Перейдите на вкладку **«Изображение»** **в** обзоре и убедитесь, что есть недавно созданный обзор изображений.

![Проверка образов](images/image-workflow-review.PNG)

Далее смотрите [руководство REST API,](../try-review-api-workflow.md) чтобы узнать, как создавать рабочие процессы программно.

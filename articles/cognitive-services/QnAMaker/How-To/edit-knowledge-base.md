# <a name="edit-qna-pairs-in-your-knowledge-base"></a>Элит пар Зна в вашей базе знаний

QnA Maker позволяет управлять содержимым базы знаний, предоставляя возможности удобного редактирования.

Пары NA добавляются из источника данных, например файла или URL, или добавляются в качестве редакционного источника. Редакционный источник указывает, что пара ЗНА была добавлена на портале зна вручную. Все пары QnA доступны для редактирования.

## <a name="add-an-editorial-qna-pair"></a>Добавить редакционную пару NA
1. Вопийте на [портале ЗНА,](https://www.qnamaker.ai/)затем выберите базу знаний, чтобы добавить пару nA.
1. На странице **EDIT** базы знаний выберите **пару Добавить пара qnA,** чтобы добавить новую пару nA.

1. В новой парной строке nA добавьте необходимые поля **вопросов** и **ответов.** Другие поля не являются обязательными. Все поля могут быть изменены в любое время.

1. Дополнительно добавьте **альтернативную формулировку.** Альтернативная формулировка — это любая форма вопроса, которая существенно отличается от исходного вопроса, но должна дать тот же ответ.

    Когда ваша база знаний опубликована, и у вас есть [активное обучение](use-active-learning.md) включено, NA Maker собирает альтернативные варианты фразировки для вас принять. Эти варианты выбираются для повышения точности прогнозирования.

1. Дополнительно добавьте **метаданные**. Для просмотра метаданных выберите **параметры просмотра** в контекстном меню. Metadata предоставляет фильтры для ответов, которые предоставляет клиентское приложение, например чат-бот.

1. Дополнительно добавьте **последующие запросы.** Последующие запросы предоставляют дополнительные пути разговора клиентскому приложению, чтобы представить пользователю.

1. Выберите **Сохранить и поезд,** чтобы увидеть прогнозы, включая новую пару NA.

## <a name="edit-a-qna-pair"></a>Изналиваю пару ЗНА

Любое поле в любой паре ЗНА может быть отредактировано, независимо от исходного источника данных. Некоторые поля могут быть не видны из-за текущих настроек **параметры представления,** найденных в панели инструментов контекста.

## <a name="delete-a-qna-pair"></a>Удаление пары QnA

Чтобы удалить пару QnA, щелкните значок **Удалить** справа в конце строки QnA. Это необратимая операция. Эту операцию невозможно отменить. Рассмотрите возможность экспорта КБ со страницы **«Публикация»** перед удалять наборы.

![Удаление пары QnA](../media/qnamaker-how-to-edit-kb/delete-qnapair.png)

## <a name="find-the-qna-pair-id"></a>Найти идентификатор пары Зна

Если вам нужно найти идентификатор пары nA, вы можете найти его в двух местах:

* Нависшие на значке удаления на строке пары ЗНА, которая вас интересует. Текст нависки включает идентификатор пары nA.
* Экспорт базы знаний. Каждая пара ЗНА в базе знаний включает идентификатор пары nA.

## <a name="add-alternate-questions"></a>Добавление альтернативных вопросов

Добавьте альтернативные вопросы к существующей паре QnA, чтобы повысить вероятность совпадения с запросом пользователя.

![Добавление альтернативных вопросов](../media/qnamaker-how-to-edit-kb/add-alternate-question.png)

## <a name="linking-qna-pairs"></a>Связывание пар ЗНА

Связывание пар NA предоставляется с [последующими подсказками.](multiturn-conversation.md) Это логическая связь между парами зна, управляемыми на уровне базы знаний. Вы можете отображать последующие запросы на портале «NA Maker».

Вы не можете связать пары nA в метаданных ответа.

## <a name="add-metadata"></a>Добавление метаданных

Добавьте наборы метаданных, сначала выбрав **параметры «Вид»,** а затем выбрав **метаданные Show.** Это отображает столбец метаданных. Затем выберите **+** знак, чтобы добавить набор метаданных. Этот набор состоит из одного ключа и одного значения.

## <a name="save-changes-to-the-qna-pairs"></a>Сохранение изменений в парах ЗНА

Периодически выбирайте **Сохранить и тренироваться** после внесения изменений, чтобы избежать потери изменений.

![Добавление метаданных](../media/qnamaker-how-to-edit-kb/add-metadata.png)

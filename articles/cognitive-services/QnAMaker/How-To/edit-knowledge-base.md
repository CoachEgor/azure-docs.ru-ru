# <a name="edit-qna-sets-in-your-knowledge-base"></a>Элит Зна устанавливает в вашей базе знаний

QnA Maker позволяет управлять содержимым базы знаний, предоставляя возможности удобного редактирования.

Наборы nA добавляются из источника данных, например файла или URL, или добавляются в качестве редакционного источника. Редакционный источник указывает, что набор qnA был добавлен на портале зна вручную. Все наборы qnA доступны для редактирования.

## <a name="add-an-editorial-qna-set"></a>Добавить редакционный набор NA
1. Вопийте на [портале ЗНА,](https://www.qnamaker.ai/)затем выберите базу знаний, чтобы добавить набор в QnA.
1. На странице **EDIT** базы знаний выберите **набор Добавить набор для** добавления нового набора nA.

1. В новой строке набора qnA добавьте необходимые поля **вопросов** и **ответов.** Другие поля не являются обязательными. Все поля могут быть изменены в любое время.

1. Дополнительно добавьте **альтернативную формулировку.** Альтернативная формулировка — это любая форма вопроса, которая существенно отличается от исходного вопроса, но должна дать тот же ответ.

    Когда ваша база знаний опубликована, и у вас есть активное обучение включено, NA Maker собирает альтернативные варианты фразировки для вас принять. Эти варианты выбираются для повышения точности прогнозирования.

1. Дополнительно добавьте **метаданные**. Для просмотра метаданных выберите **параметры просмотра** в контекстном меню. Metadata предоставляет фильтры для ответов, которые предоставляет клиентское приложение, например чат-бот.

1. Дополнительно добавьте **последующие запросы.** Последующие запросы предоставляют дополнительные пути разговора клиентскому приложению, чтобы представить пользователю.

1. Выберите **Сохранить и поезд,** чтобы увидеть прогнозы, включая новый набор NA.

## <a name="edit-a-qna-set"></a>Отредите набор NA

Любое поле в любом наборе NA может быть отредактировано, независимо от исходного источника данных. Некоторые поля могут быть не видны из-за текущих настроек **параметры представления,** найденных в панели инструментов контекста.

## <a name="delete-a-qna-set"></a>Удаление набора NA

Чтобы удалить пару QnA, щелкните значок **Удалить** справа в конце строки QnA. Это необратимая операция. Эту операцию невозможно отменить. Рассмотрите возможность экспорта КБ со страницы **«Публикация»** перед удалять наборы.

![Удаление набора Зна](../media/qnamaker-how-to-edit-kb/delete-qnapair.png)

## <a name="find-the-qna-set-id"></a>Поиск идентификатора набора nA

Если вам нужно найти идентификатор набора nA, вы можете найти его в двух местах:

* Нависшие на значке удаления на строке, установленной на nA, которая вас интересует. Текст нависки включает идентификатор набора nA.
* Экспорт базы знаний. Каждый набор NA, включающий идентификатор набора nA.

## <a name="add-alternate-questions"></a>Добавление альтернативных вопросов

Добавьте альтернативные вопросы в существующий набор qnA, чтобы повысить вероятность совпадения пользовательского запроса.

![Добавление альтернативных вопросов](../media/qnamaker-how-to-edit-kb/add-alternate-question.png)

## <a name="linking-qna-sets"></a>Связывание наборов NA

Ссылка на наборы nA предоставляется с [последующими подсказками.](multiturn-conversation.md) Это логическая связь между наборами qnA, управляемыми на уровне базы знаний. Вы можете отображать последующие запросы на портале «NA Maker».

Вы не можете связать наборы qnA в метаданных ответа.

## <a name="add-metadata"></a>Добавление метаданных

Добавьте наборы метаданных, сначала выбрав **параметры «Вид»,** а затем выбрав **метаданные Show.** Это отображает столбец метаданных. Затем выберите **+** знак, чтобы добавить набор метаданных. Этот набор состоит из одного ключа и одного значения.

## <a name="save-changes-to-the-qna-sets"></a>Сохранение изменений в наборах QnA

Периодически выбирайте **Сохранить и тренироваться** после внесения изменений, чтобы избежать потери изменений.

![Добавление метаданных](../media/qnamaker-how-to-edit-kb/add-metadata.png)

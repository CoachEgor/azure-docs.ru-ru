# <a name="language-support-for-qna-maker"></a>Языковая поддержка для QnA Maker

Язык базы знаний влияет на способность службы QnA Maker автоматически извлекать вопросы и ответы из [источников](../Concepts/data-sources-supported.md), а также на релевантность результатов, предоставляемых службой QnA Maker в ответ на запросы пользователей.

## <a name="auto-extraction"></a>Автоматическое извлечение
QnA Maker поддерживает извлечение вопросов и ответов из страницы на любом языке. Однако эффективность извлечения намного выше для указанных далее языков, так как QnA Maker использует ключевые слова для определения вопросов.

|Поддерживаемые языки| Языковые стандарты|
|-----|----|
|Английский|en-*|
|Французский|fr-*|
|Итальянский|it-*|
|Немецкий|de-*|
|Испанский|es-*|

## <a name="primary-language-detection"></a>Определение основного языка

Основной язык, используемый для обнаружения, задается для ресурса QnA Maker и всех баз знаний, созданных в этом ресурсе при добавлении первого документа или URL-адреса в первую базу знаний. Невозможно изменить язык. 

Если пользователь планирует поддерживать несколько языков, он должен иметь новый ресурс QnA Maker для каждого языка. Узнайте, как [создать базу знаний на основе языка QnA Maker](../how-to/language-knowledge-base.md).  

Проверьте основной язык, выполнив следующие действия.

1. Войдите на [портале Azure](https://portal.azure.com).  
1. Найдите и выберите ресурс службы "Поиск Azure", созданный в рамках ресурса QnA Maker. Имя ресурса службы поиска Azure будет начинаться с того же имени, что и ресурс QnA Maker и будет иметь **службу поиска**типов. Помните, что только один ресурс QnA Maker можно связать с одним ресурсом поиска Azure.
1. На странице **Обзор** поискового ресурса выберите **индексы**. 
1. Выберите индекс **testkb**.
1. Перейдите на вкладку **поля** . 
1. Просмотрите столбец **анализатора** для полей **вопросов** и **ответов** . 


## <a name="query-matching-and-relevance"></a>Сопоставление запросов и релевантность
Служба QnA Maker зависит от [анализаторов языков](https://docs.microsoft.com/rest/api/searchservice/language-support) в службе "Поиск Azure" при предоставлении результатов. Для языков en-* доступны специальные функции повторного ранжирования, позволяющие улучшить релевантность.

Хотя возможности службы "Поиск Azure" работают одинаково для поддерживаемых языков, в QnA Maker предусмотрено дополнительное средство ранжирования, расположенное над результатами поиска Azure. В этой модели ранжирования мы используем некоторые специальные функции на основе семантики и слов в EN-*, которые еще не доступны для других языков. Мы не предоставляем эти функции, так как они являются частью внутренней работы ранжирования QnA Maker. 

QnA Maker [автоматически определяет язык базы знаний](#primary-language-detection) во время создания и соответствующим образом устанавливает анализатор. Базы знаний можно создавать на указанных ниже языках.

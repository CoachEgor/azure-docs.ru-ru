# <a name="active-learning-suggestions"></a>Активные предложения по обучению

Функция _Активных предложений обучения_ позволяет улучшить качество вашей базы знаний, предлагая альтернативные вопросы, основанные на представлениях пользователей, к вашей паре вопросов и ответов. Проверенные предложения можно добавить к существующим вопросам или отклонить.

База знаний не изменяется автоматически. Для того, чтобы любое изменение вступило в силу, вы должны принять предложения. Так вы добавите новые вопросы, не меняя и не удаляя существующие.

## <a name="what-is-active-learning"></a>Что такое активное обучение?

QnA Maker изучает новые варианты вопросов с помощью механизма явных и неявных отзывов.

* [Неявная обратная связь](#how-qna-makers-implicit-feedback-works) - Ranker понимает, когда пользовательский вопрос имеет несколько ответов с оценками, которые очень близки и рассматривает это как обратную связь. Вам не нужно ничего делать для того, чтобы это произошло.
* [Явная обратная связь](#how-you-give-explicit-feedback-with-the-train-api) - Когда несколько ответов с небольшими вариациями в оценках возвращаются из базы знаний, клиентское приложение задает пользователю, какой вопрос является правильным вопросом. Явная обратная связь пользователя отправляется в SnA Maker с [помощью API Train API.](../How-to/improve-knowledge-base.md#train-api)

Оба метода предоставляют рангеру схожие запросы, которые группируются.

## <a name="how-active-learning-works"></a>Принцип работы активного обучения

Активное обучение срабатывает на основе нескольких лучших ответов, возвращенных NA Maker. Если различия в оценках между парами, которые соответствуют парам запроса, находятся в небольшом диапазоне, то запрос считается возможным предложением (как альтернативный вопрос) для каждой из возможных пар NA. Как только вы принимаете предложенный вопрос для определенной пары, он отклоняется для других пар. Вы должны помнить, чтобы сохранить и поезд, после принятия предложений.

Активное обучение предоставляет наилучшие возможные предложения, если конечные точки получают приемлемое количество разных запросов использования. При кластеризации 5 или более подобных запросов каждые 30 минут компания «NA Maker» предлагает пользовательскому запросу дизайнеру базы знаний принять или отклонить их. Все предложения группируются по сходству, и лучшие предложения для альтернативных вопросов отображаются с учетом частоты конкретных запросов пользователей.

После того, как вопросы будут предложены на портале «NA Maker», необходимо рассмотреть и принять или отклонить эти предложения. API не может управлять предложениями.

## <a name="turn-on-active-learning"></a>Включение активного обучения

По умолчанию активное обучение **отключено.**
Для активного обучения:
* Вам нужно [включить активное обучение,](../How-To/use-active-learning.md#turn-on-active-learning-for-alternate-questions) чтобы компания «NA Maker» собирала альтернативные вопросы для вашей базы знаний.
* Чтобы просмотреть предложенные альтернативные вопросы, [используйте параметры View](../How-To/improve-knowledge-base.md#view-suggested-questions) на странице «Оторит».

## <a name="how-qna-makers-implicit-feedback-works"></a>Как работает неявная обратная связь создателя

Неявная обратная связь с нейкой обратной связью с помощью неявной обратной связи с помощью этого алгоритма определяет близость оценки, а затем делает активные предложения по обучению. Для определения сходства оценок применяется сложный алгоритм. Диапазоны в следующем примере не предназначены для фиксированной, но должны использоваться в качестве руководства, чтобы понять влияние только алгоритма.

Если вопрос получает оценку с высокой степенью достоверности (например, 80 %), диапазон оценок, используемых для активного обучения, расширяется (приблизительно на 10 %). Если оценка достоверности снижается (например, до 40 %), диапазон оценок также сокращается (приблизительно на 4 %).

## <a name="how-you-give-explicit-feedback-with-the-train-api"></a>Как вы даете явную обратную связь с API train

NA Maker нуждается в явной обратной связи о том, какой из ответов был лучшим ответом. Как лучший ответ определяется до вас и может включать в себя:

* Отзывы пользователей, выбрав один из ответов.
* Бизнес-логика, например определение приемлемого диапазона баллов.
* Сочетание как отзывов пользователей, так и бизнес-логики.

Используйте [API Train,](https://docs.microsoft.com/rest/api/cognitiveservices/qnamakerruntime/runtime/train) чтобы отправить правильный ответ создателю nA, после того, как пользователь выберет его.

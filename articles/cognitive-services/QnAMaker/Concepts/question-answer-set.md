# <a name="question-and-answer-set-concepts"></a>Понятия набора вопросов и ответов

База знаний состоит из наборов вопросов и ответов.  Каждый набор имеет один ответ, и набор содержит всю информацию, связанную с этим _ответом._ Ответ может слабо напоминать строку базы данных или экземпляр структуры данных.

## <a name="question-and-answer-sets"></a>Наборы вопросов и ответов

**Необходимые** настройки в наборе вопросов и ответов:

* **вопрос** - текст пользовательского запроса, используемый для машинного обучения компании, чтобы согласовать текст вопроса пользователя с другой формулировкой, но одним и тем же ответом
* **ответ** - ответ набора — это ответ, который возвращается, когда пользовательский запрос сопоставляется с соответствующим вопросом

Каждый набор представлен **id.**

**Дополнительные** настройки для набора включают в себя:

* **Альтернативные формы вопроса** - это помогает nA Maker вернуть правильный ответ для более широкого спектра фраз вопросов
* **Метаданные**: Метаданные — это теги, связанные с парой NA, и представлены в виде пар с ключевым значением. Теги метаданных используются для фильтрации пар QnA и ограничения набора, в котором выполняется сопоставление запросов.
* **Мульти-поворот запросы**, используется для продолжения многоповоротной беседы

## <a name="editorially-add-to-knowledge-base"></a>Добавление в базу знаний в режиме редактирования

Если у вас нет уже существующего контента для заполнения базы знаний, вы можете добавить пары qnA редакционно на портале NA Maker. Узнайте, как [обновить базу знаний](../How-To/edit-knowledge-base.md).

## <a name="editing-your-knowledge-base-locally"></a>Локальное изменение базы знаний

После создания базы знаний рекомендуется вносить изменения в текст базы знаний на [портале QnA Maker](https://qnamaker.ai), а не экспортировать и повторно импортировать его через локальные файлы. Тем не менее возможны ситуации, когда вам нужно изменить базу знаний локально.

Экспортируйте базу знаний со страницы **Параметры**, затем внесите в нее изменения в Microsoft Excel. Если вы решите использовать другое приложение для отсеиваемого файла, приложение может ввести ошибки синтаксиса, поскольку оно не полностью соответствует требованиям TSV. TSV-файлы в Microsoft Excel обычно не вызывают ошибок форматирования.

Закончив внесение изменений, повторно импортируйте TSV-файл со страницы **Параметры**. Это полностью заменит текущую базу знаний на импортированную.

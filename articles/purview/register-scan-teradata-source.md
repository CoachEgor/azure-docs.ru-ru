# <a name="register-and-scan-teradata-source-preview"></a>Регистрация и сканирование источника Teradata (предварительная версия)

В этой статье описано, как зарегистрировать источник Teradata в Purview и настроить сканирование.

> [!IMPORTANT]
> В настоящее время этот источник поддерживается в предварительной версии. Попробуйте поработать с ним и оставьте свой отзыв.

## <a name="supported-capabilities"></a>Поддерживаемые возможности

Источник Teradata поддерживает **полное сканирование** для извлечения метаданных из базы данных Teradata и возвращает **происхождение данных** по нескольким информационным активам.

## <a name="prerequisites"></a>Предварительные требования

- Соединитель поддерживает хранилища данных, расположенные только в локальной сети, в виртуальной сети Azure или в виртуальном частном облаке Amazon. Поэтому для подключения вам нужно настроить [локальную среду выполнения интеграции](manage-integration-runtimes.md).

- Убедитесь, что на виртуальной машине установлена среда выполнения Java (JRE) с установленной локальной средой выполнения интеграции.
 
- Убедитесь, что на компьютере с локальной средой выполнения интеграции установлен Распространяемый компонент Visual C++ 2012 с обновлением 4. Если этот компонент не установлен, скачайте его [на этой странице](https://www.microsoft.com/download/details.aspx?id=30679).

- Вам потребуется вручную установить драйвер Teradata JDBC на локальную виртуальную машину. Исполняемый JAR-файл можно скачать на [веб-сайте Teradata](https://downloads.teradata.com/).

    > [!Note] 
    > Этот драйвер должен быть доступен для всех учетных записей на виртуальной машине. Не устанавливайте его в учетной записи пользователя.

- Поддерживаемые версии базы данных Teradata: **с 12.x по 16.x**. Убедитесь, что у вас есть доступ на чтение к сканируемому источнику Teradata.

### <a name="feature-flag"></a>Флаг функции

Регистрация и сканирование источника Teradata становятся доступны после включения флага функции. Добавьте следующий код к URL-адресу: *?feature.ext.datasource=%7b"teradata":"true"%7d* 

Например, полный URL-адрес будет выглядеть так: [https://web.purview.azure.com/?feature.ext.datasource=%7b"metadata":"true"%7d](https://web.purview.azure.com/?feature.ext.datasource=%7b"teradata":"true"%7d)

## <a name="setting-up-authentication-for-a-scan"></a>Настройка проверки подлинности для сканирования
Поддерживается только один метод для проверки подлинности в источнике Teradata: **базовая проверка подлинности в базе данных**.

## <a name="register-a-teradata-source"></a>Регистрация источника Teradata

Чтобы зарегистрировать новый источник Teradata в каталоге данных, сделайте следующее:

1. Перейдите к учетной записи Purview.
2. В области навигации слева выберите **Источники**.
3. Нажмите кнопку **Зарегистрировать**.
4. В поле **Регистрация источников** выберите **Teradata**.
5. Выберите **Продолжить**

На экране **Регистрация источников (Teradata)** выполните следующие действия:

1. Введите **Имя**, под которым этот источник данных будет включен в каталог.
2. Укажите **Узел**, который будет подключаться к источнику Teradata. Это может быть IP-адрес или полная строка подключения к серверу.
3. Выберите коллекцию или создайте новую (необязательно).
4. Щелкните **Готово**, чтобы завершить регистрацию источника данных.

## <a name="creating-and-running-a-scan"></a>Создание и выполнение проверки

Для создания и выполнения новой проверки сделайте следующее:
1. В центре управления щелкните *Среды выполнения интеграции*. Убедитесь, что локальная среда выполнения интеграции настроена. Если это не так, выполните описанные [здесь](manage-integration-runtimes.md) действия, чтобы создать локальную среду выполнения интеграции для сканирования в локальной среде или на виртуальной машине Azure с доступом к локальной сети.

2. Теперь откройте раздел **Источники**.

3. Выберите зарегистрированный источник Teradata.

4. Выберите + Новая проверка.
 
5. Укажите следующие сведения:

    - Имя: Имя сканирования.

    - Подключение через среду выполнения интеграции: выберите настроенную локальную среду выполнения интеграции

    - Метод проверки подлинности: пока поддерживается только проверка подлинности базы данных. Этот вариант будет выбран по умолчанию.

    - Имя пользователя. Пользователь, с которым выполняется подключение к серверу базы данных. Это имя пользователя должно иметь доступ на чтение к этому серверу.

    - Пароль: пароль того пользователя, который используется для подключения к серверу базы данных.

    - Схема. Список импортируемых схем в формате списка значений, разделенных точками с запятыми. Например, так: schema1; schema2. Если этот список пуст, импортируются все пользовательские схемы. Все системные схемы (например, SysAdmin) и объекты игнорируются по умолчанию.

        Для имен схем допускается синтаксис выражений в стиле SQL с использованием символов "%", например A%; %B; %C%; D обозначают следующее:
        - начинается со строки A;    
        - заканчивается строкой B;    
        - содержит строку C;    
        - строго равно D.

        Использование ключевого слова NOT и специальных символов не допускается.

    - Расположение драйвера: полный путь к расположению драйвера Teradata на виртуальной машине клиента. Драйвер JDBC для Teradata должен иметь имя com.teradata.jdbc.TeraDriver

    - Доступный объем памяти: Максимальный объем памяти (в ГБ), доступный для виртуальных машин клиента, которые будут использоваться процессами сканирования. Это значение зависит от размера сканируемого источника Teradata.

    > [!Note] 
    > В общем случае требуется примерно 2 ГБ памяти на каждые 1000 таблиц.

    :::image type="content" source="media/register-scan-teradata-source/setup-scan.png" alt-text="Настройка сканирования" border="true":::

6. Щелкните *Продолжить*.

7. Выберите триггер проверки. Можно настроить расписание или запустить сканирование один раз.

    :::image type="content" source="media/register-scan-teradata-source/scan-trigger.png" alt-text="Триггер сканирования" border="true":::

8. Просмотрите параметры сканирования и щелкните *Сохранить и выполнить*.

## <a name="viewing-your-scans-and-scan-runs"></a>Просмотр данных о проверках и их выполнении

1. Перейдите в центр управления. Выберите **Источники данных** в разделе **Источники и проверка**.

2. Выберите нужный источник данных. Вы увидите список существующих проверок для этого источника данных.

3. Выберите проверку, результаты которой нужно просмотреть.

4. На этой странице будут показаны все предыдущие проверки, а также метрики и состояние каждого выполнения проверки. Здесь также будет показано, была ли проверка запланирована или запущена вручную, ко скольким ресурсам была применена классификация, сколько всего ресурсов было обнаружено, время начала и окончания проверки, а также ее общая длительность.

## <a name="manage-your-scans"></a>Управление сканированиями

Для управления проверкой или ее удаления сделайте следующее:

1. Перейдите в центр управления. Выберите **Источники данных** в разделе **Источники и проверка**, а затем выберите нужный источник данных.

2. Выберите нужную проверку. Вы можете изменить проверку, выбрав **Изменить**.

3. Вы можете удалить проверку, выбрав **Удалить**.

# <a name="peering-service---faqs"></a>Служба пиринга - часто задаваемые вопросы

Вы можете просмотреть информацию ниже для общих вопросов.

**Может ли оператор, использующий существующий Прямой пиринг с корпорацией Майкрософт, поддерживать peering Service?**

Да, перевозчик может использовать свои существующие PNI для поддержки Peering Service. ПНИ-пиринговая служба требует разнообразия для поддержки HA. Если существующие ПНИ уже имеет разнообразие, то никакой новой инфраструктуры не требуется. Если существующие PNI нуждается в разнообразии, то она может быть увеличена.

**Может ли оператор использовать новый Direct пиринг с корпорацией Майкрософт для поддержки Peering Service?**

Да, это тоже возможно. Корпорация Майкрософт будет работать с Carrier над созданием нового прямого пиринга для поддержки Peering Service.  

**Почему Direct требует поддержки Peering Service?**

Одним из основных факторов, лежащих в основе Peering Service, является обеспечение подключения к онлайн-сервисам Майкрософт через хорошо подключенный SP. PNI всегда находятся в диапазоне Gbps и, следовательно, является основным строительным блоком для высокой пропускной связи между оператором и Microsoft.

**Каковы требования к разнообразию при прямой пирингдля для поддержки Peering Service?**

ПНИ должен поддерживать локальную избыточность и геоизбыточность. Локальная избыточность определяется как два разнообразных набора путей в определенном сайте пиринга. Геоизбыточность требует, чтобы Carrier получил дополнительную связь на другом сайте Microsoft edge в случае сбоя основного сайта. Для короткой продолжительности сбоя перевозчик может маршрут трафика через сайт резервного копирования.

**Перевозчик уже предлагает SLA и предприятия класса Интернет, как это предложение отличается?**

Некоторые перевозчики предлагают SLA и предприятия класса Интернет со своей стороны сети. В Peering Service корпорация Майкрософт предложит sLA-трафик в части сети Microsoft. Выбрав peering Service клиент получит сквозной SLA. SLA от их сайта до Microsoft edge в сети интернет-пользователей может быть покрыта isP. SLA в Глобальной сети Microsoft от Microsoft края до конечных пользователей приложение в настоящее время охвачены Microsoft.

**Если поставщик услуг уже имеет связи с корпорацией Майкрософт, используя PNI, то какие изменения необходимы для поддержки Peering Service?**

* Программное обеспечение изменяется для идентификации пользователя Peering Service и его трафика. Может потребоваться изменение политики в области реаутинга для обмена трафиком пользователя на ближайшем краю Майкрософт через подключение Peering Service.
* Убедитесь, что подключение имеет локальную избыточность и геоизбыточность.

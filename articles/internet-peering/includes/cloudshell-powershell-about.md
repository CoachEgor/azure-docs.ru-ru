Для запуска командлетов можно использовать бесплатную интерактивную оболочку Azure Cloud Shell. Она включает предварительно установленные общие инструменты Azure и настроена для использования с вашей учетной записью. Выберите **Copy,** чтобы скопировать код, и вставьте его в облачную оболочку. Затем выберите **Enter,** чтобы запустить его. Существует несколько способов запуска облачной оболочки:


|  |   |
|-----------------------------------------------|---|
| Откройте Cloud Shell в браузере. | [![https://shell.azure.com/powershell](../media/launchcloudshell.png)](https://shell.azure.com/powershell) |
| Выберите кнопку **«Обучка оболочки»** на панели инструментов в правом верхнем углу портала Azure. | [![Облачная оболочка на портале](../media/cloud-shell-menu.png)](https://portal.azure.com) |
|  |  |


Если вы не хотите использовать Azure Cloud Shell, можно установить PowerShell локально. Чтобы установить и использовать PowerShell локально, обязательно установите последнюю версию командлетов PowerShell для Azure Resource Manager. Смдлеты PowerShell обновляются часто. Обычно для получения последней функциональности функции необходимо обновить cmdlets PowerShell. Если вы этого не сможете, вы можете столкнуться с проблемами.

Чтобы найти версию PowerShell, которую вы используете локально, используйте **cmdlet 'Get-Module-ListAvailable Az'.** Если необходимо выполнить обновление, см. статью [об установке модуля Azure PowerShell](https://docs.microsoft.com/powershell/azure/azurerm/install-azurerm-ps). Для получения дополнительной информации узнайте, [как установить и настроить Azure PowerShell.](https://docs.microsoft.com/powershell/azure/azurerm/overview)

Если вы используете PowerShell на macOS, выполните последующие действия в [установке PowerShell на macOS.](https://docs.microsoft.com/powershell/scripting/install/installing-powershell-core-on-macos?view=powershell-6)

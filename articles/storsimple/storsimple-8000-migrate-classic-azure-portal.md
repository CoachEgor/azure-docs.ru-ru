# <a name="migrate-subscriptions-and-storage-accounts-associated-with-storsimple-device-manager-service"></a>Миграция подписок и учетных записей хранения, связанных со службой диспетчера устройств StorSimple

Вам может потребоваться переместить службу StorSimple в новую регистрацию или подписку. Эти сценарии миграции предполагают изменение учетных записей и изменения в центре обработки данных. Таблица ниже поможет понять, какие из этих сценариев поддерживаются. Также здесь приведены подробные инструкции по перемещению.

## <a name="account-changes"></a>Изменение учетных записей

| Ресурсы для миграции| Поддерживаются| Простой| Процесс поддержки Azure| Подход|
|-----|-----|-----|-----|-----|
| Вся подписка (включая учетные записи службы StorSimple и учетные записи хранилища) перемещается в другую регистрацию. | Да       | Нет       | **Перенос регистрации**<br>Используется:<li>при приобретении нового предложения Azure в рамках нового соглашения;</li><li>при миграции всех учетных записей и подписок из старой регистрации в новую. Сюда входят все службы Azure в рамках старой подписки.</li> | **Шаг 1. Откройте запрос в службу поддержки операции Azure Enterprise.**<li>Перейдите к [https://aka.ms/AzureEntSupport](https://aka.ms/AzureEntSupport).</li><li> выберите **Enrollment Administration** (Администрирование регистрации), а затем — **Transfer from one enrollment to a new enrollment** (Перенос из текущей регистрации в новую).<br>**Шаг 2. Введите запрашиваемые данные**<br>Укажите:<li>исходный регистрационный номер;</li><li> целевой регистрационный номер;</li><li>дату начала переноса.|
| Служба StorSimple из текущей учетной записи в новую регистрацию.    | Да       | Нет       | **Перенос учетной записи**<br>Используйте следующую команду:<li>если не требуется полный перенос регистрации;</li><li>если в новую регистрацию нужно переместить только определенные учетные записи.</li>| **Шаг 1. Откройте запрос в службу поддержки операции Azure Enterprise.**<li>Перейдите к [https://aka.ms/AzureEntSupport](https://aka.ms/AzureEntSupport).</li><li>выберите **Enrollment Administration** (Администрирование регистрации), а затем — **Transfer an EA Account to a new enrollment** (Перенос учетной записи EA в новую регистрацию).<br>**Шаг 2. Введите запрашиваемые данные**<br>Укажите:<li>исходный регистрационный номер;</li><li> целевой регистрационный номер;</li><li>дату начала переноса.|

Отображение всех элементов может занять до 10 минут.|

| Учетная запись хранения в новую подписку или группу ресурсов     | Да        | Нет             |Переместите учетную запись хранения в новую подписку или группу ресурсов |После перемещения, если ключи доступа к учетной записи хранения обновлены, пользователю потребуется настроить ключи доступа вручную для перенесенной учетной записи хранения с помощью службы диспетчера устройств StorSimple.|

Это обеспечит нормальную работу устройств StorSimple и способность распределять основные и резервные данные по категориям в Azure. Подробные инструкции по синхронизации ключей доступа, см. в статье [Rotation workflow](storsimple-8000-manage-storage-accounts.md#key-rotation-of-storage-accounts) (Рабочий процесс смены ключей).</li><li> В случае облачного устройства StorSimple, если классическая учетная запись хранения перенесена, но базовая виртуальная машина по-прежнему остается в классической модели, устройство должно функционировать надлежащим образом. Если базовая виртуальная машина для облачного устройства переносится, функции отключения и удаления не будут работать.</li><li> Необходимо создать новые облачные устройства StorSimple на портале Azure и затем выполнить отработку отказа со старых облачных устройств. Создать облачное устройство StorSimple на новом портале Azure с помощью классической учетной записи хранения нельзя. Для этого потребуется учетная запись хранилища Azure Resource Manager. Дополнительные сведения см. в статье [Развертывание и администрирование облачного устройства StorSimple в Azure (обновление 3 и более поздние версии)](storsimple-8000-cloud-appliance-u2.md).</li>|

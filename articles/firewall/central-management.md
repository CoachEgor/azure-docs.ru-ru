# <a name="azure-firewall-central-management"></a>Централизованное управление брандмауэром Azure

[!INCLUDE [Preview](../../includes/firewall-manager-preview-notice.md)]

Если вы управляете несколькими брандмауэрами, то известно, что постоянно меняющиеся правила брандмауэра затрудняют их синхронизацию. Центральным ИТ-группам требуется способ определения базовых политик брандмауэра и их применения в нескольких подразделениях. В то же время, DevOps группам нужно создать собственные локальные политики брандмауэра для повышения гибкости.

Предварительная версия диспетчера брандмауэра Azure поможет решить эти проблемы.


## <a name="azure-firewall-manager-preview"></a>Предварительная версия диспетчера брандмауэра Azure

Предварительная версия диспетчера брандмауэра Azure — это служба управления безопасностью сети, которая предоставляет централизованную политику безопасности и управление маршрутами для периметра безопасности на основе облака. Это позволяет ИТ – ИТ легко определять правила уровня сети и приложения для фильтрации трафика в нескольких экземплярах брандмауэра Azure. Вы можете охватывать различные регионы и подписки Azure в центральных и периферийных архитектурах для управления трафиком и защиты. Кроме того, он обеспечивает DevOps лучшую гибкость при использовании производных локальных политик безопасности брандмауэра, реализованных в разных организациях.

### <a name="firewall-policy"></a>Политика брандмауэра

Политика брандмауэра — это ресурс Azure, который содержит коллекции правил NAT, сети и приложений, а затем Параметры анализа угроз. Это глобальный ресурс, который можно использовать для нескольких экземпляров брандмауэра Azure в *защищенных виртуальных концентраторах* и *виртуальных сетях концентраторов*. Новые политики можно создавать с нуля или наследовать от существующих политик. Наследование позволяет DevOps создавать локальные политики брандмауэра поверх обязательных базовых политик организации. Политики работают в разных регионах и подписках.
 
Вы можете создать политику и связи брандмауэра с помощью диспетчера брандмауэра Azure. Однако можно также создать политику и управлять ею с помощью REST API, шаблонов, Azure PowerShell и интерфейса командной строки. После создания политики ее можно связать с брандмауэром в виртуальном концентраторе глобальной сети, сделав его *защищенным виртуальным концентратором* и (или) брандмауэром в виртуальной сети, сделав ее *виртуальной сетью*.

### <a name="pricing"></a>Цены

Счета за политики выставляются на основе сопоставлений брандмауэров. Политика с нулевым или одним ассоциацией брандмауэра не взимается. Плата за политику с несколькими связями брандмауэра оплачивается по фиксированной ставке. Дополнительные сведения см. на странице [цен на диспетчер брандмауэра Azure](https://azure.microsoft.com/pricing/details/firewall-manager/).

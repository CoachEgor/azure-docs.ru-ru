# <a name="azure-firewall-central-management"></a>Центральное управление Azure Firewall

[!INCLUDE [Preview](../../includes/firewall-manager-preview-notice.md)]

Если вы управляете несколькими брандмауэрами, вы знаете, что постоянно меняющиеся правила брандмауэра затрудняют их синхронизацию. Центральным ИТ-группам необходим способ определения базовых политик брандмауэра и обеспечения их выполнения в нескольких бизнес-подразделениях. В то же время команды DevOps хотят создать свои собственные локальные политики брандмауэра для повышения гибкости.

Предварительный просмотр менеджера firewall Azure может помочь решить эти проблемы.


## <a name="azure-firewall-manager-preview"></a>Предварительный просмотр менеджера брандмауэра Azure

Azure Firewall Manager Preview — это служба управления сетевой безопасностью, обеспечивающая централизованную политику безопасности и управление маршрутами для облачных периметров безопасности. Это упрощает для ИТ-команд enterprise централизованное определение правил сетевого и уровня приложений для фильтрации трафика в нескольких экземплярах Azure Firewall. Можно охватить различные области Azure и подписки в концентраторах и спомненные архитектуры для управления трафиком и защиты. Он также обеспечивает лучшую гибкость DevOps с выведенными локальными политиками безопасности брандмауэра, которые реализуются в разных организациях.

### <a name="firewall-policy"></a>Политика брандмауэра

Политика Firewall — это ресурс Azure, содержащий коллекции данных, сетевых и приложений и настройки threat Intelligence. Это глобальный ресурс, который можно использовать в нескольких экземплярах Azure Firewall в *защищенных виртуальных концентрах* и *виртуальных сетях.* Новые политики могут быть созданы с нуля или унаследованы от существующих политик. Наследование позволяет DevOps создавать локальные политики брандмауэра поверх базовой политики, утвержденной организацией. Политики работают в разных регионах и подписках.
 
Можно создать политику брандмауэра и ассоциации с менеджером брандмауэра Azure. Однако можно также создавать и управлять политикой с помощью REST API, шаблонов, Azure PowerShell и CLI. После создания политики можно связать ее с брандмауэром в виртуальном концентраторе WAN, что делает его *защищенным виртуальным концентратором* и/или брандмауэром в виртуальной сети, что делает его *Hub Virtual Network.*

### <a name="pricing"></a>Цены

Политики выставляются на основе ассоциаций брандмауэра. Политика с нулевым или одной ассоциацией брандмауэра бесплатна. Политика с несколькими ассоциациями брандмауэра выставляется по фиксированной ставке. Для получения дополнительной [Azure Firewall Manager Pricing](https://azure.microsoft.com/pricing/details/firewall-manager/)информации см.

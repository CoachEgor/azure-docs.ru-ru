# <a name="azure-firewall-forced-tunneling"></a>Принудительное туннелирование брандмауэра Azure

Вы можете настроить брандмауэр Azure для маршрутизации всего трафика, связанного с Интернетом, в назначенный следующий прыжок, а не напрямую в Интернет. Например, у вас может быть локальный брандмауэр или другое сетевое виртуальное устройство (NVA) для обработки сетевого трафика перед его передачей в Интернет.

По умолчанию принудительное туннелирование запрещено в брандмауэре Azure, чтобы обеспечить соблюдение всех исходящих зависимостей Azure. Настройки определяемых пользователем маршрутов (UDR) в *азурефиреваллсубнет* , которые имеют маршрут по умолчанию, не переключаются непосредственно в Интернет, отключены.

## <a name="forced-tunneling-configuration"></a>Конфигурация с принудительным туннелированием

Для поддержки принудительного туннелирования трафик управления службами отделяется от клиентского трафика. Для соответствующего общедоступного IP-адреса требуется дополнительная выделенная подсеть с именем *азурефиреваллманажементсубнет* (минимальный размер подсети/26). Единственным разрешенным маршрутом в этой подсети является маршрут по умолчанию к Интернету, а распространение маршрутов BGP должно быть отключено.

При наличии маршрута по умолчанию, объявленного по протоколу BGP для принудительного передачи трафика в локальную среду, необходимо создать *азурефиреваллсубнет* и *азурефиреваллманажементсубнет* перед развертыванием брандмауэра и настроить для него UDR с маршрутом по умолчанию и отключить **Распространение маршрутов шлюза виртуальной сети** .

В этой конфигурации *азурефиреваллсубнет* теперь может включать маршруты к любому локальному брандмауэру или NVA для обработки трафика перед его передачей в Интернет. Вы также можете опубликовать эти маршруты с помощью BGP в *азурефиреваллсубнет* , если в этой подсети включено **Распространение маршрутов шлюза виртуальной сети** .

Например, можно создать маршрут по умолчанию для *азурефиреваллсубнет* с VPN-шлюзом в качестве следующего прыжка для перехода на локальное устройство. Также можно включить **Распространение маршрутов шлюза виртуальной сети** , чтобы получить соответствующие маршруты к локальной сети.

![Распространение маршрутов шлюза виртуальной сети](media/forced-tunneling/route-propagation.png)

После настройки брандмауэра Azure для поддержки принудительного туннелирования отменить конфигурацию невозможно. Если удалить все другие IP-конфигурации в брандмауэре, конфигурация IP-адресов управления также будет удалена, а брандмауэр будет освобожден. Общедоступный IP-адрес, назначенный IP-конфигурации управления, не может быть удален, но можно назначить другой общедоступный IP-адрес.

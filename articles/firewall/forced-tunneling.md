# <a name="azure-firewall-forced-tunneling-preview"></a>Брандмауэр Azure принудительного туннелирования (предварительный просмотр)

Можно настроить Azure Firewall, чтобы направить весь трафик, связанный с Интернетом, в назначенный следующий переход, вместо того, чтобы переходить непосредственно в Интернет. Например, у вас может быть брандмауэр на переднем крае или другой сетевой виртуальный прибор (NVA) для обработки сетевого трафика до его перехи, прежде чем он будет передан в Интернет.

> [!IMPORTANT]
> Принудительное туннелирование Брандмауэра Azure в настоящее время находится в открытом доступе.
>
> Эта общедоступная предварительная версия предоставляется без соглашения об уровне обслуживания и не должна использоваться в рабочей среде. Некоторые функции могут не поддерживаться, иметь ограничения и быть доступными не во всех расположениях Azure. Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

По умолчанию принудительное туннелирование не допускается в брандмауэре Azure Firewall, чтобы обеспечить соответствие всех исходящих зависимостей Azure. Пользовательские конфигурации маршрута (UDR) в *AzureFirewallSubnet,* которые имеют маршрут по умолчанию, не идущий непосредственно в Интернет, отключены.

## <a name="forced-tunneling-configuration"></a>Конфигурация с принудительным туннелированием

Для поддержки принудительного туннелирования трафик управления обслуживанием отделен от трафика клиентов. Дополнительная специальная подсеть под названием *AzureFirewallManagementSubnet* (минимальный размер подсети /26) требуется с собственным ассоциированным публичным IP-адресом. Единственным маршрутом, разрешенным в этой подсети, является маршрут по умолчанию в Интернет, и распространение маршрута BGP должно быть отключено.

Если у вас есть маршрут по умолчанию, рекламируемый через BGP, чтобы заставить трафик на предпосылки, необходимо создать *AzureFirewallSubnet* и *AzureFirewallManagementSubnet* перед развертыванием брандмауэра и UDR с маршрутом по умолчанию в Интернет, и **виртуальная сеть шлюза распространения маршрута** отключен.

В этой конфигурации *AzureFirewallSubnet* теперь может включать маршруты в любой локальный брандмауэр или NVA для обработки трафика до его перехибения в Интернет. Вы также можете опубликовать эти маршруты через BGP в *AzureFirewallSubnet,* если **распространение маршрута виртуального сетевого шлюза** включено в эту подсеть.

Например, можно создать маршрут по умолчанию на *AzureFirewallSubnet* с вашим VPN шлюзом в качестве следующего перехода, чтобы добраться до вашего устройства. Или вы можете включить **распространение маршрута виртуального сетевого шлюза,** чтобы получить соответствующие маршруты в локальную сеть.

![Распространение маршрута виртуального сетевого шлюза](media/forced-tunneling/route-propagation.png)

После настройки Брандмауэра Azure для поддержки принудительного туннелирования можно отменить конфигурацию. При удалении всех других конфигураций IP-адресов в брандмауэре также удаляется конфигурация IP-адреса управления и брандмауэр удаляется. Общедоступный IP-адрес, присвоенный конфигурации IP-адреса управления, не может быть удален, но можно назначить другой общедоступный IP-адрес.

# <a name="iot-hub-dps-ip-addresses"></a>IP-адреса IoT Hub DPS

Префиксы IP-адреса для общедоступных конечных точек службы обеспечения устройств IoT Hub (DPS) периодически публикуются под [тегом сервиса](../virtual-network/service-tags-overview.md) _AzureIoTHub._ Эти префиксы IP-адреса можно использовать для управления подключением между экземпляром IoT DPS и устройствами или сетевыми активами для реализации различных целей сетевой изоляции:

| Цель | Подход |
|------|----------|
| Убедитесь, что ваши устройства и службы общаются только с конечными точками IoT Hub DPS | Используйте тег службы _AzureIoTHub,_ чтобы обнаружить экземпляры IoT Hub DPS. Навлаживправила правила ALLOW в настройках брандмауэра ваших устройств и служб для префиксов IP-адресов соответственно. Настройка правил для отбрасываемого трафика на другие IP-адреса назначения, с которыми вы не хотите, чтобы устройства или службы общались. |
| Убедитесь, что конечная точка IoT Hub DPS получает соединения только с ваших устройств и сетевых ресурсов | Используйте [функцию ip-фильтра](iot-dps-ip-filtering.md) IoT DPS для создания правил фильтрации для устройств и AA службы DPS. Эти правила фильтрации могут использоваться для размноза соединения только с ваших устройств и IP-адресов сетевых активов (см. раздел [ограничений).](#limitations-and-workarounds) | 




## <a name="best-practices"></a>Рекомендации

* При добавлении правил ALLOW в конфигурацию брандмауэра ваших устройств лучше всего предоставлять определенные [порты, используемые применимыми протоколами.](../iot-hub/iot-hub-devguide-protocols.md#port-numbers)

* Префиксы IP-адреса экземпляров IoT DPS могут быть изменены. Эти изменения периодически публикуются через теги услуг до вступления в силу. Поэтому важно, чтобы вы разрабатывали процессы для регулярного извлечения и использования новейших сервисных тегов. Этот процесс можно автоматизировать с помощью [API обнаружения тегов службы.](../virtual-network/service-tags-overview.md#service-tags-on-premises) API обнаружения тегов службы все еще находится в предварительном просмотре и в некоторых случаях может не привести полный список тегов и IP-адресов. До тех пор, пока API обнаружения, как правило, доступны, рассмотреть возможность использования [теги услуг в загружаемом формате JSON](../virtual-network/service-tags-overview.md#discover-service-tags-by-using-downloadable-json-files). 

* Используйте *AzureIoTHub. *Тег названия региона для определения префиксов IP, используемых конечными точками DPS в определенном регионе. Для учета аварийного восстановления центра обработки данных или [регионального сбоя](../iot-hub/iot-hub-ha-dr.md)также обеспечивается подключение к ip-префиксам геопарной области экземпляра DPS.

* Настройка правил брандмауэра для экземпляра DPS может блокировать подключение, необходимое для выполнения команд Azure CLI и PowerShell. Чтобы избежать проблем с подключением, можно добавить правила ALLOW для префиксов IP-адресов ваших клиентов, чтобы повторно включить клиентов CLI или PowerShell для связи с экземпляром DPS.  


## <a name="limitations-and-workarounds"></a>Ограничения и обходные решения

* Функция IP-фильтра DPS имеет ограничение в 100 правил. Этот лимит может быть поднят с помощью запросов через службу поддержки клиентов Azure. 

* Настроенные [правила фильтрации IP](iot-dps-ip-filtering.md) применяются только на конечных точках DPS, а не на связанных конечных точках IoT Hub. Фильтрация IP для связанных концентраторов IoT должна быть настроена отдельно. Для получения дополнительной информации см., [IoT концентратор IP фильтрации правил](../iot-hub/iot-hub-ip-filtering.md).

## <a name="support-for-ipv6"></a>Поддержка IPv6 

IPv6 в настоящее время не поддерживается на IoT Hub или DPS.

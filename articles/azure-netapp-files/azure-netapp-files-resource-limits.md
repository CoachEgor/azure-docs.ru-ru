# <a name="resource-limits-for-azure-netapp-files"></a>Ограничения ресурсов для службы Azure NetApp Files

Понимание ограничений ресурсов для службы Azure NetApp Files помогает в управлении томами.

## <a name="resource-limits"></a>Ограничения ресурсов

В следующей таблице описаны ограничения ресурсов для файлов NetApp Azure:

|  Ресурс  |  Ограничение по умолчанию  |  Регулируемый через запрос поддержки  |
|----------------|---------------------|--------------------------------------|
|  Количество учетных записей NetApp в регионе Azure   |  10    |  Да   |
|  Количество пулов емкости на счет NetApp   |    25     |   Да   |
|  Количество объемов на пул емкости     |    500   |    Да     |
|  Количество снимков на объем       |    255     |    нет        |
|  Количество подсетей, делегированных файлам NetApp Azure (Microsoft.NetApp/volumes) в виртуальной сети Azure    |   1   |    нет    |
|  Количество использованных ит-провайдеров в VNet (включая сразу же заглянув VNets) с azure NetApp Files   |    1000   |    Да   |
|  Минимальный размер единого пула емкости   |  4 ТиБ     |    нет  |
|  Максимальный размер единого пула емкости    |  500 ТиБ   |   нет   |
|  Минимальный размер одного тома    |    100 ГиБ    |    нет    |
|  Максимальный размер одного тома     |    100 TiB    |    нет    |
|  Максимальный размер одного файла     |    16 ТиБ    |    нет    |    
|  Максимальный размер одного каталога      |    320 МБ    |    нет    |    
|  Максимальное количество файлов[(maxfiles)](#maxfiles)на объем     |    100 млн    |    Да    |    

Для получения дополнительной информации [см.](azure-netapp-files-faqs.md#capacity-management-faqs)

## <a name="maxfiles-limits"></a>Лимиты Maxfiles<a name="maxfiles"></a> 

Объемы файлов Azure NetApp имеют лимит, называемый *maxfiles.* Пределом максимального файла является количество файлов, которые может содержать объем. Лимит максимального количества файлов Azure NetApp индексируется на основе размера (квоты) объема. Ограничение максимального размера для объема увеличивается или уменьшается в размере 20 миллионов файлов на TiB припредусмотренного размера объема. 

Служба динамически корректирует предел максимального количества для объема в зависимости от его подготовленного размера. Например, объем, настроенный изначально с размером 1 TiB, будет иметь максимум в 20 миллионов. Последующие изменения в размере объема приведут к автоматической корректировке лимита максфилов на основе следующих правил: 

|    Размер объема (квота)     |  Автоматическая корректировка лимита максфилов    |
|----------------------------|-------------------|
|    < 1 TiB                 |    20 миллионов     |
|    >1 TiB, но < 2 TiB    |    40 миллионов     |
|    >2 TiB, но < 3 TiB    |    60 млн     |
|    >No 3 TiB, но < 4 TiB    |    80 миллионов     |
|    >No 4 TiB                |    100 млн    |

Для любого размера объема можно инициировать [запрос на поддержку,](#limit_increase) чтобы увеличить лимит maxfiles более 100 миллионов.

## <a name="request-limit-increase"></a>Увеличение лимита запроса<a name="limit_increase"></a> 

Можно создать запрос поддержки Azure для увеличения регулируемых лимитов из приведенной выше таблицы. 

Из навигационного самолета портала Azure: 

1. Нажмите **Справка и поддержка**.
2. Нажмите **и новый запрос на поддержку**.
3. На вкладке Основные сведения укажите следующую информацию. 
    1. Тип выпуска: Выберите **ограничение службы и подписки (квоты).**
    2. Подписка: Выберите подписку на ресурс, который вам нужен, квота увеличена.
    3. Тип квоты: Выберите **Хранилище: ограничения файлов NetApp Azure**.
    4. Нажмите **далее: Решения**.
4. На вкладке "Подробности":
    1. В поле описания предоставьте следующую информацию для соответствующего типа ресурса:

        |  Ресурс  |    Родительские ресурсы      |    Запрошенные новые лимиты     |    Причина увеличения квот       |
        |----------------|------------------------------|---------------------------------|------------------------------------------|
        |  Учетная запись |  *Идентификатор подписки*   |  *Запрошенновый новый максимальный номер **счета***    |  *Какой сценарий или случай использования побудили запрос?*  |
        |  пул    |  *Идентификатор подписки, учетная запись URI*  |  *Запрошенный новый максимальный номер **пула***   |  *Какой сценарий или случай использования побудили запрос?*  |
        |  Том  |  *Идентификатор подписки, учетная запись URI, Пул URI*   |  *Запрошено новое максимальное количество **громкости***     |  *Какой сценарий или случай использования побудили запрос?*  |
        |  Максфилы  |  *Идентификатор подписки, учетная запись URI, бассейн URI, Том URI*   |  *Запрошенный новый максимальный номер **максфилов***     |  *Какой сценарий или случай использования побудили запрос?*  |    

    2. Укажите соответствующий метод поддержки и предоставьте информацию о контракте.

    3. Нажмите **далее: Просмотрите и создайте** для создания запроса.

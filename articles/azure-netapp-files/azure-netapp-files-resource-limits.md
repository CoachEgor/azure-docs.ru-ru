# <a name="resource-limits-for-azure-netapp-files"></a>Ограничения ресурсов для службы Azure NetApp Files

Понимание ограничений ресурсов для службы Azure NetApp Files помогает в управлении томами.

## <a name="resource-limits"></a>Ограничения ресурсов

В следующей таблице описаны ограничения ресурсов для службы файлов Azure NetApp:

|  Ресурс  |  Ограничение по умолчанию  |  Также может, обратитесь в службу поддержки  |
|----------------|---------------------|--------------------------------------|
|  Количество учетных записей NetApp на каждую подписку Azure   |  10    |  Да   |
|  Количество пулов емкости одной учетной записи NetApp   |    25     |   Да   |
|  Число томов в пуле ресурсов     |    500   |    Да     |
|  Количество моментальных снимков на том       |    255     |    Нет        |
|  Количество подсетей, делегировать NetApp службы файлов Azure (Microsoft.NetApp/volumes) каждой виртуальной сети Azure    |   1   |    Нет    |
|  Минимальный размер пула емкость одного   |  4 ТиБ     |    Нет  |
|  Максимальный размер пула емкость одного    |  500 ТиБ   |   Нет   |
|  Минимальный размер одного тома    |    100 ГиБ    |    Нет    |
|  Максимальное значение квоты для одного тома *   |   92 Тиб   |    Нет   |
|  Максимальный размер одного тома *     |    100 (тиб)    |    Нет       |

* Тома может быть вручную создан или изменен для максимально 92 Тиб. Тем не менее том может расширяться до 100 ТБ в избыточные сценарии. См. в разделе [модели затрат для службы файлов Azure NetApp](azure-netapp-files-cost-model.md) сведения о превышением емкости. 

## <a name="request-limit-increase"></a>Запросить увеличение лимита 

Можно создать запрос на поддержку Azure для увеличения настраиваемого ограничения из приведенной выше таблице. 

Плоскостью переход на портале Azure: 

1. Нажмите кнопку **Справка и поддержка**.
2. Нажмите кнопку **+ создать запрос в службу поддержки**.
3. На вкладке "Основные" укажите следующие сведения: 
    1. Тип проблемы. Выберите **ограничения службы и подписки (квоты)**.
    2. Подписки: Выберите подписку, для ресурса, который необходимо увеличить квоту.
    3. Тип квоты: Выберите **хранилища: Служба файлов Azure NetApp ограничивает**.
    4. Щелкните **Далее: Решения**.
4. На вкладке "Сведения":
    1. В поле «Описание» укажите следующие сведения для соответствующего типа ресурсов:

        |  Ресурс  |    Родительскими ресурсами      |    Запрошенный новых ограничений     |    Причина увеличения квоты       |
        |----------------|------------------------------|---------------------------------|------------------------------------------|
        |  Учетная запись. |  *Идентификатор подписки*   |  *Запрошенное новое максимальное **учетной записи** номер*    |  *Какой сценарий или вариант использования предложение запроса?*  |
        |  Пул    |  *Идентификатор подписки, URI учетной записи*  |  *Запрошенное новое максимальное **пула** номер*   |  *Какой сценарий или вариант использования предложение запроса?*  |
        |  Том  |  *Идентификатор подписки, URI, учетной записи пула URI*   |  *Запрошенное новое максимальное **тома** номер*     |  *Какой сценарий или вариант использования предложение запроса?*  |

    2. Укажите соответствующий поддерживает метод и укажите необходимые сведения для контракта.

    3. Щелкните **Далее: Просмотр и создание** для создания запроса.

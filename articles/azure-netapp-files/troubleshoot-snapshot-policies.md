# <a name="troubleshoot-snapshot-policies"></a>Устранение неполадок политик моментальных снимков

В этой статье описываются сценарии ошибок, которые могут возникнуть при управлении Azure NetApp Files политик моментальных снимков. Он также предоставляет решения, которые могут помочь решить проблемы.

## <a name="snapshot-policy-creation-failing-with-invalid-snapshot-policy-name"></a>Сбой создания политики моментальных снимков с недопустимым именем политики моментальных снимков

При создании политики моментальных снимков возникла ошибка, если имя политики моментального снимка недопустимо. Следующие рекомендации применимы к именам политик моментальных снимков:  

* Имя политики моментальных снимков не может содержать символы, отличные от ASCII или специальных символов.
* Имя политики моментальных снимков должно начинаться с буквы или цифры и может содержать буквы, цифры, символы подчеркивания ("_") и дефисы ("-").
* Имя политики моментальных снимков должно содержать от 1 до 64 символов.  

Необходимо изменить имя политики моментальных снимков в соответствии с рекомендациями.  

## <a name="snapshot-policy-creation-failing-with-invalid-values"></a>Сбой создания политики моментальных снимков с недопустимыми значениями 

Azure NetApp Files не удается создать политику моментальных снимков, если ввести недопустимое значение для поля, такого как `Number of snapshots to keep` или `Minute on the hour to take snapshot` .  
 
Допустимы следующие значения.   

* Значение должно быть допустимым числом.
* Значение должно находиться в диапазоне от 0 до 59.

Убедитесь, что для полей указано допустимое значение.

## <a name="snapshot-policy-creation-failing-with-total-number-of-snapshots-to-keep-exceeds-255-error"></a>Сбой создания политики моментальных снимков с "общим числом моментальных снимков для сохранения, превышающих 255" 

Каждый том может иметь не [более 255 моментальных снимков](azure-netapp-files-resource-limits.md). Максимальное значение включает сумму всех моментальных снимков по часам, дням, неделям и месяцам. Следует уменьшить `Snapshots to keep` значение и повторить попытку.

## <a name="assigning-policy-to-a-volume-failing-with-total-snapshot-policy-is-over-the-max-255-error"></a>Назначение политики для тома с ошибкой "Общая политика моментальных снимков превышает максимальную ошибку" 255 "

Каждый том может иметь не [более 255 моментальных снимков](azure-netapp-files-resource-limits.md). Если сумма всех моментальных снимков по запросу, ежечасных, ежедневных, еженедельных и месячных, превышает максимальное значение, возникает ошибка. Уменьшите `snapshots to keep` значение или удалите некоторые моментальные снимки по запросу и повторите попытку.

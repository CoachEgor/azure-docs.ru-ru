# <a name="service-levels-for-azure-netapp-files"></a>Уровни обслуживания для Azure NetApp Files
Уровни службы — атрибут емкости пула. Уровни обслуживания определяются и различаются по разрешенных максимальной пропускной способности для тома в пуле емкость, в соответствии с квотой, назначаемое тому.

## <a name="supported-service-levels"></a>Уровни обслуживания, поддерживаемых

Служба файлов Azure NetApp поддерживает три уровня обслуживания: *Ultra*, *уровня "премиум"*, и *стандартный*. 

* <a name="Ultra"></a>Ultra хранилища

    Уровень Ultra хранилища доступно до 128 Миб/с пропускной способности в 1 Тиб для назначена квота на количество. 

* <a name="Premium"></a>Хранилище класса "Премиум"

    Хранилище уровня "премиум" обеспечивает до 64 Миб/с пропускной способности в 1 Тиб для назначена квота на количество. 

* <a name="Standard"></a>Хранилище класса "Стандартный".

    Стандартный уровень хранилища предоставляет до 16 Миб/с пропускной способности в 1 Тиб для назначена квота на количество.

## <a name="throughput-limits"></a>Ограничения пропускной способности

Предельная пропускная способность для тома определяется сочетанием следующих факторов:
* Обновления для пула ресурсов, к которому принадлежит тома
* Квоты, назначаемое тому  

Эта концепция показана на следующей схеме:

![Иллюстрация уровня службы](../media/azure-netapp-files/azure-netapp-files-service-levels.png)

В примере 1 выше, том из пула ресурсов с уровнем хранилища уровня "премиум", назначенный 2 Тиб квоты будет назначен ограничение пропускной способности в 128 Миб/с (2 Тиб * 64 Миб/с). Этот сценарий применяется независимо от размера емкости пула и фактическим объемом потребления.

В примере 2 выше, том из пула ресурсов с уровнем хранилища уровня "премиум", назначенный содержит 100 Гиб квоты будет назначен ограничение пропускной способности в ширину в 6,25 Миб/с (0.09765625 Тиб * 64 Миб/с). Этот сценарий применяется независимо от размера емкости пула и фактическим объемом потребления.

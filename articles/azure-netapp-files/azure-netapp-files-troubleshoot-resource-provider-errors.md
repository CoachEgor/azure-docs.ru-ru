# <a name="troubleshoot-azure-netapp-files-resource-provider-errors"></a>Устранение ошибок с поставщиком ресурсов Azure NetApp Files
В этой статье описываются распространенные поставщика ресурсов Azure NetApp файлы ошибок, их причины, решения и обходные пути. 

<a name="error_01"></a>***Хранилище ключей не настроен.***   
Azure Key Vault сохраняет необходимые учетные данные для доступа к основной API. Эта ошибка означает, что хранилище ключей Azure не был получен полный учетные данные для доступа к основной API.

* Причина:  
Хранилище ключей не получил правильные учетные данные или учетные данные являются неполными.  

* Решение   
Служба файлов Azure NetApp использует хранилище ключей Azure. Azure Key Vault выполняет проверку подлинности с помощью токена из Azure Active Directory. Таким образом владелец приложения должен зарегистрировать приложение в Azure Active Directory.

* Возможное решение   
Отсутствует.  Хранилище ключей необходимо настроить правильно по использованию Azure NetApp файлов.  

<a name="error_02"></a>***Создание маркера не может изменяться.***   
Эта ошибка возникает при попытке изменить создания токен, после создания тома.
Создание маркера необходимо задать при создании тома и нельзя будет изменить позже.

* Причина:   
Вы пытаетесь изменить создания маркера, созданный том, что операция не поддерживается.

* Решение   
После создания тома, рекомендуется удалить параметр из запроса, чтобы закрыть сообщение об ошибке.

* Возможное решение   
Если вам нужно изменить токен создания, можно создать новый том с новым маркером создания и затем перенести данные с новым томом.


<a name="error_03"></a>***Создание маркера должна быть менее 16 символов.***   
Эта ошибка возникает при создания маркер не соответствует требованиям длины. Длина создания маркера должен быть по крайней мере 16 символов.

* Причина:   
Создание маркера не соответствует требованиям длины.  При создании тома с помощью API создания маркер является обязательным. Если вы используете портал, токен может быть создана автоматически.

* Решение   
Увеличение длины создания маркера. Например можно добавить еще одно слово в начале или конце создания маркера.

* Возможное решение   
Минимальная требуемая длина маркера создания не могут быть пропущены.  Можно использовать префикс или суффикс к длине создания маркера.


<a name="error_04"></a>***Ошибка при удалении тома, который не был найден в службе файлов Azure NetApp.***   
Эта ошибка возникла из-за внутренней реестра ресурсов не синхронизирован.

* Причина:   
Тома могут оставаться отображается на портале некоторое время после его удаления. При удалении тома с помощью API, это возможно, что том был указан неправильно. Ошибка также может быть вызвана устаревший браузер кэша.

* Решение   
Очистка кэша при использовании портала. Также есть внутренний кэш, которое обновляется каждые 10 минут.  Можно попытаться очистить кэш еще раз.  Если проблема повторится через 10 минут, можно создать запрос в службу поддержки.

* Возможное решение   
В то же время использовать тома и игнорировать существующий.


<a name="error_05"></a>***Ошибка при вставке новый том расположенный NetApp службы файлов Azure.***   
Эта ошибка возникает потому, что внутренних ресурсов не синхронизировано.

* Причина:   
Том может отображаться на портале некоторое время после его удаления. При удалении тома с помощью API, это возможно, что том был указан неправильно.

* Решение   
Если вы используете портал, том уже создан.  Том должен отображаться автоматически. Если проблема повторится, можно создать запрос в службу поддержки.

* Возможное решение   
Можно создать том с другим именем и различных создания маркера.


<a name="error_06"></a>***Путь к файлу может содержать буквы, цифры и дефисы (""-"») только.***   
Эта ошибка происходит, когда путь к файлу содержит неподдерживаемые символы, например, точку (".»), запятая («,»), символ подчеркивания (»\_«), или знак доллара («$»).

* Причина:   
Путь к файлу содержит неподдерживаемые символы, например, точку (".»), запятая («,»), символ подчеркивания (»\_«), или знак доллара («$»).

* Решение   
Удалите символы, которые не являются буквы, цифры и дефисы ("-») из введенный путь к файлу.

* Возможное решение   
Можно заменить символ подчеркивания с дефиса или используйте регистр вместо пробелов, обозначающей начало нового слов (например, с помощью «NewVolume» вместо «новый том»).


<a name="error_07"></a>***Идентификатор тома нельзя изменить.***   
Эта ошибка возникает при попытке изменить идентификатор тома.  Идет изменение идентификатора тома операция не поддерживается.

* Причина:   
Идентификатор файловой системы задается при создании тома. Идентификатор тома нельзя будет изменить.

* Решение   
Отсутствует.

* Возможное решение   
Отсутствует.  Идентификатор тома создается при создании тома и нельзя будет изменить.


<a name="error_08"></a>***Недопустимое значение "{0}" получена для {1}.***   
Это сообщение указывает на ошибку в полях для RuleIndex, AllowedClients, UnixReadOnly, UnixReadWrite, Nfsv3 и Nfsv4.

* Причина:   
Сбой запроса проверки входных данных для по крайней мере одно из следующих полей: RuleIndex, AllowedClients, UnixReadOnly, UnixReadWrite, Nfsv3 и Nfsv4.

* Решение   
Не забудьте установить все обязательные и не конфликтующие параметры в командной строке. Например нельзя задать параметры UnixReadOnly и UnixReadWrite в то же время.

* Возможное решение   
См. в разделе решения.  


<a name="error_09"></a>***Отсутствует значение для "{0}".***   
Эта ошибка указывает, что обязательный атрибут отсутствует в запросе, хотя бы для одного из следующих параметров: RuleIndex, AllowedClients, UnixReadOnly, UnixReadWrite, Nfsv3 и Nfsv4.

* Причина:   
Сбой запроса проверки входных данных для по крайней мере одно из следующих полей: RuleIndex, AllowedClients, UnixReadOnly, UnixReadWrite, Nfsv3 и Nfsv4.

* Решение

* Возможное решение   
См. в разделе решения.  


<a name="error_10"></a> ***{0} уже используется.***

* Причина:   
Вы пытаетесь создать том с именем, которое совпадает со значением существующего тома.

* Решение   
Используйте уникальное имя, при создании тома.

* Возможное решение   
При необходимости можно изменить имя существующий том, что новый том можно использовать указанное имя.

<a name="error_21"></a>***Значение, полученное для неизвестное свойство "{0}".***   
Эта ошибка возникает, когда несуществующего свойства предоставляются для ресурсов, таких как тома, моментальный снимок или целевой объект подключения.

* Причина:   
Запрос содержит набор свойств, которые могут использоваться с каждым ресурсом.  Не может содержать какие-либо несуществующего свойства в запросе.

* Решение   
Убедитесь, что имена всех свойств указаны правильно, и свойства доступны для подписки и ресурсов.

* Возможное решение

<a name="error_22"></a>***Операция обновления не поддерживается для этого типа ресурса.***   
Можно обновить только тома. Эта ошибка возникает при попытке выполнить операцию обновления не поддерживается, например, обновление моментального снимка.

* Причина:   
Ресурс, который вы пытаетесь провести обновление не поддерживает операции обновления.  Только тома, которые были изменены.

* Решение   
Отсутствует.  Ресурс, который вы пытаетесь провести обновление не поддерживает операции обновления. Таким образом его нельзя изменить.

* Возможное решение   
Для тома создайте новый ресурс с обновлением на месте и переноса данных.


<a name="error_23"></a>***Число элементов: {0} для объекта: {1} находится вне диапазона min-max.***   
Эта ошибка возникает, если правила политики экспорта не соответствует требованию вне допустимого диапазона.  Если вы определите политику экспорта, он должен иметь один экспорт правило политики, по меньшей мере и пять правил политики экспорта на максимально.

* Причина:   
Определенной политики экспорта не соответствует диапазона адресов.  

* Решение   
Убедитесь, что индекс уже не используется и который находится в диапазоне от 1 до 5.

* Возможное решение   
Это не обязательно использовать политику экспорта на томах. Таким образом политику экспорта можно опустить целиком в том случае, если необходимо иметь правила политики экспорта.


<a name="error_24"></a>***Повторяющееся значение ошибки для объекта {0}.***   
Эта ошибка возникает, если политику экспорта не определен с уникальным индексом.  При определении политики экспорта, все правила политики экспорта должна иметь уникальный индекс от 1 до 5.

* Причина:   
Политику определенных экспорта не соответствует требованиям для правил политики экспорта. Необходимо иметь правила политики один экспорт по меньшей мере и пять правил политики экспорта на максимально.  

* Решение   
Убедитесь, что индекс уже не используется, и что она находится в диапазоне от 1 до 5.

* Возможное решение   
Используйте другой индекс для правила, которое вы пытаетесь установить.

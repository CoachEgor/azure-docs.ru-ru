# <a name="azure-internet-analyzer-troubleshooting"></a>Устранение неполадок в Интернет-анализаторе Azure

Эта статья содержит этапы устранения неполадок для общих проблем Internet Analyzer.

## <a name="things-to-keep-in-mind"></a>Важные аспекты
- Скрипт клиента должен быть встроен в веб-сайт **HTTPS.** Измерения не будут собраны, если скрипт работает в простом тексте **(http://)** или локальном веб-сайте **(file://).**
- Данные измерений будут собираться только в том случае, если клиентский скрипт профиля Internet Analyzer был встроен в приложение, получающее реальный пользовательский трафик. Синтетический трафик (например, тесты производительности WebApp Azure) обычно не выполняет встроенный код Javascript, поэтому никакие измерения не будут сгенерированы таким типом трафика.

## <a name="azure-portal"></a>Портал Azure
**"Система показателей не была создана для выбранной комбинации фильтров" в разделе Scorecards**
- Счетные карточки генерируются ежедневно (в конце каждого дня, время UTC).
- Счетные карточки генерируются только в том случае, если было собрано более 100 измерений для выбранной комбинации фильтров (Test, Time Period, Country и т.д.).

**"Общий счет измерений" равен нулю для одной или обеих конечных точек в тесте**
- Временные ряды и количество измерений вычисляются один раз в час, так что вам нужно будет ждать, по крайней мере, такое количество времени для новых данных измерений, чтобы показать.
- Internet Analyzer подсчитывает только успешные измерения (т.е. ответы HTTP 200) для своего анализа. Если одна или обе конечные точки в тесте недоступны или возвращаются код не-200 HTTP, они будут отображаться с нулевыми общими измерениями.

# <a name="azure-internet-analyzer-troubleshooting"></a>Устранение неполадок Azure Internet Analyzer

Эта статья содержит инструкции по устранению распространенных проблем с анализатором Internet Analyzer.

## <a name="things-to-keep-in-mind"></a>Важные аспекты
- Клиентский сценарий должен быть внедрен в веб-сайт **HTTPS** . Измерения не будут собираться, если скрипт выполняется в виде открытого текста (**http://** ) или локального (**File://** ) веб-сайта.
- Данные измерений будут собираться только в том случае, если клиентский скрипт профиля Internet Analyzer внедрен в приложение, получающее реальный пользовательский трафик. Искусственный трафик (например, тесты производительности Azure WebApp) обычно не выполняет внедренный код JavaScript, поэтому никакие измерения не будут создаваться этим типом трафика.

## <a name="azure-portal"></a>Портал Azure
**"Для выбранной комбинации фильтра в разделе" системы показателей "не была создана система показателей**
- Системы показателей создаются ежедневно (в конце каждого дня, в формате UTC).
- Системы показателей создаются, только если для выбранной комбинации фильтров было собрано более 100 измерений (тест, период времени, страна и т. д.).

**"Общее количество измерений" равно нулю для одной или обеих конечных точек в тесте**
- Временные ряды и счетчики измерений вычисляются один раз в час, поэтому необходимо подождать хотя бы это время для отображения новых данных измерения.
- Анализатор Internet Analyzer считает только успешные измерения (т. е. ответы HTTP 200) для анализа. Если одна или обе конечные точки в тесте недостижимы или не возвращают код HTTP, отличный от 200, они будут отображаться с нулевым количеством измерений.

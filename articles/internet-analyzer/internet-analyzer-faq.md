# <a name="azure-internet-analyzer-faq-preview"></a>Лазурный анализатор Интернета часто задаваемые вопросы (Предварительный просмотр)

Это часто задаваемые вопросы для Azure Internet Analyzer- если у вас есть дополнительные вопросы, перейдите на [форум обратной связи](https://aka.ms/internetAnalyzerFeedbackForum) и опубликуйте свой вопрос. Когда часто задают вопрос, мы добавляем его в эту статью, чтобы его можно было найти быстро и легко.

## <a name="how-do-i-participate-in-the-preview"></a>Как принять участие в предварительном просмотре?

Предварительный просмотр доступен для выбора клиентов. Если вы заинтересованы в присоединении к предварительному просмотру, пожалуйста, сделайте следующее:

1. Войдите на [портал Azure](https://ms.portal.azure.com).
2. Перейдите на страницу **подписки.**
3. Нажмите на подписку Azure, с помощью которого вы планируете использовать Internet Analyzer.
4. Перейдите к настройкам **поставщиков ресурсов** для подписки.
5. Поиск для **Microsoft.Network** и нажмите на кнопку **Регистр** (или **перерегистрация)** кнопку.
![запрос доступа](./media/ia-faq/request-preview-access.png)

6. [Запросить одобрение,](https://aka.ms/internetAnalyzerContact) предоставив нам свой адрес электронной почты и идентификатор подписки Azure, который использовался для запроса доступа.
7. Как только ваш запрос будет одобрен, вы получите подтверждение по электронной почте и сможете создавать/обновлять/изменять ресурсы Internet Analyzer из недавно разрешенной подписки Azure.

## <a name="do-i-need-to-embed-the-client-to-run-a-test"></a>Нужно ли вставлять клиента для выполнения теста?

Да, клиент Internet Analyzer должен быть установлен в вашем приложении для сбора метрик, специфических для ваших пользователей. [Узнайте, как установить клиента.](internet-analyzer-embed-client.md) 

## <a name="do-i-get-billed-for-participating-in-the-preview"></a>Получу ли мне счет за участие в предварительном просмотре?
Нет, Анализатор Интернета Azure может использоваться в предварительном просмотре. Во время предварительного просмотра соглашения об уровне обслуживания не существует.

## <a name="what-scenarios-is-internet-analyzer-designed-to-address"></a>Какие сценарии предназначен для интернет-анализатора?

Internet Analyzer разработан, чтобы дать вам информацию о производительности сетей на основе пользователей. Чтобы помочь пользователям принимать оптимальные решения о производительности, Internet Analyzer сравнивает производительность двух конечных точек Интернета с использованием различных пользователей. Анализатор Интернета может ответить на множество вопросов. Некоторые из наиболее распространенных приведены ниже.

* Как миграция в облако сказывается на производительности? 
    * *Предлагаемый тест: Пользовательский (текущая инфраструктура помещения) против Azure (любая предварительно настроенная конечная точка)*
* Каковы преимущества размещения данных на граничном устройстве и в центре обработки данных? 
    *  *Предлагаемый тест: Azure vs. Azure Front Door, Azure vs. Azure CDN от Microsoft*
* Каково преимущество Azure Front Door в контексте производительности?
    *  *Предлагаемый тест: Пользовательские / Azure / CDN против Azure передняя дверь*
* Каково преимущество Azure CDN от Майкрософт в контексте производительности? 
    *  *Предлагаемый тест: Пользовательские / Azure / AFD против Azure CDN от Microsoft*
* Как работает Azure CDN от Майкрософт? 
    *  *Предлагаемый тест: Пользовательский (прочий конечный пункт CDN) против Azure CDN от Microsoft*
* Какое облако лучше подходит для пользователей в каждом регионе? 
    *  *Предлагаемый тест: Пользовательский (другой облачный сервис) против Azure (любая предварительно настроенная конечная точка)*

## <a name="which-tests-can-i-run-in-preview"></a>Какие тесты можно выполнить при предварительном просмотре?

Каждый тест, который вы создаете в Internet Analyzer, состоит из двух конечных точек: Endpoint A и Endpoint B. Любая из следующих комбинаций может быть запущена в виде тестов:  
* Две предварительно настроенные конечные точки,
* Один обычай и одна предварительно настроенная конечная точка, или
* Две [пользовательские конечные точки](internet-analyzer-custom-endpoint.md) (одна пользовательская конечная точка должна находиться в Azure).

При предварительном просмотре доступны следующие преднастроенные конечные точки:
* **Лазурные регионы**
    * Южная Бразилия
    * Центральная Индия
    * Центральная часть США
    * Восточная Азия
    * Восточная часть США
    * Западная Япония
    * Северная Европа
    * Северная часть ЮАР;
    * Юго-Восточная Азия
    * Северная часть ОАЭ;
    * западная часть Соединенного Королевства  
    * Западная Европа
    * западная часть США
    * западная часть США 2
* **Несколько комбинаций регионов Azure.**
    * Восточная часть США, Южная Бразилия.
    * Восточная часть США, Восточная Азия.
    * Западная Европа, Южная Бразилия.
    * Западная Европа, Юго-Восточная Азия.
    * Западная Европа, Северная часть ОАЭ.
    * Западная часть США, восточная часть США.
    * Западная часть США, Западная Европа.
    * Западная часть США, Северная часть ОАЭ.
    * Западная Европа, Северная часть ОАЭ, Юго-Восточная Азия.
    * Западная часть США, Западная Европа, Восточная Азия.
    * Западная часть США, Северная Европа, Юго-Восточная Азия, Северная часть ОАЭ, Северная часть ЮАР. 
* **Azure + Azure Front Door**, развернутые в одной или нескольких комбинациях регионов Azure, перечисленных выше.
* **Azure + Azure CDN от Майкрософт**, развернутые в одной комбинации регионов Azure, перечисленных выше.
* **Azure + диспетчер трафика Azure**, развернутые в нескольких комбинациях регионов Azure, перечисленных выше.

## <a name="how-is-internet-analyzer-different-from-other-monitoring-services-provided-by-azure"></a>Чем анализинтернет Интернета отличается от других служб мониторинга, предоставляемых Azure?

Internet Analyzer помогает вам понять производительность ваших конечных пользователей и помогает принимать решения для повышения их производительности. В то время как другие инструменты мониторинга Azure предоставляют информацию о ваших службах Azure, Internet Analyzer фокусируется на измерении сквозной производительности Интернета для пользователей.

## <a name="how-is-measurement-data-handled-by-internet-analyzer"></a>Как обрабатываются данные измерений Интернет-анализатором?

Azure применяет [надежные процедуры обеспечения безопасности и соответствует целому ряду стандартов](https://azure.microsoft.com/support/trust-center/). Доступ к данным будет только у вас и вашей команды. Сотрудникам Майкрософт может предоставляться ограниченный доступ к данным только при определенных условиях и с вашего ведома. Он зашифрован в пути и в покое.

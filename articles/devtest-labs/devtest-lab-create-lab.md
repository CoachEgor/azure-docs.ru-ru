# <a name="create-a-lab-in-azure-devtest-labs"></a>Создание лаборатории в лаборатории для разработки и тестирования Azure

Azure DevTest Labs — это инфраструктура, которая содержит группу ресурсов, таких как виртуальные машины, позволяющую более эффективно управлять этими ресурсами за счет установки ограничений и квот. В этой статье показано, как создать лабораторию с помощью портала Azure.

## <a name="prerequisites"></a>Предварительные условия

Чтобы создать лабораторию, вам потребуется:

* Подписка Azure. Дополнительные сведения о вариантах приобретения Azure см. на страницах [Как приобрести Azure](https://azure.microsoft.com/pricing/purchase-options/) и [Бесплатный ознакомительный период в один месяц](https://azure.microsoft.com/pricing/free-trial/). Для создания лаборатории необходимо быть владельцем подписки.

## <a name="get-started-with-azure-devtest-labs-in-minutes"></a>Быстрое начало работы с Azure DevTest Labs

Перейдя по следующей ссылке, вы перейдете на страницу портал Azure, которая позволит начать создание лабораторной работы в Azure DevTest Labs.

[Быстрое начало работы с Azure DevTest Labs](https://go.microsoft.com/fwlink/?LinkID=627034&clcid=0x409)

## <a name="fill-out-settings-for-your-new-account"></a>Заполните параметры для новой учетной записи

На странице **Создание DevTest Labs** заполните следующие параметры.

> [!TIP]
> В нижней части каждой страницы находится ссылка, позволяющая **скачать шаблон для автоматизации**.

### <a name="basic-settings"></a>Основные параметры

По умолчанию отображается вкладка **Основные параметры** . Заполните следующие значения:

|Имя|Описание|
|---|---|
|**Подписка** | Обязательный. Выберите **подписку** , которую необходимо связать с лабораторией.|
|**Группа ресурсов**| Обязательный. Введите **имя для группы ресурсов** лаборатории. Создайте новый, если он не существует.|
|**Имя лаборатории**| Обязательный. Введите **имя** лаборатории.|
|**Расположение**|Обязательный. Выберите расположение, в котором будет храниться лаборатория.|
|**Общедоступные среды**| См. раздел [Настройка и использование общедоступных сред](devtest-lab-configure-use-public-environments.md).

### <a name="auto-shutdown-settings"></a>Параметры автоматического завершения работы

Перейдите на страницу **автоматического завершения работы** , чтобы просмотреть ее параметры. Функция автоматического завершения работы позволяет автоматически завершать работу всех компьютеров в лаборатории в запланированное время каждый день.

На странице можно включить **Автоматическое завершение работы** и определить параметры для автоматического завершения работы всех виртуальных машин лаборатории. Компонент автоматического завершения работы предназначен главным образом для экономии затрат. При этом вы можете указать, когда следует автоматически завершать работу виртуальной машины. Вы можете изменить настройки автоматического завершения работы после создания лаборатории. Для этого нужно выполнить инструкции из руководства по [управлению всеми политиками для лаборатории в Azure DevTest Labs](./devtest-lab-set-lab-policy.md#set-auto-shutdown).

### <a name="networking"></a>Сетевое взаимодействие

При создании лаборатории будет создана сеть по умолчанию.

Перейдите на вкладку " **сеть** ", чтобы изменить или настроить параметр. Например, выберите существующую виртуальную сеть.

### <a name="tags"></a>Теги

Введите **имя** и **значение** для параметра **Теги**. Так вы сможете создать настраиваемые теги, добавляемые в каждый ресурс, который создается в лаборатории. Теги помогают администрировать и группировать ресурсы лаборатории по категориям. См. дополнительные сведения о тегах, включая способы [добавления тегов](devtest-lab-add-tag.md) после создания лаборатории.

### <a name="review-and-create"></a>Проверка и создание

После этого нажмите кнопку **создать**. Вы можете отслеживать состояние процесса создания лаборатории, просмотрев область **уведомления** в правом верхнем углу страницы портала. 

## <a name="completed-the-creation"></a>Создание завершено

После завершения работы кнопка **Переход к ресурсу** появляется в нижней части страницы и в окне уведомления. Можно также обновить страницу **DevTest Labs**, чтобы увидеть только что созданную лабораторию в списке лабораторий.  

Нажмите кнопку **"переход к ресурсу** ", и вы увидите на домашней странице новой учетной записи DevTest Labs.

Вы также можете искать **DevTest Labs** в портал Azure. Выберите новую учетную запись из списка и перейдите на домашнюю страницу.

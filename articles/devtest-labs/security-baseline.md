# <a name="azure-security-baseline-for-azure-devtest-labs"></a>Базовый план безопасности Azure для Azure DevTest Labs

Базовый план безопасности Azure для Azure DevTest Labs содержит рекомендации, которые помогут повысить уровень безопасности развертывания.

Базовая информация для этой службы взята из [теста безопасности Azure версии 1,0](https://docs.microsoft.com/azure/security/benchmarks/overview), который содержит рекомендации по обеспечению безопасности облачных решений в Azure.

Дополнительные сведения см. в статье [Обзор базовой конфигурации безопасности Azure](https://docs.microsoft.com/azure/security/benchmarks/security-baselines-overview).

## <a name="logging-and-monitoring"></a>Ведение журналов и мониторинг

*Дополнительные сведения см. в статье [Управление безопасностью: ведение журналов и мониторинг](https://docs.microsoft.com/azure/security/benchmarks/security-control-logging-monitoring).*

### <a name="21-use-approved-time-synchronization-sources"></a>2.1. Использование утвержденных источников синхронизации времени
**Руководство.** Корпорация Майкрософт хранит источники времени для ресурсов Azure. Однако вы можете управлять параметрами синхронизации времени для ресурсов вычислений.

Сведения о настройке синхронизации времени для ресурсов Azure см. в следующей статье. [Синхронизация времени для виртуальных машин Windows в Azure](https://docs.microsoft.com/azure/virtual-machines/windows/time-sync). 

**Мониторинг центра безопасности Azure:** Сейчас недоступно

**Ответственность:** NNTP

### <a name="22-configure-central-security-log-management"></a>2.2. Настройка централизованного управления журналами безопасности
**Руководство.** Включите параметры диагностики журнала действий Azure и отправьте журналы в рабочую область Log Analytics, в концентратор событий Azure или в учетную запись хранения Azure для архивации. Журналы действий позволяют получить представление об операциях, выполненных на экземплярах Azure DevTest Labs на уровне плоскости управления. С помощью данных журнала действий Azure можно определить "что, кто и когда" для любых операций записи (размещение, публикация, удаление), выполненных на уровне плоскости управления для экземпляров DevTest Labs.

Дополнительные сведения см. [в разделе Создание параметров диагностики для отправки журналов платформы и метрик в разные места назначения](../azure-monitor/platform/diagnostic-settings.md).

**Мониторинг центра безопасности Azure:** Сейчас недоступно

**Ответственность:** Пользовательских

### <a name="23-enable-audit-logging-for-azure-resources"></a>2.3. Включение журналов аудита для ресурсов Azure
**Руководство.** Включите параметры диагностики журнала действий Azure и отправьте журналы в рабочую область Log Analytics, в концентратор событий Azure или в учетную запись хранения Azure для архивации. Журналы действий позволяют получить представление об операциях, выполненных на экземплярах Azure DevTest Labs на уровне плоскости управления. С помощью данных журнала действий Azure можно определить "что, кто и когда" для любых операций записи (размещение, публикация, удаление), выполненных на уровне плоскости управления для экземпляров DevTest Labs.

Дополнительные сведения см. [в разделе Создание параметров диагностики для отправки журналов платформы и метрик в разные места назначения](../azure-monitor/platform/diagnostic-settings.md).

**Мониторинг центра безопасности Azure:** Сейчас недоступно

**Ответственность:** Пользовательских

### <a name="24-collect-security-logs-from-operating-systems"></a>2.4. Сбор журналов безопасности из операционных систем
**Руководство.** Azure DevTest Labs виртуальные машины создаются и принадлежат клиенту. Таким образом, ответственность за ее мониторинг несет Организация. С помощью центра безопасности Azure можно отслеживать операционную систему. Данные, собираемые центром безопасности из операционной системы, включают в себя тип ОС и версию, ОПЕРАЦИОНную систему (журналы событий Windows), запущенные процессы, имя компьютера, IP-адреса и вошедший в систему пользователь. Агент Log Analytics также собирает файлы аварийного дампа.

См. сведения в следующих статьях: 

- [Как получить журналы внутреннего узла виртуальной машины Azure с помощью Azure Monitor](../azure-monitor/learn/quick-collect-azurevm.md)
- [Общие сведения о сборе данных в центре безопасности Azure](../security-center/security-center-enable-data-collection.md)

**Мониторинг центра безопасности Azure:** Да

**Ответственность:** Пользовательских

### <a name="25-configure-security-log-storage-retention"></a>2.5. Настройка хранения журнала безопасности
***Руководство.** В Azure Monitor задайте срок хранения журнала для рабочих областей Log Analytics, связанных с экземплярами Azure DevTest Labs в соответствии с правилами соответствия вашей организации.

Дополнительные сведения см. в следующей статье: [как задать параметры хранения журнала](../azure-monitor/platform/manage-cost-storage.md#change-the-data-retention-period) .

**Мониторинг центра безопасности Azure:** Неприменимо

**Ответственность:** Пользовательских

### <a name="26-monitor-and-review-logs"></a>2.6. Мониторинг и просмотр журналов
**Руководство.** Включите параметры диагностики журнала действий Azure и отправьте журналы в рабочую область Log Analytics. Выполнение запросов в Log Analytics для поиска терминов, выявления тенденций, анализа закономерностей и предоставления многих других аналитических сведений на основе данных журнала действий, которые могли быть собраны для Azure DevTest Labs.

См. сведения в следующих статьях:

- [Как включить параметры диагностики для журнала действий Azure](../azure-monitor/platform/diagnostic-settings.md)
- [Получение и анализ журналов действий Azure в Log Analytics рабочей области в Azure Monitor](../azure-monitor/platform/activity-log.md)

**Мониторинг центра безопасности Azure:** Неприменимо

**Ответственность:** Пользовательских

### <a name="27-enable-alerts-for-anomalous-activity"></a>2.7. Включение оповещений об аномальных действиях
**Руководство.** Используйте рабочую область Azure Log Analytics для мониторинга и оповещения о аномальных действиях в журналах безопасности и событиях, связанных с Azure DevTest Labs.

Дополнительные сведения см. в следующей статье: [как предупредить о данных журнала Log Analytics](../azure-monitor/learn/tutorial-response.md)

**Мониторинг центра безопасности Azure:** Сейчас недоступно

### <a name="210-enable-command-line-audit-logging"></a>2.10. Включение ведения журнала аудита для командной строки
**Руководство.** Azure DevTest Labs создает компьютеры вычислений Azure, которые принадлежат и управляются клиентом. Используйте Microsoft Monitoring Agent на всех поддерживаемых виртуальных машинах Windows Azure для регистрации события создания процесса и поля командной строки. Для поддерживаемых виртуальных машин Linux в Azure можно вручную настроить ведение журнала консоли для каждого узла и использовать syslog для хранения данных. Кроме того, используйте рабочую область Log Analytics Azure Monitor, чтобы просматривать журналы и выполнять запросы к данным журнала с виртуальных машин Azure.

- [Сбор данных в центре безопасности Azure](../security-center/security-center-enable-data-collection.md#data-collection-tier)
- [Как выполнять пользовательские запросы в Azure Monitor](../azure-monitor/log-query/get-started-queries.md)
- [Источники данных системного журнала в Azure Monitor](../azure-monitor/platform/data-sources-syslog.md)

**Мониторинг центра безопасности Azure:** Да

**Ответственность:** Пользовательских

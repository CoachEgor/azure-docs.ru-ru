# <a name="tutorial-create-a-host-pool-by-using-the-azure-marketplace"></a>Руководство. Создание пула узлов с помощью Azure Marketplace

В этом руководстве вы узнаете, как создать пул узлов в клиенте виртуальных рабочих столов Windows с помощью предложения Microsoft Azure Marketplace.

Пулы узлов — это коллекция, состоящая из одной или нескольких идентичных виртуальных машин в средах клиента Виртуального рабочего стола Windows. Каждый пул узлов может содержать группы приложений, с которыми пользователи могут взаимодействовать, как с физическим рабочим столом.

В этом руководстве рассматриваются следующие задачи.

> [!div class="checklist"]
>
> * Создание пула узла на Виртуальном рабочем столе Windows.
> * Создание группы ресурсов с виртуальными машинами в подписке Azure.
> * Присоединение виртуальных машин к домену Active Directory.
> * Регистрация виртуальных машин с помощью Виртуального рабочего стола Windows.

## <a name="prerequisites"></a>предварительные требования

* Клиент в виртуальном рабочем столе. В предыдущем [руководстве](tenant-setup-azure-active-directory.md) создается клиент.
* [Модуль PowerShell для виртуальных рабочих столов Windows](/powershell/windows-virtual-desktop/overview/).

Получив этот модуль, выполните следующий командлет, чтобы войти в учетную запись:

```powershell
Add-RdsAccount -DeploymentUrl "https://rdbroker.wvd.microsoft.com"
```

## <a name="sign-in-to-azure"></a>Вход в Azure

Войдите на [портал Azure](https://portal.azure.com).

## <a name="run-the-azure-marketplace-offering-to-provision-a-new-host-pool"></a>Запустите предложение Azure Marketplace для подготовки пула узлов.

Запустите предложение Azure Marketplace для подготовки пула узлов:

1. На **домашней странице** или в меню портала Azure выберите **Создать ресурс**.
1. В окне поиска Marketplace введите **Виртуальный рабочий стол Windows**.
1. Выберите **Windows Virtual Desktop — Provision a host pool** (Виртуальный рабочий стол Windows — подготовка пула узлов) и щелкните **Создать**.

После этого следуйте инструкциям в следующем разделе, чтобы ввести сведения для соответствующих вкладок.

### <a name="basics"></a>Основы

Вот что вы можете сделать на вкладке " **основные** ":

1. Выберите **подписку**.
1. В поле **Группа ресурсов**выберите **создать** и укажите имя новой группы ресурсов.
1. Выберите **регион**.
1. Введите имя пула узлов, уникальное в клиенте виртуальных рабочих столов Windows.
1. Выберите **тип рабочего стола**. Если выбрать **личное**, каждый пользователь, который подключается к этому пулу узлов, будет окончательно назначен виртуальной машине.
1. Введите пользователей, которые могут входить в клиенты виртуальных рабочих столов Windows и получить доступ к рабочему столу. Используйте список с разделителями-запятыми. Например, если требуется назначить `user1@contoso.com` и `user2@contoso.com` доступа, введите *`user1@contoso.com,user2@contoso.com`*
1. В поле **расположение метаданных службы**выберите то же расположение, что и у виртуальной сети, имеющей подключение к серверу Active Directory.

   >[!IMPORTANT]
   >Если вы используете чистое решение Azure Active Directory доменных служб (Azure AD DS) и Azure Active Directory (Azure AD), не забудьте развернуть пул узлов в том же регионе, что и AD DS Azure, чтобы избежать ошибок подключения к домену и учетных данных.

1. Нажмите кнопку **Далее: Настройка виртуальных машин**.

### <a name="configure-virtual-machines"></a>Настройка виртуальных машин

Для вкладки **Настройка виртуальных машин** выполните следующие действия.

1. Либо примите значения по умолчанию, либо настройте число и размер виртуальных машин.

    >[!NOTE]
    >Если конкретный размер виртуальной машины, который вы ищете, не отображается в селекторе размера, это связано с тем, что мы еще не перешли на средство Azure Marketplace. Чтобы запросить размер, создайте запрос или исголосуйте существующий запрос на [форуме UserVoice для виртуальных рабочих столов Windows](https://windowsvirtualdesktop.uservoice.com/forums/921118-general).

1. Введите префикс для имен виртуальных машин. Например, если ввести *префикс*, виртуальные машины будут называться **префиксом-0**, **prefix-1**и т. д.
1. Нажмите кнопку **Далее: параметры виртуальной машины**.

### <a name="virtual-machine-settings"></a>Параметры виртуальной машины

Для вкладки **Параметры виртуальной машины** выполните следующие действия.

1. Выберите **Источник образа** и введите соответствующую информацию о том, как его искать и хранить. Параметры различаются для хранилища BLOB-объектов, управляемого образа и коллекции.

   Если вы решили не использовать управляемые диски, выберите учетную запись хранения, содержащую *VHD* -файл.
1. Введите имя участника-пользователя и пароль. Эта учетная запись должна быть учетной записью домена, которая будет присоединять виртуальные машины к домену Active Directory. Эти же имя пользователя и пароль будут созданы на виртуальных машинах в качестве локальной учетной записи. Позже вы сможете сбросить эти локальные учетные записи.

   >[!NOTE]
   > Если вы присоединяете виртуальные машины к среде Azure AD DS, убедитесь, что пользователь, присоединяемый к домену, является членом [группы администраторов контроллера домена AAD](../active-directory-domain-services/tutorial-create-instance-advanced.md#configure-an-administrative-group).
   >
   > Учетная запись также должна входить в управляемый домен Azure AD DS или клиент Azure AD. Учетные записи из внешних каталогов, связанных с клиентом Azure AD, не смогут пройти проверку подлинности в процессе присоединения к домену.

1. Выберите **виртуальную сеть** с подключением к серверу Active Directory, а затем выберите подсеть для размещения виртуальных машин.
1. Выберите **Далее: сведения о виртуальном рабочем столе Windows**.

### <a name="windows-virtual-desktop-tenant-information"></a>Сведения о клиенте Виртуального рабочего стола Windows

Для вкладки **сведения о клиенте виртуальных рабочих столов Windows** выполните следующие действия.

1. Введите **имя группы клиента Виртуального рабочего стола Windows** в группе вашего клиента. Оставьте значение по умолчанию, если вам не было предоставлено определенное имя группы клиента.
1. В поле **Windows Virtual Desktop tenant name** (Имя клиента Виртуального рабочего стола Windows) введите имя клиента, в котором будет создан этот пул узлов.
1. Укажите тип учетных данных, которые вы хотите использовать для аутентификации в качестве владельца служб удаленных рабочих столов клиента Виртуального рабочего стола Windows. Введите имя участника-пользователя или субъект-службу и пароль.

   Если вы завершили учебник [Создание субъектов-служб и назначений ролей с помощью PowerShell](./create-service-principal-role-powershell.md), выберите **Субъект-служба**.

1. В качестве **субъекта-службы**для **идентификатора клиента Azure AD**введите учетную запись администратора клиента для экземпляра Azure AD, содержащего субъект-службу. Поддерживаются только субъекты-службы с использованием пароля.
1. Нажмите кнопку **Далее: Проверка и создание**.

## <a name="complete-setup-and-create-the-virtual-machine"></a>Завершение настройки и создание виртуальной машины

На странице **Обзор и создание**проверьте сведения о настройке. Если необходимо изменить что-либо, вернитесь назад и внесите необходимые изменения. Когда будете готовы, выберите **создать** , чтобы развернуть пул узлов.

В зависимости от количества виртуальных машин, которые вы создаете, этот процесс может занять 30 минут или более.

>[!IMPORTANT]
> Для защиты среды виртуальных рабочих столов Windows в Azure рекомендуется не открывать входящий порт 3389 на виртуальных машинах. Виртуальному рабочему столу Windows не требуется открытый входящий порт 3389, чтобы пользователи могли получить доступ к виртуальным машинам пула узлов.
>
> Если необходимо открыть порт 3389 для устранения неполадок, рекомендуется использовать JIT-доступ. Дополнительные сведения см. в статье [Защита портов управления с помощью JIT-доступа](../security-center/security-center-just-in-time.md).

## <a name="optional-assign-additional-users-to-the-desktop-application-group"></a>(Необязательно.) Назначение дополнительным пользователям группы классических приложений

После того как Azure Marketplace создаст пул, можно назначить дополнительных пользователей группе приложений для настольных систем. Если вы не хотите добавлять дополнительные сведения, пропустите этот раздел.

Чтобы назначить пользователей группе приложений для настольных систем:

1. Откройте окно PowerShell.

1. Выполните следующую команду, чтобы войти в среду виртуальных рабочих столов Windows:

   ```powershell
   Add-RdsAccount -DeploymentUrl "https://rdbroker.wvd.microsoft.com"
   ```

1. Добавьте пользователей в группу приложений для настольных компьютеров с помощью следующей команды:

   ```powershell
   Add-RdsAppGroupUser <tenantname> <hostpoolname> "Desktop Application Group" -UserPrincipalName <userupn>
   ```

   Имя участника-пользователя должно совпадать с удостоверением пользователя в Azure AD, например *user1@contoso.com* . Если вы хотите добавить нескольких пользователей, выполните команду для каждого пользователя.

Пользователи, добавляемые в группу приложений для настольных систем, могут входить в виртуальные рабочие столы Windows с поддерживаемыми удаленный рабочий стол клиентами и просматривать ресурс для рабочего стола сеанса.

Поддерживаемые клиенты:

* [клиент удаленного рабочего стола для Windows 7 и Windows 10](connect-windows-7-and-10.md);
* [веб-клиент Виртуального рабочего стола Windows](connect-web.md).

# <a name="troubleshoot-windows-7-virtual-machines-in-windows-virtual-desktop"></a>Устранение неполадок виртуальных машин Windows 7 в виртуальном рабочем столе Windows

Используйте эту статью для устранения неполадок, возникающих при настройке виртуальных машин узла сеансов виртуальных рабочих столов Windows.

## <a name="known-issues"></a>Известные проблемы

Windows 7 на виртуальных рабочих столах Windows не поддерживает следующие функции:

- Виртуализированные приложения (RemoteApp)
- Перенаправление часового пояса
- Автоматическое масштабирование DPI

Виртуальные рабочие столы Windows могут виртуализировать только все настольные компьютеры под Windows 7.

В то время как автоматическое масштабирование DPI не поддерживается, можно вручную изменить разрешение на виртуальной машине, щелкнув значок в клиенте удаленный рабочий стол правой кнопкой мыши и выбрав **разрешение**.

## <a name="error-cant-access-the-remote-desktop-user-group"></a>Ошибка: не удается получить доступ к группе пользователей удаленный рабочий стол

Если виртуальному рабочему столу Windows не удается найти учетные данные пользователя или пользователя в группе удаленный рабочий стол, может появиться одно из следующих сообщений об ошибке:

- "Этот пользователь не является членом удаленный рабочий стол группы пользователей"
- "Необходимо предоставить разрешения на вход с помощью службы удаленных рабочих столов"

Чтобы устранить эту ошибку, добавьте пользователя в удаленный рабочий стол группу пользователей:

1. Перейдите на портал Azure.
2. Выберите виртуальную машину, на которой вы видели сообщение об ошибке.
3. Выберите **выполнить команду**.
4. Выполните следующую команду с `<username>` замените именем пользователя, которого нужно добавить:
   
   ```cmd
   net localgroup "Remote Desktop Users" <username> /add
   ```

# <a name="troubleshoot-windows-7-virtual-machines-in-windows-virtual-desktop"></a>Устранение неполадок с виртуальной машиной Windows 7 в службе "Виртуальный рабочий стол Windows"

Используйте эту статью для устранения неполадок при настройке виртуальных компьютеров (VM) с настройкой виртуальных компьютеров для настольных компьютеров Windows Virtual Desktop.

## <a name="known-issues"></a>Известные проблемы

Windows 7 на Windows Virtual Desktops не поддерживает следующие функции:

- Виртуализированные приложения (RemoteApps)
- Перенаправление часового пояса
- Автоматическое масштабирование ДОИ

Windows Virtual Desktop может виртуализировать только полные рабочие столы для Windows 7.

В то время как автоматическое масштабирование DPI не поддерживается, вы можете вручную изменить разрешение на вашей виртуальной машине, нажав правой кнопкой кнопку значок в клиенте удаленного рабочего стола и выбрав **Разрешение.**

## <a name="error-cant-access-the-remote-desktop-user-group"></a>Ошибка: не уможет получить доступ к группе пользователей удаленного рабочего стола

Если Windows Virtual Desktop не может найти учетные данные пользователей в группе пользователей удаленного рабочего стола, вы можете увидеть одно из следующих сообщений об ошибках:

- "Этот пользователь не является членом группы пользователей удаленного рабочего стола"
- "Вы должны быть предоставлены разрешения на входе через удаленные настольные услуги"

Чтобы исправить эту ошибку, добавьте пользователя в группу пользователей удаленного рабочего стола:

1. Откройте портал Azure.
2. Выберите виртуальную машину, на которую вы видели сообщение об ошибке.
3. Выберите **выполнить команду**.
4. Выполнить следующую `<username>` команду с заменой имени пользователя, которое вы хотите добавить:
   
   ```cmd
   net localgroup "Remote Desktop Users" <username> /add
   ```

# <a name="windows-virtual-desktop-environment"></a>Среда Виртуального рабочего стола Windows

Windows Virtual Desktop — это сервис, который предоставляет пользователям легкий и безопасный доступ к виртуализированным рабочим столам и удаленным приложениям. Эта тема расскажет вам немного больше об общей структуре среды Windows Virtual Desktop.

## <a name="tenants"></a>Клиенты

Арендатор Windows Virtual Desktop является основным интерфейсом для управления средой Windows Virtual Desktop. Каждый арендатор Windows Virtual Desktop должен быть связан с active-каталогом Azure Active, содержащим пользователей, которые войдут в среду. С арендатора Windows Virtual Desktop можно начать создавать пулы хостов для выполнения рабочих нагрузок пользователей.

## <a name="host-pools"></a>Бассейны для хозяев

Пул хоста представляет собой набор виртуальных машин Azure, которые регистрируются на Windows Virtual Desktop в качестве хостов сеансов при запуске агента Windows Virtual Desktop. Все виртуальные машины хоста сеансов в пуле хоста должны быть получены из одного и того же изображения для согласованного пользовательского интерфейса.

Бассейн хоста может быть одним из двух типов:

- Личный, где каждый хост сеанса назначается отдельным пользователям.
- Пул, где хосты сеансов могут принимать соединения от любого пользователя, уполномоченного группе приложений в пуле хоста.

Вы можете установить дополнительные свойства в пуле хоста, чтобы изменить его поведение балансировки нагрузки, сколько сеансов, которые может пройти каждый хост сеанса, и что пользователь может сделать для сеанса хостов в пуле хоста при входе в их сеансы Windows Virtual Desktop. Вы управляете ресурсами, публикуемыми пользователями через группы приложений.

## <a name="app-groups"></a>Группы приложений

Группа приложений представляет собой логическую группу приложений, установленных на хостах сеансов в пуле узлов. Группа приложений может быть одним из двух типов:

- RemoteApp, где пользователи получают доступ к удаленным приложениям, которые вы индивидуально выбираете и публикуете в группе приложений
- Рабочий стол, где пользователи получают доступ к полному рабочему столу

По умолчанию группа настольных приложений (названная "Desktop Application Group") автоматически создается при создании пула хостов. Вы можете удалить эту группу приложений в любое время. Тем не менее, вы не можете создать другую группу настольных приложений в пуле хостов, пока существует группа настольных приложений. Чтобы опубликовать RemoteApps, необходимо создать группу приложений RemoteApp. Можно создать несколько групп приложений RemoteApp для размещения различных сценариев работы. Различные группы приложений RemoteApp также могут содержать перекрывающиеся удаленные приложения.

Чтобы публиковать ресурсы пользователям, необходимо назначить их группам приложений. При назначении пользователей группам приложений следует учитывать следующие вещи:

- Пользователь не может быть назначен как группе настольного приложения, так и группе приложений RemoteApp в одном пуле хостов.
- Пользователь может быть назначен нескольким группам приложений в одном пуле хоста, и его канал будет накапливаться обеими группами приложений.

## <a name="tenant-groups"></a>Группы арендаторов

В Windows Virtual Desktop, Windows Virtual Desktop арендатора, где большинство настройки и конфигурации происходит. Арендатор Windows Virtual Desktop содержит пулы хостов, группы приложений и назначения пользователей группы приложений. Однако могут возникнуть определенные ситуации, когда вам необходимо управлять несколькими арендаторами Windows Virtual Desktop одновременно, особенно если вы являетесь поставщиком облачных услуг (CSP) или хостинг-партнером. В таких ситуациях можно использовать пользовательскую группу клиентов Windows Virtual Desktop для размещения клиентов windows Virtual Desktop и централизованного управления доступом. Однако, если вы управляете только одним арендатором Windows Virtual Desktop, концепция группы клиентов не применяется, и вы можете продолжать работать и управлять вашим арендатором, который существует в группе клиентов по умолчанию.

## <a name="end-users"></a>конечные пользователи;

После того как вы назначили пользователей в свои группы приложений, они могут подключиться к развертыванию Виртуального рабочего стола Windows с любым из клиентов Windows Virtual Desktop.

# <a name="host-pool-load-balancing-methods"></a>Методы балансировки нагрузки пула узлов

Обзор виртуального рабочего стола Windows поддерживает два метода балансировки нагрузки. Каждый метод определяет, какой узел сеансов будет размещаться сеанса пользователя при подключении к ресурсу в пуле узлов.

В виртуальный рабочий стол Windows доступны следующие методы балансировки нагрузки:

- Балансировка нагрузки по ширине позволяет равномерно распределить пользовательских сеансов между узлами в пуле узлов сеанса.
- Балансировка нагрузки преимущественно в глубину позволяет насытить узла сеансов с помощью пользовательских сеансов в пуле узлов. После первого сеанса достигает предела порогового значения сеанса, подсистема балансировки нагрузки направляет новые подключения пользователя к следующего узла сеансов в пуле узлов, пока не достигнет предела и т. д.

Каждый пул узлов можно настроить только один тип балансировки нагрузки относящиеся к ней. Тем не менее оба балансировки нагрузки методы совместно используют следующие поведения независимо от размещения пула в которых они расположены:

- Если у пользователя уже имеется сеанс в пуле узлов и подключается к этому сеансу, подсистемы балансировки нагрузки успешно перенаправит их на узел сеанса с существующим сеансам. Это поведение применяется, даже если свойство AllowNewConnections этого узла сеанса имеет значение False.
- Если у пользователя нет уже сеанса в пуле узлов, подсистемы балансировки нагрузки не рекомендуется узлах сеансов во время балансировки нагрузки, свойство которого AllowNewConnections имеет значение False.

## <a name="breadth-first-load-balancing-method"></a>Метод балансировки нагрузки по ширине

Метод балансировки нагрузки по ширине позволяет распространять пользовательских соединений оптимизировать для этого сценария. Этот метод идеально подходит для организаций, которым требуется обеспечить удобство работы для пользователей, подключающихся к их в составе пула среде виртуального рабочего стола.

Метод преимущественно сначала запрашивает узлах сеансов, которые позволяют новые подключения. Этот метод выбирает затем узла сеансов с наименьшим числом сеансов. Если имеется набор функций, этот метод выбирает первый узел сеанса в запросе.

## <a name="depth-first-load-balancing-method"></a>Метод балансировки нагрузки преимущественно в глубину

Метод балансировки нагрузки преимущественно в глубину позволяет насытить одного узла сеансов за раз, чтобы оптимизировать для этого сценария. Этот метод является идеальным для стороннюю организаций, желающих более детальный контроль над число виртуальных машин, которые они выделено узлов в пуле.

Метод преимущественно в глубину сначала запрашивает узлах сеансов, которые разрешить новым подключениям и еще не прошли его превышении максимальное число сеансов. Затем этот метод выбирает узла сеансов с наибольшим количеством сеансов. Если имеется набор функций, этот метод выбирает первый узел сеанса в запросе.

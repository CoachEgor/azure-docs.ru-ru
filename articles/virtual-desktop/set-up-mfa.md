# <a name="set-up-azure-multi-factor-authentication"></a>Настройка многофакторной аутентификации Azure

Клиент Windows для Windows Virtual Desktop является отличным вариантом для интеграции Windows Virtual Desktop с локальной машиной. Однако при настройке учетной записи Windows Virtual Desktop в клиент Windows необходимо принять определенные меры, чтобы обезопасить себя и своих пользователей.

При первом входе в систему клиент запрашивает ваше имя пользователя, пароль и MFA Azure. После этого при следующем входе в систему клиент запомнит ваш токен из корпоративного приложения Azure Active Directory (AD). Когда вы выбираете **Помните меня,** ваши пользователи могут войти в систему после перезагрузки клиента без необходимости повторно ввести свои учетные данные.

Хотя запоминание учетных данных удобно, это также может сделать развертывание в сценариях Enterprise или персональных устройствах менее безопасным. Чтобы защитить пользователей, необходимо убедиться, что клиент продолжает запрашивать учетные данные Azure Multi-Factor Authentication (MFA). В этой статье будет показан анакгия политики условного доступа для Windows Virtual Desktop для включения этой настройки.

## <a name="prerequisites"></a>Предварительные требования

Вот что вам нужно, чтобы начать работу:

- Назначьте всем пользователям одну из следующих лицензий:
  - Microsoft 365 E3 или E5
  - Активный каталог Azure Premium P1 или P2
  - Корпоративная мобильность - Безопасность E3 или E5
- Группа Active Directory Azure с пользователями, назначенными в качестве членов группы.
- Включите Azure MFA для всех пользователей. Для получения дополнительной информации о том, как это сделать, см. [Как потребовать двухэтапной проверки для пользователя](/active-directory/authentication/howto-mfa-userstates).

>[!NOTE]
>Следующие настройки также относится к [Windows Virtual Desktop веб-клиента.](https://rdweb.wvd.microsoft.com/webclient/index.html)

## <a name="opt-in-to-the-conditional-access-policy"></a>Займите политику условного доступа

1. Открыть **активный каталог Azure**.

2. Перейдите на вкладку **"Все приложения".** В меню «Тип приложения» выберите **корпоративные приложения,** затем ищите **windows Virtual Desktop Client.**

    ![Скриншот вкладки «Все приложения». Пользователь ввел "окна виртуального настольного клиента" в панель поиска, и приложение появилось в результатах поиска.](media/all-applications-search.png)

3. Выберите **Условный доступ**.

    ![Скриншот, показывающий, как пользователь парит курсор мыши над вкладкой «Условный доступ».](media/conditional-access-location.png)

4. Выберите **новую политику**.

   ![Скриншот страницы условного доступа. Пользователь зависает над новой кнопкой политики.](media/new-policy-button.png)

5. Введите **имя** для **правила,** затем **выберите** имя **группы,** созданной в предварительных условиях.

6. **Выберите,** затем выберите **Done**.

7. Далее откройте **облачные приложения или действия.**

8. На панели **Select** выберите приложение **Windows Virtual Desktop** Enterprise.

    ![Скриншот страницы приложений или действий облака. Пользователь выбрал приложение Windows Virtual Desktop, выбрав рядом с ним контрольный знак. Выбранное приложение выделено красным цветом.](media/cloud-apps-select.png)
    
    >[!NOTE]
    >Вы также должны увидеть приложение Windows Virtual Desktop Client, выбранное на левой стороне экрана, как показано на следующем изображении. Для работы политики нужны как Windows Virtual Desktop, так и windows Virtual Desktop Client Enterprise.
    >
    > ![Скриншот страницы приложений или действий облака. Приложения Windows Virtual Desktop и Windows Virtual Desktop Client выделены красным цветом.](media/cloud-apps-enterprise-selected.png)

9. **Выберите**

10. Далее, открытый **Грант** 

11. Выберите **Требуется многофакторная аутентификация,** а затем выберите **Требуют одного из выбранных элементов управления.**
   
    ![Скриншот страницы Гранта. Выбрана «требует многофакторная аутентификация».](media/grant-page.png)

    >[!NOTE]
    >Если в вашей организации есть устройства, зарегистрированные на MDM, и вы не хотите, чтобы они отображали запрос МИД, вы также можете выбрать **устройство Require, которое должно быть помечено как соответствующее.**

12. Выберите **сессию**.

13. Установите **частоту ввоза** на **Active,** затем измените ее значение до **1 часа.**

    ![Скриншот страницы сессии. Меню сеансов показывает, что меню частоты выпадения с частоты входе были изменены на "1" и "Часы".](media/sign-in-frequency.png)
   
    >[!NOTE]
    >Активные сеансы в среде Windows Virtual Desktop будут продолжать работать при изменении политики. Однако, если вы отключите или подпишите, вам нужно будет предоставить свои учетные данные снова через 60 минут. При изменении настроек можно продлить период тайм-аута столько, сколько захочется (до тех пор, пока он соответствует политике безопасности организации).
    >
    >Настройка по умолчанию представляет собой окно в 90 дней, что означает, что клиент будет просить пользователей войти снова, когда они пытаются получить доступ к ресурсу после того, как неактивны на своей машине в течение 90 дней или дольше.

14. Включить политику.

15. Выберите **Создать** для подтверждения политики.

Готово! Не стесняйтесь протестировать политику, чтобы убедиться, что ваш список разрешений работает по назначению.

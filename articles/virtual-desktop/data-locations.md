# <a name="data-locations-for-windows-virtual-desktop"></a>Местоположение данных для виртуального рабочего стола Windows

Windows Virtual Desktop в настоящее время доступна для всех географических местоположений. Первоначально метаданные службы могут храниться только в географии США (США). Администраторы могут выбрать место для хранения пользовательских данных при создании виртуальных компьютеров и связанных с ними служб хранения, таких как файловые серверы. Узнайте больше о географических регионах Azure на [карте центра обработки данных Azure.](https://azuredatacentermap.azurewebsites.net/)

>[!NOTE]
>Корпорация Майкрософт не контролирует и не ограничивает регионы, в которых вы или пользователи могут получить доступ к данным пользователя и приложения.

>[!IMPORTANT]
>Windows Virtual Desktop хранит глобальную информацию о метаданных, такие как имена арендаторов, имена пулов, имена групп приложений и имена основных пользователей в центре обработки данных, расположенном в США. Сохраненные метаданные шифруются в состоянии покоя, а георезервные зеркала сохраняются на территории Соединенных Штатов. Все данные клиентов, такие как настройки приложения и пользовательские данные, находятся в месте, которое выбирает клиент, и не управляются службой.

Метаданные службы реплицируются в Соединенных Штатах для целей аварийного восстановления.

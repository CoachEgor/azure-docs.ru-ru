# <a name="data-locations-for-windows-virtual-desktop"></a>Расположения данных для виртуального рабочего стола Windows

>[!IMPORTANT]
>Это содержимое применимо к обновлению за весну 2020 года с объектами Azure Resource Manager для Виртуального рабочего стола Windows. Если вы используете выпуск Виртуального рабочего стола Windows за осень 2019 года без объектов Azure Resource Manager, см. [эту статью](./virtual-desktop-fall-2019/data-locations-2019.md).
>
> Обновление Виртуального рабочего стола Windows за весну 2020 года пока предоставляется как общедоступная предварительная версия. без соглашений об уровне обслуживания. Мы не рекомендуем использовать ее для выполнения производственных рабочих нагрузок. Некоторые функции могут не поддерживаться или их возможности могут быть ограничены. 
> Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

Виртуальный рабочий стол Windows в настоящее время доступен для всех географических расположений. Администраторы могут выбрать расположение для хранения пользовательских данных при создании виртуальных машин пула узлов и связанных служб, таких как файловые серверы. Дополнительные сведения о географических регионах Azure см. на [карте центров обработки данных Azure](https://azuredatacentermap.azurewebsites.net/).

>[!NOTE]
>Корпорация Майкрософт не контролирует и не ограничивает регионы, в которых вы или ваши пользователи могут получить доступ к данным пользователя и конкретного приложения.

>[!IMPORTANT]
>Виртуальные рабочие столы Windows хранят глобальные сведения о метаданных, такие как имена клиентов, имена пулов узлов, имена групп приложений и имена участников-пользователей в центре обработки данных. Каждый раз, когда клиент создает объект службы, он должен ввести расположение для объекта службы. Место, где они вводятся, определяет, где будут храниться метаданные для объекта. Клиент выберет регион Azure, и метаданные будут храниться в соответствующем географическом регионе. Список всех регионов Azure и связанные географические области см. в статье [географические регионы Azure](https://azure.microsoft.com/global-infrastructure/geographies/).

В данный момент поддерживается хранение метаданных только в США (US) Azure geography. Хранимые метаданные шифруются при хранении, а геоизбыточные зеркала поддерживаются в пределах географической области. Все клиентские данные, такие как параметры приложения и данные пользователя, находятся в расположении, выбранном клиентом и не управляемом службой. Дополнительные географические области станут доступны по мере роста службы.

Метаданные службы реплицируются в географическом виде Azure для целей аварийного восстановления.

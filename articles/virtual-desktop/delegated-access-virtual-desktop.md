# <a name="delegated-access-in-windows-virtual-desktop"></a>Делегированный доступ в Виртуальном рабочем столе Windows

Windows Virtual Desktop имеет делегированную модель доступа, которая позволяет определить объем доступа, который может иметь конкретный пользователь, назначив им роль. Назначение ролей состоит из трех компонентов: принцип безопасности, определение роли и область охвата. Модель делегированного доступа Windows Virtual Desktop основана на модели Azure RBAC. Чтобы узнать больше об конкретных назначениях ролей и их компонентах, [см.](../role-based-access-control/built-in-roles.md)

Делегат настольный компьютер Windows Virtual Desktop поддерживает следующие значения для каждого элемента назначения ролей:

* Субъект безопасности
    * Пользователи
    * Субъекты-службы
* Определение роли
    * Встроенные роли
* Область
    * Группы арендаторов
    * Клиенты
    * Бассейны для хозяев
    * Группы приложений

## <a name="built-in-roles"></a>Встроенные роли

Делегированный доступ в Windows Virtual Desktop имеет несколько встроенных определений ролей, которые можно назначить пользователям и директорам служб.

* Владелец RDS может управлять всем, включая доступ к ресурсам.
* Вкладчик RDS может управлять всем, кроме доступа к ресурсам.
* Читатель RDS может просматривать все, но не может вносить какие-либо изменения.
* Оператор RDS может просматривать диагностические действия.

## <a name="powershell-cmdlets-for-role-assignments"></a>Смдлеты PowerShell для ролевых заданий

Для создания, просмотра и удаления назначений ролей можно выполнить следующие cmdlets:

* **Get-RdsRoleAssignment** отображает список ролевых заданий.
* **New-RdsRoleAssignment** создает новое назначение ролей.
* **Удалить-RdsRoleAssignment** удаляет ролевые назначения.

### <a name="accepted-parameters"></a>Принятые параметры

Вы можете изменить основные три cmdlets со следующими параметрами:

* **AadTenantId**: указывает идентификатор арендатора Azure Active Directory, из которого является директор службы.
* **AppGroupName**: название группы приложений удаленного рабочего стола.
* **Диагностика**: указывает область диагностики. (Должен быть в паре с **инфраструктурой** или параметры **арендатора.)**
* **HostPoolName**: название пула удаленного рабочего стола.
* **Инфраструктура**: указывает область инфраструктуры.
* **RoleDefinitionName**: название роли управления доступом на основе удаленного рабочего стола, назначенной пользователю, группе или приложению. (Например, владелец служб удаленного рабочего стола, владелец удаленных настольных служб и так далее.)
* **ServerPrincipleName**: название приложения Active Directory Azure.
* **SignInName**: адрес электронной почты пользователя или основное имя пользователя.
* **TenantName**: имя арендатора удаленного рабочего стола.

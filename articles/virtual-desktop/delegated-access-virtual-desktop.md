# <a name="delegated-access-in-windows-virtual-desktop-preview"></a>Делегированный доступ в Апробация виртуального рабочего стола Windows

Обзор виртуального рабочего стола Windows имеет модель делегированный доступ, который позволяет определять объем доступа, которые пользователю разрешено иметь, назначив им роли. Назначение ролей состоит из трех компонентов: субъект безопасности, определение роли и области. Виртуальный рабочий стол Windows модели делегированного доступа основан на модели Azure RBAC. Дополнительные сведения о конкретных назначений ролей и их компоненты, см. в разделе [Обзор контроля доступа Azure на основе ролей](https://docs.microsoft.com/azure/active-directory/role-based-access-built-in-roles).

Виртуальный рабочий стол Windows делегированный доступ поддерживает следующие значения для каждого элемента назначения роли:

* Субъект безопасности
    * Пользователи
    * Субъекты-службы
* Определение роли
    * Встроенные роли
* `Scope`
    * Группы клиентов
    * Клиенты
    * Пулы узлов
    * Группы приложений

## <a name="built-in-roles"></a>Встроенные роли

Делегированный доступ в виртуальный рабочий стол Windows имеет несколько определений из встроенных ролей, которые можно назначить для пользователей и субъекты-службы.

* Служба удаленных рабочих СТОЛОВ владелец может управлять всем, включая доступ к ресурсам.
* Служба удаленных рабочих СТОЛОВ участник может управлять всем, но с доступом к ресурсам.
* Модуль чтения служб удаленных рабочих СТОЛОВ может все просматривать, но не может вносить изменения.
* Оператор служб удаленных рабочих СТОЛОВ можно просмотреть диагностические действия.

## <a name="powershell-cmdlets-for-role-assignments"></a>Командлеты PowerShell для назначения ролей

Можно выполнить следующие командлеты, чтобы создавать, просматривать и удалять назначения ролей:

* **Get-RdsRoleAssignment** отображает список назначений ролей.
* **Новый RdsRoleAssignment** создает новое назначение роли.
* **Remove-RdsRoleAssignment** удаление назначений ролей.

### <a name="accepted-parameters"></a>Допустимые параметры

Вы можете изменить три основные командлеты со следующими параметрами:

* **AadTenantId**: Указывает идентификатор клиента Azure Active Directory, из которого входит субъекта-службы.
* **AppGroupName**: имя группы приложения удаленного рабочего стола.
* **Диагностика**: указывает область диагностики. (Должны быть связаны с любым **инфраструктуры** или **клиента** параметров.)
* **HostPoolName**: имя пула узла удаленного рабочего стола.
* **Инфраструктура**: указывает область инфраструктуры.
* **RoleDefinitionName**: имя роли управления доступом на основе ролей служб удаленных рабочих столов, назначенной пользователя, группы или приложения. (Например, владелец службы удаленного рабочего стола, чтения служб удаленного рабочего стола и т. д.)
* **ServerPrincipleName**: имя приложения Azure Active Directory.
* **SignInName**: адрес электронной почты пользователя или имя участника-пользователя.
* **TenantName**: имя клиента удаленного рабочего стола.

# <a name="create-a-profile-container-for-a-host-pool-using-a-file-share"></a>Создание контейнера профиля для пула узлов с помощью общей папки

Служба Windows Virtual Desktop предлагает контейнеры профиля FSLogix в качестве рекомендуемого решения для профиля пользователя. Мы не рекомендуем использовать решение user Profile Disk (UPD), которое будет универслено в будущих версиях Windows Virtual Desktop.

В этой статье будет рассказать вам, как настроить раздел контейнера для контейнеров профиля FSLogix для пула хоста, используя виртуальную машинную долю файлов. Для получения дополнительной документации [FSLogix site](https://docs.fslogix.com/)FSLogix, см.

>[!NOTE]
>Если вы ищете материал сравнения о различных вариантах хранения контейнеров [Storage options for FSLogix profile containers](store-fslogix-profile.md)FSLogix Profile Container в Azure, см.

## <a name="create-a-new-virtual-machine-that-will-act-as-a-file-share"></a>Создание новой виртуальной машины, которая будет выступать в качестве совместного файла

При создании виртуальной машины, не забудьте разместить его на той же виртуальной сети, как хост пул виртуальных машин или на виртуальную сеть, которая имеет подключение к пулу хоста виртуальных машин. Вы можете создать виртуальную машину несколькими способами:

- [Создание виртуальной машины из изображения галереи Azure](../virtual-machines/windows/quick-create-portal.md#create-virtual-machine)
- [Создание виртуальной машины из управляемого изображения](../virtual-machines/windows/create-vm-generalized-managed.md)
- [Создание виртуальной машины из неуправляемого изображения](https://github.com/Azure/azure-quickstart-templates/tree/master/101-vm-from-user-image)

После создания виртуальной машины, присоединиться к нему в домен, делая следующие вещи:

1. [Подключайтесь к виртуальной машине](../virtual-machines/windows/quick-create-portal.md#connect-to-virtual-machine) с предоставленными учетными данными при создании виртуальной машины.
2. На виртуальной машине запустите **панель управления** и выберите **систему.**
3. Выберите **имя компьютера,** выберите **настройки изменения,** а затем выберите **Изменение...**
4. Выберите **домен,** а затем введите домен Active Directory в виртуальной сети.
5. Authenticate с учетной записью домена, которая имеет привилегии для доменных машин.

## <a name="prepare-the-virtual-machine-to-act-as-a-file-share-for-user-profiles"></a>Подготовьте виртуальную машину, чтобы она действовала в качестве общего файла для профилей пользователей

Ниже приведены общие инструкции о том, как подготовить виртуальную машину, чтобы выступать в качестве файла для профилей пользователей:

1. Добавьте пользователей Active Desktop Active Directory Windows в [группу безопасности Active Directory.](/windows/security/identity-protection/access-control/active-directory-security-groups/) Эта группа безопасности будет использоваться для проверки подлинности пользователей Windows Virtual Desktop в виртуальной машине обмена файлами, которую вы только что создали.
2. [Подключение к файлу доля виртуальной машины.](../virtual-machines/windows/quick-create-portal.md#connect-to-virtual-machine)
3. На файле доля виртуальной машины, создать папку на **диске C,** который будет использоваться в качестве доли профиля.
4. Нажмите правой кнопкой новой папки, выберите **Свойства**, выберите **Обмен**, затем выберите **Расширенный обмена ...**.
5. Выберите **Доля этой папки,** выберите **Разрешения ...**, затем выберите **Добавить ...**.
6. Поиск группы безопасности, к которой вы добавили Windows Virtual Desktop пользователей, а затем убедитесь, что группа имеет **полный контроль.**
7. После добавления группы безопасности, правой кнопкой мыши папку, выберите **Свойства**, выберите **Обмен**, а затем скопировать вниз **по сети Путь** для использования для последующего использования.

Для получения дополнительной информации [FSLogix documentation](/fslogix/fslogix-storage-config-ht/)о разрешениях см.

## <a name="configure-the-fslogix-profile-container"></a>Настройка контейнера профиля FSLogix

Чтобы настроить виртуальные машины с помощью программного обеспечения FSLogix, сделайте следующее на каждой машине, зарегистрированной в пуле хоста:

1. [Подключайтесь к виртуальной машине](../virtual-machines/windows/quick-create-portal.md#connect-to-virtual-machine) с предоставленными учетными данными при создании виртуальной машины.
2. Запустите интернет-браузер и перейдите по [этой ссылке,](https://go.microsoft.com/fwlink/?linkid=2084562) чтобы загрузить агент FSLogix.
3. Перейдите \\ \\либо к\\выпуску Win32, \\ \\либо к выпуску X64\\в файле .zip и запустите **FSLogixAppsSetup** для установки агента FSLogix.  Чтобы узнать больше о том, как установить FSLogix, [см.](/fslogix/install-ht/)
4. Перейдите к **программе Файлы** > **FSLogix** > **Apps,** чтобы подтвердить установленного агента.
5. Из меню «Старт» запустите **RegEdit** в качестве администратора. Перейдите на **\\\\программное обеспечение\\HKEY_LOCAL_MACHINE компьютером FSLogix**.
6. Создайте ключ под названием **Профили**.
7. Создайте следующие значения для ключа профилей:

| name                | Тип               | Данные/значение                        |
|---------------------|--------------------|-----------------------------------|
| Активировано             | DWORD              | 1                                 |
| VHDLocations        | Многострочное значение | "Сетевой путь для обмена файлами"     |

>[!IMPORTANT]
>Чтобы усилить защиту среды Виртуального рабочего стола Windows в Azure, мы рекомендуем не открывать входящий порт 3389 для виртуальных машин. Виртуальный рабочий стол Windows не требует открытия входящего порта 3389, чтобы пользователи могли получить доступ к виртуальным машинам пула узла. Если вам все же нужно открыть порт 3389 для устранения неполадок, мы рекомендуем использовать[JIT-доступ к виртуальным машинам](../security-center/security-center-just-in-time.md).

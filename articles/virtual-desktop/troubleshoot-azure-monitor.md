# <a name="troubleshoot-azure-monitor-for-windows-virtual-desktop-preview"></a>Устранение неполадок Azure Monitor для виртуальных рабочих столов Windows (Предварительная версия)

>[!IMPORTANT]
>Azure Monitor для виртуальных рабочих столов Windows в настоящее время находится в общедоступной предварительной версии. без соглашений об уровне обслуживания. Мы не рекомендуем использовать ее для выполнения производственных рабочих нагрузок. Некоторые функции могут не поддерживаться или их возможности могут быть ограничены. Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

В этой статье представлены известные проблемы и способы решения распространенных проблем в Azure Monitor для виртуальных рабочих столов Windows (Предварительная версия).

## <a name="the-configuration-workbook-isnt-working-properly"></a>Книга конфигурации не работает должным образом

Если книга настройки Azure Monitor не работает, можно использовать эти ресурсы, чтобы вручную настроить ее части:

- Чтобы вручную включить диагностику или получить доступ к рабочей области Log Analytics, см. статью [Отправка диагностических сведений о виртуальных рабочих столах Windows в log Analytics](diagnostics-log-analytics.md).
- Сведения об установке расширения Log Analytics на узле вручную см. в разделе [log Analytics расширение виртуальной машины для Windows](../virtual-machines/extensions/oms-windows.md).
- Сведения о настройке новой рабочей области Log Analytics см. в разделе [создание log Analytics рабочей области на портал Azure](../azure-monitor/learn/quick-create-workspace.md).
- Сведения о добавлении или удалении счетчиков производительности см. в разделе [Настройка счетчиков производительности](../azure-monitor/platform/data-sources-performance-counters.md).
- Сведения о настройке событий для Log Analytics рабочей области см. в разделе [Получение источников данных журнала событий Windows с помощью агента log Analytics](../azure-monitor/platform/data-sources-windows-events.md).

Кроме того, проблема может быть вызвана недостатком ресурсов или отсутствием необходимых разрешений.

Если у подписки нет ресурсов виртуальных рабочих столов Windows, она не будет отображаться в параметре *подписки* .

Если у вас нет доступа на чтение правильных подписок, они не будут отображаться в параметре *подписки* , и данные на панели мониторинга отображаться не будут. Чтобы решить эту проблему, обратитесь к владельцу подписки и запросите доступ для чтения.

## <a name="my-data-isnt-displaying-properly"></a>Мои данные отображаются неправильно

Если данные не отображаются должным образом, то во время процесса настройки Azure Monitor произошла ошибка. Сначала убедитесь, что все поля в книге конфигурации заполнены, как описано в статье [использование Azure Monitor для виртуальных рабочих столов Windows для мониторинга развертывания](azure-monitor.md). Параметры для новых и существующих сред можно изменить в любое время. Если у вас нет счетчиков или событий, связанные с ними данные не будут отображаться в портал Azure.

Если вы не пропустили какие-либо сведения, но данные не отображаются должным образом, может возникнуть ошибка в запросе или источниках данных. 

Если вы не видите ошибок установки и все равно не видите ожидаемых данных, может потребоваться подождать 15 минут и обновить канал. Azure Monitor имеет период задержки в 15 минут для заполнения данных журнала. Дополнительные сведения см. [в разделе время приема данных журнала в Azure Monitor](../azure-monitor/platform/data-ingestion-time.md).

Наконец, если вы не пропустили какие-либо сведения, но данные не отображаются, возможно, возникла проблема в запросе или источниках данных. Если это так, может потребоваться обратиться в службу поддержки для устранения проблемы.

## <a name="i-want-to-customize-azure-monitor-for-windows-virtual-desktop"></a>Я хочу настроить Azure Monitor для виртуальных рабочих столов Windows

Azure Monitor для виртуальных рабочих столов Windows использует книги Azure Monitor. Книги позволяют сохранить копию шаблона книги виртуальных рабочих столов Windows и внести собственные изменения.

Настраиваемые шаблоны не обновляются, когда группа продуктов обновляет исходный шаблон. Это происходит в средстве «книги», поэтому необходимо сохранить копию обновленного шаблона и повторно создать настройки для внедрения обновлений. Дополнительные сведения см. в разделе [Устранение неполадок с использованием аналитических данных на основе книг](../azure-monitor/insights/troubleshoot-workbooks.md) и [общих сведений о книгах](../azure-monitor/platform/workbooks-overview.md).

## <a name="known-issues"></a>Известные проблемы

Это проблемы, с которыми мы работаем в настоящее время и которые мы работаем над исправлением:

- В настоящее время для мониторинга можно одновременно выбрать только одну подписку, группу ресурсов и пул узлов. Поэтому при использовании страницы отчеты пользователя для понимания взаимодействия с пользователем необходимо убедиться, что у вас есть правильный пул узлов, который пользователь использовал, или их данные не заполняют визуальные элементы.

- В настоящее время сохранение избранных параметров в Azure Monitor не поддерживается, если не сохранить пользовательский шаблон книги. Это означает, что ИТ администраторам придется вводить имя подписки, имена групп ресурсов и настройки пула узлов при каждом открытии Azure Monitor для виртуальных рабочих столов Windows.

- В настоящее время нет способа экспортировать данные из Azure Monitor для виртуального рабочего стола Windows в Excel.

- Все оповещения уровня серьезности 1 Azure Monitor для всех продуктов в выбранной подписке будут отображаться на странице обзора. Это зависит от структуры, поскольку оповещения от других продуктов в подписке могут повлиять на виртуальные рабочие столы Windows. Сейчас запрос ограничивается предупреждениями с уровнем серьезности 1, за исключением предупреждений с высоким приоритетом 0 на странице обзора.

- Некоторые сообщения об ошибках не обрабатываются понятным образом, а не все сообщения об ошибках описаны в документации.

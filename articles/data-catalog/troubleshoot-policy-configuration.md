# <a name="troubleshooting-azure-data-catalog"></a>Устранение проблем с Каталогом данных Azure

В этой статье описаны общие проблемы устранения неполадок для ресурсов каталога данных Azure. 

## <a name="functionality-limitations"></a>Ограничения функциональности

При использовании каталога данных Azure ограничена следующая функциональность:

- Учетные записи с типом **Гостевой Роли** не поддерживаются. Вы не можете добавлять гостевые учетные записи [https://www.azuredatacatalog.com](https://www.azuredatacatalog.com)в качестве пользователей каталога данных Azure, а гостевые пользователи не могут использовать портал в .

- Создание ресурсов каталога данных Azure с использованием шаблонов управления ресурсами Azure или команд Azure PowerShell не поддерживается.

- Ресурс каталога данных Azure не может быть перемещен между арендаторами Azure.

## <a name="azure-active-directory-policy-configuration"></a>Настройка политики Azure Active Directory

Бывает, что пользователь может войти на портал каталога данных Azure, но при попытке входа в средство регистрации источника данных отображается сообщение об ошибке, которая запрещает вход в систему. Эта ошибка может возникнуть, когда вы находитесь в сети компании или когда вы подключаетесь из-за пределов сети компании.

Средство регистрации использует *проверку подлинности на основе форм* для проверки пользователя, который входит в систему, с помощью данных Azure Active Directory. Для успешного входа в систему администратор Azure Active Directory должен включить проверку подлинности на основе форм в *глобальной политике проверки подлинности*.

Глобальная политика проверки подлинности позволяет включать методы проверки подлинности отдельно для подключений в интрасети и экстрасети, как показано на следующем рисунке. Ошибки во въезде могут возникнуть, если проверка подлинности форм не включена для сети, из которой вы подключаетесь.

 ![Глобальная политика проверки подлинности Azure Active Directory](./media/troubleshoot-policy-configuration/global-auth-policy.png)

Дополнительные сведения см. в статье [Настройка политик проверки подлинности](/previous-versions/windows/it-pro/windows-server-2012-R2-and-2012/dn486781(v=ws.11)).

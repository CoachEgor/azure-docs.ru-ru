# <a name="create-an-azure-shared-image-gallery-using-the-portal"></a>Создание общей галереи изображений Azure с помощью портала

[Коллекция общих образов](shared-image-galleries.md) упрощает обмен пользовательскими образами в вашей организации. Пользовательские образы похожи на образы магазина, однако их можно создавать самостоятельно. Пользовательские образы можно использовать для начальной загрузки задач развертывания, например при предварительной загрузке приложений, конфигураций приложений и других конфигураций операционной системы. 

Коллекция общих образов позволяет предоставлять общий доступ к пользовательским образам виртуальной машины другим пользователям вашей организации в пределах региона или между регионами в клиенте AAD. Выберите образы, к которым нужно предоставить общий доступ, регионы, где они будут доступны, и пользователей, которым будет доступно совместное использование. Вы можете создать несколько коллекций, чтобы можно было логически группировать общие образы. 

Коллекция — это ресурс верхнего уровня, который обеспечивает полное управление доступом на основе ролей (RBAC). Образы могут быть версионными, и вы можете копировать каждую версию образа в другой набор регионов Azure. Коллекция работает только с Управляемыми образами.

Функция коллекции общих образов имеет несколько типов ресурсов. Мы будем использовать или создавать в этой статье следующие ресурсы:

| Ресурс | Описание|
|----------|------------|
| **Управляемое изображение** | Базовое изображение, которое может быть использовано в одиночку или использовано для создания **версии изображения** в картинной галерее. Управляемые изображения создаются из [обобщенных](shared-image-galleries.md#generalized-and-specialized-images) VM. Управляемый образ — это специальный тип виртуального жесткого диска, который может применяться для создания нескольких виртуальных машин. Теперь его можно использовать для создания версий общих образов. |
| **Снимок** | Копия VHD, которая может быть использована для использования **версии изображения.** Снимки могут быть взяты из [специализированных](shared-image-galleries.md#generalized-and-specialized-images) VM (тот, который не был обобщен), а затем использовать сярвидев дисков данных, для создания специализированной версии изображения.
| **Фотогалерея** | Как и Azure Marketplace, **коллекция образов** — это репозиторий для управления и совместного использования образов, но в отличие от Azure Marketplace доступ к коллекции образов контролируете вы. |
| **Определение образа** | Изображения определяются в галерее и содержат информацию об изображении и требованиях к его использованию в организации. Вы можете включить информацию, например, является ли изображение обобщенным или специализированным, операционную систему, минимальные и максимальные требования к памяти, а также примечания к выпуску. Это определение типа образа. |
| **Версия образа** | **Версия образа** используется для создания виртуальной машины с помощью коллекции. В зависимости от требований для вашей среды, у вас может быть несколько версий образа. Так же как и управляемый образ при использовании **версии образа** для создания виртуальной машины, версия образа используется для создания новых дисков для виртуальной машины. Версии образов можно использовать несколько раз. |

## <a name="before-you-begin"></a>Подготовка к работе

Чтобы завершить пример в этой статье, необходимо иметь существующее управляемое изображение обобщенный VM, или снимок специализированного VM. Вы можете следить за [учебником: Создайте пользовательское изображение VM Azure с Помощью Azure PowerShell](tutorial-custom-images.md) для создания управляемого изображения или [создайте снимок](../windows/snapshot-copy-managed-disk.md) для специализированного VM. Как для управляемых изображений, так и для снимков размер диска данных не может быть более чем 1 ТБ.

При работе со статьей по мере необходимости заменяйте имена групп ресурсов и виртуальных машин.

## <a name="create-vms"></a>Создание виртуальных машин 

Теперь вы можете создать один или несколько новых VMs. В этом примере создается виртуальная машина *myVMfromImage* в группе ресурсов *myResourceGroup* в регионе *Восточная часть США*.

1. Перейдите к своему определению изображения. Вы можете использовать фильтр ресурсов, чтобы показать все доступные определения изображений.
1. На странице для определения изображения выберите **Создать VM** из меню в верхней части страницы.
1. Для **группы ресурсов**выберите **Создать новый** и введите *myResourceGroup* для имени.
1. В **виртуальном названии машины**, введите *myVM*.
1. В поле **Страна или регион** выберите *Восточная часть США*.
1. Для **вариантов доступности**оставьте по умолчанию *не требуется избыточность инфраструктуры.*
1. Значение для **изображения** автоматически заполняется версией `latest` изображения, если вы начали со страницы для определения изображения.
1. Для **размера**выберите размер VM из списка доступных размеров, а затем выберите **Select.**
1. Под **учетной записью Администратора,** если источник VM был обобщен, введите имя **пользователя** и **общедоступный ключ SSH.** Если источник VM был специализированным, эти параметры будут серыми, потому что используется информация от источника VM.
1. Если вы хотите разрешить удаленный доступ к VM, под **открытым входящими портами,** выберите **Разрешить выбранные порты,** а затем выберите **SSH (22)** из выпадения вниз. Если вы не хотите разрешить удаленный доступ к VM, не оставляйте **ни одного выбранного** для **открытых входящих портов.**
1. Когда вы закончите, выберите кнопку **«Обзор» и создайте** кнопку в нижней части страницы.
1. После того, как VM проходит проверку, выберите **Создать** в нижней части страницы, чтобы начать развертывание.


## <a name="clean-up-resources"></a>Очистка ресурсов

Если группа ресурсов, виртуальная машина и все связанные с ними ресурсы вам больше не требуются, их можно удалить. Для этого выберите группу ресурсов для виртуальной машины, выберите действие **Удалить** и подтвердите имя удаляемой группы ресурсов.

Если вы хотите удалить отдельные ресурсы, необходимо удалить их в обратном порядке. Например, чтобы удалить определение изображения, необходимо удалить все версии изображений, созданные из этого изображения.

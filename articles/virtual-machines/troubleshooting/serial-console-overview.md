# <a name="azure-serial-console"></a>Серийная консоль Azure

Серийная консоль на портале Azure предоставляет доступ к текстовой консоли для виртуальных машин и экземплярам масштабируемых наборов виртуальных машин под управлением Linux или Windows. Эта Серийная консоль подключается через последовательный порт ttyS0 или COM1 виртуальной машины или экземпляра масштабируемого набора виртуальных машин, предоставляя доступ вне зависимости от состояния сети или операционной системы. Доступ к Серийной консоли можно получить только с помощью портала Azure и разрешен только для тех пользователей, которые имеют роль доступа "Участник" или более высокого уровня для виртуальной машины или масштабируемого набора виртуальных машин.

Серийная консоль работает аналогичным образом для виртуальных машин и экземпляров масштабируемых наборов виртуальных машин. В этом документе все упоминания виртуальных машин будут неявно включать экземпляры масштабируемых наборов виртуальных машин, если не указано иное.

> [!NOTE]
> Серийная консоль общедоступна в глобальных регионах Azure и находится на этапе общедоступной предварительной версии в Azure для государственных организаций. Она пока недоступна в облаке Azure для Китая.

## <a name="prerequisites-to-access-the-azure-serial-console"></a>Необходимые условия для доступа к Серийной консоли Azure
Чтобы получить доступ к Серийной консоли в экземпляре масштабируемого набора виртуальных машин или виртуальной машине, вам потребуется следующее.

- Для виртуальной машины должна быть включена диагностика загрузки.
- В виртуальной машине должна существовать учетная запись пользователя, использующая проверку подлинности по паролю. Вы можете создать пользователя с паролем, используя функцию [сброса пароля](https://docs.microsoft.com/azure/virtual-machines/extensions/vmaccess#reset-password) расширения для доступа к виртуальной машине. В разделе **Поддержка и устранение неполадок** выберите **Сбросить пароль**.
- Учетная запись Azure, обращающаяся к Серийной консоли, должна иметь [роль участника виртуальной машины](../../role-based-access-control/built-in-roles.md#virtual-machine-contributor) для виртуальной машины и учетной записи хранения [диагностики загрузки](boot-diagnostics.md).

> [!NOTE]
> Классические развертывания не поддерживаются. Виртуальная машина или экземпляр масштабируемого набора виртуальных машин должны использовать модель развертывания Azure Resource Manager.

## <a name="get-started-with-the-serial-console"></a>Начало работы с Серийной консолью
Серийная консоль для виртуальной машины и масштабируемого набора виртуальных машин доступна только на портале Azure.

### <a name="serial-console-for-virtual-machines"></a>Серийная консоль для виртуальных машин
Чтобы получить доступ к Серийной консоли, просто выберите **Серийная консоль** в разделе **Поддержка и устранение неполадок**.
  1. Откройте [портал Azure](https://portal.azure.com).

  1. Перейдите к разделу **Все ресурсы** и выберите виртуальную машину. Откроется страница обзора для виртуальной машины.

  1. Прокрутите вниз до раздела **Поддержка и устранение неполадок** и выберите **Последовательная консоль**. Откроется новая область с последовательной консолью, и запустится подключение.

     ![Окно Серийной консоли Linux](./media/virtual-machines-serial-console/virtual-machine-linux-serial-console-connect.gif)

### <a name="serial-console-for-virtual-machine-scale-sets"></a>Серийная консоль для масштабируемых наборов виртуальных машин
Серийная консоль доступна для масштабируемых наборов виртуальных машин, доступных на каждом экземпляре в масштабируемом наборе. Вам потребуется перейти к отдельному экземпляру масштабируемого набора виртуальных машин, прежде чем появится кнопка **Серийная консоль**. Если в масштабируемом наборе виртуальных машин не включена диагностика загрузки, измените модель масштабируемого набора виртуальных машин, чтобы включить диагностику загрузки, а затем обновите все экземпляры до новой модели, чтобы обеспечить доступ к Серийной консоли.
  1. Откройте [портал Azure](https://portal.azure.com).

  1. Перейдите к разделу **Все ресурсы** и выберите масштабируемый набор виртуальных машин. Открывается страница обзора для масштабируемого набора виртуальных машин.

  1. Перейдите к разделу **Экземпляры**.

  1. Выберите экземпляр масштабируемого набора виртуальных машин.

  1. В разделе **Поддержка и устранение неполадок** щелкните **Серийная консоль**. Откроется новая область с последовательной консолью, и запустится подключение.

     ![Серийная консоль масштабируемого набора виртуальных машин Linux](./media/virtual-machines-serial-console/vmss-start-console.gif)


### <a name="tls-12-in-serial-console"></a>TLS 1.2 в Серийной консоли
Серийная консоль использует TLS 1.2 сквозным образом для защиты всех сообщений в службе. Серийная консоль имеет зависимость от управляемой пользователем учетной записи хранения для диагностики загрузки, и TLS 1.2 должен быть настроен отдельно для этой учетной записи хранения. Соответствующие инструкции находятся [здесь](https://docs.microsoft.com/azure/storage/common/storage-security-tls).

Помимо доступа через консоль к виртуальной машине, вы также можете использовать Серийную консоль Azure для следующих задач.
* Отправка [команды системного запроса на виртуальную машину](./serial-console-nmi-sysrq.md).
* Отправка [немаскируемого прерывания на виртуальную машину](./serial-console-nmi-sysrq.md).
* Нормальная [перезагрузка или принудительного отключение и включение виртуальной машины](./serial-console-power-options.md).

# <a name="azure-serial-console"></a>Серийная консоль Azure

Система Ная в портале Azure предоставляет доступ к текстовой консоли для виртуальных машин (VMs) и виртуальному набору машин, установленным экземплярами под управлением Linux или Windows. Это последовательное соединение подключается к ttyS0 или COM1 серийный порт VM или виртуальной машины масштаб набора экземпляра, обеспечивая доступ независимо от сети или операционной системы государства. К серийной консоли можно получить доступ только с помощью портала Azure и допускаются только те пользователи, которые имеют роль участника доступа или выше к набору VM или виртуальной машины.

Серийная консоль работает таким же образом для виртуальных и виртуальных машин масштаба набора экземпляров. В этом документе все упоминания в Виртуальные мва неявно включают экземпляры набора виртуальной шкалы машин, если не указано иное.

> [!NOTE]
> Серийная консоль обычно доступна в глобальных регионах Azure и в публичном предварительном просмотре в правительстве Azure. Он пока не доступен в облаке Azure China.

## <a name="prerequisites-to-access-the-azure-serial-console"></a>Предпосылки для доступа к серийной консоли Azure
Чтобы получить доступ к серийной консоли на вашем VM или виртуальной шкале машины набор экземпляр, вам нужно следующее:

- Диагностика загрузки должна быть включена для VM
- Учетная запись пользователя, используюва учетную запись паролей, должна существовать в VM. Вы можете создать пользователя на основе паролей с функцией [сбросить пароль](https://docs.microsoft.com/azure/virtual-machines/extensions/vmaccess#reset-password) расширения доступа VM. В разделе **Поддержка и устранение неполадок** выберите **Сбросить пароль**.
- Доступ к учетной записи Azure Serial Console должен иметь [роль виртуального вклада в виртуоза](../../role-based-access-control/built-in-roles.md#virtual-machine-contributor) как для VM, так и для учетной [записи хранения диагностики ботинок](boot-diagnostics.md)

> [!NOTE]
> Классические развертывания не поддерживаются. В экземпляре набора VM или виртуальной машины должен использоваться модель развертывания диспетчера ресурсов Azure.

## <a name="get-started-with-the-serial-console"></a>Начало работы с серийной консоли
Серийная консоль для виртуальных вс-ам и набор виртуальной машины доступна только через портал Azure:

### <a name="serial-console-for-virtual-machines"></a>Серийная консоль для виртуальных машин
Серийная консоль для VMs так же проста, как нажатие на **серийную консоль** в разделе **Поддержки и устранения неполадок** на портале Azure.
  1. Откройте [портал Azure](https://portal.azure.com).

  1. Перейдите на **все ресурсы** и выберите виртуальную машину. Откроется страница обзора для виртуальной машины.

  1. Прокрутите вниз до раздела **Поддержка и устранение неполадок** и выберите **Последовательная консоль**. Откроется новая область с последовательной консолью, и запустится подключение.

     ![Linux Серийный консоль окно](./media/virtual-machines-serial-console/virtual-machine-linux-serial-console-connect.gif)

### <a name="serial-console-for-virtual-machine-scale-sets"></a>Серийная консоль для виртуальных наборов шкалы машин
Серийная консоль доступна для виртуальных наборов масштабов машины, доступных в каждом экземпляре в пределах набора шкалы. Перед тем, как увидеть кнопку **Кнопка консоли Serial,** вам придется перейти к отдельному экземпляру виртуальной шкалы машины. Если ваш виртуальный набор масштабов машины не имеет диагностики загрузки включен, убедитесь, что вы обновите модель набора виртуальной шкалы машины, чтобы включить диагностику загрузки, а затем обновить все экземпляры до новой модели, чтобы получить доступ к серийной консоли.
  1. Откройте [портал Azure](https://portal.azure.com).

  1. Перейдите на **все ресурсы** и выберите набор виртуальной шкалы машин. Открывается страница обзора для набора виртуальной шкалы машин.

  1. Перейдите в **инстанции**

  1. Выберите экземпляр набора виртуального масштаба машины

  1. Из раздела **«Поддержка и устранение неполадок»** выберите **серийную консоль.** Откроется новая область с последовательной консолью, и запустится подключение.

     ![Linux виртуальный машинный набор Серийный консоль](./media/virtual-machines-serial-console/vmss-start-console.gif)

# <a name="troubleshoot-unexpected-reboots-of-vms-with-attached-vhds"></a>Устранение непредвиденных перезагрузок виртуальных машин с подключенными VHD

Если к виртуальной машине Azure подключено много VHD, размещенных в одной учетной записи хранения, могут быть превышены целевые показатели масштабируемости для отдельной учетной записи хранения. Из-за этого виртуальная машина может неожиданно перезагрузиться. Проверьте мельчайшие метрики для учетной записи хранилища **(TotalRequests**/**TotalIngress**/**TotalEgress)** для спайков, превышающих целевые показатели масштабируемости для учетной записи хранилища. Чтобы определить, производилось ли регулирование для учетной записи хранения, прочитайте раздел [Метрики показывают увеличение значения PercentThrottlingError](../../storage/common/storage-monitoring-diagnosing-troubleshooting.md#metrics-show-an-increase-in-PercentThrottlingError).

Как правило, каждая отдельная операция ввода или вывода, выполняемая виртуальной машиной с виртуальным жестким диском, преобразуется в операцию **Get Page** или **Put Page** на уровне базового страничного BLOB-объекта. Поэтому вы можете использовать предполагаемое число операций ввода-вывода в секунду, установленное для среды, для корректировки допустимого числа виртуальных жестких дисков в одной учетной записи хранения в соответствии с особенностями функционирования приложения. Корпорация Майкрософт рекомендует использовать в одной учетной записи хранения не более 40 дисков. Для получения дополнительной информации о целевых [Scalability targets for standard storage accounts](../../storage/common/scalability-targets-standard-account.md)показателях масштабируемости для стандартных учетных записей хранения см. Для получения дополнительной информации о целевых показателях [Scalability targets for premium page blob storage accounts](../../storage/blobs/scalability-targets-premium-page-blobs.md)масштабируемости для учетных записей хранения капля премиум-страниц см.

Если целевые показатели масштабируемости для учетной записи хранения превышены, распределите виртуальные жесткие диски между несколькими учетными записями хранения, чтобы снизить интенсивность использования каждой из них.

# <a name="power-options-available-from-the-azure-serial-console"></a>Параметры питания, доступные на серийной консоли Azure

Azure Serial Console предоставляет несколько мощных инструментов для управления питанием на вашем VM или виртуальном наборе машин. Эти варианты управления питанием могут ввести в заблуждение некоторые, так что это обзор каждого инструмента и его предполагаемого использования случае.

Функция серийной консоли | Описание | Вариант использования
:----------------------|:------------|:---------
Перезапуск виртуальной машины | Изящный перезапуск вашего VM или виртуальной машины шкалы набора экземпляра. Эта операция такая же, как вызов функции перезагрузки, доступной на странице «Обзор». | В большинстве случаев эта опция должна стать первым инструментом при попытке перезапустить VM. Ваше соединение Serial Console будет испытывать краткое прерывание и автоматически возобновится, как только VM перезаработает.
Сброс виртуальной машины | Сильный цикл питания вашего VM или виртуальной шкалы машин, установленный платформой Azure. | Эта опция используется для немедленной перезагрузки операционной системы независимо от ее текущего состояния. Поскольку эта операция не является изящной, существует риск потери данных или повреждения. В подключении Serial Console нет перерыва, что может быть полезно для отправки команд в начале загрузки (например, добраться до GRUB на Linux VM или безопасном режиме в Windows VM).
SysRq - Перезагрузка (b) | Системный запрос на принудительное перезапуск гостя. | Эта функция применима только к операционным системам Linux и требует [включения SysRq](./serial-console-nmi-sysrq.md#system-request-sysrq) в операционную систему. Если операционная система настроена правильно для SysRq, эта команда приведет к перезапуску ОС.
NMI (Немаскируемый прерывание) | Команда прерывания, которая будет доставлена в операционную систему | Эта операция доступна как для [Windows,](./serial-console-windows.md#use-the-serial-console-for-nmi-calls) так и для [Linux](./serial-console-nmi-sysrq.md#non-maskable-interrupt-nmi) VMs, и требует включения NMI. Отправка NMI, как правило, приводит к сбою операционной системы. Можно настроить операционную систему для создания файла дампа, а затем перезапустить после получения NMI, что может быть полезно при низкоуровневой отладке.

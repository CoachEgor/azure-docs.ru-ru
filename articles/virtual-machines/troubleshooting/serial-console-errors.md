# <a name="common-errors-within-the-azure-serial-console"></a>Распространенные ошибки в последовательной консоли Azure
В серийной консоли Azure имеется множество ошибок. Это список этих ошибок и мер по смягчению последствий для них.

## <a name="common-errors"></a>Распространенные ошибки

Error                             |   Меры по снижению риска
:---------------------------------|:--------------------------------------------|
«Azure Serial Console требует включения диагностики ботинок. Нажмите здесь, чтобы настроить диагностику загрузки для вашей виртуальной машины". | Убедитесь, что набор VM или виртуальной машины имеет [диагностику загрузки](boot-diagnostics.md) включен. Если вы используете серийную консоль на примере виртуального масштаба машины, убедитесь, что ваш экземпляр имеет последнюю модель.
«Azure Serial Console требует работы виртуальной машины. Используйте кнопку "Пуск" выше, чтобы запустить виртуальную машину".  | Пример набора VM или виртуальной машины должен находиться в запущенном состоянии для доступа к серийной консоли (ваш VM не должен быть остановлен или локализован). Убедитесь, что ваш VM или виртуальный набор масштаба машины набор экземпляр работает и попробуйте еще раз.
"Azure Serial Console не включена для этой подписки, свяжитесь с администратором подписки, чтобы включить". | Серийная консоль Azure может быть отключена на уровне подписки. Если вы являетесь администратором подписки, вы можете [включить и отключить серийную консоль Azure.](./serial-console-enable-disable.md) Если вы не являетесь администратором подписки, вы должны обратиться к администратору подписки для дальнейших шагов.
При доступе к учетной записи хранения для диагностики загрузки виртуальной машины обнаружен ответ "Запрещено". | Убедитесь, что в системе диагностики загрузки нет брандмауэра учетной записи. Для функционирования последовательной консоли требуется доступная учетная запись хранения для диагностики загрузки. Последовательная консоль не может работать с учетными записями хранения с включенным брандмауэром и диагностикой загрузки.
У вас нет нужных разрешений для использования последовательной консоли этой виртуальной машины. Убедитесь в наличии по крайней мере разрешений роли участника виртуальной машины.| Серийный доступ к консоли требует, чтобы вы имели доступ к уровню вклада или выше на вашем наборе VM или виртуальной машины. Для получения дополнительной информации смотрите [страницу обзора](serial-console-overview.md).
Учетная запись хранилища', используемая для диагностики загрузки на этом VM, не может быть найдена. Проверить, что диагностика загрузки включена для этого VM, эта учетная запись хранения не была удалена, и у вас есть доступ к этой учетной записи хранения. | Двойная проверка того, что вы не удалили учетную запись хранения загрузочной диагностики для вашего набора VM или виртуальной машины
Серийное подключение консоли к VM столкнулось с ошибкой: 'Bad Request' (400) | Это может произойти, если ваша диагностика загрузки URI неверна. Например, вместо "https://" использовалась "http://". Диагностика загрузки URI может быть исправлена с помощью этой команды:`az vm boot-diagnostics enable --name vmName --resource-group rgName --storage https://<storageAccountUri>.blob.core.windows.net/`
У вас нет необходимых разрешений для записи на учетную запись хранения диагностики загрузки для этого VM. Пожалуйста, убедитесь, что у вас есть по крайней мере разрешения вкладчика VM | Серийный доступ к консоли требует доступа на уровне вклада в учетную запись хранения диагностики ботинок. Для получения дополнительной информации смотрите [страницу обзора](serial-console-overview.md).
Невозможно определить группу ресурсов для учетной * &lt;&gt;* записи хранения загрузочной диагностики STORAGEACCOUNTACCOUNTNAME. Убедитесь, что для этой виртуальной машины включена диагностика загрузки и у вас есть доступ к этой учетной записи хранения. | Серийный доступ к консоли требует доступа на уровне вклада в учетную запись хранения диагностики ботинок. Для получения дополнительной информации смотрите [страницу обзора](serial-console-overview.md).
Подготовка к этому ВМ пока не увенчалась успехом. Пожалуйста, убедитесь, что VM полностью развернут и повторно попробуйте серийное соединение консоли. | Ваш набор VM или виртуальной машины масштаба все еще может быть подготовка. Подождите некоторое время и попробуйте еще раз.
Веб-сокет закрыт или его не удалось открыть. | Возможно, потребуется добавить `*.console.azure.com`доступ к брандмауэру. Более подробный, но более длительный подход заключается в том, чтобы разрешить доступ брандмауэра к [IP-диапазонам Microsoft Azure Datacenter,](https://www.microsoft.com/download/details.aspx?id=41653)которые меняются довольно регулярно.
Серийная консоль не работает с учетной записью хранения данных С помощью Azure Data Lake Storage 2 с иерархическими пространствами имен. | Это известная проблема с иерархическими пространствами имен. Чтобы смягчить последствия, убедитесь, что учетная запись хранения загрузок VM не будет создана с помощью Azure Data Lake Storage Gen2. Эта опция может быть установлена только при создании учетной записи хранилища. Возможно, вам придется создать отдельную учетную запись хранения загрузок без azure Data Lake Storage Gen2, что позволило смягчить эту проблему.

Чтобы смягчить последствия, запустите Cloud Shell и [выполните шаги, необходимые](https://docs.microsoft.com/azure/cloud-shell/persisting-shell-storage#unmount-clouddrive-1) для переобеспечения резервной учетной записи хранилища для Cloud Shell в текущей подписке.

# <a name="common-errors-within-the-azure-serial-console"></a>Распространенные ошибки в последовательной консоли Azure
В последовательной консоли Azure есть набор известных ошибок. Это список этих ошибок и этапов их устранения.

## <a name="common-errors"></a>Распространенные ошибки

Error                             |   Меры по снижению риска
:---------------------------------|:--------------------------------------------|
"Последовательной консоли Azure требуется включить диагностику загрузки. Щелкните здесь, чтобы настроить диагностику загрузки для виртуальной машины ". | Убедитесь, что в ВИРТУАЛЬНОЙ машине или масштабируемом наборе виртуальных машин включена [Диагностика загрузки](boot-diagnostics.md) . При использовании последовательной консоли в экземпляре масштабируемого набора виртуальных машин убедитесь, что у экземпляра есть последняя модель.
"Для выполнения последовательной консоли Azure требуется виртуальная машина. Используйте кнопку запустить выше, чтобы запустить виртуальную машину ".  | Чтобы получить доступ к последовательной консоли, ВИРТУАЛЬная машина или экземпляр масштабируемого набора виртуальных машин должны находиться в запущенном состоянии (виртуальная машина не должна быть остановлена или освобождена). Убедитесь, что виртуальная машина или экземпляр масштабируемого набора виртуальных машин работает, и повторите попытку.
"Последовательная консоль Azure не включена для этой подписки, обратитесь к администратору подписки, чтобы включить". | Последовательную консоль Azure можно отключить на уровне подписки. Если вы являетесь администратором подписки, вы можете [включить и отключить последовательную консоль Azure](./serial-console-enable-disable.md). Если вы не являетесь администратором подписки, вы должны обратиться к администратору подписки, чтобы выполнить дальнейшие действия.
При доступе к учетной записи хранения для диагностики загрузки виртуальной машины обнаружен ответ "Запрещено". | Убедитесь, что в системе диагностики загрузки нет брандмауэра учетной записи. Для функционирования последовательной консоли требуется доступная учетная запись хранения для диагностики загрузки. Последовательная консоль не может работать с учетными записями хранения с включенным брандмауэром и диагностикой загрузки.
У вас нет нужных разрешений для использования последовательной консоли этой виртуальной машины. Убедитесь в наличии по крайней мере разрешений роли участника виртуальной машины.| Для доступа к последовательной консоли требуется доступ на уровне участника или выше в масштабируемом наборе виртуальных машин. Дополнительные сведения см. на [странице Обзор](serial-console-overview.md).
Не удалось найти учетную запись хранения "", используемую для диагностики загрузки на этой виртуальной машине. Убедитесь, что для этой виртуальной машины включена Диагностика загрузки, эта учетная запись хранения не была удалена и у вас есть доступ к этой учетной записи хранения. | Проверьте, не удалили ли вы учетную запись хранения для диагностики загрузки для виртуальной машины или масштабируемого набора виртуальных машин.
При подключении последовательной консоли к виртуальной машине произошла ошибка: "Недопустимый запрос" (400) | Это может произойти, если URI диагностики загрузки неверен. Например, вместо "https://" был использован "http://". Универсальный код ресурса диагностики загрузки можно исправить с помощью следующей команды:`az vm boot-diagnostics enable --name vmName --resource-group rgName --storage https://<storageAccountUri>.blob.core.windows.net/`
У вас нет необходимых разрешений для записи в учетную запись хранения диагностики загрузки для этой виртуальной машины. Убедитесь, что у вас есть по крайней мере разрешения участника виртуальной машины. | Серийная консоль доступу требуется доступ на уровне участника к учетной записи хранения для диагностики загрузки. Дополнительные сведения см. на [странице Обзор](serial-console-overview.md).
Не удалось определить группу ресурсов для учетной записи хранения для диагностики загрузки * &lt; STORAGEACCOUNTNAME &gt; *. Убедитесь, что для этой виртуальной машины включена диагностика загрузки и у вас есть доступ к этой учетной записи хранения. | Серийная консоль доступу требуется доступ на уровне участника к учетной записи хранения для диагностики загрузки. Дополнительные сведения см. на [странице Обзор](serial-console-overview.md).
Подготовка для этой виртуальной машины еще не выполнена. Убедитесь, что виртуальная машина полностью развернута, и повторите подключение к последовательной консоли. | Возможно, ваша виртуальная машина или масштабируемый набор виртуальных машин все еще подготавливается. Подождите некоторое время и повторите попытку.
Веб-сокет закрыт или его не удалось открыть. | Возможно, потребуется добавить брандмауэр к `*.console.azure.com` . Более подробный, но более длинный подход заключается в том, чтобы разрешить брандмауэру доступ к [диапазонам IP-адресов центра обработки данных Microsoft Azure](https://www.microsoft.com/download/details.aspx?id=41653), которые меняются довольно часто.
Серийная консоль не работает с учетной записью хранения, используя Azure Data Lake Storage 2-го поколения с иерархическими пространствами имен. | Это известная проблема с иерархическими пространствами имен. Чтобы устранить эту проблемы, убедитесь, что учетная запись хранения для диагностики загрузки виртуальной машины не создана с помощью Azure Data Lake Storage 2-го поколения. Этот параметр можно задать только при создании учетной записи хранения. Чтобы устранить эту ошибку, может потребоваться создать отдельную учетную запись хранения для диагностики загрузки, не Azure Data Lake Storage 2-го поколения включена.
При подключении последовательной консоли к виртуальной машине произошла ошибка: "запрещено" (Субскриптионнотенаблед) — имя подписки не определено, идентификатор \<subscription id> находится в невключенном состоянии без определения | Эта проблема может возникать, если подписка, в которой пользователь создал свою учетную запись хранения Cloud Shell, была отключена. Чтобы устранить эту проблемы, запустите Cloud Shell и [выполните действия, необходимые](../../cloud-shell/persisting-shell-storage.md#unmount-clouddrive-1) для повторной инициализации резервной учетной записи хранения для Cloud Shell в текущей подписке.

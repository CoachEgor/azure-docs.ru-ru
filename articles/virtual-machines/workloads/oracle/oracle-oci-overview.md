# <a name="oracle-application-solutions-integrating-microsoft-azure-and-oracle-cloud-infrastructure"></a>Решения для приложений Oracle интеграция Microsoft Azure и облачной инфраструктуры Oracle

Корпорация Майкрософт и Oracle прилагаются к обеспечению низкой задержки и высокой пропускной способности подключения между облаками, что позволяет использовать преимущества обоих облаков. 

Используя это подключение между облаками, вы можете секционировать многоуровневое приложение для выполнения уровня базы данных в облачной инфраструктуре Oracle (OCI), а также приложения и других уровней на Microsoft Azure. Работа аналогична выполнению всего стека решения в одном облаке. 

Если вы хотите полностью развернуть решения Oracle в инфраструктуре Azure, см. раздел [образы виртуальных машин Oracle и их развертывание на Microsoft Azure](oracle-vm-solutions.md).

## <a name="scenario-overview"></a>Обзор сценария

Подключение между облаками предоставляет решение для запуска ведущих приложений Oracle и собственных пользовательских приложений на виртуальных машинах Azure, в то же время обеспечивая преимущества размещенных служб баз данных в OCI. 

> [!IMPORTANT]
> Oracle будет сертифицировать эти приложения для запуска в Azure при использовании решения Azure/Oracle Cloud Interconnect с 2020 мая.

Ниже перечислены приложения, которые можно выполнять в конфигурации с кросс-облаком.

* Набор E-Business
* JD Edwards EnterpriseOne
* PeopleSoft
* Розничные приложения Oracle
* Oracle Hyperion Financial Management

На следующей схеме представлен общий обзор подключенного решения. Для простоты на схеме показан только уровень приложения и уровень данных. В зависимости от архитектуры приложения решение может включать дополнительные уровни, такие как веб-уровень в Azure. Дополнительные сведения приведены в следующем разделе.

## <a name="region-availability"></a>Доступность по регионам 

Возможность подключения между облаками ограничена следующими регионами:
* Azure Восточная часть США (eastus) & OCI Эшберн (Восточная часть США)
* Azure южная часть Соединенного Королевства (uksouth) & OCI Лондон (южная часть Соединенного Королевства)
* Центр Azure для Канады (канадацентрал) & OCI Торонто (Юго-Восточная Канада)
* Azure Западной Европа (westeurope) & OCI Амстердам (нидерландский северо-запад)
* Восточная Япония Azure (жапанеаст) & OCI Токио (Восточная Япония)

## <a name="networking"></a>Сеть

Корпоративные клиенты часто выбирают изменить и развертывают рабочие нагрузки в нескольких облаках для различных бизнес-и эксплуатационных причин. Чтобы изменить, клиенты соединяют облачные сети через Интернет, VPN-подключение IPSec или решение с прямым подключением поставщика облачных служб через локальную сеть. При взаимоподключении к облачным сетям могут требоваться значительные инвестиции во время, деньги, проектирование, закупки, установку, тестирование и эксплуатацию. 

Для решения этих проблем с клиентами Oracle и корпорация Майкрософт включили интегрированную работу с несколькими облаками. Межоблачные сети устанавливаются путем подключения канала [ExpressRoute](../../../expressroute/expressroute-introduction.md) в Microsoft Azure с каналом [фастконнект](https://docs.cloud.oracle.com/iaas/Content/Network/Concepts/fastconnectoverview.htm) в OCI. Такое подключение возможно, когда расположение пиринга Azure ExpressRoute находится в том же расположении, что и Фастконнект OCI. Эта установка обеспечивает безопасное и быстрое подключение между двумя облаками без необходимости промежуточного поставщика услуг.

С помощью ExpressRoute и Фастконнект клиенты могут выполнять одноранговую виртуальную сеть в Azure с помощью виртуальной облачной сети в OCI при условии, что пространство частных IP-адресов не перекрывается. Пиринг между двумя сетями позволяет ресурсу в виртуальной сети взаимодействовать с ресурсом в виртуальной сети OCI, как если бы они одновременно направляются в одну сеть.

## <a name="network-security"></a>Безопасность сети

Сетевая безопасность является важнейшим компонентом любого корпоративного приложения и является центральной для этого решения для нескольких облаков. Весь трафик, переходящий через ExpressRoute и Фастконнект, проходит через частную сеть. Такая конфигурация обеспечивает безопасную связь между виртуальной сетью Azure и сетью виртуальной сети Oracle. Вам не нужно предоставлять общедоступный IP-адрес для виртуальных машин в Azure. Аналогичным образом шлюз Интернета в OCI не нужен. Все взаимодействие происходит через частный IP-адрес компьютеров.

Кроме того, можно настроить [списки безопасности](https://docs.cloud.oracle.com/iaas/Content/Network/Concepts/securitylists.htm) в сети OCI и правила безопасности (подключенные к [группам безопасности сети](../../../virtual-network/security-overview.md)Azure). Используйте эти правила для управления трафиком, передаваемым между компьютерами в виртуальных сетях. Правила сетевой безопасности можно добавлять на уровне компьютера, на уровне подсети, а также на уровне виртуальной сети.
 
## <a name="identity"></a>Идентификация

Удостоверение является одной из основных принципов партнерства связи между корпорацией Майкрософт и Oracle. Для интеграции [облачной службы Oracle Identity](https://docs.oracle.com/en/cloud/paas/identity-cloud/index.html) (идкс) с [Azure Active Directory](../../../active-directory/index.yml) (Azure AD) была выполнена значительная работа. Azure AD — это облачная служба управления удостоверениями и доступом Майкрософт. Она помогает пользователям входить в различные ресурсы и получать к ним доступ. Azure AD также позволяет управлять пользователями и их разрешениями.

В настоящее время такая интеграция позволяет управлять в одном централизованном расположении, которое Azure Active Directory. Azure AD синхронизирует все изменения в каталоге с соответствующим каталогом Oracle и используется для единого входа в решения Oracle для кросс-облака.

# <a name="oracle-application-solutions-integrating-microsoft-azure-and-oracle-cloud-infrastructure-preview"></a>Решения Oracle приложений, интеграции Microsoft Azure и Oracle облачной инфраструктуры (Предварительная версия)

Корпорации Майкрософт и Oracle заключили Партнерское соглашение для обеспечения низкой задержкой и высокой пропускной способностью подключения нескольких облаках, что позволяет воспользоваться преимуществами лучшее из обоих облаков. 

Используя это подключение нескольких облаков, можно разделить многоуровневого приложения для запуска вашего уровня базы данных в Oracle облачной инфраструктуры Interface (OCI) и приложение и другие уровни в Microsoft Azure. Процесс выполняется аналогично выполнению полнофункциональных решений в одно облако. 

> [!IMPORTANT]
> Эта возможность нескольких облаков сейчас находится на этапе предварительной версии и некоторые [применяются ограничения](#preview-limitations). Предварительные версии предоставляются при условии, что вы принимаете [дополнительные условия использования](https://azure.microsoft.com/support/legal/preview-supplemental-terms/). Некоторые аспекты этой функции могут быть изменены до выхода общедоступной версии.

Если вы заинтересованы в развертывании решения Oracle полностью в инфраструктуре Azure, см. в разделе [образов виртуальных Машин Oracle и их развертывание в Microsoft Azure](oracle-vm-solutions.md).

## <a name="scenario-overview"></a>Обзор сценария

Подключение нескольких облаков предоставляет решение для запуска приложения Oracle в отрасли и собственными приложениями, на виртуальных машинах Azure вместе с выгодами преимущества размещенной базой данных служб в OCI. 

Приложения могут работать в облачной конфигурации включают:

* E-Business Suite
* JD Edwards EnterpriseOne
* PeopleSoft
* Приложения для розничной торговли Oracle
* Управление финансами Hyperion Oracle

На следующей диаграмме показан общий обзор подключенном решении. Для простоты на схеме показаны только уровня приложений и уровня данных. В зависимости от архитектуры приложения решение может содержать дополнительные уровни, например веб-уровня в Azure. Дополнительные сведения приведены в следующем разделе.

## <a name="networking"></a>Сеть

Корпоративные клиенты часто предпочитают необходимо изменить и развертывания рабочих нагрузок по несколько облаков для бизнеса, так и эксплуатационным поводам, предусмотрено. Необходимо изменить, клиенты соединения, которые используются облачными сетями через Интернет, VPN-Подключение IPSec, или с помощью решения прямого подключения поставщика облачных служб с помощью вашей локальной сети. Предстают облачных сетях может потребоваться значительные инвестиции в время, деньги, проектирования, закупок, установки, тестирования и эксплуатации. 

Чтобы решить эти проблемы клиентов, Oracle и Майкрософт включить несколько облачных расширенные возможности интеграции. Сети, охватывающей несколько облачных устанавливается путем подключения [ExpressRoute](../../../expressroute/expressroute-introduction.md) канала в Microsoft Azure с помощью [FastConnect](https://docs.cloud.oracle.com/iaas/Content/Network/Concepts/fastconnectoverview.htm) канал в OCI. Это подключение можно где расположения пиринга Azure ExpressRoute — вблизи от или в одном расположении пиринга как OCI FastConnect. Эта конфигурация обеспечивает безопасную и быстрого подключения между двумя облаками без необходимости для поставщика промежуточной службы.

С помощью ExpressRoute и FastConnect, клиентам можно установить равноправное подключение виртуальной сети в Azure с помощью облачной виртуальной сети в OCI, при условии, что диапазон частных IP-адресов не перекрываются. Пиринга между двумя сетями позволяет ресурсу в виртуальной сети для взаимодействия с ресурсом в виртуальном облаке OCI сети, как, если они находятся в той же сети.

## <a name="network-security"></a>Безопасность сети

Сетевая безопасность является важнейшим компонентом любого приложения enterprise и является центральным элементом этого несколько облачных решений. Любой трафик, передаваемый через ExpressRoute и FastConnect передает через частную сеть. Эта конфигурация обеспечивает безопасный обмен данными между виртуальной сетью Azure и сеть виртуальном облаке Oracle. Укажите общедоступный IP-адрес, чтобы все виртуальные машины в Azure не требуется. Аналогичным образом не требуется шлюз Интернета в OCI. Весь обмен данными осуществляется через частный IP-адрес машин.

Кроме того, можно настроить [списки безопасности](https://docs.cloud.oracle.com/iaas/Content/Network/Concepts/securitylists.htm) на вашей OCI облачной виртуальной сети и правил безопасности (присоединен к Azure [группы безопасности сети](../../../virtual-network/security-overview.md)). Эти правила используются для управления трафиком, передающимся между машинами в виртуальных сетях. Можно добавить правила сетевой безопасности на уровне компьютера, на уровне подсети, а также на уровне виртуальной сети.
 
## <a name="identity"></a>Удостоверение

Идентификация является одной из основных ключевых компонентов партнерству Майкрософт и Oracle. Значительный объем работы были проведены интегрировать [облачную службу удостоверений Oracle](https://docs.oracle.com/en/cloud/paas/identity-cloud/index.html) (IDCS) с [Azure Active Directory](../../../active-directory/index.yml) (Azure AD). Azure AD — облачной идентификацией и доступом служба управления корпорации Майкрософт. Он помогает пользователям вход и доступ к различным ресурсам. Azure AD также позволяет управлять пользователями и их разрешения.

В настоящее время такая интеграция позволяет управлять в одном централизованном месте, что такое Azure Active Directory. Azure AD синхронизирует все изменения в каталоге с помощью соответствующего каталога Oracle и используется для единого входа для облачных решениях Oracle.

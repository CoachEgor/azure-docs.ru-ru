# <a name="azure-white-papers-about-mainframe-topics"></a>Белые документы Azure о темах мейнфрейма

Получайте ресурсы о миграции мейнфреймов, реразмещении и перемещении систем ibm в Microsoft Azure.

### <a name="demystifying-mainframe-to-azure-migration"></a>[Демистификация мэйнфрейма для миграции Azure](https://azure.microsoft.com/resources/demystifying-mainframe-to-azure-migration/)

Azure предоставляет альтернативу мэйнфрейма, способную предоставить эквивалентные функциональные возможности и функции. В этом документе Ларри Мида из группы AzureCAT рассматриваются основные компоненты мейнфрейма IBM z/OS и эквиваленты Azure. В нем также содержится «дорожная карта» для начала разговора с лицами, принимающими решения в отношении ИТ, которые присоединяются к устаревшей философии мэйнфрейма.

### <a name="move-mainframe-compute-and-storage-to-azure"></a>[Перемещение вычислений и хранения мейнфреймов в Azure](https://azure.microsoft.com/resources/move-mainframe-compute-and-storage-to-azure/)

Для выполнения рабочих нагрузок мейнфреймов в Microsoft Azure необходимо знать, как возможности мейнфрейма соотносятся с Azure. На основе мейнфрейма IBM z14 в этом руководстве Ларри Мида из AzureCAT исследуется, как получить сопоставимые результаты на Azure. Массово масштабируемые ресурсы вычислений и хранения данных в Azure обеспечивают немедленные преимущества для организаций, работающих с рабочими нагрузками мейнфрейма.

### <a name="microsoft-azure-government-cloud-for-mainframe-applications"></a>[Облако правительства Microsoft Azure для приложений с мэйнфреймом](https://azure.microsoft.com/resources/microsoft-azure-government-cloud-for-mainframe-applications/)

Планирование миграции приложений является идеальным временем для добавления стоимости и гибкости даже хорошо зарекомендовавших себя рабочих нагрузок мэйнфрейма. В этом кратком руководстве Ларри Мид (Larry Mead) из AzureCAT показывает, как правительственные учреждения США и их партнеры могут использовать azure Government для применения мэйнфреймов, и миграция может быть не такой сложной, как вы думаете. Правительство Azure предоставляет преимущества мэйнфрейма в более экономичных и гибких условиях. Кроме того, правительство Лазурного государства получило Временное управление по эксплуатации (P-ATO) для FedRAMP High Impact.

### <a name="deploy-ibm-db2-purescale-on-azure"></a>[Развертывание IBM DB2 чистый scale на Azure](https://azure.microsoft.com/resources/deploy-ibm-db2-purescale-on-azure/)

Узнайте из нашего опыта работы с предприятием, которое недавно переразместило свою среду IBM DB2 в Azure. Это руководство было написано членами команды AzureCAT и DMJ, которые были там: Ларри Мид, Бенджамин Гинебертьер, Алессандро Vozza, и Джонатан Фрост. Они описывают шаги, которые они предприняли во время этой миграции. Их выводы были рассмотрены членами команды IBM Db2 pureScale. Сценарии установки, доступные на GitHub, основаны на архитектуре, используемой командой для типичной рабочей нагрузки OLTP среднего размера.

### <a name="install-tmaxsoft-openframe-on-azure"></a>[Установка TmaxSoft OpenFrame на Azure](https://azure.microsoft.com/resources/install-tmaxsoft-openframe-on-azure/)

Модернизация инфраструктуры в облачном масштабе. TmaxSoft OpenFrame позволяет легко поднять существующие ресурсы мэйнфрейма и перенести их в Azure. В этой статье Стив Рид (AzureCAT) и Manoj Aerroju (TmaxSoft) объясняют, как создать среду OpenFrame, подходящую для разработки, демонстрации, тестирования и производственных нагрузок.

### <a name="ibm-mainframe-to-microsoft-azure-reference-architecture"></a>[Мейнфрейм IBM для справочной архитектуры Microsoft Azure](https://www.astadia.com/whitepaper/ibm-mainframe-to-microsoft-azure)

Этот информационный документ отражает более чем 25-летний опыт Astadia по модернизации платформы мэйнфреймов. Они объясняют преимущества и проблемы, связанные с модернизацией. В этом руководстве содержится обзор мейнфрейма IBM и эталонной архитектуры IBM mainframe-to-Azure. В нем также содержится представление о методологии успеха Astadia.

### <a name="deploying-mainframe-applications-to-microsoft-azure"></a>[Развертывание мейнфреймов в Microsoft Azure](https://www.microfocus.com/media/white-paper/deploying_mainframe_applications_to_microsoft_azure_wp.pdf)

Решения Micro Focus освобождают вас от ограничений несвободного оборудования и программного обеспечения на мэйнфрейме. В этом руководстве Micro Focus объясняет, как вместо этого развертывать приложения COBOL и PL/I, работающие в мэйнфреймах IBM, в облаке.

### <a name="breathe-new-life-into-mainframes"></a>[Вдохните новую жизнь в мэйнфреймы](https://www.infosys.com/services/modernization/breathe-new-life-mainframes.html)

 Mainframes являются все более сложными для бизнеса для поддержания. В этом информационном документе Infosys и Microsoft освещается выигрышная стратегия успешной миграции мэйнфреймов. Примеры использования и примеры иллюстрируют варианты.

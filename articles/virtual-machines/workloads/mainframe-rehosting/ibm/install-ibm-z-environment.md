# <a name="install-ibm-zdt-devtest-environment-on-azure"></a>Установка IBM zD & T среды разработки и тестирования в Azure

Чтобы создать среду разработки и тестирования для рабочих нагрузок мэйнфреймов в системах IBM Z, можно развернуть IBM Z разработки и тестовой среды (zD & T) в инфраструктуре виртуальных машин (ВМ) Azure как услуга (IaaS).

С помощью zD & T, можно воспользоваться преимуществами экономии средств x86 платформу для менее важных сред разработки и тестирования и затем обновления назад в рабочей среде Z системы Push-уведомлений. Дополнительные сведения см. в разделе [IBM ZD & T инструкции по установке](https://www-01.ibm.com/support/docview.wss?uid=swg24044565#INSTALL).

Azure и Azure Stack поддерживают следующие версии:

- zD & T Personal Edition
- zD & параллельных Sysplex T
- zD & T Enterprise Edition

Все выпуски zD & T запустить только на x86 систем Linux, а не Windows Server. Enterprise Edition поддерживается в Red Hat Enterprise Linux (RHEL) или Ubuntu/Debian. Для Azure доступны образы RHEL и Debian виртуальной Машины.

В этой статье показано, как настроить zD & T Enterprise Edition в Azure, поэтому zD & T Enterprise Edition веб-сервера можно использовать для создания и управления средами. Установка zD & T не устанавливает любой среды. Необходимо создать их отдельно как пакеты установки. Например распределения управляемой разработчикам приложения (ADCD) представляют собой образы томов тестовых сред. Они содержатся в образах zip в корпорации IBM распространения мультимедиа. См. в разделе Практическое [Настройка ADCD среды в Azure](demo.md).

Дополнительные сведения см. в разделе [zD & T Обзор](https://www.ibm.com/support/knowledgecenter/en/SSTQBD_12.0.0/com.ibm.zdt.overview.gs.doc/topics/c_product_overview.html) в центре знаний IBM.

В этой статье показано, как настроить Z разработки и тестовой среды (zD & T) Enterprise Edition в Azure. Затем можно использовать zD & T Enterprise Edition веб-сервер для создания и Z комплексное управление средами в Azure.

## <a name="prerequisites"></a>Технические условия

> [!NOTE]
> IBM разрешает zD & T Enterprise Edition для установки в только в средах разработки и тестирования —*не* рабочих средах.

- Подписка Azure. Если у вас еще нет подписки Azure, создайте [бесплатную учетную запись](https://azure.microsoft.com/free/?WT.mc_id=A261C142F), прежде чем начать работу.

- Вам нужен доступ к мультимедиа, которое доступно только для IBM клиентов и партнеров. Дополнительные сведения, обратитесь к своему представителю IBM или контактные данные см. в разделе на [zD & T](https://www.ibm.com/us-en/marketplace/z-systems-development-test-environment) веб-сайта.

- Объект [сервер лицензирования](https://www.ibm.com/support/knowledgecenter/en/SSTQBD_12.0.0/com.ibm.zsys.rdt.tools.user.guide.doc/topics/zdt_ee.html). Это необходимо для доступа к средам. Способ создания зависит от того, как лицензировать программное обеспечение от IBM:

     - **Сервер лицензирования на основе оборудования** требует устройства USB, содержащий Rational маркеров, необходимые для доступа к все части программного обеспечения. Вам необходимо получить от IBM.

     - **Сервер лицензирования на основе программного обеспечения** необходимо настроить централизованный сервер для управление ключами лицензирования. Этот метод является предпочтительным и требует настройки ключей, получаемых от IBM на сервере управления.

## <a name="create-the-base-image-and-connect"></a>Создание базового образа и подключение

1. На портале Azure [Создание виртуальной Машины](/azure/virtual-machines/linux/quick-create-portal) с необходимой конфигурации операционной системы. В этой статье предполагается, что виртуальная машина B4ms (с 4 виртуальных ЦП и 16 ГБ памяти) под управлением Ubuntu 16.04.

2. После создания виртуальной Машины, откройте входящие порты 22 для SSH, 21, для FTP-сервера и 9443 для веб-сервера.

3. Получение учетных данных SSH, показанный на **Обзор** колонку виртуальной машины с помощью **Connect** кнопки. Выберите **SSH** и скопировать команды входа SSH в буфер обмена.

4. Выполните вход на [оболочку Bash](/azure/cloud-shell/quickstart) из вашего локального компьютера и вставьте команду. Он будет иметь вид **ssh\<идентификатор пользователя\>\@\<IP-адрес\>** . При появлении запроса учетных данных, введите их для установления соединения в домашнем каталоге.

## <a name="copy-the-installation-file-to-the-server"></a>Скопируйте файл установки на сервер

Установочный файл веб-сервера является **ZDT\_установить\_EE\_V12.0.0.1.tgz**. Он включен в носитель, предоставленный компанией IBM. К виртуальной Машине Ubuntu, необходимо передать этот файл.

1. В командной строке введите следующую команду, чтобы убедиться, что все, что находится в актуальном состоянии в только что созданного образа:

    ```
    sudo apt-get update
    ```

2. Создайте каталог для установки:

    ```
    mkdir ZDT
    ```

3. Скопируйте файл с локального компьютера к виртуальной Машине:

    ```
    scp ZDT_Install_EE_V12.0.0.1.tgz  your_userid@<IP Address /ZDT>   =>
    ```
    
> [!NOTE]
> Эта команда копирует файл установки в каталог ZDT в домашнем каталоге, который зависит от того, запускается ли ваш клиент Windows или Linux.

## <a name="install-the-enterprise-edition"></a>Установите выпуск Enterprise

1. Перейдите в каталог ZDT и распаковки ZDT\_установить\_EE\_V12.0.0.1.tgz файл, используя следующие команды:

    ```
    cd ZDT
    chmod 755 ZDT\_Install\_EE\_V12.0.0.0.tgz
    ```

2. Запустите установщик:

    ```
    ./ZDT_Install_EE_V12.0.0.0.x86_64
    ```

3. Выберите **1** Установка Enterprise Server.

4. Нажмите клавишу **ввод** и внимательно читайте лицензионные соглашения. В конце лицензии, введите **Да** для продолжения.

5. При появлении запроса на изменение пароля для созданного пользователя **ibmsys1**, используйте команду **sudo passwd ibmsys1** и введите новый пароль.

6. Чтобы проверить успешность установки введите

    ```
    dpkg -l | grep zdtapp
    ```

7. Убедитесь, что выходные данные содержат строку **zdtapp 12.0.0.0**, означающее, газа пакета успешно установлены

### <a name="starting-enterprise-edition"></a>Начиная с выпуска Enterprise Edition

Имейте в виду, что при запуске веб-сервер, он выполняется под Идентификатором пользователя zD & T, который был создан во время установки.

1. Чтобы запустить веб-сервера, используйте корневой идентификатор пользователя в выполните следующую команду:

    ```
    sudo /opt/ibm/zDT/bin/startServer
    ```

2. Скопируйте выходные данные URL-адрес с помощью сценария, который выглядит следующим образом:

    ```
    https://<your IP address or domain name>:9443/ZDTMC/login.htm
    ```

3. Вставьте URL-адрес в веб-браузер, чтобы открыть компонент управления для вашей установки T & zD.

# <a name="install-ibm-zdt-devtest-environment-on-azure"></a>Установка IBM zD&T dev/test-среде на Azure

Для создания среды для разработки/тестов для рабочих нагрузок на мэйнфрейм на IBM и системах можно развернуть IBM и тестовую среду (zD&T) на инфраструктуре виртуальной машины Azure (VM) в качестве службы (IaaS).

С zD&T вы можете воспользоваться экономией средств платформы x86 для менее критических сред разработки и тестирования, а затем отодвинуть обновления обратно в производственную среду системы. Для получения более подробной [информации,](https://www-01.ibm.com/support/docview.wss?uid=swg24044565#INSTALL)см&.

Azure и Azure Stack поддерживают следующие версии:

- zD&T Личное издание
- zD&T Параллельный Sysplex
- zD&T Enterprise Edition

Все выпуски zD&T работают только на системах x86 Linux, а не на Windows Server. Enterprise Edition поддерживается либо на Red Hat Enterprise Linux (RHEL) или Ubuntu/Debian. Изображения RHEL и Debian VM доступны для Azure.

В этой статье показано, как настроить zD&T Enterprise Edition на Azure, чтобы вы могли использовать веб-сервер zD&T Enterprise Edition для создания и управления средами. Установка zD&T не устанавливает никаких сред. Вы должны создать их отдельно в виде пакетов установки. Например, разработчики приложений, контролируемые дистрибутивами (ADCD) — это объемные изображения тестовых сред. Они содержатся в почтовых изображениях на медиа-дистрибутиве, доступных от IBM. Узнайте, как [настроить среду ADCD в Azure.](demo.md)

Для получения дополнительной информации смотрите [обзор zD&T](https://www.ibm.com/support/knowledgecenter/en/SSTQBD_12.0.0/com.ibm.zdt.overview.gs.doc/topics/c_product_overview.html) в Центре знаний IBM.

В этой статье показано, как настроить корпоративное издание «Разработка и тестовая среда» (zD&T) на Azure. Затем можно использовать веб-сервер zD&T Enterprise Edition для создания и управления средами на основе azure.

## <a name="prerequisites"></a>Предварительные требования

> [!NOTE]
> IBM позволяет zD&T Enterprise Edition, которые будут установлены только в средах dev/test,*а не* в производственных средах.

- Подписка Azure. Если у вас еще нет подписки Azure, создайте [бесплатную учетную запись](https://azure.microsoft.com/free/?WT.mc_id=A261C142F), прежде чем начать работу.

- Вам нужен доступ к средствам массовой информации, которые доступны только клиентам и партнерам IBM. Для получения дополнительной информации свяжитесь с представителем IBM или ознакомьтесь с контактной информацией на веб-сайте [zD&T.](https://www.ibm.com/us-en/marketplace/z-systems-development-test-environment)

- Сервер [лицензирования](https://www.ibm.com/support/knowledgecenter/en/SSTQBD_12.0.0/com.ibm.zsys.rdt.tools.user.guide.doc/topics/zdt_ee.html). Это необходимо для доступа к средам. Способ его создания зависит от того, как вы лицензируете программное обеспечение от IBM:

     - **Сервер лицензирования на основе оборудования** требует USB аппаратного устройства, которое содержит rational токены, необходимые для доступа ко всем частям программного обеспечения. Вы должны получить это от IBM.

     - **Сервер лицензирования на основе программного обеспечения** требует настройки централизованного сервера для управления ключами лицензирования. Этот метод является предпочтительным и требует настройки ключей, которые вы получаете от IBM на сервере управления.

## <a name="create-the-base-image-and-connect"></a>Создание базового изображения и подключение

1. На портале Azure [создайте VM](/azure/virtual-machines/linux/quick-create-portal) с нужной конфигурацией операционной системы. Эта статья предполагает B4ms VM (с 4 vCPUs и 16 ГБ памяти) работает Ubuntu 16.04.

2. После создания VM откройте входящие порты 22 для SSH, 21 для FTP и 9443 для веб-сервера.

3. Получите учетные данные **Overview** SSH, отображаемые на лезвии Обзора VM с помощью кнопки **Connect.** Выберите вкладку **SSH** и скопируйте команду входа SSH на буфер обмена.

4. Зайдите в [оболочку Bash](/azure/cloud-shell/quickstart) с локального компьютера и вставьте команду. Это будет в форме **\<Ssh\>\@\<пользователя\>ID IP-адрес**. Когда вам будет предложено настроить учетные данные, введите их, чтобы установить подключение к каталогу Home.

## <a name="copy-the-installation-file-to-the-server"></a>Копирование файла установки на сервер

Файл установки для веб-сервера является **«DT\_Установить\_EE\_V12.0.0.1.tgz**. Он включен в средства массовой информации, поставляемые IBM. Вы должны загрузить этот файл на ваш Ubuntu VM.

1. Из командной строки введите следующую команду, чтобы убедиться, что все в актуальном состоянии в недавно созданном изображении:

    ```
    sudo apt-get update
    ```

2. Создайте каталог для установки для:

    ```
    mkdir ZDT
    ```

3. Копируйте файл с локальной машины на VM:

    ```
    scp ZDT_Install_EE_V12.0.0.1.tgz  your_userid@<IP Address /ZDT>   =>
    ```
    
> [!NOTE]
> Эта команда копирует файл установки в каталоге здуки «DT» в каталоге Home, который зависит от того, работает ли клиент с Windows или Linux.

## <a name="install-the-enterprise-edition"></a>Установка корпоративного издания

1. Перейдите в каталог здуки и распакуйте файл «DT\_Install\_EE V12.0.0.tgz»\_с использованием следующих команд:

    ```
    cd ZDT
    chmod 755 ZDT\_Install\_EE\_V12.0.0.0.tgz
    ```

2. Выполнить установщик:

    ```
    ./ZDT_Install_EE_V12.0.0.0.x86_64
    ```

3. Выберите **1** для установки Enterprise Server.

4. Press **Enter** и внимательно прочитайте лицензионные соглашения. В конце лицензии введите **Yes** для продолжения.

5. Когда предложено изменить пароль для вновь созданного пользователя, **ibmsys1**, используйте команду **sudo passwd ibmsys1** и введите новый пароль.

6. Проверить, была ли установка успешной,

    ```
    dpkg -l | grep zdtapp
    ```

7. Проверить, что выход содержит строку **zdtapp 12.0.0.0**, указывая, что пакет газа был установлен успешно

### <a name="starting-enterprise-edition"></a>Запуск корпоративного издания

Имейте в виду, что, когда веб-сервер запускается, он работает под zD&T идентификатор пользователя, который был создан в процессе установки.

1. Чтобы запустить веб-сервер, используйте идентификатор root User для выполнения следующей команды:

    ```
    sudo /opt/ibm/zDT/bin/startServer
    ```

2. Копирование вывода URL-адреса по скрипту, который выглядит следующим образом:

    ```
    https://<your IP address or domain name>:9443/ZDTMC/login.htm
    ```

3. Вставьте URL-адрес в веб-браузер, чтобы открыть компонент управления для установки zD&T.

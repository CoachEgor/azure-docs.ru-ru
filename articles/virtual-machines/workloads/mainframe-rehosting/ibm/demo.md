# <a name="set-up-an-application-developers-controlled-distribution-adcd-in-ibm-zdt-v1"></a>Настройка контролируемого дистрибуции разработчиков приложений (ADCD) в IBM zD&T v1

Вы можете запустить среду IBM - Development and Test Environment (zD&T) на виртуальных машинах Azure (VM). Эта среда имитирует архитектуру серии IBM. Он может размещать различные операционные системы или установки серии (также называемых «Инстанции или пакеты»), которые доступны через индивидуальные пакеты, называемые контролируемыми дистрибутивами разработчиков приложений IBM (ADCD).

В этой статье показано, как настроить экземпляр ADCD в среде zD&T в Azure. ADCDs создают полные реализации операционной системы серии серии для сред разработки и тестирования, которые работают в zD&T.

Как и zD&T, ADCDs доступны только клиентам и партнерам IBM и предназначены исключительно для целей разработки и тестирования. Они не должны использоваться для производственных сред. Многочисленные пакеты установки IBM доступны для скачивания через [Passport Advantage](https://www.ibm.com/support/knowledgecenter/en/SSTQBD_12.0.0/com.ibm.zsys.rdt.guide.adcd.doc/topics/installation_ps.html) или [IBM PartnerWorld.](https://www.ibm.com/partnerworld/public)

## <a name="prerequisites"></a>Предварительные требования

- Подписка Azure. Если у вас еще нет подписки Azure, создайте [бесплатную учетную запись](https://azure.microsoft.com/free/?WT.mc_id=A261C142F), прежде чем начать работу.

- Среда [zD&T,][ibm-install-z] ранее созданная на Azure. Эта статья предполагает, что вы используете тот же ubuntu 16.04 VM изображение, созданное ранее.

- Доступ к мультимедиа ADCD через IBM PartnerWorld или Passport Advantage.

- Сервер [лицензирования](https://www.ibm.com/support/knowledgecenter/en/SSTQBD_12.0.0/com.ibm.zsys.rdt.tools.user.guide.doc/topics/zdt_ee.html). Это необходимо для запуска IBM zD&T. Способ его создания зависит от того, как вы лицензируете программное обеспечение от IBM:

  - **Сервер лицензирования на основе оборудования** требует USB аппаратного устройства, которое содержит rational токены, необходимые для доступа ко всем частям программного обеспечения. Вы должны получить это от IBM.

  - **Сервер лицензирования на основе программного обеспечения** требует настройки централизованного сервера для управления ключами лицензирования. Этот метод является предпочтительным и требует настройки ключей, которые вы получаете от IBM на сервере управления.

## <a name="download-the-installation-packages-from-passport-advantage"></a>Скачать пакеты установки из преимущества паспорта

Необходим доступ к носителям ADCD. Приведенные ниже шаги предполагают, что вы являетесь клиентами IBM и можете использовать преимущество паспорта. Партнеры IBM могут использовать [IBM PartnerWorld.](https://www.ibm.com/partnerworld/public)

> [!NOTE]
> В этой статье предполагается, что Windows PC используется для доступа к порталу Azure и загрузки мультимедиа IBM. Если вы используете рабочий стол Mac или Ubuntu, команды и процесс получения носителя IBM могут незначительно отличаться.

1. Войти в [паспорт Преимущество](https://www.ibm.com/software/howtobuy/passportadvantage/paocustomer).

2. Выберите **загрузку программного обеспечения** и **доступ к средствам массовой информации**.

3. Выберите **предложение программы и номер соглашения,** и нажмите **Продолжить**.

4. Введите описание детали или номер детали и нажмите **Finder.**

5. Дополнительно нажмите на алфавитный список заказов, чтобы отобразить и просмотреть продукт по имени.

6. Выберите **все операционные системы** в **поле операционной системы**и **все языки** **на языках**поля. Затем нажмите **Go**.

7. Нажмите **Выберите отдельные файлы,** чтобы расширить список и отобразить отдельные носители для загрузки.

8. Проверьте пакет (ы), который вы хотите загрузить, выберите **Скачать,** а затем скачать файлы в каталог, который вы хотите.

## <a name="upload-the-adcd-packages"></a>Загрузить пакет ADCD (ы)

Теперь, когда у вас есть пакет (ы), вы должны загрузить его на ваш VM на Azure.

1. На портале Azure инициируйте **сеанс ssh** с созданным вами Ubuntu VM. Перейдите на VM, выберите лезвие **Обзор,** а затем выберите **Connect.**

2. Выберите вкладку **SSH,** а затем скопируйте команду ssh на буфер обмена.

3. Зайдите на ваш VM, используя ваши учетные данные и [клиент SSH](/azure/virtual-machines/linux/use-remote-desktop) выбора. Эта демо-версия использует расширения Linux для Windows 10, которая добавляет оболочку Bash в запрос команды Windows. PuTTY работает так же хорошо.

4. При входе в систему создайте каталог для загрузки пакетов IBM. Имейте в виду, Linux является случай чувствительным. Например, эта демо-версия предполагает, что пакеты загружаются в:

        /home/MyUserID/ZDT/adcd/nov2017/volumes

5. Загрузите файлы с помощью клиента SSH, такого как[WinSCP.](https://winscp.net/eng/index.php) Поскольку SCP является частью SSH, он использует порт 22, который является то, что SSH использует. Если ваш локальный компьютер не является Windows, вы можете ввести [команду scp](http://man7.org/linux/man-pages/man1/scp.1.html) в сеансе SSH.

6. Инициировать загрузку в созданный каталог Azure VM, который становится хранилищем изображений для zD&T.

    > [!NOTE]
    > Убедитесь, что **ADCDTOOLS. XML** включен в каталог загрузки в **домашний дом/MyUserID/ЗдТ/Adcd/nov2017.** Оно понадобится вам позднее.

7. Подождите, пока файлы будут загружены, что может занять некоторое время в зависимости от подключения к Azure.

8. Когда загрузка завершена, перейдите в каталог объемов и разгерметите все тома **ГЗ:**

    ```
        gunzip \*.gz
    ```
    
![Исследователь файлов, показывающий разгержаемые объемы ГЦ](media/01-gunzip.png)

## <a name="configure-the-image-storage"></a>Настройка хранилища изображений

Следующим шагом является настройка zD&T для использования загруженного пакета (ы). Процесс хранения изображений в zD&T позволяет монтировать и использовать изображения. Он может использовать SSH или FTP.

1. Запуск **zDTServer**. Для этого вы должны быть на корневом уровне. Введите следующие две команды в порядке:
    ```
        sudo su -
        /opt/ibm/zDT/bin/startServer
    ```
2. Обратите внимание на вывод URL-адреса команды и используйте этот URL для доступа к веб-серверу. Он похож на:
     > https://(ваше имя VM или IP-адрес): 9443/ ЗДТМК/index.html
     >
     > Помните, что ваш веб-доступ использует порт 9443. Используйте это для входа на веб-сервер. Идентификатор пользователя для zD&T является **zdtadmin** и пароль **пароль**.

    ![IBM zD&T Enterprise Edition Добро пожаловать экран](media/02-welcome.png)

3. На странице **Быстрый старт,** под **настройкой,** выберите **Хранилище изображений.**

     ![IBM zD&T Enterprise Edition Быстрый стартовый экран](media/03-quickstart.png)

4. На странице **хранения изображений выберите** **протокол передачи файлов SSH.**

5. Для **имени хост,** введите **Localhost** и введите путь каталога, где вы загрузили изображения. Например, /дом/MyUserID/ЗДТ/адкд/nov2017/тома.

6. Введите **идентификатор пользователя** и **пароль** для VM. Не используйте идентификатор пользователя и пароль пользователя&и пароль.

7. Проверьте соединение, чтобы убедиться, что у вас есть доступ, а затем выберите **Сохранить,** чтобы сохранить конфигурацию.

## <a name="configure-the-target-environments"></a>Настройка целевых сред

Следующим шагом является настройка целевой среды zD&T. Это эмулированной среде, где ваши изображения работают.

1. На странице **«Быстрый старт»** под **настройкой**выберите **среды Target.**

2. На странице **настройки целевых сред** выберите **Добавить цель**.

3. Выберите **Linux**. IBM поддерживает два типа сред, Linux и Облако (OpenStack), но это демо работает на Linux.

4. На странице **целевой среды Добавить,** для **имени host,** введите **localhost**. Держите **SSH порт** установлен до **22**.

5. В поле **метки Target Environment** введите этикетку, такую как **MyCICS.**

     ![Добавление экрана целевой среды](media/04-add-target.png)

## <a name="configure-adcd-and-deploy"></a>Настройка ADCD и развертывание

После завершения предыдущих этапов конфигурации необходимо настроить zD&T для использования пакета (ы) и целевой среды. Опять же, вы используете процесс хранения изображений в zD&T, что позволяет монтировать и использовать изображения. Он может использовать SSH или FTP.

1. На странице **Быстрый старт,** под **настройкой,** выберите **ADCD**. Появляется набор инструкций, в котором указаны шаги, которые необходимо выполнить до установки пакета ADCD. Это объясняет, почему мы назвали целевой каталог так, как мы это делали ранее.

2. Предполагая, что все изображения были загружены в правильные каталоги, нажмите НА IMAGE из ссылки **ADCD,** отображаемой в правом нижнем правом (показано в шаге 7 на следующем скриншоте).

     ![IBM zD&T Enterprise Edition - Настройка экрана ADCD](media/05-adcd.png)

## <a name="create-the-image"></a>Создание образа

Когда предыдущий шаг конфигурации завершен, появляется изображение, используя страницу **Компонентов ADCD.**

1. Выберите том (ноябрь 2017 в данном случае) для отображения различных пакетов, которые находятся в этом объеме.

2. Для этого демо выберите **систему управления информацией о клиентах (CICS) - 5.3**.

3. В поле **имени изображения** введите имя для изображения, такое как **MyCICS Image.**

4. Выберите кнопку **«Создание изображения»** в правом нижнем правом.

     ![IBM zD&T Enterprise Edition - Создайте изображение с помощью экрана компонентов ADCD](media/06-adcd.png)

5. В окне, которое появляется, сообщая вам, что изображение было развернуто успешно, выберите **изображения развертывание.**

6. На **развертывании изображения на странице целевой среды** выберите изображение, созданное на предыдущей странице **(MyCICS Image)** и целевую среду, созданную ранее (**MyCICS**).

7. На следующем экране предоставьте свои учетные данные для VM (т.е. не учетных данных ztadmin).

8. В панели Свойств введите число **центральных процессоров (CPs),** количество **системной памяти (GB)** и **каталог развертывания** для работаемого изображения. Так как это демо, держать его небольшим.

9. Убедитесь, что поле выбрано для **автоматического выпуска команды IPL z/OS после развертывания.**

     ![Экран свойств](media/07-properties.png)

10. Выберите **Полный**.

11. Выберите **изображение развертывания** из **изображения развертывание на страницу целевой среды.**

Изображение теперь может развертываться и готово к установке эмулятором терминала 3270.

> [!NOTE]
> Если вы получили ошибку, заявив, что у вас недостаточно дискового пространства, обратите внимание, что регион требует 151 Гб.

Поздравляем! Теперь вы работаете с средой мейнфрюра IBM в Azure.

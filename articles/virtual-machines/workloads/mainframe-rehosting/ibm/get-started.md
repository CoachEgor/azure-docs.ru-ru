# <a name="ibm-workloads-on-azure"></a>Рабочие нагрузки IBM на Azure

Многие рабочие нагрузки IBM на основе z/OS могут быть воспроизведены в Azure без потери функциональности и без того, чтобы пользователи даже не замечали изменений в своих базовых системах. Повторное размещение приложений в Azure дает вам необходимые функции, похожие на мэйнфрейм, а также эластичность, доступность и потенциальную экономию затрат облака.

Azure поддерживает интеграцию с существующими средами мейнфрейма IBM, позволяя перенести аппликаторы, которые имеют смысл, запускать гибридные решения там, где это необходимо, и мигрировать с течением времени. Хотя можно полностью переписать существующие программы на основе мэйнфрейма для Azure, чаще их перехибывают. Переписывание добавляет затраты, сложность и время для миграционных проектов. С повторного хостинга, вы можете:

- Переместите приложения в облачный эмулятор.

- Переносите базу данных в облачную базу данных.

- Замените модули и код с помощью двигателей преобразования кода.

Кроме того, программное обеспечение IBM, в том числе WebSphere и МЗ, теперь находится в Azure Marketplace. Имея лицензию на программное обеспечение IBM, вы можете воспользоваться преимуществами масштабирования инфраструктуры по требованию, предоставляемой Azure, чтобы быстро запустить виртуальную машину.

Доступна обширная партнерская экосистема, которая поможет перенести мэйнфреймы IBM в Azure. Большинство придерживаются прагматичного подхода к повторному использованию, где это возможно, прежде чем приступать к поэтапному развертыванию перезаписи или замены приложений. Получите дополнительные рекомендации и помощь от партнеров в [центре миграции Azure Mainframe.](https://azure.microsoft.com/migration/mainframe/)

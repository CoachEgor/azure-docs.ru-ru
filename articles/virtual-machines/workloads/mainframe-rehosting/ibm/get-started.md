# <a name="ibm-workloads-on-azure"></a>IBM рабочих нагрузок в Azure

Многих рабочих нагрузок мэйнфреймов IBM для z/ОС можно реплицировать в Azure без потери функциональности и без даже обнаружить изменения в их базовых систем для пользователей. Повторное размещение приложений на платформе Azure позволяет мэйнфреймов по принципу функций, нужно плюс эластичности, доступности, и экономию облака.

Azure предоставляет возможности интеграции с существующими средами мэйнфреймов IBM, что позволяет перенести applicates, который имеет смысла, запуска гибридных решений там, где требуется и перенести со временем. Несмотря на то, что можно полностью переписывать существующие мэйнфрейма программы для Azure, он является более распространенным, повторное размещение их. Перезапись добавляет стоимости, сложности и время проектов миграции. С помощью повторного размещения, можно:

- Переместите приложения в эмулятор на основе облака.

- Перенос базы данных в базы данных на основе облака.

- Замените модули и код, с помощью модулей кода преобразования.

Кроме того программное обеспечение IBM, включая WebSphere и магический Квадрант, теперь находится в Azure Marketplace. Благодаря лицензии на программное обеспечение IBM можно быстро запускать виртуальные машины благодаря предоставленной Azure функции масштабирования инфраструктуры по требованию.

Обширную партнерскую экосистему могут помочь вам перенести системы мэйнфреймов IBM в Azure. Большинство следуют Прагматичный подход повторного использования везде, где это возможно, перед тем, как поэтапного развертывания перезаписи или замена предложений. Получить дополнительные рекомендации и воспользоваться советами от партнеров на [Центр миграции Azure мэйнфреймов](https://azure.microsoft.com/migration/mainframe/).

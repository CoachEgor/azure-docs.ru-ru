# <a name="mainframe-workloads-supported-on-azure"></a>Рабочие нагрузки mainframe, поддерживаемые на Azure

Перечисленные здесь решения помогут перенести рабочую нагрузку мейнфрейма в Microsoft Azure. Некоторые рабочие нагрузки можно перенести без особых трудностей. Другие рабочие нагрузки, зависят от устаревшего программного обеспечения системы, могут быть переразмещены. 

Для получения дополнительной помощи с эмуляцией и службами мейнфрейма обратитесь в [центр миграции Azure Mainframe.](https://azure.microsoft.com/migration/mainframe/)

## <a name="migrate-mainframe-closer-to-azure"></a>Мигрировать мейнфрейм ближе к Azure

- Инструменты разработки и интеграции [Micro Focus Visual COBOL.](https://www.microfocus.com/products/visual-cobol/)
- [Микро Фокус PL / I](https://www.microfocus.com/campaign/download/pli-modernization/) наследие компилятор для платформы .NET, поддерживая мейнфрейм PL / I синтаксис, типы данных и поведение.
- Интеграционная платформа для интеграции [Micro Focus Enterprise Server.](https://www.microfocus.com/products/enterprise-suite/enterprise-server/)
- Инструменты разработки и интеграции [Fujitsu NetCOBOL.](https://www.fujitsu.com/global/products/software/developer-tool/netcobol/)
- Инструменты разработки и интеграции [NTT Data Enterprise COBOL.](https://us.nttdata.com/en/digital/application-development-and-modernization)
- [NTT Open PL/I](https://us.nttdata.com/en/digital/application-development-and-modernization) наследие компилятор для платформы .NET, поддерживая мейнфрейм PL / I синтаксис, типы данных и поведение.
- Инструменты разработки и интеграции [компилятора Raincode COBOL.](https://www.raincode.com/products/cobol/)
- [Компилятор Raincode PL/I](https://www.raincode.com/products/pli/) для платформы .NET поддерживает синтаксис мейнфрейма PL/I, типы данных и поведение.
- [Компилятор Raincode ASM370](https://www.raincode.com/technical-landscape/asm370/) для синтезатора мэйнфрейма 370 и HLASM.
- [ASNA Visual RPG для](https://asna.com/us/products/visual-rpg) компилятора .NET для рамочного модули .NET через подключаемый модуль Visual Studio.
- [Современные системы CTU (COBOL-To-Universal)](https://modernsystems.com/automatic-cobol-to-java-conversion/) инструменты разработки и интеграции.

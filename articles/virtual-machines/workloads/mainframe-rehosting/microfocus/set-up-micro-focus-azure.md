# <a name="install-micro-focus-enterprise-server-40-and-enterprise-developer-40-on-azure"></a>Установка Micro Focus Enterprise Server 4.0 и Enterprise Developer 4.0 на Azure

В этой статье показано, как настроить [Micro Focus Enterprise Server 4.0](https://www.microfocus.com/documentation/enterprise-developer/es30/) и Micro Focus Enterprise Developer [4.0](https://www.microfocus.com/documentation/enterprise-developer/ed_30/) на Azure.

Общей рабочей нагрузкой на Azure является среда разработки и тестирования. Этот сценарий является общим, потому что это так экономически эффективным и легко развернуть и снести. С помощью Enterprise Server Micro Focus создала одну из крупнейших доступных платформ для рехостинга мэйнфреймов. Вы можете запускать рабочие нагрузки z/OS на менее дорогой платформе x86 на Azure, используя виртуальные машины Windows или Linux (VM).

> [!NOTE]
> Скоро: Инструкции по настройке [Micro Focus Enterprise Server 5.0](https://techcommunity.microsoft.com/t5/azurecat/micro-focus-enterprise-server-5-0-quick-start-template-on-azure/ba-p/1160110) на VMs Azure.

В этой настройке используются VMs-м устройствами Azure под управлением Windows Server 2016 с уже установленным изображением Azure Marketplace с microsoft S'L Server 2017. Эта настройка также применяется к Azure Stack.

Соответствующей средой разработки для Enterprise Server является Enterprise Developer, который работает либо на Microsoft Visual Studio 2017, либо позже, Visual Studio Community (бесплатно для скачивания), либо на Eclipse. В этой статье показано, как развернуть его с помощью виртуальной машины Windows Server 2016, которая поставляется с Visual Studio 2017 или позже установлен.

## <a name="prerequisites"></a>Предварительные требования

Перед тем, как приходить к работе, ознакомьтесь с этими предпосылками:

- Подписка Azure. Если у вас еще нет подписки Azure, создайте [бесплатную учетную запись](https://azure.microsoft.com/free/?WT.mc_id=A261C142F), прежде чем начать работу.

- Программное обеспечение Micro Focus и действительная лицензия (или пробная лицензия). Если вы являетесь существующим клиентом Micro Focus, свяжитесь со своим представителем Micro Focus. В противном случае [запросите пробную версию.](https://www.microfocus.com/products/enterprise-suite/enterprise-server/trial/)

- Получите документацию для [корпоративного сервера и корпоративного разработчика.](https://www.microfocus.com/documentation/enterprise-developer/#")

> [!NOTE]
> Наилучшей практикой является создание туннеля виртуальной частной сети (VPN) от сайта или скачкопримежой службы, чтобы вы могли контролировать доступ к VMs-им. Azure.

## <a name="install-enterprise-server"></a>Установка Enterprise Server

1. Для повышения безопасности и управляемости рассмотрите возможность создания новой группы ресурсов только для этого проекта, например, **RGMicroFocusEntServer.** Используйте первую часть имени в Azure, чтобы выбрать тип ресурса, чтобы упростить обнаружение в списке.

2. Создайте виртуальную машину. На Azure Marketplace выберите пожелательную виртуальную машину и операционную систему. Вот рекомендуемая настройка:

    - **Корпоративный сервер**: Выберите ES2 v3 VM (с 2 vCPUs и 16 ГБ памяти) с Windows Server 2016 и установленным сервером SL 2017. Это изображение доступно на рынке Azure Marketplace. Корпоративный сервер также может использовать базу данных Azure S'L.

    - **Корпоративный разработчик**: Выберите B2ms VM (с 2 vCPUs и 8 ГБ памяти) с Windows 10 и Visual Studio установлен. Это изображение доступно на рынке Azure Marketplace.

3. В разделе **Основы** введите имя пользователя и пароль. Выберите **подписку** и **местоположение/регион,** который вы хотели бы использовать для вс-ми. Выберите **RGMicroFocusEntServer** для группы ресурсов.

4. Поместите оба виртуальных в одну и ту же виртуальную сеть, чтобы они могли общаться друг с другом.

5. Примите значения по умолчанию для остальных настроек. Помните имя пользователя и пароль, которые вы создаете для администратора этих ВМ.

6. Когда виртуальные машины были созданы, открыть входящие порты 9003, 86 и 80 для HTTP и 3389 для RDP на машине Enterprise Server и 3389 на машине разработчика.

7. Чтобы войти в виртуальную машину Enterprise Server на портале Azure, выберите ES2 v3 VM. Перейдите в раздел **«Обзор»** и выберите **Connect** для запуска сеанса RDP. Восвайтесь, используя учетные данные, созданные для VM.

8. Из сеанса RDP загрузите следующие два файла. Поскольку вы используете Windows, вы можете перетащить и выбросить файлы в сеанс RDP:

    - **es\_40.exe**, файл установки Enterprise Server.

    - **mflic**, соответствующий файл лицензии-Enterprise Server не будет загружаться без него.

9. Дважды щелкните файл, чтобы начать установку. В первом окне выберите место установки и примите лицензионное соглашение с конечным пользователем.

     ![Экран настройки сервера Micro Focus Enterprise Server](media/01-enterprise-server.png)

     По завершении настройки появляется следующее сообщение:

     ![Экран настройки сервера Micro Focus Enterprise Server](media/02-enterprise-server.png)

### <a name="check-for-updates"></a>Проверка обновлений

После установки обязательно проверяйте наличие дополнительных обновлений, так как вместе с Enterprise Server установлен ряд предпосылок, таких как Redistributable и .NET Framework.

### <a name="upload-the-license"></a>Загрузить лицензию

1. Запустите управление лицензии Micro Focus.

2. Нажмите **Start** \> **Micro Focus Лицензия менеджер** \> **администрации**, а затем нажмите **на вкладку Установить.** Выберите тип формата лицензии для загрузки: лицензионный файл или 16-символ лицензионного кода. Например, для файла в **файле Лицензии**просмотрите файл **mflic,** загруженный ранее в VM, и выберите **Лицензии установки.**

     ![Micro Фокус Лицензия Администрация диалоговая коробка](media/03-enterprise-server.png)

3. Проверить, что корпоративный сервер загружается. Попробуйте запустить сайт Enterprise Server Administration <http://localhost:86/> из браузера, используя этот URL. Страница администрирования корпоративного сервера отображается в показано.

     ![Страница администрирования корпоративных серверов](media/04-enterprise-admin.png)

## <a name="install-enterprise-developer-on-the-developer-machine"></a>Установка корпоративного разработчика на машину разработчика

1. Выберите группу ресурсов, созданную ранее (например, **RGMicroFocusEntServer),** затем выберите изображение разработчика.

2. Чтобы войти в виртуальную машину, перейдите в раздел **Обзор** и выберите **Connect**. Этот знак запускает сеанс RDP. Восвайтесь, используя учетные данные, созданные для VM.

3. Из сеанса RDP загрузите следующие два файла (перетащите и упаж, если хотите):

    - **edvs2017.exe**, файл установки Enterprise Server.

    - **mflic**, соответствующий файл лицензии (Предпринимательский разработчик не будет загружаться без него).

4. Дважды щелкните файл **edvs2017.exe,** чтобы начать установку. В первом окне выберите место установки и примите лицензионное соглашение с конечным пользователем. Если вы хотите, **выберите Установить Rumba 9.5** для установки этого эмулятора терминала, который вам, вероятно, понадобится.

     ![Micro Фокус Предприятие Разработчик для визуальной студии 2017 Настройка диалоговая коробка](media/04-enterprise-server.png)

5. После завершения настройки появляется следующее сообщение:

     ![Настройка успешного сообщения](media/05-enterprise-server.png)

6. Запустите менеджер лицензии Micro Focus так же, как и для Enterprise Server. Выберите **Start** \> **Micro Focus Лицензия менеджер** \> **администрации**и нажмите **на вкладку Установить.**

7. Выберите тип формата лицензии для загрузки: лицензионный файл или 16-символьный лицензионный код. Например, для файла в **файле Лицензии**просмотрите файл **mflic,** загруженный ранее в VM, и выберите **Лицензии установки.**

     ![Micro Фокус Лицензия Администрация диалоговая коробка](media/07-enterprise-server.png)

При загрузке корпоративного разработчика развертывание среды разработки и тестирования Micro Focus в Azure завершено!

# <a name="move-mainframe-storage-to-azure"></a>Переместить хранилище мэйнфреймов в Azure

Для запуска рабочих нагрузок мэйнфреймов на Microsoft Azure, необходимо знать, как сравнить возможности главного компьютера и в Azure. Ресурсы службы хранилища с высокой масштабируемостью может помочь организациям, которые начинаются для модернизации без оставления приложения, которые они используют.

Azure предоставляет функции в стиле мэйнфреймов и емкость хранилища, сравнимой с системами на базе z14 IBM (самые последние модель на момент написания этой статьи). В этой статье рассказывается, как получить сравнимые результаты в Azure.

## <a name="mainframe-storage-at-a-glance"></a>Хранилище мэйнфреймов с первого взгляда

Мэйнфреймов IBM характеризует хранилища двумя способами. Первый — запоминающего устройства прямого доступа (DASD). Второй — последовательного хранения. Для управления хранилищем, мэйнфрейм предоставляет данные средства хранения управления подсистемы (DFSMS). Он управляет доступа к данным для различных устройств хранения.

[DASD](https://en.wikipedia.org/wiki/Direct-access_storage_device) ссылается на отдельное устройство для дополнительного хранилища (не в памяти), что позволяет уникальный адрес для прямого доступа к данным. Первоначально термин DASD применяется к вращается диски, магнитных цилиндры или ячейки с данными. Однако теперь термин также можно применить к твердотельные накопители (SSD), сети хранения (данных SAN), хранилище (NAS) и оптические диски к сети. В целях этого документа DASD ссылается на вращающихся дисках, SANs и SSDs.

В отличие от хранилища DASD последовательного хранилища на мэйнфрейме, ссылается на устройствах, таких как ленточные накопители, где данные организован отправной точки, а затем чтения или записи в строку.

Запоминающие устройства обычно присоединяются с помощью подключения fiber (FICON) или осуществляется непосредственно в с помощью шины главного компьютера и операции ввода-ВЫВОДА [HiperSockets](https://www.ibm.com/support/knowledgecenter/zosbasics/com.ibm.zos.znetwork/znetwork_85.htm), это технология IBM для высокоскоростной связи между разделами на сервере с низкоуровневой оболочки.

Большинство систем мэйнфреймов отдельное хранилище на два типа:

- *Интернет-хранилище* (также известен как "Горячий") необходим для выполнения повседневных операций. DASD хранилище обычно используется для этой цели. Тем не менее последовательные хранилище, такие как ежедневное резервное копирование на ленту (логический или физический), также может использоваться для этой цели.

- *Архивное хранилище* монтируется в определенный момент времени не гарантируется (также называется "холодное" хранилище). Вместо этого он подключается и доступным при необходимости. Архивное хранилище часто реализуется с помощью резервного копирования на ленту, последовательные (логическое или физическое) для хранения.

## <a name="mainframe-versus-io-latency-and-iops"></a>Мэйнфреймов и задержка ввода-ВЫВОДА и операций ввода-ВЫВОДА

Мэйнфреймы часто используются для приложений, требующих высокой производительности операций ввода-ВЫВОДА и малая задержка ввода-ВЫВОДА. Они делают это с помощью подключений FICON устройства ввода-ВЫВОДА и HiperSockets. При использовании HiperSockets приложения и устройства прямого подключения к большой ЭВМ канала ввода-ВЫВОДА достигается задержка в микросекундах.

## <a name="azure-storage-at-a-glance"></a>Краткий обзор службы хранилища Azure

Azure инфраструктуры как услуга ([IaaS](https://azure.microsoft.com/overview/what-is-iaas/)) возможности хранения предоставляют емкость сопоставимых мэйнфреймов.

Корпорация Майкрософт предлагает, накопленные в несколько петабайт хранилище для приложений, размещенных в Azure, и у вас есть несколько вариантов хранения. Эти диапазон из хранилища SSD для обеспечения высокой производительности хранилище обеспечивает экономичное BLOB-объектов для хранения данных и архивы. Кроме того, Azure обеспечивает промежуточный вариант избыточности данных для хранения — то, что требует больших усилий, чтобы настроить в среде мэйнфреймов.

## <a name="azure-hot-online-and-cold-archive-storage"></a>Azure (Интернет) горячего и холодного (архив) хранилища

Тип хранилища для данной системы зависит от требований системы, включая размер хранилища, пропускной способности и операций ввода-ВЫВОДА. Для хранения типа DASD на мэйнфрейме приложений на платформе Azure обычно используйте диск хранилище дисков Azure. Для больших ЭВМ архивное хранилище хранилище BLOB-объектов используется в Azure.

Диски SSD обеспечивают самую высокую производительность хранилища в Azure. (На момент написания этого документа) доступны следующие параметры:

| type         | Размер           | IOPS                  |
|--------------|----------------|-----------------------|
| Диск SSD категории "Ультра"    | 4 ГБ до 64 ТБ  | 1200 в 160,000 операций ввода-ВЫВОДА |
| SSD (цен. категория "Премиум")  | 32 ГБ до 32 ТБ | 12 до 15 000 операций ввода-ВЫВОДА     |
| SSD (цен. категория "Стандартный") | 32 ГБ до 32 ТБ | 12 до 2000 операций ввода-ВЫВОДА      |

Хранилище BLOB-объектов предоставляет наибольший объем хранилища в Azure. В дополнение к размер хранилища Azure предоставляет хранилище с управляемых и неуправляемых. С помощью управляемого хранилища Azure берет на себя управление базовые учетные записи хранения. Неуправляемые хранилище пользователь несет ответственность по настройке учетных записей хранения Azure соответствующего размера для удовлетворения требований к хранилищу.

# <a name="supported-scenarios-for-hana-large-instances"></a>Поддерживаемые сценарии для крупных экземпляров HANA
В этой статье описаны поддерживаемые сценарии и детали архитектуры для HANA Large Instances (HLI).

>[!NOTE]
>Если требуемый сценарий не указан в этой статье, обратитесь в группу управления службами Майкрософт для оценки ваших требований.
Перед настройкой блока HLI проверьте дизайн с помощью SAP или партнера по реализации услуг.

## <a name="terms-and-definitions"></a>Термины и определения
Давайте разберемся с терминами и определениями, которые используются в этой статье:

- **SID**: Системный идентификатор для системы HANA
- **HLI**: Хана Крупных Инстанций
- **DR**: Аварийное восстановление
- **Нормальная DR**: Установка системы с выделенным ресурсом только для целей DR
- **Многоцелевая СИСТЕМА**DR-сайта, настроенная на использование непроизводственной среды наряду с производственным экземпляром, настроенным для события DR 
- **Одноsid**: Система с одним установленным экземпляром
- **Multi-SID**: Система с несколькими моментами настроена; также называется среда MCOS
- **HSR**: Репликация системы SAP HANA

## <a name="overview"></a>Обзор
HANA Large Instances поддерживает различные архитектуры, которые помогут вам выполнить ваши бизнес-требования. Следующие разделы охватывают архитектурные сценарии и детали их конфигурации. 

Производный дизайн архитектуры осуществляется исключительно с точки зрения инфраструктуры, и для развертывания HANA необходимо проконсультироваться с SAP или партнерами по реализации. Если ваши сценарии не перечислены в этой статье, обратитесь в группу учетных записей Майкрософт, чтобы просмотреть архитектуру и найти для вас решение.

> [!NOTE]
> Эти архитектуры полностью соответствуют дизайну интеграции данных (TDI) и поддерживаются SAP.

В этой статье описаны детали двух компонентов в каждой поддерживаемой архитектуре:

- Ethernet
- Хранилище

### <a name="ethernet"></a>Ethernet

Каждый подготовленный сервер поставляется с предварительно настроенными с наборами интерфейсов Ethernet. Интерфейсы Ethernet, настроенные на каждое блок HLI, разделены на четыре типа:

- **A:** Используется для или по доступу к клиенту.
- **B**: Используется для связи узла к узлам. Этот интерфейс настроен на всех серверах (независимо от запрашиваемых топологий), но используется только для сценариев масштабирования.
- **C**: Используется для подключения к узлам к хранению.
- **D**: Используется для подключения устройства узла к iSCSI для установки STONITH. Этот интерфейс настраивается только при запросе установки HSR.  

| Логический интерфейс NIC | Тип SKU | Имя с ОС SUSE | Имя с ОС RHEL | Вариант использования|
| --- | --- | --- | --- | --- |
| Объект | Тип I | eth0.tenant | eno1.tenant | Клиент-к-HLI |
| B | Тип I | eth2.tenant | eno3.tenant | Узлы к узлам|
| C | Тип I | eth1.tenant | eno2.tenant | Узла для хранения |
| D | Тип I | eth4.tenant | eno4.tenant | STONITH |
| Объект | Тип II | vlan\<арендаторАno> | team0.tenant | Клиент-к-HLI |
| B | Тип II | vlan\<tenantNo 2> | team0.tenant+2 | Узлы к узлам|
| C | Тип II | vlan\<tenantNo 1> | team0.tenant+1 | Узла для хранения |
| D | Тип II | vlan\<tenantNo 3> | team0.tenant+3 | STONITH |

Вы выбираете интерфейс на основе топологии, настроенной на блок HLI. Например, интерфейс "B" настроен для связи узла к узлам, что полезно при настройке топологии масштабирования. Этот интерфейс не используется для одного узла, масштабирования конфигураций. Для получения дополнительной информации об использовании интерфейса просмотрите требуемые сценарии (позже в этой статье). 

При необходимости вы можете самостоятельно определить дополнительные карты NIC. Однако конфигурации существующих NICs *не могут* быть изменены.

>[!NOTE]
>Вы можете найти дополнительные интерфейсы, которые являются физическими интерфейсами или склеивания. Следует учитывать только ранее упомянутые интерфейсы для корпуса использования. Любые другие могут быть проигнорированы.

Распределение для единиц с двумя назначенными IP-адресами должно выглядеть следующим образом:

- Ethernet "A" должен иметь назначенный IP-адрес, который находится в диапазоне IP-адресов сервера, который вы представили корпорации Майкрософт. Этот IP-адрес должен быть сохранен в *каталоге /etc/hosts* of the OS.

- Ethernet "C" должен иметь назначенный IP-адрес, который используется для связи с NFS. Этот адрес *не* должен быть сохранен в каталоге *и т.д./хостов,* чтобы разрешить трафик экземпляра в пределах арендатора.

Для репликации системы HANA или развертывания масштаба HANA конфигурация лезвия с двумя назначенными IP-адресами не подходит. Если у вас есть только два назначенных IP-адреса и вы хотите развернуть такую конфигурацию, свяжитесь с SAP HANA по управлению службой Azure. Они могут назначить вам третий IP-адрес в третьем VLAN. Для подразделений HANA Large Instances с тремя назначенными IP-адресами в трех портах NIC применяются следующие правила использования:

- Ethernet "A" должен иметь назначенный IP-адрес, который за пределами диапазона IP-адреса пула сервера, который вы представили корпорации Майкрософт. Этот IP-адрес не должен содержаться в каталоге *Ит-хостов* ОС.

- Ethernet "B" должен поддерживаться исключительно в *каталоге и т.д./хозяев* для связи между различными экземплярами. Это IP-адреса, которые должны поддерживаться в масштабных конфигурациях HANA в качестве IP-адресов, которые HANA использует для межузловой конфигурации.

- Ethernet "C" должен иметь назначенный IP-адрес, который используется для связи с хранением NFS. Этот тип адреса не должен поддерживаться в каталоге *и т.д./хозяев.*

- Ethernet "D" следует использовать исключительно для доступа к устройствам STONITH для Pacemaker. Этот интерфейс необходим при настройке репликации системы HANA и хотите добиться автоматического сбоя операционной системы с помощью устройства на основе SBD.


### <a name="storage"></a>Хранилище
Хранение предварительно настроено на основе запрашиваемых топологических данных. Размеры громкости и точки крепления варьируются в зависимости от количества серверов, количества SkUS и настроенной топологии. Для получения дополнительной информации просмотрите требуемые сценарии (позже в этой статье). Если вам требуется больше памяти, вы можете приобрести его с шагом в 1-ТБ.

>[!NOTE]
>Точка крепления /usr/sap/\<SID> является символической ссылкой на /hana/shared точку крепления.

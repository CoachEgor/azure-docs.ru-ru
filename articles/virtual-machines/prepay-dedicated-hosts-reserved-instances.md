# <a name="save-costs-with-a-reserved-instance-of-azure-dedicated-hosts"></a>Экономия затрат с помощью зарезервированных экземпляров выделенных хостов Azure

При фиксации зарезервированного экземпляра выделенных узлов Azure можно сэкономить деньги. Скидка на бронирование автоматически применяется к числу выделенных узлов, которые соответствуют сфере бронирования и атрибутам. Вам не нужно назначать бронирование специальному хозяину, чтобы получить скидки. Зарезервированная покупка экземпляра охватывает только вычислительную часть вашего использования и включает расходы на лицензирование программного обеспечения. Смотрите [обзор выделенных хостов Azure для виртуальных машин.](https://docs.microsoft.com/azure/virtual-machines/windows/dedicated-hosts)

## <a name="determine-the-right-dedicated-host-sku-before-you-buy"></a>Определите правильный выделенный хост SKU перед покупкой


Перед покупкой бронирования необходимо определить, какой специальный хост вам нужен. SKU определяется для выделенного хоста, представляющего серию и тип VM. 

Начните с переопределения поддерживаемых размеров для [виртуальной машины Windows](https://docs.microsoft.com/azure/virtual-machines/windows/sizes) или [Linux](https://docs.microsoft.com/azure/virtual-machines/linux/sizes?toc=%2fazure%2fvirtual-machines%2flinux%2ftoc.json) для определения серии VM.

Затем проверьте, поддерживается ли он в выделенных хостах Azure. [Страница цен Azure Dedicated Hosts](https://aka.ms/ADHPricing) содержит полный список выделенных хостов SKUS, информацию о процессорах и различные варианты ценообразования (включая зарезервированные экземпляры).

Вы можете найти несколько SKUs, поддерживающих вам серию VM (с различными типами). Определите лучший SKU, сравнив емкость хоста (количество vCPUs). Обратите внимание, что вы сможете применить бронирование для нескольких выделенных хостов SKUs, поддерживающих одну и ту же серию VM (например, DSv3_Type1 и DSv3_Type2), но не в различных сериях VM (например, DSv3 и ESv3).



## <a name="purchase-restriction-considerations"></a>Соображения ограничения на покупку

Зарезервированные экземпляры доступны для большинства выделенных размеров хоста, за некоторыми исключениями.

Скидки на бронирование не распространяются на следующие:

- **Облака** - Бронирование не доступно для покупки в Германии или Китае регионов.

- **Недостаточная квота** - резервирование, которое присваивается одной подписке, должно иметь квоту vCPU, доступную в подписке для нового зарезервированного экземпляра. Например, если целевая подписка имеет ограничение квоты 10 vCPUs для DSv3-Series, то вы не можете купить специальные хосты бронирования, поддерживающие эту серию. Проверка квот для бронирования включает в себя VMs и выделенные хосты, уже развернутые в подписке. Для решения этой проблемы можно [создать запрос на](https://docs.microsoft.com/azure/azure-supportability/resource-manager-core-quotas-request) увеличение квот.

- **Ограничения емкости** - В редких случаях Azure ограничивает покупку новых резерваций для подмножества специализированных хост-скюсов из-за низкой емкости в регионе.

## <a name="buy-a-reservation"></a>Покупка резервирования

На [портале Azure](https://portal.azure.com/#blade/Microsoft_Azure_Reservations/CreateBlade/referrer/documentation/filters/%7B%22reservedResourceType%22%3A%22VirtualMachines%22%7D)можно приобрести зарезервированный экземпляр экземпляра выделенного хоста Azure.

Оплата бронирования [авансом или ежемесячными платежами.](https://docs.microsoft.com/azure/billing/billing-monthly-payments-reservations) Эти требования применяются к покупке зарезервированного экземпляра Dedicated Host:

- Вы должны быть в роли владельца, по крайней мере, одна подписка EA или подписка с оплатой по мере использования.

- Для подписок EA опция **Add Reserved Instances** должна быть включена на [портале EA.](https://ea.azure.com/) Если этот параметр отключен, необходимо быть администратором подписки EA.

- В рамках программы для поставщиков облачных решений (CSP) только агенты администрирования или агенты продаж могут приобрести резервирования.

Требования для приобретения экземпляра.

1. Войдите на  [портал Azure](https://portal.azure.com/).

2. Выберите **Все услуги** \> **Бронирование**.

3. Выберите **Добавить,** чтобы купить новое бронирование, а затем нажмите **Выделенные Хосты**.

4. Заполните обязательные поля. Запуск экземпляров Выделенных Хостов, которые соответствуют выбранным атрибутам, чтобы получить скидку на бронирование. Фактическое количество экземпляров выделенного хоста, которые получают скидку, зависит от выбранной области и количества.

Если у вас есть соглашение EA, можно использовать **опцию** Добавить больше для быстрого добавления дополнительных экземпляров. Опция недоступна для других типов подписки.

## <a name="usage-data-and-reservation-utilization"></a>Данные об использовании и использовании резервирования

В данных об использовании указана нулевая эффективная цена за использование, на которое распространяется скидка на резервирование. Вы можете увидеть, какой экземпляр VM получил скидку на бронирование для каждого бронирования.

Для получения дополнительной информации о том, как скидки на резервирование отображаются в данных об использовании, [см.](https://docs.microsoft.com/azure/billing/billing-understand-reserved-instance-usage-ea)  Если у вас есть индивидуальная подписка, [см. Использование бронирования Azure для подписки Pay-As-You-Go.](https://docs.microsoft.com/azure/billing/billing-understand-reserved-instance-usage)

## <a name="change-a-reservation-after-purchase"></a>Изменение бронирования после покупки

После приобретения в резервирование можно внести следующие типы изменений:

- Обновить область резервирования

- Гибкость размера экземпляра (если это применимо)

- владельца;

Можно также разделить резервирование на более мелкие куски и объединить уже разделенные резервии. Ни одно из изменений не приводит к новой коммерческой транзакции или изменению даты окончания резервирования.

Вы не можете внести следующие типы изменений после покупки, непосредственно:

- Существующий регион резервации

- номер SKU

- Количество

- Duration

Тем не менее, вы можете *обменять* бронирование, если вы хотите внести изменения.

## <a name="cancel-exchange-or-refund-reservations"></a>Отмена, обмен резервирования, возмещение средств за резервирование

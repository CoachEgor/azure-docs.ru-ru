# <a name="in-place-change-of-sql-server-edition-on-azure-vm"></a>Смена на месте издания сервера S'L на Azure VM

В этой статье описывается, как изменить издание сервера S'L на виртуальной машине Windows в Azure. 

Издание S'L Server определяется ключом продукта и определяется в процессе установки с использованием носителя установки. Издание диктует, какие [функции](/sql/sql-server/editions-and-components-of-sql-server-2017) доступны в продукте S'L Server. Вы можете изменить издание s'L Server с помощью носителей установки и либо понизить его, чтобы снизить стоимость, либо обновить, чтобы включить больше функций.

После того, как издание сервера S'L server было изменено внутренне на VM сервера S'L, необходимо обновить свойство издания сервера S'L на портале Azure для целей выставления счетов. 

## <a name="prerequisites"></a>Предварительные требования

Для изменения в издании сервера S'L необходимо следующее: 

- [Подписка Azure](https://azure.microsoft.com/free/).
- [VM сервера S'L на Windows,](https://docs.microsoft.com/azure/virtual-machines/windows/sql/virtual-machines-windows-portal-sql-server-provision) зарегистрированный с [поставщиком ресурсов S'L VM.](virtual-machines-windows-sql-register-with-resource-provider.md)
- Настройка мультимедиа с **желаемым изданием** сервера S'L. Клиенты, имеющие [software Assurance,](https://www.microsoft.com/licensing/licensing-programs/software-assurance-default) могут получить свои средства установки в [Центре лицензирования объемов.](https://www.microsoft.com/Licensing/servicecenter/default.aspx) Клиенты, у которых нет программного обеспечения, могут использовать медиа-носитель настройки с изображения Azure Marketplace S'L Server VM, которое имеет желаемое издание (обычно находится в). `C:\SQLServerFull` 


## <a name="upgrade-an-edition"></a>Обновление издания

> [!WARNING]
> Обновление издания сервера S'L позволит перезапустить службу для сервера S'L, наряду с любыми сопутствующими службами, такими как аналитические службы и R-сервисы. 

Для обновления издании сервера S'L, получить медиа-носитель настройки сервера S'L для желаемого издания сервера S'L, а затем сделать следующее:

1. Откройте Setup.exe от средств установки сервера S'L. 
1. Перейдите на **техническое обслуживание** и выберите вариант **обновления Edition.** 

   ![Выбор для обновления издания сервера S'L](media/virtual-machines-windows-sql-change-edition/edition-upgrade.png)

1. Выберите **Далее,** пока не достигнете **страницы Готово обновить издание,** а затем выберите **Обновление.** Окно настройки может перестать отвечать в течение нескольких минут, пока изменение вступит в силу. **Полная** страница подтвердит, что обновление издания завершено. 

После обновления выпуска сервера S'L, измените свойство издания виртуальной машины сервера S'L Server на портале Azure. Это позволит обновить метаданные и выставление счетов, связанных с этим VM.

## <a name="downgrade-an-edition"></a>Понизить издание

Чтобы понизить рейтинг сервера S'L Server, необходимо полностью удалить сервер S'L и переустановить его снова с помощью желаемого носителя настройки издания. 

> [!WARNING]
> Установка сервера S'L может привести к дополнительным простоям. 

Вы можете понизить выпуск сервера S'L, выполнив следующие действия:

1. Резервное копирование всех баз данных, включая системные базы данных. 
1. Перемещение баз данных системы (мастер, модель и msdb) на новое место. 
1. Полностью удалить сервер S'L и все связанные с ними службы. 
1. Перезапустите виртуальную машину. 
1. Установка сервера S'L, используя средства массовой информации с желаемым изданием сервера S'L.
1. Установите новейшие пакеты услуг и кумулятивные обновления.  
1. Замените новые системные базы данных, созданные во время установки, системными базами данных, которые ранее были перемещены в другое место. 

После понижения рейтинга сервера S'L Server измените свойство издании виртуальной машины сервера S'L Server на портале Azure. Это позволит обновить метаданные и выставление счетов, связанных с этим VM.

## <a name="change-edition-in-portal"></a>Изменение издания на портале 

После того, как вы изменили издание S'L Server с помощью установки носителя, и вы зарегистрировали ваш S'L Server VM с [поставщиком ресурсов S'L VM,](virtual-machines-windows-sql-register-with-resource-provider.md)вы можете использовать портал Azure для изменения свойства Edition в S'L Server VM для целей выставления счетов. Для этого выполните следующие действия. 

1. Войдите на [портал Azure](https://portal.azure.com). 
1. Перейдите на свой виртуальный машинный ресурс S'L Server. 
1. Под **настройками**выберите **настройку.** Затем выберите желаемое издание сервера S'L из списка выпадающих версий под **Edition.** 

   ![Метаданные изменения издания](media/virtual-machines-windows-sql-change-edition/edition-change-in-portal.png)

1. Просмотрите предупреждение, в которое говорится, что сначала необходимо изменить издание s'L Server, и что свойство издания должно соответствовать изданию S'L Server. 
1. Выберите **применить для** применения изменений метаданных издания. 


## <a name="remarks"></a>Remarks

- Свойство издания для S'L Server VM должно соответствовать изданию экземпляра экземпляра сервера S'L, установленного для всех виртуальных машин S'L Server, включая как платные по мере по мере продажи, так и собственные типы лицензий.
- Если вы отбросите ресурс S'L Server VM, вы вернетесь к жестко закодированному выпуску изображения.
- Возможность изменения издания является особенностью поставщика ресурсов S'L VM. Развертывание изображения Azure Marketplace через портал Azure автоматически регистрирует VM сервера S'L с поставщиком ресурсов. Тем не менее, клиенты, которые самостоятельно устанавливают сервер S'L Server, должны будут вручную [зарегистрировать свой VM сервера S'L.](virtual-machines-windows-sql-register-with-resource-provider.md)
- Добавление VM сервера S'L в набор доступности требует воссоздания VM. Любые ВМ, добавленные в набор доступности, будут возвращены в издание по умолчанию, и издание необходимо будет изменить снова.

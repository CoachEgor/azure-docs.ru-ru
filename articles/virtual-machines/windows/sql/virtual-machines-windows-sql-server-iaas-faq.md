# <a name="frequently-asked-questions-for-sql-server-running-on-windows-virtual-machines-in-azure"></a>Часто задаваемые вопросы об SQL Server на виртуальных машинах Windows в Azure

> [!div class="op_single_selector"]
> * [Windows](virtual-machines-windows-sql-server-iaas-faq.md)
> * [Linux](../../linux/sql/sql-server-linux-faq.md)

В этой статье содержатся ответы на некоторые распространенные вопросы о запуске [SQL Server на виртуальных машинах Windows в Azure](https://azure.microsoft.com/services/virtual-machines/sql-server/).

> [!NOTE]
> Эта статья посвящена проблемам с SQL Server на виртуальных машинах Windows. Если вы используете SQL Server на виртуальных машинах Linux, см. [вопросы и ответы о виртуальных машинах Linux](../../linux/sql/sql-server-linux-faq.md).

## <a name="images"></a><a id="images"></a>Изображения

1. **Какие образы доступны в коллекции виртуальных машин SQL Server?** 

   В Azure доступны образы виртуальных машин для всех поддерживаемых основных выпусков SQL Server во всех выпусках Windows и Linux. Дополнительные сведения см. в полном списке образов виртуальных машин [Windows](virtual-machines-windows-sql-server-iaas-overview.md#payasyougo) и [Linux](../../linux/sql/sql-server-linux-virtual-machines-overview.md#create).

1. **Обновляются ли доступные образы в коллекции виртуальных машин SQL Server?**

   Каждые два месяца образы SQL Server в коллекции виртуальных машин обновляются до последних версий Windows и Linux. Образы Windows содержат все обновления, отмеченные как важные в Центре обновления Windows, в частности важные обновления системы безопасности и пакеты обновления SQL Server. Образы Linux содержат последние обновления системы. Накопительные пакеты обновления SQL Server в Linux и Windows обрабатываются по-разному. Для Linux накопительные пакеты обновления SQL Server также включены в обновление. Но виртуальные машины Windows сейчас не поддерживают накопительные пакеты обновления для SQL Server или Windows Server.

1. **Можно ли удалить образы виртуальных машин SQL Server из коллекции?**

   Да. Azure поддерживает только один образ для основного номера версии и выпуска. Например, при выпуске нового пакета обновления SQL Server в коллекцию Azure для него добавляется новый образ. Образ SQL Server для предыдущего пакета обновления немедленно удаляется с портала Azure. Но он доступен для подготовки из PowerShell в течение следующих трех месяцев. После трех месяцев предыдущий образ пакета обновления становится недоступным. Эта политика удаления также применяется, если версия SQL Server перестает поддерживаться по окончании жизненного цикла.


1. **Можно ли развернуть старое изображение сервера S'L, которое не отображается на портале Azure?**

   Да, с помощью PowerShell. Дополнительные сведения о том, как развернуть виртуальные машины SQL Server с помощью PowerShell, см. в статье [Как подготовить виртуальные машины SQL Server с помощью Azure PowerShell](virtual-machines-windows-ps-sql-create.md).

1. **Могу ли я создать обобщенное изображение сервера Azure S'L Server Marketplace, в нее, и использовать его для развертывания vMs?**

   Да, но затем необходимо [зарегистрировать каждый S'L Server VM с поставщиком ресурсов S'L Server VM](virtual-machines-windows-sql-register-with-resource-provider.md) для управления вашим S'L Server VM на портале, а также использовать такие функции, как автоматическое исправление и автоматическое резервное копирование. При регистрации в поставщике ресурсов необходимо также указать тип лицензии для каждого VM сервера S'L. 

1. **Могу ли я использовать свой собственный VHD для развертывания VM сервера S'L?**

   Да, но затем необходимо [зарегистрировать каждый S'L Server VM с поставщиком ресурсов S'L Server VM](virtual-machines-windows-sql-register-with-resource-provider.md) для управления вашим S'L Server VM на портале, а также использовать такие функции, как автоматическое исправление и автоматическое резервное копирование.

1. **Можно ли настраивать конфигурации, которых нет в коллекции виртуальных машин (например Windows 2008 R2 + SQL Server 2012)?**

   Нет. С помощью портала Azure или [PowerShell](virtual-machines-windows-ps-sql-create.md) вы можете выбрать только один из представленных в коллекции образов виртуальных машин с SQL Server. Тем не менее, у вас есть возможность развернуть Windows VM и самостоятельно установить сервер S'L к нему. Затем необходимо [зарегистрировать свой S'L Server VM с помощью поставщика ресурсов S'L Server VM](virtual-machines-windows-sql-register-with-resource-provider.md) для управления Вашим S'L Server VM на портале, а также использовать такие функции, как автоматическое исправление и автоматическое резервное копирование. 


## <a name="creation"></a>Создание

1. **Как создать виртуальную машину Azure с SQL Server?**

   Самый простой способ заключается в создании виртуальной машины, которая включает в себя сервер S'L. В учебнике по регистрации на Azure и созданию VM сервера S'L с момента создания с моделям [S'L Server на портале Azure](virtual-machines-windows-portal-sql-server-provision.md)см. Вы можете выбрать образ виртуальной машины, использующей лицензирование SQL Server с посекундной оплатой, или же образ, который позволяет применить собственную лицензию SQL Server. Кроме того, вы можете вручную установить SQL Server на виртуальной машине, используя выпуск с бесплатной лицензией (Developer или Express) или предоставив существующую локальную лицензию. Обязательно [зарегистрируйте свой S'L Server VM с помощью поставщика ресурсов S'L Server VM](virtual-machines-windows-sql-register-with-resource-provider.md) для управления вашим S'L Server VM на портале, а также используйте такие функции, как автоматическое исправление и автоматическое резервное копирование. При использовании собственной лицензии у вас должно быть преимущество [Перемещение лицензий в рамках программы Software Assurance в Azure](https://azure.microsoft.com/pricing/license-mobility/). Дополнительные сведения см. в [руководстве по выбору ценовой категории для виртуальных машин Azure SQL Server](virtual-machines-windows-sql-server-pricing-guidance.md).

1. **Как перенести локальную базу данных SQL Server в облако?**

   Сначала создайте виртуальную машину Azure с экземпляром SQL Server. Затем перенесите локальные базы данных в этот экземпляр. Стратегии переноса данных см. в статье [Миграция базы данных SQL Server в экземпляр SQL Server на виртуальной машине Azure](virtual-machines-windows-migrate-sql.md).

## <a name="licensing"></a>Лицензирование

1. **Как установить лицензированную копию SQL Server в виртуальной машине Azure?**

   Есть три способа сделать это. Если вы являетесь клиентом корпоративного соглашения (EA), вы можете предоставить одно из [виртуальных машинных изображений, поддерживающих лицензии,](virtual-machines-windows-sql-server-iaas-overview.md#BYOL)которое также известно как bring-your-own-license (BYOL). Если у вас есть [гарантия программного обеспечения,](https://www.microsoft.com/en-us/licensing/licensing-programs/software-assurance-default)вы можете включить [гибридное преимущество Azure](virtual-machines-windows-sql-ahb.md) на существующем изображении с оплатой по мере оплаты (PAYG). Или вы можете скопировать среду установки сервера S'L на Windows Server VM, а затем установить сервер S'L на VM. Обязательно зарегистрируйте свой S'L Server VM у [поставщика ресурсов](virtual-machines-windows-sql-register-with-resource-provider.md) для таких функций, как управление порталом, автоматическое резервное копирование и автоматическое исправление. 

1. **Можно ли настроить виртуальную машину для использования собственной лицензии SQL Server, если она была создана из одного из образов коллекции с оплатой по мере использования?**

   Да. Вы можете легко переключить изображение галереи с оплатой по мере вас (PAYG), чтобы принести свою собственную лицензию (BYOL), включив [гибридную выгоду Azure.](https://azure.microsoft.com/pricing/hybrid-benefit/faq/)  Дополнительные сведения см. в разделе [Изменение модели лицензирования для виртуальной машины SQL Server в Azure](virtual-machines-windows-sql-ahb.md). Сейчас эта возможность доступна только клиентам общедоступного облака.

1. **Будет ли переход на другую модель лицензирования сопровождаться простоем SQL Server?**

   Нет. [Изменение модели лицензирования](virtual-machines-windows-sql-ahb.md) не требует простоев для сервера S'L, так как изменение вступает в силу немедленно и не требует перезагрузки VM. Тем не менее, для регистрации вашего S'L Server VM с поставщиком ресурсов S'L Server VM, [расширение S'L IaaS](virtual-machines-windows-sql-server-agent-extension.md) является необходимым условием, и установка расширения S'L IaaS в _полном_ режиме перезапускает сервис S'L Server. Таким образом, если необходимо установить расширение S'L IaaS, либо установите его в _легком_ режиме для ограниченной функциональности, либо установите его в _полном_ режиме во время окна обслуживания. Расширение IaaS, установленное в _легком_ режиме, может быть переведено в _полный_ режим в любое время, но требует перезапуска службы Сервера S'L. 
   
1. **Можно ли переключить модель лицензирования на ВМ сервера S'L, развернутую с помощью классической модели?**

   Нет. Изменение модели лицензирования не поддерживается на классическом VM. Вы можете перенести VM в модель менеджера ресурсов Azure и зарегистрироваться в поставщике ресурсов S'L Server VM. После регистрации VM в ресурсном провайдере S'L Server VM на VM будут доступны изменения в модели лицензирования.

1. **Могу ли я использовать портал Azure для управления несколькими экземплярами на одном и том же VM?**

   Нет. Управление порталом — это функция, предоставляемая поставщиком ресурсов S'L Server VM, который опирается на расширение агента S'L Server IaaS. Таким образом, те же ограничения применяются к поставщику ресурсов, как и к расширению. Портал может управлять только одним экземпляром по умолчанию, либо одним названным экземпляром, если он был настроен правильно. Для получения более подробной [SQL Server IaaS agent extension](virtual-machines-windows-sql-server-agent-extension.md)информации об этих ограничениях, см. 

1. **Могут ли из подписок CSP активировать Преимущество гибридного использования Azure?**

   Да, Преимущество гибридного использования Azure доступно для подписок CSP. Клиентам CSP следует сначала развернуть образ с оплатой по мере использования, а затем [изменить модель лицензирования](virtual-machines-windows-sql-ahb.md), чтобы использовать собственную лицензию.
   
 
1. **Нужно ли платить за лицензирование SQL Server на виртуальной машине Azure, если она используется только для резервирования или отработки отказа?**

   Чтобы иметь бесплатную пассивную лицензию для группы резервного вторичного доступа или failover кластерных экземпляр, вы должны соответствовать всем следующим критериям, изложенным в [условиях лицензирования продукта:](https://www.microsoft.com/licensing/product-licensing/products)

   1. У вас есть [мобильность лицензии](https://www.microsoft.com/licensing/licensing-programs/software-assurance-license-mobility?activetab=software-assurance-license-mobility-pivot:primaryr2) через [Software Assurance](https://www.microsoft.com/licensing/licensing-programs/software-assurance-default?activetab=software-assurance-default-pivot%3aprimaryr3). 
   1. Пассивный экземпляр s'L Server не обслуживает данные сервера S'L для клиентов или выполняет активные рабочие нагрузки s'L Server. Он используется только для синхронизации с основным сервером и в противном случае поддерживать пассивную базу данных в теплом состоянии ожидания. Если он обслуживает данные, например отчеты клиентам, выполняющим активные рабочие нагрузки s'L Server, или выполняющие любую работу, кроме того, что указано в терминах продукта, это должен быть платный лицензированный экземпляр S'L Server. На вторичном экземпляре допускается следующее действие: проверка согласованности базы данных или CheckDB, полное резервное копирование, резервные копии журналов транзакций и мониторинг данных об использовании ресурсов. Вы также можете запускать основной и соответствующий экземпляр аварийного восстановления одновременно в течение коротких периодов тестирования аварийного восстановления каждые 90 дней. 
   1. Активная лицензия S'L Server покрывается Программным обеспечением и позволяет использовать **один** пассивный вторичный экземпляр S'L Server, с таким же количеством вычислений, как и лицензированный активный сервер. 
   1. Вторичный Сервер VM использует лицензию [аварийного восстановления](virtual-machines-windows-sql-high-availability-dr.md#free-dr-replica-in-azure) на портале Azure.
   
1. **Что считается пассивным экземпляром?**

   Пассивный экземпляр s'L Server не обслуживает данные сервера S'L для клиентов или выполняет активные рабочие нагрузки s'L Server. Он используется только для синхронизации с основным сервером и в противном случае поддерживать пассивную базу данных в теплом состоянии ожидания. Если он обслуживает данные, например отчеты клиентам, выполняющим активные рабочие нагрузки s'L Server, или выполняющие любую работу, кроме того, что указано в терминах продукта, это должен быть платный лицензированный экземпляр S'L Server. На вторичном экземпляре допускается следующее действие: проверка согласованности базы данных или CheckDB, полное резервное копирование, резервные копии журналов транзакций и мониторинг данных об использовании ресурсов. Вы также можете запускать основной и соответствующий экземпляр аварийного восстановления одновременно в течение коротких периодов тестирования аварийного восстановления каждые 90 дней.
   

1. **Какие сценарии могут использовать преимущества Distaster Recovery (DR)?**

   В [руководстве по лицензированию](https://aka.ms/sql2019licenseguide) приводятся сценарии, в которых можно использовать преимущество в случае аварийного восстановления. Обратитесь к условиям продукта и поговорите с вашими контактами по лицензированию или менеджером по учетным данным для получения дополнительной информации.

1. **Какие подписки поддерживают преимущества аварийного восстановления (DR)?**

   Комплексные программы, которые предлагают эквивалентные права подписки Software Assurance в качестве фиксированной выгоды, поддерживают преимущества DR. Это включает в себя. но не ограничивается, Открытое значение (OV), подписка на open Value (OVS), Enterprise Agreement (EA), Соглашение о корпоративной подписке (EAS), и Регистрация серверов и облаков (SCE). Обратитесь к [условиям продукта](https://www.microsoft.com/licensing/product-licensing/products) и поговорите со своими контактами лицензирования или менеджером по лицензированию для получения дополнительной информации. 

   
 ## <a name="resource-provider"></a>Поставщик ресурсов

1. **Принесет ли регистрация моего VM новым поставщиком ресурсов S'L Server VM дополнительными расходами?**

   Нет. Поставщик ресурсов S'L Server VM просто обеспечивает дополнительную управляемость для сервера S'L на Azure VM без каких-либо дополнительных затрат. 

1. **Доступен ли поставщик ресурсов S'L Server VM для всех клиентов?**
 
   Да, до тех пор, пока ВМ сервера S'L была развернута в общедоступном облаке с помощью модели Resource Manager, а не классической модели. Все остальные клиенты могут зарегистрироваться в новом поставщике ресурсов S'L Server VM. Однако использовать собственную лицензию могут только клиенты, имеющие преимущество в обеспечении [программного обеспечения,](https://www.microsoft.com/licensing/licensing-programs/software-assurance-default?activetab=software-assurance-default-pivot%3aprimaryr3) путем активации [гибридной выгоды Azure (AHB)](https://azure.microsoft.com/pricing/hybrid-benefit/) на VM сервера S'L. 

1. **Что происходит с ресурсо-провайдером _(Ресурсом Microsoft.SqlVirtualMachine),_ если ресурс VM перемещается или отбрасывается?** 

   Когда ресурс Microsoft.Compute/VirtualMachine удаляется или перемещается, связанному с ним ресурсу виртуальной машины Microsoft.Sql приходит уведомление об асинхронной репликации операции.

1. **Что произойдет с VM, если ресурсный провайдер _(Microsoft.SqlVirtualMachine)_ ресурс будет удален?**

    Удаление ресурса Microsoft.SqlVirtualMachine не влияет на ресурс Microsoft.Compute/VirtualMachine. Тем не менее изменения лицензирования по умолчанию возвращают исходный источник образов. 

1. **Можно ли зарегистрировать саморазвернутые vMs сервера S'L с помощью поставщика ресурсов S'L Server VM?**

    Да. Если вы развернули SQL Server с собственного носителя и установили расширение SQL IaaS, вы можете зарегистрировать виртуальную машину SQL в поставщике ресурсов, чтобы получить преимущества управления, предоставляемые расширением SQL IaaS. Тем не менее, вы не можете преобразовать саморазвернутый VM сервера S'L в оплату по мере входном оплаты.


   


## <a name="administration"></a>Администрирование

1. **Могу ли я установить второй экземпляр сервера S'L на том же VM? Могу ли я изменить установленные функции экземпляра по умолчанию?**

   Да. Носитель установки Сервера S'L расположен в папке на диске **C.** Выполнить **Setup.exe** из этого места, чтобы добавить новые экземпляры сервера S'L на машине или изменить другие установленные функции сервера S'L. Обратите внимание, что некоторые функции, такие как автоматическое резервное копирование, автоматизированное исправление и интеграция убежища ключей Azure, работают только в отношении экземпляра по умолчанию или названного экземпляра, который был настроен правильно (см. Вопрос 3). 

1. **Можно ли удалить экземпляр по умолчанию SQL Server?**

   Да, но нужно учесть некоторые факторы. Во-первых, биллинг, связанный с сервером, может по-прежнему происходить в зависимости от модели лицензии для VM. Во-вторых, как указано в предыдущем ответе, есть функции, которые опираются на [расширение агента Сервера IaaS.](virtual-machines-windows-sql-server-agent-extension.md) Если вы удалите экземпляр по умолчанию, не удалив расширение IaaS, расширение продолжает искать экземпляр по умолчанию и может генерировать ошибки журнала событий. У этих ошибок два источника: **служба управления учетными данными Microsoft SQL Server** и **агент IaaS Microsoft SQL Server**. Пример одной из ошибок показан ниже.

      При подключении к SQL Server произошла ошибка, связанная с сетью или с определенным экземпляром. Сервер не найден или недоступен.

   Если вы решите удалить экземпляр по умолчанию, также удалите [расширение агента сервера IaaS.](virtual-machines-windows-sql-server-agent-extension.md) 

1. **Могу ли я использовать названный экземпляр сервера S'L с расширением IaaS?**
   
   Да, если указанный экземпляр является единственным экземпляром на сервере S'L, и если исходный экземпляр по умолчанию был [неустановлен должным образом.](virtual-machines-windows-sql-server-agent-extension.md#install-on-a-vm-with-a-single-named-sql-server-instance) Если экземпляра по умолчанию нет и есть несколько названных экземпляров на одном Сервере VM, расширение агента S'L Server IaaS не будет установлено. 

1. **Могу ли я полностью удалить сервер S'L с VM сервера S'L?**

   Да, но с вас будет по-прежнему взиматься плата за ваш VM сервера S'L, как описано в [руководстве по ценообразованию для VMs-серверов S'L Server.](virtual-machines-windows-sql-server-pricing-guidance.md) Если вам больше не нужен SQL Server, можно развернуть новую виртуальную машину и перенести туда данные и приложения. Затем виртуальную машину SQL Server можно удалить.
   
## <a name="updating-and-patching"></a>Установка обновлений и исправлений

1. **Как изменить версию или выпуск SQL Server на виртуальной машине Azure?**

   Клиенты могут изменить версию или выпуск SQL Server с помощью установочного носителя, который содержит требуемую версию или выпуск SQL Server. Когда выпуск будет изменен, измените свойство выпуска виртуальной машины на портале Azure, чтобы для нее отображались точные сведения о выставлении счетов. Для получения дополнительной [информации, см.](virtual-machines-windows-sql-change-edition.md) Для различных версий сервера S'L не существует разницы в выставлении счетов, поэтому после изменения версии сервера S'L, никаких дальнейших действий не требуется.

1. **Где я могу получить настройку носителя, чтобы изменить издание или версию сервера S'L?**

   Клиенты, имеющие [гарантию программного обеспечения,](https://www.microsoft.com/licensing/licensing-programs/software-assurance-default) могут получить свои средства установки из [Центра лицензирования объемов.](https://www.microsoft.com/Licensing/servicecenter/default.aspx) Клиенты, не имеюющие гарантии программного обеспечения, могут использовать средства настройки с изображения Marketplace S'L Server VM, которое имеет желаемое издание.
   
1. **Как обновления и пакеты обновления применяются к виртуальной машине SQL Server?**

   Виртуальные машины позволяют контролировать сервер, в том числе время и способ применения обновлений. Что касается операционной системы, вы можете вручную применять обновления для Windows или включить службу расписаний под названием [Автоматическое исправление](virtual-machines-windows-sql-automated-patching.md). Эта служба устанавливает все обновления, помеченные как важные, включая обновления SQL Server из этой категории. Другие необязательные обновления для SQL Server необходимо устанавливать вручную.

1. **Могу ли я обновить свой экземпляр S'L Server 2008 / 2008 R2 после регистрации в поставщике ресурсов S'L Server VM?**

   Да. Вы можете использовать любые средства установки для обновления версии и издания сервера S'L, а затем вы можете обновить [режим расширения S'L IaaS](virtual-machines-windows-sql-register-with-resource-provider.md#management-modes)) от _агента до_ _полного._ Это даст вам доступ ко всем преимуществам расширения S'L IaaS, таким как управляемость портала, автоматизированное резервное копирование и автоматическое исправление. 

1. **Как я могу получить бесплатные расширенные обновления безопасности для моего конца поддержки S'L Server 2008 и S'L Server 2008 R2 экземпляров?**

   Вы можете получить [бесплатные расширенные обновления безопасности,](virtual-machines-windows-sql-server-2008-eos-extend-support.md) переместив ваш сервер s'L как это в виртуальную машину Azure S'L. Для получения дополнительной [информации](/sql/sql-server/end-of-support/sql-server-end-of-life-overview)см. 
  
   

## <a name="general"></a>Общие сведения

1. **Поддерживаются ли экземпляры кластера сбоя s'L Server (FCI) на MMs Azure?**

   Да. Можно установить экземпляр кластера сотказом, используя либо [премиум-файлы (PFS),](virtual-machines-windows-portal-sql-create-failover-cluster-premium-file-share.md) либо [непосредственное пространство для хранения данных (S2D)](virtual-machines-windows-portal-sql-create-failover-cluster.md) для подсистемы хранения. Премиум-файлы обеспечивают IOPS и пропускную способность, которая будет отвечать потребностям многих рабочих нагрузок. Для рабочих нагрузок с интенсивным io-интенсивным следует использовать места хранения, непосредственно основанные на выкованные премиальные или ультра-диски. Кроме того, можно использовать сторонние решения кластеризации или хранения, как описано в статье [Высокий уровень доступности и аварийное восстановление для SQL Server на виртуальных машинах Azure](virtual-machines-windows-sql-high-availability-dr.md#azure-only-high-availability-solutions).

   > [!IMPORTANT]
   > В настоящее время _полное_ [расширение агента сервера IaaS](virtual-machines-windows-sql-server-agent-extension.md) не поддерживается для S'L Server FCI в Azure. Мы рекомендуем удалить _полное_ расширение от VMs, которые участвуют в FCI, и установить расширение в _легком_ режиме вместо. Это расширение поддерживает такие функции, как автоматическое резервное копирование и патчинг, а также некоторые функции портала для сервера S'L. Эти функции не будут работать для VMs сервера S'L после того, как _полный_ агент не будет установлен.

1. **В чем разница между VMs-мис-сервером и службой баз данных S'L?**

   По существу выполнение SQL Server в виртуальной машине Azure не отличается от выполнения SQL Server в удаленном центре данных. В свою очередь, служба [База данных SQL](../../../sql-database/sql-database-technical-overview.md) предлагается на основе модели "база данных как услуга". При использовании Базы данных SQL у вас нет доступа к компьютерам, на которых размещаются ваши базы данных. Полное сравнение доступно в статье [Вы можете выбрать компонент SQL Server в облаке: база данных SQL Azure (PaaS) или SQL Server на виртуальных машинах Azure (IaaS)](../../../sql-database/sql-database-paas-vs-sql-server-iaas.md).

1. **Как установить SQL Server Data Tools в виртуальной машине Azure?**

    Скачайте инструменты SQL Server Data Tools со страницы [Microsoft SQL Server Data Tools ― Business Intelligence для Visual Studio 2013](https://www.microsoft.com/download/details.aspx?id=42313) и установите их.

1. **Поддерживаются ли распределенные транзакции с MSDTC на вмМИ СЗЛ Сервер?**
   
    Да. Местный DTC поддерживается для S'L Server 2016 SP2 и больше. Тем не менее, приложения должны быть протестированы при использовании всегда на группы доступности, как транзакции в полете во время сбоя не удастся и должны быть повторены. Кластерный DTC доступен начиная с Windows Server 2019.

# <a name="create-a-proximity-placement-group-using-the-portal"></a>Создание группы размещения близости с помощью портала

Чтобы получить ВМ как можно ближе, достигнув минимально возможной задержки, вы должны развернуть их в [группе размещения близости.](co-location.md#proximity-placement-groups)

Группа размещения близости — это логическая группировка, используемая для того, чтобы обеспечить физические расположения ресурсов Azure вычислений близко друг к другу. Группы размещения близости полезны для рабочих нагрузок, где требуется низкая задержка.


## <a name="create-the-proximity-placement-group"></a>Создание группы размещения близости

1. Введите **группу размещения близости** в поиске.
1. Под **службами** в результатах поиска выберите **группы размещения близости.**
1. На странице **групп размещения Близости** **выберите Добавить**.
1. На вкладке **Basics** (Основные) в разделе **Project Details** (Сведения о проекте) убедитесь, что выбрана правильная подписка.
1. В **группе ресурсов** выберите **создать новую** для создания новой группы или выберите существующую группу ресурсов из выпадающих.
1. В **регионе** выберите место, где требуется создание группы размещения несбываний.
1. В **Proximity размещения группы имя** типа имя, а затем выберите Обзор и **создать**.
1. После проверки выберите **Создать** для создания группы размещения близости.

    ![Скриншот создания группы размещения близости](./media/ppg/ppg.png)


## <a name="create-a-vm"></a>Создание виртуальной машины

1. Создавая VM на портале, перейдите на вкладку **Advanced.** 
1. При выборе **группы размещения Proximity** выберите правильную группу размещения. 

    ![Скриншот раздела группы размещения при создании нового VM на портале](./media/ppg/vm-ppg.png)

1. Когда вы закончите делать все другие необходимые выборы, выберите **Обзор и создать**.
1. После проверки выберите **«Создать»** для развертывания VM в группе размещения.




## <a name="next-steps"></a>Дальнейшие действия

Вы также можете использовать [Azure PowerShell](proximity-placement-groups.md) для создания групп размещения непосредственной близости.

# <a name="connectivity-architecture-in-azure-database-for-mysql"></a>Архитектура подключений к базе данных Azure для MySQL
В этой статье объясняется база данных Azure для MySQL архитектура подключений к также, как трафик направляется в базе данных Azure для экземпляра MySQL от клиентов как внутри, так и за пределами Azure.

## <a name="connectivity-architecture"></a>Архитектура подключения
Подключение к базе данных Azure для MySQL установлено через шлюз, который отвечает на входящие подключения к физическому расположению сервера в наших кластерах. Следующая диаграмма иллюстрирует поток трафика.

![Общие сведения об архитектуре подключения](./media/concepts-connectivity-architecture/connectivity-architecture-overview-proxy.png)

Как клиент подключения к базе данных, они получают строку подключения, которая подключается к шлюзу. Этого шлюза есть общедоступный IP-адрес, который прослушивает порт 3306. Внутри базы данных кластера трафик перенаправляется в соответствующую базу данных Azure для MySQL. Таким образом Чтобы подключиться к серверу, например, от корпоративных сетей, бывает необходимо открыть брандмауэр на стороне клиента так, чтобы разрешить исходящий трафик иметь возможность связаться с нашей шлюзы. Ниже можно найти полный список IP-адреса, используемые шлюзы на регион.

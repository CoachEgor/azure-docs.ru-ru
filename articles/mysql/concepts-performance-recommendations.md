# <a name="performance-recommendations-in-azure-database-for-mysql"></a>Рекомендации по производительности в базе данных Azure для MySQL

**Область применения** База данных Azure для MySQL 5.7

> [!NOTE]
> Рекомендации по повышению производительности доступен в предварительной версии. Поддержка рекомендаций по производительности на портале Azure будет развертываться и еще недоступен в вашем регионе.

Рекомендации по повышению производительности функция анализирует свои базы данных и создание настраиваемых предлагаемых для повышения производительности. Для получения рекомендаций, анализа рассматривает различные характеристики базы данных, включая схему. Включить [Query Store](concepts-query-store.md) на сервере, чтобы полностью использовать функцию рекомендации по повышению производительности. Если схема производительности имеет значение OFF, включение Query Store позволяет performance_schema и подмножество instruments схемы производительности требуется для функции. После реализации любой рекомендации по производительности, следует проверить производительность, чтобы оценить влияние этих изменений.

## <a name="permissions"></a>Разрешения

Для выполнения анализа с помощью функции рекомендаций по повышению производительности требуются разрешения **владельца** или **участника**.

## <a name="performance-recommendations"></a>Рекомендации по производительности

Функция [рекомендаций по повышению производительности](concepts-performance-recommendations.md) анализирует рабочие нагрузки на сервере и определяет, какие индексы могут помочь в повышении производительности.

Откройте **рекомендации по повышению производительности** из **Intelligent производительности** части строки меню на странице портала Azure для сервера MySQL.

![Целевая страница рекомендаций по повышению производительности](./media/concepts-performance-recommendations/performance-recommendations-page.png)

Выберите **анализ** и выберите базу данных, который начнет процесс анализа. В зависимости от рабочей нагрузки анализ может занять несколько минут. По окончании анализа на портале появится уведомление. Анализ выполняет глубокое изучение базы данных. Мы рекомендуем выполнять анализ в периоды наименьшей нагрузки.

**Рекомендации** окно будет отображаться список рекомендаций, если один и идентификатор связанного запроса, создавшего эту рекомендацию. Идентификатором запроса, можно использовать [mysql.query_store](concepts-query-store.md#mysqlquery_store) представления для получения дополнительных сведений о запросе.

![Новая страница рекомендаций по производительности](./media/concepts-performance-recommendations/performance-recommendations-result.png)

Рекомендации не применяются автоматически. Чтобы применить рекомендацию, скопируйте текст запроса и запустите его из клиента по выбору. Не забудьте проверить и мониторинга для оценки рекомендаций.

## <a name="recommendation-types"></a>Типы рекомендаций

В настоящее время только *Create Index* поддерживаются рекомендации.

### <a name="create-index-recommendations"></a>Рекомендации по созданию индексов

*Создание индекса* рекомендации предложить новые индексы для ускорения наиболее часто требует много времени или выполнения запросов в рабочей нагрузке. Для этого типа рекомендаций требуется [Query Store](concepts-query-store.md) требуется включить. Query Store собирает сведения о запросе и предоставляет подробные среды выполнения и частоту статистики запросов, анализ для внесения рекомендации.

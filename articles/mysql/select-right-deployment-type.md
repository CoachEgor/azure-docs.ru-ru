# <a name="choose-the-right-mysql-server-option-in-azure"></a>Выбор правильного сервера MySQL в Azure

В Azure рабочие нагрузки сервера MySQL могут работать на виртуальной машине с размещенной инфраструктурой (IaaS) или в качестве размещенной службы (PaaS). PaaS имеет несколько вариантов развертывания, и в каждом варианте развертывания существуют уровни служб. При принятии решения об использовании PaaS или IaaS необходимо определить, требуется ли управлять базой данных, применять исправления и создавать резервные копии, или вы хотите делегировать эти операции в Azure.</br>

В зависимости от ответа на этот вопрос рассмотрите следующие варианты: <br/>

**База данных Azure для MySQL** — это полностью управляемое ядро СУБД MySQL, основанное на стабильной версии Community Edition. Эта реляционная база данных как услуга (DBaaS), размещенная в облаке Azure, относится к категории "платформа как услуга" (PaaS). Используя управляемый экземпляр MySQL в Azure, вы можете использовать встроенные функции и функции, требующие расширенной настройки при использовании сервера MySQL (локально или на виртуальной машине Azure). При использовании MySQL в качестве услуги вы платите по мере использования с вариантами увеличения или уменьшения масштаба для повышения мощности без прерывания. Кроме того, в отличие от автономного сервера MySQL, база данных Azure для MySQL имеет дополнительные функции, такие как встроенная высокая доступность, аналитика и управление. <br/><br/>
**MySQL на виртуальной машине Azure** входит в состав категории "инфраструктура как услуга" (IaaS) и позволяет запускать сервер MySQL в полностью управляемой виртуальной машине в облаке Azure. Все последние версии и выпуски MySQL можно установить на виртуальную машину IaaS. Наиболее значительная разница между базой данных Azure для MySQL заключается в том, что MySQL на виртуальных машинах Azure позволяет управлять ядром СУБД. Тем не менее, этот элемент управления полагается на дополнительную ответственность за управление виртуальными машинами и выполнение многочисленных задач администратора баз данных, таких как обслуживание или исправление сервера СУБД, проектирование восстановления и высокий уровень доступности и т. д.

## <a name="business-motivations-for-choosing-paas-or-iaas"></a>Бизнес-мотивация для выбора PaaS или IaaS

Существует несколько факторов, которые могут повлиять на выбор PaaS или IaaS для размещения баз данных MySQL.

### <a name="cost"></a>Стоимость

Независимо от того, является ли вы стесненных запуска для денежных средств или группы в установленной компании, которая работает с ограниченными ограничениями бюджета, при определении наилучшего решения для размещения баз данных часто используется ограниченное финансирование. В этом разделе описываются основы выставления счетов и лицензирования в Azure в отношении базы данных Azure для MySQL и MySQL на виртуальных машинах Azure.

#### <a name="billing"></a>Выставление счетов

Сейчас база данных Azure для MySQL доступна как услуга на нескольких уровнях с различными ценами на ресурсы, каждый из которых оплачивается каждый час по фиксированной ставке. Последние сведения о поддерживаемых уровнях служб, размерах вычислений и объемах хранения см. в разделе [модель приобретения на основе виртуальное ядро](https://docs.microsoft.com/azure/mysql/concepts-pricing-tiers). Вы можете менять уровни служб и объем вычислительных ресурсов в динамическом режиме в соответствии с различными требованиями приложения к пропускной способности. За исходящий трафик Интернета взимается плата за обычные [скорости передачи данных](https://azure.microsoft.com/pricing/details/data-transfers/).

С помощью базы данных Azure для MySQL программное обеспечение базы данных автоматически настраивается, устанавливается в пакет исправлений и обновляется корпорацией Майкрософт, что сокращает затраты на администрирование. Кроме того, [встроенные функции резервного копирования](https://docs.microsoft.com/azure/mysql/concepts-backup) помогают значительно экономить средства, особенно при наличии большого количества баз данных. С помощью MySQL на виртуальных машинах Azure можно выбрать и запустить любую версию MySQL. Независимо от используемой версии MySQL вы платите за подготовленную виртуальную машину и затраты на конкретный тип лицензии, используемый для MySQL.

База данных Azure для MySQL обеспечивает высокий уровень доступности для любого рода прерываний на уровне узла и по-прежнему поддерживает соглашение об уровне обслуживания 99,99% для службы. Однако для обеспечения высокой доступности базы данных (HA) в виртуальных машинах клиент должен пройти через варианты высокого уровня доступности, доступные в базе данных MySQL, например в случае [репликации MySQL](https://dev.mysql.com/doc/refman/8.0/en/replication.html). Использование поддерживаемого режима высокой доступности не обеспечивает дополнительного соглашения об уровне обслуживания, но позволяет достичь доступности базы данных > 99,99% за счет дополнительных затрат и административных издержек.

Дополнительную информацию см. в следующих ресурсах:
* [Цены на базу данных Azure для MySQL](https://azure.microsoft.com/pricing/details/mysql/)
* [Цены на виртуальные машины](https://azure.microsoft.com/pricing/details/virtual-machines/)
* [Калькулятор стоимости — оцените свои расходы](https://azure.microsoft.com/pricing/calculator/)

### <a name="administration"></a>Управление

Во многих организациях решение о переходе на облачную службу — это практически то же самое, что и разгрузка сложности администрирования. С помощью IaaS и PaaS Корпорация Майкрософт управляет базовой инфраструктурой и автоматически реплицирует все данные для обеспечения аварийного восстановления, настраивает и обновляет программное обеспечение базы данных, управляет балансировкой нагрузки и выполняет прозрачную отработку отказа при наличии сбой сервера.

* **С помощью базы данных Azure для MySQL**можно продолжить Администрирование базы данных, но больше не требуется управлять ядром СУБД, операционной системой или оборудованием. Под вашим управлением остаются базы данных и учетные записи, индексы и оптимизация запросов, аудит и безопасность. Кроме того, Настройка высокого уровня доступности для другого центра обработки данных требует минимальной настройки или администрирования.<br/><br/>
* **С помощью MySQL на виртуальных машинах Azure**вы имеете полный контроль над конфигурацией операционной системы и экземпляра сервера MySQL. С помощью виртуальной машины вы решаете, когда следует обновлять операционную систему и программное обеспечение базы данных, а также когда устанавливать дополнительное программное обеспечение, например антивирусное приложение. Некоторые предлагаемые функции автоматизации позволяют значительно упростить процессы исправления, резервного копирования и обеспечения высокой доступности. Кроме того, вы можете контролировать размер виртуальной машины, количество дисков и их конфигурации хранения. Дополнительные сведения см. в статье размеры виртуальных машин и облачных служб для Azure.

### <a name="time-to-move-to-azure-br"></a>Время на перемещение в Azure <br/>
* **База данных Azure для MySQL** — это подходящее решение для приложений, разработанных в облаке, когда производительность разработки и быстрый доступ к новым решениям очень важны. С помощью программных средств, аналогичных АДМИНИСТРАТОРу, служба идеально подходит для архитекторов и разработчиков облака, поскольку она снижает потребность в управлении базовой операционной системой и базой данных.<br/><br/>
* **MySQL на виртуальных машинах Azure** идеально подходит для существующих или новых приложений, которым требуется база данных MySQL или доступ к функциям в базе данных MySQL в Windows и Linux, и вы хотите избежать времени и расходов на приобретение нового локального оборудования. Этот вариант также подходит для переноса существующих локальных приложений и баз данных в Azure "как есть" — для случаев, когда экземпляр базы данных Azure для MySQL не подходит. Поскольку нет необходимости изменять уровни представления, приложения и данных, вы экономите время и бюджеты при повторном проектировании существующего решения. Вместо этого вы можете сосредоточиться на переносе всех ваших решений в Azure и в устранении некоторых оптимизаций производительности, которые могут потребоваться для платформы Azure.

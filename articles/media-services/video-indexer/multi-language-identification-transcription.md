# <a name="automatically-identify-and-transcribe-multi-language-content-preview"></a>Автоматическая идентификация и транскрибная многоязычная документация (предварительный просмотр)

Video Indexer поддерживает автоматическую идентификацию языка и транскрипцию в многоязычном контенте. Этот процесс включает в себя автоматическое определение разговорного языка в различных сегментах от аудио, отправка каждого сегмента файла мультимедиа для транскрибизации и объединения транскрипции обратно в одну единую транскрипцию. 

## <a name="choosing-multilingual-identification-on-indexing-with-portal"></a>Выбор многоязычной идентификации при индексации с помощью портала

Вы можете выбрать **многоязычное обнаружение** при загрузке и индексации видео. Кроме того, при повторной индексации видео можно выбрать **многоязычное** обнаружение. Следующие шаги описывают, как переиндексировать:

1. Откройте веб-сайт [Индексатора видео](https://vi.microsoft.com/) и выполните вход.
1. Перейдите на страницу **Библиотеки** и нависните над названием видео, которое вы хотите переиндексировать. 
1. В правом нижнем углу нажмите кнопку **Re-index video.** 
1. В диалоге **видео Re-index** выберите **многоязычное обнаружение** из окна для выпадающих **исходных материалов video** source.

    * Когда видео проиндексировано как многоязычное, страница проницательности будет включать эту опцию, и появится дополнительный тип понимания, позволяющий пользователю просматривать, какой сегмент транскрибируется на каком языке "Говорящий язык".
    * Перевод на все языки полностью доступен из многоязычной стенограммы.
    * Все остальные идеи будут отображаться в главном языке обнаружены - это язык, который появился наиболее в аудио.
    * Закрытые субтитры на плеере также доступны на нескольких языках.

## <a name="choosing-multilingual-identification-on-indexing-with-api"></a>Выбор многоязычной идентификации при индексации с помощью API

При индексации или [переиндексации](https://api-portal.videoindexer.ai/docs/services/operations/operations/Re-Index-Video?) видео с помощью API выберите опцию `multi-language detection` в параметре. `sourceLanguage`

### <a name="model-output"></a>Выходные данные модели

Модель получит все языки, обнаруженные в видео, в одном списке

```json
"sourceLanguage": null,
"sourceLanguages": [
    "es-ES",
    "en-US"
],
```

Кроме того, каждый экземпляр в разделе транскрипции будет включать язык, на котором он был транскрибирован

```json
{
  "id": 136,
  "text": "I remember well when my youth Minister took me to hear Doctor King I was a teenager.",
  "confidence": 0.9343,
  "speakerId": 1,
  "language": "en-US",
  "instances": [
    {
       "adjustedStart": "0:21:10.42",
       "adjustedEnd": "0:21:17.48",
       "start": "0:21:10.42",
       "end": "0:21:17.48"
    }
  ]
},
```

## <a name="guidelines-and-limitations"></a>Рекомендации и ограничения

* Набор поддерживаемых языков: английский, французский, немецкий, испанский.
* Поддержка многоязычного контента с тремя поддерживаемыми языками.
* Если аудио содержит языки, не ведомые выше, результат является неожиданным.
* Минимальная длина сегмента для обнаружения для каждого языка - 15 секунд.
* Смещение обнаружения языка составляет в среднем 3 секунды.
* Ожидается, что речь будет непрерывным. Частые переигрывания языков могут повлиять на производительность моделей.
* Выступление носителей языка может повлиять на производительность модели (например, когда носители используют родной язык и переходят на другой язык).
* Модель предназначена для распознавания спонтанной разговорной речи с разумной аудиоакустикой (не голосовыми командами, пением и т.д.).
* Создание и редактирование проекта в настоящее время недоступно для многоязычных видео.
* Пользовательские языковые модели недоступны при использовании многоязыкового обнаружения.
* Добавление ключевых слов не поддерживается.
* При экспорте закрытых файлов заголовков указание языка не отображается.
* API стенограммы обновления не поддерживает файлы нескольких языков.

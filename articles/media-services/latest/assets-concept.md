# <a name="assets-in-azure-media-services-v3"></a>Активы в Azure Media Services v3

В Службах медиа Azure [актив](https://docs.microsoft.com/rest/api/media/assets) является основной концепцией. Это место, где вы введаете мультимедиа (например, через загрузку или живой глоток), выходные медиа (из вывода задания) и публикуете мультимедиа из (для потоковой передачи). 

Актив отображается в контейнере с каплями в [учетной записи хранения Azure,](storage-account-concept.md) а файлы в активе хранятся в виде блоков в этом контейнере. Активы содержат информацию о цифровых файлах, хранящихся в Хранилище Azure (включая видео, аудио, изображения, коллекции эскизов, текстовые треки и закрытые файлы заголовков).

Если в учетной записи используется хранилище общего назначения версии 2 (GPv2), Службы мультимедиа поддерживают уровни больших двоичных объектов. Это хранилище позволяет перемещать файлы в [холодный или архивный уровень хранилища](https://docs.microsoft.com/azure/storage/blobs/storage-blob-storage-tiers). **Хранилище архивов** подходит для архивирования исходных файлов, когда они больше не нужны (например, после их кодирования).

**Архивный уровень хранилища** рекомендуется только для очень больших исходных файлов, которые уже были закодированы, а выходные данные задания кодирования были помещены в выходной контейнер больших двоичных объектов. Капли в выходном контейнере, которые вы хотите связать с активом и использовать для потоковой передачи или анализа содержимого, должны существовать в **горячем** или **прохладном** уровне хранения.

## <a name="naming"></a>Именование 

### <a name="assets"></a>Активы

Имена активов должны быть уникальными. Имена ресурсов media Services v3 (например, «Активы, вакансии, преобразования») подпадают под ограничения на наименование ресурсов менеджера ресурсов Azure. Для получения дополнительной [информации см.](media-services-apis-overview.md#naming-conventions)

### <a name="blobs"></a>BLOB-объекты

Названия файлов/капли в активе должны следовать как [требованиям к названию капли,](https://docs.microsoft.com/rest/api/storageservices/Naming-and-Referencing-Containers--Blobs--and-Metadata) так и [требованиям к названию NTFS.](https://docs.microsoft.com/windows/win32/fileio/naming-a-file) Причина этих требований заключается в том, что файлы могут быть скопированы из хранилища капли на локальный диск NTFS для обработки.

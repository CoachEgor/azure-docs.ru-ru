# <a name="media-services-terminology-and-concepts"></a>Терминология и концепции медиа-услуг

В этой теме приводится краткий обзор терминологии и концепций медиаслужбы Azure Media Services. В статье также содержатся ссылки на статьи с подробным объяснением концепций и функциональности Media Services v3.

Основные концепции, описанные в этих темах, должны быть пересмотрены до начала разработки.

> [!NOTE]
> В настоящее время вы можете использовать [портал Azure,](https://portal.azure.com/) чтобы: управлять медиа-службами v3 [Live Events,](live-events-outputs-concept.md)просматривать (не управлять) [v3 Assets](assets-concept.md)и получать [информацию о доступе к AIS.](access-api-portal.md)
> Для всех других задач управления (например, [Transforms and Jobs](transforms-jobs-concept.md) and [Content protection)](content-protection-overview.md)используйте [REST API,](https://aka.ms/ams-v3-rest-ref) [CLI](https://aka.ms/ams-v3-cli-ref)или один из поддерживаемых [SDK.](media-services-apis-overview.md#sdks)

## <a name="media-services-v3-terminology"></a>Медиа Услуги v3 терминология

|Термин|Описание|
|---|---|
|Событие в прямом эфире|**Live Event** представляет собой конвейер для глотания, транскодирования (по желанию) и упаковки живых потоков видео, аудио и метаданных в реальном времени.<br/><br/>Для клиентов, мигрирующих из AIS Media Services v2, **Live Event** заменяет сущность **канала** в v2. Для получения дополнительной информации [см.](migrate-from-v2-to-v3.md)|
|Потоковая точка/Упаковка/Происхождение|**Потоковая конечная точка** представляет собой динамическую (точно в срок) упаковочную и отправную службу, которая может доставлять содержимое в реальном времени и по требованию непосредственно в приложение игрока-клиента. Он использует один из общих протоколов потокового мультимедиа (HLS или DASH). Кроме того, **Streaming Endpoint** обеспечивает динамическое (точно в срок) шифрование ведущих в отрасли систем управления цифровыми правами (DRM).<br/><br/>В индустрии потоковой передачи мультимедиа эта услуга обычно называется **Packager** или **Origin.**  Другие общие термины в отрасли для этой возможности включают JITP (только в срок-пакет) или JITE (просто в срок шифрования).

## <a name="media-services-v3-concepts"></a>Медиа-услуги v3 концепции

|Основные понятия|Описание|Ссылки|
|---|---|---|
|Активы и загрузка контента|Чтобы начать управление, шифрование, кодирование, анализ и потоковую передачу медиаконтента в Azure, необходимо создать учетную запись Media Services и загрузить цифровые файлы в **ресурсы.**|[Передача в облако и хранение](storage-account-concept.md)<br/><br/>[Концепция активов](assets-concept.md)|
|Кодирование содержимого|После загрузки высококачественных цифровых медиафайлов в ресурсы можно закодировать их в форматы, которые можно воспроизводить на самых разных браузерах и устройствах. <br/><br/>Чтобы закодировать с помощью Media Services v3, необходимо создать **Преобразования** и **вакансии.**|[Преобразования и задания](transforms-jobs-concept.md)<br/><br/>[Кодирование с помощью Служб мультимедиа](encoding-concept.md)|
|Анализ содержимого (Индексатор видео)|Media Services v3 позволяет извлекать информацию из видео- и аудиофайлов с помощью пресетов Media Services v3. Чтобы проанализировать ваш контент с помощью пресетов Media Services v3, необходимо создать **«Преобразования»** и **«Рабочие места».**<br/><br/>Если вы хотите получить более подробную информацию, используйте [Video Indexer](https://docs.microsoft.com/azure/media-services/video-indexer/) напрямую.|[Анализ видео- и аудиофайлов](analyzing-video-audio-files-concept.md)|
|Упаковка и доставка|Как только ваш контент будет закодирован, вы можете воспользоваться **преимуществами динамической упаковки.** В службах мультимедиа **потоковая конечная точка** — это динамическая упаковочная служба, используемая для доставки медиаконтента игрокам-клиентам. Чтобы сделать видео в активе вывода доступными для клиентов для воспроизведения, необходимо создать **потоковое locator,** а затем создать потоковые URL-адреса. <br/><br/>При создании **потокового Locator**, в дополнение к названию актива, необходимо указать **политику потоковой передачи.** **Политики потоковой передачи** позволяют определять потоковые протоколы и параметры шифрования (если таковые имеется) для **потоковых локаторов.** Динамическая упаковка используется независимо от того, транслируете ли вы контент в прямом эфире или по требованию. <br/><br/>Динамические **манифесты** Media Services можно использовать только для потоковой передачи только определенных воспроизведений или подклипов вашего видео.|[Динамическая упаковка](dynamic-packaging-overview.md)<br/><br/>[Потоковые точки](streaming-endpoint-concept.md)<br/><br/>[Указатели потоковой передачи](streaming-locators-concept.md)<br/><br/>[Политики потоковой передачи](streaming-policy-concept.md)<br/><br/>[Динамические манифесты](filters-dynamic-manifest-overview.md)<br/><br/>[Фильтры](filters-concept.md)|
|Защита содержимого|С помощью медиа-сервисов вы можете предоставлять содержимое в реальном времени и по требованию, зашифрованное динамически с помощью Advanced Encryption Standard (AES-128) или/и любой из трех основных систем DRM: Microsoft PlayReady, Google Widevine и Apple FairPlay. Они также обеспечивают службу доставки ключей AES и лицензий DRM (PlayReady, Widevine и FairPlay) авторизованным клиентам. <br/><br/>Если указаны параметры шифрования в потоке, создайте **политику ключа содержимого** и свяжете ее с **потоковым locator.** **Политика ключей содержимого** позволяет настроить способ доставки ключа содержимого конечным клиентам.<br/><br/> Попробуйте повторно использовать политики всякий раз, когда нужны одни и те же параметры.| [Политики ключа содержимого](content-key-policy-concept.md)<br/><br/>[Защита содержимого](content-protection-overview.md)|
|Потоковая передача в реальном времени|Службы мультимедиа дают возможность предоставлять клиентам события прямой трансляции в облаке Azure. **События потоковой трансляции** отвечают за прием и обработку видеопотоков. При создании **Live Event**создается конечная точка ввода, которую можно использовать для отправки живого сигнала с удаленного кодера. После того, как поток, впадающий в **Live Event,** вы можете начать потоковое событие, создав **актив,** **Live Output**и **Потоковое Locator.** **Live Output** будет архивировать поток в **актив** и сделать его доступным для зрителей через **конечную точку потоковой передачи.** В прямом эфире событие может быть установленлибо либо *сквозной* (на месте жить кодер посылает несколько битрат поток) или *жить кодирования* (на месте живой кодер посылает один битрат поток). |[Общие сведения о потоковой трансляции](live-streaming-overview.md)<br/><br/>[События и выходные данные прямой трансляции](live-events-outputs-concept.md)|
|Мониторинг с помощью Event Grid|Чтобы увидеть ход выполнения задания, используйте **Event Grid.** Службы мультимедиа также излучают типы событий в реальном времени. С помощью службы "Сетка событий Azure" приложения могут ожидать передачи данных и реагировать на события, поступающие буквально из всех служб Azure и пользовательских источников. |[Обработка событий Сетки событий](reacting-to-media-services-events.md)<br/><br/>[Схемы](media-services-event-schemas.md)|
|Мониторинг с помощью Azure Monitor|Мониторинг метрик и диагностических журналов, которые помогут вам понять, как ваши приложения работают с Azure Monitor.|[Метрики и журналы диагностики](media-services-metrics-diagnostic-logs.md)<br/><br/>[Схемы для журналов диагностики](media-services-diagnostic-logs-schema.md)|
|Клиенты в Проигрывателе|Вы можете использовать Медиаплеер Azure для воспроизведения медиаконтента, потокового media Services на самых разнообразных браузерах и устройствах. Azure Media Player использует отраслевые стандарты, такие как HTML5, Расширения медиа-источников (MSE) и зашифрованные медиа-расширения (EME), чтобы обеспечить обогащенный адаптивный поток. |[Обзор Проигрывателя мультимедиа Azure](use-azure-media-player.md)|

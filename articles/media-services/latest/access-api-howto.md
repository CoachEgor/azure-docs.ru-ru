# <a name="get-credentials-to-access-media-services-api"></a>Получить учетные данные для доступа к API медиа-услуг  

При использовании аутентификации Azure AD для доступа к API-медиаслужбам Azure у вас есть два варианта проверки подлинности:

- **Проверка подлинности службы** (рекомендуется)

    Аутентификация службы. Этот метод аутентификации обычно используют приложения, которые выполняют службы управляющей программы, службы среднего уровня или запланированные задания: веб-приложения, приложения-функции, приложения логики, интерфейсы API или микрослужбы.
- **Аутентификация пользователей**

    Аутентификация пользователя, который использует приложение для взаимодействия с ресурсами служб мультимедиа. Интерактивное приложение должно сначала запросить у пользователя учетные данные. Примером может послужить приложение консоли управления, которое используется авторизованными пользователями для мониторинга заданий кодирования или потоковой трансляции. 

В этой статье описаны шаги для получения учетных данных для доступа к API медиа-услуг. Выберите один из следующих вкладок.

## <a name="prerequisites"></a>Предварительные требования

- Учетная запись Azure. Если у вас нет учетной записи Azure, начните с получения [бесплатной пробной версии](https://azure.microsoft.com/pricing/free-trial/). 
- Учетная запись служб мультимедиа. Дополнительные сведения см. в статье [Создание учетной записи служб мультимедиа Azure с помощью портала Azure](create-account-howto.md).

## <a name="use-the-azure-portal"></a>Использование портала Azure

### <a name="api-access"></a>Доступ через API 

Страница **доступа API** позволяет выбрать метод проверки подлинности, который вы хотите использовать для подключения к API. Страница также предоставляет значения, необходимые для подключения к API.

1. На [портале Azure](https://portal.azure.com/) выберите свою учетную запись служб мультимедиа.
2. Выберите способ подключения к API служб мультимедиа.
3. В **API Connect to Media Services**выберите версию API API Media Services, к ней вы хотите подключиться (V3 является последней версией сервиса).

### <a name="service-principal-authentication--recommended"></a>Проверка подлинности службы (рекомендуется)

Аутентифицирует службу с помощью приложения Active Directory (Azure AD) и секрета. Это рекомендуется для любых служб среднего уровня, зазвоняющих в API медиасервисов. Примерами являются веб-приложения, функции, логические приложения, AI и микрослужбы. Это рекомендуемый метод проверки подлинности.

#### <a name="manage-your-azure-ad-app-and-secret"></a>Управление приложением Azure AD и секретом

**Управление приложением AAD и секретным** разделом позволяет выбрать или создать новое приложение Azure AD и создать секрет. В целях безопасности секрет не может быть показан после того, как лезвие закрыто. Приложение использует идентификатор приложения и секрет для проверки подлинности для получения действительного токена для медиа-услуг.

Убедитесь, что у вас есть достаточное разрешение на регистрацию приложения у арендатора Azure AD и присвоите приложение роли в подписке Azure. Для получения дополнительной [информации см.](https://docs.microsoft.com/azure/active-directory/develop/howto-create-service-principal-portal#required-permissions)

#### <a name="connect-to-media-services-api"></a>Подключение к API медиа-услуг

**API Connect to Media Services** предоставляет вам значения, которые используются для подключения основного приложения службы. Вы можете получить значения текста или скопировать блоки JSON или XML.

### <a name="user-authentication"></a>Аутентификация пользователей

Эта опция может быть использована для проверки подлинности сотрудника или члена Active Directory Azure, который использует приложение для взаимодействия с ресурсами медиа-служб. Интерактивное приложение должно сначала запросить у пользователя учетные данные. Этот метод проверки подлинности должен использоваться только для приложений управления.

#### <a name="connect-to-media-services-api"></a>Подключение к API медиа-услуг

Копируйте свои учетные данные для подключения приложения пользователя из раздела **API подключения к медиа-службам.** Вы можете получить значения текста или скопировать блоки JSON или XML.

# <a name="manage-azure-media-services-v3-accounts"></a>Управление учетными записями служб мультимедиа Azure v3

Чтобы начать шифрование, кодирование, анализ, потоковую передачу мультимедийного содержимого и управление им в Azure, необходимо создать учетную запись Служб мультимедиа. При создании учетной записи Служб мультимедиа необходимо предоставить имя ресурса учетной записи хранения Azure. Указанная учетная запись хранения присоединена к учетной записи Служб мультимедиа. Учетная запись Служб мультимедиа и все связанные учетные записи хранения должны размещаться в одной подписке Azure. Дополнительные сведения см. в разделе [учетные записи хранения](storage-account-concept.md).

## <a name="moving-a-media-services-account-between-subscriptions"></a>Перемещение между подписками учетную запись служб мультимедиа 

Если вам нужно переместить учетную запись служб мультимедиа в новую подписку, необходимо сначала переместить всю группу ресурсов, содержащую учетную запись служб мультимедиа в новую подписку. Необходимо переместить все подключенные ресурсы: Учетные записи хранения Azure, профили Azure CDN, и т.д. Дополнительные сведения см. в статье [Перемещение ресурсов в новую группу ресурсов или подписку](../../azure-resource-manager/resource-group-move-resources.md). Как и все ресурсы в Azure, перемещение группы ресурсов может занять некоторое время.

> [!NOTE]
> Службы мультимедиа v3 поддерживает многопользовательской модели.

### <a name="considerations"></a>Рекомендации

* Создание резервных копий всех данных в учетной записи перед миграцией в другую подписку.
* Необходимо остановить все конечные точки потоковой передачи и потоковая передача ресурсов. Пользователей не будет иметь возможность доступа к содержимому в течение перемещение группы ресурсов. 

> [!IMPORTANT]
> Не запускайте конечной точки потоковой передачи до успешного завершения перемещения.

### <a name="troubleshoot"></a>Устранение неполадок 

Если учетная запись служб мультимедиа или связанной учетной записи хранилища Azure «отключены» после перемещения группы ресурсов, попробуйте смены ключей учетной записи хранения. Смена ключей учетной записи хранения «разъединенное» состояние учетной записи служб мультимедиа не устраняется, отправьте новый запрос на поддержку из «Поддержка и устранение неполадок» меню в учетной записи служб мультимедиа.

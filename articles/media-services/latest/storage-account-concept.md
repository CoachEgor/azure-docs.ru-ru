# <a name="azure-storage-accounts"></a>Учетные записи хранения Azure

Чтобы начать шифрование, кодирование, анализ, потоковую передачу мультимедийного содержимого и управление им в Azure, необходимо создать учетную запись Служб мультимедиа. При создании учетной записи Служб мультимедиа необходимо предоставить имя ресурса учетной записи хранения Azure. Указанная учетная запись хранения присоединена к учетной записи Служб мультимедиа.

Учетная запись Служб мультимедиа и все связанные учетные записи хранения должны размещаться в одной подписке Azure. Настоятельно рекомендуется использовать учетные записи хранилища в том же месте, что и учетная запись Media Services, чтобы избежать дополнительных затрат на задержку и выемку данных.

Необходимо иметь **основную** учетную запись хранения. У вас может быть любое количество **вторичных** учетных записей хранения, связанных с учетной записью Служб мультимедиа. Службы мультимедиа поддерживают учетные записи **общего назначения версии 2** (GPv2) или **общего назначения версии 1** (GPv1). Учетные записи Blob не допускаются как **Первоначальное общество.**

Мы рекомендуем использовать GPv2, чтобы вы могли воспользоваться преимуществами новейших функций и производительности. Дополнительные сведения об учетных записях хранения см. в статье [Общие сведения об учетной записи хранения Azure](../../storage/common/storage-account-overview.md).

> [!NOTE]
> Только уровень горячего доступа поддерживается для использования в службах мультимедиа Azure, хотя другие уровни доступа могут быть использованы для снижения затрат на хранение содержимого, которое не используется активно.

Существуют различные SK-усы, которые вы можете выбрать для своей учетной записи хранения. Дополнительные сведения см. в статье об [учетных записях хранения](https://docs.microsoft.com/cli/azure/storage/account?view=azure-cli-latest). Если вы хотите поэкспериментировать с учетными записями хранения, используйте `--sku Standard_LRS`. Однако при выборе SKU для `--sku Standard_RAGRS`производства следует учитывать, что обеспечивает географическое воспроизведение для непрерывности бизнеса.

## <a name="assets-in-a-storage-account"></a>Ресурсы в учетной записи хранения

В Media Services v3 AIS хранилища используются для загрузки файлов в активы. Для получения дополнительной информации [см. Активы в Azure Media Services v3](assets-concept.md).

> [!Note]
> Не пытайтесь изменять содержимое контейнеров с каплями, которые были созданы SDK Media Services без использования AA-иносов Media Services.

## <a name="storage-side-encryption"></a>Шифрование на стороне хранилища

Чтобы защитить свои активы в состоянии покоя, активы должны быть зашифрованы шифрованием стороны хранения. В следующей таблице показано, как происходит шифрование на стороне хранилища в службах мультимедиа версии 3.

|Вариант шифрования|Описание|Службы мультимедиа версии 3|
|---|---|---|
|Шифрование медиаслужб| Шифрование AES-256, ключ, управляемый Службой сми. |Не поддерживается. <sup>(1)</sup>|
|[Шифрование службы хранения данных в состоянии покоя](https://docs.microsoft.com/azure/storage/common/storage-service-encryption)|Шифрование на стороне сервера, предлагаемое Azure Storage, ключом, управляемым Azure или клиентом.|Поддерживается.|
|[Шифрование на стороне клиента](https://docs.microsoft.com/azure/storage/common/storage-client-side-encryption)|Шифрование на стороне клиента, предлагаемое хранилищем Azure, ключом, управляемым клиентом в Key Vault.|Не поддерживается.|

<sup>1</sup> В Media Services v3 шифрование хранилища (шифрование AES-256) поддерживается только для обратной совместимости, когда ваши активы были созданы с Media Services v2, что означает, что v3 работает с существующими зашифрованными ресурсами хранения, но не позволяет создавать новые.

## <a name="storage-account-errors"></a>Ошибки учетной записи хранения

Состояние "Отключено" учетной записи Служб мультимедиа указывает, что эта учетная запись больше не имеет доступа к одной или нескольким подключенным учетным записям хранения из-за изменения ключей доступа к хранилищу. Ключи доступа к хранилищу в актуальном состоянии необходимы Службам мультимедиа для выполнения различных задач в учетной записи.

Ниже приведены основные сценарии, которые могут лишить учетную запись Служб мультимедиа доступа к подключенным учетным записям хранения.

|Проблема|Решение|
|---|---|
|Учетная запись Служб мультимедиа или подключенные учетные записи хранения были перенесены в отдельные подписки. |Мигрируйте учетную запись хранилища (ы) или учетную запись Media Services, чтобы все они были в одной подписке. |
|Учетная запись Служб мультимедиа использует подключенную учетную запись хранения в другой подписке, так как это ранняя учетная запись Служб мультимедиа, которая была создана, когда такая возможность поддерживалась. Все ранние учетные записи медиа-служб были преобразованы в современные учетные записи на основе ресурсов Azure Исключительна и будут иметь отключенное состояние. |Перенести учетную запись хранения или учетную запись Media Services, чтобы все они были в одной подписке.|

## <a name="azure-storage-firewall"></a>Брандмауэр хранения azure

Медиаслужбы Azure не поддерживают учетные записи хранения с включенным брандмауэром хранилища Azure или [частными конечными точками.](https://docs.microsoft.com/azure/storage/common/storage-network-security)

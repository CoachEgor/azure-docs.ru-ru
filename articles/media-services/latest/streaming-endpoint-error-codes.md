# <a name="streaming-endpoint-origin-errors"></a>Потоковая передача ошибки конечной точки (источник) 

В этом разделе описаны ошибки, которые могут поступать из служб мультимедиа Azure [службы конечной точки потоковой передачи](streaming-endpoint-concept.md).

## <a name="400-bad-request"></a>400 — недопустимый запрос

Запрос содержит недопустимые сведения и отклонен с эти коды ошибок и связано с одной из следующих причин:

|Код ошибки|Шестнадцатеричное значение |Описание ошибки|
|---|---|---|
|MPE_BAD_URL_SYNTAX |0x80890201|Ошибка синтаксиса или формат для URL-адрес. Примеры запросов для недопустимый тип, что недопустимый фрагмент или недопустимый дорожки. |
|MPE_ENC_ENCRYPTION_NOT_SPECIFIED_IN_URL |0x8088024C|Запрос не имеет шифрования тег в URL-адрес. Запросы CMAF требуется тег шифрования в URL-адрес. Другие протоколы, которые настроены с более чем один тип шифрования также требуется тег шифрования для устранения неоднозначности. |
|MPE_STORAGE_BAD_URL_SYNTAX |0x808900E9|Запрос к хранилищу для выполнения запроса завершилась с ошибкой недопустимый запрос. |

## <a name="403-forbidden"></a>403. Запрещено

Запрос запрещен по одной из следующих причин:

|Код ошибки|Шестнадцатеричное значение |Описание ошибки|
|---|---|---|
|MPE_STORAGE_AUTHENTICATION_FAILED |0x808900EA|Сбой запроса к хранилищу для выполнения запроса с сбоя проверки подлинности. Это может произойти, если были повернут хранилища ключей и службе не удалось синхронизировать ключи хранилища. <br/><br/>Azure обратитесь в службу поддержки, выбрав [Справка и поддержка](https://portal.azure.com/#blade/Microsoft_Azure_Support/HelpAndSupportBlade/newsupportrequest) на портале Azure.|
|MPE_STORAGE_INSUFFICIENT_ACCOUNT_PERMISSIONS |0x808900EB |Ошибка операции хранилища, сбой при доступе из-за недостатка разрешений учетной записи. |
|MPE_STORAGE_ACCOUNT_IS_DISABLED |0x808900EC |Сбой запроса в службу хранилища для выполнения запроса учетную запись хранения будет отключен. |
|MPE_STORAGE_AUTHENTICATION_FAILURE |0x808900F3 |Ошибка операции хранилища, сбой при доступе из-за общих ошибок. |
|MPE_OUTPUT_FORMAT_BLOCKED |0x80890207 |Формат вывода блокируется из-за конфигурации в StreamingPolicy. |
|MPE_ENC_ENCRYPTION_REQUIRED |0x8088021E |Шифрование является необходимым для содержимого, политики доставки является обязательной для формата выходных данных. |
|MPE_ENC_ENCRYPTION_NOT_SET_IN_DELIVERY_POLICY |0x8088024D |Шифрование не задано в настройках политики доставки. |

## <a name="409-conflict"></a>409. Конфликт

Идентификатор, предоставленный для ресурса на `PUT` или `POST` операции был занят существующего ресурса. Для устранения этой проблемы необходимо используйте другой идентификатор для ресурса.

|Код ошибки|Шестнадцатеричное значение |Описание ошибки|
|---|---|---|
|MPE_STORAGE_CONFLICT  |0x808900EE  |Ошибка операции хранилища, ошибка конфликта.  |

Операция указала eTag, отличный от версии, доступной на сервере, то есть ошибки оптимистичного параллелизма. Повторите запрос после считывания последней версии ресурса и обновления eTag в запросе.

|MPE_STORAGE_PRECONDITION_FAILED| 0x808900EF|Ошибка операции хранилища, — сбой.|

## <a name="503-service-unavailable"></a>503 — Служба недоступна

Серверу не удается получить запросы. Эта ошибка может быть вызвана большим количеством запросов к службе. Механизм регулирования служб мультимедиа ограничивает использование ресурсов для приложений, создающих избыточный запрос к службе.

> [!NOTE]
> Просмотрите сообщение и строку кода ошибки, чтобы получить более подробные сведения о причине возникновения ошибки 503. Эта ошибка не всегда указывает на регулирование.
> 

|Код ошибки|Шестнадцатеричное значение |Описание ошибки|
|---|---|---|
|MPE_STORAGE_SERVER_BUSY|0x808900E8|Ошибка операции хранилища, получил ошибка занятости сервера HTTP 503.|

# <a name="manage-azure-media-services-v2-accounts"></a>Управление учетными записями Azure Media Services v2

Чтобы начать шифрование, кодирование, анализ, потоковую передачу мультимедийного содержимого и управление им в Azure, необходимо создать учетную запись Служб мультимедиа. При создании учетной записи Служб мультимедиа необходимо предоставить имя ресурса учетной записи хранения Azure. Указанная учетная запись хранения присоединена к учетной записи Служб мультимедиа. Учетная запись Служб мультимедиа и все связанные учетные записи хранения должны размещаться в одной подписке Azure.  

## <a name="moving-a-media-services-account-between-subscriptions"></a>Перемещение учетной записи медиа-услуг между подписками 

Если вам необходимо перевести учетную запись Media Services на новую подписку, необходимо сначала перевести всю группу ресурсов, содержащую учетную запись Media Services, в новую подписку. Необходимо переместить все прилагаемые ресурсы: учетные записи хранения Azure, профили Azure CDN и т.д. Для получения дополнительной информации [см. Перемещение ресурсов в новую группу ресурсов или подписку.](../../azure-resource-manager/management/move-resource-group-and-subscription.md) Как и в случае с любыми ресурсами в Azure, перемещение группы ресурсов может занять некоторое время.

Media Services v2 не поддерживает модель мультитенантности. Если необходимо перевести учетную запись Media Services в подписку в новом арендаторе, создайте новое приложение Azure Active Directory (Azure AD) в новом арендаторе. Затем переместите учетную запись в подписку в новом арендаторе. После завершения переезда клиента можно начать использовать приложение Azure AD от нового клиента для доступа к учетной записи Media Services с помощью AI v2. 

> [!IMPORTANT]
> Для доступа к API службы безопасности media Services v2 необходимо сбросить информацию [об аутентификации Azure AD.](media-services-portal-get-started-with-aad.md)  
### <a name="considerations"></a>Рекомендации

* Создавайте резервные копирования всех данных в вашей учетной записи перед переходом на другую подписку.
* Вам нужно остановить все потоковые точки и живые потоковые ресурсы. Пользователи не смогут получить доступ к вашему содержимому в течение всего времени перемещения группы ресурсов. 

> [!IMPORTANT]
> Не запускайте конечную точку потоковой передачи до тех пор, пока перемещение не завершится успешно.

### <a name="troubleshoot"></a>Устранение неполадок 

Если учетная запись медиаслужбы или связанная с ней учетная запись Хранилища Azure становятся «отключенными» после перемещения группы ресурсов, попробуйте повернуть ключи учетной записи хранилища. Если в счету хранения нет решения проблемы "отключенного" статуса учетной записи Media Services, подайте новый запрос на поддержку из меню "Поддержка и устранение неполадок" в учетной записи Media Services.

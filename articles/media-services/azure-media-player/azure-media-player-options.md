# <a name="options"></a>Параметры #

## <a name="setting-options"></a>Значения параметра ##

Встраиваемый код Мультиплейплеела Azure — это просто видеотег HTML5, поэтому для многих вариантов можно использовать стандартные атрибуты тегов для настройки параметров.

`<video controls autoplay ...>`

Кроме того, можно использовать атрибут настройки данных для предоставления опций в формате [JSON.](http://json.org/example.html) Это также, как вы бы установить параметры, которые не являются стандартными для видео-тега.

`<video data-setup='{ "controls": true, "autoplay": false }'...>`

Наконец, если вы не используете атрибут настройки данных для запуска установки игрока, вы можете пройти в объект с вариантами игрока в качестве второго аргумента в функции настройки JavaScript.

`amp("vid1", { "controls": true, "autoplay": false });`

> [!NOTE]
> Параметры в конструкторе устанавливаются только при первой инициализации до установки источника.  Если вы хотите изменить параметры на том же инициализированном элементе медиаплеера Azure, необходимо обновить параметры перед изменением источника. Вы можете обновить параметры в `myPlayer.options({/*updated options*/});`JavaScript с помощью . Обратите внимание, что будут затронуты только измененные параметры, все остальные ранее установленные параметры будут сохраняться.

## <a name="individual-options"></a>Индивидуальные варианты ##

> [!NOTE]
>Атрибуты video Tag могут быть только истинными или ложными (boolean), вы просто включаете атрибут (знак без равных), чтобы включить его или исключить его, чтобы выключить его. Например, чтобы включить элементы управления: WRONG `<video controls="true" ...>` RIGHT `<video controls ...>` Самая большая проблема, с которой люди столкнулись, пытается установить эти значения на ложные, используя ложное значение (например, элементы управления,"ложный"), что на самом деле делает противоположное и устанавливает значение, чтобы быть правдой, потому что атрибут все еще включен.

### <a name="controls"></a>controls ###

Параметр управления устанавливает, есть ли у игрока элементы управления, с которыми пользователь может взаимодействовать. Без элементов управления единственный способ начать воспроизведение видео с атрибутом autoplay или через API.

`<video controls ...>` или `{ "controls": true }`

### <a name="autoplay"></a>autoplay ###

Если автоплей соответствует действительности, видео начнет воспроизводиться, как только страница будет загружена (без какого-либо взаимодействия с пользователем).

> [!NOTE]
> Эта опция не поддерживается мобильными устройствами, такими как Windows Phone, Apple iOS и Android. Мобильные устройства блокируют функцию автоматического воспроизведения, чтобы предотвратить чрезмерное использование ежемесячных планов данных потребителя (часто дорогих). Для запуска видео в этом случае требуется пользовательское прикосновение/клик.

`<video autoplay ...>`Или`{ "autoplay": true }`

### <a name="poster"></a>Плакат ###
Атрибут плаката устанавливает изображение, которое отображается до начала воспроизведения видео. Это часто кадр видео или пользовательский экран заголовка. Как только пользователь нажимает воспроизведение изображение исчезнет.

`<video poster="myPoster.jpg" ...>` или `{ "poster": "myPoster.jpg" }`

### <a name="width"></a>width ###

Атрибут ширины устанавливает ширину дисплея видео.

`<video width="640" ...>` или `{ "width": 640 }`

### <a name="height"></a>height ###

Атрибут высоты устанавливает высоту дисплея видео.

`<video height="480" ...>` или `{ "height": 480 }`

### <a name="plugins"></a>plugins ###

Плагины JSON определяет, какие плагины загружаются с этим экземпляром AMP позволяет настроить любые варианты, которые плагин может иметь.

   `<video... data-setup='{plugins: { "contentTitle": {"name": "Azure Medi Services Overview"}}}'...>`

Для получения дополнительной информации о разработке плагинов и использовании, [см.](azure-media-player-writing-plugins.md)

### <a name="other-options"></a>другие варианты ###

Другие параметры могут `<video>` быть установлены `data-setup` на тег, используя параметр, который принимает JSON.
`<video ... data-setup='{"nativeControlsForTouch": false}'>`

#### <a name="nativecontrolsfortouch"></a>роднойКонтрол-Фортуш ####

Это явно установлено на ложное. При установке на ложную связь она позволит отображать кожу медиаплеера Azure на разных платформах.  Кроме того, в отличие от названия, сенсорный будет по-прежнему включен.

### <a name="fluid"></a>Жидкости ###

Установив эту опцию на истинный элемент видео займет полную ширину родительского контейнера и высота будет скорректирована в соответствии с видео со стандартным соотношением сторон 16:9.

`<video ... data-setup='{"fluid": true}'>`

`fluid`опция перекрывает явные `width` настройки и `height` настройки. Эта опция доступна только в `2.0.0` версии Media Player Azure и позже.

### <a name="playbackspeed"></a>playbackSpeed ###

`playbackSpeed`опцион управляет управлением воспроизведением speed и набором настроек скорости воспроизведения, доступных для пользователя. `playbackSpeed`берет объект. Для того, чтобы включить контроль скорости `enabled` воспроизведения на панели управления, свойство объекта должно быть установлено в истину. Пример обеспечения скорости воспроизведения в разметке:

`<video ... data-setup='{"playbackSpeed": {"enabled": true}}'>`


Другие свойства `playbackSpeed` параметра приведены объектом [PlaybackSpeedOptions.](https://docs.microsoft.com/javascript/api/azuremediaplayer/amp.player.playbackspeedoptions)

Пример настройки параметров скорости воспроизведения в JavaScript:

```javascript
    var myPlayer = amp('vid1', {
        fluid: true,
        playbackSpeed: {
            enabled: true,
            initialSpeed: 1.0,
            speedLevels: [
                { name: "x4.0", value: 4.0 },
                { name: "x3.0", value: 3.0 },
                { name: "x2.0", value: 2.0 },
                { name: "x1.75", value: 1.75 },
                { name: "x1.5", value: 1.5 },
                { name: "x1.25", value: 1.25 },
                { name: "normal", value: 1.0 },
                { name: "x0.75", value: 0.75 },
                { name: "x0.5", value: 0.5 },
            ]
        }
    });
```

Эта опция доступна только в версии МедиаПлера Azure 2.0.0 и позже.

### <a name="staledatatimelimitinsec"></a>staleDataTimeLimitInSec ###

Опция `staleDataTimeLimitInSec` — это оптимизация, которая позволяет настроить количество устаревших данных, которые вы хотели бы сохранить в буферах mediaSource. Эта функция отключена по умолчанию.

### <a name="cea708captionssettings"></a>cea708CaptionsНастройки ###

Установка, включенная в истину, позволяет отображать прямые субтитры CEA в ваших живых потоках и живых архивах. Атрибут метки не требуется, если не включенный игрок вернется к метке по умолчанию.

```javascript
     cea708CaptionsSettings: {
                enabled: true,
                srclang: 'en',
                label: 'Live CC'
            }
```

Эта опция доступна только в версии МедиаПлера Azure 2.1.1 и позже.

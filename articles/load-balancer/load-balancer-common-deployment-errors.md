# <a name="troubleshoot-common-azure-deployment-errors-with-azure-load-balancer"></a>Устранение распространенных ошибок развертывания Azure с помощью Azure Load Balancer

В этой статье описаны некоторые распространенные ошибки развертывания Azure Load Balancer и приведены сведения для устранения ошибок. Если вам нужны сведения о коде ошибки и эта информация не указана в этой статье, сообщите нам об этом. В нижней части этой страницы можно оставить отзыв. Отзывы отписываются с проблемами GitHub.

## <a name="error-codes"></a>Коды ошибок

| Код ошибки | Сведения и устранение рисков |
| ------- | ---------- |
|дифферентскулоадбаланцерсандпублиЦипаддрессноталловед| Номера SKU общедоступного IP-адреса и номера SKU Load Balancer должны совпадать. Убедитесь, что Azure Load Balancer и номера SKU общедоступных IP-адресов совпадают. Для рабочих нагрузок рекомендуется использовать стандартный SKU. Дополнительные сведения о [различиях в SKU](./skus.md)  |
|дифферентскулоадбаланцерандпублиЦипаддрессноталловединвмсс | Масштабируемые наборы виртуальных машин по умолчанию имеют базовые подсистемы балансировки нагрузки, если SKU не заданы или развернуты без стандартных общедоступных IP-адресов. Повторно разверните масштабируемый набор виртуальных машин с общедоступными IP-адресами на отдельных экземплярах, чтобы убедиться, Load Balancer (цен. категория "Стандартный") выбрано, или просто выберите стандартную балансировку нагрузки при развертывании масштабируемого набора виртуальных машин из портал Azure. |
|максаваилабилитисетсинлоадбаланцерреачед | Серверный пул Load Balancer может содержать не более 150 групп доступности. Если у вас нет наборов доступности, явно определенных для виртуальных машин в внутреннем пуле, каждая отдельная виртуальная машина переходит в собственную группу доступности. Таким образом, при развертывании отдельных виртуальных машин 150 будет предбы предположить, что у нее будут наборы доступности 150, таким образом, достижение этого ограничения. В качестве обходного пути можно развернуть группу доступности и добавить в нее дополнительные виртуальные машины. |
|рулесофсамелоадбаланцертипеусесамебаккендпортпротоколандипконфиг| Нельзя использовать более одного правила для конкретного типа балансировщика нагрузки (внутренний, общедоступный) с одним и тем же внутренним портом и протоколом, на который ссылается тот же масштабируемый набор виртуальных машин. Обновите правило, чтобы изменить это создание правила дублирования. |
|рулесофсамелоадбаланцертипеусесамебаккендпортпротоколандвмссипконфиг| Нельзя использовать более одного правила для конкретного типа балансировщика нагрузки (внутренний, общедоступный) с одним и тем же внутренним портом и протоколом, на который ссылается тот же масштабируемый набор виртуальных машин. Обновите параметры правила, чтобы изменить это создание правила дублирования. |
|аносеринтерналлоадбаланцерексистс| В серверной части Load Balancer может быть только один Load Balancer типа internal, который является одним и тем же набором виртуальных машин или сетевых интерфейсов. Обновите развертывание, чтобы убедиться, что вы создаете только один Load Balancer одного и того же типа. |
|каннотусеинактивехеалспробе| Проба не используется ни одним правилом, настроенным для работоспособности масштабируемого набора виртуальных машин. Убедитесь, что настроенная проба используется активно. |
|вмскалесетканнотусемултиплелоадбаланцерсофсаметипе| Нельзя использовать несколько подсистем балансировки нагрузки одного типа (внутренний, Открытый). Можно использовать максимум один внутренний и один общедоступный Load Balancer. |
|вмскалесетканнотреференцелоадбаланцервхенларжескалеоркроссаз | Базовый Load Balancer не поддерживается для масштабируемых наборов виртуальных машин с несколькими размещениями или масштабируемого набора виртуальных машин с зоной с перекрестной доступностью. Вместо этого используйте Load Balancer (цен. категория "Стандартный"). |
|ресаурцедеплойментфаилуре| Если балансировщик нагрузки находится в состоянии сбоя, выполните следующие действия, чтобы вернуть его из состояния сбоя:<ol><li>Перейдите к https://resources.azure.comи выполните вход с помощью учетных данных портал Azure.</li><li>Выберите **чтение и запись**.</li><li>В левой части разверните узел **подписки**, а затем разверните подписку с Load Balancer для обновления.</li><li>Разверните узел **ResourceGroups**, а затем разверните группу ресурсов с Load Balancer для обновления.</li><li>Выберите **Microsoft. Network** > **LoadBalancers**, а затем выберите Load Balancer для обновления **LoadBalancer_1**.</li><li>На странице Отображение для **LoadBalancer_1**выберите **получить** > **изменение**.</li><li>Обновление значения **ProvisioningState** с " **не** выполнено **".**</li><li>Выберите **PUT**.</li></ol>|

# <a name="administer-group-policy-in-an-azure-ad-domain-services-managed-domain"></a>Администрирование групповая политика в управляемом домене доменных служб Azure AD

Параметры для объектов пользователей и компьютеров в доменных службах Azure Active Directory (Azure AD DS) часто управляются с помощью групповая политика объектов (GPO). AD DS Azure включает встроенные объекты групповой политики для контейнеров *AADDC Users* и *AADDC Computers* . Эти встроенные объекты групповой политики можно настроить для настройки групповая политика в соответствии с потребностями среды. Члены группы *администраторов контроллера домена Azure AD* имеют групповая политика привилегии администрирования в домене AD DS Azure, а также могут создавать пользовательские объекты групповой политики и подразделения (OU). Дополнительные сведения о том, что такое групповая политика и как он работает, см. в разделе [Общие сведения о групповая политика][group-policy-overview].

В этой статье показано, как установить средства управления групповая политика, а затем изменить встроенные объекты групповой политики и создать пользовательские объекты групповой политики.

## <a name="before-you-begin"></a>Перед началом работы

Для работы с этой статьей необходимы следующие ресурсы и привилегии:

* Активная подписка Azure.
    * Если у вас нет подписки Azure, [Создайте учетную запись](https://azure.microsoft.com/free/?WT.mc_id=A261C142F).
* Azure Active Directory клиент, связанный с подпиской, синхронизированный с локальным каталогом или облачным каталогом.
    * При необходимости [создайте Azure Active Directory клиента][create-azure-ad-tenant] или [свяжите подписку Azure с вашей учетной записью][associate-azure-ad-tenant].
* В клиенте Azure AD включен и настроен управляемый домен доменных служб Azure Active Directory.
    * При необходимости выполните инструкции из руководства по [созданию и настройке Azure Active Directory экземпляра доменных служб][create-azure-ad-ds-instance].
* Виртуальная машина управления Windows Server, присоединенная к управляемому домену AD DS Azure.
    * При необходимости завершите учебник, чтобы [создать виртуальную машину Windows Server и присоединить ее к управляемому домену][create-join-windows-vm].
* Учетная запись пользователя, которая является членом группы *администраторов контроллера домена Azure AD* в клиенте Azure AD.

## <a name="install-group-policy-management-tools"></a>Установка средств управления групповая политика

Для создания и настройки групповая политика объектов (GPO) необходимо установить средства управления групповая политика. Эти средства можно установить в качестве компонента Windows Server. Дополнительные сведения об установке средств администрирования на клиенте Windows см. в разделе Install [средства удаленного администрирования сервера (RSAT)][install-rsat].

1. Войдите на виртуальную машину управления. Инструкции по подключению с помощью портал Azure см. в разделе [Подключение к виртуальной машине Windows Server][connect-windows-server-vm].
1. **Диспетчер сервера** должны открываться по умолчанию при входе в виртуальную машину. В противном случае в меню **Пуск** выберите **Диспетчер сервера**.
1. В области *панель мониторинга* окна **Диспетчер сервера** выберите **Добавить роли и компоненты**.
1. На странице **перед началом** работы *мастера добавления ролей и компонентов*нажмите кнопку **Далее**.
1. Для параметра *тип установки*оставьте установленным флажок установить на основе **ролей или компонентов** и нажмите кнопку **Далее**.
1. На странице **Выбор сервера** выберите текущую виртуальную машину из пула серверов, например *myvm.contoso.com*, а затем нажмите кнопку **Далее**.
1. На странице **Роли сервера** нажмите кнопку **Далее**.
1. На странице **Функции** выберите **Управление групповой политикой**.

    ![Установка "групповая политика Management" на странице "компоненты"](./media/active-directory-domain-services-admin-guide/install-rsat-server-manager-add-roles-gp-management.png)

1. На странице **Подтверждение** нажмите кнопку **установить**. Установка средств управления групповая политика может занять одну или две минуты.
1. После завершения установки компонентов нажмите кнопку **Закрыть** , чтобы выйти из мастера **добавления ролей и компонентов** .

## <a name="open-the-group-policy-management-console-and-edit-an-object"></a>Открытие консоль управления групповыми политиками и изменение объекта

Объекты групповой политики по умолчанию (GPO) существуют для пользователей и компьютеров в управляемом домене Azure AD DS. Функция управления групповая политика, установленная из предыдущего раздела, позволяет просматривать и редактировать существующий объект GPO. В следующем разделе вы создадите пользовательский объект групповой политики.

> [!NOTE]
> Для администрирования групповой политики в управляемом домене Azure AD DS необходимо войти в учетную запись пользователя, которая является членом группы *администраторов контроллера домена AAD* .

1. На начальном экране выберите **Администрирование**. Отобразится список доступных средств управления, включая **управление групповая политика** , установленное в предыдущем разделе.
1. Чтобы открыть консоль управления групповыми политиками (GPMC), выберите **управление групповая политика**.

    ![Откроется консоль управления групповыми политиками, готовая к изменению объектов групповой политики.](./media/active-directory-domain-services-admin-guide/gp-management-console.png)

Существует два встроенных групповая политика объектов (GPO) в домене под управлением AD DS Azure — один для контейнера *Компьютеры AADDC* и один для контейнера *Пользователи AADDC* . Эти объекты групповой политики можно настроить так, чтобы при необходимости настроить групповую политику в управляемом домене Azure AD DS.

1. В консоли **управления групповая политика** разверните узел **лес: contoso.com** . Затем разверните узлы **домены** .

    Существуют два встроенных контейнера для *AADDC компьютеров* и *пользователей AADDC*. К каждому из этих контейнеров применяется объект групповой политики по умолчанию.

    ![Встроенные объекты групповой политики, применяемые к контейнерам "AADDC Computers" и "AADDC Users" по умолчанию](./media/active-directory-domain-services-admin-guide/builtin-gpos.png)

1. Эти встроенные объекты групповой политики можно настроить для настройки конкретных групповых политик в управляемом домене Azure AD DS. Щелкните правой кнопкой мыши один из объектов групповой политики, например *AADDC Computers*, а затем выберите **изменить...** .

    ![Выберите параметр "Изменить" один из встроенных объектов групповой политики](./media/active-directory-domain-services-admin-guide/edit-builtin-gpo.png)

1. Откроется средство редактор "Управление групповыми политиками", позволяющее настроить объект групповой политики, например *политики учетной записи*.

    ![Настройка параметров объекта групповой политики при необходимости](./media/active-directory-domain-services-admin-guide/gp-editor.png)

    По завершении выберите **файл > сохранить** , чтобы сохранить политику. По умолчанию компьютеры обновляют групповая политика каждые 90 минут и применяют внесенные изменения.

## <a name="create-a-custom-group-policy-object"></a>Создание пользовательского групповая политика объекта

Для группирования аналогичных параметров политики часто создаются дополнительные объекты групповой политики вместо применения всех необходимых параметров в отдельном объекте групповой политики по умолчанию. С помощью Azure AD DS можно создать или импортировать собственные объекты групповой политики и связать их с пользовательским подразделением. Если необходимо сначала создать настраиваемое подразделение, см. статью [Создание настраиваемого подразделения в управляемом домене Azure AD DS](create-ou.md).

1. В консоли **управления групповая политика** выберите Пользовательское подразделение (OU), например *MyCustomOU*. Щелкните правой кнопкой мыши подразделение и выберите **создать объект GPO в этом домене и свяжите его...** :

    ![Создание пользовательского объекта групповой политики в консоли управления групповая политика](./media/active-directory-domain-services-admin-guide/gp-create-gpo.png)

1. Укажите имя для нового объекта групповой политики, например *мой пользовательский объект групповой политики*, а затем нажмите кнопку **ОК**. При необходимости можно создать этот пользовательский объект групповой политики на основе существующего объекта групповой политики и набора параметров политики.

    ![Укажите имя для нового пользовательского объекта групповой политики](./media/active-directory-domain-services-admin-guide/gp-specify-gpo-name.png)

1. Пользовательский объект групповой политики создается и связывается с пользовательским подразделением. Чтобы настроить параметры политики, щелкните правой кнопкой мыши настраиваемый объект групповой политики и выберите пункт **изменить...** :

    ![Выберите параметр "Изменить" для настраиваемого объекта групповой политики](./media/active-directory-domain-services-admin-guide/gp-gpo-created.png)

1. Откроется **редактор "Управление групповыми политиками"** , позволяющий настроить объект групповой политики:

    ![Настройка параметров объекта групповой политики при необходимости](./media/active-directory-domain-services-admin-guide/gp-customize-gpo.png)

    По завершении выберите **файл > сохранить** , чтобы сохранить политику. По умолчанию компьютеры обновляют групповая политика каждые 90 минут и применяют внесенные изменения.

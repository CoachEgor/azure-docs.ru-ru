# <a name="administer-group-policy-in-an-azure-ad-domain-services-managed-domain"></a>Администрирование групповой политики в домене домена Azure AD, управляемом

Настройки для пользовательских и компьютерных объектов в службах функционального домена Azure Active Directory Domain Services (Azure AD DS) часто управляются с помощью объектов групповой политики (ГПО). Azure AD DS включает встроенные графические процессоры для *пользователей AADDC* и контейнеры *AADDC Computers.* Вы можете настроить встроенные графические процессоры для настройки групповой политики по мере необходимости для вашей среды. Члены *группы администраторов Azure AD DC* имеют привилегии администрирования групповой политики в домене Azure AD DS, а также могут создавать пользовательские ГП и организационные единицы (OUs). Более подробную информацию о том, что [Group Policy overview][group-policy-overview]такое групповая политика и как она работает, см.

В гибридной среде групповые политики, настроенные в среде AD DS, не синхронизированы с Azure AD DS. Чтобы определить настройки конфигурации для пользователей или компьютеров в Azure AD DS, отспособите один из GP-по умолчанию или создайте пользовательский GPO.

В этой статье показано, как установить инструменты управления политикой группы, а затем отсваивать встроенные ГП и создавать пользовательские ГП.

## <a name="before-you-begin"></a>Перед началом

Для завершения этой статьи необходимы следующие ресурсы и привилегии:

* Активная подписка Azure.
    * Если у вас еще нет подписки Azure, создайте [учетную запись](https://azure.microsoft.com/free/?WT.mc_id=A261C142F).
* Связанный с вашей подпиской клиент Azure Active Directory, синхронизированный с локальным или облачным каталогом.
    * Если потребуется, [создайте клиент Azure Active Directory][create-azure-ad-tenant] или [свяжите подписку Azure со своей учетной записью][associate-azure-ad-tenant].
* Управляемый домен доменных служб Azure Active Directory, включенный и настроенный в клиенте Azure AD.
    * При необходимости заполните учебник для [создания и настройки экземпляра служб доменов Active Directory Службы Azure.][create-azure-ad-ds-instance]
* VM управления Windows Server, который соединен с доменом Azure AD DS.
    * При необходимости заполните учебник для [создания Windows Server VM и присоединитесь к управляемому домену.][create-join-windows-vm]
* Учетная запись пользователя, входящая в группу *администраторов Azure AD DC* в клиенте Azure AD.

> [!NOTE]
> Административные шаблоны групповой политики можно использовать, копируя новые шаблоны на рабочую станцию управления. Копирование файлов *.admx* в `%SYSTEMROOT%\PolicyDefinitions` и копирование локализуаконкретных *файлов .adml,* `%SYSTEMROOT%\PolicyDefinitions\[Language-CountryRegion]`где `Language-CountryRegion` соответствует языку и области файлов *.adml.*
>
> Например, скопируйте английскую, американскую версию файлов `\en-us` *.adml* в папку.
>
> Кроме того, можно централизованно хранить административный шаблон группы политики на контроллерах доменов, которые являются частью домена Azure AD DS. Для получения дополнительной информации см. [Как создать и управлять Центральным магазином для административных шаблонов политики группы в Windows.](https://support.microsoft.com/help/3087759/how-to-create-and-manage-the-central-store-for-group-policy-administra)

## <a name="install-group-policy-management-tools"></a>Установка инструментов управления групповыми политиками

Для создания и настройки группового объекта политики (ГПО) необходимо установить инструменты управления политикой группы. Эти инструменты могут быть установлены в качестве функции в Windows Server. Для получения дополнительной информации о том, как установить административные инструменты на клиенте Windows, [см.][install-rsat]

1. Вопийте в своем менеджменте VM. Для нас можно ознакомиться с информацией о том, как подключиться с помощью портала Azure, [см.][connect-windows-server-vm]
1. При входе в виртуальную машину по умолчанию должен открываться **диспетчер сервера**. Если это не произойдет, откройте меню **Пуск** и выберите **Диспетчер сервера**.
1. На *панели управления* в окне **диспетчера серверов** щелкните **Добавить роли и компоненты**.
1. На странице **Перед началом работы** в *мастере добавления ролей и компонентов* щелкните **Далее**.
1. В разделе *Тип установки* оставьте флажок **Установка ролей или компонентов** и щелкните **Далее**.
1. На странице **выбора сервера** выберите текущий VM из пула серверов, например *myvm.aaddscontoso.com,* затем выберите **Следующий.**
1. На странице **Роли сервера** нажмите кнопку **Далее**.
1. На странице **Функции** выберите **Управление групповой политикой**.

    ![Установка «Управление групповой политикой» со страницы «Особенности»](./media/active-directory-domain-services-admin-guide/install-rsat-server-manager-add-roles-gp-management.png)

1. На странице **Подтверждение** щелкните **Установить**. Установка инструментов управления политикой группы может занять минуту или две.
1. Когда установка компонента завершится, нажмите кнопку **Закрыть**, чтобы выйти из мастера **добавления ролей и компонентов**.

## <a name="open-the-group-policy-management-console-and-edit-an-object"></a>Откройте консоль управления политиками группы и отсваивай объект

Объекты политики группы по умолчанию (ГПО) существуют для пользователей и компьютеров в домене Azure AD DS. С помощью функции управления политикой группы, установленной из предыдущего раздела, давайте просмотрим и отсванем существующее GPO. В следующем разделе вы создаете пользовательский GPO.

> [!NOTE]
> Для администрирования групповой политики в домене Azure AD DS необходимо ввести в учетную запись пользователя, являющиеся членом *группы администраторов AAD DC.*

1. С начального экрана выберите **Административные инструменты.** Отображается список доступных инструментов управления, включая **управление групповой политикой,** установленный в предыдущем разделе.
1. Чтобы открыть консоль управления политикой группы (GPMC), выберите **Управление политикой группы.**

    ![Консоль управления политиками группы открывает готовый к отсечению объектов групповой политики](./media/active-directory-domain-services-admin-guide/gp-management-console.png)

В домене Azure AD DS есть два встроенных групповых объекта политики (ГПО) - один для контейнера *AADDC Computers* и один для контейнера *AADDC Users.* Вы можете настроить эти г-жи для настройки групповой политики по мере необходимости в домене Azure AD DS.

1. На консоли **Управления политиками группы** расширьте **Forest: aaddscontoso.com** узла. Затем расширьте узлы **доменов.**

    Два встроенных контейнера существуют для *AADDC Компьютеры* и *AADDC пользователей.* Каждый из этих контейнеров имеет GPO по умолчанию применяется к ним.

    ![Встроенные ГП, применяемые к контейнерам AADDC Computers и AADDC Users](./media/active-directory-domain-services-admin-guide/builtin-gpos.png)

1. Эти встроенные г-жи можно настроить для настройки определенных групповых политик в домене Azure AD DS. Правый выбор одного из GPOs, таких как *AADDC Компьютеры GPO*, а затем выбрать **edit...**.

    ![Выберите опцию «Отсечение» одного из встроенных графических процессоров](./media/active-directory-domain-services-admin-guide/edit-builtin-gpo.png)

1. Открывается инструмент редактора отдела управления политиками группы, который позволяет настроить GPO, например *политики учетной записи:*

    ![Настройка GPO для настройки настроек по мере необходимости](./media/active-directory-domain-services-admin-guide/gp-editor.png)

    При этом выберите **файл > сохранить,** чтобы сохранить политику. Компьютеры обновлять групповую политику по умолчанию каждые 90 минут и применять внесенные изменения.

## <a name="create-a-custom-group-policy-object"></a>Создание пользовательского объекта политики группы

Чтобы сгруппировать аналогичные параметры политики, вы часто создаете дополнительные ГП вместо того, чтобы применять все необходимые настройки в едином GPO по умолчанию. С помощью Azure AD DS можно создавать или импортировать собственные объекты политики пользовательских групп и связывать их с пользовательским OU. Если вам нужно сначала создать пользовательский OU, см. [создать пользовательский OU в домене Azure AD DS, управляемом DS.](create-ou.md)

1. В консоли **Управления политиками группы** выберите пользовательскую организационную единицу (OU), например *MyCustomOU.* Правый выбор OU и выбрать **Создать GPO в этом домене, и связать его здесь ...**:

    ![Создание пользовательского GPO в консоли управления политиками группы](./media/active-directory-domain-services-admin-guide/gp-create-gpo.png)

1. Укажите имя для нового GPO, *например, My custom GPO,* затем выберите **OK.** Вы можете дополнительно основывать этот пользовательский GPO на существующем GPO и наборе вариантов политики.

    ![Указать имя для нового пользовательского GPO](./media/active-directory-domain-services-admin-guide/gp-specify-gpo-name.png)

1. Пользовательский GPO создается и связан с пользовательским OU. Чтобы теперь настроить параметры политики, правильно выберите пользовательский GPO и выберите **Edit...**:

    ![Выберите опцию 'Edit' ваш пользовательский GPO](./media/active-directory-domain-services-admin-guide/gp-gpo-created.png)

1. **Открывается редактор отдела управления политикой группы,** который позволяет настроить GPO:

    ![Настройка GPO для настройки настроек по мере необходимости](./media/active-directory-domain-services-admin-guide/gp-customize-gpo.png)

    При этом выберите **файл > сохранить,** чтобы сохранить политику. Компьютеры обновлять групповую политику по умолчанию каждые 90 минут и применять внесенные изменения.

# <a name="resolve-mismatched-directory-errors-for-existing-azure-ad-domain-services-managed-domains"></a>Устранение ошибок несоответствия каталога для существующих управляемых доменов доменных служб Azure AD

Если управляемый домен Azure Active Directory Domain Domain Services (Azure AD DS) показывает несовпадание ошибки клиента, вы не можете администрировать управляемый домен до ее решения. Эта ошибка возникает, если базовая виртуальная сеть Azure перемещается в другой каталог Azure AD.

В этой статье объясняется, почему происходит ошибка и как ее устранить.

## <a name="what-causes-this-error"></a>Что вызывает эту ошибку?

Ошибка каталога происходит, когда управляемый домен Azure AD DS и виртуальная сеть принадлежат двум разным арендаторам Azure AD. Например, у вас может быть управляемый dS-домен Azure AD под названием *aaddscontoso.com,* который работает в аготенте Contoso Azure AD. Однако виртуальная сеть Azure для управляемого домена является частью ad-арендатора Fabrikam Azure.

Azure использует элементы управления доступом на основе ролей (RBAC) для ограничения доступа к ресурсам. При вмене DS Azure AD в анзаге нанимает Azure AD, хэши учетных данных синхронизируются с управляемым доменом. Эта операция требует, чтобы вы были админ-амином для каталога Azure AD, а доступ к учетным данным должен контролироваться. Для развертывания ресурсов в виртуальной сети Azure и управления трафиком необходимо иметь административные привилегии в виртуальной сети, в которой развертывается DD Azure AD.

Для того чтобы RBAC работал ажиотаж и обеспечивал доступ ко всем ресурсам, которые использует Azure AD DS, управляемый домен и виртуальная сеть должны принадлежать одному и тому же арендатору Azure AD.

В среде Resource Manager применяются следующие правила:

- Каталог Azure AD может иметь несколько подписок Azure.
- Подписка Azure может иметь несколько ресурсов, таких как виртуальные сети.
- Для каталога Azure AD включен один управляемый домен доменных служб Azure AD.
- Управляемый домен доменных служб Azure AD можно включить в виртуальной сети, принадлежащей любой из подписок Azure в пределах одного и того же клиента Azure AD.

### <a name="valid-configuration"></a>Действительная конфигурация

В следующем примере сценарий развертывания управляемый домен Contoso Azure AD dS включен в аниманте Contoso Azure AD. Управляемый домен развертывается в виртуальной сети, которая принадлежит подписке Azure, принадлежащей арендатору Contoso Azure AD. Как управляемый домен, так и виртуальная сеть принадлежат одному и тому же арендатору Azure AD. Эта конфигурация примера действительна и полностью поддерживается.

![Действительная конфигурация dD-клиента Azure AD с управляемым доменом и виртуальной частью сети того же арендатора Azure AD](./media/getting-started/valid-tenant-config.png)

### <a name="mismatched-tenant-configuration"></a>Конфигурация клиента с несоответствием

В этом примере в развертывании в аниманте Contoso Azure AD dD включен опорный домен Contoso Azure AD. Однако управляемый домен развертывается в виртуальной сети, которая принадлежит подписке Azure, принадлежащей арендатору Fabrikam Azure AD. Управляемый домен и виртуальная сеть принадлежат двум различным арендаторам Azure AD. Эта конфигурация примера является несовпадением арендатора и не поддерживается. Виртуальная сеть должна быть перемещена в тот же арендатор Azure AD, что и управляемый домен.

![Конфигурация клиента с несоответствием](./media/getting-started/mismatched-tenant-config.png)

## <a name="resolve-mismatched-tenant-error"></a>Устранение несовместимой ошибки клиента

Следующие два варианта разрешают ошибку несоответствия каталога:

* [Удалите управляемый доменом Azure AD DS](delete-aadds.md) из существующего каталога Azure AD. Создайте в том же каталоге Azure AD [ищему замену домена Azure AD, который](tutorial-create-instance.md) вы хотите использовать. При готове присоединяйтесь ко всем машинам, ранее присоединимым к удаленному домену, к воссозданному управляемому домену.
* [Переместите подписку Azure,](../cost-management-billing/manage/billing-subscription-transfer.md) содержащую виртуальную сеть, в тот же каталог Azure AD, что и в домен Azure AD DS.

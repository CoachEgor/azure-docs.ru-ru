# <a name="troubleshoot-domain-join-problems-with-an-azure-ad-domain-services-managed-domain"></a>Проблемы соединения доменов troubleshoot с управляемым доменом Azure AD

При попытке присоединиться к виртуальной машине (VM) или подключить приложение к домену AnAzure Active Directory Domain Services (AD DS), вы можете получить ошибку, которую вы не можете сделать. Чтобы устранить проблемы соединения домена, просмотрите, в каком из следующих моментов у вас возникла проблема:

* Если вы не получили запрос на проверку подлинности, VM или приложение не могут подключиться к домену Azure AD DS.
    * Начните устранить [проблемы подключения для домена-соединения.](#connectivity-issues-for-domain-join)
* Если вы получили ошибку во время проверки подлинности, подключение к домену Azure AD DS успешно.
    * Начните устранение [проблем, связанных с учетными данными, во время соединения домена.](#credentials-related-issues-during-domain-join)

## <a name="connectivity-issues-for-domain-join"></a>Проблемы подключения для соединения доменов

Если VM не может найти домен СУ СС Azure AD, обычно возникает проблема с подключением к сети или конфигурацией. Просмотрите следующие шаги по устранению неполадок, чтобы найти и устранить проблему:

1. Убедитесь, что VM подключен к той же или загляной виртуальной сети, включенной для Azure AD DS. Если нет, То VM не может найти и подключиться к домену, чтобы присоединиться.
    * Если VM не подключен к той же виртуальной сети, подтвердите, что виртуальная сеть пиринга или VPN соединение *является активным* или *подключенным,* чтобы позволить трафику течь правильно.
1. Попробуйте пинг домена, используя доменное имя доменного домена `ping aaddscontoso.com`Azure AD DS, например.
    * Если ответ на пинг не удается, попробуйте пинг IP-адреса для домена отображается на странице обзора на `ping 10.0.0.4`портале для Azure AD DS управляемый домен, например.
    * Если вы можете успешно пинг IP-адрес, но не домен, DNS может быть неправильно настроен. Убедитесь, что вы настроили управляемый DNS-сервер Azure AD DS для виртуальной сети.
1. Попробуйте промыть кэш DNS-решатера на `ipconfig /flushdns`виртуальной машине, например.

### <a name="network-security-group-nsg-configuration"></a>Конфигурация группы сетевой безопасности (NSG)

При создании домена Azure AD DS, управляемого DS, также создается группа сетевой безопасности с соответствующими правилами для успешной работы домена. При необходимости декретирования или создания дополнительных правил группы сетевой безопасности можно непреднамеренно блокировать порты, необходимые для DS Azure AD, для предоставления услуг соединения и аутентификации. Эти правила группы сетевой безопасности могут вызвать такие проблемы, как невыполнение синхронизации паролей, невозможность ввода в систему пользователей или проблемы с соединением домена.

Если у вас по-прежнему возникает проблема с подключением, просмотрите следующие этапы устранения неполадок:

1. Проверьте состояние работоспособности домена Azure AD DS на портале Azure. Если у вас есть оповещение для *AADDS001,* правило группы безопасности сети блокирует доступ.
1. Просмотрите [требуемые порты и правила группы сетевой безопасности.][network-ports] Убедитесь, что никакие правила группы сетевой безопасности не применяются к VM или виртуальной сети, которую вы подключаете из блокирования этих сетевых портов.
1. Как только проблемы конфигурации группы сетевой безопасности будут устранены, оповещение *AADDS001* исчезает со страницы работы примерно через 2 часа. Теперь, когда подключение к сети доступно, попробуйте снова присоединиться к VM-домену.

## <a name="credentials-related-issues-during-domain-join"></a>Проблемы, связанные с учетных данных во время соединения домена

Если вы получаете диалоговую коробку, в которой запрашивается учетные данные для присоединения к домену Azure AD DS, VM может подключиться к домену с помощью виртуальной сети Azure. Процесс соединения домена не удается проверить подлинность домена или авторизации для завершения процесса соединения домена с использованием учетных данных.

Чтобы устранить проблемы, связанные с учетных данных, просмотрите следующие шаги устранения неполадок:

1. Для указания учетных данных рекомендуется использовать формат имени участника-пользователя, например `dee@aaddscontoso.onmicrosoft.com`. Убедитесь, что этот UPN настроен правильно в Azure AD.
    * *SAMAccountName* для вашей учетной записи может быть автоматически генерируемым, если в вашем арендаторе есть несколько пользователей с одной и той же приставкой UPN или если префикс UPN слишком длинный. Таким образом, формат *SAMAccountName* для вашей учетной записи может отличаться от того, что вы ожидаете или используете в вашем домене.
1. Попробуйте использовать учетные данные для учетной записи пользователя, которая является частью домена Azure AD DS, чтобы присоединиться к VMs-минам в управляемый домен.
1. Убедитесь, что вы [включили синхронизацию паролей][enable-password-sync] и ждали достаточно долго для первоначальной синхронизации паролем для завершения.

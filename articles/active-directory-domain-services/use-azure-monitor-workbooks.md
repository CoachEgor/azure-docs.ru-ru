# <a name="review-security-audit-events-in-azure-ad-domain-services-using-azure-monitor-workbooks"></a>Просмотр событий аудита безопасности в службах домена Azure AD с помощью рабочих книг Azure Monitor

Чтобы помочь вам понять состояние домена Active Directory Domain Services (Azure AD DS), можно включить события аудита безопасности. Эти события аудита безопасности могут быть рассмотрены с помощью справочников Azure Monitor, которые объединяют текст, запросы аналитики и параметры в богатые интерактивные отчеты. Azure AD DS включает шаблоны рабочих книг для обзора безопасности и активности учетных записей, которые позволяют копаться в событиях аудита и управлять средой.

В этой статье показано, как использовать рабочие книги Azure Monitor для просмотра событий аудита безопасности в Azure AD DS.

## <a name="before-you-begin"></a>Подготовка к работе

Для завершения этой статьи необходимы следующие ресурсы и привилегии:

* Активная подписка Azure.
    * Если у вас нет подписки НаAz, [создайте учетную запись](https://azure.microsoft.com/free/?WT.mc_id=A261C142F).
* Связанный с вашей подпиской клиент Azure Active Directory, синхронизированный с локальным или облачным каталогом.
    * Если потребуется, [создайте клиент Azure Active Directory][create-azure-ad-tenant] или [свяжите подписку Azure со своей учетной записью][associate-azure-ad-tenant].
* Управляемый домен доменных служб Azure Active Directory, включенный и настроенный в клиенте Azure AD.
    * При необходимости заполните учебник для [создания и настройки экземпляра служб доменов Active Directory Службы Azure.][create-azure-ad-ds-instance]
* События аудита безопасности включены для вашего домена Azure Active Directory Domain Services, управляемый доменом, который перетекает данные в рабочее пространство Log Analytics.
    * При необходимости [включите аудит безопасности для служб домена Active Directory.][enable-security-audits]

## <a name="azure-monitor-workbooks-overview"></a>Обзор трудовых книжек Azure Monitor

При включении событий аудита безопасности в DS Azure AD может быть трудно анализировать и выявлять проблемы в управляемом домене. Azure Monitor позволяет агрегировать эти события аудита безопасности и заготавлив запрос данных. С помощью рабочих книг Azure Monitor можно визуализировать эти данные, чтобы быстрее и проще выявлять проблемы.

Шаблоны Workbook являются кураторскими отчетами, которые предназначены для гибкого повторного использования несколькими пользователями и группами. При открытии шаблона рабочей книги загружаются данные из среды Azure Monitor. Вы можете использовать шаблоны без влияния на других пользователей в вашей организации, и можете сохранить свои собственные трудовые книжки на основе шаблона.

Azure AD DS включает в себя следующие два шаблона книжки:

* Отчет о обзоре безопасности
* Отчет о деятельности учетной записи

Для получения дополнительной информации о том, как отсеивать и управлять трудовыми книжками, можно ознакомиться на [обзоре трудовых книжек Azure Monitor.](../azure-monitor/platform/workbooks-overview.md)

## <a name="use-the-security-overview-report-workbook"></a>Используйте рабочую книгу отчета отчета об обзоре безопасности

Чтобы лучше понять использование и определить потенциальные угрозы безопасности, отчет о обзоре безопасности обобщает данные ввески и определяет учетные записи, которые вы можете проверить. Вы можете просматривать события в определенном диапазоне дат и просматривать определенные события, например, плохие попытки пароля или место отключения учетной записи.

Чтобы получить доступ к шаблону рабочей книги для отчета об обзоре безопасности, выполните следующие шаги:

1. Поиск и выбор **служб домена Active Directory На** портале Azure.
1. Выберите управляемый домен, например *aaddscontoso.com*
1. Из меню на левой стороне, выберите **Мониторинг > трудовые книжки**

    ![Выберите опцию меню «Рабочие книги» на портале Azure](./media/use-azure-monitor-workbooks/select-workbooks-in-azure-portal.png)

1. Выберите **отчет об обзоре безопасности**.
1. Из выпадающих меню в верхней части рабочей книги выберите подписку Azure, а затем рабочее пространство Azure Monitor. Выберите **диапазон времени,** например, *за последние 7 дней.*

    ![Выберите опцию меню «Рабочие книги» на портале Azure](./media/use-azure-monitor-workbooks/select-query-filters.png)

    Параметры **представления плитки** и **диаграммы** также могут быть изменены для анализа и визуализации данных по мере пожеланий

1. Чтобы просверлить в определенный тип события, выберите одну из карт **результата регистрации,** таких как *учетная запись заблокирована,* как показано в следующем примере:

    ![Пример данных отчета об обзоре безопасности, визуализированных в трудовых книжках Azure Monitor](./media/use-azure-monitor-workbooks/example-security-overview-report.png)

1. Нижняя часть отчета обзора безопасности ниже диаграммы затем разбивает выбранный тип активности. Можно фильтровать имена пользователей, участвующие на правой стороне, как показано в следующем примере отчета:

    [![](./media/use-azure-monitor-workbooks/account-lockout-details-cropped.png "Details of account lockouts in Azure Monitor Workbooks")](./media/use-azure-monitor-workbooks/account-lockout-details.png#lightbox)

## <a name="use-the-account-activity-report-workbook"></a>Использование рабочей книги отчета о деятельности учетной записи

Чтобы помочь устранению проблем для определенной учетной записи пользователя, отчет о деятельности учетной записи разбивает подробную информацию о журнале событий аудита. Вы можете просмотреть, когда при входе в систему было предоставлено плохое имя пользователя или пароль, и источник попытки входной записи.

Чтобы получить доступ к шаблону рабочей книги для отчета о деятельности учетной записи, выполните следующие шаги:

1. Поиск и выбор **служб домена Active Directory На** портале Azure.
1. Выберите управляемый домен, например *aaddscontoso.com*
1. Из меню на левой стороне, выберите **Мониторинг > трудовые книжки**
1. Выберите **отчет о деятельности учетной записи**.
1. Из выпадающих меню в верхней части рабочей книги выберите подписку Azure, а затем рабочее пространство Azure Monitor. Выберите **диапазон времени,** например, *за последние 30 дней,* затем, как вы хотите, чтобы **представление плитки** представляло данные. Вы можете фильтровать по **имени пользователя учетной записи,** например, *felix*, как показано в следующем примере отчета:

    [![](./media/use-azure-monitor-workbooks/account-activity-report-cropped.png "Account activity report in Azure Monitor Workbooks")](./media/use-azure-monitor-workbooks/account-activity-report.png#lightbox)

    Область ниже диаграммы показывает отдельные события ввески вместе с информацией, такой как результат действия и рабочая станция источника. Эта информация может помочь определить повторяющиеся источники событий, вступивных в систему, которые могут вызвать блокировку учетной записи или указать на потенциальную атаку.

Как и в отчете об обзоре безопасности, можно просверлить различные плитки в верхней части отчета, чтобы визуализировать и проанализировать данные по мере необходимости.

## <a name="save-and-edit-workbooks"></a>Сохранить и отсвазать трудовые книжки

Два шаблона рабочих книг, предоставленные Azure AD DS, являются хорошим местом для начала с собственного анализа данных. Если вам нужно получить более подробную информацию запросов и исследований, вы можете сохранить свои собственные трудовые книжки и отойти от запросов.

1. Чтобы сохранить копию одного из шаблонов рабочей книги, выберите **> Сохранить как > общие отчеты,** а затем предоставить имя и сохранить его.
1. Из собственной копии шаблона выберите **Edit,** чтобы войти в режим отобрачения. Вы можете выбрать синюю кнопку **«Изменить»** рядом с любой частью отчета и изменить ее.

Все диаграммы и таблицы в справочниках Azure Monitor генерируются с помощью запросов Kusto. Для получения дополнительной информации о создании собственных запросов [Kusto queries tutorial][kusto-queries] [см.][azure-monitor-queries]

# <a name="password-and-account-lockout-policies-on-managed-domains"></a>Политики паролей и блокировки учетных записей в управляемых доменах

Для управления безопасностью пользователей в службах домена Azure Active Directory Domain Services (Azure AD DS) можно определить мелкозернистые политики паролей, которые управляют настройками блокировки учетной записи или минимальной длиной и сложностью паролей. Политика тонкозернистого пароля по умолчанию создается и применяется ко всем пользователям в домене Azure AD DS. Для обеспечения детального контроля и удовлетворения конкретных потребностей в бизнесе или соответствии требованиям могут быть созданы и применены дополнительные политики к определенным группам пользователей.

В этой статье показано, как создать и настроить мелкозернистой политики паролей в Azure AD DS с помощью Административного центра Active Directory.

> [!NOTE]
> Политики паролей доступны только для доменов Azure AD DS, созданных с помощью модели развертывания ресурсного менеджера. Для старых управляемых доменов, созданных с помощью Classic, [перейти от классической виртуальной сетевой модели к менеджеру ресурсов.][migrate-from-classic]

## <a name="before-you-begin"></a>Подготовка к работе

Для завершения этой статьи необходимы следующие ресурсы и привилегии:

* Активная подписка Azure.
  * Если у вас нет подписки НаAz, [создайте учетную запись](https://azure.microsoft.com/free/?WT.mc_id=A261C142F).
* Связанный с вашей подпиской клиент Azure Active Directory, синхронизированный с локальным или облачным каталогом.
  * Если потребуется, [создайте клиент Azure Active Directory][create-azure-ad-tenant] или [свяжите подписку Azure со своей учетной записью][associate-azure-ad-tenant].
* Управляемый домен доменных служб Azure Active Directory, включенный и настроенный в клиенте Azure AD.
  * При необходимости заполните учебник для [создания и настройки экземпляра служб доменов Active Directory Службы Azure.][create-azure-ad-ds-instance]
  * Экземпляр Azure AD DS должен быть создан с помощью модели развертывания диспетчера ресурсов. При необходимости [перейти от классической виртуальной сетевой модели к менеджеру ресурсов.][migrate-from-classic]
* VM управления Windows Server, который соединен с доменом Azure AD DS.
  * При необходимости, завершить учебник для [создания управления VM][tutorial-create-management-vm].
* Учетная запись пользователя, входящая в группу *администраторов Azure AD DC* в клиенте Azure AD.

## <a name="default-password-policy-settings"></a>Настройки политики паролей по умолчанию

Политики в отношении готовых паролей (FGPP) позволяют применять определенные ограничения для политики блокировки паролей и учетных записей для разных пользователей в домене. Например, для защиты привилегированных учетных записей можно применить более строгие настройки блокировки учетных записей, чем обычные непривилегированные учетные записи. Можно создать несколько FGP-провайдеров в домене Azure AD DS и указать порядок приоритета их применения к пользователям.

Политики распределяются через групповую ассоциацию в домене Azure AD DS, и любые внесенные изменения применяются при следующем входе пользователя. Изменение политики не открывает учетную запись пользователя, которая уже заблокирована.

Политики паролей ведут себя немного по-другому в зависимости от того, как была создана учетная запись пользователя, к которой они были применены. В Azure AD DS можно создать учетную запись пользователя:

* Учетная запись пользователя может быть синхронизирована с Azure AD. Это включает в себя учетные записи пользователей только для облаков, созданные непосредственно в Azure, и гибридные учетные записи пользователей, синхронизированные с предварительной средой AD DS с помощью Azure AD Connect.
    * Большинство учетных записей пользователей в Azure AD DS создаются в процессе синхронизации с Azure AD.
* Учетная запись пользователя может быть создана вручную в домене Azure AD DS, управляемом DS, и не существует в Azure AD.

Все пользователи, независимо от того, как они созданы, имеют следующие политики блокировки учетных записей, применяемые политикой паролей по умолчанию в Azure AD DS:

* **Продолжительность блокировки счета:** 30
* **Количество разрешенных неудачных попыток логово:** 5
* **Сбросить неудачные попытки входа в строя:** 30 минут
* **Максимальный возраст паролей (срок службы):** 90 дней

С этими параметрами по умолчанию учетные записи пользователей блокируются на 30 минут, если в течение 2 минут используются пять недействительных паролей. Учетные записи автоматически разблокируются через 30 минут.

Блокировки учетных записей происходят только в управляемом домене. Учетные записи пользователей заблокированы только в Azure AD DS, и только из-за неудачных попыток вступления в систему в отношении управляемого домена. Учетные записи пользователей, синхронизированные в Azure AD или в закрытых помещениях, не заблокированы в исходных каталогах, только в Azure AD DS.

Если у вас есть политика паролей Azure AD, которая определяет максимальный возраст паролей, превышающее 90 дней, возраст пароля применяется к политике по умолчанию в Azure AD DS. Можно настроить пользовательскую политику паролей для определения другого возраста максимального возраста паролей в Azure AD DS. Будьте осторожны, если у вас более короткий максимальный возраст паролей, настроенный в политике dS Azure AD, чем в Azure AD или в предварительной среде AD DS. В этом случае пароль пользователя может истечь в Azure AD DS до того, как его подделают для изменения в Azure AD или в среде AD DS.

Для учетных записей пользователей, созданных вручную в домене Azure AD DS, из политики по умолчанию также применяются следующие дополнительные настройки паролей. Эти параметры не применяются к учетным записям пользователей, синхронизированным с Azure AD, так как пользователь не может обновить свой пароль непосредственно в Azure AD DS.

* **Минимальная длина пароля (символы):** 7
* **Требование соответствия паролей требованиям к сложности.**

Вы не можете изменить настройки блокировки учетной записи или паролей в политике пароля по умолчанию. Вместо этого *члены группы администраторов AAD DC* могут создавать пользовательские политики паролей и настраивать их на переопределение (превалировать над) встроенной политики по умолчанию, как показано в следующем разделе.

## <a name="create-a-custom-password-policy"></a>Создание пользовательской политики паролей

При создании и запуске приложений в Azure можно настроить пользовательскую политику паролей. Например, можно создать политику для установки различных параметров политики блокировки учетной записи.

Пользовательские политики паролей применяются к группам в домене Azure AD DS. Эта конфигурация фактически перекрывает политику по умолчанию.

Для создания пользовательской политики паролей используется административные инструменты Active Directory из vM, связанного с доменом. Управляющий центр Active Directory позволяет просматривать, отсвапывать и создавать ресурсы в домене Azure AD DS, включая OUS.

> [!NOTE]
> Чтобы создать пользовательскую политику паролей в домене Azure AD DS, необходимо ввести в систему учетную запись пользователя, являющиеся членом *группы администраторов AAD DC.*

1. С начального экрана выберите **Административные инструменты.** Список доступных инструментов управления показано, что были установлены в учебнике для [создания управления VM][tutorial-create-management-vm].
1. Для создания и управления ОУ выберите **Active Directory Administrative Center** из списка административных инструментов.
1. В левой панели выберите идомет Azure AD DS, например *aaddscontoso.com.*
1. Откройте контейнер **системы,** затем **контейнер настроек Password.**

    Отображается встроенная политика паролей для домена Azure AD DS. Вы не можете изменить эту встроенную политику. Вместо этого создайте пользовательскую политику паролей для отмены политики по умолчанию.

    ![Создание политики паролей в Административном центре Active Directory](./media/password-policy/create-password-policy-adac.png)

1. В панели **задач** справа выберите **новые настройки > password.**
1. В диалоге **настройки параметров создания** введите имя для политики, например *MyCustomFGPP.*
1. При наличии нескольких политик паролей политика с наивысшим приоритетом или приоритетом применяется к пользователю. Чем меньше число, тем выше приоритет. Политика паролей по умолчанию имеет приоритет *200*.

    Установите приоритет для пользовательской политики пароля, чтобы переопределить по умолчанию, например *1.*

1. Изымите другие настройки политики паролей по желанию. Помните следующие ключевые моменты:

    * Такие параметры, как сложность пароля, возраст или время действия только для пользователей, созданных вручную в домене Azure AD DS.
    * Настройки блокировки учетных записей применяются ко всем пользователям, но действуют только в управляемом домене, а не в самом Azure AD.

    ![Создание пользовательской тонкозернистого политики паролей](./media/password-policy/custom-fgpp.png)

1. Uncheck **Защитите от случайного удаления.** Если эта опция выбрана, вы не можете сохранить FGPP.
1. В разделе **«Непосредственно применяется к разделу»** выберите кнопку **«Добавить».** В диалоге **«Избранные пользователи» или «Группы»** выберите кнопку **«Место положений».**

    ![Выберите пользователей и группы для применения политики паролей к](./media/password-policy/fgpp-applies-to.png)

1. Политики паролей могут применяться только к группам. В диалоге **Локаций** расширьте доменное имя, *например, aaddscontoso.com,* затем выберите OU, например, **AADDC Users.** Если у вас есть пользовательский OU, который содержит группу пользователей, которые вы хотите подать заявку, выберите этот OU.

    ![Выберите OU, к которому принадлежит группа](./media/password-policy/fgpp-container.png)

1. Введите имя группы, к которым вы хотите применить политику, а затем выберите **Проверить имена,** чтобы проверить, что группа существует.

    ![Поиск и выбор группы для применения FGPP](./media/password-policy/fgpp-apply-group.png)

1. С названием выбранной вами группы, которая теперь отображается в разделе **Directly Applies To,** выберите **OK,** чтобы сохранить пользовательскую политику паролей.

# <a name="known-issues-network-configuration-alerts-in-azure-active-directory-domain-services"></a>Известные проблемы: оповещения о конфигурации сети в службах домена Active Directory

Чтобы приложения и службы правильно связывались с службами функционального каталога Azure Active Directory Domain (Azure AD DS), для обеспечения трафика должны быть открыты определенные сетевые порты. В Azure вы контролируете поток трафика с помощью групп сетевой безопасности. Состояние работоспособности домена Azure AD DS отображает оповещение, если не установлены необходимые правила группы безопасности сети.

Эта статья поможет вам понять и решить общие оповещения для группы сетевой безопасности проблемы конфигурации.

## <a name="alert-aadds104-network-error"></a>Оповещение AADDS104: ошибка сети

### <a name="alert-message"></a>Сообщение оповещения

*Корпорация Майкрософт не может связаться с контроллерами доменов для этого управляемого домена. Это может произойти, если группа сетевой безопасности (NSG), настроенная на виртуальную сеть, блокирует доступ к управляемому домену. Другой возможной причиной является, если есть пользовательский маршрут, который блокирует входящий трафик из Интернета.*

Недействительные правила группы сетевой безопасности являются наиболее распространенной причиной сетевых ошибок для Azure AD DS. Группа сетевой безопасности виртуальной сети должна обеспечить доступ к определенным портам и протоколам. Если эти порты заблокированы, платформа Azure не может контролировать или обновлять управляемый домен. Также влияет синхронизация между каталогом Azure AD и доменом Azure AD DS. Убедитесь, что порты по умолчанию открыты, чтобы избежать прерывания обслуживания.

## <a name="default-security-rules"></a>Правила безопасности по умолчанию

Следующие правила входящих и исходящих безопасности по умолчанию применяются к группе сетевой безопасности для домена Azure AD DS. Эти правила обеспечивают безопасность DD Azure AD и позволяют платформе Azure отслеживать, управлять и обновлять управляемый домен. Вы также можете иметь дополнительное правило, которое позволяет входящих трафика, если вы [настраиваете безопасный LDAP][configure-ldaps].

## <a name="verify-and-edit-existing-security-rules"></a>Проверка и изменение существующих правил безопасности

Чтобы проверить существующие правила безопасности и убедиться, что порты по умолчанию открыты, выполните следующие шаги:

1. На портале Azure поиск и выбор **групп безопасности сети.**
1. Выберите группу сетевой безопасности, связанную с управляемым доменом, например *AADDS-contoso.com-NSG.*
1. На странице **«Обзор»** отображаются существующие правила безопасности входящих и исходящих.

    Просмотрите входящие и исходящие правила и сравните со списком требуемых правил в предыдущем разделе. При необходимости выберите, а затем удалите все пользовательские правила, которые блокируют необходимый трафик. Если какие-либо из требуемых правил отсутствуют, добавьте правило в следующем разделе.

    После добавления или удаления правил, позволяющих разрешить необходимый трафик, работоспособность домена Azure AD автоматически обновляется в течение двух часов и удаляет оповещение.

### <a name="add-a-security-rule"></a>Добавление правила безопасности

Чтобы добавить недостающие правила безопасности, выполните следующие шаги:

1. На портале Azure поиск и выбор **групп безопасности сети.**
1. Выберите группу сетевой безопасности, связанную с управляемым доменом, например *AADDS-contoso.com-NSG.*
1. В **настройках** в левой панели щелкните *правила безопасности Входящих* или *правила безопасности Исходящих* в зависимости от правила, которое необходимо добавить.
1. Выберите **Добавить,** затем создать необходимое правило на основе порта, протокола, направления и т.д. Когда готовы, выберите **OK**.

Добавление и отображаемый в списке правила безопасности занимает несколько минут.

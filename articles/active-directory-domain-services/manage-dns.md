# <a name="administer-dns-on-an-azure-ad-domain-services-managed-domain"></a>Администрирование DNS в управляемых доменах доменных служб Azure AD
Доменные службы Azure Active Directory включают DNS-сервер, который обеспечивает разрешение доменных имен (DNS) для управляемого домена. Иногда вам может потребоваться настроить DNS в управляемом домене. У вас может возникнуть необходимость создать записи DNS для компьютеров, не присоединенных к домену, настроить виртуальные IP-адреса для балансировщиков нагрузки или внешние DNS-серверы пересылки. По этой причине пользователи, принадлежащие к группе "Администраторы контроллера домена AAD", получают права на администрирование DNS в управляемом домене.

[!INCLUDE [active-directory-ds-prerequisites.md](../../includes/active-directory-ds-prerequisites.md)]

## <a name="before-you-begin"></a>Перед началом работы
Для выполнения задач, перечисленных в этой статье, необходимо следующее:

1. Действующая **подписка Azure**.
2. **Каталог Azure AD** — синхронизированный с локальным каталогом или каталогом только для облака.
3. **Доменные службы Azure AD** должны быть включены для каталога Azure AD. Если это еще не сделано, выполните все задачи, описанные в [руководстве по началу работы](create-instance.md).
4. **Виртуальная машина, присоединенная к домену** , для администрирования домена, управляемого доменными службами Azure AD. Если у вас нет такой виртуальной машины, выполните все задачи, описанные в статье [Присоединение виртуальной машины Windows Server к управляемому домену](active-directory-ds-admin-guide-join-windows-vm.md).
5. Для администрирования DNS управляемого домена потребуются учетные данные **учетной записи пользователя, входящей в группу "Администраторы контроллера домена AAD"** в вашем каталоге.

<br>

## <a name="task-1---create-a-domain-joined-virtual-machine-to-remotely-administer-dns-for-the-managed-domain"></a>Задача 1. Создание присоединенной к домену виртуальной машины для удаленного администрирования DNS управляемого домена
Управляемыми доменами доменных служб Azure AD можно управлять удаленно с помощью привычных средств администрирования Active Directory, таких как центр администрирования Active Directory или AD PowerShell. Аналогичным образом DNS для управляемого домена можно управлять удаленно с помощью средств администрирования DNS-сервера.

Администраторы в каталоге Azure AD не имеют привилегий для подключения к контроллерам доменов в управляемом домене с помощью удаленного рабочего стола. Участники группы "Администраторы контроллера домена AAD" могут администрировать DNS для управляемых доменов удаленно с помощью средств DNS-сервера с компьютера Windows Server или клиентского компьютера, присоединенного к управляемому домену. Средства DNS-сервера входят в дополнительный компонент средств удаленного администрирования сервера (RSAT).

Первой задачей является создание виртуальной машины Windows Server, которая присоединена к управляемому домену. Инструкции см. в статье [Присоединение виртуальной машины Windows Server к управляемому домену](active-directory-ds-admin-guide-join-windows-vm.md).

## <a name="task-2---install-dns-server-tools-on-the-virtual-machine"></a>Задача 2. Установка инструментов DNS-сервера на виртуальную машину
Чтобы установить средства администрирования DNS на присоединенной к домену виртуальной машине, выполните следующие действия. Для получения дополнительных сведений об [установке и использовании средств удаленного администрирования сервера](https://technet.microsoft.com/library/hh831501.aspx) перейдите на сайт TechNet.

1. Перейдите на портал Azure. Выберите **Все ресурсы** на панели слева. Найдите и выберите виртуальную машину, созданную в задаче 1.
2. Нажмите кнопку **Подключиться** на вкладке "Обзор". Будет создан и скачан файл протокола удаленного рабочего стола (RDP-файл).

    ![Подключение к виртуальной машине Windows](./media/active-directory-domain-services-admin-guide/connect-windows-vm.png)
3. Чтобы подключиться к виртуальной машине, откройте скачанный RDP-файл. При появлении запроса нажмите кнопку **Подключиться**. Укажите учетные данные пользователя, принадлежащего к группе "Администраторы контроллера домена AAD". Например, bob@domainservicespreview.onmicrosoft.com. При входе в систему может появиться предупреждение о сертификате. Нажмите "Да" или "Продолжить" для соединения.

4. На начальном экране откройте **диспетчер серверов**. На центральной панели окна диспетчера серверов щелкните **Добавить роли и компоненты** .

    ![Запуск диспетчера серверов на виртуальной машине](./media/active-directory-domain-services-admin-guide/install-rsat-server-manager.png)
5. На странице **Перед началом работы** **мастера добавления ролей и компонентов** нажмите кнопку **Далее**.

    ![Страница "Перед началом работы"](./media/active-directory-domain-services-admin-guide/install-rsat-server-manager-add-roles-begin.png)
6. На странице **Тип установки** оставьте флажок **Установка ролей или компонентов** и нажмите кнопку **Далее**.

    ![Страница "Тип установки"](./media/active-directory-domain-services-admin-guide/install-rsat-server-manager-add-roles-type.png)
7. На странице **Выбор сервера** выберите текущую виртуальную машину из пула серверов и нажмите кнопку **Далее**.

    ![Страница "Выбор сервера"](./media/active-directory-domain-services-admin-guide/install-rsat-server-manager-add-roles-server.png)
8. На странице **Роли сервера** нажмите кнопку **Далее**.
9. На странице **Компоненты** разверните узел **Средства удаленного администрирования сервера**, а затем узел **Средства администрирования ролей**. Из списка средств администрирования ролей выберите компонент **Средства DNS-сервера**.

    ![Страница "Компоненты"](./media/active-directory-domain-services-admin-guide/install-rsat-server-manager-add-roles-dns-tools.png)
10. На странице **Подтверждение** щелкните **Установить**, чтобы установить компонент "Средства DNS-сервера" на виртуальную машину. По завершении установки компонента нажмите кнопку **Закрыть**, чтобы выйти из мастера **добавления ролей и компонентов**.

    ![Страница подтверждения](./media/active-directory-domain-services-admin-guide/install-rsat-server-manager-add-roles-dns-confirmation.png)

## <a name="task-3---launch-the-dns-management-console-to-administer-dns"></a>Задача 3. Запуск консоли управления DNS для администрирования DNS
Теперь можно использовать средства DNS-сервера Windows Server для администрирования DNS в управляемом домене.

> [!NOTE]
> Чтобы администрировать DNS управляемого домена, необходимо быть участником группы "Администраторы контроллера домена AAD".
>
>

1. На начальном экране щелкните **Администрирование**. Должна открыться консоль **DNS** , установленная на виртуальной машине.

    ![Администрирование — консоль DNS](./media/active-directory-domain-services-admin-guide/install-rsat-dns-tools-installed.png)
2. Щелкните элемент **DNS** , чтобы запустить консоль управления DNS.
3. В диалоговом окне **Подключение к DNS-серверу** выберите пункт **Указанный ниже компьютер** и введите DNS-имя управляемого домена (например, contoso100.com).

    ![Консоль DNS — подключение к домену](./media/active-directory-domain-services-admin-guide/dns-console-connect-to-domain.png)
4. Консоль DNS подключится к управляемому домену.

    ![Консоль DNS — администрирование домена](./media/active-directory-domain-services-admin-guide/dns-console-managed-domain.png)
5. Теперь можно использовать консоль DNS для добавления записей DNS для компьютеров в виртуальной сети, в которой вы активировали доменные службы AAD.

> [!WARNING]
> Будьте внимательны при администрировании DNS для управляемого домена с помощью средств администрирования DNS. Следите за тем, чтобы **не удалить и не изменить встроенные записи DNS, которые используются в домене доменными службами**. Встроенные записи DNS включают в себя записи DNS домена, записи сервера имен и другие записи, используемые для обнаружения контроллера домена. Изменение этих записей приведет к нарушению работы доменных служб в виртуальной сети.
>
>

Дополнительные сведения об управлении DNS см. в статье [о средствах DNS на сайте TechNet](https://technet.microsoft.com/library/cc753579.aspx).

# <a name="check-the-health-of-an-azure-active-directory-domain-services-managed-domain"></a>Проверьте работоспособность домена Active Directory Domain Services, управляемого исправления Azure Active Directory

Службы функционального каталога Azure Active Directory Domain (Azure AD DS) выполняют некоторые фоновые задачи, чтобы сохранить управляемый домен здоровым и актуальным. Эти задачи включают в себя прием резервных данных, применение обновлений безопасности и синхронизацию данных из Azure AD. При наличии проблем с доменом Azure AD DS эти задачи могут быть успешно завершены. Для рассмотрения и устранения любых проблем можно проверить состояние работоспособности домена Azure AD DS с помощью портала Azure.

В этой статье показано, как просматривать состояние здоровья DS Azure AD и понимать информацию или оповещения, показанные.

## <a name="view-the-health-status"></a>Просмотр состояния работоспособности

Состояние работоспособности для домена Azure AD DS, управляемого DS, просматривается с помощью портала Azure. Можно увидеть информацию о последнем времени резервного копирования и синхронизации с Azure AD, а также любые оповещения, указывающие на проблему со здоровьем управляемого домена. Чтобы просмотреть состояние работоспособности для домена Azure AD DS, выполните следующие шаги:

1. На портале Azure ищите и выбирайте **службы домена Azure AD.**
1. Выберите идомет Azure AD DS, *например, aaddscontoso.com.*
1. На левой стороне окна ресурса Azure AD DS выберите **«Здоровье».** На следующем скриншоте приведен ы работы с управляемым dS Azure AD и состоянием последнего резервного копирования и синхронизации Azure AD:

    ![Обзор страницы работы на портале Azure, показывающий состояние доменных служб активных каталогов Azure](./media/check-health/health-page.png)

*Последняя оцениваемая* метка времени страницы работы показывает, когда последний раз проверялся домен Azure AD DS. Здоровье домена Azure AD DS оценивается каждый час. Если вы внесете какие-либо изменения в домен Azure AD DS, подождите до следующего цикла оценки, чтобы просмотреть обновленное состояние работы.

Состояние в правом верхнем правом положении указывает на общее состояние домена Azure AD DS. Статус факторы всех существующих предупреждений на вашем домене. В следующей таблице подробно описаны доступные индикаторы состояния:

## <a name="understand-monitors-and-alerts"></a>Понимание мониторов и оповещений

Состояние работоспособности для домена Azure AD DS показывает два типа информации - *мониторы*и *оповещения.* Мониторы показывают время выполнения основных фоновых задач. Оповещения предоставляют информацию или предложения по улучшению стабильности управляемого домена.

### <a name="monitors"></a>Мониторы

Мониторы являются областями домена Azure AD DS, которые регулярно проверяются. При наличии активных оповещений для домена Azure AD DS есть активные оповещения, это может привести к тому, что один из мониторов сообщит о непой. Службы домена Azure AD в настоящее время ведут мониторингы для следующих областей:

* Резервное копирование
* Синхронизация с Azure AD

#### <a name="backup-monitor"></a>Монитор резервного копирования

Монитор резервного копирования проверяет, что автоматизированные регулярные резервные копирования управляемого домена Azure AD DS успешно запускались. В следующей таблице подробно описано доступный статус монитора резервного копирования:

| Значение столбца сведений | Объяснение |
| --- | --- |
| Никогда не резервное копирование | Это состояние является нормальным для новых доменов Azure AD DS. Первая резервная работа должна быть создана через 24 часа после развертывания домена Azure AD DS. Если этот статус сохраняется, [откройте запрос поддержки Azure.][azure-support] |
| "Last backup was taken 1 to 14 days ago" (Последняя резервная копия была создана 1–14 дней назад) | Этот диапазон времени является ожидаемым состоянием для резервного монитора. Автоматизированные регулярные резервные копирования должны происходить в этот период. |
| "Last backup was taken more than 14 days ago" (Последняя резервная копия была создана более 14 дней назад) | Промежуток времени, превышающее две недели, указывает на проблему с автоматизированным регулярным резервным копированием. Активные критические оповещения могут предотвратить резервное копирование домена Azure AD DS. Устраните все активные оповещения для домена Azure AD DS. Если монитор резервного копирования не обновляет статус, чтобы сообщить о недавнем резервном копировании, [откройте запрос поддержки Azure:][azure-support] |

#### <a name="synchronization-with-azure-ad-monitor"></a>Синхронизация с монитором Azure AD

Управляемый домен Azure AD DS регулярно синхронизируется с Active Directory Azure. Количество пользователей и групповых объектов, а также количество изменений, внесенных в каталог Azure AD с момента последней синхронизации, влияют на время синхронизации. Если домен Azure AD DS был синхронизирован более трех дней назад, проверьте и разрешите все активные оповещения. Если монитор синхронизации не обновляет статус для отображения последних синхронизаций после обращения к активным оповещениям, [откройте запрос поддержки Azure.][azure-support]

### <a name="alerts"></a>видны узлы

Оповещения генерируются для проблем в домене Azure AD DS, которые необходимо адресовать службе для правильной работы. Каждое оповещение объясняет проблему и дает URL,который описывает конкретные шаги для решения проблемы. Для получения дополнительной информации о возможных предупреждениях и их разрешениях [см.](troubleshoot-alerts.md)

Оповещения о состоянии здоровья классифицируются по следующим уровням серьезности:

 * **Критические оповещения** – это проблемы, которые серьезно влияют на управляемый домен Azure AD DS. Эти предупреждения должны быть немедленно рассмотрены. Платформа Azure не может контролировать, управлять, исправлять и синхронизировать управляемый домен до тех пор, пока проблемы не будут устранены.
 * Предупреждения о **предупреждении** уведомляют вас о проблемах, которые могут повлиять на управляемые операции домена Azure AD при сохранении проблемы. Эти предупреждения также предлагают рекомендации по обеспечению безопасности управляемого домена.
 * **Информационные оповещения** — это уведомления, которые не оказывают негативного влияния на управляемый домен Azure AD DS. Информационные оповещения дают некоторое представление о том, что происходит в управляемом домене.

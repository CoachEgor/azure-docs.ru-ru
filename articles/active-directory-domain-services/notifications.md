# <a name="notification-settings-in-azure-ad-domain-services"></a>Параметры уведомлений в доменных службах Azure AD

Уведомления для доменных служб Azure AD позволяют немедленно получать информацию об обнаруженных проблемах с работоспособностью управляемого домена.  

Эта функция доступна только для управляемых доменов, которые не включены в классические виртуальные сети.


## <a name="how-to-check-your-azure-ad-domain-services-email-notification-settings"></a>Проверка параметров уведомлений по электронной почте для доменных служб Azure AD

1. Перейдите на [страницу доменных служб Azure AD](https://portal.azure.com/#blade/HubsExtension/Resources/resourceType/Microsoft.AAD%2FdomainServices) на портале Azure.
2. Выберите из таблицы управляемый домен.
3. На панели навигации слева выберите **Параметры уведомлений**.

На этой странице перечислены все получатели электронных сообщений с уведомлениями в отношении доменных служб Azure AD.

## <a name="what-does-an-email-notification-look-like"></a>Как выглядит уведомление по электронной почте?

На следующем изображении представлен пример уведомления, отправляемого по электронной почте:

![Пример уведомления по электронной почте](./media/active-directory-domain-services-alerts/email-alert.png)

В этом сообщении указан управляемый домен, к которому относится оповещение, а также время его обнаружения и ссылка на страницу с данными о работоспособности доменных служб Azure AD на портале Azure.

> [!WARNING]
> Сначала убедитесь, что сообщение отправлено проверенным отправителем Майкрософт, и только после этого переходите по ссылкам в нем. Наши электронные сообщения всегда поступают с адреса azure-noreply@microsoft.com.


## <a name="why-would-i-receive-email-notifications"></a>Зачем мне получать уведомления по электронной почте?

Доменные службы Azure AD отправляют уведомления по электронной почте о важных изменениях в домене.  Эти уведомления создаются только для важных событий, которые могут существенно повлиять на службу и требуют немедленных действий. Каждое уведомление по электронной почте активируется оповещением об управляемом домене. Эти же оповещения отображаются на портале Azure, их можно просмотреть на [странице с данными о работоспособности доменных служб Azure AD](check-health.md).

Доменные службы Azure AD не отправляют в этот список рассылки никакие рекламные сообщения, обновления или коммерческие предложения.

## <a name="when-will-i-receive-email-notifications"></a>Когда я буду получать уведомления по электронной почте?

Уведомление будет отправлено незамедлительно при появлении [нового предупреждения](troubleshoot-alerts.md) касательно вашего управляемого домена. Если предупреждение не будет устранено, каждые четыре дня по электронной почте будет отправляться напоминание о нем.

## <a name="who-should-receive-the-email-notifications"></a>Кто должен получать уведомления по электронной почте?


 Рекомендуем включить в список получателей электронной почты от доменных служб Azure AD тех людей, которые могут администрировать и вносить изменения управляемого домена. Этот список рассылки следует считать группой экстренного реагирования на возникшие проблемы. Если вы хотите добавить более пяти дополнительных электронных адресов, рекомендуем создать отдельный список рассылки и добавить к списку получателей уведомлений.

Вы можете добавить до пяти дополнительных адресов электронной почты в список получателей уведомлений касательно доменных служб Azure AD. Кроме того, вы можете настроить получение по электронной почте уведомлений о доменных службах Azure AD для всех глобальных администраторов из каталога и всех членов группы "Администраторы AAD DC". Доменные службы Azure AD будут отправлять уведомления только на 100 адресов электронной почты, включая список глобальных администраторов и администраторов контроллера домена AAD.


## <a name="how-to-add-an-additional-email-recipient"></a>Добавление дополнительных получателей электронной почты

> [!WARNING]
> Изменения, вносимые в параметры уведомлений, влияют на весь управляемый домен, а не только на текущего пользователя.

1. Откройте [страницу доменных служб Azure AD](https://portal.azure.com/#blade/HubsExtension/Resources/resourceType/Microsoft.AAD%2FdomainServices) на портале Azure.
2. Щелкните нужный управляемый домен.
3. На панели навигации слева выберите **Параметры уведомлений**.
4. Чтобы добавить адрес электронной почты, введите его в таблице дополнительных получателей.
5. Щелкните "Сохранить" в области навигации сверху.

## <a name="frequently-asked-questions"></a>Часто задаваемые вопросы

#### <a name="i-received-an-email-notification-for-an-alert-but-when-i-logged-on-to-the-azure-portal-there-was-no-alert-what-happened"></a>По электронной почте мне пришло уведомление о предупреждении, но у меня не получилось найти это предупреждение на портале Azure. Что произошло?

Если предупреждение было устранено, оно исчезает с портала Azure. Скорее всего, проблему с управляемым доменом устранил кто-то другой из списка получателей уведомлений или она была решена автоматически доменными службами Azure AD.


#### <a name="why-can-i-not-edit-the-notification-settings"></a>Почему я не могу изменить параметры уведомлений?

Если вы не можете получить доступ к странице параметров уведомлений на портале Azure, у вас нет разрешений на внесение изменений касательно доменных служб Azure AD. Обратитесь к глобальному администратору, чтобы он предоставил вам разрешения на изменение ресурсов доменных служб Azure AD или удалил вас из списка получателей.

#### <a name="i-dont-seem-to-be-receiving-email-notifications-even-though-i-provided-my-email-address-why"></a>Я не получаю уведомления по электронной почте, хотя мой адрес электронной почты был указан. Почему?

Проверьте, не попало ли уведомление в папку спама или нежелательной почты, и внесите адрес отправителя (azure-noreply@microsoft.com) в список разрешений.

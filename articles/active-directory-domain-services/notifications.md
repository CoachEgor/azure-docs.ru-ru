# <a name="configure-email-notifications-for-issues-in-azure-active-directory-domain-services"></a>Настройка уведомлений по электронной почте для проблем в Azure Active Directory доменных служб

Работоспособность управляемого домена Azure Active Directory доменных служб (Azure AD DS) отслеживается платформой Azure. На странице состояние работоспособности в портал Azure отображаются оповещения для управляемого домена. Чтобы своевременно получать ответы на вопросы, уведомления по электронной почте можно настроить для создания отчетов об оповещениях о работоспособности, как только они будут обнаружены в управляемом домене Azure AD DS.

В этой статье показано, как настроить получателей уведомлений по электронной почте для управляемого домена AD DS Azure.

## <a name="email-notification-overview"></a>Обзор уведомлений по электронной почте

Чтобы предупредить о проблемах с управляемым доменом AD DS Azure, можно настроить уведомления по электронной почте. В этих уведомлениях по электронной почте указывается управляемый домен Azure AD DS, на котором имеется оповещение, а также время обнаружения и ссылка на страницу работоспособности в портал Azure. Затем можно выполнить указанные рекомендации по устранению неполадок, чтобы устранить неполадки.

В следующем примере уведомления по электронной почте указывают на то, что в управляемом домене AD DS Azure создано критическое предупреждение или оповещение:

![Пример уведомления по электронной почте](./media/active-directory-domain-services-alerts/email-alert.png)

> [!WARNING]
> Прежде чем щелкать ссылки в сообщении, обязательно убедитесь, что электронная почта получена от проверенного отправителя Майкрософт. Уведомления по электронной почте всегда берутся из адреса `azure-noreply@microsoft.com`.

### <a name="why-would-i-receive-email-notifications"></a>Зачем мне получать уведомления по электронной почте?

AD DS Azure отправляет уведомления по электронной почте о важных обновлениях управляемого домена. Эти уведомления предназначены только для срочных проблем, которые влияют на службу и должны быть немедленно решены. Каждое уведомление по электронной почте активируется предупреждением в управляемом домене Azure AD DS. Оповещения также отображаются в портал Azure и могут быть просмотрены на [странице работоспособности AD DS Azure][check-health].

AD DS Azure не отправляет сообщения электронной почты для рекламы, обновлений или продаж.

### <a name="when-will-i-receive-email-notifications"></a>Когда я буду получать уведомления по электронной почте?

Уведомление отправляется немедленно при обнаружении [нового оповещения][troubleshoot-alerts] в управляемом домене Azure AD DS. Если предупреждение не разрешено, дополнительные уведомления отправляются в виде напоминания каждые четыре дня.

### <a name="who-should-receive-the-email-notifications"></a>Кто должен получать уведомления по электронной почте?

Список получателей электронной почты для AD DS Azure должен состоять из пользователей, которые могут администрировать и вносить изменения в управляемый домен. Этот список электронной почты следует рассматривать как "первые отвечающие" на любые оповещения и проблемы.

Для уведомлений по электронной почте можно добавить до пяти дополнительных получателей электронной почты. Если требуется более пяти получателей уведомлений по электронной почте, создайте список рассылки и добавьте его в список уведомлений.

Кроме того, можно выбрать, чтобы все *Глобальные администраторы* каталога Azure AD и все члены группы *администраторов контроллера домена AAD* получали уведомления по электронной почте. Azure AD DS отправляет уведомления только до 100 адресов электронной почты, включая список глобальных администраторов и администраторов AAD контроллеров домена.

## <a name="configure-email-notifications"></a>Настройка уведомлений по электронной почте

Чтобы проверить существующие получатели уведомлений по электронной почте или добавить дополнительных получателей, выполните следующие действия.

1. В портал Azure найдите и выберите **доменные службы Azure AD**.
1. Выберите управляемый домен Azure AD DS, например *contoso.com*.
1. В левой части окна ресурсов Azure AD DS выберите **Параметры уведомлений**. Будут показаны существующие получатели уведомлений по электронной почте.
1. Чтобы добавить получателя электронной почты, введите адрес электронной почты в таблице дополнительные получатели.
1. По завершении нажмите кнопку **сохранить** в верхней части окна навигации.

> [!WARNING]
> При изменении параметров уведомления параметры уведомлений для всего управляемого домена Azure AD DS обновляются, а не просто.

## <a name="frequently-asked-questions"></a>Часто задаваемые вопросы

### <a name="i-received-an-email-notification-for-an-alert-but-when-i-logged-on-to-the-azure-portal-there-was-no-alert-what-happened"></a>По электронной почте мне пришло уведомление о предупреждении, но у меня не получилось найти это предупреждение на портале Azure. Что произошло?

Если предупреждение устранено, предупреждение удаляется из портал Azure. Наиболее вероятная причина заключается в том, что кто-то, кто получит уведомления по электронной почте, разрешил оповещение в управляемом домене Azure AD DS или он был разрешен Авторазрешение платформой Azure.

### <a name="why-can-i-not-edit-the-notification-settings"></a>Почему я не могу изменить параметры уведомлений?

Если вы не можете получить доступ к странице параметров уведомлений в портал Azure, у вас нет разрешений на изменение управляемого домена AD DS Azure. Необходимо обратиться к глобальному администратору, чтобы получить разрешения на изменение ресурса AD DS Azure или удалить из списка получателей.

### <a name="i-dont-seem-to-be-receiving-email-notifications-even-though-i-provided-my-email-address-why"></a>Я не получаю уведомления по электронной почте, хотя мой адрес электронной почты был указан. Почему?

Проверьте нежелательную почту или папку нежелательной почты в электронной почте на наличие уведомления и убедитесь, что отправитель `azure-noreply@microsoft.com`.

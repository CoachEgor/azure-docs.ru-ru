# <a name="configure-email-notifications-for-issues-in-azure-active-directory-domain-services"></a>Настройка уведомлений по электронной почте для проблем в службах домена Active Directory

Работоспособность домена Active Directory Domain Services Azure (Azure AD DS) контролируется платформой Azure. Страница состояния работоспособности на портале Azure отображает все оповещения для управляемого домена. Чтобы убедиться, что на проблемы своевременно реагируют, уведомления по электронной почте могут быть настроены для отчета о оповещениях о работоспособности, как только они обнаруживаются в домене Azure AD DS.

В этой статье показано, как настроить получателей уведомлений по электронной почте для домена Azure AD DS.

## <a name="email-notification-overview"></a>Обзор уведомления по электронной почте

Чтобы предупредить вас о проблемах с доменом Azure AD DS, можно настроить уведомления по электронной почте. В этих уведомлениях по электронной почте указывается управляемый домен Azure AD DS, на котором присутствует оповещение, а также указывается время обнаружения и ссылка на страницу работоспособности на портале Azure. Затем вы можете следовать предоставленным рекомендациям по устранению неполадок для устранения проблем.

В следующем примере уведомление по электронной почте указывает на критическое предупреждение или оповещение, генерируемое на домене Azure AD DS:

![Пример уведомления по электронной почте](./media/active-directory-domain-services-alerts/email-alert.png)

> [!WARNING]
> Всегда убедитесь, что письмо исходит от проверенного отправителя Microsoft, прежде чем нажать на ссылки в сообщении. Уведомления по электронной `azure-noreply@microsoft.com` почте всегда приходят с адреса.

### <a name="why-would-i-receive-email-notifications"></a>Зачем мне получать уведомления по электронной почте?

Azure AD DS отправляет уведомления по электронной почте для важных обновлений об управляемом домене. Эти уведомления касаются только неотложных вопросов, влияющих на обслуживание, и их следует решать немедленно. Каждое уведомление по электронной почте вызывается оповещением на домене Azure AD DS. Оповещения также отображаются на портале Azure и могут быть просмотрены на [странице работоспособности Azure AD DS.][check-health]

Azure AD DS не отправляет электронные письма для рекламы, обновлений или целей продаж.

### <a name="when-will-i-receive-email-notifications"></a>Когда я буду получать уведомления по электронной почте?

Уведомление отправляется сразу же, когда [новое оповещение][troubleshoot-alerts] найдено на домене Azure AD DS. Если оповещение не устранено, дополнительные уведомления по электронной почте отправляются в качестве напоминания каждые четыре дня.

### <a name="who-should-receive-the-email-notifications"></a>Кто должен получать уведомления по электронной почте?

Список получателей электронной почты для Azure AD DS должен состоять из людей, которые могут администрировать и вносить изменения в управляемый домен. Этот список адресов электронной почты следует рассматривать как ваших "первых ответчиков" на любые оповещения и проблемы.

Вы можете добавить до пяти дополнительных получателей писем для уведомлений по электронной почте. Если для уведомлений по электронной почте требуется более пяти получателей, создайте список рассылки и добавьте его в список уведомлений.

Вы также можете получить все *глобальные администраторы* каталога Azure AD, и каждый член *группы администраторов AAD DC* получает уведомления по электронной почте. Azure AD DS отправляет уведомления только на до 100 адресов электронной почты, включая список глобальных администраторов и администраторов AAD DC.

## <a name="configure-email-notifications"></a>Настройка уведомлений по электронной почте

Чтобы просмотреть существующие получатели уведомлений по электронной почте или добавить дополнительные получатели, выполните следующие шаги:

1. На портале Azure ищите и выбирайте **службы домена Azure AD.**
1. Выберите идомет Azure AD DS, *например, aaddscontoso.com.*
1. На левой стороне окна ресурса Azure AD DS выберите **параметры уведомления.** Отображаются существующие получатели уведомлений по электронной почте.
1. Чтобы добавить получателя электронной почты, введите адрес электронной почты в дополнительной таблице получателей.
1. При этом выберите **Сохранить** на верхней рукой навигации.

> [!WARNING]
> При изменении настроек уведомлений обновляются параметры уведомлений для всего домена Azure AD DS, а не только самостоятельно.

## <a name="frequently-asked-questions"></a>Вопросы и ответы

### <a name="i-received-an-email-notification-for-an-alert-but-when-i-logged-on-to-the-azure-portal-there-was-no-alert-what-happened"></a>По электронной почте мне пришло уведомление о предупреждении, но у меня не получилось найти это предупреждение на портале Azure. Что произошло?

Если оповещение удалено, оповещение очищается с портала Azure. Наиболее вероятной причиной является то, что кто-то другой, который получает уведомления по электронной почте, решил оповещение в домене Azure AD DS или был автоматически решен платформой Azure.

### <a name="why-can-i-not-edit-the-notification-settings"></a>Почему я не могу изменить параметры уведомлений?

Если вы не можете получить доступ к странице настроек уведомлений на портале Azure, у вас нет разрешений на отодевательную часть домена Azure AD DS. Необходимо связаться с глобальным администратором, чтобы получить разрешения на отображдение ресурса Azure AD DS или быть удалены из списка получателей.

### <a name="i-dont-seem-to-be-receiving-email-notifications-even-though-i-provided-my-email-address-why"></a>Я не получаю уведомления по электронной почте, хотя мой адрес электронной почты был указан. Почему?

Проверьте свой спам или нежелательной папки в вашей `azure-noreply@microsoft.com`электронной почте для уведомления и убедитесь, что для отправителя .

# <a name="azure-ad-domain-services---troubleshooting-secure-ldap-configuration"></a>Доменные службы Azure AD: устранение неполадок при настройке защищенного протокола LDAP

В этой статье приведены решения распространенных проблем при [настройке защищенного протокола LDAP](configure-ldaps.md) для доменных служб Azure AD.

## <a name="aadds101-secure-ldap-network-security-group-configuration"></a>AADDS101. Настройка группы безопасности сети защищенного протокола LDAP

**Оповещение.**

*Защищенный протокол LDAP через Интернет включен в управляемом домене. Однако доступ к порту 636 не заблокирован с помощью группы безопасности сети. Это может привести к тому, что учетные записи пользователей могут подвергнуться атакам с использованием метода подбора пароля.*

### <a name="secure-ldap-port"></a>Порт защищенного протокола LDAP

Если защищенный протокол LDAP включен, мы рекомендуем создать дополнительные правила, чтобы разрешить входящий доступ по этому протоколу только для определенных IP-адресов. Используя эти правила, вы защитите домен от атак методом подбора, которые могут представлять угрозу безопасности. Порт 636 обеспечивает безопасный доступ к управляемому домену. Чтобы обновить группу безопасности сети для включения доступа по защищенному протоколу LDAP, сделайте следующее:

1. Перейдите на [вкладку группы безопасности сети](https://portal.azure.com/#blade/HubsExtension/Resources/resourceType/Microsoft.Network%2FNetworkSecurityGroups) на портале Azure.
2. Из таблицы выберите группу NSG, связанную с доменом.
3. Щелкните **Правила безопасности для входящего трафика**.
4. Создайте правило порта 636.
   1. Щелкните **Добавить** в верхней панели навигации.
   2. Выберите **IP-адреса** для источника.
   3. Укажите диапазоны порта источника для этого правила.
   4. Введите "636" для диапазонов конечного порта.
   5. Выберите протокол **TCP**.
   6. Введите соответствующее имя правила, описание и приоритет. Приоритет для этого правила должен быть выше приоритета "Запретить все" (при его наличии).
   7. Последовательно выберите **ОК**.
5. Убедитесь, что вы создали правило.
6. Проверьте работоспособность домена через два часа, чтобы убедиться, что шаги выполнены правильно.

> [!TIP]
> Для беспроблемной работы доменных служб Azure AD нужен не только порт 636. Чтобы узнать больше, ознакомьтесь со статьями [Рекомендации по сетям для доменных служб Azure AD](network-considerations.md) и [Узнайте, как устранить неполадки при настройке групп безопасности сети в доменных службах Azure AD](alert-nsg.md).
>

## <a name="aadds502-secure-ldap-certificate-expiring"></a>AADDS502. Срок действия сертификата защищенного протокола LDAP истекает

**Оповещение.**

*Срок действия защищенного сертификата LDAP для управляемого домена истекает [дата]].*

**Способы устранения:**

Создайте сертификат защищенного протокола LDAP, выполнив инструкции в статье о [настройке защищенного протокола LDAP](configure-ldaps.md).

# <a name="known-issues-secure-ldap-alerts-in-azure-active-directory-domain-services"></a>Известные проблемы: Безопасные оповещения LDAP в службах домена Active Directory

Приложения и службы, которые используют легкий протокол доступа к каталогам (LDAP) для связи с службами домена Azure Active Directory Domain Services (Azure AD DS), могут быть [настроены на безопасное LDAP.](tutorial-configure-ldaps.md) Соответствующий сертификат и необходимые сетевые порты должны быть открыты для правильной работы LDAP.

Эта статья поможет вам понять и разрешить общие оповещения с защищенным доступом К LDAP в Azure AD DS.

## <a name="aadds101-secure-ldap-network-configuration"></a>AADDS101: Безопасная конфигурация сети LDAP

### <a name="alert-message"></a>Сообщение оповещения

*Безопасная LDAP через Интернет включена для управляемого домена. Однако доступ к порту 636 не заблокирован с помощью группы сетевой безопасности. Это может подвергнуть учетные записи пользователей управляемого домена паролям атак грубой силы.*

### <a name="resolution"></a>Решение

При входной безопасной LDAP рекомендуется создавать дополнительные правила, ограничивающие входящий доступ LDAPS к определенным IP-адресам. Эти правила защищают управляемый домен Azure AD DS от атак грубой силы. Чтобы обновить группу сетевой безопасности, чтобы ограничить доступ к tCP port 636 для безопасного LDAP, выполните следующие шаги:

1. На портале Azure поиск и выбор **групп безопасности сети.**
1. Выберите группу сетевой безопасности, связанную с управляемым доменом, например *AADDS-contoso.com-NSG,* затем выберите **правила безопасности входящего**
1. **Добавьте** правило для порта TCP 636. При необходимости выберите **Advanced** в окне для создания правила.
1. Для **источника**выберите *IP-адреса* из выпадающего меню. Введите исходные IP-адреса, которые вы хотите предоставить доступ для безопасного трафика LDAP.
1. Выберите *Любой* в качестве **назначения**, а затем введите *636* для **назначения диапазонов порта.**
1. Установите **Протокол** как *TCP* и **действия,** чтобы *разрешить*.
1. Укажите приоритет для правила, а затем введите имя, такое как *RestrictLDAPS.*
1. При готовности **выберите Добавить** для создания правила.

Здоровье домена Azure AD DS автоматически обновляется в течение двух часов и удаляет оповещение.

> [!TIP]
> Порт TCP 636 – не единственное правило, необходимое для бесперебойной работанадия Azure AD DS. Чтобы узнать больше, смотрите [группы безопасности Azure AD Network и необходимые порты.](network-considerations.md#network-security-groups-and-required-ports)

## <a name="aadds502-secure-ldap-certificate-expiring"></a>AADDS502. Срок действия сертификата защищенного протокола LDAP истекает

### <a name="alert-message"></a>Сообщение оповещения

*Срок действия защищенного сертификата LDAP для управляемого домена истекает [дата]].*

### <a name="resolution"></a>Решение

Создайте заменяющий защищенный сертификат LDAP, выдвивая следующие шаги по [созданию сертификата для безопасного LDAP.](tutorial-configure-ldaps.md#create-a-certificate-for-secure-ldap) Примените сертификат замены к Azure AD DS и раздайте сертификат любым клиентам, которые подключаются с помощью защищенного LDAP.

# <a name="understand-the-health-states-and-resolve-suspended-domains-in-azure-active-directory-domain-services"></a>Сведения о состояниях работоспособности и разрешении приостановленных доменов в доменных службах Azure Active Directory

Когда доменные службы Azure Active Directory (Azure AD DS) в течение длительного периода времени не обслуживают управляемый домен, его работа приостанавливается. Если управляемый домен остается в приостановленном состоянии, он автоматически удаляется. Чтобы предотвратить работоспособность управляемого домена Azure AD DS и избежать приостановки, разрешите все оповещения как можно быстрее.

В этой статье объясняется, почему управляемые домены приостановлены и как восстановить приостановленный домен.

## <a name="overview-of-managed-domain-states"></a>Общие сведения о состояниях управляемых доменов

С помощью жизненного цикла управляемого домена AD DS Azure существуют различные состояния, указывающие на его работоспособность. Если управляемый домен сообщает о неполадке, быстро устраните основную причину, чтобы предотвратить снижение состояния.

![Ход выполнения состояний, которые управляемый домен AD DS Azure осуществляет приостановку](media/active-directory-domain-services-suspension/suspension-timeline.PNG)

Управляемый домен AD DS Azure может находиться в одном из следующих состояний:

* [Выполнение](#running-state)
* [Требует внимания](#needs-attention-state)
* [Режим](#suspended-state)
* [Удаленной](#deleted-state)

## <a name="running-state"></a>Состояние выполнения

Управляемый домен AD DS Azure, который правильно настроен и работает без проблем, находится в состоянии *выполняется* . Это требуемое состояние для управляемого домена.

### <a name="what-to-expect"></a>Чего следует ожидать

* Платформа Azure может регулярно отслеживать работоспособность управляемого домена.
* Контроллеры домена для управляемого домена исправлены и обновлены регулярно.
* Изменения из Azure Active Directory регулярно синхронизируются с управляемым доменом.
* Для управляемого домена выполняются регулярные резервные копии.

## <a name="needs-attention-state"></a>Требуется состояние предупреждения

Управляемый домен AD DS Azure с одной или несколькими проблемами, которые необходимо исправить, находится в состоянии " *требуется вмешательство* ". На странице работоспособность управляемого домена перечислены предупреждения и указано, где возникла проблема. Некоторые оповещения являются временными и автоматически решаются платформой Azure. Для других оповещений можно устранить проблему, выполнив приведенные действия. Это критическое оповещение. Отправьте [запрос в службу поддержки Azure][azure-support] для получения дополнительных сведений об устранении неполадок.

Одним из примеров предупреждения является наличие группы безопасности сети. В этой конфигурации Платформа Azure может не поддерживать обновление и мониторинг управляемого домена. Создается предупреждение и изменяется состояние, *требующее внимания*.

Дополнительные сведения см. [в статье Устранение неполадок оповещений для управляемого домена AD DS Azure][resolve-alerts].

### <a name="what-to-expect"></a>Чего следует ожидать

Если управляемый домен AD DS Azure находится в состоянии " *требуется вмешательство* ", Платформа Azure не может регулярно выполнять мониторинг, исправление, обновление или архивацию данных. В некоторых случаях, например с неверной конфигурацией сети, контроллеры домена для управляемого домена могут быть недоступными.

* Управляемый домен находится в неработоспособном состоянии, и текущий мониторинг работоспособности может завершиться до тех пор, пока не будет устранено предупреждение.
* Контроллеры домена для управляемого домена не могут быть исправлены или обновлены.
* Изменения из Azure Active Directory не могут быть синхронизированы с управляемым доменом.
* Резервное копирование для управляемого домена может быть не выполнено.
* При разрешении некритических предупреждений, влияющих на управляемый домен, работоспособность должна вернуться в состояние *выполнения* .
* Критические оповещения активируются для проблем конфигурации, когда Платформа Azure не может связаться с контроллерами домена. Если эти критические предупреждения не разрешены в течение 15 дней, управляемый домен переходит в состояние *suspended* .

## <a name="suspended-state"></a>Приостановленное состояние

Управляемый домен AD DS Azure переходит в состояние **suspended** по одной из следующих причин:

* Одна или несколько проблем в критических оповещениях не были разрешены в течение 15 дней.
    * Критические предупреждения могут быть вызваны неправильной конфигурацией, которая блокирует доступ к ресурсам, необходимым для Azure AD DS. Например, в случае оповещения "[AADDS104: ошибка сети][alert-nsg] в управляемом домене не была устранена в течение 15 дней".
* Возникла ошибка выставления счетов в подписке Azure, или истек срок действия подписки Azure.

Управляемые домены приостанавливаются, когда Платформа Azure не может управлять, отслеживать, исправлять или создавать резервные копии домена. Управляемый домен остается в *приостановленном* состоянии в течение 15 дней. Чтобы обеспечить доступ к управляемому домену, немедленно устраните критические предупреждения.

### <a name="what-to-expect"></a>Чего следует ожидать

Если управляемый домен AD DS Azure находится в *приостановленном* состоянии, возникают следующие действия:

* Контроллеры домена для управляемого домена перестают быть подготовлены и недоступны в виртуальной сети.
* Защищенный протокол LDAP доступ к управляемому домену через Интернет, если он включен, перестает работать.
* При проверке подлинности в управляемом домене возникают сбои, вход на виртуальные машины, присоединенные к домену, или подключение по протоколу LDAP/LDAPs.
* Резервные копии для управляемого домена больше не выполняются.
* Синхронизация с Azure AD прекращена.

### <a name="how-do-you-know-if-your-managed-domain-is-suspended"></a>Выявление приостановки работы управляемого домена

Вы увидите [оповещение][resolve-alerts] на странице работоспособности AD DS Azure в портал Azure, в которой заметка о том, что домен приостановлен. Состояние домена также будет отображаться как *suspended*.

### <a name="restore-a-suspended-domain"></a>Восстановление приостановленного домена

Чтобы восстановить работоспособность управляемого домена AD DS Azure, который находится в *приостановленном* состоянии, выполните следующие действия.

1. В портал Azure найдите и выберите **доменные службы**.
1. Выберите управляемый домен AD DS Azure из списка, например *contoso.com*, а затем выберите **работоспособность**.
1. Выберите предупреждение, например *AADDS503* или *AADDS504*, в зависимости от причины приостановки.
1. Выберите ссылку для разрешения, указанную в предупреждении, и выполните действия по ее устранению.

Управляемый домен можно восстановить только до даты последнего резервного копирования. Дата последнего резервного копирования отображается на странице **работоспособность** управляемого домена. Любые изменения, произошедшие после создания последней резервной копии, не подлежат восстановлению. Резервные копии для управляемого домена хранятся в течение 30 дней. Резервные копии, которым больше 30 дней, удаляются.

После разрешения предупреждений, когда управляемый домен находится в *приостановленном* состоянии, [откройте запрос в службу поддержки Azure][azure-support] , чтобы вернуться в работоспособное состояние. Если резервная копия составляет менее 30 дней, служба поддержки Azure может восстановить управляемый домен.

## <a name="deleted-state"></a>Удаленное состояние

Если управляемый домен AD DS Azure остается в *приостановленном* состоянии в течение 15 дней, он удаляется. Этот процесс недоступен для восстановления.

### <a name="what-to-expect"></a>Чего следует ожидать

Когда управляемый домен AD DS Azure переходит в *Удаленное* состояние, отображается следующее поведение:

* Все ресурсы и резервные копии управляемого домена будут удалены.
* Вы не можете восстановить управляемый домен, и вам нужно создать заменяющий управляемый домен для повторного использования Azure AD DS.
* После удаления вам не выставляются счета за управляемый домен.

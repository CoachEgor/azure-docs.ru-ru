# <a name="enable-azure-active-directory-domain-services-using-the-azure-portal"></a>Включение доменных служб Azure Active Directory с помощью портала Azure.


## <a name="before-you-begin"></a>Перед началом работы
См. статью [Рекомендации по сетям для доменных служб Azure AD](active-directory-ds-networking.md).


## <a name="task-2-configure-network-settings"></a>Задача 2. Настройка сетевых параметров
Следующая задача по настройке — создать виртуальную сеть Azure c выделенной подсетью. Вам потребуется включить доменные службы Azure Active Directory в этой подсети. Можно также выбрать существующую виртуальную сеть и создать выделенную подсеть внутри нее.

1. Щелкните **Виртуальная сеть**, чтобы выбрать виртуальную сеть.
    > [!NOTE]
    > **Классические виртуальные сети не поддерживаются для новых развертываний.** Классические виртуальные сети не поддерживаются для новых развертываний. Существующие управляемые домены, развернутые в классических виртуальных сетях, по-прежнему поддерживаются. В ближайшем будущем корпорация Майкрософт предоставит возможность переносить имеющийся управляемый домен из классической виртуальной сети в виртуальную сеть Azure Resource Manager.
    >

2. На странице **Выбор виртуальной сети** представлены все существующие виртуальные сети. Отображаются только виртуальные сети, принадлежащие к группе ресурсов и расположению Azure, которые выбраны странице **Основное** мастера.
3. Выберите виртуальную сеть, в которой следует включить доменные службы Azure AD. Можно выбрать существующую учетную запись хранения или создать новую.

   > [!TIP]
   > **После включения доменных служб Azure AD невозможно переместить управляемый домен в другую виртуальную сеть.** Выберите нужную виртуальную сеть для включения управляемого домена. Вы не сможете переместить созданный управляемый домен в другую виртуальную сеть, не удалив его. Мы рекомендуем ознакомиться со статьей [Рекомендации по сетям для доменных служб Azure AD](active-directory-ds-networking.md), прежде чем продолжить.  
   >

4. **Создание виртуальной сети.** Щелкните **Создать новую** для создания виртуальной сети. Используйте выделенную подсеть для доменных служб Azure Active Directory. Например, можно создать подсеть с именем "DomainServices", чтобы другие администраторы смогли с легкостью понять, что развернуто в подсети. По завершении нажмите кнопку **ОК**.

    ![Выбор виртуальной сети](./media/getting-started/domain-services-blade-network-pick-vnet.png)

   > [!WARNING]
   > Используйте адресное пространство, которое входит в пространство частных IP-адресов. Не принадлежащие вам IP-адреса, которые входят в пространство общедоступных адресов, вызывают ошибки в доменных службах Azure AD.

5. **Существующая виртуальная сеть.** Если вы планируете выбрать имеющуюся виртуальную сеть, [создайте выделенную подсеть с помощью расширения виртуальных сетей](../virtual-network/virtual-network-manage-subnet.md#add-a-subnet), а затем выберите эту подсеть. Щелкните **Виртуальная сеть**, чтобы выбрать существующую виртуальную сеть. Щелкните **Подсеть**, чтобы выбрать выделенную подсеть в существующей виртуальной сети, в которой нужно включить новый управляемый домен. По завершении нажмите кнопку **ОК**.

    ![Выбор подсети в виртуальной сети](./media/getting-started/domain-services-blade-network-pick-subnet.png)

   > [!NOTE]
   > **Рекомендации по выбору подсети**
   > 1. Используйте выделенную подсеть для доменных служб Azure Active Directory. Не развертывайте другие виртуальные машины в этой подсети. Такая конфигурация позволяет настроить группы безопасности сети (NSG) для рабочих нагрузок и виртуальных машин без нарушения целостности управляемого домена. Дополнительные сведения см. в статье [Рекомендации по сетям для доменных служб Azure AD](active-directory-ds-networking.md).
   > 2. Не используйте подсеть шлюза для развертывания доменных служб Azure Active Directory, поскольку это не является поддерживаемой конфигурацией.
   > 3. Выбранные подсети должны иметь в диапазоне 3–5 доступных IP-адресов.

6. По завершении нажмите **ОК**, чтобы перейти к странице **Группа администраторов** в мастере.

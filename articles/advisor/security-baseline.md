# <a name="azure-security-baseline-for-azure-advisor"></a>Базовый план безопасности Azure для помощника по Azure

Этот базовый план безопасности применяет рекомендации из [тестового показателя безопасности Azure версии 2,0](../security/benchmarks/overview.md) к советнику по Azure. Azure Security Benchmark содержит рекомендации по обеспечению безопасности облачных решений в Azure. Содержимое группируются по **элементам управления безопасностью** , определенным в тестовом производительности системы безопасности Azure, и связанным рекомендациям, применимым к помощнику по Azure. **Элементы управления** , неприменимые к помощнику по Azure, были исключены.

Чтобы узнать, как Azure Advisor полностью сопоставляется с тестовым показателем безопасности Azure, ознакомьтесь с [полным файлом сопоставления базовых показателей безопасности Azure Advisor](https://github.com/MicrosoftDocs/SecurityBenchmarks/tree/master/Azure%20Offer%20Security%20Baselines).

### <a name="im-1-standardize-azure-active-directory-as-the-central-identity-and-authentication-system"></a>IM-1. Стандартизация Azure Active Directory как центральной системы идентификации и проверки подлинности

**Руководство**. Помощник по azure использует Azure Active Directory (Azure AD) в качестве службы управления удостоверениями и доступом по умолчанию. Стандартизация Azure AD для управления удостоверениями и доступом в вашей организации в:

- Microsoft Cloud ресурсы, такие как портал Azure, служба хранилища Azure, виртуальная машина Azure (Linux и Windows), Azure Key Vault, PaaS и приложения SaaS
- Ресурсы Организации, такие как приложения в Azure или ресурсы корпоративной сети;

Обеспечьте безопасность Azure AD с высоким приоритетом в целях безопасности в облаке вашей организации. Azure AD также предоставляет удостоверение безопасного удостоверения, помогающее оценить безопасность идентификации по сравнению с рекомендациями Майкрософт. Используйте оценку, чтобы понять, насколько точно ваша конфигурация соответствует рекомендациям, и улучшить состояние безопасности.

Обратите внимание, что Azure AD поддерживает внешние удостоверения, которые позволяют пользователям без учетная запись Майкрософт входить в свои приложения и ресурсы с их внешним удостоверением.

- [Создание и настройка экземпляра Azure AD](../active-directory/fundamentals/active-directory-access-create-new-tenant.md) 

- [Использование внешних поставщиков удостоверений для приложения](../active-directory/external-identities/identity-providers.md) 

- [Оценка безопасности идентификации в Azure Active Directory](../active-directory/fundamentals/identity-secure-score.md)

### <a name="im-3-use-azure-ad-single-sign-on-sso-for-application-access"></a>IM-3. Использование единого входа (SSO) Azure AD для доступа к приложениям

**Руководство**. Помощник по azure использует Azure Active Directory (Azure AD) для предоставления управления удостоверениями и доступом к ресурсам Azure, облачным приложениям и локальным приложениям. Сюда входят корпоративные удостоверения, например сотрудники, а также внешние удостоверения, например партнеры и поставщики. 

Единый вход используется для управления доступом к данным и ресурсам Организации в локальной среде и в облаке, а также для обеспечения безопасного доступа к ним. Подключайте пользователей, приложения и устройства к Azure AD, чтобы обеспечить простой, безопасный доступ и более широкие возможности видимости и управления.

### <a name="im-4-use-strong-authentication-controls-for-all-azure-active-directory-based-access"></a>IM-4. Использование элементов управления строгой проверки подлинности для любого доступа на основе Azure Active Directory

**Руководство**. Помощник по azure использует Azure Active Directory (Azure AD), который поддерживает надежные элементы управления проверкой подлинности с помощью многофакторной проверки подлинности и надежные методы, не требующие пароля.
- Многофакторная проверка подлинности. Включите многофакторную проверку подлинности Azure AD и следуйте рекомендациям по управлению удостоверениями и доступом центра безопасности Azure для некоторых рекомендаций в настройке многофакторной проверки подлинности. Многофакторная проверка подлинности может быть применена ко всем, выбирать пользователей или на уровне отдельных пользователей в зависимости от условий входа и факторов риска.
- Проверка подлинности с паролем — доступны три варианта проверки подлинности с паролем. К ним относятся, Windows Hello для бизнеса, Microsoft Authenticator приложение и локальные методы проверки подлинности, такие как смарт-карты.

Для администраторов и привилегированных пользователей убедитесь, что используется самый высокий уровень строгой проверки подлинности с последующим развертыванием соответствующей политики строгой проверки подлинности для других пользователей.

### <a name="im-5-monitor-and-alert-on-account-anomalies"></a>IM-5. Мониторинг аномалий учетных записей и соответствующее оповещение

**Руководство**. Помощник по Azure интегрируется с Azure Active Directory (Azure AD), в котором предоставляются следующие источники данных:
- Вход. отчет о входе содержит сведения об использовании управляемых приложений и действиях входа пользователей.
- Журналы аудита — возможность отслеживания с помощью журналов всех изменений, внесенных при использовании разных функций в Azure AD. Примеры журналов аудита включают изменения, внесенные в такие ресурсы в Azure AD, как добавление или удаление пользователей, приложений, групп, ролей и политик.
- Рискованный вход. рискованный вход — это индикатор попытки входа, которая могла быть выполнена пользователем, который не является законным владельцем учетной записи пользователя.
- Пользователи, находящиеся в группе риска. Такая пометка означает, что конфиденциальность учетной записи пользователя, возможно, нарушена.

Используйте эти источники данных для интеграции с Azure Monitor, Sentinel Azure или сторонними системами SIEM. Настройка оповещений в центре безопасности Azure для определенных подозрительных действий, например чрезмерного числа неудачных попыток проверки подлинности, нерекомендуемых учетных записей в подписке.

Расширенная защита от угроз Azure (ATP) — это решение для обеспечения безопасности, которое может использовать сигналы Active Directory для выявления, обнаружения и изучения дополнительных угроз, скомпрометированных удостоверений и внутренних вредоносных действий.

### <a name="lt-4-enable-logging-for-azure-resources"></a>LT-4: Включение ведения журнала для ресурсов Azure

### <a name="gs-7-define-logging-and-threat-response-strategy"></a>GS-7. Определение стратегии ведения журнала и реагирования на угрозы

**Руководство**. Создайте стратегию ведения журнала и реагирования на угрозы для быстрого обнаружения и устранения угроз при соблюдении требований к соответствию. Определите приоритет для аналитиков с помощью высококачественных оповещений и эффективного взаимодействия, чтобы они могли сосредоточиться на угрозах, а не на интеграции и выполнении действий вручную. 

Эта стратегия должна включать задокументированные руководство, политику и стандарты для следующих элементов: 

-   Роль и обязанности организации по операциям безопасности (SecOps). 

-   Четко определенный процесс реагирования на инциденты, согласованный с NIST или другой платформой отрасли. 

-   Запись и хранение журналов для поддержки обнаружения угроз, реагирования на инциденты и соответствия требованиям.

-   Централизованный просмотр сведений и корреляционный анализ информации об угрозах, использовании SIEM, собственных возможностях Azure и других источниках. 

-   Взаимодействие с вашими клиентами, поставщиками и публичными сторонами, а также план их уведомления.

-   Использование собственных платформ Azure и платформ сторонних производителей для обработки инцидентов. Например, для ведения журнала и обнаружения угроз, судебных экспертиз, исправления и удаления последствий атак.

-   Процессы обработки инцидентов и действия, выполняемые после инцидента, например использование полученного опыта и хранение доказательств.

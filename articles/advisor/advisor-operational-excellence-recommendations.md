# <a name="achieve-operational-excellence-with-azure-advisor"></a>Эффективное функционирование с помощью помощника по Azure

Рекомендации по эффективному использованию Azure Advisor помогают клиентам повысить эффективность процессов и рабочих процессов, а также рекомендации по управлению ресурсами и развертыванию. Эти рекомендации можно получить из помощника на вкладке " **эффективность** работы" панели мониторинга Advisor.

## <a name="create-azure-service-health-alerts-to-be-notified-when-azure-issues-affect-you"></a>Создание оповещений о Работоспособности служб Azure, чтобы узнавать о возникновении проблем в Azure

Мы рекомендуем настроить оповещения о работоспособности служб Azure, чтобы получать уведомления на случай возникновения проблем со службами Azure. [Работоспособность служб Azure](https://azure.microsoft.com/features/service-health/) — это бесплатная служба, которая предоставляет персонализированные инструкции и поддержку в случае возникновения проблем со службой Azure. Помощник определяет подписки, для которых не настроены оповещения, и рекомендует создать их.

## <a name="design-your-storage-accounts-to-prevent-hitting-the-maximum-subscription-limit"></a>Разработайте учетные записи хранения, чтобы не допустить превышения максимального числа подписок.

Регион Azure может поддерживать не более 250 учетных записей хранения на подписку. После достижения этого ограничения вы не сможете создавать дополнительные учетные записи хранения в этой комбинации региона или подписки. Advisor проверит ваши подписки и Рекомендуемые области, чтобы создать меньшее количество учетных записей хранения для всех, которые близки к максимальному пределу.

## <a name="ensure-you-have-access-to-azure-cloud-experts-when-you-need-it"></a>Обеспечение доступа к экспертам по облаку Azure, когда это необходимо

При выполнении критически важной рабочей нагрузки важно иметь доступ к службе технической поддержки, когда он требуется. Помощник идентифицирует потенциальные критически важные для бизнеса подписки, в плане поддержки которых нет технической поддержки, и советует выполнить обновление до варианта, в котором присутствует техническая поддержка.

## <a name="repair-invalid-log-alert-rules"></a>Исправить недопустимые правила генерации оповещений журнала

Помощник по Azure будет обнаруживать правила генерации оповещений, в разделе условий которых указаны недопустимые запросы. Правила генерации оповещений журнала создаются в Azure Monitor и используются для выполнения запросов аналитики через заданные интервалы. Результаты запроса определяют, нужно ли активировать оповещение. Со временем запросы аналитики могут стать недопустимыми из-за изменений в ресурсах, таблицах или командах, на которые ссылается запрос. Помощник порекомендует исправить запрос в правиле оповещения, чтобы предотвратить его автоматическое отключение и обеспечить мониторинг ресурсов в Azure. [Дополнительные сведения об устранении неполадок в правилах оповещений](https://aka.ms/aa_logalerts_queryrepair)

## <a name="follow-best-practices-using-azure-policy"></a>Соблюдение рекомендаций с помощью политики Azure

Политика Azure — это служба в Azure, которая используется для создания и присваивания политик, а также управления ими. Эти политики обеспечивают соблюдение различных правил и выполнение действий с вашими ресурсами. Ниже приведены рекомендации по политикам Azure, которые помогут достичь эксплуатационных ексцелленци: 
1. Управление тегами с помощью политики Azure. Эта политика добавляет или заменяет указанный тег и значение при создании или обновлении любого ресурса. Существующие ресурсы можно исправить, активировав задачу исправления. Кроме того, не изменяет теги в группах ресурсов.
2. Соблюдение требований к географическому соответствию с помощью политики Azure. Политика позволяет ограничить расположения, которые может указывать организация при развертывании ресурсов. 
3. Укажите разрешенные номера SKU виртуальных машин для развертываний. Эта политика позволяет указать набор номеров SKU виртуальных машин, которые может развернуть Ваша организация.
4. Принудительно применить "аудит виртуальных машин, не использующих управляемые диски" с помощью политики Azure
5. Используйте команду "наследовать тег от групп ресурсов" с помощью политики Azure. Политика добавляет или заменяет указанный тег и значение из родительской группы ресурсов при создании или обновлении ресурсов. Существующие ресурсы можно исправить, активировав задачу исправления.

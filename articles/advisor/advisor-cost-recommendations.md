# <a name="reduce-service-costs-using-azure-advisor"></a>Снижение затрат на обслуживание с помощью средства "Помощник по Azure"

Advisor помогает оптимизировать и уменьшить общие расходы на Azure, выявляя простаивающие и недостаточно нагруженные ресурсы. Рекомендации по затратам можно получить на вкладке **Стоимость** панели мониторинга Помощника.

## <a name="optimize-virtual-machine-spend-by-resizing-or-shutting-down-underutilized-instances"></a>Оптимизация затрат на виртуальные машины путем изменения размеров или завершения работы недостаточно нагруженных экземпляров 

Хотя в некоторых сценариях приложение может намеренно обеспечивать низкую загрузку, часто можно сократить затраты, управляя размером и числом виртуальных машин. Помощник отслеживает использование виртуальных машин в течение 7 дней и затем определяет недостаточно нагруженными виртуальные машины. Виртуальных машин считаются недостаточно нагруженными если их использование ресурсов ЦП составляет 5% или меньше и эффективность использования сети составляет менее % 2, или если текущей рабочей нагрузки может быть размещен на имеющемся меньший размер виртуальной машины.

Помощник показывает предполагаемую стоимость продолжения работы виртуальной машины, поэтому вы можете завершить ее работу или изменить ее размер.

Если вы хотите интенсивнее определять недостаточно нагруженные виртуальные машины, то можете настроить правило среднего показателя использования ЦП для каждой подписки.

## <a name="reduce-costs-by-eliminating-unprovisioned-expressroute-circuits"></a>Сокращение расходов за счет исключения неподготовленных каналов ExpressRoute

Помощник определяет каналы ExpressRoute, которые находятся в состоянии поставщика *Не подготовлено* больше месяца, и рекомендует удалить такие каналы, если вы не планируете их подготовить с помощью поставщика соединения.

## <a name="reduce-costs-by-deleting-or-reconfiguring-idle-virtual-network-gateways"></a>Сокращение расходов за счет удаления или перенастройки неактивных шлюзов виртуальной сети

Помощник определяет шлюзы виртуальной сети, которые простаивают более 90 дней. Поскольку эти шлюзы оплачиваются ежечасно, следует рассмотреть возможность их перенастройки или удаления, если вы больше не собираетесь их использовать. 

## <a name="buy-reserved-virtual-machine-instances-to-save-money-over-pay-as-you-go-costs"></a>Приобретение зарезервированных экземпляров виртуальных машин для экономии средств по сравнению с применением оплаты по мере использования

Помощник рассмотрит данные об использовании виртуальной машины за последние 30 дней и определит, можно ли сэкономить на приобретении резервирования Azure. Он предоставит вам данные о регионах и размерах, в которых можно сэкономить больше всего средств, и покажет ожидаемую экономию в результате приобретения резервирований. Приобретая резервирования Azure, можно предварительно оплатить базовую стоимость виртуальных машин. Скидки будут автоматически применяться к новым или имеющимся виртуальным машинам с тем же размером и регионом, для которых приобретены резервирования. [Узнайте больше о зарезервированных экземплярах виртуальных машин Azure.](https://azure.microsoft.com/pricing/reserved-vm-instances/)

Помощник по также будет уведомлять вас о зарезервированные экземпляры, которые у вас есть которых истекает в течение 30 дней. Мастер может порекомендовать приобрести новые зарезервированные экземпляры, чтобы избежать оплаты оплаты.

## <a name="delete-unassociated-public-ip-addresses-to-save-money"></a>Удалить несвязанные общедоступные IP-адреса для экономии

Помощник определяет общедоступные IP-адреса, которые не связаны в настоящее время к ресурсам Azure, такие как виртуальные машины или подсистемы балансировки нагрузки. Эти общедоступный IP-адреса можно получить с помощью номинальная плата. Если вы не планируете их использовать, их удаление может привести к снижению затрат.

## <a name="delete-azure-data-factory-pipelines-that-are-failing"></a>Удалить конвейеров фабрики данных Azure, завершающихся неудачей

Помощник по Azure обнаружит конвейеров фабрики данных Azure, которые многократно ошибкой и рекомендуется устранить проблемы и удалить конвейеры сбоя, если они больше не требуются. Вы будете оплачивать эти конвейеры даже если, однако они не обслуживают вы пока они выполняются. 

## <a name="use-standard-snapshots-for-managed-disks"></a>Используйте стандартный моментальные снимки для управляемых дисков
Чтобы сохранить 60% стоимости, рекомендуется хранить моментальных снимков в стандартном хранилище, независимо от типа хранилища на родительском диске. Это параметр по умолчанию для моментальных снимков управляемые диски. Помощник по Azure определит моментальных снимков, которые хранятся хранилище класса Premium и миграция создания моментальных снимков с уровня "премиум" в стандартное хранилище рекомендуется использовать. [Дополнительные сведения о ценах на управляемый диск](https://aka.ms/aa_manageddisksnapshot_learnmore)

## <a name="how-to-access-cost-recommendations-in-azure-advisor"></a>Как получить доступ к рекомендациям по затратам в Помощнике по Azure?

1. Войдите на [портал Azure](https://portal.azure.com) и откройте [Помощник](https://aka.ms/azureadvisordashboard).

2.  На панели мониторинга Помощника щелкните вкладку **Стоимость**.

# <a name="reduce-service-costs-using-azure-advisor"></a>Снижение затрат на обслуживание с помощью средства "Помощник по Azure"

Advisor помогает оптимизировать и уменьшить общие расходы на Azure, выявляя простаивающие и недостаточно нагруженные ресурсы.Рекомендации по затратам можно получить на вкладке **Стоимость** панели мониторинга Помощника.

## <a name="optimize-virtual-machine-spend-by-resizing-or-shutting-down-underutilized-instances"></a>Оптимизация затрат на виртуальные машины путем изменения размеров или завершения работы недостаточно нагруженных экземпляров 

Хотя в некоторых сценариях приложение может намеренно обеспечивать низкую загрузку, часто можно сократить затраты, управляя размером и числом виртуальных машин. Советник передовых моделей оценки считает виртуальные машины для выключения, когда P95th из максимальной стоимости максимального значения использования процессора составляет менее 3%, а использование сети составляет менее 2% в течение 7-дневного периода. Виртуальные машины учитываются для правильного размера, когда можно поместить текущую нагрузку в меньшем SKU (в пределах того же семейства SKU) или меньшее число экземпляров, так что текущая нагрузка не превышает 80% использования, когда непользователь облицованная рабочей нагрузкой и не выше 40% при рабочей нагрузке, обнажённые пользователем. Здесь тип рабочей нагрузки определяется путем анализа характеристик использования процессора рабочей нагрузки.

Рекомендуемые действия выключаются или изменяются, относясь к рекомендованному ресурсу. Advisor показывает вам предполагаемую экономию затрат для рекомендуемых действий - изменения размера или выключения. Кроме того, для повторного размера рекомендуемых действий, Советник предоставляет текущую и целевую информацию SKU. 

Если вы хотите быть более агрессивными при выявлении недостаточно используемых виртуальных машин, вы можете настроить правило использования процессора на основе подписки.

## <a name="reduce-costs-by-eliminating-unprovisioned-expressroute-circuits"></a>Сокращение расходов за счет исключения неподготовленных каналов ExpressRoute

Помощник определяет каналы ExpressRoute, которые находятся в состоянии поставщика *Не подготовлено* больше месяца, и рекомендует удалить такие каналы, если вы не планируете их подготовить с помощью поставщика соединения.

## <a name="reduce-costs-by-deleting-or-reconfiguring-idle-virtual-network-gateways"></a>Сокращение расходов за счет удаления или перенастройки неактивных шлюзов виртуальной сети

Advisor определяет виртуальные сетевые шлюзы, которые простаивали более 90 дней. Поскольку эти шлюзы оплачиваются ежечасно, следует рассмотреть возможность их перенастройки или удаления, если вы больше не собираетесь их использовать. 

## <a name="buy-reserved-virtual-machine-instances-to-save-money-over-pay-as-you-go-costs"></a>Приобретение зарезервированных экземпляров виртуальных машин для экономии средств по сравнению с применением оплаты по мере использования

Помощник рассмотрит данные об использовании виртуальной машины за последние 30 дней и определит, можно ли сэкономить на приобретении резервирования Azure. Он предоставит вам данные о регионах и размерах, в которых можно сэкономить больше всего средств, и покажет ожидаемую экономию в результате приобретения резервирований. Приобретая резервирования Azure, можно предварительно оплатить базовую стоимость виртуальных машин. Скидки будут автоматически применяться к новым или имеющимся виртуальным машинам с тем же размером и регионом, для которых приобретены резервирования. [Узнайте больше о зарезервированных экземплярах виртуальных машин Azure.](https://azure.microsoft.com/pricing/reserved-vm-instances/)

Advisor также уведомит вас о зарезервированных случаях, которые истекают в течение следующих 30 дней. Он будет рекомендовать вам приобрести новые зарезервированные экземпляры, чтобы избежать оплаты по мере использования цен.

## <a name="delete-unassociated-public-ip-addresses-to-save-money"></a>Удаление несвязанных публичных IP-адресов, чтобы сэкономить деньги

Советник определяет общедоступные IP-адреса, которые в настоящее время не связаны с ресурсами Azure, такими как балансы нагрузки или VMs. Эти публичные IP-адреса поставляются с номинальной платой. Если вы не планируете использовать их, их удаляние может привести к экономии средств.

## <a name="delete-azure-data-factory-pipelines-that-are-failing"></a>Удаление конвейеров Фабрики данных Azure, которые завершаются сбоем

Советник Azure обнаружит конвейеры Azure Data Factory, которые неоднократно выходят из строя, и рекомендует устранить проблемы или удалить неисправные конвейеры, если они больше не нужны. Вам будет выставлен счет за эти трубопроводы, даже если они не обслуживают вас, пока они не удается. 

## <a name="use-standard-snapshots-for-managed-disks"></a>Используйте стандартные снимки для управляемых дисков
Чтобы сэкономить 60 % стоимости, рекомендуем хранить моментальные снимки в Службе хранилища (цен. категория 'Стандартный') независимо от типа хранилища родительского диска. Эта опция является опцией по умолчанию для снимков управляемых дисков. Советник Azure идентифицирует снимки, которые хранятся Premium Storage, и рекомендует переходить с Премиум-хранилища на стандартные хранилища. [Подробнее о ценах на управляемые диски](https://aka.ms/aa_manageddisksnapshot_learnmore)

## <a name="utilize-lifecycle-management"></a>Использование управления жизненным циклом
Azure Advisor будет использовать сведения о количестве объектов хранения azure blob, общем размере и транзакциях, чтобы определить, лучше всего ли будет подходить один или несколько учетных записей хранилища для управления жизненным циклом. Это подскажет вам создать правила управления жизненным циклом, чтобы автоматически уровень вашей данных Cool или Архив для оптимизации расходов на хранение при сохранении данных в Azure blob хранения для совместимости приложений.

## <a name="how-to-access-cost-recommendations-in-azure-advisor"></a>Как получить доступ к рекомендациям по затратам в Помощнике по Azure?

1. Войдите на [портал Azure](https://portal.azure.com).

1. Поиск и выберите [**советника**](https://aka.ms/azureadvisordashboard) с любой страницы.

1. На панели мониторинга **Advisor** выберите вкладку **«Стоимость».**

# <a name="use-one-click-ingestion-to-ingest-data-into-azure-data-explorer"></a>Использование приема одним щелчком для приема данных в Azure обозреватель данных

В этой статье показано, как использовать прием одним щелчком для быстрого приема новой таблицы в форматах JSON или CSV из хранилища в обозреватель данных Azure. После приема данных можно изменить таблицу и выполнить запросы с помощью веб-интерфейса.

## <a name="prerequisites"></a>Предварительные требования

* Если у вас еще нет подписки Azure, создайте [бесплатную учетную запись](https://azure.microsoft.com/free/) Azure, прежде чем начинать работу.
* Выполните [вход в приложение](https://dataexplorer.azure.com/).
* Создание [кластера Azure обозреватель данных и базы данных](create-cluster-database-portal.md)
* Вход в [веб-интерфейс](https://dataexplorer.azure.com/) и [Добавление подключения к кластеру](/azure/data-explorer/web-query-data#add-clusters)
* Источник данных в службе хранилища Azure.

## <a name="ingest-new-data"></a>Прием новых данных

1. Щелкните правой кнопкой мыши *базу данных* или строку *таблицы* в левом меню веб-интерфейса и выберите пункт принять **новые данные (Предварительная версия)** .

    ![Выбор одного приема щелчка в веб-ИНТЕРФЕЙСе](media/ingest-data-one-click/one-click-ingestion-in-webui.png)   
 
1. В окне прием **новых данных (Предварительная версия)** на вкладке **Исходный код** введите **сведения о проекте**.

    * **Таблица**: выберите имя существующей таблицы из раскрывающегося списка или щелкните **создать** , чтобы создать новую таблицу.
    * Выберите **тип приема** > **из хранилища** или **из файла**.
        * Если вы выбрали **из хранилища**, введите **ссылку на хранилище** , чтобы добавить URL-адрес в хранилище. Используйте [URL-адрес SAS BLOB](/azure/vs-azure-tools-storage-explorer-blobs#get-the-sas-for-a-blob-container) для частных учетных записей хранения. 
        * Если вы выбрали **из файла**, нажмите кнопку **Обзор** и перетащите файл в поле.
    * Выберите **изменить схему** , чтобы просмотреть и изменить конфигурацию столбца таблицы.
 
    ![сведения об источнике приема одного щелчка](media/ingest-data-one-click/one-click-ingestion-source.png) 

    > [!TIP]
    > Если выбрать параметр принять **новые данные (Предварительная версия)** в строке *таблицы* , то выбранное имя таблицы появится в **сведениях о проекте**.

1. Если выбрана существующая таблица, откроется окно **Map Columns** , в котором сопоставляются столбцы исходных данных с целевыми столбцами таблицы. 
    * Используйте **параметр опустить столбец** , чтобы удалить целевой столбец из таблицы. 
    * Используйте **новый столбец** , чтобы добавить новый столбец в таблицу. 

    ![Окно "схема столбцов"](media/ingest-data-one-click/one-click-map-columns-window.png)

1. На вкладке **схема** :

    * Выберите **тип сжатия** из раскрывающегося списка > **несжатые** **файлы или gzip**.
    * Выберите **Формат данных** из раскрывающегося списка **> JSON**, **CSV**, **TSV**, **сксв**, **сохсв**, **тсве**или **ПСВ**. 
        * Выбрав формат **JSON** , выберите **уровни JSON**: 1-10. Уровни влияют на отображение данных столбца таблицы. 
        * При выборе формата, отличного от JSON: установите флажок **включить имена столбцов** , чтобы игнорировать строку заголовка файла.    
    * **Имя сопоставления** задается автоматически, но может быть изменено.
    * Если выбрана существующая таблица, можно нажать кнопку **Map Columns (связать столбцы** ), чтобы открыть окно « **Map Columns** ».

    ![Схема CSV формата приема одного щелчка. png](media/ingest-data-one-click/one-click-csv-format.png)

1. В **редакторе**выберите **V** справа, чтобы открыть редактор. В редакторе можно просматривать и копировать автоматические запросы, созданные на основе входных данных. 

1.  В таблице: 
    * Щелкните правой кнопкой мыши заголовки новых столбцов, чтобы **изменить тип данных**, **Переименовать столбец**, **удалить столбец**, **Сортировать по возрастанию**или **Сортировать по убыванию**. В существующих столбцах доступна только сортировка данных. 
    * Дважды щелкните имя нового столбца, чтобы изменить его.

1. Выберите **начать** прием для создания таблицы, создания сопоставления и приема данных.

    ![Схема формата JSON для приема одного щелчка](media/ingest-data-one-click/one-click-json-format.png) 
 
## <a name="query-data"></a>Запрос данных

1. В окне **приема данных завершено** все три шага будут отмечены зелеными флажками, если прием данных успешно завершился. 
 
    ![прием данных одним щелчком завершен](media/ingest-data-one-click/one-click-data-ingestion-complete.png)

1. Выберите **V** , чтобы открыть запрос. Скопируйте в веб-интерфейс, чтобы изменить запрос.

1. Меню справа содержит **быстрые запросы** и **средства**. 

    * **Быстрые запросы** включают ссылки на веб-интерфейс с примерами запросов.
    * В состав **средств** входит ссылка на пользовательский веб-интерфейс с **командами Drop** , которые позволяют устранять неполадки, выполняя соответствующую команду `.drop`.

    > [!TIP]
    > Данные могут быть потеряны с помощью команд `.drop`. Внимательно используйте их.

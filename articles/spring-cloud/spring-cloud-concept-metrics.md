# <a name="metrics-for-azure-spring-cloud"></a>Метрики для Azure Веснного облака

Azure Monitor обозреватель метрик является компонентом портал Microsoft Azure, который позволяет строить диаграммы, визуально сопоставлять тенденции и изучать пики и DIP в метриках. Используйте обозреватель метрик для изучения работоспособности и использования ресурсов. В Azure с пружинным облаком мы предлагаем два варианта просмотра метрик: диаграммы на странице " **Обзор приложений** " и "метрики уровня обслуживания".

## <a name="application-overview-page"></a>Страница "Обзор приложения"

На странице **Обзор приложений** каждого приложения представлена диаграмма метрик, позволяющая выполнять быструю проверку состояния приложения.  Перейдите на страницу облачной службы Azure весны и выберите **панель мониторинга приложений**, а затем выберите приложение из списка.  

Мы предоставляем 5 диаграмм с метриками, обновленными каждую минуту в следующих целях:

* **Ошибки HTTP-сервера**: число ошибок для HTTP-запросов к приложению.
* **Данные в**: байтов, полученных приложением.
* **Исходящие данные**: отправленные в приложение байты.
* **Запросы**: запросы, полученные приложением.
* **Среднее время отклика**: среднее время отклика приложения.

Можно выбрать диапазон времени для диаграммы от 1 часа до 7 дней.

## <a name="service-level-metric-queries"></a>Запросы метрик уровня службы

Azure Веснного облака позволяет отслеживать различные метрики приложений. Ознакомьтесь с [руководством](https://docs.microsoft.com/azure/azure-monitor/platform/metrics-getting-started) по началу работы с метриками Azure Monitor, чтобы узнать больше об этой службе.

Чтобы проверить данные метрики, выберите метрику, **агрегирование**и диапазон времени.  Эти понятия описаны ниже.

### <a name="aggregation"></a>Агрегирование 

Azure опрашивает и обновляет метрики каждую минуту. Azure предоставляет три способа агрегирования данных за выбранный период времени:

* **Total**: суммировать все метрики в качестве целевых выходных данных.
* **Среднее**: используйте среднее значение периода в качестве целевого выходных данных.
* **Макс./мин**. Использование значения max/min в периоде в качестве целевого выходных данных.

### <a name="time-range"></a>Диапазон времени

Выберите диапазон времени по умолчанию или определите свой собственный.

### <a name="modifying-the-granularity-of-your-metric-query"></a>Изменение гранулярности запроса метрики

По умолчанию Azure собирает метрики для всех приложений облачной службы Azure весны. Чтобы проверить метрики на уровне приложения или экземпляра, используйте функцию Filter.  
Выберите **Добавить фильтр**, задайте для свойства значение **приложение** и выберите целевое приложение, которое требуется отслеживать. При необходимости используйте параметр **Применить разделение** , чтобы нарисовать отдельные линии для каждого приложения в диаграмме.

>[!TIP]
> Вы можете создать собственные диаграммы на странице метрики и закрепить их на **панели мониторинга**. Начните с именования диаграммы.  Затем выберите **закрепить на панели мониторинга в правом верхнем углу**. Теперь вы можете проверить приложение на **панели мониторинга**портала.

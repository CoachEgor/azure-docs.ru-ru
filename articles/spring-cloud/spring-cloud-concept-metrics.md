# <a name="metrics-for-azure-spring-cloud"></a>Метрики для Azure Spring Cloud

Обозреватель метрик Azure Monitor является компонентом портала Microsoft Azure, который позволяет создавать диаграммы, визуально коррелируемые тренды и изучать пики и спады в метриках. Используйте обозреватель метрик для анализа работоспособности и использования ресурсов. В Azure Spring Cloud доступны два варианта просмотра метрик: диаграммы на странице **Обзор приложения** и на странице"Метрики уровня обслуживания".

## <a name="application-overview-page"></a>Страница обзора приложения

На каждой странице **Обзор приложения** представлена диаграмма метрик, позволяющая выполнять быструю проверку состояния приложения.  Перейдите на страницу службы Azure Spring Cloud и выберите **Панель мониторинга приложения**, а затем выберите приложение в списке.  

Мы предоставляем 5 диаграмм с метриками, которые обновляются каждую минуту, отображая следующие данные.

* **Ошибки HTTP-сервера**: количество ошибок HTTP-запросов к приложению.
* **Входные данные**: байтов, полученных приложением.
* **Выходные данные**: байтов, отправленных в приложение.
* **Запросы**: запросы, полученные приложением.
* **Среднее время ответа**: среднее время отклика от приложения.

Для диаграммы можно выбрать диапазон времени от 1 часа до 7 дней.

## <a name="service-level-metric-queries"></a>Запросы метрик уровня службы

Azure Spring Cloud позволяет отслеживать различные метрики приложений. Дополнительные сведения об этой службе см. в [руководстве по началу работы](https://docs.microsoft.com/azure/azure-monitor/platform/metrics-getting-started) с метриками Azure Monitor.

Чтобы проверить данные метрик, выберите метрику, **агрегирование** и диапазон времени.  Эти понятия описаны ниже.

### <a name="aggregation"></a>Агрегирование 

Azure опрашивает и обновляет метрики каждую минуту. Azure предоставляет три способа агрегирования данных за выбранный период времени:

* **Total**: Суммировать все метрики в качестве целевых выходных данных.
* **Average**: Использовать среднее значение за период в качестве целевых выходных данных.
* **Макс./мин.** : Использовать максимальное и минимальное значения за период в качестве целевых выходных данных.

### <a name="time-range"></a>Диапазон времени

Выберите диапазон времени по умолчанию или определите собственный диапазон.

### <a name="modifying-the-granularity-of-your-metric-query"></a>Изменение точности запроса метрики

По умолчанию Azure собирает метрики для всех приложений службы Azure Spring Cloud. Чтобы проверить метрики на уровне приложения или экземпляра, используйте функцию фильтра.  
Выберите **Добавить фильтр**, задайте для свойства значение **Приложение** и выберите целевое приложение, которое требуется отслеживать. При необходимости используйте параметр **Apply splitting** (Разделить), чтобы нарисовать отдельные линии для каждого приложения на диаграмме.

>[!TIP]
> Вы можете создать собственные диаграммы на странице метрики и закрепить их на **Панели мониторинга**. Начните с названия диаграммы.  Затем выберите **Pin to dashboard in the top right corner** (Закрепить на панели в верхнем правом углу). Теперь можно проверить приложение на **Панели мониторинга** на портале.

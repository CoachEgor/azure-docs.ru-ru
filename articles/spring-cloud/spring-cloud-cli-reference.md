## <a name="az-spring-cloud-create"></a>аз пружинное облако создать

Создайте новое приложение с развертыванием по умолчанию в облачности Azure Spring.

```azurecli
az spring-cloud create  --name -n
                        --resource-group -g
                        --location -l
                        --no-wait
```

| Необходимые параметры | |
| --- | ---: |
| --имя -n | Название для этого экземпляра Облака весна Azure. |
| --resource-group -g | Определяет группу ресурсов для этого приложения.  Настройка группы по умолчанию с использованием`az configure --defaults group=<name>` |

| Необязательные параметры | |
| --- | ---: |
| --location -l | Определяет местоположение сервера для этого приложения.  Поиск действительных местоположений с помощью`az account list-locations` |
| --no-wait | Не выполняйте длительные операции.

### <a name="examples"></a>Примеры

Создание нового весеннего облака Azure в WestUS

```azurecli
az spring-cloud create -n MyService -g MyResourceGroup -l westus
```

## <a name="az-spring-cloud-app-create"></a>приложение az spring-cloud создает

Создайте новое приложение в облачности Azure Spring.

```azurecli
az spring-cloud app create --name -n
                            --resource-group -g
                            --service -s
                            --cpu
                            --enable-persistent-storage
                            --instance-count
                            --is-public
                            --memory
```

| Необходимые параметры | |
| --- | ---: |
| --имя -n | Имя приложения |
| --resource-group -g | Имя группы ресурсов.  Вы можете настроить расположение по умолчанию с помощью `az configure --defaults group=<name>`. |
| --сервис -s | Название весеннего облака Azure.  Вы можете настроить службу `az configure --defaults spring-cloud=<name>`по умолчанию с помощью. |

| Необязательные параметры | |
| --- | ---: |
| --cpu | Количество виртуальных ядер в экземпляре.  Значение по умолчанию: 1. |
| --включить-стойкий-хранение | .  Если это так, монтирует 50 ГБ диска с траекторией по умолчанию. |
| --instance-count | Количество экземпляров.  Значение по умолчанию: 1. |
| --это-общественный | .  Если это так, присваивает общественное достояние. |
| --memory | Количество ГБ памяти на экземпляр.  Значение по умолчанию: 1. |

### <a name="examples"></a>Примеры

Создайте приложение с конфигурацией по умолчанию.

```azurecli
az spring-cloud app create -n MyApp -s MyService
```

Создайте общедоступное приложение с 3 экземплярами.  Каждый экземпляр имеет 3 ГБ памяти и 2 процессорных ядра.

```azurecli
az spring-cloud app create -n MyApp -s MyService --is-public true --cpu 2 --memory 3
```

## <a name="az-spring-cloud-app-deploy"></a>развертывание приложения az spring-cloud

Развертывание приложения в облачное облако Azure Spring из исходного кода или предварительно построенного двоичного файла и обновление соответствующих конфигураций.

```azurecli
az spring cloud app deploy  --name -n
                            --resource-group -g
                            --service -s
                            --cpu
                            --deployment -d
                            --env
                            --instance-count
                            --jar-path
                            --jvm-options
                            --memory
                            --no-wait
                            --runtime-version
                            --target-module
                            --version
```

| Необходимые параметры | |
| --- | ---: |
| --имя -n | Имя приложения |
| --resource-group -g | Имя группы ресурсов.  Вы можете настроить расположение по умолчанию с помощью `az configure --defaults group=<name>`. |
| --сервис -s | Название весеннего облака Azure.  Вы можете настроить службу `az configure --defaults spring-cloud=<name>`по умолчанию с помощью. |

| Необязательные параметры | |
| --- | ---: |
| --cpu | Количество виртуальных ядер ИПЦ в каждом экземпляре. |
| --развертывание -d | Название существующего развертывания приложения.  По умолчанию развертывание производства, если не указано. |
| --энв | Пространственно-разделенные переменные среды в формате «ключ»значение». |
| --instance-count | Количество экземпляров. |
| --банк-путь | Если это предусмотрено, развернуть банку из заданного пути. В противном случае разместите текущую папку в виде смолы. |
| --jvm-опции | Строка, содержащая параметры JVM.  Используйте ''' вместо '', чтобы избежать ошибок разбора оболочки. Например, `--jvm-options='-Xms1024m -Xmx2048m`. |
| --memory | Количество ГБ памяти на экземпляр. |
| --no-wait | Не ждите завершения длительных операций. |
| --бег-версия | Запуск версии языка, используемого в приложении.  Разрешенные `Java_11`значения: `Java_8`. |
| --целевой модуль | Детский модуль, который будет развернут.  Требуется, когда несколько пакетов банку построены из исходного кода. |
| --version | Версия развертывания.  Без изменений, если не установлен. |

### <a name="examples"></a>Примеры

Развертывание исходного кода в приложении. Это позволит упаковать текущий каталог, построить двоичный с помощью службы Pivotal Build, а затем развернуть в приложение.

```azurecli
az spring-cloud app deploy -n MyApp -s MyService
```

Развертывание предварительно построенной банки в приложении с использованием вариантов JVM и переменных среды.

```azurecli
az spring-cloud app deploy -n MyApp -s MyService --jar-path app.jar --jvm-options="-XX:+UseG1GC -XX:+UseStringDeduplication" --env foo=bar
```

Развертывание исходного кода для конкретного развертывания приложения.

```azurecli
az spring-cloud app deploy -n MyApp -s Myspring-cloud -d green-deployment
```

## <a name="az-spring-cloud-app-scale"></a>az весна-облако приложение масштаба

Ручно масштабируйте приложение или его развертывание.

```azurecli
az spring-cloud app scale --name -n
                          --resource-group -g
                          --service -s
                          --cpu
                          --deployment -d
                          --instance-count
                          --memory
                          --no-wait
```

| Необходимые параметры | |
| --- | ---: |
| --имя -n | Имя приложения |
| --resource-group -g | Имя группы ресурсов.  Вы можете настроить расположение по умолчанию с помощью `az configure --defaults group=<name>`. |
| --сервис -s | Название весеннего облака Azure.  Вы можете настроить службу `az configure --defaults spring-cloud=<name>`по умолчанию с помощью. |

| Необязательные параметры | |
| --- | ---: |
| --cpu | Количество виртуальных ядер процессора в экземпляре приложения. |
| --развертывание -d | Название существующего развертывания приложения.  По умолчанию развертывание производства, если не указано. |
| --instance-count | Количество экземпляров этого приложения. |
| --memory | Количество ГБ памяти на экземпляр приложения. |
| --no-wait | Не ждите завершения длительных операций. |

### <a name="examples"></a>Примеры

Масштабируйте приложение до 4 ядер процессора и 8 ГБ памяти на экземпляр.

```azurecli
az spring-cloud app scale -n MyApp -s MyService --cpu 3 --memory 8
```

Масштабируйте развертывание приложения до 5 экземпляров.

```azurecli
az spring-cloud app scale -n MyApp -s MyService -d green-deployment --instance-count 5
```

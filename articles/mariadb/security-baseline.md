# <a name="azure-security-baseline-for-azure-database-for-mariadb"></a>Базовый унизел безопасности Azure для базы данных Azure для MariaDB

Базовый унизел безопасности Azure для базы данных Azure для MariaDB содержит рекомендации, которые помогут вам улучшить безопасность развертывания.

Базовая версия этой службы взята из [версии 1.0 Azure Security Benchmark,](https://docs.microsoft.com/azure/security/benchmarks/overview)которая содержит рекомендации о том, как можно обезопасить свои облачные решения в Azure с нашими рекомендациями по передовым практикам.

Для получения дополнительной информации смотрите [обзор базовых линий безопасности Azure](https://docs.microsoft.com/azure/security/benchmarks/security-baselines-overview).

### <a name="11-protect-resources-using-network-security-groups-or-azure-firewall-on-your-virtual-network"></a>1.1: Защита ресурсов, использующих группы сетевой безопасности или брандмауэр Azure в виртуальной сети

**Руководство**: Налаживать частную ссылку для базы данных Azure для MariaDB с частными конечными точками. Приватный канал позволяет подключаться к различным службам PaaS в Azure через частную конечную точку. Azure Private Link по существу приносит услуги Azure в вашей частной виртуальной сети (VNet). Трафик между виртуальной сетью и экземпляром MariaDB перемещается по магистральной сети Майкрософт.

Кроме того, вы можете использовать конечные точки виртуального сетевого обслуживания для защиты и ограничения доступа к базе данных Azure для реализаций MariaDB. Правила виртуальной сети — это одна функция безопасности брандмауэра, которая контролирует, принимает ли база данных Azure для MariaDB сообщения, отправляемые из определенных подсетей в виртуальных сетях.

Вы также можете обеспечить базу данных Azure для MariaDB с помощью правил брандмауэра. Брандмауэр сервера предотвращает доступ к серверу базы данных до тех пор, пока вы не укажете, какие компьютеры имеют разрешение. Для настройки брандмауэра можно создать правила брандмауэра, которые указывают диапазон допустимых IP-адресов. Правила брандмауэра можно создавать на уровне сервера.

Как настроить частную ссылку для базы данных Azure для MariaDB:https://docs.microsoft.com/azure/mariadb/howto-configure-privatelink-portal

Как создавать и управлять конечными точками обслуживания VNet и правилами VNet в базе данных Azure для сервера MariaDB:https://docs.microsoft.com/azure/mariadb/howto-manage-vnet-portal

Как настроить базу данных Azure для правил брандмауэра MariaDB:https://docs.microsoft.com/azure/mariadb/concepts-firewall-rules

**Мониторинг Центра безопасности Azure:** Да

### <a name="19-maintain-standard-security-configurations-for-network-devices"></a>1.9: Поддержание стандартных конфигураций безопасности для сетевых устройств

**Руководство**: Определение и реализация стандартных конфигураций безопасности для настроек сети и сетевых ресурсов, связанных с базой данных Azure для экземпляров MariaDB с Azure Policy. Используйте псевдонимы Azure Policy в пространствах имен "Microsoft.DBforMariaDB" и "Microsoft.Network" для создания пользовательских политик для аудита или обеспечения сетевой конфигурации базы данных Azure для экземпляров MariaDB. - Сервер MariaDB должен использовать виртуальную конечную точку сетевого обслуживания

*Для получения дополнительной информации [см.](https://docs.microsoft.com/azure/security/benchmarks/security-control-data-recovery)*

### <a name="91-ensure-regular-automated-back-ups"></a>9.1: Обеспечить регулярные автоматизированные резервные истектоли

**Руководство**: База данных Azure для MariaDB занимает полное, дифференциальное и резервное копирование журналов транзакций.  В службе "База данных Azure для MariaDB" для сервера автоматически создаются резервные копии, которые сохраняются в локально избыточном или геоизбыточном хранилище, настроенном пользователем. Резервные копии можно использовать для восстановления сервера до точки во времени. Резервное копирование и восстановление данных являются важной частью любой стратегии непрерывности бизнес-процессов. Таким образом данные защищаются от случайного повреждения или удаления.  По умолчанию срок хранения резервных копий составляет 7 дней. При необходимости вы увеличить его вплоть до 35 дней. Все резервные копии шифруются с помощью 256-битового шифрования AES.

Понять резервные копирования для MariaDB:https://docs.microsoft.com/azure/mariadb/concepts-backup

Поймите начальную конфигурацию MariaDB:https://docs.microsoft.com/azure/mariadb/tutorial-design-database-using-portal

**Руководство**: База данных Azure для MariaDB автоматически создает резервные копии серверов и хранит их в локально настроенном локально модном или гео-избыточном хранилище. Резервные копии можно использовать для восстановления сервера до точки во времени.  Резервное копирование и восстановление данных являются важной частью любой стратегии непрерывности бизнес-процессов. Таким образом данные защищаются от случайного повреждения или удаления.

При использовании Key Vault для шифрования данных на стороне клиента для данных, хранящихся на сервере MariaDB, обеспечите регулярное автоматическое резервное копирование ключей.

### <a name="93-validate-all-backups-including-customer-managed-keys"></a>9.3: Проверка всех резервных ups, включая управляемые ключей клиента

**Руководство**: В базе данных Azure для MariaDB выполните восстановление из резервных данных исходного сервера для периодического тестирования резервных ups. Доступны два типа восстановления: восстановление point-in-time и Geo-restore. Восстановление до точки во времени доступно для любого типа избыточности резервного копирования. При таком восстановлении новый сервер создается в том же регионе, где расположен исходный сервер. Географическое восстановление доступно, если для сервера настроено геоизбыточное хранилище, и позволяет восстановить сервер в другом регионе.

Предполагаемое время восстановления будет зависеть от нескольких факторов, включая размер базы данных, размер журнала транзакций, пропускную способность сети и общее количество баз данных, восстанавливаемых в том же регионе и в то же время. Обычно время восстановления составляет менее 12 часов.

Понимание резервного копирования и восстановление в базе данных Azure для MariaDB:https://docs.microsoft.com/azure/mariadb/concepts-backup#restore

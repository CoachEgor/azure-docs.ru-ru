# <a name="prepay-for-azure-database-for-mariadb-compute-resources-with-reserved-capacity"></a>Предоплата для базы данных Azure для вычисляемых ресурсов MariaDB с зарезервированной емкостью

База данных Azure для MariaDB теперь помогает вам сэкономить деньги, предоплату за вычислительные ресурсы по сравнению с ценами с оплатой по мере работы. С помощью зарезервированной емкости Azure для зарезервированной мощности MariaDB вы берете на себя авансовые обязательства на серверах MariaDB в течение одного или трех лет, чтобы получить значительную скидку на вычислительные затраты. Чтобы приобрести базу данных Azure для зарезервированных емкостей MariaDB, необходимо указать область Azure, тип развертывания, уровень производительности и термин. </br>

Вам не нужно присваивать резервирование конкретной базе данных Azure для серверов MariaDB. Уже запущенная база данных Azure для MariaDB или недавно развернутая, автоматически получит выгоду от зарезервированных цен. Приобретая резервирование, вы предварительно оплачиваете стоимость вычислений на один или три года. Как только вы покупаете бронирование, база данных Azure для расчета MariaDB, которая соответствует атрибутам бронирования, больше не взимается по тарифам с оплатой по мере покупки. Бронирование не распространяется на расходы на программное обеспечение, сеть или хранение данных, связанные с сервером базы данных MariaDB. В конце срока бронирования срок действия платежного пособия истекает, а база данных Azure для MariaDB выставляется по цене оплаты по мере того, как вы получаете плату. Резервирования не возобновляются автоматически. Для получения информации [Azure Database for MariaDB reserved capacity offering](https://azure.microsoft.com/pricing/details/mariadb/)о ценах см. </br>

Базу данных Azure для зарезервированной мощности MariaDB можно купить на [портале Azure.](https://portal.azure.com/) Чтобы купить зарезервированную емкость:

* Вы должны быть в роли владельца, по крайней мере один предприятие или индивидуальную подписку с оплатой по мере использования ставок.
* Для подписок с соглашением Enterprise параметр **Добавить зарезервированные экземпляры** следует включить на [портале EA](https://ea.azure.com/). Или, если этот параметр отключен, необходимо быть администратором подписки EA.
* Для программы поставщика облачных решений (CSP) только агенты-админы или агенты по продажам могут приобрести базу данных Azure для зарезервированной емкости MariaDB. </br>

Подробная информация о том, как корпоративные клиенты и клиенты Pay-As-You-Go взимаются за покупки бронирования, см. понимание [использования бронирования Azure для вашего корпоративного регистрации](https://docs.microsoft.com/azure/billing/billing-understand-reserved-instance-usage-ea) и [понимания использования бронирования Azure для подписки Pay-As-You-Go.](https://docs.microsoft.com/azure/billing/billing-understand-reserved-instance-usage)


## <a name="determine-the-right-server-size-before-purchase"></a>Определить правильный размер сервера перед покупкой

Размер резервирования должен основываться на общем количестве вычислений, используемых в существующих или скоро развернутых базах данных в конкретном регионе, и с использованием одного и того же уровня производительности и аппаратного обеспечения.</br>

Например, предположим, что вы работаете с одной общей целью, Gen5 - 32 vCore MariaDB базы данных, и две оптимизированные памяти, Gen5 - 16 vCore MariaDB баз данных. Кроме того, предположим, что вы планируете развернуть в течение следующего месяца дополнительную общую цель, Gen5 - 32 vCore сервер базы данных, и один оптимизированный, Gen5 - 16 vCore сервер базы данных. Предположим, что вы знаете, что вам понадобятся эти ресурсы, по крайней мере 1 год. В этом случае вы должны приобрести 64 (2x32) vCores, 1 год резервирования для одной базы данных общего назначения - Gen5 и 48 (2x16 16) vCore 1 год резервирования для одной базы данных памяти оптимизированы - Gen5


## <a name="buy-azure-database-for-mariadb-reserved-capacity"></a>Купить базу данных Azure для зарезервированных мощностей MariaDB

1. Войдите на [портал Azure](https://portal.azure.com/).
2. Выберите **Все услуги** > **Бронирование**.
3.  Выберите **Добавить,** а затем в панели бронирования покупок, выберите **базу данных Azure для MariaDB,** чтобы приобрести новое бронирование для баз данных MariaDB.
4.  Заполните требуемые поля. Существующие или новые базы данных, которые соответствуют выбранным атрибутам, чтобы получить скидку зарезервированной емкости. Фактическое количество базы данных Azure для серверов MariaDB, которые получают скидку, зависит от выбранной области и количества.


![Обзор зарезервированных цен](media/concepts-reserved-pricing/mariadb-reserved-price.png)


В следующей таблице описаны необходимые поля.

| Поле | Описание |
| :------------ | :------- |
| Подписка   | Подписка, используемая для оплаты базы данных Azure для резервирования зарезервированных емок MariaDB. С способа оплаты по подписке взимается авансовые расходы для базы данных Azure для резервирования зарезервированных емок MariaDB. Тип подписки должен быть корпоративным соглашением (номерами предложений: MS-A'A'-0017P или MS-A'A'-0148P) или индивидуальным соглашением с ценами с оплатой по мере использования (номера предложений: MS-A'R-0003P или MS-A'A'-0023P). Для подписки с соглашением Enterprise плата вычитается из баланса денежных обязательств или относится к избыточным расходам. Для индивидуальной подписки с оплатой по мере использования, сборы выставляются на кредитную карту или метод оплаты счета-фактуры по подписке.
| Область | Область резервирования vCore может охватывать одну подписку или несколько подписок (общий объем). Если выбрать: </br></br> **Общая**скидка на бронирование vCore применяется к базе данных Azure для серверов MariaDB, работающих в любых подписках в контексте выставления счетов. Для клиентов с соглашением Enterprise общая область действует в рамках регистрации и включает в себя все подписки, заданные при регистрации. Для клиентов с подпиской с оплатой по мере использования общая область включает в себя все подписки с оплатой по мере использования, созданные администратором учетной записи.</br></br> **Единая подписка,** скидка на бронирование vCore применяется к базе данных Azure для серверов MariaDB в этой подписке. </br></br> **Единая группа ресурсов,** скидка на бронирование применяется к базе данных Azure для серверов MariaDB в выбранной подписке и выбранной группе ресурсов в рамках этой подписки.
| Регион | Область Azure, охватываемая базой данных Azure для резервирования резервных емок MariaDB.
| Тип развертывания | База данных Azure для типа ресурсов MariaDB, для чего вы хотите купить бронирование.
| Уровень производительности | Уровень обслуживания для базы данных Azure для серверов MariaDB.
| Термин | Один год
| Количество | Объем вычислительных ресурсов, приобретаемых в базе данных Azure для резервирования резервных емок MariaDB. Количество — это количество vCore s в выбранном регионе Azure и уровне производительности, которые зарезервированы и получат скидку на выставление счетов. Например, если вы работаете или планируете запустить базу данных Azure для серверов MariaDB с общей вычислительной мощностью Gen5 16 vCores в восточном регионе США, то вы укажете количество как 16, чтобы максимизировать выгоду для всех серверов.

# <a name="manage-an-azure-database-for-mariadb-server-using-the-azure-portal"></a>Управление базой данных Azure для сервера MariaDB с помощью портала Azure
В этой статье показано, как управлять базой данных Azure для серверов MariaDB. Задачи управления включают вычисление и масштабирование системы хранения данных, сбросить пароли администратора и просматривать данные сервера.

## <a name="sign-in"></a>Вход
Войдите на [портал Azure](https://portal.azure.com).

## <a name="create-a-server"></a>Создание сервера
Посетите [быстрый запуск,](quickstart-create-mariadb-server-database-using-azure-portal.md) чтобы узнать, как создавать и начать работу с базой данных Azure для сервера MariaDB.

## <a name="scale-compute-and-storage"></a>Масштабирование вычислений и хранения

После создания сервера вы можете масштабировать сяраженный между уровнями общей цели и оптимизации памяти по мере изменения потребностей. Вы также можете масштабировать вычисления и память, увеличивая или уменьшая vCores. Хранение может быть масштабировано (однако, вы не можете масштабировать хранилище вниз).

### <a name="scale-between-general-purpose-and-memory-optimized-tiers"></a>Масштаб ирование между уровнями общей цели и оптимизацией памяти

Вы можете масштабироваться от общего назначения до оптимизации памяти и наоборот. Изменение к и из базового уровня после создания сервера не поддерживается. 

1. Выберите сервер на портале Azure. Выберите **уровень ценообразования,** расположенный в разделе **Настройки.**

2. Выберите **общее назначение** или **оптимизированную память,** в зависимости от того, что вы масштабируете. 

    ![изменение ценообразования-уровня](./media/howto-create-manage-server-portal/change-pricing-tier.png)

    > [!NOTE]
    > Изменение ярусов приводит к перезагрузке сервера.

4. Выберите **OK** для сохранения изменений.


### <a name="scale-vcores-up-or-down"></a>Масштаб vCores вверх или вниз

1. Выберите сервер на портале Azure. Выберите **уровень ценообразования,** расположенный в разделе **Настройки.**

2. Измените параметр **Виртуальное ядро**, переместив ползунок на нужное значение.

    ![Масштабирование вычислений](./media/howto-create-manage-server-portal/scaling-compute.png)

    > [!NOTE]
    > Масштабирование vCores приводит к перезагрузке сервера.

3. Выберите **OK** для сохранения изменений.


### <a name="scale-storage-up"></a>Масштабирование хранилища

1. Выберите сервер на портале Azure. Выберите **уровень ценообразования,** расположенный в разделе **Настройки.**

2. Измените настройки **хранилища,** переместив ползунок в нужное значение.

    ![Масштабирование хранилища](./media/howto-create-manage-server-portal/scaling-storage.png)

    > [!NOTE]
    > Хранение не может быть уменьшено.

3. Выберите **OK** для сохранения изменений.


## <a name="update-admin-password"></a>Обновление пароля админ
Изменить пароль роли администратора можно с помощью портала Azure.

1. Выберите сервер на портале Azure. В окне **«Обзор»** выберите **пароль сбросить**.

   ![общие сведения](./media/howto-create-manage-server-portal/overview-reset-password.png)

2. Введите новый пароль и подтвердите его. Текстовый ящик подскажет вам о требованиях сложности паролей.

   ![Сброс пароля](./media/howto-create-manage-server-portal/reset-password.png)

3. Выберите **OK,** чтобы сохранить новый пароль.


## <a name="delete-a-server"></a>Удаление сервера

Вы можете удалить свой сервер, если он вам больше не нужен. 

1. Выберите сервер на портале Azure. В окне **Обзора** выберите **Удалить**.

    ![удалить](./media/howto-create-manage-server-portal/overview-delete.png)

2. Введите имя сервера в поле ввода, чтобы подтвердить, что это сервер, который вы хотите удалить.

    ![Подтверждение удаления](./media/howto-create-manage-server-portal/confirm-delete.png)

    > [!NOTE]
    > Удалять сервер необратимо.

3. Выберите **Удалить**.

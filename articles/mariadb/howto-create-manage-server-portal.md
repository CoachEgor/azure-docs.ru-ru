# <a name="create-and-manage-azure-database-for-mariadb-server-using-azure-portal"></a>Создание сервера базы данных Azure для MariaDB и управление им с помощью портал Azure
В этом разделе описывается, как быстро создать новый сервер базы данных Azure для MariaDB. Он также содержит сведения о том, как управлять сервером с помощью портала Azure. Управление сервером включает в себя просмотр сведений о сервере и базах данных, сброс пароля, масштабирование ресурсов и удаление сервера.

## <a name="log-in-to-the-azure-portal"></a>Войдите на портал Azure.
Войдите на [портал Azure](https://portal.azure.com).

## <a name="create-an-azure-database-for-mariadb-server"></a>Создание сервера Базы данных Azure для MariaDB
Выполните следующие действия, чтобы создать базу данных Azure для сервера MariaDB с именем "mydemoserver".

1. Нажмите кнопку **Создать ресурс** в верхнем левом углу на портале Azure.

2. На странице Создание выберите **базы**данных, а затем на странице базы данных выберите **база данных Azure для MariaDB**.

    > Сервер базы данных Azure для MariaDB создается с определенным набором ресурсов [вычислений и хранилища](./concepts-pricing-tiers.md) . База данных создается в группе ресурсов Azure и на сервере базы данных Azure для MariaDB.

   ![create-new-server](./media/howto-create-manage-server-portal/create-new-server.png)

3. Заполните форму базы данных Azure для MariaDB, используя следующие сведения:

    | **Поле формы** | **Описание поля** |
    |----------------|-----------------------|
    | *Имя сервера* | mydemoserver (глобально уникальное имя сервера) |
    | *Подписка* | mysubscription (выберите из раскрывающегося меню) |
    | *Группа ресурсов* | myresourcegroup (создайте новую группу ресурсов или используйте существующую) |
    | *Выбрать источник* | Пусто (создать пустой сервер MariaDB) |
    | *Имя для входа администратора сервера* | myadmin (укажите имя учетной записи администратора) |
    | *Пароль* | задайте пароль учетной записи администратора |
    | *Подтверждение пароля.* | Подтвердите пароль учетной записи администратора |
    | *Location* | Юго-Восточная Азия (выберите "Северная Европа" или "Западная часть США") |
    | *Версия* | 10,3 (выбор версии сервера базы данных Azure для MariaDB) |

   ![create-new-server](./media/howto-create-manage-server-portal/form-field.png)

4. Нажмите кнопку **настроить сервер** , чтобы указать уровень служб и производительности для нового сервера. Выберите вкладку **Общего назначения**. *Поколение 5*, *4 виртуальных ядра*, *100 ГБ* и *7 дней* — это значения по умолчанию для параметров **Поколение вычислительных ресурсов**, **Виртуальное ядро**, **Хранилище** и **Срок хранения резервных копий**. Эти ползунки можно оставить как есть. Чтобы включить резервное копирование сервера в геоизбыточном хранилище, выберите **Геоизбыточное** в разделе **параметров избыточности резервного копирования**.

   ![create-server-pricing-tier](./media/howto-create-manage-server-portal/create-server-pricing-tier.png)

5. Щелкните **проверить + создать** , чтобы открыть экран Проверка и проверить все сведения. Нажмите кнопку **Создать**, чтобы подготовить сервер. Подготовка занимает несколько минут.

    > Установите флажок **Закрепить на панели мониторинга**, чтобы с легкостью отслеживать процесс развертывания.

## <a name="update-an-azure-database-for-mariadb-server"></a>Обновление сервера базы данных Azure для MariaDB
После подготовки нового сервера пользователь имеет несколько параметров для настройки существующего сервера, включая сброс пароля администратора, изменение ценовой категории и масштабирование сервера путем изменения Виртуальное ядро или хранилища.

### <a name="change-the-administrator-user-password"></a>Изменение пароля администратора
1. В области **Обзор** для сервера нажмите кнопку **сброс пароля** для отображения окна сброса пароля.

   ![обзор](./media/howto-create-manage-server-portal/overview.png)

2. Введите новый пароль и подтвердите его в окне, как показано ниже:

   ![Сброс пароля](./media/howto-create-manage-server-portal/reset-password.png)

3. Нажмите кнопку **ОК** , чтобы сохранить новый пароль.

### <a name="change-the-pricing-tier"></a>Изменение ценовой категории
> [!NOTE]
> Масштабирование поддерживается только из общего назначения в оптимизированные для памяти уровни служб и наоборот. Обратите внимание, что переход на базовую ценовую категорию и из нее после создания сервера не поддерживается в базе данных Azure для MariaDB.
> 
1. Щелкните элемент **Ценовая категория**, расположенный в области **Параметры**.
2. Выберите **ценовую категорию** , на которую вы хотите перейти.

    ![изменение ценовой категории](./media/howto-create-manage-server-portal/change-pricing-tier.png)

4. Нажмите кнопку **OK**, чтобы сохранить изменения. 

### <a name="scale-vcores-updown"></a>Увеличение или уменьшение масштаба виртуальных ядер

1. Щелкните элемент **Ценовая категория**, расположенный в области **Параметры**.

2. Измените параметр **Виртуальное ядро**, переместив ползунок на нужное значение.

    ![Масштабирование вычислений](./media/howto-create-manage-server-portal/scale-compute.png)

3. Нажмите кнопку **OK**, чтобы сохранить изменения.

### <a name="scale-storage-up"></a>Увеличение масштаба хранилища

1. Щелкните элемент **Ценовая категория**, расположенный в области **Параметры**.

2. Измените параметр **Хранилище**, переместив ползунок на нужное значение.

    ![Масштабирование хранилища](./media/howto-create-manage-server-portal/scale-storage.png)

3. Нажмите кнопку **OK**, чтобы сохранить изменения.

## <a name="delete-an-azure-database-for-mariadb-server"></a>Удаление сервера базы данных Azure для MariaDB

1. В области **Обзор** сервера нажмите кнопку **Удалить**, чтобы открыть запрос на подтверждение удаления.

    ![delete](./media/howto-create-manage-server-portal/delete.png)

2. Введите имя сервера в поле ввода для дополнительного подтверждения.

    ![Подтверждение удаления](./media/howto-create-manage-server-portal/confirm.png)

3. Нажмите кнопку **Удалить**, чтобы подтвердить удаление сервера. Дождитесь появления всплывающего окна "успешно удаленные MariaDB Server" на панели уведомлений.

## <a name="list-the-azure-database-for-mariadb-databases"></a>Вывод списка баз данных Azure для MariaDB
Прокрутите область **Обзор** сервера вниз до появления плитки базы данных, расположенной внизу. Все базы данных на сервере отображаются в таблице.

   ![show-databases](./media/howto-create-manage-server-portal/show-databases.png)

## <a name="show-details-of-an-azure-database-for-mariadb-server"></a>Отображение сведений о сервере базы данных Azure для MariaDB
Щелкните элемент **Свойства**, расположенный в области **Параметры**, для просмотра подробных сведений о сервере.

![свойства](./media/howto-create-manage-server-portal/properties.png)

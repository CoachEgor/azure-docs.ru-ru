# <a name="security-in-azure-database-for-mariadb"></a>Безопасность в Базе данных Azure для MariaDB

Существует несколько уровней безопасности, которые доступны для защиты данных в базе данных Azure для сервера MariaDB. В этой статье излагаются эти параметры безопасности.

## <a name="information-protection-and-encryption"></a>Защита и шифрование информации

### <a name="in-transit"></a>В пути
База данных Azure для MariaDB защищает ваши данные, шифруя данные в пути с помощью Transport Layer Security. Шифрование (SSL/TLS) осуществляется по умолчанию.

### <a name="at-rest"></a>На отдыхе
В Базе данных Azure для MariaDB используется криптографический модуль, соответствующий стандарту FIPS 140-2, для шифрования неактивных данных. Данные, включая резервные, шифруются на диске, за исключением временных файлов, созданных во время выполнения запросов. Служба использует 256-разрядный шифр AES, включенный в шифрование службы хранилища Azure. Ключами управляет система. Шифрование хранилища всегда включено, и его нельзя отключить.


## <a name="network-security"></a>Безопасность сети
Сначала подключение к базе данных Azure для сервера MariaDB направляется через региональный шлюз. Шлюз имеет общедоступный IP-адрес, в то время как IP-адреса сервера защищены. Для получения дополнительной информации о шлюзе посетите [статью архитектуры подключения](concepts-connectivity-architecture.md).  

Недавно созданная база данных Azure для сервера MariaDB имеет брандмауэр, который блокирует все внешние соединения. Хотя они достигают шлюза, они не имеют права подключаться к серверу. 

### <a name="ip-firewall-rules"></a>Правила брандмауэра для IP-адресов
Правила IP-брандмауэра предоставляют доступ к серверам на основе исходного IP-адреса каждого запроса. Для получения дополнительной информации смотрите [обзор правил брандмауэра.](concepts-firewall-rules.md)

### <a name="virtual-network-firewall-rules"></a>Правила брандмауэра для виртуальной сети
Конечные точки виртуального сетевого обслуживания расширяют виртуальное сетевое подключение по сравнению с магистралью Azure. Используя правила виртуальной сети, вы можете включить базу данных Azure для сервера MariaDB, чтобы разрешить соединения из выбранных подсетей в виртуальной сети. Для получения дополнительной информации смотрите [обзор конечных точек виртуального сетевого обслуживания.](concepts-data-access-security-vnet.md)


## <a name="access-management"></a>Управление доступом

При создании базы данных Azure для сервера MariaDB вы предоставляете учетные данные для пользователя администратора. Этот администратор может быть использован для создания дополнительных пользователей MariaDB.


## <a name="threat-protection"></a>Защита от угроз

Вы можете выбрать для [расширенной защиты от угроз,](concepts-data-access-and-security-threat-protection.md) которая обнаруживает аномальные действия, указывающие на необычные и потенциально опасные попытки доступа или эксплуатировать серверы.

[Регистрация аудита](concepts-audit-logs.md) доступна для отслеживания активности в базах данных.

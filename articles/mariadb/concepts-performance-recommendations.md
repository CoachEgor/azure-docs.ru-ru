# <a name="performance-recommendations-in-azure-database-for-mariadb"></a>Рекомендации по повышению производительности в Базе данных Azure для MariaDB

**Применяется к:** База данных Azure для MariaDB 10.2

Функция «Рекомендации по производительности» анализирует базы данных для создания индивидуальных предложений для повышения производительности. Для подготовки рекомендаций в анализе рассматриваются различные характеристики базы данных, включая схему. Включите [магазин запросов](concepts-query-store.md) на сервере, чтобы в полной мере использовать функцию Рекомендации производительности. Если схема производительности OFF, включение Магазина запросов позволяет performance_schema и подмножество инструментов схемы производительности, необходимых для функции. После реализации какой-либо рекомендации по производительности следует проверить производительность для оценки влияния этих изменений.

## <a name="permissions"></a>Разрешения

Для выполнения анализа с помощью функции рекомендаций по повышению производительности требуются разрешения **владельца** или **участника**.

## <a name="performance-recommendations"></a>Рекомендации по производительности

Функция [рекомендаций по повышению производительности](concepts-performance-recommendations.md) анализирует рабочие нагрузки на сервере и определяет, какие индексы могут помочь в повышении производительности.

Рекомендации **по производительности** от раздела **Интеллектуальная производительность** панели меню на странице портала Azure для вашего сервера MariaDB.

![Целевая страница рекомендаций по повышению производительности](./media/concepts-performance-recommendations/performance-recommendations-page.png)

Выберите **анализ** и выберите базу данных, которая начнет анализ. В зависимости от рабочей нагрузки анализ может занять несколько минут. По окончании анализа на портале появится уведомление. Анализ выполняет глубокое изучение вашей базы данных. Мы рекомендуем вам проводить анализ во время внепиковых периодов.

В окне **Рекомендаций** будет отображаться список рекомендаций, если таковые были найдены, и связанный с этим идентификатор запроса, который сгенерировал эту рекомендацию. С идентификатором запроса можно использовать представление [mysql.query_store,](concepts-query-store.md#mysqlquery_store) чтобы узнать больше о запросе.

![Новая страница рекомендаций по производительности](./media/concepts-performance-recommendations/performance-recommendations-result.png)

Рекомендации не применяются автоматически. Чтобы применить рекомендацию, скопируйте текст запроса и запустите его от клиента по выбору. Не забудьте проверить и контролировать, чтобы оценить рекомендацию.

## <a name="recommendation-types"></a>Типы рекомендаций

В настоящее время поддерживаются только рекомендации *Create Index.*

### <a name="create-index-recommendations"></a>Рекомендации по созданию индексов

*Рекомендации по созданию индекса* предлагают новые индексы для ускорения наиболее часто заражаемых или трудоемких запросов в рабочей нагрузке. Этот тип рекомендаций требует включения [Магазина запросов.](concepts-query-store.md) Магазин запросов собирает информацию о запросах и предоставляет подробную статистику выполнения запросов и частоты, которую анализ использует для выполнения рекомендации.

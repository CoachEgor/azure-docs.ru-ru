# <a name="upcoming-deprecation-of-dr-between-customer-owned-sites-using-hyper-v-and-between-sites-managed-by-scvmm-to-azure"></a>Предстоящие нерекомендуемые средства аварийного восстановления между клиентскими сайтами с помощью Hyper-V и между сайтами, управляемыми SCVMM, в Azure

В этой статье описывается предстоящий план устаревания, соответствующие последствия и альтернативные варианты, доступные для клиентов в следующих сценариях, которые поддерживались Azure Site Recovery. 

- Аварийное восстановление между виртуальными машинами Hyper-V, управляемыми SCVMM между сайтами, принадлежащими заказчику 
- Аварийное восстановление виртуальных машин Hyper-V под управлением SCVMM в Azure

> [!IMPORTANT]
> В настоящее время эти сценарии отмечены для упрощения, и клиенты должны предпринять действия по исправлению с самого раннего, чтобы избежать сбоев в их среде. 
 

## <a name="what-changes-should-you-expect"></a>Какие изменения следует рассчитывать?

- Начиная с 2019 ноября вы не сможете использовать новые пользовательские платы для этих сценариев. **Существующие операции репликации и управления** , включая отработку отказа, тестовую отработку отказа, мониторинг и т. д., **не будут затронуты**.

- После того как сценарии устарели, они будут иметь следующие последствия, если клиент не будет следовать рекомендациям.

    - Аварийное восстановление между виртуальными машинами Hyper-V, управляемыми SCVMM, между сайтами, принадлежащими заказчику. репликация будет продолжать работать, так как базовая возможность реплики Hyper-V будет продолжать работать, но клиенты не смогут просматривать, управлять и выполнять операции, связанные с аварийным восстановлением. с помощью Azure Сире Recovery в портал Azure. 
    - Аварийное восстановление виртуальных машин Hyper-V, управляемых SCVMM, в Azure. существующие репликации будут нарушены, и клиенты не смогут просматривать, управлять или выполнять операции, связанные с аварийным восстановлением, с помощью Azure Site Recovery


## <a name="recommended-actions-to-be-taken"></a>Рекомендуемые действия

Ниже приведены альтернативные варианты, которые клиент должен гарантировать, чтобы Стратегия аварийного восстановления не влияла, когда сценарий считается устаревшим. 

- Выберите, чтобы [начать использовать Azure в качестве цели аварийного восстановления для виртуальных машин на узлах Hyper-V](hyper-v-azure-tutorial.md).

> [!IMPORTANT]
> Обратите внимание, что ваша локальная среда по-прежнему может иметь СКВМММ, но вы настроите ASR со ссылками только на узлы Hyper-V.

- Выберите продолжение репликации "сеть — сеть", но используя базовое [решение реплики Hyper-V](https://docs.microsoft.com/windows-server/virtualization/hyper-v/manage/set-up-hyper-v-replica), но не сможет управлять КОНФИГУРАЦИЯми аварийного восстановления с помощью Azure Site Recovery в портал Azure.

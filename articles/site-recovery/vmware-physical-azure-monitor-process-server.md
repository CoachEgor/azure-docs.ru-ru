# <a name="monitor-the-process-server"></a>Мониторинг сервера обработки

В этой статье описывается, как контролировать процесс [восстановления сайта.](site-recovery-overview.md)

- Сервер процесса используется при настройке аварийного восстановления внутренних VM VMw и физических серверов в Azure.
- По умолчанию сервер процесса работает на сервере конфигурации. Он устанавливается по умолчанию при развертывании сервера конфигурации.
- В качестве опции для масштабирования и обработки большего количества реплицированных машин и больших объемов трафика репликации можно развернуть дополнительные серверы масштабирования.

[Подробнее](vmware-physical-azure-config-process-server-overview.md) о роли и развертывании серверов процессов.

## <a name="monitoring-overview"></a>Общие сведения о мониторинге

Поскольку сервер процесса имеет так много ролей, особенно в реплицировании кэширования данных, сжатии и передаче в Azure, важно постоянно контролировать работоспособность сервера процесса.

Существует ряд ситуаций, которые обычно влияют на производительность сервера процесса. Проблемы, влияющие на производительность, будут иметь каскадное воздействие на работу VM, в конечном итоге подталкивая как сервер процесса, так и его реплицированные машины в критическое состояние. Ситуации включают в себя:

- Большое количество вм-случаев используют сервер процесса, приближаясь или превышая рекомендуемые ограничения.
- VMs, использующие процесссервер, имеют высокую скорость оттока.
- Пропускная память сети между VMs и сервером процесса недостаточна для загрузки данных репликации на сервер процесса.
- Пропускная часть сети между сервером процесса и Azure недостаточна для загрузки данных репликации с сервера процесса в Azure.

Все эти проблемы могут повлиять на цель точки восстановления (RPO) VMs. 

**Почему?** Потому что генерация точки восстановления для VM требует, чтобы все диски на VM имели общую точку. Если один диск имеет высокую скорость оттока, репликация идет медленно, или сервер процесса не является оптимальным, это влияет на то, насколько эффективно создаются точки восстановления.

## <a name="monitor-proactively"></a>Упреждающий мониторинг

Чтобы избежать проблем с сервером процесса, важно:

- Понимание конкретных требований к серверам процессов, использующих [рекомендации по емкости и размеру,](site-recovery-plan-capacity-vmware.md#capacity-considerations)и убедитесь, что серверы процессов развернуты и работают в соответствии с рекомендациями.
- Мониторинг оповещений и устранения неполадок по мере их возникновения, чтобы эффективно отслеживать работу серверов обработки.

## <a name="monitor-process-server-health"></a>Мониторинг работоспособности сервера процесса

Вы можете следить за состоянием работоспособности серверов процесса следующим образом: 

1. Чтобы контролировать работоспособность репликации и состояние реплицируемой машины, а также ее сервера процесса, в хранилище > **реплицированных элементов,** нажмите на машину, которую вы хотите контролировать.
2. В **Replication Health**вы можете следить за состоянием здоровья VM. Нажмите на состояние, чтобы просверлить сведения об ошибках.

    ![Обработка работоспособности сервера в панели мониторинга VM](./media/vmware-physical-azure-monitor-process-server/vm-ps-health.png)

4. В **Process Server Health**можно следить за состоянием сервера процесса. Просверлите вниз для деталей.

    ![Детали сервера обработки в панели мониторинга VM](./media/vmware-physical-azure-monitor-process-server/ps-summary.png)

5. Здоровье также можно контролировать с помощью графического представления на странице VM.
    - Сервер процесса масштабирования будет выделен оранжевым цветом, если с ним есть предупреждения, и красным цветом, если у него есть критические проблемы. 
    - Если сервер процесса работает в развертывании по умолчанию на сервере конфигурации, то сервер конфигурации будет выделен соответствующим образом.
    - Чтобы сверлить вниз, нажмите на сервер конфигурации или процесс сервера. Обратите внимание на любые проблемы и любые рекомендации по исправлению положения.

Вы также можете контролировать серверы процессов в хранилище под **инфраструктурой восстановления сайта.** В **управлении инфраструктурой восстановления сайта**нажмите **На серверы конфигурации.** Выберите сервер конфигурации, связанный с сервером процесса, и просверлите детали сервера процесса.

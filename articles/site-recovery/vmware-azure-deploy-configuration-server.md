# <a name="deploy-a-configuration-server"></a>Развертывание сервера конфигурации.

При использовании [Azure Site Recovery](site-recovery-overview.md) для аварийного восстановления виртуальных машин VMware и физических серверов в Azure развертывается локальный сервер конфигурации. Сервер конфигурации используется для управления обменом данными между локальной средой VMware и Azure. Он также управляет репликацией данных. В этой статье рассматриваются действия, необходимые для развертывания сервера конфигурации при репликации виртуальных машин VMware в Azure. Если вам нужно настроить сервер конфигурации для физического репликации сервера, [см.](physical-azure-set-up-source.md)

> [!TIP]
> Чтобы узнать о роли сервера конфигурации в архитектуре [VMware to Azure disaster recovery architecture](vmware-azure-architecture.md)восстановления сайтов Azure, см.

## <a name="deploy-a-configuration-server-through-an-ova-template"></a>Развертывание сервера конфигурации через шаблон OVA

Сервер конфигурации должен быть настроен как высокодоступный VMware VM с определенными минимальными требованиями к оборудованию и размеру. Для удобного и простого развертывания Site Recovery предоставляет загружаемый шаблон Open Virtualization Application (OVA) для настройки сервера конфигурации, который соответствует всем перечисленным здесь требованиям.

    - В соответствии с **регистрацией** **приложений, пользователи могут зарегистрировать приложения,** выберите **Да**.

## <a name="download-the-template"></a>Скачивание шаблона

1. В хранилище, перейдите к **подготовке инфраструктуры** > **источник**.
2. В колонке **Подготовка источника** выберите **+ Сервер конфигурации**.
3. В колонке **Add Server** (Добавление сервера) в поле **Server type** (Тип сервера) должно быть указано **Configuration server for VMware** (Сервер конфигурации для VMware).
4. Скачать шаблон OVA для сервера конфигурации.

   > [!TIP]
   >Вы также можете скачать последнюю версию шаблона сервера конфигурации непосредственно из [Центра загрузки Microsoft.](https://aka.ms/asrconfigurationserver)

> [!NOTE]
> Лицензия с шаблоном OVA является лицензией оценки, которая действительна в течение 180 дней. По истечении этого срока вы должны приобрести лицензию.

## <a name="import-the-template-in-vmware"></a>Импорт шаблона в VMware

1. Подключитесь к серверу VMware vCenter или узлу vSphere ESXi с помощью клиента VMWare vSphere.
2. В меню **файла** выберите **шаблон Deploy OVF,** чтобы запустить мастер **шаблона Deploy OVF.**

     ![Развертывание шаблона OVF](./media/vmware-azure-deploy-configuration-server/vcenter-wizard.png)

3. Укажите расположение скачанного шаблона OVF в разделе **Выбрать источник**.
4. В разделе **Просмотр сведений** выберите **Далее**.
5. Примите значения параметров по умолчанию в разделах **Select name and folder** (Выбор имени и папки) и **Select configuration** (Выбор конфигурации).
6. В разделе **Select storage** (Выбор хранилища), чтобы улучшить производительность, выберите **Thick Provision Eager Zeroed** (Быстрое обнуление плотной подготовки) в области **Select virtual disk format** (Выбор формата виртуального диска). Использование параметра тонкого подготовки может повлиять на производительность сервера конфигурации.
7. На остальных страницах мастера примите значение параметров по умолчанию.
8. На странице **Ready to complete** (Все готово к выполнению) сделайте следующее:

    * Чтобы настроить VM с настройками по умолчанию, выберите **power on после завершения развертывания.** > **Finish**
    * Чтобы добавить дополнительный сетевой интерфейс, снимите флажок **Power on after deployment** (Включить после развертывания), а затем щелкните **Finish** (Готово). По умолчанию шаблон сервера конфигурации развертывается с одним сетевым адаптером. Вы можете добавить дополнительные сетевые адаптеры после развертывания.

> [!IMPORTANT]
> Не изменяйте конфигурации ресурсов, такие как память, ядра и ограничения процессора, или изменяйте или удаляйте установленные службы или файлы на сервере конфигурации после развертывания. Эти типы изменений влияют на регистрацию сервера конфигурации службами Azure и производительность сервера конфигурации.

## <a name="add-an-additional-adapter"></a>Добавление дополнительного адаптера

> [!NOTE]
> Два NICs необходимы, если вы планируете сохранить IP-адреса исходных машин на failover и хотите потерпеть неудачу обратно в на месте позже. Один NIC подключен к исходным машинам, а другой NIC используется для подключения Azure.

Если вы хотите добавить дополнительный сетевой адаптер к серверу конфигурации, добавьте его до регистрации сервера в хранилище. Возможность добавления дополнительных адаптеров не поддерживается после регистрации.

1. В списке клиента vSphere щелкните виртуальную машину правой кнопкой мыши и выберите **Edit Settings** (Изменить параметры).
2. В разделе **Hardware** (Оборудование) выберите **Add** > **Ethernet Adapter** (Добавить > Адаптер Ethernet). Затем выберите **Следующий**.
3. Выберите тип адаптера и сеть.
4. Чтобы подключить виртуальный сетевой адаптер на включенной виртуальной машине, выберите **Connect at power on** (Подключиться, когда включено). Затем выберите **Следующая** > **отделка** > **OK**.

## <a name="register-the-configuration-server-with-azure-site-recovery-services"></a>Регистрация сервера конфигурации с помощью служб Azure Site Recovery

1. Из консоли клиента VMWare vSphere включите виртуальную машину.
2. Виртуальная машина загружается в среду установки Windows Server 2016. Примите лицензионное соглашение и введите пароль администратора.
3. После установки войдите в виртуальную машину от имени администратора.
4. При первом входе в систему в течение нескольких секунд запускается инструмент восстановления azure Site.
5. Введите имя, которое использовалось для регистрации сервера конфигурации в службе Site Recovery. Затем выберите **Следующий**.
6. Средство проверяет, может ли виртуальная машина подключиться к Azure. После установки подключения выберите **Sign in** (Вход), чтобы войти в подписку Azure.</br>
    а. Учетные данные должны иметь доступ к хранилищу, в котором вы хотите зарегистрировать сервер конфигурации.</br>
    b. Убедитесь, что выбранная учетная запись пользователя имеет разрешение на создание приложения в Azure. Для включения необходимых разрешений следуйте инструкциям в разделе [Azure Active Directory permission.](#azure-active-directory-permission-requirements)
7. Средство выполняет некоторые задачи конфигурации, а затем перезагружается.
8. Снова войдите в систему. Мастер управления сервером конфигурации запускается автоматически в течение нескольких секунд.

### <a name="configure-settings"></a>Настройка параметров

1. В мастере управления сервером конфигурации выберите **Setup connectivity** (Настройка подключения). Из выпадающих коробок сначала выберите NIC, который встроенный сервер процесса использует для обнаружения и установки службы мобильности на исходных машинах. Затем выберите NIC, который сервер конфигурации использует для подключения к Azure. Нажмите кнопку **Сохранить**. Этот параметр уже нельзя изменить после настройки. Не изменяйте IP-адрес сервера конфигурации. Убедитесь, что IP, назначенный серверу конфигурации, является статическим IP, а не IP-адресом DHCP.
2. В **хранилище «Выберите службы восстановления»** войдите в Microsoft Azure с учетными данными, используемыми в шаге 6 [Регистрации сервера конфигурации с службами восстановления сайта Azure.](#register-the-configuration-server-with-azure-site-recovery-services)
3. После ввода в систему выберите подписку Azure и соответствующую группу ресурсов и хранилище.

    > [!NOTE]
    > После регистрации нет гибкости для изменения хранилища служб восстановления.
    > Изменение хранилища служб восстановления требует отмежевания сервера конфигурации от текущего хранилища, и репликация всех защищенных виртуальных машин под сервером конфигурации остановлена. Чтобы узнать больше, смотрите [Управление сервером конфигурации для аварийного восстановления VMware VM.](vmware-azure-manage-configuration-server.md#register-a-configuration-server-with-a-different-vault)

4. На **установке стороннего программного обеспечения:**

    |Сценарий   |Инструкции  |
    |---------|---------|
    |Могу ли я скачать и установить MyS'L вручную?     |  Да. Загрузите приложение MyS'L, поместите его в папку **C: «Темп»ASRSetup**, а затем установите вручную. После того как вы принимаете условия и выбрать **Скачать и установить**, портал говорит *уже установлен*. Можно перейти к следующему шагу.       |
    |Можно ли избежать скачивания MySQL через Интернет?     |   Да. Поместите установщик MySQL в папку **C:\Temp\ASRSetup**. Примите условия, выберите **Скачать и установить**, и портал использует установщик вы добавили для установки приложения. После завершения монтажа приступайте к следующему шагу.    |
    |Я хочу загрузить и установить MyS'L через восстановление сайта Azure.    |  Примите лицензионное соглашение, и выберите **Скачать и установить**. После завершения монтажа приступайте к следующему шагу.       |

5. При **конфигурации прибора Validate,** предпосылки проверяются перед продолжением.
6. На **сервере TCenter Server/vSphere ESXi**введите F-DN или IP-адрес сервера vCenter или хостеаv vSphere, где находятся vMs-файлы, которые вы хотите воспроизвести. Введите порт, от которого ожидается передача данных на сервер. Введите понятное имя для сервера VMware в хранилище.
7. Введите учетные данные, которые будут использоваться сервером конфигурации для подключения к серверу VMware. Site Recovery использует эти учетные данные для автоматического обнаружения виртуальных машин VMware, доступных для репликации. Выберите **Добавить** > **Продолжить**. Введенные здесь учетные данные сохраняются локально.
8. На **настройке виртуальных учетных данных машины**, введите имя пользователя и пароль виртуальных машин, чтобы автоматически установить услугу мобильности во время репликации. Для компьютеров **Windows** учетной записи должны быть предоставлены права локального администратора на компьютерах, которые требуется реплицировать. Для **Linux** предоставьте сведения для учетной записи root.
9. Выберите **Finalize configuration** (Завершение конфигурации) для завершения регистрации.
10. После завершения регистрации откройте портал Azure и убедитесь, что сервер конфигурации и сервер VMware перечислены на**серверах конфигурации** **Recovery Services Vault** > **Manage** > **Site Recovery Infrastructure** > Configuration Servers.

## <a name="faqs"></a>Часто задаваемые вопросы

* Как долго предоставляется лицензия на сервере конфигурации, развернутом через OVF? Что произойдет, если я не активирую лицензию повторно?

    Лицензия, предоставленная шаблоном OVA, является лицензией оценки, действительной в течение 180 дней. До истечения срока действия необходимо активировать лицензию. В противном случае это может привести к частому отключению сервера конфигурации и помехе для действий репликации. Для получения дополнительной информации [см. Управление сервером конфигурации для аварийного восстановления VMware VM.](vmware-azure-manage-configuration-server.md#update-windows-license)

* Могу ли я использовать VM, где установлен сервер конфигурации для различных целей?

    Нет. Используйте VM с единственной целью сервера конфигурации. Убедитесь, что вы будете следовать всем спецификациям, указанным в [Предпосылках](#prerequisites) для эффективного управления аварийно-спасательным процессом.
* Можно ли сменить хранилище, уже зарегистрированное на сервере конфигурации, на новое хранилище?

    Нет. После того, как хранилище зарегистрировано на сервере конфигурации, его нельзя изменить.
* Могу ли я использовать один и тот же сервер конфигурации для защиты как физических, так и виртуальных машин?

    Да. Тот же сервер конфигурации может быть использован для репликации физических и виртуальных машин. Тем не менее, физическая машина может быть невозвращена только к VMware VM.
* Какова цель сервера конфигурации и где он используется?

    Чтобы узнать больше о сервере конфигурации и его функциональных возможностях, [см.](vmware-azure-architecture.md)
* Где я могу найти последнюю версию сервера конфигурации?

    Ознакомьтесь с инструкциями по [обновлению сервера конфигурации на портале](vmware-azure-manage-configuration-server.md#upgrade-the-configuration-server). Для получения инструкций о том, как обновить все компоненты восстановления сайта, [см.](https://aka.ms/asr_how_to_upgrade)
* Где можно загрузить парольную фразу для сервера конфигурации?

    Чтобы загрузить пароль, [см. Управление сервером конфигурации для аварийного восстановления VMware VM.](vmware-azure-manage-configuration-server.md#generate-configuration-server-passphrase)
* Могу ли я изменить парольную фразу?

    Нет. Не изменяйте пароль сервера конфигурации. Изменение фразы прерывает репликацию защищенных машин и приводит к критическому состоянию работоспособности.
* Где можно загрузить ключи регистрации хранилища?

    В **Хранилище служб восстановления**выберите **Серверы** > **конфигурации инфраструктуры** > **восстановления сайта.** В списке **Серверы** выберите **Скачать ключ регистрации**, чтобы загрузить файл учетных данных хранилища.
* Могу ли я клонировать существующий сервер конфигурации и использовать его для оркестровки репликации?

    Нет. Использование компонента сервера клонированной конфигурации не поддерживается. Клонирование сервера процесса масштабирования также является неподдерживаемым сценарием. Компоненты клонирования сайта восстановления влияет на текущие репликации.

* Могу ли я изменить IP-адрес сервера конфигурации?

    Нет. Не изменяйте IP-адрес сервера конфигурации. Убедитесь, что все ипки, назначенные серверу конфигурации, являются статическими ИП, а не ИПКами DHCP.
* Можно ли настроить сервер конфигурации в Azure?

    Настройка сервера конфигурации в предварительной среде с прямой прямой линией с v-Center и минимизация пропотеченных передачи данных. Резервные копии сервера конфигурации можно создавать по расписанию для [восстановления размещения](vmware-azure-manage-configuration-server.md#failback-requirements).

* Могу ли я изменить драйвер кэша на сервере конфигурации или сервере процесса масштабирования?

    Нет, драйвер кэша не может быть изменен после завершения настройки.

Для получения дополнительной часто задаваемых вопросов на серверах конфигурации [см.](vmware-azure-common-questions.md#configuration-server)

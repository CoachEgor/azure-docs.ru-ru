# <a name="set-up-disaster-recovery-for-azure-vms-after-migration-to-azure"></a>Настройка аварийного восстановления виртуальных машин Azure после миграции в Azure 


Следуйте этой статье, если вы [перенесли на емвице-сообщения Azure с](tutorial-migrate-on-premises-to-azure.md) помощью службы [восстановления сайта,](site-recovery-overview.md) и теперь необходимо настроить вневые технологии для аварийного восстановления в второстепенном регионе Azure. В статье описывается, как обеспечить установку агента Azure VM на мигрированных ВМ, а также как удалить службу восстановления сайта, которая больше не нужна после миграции.



## <a name="verify-migration"></a>Проверка миграции

Прежде чем настраивать аварийное восстановление, убедитесь, что миграция завершена правильно. Для успешного завершения миграции после отработки отказа следует выбрать параметр **Завершение миграции** для каждой виртуальной машины, которую требуется перенести. 

## <a name="verify-the-azure-vm-agent"></a>Проверить агент Azure VM

Каждый VM Azure должен иметь установлен [агент Azure VM.](../virtual-machines/extensions/agent-windows.md) Чтобы воспроизвести VMs Azure, восстановление сайта устанавливает расширение на агенте.

- Если машина работает в версии 9.7.0.0 или позже службы восстановления сайта, агент Azure VM автоматически устанавливается службой мобильности на Windows VMs. На более ранних версиях службы Mobility вы установите агент вручную.
- Для Linux VMs необходимо установить vM-агент Azure вручную. Агент виртуальной машины Azure необходимо устанавливать вручную, только если служба Mobility Service, установленная на перенесенном компьютере, имеет версию 9.6 или более раннюю.


### <a name="install-the-agent-on-windows-vms"></a>Установка агента на Windows VMs

Если вы работаете с версией службы восстановления сайта раньше 9.7.0.0, или у вас есть другая необходимость установить агент вручную, сделайте следующее:  

1. Убедитесь, что у вас есть разрешения на админ на VM.
2. Скачать [vM Агент установки](https://go.microsoft.com/fwlink/?LinkID=394789&clcid=0x409).
3. Выполнить файл установки.

#### <a name="validate-the-installation"></a>проверка установки
Чтобы проверить, установлен ли агент:

1. На виртуальной машине Azure в папке C:\WindowsAzure\Packages должен находиться файл WaAppAgent.exe.
2. Щелкните этот файл правой кнопкой мыши, выберите пункт **Свойства** и перейдите на вкладку **Подробно**.
3. В поле **Версия продукта** должно отображаться значение 2.6.1198.718 или выше.

[Подробнее](https://docs.microsoft.com/azure/virtual-machines/extensions/agent-windows) об установке агента для Windows.

### <a name="install-the-agent-on-linux-vms"></a>Установка агента на Linux VMs

Установите [VM-агент Azure Linux](../virtual-machines/extensions/agent-linux.md) вручную следующим образом:

1. Убедитесь, что у вас есть разрешения на админ на машине.
2. Мы настоятельно рекомендуем установить агент Linux VM с помощью RPM или пакета DEB из репозитория пакета дистрибутива. Все [поставщики поддерживаемых дистрибутивов](https://docs.microsoft.com/azure/virtual-machines/linux/endorsed-distros) встраивают пакет агента Linux для Azure в свои образы и репозитории.
    - Мы настоятельно рекомендуем обновлять агент только посредством репозитория дистрибутивов.
    - Мы не рекомендуем устанавливать агент Linux VM непосредственно с GitHub и обновлять его.
    -  Если последняя версия агента для вашего дистрибутива недоступна, обратитесь в службу поддержки дистрибутива, чтобы получить инструкции по установке последней версии агента. 

#### <a name="validate-the-installation"></a>проверка установки 

1. Запустите эту команду: **ps -e,** чтобы убедиться, что агент Azure работает на Linux VM.
2. Если нужный процесс не запущен, выполните следующие команды для его перезапуска:
    - Для Ubuntu: **сервис walinuxagent начала**
    - Для других дистрибутивов: **запуск службы waagent**


## <a name="uninstall-the-mobility-service"></a>Удаление службы Mobility Service

1. Вручную удалить службу мобильности из Azure VM, используя один из следующих методов. 
    - Windows. На панели управления выберите **Установка и удаление программ** и удалите компонент **Microsoft Azure Site Recovery Mobility Service/Главный целевой сервер**. В командной строке с повышенными привилегиями выполните следующую команду:
        ```
        MsiExec.exe /qn /x {275197FC-14FD-4560-A5EB-38217F80CBD1} /L+*V "C:\ProgramData\ASRSetupLogs\UnifiedAgentMSIUninstall.log"
        ```
    - Linux: войдите как привилегированный пользователь. В окне терминала перейдите в расположение **/user/local/ASR** и выполните следующую команду:
        ```
        ./uninstall.sh -Y
        ```
2. Перед настройкой репликации перезапустите виртуальную машину.

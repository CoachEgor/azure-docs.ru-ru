# <a name="set-up-disaster-recovery-for-azure-vms-after-migration-to-azure"></a>Настройка аварийного восстановления виртуальных машин Azure после миграции в Azure 


Выполните в этой статье, если вы уже [миграции локальных компьютеров к виртуальным машинам Azure](tutorial-migrate-on-premises-to-azure.md) с помощью [Site Recovery](site-recovery-overview.md) службы и теперь требуется получить виртуальные машины для аварийного восстановления в дополнительный регион Azure. В статье описывается, как убедиться, что агент виртуальной Машины Azure установлен на перенесенных виртуальных машинах, а также способ удаления службы мобильности Site Recovery, который имеет больше не нужны после миграции.



## <a name="verify-migration"></a>Проверка миграции

Прежде чем настраивать аварийное восстановление, убедитесь, что миграция завершена правильно. Для успешного завершения миграции после отработки отказа следует выбрать параметр **Завершение миграции** для каждой виртуальной машины, которую требуется перенести. 

## <a name="verify-the-azure-vm-agent"></a>Убедитесь, агент виртуальной Машины Azure

Каждая виртуальная машина Azure должен иметь [агент виртуальной Машины Azure](../virtual-machines/extensions/agent-windows.md) установлен. Чтобы реплицировать виртуальные машины Azure, Site Recovery устанавливает расширение агента.

- Если компьютер работает под управлением версии 9.7.0.0 или более поздней версии службы мобильности Site Recovery, агент виртуальной Машины Azure автоматически устанавливаются службы Mobility service на виртуальных машинах Windows. В более ранних версиях службы Mobility service необходимо установить агент автоматически.
- Для виртуальных машин Linux необходимо вручную установить агент виртуальной Машины Azure. Необходимо только установить агент виртуальной Машины Azure, если мобильной службы, установленной на перенесенном компьютере, имеет версию 9.6 или более ранней версии.


### <a name="install-the-agent-on-windows-vms"></a>Установите агент на виртуальных машинах Windows

Если вы используете версию службы мобильности Site Recovery более ранних, чем 9.7.0.0 или других требуется, чтобы установить агент вручную, сделайте следующее:  

1. Убедитесь, что у вас есть разрешения администратора на виртуальной Машине.
2. Скачайте [установщика агента виртуальной Машины](https://go.microsoft.com/fwlink/?LinkID=394789&clcid=0x409).
3. Запустите файл установщика.

#### <a name="validate-the-installation"></a>проверка установки
Чтобы проверить, что агент установлен:

1. На виртуальной машине Azure в папке C:\WindowsAzure\Packages должен находиться файл WaAppAgent.exe.
2. Щелкните этот файл правой кнопкой мыши, выберите пункт **Свойства** и перейдите на вкладку **Подробно**.
3. В поле **Версия продукта** должно отображаться значение 2.6.1198.718 или выше.

[Дополнительные сведения](https://docs.microsoft.com/azure/virtual-machines/extensions/agent-windows) об установке агента для Windows.

### <a name="install-the-agent-on-linux-vms"></a>Установите агент на виртуальных машинах Linux

Установка [виртуальной Машины Linux Azure](../virtual-machines/extensions/agent-linux.md) агент вручную следующим образом:

1. Убедитесь, что у вас есть разрешения администратора на компьютере.
2. Мы настоятельно рекомендуем установить агент виртуальной Машины Linux, с помощью пакета RPM или DEB из репозитория пакета дистрибутива. Все [поставщики поддерживаемых дистрибутивов](https://docs.microsoft.com/azure/virtual-machines/linux/endorsed-distros) встраивают пакет агента Linux для Azure в свои образы и репозитории.
    - Мы настоятельно рекомендуем обновлять агент только через репозиторий дистрибутива.
    - Мы не рекомендуем устанавливать агент виртуальной Машины Linux непосредственно из GitHub и обновлять его.
    -  Если последняя версия агента для вашего дистрибутива недоступна, обратитесь в службу поддержки дистрибутива, чтобы получить инструкции по установке последней версии агента. 

#### <a name="validate-the-installation"></a>проверка установки 

1. Выполните следующую команду: **ps -e** для убедитесь, что запущен агент Azure на виртуальной Машине Linux.
2. Если нужный процесс не запущен, выполните следующие команды для его перезапуска:
    - Для Ubuntu: **службы начала walinuxagent**
    - Для других дистрибутивов: **service waagent start**


## <a name="uninstall-the-mobility-service"></a>Удаление службы Mobility Service

1. Вручную удалите службу мобильности из виртуальной Машины Azure, с помощью одного из следующих методов. 
    - Windows. На панели управления выберите **Установка и удаление программ** и удалите компонент **Microsoft Azure Site Recovery Mobility Service/Главный целевой сервер**. В командной строке с повышенными привилегиями выполните следующую команду:
        ```
        MsiExec.exe /qn /x {275197FC-14FD-4560-A5EB-38217F80CBD1} /L+*V "C:\ProgramData\ASRSetupLogs\UnifiedAgentMSIUninstall.log"
        ```
    - Linux: войдите как привилегированный пользователь. В окне терминала перейдите в расположение **/user/local/ASR** и выполните следующую команду:
        ```
        ./uninstall.sh -Y
        ```
2. Перед настройкой репликации перезапустите виртуальную машину.

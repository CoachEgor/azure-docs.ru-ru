# <a name="exclude-disks-from-replication"></a>Исключение дисков из репликации

В этой статье описывается, как исключить диски при репликации Hyper-V VM в Azure. Возможно, по ряду причин можно исключить диски из репликации:

- Убедитесь, что неважные данные, вспененные на исключенном диске, не будут реплицироваться.
- Оптимизируйте потребляемую пропускную способность репликации или целевые ресурсы, исключив диски, которые не нужно реплицировать.
- Сохраняйте ресурсы хранения и сети, не реплицируя данные, которые вам не нужны.

Прежде чем исключить диски из репликации:

- Дополнительные сведения об исключении дисков см.в [этой статье](exclude-disks-replication.md).
- Просмотрите [типичные сценарии исключений](exclude-disks-replication.md#typical-scenarios) и [примеры,](exclude-disks-replication.md#example-1-exclude-the-sql-server-tempdb-disk) которые показывают, как исключение диска влияет на репликацию, сбой и сбой.

## <a name="before-you-start"></a>Перед началом работы

Прежде чем начать, обратите внимание на следующее:

- **Репликация**: По умолчанию все диски на машине реплицируются.
- **Тип диска**:
    - Можно исключить из репликации основные диски.
    - Вы не можете исключать диски операционной системы.
    - Мы не рекомендуем исключать динамические диски. Восстановление сайта не может определить, какой VHD является основным или динамичным в гостевом VM.  Если не исключать все зависимые динамические объемные диски, защищенный динамический диск становится неудавшимся диском на сбой над VM, и данные на этом диске недоступны.
- **Добавить/удалить/исключить диски:** После включения репликации нельзя добавлять/удалять/исключать диски для репликации. Если вы хотите добавить/удалить или исключить диск, необходимо отключить защиту Для VM, а затем включить его снова.
- **Failover**: После неудачи, если не удалось над приложениями необходимо исключить диски для того, чтобы работать, необходимо создать эти диски вручную. Кроме того, можно интегрировать автоматизацию Azure в план восстановления, чтобы создать диск во время сбоя машины.
- **Failback**: Когда вы не возвращаетесь на свой сайт после сбоя, диски, созданные вручную в Azure, не возвращаются обратно. Например, если вы не смогли выполнить три диска и создать два диска непосредственно на Azure VM, только три диска, которые не были перевернуты, затем сбой обратно. Нельзя включать диски, созданные вручную в отказе или в обратном воспроизведении виртуальных вывешие.

## <a name="exclude-disks"></a>Исключение дисков

1. Чтобы исключить диски при [включении репликации](site-recovery-hyper-v-site-to-azure.md) для Hyper-V VM, после выбора VMs, которые вы хотите воспроизвести, на странице**свойств** >  **включения репликации** > **нанастройки свойств** просмотрите **диски для копирования** столбца. По умолчанию все диски выбираются для репликации.
2. Если вы не хотите воспроизводить определенный диск, в **дисках, чтобы воспроизвести** четкий выбор для любых дисков, которые вы хотите исключить.

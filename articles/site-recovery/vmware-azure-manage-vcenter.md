# <a name="manage-vmware-vcenter-server"></a>Управление VMware vCenter Server

В этой статье кратко излагаются действия управления на vMware vCenter Server в [восстановлении сайта Azure.](site-recovery-overview.md)

## <a name="verify-prerequisites-for-vcenter-server"></a>Проверка предпосылок для vCenter Server

Предпосылки для серверов и vMs vCenter при аварийном восстановлении VM VMw в Azure перечислены в [матрице поддержки.](vmware-physical-azure-support-matrix.md#replicated-machines)

## <a name="set-up-an-account-for-automatic-discovery"></a>Настройка учетной записи для автоматического обнаружения

При настройке аварийного восстановления для нахлок VMware VMs восстановления сайта необходим доступ к хостам vCenter Server/vSphere. Затем сервер процесса восстановления сайта может автоматически обнаруживать вм-исправные вывешивать и по мере необходимости выводить их из строя. По умолчанию сервер процесса работает на сервере конфигурации восстановления сайта. Добавьте учетную запись для сервера конфигурации для подключения к хослу vCenter Server/vSphere следующим образом:

1. Войдите на сервер конфигурации.
1. Откройте инструмент сервера конфигурации _(cspsconfigtool.exe)_ с помощью ярлыка Desktop.
1. На вкладке **«Управление счетом»** нажмите **«Добавить учетную запись».**

   ![add-account](./media/vmware-azure-manage-vcenter/addaccount.png)

1. Укажите сведения об учетной записи и нажмите кнопку **ОК**, чтобы добавить учетную запись. Учетная запись должна обобщить привилегии в таблице разрешений учетной записи.

   > [!NOTE]
   > Синхронизация информации об учетной записи с восстановлением сайта занимает около 15 минут.

## <a name="add-vmware-server-to-the-vault"></a>Добавление сервера VMware в хранилище

При настройке аварийного восстановления для нахлок VMware VMs добавляется vCenter Server/vSphere хост, на котором вы обнаруживаете VM в хранилище восстановления сайта, следующим образом:

1. В хранилище > конфигурации **инфраструктуры** > восстановления сайта**Severs**, открыть сервер конфигурации.
1. В **подробностях страницы** нажмите **vCenter**.
1. В **Add vCenter**укажите дружественное имя для хоста vSphere или сервера vCenter.
1. Укажите IP-адрес или F-DN сервера.
1. Если серверы VMware настроены на прослушивание запросов через другой порт, оставьте порт 443.
1. Выберите учетную запись, используемую для подключения к vCenter vMwили vCenter или vSphere ESXi. После этого щелкните **OK**.

## <a name="modify-credentials"></a>Изменение учетных данных

При необходимости можно изменить учетные данные, используемые для подключения к хостам vCenter Server/vSphere:

1. Войдите на сервер конфигурации.
1. Откройте инструмент сервера конфигурации _(cspsconfigtool.exe)_ с помощью ярлыка Desktop.
1. На вкладке **Управление учетными записями** щелкните **Добавить учетную запись**.

   ![add-account](./media/vmware-azure-manage-vcenter/addaccount.png)

1. Предоставьте новые сведения об учетной записи и нажмите **OK**. Учетная запись нуждается в разрешениях, перечисленных в таблице [разрешений учетной записи.](#account-permissions)
1. В хранилище > конфигурация **инфраструктуры** > восстановления сайта**Severs**откройте сервер конфигурации.
1. В **подробностях**нажмите **Refresh Server**.
1. После завершения задания Refresh Server выберите сервер vCenter Server.
1. В **Резюме**выберите недавно добавленную учетную запись в **vCenter server/vSphere host account**и нажмите **Сохранить.**

   ![modify-account](./media/vmware-azure-manage-vcenter/modify-vcente-creds.png)

## <a name="delete-a-vcenter-server"></a>Удаление сервера vCenter

1. В хранилище > конфигурация **инфраструктуры** > восстановления сайта**Severs**откройте сервер конфигурации.
1. На странице **сведений** выберите сервер vCenter.
1. Нажмите кнопку **Удалить**.

   ![delete-account](./media/vmware-azure-manage-vcenter/delete-vcenter.png)

## <a name="modify-the-ip-address-and-port"></a>Изменение IP-адреса и порта

Можно изменить IP-адрес vCenter Server или порты, используемые для связи между сервером и восстановлением сайта. По умолчанию восстановление сайта получает доступ к информации vCenter Server/vSphere через порт 443.

1. В хранилище > > **серверов конфигурации** **инфраструктуры восстановления сайта**нажмите на сервер конфигураций, к которому добавлен сервер vCenter Server.
1. На **серверах vCenter**нажмите на сервер vCenter, который вы хотите изменить.
1. В **Резюме**— обновите IP-адрес и порт и сохраните изменения.

   ![add_ip_new_vcenter](media/vmware-azure-manage-vcenter/add-ip.png)

1. Чтобы изменения вступили в силу, дождитесь 15 минут или [обновите сервер конфигурации.](vmware-azure-manage-configuration-server.md#refresh-configuration-server)

## <a name="migrate-all-vms-to-a-new-server"></a>Перенести все вмхэ на новый сервер

Если вы хотите перенести все визм для использования нового vCenter Server, вам просто необходимо обновить IP-адрес, назначенный vCenter Server. Не добавляйте еще одну учетную запись VMware, так как это может привести к дублированию записей. Обновление адреса следующим образом:

1. В хранилище > > **серверов конфигурации** **инфраструктуры восстановления сайта**нажмите на сервер конфигураций, к которому добавлен сервер vCenter Server.
1. В разделе **серверов vCenter** нажмите на сервер vCenter, с которого вы хотите перенестись.
1. В **Резюме**обновите IP-адрес нового vCenter Server и сохраните изменения.
1. Как только IP-адрес обновляется, восстановление сайта начинает получать информацию об открытии VM от нового vCenter Server. Это не влияет на текущие действия репликации.

## <a name="migrate-a-few-vms-to-a-new-server"></a>Перенести несколько вм: на новый сервер

Если вы хотите перенести только несколько ваших репликационных вс-миноносцифицировать к новому серверу vCenter, сделайте следующее:

1. [Добавьте](#add-vmware-server-to-the-vault) новый сервер vCenter на сервер конфигурации.
1. [Отключить репликацию](site-recovery-manage-registration-and-protection.md#disable-protection-for-a-vmware-vm-or-physical-server-vmware-to-azure) для vMs, которые будут перемещаться на новый сервер.
1. В VMware переносите vMs на новый vCenter Server.
1. [Снова включить репликацию](vmware-azure-tutorial.md#enable-replication) для мигрированных VMs, выбрав новый сервер vCenter Server.

## <a name="migrate-most-vms-to-a-new-server"></a>Перенести большинство вм: на новый сервер

Если количество вс-м всм, которые вы хотите перенести на новый vCenter Server, превышает количество vMs, которые останутся на исходном сервере vCenter Server, сделайте следующее:

1. [Обновление IP-адреса,](#modify-the-ip-address-and-port) назначенного vCenter Server в настройках сервера конфигурации, на адрес нового vCenter Server.
1. [Отключить репликацию](site-recovery-manage-registration-and-protection.md#disable-protection-for-a-vmware-vm-or-physical-server-vmware-to-azure) для нескольких vMs, которые остаются на старом сервере.
1. [Добавьте старый vCenter Server](#add-vmware-server-to-the-vault) и его IP-адрес на сервер конфигурации.
1. [Повторное воспроизведение](vmware-azure-tutorial.md#enable-replication) для вс-в., которые остаются на старом сервере.

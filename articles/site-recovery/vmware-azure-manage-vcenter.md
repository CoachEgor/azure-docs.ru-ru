# <a name="manage-vmware-vcenter-server"></a>Управление сервером VMware vCenter

В этой статье рассматриваются различные операции Site Recovery, которые могут выполняться на сервере VMware vCenter Server. Прежде чем начать, убедитесь, что соблюдены [предварительные требования](vmware-physical-azure-support-matrix.md#replicated-machines).


## <a name="set-up-an-account-for-automatic-discovery"></a>Настройка учетной записи для автоматического обнаружения

Службе Site Recovery нужен доступ к учетным записям VMware для отработки отказа и восстановления размещения, а также для автоматического обнаружения виртуальных машин на сервере обработки. Создайте учетную запись для доступа следующим образом:

1. Войдите на компьютер сервера конфигурации.
2. Откройте файл cspsconfigtool.exe с помощью ярлыка на рабочем столе.
3. На вкладке **Управление учетными записями** щелкните **Добавить учетную запись**.

   ![add-account](./media/vmware-azure-manage-vcenter/addaccount.png)
1. Укажите сведения об учетной записи и нажмите кнопку **ОК**, чтобы добавить учетную запись.  Для учетной записи требуются разрешения, представленные в приведенной ниже таблице. 

Чтобы сведения учетной записи синхронизировались со службой Site Recovery, потребуется около 15 минут.

## <a name="add-vmware-server-to-the-vault"></a>Добавление сервера VMware в хранилище

1. На портале Azure откройте свое хранилище и последовательно выберите **Site Recovery Infrastructure (Инфраструктура Site Recovery)**  > **Configuration Severs (Серверы конфигураций)** , а затем откройте страницу сервера конфигурации.
2. На странице **сведений** щелкните **+vCenter**.

## <a name="modify-credentials"></a>Изменение учетных данных

Измените учетные данные, используемые для подключения к серверу vCenter или узлу ESXi, следующим образом:

1. Войдите на сервер конфигурации и запустите файл cspsconfigtool.exe. с рабочего стола.
2. На вкладке **Управление учетными записями** щелкните **Добавить учетную запись**.

   ![add-account](./media/vmware-azure-manage-vcenter/addaccount.png)
3. Укажите сведения о новой учетной записи и нажмите кнопку **ОК**, чтобы добавить учетную запись. Учетной записи следует назначить разрешения, перечисленные [выше](#account-permissions).
4. На портале Azure откройте хранилище и последовательно выберите **Site Recovery Infrastructure (Инфраструктура Site Recovery)**  > **Configuration Severs (Серверы конфигураций)** , а затем откройте страницу сервера конфигурации.
5. На странице **сведений** щелкните **Refresh Server** (Обновить сервер).
6. Когда завершится задание обновления сервера, выберите сервер vCenter, чтобы открыть для него страницу **сводки**.
7. Выберите только что добавленную учетную запись в поле **vCenter server/vSphere host account** (Учетная запись сервера vCenter Server и узла vSphere) и щелкните **Сохранить**.

   ![modify-account](./media/vmware-azure-manage-vcenter/modify-vcente-creds.png)

## <a name="delete-a-vcenter-server"></a>Удаление сервера vCenter

1. На портале Azure откройте свое хранилище и последовательно выберите **Site Recovery Infrastructure (Инфраструктура Site Recovery)**  > **Configuration Severs (Серверы конфигураций)** , а затем откройте страницу сервера конфигурации.
2. На странице **сведений** выберите сервер vCenter.
3. Нажмите кнопку **Удалить**.

   ![delete-account](./media/vmware-azure-manage-vcenter/delete-vcenter.png)

## <a name="modify-the-vcenter-ip-address-and-port"></a>Изменение IP-адреса и порта vCenter

1. Войдите на портал Azure.
2. Перейдите к **хранилищу служб восстановления** > **Site Recovery инфраструктуры** > **серверы конфигурации**.
3. Щелкните сервер конфигурации, которому назначен vCenter.
4. В разделе **серверы vCenter Server** щелкните vCenter, который нужно изменить.
5. На странице Сводка vCenter обновите IP-адрес и порт vCenter в соответствующих полях, а затем сохраните изменения.

   ![add_ip_new_vcenter](media/vmware-azure-manage-vcenter/add-ip.png)

6. Чтобы изменения стали эффективными, подождите 15 минут или [Обновите сервер конфигурации](vmware-azure-manage-configuration-server.md#refresh-configuration-server).

## <a name="migrate-all-protected-virtual-machines-to-a-new-vcenter"></a>Перенос всех защищенных виртуальных машин в новый vCenter

Чтобы перенести все виртуальные машины в новый сервер vCenter, не добавляйте еще одну учетную запись vCenter. Это может привести к дублированию записей. Просто обновите IP-адрес нового vCenter:

1. Войдите на портал Azure.
2. Перейдите к **хранилищу служб восстановления** > **Site Recovery инфраструктуры** > **серверы конфигурации**.
3. Щелкните сервер конфигурации, которому назначен старый vCenter.
4. В разделе **серверы vCenter Server** щелкните vCenter, из которого планируется выполнить миграцию.
5. На странице Сводка vCenter обновите IP-адрес нового vCenter в поле **имя узла vCenter Server/vSphere или IP-адрес**. Сохраните изменения.

Как только IP-адрес будет обновлен, Site Recovery компоненты начнут получать сведения об обнаружении виртуальных машин из нового vCenter. Это не повлияет на текущие действия репликации.

## <a name="migrate-few-protected-virtual-machines-to-a-new-vcenter"></a>Перенос нескольких защищенных виртуальных машин в новый vCenter

> [!NOTE]
> Этот раздел применим только при переносе нескольких защищенных виртуальных машин в новый сервер vCenter. Если вы хотите защитить новый набор виртуальных машин из нового vCenter, [добавьте новые сведения о vCenter на сервер конфигурации](#add-vmware-server-to-the-vault) и начните с **[включения защиты](vmware-azure-tutorial.md#enable-replication)** .

Перемещение нескольких виртуальных машин в новый сервер vCenter:

1. [Добавьте новые сведения о vCenter на сервер конфигурации](#add-vmware-server-to-the-vault).
2. [Отключите репликацию виртуальных машин](site-recovery-manage-registration-and-protection.md#disable-protection-for-a-vmware-vm-or-physical-server-vmware-to-azure) , которые вы планируете перенести.
3. Завершите перенос выбранных виртуальных машин в новый vCenter.
4. Теперь Защитите перенесенные виртуальные машины, [выбрав новый vCenter при включении защиты](vmware-azure-tutorial.md#enable-replication).

> [!TIP]
> Если число переносимых виртуальных **машин превышает число** виртуальных машин, хранящихся в старом vCenter, обновите IP-адрес нового vCenter, используя инструкции, приведенные здесь. Для нескольких виртуальных машин, которые хранятся в старом vCenter, [отключите репликацию](site-recovery-manage-registration-and-protection.md#disable-protection-for-a-vmware-vm-or-physical-server-vmware-to-azure). [добавьте новые сведения о vCenter на сервер конфигурации](#add-vmware-server-to-the-vault)и запустите **[включить защиту](vmware-azure-tutorial.md#enable-replication)** .

## <a name="frequently-asked-questions"></a>Часто задаваемые вопросы

1. Если защищенные виртуальные машины перемещаются с одного узла ESXi на другой, будет ли это влиять на репликацию?

    Нет, это не повлияет на текущую репликацию. Однако [обязательно разверните главный целевой сервер с достаточными привилегиями](vmware-azure-reprotect.md#deploy-a-separate-master-target-server) .

2. Какие номера портов используются для обмена данными между vCenter и другими компонентами Site Recovery?

    Порт по умолчанию — 443. Сервер конфигурации будет получать доступ к сведениям об узле vCenter/vSphere через этот порт. Если вы хотите обновить эту информацию, щелкните [здесь](#modify-the-vcenter-ip-address-and-port).

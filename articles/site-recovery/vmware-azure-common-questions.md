# <a name="common-questions-about-vmware-to-azure-replication"></a>Часто задаваемые вопросы о репликации из VMware в Azure

В этой статье ответы на общие вопросы могут возникнуть при развертывании аварийного восстановления виртуальных машин VMware (VM) в Azure.

## <a name="general"></a>Общие сведения

### <a name="what-do-i-need-for-vmware-vm-disaster-recovery"></a>Что нужно для аварийного восстановления VMware VM?

[Узнайте о компонентах, участвующих](vmware-azure-architecture.md) в аварийном восстановлении VMw.

### <a name="can-i-use-site-recovery-to-migrate-vmware-vms-to-azure"></a>Могу ли я использовать восстановление сайта для переноса VMware VMs в Azure?

Да. В дополнение к использованию восстановления сайта для настройки полного аварийного восстановления для VMware VMs, вы также можете использовать восстановление сайта для переноса на места VMware VMs в Azure. В этом сценарии вы воспроизводите на местах VMware VMs для хранения Azure. Затем выполняется отработка отказа из локальной среды в Azure. После отработки отказа приложения и рабочие нагрузки доступны и выполняются на виртуальных машинах Azure. Этот процесс подосеваю настройке полного аварийного восстановления, за исключением того, что при миграции вы не можете выполнить обратный ход из Azure.

### <a name="does-my-azure-account-need-permissions-to-create-vms"></a>Требуются ли учетной записи Azure разрешения на создание виртуальных машин?

Если вы являетесь администратором подписки, у вас есть необходимые разрешения на репликацию. Если вы не являетесь администратором, вам нужны разрешения для этих действий:

- Создайте VM Azure в группе ресурсов и виртуальную сеть, которую вы указываете при настройке восстановления сайта.
- Запись на выбранную учетную запись хранения или управляемый диск на основе вашей конфигурации.

[Узнайте больше](site-recovery-role-based-linked-access-control.md#permissions-required-to-enable-replication-for-new-virtual-machines) об необходимых разрешениях.

### <a name="what-applications-can-i-replicate"></a>Какие приложения можно реплицировать?

Вы можете воспроизвести любое приложение или рабочую нагрузку, работая на VMware VM, которая отвечает [требованиям репликации.](vmware-physical-azure-support-matrix.md#replicated-machines)

- Восстановление сайта поддерживает репликацию приложений, так что приложения могут быть сбой и не обратно в интеллектуальное состояние.
- Восстановление сайта интегрируется с такими приложениями Майкрософт, как SharePoint, Exchange, Dynamics, S'L Server и Active Directory. Она также тесно сотрудничает с ведущими поставщиками, включая Oracle, SAP, IBM и Red Hat.

[Подробнее](site-recovery-workload.md) о защите рабочей нагрузки.

### <a name="can-i-use-a-guest-os-server-license-on-azure"></a>Могу ли я использовать лицензию сервера гостевой ОС в Azure?

Да, клиенты Microsoft Software Assurance могут использовать программу [Преимущество гибридного использования Azure](https://azure.microsoft.com/pricing/hybrid-benefit/), чтобы сэкономить на стоимости лицензирования компьютеров под управлением Windows Server, которые переносятся в Azure, или использовать Azure для аварийного восстановления.

## <a name="security"></a>Безопасность

### <a name="what-access-to-vmware-servers-does-site-recovery-need"></a>Какой доступ к серверам VMware нужен восстановлению сайта?

Site Recovery требуется доступ к серверам VMware, чтобы:

- Настройка VMware VM под управлением сервера конфигурации восстановления сайта.
- автоматически обнаруживать виртуальные машины для репликации.

### <a name="what-access-to-vmware-vms-does-site-recovery-need"></a>Какой доступ к VMware VMs нужен для восстановления сайта?

- Для репликации VMware VM необходимо установить и запустить службу восстановления сайта. Можно развернуть инструмент вручную или указать, что восстановление сайта делает push-установку службы при включании репликации для VM.
- Во время репликации виртуальные машины обмениваются данными с Site Recovery следующим образом:
    - VM общаются с сервером конфигурации на порте HTTPS 443 для управления репликацией.
    - VM отправляют данные репликации на сервер процесса в порту HTTPS 9443. (Эта настройка может быть изменена.)
    - если включить согласованность между виртуальными машинами, они будут обмениваться данными друг с другом через порт 20004.

### <a name="is-replication-data-sent-to-site-recovery"></a>Отправляются ли данные репликации в Site Recovery?

Нет, восстановление сайта не перехватывает реплицированные данные и не имеет никакой информации о том, что работает на ваших ВМ. Обмен данными репликации осуществляется между гипервизорами VMware и Хранилищем Azure. Служба Site Recovery не обладает способностью перехватывать эти данные. В службу Site Recovery передаются только метаданные, необходимые для управления репликацией и отработкой отказа.

Восстановление сайта сертифицировано для ISO 27001:2013 и 27018, HIPAA и DPA. Он проходит оценку SOC2 и FedRAMP JAB.

### <a name="is-there-any-difference-in-cost-between-replicating-to-storage-or-directly-to-managed-disks"></a>Есть ли разница в стоимости между репликацией для хранения или непосредственно управляемых дисков?

Управляемые диски заряжаются немного иначе, чем учетные записи хранилища. [Подробнее](https://azure.microsoft.com/pricing/details/managed-disks/) о ценах на управляемые диски.

Обычно вы увидите увеличение затрат на транзакции, понесенные на учетные записи хранения GPv2, так как восстановление сайта Azure является большим объемом транзакций. [Прочитайте больше,](../storage/common/storage-account-upgrade.md#pricing-and-billing) чтобы оценить изменения.

## <a name="managed-disks"></a>Управляемые диски

### <a name="where-does-site-recovery-replicate-data-to"></a>Где происходит воспроизведение данных на восстановлении сайта?

Восстановление сайта реплицирует на местах VMware VMs и физические серверы на управляемые диски в Azure.

- Сервер процесса восстановления сайта записывает журналы репликации в учетную запись хранения кэша в целевом регионе.
- Эти журналы используются для создания точек восстановления на дисках, управляемых Azure, которые имеют приставку **asrseeddisk.**
- При сбою происходит, точка восстановления, выбранная вы выбираете, используется для создания нового диска, управляемого целевым. Этот управляемый диск прикрепляется к VM в Azure.
- ВМ, которые ранее были реплицированы на учетную запись хранения (до марта 2019 года), не затрагиваются.

### <a name="can-i-replicate-new-machines-to-storage-accounts"></a>Могу ли я воспроизвести новые машины для хранения счетов?

Нет. Начиная с марта 2019 года на портале Azure можно реплицировать только управляемые диски Azure.

Репликация новых ВМ на учетную запись хранилища доступна только с помощью PowerShell или REST API (версия 2018-01-10 или 2016-08-10).

### <a name="what-are-the-benefits-of-replicating-to-managed-disks"></a>Каковы преимущества репликации управляемых дисков?

[Узнайте, как](https://azure.microsoft.com/blog/simplify-disaster-recovery-with-managed-disks-for-vmware-and-physical-servers/) Восстановление сайта упрощает аварийное восстановление с помощью управляемых дисков.

### <a name="can-i-change-the-managed-disk-type-after-a-machine-is-protected"></a>Могу ли я изменить тип управляемого диска после защиты машины?

Да, вы можете легко [изменить тип управляемого диска](https://docs.microsoft.com/azure/virtual-machines/windows/convert-disk-storage) для текущих репликаций. Прежде чем изменить тип, убедитесь, что на управляемом диске не генерируется URL-адреса общей подписи доступа:

1. Перейдите на ресурс **Управляемого диска** на портале Azure и проверьте, есть ли у вас на лезвии **«Обзор»** общий баннер подписи доступа.
1. Если баннер присутствует, выберите его, чтобы отменить текущий экспорт.
1. Измените тип диска в течение следующих нескольких минут. Если вы измените тип управляемого диска, подождите, пока новые точки восстановления будут сгенерированы восстановлением сайта Azure.
1. Используйте новые точки восстановления для любого теста failover или неудачи в будущем.

### <a name="can-i-switch-replication-from-managed-disks-to-unmanaged-disks"></a>Могу ли я переключить репликацию с управляемых дисков на неуправляемые?

Нет. Переход от управляемого к неуправляемому не поддерживается.

## <a name="replication"></a>Репликация

### <a name="what-are-the-replicated-vm-requirements"></a>Каковы требования к реплицируемой виртуальной машине?

[Узнайте больше](vmware-physical-azure-support-matrix.md#replicated-machines) о требованиях к поддержке VMware VMs и физических серверов.

### <a name="how-often-can-i-replicate-to-azure"></a>Как часто можно выполнять репликацию в Azure?

Этот процесс является непрерывным при репликации виртуальных машин VMware в Azure.

### <a name="can-i-extend-replication"></a>Можно ли расширить репликацию?

Расширенная репликация и цепочка репликации не поддерживаются. Запросите эту функцию на [форуме отзывов и предложений](https://feedback.azure.com/forums/256299-site-recovery/suggestions/6097959).

### <a name="can-i-do-an-offline-initial-replication"></a>Можно ли выполнить начальную репликацию в автономном режиме?

Репликация в автономном режиме не поддерживается. Запросите эту функцию на [форуме отзывов и предложений](https://feedback.azure.com/forums/256299-site-recovery/suggestions/6227386-support-for-offline-replication-data-transfer-from).

### <a name="what-is-asrseeddisk"></a>Что такое ассеедивый диск?

Для каждого исходного диска данные реплицируются на управляемый диск в Azure. Этот диск имеет префикс **asrseeddisk**. Он хранит копию исходного диска и все моментальные моменты точки восстановления.

### <a name="can-i-exclude-disks-from-replication"></a>Могу ли я исключить диски из репликации?

Да, вы можете исключить диски.

### <a name="can-i-replicate-vms-that-have-dynamic-disks"></a>Могу ли я воспроизвести вдыхать вс-миноносцы с динамическими дисками?

Динамические диски могут быть реплицированы. Диск операционной системы должен представлять собой базовый диск.

### <a name="if-i-use-replication-groups-for-multi-vm-consistency-can-i-add-a-new-vm-to-an-existing-replication-group"></a>Если я использую группы репликации для согласованности нескольких виртуальных машин, могу ли я добавить новую виртуальную машину в имеющуюся группу репликации?

Да, вы можете добавить новые виртуальные машины в имеющуюся группу репликации при включении для них репликации. Но:

- Вы не можете добавить VM в существующую группу репликации после начала репликации.
- Нельзя создать группу репликации для существующих VM.

### <a name="can-i-modify-vms-that-are-replicating-by-adding-or-resizing-disks"></a>Можно ли изменить виртуальные машины, которые реплицируются, путем добавления дисков или изменения их размера?

Для репликации VMware в Azure можно изменить размер диска исходных VMs. Если вы хотите добавить новые диски, необходимо добавить диск и подпяивая защиту для VM.

### <a name="can-i-migrate-on-premises-machines-to-a-new-vcenter-server-without-impacting-ongoing-replication"></a>Могу ли я перенести внутренние машины на новый vCenter Server без влияния на текущую репликацию?

Нет. Изменение VMware Vcenter или миграции повлияет на текущее репликацию. Настройте восстановление сайта с помощью нового сервера vCenter и снова включите репликацию для машин.

### <a name="can-i-replicate-to-a-cache-or-target-storage-account-that-has-a-virtual-network-with-azure-firewalls-configured-on-it"></a>Могу ли я воспроизвести кэш или учетную запись целевого хранилища с виртуальной сетью (с Azure Firewalls), настроенной на нее?

Нет, восстановление сайта не поддерживает репликацию в хранилище Azure в виртуальных сетях.

## <a name="configuration-server"></a>Сервер конфигурации

### <a name="what-does-the-configuration-server-do"></a>Что делает сервер конфигурации?

На сервере конфигурации выполняются локальные компоненты Site Recovery, включая:

- Сам сервер конфигурации. Сервер координирует связь между компонентами и Azure и управляет репликацией данных.
- Сервер процесса, который действует как шлюз репликации. Этот сервер:
    1. Получает данные репликации.
    2. оптимизирует данные с помощью кэширования, сжатия и шифрования;
    3. Отправляет данные в хранилище Azure.
  Технологический сервер также выполняет push-установку Службы мобильности на VMs и выполняет автоматическое обнаружение предприимки VMw.
- Основной целевой сервер, который обрабатывает данные репликации во время сбоя из Azure.

См. дополнительные сведения о [компонентах и процессах сервера конфигурации](vmware-azure-architecture.md).

### <a name="where-do-i-set-up-the-configuration-server"></a>Где настраивается сервер конфигурации?

Вам нужен один, высокодоступный, наемный VMware VM для сервера конфигурации. Для физического восстановления аварийного сообщения сервера установите сервер конфигурации на физическом компьютере.

### <a name="what-do-i-need-for-the-configuration-server"></a>Что мне нужно для сервера конфигурации?

Ознакомьтесь с [предварительными требованиями.](vmware-azure-deploy-configuration-server.md#prerequisites)

### <a name="can-i-manually-set-up-the-configuration-server-instead-of-using-a-template"></a>Можно ли вручную настроить сервер конфигурации, не используя шаблон?

Мы рекомендуем [создать сервер конфигурации VM](vmware-azure-deploy-configuration-server.md) с помощью последней версии шаблона Open Virtualization Format (OVF). Если вы не можете использовать шаблон (например, если у вас нет доступа к серверу VMware), [загрузите](physical-azure-set-up-source.md) файл настройки с портала и навяжжнажмите сервер конфигурации.

### <a name="can-a-configuration-server-replicate-to-more-than-one-region"></a>Может ли сервер конфигурации выполнять репликацию больше, чем в один регион?

Нет. Чтобы воспроизвести более чем в одном регионе, нужен сервер конфигурации в каждом регионе.

### <a name="can-i-host-a-configuration-server-in-azure"></a>Можно ли разместить сервер конфигурации в Azure?

Хотя это возможно, Azure VM, работающий на сервере конфигурации, должен общаться с вашей инфраструктурой VMware и VM. Это сообщение добавляет задержку и влияет на текущую репликацию.

### <a name="how-do-i-update-the-configuration-server"></a>Как обновить сервер конфигурации?

[Узнайте,](vmware-azure-manage-configuration-server.md#upgrade-the-configuration-server) как обновить сервер конфигурации.

- Вы можете найти последнюю информацию об обновлении на [странице обновлений Azure.](https://azure.microsoft.com/updates/?product=site-recovery)
- Вы можете скачать последнюю версию с портала. Или вы можете скачать последнюю версию сервера конфигурации непосредственно из [Центра загрузки Майкрософт.](https://aka.ms/asrconfigurationserver)
- Если версия более чем на четыре версии старше [support statement](https://aka.ms/asr_support_statement) текущей версии, см.

### <a name="should-i-back-up-the-configuration-server"></a>Должен ли я резервное копирование сервера конфигурации?

Мы рекомендуем создавать плановые резервные копии сервера конфигурации.

- Для успешного сбоя возврата, VM время сбоя обратно должны существовать в базе данных сервера конфигурации.
- Сервер конфигурации должен работать и находиться в подключенном состоянии.
- [Узнайте больше](vmware-azure-manage-configuration-server.md) об общих задачах управления сервером конфигурации.

### <a name="when-im-setting-up-the-configuration-server-can-i-download-and-install-mysql-manually"></a>Можно ли скачать и установить MySQL вручную при настройке сервера конфигурации?

Да. Скачайте среду MySQL и поместите ее в папку C:\Temp\ASRSetup. Затем установите его вручную. После настройки виртуальной машины сервера конфигурации и принятия условий для MySQL в поле **Скачать и установить** будет показано значение **Уже установлено**.

### <a name="can-i-avoid-downloading-mysql-but-let-site-recovery-install-it"></a>Можно ли не скачивать MySQL и разрешить Site Recovery сделать это?

Да. Скачайте установщик MySQL и поместите его в папку C:\Temp\ASRSetup. При настройке сервера конфигурации VM, принять условия и выбрать **Скачать и установить.** Портал будет использовать установщик, который вы добавили для установки MyS'L.

### <a name="can-i-use-the-configuration-server-vm-for-anything-else"></a>Можно ли использовать виртуальную машину сервера конфигурации для чего-либо еще?

Нет. Используйте VM только для сервера конфигурации.

### <a name="can-i-clone-a-configuration-server-and-use-it-for-orchestration"></a>Можно ли клонировать сервер конфигурации и использовать его для оркестрации?

Нет. Настройка нового сервера конфигурации, чтобы избежать проблем с регистрацией.

### <a name="can-i-change-the-vault-in-which-the-configuration-server-is-registered"></a>Могу ли я изменить хранилище, в котором зарегистрирован сервер конфигурации?

Нет. После того, как хранилище связано с сервером конфигурации, его нельзя изменить. [Узнайте](vmware-azure-manage-configuration-server.md#register-a-configuration-server-with-a-different-vault) о регистрации сервера конфигурации с другим хранилищем.

### <a name="can-i-use-the-same-configuration-server-for-disaster-recovery-of-both-vmware-vms-and-physical-servers"></a>Могу ли я использовать один и тот же сервер конфигурации для аварийного восстановления как VMware VMs, так и физических серверов?

Да, но обратите внимание, что физическая машина может быть невозвращена только к VMware VM.

### <a name="where-can-i-download-the-passphrase-for-the-configuration-server"></a>Где можно скачать парольную фразу для сервера конфигурации?

[Узнайте,](vmware-azure-manage-configuration-server.md#generate-configuration-server-passphrase) как скачать пароль.

### <a name="where-can-i-download-vault-registration-keys"></a>Где можно загрузить ключи регистрации хранилища?

В хранилище служб восстановления выберите **серверы конфигурации** в > **управлении** **инфраструктурой восстановления сайта.** Затем, в **серверах,** выберите **ключ регистрации скачать,** чтобы загрузить файл учетных данных хранилища.

### <a name="can-a-single-configuration-server-be-used-to-protect-multiple-vcenter-instances"></a>Можно ли использовать один сервер конфигурации для защиты нескольких экземпляров vCenter?

Да, один сервер конфигурации может защитить VMs через несколько vCenters.  Количество экземпляров vCenter может быть добавлено на сервер конфигурации, однако ограничения на количество вс-м внимания одного сервера конфигурации могут быть заполнены.

### <a name="can-a-single-configuration-server-protect-multiple-clusters-within-vcenter"></a>Может ли один сервер конфигурации защитить несколько кластеров в vCenter?

Да, восстановление сайта Azure может защитить высочения в различных кластерах.

## <a name="process-server"></a>Сервер обработки

### <a name="why-am-i-unable-to-select-the-process-server-when-i-enable-replication"></a>Почему при включании репликации не удается выбрать сервер процесса?

Обновления в версиях 9.24 и позже теперь отображают [работоспособность сервера процесса при включании репликации.](vmware-azure-enable-replication.md#enable-replication) Эта функция помогает избежать регулирования сервера процесса и свести к минимуму использование неработоспособных серверов процесса.

### <a name="how-do-i-update-the-process-server-to-version-924-or-later-for-accurate-health-information"></a>Как обновить сервер процесса до версии 9.24 или позже для получения точной информации о работоспособности?

Начиная с [версии 9.24,](service-updates-how-to.md#links-to-currently-supported-update-rollups)были добавлены дополнительные оповещения, указывающие на работоспособность сервера процесса. [Обновите компоненты восстановления сайта до версии 9.24 или позже,](service-updates-how-to.md#links-to-currently-supported-update-rollups) чтобы все оповещения были сгенерированы.

### <a name="how-can-i-ensure-high-availability-of-the-process-server"></a>Как обеспечить высокую доступность сервера процесса?

Настраивая более одного сервера процессов, конструкция обеспечивает гибкость для перемещения защищенных машин с неработоспособного сервера процесса на рабочий сервер процесса. Перемещение машины с одного сервера процесса на другой должно быть инициировано явно/вручную через определенные шаги здесь: [перемещение ВМ между серверами процесса.](vmware-azure-manage-process-server.md#move-vms-to-balance-the-process-server-load)

## <a name="failover-and-failback"></a>Отработка отказа и восстановление размещения

### <a name="can-i-use-the-on-premises-process-server-for-failback"></a>Могу ли я использовать сервер внутреннего процесса для сбоя?

Мы настоятельно рекомендуем создать сервер процессов в Azure для целей отказа, чтобы избежать переноса данных. Кроме того, в случае, если вы отделили сеть исходных всфой с контактной сетью Azure на сервере конфигурации, необходимо использовать сервер процесса, созданный в Azure, для обратной связи.

### <a name="can-i-keep-the-ip-address-on-failover"></a>Могу ли я сохранить IP-адрес на отказе?

Да, вы можете сохранить IP-адрес на failover. Убедитесь, что вы укажете целевой IP-адрес в настройках **вычислений и сети** для VM до сбоя. Кроме того, выключите машины во время сбоя, чтобы избежать конфликтов IP-адресов во время сбоя.

### <a name="can-i-change-the-target-vm-size-or-vm-type-before-failover"></a>Могу ли я изменить целевой размер VM или тип VM до неудачи?

Да, вы можете изменить тип или размер VM в любое время до неудачи. На портале используйте настройки **вычислений и сети** для реплицируемого VM.

### <a name="how-far-back-can-i-recover"></a>До какой точки во времени можно выполнить восстановление?

Для VMware to Azure самая старая точка восстановления, которая может быть возможной для использования, составляет 72 часа.

### <a name="how-do-i-access-azure-vms-after-failover"></a>Как получить доступ к виртуальным машина Azure после отработки отказа?

После сбоя вы можете получить доступ к VMs Azure через безопасное подключение к Интернету, через VPN от сайта к сайту или через Azure ExpressRoute. Чтобы подключиться, вы должны подготовить несколько вещей. Ознакомьтесь с [дополнительными сведениями](site-recovery-test-failover-to-azure.md#prepare-to-connect-to-azure-vms-after-failover).

### <a name="is-failed-over-data-resilient"></a>Устойчивость ли неудавшихся данных?

Среда Azure разработана для обеспечения отказоустойчивости. Восстановление сайта разработано для сбоя во вторичном центре обработки данных Azure, как того требует соглашение уровня обслуживания Azure (SLA). При сбое происходит сбой, мы удостоверяемся, что ваши метаданные и хранилища остаются в том же географическом регионе, что вы выбрали для вашего хранилища.

### <a name="is-failover-automatic"></a>Отработка отказа выполняется автоматически?

[Failover](site-recovery-failover.md) не является автоматическим. Вы начинаете неудачу, сделав один выбор на портале, или вы можете использовать [PowerShell,](/powershell/module/az.recoveryservices) чтобы вызвать неудачу.

### <a name="can-i-fail-back-to-a-different-location"></a>Можно ли выполнить восстановление размещения в другое расположение?

Да. Если вы не смогли перейти в Azure, вы можете не вернуться в другое место, если исходное место недоступно. Ознакомьтесь с [дополнительными сведениями](concepts-types-of-failback.md#alternate-location-recovery-alr).

### <a name="why-do-i-need-a-vpn-or-expressroute-with-private-peering-to-fail-back"></a>Зачем мне нужен VPN или ExpressRoute с частными вглядывающимися, чтобы потерпеть неудачу?

При отказе от возврата из Azure данные из Azure копируется обратно в ваш закрытый VM, и требуется частный доступ.

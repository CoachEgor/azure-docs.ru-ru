# <a name="common-questions---vmware-to-azure-replication"></a>Часто задаваемые вопросы о репликации из VMware в Azure

В этой статье содержатся ответы на часто задаваемые вопросы, при развертывании аварийного восстановления локальных виртуальных машин VMware в Azure. 

## <a name="general"></a>Общие сведения 
### <a name="what-do-i-need-for-vmware-vm-disaster-recovery"></a>Что нужно для аварийного восстановления виртуальных Машин VMware?

[Дополнительные сведения о](vmware-azure-architecture.md) компоненты, участвующие в аварийное восстановление виртуальных машин VMware. 

### <a name="can-i-use-site-recovery-to-migrate-vmware-vms-to-azure"></a>Можно ли использовать Site Recovery для переноса виртуальных машин VMware в Azure?

Да, помимо использования Site Recovery, чтобы настроить полное аварийное восстановление для виртуальных машин VMware, можно также использовать Site Recovery для переноса локальных виртуальных машин VMware в Azure. В этом сценарии выполняется репликация локальных виртуальных машин в хранилище Azure. Затем выполняется отработка отказа из локальной среды в Azure. После отработки отказа приложения и рабочие нагрузки доступны и выполняются на виртуальных машинах Azure. Процесс аналогичен настройке полное аварийное восстановление, за исключением того, что при миграции невозможно восстановить размещение из Azure.


### <a name="does-my-azure-account-need-permissions-to-create-vms"></a>Требуются ли учетной записи Azure разрешения на создание виртуальных машин?
Если вы являетесь администратором подписки, у вас есть необходимые разрешения на репликацию. Если вы не являетесь, необходимо иметь разрешения для создания виртуальной Машины Azure в группе ресурсов и виртуальную сеть, указанную вами при настройке Site Recovery и разрешения на запись в выбранной учетной записи хранения или управляемый диск на основе конфигурации. [Узнайте больше](site-recovery-role-based-linked-access-control.md#permissions-required-to-enable-replication-for-new-virtual-machines).

### <a name="what-applications-can-i-replicate"></a>Какие приложения можно реплицировать?
Можно реплицировать любые приложения или рабочие нагрузки, выполняемые в виртуальной машине VMware, которые соответствуют [требованиям к репликации](vmware-physical-azure-support-matrix.md##replicated-machines).
- Служба Site Recovery также поддерживает репликацию уровня приложения, позволяя выполнять отработку отказа приложений и восстанавливать их размещение до программируемого состояния.
- Site Recovery интегрируется с приложениями Майкрософт, включая SharePoint, Exchange, Dynamics, SQL Server и Active Directory, а также тесно взаимодействует с решениями ведущих производителей, в том числе Oracle, SAP, IBM и Red Hat.
- [Подробнее](site-recovery-workload.md) о защите рабочей нагрузки.

### <a name="can-i-use-a-guest-os-server-license-on-azure"></a>Можно использовать гостевой ОС серверная лицензия на Azure?
Да, клиенты Microsoft Software Assurance могут использовать программу [Преимущество гибридного использования Azure](https://azure.microsoft.com/pricing/hybrid-benefit/), чтобы сэкономить на стоимости лицензирования **компьютеров под управлением Windows Server**, которые переносятся в Azure, или использовать Azure для аварийного восстановления.

## <a name="security"></a>Безопасность

### <a name="what-access-does-site-recovery-need-to-vmware-servers"></a>Какой доступ к серверам VMware требуется Site Recovery?
Site Recovery требуется доступ к серверам VMware, чтобы:

- Настройка виртуальной Машины VMware под управлением сервера конфигурации Site Recovery,
- автоматически обнаруживать виртуальные машины для репликации. 


### <a name="what-access-does-site-recovery-need-to-vmware-vms"></a>Какой доступ к виртуальным машинам VMware требуется Site Recovery?

- На реплицируемой виртуальной машине VMware должна быть установлена и запущена служба Site Recovery Mobility Service. Можно развернуть средство вручную или указать, что служба Site Recovery должна принудительно установить Mobility Service при включении репликации для виртуальной машины. 
- Во время репликации виртуальные машины обмениваются данными с Site Recovery следующим образом:
    - виртуальные машины обмениваются данными с сервером конфигурации через порт HTTPS 443, чтобы управлять репликацией;
    - виртуальные машины отправляют данные репликации на сервер обработки через порт HTTPS 9443 (можно изменить);
    - если включить согласованность между виртуальными машинами, они будут обмениваться данными друг с другом через порт 20004.


### <a name="is-replication-data-sent-to-site-recovery"></a>Отправляются ли данные репликации в Site Recovery?
Нет, служба Site Recovery не перехватывает реплицируемые данные и не получает сведения о компонентах, запущенных на ваших виртуальных машинах. Данные репликации передаются между гипервизорами VMware и хранилищем Azure. Служба Site Recovery не обладает способностью перехватывать эти данные. В службу Site Recovery передаются только метаданные, необходимые для управления репликацией и отработкой отказа.  

Site Recovery имеет сертификаты ISO 27001:2013, 27018, HIPAA, DPA и сейчас проходит проверки на соответствие SOC2 и FedRAMP JAB.

## <a name="managed-disks"></a>Управляемые диски

### <a name="where-does-site-recovery-replicate-data-to"></a>Где Site Recovery реплицировать данные?

Site Recovery реплицирует локальные виртуальные машины VMware и физических серверов для использования управляемых дисков в Azure.
- Сервер обработки Site Recovery записывает журналы репликации для учетной записи хранения кэша в целевом регионе.
- Эти журналы используются для создания точек восстановления на управляемых дисках.
- При отработке отказа, выбранная точка восстановления используется для создания управляемого диска целевой объект.
- Виртуальные машины, которые ранее были реплицированы в учетную запись хранения (до марта 2019 г.) не затрагиваются.


### <a name="can-i-replicate-new-machines-to-storage-accounts"></a>Можно ли реплицировать новых машин в учетных записях хранения?

Нет, марта 2019 г., начиная с версии на портале, вы можете реплицировать только в Azure управляемые диски. 

Репликации новых виртуальных машин в учетную запись хранилища доступен только с помощью PowerShell или REST API (версии 2018-01-10 "или" 2016-08-10).

### <a name="what-are-the-benefits-in-replicating-to-managed-disks"></a>Каковы преимущества в репликации для использования управляемых дисков?

[Узнайте, как](https://azure.microsoft.com/blog/simplify-disaster-recovery-with-managed-disks-for-vmware-and-physical-servers/) Site Recovery упрощает аварийное восстановление с управляемыми дисками.


### <a name="can-i-change-the-managed-disk-type-after-machine-is-protected"></a>Можно ли изменить тип управляемого диска, защищенном компьютере?

Да, вы можете легко [изменить тип управляемого диска](https://docs.microsoft.com/azure/virtual-machines/windows/convert-disk-storage). Перед изменением типа, убедитесь, что вы отменяете URL-адрес SAS для диска, перейдя к ресурсу управляемого диска на портале Azure. В колонке "Обзор" отмените все текущие экспорт. Как только URL-адрес SAS будет отозван, измените тип диска в течение нескольких минут. Тем не менее если изменить тип управляемого диска, дождитесь точек восстановления, новые создаваться с помощью Azure Site Recovery. Используйте новые точки восстановления для тестовой отработки отказа или отработки отказа, в дальнейшем.

### <a name="can-i-switch-replication-from-managed-disks-to-unmanaged-disks"></a>Можно ли перейти с репликации из управляемых дисков для неуправляемых дисков?

Нет, переключение с управляемым и неуправляемым кодом не поддерживается.

### <a name="can-i-exclude-disks-from-replication"></a>Исключение дисков из репликации

### <a name="if-i-use-replication-groups-for-multi-vm-consistency-can-i-add-a-new-vm-to-an-existing-replication-group"></a>Если я использую группы репликации для согласованности нескольких виртуальных машин, могу ли я добавить новую виртуальную машину в имеющуюся группу репликации?
Да, вы можете добавить новые виртуальные машины в имеющуюся группу репликации при включении для них репликации.
- Невозможно добавить виртуальную Машину в существующую группу репликации, после инициации репликации.
- Невозможно создать группу репликации для существующих виртуальных машин.

### <a name="can-i-modify-vms-that-are-replicating-by-adding-or-resizing-disks"></a>Можно ли изменить виртуальные машины, которые реплицируются, путем добавления дисков или изменения их размера?

Для репликации VMware в Azure можно изменить размер диска. Если вы хотите добавить новые диски, вам необходимо добавить диск и повторно включить защиту для виртуальной машины.

### <a name="can-i-migrate-on-premises-machines-to-a-new-vcenter-server-without-impacting-ongoing-replication"></a>Можно ли перенести локальные компьютеры в новый сервер vCenter Server без влияния на выполняющуюся репликацию?
Нет, изменение сервера vCenter или миграция повлияет на текущую репликацию. Необходимо установить Site Recovery с новый сервер vCenter Server и снова включите репликацию для машин.

### <a name="can-i-replicate-to-a-cachetarget-storage-account-which-has-a-vnet-with-azure-storage-firewalls-configured-on-it"></a>Можно ли реплицировать в учетную запись хранения кэша/target, имеющая виртуальной сети (с помощью брандмауэров службы хранилища Azure), настроенные на ней?
Нет, Site Recovery не поддерживает репликацию в хранилище в виртуальной сети.

## <a name="configuration-server"></a>Сервер конфигурации

### <a name="what-does-the-configuration-server-do"></a>Что делает сервер конфигурации?

На сервере конфигурации выполняются локальные компоненты Site Recovery, включая:
- Конфигурация сам сервер, для управления обменом данными между локальной сетью и Azure, а также репликацией данных.
- Сервер обработки, который действует как шлюз репликации. Он получает данные репликации, оптимизирует их путем кэширования, сжатия и шифрования, а затем отправляет эти данные в службу хранилища Azure. Сервер обработки также устанавливает службу Mobility Service на виртуальных машинах Push-уведомлений и выполняет автоматическое обнаружение локальных виртуальных машин VMware.
- Главный целевой сервер, который обрабатывает данные репликации при восстановлении размещения с переносом из Azure.

См. дополнительные сведения о [компонентах и процессах сервера конфигурации](vmware-azure-architecture.md).

### <a name="where-do-i-set-up-the-configuration-server"></a>Где настраивается сервер конфигурации?
Для сервера конфигурации требуется одна высокодоступная локальная виртуальная машина VMware. Для аварийного восстановления физических серверов можно установить сервер конфигурации на физическом компьютере.

### <a name="what-do-i-need-for-the-configuration-server"></a>Что нужно для сервера конфигурации?

Ознакомьтесь с [предварительными требованиями.](vmware-azure-deploy-configuration-server.md#prerequisites)

### <a name="can-i-manually-set-up-the-configuration-server-instead-of-using-a-template"></a>Можно ли вручную настроить сервер конфигурации, не используя шаблон?
Мы рекомендуем вам [Создание виртуальной Машины сервера конфигурации](vmware-azure-deploy-configuration-server.md) с последней версией шаблона OVF. Если для какой-то причине вы не можете, например у вас нет доступа к серверу VMware, вы можете [загрузить](physical-azure-set-up-source.md) файл установки на портале и настроить сервер конфигурации.

### <a name="can-a-configuration-server-replicate-to-more-than-one-region"></a>Может ли сервер конфигурации выполнять репликацию больше, чем в один регион?
№ Для этого необходим сервер конфигурации в каждом регионе.

### <a name="can-i-host-a-configuration-server-in-azure"></a>Можно ли разместить сервер конфигурации в Azure?
Это возможно, однако на виртуальной машине Azure, где запущен сервер конфигурации, необходимо настроить обмен данными с локальными инфраструктурой и виртуальными машинами VMware. Это приводит к увеличению задержки и влияет на текущую репликацию.

### <a name="how-do-i-update-the-configuration-server"></a>Как обновить сервер конфигурации?

[Узнайте,](vmware-azure-manage-configuration-server.md#upgrade-the-configuration-server) как обновить сервер конфигурации.
- Последние сведения об обновлении можно найти на [страницы обновления Azure](https://azure.microsoft.com/updates/?product=site-recovery).
- Последнюю версию можно загрузить с портала. Кроме того, можно напрямую скачать последнюю версию сервера конфигурации с [центра загрузки Майкрософт](https://aka.ms/asrconfigurationserver).
- Если ваша версия — более чем в четырех версиях, предшествующих текущей версии, см. наш [поддерживает инструкцию](https://aka.ms/asr_support_statement) рекомендации по обновлению.

### <a name="should-i-back-up-the-configuration-server"></a>Нужно резервное копирование сервера конфигурации?
Мы рекомендуем создавать плановые резервные копии сервера конфигурации.
- Для успешного восстановления размещения виртуальной Машины, будет восстановлен в исходном должен существовать в базе данных сервера конфигурации.
- Сервер конфигурации должен быть запущен и подключен.
- [Дополнительные сведения](vmware-azure-manage-configuration-server.md) о распространенных задач управления сервером конфигурации.

### <a name="when-im-setting-up-the-configuration-server-can-i-download-and-install-mysql-manually"></a>Можно ли скачать и установить MySQL вручную при настройке сервера конфигурации?

Да. Скачайте среду MySQL и поместите ее в папку **C:\Temp\ASRSetup**. Затем установите ее вручную. После настройки виртуальной машины сервера конфигурации и принятия условий для MySQL в поле **Скачать и установить** будет показано значение **Уже установлено**.

### <a name="can-i-avoid-downloading-mysql-but-let-site-recovery-install-it"></a>Можно ли не скачивать MySQL и разрешить Site Recovery сделать это?

Да. Скачайте установщик MySQL и поместите его в папку **C:\Temp\ASRSetup**.  При настройке виртуальной Машины сервера конфигурации, примите условия и щелкнуть **загрузить и установить**. На портале будет использоваться установщика, вы добавили установить MySQL.
 
### <a name="can-i-use-the-configuration-server-vm-for-anything-else"></a>Можно ли использовать виртуальную машину сервера конфигурации для чего-либо еще?
Нет, виртуальную машину следует использовать только для сервера конфигурации. 

### <a name="can-i-clone-a-configuration-server-and-use-it-for-orchestration"></a>Можно ли клонировать сервер конфигурации и использовать его для оркестрации?
Нет, следует установить сервер новую конфигурацию во избежание возникновения проблемы с регистрацией.

### <a name="can-i-change-the-vault-in-which-the-configuration-server-is-registered"></a>Можно ли изменить хранилище, в котором зарегистрирован сервер конфигурации?
№ После того как хранилище связан с сервером конфигурации, его нельзя изменить. Просмотрите [в этой статье](vmware-azure-manage-configuration-server.md#register-a-configuration-server-with-a-different-vault) Дополнительные сведения о повторной регистрации.

### <a name="can-i-use-the-same-configuration-server-for-disaster-recovery-of-both-vmware-vms-and-physical-servers"></a>Можно ли использовать тот же сервер конфигурации для аварийного восстановления виртуальных машин и физических серверов VMware
Да, но Обратите внимание, что этот физический компьютер может быть выполнена только обратно на виртуальную Машину VMware.

### <a name="where-can-i-download-the-passphrase-for-the-configuration-server"></a>Где можно скачать парольную фразу для сервера конфигурации?
[Узнайте, как](vmware-azure-manage-configuration-server.md#generate-configuration-server-passphrase) скачать парольную фразу.

### <a name="where-can-i-download-vault-registration-keys"></a>Где можно загрузить ключи регистрации хранилища?

В хранилище служб восстановления, нажмите кнопку **серверы конфигурации** в **инфраструктура Site Recovery** > **управление**. Затем в **серверы**выберите **скачать ключ регистрации** скачать файл учетных данных хранилища.

## <a name="process-server"></a>Сервер обработки

### <a name="unable-to-select-process-server-during-enable-replication"></a>Не удалось выбрать сервер обработки во время включения репликации

9.24 версии внесены усовершенствования для обеспечения [рекомендации в продукте](vmware-azure-manage-process-server.md#process-server-selection-guidance) о том, когда для установки сервера обработки масштабирования. Это позволяет избежать регулирования процесса сервера и избежать использования неработоспособное процесса сервера.

### <a name="what-should-i-do-to-obtain-accurate-health-status-of-process-server"></a>Что делать для получения точных работоспособность сервера обработки

Обновите компоненты Site Recovery для [последние версии](service-updates-how-to.md#links-to-currently-supported-update-rollups) (по крайней мере 9.24 или более поздней версии).

## <a name="failover-and-failback"></a>Отработка отказа и восстановление размещения
### <a name="can-i-use-the-process-server-at-on-premises-for-failback"></a>Можно ли использовать сервер обработки в локальной для восстановления размещения?
Настоятельно рекомендуется создание сервера обработки в Azure для восстановления размещения, чтобы избежать задержки передачи данных. Кроме того в случае, если разделены исходной сети виртуальных машин с сетью Azure с выходом на сервере конфигурации, очень важно использовать сервер обработки, созданные в Azure для восстановления размещения.

### <a name="can-i-retain-the-ip-address-on-failover"></a>Можно ли сохранить IP-адрес при отработке отказа?
Да, вы можете сохранить IP-адрес при отработке отказа. Убедитесь, что целевой IP-адрес указан в параметрах «Вычисления и сеть» для виртуальной Машины до отработки отказа. Кроме того завершите работу виртуальных машин во время отработки отказа, чтобы избежать конфликтов IP-адресов во время восстановления размещения.

### <a name="can-i-change-the-target-vm-size-or-vm-type-before-failover"></a>Можно ли изменить размер целевой виртуальной Машины или тип виртуальной Машины до отработки отказа?
Да, вы можете изменить тип и размер виртуальной Машины в любое время перед выполнением отработки отказа на параметры вычислений и сети реплицированной виртуальной машины, на портале.

### <a name="how-far-back-can-i-recover"></a>До какой точки во времени можно выполнить восстановление?
Для репликации VMware в Azure самая давняя точка восстановления — 72 часа.

### <a name="how-do-i-access-azure-vms-after-failover"></a>Как получить доступ к виртуальным машина Azure после отработки отказа?
После отработки отказа доступ к виртуальным машинам Azure можно получить через безопасное подключение к Интернету, VPN-подключение "сеть — сеть" или с помощью Azure ExpressRoute. Для подключения потребуется выполнить несколько подготовительных действий. [Подробнее](site-recovery-test-failover-to-azure.md#prepare-to-connect-to-azure-vms-after-failover)

### <a name="is-failed-over-data-resilient"></a>Устойчивы ли данные, для которых была выполнена отработка отказа?

Среда Azure разработана для обеспечения отказоустойчивости. Служба Site Recovery рассчитана на отработку отказа в дополнительный центр обработки данных Azure в соответствии с соглашением об уровне обслуживания Azure. Когда происходит отработка отказа, Azure обеспечивает хранение метаданных и хранилищ в географическом регионе, выбранном для хранилища.

### <a name="is-failover-automatic"></a>Отработка отказа выполняется автоматически?
[Отработка отказа](site-recovery-failover.md) не выполняется автоматически. Инициировать отработку отказа одним щелчком на портале или воспользоваться [PowerShell](/powershell/module/az.recoveryservices) для активации отработки отказа.

### <a name="can-i-fail-back-to-a-different-location"></a>Можно ли выполнить восстановление размещения в другое расположение?
Да, если отработка отказа выполнена в Azure, можно восстановить размещение в другом расположении, если изначальное расположение недоступно. [Узнайте больше](concepts-types-of-failback.md#alternate-location-recovery-alr).

### <a name="why-do-i-need-a-vpn-or-expressroute-to-fail-back"></a>Почему для восстановления размещения требуется VPN или ExpressRoute?
При восстановлении размещения из Azure данные из Azure копируются обратно на локальную виртуальную машину и требуется закрытый доступ.

### <a name="can-i-resize-the-azure-vm-after-failover"></a>Можно ли изменить размер виртуальной машины Azure после отработки отказа?
Нет, изменить размер или тип целевой виртуальной машины после отработки отказа невозможно.

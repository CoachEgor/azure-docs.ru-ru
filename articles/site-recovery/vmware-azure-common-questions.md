# <a name="common-questions-about-vmware-to-azure-replication"></a>Часто задаваемые вопросы о VMware для репликации в Azure

В этой статье содержатся ответы на часто задаваемые вопросы, которые могут устанавливаться во время развертывания аварийного восстановления локальных виртуальных машин VMware (ВМ) в Azure.

## <a name="general"></a>Общие сведения

### <a name="what-do-i-need-for-vmware-vm-disaster-recovery"></a>Что нужно для аварийного восстановления виртуальных Машин VMware?

[Узнайте о компонентах, используемых](vmware-azure-architecture.md) в аварийное восстановление виртуальных машин VMware.

### <a name="can-i-use-site-recovery-to-migrate-vmware-vms-to-azure"></a>Можно ли использовать Site Recovery для переноса виртуальных машин VMware в Azure?

Да. Помимо использования Site Recovery, чтобы настроить полное аварийное восстановление для виртуальных машин VMware, можно также использовать Site Recovery для переноса локальных виртуальных машин VMware в Azure. В этом случае репликация локальных виртуальных машин VMware в службу хранилища Azure. Затем выполняется отработка отказа из локальной среды в Azure. После отработки отказа приложения и рабочие нагрузки доступны и выполняются на виртуальных машинах Azure. Процесс аналогичен настройке полное аварийное восстановление, за исключением того, что при миграции невозможно восстановить размещение из Azure.

### <a name="does-my-azure-account-need-permissions-to-create-vms"></a>Требуются ли учетной записи Azure разрешения на создание виртуальных машин?

Если вы являетесь администратором подписки, у вас есть необходимые разрешения на репликацию. Если вы не являетесь администратором, необходимо указать разрешения для выполнения этих действий.

- Создайте виртуальную Машину Azure в ресурсе группы и виртуальные сети, которые были заданы при настройке Site Recovery.
- Запись в выбранной учетной записи хранения или управляемого диска, в зависимости от конфигурации.

[Дополнительные сведения](site-recovery-role-based-linked-access-control.md#permissions-required-to-enable-replication-for-new-virtual-machines) о требуемых разрешениях.

### <a name="what-applications-can-i-replicate"></a>Какие приложения можно реплицировать?

Вы можете реплицировать любое приложение или рабочую нагрузку, запущенную на виртуальной Машине VMware, который соответствует [требования к репликации](vmware-physical-azure-support-matrix.md#replicated-machines).

- Site Recovery поддерживает репликацию уровня приложения, чтобы приложения могли отработки отказа и выполнено восстановление размещения программируемого состояния.
- Site Recovery интегрируется с приложениями Майкрософт, таких как SharePoint, Exchange, Dynamics, SQL Server и Active Directory. Он также тесно сотрудничает с ведущими поставщиками, включая Oracle, SAP, IBM и Red Hat.

[Подробнее](site-recovery-workload.md) о защите рабочей нагрузки.

### <a name="can-i-use-a-guest-os-server-license-on-azure"></a>Можно использовать гостевой ОС серверная лицензия на Azure?

Да, можно использовать клиенты Microsoft Software Assurance [Преимущество гибридного использования Azure](https://azure.microsoft.com/pricing/hybrid-benefit/) чтобы сэкономить на стоимости лицензирования компьютеров Windows Server, которые переносятся в Azure, или использовать Azure для аварийного восстановления.

## <a name="security"></a>Безопасность

### <a name="what-access-to-vmware-servers-does-site-recovery-need"></a>Какой доступ к серверам VMware требуется Site Recovery?

Site Recovery требуется доступ к серверам VMware, чтобы:

- Настройка виртуальной Машины VMware под управлением сервера конфигурации Site Recovery.
- автоматически обнаруживать виртуальные машины для репликации.

### <a name="what-access-to-vmware-vms-does-site-recovery-need"></a>Какой доступ к виртуальным машинам VMware требуется Site Recovery?

- Чтобы реплицировать, виртуальной Машины VMware требуется Site Recovery Mobility service установлена и запущена. Можно развернуть средство вручную, или можно указать, что Site Recovery выполнить принудительную установку службы при включении репликации для виртуальной Машины.
- Во время репликации виртуальные машины обмениваются данными с Site Recovery следующим образом:
    - Виртуальные машины, обмениваться данными с сервером конфигурации через порт HTTPS 443 для управления репликацией.
    - Виртуальные машины отправляют данные репликации на сервер обработки через порт HTTPS 9443. (Этот параметр можно изменить.)
    - если включить согласованность между виртуальными машинами, они будут обмениваться данными друг с другом через порт 20004.

### <a name="is-replication-data-sent-to-site-recovery"></a>Отправляются ли данные репликации в Site Recovery?

Нет, Site Recovery не перехватывает реплицируемые данные и не получает сведения о компонентах, запущенных на виртуальных машинах. Данные репликации передаются между гипервизорами VMware и службы хранилища Azure. Служба Site Recovery не обладает способностью перехватывать эти данные. В службу Site Recovery передаются только метаданные, необходимые для управления репликацией и отработкой отказа.  

Site Recovery сертифицирован для ISO 27001: 2013 и 27018, HIPAA и DPA. Он находится в процессе SOC2 и FedRAMP JAB.

## <a name="managed-disks"></a>Управляемые диски

### <a name="where-does-site-recovery-replicate-data-to"></a>Где Site Recovery реплицировать данные?

Site Recovery реплицирует локальные виртуальные машины VMware и физических серверов для использования управляемых дисков в Azure.

- Сервер обработки Site Recovery записывает журналы репликации для учетной записи хранения кэша в целевом регионе.
- Эти журналы используются для создания точек восстановления на дисках, управляемое Azure, которые имеют префикс **asrseeddisk**.
- При отработке отказа, выбранная точка восстановления используется для создания нового управляемого диска целевой объект. Этот управляемый диск подключен к виртуальной Машине в Azure.
- Виртуальные машины, которые ранее были реплицированы в учетную запись хранения (до марта 2019 г.) не затрагиваются.

### <a name="can-i-replicate-new-machines-to-storage-accounts"></a>Можно ли реплицировать новых машин в учетных записях хранения?

№ Начиная с марта 2019 г., на портале Azure можно реплицировать только в Azure управляемые диски.

Репликация новых виртуальных машин в учетную запись хранилища доступна только с помощью PowerShell или REST API (версии 2018-01-10 "или" 2016-08-10).

### <a name="what-are-the-benefits-of-replicating-to-managed-disks"></a>Каковы преимущества репликации для использования управляемых дисков?

[Узнайте, как](https://azure.microsoft.com/blog/simplify-disaster-recovery-with-managed-disks-for-vmware-and-physical-servers/) Site Recovery упрощает аварийное восстановление с управляемыми дисками.

### <a name="can-i-change-the-managed-disk-type-after-a-machine-is-protected"></a>Можно ли изменить тип управляемого диска, защищенном компьютере?

Да, вы можете легко [изменить тип управляемого диска](https://docs.microsoft.com/azure/virtual-machines/windows/convert-disk-storage) для выполняющаяся. Прежде чем изменить тип, убедитесь, что без подписанного URL-адрес создается на управляемом диске:

1. Перейдите к **управляемого диска** ресурсов на портале Azure и проверьте, есть ли у вас баннер URL-адреса подписи общего доступа **Обзор** колонке.
1. Если присутствует баннер, выберите его, чтобы отменить текущие экспорт.
1. Измените тип диска в течение нескольких минут. Если изменить тип управляемого диска, дождитесь точек восстановления, новые создаваться с помощью Azure Site Recovery.
1. Используйте новые точки восстановления для тестовой отработки отказа или отработки отказа в будущем.

### <a name="can-i-switch-replication-from-managed-disks-to-unmanaged-disks"></a>Можно ли перейти с репликации из управляемых дисков для неуправляемых дисков?

№ Переключение с управляемым и неуправляемым кодом не поддерживается.

[Дополнительные сведения](vmware-physical-azure-support-matrix.md#replicated-machines) о требованиях к поддержке для виртуальных машин VMware и физических серверов.

### <a name="how-often-can-i-replicate-to-azure"></a>Как часто можно выполнять репликацию в Azure?

Этот процесс является непрерывным при репликации виртуальных машин VMware в Azure.

### <a name="can-i-extend-replication"></a>Можно ли расширить репликацию?

Расширенная репликация и цепочка репликации не поддерживаются. Запросите эту функцию на [форуме отзывов и предложений](https://feedback.azure.com/forums/256299-site-recovery/suggestions/6097959).

### <a name="can-i-do-an-offline-initial-replication"></a>Можно ли выполнить начальную репликацию в автономном режиме?

Репликация в автономном режиме не поддерживается. Запросите эту функцию на [форуме отзывов и предложений](https://feedback.azure.com/forums/256299-site-recovery/suggestions/6227386-support-for-offline-replication-data-transfer-from).

### <a name="what-is-asrseeddisk"></a>Что такое asrseeddisk?

Для каждого диска источника данные реплицируются в управляемый диск в Azure. Этот диск имеет префикс **asrseeddisk**. Он сохраняет копию исходного диска и все моментальные снимки точки восстановления.

### <a name="can-i-exclude-disks-from-replication"></a>Исключение дисков из репликации

Да, вы можете исключить диски.

### <a name="can-i-replicate-vms-that-have-dynamic-disks"></a>Можно ли реплицировать виртуальные машины с динамическими дисками?

Динамические диски могут быть реплицированы. Диск операционной системы должен представлять собой базовый диск.

### <a name="if-i-use-replication-groups-for-multi-vm-consistency-can-i-add-a-new-vm-to-an-existing-replication-group"></a>Если я использую группы репликации для согласованности нескольких виртуальных машин, могу ли я добавить новую виртуальную машину в имеющуюся группу репликации?

Да, вы можете добавить новые виртуальные машины в имеющуюся группу репликации при включении для них репликации. Тем не менее:

- Невозможно добавить виртуальную Машину в существующую группу репликации, после начала репликации.
- Невозможно создать группу репликации для существующих виртуальных машин.

### <a name="can-i-modify-vms-that-are-replicating-by-adding-or-resizing-disks"></a>Можно ли изменить виртуальные машины, которые реплицируются, путем добавления дисков или изменения их размера?

Для репликации VMware в Azure можно изменить размер диска. Если вы хотите добавить новые диски, необходимо добавить диск и повторно включить защиту виртуальной Машины.

### <a name="can-i-migrate-on-premises-machines-to-a-new-vcenter-server-without-impacting-ongoing-replication"></a>Можно ли перенести локальные компьютеры в новый сервер vCenter Server без влияния на выполняющуюся репликацию?

№ Изменение VMware Vcenter или миграции повлияет на текущую репликацию. Настройте Site Recovery с новый сервер vCenter Server и снова включите репликацию для машин.

### <a name="can-i-replicate-to-a-cache-or-target-storage-account-that-has-a-virtual-network-with-azure-firewalls-configured-on-it"></a>Можно реплицировать в учетную запись хранилища кэша или целевой объект, имеющий виртуальной сети (с помощью брандмауэров Azure), настроенные на ней?

Нет, Site Recovery не поддерживает репликацию в хранилище Azure в виртуальных сетях.

## <a name="configuration-server"></a>Сервер конфигурации

### <a name="what-does-the-configuration-server-do"></a>Что делает сервер конфигурации?

На сервере конфигурации выполняются локальные компоненты Site Recovery, включая:

- Сам сервер конфигурации. Сервер управления обменом данными между компонентами локальной средой и Azure, а также репликацией данных.
- Сервер обработки, который выступает в качестве шлюза репликации. Этот сервер:
    1. Получает данные репликации.
    2. оптимизирует данные с помощью кэширования, сжатия и шифрования;
    3. Отправляет данные в службу хранилища Azure.
  Сервер обработки также не принудительной установки службы Mobility Service на виртуальных машинах и выполняет автоматическое обнаружение локальных виртуальных машин VMware.
- Главный целевой сервер, который обрабатывает данные репликации при восстановлении размещения из Azure.

См. дополнительные сведения о [компонентах и процессах сервера конфигурации](vmware-azure-architecture.md).

### <a name="where-do-i-set-up-the-configuration-server"></a>Где настраивается сервер конфигурации?

Вам потребуется VMware единый, высокой доступности в локальной виртуальной Машины сервера конфигурации. Для аварийного восстановления физических серверов установки сервера конфигурации на физическом компьютере.

### <a name="what-do-i-need-for-the-configuration-server"></a>Что нужно для сервера конфигурации?

Ознакомьтесь с [предварительными требованиями.](vmware-azure-deploy-configuration-server.md#prerequisites)

### <a name="can-i-manually-set-up-the-configuration-server-instead-of-using-a-template"></a>Можно ли вручную настроить сервер конфигурации, не используя шаблон?

Мы рекомендуем вам [Создание виртуальной Машины сервера конфигурации](vmware-azure-deploy-configuration-server.md) с помощью последней версии шаблона Open виртуальной машины Format (OVF). Если шаблон не может использовать (например, если у вас нет доступа к серверу VMware), [загрузить](physical-azure-set-up-source.md) файл установки на портале и настроить сервер конфигурации.

### <a name="can-a-configuration-server-replicate-to-more-than-one-region"></a>Может ли сервер конфигурации выполнять репликацию больше, чем в один регион?

№ Для репликации в нескольких регионах, необходим сервер конфигурации в каждом регионе.

### <a name="can-i-host-a-configuration-server-in-azure"></a>Можно ли разместить сервер конфигурации в Azure?

Несмотря на то, что это возможно, виртуальной Машине Azure под управлением сервера конфигурации необходимо настроить обмен данными с вашей локальной инфраструктуре VMware и виртуальные машины. Это взаимодействие приводит к увеличению задержки и влияет на текущую репликацию.

### <a name="how-do-i-update-the-configuration-server"></a>Как обновить сервер конфигурации?

[Узнайте,](vmware-azure-manage-configuration-server.md#upgrade-the-configuration-server) как обновить сервер конфигурации.

- Последние сведения об обновлении можно найти на [страницы обновления Azure](https://azure.microsoft.com/updates/?product=site-recovery).
- Последнюю версию можно загрузить с портала. Или можно загрузить последнюю версию сервера конфигурации непосредственно из [центра загрузки Майкрософт](https://aka.ms/asrconfigurationserver).
- Если ваша версия — более чем в четырех версиях, предшествующих текущей версии, см. в разделе [поддерживает инструкцию](https://aka.ms/asr_support_statement) рекомендации по обновлению.

### <a name="should-i-back-up-the-configuration-server"></a>Нужно резервное копирование сервера конфигурации?

Мы рекомендуем создавать плановые резервные копии сервера конфигурации.

- Для успешного восстановления размещения виртуальной Машины, будет восстановлен в исходном должен существовать в базе данных сервера конфигурации.
- Сервер конфигурации должен быть запущен и подключен.
- [Дополнительные сведения](vmware-azure-manage-configuration-server.md) о распространенных задач управления сервером конфигурации.

### <a name="when-im-setting-up-the-configuration-server-can-i-download-and-install-mysql-manually"></a>Можно ли скачать и установить MySQL вручную при настройке сервера конфигурации?

Да. Скачать MySQL и поместите его в папку C:\Temp\ASRSetup. Затем установите драйвер вручную. После настройки виртуальной машины сервера конфигурации и принятия условий для MySQL в поле **Скачать и установить** будет показано значение **Уже установлено**.

### <a name="can-i-avoid-downloading-mysql-but-let-site-recovery-install-it"></a>Можно ли не скачивать MySQL и разрешить Site Recovery сделать это?

Да. Загрузите установщик MySQL и поместите его в папку C:\Temp\ASRSetup. При настройке виртуальной Машины сервера конфигурации, примите условия и выберите **загрузить и установить**. На портале будет использоваться установщик, который вы добавили установить MySQL.

### <a name="can-i-use-the-configuration-server-vm-for-anything-else"></a>Можно ли использовать виртуальную машину сервера конфигурации для чего-либо еще?

№ Используйте виртуальную Машину только для сервера конфигурации.

### <a name="can-i-clone-a-configuration-server-and-use-it-for-orchestration"></a>Можно ли клонировать сервер конфигурации и использовать его для оркестрации?

№ Настройте сервер новую конфигурацию, чтобы избежать проблемы с регистрацией.

### <a name="can-i-change-the-vault-in-which-the-configuration-server-is-registered"></a>Можно ли изменить хранилище, в котором зарегистрирован сервер конфигурации?

№ После того как хранилище связан с сервером конфигурации, его нельзя изменить. [Узнайте,](vmware-azure-manage-configuration-server.md#register-a-configuration-server-with-a-different-vault) о регистрации сервера конфигурации в другом хранилище.

### <a name="can-i-use-the-same-configuration-server-for-disaster-recovery-of-both-vmware-vms-and-physical-servers"></a>Можно использовать тот же сервер конфигурации для аварийного восстановления виртуальных машин VMware и физических серверов

Да, но Обратите внимание на то, что физический компьютер может быть выполнена только с виртуальной Машины VMware.

### <a name="where-can-i-download-the-passphrase-for-the-configuration-server"></a>Где можно скачать парольную фразу для сервера конфигурации?

[Узнайте,](vmware-azure-manage-configuration-server.md#generate-configuration-server-passphrase) загрузке парольную фразу.

### <a name="where-can-i-download-vault-registration-keys"></a>Где можно загрузить ключи регистрации хранилища?

В хранилище служб восстановления, выберите **серверы конфигурации** в **инфраструктура Site Recovery** > **управление**. Затем в **серверы**выберите **скачать ключ регистрации** скачать файл учетных данных хранилища.

## <a name="process-server"></a>Сервер обработки

### <a name="why-am-i-unable-to-select-the-process-server-when-i-enable-replication"></a>Почему мне не удается выбрать сервер обработки, при включении репликации?

Обновления версий 9,24 и более поздних теперь отображает [исправность сервера обработки при включении репликации](vmware-azure-enable-replication.md#enable-replication). Эта функция позволяет избежать регулирования сервер обработки и свести к минимуму влияние неработоспособные серверы.

### <a name="how-do-i-update-the-process-server-to-version-924-or-later-for-accurate-health-information"></a>Как обновить сервер обработки до версии 9.24 или более поздней сведения о точной работоспособности?

Начиная с версии [версии 9,24](service-updates-how-to.md#links-to-currently-supported-update-rollups), были добавлены дополнительные оповещения о работоспособности сервера обработки. [Обновите компоненты Site Recovery до версии 9.24 или более поздней версии](service-updates-how-to.md#links-to-currently-supported-update-rollups) , чтобы все оповещений.

## <a name="failover-and-failback"></a>Отработка отказа и восстановление размещения

### <a name="can-i-use-the-on-premises-process-server-for-failback"></a>Можно использовать локальный сервер обработки для восстановления размещения?

Настоятельно рекомендуется создать сервер обработки в Azure для восстановления размещения, чтобы избежать задержки передачи данных. Кроме того в случае, если разделены исходной сети виртуальных машин с сетью Azure с выходом на сервере конфигурации, очень важно использовать сервер обработки, созданные в Azure для восстановления размещения.

### <a name="can-i-keep-the-ip-address-on-failover"></a>Можно сохранить IP-адрес при отработке отказа?

Да, вы можете сохранить IP-адрес при отработке отказа. Убедитесь, что целевой IP-адрес в **вычисления и сеть** параметры для виртуальной Машины до отработки отказа. Кроме того завершите работу виртуальных машин во время отработки отказа, чтобы избежать конфликтов IP-адресов во время восстановления размещения.

### <a name="can-i-change-the-target-vm-size-or-vm-type-before-failover"></a>Можно ли изменить размер целевой виртуальной Машины или тип виртуальной Машины до отработки отказа?

Да, вы можете изменить тип и размер виртуальной Машины в любое время перед выполнением отработки отказа. На портале, использовать **вычисления и сеть** параметры для реплицируемой виртуальной Машины.

### <a name="how-far-back-can-i-recover"></a>До какой точки во времени можно выполнить восстановление?

VMware в Azure самая старая точка восстановления, которые можно использовать — 72 часа.

### <a name="how-do-i-access-azure-vms-after-failover"></a>Как получить доступ к виртуальным машина Azure после отработки отказа?

После отработки отказа виртуальных машин Azure доступны через безопасное Интернет-подключение, через виртуальную частную сеть сеть или через Azure ExpressRoute. Чтобы подключиться, необходимо подготовить следующее. [Узнайте больше](site-recovery-test-failover-to-azure.md#prepare-to-connect-to-azure-vms-after-failover).

### <a name="is-failed-over-data-resilient"></a>Устойчив данных выполнена отработка отказа?

Среда Azure разработана для обеспечения отказоустойчивости. Site Recovery рассчитана на отработку отказа в дополнительный ЦОД Azure, требуемые для Azure соглашение уровня обслуживания (SLA). При отработке отказа, мы обеспечивает хранение метаданных и хранилищ в географическом регионе, выбранном для хранилища.

### <a name="is-failover-automatic"></a>Отработка отказа выполняется автоматически?

[Отработка отказа](site-recovery-failover.md) не выполняется автоматически. Запуск отработки отказа, выбрав один вариант в на портале или воспользоваться [PowerShell](/powershell/module/az.recoveryservices) для активации отработки отказа.

### <a name="can-i-fail-back-to-a-different-location"></a>Можно ли выполнить восстановление размещения в другое расположение?

Да. При отработке отказа в Azure, можно выполнить обратно в другом месте Если изначальное недоступен. [Узнайте больше](concepts-types-of-failback.md#alternate-location-recovery-alr).

### <a name="why-do-i-need-a-vpn-or-expressroute-with-private-peering-to-fail-back"></a>Зачем мне VPN или ExpressRoute с частным пирингом для восстановления размещения?

При восстановлении размещения из Azure данных из Azure копируется обратно на локальной виртуальной Машины, и закрытый доступ является обязательным.

### <a name="can-i-resize-the-azure-vm-after-failover"></a>Можно ли изменить размер виртуальной машины Azure после отработки отказа?

Нет, не может изменить размер или тип целевой виртуальной Машине после отработки отказа.

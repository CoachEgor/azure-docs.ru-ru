# <a name="exclude-disks-from-vmware-vm-replication-to-azure"></a>Исключить диски из репликации VMware VM в Azure

В этой статье описывается, как исключить диски при репликации VM VMw в Azure для аварийного восстановления. Возможно, по ряду причин можно исключить диски из репликации:

- Убедитесь, что неважные данные, вспененные на исключенном диске, не будут реплицироваться.
- Оптимизируйте потребляемую пропускную способность репликации или целевые ресурсы, исключив диски, которые не нужно реплицировать.
- Сохраняйте ресурсы хранения и сети, не реплицируя данные, которые вам не нужны.

Прежде чем исключить диски из репликации:

- Дополнительные сведения об исключении дисков см.в [этой статье](exclude-disks-replication.md).
- Просмотрите [типичные сценарии исключений](exclude-disks-replication.md#typical-scenarios) и [примеры,](exclude-disks-replication.md#example-1-exclude-the-sql-server-tempdb-disk) которые показывают, как исключение диска влияет на репликацию, сбой и сбой.

## <a name="before-you-start"></a>Перед началом работы

 Прежде чем начать, обратите внимание на следующее:

- **Репликация**: По умолчанию все диски на машине реплицируются.
- **Тип диска**: Только основные диски могут быть исключены из репликации. Вы не можете исключать диски операционной системы или динамические диски.
- **Служба мобильности**: Чтобы исключить диск из репликации, необходимо вручную установить службу Мобильности на машине, прежде чем включить репликацию. Вы не можете использовать установку push, так как этот метод устанавливает службу мобильности на VM только после включения репликации.  
- **Добавить/удалить/исключить диски:** После включения репликации нельзя добавлять/удалять/исключать диски для репликации. Если вы хотите добавить/удалить или исключить диски, необходимо отключить защиту машины, а затем включить ее снова.
- **Failover**: После неудачи, если не удалось над приложениями необходимо исключены диски для того, чтобы работать, необходимо создать эти диски вручную. Кроме того, можно интегрировать автоматизацию Azure в план восстановления, чтобы создать диск во время сбоя машины.
- **Failback-Windows**: Когда вы не возвращаетесь на свой сайт после сбоя, диски Windows, которые вы создаете вручную в Azure, не выходят из строя. Например, если вы не сможете выполнить три диска и создать два диска непосредственно на VMs Azure, только три диска, которые были несуминуты, будут возвращены.
- **Failback-Linux:** Для отказов машин Linux диски, которые вы создаете вручную в Azure, не возвращаются назад. Например, если вы не сможете выполнить три диска и создать два диска непосредственно на VMs Azure, все пять из них будут возвращены. Нельзя исключать диски, созданные вручную в отказе или в повторной защите VMs.



## <a name="exclude-disks-from-replication"></a>Исключение дисков из репликации

1. Когда вы [включите репликацию](site-recovery-hyper-v-site-to-azure.md) для VMware VM, после выбора vMs, которые вы хотите воспроизвести, на странице**свойств** >  **включения репликации** > **настраивайте свойства,** просмотрите **диски для копирования** столбца. По умолчанию все диски выбираются для репликации.
2. Если вы не хотите воспроизводить определенный диск, в **дисках, чтобы воспроизвести** четкий выбор для любых дисков, которые вы хотите исключить.

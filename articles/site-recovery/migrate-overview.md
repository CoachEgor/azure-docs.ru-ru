# <a name="about-migration"></a>Общие сведения о миграции

Используйте службу "миграция Azure", чтобы перенести виртуальные машины и серверы в Azure, а не в службу Azure Site Recovery. Дополнительные [сведения](../migrate/migrate-services-overview.md) о службе "миграция Azure".


## <a name="why-use-azure-migrate"></a>Зачем использовать службу "Миграция Azure"?

Использование функции "миграция Azure" для миграции предоставляет ряд преимуществ.
 
 
- Служба "миграция Azure" предоставляет централизованный центр для обнаружения, оценки и миграции в Azure.
- Использование службы "миграция Azure" обеспечивает взаимодействие и дальнейшее расширение возможностей с помощью средств миграции Azure, других служб Azure и сторонних средств.
- Миграция Azure: средство миграции сервера предназначено для миграции сервера в Azure. Она оптимизирована для миграции. Вам не нужно изучать основные понятия и сценарии, не относящиеся непосредственно к миграции. 
- Плата за использование средств для миграции в течение 180 дней не взимается, с момента запуска репликации для виртуальной машины. Это дает вам время для завершения миграции. Вы платите только за ресурсы хранилища и сети, используемые при репликации, а также для расходов на вычисление, потребляемое во время миграции тестов.
- Для виртуальных машин VMware служба "миграция Azure" обеспечивает миграцию без агента в дополнение к миграции на основе агентов.
- Мы раснесем приоритеты новых функций миграции для средства миграции Azure: только для сервера.

## <a name="when-to-use-site-recovery"></a>Когда следует использовать Site Recovery?

Следует использовать Site Recovery:

- Для аварийного восстановления локальных компьютеров в Azure.
- Для аварийного восстановления виртуальных машин Azure между регионами Azure.

Хотя мы рекомендуем использовать миграцию Azure для переноса локальных серверов в Azure, если вы уже начали переход с Site Recovery, можно продолжить его использование для завершения миграции.

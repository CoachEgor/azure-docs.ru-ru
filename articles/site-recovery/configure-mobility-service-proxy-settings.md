# <a name="configure-mobility-service-proxy-settings-for-azure-to-azure-disaster-recovery"></a>Настройка настройки прокси-серверов службы мобильности для Azure для аварийного восстановления

В этой статье содержатся рекомендации по настройке конфигураций сетей на целевой виртуальной машине Azure (VM) при репликации и восстановлении виртуальных технологий Azure из одного региона в другой с помощью [Azure Site Recovery.](site-recovery-overview.md)

Цель юрисконсульта заключается в том, чтобы предусмотреть шаги по настройке параметров прокси для службы восстановления мобильности сайтов Azure в сценарии аварийного восстановления Azure. 

Прокси - это сетевые шлюзы, которые позволяют/отпускать сетевые соединения в конечные точки. Обычно прокси — это машина за пределами клиентской машины, которая пытается получить доступ к конечным точкам сети. Список обхода позволяет клиенту делать соединения непосредственно к конечным точкам, не проходя через прокси. Имя пользователя и пароль могут быть дополнительно установлены для прокси-серверами сетевыми админами, чтобы только проверенные клиенты могли использовать прокси. 

## <a name="before-you-start"></a>Перед началом работы

Узнайте, как Site Recovery обеспечивает аварийное восстановление для [этого сценария](azure-to-azure-architecture.md).
Понимание [руководства по сети](azure-to-azure-about-networking.md) при репликации и восстановлении vMs Azure из одного региона в другой с помощью [восстановления сайта Azure.](site-recovery-overview.md)
Убедитесь, что ваш прокси настроен соответствующим образом в зависимости от потребностей вашей организации.

## <a name="configure-the-mobility-service"></a>Настройка службы мобильности

Mobility Service поддерживает только непроверенные прокси- и службы. Он предоставляет два способа ввода прокси-детали для связи с конечными точками восстановления сайта. 

### <a name="method-1-auto-detection"></a>Метод 1: Автоматическое обнаружение

Служба мобильности автоматически обнаруживает настройки прокси из настроек среды или IE Настройки (только Windows) во время репликации включить. 

- ОС Windows: Во время репликации включить, Мобильность служба обнаруживает настройки прокси, как настроены в Internet Explorer для локального пользователя системы. Чтобы настроить прокси для учетной записи Локальной системы, администратор может использовать psexec для запуска запроса команды, а затем Internet Explorer. 
- ОС Windows: Настройки прокси настроены как переменные среды http_proxy и no_proxy. 
- Linux OS: Настройки прокси настроены в /etc/profile или /etc/environment в качестве переменных среды http_proxy, no_proxy. 
- Автоматическое обнаружение параметров прокси сохраняются в прокси-файле proxyInfo.conf службы мобильности 
- Местоположение по умолчанию ProxyInfo.conf 
    - Окна: C:'ProgramData-Microsoft Azure Восстановление сайта/Config-ProxyInfo.conf 
    - Linux: /usr/local/InMage/config/ProxyInfo.conf


### <a name="method-2-provide-custom-application-proxy-settings"></a>Метод 2: Предоставьте настройки прокси-серверов пользовательского приложения

В этом случае клиент предоставляет настройки прокси-сервера пользовательского приложения в файле конфигурации MobilityInfo.conf. Этот метод позволяет клиентам предоставлять прокси только для службы мобильности или другой прокси для службы восстановления azure, чем прокси (или нет прокси) для остальной части приложений на машине.

## <a name="proxy-template"></a>Шаблон прокси
ProxyInfo.conf содержит следующий шаблон «прокси» Адрес:http://1.2.3.4 Порт-5678 BypassList-hypervrecoverymanager.windowsazure.com,login.microsoftonline.com,blob.core.windows.net. BypassList не поддерживает подстановочные знаки, такие как «к.windows.net», но предоставление windows.net достаточно хорошо, чтобы обойти.

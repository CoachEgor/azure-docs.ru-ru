# <a name="configure-and-manage-replication-policies-for-vmware-disaster-recovery"></a>Настройка и управление политиками репликации для аварийного восстановления VMware

Из этой статьи вы узнаете, как настроить политику репликации при репликации виртуальных машин VMware в Azure с помощью [Azure Site Recovery](site-recovery-overview.md).

## <a name="create-a-policy"></a>Создание политики

1. Выберите **Управляйте** > **инфраструктурой восстановления сайта.**
2. В разделе **For VMware and Physical machines** (Для виртуальных машин VMware и физических компьютеров) выберите **Политики репликации**.
3. Щелкните **+ Политика репликации** и укажите имя политики.
4. В **пороге RPO**укажите лимит RPO. Если показатель непрерывной репликации превышает это значение, создаются оповещения.
5. В поле **Хранение точки восстановления** укажите период хранения каждой точки восстановления (в часах). Защищенные компьютеры будут восстанавливаться до любой точки в пределах этого периода. Для компьютеров, реплицируемых в хранилище уровня "Премиум", поддерживается хранение до 24 часов. Для стандартного хранилища поддерживается хранение до 72 часов.
6. В **частоте снимков, совместимых с приложением,** выберите из выпадения, как часто (в часах) должны быть созданы точки восстановления, содержащие последовательные моментальные снимки приложения. Если вы хотите отключить генерацию точек согласованности приложений, выберите значение "Off" при выпадении.
7. Нажмите кнопку **ОК**. Политика будет создана в течение 30–60 секунд.

Когда вы создаете политику репликации, по умолчанию автоматически создается соответствующая политика для восстановления размещения с суффиксом failback. Чтобы изменить созданную политику, выберите ее и щелкните **Изменить параметры**.

## <a name="associate-a-configuration-server"></a>Связывание сервера конфигурации

Свяжите политику репликации с локальным сервером конфигурации.

1. Щелкните **Связать** и выберите сервер конфигурации.

    ![Связывание сервера конфигурации](./media/vmware-azure-set-up-replication/associate1.png)
2. Нажмите кнопку **ОК**. Связь сервера конфигурации с политикой будет задана в течение 1–2 минут.

    ![Связывание сервера конфигурации](./media/vmware-azure-set-up-replication/associate2.png)

## <a name="edit-a-policy"></a>Изменение политики

После создания политики репликации можно изменить политику репликации.

- Изменения в политике применяются ко всем машинам, использующим политику.
- Если вы хотите связать реплицированные машины с другой политикой репликации, необходимо отключить и обеспечить защиту соответствующих машин.

Изыскните политику следующим образом:
1. Выберите **Политики** >  > **репликации инфраструктуры восстановления инфраструктуры****сайта.**
2. Выберите политику репликации, которые вы хотите изменить.
3. Нажмите **настройки отстекать**и по мере необходимости обновите часы удержания порога/точки восстановления RPO/приложение-согласованное частотное поле моментального снимка.
4. Если вы хотите отключить генерацию точек согласованности приложений, выберите значение "Off" при выпадении **частоты снимка приложения.**
5. Нажмите **Сохранить**. Политика должна быть обновлена через 30-60 секунд.



## <a name="disassociate-or-delete-a-replication-policy"></a>Отмена связи или удаление политики репликации

1. Выберите политику репликации.
    а. Чтобы отменить связь между политикой и сервером конфигурации, убедитесь, что реплицируемые виртуальные машины не используют эту политику. Затем щелкните **Отменить связь**.
    b. Чтобы удалить политику, убедитесь что она не связана с сервером конфигурации. Затем нажмите **Удалить**. Удаление занимает 30–60 секунд.
2. Нажмите кнопку **ОК**.

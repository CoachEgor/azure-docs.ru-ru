# <a name="about-disaster-recovery-for-on-premises-apps"></a>Сведения об аварийном восстановлении для локальных приложений

В этой статье описаны рабочие нагрузки и приложения, которые можно защитить для аварийного восстановления с помощью службы [восстановления сайта Azure.](site-recovery-overview.md)

## <a name="overview"></a>Обзор

Организации нуждаются в стратегии непрерывности бизнеса и аварийного восстановления (BCDR), чтобы сохранить рабочие нагрузки и данные в безопасности и в запланированные и незапланированные простои. И, восстановить к обычным условиям труда.

Site Recovery — это служба Azure, которая помогает реализовать стратегию BCDR. Она позволяет выполнять репликацию в облако или на дополнительный сайт с учетом особенностей приложения. Вы можете использовать восстановление сайта для управления репликацией, выполнения тестирования аварийного восстановления и выполнения сбоев и сбоев. Ваши приложения могут работать на компьютерах на базе Windows или Linux, физических серверах, VMware или Hyper-V.

Восстановление сайта интегрируется с такими приложениями Майкрософт, как SharePoint, Exchange, Dynamics, S'L Server и Active Directory. Корпорация Майкрософт тесно сотрудничает с ведущими поставщиками, включая Oracle, SAP и Red Hat. Решения репликации можно настроить для нескольких приложений.

## <a name="why-use-site-recovery-for-application-replication"></a>Причины использования Site Recovery для репликации приложений

Site Recovery помогает реализовать защиту и восстановление на уровне приложения следующим образом:

- App-agnostic и обеспечивает репликацию для любых рабочих нагрузок, работающих на поддерживаемой машине.
- Почти синхронная репликация с целями пунктов восстановления (RPO) всего за 30 секунд для удовлетворения потребностей наиболее важных бизнес-приложений.
- Моментальные снимки с согласованием для одноуровневых и многоуровневых приложений.
- Интеграция с s'L Server AlwaysOn и партнерство с другими технологиями репликации на уровне приложений. Например, репликация Active Directory, группы доступности баз данных S'L alwaysOn и Exchange Database (DAG).
- Гибкие планы восстановления, которые позволяют восстановить весь стек приложения одним щелчком мыши, а также включить внешние скрипты и ручные действия в план.
- Расширенное управление сетью в восстановлении сайта и Azure для упрощения требований к сети приложений. Управление сетью, такие как возможность резервирования IP-адресов, настройка балансировки нагрузки и интеграция с Менеджером движения Azure для переключения сетей с низким временем восстановления (RTO).
- обширная библиотека автоматизации, содержащая готовые к работе и учитывающие особенности приложений сценарии, которые можно скачать и интегрировать с планами восстановления.

## <a name="replicate-active-directory-and-dns"></a>Репликация Active Directory и DNS

Инфраструктура Active Directory и DNS необходима для большинства корпоративных приложений. Во время аварийного восстановления необходимо защитить и восстановить эти компоненты инфраструктуры, прежде чем восстанавливать рабочие нагрузки и приложения.

Azure Site Recovery позволяет создать полностью автоматизированный план аварийного восстановления для Active Directory и DNS. Например, чтобы выполнить сбой в сравнении с SharePoint и SAP от основного к вторичному сайту, можно настроить план восстановления, который сначала не выполняется в течение Active Directory. Затем используйте дополнительный план восстановления для восстановления, специфичный для использования в приложениях, которые полагаются на Active Directory.

[Подробнее](site-recovery-active-directory.md) о аварийном восстановлении для Active Directory и DNS.

## <a name="protect-sql-server"></a>Защита SQL Server

Сервер S'L Server обеспечивает основу служб передачи данных для многих бизнес-приложений в центре обработки данных. Восстановление сайта может быть использовано с помощью технологий S'L Server HA/DR для защиты многоуровневых корпоративных приложений, которые используют Сервер S'L.

Преимущества службы Site Recovery:

- Простое и экономичное решение аварийного восстановления для SQL Server. Репликация нескольких версий и выпусков автономных серверов и кластеров SQL Server в Azure или на дополнительный сайт.
- Интеграция с группами доступности SQL AlwaysOn для управления отработкой отказа и восстановлением размещения с использованием планов восстановления Azure Site Recovery.
- Планы комплексного восстановления для всех уровней приложения, включая базы данных SQL Server.
- Масштабирование сервера S'L для пиковых нагрузок с помощью восстановления сайта, _разрывая_ их в более крупные размеры виртуальных машин IaaS в Azure.
- Простое тестирование аварийного восстановления SQL Server. Можно запускать тестовые отработки отказа и выполнять проверки на соответствие без влияния на рабочую среду.

[Узнайте больше](site-recovery-sql.md) о аварийном восстановлении сервера S'L.

## <a name="protect-sharepoint"></a>Защита SharePoint

Azure Site Recovery помогает защитить развертывания SharePoint следующим образом:

- Устраняется необходимость в резервной ферме аварийного восстановления и соответствующие расходы на инфраструктуру. Используйте восстановление сайта для воспроизведения всей фермы (уровень веб-приложений и баз данных) в Azure или на второстепенный сайт.
- Упрощается развертывание приложения и управлением им. Обновления, развернутые на основном сайте, автоматически реплицируются. Обновления доступны после сбоя и восстановления фермы на вторичном сайте. Снижает сложность управления и затраты, связанные с поддержанием резервной фермы в актуальном состоянии.
- Упрощается разработка и тестирование приложений SharePoint благодаря созданию реплики рабочей среды по запросу для тестирования и отладки.
- Упрощается переход в облако благодаря использованию Site Recovery для переноса развертываний SharePoint в Azure.

[Подробнее](site-recovery-sharepoint.md) о аварийном восстановлении sharePoint.

## <a name="protect-dynamics-ax"></a>Защита Dynamics AX

## <a name="protect-remote-desktop-services"></a>Защита служб удаленного рабочего стола

С помощью восстановления сайта Azure можно воспроизвести следующие службы:

- Replicate управляемые или неуправляемые объединенные виртуальные рабочие столы на вторичном сайте.
- Воспроизвести удаленные приложения и сеансы на второстепенный сайт или Azure.

## <a name="protect-exchange"></a>Защита Exchange

Site Recovery защищает Exchange следующим образом.

- Для небольших развертываний Exchange, например отдельного или изолированного сервера, Site Recovery обеспечивает репликацию и отработку отказа в Azure или на дополнительный сайт.
- Для более крупных развертываний Site Recovery интегрируется с группами DAG Exchange.
- Exchange DAG рекомендуется в качестве основного решения для аварийного восстановления Exchange на уровне предприятия.  Планы восстановления Site Recovery могут включать в себя группы DAG для координации отработки отказа DAG на различных сайтах.

## <a name="protect-sap"></a>Защита SAP

Site Recovery обеспечивает защиту развертывания SAP благодаря следующим возможностям:

- защита рабочих приложений SAP NetWeaver и других приложений, запущенных в локальной среде, с помощью репликации компонентов в Azure;
- защита рабочих приложений SAP NetWeaver и других приложений, запущенных в Azure, с помощью репликации компонентов в другой центр обработки данных Azure;
- упрощение миграции в облако благодаря использованию Site Recovery для переноса развертывания SAP в Azure;
- упрощение обновления, тестирования и создания прототипов проектов SAP путем создания рабочего клона по запросу для тестирования приложений SAP.

[Подробнее](site-recovery-sap.md) о аварийном восстановлении sAP.

## <a name="protect-internet-information-services"></a>Защита информационных услуг в Интернете

Используйте восстановление сайта для защиты развертывания Интернет-информационных служб (IIS) следующим образом:

Azure Site Recovery обеспечивает аварийное восстановление путем репликации важных компонентов из вашей среды на "холодный" удаленный сайт или в общедоступное облако, например Microsoft Azure. Поскольку виртуальные машины с веб-сервером и базой данных реплицируются на сайт восстановления, нет необходимости в отдельном резервном копировании файлов или сертификатов конфигурации. Сопоставления и привязки приложения, которые зависят от переменных среды, измененных после отработки отказа, можно обновить с помощью скриптов, интегрированных в планы аварийного восстановления. Виртуальные машины воспитываются на месте восстановления только во время сбоя. Восстановление сайта Azure также поможет вам организовать сквозной неудачу, предоставив вам следующие возможности:

- Последовательность завершения работы и запуска виртуальных машин на разных уровнях.
- Добавление скриптов для обновления зависимостей приложений и привязок к виртуальным машинам после их запуска. С помощью скриптов можно также обновить DNS-сервер, чтобы он указывал на сайт восстановления.
- Распределение IP-адресов для виртуальных машин до неудачи путем отображения первичных и восстановительных сетей и использования скриптов, которые не должны быть обновлены после неудачи.
- Возможность сбоя в одном клике для нескольких веб-приложений, что исключает возможности для путаницы во время аварии.
- Возможность тестирования планов восстановления в изолированной среде для детализации аварийного восстановления.

[Узнайте больше](site-recovery-iis.md) о аварийном восстановлении для IIS.

## <a name="protect-citrix-xenapp-and-xendesktop"></a>Защита Citrix XenApp и XenDesktop

Site Recovery обеспечивает защиту развертывания Citrix XenApp и XenDesktop благодаря следующим возможностям:

- Позволяет защитить развертывание Citrix XenApp и XenDesktop. Воспроизвести различные слои развертывания в Azure: Active Directory, DNS-сервер, сервер базы данных S'L, контроллер доставки Citrix, сервер StoreFront, XenApp Master (VDA), лицензионный сервер Citrix XenApp.
- Упрощение миграции в облако благодаря использованию Site Recovery для переноса развертывания Citrix XenApp и XenDesktop в Azure.
- Упрощение тестирования Citrix XenApp и XenDesktop благодаря созданию реплики рабочей среды по запросу для тестирования и отладки.
- Это решение применяется только к виртуальным рабочим столам Windows Server, а не к виртуальным настольным компьютерам клиентов. Виртуальные рабочие столы клиентов пока не поддерживаются для лицензирования в Azure. Дополнительные сведения о лицензировании рабочих столов клиента и сервера в Azure см. [здесь](https://azure.microsoft.com/pricing/licensing-faq/).

[Узнайте больше](site-recovery-citrix-xenapp-and-xendesktop.md) о аварийном восстановлении для развертывания Citrix XenApp и XenDesktop. Или, вы можете сослаться на [документ Citrix](https://aka.ms/citrix-xenapp-xendesktop-with-asr).

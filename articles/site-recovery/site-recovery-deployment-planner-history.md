# <a name="azure-site-recovery-deployment-planner-version-history"></a>Журнала версий планировщика развертывания для Azure Site Recovery

Эта статья содержит журнал всех версий планировщика развертывания Azure Site Recovery, а также исправления, известные ограничения в каждом и даты их выхода.

## <a name="version-24"></a>Версии 2.4

**Дата выпуска: 17 апреля 2019 г.**

**Исправления:**

- Улучшенная совместимость операционной системы, в частности в том случае, когда обработка ошибок на основе локализации.
- Добавлена виртуальных машин с помощью до 20 Мбит/с, данных измените скорость в контрольном списке совместимости.
- Улучшенные сообщения об ошибках
  - Добавлена поддержка vCenter 6.7.
  - Добавлена поддержка для рабочей станции Windows Server 2019 и Red Hat Enterprise Linux (RHEL).



## <a name="version-23"></a>Версия 2.3

**Дата выпуска: 3 декабря 2018 г.**

**Исправления:**

- Устранена проблема, препятствующая планировщике создавая отчет с заданным целевым объектом расположением и подпиской.

## <a name="version-22"></a>Версии 2.2 

**Дата выпуска: 25 апреля 2018 г.**

**Исправления:**

- Операции GetVMList.
  - Исправлена проблема, которая вызывала GetVMList с ошибкой, если указанная папка не существует. Он теперь создает каталог по умолчанию либо создает каталог, указанный в параметре outputfile.
  - Добавить дополнительные причины сбоя для GetVMList.
- Добавлены сведения типа виртуальной Машины как столбец в таблице совместимых виртуальных машин отчета планировщика развертывания.
- Hyper-V для аварийного восстановления Azure:
  - Исключенные виртуальных машин с помощью общих виртуальных жестких дисков и сквозные диски из профилирования. Операция Startprofiling показывает список исключенных виртуальных машин в консоли.
  - Добавлена виртуальные машины с более чем 64 дисками в список несовместимых виртуальных машин.
  - Обновлены начальной репликации (IR) и коэффициент сжатия разностной репликации (DR).
  - Добавлена ограниченная поддержка хранилища SMB.

## <a name="version-21"></a>Версии 2.1

**Дата выпуска: 3 января 2018 г.**

**Исправления:**

- Обновление отчетов Excel.
- Исправленные ошибки в операция GetThroughput.
- Добавлен параметр, чтобы ограничить количество виртуальных машин для профилирования или для формирования отчета. Ограничение по умолчанию — 1000 виртуальных машин.
- VMware для аварийного восстановления Azure:
  - Устранена проблема, из виртуальной Машины Windows Server 2016 в несовместимая таблицы. 
  - Обновление сообщения о совместимости для виртуальных машин Windows Extensible Firmware Interface (EFI).
- Обновить VMware в Azure и Hyper-V в Azure, скорость обработки данных виртуальной Машины предел каждой виртуальной Машины. 
- Повышенная надежность для синтаксического анализа файла списка виртуальных Машин.

## <a name="version-201"></a>Версия 2.0.1

**Дата выпуска: 7 декабря 2017 г.**

**Исправления:**

- Добавлены рекомендации для оптимизации пропускной способности сети.

## <a name="version-20"></a>Версии 2.0

**Дата выпуска: 28 ноября 2017 г.**

**Исправления:**

- Добавлена поддержка Hyper-V для аварийного восстановления Azure.
- Калькулятор дополнительной платы.
- Добавлена проверка версии операционной системы для VMware для аварийного восстановления Azure для определения того, является ли виртуальная машина совместимы для защиты. Средство использует строку версии ОС, возвращенный сервером vCenter, для этой виртуальной Машины. Это именно та версия гостевой ОС, пользователь выбрал при создании виртуальной Машины в VMware.

**Известные ограничения:**

- Для Hyper-V для аварийного восстановления Azure, виртуальной Машины с именем, содержащим символы, например: `,`, `"`, `[`, `]`, и ``` ` ``` не поддерживаются. Профилирование, создание отчетов не удастся или будет иметь неверный результат.
- Виртуальная машина с именем, содержащим запятой не поддерживается для VMware для аварийного восстановления Azure. Профилирование, происходит сбой создания отчетов или будет иметь неверный результат.

## <a name="version-131"></a>Версия 1.3.1

**Дата выпуска: 19 июля 2017 г.** 

**Исправления:**

- Добавлена поддержка дисков большого объема (> 1 ТБ) при создании отчетов. Теперь вы можете использовать планировщик для планирования репликации виртуальных машин с дисками размером более 1 ТБ (до 4095 ГБ).
Дополнительные сведения о [поддержке дисков большого размера в Azure Site Recovery](https://azure.microsoft.com/blog/azure-site-recovery-large-disks/).

## <a name="version-13"></a>Версии 1.3

**Дата выпуска: 9 мая 2017 г.**

**Исправления:**

- Добавлена поддержка для управляемого диска при создании отчетов. Вычисляется на основе количества виртуальных машин, которые могут быть помещены в одной учетной записью, если управляемый диск выбран для отработки отказа или тестовой отработки отказа.

## <a name="version-12"></a>версии 1.2

**Дата выпуска: 7 апреля 2017 г.**

**Исправления:**

- Добавлена загрузки (BIOS или EFI) проверки типов для каждой виртуальной Машины определить совместимые или несовместимые для защиты виртуальной Машины.
- Добавлены ОС введите сведения для каждой виртуальной машины в совместимых и несовместимых виртуальных машин листов.
- Добавлена поддержка операция GetThroughput для регионов для государственных организаций США и Microsoft Azure в Китае.
- Добавлено несколько дополнительных проверок необходимых компонентов для vCenter и сервера ESXi.
- Устранена проблема, из неправильный отчет, создаваемыми настройки параметров языкового стандарта для локализованной.

## <a name="version-11"></a>Версия 1.1

**Дата выпуска: 9 марта 2017 г.**

**Исправления:**

- Устранена проблема, препятствующая профилирование виртуальных машин, при наличии двух или нескольких виртуальных машин с тем же именем или IP-адрес, расположенных на разных узлах ESXi в vCenter.
- Исправлена проблема, копирования и поиска, необходимо отключить для совместимых и несовместимых виртуальных машин листов.

## <a name="version-10"></a>Версия 1.0

**Дата выпуска: 23 февраля 2017 г.**

**Известные ограничения:**

- Поддерживает только для VMware в Azure аварийного восстановления. Для Hyper-V в Azure аварийного восстановления, используйте [планировщик ресурсов Hyper-V](./site-recovery-capacity-planning-for-hyper-v-replication.md).
- Не поддерживает операцию getthroughput, доступная для регионов для государственных организаций США и Microsoft Azure в Китае.
- Профиль средство виртуальных машин, если сервер vCenter имеет два или несколько виртуальных машин с тем же именем или IP-адрес, расположенных на разных узлах ESXi.
В этой версии программа пропускает профилирование виртуальных машин, имена или IP-адреса которых заданы в качестве значения параметра VMListFile несколько раз. Возможное решение — выполнить профилирование виртуальных машин на узле ESXi, а не на сервере vCenter Server. Убедитесь, что запустить один экземпляр для каждого узла ESXi.

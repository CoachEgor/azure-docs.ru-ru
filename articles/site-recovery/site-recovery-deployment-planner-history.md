## <a name="version-10"></a>Версия 1.0

**Дата выпуска: 23 февраля 2017 г.**

**Известные ограничения:**

- Поддерживает только сценарии аварийного восстановления VMware to Azure. Для сценариев аварийного восстановления Hyper-V в Azure используйте [инструмент планировщика емкости Hyper-V.](./site-recovery-capacity-planning-for-hyper-v-replication.md)
- Не поддерживает операцию GetThroughput для правительства США и регионов Microsoft Azure.
- Инструмент не может профилировать VMs, если сервер vCenter имеет два или более VMs с тем же именем или IP-адресом в различных хостах ESXi.
В этой версии программа пропускает профилирование виртуальных машин, имена или IP-адреса которых заданы в качестве значения параметра VMListFile несколько раз. Возможное решение — выполнить профилирование виртуальных машин на узле ESXi, а не на сервере vCenter Server. Убедитесь в том, что для каждого узла ESXi будет запущен один экземпляр.

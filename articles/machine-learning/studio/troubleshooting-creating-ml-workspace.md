# <a name="troubleshooting-guide-create-and-connect-to-an-azure-machine-learning-studio-classic-workspace"></a>Руководство по устранению неполадок: Создание и подключение к рабочей студии Машинного обучения Azure (классическое)

[!INCLUDE [Notebook deprecation notice](../../../includes/aml-studio-notebook-notice.md)]

Это руководство предоставляет решения для некоторых часто встречающихся проблем при настройке Azure Machine Learning Learning Studio (классические) рабочие области.

## <a name="workspace-owner"></a>Владелец рабочей области
Чтобы открыть рабочее пространство в студии машинного обучения (классический), вы должны войти в учетную запись Майкрософт, которую вы использовали для создания рабочего пространства, или вам нужно получить приглашение от владельца присоединиться к рабочему пространству. На портале Azure можно управлять рабочей областью, что включает в себя возможность настроить доступ.

Для получения дополнительной информации об управлении рабочим пространством можно ознакомиться на сайте [Manage an Azure Machine Learning Studio (классическое) рабочее пространство.]

[Управление Лазурной студии машинного обучения (классическое) рабочее пространство]: manage-workspace.md

## <a name="allowed-regions"></a>Разрешенные регионы
Сейчас машинное обучение доступно только в ограниченном числе регионов. Если подписка не включает один из этих регионов, вы можете увидеть сообщение об ошибке: "У вас нет подписки в разрешенных регионах".

Чтобы запросить добавление этого региона в вашу подписку, на портале Azure создайте запрос в службу поддержки Майкрософт, а затем в качестве типа проблемы выберите **Выставление счетов** и следуйте указаниям, чтобы отправить запрос.

## <a name="storage-account"></a>Учетная запись хранения
Для хранения данных службе машинного обучения требуется учетная запись хранения. Вы можете использовать существующую учетную запись хранения, или вы можете создать новую учетную запись хранения при создании нового рабочего пространства Машинного Обучения (классическое) рабочее пространство (если у вас есть квота для создания новой учетной записи хранилища).

После создания нового рабочего пространства Machine Learning Studio можно войти в студию машинного обучения (классический) с помощью учетной записи Майкрософт, которую вы использовали для создания рабочего пространства. Если вы столкнулись с сообщением об ошибке, "Workspace не найдено" (по аналогии со следующим скриншотом), пожалуйста, используйте следующие шаги, чтобы удалить файлы cookie браузера.

![Рабочая область не найдена.](media/troubleshooting-creating-ml-workspace/screen3.png)

**Удаление файлов cookie браузера**

1. Если вы используете Internet Explorer, нажмите кнопку **Сервис** в правом верхнем углу и выберите **Свойства браузера**.  

   ![Свойства браузера](media/troubleshooting-creating-ml-workspace/screen4.png)

2. На вкладке **Общие** нажмите кнопку **Удалить**.

   ![Вкладка «Общее»](media/troubleshooting-creating-ml-workspace/screen5.png)

3. В диалоговом окне **Удаление истории обзора** установите флажок **Файлы cookie и данные веб-сайтов** и нажмите кнопку **Удалить**.

   ![Удалить файлы cookie](media/troubleshooting-creating-ml-workspace/screen6.png)

После удаления файлов cookie перезапустите браузер, а затем перейдите на страницу [Microsoft Azure Machine Learning Studio (классическая).](https://studio.azureml.net) Введите данные той же учетной записи Майкрософт, использованной для создания рабочей области.

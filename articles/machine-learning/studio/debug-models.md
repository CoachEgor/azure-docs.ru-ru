# <a name="debug-your-model-in-azure-machine-learning-studio-classic"></a>Отладка модели в Машинное обучение Azure Studio (классическая модель)

При запуске модели могут возникнуть следующие ошибки:

* модуль [обучение модели][train-model] выдает ошибку 
* модуль [оценки модели][score-model] выдает неверные результаты 

В этой статье объясняются потенциальные причины появления этих ошибок.


## <a name="train-model-module-produces-an-error"></a>Модуль "Обучение модели" выдает ошибку

![image1](./media/debug-models/train_model-1.png)

Модуль [обучение модели][train-model] принимает два входа:

1. Тип модели машинного обучения из коллекции моделей, предоставляемых Машинное обучение Azure Studio (классическая модель).
2. Данные для обучения со столбцом метки, где указывается переменная, которую нужно спрогнозировать (остальные столбцы считаются функциями).

Модуль может выдавать ошибку в таких случаях:

1. Столбец метки указан неправильно. Это может произойти, если в качестве метки выбрано более одного столбца или выбран недопустимый индекс столбца. Пример второго случая: указан индекс столбца 30, тогда как входной набор данных содержит только 25 столбцов.

2. Набор данных не содержит столбцов с признаками. Например, если входной набор данных содержит только один столбец, выбранный в качестве метки, для построения модели не будет ни одного признака. В этом случае модуль [обучение модели][train-model] выдает ошибку.

3. Во входном наборе данных (признаки или метка) в качестве значения указана бесконечность.

## <a name="score-model-module-produces-incorrect-results"></a>Модуль "Score Model" (Оценка модели) выдает неправильные результаты

![рисунок 2](./media/debug-models/train_test-2.png)

В обычном эксперименте по обучению и тестированию для контролируемого обучения модуль [Split Data (разделение данных][split] ) разделяет исходный набор данных на две части: одна часть используется для обучения модели, а одна часть используется для оценки того, насколько хорошо работает обученная модель. Затем обученная модель оценивает тестовые данные, после чего оцениваются результаты и определяется точность ее работы.

Для модуля [оценки модели][score-model] требуются два входа:

1. Вывод обученной модели из модуля [обучение модели][train-model] .
2. Набор оцениваемых данных, отличный от набора данных, использованного для обучения модели.

Возможно, несмотря на успешность эксперимента, модуль [оценки модели][score-model] выдает неверные результаты. Вот в каких случаях это может произойти.

1. Если для указанной метки задано значение «Категория» и модель регрессии обучена по данным, то в модуле « [Оценка модели][score-model] » будет сформирован неверный результат. Это обусловлено тем, что для регрессии требуется непрерывная переменная ответа. В этом случае лучше использовать модель классификации. 

2. Аналогичным образом, если модель классификации обучается с помощью данных, которые содержат числа с плавающей запятой в столбце метки, модель может выдать недопустимые результаты. Это обусловлено тем, что для классификации требуется дискретная переменная отклика, с которой можно использовать только значения, соответствующие конечному и малому набору классов.

3. Если набор данных для оценки не содержит все функции, используемые для обучения модели, то [модель оценки][score-model] выдает ошибку.

4. Если строка в наборе данных оценки содержит отсутствующее значение или бесконечное значение для любой из его функций, то [модель оценки][score-model] не создает никаких выходных данных, соответствующих этой строке.

5. [Модель оценки][score-model] может формировать идентичные выходные данные для всех строк в наборе данных оценки. Это может случиться, например, при классификации с помощью лесов принятия решений, когда минимальное количество примеров на листовой узел превышает количество доступных обучающих примеров.

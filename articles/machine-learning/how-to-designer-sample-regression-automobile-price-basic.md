# <a name="use-regression-to-predict-car-prices-with-azure-machine-learning-designer"></a>Использование регрессии для прогнозирования цен на автомобили с помощью конструктора Машинного обучения Azure

**Конструктор (предварительная версия) — пример 1**

[!INCLUDE [applies-to-skus](../../includes/aml-applies-to-enterprise-sku.md)]

Узнайте, как создать регрессионную модель машинного обучения с помощью конструктора (предварительная версия) без необходимости писать код.

Этот конвейер обучает **линейный регрессор** для прогнозирования цен на автомобили на основе таких технических признаков, как марка, модель, мощность и размер. Проблема, которую мы решаем, называется задачей регрессии. Но вы можете выполнять базовые действия, описанные в этом примере, для решения любых задач машинного обучения, будь то регрессия, классификация, кластеризация и т. д.

Базовые этапы обучения модели машинного обучения:

1. Получение данных
1. Предварительная обработка данных.
1. Обучение модели
1. Анализ модели

Вот финальная версия графа конвейера. В этой статье рассматривается целесообразность использования каждого модуля, чтобы вы могли принимать соответствующие решения самостоятельно.

## <a name="get-the-data"></a>Получение данных

В этом примере используется набор **необработанных данных о ценах на автомобили** из репозитория машинного обучения UCI. Этот набор данных содержит 26 столбцов с информацией об автомобилях, включая сведения о марке, модели, цене, признаках (например, числе цилиндров), расходе топлива и оценке страховых рисков. Задача этого примера — спрогнозировать цену автомобиля.

## <a name="pre-process-the-data"></a>Предварительная обработка данных

Основные задачи подготовки данных включают очистку, интеграцию, преобразование, сокращение, дискретизацию и квантование. В конструкторе есть модули для выполнения этих операций и других задач предварительной обработки данных. Они включены в группу модулей **преобразования данных** на панели слева.

С помощью модуля **выбора столбцов в наборе данных** исключите нормализованные потери с большим числом отсутствующих значений. Затем используйте модуль **очистки отсутствующих данных**, чтобы удалить строки с отсутствующими значениями. Так вы создадите чистый набор данных для обучения.

![Предварительная обработка данных](./media/how-to-designer-sample-regression-automobile-price-basic/data-processing.png)

## <a name="train-the-model"></a>Обучение модели

Есть различные задачи машинного обучения. Среди распространенных — классификация, кластеризация, регрессия и системы рекомендаций. Для решения каждой из таких задач может потребоваться отдельный алгоритм. Выбор алгоритма часто зависит от требований сценария пользователя. Когда вы выберете алгоритм, нужно будет настроить его параметры для обучения более точной модели. Затем необходимо проанализировать все модели с использованием таких метрик, как точность, понятность и эффективность.

Так как задача этого примера — прогнозирование цен на автомобили, а столбец меток (цена) содержит реальные числа, оптимальным вариантом будет использовать регрессионную модель. Учитывая, что число признаков относительно невелико (меньше 100) и эти признаки не разрежены, граница решения, скорее всего, будет нелинейной. Поэтому для этого конвейера используется **регрессия леса принятия решений**.

Используйте модуль **разделения данных**. Он разделяет входные данные в случайном порядке таким образом, чтобы набор данных для обучения содержал 70 % исходных данных, а набор данных для тестирования — 30 % исходных данных.

## <a name="test-evaluate-and-compare"></a>Проверка, анализ и сравнение

Для повышения объективности при анализе модели разделите набор данных таким образом, чтобы для обучения и тестирования модели использовались разные наборы данных.

После обучения модели вы можете использовать модули **оценки модели** и **анализа модели** для создания прогнозируемых результатов и анализа моделей.

Модуль **оценки модели** выполняет прогнозирование для тестового набора данных с использованием обученной модели. Чтобы проверить результат, выберите порт вывода модуля **оценки модели** и включите **визуализацию**.

![Оценка результата](./media/how-to-designer-sample-regression-automobile-price-basic/sample1-score-1225.png)

Передайте оценки в модуль **анализа модели** для создания метрик оценки. Чтобы проверить результат, выберите порт вывода модуля **анализа модели** и включите **визуализацию**.

![Анализ результата](./media/how-to-designer-sample-regression-automobile-price-basic/sample1-evaluate-1225.png)

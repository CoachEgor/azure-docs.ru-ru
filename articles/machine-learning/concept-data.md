# <a name="data-access-in-azure-machine-learning"></a>Доступ к данным в машинном обучении Azure

Машинное обучение Azure упрощает подключение к данным в облаке.  Он обеспечивает слой абстракции над базовым сервисом хранения, так что вы можете безопасно получить доступ и работать с вашими данными без необходимости писать код, специфичный для вашего типа хранилища. Azure Machine Learning также предоставляет следующие возможности данных:

*    Версия и отслеживание линии данных
*    Маркировка данных 
*    Мониторинг смещения данных
*    Совместимость с пандами и Spark DataFrames

## <a name="data-workflow"></a>Рабочий процесс данных

Когда вы будете готовы использовать данные в облачном решении для хранения данных, мы рекомендуем следующий рабочий процесс доставки данных. Этот рабочий процесс предполагает, что у вас есть [учетная запись хранения Azure](https://docs.microsoft.com/azure/storage/common/storage-quickstart-create-account?tabs=azure-portal) и данные в облачном сервисе хранения данных в Azure. 

1. Создайте [хранилище данных По](#datastores) для обработки данных о машинном обучении Azure для хранения информации о подключении к хранилищу Azure.

2. Из этого хранилища данных создайте [набор данных Azure Machine Learning,](#datasets) чтобы указать на определенный файл (ы) в базовом хранилище. 

3. Использовать этот набор данных в эксперименте машинного обучения можно либо
    1. Установите его к вычислительной цели вашего эксперимента для обучения модели.

        **Или** 

    1. Потребляйте его непосредственно в решениях для машинного обучения Azure, таких как, запуски экспериментов по автоматическому машинному обучению (автоматизированный ML), конвейеры машинного обучения или [конструктор машинного обучения Azure.](concept-designer.md)

4. Создание [мониторов набора данных](#data-drift) для набора выходных данных модели для обнаружения дрейфа данных. 

5. При обнаружении дрейфа данных обновите набор входных данных и переурядитесь.

Следующая диаграмма обеспечивает наглядное демонстрацию этого рекомендуемого рабочего процесса.

![Данные-концептуальные диаграммы](./media/concept-data/data-concept-diagram.svg)

## <a name="datastores"></a>Хранилища данных

Магазины данных Azure Machine Learning надежно хранят информацию о подключении к хранилищу Azure, поэтому вам не нужно кодировать ее в своих скриптах. [Зарегистрируйтесь и создайте хранилище данных](how-to-access-data.md) для легкого подключения к учетной записи хранилища и получите доступ к данным в базовой службе хранения Azure. 

Поддерживаемые облачные службы хранения данных в Azure, которые могут быть зарегистрированы в качестве хранилищ данных:

+ Контейнер Azure Blob
+ Общая папка Azure
+ Azure Data Lake;
+ Azure Data Lake 2-го поколения
+ База данных SQL Azure
+ База данных Azure для PostgreSQL
+ Файловая система Databricks
+ База данных Azure для MySQL

## <a name="datasets"></a>Наборы данных

Наборы данных машинного обучения Azure — это ссылки, уотняющие данные в службе хранения данных. Они не являются копиями ваших данных, поэтому никаких дополнительных расходов на хранение не понесено. Чтобы взаимодействовать с данными в хранилище, [создайте набор данных](how-to-create-register-datasets.md) для упаковки данных в расходный объект для задач машинного обучения. Зарегистрируйте набор данных в рабочем пространстве для обмена и повторного использования его в различных экспериментах без сложностей, пригласив их.

Наборы данных могут создаваться из локальных файлов, общедоступных URL-адресов, [открытых наборов данных Azure](https://azure.microsoft.com/services/open-datasets/)или служб хранения данных Azure через хранилища данных. Чтобы создать набор данных из фрейма данных панд в памяти, напишите данные в локальный файл, например паркет, и создайте набор данных из этого файла.  

Мы поддерживаем 2 типа наборов данных: 
+ [TabularDataset](https://docs.microsoft.com/python/api/azureml-core/azureml.data.tabulardataset?view=azure-ml-py) представляет данные в табулярном формате путем анализа предоставленного файла или списка файлов. Вы можете загрузить TabularDataset в Pandas или Spark DataFrame для дальнейших манипуляций и очищения. Для полного списка форматов данных, из которые можно [TabularDatasetFactory class](https://aka.ms/tabulardataset-api-reference)создать TabularDatasets, см.

+ [FileDataset](https://docs.microsoft.com/python/api/azureml-core/azureml.data.file_dataset.filedataset?view=azure-ml-py) ссылается на отдельные или несколько файлов в ваших хранилищах данных или общедоступных URL-адресах. Вы можете [скачать или смонтировать файлы,](how-to-train-with-datasets.md#option-2--mount-files-to-a-remote-compute-target) на которые ссылаются FileDatasets, в вашу вычислительную цель.

Дополнительные возможности наборов данных можно найти в следующей документации:

+ Линия набора [данных версии и отслеживания.](how-to-version-track-datasets.md)
+ [Мониторинг набора данных,](how-to-monitor-datasets.md) чтобы помочь с обнаружением дрейфа данных.

## <a name="data-labeling"></a>Маркировка данных

Добавление меток к большим объемам данных часто усложняет проекты машинного обучения. Те, с компонентом компьютерного зрения, такие как классификация изображений или обнаружения объектов, как правило, требуют тысячи изображений и соответствующих меток.

Решение "Машинное обучение Azure" служит централизованным расположением для создания, администрирования и мониторинга проектов маркировки. Проекты маркировки помогают упорядочивать данные, метки и координировать работу команды, то есть более эффективно управлять задачами присвоения меток. Сейчас поддерживаются такие задачи: классификация изображений (с несколькими метками или классами) и идентификация объектов с использованием ограничивающих прямоугольников.

Создайте [проект маркировки данных](how-to-create-labeling-projects.md)и выведивайте набор данных для использования в экспериментах машинного обучения.

<a name="drift"></a>

## <a name="data-drift"></a>Дрейф данных

В контексте машинного обучения дрейф данных представляет собой изменение входных данных модели, что приводит к снижению производительности модели. Это одна из главных причин, по которым точность модели ухудшается с течением времени, таким образом, мониторинг дрейфа данных помогает обнаружить проблемы с производительностью модели.

Ознакомьтесь со [статьей «Создайте набор данных»,](how-to-monitor-datasets.md) чтобы узнать больше о том, как обнаруживать и предупреждать о дрейфе данных на новых данных в наборе данных.

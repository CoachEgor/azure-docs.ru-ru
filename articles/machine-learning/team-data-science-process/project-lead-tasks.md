# <a name="project-lead-tasks-in-the-team-data-science-process"></a>Ведущие задачи проекта в процессе командных данных

В этой статье описаны задачи, которые *руководитель проекта* завершает для создания репозитория для своей проектной группы в [процессе подготовки данных team](overview.md) (TDSP). TDSP — это платформа, разработанная корпорацией Майкрософт, которая обеспечивает структурированную последовательность действий для эффективного выполнения облачных решений для прогнозной аналитики. TDSP разработан, чтобы помочь улучшить сотрудничество и обучение в команде. Для набросков ролей персонала и связанных с ними задач для группы [Team Data Science Process roles and tasks](roles-tasks.md)по анализу данных, стандартизирующей TDSP, см.

Руководитель проекта управляет повседневной деятельностью отдельных ученых по обработке данных по конкретному проекту по науке о данных в TDSP. На следующей диаграмме отображается рабочий процесс для выполнения задач, ведущих по проектам:

![Рабочий процесс выполнения задач проекта](./media/project-lead-tasks/project-leads-1-tdsp-creating-projects.png)

Этот учебник охватывает Шаг 1: Создание репозитория проекта и репозиторий проекта Step 2: Seed из репозитория ProjectTemplate вашей команды. 

Для шага 3: Создайте элемент работы функций для проекта и шаг 4: Добавить истории для этапов проекта, [см. Agile развития проектов науки о данных](agile-development.md).

Для шага 5: Создание и настройка ресурсов хранения/анализа и совместное использование, при необходимости, [см.](team-lead-tasks.md#create-team-data-and-analytics-resources)

Для шага 6: Настройте контроль безопасности репозитория проекта, [см. Добавить членов группы и настроить разрешения.](team-lead-tasks.md#add-team-members-and-configure-permissions)

> [!NOTE] 
> В этой статье используется Azure Repos для настройки проекта TDSP, поскольку именно так можно реализовать TDSP в корпорации Майкрософт. Если ваша команда использует другую платформу хостинга кода, задачи руководства проекта остаются теми же, но способ их выполнения может быть разным.

## <a name="prerequisites"></a>Предварительные требования

Этот учебник предполагает, что [ваш руководитель группы](group-manager-tasks.md) и [руководитель группы](team-lead-tasks.md) создали следующие ресурсы и разрешения:

- **Организация** Azure DevOps для вашего подразделения данных
- Командный **проект** для вашей группы по анализу данных
- Шаблон команды и **хранилища** утилит
- **Разрешения** на организацию должны создавать и отсеивать репозитории для проекта

Чтобы клонировать репозитории и изменять содержимое локальной машины или виртуальной машины Data Science (DSVM), или настроить хранилище файлов Azure и установить его на DSVM, необходимо также рассмотреть этот контрольный список:

- Подписка Azure.
- Git установлен на вашей машине. Если вы используете DSVM, Git предустановлен. Если система не установлена, см. [описание платформ и средств](platforms-and-tools.md#appendix).
- Если вы хотите использовать DSVM, Windows или Linux DSVM, созданные и настроенные в Azure. Для получения дополнительной информации и инструкций, [см.](/azure/machine-learning/data-science-virtual-machine/)
- Для Windows DSVM на вашей машине установлен [Git Credential Manager (GCM).](https://github.com/Microsoft/Git-Credential-Manager-for-Windows) В *README.md* файла прокрутите вниз раздел **Загрузить и установить** и выберите **последний установщик.** Скачать установщик *.exe* со страницы установки и запустить его. 
- Для Linux DSVM, sSH общедоступный ключ, настроенный на DSVM и добавленный в Azure DevOps. Для получения дополнительной информации и инструкций смотрите раздел **«Создание sSH»** в разделе «Открытый ключ» в [приложении Платформы и инструменты.](platforms-and-tools.md#appendix) 

## <a name="create-a-project-repository-in-your-team-project"></a>Создание репозитория проекта в проекте группы

Чтобы создать репозиторий проекта в проекте **MyTeam** вашей команды:

1. Перейдите на **страницу** проекта вашей команды по *адресу\//\<https: имя сервера\<>/имя организации>/имя\<команды>, *например, **https:/dev.azure.com/DataScienceUnit/MyTeam,\/** и выберите **Репос** из левой навигации. 
   
1. Выберите имя репозитория в верхней части страницы, а затем выберите **новый репозиторий** из выпадения.
   
   ![Выберите новый репозиторий](./media/project-lead-tasks/project-leads-9-select-repos.png)
   
1. При создании нового диалога **репозитория** убедитесь, что **Git** выбран под **типом**. Введите *DSProject1* под **названием Repository,** а затем выберите **Создать.**
   
   ![Создание репозитория](./media/project-lead-tasks/project-leads-3-create-project-repo-2.png)
   
1. Подтвердите, что вы можете увидеть новый репозиторий **DSProject1** на странице настроек проекта. 
   
   ![Репозиторий проекта в настройках проекта](./media/project-lead-tasks/project-leads-4-create-project-repo-3.png)

## <a name="import-the-team-template-into-your-project-repository"></a>Импорт шаблона группы в репозиторий проекта

Чтобы заполнить репозиторий проекта с помощью содержимого репозитория шаблона группы:

1. На странице проекта **«Резюме»** группы выберите **РЕПО** в левой навигации. 
   
1. Выберите имя репозитория в верхней части страницы и выберите **DSProject1** из выпадения.
   
1. На **DSProject1 пустая** страница, выберите **Импорт**. 
   
   ![Выберите Импорт](./media/project-lead-tasks/project-leads-5-create-project-repo-4.png)
   
1. В **диалоге репозитория «Импорт»** выберите **Git** в качестве **типа Исходного кода**и введите URL-адрес для репозитория **TeamTemplate** под **URL-адресом clone.** *URL-адрес\//\<https: имя\<сервера>/ название организации\<>/ название команды>/_git/\<имя репозитория шаблона>. * Например: **https:\//dev.azure.com/DataScienceUnit/MyTeam/_git/TeamTemplate**. 
   
1. Выберите **Импортировать**. Содержимое репозитория шаблона группы импортируется в репозиторий проекта. 
   
   ![Репозиторий шаблона группы импорта](./media/project-lead-tasks/project-leads-6-create-project-repo-5.png)

Если вам необходимо настроить содержимое репозитория проекта в соответствии с конкретными потребностями проекта, можно добавить, удалить или изменить файлы репозитория и папки. Вы можете работать непосредственно в Azure Repos или клонировать репозиторий локальной машине или DSVM, вносить изменения, а также фиксировать и продвигать обновления в общий репозиторий проекта. Следуйте инструкциям по [настройке содержимого репозиториев команды.](team-lead-tasks.md#customize-the-contents-of-the-team-repositories)

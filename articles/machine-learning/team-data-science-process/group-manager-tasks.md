# <a name="team-data-science-process-group-manager-tasks"></a>Задачи диспетчера группового командного процесса обработки и анализа данных

В этой статье описаны задачи, выполняемые *менеджером группы* для организации по науке о данных. Менеджер группы управляет всем подразделением по науке о данных на предприятии. Подразделение по анализу данных может иметь несколько групп, каждая из которых работает над многими проектами в области науки о данных в различных бизнес-вертикалях. Целью руководителя группы является создание совместной группы, которая стандартизирует [процесс оценки данных команды](overview.md) (TDSP). Для набросков всех кадровых ролей и связанных с ними задач, обрабатываемых группой по анализу данных, стандартизирующей TDSP, [см.](roles-tasks.md)

На следующей диаграмме показаны шесть основных задач настройки менеджера группы. Руководители групп могут делегировать свои задачи суррогатам, но задачи, связанные с ролью, не меняются.

![Задачи менеджера группы](./media/group-manager-tasks/tdsp-group-manager.png)

1. Навлаживать **для группы организацию Azure DevOps.**
2. Создайте **проект GroupCommon** по умолчанию в организации Azure DevOps.
3. Создание репозитория **GroupProjectTemplate** в Azure Repos.
4. Создайте репозиторий **GroupUtilities** в Azure Repos.
5. Импортируйте содержимое репозиториев **ProjectTemplate** и **утилиты** команды Microsoft TDSP в общие репозитории группы.
6. Настройка **членства** и разрешений для членов **группы** для доступа к группе.

Следующий учебник проходит через шаги в деталях. 

> [!NOTE] 
> В этой статье используется Azure DevOps для настройки среды группы TDSP, поскольку именно так можно реализовать TDSP в корпорации Майкрософт. Если ваша группа использует другие платформы для хостинга кода или разработки, задачи менеджера группы остаются прежними, но способ их выполнения может быть разным.

## <a name="set-up-the-group-common-repositories"></a>Настройка групповых общих репозиториев

В Azure Repos размещаются следующие типы репозиториев для вашей группы:

- **Групповые общие репозитории**: репозитории общего назначения, которые несколько групп в рамках подразделения по науке о данных могут принять для многих проектов по науке о данных. 
- **Репозитории групп:** Репозитории для определенных групп в подразделении по науке о данных. Эти репозитории специфичны для потребностей группы и могут использоваться для нескольких проектов в рамках этой группы, но не являются достаточно общими для использования в нескольких группах в подразделении по науке о данных.
- **Репозитории проектов**: Репозитории для конкретных проектов. Такие репозитории могут быть недостаточно общими для нескольких проектов в группе или для других групп в подразделении по анализу данных.

Чтобы настроить общие репозитории группы в проекте: 
- Переименуй репозиторий **GroupCommon** в **GroupProjectTemplate**
- Создание нового репозитория **GroupUtilities**

### <a name="rename-the-default-project-repository-to-groupprojecttemplate"></a>Переименуй репозиторий проекта по умолчанию в GroupProjectTemplate

Переименовать репозиторий проекта **GroupCommon** в **GroupProjectTemplate:**

1. На странице **резюме** проекта **GroupCommon** выберите **REpos**. Это действие приведет вас к репозиторию **GroupCommon** проекта GroupCommon, который в настоящее время пуст.
   
1. В верхней части страницы опустите стрелку рядом с **GroupCommon** и выберите **репозитории управления.**
   
   ![Управление репозиториями](./media/group-manager-tasks/rename-groupcommon-repo-3.png)
   
1. На странице **Настройки проекта** выберите **GroupCommon** **...** **Rename repository** 
   
   ![Выберите... а затем выберите репозиторий переименования](./media/group-manager-tasks/rename-groupcommon-repo-4.png)
   
1. Во **время переименования всплывающем всплывающем** центре GroupCommon введите *GroupProjectTemplate,* а затем выберите **Rename**. 
   
   ![Репозиторий переименования](./media/group-manager-tasks/rename-groupcommon-repo-6.png)

### <a name="create-the-grouputilities-repository"></a>Создание репозитория GroupUtilities

Для создания репозитория **GroupUtilities:**

1. На странице **резюме** проекта **GroupCommon** выберите **REpos**. 
   
1. В верхней части страницы, падение стрелка рядом с **GroupProjectTemplate** и выбрать **новый репозиторий**.
   
   ![Выберите новый репозиторий](./media/group-manager-tasks/create-grouputilities-repo-1.png)
   
1. В создании нового диалога **репозитория** выберите **Git** в качестве **типа,** введите *GroupUtilities* в качестве **имени репозитория,** а затем выберите **Создать.**
   
   ![Создание репозитория GroupUtilities](./media/group-manager-tasks/create-grouputilities-repo-2.png)
   
1. На странице **Настройки проекта** выберите **репозитории** под **РЕпосом** в левой навигации, чтобы просмотреть два репозитория группы: **GroupProjectTemplate** и **GroupUtilities.**
   
   ![Два групповых репозитория](./media/group-manager-tasks/two-repositories.png)

## <a name="import-the-microsoft-tdsp-team-repositories"></a>Импорт репозиторий группы Microsoft TDSP

В этой части учебника вы импортируете содержимое репозиториев **ProjectTemplate** и **Утилиты,** управляемых командой Microsoft TDSP, в репозитории **GroupProjectTemplate** и **GroupUtilities.** 

Для импорта репозиториев команды TDSP:

1. На главной странице проекта **GroupCommon** выберите **РЕПО** в левой навигации. Открывается репо **GroupProjectTemplate** по умолчанию. 
   
1. На **GroupProjectTemplate пустая** страница, выберите **Импорт**. 
   
   ![Выберите Импорт](./media/group-manager-tasks/import-repo.png)
   
1. В **диалоге репозитория Импорта Git** выберите **Git** в качестве **типа Исхода**и введите *https:\//github.com/Azure/Azure-TDSP-ProjectTemplate.git* **URL-адреса клона.** Затем выберите **Импорт**. Содержимое репозитория команды Microsoft TDSP ProjectTemplate импортируется в репозиторий GroupProjectTemplate. 
   
   ![Импортировать репозиторий команды Microsoft TDSP](./media/group-manager-tasks/import-repo-2.png)
   
1. В верхней части страницы **РЕПО** отбросьте и выберите репозиторий **GroupUtilities.**
   
1. Повторите процесс импорта для импорта содержимого репозитория **утилит** ыmicrosoftиt TDSP, *https:\//github.com/Azure/Azure-TDSP-Utilities.git,* в репозиторий **GroupUtilities.** 
   
Каждый из двух репозиториев группы теперь содержит все файлы, за исключением файлов в каталоге *.git,* из соответствующего репозитория команды Microsoft TDSP. 

## <a name="customize-the-contents-of-the-group-repositories"></a>Настройка содержимого групповых репозиториев

Если вы хотите настроить содержимое репозиториев группы для удовлетворения конкретных потребностей вашей группы, вы можете сделать это сейчас. Можно изменить файлы, изменить структуру каталога или добавить файлы, разработанные группой или полезные для группы.

### <a name="make-changes-in-azure-repos"></a>Вносить изменения в Репос Azure

Для настройки содержимого репозитория:

1. На странице **резюме** проекта **GroupCommon** выберите **REpos**. 
   
1. В верхней части страницы выберите репозиторий, который вы хотите настроить.

1. В структуре каталога репо перейдите в папку или файл, который вы хотите изменить. 
   
   - Чтобы создать новые папки или файлы, выберите стрелку рядом с **Новым**. 
     
     ![Создание нового файла](./media/group-manager-tasks/new-file.png)
     
   - Чтобы загрузить файлы, выберите **Файл загрузки (ы)**. 
     
     ![Upload files](./media/group-manager-tasks/upload-files.png)
     
   - Чтобы отсеять существующие файлы, перейдите к файлу, а затем выберите **Edit.** 
     
     ![Отсечение файла](./media/group-manager-tasks/edit-file.png)
     
1. После добавления или редактирования файлов выберите **Commit**.
   
   ![Фиксация изменений](./media/group-manager-tasks/commit.png)

### <a name="make-changes-using-your-local-machine-or-dsvm"></a>Вносить изменения с помощью локальной машины или DSVM

Если вы хотите внести изменения с помощью локальной машины или DSVM и нажать изменения до группы репозиториев, убедитесь, что у вас есть предпосылки для работы с Git и DSVMs:

- Подписка Azure, если вы хотите создать DSVM.
- Git установлен на вашей машине. Если вы используете DSVM, Git предустановлен. Если система не установлена, см. [описание платформ и средств](platforms-and-tools.md#appendix).
- Если вы хотите использовать DSVM, Windows или Linux DSVM, созданные и настроенные в Azure. Для получения дополнительной информации и инструкций, [см.](/azure/machine-learning/data-science-virtual-machine/)
- Для Windows DSVM на вашей машине установлен [Git Credential Manager (GCM).](https://github.com/Microsoft/Git-Credential-Manager-for-Windows) В *README.md* файла прокрутите вниз раздел **Загрузить и установить** и выберите **последний установщик.** Скачать установщик *.exe* со страницы установки и запустить его. 
- Для Linux DSVM, sSH общедоступный ключ, настроенный на DSVM и добавленный в Azure DevOps. Для получения дополнительной информации и инструкций смотрите раздел **«Создание sSH»** в разделе «Открытый ключ» в [приложении Платформы и инструменты.](platforms-and-tools.md#appendix) 

Во-первых, скопируйте или *клонируйте* репозиторий для локальной машины. 
   
1. На странице **резюме** проекта **GroupCommon** выберите **Репос,** а в верхней части страницы выберите репозиторий, который вы хотите клонировать.
   
1. На странице репо выберите **клон** в правом верхнем углу.
   
1. В **диалоге репозитория клонов** выберите **HTTPS** для подключения HTTP или **SSH** для соединения SSH и скопируйте URL-адрес клона под **командной строкой** на буфер обмена.
   
   ![Клонирование репозиториев](./media/group-manager-tasks/clone.png)
   
1. На локальной машине создайте следующие каталоги:
   
   - Для Windows: **C:'GitRepos-GroupCommon**
   - Для Linux, **$/GitRepos/GroupCommon** в вашем домашнем каталоге 
   
1. Изменение каталога, который вы создали.
   
1. В Git Bash, запустить команду`git clone <clone URL>.`
   
   Например, одна из следующих команд клонирует репозиторий **GroupUtilities** в каталог *GroupCommon* на локальной машине. 
   
   **Соединение HTTPS:**
   
   ```bash
   git clone https://DataScienceUnit@dev.azure.com/DataScienceUnit/GroupCommon/_git/GroupUtilities
   ```
   
   **Соединение SSH:**
   
   ```bash
   git clone git@ssh.dev.azure.com:v3/DataScienceUnit/GroupCommon/GroupUtilities
   ```

После внесения любых изменений в локальный клон репозитория можно нажать на изменения в общие репозитории общей группы. 

Выполнить следующие команды Git Bash из вашего локального каталога **GroupProjectTemplate** или **GroupUtilities.**

```bash
git add .
git commit -m "push from local"
git push
```

> [!NOTE]
> Если это первый раз, когда вы совершаете в репозиторий Git, возможно, потребуется `git commit` настроить глобальные параметры *user.name* и *user.email* перед запуском команды. Выполните эти две команды:
> 
> `git config --global user.name <your name>`
> 
> `git config --global user.email <your email address>`
> 
> Если вы совершаете несколько репозиториев Git, используйте одно и то же имя и адрес электронной почты для всех из них. Использование одного и того же имени и адреса электронной почты удобно при создании панелей мониторинга Power BI для отслеживания действий Git в нескольких репозиториях.

## <a name="add-group-members-and-configure-permissions"></a>Добавление членов группы и настройка разрешений

Чтобы добавить членов в группу:

1. В Azure DevOps на главной странице проекта **GroupCommon** выберите параметры проекта из левой **навигации.** 
   
1. Из **настройки проекта** левая навигация, выберите **команды**, затем на странице **команды,** выберите **Team GroupCommon**. 
   
   ![Настройка команд](./media/group-manager-tasks/teams.png)
   
1. На странице **профиля команды** выберите **Добавить**.
   
   ![Добавление в команду GroupCommon](./media/group-manager-tasks/add-to-team.png)
   
1. В диалоге **пользователей и групп add,** поиск и выбор участников, чтобы добавить в группу, а затем выберите **Сохранить изменения.** 
   
   ![Добавление пользователей и групп](./media/group-manager-tasks/add-users.png)
   

Для настройки разрешений для участников:

1. Из **настройки проекта левой навигации,** выберите **Разрешения**. 
   
1. На странице **«Разрешения»** выберите группу, в которая вы хотите добавить участников. 
   
1. На странице для этой группы выберите **участников,** а затем выберите **Добавить.** 
   
1. Во всплывающем **центре invite участники** ищут и выбирайте участников, чтобы добавить в группу, а затем выберите **Сохранить**. 
   
   ![Разрешение на предоставление членам](./media/group-manager-tasks/grant-permissions.png)

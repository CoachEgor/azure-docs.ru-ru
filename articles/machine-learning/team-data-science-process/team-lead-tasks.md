# <a name="tasks-for-the-team-lead-on-a-team-data-science-process-team"></a>Задачи для руководства группой в команде по анализу данных

В этой статье описаны задачи, которые *возглавляет группа,* для их группы по науке о данных. Целью группы является создание совместной группы, которая стандартизирует [процесс team Data Science Process](overview.md) (TDSP). TDSP разработан, чтобы помочь улучшить сотрудничество и обучение в команде. 

TDSP — это гибкая итеративная методология науки о данных для эффективного предоставления решений для прогнозной аналитики и интеллектуальных приложений. Этот процесс перегоняет лучшие практики и структуры от Microsoft и отрасли.  Целью является успешная реализация инициатив в области науки о данных и полное осознание преимуществ их аналитических программ. Для набросков ролей персонала и связанных с ними задач для группы [Team Data Science Process roles and tasks](roles-tasks.md)по анализу данных, стандартизирующей TDSP, см.

Руководитель группы управляет группой, состоящей из нескольких ученых-данных в подразделении по анализу данных предприятия. В зависимости от размера и структуры подразделения по науке о данных [руководитель группы](group-manager-tasks.md) и руководитель группы могут быть одно и тем же человеком, или они могут делегировать свои задачи суррогатам. Но сами задачи не изменяются. 

Следующая диаграмма показывает рабочий процесс для задач, которые приводит команда для настройки среды команды:

![Рабочий процесс выполнения задач группы](./media/team-lead-tasks/team-leads-1-creating-teams.png)

1. Создайте **командный проект** в организации группы в Azure DevOps. 
  
1. Переименуй репозиторий команды по умолчанию в **TeamUtilities.**
  
1. Создайте новый репозиторий **TeamTemplate** в командном проекте. 
  
1. Импортируйте содержимое репозиториев **GroupUtilities** и **GroupProjectTemplate** в репозитории **TeamUtilities** и **TeamTemplate.** 
  
1. Настройка **управления безопасностью** путем добавления членов группы и настройки их разрешений.
  
1. При необходимости создавайте командные данные и ресурсы аналитики:
   - Добавление утилит для конкретных команд в репозиторий **TeamUtilities.** 
   - Создайте **хранилище файлов Azure** для хранения ресурсов данных, которые могут быть полезны для всей команды. 
   - Установите хранилище файлов Azure в **виртуальную машину Data Science** (DSVM) группы и добавьте в него ресурсы данных.

Следующий учебник проходит через шаги в деталях.

> [!NOTE] 
> В этой статье используются Azure DevOps и DSVM для настройки среды группы TDSP, потому что именно так можно реализовать TDSP в корпорации Майкрософт. Если ваша команда использует другие платформы хостинга кода или разработки, задачи руководства команды остаются прежними, но способ их выполнения может быть разным.

- **Организация** Azure DevOps для вашего подразделения данных
- Репозитории **GroupProjectTemplate** и **GroupUtilities,** заполненные содержимым репозиториев **ProjectTemplate** и **утилит** ый ствами команды Microsoft TDSP
- Разрешения на организацию должны создавать проекты и репозитории для вашей команды

Чтобы клонировать репозитории и изменять их содержимое на локальной машине или DSVM, или настроить хранилище файлов Azure и установить его на DSVM, вам нужно следующее:

- Подписка Azure.
- Git установлен на вашей машине. Если вы используете DSVM, Git предустановлен. Если система не установлена, см. [описание платформ и средств](platforms-and-tools.md#appendix).
- Если вы хотите использовать DSVM, Windows или Linux DSVM, созданные и настроенные в Azure. Для получения дополнительной информации и инструкций, [см.](/azure/machine-learning/data-science-virtual-machine/)
- Для Windows DSVM на вашей машине установлен [Git Credential Manager (GCM).](https://github.com/Microsoft/Git-Credential-Manager-for-Windows) В *README.md* файла прокрутите вниз раздел **Загрузить и установить** и выберите **последний установщик.** Скачать установщик *.exe* со страницы установки и запустить его. 
- Для Linux DSVM, sSH общедоступный ключ, настроенный на DSVM и добавленный в Azure DevOps. Для получения дополнительной информации и инструкций смотрите раздел **«Создание sSH»** в разделе «Открытый ключ» в [приложении Платформы и инструменты.](platforms-and-tools.md#appendix) 

## <a name="create-a-team-project-and-repositories"></a>Создание командного проекта и репозиториев

В этом разделе вы создаете следующие ресурсы в организации Azure DevOps группы:

- Проект **MyTeam** в Azure DevOps
- Репозиторий **TeamTemplate**
- Репозиторий **TeamUtilities**

Имена, указанные для репозиториев и каталогов в этом учебнике, предполагают, что необходимо создать отдельный проект для вашей собственной команды в рамках более крупной организации по науке о данных. Однако вся группа может выбрать работу в рамках одного проекта, созданного менеджером группы или администратором организации. Затем все группы по анализу данных создают репозитории в рамках этого единственного проекта. Этот сценарий может быть действителен для:
- Небольшая группа по науке о данных, которая не имеет нескольких групп по науке о данных. 
- Большая группа по науке о данных с несколькими группами по анализу данных, которая, тем не менее, хочет оптимизировать взаимодействие между командами с такими видами деятельности, как планирование спринта на групповом уровне. 

Если команды хотят иметь свои репозитории в рамках одного группового проекта, руководители группдолжны создавать репозитории с именами, такими как * \<TeamName>Template* и * \<TeamName>Utilities.* Например: *TeamATemplate* и *TeamAUtilities*. 

В любом случае, приводит группы, чтобы члены их команды знали, какие шаблоны и хранилища утилит для настройки и клонирования. Руководители проекта должны выполнять [задачи, стоящие перед руководством проекта, для группы по науке о данных](project-lead-tasks.md) для создания репозиторий проектов, будь то в рамках отдельных проектов или одного проекта. 

### <a name="create-the-myteam-project"></a>Создание проекта MyTeam

Чтобы создать отдельный проект для вашей команды:

1. В вашем веб-браузере перейдите на домашнюю страницу организации Azure DevOps вашей группы по адресу *URL\//\<https: название сервера\<>/название организации>* и выберите **новый проект.** 
   
   ![Выберите новый проект](./media/team-lead-tasks/team-leads-2-create-new-team.png)
   
1. В диалоге **проекта Create** введите имя команды *MyTeam,* под **названием Project,** а затем выберите **Advanced.** 
   
1. Под **контролем версии,** выберите **Git,** и под **процессом элемента работы,** выберите **Agile.** Затем выберите **Создать**. 
   
   ![Создание проекта](./media/team-lead-tasks/team-leads-3-create-new-team-2.png)
   
Открывается страница **резюме** проекта, url-адрес *страницы https:\//\<название сервера>/название\<организации>/имя\<команды>. *

### <a name="rename-the-myteam-default-repository-to-teamutilities"></a>Переименуй репозиторий по умолчанию MyTeam в TeamUtilities

1. На странице **резюме** проекта **MyTeam,** под **какой услугой вы хотели бы начать с?**, выберите **REPOS**. 
   
   ![Выберите РЕПО](./media/team-lead-tasks/team-leads-6-rename-team-project-repo.png)
   
1. На странице репо **MyTeam** выберите репозиторий **MyTeam** в верхней части страницы, а затем выберите **Управление репозиториями** из выпадения. 
   
   ![Выберите Репозитории управления](./media/team-lead-tasks/team-leads-7-rename-team-project-repo-2.png)
1. На странице **Настройки проекта** выберите **...** **MyTeam** **Rename repository** 
   
   ![Выберите репозиторий переименования](./media/team-lead-tasks/team-leads-8-rename-team-project-repo-3.png)
   
1. Во **время повторного всплывающих платежей репозитория MyTeam** введите *TeamUtilities,* а затем выберите **Переименовать.** 

### <a name="create-the-teamtemplate-repository"></a>Создание репозитория TeamTemplate

1. На странице **Настройки проекта** выберите **новый репозиторий.** 
   
   ![Выберите новый репозиторий](./media/team-lead-tasks/team-leads-9-create-team-utilities.png)
   
   Или выберите **РЕпос** из левой навигации на странице **резюме** проекта **MyTeam,** выберите репозиторий в верхней части страницы, а затем выберите **новый репозиторий** из выпадения.
   
1. При создании нового диалога **репозитория** убедитесь, что **Git** выбран под **типом**. Введите *TeamTemplate* под **именем repository,** а затем выберите **Создать**.
   
   ![Создание репозитория](./media/team-lead-tasks/team-leads-10-create-team-utilities-2.png)
   
1. Подтвердите, что на странице настроек проекта можно увидеть два репозитория **TeamUtilities** и **TeamTemplate.** 
   
   ![Два репозитория группы](./media/team-lead-tasks/team-leads-11-two-repo-in-team.png)

### <a name="import-the-contents-of-the-group-common-repositories"></a>Импорт содержимого групповых общих репозиториев

Заполнить репозитории вашей группы с помощью содержимого общих репозиторий группы, настроенных менеджером группы:

1. На главной странице проекта **MyTeam** выберите **РЕПО** в левой навигации. Если вы получаете сообщение о том, что шаблон **MyTeam** не найден, выберите ссылку в **противном случае, перейдите в репозиторий TeamTemplate по умолчанию.** 
   
   Открывается репозиторий **TeamTemplate** по умолчанию. 
   
1. На **странице TeamTemplate пустая** страница, выберите **Импорт**. 
   
   ![Выберите Импорт](./media/team-lead-tasks/import-repo.png)
   
1. В **диалоге репозитория «Импорт»** выберите **Git** в качестве **типа Исходного кода**и введите URL-адрес для общего репозитория шаблонов группы под **URL-адресом clone.** URL *https:\//\<имя сервера\<>/ название\<организации>/_git/ имя репозитория>*. Например: *https:\//dev.azure.com/DataScienceUnit/GroupCommon/_git/GroupProjectTemplate*. 
   
1. Выберите **Импортировать**. Содержимое репозитория шаблона группы импортируется в репозиторий шаблона команды. 
   
   ![Общий репозиторий группы импорта](./media/team-lead-tasks/import-repo-2.png)
   
1. В верхней части страницы **РЕпос** проекта отпадет и выберите репозиторий **TeamUtilities.**
   
1. Повторите процесс импорта, чтобы импортировать содержимое общего репозитория общих утилит группы, например *GroupUtilities,* в репозиторий **TeamUtilities.** 
   
Каждый из двух репозиториев вашей группы теперь содержит файлы из соответствующего общего репозитория группы. 

### <a name="customize-the-contents-of-the-team-repositories"></a>Настройка содержимого репозиториев группы

Если вы хотите настроить содержимое репозиториев вашей команды в соответствии с конкретными потребностями вашей команды, вы можете сделать это сейчас. Можно изменить файлы, изменить структуру каталога или добавить файлы и папки.

Чтобы изменить, загрузить или создать файлы или папки непосредственно в Azure DevOps:

1. На странице **резюме** проекта **MyTeam** выберите **REpos**. 
   
1. В верхней части страницы выберите репозиторий, который вы хотите настроить.

1. В структуре каталога репо перейдите в папку или файл, который вы хотите изменить. 
   
   - Чтобы создать новые папки или файлы, выберите стрелку рядом с **Новым**. 
     
     ![Создание нового файла](./media/team-lead-tasks/new-file.png)
     
   - Чтобы загрузить файлы, выберите **Файл загрузки (ы)**. 
     
     ![Upload files](./media/team-lead-tasks/upload-files.png)
     
   - Чтобы отсеять существующие файлы, перейдите к файлу, а затем выберите **Edit.** 
     
     ![Отсечение файла](./media/team-lead-tasks/edit-file.png)
     
1. После добавления или редактирования файлов выберите **Commit**.
   
   ![Фиксация изменений](./media/team-lead-tasks/commit.png)

Для работы с репозиториями на локальной машине или DSVM сначала копируете или *клонируете* репозитории в локальной машине, а затем совершаете и передвигаете изменения в общие репозитории группы, 

Для клонирования репозиторий:

1. На странице **Резюме** проекта **MyTeam** выберите **Репос,** а в верхней части страницы выберите репозиторий, который вы хотите клонировать.
   
1. На странице репо выберите **клон** в правом верхнем углу.
   
1. В **диалоге репозитория клонов** под **командной строкой**выберите **HTTPS** для подключения HTTP или **SSH** для соединения SSH и скопируйте URL-адрес клона на буфер обмена.
   
   ![URL-адрес копирования клона](./media/team-lead-tasks/clone.png)
   
1. На локальной машине создайте следующие каталоги:
   
   - Для Windows: **C:'GitRepos/MyTeam**
   - Для Linux, **$home/GitRepos/MyTeam** 
   
1. Изменение каталога, который вы создали.
   
1. В Git Bash запустите `git clone <clone URL>` \<команду, где клон URL> является URL, который вы скопировали из диалога **клонов.**
   
   Например, используйте одну из следующих команд для клонирования репозитория **TeamUtilities** в каталог *MyTeam* на локальной машине. 
   
   **Соединение HTTPS:**
   
   ```bash
   git clone https://DataScienceUnit@dev.azure.com/DataScienceUnit/MyTeam/_git/TeamUtilities
   ```
   
   **Соединение SSH:**
   
   ```bash
   git clone git@ssh.dev.azure.com:v3/DataScienceUnit/MyTeam/TeamUtilities
   ```

После внесения любых изменений в локальный клон репозитория совершайте и нажимайте изменения в общие репозитории группы. 

Выполнить следующие команды Git Bash из вашего локального каталога **GitRepos-MyTeam-TeamTemplate** или **GitRepos-MyTeam-TeamUtilities.**

```bash
git add .
git commit -m "push from local"
git push
```

> [!NOTE]
> Если это первый раз, когда вы совершаете в репозиторий Git, возможно, потребуется `git commit` настроить глобальные параметры *user.name* и *user.email* перед запуском команды. Выполните эти две команды:
> 
> `git config --global user.name <your name>`
> 
> `git config --global user.email <your email address>`
> 
> Если вы совершаете несколько репозиториев Git, используйте одно и то же имя и адрес электронной почты для всех из них. Использование одного и того же имени и адреса электронной почты удобно при создании панелей мониторинга Power BI для отслеживания действий Git в нескольких репозиториях.

## <a name="add-team-members-and-configure-permissions"></a>Добавление членов группы и настройка разрешений

Чтобы добавить членов в команду:

1. В Azure DevOps на главной странице проекта **MyTeam** выберите настройки проекта из левой **навигации.** 
   
1. Из **настройки проекта** левая навигация, выберите **команды**, затем на странице **команды,** выберите **MyTeam команды**. 
   
   ![Настройка команд](./media/team-lead-tasks/teams.png)
   
1. На странице **профиля команды** выберите **Добавить**.
   
   ![Добавить в команду MyTeam](./media/team-lead-tasks/add-to-team.png)
   
1. В диалоге **пользователей и групп add,** поиск и выбор участников, чтобы добавить в группу, а затем выберите **Сохранить изменения.** 
   
   ![Добавление пользователей и групп](./media/team-lead-tasks/add-users.png)
   

Для настройки разрешений для членов команды:

1. Из **настройки проекта левой навигации,** выберите **Разрешения**. 
   
1. На странице **«Разрешения»** выберите группу, в которая вы хотите добавить участников. 
   
1. На странице для этой группы выберите **участников,** а затем выберите **Добавить.** 
   
1. Во всплывающем **центре invite участники** ищут и выбирайте участников, чтобы добавить в группу, а затем выберите **Сохранить**. 
   
   ![Разрешение на предоставление членам](./media/team-lead-tasks/grant-permissions.png)

## <a name="create-team-data-and-analytics-resources"></a>Создание ресурсов группы данных и аналитики

Этот шаг является необязательным, но совместное использование ресурсов данных и аналитики со всей командой имеет преимущества в производительности и затратах. Члены группы могут выполнять свои проекты на общих ресурсах, экономить на бюджетах и более эффективно сотрудничать. Вы можете создать хранилище файлов Azure и установить его на DSVM, чтобы поделиться с членами команды. 

Для получения информации о совместном использовании других ресурсов с вашей командой, таких как кластеры Azure HDInsight Spark, [см.](platforms-and-tools.md) Эта тема содержит рекомендации с точки зрения науки о данных по выбору ресурсов, подходящих для ваших нужд, и ссылки на страницы продуктов и другие соответствующие и полезные учебники.

>[!NOTE]
> Чтобы избежать передачи данных через центры обработки данных, которые могут быть медленными и дорогостоящими, убедитесь, что группа ресурсов Azure, учетная запись хранения и DSVM размещаются в одном и том же регионе Azure. 

### <a name="create-azure-file-storage"></a>Создание хранилища файлов Azure

1. Выполнить следующий скрипт для создания хранилища файлов Azure для ресурсов данных, которые полезны для всей вашей команды. Скрипт подскажет вам информацию о подписке Azure, так что это готово для ввода. 

   - На машине Windows запустите скрипт из запроса команды PowerShell:
     
     ```powershell
     wget "https://raw.githubusercontent.com/Azure/Azure-MachineLearning-DataScience/master/Misc/TDSP/CreateFileShare.ps1" -outfile "CreateFileShare.ps1"
     .\CreateFileShare.ps1
     ```
     
   - На машине Linux запустите скрипт из оболочки Linux:
     
     ```shell
     wget "https://raw.githubusercontent.com/Azure/Azure-MachineLearning-DataScience/master/Misc/TDSP/CreateFileShare.sh"
     bash CreateFileShare.sh
     ```
   
1. Войти в учетную запись Microsoft Azure по запросу и выбрать подписку, которая вы хотите использовать.
   
1. Выберите учетную запись хранилища для использования или создайте новую под выбранной подпиской. Для имени хранения файлов Azure можно использовать символы, цифры и дефисы в нижнем регистре.
   
1. Чтобы облегчить монтаж и совместное использование хранилища, нажмите Enter или введите *Y,* чтобы сохранить информацию о хранении файлов Azure в текстовый файл в текущем каталоге. Вы можете проверить этот текстовый файл в репозитории **TeamTemplate,** в идеале под **Docs и DataDictionaries,** чтобы все проекты в вашей команде могли получить к нему доступ. Вам также нужна информация о файлах для установки хранилища файлов Azure в DSVM Azure в следующем разделе. 
   
### <a name="mount-azure-file-storage-on-your-local-machine-or-dsvm"></a>Хранилище файлов Azure на локальной машине или DSVM

1. Чтобы смонтировать хранилище файлов Azure в локальной машине или DSVM, используйте следующий скрипт.
   
   - На машине Windows запустите скрипт из запроса команды PowerShell:
     
     ```powershell
     wget "https://raw.githubusercontent.com/Azure/Azure-MachineLearning-DataScience/master/Misc/TDSP/AttachFileShare.ps1" -outfile "AttachFileShare.ps1"
     .\AttachFileShare.ps1
     ```
     
   - На машине Linux запустите скрипт из оболочки Linux:
     
     ```shell
     wget "https://raw.githubusercontent.com/Azure/Azure-MachineLearning-DataScience/master/Misc/TDSP/AttachFileShare.sh"
     bash AttachFileShare.sh
     ```
   
1. Нажмите Введите или введите *Y* для продолжения, если вы сохранили файл информации о хранении файлов Azure на предыдущем этапе. Введите полный путь и имя созданного файла. 
   
   Если у вас нет файла информации о хранении файлов Azure, введите *n*и следуйте инструкциям для ввода подписки, учетной записи хранения azure и информации о хранении файлов Azure.
   
1. Введите имя локального или TDSP-диска, чтобы установить долю файла. На экране отображается список существующих имен дисков. Предоставьте имя диска, которое еще не существует.
   
1. Подтвердите, что новый диск и хранилище успешно установлены на вашей машине.

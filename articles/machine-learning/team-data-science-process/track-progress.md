# <a name="track-the-progress-of-data-science-projects"></a>Отслеживайте ход проектов в области науки о данных

Руководители групп данных, руководители групп и руководители проектов могут отслеживать ход своих проектов.  Руководители хотят знать, какая работа была проделана, кто выполнил работу, и какая работа остается.   Управление ожиданиями является важным элементом успеха.

## <a name="azure-devops-dashboards"></a>Панели мониторинга Azure DevOps

Если вы используете Azure DevOps, можно создать панели мониторинга для отслеживания действий и рабочих элементов, связанных с данным проектом Agile. Для получения дополнительной информации [Dashboards, reports, and widgets](/azure/devops/report/dashboards/)о панелях мониторинга см.

Для получения инструкций о том, как создавать и настраивать панели мониторинга и виджеты в Azure DevOps, см.

- [Добавление панелей мониторинга и управление ими](/azure/devops/report/dashboards/dashboards)
- [Добавление виджетов на приборную панель](/azure/devops/report/dashboards/add-widget-to-dashboard)

## <a name="example-dashboard"></a>Пример панели мониторинга

Вот простой пример панели мониторинга, которая отслеживает действия спринта проекта Agile data, включая количество коммитов в связанных репозиториях. 

- Плитка **отсчета** показывает количество дней, остающихся в текущем спринте. 

- Две **плитки кода** показывают количество коммитов в двух репозиториях проекта за последние семь дней. 

- **Рабочие элементы для проекта клиента TDSP** показывают результаты запроса для всех рабочих элементов и их статуса. 

- На **накопительной диаграмме потока** (CFD) отображается количество закрытых и активных рабочих элементов.

- **Диаграмма сгорания** показывает работу, которую еще предстоит завершить против оставшегося времени в спринте.

- Диаграмма **сгорания** показывает завершенную работу по сравнению с общим объемом работы в спринте.

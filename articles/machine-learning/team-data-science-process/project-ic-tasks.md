# <a name="tasks-for-an-individual-contributor-in-the-team-data-science-process"></a>Задачи отдельного участника в командном процессе обработки и анализа данных

В этой теме излагаются задачи, которые *отдельный участник* выполняет для создания проекта в [процессе team Data Science Process](overview.md) (TDSP). Цель состоит в том, чтобы работать в совместной рабочей группе, которая стандартизирует TDSP. TDSP разработан, чтобы помочь улучшить сотрудничество и обучение в команде. Для набросков ролей персонала и связанных с ними задач, которые обрабатываются группой [Team Data Science Process roles and tasks](roles-tasks.md)по анализу данных, стандартизирующей TDSP, см.

На следующей диаграмме показаны задачи, которые проектировать отдельных участников (данных ученых) для настройки среды своей команды. Для получения инструкций о том, как выполнить проект по науке о данных в рамках TDSP, [см.](project-execution.md) 

![Индивидуальные задачи вкладчика](./media/project-ic-tasks/project-ic-1-tdsp-data-scientist.png)

- **ProjectRepository** — это репозиторий, который ведет команда проекта для обмена шаблонами и активами проекта.
- **TeamUtilities** — это хранилище утилит, которые ваша команда поддерживает специально для вашей команды. 
- **GroupUtilities** — это хранилище, поддерживаемое группой для совместного обмена полезными утилитами по всей группе. 

> [!NOTE] 
> В этой статье используются Azure Repos и виртуальная машина Data Science (DSVM) для настройки среды TDSP, поскольку именно таким образом можно реализовать TDSP в корпорации Майкрософт. Если ваша команда использует другие платформы хостинга кода или разработки, отдельные задачи вкладчика одинаковы, но способ их выполнения может быть разным.

## <a name="prerequisites"></a>Предварительные требования

Этот учебник предполагает, что следующие ресурсы и разрешения были созданы [вашим менеджером группы,](group-manager-tasks.md) [руководителем группы](team-lead-tasks.md)и [руководителем проекта:](project-lead-tasks.md)

- **Организация** Azure DevOps для вашего подразделения по науке о данных
- **Репозиторий проекта,** созданный проектом, приводит к обмену шаблонами и активами проекта
- Репозитории **GroupUtilities** и **TeamUtilities,** созданные менеджером группы и руководителем группы, если это применимо
- Хранилище **файлов** Azure, настроенное для общих ресурсов для вашей команды или проекта, если это применимо
- **Разрешения** на клонирование и отодвилость в репозиторий проекта 

Чтобы клонировать репозитории и изменять содержимое локальной машины или DSVM, или монтировать хранилище файлов Azure в DSVM, необходимо рассмотреть этот контрольный список:

- Подписка Azure.
- Git установлен на вашей машине. Если вы используете DSVM, Git предустановлен. Если система не установлена, см. [описание платформ и средств](platforms-and-tools.md#appendix).
- Если вы хотите использовать DSVM, Windows или Linux DSVM, созданные и настроенные в Azure. Для получения дополнительной информации и инструкций, [см.](/azure/machine-learning/data-science-virtual-machine/)
- Для Windows DSVM на вашей машине установлен [Git Credential Manager (GCM).](https://github.com/Microsoft/Git-Credential-Manager-for-Windows) В *README.md* файла прокрутите вниз раздел **Загрузить и установить** и выберите **последний установщик.** Скачать установщик *.exe* со страницы установки и запустить его. 
- Для Linux DSVM, sSH общедоступный ключ, настроенный на DSVM и добавленный в Azure DevOps. Для получения дополнительной информации и инструкций смотрите раздел **«Создание sSH»** в разделе «Открытый ключ» в [приложении Платформы и инструменты.](platforms-and-tools.md#appendix) 
- Информация о хранении файлов Azure для любого хранилища файлов Azure, необходимого для установки в DSVM. 

## <a name="clone-repositories"></a>Репозитории клонов

Чтобы работать с репозиториями локально и продвигать изменения в общие репозитории группы и проекта, сначала вы копируете или *клонируете* репозитории локальной машины. 

1. В Azure DevOps перейдите на страницу проекта вашей команды По *\//\<адресу https: имя сервера\<>/имя организации>/имя\<команды>, *например, **https:\//dev.azure.com/DataScienceUnit/MyTeam**.
   
1. Выберите **РЕПО** в левой навигации и в верхней части страницы, выберите репозиторий, который вы хотите клонировать.
   
1. На странице репо выберите **клон** в правом верхнем углу.
   
1. В **диалоге репозитория клонов** выберите **HTTPS** для подключения HTTP или **SSH** для соединения SSH и скопируйте URL-адрес клона под **командной строкой** на буфер обмена.
   
   ![Клонирование репозиториев](./media/project-ic-tasks/clone.png)
   
1. На локальной машине или DSVM создайте следующие каталоги:
   
   - Для Windows: **C:'GitRepos**
   - Для Linux: **$home/GitRepos**
   
1. Изменение каталога, который вы создали.
   
1. В Git Bash запустите команду `git clone <clone URL>` для каждого репозитория, который вы хотите клонировать. 
   
   Например, следующие команды клонирует репозиторий **TeamUtilities** в каталог *MyTeam* на локальной машине. 
   
   **Соединение HTTPS:**
   
   ```bash
   git clone https://DataScienceUnit@dev.azure.com/DataScienceUnit/MyTeam/_git/TeamUtilities
   ```
   
   **Соединение SSH:**
   
   ```bash
   git clone git@ssh.dev.azure.com:v3/DataScienceUnit/MyTeam/TeamUtilities
   ```
   
1. Подтвердите, что в локальном каталоге проектов можно увидеть папки для клонированных репозиториев.
   
   ![Три локальные папки репозитория](./media/project-ic-tasks/project-ic-5-three-repo-cloned-to-ic-linux.png)

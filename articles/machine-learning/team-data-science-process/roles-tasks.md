# <a name="team-data-science-process-roles-and-tasks"></a>Роли и задачи в команде обработки и анализа данных

Процесс team Data Science Process (TDSP) — это платформа, разработанная корпорацией Майкрософт, которая обеспечивает структурированную методологию для эффективного построения решений для прогнозной аналитики и интеллектуальных приложений. В этой статье излагаются ключевые роли персонала и связанные с ними задачи для группы по анализу данных, стандартизирующей этот процесс.

Эта вводная статья ссылки на учебники о том, как настроить среду TDSP. В этих учебниках содержатся подробные рекомендации по использованию проектов Azure DevOps, репозиторий Репосazии и советов Azure.  Мотивирующей целью является переход от концепции к моделированию и к развертыванию.

В учебниках используются Azure DevOps, так как можно реализовать TDSP в корпорации Майкрософт. Azure DevOps облегчает сотрудничество, интегрируя безопасность на основе ролей, управление рабочими деталями и отслеживание, а также хостинг кода, общий доступ и исходный контроль. В учебниках также используется [виртуальная машина](https://aka.ms/dsvm) Azure Data Science (DSVM) в качестве рабочего стола аналитики, который имеет несколько популярных инструментов для анализа данных, предварительно настроенных и интегрированных с программным обеспечением Microsoft и службами Azure. 

Вы можете использовать учебники для реализации TDSP с помощью других инструментов хостинга кода, гибкого планирования и средств разработки и сред, но некоторые функции могут быть недоступны.

## <a name="structure-of-data-science-groups-and-teams"></a>Структура групп и групп по анализу данных

Функции науки о данных на предприятиях часто организуются в следующей иерархии:

- Группа по анализу данных
  - Команда науки о данных/с в группе

В такой структуре есть групповые зацепки и командные зацепки. Как правило, проект по науке о данных выполняется группой по науке о данных. Группы по анализу данных имеют проектные руководители для выполнения задач по управлению проектами и управлению, а отдельные специалисты и инженеры по обработке данных выполняют научно-технические данные и части проекта. Первоначальная настройка и управление проектом выполняется группой, командой или разработчиками проекта.

## <a name="definition-and-tasks-for-the-four-tdsp-roles"></a>Определение и задачи для четырех ролей TDSP
Исходя из предположения, что подразделение по анализу данных состоит из групп в группе, для персонала TDSP существует четыре различные роли:

1. **Менеджер группы**: Управляет всем подразделением по науке о данных на предприятии. В подразделение обработки и анализа данных может входить несколько команд, каждая из которых выполняет несколько проектов по обработке и анализу данных для разных подразделений компании. Менеджер группы может делегировать свои задачи подчиненным, но задачи для этой роли не изменяются.
   
2. **Руководитель группы**: Управляет командой в подразделении по науке данных предприятия. Команда состоит из нескольких специалистов по анализу данных. Для небольшого подразделения по науке о данных менеджер группы и руководитель группы могут быть одно и тем же человеком.
   
3. **Руководитель проекта**: Управляет повседневной деятельностью отдельных ученых по обработке данных по конкретному проекту науки о данных.
   
4. **Индивидуальные авторы проекта:** специалисты по обработке данных, бизнес-аналитики, инженеры по обработке данных, архитекторы и другие, которые выполняют проект по науке о данных.

> [!NOTE]
> В зависимости от структуры и размера предприятия, один человек может играть более одной роли, или более одного человека может заполнить роль.

## <a name="group-manager-tasks"></a>Задачи менеджера группы

Менеджер группы или назначенный системный администратор TDSP выполняет следующие задачи по принятию TDSP:

- Создает **организацию** Azure DevOps и групповой проект в рамках организации. 
- Создает **репозиторий шаблона проекта** в группе Azure DevOps и загорит его из репозитория шаблона проекта, разработанного командой Microsoft TDSP. Репозиторий шаблона проекта Microsoft TDSP предоставляет:
  - **Стандартизированная структура каталога,** включая каталоги данных, кода и документов.
  - Набор **стандартизированных шаблонов документов** для руководства эффективным процессом анализа данных.
- Создает **репозиторий утилиты**и заседает из репозитория утилиты, разработанного командой Microsoft TDSP. Репозиторий утилиты TDSP от корпорации Майкрософт предоставляет набор полезных утилит, чтобы сделать работу ученого по обработке данных более эффективной. Утилита репозитория Майкрософт включает утилиты для интерактивного исследования данных, анализа, отчетности и базового моделирования и отчетности.
- Настройка **политики управления безопасностью** для учетной записи организации.

Для получения подробных [Group Manager tasks for a data science team](group-manager-tasks.md)инструкций см.

## <a name="team-lead-tasks"></a>Задачи руководителя команды

Руководитель группы или назначенный администратор проекта выполняет следующие задачи по принятию TDSP:

- Создает командный **проект** в организации Azure DevOps группы.
- Создает **репозиторий шаблона проекта** в проекте и загонит его из репозитория шаблона группового проекта, созданного менеджером группы или делегатом.
- Создает **репозиторий утилиты команды,** загонит его из репозитория утилиты группы и добавляет утилиты для конкретных команд в репозиторий.
- Дополнительно [создаетхранилище файлов Azure](https://azure.microsoft.com/services/storage/files/) для хранения полезных ресурсов данных для группы. Другие члены команды могут подключать это облачное хранилище файлов к рабочим столам аналитики.
- Дополнительно монтирует хранилище файлов Azure на **DSVM** группы и добавляет в него ресурсы группы данных.
- Настройка **управления безопасностью** путем добавления членов группы и настройки их разрешений.

Подробные инструкции можно узнать [для группы по науке о данных.](team-lead-tasks.md)


## <a name="project-lead-tasks"></a>Задачи руководителя проекта

Руководитель проекта выполняет следующие задачи по принятию TDSP:

- Создает **репозиторий проекта** в командном проекте и загорит его из репозитория шаблона проекта.
- Дополнительно **создаетхранилище файлов Azure** для хранения ресурсов данных проекта.
- Дополнительно монтирует хранилище файлов Azure в **DSVM** и добавляет в него ресурсы данных проекта.
- Настройка **управления безопасностью** путем добавления участников проекта и настройки их разрешений.

Подробные инструкции [см.](project-lead-tasks.md)

## <a name="project-individual-contributor-tasks"></a>Задачи отдельного участника проекта

Индивидуальный участник проекта, обычно специалист по обработке данных, выполняет следующие задачи с помощью TDSP:

- Клонирует **репозиторий проекта,** созданный ведущим проектом.
- Дополнительно монтирует общую команду и проектирует **хранилище файлов Azure** на их **виртуальную машину Data Science** (DSVM).
- Выполняет проект.

Подробные инструкции по посадке на проект можно узнать [из задач Project Individual Contributor для группы по науке о данных.](project-ic-tasks.md)

## <a name="data-science-project-execution-workflow"></a>Рабочий процесс выполнения проекта Data science

Следуя соответствующим учебникам, специалисты по обработке данных, руководители проектов и руководители групп могут создавать рабочие элементы для отслеживания всех задач и этапов проекта от начала до конца. Использование Azure Repos способствует сотрудничеству между учеными в области данных и гарантирует, что артефакты, созданные при выполнении проекта, контролируются и передаются всем участникам проекта. Azure DevOps позволяет связывать рабочие элементы Azure Boards с ветвями репозитория Репосaz и легко отслеживать, что было сделано для рабочего элемента.

Следующая цифра описывает рабочий процесс TDSP для выполнения проекта:

![Типичный рабочий процесс проекта по науке данных](./media/roles-tasks/overview-project-execute.png)

Шаги рабочего процесса могут быть сгруппированы в три действия:

- Проект ведет планирование спринта
- Специалисты по обработке данных разрабатывают артефакты на `git` ветвях для решения рабочих элементов
- Ведущие проекта или другие члены группы делают обзоры кода и объединяют рабочие ветви в основной ветви

Подробные инструкции по выполнению [Agile development of data science projects](agile-development.md)рабочего процесса по проекту см.

# <a name="how-to-run-jupyter-notebooks-in-your-workspace"></a>Запуск записных книжек Jupyter в рабочей области
[!INCLUDE [applies-to-skus](../../includes/aml-applies-to-basic-enterprise-sku.md)]

Узнайте, как запускать записные книжки Jupyter непосредственно в рабочей области в Машинное обучение Azure Studio. Хотя вы можете запустить [Jupyter](https://jupyter.org/) или [JupyterLab](https://jupyterlab.readthedocs.io), вы также можете редактировать и запускать свои записные книжки, не закрывая рабочую область.

Узнайте, как это можно делать:

* Создание записных книжек Jupyter в рабочей области
* Запуск эксперимента из записной книжки
* Изменение окружения записных книжек
* Найти сведения об экземплярах вычислений, используемых для запуска записных книжек

## <a name="prerequisites"></a>Предварительные требования

* Подписка Azure. Если у вас еще нет подписки Azure, [создайте бесплатную учетную запись](https://aka.ms/AMLFree), прежде чем начинать работу.
* Рабочая область машинного обучения. См. сведения о [создании рабочей области машинного обучения Azure](how-to-manage-workspace.md).

## <a name="create-notebooks"></a><a name="create"></a>Создание записных книжек

В рабочей области Машинное обучение Azure создайте новую записную книжку Jupyter и начните работу. Вновь созданная Записная книжка хранится в хранилище рабочей области по умолчанию. Эта записная книжка может быть предоставлена любому пользователю, имеющему доступ к рабочей области. 

Чтобы создать новую записную книжку, сделайте следующее: 

1. Откройте рабочую область в [машинное обучение Azure Studio](https://ml.azure.com).
1. В левой части выберите **записные книжки**. 
1. Щелкните значок **создать новый файл** над списком **файлов пользователей** в разделе **Мои файлы** .

    :::image type="content" source="media/how-to-run-jupyter-notebooks/create-new-file.png" alt-text="Создать файл":::

1. Присвойте файлу имя. 
1. Для Jupyter Notebook файлов выберите в качестве типа файла **записную книжку Python** .
1. Выберите каталог файлов.
1. Щелкните **Создать**.

> [!TIP]
> Кроме того, можно создавать текстовые файлы.  Выберите **текст** в качестве типа файла и добавьте расширение к имени (например, MyFile.py или MyFile. txt).  

Вы также можете отправлять папки и файлы, включая записные книжки, с помощью инструментов, которые находятся в верхней части страницы записных книжек.  В разделе Предварительный просмотр отображаются записные книжки и большинство типов текстовых файлов.  Для большинства других типов файлов Предварительная версия недоступна.

### <a name="clone-samples"></a>Клонирование примеров

Рабочая область содержит папку **Samples** с записными книжками, предназначенными для ознакомления с пакетом SDK и служить примерами для собственных проектов машинного обучения.  Эти записные книжки можно клонировать в собственную папку в контейнере хранилища рабочей области.  

Пример см. в разделе [учебник. Создание первого эксперимента ML](tutorial-1st-experiment-sdk-setup.md#azure).

### <a name="a-nameterminal-use-files-from-git-and-version-my-files"></a><a name="terminal">Использовать файлы из Git и версии My Files

Доступ ко всем операциям Git можно получить с помощью окна терминала. Все файлы и папки Git будут храниться в файловой системе рабочей области.

> [!NOTE]
> Добавьте файлы и папки в папку **~/клаудфилес/коде/усерс** , чтобы они отображались во всех средах Jupyter.

Чтобы получить доступ к терминалу, выполните следующие действия.

1. Откройте рабочую область в [машинное обучение Azure Studio](https://ml.azure.com).
1. В левой части выберите **записные книжки**.
1. Выберите любую записную книжку, расположенную в разделе **User Files (файлы пользователей** ) в левой части.  Если у вас нет записных книжек, сначала [Создайте записную книжку](#create) .
1. Выберите целевой объект **вычислений** или создайте новый и дождитесь его запуска.
1. Выберите значок **Открыть терминал** .

    :::image type="content" source="media/how-to-run-jupyter-notebooks/open-terminal.png" alt-text="Откройте терминал":::

1. Если значок не отображается, нажмите кнопку **...** справа от целевого объекта вычислений и выберите **Открыть терминал** .

    :::image type="content" source="media/how-to-run-jupyter-notebooks/alt-open-terminal.png" alt-text="Открыть терминал из...":::


Дополнительные сведения о [клонировании репозиториев Git в файловую систему рабочей области](concept-train-model-git-integration.md#clone-git-repositories-into-your-workspace-file-system).


### <a name="share-notebooks-and-other-files"></a>Общий доступ к записным книжкам и другим файлам

Скопируйте и вставьте URL-адрес для совместного использования записной книжки или файла.  Только другие пользователи рабочей области смогут получить доступ к этому URL-адресу.  Дополнительные сведения о [предоставлении доступа к рабочей области](how-to-assign-roles.md).

## <a name="edit-a-notebook"></a>Изменение записной книжки

Чтобы изменить записную книжку, откройте любую записную книжку, расположенную в разделе **User Files** рабочей области. Щелкните ячейку, которую вы хотите изменить. 

Если выполняется выполняющийся вычислительный экземпляр, можно также использовать функцию завершения кода на платформе [IntelliSense](https://code.visualstudio.com/docs/editor/intellisense)в любой записной книжке Python.

Вы также можете запустить Jupyter или JupyterLab на панели инструментов записной книжки.  Машинное обучение Azure не предоставляет обновления и исправляет ошибки из Jupyter или JupyterLab, так как они являются продуктами с открытым исходным кодом вне границ служба поддержки Майкрософт.

|---------|---------|

## <a name="delete-a-notebook"></a>Удаление записной книжки

Вы *не можете* удалить **образцы** записных книжек.  Эти записные книжки входят в состав студии и обновляются каждый раз при публикации нового пакета SDK.  

Удалить записные книжки для **файлов пользователей** *можно* одним из следующих способов:

* В студии выберите **...** в конце папки или файла.  Убедитесь, что используется поддерживаемый браузер (Microsoft ребр, Chrome или Firefox).
* На любой панели инструментов записной книжки выберите [**Открыть терминал**](#terminal) , чтобы открыть окно терминала для вычислительного экземпляра.
* В Jupyter или JupyterLab со своими инструментами.

## <a name="run-an-experiment"></a>Выполнение эксперимента

Чтобы запустить эксперимент из записной книжки, сначала подключитесь к выполняющемуся [вычислительному экземпляру](concept-compute-instance.md). Если у вас нет вычислительного экземпляра, выполните следующие действия, чтобы создать его. 

1. Выберите **+** на панели инструментов записной книжки. 
2. Назовите вычисление и выберите **Размер виртуальной машины**. 
3. Щелкните **Создать**.
4. Вычислительный экземпляр автоматически подключается к записной книжке и теперь может выполнять ячейки.

Вы можете просматривать и использовать созданные вами экземпляры вычислений.  **Файлы пользователей** хранятся отдельно от виртуальной машины и являются общими для всех экземпляров вычислений в рабочей области.

### <a name="view-logs-and-output"></a>Просмотр журналов и выходных данных

Используйте [мини-приложения записной книжки](https://docs.microsoft.com/python/api/azureml-widgets/azureml.widgets?view=azure-ml-py) для просмотра хода выполнения и журналов. Мини-приложение является асинхронным и предоставляет обновления до тех пор, пока обучение не будет завершено. Машинное обучение Azure мини-приложений также поддерживаются в Jupyter и Жуптерлаб.

## <a name="change-the-notebook-environment"></a>Изменение окружения записных книжек

Панель инструментов записной книжки позволяет изменить среду, в которой выполняется Записная книжка.  

Эти действия не будут изменять состояние записной книжки или значения любых переменных в записной книжке.

|Действие  |Результат  |
|---------|---------| --------|
|Завершение ядра     |  Останавливает любую работающую ячейку. При выполнении ячейки автоматически перезапускается ядро. |
|Переход к другой разделу рабочей области     |     Работающие ячейки остановлены. |

Эти действия будут сбрасывать состояние записной книжки и сбрасывать все переменные в записной книжке.

|Действие  |Результат  |
|---------|---------| --------|
| Изменение ядра | Записная книжка использует новое ядро |
| Переключить вычисление    |     Записная книжка автоматически использует новое вычисление. |
| Сбросить вычисление | Запуск при попытке запустить ячейку |
| Прерывать вычисление     |    Ни одна ячейка не будет выполняться  |
| Открытие записной книжки в Jupyter или JupyterLab     |    Записная книжка открыта на новой вкладке.  |

### <a name="add-new-kernels"></a>Добавить новые ядра

В записной книжке будут автоматически найдены все ядра Jupyter, установленные на подключенном вычислительном экземпляре.  Добавление ядра к вычислительному экземпляру:

1. Выберите [**Открыть терминал**](#terminal) на панели инструментов записной книжки.
1. Используйте окно терминала для создания новой среды.
1. Активируйте среду.  Например, после создания `newenv`:

    ```shell
    source activate newenv
    python -m ipykernel install --user --name newenv --display-name "Python (newenv)"
    ```

Можно установить любой из [доступных ядер Jupyter](https://github.com/jupyter/jupyter/wiki/Jupyter-kernels) .

### <a name="status-indicators"></a>Индикаторы состояния

Индикатор рядом с раскрывающимся списком " **Вычисление** " показывает свое состояние.  Состояние также отображается в самом раскрывающемся списке.  

|Цвет |Состояние вычислений |
|---------|---------| 
| Зеленый | Выполнение вычислений |
| Красный |Сбой при вычислении | 
| Черный | Вычисление остановлено |
|  Светло-синий |Вычисление создания, запуск, перезапуск, Настройка |
|  Серый |Вычисление удаления, остановка |

Индикатор рядом с раскрывающимся списком **ядро** показывает его состояние.

|Цвет |Состояние ядра |
|---------|---------|
|  Зеленый |Ядро подключено, бездействует, занято|
|  Серый |Ядро не подключено |

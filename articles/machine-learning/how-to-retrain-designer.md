# <a name="retrain-models-with-azure-machine-learning-designer-preview"></a>Переобучение моделей с помощью конструктора Машинного обучения Azure (предварительная версия)
[!INCLUDE [applies-to-skus](../../includes/aml-applies-to-enterprise-sku.md)]

В этой статье вы узнаете, как использовать дизайнера машинного обучения Azure для переподготовки модели машинного обучения. Узнайте, как использовать опубликованные конвейеры для автоматизации рабочих процессов машинного обучения для переподготовки.

Вы узнаете, как выполнять следующие задачи:

> [!div class="checklist"]
> * обучить модель машинного обучения;
> * Создайте параметр конвейера.
> * Опубликуйте свой конвейер обучения.
> * Переподготовка модели.

## <a name="prerequisites"></a>Предварительные требования

* Подписка Azure. Если у вас нет подписки На Azure, создайте [бесплатную учетную запись](https://aka.ms/AMLFree).
* Рабочее пространство для машин Azure Machine Learning с помощью Корпоративного SKU.

Эта статья предполагает, что у вас есть базовые знания о строительстве трубопроводов в конструкторе. Вы можете ознакомиться с [руководством](tutorial-designer-automobile-price-train-score.md) по началу работы с конструктором. 

### <a name="sample-pipeline"></a>Пример конвейера

Конвейер, используемый в этой статье, представляет собой измененную версию той, которая содержится в [примере 3: прогноз дохода.](how-to-designer-sample-classification-predict-income.md) Он использует модуль [данных импорта](algorithm-module-reference/import-data.md) вместо набора данных образца, чтобы показать вам, как обучить модель с помощью собственных данных.

![Скриншот, на который показан измененный образец конвейера с полем, выделяющим модуль данных импорта](./media/how-to-retrain-designer/modified-sample-pipeline.png)

## <a name="train-a-machine-learning-model"></a>Обучение модели машинного обучения.

Для переподготовки модели нужна начальная модель. В этом разделе вы узнаете, как обучить модель и получить доступ к сохраненной модели с помощью дизайнера.

1. Выберите модуль **данных импорта.**
1. На панели свойств укажите источник данных.

   ![Скриншот, на которых показана примерконфигурация модуля данных импорта](./media/how-to-retrain-designer/import-data-settings.png)

   В этом примере данные хранятся в [хранилище данных Azure.](how-to-access-data.md) Если у вас еще нет хранилища данных, его можно создать сейчас, выбрав **новый магазин данных.**

1. Укажите путь к данным. Вы также можете выбрать **путь просмотра** для просмотра в хранилище данных. 
1. Выберите **Выполнить** в верхней части холста.
    
   > [!NOTE]
   > Если вы уже установили вычисление по умолчанию для этого проекта конвейера, конвейер будет работать автоматически. В противном случае, вы можете следить за запросами на панели настроек, чтобы установить один сейчас.

### <a name="find-your-trained-model"></a>Найти обученную модель

Конструктор сохраняет все выходы конвейера, включая обученные модели, на учетную запись хранения по умолчанию. Вы также можете получить доступ к обученной модели непосредственно в конструкторе:

1. Подождите, пока конвейер закончит работу.
1. Выберите модуль **Train Model** (Обучение модели).
1. На панели настроек выберите **журналы выходов.**
1. Выберите значок **вывода View** и следуйте инструкциям во всплывающем окне, чтобы найти обученную модель.

![Скриншот, который показывает, как скачать обученную модель](./media/how-to-retrain-designer/trained-model-view-output.png)

## <a name="create-a-pipeline-parameter"></a>Создание параметра конвейера

Добавление параметров конвейера к динамически установленным переменным во время выполнения. Для этого конвейера добавьте параметр для пути обучения данных, чтобы можно было переучивать модель на новом наборе данных.

1. Выберите модуль **данных импорта.**
1. В настройках панели выберите эллипсы над полем **Пути.**
1. Выберите **Добавить к параметру конвейера**.
1. Укажите имя параметра и значение по умолчанию.

   > [!NOTE]
   > Вы можете проверить и отсеить параметры конвейера, выбрав значок **передачи настроек** рядом с названием проекта конвейера. 

![Скриншот, который показывает, как создать параметр конвейера](media/how-to-retrain-designer/add-pipeline-parameter.png)

## <a name="publish-a-training-pipeline"></a>Публикация учебного конвейера

Когда вы публикуете конвейер, он создает конечную точку конвейера. Конечные точки трубопровода позволяют повторно использовать и управлять конвейерами для повторяемости и автоматизации. В этом примере вы настроили конвейер для переподготовки.

1. Выберите **Опубликовать** выше дизайнерского холста.
1. Выберите или создайте конечную точку конвейера.

   > [!NOTE]
   > Можно опубликовать несколько конвейеров в одну конечную точку. Каждому конвейеру в конечном пункте дается номер версии, который можно указать при вызове конечной точки конвейера.

1. Нажмите кнопку **Опубликовать**.

## <a name="retrain-your-model"></a>Повторное обучение модели

Теперь, когда у вас есть опубликованный конвейер обучения, вы можете использовать его для переподготовки модели с помощью новых данных. Вы можете отправить запуски из конечных точек конвейера с портала Azure или отправить их программно.

### <a name="submit-runs-by-using-the-designer"></a>Отправить работает с помощью дизайнера

Используйте следующие шаги, чтобы отправить конечную точку конвейера от конструктора:

1. Перейдите на страницу **Endpoints.**
1. Выберите вкладку **конечных точек трубопровода.**
1. Выберите конечную точку конвейера.
1. Выберите вкладку **«Опубликовано конвейеров».**
1. Выберите выбранный конвейер, который необходимо запустить.
1. Нажмите кнопку **Отправить**.
1. В поле диалога настройки можно указать новое значение для значения пути ввода данных. Это значение указывает на новый набор данных.

![Скриншот, который показывает, как настроить параметризированный запуск трубопровода в конструкторе](./media/how-to-retrain-designer/published-pipeline-run.png)

### <a name="submit-runs-by-using-code"></a>Отправка работает с помощью кода

В панели обзора можно найти конечную точку REST опубликованного конвейера. Позвонив в конечную точку, можно переквалифицировать опубликованный конвейер.

Чтобы сделать вызов REST, вам нужен заголовок аутентификации типа OAuth 2.0. Для получения информации о настройке аутентификации в рабочем пространстве и совершении параметризированного вызова REST [см.](tutorial-pipeline-batch-scoring-classification.md#publish-and-run-from-a-rest-endpoint)

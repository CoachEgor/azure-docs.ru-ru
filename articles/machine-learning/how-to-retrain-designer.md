# <a name="retrain-models-with-azure-machine-learning-designer-preview"></a>Переобучение моделей с помощью конструктора Машинного обучения Azure (предварительная версия)
[!INCLUDE [applies-to-skus](../../includes/aml-applies-to-enterprise-sku.md)]

В этой статье вы узнаете, как использовать дизайнера машинного обучения Azure для переподготовки модели машинного обучения. Для автоматизации рабочего процесса и набора параметров для обучения модели новым данным будут использоваться опубликованные конвейеры. 

Вы узнаете, как выполнять следующие задачи:

> [!div class="checklist"]
> * обучить модель машинного обучения;
> * Создайте параметр конвейера.
> * Опубликуйте свой конвейер обучения.
> * Переподготовка модели с новыми параметрами.

## <a name="prerequisites"></a>Предварительные требования

* Рабочая область Машинного обучения Azure с номером SKU Enterprise.
* Набор данных, доступный для конструктора. Может иметь одно из следующих значений.
   * Набор данных, зарегистрированный в Azure Machine Learning
    
     **-или-**
   * Файл данных, хранящийся в хранилище данных Azure Machine Learning.
   
Для получения информации о доступе к данным с помощью дизайнера [см. Как импортировать данные в конструктор.](how-to-designer-import-data.md)

Эта статья также предполагает, что у вас есть базовые знания о строительстве трубопроводов в конструкторе. Для управляемого введения, завершить [учебник](tutorial-designer-automobile-price-train-score.md).

Конвейер, используемый в этой статье, представляет собой измененную версию [Sample 3: Прогноз дохода.](samples-designer.md#classification-samples) Конвейер использует модуль [«Импортные данные»](algorithm-module-reference/import-data.md) вместо набора данных образца, чтобы показать, как обучать модели с использованием собственных данных.

## <a name="create-a-pipeline-parameter"></a>Создание параметра конвейера

Создавайте параметры конвейера для динамических переменных во время выполнения. В этом примере вы измените путь обучения данных с фиксированного значения на параметр, чтобы можно было переквалифицировать модель на различные данные.

1. Выберите модуль **данных импорта.**

    > [!NOTE]
    > В этом примере используется модуль данных импорта для доступа к данным в зарегистрированном хранилище данных. Тем не менее, вы можете следовать аналогичным шагам, если вы используете альтернативные шаблоны доступа к данным.

1. В панели детали модуля, справа от холста, выберите источник данных.

1. Введите путь к вашим данным. Вы также можете выбрать **путь просмотра** для просмотра дерева файлов. 

1. Мышь над полем **Пути** и выберите эллипсы над появляются **полем Пути.**

    ![Скриншот, который показывает, как создать параметр конвейера](media/how-to-retrain-designer/add-pipeline-parameter.png)

1. Выберите **Добавить к параметру конвейера**.

1. Укажите имя параметра и значение по умолчанию.

   > [!NOTE]
   > Вы можете проверить и отсеить параметры конвейера, выбрав значок **передачи настроек** рядом с названием проекта конвейера. 

1. Щелкните **Сохранить**.

1. Отправить пробег конвейера.

## <a name="find-a-trained-model"></a>Найти обученную модель

Конструктор сохраняет все выходные данные конвейера, включая обученные модели, для учетной записи хранения рабочего пространства по умолчанию. Вы также можете получить доступ к обученным моделям непосредственно в дизайнере:

1. Подождите, пока конвейер закончит работу.
1. Выберите модуль **Train Model** (Обучение модели).
1. В модуле детали панели, справа от холста, выберите **Выходы и журналы**.
1. Вы можете найти свою модель в **других выводах** вместе с журналами выполнения.
1. В качестве альтернативы выберите значок **вывода view.** Отсюда вы можете следовать инструкциям в диалоге, чтобы перейти непосредственно к вашему магазину данных. 

![Скриншот, который показывает, как скачать обученную модель](./media/how-to-retrain-designer/trained-model-view-output.png)

## <a name="publish-a-training-pipeline"></a>Публикация учебного конвейера

Опубликовать конвейер в конечную точку конвейера, чтобы легко использовать конвейеры в будущем. Конечная точка конвейера создает конечную точку REST для ввода конвейера в будущем. В этом примере конечная точка конвейера позволяет повторно использовать конвейер для переподготовки модели на разных данных.

1. Выберите **Опубликовать** выше дизайнерского холста.
1. Выберите или создайте конечную точку конвейера.

   > [!NOTE]
   > Можно опубликовать несколько конвейеров в одну конечную точку. Каждому конвейеру в данной точке указания номера версии, который можно указать при вызове конечной точки конвейера.

1. Нажмите кнопку **Опубликовать**.

## <a name="retrain-your-model"></a>Повторное обучение модели

Теперь, когда у вас есть опубликованный конвейер обучения, вы можете использовать его для переподготовки модели на новых данных. Вы можете отправлять трассы из конечных точек конвейера из рабочего пространства студии или программно.

### <a name="submit-runs-by-using-the-designer"></a>Отправить работает с помощью дизайнера

Используйте следующие шаги, чтобы отправить параметризированную конечную точку конвейера от конструктора:

1. Перейдите на страницу **Endpoints** в рабочем пространстве студии.
1. Выберите вкладку **конечных точек трубопровода.** Затем выберите конечную точку конвейера.
1. Выберите вкладку **«Опубликовано конвейеров».** Затем выберите версию конвейера, которую вы хотите запустить.
1. Нажмите кнопку **Submit** (Отправить).
1. В поле диалога настройки можно указать значения параметров для выполнения. В этом примере обновите путь данных для обучения модели с помощью набора данных, не относящегосшаму в США.

![Скриншот, который показывает, как настроить параметризированный запуск трубопровода в конструкторе](./media/how-to-retrain-designer/published-pipeline-run.png)

### <a name="submit-runs-by-using-code"></a>Отправка работает с помощью кода

В панели обзора можно найти конечную точку REST опубликованного конвейера. Позвонив в конечную точку, можно переквалифицировать опубликованный конвейер.

Чтобы сделать вызов REST, вам нужен заголовок аутентификации типа OAuth 2.0. Для получения информации о настройке аутентификации в рабочем пространстве и совершении параметризированного вызова REST [см.](tutorial-pipeline-batch-scoring-classification.md#publish-and-run-from-a-rest-endpoint)

# <a name="sample-3---classification-predict-credit-risk"></a>Пример 3. Классификация: Прогнозирование кредитного риска

Узнайте, как создать классификатор машинного обучения, не написав ни одной строки кода с помощью визуального интерфейса. Этот пример обучает **высококлассное дерево принятия решений** для прогнозирования кредитного риска (высокое или низкое) на основе сведений о кредитных приложениях, таких как история кредита, возраст и количество кредитных карт.

Так как вопрос отвечает "какой из них?" Это называется проблемой классификации. Тем не менее можно применить тот же фундаментальный процесс для решения любых проблем машинного обучения, будь то регрессия, классификация, кластеризация и т. д.

## <a name="data"></a>Data

В примере используется набор данных с кредитными картами для Германии из репозитория Ирвин UC.
Набор данных содержит 1 000 выборок с 20 компонентами и одной меткой. Каждый пример представляет человека. Функции включают в себя числовые и функции категорий. Значения функций категории см. на [веб-сайте UCI](https://archive.ics.uci.edu/ml/datasets/Statlog+%28German+Credit+Data%29) . Последний столбец — это метка, которая обозначает кредитный риск и имеет только два возможных значения: высокий кредитный риск = 2 и низкий кредитный риск = 1.

## <a name="experiment-summary"></a>Сводка по экспериментам

Чтобы создать эксперимент, выполните следующие действия.

1. Перетащите на холст эксперимента модуль набора данных для немецкой кредитной карты UCI.
1. Добавьте модуль **изменения метаданных** , чтобы мы могли добавить значимые имена для каждого столбца.
1. Добавьте модуль **Split Data (разделение данных** ) для создания обучающих и проверочных наборов. Задайте в качестве доли строк в первом выходном наборе данных значение 0,7. Этот параметр указывает, что 70% данных будут выведены по левому порту модуля, а остальные — к нужному порту. Мы используем левый набор данных для обучения и правую единицу для тестирования.
1. Добавьте модуль **высококлассного дерева принятия решений с двумя классами** для инициализации классификатора дерева повышенных решений.
1. Добавьте модуль **обучение модели** . Подключите классификатор из предыдущего шага к левому порту ввода для **модели обучения**. Добавьте обучающий набор (левый порт вывода **разделенных данных**) к правому порту ввода **модели обучения**. **Модель обучения** будет обучать классификатор.
1. Добавьте модуль **Оценка модели** и подключите к нему модуль **обучение модели** . Затем добавьте набор тестов (правый порт **разделенных данных**) в **модель оценки**. **Модель оценки** сделает прогнозы. Для просмотра прогнозов и вероятностей положительного класса можно выбрать его выходной порт.
1. Добавьте модуль « **Анализ модели** » и подключите счет с расчетом на левый порт ввода. Чтобы просмотреть результаты оценки, выберите порт вывода модуля **Анализ модели** и щелкните **визуализировать**.

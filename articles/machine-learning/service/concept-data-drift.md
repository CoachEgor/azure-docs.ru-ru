# <a name="what-is-data-drift-monitoring-preview"></a>Что такое мониторинг отклонения данных (Предварительная версия)?

Смещение данных — это изменение распределения данных. В контексте машинного обучения обученные модели машинного обучения могут испытывать снижение производительности предсказания из-за смещения. Отслеживание различий между обучающими данными и данными, используемыми для прогнозирования, может помочь выявить проблемы с производительностью.

Модели машинного обучения так же хороши, как и данные, используемые для их обучения. Развертывание моделей в рабочей среде без наблюдения за производительностью может привести к незамеченным и негативным последствиям. С помощью мониторинга несмещенных данных можно обнаружить и адаптироваться к смещению данных. 

## <a name="when-to-monitor-for-data-drift"></a>Когда следует отслеживать смещение данных?

Ниже перечислены метрики, которые можно отслеживать:

+ Величина смещения (коэффициент смещения)
+ Причина смещения (отклонения от вклада в функцию)
+ Метрики расстояния (Dresdner, энергия и т. д.)

С помощью этого мониторинга можно настроить предупреждения или действия при обнаружении отклонения, а анализу данных может исследовать основную причину проблемы. 

Если вы считаете, что входные данные для развернутой модели могут измениться, следует рассмотреть возможность определения расхождения данных.

## <a name="how-data-drift-is-monitored-in-azure-machine-learning-service"></a>Как отслеживается смещение данных в службе Машинное обучение Azure

При использовании **службы машинное обучение Azure**отклонения данных отслеживаются через наборы данных или развертывания. Для отслеживания смещения данных можно указать базовый набор данных — обычно это набор обучающих данных для модели. Второй набор данных — обычно входные данные модели, собранные из развертывания, тестируются на основе базового набора данных. Оба набора данных являются [профилированием](how-to-explore-prepare-data.md#explore-with-summary-statistics) и входными данными для службы отслеживания смещения данных. Модель машинного обучения обучена для обнаружения различий между двумя наборами данных. Производительность модели преобразуется в коэффициент смещения, который измеряет величину смещения между двумя наборами данных. Используя [интерпретируемость модели](machine-learning-interpretability-explainability.md) , вычисляются функции, влияющие на коэффициент смещения. В профиле набора данных сохранится статистическая информация о каждом компоненте. 

## <a name="data-drift-metric-output"></a>Выходные данные метрики смещения данных

Существует несколько способов просмотра метрик смещения.

* Используйте мини-приложение [Jupyter.](https://docs.microsoft.com/python/api/azureml-widgets/azureml.widgets?view=azure-ml-py) `RunDetails`
* Используйте функцию для любого `datadriftRun` объекта. `get_metrics()`
* Просмотр метрик в портал Azure модели

Следующие метрики сохраняются в каждой итерации выполнения для задачи «смещение данных».

|Метрика|Описание|
--|--|
wasserstein_distance|Статистическое расстояние, определенное для одномерного числового распределения.|
energy_distance|Статистическое расстояние, определенное для одномерного числового распределения.|
datadrift_coefficient|Формально Matthews коэффициент корреляции — вещественное число в диапазоне от-1 до 1. В контексте смещения значение 0 означает отсутствие смещения, а 1 — Максимальное смещение.|
datadrift_contribution|Важность признаков компонентов, которые вносят вклад в отклонения.|

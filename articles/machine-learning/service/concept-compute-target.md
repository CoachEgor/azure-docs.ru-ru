#  <a name="what-are-compute-targets-in-azure-machine-learning-service"></a>Каковы целевые объекты вычисления в службе машинного обучения Azure? 

Объект **целевого объекта вычислений** является указанного ресурса или среде вычислений, где выполняется сценарий обучения или узел развертывания службы. Это расположение может быть на локальном компьютере или облачных вычислительных ресурсов. С помощью вычислений, целевые объекты позволяют легко позже изменить среду службы вычислений без изменения кода.  

В обычной модели жизненного цикла разработки которые могут быть:
1. Для начала разработки и экспериментировать с небольшой объем данных. На этом этапе мы рекомендуем в локальной среде (локальный компьютер или виртуальную Машину на основе облака) в качестве целевого объекта вычислений. 
2. Масштаб более крупный набор данных или распределенного обучения, с помощью одного из этих [целевых объектов вычислений обучения](#train).  
3. Как только модель готова, развернуть его в веб-размещения среды или устройства Интернета вещей с помощью одного из этих [целевых объектов вычислений развертывания](#deploy).

Вычислительные ресурсы для целевыми объектами вычислений, присоединяются к [рабочей области](concept-workspace.md). Для вычислительных ресурсов, отличных от локальной системы являются общими для пользователей рабочей области.

## <a name="azure-machine-learning-compute-managed"></a>Вычислений машинного обучения Azure (управляемый код)

Управляемый вычислительный ресурс создается и управляется службой машинного обучения Azure. Этот вычислений оптимизирован для рабочих нагрузок машины обучения. Azure вычислений для машинного обучения является единственным управляемой вычислительной по состоянию на 30 мая 2019 г. В будущем могут добавляться дополнительные управляемые вычислительные ресурсы.

Вычислений машинного обучения Azure можно использовать для обучения, а также для определения пакета (Предварительная версия).  С этим ресурсом вычислений у вас есть:

* Одним или несколькими node кластера
* Автоматически масштабируется при каждой отправке запуска 
* Автоматического управления кластером и планирования заданий 
* поддерживает ресурсы ЦП и GPU;

Можно создать экземпляры вычислений машинного обучения Azure на портале Azure, с помощью пакета SDK или с помощью интерфейса командной строки. При создании автоматически является частью рабочей области в отличие от других типов целевых объектов вычислений.

## <a name="unmanaged-compute"></a>Неуправляемые вычислений

— Целевой объект вычислений неуправляемый *не* управляется службой машинного обучения Azure. Создать этот тип целевого объекта вычислений за пределами машинного обучения Azure, а затем присоединить его к рабочей области. Неуправляемые вычислительные ресурсы можно требуются дополнительные действия для поддержания или для повышения производительности для рабочих нагрузок машин обучения.

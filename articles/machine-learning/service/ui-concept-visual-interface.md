# <a name="what-is-the-visual-interface-for-azure-machine-learning-service"></a>Что такое визуального интерфейса для службы машинного обучения Azure? 

Визуальный интерфейс (Предварительная версия) для службы машинного обучения Azure предоставляет возможность подготовки данных, обучение, тестирование, развертывание, управление и отслеживание моделей машинного обучения без написания кода.

Никакое программирование не требуется, визуально подключения [наборы данных](#dataset) и [модули](#module) для построения модели.

Службы машинного обучения Azure использует визуальный интерфейс [рабочей области](concept-workspace.md) для:

+ Запись артефакты [поэкспериментировать](#experiment) выполняется в рабочую область.
+ Доступ [наборы данных](#dataset).
+ Используйте [вычислительные ресурсы](#compute) в рабочей области, чтобы запустить эксперимент. 
+ Зарегистрировать [моделей](concept-azure-machine-learning-architecture.md#models).
+ [Развертывание](#deployment) модели как веб-служб на вычислительные ресурсы в рабочей области.

![Обзор visual интерфейса](media/ui-concept-visual-interface/overview.png)

## <a name="workflow"></a>Рабочий процесс

Визуальный интерфейс дает visual, интерактивные холст, чтобы быстро создавать, тестировать и выполнять итерацию модели. 

+ Вы и перетащите [модули](#module) на холст.
+ Объединять модули форму [поэкспериментировать](#experiment).
+ Запустите эксперимент с использованием вычислительных ресурсов рабочей области службы машинного обучения.
+ Итерации разработки модели, изменив эксперимента и запустить ее еще раз.
+ Когда вы будете готовы, преобразование вашей **обучающий эксперимент** для **прогнозный эксперимент**.
+ [Развертывание](#deployment) прогнозный эксперимент как веб-службы, чтобы модель может осуществляться другими пользователями.

## <a name="experiment"></a>Эксперимент

Создать эксперимент с нуля, или использовать существующий пример эксперимента в качестве шаблона.  Каждый раз при запуске эксперимента, артефакты хранятся в рабочей области.

Эксперимент состоит из наборов данных и аналитических модулей, которые соединяются вместе для создания модели. В частности, правильный эксперимент имеет следующие характеристики.

* Наборы данных могут быть связаны только с модулями.
* Модули могут быть связаны для наборов данных или других модулей.
* Все входные порты для модулей должны иметь связь с потоком данных.
* Все необходимые параметры для каждого модуля необходимо определить.

Пример простого эксперимента, см. в разделе [краткое руководство: Подготовка и визуализируйте данные без написания кода в машинном обучении Azure](ui-quickstart-run-experiment.md).

Подробное пошаговое руководство по решения для прогнозной аналитики, см. в разделе [руководства: Прогнозирования стоимости автомобиля в визуальный интерфейс](ui-tutorial-automobile-price-train-score.md).

## <a name="dataset"></a>Выборка

Набор данных — данные, которые будут переданы визуальный интерфейс для использования в процессе моделирования. Несколько примеров наборов данных, можно поэкспериментировать с и отправлять дополнительные наборы данных по мере необходимости.

## <a name="module"></a>модуль

Модуль — это алгоритм, который можно выполнять с данными. Визуальный интерфейс имеет ряд модулей, начиная с функций ввода данных для обучения, оценки и процессы проверки.

Модуль может иметь ряд параметров, которые можно использовать для настройки внутренних алгоритмов модуля. При выборе модуля на холсте его параметры отображаются на панели свойств справа от холста. Подстройка модели осуществляется изменением параметров в этой области.

![Свойства модуля](media/ui-concept-visual-interface/properties.png)

Разобраться в огромной библиотеке доступных алгоритмов машинного обучения, см. в разделе [алгоритмам и модулям Обзор справочника](../algorithm-module-reference/module-reference.md)

## <a name="compute"></a> Вычислительные ресурсы

Использование вычислительных ресурсов в рабочей области на запуск эксперимента или узле развернутых моделей как веб-службы. Поддерживаются следующие целевые объекты вычислений:


| Целевой объект вычисления | Обучение | Развертывание |
| ---- |:----:|:----:|
| Вычислительная среда Машинного обучения Azure | ✓ | |
| Служба Azure Kubernetes | | ✓ |

Вычислить целевые объекты присоединяются к машинного обучения [рабочей области](concept-workspace.md). Управление целевыми объектами вычислений в рабочей области в [портала Azure](https://portal.azure.com).

## <a name="deployment"></a>Развертывание

После подготовки модели прогнозной аналитики можно развернуть ее как веб-службы прямо из визуального интерфейса.

Веб-служб обеспечивают интерфейс между приложением и оценки модели. Внешнее приложение может взаимодействовать с моделью оценки в режиме реального времени. Вызов веб-служба возвращает результаты прогноза внешнему приложению. Чтобы вызвать веб-службу, передайте ключ API, созданный при развертывании этой службы. Веб-служба работает на основе REST, выбор популярных архитектура для веб-программирования проектов.

Дополнительные сведения о развертывании модели, см. в разделе [руководства: Развертывание модели машинного обучения в визуальный интерфейс](ui-tutorial-automobile-price-deploy.md).

# <a name="what-is-azure-machine-learning-designer-preview"></a>Что такое Машинное обучение Azure конструктор (Предварительная версия)? 
[!INCLUDE [applies-to-skus](../../../includes/aml-applies-to-enterprise-sku.md)]

Конструктор для Машинное обучение Azure позволяет выполнять подготовку данных, обучение, тестирование, развертывание, управление и мониторинг моделей машинного обучения без написания кода.

Программирование не требуется, вы визуально подключаете [наборы данных](#datasets) и [модули](#module) для создания модели.

Конструктор использует [рабочую область](concept-workspace.md) машинное обучение Azure для:

+ Создание, изменение и запуск [конвейеров](#pipeline) в рабочей области.
+ Доступ к [наборам данных](#datasets).
+ Используйте [ресурсы вычислений](#compute) в рабочей области для запуска конвейера. 
+ Регистрация [моделей](concept-azure-machine-learning-architecture.md#models).
+ [Опубликуйте](#publish) конвейеры как конечные точки RESTful.
+ [Разверните](#deployment) модели в качестве конечных точек конвейера (для вывода пакета) или конечных точек реального времени в ресурсах вычислений в рабочей области.

![Общие сведения о конструкторе](media/ui-concept-visual-interface/overview.png)

## <a name="workflow"></a>Рабочий процесс

Конструктор предоставляет интерактивный визуальный холст для быстрой сборки, тестирования и итерации модели. 

+ На холсте можно перетаскивать [наборы данных](#datasets) и [модули](#module) .
+ Соедините модули вместе, чтобы сформировать [конвейер](#pipeline).
+ Запустите конвейер, используя ресурс COMPUTE рабочей области службы Машинное обучение.
+ Выполните итерацию по структуре модели, изменив конвейер и запустив ее снова.
+ Когда вы будете готовы, преобразуйте **обучающий конвейер** в **конвейер вывода**.
+ [Опубликуйте](#publish) конвейер в качестве КОНЕЧНОЙ точки RESTful, если хотите повторно отправить его без создания кода Python.
+ [Разверните](#deployment) конвейер вывода в качестве конечной точки конвейера или конечной точки в режиме реального времени, чтобы предоставить доступ к модели другим пользователям.

## <a name="pipeline"></a>Конвейер

Создайте [конвейер](concept-azure-machine-learning-architecture.md#ml-pipelines) машинного обучения с нуля или используйте существующий пример конвейера в качестве шаблона. Каждый раз, когда вы запускаете конвейер, артефакты хранятся в рабочей области. Запуски конвейеров группируются в [эксперименты](concept-azure-machine-learning-architecture.md#experiments).

Конвейер состоит из наборов данных и аналитических модулей, которые соединяются вместе для создания модели. В частности, допустимый конвейер имеет следующие характеристики:

* Наборы данных могут быть подключены только к модулям.
* Модули могут быть подключены либо к наборам данных, либо к другим модулям.
* Все входные порты для модулей должны иметь подключение к потоку данных.
* Необходимо задать все обязательные параметры для каждого модуля.


Сведения о том, как приступить к работе с конструктором, см. [в разделе Учебник. Прогноз стоимости автомобилей с помощью конструктора](tutorial-designer-automobile-price-train-score.md).

## <a name="datasets"></a>Наборы данных

Набор данных машинного обучения облегчает доступ к данным и работу с ними. В конструктор включается ряд образцов наборов данных, с помощью которых можно экспериментировать с. Вы можете [зарегистрировать](./how-to-create-register-datasets.md) больше наборов данных по мере необходимости.

## <a name="module"></a>модуль

Модуль — это алгоритм, который можно выполнять с данными. В конструкторе имеется ряд модулей, отменяющихся от функций передачи данных до процессов обучения, оценки и проверки.

Модуль может иметь ряд параметров, которые можно использовать для настройки внутренних алгоритмов модуля. При выборе модуля на холсте параметры модуля отображаются на панели свойств справа от холста. Подстройка модели осуществляется изменением параметров в этой области.

![Свойства модуля](media/ui-concept-visual-interface/properties.png)

Дополнительные сведения о работе с библиотекой доступных алгоритмов машинного обучения см. в статье [Справочник по алгоритму & модуля](../algorithm-module-reference/module-reference.md) .

## <a name="compute"></a>Ресурсы для вычислений

Используйте COMPUTE Resources из рабочей области для запуска конвейера и размещения развернутых моделей в качестве конечных точек в реальном времени или конечных точек конвейера (для вывода пакета). Поддерживаются следующие целевые объекты вычислений:

| Целевой объект вычисления | Обучение | Развертывание |
| ---- |:----:|:----:|
| Вычислительная среда Машинного обучения Azure | ✓ | |
| Служба Azure Kubernetes | | ✓ |

Целевые объекты вычислений присоединяются к [рабочей области](concept-workspace.md)машинное обучение. Вы управляете целевыми объектами вычислений в рабочей области в [машинное обучение Azure Studio](https://ml.azure.com).

## <a name="publish"></a>Опубликовать

После подготовки конвейера его можно опубликовать как конечную точку RESTFUL. [Публишедпипелине](https://docs.microsoft.com/python/api/azureml-pipeline-core/azureml.pipeline.core.graph.publishedpipeline?view=azure-ml-py) можно отправить без кода Python, в котором он был создан.

Кроме того, Публишедпипелине можно использовать для повторной отправки конвейера с различными значениями Пипелинепараметер и входными данными.

## <a name="deployment"></a>Развертывание

Когда прогнозная модель будет готова, разверните ее как конечную точку конвейера или конечную точку в режиме реального времени прямо из конструктора.

Конечная точка конвейера — это Публишедпипелине, который позволяет отправить конвейер с различными значениями Пипелинепараметер и входными данными для вывода пакетов.

Конечная точка в режиме реального времени предоставляет интерфейс между приложением и моделью оценки. Внешнее приложение может взаимодействовать с моделью оценки в режиме реального времени. Вызов конечной точки в режиме реального времени возвращает результаты прогноза внешнему приложению. Чтобы вызвать конечную точку в режиме реального времени, необходимо передать ключ API, который был создан при развертывании конечной точки. Эта конечная точка основана на популярном варианте архитектуры для проектов веб-программирования.

Сведения о развертывании модели см. в разделе [учебник. Развертывание модели машинного обучения с помощью конструктора служб](tutorial-designer-automobile-price-deploy.md).

## <a name="moving-from-the-visual-interface-to-the-designer"></a>Перемещение из визуального интерфейса в конструктор

Визуальный интерфейс (Предварительная версия) обновлен и теперь Машинное обучение Azure конструкторе (Предварительная версия). Конструктор изменил архитектуру для использования серверной части на основе конвейера, которая полностью интегрируется с другими функциями Машинное обучение Azure. 

В результате этих обновлений некоторые понятия и термины визуального интерфейса были изменены или переименованы. Наиболее важные концептуальные изменения см. в таблице ниже. 

| Понятие в конструкторе | Ранее в интерфейсе визуализации |
| ---- |:----:|
| Черновик конвейера | Эксперимент |
| Конечная точка в режиме реального времени | Веб-служба |

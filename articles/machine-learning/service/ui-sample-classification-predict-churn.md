# <a name="sample-5---classification-predict-churn-appetency-and-up-selling"></a>Пример 5 - классификации. Прогнозирование оттока стремления и сопутствующих продаж 

Узнайте, как для создания эксперимента сложного машинного обучения, не написав ни строчки кода, с помощью визуального интерфейса.

Этот эксперимент предусматривает обучение, **двухклассовое увеличивающееся дерево принятия решений** классификаторами, чтобы спрогнозировать общие задачи для систем управления (CRM) для связи клиента: обработка стремления и сопутствующих продаж. Значения данных и меток разбиваются в нескольких источниках данных и зашифровываться анонимизации сведения о заказчике, тем не менее, можно по-прежнему использовать визуального интерфейса для объединения наборов данных и обучения модели с помощью скремблированных значения.

Так как мы пытаемся ответить на вопрос «Какой из них?» Это называется проблемой классификации. Тем не менее можно применить те же действия, в этом эксперименте решать любого типа задачи машинного обучения, будь то регрессии, классификацию, кластеризацию и т. д.

## <a name="data"></a>Data

Данные, которые мы используем для этого эксперимента, с KDD Cup 2009. Набор данных содержит 50 000 строк и столбцов 230 компонентов. Задача — прогнозирование оттока стремления и сопутствующих продаж для клиентов, использующих эти функции. Дополнительные сведения о данных и задачи, см. в разделе [KDD веб-сайт](https://www.kdd.org/kdd-cup/view/kdd-cup-2009).

## <a name="experiment-summary"></a>Сводка эксперимента

Пример эксперимента визуальный интерфейс показывает двоичный классификатор прогнозирования оттока, стремления и сопутствующих продаж, общие задачи для управления связь с клиентами (CRM).

Во-первых мы делаем некоторые простые обработки данных.

- Необработанного набора данных содержит много недостающих значений. Мы используем **Очистка недостающих данных** модуля, чтобы заменить отсутствующие значения с 0.

    ![Очистка набора данных](./media/ui-sample-classification-predict-churn/cleaned-dataset.png)

- Функции и соответствующие оттока, стремления, и сопутствующих продаж меток находятся в разных наборов данных. Мы используем **Добавление столбцов** модуля для добавления столбцов меток столбцов признаков. Первый столбец, **Col1**, столбец метки. Остальные столбцы, **Var1**, **Var2**, и т. д., являются столбцы признаков.

    ![Добавление столбца набора данных](./media/ui-sample-classification-predict-churn/added-column1.png)

- Мы используем **разбиение данных** модуль разбить набор данных на обучение и тестовых наборов.

    Затем мы используем двоичный классификатор увеличивающегося дерева принятия решений с параметрами по умолчанию для создания моделей прогнозирования. Мы создаем одной модели каждой задачи, то есть одна модель для прогнозирования сопутствующих продаж, стремления и обновлений.

## <a name="results"></a>Результаты

Визуализировать выходные данные **Evaluate Model** модуля, чтобы увидеть производительность модели в тестовом наборе. Для задачи сопутствующих продаж кривая ROC показывает, что модель не лучше, чем случайное модели. Площадь под кривой (AUC) — 0.857. В пороговое значение 0,5 точность составляет 0,7 в, отзыв — 0.463 и показатель F1 — 0.545.

![анализ результатов;](./media/ui-sample-classification-predict-churn/evaluate-result.png)

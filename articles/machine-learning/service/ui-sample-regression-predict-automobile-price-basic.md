## <a name="get-the-data"></a>Получение данных

В этом эксперименте мы используем **данные на автомобили (необработанные)** набор данных, который является из репозитория UCI машинного обучения. Набор данных содержит 26 столбцов, содержащих сведения об автомобилях, включая создание, модели, цены, автомобиля функции (например, число цилиндров), MPG и оценки страховых рисков. Цель этого эксперимента — для прогнозирования стоимости автомобиля.

## <a name="pre-process-the-data"></a>Предварительно обработать данные

Основной к ним относятся, очистке данных, интеграция, преобразования, сокращение и дискретизации или дискретизации. В визуальном интерфейсе, можно найти модули для выполнения этих операций и другие данные предварительной обработки задач в **преобразования данных** группе на левой панели.

Мы используем **Выбор столбцов в наборе данных** модуля, чтобы исключить нормированные потери с большим количеством отсутствующих значений. Затем мы используем **Очистка недостающих данных** Чтобы удалить строки с отсутствующими значениями. Это позволяет создать чистую набор обучающих данных.

![Предварительная обработка данных](./media/ui-sample-regression-predict-automobile-price-basic/data-processing.png)

## <a name="train-the-model"></a>Обучение модели

Различаются задач машинного обучения. Распространенные задачи машинного обучения включают классификацию, кластеризацию, регрессии и с системой рекомендаций, каждый из которых может потребоваться другой алгоритм. Выбор алгоритма часто зависит от требований варианта использования. После его выбора алгоритма, необходимо настроить его параметры для обучения более точную модель. Затем необходимо оценить все модели на основе метрик, таких как точность, читаемость и эффективность.

Поскольку целью этого эксперимента является прогнозирование цен на автомобили, а поскольку столбца "label" (цена) содержит вещественных чисел, модель регрессии будет хорошим выбором. Учитывая, что количество функций, относительно невелико (меньше 100), и эти функции не являются разреженными, вероятно, будет нелинейной границы принятия решений. Поэтому мы используем **Регрессия леса принятия решений** для этого эксперимента.

Мы используем **разбиение данных** модуль случайным образом разделить входные данные, чтобы в обучающем наборе данных содержит 70% исходных данных и проверочного набора данных содержит 30% исходных данных.

## <a name="test-evaluate-and-compare"></a>Тестирование, оценка и сравнение

 Мы разбить набор данных и использовать разные наборы данных для обучения и проверки модели, чтобы дополнительные цели оценки модели.

После обучения модели, мы используем **Score Model** и **Evaluate Model** модулей, чтобы создать результаты прогнозирования и анализ моделей.

**Оценка модели** создает прогнозы для тестового набора данных с помощью обученной модели. Чтобы проверить результат, выберите порт вывода **Score Model** , а затем выберите **визуализировать**.

![Результат оценки](./media/ui-sample-regression-predict-automobile-price-basic/score-result.png)

Затем мы передаем оценки **Evaluate Model** модуль для создания показателей оценки. Чтобы проверить результат, выберите порт вывода **Evaluate Model** , а затем выберите **визуализировать**.

![Результат оценки](./media/ui-sample-regression-predict-automobile-price-basic/evaluate-result.png)

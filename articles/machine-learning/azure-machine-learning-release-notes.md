# <a name="azure-machine-learning-release-notes"></a>Заметки о машинном обучении Azure

В этой статье узнайте о версиях машинного обучения Azure.  Для получения полного справочного контента SDK посетите основную страницу [**SDK для Python**](https://docs.microsoft.com/python/api/overview/azure/ml/intro?view=azure-ml-py) в Azure Machine Learning.

Ознакомьтесь со [списком известных проблем](resource-known-issues.md), чтобы получить сведения об известных ошибках и способах их устранения.

## <a name="2020-03-23"></a>2020-03-23

### <a name="azure-machine-learning-sdk-for-python-v120"></a>Azure Машинное обучение SDK для Python v1.2.0

+ **Критические изменения**
  + Поддержка капли для питона 2.7

+ **Исправления и усовершенствования ошибок**
  + **azure-cli-ml**
    + Добавляет "--подписка-id" к `az ml model/computetarget/service` командам в CLI
    + Добавление поддержки для передачи управляемых клиентом ключей (CMK) vault_url, key_name и key_version для развертывания ACI
  + **azureml-automl-core** 
    + Включено настраиваемый вычисление с постоянным значением для задач прогнозирования данных X и y.
    + Исправлена проблема с отображением сообщений об ошибках пользователю.    
  + **лазурьный-автомл-время выполнения**
    + Исправлена проблема с прогнозированием наборов данных, содержащей зерна только с одним рядом
    + Снижение объема памяти, требуемого задачами прогнозирования.
    + Добавлены лучшие сообщения об ошибках, если столбец времени имеет неправильный формат.
    + Включено настраиваемый вычисление с постоянным значением для задач прогнозирования данных X и y.
  + **лазурный ядер**
    + Добавьте поддержку для загрузки ServicePrincipal из переменных среды: AZUREML_SERVICE_PRINCIPAL_ID, AZUREML_SERVICE_PRINCIPAL_TENANT_ID и AZUREML_SERVICE_PRINCIPAL_PASSWORD
    + Введен новый `support_multi_line` `Dataset.Tabular.from_delimited_files`параметр:`support_multi_line=False`По умолчанию ( ), все разрывы строк, в том числе в цитируемых значениях поля, будут интерпретироваться как разрыв записи. Чтение данных таким образом быстрее и оптимизировано для параллельного выполнения на нескольких ядрах процессора. Однако это может привести к бесшумному производству большего количества записей с несогласованными значениями полей. Это должно быть `True` установлено, когда делимитированные файлы, как известно, содержат проломы котируемых строк.
    + Добавлена возможность регистрации ADLS Gen2 в ClI машинного обучения Azure
    + Переименован параметр "fine_grain_timestamp" в "метку времени" и параметр "coarse_grain_timestamp" для метод partition_timestampа with_timestamp_columns() в TabularDataset, чтобы лучше отражать использование параметров.
    + Увеличена максимальная длина названия эксперимента до 255.
  + **azureml-interpret**
    + Обновленная лазурная интерпретация для интерпретации сообщества 0,7.
  + **лазурит-sdk**
    + Изменение зависимостей с совместимой версией Tilde для поддержки исправления в предварительном выпуске и стабильных релизах.

У нас есть новый в линии способ генерации схемах чванство из любой функции Python через декораторы. Мы разработали этот код с открытым исходным кодом, и наш протокол генерации схем не связан с платформой Azure ML.

+ **Azure ML CLI, как правило, доступен (GA)**<br/> Модели теперь могут быть развернуты с одной командой CLI. Мы получили общую обратную связь с клиентами, что никто не развертывает модель ML из ноутбука Jupyter. Обновлена [**справочная документация CLI.**](https://aka.ms/azmlcli)

### <a name="azure-machine-learning-compute"></a>Вычислительная среда Машинного обучения Azure;
В этом выпуске мы объявляем о новых возможностях управляемых вычислений в [Вычислительной среде Машинного обучения Azure](how-to-set-up-training-targets.md#amlcompute). Этот целевой объект вычислений заменяет среду вычислений Azure Batch AI для Машинного обучения Azure.

Целевой объект вычислений:
+ Используется для обучения модели и пакетного выводов/скоринга
+ это одно- или многоузловая вычислительная среда;
+ выполняет управление кластерами и планирование заданий для пользователя;
+ автоматически масштабируется по умолчанию;
+ поддерживает ресурсы ЦП и GPU;
+ позволяет использовать виртуальные машины с низким приоритетом для уменьшения затрат.

Вычислительная среда Машинного обучения Azure может быть создана в среде Python с помощью портала Azure или интерфейса командной строки. Ее необходимо создать в регионе вашей рабочей области и невозможно подключить к другой рабочей области. Этот целевой объект вычислений использует контейнер Docker для работы и упаковывает зависимости для репликации этой среды на всех узлах.

> [!Warning]
> Мы рекомендуем создать новую рабочую область для использования Вычислительной среды Машинного обучения Azure. Существует небольшая вероятность того, что пользователи, пытающиеся создать Вычислительную среду Машинного обучения Azure из существующей рабочей области, увидят сообщение об ошибке. Существующая вычислительная среда в вашей рабочей области должна продолжить работу без изменений.

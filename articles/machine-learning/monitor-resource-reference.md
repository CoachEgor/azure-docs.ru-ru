# <a name="azure-machine-learning-monitoring-data-reference"></a>Ссылка на данные мониторинга машинного обучения Azure

Узнайте о данных и ресурсах, собранных Azure Monitor из рабочего пространства машинного обучения Azure. Подробнее о сборе и анализе данных мониторинга можно просмотреть данные [мониторинга« «Мониторинг машинного обучения».](monitor-azure-machine-learning.md)

### <a name="metrics"></a>Метрики

В следующих таблицах приведены метрики платформы, собранные для Azure Machine Learning, все метрики хранятся в рабочем пространстве поиска **машин nameure Machine Learning.**

**Модели**

| Метрика | Единицы | Описание |
| ----- | ----- | ----- |
| Развертывание модели не удалось | Count | Количество невыполненных развертываний моделей. |
| Развертывание модели началось | Count | Количество запущенных развертываний моделей. |
| Развертывание модели удалось | Count | Количество успешного развертывания моделей. |
| Модель ный регистр не удалось | Count | Количество регистраций моделей, которые не удалось. |
| Модельный регистр удалось | Count | Количество регистраций моделей, которые удалось. |

**Квоты**

Информация о квотах предназначена только для вычислений машинного обучения Azure.

| Метрика | Единицы | Описание |
| ----- | ----- | ----- |
| Активные ядра | Count | Количество активных вычислительных ядер. |
| Активные узлы | Count | Количество активных узлов. |
| Ядра холостого хода | Count | Количество простаивающих вычислительных ядер. |
| Узлы простоя | Count | Количество простаивающих вычислительных узлов. |
| Оставляя ядра | Count | Количество осваивающихя ядер. |
| Оставляя узлы | Count | Количество освящающихся узлов. |
| Упреждаемые ядра | Count | Количество упреждающих ядер. |
| Упреждаемые узлы | Count | Количество упреждаемых узлов. |
| Процент использования квот | Процент | Процент используемой квоты. |
| Общее число ядер | Count | Общие ядра. |
| Всего узлов | Count | Общие узлы. |
| Непригодные ядра | Count | Количество непригодных ядер. |
| Непригодные для узла | Count | Количество непригодных узлов. |

Ниже приведены измерения, которые могут быть использованы для фильтрации метрик квот:

| Измерение | Метрика (ы) доступна с | Описание |
| ---- | ---- | ---- |
| Имя кластера, | Все метрики квот | Название экземпляра вычислений. |
| Имя семьи Vm | Процент использования квот | Название семейства VM, используемое кластером. |
| Vm Приоритет | Процент использования квот | Приоритет ВМ.

**Запустить**

Информация о тренировочных забегах.

| Метрика | Единицы | Описание |
| ----- | ----- | ----- |
| Завершенные запуски | Count | Количество завершенных запусков. |
| Неудачные запуски | Count | Количество неудачных запусков. |
| Запущенные запуски | Count | Количество запущенных запусков. |

Ниже приведены измерения, которые могут быть использованы для фильтрации метрик выполнения:

| Измерение | Описание |
| ---- | ---- |
| ВычислитьТип | Тип вычислений, используемый в запуске. |
| PipelineStepType | Тип [PipelineStep,](https://docs.microsoft.com/python/api/azureml-pipeline-core/azureml.pipeline.core.pipelinestep?view=azure-ml-py) используемый в беге. |
| ОпубликованоPipelineId | Идентификатор опубликованного конвейера, используемый в запуске. |
| RunType | Тип запуска. |

Действительные значения для измерения RunType:

| Значение | Описание |
| ----- | ----- |
| Эксперимент | Нетрубопроводные запуски. |
| PipelineRun | Запуск конвейера, который является родителем StepRun. |
| СтепРун | Запуск для шага конвейера. |
| ПовторноиспользованныйСтепран | Запуск шага конвейера, который повторно использует предыдущий запуск. |

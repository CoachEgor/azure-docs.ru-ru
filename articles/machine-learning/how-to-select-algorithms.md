# <a name="how-to-select-algorithms-for-azure-machine-learning"></a>Как выбрать алгоритмы для машинного обучения Azure

Общий вопрос: «Какой алгоритм машинного обучения я должен использовать?» Выбранный алгоритм зависит в первую очередь от двух различных аспектов сценария науки о данных:

 - **Что вы хотите делать с вашими данными?** В частности, на какой бизнес-вопрос вы хотите ответить, изучая ваши прошлые данные?

 - **Каковы требования сценария науки о данных?** В частности, какова точность, время обучения, линейность, количество параметров и количество функций, которые поддерживает ваше решение?

## <a name="accuracy"></a>Точность

Точность в машинном обучении измеряет эффективность модели как пропорции истинных результатов к общему объему случаев. В конструкторе машинного обучения [модуль Evaluate Model](https://docs.microsoft.com/azure/machine-learning/algorithm-module-reference/evaluate-model?WT.mc_id=docs-article-lazzeri) вычисляет набор стандартных в отрасли показателей оценки. Этот модуль можно использовать для измерения точности обученной модели.

Получение наиболее точного ответа не всегда необходимо. Иногда достаточно приближенного ответа в зависимости от того, для чего он используется. Если это так, вы можете быть в состоянии сократить время обработки резко, придерживаясь более приблизительные методы. Приблизительные методы также естественно, как правило, чтобы избежать переоборудования.

Существует три способа использования модуля Модели оценки:

- Создание баллов по учебным данным для оценки модели
- Создание баллов по модели, но сравнить эти баллы на баллы на зарезервированном наборе тестирования
- Сравните баллы для двух разных, но связанных моделей, используя один и тот же набор данных

Полный список метрик и подходов, которые можно использовать для оценки точности моделей машинного обучения, [см.](https://docs.microsoft.com/azure/machine-learning/algorithm-module-reference/evaluate-model?WT.mc_id=docs-article-lazzeri)

## <a name="training-time"></a>Время обучения

В контролируемом обучении обучение означает использование исторических данных для построения модели машинного обучения, которая сводит к минимуму ошибки. Количество минут или часов, необходимых для обучения модели, сильно отличается для различных алгоритмов. Время обучения часто тесно связано с точностью; один обычно сопровождает другое. 

Кроме того, некоторые алгоритмы более чувствительны к количеству точек данных, чем другие. Вы можете выбрать конкретный алгоритм, потому что у вас есть ограничение по времени, особенно когда набор данных большой.

В конструкторе машинного обучения создание и использование модели машинного обучения, как правило, представляет собой трехступенчатый процесс:

1.  Настроили модель, выбрав определенный тип алгоритма, а затем определив его параметры или гиперпараметры. 

2.  Предоставьте набор данных, который помечен и имеет данные, совместимые с алгоритмом. Подключите как данные, так и модель к [модулю Train Model.](https://docs.microsoft.com/azure/machine-learning/algorithm-module-reference/train-model?WT.mc_id=docs-article-lazzeri)

3.  После завершения обучения используйте обученную модель с одним из [модулей скоринга,](https://docs.microsoft.com/azure/machine-learning/algorithm-module-reference/score-model?WT.mc_id=docs-article-lazzeri) чтобы делать прогнозы на новые данные.

## <a name="linearity"></a>Линейность

Линейность в статистике и машинном обучении означает, что существует линейная связь между переменной и постоянной в наборе данных. Например, алгоритмы линейной классификации предполагают, что классы могут быть разделены прямой линией (или ее высокомерным аналогом).

Линейность используется во многих алгоритмах машинного обучения. В дизайнере машинного обучения Azure они включают: 

- [Многоклассная логистическая регрессия](https://docs.microsoft.com/azure/machine-learning/algorithm-module-reference/multiclass-logistic-regression?WT.mc_id=docs-article-lazzeri)
- [Двухклассная логистическая регрессия](https://docs.microsoft.com/azure/machine-learning/algorithm-module-reference/two-class-logistic-regression?WT.mc_id=docs-article-lazzeri)
- [Машины вектора поддержки](https://docs.microsoft.com/azure/machine-learning/algorithm-module-reference/two-class-support-vector-machine?WT.mc_id=docs-article-lazzeri)  

Алгоритмы линейной регрессии предполагают, что тренды данных следуют прямой линии. Это предположение не плохо для некоторых проблем, но для других это снижает точность. Несмотря на свои недостатки, линейные алгоритмы популярны как первая стратегия. Обычно они алгоритмически просты и быстро осваиваются.

![Данные с нелинейным трендом](./media/how-to-select-algorithms/nonlinear-trend.png)

***Данные с нелинейным трендом:*** *Использование метода линейной регрессии приведет к повешению гораздо больших ошибок, чем это необходимо.*

## <a name="number-of-parameters"></a>Количество параметров

Параметры являются теми регуляторами, которые специалист по данным поворачивает при настройке алгоритма. Это числа, влияющие на поведение алгоритма, такие как допуск ошибок или количество итераций, или варианты между вариантами поведения алгоритма. Время обучения и точность алгоритма иногда могут быть чувствительны к получению только правильные настройки. Как правило, алгоритмы с большим количеством параметров требуют наибольшего проб и ошибок, чтобы найти хорошую комбинацию.

Кроме того, в конструкторе machine Learning есть [модуль Tune Model Hyperparameters:](https://docs.microsoft.com/azure/machine-learning/algorithm-module-reference/tune-model-hyperparameters?WT.mc_id=docs-article-lazzeri) цель этого модуля состоит в том, чтобы определить оптимальные гиперпараметры модели машинного обучения. Модуль строит и тестирует несколько моделей с помощью различных комбинаций настроек. Он сравнивает метрики по всем моделям, чтобы получить комбинации настроек. 

Хотя это отличный способ убедиться, что вы охватили пространство параметра, время, необходимое для обучения модели увеличивается в геометрической прогрессии с числом параметров. Плюсом является то, что наличие большого количества параметров обычно означает, что алгоритм имеет большую гибкость. Он часто может достичь очень хорошей точности, при условии, что вы можете найти правильное сочетание параметров параметров.

## <a name="number-of-features"></a>Количество функций

В машинном обучении функция является количественной переменной явления, которое вы пытаетесь проанализировать. Для некоторых типов данных количество функций может быть очень большим по сравнению с количеством точек данных. Это часто происходит с генетическими или текстовыми данными. 

Большое количество функций может увязнуть некоторые алгоритмы обучения, что делает время обучения неосуществимо долго. [Векторные машины поддержки](https://docs.microsoft.com/azure/machine-learning/algorithm-module-reference/two-class-support-vector-machine?WT.mc_id=docs-article-lazzeri) особенно хорошо подходят для сценариев с большим количеством функций. По этой причине они использовались во многих приложениях, от поиска информации до классификации текста и изображений. Машины вектора поддержки могут использоваться как для задач классификации, так и для регрессии.

Выбор функций относится к процессу применения статистических тестов к входным данным с учетом определенного результата. Цель состоит в том, чтобы определить, какие столбцы являются более предсказуемыми для вывода. [Модуль выбора функций на основе фильтра](https://docs.microsoft.com/azure/machine-learning/algorithm-module-reference/filter-based-feature-selection?WT.mc_id=docs-article-lazzeri) в конструкторе машинного обучения предоставляет несколько алгоритмов выбора функций на выбор. Модуль включает в себя методы корреляции, такие как корреляция Пирсона и значения чи-квадрата.

Вы также можете использовать [модуль важности функций перестановки](https://docs.microsoft.com/azure/machine-learning/algorithm-module-reference/permutation-feature-importance?WT.mc_id=docs-article-lazzeri) для вычисления набора оценок важности объектов для набора данных. Затем вы можете использовать эти баллы, чтобы помочь вам определить лучшие функции для использования в модели.

# <a name="two-class-logistic-regression-module"></a>Модуль Двухклассовой логистической регрессии

В этой статье описывается модуль визуального интерфейса (Предварительная версия) для службы машинного обучения Azure.

Этот модуль используется для создания модели логистической регрессии, которая может использоваться для прогнозирования результатов двух (и только два). 

Логистическая Регрессия является известным статистическим методом, который используется для моделирования различных проблем. Этот алгоритм является *контролируемого обучения* метода.  Таким образом необходимо указать набор данных, которая уже содержит результаты для обучения модели.  

### <a name="about-logistic-regression"></a>О логистической регрессии  

Логистическая Регрессия — известный метод в статистике, которая используется для прогнозирования вероятности результата и особенно широко применяется для задач классификации. Алгоритм прогнозирует вероятность возникновения события путем подгонки данных логистической функции.
  
В этом модуле алгоритм классификации оптимизирован для переменных дихотомических или binary. Если вам требуется для классификации нескольких результатов, используйте [Мультиклассовая логистическая Регрессия](./multiclass-logistic-regression.md) модуля.

##  <a name="how-to-configure"></a>Процесс настройки  

Для обучения этой модели, необходимо указать набор данных, содержащую столбец метки или класса. Так как этот модуль предназначен для проблемы двухклассовой, столбец метки или класс должны содержать ровно два значения. 

Например метка столбца может быть [проголосовали] с возможными значениями «Yes» или «No». Или, возможно, с возможными значениями «Высокая» или «Low» [кредитного риска,]. 
  
1.  Добавить **Двухклассовой логистической регрессии** эксперимент модуль.  
  
2.  Укажите способ модели для обучения, задав **Создание режима учителя** параметр.  
  
    -   **Единый параметр**: Если вы знаете, как вы хотите настроить модель, вы можете предоставить определенный ряд значений в качестве аргументов.  
  
3.  Для **отклонение оптимизации**, укажите пороговое значение для использования при оптимизации модели. Если улучшение между итерациями падает ниже заданного порогового значения, алгоритм считается конвергированы над решением, и останавливает обучения.  
  
4.  Для **вес регуляризации L1** и **вес регуляризации L2**, введите значение, которое используется для параметров регуляризации L1 и L2. Ненулевое значение рекомендуется для обоих.  
  
     *Регуляризация* — это метод предотвращения образования лжевзаимосвязей по penalizing моделей с коэффициент экстремальных значений. Регуляризация работает Добавление штрафа, связанного со значениями коэффициентов, к погрешности гипотезы. Таким образом более штраф к точной модели с extreme коэффициент значения будет, но менее точной модели с более приемлемыми значениями штраф будет меньше.  
  
     Унификация L1 и L2 имеют другой эффект и использует.  
  
    -   L1 могут применяться к разреженным моделям, что удобно при работе с многомерными данными.  
  
    -   Напротив регуляризации L2 является предпочтительным для данных, который не является разреженным.  
  
     Этот алгоритм поддерживает линейное сочетание значений регуляризации L1 и L2: то есть если <code>x = L1</code> и <code>y = L2</code>, затем <code>ax + by = c</code> определяет линейный диапазон условий регуляризации.  
  
    > [!NOTE]
    >  Хотите узнать больше о регуляризации L1 и L2? В следующей статье представлены обсуждение как регуляризации L1 и L2 отличаются, и как они влияют на Подгонка модели, примеры программного кода для модели нейронной сети и логистической регрессии:  [Унификация L1 и L2 для машинного обучения](https://msdn.microsoft.com/magazine/dn904675.aspx)  
    >
    > Различные сочетания линейный L1 и L2 условия целей было разработано для моделей логистической регрессии: например, [эластичной net регуляризации](https://wikipedia.org/wiki/Elastic_net_regularization). Рекомендуется ссылаться на эти сочетания, для определения линейное сочетание, действующую в модели.
      
5.  Для **объем памяти для L-BFGS**, укажите объем памяти для *L-BFGS* оптимизации.  
  
     L-BFGS — это «ограниченным использованием памяти Бройдена-Флетчера-Гольдфарба-Шанно». Это алгоритм оптимизации, который широко применяется для оценки параметра. Этот параметр указывает число сохраняемых последних позиций и градиентов для вычисления следующего шага.  
  
     Этот параметр оптимизации ограничивает объем памяти, который используется для вычисления следующего шага и направления. При указании меньше памяти, обучение проходит быстрее, но менее точным.  
  
6.  Для **начальное значение случайного числа**, введите целое значение. Определение начальное значение важно в том случае, если результаты должны быть воспроизводимой по нескольким запускам и того же эксперимента.  
  
  
8. Добавьте маркированного набора данных в эксперимент и соедините один из [обучающих модулей](module-reference.md).  
  
    -   Если задать **Создание режима учителя** для **один параметр**, использовать [Обучение модели](./train-model.md) модуля.  
  
9. Запустите эксперимент.  
  
## <a name="results"></a>Результаты

После завершения обучения:

+ Чтобы просмотреть сводку параметров модели, а также весовые показатели признака, узнали из обучения, щелкните правой кнопкой мыши выходные данные [Обучение модели](./train-model.md) и выберите **визуализировать**.   
  
+ Чтобы получить прогноз на основе новых данных, используйте обученную модель и новые данные как входной файл [Score Model](./score-model.md) модуля.

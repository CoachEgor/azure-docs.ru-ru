# <a name="edit-metadata-module"></a>Изменение метаданных модуля

В этой статье описывается модуль визуального интерфейса (Предварительная версия) для службы машинного обучения Azure.

Изменение данных модуль используется для изменения метаданных, который связан со столбцами в наборе данных. После использования модуля изменения метаданных изменит значение и тип набора данных.

Типичные изменения метаданных могут включать:
  
+ Обработка логических или числовых столбцов в виде категориальных значений.
  
+ Указание столбца, содержащего **класс** метки или содержит значения, которые необходимо категоризировать или спрогнозировать.
  
+ Пометка столбцы в качестве функции.
  
+ Изменение значений даты и времени в числовые значения или наоборот.
  
+ Переименование столбцов.
  
 Каждый раз, когда необходимо изменить определение столбца, обычно в соответствии с требованиями для подчиненного модуля с помощью редактирования метаданных. Например, некоторые модули работают только с определенных типов данных или требуют флагов для столбцов, таких как `IsFeature` или `IsCategorical`.  
  
 После выполнения нужной операции можно сбросить метаданные в исходное состояние.
  
## <a name="configure-edit-metadata"></a>Настройка Edit Metadata
  
1. В машинном обучении Azure добавьте модуль Edit Metadata эксперимента и подключите набор данных, который вы хотите обновить. Можно найти набор данных в группе **преобразования данных** в **управление** категории.
  
1. Выберите **запустить средство выбора столбцов** и выберите столбец или набор столбцов для работы с. Можно выбрать столбцы по отдельности по имени или индексу, или вы можете выбрать группу столбцов по типу.  
  
1. Выберите **тип данных** параметр, если необходимо назначить другой тип данных к выбранным столбцам. Может потребоваться изменить тип данных для определенных операций. Например если исходный набор данных содержит числа, обрабатывается как текст, необходимо изменить их в числовой тип данных перед использованием математические операции.

    + Поддерживаемые типы данных **строка**, **целое число**, **двойные**, **логическое**, и **DateTime**.

    + Если выбрать несколько столбцов, необходимо применить изменения метаданных для *все* выбранных столбцов. Например предположим, что выбрано два или три числовых столбцов. Можно изменить их все в строку в тип данных и переименовать их в рамках одной операции. Тем не менее нельзя изменить один столбец данных в строковый тип и еще один столбец из значение с плавающей запятой в целое число.
  
    + Если вы не укажете новый тип данных, метаданные столбца не изменятся.

    + Тип столбца и значения изменится после выполнения операции изменения метаданных. Исходный тип данных в любое время можно восстановить с помощью изменения метаданных, чтобы сбросить тип данных столбца.  

    > [!NOTE]
    > При изменении любого типа, создаваемого числа **DateTime** введите, оставьте **формат даты и времени** пустым. Сейчас эта возможность недоступна для указания целевой формат данных.  

1. Выберите **Категориальный** параметр, чтобы указать, что значения в выбранных столбцах должны рассматриваться как категории.

    Например, может содержать столбец, содержащий числа, 0, 1 и 2, но знать, что действительно значат «Smoker», «Non-smoker» и «Неизвестно». В этом случае, помечая столбец как категориальный убедитесь, что значения используются только для группирования данных, а не в числовых вычислений.
  
1. Используйте **поля** параметр, если вы хотите изменить способ, что машинного обучения Azure использует данные в модели.

    + **Компонент**. Используйте этот параметр, чтобы отметить столбцы как компонент в модулях, которые работают только со столбцами компонентов. По умолчанию все столбцы изначально рассматриваются как компоненты.  
  
    + **Метка** — Используйте этот параметр, чтобы пометить метку, которая также называется прогнозируемого атрибута или целевой переменной. Во многих модулях требуют наличия ровно одна метка столбца в наборе данных.

        Во многих случаях машинного обучения Azure может определить, что столбец содержит метку класса. Задавая эти метаданные, гарантирует, что столбец определяется правильно. Если этот параметр не изменяет значения данных. Меняется только способ, что некоторые алгоритмы машинного обучения, обрабатывать данные.
  
    > [!TIP]
    > У вас есть данные, которые не помещаются на следующие категории? Например набор данных может содержать значения, такие как уникальные идентификаторы, которые не полезны как переменные. Иногда такие идентификаторы могут вызвать проблемы при использовании в модели.
    >
    > К счастью машинного обучения Azure отслеживает все ваши данные, чтобы не нужно удалить такие столбцы из набора данных. Если вам нужно выполнять операции с некоторые специальные набор столбцов, просто удалите все остальные столбцы временно с помощью [Выбор столбцов в наборе данных](select-columns-in-dataset.md) модуля. Позже можно объединить столбцы обратно в набор данных с помощью [Добавление столбцов](add-columns.md) модуля.  
  
1. Используйте следующие параметры, чтобы очистить выбранные ранее параметры и восстановить значения по умолчанию метаданные.  
  
    + **Отменить Признак**: Используйте этот параметр для удаления флага компонента.  
  
         Изначально все столбцы рассматриваются как компоненты. Для модулей, которые выполняют математические операции может потребоваться использовать этот параметр, чтобы не допустить числовых столбцов, которые обрабатываются как переменные.
  
    + **Отменить метку**: Используйте этот параметр, чтобы удалить **метка** метаданные из указанного столбца.  
  
    + **Отменить показатель**: Используйте этот параметр, чтобы удалить **Оценка** метаданные из указанного столбца.  
  
         В настоящее время невозможно явно столбец помечается как оценка в машинном обучении Azure. Тем не менее некоторые операции к столбец не будет отмечена как оценку внутренним образом. Кроме того пользовательский R-модуль может выходные значения оценки.

1. Для **новые имена столбцов**, введите новое имя выбранного столбца или столбцов.  
  
    + Имена столбцов можно использовать только символы, которые поддерживаются UTF-8 кодирования. Не допускаются пустые строки, значения NULL или имена, состоящие только из пробелов.  
  
    + Чтобы переименовать несколько столбцов, введите имена как список с разделителями запятыми в порядке индексов столбцов.  
  
    + Необходимо переименовать всех выделенных столбцов. Не пропустите или пропустить столбцы.  
  
1. Запустите эксперимент.

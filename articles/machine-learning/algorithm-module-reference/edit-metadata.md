# <a name="edit-metadata-module"></a>Модуль «Эдит Метадата»

В этой статье описывается модуль, включенный в конструктор машинного обучения Azure (предварительный просмотр).

Используйте модуль Edit Metadata для изменения метаданных, связанных с столбцов в наборе данных. Значение и тип данных набора данных изменятся после использования модуля Edit Metadata.

Типичные изменения метаданных могут включать следующее.
  
+ Лечение булеаевых или численных столбцов как категориальных значений.
  
+ Указывая, какой столбец содержит метку **класса** или содержит значения, которые вы хотите классифицировать или предсказать.
  
+ Маркировка столбцов как объектов.
  
+ Изменение значений даты/времени на числовые значения или наоборот.
  
+ Переименование столбцов.
  
 Используйте Edit Metadata в любое время, когда вам нужно изменить определение столбца, как правило, для удовлетворения требований к модулю ниже по течению. Например, некоторые модули работают только с определенными типами `IsFeature` `IsCategorical`данных или требуют флагов на столбцах, таких как или .  
  
 После выполнения требуемой операции можно сбросить метаданные в исходное состояние.
  
## <a name="configure-edit-metadata"></a>Настройка Edit Metadata
  
1. В конструкторе машинного обучения Azure добавьте модуль Edit Metadata в конвейер и подключите набор данных, который необходимо обновить. Вы можете найти модуль в категории **преобразования данных.**
  
1. Нажмите **колонку Edit** в правой панели модуля и выберите столбец или набор столбцов для работы. Вы можете выбрать столбцы по отдельности по имени или индексу, или вы можете выбрать группу столбцов по типу.  
  
1. Выберите опцию **типа данных,** если необходимо назначить выбранным столбикам другой тип данных. Возможно, потребуется изменить тип данных для определенных операций. Например, если в наборе исходных данных есть номера, обрабатываемые как текст, необходимо изменить их на тип числовых данных перед использованием математических операций.

    + Поддерживаемые типы **данных: String,** **Integer,** **Double,** **Boolean**и **DateTime**.

    + При выборе нескольких столбцов необходимо применить изменения метаданных ко *всем* выбранным столбцам. Например, предположим, что вы выбираете два или три числовые столбцы. Вы можете изменить их все в тип строки данных и переименовать их в одной операции. Однако нельзя изменять один столбец на тип данных строки, а другой — с поплавка на ряд.
  
    + Если не указано новый тип данных, метаданные столбца остаются неизменными.

    + Тип и значения столбцов изменятся после выполнения операции Edit Metadata. Восстановить исходный тип данных можно в любое время, используя Edit Metadata для сбросить тип данных столбца.  

    > [!NOTE]
    > Если вы измените любой тип номера на тип **DateTime,** оставьте поле **формата DateTime** пустым. В настоящее время невозможно указать формат целевых данных.  

1. Выберите **категориальный** вариант, чтобы указать, что значения в выбранных столбцах следует рассматривать как категории.

    Например, у вас может быть столбец, содержащий числа 0, 1 и 2, но знайте, что цифры на самом деле означают "Курение", "Некурящий" и "Неизвестный". В этом случае, пометив столбец как категоричный, вы гарантируете, что значения используются только для группирования данных, а не в числовых расчетах.
  
1. Используйте опцию **Fields,** если требуется изменить способ использования данных в модели.

    + **Функция**: Используйте эту опцию для пометки столбца в качестве функции в модулях, которые работают только на столбцах объектов. По умолчанию все столбцы изначально считаются функциями.  
  
    + **Метка**: Используйте эту опцию для обозначения метки, которая также известна как предсказуемый атрибут или переменная цели. Многие модули требуют, чтобы в наборе данных присутствовала ровно одна столбица метки.

        Во многих случаях Azure Machine Learning может сделать вывод, что столбец содержит метку класса. Установив эти метаданные, можно убедиться, что столбец идентифицирован правильно. Установка этой опции не изменяет значения данных. Он изменяет только способ обработки данных некоторыми алгоритмами машинного обучения.
  
    > [!TIP]
    > Есть ли у вас данные, которые не вписываются в эти категории? Например, набор данных может содержать такие значения, как уникальные идентификаторы, которые не являются полезными в качестве переменных. Иногда такие иди могут вызвать проблемы при использовании в модели.
    >
    > К счастью, Azure Machine Learning сохраняет все данные, так что вам не придется удалять такие столбцы из набора данных. При необходимости выполнения операций на специальном наборе столбцов просто временно удалите все остальные столбцы, используя выбранные столбцы в модуле [Dataset.](select-columns-in-dataset.md) Позже можно объединить столбцы обратно в набор данных с помощью модуля [Add Columns.](add-columns.md)  
  
1. Используйте следующие параметры, чтобы очистить предыдущие выделения и восстановить метаданные к значениям по умолчанию.  
  
    + **Четкая функция**: Используйте эту опцию для удаления флага функции.  
  
         Все столбцы изначально рассматриваются как объекты. Для модулей, выполняющие математические операции, может потребоваться использовать эту опцию, чтобы не допустить, чтобы числовые столбцы рассматривались как переменные.
  
    + **Четкая метка**: Используйте эту опцию для удаления метаданных **метки** из указанного столбца.  
  
    + **Четкая оценка**: Используйте эту опцию, чтобы удалить метаданные **оценки** из указанного столбца.  
  
         В настоящее время нельзя пометить столбец как оценку в Azure Machine Learning. Однако некоторые операции приводят к тому, что столбец помечается как оценка внутренне. Кроме того, пользовательский модуль R может вывести значения баллов.

1. Для **новых имен столбцов**введите новое имя выбранного столбца или столбцов.  
  
    + Названия столбцов могут использовать только символы, поддерживаемые кодированием UTF-8. Пустые строки, нулевки или имена, полностью состоящие из пробелов, не допускаются.  
  
    + Чтобы переименовать несколько столбцов, введите имена в виде списка, разделенного запятой, в порядке индексов столбцов.  
  
    + Все выбранные столбцы должны быть переименованы. Нельзя опускать или пропускать столбцы.  
  
1. Отправить конвейер.

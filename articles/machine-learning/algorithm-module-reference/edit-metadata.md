# <a name="edit-metadata-module"></a>Изменение метаданных модуля

В этой статье описывается модуль визуального интерфейса (Предварительная версия) для службы машинного обучения Azure.

Этот модуль используется, чтобы изменить метаданные, связанные со столбцами в наборе данных. Значение и тип набора данных изменяется после использования **изменить метаданные** модуля. 
 
Типичные изменения метаданных могут включать:
  
+ Рассматривая логических или числовых столбцов как категориальных значений  
  
+ Указание столбца, содержащего *класс* метку или значения, которые необходимо категоризировать или спрогнозировать  
  
+ Пометка столбцы компонентов
  
+ Изменение даты и времени значения в числовое значение (или наоборот)  
  
+ Переименование столбцов
  
 Используйте [изменить метаданные каждый раз, когда необходимо изменить определение столбца, обычно в соответствии с требованиями для подчиненного модуля. Например, некоторые модули можно работать только с конкретными типами данных или требуют флагов для столбцов, таких как `IsFeature` или `IsCategorical`.  
  
 После выполнения нужной операции, можно сбросить метаданные в исходное состояние. 
  
## <a name="configure-edit-metadata"></a>Настройка Edit Metadata
  
1.  В машинном обучении Azure, добавьте [Edit Metadata](./edit-metadata.md) вашего эксперимента и подключите набор данных, необходимо обновить. Его можно найти в разделе **преобразования данных**в **управление** категории.
  
2.  Нажмите кнопку **запустить средство выбора столбцов** и выберите столбец или набор столбцов для работы с. Можно выбрать столбцы по отдельности, по имени или индексу, или вы можете выбрать группу столбцов по типу.  
  
3.  Выберите **тип данных** параметр, если необходимо назначить другой тип данных к выбранным столбцам. Изменение данных типа могут понадобиться для определенных операций: например, если исходный набор данных содержит числа, обрабатывается как текст, необходимо изменить их в числовой тип данных перед использованием математические операции. 

    + Поддерживаемые типы данных: `String`, `Integer`, `Double`, `Boolean`, `DateTime`. 

    + Если выбрано несколько столбцов, необходимо применить изменения метаданных для **все** выбранных столбцов. Например предположим, что можно выбрать числовые столбцы 2 – 3. Можно изменить их все в строковый тип данных и переименовать их в рамках одной операции. Тем не менее нельзя изменить один столбец данных в строковый тип и еще один столбец из значение с плавающей запятой в целое число.
  
    + Если вы не укажете новый тип данных, метаданные столбца не меняется. 
    
    + Тип столбца и значения изменяются после того, как выполнять [изменить метаданные](./edit-metadata.md) операции. Исходный тип данных в любое время можно восстановить с помощью [изменить метаданные](./edit-metadata.md) сбросить тип данных столбца.  

    > [!NOTE]
    > При изменении любого типа, создаваемого числа **DateTime** введите, оставьте **формат даты и времени** пустым. В настоящее время не можно указать целевой формат данных.  

      
4.  Выберите **Категориальный** параметр, чтобы указать, что значения в выбранных столбцах должны рассматриваться как категории. 

    Например может содержать столбец, содержащий числа 0; 1 и 2, но знаете, что действительно значат «Smoker», «Non-smoker» и «Неизвестно». В этом случае, помечая столбец как категориальный можно убедитесь, что значения не используются в числовых вычислений, только для группирования данных. 
  
5.  Используйте **поля** параметр, если вы хотите изменить способ, что машинного обучения Azure использует данные в модели.

    + **Компонент**. Используйте этот параметр, чтобы отметить столбцы как компонент, для использования с модулями, которые работают только со столбцами компонентов. По умолчанию все столбцы изначально рассматриваются как компоненты.  
  
    + **Метка** — Используйте этот параметр, чтобы пометить метки (также известный как прогнозируемый атрибут, или целевой переменной). Во многих модулях требуется по крайней мере один (и только один) столбца "Метка" присутствовать в наборе данных. 
    
        Во многих случаях машинного обучения Azure может определить, что столбец содержит метку класса, но, установив эти метаданные можно обеспечить, что столбец определяется правильно. Если этот параметр не изменяет значения данных, является единственным способом, что некоторые алгоритмы машинного обучения, обрабатывать данные.
  

  
    > [!TIP]
    >  У вас есть данные, которые не помещаются на следующие категории?  Например набор данных может содержать значения, такие как уникальные идентификаторы, которые не используются как переменные. Иногда идентификаторов может вызвать проблемы при использовании в модели. 
    >   
    >  К счастью «на самом деле» машинного обучения Azure защищает ваши данные, поэтому не нужно удалить такие столбцы из набора данных. Если вам нужно выполнять операции с некоторые специальные набор столбцов, просто удалите все остальные столбцы временно с помощью [Выбор столбцов в наборе данных](./select-columns-in-dataset.md) модуля. Позже можно объединить столбцы обратно в набор данных с помощью [Добавление столбцов](./add-columns.md) модуля.  
  
6. Используйте следующие параметры, чтобы очистить выбранные ранее параметры и восстановить значения по умолчанию метаданные.  
  
    + **Отменить Признак**: Используйте этот параметр для удаления флага компонента.  
  
         Так как все столбцы изначально рассматриваются как компоненты, модули, которые выполняют математические операции, может потребоваться использовать этот параметр для предотвращения числовых столбцов, которые обрабатываются как переменные.
  
    + **Отменить метку**: Используйте этот параметр, чтобы удалить **метка** метаданные из указанного столбца.  
  
    + **Отменить показатель**: Используйте этот параметр, чтобы удалить **Оценка** метаданные из указанного столбца.  
  
         В настоящее время возможность явно помечать столбец в качестве оценки недоступна в машинном обучении Azure. Тем не менее некоторые операции к столбец не будет отмечена как оценку внутренним образом. Кроме того пользовательский R-модуль может выходные значения оценки.
  
  
7.  Для **новые имена столбцов**, введите новое имя выбранного столбца или столбцов.  
  
    + Имена столбцов можно использовать только символы, которые поддерживаются UTF-8 кодирования. Не допускаются пустые строки, значения NULL или имена, состоящие только из пробелов.  
  
    + Чтобы переименовать несколько столбцов, введите имена в виде списка с разделителями запятыми в порядке индексов столбцов.  
  
    + Необходимо переименовать всех выделенных столбцов. Не пропустите или пропустить столбцы.  
  
  
8.  Запустите эксперимент.

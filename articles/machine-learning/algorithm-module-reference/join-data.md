# <a name="join-data"></a>Объединение данных

В этой статье описывается использование **объединить данные** модуля в визуальном интерфейсе службы машинного обучения Azure для слияния двух наборов данных, используя операцию соединения стиле базы данных.  

## <a name="how-to-configure-join-data"></a>Как настроить соединение данных

Для соединения двух наборов данных, они должны быть связаны с ключевой столбец. Также поддерживаются составные ключи, с помощью нескольких столбцов. 

1. Добавить наборы данных, которые вы хотите объединить, а затем перетащите **соединения данных** модуля в эксперименте. 

    Можно найти модуль в **преобразования данных** категорию в разделе **манипуляции**.

1. Подключите наборы данных для **соединения данных** модуля. 
 
1. Выберите **запустить средство выбора столбцов** выбрать ключевые столбцы. Не забудьте выбрать столбцы для левого и правого входных данных.

    Для одного ключа:

    Выберите один ключевой столбец для обоих входных параметров.
    
    Для составного ключа:

    Выберите все ключевые столбцы из входных данных слева и входные данные справа, в том же порядке. **Объединить данные** модуль соединит таблицы, если все ключевые столбцы совпадают. Установите флажок **допускает дубликатов и сохранить порядок столбцов в выделенном фрагменте** Если порядок столбцов не совпадает с исходной таблицей. 

    ![средство выбора столбцов](media/module/join-data-column-selector.png)


1. Выберите **учитывать регистр** параметр, если вы хотите сохранить регистр символов текста столбца соединения. 
   
1. Используйте **тип соединения** раскрывающийся список, чтобы указать способ объединения наборов данных.  
  
    * **Внутреннее соединение**: *Внутреннее соединение* является наиболее распространенной операцией соединения. Она возвращает соединенные строки только в том случае, если значения ключевых столбцов совпадают.  
  
    * **Левое внешнее соединение**: Объект *левое внешнее соединение* возвращает объединенные строки для всех строк из левой таблицы. Когда строки в левой таблице не имеет совпадающей строки в правой таблице, возвращаемая строка содержит отсутствующие значения для всех столбцов, полученных из правой таблицы. Можно также указать значение замены недостающих значений.  
  
    * **Полное внешнее соединение**: Объект *полное внешнее соединение* возвращает все строки из левой таблицы (**table1**) и из правой таблицы (**table2**).  
  
         Для каждой строки в любой из таблиц, не имеющие соответствующих строк в другой то результат включает строки, содержащей отсутствующих значений.  
  
    * **Оставить полусоединение**: Объект *левое полусоединение* возвращает только значения из левой таблицы, если значения ключевых столбцов совпадают.  

1. Для параметра **сохранить правые ключевые столбцы в объединенной таблице**:

    * Выберите этот параметр, чтобы просмотреть ключи из обеих входных таблиц.
    * Снимите флажок, чтобы возвращать только ключевые столбцы из входных данных слева.

1. Выполнение эксперимента, или выберите модуль объединения данных и выбора **запустить выбранный** выполнить соединение.

1. Чтобы просмотреть результаты, щелкните правой кнопкой мыши **соединения данных** > **результирующий набор данных** > **визуализировать**.

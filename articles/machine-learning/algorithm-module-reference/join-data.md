# <a name="join-data"></a>Объединение данных

В этой статье описывается, как использовать модуль **Join Data** в конструкторе машинного обучения Azure (предварительный просмотр) для объединения двух наборов данных с помощью операции соединения в стиле базы данных.  

## <a name="how-to-configure-join-data"></a>Как настроить данные соединения

Для выполнения соединения на двух наборах данных они должны быть связаны с ключевым столбцом. Также поддерживаются композитные клавиши с использованием нескольких столбцов. 

1. Добавьте наборы данных, которые вы хотите объединить, а затем перетащите модуль **Join Data** в конвейер. 

    Вы можете найти модуль в категории **преобразования данных,** под **манипуляцией.**

1. Подключите наборы данных к модулю **«Соединить данные».** 
 
1. Выберите **селектор колонки Запуска,** чтобы выбрать ключевой столбец(ы). Не забудьте выбрать столбцы для левого и правого входов.

    Для одного ключа:

    Выберите один столбец ключа для обоих входов.
    
    Для композитного ключа:

    Выберите все ключевые столбцы из левого ввода и правый вход в том же порядке. Модуль **Join Data** присоединится к таблицам, когда все ключевые столбцы совпадают. Проверьте **опцию Разрешить дубликаты и сохранить столбец порядка в выборе,** если столбец порядок не совпадает с исходной таблицей. 

    ![средство выбора столбцов](media/module/join-data-column-selector.png)


1. Выберите опцию **«Матч-кейс»,** если вы хотите сохранить чувствительность к делу на присоединении к текстовой колонке. 
   
1. Используйте список отбрасываемых **типов Join,** чтобы указать, как должны быть объединены наборы данных.  
  
    * **Внутреннее соединение**: *внутреннее соединение* является наиболее распространенной операцией соединения. Она возвращает соединенные строки только при совпадении значений ключевых столбцов.  
  
    * **Левая внешняя присоединение**: *Левый внешний соединить* возвращается присоединился строк для всех строк из левого стола. Если строка в левой таблице не имеет соответствующих строк в правой таблице, возвращенная строка содержит недостающие значения для всех столбцов, которые приходят из правой таблицы. Можно также указать значение замены недостающих значений.  
  
    * **Полное внешнее соединение**: *Полное внешнее соединение* возвращает все строки из левой таблицы **(таблица1)** и из правого стола **(таблица2**).  
  
         Для каждой строки в любой таблице, не имеющая соответствующих строк в другой, результат включает строку, содержащую недостающие значения.  
  
    * **Левая полусоединение**: *Левый полусоединение* возвращает только значения из левой таблицы, когда значения ключевых столбцов совпадают.  

1. Для опции **Держите правые ключевые столбцы в объединенной таблице:**

    * Выберите эту опцию для просмотра ключей из обеих таблиц ввода.
    * Отберите только для возврата ключевых столбцов с левого ввода.

1. Отправить конвейер.

1. Чтобы просмотреть результаты, нажмите правой кнопкой **мыши Присоединение данных** и выберите **Визуализация**.

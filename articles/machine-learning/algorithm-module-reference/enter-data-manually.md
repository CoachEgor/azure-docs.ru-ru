# <a name="enter-data-manually-module"></a>Введите данные вручную модуля

В этой статье описывается модуль визуального интерфейса (Предварительная версия) для службы машинного обучения Azure.

Этот модуль используется для создания небольшой набор данных, введя значения. Набор данных может содержать несколько столбцов.
  
Этот модуль может пригодиться в следующих сценариях:  
  
- Создание небольшой набор значений для тестирования  
  
- Создание краткого списка меток
  
- Ввод списка имен столбцов для вставки в наборе данных

## <a name="enter-data-manually"></a>Ввод данных вручную 
  
1.  Добавить [ввести данные вручную](./enter-data-manually.md) эксперимент модуль. Можно найти этот модуль в **ввод и вывод данных** категории в машинном обучении Azure. 
  
2.  Для **DataFormat**, выберите один из следующих вариантов. Эти параметры определяют, каким образом следует анализировать данные, предоставляемые. Требования для каждого формата сильно отличаются, поэтому не забудьте прочитать в соответствующих разделах.  
  
    -   **ARFF**. Формат файла связи атрибутов, используемых Weka.   
  
    -   **CSV-ФАЙЛ**. Формат значений с разделителями запятыми. Дополнительные сведения см. в разделе [преобразовать в CSV-ФАЙЛ](./convert-to-csv.md).  
  
    -   **SVMLight**. Формат, используемый Vowpal Wabbit и другие платформы машинного обучения.  
  
    -   **TSV**. Формат значения, разделенные табуляцией.

     Если вы выберите формат и не предоставляют данные, которые соответствуют спецификации формата, возникает ошибка времени выполнения.
  
3.  Щелкните внутри **данных** текстовое поле, чтобы начать ввод данных. Следующие форматы требуют особого внимания:  
  
    - **CSV-ФАЙЛ**:  Чтобы создать несколько столбцов, вставьте в текст с разделителями запятыми, или введите несколько столбцов, используйте запятые между полями.
  
        При выборе **HasHeader** вариант, можно использовать первую строку значений заголовка столбца.  
  
        Если выбран этот параметр, имена столбцов, Col1, Col2 и так далее, будут использоваться. Можно добавить или изменить столбцы имена позже с помощью [изменить метаданные](./edit-metadata.md).  
  
    - **TSV**: Чтобы создать несколько столбцов, вставьте шрифтом, разделенные табуляцией, или введите несколько столбцов, используя вкладки между полями.  
  
        При выборе **HasHeader** вариант, можно использовать первую строку значений заголовка столбца.  
  
        Если выбран этот параметр, имена столбцов, Col1, Col2 и так далее, будут использоваться. Можно добавить или изменить столбцы имена позже с помощью [изменить метаданные](./edit-metadata.md).  
  
    -   **ARFF**:  Вставьте в существующий файл формата ARFF. При вводе значения напрямую, не забудьте добавить необязательный заголовок и обязательный атрибут поля в начале данных. 
    
        Например простой список удалось добавить следующие строки заголовка и атрибутов. Заголовок столбца будет `SampleText`.
    
        ```text
        % Title: SampleText.ARFF  
        % Source: Enter Data module  
        @ATTRIBUTE SampleText STRING  
        @DATA  
        \<type first data row here>  
        ```

    -   **SVMLight**: Введите или вставьте значения в формате SVMLight.  
  
        Например в следующем примере представлена первые несколько строк набора данных о пожертвованиях крови, в формате SVMight:  
  
        ```text  
        # features are [Recency], [Frequency], [Monetary], [Time]  
        1 1:2 2:50 3:12500 4:98   
        1 1:0 2:13 3:3250 4:28   
        ```  
  
        При запуске [ввести данные вручную](./enter-data-manually.md) модуля, эти строки преобразуются в набор данных, столбцы и индексировать значения следующим образом:  
  
        |Col1|Col2|Col3|Col4|Метки|  
        |-|-|-|-|-|  
        |0.00016|0.004|0.999961|0.00784|1|  
        |0|0.004|0.999955|0.008615|1|  
  
4.  Нажмите клавишу ВВОД после каждой строки, чтобы начать новую строку.  
  
     **Убедитесь, что клавишу ВВОД после последней строке.** 
     
     При нажатии клавиши ВВОД несколько раз, чтобы добавить несколько пустые строки в конце, удаления обрезанной окончательный пустой строки, но другие пустые строки рассматриваются как отсутствующие значения.  
  
     При создании строки с отсутствующими значениями, можно всегда отфильтровать их позже.  
  
5.  Щелкните модуль правой кнопкой мыши и выберите **запустить выбранные** для анализа данных и загрузить их в рабочую область как набор данных.  
  
     Чтобы просмотреть набор данных, щелкните порт вывода и выберите **визуализировать**.

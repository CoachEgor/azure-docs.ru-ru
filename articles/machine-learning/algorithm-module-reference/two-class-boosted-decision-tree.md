# <a name="two-class-boosted-decision-tree-module"></a>Модуль two-Class Boosted Decision Tree

В этой статье описывается модуль визуального интерфейса (Предварительная версия) для службы машинного обучения Azure.

Этот модуль используется для создания модели машинного обучения, которая основана на алгоритме дерева принятия решений. 

Увеличивающееся дерево принятия решений является это метод, в котором дереве второй исправляет ошибки первый дерева совокупного обучения, в дереве третий исправляет ошибки деревьев первого и второго и т. д.  Прогнозы основываются на весь набор деревьев вместе, делает прогноз.
  
Как правило при правильной настройке повышенные деревья принятия решений — это самый простой методы, позволяющие добиться максимальной производительности на самых разнообразных задач машинного обучения. Тем не менее они также один из учеников более интенсивно использующих память, и текущая реализация содержит все, что в памяти. Таким образом модель дерева принятия решений не может иметь возможность обрабатывать большие наборы данных, который может обработать некоторые из линейных учеников.

## <a name="how-to-configure"></a>Процесс настройки

Этот модуль создает необученную модель классификации. Поскольку классификации — это метод контролируемого обучения, для обучения модели, достаточно *с тегами набора данных* , содержащий столбец меток со значением для всех строк.

Вы можете обучить модель с помощью такого рода [Обучение модели](././train-model.md). 

1.  В машинном обучении Azure, добавьте **повышенного дерева принятия решений** эксперимент модуль.
  
2.  Укажите способ модели для обучения, задав **Создание режима учителя** параметр.
  
    + **Единый параметр**: Если вы знаете, как вы хотите настроить модель, вы можете предоставить определенный ряд значений в качестве аргументов.
  
  
3.  Для **максимальное количество листьев каждого дерева**, задать максимальное число конечных узлов (листьев), которые могут быть созданы в любом дереве.
  
     Увеличение этого значения, потенциально увеличивают размер дерева и повысите точность, риском лжевзаимосвязей и больше времени обучения.
  
4.  Для **минимальное число выборок для конечного узла**, указать количество вариантов, необходимых для создания любого конечного узла (листа) в дереве.  
  
     Увеличение этого значения, увеличьте пороговое значение для создания новых правил. Например значением по умолчанию 1, даже один случай может привести к новое правило, чтобы создать. Если вы увеличите значение до 5, обучающие данные должны будут содержать по крайней мере пять вариантов, которые удовлетворяют тем же условиям.
  
5.  Для **скорости обучения**, введите число от 0 до 1, который определяет размер шага во время обучения.  
  
     Скорость обучения определяет, как быстро или медленно ученик сходится оптимальное решение. Если размер шага, слишком велико, может перенапряжение оптимальное решение. Если размер шага, слишком мал, обучение занимает больше времени, на которой наилучшим решением.
  
6.  Для **количество созданных деревьев**, указывают общее число деревьев принятия решений для создания в ансамбле. Создавая больше деревьев принятия решений, вы можете потенциально получить большее покрытие, но увеличивает время обучения.
  
     Это значение также определяет число деревьев, отображаемое, когда визуализация обученной модели. Если требуется просмотреть или распечатать одно дерево, значение равно 1. Тем не менее, при этом только одно дерево создается (дерево с начальным набором параметров) и выполняются без дальнейших итераций.
  
7.  Для **начальное значение случайного числа**, при необходимости введите неотрицательное целое число для использования в качестве случайного начального значения. Указав начальное значение, вы обеспечите повторяемость между запусками с теми же параметрами и данными.  
  
     Случайное начальное значение имеет значение по умолчанию 0, это означает, что начальное значение получается из системных часов.  Последующие запуски с помощью случайное начальное значение может иметь различные результаты.
  

9. Обучение модели.
  
    + Если задать **Создание режима учителя** для **один параметр**, подключите набор данных, заключенные в теги и [Обучение модели](./train-model.md) модуля.  
  
   
## <a name="results"></a>Результаты

После обучения модели, щелкните правой кнопкой мыши выходные данные [Обучение модели](./train-model.md) для просмотра результатов:

+ Для просмотра дерева, который был создан на каждой итерации, выберите **визуализировать**. 
+ Чтобы детализировать разбиений и отображения списка правил для каждого узла, щелкните каждое дерево.

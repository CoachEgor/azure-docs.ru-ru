# <a name="two-class-boosted-decision-tree-module"></a>Двухклассный модуль Boosted Decision Tree

В этой статье описывается модуль в дизайнере машинного обучения Azure (предварительный просмотр).

Используйте этот модуль для создания модели машинного обучения, основанной на алгоритме ускоренных деревьев решений. 

Увеличенное дерево решений – это метод обучения ансамблю, в котором второе дерево исправляет ошибки первого дерева, третье дерево исправляет ошибки первого и второго деревьев и так далее.  Прогнозы основаны на весь ансамбль деревьев вместе, что делает прогноз.
  
Как правило, при правильной настройке, увеличенные деревья решений являются самыми простыми методами, с помощью которых можно получить максимальную производительность на самых разнообразных задачах машинного обучения. Тем не менее, они также являются одним из наиболее памяти интенсивных учащихся, и текущая реализация держит все в памяти. Таким образом, увеличенная модель дерева решений может оказаться не в состоянии обрабатывать большие наборы данных, с которыми могут обрабатывать сяочливые данные.

## <a name="how-to-configure"></a>Порядок настройки

Этот модуль создает неподготовленную модель классификации. Поскольку классификация является контролируемым методом обучения для обучения модели, требуется *помеченный набор данных,* который включает столбец метки со значением для всех строк.

Вы можете обучить этот тип модели с помощью [модели train.](././train-model.md) 

1.  В машинном обучении Azure добавьте модуль **«Повышение дерева решений»** в конвейер.
  
2.  Укажите, как нужно обучить модель, установив опцию **«Создать тренер».**
  
    + **Единый параметр**: Если вы знаете, как настроить модель, вы можете предоставить определенный набор значений в качестве аргументов.
  
    + **Диапазон параметров**: Если вы не уверены в лучших параметрах, вы можете найти оптимальные параметры с помощью модуля [Tune Model Hyperparameters.](tune-model-hyperparameters.md) Вы предоставляете некоторый диапазон значений, и тренер итерирует несколько комбинаций параметров, чтобы определить комбинацию значений, которая дает наилучший результат.
  
3.  Для **максимального количества листьев на дерево**укажите максимальное количество узлов терминала (листьев), которые могут быть созданы в любом дереве.
  
     Увеличив это значение, вы потенциально увеличите размер дерева и повысите точность, но это может привести к возникновению лжевзаимосвязей и более длительному времени обучения.
  
4.  Для **минимального количества образцов на узла листа**укажите количество случаев, необходимых для создания любого терминального узла (листа) в дереве.  
  
     Увеличив это значение, вы увеличиваете пороговое значение для создания новых правил. Например, при использовании значения по умолчанию 1, даже один случай может привести к созданию нового правила. Если увеличить его до 5, для создания правила обучающие данные должны будут содержать не менее пяти вариантов.
  
5.  Для **обучения,** введите число между 0 и 1, который определяет размер шага во время обучения.  
  
     Скорость обучения определяет, насколько быстро или медленно учащийся сходится на оптимальном решении. Если размер шага слишком велик, возможно, вы перечеркните оптимальное решение. Если размер шага слишком мал, обучение занимает больше времени, чтобы сблизиться с лучшим решением.
  
6.  Для **количества деревьев, построенных,** указать общее количество деревьев решения для создания в ансамбле. Создавая больше деревьев принятия решений, вы можете потенциально получить большее покрытие, но время на обучение при этом увеличится.
  
     Это значение также контролирует количество деревьев, отображаемых при визуализации обученной модели. если вы хотите увидеть или распечатать одно дерево, установите значение до 1. Однако при этом производится только одно дерево (дерево с исходным набором параметров) и никаких дальнейших итераций не выполняется.
  
7.  Для **случайных семян числа,** дополнительно введите неотрицательный целый ряд для использования в качестве случайного значения семян. Определение семени обеспечивает воспроизводимость на трассах с одинаковыми данными и параметрами.  
  
     Случайное семя устанавливается по умолчанию до 0, что означает начальное значение семян, полученное из системных часов.  Последовательные запуски с использованием случайного семени могут иметь разные результаты.
  

9. Обучение модели.
  
    + При установке **режима "Создание тренера"** на **единый параметр**подключите помеченный набор данных и модуль [модели train.](./train-model.md)  
   
## <a name="results"></a>Результаты

После завершения обучения:

+ Чтобы сохранить моментальный снимок обученной модели, выберите вкладку **Выводы** в правой панели **модуля модели Train.** Выберите значок **набора данных Регистра,** чтобы сохранить модель в качестве многоразового модуля.

+ Чтобы использовать модель для скоринга, добавьте модуль **Модели оценки** в конвейер.

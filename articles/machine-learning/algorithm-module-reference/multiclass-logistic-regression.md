# <a name="multiclass-logistic-regression-module"></a>Модуль мультиклассовой логистической регрессии

В этой статье описывается модуль визуального интерфейса (Предварительная версия) для службы машинного обучения Azure.

Этот модуль используется для создания модели логистической регрессии, которая может использоваться для прогнозирования нескольких значений.

Классификация с помощью логистической регрессии — это метод контролируемого обучения, почему и требует набора данных с меткой. Обучение модели, предоставляя модель и набор данных с метками в качестве входных данных модуля например [Обучение модели](./train-model.md). Затем обученную модель можно использовать для прогнозирования новых примеров входных данных.

Машинное обучение Azure также предоставляет [Двухклассовой логистической регрессии](./two-class-logistic-regression.md) модуль, который подходит для классификации, двоичный или дихотомических переменных.

## <a name="about-multiclass-logistic-regression"></a>О мультиклассовой логистической регрессии

Логистическая Регрессия — известный метод в статистике, которая используется для прогнозирования вероятности результата и широко применяется для задач классификации. Алгоритм прогнозирует вероятность возникновения события путем подгонки данных логистической функции. 

В мультиклассовой логистической регрессии классификатор может использоваться для прогнозирования нескольких результатов.

## <a name="configure-a-multiclass-logistic-regression"></a>Настройка мультиклассовой логистической регрессии

1. Добавить **Мультиклассовая логистическая Регрессия** эксперимента.

2. Укажите способ модели для обучения, задав **Создание режима учителя** параметр.

    + **Единый параметр**: Используйте этот параметр, если вы знаете, как вы хотите настроить модель и предоставляют ряд значения в качестве аргументов.

    + **Диапазон параметров**: Используйте этот параметр, если вы не знаете оптимальных параметров и хотите используете очистку параметров.

3. **Отклонение оптимизации**, укажите пороговое значение для конвергенции оптимизатора. Если улучшение между итерациями меньше, чем пороговое значение, алгоритм прекращает и возвращает текущую модель.

4. **Вес регуляризации L1**, **вес регуляризации L2**: Введите значение, которое используется для параметров регуляризации L1 и L2. Ненулевое значение рекомендуется для обоих.

    Регуляризация — это метод предотвращения образования лжевзаимосвязей путем применения штрафных коэффициентов к моделям со значениями extreme коэффициент. Регуляризация работает Добавление штрафа, связанного со значениями коэффициентов, к погрешности гипотезы. К точной модели с extreme коэффициент значения будет более штраф. при этом менее точной модели с более приемлемыми значениями штраф будет меньше.

     Унификация L1 и L2 имеют другой эффект и использует. L1 могут применяться к разреженным моделям, что удобно при работе с многомерными данными. Напротив регуляризации L2 является предпочтительным для данных, который не является разреженным.  Этот алгоритм поддерживает линейное сочетание значений регуляризации L1 и L2: то есть если `x = L1` и `y = L2`, `ax + by = c` определяет линейный диапазон условий регуляризации.

     Различные сочетания линейный L1 и L2 условия целей было разработано для моделей логистической регрессии, такие как [эластичной net регуляризации](https://wikipedia.org/wiki/Elastic_net_regularization).

6. **Начальное значение случайного числа**: Введите целое значение для использования в качестве начального значения для алгоритма, если вы хотите, чтобы результаты должны повторяться при каждом выполнении. В противном случае — значение часов системы используется в качестве начального значения, что может привести к немного другие результаты при запусках и того же эксперимента.

8. Подключите набор данных с меткой и один из модулей обучения.

    + Если задать **Создание режима учителя** для **один параметр**, использовать [Обучение модели](./train-model.md) модуля.

9. Запустите эксперимент.

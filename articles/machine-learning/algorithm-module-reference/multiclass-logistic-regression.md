# <a name="multiclass-logistic-regression-module"></a>Многоклассный модуль логистической регрессии

В этой статье описывается модуль в дизайнере машинного обучения Azure (предварительный просмотр).

Используйте этот модуль для создания логистической модели регрессии, которая может быть использована для прогнозирования нескольких значений.

Классификация с использованием логистической регрессии является контролируемым методом обучения, и поэтому требует помеченного набора данных. Вы тренируете модель, предоставляя модель и помеченный набор данных в качестве ввода в модуль, такой как [Train Model.](./train-model.md) Затем обученная модель может использоваться для прогнозирования значений для новых примеров ввода.

Azure Machine Learning также предоставляет [двухклассный модуль Logistic Regression,](./two-class-logistic-regression.md) который подходит для классификации бинарных или дихотомических переменных.

## <a name="about-multiclass-logistic-regression"></a>О многоклассной логистической регрессии

Логистическая регрессия является хорошо известным методом в статистике, который используется для прогнозирования вероятности результата и популярен для классификационных задач. Алгоритм прогнозирует вероятность возникновения события путем установки данных на логистическую функцию. 

В многоклассной логистической регрессии классификатор может использоваться для прогнозирования нескольких исходов.

## <a name="configure-a-multiclass-logistic-regression"></a>Настройка многоклассовой логистической регрессии

1. Добавьте в конвейер **модуль Multiclass Logistic Regression.**

2. Укажите, как нужно обучить модель, установив опцию **«Создать тренер».**

    + **Единый параметр**: Используйте эту опцию, если вы знаете, как настроить модель, и предоставить определенный набор значений в качестве аргументов.

    + **Диапазон параметров**: Выберите этот параметр, если вы не уверены в лучших параметрах, и хотите запустить развертки параметров. Выберите диапазон значений для итерации, и [Hyperparameters Tune Model](tune-model-hyperparameters.md) итерирует все возможные комбинации параметров, которые вы предоставили, чтобы определить гиперпараметры, которые дают оптимальные результаты.  

3. **Допуск оптимизации,** укажите пороговое значение для конвергенции оптимизатора. Если улучшение между итерациями меньше, чем пороговое значение, алгоритм прекращает работу и возвращает текущую модель.

4. **Вес регуляризации L1,** **вес регуляризации L2**: Введите значение для использования для параметров регуляризации L1 и L2. Для обоих параметров рекомендуется использовать ненулевое значение.

    Регуляризация — это метод предотвращения переоборудования путем наказания моделей с экстремальными значениями коэффициентов. Регуляризация работает путем добавления штрафа, связанного с значениями коэффициента, к ошибке гипотезы. Точная модель с экстремальными значениями коэффициентов будет наказываться больше, но менее точная модель с более консервативными значениями будет наказываться меньше.

     Регуляризации L1 и L2 отличаются результатами и способом применения. L1 можно применять к разреженным моделям, что удобно при работе с многомерными данными. L2, напротив, предпочтительнее использовать для неразреженных данных.  Этот алгоритм поддерживает линейное сочетание значений регуляризации `x = L1` L1 `ax + by = c` и L2: то есть, если и, `y = L2`определяет линейный диапазон терминов регуляризации.

     Для моделей логистической регрессии были разработаны различные линейные комбинации терминов L1 и L2, такие как [эластичная чистая регуляризация.](https://wikipedia.org/wiki/Elastic_net_regularization)

6. **Случайное количество семян**: Введите целые значения для использования в качестве семян для алгоритма, если вы хотите, чтобы результаты были повторяемыми в течение нескольких запусков. В противном случае в качестве семени используется значение системных часов, которое может дать несколько иные результаты в ходе одного и того же трубопровода.

8. Подключите помеченный набор данных и один из модулей поездов:

    + При установке **режима создания тренажера** на **единый параметр**используйте модуль [Модели поезда.](./train-model.md)

9. Отправить конвейер.

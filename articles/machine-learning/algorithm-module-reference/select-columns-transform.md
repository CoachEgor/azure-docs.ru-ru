# <a name="select-columns-transform"></a>Преобразование "Выбор столбцов"

В этой статье описывается, как использовать модуль Преобразования избранных столбцов в конструкторе машинного обучения Azure (предварительный просмотр). Цель юнита «Преобразование столбцов» заключается в обеспечении того, чтобы предсказуемый, последовательный набор столбцов использовался в операциях машинного обучения ниже по течению.

Этот модуль полезен для таких задач, как скоринг, которые требуют определенных столбцов. Изменения в доступных столбцах могут привести к разрыву конвейера или изменению результатов.

Для создания и сохранения набора столбцов используется Преобразование Select Columns. Затем используйте модуль [Apply Transformation,](apply-transformation.md) чтобы применить эти выделения к новым данным.

## <a name="how-to-use-select-columns-transform"></a>Как использовать Преобразование избранных столбцов

Этот сценарий предполагает, что вы хотите использовать выбор функций для создания динамического набора столбцов, которые будут использоваться для обучения модели. Чтобы гарантировать, что выбор столбцов одинаков для процесса скоринга, вы используете модуль Преобразования избранных столбцов для захвата выделений столбцов и их применения в других местах конвейера.

1. Добавьте набор входных данных в конвейер в проектировщике.

2. Добавьте экземпляр [выбора функций на основе фильтра.](filter-based-feature-selection.md)

3. Подключите модули и назначаем модуль выбора функций, чтобы автоматически найти ряд лучших функций в наборе входных данных.

4. Добавьте экземпляр [модели train Model](train-model.md) и используйте выход [фильтровального выбора функций](filter-based-feature-selection.md) в качестве ввода для обучения.

    > [!IMPORTANT]
    > Поскольку значение функции основано на значениях в столбце, вы не можете заранее знать, какие столбцы могут быть доступны для ввода [модели train Model.](train-model.md)  
5. Прикрепите экземпляр модуля Преобразования избранных столбцов. 

    Этот шаг генерирует выделение столбца в качестве преобразования, которое может быть сохранено или применено к другим наборам данных. Этот шаг гарантирует, что столбцы, идентифицированные в выборе объектов, сохраняются для повторного использования других модулей.

6. Добавьте модуль [модели оценки.](score-model.md) 

   *Не подключайте набор входных данных.* Вместо этого добавьте модуль [Apply Transformation](apply-transformation.md) и подключите выход преобразования выбора функций.

   > [!IMPORTANT]
   > Вы не можете ожидать, чтобы применить [фильтр на основе выбора функций](filter-based-feature-selection.md) для набора данных скоринга и получить те же результаты. Поскольку выбор функций основан на значениях, он может выбрать другой набор столбцов, что приведет к сбою операции скоринга.
7. Отправить конвейер.

Этот процесс сохранения и затем применения выбора столбцов гарантирует, что та же схема данных доступна для обучения и оценки.

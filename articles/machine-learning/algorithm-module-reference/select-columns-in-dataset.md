# <a name="select-columns-in-dataset-module"></a>Выберите столбцы в модуле набора данных

В этой статье описывается модуль в дизайнере машинного обучения Azure (предварительный просмотр).

Используйте этот модуль, чтобы выбрать подмножество столбцов для использования в операциях ниже по течению. Модуль физически не удаляет столбцы из исходного набора данных; вместо этого он создает подмножество столбцов, подобно *представлению* базы данных или *проекции.*

Этот модуль полезен, когда необходимо ограничить столбцы, доступные для операции ниже по течению, или если вы хотите уменьшить размер набора данных, удалив ненужные столбцы.

Столбцы в наборе данных выходят в том же порядке, что и в исходных данных, даже если вы указываете их в другом порядке.

## <a name="how-to-use"></a>Использование

Этот модуль не имеет параметров. Селектор столбцов выбирает столбцы для включения или исключения столбцов.

### <a name="choose-columns-by-name"></a>Выберите столбцы по имени

В модуле есть несколько вариантов выбора столбцов по имени: 

+ Фильтр и поиск

    Нажмите на опцию **BY NAME.**

    Если вы подключили уже заполненный набор данных, должен появиться список доступных столбцов. Если столбцы не отображаются, возможно, потребуется запустить модули вверх по течению для просмотра списка столбцов.

    Чтобы отфильтровать список, введите в поле поиска. Например, если вы `w` введете письмо в поле поиска, список фильтруется, чтобы показать имена столбцов, которые содержат букву. `w`

    Выберите столбцы и нажмите кнопку правой стрелки, чтобы переместить выбранные столбцы в список в правой панели.

    + Чтобы выбрать непрерывный диапазон названий столбцов, нажмите **Shift и нажмите кнопку**.
    + Чтобы добавить отдельные столбцы в выбор, нажмите **Ctrl и нажмите**кнопку .

    Нажмите кнопку галочку, чтобы сохранить и закрыть.

+ Использование имен в сочетании с другими правилами

    Нажмите на опцию **WITH RULES.**
    
    Выберите правило, например отображение столбцов определенного типа данных.

    Затем щелкните отдельные столбцы этого типа по имени, чтобы добавить их в список выбора.

+ Введите или вставьте запятой разделенный список имен столбцов

    Если набор данных широк, может быть проще использовать индексы или сгенерированные списки имен, а не выбирать столбцы по отдельности. Предполагая, что вы подготовили список заранее:

    1. Нажмите на опцию **WITH RULES.** 
    2. Выберите **Нет столбцов,** выберите **Включить**, а затем нажмите внутри текстового окна с красным восклицательным знаком. 
    3. Вставьте или введите список, разделенный запятой, ранее проверенных имен столбцов. Вы не можете сохранить модуль, если какой-либо столбец имеет недействительное имя, поэтому не забудьте проверить имена заранее.
    
    Этот метод можно также использовать для указания списка столбцов, используя их значения индекса. 

### <a name="choose-by-type"></a>Выбор по типу

Если вы используете опцию **WITH RULES,** можно применить несколько условий на выборе столбца. Например, может потребоваться получить только столбцы функций типа численных данных.

Опция **BEGIN WITH** определяет вашу отправную точку и важна для понимания результатов. 

+ Если вы выберете опцию **ALL COLUMNS,** все столбцы добавляются в список. Затем необходимо использовать опцию **Exclude** для *удаления* столбцов, отвечающих определенным условиям. 

    Например, можно начать со всех столбцов, а затем удалить столбцы по имени или по типу.

+ Если вы выберете опцию **NO COLUMNS,** список столбцов начинается пустым. Затем вы указываете условия для *добавления* столбцов в список. 

    Если вы применяете несколько правил, каждое условие является **добавкой.** Например, например, вы начинаете с столбцов, а затем добавляете правило, чтобы получить все числовые столбцы. В наборе данных о ценах на автомобили это приводит к 16 столбиков. Затем вы нажимаете **+** на знак, чтобы добавить новое условие, и выберите Включить все **функции.** Полученный набор данных включает в себя все числовые столбцы, а также все столбцы функций, включая некоторые столбцы функций строки.

### <a name="choose-by-column-index"></a>Выберите по индексу столбцов

Индекс столбцов относится к порядку столбца в исходном наборе данных.

+ Столбцы пронумероватемы последовательно, начиная с 1.  
+ Чтобы получить диапазон столбцов, используйте дефис. 
+ Открытые спецификации, `1-` такие `-3` как или не допускаются.
+ Двойные значения индекса (или названия столбцов) не допускаются, что может привести к ошибке.

Например, если предположить, что в наборе данных имеется не менее восьми столбцов, можно вставить в любом из следующих примеров доступ нескольких несмежеющих столбцов: 

+ `8,1-4,6`
+ `1,3-8`
+ `1,3-6,4` 

окончательный пример не приводит к ошибке; однако, он возвращает один `4`экземпляр столбца .



### <a name="change-order-of-columns"></a>Изменение порядка столбцов

Опция **Разрешить дубликаты и сохранить столбец порядок в выборе** начинается с пустого списка, и добавляет столбцы, которые вы указать по имени или индексу. В отличие от других вариантов, которые всегда возвращают столбцы в их "естественном порядке", этот вариант выводит столбцы в порядке, который вы называете или перечисляете их. 

Например, в наборе данных с столбиками Col1, Col2, Col3 и Col4 можно изменить порядок столбцов и оставить столбец 2, указав любой из следующих списков:

+ `Col4, Col3, Col1`
+ `4,3,1`

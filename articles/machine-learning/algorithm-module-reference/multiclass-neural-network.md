# <a name="multiclass-neural-network-module"></a>Модуль мультиклассовой нейронной сети

В этой статье описывается модуль визуального интерфейса (Предварительная версия) для службы машинного обучения Azure.

Этот модуль используется для создания модели нейронной сети, который может использоваться для прогнозирования цели, имеющей несколько значений. 

Например нейронные сети такого рода может использоваться в сложных задачах машинного распознавания образов, таких как распознавание цифр или букв, классификация документов и распознавание шаблонов.

Классификация с использованием нейронных сетей — это метод контролируемого обучения, почему и требует *с тегами набора данных* , содержащий столбец меток.

Вы можете обучить модель, предоставляя модели и маркированному набору данных в качестве входных данных [Обучение модели](./train-model.md). Затем обученную модель можно использовать для прогнозирования новых примеров входных данных.  

## <a name="about-neural-networks"></a>О нейронных сетей

Нейронная сеть представляет набор взаимосвязанных слоев. Входные данные являются первый слой и подключены к выходной слой с ациклического графа, состоящая из взвешенных ребер и узлов.

Между входного и выходного слоев можно вставить несколько скрытых слоев. Большинство задач прогнозирования можно легко выполнить с только одним или несколькими скрытыми слоями. Тем не менее последние исследования показали, что глубокие нейронные сети (DNN) с помощью множества слоев могут быть эффективны в решении сложных задач, таких как распознавание изображений или речи. Для моделирования возрастающих уровней семантической глубины используются последовательные слои.

Связь между входными и выходными данными может быть получен из обучении нейронной сети на входных данных. Направление графа переходит от входных данных через скрытый слой и выходной слой. Все узлы в слое соединены взвешенными ребрами с узлами следующего слоя.

Чтобы вычислить данные на выходе сети для отдельного входного, значение вычисляется с каждого узла на скрытых слоях и на выходном слое. Значение устанавливается путем вычисления взвешенной суммы значений узлов предыдущего уровня. Функции активации затем применяется к этой взвешенной суммы.

## <a name="configure-multiclass-neural-network"></a>Настройка Мультиклассовой нейронной сети

1. Добавить **Мультиклассовая Нейронная сеть** эксперимент в интерфейсе модуль. Этот модуль, в разделе можно найти **машинного обучения**, **инициализировать**в **классификации** категории.

2. **Создание режима учителя**: Используйте этот параметр для указания способа обучения модели:

    - **Единый параметр**: Выберите этот параметр, если вы уже знаете, как вы хотите настроить модель.

    

3. **Скрытые спецификации слой**: Выберите тип сетевой архитектуры для создания.

    - **Полностью подключен случай**: Выберите этот параметр, чтобы создать модель, используя в архитектуре нейронной сети по умолчанию. Для моделей мультиклассовой нейронной сети значения по умолчанию задаются следующим образом:

        - Один скрытый слой
        - Выходной слой полностью подключен к скрытому слою.
        - Скрытый слой полностью подключен ко входному слою.
        - Количество узлов во входном слое определяется количеством признаков в обучающих данных.
        - Количество узлов в скрытом слое можно задать для пользователя. Количество по умолчанию — 100.
        - Количество узлов в выходном слое зависит от количества классов.
  
   

5. **Количество скрытых узлов**: Этот параметр позволяет настроить количество скрытых узлов в архитектуре по умолчанию. Введите количество скрытых узлов. По умолчанию используется один скрытый слой с 100 узлов.

6. **Скорость обучения**: Определите размер шага, выполненный на каждой итерации, до осуществления коррекции. Увеличьте значение скорости обучения может привести к ускорить Сходимость модели, но возможны Промахи на локальных минимумах.

7. **Число итераций обучения**: Укажите максимальное количество раз, алгоритм должен обрабатывать обучающих вариантов.

8. **Начальных весовых коэффициентов обучения диаметр**: Укажите весовые коэффициенты узлов в начале процесса обучения.

9. **Импульс**: Укажите весовой коэффициент, применяемый во время обучения к узлам из предыдущих итераций.
  
11. **Случайный порядок примеров**: Выберите этот параметр, чтобы в случайном порядке случаях между итерациями.

    Если вы отключите этот параметр, случаи обрабатываются в точно в том порядке каждый раз при запуске эксперимента.

12. **Начальное значение случайного числа**: Введите значение для использования в качестве начального значения, если вы хотите проверить повторяемость несколькими запусками одного эксперимента.

14. Подключите обучающий набор данных и один из [обучающих модулей](module-reference.md): 

    - Если задать **Создание режима учителя** для **один параметр**, использовать [Обучение модели](train-model.md).  
  

## <a name="results"></a>Результаты

После завершения обучения:

- Чтобы просмотреть сводку параметров модели, а также функцию весовые коэффициенты, узнали из обучения и другие параметры нейронной сети, щелкните правой кнопкой мыши выходные данные [Обучение модели](./train-model.md) и выберите **визуализировать**.  

- Чтобы сохранить моментальный снимок обученной модели, щелкните правой кнопкой мыши **модели Trained** выходных данных и выберите **Сохранить как Обученную модель**. Эта модель не обновляется при последующих запусках одного эксперимента.

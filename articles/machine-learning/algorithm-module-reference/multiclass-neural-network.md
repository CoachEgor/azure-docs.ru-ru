# <a name="multiclass-neural-network-module"></a>Многоклассовый модуль нейронной сети

В этой статье описывается модуль в дизайнере машинного обучения Azure (предварительный просмотр).

Этот модуль применяется для создания модели нейронной сети, которая может использоваться для прогнозирования цели, имеющей несколько значений. 

Например, нейронные сети такого рода могут использоваться в сложных задачах компьютерного зрения, таких как распознавание цифр или букв, классификация документов и распознавание шаблонов.

Классификация с использованием нейронных сетей является контролируемым методом обучения, и поэтому требует *тегами набора данных,* который включает столбец метки.

Вы можете обучить модель, предоставив модель и помеченный набор данных в качестве ввода модели [train Model.](./train-model.md) Затем обученная модель может использоваться для прогнозирования значений для новых примеров ввода.  

## <a name="about-neural-networks"></a>О нейронных сетях

Нейронная сеть представляет собой набор взаимосвязанных слоев. Входные данные являются первым слоем и соединены с выходным слоем ациклическим графиком, состоящим из взвешенных краев и узлов.

Между входной и выходной слоями можно вставить несколько скрытых слоев. Большинство задач прогнозирования можно легко выполнить только с одним или несколькими скрытыми слоями. Однако недавние исследования показали, что глубокие нейронные сети (DNN) со многими слоями могут быть эффективными в сложных задачах, таких как распознавание изображений или речи. Последовательные слои используются для моделирования возрастающих уровней семантической глубины.

Взаимосвязь между входными данными и выходами изучена из обучения нейронной сети входным данным. Направление графика происходит от входов через скрытый слой и к выходу слоя. Все узлы в слое соединены взвешенными краями с узлами следующего слоя.

Для вычисления вывода сети для конкретного ввода значение рассчитывается на каждом узло в скрытых слоях и в выходном слое. Значение устанавливается путем расчета средневзвешенной суммы значений узлов из предыдущего слоя. Функция активации затем применяется к этой взвешенной сумме.

## <a name="configure-multiclass-neural-network"></a>Настройка многоклассовой нейронной сети

1. Добавьте модуль **MultiClass Neural Network** в конвейер в проектировщике. Вы можете найти этот модуль под **машинного обучения**, **Инициализация**, в категории **классификации.**

2. **Создайте режим тренера:** Используйте эту опцию, чтобы указать, как вы хотите, чтобы модель была обучена:

    - **Единый параметр**: Выберите этот параметр, если вы уже знаете, как настроить модель.

    - **Диапазон параметров**: Выберите этот параметр, если вы не уверены в лучших параметрах, и хотите запустить развертки параметров. Выберите диапазон значений для итерации, и [Hyperparameters Tune Model](tune-model-hyperparameters.md) итерирует все возможные комбинации параметров, которые вы предоставили, чтобы определить гиперпараметры, которые дают оптимальные результаты.  

3. **Скрытый слой спецификации:** Выберите тип сетевой архитектуры для создания.

    - **Полностью подключенный случай**: Выберите эту опцию для создания модели с использованием архитектуры нейронной сети по умолчанию. Для моделей многоклассных нейронных сетей по умолчанию следующие:

        - Один скрытый слой
        - Выходной слой полностью подключен к скрытому слою.
        - Скрытый слой полностью подключен к вхотозному слою.
        - Количество узлов во входном слое определяется количеством признаков в обучающих данных.
        - Количество узлов в скрытом слое может быть установлено пользователем. Значение по умолчанию — 100.
        - Количество узлов в выходном слое зависит от количества классов.
  
   

5. **Количество скрытых узлов:** Эта опция позволяет настроить количество скрытых узлов в архитектуре по умолчанию. Введите число скрытых узлов. По умолчанию это один скрытый слой с 100 узлами.

6. **Скорость обучения**: Определите размер шага, сделанного при каждой итерации, до коррекции. Большее значение для скорости обучения может привести к более быстрому сближению модели, но может вывести из-под выпадения локальных минимумов.

7. **Количество итераций обучения**: Укажите максимальное количество раз, когда алгоритм должен обрабатывать обучающие случаи.

8. **Первоначальный диаметр веса обучения**: Укажите веса узла в начале учебного процесса.

9. **Импульс**: Укажите вес, который следует применять во время обучения к узлам из предыдущих итераций.
  
11. **Перемешать примеры**: Выберите эту опцию для перетасовки случаев между итерациями.

    При невыборе этой опции случаи обрабатываются в точно таком же порядке при запуске конвейера.

12. **Случайное количество семян**: Введите значение для использования в качестве семян, если вы хотите обеспечить повторяемость через пробеги одного и того же конвейера.

14. Подключите набор обучаемых данных и один из [учебных модулей:](module-reference.md) 

    - Если вы установите **режим создания тренера** для single **Parameter,** используйте [модель train.](train-model.md)  
  

## <a name="results"></a>Результаты

После завершения обучения:

- Чтобы сохранить моментальный снимок обученной модели, выберите вкладку **Выводы** в правой панели **модуля модели Train.** Выберите значок **набора данных Регистра,** чтобы сохранить модель в качестве многоразового модуля.

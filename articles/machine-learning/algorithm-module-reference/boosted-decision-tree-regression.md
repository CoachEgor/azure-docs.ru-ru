# <a name="boosted-decision-tree-regression-module"></a>Модуль увеличивающегося Регрессия дерева принятия решений

В этой статье описывается модуль визуального интерфейса (Предварительная версия) для службы машинного обучения Azure.

Этот модуль используется для создания совокупности деревьев регрессии. *Бустинг* означает, что каждое дерево зависит от предыдущих деревьев. Алгоритм обучается путем обеспечения соответствия остальным деревьям, расположенным перед его. Таким образом бустинг в совокупности деревьев принятия решений, как правило, для повышения точности с небольшим риском снижения покрытия.  
  
Этот метод регрессии — это метод контролируемого обучения и поэтому требует *с меткой набора данных*. Столбец меток должен содержать числовые значения.  

> [!NOTE]
> Этот модуль используется только с наборами данных, использующих числовые переменные.  

После определения модели, выполнить его обучение с помощью [Обучение модели](./train-model.md).

> [!TIP]
> Хотите узнать больше о деревьях, которые были созданы? После обучения модели, щелкните правой кнопкой мыши выходные данные [Обучение модели](./train-model.md) модуля и выберите **визуализировать** для просмотра дерева, который был создан на каждой итерации. Можно детализировать разбиений для каждого дерева и просмотреть правила, для каждого узла.  
  
## <a name="more-about-boosted-regression-trees"></a>Дополнительные сведения о деревьях регрессии повышенного  

Бустинг — один из нескольких классических методов создания моделей, а также бэггинга, случайные леса и т. д.  В машинном обучении Azure повышенные деревья принятия решений использовать эффективную реализацию алгоритма градиентного бустинга MART. Градиентный бустинг — это метод для проблем регрессии машинного обучения. Он создает каждое дерево регрессии образом пошагово, используя стандартную функцию потерь для измерения ошибок в каждом шаге и их исправления в следующем. Таким образом модель прогнозирования — фактически является набором более слабых моделей прогнозирования.  
  
В проблем регрессии повышая серия деревьев пошагово и затем выбирается оптимальное дерево с помощью произвольной дифференцируемой функции потерь.

## <a name="how-to-configure-boosted-decision-tree-regression"></a>Как настроить Регрессия повышенного дерева принятия решений

1.  Добавить **повышенного дерева принятия решений** эксперимент модуль. Этот модуль, в разделе можно найти **машинного обучения**, **инициализировать**в разделе **регрессии** категории. 
  
2.  Укажите способ модели для обучения, задав **Создание режима учителя** параметр.  
  
    -   **Единый параметр**: Выберите этот параметр, если вы знаете, как вы хотите настроить модель и предоставляют ряд значения в качестве аргументов.  
   
  
3. **Максимальное количество листьев каждого дерева**: Укажите максимальное число конечных узлов (листьев), которые могут быть созданы в любом дереве.  

    Увеличение этого значения, потенциально увеличивают размер дерева и повысите точность, риском лжевзаимосвязей и больше времени обучения.  

4. **Минимальное число образцов на листовой узел**: Указывает минимальное количество вариантов, необходимых для создания любого конечного узла (листа) в дереве.

    Увеличение этого значения, увеличьте пороговое значение для создания новых правил. Например значением по умолчанию 1, даже один случай может привести к новое правило, чтобы создать. Если вы увеличите значение до 5, обучающие данные должны будут содержать по крайней мере 5 случаев, отвечающие тем же условиям.

5. **Скорости обучения**: Введите число от 0 до 1, который определяет размер шага во время обучения. Скорость обучения определяет, как быстро или медленно ученик сходится оптимальное решение. Если размер шага слишком велик, может перенапряжение оптимальное решение. Если размер шага, слишком мал, обучение занимает больше времени, на которой наилучшим решением.

6. **Количество созданных деревьев**: Укажите общее число деревьев принятия решений для создания в ансамбле. Создавая больше деревьев принятия решений, вы можете потенциально получить большее покрытие, но увеличивает время обучения.

    Это значение также определяет число деревьев, отображаемое, когда визуализация обученной модели. Если вы хотите см. в разделе или распечатать дерево одиночная, можно задать значение 1; Тем не менее, создается только одно дерево (дерево с начальным набором параметров) и выполняются без дальнейших итераций.

7. **Начальное значение случайного числа**: Введите целое неотрицательное необязательно для использования в качестве случайного начального значения. Указав начальное значение, вы обеспечите повторяемость между запусками с теми же параметрами и данными.

    Случайное начальное значение по умолчанию присваивается 0, это означает, что начальное значение получается из системных часов.
  
8. **Разрешить неизвестные категориальные уровни**: Выберите этот параметр, чтобы создать группу для неизвестных значений в наборы для обучения и проверки. Если вы отключите этот параметр, модель сможет принимать только значения, которые содержатся в обучающих данных. Модель может быть менее точной для известных значений, но он может предоставить более точные прогнозы для новых (неизвестных) значений.

9. Добавление набора данных для обучения и один из модулей обучения:

    - Если задать **Создание режима учителя** равным **один параметр**, использовать [Обучение модели](train-model.md) модуля.  
  
    

10. Запустите эксперимент.  
  
### <a name="results"></a>Результаты

После завершения обучения:

+ Для просмотра дерева, который был создан на каждой итерации, щелкните правой кнопкой мыши выходные данные [Обучение модели](train-model.md) модуля и выберите **визуализировать**.
  
     Щелкните каждое дерево, чтобы детализировать разбиений и просмотреть правила для каждого узла.  

+ Чтобы использовать модель для оценки, подключите его к [оценка модели](./score-model.md), для прогнозирования новых примеров входных данных.

+ Чтобы сохранить моментальный снимок обученной модели, щелкните правой кнопкой мыши **модели Trained** выходные данные модуля обучения и выберите **Сохранить как**. Копия обученную модель, которую можно сохранить не обновляется при последующих запусках эксперимента.

# <a name="two-class-support-vector-machine-module"></a>Модуль two-Class Support Vector Machine

В этой статье описывается модуль визуального интерфейса (Предварительная версия) для службы машинного обучения Azure.

Этот модуль используется для создания модели, которая основана на алгоритме машины опорных векторов. 

Метод опорных векторов (SVM) — это хорошо исследованный класс методов контролируемого обучения. Эта конкретная реализация подходит для прогнозирования из двух возможных значений, на основе либо непрерывных или категориальных переменных.

После определения параметров модели, обучения модели, используя обучающие модули и *с тегами набора данных* , содержит столбец метки или результат.

## <a name="about-support-vector-machines"></a>О методе опорных векторов

Метод опорных векторов являются одними из первых алгоритмов машинного обучения и модели SVM используются во многих приложениях, от извлечения информации до классификации текста и изображений. SVM можно использовать для задач классификации и регрессии.

Эта модель SVM является контролируемого обучения модели, нуждающейся в данных с меткой. В процессе обучения, алгоритм анализирует входные данные и распознает шаблоны в пространство многомерных функций называется *гиперплоскостью*.  Все примеры входных представляются в виде точек в этом пространстве и были сопоставлены с выходными категорий, так что категорий разделены широкими и снимите пропуск максимально.

Для прогноза алгоритм SVM назначает новые образцы в одну категорию или другое, их сопоставления в этой же области. 

## <a name="how-to-configure"></a>Процесс настройки 

Для этого типа модели рекомендуется нормализовать набора данных перед его использованием для обучения классификатора.
  
1.  Добавить **Two-Class Support Vector Machine** эксперимент модуль.  
  
2.  Укажите способ модели для обучения, задав **Создание режима учителя** параметр.  
  
    -   **Единый параметр**: Если вы знаете, как вы хотите настроить модель, вы можете предоставить определенный ряд значений в качестве аргументов.  

3.  Для **число итераций**, введите число, которое обозначает количество итераций, используемых при построении модели.  
  
     Этот параметр можно использовать для достижения компромисса между скоростью и точностью обучения.  
  
4.  Для **лямбда-выражение**, введите значение для использования в качестве веса для регуляризации L1.  
  
     Это коэффициент регуляризации можно использовать для настройки модели. Большие значения ведут к ухудшению более сложных моделей.  
  
5.  Выберите параметр **функции нормализации**, если вы хотите компоненты сначала обучения.
  
     При применении нормализации перед обучением точки данных по центру в среднем значении и масштабировать, чтобы иметь одну единицу стандартного отклонения.
  
6.  Выберите параметр **проект для единичной сфере**, нормализовать коэффициенты.
  
     Прогнозирование значений в пространство единицы означает, что перед обучением точки данных по центру в 0 и масштабируется до иметь одну единицу стандартного отклонения.
  
7.  В **начальное значение случайного числа**, введите целое значение для использования в качестве начальное значение, если вы хотите обеспечить повторяемость между запусками.  В противном случае — значение часов системы используется в качестве начального значения, которое можно привести немного другие результаты между запусками.
  
9. Подключение набора данных с меткой и одна из [обучающих модулей](module-reference.md):
  
    -   Если задать **Создание режима учителя** для **один параметр**, использовать [Обучение модели](train-model.md) модуля.
  

10. Запустите эксперимент.

# <a name="export-data-module"></a>Модуль экспортных данных

В этой статье описывается модуль в дизайнере машинного обучения Azure (предварительный просмотр).

Используйте этот модуль для сохранения результатов, промежуточных данных и рабочих данных из конвейеров в облачные места хранения. 

Этот модуль поддерживает экспорт данных в следующие службы облачных данных:

- Контейнер Azure Blob
- Общая папка Azure
- Azure Data Lake;
- Azure Data Lake 2-го поколения

Перед экспортом данных необходимо сначала зарегистрировать хранилище данных в рабочем пространстве Azure Machine Learning. Для получения дополнительной информации смотрите [данные доступа в службах хранения данных Azure.](../how-to-access-data.md)

## <a name="how-to-configure-export-data"></a>Как настроить экспортные данные

1. Добавьте модуль **экспортных данных** в конвейер в проектировщике. Этот модуль можно найти в категории **ввода и вывода.**

1. Подключите **экспортные данные** к модулю, содержащему данные, которые необходимо экспортировать.

1. Выберите **экспортные данные,** чтобы открыть панель **Свойств.**

1. Для **Datastore**выберите существующий магазин данных из списка выпадающих данных. Можно также создать новый хранилище данных. Проверьте, как можно ознакомиться с [данными Access в службах хранения данных Azure.](../how-to-access-data.md)

1. Флажок, **Регенерировать выход**, решает, следует ли выполнить модуль для регенерации вывода в течение времени выполнения. 

    Это по умолчанию невыбранный, что означает, что если модуль был выполнен с теми же параметрами ранее, система будет повторно использовать выход из последнего запуска, чтобы уменьшить время выполнения. 

    Если он выбран, система снова выполнит модуль для регенерации вывода.

1. Определите путь в хранилище данных, где находятся данные. Путь является относительным путем. Пустые пути или пути URL не допускаются.


1. Для **формата файла**выберите формат, в котором должны храниться данные.
 
1. Отправить конвейер.

# <a name="export-data-module"></a>Модуль экспорта данных

В этой статье описывается модуль визуального интерфейса (Предварительная версия) для службы машинного обучения Azure.

Этот модуль используется для сохранения результатов, интеллектуальному анализу данных и рабочие данные из экспериментов в облаке места хранения за пределами машинного обучения Azure.

Этот модуль поддерживает экспортом или сохранением данных в следующие облачные службы данных:


- **Экспорт в хранилище BLOB-объектов Azure**: Сохраняет данные в службу BLOB-объектов в Azure. Данные в службе BLOB-объектов может быть общий доступ или сохранен в защищенных хранилищах данных приложения.

  
## <a name="how-to-configure-export-data"></a>Настройка экспорта данных

1. Добавить **Экспорт данных** эксперимент в интерфейсе модуль. Можно найти этот модуль в **входные и выходные данные** категории.

2. Подключение **Экспорт данных** модулю, который содержит данные, необходимо экспортировать.

3. Дважды щелкните **Экспорт данных** открыть **свойства** области.

4. Для **назначение данных**, выберите тип облачного хранилища, в котором вы будете хранить ваши данные. Если внести изменения в этот параметр, все остальные свойства будут сброшены. Поэтому не забудьте сначала выберите этот параметр!

5. В качестве способа учетной записи имени и проверки подлинности требуется для доступа к указанной учетной записи хранения.

    **Экспорт в хранилище BLOB-объектов** является единственным параметром в закрытой предварительной версии. Ниже показано, как установите модуль.
    1. Служба BLOB-объектов Azure — для хранения больших объемов данных, включая двоичные данные. Существует два типа хранилища BLOB-объектов: общедоступным большим двоичным объектам и большие двоичные объекты, требуются учетные данные для входа.

    2. Для **тип проверки подлинности**, выберите **открытый (SAS)** Если вы знаете, что хранилище поддерживает доступ по URL-адрес SAS.

          URL-адрес SAS — это специальный тип URL-адрес, который может быть создан с помощью служебной программы службы хранилища Azure и доступна только в течение ограниченного времени.  Он содержит все сведения, необходимые для проверки подлинности и загрузки.

        Для **URI**введите или вставьте полный URI, определяющий учетной записи и общедоступный большой двоичный объект.

        Формат файла CSV и TSV поддерживаются.

    3. Закрытый учетные записи, выберите **учетной записи**и укажите имя учетной записи и ключ учетной записи, таким образом, эксперимент можно записать в учетную запись хранения.

         - **Имя учетной записи**. Введите или вставьте имя учетной записи, где требуется сохранить данные. Например, если полный URL-адрес учетной записи хранения `http://myshared.blob.core.windows.net`, следует ввести `myshared`.

        - **Ключ учетной записи**. Вставьте ключ доступа к хранилищу, которая связана с учетной записью.

        -  **Путь к контейнеру, каталогу или BLOB-объектов**: Введите имя большого двоичного объекта, где будут храниться экспортированные данные. Например, чтобы сохранить результаты эксперимента в большой двоичный объект с именем **results01.csv** в контейнере **прогнозов** в учетной записи с именем **mymldata**, полный URL-адрес большой двоичный объект может быть `http://mymldata.blob.core.windows.net/predictions/results01.csv`.

            Таким образом, в поле **путь к контейнеру, каталогу или BLOB-объектов**, следует указать контейнера и имя BLOB-объектов, как следует за: `predictions/results01.csv`

        - Если указано имя большого двоичного объекта, который еще не существует, Azure создает большой двоичный объект.

       -  При записи в существующий большой двоичный объект, можно указать перезаписи текущее содержимое большого двоичного объекта, установив свойство **хранилище больших двоичных объектов в режим записи**. По умолчанию это свойство имеет значение **ошибка**, это означает, что всякий раз, когда найден существующий файл больших двоичных объектов с тем же именем, возникает ошибка.


    4. Для **формат файла для файла большого двоичного объекта**, выберите формат, в котором должны быть сохранены данные.

        - **CSV-ФАЙЛ**: Значения с разделителями запятыми (CSV) являются формат хранения по умолчанию. Чтобы экспортировать заголовки столбцов вместе с данными, выберите параметр **строки заголовка BLOB-объектов для записи**.  Дополнительные сведения о запятая - формат с разделителями используются в машинном обучении Azure, см. в разделе [преобразовать в CSV-ФАЙЛ](./convert-to-csv.md).

        - **TSV**: Формат с разделителями табуляции (TSV) совместим со многими средствами обучения машины. Чтобы экспортировать заголовки столбцов вместе с данными, выберите параметр **строки заголовка BLOB-объектов для записи**.  

 
    5. **Использовать кэшированные результаты**: Выберите этот параметр, если вы хотите избежать перезаписи результаты в файл больших двоичных объектов, каждый раз при запуске эксперимента. Если вносить изменения в параметрах модуля, эксперимент записывает результаты только в первый раз, этот модуль выполняется, или при внесении изменений в данных.

    6. Запустите эксперимент.

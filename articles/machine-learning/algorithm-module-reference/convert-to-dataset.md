# <a name="convert-to-dataset"></a>Преобразовать в набор данных

В этой статье описывается использование модуля [Convert to DataSet](convert-to-dataset.md) в машинное обучение Azure Designer (Предварительная версия) для преобразования любых данных из конвейера во внутренний формат, используемый конструктором.
  
В большинстве случаев преобразование не требуется, поскольку Машинное обучение Azure неявно преобразует данные в собственный формат набора данных при выполнении любой операции с данными. 

Однако сохранение данных в формате набора данных рекомендуется, если вы выполнили некоторую нормализацию или очистку набора данных, и вы хотите убедиться, что изменения используются в дальнейших конвейерах.  
  
> [!NOTE]
> [Преобразование в набор данных](convert-to-dataset.md) изменяет только формат данных. Она не сохраняет новую копию данных в рабочей области. Чтобы сохранить набор данных, дважды щелкните порт вывода, выберите **Сохранить как набор данных**и введите новое имя.  
  
## <a name="how-to-use-convert-to-dataset"></a>Использование CONVERT в наборе данных  

Рекомендуется использовать модуль [редактирования метаданных](edit-metadata.md) для подготовки набора данных перед использованием инструкции [Convert to DataSet](convert-to-dataset.md).  Можно добавлять или изменять имена столбцов, настраивать типы данных и т. д.

1.  Добавьте модуль [Преобразование в набор данных](convert-to-dataset.md) в конвейер. Этот модуль можно найти в категории **Преобразование данных** в конструкторе. 

2. Подключите его к любому модулю, который выводит набор данных.   

    Пока данные являются [табличными](https://docs.microsoft.com/python/api/azureml-core/azureml.data.tabulardataset?view=azure-ml-py), их можно преобразовать в набор данных. Сюда относятся данные, загруженные с помощью [импорта данных](import-data.md), данные, созданные с помощью команды [Ввод данных вручную](enter-data-manually.md), или наборы данных, преобразованные с помощью функции [Применить преобразование](apply-transformation.md).

3.  В раскрывающемся списке **действие** укажите, хотите ли вы выполнить очистку данных перед сохранением набора данных:  
  
    - **Нет**: Используйте данные как есть.  
  
    - **Сетмиссингвалуе**: задайте в наборе данных определенное значение для отсутствующего значения. Заполнитель по умолчанию — символ вопросительного знака (?), но можно использовать **настраиваемый параметр отсутствующих значений** для ввода другого значения. Например, если ввести "такси" для **пользовательского отсутствующего значения**, все "такси" в наборе данных будет изменено на "отсутствующие значения".
  
    - **Реплацевалуес**: Используйте этот параметр, чтобы указать одно точное значение, которое должно быть заменено любым другим точным значением.  Можно заменить отсутствующие значения или пользовательские значения, установив метод **Replace** :- **Missing**: Выберите этот параметр, чтобы заменить отсутствующие значения во входном наборе данных. В поле **новое значение**введите значение, чтобы заменить отсутствующие значения на.
            - **Custom**: Выберите этот параметр, чтобы заменить пользовательские значения во входном наборе данных. В поле **настраиваемое значение**введите значение, которое необходимо найти. Например, если данные содержат строку `obs` используемую в качестве заполнителя для отсутствующих значений, следует ввести `obs`. В поле **новое значение**введите новое значение, чтобы заменить исходную строку.
  
    Обратите внимание, что операция **реплацевалуес** применяется только к точным совпадениям. Например, эти строки не затрагиваются: `obs.`, `obsolete`.  
 
  
5.  Запустите конвейер или щелкните правой кнопкой мыши модуль [преобразовать в набор данных](convert-to-dataset.md) и выберите пункт **Выполнить выбранное**.  

## <a name="results"></a>Результаты

+  Чтобы сохранить результирующий набор данных с новым именем, щелкните правой кнопкой мыши выход [преобразовать в набор данных](convert-to-dataset.md) и выберите **Сохранить как набор данных**.  
  
## <a name="technical-notes"></a>Технические примечания  

В этом разделе содержатся сведения о реализации, советы и ответы на часто задаваемые вопросы.

-   Любой модуль, который принимает набор данных как входные данные, может также принимать данные в CSV-или TSV-файл. Перед выполнением кода модуля выполняется предварительная обработка входных данных, что эквивалентно выполнению модуля [Convert to DataSet](convert-to-dataset.md) на входе.  
  
-   Преобразование из формата SVMLight в набор данных невозможно.  
  
-   При указании пользовательской операции замены операция поиска и замены применяется к полным значениям. частичные совпадения не допускаются. Например, можно заменить 3 на-1 или на 33, но нельзя заменить 3 в двузначное число, например 35.  
  
-   Для настраиваемых операций замены замена не будет выполнена автоматически, если вы используете в качестве замены любой символ, который не соответствует текущему типу данных столбца.

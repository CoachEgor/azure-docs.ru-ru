# <a name="convert-to-dataset"></a>Преобразование в набор данных

В этой статье описывается, как использовать модуль Convert to Dataset в конструкторе машинного обучения Azure (предварительный просмотр) для преобразования любых данных для конвейера во внутренний формат разработчика.
  
Конверсия не требуется в большинстве случаев. Azure Machine Learning неявно преобразует данные в родной формат набора данных при выполнении любой операции на данных. 

Мы рекомендуем сохранять данные в формате набора данных, если вы выполнили какую-либо нормализацию или очистку набора данных, и вы хотите убедиться, что изменения используются в других конвейерах.  
  
> [!NOTE]
> Преобразование в набор данных изменяет только формат данных. Он не сохраняет новую копию данных в рабочей области. Чтобы сохранить набор данных, дважды щелкните в выходный порт, выберите **Сохранить в качестве набора данных**и введите новое имя.  
  
## <a name="how-to-use-convert-to-dataset"></a>Как использовать Преобразование в набор данных  

Мы рекомендуем использовать модуль [Edit Metadata](edit-metadata.md) для подготовки набора данных перед использованием «Преобразовать в набор данных». Можно добавлять или изменять имена столбцов, корректировать типы данных и вносить другие изменения по мере необходимости.

1.  Добавьте модуль «Преобразование» в модуль Dataset в конвейер. Этот модуль можно найти в категории **преобразования данных** в проектировщике. 

2. Подключите его к любому модулю, который выводит набор данных.   

    До тех пор, пока данные будут [табликами,](https://docs.microsoft.com/python/api/azureml-core/azureml.data.tabulardataset?view=azure-ml-py)можно преобразовать их в набор данных. Это включает данные, загруженные через [данные импорта,](import-data.md)данные, созданные через [Enter Data Вручную,](enter-data-manually.md)или наборы данных, преобразованные через [Apply Transformation.](apply-transformation.md)

3.  В списке выпадающих **действий** укажите, хотите ли вы сделать очистку данных перед сохранением набора данных:  
  
    - **Нет**: Используйте данные как есть.  
  
    - **SetMissingValue**: Установите определенное значение для отсутствующих значений в наборе данных. Заполнитель по умолчанию — это символ знака вопроса (?), но для ввода другого значения можно использовать опцию **пользовательского отсутствующего значения.** Например, если вы введете **такси** для **пользовательского отсутствующее значение,** то все экземпляры **такси** в наборе данных будут изменены на недостающее значение.
  
    - **ReplaceValues**: Используйте эту опцию, чтобы указать одно точное значение, необходимое для замены на любое другое точное значение. Вы можете заменить недостающие значения или пользовательские значения, установив метод **Replace:**

      - **Отсутствует:** Выберите этот вариант, чтобы заменить недостающие значения в наборе входных данных. Для **нового значения**введите значение, чтобы заменить недостающие значения.
      - **Пользовательские**: Выберите эту опцию, чтобы заменить пользовательские значения в наборе входных данных. Для **пользовательского значения**введите значение, которое вы хотите найти. Например, если данные `obs` содержат строку, используемую в качестве `obs`заполнителя для отсутствующих значений, вы вводите. Для **нового значения**введите новое значение, чтобы заменить исходную строку.
  
    Обратите внимание, что операция **ReplaceValues** применяется только к точным совпадениям. Например, эти строки не `obs.`будут `obsolete`затронуты: .  
 
  
5.  Отправить конвейер.  

## <a name="results"></a>Результаты

+  Чтобы сохранить полученный набор данных с новым именем, выберите **набор данных** регистра значка под вкладкой **Выводы** в правой панели модуля.  
  
## <a name="technical-notes"></a>Технические примечания  

-   Любой модуль, который принимает набор данных в качестве ввода, также может принимать данные в файле CSV или файле TSV. Перед запуском кода модуля входные данные обрабатываются предварительно. Предварительная обработка эквивалентна запуску модуля Convert to Dataset на входе.  
  
-   Нельзя преобразовать из формата SVMLight в набор данных.  
  
-   Когда вы указываете операцию замены пользовательской системы, операция поиска и замены применяется к полным значениям. Частичные совпадения не допускаются. Например, вы можете заменить 3 на -1 или 33, но вы не можете заменить 3 в двухзначном количестве, например 35.  
  
-   При пользовательских операциях замены замена автоматически завершится ошибкой, если использовать в качестве альтернативы любой символ, не соответствующий текущему типу данных столбца.

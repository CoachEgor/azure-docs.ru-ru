# <a name="web-service-inputoutput"></a>Входные и выходные данные веб-службы

В этой статье описывается модуль **ввода веб-службы** и модуль **вывода веб-службы** в машинное обучение Azure Designer (Предварительная версия).

**Входной модуль веб-службы** может подключаться только к порту ввода с типом **датафрамедиректори**. И модуль **вывода веб-службы** можно подключить только из выходного порта с типом **датафрамедиректори**. Два модуля можно найти в дереве модулей в категории **веб-служба** . 

Модуль **входных данных веб-службы** используется для указания того, где пользователь вводит данные в конвейер и модуль **вывода веб-службы** , чтобы указать, где возвращаются пользовательские данные в конвейере вывода в режиме реального времени.

## <a name="how-to-use-web-service-inputoutput"></a>Использование входных и выходных данных веб-службы

- При создании конвейера вывода в режиме реального времени из конвейера обучения автоматически добавляется модуль выходных данных **веб-службы** и **веб-службы** , который показывает, где пользователь вводит данные в конвейер и где возвращаются данные. 

    Дополнительные сведения о [создании конвейера вывода в режиме реального времени](https://docs.microsoft.com/azure/machine-learning/tutorial-designer-automobile-price-deploy#create-a-real-time-inference-pipeline).

    > [!NOTE]
    > Автоматическое создание конвейера вывода в режиме реального времени — это процесс, основанный на правилах, не гарантирует правильность. Вы можете вручную добавить или удалить модули **ввода и вывода веб-службы** для удовлетворения ваших требований. Убедитесь, что в конвейере вывода в режиме реального времени существует по крайней мере один модуль **входных данных веб-службы** и один модуль **вывода веб-службы** . При наличии нескольких модулей для работы с **веб-службами** или **выходных данных веб-службы** убедитесь, что они имеют уникальные имена, и вы можете ввести имя на правой панели модуля.

- Вы также можете вручную создать конвейер вывода в режиме реального времени, добавив **входные модули веб-службы** и **выходные данные веб-службы** в неотправленный конвейер.

    > [!NOTE]
    >  Тип конвейера будет определен при первой отправке. Поэтому не забудьте добавить **входной веб-службы** и модуль **вывода веб-службы** перед первой отправкой, если требуется создать конвейер вывода в режиме реального времени.

   В приведенном ниже примере показано, как вручную создать конвейер вывода в режиме реального времени из модуля **выполнение скрипта Python** . 

   ![Пример](media/module/web-service-input-output-example.png)
   
   После того как вы отправите конвейер и выполнение завершится успешно, вы сможете развернуть конечную точку в режиме реального времени.
   
   > [!NOTE]
   >  В приведенном выше примере **введите данные вручную** — схема данных для входа веб-службы, которая необходима для развертывания конечной точки в режиме реального времени. Как правило, необходимо всегда подключать модуль или набор данных к порту, к которому подключены **входные данные веб-службы** , чтобы предоставить схему данных.

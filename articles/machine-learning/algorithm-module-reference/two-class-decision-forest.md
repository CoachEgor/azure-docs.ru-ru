# <a name="two-class-decision-forest-module"></a>Модуль Двухклассовый лес принятия решений

В этой статье описывается модуль визуального интерфейса (Предварительная версия) для службы машинного обучения Azure.

Этот модуль используется для создания модели, основанной на алгоритме лесов принятия решений машинного обучения.  

Леса принятия решений являются быстрыми защищенными моделями совокупности. Этот модуль является хорошим выбором, если требуется использовать для прогнозирования цели, не более двух результатов. 

## <a name="understanding-decision-forests"></a>Основные сведения о лесах принятия решений

Этот алгоритм леса принятия решений — это метод совокупного обучения предназначен для задач классификации. Методы совокупности основаны на общий принцип, вместо того чтобы полагаться на одну модель, можно получить лучшие результаты и более обобщенной модели, создавая несколько связанных моделей и их объединения каким-либо образом. Как правило модели совокупности обеспечивают большее покрытие и точность, чем одно дерево принятия решений. 

Существует много способов для создания отдельных моделей и объединить их в совокупности. Эта конкретная реализация леса принятия решений работает путем создания нескольких деревьев принятия решений и затем **голосования** за наиболее популярный класс выходных данных. Голосование является одним из better-known методы для создания результатов в совокупности модели. 

+ Создаются много отдельных классификации деревьев, с помощью весь набор данных, но разные (обычно в случайном порядке) отправные точки. Это отличается от подход случайного леса, в котором отдельных деревьев принятия решений может использовать только со случайными часть данных или функций.
+ Каждое дерево в дереве леса принятия решений выводит гистограмму ненормализованной частота меток. 
+ Процесс статистической схемы суммирует эти гистограммы и нормализует результат для получения «вероятности» для каждой метки. 
+ Деревья, имеющих высокий уровень прогнозирования будет иметь больший вес в окончательное решение совокупности.

В целом, деревья принятия решений имеют много преимуществ для задач классификации:
  
- Их можно записать границы нелинейного принятия решений.
- Можно обучить и прогнозирования на основе больших объемов данных, так как они эффективны в вычисление и использование памяти.
- Выбор компонентов интегрировано в процесс обучения и классификации.  
- Деревья позволяет разместить искаженных данных и множество функций.  
- Они являются непараметрическими моделями, это означает, что они могут обрабатывать данные с различными распределениями. 

Тем не менее простые деревья можно снижение вероятности появления лжевзаимосвязей в данных и менее обобщению чем совокупности дерева.

Дополнительные сведения см. в разделе [леса принятия решений](http://go.microsoft.com/fwlink/?LinkId=403677).  

## <a name="how-to-configure"></a>Процесс настройки
  
1.  Добавить **Двухклассовый лес принятия решений** эксперимент в машинном обучении Azure и откройте модуль **свойства** области модуля. 

    Модуль в разделе можно найти **машинного обучения**. Разверните **инициализировать**, а затем **классификации**.  
  
2.  Для **метод повторной выборки**, выберите метод, используемый для создания отдельных деревьев.  Можно выбрать из **Bagging** или **реплицировать**.  
  
    -   **Bagging**: Также называется бэггинга *начальной загрузки статистической обработки*. В этом методе каждое дерево происходит при явном на новый образец, созданный случайным образом выборки с заменой исходного набора данных, пока не получится размер исходного набора данных.  
  
         Выходные данные из моделей объединяются с *голосования*, которая является формой статистической обработки. Каждое дерево в лес принятия решений классификации выводит гистограмму Ненормализованный частота меток. Статистическая схема представляет суммировать эти гистограммы и нормализации для получения «вероятности» для каждой метки. Таким образом деревья, имеющих высокий уровень прогнозирования будет иметь больший вес в окончательное решение совокупности.  
  
         Дополнительные сведения см. в статье Википедии запись для начальной загрузки статистической обработки.  
  
    -   **Реплицировать**: В репликации каждое дерево обучается на точно те же входные данные. Определение, какие разбиения используется предикат для каждого узла дерева остается случайных и деревья будут различными.   
  
3.  Укажите способ модели для обучения, задав **Создание режима учителя** параметр.  
  
    -   **Единый параметр**: Если вы знаете, как вы хотите настроить модель, вы можете предоставить определенный ряд значений в качестве аргументов.
  
4.  Для **число деревьев принятия решений**, введите максимальное число деревьев принятия решений, которые могут быть созданы в ансамбле. Создавая больше деревьев принятия решений, вы можете потенциально получить большее покрытие, но увеличивает время обучения.  
  
    > [!NOTE]
    >  Это значение также определяет число деревьев, отображаемое, когда визуализация обученной модели. Если требуется просмотреть или распечатать одно дерево, можно задать значение 1. Тем не менее, может быть только одно дерево созданных (дерево с начальным набором параметров) и выполняются без дальнейших итераций.
  
5.  Для **Максимальная глубина деревьев принятия решений**, введите число, ограничивающее максимальную глубину дерева принятия решений. Увеличение глубины дерева может повысить точность риском некоторые лжевзаимосвязей и увеличения времени обучения.
  
6.  Для **число случайных разбиений для каждого узла**, введите число разбиений при создании каждого узла дерева. Объект *разделить* означает, что компоненты на каждом уровне дерева (узле) распределяются случайным образом.
  
7.  Для **минимальное число выборок для конечного узла**, указать минимальное количество вариантов, которые необходимы для создания любого конечного узла (листа) в дереве.
  
     Увеличение этого значения, увеличьте пороговое значение для создания новых правил. Например значением по умолчанию 1, даже один случай может привести к новое правило, чтобы создать. Если вы увеличите значение до 5, обучающие данные должны будут содержать по крайней мере пять вариантов, которые удовлетворяют тем же условиям.  
  
8.  Выберите **разрешить неизвестные значения для категориальных признаков** параметр, чтобы создать группу для неизвестных значений в наборах обучения или проверки. Модель может быть менее точной для известных значений, но он может предоставить более точные прогнозы для новых (неизвестных) значений. 

     Если вы отключите этот параметр, модель сможет принимать только значения, которые содержатся в обучающих данных.
  
9. Подключить набор данных с меткой и одна из [обучающих модулей](module-reference.md):  
  
    -   Если задать **Создание режима учителя** для **один параметр**, использовать [Обучение модели](./train-model.md) модуля.  
  
    
## <a name="results"></a>Результаты

После завершения обучения:

+ Для просмотра дерева, который был создан на каждой итерации, щелкните правой кнопкой мыши выходные данные [Обучение модели](./train-model.md) модуля и выберите **визуализировать**.
  
    Щелкните каждое дерево, чтобы детализировать разбиений и просмотреть правила для каждого узла.

+ Чтобы сохранить снимок модели, щелкните правой кнопкой мыши **обученной модели** выходных данных и выберите **сохранить модель**. Сохраненная модель не обновляется при последующих запусках эксперимента.

+ Чтобы использовать модель для оценки, добавьте **Score Model** эксперимента.

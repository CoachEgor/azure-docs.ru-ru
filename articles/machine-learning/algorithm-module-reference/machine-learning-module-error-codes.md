# <a name="exceptions-and-error-codes-for-algorithm--module-reference"></a>Исключения и коды ошибок для ссылки на модуль & алгоритма

Сведения о сообщениях об ошибках и кодах исключений, которые могут возникнуть при использовании модулей в Машинное обучение Azure Designer (Предварительная версия). 

Чтобы устранить эту проблему, просмотрите сведения об ошибке в этой статье, чтобы ознакомиться с общими причинами. Есть два способа получить полный текст сообщения об ошибке в конструкторе:  
 
- Щелкните ссылку, **Просмотрите журнал вывода**в правой области и прокрутите вниз. Подробное сообщение об ошибке отображается в двух последних строках окна.  
  
- Выберите модуль с ошибкой и щелкните красный значок X. Отображается только соответствующий текст ошибки.  
  
Если текст сообщения об ошибке не полезен, отправьте нам сведения о контексте и любых нужных дополнениях или изменениях, отправив отзыв.


## <a name="error-0001"></a>Ошибка 0001  
 Исключение возникает, если не удалось найти один или несколько указанных столбцов набора данных.  
  
 Эта ошибка возникает при выборе столбца для модуля, но выбранные столбцы не существуют во входном наборе данных. Эта ошибка может возникать, если вы вручную ввели имя столбца или если селектор столбцов предоставил предлагаемый столбец, который не существовал в наборе данных при выполнении конвейера.  
  
**Решение:** Повторное посещение модуля вызовет это исключение и проверка правильности имени или имен столбца и наличия всех связанных столбцов.  
  
|Сообщения об исключении|  
|------------------------|  
|Не удалось найти один или несколько указанных столбцов|  
|Столбец с именем или индексом "{0}" не найден|  
|Столбец с именем или индексом "{0}" не существует в "{1}"|  
 

## <a name="error-0002"></a>Ошибка 0002  
 Исключение возникает, если один или несколько параметров не удалось проанализировать или преобразовать из указанного типа в тип, необходимый для целевого типа метода.  
  
 Эта ошибка возникает в Машинное обучение Azure при указании параметра в качестве входных данных, а тип значения отличается от ожидаемого типа, и неявное преобразование не может быть выполнено.  
  
**Решение:** Проверьте требования к модулю и определите требуемый тип значения (строка, целое число, двойное и т. д.).  
  
|Сообщения об исключении|  
|------------------------|  
|Не удалось проанализировать параметр|  
|Не удалось проанализировать параметр "{0}"|  
|Не удалось выполнить синтаксический анализ (преобразование) параметра "{0}" в "{1}"|  
|Не удалось преобразовать параметр "{0}" из "{1}" в "{2}"|  
|Не удалось преобразовать значение параметра "{0}" "{1}" из "{2}" в "{3}"|  
|Не удалось преобразовать значение "{0}" в столбце "{1}" из "{2}" в "{3}" с использованием указанного формата "{4}"|  
  

## <a name="error-0003"></a>Ошибка 0003  
 Исключение возникает, если один или несколько входных данных имеют значение null или пусты.  
  
 Эта ошибка возникает в Машинное обучение Azure, если какие-либо входные или параметры модуля пусты или равны NULL.  Эта ошибка может возникать, например, если не введено какое-либо значение для параметра. Это также может произойти, если выбрать набор данных, содержащий отсутствующие значения, или пустой набор данных.  
  
**Способы устранения:**
 
+ Откройте модуль, который создал исключение, и убедитесь, что указаны все входные данные. Убедитесь, что указаны все необходимые входные данные. 
+ Убедитесь, что данные, загружаемые из хранилища Azure, доступны, а имя или ключ учетной записи не изменились.  
+ Проверьте входные данные на наличие отсутствующих значений или значения NULL.
+ При использовании запроса к источнику данных убедитесь, что данные возвращаются в нужном формате. 
+ Проверьте наличие опечаток или других изменений в спецификации данных.
  
|Сообщения об исключении|  
|------------------------|  
|Один или несколько входных данных имеют значение null или пусты.|  
|Входные данные "{0}" пусты или равны NULL|

## <a name="error-0007"></a>Ошибка 0007  
 Исключение возникает, если параметр больше определенного значения.  
  
 Эта ошибка возникает в Машинное обучение Azure, если в свойствах модуля указано значение, превышающее разрешенное. Например, можно указать данные, находящиеся за пределами диапазона поддерживаемых дат, или указать, что при наличии только трех столбцов будут использоваться пять столбцов. 
 
 Эта ошибка может также возникать, если вы указываете два набора данных, которые необходимо точно сопоставить. Например, при переименовании столбцов и задании столбцов по индексу количество указываемых имен должно совпадать с числом индексов столбцов. Другим примером может быть математическая операция, использующая два столбца, где столбцы должны иметь одинаковое число строк. 
  
**Способы устранения:**
 
 + Откройте рассматриваемый модуль и проверьте параметры числовых свойств.
 + Убедитесь, что значения параметров находятся в пределах поддерживаемого диапазона значений для этого свойства.
 + Если модуль принимает несколько входов, убедитесь, что входные данные имеют одинаковый размер.

 + Проверьте, изменился ли набор данных или источник данных. Иногда значение, которое работало с предыдущей версией данных, завершится ошибкой после того, как число столбцов, тип данных столбца или размер данных изменились.  
  
|Сообщения об исключении|

## <a name="error-0009"></a>Ошибка 0009  
 Исключение возникает, если имя учетной записи хранения Azure или имя контейнера указано неправильно.  
  
Эта ошибка возникает в Машинное обучение Azure конструкторе при указании параметров для учетной записи хранения Azure, но имя или пароль не могут быть разрешены. Ошибки паролей или имен учетных записей могут происходить по многим причинам.
 
 + Неверный тип учетной записи. Некоторые новые типы учетных записей не поддерживаются в конструкторе Машинное обучение. Дополнительные сведения см. в разделе [Импорт данных](import-data.md) .
 + Введено неправильное имя учетной записи
 + Учетная запись больше не существует
 + Неправильный пароль для учетной записи хранения или она изменилась
 + Вы не указали имя контейнера, или контейнер не существует
 + Путь к файлу не был полностью указан (путь к BLOB-объекту)
   
**Способы устранения:**

Такие проблемы часто возникают при попытке вручную ввести имя учетной записи, пароль или путь к контейнеру. Рекомендуется использовать новый мастер для модуля [Импорт данных](import-data.md) , который помогает искать и проверять имена.

Также проверьте, удалена ли учетная запись, контейнер или большой двоичный объект. Используйте другую служебную программу службы хранилища Azure, чтобы убедиться, что имя учетной записи и пароль введены правильно и что контейнер существует. 

Некоторые новые типы учетных записей не поддерживаются Машинное обучение Azure. Например, новые "горячий" или "холодный" тип хранилища нельзя использовать для машинного обучения. Как классические учетные записи хранения, так и учетные записи хранения, созданные как "Общее назначение", работают нормально.

Если указан полный путь к большому двоичному объекту, убедитесь, что путь указан как **Container/blobname**и что контейнер и BLOB-объект существуют в учетной записи.  
  
 Путь не должен содержать начальную косую черту. Например, **/контаинер/блоб** неверен и должен быть введен в качестве **контейнера или большого двоичного объекта**.  

  
|Сообщения об исключении|  
|------------------------|  
|Неверное имя учетной записи хранения Azure или имя контейнера.|  
|Имя учетной записи хранения Azure "{0}" или имя контейнера "{1}" неверно; Ожидалось имя контейнера контейнера или BLOB-объекта формата.|

## <a name="error-0011"></a>Ошибка 0011  
 Исключение возникает, если переданный аргумент набора столбцов не применяется ни к одному из столбцов набора данных.  
  
 Эта ошибка возникает в Машинное обучение Azure, если указанный выбор столбцов не совпадает ни с одним из столбцов в данном наборе данных.  
  
 Кроме того, эту ошибку можно получить, если вы не выбрали столбец и для работы модуля требуется хотя бы один столбец.  
  
**Решение:** Измените выбор столбца в модуле, чтобы он применялся к столбцам в наборе данных.  
  
 Если для модуля требуется выбрать конкретный столбец, например столбец меток, убедитесь, что выбран нужный столбец.  
  
 Если выбраны недопустимые столбцы, удалите их и перезапустите конвейер.

|------------------------|  
|Указанный набор столбцов не применяется ни к одному из столбцов набора данных.|  
|Указанный набор столбцов "{0}" не применяется ни к одному из столбцов набора данных.|

## <a name="error-0014"></a>Ошибка 0014  
 Исключение возникает, если количество уникальных значений столбца больше допустимого значения.  
  
 Эта ошибка возникает, когда столбец содержит слишком много уникальных значений.  Например, эта ошибка может возникать, если указать, что столбец должен обрабатываться как данные категории, но в столбце слишком много уникальных значений, чтобы разрешить завершение обработки. Эта ошибка может также возникнуть в случае несоответствия числа уникальных значений в двух входных данных.   
  
**Способы устранения:**

Откройте модуль, создавший ошибку, и найдите столбцы, используемые в качестве входных данных. Для некоторых модулей можно щелкнуть правой кнопкой мыши входные данные набора данных и выбрать **визуализировать** , чтобы получить статистику по отдельным столбцам, включая количество уникальных значений и их распределение.

Для столбцов, которые предполагается использовать для группирования или категоризации, выполните шаги по сокращению числа уникальных значений в столбцах. В зависимости от типа данных столбца их можно уменьшить различными способами.

## <a name="error-0021"></a>Ошибка 0021  
 Исключение возникает, если число строк в некоторых наборах данных, переданных в модуль, слишком мало.

Например, эта ошибка может возникать, если входной набор данных пуст или если выполняется попытка выполнить операцию, требующую допустимого минимального количества строк.  + Откройте модуль, который вернул ошибку, и проверьте входной набор данных и свойства модуля. 
 + Убедитесь, что входной набор данных не пуст и имеется достаточно строк данных для удовлетворения требований, описанных в разделе Справка по модулям.

 + При выполнении операции с вышестоящим потоком данных, который может повлиять на тип данных, или на количество значений, таких как очистка, разделение или операции объединения, проверьте выходные данные этих операций, чтобы определить количество возвращаемых строк.

 Эта ошибка возникает, если набор данных подключен к модулю, для которого требуется несколько столбцов, включая функции, но вы указали только столбец меток.  
  
**Решение:** Выберите по крайней мере один столбец компонентов для включения в набор данных.  
  
|Сообщения об исключении|  
|------------------------|  
|Указанный набор столбцов не содержит других столбцов, кроме столбца меток.|  
  

## <a name="error-0141"></a>Ошибка 0141  
 Исключение возникает, если число выбранных числовых столбцов и уникальных значений в столбцах и строках категорий не слишком мало.  
  
 Эта ошибка в Машинное обучение Azure возникает, если в выбранном столбце недостаточно уникальных значений для выполнения операции.  
  
**Решение:** Некоторые операции выполняют статистические операции с характеристиками и столбцами категорий, и если значения недостаточно, операция может завершиться ошибкой или вернуть недопустимый результат. Проверьте набор данных, чтобы узнать, сколько значений есть в столбцах "функция" и "метка", и определите, является ли операция, которую вы пытаетесь выполнить, статистически допустимой.  
  
 Если исходный набор данных является допустимым, можно также проверить, изменились ли данные в некоторых восходящих операциях обработки данных или операции с метаданными, и удалить некоторые значения.  
  
 Если в вышестоящей операции есть разбиение, выборка или ресамплинг, убедитесь, что выходные данные содержат ожидаемое число строк и значений.  
  
|Сообщения об исключении|  
|------------------------|  
|Число выбранных числовых столбцов и уникальных значений в столбцах категорий и строк слишком мало.|  
|Общее число выбранных числовых столбцов и уникальных значений в столбцах категорий и строк (в настоящее время {0}) должно быть не менее {1}|

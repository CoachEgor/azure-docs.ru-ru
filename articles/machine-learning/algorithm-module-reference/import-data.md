# <a name="import-data-module"></a>Импорт модуля данных

В этой статье описывается модуль визуального интерфейса (Предварительная версия) для службы машинного обучения Azure.

Этот модуль используется для загрузки данных в эксперимент машинного обучения из существующих облачных служб данных.  
Теперь модуль содержит мастер, чтобы помочь вам выбрать вариант хранения данных и выбрать один из существующих подписок и учетных записей, чтобы быстро настроить все параметры. Необходимо изменить существующее подключение к данным? Нет проблем; Мастер загружает все предыдущие сведения о конфигурации, чтобы у вас нет еще раз начинать с нуля. 
  
После определения данные и подключения к источнику, [импорта данных](./import-data.md) выводит тип данных каждого столбца, на основе значений, которые она содержит и загружает данные в рабочей области машинного обучения Azure. Выходные данные [импорта данных](./import-data.md) представляет собой набор данных, который может использоваться с любой эксперимент.

  
При изменении источника данных, можно обновить набор данных и добавить новые данные путем запуска [импорта данных](./import-data.md). Тем не менее, если вы не хотите повторно считывается из источника при каждом запуске эксперимента, выберите **использовать кэшированные результаты** значение TRUE для параметра. Если этот параметр выбран, модуль проверяет, запускалось ли эксперимента ранее с помощью тех же исходной и одинаковые входные параметры. Если обнаруживается предыдущего запуска, использовать данные в кэше, вместо повторной загрузки данных из источника.

## <a name="how-to-use-import-data"></a>Как использовать Импорт данных
 
1. Добавить **импорта данных** эксперимент модуль. Можно найти этот модуль в **ввод и вывод данных** категории в интерфейсе.

2. Нажмите кнопку **запустить мастер импорта данных** для настройки источника данных с помощью мастера.

    Мастер получает имя учетной записи и учетные данные и настроить другие параметры справки. При редактировании существующей конфигурации, он сначала загружает текущие значения.

3. Если вы не хотите использовать мастер, нажмите кнопку **источника данных**и выберите тип облачного хранилища, выполняется чтение. 

    Дополнительные параметры зависят от типа хранилища выбран, и защищена ли хранилище или нет. Может потребоваться указать имя учетной записи, тип файла или учетные данные. Некоторые источники не требуют проверки подлинности; для других пользователей необходимо знать имя учетной записи, ключ или имя контейнера.

4. Выберите **использовать кэшированные результаты** параметр, если вы хотите кэшировать набор данных для повторного использования при последующих запусках.

    Предположим, что произошло вносить изменения в параметрах модуля, эксперимент загружает время данных только первый модуль выполняется, а затем использует кэшированную версию набора данных.

    Снимите этот флажок, если необходимо перезагрузить данные каждый раз при запуске эксперимента.

5. Запустите эксперимент.

    При импорте данных загружает данные в интерфейсе, он выводит тип данных каждого столбца, на основе значений, он содержит, числовых или категориальных.

    - Если заголовок присутствует, заголовок используется для именования столбцов выходного набора данных.

    - Если нет существующих заголовков столбцов в данных, новые имена столбцов создаются с помощью формата col1, col2... , coln *.

## <a name="results"></a>Результаты

По завершении импорта, щелкните выходной набор данных и выберите **визуализировать** для просмотра, если данные были успешно импортированы.

Если вы хотите сохранить данные для повторного использования, вместо того, чтобы импортировать новый набор данных при каждом запуске эксперимента, щелкните правой кнопкой мыши результат и выберите **сохранение набора данных**. Выберите имя для набора данных. Сохраненного набора данных сохраняет данные во время сохранения и данные не обновляются при повторном запуске эксперимента, даже при изменении набора данных в эксперименте. Это может быть удобно для создания моментальных снимков данных.

После импорта данных, может потребоваться некоторые дополнительные приготовления для моделирования и анализа:


- Используйте [изменить метаданные](./edit-metadata.md) Чтобы изменить имена столбцов, для обработки в качестве другого типа данных столбец или указывает, что некоторые столбцы меток или функции.

- Используйте [Выбор столбцов в наборе данных](./select-columns-in-dataset.md) для выбора подмножества столбцов для преобразования или использовать в моделировании. Преобразованные или удаленные столбцы можно легко повторно присоединить к исходный набор данных с помощью [Добавление столбцов](./add-columns.md) модуля.  

- Используйте [секционирование и выборка](./partition-and-sample.md) разделить набор данных, сделать выборку или получить n первых строк.

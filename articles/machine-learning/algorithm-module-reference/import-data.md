# <a name="import-data-module"></a>Модуль данных об импорте

В этой статье описывается модуль в дизайнере машинного обучения Azure (предварительный просмотр).

Используйте этот модуль для загрузки данных в конвейер машинного обучения из существующих служб облачных данных. 

> [!Note]
> Вся функциональность, предоставляемая этим модулем, может быть выполнена **хранилищем данных** и **наборами данных** на целевой странице worksapce. Мы рекомендуем использовать **хранилище данных** и **набор данных,** который включает в себя дополнительные функции, такие как мониторинг данных. Чтобы узнать больше, смотрите [как получить доступ к данным](../how-to-access-data.md) и как зарегистрировать статью [наборов данных.](../how-to-create-register-datasets.md)
> После регистрации набора данных вы можете найти его в категории **наборов** -> **данных My Datasets** в интерфейсе конструктора. Этот модуль зарезервирован для studio (классический) пользователей для знакомого опыта. 
>

Модуль **импортных данных** считывает данные из следующих источников:

- URL через HTTP
- Облачные хранилища Azure через [**хранилища данных)**](../how-to-access-data.md)
    - Контейнер Azure Blob
    - Общая папка Azure
    - Azure Data Lake;
    - Azure Data Lake 2-го поколения
    - База данных SQL Azure
    - Azure PostgreS'L    

Прежде чем использовать облачное хранилище, сначала необходимо зарегистрировать хранилище данных в рабочем пространстве Azure Machine Learning. Для получения дополнительной информации [см.](../how-to-access-data.md) 

После определения нужных данных и подключения к источнику **[данные импорта](./import-data.md)** позволяют сделать вывод о типе данных каждого столбца на основе содержащихся значений и загружают данные в конвейер конструктора. Выход **данных импорта** — это набор данных, который можно использовать с любым конвейером конструктора.

При изменении исходных данных можно обновить набор данных и добавить новые [данные, переуславая данные импорта.](./import-data.md)

## <a name="how-to-configure-import-data"></a>Как настроить данные об импорте

1. Добавьте модуль **данных импорта** в конвейер. Этот модуль можно найти в категории **ввода данных и вывода данных** в категории конструктора.

1. Выберите модуль, чтобы открыть правую панель.

1. Выберите **источник данных**и выберите тип источника данных. Это может быть HTTP или хранилище данных.

    Если вы выбираете хранилище данных, можно выбрать существующие хранилища данных, которые уже зарегистрированы в рабочем пространстве Azure Machine Learning, или создать новый магазин данных. Затем определите способ импорта данных в хранилище данных. Вы можете легко просмотреть путь, нажмите **Просмотр Путь** ![импорт-данные-путь](media/module/import-data-path.png)

1. Выберите схему предварительного просмотра для фильтрации столбцов, которые вы хотите включить. Можно также определить расширенные настройки, такие как Delimiter в парасинговых опциях.

    ![импорт-дата-предварительный просмотр](media/module/import-data.png)

1. Флажок, **Регенерировать выход**, решает, следует ли выполнить модуль для регенерации вывода в течение времени выполнения. 

    Это по умолчанию невыбранный, что означает, что если модуль был выполнен с теми же параметрами ранее, система будет повторно использовать выход из последнего запуска, чтобы уменьшить время выполнения. 

    Если он выбран, система снова выполнит модуль для регенерации вывода. Поэтому выберите эту опцию, когда базовые данные в хранилище обновляются, это может помочь получить последние данные.


1. Отправить конвейер.

    Когда данные импорта загружают данные в конструктор, он выражает тип данных каждого столбца на основе значений, которые он содержит, как числетельный, так и категоричный.

    Если заголовок присутствует, то он используется для именования столбцов результирующего набора данных.

    Если в данных нет существующих заголовков столбцов, новые имена столбцов генерируются с помощью формата col1, col2,... , coln.

## <a name="results"></a>Результаты

При завершении импорта щелкните набор выходных данных и выберите **Визуализацию,** чтобы увидеть, были ли данные успешно импортированы.

Если требуется сохранить данные для повторного использования, а не импортировать новый набор данных при запуске конвейера, выберите значок **набора данных Регистра** под вкладкой **Выводы** в правой панели модуля. Выберите имя для набора данных. Сохраненный набор данных сохраняет данные во время экономии, набор данных не обновляется при повторном запуске конвейера, даже если набор данных в конвейере изменяется. Это может быть полезно для получения снимков данных.

После импорта данных может потребоваться дополнительная подготовка к моделированию и анализу:

- Используйте [Edit Metadata](./edit-metadata.md) для изменения имен столбцов, для обработки столбца в качестве другого типа данных или для указания того, что некоторые столбцы являются метками или функциями.

- Используйте [избранные столбцы в наборе данных](./select-columns-in-dataset.md) для выбора подмножества столбцов для преобразования или использования в моделировании. Преобразованные или удаленные столбцы могут быть легко объединены с исходным набором данных с помощью модуля [Add Columns.](./add-columns.md)  

- Используйте [раздел и образец](./partition-and-sample.md) для разделения набора данных, выполнения выборки или получения верхних строк n.

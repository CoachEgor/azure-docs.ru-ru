# <a name="normalize-data-module"></a>Модуля нормализации данных

В этой статье описывается модуль в дизайнере машинного обучения Azure (предварительный просмотр).

Используйте этот модуль для преобразования набора данных путем *нормализации.*

Нормализация — это метод, часто применяемый в рамках подготовки данных к машинному обучению. Цель нормализации заключается в изменении значений численных столбцов в наборе данных для использования общей шкалы, не искажая различий в диапазонах значений или теряя информацию. Нормализация также необходима для некоторых алгоритмов для правильной моделии данных.

Например, предположим, что набор входных данных содержит один столбец со значениями от 0 до 1, а другой столбец со значениями от 10 000 до 100 000. Большая разница в *масштабе* чисел может вызвать проблемы при попытке объединить значения как функции во время моделирования.

*Нормализация* позволяет избежать этих проблем, создавая новые значения, которые поддерживают общее распределение и соотношения в исходных данных, сохраняя при этом значения в масштабе, применяемом во всех числовых столбцах, используемых в модели.

Этот модуль предлагает несколько вариантов преобразования численных данных:

- Вы можете изменить все значения в масштабе 0-1 или трансформировать значения, представляя их как процентильные ряды, а не абсолютные значения.
- Нормализацию можно применить к одному столбцу или к нескольким столбцам в одном наборе данных.
- Если необходимо повторить конвейер или применить те же шаги нормализации к другим данным, можно сохранить шаги при преобразовании нормализации и применить их к другим наборам данных, которые имеют ту же схему.

> [!WARNING]
> Для некоторых алгоритмов требуется, чтобы данные были нормализованы до обучения модели. Другие алгоритмы выполняют собственное масштабирование или нормализацию данных. Поэтому, выбирая алгоритм машинного обучения для использования в построении прогностической модели, обязательно просмотрите требования к данным алгоритма, прежде чем применять нормализацию к обучающим данным.

##  <a name="configure-normalize-data"></a>Настройка нормализации данных

С помощью этого модуля можно применять только один метод нормализации. Таким образом, один и тот же метод нормализации применяется ко всем выбранным столбцов. Для использования различных методов нормализации используйте второй экземпляр **нормализовать данные.**

1. Добавьте модуль **нормализации данных** в конвейер. Вы можете найти модуль в Azure Machine Learning, в разделе **Преобразование данных,** в категории **Масштаб и уменьшить.**

2. Подключите набор данных, содержащий по крайней мере один столбец всех чисел.

3. Используйте селектор столбцов, чтобы выбрать числовые столбцы для нормализации. Если вы не выбираете отдельные столбцы, по умолчанию **все** столбцы численного типа в ввода включены, и тот же процесс нормализации применяется ко всем выбранным столбикам. 

    Это может привести к странным результатам, если вы включаете числовые столбцы, которые не должны быть нормализованы! Всегда проверяйте столбцы тщательно.

    Если числовые столбцы не обнаружены, проверьте метаданные столбца, чтобы убедиться, что тип данных столбца является поддерживаемым числовым типом.

    > [!TIP]
    > Чтобы обеспечить наличие столбцов определенного типа в качестве ввода, перед **нормализацией данных**попробуйте использовать выбранные столбцы в модуле [Dataset.](./select-columns-in-dataset.md)

4. **Используйте 0 для постоянных столбцов при проверке:** Выберите эту опцию, когда любой числовой столбец содержит одно неизменное значение. Это гарантирует, что такие столбцы не используются в операциях нормализации.

5. Из списка выпадающих **методов Трансформации** выберите единую математическую функцию для всех выбранных столбцов. 
  
    - **Zscore**: Преобразует все значения в z-score.
    
      Значения в столбце преобразуются по следующей формуле:  
  
      ![нормализация с использованием z&#45;баллов](media/module/aml-normalization-z-score.png)
  
      Среднее и стандартное отклонения вычисляются для каждого столбца отдельно. Используется стандартное отклонение совокупности.
  
    - **MinMax**: Нормализуин мин-макс линейно перемасштабирует каждую функцию с интервалом 0,1.
    
      Масштабирование в интервале [0,1] осуществляется путем сдвига значений каждого компонента таким образом, чтобы минимальное значение было равно 0, а затем деления на новое максимальное значение (которое представляет собой разницу между первоначальными максимальными и минимальными значениями).
      
      Значения в столбце преобразуются по следующей формуле:  
  
      ![нормализация с использованием функции мин&#45;макс](media/module/aml-normalization-minmax.png "AML_normalization-минмакс")  
  
    - **Логистика**: Значения в столбце преобразуются по следующей формуле:

      ![формула для нормализации с помощью логистической функции](media/module/aml-normalization-logistic.png "AML_normalization логистика")  
  
    - **LogNormal**: Этот параметр преобразует все значения в логнормальное масштаб.
  
      Значения в столбце преобразуются по следующей формуле:
  
      ![формула журнала&#45;нормальное распределение](media/module/aml-normalization-lognormal.png "AML_normalization-логнормально")
    
      Здесь - параметры распределения, вычисляемые эмпирически из данных в качестве максимальных оценок вероятности, для каждого столбца по отдельности.  
  
    - **TanH**: Все значения преобразуются в гиперболическую касательную.
    
      Значения в столбце преобразуются по следующей формуле:
    
      ![нормализация с помощью функции гиперболического тангенса](media/module/aml-normalization-tanh.png "AML_normalization-тан")

6. Отправить конвейер или дважды щелкните модуль **нормализации данных** и выберите **выбранный run.** 

## <a name="results"></a>Результаты

Модуль **нормализации данных** генерирует два вывода:

- Чтобы просмотреть преобразованные значения, нажмите вправо на модуль и выберите **Визуализацию.**

    По умолчанию значения преобразуются на месте. Если требуется сравнить преобразованные значения с исходными значениями, используйте модуль [Add Columns](./add-columns.md) для рекомбинации наборов данных и просмотра столбцов бок о бок.

- Чтобы сохранить преобразование, чтобы можно было применить тот же метод нормализации к другому набору данных, выберите модуль и выберите **набор данных Регистра** под вкладкой **Выводы** в правой панели.

    Затем можно загрузить сохраненные преобразования из **группы Преобразований** левой навигационной панели и применить его к набору данных с той же схемой с помощью [Apply Transformation.](apply-transformation.md)

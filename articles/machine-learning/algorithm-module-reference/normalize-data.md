# <a name="normalize-data-module"></a>Нормализация данных модуля

В этой статье описывается модуль визуального интерфейса (Предварительная версия) для службы машинного обучения Azure.

Этот модуль используется для преобразования в объект dataset с помощью *нормализации*.

Нормализация — это метод, который часто применяется в ходе подготовки данных для машинного обучения. Цель нормализации — для изменения значений числовых столбцов в наборе данных для использования общей шкале, без искажение различия в диапазоны значений или потери данных. Нормализация также является обязательным для некоторых алгоритмов для моделирования данных правильно.

Например предположим, что входной набор данных содержит один столбец со значениями от 0 до 1 и другой столбец в диапазоне от 10 000 до 100 000. Содержит огромное разница в *масштабирования* чисел может вызвать проблемы при попытке объединить значения, как компоненты во время моделирования.

*Нормализация* позволяет избежать этих проблем, создав новые значения, которые поддерживают общее распределение и отношения в исходных данных, сохраняя значения в шкалу, применять на всех числовых столбцов, используемых в модели.

Этот модуль предоставляет несколько вариантов для преобразования числовых данных.

- Все значения можно изменить на 0-1 масштабировать или преобразовывать значения, представляя их как ранжирует процентиля, а не абсолютные значения.
- Нормализацию можно применить к одному столбцу или к нескольким столбцам в тот же набор данных.
- Если вам нужно повторять эксперимент, или применить те же шаги нормализации для других данных, вы сохраните в виде нормализации преобразования и применить его к другим наборам данных, имеют одинаковую схему.

> [!WARNING]
> Некоторые алгоритмы требуют, что данные были нормализованы до обучения модели. Другие алгоритмы выполняют свои собственные масштабирование или нормализацию данных. Таким образом при выборе алгоритма для использования при создании прогнозной модели машинного обучения, обязательно ознакомьтесь с требованиями к данным алгоритма перед применением нормализации обучающими данными.

##  <a name="configure-normalize-data"></a>Настройка Нормализация данных

Можно применить только один метод нормализации одновременно с помощью этого модуля. Таким образом один и тот же метод нормализации применяется ко всем столбцам, которые можно выбрать. Чтобы использовать методы разных нормализации, используйте второй экземпляр **Нормализация данных**.

1. Добавить **Нормализация данных** эксперимент модуль. Модуль в службе машинного обучения Azure можно найти в разделе **преобразования данных**в **масштабирование и сокращение** категории.

2. Подключите набор данных, содержащий по крайней мере один столбец всех чисел.

3. Используйте средство выбора столбцов, чтобы выбрать числовые столбцы для нормализации. Если вы не выберете отдельных столбцов, по умолчанию **все** столбцы числового типа во входных данных, которые, и том же процессе нормализации применяется для всех выбранных столбцов. 

    Это может привести к неожиданным результатам, если включить числовых столбцов, которые не должны быть нормализованы! Всегда проверяйте столбцы.

    При обнаружении нет числовых столбцов, проверьте метаданные столбца для проверки является поддерживаемым типом числового типа данных столбца.

    > [!TIP]
    > Чтобы убедиться, что приведены в качестве входных данных столбцы определенного типа, попробуйте использовать [Выбор столбцов в наборе данных](./select-columns-in-dataset.md) модуля перед **Нормализация данных**.

4. **Константа столбцы, если этот флажок установлен, укажите 0**:  Выберите этот параметр, если любой числовой столбец содержит единственное неизменное значение. Это гарантирует, что такие столбцы не используются в операциях нормализации.

5. Из **метода преобразования** раскрывающемся списке выберите один математической функции, чтобы применить все выбранные столбцы. 
  
    - **Zscore**: Преобразует все значения z показателя.
    
      Значения в столбце преобразуются по следующей формуле:  
  
      ![Нормализация с помощью z&#45;оценок](media/module/aml-normalization-z-score.png)
  
      Среднее и стандартное отклонения вычисляются для каждого столбца отдельно. Используется стандартное отклонение совокупности.
  
    - **MinMax**: Минимаксный метод нормализации линейно масштабирует каждый компонент в интервале [0,1].
    
      Масштабирование в интервале [0,1] осуществляется путем сдвига значений каждого компонента, чтобы минимальное значение равно 0, а затем деления на новое максимальное значение (который отличается от исходного максимальными и минимальными значениями).
      
      Значения в столбце преобразуются по следующей формуле:  
  
      ![Нормализация с помощью минимального значения на&#45;Макс. функция](media/module/aml-normalization-minmax.png "AML_normalization minmax")  
  
    - **Логистической**: Значения в столбце преобразуются по следующей формуле:

      ![Формула для нормализации с помощью логистической функции](media/module/aml-normalization-logistic.png "AML_normalization логистической")  
  
    - **Логарифмическое нормальное**: Этот параметр преобразует все значения в логнормальном масштабе.
  
      Значения в столбце преобразуются по следующей формуле:
  
      ![формулы журнала&#45;нормального распределения](media/module/aml-normalization-lognormal.png "логарифмически нормальное AML_normalization")
    
      Здесь μ и σ являются параметрами распределения, вычисляется эмпирически из данных как максимальное оценки для каждого столбца отдельно.  
  
    - **TanH**: Все значения преобразуются в гиперболический тангенс.
    
      Значения в столбце преобразуются по следующей формуле:
    
      ![Нормализация с помощью функции гиперболического тангенса](media/module/aml-normalization-tanh.png "AML_normalization tanh")

6. Запустите эксперимент, или дважды щелкните **Нормализация данных** модуля и выберите **выполнить выбранные**. 

## <a name="results"></a>Результаты

**Нормализация данных** модуль создает два выхода:

- Чтобы просмотреть преобразованные значения, щелкните модуль правой кнопкой мыши **преобразования набора данных**и нажмите кнопку **визуализировать**.

    По умолчанию значения преобразуются на месте. Если вы хотите сравнить преобразованные значения исходные значения, используйте [Добавление столбцов](./add-columns.md) модуль, чтобы перекомпоновать наборы данных и просмотрите столбцы рядом друг с другом.

- Чтобы сохранить преобразование можно применить к другому набору данных аналогично один и тот же метод нормализации, щелкните модуль правой кнопкой мыши **функции преобразования**и нажмите кнопку **Сохранить как преобразование**.

    Теперь можно загрузить сохраненные преобразования из **преобразует** группе на панели навигации слева и применить его к набору данных с той же схемой, с помощью [. / Применение преобразования](apply-transformation.md).

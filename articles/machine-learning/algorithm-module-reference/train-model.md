# <a name="train-model-module"></a>Модуль "Обучение модели"

В этой статье описывается модуль визуального интерфейса (Предварительная версия) для службы машинного обучения Azure.

Этот модуль используется для обучения модели классификации или регрессии. Обучение сразу после определения модели и задать его параметры и требует данных с тегами. Можно также использовать **Обучение модели** для повторного обучения существующей модели новыми данными. 

## <a name="how-the-training-process-works"></a>Как работает в процессе обучения

В машинном обучении Azure Создание и использование модели машинного обучения обычно имеет три этапа. 

1. Для настройки модели, путем выбора определенного типа алгоритма и определения его параметры или гиперпараметров. Выберите один из следующих типов моделей. 

    + **Классификации** моделей, основанных на нейронных сетей, деревья принятия решений и леса принятия решений и другие алгоритмы.
    + **Регрессия** модели, которая может включать стандартный линейной регрессии и с использованием других алгоритмов, включая нейронные сети и Байесовской регрессии.  

2. Предоставляют набор данных, который имеет метку и содержит данные, совместимого с алгоритмом. Подключение данных и модели, чтобы **Обучение модели**.

    Создает какое обучение — это определенные двоичный формат, iLearner, который инкапсулирует шаблонов статистики, полученные из данных. Нельзя напрямую изменять и читать этот формат; Тем не менее эту обученную модель можно использовать другие модули. 
    
    Также можно просмотреть свойства модели. Дополнительные сведения см. в разделе результатов.

3. После завершения обучения, используйте обученную модель с одним из [оценки модули](./score-model.md), для создания прогнозов на основе новых данных.

## <a name="how-to-use-train-model"></a>Как использовать **Обучение модели**  
  
1.  В машинном обучении Azure настройте модель классификации или регрессии.
    
2. Добавить **Обучение модели** эксперимента.  Этот модуль, в разделе можно найти **машинного обучения** категории. Разверните **Train**, а затем перетащите **Обучение модели** модуля в эксперименте.
  
3.  На левой входных данных, необученная режима подключения. Присоединение к правой входные данные из набора данных для обучения **Обучение модели**.

    В обучающем наборе данных может содержать столбец метки. Все строки без меток учитываются.
  
4.  Для **столбца "Метка"**, нажмите кнопку **запустить средство выбора столбцов**и выбрать один столбец, содержащий результаты модели можно использовать для обучения.
  
    - Для проблем классификации, столбца "label" должен содержать **категориальные** значения или **дискретных** значения. Примеры могут включать оценка Да/Нет, код классификацию заболеваний или имя или группу по уровню доходов.  Если вы выберете некатегориального столбца, модуль вернет ошибку во время обучения.
  
    -   Для проблем регрессии столбец меток должен содержать **числовых** данных, представляющий переменную ответа. В идеале числовых данных представляет непрерывной шкале. 
    
    Примерами могут быть оценку кредитного риска, прогнозируемое время на отказ для жесткого диска или прогнозируемое количество звонков в центр обработки вызовов на заданного дня или времени.  Если вы не выберете числового столбца, может появиться ошибка.
  
    -   Если вы не укажете столбец метки, машинное обучение Azure попытается определить подходящий столбец метки, используя метаданные набора данных. Если он выбирает неверный столбец, используйте средство выбора столбцов, чтобы исправить ее.
  
    > [!TIP] 
    > Если вы испытываете проблемы с помощью средства выбора столбцов, см. в статье [Выбор столбцов в наборе данных](./select-columns-in-dataset.md) советы. Он описывает некоторые распространенные сценарии и советы по использованию **ПРАВИЛА с** и **по ИМЕНИ** параметры.
  
5.  Запустите эксперимент. Если у вас есть большой объем данных, это может занять некоторое время.

## <a name="bkmk_results"></a> результаты

Обучение модели:

+ Чтобы просмотреть параметры модели и веса признаков, щелкните правой кнопкой мыши выходные данные и выберите **визуализировать**.
+ Чтобы использовать модель в эксперименты, щелкните модель правой кнопкой мыши и выберите **сохранить модель**. Введите имя для модели. 

    Это модель сохраняется в качестве моментального снимка, который не обновляется путем повторяющихся запусков эксперимента.
+ Чтобы использовать модель прогнозирования новых значений, подключите его к [Score Model](./score-model.md) модуля, а также новые входные данные.

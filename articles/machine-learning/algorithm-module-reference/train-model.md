# <a name="train-model-module"></a>Модуль "Обучение модели"

В этой статье описывается модуль в дизайнере машинного обучения Azure (предварительный просмотр).

Используйте этот модуль для обучения модели классификации или регрессии. Обучение происходит после того, как вы определили модель и установили ее параметры, и требует помеченных данных. Вы также можете использовать **модель Train** для переподготовки существующей модели с новыми данными. 

## <a name="how-the-training-process-works"></a>Как работает тренировочный процесс

В Azure Machine Learning создание и использование модели машинного обучения, как правило, является трехступенчатым процессом. 

1. Вы настраиваете модель, выбирая определенный тип алгоритма и определяя его параметры или гиперпараметры. Выберите любой из следующих типов моделей: 

    + **Модели классификации,** основанные на нейронных сетях, деревьях решений и лесах решений и других алгоритмах.
    + **Регрессионные** модели, которые могут включать стандартную линейную регрессию, или которые используют другие алгоритмы, включая нейронные сети и байесовскую регрессию.  

2. Предоставьте набор данных, который помечен и имеет данные, совместимые с алгоритмом. Подключите как данные, так и модель к **модели train.**

    Обучение производит конкретный двоичный формат, iLearner, который инкапсулирует статистические модели, извлеченные из данных. Вы не можете напрямую изменять или читать этот формат; однако, другие модули могут использовать эту обученную модель. 
    
    Вы также можете просмотреть свойства модели. Для получения дополнительной информации смотрите раздел Результаты.

3. После завершения обучения используйте обученную модель с одним из [модулей скоринга,](./score-model.md)чтобы делать прогнозы на новые данные.

## <a name="how-to-use-train-model"></a>Как использовать модель поезда 
  
1.  В Azure Machine Learning наверстываем модель классификации или регрессии.
    
2. Добавьте модуль **Модели поезда** в конвейер.  Вы можете найти этот модуль в категории **машинного обучения.** Расширьте **поезд,** а затем перетащите модуль **Модели поезда** в конвейер.
  
3.  На левом входе прикрепите неподготовленный режим. Прикрепите набор обучаемых данных к правому ввоза **модели Train Model.**

    Набор обучаемых данных должен содержать столбец метки. Любые строки без меток игнорируются.
  
4.  Для **столбца Label**щелкните **столбец «Оторите»** в правой панели модуля и выберите один столбец, содержащий результаты, которые модель может использовать для обучения.
  
    - Для проблем классификации столбец метки должен содержать либо **категориальные** значения, либо **дискретные** значения. Некоторыми примерами могут быть рейтинг «да/нет», код классификации заболеваний или название или группа доходов.  Если вы выберете некатегоричный столбец, модуль вернет ошибку во время тренировки.
  
    -   Для проблем регрессии столбец метки должен содержать **числовые** данные, представляющие переменную ответа. В идеале числовые данные представляют собой непрерывную шкалу. 
    
    Примерами могут быть оценка кредитного риска, прогнозируемое время сбоя для жесткого диска или прогнозируемое количество звонков в колл-центр в данный день или время.  Если вы не выберете числовой столбец, вы можете получить ошибку.
  
    -   Если не указан используемый столбец метки, машинное обучение Azure попытается определить подходящий столбец метки с помощью метаданных набора данных. Если он выбирает неправильный столбец, используйте селектор столбца, чтобы исправить его.
  
    > [!TIP] 
    > Если у вас возникли проблемы с [Select Columns in Dataset](./select-columns-in-dataset.md) использованием селектора столбцов, см. В нем описаны некоторые общие сценарии и советы по использованию с **РУЛЕИ** и **BY NAME** вариантов.
  
5.  Отправить конвейер. Если у вас есть много данных, это может занять некоторое время.

## <a name="results"></a><a name="bkmk_results"></a>Результаты

После обучения модели:


+ Чтобы использовать модель в других конвейерах, выберите модуль и выберите значок **набора данных Регистра** под вкладкой **Выходов** в правой панели. Вы можете получить доступ к сохраненным моделям в палитре модуля под **наборами данных.**

+ Чтобы использовать модель для прогнозирования новых значений, подключите ее к модулю [Модели оценки](./score-model.md) вместе с новыми входными данными.

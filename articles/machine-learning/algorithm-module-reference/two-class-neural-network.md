# <a name="two-class-neural-network-module"></a>Модуль two-Class нейронной сети

В этой статье описывается модуль визуального интерфейса (Предварительная версия) для службы машинного обучения Azure.

Этот модуль используется для создания модели нейронной сети, которая может использоваться для прогнозирования цели, имеющей только два значения.

Классификация с использованием нейронных сетей — это метод контролируемого обучения, почему и требует *с тегами набора данных*, который включает столбец меток. Например эту модель нейронной сети может использовать для прогнозирования двоичных результатов, таких как ли у пациента определенного заболевания или является ли он компьютером может привести к отказу в пределах указанного периода времени.  

После определения модели, выполнить его обучение, предоставляя маркированного набора данных и модели в качестве входных данных [Обучение модели](./train-model.md). Затем обученную модель можно использовать для прогнозирования значений для новых входных данных.

### <a name="more-about-neural-networks"></a>Дополнительные сведения о нейронных сетей

Нейронная сеть представляет набор взаимосвязанных слоев. Входные данные являются первый слой и подключены к выходной слой с ациклического графа, состоящая из взвешенных ребер и узлов.

Между входного и выходного слоев можно вставить несколько скрытых слоев. Большинство задач прогнозирования можно легко выполнить с только одним или несколькими скрытыми слоями. Тем не менее последние исследования показали, что глубокие нейронные сети (DNN) с помощью множества слоев могут быть эффективны в решении сложных задач, таких как распознавание изображений или речи. Для моделирования возрастающих уровней семантической глубины используются последовательные слои.

Связь между входными и выходными данными может быть получен из обучении нейронной сети на входных данных. Направление графа переходит от входных данных через скрытый слой и выходной слой. Все узлы в слое соединены взвешенными ребрами с узлами следующего слоя.

Чтобы вычислить данные на выходе сети для отдельного входного, значение вычисляется с каждого узла на скрытых слоях и на выходном слое. Значение устанавливается путем вычисления взвешенной суммы значений узлов предыдущего уровня. Функции активации затем применяется к этой взвешенной суммы.
  
## <a name="how-to-configure"></a>Процесс настройки

1.  Добавить **Двухклассовой нейронной сети** эксперимент модуль. Этот модуль, в разделе можно найти **машинного обучения**, **инициализировать**в **классификации** категории.  
  
2.  Укажите способ модели для обучения, задав **Создание режима учителя** параметр.  
  
    -   **Единый параметр**: Выберите этот параметр, если вы уже знаете, как вы хотите настроить модель.  

3.  Для **скрытых слоев спецификации**, выберите тип сетевой архитектуры для создания.  
  
    -   **Полностью подключен случай**: Используются в архитектуре нейронной сети по умолчанию, определенные для двухклассовой нейронной сети следующим образом:
  
        -   Содержит один скрытый слой.
  
        -   Выходной слой полностью подключен к скрытому слою и скрытый слой полностью подключен ко входному слою.
  
        -   Количество узлов во входном слое равно числа признаков в обучающих данных.
  
        -   Пользователь задал на количество узлов в скрытом слое. По умолчанию используется значение 100.
  
        -   Количество узлов равно количеству классов. Двухклассовая Нейронная сеть это означает, что все входные данные должны сопоставляться с одним из двух узлов в выходном слое.

5.  Для **скорость обучения**, определить размер шага, выполненный на каждой итерации, до осуществления коррекции. Увеличьте значение скорости обучения может привести к ускорить Сходимость модели, но возможны Промахи на локальных минимумах.

6.  Для **число итераций обучения**, задать максимальное количество повторов, алгоритм должен обрабатывать обучающих вариантов.

7.  Для **начальных весовых коэффициентов обучения диаметр**, укажите весовые коэффициенты узлов в начале процесса обучения.

8.  Для **импульс**, укажите весовой коэффициент, применяемый во время обучения к узлам из предыдущих итераций  

10. Выберите **случайный порядок примеров** параметр, чтобы в случайном порядке случаях между итерациями. Если вы отключите этот параметр, случаи обрабатываются в точно в том порядке каждый раз при запуске эксперимента.
  
11. Для **начальное значение случайного числа**, введите значение для использования в качестве начального значения.
  
     Указание начального значения полезно при необходимо проверить повторяемость несколькими запусками одного эксперимента.  В противном случае — значение часов системы используется в качестве начального значения, которое может привести к немного другие результаты при каждом запуске эксперимента.
  
13. Добавьте маркированного набора данных в эксперимент и соедините один из [обучающих модулей](module-reference.md).  
  
    -   Если задать **Создание режима учителя** для **один параметр**, использовать [Обучение модели](train-model.md) модуля.  
  
14. Запустите эксперимент.

## <a name="results"></a>Результаты

После завершения обучения:

+ Чтобы просмотреть сводку параметров модели, а также функцию весовые коэффициенты, узнали из обучения и другие параметры нейронной сети, щелкните правой кнопкой мыши выходные данные [Обучение модели](./train-model.md)и выберите **визуализировать**.  

+ Чтобы сохранить моментальный снимок обученной модели, щелкните правой кнопкой мыши **модели Trained** выходных данных и выберите **Сохранить как Обученную модель**. Эта модель не обновляется при последующих запусках одного эксперимента.

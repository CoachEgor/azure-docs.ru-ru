# <a name="two-class-neural-network-module"></a>Двухклассный модуль нейронной сети

В этой статье описывается модуль в дизайнере машинного обучения Azure (предварительный просмотр).

Этот модуль применяется для создания модели нейронной сети, которая может использоваться для прогнозирования цели, имеющей только два значения.

Классификация с использованием нейронных сетей является контролируемым методом обучения, и поэтому требует *тегами набор данных*, который включает столбец метки. Например, можно использовать эту модель нейронной сети для прогнозирования бинарных исходов, таких как, есть ли у пациента определенное заболевание, или же машина может выйти из строя в течение определенного периода времени.  

После определения модели обучите ее, предоставив помеченный набор данных и модель в качестве ввода модели [train Model.](./train-model.md) Затем обученная модель может использоваться для прогнозирования значений новых входов.

### <a name="more-about-neural-networks"></a>Подробнее о нейронных сетях

Нейронная сеть представляет собой набор взаимосвязанных слоев. Входные данные являются первым слоем и соединены с выходным слоем ациклическим графиком, состоящим из взвешенных краев и узлов.

Между входной и выходной слоями можно вставить несколько скрытых слоев. Большинство задач прогнозирования можно легко выполнить только с одним или несколькими скрытыми слоями. Однако недавние исследования показали, что глубокие нейронные сети (DNN) со многими слоями могут быть эффективными в сложных задачах, таких как распознавание изображений или речи. Последовательные слои используются для моделирования возрастающих уровней семантической глубины.

Взаимосвязь между входными данными и выходами изучена из обучения нейронной сети входным данным. Направление графика происходит от входов через скрытый слой и к выходу слоя. Все узлы в слое соединены взвешенными краями с узлами следующего слоя.

Для вычисления вывода сети для конкретного ввода значение рассчитывается на каждом узло в скрытых слоях и в выходном слое. Значение устанавливается путем расчета средневзвешенной суммы значений узлов из предыдущего слоя. Функция активации затем применяется к этой взвешенной сумме.
  
## <a name="how-to-configure"></a>Порядок настройки

1.  Добавьте **двухклассный** модуль нейронной сети в конвейер. Вы можете найти этот модуль под **машинного обучения**, **Инициализация**, в категории **классификации.**  
  
2.  Укажите, как нужно обучить модель, установив опцию **«Создать тренер».**  
  
    -   **Единый параметр**: Выберите этот параметр, если вы уже знаете, как настроить модель.

    -   **Диапазон параметров**: Если вы не уверены в лучших параметрах, вы можете найти оптимальные параметры с помощью модуля [Tune Model Hyperparameters.](tune-model-hyperparameters.md) Вы предоставляете некоторый диапазон значений, и тренер итерирует несколько комбинаций параметров, чтобы определить комбинацию значений, которая дает наилучший результат.  

3.  Для **спецификации скрытого слоя**выберите тип сетевой архитектуры для создания.  
  
    -   **Полностью подключенный случай**: Использует архитектуру нейронной сети по умолчанию, определяемую для двухклассных нейронных сетей следующим образом:
  
        -   Имеет один скрытый слой.
  
        -   Выходной слой полностью подключен к скрытому слою, а скрытый слой полностью подключен к вхотвореному.
  
        -   Количество узлов в вхотозавном слое равно числу функций в обучающих данных.
  
        -   Количество узлов в скрытом слое устанавливается пользователем. По умолчанию используется значение 100.
  
        -   Количество узлов равно числу классов. Для двухклассной нейронной сети это означает, что все входные данные должны отображаться на одном из двух узлов в выходном слое.

5.  Для **скорости обучения**определите размер шага, сделанного при каждой итерации, до коррекции. Более высокое значение скорости обучения может ускорить сходимость модели, однако возможны промахи на локальных минимумах.

6.  Для **количества итераций обучения**укажите максимальное количество раз, когда алгоритм должен обрабатывать обучающие случаи.

7.  Для **начального диаметра обучения**укажите вес узла в начале учебного процесса.

8.  Для **импульса,** укажите вес, чтобы применить во время обучения к узлам из предыдущих итераций  

10. Выберите опцию **Shuffle examples** для перетасовки случаев между итерациями. При невыборе этой опции случаи обрабатываются в точно таком же порядке при запуске конвейера.
  
11. Для **семян случайного числа,** введите значение для использования в качестве семян.
  
     Указать значение семян полезно, когда требуется обеспечить повторяемость в ходе одного и того же конвейера.  В противном случае значение системных часов используется в качестве семени, что может привести к несколько иным результатам при запуске конвейера.
  
13. Добавьте в конвейер помеченный набор данных и подключите один из [учебных модулей.](module-reference.md)  
  
    -   При установке **режима создания тренажера** на **единый параметр**используйте модуль [Модели поезда.](train-model.md)  
  
14. Отправить конвейер.

## <a name="results"></a>Результаты

После завершения обучения:

+ Чтобы сохранить моментальный снимок обученной модели, выберите вкладку **Выводы** в правой панели **модуля модели Train.** Выберите значок **набора данных Регистра,** чтобы сохранить модель в качестве многоразового модуля.

+ Чтобы использовать модель для скоринга, добавьте модуль **Модели оценки** в конвейер.

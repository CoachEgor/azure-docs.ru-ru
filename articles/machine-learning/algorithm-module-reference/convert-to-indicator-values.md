# <a name="convert-to-indicator-values"></a>Преобразование в значения индикатора
В этой статье описывается модуль дизайнера машинного обучения Azure.

Используйте модуль **Convert to Indicator Values** в конструкторе Машинного Обучения Azure для преобразования столбцов, содержащих категоричные значения, в ряд столбцов бинарных индикаторов.  

Этот модуль также выводит определение преобразования, используемого для преобразования в значения индикатора. Это преобразование можно повторно использовать на других наборах данных, которые имеют ту же схему, с помощью модуля [Apply Transformation.](apply-transformation.md)

## <a name="how-to-configure-convert-to-indicator-values"></a>Как настроить Преобразование в значения индикаторов

1.  Найдите **значение Convert to Indicator и** перетащите его в проект конвейера. Этот модуль можно найти в категории **«Трансформация данных».**
    > [!NOTE]
    > Модуль [Edit Metadata](edit-metadata.md) перед модулем **Convert to Indiciator Values** можно использовать для обозначения целевого столбца как категоричный.

1. Подключите модуль **«Преобразование» к** индикаторным значениям к набору данных, содержащему столбцы, которые необходимо преобразовать. 

1. Выберите **столбец «Отображим»** для выбора одной или нескольких категориальных столбцов.

1. Выберите опцию **категориальных столбцов Overwrite,** если вы хотите вывести **только** новые столбцы Boolean. По умолчанию этот параметр выключен.
    

    > [!TIP]
    >  Если вы выберете вариант перезаписи, исходная колонка фактически не удаляется или изменяется. Вместо этого новые столбцы генерируются и представляются в наборе данных вывода, а исходный столбец остается доступным в рабочем пространстве. Если вам нужно увидеть исходные данные, вы можете использовать модуль [Добавления столбцов](add-columns.md) в любое время, чтобы добавить исходную колонку обратно.

1. Отправить конвейер.

## <a name="results"></a>Результаты

Предположим, у вас есть столбец с баллами, указывающими на высокую, среднюю или низкую вероятность сбоя сервера.  

| Идентификатор сервера | Счет неудачи |
| --------- | ------------- |
| 10301     | Низкий           |
| 10302     | Средний        |
| 10303     | Высокий          |

При применении **Значение показателя**convert переходит в одну колонку меток в несколько столбцов, содержащих значения Boolean:  

| Идентификатор сервера | Счет неудачи - Низкий | Счет неудачи - Средний | Счет неудачи - Высокий |
| --------- | ------------------- | ---------------------- | -------------------- |
| 10301     | 1                   | 0                      | 0                    |
| 10302     | 0                   | 1                      | 0                    |
| 10303     | 0                   | 0                      | 1                    |

Вот как работает преобразование:  

-   В столбце **оценки «Отказ»,** описывающей риск, есть только три возможных значения (высокие, средние и низкие) и отсутствующие значения. Таким образом, создаются ровно три новых столбца.  

-   Новые столбцы индикаторов названы на основе заголовков столбцов и значений исходной колонки, используя этот шаблон: * \<столбец источника>- \<значение данных>. *  

-   В одном столбце индикатора должно быть 1 и 0 во всех других столбцах индикаторов, так как каждый сервер может иметь только один рейтинг риска.  

Теперь в модели машинного обучения можно использовать три столбца индикатора в качестве функций.

Модуль возвращает два выхода:

- **Набор данных результатов**: Набор данных с преобразуемыми столбиками значений индикатора. Колонны, не выбранные для очистки, также "проходят".
- **Преобразование значений индикатора:** преобразование данных, используемое для преобразования в значения индикатора, которое может быть сохранено в рабочем пространстве и применено к новым данным позже.

## <a name="apply-a-saved-indicator-values-operation-to-new-data"></a>Применение операции значений сохраненных индикаторов к новым данным

Если вам нужно часто повторять операции значений индикатора, можно сохранить шаги манипуляции с данными в качестве *преобразования* для повторного использования с тем же набором данных. Это полезно, если необходимо часто реимпортировать, а затем очистить данные, которые имеют ту же схему.

1. Добавьте модуль [«Применить трансформацию»](apply-transformation.md) в конвейер.

1. Добавьте набор данных, который необходимо очистить, и подключите набор данных к правому вхотворного порту.

1. Расширьте группу **преобразования данных** в левом стеле конструктора. Найдите сохраненное преобразование и перетащите ее в конвейер.

1. Подключите сохраненную трансформацию к левому вхотвому порту [Apply Transformation.](apply-transformation.md)

   При применении сохраненного преобразования невозможно выбрать, какие столбцы преобразуются. Это происходит потому, что преобразование определено и автоматически применяется к типам данных, указанным в исходной операции.

1. Отправить конвейер.
 
## <a name="technical-notes"></a>Технические примечания  

Этот раздел содержит детали реализации, советы и ответы на часто задаваемые вопросы.

### <a name="usage-tips"></a>Советы по использованию

-   Преобразовать в столбцы индикаторов можно только столбцы, помеченные как категориальные. Если вы видите следующую ошибку, вполне вероятно, что один из выбранных столбцов не является категоричным:  

     Ошибка 0056: Колонка с именем столбца> \<не находится в разрешенной категории.  

     По умолчанию большинство строковых столбцов обрабатываются как функции строки, поэтому их следует четко отметить как категоричные с помощью [Edit Metadata.](edit-metadata.md)  

-   Количество столбцов, которые можно преобразовать в столбцы индикаторов, не ограничено. Однако, поскольку каждый столбец значений может дать несколько столбцов индикаторов, может потребоваться преобразовать и просмотреть сразу несколько столбцов одновременно.  

-   Если столбец содержит недостающие значения, создается отдельный столбец индикатора для отсутствующих категорий с этим именем: * \<столбец источника>-Missing*  

-   Если столбец, конвертируемый в значения индикатора, содержит числа, они должны быть помечены как категоричные, как и любой другой столбец объекта. После того как вы сделали это, цифры рассматриваются как дискретные значения. Например, если у вас есть числовой столбец со значениями MPG от 25 до 30, для каждого дискретного значения будет создан новый столбец индикатора:  

    | Убедитесь,       | Шоссе mpg -25 | Шоссе mpg -26 | Шоссе mpg -27 | Шоссе mpg -28 | Шоссе mpg -29 | Шоссе mpg -30 |
    | ---------- | --------------- | --------------- | --------------- | --------------- | --------------- | --------------- |
    | Контосо Автомобили | 0               | 0               | 0               | 0               | 0               | 1               |

- Чтобы избежать добавления слишком большого количества измерений в набор данных. Мы рекомендуем сначала проверить количество значений в столбце, а также бен или количественно данные надлежащим образом.

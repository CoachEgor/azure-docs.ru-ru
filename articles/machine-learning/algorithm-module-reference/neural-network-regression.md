# <a name="neural-network-regression-module"></a>Модуль регрессии нейронных сетей

*Создает модели регрессии с помощью алгоритма нейронной сети*  
  
 Категория: Машинное обучение / Инициализация модели / Регрессия
  
## <a name="module-overview"></a>Обзор модуля  

В этой статье описывается модуль в дизайнере машинного обучения Azure (предварительный просмотр).

Используйте этот модуль для создания регрессионной модели с помощью настраиваемого алгоритма нейронной сети.
  
 Несмотря на то, что нейронные сети широко используются для углубленного обучения и моделирования сложных задач, таких как распознавание изображений, они легко адаптируются к задачам регрессии. Любой класс статистических моделей можно назвать нейронной сетью, если эти модели используют адаптивные весовые коэффициенты и могут использоваться для аппроксимации нелинейных функций входных данных. Таким образом, регрессия нейронной сети подходит для задач, которые нельзя решить с помощью более традиционных моделей.
  
 Регрессия нейронной сети является контролируемым методом обучения, и поэтому требует *тегами набора данных,* который включает столбец метки. Так как модель регрессии прогнозирует числовое значение, столбец меток должен принадлежать к числовому типу данных.  
  
 Вы можете обучить модель, предоставив модель и помеченный набор данных в качестве ввода модели [train Model.](./train-model.md) Затем обученная модель может использоваться для прогнозирования значений для новых примеров ввода.  
  
## <a name="configure-neural-network-regression"></a>Настройка регрессии нейронной сети 

Нейронные сети могут быть тщательно настроены. В этом разделе описывается, как создать модель с помощью двух методов:
  
+ [Создание модели нейронной сети с помощью архитектуры по умолчанию](#bkmk_DefaultArchitecture)  
  
    Если вы принимаете архитектуру нейронной сети по умолчанию, используйте панель **Свойств** для набора параметров, контролирующих поведение нейронной сети, таких как количество узлов в скрытом слое, скорость обучения и нормализация.

    Начните здесь, если вы новичок в нейронных сетях. Модуль поддерживает множество настроек, а также настройки модели, без глубоких знаний нейронных сетей. 

+ Определение пользовательской архитектуры для нейронной сети 

    Используйте эту опцию, если вы хотите добавить дополнительные скрытые слои или полностью настроить архитектуру сети, ее соединения и функции активации.
    
    Этот вариант лучше всего, если вы уже несколько знакомы с нейронными сетями. Для определения сетевой архитектуры используется язык Net.  

##  <a name="create-a-neural-network-model-using-the-default-architecture"></a><a name="bkmk_DefaultArchitecture"></a>Создание модели нейронной сети с помощью архитектуры по умолчанию

1.  Добавьте модуль **регрессии нейронной сети** в конвейер в проектировщике. Вы можете найти этот модуль под **машинного обучения**, **Инициализация**, в категории **Регрессия.** 
  
2. Укажите, как вы хотите, чтобы модель была обучена, установив опцию **режим создания.**  
  
    -   **Единый параметр**: Выберите этот параметр, если вы уже знаете, как настроить модель.

    -   **Диапазон параметров**: Выберите этот параметр, если вы не уверены в лучших параметрах, и хотите запустить развертки параметров. Выберите диапазон значений для итерации, и [Hyperparameters Tune Model](tune-model-hyperparameters.md) итерирует все возможные комбинации параметров, которые вы предоставили, чтобы определить гиперпараметры, которые дают оптимальные результаты.   

3.  В **спецификации скрытого слоя**выберите **полностью подключенный корпус.** Эта опция создает модель с использованием архитектуры нейронной сети по умолчанию, которая для модели регрессии нейронной сети имеет следующие атрибуты:  
  
    + Сеть имеет ровно один скрытый слой.
    + Выходной слой полностью связан со скрытым слоем, который, в свою очередь, полностью связан с входным слоем.
    + Количество узлов в скрытом слое может быть задано пользователем (значение по умолчанию — 100).  
  
    Поскольку количество узлов в входном слое определяется количеством функций в обучающих данных, в модели регрессии может быть только один узло в выходном слое.  
  
4. Для **количества скрытых узлов**введите количество скрытых узлов. По умолчанию это один скрытый слой с 100 узлами. (Эта опция недоступна, если вы определяете пользовательскую архитектуру с помощью Сети.)
  
5.  Для **скорости обучения**введите значение, определяющее шаг, сделанный при каждой итерации, до коррекции. Более высокое значение скорости обучения может ускорить сходимость модели, однако возможны промахи на локальных минимумах.

6.  Для **количества обучающих итераций**укажите максимальное количество раз, когда алгоритм обрабатывает обучающие случаи.


8.  Для **импульса,** введите значение, чтобы применить во время обучения в качестве веса на узлах от предыдущих итераций.

10. Выберите опцию, **Примеры Shuffle,** чтобы изменить порядок дел между итерациями. При невыборе этой опции случаи обрабатываются в точно таком же порядке при запуске конвейера.
  
11. Для **семян случайного числа**можно дополнительно ввести значение для использования в качестве семени. Указать значение семян полезно, когда требуется обеспечить повторяемость в ходе одного и того же конвейера.
  
13. Подключите набор обучаемых данных и один из [учебных модулей:](module-reference.md) 
  
    -   Если вы установите **режим создания тренера** для single **Parameter,** используйте [модель train.](./train-model.md)  
  
   
14. Отправить конвейер.  

## <a name="results"></a>Результаты

После завершения обучения:

- Чтобы сохранить моментальный снимок обученной модели, выберите вкладку **Выводы** в правой панели **модуля модели Train.** Выберите значок **набора данных Регистра,** чтобы сохранить модель в качестве многоразового модуля.

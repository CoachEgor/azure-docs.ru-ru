# <a name="apply-sql-transformation"></a>Применить преобразование SQL

В этой статье описывается модуль Машинное обучение Azure конструктора (Предварительная версия).

С помощью модуля Apply SQL преобразование можно:
  
-   Создайте таблицы для результатов и сохраните их в переносимой базе данных.  
  
-   Выполните пользовательские преобразования типов данных или создайте статистические выражения.  
  
-   Выполнение инструкций SQL-запросов для фильтрации или изменения данных и возврата результатов запроса в виде таблицы данных.  

> [!IMPORTANT]
> Ядро SQL, используемое в этом модуле, — **SQLite**. Дополнительные сведения о синтаксисе SQLite см. в разделе [SQL как понятное для SQLite](https://www.sqlite.org/index.html) .  

## <a name="how-to-configure-apply-sql-transformation"></a>Настройка преобразования «применение SQL»  

Модуль может принимать до трех наборов данных в качестве входных данных. При ссылке на наборы данных, подключенные к каждому порту ввода, необходимо использовать имена `t1`, `t2`и `t3`. Номер таблицы указывает индекс входного порта.  
  
Оставшийся параметр — SQL-запрос, использующий синтаксис SQLite. При вводе нескольких строк в текстовом поле **скрипт SQL** используйте точку с запятой для завершения каждой инструкции. В противном случае разрывы строк преобразуются в пробелы.  

Этот модуль поддерживает все стандартные инструкции в синтаксисе SQLite. Список неподдерживаемых инструкций см. в разделе [Технические примечания](#technical-notes) .

##  <a name="technical-notes"></a>Технические примечания  

В этом разделе содержатся сведения о реализации, советы и ответы на часто задаваемые вопросы.

-   Входные данные всегда требуются для порта 1.  
  
-   Для идентификаторов столбцов, содержащих пробелы или другие специальные символы, всегда заключите идентификатор столбца в квадратные или двойные кавычки при ссылке на столбец в предложениях `SELECT` или `WHERE`.  
  
### <a name="unsupported-statements"></a>Неподдерживаемые инструкции  

Хотя SQLite поддерживает большую часть стандарта ANSI SQL, он не включает множество функций, поддерживаемых коммерческими системами реляционных баз данных. Дополнительные сведения см. в разделе [SQL как понятное для SQLite](http://www.sqlite.org/lang.html). Кроме того, при создании инструкций SQL учитывайте следующие ограничения.  
  
- SQLite использует динамическую типизацию значений, вместо того чтобы назначать тип столбцу в большинстве систем реляционных баз данных. Он слабо типизирован и допускает неявное преобразование типов.  
  
- `LEFT OUTER JOIN` реализован, но не `RIGHT OUTER JOIN` или `FULL OUTER JOIN`.  

- Операторы `RENAME TABLE` и `ADD COLUMN` можно использовать с командой `ALTER TABLE`, но другие предложения не поддерживаются, включая `DROP COLUMN`, `ALTER COLUMN`и `ADD CONSTRAINT`.  
  
- Можно создать представление в SQLite, но после этого представления будут доступны только для чтения. В представлении нельзя выполнять инструкцию `DELETE`, `INSERT`или `UPDATE`. Однако можно создать триггер, который срабатывает при попытке `DELETE`, `INSERT`или `UPDATE` представления и выполнения других операций в теле триггера.  
  

Помимо списка неподдерживаемых функций, предоставляемых на официальном веб-сайте SQLite, следующий вики-сайт предоставляет список других неподдерживаемых функций: [SQLite — неподдерживаемый SQL](http://www2.sqlite.org/cvstrac/wiki?p=UnsupportedSql)

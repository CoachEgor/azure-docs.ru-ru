# <a name="apply-sql-transformation"></a>Применение преобразования SQL

В этой статье описывается модуль дизайнера машинного обучения Azure (предварительный просмотр).

Используя модуль преобразования прикладного S'L, вы можете:
  
-   создать таблицу для результатов, а затем сохранить наборы данных в переносимой базе данных;  
  
-   выполнять пользовательские преобразования типов данных или создавать статистические выражения;  
  
-   выполнять инструкции SQL-запроса для фильтрации или изменения данных и получения результатов запроса в виде таблицы данных.  

> [!IMPORTANT]
> Движок S'L, используемый в этом модуле, является **S'Lite.** Для получения более подробной информации [SQL as Understood by SQLite](https://www.sqlite.org/index.html) о синтаксисе s'Lite, для получения дополнительной информации, см.  

## <a name="how-to-configure-apply-sql-transformation"></a>Как настроить применение трансформации S'L  

В модуле может содержаться до трех наборов в качестве входных данных. При ссылке на наборы данных, подключенных к `t1` `t2`каждому `t3`порту ввода, необходимо использовать имена и: Номер таблицы соответствует индексу входного порта.  
  
Оставшийся параметр — SQL-запрос, в котором используется синтаксис SQLite. При вводе нескольких строк в текстовом окне **скриптом,** используйте полуколонну, чтобы завершить каждую выписку. В противном случае разрывы строки преобразуются в пробелы.  

Этот модуль поддерживает все стандартные операторы синтаксиса SQLite. Список неподдерживаемых заявлений можно узнать в разделе [Технические примечания.](#technical-notes)

##  <a name="technical-notes"></a>Технические примечания  

Этот раздел содержит детали реализации, советы и ответы на часто задаваемые вопросы.

-   Ввод всегда требуется в порту 1.  
  
-   Для идентификаторов столбцов, содержащих пробел или другие специальные символы, всегда прикладывайте идентификатор столбца в квадратные скобки или двойные кавычки, когда речь идет о столбце в `SELECT` или `WHERE` положениях.  
  
### <a name="unsupported-statements"></a>Неподдерживаемые заявления  

Хотя SQLite поддерживает большую часть стандарта ANSI SQL, в него не включены несколько функций, поддерживаемых коммерческими системами реляционных баз данных. Для получения более подробной информации, [см.](http://www.sqlite.org/lang.html) Кроме того, следует знать о следующих ограничениях при создании инструкций по S'L:  
  
- SQLite использует динамическую типизацию значений, а не присвоение типа столбцу, как в большинстве систем реляционных баз данных. Кроме того, для нее характерна слабая типизация. Система также позволяет выполнять неявное преобразование типов.  
  
- `LEFT OUTER JOIN`реализуется, но `RIGHT OUTER JOIN` `FULL OUTER JOIN`нет или .  

- Можно использовать инструкции `RENAME TABLE` и `ADD COLUMN` с командой `ALTER TABLE`, но другие предложения не поддерживаются, в том числе `DROP COLUMN`, `ALTER COLUMN` и `ADD CONSTRAINT`.  
  
- Вы можете создать VIEW в SQLite, но последующие представления будут доступны только для чтения. Невозможно выполнить инструкцию `DELETE`, `INSERT` или `UPDATE` для представления. Но вы можете создать триггер, срабатывающий при попытке выполнить инструкцию `DELETE`, `INSERT` или `UPDATE` для представления, и выполнить другие операции в теле триггера.  
  

В дополнение к списку неподдерживаемых функций, представленных на официальном сайте S'Lite, в следующем вики приводится список других неподдерживаемых функций: [S'Lite - Неподдерживаемые S'L](http://www2.sqlite.org/cvstrac/wiki?p=UnsupportedSql)

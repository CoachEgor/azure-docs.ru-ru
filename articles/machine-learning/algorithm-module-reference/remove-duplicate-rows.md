# <a name="remove-duplicate-rows-module"></a>Удалить модуль дубликатов строк

В этой статье описывается модуль в дизайнере машинного обучения Azure (предварительный просмотр).

Используйте этот модуль для удаления потенциальных дубликатов из набора данных.

Например, предположим, что ваши данные выглядят следующим образом и представляют несколько записей для пациентов. 

| Пациенти | Initials| пол;|Возраст|Признался|
|----|----|----|----|----|
|1|Ф.М.| M| 53| Январь|
|2| Ф.А.М.| M| 53| Январь|
|3| Ф.А.М.| M| 24| Январь|
|3| Ф.М.| M| 24| Февраль|
|4| Ф.М.| M| 23| Февраль|
| | Ф.М.| M| 23| |
|5| Ф.А.М.| M| 53| |
|6| Ф.А.М.| M| NaN| |
|7| Ф.А.М.| M| NaN| |

Очевидно, что этот пример имеет несколько столбцов с потенциально дублирующими данными. Являются ли они на самом деле дубликаты зависит от вашего знания данных. 

+ Например, вы можете знать, что многие пациенты имеют одно и то же имя. Вы не устраните дубликаты, используя любые столбцы имен, только столбец **идентификатора.** Таким образом, отфильтровываются только строки с дублирующими значениями идентификации, независимо от того, имеют пациенты одно и то же имя или нет.

+ Кроме того, можно разрешить дубликаты в поле идентификаторов и использовать некоторые другие комбинации файлов для поиска уникальных записей, таких как имя, фамилия, возраст и пол.  

Чтобы установить критерии дублирования строки или нет, указано один столбец или набор столбцов для использования в качестве **ключей.** Две строки считаются дубликатами только тогда, когда значения во **всех** ключевых столбцах равны. Если какая-либо строка имеет отсутствующее значение для **ключей,** они не будут считаться дублирующими строками. Например, если гендер и возраст устанавливаются как ключи в таблице выше, строки 6 и 7 не являются дублирующими строками, учитывая, что они имеют отсутствующее значение в Age.

При запуске модуля он создает набор данных кандидата и возвращает набор строк, которые не имеют дубликатов в указанном наборе столбцов.

> [!IMPORTANT]
> Набор исходных данных не изменяется; этот модуль создает новый набор данных, который фильтруется, чтобы исключить дубликаты, на основе критериев, которые вы указываете.

## <a name="how-to-use-remove-duplicate-rows"></a>Как использовать Удалить дублирующие строки

1. Добавьте модуль в конвейер. Вы можете найти модуль **Удалить дублирующие строки** под **преобразованием данных,** **манипуляции**.  

2. Подключите набор данных, который необходимо проверить для дубликатов строк.

3. В панели **Свойств,** под **выражением фильтра выбора столбца ключа,** нажмите **селектор колонки запуска,** чтобы выбрать столбцы для использования в определении дубликатов.

    В этом контексте **ключ** не означает уникальный идентификатор. Все столбцы, выбранные с помощью селектора столбцов, обозначены как **ключевые столбцы.** Все невыбранные столбцы считаются неключевыми столбцов. Сочетание столбцов, которые вы выбираете в качестве ключей, определяет уникальность записей. (Подумайте об этом как о заявлении, которое использует несколько присоединяется к равенству.)

    Примеры:

    + "Я хочу убедиться, что идентификаторы уникальны": Выберите только колонку ID.
    + "Я хочу убедиться, что сочетание имени, фамилии и идентификатора является уникальным": Выберите все три столбца.

4. Используйте **первый дубликат флажка Retain,** чтобы указать, какую строку вернуть при обнаружении дубликатов:

    + При выборе первая строка возвращается, а другие отбрасываются. 
    + Если вы отопроводите эту опцию, последний дубликат строки сохраняется в результатах, а другие отбрасываются. 

5. Отправить конвейер.

6. Чтобы просмотреть результаты, нажмите на модуль и выберите **Визуализацию.** 

> [!TIP]
> Если результаты трудно понять, или если вы хотите исключить некоторые столбцы из рассмотрения, можно удалить столбцы с помощью [выбранных столбцов в](./select-columns-in-dataset.md) модуле Набора данных.

# <a name="remove-duplicate-rows-module"></a>Снятие модуля повторяющихся строк

В этой статье описывается модуль визуального интерфейса (Предварительная версия) для службы машинного обучения Azure.

Этот модуль используется для удаления возможные дубликаты из набора данных.

Предположим, например, данных выглядит следующим образом и представляет несколько записей для пациентов. 

| PatientID | Initials| пол;|Age|Это означает, что|
|----|----|----|----|----|
|1|F.M.| M| 53| Янв|
|2| F.A.M.| M| 53| Янв|
|3| F.A.M.| M| 24| Янв|
|3| F.M.| M| 24| Feb|
|4\.| F.M.| M| 23| Feb|
| | F.M.| M| 23| |
|5| F.A.M.| M| 53| |
|6| F.A.M.| M| NaN| |
|7| F.A.M.| M| NaN| |

Очевидно, что в этом примере имеется несколько столбцов с потенциально повторяющиеся данные. Являются ли они фактически дубликаты зависит от знания о данных. 

+ Например вы знаете, что много пациентов имеют одинаковое имя. Разве удаления повторяющихся элементов, используя любое имя, только столбцы **идентификатор** столбца. Таким образом, только строки с повторяющимися значениями идентификатора, отфильтровываются, независимо от наличия у пациентов тем же именем или нет.

+ Кроме того можно разрешить дубликаты в поле идентификатора и использовать любые другие комбинации файлов для поиска уникальных записей, таких как имя, последнее имя, возраст и пол.  

Чтобы задать критерии для ли повторяющиеся строки или нет, необходимо указать один столбец или набор столбцов для использования в качестве **ключи**. Две строки считаются повторяющимися, только если значения в **все** ключевых столбцов равны. Если отсутствует значение для любой строки **ключи**, они не будет учитываться повторяющиеся строки. Например пол и возраст задаются, как ключи в над таблицей строки 6 и 7 не учитывая повторяющиеся строки после отсутствующее значение в возраст.

При запуске модуля, он создает предлагаемый набор данных и возвращает набор строк, которые не должны содержать повторяющихся по всему набору столбцов, которые вы указали.

> [!IMPORTANT]
> Исходный набор данных не изменяется; Этот модуль создает новый набор данных, который фильтруется, чтобы исключить повторяющиеся значения, в зависимости от заданных критериев.

## <a name="how-to-use-remove-duplicate-rows"></a>Как использовать удаление повторяющихся строк

1. Добавьте модуль в свой эксперимент. Можно найти **удаление повторяющихся строк** модуля **преобразования данных**, **манипуляции**.  

2. Подключите набор данных, который вы хотите проверить повторяющиеся строки.

3. В **свойства** панели в разделе **ключа выражение фильтра для выбора столбца**, нажмите кнопку **запустить средство выбора столбцов**, чтобы выбрать столбцы, используемые для обнаружения дубликатов.

    В этом контексте **ключ** не означает уникальный идентификатор. Все столбцы, выбранные с помощью средства выбора столбцов определяются как **ключевые столбцы**. Все невыбранные столбцы считаются неключевыми столбцами. Сочетание столбцов, которые выбраны в качестве ключей определяет уникальность записей. (Рассматривать как инструкция SQL, которая использует несколько соединений равенства.)

    Примеры:

    + «Я хочу убедиться, что идентификаторы являются уникальными:» Выберите только столбец идентификатора.
    + «Я хочу гарантировать уникальность сочетание имени, фамилии и идентификатор:» Выберите все три столбца.

4. Используйте **сохранить первую из повторяющихся строк** флажок, чтобы указать, какие строки должны быть возвращены при обнаружении дубликатов:

    + Если выбрано, возвращается первая строка и другие удаляются. 
    + Если этот флажок снят, последняя из повторяющихся строк хранятся в результатах и другие не учитываются. 

5. Запустите эксперимент.

6. Чтобы просмотреть результаты, щелкните модуль правой кнопкой мыши **результирующий набор данных**и нажмите кнопку **визуализировать**. 

> [!TIP]
> Если результаты сложны для понимания, или если вы хотите исключить некоторые столбцы из рассмотрения, можно удалить столбцы с помощью [Выбор столбцов в наборе данных](./select-columns-in-dataset.md) модуля.

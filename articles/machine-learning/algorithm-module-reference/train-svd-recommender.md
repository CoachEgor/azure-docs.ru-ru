# <a name="train-svd-recommender"></a>Обучение рекомендателя SVD

В этой статье описывается, как использовать модуль Train SVD Recommender в дизайнере машинного обучения Azure (предварительный просмотр). Используйте этот модуль для обучения рекомендательной модели на основе алгоритма разложения единого значения (SVD).  

Модуль Train SVD Recommender считывает набор данных о трехсетовых рейтингах пользователей. Он возвращает обученный рекомендатель SVD. Затем можно использовать обученную модель для прогнозирования оценок или создания рекомендаций, используя модуль [Рекомендатора Score SVD.](score-svd-recommender.md)  


  
## <a name="more-about-recommendation-models-and-the-svd-recommender"></a>Подробнее о моделях рекомендаций и рекомендации SVD  

Основная цель системы рекомендаций заключается в том, чтобы рекомендовать один или несколько *элементов* *пользователям* системы. Примерами элемента могут быть фильм, ресторан, книга или песня. Пользователем может быть лицо, группа лиц или другая организация с предпочтениями элемента.  

Существует два основных подхода к системам рекомендательных рекомендаций: 

+ Подход, **основанный на контенте,** использует функции как для пользователей, так и для элементов. Пользователей можно описать по таким свойствам, как возраст и пол. Элементы могут быть описаны по таким свойствам, как автор и производитель. Вы можете найти типичные примеры систем рекомендаций на основе контента на сайтах социальных сватовства. 
+ **Совместная фильтрация** использует только идентификаторы пользователей и элементов. Он получает неявную информацию об этих сущностях из (разреженной) матрицы оценок, присвоенных пользователями, к элементам. Мы можем узнать о пользователе из элементов, которые они оценили, и от других пользователей, которые оценили те же элементы.  

Рекомендатор SVD использует идентификаторы пользователей и элементов, а также матрицу оценок, приведенных пользователями к элементам. Это *совместный рекомендатель*. 

Для получения дополнительной информации о рекомендации SVD, см. [Matrix factorization techniques for recommender systems](https://datajobs.com/data-science-repo/Recommender-Systems-[Netflix].pdf)


## <a name="how-to-configure-train-svd-recommender"></a>Как настроить поезд SVD Рекомендатор  

### <a name="prepare-data"></a>Подготовка данных

Перед использованием модуля данные ввода должны находиться в формате, который ожидает модель рекомендаций. Требуется набор обучающих данных о трехкратных рейтингах пользователей.

+ Первый столбец содержит идентификаторы пользователей.
+ Вторая колонка содержит идентификаторы элементов.
+ Третья колонка содержит рейтинг для пары пользователь-элемент. Значения рейтинга должны быть численным типом.  

Набор данных **Movie Ratings** в конструкторе машинного обучения Azure (выберите **наборы данных,** а затем **образцы)** демонстрирует ожидаемый формат:

![Рейтинги фильмов](media/module/movie-ratings-dataset.png)

Из этого примера можно увидеть, что один пользователь оценил несколько фильмов. 

### <a name="train-the-model"></a>Обучение модели

1.  Добавьте модуль Train SVD Recommender в конвейер в проектировщике и подключите его к обучающие данные.  
   
2.  Для **числа факторов,** укажите количество факторов, которые можно использовать с рекомендателем.  
    
    Каждый фактор измеряет, сколько пользователь связан с элементом. Число факторов также размерность скрытого пространства фактора. С увеличением числа пользователей и элементов, лучше установить большее количество факторов. Но если число слишком велико, производительность может снизиться.
    
3.  **Количество итераций рекомендательных алгоритмов** указывает, сколько раз алгоритм должен обрабатывать входные данные. Чем выше это число, тем точнее прогнозы. Тем не менее, большее число означает более медленное обучение. Значение по умолчанию — 30.

4.  Для **обучения,** введите число между 0.0 и 2.0, который определяет размер шага для обучения.

    Скорость обучения определяет размер шага при каждой итерации. Если размер шага слишком велик, возможно, вы перечеркните оптимальное решение. Если размер шага слишком мал, обучение занимает больше времени, чтобы найти лучшее решение. 
  
5.  Отправить конвейер.

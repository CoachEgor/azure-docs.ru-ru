# <a name="permutation-feature-importance"></a>Значение функции перестановки

В этой статье описывается, как использовать модуль важности функций перестановки в конструкторе машинного обучения Azure (предварительный просмотр) для вычисления набора баллов важности функций для набора данных. Вы используете эти оценки, чтобы помочь вам определить лучшие функции для использования в модели.

В этом модуле значения объектов случайным образом перетасовываются по одному столбику за раз. Производительность модели измеряется до и после. Вы можете выбрать один из стандартных показателей для измерения производительности.

Оценки, которые возвращается модуль, представляют *собой изменение* производительности обученной модели после перестановки. Важные функции, как правило, более чувствительны к процессу перетасовки, поэтому они приведут к более высокой важности баллов. 

В этой статье приводится обзор функции перестановки, ее теоретической основы и ее применения в машинном обучении: [Важность функции перестановки](https://blogs.technet.com/b/machinelearning/archive/2015/04/14/permutation-feature-importance.aspx).  

## <a name="how-to-use-permutation-feature-importance"></a>Как использовать важность функции перестановки

Создание набора оценок функций требует наличия уже обученной модели, а также набора тестовых данных.  

1.  Добавьте модуль значения функции перестановки в конвейер. Вы можете найти этот модуль в категории **Выбор функций.** 

2.  Подключите обученную модель к левому вхустройству. Модель должна быть регрессионной моделью или моделью классификации.  

3.  На правом входе подключите набор данных. Предпочтительно выбрать тот, который отличается от набора данных, который вы использовали для обучения модели. Этот набор данных используется для оценки на основе обученной модели. Он также используется для оценки модели после изменения значений объектов.  

4.  Для **случайных семян,** введите значение для использования в качестве семян для рандомизации. Если вы указываете 0 (по умолчанию), номер генерируется на основе системных часов.

     Значение семян является необязательным, но вы должны предоставить значение, если вы хотите воспроизводимости через пробеги того же конвейера.  

5.  Для **измерения производительности**выберите единую метрику для использования при выполнении компьютерной модели после перестановки.  

     Дизайнер машинного обучения Azure поддерживает следующие метрики, в зависимости от того, оцениваете ли вы модель классификации или регрессии:  

    -   **Классификации**

        Точность, точность, отзыв  

    -   **Регрессии**

        Точность, напомним, средняя абсолютная ошибка, Корневая средняя квадратная ошибка, Относительная абсолютная ошибка, Относительная квадратная ошибка, Коэффициент определения  

     Более подробное описание этих показателей оценки и способов [Evaluate Model](evaluate-model.md)их расчета см.  

6.  Отправить конвейер.  

7.  Модуль выводит список столбцов объектов и оценки, связанные с ними. Список ранжируется в порядке убывания баллов.  


##  <a name="technical-notes"></a>Технические примечания

Значение функции перестановки работает путем случайного изменения значений каждого столбца объекта, по одному столбце за раз. Затем он оценивает модель. 

Рейтинги, которые предоставляет модуль, часто отличаются от тех, которые вы получаете от [выбора функций на основе фильтра.](filter-based-feature-selection.md) Выбор функций на основе фильтра вычисляет баллы *до* создания модели. 

Разница заключается в том, что значение функции перестановки не измеряет связь между объектом и целевым значением. Вместо этого он фиксирует, какое влияние каждая функция оказывает на прогнозы модели.

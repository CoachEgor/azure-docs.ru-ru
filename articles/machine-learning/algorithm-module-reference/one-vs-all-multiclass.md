# <a name="one-vs-all-multiclass"></a>Многоклассовая классификация "один-все"

В этой статье описывается, как использовать модуль One-vs-All Multiclass в дизайнере машинного обучения Azure (предварительный просмотр). Цель состоит в том, чтобы создать модель классификации, которая может предсказать несколько классов, используя подход *«один против всех».*

Этот модуль удобно использовать при создании моделей для предсказания трех и более возможных значений, если они зависят от непрерывных или категориальных прогностических переменных. Этот метод также позволяет использовать методы двоичной классификации для проблем, решение которых требует использования нескольких выходных классов.

### <a name="more-about-one-versus-all-models"></a>Подробнее о моделях «один против всех»

Некоторые алгоритмы классификации позволяют использовать более двух классов по дизайну. Другие ограничивают возможные результаты одним из двух значений (двоичная или двухклассная модель). Но даже алгоритмы двоичной классификации могут быть адаптированы для многоклассовых задач классификации с помощью различных стратегий. 

Этот модуль реализует метод «один против всех», в котором создается двоичная модель для каждого из нескольких классов вывода. Модуль оценивает каждую из этих бинарных моделей для отдельных классов в отличие от его дополнения (все другие классы в модели), как будто это двоичная проблема классификации. Модуль затем выполняет прогноз, запустив эти двоичные классификаторы и выбрав прогноз с наивысшим баллом доверия.  

По сути, модуль создает ансамбль отдельных моделей, а затем объединяет результаты, чтобы создать единую модель, которая предсказывает все классы. Любой двоичный классификатор может быть использован в качестве основы для модели «один против всех».  

Например, предположим, что вы настраиваете [двухклассную модель векторной группы поддержки](two-class-support-vector-machine.md) и предоставляете ее в качестве ввода в модуль «Один против всех». Модуль создаст двухклассные модели векторных машин поддержки для всех членов класса вывода. Затем будет применяться метод "один против всех" для объединения результатов для всех классов.  

## <a name="how-to-configure-the-one-vs-all-multiclass-classifier"></a>Как настроить классификатор «один против всех»  

Этот модуль создает ансамбль бинарных моделей классификации для анализа нескольких классов. Чтобы использовать этот модуль, необходимо сначала настроить и обучить *двоичную модель классификации.* 

Двоичная модель подключается к многоклассовому модулю «один против всех». Затем вы тренируете ансамбль моделей с помощью [модели Train Model](train-model.md) с помеченным набором обучающих данных.

При объединении моделей One-vs-All Multiclass создает несколько моделей двоичной классификации, оптимизирует алгоритм для каждого класса, а затем объединяет модели. Модуль выполняет эти задачи, даже несмотря на то, что набор обучаемых данных может иметь несколько значений класса.

1. Добавьте модуль One-vs-All Multiclass в конвейер в проектировщике. Вы можете найти этот модуль под **машинное обучение - Инициализация**, в категории **классификации.**

   Классификатор One-vs-All Multiclass не имеет собственных настраиваемых параметров. Любые настройки должны быть сделаны в модели двоичной классификации, которая предоставляется в качестве ввода.

2. Добавьте в конвейер модель двоичной классификации и настройте эту модель. Например, можно использовать [двухклассную машину векторов поддержки](two-class-support-vector-machine.md) или [двухклассное повышаельное дерево решений.](two-class-boosted-decision-tree.md)

3. Добавьте модуль [Модели поезда](train-model.md) в конвейер. Соедините неподготовленный классификатор, который является выходом One-vs-All Multiclass.

4. На другом входе [модели train Model](train-model.md)подключите помеченный набор обучаемых данных с несколькими значениями класса.

5. Отправить конвейер.

## <a name="results"></a>Результаты

После завершения обучения можно использовать модель для прогнозирования нескольких классов.

Кроме того, можно передать неподготовленный классификатор [модели Cross-Validate](cross-validate-model.md) для перекрестной проверки по набору данных с помеченной проверкой.

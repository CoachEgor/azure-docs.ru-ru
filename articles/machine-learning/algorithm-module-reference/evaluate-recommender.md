# <a name="evaluate-recommender"></a>Оценка рекомендателя

В этой статье описывается, как использовать модуль «Рекомендация по оценке» в конструкторе машинного обучения Azure (предварительный просмотр). Цель состоит в том, чтобы измерить точность прогнозов, сделанных рекомендательной моделью. Используя этот модуль, вы можете оценить различные виды рекомендаций:  
  
-   Рейтинги, прогнозируемые для пользователя и товара    
-   Элементы, рекомендуемые для пользователя  
  
При создании прогнозов с помощью модели рекомендаций для каждого из этих поддерживаемых типов предсказаний возвращаются несколько иные результаты. Модуль «Оценка рекомендателя» выводит вид прогноза из формата столбца набора набранных данных. Например, набор набранных данных может содержать:

- Пользователь-элемент-рейтинг втрое
- Пользователи и рекомендуемые ими элементы

Модуль также применяет соответствующие метрики производительности, основанные на типе прогноза. 

  
## <a name="how-to-configure-evaluate-recommender"></a>Как настроить Оценка Рекомендация

Модуль «Оценка рекомендателя» сравнивает результаты прогнозирования, используя рекомендательную модель с соответствующими данными «наземной правды». Например, модуль [Score SVD Recommender](score-svd-recommender.md) производит наборы данных, которые можно проанализировать с помощью Evaluate Recommender.

### <a name="requirements"></a>Требования

Оценить Рекомендацию требует следующих наборов данных в качестве ввода. 
  
#### <a name="test-dataset"></a>Проверочный набор данных

Набор данных тестирования содержит данные "наземной правды" в виде трехкратных рейтингов пользователей.  

#### <a name="scored-dataset"></a>Оцененный набор данных

Набор набранных данных содержит прогнозы, генерируемые рекомендательной моделью.  
  
Столбцы в этом втором наборе данных зависят от того, какой прогноз выполнялся во время процесса подсчета очков. Например, набор набранных данных может содержать любой из следующих:

- Пользователи, элементы и рейтинги, которые пользователь, скорее всего, даст для элемента
- Список пользователей и предметов, рекомендованных для них 

### <a name="metrics"></a>Метрики

Показатели производительности модели генерируются в зависимости от типа ввода. В следующих разделах приведены подробные сведения.

## <a name="evaluate-predicted-ratings"></a>Оценка прогнозируемых рейтингов  

При оценке прогнозируемых оценок набор данных (второй вход для оценки рекомендаций) должен содержать тройки, отвечающие этим требованиям:
  
-   Первый столбец набора данных содержит идентификаторы пользователей.    
-   Вторая колонка содержит идентификаторы элементов.  
-   Третья колонка содержит соответствующие оценки пользовательского элемента.  
  
> [!IMPORTANT] 
> Для того чтобы оценка увенчалась `User` `Item`успехом, имена столбцов должны быть, и `Rating`соответственно.  
  
Оценить Recommender сравнивает рейтинги в наборе данных «земляная правда» с прогнозируемыми рейтингами набранного набора данных. Затем он вычисляет среднее абсолютное погрешность (MAE) и корневую среднее квадратную ошибку (RMSE).



## <a name="evaluate-item-recommendations"></a>Оценка рекомендаций по номенклатуре

При оценке рекомендаций по элементу используйте набор данных, который включает рекомендуемые элементы для каждого пользователя:
  
-   Первый столбец набора данных должен содержать идентификатор пользователя.    
-   Все последующие столбцы должны содержать соответствующие рекомендуемые идентификаторы элементов, упорядоченные по тому, насколько актуален элемент для пользователя. 

Перед подключением этого набора данных мы рекомендуем сортировать набор данных таким образом, чтобы наиболее релевантные элементы были на первом место.  

> [!IMPORTANT] 
> Для оценки Рекомендатор работать, `User` `Item 1`названия `Item 2` `Item 3` столбцов должны быть , , и так далее.  
  
Evaluate Recommender вычисляет средний нормализованный дисконтированный совокупный прирост (NDCG) и возвращает его в набор выходных данных.  
  
Поскольку невозможно узнать фактическую «истину земли» для рекомендуемых элементов, Evaluate Recommender использует оценки пользовательского элемента в тестовом наборе данных в качестве прибыли при вычислении NDCG. Для оценки модуля оценки рекомендаций должна быть только для получения рекомендаций по элементам с рейтингами «земляная правда» (в наборе тестовых данных).

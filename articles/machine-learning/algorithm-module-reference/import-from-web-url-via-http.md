# <a name="import-from-web-url-via-http-module"></a>Импорт из URL-адреса через модуль HTTP

В этой статье описывается модуль визуального интерфейса (Предварительная версия) для службы Машинное обучение Azure.

Этот модуль используется для чтения данных из общедоступной веб-страницы для использования в эксперименте машинного обучения.

К данным, опубликованным на веб-странице, применяются следующие ограничения.

- Данные должны быть в одном из поддерживаемых форматов: CSV, TSV, ARFF или SvmLight. Другие данные приведут к ошибкам.
- Проверка подлинности не требуется или не поддерживается. Данные должны быть общедоступными. 

Получить данные можно двумя способами: с помощью мастера настройте источник данных или настройте его вручную.

## <a name="use-the-data-import-wizard"></a>Использование мастера импорта данных

1. Добавьте модуль **Импорт данных** в эксперимент. Модуль можно найти в интерфейсе в категории **входные и выходные данные** .

2. Щелкните **запустить мастер импорта данных** и выберите URL-адрес через HTTP.

3. Вставьте URL-адрес и выберите формат данных.

4. По завершении настройки.

Чтобы изменить существующее подключение к данным, запустите мастер еще раз. Мастер загрузит все предыдущие сведения о конфигурации, чтобы не приступить к повторному запуску с нуля.

## <a name="manually-set-properties-in-the-import-data-module"></a>Установка свойств вручную в модуле "Импорт данных"

Следующие шаги описывают настройку источника импорта вручную.

1. Добавьте модуль [Импорт данных](import-data.md) в эксперимент. Модуль можно найти в интерфейсе в категории **входные и выходные данные** .

2. В качестве **источника данных**выберите **URL-адрес в Интернете через HTTP**.

3. В поле **URL-адрес**введите или вставьте полный URL-адрес страницы, содержащей данные, которые требуется загрузить.

    URL-адрес должен содержать URL-адрес сайта и полный путь, с именем и расширением файла, на страницу, содержащую данные для загрузки.

    Например, на следующей странице приведен набор данных IRI из репозитория машинного обучения штата Калифорния, Ирвин:

    `https://archive.ics.uci.edu/ml/machine-learning-databases/iris/iris.data`

4. В поле **Формат данных**выберите один из поддерживаемых форматов данных из списка.

    Рекомендуется всегда проверять данные, чтобы определить формат. На странице UC Ирвин используется формат CSV. Другие поддерживаемые форматы данных: TSV, ARFF и SvmLight.

5. Если данные представлены в формате CSV или TSV, используйте параметр « **файл содержит строку заголовка** », чтобы указать, содержат ли исходные данные строку заголовка. Строка заголовка используется для назначения имен столбцов.

6. Установите флажок **использовать кэшированные результаты** , если вы не предполагаете, что данные изменяют много, или если вы хотите избежать перезагрузки данных при каждом запуске эксперимента.

    Если выбран этот параметр, эксперимент загружает данные при первом запуске модуля, а затем использует кэшированную версию набора данных.

    Если требуется перезагрузить набор данных при каждой итерации набора данных эксперимента, снимите флажок **использовать кэшированные результаты** . Кроме того, результаты перегружаются при наличии изменений параметров [импорта данных](import-data.md).

7. Запустите эксперимент.

## <a name="results"></a>Результаты

По завершении щелкните выходной набор данных и выберите **визуализировать** , чтобы узнать, успешно ли импортированы данные.

# <a name="import-from-web-url-via-http-module"></a>Импорт из URL-адрес с помощью HTTP-модуля

В этой статье описывается модуль визуального интерфейса (Предварительная версия) для службы машинного обучения Azure.

Этот модуль используется для чтения данных из общедоступной веб-страницы для использования в эксперимента машинного обучения.

К данным, опубликованным на веб-странице, применяются следующие ограничения:

- Данные должны быть в одном из поддерживаемых форматов: CSV, TSV, ARFF или SvmLight. Другие данные будут вызывать ошибки.
- Проверка подлинности не требуется или поддерживается. Данные должны быть общедоступными. 

Существует два способа для получения данных: с помощью мастера настройки источника данных, или настройте его вручную.

## <a name="use-the-data-import-wizard"></a>Использование мастера импорта данных

1. Добавить **импорта данных** эксперимент модуль. Модуль в интерфейсе, можно найти в **ввод и вывод данных** категории.

2. Нажмите кнопку **запустить мастер импорта данных** и выберите URL-адрес по протоколу HTTP.

3. Вставьте URL-адрес и выберите формат данных.

4. Когда настройка завершена.

Чтобы изменить существующее подключение к данным, запустите мастер еще раз. Мастер загружает все предыдущие сведения о конфигурации, чтобы не нужно снова начать с нуля

## <a name="manually-set-properties-in-the-import-data-module"></a>Вручную задать свойства в модуле импорта данных

Следующие шаги описывают, как вручную настроить источника импорта.

1. Добавить [импорта данных](import-data.md) эксперимент модуль. Модуль в интерфейсе, можно найти в **ввод и вывод данных** категории.

2. Для **источника данных**выберите **URL-адрес по протоколу HTTP**.

3. Для **URL-адрес**введите или вставьте полный URL-адрес страницы, которая содержит данные, которые вы хотите загрузить.

    Этот адрес должен содержать URL-адрес сайта и полный путь, имя файла и расширение, для страницы, содержащей данные для загрузки.

    Например следующая страница содержит набор данных Iris из репозитория машинного обучения Калифорнийского Университета, Ирвин:

    `https://archive.ics.uci.edu/ml/machine-learning-databases/iris/iris.data`

4. Для **формат данных**, выберите один из данных, поддерживаемых форматов в списке.

    Мы рекомендуем всегда проверяйте данные заранее, чтобы определить формат. На странице UC Irvine используется формат CSV. Другие поддерживаемые форматы данных: TSV, ARFF и SvmLight.

5. Если данные находятся в формате CSV или TSV, используйте **файл содержит строку заголовков** параметр, чтобы указать ли источник данных включает строку заголовка. Строки заголовка используется для назначения имен столбцов.

6. Выберите **использовать кэшированные результаты** параметры, если вы не ожидается, что так сильно меняются данные, или если вы хотите избежать повторной загрузки данных каждый раз при запуске эксперимента.

    Если этот параметр выбран, эксперимент загружает время данных первый модуль выполняется, а затем использует кэшированную версию набора данных.

    Если вы хотите перезагрузить набора данных на каждой итерации эксперимента набора данных, отмените выбор **использовать кэшированные результаты** параметр. Если каких-либо изменений с параметрами, также загружаются результаты [импорта данных](import-data.md).

7. Запустите эксперимент.

## <a name="results"></a>Результаты

По завершении щелкните выходной набор данных и выберите **визуализировать** для просмотра, если данные были успешно импортированы.

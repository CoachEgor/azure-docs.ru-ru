# <a name="module-assign-data-to-clusters"></a>Модуль: Назначение данных кластерам

В этой статье описывается использование *назначение данных кластерам* модуля в визуальном интерфейсе машинного обучения Azure. Модуль создает прогнозов при помощи модели кластеризации, обучена на *кластеризация K-средних* алгоритм.

Назначение данных модуль кластеров возвращает набор данных, содержащий возможные назначения для каждой новой точки данных. 


## <a name="how-to-use-assign-data-to-clusters"></a>Как использовать назначение данных кластерам
  
1. В визуальном интерфейсе машинного обучения Azure найдите предварительно обученной модели кластеризации. Можно создать и обучить модель кластеризации с помощью любого из следующих методов:  
  
    - Настройка алгоритма кластеризации K-средних с помощью [кластеризация K-средних](k-means-clustering.md) модуль и обучение модели на основе набора данных и модуля обучения модели кластеризации (в этой статье).  
  
    - Можно также добавить существующей обученной модели кластеризации из **сохранения моделей** в рабочей группе.

2. Присоединение обученной модели к левому порту ввода **назначение данных кластерам**.  

3. Подключите новый набор данных в качестве входных данных. 

   В этом наборе данных метки являются необязательными. Как правило Кластеризация является методом неконтролируемого обучения. Вы не должны заранее знать категории. Тем не менее входные столбцы должны быть так же, как столбцы, которые использовались при обучении, модели кластеризации, иначе возникает ошибка.

    > [!TIP]
    > Чтобы уменьшить количество столбцов, которые записываются в интерфейс кластера прогнозы, используйте [Выбор столбцов в наборе данных](select-columns-in-dataset.md)и выбрать подмножество столбцов. 
    
4. Оставьте **проверки Append или снимите флажок для результата только** флажок установленным, если результаты должны содержать полное входного набора данных, включая столбец, отображающий результатов (назначений кластерам).
  
    Если снять этот флажок, то возвращаются только результаты. Этот параметр может быть полезен при создании прогнозов как часть веб-службы.
  
5.  Запустите эксперимент.  
  
### <a name="results"></a>Результаты

+  Для просмотра значений в наборе данных, щелкните модуль правой кнопкой мыши **привести наборы данных**, а затем выберите **визуализировать**.

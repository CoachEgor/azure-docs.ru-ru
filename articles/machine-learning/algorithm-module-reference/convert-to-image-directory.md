# <a name="convert-to-image-directory"></a>Преобразование в каталог изображений

В этой статье описывается, как использовать модуль Convert to Image Directory для преобразования набора данных изображений в тип данных "Каталог изображений", который представляет собой стандартизированный формат данных в задачах, связанных с изображениями, таких как классификация изображений в Машинное обучение Azure конструкторе (Предварительная версия).

## <a name="how-to-use-convert-to-image-directory"></a>Использование Convert to Image Directory  

1.  Добавьте в эксперимент модуль **преобразовать в каталог изображений** . Этот модуль можно найти в категории "Преобразование данных" Компьютерное зрение/Image "в списке модулей. 

2.  [Зарегистрируйте набор данных образа](https://docs.microsoft.com/azure/machine-learning/how-to-create-register-datasets) и подключите его к порту ввода модуля. Убедитесь, что в входном наборе данных есть изображение. 
    Поддерживаются следующие форматы наборов данных:

    - Сжатый файл в следующих расширениях: ". zip", ". tar", ". gz", ". bz2".
    - Папка, содержащая изображения. **Настоятельно рекомендуем сначала сжать такую папку, а затем использовать сжатый файл в качестве набора данных**.

    > [!WARNING]
    > Модуль **импорта данных** **нельзя** использовать для импорта набора данных изображений, так как тип выходных данных модуля **Import Data** — это каталог данных, содержащий только строку пути к файлу.
    

    > [!NOTE]
    > Если используется набор данных изображений в защищенном учебном обучении, то метка является обязательной.
    > Для задачи классификации изображений метка может быть создана как изображение "Категория" в выходных данных модуля, если этот набор данных изображений организован в формате торчвисион Имажефолдер. В противном случае только изображения сохраняются без метки. Ниже приведен пример того, как можно организовать набор данных изображений для получения метки, использовать категорию изображений в качестве имени вложенной папки. Дополнительные сведения см. в разделе [торчвисион DataSets](https://pytorch.org/docs/stable/torchvision/datasets.html#imagefolder) .
    >
    > ```
    > root/dog/xxx.png
    > root/dog/xxy.png
    > root/dog/xxz.png
    >
    > root/cat/123.png
    > root/cat/nsdf3.png
    > root/cat/asd932_.png
    > ```

3.  Отправьте конвейер.

## <a name="results"></a>Результаты

Выходные данные **преобразования в модуль каталога изображений** находятся в формате каталога изображений и могут быть подключены к другим модулям, связанным с изображениями, для которых формат входного порта также является каталогом образа.

# <a name="convert-to-image-directory"></a>Преобразование в каталог изображений

В этой статье описывается использование модуля Convert to Image Directory для преобразования набора данных изображений в тип данных "Каталог изображений", который является стандартизированным форматом данных в задачах, связанных с образами, таких как классификация изображений в конструкторе Машинное обучение Azure.

## <a name="how-to-use-convert-to-image-directory"></a>Использование Convert to Image Directory  

1.  Добавьте в холст модуль **преобразовать в каталог изображений** . Этот модуль можно найти в категории "Преобразование данных" Компьютерное зрение/Image "в списке модулей. 

2.  Входной модуль **Convert to Image Directory** должен представлять собой файловый набор данных. [Зарегистрируйте набор данных образа](https://docs.microsoft.com/azure/machine-learning/how-to-create-register-datasets) и подключите его к порту ввода модуля. Убедитесь, что в входном наборе данных есть изображение. В настоящее время конструктор не поддерживает визуализацию набора данных изображений.
 
    Поддерживаются следующие форматы наборов данных:

    - Сжатый файл в следующих расширениях: ". zip", ". tar", ". gz", ". bz2".
    - Папка, содержащая изображения. **Настоятельно рекомендуем сначала сжать такую папку, а затем использовать сжатый файл в качестве набора данных**.

    > [!WARNING]
    > Модуль **импорта данных** **нельзя** использовать для импорта набора данных изображений, так как тип выходных данных модуля **Import Data** — это каталог данных, содержащий только строку пути к файлу.
    

    > [!NOTE]
    > - При использовании набора данных Image в защищенном обучении необходимо указать метку набора данных для обучения.
    > - Для задачи классификации изображений метка может быть создана как изображение "Категория" в выходных данных модуля, если этот набор данных изображений организован в формате торчвисион Имажефолдер. В противном случае только изображения сохраняются без метки. Ниже приведен пример того, как можно организовать набор данных изображений для получения метки, использовать категорию изображений в качестве имени вложенной папки. 
    > - Вам не нужно отправлять одинаковое количество образов в каждой папке категории.
    > - Поддерживаются образы с такими расширениями (в нижнем регистре): ". jpg", ". JPEG", ". png", ". ppm", ". bmp", ". PGM", ". tif", ". TIFF", ". вебп". В одной папке можно также использовать несколько типов образов.    
    > - Дополнительные сведения см. в разделе [торчвисион DataSets](https://pytorch.org/docs/stable/torchvision/datasets.html#imagefolder) .
    >
    > ```
    > Your_image_folder_name/Category_1/xxx.png
    > Your_image_folder_name/Category_1/xxy.jpg
    > Your_image_folder_name/Category_1/xxz.jpeg
    >
    > Your_image_folder_name/Category_2/123.png
    > Your_image_folder_name/Category_2/nsdf3.png
    > Your_image_folder_name/Category_2/asd932_.png
    > ```
    > - Если для оценки используется набор данных Image, то набор данных входных файлов этого модуля должен содержать Неклассифицированные изображения.
    
3.  Отправьте конвейер. Этот модуль можно запустить либо на GPU, либо на ЦП.

## <a name="results"></a>Результаты

Выходные данные **преобразования в модуль каталога изображений** находятся в формате каталога изображений и могут быть подключены к другим модулям, связанным с изображениями, для которых формат входного порта также является каталогом образа.

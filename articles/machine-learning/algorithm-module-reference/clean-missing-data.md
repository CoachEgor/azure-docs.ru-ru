# <a name="clean-missing-data-module"></a>Модуль «Чистые отсутствующие данные»

В этой статье описывается модуль в дизайнере машинного обучения Azure (предварительный просмотр).

Используйте этот модуль для удаления, замены или вывода недостающих значений. 

Специалисты по обработке данных часто проверяют данные на наличие недостающих значений, а затем выполняют различные операции по фиксации данных или вставке новых значений. Цельтаких операций очистки заключается в предотвращении проблем, вызванных отсутствием данных, которые могут возникнуть при обучении модели. 

Этот модуль поддерживает несколько типов операций для "очистки" недостающих значений, в том числе:

+ Замена недостающих значений заполнителем, средним или другим значением
+ Полностью удаление строк и столбцов, в которых отсутствуют значения
+ Вывод значений на основе статистических методов


Использование этого модуля не меняет набор исходных данных. Вместо этого он создает новый набор данных в рабочем пространстве, который можно использовать в последующем рабочем процессе. Вы также можете сохранить новый, очищенный набор данных для повторного использования.

Этот модуль также выводит определение преобразования, используемого для очистки недостающих значений. Это преобразование можно повторно использовать на других наборах данных, которые имеют ту же схему, используя модуль [Apply Transformation.](./apply-transformation.md)  

## <a name="how-to-use-clean-missing-data"></a>Как использовать чистые отсутствующие данные

Этот модуль позволяет определить операцию очистки. Вы также можете сохранить операцию очистки, так что вы можете применить его позже к новым данным. Ознакомьтесь со следующими разделами, как создать и сохранить процесс очистки: 
 
+ [Замена недостающих значений](#replace-missing-values)
  
+ [Применение преобразования очистки к новым данным](#apply-a-saved-cleaning-operation-to-new-data)
 
> [!IMPORTANT]
> Метод очистки, используемый для обработки недостающих значений, может существенно повлиять на результаты. Мы рекомендуем вам экспериментировать с различными методами. Рассмотрим как обоснование использования того или иного метода, так и качество результатов.

### <a name="replace-missing-values"></a>Замените отсутствующие значения  

Каждый раз, когда вы применяете модуль [Clean Missing Data](./clean-missing-data.md) к набору данных, одиниковая операция очистки применяется ко всем выбранным столбцов. Поэтому, если вам нужно очистить различные столбцы с помощью различных методов, используйте отдельные экземпляры модуля.

1.  Добавьте модуль [Clean Missing Data](./clean-missing-data.md) в конвейер и подключите набор данных с недостающим значениями.  
  
2.  Для **очистки столбцов**выберите столбцы, содержащие недостающие значения, которые вы хотите изменить. Вы можете выбрать несколько столбцов, но необходимо использовать один и тот же метод замены во всех выбранных столбцах. Поэтому обычно необходимо почищать строки и числовые столбцы по отдельности.

    Например, чтобы проверить недостающие значения во всех числовых столбцах:

    1. Выберите модуль **Clean Missing Data** и нажмите на **столбец Edit** в правой панели модуля.

    3. Для **включения**— выберите **типы столбцов** из списка выпадающих, а затем выберите **Число.** 
  
    Любой метод очистки или замены, который вы выберете, должен быть применим ко **всем** столбцам в выборе. Если данные в каком-либо столбце несовместимы с указанной операцией, модуль возвращает ошибку и останавливает конвейер.
  
3.  Для **минимального коэффициента отсутствующих значений**укажите минимальное количество недостающих значений, требуемых для выполнения операции.  
  
    Вы используете эту опцию в сочетании с **максимальным коэффициентом недостающего значения** для определения условий, при которых операция очистки выполняется на наборе данных. Если слишком много или слишком мало строк, не хватает значений, операция не может быть выполнена. 
  
    Вводимый номер представляет **отношение** недостающих значений ко всем значениям в столбце. По умолчанию **свойство минимального значения недостающих значений** устанавливается до 0. Это означает, что недостающие значения очищаются, даже если есть только одно недостающее значение. 

    > [!WARNING]
    > Это условие должно быть выполнено каждым столбцом для того, чтобы указанная операция была применена. Например, предположим, что вы выбрали три столбца, а затем установите минимальное соотношение недостающих значений до .2 (20%), но только один столбец на самом деле имеет 20% недостающих значений. В этом случае операция очистки будет применяться только к столбце с более чем 20% недостающих значений. Таким образом, другие столбцы будут неизменными.
    > 
    > В случае каких-либо сомнений в изменении недостающих значений выберите параметр **Создать столбец индикации недостающих значений**. К набору данных придавает столбец, указывающий, соответствует ли каждая колонка указанным критериям для минимальных и максимальных диапазонов.  
  
4. Для **максимального коэффициента отсутствующих значений**укажите максимальное количество недостающих значений, которые могут присутствовать для выполнения операции.   
  
    Например, можно выполнить недостающую замену значений только в том случае, если 30% или меньше строк содержат недостающие значения, но оставляйте значения как-это, если более 30% строк имеют недостающие значения.  
  
    Его следует указать как отношение недостающих значений ко всем значениям в столбце. По умолчанию **максимальное коэффициент недостающих значений** устанавливается на 1. Это означает, что недостающие значения очищаются, даже если 100% значений в столбце отсутствуют.  
  
   
  
5. Для **режима очистки**выберите один из следующих вариантов замены или удаления недостающих значений:  
  
  
    + **Пользовательское значение замены**: Используйте эту опцию, чтобы указать значение заполнителя (например, 0 или NA), которое применяется ко всем недостающим значениям. Значение, указанное в качестве замены, должно быть совместимо с типом данных столбца.
  
    + **Заменить средним:** Вычисляет столбец означает и использует среднее значение замены для каждого отсутствующих значений в столбце.  
  
        Применяется только для столбцов с типами данных Integer, Double или Boolean.  
  
    + **Заменить медиану:** Вычисляет медианное значение столбца и использует медианное значение в качестве замены для любого отсутствующего значения в столбце.  
  
        Применяется только для столбцов с типом данных Integer или Double. 
  
    + **Заменить режим:** Вычисляет режим для столбца и использует режим в качестве значения замены для каждого отсутствующее значение в столбце.  
  
        Применяется только для столбцов с типом данных Integer, Double, Boolean или категориальным типом. 
  
    + **Удалить всю строку:** Полностью удаляет любую строку в наборе данных, которая имеет одно или недостающие значения. Этот вариант удобно использовать, если недостающее значение может рассматриваться как случайный пропуск.  
  
    + **Удалить весь столбец:** Полностью удаляет любой столбец в наборе данных, который имеет одно или недостающие значения.  
  
    
  
6. Значение **замены** опции доступно, если вы выбрали опцию, **пользовательское значение замены.** Введите новое значение для использования в качестве значения замены для всех недостающих значений в столбце.  
  
    Обратите внимание, что эту опцию можно использовать только в столбцах, в которых есть Integer, Double, Boolean или String.
  
7. **Создание столбца индикатора недостающих значений:** Выберите этот параметр, если вы хотите вывести некоторое представление о том, соответствовали ли значения в столбце критериям для очистки значения. Эта опция особенно полезна при настройке новой операции очистки и хотите убедиться, что она работает так, как задумано.
  
8. Отправить конвейер.

### <a name="results"></a>Результаты

Модуль возвращает два выхода:  

-   **Очищенный набор данных**: Набор данных, состоящий из выбранных столбцов, с отсутствующими значениями, обрабатываемыми в указанных, наряду с столбцом индикатора, если вы выбрали этот вариант.  

    Колонны, не выбранные для очистки, также "проходят".  
  
-  **Преобразование очистки:** Преобразование данных, используемое для очистки, которое может быть сохранено в рабочем пространстве и применено к новым данным позже.

### <a name="apply-a-saved-cleaning-operation-to-new-data"></a>Применение операции очистки к новым данным  

Если вам нужно часто повторять операции очистки, мы рекомендуем сохранить рецепт для очистки данных в качестве *преобразования,* чтобы повторно использовать с тем же набором данных. Сохранение преобразования очистки особенно полезно, если необходимо часто повторно импортировать, а затем очищать данные с той же схемой.  
      
1.  Добавьте модуль [«Применить трансформацию»](./apply-transformation.md) в конвейер.  
  
2.  Добавьте набор данных, который необходимо очистить, и подключите набор данных к правому вхотворного порту.  
  
3.  Расширьте группу **Transforms** в левом стеле дизайнера. Найдите сохраненное преобразование и перетащите ее в конвейер.  

4.  Подключите сохраненную трансформацию к левому вхотвому порту [Apply Transformation.](./apply-transformation.md) 

    При применении сохраненного преобразования нельзя выбрать столбцы, к которым применяется преобразование. Это связано с тем, что преобразование уже определено и автоматически применяется к столбику, указанному в исходной операции.

    Однако предположим, что вы создали преобразование на подмножестве числовых столбцов. Это преобразование можно применить к набору данных смешанных типов столбцов, не вызывая ошибки, поскольку недостающие значения изменяются только в соответствующих числовых столбцах.

6.  Отправить конвейер.

# <a name="clean-missing-data-module"></a>Clean Missing Data модуля

В этой статье описывается модуль визуального интерфейса (Предварительная версия) для службы машинного обучения Azure.

Этот модуль используется для удаления, замены или вывести отсутствующих значений. 

Специалисты по анализу данных часто проверять данные для отсутствующих значений и выполнить различные операции, чтобы исправить данные или вставить новые значения. Такие операции очистки предназначена для того, чтобы предотвратить возникновение проблем из-за отсутствующих данных, которые могут возникнуть при обучении модели. 

Этот модуль поддерживает несколько типов операций для «очистки» недостающих значений, включая:

+ Отсутствующие значения, заменив заполнитель, среднее или другое значение
+ Полностью удалить строки и столбцы с отсутствующими значениями
+ Вывод типа значения, основанные на статистических методов


Использование этого модуля не изменяет исходный набор данных. Вместо этого он создает новый набор данных в рабочей области, который можно использовать в последующем рабочем процессе. Можно также сохранить новый очищенный набор для повторного использования.

Этот модуль также выдает определение преобразования, примененного для очистки недостающих значений. Это преобразование можно повторно использовать с другими наборами данных, который имеет ту же схему, с помощью [применение преобразования](./apply-transformation.md) модуля.  

## <a name="how-to-use-clean-missing-data"></a>Как использовать Очистка недостающих данных

Этот модуль позволяет определять операции очистки. Можно также сохранить операции очистки, чтобы позднее его можно применить к новым данным. См. в разделе приведены ссылки на описание того, как создать и сохранить процесса очистки: 
 
+ Для замены отсутствующих значений
  
+ Можно применять к новым данным очистки преобразование
 
> [!IMPORTANT]
> Метод очистки, который используется для обработки недостающих значений может значительно повлиять на результаты. Мы рекомендуем экспериментировать с помощью различных методов. Рассмотрим оба обоснования для использования конкретного метода и качество результатов.

### <a name="replace-missing-values"></a>Замените отсутствующие значения  

При каждом применении [Очистка недостающих данных](./clean-missing-data.md) модуль для набора данных, та же операция очистки применяется ко всем столбцам, которые можно выбрать. Поэтому если необходимо очистить различных столбцов, с помощью разных методов, рекомендуется используйте отдельные экземпляры модуля.

1.  Добавить [Очистка недостающих данных](./clean-missing-data.md) вашего эксперимента и подключите набор данных, в котором отсутствуют некоторые значения.  
  
2.  Для **столбцы для очистки**, выберите столбцы, содержащие недостающие значения, которые вы хотите изменить. Можно выбрать несколько столбцов, но необходимо использовать один и тот же метод замены всех выбранных столбцов. Таким образом обычно необходимо очистить строковых столбцов и числовые столбцы отдельно.

    Например, чтобы проверить наличие недостающих значений во всех числовых столбцах:

    1. Откройте средство выбора столбцов и выберите **ПРАВИЛА с**.
    2. Для **НАЧИНАЮТСЯ с**выберите **СТОЛБЦОВ нет**.

        Можно также начать со всеми СТОЛБЦАМИ, а затем исключить столбцы. Изначально правила не отображаются Если сначала щелкните **все СТОЛБЦЫ**, но вы сможете щелкнуть **СТОЛБЦОВ нет** и нажмите кнопку **все СТОЛБЦЫ** еще раз, чтобы начать со всеми столбцами, а затем отфильтровать столбцы (исключить) на основе имени, типа данных, или столбцы индекса.

    3. Для **Include**выберите **тип столбца** из раскрывающегося списка, а затем выберите **числовых**, или более конкретные числовой тип. 
  
    Метод очистки или замены, выбираемая должна быть применима к **все** столбцы в выделении. Если данные в любом столбце, несовместимые с заданной операцией, модуль возвращает сообщение об ошибке и прекращает эксперимента.
  
3.  Для **коэффициентом содержания недостающих значений менее**, укажите минимальное количество недостающих значений, необходимых для выполнения операции.  
  
    Используйте этот параметр в сочетании с **коэффициентом содержания недостающих значений более** для определения условий, при которых операции очистки выполняется на основе набора данных. Если имеется слишком много или слишком мало строк, в которых отсутствуют значения, невозможно выполнить операцию. 
  
    Число, вводимое представляет **соотношение** недостающих значений ко всем значениям в столбце. По умолчанию **коэффициент содержания недостающих значений менее** свойство имеет значение 0. Это означает, что недостающие значения будут удалены, даже если имеется только одно значение. 

    > [!WARNING]
    > Это условие необходимо выполнение каждого столбца в порядке для указанной операции для применения. Предположим, например, можно выбрать три столбца, а затем установить минимального содержания недостающих значений значение.2 (20%), но только один столбец фактически имеет 20% недостающих значений. В этом случае операция очистки будет применена только к столбцу с более чем 20% недостающих значений. Таким образом другие столбцы бы без изменений.
    > 
    > Если у вас есть сомнения по поводу ли изменении недостающих значений, выберите параметр **создать столбец индикации недостающих значений**. Столбец добавляется в набор данных, чтобы указать, выполнены ли каждый столбец указанным критериям для минимального и максимального диапазонов.  
  
4. Для **коэффициентом содержания недостающих значений более**, указать максимальное количество недостающих значений, которые могут присутствовать для выполнения операции.   
  
    Например, может потребоваться выполнить Подстановка отсутствующего значения только в том случае, если 30% или меньше строк недостающими значениями, но не изменяйте значения как-является наличие недостающих значений более чем 30% строк.  
  
    Номер определяется как отношение недостающих значений ко всем значениям в столбце. По умолчанию **коэффициентом содержания недостающих значений более** имеет значение 1. Это означает, что недостающие значения удалены, даже если отсутствуют 100% значений в столбце.  
  
   
  
5. Для **режим очистки**, выберите один из следующих параметров для замены или удаления отсутствующих значений:  
  
  
    + **Настраиваемое заменяющее значение**: Используйте этот параметр, чтобы указать значение заполнителя (например, 0 или н/д), которое применяется для всех недостающих значений. Значение, указанное в качестве замены должен быть совместим с типом данных столбца.
  
    + **Заменить средним**: Вычисляет среднее значение столбца и использует среднее значение в качестве замены для каждого недостающего значения столбца.  
  
        Применяется только для столбцов, имеющих Integer, Double или логические типы данных.  
  
    + **Замена медианным значением**: Вычисляет значение медианы столбца и использует значение медианы в качестве замены для каждого недостающего значения в столбце.  
  
        Применяется только к столбцам с типами данных Integer или Double. 
  
    + **Заменить режим**: Вычисляет моды столбца и использует режим в качестве замены для каждого недостающего значения столбца.  
  
        Применяется к столбцам с типами данных Integer, Double, Boolean или Категориальный. 
  
    + **Удаление всей строки**: Полностью удаляет любую строку в наборе данных, который имеет один или несколько недостающих значений. Это полезно, если недостающее значение может рассматриваться как случайный пропуск.  
  
    + **Удалить весь столбец**: Полностью удаляет любой столбец в наборе данных, который имеет один или несколько недостающих значений.  
  
    
  
6. Параметр **Замещающее значение** доступен, если выбран параметр, **настраиваемое значение подстановки**. Введите новое значение для использования в качестве замены для всех недостающих значений в столбце.  
  
    Обратите внимание на то, что этот параметр можно использовать только в столбцах с типами данных Integer, Double, Boolean или Date. Для столбцов даты заменяющее значение также можно ввести в виде количество 100-наносекундных тактов с момента 1/1/0001 12:00 утра  
  
7. **Создать столбец индикации недостающих значений**: Выберите этот параметр, если требуется направить вывод некоторые указания ли значения в столбце соответствующего критериям для очистки недостающих значений. Этот параметр особенно полезен при настройке новой операции очистки и хотим убедиться, что оно работает, как и было задумано.
  
8. Запустите эксперимент.

### <a name="results"></a>Результаты

Модуль возвращает два выхода:  

-   **Очищенный набор данных**: Набор данных, состоящий из выделенных столбцов с недостающими значениями обрабатываются как указано, а также столбец индикации, если выбран этот параметр.  

    Столбцы, не выбранные для очистки также «передаются через».  
  
-  **Преобразование очистки**: Преобразование данных использоваться как чистящая, который можно сохранить в рабочей области и применить к новым данным, более поздней версии.

### <a name="apply-a-saved-cleaning-operation-to-new-data"></a>Применить к новым данным сохраненные операции очистки  

Если необходимо повторить очистки часто, рекомендуется сохранять ваши рецепт для очистки данных в качестве *преобразования*, для повторного использования с одним набором данных. Сохранение очистки преобразования будет особенно полезен, если необходимо часто повторно импортировать и затем выполните очистку данных, который имеет ту же схему.  
      
1.  Добавить [применение преобразования](./apply-transformation.md) эксперимент модуль.  
  
2.  Добавьте набор данных, которые необходимо очистить и подключите набор данных, чтобы правый входной порт.  
  
3.  Разверните **преобразует** в левой части интерфейса. Найдите сохраненного преобразования и перетащите его в эксперимент.  
  
4.  Подключение к левому порту ввода из сохраненного преобразования [применение преобразования](./apply-transformation.md). 

    При применении сохраненное преобразование, нельзя выбрать столбцы, к которым применяются преобразования. Том, что преобразование уже определен и автоматически применяется к столбцам, указанным в исходной операции.

    Тем не менее Предположим, что вы создали преобразование на подмножестве числовых столбцов. Это преобразование можно применить к набору данных типов смешанных столбцов без возникновения ошибки, так, как отсутствующие значения изменяются только в соответствующие столбцы числовые.

6.  Запустите эксперимент.

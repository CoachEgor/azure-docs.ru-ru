# <a name="two-class-averaged-perceptron-module"></a>Двухклассный модуль Перцептрона

В этой статье описывается модуль в дизайнере машинного обучения Azure (предварительный просмотр).

Используйте этот модуль для создания модели машинного обучения на основе усредненный алгоритм перцептрона.  
  
Этот алгоритм классификации является контролируемым методом обучения и требует *тегами набора данных,* который включает столбец метки. Вы можете обучить модель, предоставив модель и помеченный набор данных в качестве ввода модели [train Model.](./train-model.md) Затем обученная модель может использоваться для прогнозирования значений для новых примеров ввода.  

### <a name="about-averaged-perceptron-models"></a>О усредненный перцептрон модели

*Усредненный метод перцептрона* является ранней и простой версией нейронной сети. В этом подходе входные данные классифицируются на несколько возможных выходов на основе линейной функции, а затем в сочетании с набором весов, которые вытекают из вектора функции, отсюда и название "перцептрон".

Более простые модели перцептрона подходят для обучения линейно отделяемых шаблонов, тогда как нейронные сети (особенно глубокие нейронные сети) могут моделировать более сложные границы класса. Однако перцептроны работают быстрее, и их можно использовать в непрерывном обучении, так как перцептроны обрабатывают случаи последовательно.

## <a name="how-to-configure-two-class-averaged-perceptron"></a>Как настроить двухклассный средний перцептрон

1.  Добавьте модуль **Среднего перцептрона двух классов** в конвейер.  

2.  Укажите, как нужно обучить модель, установив опцию **«Создать тренер».**  
  
    -   **Единый параметр**: Если вы знаете, как настроить модель, предоставьте определенный набор значений в качестве аргументов.

    -   **Диапазон параметров**: Выберите этот параметр, если вы не уверены в лучших параметрах, и хотите запустить развертки параметров. Выберите диапазон значений для итерации, и [Hyperparameters Tune Model](tune-model-hyperparameters.md) итерирует все возможные комбинации параметров, которые вы предоставили, чтобы определить гиперпараметры, которые дают оптимальные результаты.  
  
3.  Для **обучения скорость**, укажите значение для *обучения.* Значения скорости обучения контролируют размер шага, который используется при стохастический градиентный спуск каждый раз, когда модель тестируется и исправляется.
  
     Делая скорость меньше, вы тестируете модель чаще, с риском того, что вы можете застрять в местном плато. При увеличении шага схождение будет выполняться быстрее, но появляется риск превышения истинного минимума.
  
4.  Для **максимального количества итераций**введите количество раз, когда вы хотите, чтобы алгоритм изучил данные обучения.  
  
     Ранняя остановка часто обеспечивает лучшее обобщение. При повышении количества итераций улучшается подгонка, но при этом появляется риск возникновения лжевзаимосвязей.
  
5.  Для **семян случайного числа,** дополнительно введите целый ряд значение для использования в качестве семян. Использование семян рекомендуется, если вы хотите обеспечить воспроизводимость трубопровода через пробеги.  
  
1.  Подключите набор обучаемых данных и один из учебных модулей:
  
    -   При установке **режима создания тренажера** на **единый параметр**используйте модуль [Модели поезда.](train-model.md)

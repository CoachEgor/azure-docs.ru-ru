# <a name="two-class-averaged-perceptron-module"></a>Модуль Двухклассовый усредненный Перцептрон

В этой статье описывается модуль визуального интерфейса (Предварительная версия) для службы машинного обучения Azure.

Этот модуль используется для создания модели машинного обучения на основе алгоритма усредненного перцептрона.  
  
Этот алгоритм классификации — это метод контролируемого обучения и требуется *с тегами набора данных*, который включает в себя столбец метки. Вы можете обучить модель, предоставляя модели и маркированному набору данных в качестве входных данных [Обучение модели](./train-model.md). Затем обученную модель можно использовать для прогнозирования новых примеров входных данных.  

### <a name="about-averaged-perceptron-models"></a>О модели усредненного перцептрона

*Среднее значение метод перцептрона* является более ранней и простой версией нейронной сети. В этом случае входные данные делятся на несколько возможных значений на основе линейной функций, а затем объединяется с набором весовых коэффициентов, которые являются производными от вектора компонента, отсюда и название «перцептрон.»

Более простые модели перцептрона подходят для обучения линейно отделяемых шаблонов, тогда как нейронные сети (особенно глубокие нейронные сети) могут моделировать более сложные границы класса. Однако перцептроны работают быстрее, и так как они обрабатывают случаи последовательно перцептронов можно использовать в непрерывном обучении.

## <a name="how-to-configure-two-class-averaged-perceptron"></a>Как настроить Двухклассовый усредненный Перцептрон

1.  Добавить **Двухклассовый усредненный Перцептрон** эксперимент модуль.  

2.  Укажите способ модели для обучения, задав **Создание режима учителя** параметр.  
  
    -   **Единый параметр**: Если вы знаете, как вы хотите настроить модель, предоставить определенный ряд значений в качестве аргументов.
  
3.  Для **скорости обучения**, укажите значение для *скорости обучения*. Скорость обучения значения контролировать размер шага, который используется в вероятностного градиентного спуска каждый раз, тестировании и корректировке модели.
  
     Уменьшая частоту модель тестируется чаще, с риском, что вы может застрять в локальном плато. Выполняя шаг большего размера, вы может ускорить Сходимость, риском схождение истинного минимума.
  
4.  Для **максимальное число итераций**, введите номер раз требуется алгоритм для изучения обучающих данных.  
  
     Ранняя остановка часто обеспечивает лучшее обобщение. Увеличение количества итераций улучшается Подгонка, риском чрезмерно высокой точности.
  
5.  Для **начальное значение случайного числа**, при необходимости введите целое значение для использования в качестве начального значения. Если вы хотите убедиться, что выполняется для обеспечения повторяемости эксперимента по, рекомендуется использовать начальное значение.  
  
1.  Подключите обучающий набор данных и один из модулей обучения.
  
    -   Если задать **Создание режима учителя** для **один параметр**, использовать [Обучение модели](train-model.md) модуля.

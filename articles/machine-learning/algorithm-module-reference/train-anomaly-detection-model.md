# <a name="train-anomaly-detection-model"></a>Обучение модели обнаружения аномалий

В этой статье описывается, как использовать модуль **модели обнаружения аномалий поезд** в Azure Machine Learning designer (предварительный просмотр) для создания обученной модели обнаружения аномалий.

Модуль принимает в качестве ввода набор параметров модели для модели обнаружения аномалий и немаркированный набор данных. Он возвращает обученную модель обнаружения аномалий вместе с набором меток для обучаемых данных.  

Для получения дополнительной информации об алгоритмах обнаружения аномалий, представленных в проектировщике, см. 

+ [Обнаружение аномалий на основе PCA](pca-based-anomaly-detection.md)  

## <a name="how-to-configure-train-anomaly-detection-model"></a>Как настроить модель обнаружения аномалий поезд 

1.  Добавьте модуль **модели обнаружения аномалий поезд** в конвейер в проектировщике. Этот модуль можно найти в категории **обнаружения аномалий.**

2. Подключите один из модулей, предназначенных для обнаружения аномалий, таких как [обнаружение аномалий на основе PCA](pca-based-anomaly-detection.md)

    Другие типы моделей не поддерживаются; при запуске конвейера вы получите ошибку: все модели должны иметь один и тот же тип ученика.  

3.  Настроили модуль обнаружения аномалий, выбрав столбец метки и установив другие параметры, специфические для алгоритма.  

4.  Прикрепите набор обучаемых данных к правому ввоза **модели обнаружения аномалий поездной аномалии.**  

5.  Отправить конвейер.  

## <a name="results"></a>Результаты

После завершения обучения:

+ Чтобы просмотреть параметры модели, нажмите правой кнопкой мыши на модуле и выберите **Визуализацию.** 

+ Для создания прогнозов используйте [модель оценки](score-model.md) с новыми входными данными.

+ Чтобы сохранить моментальный снимок обученной модели, выберите модуль и нажмите на значок **набора данных Регистра** под вкладкой **Outputs-logs** в правой панели.

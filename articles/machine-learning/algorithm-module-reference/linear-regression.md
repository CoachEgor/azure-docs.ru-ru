# <a name="linear-regression-module"></a>Модуль линейной регрессии
В этой статье описывается модуль визуального интерфейса (Предварительная версия) для службы машинного обучения Azure.

Этот модуль используется для создания модели линейной регрессии для использования в эксперименте.  Линейная регрессия пытается установить линейную связь между одной или нескольких независимых переменных и числового результата или зависимой переменной. 

Этот модуль используется для определения метода, линейной регрессии и затем обучающая модель, используя набор данных с меткой. Затем обученную модель можно использовать для создания прогнозов.

## <a name="about-linear-regression"></a>О линейной регрессии

Линейная регрессия является распространенные статистический метод, который применяет в машинном обучении и улучшены за счет многие новые методы для подгонки линии, а также измерение ошибки. В наиболее общем смысле Регрессия относится к прогнозирования числовых целевого объекта. Линейная регрессия по-прежнему хорошо подходит при простой модели для основных задач прогнозирования. Линейную регрессию также обычно используют для работы с многомерными, разреженными наборами данных с отсутствием сложности.

Машинное обучение Azure поддерживает широкий спектр моделей регрессии, в дополнение к линейной регрессии. Тем не менее термин «Регрессия» может интерпретироваться слабо и некоторые типы регрессии, в другие инструменты не поддерживаются.

+ Классический регрессии проблема заключается в единый независимой переменной и зависимой переменной. Это называется *простой регрессии*.  Этот модуль поддерживает простой регрессии.

+ *Множественная Линейная регрессия* включает в себя два или несколько независимых переменных, которые влияют на одной зависимой переменной. Проблемы, в которых используются несколько источников входных данных для прогнозирования одного числового результата также называются *многофакторная Линейная регрессия*.

    **Линейной регрессии** модуля можно решения этих проблем, как и большинство других модулей регрессии.

+ *Многометочной регрессией* является задача прогнозирования нескольких зависимых переменных в рамках одной модели. Например в многометочной логистической регрессии пример можно назначить нескольким разным меткам. (Это отличается от задача прогнозирования нескольких уровнях в рамках одного класса переменной.)

    Этот тип регрессии в машинном обучении Azure не поддерживается. Для прогнозирования нескольких переменных, создайте отдельный ученик для каждого выхода, который требуется спрогнозировать.

Лет специалисты по статистике разрабатывают более усовершенствованные методы регрессии. Это справедливо даже для линейной регрессии. Этот модуль поддерживает два метода для измерения ошибок и соответствуют линии регрессии: метод наименьших квадратов и метод градиентного спуска.

- **Градиентный спуск** — это метод, который сводит к минимуму количество ошибок на каждом этапе процесса обучения модели. Существует множество вариантов градиентного спуска, и его оптимизацию для различных проблем обучения тщательно изучили. Если выбран этот параметр для **метод решения**, можно установить различные параметры для управления размером шага, скоростью обучения и т. д. Этот параметр также поддерживает использование встроенной гиперпараметров.

- **Обычный метод наименьших квадратов** является одним из наиболее часто используемые методы в моделях линейной регрессии. Например метод наименьших квадратов — это метод, который используется в пакете анализа для Microsoft Excel.

    Обычный метод наименьших квадратов ссылается на функцию потерь, которая вычисляет погрешность как сумму квадрата расстояние от фактического значения и спрогнозированной линии и подгоняет модель, минимизируя квадратичную погрешность. Этот способ предполагает прочную линейную связь между входными данными и зависимой переменной.

## <a name="configure-linear-regression"></a>Настройка линейной регрессии

Этот модуль поддерживает два метода для подгонки модель регрессии с различными параметрами.

+ [Создание модели регрессии с помощью метода вероятностного градиентного спуска](#bkmk_GradientDescent)

    Градиентный спуск представляет собой улучшенную функцию потерь для моделей, являются более сложными, или в которых слишком мало обучающих данных, учитывая количество переменных.



+ [Вписать модели регрессии с помощью обычного метода наименьших квадратов](#bkmk_OrdinaryLeastSquares)

    Для небольших наборов данных лучше обычный метод наименьших квадратов. Это следует получить аналогичные результаты в Excel.

## <a name="bkmk_OrdinaryLeastSquares"></a> Создание модели регрессии с помощью обычного метода наименьших квадратов

1. Добавить **модели линейной регрессии** эксперимент в интерфейсе модуль.

    Можно найти этот модуль в **машинного обучения** категории. Разверните **Initialize Model**, разверните **регрессии**, а затем перетащите **модели линейной регрессии** эксперимент модуль.

2. В **свойства** области в **метод решения** раскрывающемся списке выберите **обычного метода наименьших квадратов**. Этот параметр определяет метод вычисления, используемое для поиска линии регрессии.

3. В **вес регуляризации L2**, введите значение для использования в качестве веса для регуляризации L2. Мы рекомендуем использовать ненулевое значение во избежание чрезмерно высокой точности.

     Дополнительные сведения о влиянии регуляризации Подгонка модели, см. в статье: [Унификация L1 и L2 для машинного обучения](https://msdn.microsoft.com/magazine/dn904675.aspx)

4. Выберите параметр **Include определяющий смещение член**, если вы хотите просмотреть термин регрессии.

    Снимите этот флажок, если вам не нужно просмотреть формулу регрессии.

5. Для **начальное значение случайного числа**, при необходимости можно ввести начальное значение для генератора случайных чисел, используемого моделью.

    С помощью начального значения полезно в том случае, если вы хотите обеспечить одинаковые результаты в различных запусках и того же эксперимента. В противном случае — значение по умолчанию является использование значения из системных часов.


7. Добавить [Обучение модели](./train-model.md) вашего эксперимента и подключите набор данных с меткой.

8. Запустите эксперимент.

+ Чтобы получить прогноз, подключение обученной модели для [Score Model](./score-model.md) модуль, а также набор новых значений. 


## <a name="bkmk_GradientDescent"></a> Создание модели регрессии с помощью метода вероятностного градиентного спуска

1. Добавить **модели линейной регрессии** эксперимент в интерфейсе модуль.

    Можно найти этот модуль в **машинного обучения** категории. Разверните **Initialize Model**, разверните **регрессии**и перетащите **модели линейной регрессии** эксперимент модуль

2. В **свойства** области в **метод решения** раскрывающегося списка выберите **метода вероятностного градиентного спуска** как метод вычисления, используемое для поиска линии регрессии.

3. Для **Создание режима учителя**, указать, нужно ли выполнять Обучение модели с помощью предопределенного набора параметров или если вы хотите оптимизировать модель с помощью очистку параметров.

    + **Единый параметр**: Если вы знаете, как вы хотите настроить сеть линейной регрессии, вы можете предоставить определенный ряд значений в качестве аргументов.

   
4. Для **скорости обучения**, укажите начальную скорость обучения для оптимизатора вероятностного градиентного спуска.

5. Для **количество эпох**, тип, значение, указывающее, сколько раз алгоритм должен пройти по примерам. Для наборов данных с небольшим числом примеров это число должно быть больше для достижения конвергенции.

6. **Функции нормализации**: Если вы уже нормализуются числовые данные, используемые для обучения модели, можно отменить выбор этого параметра. По умолчанию модуль нормализует все числовые значения в диапазоне от 0 до 1.

    > [!NOTE]
    > 
    > Не забывайте применить один и тот же метод нормализации к новым данным, используемые для оценки.

7. В **вес регуляризации L2**, введите значение для использования в качестве веса для регуляризации L2. Мы рекомендуем использовать ненулевое значение во избежание чрезмерно высокой точности.

    Дополнительные сведения о влиянии регуляризации Подгонка модели, см. в статье: [Унификация L1 и L2 для машинного обучения](https://msdn.microsoft.com/magazine/dn904675.aspx)


9. Выберите параметр **уменьшить скорость обучения**, если вы хотите скорость обучения, чтобы уменьшить в ходе итерации.  

10. Для **начальное значение случайного числа**, при необходимости можно ввести начальное значение для генератора случайных чисел, используемого моделью. С помощью начального значения полезно в том случае, если вы хотите обеспечить одинаковые результаты в различных запусках и того же эксперимента.


12. Добавьте набор данных с метками и один из модулей обучения.

    Если вы не используете очистку параметров, используйте [Обучение модели](train-model.md) модуля.

13. Запустите эксперимент.

# <a name="train-models-with-azure-machine-learning"></a>Обучение моделей с помощью Машинного обучения Azure

Azure Machine Learning предоставляет несколько способов обучения моделей, от кодовых первых решений с использованием SDK до низкокодовых решений, таких как автоматизированное машинное обучение и визуальный дизайнер. Используйте следующий список, чтобы определить, какой метод обучения подходит именно вам:

+ [Azure Machine Learning SDK для Python](#python-sdk): Python SDK предоставляет несколько способов обучения моделей, каждый из которых имеет различные возможности.

    | Метод обучения | Описание |
    | ----- | ----- |
    | [Конфигурация запуска](#run-configuration) | **Общий способ обучения моделей** заключается в использовании сценария обучения и конфигурации выполнения. Конфигурация запуска предоставляет информацию, необходимую для настройки учебной среды, используемой для обучения модели. Можно взять конфигурацию запуска, сценарий обучения и цель вычислений (учебная среда) и запустить учебную работу. |
    | [Автоматизированное машинное обучение](#automated-machine-learning) | Автоматизированное машинное обучение позволяет **обучать модели без обширных знаний о данных или программирования.** Для людей с наукой о данных и программирования фон, он обеспечивает способ сэкономить время и ресурсы путем автоматизации выбора алгоритма и гиперпараметр настройки. Вам не придется беспокоиться об определении конфигурации выполнения при использовании автоматизированного машинного обучения. |
    | [Оценщики](#estimators) | Классы оценщиков **упрощают обучение моделей на основе популярных рамок машинного обучения.** Есть оценщик классов для **Scikit-учиться**, **PyTorch**, **TensorFlow**, и **Chainer**. Существует также общий оценщик, который может быть использован с инфраструктурами, которые еще не имеют выделенного класса оценщика. Вам не придется беспокоиться об определении конфигурации выполнения при использовании оценщиков. |
    | [Конвейер машинного обучения](#machine-learning-pipeline) | Трубопроводы не являются другим методом обучения, а **способом определения рабочего процесса с использованием модульных, многоразовых шагов,** которые могут включать обучение как часть рабочего процесса. Конвейеры машинного обучения поддерживают сярют с помощью автоматизированного машинного обучения, оценщиков и конфигурации выполнения для обучения моделей. Поскольку трубопроводы не ориентированы конкретно на подготовку, причины использования трубопровода более разнообразны, чем другие методы обучения. Как правило, можно использовать конвейер, когда:<br>Вы хотите **запланировать неучтенные процессы,** такие как длительные учебные задания или подготовка данных.<br>Используйте **несколько шагов,** которые координируются через неоднородные вычислительные ресурсы и места хранения.<br>Использовать конвейер в качестве **многоразового шаблона** для конкретных сценариев, таких как переподготовка или пакетный скоринг.<br>* **Отслеживайте и версии источников данных, входов и выходов** для рабочего процесса.<br>Ваш рабочий процесс **реализуется различными группами, которые работают над определенными шагами независимо.** Затем шаги могут быть объединены в конвейер для реализации рабочего процесса. |

+ [Azure Machine Learning SDK для Python:](#r-sdk)SDK использует пакет сетчатой связи с Python SDK от Azure Machine Learning. Это позволяет получить доступ к основным объектам и методам, реализованным в Python SDK, из любой среды R.

+ **Дизайнер:** Дизайнер машинного обучения Azure (предварительный просмотр) обеспечивает легкую точку входа в машинное обучение для создания доказательств концепций или для пользователей с небольшим опытом кодирования. Это позволяет обучать модели с помощью перетаскивания и падения веб-uI. Вы можете использовать код Python как часть дизайна или поезд модели без написания какого-либо кода.

+ **CLI**: Машинное обучение CLI предоставляет команды для общих задач с Azure Machine Learning и часто используется для **выполнения сценариев и автоматизации задач.** Например, после создания скрипа или конвейера обучения можно использовать CLI для запуска учебного запуска по расписанию или при обновлении файлов данных, используемых для обучения. Для моделей обучения, он предоставляет команды, которые представляют обучение рабочих мест. Он может отправлять задания с помощью конфигураций выполнения или конвейеров.

Каждый из этих методов обучения может использовать различные типы вычислительных ресурсов для обучения. В совокупности эти ресурсы называются [__вычислительными целями.__](concept-azure-machine-learning-architecture.md#compute-targets) Целью вычислений может быть локальная машина или облачный ресурс, например, Azure Machine Learning Compute, Azure HDInsight или удаленная виртуальная машина.

### <a name="run-configuration"></a>Конфигурация запуска

Общее учебное задание с Azure Machine Learning можно определить с помощью [RunConfiguration.](https://docs.microsoft.com/python/api/azureml-core/azureml.core.runconfiguration?view=azure-ml-py) Затем используется конфигурация выполнения, а также сценарий обучения (ы) для обучения модели на вычислительной цели.

Вы можете начать с конфигурации выполнения для локального компьютера, а затем переключиться на один для облачной вычислительной цели по мере необходимости. При изменении цели вычислений изменяется только используемая конфигурация выполнения. Запуск также регистрирует информацию о заданиях обучения, например, входы, выходы и журналы.

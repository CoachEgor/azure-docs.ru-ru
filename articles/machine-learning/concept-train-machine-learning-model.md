# <a name="train-models-with-azure-machine-learning"></a>Обучение моделей с помощью Машинного обучения Azure

Машинное обучение Azure предоставляет несколько способов обучения моделей, от первых решений, использующих пакет SDK, до небольших кодовых решений, таких как автоматизированное машинное обучение и визуальный конструктор. Используйте следующий список, чтобы определить, какой метод обучения подходит вам:

+ [Пакет SDK для машинное обучение Azure для Python](#python-sdk): пакет SDK для Python предоставляет несколько способов обучения моделей, каждый из которых имеет различные возможности.

    | Метод обучения | Описание |
    | ----- | ----- |
    | [Конфигурация запуска](#run-configuration) | **Универсальный способ обучения моделей** — использование обучающего скрипта и конфигурации запуска. Конфигурация запуска предоставляет сведения, необходимые для настройки среды обучения, используемой для обучения модели. Вы можете выполнить настройку запуска, сценарий обучения и целевой объект вычислений (среда обучения) и запустить задание обучения. |
    | [Автоматическое машинное обучение](#automated-machine-learning) | Автоматическое машинное обучение позволяет **обучать модели без обширных данных или знаний по программированию**. Для людей с обработкой и обработкой данных и программирования это позволяет экономить время и ресурсы путем автоматизации выбора алгоритма и настройки параметров. При использовании автоматического машинного обучения не нужно беспокоиться об определении конфигурации запуска. |
    | [Интуицией](#estimators) | Классы оценщиков упрощают **обучение моделей на основе популярных платформ машинного обучения**. Существуют классы средства оценки для **Scikit-изучать**, **PyTorch**, **TensorFlow**и **chain**. Также существует универсальный механизм оценки, который можно использовать с платформами, у которых еще нет выделенного класса средства оценки. Не нужно беспокоиться об определении конфигурации запуска при использовании средств оценки. |
    | [Конвейер машинного обучения](#machine-learning-pipeline) | Конвейеры — это не другой метод обучения, но **способ определения рабочего процесса с помощью модульных, многократно используемых действий**, которые могут включать обучение как часть рабочего процесса. Конвейеры машинного обучения поддерживают использование автоматизированного машинного обучения, средств оценки и настройки запуска для обучения моделей. Поскольку конвейеры не сосредоточены на обучении, причины использования конвейера более разнообразны, чем другие методы обучения. Как правило, конвейер можно использовать в следующих случаях:<br>* Необходимо **запланировать автоматические процессы** , такие как длительное выполнение заданий обучения или подготовка данных.<br>* Используйте **несколько шагов** , которые согласовываются между разнородными вычислительными ресурсами и местами хранения.<br>* Использование конвейера в качестве **многократно используемого шаблона** для конкретных сценариев, таких как переобучение или Пакетная оценка.<br>* **Отслеживание и управление версиями, входными и выходными данными** для рабочего процесса.<br>* Рабочий процесс **реализуется различными командами, которые работают с конкретными действиями независимо друг от друга**. Затем можно объединить шаги в конвейере для реализации рабочего процесса. |

+ Пакет SDK [для машинное обучение Azure для Python](#r-sdk): пакет SDK используется для привязки к пакету SDK для python для машинное обучение Azure. Это позволяет получить доступ к основным объектам и методам, реализованным в пакете SDK для Python, из любой среды R.

+ **Конструктор**: машинное обучение Azure Designer (Предварительная версия) предоставляет простую точку входа в машинное обучение для создания экспериментов или для пользователей с небольшим опытом программирования. Он позволяет обучить модели с помощью веб-интерфейса перетаскивания. Можно использовать код Python как часть проекта или обучать модели без написания кода.

+ **CLI**. CLI машинного обучения предоставляет команды для распространенных задач с машинное обучение Azure и часто используется для **создания скриптов и автоматизации задач**. Например, после создания обучающего скрипта или конвейера вы можете использовать интерфейс командной строки для запуска обучения по расписанию или при обновлении файлов данных, используемых для обучения. Для учебных моделей предоставляются команды, которые отправляют задания обучения. Он может отправлять задания с помощью конфигураций запуска или конвейеров.

Каждый из этих методов обучения может использовать различные типы ресурсов вычислений для обучения. В совокупности эти ресурсы называются [__целевыми объектами вычислений__](concept-azure-machine-learning-architecture.md#compute-targets). Целевым объектом вычислений может быть локальный компьютер или облачный ресурс, например Машинное обучение Azureное вычисление, Azure HDInsight или удаленная виртуальная машина.

### <a name="run-configuration"></a>Конфигурация запуска

Универсальное обучающее задание с Машинное обучение Azure можно определить с помощью [RunConfiguration](https://docs.microsoft.com/python/api/azureml-core/azureml.core.runconfiguration?view=azure-ml-py). Затем используется конфигурация запуска, а также скрипты обучения для обучения модели на целевом объекте вычислений.

Вы можете начать с конфигурации запуска для локального компьютера, а затем переключиться на одну облачную целевую платформу вычислений по мере необходимости. При изменении целевого объекта вычислений изменяется только используемая конфигурация запуска. В ходе выполнения также записываются сведения о задании обучения, такие как входные данные, выходные данные и журналы.

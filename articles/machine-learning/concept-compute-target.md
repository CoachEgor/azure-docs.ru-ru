#  <a name="what-are-compute-targets-in-azure-machine-learning"></a>Каковы цели вычислений в машинном обучении Azure? 

Вычислительная **цель** — это назначенный вычислительный ресурс/среда, где вы запустите сценарий обучения или размещаете развертывание службы. Это место может быть локальной машиной или облачным вычислительным ресурсом. Использование вычислительных целей позволяет вам впоследствии изменить среду вычислений без необходимости изменения кода.  

В типичном жизненном цикле разработки модели можно:
1. Начните с разработки и экспериментирования на небольшом объеме данных. На этом этапе мы рекомендуем локальной среде (локальный компьютер или облачный VM) в качестве вычислительной цели. 
2. Масштабируйте масштабирование до больших данных или делайте распределенное обучение с помощью одной из этих [целей обучения.](#train)  
3. Как только модель будет готова, разместите ее в среде веб-хостинга или устройстве IoT с одной из этих [целей вычислений развертывания.](#deploy)

Вычислительные ресурсы, которые используются для целевых показателей вычислений, прикрепляются к [рабочему пространству.](concept-workspace.md) Вычислить ресурсы, кроме локальной машины, совместно используются пользователями рабочего пространства.

## <a name="azure-machine-learning-compute-managed"></a>Вычислить машинного обучения Azure (управляемый)

Управляемый вычислительный ресурс создается и управляется Azure Machine Learning. Этот компьютер оптимизирован для рабочих нагрузок машинного обучения. Вычислительные кластеры и [вычислительные экземпляры](concept-compute-instance.md) Azure Machine Learning являются единственными управляемыми вычислениями. В будущем могут быть добавлены дополнительные управляемые вычислительные ресурсы.

Можно создать экземпляры вычислений машин Azure Machine Learning (предварительный просмотр) или вычислить кластеры в:

| | Студия машинного обучения Azure. | Портал Azure | SDK | Шаблон Resource Manager | CLI |
|---| ----- | ----- | ----- | ----- | ----- |
| Вычислительная операция | да | да | да | да |  |
| Вычислительный кластер | да | да | да | да | да |

При создании эти вычислительные ресурсы автоматически являются частью рабочего пространства в отличие от других видов вычислительных целей.

### <a name="compute-clusters"></a>Вычислительные кластеры

Вы можете использовать вычислительные кластеры Azure Machine Learning для обучения и для сбора пакетов (предварительный просмотр).  С помощью этого вычислительного ресурса у вас есть:

* Одно- или многоузловой кластер
* Автомасштабы каждый раз, когда вы отправляете пробег 
* Автоматическое управление кластерами и планирование заданий 
* поддерживает ресурсы ЦП и GPU;



## <a name="unmanaged-compute"></a>Неуправляемые вычисления

Неуправляемая вычислительная цель *не* управляется программным управлением Azure Machine Learning. Вы создаете этот тип вычислительной цели за пределами машинного обучения Azure, а затем прикрепляете ее к рабочему пространству. Неуправляемые вычислительные ресурсы могут потребовать дополнительных шагов для поддержания или повышения производительности рабочих нагрузок машинного обучения.

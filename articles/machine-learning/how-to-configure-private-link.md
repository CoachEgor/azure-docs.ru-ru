# <a name="configure-azure-private-link-for-an-azure-machine-learning-workspace"></a>Научите частную ссылку Azure для рабочего пространства для обучения машин Azure

В этом документе вы узнаете, как использовать частную ссылку Azure с рабочим пространством машинного обучения Azure. Частная ссылка Azure позволяет подключиться к рабочему пространству с помощью частной конечной точки. Частная конечная точка представляет собой набор частных IP-адресов в вашей виртуальной сети. Затем можно ограничить доступ к рабочему пространству только через частные IP-адреса. Private Link помогает снизить риск эксфильтрации данных. Подробнее о частных конечных точках читайте в статье [Azure Private Link.](/azure/private-link/private-link-overview)

> [!IMPORTANT]
> Частная ссылка Azure не влияет на плоскость управления Azure (операции управления), такие как удаляние рабочего пространства или управление вычислительными ресурсами. Например, создание, обновление или утяжение цели вычислений. Эти операции выполняются через общедоступный Интернет в обычном режиме.
>
> Предварительный просмотр экземпляров Azure Machine Learning не поддерживается в рабочей области, где включена приватная ссылка.

## <a name="create-a-workspace-that-uses-a-private-endpoint"></a>Создание рабочего пространства, используюго частную конечную точку

В настоящее время мы поддерживаем только создание частной конечных точек при создании нового рабочего пространства машинного обучения Azure. Следующие шаблоны предусмотрены для нескольких популярных конфигураций:

> [!TIP]
> Автоматическое утверждение управляет автоматизированным доступом к ресурсу с включенным частным соединением. Для получения дополнительной информации [смотрите, что такое служба частной ссылки Azure](../private-link/private-link-service-overview.md).

* [Рабочее пространство с ключами, управляемыми клиентом, и автоматическое одобрение для Private Link](#cmkaapl)
* [Рабочее пространство с ключами, управляемыми клиентом, и ручным утверждением для Private Link](#cmkmapl)
* [Рабочее пространство с ключами, управляемыми корпорацией Майкрософт, и автоматическое утверждение для Частной ссылки](#mmkaapl)
* [Рабочее пространство с ключами, управляемыми корпорацией Майкрософт, и ручным утверждением для Частной ссылки](#mmkmapl)

При развертывании шаблона необходимо предоставить следующую информацию:

* имя рабочей области.
* Azure региона для создания ресурсов в
* Рабочее издание (Основное или пределение)
* Если следует ввести высокие параметры конфиденциальности рабочего пространства
* Если шифрование рабочего пространства с ключом, управляемым клиентом, должно быть включено, а связанные значения для ключа
* Название и подсеть виртуальной сети
* Имя подсети

После того, как шаблон был представлен и подготовка завершена, группа ресурсов, содержащая рабочее пространство, будет содержать три новых типа артефактов, связанных с Private Link:

* Частная конечная точка
* сетевому интерфейсу
* Частная зона DNS

Рабочее пространство также содержит виртуальную сеть Azure, которая может общаться с рабочей областью через частную конечную точку.

### <a name="deploy-the-template-using-the-azure-portal"></a>Развертывание шаблона с помощью портала Azure

1. Выполните инструкции из раздела [Развертывание ресурсов с помощью настраиваемого шаблона](https://docs.microsoft.com/azure/azure-resource-manager/resource-group-template-deploy-portal#deploy-resources-from-custom-template). Когда вы приедете на экран __шаблона Edit,__ вставьте в один из шаблонов из конца этого документа.
1. Щелкните __Сохранить__, чтобы применить этот шаблон. Укажите описанные ниже сведения и подтвердите согласие с предложенными условиями.

   * Подписка: Выберите подписку Azure для использования для этих ресурсов.
   * Группа ресурсов: Выберите или создайте группу ресурсов для хранения служб.
   * Название рабочего пространства: имя, которое будет использоваться для создаваемого рабочего пространства Azure Machine Learning. Для имени рабочего пространства допускается длина от 3 до 33 символов. Имя может содержать только буквы, цифры и дефис ("-").
   * Местонахождение: Выберите место, где будут созданы ресурсы.

Дополнительные сведения см. в разделе [Развертывание ресурсов с помощью настраиваемого шаблона](../azure-resource-manager/templates/deploy-portal.md#deploy-resources-from-custom-template).

## <a name="using-a-workspace-over-a-private-endpoint"></a>Использование рабочей области над частной конечной точкой

Поскольку связь с рабочим пространством разрешена только из виртуальной сети, любые среды разработки, которые используют рабочее пространство, должны быть членами виртуальной сети. Например, виртуальная машина в виртуальной сети или машина, подключенная к виртуальной сети с помощью VPN-шлюза.

> [!IMPORTANT]
> Чтобы избежать временного нарушения подключения, корпорация Майкрософт рекомендует промывки кэша DNS на машинах, подключенных к рабочему пространству, после включения Private Link. 

Для получения информации о виртуальных машинах Azure [см.](/azure/virtual-machines/)

Для получения информации о ШЛюзах VPN, смотрите [Что такое VPN шлюз](/azure/vpn-gateway/vpn-gateway-about-vpngateways).

## <a name="using-azure-storage"></a>Использование службы хранилища Azure

Чтобы обезопасить учетную запись хранения Azure, используемую рабочим пространством, поместите ее в виртуальную сеть.

Для получения информации о размещении учетной записи хранилища в виртуальной сети [см.](how-to-enable-virtual-network.md#use-a-storage-account-for-your-workspace)

## <a name="using-azure-key-vault"></a>Использование Azure Key Vault

Чтобы обезопасить хранилище ключей Azure, используемое в рабочем пространстве, вы можете поместить его в виртуальную сеть или включить для него Private Link.

Для получения информации о размещении хранилища ключей в виртуальной сети [см.](how-to-enable-virtual-network.md#use-a-key-vault-instance-with-your-workspace)

Для получения информации о включении Private Link для хранилища ключей можно ознакомиться на примере [хранилища ключей Integrate Key Vault с помощью Частной ссылки Azure.](/azure/key-vault/private-link-service)

## <a name="using-azure-kubernetes-services"></a>Использование служб Azure Kubernetes

Чтобы обезопасить службы Azure Kubernetes, используемые рабочим пространством, поместите их в виртуальную сеть. Для получения дополнительной информации [см.](how-to-enable-virtual-network.md#aksvnet)

> [!WARNING]
> Azure Machine Learning не поддерживает использование службы Azure Kubernetes с включенной частной ссылкой.

## <a name="azure-container-registry"></a>Реестр контейнеров Azure

Для получения информации о безопасности реестра контейнеров Azure в виртуальной сети можно получить информацию о регистрации [контейнеров Azure.](how-to-enable-virtual-network.md#use-azure-container-registry)

> [!IMPORTANT]
> Если вы используете Private Link для рабочего пространства Azure Machine Learning и размещаете реестр контейнеров Azure для рабочего пространства в виртуальной сети, необходимо также применить следующий шаблон менеджера ресурсов Azure. Этот шаблон позволяет вашему рабочему пространству общаться с ACR по частной ссылке.

```json
{
  "$schema": "https://schema.management.azure.com/schemas/2015-01-01/deploymentTemplate.json#",
  "contentVersion": "1.0.0.0",
  "parameters": {
      "keyVaultArmId": {
      "type": "string"
      },
      "workspaceName": {
      "type": "string"
      },
      "containerRegistryArmId": {
      "type": "string"
      },
      "applicationInsightsArmId": {
      "type": "string"
      },
      "storageAccountArmId": {
      "type": "string"
      },
      "location": {
      "type": "string"
      }
  },
  "resources": [
      {
      "type": "Microsoft.MachineLearningServices/workspaces",
      "apiVersion": "2019-11-01",
      "name": "[parameters('workspaceName')]",
      "location": "[parameters('location')]",
      "identity": {
          "type": "SystemAssigned"
      },
      "sku": {
          "tier": "enterprise",
          "name": "enterprise"
      },
      "properties": {
          "sharedPrivateLinkResources":
  [{"Name":"Acr","Properties":{"PrivateLinkResourceId":"[concat(parameters('containerRegistryArmId'), '/privateLinkResources/registry')]","GroupId":"registry","RequestMessage":"Approve","Status":"Pending"}}],
          "keyVault": "[parameters('keyVaultArmId')]",
          "containerRegistry": "[parameters('containerRegistryArmId')]",
          "applicationInsights": "[parameters('applicationInsightsArmId')]",
          "storageAccount": "[parameters('storageAccountArmId')]"
      }
      }
  ]
}
```

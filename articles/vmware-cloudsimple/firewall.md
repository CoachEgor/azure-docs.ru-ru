# <a name="set-up-firewall-tables-and-rules-for-private-clouds"></a>Настройка таблиц и правил брандмауэра для частных облаков

Таблицы брандмауэров и связанные с ними правила позволяют указать ограничения на трафик, которые применяются к определенным подсетям и VLAN.

* Подсеть может быть связана с одной таблицей брандмауэра.
* Таблица брандмауэра может быть связана с несколькими подсетями.

## <a name="add-a-new-firewall-table"></a>Добавление новой таблицы брандмауэра

1. [Получите доступ к порталу CloudSimple](access-cloudsimple-portal.md) и выберите **сеть** в боковом меню.
2. Выберите **таблицы брандмауэра**.
3. Выберите **Создать таблицу брандмауэра.**

    ![Страница VLAN/subnet](media/firewall-tables-page.png)

4. Введите имя для таблицы.
5. В списке значатся правило по умолчанию для таблицы. Нажмите **Создать новое правило,** чтобы создать дополнительное правило. Подробности приведены в следующей процедуре.
6. Нажмите **Сделано,** чтобы сохранить таблицу брандмауэра.

> [!IMPORTANT]
> Можно создать до двух таблиц брандмауэра в частном облаке.

## <a name="firewall-rules"></a>Правила брандмауэра

Правила брандмауэра определяют, как брандмауэр относится к определенным типам трафика. Вкладка **Правила** для выбранной таблицы брандмауэра перечисляет все связанные с этим правила.

![Таблица правил брандмауэра](media/firewall-rules-tab.png)

## <a name="create-a-firewall-rule"></a>Создание правила брандмауэра

1. Отобразите настройки для создания правила брандмауэра в любом из этих способов:
    * Нажмите **Добавить правило** при создании таблицы брандмауэра.
    * Выберите определенную таблицу брандмауэра на странице **Network > Firewall Tables** и нажмите **«Создать новое правило брандмауэра».**
2. Настройка правила следующим образом:
    * **Имя**. Дайте правилу имя.
    * **Приоритет**. Присвоите приоритет правилу. Правила с меньшими числами выполняются в первую очередь.
    * **Тип трафика**. Выберите, предназначено ли это правило для частного облака, Интернета или VPN-трафика (без гражданства) или для общедоступного IP-адреса (stateful).
    * **Протокол**. Выберите протокол, охватываемый правилом (TCP, UDP или любой протокол).
    * **Направление**. Выберите, предназначено ли правило для входящего или исходящего трафика. Необходимо определить отдельные правила для входящего и исходящего трафика.
    * **Действие**. Выберите действие, если правило совпадает (позволяет или отрицает).
    * **Источник**. Укажите источники, охватываемые правилом (блок CIDR, внутренний или любой источник).
    * **Диапазон исходного порта**. Укажите диапазон портов, подпадающих под правило.
    * **Направление**. Выберите входящий или исходящий.
    * **Пункт назначения**. Укажите направления, охватываемые правилом (блок CIDR, внутренний или любой источник).
    * **Диапазон исходного порта**. Укажите диапазон портов, подпадающих под правило.

    ![Правило добавления таблицы брандмауэра](media/firewall-rule-create.png)

3. Нажмите **«Сделано»,** чтобы сохранить правило, и добавьте его в список правил для таблицы брандмауэра.

> [!IMPORTANT]
> Каждая таблица Firewall может иметь до 10 входящих правил и 20 исходящих правил. Эти ограничения могут быть увеличены, [обратившись в службу поддержки.](https://portal.azure.com/#blade/Microsoft_Azure_Support/HelpAndSupportBlade/newsupportrequest)

## <a name="attach-vlanssubnets"></a><a name="attach-vlans-subnet"></a>Прикрепите VLANs/подсети

После определения таблицы брандмауэра можно указать подсети, на которые распространяются правила в таблице.

1. На странице**таблицы брандмауэра** **Сети** > выберите таблицу брандмауэра.
2. Откройте **прилагаемую вкладку VLANs/Subnet.**
3. Нажмите **Прикрепите к VLAN/Subnet**.
4. Выберите частное облако и VLAN. Отображается связанное название подсети и блок CIDR.
5. Щелкните элемент **Отправить**.

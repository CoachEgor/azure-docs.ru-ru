# <a name="how-to-secure-your-avs-private-cloud-environment"></a>Как защитить среду частного облака AVS

Определение управления доступом на основе ролей (RBAC) для службы AVS, портала AVS и частного облака AVS из Azure. Пользователи, группы и роли для доступа к vCenter из частного облака AVS указываются с помощью единого входа VMware. 

## <a name="rbac-for-avs-service"></a>RBAC для службы AVS

Для создания службы AVS требуется роль **владельца** или **участника** в подписке Azure. По умолчанию все владельцы и участники могут создать службу AVS и получить доступ к порталу AVS для создания частных облаков AVS и управления ими. Для каждого региона можно создать только одну службу AVS. Чтобы ограничить доступ определенным администраторам, выполните описанную ниже процедуру.

1. Создание службы AVS в новой **группе ресурсов** на портал Azure
2. Укажите RBAC для группы ресурсов.
3. Приобретение узлов и использование той же группы ресурсов, что и служба AVS

Только пользователи, имеющие права **владельца** или **участника** в группе ресурсов, увидят службу AVS и запустили портал AVS.

Дополнительные сведения о RBAC см. в статье [что такое управление доступом на основе ролей (RBAC) для ресурсов Azure](../role-based-access-control/overview.md).

## <a name="rbac-for-avs-private-cloud-vcenter"></a>RBAC для частного облака AVS

При создании частного облака AVS в домене единого входа vCenter создается `CloudOwner@AVS.local` пользователя по умолчанию. У пользователя Клаудовнер есть права на управление vCenter. Дополнительные источники удостоверений добавляются в службу единого входа vCenter для предоставления доступа разным пользователям. Предварительно определенные роли и группы настраиваются в vCenter, который можно использовать для добавления дополнительных пользователей.

### <a name="add-new-users-to-vcenter"></a>Добавление новых пользователей в vCenter

1. [Эскалировать привилегии](escalate-private-cloud-privileges.md) для **CloudOwner@AVS.local** пользователя в частном облаке AVS.
2. Вход в vCenter с помощью **CloudOwner@AVS.local**
3. [Добавление пользователей с единым входом vCenter](https://docs.vmware.com/en/VMware-vSphere/5.5/com.vmware.vsphere.security.doc/GUID-72BFF98C-C530-4C50-BF31-B5779D2A4BBB.html).
4. Добавление пользователей в [группы единого входа vCenter](https://docs.vmware.com/en/VMware-vSphere/5.5/com.vmware.vsphere.security.doc/GUID-CDEA6F32-7581-4615-8572-E0B44C11D80D.html).

Дополнительные сведения о предопределенных ролях и группах см. в статье [модель разрешений частного облака AVS статьи о VMware vCenter](learn-private-cloud-permissions.md) .

Вы можете добавить дополнительных поставщиков удостоверений для домена единого входа vCenter вашего частного облака AVS. Поставщики удостоверений обеспечивают проверку подлинности и группы единого входа vCenter, обеспечивающие авторизацию для пользователей.

* [Использование Azure AD в качестве поставщика удостоверений](azure-ad.md) в частном ОБЛАКе AVS

1. [Эскалировать привилегии](escalate-private-cloud-privileges.md) для **CloudOwner@AVS.local** пользователя в частном облаке AVS.
2. Вход в vCenter с помощью **CloudOwner@AVS.local**
3. Добавьте пользователей из поставщика удостоверений в [группы единого входа vCenter](https://docs.vmware.com/en/VMware-vSphere/5.5/com.vmware.vsphere.security.doc/GUID-CDEA6F32-7581-4615-8572-E0B44C11D80D.html).

## <a name="secure-network-on-your-avs-private-cloud-environment"></a>Защита сети в среде частного облака AVS

Безопасность сети в частной облачной среде AVS контролируется путем защиты сетевого доступа и управления сетевым трафиком между ресурсами.

### <a name="access-to-avs-private-cloud-resources"></a>Доступ к ресурсам частного облака AVS

Доступ к ресурсам для частного облака AVS и ресурсов через безопасное сетевое подключение:

* **[Подключение ExpressRoute](on-premises-connection.md)** . ExpressRoute обеспечивает безопасное подключение с высокой пропускной способностью и низкой задержкой из локальной среды. Использование подключения позволяет локальным службам, сетям и пользователям получать доступ к службе "частный облако" AVS.
* **[VPN-шлюз типа "сеть — сеть](vpn-gateway.md)** ". VPN типа "сеть — сеть" предоставляет доступ к ресурсам частного облака AVS из локальной среды через безопасный туннель. Вы можете указать, какие локальные сети могут отправлять и получать сетевой трафик в частное облако AVS.
* **[VPN-шлюз типа "точка — сеть](vpn-gateway.md#set-up-a-site-to-site-vpn-gateway)** ". Используйте VPN-подключение типа "точка — сеть" для быстрого удаленного доступа к vCenter закрытого облака AVS.

### <a name="control-network-traffic-in-avs-private-cloud"></a>Управление сетевым трафиком в частном облаке AVS

Таблицы и правила брандмауэра управляют сетевым трафиком в частном облаке AVS. Таблица брандмауэра позволяет управлять сетевым трафиком между исходной сетью или IP-адресом и целевой сетью или IP-адресом в зависимости от сочетания правил, определенных в таблице.

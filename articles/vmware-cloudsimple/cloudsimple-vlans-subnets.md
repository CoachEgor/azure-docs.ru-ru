# <a name="vlans-and-subnets-overview"></a>Обзор VLAN и подсетей

CloudSimple предоставляет сеть в регионе, где развертывается служба CloudSimple.  Сеть представляет собой единое адресное пространство TCP Layer 3 с включенной по умолчанию реуктором.  Все частные облака и подсети, созданные в этом регионе, могут общаться друг с другом без какой-либо дополнительной конфигурации.  Вы можете создавать распределенные группы портов на vCenter с помощью VLANs.

![Топология CloudSimple Сети](media/cloudsimple-network-topology.png)

## <a name="vlans"></a>Виртуальные локальные сети

Для каждого частного облака создается VLAN (сеть layer 2).  Трафик уровня 2 находится в пределах границ частного облака, что позволяет изолировать локальный трафик в частном облаке.  VLAN, созданный в частном облаке, может использоваться для создания распределенных групп портов только в этом частном облаке.  Созданный vLAN в частном облаке автоматически настраивается на все коммутаторы, подключенные к хостам частного облака.

## <a name="subnets"></a>Подсети

Вы можете создать подсеть при создании VLAN, определив адресное пространство подсети. IP-адрес из пространства адреса присваивается в качестве шлюза подсети. На каждого клиента и регион назначается одно частное адресное пространство уровня 3. Вы можете настроить любое неперекрывающееся адресное пространство RFC 1918 с помощью своей предприимчной сети или виртуальной сети Azure в вашем сетевом регионе.

Все подсети могут общаться друг с другом по умолчанию, уменьшая накладные расходы на конфигурацию для реуктора между частными облаками. Данные между Востоком и Западом на ПК в том же регионе остаются в одной сети уровня 3 и передаются по локальной сетевой инфраструктуре в регионе. Для связи между частными облаками в регионе не требуется никакого выходного входа. Этот подход устраняет любой штраф за производительность WAN/egress при развертывании различных рабочих нагрузок в различных частных облаках.

## <a name="vspherevsan-subnets-cidr-range"></a>vSphere/vSAN подсети CIDR диапазон

Частное облако создается как изолированный стек VMware (esXi хосты, vCenter, vSAN и NSX) среда, управляемая сервером vCenter.  Компоненты управления развертываются в сети, выбранной для подсетей vSphere/vSAN CIDR.  Диапазон сети CIDR делится на различные подсети во время развертывания.

* Минимальный vSphere/vSAN подсети CIDR диапазон префикс: **/24**
* Максимальная префикс диапазона диапазона vSphere/vSAN: **/21**

> [!CAUTION]
> IP-адреса в диапазоне vSphere/vSAN CIDR зарезервированы для использования инфраструктурой Private Cloud.  Не используйте IP-адрес в этом диапазоне на любой виртуальной машине.

### <a name="vspherevsan-subnets-cidr-range-limits"></a>vSphere/vSAN подсети CIDR пределы диапазона

Выбор подсетей vSphere/vSAN CIDR влияет на размер вашего частного облака.  В следующей таблице отображается максимальное количество узлов, которые вы можете иметь в зависимости от размера подсетей vSphere/vSAN CIDR.

| Уточненная длина префикса vSphere/vSAN CIDR | Максимальное количество узлов |
|---------------------------------------------------|-------------------------|
| /24 | 26 |
| /23 | 58 |
| /22 | 118 |
| /21 | 220 |

### <a name="management-subnets-created-on-a-private-cloud"></a>Подсети управления, созданные в частном облаке

При создании частного облака создаются следующие подсети управления.

* **Управление системой**. VLAN и подсеть для сети управления хостов ESXi, DNS-сервер, сервер vCenter.
* **VMotion**. VLAN и подсеть для сети vMotion хостов ESXi.
* **VSAN**. VLAN и подсеть для сети vSAN хостов ESXi.
* **NsxtEdgeUplink1**. VLAN и подсеть для VLAN uplinks к внешней сети.
* **NsxtEdgeUplink2**. VLAN и подсеть для VLAN uplinks к внешней сети.
* **NsxtEdgeTransport**. VLAN и подсеть для транспортных зон контролируют охват сетей уровня 2 в NSX-T.
* **NsxtHostTransport**. VLAN и подсеть для транспортной зоны.

# <a name="vlans-and-subnets-overview"></a>Общие сведения о виртуальных локальных сетей и подсетей

CloudSimple предоставляет сети на регион, где развернута служба CloudSimple.  Сеть не одно пространство адресов TCP уровня 3 с маршрутизацией включен по умолчанию.  Все частные облака и подсетей, созданные в данном регионе могут взаимодействовать друг с другом без дополнительной настройки.  Можно создать группы портов, распределенных на vCenter, используя виртуальные локальные сети.

![Топология сети CloudSimple](media/cloudsimple-network-topology.png)

## <a name="vlans"></a>Виртуальные локальные сети

Виртуальные локальные сети (network уровня 2) создаются для каждого частного облака.  Трафик уровня 2, гарантирующие частного облака, чтобы изолировать локальный трафик в частном облаке.  Виртуальной локальной сети, созданные в частном облаке, можно использовать для создания групп распределенных порт только в этом частном облаке.  Виртуальной локальной сети, созданные в частном облаке настраивается автоматически для всех коммутаторов, подключенных к узлам частного облака.

## <a name="subnets"></a>Подсети

При создании виртуальной локальной сети, определив адресное пространство подсети, можно создать подсеть. IP-адрес из адресного пространства назначается в качестве подсети шлюза. Одно пространство частных адресов третьего уровня назначается каждому клиенту и региону. RFC 1918 неперекрывающихся адресное пространство, можно настроить в локальной сети или виртуальной сети Azure, в вашем регионе сети.

Все подсети могут взаимодействовать друг с другом по умолчанию, уменьшая конфигурации затраты для маршрутизации между частных облаков. Восток Запад данных между компьютерами в одном регионе остается в той же сети третьего уровня и передает по инфраструктуре локальной сети в регионе. Нет исходящий трафик необходим для связи между частных облаков в регионе. Такой подход позволяет избежать снижения производительности глобальной сети и входящего трафика в развертывание разных рабочих нагрузок в различных частных облаках.

## <a name="vspherevsan-subnets-cidr-range"></a>диапазон CIDR подсети vSphere/виртуальной сети хранения данных

Частное облако создается как изолированное стека VMware (ESXi узлов vCenter, виртуальной сети хранения данных и NSX) среде под управлением сервера vCenter.  Компоненты управления развернуты в сети, выбранной для **vSphere или vSAN подсети CIDR**.  Диапазон CIDR сети состоит из разных подсетях во время развертывания.

Префикс диапазона минимальный vSphere или vSAN подсети CIDR: **/24** максимально vSphere или vSAN подсети CIDR диапазон префикс:   **/21**

> [!CAUTION]
> IP-адреса в диапазоне CIDR vSphere или vSAN зарезервированы для использования в инфраструктуре частного облака. Не используйте IP-адрес в этом диапазоне на любой виртуальной машине.


### <a name="vspherevsan-subnets-cidr-range-limits"></a>vSphere или vSAN подсети CIDR диапазон ограничения

Выбор размера диапазона CIDR подсети vSphere или vSAN оказывает влияние на размер вашего частного облака.  В следующей таблице показано максимальное число узлов, которые можно зависимости от размера vSphere или vSAN подсети CIDR.

| Длина префикса заданных подсетей vSphere или vSAN CIDR | Максимальное количество узлов |
|---------------------------------------------------|-------------------------|
| /24 | 26 |
| /23 | 58 |
| /22 | 118 |
| /21 | 220 |

### <a name="management-subnets-created-on-a-private-cloud"></a>Подсети управления, создать для частного облака

Следующие подсети управления создаются при создании частного облака. 

* **Управление системой** -виртуальной локальной сети и подсеть для управления узлами ESXi сети, DNS-сервер, сервер vCenter.
* **VMotion** -виртуальной локальной сети и подсети для сети vMotion узлов ESXi.
* **Виртуальная сеть хранения данных** -виртуальной локальной сети и подсети для сети виртуальной сети хранения данных на узлах ESXi.
* **NsxtEdgeUplink1** -виртуальной локальной сети и подсети для исходящей связи виртуальной локальной сети к внешней сети.
* **NsxtEdgeUplink2** -виртуальной локальной сети и подсети для исходящей связи виртуальной локальной сети к внешней сети.
* **NsxtEdgeTransport** -виртуальной локальной сети и подсети для зон транспорта управления возможностей уровня 2 сетей в NSX-T.
* **NsxtHostTransport** -виртуальной локальной сети и подсети для узла передачи зоны.

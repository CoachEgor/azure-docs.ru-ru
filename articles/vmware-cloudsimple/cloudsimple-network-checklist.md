# <a name="networking-prerequisites-for-azure-vmware-solution-by-cloudsimple"></a>Сетевые предпосылки для решения Azure VMware от CloudSimple

Решение Azure VMware от CloudSimple предлагает частную облачную среду VMware, доступную для пользователей и приложений из закрытых сред, устройств, управляемых предприятием, и ресурсов Azure. Подключение осуществляется через сетевые службы, такие как VPN и соединения Azure ExpressRoute. Некоторые из этих сетевых служб требуют указания диапазонов сетевых адресов для включения служб. 

Таблицы в этой статье описывают набор диапазонов адресов и соответствующие службы, которые используют указанные адреса. Некоторые адреса являются обязательными, а некоторые зависят от служб, которые вы хотите развернуть. Эти адресные пространства не должны пересекаться ни с одной из ваших подсетей, подсетями Виртуальной сети Azure или запланированными подсетями рабочей нагрузки CloudSimple.

## <a name="network-address-ranges-required-for-creating-a-private-cloud"></a>Диапазоны сетевых адресов, необходимых для создания частного облака

При создании сервиса CloudSimple и частного облака необходимо соблюдать указанные диапазоны бесклассовой интер-доменной сети (CIDR) следующим образом.

| Имя/используется для     | Описание                                                                                                                            | Диапазон адресов            |
|-------------------|----------------------------------------------------------------------------------------------------------------------------------------|--------------------------|
| Шлюз CIDR      | Требуется для краевых сервисов (VPN шлюзов).  Этот CIDR необходим во время создания службы CloudSimple и должен быть из пространства RFC 1918. | /28                      |
| vSphere/vSAN CIDR | Требуется для сетей управления VMware. Этот CIDR должен быть указан во время создания частного облака.                                    | /24 или /23 или /22 или /21 |

## <a name="network-address-range-required-for-azure-network-connection-to-an-on-premises-network"></a>Диапазон сетевых адресов, необходимых для подключения сети Azure к предприимческой сети

Подключение от [предприимчой к частной облачной сети через ExpressRoute](on-premises-connection.md) устанавливает соединение Global Reach.  Соединение использует протокол Border Gateway (BGP) для обмена маршрутами между вашей приложечной сетью, частной облачной сетью и сетями Azure.

| Имя/используется для             | Описание                                                                                                                                                                             | Диапазон адресов |
|---------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|---------------|
| ЭкспрессРут Пиринг CIDR | Требуется при использовании ExpressRoute Global Reach для подключения на месте. Этот CIDR должен быть предоставлен, когда запрос на подключение Global Reach осуществляется через билет поддержки. | /29           |

## <a name="network-address-range-required-for-using-a-site-to-site-vpn-connection-to-an-on-premises-network"></a>Диапазон адресов сети, необходимый для использования VPN-соединения от сайта к сайту в предприимки

Подключение от [предварительной сети к частной облачной сети с помощью VPN](vpn-gateway.md) от сайта требует следующих IP-адресов, предварительной сети и идентификаторов. 

| Диапазон адресов/адресов | Описание                                                                                                                                                                                                                                                           |
|-----------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Одноранговый IP               | На-припомещении VPN шлюз публичный IP-адрес. Требуется установить VPN-соединение между центром обработки данных и регионом облачных технологий. Этот IP-адрес необходим во время создания VPN шлюза от сайта к сайту.                                         |
| Одноранговый идентификатор       | Одноранговый идентификатор наместерарователь VPN шлюза. Это, как правило, то же самое, как **равный IP**.  Если уникальный идентификатор указан на вашем предварительном VPN шлюзе, идентификатор должен быть указан.  Одноранговый идентификатор необходим во время создания VPN шлюза от сайта к сайту.   |
| Натерритории сети   | На префиксах, которые нуждаются в доступе к сетям CloudSimple в регионе.  Включите все префиксы из предварительной сети, которая получит доступ к сети CloudSimple, включая клиентскую сеть, откуда пользователи получат доступ к сети.                                         |

## <a name="network-address-range-required-for-using-point-to-site-vpn-connections"></a>Диапазон адресов сети, необходимый для использования VPN-соединений от точки к сайту

VPN-соединение от точки к сайту позволяет получить доступ к сети CloudSimple с клиентской машины.  [Чтобы настроить пункт к сайту VPN,](vpn-gateway.md)необходимо указать следующий диапазон адресов сети.

| Диапазон адресов/адресов | Описание                                                                                                                                                                                                                                                                                                  |
|-----------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Клиентская подсеть         | Адреса DHCP предоставляются клиентской подсетью при подключении с помощью VPN от точки к сайту. Эта подсеть необходима в то время как вы создаете VPN шлюз от точки к сайту на портале CloudSimple.  Сеть разделена на две подсети; один для соединения UDP, а другой для соединений TCP. |

# <a name="firewall-tables-overview"></a>Обзор таблиц брандмауэра

Таблица брандмауэра перечисляет правила фильтрации сетевого трафика на ресурсы Private Cloud и из нее. Вы можете применить таблицы брандмауэра к VLAN/subnet. Правила контролируют сетевой трафик между исходной сетью или IP-адресом и сетью назначения или IP-адресом.

## <a name="firewall-rules"></a>Правила брандмауэра

В следующей таблице описаны параметры в правиле брандмауэра.

| Свойство | Сведения |
| ---------| --------|
| **Название** | Имя, которое однозначно определяет правило брандмауэра и его назначение. |
| **Priority** | Число между 100 и 4096, с 100 является наивысшим приоритетом. Правила обрабатываются в порядке приоритета. Когда трафик сталкивается с правилом, обработка правил останавливается. В результате не обрабатываются правила с более низкими приоритетами, которые имеют те же атрибуты, что и правила с более высокими приоритетами.  Позаботьтесь, чтобы избежать противоречивых правил. |
| **Отслеживание состояния** | Отслеживание может быть без апоссов (частное облако, Интернет или VPN) или stateful (общественный IP).  |
| **Протокол** | Варианты включают любой, TCP или UDP. Если вам нужен ICMP, используйте Any. |
| **Направление** | Определяет тип трафика (входящий или исходящий), к которому применяется правило. |
| **Действие** | Разрешить или отказать в типе трафика, определенного в правиле. |
| **Источник** | IP-адрес, бесклассовый междоменный блок (CIDR) (например, 10.0.0.0/24) или Любой.  Определение диапазона, тега обслуживания или группы безопасности приложений позволяет создавать меньше правил безопасности. |
| **Источник Порт** | Порт, из которого происходит сетевой трафик.  Можно указать отдельный порт или диапазон портов, таких как 443 или 8000-8080. Если указать диапазон, можно создавать меньше правил безопасности. |
| **Назначения** | IP-адрес, бесклассовый междоменный блок (CIDR) (например, 10.0.0.0/24) или Любой.  Определение диапазона, тега обслуживания или группы безопасности приложений позволяет создавать меньше правил безопасности.  |
| **Порт назначения** | Порт, в который течет сетевой трафик.  Можно указать отдельный порт или диапазон портов, таких как 443 или 8000-8080. Если указать диапазон, можно создавать меньше правил безопасности.|

### <a name="stateless"></a>Без отслеживания состояния

Правило без состояния смотрит только на отдельные пакеты и фильтрует их на основе правила.  
Для движения в обратном направлении могут потребоваться дополнительные правила.  Используйте правила без гражданства для движения между следующими точками:

* Подсети частных облаков
* Подсеть и подсеть Private Cloud
* Интернет-трафик из частных облаков

### <a name="stateful"></a>С отслеживанием состояния

 Правило состояния знает о соединениях, которые проходят через него. Запись потока создается для имеющихся подключений. Обмен данными разрешен или запрещен в зависимости от состояния подключения записи потока.  Используйте этот тип правил для общедоступных IP-адресов для фильтрации трафика из Интернета.

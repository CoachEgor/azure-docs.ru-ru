# <a name="shrink-an-avs-private-cloud"></a>Сжатие частного облака AVS

AVS обеспечивает гибкость динамического сжатия частного облака AVS. Частное облако AVS состоит из одного или нескольких кластеров vSphere. Каждый кластер может иметь от 3 до 16 узлов. При сжатии частного облака AVS вы удаляете узел из существующего кластера или удаляете весь кластер. 

## <a name="before-you-begin"></a>Перед началом работы

Прежде чем сжимать частное облако AVS, должны быть выполнены следующие условия. Кластер управления (Первый кластер) создается при создании частного облака AVS. Его нельзя удалить.

* Кластер vSphere должен содержать три узла. Невозможно сжать кластер с тремя узлами.
* Общий объем потребляемого хранилища не должен превышать общую емкость после сжатия кластера.
* Проверьте, не запрещено ли в каких-либо правилах планировщика распределенных ресурсов (DRS) значение vMotion виртуальной машины. Если правила имеются, отключите или удалите правила. Правила DRS включают в себя виртуальную машину для размещения правил сходства.


## <a name="sign-in-to-azure"></a>Войдите в Azure

Войдите на портал Azure по адресу [https://portal.azure.com](https://portal.azure.com).

## <a name="shrinking-an-avs-private-cloud"></a>Сжатие частного облака AVS

1. [Доступ к порталу AVS](access-cloudsimple-portal.md).

2. Откройте страницу **ресурсы** .

3. Щелкните частное облако AVS, которое необходимо сжать.

4. На странице Сводка нажмите кнопку **Сжать**.

    ![Сжатие частного облака AVS](media/shrink-private-cloud.png)

5. Выберите кластер, который необходимо сжать или удалить. 

    ![Сжатие частного облака AVS — выбор кластера](media/shrink-private-cloud-select-cluster.png)

6. Выберите **удалить один узел** или **удалить весь кластер**. 

7. Проверка емкости кластера

8. Нажмите кнопку **Отправить** , чтобы сжать частное облако AVS.

Начинается сжатие частного облака AVS. Вы можете отслеживать ход выполнения задач. Процесс сжатия может занять несколько часов в зависимости от данных, которые необходимо повторно синхронизировать в vSAN.

> [!NOTE]
> 1. Если вы сжимаете частное облако, удаляя последний или единственный кластер в центре обработки данных, центр обработки данных не будет удален.
> 2. При возникновении любого нарушения правила DRS узел не удаляется из кластера, а описание задачи показывает, что удаление узла приведет к нарушению правил DRS в кластере.

# <a name="shrink-a-cloudsimple-private-cloud"></a>Сокращение облачного частного облака

CloudSimple обеспечивает гибкость для динамического сжатия частного облака.  Частное облако состоит из одного или нескольких кластеров vSphere. Каждый кластер может иметь от 3 до 16 узлов. При сжатии приватного облака удаляется узла из существующего кластера или удаляется весь кластер. 

## <a name="before-you-begin"></a>Перед началом

При сжатии частного облака необходимо вынести следующие условия.  Кластер управления (первый кластер), созданный при создании приватного облака, не может быть удален.

* Кластер vSphere должен иметь три узла.  Кластер с тремя узлами не может быть сжат.
* Общий объем потребляемого хранилища не должен превышать общую емкость после сжатия кластера.
* Проверьте, не предотвращает ли какое-либо правила распределенного ресурсного расписания (DRS) vMotion виртуальной машины.  Если правила присутствуют, отключать или удалять правила.  Правила DRS включают виртуальную машину для размещения правил сродства.

## <a name="sign-in-to-azure"></a>Вход в Azure

Войти на портал Azure по адресу [https://portal.azure.com](https://portal.azure.com).

## <a name="shrink-a-private-cloud"></a>Сжатие частного облака

1. [Доступ к порталу CloudSimple](access-cloudsimple-portal.md).

2. Откройте страницу **Ресурсов.**

3. Нажмите на приватное облако, необходимое для сжатия

4. На странице резюме нажмите **Кнопка**.

    ![Сжатие частного облака](media/shrink-private-cloud.png)

5. Выберите кластер, который необходимо уменьшить или удалить. 

    ![Сжатие частного облака - выберите кластер](media/shrink-private-cloud-select-cluster.png)

6. Выберите **Удалить один узла** или **удалить весь кластер.** 

7. Проверка емкости кластера

8. Нажмите **Отправить,** чтобы уменьшить частное облако.

Начинается сжатие частного облака.  Вы можете контролировать ход выполнения задач.  Процесс сжатия может занять несколько часов в зависимости от данных, которые должны быть синхронизированы на vSAN.

> [!NOTE]
> 1. При сжатии частного облака путем удаления последнего или единственного кластера в центре обработки данных центр обработки данных не будет удален.
> 2. При возникновении какого-либо нарушения правила DRS узел не удаляется из кластера, а описание задачи показывает, что удаление узла будет нарушением правил DRS в кластере.

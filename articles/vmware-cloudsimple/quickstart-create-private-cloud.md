# <a name="quickstart---configure-a-private-cloud-environment"></a>Краткое руководство. Настройка среды с частным облаком

В этой статье вы научитесь создавать CloudSimple частного облака и настройте среду частного облака.

## <a name="sign-in-to-azure"></a>Вход в Azure
Войдите на портал Azure по адресу [https://portal.azure.com](https://portal.azure.com).

## <a name="create-a-private-cloud"></a>Создание частного облака

1. Выбор пункта **Все службы**.
2. Поиск **CloudSimple служб**.
3. Выберите службу CloudSimple, на котором вы хотите создать частным облаком.
4. Из сводки, щелкните **Создание частного облака** открыть новую вкладку браузера для CloudSimple портала.  Если будет предложено, войдите с помощью Azure войти в учетные данные.  

    ![Создание частного облака из Azure](media/create-private-cloud-from-azure.png)

5. На портале CloudSimple укажите имя для вашего частного облака
6. Выберите **расположение** частного облака
7. Выберите **тип узла** подготовки в Azure.  Вы можете выбрать [CS28 или CS36](cloudsimple-node.md#vmware-solution-by-cloudsimple-nodes-sku). Последний вариант включает в себя Максимальная емкость вычислительных ресурсов и памяти.
8. Укажите **число узлов**.  Менее трех узлов, необходимы для создания частного облака

    ![Создание частного облака — основные сведения](media/create-private-cloud-basic-info.png)

9. Щелкните **Далее: Дополнительные параметры**.
10. Введите диапазон CIDR для подсети vSphere/виртуальной сети хранения данных. Убедитесь, что диапазон CIDR не пересекается с другими в локальной или других подсетей Azure.

    ![Создание частного облака — Дополнительные параметры](media/create-private-cloud-advanced-options.png)

11. По завершении выберите **Next: Просмотр и создание**.
12. Проверьте параметры. Если вам нужно изменить какие-либо параметры, нажмите кнопку **Назад**.
13. Нажмите кнопку **Создать**.

Будет запущен процесс подготовки частного облака.  Может занять не более двух часов для частного облака, должны быть подготовлены.

## <a name="launch-cloudsimple-portal"></a>Запуск портала CloudSimple

Портал CloudSimple доступны на портале Azure.  Портал CloudSimple запускаются с Azure войти в учетные данные с помощью единого входа (SSO).  Доступ к порталу CloudSimple вам потребуется авторизовать **авторизации службы CloudSimple** приложения.  Дополнительные сведения о предоставлении разрешений см. в разделе [согласие на CloudSimple авторизации службы приложений](https://docs.azure.cloudsimple.com/access-cloudsimple-portal/#consent-to-cloudsimple-service-authorization-application)

1. Выбор пункта **Все службы**.
2. Поиск **CloudSimple служб**.
3. Выберите службу CloudSimple, на котором вы хотите создать частным облаком.
4. Из сводки, щелкните **перейдите на портал CloudSimple** открыть новую вкладку браузера для CloudSimple портала.  Если будет предложено, войдите с помощью Azure войти в учетные данные.  

    ![Запуск портала CloudSimple](media/launch-cloudsimple-portal.png)

## <a name="create-point-to-site-vpn"></a>Создание типа "точка сеть"

VPN-подключения между сайтами — это самый простой способ подключения к частным облаком с вашего компьютера. Используйте VPN-подключение между сайтами, если вы подключаетесь к частному облаку, удаленно.  Для быстрого доступа к частным облаком выполните следующие действия.  Доступ к области CloudSimple из вашей локальной сети можно сделать с помощью [сеть-сеть VPN](https://docs.azure.cloudsimple.com/vpn-gateway/) или [Azure ExpressRoute](https://docs.azure.cloudsimple.com/on-premises-connection/).

### <a name="create-gateway"></a>Создание шлюза

1. Запустите CloudSimple портала и выберите **сети**.
2. Выберите **VPN-шлюз**.
3. Нажмите кнопку **новый VPN-шлюз**.

    ![Создание VPN-шлюза](media/create-vpn-gateway.png)

4. Для **конфигурации шлюза**, укажите следующие параметры и нажмите кнопку **Далее**.

    * Выберите **точка-сеть VPN** как тип шлюза.
    * Введите имя для идентификации шлюза.
    * Укажите расположение Azure, где развернута служба CloudSimple.
    * Укажите подсеть клиента для шлюза между сайтами.  При подключении из этой подсети получает адреса DHCP.

5. Для **подключение/пользователя**, укажите следующие параметры и нажмите кнопку **Далее**.

    * Автоматически разрешает все текущие и будущие пользователи для доступа к частному облаку через этот шлюз между сайтами, выберите **автоматически добавить всех пользователей**. При выборе этого параметра автоматически выбираются все пользователи в списке пользователей. Можно переопределить автоматическую настройку, отменив выделение отдельных пользователей в списке.
    * Чтобы выбрать только отдельным пользователям, установите флажки в списке пользователей.

6. В разделе виртуальные локальные сети и подсети можно указать, управления и пользовательского виртуальные локальные сети и подсети для шлюза и подключений.

    * **Автоматически добавлять** параметры задать глобальные политики для этого шлюза. Параметры применяются для текущего шлюза. Параметры можно переопределить в **выберите** области.
    * Выберите **добавить управление виртуальных локальных сетей, подсетей, частных облаков**. 
    * Чтобы добавить все определяемые пользователем виртуальных локальных сетей, подсетей, щелкните **Добавьте пользовательские виртуальные локальные сети и подсети**. 
    * **Выберите** параметры переопределяют глобальные параметры в разделе **автоматического добавления**. 

7. Нажмите кнопку **Далее** для просмотра параметров. Щелкните значок редактирования вносить изменения.
8. Нажмите кнопку **создать** для создания VPN-шлюза.

### <a name="connect-to-cloudsimple-using-point-to-site-vpn"></a>Подключение к CloudSimple, с помощью типа "точка сеть"

VPN-клиента необходим для подключения к CloudSimple с компьютера.  Скачайте [клиента OpenVPN](https://openvpn.net/community-downloads/) для Windows или [вязкость](https://www.sparklabs.com/viscosity/download/) для macOS и OS X.

1. Запустите CloudSimple портала и выберите **сети**.
2. Выберите **VPN-шлюз**.
3. В списке VPN-шлюзов щелкните VPN-шлюза между сайтами.
4. Выберите **Пользователи**.
5. Щелкните **загрузки моей конфигурации VPN**

    ![Загрузка конфигурации VPN](media/download-p2s-vpn-configuration.png)

6. Импорт конфигурации на VPN-клиента

    * Инструкции по [импорте конфигурации на клиенте Windows](https://openvpn.net/vpn-server-resources/connecting-to-access-server-with-windows/#openvpn-open-source-openvpn-gui-program)
    * Инструкции по [Импорт конфигурации в macOS или OS X](https://www.sparklabs.com/support/kb/article/getting-started-with-viscosity-mac/#creating-your-first-connection)

7. Подключение к CloudSimple

## <a name="create-a-vlan-for-your-workload-vms"></a>Создание виртуальной локальной сети для рабочих нагрузок виртуальных машин

После создания частного облака, создайте виртуальную ЛС, где будет развернут рабочей нагрузки и приложения виртуальных машин.

1. На портале CloudSimple выберите **сети**.
2. Нажмите кнопку **виртуальной локальной сети и подсети**.
3. Нажмите кнопку **создать виртуальной локальной сети и подсети**

    ![Создание виртуальной локальной сети и подсети](media/create-new-vlan-subnet.png)

4. Выберите **частного облака** для новой виртуальной локальной сети и подсети.
5. Выберите идентификатор виртуальной локальной сети из списка.  
6. Введите имя подсети для идентификации подсети.
7. Укажите диапазон CIDR подсети и маски.  Этот диапазон не должен пересекаться с любой существующей подсети.
8. Нажмите кнопку **Submit**(Отправить).

    ![Создать данные о виртуальной локальной сети и подсети](media/create-new-vlan-subnet-details.png)

Будет создан виртуальной локальной сети или подсети.  Теперь можно использовать этот идентификатор виртуальной локальной сети для создания группы распределенных порт в vCenter частного облака. 

## <a name="connect-your-environment-to-an-azure-virtual-network"></a>Подключить среду виртуальной сети Azure

CloudSimple предоставляет канал ExpressRoute для частного облака. Виртуальной сети в Azure можно подключиться к каналу ExpressRoute. Дополнительные сведения о настройке подключения выполните действия, описанные в [Azure виртуальной сети с помощью ExpressRoute](https://docs.azure.cloudsimple.com/cloudsimple-azure-network-connection/)

## <a name="sign-in-to-vcenter"></a>Войдите на сервер vCenter

Теперь можно входить к vCenter для настройки виртуальных машин и политик.

1. Для доступа к vCenter, запустите на портале CloudSimple. На домашней странице в разделе **общих задач**, нажмите кнопку **запуска клиента vSphere**.  Выберите частное облако, а затем нажмите кнопку **запуска клиента vSphere** о частном облаке.

    ![Запустите клиент vSphere](media/launch-vcenter-from-cloudsimple-portal.png)

2. Выберите предпочтительный vSphere клиента для доступа к vCenter и войдите, используя имя пользователя и пароль.  Значения по умолчанию являются:
    * Имя пользователя: **CloudOwner@cloudsimple.local**
    * Пароль: **CloudSimple123!**  

VCenter экраны следующих процедур: от клиента vSphere (HTML5).

## <a name="change-your-vcenter-password"></a>Изменить пароль vCenter

CloudSimple рекомендует изменить пароль при первом входе в vCenter.  
Пароль должен соответствовать следующим требованиям:

* Максимальное время существования: Пароль должен меняться каждые 365 дней
* Запретить повторное использование: Пользователи, не может использовать любой из пяти предыдущих паролей
* Продолжительность: 8 - 20 символов
* Специальный символ: По крайней мере один специальный символ
* Буквенные символы: По крайней мере одну прописную букву, по крайней мере одну строчную букву, a – z и A – Z
* Номера: По крайней мере одной цифре, 0 – 9
* Максимальное идентичные соседних символов: Три

    Пример: "Копия" или CCC допустима в качестве части пароля, но не CCCC.

Если задать пароль, который не соответствует требованиям:

* Если вы используете Flash клиента vSphere, она сообщает об ошибке
* Если вы используете клиент HTML5, она не сообщает об ошибке. Клиент не принимает изменение и старый пароль продолжает работать.

## <a name="change-nsx-administrator-password"></a>Изменение пароля администратора NSX

NSX manager развертывается с паролем по умолчанию.  Мы рекомендуем изменить пароль после создания вашего частного облака.

   * Имя пользователя: **администратора**
   * Пароль: **CloudSimple123!**

Полное доменное имя (FQDN) и IP-адрес диспетчера NSX можно найти на портале CloudSimple.

1. Запустите CloudSimple портала и выберите **ресурсы**.
2. Щелкните частное облако, который вы хотите использовать.
3. Выберите **сеть управления vSphere**
4. Используйте полное доменное имя или IP-адрес **NSX Manager** и подключитесь с помощью веб-браузер. 

    ![Найти полное доменное имя диспетчера NSX](media/private-cloud-nsx-manager-fqdn.png)

Чтобы изменить пароль, следуйте инструкциям в [управление пароль](https://docs.vmware.com/en/VMware-NSX-T-Data-Center/2.4/administration/GUID-DB31B304-66A5-4516-9E55-2712D12B4F27.html).

> [!WARNING]
> По умолчанию пароль администратора NSX срок действия истекает через 90 дней.

## <a name="create-a-port-group"></a>Создайте группу портов

Чтобы создать группу портов, распределенных в vSphere:

1. Следуйте инструкциям в разделе «Добавление группу распределенных портов» в [vSphere руководство по сетевому взаимодействию](https://docs.vmware.com/en/VMware-vSphere/6.5/vsphere-esxi-vcenter-server-65-networking-guide.pdf).
2. При настройке порта распределенной группы, укажите идентификатор виртуальной локальной сети, созданные в [создайте виртуальной локальной сети для виртуальных машин рабочей нагрузки](#create-a-vlan-for-your-workload-vms).

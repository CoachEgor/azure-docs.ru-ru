# <a name="set-up-cloudsimple-private-cloud-as-a-disaster-recovery-site-for-on-premises-vmware-workloads"></a>Настройка облачного частного облака Cloud как места аварийного восстановления для рабочих нагрузок VMware

Ваше облачное частное облако может быть настроено как место восстановления для корпоративных приложений, чтобы обеспечить непрерывность бизнеса в случае аварии. Решение для восстановления основано на виртуальной репликации зерто в качестве платформы репликации и оркестровки. Критические инфраструктуры и приложения виртуальные машины могут быть воспроизведены непрерывно из вашего собственного vCenter в вашем частном облаке. Вы можете использовать частное облако для тестирования на сбой и обеспечения доступности приложения во время аварии. Аналогичный подход можно использовать для создания частного облака в качестве основного сайта, защищенного сайтом восстановления в другом месте.

> [!NOTE]
> Обратитесь к документу Зерто [Размер соображения для виртуального репликации Зерто](https://s3.amazonaws.com/zertodownload_docs/5.5U3/Zerto%20Virtual%20Replication%20Sizing.pdf) для руководящих принципов по размеру среды аварийного восстановления.

Решение CloudSimple:

* Устраняет необходимость создания центра обработки данных специально для аварийного восстановления (DR).
* Позволяет использовать места Azure, где CloudSimple развернут для географической устойчивости во всем мире.
* Предоставляет возможность сократить затраты на развертывание и общую стоимость владения DR.

Решение требует:

* Установка, настройка и управление зерто в вашем частном облаке.
* Предоставьте свои собственные лицензии для компании «Зерто», когда частное облако является защищенным сайтом. Вы можете спарить Зерто, работающий на сайте CloudSimple, с вашим местным сайтом для лицензирования.

### <a name="prerequisites"></a>Предварительные требования

Для того, чтобы виртуальная репликация зерто из вашей среды в приватную среду в ваше частное облако, выполните следующие предпосылки.

1. [Настройте VPN-соединение от сайта к сайту между вашей предварительной сетью и вашим облачным частным облаком.](set-up-vpn.md)
2. [Настройка поиска DNS таким образом, чтобы компоненты управления private Cloud были перенаправлены на серверы Private Cloud DNS.](on-premises-dns-setup.md)  Чтобы обеспечить переадресировку поиска DNS, создайте вход в зону `*.cloudsimple.io` переадресов в вашем предприимчном DNS-сервере для серверов CloudSimple DNS.
3. Настройка поиска DNS таким образом, чтобы компоненты vCenter были перенаправлены на предприимки DNS-серверы.  DNS-серверы должны быть доступны из вашего Облачного частного облака через VPN от сайта к сайту. Для получения помощи отправьте [запрос на поддержку,](https://portal.azure.com/#blade/Microsoft_Azure_Support/HelpAndSupportBlade/newsupportrequest)предоставив следующую информацию.  

    * Предварительное доменное имя DNS
    * Внутренние IP-адреса DNS-сервера

4. Установите сервер Windows в личном облаке. Сервер используется для установки виртуального менеджера зерто.
5. [Эскалация привилегий CloudSimple](escalate-private-cloud-privileges.md).
6. Создайте нового пользователя в вашем Private Cloud vCenter с административной ролью для использования в качестве учетной записи службы для виртуального менеджера зерто.

### <a name="optional-configuration-on-your-private-cloud"></a>Дополнительная конфигурация в частном облаке

1. Создайте один или несколько пулов ресурсов в вашем Private Cloud vCenter для использования в качестве целевых пулов ресурсов для вс-клиентов из вашей предварительной среды.
2. Создайте одну или несколько папок в вашем Private Cloud vCenter для использования в качестве целевых папок для вс-клиентов из вашей предварительной среды.
3. Создание VLANs для сети сбоев и настройка правил брандмауэра. Откройте [запрос на](https://portal.azure.com/#blade/Microsoft_Azure_Support/HelpAndSupportBlade/newsupportrequest) помощь.
4. Создание распределенных групп портов для сети сбоев и тестовой сети для тестирования сбоев в сбое в вд.
5. Установите [серверы DHCP и DNS](dns-dhcp-setup.md) или используйте контроллер домена Active Directory в среде Private Cloud.

### <a name="set-up-zvm-and-vra-on-your-private-cloud"></a>Настройка ВММ и VRA в личном облаке

1. Установка виртуального менеджера «Зерто» на сервере Windows в личном облаке.
2. Вопием в в «VM» с помощью учетной записи службы, созданной в предыдущих шагах.
3. Настройка лицензирования для виртуального менеджера зерто.
4. Установка виртуального прибора репликации (VRA) на хосты ESXi вашего частного облака.
5. Соедините ваше частное облако ЗВМ с вашим наемным ТВМ.

### <a name="set-up-zerto-virtual-protection-group"></a>Настройка группы виртуальной защиты Зерто

1. Создайте новую виртуальную группу защиты (VPG) и уточните приоритет для VPG.
2. Выберите виртуальные машины, которые требуют защиты для непрерывности бизнеса и настроить заказ на загрузку, если это необходимо.
3. Выберите сайт восстановления в качестве частного облака и сервер восстановления по умолчанию в качестве кластера Private Cloud или созданной группы ресурсов. Выберите **vsanDatastore** для хранилища данных восстановления в личном облаке.

    ![VPG](media/cloudsimple-zerto-vpg.png)

    > [!NOTE]
    > Вы можете настроить опцию хоста для отдельных VMs в соответствии с опцией VM Настройки.

4. Настройка вариантов хранения по мере необходимости.
5. Укажите сети восстановления для использования для сети сбоев и тестов для сбоя в качестве распределенных групп портов, созданных ранее, и настроить сценарии восстановления по мере необходимости.
6. Настройте настройки сети для отдельных VMs при необходимости и создайте VPG.
7. Тест неудачи после завершения репликации.

# <a name="vpn-gateways-overview"></a>Обзор VPN шлюзов

VPN шлюз используется для отправки зашифрованного трафика между сетью cloudSimple в месте расположения в открытом доступе или компьютером через общедоступный Интернет.  Каждый регион может иметь один VPN шлюз, который может поддерживать несколько соединений. При создании нескольких подключений к одному VPN-шлюзу все VPN-туннели совместно используют доступную для этого шлюза пропускную способность.

CloudSimple предоставляет два вида VPN шлюзов:

* VPN шлюз от сайта к сайту
* Точечный VPN шлюз

## <a name="site-to-site-vpn-gateway"></a>VPN шлюз от сайта к сайту

VPN шлюз от сайта к сайту используется для отправки зашифрованного трафика между сетью cloudSimple и центром обработки данных. Используйте это соединение для определения диапазона подсетей/CIDR для сетевого трафика между вашей локальным сетью и сетью CloudSimple region.

VPN шлюз позволяет потреблять услуги из закрытых помещений в личном облаке, а также услуги в вашем частном облаке из предварительной сети.  CloudSimple предоставляет VPN-сервер, основанный на политике, для создания соединения из вашей предварительной сети.

Используйте чехлы для VPN от сайта к сайту:

* Доступность вашего Private Cloud vCenter с любой рабочей станции в вашей предварительной сети.
* Использование вашего наместеранного Active Directory в качестве источника идентификации vCenter.
* Удобная передача шаблонов VM, II и других файлов из ваших закрытых ресурсов в ваш Private Cloud vCenter.
* Доступность рабочих нагрузок, работающих в вашем частном облаке, из вашей предварительной сети.

### <a name="cryptographic-parameters"></a>Криптографические параметры

VPN-соединение от сайта к сайту использует следующие криптографические параметры по умолчанию для создания безопасного соединения.  При создании соединения с вашего собственного VPN-устройства используйте любой из следующих параметров, которые поддерживаются вашим наемным VPN шлюзом.

#### <a name="phase-1-proposals"></a>Предложения на этапе 1

| Параметр | Предложение 1 | Предложение 2 | Предложение 3 |
|-----------|------------|------------|------------|
| Версия IKE | IKEv1 | IKEv1 | IKEv1 |
| Шифрование | AES 128 | AES 256 | AES 256 |
| Хэш-алгоритм| ША 256 | ША 256 | ША 1 |
| Диффи Хеллман Групп (DH Group) | 2 | 2 | 2 |
| Время существования | 28 800 сек | 28 800 сек | 28 800 сек |
| Размер данных | 4 ГБ | 4 ГБ | 4 ГБ |

#### <a name="phase-2-proposals"></a>Предложения на этапе 2

| Параметр | Предложение 1 | Предложение 2 | Предложение 3 |
|-----------|------------|------------|------------|
| Шифрование | AES 128 | AES 256 | AES 256 |
| Хэш-алгоритм| ША 256 | ША 256 | ША 1 |
| Идеальная группа передовой тайны (PFS Group) | None | None | None |
| Время существования | 1800 секунд | 1800 секунд | 1800 секунд |
| Размер данных | 4 ГБ | 4 ГБ | 4 ГБ |


> [!IMPORTANT]
> Установите TCP MSS Clamping на 1200 на вашем устройстве VPN. Или, если ваши VPN устройства не поддерживают MSS зажима, вы можете в качестве альтернативы установить MTU на туннель интерфейс е1240 байтов вместо.

## <a name="point-to-site-vpn-gateway"></a>Точечный VPN шлюз

VPN от точки к сайту используется для отправки зашифрованного трафика между сетью региона CloudSimple и клиентским компьютером.  VPN от точки к месту — это самый простой способ получить доступ к сети Private Cloud, включая vCenter private Cloud и vpn-нагрузок.  Используйте VPN-соединение от точки к сайту, если вы подключаетесь к частному облаку удаленно.

# <a name="cloudsimple-private-cloud-permission-model-of-vmware-vcenter"></a>Модель разрешения на частное облако Cloud Компании VMware vCenter

CloudSimple сохраняет полный административный доступ к среде Private Cloud. Каждому клиенту CloudSimple предоставляется достаточно административных привилегий, чтобы иметь возможность развертывать и управлять виртуальными машинами в своей среде.  При необходимости можно временно увеличить свои привилегии для выполнения административных функций.

## <a name="cloud-owner"></a>Владелец облака

При создании частного облака пользователь **CloudOwner** создается в домене vCenter Single Sign-On с доступом **к облачному владельцу-роли** для управления объектами в частном облаке. Этот пользователь также может настроить дополнительные [источники идентификации vCenter](set-vcenter-identity.md)и других пользователей в Private Cloud vCenter.

> [!NOTE]
> Пользователь по умолчанию для вашего cloudowner@cloudsimple.local VCenter CloudSimple Private Cloud — это время создания частного облака.

## <a name="user-groups"></a>Пользовательские группы

Группа под названием **Cloud-Owner-Group** создается во время развертывания частного облака. Пользователи этой группы могут управлять различными частями среды vSphere в частном облаке. Этой группе автоматически предоставляются привилегии **Cloud-Owner-Role,** и пользователь **CloudOwner** добавляется в качестве члена этой группы.  CloudSimple создает дополнительные группы с ограниченными привилегиями для удобства управления.  Вы можете добавить любого пользователя в эти предварительно созданные группы, и привилегии, определяемые ниже, автоматически назначаются пользователям в группах.

### <a name="pre-created-groups"></a>Предварительно созданные группы

| Имя группы | Назначение | Роль |
| -------- | ------- | ------ |
| Облако-Владелец-Группа | Члены этой группы имеют административные привилегии для Private Cloud vCenter | [Облако-владелец-роль](#cloud-owner-role) |
| Облако-Глобальный кластер-Админ-группа | Члены этой группы имеют административные привилегии в кластере Private Cloud vCenter | [Облако-Кластер-Админ-Роль](#cloud-cluster-admin-role) |
| Облако-Глобальный-Хранение-Администратор-Группа | Члены этой группы могут управлять хранением на Private Cloud vCenter | [Облако-Хранение-Администратор-Роль](#cloud-storage-admin-role) |
| Облако-Глобальный-Сеть-Админ-группы | Члены этой группы могут управлять сетевыми и распределенными группами портов на Private Cloud vCenter | [Облако-Сеть-Админ-Роль](#cloud-network-admin-role) |
| Облако-Глобал-ВМ-Админ-Группа | Члены этой группы могут управлять виртуальными машинами на Private Cloud vCenter | [Облако-VM-Админ-Роль](#cloud-vm-admin-role) |

Чтобы предоставить отдельным пользователям разрешения на управление приватным облаком, создайте учетные записи пользователей, добавляя к соответствующим группам.

> [!CAUTION]
> Новые пользователи должны быть добавлены только в *Cloud-Owner-Group*, *Cloud-Global-Cluster-Admin-Group,* *Cloud-Global-Storage-Admin-Group,* *Cloud-Global-Network-Admin-Group* или, *Cloud-Global-VM-Admin-Group.*  Пользователи, добавленные в группу *администраторов,* будут удалены автоматически.  Только учетные записи служб должны быть добавлены в группу *администраторов,* а учетные записи службы не должны использоваться для входиных в веб-uI vSphere.

# <a name="accessing-your-cloudsimple-private-cloud-environment-and-applications-from-on-premises"></a>Доступ к среде облачного облака Cloud и приложениям из закрытых

Соединение можно настроить от предприимчезной сети к CloudSimple с помощью Azure ExpressRoute или VPN от сайта к сайту.  Получите доступ к vCenter CloudSimple Private Cloud и любым рабочим нагрузкам, которые вы используете в частном облаке, используя соединение.  Вы можете управлять тем, какие порты открываются на подключении, используя брандмауэр в вашей предварительной сети.  В этой статье рассматриваются некоторые из типичных требований к портам приложений.  Для любых других приложений обратитесь к заявочной документации для требований порта.

## <a name="ports-required-for-accessing-vcenter"></a>Порты, необходимые для доступа к vCenter

Чтобы получить доступ к вашему private Cloud vCenter и менеджеру NSX-T, порты, определяемые в таблице ниже, должны быть открыты на закрытом брандмауэре.  

| Порт       | Источник                           | Назначение                      | Назначение                                                                                                                |
|------------|----------------------------------|----------------------------------|------------------------------------------------------------------------------------------------------------------------|
| 53 (UDP)   | Локальные серверы DNS          | Частные облачные DNS-серверы        | Требуется для пересылки DNS поиска *az.cloudsimple.io* на частные облачные DNS-серверы из предварительной сети.       |
| 53 (UDP)   | Частные облачные DNS-серверы        | Локальные серверы DNS          | Требуется для пересылки DNS искать на месте доменных имен из Private Cloud vCenter на закрытые DNS-серверы. |
| 80 (TCP)   | Локальная сеть              | Сеть управления частным облаком | Требуется для перенаправления URL-адреса vCenter с *http* *s https*.                                                           |
| 443 (TCP)  | Локальная сеть              | Сеть управления частным облаком | Требуется для доступа к vCenter и NSX-T менеджер из предварительной сети.                                             |
| 8000 (TCP) | Локальная сеть              | Сеть управления частным облаком | Требуется для vMotion виртуальных машин из наемных помещений в Private Cloud.                                            |
| 8000 (TCP) | Сеть управления частным облаком | Локальная сеть              | Требуется для vMotion виртуальных машин от Private Cloud до наместера.                                            |

## <a name="ports-required-for-using-on-premises-active-directory-as-an-identity-source"></a>Порты, необходимые для использования в предварительном активном каталоге в качестве источника идентификации

Для настройки наместеранного активного каталога в качестве источника идентификации в Private Cloud vCenter необходимо открыть порты, определенные в таблице.  Для шагов конфигурации [используйте Azure AD в качестве поставщика идентификационных данных для vCenter в облачном частном облаке CloudSimple Private Cloud.](https://docs.azure.cloudsimple.com/azure-ad/)

| Порт         | Источник                           | Назначение                                         | Назначение                                                                                                                                          |
|--------------|----------------------------------|-----------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------|
| 53 (UDP)      | Частные облачные DNS-серверы        | Локальные серверы DNS                             | Требуется для пересылки DNS искать на месте активных доменных имен каталога от Private Cloud vCenter на закрытые DNS-серверы.          |
| 389 (TCP или UDP) | Сеть управления частным облаком | На-предприимчивые активные контроллеры доменов каталога     | Требуется для связи LDAP с сервера Private Cloud vCenter для активных контроллеров доменов каталогов для проверки подлинности пользователей.                |
| 636 (TCP)     | Сеть управления частным облаком | На-предприимчивые активные контроллеры доменов каталога     | Требуется для безопасной связи LDAP (LDAPS) с сервера Private Cloud vCenter для активных контроллеров доменов каталогов для проверки подлинности пользователей. |
| 3268 (TCP)    | Сеть управления частным облаком | На-предежетных активных серверов глобального каталога каталога | Требуется для связи LDAP в многодоменных развертываниях контроллеров.                                                                        |
| 3269 (TCP)    | Сеть управления частным облаком | На-предежетных активных серверов глобального каталога каталога | Требуется для связи LDAPS в многодоменных развертываниях контроллеров.                                                                       |                                           |

## <a name="common-ports-required-for-accessing-workload-virtual-machines"></a>Общие порты, необходимые для доступа к виртуальным рабочим машинам

Для доступа к рабочей нагрузке виртуальных машин, работающих в Private Cloud, требуется, чтобы порты открывались на вашем закрытом брандмауэре.  В таблице ниже показаны некоторые из требуемых общих портов и их назначение.  Для любого приложения конкретные портовые требования относятся к заявочной документации.

| Порт         | Источник                         | Назначение                          | Назначение                                                                              |
|--------------|--------------------------------|--------------------------------------|--------------------------------------------------------------------------------------|
| 22 (TCP)      | Локальная сеть            | Сеть рабочей нагрузки Private Cloud       | Безопасный доступ к устройствам Linux виртуальных машин, работающих на Private Cloud.              |
| 3389 (TCP)    | Локальная сеть            | Сеть рабочей нагрузки Private Cloud       | Удаленный рабочий стол к окнам виртуальных машин, работающих на Private Cloud.                 |
| 80 (TCP)      | Локальная сеть            | Сеть рабочей нагрузки Private Cloud       | Доступ к любым веб-серверам, развернутым на виртуальных машинах, работающих в Private Cloud.        |
| 443 (TCP)     | Локальная сеть            | Сеть рабочей нагрузки Private Cloud       | Доступ к любым защищенным веб-серверам, развернутым на виртуальных машинах, работающих в Private Cloud. |
| 389 (TCP или UDP) | Сеть рабочей нагрузки Private Cloud | Натерритории активной сети каталогов | Присоединяйтесь к виртуальным компьютерам рабочей нагрузки Windows в домена активного каталога.       |
| 53 (UDP)      | Сеть рабочей нагрузки Private Cloud | Локальная сеть                  | DNS-сервисный доступ для рабочих компьютеров виртуальной загрузки на предприимки DNS-серверов.         |

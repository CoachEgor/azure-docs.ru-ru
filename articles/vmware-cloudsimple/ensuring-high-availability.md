## <a name="fast-replacement-of-failed-node"></a>Быстрая замена неисправного узла

Программное обеспечение CloudSimple Control plane непрерывно отслеживает работоспособность кластеров VMware и обнаруживает, когда узла ESXi выходит из строя. Затем он автоматически добавляет новый хост ESXi в пораженный кластер VMware из пула легкодоступных узлов и выводит неузлый из кластера. Эта функциональность гарантирует быстрое восстановление резервной емкости в кластере VMware, так что устойчивость кластера, обеспечиваемый vSAN и VMware HA, будет восстановлена.

## <a name="placement-groups"></a>Группы размещения

Пользователь, создающий частное облако, может выбрать область Azure и группу размещения в выбранном регионе. Группа размещения представляет собой набор узлов, разбросанных по нескольким стойкам, но в пределах одного и того же сегмента сети позвоночника. Узлы в одной и той же группе размещения могут достигать друг друга с помощью максимум двух дополнительных коммутаторов. Группа размещения всегда находится в пределах единой зоны доступности Azure и охватывает несколько стоек. Самолет управления CloudSimple распределяет узлы частного облака по нескольким стойкам, основываясь на наилучших усилиях. Узлы в разных группах размещения гарантированно размещаются в разных стеллажах.

## <a name="availability-zones"></a>Зоны доступности

Зоны доступности — это предложение с высокой доступностью, которое защищает ваши приложения и данные от сбоев в работе центра обработки данных. Зоны доступности являются особыми физическими местами в регионе Azure. Каждая зона состоит из одного или нескольких центров обработки данных, оснащенных независимыми системами электроснабжения, охлаждения и сетевого взаимодействия. Каждый регион имеет одну зону доступности. Для получения дополнительной [информации](../availability-zones/az-overview.md)см.

## <a name="redundant-azure-expressroute-circuits"></a>Излишние схемы Azure ExpressRoute

Подключение центра обработки данных к Azure vNet с помощью ExpressRoute имеет избыточные схемы для обеспечения высокодоступной сетевой связи.

## <a name="redundant-networking-services"></a>Избыточные сетевые услуги

Все сетевые сервисы CloudSimple для частного облака (включая VLAN, брандмауэр, общедоступные IP-адреса, Интернет и VPN) разработаны таким образом, чтобы быть очень доступными и способны поддерживать службу SLA.

## <a name="azure-layer-7-load-balancer-for-stateless-vmware-vms"></a>Azure Слой 7 Балансер нагрузки для абонемативных VMw

Пользователи могут поместить балансер загрузки Azure Layer 7 перед vMs-миностами без состояния, работающим в среде VMware, для достижения высокой доступности веб-уровня.

## <a name="azure-regions"></a>Регионы Azure

Регион Azure — это набор центров обработки данных, развернутых в пределах определяемого по периметру задержки и соединенных через выделенную региональную сеть с низкой задержкой. Для получения подробной информации [см.](https://azure.microsoft.com/global-infrastructure/regions)

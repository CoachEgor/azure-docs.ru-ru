# <a name="quickstart---consume-vmware-vms-on-azure"></a>Краткое руководство. Использование виртуальных машин VMware в Azure

Чтобы создать виртуальную машину на портале Azure, используйте шаблоны виртуальных машин, CloudSimple администратор включил для вашей подписки. Эти шаблоны виртуальных Машин находятся на инфраструктура VMware.

## <a name="cloudsimple-vm-creation-on-azure-requires-a-vm-template"></a>Создание виртуальной Машины CloudSimple в Azure требуется шаблон виртуальной Машины

Создание виртуальной машины в частном облаке из vCenter пользовательского интерфейса. Чтобы создать шаблон, следуйте инструкциям в [клонировать виртуальную машину в шаблон в веб-клиенте vSphere](https://docs.vmware.com/en/VMware-vSphere/6.7/com.vmware.vsphere.vm_admin.doc/GUID-FE6DE4DF-FAD0-4BB0-A1FD-AFE9A40F4BFE.html). Store шаблона виртуальной Машины в частном облаке vCenter.

## <a name="create-a-virtual-machine-in-the-azure-portal"></a>Создание виртуальной машины на портале Azure

1. В меню слева щелкните **+** или **создать ресурс**.

2. В меню слева щелкните **вычислений**, а затем нажмите кнопку **CloudSimple виртуальной машины**.

3. Нажмите кнопку **Подтверждение** чтобы убедиться, что вы хотите создать новую виртуальную Машину.

4. Задайте базовой конфигурации, как описано в следующей таблице и нажмите кнопку **далее: Размер**.

    | Поле | ОПИСАНИЕ |
    | ------------ | ------------- |
    | Подписка | Подписка Azure, связанные с развертыванием частного облака.  |
    | Группа ресурсов | Группа развертывания, к которому будет назначен виртуальной Машине. Можно выбрать существующую группу или создайте новую. |
    | ИМЯ | Имя для идентификации виртуальной Машины.  |
    | Location | Регион Azure, в котором размещается эта виртуальная машина.  |
    | Пул ресурсов | Физические ресурсы для виртуальной Машины. Выберите доступные пулы ресурсов. |
    | vSphere шаблона | Тип шаблона операционной системы для виртуальной Машины.  |
    | Имя пользователя | Имя пользователя администратора виртуальной Машины. |
    | Подтверждение пароля | Пароль администратора виртуальной Машины.  |

5. Выберите количество ядер и объем памяти для виртуальной Машины.

6. (Необязательно) Если вы хотите предоставить полной виртуализации ЦП для гостевой операционной системы, выберите **предоставлять для гостевой ОС** флажок.
Установка этого флажка позволяет приложений, требующих виртуализацию оборудования для запуска на виртуальных машинах без двоичную трансляцию или паравиртуализации. Дополнительные сведения см. в статье VMware [предоставлять VMware оборудования виртуализации](https://docs.vmware.com/en/VMware-vSphere/6.5/com.vmware.vsphere.vm_admin.doc/GUID-2A98801C-68E8-47AF-99ED-00C63E4857F6.html).

7. Щелкните **Далее: Конфигурации**.

8. Настройка сетевых интерфейсов и диски, как описано в следующих таблицах.

    Для сетевых интерфейсов, нажмите кнопку **добавить сетевой интерфейс** и настройте следующие параметры.

    | Контроль | ОПИСАНИЕ |
    | ------------ | ------------- |
    | ИМЯ | Введите имя для определения интерфейса.  |
    | Сеть | Выберите из списка сетей, настроенных в vSphere вашего частного облака.  |
    | Адаптер | Выберите адаптер vSphere в списке доступных типов, которые настроены для виртуальной Машины. Дополнительные сведения см. в статье базы знаний VMware [Выбор сетевой адаптер для виртуальной машины](https://kb.vmware.com/s/article/1001805). |
    | Включение при загрузке | Выберите, следует ли включить оборудовании сетевого Адаптера при загрузке виртуальной Машины. По умолчанию используется **включить**. |

    Для дисков, нажмите кнопку **добавить диск** и настройте следующие параметры.

    | Элемент | ОПИСАНИЕ |
    | ------------ | ------------- |
    | ИМЯ | Введите имя для идентификации диска.  |
    | Размер | Выберите один из доступных размеров.  |
    | SCSI-контроллер | Выберите SCSI-контроллер. Доступные контроллеры зависят от других поддерживаемых операционных систем.  |
    | Mode | Определяет, как диск участвует в моментальных снимках. Выберите один из следующих вариантов: <br> -Независимые от постоянных: Все данные, записанные на диск записываются без возможности восстановления.<br> -Независимые непостоянный: Изменения записываются на диск отменяются выключение питания или сброс виртуальной машины.  Независимый режим непостоянный позволяет всегда перезапустите виртуальную Машину в том же состоянии. Дополнительные сведения см. в разделе [документации по VMware.](https://docs.vmware.com/en/VMware-vSphere/6.5/com.vmware.vsphere.vm_admin.doc/GUID-8B6174E6-36A8-42DA-ACF7-0DA4D8C5B084.html)

9. Проверьте параметры. Чтобы внести изменения, щелкните вкладки сверху.

10. Нажмите кнопку **создать** для сохранения настроек и создания виртуальной Машины.

# <a name="set-up-vcenter-on-your-private-cloud-for-vmware-vrealize-automation"></a>Настройка vCenter на вашем частном облаке для VMware vRealize автоматизации

## <a name="install-vrealize-automation-in-your-on-premises-environment"></a>Установка vRealize Автоматизация в вашей предварительной среде

1. Вопийте на серверном приборе vRealise Automation IaaS в качестве администратора IaaS, созданного для вас облачной службой поддержки.
2. Развертывание агента vSphere для конечной точки vRealize Automation.
    1. Перейти к https://*vra-url*:5480/installer, где *vra-URL является URL-адресом,* который вы используете для доступа к uI администрирования администрирования vRealize Automation.
    2. Нажмите на **установку IaaS,** чтобы загрузить установщик.<br>
    Конвенция именования для файла установки setup_*vra-url*@5480.exe.
    3. Запустите установщик. На экране приветствия щелкните **Далее**.
    4. Примите EULA и нажмите **далее**.
    5. Предоставьте информацию о входе, нажмите **Accept Certificate,** а затем нажмите **Далее**.
    ![vRA учетные данные](media/configure-vra-endpoint-login.png)
    6. Выберите **пользовательские установки** и **прокси-агентов** и нажмите **далее**.
    ![vRA тип установки](media/configure-vra-endpoint-install-type.png)
    7. Введите информацию о входе сервера IaaS и нажмите **кнопку Next**. Если вы используете Active Directory, введите имя пользователя в формате **domain.user.** В противном случае, использовать **user@domain** формат.
    ![vRA Войти информация](media/configure-vra-endpoint-account.png)
    8. Для настроек прокси введите **vSphere** для **типа Agent.** Введите имя агента.
    9. Введите сервер IaaS F'DN в **хост обслуживания менеджера** и полях **Web Service Model Manager.** Нажмите **тест,** чтобы проверить соединение для каждого из значений F-DN. Если тест не удается, измените настройки DNS таким образом, чтобы устранить хост-имя сервера IaaS.
    10. Введите имя конечной точки сервера vCenter для частного облака. Запись имени для использования позже в процессе настройки.

        ![vRA установить прокси](media/configure-vra-endpoint-proxy.png)

    11. Нажмите кнопку **Далее**.
    12. Нажмите кнопку **Установить**.

## <a name="configure-the-vsphere-agent"></a>Настройка агента vSphere

1. Перейти к https://*vra-url*/vcac и войти в качестве **ConfigurationAdmin**.
2. Выберите**конечные** > **точки** **инфраструктуры.** > 
3. Выберите **новый** > **виртуальный** > **vSphere**.
4. Введите имя конечных точек vSphere, указанное в предыдущей процедуре.
5. Для **адреса**введите URL-адрес Private Cloud vCenter Server в формате https://*vcenter-fqdn/sdk,* где *vcenter-fqdn* — это название сервера vCenter.
6. Введите учетные данные для административного пользователя vRealise Automation IaaS, созданного для вас службой поддержки CloudSimple.
7. Нажмите **На тестовое соединение** для проверки учетных данных пользователей. Если тест не удается, проверьте URL,, информацию об учетной записи, [имя и имя конечных точек](#verify-the-endpoint-name) и тест снова.
8. После успешного тестирования нажмите **OK,** чтобы создать конечную точку vSphere.
    ![vRA конечный доступ к конфигурации](media/configure-vra-endpoint-vra-edit.png)

### <a name="verify-the-endpoint-name"></a>Проверить имя конечных точек

Чтобы определить правильное имя конечной точки сервера vCenter, сделайте следующее:

1. Откройте командный запрос на приборе IaaS.
2. Изменение каталога на C: «Файлы программы (x86)»VMware'vCAC-Агенты-имя, где *имя агента* — это имя, назначенное в конечную точку сервера vCenter.
3. Выполните следующую команду.

```
..\..\Server\DynamicOps.Vrm.VRMencrypt.exe VRMAgent.exe.config get
```

Выход аналогичен следующему. Значение `managementEndpointName` поля — это имя конечной точки.

```
managementEndpointName: cslab1pc3-vc
doDeletes: true
```

# <a name="choose-a-load-balancing-solution-for-cloudsimple-private-clouds"></a>Выберите решение для балансировки нагрузки для частных облаков CloudSimple

При развертывании приложения в облачном частном облаке можно выбрать любой из нескольких вариантов балансировки нагрузки.

Вы можете выбрать виртуальный или программно-ориентированный балансоровостой нагрузки в частном облаке CloudSimple или даже использовать балансистер нагрузки Azure L7, работающий в подписке Azure, на передний конец вашего веб-уровня VMs, работающих в облачном частном облаке. Здесь мы перечислим несколько вариантов:

## <a name="virtual-load-balancers"></a>Виртуальные балансиваторы нагрузки

Вы можете развернуть виртуальные приборы балансировки нагрузки в вашей среде VMware через интерфейс vCenter и настроить их на переднем конце вашего движения приложений.

Некоторые популярные поставщики: http://nginx.org/en/docs/http/load_balancing.html NginX: F5- https://www.f5.com/products/big-ip-services/virtual-editions BigIP - Менеджер трафика: Citrix ADC:https://www.citrix.com/products/citrix-adc/

## <a name="azure-l7-load-balancer"></a>Балансизатор нагрузки Azure L7

При использовании Azure Application Gateway в качестве балансиста нагрузки L7 для приложения, работаемого в частном облаке, вам не нужно управлять программным обеспечением для балансирования нагрузки. Программное обеспечение балансосивации нагрузки управляется Azure. Все вс-адреса веб-уровня в частном облаке используют частные IP-адреса и не требуют дополнительных правил NAT или общедоступных IP-адресов для разрешения имен. Веб-уровень VMs общаться с Azure application Gateway через частное, с низкой задержкой, высокой пропускной способностью соединения.

Чтобы узнать больше о том, как настроить это решение, обратитесь к руководству по решению по использованию Azure Application Gateway в качестве балансиста нагрузки L7.

## <a name="azure-internal-load-balancer"></a>Балансизатор внутренней нагрузки Azure

Если вы решите запустить приложение в гибридном развертывании, где веб-фронтовой уровень работает в рамках Azure vNet в подписке Azure и уровень DB приложения работает в VMware VMs в облачном частном облаке, вы можете использовать внутреннюю нагрузку Azure балансер (балансер нагрузки L4) перед вашим ими уровня DB для управления трафиком.

Чтобы узнать больше, смотрите документацию [баланса внутренней нагрузки](../load-balancer/concepts-limitations.md#internalloadbalancer) Azure.

## <a name="global-server-load-balancer"></a>Глобальный балансировочный баланс нагрузки сервера

Если вы ищете балансиватор нагрузки на основе DNS, то вы можете использовать сторонние решения, доступные в Azure Marketplace, либо использовать универсиадное решение Azure.

Azure Traffic Manager — это балансер на основе трафика на основе DNS, который позволяет оптимально распределять трафик среди служб в глобальных регионах Azure и в помещениях, обеспечивая при этом высокую доступность и отзывчивость. Чтобы узнать больше, смотрите документацию менеджера [движения](../traffic-manager/traffic-manager-configure-geographic-routing-method.md) Azure.

# <a name="create-an-avs-private-cloud"></a>Создание частного облака AVS

Частное облако AVS — это изолированный стек VMware, который поддерживает узлы ESXi, vCenter, vSAN и НСКС. Частные облака AVS управляются с помощью портала AVS. Они имеют собственный сервер vCenter в собственном домене управления. Стек работает на выделенных узлах и изолированных аппаратных узлах без операционной системы.

Создание частного облака AVS помогает решать разнообразные распространенные потребности в сетевой инфраструктуре.

* **Рост**. Если вы достигли точки обновления оборудования для существующей инфраструктуры, частное облако AVS позволяет расширить, не требующее новых инвестиций в оборудование.

* **Быстрое расширение**. Если возникли какие либо временные или незапланированные потребности в емкости, частное облако AVS позволяет создать дополнительную емкость без задержки.

* **Повышенная защита**. Благодаря частному облаку AVS из трех и более узлов вы получаете автоматическую избыточность и защиту высокого уровня доступности.

* **Долгосрочные потребности в инфраструктуре**. Если ваши центры обработки данных находятся в емкости или вы хотите изменить структуру, чтобы снизить затраты, частное облако AVS позволяет выпустить центры обработки данных и перейти на облачное решение, в то же время сохраняя совместимость с корпоративными операциями.

При создании частного облака AVS вы получаете один кластер vSphere и все виртуальные машины управления, созданные в этом кластере.

## <a name="before-you-begin"></a>Перед началом работы

Прежде чем можно будет создать частное облако AVS, необходимо подготовить узлы. Дополнительные сведения о подготовке узлов см. в статье [подготовка узлов для решений VMware для Azure (AVS)](create-nodes.md).

Выделите диапазон CIDR для подсетей vSphere/vSAN для частного облака AVS. Частное облако AVS создается в виде изолированной среды стека VMware (с узлами ESXi, vCenter, vSAN и НСКС), управляемыми сервером vCenter. Компоненты управления развертываются в сети, выбранной для подсетей vSphere/vSAN CIDR. Диапазон CIDR сети делится на разные подсети во время развертывания. Адресное пространство подсети vSphere/vSAN должно быть уникальным. Она не должна пересекаться с сетью, которая взаимодействует с средой AVS. Сети, взаимодействующие с AVS, включают в себя локальные сети и виртуальные сети Azure. Дополнительные сведения о подсетях vSphere/vSAN см. в разделе Общие сведения о виртуальных ЛС и подсетях.

* Минимальная vSphere/vSAN подсети префикс диапазона CIDR:/24
* Максимальное число подсетей vSphere/vSAN. префикс диапазона CIDR:/21


## <a name="access-the-avs-portal"></a>Доступ к порталу AVS

Доступ к [порталу AVS](access-cloudsimple-portal.md).

## <a name="create-a-new-avs-private-cloud"></a>Создание частного облака AVS

1. Выбор пункта **Все службы**.
2. Поиск **служб AVS**.
3. Выберите службу AVS, на которой нужно создать частное облако AVS.
4. В разделе **Обзор**щелкните **создать частное облако AVS** , чтобы открыть новую вкладку браузера для портала AVS. При появлении запроса войдите с помощью учетных данных для входа в Azure.

    ![Создание частного облака AVS из Azure](media/create-private-cloud-from-azure.png)

5. На портале AVS укажите имя для частного облака AVS.
6. Выберите **Расположение** для частного облака AVS.
7. Выберите **тип узла**в соответствии с тем, что вы приобрели в Azure.
8. Укажите **количество узлов**. Для создания частного облака AVS требуется по крайней мере три узла.
5. На портале Клаудсимпле укажите имя частного облака.
6. Выберите **Расположение** для частного облака.
7. Выберите **тип узла**в соответствии с тем, что подготовлено в Azure.
8. Укажите **количество узлов**.  Для создания частного облака требуется по крайней мере три узла.

    ![Создание частного облака AVS — основные сведения](media/create-private-cloud-basic-info.png)

9. Нажмите кнопку **Далее: дополнительные параметры**.
10. Введите диапазон CIDR для подсетей vSphere/vSAN. Убедитесь, что диапазон CIDR не пересекается с какими-либо локальными или другими подсетями Azure (виртуальными сетями) или с подсетью шлюза.

    **Параметры диапазона CIDR:** /24,/23,/22 или/21. Диапазон CIDR/24 поддерживает до девяти узлов, а также диапазон CIDR/23 поддерживает до 41 узлов, а диапазон значений CIDR/22 и/21 поддерживает до 64 узлов (максимальное число узлов в частном облаке AVS).

    > [!IMPORTANT]
    > IP-адреса в диапазоне CIDR vSphere/vSAN зарезервированы для использования в инфраструктуре частного облака AVS. Не используйте этот IP-адрес в этом диапазоне на любой виртуальной машине.

11. Нажмите кнопку **Далее: Проверьте и создайте**.
12. Проверьте параметры. Если необходимо изменить какие бы то ни было параметры, нажмите кнопку **назад**.
13. Нажмите кнопку **Создать**.

Начинается процесс подготовки частного облака AVS. Подготовка частного облака AVS может занять до двух часов.

Инструкции по развертыванию существующего частного облака AVS см. в разделе [расширение частного облака AVS](expand-private-cloud.md).

# <a name="create-a-cloudsimple-private-cloud"></a>Создание частного облака Клаудсимпле

Создание частного облака позволяет решать различные распространенные потребности в сетевой инфраструктуре.

* **Рост**. Если вы достигли точки обновления оборудования для существующей инфраструктуры, частное облако позволяет расширить, не требующее новых инвестиций в оборудование.

* **Быстрое расширение**. Если возникли какие-либо временные или незапланированные потребности в емкости, частное облако позволяет создать дополнительную емкость без задержки.

* **Повышенная защита**. С частным облаком из трех и более узлов вы получаете автоматическую избыточность и защиту высокого уровня доступности.

* **Долгосрочные потребности**в инфраструктуре. Если ваши центры обработки данных находятся в емкости или вы хотите изменить структуру, чтобы снизить затраты, частное облако позволяет выключать центры обработки данных и переходить на облачное решение, в то время как оно остается совместимым с корпоративными операциями.

При создании частного облака вы получаете один кластер vSphere и все виртуальные машины управления, созданные в этом кластере.

## <a name="before-you-begin"></a>Перед началом работы

Прежде чем можно будет создать частное облако, необходимо подготовить узлы.  Дополнительные сведения о подготовке узлов см. в статье [подготовка узлов для решения VMware с помощью клаудсимпле-Azure](create-nodes.md) .

Выделите диапазон CIDR для подсетей vSphere/vSAN для частного облака. Частное облако создается в виде изолированного стека VMware (ESXi hosts, vCenter, vSAN и НСКС), управляемого сервером vCenter. Компоненты управления развертываются в сети, выбранной для подсетей vSphere/vSAN CIDR. Диапазон CIDR сети делится на разные подсети во время развертывания.  Адресное пространство подсети vSphere/vSAN должно быть уникальным. Она не должна перекрываться с сетью, которая взаимодействует с средой Клаудсимпле.  Сети, взаимодействующие с Клаудсимпле, включают в себя локальные сети и виртуальные сети Azure.  Дополнительные сведения о подсетях vSphere/vSAN см. в разделе [Общие сведения о виртуальных ЛС и подсетях](cloudsimple-vlans-subnets.md).

* Минимальная vSphere/vSAN подсети префикс диапазона CIDR:/24 
* Максимальное число подсетей vSphere/vSAN. префикс диапазона CIDR:/21

## <a name="create-a-new-private-cloud"></a>Создание частного облака

1. На странице **ресурсы** щелкните **создать частное облако**.

    ![Создание частного облака — начало работы](media/create-pc-button.png)

2. Выберите расположение для размещения ресурсов частного облака.

3. Выберите тип узла CS28 или CS36 йоу'ев, подготовленный для частного облака. Последний вариант включает максимальный объем вычислений и объем памяти.

4. Выберите число узлов для частного облака. Вы можете выбрать не более чем количество узлов, которые йоу'ев [подготовлены](create-nodes.md).

    ![Создание частного облака — основные параметры](media/create-private-cloud-basic-info.png)

5. Щелкните **Далее: Дополнительные параметры**.

6. Введите диапазон CIDR для подсетей vSphere/vSAN. Убедитесь, что диапазон CIDR не пересекается с какими-либо локальными или другими подсетями Azure (виртуальными сетями) или с подсетью шлюза.  Не используйте диапазон CIDR, определенный в виртуальных сетях Azure.
    
    **Параметры диапазона CIDR:** /24,/23,/22 или/21. Диапазон CIDR/24 поддерживает до девяти узлов, а также диапазон CIDR/23 поддерживает до 41 узлов, а диапазон значений CIDR/22 и/21 поддерживает до 64 узлов (максимальное количество узлов в частном облаке).

    > [!CAUTION]
    > IP-адреса в диапазоне CIDR vSphere/vSAN зарезервированы для использования инфраструктурой частного облака.  Не используйте этот IP-адрес в этом диапазоне на любой виртуальной машине.

7. Щелкните **Далее: Проверьте и создайте**.

8. Проверьте параметры. Если необходимо изменить какие бы то ни было параметры, нажмите кнопку **назад**.

9. Нажмите кнопку **Создать**.

Подготовка частного облака начнется после нажатия кнопки Создать.  Вы можете отслеживать ход выполнения на странице [задач](https://docs.azure.cloudsimple.com/activity/#tasks) на портале клаудсимпле.  Подготовка может занять от 30 минут до двух часов.  После завершения подготовки вы получите сообщение электронной почты.

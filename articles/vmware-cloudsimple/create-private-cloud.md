# <a name="create-a-cloudsimple-private-cloud"></a>Создание частного облака CloudSimple

Создание частного облака помогает используется целый ряд распространенных задач для сетевой инфраструктуры.

* **Рост**. Если достигнута точка обновления оборудования для существующей инфраструктуры частного облака позволяет расширить без новых вложений оборудования, который требуется.

* **Быстрое расширение**. Если любые временные или незапланированных емкость должна возникнуть, частного облака позволяет создать дополнительную емкость без задержек.

* **Увеличенная степень защиты**. С помощью частного облака из трех или более узлов вы получаете автоматической избыточности и защиты высокого уровня доступности.

* **Долгосрочное инфраструктура должна**. Если достигнуто предельное число свои центры обработки данных или требуется изменить структуру к снижению затрат, частного облака можно снять с учета центров обработки данных и перенести в Облачное решение, оставаясь совместим с работой enterprise.

При создании частного облака, вы получаете кластер с одной vSphere и все виртуальные машины, созданные в этом кластере управления.

## <a name="before-you-begin"></a>Перед началом работы

Узлы должны быть подготовлены, перед созданием вашего частного облака.  Дополнительные сведения о подготовке узлов см. в разделе [Подготовка узлов VMware решения с CloudSimple - Azure](create-nodes.md) статьи.

## <a name="sign-in-to-azure"></a>Вход в Azure

Войдите на портал Azure по адресу [https://portal.azure.com](https://portal.azure.com).

## <a name="access-the-cloudsimple-portal"></a>Доступ к порталу CloudSimple

Доступ [CloudSimple портала](access-cloudsimple-portal.md).

## <a name="create-a-new-private-cloud"></a>Создание нового частного облака

1. На **ресурсы** щелкните **новый частного облака**.

    ![Создание частного облака — запуск](media/create-pc-button.png)

2. Выберите расположение для размещения ресурсов частного облака.

3. Выберите CS28 или CS36 узел типа you'ev подготовлены для частного облака. Последний вариант включает в себя Максимальная емкость вычислительных ресурсов и памяти.

4. Выберите количество узлов для частного облака. Можно выбрать не более число доступных узлов, you'ev [подготовлены](create-nodes.md).

    ![Создание частного облака — основные параметры](media/create-private-cloud-basic-info.png)

5. Щелкните **Далее: Дополнительные параметры**.

6. Введите диапазон CIDR для подсети vSphere/виртуальной сети хранения данных. Убедитесь, что диапазон CIDR не пересекается с любым из собственных или других подсетей Azure (виртуальные сети) или с подсетью шлюза.  Не используйте любой диапазон CIDR, определенные в виртуальных сетях Azure.
    
    **Параметры диапазона CIDR:** диапазоном/24, /23, / 22 до или /21. A/24 диапазон CIDR поддерживает до девяти узлов, /23 диапазон CIDR поддерживает до 41 узлы "," / 22 до "и" /21 диапазон CIDR поддерживает до 64 узлов (максимальное количество узлов в частном облаке).

    > [!CAUTION]
    > IP-адреса в диапазоне CIDR vSphere или vSAN зарезервирован для использования в инфраструктуре частного облака.  Не используйте IP-адрес в этом диапазоне на любой виртуальной машине.

7. Щелкните **Далее: Просмотр и создание**.

8. Проверьте параметры. Если вам нужно изменить какие-либо параметры, нажмите кнопку **Назад**.

9. Нажмите кнопку **Создать**.

Частное облако подготовки начнется, как только вы нажмете кнопку Создать.  Вы можете отслеживать ход выполнения из [задачи](https://docs.azure.cloudsimple.com/activity/#tasks) странице CloudSimple портала.  Подготовка может занять 30 минут до двух часов.  После завершения подготовки вы получите сообщение электронной почты.

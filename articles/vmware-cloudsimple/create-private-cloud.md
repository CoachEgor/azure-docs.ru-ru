# <a name="create-a-cloudsimple-private-cloud"></a>Создание облачного частного облака

Частное облако — это изолированный стек VMware, который поддерживает хосты ESXi, vCenter, vSAN и NSX. Частные облака управляются через портал CloudSimple. Они имеют свой собственный сервер vCenter в своей собственной области управления. Стек работает на выделенных узлах и изолированных голых металлических аппаратных узлов.

Создание частного облака помогает удовлетворить различные общие потребности в сетевой инфраструктуре:

* **Рост**. Если вы достигли точки обновления оборудования для существующей инфраструктуры, частное облако позволяет расширяться без каких-либо новых инвестиций в оборудование.

* **Быстрое расширение**. При возникновении каких-либо временных или незапланированных потребностей в емкости частное облако позволяет создавать дополнительную емкость без задержек.

* **Повышенная защита.** С помощью частного облака из трех или более узлов вы получаете автоматическую избыточность и высокую защиту доступности.

* **Долгосрочные потребности в инфраструктуре.** Если центры обработки данных работают или вы хотите реструктурировать, чтобы снизить затраты, частное облако позволяет вывести центры обработки данных из эксплуатации и перейти на облачное решение, оставаясь совместимым с корпоративными операциями.

При создании частного облака вы получаете единый кластер vSphere и все вычисления, созданные в этом кластере.

## <a name="before-you-begin"></a>Перед началом

Узлы должны быть подготовлены, прежде чем вы сможете создать частное облако. Для получения дополнительной информации о [Provision nodes for Azure VMware Solution by CloudSimple](create-nodes.md)подготовке узлов см.

Выделите диапазон CIDR для подсетей vSphere/vSAN для частного облака. Частное облако создается как изолированная среда стека VMware (с хостами ESXi, vCenter, vSAN и NSX), управляемой сервером vCenter. Компоненты управления развертываются в сети, которая выбрана для подсетей vSphere/vSAN CIDR. Диапазон сети CIDR делится на различные подсети во время развертывания. Адресное пространство vSphere/vSAN должно быть уникальным. Он не должен пересекаться с любой сетью, которая общается с средой CloudSimple. Сети, взаимодействуют с CloudSimple, включают в себя прикрытые сети и виртуальные сети Azure. Для получения дополнительной информации о подсетях vSphere/vSAN см.

* Минимальный vSphere/vSAN подсети CIDR диапазон префикс: /24
* Максимальная префикс диапазона диапазона vSphere/vSAN: /21

## <a name="create-a-new-private-cloud"></a>Создание нового приватного облака

1. Выберите **все услуги**.
2. Поиск **облачных сервисов CloudSimple**.
3. Выберите сервис CloudSimple, на котором вы хотите создать частное облако.
4. От **обзора**нажмите **Создать частное облако,** чтобы открыть новую вкладку браузера для портала CloudSimple. Если побуждение, вопийте со своим знаком Azure в учетных данных.

    ![Создание частного облака из Azure](media/create-private-cloud-from-azure.png)

5. На портале CloudSimple укажите имя для вашего частного облака.
6. Выберите **Место для** частного облака.
7. Выберите **тип узла,** в соответствии с тем, что вы приковали к Azure.
8. Укажите **количество узлов**.  Для создания частного облака требуется не менее трех узлов.

    ![Создание частного облака - Основная информация](media/create-private-cloud-basic-info.png)

9. Нажмите **далее: Расширенные опции**.
10. Введите диапазон CIDR для подсетей vSphere/vSAN. Убедитесь, что диапазон CIDR не пересекается ни с одной из ваших подсетей Azure (виртуальными сетями) или с подсетью шлюза.

    **Варианты диапазона CIDR:** /24, /23, /22, или /21. Диапазон A /24 CIDR поддерживает до девяти узлов, диапазон CIDR /23 поддерживает до 41 узла, а диапазон /22 и /21 CIDR поддерживает до 64 узлов (максимальное количество узлов в частном облаке).

    > [!IMPORTANT]
    > IP-адреса в диапазоне vSphere/vSAN CIDR зарезервированы для использования инфраструктурой Private Cloud.  Не используйте IP-адрес в этом диапазоне на любой виртуальной машине.

11. Нажмите **далее: Просмотрите и создайте**.
12. Просмотрите настройки. Если вам нужно изменить какие-либо настройки, нажмите **Предыдущий**.
13. Нажмите кнопку **Создать**.

Начинается процесс подготовки частного облака. Подготовка частного облака может занять до двух часов.

Для получения инструкций по расширению существующего частного облака [см.](expand-private-cloud.md)

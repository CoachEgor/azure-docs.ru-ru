# <a name="expand-a-cloudsimple-private-cloud"></a>Расширьте облачное частное облако

CloudSimple обеспечивает гибкость для динамического расширения частного облака. Вы можете начать с меньшей конфигурации, а затем расширить, как вам нужно большей емкости. Или вы можете создать частное облако на основе текущих потребностей, а затем расширить по мере роста потребления.

Частное облако состоит из одного или нескольких кластеров vSphere. Каждый кластер может иметь от 3 до 16 узлов.  При расширении частного облака необходимо добавить узлы к существующему кластеру или создать новый кластер. Для расширения существующего кластера дополнительные узлы должны быть того же типа (SKU), что и существующие узлы. Для создания нового кластера узлы могут быть другого типа. Для получения дополнительной информации о лимитах частного облака смотрите раздел «Ограничения» в статье [обзора частного облака CloudSimple.](cloudsimple-private-cloud.md)

Частное облако создается с **центром обработки данных** по умолчанию на vCenter.  Каждый центр обработки данных служит управленческим предприятием высшего уровня.  Для нового кластера CloudSimple предоставляет возможность добавления в существующий центр обработки данных или создания нового центра обработки данных.

В рамках новой конфигурации кластера CloudSimple настраивает инфраструктуру VMware.  Настройки включают настройки хранения для групп дисков vSAN, высокой доступности VMware и распределенного планировщика ресурсов (DRS).

Частное облако может быть расширено несколько раз. Расширение может быть сделано только тогда, когда вы остаетесь в пределах общего узла. Каждый раз, когда вы расширяете приватное облако, вы добавляете в существующий кластер или создаете новый.

## <a name="before-you-begin"></a>Перед началом

Узлы должны быть подготовлены, прежде чем вы сможете расширить частное облако.  Более подробную информацию о подготовке узлов можно найти в статье [«Положение» для решения VMware от CloudSimple - Azure.](create-nodes.md)  Для создания нового кластера необходимо иметь не менее трех доступных узлов одного и того же SKU.

## <a name="sign-in-to-azure"></a>Вход в Azure

Войти на портал Azure по адресу [https://portal.azure.com](https://portal.azure.com).

## <a name="expand-a-private-cloud"></a>Расширение частного облака

1. [Доступ к порталу CloudSimple](access-cloudsimple-portal.md).

2. Откройте страницу **ресурсов** и выберите частное облако, для которого вы хотите расширить.

3. В разделе резюме нажмите **Expand**.

    ![Расширить частное облако](media/resources-expand-private-cloud.png)

4. Выберите, следует ли расширить существующий кластер или создать новый кластер vSphere. При внесении изменений сводка информации на странице обновляется.

    * Чтобы расширить существующий кластер, нажмите **«Расширить существующий кластер».** Выберите кластер, который необходимо расширить, и введите число узлов для добавления. Каждый кластер может иметь максимум 16 узлов.
    * Чтобы добавить новый кластер, нажмите **Создать новый кластер.** Введите имя кластера. Выберите существующий центр обработки данных или введите имя для создания нового центра обработки данных. Выберите тип узла. При создании нового кластера vSphere можно выбрать другой тип узлов, но не при расширении существующего кластера vSphere. Выберите количество узлов. Каждый новый кластер должен иметь не менее трех узлов.

    ![Расширить частное облако - добавить узлы](media/resources-expand-private-cloud-add-nodes.png)

5. Нажмите **Отправить,** чтобы расширить частное облако.

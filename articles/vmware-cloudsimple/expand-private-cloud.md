# <a name="expand-a-cloudsimple-private-cloud"></a>Разверните CloudSimple частного облака

CloudSimple предоставляет возможность динамически расширять частного облака. Вы можете начать с исходной небольшой конфигурации, затем разверните необходимое пропускную способность. Или можно создать частное облако на основе текущих потребностей и раскройте при увеличении потребления.

Частное облако состоит из одного или нескольких кластерах vSphere. Каждый кластер может содержать 3 до 16 узлов.  При развертывании частного облака, можно добавить узлы в существующий кластер, или создать новый кластер. Чтобы развернуть кластер, дополнительные узлы должны быть одного типа (SKU), что и существующие узлы. Для создания нового кластера, узлы могут быть другого типа. Дополнительные сведения об ограничениях частного облака см. в разделе ограничивает статьи [обзор частного облака CloudSimple](cloudsimple-private-cloud.md) статьи.

Частное облако создается по умолчанию **центра обработки данных** на vCenter.  Каждый центр обработки данных служит объектом управления верхнего уровня.  Для нового кластера CloudSimple предоставляет выбор из добавления имеющегося центра обработки данных или создания нового центра обработки данных.

В рамках новой конфигурации кластера CloudSimple настраивает инфраструктура VMware.  Параметры включают параметры хранилища для группы дисков виртуальной сети хранения данных, высокий уровень доступности VMware и Distributed Resource Scheduler (DRS).

Частное облако можно развернуть несколько раз. Расширения можно сделать только в том случае, если вы остаетесь в пределах общего узла. Каждый раз, разверните в частном облаке, добавление к существующему кластеру или создайте новую.

## <a name="before-you-begin"></a>Перед началом работы

Узлы должны быть подготовлены перед развертыванием частным облаком.  Дополнительные сведения о подготовке узлов см. в разделе [Подготовка узлов VMware решения с CloudSimple - Azure](create-nodes.md) статьи.  Для создания нового кластера, необходимо иметь по крайней мере три доступные узлы одного номера SKU.

## <a name="sign-in-to-azure"></a>Вход в Azure

Войдите на портал Azure по адресу [https://portal.azure.com](https://portal.azure.com).

## <a name="expand-a-private-cloud"></a>Разверните частного облака

1. [Доступ к порталу CloudSimple](access-cloudsimple-portal.md).

2. Откройте **ресурсы** страницы и выберите частное облако, для которого вы хотите развернуть.

3. В разделе сводки нажмите кнопку **Expand**.

    ![Разверните частного облака](media/resources-expand-private-cloud.png)

4. Выберите, следует ли развернуть существующего кластера или создать новый кластер vSphere. При внесении изменений, обновляется сводные данные на странице.

    * Чтобы развернуть существующий кластер, щелкните **разверните существующий кластер**. Выберите кластер, который требуется развернуть и введите количество узлов для добавления. Каждый кластер может иметь до 16 узлов.
    * Чтобы добавить новый кластер, щелкните **создать новый кластер**. Введите имя кластера. Выберите существующий центр данных, или введите имя для создания нового центра обработки данных. Выберите тип узла. При создании нового кластера vSphere, но не в том случае, при расширении существующего кластера vSphere можно выбрать разные типы узла. Выберите количество узлов. Каждый новый кластер должен содержать по крайней мере три узла.

    ![Разверните частного облака — Добавление узлов](media/resources-expand-private-cloud-add-nodes.png)

5. Нажмите кнопку **отправить** для развертывания частного облака.

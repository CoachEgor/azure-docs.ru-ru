# <a name="configure-a-high-availability-connection-from-on-premises-to-cloudsimple-vpn-gateway"></a>Налаживание высокой доступности соединения с предприимчивого к VPN-шлюзу CloudSimple

Сетевые администраторы могут настроить высокодоступное VPN-соединение IPsec Site-to-Site из своей собственной среды на VPN-шлюз CloudSimple.

В этом руководстве представлены шаги по настройке предпосылок для подключения к VPN-сайту IPsec Site к сайту. Подробные шаги специфичны для типа предприимного брандмауэра. В качестве примеров в этом руководстве приводятся шаги для двух типов брандмауэров: Cisco ASA и Palo Alto Networks.

## <a name="before-you-begin"></a>Перед началом

Выполните следующие задачи перед настройкой предпосылок брандмауэра.

1. Убедитесь, что организация [подготовила](create-nodes.md) необходимые узлы и создала по крайней мере одно облачное частное облако.
2. [Настройте VPN шлюз от сайта к сайту](vpn-gateway.md#set-up-a-site-to-site-vpn-gateway) между вашей предварительной сетью и вашим облачным частным облаком CloudSimple Private Cloud.

Смотрите [обзор VPN шлюзов](cloudsimple-vpn-gateways.md) для поддерживаемых предложений фазы 1 и фазы 2.

## <a name="configure-on-premises-cisco-asa-firewall"></a>Настройка на месте брандмауэра Cisco ASA

Инструкции в этом разделе применяются к версии Cisco ASA 8.4 и позже. В примере конфигурации используется и настраивается в режиме IKEv1 и настроена версия программного обеспечения для адаптивной безопасности Cisco.

Для работы VPN от сайта к сайту необходимо разрешить UDP 500/4500 и ESP (IP-протокол 50) от первичного и вторичного публичного IP-адреса CloudSimple (peer IP) на внешнем интерфейсе предприимчивого шлюза Cisco ASA VPN.

### <a name="1-configure-phase-1-ikev1"></a>1. Настройка фазы 1 (IKEv1)

Чтобы включить фазу 1 (IKEv1) на внешнем интерфейсе, введите следующую команду CLI в брандмауэре Cisco ASA.

```crypto ikev1 enable outside```

### <a name="2-create-an-ikev1-policy"></a>2. Создание полиса IKEv1

Создайте политику IKEv1, которая определяет алгоритмы и методы, которые будут использоваться для хэширования, аутентификации, группы Диффи-Хеллмана, срока службы и шифрования.

```
crypto ikev1 policy 1
authentication pre-share
encryption aes-256
hash sha
group 2
lifetime 28800
```

### <a name="3-create-a-tunnel-group"></a>3. Создание группы туннелей

Создайте группу туннелей под атрибутами IPsec. Настройка однорангового IP-адреса и предварительного общего ключа туннеля, который вы устанавливаете при [настройке VPN шлюза от сайта к сайту.](vpn-gateway.md#set-up-a-site-to-site-vpn-gateway)

```
tunnel-group <primary peer ip> type ipsec-l2l
tunnel-group <primary peer ip> ipsec-attributes
ikev1 pre-shared-key *****

tunnel-group <secondary peer ip> type ipsec-l2l
tunnel-group <secondary peer ip> ipsec-attributes
ikev1 pre-shared-key *****
```

### <a name="4-configure-phase-2-ipsec"></a>4. Настройка фазы 2 (IPsec)

Для настройки фазы 2 (IPsec) создайте список управления доступом (ACL), который определяет трафик для шифрования и туннелирования. В следующем примере интерес к нему представляет сяртование из туннеля, полученного из локальной подсети (10.16.1.0/24) в удаленную подсеть Private Cloud (192.168.0.0/24). ACL может содержать несколько записей, если между сайтами есть несколько подсетей.

В версиях Cisco ASA 8.4 и позже могут создаваться объекты или группы объектов, которые служат контейнерами для сетей, подсетей, IP-адресов хоста или нескольких объектов. Создайте объект для локального и объект для удаленных подсетей и используйте их для криптоA ACL и заявлений NAT.

#### <a name="define-an-on-premises-local-subnet-as-an-object"></a>Определение локальной подсети как объекта

```
object network AZ_inside
subnet 10.16.1.0 255.255.255.0
```

#### <a name="define-the-cloudsimple-remote-subnet-as-an-object"></a>Определите удаленную подсеть CloudSimple как объект

```
object network CS_inside
subnet 192.168.0.0 255.255.255.0
```

#### <a name="configure-an-access-list-for-the-traffic-of-interest"></a>Настройка списка доступа для трафика интересов

```
access-list ipsec-acl extended permit ip object AZ_inside object CS_inside
```

### <a name="5-configure-the-transform-set"></a>5. Настройка набора преобразований

Настроили набор преобразования (TS), который ```ikev1```должен включать ключевое слово. Атрибуты шифрования и хэша, указанные в ТС, должны соответствовать параметрам, перечисленным в [конфигурации По умолчанию для шлюзов CloudSimple VPN.](cloudsimple-vpn-gateways.md)

```
crypto ipsec ikev1 transform-set devtest39 esp-aes-256 esp-sha-hmac 
```

### <a name="6-configure-the-crypto-map"></a>6. Настройка криптокарты

Нанастройка криптокарты, которая содержит следующие компоненты:

* Одноранговый IP-адрес
* Определенная ACL, которая содержит трафик интересов
* Преобразование набора

```
crypto map mymap 1 set peer <primary peer ip> <secondary peer ip>
crypto map mymap 1 match address ipsec-acl
crypto map mymap 1 set ikev1 transform-set devtest39
```

### <a name="7-apply-the-crypto-map"></a>7. Примените криптокарту

Примените криптокарту на внешнем интерфейсе:

```crypto map mymap interface outside```

### <a name="8-confirm-applicable-nat-rules"></a>8. Подтвердите применимые правила NAT

Ниже приводится правило NAT, которое используется. Убедитесь, что VPN трафик не подвергается какой-либо другой правила NAT.

```nat (inside,outside) source static AZ_inside AZ_inside destination static CS_inside CS_inside```

## <a name="configure-on-premises-palo-alto-networks-firewall"></a>Настройка на месте Пало-Альто Сети брандмауэр

Инструкции в этом разделе применяются к версии Palo Alto Networks 7.1 и позже. В этом примере конфигурации, Palo Alto Networks VM-Серии Software Version 8.1.0 развернута и настроена в режиме IKEv1.

Для работы VPN от сайта к сайту необходимо разрешить UDP 500/4500 и ESP (IP-протокол 50) от первичного и вторичного публичного IP-адреса CloudSimple (peer IP) на внешнем интерфейсе шлюза Palo Alto Networks.

### <a name="1-create-primary-and-secondary-tunnel-interfaces"></a>1. Создание первичных и вторичных туннельных интерфейсов

Вопийте в брандмауэре Пало-Альто, выберите **Network** > **Interfaces** > **Tunnel** > **Add,** настройте следующие поля и **нажмите OK.**

* Название интерфейса. Первое поле автозаселено с ключевым словом «туннель». В соседнем поле введите любое число между 1 до 9999. Этот интерфейс будет использоваться в качестве основного интерфейса туннеля для переноса трафика от сайта к сайту между центром обработки данных и частным облаком.
* Комментарий Введите комментарии для легкой идентификации цели туннеля
* Профиль netflow. Оставьте значение по умолчанию.
* Config. Назначить интерфейс для: Виртуальный маршрутизатор: Выберите **по умолчанию.** 
        Зона безопасности: Выберите зону для надежного LAN-трафика. В этом примере название зоны для LAN-трафика — «Доверие».
* IP версии 4. Нажмите **Добавить** и добавить любой неперекрывающийся неиспользованный /32 IP-адрес в вашей среде, который будет назначен интерфейсу основного туннеля и будет использоваться для мониторинга туннелей (объясняется позже).

Поскольку эта конфигурация предназначена для VPN с высокой доступностью, требуется два туннельных интерфейса: один первичный и один вторичный. Повторите предыдущие шаги для создания вторичного интерфейса туннеля. Выберите другой идентификатор туннеля и другой неиспользованный IP-адрес /32.

### <a name="2-set-up-static-routes-for-private-cloud-subnets-to-be-reached-over-the-site-to-site-vpn"></a>2. Настройка статических маршрутов для подсетей Private Cloud, которые будут использоваться через VPN сайта к сайту

Маршруты необходимы для того, чтобы закрытые подсети достигли частных облачных подсетей CloudSimple.

Выберите **сеть** > **виртуальных маршрутизаторов** > *по умолчанию* > **Статических маршрутов** > **Добавить,** настроить следующие поля, и нажмите **OK**.

* Название Введите любое имя для легкой идентификации цели маршрута.
* Целевой объект. Укажите подсети облачных технологий CloudSimple, которые будут использоваться по интерфейсам туннелей S2S из предварительных
* Интерфейс. Выберите интерфейс первичного туннеля, созданный в шаге-1 (раздел-2) из выпадения. В этом примере это туннель.20.
* Следующий хоп. Выберите **Отсутствует**.
* Расстояние админа. Оставьте значение по умолчанию.
* Метрические. Введите любое значение от 1 до 65535. Ключ должен ввести более низкую метрику для маршрута, соответствующую интерфейсу первичного туннеля по сравнению с маршрутом, соответствующим вторичным интерфейсом туннеля, что делает прежний маршрут предпочтительным. Если tunnel.20 имеет метрическое значение 20 в отличие от метрического значения 30 для tunnel.30, то предпочтительнее туннель.20.
* Таблица маршрутов. Оставьте значение по умолчанию.
* Профиль BFD. Оставьте значение по умолчанию.
* Мониторинг пути. не устанавливайте флажок.

Повторите предыдущие шаги, чтобы создать другой маршрут для подсетей Private Cloud для использования в качестве вторичного/резервного маршрута через вторичный интерфейс туннеля. На этот раз выберите другой идентификатор туннеля и более высокую метрику, чем для основного маршрута.

### <a name="3-define-the-cryptographic-profile"></a>3. Определите криптографический профиль

Определите криптографический профиль, который определяет протоколы и алгоритмы идентификации, аутентификации и шифрования, которые будут использоваться для настройки VPN туннелей в IKEv1 Phase 1.

Выберите **Сеть** > **Расширить сети профилей** > **IKE Crypto** > **Добавить,** настроить следующие поля, и нажмите **OK**.

* Название Введите любое имя криптографического профиля IKE.
* DH Group. Нажмите **Добавить** и выберите соответствующую группу DH.
* Шифрование. Нажмите **Добавить** и выберите подходящий метод шифрования.
* аутентификация; Нажмите **Добавить** и выберите подходящий метод проверки подлинности.
* Ключевая продолжительность жизни. Оставьте значение по умолчанию.
* ИКЕВ2 Аутентификация несколько. Оставьте значение по умолчанию.

### <a name="4-define-ike-gateways"></a>4. Определите шлюзы IKE

Определите шлюзы IKE для установления связи между узлами на каждом конце vpn-туннеля.

Выберите **Сеть** > **Расширить сети профилей** > **IKE шлюзы** > **Добавить,** настроить следующие поля, и нажмите **OK**.

Общая вкладка:

* Название Введите название шлюза IKE, который будет всматриваться с основным сверстником CloudSimple VPN.
* Версия. Выберите **только режим IKEv1.**
* Тип адреса. Выберите **IPv4**.
* Интерфейс. Выберите общедоступный интерфейс или внешний интерфейс.
* Местный IP-адрес. Оставьте значение по умолчанию.
* Тип одноранговой IP-адреса. Выберите **IP**.
* Адрес одноранговой раны. Введите основной одноранговой IP-адрес CloudSimple VPN.
* аутентификация; Выберите **предварительно общий ключ**.
* Предварительно совместное ключ / Подтвердите предварительное совместное ключ. Введите предварительно общий ключ, чтобы соответствовать ключу шлюза CloudSimple VPN.
* Местная идентификация. Введите общедоступный IP-адрес предварительного брандмауэра Palo Alto.
* Идентификация сверстников. Введите основной одноранговой IP-адрес CloudSimple VPN.

Вкладка Расширенные опции:

* Включить пассивный режим. не устанавливайте флажок.
* Включить NAT Traversal. Оставьте без внимания, если в помещении брандмауэра Palo Alto нет ни одного устройства NAT. В противном случае выберите флажок.

IKEv1:

* Режим обмена. Выберите **основной**.
* IKE Крипто Профиль. Выберите профиль IKE Crypto, созданный ранее. Не остановить поле фрагментации включить.
* Обнаружение мертвых сверстников. Оставьте коробку без внимания.

Повторите предыдущие шаги для создания вторичного шлюза IKE.

### <a name="5-define-ipsec-crypto-profiles"></a>5. Определите профили IPSEC Crypto

Выберите **Сеть** > **Расширить сети профилей** > **IPSEC Crypto** > **Добавить,** настроить следующие поля, и нажмите **OK**.

* Название Введите имя криптографического профиля IPsec.
* Протокол IPsec. Выберите **ESP**.
* Шифрование. Нажмите **Добавить** и выберите подходящий метод шифрования.
* аутентификация; Нажмите **Добавить** и выберите подходящий метод проверки подлинности.
* DH Group. Выберите **no-pfs**.
* Жизни. Установить как 30 минут.
* "Включено". Оставьте коробку без внимания.

Повторите предыдущие шаги для создания еще одного криптопрофиля IPsec, который будет использоваться в качестве вторичного пэра CloudSimple VPN. Тот же профиль IPSEC Crypto также может быть использован как для первичных, так и для вторичных туннелей IPsec (см. следующую процедуру).

### <a name="6-define-monitor-profiles-for-tunnel-monitoring"></a>6. Определить профили мониторов для мониторинга туннелей

Выберите **Сеть** > **Расширить сети профилей** > **монитор** > **добавить,** настроить следующие поля, и нажмите **OK**.

* Название Введите любое название профиля Монитора, которое будет использоваться для мониторинга туннеля для упреждающей реакции на сбой.
* Действий. Выберите **Fail Over**.
* Интервал. Введите значение **3**.
* Пороговое значение. Введите значение **7**.

### <a name="7-set-up-primary-and-secondary-ipsec-tunnels"></a>7. Создание первичных и вторичных туннелей IPsec.

Выберите **сеть** > **IPsec туннели** > **Добавить,** настроить следующие поля, и нажмите **OK**.

Общая вкладка:

* Название Введите любое имя для основного туннеля IPSEC, который будет заглянул с первичным сверстником CloudSimple VPN.
* Интерфейс туннеля. Выберите интерфейс первичного туннеля.
* Введите , Оставьте значение по умолчанию.
* Тип адреса. Выберите **IPv4**.
* IKE Шлюз. Выберите основной шлюз IKE.
* Профиль IPsec Crypto. Выберите основной профиль IPsec. Выберите **Опции Show Advanced.**
* Включить защиту повтора. Оставьте значение по умолчанию.
* Копировать ЗАголовок TOS. Оставьте коробку без внимания.
* Монитор туннеля. Проверь коробку.
* Направление IP. Введите любой IP-адрес, принадлежащий подсети CloudSimple Private Cloud, разрешенную по подключению к сайту к сайту. Убедитесь, что интерфейсы туннеля (например, туннель.20 - 10.64.5.2/32 и tunnel.30 - 10.64.6.2/32) на Пало-Альто могут быть доступны для достижения IP-адреса CloudSimple Private Cloud через VPN сайта.ru. Посмотреть следующую конфигурацию для идотов прокси.
* Профиль. Выберите профиль монитора.

Вкладка прокси-идентипов: Нажмите **IPv4** > **Добавить** и настроить следующее:

* Идентификатор прокси. Введите любое имя для интересного трафика. Там может быть несколько прокси-идол осуществляется внутри одного туннеля IPsec.
* Локальный. Укажите локальные подсети, которым разрешено общаться с подсетями Private Cloud через VPN-сайт.
* Удаленного. Укажите удаленные подсети Private Cloud, которым разрешено общаться с локальными подсетями.
* Протокол. Выберите **любой**из.

Повторите предыдущие шаги по созданию еще одного туннеля IPsec для использования для вторичного пэра CloudSimple VPN.

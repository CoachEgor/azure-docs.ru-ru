# <a name="manage-private-cloud-resources-and-activity"></a>Управление ресурсами частного облака и действия

Частные облака управляются CloudSimple портале.  Проверьте состояние, доступных ресурсов, активности в частном облаке и другие параметры на портале CloudSimple.

## <a name="sign-in-to-azure"></a>Вход в Azure

Войдите на портал Azure по адресу [https://portal.azure.com](https://portal.azure.com).

## <a name="access-the-cloudsimple-portal"></a>Доступ к порталу CloudSimple

Доступ [CloudSimple портала](access-cloudsimple-portal.md).

## <a name="view-the-list-of-private-clouds"></a>Просмотрите список частных облаков

**Частных облаков** вкладке **ресурсы** странице перечислены все частные облака в вашей подписке. Сведения включают имя, количество vSphere кластеров, расположение, текущее состояние данных частного облака и ресурсов.

![Страница частного облака](media/manage-private-cloud.png)

Выберите частное облако для получения дополнительных сведений и действия.

## <a name="private-cloud-summary"></a>Сводка по частному облаку

Просмотрите полную сводку выбранного частного облака.  Страница «Сводка» включает в себя DNS-серверов, развернутых на частного облака.  Можно настроить DNS пересылки на локальный DNS-серверы к серверам облачной частной зоны DNS.  Дополнительные сведения о перенаправление запросов DNS, см. в разделе [Настройка DNS для разрешения имен для сервера vCenter частное облако из локальной](https://docs.azure.cloudsimple.com/on-premises-dns-setup/).

![Сводка частного облака](media/private-cloud-summary.png)

### <a name="available-actions"></a>Доступные действия

* [Запуск клиента vSphere](https://docs.azure.cloudsimple.com/vsphere-access/). Доступ к vCenter, для этого частного облака.
* [Приобрести узлы](create-nodes.md). Добавьте узлы в этом частном облаке.
* [Разверните](expand-private-cloud.md). Добавьте узлы в этом частном облаке.
* **Обновить**. Обновите данные на этой странице.
* **Удалить**. Частное облако можно удалить в любое время. **Перед удалением, убедитесь, что необходимо создать резервные копии всех систем и данных.** Удаление частного облака удалит все виртуальные машины, vCenter конфигурации и данных. Нажмите кнопку **удалить** в разделе сводки для выбранного частного облака. После удаления все данные частного облака удаляются в процессе безопасной, соответствующей высоким требованиям стирания.
* [Изменить права vSphere](escalate-private-cloud-privileges.md).  Эскалировать ваши права доступа на этом частного облака.

## <a name="private-cloud-vlanssubnets"></a>Частные облака виртуальные локальные сети и подсети

Просмотр списка определенных виртуальных локальных сетей, подсетей для выбранного частного облака.  Этот список включает управление создания виртуальных локальных сетей, подсетей при создании частного облака.

![Частное облако — виртуальные локальные сети и подсети](media/private-cloud-vlans-subnets.png) 

### <a name="available-actions"></a>Доступные действия

* [Добавление виртуальных локальных СЕТЕЙ, подсетей](https://docs.azure.cloudsimple.com/create-vlan-subnet/). Добавьте виртуальной ЛС или подмножества этого частного облака.

Выберите подсеть виртуальной локальной сети или для следующих действий.
* [Присоединить таблицу брандмауэра](https://docs.azure.cloudsimple.com/firewall/). Присоедините таблицу брандмауэра для этого частного облака.
* **Edit** (Изменение)
* **Удалить** (только пользовательские виртуальные локальные сети и подсети)

## <a name="private-cloud-activity"></a>Действие частного облака

Просмотрите следующие сведения для выбранного частного облака.  Сведения о действии является отфильтрованный список всех действий для выбранного частного облака.  На этой странице отображаются до 25 последние действия.

* Последние оповещения
* Последние события
* Недавние задачи
* Последние аудита

![Частное облако — действий](media/private-cloud-activity.png)

## <a name="cloud-racks"></a>Облако стоек

Cloud стойках являются стандартными блоками частного облака. Каждая стойка содержит это единица мощности. CloudSimple автоматически настраивает стойках облака, с учетом выбранных при создании или развертывание частного облака.  Просмотреть полный список облачных стойках, включая частное облако, в которых будет назначен.

![Частное облако — стойках облака](media/private-cloud-cloudracks.png)

## <a name="vsphere-management-network"></a>vSphere сеть управления

Список ресурсов для управления VMware и виртуальные машины, которые в настоящий момент на частного облака. Сведения включают версию программного обеспечения, полное доменное имя (FQDN) и IP-адреса ресурсов.

![Частное облако — vSphere сеть управления](media/private-cloud-vsphere-management-network.png)

# <a name="frequently-asked-questions-about-vmware-solution-by-cloudsimple"></a>Часто задаваемые вопросы о решении VMware, CloudSimple

Часто задаваемые вопросы и ответы о решении Azure VMware, CloudSimple, которые помогут вам понять, службы и способы ее использования. Вопросы и ответы упорядочены по следующим категориям:

* Служба CloudSimple
* Возможность подключения
* Сеть
* Безопасность
* Службы вычислений
* Хранилище
* VMware
* Интеграция Azure.
 
## <a name="cloudsimple-service"></a>Служба CloudSimple

**Что такое решения Azure VMware с CloudSimple?**

Решение Azure VMware, CloudSimple преобразует и расширяет рабочих нагрузок VMware в закрытой облака в Azure за несколько минут. Решение подготавливает, управляет инфраструктурой и координирует рабочие нагрузки между локальной сетью и Azure. Поскольку приложения выполняются точно же локально и в Azure, вы воспользоваться преимуществами эластичности и службы в облаке без необходимости изменения архитектуры приложений сложности. CloudSimple снижает совокупную стоимость владения с помощью модели потребления облака, предоставляемой по запросу подготовки, оплата как роста и оптимизация емкости. Функции, преимущества и сценарии, см. в разделе [что такое решения Azure VMware с CloudSimple?](cloudsimple-vmware-solutions-overview.md).

**Что такое частное облако CloudSimple?**

Подготовьте закрытой облако, которое состоит из высокопроизводительными вычислительными ресурсами, хранилища и сетевой среды, развернутые в инфраструктуре Microsoft Azure (место оборудования и центра обработки данных) в расположениях Azure. Частное облако предоставляет собственной платформы VMware как услугу. В терминах VMware каждого частного облака содержит только один экземпляр сервера vCenter. VCenter Server управляет несколько узлов ESXi, содержащихся в один или несколько кластеров vSphere, а также соответствующего хранилища виртуальной сети хранения данных. Служба CloudSimple может содержать несколько частных облаков в вашей подписке Azure. Дополнительные сведения о частных облаках, см. в разделе [обзор частного облака](cloudsimple-private-cloud.md).

**Где доступна служба CloudSimple?**

CloudSimple доступна в восточной части США и Западная часть США.

**Как включить подписку CloudSimple?**

Обратитесь к своему представителю корпорации Майкрософт в [ azurevmwaresales@microsoft.com ](mailto:azurevmwaresales@microsoft.com) Чтобы включить подписку для службы CloudSimple. Отправьте идентификатор вашей подписки по электронной почте, для которого требуется включить службу CloudSimple. 

**Как получить доступ к порталу CloudSimple?**

Получить доступ к порталу CloudSimple на портале Azure. Сведения о том, как получить доступ к порталу CloudSimple, см. в разделе [доступ к решение VMware порталом CloudSimple на портале Azure](https://docs.azure.cloudsimple.com/access-cloudsimple-portal).

**Как увеличить мощность для частного облака?**

Подготовка узлов на портале Azure и разверните частного облака на портале CloudSimple. Вы можете развернуть частного облака, путем добавления узлов к существующему кластеру vSphere, или путем создания нового кластера vSphere. Сведения о процедуре см. в разделе [развернуть в частном облаке CloudSimple](https://docs.azure.cloudsimple.com/expand-private-cloud).

**Что произойдет с моей частного облака, во время обслуживания?**

CloudSimple предоставляет периодических уведомлений дней до запланированного обслуживания. Обслуживание выполняется в непрерывном процессе способ обеспечить доступность частного облака. Обслуживание может быть следующих типов:

- **Инфраструктура CloudSimple**: Инфраструктура CloudSimple предназначен для обеспечения высокой доступности. Во время обслуживания подключение и доступность частного облака за счет обновление избыточные компоненты во время без какого-либо влияния. Имеется доступ к vCenter частного облака, все виртуальные машины, подключение к Интернету из частного облака и подключения к локальной или Azure.
- **Портал CloudSimple**: Во время обслуживания некоторые функции на портале CloudSimple может быть недоступен или может быть отключено. Уведомления об обслуживании содержит сведения о что можно сделать на портале.

## <a name="connectivity"></a>Возможность подключения

**Каковы Мои возможности подключения к сети CloudSimple регионе?**

CloudSimple предоставляет три различные параметры подключения для подключения к сети CloudSimple регион. Все три варианта могут использоваться совместно:

- Azure ExpressRoute с подключением между локальным центром обработки данных к сети региона CloudSimple: Высокоскоростной низкой задержкой безопасного частного подключения, связывающим локального канала ExpressRoute с каналом CloudSimple ExpressRoute с помощью глобальным доступом. Чтобы настроить подключение, см. в разделе [соединиться CloudSimple с помощью ExpressRoute между локальной](https://docs.azure.cloudsimple.com/on-premises-connection).
- ExpressRoute с подключением между вашей виртуальной сети Azure к сети CloudSimple региона: Высокоскоростная и малой задержкой безопасного частного подключения, связывающим виртуальной сети в Azure с каналом CloudSimple ExpressRoute с помощью шлюзов виртуальной сети. Чтобы настроить подключение, см. в разделе [подключить среду частного облака CloudSimple к виртуальной сети Azure с помощью ExpressRoute](https://docs.azure.cloudsimple.com/azure-expressroute-connection).
- Сеть сеть VPN-подключение из локального центра обработки данных к сети CloudSimple региона: Безопасный виртуальной частной сети от локальных VPN-устройства для вашего региона CloudSimple частного облака. Чтобы настроить подключение, см. в разделе [настроить VPN-подключение между вашей локальной сети и частного облака CloudSimple](https://docs.azure.cloudsimple.com/set-up-vpn).

**Как подключаться к частному облаку?**

Можно просмотреть сведения о вашем частном облаке, на портале CloudSimple. Чтобы подключиться к vCenter, соответствующий частного облака, убедитесь, что сетевое подключение устанавливается с помощью сеть сеть "," точка сеть "или" ExpressRoute. Затем чтобы открыть портал CloudSimple на портале Azure. Выберите **запуска клиента vSphere** на домашней странице или на странице сведений о частном облаке.

**Что такое преимущество канала ExpressRoute?**

Канал Azure ExpressRoute обеспечивает безопасное подключение высокоскоростной и малой задержкой. CloudSimple предоставляет выделенного канала ExpressRoute на регион на каждого клиента. С помощью этого канала, можно установить безопасное подключение из локальных и подписке Azure.

**Что такое сетевые затраты для подключения к и из CloudSimple? Взимается ли плата за любой исходящий трафик и из CloudSimple в Azure? Взимается ли плата за любой исходящий трафик между регионами?**

Не взимается плата за исходящий трафик. Azure стандартные ставки на любой исходящий трафик из виртуальной сети или канала ExpressRoute на предприятии.

## <a name="networking"></a>Сеть

**Какие сетевые возможности, доступные для моей частного облака?**

Вы можете подготовить виртуальные локальные сети и подсети и таблицы брандмауэра. Можно назначить общедоступный IP-адресов и сопоставляются с виртуальной машины, которая выполняется в вашем частном облаке. Дополнительные сведения см. в разделе [виртуальные ЛС и подсети Обзор](cloudsimple-vlans-subnets.md), [Общие сведения о брандмауэре таблиц](cloudsimple-firewall-tables.md), и [Обзор адрес общедоступного IP-адреса](cloudsimple-public-ip-address.md).

**Как настроить разные подсети для моих приложений в моей частного облака?**

Можно создать виртуальные локальные сети в вашем частном облаке с портала CloudSimple. После создания виртуальной локальной сети, можно создать группу портов, распределенных на vCenter частного облака с помощью виртуальной локальной сети и создавать виртуальные машины, подключенные к группе распределенных порт. Можно включить таблицу брандмауэра для виртуальной локальной сети или подсети и определить правила брандмауэра для защиты сетевого трафика.

**Какие параметры брандмауэра доступны для моей частных облаков?**

Можно настроить правила для трафика Север Юг и Восток запад. Правила определены в таблице брандмауэра. В таблице брандмауэра можно прикреплять к виртуальные локальные сети в вашем частном облаке. Процедура установки, см. в разделе [Настройка брандмауэра таблиц и правил для частных облаков](https://docs.azure.cloudsimple.com/firewall).

**Можно ли назначить общедоступный IP-адресов для виртуальных машин в моей среде частного облака?**

На портале CloudSimple можно легко выделить новый общедоступный IP-адрес и связать его с помощью частного IP-адрес виртуального компьютера или устройства. Также можно создать новые правила брандмауэра или применить правила брандмауэра, чтобы разрешить трафик от определенных портов и конкретные наборы IP-адресов на портале. Процедура установки, см. в разделе [выделения общедоступных IP-адресов для частных облачных средах](https://docs.azure.cloudsimple.com/public-ips).

## <a name="security"></a>Безопасность

**Каковы параметры безопасности на CloudSimple?**

CloudSimple частное облако предоставляет следующие средства безопасности для защиты частной облачной среде:

- **Данные, шифрование неактивных данных:** Можно шифровать данные при хранении, которая находится в виртуальной сети хранения данных хранилище в частном облаке. Виртуальная сеть хранения данных поддерживает сервер управления ключами, который можно развернуть в виртуальной сети или в локальной среде Azure. Дополнительные сведения см. в разделе [настройки виртуальной сети хранения данных шифрования для частного облака CloudSimple](https://docs.azure.cloudsimple.com/vsan-encryption).
- **Сетевая безопасность:** Управление потоком сетевого трафика от и для частного облака из Интернета, в локальной среде и в пределах подсетей частного облака с помощью правил брандмауэра.
- **Безопасность и конфиденциальность подключения:** Безопасность, частное подключение между вашей локальной сетью и вашей подписке Azure.

## <a name="compute"></a>Службы вычислений

**Какого рода узлы доступны?**

CloudSimple предлагает два типа узлов:

* **Узел CS28**: CPU:2 x 2,2 ГГц, 28 общее количество ядер, 48 HT. ОЗУ. 256 ГБ. Хранилище. Кэш NVMe 1600 ГБ, 5760 ГБ данных (флэш-памяти). Сеть. 2x25Gbe сетевого адаптера.
* **Узел CS36**: ЦП 2 x 2,3 ГГц, 36 ядер, 72 HT. ОЗУ. 512 ГБ. Хранилище. Данные 11,520 ГБ кэша NVMe 3200 ГБ (флэш-памяти). Сеть. 2x25Gbe сетевого адаптера.

**Способ обработки сбоев оборудования?**

Всю инфраструктуру CloudSimple постоянно отслеживается CloudSimple платформы и ее команды операций по обслуживанию. При обнаружении сбоя оборудования, добавляется новый узел частного облака. Удаления неисправного узла, чтобы обеспечить высокий уровень доступности вашего частного облака.

## <a name="storage"></a>Хранилище

**Какой тип хранилища поддерживается в частном облаке?**

Предлагает CloudSimple **хранения vSAN VMware флэш-памяти** с каждого частного облака. Каждый vSphere создается с свой собственный хранилище данных vSAN. Дополнительные сведения см. в разделе [компоненты VMware частного облака — виртуальной сети хранения данных хранилище](https://docs.azure.cloudsimple.com/vmware-components/#vsan-storage).

**Такое шифрование данных поддерживается?**
Да. Можно выполнить настройку виртуальной сети хранения данных хранилища в вашем частном облаке, чтобы использовать сервер управления ключами (KMS), развернутых локально или на платформе Azure для шифрования данных, хранящихся в виртуальной сети хранения данных.

**Как обрабатываются неисправных дисков?**

Мониторинг CloudSimple постоянно отслеживает все аппаратные компоненты частного облака. При обнаружении сбоя диска или диска определяется как сбой в зависимости от эвристики, новый узел автоматически добавляется в частном облаке. Узел с сбоя или недоступный диск удаляется из частного облака.

## <a name="vmware"></a>VMware

**Выполнение отправки большого объема и миграции приложений и данных из локальной?**

CloudSimple предоставляет собственное решение vSphere VMware. Можно использовать любое средство, используемое для массового переноса данных с частным облаком CloudSimple. Ниже приведены некоторые из доступных вариантов.

- HCX VMware для массового переноса данных.
- "Холодных" Перенос данных с помощью Storage vMotion на платформу для CloudSimple.

**Можно установить средства VMware?**

CloudSimple предоставляет собственное решение vSphere VMware. Любой инструмент, используемый для управления средой vSphere на CloudSimple может использоваться в локальной. CloudSimple поддерживает модель bring-your собственной лицензии (BYOL) для установки средств VMware.

**Как управляются обновления?**

CloudSimple управляет и обновляет все компоненты инфраструктуры частного облака в виде простого непрерывном процессе. Как только он уточняется CloudSimple, обновления или исправления, выпущенные VMware или инфраструктуры поставщиков запланировано к обновлению.

CloudSimple не выполняет модернизации или обновлении приложений, установленных в частном облаке.

## <a name="azure-integration"></a>Интеграция Azure.

**Какие службы Azure поддерживаются?**

CloudSimple предоставляет подключение Azure ExpressRoute к вашей подписке в Azure. Все службы, работающих в вашей подписке наличие сетевого подключения для частного облака и могут подключаться к частного облака. Примеры:

- **Azure Active Directory**. Использование Azure Active Directory identity источник CloudSimple vCenter.
- **Служба хранилища Azure**. Использование хранилища для хранения резервных копий, изображения и другие данные о вашем частном облаке.
- **Гибридные приложения**: Вы можете создать приложение архитектуру, которая охватывает общедоступных и частных облаков. Например созданием веб-серверов в Azure, приложение access и серверов баз данных в частном облаке CloudSimple.
- **Azure Monitor** и **центра безопасности**: Рабочая нагрузка, запущенная в VMware можно использовать монитор и центр безопасности для ведения журнала, показатели производительности и управление безопасностью.

**Как сопоставить клиентов VMware в Azure?**

CloudSimple предоставляет уникальные возможности для управления в частном облаке виртуальных машин VMware на портале Azure. Пул ресурсов vCenter настроены ограничения ресурсов, нужных могут сопоставляться с подпиской, глобальным администратором. 

**Какие преимущества лицензирования получить с помощью Azure?**

С помощью CloudSimple можно воспользоваться преимуществами гибридного использования Azure и сэкономьте до 90% на лицензии, чтобы сохранить инвестиции в лицензии Microsoft и сократите совокупную стоимость владения, по сравнению с других облаках. Также получить расширенные обновления для системы безопасности для Windows Server 2008 и Microsoft SQL Server 2008. Свести затраты с BYOL в облако для стандартных приложений, как Veeam, Zerto и другим пользователям.

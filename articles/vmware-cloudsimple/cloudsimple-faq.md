# <a name="frequently-asked-questions-about-vmware-solution-by-cloudsimple"></a>Часто задаваемые вопросы о решении VMware, CloudSimple

Часто задаваемые вопросы и ответы о решении VMware, CloudSimple, которые помогут вам понять, службы и способы ее использования.  Вопросы и ответы упорядочены по следующим категориям.

* Служба CloudSimple
* Соединение
* Сеть
* Безопасность
* Среда выполнения приложений
* Хранилище
* VMware
* Интеграция Azure
  
## <a name="cloudsimple-service"></a>Служба CloudSimple

**Что такое решение VMware, CloudSimple?**

**Решение VMware, CloudSimple** преобразует и расширяет рабочих нагрузок VMware в закрытой облака в Azure за несколько минут. Мы берем подготовки, управления инфраструктурой и оркестрации рабочих нагрузок между локальной сетью и Azure. Поскольку приложения выполняются точно же локально и в Azure, вы воспользоваться преимуществами эластичности и службы в облаке без необходимости изменения архитектуры приложений сложности. CloudSimple снижает совокупную стоимость владения с помощью модели потребления облака, предоставляемой по запросу подготовки, оплата как роста и оптимизация емкости.  См. в разделе [что такое решение VMware в Azure по CloudSimple](cloudsimple-vmware-solutions-overview.md) для функции, преимущества и сценарии.

**Что такое CloudSimple частного облака?**

Подготовьте облака закрытой состоящий из высокопроизводительные вычисления, хранение и сетевое взаимодействие среде, развернутой в инфраструктуре Microsoft Azure (место оборудования и центра обработки данных) в расположениях Azure.  Частное облако предоставляет собственной платформы VMware «как услуга». В терминах VMware каждого частного облака содержит только один экземпляр сервера vCenter. VCenter Server управляет несколько узлов ESXi, содержащихся в один или несколько кластеров vSphere, а также соответствующего хранилища виртуальной сети хранения данных (vSAN). Служба CloudSimple может содержать несколько частных облаков в вашей подписке Azure.  Дополнительные сведения о частных облаках, см. в разделе [обзор частного облака](cloudsimple-private-cloud.md).

**Где доступна служба CloudSimple?**

CloudSimple доступна в восточной части США и Западная часть США.

**Как включить подписку CloudSimple?**

Вы можно обратиться к своему представителю корпорации Майкрософт в [ azurevmwaresales@microsoft.com ](mailto:azurevmwaresales@microsoft.com) Чтобы включить подписку для службы CloudSimple. Отправьте идентификатор вашей подписки по электронной почте, для которого требуется включить службу CloudSimple.  

**Как получить доступ к порталу CloudSimple?**

Вам доступ к порталу CloudSimple с портала Azure.  См. в разделе [доступ к VMware решения с портала CloudSimple с портала Azure](https://docs.azure.cloudsimple.com/access-cloudsimple-portal) приводятся сведения о доступе к CloudSimple портала. 

**Как увеличить мощность для частного облака?**

Вы приобрели узлов на портале Azure и разверните узел вашего частного облака на портале CloudSimple.  Вы можете развернуть вашего частного облака, добавив дополнительные узлы в существующий кластер vSphere, или путем создания нового кластера vSphere.  См. в разделе [разверните CloudSimple частного облака](https://docs.azure.cloudsimple.com/expand-private-cloud) статьи для процедуры.

**Что произойдет с моей частного облака, во время обслуживания?**

CloudSimple предоставляет периодические уведомления, предшествующего планового обслуживания.  Обслуживание проводится некритическими так, чтобы обеспечить доступность частного облака.  Обслуживание может быть следующих типов:

1. **CloudSimple инфраструктуры:**  Инфраструктура CloudSimple предназначен для обеспечения высокой доступности.  Во время обслуживания подключение и доступность вашего частного облака за счет обновление избыточные компоненты во время без какого-либо влияния.  Вы получите доступ к vCenter частного облака, все виртуальные машины, подключение к Интернету с частным облаком и подключения к локальным или Azure.
2. **Портал CloudSimple:** Во время обслуживания некоторые функции на портале CloudSimple не может быть недоступен или отключен.  Уведомление об обслуживании будет включать сведения о что можно сделать на портале.

## <a name="connectivity"></a>Соединение

**Каковы Мои возможности подключения к сети региона CloudSimple?**

CloudSimple предоставляет три различные параметры подключения для подключения к сети CloudSimple регион.  Все три могут использоваться совместно.

1. ExpressRoute с подключением между локальным центром обработки данных к сети региона CloudSimple - частное подключение с безопасности высокоскоростной высокоскоростной мост локального канала ExpressRoute с каналом CloudSimple ExpressRoute с помощью глобальным доступом. См. в разделе [соединиться CloudSimple, с помощью ExpressRoute между локальной](https://docs.azure.cloudsimple.com/on-premises-connection) статье для настройки соединения.
2. ExpressRoute с подключением между вашей виртуальной сети Azure к сети региона CloudSimple - частное подключение с безопасности высокоскоростной низкой задержкой мост виртуальной сети в Azure с каналом CloudSimple ExpressRoute с использованием шлюзов виртуальной сети.  См. в разделе [подключить среду CloudSimple частного облака к виртуальной сети Azure с помощью ExpressRoute](https://docs.azure.cloudsimple.com/azure-expressroute-connection) статье для настройки соединения.
3. VPN-подключение сайт-сайт из локального центра обработки данных к сети региона CloudSimple - Secure виртуальной частной сети от локальных VPN-устройства для вашего региона CloudSimple частного облака.  См. в разделе [Настройка VPN-подключение между вашей локальной сети и частного облака CloudSimple] статьи по настройке VPN-подключение.

**Как подключаться для частного облака?**

На портале CloudSimple можно просмотреть сведения частного облака. Чтобы подключиться к vCenter, соответствующие частным облаком, убедитесь, что сетевое подключение является установленным с помощью сеть-сеть "," точка-сеть или ExpressRoute. Затем, чтобы открыть портал CloudSimple портале Azure и щелкните *запуска клиента vSphere* на домашней странице или на странице сведений о частном облаке.

**Что такое преимущество канала ExpressRoute?**

Канал Azure ExpressRoute обеспечивает безопасное подключение высокоскоростной низкой задержкой.  CloudSimple предоставляет выделенного канала ExpressRoute на регион на каждого клиента.  С помощью этого канала, можно установить безопасное подключение от локальной или подписке Azure.

**Каковы затраты сети для подключения с CloudSimple. Плата за исходящий трафик с CloudSimple в Azure? В регионах?**

Не взимается плата за любой исходящий трафик.  Azure стандартные ставки на любой исходящий трафик из виртуальной сети или локальному каналу ExpressRoute.

## <a name="networking"></a>Сеть

**Какие сетевые возможности, доступные для моей частного облака?**

Можно подготовить виртуальные ЛС (и их подсети), брандмауэра таблиц и назначить общедоступные IP-адреса и сопоставляются с виртуальной машины, выполняющиеся у вашего частного облака.  Дополнительные сведения см. в разделе [Общие сведения о виртуальных локальных сетей, подсетей](cloudsimple-vlans-subnets.md), [обзоре брандмауэра таблиц](cloudsimple-firewall-tables.md), и [Обзор адрес общедоступного IP-адреса](cloudsimple-public-ip-address.md) статей.

**Как настроить разные подсети для моих приложений в моей частного облака?**

Можно создать виртуальные локальные сети на вашего частного облака с портала CloudSimple.  После создания виртуальной локальной сети, можно создать группу портов, распределенных на vCenter частного облака, используя виртуальную локальную сеть и создать виртуальные машины, подключенные к группе распределенных порт.  Можно включить таблицу брандмауэра для виртуальной локальной сети и подсети и определить правила брандмауэра для защиты сетевого трафика.

**Какие параметры брандмауэра доступны для моей Частные облака?**

Можно настроить правила для трафика Север Юг и Восток запад.  Правила определены в таблице брандмауэра.  В таблице брандмауэра можно прикреплять к создания VLAN на частным облаком.  См. в разделе [Настройка брандмауэра таблиц и правил для частных облаков](https://docs.azure.cloudsimple.com/firewall) статьи для процедуры настройки.

**Можно ли назначить общедоступный IP-адресов для виртуальных машин в моей среде частного облака?**

На портале CloudSimple можно легко выделить новый общедоступный IP-адрес и связать его с помощью частного IP-адрес виртуального компьютера или устройства.  Кроме того, можно также создать новые правила брандмауэра или применить существующего правила брандмауэра, чтобы разрешить трафик из определенных портов и/или определенного набора IP-адресов на портале. См. в разделе [выделить общедоступные IP-адреса для частного облака среды](https://docs.azure.cloudsimple.com/public-ips) для процедуры настройки.

## <a name="security"></a>Безопасность

**Каковы параметры безопасности на CloudSimple?**

CloudSimple частное облако предоставляет следующие средства безопасности для защиты вашей среде частного облака:

1. **Данные, шифрование неактивных данных**: Можно шифровать данные при хранении, находящихся на хранилище сети хранения данных в частным облаком. Виртуальная сеть хранения данных поддерживает сервер управления ключами, который можно развернуть в Azure виртуальной сети или в локальной среде.  См. в разделе [настройки виртуальной сети хранения данных шифрования CloudSimple частным облаком](https://docs.azure.cloudsimple.com/vsan-encryption) для получения дополнительных сведений.
2. **Сетевая безопасность**: Управление потоком сетевого трафика из нее вашего частного облака из Интернета, в локальной, так и в пределах подсетей частного облака с помощью правил брандмауэра.
3. **Безопасность, частное подключение**: Защитите частное подключение между вашей локальной сетью и вашей подписке Azure.

## <a name="compute"></a>Среда выполнения приложений

**Какого рода узлы доступны?**

CloudSimple предлагает два типа узлов:

* **Узел CS28:** CPU:2 x 2,2 ГГц, 28 общее количество ядер, 48 HT.  ОЗУ. 256 ГБ.  Хранилище. Кэш NVMe ГБ 1600, 5760 ГБ данных (флэш-памяти). Сеть. сетевой Адаптер 2x25Gbe
* **Узел CS36:** ЦП 2 x 2,3 ГГц, общее 36 ядер, 72 HT.  ОЗУ. 512 ГБ.  Хранилище. Данные 11,520 ГБ кэша NVMe 3200 ГБ (флэш-памяти).  Сеть. сетевой Адаптер 2x25Gbe

**Способ обработки сбоев оборудования?**

Всю инфраструктуру CloudSimple постоянно отслеживается CloudSimple платформы и нашими командами разработчиков операций службы.  При обнаружении сбоя оборудования, добавляется новый узел вашего частного облака, а неисправного узла удаляется обеспечения высокого уровня доступности частного облака.

## <a name="storage"></a>Хранилище

**Какой тип хранилища поддерживается в частное облако?**

Предлагает CloudSimple **хранения vSAN VMware флэш-памяти** с каждого частного облака.  Каждый vSphere создается с свой собственный хранилище данных vSAN.  См. в разделе [компоненты VMware частного облака — виртуальной сети хранения данных хранилище](https://docs.azure.cloudsimple.com/vmware-components/#vsan-storage) Дополнительные сведения.

**Такое шифрование данных поддерживается?**
Да.  Можно настроить в виртуальной сети хранения данных хранилище вашего частного облака для использования себя сервер управления ключами (KMS), который является развернут локально или в Azure для шифрования данных, хранящихся в виртуальной сети хранения данных

**Как обрабатываются неисправных дисков?**

Мониторинг CloudSimple постоянно отслеживает все аппаратные компоненты частного облака.  При обнаружении любого сбоя диска или любой диск определяется как сбоя (на основании эвристики), новый узел автоматически добавляется в частное облако.  Узел с сбоя или недоступный диск удаляется из частного облака.

## <a name="vmware"></a>VMware

**Выполнение крупномасштабных отправки или переноса приложений и данных из локальной?**

CloudSimple предоставляет собственное решение VMware vSphere.  Можно использовать любое средство, используемое для массового переноса данных с CloudSimple частного облака.  Ниже приведены некоторые из доступных вариантов.

1. HCX VMware для массового переноса данных.
2. "Холодных" миграции данных с использованием Storage vMotion на платформу для CloudSimple.

**Можно установить средства VMware?**

CloudSimple предоставляет собственное решение VMware vSphere.  Можно использовать любое средство, используемое для управления vSphere локальная среда на CloudSimple.  CloudSimple поддерживает модель Bring-Your-собственной лицензии (BYOL) для установки средств VMware.

**Как управляются обновления?**

CloudSimple управляет и обновляет все компоненты инфраструктуры частного облака в виде простого некритическими.  Как только он уточняется CloudSimple обновления или исправления, выпущенные VMware или инфраструктуры поставщиков будет запланировано для обновления.

CloudSimple не выполняет модернизации или обновлении приложений установлены на частного облака.

## <a name="azure-integration"></a>Интеграция Azure

**Какие службы Azure поддерживаются?**

CloudSimple предоставляет подключение Azure ExpressRoute к вашей подписке в Azure.  Все службы, работающие в подписке доступно сетевое подключение к частным облаком и могут подключаться к частным облаком.  Примеры:

1. **Azure Active Directory** источник удостоверений CloudSimple vCenter
2. **Служба хранилища Azure** для хранения резервных копий, изображения и другие данные из вашего частного облака
3. **Гибридные приложения** -созданием архитектуры приложения, охватывающий общедоступных и частных облаков.  Например можно создать веб-серверы в Azure, доступ к БД и серверов приложений на CloudSimple частного облака.
4. **Azure monitor** и **центр безопасности Azure** -рабочая нагрузка, выполняемая в VMware можно использовать их для ведения журнала, показатели производительности и управление безопасностью.

**Как сопоставить клиентов VMware в Azure?**

CloudSimple предоставляет уникальные возможности управления виртуальными машинами VMware, о частном облаке с портала Azure.  Пул ресурсов vCenter (настраивается с ограничениями требуемого ресурса) могут сопоставляться с подпиской глобальным администратором.  

**Какие преимущества лицензирования получить с помощью Azure?**

С помощью CloudSimple можно воспользоваться преимуществами гибридного использования Azure и сэкономьте до 90% на лицензии, сохраняя инвестиции в лицензии Майкрософт, снижению совокупной стоимости владения, по сравнению с других облаках. Кроме того получить расширенные обновления для системы безопасности для Windows Server 2008 и Microsoft SQL Server 2008.  Усложнять затрат с использованием собственной лицензии (BYOL) в облако для стандартных приложений, таких как Veeam, Zerto и другие.

# <a name="frequently-asked-questions-about-vmware-solution-by-avs"></a>Часто задаваемые вопросы о решении VMware с помощью AVS

## <a name="avs-service"></a>Служба AVS

**Что такое решения Azure VMware (AVS)?**

Решения Azure VMware (AVS) преобразуют и расширяют рабочие нагрузки VMware в частные, выделенные облака в Azure за считанные минуты. AVS осуществляет подготовку, управление инфраструктурой и организацию рабочих нагрузок между локальной средой и Azure. Так как ваши приложения работают в локальной среде и в Azure, вы обладаете преимуществами эластичности и служб облака, не усложняя создание архитектуры приложений. AVS снижает общую стоимость владения с помощью модели использования облака, которая обеспечивает подготовку по требованию, оплату по мере роста и оптимизацию емкости. Дополнительные сведения о функциях, преимуществах и сценариях см. [в статье что такое решение VMware в Azure с помощью AVS](cloudsimple-vmware-solutions-overview.md) .

**Что такое частное облако AVS?**

Частное облако AVS — это закрытое выделенное облако, которое состоит из высокопроизводительных вычислений, хранилища и сетевой среды, развернутой на Microsoft Azureной инфраструктуре (в пространстве оборудования и в центре обработки данных) в расположениях Azure. Частное облако AVS предоставляет собственную платформу VMware как услугу. В терминах VMware Каждое частное облако AVS содержит ровно один экземпляр vCenter Server. VCenter Server управляет несколькими узлами ESXi, содержащимися в одном или нескольких кластерах vSphere, вместе с соответствующим хранилищем виртуальной сети SAN (vSAN). Служба AVS может содержать несколько частных облаков AVS в подписке Azure. Дополнительные сведения см. в статье [Общие сведения о частном облаке AVS](cloudsimple-private-cloud.md).

**Где доступна служба AVS?**

В ближайшее время для AVS доступны регионы Восточной части США, Западная часть США и Западная Европа с дополнительными регионами.

**Разделы справки включить мою подписку для AVS?**

Вы можете обратиться к учетная запись Майкрософт представителя по адресу [azurevmwaresales@microsoft.com](mailto:azurevmwaresales@microsoft.com) , чтобы включить в подписку службу AVS. Укажите идентификатор подписки в сообщении электронной почты, для которого требуется включить службу AVS. 

**Разделы справки доступ к порталу AVS?**

Доступ к порталу AVS можно получить из портал Azure. Дополнительные сведения см. в статье [доступ к порталу решений VMware (AVS) из портал Azure](access-cloudsimple-portal.md).

**Разделы справки увеличить емкость для частного облака AVS?**

Чтобы увеличить емкость, приобретите дополнительные узлы из портал Azure а затем с помощью узлов разверните частное облако AVS на портале AVS. Вы можете добавить дополнительные узлы в существующий кластер vSphere или добавить их в новый кластер vSphere. Дополнительные сведения см. в разделе [расширение частного облака AVS](expand-private-cloud.md).

**Что происходит с моим частным облаком AVS во время обслуживания?**

AVS предоставляет уведомление через несколько дней до интервала запланированного обслуживания. Обслуживание выполняется без нарушения работы, чтобы обеспечить доступность частного облака AVS. Обслуживание может иметь следующие типы:

* **Инфраструктура AVS**. Инфраструктура AVS разработана с высокой доступностью. Во время этого типа интервала обслуживания избыточные компоненты обновляются по одному за раз, чтобы избежать перерыва в работе служб. Вы сохраняете доступ к службе Virtual Machine Cloud vCenter, всем виртуальным машинам, Интернет-подключению из частного облака AVS и подключение к локальной среде или Azure.
* **Портал AVS**. В течение этого типа интервала обслуживания некоторые функции на портале AVS могут быть отключены или недоступны. Уведомление до интервала обслуживания включает сведения об ограничениях функций во время обслуживания.

## <a name="connectivity"></a>Подключение

**Какие варианты подключения относятся к сети регионов AVS?**

AVS предоставляет следующие варианты подключения для подключения к сети регионов AVS. В одно и то же время можно использовать несколько параметров.

* **Подключение ExpressRoute из локального центра обработки данных к сети регионов AVS**. Это высокоскоростное безопасное частное подключение с низкой задержкой, которое использует Global Reach для моста локального канала ExpressRoute в цепь ExpressRoute. Инструкции по настройке подключения см. в статье [подключение из локальной среды к AVS с помощью ExpressRoute](on-premises-connection.md).
* **Подключение ExpressRoute из виртуальной сети Azure к сети регионов AVS**. Это высокоскоростное безопасное частное подключение с низкой задержкой, которое использует шлюзы виртуальной сети для моста виртуальной сети в Azure до канала ExpressRoute для AVS. Инструкции по настройке подключения см. в статье [Подключение среды частного облака AVS к виртуальной сети Azure с помощью ExpressRoute](azure-expressroute-connection.md).
* **VPN-подключение типа "сеть — сеть" из локального центра обработки данных к сети "регион AVS**". Это безопасная виртуальная частная сеть из локального VPN-устройства в регион частного облака AVS. Дополнительные сведения см. [в разделе Настройка VPN-шлюзов в сети AVS](vpn-gateway.md).

**Разделы справки подключиться к частному облаку AVS?**

Вы можете просмотреть сведения о частном облаке AVS на портале AVS. Чтобы подключиться к vCenter, который соответствует частному облаку AVS, сначала убедитесь, что сетевое подключение установлено с помощью VPN типа "сеть — сеть", VPN-подключение типа "точка — сеть" или ExpressRoute. Затем запустите портал AVS из портал Azure и щелкните **запустить клиент vSphere** на домашней странице или на странице сведений о частном облаке AVS.

**Каковы преимущества каналов ExpressRoute?**

Канал ExpressRoute Azure — это высокоскоростное безопасное подключение с низкой задержкой. AVS предоставляет выделенный канал ExpressRoute для каждого клиента. С помощью этого канала можно установить безопасное подключение из локальной среды или подписки Azure.

**Каковы расходы на сеть для подключения к AVS? Взимается ли плата за исходящий трафик между AVS и Azure или между регионами?**

Для исходящего трафика не взимается плата за AVS. Тарифы Azure уровня "Стандартный" применяются ко всем исходящим трафикам из виртуальной сети или из локального канала ExpressRoute.

## <a name="networking"></a>Работа в сети

**Какие сетевые возможности доступны для частного облака AVS?**

Вы можете подготавливать виртуальные ЛС (и их подсети) и таблицы брандмауэров, а также назначать общедоступные IP-адреса, которые сопоставляются виртуальной машине, работающей в частном облаке AVS. Дополнительные сведения о сетевых функциях см. в разделе [Общие сведения о виртуальных ЛС и подсетях](cloudsimple-vlans-subnets.md), обзор [таблиц брандмауэра](cloudsimple-firewall-tables.md)и [Общие сведения о общедоступном IP-адресе](cloudsimple-public-ip-address.md).

**Разделы справки настроить разные подсети для приложений в частном облаке AVS?**

Виртуальные ЛС создаются в частном облаке AVS на портале AVS. После создания виртуальной ЛС можно создать группу распределенных портов в частном облаке "AVS" с помощью виртуальной ЛС и создать виртуальные машины, подключенные к группе распределенных портов. Вы можете включить таблицы брандмауэра для VLAN или подсети и определить правила брандмауэра для защиты сетевого трафика.

**Какие параметры брандмауэра доступны для частных облаков AVS?**

Можно настроить правила для трафика «Север-Юг» и «Восток-Западная». Правила определяются в таблице брандмауэра. Таблицу брандмауэра можно подключить к виртуальным ЛС в частном облаке AVS. Дополнительные сведения см. в разделе [Настройка таблиц брандмауэра и правил для частных облаков AVS](firewall.md).

**Можно ли назначить общедоступные IP-адреса виртуальным машинам в среде частного облака AVS?**

На портале AVS можно выделить новый общедоступный IP-адрес и связать его с частным IP-адресом виртуальной машины или устройства. Можно также создать новые правила брандмауэра или применить существующие правила брандмауэра, чтобы разрешить трафик с конкретных портов и IP-адресов на портале. Дополнительные сведения см. в разделе [выделение общедоступных IP-адресов для среды частного облака AVS](public-ips.md).

## <a name="security"></a>Безопасность

**Что такое мои параметры безопасности в AVS?**

AVS предоставляет следующие функции безопасности для защиты среды частного облака AVS:

* **Шифрование неактивных данных**. Вы можете шифровать неактивных данных, хранящихся в хранилище vSAN в частном облаке AVS. vSAN поддерживает внешние серверы управления ключами, которые можно развернуть в виртуальной сети Azure или в локальной среде. Дополнительные сведения см. в статье [Настройка шифрования vSAN для частного облака AVS](vsan-encryption.md).
* **Сетевая безопасность**. Управление сетевым трафиком с помощью правил брандмауэра, применяемых между частным облаком AVS и Интернетом, частным облаком AVS и локальной средой, а также в подсетях частного облака AVS.
* **Безопасное, частное подключение**. Между локальной сетью и подпиской Azure устанавливается безопасное, частное подключение.

## <a name="compute"></a>Среда выполнения приложений

**Какие типы узлов доступны?**

AVS предлагает следующие типы узлов:

* **CS28 узел:** ЦП: 2x 2,2 ГГц, всего 28 ядер, 48 НТ.  ОЗУ: 256 ГБ.  Хранилище: 1600 ГБ кэша NVMe, 5760 ГБ данных (все Flash). Сеть: 4x25Gbe NIC
* **CS36 узел:** ЦП 2x 2,3 ГГц, всего 36 ядер, 72 HT.  ОЗУ: 512 ГБ.  Хранилище: 3200 ГБ кэша NVMe 11520 ГБ данных (все Flash).  Сеть: 4x25Gbe NIC
* **CS36m узел:** ЦП 2x 2,3 ГГц, всего 36 ядер, 72 HT.  ОЗУ: 576 ГБ.  Хранилище: 3200 ГБ кэша NVMe 13360 ГБ данных (все Flash).  Сеть: 4x25Gbe NIC

**Как обрабатываются сбои оборудования?**

Вся инфраструктура AVS постоянно отслеживается платформой AVS и группами операций службы. При обнаружении сбоя оборудования в частное облако AVS добавляется новый узел, и неисправный узел удаляется.

## <a name="storage"></a>Хранилище

**Какой тип хранилища поддерживается в частном облаке AVS?**

AVS предлагает все — флэш-накопитель VMware vSAN в каждом частном облаке AVS. Каждый vSphere создается с собственным хранилищем данных vSAN. Дополнительные сведения см. в разделе [компоненты частного облака VMware, vSAN хранилище](vmware-components.md#vsan-storage).

**Поддерживается ли шифрование данных?**
Да. Вы можете настроить хранилище vSAN в частном облаке AVS, чтобы использовать сервер управления ключами (KMS), развернутый локально или в Azure для шифрования данных, хранящихся в vSAN.

**Как обрабатываются неисправные диски?**

AVS постоянно отслеживает все аппаратные компоненты частного облака AVS. При обнаружении сбоя диска или обнаружении диска как сбой (на основе эвристики) в частное облако AVS автоматически добавляется новый узел. Узел с неисправным или неисправным диском удаляется из частного облака AVS.

## <a name="vmware"></a>VMware

**Разделы справки выполнить крупномасштабную передачу или перенос приложений и данных из локальной среды?**

AVS предоставляет собственное VMware vSphere решение. Для частного облака AVS можно использовать все средства VMware, предназначенные для переноса данных. Доступные параметры:

* VMware ХККС для выполнения групповой миграции данных.
* Холодное перемещение данных с помощью хранилища данных vMotion из локальной среды в AVS.

**Можно ли установить любые инструменты VMware?**

AVS предоставляет собственное VMware vSphere решение. Все инструменты VMware, используемые для управления локальной средой vSphere, можно использовать в AVS. AVS поддерживает модель с собственной лицензией (BYOL) для установки средств VMware.

**Как осуществляется управление обновлениями и обновлением?**

AVS управляет и обновляет все компоненты инфраструктуры частного облака AVS без нарушения работы. Все обновления и исправления безопасности, выпущенные VMware или поставщиками инфраструктуры, планируются на обновление сразу после того, как они будут дополнены AVS.

AVS не выполняет обновления или обновления приложений, установленных в частном облаке AVS.

## <a name="azure-integration"></a>Интеграция с Azure

**Какие службы Azure поддерживаются?**

AVS предоставляет подключение Azure ExpressRoute к вашей подписке в Azure. Все службы, работающие в вашей подписке, могут подключаться к частному облаку AVS. Примеры.

* **Azure Active Directory** в качестве источника удостоверений для системы AVS vCenter.
* Служба **хранилища Azure** для хранения резервных копий, образов и других данных из частного облака AVS.
* **Гибридные приложения** с архитектурой приложений, охватывающей общедоступные и закрытые облака AVS. Например, вы можете создавать веб серверы в Azure, которые обращаются к приложениям и серверам баз данных в частном облаке AVS.
* **Azure Monitor** и **Центр безопасности Azure** для рабочих нагрузок, использующих журналы поддержки VMware, метрики производительности и управление безопасностью.

**Разделы справки сопоставлять мои клиенты VMware с Azure?**

AVS обеспечивает уникальную возможность управления виртуальными машинами VMware в частном облаке AVS из портал Azure. Пул ресурсов vCenter, настроенный с требуемыми ограничениями ресурсов, может быть сопоставлен с вашей подпиской глобальным администратором. 

**Какие преимущества лицензирования можно получить с помощью Azure?**

С AVS вы можете воспользоваться преимуществами гибридного использования Azure и сэкономить до 90% лицензий. Это преимущество позволяет сохранить ваши инвестиции в лицензии Майкрософт и снизить совокупную стоимость владения относительно других облачных решений. Вы также получаете расширенные обновления для системы безопасности для Windows Server 2008 и Microsoft SQL Server 2008. Модель с собственной лицензией (BYOL) позволяет снизить затраты для таких распространенных приложений, как Veeam и Zerto.

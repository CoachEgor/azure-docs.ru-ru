# <a name="frequently-asked-questions-about-vmware-solution-by-cloudsimple"></a>Часто задаваемые вопросы о vMware Solution от CloudSimple

## <a name="cloudsimple-service"></a>Сервис CloudSimple

**Что такое решение Azure VMware от CloudSimple?**

Решение Azure VMware от CloudSimple преобразует и расширяет рабочие нагрузки VMware на частные, выделенные облака на Azure за считанные минуты. CloudSimple занимается подготовкой, управлением инфраструктурой и организацией рабочих нагрузок между наймом и Azure. Поскольку ваши приложения работают точно так же на месте и в Azure, вы получаете выгоду от эластичности и услуг облака без сложности переквалификации приложений. CloudSimple снижает общую стоимость владения с помощью модели облачного потребления, которая обеспечивает подготовку по требованию, оплату по мере роста и оптимизацию емкости.  Ознакомьтесь с преимуществами, преимуществами и сценариями [решения VMware на Azure от CloudSimple.](cloudsimple-vmware-solutions-overview.md)

**Что такое облачное частное облако?**

Облачное частное облако — это частное специализированное облако, состоящее из высокопроизводительной вычислительной, системной и сетевой среды, развернутой в инфраструктуре Microsoft Azure (аппаратное обеспечение и пространство центра обработки данных) в местах Расположения Azure.  Частное облако предоставляет родную платформу VMware как услугу. С точки зрения VMware каждое частное облако содержит ровно один экземпляр сервера vCenter Server. VCenter Server управляет несколькими узлами ESXi, содержащимися в одном или нескольких кластерах vSphere, наряду с соответствующим виртуальным хранилищем SAN (vSAN). Служба CloudSimple может содержать несколько частных облаков в подписке Azure.  Для получения более подробной информации [см.](cloudsimple-private-cloud.md)

**Где доступен сервис CloudSimple?**

CloudSimple доступен в регионах Восточной США, Западной Ивсеи и Западной Европы, в скором времени будут выделены дополнительные регионы.

**Как включить подписку на CloudSimple?**

Вы можете связаться с [azurevmwaresales@microsoft.com](mailto:azurevmwaresales@microsoft.com) представителем учетной записи Майкрософт, чтобы включить подписку на сервис CloudSimple. Предоставьте свой идентификатор подписки в электронной почте, для которого вы хотите, чтобы сервис CloudSimple был включен.  

**Как получить доступ к порталу CloudSimple?**

Вы получаете доступ к порталу CloudSimple с портала Azure.  Подробнее о решении [VMware с сайта CloudSimple можно найти на портале Azure.](access-cloudsimple-portal.md)

**Как увеличить емкость частного облака?**

Чтобы увеличить емкость, приобретите дополнительные узлы с портала Azure, а затем используйте узлы для расширения частного облака с портала CloudSimple.  Можно добавить дополнительные узлы в существующий кластер vSphere или добавить их в новый кластер vSphere.  Для получения подробной информации [см.](expand-private-cloud.md)

**Что происходит с моим приватное облако во время обслуживания?**

CloudSimple предоставляет уведомления за несколько дней до запланированного интервала обслуживания.  Обслуживание осуществляется ненарушая образом, чтобы обеспечить доступность вашего частного облака.  Обслуживание может быть следующих типов:

* **Инфраструктура CloudSimple**.  Инфраструктура CloudSimple предназначена для высокой доступности.  Во время этого типа интервала обслуживания избыточные компоненты обновляются по одному, чтобы избежать прерывания обслуживания. Вы поддерживаете доступ к private Cloud vCenter, всем виртуальным машинам, интернет-соединению из вашего частного облака и подключению к найму или Azure.
* **CloudSimple портал**. Во время этого интервала обслуживания некоторые функции на портале CloudSimple могут быть отключены или недоступны.  Уведомление до интервала обслуживания содержит сведения об ограничениях функций во время технического обслуживания.

## <a name="connectivity"></a>Соединение

**Каковы мои варианты подключения к региональной сети CloudSimple?**

CloudSimple предоставляет следующие варианты подключения для подключения к вашей региональной сети CloudSimple. Несколько вариантов могут быть использованы в то же время.

* **Подключение ExpressRoute от вашего центра обработки данных к сети cloudSimple региона.** Это высокоскоростное, низкое затягивание, безопасное частное соединение, которое использует Global Reach для подключения вашей трассы ExpressRoute к схеме CloudSimple ExpressRoute. Для получения инструкций по настройке соединения [см.](on-premises-connection.md)
* **Подключение ExpressRoute от виртуальной сети Azure к вашей региональной сети CloudSimple.** Это высокоскоростное, низкое замещение, безопасное частное соединение, которое использует виртуальные сетевые шлюзы для подключения вашей виртуальной сети на Azure к вашей схеме CloudSimple ExpressRoute. Для получения инструкций по настройке соединения [см.](azure-expressroute-connection.md)
* **VPN-соединение от сайта к сайту от вашего центра обработки данных к сети CloudSimple.** Это безопасная виртуальная частная сеть от вашего собственного VPN-устройства до региона CloudSimple Private Cloud.  Для получения подробной информации смотрите [Настройка VPN шлюзов в сети CloudSimple.](vpn-gateway.md)

**Как подключиться к приватному облаку?**

Вы можете просматривать сведения о вашем частном облаке на портале CloudSimple. Чтобы подключиться к vCenter, который соответствует вашему частному облаку, сначала убедитесь, что сетевое соединение установлено с помощью VPN сайта к сайту, VPN от point-to-Site или ExpressRoute. Затем запустите портал CloudSimple с портала Azure и нажмите кнопку **«Клиент запуска vSphere»** на главной странице или на странице данных частного облака.

**В чем преимущество схем ExpressRoute?**

Схема Azure ExpressRoute представляет собой высокоскоростное, низкое замедляемые и безопасные соединения.  CloudSimple предоставляет специальную схему ExpressRoute в регионе на одного клиента.  Используя эту схему, можно установить безопасное соединение из предварительной подписки или подписки Azure.

**Сколько стоит сеть для подключения к CloudSimple?  Применяются ли какие-либо сборы между CloudSimple и Azure или в разных регионах?**

Плата CloudSimple за выход из сети не взимается.  Стандартные тарифы Azure применяются к любому трафику из вашей виртуальной сети или с вашей предварительной схемы ExpressRoute.

## <a name="networking"></a>Сети

**Какие сетевые функции доступны для моего частного облака?**

Вы можете предоставить VLANs (и их подсети) и таблицы брандмауэров, а также назначить общедоступные IP-адреса, которые карта виртуальная машина работает в частном облаке. Для получения подробной информации о функциях сетей, [см. Обзор VLAN и подсетей,](cloudsimple-vlans-subnets.md) [обзор таблиц Брандуолла](cloudsimple-firewall-tables.md)и [обзор общедоступных IP-адресов.](cloudsimple-public-ip-address.md)

**Как настроить различные подсети для приложений в приватном облаке?**

Вы создаете VLANs на вашем частном облаке с портала CloudSimple.  После создания VLAN можно создать распределенную группу портов в vCenter Private Cloud с помощью VLAN и создать виртуальные машины, подключенные к распределенной группе портов.  Можно включить таблицы брандмауэра для Подсети VLAN/subnet и определить правила брандмауэра для обеспечения сетевого трафика.

**Какие настройки брандмауэра доступны для моих частных облаков?**

Вы можете настроить правила для движения с севера на юг и с востока на запад.  Правила определяются в таблице брандмауэра.  Таблица брандмауэра может быть прикреплена к VLANs в вашем частном облаке.  Для получения подробной информации смотрите [таблицы брандмауэров и правила для частных облаков.](firewall.md)

**Могу ли я присвоить общедоступные IP-адреса в среде вне веможно сями в среде Private Cloud?**

На портале CloudSimple можно выделить новый общедоступный IP-адрес и связать его с частным IP-адресом виртуальной машины или устройства.  Можно также создать новые правила брандмауэра или применить существующие правила брандмауэра, чтобы разрешить трафик из определенных портов и IP-адресов на портале. Для получения подробной информации [см.](public-ips.md)

## <a name="security"></a>Безопасность

**Каковы мои параметры безопасности в CloudSimple?**

CloudSimple предоставляет следующие функции безопасности для обеспечения безопасности среды private Cloud:

* **Данные в состоянии шифрования.** Вы можете шифровать данные в состоянии покоя, проживающие на хранилище vSAN в вашем частном облаке. vSAN поддерживает внешние серверы управления ключами, которые могут быть развернуты в вашей электронной или предварительной среде Azure.  Для получения подробной информации [см.](vsan-encryption.md)
* **Сетевая безопасность**. Управление потоком сетевого трафика с помощью правил брандмауэра, которые применяются между вашим частным облаком и Интернетом, вашим частным облаком и предварительной средой или в подсетях вашего частного облака.
* **Безопасное, личное соединение**. Между вашей предварительной сетью и подпиской Azure устанавливается безопасное частное соединение.

## <a name="compute"></a>Службы вычислений

**Какие хосты доступны?**

CloudSimple предлагает следующие типы хостов:

* **Узлы CS28:** Процессор: 2x 2,2 ГГц, всего 28 ядер, 48 HT.  Оперативная память: 256 ГБ.  Хранение: 1600 ГБ кэша NVMe, данные 5760 ГБ (All-Flash). Сеть: 4x25Gbe NIC
* **Узлы CS36:** Процессор 2x 2,3 ГГц, всего 36 ядер, 72 HT.  Оперативная память: 512 ГБ.  Хранение: 3200 ГБ Кэш NVMe 11520 ГБ данных (All-Flash).  Сеть: 4x25Gbe NIC
* **Узло CS36m:** Процессор 2x 2,3 ГГц, всего 36 ядер, 72 HT.  Оперативная память: 576 ГБ.  Хранение: 3200 ГБ Кэш NVMe 13360 ГБ данных (All-Flash).  Сеть: 4x25Gbe NIC

**Как обрабатываются какие-либо аппаратные сбои?**

Вся инфраструктура CloudSimple постоянно контролируется платформой CloudSimple и нашими командами по работе с сервисом.  При обнаружении сбоя оборудования в приватное облако добавляется новый узла и удаляется неузлый узла.

## <a name="storage"></a>Хранилище

**Какой тип хранилища поддерживается в частном облаке?**

CloudSimple предлагает все вспышки VMware vSAN хранения с каждым частным облаком.  Каждый vSphere создается со своим собственным хранилищем данных vSAN.  Для получения подробной информации [см.](vmware-components.md#vsan-storage)

**Поддерживается ли шифрование данных?**
Да.  Вы можете настроить хранилище vSAN в личном облаке, чтобы использовать сервер управления ключами (KMS), который развертывается локально или в Azure для шифрования данных, хранящихся на vSAN.

**Как обрабатываются неудавные диски?**

CloudSimple постоянно отслеживает все аппаратные компоненты частного облака.  Если сбой диска обнаружен или диск идентифицируется как сбой (на основе эвристики), новый узлы автоматически добавляются в частное облако.  Узла с неисправным или неисправным диском удаляется из приватного облака.

## <a name="vmware"></a>VMware

**Как выполнять масштабную загрузку или миграцию приложений и данных из помещений?**

CloudSimple предлагает нативное решение VMware vSphere.  Все инструменты VMware для массовой миграции данных могут быть использованы с вашим частным облаком.  Доступные параметры: 

* VMware HCX для массовой миграции данных.
* Холодная миграция данных с помощью хранения vMotion из наемных помещений в CloudSimple.

**Могу ли я установить какие-либо инструменты VMware?**

CloudSimple предлагает нативное решение VMware vSphere.  Все инструменты VMware, используемые для управления вашей предварительной средой vSphere, могут быть использованы в CloudSimple.  CloudSimple поддерживает модель собственной лицензии (BYOL) для установки инструментов VMware.

**Как управляется обновление и обновление?**

CloudSimple управляет и обновляет все компоненты инфраструктуры вашего частного облака бесшовным образом.  Все обновления и патчи безопасности, выпущенные VMware или поставщиками инфраструктуры, планируется обновить, как только они будут квалифицированы CloudSimple.

CloudSimple не выполняет обновления или обновления приложений, установленных в частном облаке.

## <a name="azure-integration"></a>Интеграция с Azure

**Какие службы Azure поддерживаются?**

CloudSimple предоставляет подключение Azure ExpressRoute к подписке на Azure.  Все службы, работающие в подписке, могут подключиться к вашему частному облаку.  Примеры приведены ниже.

* **Активный каталог Azure** в качестве источника идентификации для вашего Облачного vCenter.
* **Хранилище Azure** для хранения резервных копий, изображений и других данных из вашего частного облака.
* **Гибридные приложения** с архитектурой приложений, которая охватывает общедоступные и частные облака.  Например, можно создать веб-серверы в Azure, которые будут получать доступ к серверам приложений и баз данных в частном облаке.
* **Мониторинг Azure** и **центр безопасности Azure** для рабочих нагрузок, работающих при регистрации поддержки VMware, метрики производительности и управлении безопасностью.

**Как сопоставить мои хонереты VMware в Azure?**

CloudSimple предоставляет уникальную возможность управлять VMware VMs на частном облаке с портала Azure.  Пул ресурсов vCenter, настроенный с желаемыми ограничениями ресурсов, может быть отображен к вашей подписке глобальным администратором.  

**Какие преимущества лицензирования я получаю с Azure?**

С помощью CloudSimple вы можете воспользоваться преимуществами преимущества гибридного использования Azure и сэкономить до 90% лицензий. Это преимущество сохраняет ваши инвестиции в лицензии Майкрософт и снижает ваш ТКО по сравнению с другими облачными решениями. Вы также получаете расширенные обновления безопасности для Windows Server 2008 и Microsoft S'L Server 2008.  Модель bring-your-own-license (BYOL) позволяет снизить затраты на обычные приложения, такие как Veeam и Зерто.

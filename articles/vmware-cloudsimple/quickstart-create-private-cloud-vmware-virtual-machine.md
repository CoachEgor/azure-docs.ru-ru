# <a name="create-vmware-virtual-machines-on-your-private-cloud"></a>Создание виртуальных машин VMware на вашем частном облаке

Чтобы создать виртуальные машины в частном облаке, начните с доступа к порталу CloudSimple с портала Azure.

## <a name="sign-in-to-the-azure-portal"></a>Вход на портал Azure

Войти на портал Azure по адресу [https://portal.azure.com](https://portal.azure.com).

## <a name="access-the-cloudsimple-portal"></a>Доступ к порталу CloudSimple

1. Выберите **все услуги**.
2. Поиск **облачных сервисов CloudSimple**.
3. Выберите сервис CloudSimple, на котором вы хотите создать частное облако.
4. Со страницы **«Обзор»** нажмите **«Перейдите на портал CloudSimple»,** чтобы открыть новую вкладку браузера для портала CloudSimple.  Если побуждение, вопийте со своим знаком Azure в учетных данных.  

    ![Запуск портала CloudSimple](media/launch-cloudsimple-portal.png)

## <a name="launch-vcenter-web-ui"></a>Запуск vCenter веб-ui

Теперь вы можете запустить vCenter для настройки виртуальных машин и политик.

Чтобы получить доступ к vCenter, начните с портала CloudSimple. На главной странице, под **общими задачами,** нажмите **Запуск vSphere Клиента**.  Выберите частное облако, а затем нажмите **Кнопка Запуска vSphere Клиента** в частном облаке.

   ![Запуск vSphere Клиент](media/launch-vcenter-from-cloudsimple-portal.png)

## <a name="upload-an-iso-or-vsphere-template"></a>Загрузить шаблон ISO или vSphere

  > [!WARNING]
  > Для загрузки ISO используйте клиент vSphere HTML5.  Использование клиента Flash может привести к ошибке.

1. Получите шаблон ISO или vSphere, который вы хотите загрузить в vCenter, чтобы создать VM и иметь его в локальной системе.
2. В vCenter щелкните значок **диска** и выберите **vsanDatastore**. Нажмите **файлы,** а затем нажмите **New Folder**.
    ![vCenter ISO](media/vciso00.png)

3. Создайте папку под названием 'ISOs и шаблоны'.

4. Перейдите к папке IIOs в IsOs и шаблонах и нажмите **Upload Files.** Следуйте инструкциям на экране для загрузки ISO.

## <a name="create-a-virtual-machine-in-vcenter"></a>Создание виртуальной машины в vCenter

1. В vCenter щелкните **значок Хостов и кластеров.**

2. Право нажмите **рабочей нагрузки** и выберите **новую виртуальную машину**.
    ![Создание виртуальной машины](media/vcvm01.png)

3. Выберите **Создать новую виртуальную машину** и нажмите **кнопку Next**.
    ![Создание виртуальной машины](media/vcvm02.png)

4. Назовите машину, выберите местоположение **Workload VM** и нажмите **далее.**
    ![Создание виртуальной машины](media/vcvm03.png)

5. Выберите ресурс **вычисления рабочей нагрузки** и нажмите **далее.**
    ![Создание виртуальной машины](media/vcvm04.png)

6. Выберите **vsanDatastore** и нажмите **далее**.
    ![Создание виртуальной машины](media/vcvm05.png)

7. Держите выбор совместимости esXi по умолчанию 6.5 и нажмите **далее.**
    ![Создание виртуальной машины](media/vcvm06.png)

8. Выберите гостевую ОС ISO для VM, которую вы создаете, и нажмите **Далее.**
    ![Создание виртуальной машины](media/vcvm07.png)

9. Выберите варианты жесткого диска и сети. Для нового диска CD/DVD выберите **файл Datastore ISO.**  Если вы хотите разрешить трафик с общественного IP-адреса на этот VM, выберите сеть как **vm-1**.
    ![Создание виртуальной машины](media/vcvm08.png)

10. Открывается окно выбора. Выберите файл, который вы ранее загрузили в папку IIOs и templates, и нажмите **OK.**
    ![Создание виртуальной машины](media/vcvm10.png)

11. Просмотрите настройки и **нажмите OK,** чтобы создать VM.
    ![Создание виртуальной машины](media/vcvm11.png)

VM теперь добавляется к вычислительным ресурсам рабочей нагрузки и готов к использованию. 
![Создание виртуальной машины](media/vcvm12.png)

Основная настройка завершена. Вы можете начать использовать приватное облако по аналогии с тем, как вы будете использовать свою инфраструктуру VM.

Следующие разделы содержат дополнительную информацию о настройке DNS и DHCP серверов для рабочих нагрузок Private Cloud и изменении конфигурации сетей по умолчанию.

## <a name="add-users-and-identity-sources-to-vcenter-optional"></a>Добавление пользователей и источников идентификации в vCenter (необязательно)

CloudSimple присваивает учетную запись `cloudowner@cloudsimple.local`пользователя vCenter по умолчанию с именем пользователя. Для начала работы не требуется дополнительная настройка учетной записи.  CloudSimple обычно присваивает администраторам привилегии, необходимые для выполнения обычных операций.  Навлаживайте активный каталог или AD Azure в качестве [дополнительного источника идентификации](set-vcenter-identity.md) в личном облаке.

## <a name="create-a-dns-and-dhcp-server-optional"></a>Создание DNS и DHCP-сервера (необязательно)

Приложения и рабочие нагрузки, работающие в среде Private Cloud, требуют разрешения имен и dhCP-сервисов для поиска и назначения IP-адресов. Для предоставления этих услуг требуется надлежащая инфраструктура DHCP и DNS. Вы можете настроить виртуальную машину в vCenter для предоставления этих услуг в среде Private Cloud.

Предварительные требования

* Распределенная группа портов с vLAN настроена

* Настройка маршрута на предприимские или интернет-dNS-серверы

* Виртуальный шаблон машины или ISO для создания виртуальной машины

## <a name="customize-networking-configuration-optional"></a>Настройка конфигурации сетей (необязательно)

Страницы Сети на портале CloudSimple позволяют указать конфигурацию для таблиц брандмауэра и общедоступных IP-адресов для VMs.

### <a name="allocate-public-ips"></a>Выделение публичных ИС

1. Перейдите на **сетевое > общественное IP** на портале CloudSimple.
2. Нажмите **Выделить общественный IP**.
3. Введите имя для идентификации записи IP-адреса.
4. Сохраняйте местоположение по умолчанию.
5. Используйте ползунок, чтобы при желании изменить тайм-аут для простоя.
6. Введите локальный IP-адрес, на который требуется присвоить общедоступный IP-адрес.
7. При желании введите связанное имя DNS.
8. Нажмите кнопку **Готово**.

    ![Общедоступный IP-адрес](media/quick-create-pc-public-ip.png)

Начинается задача выделения общедоступного IP-адреса. Вы можете проверить состояние задачи на странице **«Задачи активности > задачи».** Когда выделение завершено, новая запись отображается на странице публичных IPs.

VM, на который должен быть отображен этот IP-адрес, должен быть настроен с указанным выше локальным адресом. Процедура настройки IP-адреса характерна для операционной системы VM. Проконсультируйтесь с документацией для вашей операционной системы VM для правильной процедуры.

#### <a name="example"></a>Пример

Например, вот подробности для Ubuntu 16.04.

Добавьте статический метод в конфигурацию семейства inet address в файле /etc/network/interfaces. Измените значения адреса, netmask и шлюза. Для этого примера мы используем интерфейс эт0, внутренний IP-адрес 192.168.24.10, адрес шлюза 192.168.24.1 и netmask 255.255.255.0. Для вашей среды доступная информация о подсети предоставляется в приветственном письме.

```
sudo vi /etc/network/interfaces
```

```
auto eth0
Iface eth0 inet static
iface eth0 inet static
address 192.168.24.10
netmask 255.255.255.0
gateway 192.168.24.1
dns-nameservers 8.8.8.8
dns-domain acme.com
dns-search acme.com
```
Вручную отключить интерфейс.

```
sudo ifdown eth0
```
Вручную включить интерфейс снова.

```
sudo ifup eth0
```

По умолчанию, весь входящий трафик из Интернета **отказано**. Если вы хотите открыть любой другой порт, создайте [таблицу брандмауэра.](firewall.md)

После настройки внутреннего IP-адреса в виде статического IP-адреса убедитесь, что вы можете связаться с Интернетом из Нутри ВМ.

```
ping 8.8.8.8
```
Также убедитесь, что вы можете связаться с VM из Интернета с помощью публичного IP-адреса.

Убедитесь, что любые правила iptable на VM не блокируют входящий порт 80.
        
```
netstat -an | grep 80
```

Запустите сервер http, который слушает на порту 80.
       
```
python2.7 -m SimpleHTTPServer 80
```

или диспетчер конфигурации служб

```
python3 -m http.server 80
```
Запустите браузер на рабочем столе и направьте его на порт 80 для общедоступного IP-адреса для просмотра файлов на Вашем VM.

### <a name="default-cloudsimple-firewall-rules-for-public-ip"></a>Правила брандмауэра CloudSimple для общедоступного IP

* VPN трафик: Все трафик между (от / к) VPN и все сети рабочей нагрузки и сети управления допускается.
* Внутренний трафик частного облака: Разрешен весь трафик между (от/к) рабочими сетями и сетью управления (см. выше).
* Интернет-трафик:
  * Весь входящий трафик из Интернета не имеет рабочих сносов и сети управления.
  * Допускается весь исходящий трафик в Интернет из сетей рабочей нагрузки или сети управления.

Вы также можете изменить способ обеспечения безопасности трафика с помощью функции Правил брандмауэра. Для получения дополнительной информации [см.](firewall.md)

# <a name="cloudsimple-private-cloud-overview"></a>Обзор облачного частного облака

CloudSimple преобразует и расширяет рабочие нагрузки VMware до общедоступных облаков за считанные минуты. Используя сервис CloudSimple, вы можете развернуть VMware на голой металлической инфраструктуре Azure. Развертывание в местах развертывания Azure полностью интегрируется с остальным облаком Azure.

Решение CloudSimple обеспечивает полную непрерывность работы VMware. Это решение дает вам преимущества общедоступного облака:

* Эластичность
* Инновации
* Эффективность

С помощью CloudSimple вы получаете выгоду от модели облачного потребления, которая снижает общую стоимость владения. Он также предлагает подготовку по требованию, оплату по мере роста, и оптимизацию емкости.

CloudSimple полностью совместим с:

* Существующие инструменты
* Навыки
* Процессы

Эта совместимость позволяет группам управлять рабочими нагрузками в облаке Azure, не нарушая эти типы политик:

* Сеть
* Безопасность  
* Защита данных  
* Аудит

CloudSimple управляет инфраструктурой и всеми необходимыми сетевыми и управленческими службами. Сервис CloudSimple позволяет вашей команде сосредоточиться на:

* Бизнес-ценность
* Подготовка приложений
* Непрерывность бизнес-процессов
* Поддержка
* Принудительное применение политики

## <a name="private-cloud-environment-overview"></a>Обзор среды частного облака

Частное облако — это изолированный стек VMware, который поддерживает:

* Хосты ESXi
* vCenter
* vSAN
* Nsx

Частные облака управляются через портал CloudSimple. Они имеют свой собственный сервер vCenter в своей собственной области управления.

Стек работает на:

* Специальные узлы
* Изолированные голые металлические аппаратные узлы

Пользователи потребляют стек с помощью нативных инструментов VMware, в ключая:

* vCenter
* Менеджер NSX

Развертывание выделенных узлов можно в местах расположения Azure. Затем вы можете управлять ими с помощью Azure и CloudSimple. Частное облако состоит из одного или нескольких кластеров vSphere, и каждое кластер содержит от 3 до 16 узлов.

Вы можете создать частное облако, используя приобретенные узлы с оплатой по мере использования или зарезервированные выделенные узлы.

Вы можете подключить частное облако к своей предварительной среде и сети Azure, используя следующие соединения:

* Безопасность
* Частный VPN
* Azure ExpressRoute

Среда Private Cloud предназначена для устранения отдельных точек сбоя:

* Кластеры ESXi настроены с высокой доступностью vSphere и имеют размер по крайней мере одного запасного узла для устойчивости.
* vSAN обеспечивает избыточное первичное хранилище. vSan требует, чтобы по крайней мере три узла обеспечивали защиту от одного сбоя. Можно настроить vSAN, чтобы обеспечить более высокую устойчивость для больших кластеров.
* Вы можете настроить vCenter, PSC и NSX Manager VMs с политикой хранения RAID-10 для защиты от сбоя хранилища. vSphere HA защищает от сбоев узлов и сетей.

## <a name="scenarios-for-deploying-a-private-cloud"></a>Сценарии развертывания приватного облака

Ниже приведены примеры использования примеров для развертывания private Cloud.

### <a name="data-center-retirement-or-migration"></a>Выход на пенсию или миграция центра обработки данных

* Получите дополнительную емкость, когда вы достигнете пределов существующего центра обработки данных или обновите оборудование.
* Добавьте необходимую емкость в облако и устраните головную боль, при этом управление аппаратными обновлениями.
* Снижение риска и затрат облачных миграций по сравнению с трудоемкими преобразованиями или реархитектурой.
* Используйте знакомые инструменты и навыки VMware для ускорения миграции облаков. В облаке используйте службы Azure для модернизации приложений в вашем темпе.

### <a name="expand-on-demand"></a>Расширить спрос по требованию

* Расширяйтесь до облака для удовлетворения непредвиденных потребностей, таких как новые среды разработки или сезонные всплески емкости.
* Создавайте новые мощности по требованию и сохраняйте ее только до тех пор, пока это необходимо.
* Сократите объем предварительных инвестиций, ускорьте скорость подготовки и уменьшите сложность с помощью одной и той же архитектуры и политик и политик как в помещениях, так и в облаке.

### <a name="disaster-recovery-and-virtual-desktops-in-the-azure-cloud"></a>Аварийное восстановление и виртуальные рабочие столы в облаке Azure

* Установить удаленный доступ к данным, приложениям и рабочим столам в облаке Azure. С высокой пропускной способностью соединений, вы загружаете / загружать данные быстро, чтобы восстановить после инцидентов. Сети с низкой задержкой позволяют быстро отреагировать, которое пользователи ожидают от настольного приложения.

* Воспроизобьте все ваши политики и сети в облаке с помощью портала CloudSimple и знакомых инструментов VMware. Репликация снижает усилия и риск создания и управления реализациями DR и VDI.

### <a name="high-performance-applications-and-databases"></a>Высокопроизводительные приложения и базы данных

* Выполнить наиболее сложные рабочие нагрузки с гиперконвергентной архитектурой, предоставляемой CloudSimple.
* Выполнить Oracle, сервер Microsoft S'L, системы промежуточного программного обеспечения, а также высокопроизводительные базы данных без S'L.
* Испытайте облако как собственный центр обработки данных с высокоскоростными 25-Гбит/с сетевыми соединениями. Высокоскоростные соединения позволяют запускать гибридные приложения, которые охватывают на месте, VMware на Azure и частные рабочие нагрузки Azure, без ущерба для производительности.

### <a name="true-hybrid"></a>Настоящий гибрид

* Объединить DevOps через службы VMware и Azure.
* Оптимизируйте администрирование VMware для служб и решений Azure, которые могут быть применены во всех рабочих нагрузках.
* Доступ к общедоступным облачным службам без расширения центра обработки данных или переосвоить приложения.
* Центризуйте идентификаторы, политики управления доступом, журналирование и мониторинг приложений VMware в Azure.

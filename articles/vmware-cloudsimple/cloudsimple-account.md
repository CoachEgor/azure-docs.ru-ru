# <a name="account-management-overview"></a>Общие сведения об управлении учетной записью

При создании службы AVS она создает учетную запись в AVS. Учетная запись связана с подпиской Azure, в которой находится служба. Все пользователи с ролями **владельца** и **участника** в подписке имеют доступ к порталу AVS. Идентификатор подписки Azure и идентификатор клиента, связанные со службой AVS, находятся на [странице учетных записей](account.md).

## <a name="additional-alert-emails"></a>Дополнительные оповещения по электронной почте

Вы можете настроить идентификаторы электронной почты в AVS для получения оповещений:

* Связано со службой
* Для автоматической обработки

## <a name="avs-operator-access"></a>Доступ к оператору AVS

Вы можете управлять доступом к порталу AVS для персонала операций службы. При отправке запроса в службу поддержки персонал службы "операции входа" на портал. Операции службы исправляют все обнаруженные проблемы, и действия, выполненные в журналах аудита.

## <a name="users"></a>Пользователи

Все пользователи, у которых есть роль " **владелец** " и " **участник** " в подписке, имеют доступ к порталу AVS. При доступе к порталу пользователь создается в учетной записи AVS. Вы можете отключить доступ к порталу AVS для конкретных пользователей на странице учетные записи.

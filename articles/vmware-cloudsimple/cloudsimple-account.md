# <a name="account-management-overview"></a>Общие сведения об управлении учетной записью

При создании сервиса CloudSimple он создает учетную запись в CloudSimple.  Учетная запись связана с подпиской Azure, в которой находится служба.  Все пользователи с функциями **владельца** и **автора** в подписке имеют доступ к порталу CloudSimple.  Идентификатор подписки Azure и идентификатор клиента, связанный с сервисом CloudSimple, можно найти на [странице Счетов.](account.md)

## <a name="additional-alert-emails"></a>Дополнительные сообщения о помесье

Вы можете настроить идея электронной почты в CloudSimple для получения оповещений:

* Связанные с вашим сервисом
* Для автоматической обработки

## <a name="cloudsimple-operator-access"></a>Доступ оператора CloudSimple

Вы можете контролировать доступ к порталу CloudSimple для персонала службы.  Сотрудники службы регистра регистратесь на портале при подаче билета на поддержку.  Операции службы исправят все проблемы, о которых сообщается, и принятые меры доступны в журналах аудита.

## <a name="users"></a>Пользователи

Все пользователи, которые имеют **роль владельца** и **участника** подписки, имеют доступ к порталу CloudSimple.  При доступе к порталу пользователь создается на учетной записи CloudSimple.  Вы можете отключить доступ к порталу CloudSimple для конкретных пользователей со страницы Учетных записей.

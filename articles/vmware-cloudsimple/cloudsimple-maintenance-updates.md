# <a name="cloudsimple-maintenance-and-updates"></a>Обслуживание и обновления CloudSimple

Среда Private Cloud предназначена для того, чтобы не иметь ни одной точки сбоя.

* Кластеры ESXi настроены с vSphere High Availability (HA). Кластеры имеют размер по крайней мере один запасной узлы для устойчивости.
* Избыточное первичное хранилище обеспечивается vSAN, что требует по крайней мере трех узлов для защиты от одного сбоя. vSAN может быть настроен таким образом, чтобы обеспечить более высокую устойчивость больших кластеров.
* vCenter, PSC и NSX Manager VMs настроены с помощью хранилища RAID-10 для защиты от сбоя хранилища. VM защищены от сбоев узлов/сетей vSphere HA.
* У хостов ESXi есть избыточные вентиляторы и NICs.
* Переключения TOR и позвоночника настроены в парах HA для обеспечения устойчивости.

CloudSimple постоянно отслеживает следующие ВМ для простоя и доступности, а также обеспечивает доступность SLAs:

* Хосты ESXi
* vCenter
* Psc
* Менеджер NSX

CloudSimple также постоянно отслеживает следующие сбои:

* Жесткие диски
* Физические порты NIC
* Серверы
* Вентиляторы
* Power
* Коммутаторы
* Переключать порты

Если диск или узла выходит из строя, новый узлы автоматически добавляются в пораженный кластер VMware, чтобы немедленно вернуть его к работоспособности.

CloudSimple резервирует, поддерживает и обновляет эти элементы VMware в частных облаках:

* Esxi
* VCenter Платформа Услуги
* Контроллер
* vSAN
* Nsx

## <a name="back-up-and-restore"></a>Резервное копирование и восстановление

Резервное копирование CloudSimple включает в себя:

* Ночные дополнительные резервные копирования правил vCenter, PSC и DVS.
* vCenter родные AIS для резервного копирования компонентов на уровне приложения.
* Автоматическое резервное копирование до обновления или обновления программного обеспечения управления VMware.
* шифрование данных vCenter в источнике перед передачей данных по зашифрованном каналу TLS1.2 в Azure. Данные хранятся в blob Azure, где они реплицируются в разных регионах.

Вы можете запросить восстановление, открыв [запрос на поддержку.](https://portal.azure.com/#blade/Microsoft_Azure_Support/HelpAndSupportBlade/newsupportrequest)

## <a name="maintenance"></a>Обслуживание

CloudSimple выполняет несколько типов планового обслуживания.

### <a name="backendinternal-maintenance"></a>Бэкэнд/внутреннее техническое обслуживание

Это обслуживание обычно включает в себя перенастройку физических активов или установку программных патчей. Это не влияет на нормальное потребление обслуживаемых активов. При избытке NICs, идущих на каждую физическую стойку, нормальный сетевой трафик и операции Private Cloud не затрагиваются. Вы можете заметить влияние производительности только в том случае, если организация планирует использовать полную избыточную пропускную способность во время интервала обслуживания.

### <a name="cloudsimple-portal-maintenance"></a>Обслуживание портала CloudSimple

При обновлении системы управления или инфраструктурой CloudSimple требуется ограниченное время простоя службы. В настоящее время интервалы обслуживания могут быть столь же частыми, как один раз в месяц. Ожидается, что со временем эта частота будет снижаться. CloudSimple предоставляет уведомления для обслуживания портала и сохраняет интервал как можно более коротким. Во время интервала обслуживания портала следующие службы продолжают функционировать без какого-либо воздействия:

* Самолет управления vMware и приложения
* vCenter доступ
* Все сети и хранилища
* Весь трафик Azure

### <a name="vmware-infrastructure-maintenance"></a>Обслуживание инфраструктуры VMware

Иногда необходимо вносить изменения в конфигурацию инфраструктуры VMware.  В настоящее время эти интервалы могут происходить каждые 1-2 месяца, но частота, как ожидается, будет снижаться с течением времени. Этот тип обслуживания обычно может быть выполнен без прерывания нормального потребления служб CloudSimple. Во время интервала обслуживания VMware следующие службы продолжают функционировать без какого-либо воздействия:

* Самолет управления vMware и приложения
* vCenter доступ
* Все сети и хранилища
* Весь трафик Azure

## <a name="updates-and-upgrades"></a>Обновления и обновления

CloudSimple отвечает за управление жизненным циклом программного обеспечения VMware (ESXi, vCenter, PSC и NSX) в частном облаке.

Обновления программного обеспечения включают в себя:

* **Патчи**. Патчи безопасности или исправления ошибок, выпущенные VMware.
* **Обновления**. Незначительное изменение версии компонента стека VMware.
* **Обновления**. Основные изменения версии компонента стека VMware.

CloudSimple тестирует критический патч безопасности, как только он становится доступным от VMware. В течение недели CloudSimple выкатывает патч безопасности в среду Private Cloud.

CloudSimple обеспечивает ежеквартальные обновления технического обслуживания компонентов программного обеспечения VMware. Когда новая основная версия программного обеспечения VMware доступна, CloudSimple работает с клиентами, чтобы координировать подходящее окно обслуживания для обновления.

# <a name="getting-started-with-azure-metrics-explorer"></a>Приступая к работе с обозревателем метрик Azure

## <a name="where-do-i-start"></a>С чего начать
Обозреватель метрик Azure Monitor является компонентом портала Microsoft Azure, который позволяет создавать диаграммы, визуально коррелируемые тренды и изучать пики и спады в значениях метрик. Используйте обозреватель метрик для анализа работоспособности и использовании ресурсов. Запустите в следующем порядке:

1. [Выберите ресурс и метрику](#create-your-first-metric-chart) и вы увидите простой диаграммы. Затем [выберите диапазон времени](#select-a-time-range) , необходимые для анализа.

1. Попробуйте [применения фильтров измерения и разбиение](#apply-dimension-filters-and-splitting). Фильтры и разделение позволяют анализировать, какие сегменты метрики влияет на в целом метрики значение и определить возможные выбросы.

1. Используйте [Дополнительные параметры](#advanced-chart-settings) для настройки диаграммы перед закреплением на информационных панелях. [Настройка оповещений](alerts-metric-overview.md) получать уведомления, когда значение метрики выше или опускается ниже порогового значения.

## <a name="create-your-first-metric-chart"></a>Создайте свою первую диаграмму метрики

Чтобы создать диаграмму метрики из ресурсов, группы ресурсов, подписке или представление монитора Azure, откройте **метрики** вкладку и выполните следующие действия:

1. Используя средство выбора ресурсов, выберите ресурс, для которого требуется просмотреть метрики. (Ресурс будет выбран, если вы открыли **метрики** в контексте конкретного ресурса).

    > ![Выбрать ресурс](./media/metrics-getting-started/resource-picker.png)

2. Для некоторых ресурсов необходимо выбрать пространство имен. Пространство имен — это просто способ организовать метрики, таким образом, чтобы их можно легко найти. Например учетные записи хранения имеют отдельные пространства имен для хранения файлов, таблиц, больших двоичных объектов и очередей метрики. Многие типы ресурсов могут иметь только одно пространство имен.

3. Выберите метрику из списка доступных метрик.

    > ![Выберите метрику](./media/metrics-getting-started/metric-picker.png)

4. При необходимости можно изменить метрик статистической обработки. Например можно в диаграмму, чтобы показать минимальное, максимальное и среднее значения показателя.

> [!NOTE]
> Используйте **добавить метрику** кнопку и повторите эти шаги, если вы хотите увидеть несколько метрик, на одной диаграмме. Несколько диаграмм в одном представлении выберите **добавить диаграмму** кнопку в верхней части.

## <a name="select-a-time-range"></a>Выберите диапазон времени

По умолчанию диаграмма отображает за последние 24 часов данных метрик. Используйте **средство выбора времени** панели, чтобы изменить диапазон времени, увеличить или уменьшить масштаб на диаграмме. 

![Панель диапазон времени изменений](./media/metrics-getting-started/time-picker.png)

## <a name="apply-dimension-filters-and-splitting"></a>Применение фильтров измерения и разбиение

[Фильтрация](metrics-charts.md#apply-filters-to-charts) и [разделение](metrics-charts.md#apply-splitting-to-a-chart) являются мощными средствами диагностики для метрик, измерениями. Эти функции Показать как различные метрики сегментов («измерение значения») изменится общее значение метрики и идентифицировать возможные выбросы.

- **Фильтрация** позволяет выбрать значения измерений, которые включены в диаграмму. Например, может потребоваться Показать успешные запросы, если построение диаграмм *время ответа сервера* метрики. Необходимо применить фильтр к *успех запроса* измерения. 

- **Разделение** элементы управления ли на диаграмме отображаются отдельные строки для каждого значения измерения или статистической обработки значений в одну строку. Например можно см. в разделе одну строку для среднем времени ответа на всех экземплярах сервера, или отдельные строки для каждого сервера см. в разделе. Необходимо применить по *экземпляр сервера* измерения для просмотра отдельных строк.

См. в разделе [Примеры диаграмм](metric-chart-samples.md) , фильтрации и разделения применения. Здесь показано, что действия были использованы для настройки диаграммы.

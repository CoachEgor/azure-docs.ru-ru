# <a name="getting-started-with-azure-metrics-explorer"></a>Приступая к работе с Azure обозреватель метрик

## <a name="where-do-i-start"></a>Начало работы
Обозреватель метрик Azure Monitor является компонентом портала Microsoft Azure, который позволяет создавать диаграммы, визуально коррелируемые тренды и изучать пики и спады в значениях метрик. Используйте обозреватель метрик для изучения работоспособности и использования ресурсов. Запустите в следующем порядке:

1. [Выберите ресурс и метрику](#create-your-first-metric-chart) , и вы увидите простую диаграмму. Затем [выберите диапазон времени](#select-a-time-range) , соответствующий вашему исследованию.

1. Попробуйте [Применить фильтры измерений и разделение](#apply-dimension-filters-and-splitting). Фильтры и разделение позволяют анализировать, какие сегменты метрики вносят вклад в общее значение метрики и выдают возможные выбросы.

1. Используйте [Дополнительные параметры](#advanced-chart-settings) , чтобы настроить диаграмму перед закреплением на панелях мониторинга. [Настройте оповещения](alerts-metric-overview.md) для получения уведомлений, когда значение метрики превышает или падает ниже порогового значения.

## <a name="create-your-first-metric-chart"></a>Создание первой диаграммы метрик

Чтобы создать диаграмму метрики, в представлении ресурса, группы ресурсов, подписки или Azure Monitor откройте вкладку метрики и выполните следующие действия.

1. В средстве выбора ресурсов выберите ресурс, для которого требуется просмотреть метрики. (Ресурс предварительно выбирается, если вы открыли **метрики** в контексте определенного ресурса).

    > ![Выбрать ресурс](./media/metrics-getting-started/resource-picker.png)

2. Для некоторых ресурсов необходимо выбрать пространство имен. Пространство имен — это просто способ организации метрик, чтобы их можно было легко найти. Например, учетные записи хранения имеют отдельные пространства имен для хранения метрик файлов, таблиц, больших двоичных объектов и очередей. Многие типы ресурсов имеют только одно пространство имен.

3. Выберите метрику из списка доступных метрик.

    > ![Выберите метрику](./media/metrics-getting-started/metric-picker.png)

4. При необходимости можно изменить агрегат метрики. Например, может потребоваться, чтобы в диаграмме отображалось минимальное, максимальное или среднее значение метрики.

> [!NOTE]
> Нажмите кнопку **Добавить метрику** и повторите эти действия, если вы хотите увидеть несколько метрик, отображенных на одной диаграмме. Для нескольких диаграмм в одном представлении нажмите кнопку **Добавить диаграмму** сверху.

## <a name="select-a-time-range"></a>Выберите диапазон времени

По умолчанию на диаграмме отображаются данные метрик за последние 24 часа. Используйте панель **выбора времени** , чтобы изменить диапазон времени, увеличить или уменьшить масштаб диаграммы. 

![Изменить панель диапазона времени](./media/metrics-getting-started/time-picker.png)

> [!NOTE]
> Используйте **кисть времени** для изучения интересной области диаграммы (пика или DIP). Наведите указатель мыши в начало области, щелкните и удерживайте левую кнопку мыши, перетащите курсор на другую сторону области, а затем отпустите кнопку. Диаграмма будет уменьшаться в течение этого диапазона времени. 

## <a name="apply-dimension-filters-and-splitting"></a>Применение фильтров измерений и разделения

[Фильтрация](metrics-charts.md#apply-filters-to-charts) и [разбиение](metrics-charts.md#apply-splitting-to-a-chart) — это мощные средства диагностики для метрик, имеющих измерения. Эти функции показывают, как различные сегменты метрик ("значения измерения") влияют на общее значение метрики и позволяют выявление возможных выбросов.

- **Фильтрация** позволяет выбрать, какие значения измерения включаются в диаграмму. Например, может потребоваться отобразить успешные запросы при диаграмме метрики *времени ответа сервера* . Необходимо применить фильтр к *успешному* измерению запроса. 

- **Разделение** определяет, отображаются ли на диаграмме отдельные линии для каждого значения измерения, или объединяет значения в одну строку. Например, можно увидеть одну строку для среднего времени ответа во всех экземплярах сервера или отдельные строки для каждого сервера. Для просмотра отдельных строк необходимо применить разделение в измерении « *экземпляр сервера* ».

См. [Примеры диаграмм](metric-chart-samples.md) с примененными фильтрами и разбиением. В этой статье показаны шаги, которые использовались для настройки диаграмм.

## <a name="advanced-chart-settings"></a>Дополнительные параметры диаграммы

Можно настроить стиль диаграммы, заголовок и изменить дополнительные параметры диаграммы. После завершения настройки закрепите ее на панели мониторинга, чтобы сохранить работу. Также можно настроить оповещения метрик. Чтобы узнать об этих и других расширенных возможностях обозревателя метрик Azure Monitor, следуйте инструкциям в [документации по продукту](metrics-charts.md) .

# <a name="connect-configuration-manager-to-azure-monitor"></a>Подключение менеджера конфигурации к монитору Azure
Среду настройки конечных точек Майкрософт можно подключить к Azure Monitor, чтобы синхронизировать данные сбора устройств и ссылаться на эти коллекции в Azure Monitor и Azure Automation.  

## <a name="prerequisites"></a>Предварительные требования

Azure Monitor поддерживает текущую ветку configuration Manager, версию 1606 и выше.

>[!NOTE]
>Функция подключения менеджера конфигурации с рабочим пространством Log Analytics необязательна и не включена по умолчанию. Перед использованием ее необходимо включить. Дополнительные сведения см. в разделе [Enable optional features from updates](https://docs.microsoft.com/configmgr/core/servers/manage/install-in-console-updates#bkmk_options).

## <a name="configuration-overview"></a>Общие сведения о настройке

Следующие шаги обобщаются шаги по настройке интеграции менеджера конфигурации с Azure Monitor.  

1. В Active Directory Azure зарегистрируйте менеджера конфигурации как приложение Web Application and/or Web API и убедитесь, что у вас есть идентификатор клиента и секретный ключ клиента от регистрации от Azure Active Directory. Подробные сведения о выполнении этого шага см. в статье [Создание приложения Azure Active Directory и субъекта-службы с доступом к ресурсам с помощью портала](../../active-directory/develop/howto-create-service-principal-portal.md).

2. В Active Directory Azure [предоставите configuration Manager (зарегистрированное веб-приложение) с разрешением на доступ к Azure Monitor.](#grant-configuration-manager-with-permissions-to-log-analytics)

3. В настройке Manager добавьте соединение с помощью мастера **служб Azure.**

4. [Скачать и установить агент журнала Analytics для Windows](#download-and-install-the-agent) на компьютере под управлением configuration Manager службы подключения точки сайта роли. Агент отправляет данные менеджера конфигурации в рабочее пространство log Analytics в Azure Monitor.

5. В Azure Monitor [импортируйте коллекции от Configuration Manager](#import-collections) в качестве компьютерных групп.

6. В Azure Monitor просмотр данных от configuration Manager как [компьютерных групп.](computer-groups.md)

## <a name="grant-configuration-manager-with-permissions-to-log-analytics"></a>Предоставление Configuration Manager разрешений для Log Analytics

В следующей процедуре вы предоставляете роль *Участник* в рабочей области Log Analytics для приложения AD и субъекта-службы, созданных ранее для Configuration Manager. Если у вас еще нет рабочего пространства, [см.](../../azure-monitor/learn/quick-create-workspace.md) Это позволит Configuration Manager проходить аутентификацию и подключаться к рабочей области Log Analytics.  

> [!NOTE]
> Необходимо указать разрешения в рабочей области log Analytics для менеджера конфигурации. В противном случае при использовании мастера конфигурации в Configuration Manager вы получите сообщение об ошибке.
>

1. На портале Azure щелкните **Все службы** в нижнем левом углу. В списке ресурсов введите **log Analytics**. Как только вы начнете вводить символы, список отфильтруется соответствующим образом. Выберите **аналитику журнала**.

2. В списке рабочих областей Log Analytics выберите рабочую область, которую требуется изменить.

3. В области слева выберите **Управление доступом (IAM)**.

4. На странице "Управление доступом (IAM)" щелкните **Добавить назначение ролей **, после чего откроется **соответствующая область**.

5. В области **Добавить назначение ролей ** из раскрывающегося списка **Роли** выберите роль **Участник**.  

6. Из раскрывающегося списка **Назначение доступа к** выберите приложение Configuration Manager, ранее созданное в AD, и нажмите кнопку **ОК**.  

## <a name="download-and-install-the-agent"></a>Загрузка и установка агента

Просмотрите статью [Подключите компьютеры Windows к Azure Monitor в Azure,](agent-windows.md) чтобы понять методы, доступные для установки агента Log Analytics для Windows на компьютере, принимающем роль точки подключения Configuration Manager.  

## <a name="connect-configuration-manager-to-log-analytics-workspace"></a>Подключение менеджера конфигурации к рабочему пространству аналитики журналов

>[!NOTE]
> Для добавления подключения к журналу Analytics среда Configuration Manager должна иметь [точку подключения службы,](https://docs.microsoft.com/configmgr/core/servers/deploy/configure/about-the-service-connection-point) настроенную для онлайн-режима.

> [!NOTE]
> Необходимо подключить сайт верхнего уровня в иерархии к Azure Monitor. Если вы подключите автономный основной узел к Azure Monitor, а затем добавите сайт центральной администрации в среду, необходимо удалить и воссоздать соединение в новой иерархии.

1. В рабочей области **администрирования** менеджера конфигурации выберите **службы облака,** а затем выберите **службы Azure.** 

2. Нажмите кнопкой мыши **Azure Services** и выберите **Настройка служб Azure.** Отображается страница **службы настройки Azure.** 
   
3. На экране **Общие** убедитесь, что вы выполнили следующие действия и указали сведения по каждому пункту, а затем щелкните **Далее**.

4. На странице Службы Azure мастера служб Azure выполните следующие действия:

    1. Укажите **Имя** объекта в Configuration Manager.
    2. При желании укажите **Описание**, которое поможет идентифицировать службу.
    3. Выберите службу **Azure OMS Connector.**

    >[!NOTE]
    >В настоящее время OMS называется журналом Analytics, который является особенностью Azure Monitor.

5. Нажмите кнопку **Далее**, чтобы перейти на страницу Свойства приложения Azure мастера служб Azure.

6. На странице **приложения** Мастера служб Azure сначала выберите среду Azure из списка, а затем нажмите **«Импорт».**

7. На странице **импортных приложений** укажите следующую информацию:

    1. Укажите для приложения **имя арендатора Azure AD.**

    2. Укажите для **ИДента Натуры AD идентификатор Azure AD** для арендатора Azure AD. Эту информацию можно найти на странице Azure Active Directory **Properties.** 

    3. Укажите для **имени приложения** имя приложения.

    4. Укажите для **Идентификатор клиента**— идентификатор приложения созданного ранее приложения Azure AD.

    5. Укажите для **секретного ключа**, секретный ключ клиента созданного приложения Azure AD.

    6. Укажите для **Secret Key истечение**срока действия, срок действия вашего ключа.

    7. Укажите для **App ID URI**, App ID URI созданного ранее приложения Azure AD.

    8. Выберите **Проверить** и справа результаты должны показать **Успешно проверены!**.

8. На странице **Configuration** просмотрите информацию для проверки **подписок Azure,** **группы ресурсов Azure**и полей **рабочего пространства Управления операциями ,Management Suite** предварительно заселены, что указывает на то, что приложение Azure AD имеет достаточно ежеаков в группе ресурсов. Если поля пусты, это означает, что приложение не имеет требуемых прав. Выберите коллекции устройств для сбора и переадресова в рабочее пространство, а затем выберите **Добавить.**

9. Просмотрите параметры на странице **«Подтверждение настроек»** и выберите **далее,** чтобы начать создание и настройку соединения.

10. Когда конфигурация закончена, появляется страница **Завершения.** Выберите **Закрыть**. 

После подключения менеджера конфигурации к Azure Monitor можно добавить или удалить коллекции и просмотреть свойства соединения.

## <a name="update-log-analytics-workspace-connection-properties"></a>Обновление свойств рабочего пространства анализа журналов

Если срок действия ключа от добавления пароля или секретного ключа клиента истекает или теряется, необходимо вручную обновлять свойства подключения Log Analytics.

1. В **рабочей** области администрирования менеджера конфигурации выберите **облачные службы,** а затем выберите **РАЗъем OMS,** чтобы открыть страницу **OMS Connection Properties.**
2. На этой странице щелкните **Azure Active Directory**, чтобы просмотреть сведения о **клиенте**, **идентификаторе клиента** и **сроке действия секретного ключа клиента**. **Проверьте****секретный ключ клиента**, если срок его действия истек.

## <a name="import-collections"></a>Импорт коллекций

После добавления подключения log Analytics в configuration Manager и установлен агента на компьютере, работая над ролью точки подключения Configuration Manager, следующим шагом будет импортировать коллекции из configuration Manager в Azure Монитор как компьютерные группы.

После завершения начальной конфигурации для импорта коллекций устройств из иерархии информация о сборе извлекается каждые 3 часа, чтобы сохранить значение членства. Это можно отключить в любое время.

1. На портале Azure щелкните **Все службы** в нижнем левом углу. В списке ресурсов введите **log Analytics**. Как только вы начнете вводить символы, список отфильтруется соответствующим образом. Выберите **Рабочие области Log Analytics**.
2. Из списка рабочих областей Log Analytics выберите ту, в которой зарегистрировано приложение Configuration Manager.  
3. Выберите **Дополнительные параметры**.
4. Выберите **Группы компьютеров**, а затем щелкните **SCCM**.  
5. Выберите **Импортировать членства в коллекциях Configuration Manager** и нажмите кнопку **Сохранить**.  
   
    ![Вкладка "SCCM" на вкладке "Группы компьютеров"](./media/collect-sccm/sccm-computer-groups01.png)

## <a name="view-data-from-configuration-manager"></a>Просмотр данных из Configuration Manager

После добавления подключения log Analytics в configuration Manager и установки агента на компьютере, работая над ролью точки подключения Configuration Manager, данные агента отправляются в рабочее пространство журнала Analytics в Azure Monitor. В Azure Monitor коллекции configuration Manager отображаются как [компьютерные группы.](../../azure-monitor/platform/computer-groups.md) Группы можно просматривать на странице **Configuration Manager**, выбрав **Параметры > Группы компьютеров**.

После импорта коллекций можно увидеть число обнаруженных компьютеров с членством в коллекциях. Также можно увидеть число импортированных коллекций.

![Вкладка "SCCM" на вкладке "Группы компьютеров"](./media/collect-sccm/sccm-computer-groups02.png)

При нажатии на одну из них открывается редактор запросов журнала, отображающий либо все импортируемые группы, либо все компьютеры, принадлежащие к каждой группе. С помощью [поиска журнала](../../azure-monitor/log-query/log-query-overview.md)вы можете выполнить дальнейший углубленный анализ данных о членстве в сборе.

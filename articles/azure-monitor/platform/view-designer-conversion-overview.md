# <a name="azure-monitor-view-designer-to-workbooks-transition-guide"></a>Дизайнер представления Azure Monitor для руководства по переходу к трудовым книгам
[Дизайнер представления](view-designer.md) — это функция Azure Monitor, которая позволяет создавать пользовательские представления, чтобы помочь вам визуализировать данные в рабочем пространстве Log Analytics с диаграммами, списками и сроками. Они постепенно сворачиваются и заменяются трудовыми книжеками, которые обеспечивают дополнительную функциональность. В этой статье приводится обзор процесса преобразования существующих представлений в трудовые книжки.

## <a name="workbooks-overview"></a>Обзор трудовых книжек
[В богатых интерактивных отчетах рабочие книги](../insights/vminsights-workbooks.md) объединяют текст, [запросы журналов,](../log-query/query-language.md)метрики и параметры. Члены группы с одинаковым доступом к ресурсам Azure также могут отощоблять трудовые книжки.

Учебники полезны для таких сценариев, как:

-   Изучение использования виртуальной машины, когда вы не знаете метрик интерес заранее: использование процессора, дисковое пространство, память, сетевые зависимости и т.д. В отличие от других инструментов аналитики использования, трудовые книжки позволяют сочетать несколько видов визуализаций и анализов, что делает их отличными для такого рода исследований свободной формы.
-   Объясняя вашей команде, как работает недавно подготовленный VM, показывая метрики для ключевых счетчиков и других событий журнала.
-   Совместное использование результатов эксперимента по размеру вашего VM с другими членами вашей команды. Вы можете объяснить цели эксперимента с текстом, а затем показать каждое использование метрики и аналитики запросы, используемые для оценки эксперимента, а также четкие вызовы для ли каждая метрика была выше или ниже цели.
-   Сообщение о влиянии сбоя на использование ВМ, объединение данных, текстовых пояснений и обсуждение дальнейших шагов по предотвращению простоев в будущем.


## <a name="why-convert-view-designer-dashboards-to-workbooks"></a>Зачем преобразовывать панели мониторинга представления в трудовые книжки?

Дизайнер представления предлагает возможность создавать различные представления и визуализации на основе запросов. Но многие высокоуровневые настройки, например форматирование сеток и макетов плиток или выбор альтернативной графики для представления данных, можно выполнить с ограничениями. Впредставление конструктора ограничено в общей сложности девятью различными плитками для представления ваших данных.

Книги — это платформа, которая позволяет использовать данные в полной мере. трудовые книжки не только сохраняют все возможности, но и поддерживают дополнительные функциональные возможности через текст, метрики, параметры и многое другое. Например, трудовые книжки позволяют пользователям консолидировать плотные сетки и добавлять панели поиска, чтобы легко фильтровать и анализировать данные. 

### <a name="advantages-of-using-workbooks-over-view-designer"></a>Преимущества использования справочников по просмотру дизайнера

* Поддерживает как журналы, так и метрики.
* Позволяет как личные представления для индивидуального управления доступом, так и общие представления трудовых книжек.
* Пользовательские параметры компоновки с вкладками, калибровкой и элементами масштабирования.
* Поддержка запросов в нескольких рабочих областях log Analytics, приложениях Application Insights и подписках.
* Включает пользовательские параметры, которые динамически обновляют связанные диаграммы и визуализации.
* Галерея шаблонов поддерживает сяочку от общественности GitHub.

В то время как это руководство предлагает простые шаги, чтобы непосредственно воссоздать некоторые из широко используемых представлений дизайнера представления, трудовые книжки позволяют пользователям иметь свободу создавать и проектировать любые свои собственные пользовательские визуализации и метрики. ## <a name="how-to-start-using-workbooks"></a>Как начать использовать трудовые книжки
Открытые трудовые книжки из трудовых книжек включены в рабочие области Log Analytics в качестве элемента в боковой панели навигации, непосредственно под расположением дизайнера представления.

![Навигация по трудовым книгам](media/view-designer-conversion-overview/workbooks-nav.png)

После выбора будет отображаться галерея, в ней будут перечислены все сохраненные трудовые книжки и шаблоны для рабочего пространства.

![Галерея трудовых книжек](media/view-designer-conversion-overview/workbooks-gallery.png)

Чтобы начать новую книгу, вы можете выбрать **шаблон Пустой** под **быстрым запуском**или **новый** значок в верхней панели навигации. Чтобы просмотреть шаблоны или вернуться к сохраненным трудовым книжке, выберите элемент из галереи или ищите имя в панели поиска.

Чтобы сохранить рабочую книгу, вам нужно будет сохранить отчет с определенным названием, подпиской, группой ресурсов и местоположением.
Рабочая книга будет автоматически заполняться в тех же настройках, что и рабочее пространство Лос-Анджелеса, с той же подпиской, группой ресурсов, однако пользователи могут изменить эти настройки отчета. Рабочие книги по умолчанию сохраняются в *My Reports,* доступны только отдельному пользователю. Они также могут быть сохранены непосредственно в общих отчетах или совместно позже.

![Трудовые книжки сохранить](media/view-designer-conversion-overview/workbooks-save.png)

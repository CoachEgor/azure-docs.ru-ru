# <a name="view-designer-to-workbooks-conversion-common-tasks"></a>Просмотр дизайнера в трудовые книжки общих задач
[Дизайнер представления](view-designer.md) — это функция Azure Monitor, которая позволяет создавать пользовательские представления, чтобы помочь вам визуализировать данные в рабочем пространстве Log Analytics с диаграммами, списками и сроками. Они постепенно сворачиваются и заменяются трудовыми книжеками, которые обеспечивают дополнительную функциональность. В этой статье подробно описаны задачи, которые являются общими при преобразовании представлений в трудовые книжки.


## <a name="quickstart-with-preset-view-designer-templates"></a>Быстрый запуск с предустановленными шаблонами представления дизайнера

В рабочих областях журналной аналитики уже есть шаблоны, составленные в соответствии с некоторыми представлениями в представлении дизайнера. В категории **View Designer Guides** выберите **Руководство по переходу к дизайну View Designer,** чтобы узнать о ваших вариантах или выберите один из заданным шаблонов.

![Образцы шаблонов](media/view-designer-conversion-tasks/templates.png)

## <a name="enabling-time-range-filter"></a>Включение фильтра диапазона времени
В представлении конструктора есть встроенный фильтр диапазона времени по умолчанию, однако в трудовых книжках эта настройка не включена по умолчанию. Учебники позволяют пользователям создавать свои собственные фильтры диапазона времени, которые могут быть более применимы к их журналам данных. Шаги для создания фильтра перечислены ниже:

Выберите опцию **параметров добавления.** **Стиль** по умолчанию установлен на *таблетки.*

![Добавить Парам](media/view-designer-conversion-tasks/add-param.png)

 Выберите кнопку **«Добавить параметр».**

![Добавить параметр](media/view-designer-conversion-tasks/add-parameter.png)

Из бокового меню, в текстовом ящике **названия параметра,** введите *TimeRange*. Установить **тип параметра** как *выборщик диапазона времени*. Выберите **обязательный?** флажок.

![Параметр меню](media/view-designer-conversion-tasks/parameter-menu.png)

Сохранить параметр в верхнем левом углу бокового меню. Вы можете оставить выпадение как *неустановленное* по умолчанию или выбрать значение **TimeRange** по умолчанию, например *24 часа.* Выберите **готовое редактирование**.

Параметры могут быть использованы в запросах, {} добавляя фигурные скобки вокруг вашего параметра имени. Более подробную информацию о параметрах можно найти в [документации по параметрам.](https://github.com/microsoft/Application-Insights-Workbooks/blob/master/Documentation/Parameters/Parameters.md)

## <a name="updating-queries-with-the-timerange-parameter"></a>Обновление запросов с параметром TimeRange

### <a name="option-1-select-timerange-from-the-time-range-dropdown"></a>Вариант 1: Выберите TimeRange из отпадения диапазона времени

![Параметр времени](media/view-designer-conversion-tasks/time-parameter.png)

### <a name="option-2-update-your-log-queries"></a>Вариант 2: Обновление запросов журнала

В запросе добавляйте `| where TimeGenerated {TimeRange}` строку: как в следующем примере:

Исходный запрос
```KQL
search * 
| summarize count() by Type
```

Обновленный запрос
```KQL
search * 
| where TimeGenerated {TimeRange} 
| summarize count() by Type
```

## <a name="including-a-list"></a>Включая список
Большинство представлений представления-конструктора содержат список, и вы можете воспроизвести этот стандартный список в трудовой книжке.

![Список плитки](media/view-designer-conversion-tasks/tile-list.png)

Добавьте визуализацию, нажав **на запрос Добавить** из параметров ячейки.

![Добавить Парам](media/view-designer-conversion-tasks/add-param.png)

В представлении конструктора используется запрос по умолчанию, который соответствует синтаксису из примера Original. Это может быть обновлено путем изменения запроса на обновленную форму, как в следующем примере:

Исходный запрос
```KQL
search * 
| summarize AggregatedValue = count() by Type
```

Обновленный запрос
```KQL
search * 
| summarize Count = count() by Type
```

Это позволит создать список, который выглядит как следующее:

![Пример списка](media/view-designer-conversion-tasks/list-example.png)

## <a name="enabling-sparklines"></a>Включение искры
Общей особенностью для сеток является добавление искры для обобщения различных шаблонов данных с течением времени. Посмотреть дизайнер предлагает **функцию Enable Sparklines** для всех списков, как и трудовые книжки. Чтобы включить искры в данные, которые соответствуют конструктору представления, присоединяйтесь к данным с исходным запросом, как в следующем примере:

Исходный запрос
```KQL
search *
| summarize AggregatedValue = count() by Type) on Type
```

Обновленный запрос
```KQL
search * 
| summarize AggregatedValue = count() by Type
| join kind = inner (search * 
    | make-series Trend = count() default=0 on TimeGenerated from {TimeRange:start} to {TimeRange:end} step {TimeRange:grain} by Type) on Type
| project Type, AggregatedValue, Trend
```

Выберите **параметры столбца**.
![Настройки столбца](media/view-designer-conversion-tasks/column-settings.png)

Обновление выпадения **визгона столбца,** чтобы быть *областью Spark.*
![Спарклайны](media/view-designer-conversion-tasks/sparkline.png)

Сохраните настройки и запустите запрос снова, чтобы обновить таблицу, чтобы включить искру.

Полученная сетка будет выглядеть так же, как и следующая: ![пример Sparkline](media/view-designer-conversion-tasks/sparkline-example.png)

## <a name="advanced-cell-settings"></a>Расширенные настройки ячейки
Чтобы отразить конструктор ателье представления, можно выполнять такие задачи, как изменение размера ячеек трудовой книжки или добавление контактов и внешних ссылок в журналы.

Для доступа к **Расширенным Настройкам** выберите значок передачви в нижней части каждой ячейки.

![Дополнительные параметры](media/view-designer-conversion-tasks/advanced-settings.png)

Здесь будет отображаться меню с различными вариантами:

![Расширенные настройки настройки](media/view-designer-conversion-tasks/advanced-settings-settings.png)

Чтобы добавить пин-код и ссылку на внешний запрос, выберите соответствующие флажки. Чтобы добавить заголовок в ячейку, введите желаемое название в разделе **заголовка диаграммы.**

По умолчанию любая ячейка трудовых книжек устанавливается, чтобы занять всю ширину страницы, но вы можете настроить это путем масштабирования ячейки вниз под вкладкой **Стиль** меню **Расширенные настройки**

![Расширенный стиль настроек](media/view-designer-conversion-tasks/advanced-settings-style.png)

 
## <a name="additional-parameters"></a>Дополнительные параметры
Выберите **Добавить Параметр** для создания нового параметра в вашей трудовой книжке. 

Чтобы выбрать подписку, введите *подписку* в поле **имени параметра** в боковом меню и выберите *подписной сборщик* из типа **Параметр**

![Меню подписки](media/view-designer-conversion-tasks/subscription-filter.png)

Чтобы выбрать ресурс, введите *ресурс* в поле **имени параметра** в боковом меню и выберите *ресурсный сборщик* из типа **Параметр.**

![Меню ресурсов](media/view-designer-conversion-tasks/resource-filter.png)

Это позволит вставить выпадающие данные, чтобы позволить вам получить доступ к различным подпискам и ресурсам.

![Падение ресурсов подписки](media/view-designer-conversion-tasks/subscription-resource.png)

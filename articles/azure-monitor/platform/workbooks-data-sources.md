# <a name="azure-monitor-workbooks-data-sources"></a>Источники данных трудовых книжек Azure Monitor

Трудовые книжки совместимы с большим количеством источников данных. В этой статье вы проведете поиск истоков данных, которые в настоящее время доступны для трудовых книжек Azure Monitor.

## <a name="logs"></a>Журналы

В учебниках можно задать запрос журналов из следующих источников:

* Журналы мониторинга Azure (Ресурсы анализа приложений и рабочие области аналитики журналов.)
* Ресурсоориентированные данные (журналы активности)

Авторы книги могут использовать запросы к K'L, которые преобразуют базовые данные ресурса, чтобы выбрать набор результатов, который может быть визуализирован как текст, диаграммы или сетки.

![Скриншот интерфейса отчетов журналов трудовых книжек](./media/workbooks-overview/logs.png)

Авторы книги могут легко запрашивать на нескольких ресурсах, создавая действительно унифицированный богатый опыт отчетности.

## <a name="metrics"></a>Метрики

Ресурсы Azure излучают [метрики,](data-platform-metrics.md) к которым можно получить доступ через трудовые книжки. К метрикам можно получить доступ в трудовых книжках с помощью специализированного управления, который позволяет указать целевые ресурсы, нужные метрики и их агрегацию. Эти данные могут быть построены на диаграммах или сетках.

![Скриншот диаграмм метрик книжки cpu](./media/workbooks-overview/metrics-graph.png)

![Скриншот интерфейса метрик рабочей книги](./media/workbooks-overview/metrics.png)

## <a name="azure-resource-graph"></a>Azure Resource Graph 

Рабочие книги поддерживают запросы ресурсов и их метаданных с помощью графика ресурсов Azure (ARG). Эта функциональность в основном используется для создания пользовательских областей запросов для отчетов. Область ресурса выражается через подмножество КЗЛ, которое поддерживает ARG, что часто достаточно для общего использования случаев.

Чтобы сделать так, чтобы элемент управления запросом использовал этот источник данных, используйте выпадение типа запроса, чтобы выбрать график ресурсов Azure и выбрать целевые подписки. Используйте элемент управления запроса, чтобы добавить подмножество ARG КЗЛ, которое выбирает интересный подмножество ресурсов.


![Скриншот запроса Azure Resource Graph КЗЛ](./media/workbooks-overview/azure-resource-graph.png)

## <a name="alerts-preview"></a>Оповещения (предварительный просмотр)

В работах пользователи могут визуализировать активные оповещения, связанные с их ресурсами. Эта функция позволяет создавать отчеты, которые объединяют данные уведомлений (оповещения) и диагностическую информацию (метрики, журналы) в один отчет. Эта информация также может быть объединена для создания богатых отчетов, которые объединяют информацию в этих источниках данных.

Чтобы сделать контроль запроса, используйте этот источник данных, используйте выпадение типа запроса, чтобы выбрать оповещения и выбрать подписи, группы ресурсов или ресурсы для таргетинга. Используйте падение фильтра оповещения, чтобы выбрать интересный подмножество оповещений для ваших аналитических потребностей.

![Скриншот запроса оповещений](./media/workbooks-overview/alerts.png)

## <a name="workload-health-preview"></a>Работоспособность рабочей нагрузки (предварительный просмотр)

Azure Monitor имеет функциональность, которая активно отслеживает доступность и производительность гостевых операционных систем Windows или Linux. Azure Monitor моделирует ключевые компоненты и их взаимосвязь, критерии измерения работоспособности этих компонентов и какие компоненты предупреждают вас при обнаружении нездорового состояния. Учебники позволяют пользователям использовать эту информацию для создания богатых интерактивных отчетов.

Чтобы сделать так, чтобы элемент управления запросом использовал этот источник данных, используйте выпадение **типа запроса,** чтобы выбрать работу по работе с работой и выбрать для таргетинга подписку, группу ресурсов или ресурсы VM. Используйте падение фильтра работоспособности, чтобы выбрать интересный подмножество инцидентов для здоровья для ваших аналитических потребностей.

![Скриншот запроса оповещений](./media/workbooks-overview/workload-health.png)

## <a name="azure-resource-health"></a>Здоровье ресурсов Azure 

Учебники поддержки обеспечения работоспособности ресурсов Azure и объединения их с другими источниками данных для создания богатых интерактивных отчетов о состоянии здоровья

Чтобы сделать так, чтобы элемент управления запросом использовал этот источник данных, используйте выпадение **типа запроса,** чтобы выбрать работоспособность Azure и выбрать ресурсы для целевой. Используйте падение фильтра работоспособности, чтобы выбрать интересный подмножество проблем ресурсов для ваших аналитических потребностей.

![Скриншот запроса оповещений](./media/workbooks-overview/resource-health.png)

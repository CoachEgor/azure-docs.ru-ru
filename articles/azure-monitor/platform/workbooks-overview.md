# <a name="azure-monitor-workbooks"></a>Изучение и предоставление общего доступа к данным об использовании c интерактивными книгами в Application Insights

В рабочих учебниках обеспечивается гибкое полотно для анализа данных и создания богатых визуальных отчетов на портале Azure. Они позволяют использовать несколько источников данных со всей Azure и объединять их в единые интерактивные взаимодействия. 

## <a name="data-sources"></a>Источники данных

В учебниках можно заманить данные из нескольких источников azure. Авторы трудовых книжек могут преобразовать эти данные, чтобы получить представление о доступности, производительности, использовании и общем состоянии основных компонентов. Например, анализ журналов производительности с виртуальных машин для выявления высоких экземпляров cPU или низкой памяти и отображение результатов в виде сетки в интерактивном отчете.
  
Но реальная сила трудовых книжек заключается в способности комбинировать данные из разрозненных источников в рамках одного отчета. Это позволяет создавать композитные представления ресурсов или соединения между ресурсами, позволяющими более богатым данным и аналитическим данным, что в противном случае было бы невозможно.

В настоящее время трудовые книжки совместимы со следующими источниками данных:

* [Журналы](workbooks-data-sources.md#logs)
* [Метрики](workbooks-data-sources.md#metrics)
* [График ресурсов Azure](workbooks-data-sources.md#azure-resource-graph)
* [Оповещения (предварительная версия)](workbooks-data-sources.md#alerts-preview)
* [Здоровье рабочей нагрузки (Предварительный просмотр)](workbooks-data-sources.md#workload-health-preview)
* [Здоровье ресурсов Azure (Предварительный просмотр)](workbooks-data-sources.md#azure-resource-health)
* [Исследователь данных Azure (Предварительный просмотр)](workbooks-data-sources.md#azure-data-explorer-preview)

### <a name="workbooks-versus-workbook-templates"></a>Шаблоны трудовых книжек по сравнению с шаблонами трудовых книжек

Вы можете увидеть _трудовую книгу_ в зеленом цвете и ряд _шаблонов рабочей книги_ в фиолетовом цвете. Шаблоны служат кураторскими отчетами, предназначенными для гибкого повторного использования несколькими пользователями и группами. Открытие шаблона создает переходную трудовую книжку, наполненную содержанием шаблона. 

Вы можете настроить параметры работы на основе шаблонов и провести анализ, не опасаясь нарушить будущий опыт отчетности для коллег. Если вы откроете шаблон, внести некоторые коррективы, а затем выбрать значок сохранения вы будете сохранять шаблон в качестве рабочей книги, которая затем показать в зеленом оставляя оригинальный шаблон нетронутым. 

Под капотом шаблоны также отличаются от сохраненных трудовых книжек. Сохранение рабочей книги создает связанный ресурс Azure Resource Manager, в то время как переходная рабочая книга, созданная при простом открытии шаблона, не имеет уникального ресурса, связанного с ним. Чтобы узнать больше о том, как управление доступом управляется в трудовых книжках, проконсультируйтесь со [статьей управления доступом к трудовым книжкам.](workbooks-access-control.md)

### <a name="exploring-a-workbook-template"></a>Изучение шаблона рабочей книги

Выберите **анализ сбоев приложений,** чтобы увидеть один из шаблонов рабочей книги приложений по умолчанию.

![Скриншот шаблона анализа сбоев приложения](./media/workbooks-overview/failure-analysis.png)

Как указывалось ранее, открытие шаблона создает временную трудовую книжку для вас, чтобы иметь возможность взаимодействовать с. По умолчанию рабочая книга открывается в режиме чтения, который отображает только информацию для предполагаемого анализа, который был создан автором оригинального шаблона.

В случае этой конкретной трудовой книжки, опыт является интерактивным. Вы можете настроить подписку, целевые приложения и временной диапазон данных, которые вы хотите отобразить. После того как вы сделали эти выделения сетки http Requests также интерактивный, в результате чего выбор отдельной строки изменит то, какие данные отображаются в двух диаграммах в нижней части отчета.

## <a name="pinning-visualizations"></a>Закрепление визуализаций

Текст, запрос и метрики шаги в трудовой книжке можно прикрепить с помощью кнопки контакта на этих элементах, пока рабочая книга находится в режиме контакта, или если автор книги включил настройки для этого элемента, чтобы сделать значок контакта видимым. 

Чтобы получить доступ к режиму контакта, нажмите **Редактировать,** чтобы войти в режим редактирования, и выберите синий значок штыря в верхней панели. Индивидуальный значок контактного контакта будет отображаться над каждой соответствующей рабочей частью *Edit* поле на правой стороне экрана.

![Опыт работы с контактами](./media/workbooks-overview/pin-experience.png)

> [!NOTE]
> Состояние рабочей книги сохраняется во время пин-кода, и закрепленные трудовые книжки на панели мониторинга не будут обновляться, если базовая трудовая книжка изменена. Для того, чтобы обновить закрепленную часть рабочей книги, вам нужно будет удалить и повторно закрепить эту часть.

## <a name="dashboard-time-ranges"></a>Диапазоны времени приборной панели

Закрепленные части запроса рабочей книги будут уважать временной диапазон панели мониторинга, если закрепленный элемент настроен для использования параметра *Time Range.* Значение диапазона времени панели мониторинга будет использоваться в качестве значения параметра временного диапазона, а любое изменение диапазона времени приборной панели приведет к обновлению закрепленного элемента. Если закрепленная часть использует временной диапазон панели мониторинга, вы увидите субтитр обновления закрепленной части, чтобы показать временной диапазон приборной панели при изменении диапазона времени. 

Кроме того, закрепленные части рабочей книги, используя параметр временной диапазон, будут автоматически обновляться со скоростью, определяемой временной дальностью приборной панели. Последний раз, когда запрос был пропил, в подзаголовке закрепленной части.

Если закрепленный шаг имеет явно установленный временной диапазон (не использует параметр временной диапазон), этот временной диапазон всегда будет использоваться для панели мониторинга, независимо от настроек панели мониторинга. Подзаголовок закрепленной части не будет отображать временной диапазон панели мониторинга, а запрос не будет автоматически обновляться на приборной панели. В подзаголовке будет отображаться последний раз, когда выполняется запрос.

> [!NOTE]
> Запросы с использованием источника данных *слияния* в настоящее время не поддерживаются при закреплении на панелях мониторинга.

# <a name="overview-of-azure-platform-logs"></a>Общие сведения о журналах платформы Azure
Платформные журналы предоставляют подробную информацию о диагностике и аудите для ресурсов Azure и платформы Azure, от которой они зависят. Они автоматически генерируются, хотя необходимо настроить определенные журналы платформы для переадресовки в один или несколько пунктов назначения, которые должны быть сохранены. В этой статье приводится обзор журналов платформ, включая информацию, которая они предоставляют и как их можно настроить для сбора и анализа.

## <a name="types-of-platform-logs"></a>Типы журналов платформ
В следующей таблице перечислены определенные журналы платформы, доступные на разных уровнях Azure.

| Журнал | Уровень | Описание |
|:---|:---|:---|
| Журналы ресурсов | Ресурсы Azure | Предоставьте информацию об операциях, выполняемых в ресурсе Azure *(плоскость данных),* например, получая секрет из Убежища ключей или делая запрос в базу данных. Содержание журналов ресурсов зависит от типа службы Azure и типа ресурсов.<br><br>*Ранее журналы ресурсов назывались диагностическими журналами.*  |
| Журнал действий | Подписка Azure | Помимо обновлений о событиях Службы здоровья, в подписке извне *(план управления)* предоставляет информацию об операциях на каждом ресурсе Azure. Используйте журнал активности, чтобы определить, _что,_ _кто_, и _когда_ для любых операций записи (PUT, POST, DELETE) приняты на ресурсы в подписке. Вы также можете отслеживать состояние операции и другие ее свойства.  Для каждой подписки Azure существует единый журнал активности. |
| Активные журналы каталогов Azure | Клиент Azure |  Содержит историю активности в входе и аудиторский след изменений, внесенных в Active Directory Azure для конкретного арендатора. Смотрите, [что такое отчеты об активных каталогах Azure?](../../active-directory/reports-monitoring/overview-reports.md) для полного описания журналов активных каталогов Azure.   |

> [!NOTE]
> Журнал действий Azure используется главным образом для действий, которые происходят в Azure Resource Manager. Ресурсы, использующие классическую модель или модель RDFE, не отслеживаются. Для некоторых классических типов ресурсов в Azure Resource Manager имеется прокси-поставщик ресурсов (например, Microsoft.ClassicCompute). Если вы взаимодействуете с ресурсом классического типа через Azure Resource Manager с помощью этих прокси-поставщиков ресурсов, то такие операции отображаются в журнале действий. Если вы взаимодействуете с ресурсом классического типа за пределами прокси Azure Resource Manager, ваши действия записываются только в журнал операций. Журнал операций можно просмотреть в отдельном разделе портала.

## <a name="viewing-platform-logs"></a>Просмотр журналов платформы
Существуют различные варианты просмотра и анализа различных журналов платформы Azure.

- Просмотр журнала activity на портале Azure и событий доступа с PowerShell и CLI. Просмотр [и получение событий журналов azure Activity](activity-log-view.md) для получения подробной информации. 
- Просмотр отчетов об активной безопасности и действиях Azure на портале Azure. [Посмотреть, что такое отчеты об активном каталоге Azure?](../../active-directory/reports-monitoring/overview-reports.md)  .
- Журналы ресурсов автоматически генерируются поддерживаемыми ресурсами Azure, но они недоступны для просмотра, если вы не отправите их в [пункт назначения.](#destinations)

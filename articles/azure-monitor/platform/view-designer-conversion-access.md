# <a name="view-designer-to-workbooks-conversion-summary-and-access"></a>Просмотр дизайнера в сводке преобразования трудовых книжек и доступе
[Дизайнер представления](view-designer.md) — это функция Azure Monitor, которая позволяет создавать пользовательские представления, чтобы помочь вам визуализировать данные в рабочем пространстве Log Analytics с диаграммами, списками и сроками. Они постепенно сворачиваются и заменяются трудовыми книжеками, которые обеспечивают дополнительную функциональность. В этой статье подробно описано, как можно создать резюме обзора и разрешения, необходимые для доступа к трудовым книгам.

## <a name="creating-your-workspace-summary-from-azure-dashboard"></a>Создание резюме рабочего пространства с панели мониторинга Azure
Пользователи представления-дизайнера могут быть знакомы с наличием плитки обзора для представления набора представлений. Для поддержания визуального обзора, такого как резюме рабочего пространства представления, трудовые книжки предлагают закрепленные шаги, которые можно прикрепить к [панели мониторинга портала Azure.](../../azure-portal/azure-portal-dashboards.md) Так же, как обзор плитки в резюме Workspace, возлагали элементы трудовой книжки будет ссылка непосредственно на вид рабочей книги.

Вы можете воспользоваться преимуществами функций настройки высокого уровня, предоставляемых панелями мониторинга Azure, что позволяет автоматически обновлять, перемещать, размеры и дополнительную фильтрацию для призакрепленных элементов и визуализаций. 

![Панель мониторинга](media/view-designer-conversion-access/dashboard.png)

Создайте новую панель мониторинга Azure или выберите существующую панель мониторинга, чтобы начать закреплять элементы трудовых книжек.

Чтобы закрепить отдельный элемент, вам нужно включить значок контакта для вашего конкретного шага. Для этого выберите соответствующую кнопку **«Отредки»** для шага, а затем выберите значок передачи, чтобы открыть **Расширенные Настройки.** Проверьте возможность **всегда показывать значок штыря на этом шаге,** и значок штыря появится в верхнем правом углу вашего шага. Эта булавка позволяет прикреплять конкретные визуализации к панели мониторинга, например, плитки обзора.

![Шаг штифта](media/view-designer-conversion-access/pin-step.png)


Вы также можете прикрепить несколько визуализаций из рабочей книги или всего содержимого рабочей книги к панели мониторинга. Чтобы закрепить всю трудовую книжку, выберите **edit** в верхней панели инструментов, чтобы переключить **режим edit.** Появится значок контакта, позволяющий либо прикрепить весь элемент Workbook или все отдельные шаги и визуализации в трудовой книжке.

![Прикрепить все](media/view-designer-conversion-access/pin-all.png)



## <a name="sharing-and-viewing-permissions"></a>Разрешение на совместное использование и просмотр 
В справочниках работа можно получить как частный, либо общий документ. По умолчанию сохраненные трудовые книжки будут сохранены в соответствии с **My Reports**, что означает, что только создатель может просматривать эту трудовую книжку.

Вы можете поделиться своими трудовыми книжками, выбрав значок **«Доля»** из верхней панели инструментов в **режиме edit.** Вам будет предложено переместить свою трудовую книжку в **общие отчеты,** которые будут генерировать ссылку, которая обеспечивает прямой доступ к трудовой книжке.

Для того, чтобы пользователь просматривал общую рабочую книгу, он должен иметь доступ как к подписке, так и к группе ресурсов, под которой сохраняется рабочая книга.

![Доступ на основе подписки](media/view-designer-conversion-access/subscription-access.png)

# <a name="workbook-parameters"></a>Параметры рабочей книги

Параметры позволяют авторам книги собирать входные данные от потребителей и ссылаться на него в других частях трудовой книжки - как правило, для области набора результатов или установки правого визуального. Это ключевая возможность, которая позволяет авторам создавать интерактивные отчеты и опыт. 

Рабочие книги позволяют контролировать, как ваши элементы управления параметра представлены потребителям - текстовый ящик против падения, одно- против нескольких выбора, значения из текста, JSON, КЗЛ, или Azure Resource Graph и т.д.  

Поддерживаемые типы параметров включают:
* [Время](workbooks-time.md) - позволяет пользователю выбрать из заселенных временных диапазонов или выбрать пользовательский диапазон
* [Падение вниз](workbooks-dropdowns.md) - позволяет пользователю выбрать из значения или набора значений
* [Текст](workbooks-text.md) - позволяет пользователю вводить произвольный текст
* [Ресурс](workbooks-resources.md) - позволяет пользователю выбрать один или несколько ресурсов Azure
* [Подписка](workbooks-resources.md) - позволяет пользователю выбрать один или несколько ресурсов подписки Azure
* Тип ресурса - позволяет пользователю выбрать одно или несколько значений типа ресурсов Azure
* Местоположение - позволяет пользователю выбрать одно или несколько значений местоположения Azure

Эти значения параметров могут быть отсылки в других частях трудовых книжек либо через привязки или расширения значения.

## <a name="creating-a-parameter"></a>Создание параметра
1. Начните с пустой трудовой книжки в режиме рерит.
2. Выберите _Параметры добавления_ из ссылок в трудовой книжке.
3. Нажмите на синюю кнопку _Добавить параметр._
4. В новом параметре панели, которая всплывает ввести:
    1. Название параметра: `TimeRange` *(обратите внимание, что __имена__ параметров не **могут** включать пробелы или специальные символы)*
    2. Название дисплея: `Time Range` *(однако, имена __отображения__ могут включать пробелы, специальные символы, смайлики и т.д.)*  
    2. Тип параметра:`Time range picker`
    3. Обязательно:`checked`
    4. Доступное время диапазонов: Последний час, Последние 12 часов, Последние 24 часа, Последние 48 часов, Последние 3 дня, Последние 7 дней и Разрешить пользовательский выбор диапазона времени
5. Выберите "Сохранить" из панели инструментов для создания параметра.

   ![Изображение, показывающее создание параметра временного диапазона](./media/workbooks-parameters/time-settings.png)

Так будет выглядеть трудовая книжка в режиме чтения, в стиле "Пилы".

   ![Изображение, показывающее параметр диапазона времени в режиме чтения](./media/workbooks-parameters/parameters-time.png)

## <a name="referencing-a-parameter"></a>Ссылка на параметр
### <a name="via-bindings"></a>Через привязки
1. Добавьте элемент управления запросами в трудовую книгу и выберите ресурс Application Insights.
2. Откройте _диапазон времени_ упасть `Time Range` вниз и выберите вариант из раздела Параметры в нижней части.
3. Это связывает параметр временного диапазона с временной диапазоном диаграммы. Сфера времени запроса образца теперь длится 24 часа.
4. Выполнить запрос, чтобы увидеть результаты

    ![Изображение, показывающее параметр диапазона времени, на который ссылаются через привязки](./media/workbooks-parameters/time-binding.png)

### <a name="in-kql"></a>В КЗЛ
1. Добавьте элемент управления запросами в трудовую книгу и выберите ресурс Application Insights.
2. В КЗЛ введите фильтр области времени, используя параметр:`| where timestamp {TimeRange}`
3. Это расширяет время оценки `| where timestamp > ago(1d)`запроса до, что является значением диапазона времени параметра.
4. Выполнить запрос, чтобы увидеть результаты

    ![Изображение, показывающее временной диапазон, на который ссылаются в КЗЛ](./media/workbooks-parameters/time-in-code.png)

### <a name="in-text"></a>В тексте 
1. Добавьте элемент управления текстом в трудовую книжку.
2. В разметке введите`The chosen time range is {TimeRange:label}`
3. Выбрать _готовое редактирование_
4. Текст управления покажет текст: _Выбранный временной диапазон за последние 24 часа_

## <a name="parameter-options"></a>Параметры:
В разделе _"В тексте"_ `label` вместо его значения используется параметр. Параметры выставляют различные такие параметры в зависимости от его типа - например, сборщики временных диапазонов позволяют значение, метку, запрос, старт, конец и зерно.

Используйте `Previews` раздел панели _параметры edit,_ чтобы увидеть параметры расширения для параметра:

![Изображение, показывающее параметры параметра диапазона времени](./media/workbooks-parameters/time-previews.png)

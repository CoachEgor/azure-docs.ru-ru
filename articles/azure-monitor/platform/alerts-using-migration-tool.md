# <a name="use-the-voluntary-migration-tool-to-migrate-your-classic-alert-rules"></a>Используйте средство произвольную миграцию для переноса классической правилами генерации оповещений

Как [было объявлено ранее](monitoring-classic-retirement.md), классических оповещений в Azure Monitor, прекращается в июля 2019 г. Средство миграции для запуска миграции добровольно доступна на портале Azure и станет доступен для клиентов, использующих классические правила генерации оповещений. В этой статье пошагово продемонстрирует о том, как использовать средство миграции добровольно миграции классической правилами генерации оповещений, до начала автоматического переноса в июля 2019 г.

## <a name="benefits-of-new-alerts"></a>Преимущества новых оповещений

Классические оповещения будут заменены новой унифицированной оповещений в Azure Monitor. Новая платформа оповещений имеет следующие преимущества:

- Оповещения с использованием разнообразных многомерные метрики [многие службы Azure](alerts-metric-near-real-time.md#metrics-and-dimensions-supported)
- Оповещения поддержки новых метрик [правила генерации оповещений для нескольких ресурсов](alerts-metric-overview.md#monitoring-at-scale-using-metric-alerts-in-azure-monitor) которой значительно сократить расходы на управление много правил.
- Механизм единой уведомлений
  - [Группы действий](action-groups.md) — это механизм модульная уведомлений, который работает с всех новых типов предупреждений (метрики, журнала и журнала действий)
  - Вы также сможете воспользоваться преимуществами новых механизмов уведомлений, например SMS, голосовой и соединитель ITSM
- [Единое взаимодействие оповещений](alerts-overview.md) переводит все оповещения разных сигналы (метрик, действий и журнала) в одном месте

## <a name="before-you-migrate"></a>Перед миграцией

В рамках миграции классические правила генерации оповещений, преобразуются в эквивалентные новые правила генерации оповещений и создаются группы действий.

- Формат полезных данных уведомления, а также API-интерфейсы для создания и управления нового правила генерации оповещений отличается по сравнению с классической правил генерации оповещений, так как они поддерживают дополнительные функции. Узнайте, [как подготовиться к переносу](alerts-prepare-migration.md).

- Невозможно перенести некоторые классические правила генерации оповещений с помощью средства. [Узнайте, какие правила, не переносимые и узнать, как перенести их](alerts-understand-migration.md#which-classic-alert-rules-can-be-migrated).

    > [!NOTE]
    > Процесс миграции не влияет на вычисления классические правила генерации оповещений. Они будут продолжать работать и отсылать оповещения, пока они переносятся и приступайте нового правила генерации оповещений.


## <a name="how-to-use-the-migration-tool"></a>Как использовать средство миграции

Следующая процедура описывает способы запуска миграции классической правилами генерации оповещений на портале Azure:

1. На [портале Azure](https://portal.azure.com) выберите **Монитор**.

2. Нажмите кнопку **оповещения** щелкните **Управление правилами генерации оповещений** или **просмотреть классические оповещения**.

3. Нажмите кнопку **переход на новые правила** для перейдите на главную страницу миграции. Эта страница отображается список всех подписок и состояние переноса для них.

    ![Целевая миграции](media/alerts-migration/migration-landing.png "перенесите правила")

4. Все подписки, которые можно перенести с помощью средства будут помечены как **готовы к миграции**.

    > [!NOTE]
    > Средство миграции станет доступен на этапах для всех подписок, использующих классические правила генерации оповещений. В ранних этапах развертывания можно увидеть некоторые подписки как не готовый для миграции.

5. Выберите одну или несколько подписок и щелкните **Предварительный просмотр миграции**

6. На этой странице можно просмотреть сведения классической правил генерации оповещений, которые будут перенесены для одной подписки за раз. Вы также можете **загрузить сведения о миграции для этой подписки** в формате CSV.

    ![Предварительная версия Migration](media/alerts-migration/migration-preview.png "Предварительный просмотр миграции")

7. Укажите один или несколько **адреса электронной почты** чтобы получать уведомления о состоянии миграции. При завершении миграции или от вас требуется действие будет отправлено сообщение электронной почты.

8. Щелкните **начать миграцию**. Считывать сведения, отображаемые в диалоговом окне подтверждения и подтверждения, если вы готовы начать процесс миграции.

    >[!IMPORTANT]
    > После запуска процесс миграции для подписки, вы не сможете изменения или создания классические правила генерации оповещений для подписки. Однако классические правила генерации оповещений будет под управлением и предоставляют соответствующие уведомления, пока они не будут перенесены. Это позволяет обеспечить единство классические правила генерации оповещений и новых правил, созданных во время миграции. По завершении миграции для вашей подписки, можно не использовать классические правила генерации оповещений больше.

    ![Подтверждение миграции](media/alerts-migration/migration-confirm.png "подтвердить запуск миграции")

9. Как мы завершения миграции, или требуется действий со стороны пользователя, вы получите сообщение электронной почты на адреса электронной почты, указанных на шаге 8. Можно также периодически проверять состояние на целевой странице миграции на портале.

## <a name="frequently-asked-questions"></a>Часто задаваемые вопросы

### <a name="why-is-my-subscriptions-listed-as-not-ready-for-migration"></a>**Почему Мои подписки отображается как не готовы к миграции?**

Средство миграции станет доступен на этапах для всех клиентов. В ранних этапах, большинство или все ваши подписки может быть помечен как **не готов к миграции**. Однако до середины апреля, все подписки должны быть готовы к миграции.

Когда подписка будет готова для миграции, владельцев подписки будут получать сообщение электронной почты, уведомлением о доступности средства. Обратите внимание для этого уведомления.

### <a name="who-can-trigger-the-migration"></a>**Кто может активировать миграции?**

Пользователи, которым назначена роль Monitoring Contributor, назначенные им на уровне подписки, смогут для запуска миграции. Дополнительные сведения о [Role Based Access Control для процесса миграции](alerts-understand-migration.md#who-can-trigger-the-migration).

### <a name="how-long-is-the-migration-going-to-take"></a>**Продолжительность миграции будет принимать?**

Для большинства подписок миграции обычно завершается в течение часа. Вы можете отслеживания хода выполнения миграции на целевой странице миграции.  В течение этого времени можно обеспечить за что оповещения по-прежнему выполняются в системе классических оповещений или новой.

### <a name="what-can-i-do-if-i-run-into-an-issue-during-migration"></a>**Что делать, если у меня возникли проблемы во время миграции?**

Следуйте [Устранение неполадок в руководстве по устранению проблем, могут возникнуть во время миграции](alerts-understand-migration.md#common-issues-and-remediations). Если какие-либо действия требуются для завершения миграции, вы получите уведомление на адреса электронной почты, указанный во время миграции.

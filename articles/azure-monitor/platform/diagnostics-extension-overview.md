# <a name="azure-diagnostics-extension-overview"></a>Общие сведения о расширении система диагностики Azure
Расширение система диагностики Azure — это [агент в Azure Monitor](agents-overview.md) , собирающий данные мониторинга из операционной системы на виртуальной машине для ресурсов вычислений Azure, включая виртуальные машины. В этой статье представлен обзор расширения система диагностики Azure, включая конкретные функциональные возможности, которые он поддерживает, и параметры установки и настройки. 

> [!NOTE]
> Расширение система диагностики Azure — это один из агентов, доступных для получения данных мониторинга из операционной системы на виртуальной машине для вычислений ресурсов. Описание различных агентов и руководство по выбору соответствующих агентов для ваших требований см. в разделе [Общие сведения об агентах Azure Monitor](agents-overview.md) .

## <a name="comparison-to-log-analytics-agent"></a>Сравнение с агентом Log Analytics
Агент Log Analytics в Azure Monitor может также использоваться для получения данных мониторинга из операционной системы виртуальной машины. Вы можете использовать один или оба варианта в зависимости от требований. Подробное сравнение агентов Azure Monitor см. [в разделе Общие сведения об агентах Azure Monitor](agents-overview.md) . 

Ниже приведены основные различия, которые следует учитывать.

- Расширение система диагностики Azure можно использовать только с виртуальными машинами Azure. Агент Log Analytics можно использовать с виртуальными машинами в Azure, в других облаках и в локальной среде.
- Расширение система диагностики Azure отправляет данные в службу хранилища Azure, [Azure Monitor метрики](data-platform-metrics.md) (только для Windows) и концентраторы событий. Агент Log Analytics собирает данные в [журналы Azure Monitor](data-platform-logs.md).
- Агент Log Analytics необходим для [решений](../monitor-reference.md#insights-and-core-solutions), [Azure Monitor для виртуальных машин](../insights/vminsights-overview.md)и других служб, таких как [Центр безопасности Azure](/azure/security-center/).

## <a name="costs"></a>Затраты
Плата за расширение диагностики Azure не взимается, но вы можете взимать плату за полученные данные. Проверьте [Azure Monitor цены](https://azure.microsoft.com/pricing/details/monitor/) на место назначения, в котором будут собираться данные.

## <a name="data-collected"></a>Собираемые данные
В следующих таблицах перечислены данные, которые можно собирать с помощью расширения системы диагностики Windows и Linux.

### <a name="windows-diagnostics-extension-wad"></a>Расширение диагностики Windows (WAD)

| Источник данных | Описание |
| --- | --- |
| Журналы событий Windows   | События из журнала событий Windows. |
| Счетчики производительности | Числовые значения измеряют производительность различных аспектов операционной системы и рабочих нагрузок. |
| Журналы IIS             | Сведения об использовании веб-сайтов IIS, работающих в гостевой операционной системе. |
| Журналы приложений     | Сообщения трассировки, написанные вашим приложением. |
| Журналы .NET EventSource |События записи кода с использованием класса .NET [EventSource](https://msdn.microsoft.com/library/system.diagnostics.tracing.eventsource.aspx) |
| [Журналы трассировки событий Windows на основе манифеста](https://docs.microsoft.com/windows/desktop/etw/about-event-tracing) |Трассировка событий для событий Windows, создаваемых любым процессом. |
| Аварийные дампы (журналы)   | Сведения о состоянии процесса в случае сбоя приложения. |
| Журналы на основе файлов    | Журналы, созданные вашим приложением или службой. |
| Журналы диагностики агента | Сведения о система диагностики Azure. |


### <a name="linux-diagnostics-extension-lad"></a>Расширение системы диагностики Linux (LAD)

| Источник данных | Описание |
| --- | --- |
| Системный журнал | События, отправляемые в систему ведения журнала событий Linux.   |
| Счетчики производительности  | Числовые значения измеряют производительность различных аспектов операционной системы и рабочих нагрузок. |
| Файлы журналов | Записи, отправляемые в журнал на основе файлов.  |

## <a name="data-destinations"></a>Целевое расположение данных
Расширение системы диагностики Azure для Windows и Linux всегда соберет данные в учетную запись хранения Azure. См. статью [Установка и настройка расширения системы диагностики Microsoft Azure (WAD)](diagnostics-extension-windows-install.md) и [Использование диагностического расширения Linux для мониторинга метрик и журналов](../../virtual-machines/extensions/diagnostics-linux.md) для получения списка конкретных таблиц и больших двоичных объектов, в которых собираются эти данные.

Настройте один или несколько *приемников данных* для отправки данных в другие дополнительные назначения. В следующих разделах перечислены приемники, доступные для расширения системы диагностики Windows и Linux.

### <a name="windows-diagnostics-extension-wad"></a>Расширение диагностики Windows (WAD)

| Назначение | Описание |
|:---|:---|
| Метрики Azure Monitor | Собирайте данные производительности в метрики Azure Monitor. См. раздел [Отправка метрик гостевой ОС в базу данных метрик Azure Monitor](collect-custom-metrics-guestos-resource-manager-vm.md).  |
| Центры событий | Используйте концентраторы событий Azure для отправки данных за пределы Azure. См. статью [потоковая передача данных система диагностики Azure в концентраторы событий](diagnostics-extension-stream-event-hubs.md) . |
| Большие двоичные объекты службы хранилища Azure | Запись данных в большие двоичные объекты в службе хранилища Azure в дополнение к таблицам. |
| Application Insights | Собирайте данные из приложений, выполняющихся на виртуальной машине, чтобы Application Insights для интеграции с другим наблюдением за приложениями. См. раздел [Отправка диагностических данных в Application Insights](diagnostics-extension-to-application-insights.md). |

Вы также можете получать данные WAD из хранилища в рабочую область Log Analytics, чтобы анализировать их с помощью Azure Monitor журналов, хотя для этой функции обычно используется агент Log Analytics. Он может передавать данные непосредственно в Log Analytics рабочую область и поддерживает решения и аналитические сведения, которые предоставляют дополнительные функциональные возможности.  См. раздел [Сбор журналов диагностики Azure из службы хранилища Azure](diagnostics-extension-logs.md). 


### <a name="linux-diagnostics-extension-lad"></a>Расширение системы диагностики Linux (LAD)
LAD записывает данные в таблицы в службе хранилища Azure. Он поддерживает приемники, приведенные в следующей таблице.

| Назначение | Описание |
|:---|:---|
| Центры событий | Используйте концентраторы событий Azure для отправки данных за пределы Azure. |
| Большие двоичные объекты службы хранилища Azure | Запись данных в большие двоичные объекты в службе хранилища Azure в дополнение к таблицам. |
| Метрики Azure Monitor | Установите агент Telegraf в дополнение к LAD. См. раздел [Получение пользовательских метрик для виртуальной машины Linux с помощью агента Инфлуксдата Telegraf](collect-custom-metrics-linux-telegraf.md).


## <a name="installation-and-configuration"></a>Установка и настройка
Диагностическое расширение реализовано как [расширение виртуальной машины](/virtual-machines/extensions/overview) в Azure, поэтому оно поддерживает те же параметры установки с помощью шаблонов диспетчер ресурсов, POWERSHELL и CLI. Общие сведения об установке и обслуживании расширений виртуальных машин см. в статье [расширения и компоненты виртуальных](/virtual-machines/extensions/features-windows) машин для Windows и [компонентов для Linux](/virtual-machines/extensions/features-linux) .

Вы также можете установить и настроить диагностическое расширение Windows и Linux в портал Azure в разделе **параметры диагностики** раздела **мониторинг** в меню виртуальной машины.

Дополнительные сведения об установке и настройке расширения системы диагностики для Windows и Linux см. в следующих статьях.

- [Установка и настройка расширения системы диагностики Microsoft Azure (WAD)](diagnostics-extension-windows-install.md)
- [Использование диагностического расширения Linux для мониторинга метрик и журналов](../../virtual-machines/extensions/diagnostics-linux.md)

# <a name="what-is-monitored-by-azure-monitor"></a>Что отслеживает Azure Monitor?
В этой статье описаны различные приложения и службы, которые контролируются Azure Monitor. 

## <a name="insights-and-core-solutions"></a>Исследования и основные решения
Основные идеи и решения считаются частью Azure Monitor и следуют соглашениям об уровне поддержки и обслуживания для Azure. Они поддерживаются во всех регионах Azure, где доступен Azure Monitor.

### <a name="insights"></a>Аналитика

Исследования предоставляют индивидуальный опыт мониторинга для конкретных приложений и служб. Они собирают и анализируют как журналы, так и метрики.

| Аналитические сведения | Описание |
|:---|:---|
| [Application Insights](app/app-insights-overview.md) | Расширяемая служба управления производительностью приложений (APM) для мониторинга веб-приложений в реальном времени на любой платформе. |
| [Azure Monitor для контейнеров](insights/container-insights-overview.md) | Мониторинг производительности контейнерных нагрузок, развернутых либо в azure Container Instances, либо в управляемых кластерах Kubernetes, размещенных в службе Azure Kubernetes Service (AKS). |
| [Azure Монитор для Космосd (предварительный просмотр)](insights/cosmosdb-insights-overview.md) | Обеспечивает представление общей производительности, сбоев, емкости и работоспособности всех ресурсов Azure Cosmos DB в едином интерактивном интерфейсе. |
| [Azure Monitor для сетей (предварительный просмотр)](insights/network-insights-overview.md) | Обеспечивает полное представление о работоспособности и метриках для всего вашего сетевого ресурса. Расширенная возможность поиска помогает определить зависимости ресурсов, позволяя такие сценарии, как определение ресурса, который размещает ваш веб-сайт, просто ища имя вашего сайта. |
[Монитор Azure для групп ресурсов (предварительный просмотр)](insights/resource-group-insights.md) |  Изыскивать и диагностировать любые проблемы, с которыми сталкиваются ваши индивидуальные ресурсы, предлагая при этом контекст работоспособности и производительности группы ресурсов в целом. |
| [Монитор Azure для хранения (предварительный просмотр)](insights/storage-insights-overview.md) | Обеспечивает всесторонний мониторинг учетных записей хранилища Azure, предоставляя единое представление об производительности, емкости и доступности служб хранения Azure. |
| [Azure Monitor для vMs](insights/container-insights-overview.md) | Контролирует ваши виртуальные машины Azure (VM) и виртуальные наборы машин в масштабе. Это решение анализирует производительность и работоспособность виртуальных машин Windows и Linux, отслеживает их процессы и зависимости на других ресурсах, а также внешние процессы. |

### <a name="core-solutions"></a>Основные решения

Решения основаны на запросах журналов и представлениях, настраиваемых для конкретного приложения или службы. Они собирают и анализируют журналы только и в настоящее время deprecated с течением времени в пользу идеи.

| Решение | Описание |
|:---|:---|
| [Здоровье агента](insights/solution-agenthealth.md) | Проанализируйте работоспособность и конфигурацию агентов Log Analytics. |
| [Управление оповещениями](platform/alert-management-solution.md) | Проанализируйте оповещения, собранные менеджером по операциям Системного центра, Nagios или zabbix. |
| [Сервисная карта](insights/service-map.md) | Автоматически обнаруживает компоненты приложений в системах Windows и Linux и отображает связь между службами. |

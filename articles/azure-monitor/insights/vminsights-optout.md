# <a name="disable-monitoring-of-your-vms-in-azure-monitor-for-vms-preview"></a>Отключить мониторинг виртуальных машин в Azure Monitor для виртуальных машин (Предварительная версия)

После включения мониторинга виртуальных машин (ВМ), позже вы можете отключить мониторинг в Azure Monitor для виртуальных машин. В этой статье показано, как отключить мониторинг для одного или нескольких виртуальных машин.  

В настоящее время Azure Monitor для виртуальных машин не поддерживает выборочную отключение мониторинга виртуальной Машины. Рабочую область Log Analytics может поддерживать Azure Monitor для виртуальных машин и других решений. Он также может собирать другие данные наблюдения. Если рабочей области Log Analytics предоставляет эти службы, необходимо понимать влияние и методы отключение мониторинга, прежде чем начать.

Для реализации возможностей Azure Monitor для виртуальных машин нужны следующие компоненты:

* Рабочая область Log Analytics, где хранятся данные мониторинга из виртуальных машин и других источников.
* Набор счетчиков производительности, которые настроены в рабочей области. Коллекции обновляет конфигурацию наблюдения для всех виртуальных машин, подключенных к рабочей области.
* `InfrastructureInsights` и `ServiceMap`, который ведется наблюдение решениям в рабочей области. Эти решения обновить конфигурацию мониторинга на всех виртуальных машинах, подключенных к рабочей области.
* `MicrosoftMonitoringAgent` и `DependencyAgent`, которые являются расширениями виртуальной Машины Azure. Эти расширения собирать и отправлять данные в рабочую область.

При подготовке отключить мониторинг виртуальных машин, учитывайте следующие аспекты:

* Если Вы оценили с одной виртуальной Машины и использовать рабочую область Log Analytics предварительно выбранные по умолчанию, можно отключить, мониторинг, удаление агента зависимостей из виртуальной Машины и отключение агента Log Analytics из этой рабочей области. Этот подход используется, если вы планируете использовать виртуальную Машину для других целей и решить позже, чтобы заново установить соединение с другой рабочей областью.
* Если вы выбрали ранее существовавших рабочей области Log Analytics, которая поддерживает другие решения для мониторинга и сбора данных из других источников, не прерывая работу, не снижая рабочей области можно удалить компоненты решения из рабочей области.  

>[!NOTE]
> После удаления компонентов решения из рабочей области, вы может по-прежнему отображается состояние работоспособности с виртуальных машин Azure; в частности вы см. в разделе производительности и сопоставить данные при переходе в любом представлении, на портале. Данных перестанет отображаться в **производительности** и **карты** представления. Но **работоспособности** представления будут отображать состояние работоспособности для виртуальных машин. **Попробуйте** параметр будет доступен из выбранной виртуальной Машины Azure, поэтому вы можете повторно включить наблюдение в будущем.  

## <a name="remove-azure-monitor-for-vms-completely"></a>Полностью удалить Azure Monitor для виртуальных машин

Если вам нужна рабочая область Log Analytics, выполните следующие действия, чтобы полностью удалить Azure Monitor для виртуальных машин. Мы удалим `InfrastructureInsights` и `ServiceMap` решения из рабочей области.  

>[!NOTE]
>При использовании схемы услуги, решением для мониторинга, прежде чем вы включили Azure Monitor для виртуальных машин и по-прежнему полагаться на него, не удаляйте это решение, как описано на предыдущем шаге следующей процедуры.  
>

1. Войдите на [портале Azure](https://portal.azure.com).
2. На портале Azure щелкните **Все службы**. В списке ресурсов введите **Log Analytics**. Как только вы начнете вводить, список фильтруется по предложения на основе ввода. Выберите **Log Analytics**.
3. В списке рабочих областей Log Analytics, выберите рабочую область вы выбрали при включении Azure Monitor для виртуальных машин.
4. С левой стороны экрана выберите **решения**.  
5. В списке решений, выберите **InfrastructureInsights (имя рабочей области)** . На **Обзор** страницы для решения, щелкните **удалить**. При появлении запроса на подтверждение, выберите **Да**.  
6. В списке решений, выберите **ServiceMap (имя рабочей области)** . На **Обзор** страницы для решения, щелкните **удалить**. При появлении запроса на подтверждение, выберите **Да**.  

Перед включением Azure Monitor для виртуальных машин, если это еще не [сбора данных счетчиков производительности](vminsights-enable-overview.md#performance-counters-enabled) для виртуальных машин на базе Windows или Linux в рабочей области, [отключить эти правила](../platform/data-sources-performance-counters.md#configuring-performance-counters) для Windows и Linux.

## <a name="disable-monitoring-and-keep-the-workspace"></a>Отключение мониторинга и рабочая область  

Если рабочей области Log Analytics по-прежнему необходимо для поддержки мониторинга из других источников, выполнив следующие действия, чтобы отключить мониторинг на виртуальной Машине, которая использовалась для оценки Azure Monitor для виртуальных машин. Для виртуальных машин Azure то будут удалены агент зависимостей расширения виртуальной Машины и расширения виртуальной Машины агента Log Analytics для Windows или Linux непосредственно из виртуальной Машины. 

>[!NOTE]
>Не удалять агент Log Analytics, если: 
>
> * Служба автоматизации Azure управляет выполнять оркестрацию процессов, или для управления конфигурации или обновления виртуальной Машины. 
> * Центр безопасности Azure управляет виртуальной Машины для обеспечения безопасности и обнаружения угроз. 
>
> Если вы удалите агент Log Analytics, вы будет препятствовать этими службами и решениями упреждающему управлению виртуальной Машины. 

1. Войдите на [портале Azure](https://portal.azure.com). 
2. На портале Azure выберите **Виртуальные машины**. 
3. Выберите виртуальную машину из списка. 
4. С левой стороны экрана выберите **расширения**. На **расширения** выберите **DependencyAgent**.
5. На странице свойств расширения, выберите **удаления**.
6. На **расширения** выберите **MicrosoftMonitoringAgent**. На странице свойств расширения, выберите **удаления**.

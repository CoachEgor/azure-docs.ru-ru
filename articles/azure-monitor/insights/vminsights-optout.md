# <a name="disable-monitoring-of-your-vms-in-azure-monitor-for-vms"></a>Отработка мониторинга ваших ВМ в Azure Monitor для VMs

После включения мониторинга виртуальных машин (Виртуальных ММ) можно отключить мониторинг в Azure Monitor для виртуальных технологий. В этой статье показано, как отключить мониторинг для одного или нескольких VMs.  

В настоящее время Azure Monitor для VM не поддерживает выборочное отключение мониторинга VM. Рабочее пространство журналов Analytics может поддерживать Azure Monitor для вс-ми и других решений. Он может также собирать другие данные мониторинга. Если рабочее пространство Log Analytics предоставляет эти услуги, необходимо понять эффект и методы отключения мониторинга перед началом работы.

Для реализации возможностей Azure Monitor для виртуальных машин нужны следующие компоненты:

* Рабочее пространство Log Analytics, в котором хранятся данные мониторинга с вс-маи и других источников.
* Коллекция счетчиков производительности, настроенная в рабочем пространстве. Коллекция обновляет конфигурацию мониторинга на всех ВМ, подключенных к рабочему пространству.
* `VMInsights`, который представляет собой решение мониторинга, настроенное в рабочей области. Это решение обновляет конфигурацию мониторинга на всех вс-данах, подключенных к рабочему пространству.
* `MicrosoftMonitoringAgent`и `DependencyAgent`, которые являются расширениями Azure VM. Эти расширения собирают и отправляют данные в рабочее пространство.

Готовясь отключить мониторинг ваших вс-ацидов, имейте в виду следующие соображения:

* Если вы оценили с помощью одного VM и использовали заранее выбранное рабочее пространство Log Analytics по умолчанию, можно отключить мониторинг, удалив агент зависимости из VM и отключив агент Log Analytics от этого рабочего пространства. Этот подход уместны, если вы собираетесь использовать VM для других целей и решили позже восстановить его в другом рабочем пространстве.
* Если вы выбрали уже существующее рабочее пространство Log Analytics, которое поддерживает другие решения мониторинга и сбор данных из других источников, можно удалить компоненты решения из рабочего пространства, не прерывая и не влияя на рабочее пространство.  

>[!NOTE]
> После удаления компонентов решения из рабочего пространства можно продолжить поиск данных о производительности и картирования для вс-карт Azure. Данные в конечном итоге перестанут отображаться в представлениях **«Производительность»** и **«Карта».** Опция **Enable** будет доступна в выбранном Azure VM, чтобы в будущем можно было повторно включить мониторинг.  

## <a name="remove-azure-monitor-for-vms-completely"></a>Удалить монитор Azure для VMs полностью

Если вам все еще нужно рабочее пространство для анализа журналов, выполните следующие действия, чтобы полностью удалить Azure Monitor для vMs. Вы удалите `VMInsights` решение из рабочей области.  

1. Войдите на [портал Azure](https://portal.azure.com).
2. На портале Azure выберите **все сервисы.** В списке ресурсов введите **log Analytics**. При начале ввода список фильтрует предложения, основанные на входе. Выберите **аналитику журнала**.
3. В списке рабочих областей анализа журналов выберите выбранное рабочее место при включении Azure Monitor для вс-ми.
4. Слева выберите **Решения.**  
5. В списке решений выберите **VMInsights (имя рабочего пространства).** На странице **Обзора** для решения выберите **Удалить**. Когда попрошайке, чтобы подтвердить, выберите **Да**.

## <a name="disable-monitoring-and-keep-the-workspace"></a>Отключить мониторинг и сохранить рабочее пространство  

Если рабочее пространство log Analytics по-прежнему нуждается в поддержке мониторинга из других источников, после этих шагов, чтобы отключить мониторинг на VM, который вы использовали для оценки Azure Monitor для VMs. Для VMs-м иссоциировать сяодор ы агента зависимости VM и расширение VM-агента Log Analytics для Windows или Linux непосредственно из VM. 

>[!NOTE]
>Не удаляйте агент Log Analytics, если: 
>
> * Azure Automation управляет VM для организации процессов или управления конфигурацией или обновлениями. 
> * Центр безопасности Azure управляет VM для обнаружения угроз. 
>
> Если вы удалите агент Log Analytics, вы предотвратите упреждающее управление VM службами и решениями. 

1. Войдите на [портал Azure](https://portal.azure.com). 
2. На портале Azure выберите **Виртуальные машины**. 
3. Выберите виртуальную машину из списка. 
4. Слева выберите **расширения.** На странице **расширения** выберите **DependencyAgent**.
5. На странице свойств расширения выберите **Uninstall**.
6. На странице **расширения** выберите **MicrosoftMonitoringAgent**. На странице свойств расширения выберите **Uninstall**.

# <a name="enable-azure-monitor-for-vms-preview-for-evaluation"></a>Включить монитор Azure для виртуальных машин (Предварительная версия) для оценки

Чтобы оценить Azure Monitor для виртуальных машин (Предварительная версия) на небольшом количестве виртуальных машин Azure, одной виртуальной машины или масштабируемого набора виртуальных машин, самый простой и самый прямой подход, чтобы включить мониторинг — на портале Azure. В конце этого процесса, будет успешно начала наблюдения за ними и узнайте, если они возникли проблемы производительности или доступности. 

Прежде чем Приступая к работе, обязательно просмотрите [предварительные требования](vminsights-enable-overview.md) и убедитесь, ваша подписка и ресурсы отвечают требованиям.  

## <a name="enable-monitoring-for-a-single-azure-vm"></a>Включить мониторинг для одной виртуальной Машине Azure
Чтобы включить мониторинг виртуальной машины на портале Azure, сделайте следующее.

1. Войдите на [портале Azure](https://portal.azure.com).

1. Выберите **Виртуальные машины**.

1. Выберите виртуальную машину из списка.

1. На странице виртуальной машины в разделе **Мониторинг** выберите **Аналитические сведения (предварительная версия)**.

1. На странице **Аналитические сведения (предварительная версия)** выберите **Попробовать**.

    ![Включение Azure Monitor для виртуальной машины](./media/vminsights-enable-single-vm/enable-vminsights-vm-portal-01.png)

1. Если имеющаяся рабочая область Log Analytics расположена в той же подписке, что и кластер, выберите ее из раскрывающегося списка на странице **Azure Monitor Insights Onboarding** (Подключение к службе аналитических сведений Azure Monitor).  

    В списке предварительно выбрана рабочая область по умолчанию и расположение, в которое развертывается виртуальная машина в подписке. 

    >[!NOTE]
    >Если вы хотите создать новую рабочую область Log Analytics для хранения данных мониторинга из виртуальной Машины, следуйте инструкциям в [создать рабочую область Log Analytics](../../azure-monitor/learn/quick-create-workspace.md) в одном из поддерживаемых регионов в списке [здесь](vminsights-enable-overview.md#log-analytics).

После включения мониторинга может потребоваться около 10 минут, прежде чем вы сможете просмотреть метрики работоспособности для виртуальной машины.

![Включение Azure Monitor для обработки развертывания мониторинга виртуальных машин](./media/vminsights-enable-single-vm/onboard-vminsights-vm-portal-status.png)

## <a name="enable-monitoring-for-a-single-virtual-machine-scale-set"></a>Включить мониторинг для одной виртуальной машины масштабируемого набора

Чтобы включить мониторинг виртуальной машины Azure масштабируемого набора на портале Azure, сделайте следующее:

1. Войдите на [портале Azure](https://portal.azure.com).

2. Выберите **масштабируемые наборы виртуальных машин**.

3. В списке выберите масштабируемый набор виртуальных машин.

4. На виртуальной машине масштабируемого набора странице, в **мониторинг** выберите **Insights (Предварительная версия)**.

5. На **Insights (Предварительная версия)** странице, при наличии существующую рабочую область Log Analytics вы хотите использовать, выберите его в раскрывающемся списке.

    В списке предварительно выбрана рабочая область по умолчанию и расположение, в которое развертывается виртуальная машина в подписке. 

    ![Включение Azure Monitor для виртуальных машин для масштабируемого набора виртуальных машин](./media/vminsights-enable-single-vm/enable-vminsights-vmss-portal-01.png)

    >[!NOTE]
    >Если вы хотите создать новую рабочую область Log Analytics для хранения данных мониторинга из виртуальной Машины, следуйте инструкциям в [создать рабочую область Log Analytics](../learn/quick-create-workspace.md) в одном из поддерживаемых регионов в списке [здесь](vminsights-enable-overview.md#log-analytics).

После включения мониторинга, может занять около 10 минут, прежде чем можно просмотреть данные наблюдения для масштабируемого набора.

>[!NOTE]
>Если вы используете ручного обновления модели масштабируемого набора необходимо будет обновить экземпляры, чтобы завершить настройку.  Это можно сделать на странице экземпляров в разделе **параметры** раздел.

![Включение Azure Monitor для обработки развертывания мониторинга виртуальных машин](./media/vminsights-enable-single-vm/onboard-vminsights-vmss-portal-status-01.png)

# <a name="enable-azure-monitor-for-vms-in-the-azure-portal"></a>Включить Azure Monitor для вс-мина на портале Azure

В этой статье описывается, как включить Azure Monitor для виртуальных технологий на небольшом количестве виртуальных машин Azure (VM) с помощью портала Azure. Ваша цель состоит в том, чтобы контролировать ваши VMs и обнаружить любые проблемы производительности или доступности. 

Прежде чем начать, просмотрите [предпосылки](vminsights-enable-overview.md) и убедитесь, что ваша подписка и ресурсы соответствуют требованиям.  

## <a name="enable-monitoring-for-a-single-azure-vm"></a>Включить мониторинг для одного Azure VM
Для обеспечения мониторинга вашего Azure VM:

1. Войдите на [портал Azure](https://portal.azure.com).

1. Выберите **Виртуальные машины**.

1. Выберите виртуальную машину из списка.

1. На странице VM, в разделе **Мониторинг,** выберите **Исследования,** а затем **включить**.

    ![Включение Azure Monitor для виртуальной машины](media/vminsights-enable-single-vm/enable-vminsights-vm-portal.png)

1. Если имеющаяся рабочая область Log Analytics расположена в той же подписке, что и кластер, выберите ее из раскрывающегося списка на странице **Azure Monitor Insights Onboarding** (Подключение к службе аналитических сведений Azure Monitor).  

    В списке предварительно выбрана рабочая область по умолчанию и место развертывания виртуальной машины в подписке. 

    >[!NOTE]
    >Для создания нового рабочего пространства Log Analytics для хранения данных мониторинга из ВМ [см.](../../azure-monitor/learn/quick-create-workspace.md) Ваша рабочая область Log Analytics должна находиться в одном из [поддерживаемых регионов](vminsights-enable-overview.md#log-analytics).

6. Вы будете получать сообщения о состоянии по мере выполнения конфигурации.

    ![Включение Azure Monitor для обработки развертывания мониторинга виртуальных машин](media/vminsights-enable-single-vm/onboard-vminsights-vm-portal-status.png)

## <a name="enable-monitoring-for-a-single-virtual-machine-scale-set"></a>Включить мониторинг для одного виртуального набора масштабов машины

Для обеспечения мониторинга виртуальной шкалы масштабов azure:

1. Войдите на [портал Azure](https://portal.azure.com).

2. Выберите **наборы виртуальной шкалы машин.**

3. Из списка выберите набор виртуальной шкалы машин.

4. На странице набора виртуальной шкалы машины, в разделе **Мониторинг,** выберите **Исследования,** а затем **включить.**

5. На странице **Insights,** если вы хотите использовать существующее рабочее пространство Log Analytics, выберите его в списке выпадающих.

    Список предварительно выбирает рабочее пространство по умолчанию и местоположение, развернутое VM в подписке. 

    ![Включить Azure Monitor для виртуальных машин для набора виртуальной шкалы машин](media/vminsights-enable-single-vm/enable-vminsights-vmss-portal.png)

    >[!NOTE]
    >Для создания нового рабочего пространства Log Analytics для хранения данных мониторинга из набора виртуальной шкалы машин [см.](../learn/quick-create-workspace.md) Ваша рабочая область Log Analytics должна находиться в одном из [поддерживаемых регионов](vminsights-enable-overview.md#log-analytics).

6. Вы будете получать сообщения о состоянии по мере выполнения конфигурации.

    >[!NOTE]
    >Если для набора масштабов используется модель ручного обновления, обновите экземпляры для завершения настройки. Вы можете начать обновления со **страницы Instances** в разделе **Настройки.**
    
    ![Включение Azure Monitor для обработки развертывания мониторинга виртуальных машин](media/vminsights-enable-single-vm/onboard-vminsights-vmss-portal-status.png)

Теперь, когда вы включили мониторинг для набора VM или виртуальной машины, информация о мониторинге доступна для анализа в Azure Monitor для виртуальных технологий.

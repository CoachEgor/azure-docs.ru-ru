# <a name="enable-azure-monitor-for-vms-preview-for-evaluation"></a>Включить монитор Azure для виртуальных машин (Предварительная версия) для оценки

Можно оценить Azure Monitor для виртуальных машин (Предварительная версия) на небольшое количество виртуальных машин (ВМ) Azure или на одной шкале виртуальной Машины или виртуальной машины. Самый простой и самый простой способ включить мониторинг — на портале Azure. Задача — мониторинг виртуальных машин и обнаруживать любые проблемы с производительностью или доступности. 

Перед началом работы ознакомьтесь с [предварительные требования](vminsights-enable-overview.md) и убедитесь, что подписки, а чтобы ресурсы соответствовали требованиям.  

## <a name="enable-monitoring-for-a-single-azure-vm"></a>Включить мониторинг для одной виртуальной Машине Azure
Чтобы включить мониторинг для виртуальной Машины Azure:

1. Войдите на [портале Azure](https://portal.azure.com).

1. Выберите **Виртуальные машины**.

1. Выберите виртуальную машину из списка.

1. На странице виртуальной машины в разделе **Мониторинг** выберите **Аналитические сведения (предварительная версия)** .

1. На странице **Аналитические сведения (предварительная версия)** выберите **Попробовать**.

    ![Включение Azure Monitor для виртуальной машины](./media/vminsights-enable-single-vm/enable-vminsights-vm-portal-01.png)

1. Если имеющаяся рабочая область Log Analytics расположена в той же подписке, что и кластер, выберите ее из раскрывающегося списка на странице **Azure Monitor Insights Onboarding** (Подключение к службе аналитических сведений Azure Monitor).  

    Список заранее выбирается рабочую область по умолчанию и расположение, где развернута виртуальная машина в подписке. 

    >[!NOTE]
    >Чтобы создать новую рабочую область Log Analytics для хранения данных мониторинга из виртуальной Машины, см. в разделе [создать рабочую область Log Analytics](../../azure-monitor/learn/quick-create-workspace.md). Рабочую область Log Analytics должны принадлежать к одному из [поддерживаемые регионы](vminsights-enable-overview.md#log-analytics).

После включения мониторинга, может потребоваться подождать около 10 минут, прежде чем вы можете просмотреть метрики работоспособности для виртуальной Машины.

![Включение Azure Monitor для обработки развертывания мониторинга виртуальных машин](./media/vminsights-enable-single-vm/onboard-vminsights-vm-portal-status.png)

## <a name="enable-monitoring-for-a-single-virtual-machine-scale-set"></a>Включить мониторинг для одной виртуальной машины масштабируемого набора

Чтобы включить мониторинг масштабируемого набора виртуальных машин Azure:

1. Войдите на [портале Azure](https://portal.azure.com).

2. Выберите **масштабируемые наборы виртуальных машин**.

3. В списке выберите масштабируемый набор виртуальных машин.

4. На виртуальной машине масштабируемого набора странице, в **мониторинг** выберите **Insights (Предварительная версия)** .

5. На **Insights (Предварительная версия)** странице, если вы хотите использовать существующую рабочую область Log Analytics, выберите его в раскрывающемся списке.

    Список заранее выбирается рабочую область по умолчанию и расположение, которое можно развернуть виртуальную Машину в подписке. 

    ![Включение Azure Monitor для виртуальных машин для масштабируемого набора виртуальных машин](./media/vminsights-enable-single-vm/enable-vminsights-vmss-portal-01.png)

    >[!NOTE]
    >Чтобы создать новую рабочую область Log Analytics для хранения данных мониторинга из масштабируемого набора виртуальных машин, см. в разделе [создать рабочую область Log Analytics](../learn/quick-create-workspace.md). Рабочую область Log Analytics должны принадлежать к одному из [поддерживаемые регионы](vminsights-enable-overview.md#log-analytics).

После включения мониторинга, может потребоваться подождать около 10 минут, прежде чем можно просмотреть данные наблюдения для масштабируемого набора.

>[!NOTE]
>Если вы используете ручного обновления модели масштабируемого набора, обновите экземпляры для завершения установки. Можно запустить обновление с **экземпляров** странице **параметры** раздел.

![Включение Azure Monitor для обработки развертывания мониторинга виртуальных машин](./media/vminsights-enable-single-vm/onboard-vminsights-vmss-portal-status-01.png)

Теперь, когда вы настроили мониторинг для виртуальной Машины или виртуальной машины масштабируемого набора, данные мониторинга для анализа в Azure Monitor для виртуальных машин.

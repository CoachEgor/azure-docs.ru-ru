# <a name="enable-azure-monitor-for-single-vm-or-vmss-in-the-azure-portal"></a>Включение Azure Monitor для одной виртуальной машины или VMSS в портал Azure
В этой статье описывается, как включить Azure Monitor для виртуальных машин для одной виртуальной машины или масштабируемого набора виртуальных машин с помощью портал Azure. Эту процедуру можно использовать для следующих действий:

- Виртуальная машина Azure
- Масштабируемый набор виртуальных машин Azure
- Машина Arc Azure

Прежде чем начать, ознакомьтесь с [предварительными](vminsights-enable-overview.md) требованиями и убедитесь, что подписка и ресурсы соответствуют требованиям.  

## <a name="enable-azure-monitor-for-vms"></a>Включение Azure Monitor для виртуальных машин

1. Войдите на [портал Azure](https://portal.azure.com).

1. Выберите **виртуальные машины**, **масштабируемые наборы виртуальных машин**или **компьютеры в службе "Дуга Azure**".

1. Выберите ресурс из списка.

1. В разделе **мониторинг** меню выберите **Insights** , а затем **включите**. В следующем примере показана виртуальная машина Azure, но меню аналогично для Azure VMSS или Azure ARC.

    ![Включение Azure Monitor для виртуальной машины](media/vminsights-enable-single-vm/enable-vminsights-vm-portal.png)

1. Если виртуальная машина еще не подключена к рабочей области Log Analytics, вам будет предложено выбрать ее. Если вы еще не [создали рабочую область](../learn/quick-create-workspace.md), вы можете выбрать значение по умолчанию для расположения, в котором развернута виртуальная машина или VMSS в подписке. Эта Рабочая область будет создана и настроена, если она еще не создана.

2. При выполнении конфигурации вы получите сообщения о состоянии.

    >[!NOTE]
    >Если для масштабируемого набора используется Ручная модель обновления, обновите экземпляры, чтобы завершить установку. Обновления можно запустить на странице **экземпляры** в разделе **Параметры** .

    ![Включение Azure Monitor для обработки развертывания мониторинга виртуальных машин](media/vminsights-enable-single-vm/onboard-vminsights-vm-portal-status.png)

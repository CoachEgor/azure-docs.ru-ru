# <a name="what-is-azure-monitor-for-vms"></a>Общие сведения об Azure Monitor для виртуальных машин

Azure Monitor для виртуальных машин отслеживает виртуальные машины Azure и масштабируемые наборы виртуальных машин в нужном масштабе. Это решение анализирует производительность и работоспособность виртуальных машин Windows и Linux, отслеживает их процессы и зависимости на других ресурсах, а также внешние процессы. Она включает в себя поддержку мониторинга производительности и зависимостей приложений для вс-ми, которые размещаются на месте или в другом поставщике облачных технологий. Следующие ключевые особенности обеспечивают глубокое понимание:

- **Заранее определенные трендовые диаграммы производительности**: Отображение основных показателей производительности от гостевой операционной системы VM.

- **Карта зависимости**: Отображает взаимосвязанные компоненты с VM из различных групп ресурсов и подписок.  

>[!NOTE]
>Недавно мы [объявили об изменениях,](https://azure.microsoft.com/updates/updates-to-azure-monitor-for-virtual-machines-preview-before-general-availability-release/
) которые мы вносим в функцию «Здоровье», основываясь на отзывах, которые мы получили от наших публичных клиентов предварительного просмотра. Учитывая количество изменений, которые мы будем делать, мы собираемся прекратить предлагать функцию здоровье для новых клиентов. Существующие клиенты могут продолжать использовать функцию работоспособности. Для получения более подробной информации, пожалуйста, обратитесь к нашим [общим часто задаваемым вопросом о доступности](vminsights-ga-release-faq.md).  

Интеграция с журналами Azure Monitor обеспечивает мощную агрегацию и фильтрацию, позволяя Azure Monitor для vMs анализировать тенденции данных с течением времени. Вы можете просматривать эти данные в одном VM с виртуальной машины напрямую, или вы можете использовать Azure Monitor для доставки агрегированного представления ваших виртуальных технологий, где представление поддерживает ресурс-контекст Azure или режимрабочей контекста. Для получения дополнительной информации смотрите [обзор режимов доступа](../platform/design-logs-deployment.md#access-mode).

Пакет Azure Monitor для виртуальных машин может обеспечить прогнозируемую производительность и доступность важнейших приложений. Он определяет узкие места производительности и сетевые проблемы, а также может помочь вам понять, связана ли проблема с другими зависимостями.  

## <a name="data-usage"></a>Использование данных

При развертывании Azure Monitor для виртуальных машин данные, собранные с виртуальных машин, принимаются и хранятся в Azure Monitor. Собранные данные о производительности и зависимости хранятся в рабочей области Log Analytics. Плата в Azure Monitor для виртуальных машин выставляется за следующие элементы (на основе цен, опубликованных на [странице цен на Azure Monitor](https://azure.microsoft.com/pricing/details/monitor/)):

- данные, которые принимаются и хранятся;
- правила генерации оповещений, которые создаются;
- отправляемые уведомления. 

Размер журнала варьируется в зависимости от длины строксчетчиков производительности и может увеличиваться с увеличением количества логических дисков и сетевых адаптеров, выделенных для VM. Если у вас уже есть рабочее пространство и вы собираете эти счетчики, никаких дублирующих сборов не будет. Если вы уже используете карту обслуживания, единственное изменение, которое вы увидите, это дополнительные данные о подключении, отправленные в Azure Monitor.

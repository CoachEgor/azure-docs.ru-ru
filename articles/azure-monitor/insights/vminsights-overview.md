# <a name="what-is-azure-monitor-for-vms-preview"></a>Что такое Azure Monitor для виртуальных машин (предварительная версия)?

Azure Monitor для виртуальных машин отслеживает виртуальные машины Azure и масштабируемые наборы виртуальных машин в нужном масштабе. Это решение анализирует производительность и работоспособность виртуальных машин Windows и Linux, отслеживает их процессы и зависимости на других ресурсах, а также внешние процессы. 

Решение Azure Monitor для виртуальных машин включает поддержку мониторинга производительности и зависимостей приложений для виртуальных машин, размещенных локально или в другом поставщике облачных служб. Подробные аналитические сведения предоставляют три ключевых компонента:

* **Логические компоненты виртуальных машин Azure, выполняющих Windows и Linux**. Измеряются в соответствии с предварительно настроенными критериями работоспособности, и они предупреждают вас, когда условие оценки выполнено.  

* **Предварительно определенные диаграммы тенденций производительности**: Отображает основные метрики производительности из гостевой операционной системы виртуальной машины.

* **Карта зависимостей**. Отображает взаимосвязанные с виртуальной машиной компоненты различных групп ресурсов и подписок.  

Эти возможности упорядочены по трем перспективам:

* Здоровье
* Производительность
* Сопоставление

>[!NOTE]
>В настоящее время функция работоспособности предоставляется только для виртуальных машин Azure. Производительность и функции карты поддерживают виртуальные машины Azure, масштабируемые наборы виртуальных Машин Azure и виртуальных машин, размещенных в вашей среде или другого поставщика облачных решений.

Интеграция с журналами Azure Monitor обеспечивает мощную агрегацию и фильтрацию и позволяет проводить анализ тенденций данных с течением времени. Такой комплексный мониторинг рабочей нагрузки не может осуществляться только c помощью Azure Monitor или Сопоставления служб.  

Эти данные можно просмотреть в одной виртуальной машине напрямую из виртуальной машины или использовать Azure Monitor для предоставления статического представления своих виртуальных машин. Это представление формируется в зависимости от каждой функции:

* **Работоспособность**. Виртуальные машины, относящиеся к группе ресурсов.
* **Сопоставления** и **Производительность**. Виртуальные машины настроены на отправку отчетов в определенную рабочую область Log Analytics.

![Перспектива полезных сведений о виртуальной машине на портале Azure](./media/vminsights-overview/vminsights-azmon-directvm-01.png)

Пакет Azure Monitor для виртуальных машин может обеспечить прогнозируемую производительность и доступность важнейших приложений. Он идентифицирует важные события в операционной системе, узкие места производительности и сетевые проблемы. Azure Monitor для виртуальных машин может также помочь вам понять, связана ли проблема с другими зависимостями.  

## <a name="data-usage"></a>Использование данных 

При развертывании Azure Monitor для виртуальных машин данные, собранные с виртуальных машин, принимаются и хранятся в Azure Monitor. Критерии метрик работоспособности хранятся в Azure Monitor в базе данных временных рядов, данные о производительности и зависимостей, которые собираются, хранятся в рабочую область Log Analytics. Плата в Azure Monitor для виртуальных машин выставляется за следующие элементы (на основе цен, опубликованных на [странице цен на Azure Monitor](https://azure.microsoft.com/pricing/details/monitor/)):

* данные, которые принимаются и хранятся;
* число отслеживаемых временных рядов метрик критериев работоспособности;
* правила генерации оповещений, которые создаются;
* отправляемые уведомления. 

Размер журнала зависит от длины строк счетчиков производительности, и его можно увеличить с помощью число логических дисков и сетевых адаптеров, выделенного для виртуальной Машины. Если у вас уже есть рабочее пространство и вы собираете эти счетчики, никаких дублирующих сборов не будет. Если вы уже используете Сопоставление служб, единственное изменение, которое вы увидите — это данные дополнительного подключения, отправляемые в Azure Monitor.

# <a name="azure-monitor-for-containers-overview"></a>Обзор службы "Azure Monitor для контейнеров"

Azure Monitor для контейнеров — это функция, предназначенная для мониторинга производительности контейнерных нагрузок, развернутых для:

- Управляемые кластеры Kubernetes, размещенные на [службе Azure Kubernetes (AKS)](../../aks/intro-kubernetes.md)
- Самоуправляемые кластеры Kubernetes, размещенные на Azure с помощью [AKS Engine](https://github.com/Azure/aks-engine)
- [Инстанции контейнеров Azure](../../container-instances/container-instances-overview.md)
- Самоуправляемые кластеры Kubernetes, размещенные на [Azure Stack](https://docs.microsoft.com/azure-stack/user/azure-stack-kubernetes-aks-engine-overview?view=azs-1910) или на территории
- [Azure Red Hat OpenShift](../../openshift/intro-openshift.md)

Azure Monitor для контейнеров поддерживает кластеры под управлением операционной системы Linux и Windows Server 2019. 

Мониторинг контейнеров крайне важен, особенно если вы управляете рабочим кластером в нужном масштабе с несколькими приложениями.

С помощью Azure Monitor для контейнеров можно отслеживать производительность, собирая данные метрик памяти и процессора из контроллеров, узлов и контейнеров, доступных в Kubernetes, используя API метрик. Также собираются журналы контейнеров.  После включения мониторинга из кластеров Kubernetes метрики и журналы автоматически собираются для вас через контейнерную версию агента Log Analytics для Linux. Метрики записываются в хранилище метрик, а данные журналов — в хранилище журналов, связанное с рабочим пространством [Log Analytics.](../log-query/log-query-overview.md)

## <a name="what-does-azure-monitor-for-containers-provide"></a>В чем заключается польза Azure Monitor для контейнеров?

Azure Monitor для контейнеров обеспечивает полный опыт мониторинга с использованием различных функций Azure Monitor. Эти функции позволяют понять производительность и работоспособность кластера Kubernetes под управлением операционной системы Linux и Windows Server 2019, а также контейнерных рабочих нагрузок. С Azure Monitor для контейнеров вы можете:

* Определить, какие контейнеры AKS запущены на узле, а также данные о среднем использовании их ресурсов процессора и памяти. Эта информация может помочь выявить узкие места ресурсов.
* Определите использование ресурсов процессора и памяти группами и их контейнерами, размещенными в службе "Экземпляры контейнеров Azure".  
* Определить, где находится контейнер в контроллере или модуле pod. Эта информация может помочь просмотреть общую производительность контроллера или pod. 
* Просмотреть данные об использовании ресурсов рабочих нагрузок на узле, не относящихся к стандартным процессам, поддерживающим pod.
* Понять поведение кластера при средних и высоких нагрузках. Эта информация может помочь определить потребности в емкости, а также максимальную нагрузку, которую может выдержать кластер. 
* Настройка оповещений для упреждающего уведомления или записи, когда использование процессора и памяти в узлах или контейнерах превышает пороговые значения, или когда происходит изменение состояния работоспособности в кластере в инфраструктуре или свертывания узла.
* Интегрируйтесь с [Prometheus](https://prometheus.io/docs/introduction/overview/) для просмотра показателей приложений и рабочей нагрузки, которые он собирает из узлов и Kubernetes, используя [запросы](container-insights-log-search.md) для создания пользовательских оповещений, панелей мониторинга и детального выполнения детального анализа.
* Мониторинг контейнерных рабочих нагрузок, развернутых в [akS Engine](https://github.com/Azure/aks-engine) на месте и [AKS Engine на Azure Stack.](https://docs.microsoft.com/azure-stack/user/azure-stack-kubernetes-aks-engine-overview?view=azs-1908)
* Мониторинг контейнерных рабочих нагрузок, [развернутых в Azure Red Hat OpenShift.](../../openshift/intro-openshift.md)

    >[!NOTE]
    >Поддержка Azure Red Hat OpenShift — функция, представленная в открытом доступе.
    >

Основные различия в мониторинге кластера Windows Server по сравнению с кластером Linux:

- Метрика памяти RSS недоступна для узлов Windows и контейнеров.
- Информация о емкости диска недоступна для узлов Windows.
- Контейнерные журналы недоступны для контейнеров, работающих в узлах Windows.
- Поддержка функций Live Data (предварительный просмотр) доступна за исключением журналов контейнеров Windows.
- Контролируются только среды стручка, а не среды Docker.
- При предварительном выпуске поддерживается максимум 30 контейнеров Windows Server. Это ограничение не распространяется на контейнеры Linux. 

Ознакомьтесь со следующим видео, обеспечивающим глубокое погружение на промежуточном уровне, чтобы помочь вам узнать о мониторинге кластера AKS с помощью Azure Monitor для контейнеров.

## <a name="how-do-i-access-this-feature"></a>Как получить доступ к этому компоненту?

Вы можете получить доступ к Azure Monitor для контейнеров двумя способами: из Azure Monitor или непосредственно из выбранного кластера AKS. От Azure Monitor у вас есть глобальная перспектива всех развернутых контейнеров, которые контролируются, а какие нет, что позволяет искать и фильтровать через ваши подписки и группы ресурсов, а затем сверлить в Azure Monitor для контейнеров из выбранный контейнер.  В противном случае вы можете получить доступ к функции непосредственно из выбранного контейнера AKS со страницы AKS.

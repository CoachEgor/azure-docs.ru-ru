# <a name="integrate-system-center-operations-manager-with-azure-monitor-for-vms-map-feature"></a>Менеджер по работе с системным центром с функцией Azure Monitor для картдля высожжаемых

В Azure Monitor для виртуальных технологий можно просматривать обнаруженные компоненты приложений на виртуальных компьютерах Windows и Linux( которые работают в Azure или в вашей среде. Благодаря этой интеграции между функцией «Карта» и менеджером операций System Center можно автоматически создавать распределенные диаграммы приложений в «Менеджере операций», которые основаны на динамических картах зависимости в Azure Monitor для VMs. В этой статье описывается, как настроить группу управления управлением операциями System Center для поддержки этой функции.

>[!NOTE]
>Если вы уже развернули карту обслуживания, вы можете просматривать карты в Azure Monitor для вс-карт, который включает дополнительные функции для мониторинга работоспособности и производительности VM. Функция Карты Azure Monitor для VMs предназначена для замены отдельного решения Service Map. Дополнительные сведения см. в разделе [Общие сведения об Azure Monitor для виртуальных машин](vminsights-overview.md).

## <a name="prerequisites"></a>Предварительные требования

* Группа управления операциями системного центра (2012 R2 или более поздний срок).
* Рабочее пространство для анализа журналов, настроенное для поддержки azure Monitor для vMs.
* Один или несколько виртуальных компьютеров Windows и Linux или физических компьютеров, которые контролируются менеджером по операциям и отправляют данные в рабочее пространство Log Analytics. Для непосредственного подключения к Azure Monitor необходимо настроить серверы Linux, сообщающие об этом группе управления операциями. Для получения дополнительной информации просмотрите обзор [в журнале Collect data с агентом Log Analytics.](../platform/log-analytics-agent.md)
* Субъект-служба с доступом к подписке Azure, связанной с рабочей областью Log Analytics. Дополнительные сведения см. в руководстве по [созданию субъекта-службы](#create-a-service-principal).

## <a name="install-the-service-map-management-pack"></a>Установка пакета управления схемы услуги

Вы позволяете интегрировать сяочку между менеджером операций и функцией Карты, импортируя пакет пакет управления Microsoft.SystemCenter.ServiceMap (Microsoft.SystemCenter.ServiceMap.mpb). Скачать пакет управления можно в [Центре загрузки Майкрософт](https://www.microsoft.com/download/details.aspx?id=55763). Этот набор содержит следующие пакеты управления:

* Microsoft.ServiceMap.Application.Views;
* Microsoft.System.Center.ServiceMap.Internal;
* Microsoft.System.Center.ServiceMap.Overrides;
* Microsoft.System.Center.ServiceMap.

## <a name="configure-integration"></a>Настройка интеграции

После установки пакета управления картами обслуживания новый узло , **Карта обслуживания**, отображается в **наборе управления операциями** в **панели администрирования** консоли Операций менеджера операций.

>[!NOTE]
>[Набор услуг по управлению операциями — это набор услуг,](../terminology.md#april-2018---retirement-of-operations-management-suite-brand) включаемых в log Analytics, в настоящее время входящих в [Azure Monitor.](../overview.md)

Для настройки Azure Monitor для интеграции Карты VMs сделайте следующее:

1. Чтобы открыть мастер настройки, щелкните **Добавить рабочую область** в области **Service Map Overview** (Обзор схемы услуги).  

    ![Область "Service Map Overview" (Обзор схемы услуги)](media/service-map-scom/scom-configuration.png)

2. В окне **Настройка подключения** введите имя или идентификатор клиента, идентификатор приложения (имя пользователя или clientID) и пароль субъекта-службы, а затем нажмите кнопку **Далее**. Дополнительные сведения см. в руководстве по созданию субъекта-службы.

    ![Окно "Настройка подключения"](media/service-map-scom/scom-config-spn.png)

3. В окне **Subscription Selection** (Выбор подписки) выберите подписку Azure, группу ресурсов Azure (которая содержит рабочую область Log Analytics) и саму рабочую область Log Analytics, а затем нажмите кнопку **Далее**.

    ![Настройка Operations Manager: рабочая область](media/service-map-scom/scom-config-workspace.png)

4. В окне **Machine Group Selection** (Выбор группы компьютеров) выберите группы компьютеров решения "Сопоставление служб", которые необходимо синхронизировать с Operations Manager. Нажмите кнопку **Add/Remove Machine Groups** (Добавить или удалить группы компьютеров), выберите группы из списка **доступных групп компьютеров**и нажмите кнопку **Добавить**.  Выбрав группы, нажмите кнопку **ОК** для завершения.

    ![Настройка групп компьютеров в Operations Manager](media/service-map-scom/scom-config-machine-groups.png)

5. В окне **выбора сервера** вы настраиваете группу серверов службы с серверами, которые необходимо синхронизировать между менеджером операций и функцией Карты. Нажмите кнопку **Добавление и удаление серверов**.

    Для интеграции и успешного построения схемы распределенного приложения для сервера он должен:

   * Мониторинг, проведенный менеджером по операциям
   * Настроен для отчета в рабочее пространство журнала Analytics, настроенное с помощью Azure Monitor для вс-ми
   * входить в группу серверов решения "Сопоставление служб".

     ![Настройка Operations Manager: группа](media/service-map-scom/scom-config-group.png)

6. Дополнительно: Выберите все серверы управления ресурсный пул для связи с журналом Analytics, а затем нажмите **Добавить Рабочее пространство**.

    ![Настройка Operations Manager: пул ресурсов](media/service-map-scom/scom-config-pool.png)

    Для настройки и регистрации рабочей области Log Analytics может потребоваться около минуты. После настройки менеджер операций инициирует первую синхронизацию карты.

    ![Настройка Operations Manager: пул ресурсов](media/service-map-scom/scom-config-success.png)

## <a name="monitor-integration"></a>Мониторинг интеграции

После подключения рабочего пространства Log Analytics новая папка Service Map отображается в панели **мониторинга** консоли Operations Manager Operations.

![Operations Manager: область "Мониторинг"](media/service-map-scom/scom-monitoring.png)

Папка "Сопоставление служб" имеет четыре узла:

* **Активные оповещения:** Перечисляет все активные оповещения о связи между менеджером операций и Azure Monitor.  

  >[!NOTE]
  >Эти оповещения не являются оповещениями Log Analytics, синхронизированными с менеджером операций, они генерируются в группе управления на основе рабочих процессов, определенных в пакете управления картами обслуживания.

* **Серверы:** Списки контролируемых серверов, настроенных для синхронизации с функцией Azure Monitor для карты vMs.

    ![Operations Manager: область "Серверы"](media/service-map-scom/scom-monitoring-servers.png)

* **Представления зависимостей группы машин:** списки всех групп машин, синхронизированных с функцией «Карта». Вы можете щелкнуть любую группу, чтобы просмотреть распределенные диаграммы приложения.

    ![Схема распределенного приложения Operations Manager](media/service-map-scom/scom-group-dad.png)

* **Представления зависимостей серверов: Списки**всех серверов, синхронизированных с функцией Карты. Вы можете щелкнуть любой сервер, чтобы просмотреть его схему распределенного приложения.

    ![Схема распределенного приложения Operations Manager](media/service-map-scom/scom-dad.png)

В текущем выпуске можно настроить только одно рабочее пространство Log Analytics.

## <a name="configure-rules-and-overrides"></a>Настройка правил и переопределений

Правило, *Microsoft.SystemCenter.ServiceMapИмпорт.Правило*, периодически получает информацию из Azure Monitor для функции карты VMs. Чтобы изменить интервал синхронизации, можно переопределить правило и изменить значение параметра **IntervalMinutes.**

![Operations Manager: окно свойств переопределений](media/service-map-scom/scom-overrides.png)

* **Enabled**: позволяет включить или отключить автоматическое обновление.
* **IntervalMinutes**: Определяет время между обновлениями. Значение по умолчанию — один час. Если вы хотите чаще синхронизировать карты, можно изменить значение.
* **TimeoutSeconds**: Определяет продолжительность времени до того, как запрос выйдет из срока.
* **TimeWindowMinutes**: Определяет временное окно для запросов данных. По умолчанию составляет 60 минут, что является максимальным интервалом.

## <a name="known-issues-and-limitations"></a>Известные проблемы и ограничения

В текущей версии существуют следующие проблемы и ограничения:

* Можно подключиться только к одной рабочей области Log Analytics.
* Хотя пользователи могут вручную добавить серверы в группу серверов решения "Сопоставление служб" с помощью области **Разработка**, сопоставления для этих серверов синхронизируются не сразу. Они будут синхронизированы с функцией Azure Monitor для mMs Map в течение следующего цикла синхронизации.
* Если вы внесете какие-либо изменения в диаграммы распределенных приложений, созданные пакетом управления, эти изменения, скорее всего, будут перезаписаны при следующей синхронизации с Azure Monitor для VMs.

## <a name="concurrency-maximums-for-workload-groups"></a>Параллельные максимумы для групп рабочей нагрузки
С введением [групп рабочей нагрузки,](sql-data-warehouse-workload-isolation.md)понятие параллелизма слотов больше не применяется.  Ресурсы на запрос распределяются на процентной основе и определяются в определении рабочей группы.  Однако, даже при удалении слотов параллелизма, существует минимальное количество ресурсов, необходимых для запросов в зависимости от уровня обслуживания. ## <a name="concurrency-maximums-for-resource-classes"></a>Параллельные максимумы для классов ресурсов
Для обеспечения того, чтобы каждый запрос имеет достаточно ресурсов для эффективного выполнения, использование ресурсов отслеживается путем назначения слотов для каждого запроса. Система помещает запросы в очередь в зависимости от важности и слотов для параллелизма. Запросы ждут в очереди до тех пор, пока не будет доступно достаточное количество слотов для параллелизма. [Значение](https://docs.microsoft.com/azure/sql-data-warehouse/sql-data-warehouse-workload-importance) и параллелизм слоты определяют приоритеты процессора. Дополнительные сведения см. в статье [Анализ рабочей нагрузки](analyze-your-workload.md).

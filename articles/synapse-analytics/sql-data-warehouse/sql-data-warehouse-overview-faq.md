# <a name="azure-synapse-analytics-formerly-sql-dw-frequently-asked-questions"></a>Лазурная синапсная аналитика (ранее S'L DW) Часто задаваемые вопросы

## <a name="general"></a>Общие сведения

У. Что такое Azure Synapse?

A. Azure Synapse — это безграничная аналитическая служба, объединяющая складирование данных и аналитику больших данных. Это дает вам свободу запроса данных на ваших условиях, используя либо сервера по требованию или подготовленные ресурсы - в масштабе. Azure Synapse объединяет эти два мира с единым опытом для того, чтобы глотать, готовить, управлять и обслуживать данные для непосредственных потребностей в области би-би-и машинного обучения. Для получения дополнительной информации см., [Что такое аналитика Azure Synapse](sql-data-warehouse-overview-what-is.md).

У. Что случилось со Хранилищем данных Azure S'L?

A. Azure Synapse — это хранилище данных Azure S'L (S'L DW). Мы вывели один и тот же ведущий в отрасли хранилище данных на совершенно новый уровень производительности и возможностей. Можно продолжить работу существующих рабочих нагрузок хранилища данных в производстве с помощью Azure Synapse и автоматически воспользоваться новыми возможностями, которые находятся в предварительном просмотре. Для получения дополнительной информации смотрите [что такое аналитика Azure Synapse](sql-data-warehouse-overview-what-is.md).

У. Что такое бассейн Synapse S'L?

A. В пуле Synapse S'L относятся к функциям складирования корпоративных данных, которые обычно доступны с Azure Synapse. Для получения дополнительной информации см., [Что такое аналитика Azure Synapse](sql-data-warehouse-overview-what-is.md).

У. Как начать работу с Azure Synapse?

A. Вы можете начать работу с [бесплатной учетной записью Azure](https://azure.microsoft.com/free/sql-data-warehouse/) или [связаться с продажами для получения дополнительной информации.](https://info.microsoft.com/ww-landing-azure-sql-data-warehouse-contactme.html)

У. Что предлагает Azure Synapse для обеспечения безопасности данных?

A. Azure Synapse предлагает несколько решений для защиты данных, таких как TDE и аудит. Дополнительные сведения см. в статье [Безопасность](sql-data-warehouse-overview-manage-security.md).

У. Где я могу узнать, какие юридические или бизнес-стандарты Azure Synapse соответствуют?

A. На странице [соответствия требованиям корпорации Майкрософт](https://www.microsoft.com/trustcenter/compliance/complianceofferings) доступны различные сертификаты соответствия (такие как SOC и ISO), которые можно отфильтровать по продукту.
Во-первых, выберите по названию соответствия. Затем расширьте Azure в разделе облачных служб Microsoft in-scope на правой стороне страницы, чтобы узнать, какие службы соответствуют требованиям Azure Synapse.

У. Могу ли я подключить Power BI?

A. Да! Хотя Power BI поддерживает прямой запрос с Azure Synapse, он не предназначен для большого числа пользователей или данных в режиме реального времени. Для дальнейшего оптимизации производительности Power BI рассмотрите возможность использования Power BI поверх служб анализа Azure или аналитической службы IaaS.

У. Каковы ограничения в емкости пула Synapse S'L?

A. Текущие ограничения см. на странице [Ограничения емкости хранилища данных SQL](sql-data-warehouse-service-capacity-limits.md).

У. Почему операция масштабирования, приостановки или возобновления занимает так много времени?

A. На время операций по управлению вычислениями могут влиять несколько факторов. Распространенным аргументом для длительных операций является откат транзакций. При инициации операции масштабирования или приостановки блокируются все входящие сеансы, и запросы утрачиваются. Чтобы оставить систему в стабильном состоянии, перед началом операции необходимо выполнить откат транзакций. Чем больше число транзакций и размер их журналов, тем дольше будет задержка операции при восстановлении стабильного состояния системы.

## <a name="loading"></a>Загрузка

У. Какие драйверы клиента поддерживаются?

A. Поддержка драйверов для пула Synapse S'L можно найти на странице [строк соединения](../sql/connection-strings.md)

В: Какие форматы файлов поддерживаются PolyBase?

О. Orc, RC, Parquet и неструктурированный текст с разделителями.

В: К каким источникам данных можно подключиться с помощью PolyBase?

О: [Хранилище озер данных Azure](sql-data-warehouse-load-from-azure-data-lake-store.md) и [блоки хранения azure](sql-data-warehouse-load-from-azure-blob-storage-with-polybase.md)

В: Возможна ли выталкивание вычислений при подключении к Azure Storage Blobs или ADLS?

О: Нет, PolyBase взаимодействует только с компонентами хранения.

В. Можно ли подключиться к HDI?

О. HDI может использовать Azure Data Lake Store или WASB в качестве уровня HDFS. Если у вас есть слой HDFS, вы можете загрузить эти данные в пул Synapse S'L. Тем не менее, создание вычислений со стековой памятью для экземпляра HDI невозможно.

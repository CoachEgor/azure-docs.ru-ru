# <a name="connect-to-your-azure-synapse-workspace-using-private-links-preview"></a>Подключение к рабочей области Azure синапсе с помощью частных ссылок (Предварительная версия)

Из этой статьи вы узнаете, как создать закрытую конечную точку в рабочей области Azure синапсе. Дополнительные сведения см. в разделе [частные ссылки и частные конечные точки](https://docs.microsoft.com/azure/private-link/) .

## <a name="step-1-open-your-azure-synapse-workspace-in-azure-portal"></a>Шаг 1. Откройте рабочую область Azure синапсе в портал Azure

Выберите **Подключение к частной конечной точке** в разделе **Безопасность** , а затем выберите **+ Частная конечная точка**.
![Откройте рабочую область Azure синапсе в портал Azure](./media/how-to-connect-to-workspace-with-private-links/private-endpoint-1.png)

## <a name="step-2-select-your-subscription-and-region-details"></a>Шаг 2. Выбор сведений о подписке и регионе

На вкладке **Основные сведения** в окне **Создание частной конечной точки** выберите **подписку** и **группу ресурсов**. Присвойте **имя** частной конечной точке, которую вы хотите создать. Выберите **регион** , в котором должна быть создана частная конечная точка.

Частные конечные точки создаются в подсети. Выбранная подписка, Группа ресурсов и регион отфильтруют частные подсети конечных точек. По завершении нажмите кнопку **Далее: ресурс >** .
![Выбор сведений о подписке и регионе](./media/how-to-connect-to-workspace-with-private-links/private-endpoint-2.png)

## <a name="step-3-select-your-azure-synapse-workspace-details"></a>Шаг 3. Выбор сведений о рабочей области Azure синапсе

Выберите **подключиться к ресурсу Azure в моем каталоге** на вкладке **ресурс** . Выберите **подписку** , которая содержит рабочую область Azure синапсе. **Тип ресурса** для создания частных конечных точек в рабочей области Azure синапсе — *Microsoft. синапсе/workspaces*.

Выберите рабочую область Azure синапсе в качестве **ресурса**. У каждой рабочей области Azure синапсе есть три **целевых подресурса** , которые можно использовать для создания частной конечной точки: SQL, Склондеманд и dev.

Нажмите кнопку **Далее:>конфигурации** , чтобы перейти к следующей части установки.
![Выбор сведений о подписке и регионе](./media/how-to-connect-to-workspace-with-private-links/private-endpoint-3.png)

На вкладке **Конфигурация** выберите **виртуальную сеть** и **подсеть** , в которой должна быть создана частная конечная точка. Также необходимо создать запись DNS, которая сопоставляется с частной конечной точкой.

Выберите **Да для "** **Интеграция с частной зоной DNS** ", чтобы интегрировать закрытую конечную точку с частной зоной DNS. Если у вас нет частной зоны DNS, связанной с виртуальной сетью, создается новая частная зона DNS. По завершении выберите **Проверка и создать** .

![Выбор сведений о подписке и регионе](./media/how-to-connect-to-workspace-with-private-links/private-endpoint-4.png)

После завершения развертывания откройте рабочую область Azure синапсе в портал Azure и выберите **подключения к частным конечным точкам**. Будут показаны новые имена частной конечной точки и подключения частной конечной точки, связанные с частной конечной точкой.

![Выбор сведений о подписке и регионе](./media/how-to-connect-to-workspace-with-private-links/private-endpoint-5.png)

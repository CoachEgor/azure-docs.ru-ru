# <a name="connect-to-synapse-studio-workspace-resources-from-a-restricted-network"></a>Подключение к ресурсам рабочей области синапсе Studio из ограниченной сети

Целевой модуль чтения этой статьи — ИТ-администратор компании, управляющий ограниченной сетью компании. ИТ-администратор собирается включить сетевое подключение между Azure синапсе Studio и рабочей станцией в этой ограниченной сети.

Из этой статьи вы узнаете, как подключиться к рабочей области Azure синапсе из ограниченной сетевой среды. 

## <a name="prerequisites"></a>Предварительные требования

* **Подписка Azure** : Если у вас еще нет подписки Azure, создайте [бесплатную учетную запись](https://azure.microsoft.com/free/) Azure, прежде чем начинать работу.
* **Рабочая область Azure синапсе**. Если у вас нет синапсе Studio, создайте рабочую область синапсе из Azure синапсе Analytics. Имя рабочей области будет необходимо на шаге 4.
* **Ограниченная сеть**. Ограниченная сеть обслуживается ИТ-администратором компании. Администратор имеет разрешение на настройку политики сети. Имя виртуальной сети и ее подсеть будут необходимы на шаге 3.



## <a name="step-1-add-network-outbound-security-rules-to-the-restricted-network"></a>Шаг 1. Добавление сетевых правил безопасности для исходящего трафика в сеть с ограниченным доступом

Необходимо добавить четыре правила сетевой безопасности для исходящего трафика с четырьмя тегами службы. Дополнительные сведения о [тегах служб](/azure/virtual-network/service-tags-overview.md) 
* AzureResourceManager
* AzureFrontDoor.Frontend
* AzureActiveDirectory
* Азуремонитор (необязательно. Добавьте правило этого типа, только если вы хотите предоставить доступ к данным в корпорацию Майкрософт.)

**Azure Resource Manager** подробные сведения о правиле исходящего трафика, как показано ниже. При создании других трех правил замените значение " **тег службы назначения** ", выбрав имя тега службы " **азурефронтдур. интерфейс** ", " **AzureActiveDirectory** ", " **азуремонитор** " в раскрывающемся списке выбора.

![AzureResourceManager](./media/how-to-connect-to-workspace-from-restricted-network/arm-servicetag.png)


## <a name="step-2-create-azure-synapse-analytics-private-link-hubs"></a>Шаг 2. Создание Azure синапсе Analytics (концентраторы частных ссылок)

Вам потребуется создать Azure синапсе Analytics (частные центры ссылок) из портал Azure. Выполните поиск по запросу " **аналитика синапсе Azure (концентраторы частных ссылок)** " в портал Azure, а затем заполните требуемое поле и создайте его. 

> [!Note]
> Регион должен совпадать с областью, в которой находится рабочая область синапсе.

![Создание концентраторов частных ссылок синапсе Analytics](./media/how-to-connect-to-workspace-from-restricted-network/private-links.png)

## <a name="step-3-create-private-link-endpoint-for-synapse-studio-gateway"></a>Шаг 3. Создание конечной точки частной ссылки для шлюза синапсе Studio

Чтобы получить доступ к шлюзу синапсе Studio, необходимо создать конечную точку частного канала из портал Azure. Выполните поиск по фразе " **Частная ссылка** " с помощью портал Azure. Выберите " **создать частную конечную точку** " в " **частном канале связи** ", а затем заполните требуемое поле и создайте его. 

> [!Note]
> Регион должен совпадать с областью, в которой находится рабочая область синапсе.

![Создание закрытой конечной точки для синапсе Studio 1](./media/how-to-connect-to-workspace-from-restricted-network/plink-endpoint-1.png)

На следующей вкладке " **ресурс** " выберите концентратор частных ссылок, созданный на шаге 2 выше.

![Создание закрытой конечной точки для синапсе Studio 2](./media/how-to-connect-to-workspace-from-restricted-network/plink-endpoint-2.png)

На следующей вкладке " **Конфигурация** " 
* Выберите ограниченное имя виртуальной сети для " **виртуальной сети** ".
* Выберите подсеть ограниченной виртуальной сети для " **подсеть** ". 
* Выберите " **Да** ", чтобы " **интегрировать с частной зоной DNS** ".

![Создание закрытой конечной точки для синапсе Studio 3](./media/how-to-connect-to-workspace-from-restricted-network/plink-endpoint-3.png)

После создания конечной точки частной ссылки можно открыть страницу входа веб-инструмента синапсе Studio. Однако вы не сможете получить доступ к ресурсам в рабочей области синапсе до тех пор, пока вам не потребуется выполнить следующий шаг.

## <a name="step-4-create-private-link-endpoints-for-synapse-studio-workspace-resource"></a>Шаг 4. Создание конечных точек частной ссылки для ресурса рабочей области синапсе Studio

Чтобы получить доступ к ресурсам рабочей области синапсе Studio, необходимо создать по крайней мере одну конечную точку частной связи с типом " **dev** " **целевого подресурсов** и двумя другими дополнительными конечными точками частной связи с типами " **SQL** " или " **склондеманд** ", которые зависят от ресурсов в рабочей области синапсе Studio, к которой вы хотите получить доступ. Эта конечная точка закрытой ссылки для рабочей области синапсе Studio аналогична созданию конечной точки.  

Обратите внимание на следующие области на вкладке " **ресурс** ":
* Выберите " **Microsoft. синапсе/рабочие области** " в " **тип ресурса** ".
* Выберите " **йоурворкспаценаме** " для **ресурса** , который вы создали ранее.
* Выберите тип конечной точки в поле " **целевой вспомогательный ресурс** ":
  * **SQL** : предназначен для выполнения запросов SQL в пуле SQL.
  * **Склондеманд** : предназначен для встроенного выполнения запросов SQL.
  * **Dev** : предназначен для доступа ко всем остальным в рабочих областях синапсе Studio. Необходимо создать по крайней мере конечную точку частной ссылки для этого типа.

![Создание частной конечной точки для рабочей области синапсе Studio](./media/how-to-connect-to-workspace-from-restricted-network/plinks-endpoint-ws-1.png)

Теперь все готово. Вы можете получить доступ к ресурсу рабочей области синапсе Studio.

# <a name="azure-synapse-analytics-workspaces-preview-frequently-asked-questions"></a>Вопросы и ответы по Azure Synapse Analytics (предварительная версия рабочих областей)

В этом руководстве вы найдете наиболее часто задаваемые вопросы о Synapse Analytics.

[!INCLUDE [preview](includes/note-preview.md)]

## <a name="general"></a>Общие сведения

### <a name="q-what-is-azure-synapse-analytics"></a>Вопрос. Что такое Azure Synapse Analytics?

A. Azure Synapse представляет собой интегрированную платформу данных для бизнес-аналитики, искусственного интеллекта и непрерывной аналитики. Она объединяет на одной платформе несколько сред выполнения аналитики, таких как SQL и Apache Spark, и предоставляет унифицированные возможности:

- для защиты ресурсов аналитики, включая сеть, управление доступом с применением единого входа к артефактам пула, данных и разработки;
- для удобного отслеживания и быстрой оптимизации, реагирования на события и отладки событий, происходящих в действиях на любом слое рабочей области;
- для управления метаданными между несколькими ядрами (созданная в бессерверном пуле Apache Spark таблица будет автоматически доступна в базах данных Azure Synapse);
- для унифицированного взаимодействия с пользователем при доступе к данным (Synapse Studio объединяет на одной платформе разработчиков больших данных, инженеров по обработке данных, администраторов баз данных, специалистов по обработке и анализу данных).

Дополнительные сведения см. в разделе [Что такое Azure Synapse Analytics?](https://docs.microsoft.com/azure/synapse-analytics/overview-what-is)

### <a name="q-what-are-the-main-components-of-azure-synapse-analytics"></a>Вопрос. Какие компоненты входят в Azure Synapse Analytics?

A. Решение Azure Synapse обеспечивает следующие возможности:

- Возможности для аналитики предлагаются через выделенный пул SQL или бессерверный пул SQL (предварительная версия).
- Бессерверный пул Apache Spark (предварительная версия) с полной поддержкой Scala, Python, Spark, SQL и C#.
- Поток данных с интерфейсом для преобразования больших данных без написания кода.
- Интеграция и оркестрация данных для автоматизации процессов интеграции данных и разработки кода.
- Studio с единым веб-интерфейсом для доступа ко всем этим возможностям.

### <a name="q-how-does-azure-synapse-analytics-relate-to-azure-sql-data-warehouse"></a>Вопрос. Как Azure Synapse Analytics связана с Хранилищем данных SQL Azure?

A. Azure Synapse Analytics как платформа аналитики создана на основе Хранилища данных SQL Azure и включает выделенный пул SQL в качестве решения для хранения данных. Эта платформа сочетает в себе уровни исследования данных, приема, преобразования, подготовки и аналитики обслуживания.

## <a name="use-cases"></a>Варианты использования

### <a name="q-how-do-i-rename-a-published-artifact-dataset-notebook-sql-script-and-so-on-in-azure-synapse"></a>Вопрос. Как переименовать опубликованный артефакт (набор данных, записную книжку, скрипт SQL и пр.) в Azure Synapse?

A. Чтобы переименовать опубликованный файл артефакта, сначала создайте его копию и переименуйте новый файл, присвоив ему имя. Вам нужно вручную обновить все ссылки артефакта, указав новое имя файла и удалить старое.

### <a name="q-what-is-a-good-use-case-for-dedicated-sql-pool"></a>Вопрос. Как правильно применять выделенный пул SQL?

A. Выделенный пул SQL создает основу для хранилища данных. Это лучшее решение для работы с хранилищем данных по соотношению [цены и производительности](https://azure.microsoft.com/services/sql-data-warehouse/compare/). Выделенный пул SQL лидирует в отрасли облачных решений для хранения данных благодаря следующим возможностям:

- обслуживание больших и смешанных рабочих нагрузок без снижения производительности благодаря высокому параллелизму и изоляции рабочих нагрузок;
- защита данных с помощью современных функций, от безопасности сети до точного контроля доступа;
- широкой спектр готовых экосистем.

### <a name="q-what-is-a-good-use-case-for-serverless-apache-spark-pool-in-azure-synapse"></a>Вопрос. Как правильно применять бессерверный пул Apache Spark в Azure Synapse?

A. Главной целью мы считаем эффективную поддержку инжиниринга данных для пакетного и потокового преобразования данных в озере данных. Тесная и простая интеграция с инфраструктурой оркестрации данных существенно упрощает ввод созданных решений в эксплуатацию.

Кроме того, специалисты по обработке и анализу данных могут применять Apache Spark для:

- извлечения признаков;
- просмотра данных;
- обучения моделей.

### <a name="q-what-is-a-good-use-case-for-serverless-sql-pool-in-azure-synapse"></a>Вопрос. Как правильно применять бессерверный пул SQL в Azure Synapse?

A. Бессерверный пул SQL — это служба для создания запросов к данным в озере данных. Она позволяет упростить доступ ко всем данным, поддерживая знакомый синтаксис T-SQL для запроса данных на месте без копирования или загрузки в специализированное хранилище.

Вот несколько примеров использования:

- простое обнаружение и исследование позволяет специалистам по обработке, анализу и инжинирингу данных использовать простой метод доступа к данным в озере данных на основе запросов чтения T-SQL;
- логическое хранилище данных предоставляет специалистам по аналитике полную мощь языка T-SQL для просмотра и анализа данных, размещенных в службе хранилища Azure, а также возможность использовать привычные средства бизнес-аналитики (службы анализа Azure, Power BI Premium и т. д.) для обновления панелей мониторинга запросами Starlight Query;
- извлечение, преобразование и загрузка одним запросом позволяет специалистам по инжинирингу данных преобразовывать данные в службе хранилища Azure из одного формата в другой, фильтровать, агрегировать и т. д. с высоким уровнем параллельности, сохранением результатов запросов в службе хранилище Azure и возможностью быстро предоставить доступ для дальнейшей обработки в Starlight Query или других службах.

### <a name="q-what-is-a-good-use-case-for-data-flow-in-azure-synapse"></a>Вопрос. Как правильно применять поток данных в Azure Synapse?

A. Потоки данных позволяют специалистам по инжинирингу данных разрабатывать логику преобразования графических данных без написания кода. Полученные потоки данных выполняются как действия на платформе интеграции и оркестрации данных. Действия потока данных можно применять через уже существующие средства планирования, управления, потока и мониторинга.

## <a name="security-and-access"></a>Безопасность и доступ

A. Сквозной процесс единого входа играет важную роль в системе проверки подлинности Synapse Analytics. Передача идентификаторов и управление ими выполняются только через механизм полной интеграции с Azure AD.

### <a name="q-how-do-i-get-access-to-files-and-folders-in-the-adls-gen2"></a>Вопрос. Как получить доступ к файлам и папкам в Azure Data Lake Storage 2-го поколения?

A. Сейчас управление доступом к файлам и папкам осуществляется через Azure Data Lake Storage 2-го поколения. Дополнительные сведения см. в статье [о контроле доступа в Azure Data Lake Storage](../storage/blobs/data-lake-storage-access-control.md?toc=/azure/synapse-analytics/toc.json&bc=/azure/synapse-analytics/breadcrumb/toc.json).

### <a name="q-can-i-use-third-party-business-intelligence-tools-to-access-azure-synapse-analytics"></a>Вопрос. Можно ли использовать сторонние средства бизнес-аналитики для доступа к Azure Synapse Analytics?

A. Да, вы можете использовать для подключения к выделенному пулу SQL и к бессерверному пулу SQL сторонние бизнес-приложения, такие как Tableau и Power BI. Spark поддерживает IntelliJ.

# <a name="sfctl-events"></a>sfctl события
Извлекать события из хранилища событий (если служба EventStore уже установлена).

Система EventStore может быть добавлена с помощью обновления конфигурации к любому кластеру SFRP, работая >6,4 евро. Пожалуйста, проверьте следующий URL\: https\://docs.microsoft.com/azure/service-fabric/service-fabric-diagnostics-eventstore.

## <a name="commands"></a>Команды

|Команда|Описание|
| --- | --- |
| все приложения-список | Получает все события, связанные с приложениями. |
| все узлы-лист | Получает все события, связанные с узлами. |
| список всех разделов | Получает все события, связанные с разделами. |
| все услуги-список | Получает все события, связанные с службами. |
| список приложений | Получает события, связанные с приложением. |
| кластерный список | Получает все события, связанные с кластером. |
| список узлов | Получает события, связанные с узлами. |
| список перегородок-все реплики | Получает все события, связанные с репликами, для раздела. |
| список разделов | Получает события, связанные с разделом. |
| список перегородки-реплики | Получает события, связанные с репликой раздела. |
| сервис-лист | Получает события, связанные с обслуживанием. |

## <a name="sfctl-events-all-applications-list"></a>sfctl события все-приложения-список
Получает все события, связанные с приложениями.

Ответ — это список объектов ApplicationEvent.

### <a name="arguments"></a>Аргументы

|Аргумент|Описание|
| --- | --- |
| ---конец времени-utc (обязательно) | Окончание времени поиска запроса в ISO UTC yyyy-MM-ddTHH\:мм\:ss. |
| --начало-время-utc (обязательно) | Время начала поиска запроса в ISO UTC yyyy-MM-ddTHH\:мм\:ss. |
| --события-типы-фильтр | Это запятая разделенная строка с указанием типов FabricEvents, которые должны быть включены только в ответ. |
| --исключить-анализ-события | Эта парам отсутсвийает поиск AnalysisEvents, если верно передается. |
| --скип-корреляция-поиск | Эта парама отражает поиск информации CorrelatedEvents, если истинна передается. в противном случае CorrelationEvents обрабатываются и HasCorrelatedEvents поле в каждом FabricEvent получает заселен. |
| --timeout -t | Тайм-аут сервера для выполнения операции в считанные секунды. Этот тайм-аут определяет продолжительность времени, которую клиент готов ждать, пока запрошенная операция завершится. Значение по умолчанию для этого параметра составляет 60 секунд.  Значение по умолчанию\: 60. |

### <a name="global-arguments"></a>Глобальные аргументы

|Аргумент|Описание|
| --- | --- |
| --debug | Повышение уровня детализации журнала для включения всех журналов отладки. |
| --help -h | Отображение этого справочного сообщения и выход. |
| --output -o | Формат вывода.  Допустимые значения\: json, jsonc, table, tsv.  Значение по умолчанию\: json. |
| --query | Строка запроса JMESPath. Дополнительные сведения и примеры см. на веб-сайте http\://jmespath.org/. |
| --verbose | Повышение уровня детализации журнала. Чтобы включить полные журналы отладки, используйте параметр --debug. |

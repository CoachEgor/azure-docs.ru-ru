# <a name="sfctl-property"></a>Свойство sfctl
Хранение и запрос свойств с именами Service Fabric.

## <a name="commands"></a>Команды

|Команда|Описание|
| --- | --- |
| удалить | Удаляет указанное свойство Service Fabric. |
| get | Возвращает указанное свойство Service Fabric. |
| list | Возвращает информацию обо всех свойствах Service Fabric с указанным именем. |
| put | Создает или обновляет свойство Service Fabric. |

## <a name="sfctl-property-delete"></a>sfctl property delete
Удаляет указанное свойство Service Fabric.

Удаляет указанное свойство Service Fabric с указанным именем. Свойство должно быть создано, прежде чем его можно будет удалить.

### <a name="arguments"></a>Аргументы

|Аргумент|Описание|
| --- | --- |
| --name-id [обязательный параметр] | Имя Service Fabric без указания схемы универсального кода ресурса (URI) 'fabric\:'. |
| --property-name [обязательный параметр] | Указывает имя возвращаемого свойства. |
| --timeout -t | Время ожидания сервера для выполнения операции в секундах. Это время ожидания указывает период времени, в течение которого клиент желает ожидать завершения запрошенной операции. Значение по умолчанию для этого параметра — 60 секунд.  Значение по умолчанию\: 60. |

### <a name="global-arguments"></a>Глобальные аргументы

|Аргумент|Описание|
| --- | --- |
| --debug | Увеличение уровня детализации ведения журнала для отображения всех журналов отладки. |
| --help -h | Отображение этого справочного сообщения и выход. |
| --output -o | Формат выходных данных.  Допустимые значения\: json, jsonc, table, tsv.  Значение по умолчанию\: json. |
| --query | Строка запроса JMESPath. Дополнительные сведения см. на сайте http\://jmespath.org. |
| --verbose | Повышение уровня детализации ведения журнала. Используйте параметр --debug, чтобы получить полные журналы отладки. |

## <a name="sfctl-property-list"></a>sfctl property list
Возвращает информацию обо всех свойствах Service Fabric с указанным именем.

Имя Service Fabric может иметь одно или несколько именованных свойств для хранения пользовательской информации. Эта операция возвращает информацию об этих свойствах в списке, разбитом на страницы. Информация включает имя, значение и метаданные о каждом из свойств.

### <a name="arguments"></a>Аргументы

|Аргумент|Описание|
| --- | --- |
| --name-id [обязательный параметр] | Имя Service Fabric без указания схемы универсального кода ресурса (URI) 'fabric\:'. |
| --continuation-token | Параметр маркера продолжения используется для получения следующего набора результатов. Маркер продолжения с непустым значением добавляется в ответ API, когда результаты из системы не помещаются в одном ответе. Когда это значение передается в следующем вызове API, API возвращает следующий набор результатов. Если дальнейшие результаты отсутствуют, маркер продолжения не будет содержать значения. Значение этого параметра не должно быть указано в формате URL-адреса. |
| --include-values | Позволяет указывать, следует ли включать значения возвращаемых свойств. Если значения необходимо возвращать с метаданными, устанавливается значение true. Чтобы возвращать только метаданные свойства, устанавливается значение false. |
| --timeout -t | Время ожидания сервера для выполнения операции в секундах. Это время ожидания указывает период времени, в течение которого клиент желает ожидать завершения запрошенной операции. Значение по умолчанию для этого параметра — 60 секунд.  Значение по умолчанию\: 60. |

### <a name="global-arguments"></a>Глобальные аргументы

|Аргумент|Описание|
| --- | --- |
| --debug | Увеличение уровня детализации ведения журнала для отображения всех журналов отладки. |
| --help -h | Отображение этого справочного сообщения и выход. |
| --output -o | Формат выходных данных.  Допустимые значения\: json, jsonc, table, tsv.  Значение по умолчанию\: json. |
| --query | Строка запроса JMESPath. Дополнительные сведения см. на сайте http\://jmespath.org. |

## <a name="sfctl-property-put"></a>sfctl property put
Создает или обновляет свойство Service Fabric.

Создает или обновляет указанное свойство Service Fabric с указанным именем.

### <a name="arguments"></a>Аргументы

|Аргумент|Описание|
| --- | --- |
| --name-id [обязательный параметр] | Имя Service Fabric без указания схемы универсального кода ресурса (URI) 'fabric\:'. |
| --property-name [обязательный параметр] | Имя свойства Service Fabric. |
| --value [обязательный параметр] | Описывает значение свойства Service Fabric. Это строка JSON. <br><br> Строка json имеет два поля данных, "Вид" и "Значение". Значение "Вид" должно быть первым элементом в строке JSON и может принимать значения "двоичное", "целое64", "двойное", "строка" или "Guid". Значение должно иметь возможность сериализации данных типов. Значения "Вид" и "Данные" необходимо указать как строки. |
| --custom-id-type | Идентификатор типа пользовательского свойства. Используя это свойство, пользователь может пометить тип значения свойства. |
| --timeout -t | Значение по умолчанию\: 60. |

### <a name="global-arguments"></a>Глобальные аргументы

|Аргумент|Описание|
| --- | --- |

| --query | Строка запроса JMESPath. Дополнительные сведения см. на сайте http\://jmespath.org. |
| --verbose | Повышение уровня детализации ведения журнала. Используйте параметр --debug, чтобы получить полные журналы отладки. |

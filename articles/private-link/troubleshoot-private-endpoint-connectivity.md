# <a name="troubleshoot-azure-private-endpoint-connectivity-problems"></a>Устранение проблем с подключением к частной конечной точке Azure

В этой статье содержится пошаговозадляенное руководство для проверки и диагностики настройки подключения Azure Private Endpoint.

Azure Private Endpoint — это сетевой интерфейс, который соединяет вас в частном порядке и надежно к частной службе ссылок. Это решение помогает защитить рабочие нагрузки в Azure, предоставляя приватное подключение к ресурсам службы Azure из виртуальной сети. Это решение эффективно приносит эти услуги в виртуальную сеть.

Ниже приведены сценарии подключения, доступные в private Endpoint:

- Виртуальная сеть из того же региона
- Региональные заглянул виртуальных сетей
- Глобально заглянул виртуальных сетей
- Клиент на месте по схемам VPN или Azure ExpressRoute

## <a name="diagnose-connectivity-problems"></a>Диагностика проблем с подключением 

Просмотрите эти шаги, чтобы убедиться, что все обычные конфигурации, как ожидается, решить проблемы подключения с вашей частной настройки конечной точки.

1. Просмотрите конфигурацию private Endpoint, просматривая ресурс.

    а. Перейти к **частным ссылка центр**.

      ![Частный центр связи](./media/private-endpoint-tsg/private-link-center.png)

    b. На левой панели выберите **частные конечные точки.**
    
      ![Частные конечные точки](./media/private-endpoint-tsg/private-endpoints.png)

    c. Фильтр и выберите частную конечную точку, которую вы хотите диагностировать.

    d. Просмотрите информацию о виртуальной сети и DNS.
     - Проверка утверждения состояния **соединения.**
     - Убедитесь, что VM имеет подключение к виртуальной сети, которая размещает частные конечные точки.
     - Проверить, что информация (копия) и частный IP-адрес назначены.
    
       ![Конфигурация виртуальной сети и DNS](./media/private-endpoint-tsg/vnet-dns-configuration.png)
    
1. Используйте [Azure Monitor,](https://docs.microsoft.com/azure/azure-monitor/overview) чтобы увидеть, течет ли данные.

    а. На частном ресурсе конечных точек выберите **Monitor**.
     - Выберите **данные в** или **данные out**. 
     - Просмотрите, текут ли данные при попытке подключения к частной конечной точке. Ожидайте задержку примерно на 10 минут.
    
       ![Проверка частной телеметрии конечных точек](./media/private-endpoint-tsg/private-endpoint-monitor.png)

1.  Используйте **устранение неполадок VM Connection** из Azure Network Watcher.

    а. Выберите клиента VM.

    b. Выберите **устранение неполадок соединения,** а затем выберите вкладку **Исходящих** подключений.
    
      ![Network Watcher - Тест исходящих соединений](./media/private-endpoint-tsg/network-watcher-outbound-connection.png)
    
    c. Выберите **«Используйте сетевой наблюдатель» для детального отслеживания соединения.**
    
      ![Сетевой наблюдатель - устранение неполадок подключения](./media/private-endpoint-tsg/network-watcher-connection-troubleshoot.png)

    d. Выберите **тест по F'DN**.
     - Вставьте F-DN из частного ресурса конечных точек.
     - Обеспечить порт. Как правило, используйте 443 для хранения Azure или Azure Cosmos DB и 1336 для S'L.

    д) Выберите **тест**и проверьте результаты теста.
    
      ![Сетевой наблюдатель - Результаты испытаний](./media/private-endpoint-tsg/network-watcher-test-results.png)
    
        
1. Разрешение DNS из результатов тестирования должно иметь тот же частный IP-адрес, присвоенный частной конечной точке.

    а. Если настройки DNS неверны, выполните следующие действия:
     - Если вы используете частную зону: 
       - Убедитесь, что клиент VM виртуальная сеть связана с частной зоной.
       - Проверьте наличие записи о частной зоне DNS. Если его не существует, создайте его.
     - Если вы используете пользовательские DNS:
       - Просмотрите пользовательские настройки DNS и проверьте правильность конфигурации DNS.
       Для руководства [см. Обзор частной точки зрения: конфигурация DNS](https://docs.microsoft.com/azure/private-link/private-endpoint-overview#dns-configuration).

    b. Если подключение не работает из-за групп сетевой безопасности (NSG) или пользовательских маршрутов:
     - Просмотрите правила исходящего NSG и создайте соответствующие правила исходящего, чтобы разрешить трафик.
    
       ![Правила исходящего NSG](./media/private-endpoint-tsg/nsg-outbound-rules.png)

1. Если соединение подтвердило результаты, проблема подключения может быть связана с другими аспектами, такими как секреты, токены и пароли на уровне приложения.
   - В этом случае просмотрите конфигурацию ресурса частной ссылки, связанного с частной конечной точкой. Для получения дополнительной [Azure Private Link troubleshooting guide](troubleshoot-private-link-connectivity.md)информации см.

1. Обратитесь в группу [поддержки Azure,](https://ms.portal.azure.com/#blade/Microsoft_Azure_Support/HelpAndSupportBlade/overview) если проблема еще не решена, а проблема подключения все еще существует.

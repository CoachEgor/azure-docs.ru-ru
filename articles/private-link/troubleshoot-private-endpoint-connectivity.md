# <a name="troubleshoot-private-endpoint-connectivity-problems"></a>Устранение неполадок с подключением к частным конечным точкам

Это руководство содержит пошаговые инструкции по проверке и диагностике настройки подключения к частной конечной точке. 

Частная конечная точка Azure — это сетевой интерфейс, который обеспечивает безопасное и надежное подключение к службе частной связи. Это решение помогает защитить рабочие нагрузки в Azure, обеспечивая частное подключение к ресурсам службы Azure из виртуальной сети. Это позволяет эффективно присоединять эти службы к виртуальной сети. 

Ниже приведены сценарии подключения, доступные с частными конечными точками. 
- Виртуальная сеть из того же региона 
- виртуальные сети с региональными соединениями
- глобальные одноранговые виртуальные сети
- клиент локально через VPN или каналы Express Route

## <a name="diagnosing-connectivity-problems"></a>Диагностика проблем с подключением 
Выполните приведенные ниже шаги, чтобы убедиться в том, что все стандартные конфигурации должны устранять проблемы с подключением к вашей частной конечной точке.

1. Просмотр конфигурации частной конечной точки путем просмотра ресурса 

    а) переход к **частному центру ссылок**

      ![Частный центр ссылок](./media/private-endpoint-tsg/private-link-center.png)

    б) в области навигации слева выберите частные конечные точки.
    
      ![Частные конечные точки](./media/private-endpoint-tsg/private-endpoints.png)

    в) Отфильтруйте и выберите закрытую конечную точку, которую требуется диагностировать.

    г) проверка виртуальной сети и сведений DNS
    
     - Проверка состояния подключения **утверждена**
     - Убедитесь, что виртуальная машина подключена к виртуальной сети, в которой размещены частные конечные точки.
     - Сведения о полном доменном имени (копия) и назначенный частный IP-адрес
    
       ![Конфигурация виртуальной сети и DNS](./media/private-endpoint-tsg/vnet-dns-configuration.png)    
    
2. Использование [**Azure Monitor**](https://docs.microsoft.com/azure/azure-monitor/overview) для просмотра данных

    а) в ресурсе частной конечной точки выберите **монитор** .
     - Выберите данные или исходящий трафик и проверьте, передаются ли данные при попытке подключения к частной конечной точке. Ожидание задержки в примерно 10 минут.
    
       ![Проверка телеметрии частной конечной точки](./media/private-endpoint-tsg/private-endpoint-monitor.png)

3. Устранение неполадок подключения виртуальной машины с помощью **наблюдателя за сетями**

    а) выберите виртуальную машину клиента.

    б) выберите раздел **Устранение неполадок подключения** , вкладка **исходящее подключение** .
    
      ![Наблюдатель за сетями — проверка исходящих подключений](./media/private-endpoint-tsg/network-watcher-outbound-connection.png)
    
    в) выберите **использовать наблюдатель за сетями для подробной трассировки подключений**
    
      ![Наблюдатель за сетями — устранение неполадок подключения](./media/private-endpoint-tsg/network-watcher-connection-troubleshoot.png)

    г) выберите **тест по полному доменному имени**
     - Вставьте полное доменное имя из ресурса частной конечной точки
     - Укажите порт (*обычно 443 для службы хранилища Azure или Cosmos, 1336 для SQL...* )

    д) нажмите кнопку **тест** и проверьте результаты теста.
    
      ![Наблюдатель за сетями — результаты теста](./media/private-endpoint-tsg/network-watcher-test-results.png)
    
        
4. Разрешение DNS от результатов теста должно иметь один и тот же частный IP-адрес, назначенный частной конечной точке

    ответ.) Если параметры DNS неверны, выполните следующие действия.
     - Использование частной зоны: 
       - Убедитесь, что виртуальная сеть виртуальной машины клиента связана с частной зоной.
       - Проверьте наличие записи Частная зона DNS зоны, создайте ее, если она не существует.
    
     - Использование настраиваемой службы DNS:
       - Проверьте параметры DNS для клиента и проверьте правильность конфигурации DNS.
       Инструкции см. в [обзоре частной конечной точки — конфигурация DNS](https://docs.microsoft.com/azure/private-link/private-endpoint-overview#dns-configuration) .

    б) в случае сбоя подключения из-за NSG/определяемые пользователем маршруты
     - Ознакомьтесь с правилами исходящих подключений NSG и создайте соответствующие исходящие правила, чтобы разрешить трафик.
    
       ![Правила исходящих подключений NSG](./media/private-endpoint-tsg/nsg-outbound-rules.png)

5. Если подключение проверено, проблемы с подключением могут быть связаны с другими аспектами, такими как секреты, токены и пароли на уровне приложения.
   - В этом случае проверьте конфигурацию ресурса частной связи, связанного с частной конечной точкой. См. [руководство по устранению неполадок в частной связи](https://docs.microsoft.com/azure/private-link/private-link-connectivity-troubleshooting). 

6. Обратитесь в службу [поддержки Azure](https://ms.portal.azure.com/#blade/Microsoft_Azure_Support/HelpAndSupportBlade/overview) , если проблема не устранена, а проблема с подключением все еще существует.

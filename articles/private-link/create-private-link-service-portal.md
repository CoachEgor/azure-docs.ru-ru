# <a name="quickstart-create-a-private-link-service-by-using-the-azure-portal"></a>Краткое руководство. Создание службы "Приватный канал" с помощью портала Azure

Служба "Приватный канал Azure" — это ваша служба, которая работает на базе Приватного канала Azure. Вы можете предоставить Приватному каналу доступ к службе или ресурсу, которые работают под управлением Load Balancer Azure (цен. категория "Стандартный"). Пользователи вашей службы могут получить доступ к ней в частном порядке через свои виртуальные сети. Из этого руководства вы узнаете, как создать службу "Приватный канал" с помощью портала Azure.

Если у вас еще нет подписки Azure, [создайте бесплатную учетную запись](https://azure.microsoft.com/free/?WT.mc_id=A261C142F), прежде чем начинать работу.

## <a name="sign-in-to-the-azure-portal"></a>Вход на портал Azure

Войдите на портал Azure по адресу https://portal.azure.com.

## <a name="create-an-internal-load-balancer"></a>Создание внутреннего балансировщика нагрузки

Сначала создайте виртуальную сеть. Затем создайте внутреннюю подсистему балансировки нагрузки для использования со службой "Приватный канал".

### <a name="create-the-virtual-network"></a>Создание виртуальной сети

В этом разделе вы создадите виртуальную сеть. Вы также создадите подсеть для размещения подсистемы балансировки нагрузки, которая обращается к службе "Приватный канал".

1. Вверху с левой стороны портала выберите **Создать ресурс** > **Сети** > **Виртуальная сеть**.

1. На панели **Создать виртуальную сеть** введите или выберите приведенные ниже значения.

   - **Name** (Имя). Введите **myVNet**.
   - **Группа ресурсов**: выберите **Создать новую**, введите **myResourceGroupLB** и нажмите кнопку **ОК**.
   - **Подсеть** > **Имя**. Введите **myBackendSubnet**.

1. Нажмите кнопку **создания**.

   ![Создание виртуальной сети](../load-balancer/media/tutorial-load-balancer-basic-internal-portal/2-load-balancer-virtual-network.png)

### <a name="create-a-standard-load-balancer"></a>Создание подсистемы балансировки нагрузки уровня "Стандартный"

С помощью портала создайте внутреннюю подсистему балансировки нагрузки (цен. категория "Стандартный"). Указанные вами имя и IP-адрес автоматически настраиваются в качестве внешнего интерфейса подсистемы балансировки нагрузки.

1. На портале вверху слева последовательно выберите **Создать ресурс** > **Сети** > **Load Balancer**.

1. На странице **Создание подсистемы балансировки нагрузки** на вкладке **Основные сведения** введите или выберите следующие значения:

    | Параметр                 | Значение                                              |
    | ---                     | ---                                                |
    | **Подписка**               | Выберите свою подписку.    |
    | **Группа ресурсов**         | В поле выберите **myResourceGroupLB**.|
    | **имя**;                   | Введите **myLoadBalancer**.                                   |
    | **Регион**         | Выберите регион **Восточная часть США 2**.                                        |
    | **Тип**          | Выберите **Внутренний**.                                        |
    | **SKU**           | Выберите **Стандартная**.                          |
    | **Виртуальная сеть**           | Выберите **myVNet**.                          |
    | **Назначение IP-адресов**              | Выберите **Статический**.   |
    | **Частный IP-адрес**|Введите адрес, который находится в диапазоне IP-адресов виртуальной сети и подсети. Например, 10.3.0.7.  |

1. Оставьте значения по умолчанию для остальных параметров, а затем нажмите кнопку **Проверить и создать**.

1. На вкладке **Проверить и создать** выберите **Создать**.

### <a name="create-standard-load-balancer-resources"></a>Создание ресурсов подсистемы балансировки нагрузки (цен. категория "Стандартный")

В этом разделе вы настраиваете параметры подсистемы балансировки нагрузки для внутреннего пула адресов и зонда работоспособности. Можно также указать правила подсистемы балансировки нагрузки.

#### <a name="create-a-back-end-pool"></a>Создание серверного пула

Этот серверный пул адресов содержит IP-адреса виртуальных сетевых карт, подключенных к подсистеме балансировки нагрузки. Этот пул позволяет распределять трафик между ресурсами. Создайте серверный пул адресов с именем **myBackendPool**, чтобы включить ресурсы для распределения трафика.

1. Выберите **Все службы** в меню слева.
1. Щелкните **Все ресурсы**, а затем в списке ресурсов выберите **myLoadBalancer**.
1. В разделе **Параметры** выберите **Серверные пулы**, а затем щелкните **Добавить**.
1. На странице **Добавление серверного пула** введите имя **myBackEndPool** для серверного пула и выберите **Добавить**.

#### <a name="create-a-health-probe"></a>Создание пробы работоспособности

Чтобы подсистема балансировки нагрузки могла отслеживать состояние ресурса, используйте пробу работоспособности. Проба работоспособности динамически добавляет или удаляет ресурсы в подсистеме балансировки нагрузки на основе результатов проверки работоспособности ресурсов.

Чтобы создать пробу работоспособности для отслеживания работоспособности ресурсов, сделайте следующее:

1. Щелкните **Все ресурсы** в меню слева, а затем в списке ресурсов выберите **MyLoadBalancer**.

1. В разделе **Параметры** выберите **Проверки работоспособности**, а затем щелкните **Добавить**.

1. На странице **Add a health probe** (Добавление пробы работоспособности) введите или выберите приведенные ниже значения.

   - **Name** (Имя). Введите **myHealthProbe**.
   - **Протокол**. Выберите **TCP**.
   - **Порт**: Введите **80**.
   - **Интервал**: Введите **15**. Значение представляет собой количество секунд между попытками выполнения пробы.
   - **Пороговое значение сбоя**: Введите **2**. Это значение количества последовательных сбоев пробы, которые произойдут, прежде чем виртуальная машина будет считаться неработоспособной.

1. Щелкните **ОК**.

#### <a name="create-a-load-balancer-rule"></a>Создание правила балансировщика нагрузки

Правило подсистемы балансировки нагрузки определяет режим распределения трафика между ресурсами. Правило позволяет определить следующее:

- Конфигурация интерфейсных IP-адресов для входящего трафика.
- Серверный пул IP-адресов для получения трафика.
- Необходимые порты источника и назначения.

Правило подсистемы балансировки нагрузки с именем **myLoadBalancerRule** прослушивает порт 80 во внешнем интерфейсе **LoadBalancerFrontEnd**. Это правило активирует перенаправление трафика к серверному пулу адресов **myBackendPool** на тот же порт 80.

Чтобы создать правило подсистемы балансировки нагрузки, сделайте следующее:

1. Щелкните **Все ресурсы** в меню слева, а затем в списке ресурсов выберите **MyLoadBalancer**.

1. В разделе **Параметры** выберите **Правила балансировки нагрузки**, а затем щелкните **Добавить**.

1. На странице **Add load-balancing rule** (Добавление правила балансировки нагрузки) введите или выберите следующие значения, если они отсутствуют:

   - **Name** (Имя). Введите **myLoadBalancerRule**.
   - **Интерфейсный IP-адрес**. Введите **LoadBalancerFrontEnd**.
   - **Протокол**. Выберите **TCP**.
   - **Порт**: Введите **80**.
   - **Серверный порт**: Введите **80**.
   - **Серверный пул**. Выберите **myBackendPool**.
   - **Зонд работоспособности**. Выберите **myHealthProbe**. 

1. Щелкните **ОК**.

## <a name="create-a-private-link-service"></a>Создание службы "Приватный канал"

В этом разделе объясняется, как создать службу "Приватный канал" за подсистемой балансировки нагрузки (цен. категория "Стандартный").

1. В верхнем левом углу страницы на портале Azure выберите **Создать ресурс** > **Сеть** > **Центр приватных каналов (предварительная версия)** . Можете также воспользоваться полем поиска на портале, чтобы найти службу "Приватный канал".

1. В окне **Центр Приватных каналов — общие сведения** > **Предоставьте доступ к своей службе, чтобы другие могли подключиться к ней** выберите **Запуск**.

1. В окне **Create a private link service — Basics** (Создание службы "Приватный канал" — основы) введите или выберите следующие значения:

    | Параметр           | Значение                                                                        |
    |-------------------|------------------------------------------------------------------------------|
    | Сведения о проекте  |                                                                              |
    | **Подписка**      | Выберите свою подписку.                                                     |
    | **Группа ресурсов**    | Выберите **myResourceGroupLB**.                                                    |
    | Сведения об экземпляре: |                                                                              |
    | **имя**;              | Введите **myPrivateLinkService**. |
    | **Регион**            | Выберите регион **Восточная часть США 2**.                                                        |

1. По завершении выберите **Next: Параметры исходящего трафика**.

1. В окне **Create a private link service — Outbound settings** (Создание службы "Приватный канал" — параметры исходящего трафика) введите или выберите следующую значения:

    | Параметр                           | Значение                                                                           |
    |-----------------------------------|---------------------------------------------------------------------------------|
    | **Load Balancer**                     | Выберите **myLoadBalancer**.                                                           |
    | **IP-адрес внешнего интерфейса подсистемы балансировки нагрузки** | Выберите IP-адрес внешнего интерфейса для **myLoadBalancer**.                                |
    | **Исходная виртуальная сеть NAT**        | Выберите **myVNet**.                                                                   |
    | **Исходная подсеть NAT**                 | Выберите **myBackendSubnet**.                                                          |
    | **Включить TCP-прокси V2**               | Выберите **Да** или **Нет** в зависимости от того, ожидает ли приложение заголовок прокси-сервера TCP версии 2. |
    | **Параметры частного IP-адреса**       | Настройте способ распределения и IP-адреса для каждого IP-адреса NAT.                  |

1. По завершении выберите **Next: Безопасность доступа**

1. В окне **Create a private link service — Access security** (Создание службы "Приватный канал" — безопасность доступа) щелкните **Видимость** и выберите **Только управление доступом на основе ролей**.
  
1. Щелкните **Далее: Теги** > **Проверить и создать** (Просмотр и создание) или выберите вкладку **Проверить и создать** вверху страницы.

1. Просмотрите сведения и щелкните **Создать**.

## <a name="clean-up-resources"></a>Очистка ресурсов

После завершения работы со службой "Приватный канал", удалите группу ресурсов, чтобы очистить ресурсы, используемые при работе с этим кратким руководством.

1. Введите **myResourceGroupLB** в поле поиска в верхней части портала и выберите **myResourceGroupLB** в результатах поиска.
1. Выберите **Удалить группу ресурсов**.
1. В поле **Введите имя группы ресурсов** введите **myResourceGroup**.
1. Выберите команду **Удалить**.

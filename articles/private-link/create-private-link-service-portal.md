# <a name="quickstart-create-a-private-link-service-using-the-azure-portal"></a>Краткое руководство. Создание службы "Приватный канал" с помощью портала Azure

Служба "Приватный канал" Azure — это ссылка на вашу собственную службу, которая работает на базе Приватного канала Azure. Службу или ресурс, работающие за Azure Load Balancer, можно включить для доступа по Приватному каналу. Потребители службы могут получить доступ к вашей службе в частном порядке через свои собственные виртуальные сети. В этом кратком руководстве содержатся сведения о создании службы "Приватный канал" с помощью портала Azure

Если у вас еще нет подписки Azure, [создайте бесплатную учетную запись](https://azure.microsoft.com/free/?WT.mc_id=A261C142F), прежде чем начинать работу.


## <a name="sign-in-to-azure"></a>Вход в Azure

Войдите на портал Azure по адресу https://portal.azure.com.

## <a name="create-an-internal-load-balancer"></a>Создание внутренней подсистемы балансировки нагрузки

Сначала создайте виртуальную сеть, а затем внутреннюю подсистему балансировки нагрузки для использования со службой "Приватный канал" Azure.

### <a name="create-the-virtual-network"></a>Создание виртуальной сети

В этом разделе вы создадите виртуальную сеть и подсеть для размещения подсистемы балансировки нагрузки, которая используется для доступа к службе "Приватный канал".


1. Вверху с левой стороны портала выберите **Создать ресурс** > **Сети** > **Виртуальная сеть**.
   
1. На панели **Создание виртуальной сети** введите или выберите приведенные ниже значения.
   
   - **Имя**: введите **MyVNet**.
   - **Группа ресурсов**: выберите **Создать новую**, а затем введите **MyResourceGroupLB** и нажмите кнопку **ОК**. 
   - **Подсеть** > **Имя**. Введите **myBackendSubnet**.
   
1. Нажмите кнопку **Создать**.

   ![Создание виртуальной сети](../load-balancer/media/tutorial-load-balancer-basic-internal-portal/2-load-balancer-virtual-network.png)

### <a name="create-a-standard-load-balancer"></a>Создание подсистемы балансировки нагрузки уровня "Стандартный"

Создайте внутреннюю подсистему балансировки нагрузки уровня "Стандартный" с помощью портала. Имя и IP-адрес, созданный вами, автоматически настроен в качестве внешнего интерфейса подсистемы балансировки нагрузки.

1. На портале вверху слева последовательно выберите **Создать ресурс** > **Сети** > **Load Balancer**.
   
2. На вкладке **Основные сведения** страницы **Создание подсистемы балансировки нагрузки** введите или выберите следующие сведения, примите значения по умолчанию для остальных параметров и нажмите кнопку **Review + create** (Проверить и создать).

    | Параметр                 | Значение                                              |
    | ---                     | ---                                                |
    | Подписка               | Выберите свою подписку.    |    
    | Группа ресурсов         | В раскрывающемся списке выберите *MyResourceGroupLB*.|
    | Имя                   | *myLoadBalancer*                                   |
    | Регион         | Выберите регион **Восточная часть США 2**.                                        |
    | Тип          | Выберите **Внутренний**.                                        |
    | номер SKU           | Выберите **Стандартная**.                          |
    | Виртуальная сеть           | Выберите *MyVNet*.                          |    
    | Назначение IP-адресов              | Выберите **Статический**.   |
    | Частный IP-адрес|Введите адрес, который находится в диапазоне IP-адресов виртуальной сети и подсети, например *10.3.0.7*.  |

3. На вкладке **Review + create** (Проверить и создать) щелкните **Создать**. 
   

### <a name="create-standard-load-balancer-resources"></a>Создание ресурсов подсистемы балансировки нагрузки (цен. категория "Стандартный")

В этом разделе описано, как настроить параметры подсистемы балансировки нагрузки для серверного пула адресов и пробы работоспособности, а также указать правила подсистемы балансировки нагрузки.

#### <a name="create-a-backend-pool"></a>Создание внутреннего пула

Для распределения трафика между вашими ресурсами, внутренний пул адресов содержит IP-адреса виртуальных сетевых карт, подключенных к Load Balancer. Создайте внутренний пул адресов *myBackendPool*, чтобы включить ресурсы для балансировки нагрузки трафика.

1. В меню слева щелкните **Все службы**, выберите **Все ресурсы**, а затем из списка ресурсов выберите **myLoadBalancer**.
2. В разделе **Параметры** выберите **Серверные пулы**, затем щелкните **Добавить**.
3. На странице **Добавление серверного пула** введите имя *myBackEndPool* для серверного пула и выберите **Добавить**.



#### <a name="create-a-health-probe"></a>Создание пробы работоспособности

Чтобы подсистема балансировки нагрузки могла отслеживать состояние ресурса, необходимо использовать пробу работоспособности. Проба работоспособности динамически добавляет или удаляет ресурсы из балансировщика нагрузки на основе их ответа на проверки работоспособности. 

**Создание пробы работоспособности для отслеживания работоспособности ресурсов.**

1. Щелкните **Все ресурсы** в меню слева, а затем из списка ресурсов выберите **MyLoadBalancer**.
   
1. В разделе **Параметры** выберите **Проверки работоспособности**, а затем щелкните **Добавить**.
   
1. На странице **Add a health probe** (Добавление пробы работоспособности) введите или выберите приведенные ниже значения.
   
   - **Имя**: введите **MyHealthProbe**.
   - **Протокол**. В раскрывающемся списке выберите **TCP**. 
   - **Порт**: введите **80**. 
   - **Интервал**: введите **15**. Интервал — количество секунд между попытками выполнения пробы.
   - **Пороговое значение сбоя**: введите **2**. Это значение количества последовательных сбоев пробы, которые произойдут, прежде чем виртуальная машина будет считаться неработоспособной.
   
1. Щелкните **ОК**.

#### <a name="create-a-load-balancer-rule"></a>Создание правила балансировщика нагрузки

Правило подсистемы балансировки нагрузки определяет режим распределения трафика между ресурсами. Правило определяет интерфейсную конфигурацию IP-адресов для входящего трафика и серверный пул IP-адресов для приема трафика, а также требуемый порт источника и назначения. 

Правило подсистемы балансировки нагрузки с именем **MyLoadBalancerRule** прослушивает порт 80 во внешнем интерфейсе **LoadBalancerFrontEnd**. Это правило перенаправляет трафик к серверному пулу адресов **MyBackendPool** (также на порт 80). 

**Создание правила подсистемы балансировки нагрузки.**

1. Щелкните **Все ресурсы** в меню слева, а затем из списка ресурсов выберите **MyLoadBalancer**.
   
1. В разделе **Параметры** выберите **Правила балансировки нагрузки**, а затем щелкните **Добавить**.
   
1. На странице **Добавить правило балансировки нагрузки** введите или выберите следующие значения, если они отсутствуют:
   
   - **Name** (Имя). введите **MyLoadBalancerRule**.
   - **Интерфейсный IP-адрес**. Введите **LoadBalancerFrontEnd**, если этот параметр не указан.
   - **Протокол**. Выберите **TCP**.
   - **Порт**: введите **80**.
   - **Серверный порт**: введите **80**.
   - **Серверный пул**. Выберите **MyBackendPool**.
   - **Зонд работоспособности**. Выберите **MyHealthProbe**. 
   
1. Щелкните **ОК**.
   
## <a name="create-a-private-link-service"></a>Создание службы "Приватный канал"

В этом разделе вы создадите службу "Приватный канал" за стандартной подсистемой балансировки нагрузки.

1. В верхнем левом углу экрана на портале Azure выберите **Создать ресурс** > **Сеть** > **Центр приватных каналов (предварительная версия)** . "Приватный канал" также можно найти с помощью поиска на портале.

2. В окне **Private Link Center — Overview** (Центр приватных каналов — общие сведения) для варианта **"Предоставьте доступ к своей службе, чтобы другие могли подключиться к ней."** выберите "Пуск".

3. В окне **Create a private link service — Basics** (Создание службы приватного канала — Основы) введите или выберите следующую информацию:

    | Параметр           | Значение                                                                        |
    |-------------------|------------------------------------------------------------------------------|
    | **Сведения о проекте:**  |                                                                              |
    | Подписка      | Выберите свою подписку.                                                     |
    | Группа ресурсов    | Выберите *MyResourceGroupLB*                                                     |
    | **СВЕДЕНИЯ ОБ ЭКЗЕМПЛЯРЕ:** |                                                                              |
    | Имя              | Введите *myPrivateLinkService* |
    | Регион            | Выберите регион *Восточная часть США 2*                                                         |

4. По завершении выберите **Next: Параметры исходящего трафика**.

5. В окне **Create a private link service — Outbound settings** (Создание службы "Приватный канал" — Параметры исходящего трафика) введите или выберите следующую информацию:


    | Параметр                           | Значение                                                                           |
    |-----------------------------------|---------------------------------------------------------------------------------|
    | Load Balancer                     | Выберите *MyLoadBalancer*                                                           |
    | IP-адрес внешнего интерфейса Load Balancer | Выберите IP-адрес внешнего интерфейса MyLoadBalancer                                |
    | Исходная виртуальная сеть NAT        | Выберите *myVNet*                                                                   |
    | Исходная подсеть NAT                 | Выберите *myBackendSubnet*                                                          |
    | Включение прокси-сервера TCP версии 2               | Выберите Да/Нет в зависимости от того, ожидает ли приложение заголовок прокси-сервера TCP версии 2. |
    | Параметры частного IP-адреса       | Настройка способа распределения и IP-адреса для каждого протокола IP NAT                  |

6. По завершении выберите **Next: Безопасность доступа**

7. В окне **Create a private link service — Access security** (Создание службы приватного канала — Безопасность доступа) введите или выберите следующую информацию:

    | Параметр                                     | Значение                                                             |
    |---------------------------------------------|-------------------------------------------------------------------|
    | Видимость                                  | Выберите *Только управление доступом на основе ролей*         |
  
8. По завершении выберите **Next: Теги**, **Review + create** (Просмотр и создание) или выберите вкладку **Review + create** (Просмотр и создание) в верхней части страницы.

9. Просмотрите сведения и щелкните **Создать**.

## <a name="clean-up-resources"></a>Очистка ресурсов 
После завершения работы со службой "Приватный канал", удалите группу ресурсов, чтобы удалить ресурсы, используемые в этом кратком руководстве:

1. введите *myResourceGroupLB* в поле **Поиск** в верхней части портала и выберите *myResourceGroupLB* в результатах поиска. 
2. Выберите **Удалить группу ресурсов**. 
3. Введите myResourceGroup в поле **Введите имя группы ресурсов** и нажмите кнопку **Удалить**.

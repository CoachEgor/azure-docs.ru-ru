# <a name="ports-used-by-apache-hadoop-services-on-hdinsight"></a>Порты, используемые службами Apache Hadoop в HDInsight

В этом документе приводится список портов, используемых службами Apache Hadoop, работающих в кластерах HDInsight. Кроме того, в статье содержатся сведения о портах, которые используются для подключения к кластеру с помощью протокола SSH.

## <a name="public-ports-vs-non-public-ports"></a>Общедоступные и необщедоступные порты

Кластеры HDInsight на основе Linux публично разоблачают три порта в Интернете: 22, 23 и 443. Эти порты обеспечивают безопасный доступ кластера с помощью SSH и служб, выставленных по безопасному протоколу HTTPS.

HDInsight реализован несколькими виртуальными машинами Azure (кластерными узлами), работающими в виртуальной сети Azure. Из виртуальной сети вы можете получить доступ к портам, недоступным из Интернета. При подключении через SSH к головному узлам можно напрямую получить доступ к службам, работающим на кластерных узлах.

> [!IMPORTANT]  
> Если не указать виртуальную сеть Azure с помощью параметра конфигурации для HDInsight, она будет создана автоматически. Однако вы не можете присоединиться к другим машинам (например, другим виртуальным машинам Azure или машине разработки клиента) к этой виртуальной сети.

Чтобы присоединить дополнительные компьютеры к виртуальной сети, необходимо сначала создать виртуальную сеть, а затем указать ее при создании кластера HDInsight. Для получения дополнительной информации [см.](hdinsight-plan-virtual-network-deployment.md)

## <a name="public-ports"></a>Общедоступные порты

Все узлы в кластере HDInsight расположены в виртуальной сети Azure. Узлы не могут быть непосредственно доступны из Интернета. Общедоступный шлюз обеспечивает интернет-доступ к приведенным ниже портам. Они общие для всех типов кластеров HDInsight.

| Служба | Порт | Протокол | Описание |
| --- | --- | --- | --- |
| sshd |22 |SSH |Подключает клиенты к sshd на основном головном узле. Для получения дополнительной информации [см.](hdinsight-hadoop-linux-use-ssh-unix.md) |
| sshd |22 |SSH |Подключает клиенты к SSHD на граничном узле. Для получения дополнительной информации [см.](hdinsight-hadoop-linux-use-ssh-unix.md) |
| sshd |23 |SSH |Подключает клиенты к sshd на дополнительном головном узле. Для получения дополнительной информации [см.](hdinsight-hadoop-linux-use-ssh-unix.md) |
| Ambari |443 |HTTPS |Веб-интерфейс Ambari. Дополнительные сведения см. в статье [Управление кластерами HDInsight с помощью веб-интерфейса Ambari](hdinsight-hadoop-manage-ambari.md). |
| Ambari |443 |HTTPS |REST API Ambari. Дополнительные сведения см. в статье [Управление кластерами HDInsight с помощью REST API Ambari](hdinsight-hadoop-manage-ambari-rest-api.md). |
| WebHCat |443 |HTTPS |REST API HCatalog. Смотрите [Использование MapReduce с Curl](hadoop/apache-hadoop-use-mapreduce-curl.md) |
| HiveServer2 |443 |ODBC |Подключение к Hive с помощью ODBC. См. статью [Подключение Excel к Hadoop с помощью драйвера Microsoft Hive ODBC](hadoop/apache-hadoop-connect-excel-hive-odbc-driver.md). |
| HiveServer2 |443 |JDBC |Подключение к ApacheHive с помощью JDBC. Дополнительные сведения см. в статье [Отправка запросов в Apache Hive с помощью драйвера JDBC в HDInsight](hadoop/apache-hadoop-connect-hive-jdbc-driver.md). |

### <a name="authentication"></a>Аутентификация

Все общедоступные службы в Интернете должны проходить проверку подлинности.

| Порт | Учетные данные |
| --- | --- |
| 22 или 23 |Учетные данные пользователя SSH, указанные при создании кластера. |
| 443 |Имя для входа (по умолчанию — admin) и пароль, указанные при создании кластера. |

## <a name="non-public-ports"></a>Необщедоступные порты

> [!NOTE]  
> Некоторые службы доступны только в кластерах определенных типов. Например, служба HBase доступна только на кластерах типа HBase.

> [!IMPORTANT]  
> Некоторые службы могут работать только на одном головном узле одновременно. Если вы пытаетесь подключиться к службе на основном головном узле и получаете сообщение об ошибке, повторите попытку, используя вторичный головной узел.

# <a name="scenario-cluster-creation-fails-with-domainnotfound-error-in-azure-hdinsight"></a>Сценарий: Создание кластера не удается с DomainNotFound ошибка в Azure HDInsight

В этой статье описаны шаги устранения неполадок и возможные решения проблем при взаимодействии с кластерами Azure HDInsight.

## <a name="issue"></a>Проблема

Создание кластера HDI Secure (Пакет `DomainNotFound` безопасности предприятия) завершается с боем с сообщением об ошибке.

## <a name="cause"></a>Причина

Неправильные настройки DNS.

## <a name="resolution"></a>Решение

При развертывании кластеров, связанных с доменом, ИСР создает внутреннее имя пользователя и пароль в AAD DS (для каждого кластера) и соединяет все кластерные узлы к этому домену. Соединение домена осуществляется с помощью инструментов Samba. Выполните указанные ниже предварительные требования.

* Доменное имя должно быть решено через DNS.
* IP-адрес контроллеров доменов должен быть установлен в настройках DNS для виртуальной сети, где развертывается кластер.
* Если виртуальная сеть заглянула с виртуальной сетью AAD DS, то это должно быть сделано вручную.
* Если вы используете DNS-форварды, доменное имя должно быть правильно разослано в виртуальной сети.
* Политики безопасности (НСГ) не должны блокировать соединение домена.

### <a name="additional-debugging-steps"></a>Дополнительные шаги отладки

* Развертывание Windows VM в той же подсети, домен присоединиться к машине с помощью имени пользователя и пароля (это может быть сделано через пульт управления), или

* Развертывание ubuntu VM в той же подсети и домене присоединиться к машине
  * SSH в машину
  * sudo su
  * Выполнить скрипт с именем пользователя и паролем
  * Скрипт будет пинг, создать необходимые файлы конфигурации, а затем домен. Если это удастся, ваши настройки DNS хороши.

# <a name="scenario-apache-ambari-directory-alerts-in-azure-hdinsight"></a>Сценарий: Оповещение каталога Apache Ambari в Azure HDInsight

В этой статье описаны шаги устранения неполадок и возможные решения проблем при взаимодействии с кластерами Azure HDInsight.

## <a name="issue"></a>Проблема

Вы получаете ошибки от Apache Ambari, которые похожи на:

```
1/1 local-dirs have errors: [ /mnt/resource/hadoop/yarn/local : Cannot create directory: /mnt/resource/hadoop/yarn/local ]
1/1 log-dirs have errors: [ /mnt/resource/hadoop/yarn/log : Cannot create directory: /mnt/resource/hadoop/yarn/log ]
```

## <a name="cause"></a>Причина

Упомянутые каталоги из оповещения Ambari отсутствуют на пораженном узлах работника (ы).

## <a name="resolution"></a>Решение

Вручную создавайте недостающие каталоги на пораженном узлах рабочего (ы).

1. SSH к соответствующему узлам работника.

1. Получить корень `sudo su`пользователя: .

1. Рекурсивно создавайте необходимые каталоги.

1. Смена владельца и группы для этих каталогов.

    ```bash
    chown -R yarn /mnt/resource/hadoop/yarn/local
    chgrp -R hadoop /mnt/resource/hadoop/yarn/local
    chown -R yarn /mnt/resource/hadoop/yarn/log
    chgrp -R hadoop /mnt/resource/hadoop/yarn/log
    ```

1. От UI Apache Ambari отключите, а затем включите оповещение.

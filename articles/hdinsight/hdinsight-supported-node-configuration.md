# <a name="what-are-the-default-and-recommended-node-configurations-for-azure-hdinsight"></a>Что такое конфигурация узлов по умолчанию и рекомендуемых узлов для Azure HDInsight?

В этой статье рассматриваются по умолчанию и рекомендуемые конфигурации узлов для кластеров Azure HDInsight.

## <a name="default-and-minimum-recommended-node-configuration-and-virtual-machine-sizes-for-clusters"></a>По умолчанию и минимальная рекомендуемая конфигурация узла и виртуальные размеры машины для кластеров

В следующих таблицах указаны по умолчанию и рекомендуемые размеры виртуальной машины (VM) для кластеров HDInsight.  Эта информация необходима для понимания размеров VM для использования при создании скриптов PowerShell или Azure CLI для развертывания кластеров HDInsight.

Если в кластере требуется более 32 рабочих узлов, выберите размер головного узла с не менее 8 ядрами и 14 ГБ оперативной памяти. 

Единственными типами кластеров с дисками данных являются кластеры Kafka и HBase с включенной функцией Ускоренная записи. HDInsight поддерживает размеры дисков P30 и S30 в этих сценариях.

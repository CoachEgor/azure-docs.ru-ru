# <a name="performance-tiers-for-block-blob-storage"></a>Уровни производительности для хранения блоковых каблов

При развертывании предприятиями чувствительных к производительности облачных приложений важно иметь варианты экономичного хранения данных на разных уровнях производительности.

Хранение блоков Azure blob предлагает два различных уровня производительности:

- **Премиум**: оптимизирован для высоких транзакционных ставок и однозначной последовательной задержки хранилища
- **Стандарт**: оптимизирован для высокой емкости и высокой пропускной способности

Следующие соображения относятся к различным уровням производительности:

| Область |Стандартная производительность  |Премиум производительность  |
|---------|---------|---------|
|Доступность по регионам     |   Все регионы      | В [отдельных регионах](https://azure.microsoft.com/global-infrastructure/services/?products=storage)       |
|[Поддерживаемые типы учетных записей хранения](../common/storage-account-overview.md#types-of-storage-accounts)     |     Общее назначение v2, BlobStorage, Общее назначение v1    |    BlockBlobStorage     |
|Поддерживает [высокопроизводительные блобы блока](https://azure.microsoft.com/blog/high-throughput-with-azure-blob-storage/)     |    Да, при более чем 4 размерах MiB PutBlock или PutBlob     |    Да, при более чем 256 размерах KiB PutBlock или PutBlob    |
|Избыточность     |     Посмотреть [Типы учетных записей хранения](../common/storage-account-overview.md#types-of-storage-accounts)   |  В настоящее время поддерживает только локально-избыточное хранилище (LRS) и зон-redudant хранилище (ЗРС)<div role="complementary" aria-labelledby="zone-redundant-storage"><sup>1</sup></div>     |

Что касается затрат, то производительность премиум-класса обеспечивает оптимизированное ценообразование для приложений с высокими транзакционными ставками, чтобы снизить [общие затраты на хранение](https://azure.microsoft.com/blog/reducing-overall-storage-costs-with-azure-premium-blob-storage/) этих рабочих нагрузок.

## <a name="premium-performance"></a>Премиум производительность

Премиум-преоскладет блок-блок делает данные доступными с помощью высокопроизводительного оборудования. Данные хранятся на твердотельных дисках (SSD), которые оптимизированы с низкой задержкой. SSD обеспечивают более высокую пропускную выгоду по сравнению с традиционными жесткими дисками.

Премиум-хранилище производительности идеально подходит для рабочих нагрузок, требующих быстрого и последовательного времени отклика. Лучше всего для рабочих нагрузок, выполняемых во многих небольших транзакциях. Примеры рабочих нагрузок включают в себя:

- **Интерактивные рабочие нагрузки**. Эти рабочие нагрузки требуют мгновенных обновлений и обратной связи с пользователями, таких как электронная коммерция и картографические приложения. Например, в приложении электронной коммерции менее часто просматриваемые элементы, скорее всего, не кэшируются. Тем не менее, они должны быть немедленно отображены клиенту по требованию.

- **Аналитика**. В сценарии IoT каждую секунду в облако может быть перемещено множество небольших операций записи. Большие объемы данных могут быть взяты в, агрегированы для целей анализа, а затем удалены почти сразу. Высокие возможности приема премиум-блока капли хранения делают его эффективным для этого типа рабочей нагрузки.

- **Искусственный интеллект/машинное обучение (AI/ML).** AI/ML занимается потреблением и обработкой различных типов данных, таких как визуальные эффекты, речь и текст. Этот высокопроизводительный вычислительный тип рабочей нагрузки имеет дело с большими объемами данных, что требует быстрого реагирования и эффективного времени приема данных.

- **Преобразование данных**. Процессы, требующие постоянного редактирования, модификации и преобразования данных, требуют мгновенного обновления. Для точного представления данных потребители этих данных должны немедленно увидеть эти изменения.

## <a name="standard-performance"></a>Стандартная производительность

Стандартная производительность поддерживает различные [уровни доступа](storage-blob-storage-tiers.md) для хранения данных наиболее экономично. Он оптимизирован для высокой емкости и высокой пропускной способности больших наборов данных.

- **Наборы данных резервного копирования и аварийного восстановления.** Стандартная наценка данных предлагает экономически эффективные уровни, что делает его идеальным аргументом в пользу краткосрочных и долгосрочных наборов данных о возмещении последствий стихийных бедствий, вторичных резервных копий и архивирования данных соответствия требованиям.

- **Медиа-контент**. Изображения и видео часто доступны часто при первом создании и хранении, но этот тип содержимого используется реже по мере взросления. Стандартное хранилище производительности предлагает подходящие уровни для нужд медиаконтента. 

- **Массовая обработка данных**. Эти виды рабочих нагрузок подходят для стандартного хранения, поскольку они требуют экономично-эффективного хранения высокой пропускной связи, а не последовательной низкой задержки. Большие необработанные наборы данных ставятся для обработки и в конечном итоге переходят на более холодные уровни.

## <a name="migrate-from-standard-to-premium"></a>Переход от стандарта к премиуму

Вы не можете преобразовать существующую стандартную учетную запись хранения производительности в учетную запись хранения блоков с премиальной производительностью. Чтобы перейти на учетную запись хранения премиум-класса, необходимо создать учетную запись BlockBlobStorage и перенести данные в новую учетную запись. Для получения дополнительной информации [см.](storage-blob-create-account-block-blob.md)

Для копирования капли между учетными записями хранения можно использовать последнюю версию инструмента командной строки [AzCopy.](../common/storage-use-azcopy-blobs.md) Другие инструменты, такие как Azure Data Factory, также доступны для перемещения и преобразования данных.

## <a name="blob-lifecycle-management"></a>Управление жизненным циклом большого двоичного объекта

Управление жизненным циклом хранения blob предлагает богатую политику, основанную на правилах:

- **Премиум**: Данные expire в конце жизненного цикла.
- **Стандарт**: Переход данных на лучший уровень доступа и истексрок данных в конце его жизненного цикла.

Чтобы узнать больше, смотрите [Управление жизненного цикла хранения Azure Blob](storage-lifecycle-management-concepts.md).

Вы не можете перемещать данные, которые хранятся в премиум-аккаунте хранения blob между горячими, прохладными и архивными уровнями. Тем не менее, вы можете скопировать капли из учетной записи хранения blob блока на уровень горячего доступа в *другой* учетной записи. Чтобы скопировать данные на другую учетную запись, используйте [блок Put From URL](/rest/api/storageservices/put-block-from-url) API или [AzCopy v10.](../common/storage-use-azcopy-v10.md) **Блок Put From URL** API синхронно копирует данные на сервере. Вызов завершается только после того, как все данные будут перемещены из исходного местоположения сервера в место назначения.

# <a name="scalability-and-performance-targets-for-premium-page-blob-storage-accounts"></a>Цели масштабируемости и производительности для учетных записей хранения капля премиум-страниц

[!INCLUDE [storage-scalability-intro-include](../../../includes/storage-scalability-intro-include.md)]

## <a name="scale-targets-for-premium-page-blob-accounts"></a>Масштабирование целевых показателей для учетных записей премиум-страниц

Учетная запись хранения кабл-страниц премиум-класса оптимизирована для операций чтения/записи. Этот тип учетной записи хранения поддерживает неуправляемый диск для виртуальной машины Azure.

> [!NOTE]
> Корпорация Майкрософт рекомендует использовать управляемые диски с помощью виртуальных машин Azure (VMs), если это возможно. Для получения дополнительной информации об управляемых дисках смотрите [обзор хранения дисков Azure для Windows VMs.](../../virtual-machines/windows/managed-disks-overview.md)

Учетные записи хранения капля премиум-страницы имеют следующие целевые показатели масштабируемости:

| Общая емкость учетной записи                            | Общая пропускная способность для учетной записи локально избыточного хранилища                     |
| ------------------------------------------------- | --------------------------------------------------------------------------- |
| Емкость диска: 4 ТБ (индивидуальный диск)/ 35 ТБ (совокупный итог всех дисков) <br>Емкость для моментальных снимков: 10 ТБ | До 50 гигабит в секунду для входящих<sup>1</sup> и исходящих подключений<sup>2</sup> |

<sup>1</sup> Все данные (запросы), отправляемые в учетную запись хранилища.

<sup>2</sup> Все данные (отклики), получаемые из учетной записи хранения.

Премиум-аккаунт blob — это учетная запись общего назначения, настроенная для премиум-производительности. Рекомендуется учетные записи хранения v2 общего назначения.

Если вы используете учетные записи памяти капля премиум-страниц для неуправляемых дисков, и ваше приложение превышает целевые показатели масштабируемости одной учетной записи хранилища, microsoft рекомендует переходить на управляемые диски. Для получения дополнительной информации об управляемых дисках смотрите [обзор хранения дисков Azure для Windows VMs](../../virtual-machines/windows/managed-disks-overview.md) или [обзор хранения дисков Azure для Linux VMs.](../../virtual-machines/linux/managed-disks-overview.md)

Если вы не можете перейти на управляемые диски, создайте приложение для использования нескольких учетных записей хранения и раздела данных по этим учетным записям хранения. Например, если необходимо подключить диски объемом 51 ТБ к нескольким виртуальным машинам, распределите их между двумя учетными записями хранения. Максимальный размер, поддерживаемый одной учетной записью хранилища класса Premium, составляет 35 ТБ. Убедитесь, что в одной учетной записи хранения производительности премиум-класса никогда не будет более 35 ТБ подготовленных дисков.

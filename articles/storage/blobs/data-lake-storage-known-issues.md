# <a name="known-issues-with-azure-data-lake-storage-gen2"></a>Известные проблемы с Azure Data Lake Storage 2-го поколения

В этой статье перечислены функции и инструменты, которые еще не поддерживаются или только частично поддерживаются с учетными записями хранения, имеющих иерархического пространства имен (Gen2 хранилища Озера данных Azure).

<a id="blob-apis-disabled" />

## <a name="blob-storage-apis"></a>API хранилища BLOB-объектов

API-интерфейсы хранилища BLOB-объектов будут отключены для предотвращения проблем работу функции, которые могут возникнуть, так как интерфейсы API службы хранилища больших двоичных объектов еще не может взаимодействовать с API-интерфейсы Gen2 Озера данных Azure.

### <a name="what-to-do-with-existing-tools-applications-and-services"></a>Что делать с существующими средствами, приложений и служб

Если любой из этих использования API-интерфейсы больших двоичных объектов и вы хотите использовать их для работы с все содержимое, отправляемое в вашей учетной записи, пока не станут может взаимодействовать с API-интерфейсы Gen2 Озера данных Azure BLOB-объектов API-интерфейсы не включите иерархического пространства имен в вашей учетной записи хранилища BLOB-объектов.

С помощью учетной записи хранения без иерархическое пространство имен означает, что затем у вас нет доступа к некоторым функциям Gen2 хранилища Озера данных, таких как каталоги и файлы системный список управления доступом.

### <a name="what-to-do-with-unmanaged-virtual-machine-vm-disks"></a>Что делать с неуправляемых дисков виртуальной машины (VM)

Они зависят от Отключенные интерфейсы API хранилища BLOB-объектов, поэтому если вы хотите включить иерархическое пространство имен на учетную запись хранения, рассмотрите возможность помещения их в учетную запись хранения, у которых включена функция иерархического пространства имен.

### <a name="what-to-do-if-you-used-blob-apis-to-load-data-before-blob-apis-were-disabled"></a>Что делать, если вы использовали API-интерфейсы больших двоичных объектов для загрузки данных, прежде чем API-интерфейсы больших двоичных объектов были отключены

Если вы использовали эти API-интерфейсы для загрузки данных, прежде чем они были отключены, и существует требование рабочей среды для доступа к этим данным, обратитесь в службу поддержки Майкрософт, предоставив следующие сведения:

> [!div class="checklist"]
> * Идентификатор подписки (идентификатор GUID, а не имя).
> * Имена учетной записи хранения.
> * Ли вы активно затрагиваются в рабочей среде и если да, то какие учетные записи хранения?
> * сведения о том, необходимо ли копировать данные в другую учетную запись хранения и, если это так, то по каким причинам (укажите эти данные, даже если вы не были активно вовлечены в производство).

В таком случае мы можем восстановить доступ к API BLOB-объектов на ограниченный период времени, чтобы вы смогли скопировать эти данные в учетную запись хранения, в которой не включена функция поддержки иерархического пространства имен.

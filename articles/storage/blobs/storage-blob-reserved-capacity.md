# <a name="optimize-costs-for-blob-storage-with-reserved-capacity"></a>Optimize costs for Blob storage with reserved capacity (Оптимизация затрат на хранилище BLOB-объектов с зарезервированной емкостью)

Вы можете сэкономить на расходах на хранение данных с зарезервированной емкостью Azure Storage. Зарезервированная емкость Azure Storage предлагает вам скидку на емкость для блокирования капли и данные Azure Data Lake Storage Gen2 в стандартных учетных записях хранения данных, когда вы берете на себя обязательство резервировать в течение одного года или трех лет. Резервирование предусматривает фиксированное количество емкости для хранения на срок резервирования.

Зарезервированная емкость Хранилища Azure может значительно снизить затраты на емкости для блокирования капли и данных Azure Data Lake Storage Gen2. Достигнутая экономия зависит от продолжительности бронирования, общей емкости, которую вы выбираете для резервирования, а также уровня доступа и типа избыточности, выбранной для учетной записи хранения данных. Зарезервированная емкость предоставляет скидку на выставление счетов и не влияет на состояние ресурсов хранения Azure.

Сведения о ценах на резервирование службы хранилища Azure см. в статьях [Цены на хранение блочных BLOB-объектов](https://azure.microsoft.com/pricing/details/storage/blobs/) и [Цены на Azure Data Lake Storage 2-го поколения](https://azure.microsoft.com/pricing/details/storage/data-lake/).

## <a name="reservation-terms-for-azure-storage"></a>Условия бронирования для хранения Azure

В следующих разделах описаны условия резервирования хранилища Azure.

### <a name="reservation-capacity"></a>Емкость резервирования

Зарезервированную емкость хранилища Azure можно в единицах по 100 ТБ и 1 ПБ в месяц на один год или три года.

### <a name="reservation-scope"></a>Область резервирования

Зарезервированная емкость хранилища Azure доступна для одной подписки или для нескольких подписок (общий объем). При применительно к одной подписке скидка на бронирование применяется только к выбранной подписке. При распределении нескольких подписок скидка на бронирование распределяется между этими подписками в контексте выставления счетов клиента.

При покупке зарезервированной емкости хранилища Azure можно использовать резервирование как для блокирования капли, так и для данных Azure Data Lake Storage Gen2. Резервирование применяется к вашему использованию в пределах приобретенной области и не может быть ограничено определенной учетной записью хранения, контейнером или объектом в рамках подписки. Бронирование не может быть разделено на несколько подписок.

Резервирование хранилища Azure охватывает только объем данных, хранящихся в группе ресурсов подписки или общих ресурсах. Досрочное удаление, операции, пропускная способность и плата за передачу данных не включены в резервирование. Как только вы покупаете бронирование, плата за емкости, которая соответствует атрибутам бронирования, взимается по сниженным ставкам, а не по тарифам с оплатой по мере покупки. Для получения дополнительной информации о бронировании Azure [см.](/azure/billing/billing-save-compute-costs-reservations)

### <a name="supported-account-types-tiers-and-redundancy-options"></a>Поддерживаемые типы учетных записей, уровни и параметры избыточности

Зарезервированная емкость Хранилища Azure доступна для ресурсов в стандартных учетных записях хранения данных, включая учетные записи общего назначения v2 (GPv2) и учетные записи данных blob.

Все уровни доступа (горячие, прохладные и архивные) поддерживаются для бронирования. Для получения дополнительной информации [Azure Blob storage: hot, cool, and archive access tiers](storage-blob-storage-tiers.md)о уровнях доступа см.

Все виды избыточности поддерживаются для резервирования. Для получения дополнительной информации о [Azure Storage redundancy](../common/storage-redundancy.md)вариантах избыточности см.

> [!NOTE]
> Зарезервированная емкость Хранилища Azure недоступна для учетных записей премиум-памяти, учетных записей хранения v1 (GPv1 общего назначения), хранения данных Azure Data Lake Data Gen1, полосок, хранилища Azure Queue, хранилища таблицы Azure или файлов Azure.  

### <a name="security-requirements-for-purchase"></a>Требования к безопасности при покупке

Для покупки зарезервированной емкости:

- Вы должны быть в роли **владельца,** по крайней мере, для одного предприятия или индивидуальной подписки с оплатой по мере использования ставок.
- Для корпоративных подписок **добавление зарезервированных инстанций** должно быть включено на портале EA. Или, если этот параметр отключен, необходимо быть администратором подписки EA.
- В программе поставщика облачных решений (CSP) только агенты администраторов или агенты по продажам могут приобрести зарезервированную емкость хранилища Azure Blob.

## <a name="determine-required-capacity-before-purchase"></a>Определение требуемой емкости перед покупкой

При покупке резервирования хранилища Azure необходимо выбрать регион, уровень доступа и опцию избыточности для бронирования. Ваше бронирование действительно только для данных, хранящихся в этом регионе, уровня доступа и уровня избыточности. Например, предположим, что вы покупаете резервирование для данных на западе США для горячего уровня с использованием резерва-избыточного хранилища (ЗРС). Вы не можете использовать одно и то же резервирование для данных на Востоке США, данных на уровне архива или данных в геоизмовом хранилище (GRS). Тем не менее, вы можете приобрести другое бронирование для ваших дополнительных нужд.  

Бронирование доступно сегодня для 100 ТБ или 1 БЛОК блоков, с более высокими скидками на 1 блок ПБ. При покупке бронирования на портале Azure корпорация Майкрософт может предоставить вам рекомендации, основанные на предыдущем использовании, чтобы определить, какое бронирование следует приобрести.

## <a name="purchase-azure-storage-reserved-capacity"></a>Зарезервированная емкость хранилища Azure

Зарезервированную емкость хранилища Azure можно через [портал Azure.](https://portal.azure.com) Платите за резервирование наперед или ежемесячными платежами. Для получения дополнительной информации о покупке с ежемесячными [платежами, см.](/azure/billing/billing-monthly-payments-reservations)

Для получения помощи в определении условий [Understand the Azure Storage reserved capacity discount](../../cost-management-billing/reservations/understand-storage-charges.md)бронирования, подходящих для сценария, см.

Если вы еще не создали ресурсы хранения Azure, резервирование будет применяться всякий раз, когда вы создаете ресурс, который соответствует условиям резервирования. В любом случае срок бронирования начинается сразу после успешной покупки.

## <a name="exchange-or-refund-a-reservation"></a>Обмен или возврат бронирования

Вы можете обменять или вернуть бронирование, с определенными ограничениями. Эти ограничения описаны в следующих разделах.

Чтобы обменять или вернуть бронирование, перейдите к деталям бронирования на портале Azure. Выберите **Exchange** или **Refund,** и следуйте инструкциям, чтобы отправить запрос на поддержку. Когда запрос будет обработан, корпорация Майкрософт вышлет вам электронное письмо для подтверждения завершения запроса.

Для получения дополнительной информации о политиках бронирования Azure [см.](../../cost-management-billing/reservations/exchange-and-refund-azure-reservations.md)

### <a name="exchange-a-reservation"></a>Обмен резервированием

Обмен бронирования позволяет получить распределенный возврат на основе неиспользованной части бронирования. Затем вы можете применить возврат к покупной цене нового бронирования хранилища Azure.

Количество обменов, которые вы можете сделать, не ограничено. Кроме того, нет никакой платы, связанной с обменом. Новое бронирование, которое вы покупаете, должно быть равной или большей стоимости, чем протированный кредит от первоначального бронирования. Бронирование хранилища Azure можно обменять только на другое бронирование хранилища Azure, а не на бронирование для любой другой службы Azure.

### <a name="refund-a-reservation"></a>Возврат бронирования

Вы можете отменить бронирование хранилища Azure в любое время. При отмене, вы получите переоцененный возврат на основе оставшегося срока бронирования, минус 12 процентов досрочного прекращения плату. Максимальный возврат в год составляет $50,000.

Отмена бронирования немедленно завершает бронирование и возвращает оставшиеся месяцы корпорации Майкрософт. Оставшийся проложенный остаток, за вычетом платы, будет возвращен вашей первоначальной форме покупки.

## <a name="expiration-of-a-reservation"></a>Срок действия бронирования

По истечении срока действия резервирования любой емкость юрисобицы, используемая в рамках этого бронирования, оплачивается по тарифу pay-as-you go. Резервирования не продлеваются автоматически.

Вы получите уведомление по электронной почте за 30 дней до истечения срока бронирования, и снова на дату истечения срока действия. Чтобы продолжить использование экономии средств, предоставляемой бронированием, продлите ее не позднее срока действия.

# <a name="configure-customer-managed-keys-for-azure-storage-encryption-from-the-azure-portal"></a>Настройка управляемых пользователем ключей для шифрования службы хранилища Azure на портале Azure

[!INCLUDE [storage-encryption-configure-keys-include](../../../includes/storage-encryption-configure-keys-include.md)]

В этой статье показано, как настроить хранилище ключей с помощью управляемых клиентом ключей с помощью [портала Azure](https://portal.azure.com/). Узнайте, как создать хранилище ключей с помощью портала Azure, см. в разделе [краткое руководство: Задание и получение секрета из хранилища ключей Azure с помощью портала Azure](../../key-vault/quick-create-portal.md). 


> [!IMPORTANT]
> С помощью управляемых клиентом ключей с помощью шифрования службы хранилища Azure требует, чтобы хранилище ключей два обязательных свойства настройки **обратимое удаление** и **не следует очистить**. Эти свойства включены по умолчанию при создании нового хранилища ключей на портале Azure. Тем не менее если необходимо включить эти свойства в существующем хранилище ключей, необходимо использовать PowerShell или интерфейса командной строки Azure.

## <a name="enable-customer-managed-keys"></a>Включение управляемых клиентом ключей

Чтобы включить управляемые клиентом ключи, на портале Azure, выполните следующие действия.

1. Войдите в свою учетную запись хранения.
1. В колоне **параметров** для учетной записи хранения выберите **Шифрование**. Выберите параметр **Использовать собственный ключ**, как показано на следующем рисунке.

    ![Снимок экрана портала с параметром шифрования](./media/storage-encryption-keys-portal/ssecmk1.png)

## <a name="specify-a-key"></a>Укажите ключ

После включения управляемые клиентом ключи, вы получите возможность указать ключ должен быть сопоставлен учетной записи хранения.

### <a name="specify-a-key-as-a-uri"></a>Указание ключа в качестве URI

Чтобы указать ключ как URI, выполните следующие действия:

1. Чтобы найти URI ключа на портале Azure, перейдите к хранилищу ключей и выберите **ключи** параметр. Выберите желаемый ключ, а затем нажмите клавишу, чтобы просмотреть его параметры. Скопируйте значение **идентификатор ключа** поля, который предоставляет URI.

    ![Снимок экрана: отображение ключевой URI для хранилища ключей](media/storage-encryption-keys-portal/key-uri-portal.png)

1. В **шифрования** выберите параметры для учетной записи хранения, **введите URI ключа** параметр.
1. В поле **Ключевой URI** укажите URI.

   ![Снимок экрана, показывающий, как введите URI ключа](./media/storage-encryption-keys-portal/ssecmk2.png)

### <a name="specify-a-key-from-a-key-vault"></a>Указание ключа из хранилища ключей

Чтобы указать ключ из хранилища ключей, сначала убедитесь, что у вас есть хранилище ключей, который содержит ключ. Чтобы указать ключ из хранилища ключей, выполните следующие действия:

1. Выберите параметр **Выбрать в Key Vault**.
2. Выберите хранилище ключей, содержащее ключ, который вы хотите использовать.
3. Выберите ключ из хранилища ключей.

   ![Снимок экрана, показывающий управляемых пользователем ключей параметр](./media/storage-encryption-keys-portal/ssecmk3.png)

## <a name="update-the-key-version"></a>Обновление версии ключа

При создании новой версии ключа, вам потребуется обновить учетную запись хранения для использования новой версии. Выполните следующие действия.

1. Перейдите к своей учетной записи хранения и отображения **шифрования** параметры.
1. Укажите URI для новой версии ключа. Кроме того можно выбрать хранилище ключей и ключ еще раз, чтобы обновить версию.

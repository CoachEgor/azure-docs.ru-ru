# <a name="storage-account-container-recovery"></a>Восстановление контейнера учетной записи хранения

Служба хранилища Azure обеспечивает устойчивость данных благодаря автоматическим репликам. Однако это не может помешать коду приложения и пользователям повредить данные, случайно или намеренно. Чтобы сохранить точность данных в случае ошибок приложения или пользователя, требуются более сложные приемы, например копирование данных в дополнительное хранилище и ведение журнала аудита.

## <a name="checking-azure-storage-account-type"></a>Проверка типа учетной записи хранилища Azure

1. Перейдите на [портал Azure](https://portal.azure.com/).

2. Перейдите к учетной записи хранения.

3. В разделе **Обзор** проверьте **репликацию.**

   ![Изображение](media/storage-account-container-recovery/1.png)

4. Если тип репликации **является GRS/RA-GRS,** восстановление контейнера учетной записи возможно без гарантии. Для всех остальных типов репликации это невозможно.

5. Соберите следующую информацию и подайте запрос на поддержку в службу поддержки майкрософт.

   * Имя учетной записи хранилища:
   * Название контейнера:
   * Время удаления:

   В следующей таблице приведен обзор объема восстановления контейнеров учета хранения в зависимости от стратегии репликации.

   |Тип содержимого|LRS|ZRS|GRS|РА - ГРС| 
   |---|---|---|---|---|
   |Контейнер хранилища|нет|нет|Да|Да| 

   * Мы можем попытаться восстановить контейнер для хранения счета, но без каких-либо гарантий. 

## <a name="things-not-to-do-for-recover-to-be-successful"></a>Вещи, которые не следует делать для восстановления, чтобы быть успешным

* Пожалуйста, не воссоздайте контейнер (с тем же именем).  
* Если вы уже воссоздали контейнер, необходимо удалить контейнер перед подачей запроса на восстановление поддержки.

## <a name="steps-to-prevent-this-in-the-future"></a>Шаги по предотвращению этого в будущем

1. Чтобы избежать этого в будущем, наиболее рекомендуемой функцией для использования является [Soft Delete](https://docs.microsoft.com/azure/storage/blobs/storage-blob-soft-delete).

2. Мы также рекомендуем функцию [Snapshot.](https://docs.microsoft.com/rest/api/storageservices/Creating-a-Snapshot-of-a-Blob)

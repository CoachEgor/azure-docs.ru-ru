# <a name="azcopy-login"></a>azcopy login

Входите в Active-каталог Azure для доступа к ресурсам хранения Azure.

## <a name="synopsis"></a>Краткий обзор

Войти в Active-каталог Azure для доступа к ресурсам хранения Azure.

Чтобы быть авторизованным в учетную запись хранилища Azure, необходимо назначить роль **вкладчика хранилища Blob Data** в учетную запись пользователя в контексте учетной записи хранилища, группы родительских ресурсов или родительской подписки.

Эта команда будет кэшировать зашифрованную информацию для входа для текущего пользователя с помощью встроенных механизмов ОС.

Пожалуйста, обратитесь к примерам для получения дополнительной информации.

> [!IMPORTANT]
> Если вы установите переменную среды с помощью командной строки, эта переменная будет читаема в истории командной строки. Рассмотрите возможность очистки переменных, содержащих учетные данные из истории командной строки. Чтобы не отображаться переменные в истории, можно использовать скрипт, чтобы подсказать пользователю их учетные данные и установить переменную среды.

```azcopy
azcopy login [flags]
```

## <a name="examples"></a>Примеры

Войти в интерактивно с идентификатором клиента AAD по умолчанию:

```azcopy
azcopy login
```

Войти в интерактивно с указанным идентификатором арендатора:

```azcopy
azcopy login --tenant-id "[TenantID]"
```

Войти в систему с помощью системного удостоверения виртуальной машины (VM):

```azcopy
azcopy login --identity
```

Войти в систему с помощью установленного пользователем удостоверения Личности VM и идентификатора клиента, удостоверяющего личность службы:

```azcopy
azcopy login --identity --identity-client-id "[ServiceIdentityClientID]"
```

Войти в систему с помощью установленного пользователем удостоверения VM и идентификатора объекта идентификатора службы:

```azcopy
azcopy login --identity --identity-object-id "[ServiceIdentityObjectID]"
```

Войти в систему с помощью установленного пользователем удостоверения VM и идентификатора ресурса, удостоверяющего личность службы:

```azcopy
azcopy login --identity --identity-resource-id "/subscriptions/<subscriptionId>/resourcegroups/myRG/providers/Microsoft.ManagedIdentity/userAssignedIdentities/myID"
```

Войти в качестве основного сервиса с использованием секрета клиента. Установите переменную среды AZCOPY_SPA_CLIENT_SECRET к секрету клиента для секретной службы основной auth.

```azcopy
azcopy login --service-principal
```

Войти в систему в качестве основного сервиса с помощью сертификата и пароля. Установите переменную среды AZCOPY_SPA_CERT_PASSWORD паролем сертификата для основной авторизации службы на основе сертификата.

```azcopy
azcopy login --service-principal --certificate-path /path/to/my/cert
```

Убедитесь в том, чтобы лечить /путь/к/моему/сертификату как путь к файлу PEM или PKCS12. AzCopy не попадает в магазин системного сертификата для получения сертификата.

--сертификат-путь является обязательным при выполнении сертификата на основе услуг основной auth.

## <a name="options"></a>Параметры

|Параметр|Описание|
|--|--|
|--aad-конечная точка|Конечная точка Active Directory Для использования. Значение по`https://login.microsoftonline.com`умолчанию () является правильным для общедоступного облака Azure. Установите этот параметр при аутентификации в национальном облаке. Смотрите [конечные точки проверки подлинности Azure AD.](https://docs.microsoft.com/azure/active-directory/develop/authentication-national-cloud#azure-ad-authentication-endpoints)
Этот флаг не нужен для идентификации управляемой службы.|
|--приложение-id строка|Идентификатор приложения, удостоверяющий личность пользователя. Требуется для обслуживания основной auth.|
|--сертификат-путь строки|Путь к сертификату для проверки подлинности SPN. Требуется для сертификата на основе услуги основной auth.|
|-h, --help|Отображаем содержимое справки для команды входа.|
|--идентичность|войти в систему с использованием виртуальной машины, также известной как управляемый сервисный итог (MSI).|
|--идентичность-клиент-id строка|Идентификатор клиента, удостоверяющий личность пользователя.|
|--идентичность-объект-id строка|Идентификатор объекта, удостоверяющий личность пользователя.|
|--идентичность-ресурс-id строка|Идентификатор ресурса, удостоверяющий личность, назначенный пользователем.|
|--Сервис-основной|Войдите в систему через SPN (имя главного обслуживания) с помощью сертификата или секрета. Секрет клиента или пароль сертификата должны быть помещены в соответствующую переменную среды. Введите `AzCopy env` для просмотра имен и описаний переменных среды.|
|--тенант-id строка| идентификатор драйвера активного каталога Azure для использования для интерактивного входа устройства OAuth.|

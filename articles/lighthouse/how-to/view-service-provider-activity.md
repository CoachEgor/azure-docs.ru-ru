# <a name="view-service-provider-activity"></a>Просмотр действий поставщика служб

Клиенты, делегированные подписями для управления ресурсами, делегированными Azure, могут просматривать данные [журнала Azure Activity,](../../azure-monitor/platform/platform-logs-overview.md) чтобы увидеть все выполненные действия. Это дает клиентам полную видимость операций, выполняемых поставщиками услуг через управление ресурсами, делегированных Azure, а также операции, выполняемые пользователями в рамках собственного каталога Azure Active Directory (Azure AD) клиента.

> [!TIP]
> Мы также предоставляем встроенное определение политики Azure для аудита делегирования областей управляющему арендатору. Для получения дополнительной [информации](view-manage-service-providers.md#audit-delegations-in-your-environment)см.

## <a name="view-activity-log-data"></a>Просмотр данных журнала активности

Вы можете [просмотреть журнал действий](../../azure-monitor/platform/activity-log-view.md) из меню **Монитор** на портале Azure. Чтобы ограничить результаты конкретной подпиской, используйте фильтры для выбора конкретной подписки. Вы также можете [просматривать и получать события журнала активности](../../azure-monitor/platform/activity-log-view.md) программно.

> [!NOTE]
> Пользователи в клиенте поставщика услуг могут просматривать результаты журналов активности для делегированной подписки в клиенте- клиенте, если им была предоставлена роль [читателя](../../role-based-access-control/built-in-roles.md#reader) (или другая встроенная роль, которая включает доступ к Reader), когда эта подписка была включена для управления ресурсами Azure делегированными ресурсами.

В журнале активности вы увидите название операции и ее статус, а также дату и время ее выполнения. **Событие, инициированное столбцом,** показывает, какой пользователь выполнил операцию, будь то пользователь в клиенте, деениме которого через Azure делегировал управление ресурсами, или пользователь в собственном арендаторе клиента. Обратите внимание, что отображается имя пользователя, а не арендатор или роль, которую пользователь назначил для этой подписки.

Зарегистрированная активность доступна на портале Azure за последние 90 дней. Чтобы узнать, как хранить эти данные более 90 дней, смотрите [в рабочей области «Сбор» и анализ деятельности Azure в рабочей области Log Analytics.](../../azure-monitor/platform/activity-log-collect.md)

> [!NOTE]
> Пользователи из поставщика услуг отображаются в журнале действий, но эти пользователи и их ролевые назначения не отображаются в **Control Control (IAM)** или при получении информации о назначении ролей через AI.

## <a name="set-alerts-for-critical-operations"></a>Установка оповещений для критически химопераций

Чтобы быть в курсе критически важных операций, выполняемых поставщиками услуг (или пользователями вашего собственного арендатора), мы рекомендуем создавать [оповещения о журнале активности.](../../azure-monitor/platform/activity-log-alerts.md) Например, можно отследить все административные действия для подписки или получить уведомление об удалении любой виртуальной машины в определенной группе ресурсов. При создании оповещений они будут включать действия, выполняемые пользователями как в собственном арендаторе клиента, так и в любых управляющих арендаторах.

Для получения дополнительной [информации см.](../../azure-monitor/platform/alerts-activity-log.md)

## <a name="create-log-queries"></a>Создание запросов журналов

Можно создавать запросы для анализа зарегистрированной активности или фокусировки на конкретных элементах. Например, возможно, аудит требует, чтобы вы отчитывались обо всех действиях административного уровня, выполняемых по подписке. Можно создать запрос для фильтрации только этих действий и сортировки результатов по пользователю, дате или другому значению.

Для получения дополнительной информации смотрите [обзор запросов журналов в Azure Monitor](../../azure-monitor/log-query/log-query-overview.md).

# <a name="monitor-delegated-resources-at-scale"></a>Мониторинг делегированных ресурсов в масштабе

Как поставщик услуг вы можете подключить несколько клиентов для системы делегированного управления ресурсами Azure. [Azure Lighthouse](../overview.md) позволяет поставщикам служб выполнять операции одновременно в нескольких клиентах, что делает задачи управления более эффективными.

Эта тема показывает, как использовать [журналы мониторинга Azure](../../azure-monitor/platform/data-platform-logs.md) масштабируемым способом в группах клиентов, которыми вы управляете.

## <a name="create-log-analytics-workspaces"></a>Создание рабочих областей анализа журналов

Для сбора данных необходимо создать рабочие области Log Analytics. Эти рабочие области анализа журналов являются уникальными средами для данных, собранных Azure Monitor. Каждая рабочая область имеет свои репозиторий данных и конфигурации, при этом источники данных и решения настроены для хранения данных в определенной рабочей области.

Мы рекомендуем создавать эти рабочие области непосредственно у клиентов-арендаторов. Таким образом, их данные остаются в их арендаторов, а не экспортируется в ваш. Это также позволяет централизованно контролировать любые ресурсы или службы, поддерживаемые Log Analytics, что дает вам большую гибкость в том, какие типы данных вы контролируете.

Рабочее пространство для анализа журналов можно создать с помощью [портала Azure,](../../azure-monitor/learn/quick-create-workspace.md)используя [Azure CLI](../../azure-monitor/learn/quick-create-workspace-cli.md)или с помощью [Azure PowerShell.](../../azure-monitor/learn/quick-create-workspace-posh.md)

## <a name="deploy-policies-that-log-data"></a>Развертывание политик, регистрирую данных

После создания рабочих областей анализа журналов можно развернуть [политику Azure](../../governance/policy/index.yml) в иерархиях клиентов, чтобы диагностические данные были отправлены в соответствующее рабочее пространство каждого клиента. Точные политики, которые вы развертываете, могут варьироваться в зависимости от типов ресурсов, которые требуется контролировать.

Подробнее о создании политик [см.](../../governance/policy/tutorials/create-and-manage.md) Этот [инструмент сообщества](https://github.com/Azure/Azure-Lighthouse-samples/tree/master/tools/azure-diagnostics-policy-generator) предоставляет скрипт, который поможет вам создать политики для мониторинга определенных типов ресурсов, которые вы выбираете.

Когда вы определяете, какие политики развертывать, можно [развернуть их в делегированных подписях в масштабе.](policy-at-scale.md)

## <a name="analyze-the-gathered-data"></a>Анализ собранных данных

После развертывания политики данные будут входить в рабочие области анализа журналов, созданные в каждом клиенте. Чтобы получить информацию обо всех управляемых клиентах, можно использовать такие инструменты, как [рабочие книги Azure Monitor,](../../azure-monitor/platform/workbooks-overview.md) для сбора и анализа информации из нескольких источников данных.

# <a name="azure-blockchain-tokens-composability"></a>Композитность блокчейн-токенов Azure

[!INCLUDE [Preview note](./includes/preview.md)]

Композитность токенов обеспечивает гибкость для создания токенов для продвинутых сценариев. У вас может быть сложный сценарий, который не может быть реализован с помощью [четырех предварительно построенных шаблонов маркеров.](templates.md#base-token-types) Композитность токенов позволяет создавать собственные шаблоны токенов, добавляя или удаляя определенные поведения для создания собственного шаблона токенов. При создании нового шаблона токенов Azure Blockchain Tokens проверяет все правила грамматики токенов. Составленные шаблоны сохраняются в сервисе Azure Blockchain Tokens для выпуска на подключенных блокчейн-сетях.

Для разработки шаблона маркеров можно использовать поведение маркеров в следующих [разделах.](templates.md#token-behaviors)

## <a name="burnable-b"></a>Горящий (b)

Возможность удаления токенов из поставок.

Например, при выкупить онлайн-очки кредитной карты для подарочной карты, очки кредитной карты сжигаются.

## <a name="delegable-g"></a>Делегируемый (g)

Возможность делегировать действия, сделанные на токене, которым вы владеете.

Делегат может выполнять действия в качестве владельца токена. Например, для реализации голосования можно использовать недоисполнимый токен. Делегируемый токен позволяет владельцу токенов голосования голосовать за кого-то другого голосовать от его имени.

## <a name="logable-l"></a>Войти (l)

Возможность входа.

Например, можно выпустить токен для распространения фильма в каждом кинотеатре, показывающий определенный фильм. Для воспроизведения фильма показ должен войти в транзакцию для каждого показа, поскольку выплаты роялти выплачиваются за показ во время запуска релиза фильма. Актеры строят можно использовать токены фильма для проверки выплат за фильм, показывающий в кинотеатре в распределении.

## <a name="mint-able-m"></a>Мята-способная (м)

Возможность чеканить дополнительные токены для класса токенов. Роль minter включает в себя мята поведения.

Например, розничная компания, которая хочет реализовать программу лояльности, может использовать мотируемые токены для своей программы лояльности. Они могут чеканить дополнительные очки лояльности для своих клиентов, как их клиентская база растет.  

## <a name="non-subdividable-or-whole-d"></a>Неподделные или целые (Зд)

Ограничение для предотвращения деления токена на более мелкие части.

Например, одну картину не может быть разделена на несколько меньших частей. 

## <a name="non-transferable-t"></a>Непередаваемый (t)

Ограничение для предотвращения смены владельца атокенов.

Например, университетский диплом является токеном, не поддавшимся передаче. После получения диплома выпускнику он не может быть передан от выпускника другому лицу.

## <a name="roles-r"></a>Роли (г)

Возможность определения ролей в классе шаблона маркеров для определенных поведений.

Можно предоставить список имен ролей, которые поддерживает маркер во время создания маркера. Когда роли указаны, пользователь может назначить роли для этих поведений. В настоящее время поддерживается только роль minter.

## <a name="singleton-s"></a>Синглтон (ы)

Ограничение для обеспечения поставки одного токена.

Например, музейный артефакт является токеном синглтона. Музейные экспонаты уникальны. Токен, представляющий артефакт, имеет только один элемент в поставке.

## <a name="subdividable-d"></a>Подраздельно (d)

Возможность разделить токен на более мелкие части.

Например, доллар можно разделить на центы.

## <a name="transferable-t"></a>Трансферируемый (t)

Возможность передачи права собственности на токен.

Например, право собственности является токеном, передаваемым, который может быть передан от одного лица другому при продаже имущества.

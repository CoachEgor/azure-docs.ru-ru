# <a name="supported-azure-blockchain-service-ledger-versions"></a>Поддерживаемые версии службы блокчейн Azure

Azure Blockchain Service использует базирующуюся в Ethereum [книгу Кворума,](https://www.goquorum.com/developers) предназначенную для обработки частных транзакций в группе известных участников, идентифицированных как консорциум в Azure Blockchain Service.

В настоящее время Azure Blockchain Service поддерживает [версию Кворума 2.2.3](https://github.com/jpmorganchase/quorum/releases/tag/v2.2.3) и [менеджера транзакций Tessera.](https://github.com/jpmorganchase/tessera)

## <a name="managing-updates-and-upgrades"></a>Управление обновлениями

Версия в Кворуме осуществляется через крупные, незначительные и патч-релизы. Например, если версия Кворума составляет 2.0.1, тип выпуска будет классифицирован следующим образом:

|Значительно | Minor.  | Обновление  |
| :--- | :----- | :----- |
| 2 | 0 | 1 | 

Служба блокчейн Azure автоматически обновляет выпуски патчей Кворума для существующих запущенных членов в течение 30 дней после того, как они будут доступны из Кворума.

## <a name="availability-of-new-ledger-versions"></a>Наличие новых версий книг

Служба блокчейн Azure предоставляет новейшие крупные и незначительные версии книги Кворум в течение 60 дней с момента появления у производителя Кворума. Консорциумам предоставляется не более четырех незначительных релизов при подготовке нового члена и консорциума. Обновление от майора или незначительного выпуска в настоящее время не поддерживается. Например, если вы работаете версию 2.x, обновление до версии 3.x в настоящее время не поддерживается. Аналогичным образом, если вы работаете версию 2.2, обновление до версии 2.3 в настоящее время не поддерживается.

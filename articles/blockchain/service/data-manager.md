# <a name="what-is-blockchain-data-manager-for-azure-blockchain-service"></a>Что такое Диспетчер данных Блокчейн для службы Блокчейн Azure?

Блокчейн Диспетчер данных захваты, преобразования и доставки данных транзакций службы Блокчейн Azure в службу "Сетка событий Azure", обеспечивая надежную и масштабируемую интеграцию блокчейн главной книги с службами Azure.

В большинстве сценариев Enterprise блокчейн книга блокчейн является одной частью решения. Например, для передачи ресурса из одной сущности в другую необходим механизм для отправки транзакции. Затем нужен механизм для чтения данных книги учета, чтобы убедиться, что транзакция была принята, а полученные изменения состояния затем интегрируются с комплексным решением. В этом примере, если вы напишете интеллектуальный контракт для перемещения ресурсов, вы можете использовать Блокчейн Диспетчер данных для интеграции приложений и хранилищ данных, не связанных с сцеплением. Для примера переноса ресурса, когда ресурс передается в блокчейн, изменения состояния событий и свойств доставляются Блокчейн Диспетчер данных через службу "Сетка событий". Затем можно использовать несколько возможных обработчиков событий для сетки событий, чтобы хранить блокчейн данные из сети или реагировать на изменения состояния в режиме реального времени.

Блокчейн Диспетчер данных выполняет три основные функции: захват, преобразование и доставку.

![Функции Диспетчер данных блокчейн](./media/data-manager/functions.png)

## <a name="capture"></a>Регистрация

Каждый экземпляр Диспетчер данных Блокчейн подключается к одному узлу транзакции члена службы Блокчейн Azure. Только пользователи с доступом к узлу транзакции могут создать подключение, обеспечивающее правильное управление доступом к данным клиента. Экземпляр Блокчейн Диспетчер данных надежно захватывает все данные необработанных блоков и необработанных транзакций из узла транзакции и может масштабироваться для поддержки рабочих нагрузок предприятия.

## <a name="transform"></a>Преобразование

Диспетчер данных Блокчейн можно использовать для декодирования состояния события и свойства путем настройки приложений интеллектуального контракта в Блокчейн Диспетчер данных. Чтобы добавить интеллектуальный контракт, укажите ABI и байт-код контракта. Блокчейн Диспетчер данных использует артефакты смарт-контракта для декодирования и обнаружения адресов контрактов. После добавления приложения блокчейн в экземпляр Блокчейн Диспетчер данных динамически обнаруживает адрес смарт-контракта, когда интеллектуальный контракт развертывается в консорциуме, и отправляет Раскодированное событие и состояние свойства в настроенные места назначения.

## <a name="deliver"></a>Доставка

Блокчейн Диспетчер данных поддерживает несколько разделов "Сетка событий" с исходящими подключениями для любого заданного экземпляра Блокчейн Диспетчер данных. Данные блокчейн можно отправить в одно место назначения или отправить данные блокчейн в несколько назначений. С помощью Блокчейн Диспетчер данных можно создать масштабируемое решение для публикации данных на основе событий для любого развертывания блокчейн.

## <a name="configuration-options"></a>Варианты настройки

Вы можете настроить Диспетчер данных Блокчейн в соответствии с потребностями вашего решения. Например, можно выполнить следующие действия:

* Один экземпляр Диспетчер данных Блокчейн для члена службы Блокчейн Azure.
* Экземпляр Диспетчер данных Блокчейн для каждого узла транзакции службы Блокчейн Azure. Например, узлы закрытых транзакций могут иметь собственный экземпляр Диспетчер данных Блокчейн для поддержания конфиденциальности.
* Экземпляр Диспетчер данных Блокчейн может поддерживать несколько выходных соединений. Один экземпляр Диспетчер данных Блокчейн можно использовать для управления всеми точками интеграции публикации данных для члена службы Блокчейн Azure.

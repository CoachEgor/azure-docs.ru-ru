# <a name="azure-blockchain-service-consortium"></a>Служба Azure Блокчейн-консорциума

С помощью службы Azure Blockchain, можно создать закрытый консорциума блокчейн сетей, где каждая сеть блокчейнов может быть ограничен в конкретных участников в сети. Только участники в сеть блокчейнов закрытый консорциума можно просматривать и работать с блокчейном. Консорциум сетей в службе Azure Blockchain может содержать два типа ролей участника члена:

* **Администратор** -привилегированный участников, которые могут выполнять действия управления консорциума и может участвовать в транзакциях блокчейна.

* **Пользователь** -участников, которые не предпринимать никаких действий управления consortium, но может участвовать в транзакциях блокчейна.

Консорциум сети могут быть смесью участников ролей и может иметь произвольное число каждого типа роли. Должно существовать хотя бы одного администратора.

На следующей схеме показана сеть с несколькими участниками консорциума:

![Сетевой график закрытый консорциума](./media/consortium/network-diagram.png)

С управлением консорциума в службе Azure Blockchain вы можете управлять участниками консорциума сети. Управление консорциума основано на модели консенсус сети. В текущем выпуске предварительной версии службы Azure Blockchain предоставляет модель централизованного консенсус для управления консорциума. Любой из привилегированных участников с ролью администратора может занять консорциума операциями управления, такие как добавление или удаление участников из сети.

## <a name="roles"></a>Роли

Участники в Консорциум может быть отдельным лицам или организациям и можно назначить роли пользователя или роль администратора. В следующей таблице перечислены основные различия между двумя ролями:

| Действие | роль пользователя. | роль администратора;
|--------|:----:|:------------:|
| Создание нового элемента | Да | Да |
| Приглашать новых участников | Нет | Да |
| Установка или изменение элемента роль «участник» | Нет | Да |
| Изменение элемента отображаемого имени | Только для собственных члена | Только для собственных члена |
| Удаление членов | Только для собственных члена | Да |
| Участвовать в транзакциях блокчейна | Да | Да |

### <a name="user-role"></a>роль пользователя.

Пользователи являются участниками консорциума нет возможности администратора. Они не могут участвовать в управлении членами, связанные с в рамках соглашения. Пользователи, можно изменить их отображаемое имя члена и может удалить себя из консорциумом.

### <a name="administrator"></a>Администратор

Элементы в пределах консорциума может управлять администратор. Администратор может приглашать участников, удалялись или обновление членов роли в рамках консорциума.
Всегда должен быть хотя бы одного администратора в рамках консорциумом. Последний администратор должен указать другим участником как роль администратора, перед выходом из консорциумом.

## <a name="managing-members"></a>Элементы управления

Только администраторы могут приглашать другими участниками консорциума. Администраторы пригласить участников, с помощью идентификатора подписки Azure

После приглашения, участники могут подключиться блокчейн-консорциума путем развертывания нового члена в службе Блокчейн в Azure. Чтобы просмотреть и присоединить приглашенного consortium, необходимо указать идентификатор подписки Azure, используемый в приглашении, администратором сети.

Администраторы могут удалять любой участник из consortium, включая других администраторов. Элементы можно только удалить себя из консорциумом.

## <a name="consortium-management-smart-contract"></a>Консорциум управления смарт-контрактов

Управление консорциума в службе Azure Blockchain осуществляется через консорциума управления смарт-контрактов. При развертывании нового члена блокчейн смарт-контрактов, автоматически развертываются на узлах.

Адрес управления корневого консорциума смарт-контракта можно просмотреть на портале Azure. **RootContract адрес** находится в разделе Обзор блокчейн член.

![Адрес RootContract](./media/consortium/rootcontract-address.png)

Вы можете взаимодействовать с консорциума управления смарт-контракта с помощью управления консорциума [модуль PowerShell](manage-consortium-powershell.md), Azure создан портала или напрямую с помощью смарт-контрактов с помощью службы Azure Blockchain Ethereum Учетная запись.

## <a name="ethereum-account"></a>Учетная запись Ethereum

При создании члена создается ключ учетной записи Ethereum. Служба Azure Blockchain использует ключ для создания транзакции, связанные с управлением консорциума. Ключ учетной записи Ethereum управляет Azure Blockchain служба автоматически.

Учетная запись члена можно просмотреть на портале Azure. Член является в разделе общих сведений блокчейн член.

![Владелец учетной записи](./media/consortium/member-account.png)

Вы можете сбросить учетной записи Ethereum, щелкните на вашей учетной записи и введите новый пароль. Адрес Ethereum учетной записи и пароль будут сброшены.

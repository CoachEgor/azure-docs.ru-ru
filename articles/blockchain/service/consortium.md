# <a name="azure-blockchain-service-consortium"></a>Консорциум Azure Blockchain Service

Используя Azure Blockchain Service, можно создать частные блокчейн-сети консорциума, где каждая блокчейн-сеть может быть ограничена конкретными участниками сети. Только участники блокчейн-сети частного консорциума могут просматривать и взаимодействовать с блокчейном. Консорциумные сети в Azure Blockchain Service могут содержать два типа ролей участников:

* **Администратор** - Привилегированные участники, которые могут принимать меры по управлению консорциумом и могут участвовать в блокчейн-транзакциях.

* **Пользователь** - Участники, которые не могут предпринять какие-либо действия по управлению консорциумом, но могут участвовать в блокчейн-транзакциях.

Консорциумные сети могут быть сочетанием ролей участников и могут иметь произвольное число каждого типа ролей. Должен быть хотя бы один администратор.

На следующей диаграмме показана сеть консорциумов с несколькими участниками:

![Диаграмма сети частных консорциумов](./media/consortium/network-diagram.png)

С управлением консорциумом в Azure Blockchain Service вы можете управлять участниками сети консорциумов. Управление консорциумом основано на консенсусной модели сети. В текущем выпуске предварительного просмотра Служба блокчейн Azure предоставляет централизованную модель консенсуса для управления консорциумом. Любой привилегированный участник, на котором играет роль администрирования, может предпринять действия по управлению консорциумом, такие как добавление или удаление участников из сети.

## <a name="roles"></a>Роли

Участниками консорциума могут быть физические лица или организации, и им может быть присвоена роль пользователя или роль администратора. В следующей таблице перечислены различия на высоком уровне между двумя ролями:

| Действие | роль пользователя. | роль администратора;
|--------|:----:|:------------:|
| Создание нового участника | Да | Да |
| Приглашаем новых членов | нет | Да |
| Установка или изменение роли участника | нет | Да |
| Изменение имени дисплея участника | Только для собственного члена | Только для собственного члена |
| Удаление членов | Только для собственного члена | Да |
| Участие в блокчейн-транзакциях | Да | Да |

### <a name="user-role"></a>роль пользователя.

Пользователи являются участниками консорциума без возможностей администратора. Они не могут участвовать в управлении членами, связанными с консорциумом. Пользователи могут изменить имя дисплея и удалить себя из консорциума.

### <a name="administrator"></a>Администратор

Администратор может управлять членами в рамках консорциума. Администратор может приглашать участников, удалять или обновлять роли членов в консорциуме.
В консорциуме всегда должен быть по крайней мере один администратор. Последний администратор должен указать другого участника в качестве роли администратора перед выходом из консорциума.

## <a name="managing-members"></a>Управляющие члены

Только администраторы могут пригласить других участников в консорциум. Администраторы приглашают участников, используя свой идентификатор подписки Azure.

После приглашения участники могут присоединиться к консорциуму блокчейн, развернув нового участника в Azure Blockchain Service. Чтобы просмотреть и присоединиться к приглашенному консорциуму, необходимо указать тот же идентификатор подписки Azure, который используется в приглашении администратором сети.

Администраторы могут удалить любого участника из консорциума, включая других администраторов. Участники могут только удалить себя из консорциума.

## <a name="consortium-management-smart-contract"></a>Смарт-контракт управления консорциумом

Управление консорциумом в Azure Blockchain Service осуществляется с помощью смарт-контрактов управления консорциумом. Смарт-контракты автоматически развертываются в узлах при развертывании нового участника блокчейна.

Адрес смарт-контракта управления корневым консорциумом можно посмотреть на портале Azure. **Адрес RootContract** находится в разделе обзора участника блокчейна.

![Адрес RootContract](./media/consortium/rootcontract-address.png)

Вы можете взаимодействовать с смарт-контрактом управления консорциумом с помощью [модуля powerShell,](manage-consortium-powershell.md)портала Azure или непосредственно через смарт-контракт с помощью сгенерированной Ethereum учетной записи Ethereum.

## <a name="ethereum-account"></a>Учетная запись Ethereum

При создании элемента создается ключ учетной записи Ethereum. Служба блокчейн Azure использует ключ для создания транзакций, связанных с управлением консорциумом. Ключ учетной записи Ethereum управляется службой блокчейн Azure автоматически.

Учетную запись участника можно посмотреть на портале Azure. Учетная запись участника находится в разделе обзора участника blockchain.

![Учетная запись участника](./media/consortium/member-account.png)

Вы можете сбросить учетную запись Ethereum, нажав на учетную запись участника и введя новый пароль. Адрес учетной записи Ethereum и пароль будут сброситься.

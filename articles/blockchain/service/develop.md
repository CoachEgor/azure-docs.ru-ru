# <a name="azure-blockchain-service-development-overview"></a>Обзор разработки блокчейн-сервиса Azure

С помощью Azure Blockchain Service вы можете создавать корпоративные блокчейн-сети, чтобы включить корпоративные сценарии, такие как отслеживание активов, цифровой токен, лояльность и вознаграждение, финансовую цепочку поставок и происхождение. Следующие разделы знакомят с развитием Azure Blockchain Service для реализации корпоративных блокчейн-решений.

## <a name="connecting-to-azure-blockchain-service"></a>Подключение к блокчейн-сервису Azure

Существуют различные типы клиентов для блокчейн-сетей, включая полные узлы, световые узлы и удаленных клиентов. Azure Blockchain Service создает блокчейн-сеть, которая включает в себя узлы. Вы можете использовать различных клиентов в качестве шлюза для Azure Blockchain Service для разработки блокчейна. Служба блокчейн Azure предлагает базовую аутентификацию или ключ доступа в качестве конечной точки разработки. Ниже приведены популярные клиенты, которые вы можете использовать connect.

### <a name="visual-studio-code"></a>Visual Studio Code

Вы можете подключиться к членам консорциума, используя расширение кода Visual Studio Azure Blockchain Development Kit. После подключения к консорциуму вы можете создавать и развертывать смарт-контракты в участнике консорциума службы "Блокчейн Azure".

Для разработки сложных корпоративных блокчейн-решений необходима система разработки для подключения к различным блокчейн-сетям и управления жизненными циклами смарт-контрактов. Большинство проектов взаимодействуют как минимум с двумя блокчейн-узлами. Разработчики используют локальный блокчейн во время разработки. Когда приложение готово к тестированию или выпуску, разработчик развертывается в блокчейн-сети. Например, основная публичная сеть Ethereum или служба блокчейнAzра. Комплект разработки блокчейна Azure для расширения Ethereum в Visual Studio Code использует Трюфель. Трюфель является популярной платформой разработки блокчейна для записи, компиляции, развертывания и тестирования децентрализованных приложений на блокчейнах Ethereum. Вы также можете думать о Трюфель как рамки, которая пытается легко интегрировать смарт-контракт развития и традиционной веб-разработки.

Для получения дополнительной информации [см.](connect-vscode.md)

### <a name="metamask"></a>МетаМаск

MetaMask — это кошелек на основе браузера (удаленный клиент), клиент RPC и основной исследователь контрактов. В отличие от других браузерных кошельков, MetaMask вводит экземпляр web3 в контекст браузера JavaScript, выступая в качестве клиента RPC, который подключается к различным блокчейнам Ethereum *(mainnet,* *Ropsten testnet*, *Kovan testnet*, локальный узел RPC и т.д.). Вы можете настроить пользовательский RPC легко подключиться к Azure Blockchain Service и начать разработку блокчейна с помощью Remix.

Для получения дополнительной информации [см.](connect-metamask.md)

### <a name="geth"></a>Гет

Geth — это интерфейс командной строки для запуска полного узла Ethereum, реализованного в Го. Вам не нужно запускать полный узел, но вы можете запустить интерактивную консоль, которая обеспечивает среду выполнения JavaScript, подвергая API JavaScript взаимодействовать с Azure Blockchain Service.

Для получения дополнительной информации [см.](connect-geth.md)

## <a name="ethereum-quorum-private-transactions"></a>Ethereum Кворум частных сделок

Кворум является Ethereum основе распределенного реестра протокол с транзакцией плюс конфиденциальность контракта и новые механизмы консенсуса. Ключевые усовершенствования по сравнению с Go-Ethereum включают в себя:

* **Конфиденциальность** - Кворум поддерживает частные транзакции и частные контракты через разделение государственных и частных государств и использует одноранговой зашифрованный обмен сообщениями для направленной передачи частных данных участникам сети.
* **Альтернативные механизмы консенсуса** - доказательство работы или консенсус в отношении доказательства ставки не нужны для разрешенной сети. Кворум предлагает несколько механизмов консенсуса, которые предназначены для консорциумных цепей, таких как RAFT и IBFT.Служба блокчейн Azure использует механизм консенсуса IBFT.
* **Разрешение одноранговых узлов** - узла и разрешение на использование смарт-контрактов гарантирует, что только известные стороны могут присоединиться к сети.
* **Более высокая производительность** - Кворум предлагает более высокую производительность, чем публичный Geth.

## <a name="block-explorers"></a>Исследователя блока

Исследователя блока онлайн браузеры блокчейн которые отображают индивидуальный содержание блока, данные адреса транзакций, и историю. Информация об основных блоках доступна через Azure Monitor в службе блокчейнAzра. Однако, если вам нужна более подробная информация во время разработки, блок исследователей может быть полезным.  Следующие исследователи блока работают с Azure Blockchain Service:

* [Исследователь блокчейн-сервиса Epirus Azure](https://azuremarketplace.microsoft.com/marketplace/apps/blk-technologies.azure-blockchain-explorer-template?tab=Overview) от Web3 Labs
* [BlockScout](https://github.com/Azure-Samples/blockchain/blob/master/ledger/template/ethereum-on-azure/technology-samples/blockscout/README.md)

Вы также можете создать свой собственный исследователь блоков с помощью Blockchain Data Manager и Azure Cosmos DB, [см. Tutorial: Используйте менеджер данных Blockchain для отправки данных в Azure Cosmos DB.](data-manager-cosmosdb.md)

## <a name="tps-measurement"></a>Измерение TPS

Поскольку блокчейн используется в большем количестве корпоративных сценариев, скорость транзакций в секунду (TPS) важна для того, чтобы избежать узких мест и неэффективности системы. Высокие ставки транзакций может быть трудно поддерживать в рамках децентрализованного блокчейна. Точное измерение TPS может быть затронуто различными факторами, такими как поток сервера, размер очереди транзакций, задержка сети и безопасность. Если вам нужно измерить скорость TPS во время разработки, популярным инструментом с открытым исходным кодом является [ChainHammer.](https://github.com/drandreaskrueger/chainhammer)

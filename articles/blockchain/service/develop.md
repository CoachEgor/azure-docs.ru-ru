# <a name="azure-blockchain-service-development-overview"></a>Обзор разработки Блокчейн службы Azure

С помощью службы Azure Blockchain можно создать консорциума блокчейн сетей обеспечивает enterprise таких сценариев, как учет основных фондов, цифровой маркер, лояльности и reward, финансовых цепочки поставок и provenance. Эта статья содержит введение в службы Azure Blockchain проектные и ключевые темы для реализации блокчейна для предприятия.

## <a name="client-connection-to-azure-blockchain-service"></a>Подключение клиента к службе Azure Blockchain

Существуют различные типы клиентов для блокчейн сетями, включая полный узлов, узлов света и удаленных клиентов. Azure Blockchain служба создает сеть блокчейнов, который включает в себя узлы. Можно использовать разные клиенты как шлюз в службе Azure Blockchain для разработки блокчейна. Служба Azure Blockchain предлагает обычной проверки подлинности или ключ доступа, как конечную точку разработки. Ниже перечислены популярные клиентов, можно использовать подключение.

### <a name="metamask"></a>MetaMask

MetaMask является браузерных wallet (удаленного клиента), клиент RPC и базовым контрактом обозревателя. В отличие от других wallets браузера MetaMask внедряет экземпляр web3 в контекст браузер JavaScript, действующий как клиент RPC, который подключается к различным блокчейна Ethereum (*mainnet*, *Ropsten testnet*, *Kovan testnet*, локальный узел RPC и т. д.). Можно настроить пользовательские RPC легко подключиться к службе Azure Blockchain и запустить с помощью Remix разработки блокчейна.

### <a name="geth"></a>Geth

Geth — интерфейс командной строки для запуска полных узлах Ethereum, реализованные в Go. Нет необходимости запускать полный узел, но можно запустить его интерактивная консоль, которая обеспечивает предоставление доступа к API JavaScript для взаимодействия со службой Azure Blockchain среду выполнения JavaScript.

## <a name="development-framework-configuration"></a>Конфигурации платформы разработки

Для разработки сложных промышленных блокчейн-решения, инфраструктурой разработки требуется подключаться к сетям различных блокчейн, управление жизненным циклом смарт-контрактов, автоматизировать тестирование, развертывание смарт-контрактов с помощью сценариев и Оснастите интерактивной консоли.

Truffle — это платформа разработки популярных блокчейна для записи, компиляции, развертывания и тестирования децентрализованной приложений на Ethereum блокчейна. Можно также считать из Truffle платформу, которая пытается беспрепятственной интеграции разработки смарт-контрактов и традиционной веб-разработки.

Даже небольшой проект взаимодействует с по крайней мере два узла блокчейн: Один на компьютере разработчика, а другой представляет сеть, где разработчик развертывает их приложения. Например основной общедоступной сети Ethereum или службе Блокчейн в Azure. Truffle предоставляет систему для компиляции и развертывания артефактов для каждой сети управления и делает это образом, который значительно упрощает развертывание готового приложения. Дополнительные сведения см. в разделе [ Использовать Truffle для подключения к службе Azure Blockchain сети](connect-truffle.md).

## <a name="ethereum-quorum-private-transaction"></a>Закрытый транзакции Ethereum кворума

Кворум — это протокол на базе Ethereum распределенного реестра с транзакции, а также конфиденциальности контракта и новые механизмы консенсуса. Включить ключевых усовершенствований по сравнению с Go Ethereum:

* Конфиденциальность — кворума поддерживает закрытый транзакции и частных контракты счет разделения открытого и закрытого состояния и использует обмен сообщениями зашифрованных peer-to-peer для направленного передачи личных данных, участникам сети.
* Альтернативные механизмы консенсус - без необходимости для подтверждения работы или подтверждения из воздействие согласие предоставлено разрешение сети. Кворум предлагает несколько консенсус механизмов, которые предназначены для цепочек consortium, например СОВМЕСТНОГО и IBFT.  Службы Azure Blockchain использует механизм IBFT консенсуса.

* Однорангового разрешения - узлом и узлом разрешения с помощью смарт-контрактов, обеспечивая только известные сторонам можно подключиться к сети
* Более высокая производительность - кворума предлагает более высокую производительность, чем открытый Geth

Пошаговые инструкции см. в [руководстве Отправить транзакцию с помощью службы Azure Blockchain](send-transaction.md) пример закрытого транзакции.

## <a name="block-explorers"></a>Обучающие ресурсы для блока

Блок Проводники — online блокчейн браузеров, которые отображают содержимое отдельных блоков, адрес данные транзакций и журнал. Основной блок сведения можно найти с помощью Azure Monitor в службе Azure Blockchain, однако, если требуется более подробную информацию во время разработки, обозреватели блок может оказаться полезным.  Существуют обозревателей популярных блок с открытым исходным кодом, которые можно использовать. Ниже приведен список блок обозревателей, которые работают со службой Azure Blockchain:

* [Azure Blockchain Service Explorer](https://web3labs.com/azure-offer) Web3 лабораторных занятий
* [BlockScout](https://github.com/Azure-Samples/blockchain/blob/master/ledger/template/ethereum-on-azure/technology-samples/blockscout/README.md)

## <a name="tps-measurement"></a>Измерения транзакций в Секунду

Так как блокчейн используется в различных сценариях enterprise, транзакций в второй скорость (TPS) важно избежать узких мест и повысить эффективность работы системы. Большом количестве транзакций может быть трудно поддерживать в децентрализованной блокчейна. Точного измерения транзакций в Секунду может зависеть от различных факторов, таких как поток сервера, размер очереди транзакций, задержки в сети и безопасности. Если необходимо измерить скорость транзакций в Секунду во время разработки, — это популярный инструмент открытым исходным кодом [ChainHammer](https://github.com/drandreaskrueger/chainhammer).

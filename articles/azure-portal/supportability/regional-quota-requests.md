# <a name="standard-quota-increase-limits-by-region"></a>Стандартная квота: Увеличение лимитов по регионам

Менеджер ресурсов Azure поддерживает два типа квот vCPU для виртуальных машин:

* *VMs с оплатой по мере вхром* и *зарезервированных экземплярах VM* подчиняются *стандартной квоте vCPU.*
* *Spot VMs* подлежат *спотовой квоте VCPU.*

Стандартная квота vCPU для с оплатой по мере использования и зарезервированных виртуальных машин применяется на двух уровнях для каждой подписки в каждом регионе:

* Первый уровень — это *общий региональный лимит vCPUs*во всех сериях VM.
* Второй уровень — это *лимит vCPUs серии vCPUs серии VCPUs,* например vCPUs серии D.

Всякий раз, когда вы развертываете новое место VM, общее новое и существующее использование vCPU для этой серии VM не должно превышать утвержденную квоту vCPU для этой конкретной серии VM. Кроме того, общее количество новых и существующих vCPUs, развернутых во всех сериях VM, не должно превышать общую утвержденную региональную квоту VCPU для подписки. Если какая-либо из этих квот превышена, развертывание VM не допускается.

Вы можете запросить увеличение лимита квоты VCPU для серии VM с помощью портала Azure. Увеличение квоты серии VM автоматически увеличивает общий региональный лимит VCPU на ту же сумму.

При создании новой подписки общее число региональных vCPUs по умолчанию может быть не равно общей квоте vCPU по умолчанию для всех отдельных серий VM. Это несоответствие может привести к подписке с достаточной квотой для каждой отдельной серии VM, которую вы хотите развернуть. Однако, возможно, не будет достаточной квоты для размещения всех региональных VCPUs для всех развертываний. В этом случае необходимо отправить запрос, чтобы явно увеличить лимит общего числа региональных vCPUs. Общий региональный лимит VCPU не может превышать общую утвержденную квоту по всем сериям VM для региона.

Чтобы узнать больше о стандартных квотах vCPU, [Azure subscription and service limits, quotas, and constraints](../../azure-resource-manager/management/azure-subscription-service-limits.md) [см.](../../virtual-machines/windows/quotas.md)

Чтобы узнать больше об увеличении пределов [Spot quota: Increase limits for all VM series](low-priority-quota.md)спот VM vCPU, см.

Вы можете запросить увеличение стандартного лимита квот vCPU по регионам любым из двух способов.

## <a name="request-a-quota-increase-by-region-from-help--support"></a>Запрос на увеличение квоты по регионам от справки и поддержки

Чтобы запросить увеличение квоты vCPU по регионам от **справки и поддержки:**

1. Из меню [портала Azure](https://portal.azure.com) выберите **поддержку Справки .**

   ![Ссылка "Помощь и поддержка"](./media/resource-manager-core-quotas-request/help-plus-support.png)

1. В **справке и поддержке**выберите **новый запрос поддержки.**

    ![Новый запрос на техническую поддержку](./media/resource-manager-core-quotas-request/new-support-request.png)

1. Для поля **Тип проблемы** выберите **Ограничения службы и подписки (квоты)**.

   ![Выберите тип проблемы](./media/resource-manager-core-quotas-request/select-quota-issue-type.png)

1. Для **подписки**выберите подписку, квоту которой вы хотите увеличить.

   ![Выбор подписки](./media/resource-manager-core-quotas-request/select-subscription-support-request.png)

1. Для **типа квоты**выберите **другие запросы.**

   ![Выберите тип квоты](./media/resource-manager-core-quotas-request/regional-quotatype.png)

1. Выберите **Далее: Решения** для открытия **PROBLEM DETAILS**. В **описании**, предоставить следующую информацию:

    1. Для **модели развертывания**укажите диспетчер **ресурсов.**  
    1. Для **региона,** укажите ваш необходимый регион, например, **Восток США 2**.  
    1. Для **New Limit**укажите новый лимит VCPU для региона. Это значение не должно превышать сумму утвержденных квот для отдельных серий SKU для этой подписки.

    ![Введите сведения для запроса на квоту](./media/resource-manager-core-quotas-request/regional-details.png)

1. Выберите **Обзор и создайте** для продолжения создания запроса на поддержку.

## <a name="request-a-quota-increase-by-region-from-subscriptions"></a>Запрос на увеличение квот по регионам из подписки

Чтобы запросить увеличение квоты vCPU по регионам от **подписки:**

1. На [портале Azure](https://portal.azure.com) **ищите подписки.**

   ![Перейти к подпискам на портале Azure](./media/resource-manager-core-quotas-request/search-for-subscriptions.png)

1. Выберите подписку, квоту которой вы хотите увеличить.

   ![Выберите подписку для изменения](./media/resource-manager-core-quotas-request/select-subscription-change-quota.png)

1. В левом стеле выберите **квоты использования и**использования.

   ![Следуйте использованию и квотам ссылку](./media/resource-manager-core-quotas-request/select-usage-plus-quotas.png)

1. В правом верхнем правом положении выберите **увеличение запроса.**

   ![Выберите для увеличения квоты](./media/resource-manager-core-quotas-request/request-increase-from-subscription.png)

1. Из **типа квоты**выберите **другие запросы.**

   ![Выберите тип квоты](./media/resource-manager-core-quotas-request/regional-quotatype.png)

1. Выберите **Далее: Решения** для открытия **PROBLEM DETAILS**. В поле **описания** предоставьте следующую дополнительную информацию:

    1. Для **модели развертывания**укажите диспетчер **ресурсов.**  
    1. Для **региона,** укажите ваш необходимый регион, например, **Восток США 2**.  
    1. Для **New Limit**укажите новый лимит VCPU для региона. Это значение не должно превышать сумму утвержденных квот для отдельных серий SKU для этой подписки.

    ![Введите информацию в деталях](./media/resource-manager-core-quotas-request/regional-details.png)

1. Выберите **Обзор и создайте** для продолжения создания запроса на поддержку.

# <a name="how-to-create-an-azure-support-request"></a>Создание запроса на поддержку Azure

## <a name="overview"></a>Обзор

Azure позволяет создавать и управлять запросами на поддержку, также известными как билеты поддержки. Вы можете создавать и управлять запросами на [портале Azure,](https://portal.azure.com)который описан в этой статье. Вы также можете создавать и управлять запросами программно, используя [aPI поддержки поддержки Azure REST API.](/rest/api/support)

> [!NOTE]
> URL-адрес портала Azure специфичен для облака Azure, где развернута ваша организация.
>
>* Портал Azure для коммерческого использования:[https://portal.azure.com](https://portal.azure.com)
>* Портал Azure для Германии:[https://portal.microsoftazure.de](https://portal.microsoftazure.de)
>* Портал Azure для правительства США:[https://portal.azure.us](https://portal.azure.us)
>
>

Основываясь на отзывах клиентов, мы обновили опыт запроса поддержки, чтобы сосредоточиться на трех основных целях:

* **Оптимизированный:** Упростите поиск и упрощение запроса на поддержку.
* **Интегрированный:** Вы можете легко открыть запрос на поддержку, когда вы улавливаете проблему с ресурсом Azure, не переключая контекст.
* **Эффективно:** Соберите ключевую информацию, необходимую агенту поддержки для эффективного решения проблемы.

## <a name="getting-started"></a>Начало работы

Вы можете получить **поддержку в справке и поддержке** на портале Azure. Он доступен в меню портала Azure, глобальной заголовке или меню ресурсов для службы. Прежде чем подать запрос на поддержку, необходимо иметь соответствующие разрешения.

### <a name="role-based-access-control"></a>Управление доступом на основе ролей

Чтобы создать запрос на поддержку, вы должны быть [владельцем,](../../role-based-access-control/built-in-roles.md#owner) [участником](../../role-based-access-control/built-in-roles.md#contributor) или быть назначены роли [запроса на поддержку](../../role-based-access-control/built-in-roles.md#support-request-contributor) на уровне подписки. Для создания запроса поддержки без подписки, например в сценарии Azure Active Directory (AAD), необходимо быть [админ-](../../active-directory/users-groups-roles/directory-assign-admin-roles.md)

### <a name="go-to-help--support-from-the-global-header"></a>Перейдите на помощь и поддержку от глобального заголовка

Чтобы запустить запрос на поддержку из любой точки портала Azure:

1. Выберите **?** в глобальном заголовке. Затем выберите **Справка и поддержка**.

   ![Справка и поддержка](./media/how-to-create-azure-support-request/helpandsupportnewlower.png)

2. Выберите **новый запрос на поддержку**. Следуйте запросам, чтобы предоставить нам информацию о вашей проблеме. Мы предложим некоторые возможные решения, соберем подробную информацию о проблеме и поможем вам отправить и отследить запрос на поддержку.

   ![Новый запрос на поддержку](./media/how-to-create-azure-support-request/newsupportrequest2lower.png)

### <a name="go-to-help--support-from-a-resource-menu"></a>Перейдите на помощь и поддержку из меню ресурсов

Чтобы запустить запрос на поддержку в контексте ресурса, вы в настоящее время работаете с:

1. Из меню ресурсов, в разделе **Поддержка и устранение неполадок,** выберите **новый запрос поддержки.**

   ![Контекст](./media/how-to-create-azure-support-request/incontext2lower.png)

2. Следуйте запросам, чтобы предоставить нам информацию о проблеме, с которой вы имеете. При запуске процесса запроса поддержки с ресурса некоторые параметры предварительно выбраны для вас.

## <a name="create-a-support-request"></a>Создание запроса на обслуживание

Мы проведем вас через несколько шагов, чтобы собрать информацию о вашей проблеме и помочь вам решить ее. Каждый шаг описан в следующих разделах.

### <a name="basics"></a>Основы

Первый шаг при отправке запроса на поддержку заключается в сборе сведений о возникшей проблеме и вашем плане поддержки.

На вкладке **Основы** **Нового запроса поддержки,** используйте селекторы, чтобы начать рассказывать нам о проблеме. Во-первых, вы определите некоторые общие категории для типа проблемы и выберете соответствующую подписку. Выберите сервис, например, **виртуальную машину под управлением Windows.** Выберите ресурс, например название виртуальной машины. Опишите проблему своими словами, а затем **выберите тип проблемы,** чтобы получить более конкретную информацию.

![Колонка «Основные»](./media/how-to-create-azure-support-request/basics2lower.png)

> [!NOTE]
> Azure предоставляет неограниченную поддержку управления подпиской, которая включает выставление счетов, корректировку квот и переводы счетов. Чтобы получить техническую поддержку, у вас должен быть план поддержки. [Дополнительные сведения о планах поддержки](https://azure.microsoft.com/support/plans).
>
>

### <a name="solutions"></a>Решения

После сбора основной информации, мы затем показать вам решения, чтобы попробовать по своему усмотрению. В некоторых случаях мы можем даже провести быструю диагностику. Решения написаны инженерами Azure и позволят решить наиболее распространенные проблемы.

### <a name="details"></a>Сведения

Далее мы собираем дополнительные сведения о проблеме. Предоставление подробной и подробной информации на этом этапе помогает нам направить ваш запрос на поддержку к нужному агенту.

Если возможно, сообщите нам, когда проблема началась, и любые шаги, чтобы воспроизвести его. Можно загрузить файл, например файл журнала или выход из диагностики.

После того, как мы получим всю информацию о проблеме, выберите, как получить поддержку. В разделе **метод поддержки** **Подробная информация,** выберите тяжесть воздействия. Предоставьте свой предпочтительный метод контакта, хорошее время, чтобы связаться с вами, и ваш язык поддержки.

Затем заполните раздел **Контактная информация,** чтобы мы знали, как связаться с вами.

### <a name="review--create"></a>Отзыв и создание

Заполните всю необходимую информацию на каждой вкладке, а затем выберите **Обзор и создать**. Проверьте детали, которые вы отправите в службу поддержки. Вернитесь к любой вкладке, чтобы внести изменения, если это необходимо. После удовлетворения запроса на поддержку выберите **« Создать**— это

Агент поддержки свяжется с вами с помощью указанного вами метода. Для получения информации о первоначальном времени отклика, см [область поддержки и отзывчивость](https://azure.microsoft.com/support/plans/response/).

## <a name="all-support-requests"></a>Все запросы на поддержку

Вы можете просматривать сведения и статус запросов на поддержку, перейдя **в службу помощи и поддержать** >  **все запросы поддержки.**

![Все запросы на поддержку](./media/how-to-create-azure-support-request/allrequestslower.png)

На этой странице можно отфильтровать запросы на поддержку по **подписке, Добавленной** дате (UTC) и **Status.** **Subscription** На этой странице также можно отсортировать запросы на поддержку и выполнить их поиск.

Выберите запрос поддержки для просмотра деталей, включая серьезность и ожидаемое время, необходимое агенту службы поддержки для ответа.

Если вы хотите изменить серьезность запроса, выберите **влияние на бизнес.** Выберите один из списка разъединений для присвоения.

> [!NOTE]
> Максимальный уровень серьезности зависит от плана поддержки. [Дополнительные сведения о планах поддержки](https://azure.microsoft.com/support/plans).

# <a name="standard-quota-increase-limits-by-vm-series"></a>Стандартная квота: увеличение лимитов по сериям виртуальных машин

Azure Resource Manager поддерживает два типа квот виртуальных ЦП для виртуальных машин:
* *Виртуальные машины с оплатой по мере* использования и *зарезервированные экземпляры виртуальных машин* подвергаются *стандартной виртуальных ЦП квоте*.
* На *точечные виртуальные машины* распространяется *виртуальных ЦП квота*. 

Стандартная квота виртуальных ЦП для экземпляров с оплатой по мере использования и зарезервированные экземпляры виртуальных машин применяется на двух уровнях для каждой подписки в каждом регионе:
* Первый уровень — это *Общее виртуальных ЦП ограничение региональных стандартов* (во всех рядах виртуальных машин).
* Второй уровень — это *ограничение виртуальных ЦП для серии виртуальных машин* (например, виртуальных ЦП серии Dv3). 

При развертывании новой точки на виртуальной машине общее количество новых и существующих виртуальных ЦП использования для этой серии виртуальных машин не должно превышать утвержденную квоту виртуальных ЦП для этой конкретной серии виртуальных машин. Кроме того, общее число новых и существующих виртуальных ЦП, развернутых во всех рядах виртуальных машин, не должно превышать общее количество утвержденных региональных квот виртуальных ЦП для подписки. Если одна из этих квот превышена, развертывание виртуальной машины не разрешается.

Вы можете запросить увеличение предельной квоты виртуальных ЦП для ряда виртуальных машин с помощью портал Azure. Увеличение квоты для серии виртуальных машин автоматически увеличивает общее значение региональных виртуальных ЦП на один и тот же объем. 

Дополнительные сведения о стандартных квотах виртуальных ЦП см. в статье [квоты виртуальной машины виртуальных ЦП](https://docs.microsoft.com/azure/virtual-machines/windows/quotas) и [ограничения подписки и службы Azure](https://docs.microsoft.com/azure/azure-supportability/classic-deployment-model-quota-increase-requests). 

Дополнительные сведения о увеличении ограничения виртуальных ЦП по регионам для стандартной квоты см. в разделе [Стандартная квота: увеличение ограничений по регионам](https://docs.microsoft.com/azure/azure-supportability/regional-quota-requests). 

Дополнительные сведения об увеличении ограничений виртуальных ЦП виртуальных машин см. в разделе [Квота на место: увеличение ограничений для всех рядов виртуальных машин](https://docs.microsoft.com/azure/azure-supportability/low-priority-quota).

Вы можете запросить увеличение квоты Standard виртуальных ЦП на серию виртуальных машин одним из двух способов, как описано в следующих разделах.

## <a name="request-a-standard-quota-increase-from-the-help--support-pane"></a>Запросите увеличение стандартной квоты в области "Справка и поддержка"

Чтобы запросить стандартное увеличение квоты виртуальных ЦП на серию виртуальных машин из панели " **Справка и поддержка** ", выполните следующие действия. 

> [!NOTE]
> Вы также можете запросить увеличение квоты для нескольких регионов с помощью одного варианта поддержки. Дополнительные сведения см. в шаге 8.

1. В левой области [портал Azure](https://portal.azure.com)выберите **Справка и поддержка**.

   ![Ссылка "Справка и поддержка"](./media/resource-manager-core-quotas-request/helpsupport.png)
 
1. В области **Справка и поддержка** выберите **новый запрос в службу поддержки**. 

    ![Новый запрос на техническую поддержку](./media/resource-manager-core-quotas-request/newsupportrequest.png)

1. В раскрывающемся списке **тип проблемы** выберите **пределы службы и подписки (квоты)** .

   ![Раскрывающийся список "тип проблемы"](./media/resource-manager-core-quotas-request/issuetypedropdown.png)

1. В раскрывающемся списке **Подписка** выберите подписку, для которой необходимо увеличить квоту.

   ![Раскрывающийся список "Подписка"](./media/resource-manager-core-quotas-request/select-subscription-sr.png)
   
1. В раскрывающемся списке **тип квоты** выберите **Вычисление виртуальной машины (ядер-виртуальных ЦП), чтобы увеличить лимит подписки**. 

   ![Раскрывающийся список "тип квоты"](./media/resource-manager-core-quotas-request/select-quota-type.png)

1. На вкладке **сведения** в разделе **сведения о проблеме**выберите **указать сведения**, а затем введите дополнительные сведения, которые помогут обработать запрос.

   ![Ссылка "предоставить сведения"](./media/resource-manager-core-quotas-request/provide-details.png)

1. В области **сведений о квоте** в правом верхнем углу выполните следующие действия.

   ![Область "сведения о квоте"](./media/resource-manager-core-quotas-request/1-7.png)

   а. В раскрывающемся списке **модель развертывания** выберите нужную модель.

   b. В раскрывающемся списке **расположения** выберите расположение. Для выбранного расположения в разделе **типы**в поле **выберите тип** введите **Standard**.

   ![Область "сведения о квоте" — Типы квот](./media/resource-manager-core-quotas-request/1-8.png)

   В разделе **типы**можно запросить как стандартные, так и типы квот из одного варианта поддержки с поддержкой множественного выбора.
   
   Дополнительные сведения об увеличении пределов дисковой квоты см. в статье [точки на виртуальных машинах Azure для масштабируемых наборов виртуальных машин](https://docs.microsoft.com/azure/virtual-machine-scale-sets/use-spot).

   c. Ниже раскрывающегося списка " **стандартный** " выберите серию номеров SKU, для которой необходимо увеличить квоты.

   ![Панель "сведения о квоте" — серия номеров SKU](./media/resource-manager-core-quotas-request/1-9.png)

   d. Введите новые ограничения квоты для этой подписки. Чтобы удалить номер SKU из списка, снимите флажок рядом с номером SKU или щелкните значок **Удалить** (X). 

   ![Текстовое поле "новый предел виртуальных ЦП"](./media/resource-manager-core-quotas-request/1-10.png)

1. Чтобы запросить увеличение квоты для нескольких расположений, выберите дополнительное расположение в раскрывающемся списке, а затем выберите подходящий тип виртуальной машины. Затем можно ввести ограничение, которое будет применяться к дополнительному расположению.

   ![Дополнительные расположения в области "сведения о квоте"](./media/resource-manager-core-quotas-request/1-11.png)
   
1. Выберите **сохранить и продолжить** , чтобы продолжить создание запроса в службу поддержки.

## <a name="request-a-standard-quota-increase-from-the-subscriptions-pane"></a>Запрос увеличения стандартной квоты с панели "подписки"

Чтобы запросить стандартное увеличение квоты виртуальных ЦП на серию виртуальных машин из области **подписки** , выполните следующие действия.

> [!NOTE]
> Вы также можете запросить увеличение квоты для нескольких регионов с помощью одного варианта поддержки. Дополнительные сведения см. в разделе Шаг 7.

1. В левой области [портал Azure](https://portal.azure.com)выберите **подписки**.

   ![Ссылка "подписки"](./media/resource-manager-core-quotas-request/subscriptions.png)

1. Выберите подписку, для которой необходимо увеличить квоту.

   ![Область "подписки"](./media/resource-manager-core-quotas-request/select-subscription.png)

1. В левой области страницы **\<имя подписки >** выберите **использование и квоты**.

   ![Ссылка "использование и квоты"](./media/resource-manager-core-quotas-request/select-usage-quotas.png)

1. В правом верхнем углу выберите **запросить увеличение**.

   ![Запросить увеличение](./media/resource-manager-core-quotas-request/request-increase.png)

1. В раскрывающемся списке **тип квоты** выберите **Вычисление виртуальной машины (ядер-виртуальных ЦП), чтобы увеличить лимит подписки**.

   ![Раскрывающийся список "тип квоты"](./media/resource-manager-core-quotas-request/select-quota-type.png)
   
1. В области **сведений о квоте** в правом верхнем углу выполните следующие действия.

   ![Область "сведения о квоте"](./media/resource-manager-core-quotas-request/1-1-6.png)

   а. В раскрывающемся списке **модель развертывания** выберите нужную модель.

   b. В раскрывающемся списке **расположения** выберите расположение. 
   
   c. Для выбранного расположения в разделе **типы**выберите **выбрать тип**, а затем установите флажок **стандартный** .

   ![Флажок "Стандартный"](./media/resource-manager-core-quotas-request/1-1-7.png)
   
   В разделе **типы**можно запросить тип квоты "Стандартный" и "низкий приоритет" из одного случая поддержки с поддержкой множественного выбора.
   
   Дополнительные сведения об увеличении пределов дисковой квоты см. в статье [точки на виртуальных машинах Azure для масштабируемых наборов виртуальных машин](https://docs.microsoft.com/azure/virtual-machine-scale-sets/use-spot).

   d. Ниже раскрывающегося списка " **стандартный** " выберите серию номеров SKU, для которой необходимо увеличить квоты.

   ![Панель "сведения о квоте" — серия номеров SKU](./media/resource-manager-core-quotas-request/1-1-8.png)

   д) Введите новые ограничения квоты для этой подписки. Чтобы удалить номер SKU из списка, снимите флажок рядом с номером SKU или нажмите кнопку **Удалить** (X). 

   ![Текстовое поле "новый предел виртуальных ЦП"](./media/resource-manager-core-quotas-request/1-1-9.png)
   
1. Чтобы запросить увеличение квоты для нескольких расположений, выберите дополнительное расположение в раскрывающемся списке, а затем выберите подходящий тип виртуальной машины. 

   Этот шаг предварительно загружает серию номеров SKU, выбранную для более ранних расположений. Введите пределы квот, которые необходимо применить к дополнительным рядам.
   
   ![Дополнительные расположения в области "сведения о квоте"](./media/resource-manager-core-quotas-request/1-1-10.png)
 
1. Выберите **сохранить и продолжить** , чтобы продолжить создание запроса в службу поддержки.

# <a name="quota-increase-requests"></a>Запросы на увеличение квоты

Квоты менеджера ресурсов vCPU для виртуальных машин и виртуальных наборов масштабов машин применяются на двух уровнях для каждой подписки в каждом регионе.

Первый уровень — это лимит Total Regional vCPUs во всех сериях VM. Второй уровень — это лимит vCPUs серии VM, например vCPUs серии D. Всякий раз, когда новая виртуальная машина должна быть развернута, сумма нового и существующего использования vCPUs для этой серии VM не должна превышать квоту VCPU, утвержденную для этой конкретной серии VM. Кроме того, общий объем новых и существующих подсчетов VCPU, развернутых во всех сериях VM, не должен превышать общую региональную квоту vCPUs, утвержденную для подписки. Если какая-либо из этих квот будет превышена, развертывание VM не будет разрешено.
Вы можете запросить увеличение лимита квоты vCPUs для серии VM с портала Azure. Увеличение квоты серии VM автоматически увеличивает общий лимит региональных vCPUs на ту же сумму.

При создании новой подписки общие региональные vCPUs по умолчанию могут быть не равны сумме квот по умолчанию vCPU для всех отдельных серий VM. Этот факт может привести к подписке с достаточной квотой для каждой отдельной серии VM, которую вы хотите развернуть. Она может не хватить квот для общего регионального vCPUs для всех развертываний. В этом случае необходимо отправить запрос на явное увеличение лимита общего регионального vCPUs. Общий лимит региональных vCPUs не может превышать сумму утвержденной квоты по всем сериям VM для региона.

Для получения дополнительной информации о квотах, [Azure subscription and service limits, quotas, and constraints](../../azure-resource-manager/management/azure-subscription-service-limits.md)см. [Virtual machine vCPU quotas](../../virtual-machines/windows/quotas.md)

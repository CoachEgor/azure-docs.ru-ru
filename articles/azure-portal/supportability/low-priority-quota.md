# <a name="spot-quota-increase-limits-for-all-vm-series"></a>Точечная квота: Увеличение лимитов для всех серий VM

Spot виртуальные машины (VM) обеспечивают другую модель использования Azure. Они позволяют взять на себя более низкие затраты в обмен на то, что Azure позволит удалить виртуальные машины по мере необходимости для развертывания экземпляров VM с оплатой по мере входном оплаты или резервов. Для получения дополнительной информации [Azure spot VMs for virtual machine scale sets](../../virtual-machine-scale-sets/use-spot.md)о точечных виртуальных выхвачиваемых вымотках см.

Менеджер ресурсов Azure поддерживает два типа квот vCPU для виртуальных машин:

* *VMs с оплатой по мере вхром* и *зарезервированных экземплярах VM* подчиняются *стандартной квоте vCPU.*
* *Spot VMs* подлежат *спотовой квоте VCPU.*

Для типа квот спотовых квот vCPU квоты менеджера ресурсов vCPU применяются во всех доступных виртуальных сериях машин в качестве единого регионального лимита.

Всякий раз, когда вы развертываете новое место VM, общее новое и существующее использование vCPU для всех топовых экземпляров VM не должно превышать утвержденный предел квоты vCPU. Если квота спотов не превышена, точечное развертывание VM не допускается.

В этой статье обсуждается, как запросить увеличение лимита квот спотов оговорения с помощью портала Azure.

Чтобы узнать больше о стандартных квотах vCPU, [Azure subscription and service limits, quotas, and constraints](../../azure-resource-manager/management/azure-subscription-service-limits.md) [см.](../../virtual-machines/windows/quotas.md)

Чтобы узнать об увеличении лимита [Standard quota: Increase limits by region](regional-quota-requests.md)VCPU по регионам, см.

## <a name="request-a-quota-limit-increase-from-help--support"></a>Запрос на увеличение лимита квоты от справки и поддержки

Чтобы запросить увеличение лимита спотовых квот для всех виртуальных серий машин с помощью **поддержки Справка :**

> [!NOTE]
> Вы также можете запросить увеличение лимита квот для нескольких регионов через один случай поддержки. Для получения подробной информации см.

1. Из меню [портала Azure](https://portal.azure.com) выберите **поддержку Справки .**

   ![Ссылка на справку и поддержку](./media/resource-manager-core-quotas-request/help-plus-support.png)

1. В **справке и поддержке**выберите **новый запрос поддержки.**

    ![Создание нового запроса на поддержку](./media/resource-manager-core-quotas-request/new-support-request.png)

1. Для поля **Тип проблемы** выберите **Ограничения службы и подписки (квоты)**.

   ![Выберите тип проблемы](./media/resource-manager-core-quotas-request/select-quota-issue-type.png)

1. Для **подписки**выберите подписку, квоту которой вы хотите увеличить.

   ![Выберите подписку для увеличенной квоты](./media/resource-manager-core-quotas-request/select-subscription-support-request.png)

1. Для **типа квоты,** выберите **Вычислить-VM (cores-vCPUs) ограничение подписки увеличивается.**

   ![Выберите тип квоты](./media/resource-manager-core-quotas-request/select-quota-type.png)

1. Выберите **Далее: Решения** для открытия **PROBLEM DETAILS**. Выберите **Предоставить подробную информацию** для ввода дополнительной информации.

   ![Ссылка "Предоставить подробную информацию"](./media/resource-manager-core-quotas-request/provide-details-link.png)

1. В **деталях квоты**сделайте следующие шаги:

   1. Для **модели развертывания**выберите соответствующую модель, а для **местоположения**— месторасположение.

      ![Предоставить дополнительные сведения о квотах](./media/resource-manager-core-quotas-request/quota-details-deployment-locations.png)

   1. Для выбранного местоположения, под **типами,** в **Выберите тип,** выберите **Пятно**.

      ![Выберите тип пятна](./media/resource-manager-core-quotas-request/select-spot-type.png)

       В **разделе Типы**можно запросить как стандартные, так и спотовые типы квот из одного случая поддержки через поддержку с несколькими выборами.

       Для получения дополнительной [информации см. Стандартную квоту: Увеличение лимитов по сериям VM](per-vm-quota-requests.md).

   1. Введите новый лимит квот, который требуется для этой подписки.

      ![Выберите новую квоту для спот-VM](./media/resource-manager-core-quotas-request/spot-set-new-quota.png)

1. Чтобы запросить увеличение квоты для более чем одного местоположения, выберите дополнительное место в **местоположении,** а затем выберите подходящий тип VM. Затем можно ввести лимит, который применяется к дополнительному местоположению.

   ![Указать дополнительные местоположения в деталях квоты](./media/resource-manager-core-quotas-request/quota-details-multiple-locations.png)

1. Выберите **Сохранить и продолжить** создание запроса поддержки.

## <a name="request-a-quota-limit-increase-from-the-subscriptions-pane"></a>Запрос увеличения лимита квоты из панели подписки

Чтобы запросить увеличение лимита спотового лимита для всех серий VM из панели **подписки:**

> [!NOTE]
> Вы также можете запросить увеличение лимита квот для нескольких регионов через один случай поддержки. Для получения подробной информации см.

1. На [портале Azure](https://portal.azure.com) **ищите подписки.**

   ![Подписка в поиске портала Azure](./media/resource-manager-core-quotas-request/search-for-subscriptions.png)

1. Выберите подписку, квоту которой вы хотите увеличить.

   ![Подписки для выбора для изменений](./media/resource-manager-core-quotas-request/select-subscription-change-quota.png)

1. В левом стеле выберите **квоты использования и**использования.

   ![Ссылка "Использование и квоты"](./media/resource-manager-core-quotas-request/select-usage-plus-quotas.png)

1. В правом верхнем правом положении выберите **увеличение запроса.**

   ![Выберите для увеличения квоты](./media/resource-manager-core-quotas-request/request-increase-from-subscription.png)

1. Для **типа квоты,** выберите **Вычислить-VM (cores-vCPUs) ограничение подписки увеличивается.**

   ![Выберите тип квоты](./media/resource-manager-core-quotas-request/select-quota-type.png)

1. Выберите **Далее: Решения** для открытия **PROBLEM DETAILS**. Выберите **Предоставить подробную информацию** для ввода дополнительной информации. В **деталях квоты**введите следующую информацию:

   1. Для **модели развертывания**выберите соответствующую модель, а для **местоположения**— месторасположение.

      ![Предоставить сведения о квотах](./media/resource-manager-core-quotas-request/quota-details-deployment-locations.png)

   1. Для выбранного местоположения, под **типами,** в **Выберите тип,** выберите **Пятно**.

      ![Выберите тип пятна](./media/resource-manager-core-quotas-request/select-spot-type.png)

      Для получения дополнительной [информации см. Стандартную квоту: Увеличение лимитов по сериям VM](per-vm-quota-requests.md).

   1. Введите новый лимит квот, который требуется для этой подписки.

      ![Введите новое значение для лимита vCPU](./media/resource-manager-core-quotas-request/spot-set-new-quota.png)

1. Чтобы запросить увеличение квоты для более чем одного местоположения, выберите дополнительное место в **местоположении,** а затем выберите подходящий тип VM. Затем можно ввести лимит, который применяется к дополнительному местоположению.

   ![Выберите дополнительные места в деталях квоты](./media/resource-manager-core-quotas-request/quota-details-multiple-locations.png)

1. Выберите **Сохранить и продолжить** создание запроса поддержки.

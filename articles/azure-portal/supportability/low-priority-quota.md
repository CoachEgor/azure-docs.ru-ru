# <a name="spot-quota-increase-limits-for-all-vm-series"></a>Квота на место: увеличение ограничений для всех рядов виртуальных машин

Плашечные виртуальные машины (VM) предоставляют другую модель использования Azure. Они позволяют снизить затраты на Exchange, чтобы позволить Azure удалять виртуальные машины по мере необходимости для развертывания экземпляров виртуальных машин с оплатой по мере использования или зарезервированного экземпляра виртуальной машины. Дополнительные сведения о плашечных виртуальных машинах см. в статье [точки Azure для масштабируемых наборов виртуальных машин](https://docs.microsoft.com/azure/virtual-machine-scale-sets/use-spot).

Azure Resource Manager поддерживает два типа квот виртуальных ЦП для виртуальных машин:
* *Виртуальные машины с оплатой по мере* использования и *зарезервированные экземпляры виртуальных машин* подвергаются *стандартной виртуальных ЦП квоте*.
* На *точечные виртуальные машины* распространяется *виртуальных ЦП квота*. 

Для типа *квоты "виртуальных ЦП* " Укажите диспетчер ресурсов, что квоты виртуальных ЦП применяются ко всем доступным сериям виртуальных машин как единое Региональное ограничение.

При каждом развертывании новой точки размещения общее количество новых и существующих виртуальных цпй использования всех экземпляров виртуальных машин не должно превышать допустимый объем виртуальных цпной квоты. Если превышена квота, плашечное развертывание виртуальной машины не разрешается. 

В этой статье описывается, как запросить увеличение виртуальных ЦП квоты с точки на месте с помощью портал Azure. 

Дополнительные сведения о стандартных квотах виртуальных ЦП см. в статье [квоты виртуальной машины виртуальных ЦП](https://docs.microsoft.com/azure/virtual-machines/windows/quotas) и [ограничения подписки и службы Azure](https://aka.ms/quotalimits). 

Дополнительные сведения о увеличении ограничения виртуальных ЦП по регионам см. в разделе [Стандартная квота: региональные виртуальных ЦП увеличение лимита](https://docs.microsoft.com/azure/azure-supportability/regional-quota-requests).

## <a name="request-a-quota-limit-increase-from-the-help--support-pane"></a>Запросите увеличение квоты в области "Справка и поддержка" 

Чтобы запросить увеличение квоты на место для всех рядов виртуальных машин в области **Справка и поддержка** , выполните следующие действия.

> [!NOTE]
> Вы также можете запросить увеличение квоты для нескольких регионов с помощью одного варианта поддержки. Дополнительные сведения см. в шаге 8. 

1. В левой области [портал Azure](https://portal.azure.com)выберите **Справка и поддержка**.

   ![Ссылка "Справка и поддержка"](./media/resource-manager-core-quotas-request/helpsupport.png)
 
1. В области **Справка и поддержка** выберите **новый запрос в службу поддержки**. 

    ![Новый запрос на техническую поддержку](./media/resource-manager-core-quotas-request/newsupportrequest.png)

1. В раскрывающемся списке **тип проблемы** выберите **пределы службы и подписки (квоты)** .

   ![Раскрывающийся список "тип проблемы"](./media/resource-manager-core-quotas-request/issuetypedropdown.png)

1. В раскрывающемся списке **Подписка** выберите подписку, для которой необходимо увеличить квоту.

   ![Раскрывающийся список "Подписка"](./media/resource-manager-core-quotas-request/select-subscription-sr.png)
   
1. В раскрывающемся списке **тип квоты** выберите **Вычисление виртуальной машины (ядер-виртуальных ЦП), чтобы увеличить лимит подписки**. 

   ![Раскрывающийся список "тип квоты"](./media/resource-manager-core-quotas-request/select-quota-type.png)

1. Перейдите на вкладку **сведения** , в разделе **сведения о проблеме**выберите **указать сведения**и введите дополнительные сведения, чтобы упростить обработку запроса.

   ![Ссылка "предоставить сведения"](./media/resource-manager-core-quotas-request/provide-details.png)
   
1. В области **сведений о квоте** в правом верхнем углу выполните следующие действия.

   ![Область "сведения о квоте"](./media/resource-manager-core-quotas-request/3-7.png)

   а. В раскрывающемся списке **модель развертывания** выберите нужную модель.

   b. В раскрывающемся списке **расположения** выберите расположение. Для выбранного расположения в разделе **типы**в поле **выберите тип** введите **место**. 
   
   ![Вкладка сведений «новый запрос в службу поддержки»](./media/resource-manager-core-quotas-request/3-8.png)

    В разделе **типы**можно запросить как стандартные, так и типы квот из одного варианта поддержки с поддержкой множественного выбора. 
    
    Дополнительные сведения см. [в разделе Стандартная квота: ограничение виртуальных ЦП для серии виртуальных машин](https://docs.microsoft.com/azure/azure-supportability/per-vm-quota-requests).

   c. Введите новое ограничение квоты для этой подписки. 
 
   ![Текстовое поле "новый предел виртуальных ЦП"](./media/resource-manager-core-quotas-request/3-9.png)

1. Чтобы запросить увеличение квоты для нескольких расположений, выберите дополнительное расположение в раскрывающемся списке, а затем выберите подходящий тип виртуальной машины. Затем можно ввести ограничение, которое будет применяться к дополнительному расположению.

   ![Дополнительные расположения в области "сведения о квоте"](./media/resource-manager-core-quotas-request/3-10.png)

1. Выберите **сохранить и продолжить** , чтобы продолжить создание запроса в службу поддержки.

## <a name="request-a-quota-limit-increase-from-the-subscriptions-pane"></a>Запросите увеличение квоты на панели "подписки"

Чтобы запросить увеличение квоты на место для всех рядов виртуальных машин в области **подписки** , выполните следующие действия.

> [!NOTE]
> Вы также можете запросить увеличение квоты для нескольких регионов с помощью одного варианта поддержки. Дополнительные сведения см. в разделе Шаг 7. 

1. В левой области [портал Azure](https://portal.azure.com)выберите **подписки**.

   ![Ссылка "подписки"](./media/resource-manager-core-quotas-request/subscriptions.png)

1. Выберите подписку, для которой необходимо увеличить квоту.

   ![Область "подписки"](./media/resource-manager-core-quotas-request/select-subscription.png)

1. В левой области страницы **\<имя подписки >** выберите **использование и квоты**.

   ![Ссылка "использование и квоты"](./media/resource-manager-core-quotas-request/select-usage-quotas.png)

1. В правом верхнем углу выберите **запросить увеличение**.

   ![Запросить увеличение](./media/resource-manager-core-quotas-request/request-increase.png)

1. В раскрывающемся списке **тип квоты** выберите **Вычисление виртуальной машины (ядер-виртуальных ЦП), чтобы увеличить лимит подписки**.

   ![Раскрывающийся список "тип квоты"](./media/resource-manager-core-quotas-request/select-quota-type.png)

1. В области **сведений о квоте** в правом верхнем углу выполните следующие действия.

   ![Область "сведения о квоте"](./media/resource-manager-core-quotas-request/3-2-6.png)
 
   а. В раскрывающемся списке **модель развертывания** выберите нужную модель.

   b. В раскрывающемся списке **расположения** выберите расположение. 
   
   c. Для выбранного расположения в разделе **типы**в поле **выберите тип** введите **место**.

   ![Область "сведения о квоте"](./media/resource-manager-core-quotas-request/3-2-7.png)

   Дополнительные сведения см. [в разделе Стандартная квота: ограничение виртуальных ЦП для серии виртуальных машин](https://docs.microsoft.com/azure/azure-supportability/per-vm-quota-requests).

   d. Введите новое ограничение квоты для этой подписки.

   ![Текстовое поле "новый предел виртуальных ЦП"](./media/resource-manager-core-quotas-request/3-2-8.png)
 
1. Чтобы запросить увеличение квоты для нескольких расположений, выберите дополнительное расположение в раскрывающемся списке, а затем выберите подходящий тип виртуальной машины. Затем можно ввести ограничение, которое будет применяться к дополнительному расположению.

   ![Дополнительные расположения в области "сведения о квоте"](./media/resource-manager-core-quotas-request/3-2-9.png)
 
1. Выберите **сохранить и продолжить** , чтобы продолжить создание запроса в службу поддержки.

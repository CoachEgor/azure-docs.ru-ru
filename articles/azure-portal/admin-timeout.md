# <a name="set-directory-level-inactivity-timeout"></a>Установить тайм-аут бездействия уровня каталога

Настройка тайм-аута бездействия помогает защитить ваши ресурсы от несанкционированного доступа, если пользователи забывают обезопасить свою рабочую станцию. Когда пользователь простаивает некоторое время, его сеанс портала Azure автоматически выводится из него. Администраторы в роли [Глобального администратора](../active-directory/users-groups-roles/directory-assign-admin-roles.md#global-administrator--company-administrator) могут обеспечить максимальное время простоя до того, как сессия будет подписана. Настройка тайм-аута бездействия применяется на уровне каталога. Для получения дополнительной информации о каталогах [см.](/windows-server/identity/ad-ds/get-started/virtual-dc/active-directory-domain-services-overview)

## <a name="configure-the-inactive-timeout-setting"></a>Настройка настройки неактивного тайм-аута

Если вы являетесь глобальным администратором и хотите обеспечить настройку простоя для всех пользователей портала Azure, выполните следующие действия:

1. Войдите на [портал Azure](https://portal.azure.com).
2. Выберите настройки из глобального заголовка **страницы.**
3. Выберите тайм-аут уровня уровня ссылки на текст **настройки каталога.**

    ![Скриншот, показывающий настройки портала со ссылкой текст выделен](./media/admin-timeout/settings.png)

4. Откроется новая страница. На странице **бездействия уровня Настройки уровень тайм-аута** выберите тайм-аут уровня Включения каталога **для портала Azure,** чтобы включить настройку.
5. Далее введите **часы** и **минуты** для максимального времени, которое пользователь может простаивать до того, как сеанс автоматически запишется.
6. Нажмите кнопку **Применить**.

    ![Скриншот, показывающий страницу для установки тайм-аута бездействия уровня каталога](./media/admin-timeout/configure.png)

Чтобы подтвердить правильное настройку **политики** тайм-аута бездействия, выберите Уведомления из глобального заголовка страницы. Убедитесь, что уведомление об успехе включено в список.

  ![Скриншот, показывающий успешное сообщение уведомления для тайм-аута бездействия уровня каталога](./media/admin-timeout/confirmation.png)

Настройка вступает в силу для новых сеансов. Он не будет применяться сразу к любым пользователям, которые уже подписаны.

> [!NOTE]
> Если Глобальный администратор настроил настройку тайм-аута уровня каталога, пользователи могут переопределить политику и установить свою собственную неактивную продолжительность вывески. Тем не менее, пользователь должен выбрать временной интервал, который меньше, чем тот, который устанавливается на уровне каталога Глобальным администратором.

# <a name="iot-plug-and-play-preview-common-interfaces"></a>IoT Plug и Play Предварительный просмотр общих интерфейсов

Все устройства IoT Plug и Play, как ожидается, будут реализованы некоторые общие интерфейсы. Общие интерфейсы приносят пользу решениям IoT, поскольку они обеспечивают последовательную функциональность. [Сертификация](tutorial-build-device-certification.md) требует от устройства реализации нескольких общих интерфейсов. Можно получить общие определения интерфейса из общедоступного репозитория моделей.

## <a name="summary-of-common-interfaces"></a>Резюме общих интерфейсов

| name | ID | Описание | Реализовано SDK Azure IoT | Должен быть объявлен в модели возможностей |
| -------- | -------- | -------- | -------- | -------- | -------- |
| Информация о модели | урна:azureiot:МодельОткрытие:МодельИнформация:1 | Для устройств, чтобы объявить идентификатор модели возможностей и интерфейсы. Требуется для всех устройств IoT Plug and Play. | Да | нет |
| Информация о клиенте-близнеце Цифровой | урна:azureiot:Клиент:SDKInformation:1 | Клиент SDK для подключения устройства к Azure. Требуется для [сертификации](tutorial-build-device-certification.md) | Да | нет |
| Сведения об устройстве | урна:azureiot:Управление устройством:DeviceInformation:1 | Аппаратная и операционная система информация об устройстве. Требуется для [сертификации](tutorial-build-device-certification.md) | нет | Да |
| Определение модели | урна:azureiot:МодельОткрытие:МодельОпределение:1 | Для устройств, чтобы объявить полное определение своей модели возможностей и интерфейсов. Должны быть реализованы, когда определения модели не размещаются в репозитории моделей. | нет | Да |
| Digital Twins | урна:azureiot:МодельОткрытие:DigitalTwin:1 | Для разработчиков решений, чтобы получить идентификатор модели возможностей и идентификаторы интерфейса для цифрового близнеца. Этот интерфейс не объявляется и не реализуется устройством IoT Plug and Play. | нет | нет |

- Реализовано Azure IoT SDK - реализует ли Azure IoT SDK возможности, заявленные в интерфейсах. IoT Plug and Play устройств, которые используют Azure IoT SDK не нужно реализовать этот интерфейс.
- Должна быть объявлена в модели возможностей - Если "да", этот интерфейс должен быть объявлен в `"implements":` разделе модели возможностей устройства для этого устройства IoT Plug and Play.

## <a name="retrieve-interface-definitions-from-the-public-repository"></a>Извлечение определений интерфейса из публичного репозитория

### <a name="cli"></a>CLI

Для получения общих интерфейсов из репозитория общедоступной модели можно использовать расширение Azure IoT для Azure CLI.

```azurecli
az iot pnp interface show --interface {InterfaceID}
```

```azurecli
az iot pnp capability-model show --model {ModelID}
```

### <a name="vs-code"></a>VS Code

1. Нажмите сочетание клавиш **CTRL+SHIFT+P**, чтобы открыть палитру команд.

1. Введите **Plug and Play,** а затем выберите **ioT Plug и играть: Открытая команда репозитория модели.** Выберите **публичное репозиторий**. Общедоступный репозиторий моделей откроется в VS Code.

1. В общедоступном репозитории модели введите имя интерфейса в поле поиска.

1. Чтобы создать локальную копию интерфейса, выберите ее в результатах поиска, а затем выберите **Download.**

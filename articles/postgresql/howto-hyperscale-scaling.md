# <a name="scale-a-hyperscale-citus-server-group"></a>Масштабирование группы серверов Hyperscale (Citus)

База данных Azure для PostgreS'L - Hyperscale (Citus) обеспечивает масштабирование самообслуживания для решения повышенной нагрузки. Портал Azure упрощает добавление новых узлов рабочего до увеличения vCores существующих узлов.

## <a name="add-worker-nodes"></a>Добавление узлов работы

Чтобы добавить узлы, перейдите на вкладку **Configure** в группе серверов Hyperscale (Citus).  Перетаскивание ползунка для **подсчета рабочих узлов** изменяет значение.

![Ползунки ресурсов](./media/howto-hyperscale-scaling/01-sliders-workers.png)

Нажмите кнопку **Сохранить,** чтобы изменение значения вступило в силу.

> [!NOTE]
> После увеличения и сохранения количество узлов рабочего не может быть уменьшено с помощью ползунка.

### <a name="rebalance-shards"></a>Ребалансировка осколков

Чтобы воспользоваться преимуществами недавно добавленных узлов, необходимо сбалансировать распределенные [осколки таблицы,](concepts-hyperscale-distributed-data.md#shards)что означает перемещение некоторых осколков от существующих узлов к новым. Сначала убедитесь, что новые работники успешно закончили подготовку. Затем запустите ребалансизатор осколка, подключившись к узлом координатора кластера с psql и запустив:

```sql
SELECT rebalance_table_shards('distributed_table_name');
```

Функция `rebalance_table_shards` перебалансирует все таблицы в группе [colocation](concepts-hyperscale-colocation.md) таблицы, названной в ее аргументе. Таким образом, вам не нужно вызывать функцию для каждой распределенной таблицы, просто позвоните на представительную таблицу от каждой группы colocation.

## <a name="increase-or-decrease-vcores-on-nodes"></a>Увеличение или уменьшение vCores на узлах

> [!NOTE]
> Эта функция в настоящее время находится на стадии предварительной версии. Чтобы запросить изменение vCores для узлов в вашей серверной группе, пожалуйста, [свяжитесь с поддержкой Azure.](https://portal.azure.com/?#blade/Microsoft_Azure_Support/HelpAndSupportBlade)

В дополнение к добавлению новых узлов, вы можете увеличить возможности существующих узлов. Корректировка вычислительной мощности вверх и вниз может быть полезна для экспериментов с производительностью, а также краткосрочных или долгосрочных изменений в требованиях трафика.

Чтобы изменить vCores для всех узлов работы, отрегулируйте ползунок **vCores** под **конфигурацией (на узлы работника).** vCores узла координатора можно настроить самостоятельно. Нажмите на ссылку **конфигурации изменения** под **узлами координатора.** Появится диалог с ползунками для vCores и емкости хранилища координатора. Изменение ползунки по желанию и выбрать **OK**.

# <a name="configure-hyperscale-citus-high-availability"></a>Настройка гипермасштабы (Citus) высокой доступности

База данных Azure для PostgreS'L - Hyperscale (Citus) обеспечивает высокую доступность (HA), чтобы избежать простоя базы данных. С включенным HA каждый узла в группе серверов получит режим ожидания. Если исходный узла становится нездоровым, его резервный режим будет повышен, чтобы заменить его.

> [!IMPORTANT]
> Поскольку HA удваивает количество серверов в группе, это также удвоит стоимость.

Включение HA возможно во время создания группы серверов или после этого во вкладке **«Настройка»** для вашей группы серверов на портале Azure. Пользовательский интерфейс выглядит одинаково в любом случае. Перетащите ползунок для **высокой доступности** в ДА:

![га ползунок](./media/howto-hyperscale-high-availability/01-ha-slider.png)

Нажмите кнопку **Сохранить,** чтобы применить ваш выбор. Включение HA может занять некоторое время, поскольку группа серверов резервирует ожидания и потоковые данные к ним.

Вкладка **«Обзор»** для группы серверов будет перечислять все узлы и их ожидания, а также столбец **высокой доступности,** указывающий, успешно ли включен HA для каждого узла.

![столбец ha в обзоре группы серверов](./media/howto-hyperscale-high-availability/02-ha-column.png)

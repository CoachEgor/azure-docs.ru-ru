# <a name="prepay-for-azure-database-for-postgresql-compute-resources-with-reserved-capacity"></a>Предоплата для базы данных Azure для вычисляемых ресурсов PostgreS'L с зарезервированной емкостью

База данных Azure для PostgreS'L теперь помогает вам сэкономить деньги, предоплату за вычислительные ресурсы по сравнению с ценами с оплатой по мере работы. С помощью зарезервированной емкости Azure Database для зарезервированных мощностей PostgreS'L вы берете на себя авансовые обязательства на сервере PostgreS'L в течение одного или трех лет, чтобы получить значительную скидку на вычислительные расходы. Чтобы приобрести базу данных Azure для зарезервированных емкостей PostgreS'L, необходимо указать область Azure, тип развертывания, уровень производительности и термин. </br>

Вам не нужно присваивать резервирование конкретной базе данных Azure для серверов PostgreS'L. Уже запущенная база данных Azure для PostgreS'L или недавно развернутая база данных, автоматически получит выгоду от зарезервированных цен. Приобретая резервирование, вы предварительно оплачиваете стоимость вычислений на один или три года. Как только вы покупаете бронирование, база данных Azure для расчета PostgreS'L, которая соответствует атрибутам бронирования, больше не взимается по тарифам с оплатой по мере покупки. Бронирование не распространяется на расходы на программное обеспечение, сеть или хранение данных, связанные с серверами базы данных PostgreS'L. В конце срока бронирования истекает срок действия платежного пособия, а база данных Azure для PostgreS'L выставляется по цене оплаты по мере того, как вы получаете плату. Резервирования не возобновляются автоматически. Для получения информации о ценах см. базу [данных Azure для зарезервированных емкости PostgreS'L.](https://azure.microsoft.com/pricing/details/postgresql/) </br>

> [!IMPORTANT]
> Зарезервированные цены на емкости доступны только для базы данных Azure для развертывания [одного сервера](https://docs.microsoft.com/azure/postgresql/overview#azure-database-for-postgresql---single-server) PostgreS'L, а не для развертывания [Hyperscale Citus.](https://docs.microsoft.com/azure/postgresql/overview#azure-database-for-postgresql---hyperscale-citus)

Вы можете купить базу данных Azure для зарезервированной емкости PostgreS'L на [портале Azure.](https://portal.azure.com/) Чтобы купить зарезервированную емкость:

* Вы должны быть в роли владельца, по крайней мере один предприятие или индивидуальную подписку с оплатой по мере использования ставок.
* Для подписок с соглашением Enterprise параметр **Добавить зарезервированные экземпляры** следует включить на [портале EA](https://ea.azure.com/). Или, если этот параметр отключен, необходимо быть администратором подписки EA.
* Для программы поставщика облачных решений (CSP) только агенты-админы или агенты по продажам могут приобрести базу данных Azure для зарезервированной емкости PostgreS'L. </br>

Подробная информация о том, как корпоративные клиенты и клиенты Pay-As-You-Go взимаются за покупки бронирования, см. понимание [использования бронирования Azure для вашего корпоративного регистрации](https://docs.microsoft.com/azure/billing/billing-understand-reserved-instance-usage-ea) и [понимания использования бронирования Azure для подписки Pay-As-You-Go.](https://docs.microsoft.com/azure/billing/billing-understand-reserved-instance-usage)


## <a name="determine-the-right-server-size-before-purchase"></a>Определить правильный размер сервера перед покупкой

Размер резервирования должен основываться на общем количестве вычислений, используемых существующими или скоро развернутыми серверами в определенном регионе, и с использованием одного и того же уровня производительности и аппаратного обеспечения.</br>

Например, предположим, что вы работаете с одной общей целью, Gen5 - 32 vCore PostgreS'L базы данных, и две оптимизированные данные памяти, Gen5 - 16 vCore PostgreS'L баз данных. Кроме того, предположим, что вы планируете развернуть в течение следующего месяца дополнительную общую цель, Gen5 - 32 vCore сервер базы данных, и один оптимизированный, Gen5 - 16 vCore сервер базы данных. Предположим, что вы знаете, что вам понадобятся эти ресурсы, по крайней мере 1 год. В этом случае вы должны приобрести 64 (2x32) vCores, 1 год резервирования для одной базы данных общего назначения - Gen5 и 48 (2x16 16) vCore 1 год резервирования для одной базы данных памяти оптимизированы - Gen5


## <a name="buy-azure-database-for-postgresql-reserved-capacity"></a>Купить базу данных Azure для зарезервированных мощностей PostgreS'L

1. Войдите на [портал Azure](https://portal.azure.com/).
2. Выберите **Все услуги** > **Бронирование**.
3. Выберите **Добавить,** а затем в панели бронирования покупок, выберите **базу данных Azure для PostgreS'L,** чтобы приобрести новое бронирование для баз данных PostgreS'L.
4. Заполните требуемые поля. Существующие или новые базы данных, которые соответствуют выбранным атрибутам, чтобы получить скидку зарезервированной емкости. Фактическое количество базы данных Azure для серверов PostgreS'L, которые получают скидку, зависит от выбранной области и количества.


![Обзор зарезервированных цен](media/concepts-reserved-pricing/postgresql-reserved-price.png)


В следующей таблице описаны необходимые поля.

| Поле | Описание |
| :------------ | :------- |
| Подписка   | Подписка, используемая для оплаты базы данных Azure для резервирования зарезервированных емок PostgreS'L. С способа оплаты по подписке взимается авансовые расходы для базы данных Azure для резервирования зарезервированных емок PostgreS'L. Тип подписки должен быть корпоративным соглашением (номерами предложений: MS-A'A'-0017P или MS-A'A'-0148P) или индивидуальным соглашением с ценами с оплатой по мере использования (номера предложений: MS-A'R-0003P или MS-A'A'-0023P). Для подписки с соглашением Enterprise плата вычитается из баланса денежных обязательств или относится к избыточным расходам. Для индивидуальной подписки с оплатой по мере использования, сборы выставляются на кредитную карту или метод оплаты счета-фактуры по подписке.
| Область | Область резервирования виртуальных ядер может охватывать одну или несколько подписок (общая область). Если выбрать: </br></br> **Общая**скидка на бронирование vCore применяется к базе данных Azure для серверов PostgreS'L, работающих в любых подписках в контексте выставления счетов. Для клиентов с соглашением Enterprise общая область действует в рамках регистрации и включает в себя все подписки, заданные при регистрации. Для клиентов с подпиской с оплатой по мере использования общая область включает в себя все подписки с оплатой по мере использования, созданные администратором учетной записи.</br></br> **Единая подписка,** скидка на бронирование vCore применяется к базе данных Azure для серверов PostgreS'L в этой подписке. </br></br> **Единая группа ресурсов,** скидка на резервирование применяется к базе данных Azure для серверов PostgreS'L в выбранной подписке и выбранной группе ресурсов в рамках этой подписки.
| Регион | Область Azure, охватываемая базой данных Azure для резервирования резервных емок PostgreS'L.
| Тип развертывания | База данных Azure для типа ресурсов PostgreS'L, для которого вы хотите купить резервирование.
| Уровень производительности | Уровень обслуживания для базы данных Azure для серверов PostgreS-L.
| Термин | Один год
| Количество | Объем вычислительных ресурсов, приобретаемых в базе данных Azure для резервирования зарезервированных емок PostgreS'L. Количество — это количество vCore s в выбранном регионе Azure и уровне производительности, которые зарезервированы и получат скидку на выставление счетов. Например, если вы работаете или планируете запустить базу данных Azure для серверов PostgreS'L с общей вычислительной мощностью Gen5 16 vCores в восточном регионе США, то вы укажете количество как 16, чтобы максимизировать выгоду для всех серверов.

# <a name="auto-grow-storage-using-the-azure-portal-in-azure-database-for-postgresql---single-server"></a>Автоматическое хранение с помощью портала Azure в базе данных Azure для PostgreS-L - Единый сервер
В этой статье описывается, как можно настроить базу данных Azure для хранения серверов PostgreS'L для роста, не влияя на рабочую нагрузку.

Когда сервер достигает выделенного лимита хранилища, сервер помечается как только для чтения. Однако, если вы позволяете автоматически увеличивать объемы хранения данных, то хранилище серверов увеличивается для размещения растущих данных. Для серверов с объемом хранения менее 100 ГБ размер резерва увеличивается на 5 ГБ, как только свободное хранилище будет ниже более 1 ГБ или 10% от объема резервированного хранилища. Для серверов с более чем 100 ГБ подготовленного хранилища размер резервированного хранилища увеличивается на 5%, когда свободное пространство для хранения ниже 5% от предусмотренного размера хранилища. Максимальные лимиты хранения, указанные [здесь,](https://docs.microsoft.com/azure/postgresql/concepts-pricing-tiers#storage) применяются.

## <a name="enable-storage-auto-grow"></a>Включить автоматическое тростку хранилища 

Следуйте этим шагам, чтобы установить PostgreS'L сервер хранения автоматической расти:

1. На [портале Azure](https://portal.azure.com/)выберите существующую базу данных Azure для сервера PostgreS'L.

2. На странице сервера PostgreS'L, в **настройках,** нажмите **уровень ценообразования,** чтобы открыть страницу уровня ценообразования.

3. В разделе **Авторост** выберите **«Да»,** чтобы обеспечить автоматическое увеличение хранилища.

    ![База данных Azure для PostgreS'L - Settings_Pricing_tier - Авто-рост](./media/howto-auto-grow-storage-portal/3-auto-grow.png)

4. Чтобы сохранить изменения, нажмите кнопку **ОК**.

5. Уведомление подтвердит, что авторост был успешно включен.

    ![База данных Azure для PostgreS'L - успех авто-роста](./media/howto-auto-grow-storage-portal/5-auto-grow-successful.png)

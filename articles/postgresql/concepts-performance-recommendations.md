# <a name="performance-recommendations-in-azure-database-for-postgresql---single-server"></a>Рекомендации по производительности в базе данных Azure для PostgreS-L - Единый сервер

**Применяется к:** База данных Azure для PostgreS'L - Версии для одного сервера 9.6, 10, 11

Функция «Рекомендации по производительности» анализирует базы данных для создания индивидуальных предложений для повышения производительности. Для подготовки рекомендаций в анализе рассматриваются различные характеристики базы данных, включая схему. Включите [магазин запросов](concepts-query-store.md) на сервере, чтобы в полной мере использовать функцию Рекомендации производительности. После реализации какой-либо рекомендации по производительности следует проверить производительность для оценки влияния этих изменений. 

## <a name="permissions"></a>Разрешения
Для выполнения анализа с помощью функции рекомендаций по повышению производительности требуются разрешения **владельца** или **участника**.

## <a name="performance-recommendations"></a>Рекомендации по производительности
Функция [рекомендаций по повышению производительности](concepts-performance-recommendations.md) анализирует рабочие нагрузки на сервере и определяет, какие индексы могут помочь в повышении производительности.

**Рекомендации по открытой производительности** из раздела **«Интеллектуальная производительность»** в панели меню на странице портала Azure для сервера PostgreS'L.

![Целевая страница рекомендаций по повышению производительности](./media/concepts-performance-recommendations/performance-recommendations-page.png)

Выберите **анализ** и выберите базу данных, которая начнет анализ. В зависимости от рабочей нагрузки анализ th может занять несколько минут. По окончании анализа на портале появится уведомление. Анализ выполняет глубокое изучение вашей базы данных. Мы рекомендуем вам проводить анализ во время внепиковых периодов. 

В окне **Рекомендаций** будет указан список рекомендаций, если таковые были найдены.

![Новая страница рекомендаций по производительности](./media/concepts-performance-recommendations/performance-recommendations-result.png)

Рекомендации не применяются автоматически. Чтобы применить рекомендацию, скопируйте текст запроса и запустите его от клиента по выбору. Не забудьте проверить и контролировать, чтобы оценить рекомендацию. 

## <a name="recommendation-types"></a>Типы рекомендаций

В настоящее время поддерживаются два типа рекомендаций: *Создание индекса* и *Индекс падения.*

### <a name="create-index-recommendations"></a>Рекомендации по созданию индексов
*Рекомендации по созданию индекса* предлагают новые индексы для ускорения наиболее часто заражаемых или трудоемких запросов в рабочей нагрузке. Этот тип рекомендаций требует включения [Магазина запросов.](concepts-query-store.md) Магазин запросов собирает информацию о запросах и предоставляет подробную статистику выполнения запросов и частоты, которую анализ использует для выполнения рекомендации.

### <a name="drop-index-recommendations"></a>Рекомендации по удалению индексов
Помимо обнаружения недостающих индексов, База данных Azure для PostgreS'L анализирует производительность существующих индексов. Если индекс используется либо редко, либо излишни, анализатор рекомендует его сбросить.

## <a name="considerations"></a>Рекомендации
* Рекомендации по производительности недоступны для [ремиссий для чтения.](concepts-read-replicas.md)

# <a name="security-in-azure-database-for-postgresql---single-server"></a>Безопасность в базе данных Azure для PostgreS-L - Единый сервер

Существует несколько уровней безопасности, которые доступны для защиты данных на базе данных Azure для сервера PostgreS'L. В этой статье излагаются эти параметры безопасности.

## <a name="information-protection-and-encryption"></a>Защита и шифрование информации

### <a name="in-transit"></a>В пути
База данных Azure для PostgreS'L защищает ваши данные, шифруя данные в пути с помощью Безопасности уровня транспорта. Шифрование (SSL/TLS) осуществляется по умолчанию.

### <a name="at-rest"></a>На отдыхе
В службе "База данных Azure для PostgreSQL" используется проверенный криптографический модуль FIPS 140-2 для шифрования неактивных данных. Данные, включая резервные, шифруются на диске, за исключением временных файлов, созданных во время выполнения запросов. Служба использует 256-разрядный шифр AES, включенный в шифрование службы хранилища Azure. Ключами управляет система. Шифрование хранилища всегда включено, и его нельзя отключить.


## <a name="network-security"></a>Безопасность сети
Сначала подключение к базе данных Azure для сервера PostgreS'L направляется через региональный шлюз. Шлюз имеет общедоступный IP-адрес, в то время как IP-адреса сервера защищены. Для получения дополнительной информации о шлюзе посетите [статью архитектуры подключения](concepts-connectivity-architecture.md).  

Недавно созданная база данных Azure для сервера PostgreS'L имеет брандмауэр, который блокирует все внешние соединения. Хотя они достигают шлюза, они не имеют права подключаться к серверу. 

### <a name="ip-firewall-rules"></a>Правила брандмауэра для IP-адресов
Правила IP-брандмауэра предоставляют доступ к серверам на основе исходного IP-адреса каждого запроса. Для получения дополнительной информации смотрите [обзор правил брандмауэра.](concepts-firewall-rules.md)

### <a name="virtual-network-firewall-rules"></a>Правила брандмауэра для виртуальной сети
Конечные точки виртуального сетевого обслуживания расширяют виртуальное сетевое подключение по сравнению с магистралью Azure. Используя правила виртуальной сети, вы можете включить базу данных Azure для сервера PostgreS'L, чтобы разрешить соединения из выбранных подсетей в виртуальной сети. Для получения дополнительной информации смотрите [обзор конечных точек виртуального сетевого обслуживания.](concepts-data-access-and-security-vnet.md)

### <a name="private-ip"></a>Частный IP-адрес
Частная ссылка позволяет подключиться к базе данных Azure для одного сервера PostgreS'L в Azure через частную конечную точку. Azure Private Link по существу приносит услуги Azure в вашей частной виртуальной сети (VNet). К ресурсам PaaS можно получить доступ с помощью частного IP-адреса, как и любого другого ресурса в VNet. Для получения дополнительной [private link overview](concepts-data-access-and-security-private-link.md) информации см.


## <a name="access-management"></a>Управление доступом

При создании базы данных Azure для сервера PostgreS'L вы предоставляете учетные данные для роли администратора. Эта роль администратора может быть использована для создания дополнительных [ролей PostgreS'L.](https://www.postgresql.org/docs/current/user-manag.html)

Вы также можете подключиться к серверу с помощью [проверки подлинности Azure Active Directory (AAD).](concepts-aad-authentication.md)


## <a name="threat-protection"></a>Защита от угроз

Вы можете выбрать для [расширенной защиты от угроз,](concepts-data-access-and-security-threat-protection.md) которая обнаруживает аномальные действия, указывающие на необычные и потенциально опасные попытки доступа или эксплуатировать серверы.

[Регистрация аудита](concepts-audit.md) доступна для отслеживания активности в базах данных.

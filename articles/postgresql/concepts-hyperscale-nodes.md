# <a name="nodes-in-azure-database-for-postgresql--hyperscale-citus"></a>Узлы в базе данных Azure для PostgreS'L - Гипермасштаб (Citus)

Тип хостинга Hyperscale (Citus) позволяет базе данных Azure для серверов PostgreS'L (называемые узлами) координировать друг с другом в архитектуре "общих ничего". Узлы в группе серверов в совокупности хранят больше данных и используют больше ядер процессора, чем это было бы возможно на одном сервере. Архитектура также позволяет масштабировать базу данных, добавляя больше узлов в группу серверов.

## <a name="coordinator-and-workers"></a>Координатор и рабочие

Каждая группа серверов имеет узла координатора и несколько сотрудников. Приложения отправляют свои запросы в узлы координатора, который передает его соответствующим работникам и аккумулирует их результаты. Приложения не могут подключаться непосредственно к работникам.

Для каждого запроса координатор либо направляет его в один узлы рабочего, либо параллелизирует его по нескольким в зависимости от того, живут ли необходимые данные на одном узлаили или несколько. Координатор решает, что делать, консультируя таблицы метаданных. В этих таблицах отслеживают имена DNS и работоспособность узлов работы, а также распределение данных по узлам.

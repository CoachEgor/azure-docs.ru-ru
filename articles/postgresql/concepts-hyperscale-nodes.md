# <a name="nodes-in-azure-database-for-postgresql--hyperscale-citus-preview"></a>Узлы в базе данных Azure для PostgreSQL — Гипермасштабируемость (Citus) (Предварительная версия)

Гипермасштабируемого (Citus) (Предварительная версия), тип размещения базы данных Azure позволяет серверы PostgreSQL (называемые узлами) для координации друг с другом в архитектуре «» без общих ресурсов. Узлы в группе серверов совместно хранить больше данных и использовать большее число ядер ЦП, чем на одном сервере. Архитектура также позволяет базе данных для масштабирования, добавив дополнительные узлы в группу серверов.

## <a name="coordinator-and-workers"></a>Координатора и рабочих ролей

Каждая группа сервера имеет координатором становится другой узел и нескольких рабочих ролей. Приложения отправляют запросы к узлу координатора, который передает его в соответствующие рабочие роли и собирает их результаты. Приложения не смогут подключаться непосредственно к рабочих ролей.

Для каждого запроса координатор направляет его в одном рабочем узле или его распараллеливание нескольких в зависимости от ли необходимые данные находятся на одном узле или нескольких. Координатор определяет необходимые действия по журналу таблицы метаданных. Эти таблицы отслеживания DNS-имена и работоспособности рабочих узлов и распределение данных между узлами.

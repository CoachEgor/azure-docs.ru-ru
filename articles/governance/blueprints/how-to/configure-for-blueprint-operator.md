# <a name="configure-your-environment-for-a-blueprint-operator"></a>Настройка среды для роли "Оператор схем"

Управление определениями чертежа и назначениями чертежей может быть назначено различным группам. Обычно архитектор ы или команда управления отвечают за управление жизненным циклом определений чертежей, в то время как оперативная группа отвечает за управление назначениями этих централизованно контролируемых определений чертежей.

Встроенный элемент управления доступом blueprint **Operator** Operator (RBAC) разработан специально для использования в этом типе сценария. Роль позволяет группам типов операций управлять назначением определений чертежей организаций, но не возможностью их изменения. Для этого требуется определенная конфигурация в среде Azure, и в этой статье объясняются необходимые шаги.

## <a name="grant-permission-to-the-blueprint-operator"></a>Разрешение на предоставление гранта Оператору проекта

Первым шагом является предоставление роли **оператора Blueprint** учетной записи или группы безопасности (рекомендуется), которая будет назначать чертежи. Это действие должно быть выполнено на самом высоком уровне в иерархии групп управления, которая охватывает все группы управления и подписки, к которым должна иметь доступ к назначению чертежей. При предоставлении этих разрешений рекомендуется соблюдать принцип наименьших привилегий.

1. (Рекомендуется) [Создание группы безопасности и добавление участников](../../../active-directory/fundamentals/active-directory-groups-create-azure-portal.md)

1. [Добавление ролевой уступки](../../../role-based-access-control/role-assignments-portal.md#add-a-role-assignment) **оператора Blueprint** в учетную запись или группу безопасности

## <a name="user-assign-managed-identity"></a>Управляемый имитающий иственнии пользователя

Определение чертежа может использовать как назначенные системой, так и назначенные пользователем управляемые идентификаторы. Однако при использовании роли **оператора Blueprint** определение чертежа должно быть настроено для использования управляемого удостоверения, назначенного пользователем. Кроме того, учетная запись или группа безопасности, которой предоставляется роль **оператора Blueprint,** должна быть предоставлена роль **оператора управляемой идентификации** в управляемой управляемой идентификации. Без этого разрешения назначения чертежей не выполняются из-за отсутствия разрешений.

1. [Создание назначенного пользователем управляемого удостоверения](../../../active-directory/managed-identities-azure-resources/how-to-manage-ua-identity-portal.md#create-a-user-assigned-managed-identity) для использования назначенным чертежем

1. [Добавьте в](../../../role-based-access-control/role-assignments-portal.md#add-a-role-assignment) группу учетной записи или группы безопасности назначение ролевых задач **оператора управляемой идентификации.** Область назначения роли к новому управляемому иполочному ипоните, назначенному пользователем.

1. Как **оператор Blueprint**, [назначить план,](../create-blueprint-portal.md#assign-a-blueprint) который использует новый пользовательский назначенный управляемый иноеудостоверения.

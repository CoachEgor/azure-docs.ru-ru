# <a name="control-mapping-of-the-hipaa-hitrust-blueprint-sample"></a>Сопоставление элементов управления примера схемы HIPAA HITRUST

В статье подробно описано, как пример схемы Azure Blueprints HIPAA HITRUST сопоставляется с элементами управления HIPAA HITRUST. См. сведения об [элементах управления HIPAA HITRUST](https://www.hhs.gov/hipaa/for-professionals/security/laws-regulations/index.html).

Далее представлены сопоставления для элементов управления **HIPAA HITRUST**. Используйте панель навигации справа для перехода непосредственно к сопоставлению конкретных элементов управления. Многие сопоставленные элементы управления реализуются с помощью инициативы [Политика Azure](../../../policy/overview.md). Чтобы просмотреть полную инициативу, откройте раздел **Политика** на портале Azure и перейдите на страницу **Определения**. Найдите и выберите функцию **\[Предварительная версия\]: аудит элементов управления HIPAA HITRUST**. Это встроенная инициатива политики.

> [!IMPORTANT]
> Каждый элемент управления ниже связан с одним или несколькими определениями [Политики Azure](../../../policy/overview.md). Такие политики помогут вам в [оценке соответствия](../../../policy/how-to/get-compliance-data.md) с помощью элементов управления, но часто полное или точное соответствие между элементом управления и одной или несколькими политиками отсутствует. Поэтому состояние **Соответствует** в Политике Azure применимо только к самим политикам и не означает полное соответствие всем требованиям элемента управления. Кроме того, стандарт соответствия включает элементы управления, на которые сейчас не распространяются определения Политики Azure. Следовательно, сведения о соответствии в Политике Azure — это только частичное представление общего состояния соответствия. Связи между элементами управления и определениями Политики Azure для этого примера схемы соответствия могут измениться в будущем. Историю изменений можно просмотреть на странице [журнала фиксаций в GitHub](https://github.com/MicrosoftDocs/azure-docs/commits/master/articles/governance/blueprints/samples/HIPAA-HITRUST/control-mapping.md).

## <a name="control-against-malicious-code"></a>Защита от вредоносного кода

Эта схема помогает управлять защитой конечных точек, в том числе от вредоносного кода, назначая определения [Политики Azure](../../../policy/overview.md), которые контролируют отсутствие защиты конечных точек на виртуальных машинах в Центре безопасности Azure и принудительно применяют защиту от вредоносных программ на виртуальных машинах Windows.

- Развертывание стандартного расширения защиты от вредоносных программ IaaS (Майкрософт) для Windows Server
- Необходимо включить журналы диагностики в учетных записях пакетной службы
- На ваших компьютерах должны быть установлены обновления системы


## <a name="management-of-removable-media"></a>Управление съемными носителями

На основе требований к уровню классификации данных организация регистрирует носители (включая ноутбуки) перед использованием, налагает обоснованные ограничения на использование этих носителей и обеспечивает соответствующий уровень физической и логической защиты (включая шифрование) для носителей, которые содержат защищаемые сведения, вплоть до момента их правильного уничтожения или очистки.

- Требование шифрования для учетных записей Data Lake Store.
- Предохранитель TDE управляемого экземпляра SQL должен быть зашифрован с помощью вашего собственного ключа
- Шифрование дисков должно применяться к виртуальным машинам.
- Прозрачное шифрование данных должно быть включено в базах данных SQL.


## <a name="information-exchange-policies-and-procedures"></a>Политики и процедуры обмена информацией

Поставщики облачных служб используют признанную в отрасли платформу виртуализации и стандартные форматы виртуализации (например, OVF — Open Virtualization Format) для поддержки взаимодействия, а также документируют все дополнительные изменения, внесенные в используемые низкоуровневые оболочки, и все доступные для проверки клиентом перехватчики виртуализации в своих решениях.

- Развертывание необходимых компонентов для аудита виртуальных машин Windows, на которых не установлены указанные приложения

## <a name="control-of-operational-software"></a>Управление операционным ПО 

Организация выявляет несанкционированное программное обеспечение в информационной системе, в том числе на серверах, рабочих станциях и переносных компьютерах, применяет политику, которая разрешает все подключения и запрещает все подключения с исключениями для блокировки известного несанкционированного ПО в информационной системе, а также периодически (не реже раза в год) проверяет и обновляет список несанкционированного ПО.

- \[Предварительная версия\] Отображение результатов аудита для конфигураций виртуальных машин Windows в разделе "Параметры безопасности — аудит"
- \[Предварительная версия\] Отображение результатов аудита для конфигураций виртуальных машин Windows в разделе "Политики аудита системы — управление учетными записями"

## <a name="change-control-procedures"></a>Процедуры управления изменениями

Целостность всех образов виртуальных машин постоянно поддерживается путем ведения журналов и создания предупреждений о любых изменениях, внесенных в образы виртуальных машин, с предоставлением владельцам и (или) клиентам информации о результатах таких изменений или перемещений электронными методами (например, порталы или оповещения) с последующей проверкой целостности образов.

- \[Предварительная версия\] Отображение результатов аудита для конфигураций виртуальных машин Windows в разделе "Политики аудита системы — подробное отслеживание"

## <a name="inventory-of-assets"></a>Каталог ресурсов 

Поддерживается каталог ресурсов и служб

- Необходимо включить журналы диагностики в службах поиска.
- \[Предварительная версия\] Развертывание требований к аудиту конфигураций виртуальных машин Windows в разделе "Параметры безопасности — сервер для сетей Майкрософт"
- \[Предварительная версия\] Развертывание требований по аудиту конфигураций виртуальных машин Windows в разделе "Административные шаблоны — сеть"

## <a name="control-of-technical-vulnerabilities"></a>Управление техническими уязвимостями 

Для всех системных и сетевых компонентов существует стандарт защищенной конфигурации.

- Аудит виртуальных машин, для которых не настроено аварийное восстановление
- Необходимо включить оценку уязвимости на управляемых экземплярах SQL.
- Уязвимости должны быть устранены с помощью решения для оценки уязвимостей.

## <a name="segregation-in-networks"></a>Разделение в сетях

В организации существуют шлюзы безопасности (например, брандмауэры), которые применяют политики безопасности и фильтруют трафик между доменами, блокируют несанкционированный доступ, поддерживают разделение между внутренней проводной, внутренней беспроводной и внешней (Интернет) сетями с организацией сетей периметра, а также применяют политики управления доступом для каждого домена.

- Автоматическая подготовка агента мониторинга безопасности
- Развертывание наблюдателя сети при создании виртуальных сетей

## <a name="input-data-validation"></a>Проверка входных данных

Для любых общедоступных веб-приложений применяются брандмауэры уровня приложения для управления трафиком. Для каждого из общедоступных приложений, не имеющих веб-интерфейсов, в организации реализован сетевой брандмауэр, соответствующий типу этого приложения. Если трафик к общедоступному приложению шифруется, это устройство размещается за шлюзом шифрования или умеет расшифровывать трафик до выполнения анализа.

- \[Предварительная версия\] Отображение результатов аудита для конфигураций виртуальных машин Windows в разделе "Свойства брандмауэра Windows"


## <a name="network-connection-control"></a>Управление сетевым подключением

Сетевой трафик контролируется в соответствии с корпоративной политикой контроля доступа с применением брандмауэра или других сетевых средств ограничения для каждой точки доступа к сети или управляемого интерфейса внешней службы телекоммуникации.

- Доступ через подключенные к Интернету конечные точки должен быть ограничен.
- Предохранитель TDE управляемого экземпляра SQL должен быть зашифрован с помощью вашего собственного ключа
- Удаленная отладка для приложений API должна быть отключена.

## <a name="network-controls"></a>Элементы управления сетью

Организация использует защищенные и зашифрованные каналы связи для миграции физических серверов, приложений или данных на виртуализированные серверы.

- К виртуальным машинам должно применяться шифрование дисков.
- На серверах SQL предохранитель TDE должен быть зашифрован с использованием вашего ключа
- \[Предварительная версия\] Отображение результатов аудита для конфигураций виртуальных машин Windows в разделе "Параметры безопасности — сетевой доступ"
- Аудит неограниченного сетевого доступа к учетным записям хранения
- \[Предварительная версия\] Отображение результатов аудита для конфигураций виртуальных машин Windows в разделе "Свойства брандмауэра Windows"
- Развертывание параметров диагностики из групп безопасности сети
- Доступ через подключенные к Интернету конечные точки должен быть ограничен.

## <a name="sensitive-system-isolation"></a>Изоляция конфиденциальной системы

Общие системные ресурсы (например, регистры, основная память и дополнительное хранилище) возвращаются в систему и защищаются от раскрытия для других систем, приложений и пользователей, а у пользователей нет возможности намеренно или случайно получить доступ к остающимся данных.

- Необходимо перенести виртуальные машины в новые ресурсы Azure Resource Manager.

## <a name="security-of-network-services"></a>Безопасность сетевых служб

Согласованные службы, предоставляемые поставщиком или диспетчером сетевых услуг, официально управляются и отслеживаются для гарантии безопасности их предоставления.

- Необходимо перенести виртуальные машины в новые ресурсы Azure Resource Manager.

## <a name="network-routing-control"></a>Управление сетевой маршрутизацией

Реализованы элементы управления маршрутизацией через шлюзы безопасности (например, брандмауэры) между внутренними и внешними сетями (например, Интернетом и сетями сторонних организаций).

- Адаптивные элементы управления приложениями должны быть включены на виртуальных машинах.

## <a name="information-exchange-policies-and-procedures"></a>Политики и процедуры обмена информацией

Организация ограничивает использование переносимых запоминающих устройств, управляемых ею, уполномоченными пользователями из внешних информационных систем.

- Аудит неограниченного сетевого доступа к учетным записям хранения
- Удаленная отладка для веб-приложений должна быть отключена
- Приложение API должно быть доступно только по протоколу HTTPS.

## <a name="electronic-messaging"></a>Электронная система обмена сообщениями

Утверждения можно получить перед использованием внешних общедоступных служб, в том числе систем обмена мгновенными сообщениями или общего доступа к файлам.

- \[Предварительная версия\] Отображение результатов аудита виртуальных машин Linux без разрешений 0644 для файла паролей

## <a name="on-line-transactions"></a>Транзакции в Интернете

Организация требует использовать шифрование и электронные подписи при взаимодействии между всеми сторонами транзакции. Организация гарантирует, что сведения о транзакциях хранятся за пределами общедоступных сред (например, на платформе хранилища в интрасети организации), а также не сохраняются и не предоставляются на носителях с возможностью доступа напрямую из Интернета. Если используется доверенный центр (например, для выдачи и обслуживания цифровых подписей или цифровых сертификатов), защита полностью интегрирована в комплексный процесс управления сертификатами и подписями.

- К виртуальным машинам должно применяться шифрование дисков.
- \[Предварительная версия\] Отображение результатов аудита виртуальных машин Windows без указанных сертификатов в доверенных корневых центрах сертификации

## <a name="password-management"></a>Управление паролями

Пароли шифруются во время передачи и хранения на всех компонентах системы.

- \[Предварительная версия\] Отображение результатов аудита виртуальных машин Windows без настроенной сложности пароля

## <a name="user-authentication-for-external-connections"></a>Проверка подлинности пользователей для внешних подключений

Используются надежные методы проверки подлинности, такие как многофакторная аутентификация, RADIUS или Kerberos (для привилегированного доступа) и CHAP (для шифрования учетных данных при коммутируемом доступе), для всех внешних подключений к сети Организации.

- MFA должна быть включена для учетных записей с разрешениями на запись в вашей подписке.
- К виртуальным машинам должно применяться JIT-управление доступом.

## <a name="user-identification-and-authentication"></a>Идентификация и проверка подлинности пользователей

Пользователи, которые выполняют привилегированные функции (например, администрирование системы), используют отдельные учетные записи при выполнении этих привилегированных функций. Используются методы многофакторной проверки подлинности в соответствии с корпоративной политикой (например, для удаленного доступа к сети).

- MFA должна быть включена для учетных записей с разрешениями на запись в вашей подписке.
- К виртуальным машинам должно применяться JIT-управление доступом.

## <a name="privilege-management"></a>Управление правами

Доступ к функциям управления или консолям администрирования систем, на которых размещаются виртуализированные системы, предоставляется только сотрудникам согласно принципу наименьших привилегий с применением технических элементов управления.

- К виртуальным машинам должно применяться JIT-управление доступом.
- \[Предварительная версия\] В службах Kubernetes следует использовать управление доступом на основе ролей (RBAC)

## <a name="review-of-user-access-rights"></a>Обзор прав доступа пользователей

Организация поддерживает документированный список полномочных пользователей для информационных ресурсов.

- \[Предварительная версия\] Отображение результатов аудита для конфигураций виртуальных машин Windows в разделе "Параметры безопасности — учетные записи"

## <a name="remote-diagnostic-and-configuration-port-protection"></a>Удаленная диагностика и защита портов конфигурации

Порты, службы и аналогичные приложения, установленные на компьютере или в сетевых системах, которые не требуются явным образом для выполнения бизнес-функций, отключены или удалены.

- Необходимо закрыть порты управления на виртуальных машинах.
- Уязвимости в конфигурации безопасности в масштабируемых наборах виртуальных машин должны быть устранены.

## <a name="audit-logging"></a>Ведение журнала аудита

Поддерживаются журналы отправляемых и получаемых сообщений с такими данными, как дата, время, источник и получатель сообщения, но без содержимого сообщений. Аудит доступен всегда, пока система активна. При этом отслеживаются ключевые события, успешные и неудачные попытки доступа к данным, изменение конфигурации безопасности, использование привилегированных или служебных программ, срабатывание оповещений, активация и деактивация систем защиты (например, A/V и IDS), активация и деактивация механизмов идентификации и проверки подлинности, а также создание и удаление объектов на уровне системы.

- Необходимо включить журналы диагностики в концентраторе событий
- Обновления системы должны быть установлены в масштабируемых наборах виртуальных машин.

## <a name="monitoring-system-use"></a>Мониторинг использования системы

В корпоративной среде развернуты автоматизированные системы для мониторинга за основными событиями и аномальными действиями, а также для анализа системных журналов, работа которых регулярно проверяется. Мониторинг охватывает привилегированные операции, авторизованный доступ или попытки несанкционированного доступа, включая попытки доступа к отключенным учетным записям, а также системные оповещения и сбои.

- Необходимо включить журналы диагностики в масштабируемых наборах виртуальных машин.

## <a name="segregation-of-duties"></a>Разделение обязанностей

Используется разделение обязанностей для ограничения риска несанкционированного или непреднамеренного изменения информации и конфигурации систем. Пользователи не могут получать доступ к информационным системам, а также изменять или использовать их без авторизации или обнаружения. Для лиц, ответственных за администрирование и управление доступом, предоставляется минимально необходимый уровень доступа с учетом ролей и обязанностей каждого пользователя, и у этих лиц нет доступа к функциям аудита, связанным с этими элементами управления.

- MFA должна быть включена для учетных записей с разрешениями на запись в вашей подписке.

## <a name="administrator-and-operator-logs"></a>Журналы администратора и оператора

Организация обеспечивает правильное ведение журнала для аудита действий администратора и регулярно просматривает журналы системных администраторов и операторов.

- MFA должна быть включена для учетных записей с разрешениями на запись в вашей подписке.

## <a name="identification-of-risks-related-to-external-parties"></a>Определение рисков, связанных с внешними сторонами

Подключения для обеспечения удаленного доступа между организацией и внешними сторонами шифруются

- К виртуальным машинам должно применяться шифрование дисков.

## <a name="business-continuity-and-risk-assessment"></a>Непрерывность бизнес-процессов и оценка рисков

Организация определяет критически важные бизнес-процессы и интегрирует требования к управлению информационной безопасностью, связанные с непрерывностью бизнес-процессов, с другими требованиями к непрерывности в контексте операций, персонала, материалов, транспорта и инфраструктуры.

- \[Предварительная версия\] Отображение результатов аудита для конфигураций виртуальных машин Windows в разделе "Параметры безопасности — консоль восстановления"

# <a name="control-mapping-of-the-nist-sp-800-53-r4-blueprint-sample"></a>Сопоставление элементов управления для образца схемы "NIST SP 800-53 R4"

В статье подробно описано, как образец схемы Azure Blueprints "NIST SP 800-53 R4" сопоставляется с элементами управления NIST SP 800-53 R4. См. подробнее об [элементах управления NIST SP 800-53](https://nvd.nist.gov/800-53).

Далее представлены сопоставления для элементов управления **NIST SP 800-53 (редакция 4)** . Используйте панель навигации справа для перехода непосредственно к сопоставлению конкретных элементов управления. Многие сопоставленные элементы управления реализуются с помощью инициативы [Политика Azure](../../../policy/overview.md). Чтобы просмотреть полную инициативу, откройте раздел **Политика** на портале Azure и перейдите на страницу **Определения**. Найдите и выберите функцию **[Предварительный просмотр]: Аудит элементов управления NIST SP 800-53 R4 и развертывание определенных расширений виртуальной машины для обеспечения требований аудита**.

## <a name="ac-2-account-management"></a>Управление учетными записями AC-2

Эта схема поможет вам проверить учетные записи, которые могут не соответствовать требованиям к управлению учетными записями, которые действуют в вашей организации. Эта схема назначает пять определений Политики Azure, которые проводят аудит внешних учетных записей с разрешениями на чтение, запись и управление для подписок и устаревших учетных записей. Проверив учетные записи, обнаруженные этими политиками, вы сможете предпринять необходимые действия для выполнения требований к управлению учетными записями.

- [Предварительный просмотр]: Audit deprecated accounts on a subscription
- [Предварительный просмотр]: Audit deprecated accounts with owner permissions on a subscription
- [Предварительный просмотр]: Audit external accounts with owner permissions on a subscription
- [Предварительный просмотр]: Audit external accounts with read permissions on a subscription
- [Предварительный просмотр]: Audit external accounts with write permissions on a subscription

## <a name="ac-2-7-account-management--role-based-schemes"></a>AC-2 (7) Управление учетными записями | Схемы на основе ролей

В Azure реализовано [управление доступом на основе ролей](../../../../role-based-access-control/overview.md) (RBAC). Оно упрощает управление пользователями, которые обращаются к ресурсам Azure. С помощью портала Azure вы можете просмотреть, у кого есть доступ к ресурсам Azure и какие им назначены разрешения. В этой схеме назначаются еще два определения [Политики Azure](../../../policy/overview.md) для аудита использования аутентификации Azure Active Directory для SQL Server и Service Fabric. Использование проверки подлинности Azure Active Directory упрощает контроль разрешений и обеспечивает централизованное управление удостоверениями для пользователей баз данных и других служб Майкрософт.

- Аудит подготовки администратора Azure Active Directory для SQL Server
- Аудит использования Azure Active Directory для проверки подлинности клиента в Service Fabric

## <a name="ac-2-12-account-management--account-monitoring--atypical-usage"></a>AC-2 (12) Управление учетными записями | Мониторинг учетной записи и нетипичное использование

JIT-доступ к виртуальной машине блокирует входящий трафик к виртуальным машинам Azure, что снижает уязвимость к атакам и обеспечивает удобный доступ к виртуальным машинам, когда это необходимо. Все запросы JIT-доступа к виртуальным машинам регистрируются в журнале действий, что позволяет отслеживать нетипичное использование. Эта схема назначает определение [Политики Azure](../../../policy/overview.md), которое поможет вам отслеживать виртуальные машины с поддержкой JIT-доступа, для которых он еще не настроен.

- [Предварительный просмотр] Мониторинг возможного JIT-доступа к сети в Центре безопасности Azure

## <a name="ac-4-information-flow-enforcement"></a>AC-4 Управление потоком информации

Механизм CORS (общий доступ к ресурсам независимо от источника) позволяет обращаться к ресурсам служб приложений из внешних доменов. Корпорация Майкрософт рекомендует разрешать доступ только из тех доменов, которым необходимо взаимодействовать с вашими интерфейсами API, функциями и (или) веб-приложениями. Эта схема назначает определение [Политики Azure](../../../policy/overview.md) для мониторинга ограничений доступа к ресурсам CORS в Центре безопасности Azure.
Понимание реализаций CORS поможет вам убедиться, что элементы управления потоками информации реализованы правильно.

- [Предварительный просмотр]: Audit CORS resource access restrictions for a Web Application

## <a name="ac-5-separation-of-duties"></a>AC-5 Разделение обязанностей

Наличие только одного владельца подписки Azure не позволит организовать административную избыточность. И наоборот, слишком большое число владельцев подписки Azure может увеличить вероятность бреши в системе безопасности через скомпрометированную учетную запись владельца. Эта схема помогает сохранить требуемое число владельцев подписки Azure, назначая два определения [Политики Azure](../../../policy/overview.md) для аудита числа владельцев для подписок Azure. Также эта схема назначает четыре определения Политики Azure, которые помогут вам управлять членством в группе администраторов на виртуальных машинах Windows. Управление разрешениями владельца подписки и администратора виртуальной машины помогает реализовать правильное разделение обязанностей.

- [Предварительный просмотр]: Audit maximum number of owners for a subscription
- [Предварительный просмотр]: Audit minimum number of owners for subscription
- Проверка отсутствия указанных членов в группе администраторов на виртуальных машинах Windows
- Проверка наличия указанных членов в группе администраторов на виртуальных машинах Windows
- Развертывание расширения виртуальной машины для проверки отсутствия указанных членов в группе администраторов на виртуальных машинах Windows
- Развертывание расширения виртуальной машины для проверки наличия указанных членов в группе администраторов на виртуальных машинах Windows

Наблюдая за учетными записями без включенной многофакторной проверки подлинности, вы можете определить учетные записи, которые с большей вероятностью могли быть скомпрометированы.

## <a name="sc-7-boundary-protection"></a>SC-7 Защита границ

Эта схема упрощает управление системными границами и контроль над ними, назначая определение [Политики Azure](../../../policy/overview.md) для мониторинга групп безопасности сети с разрешительными правилами. Эта схема также назначает определение политики, которое отслеживает рекомендации по укреплению безопасности для группы сетевой безопасности в Центре безопасности Azure. Центр безопасности Azure анализирует трафик виртуальных машин с выходом в Интернет и предоставляет рекомендации по правилам для групп безопасности сети с целью уменьшить потенциальную поверхность атак.
Эта схема также назначает три определения Политики Azure для мониторинга незащищенных конечных точек, приложений и учетных записей хранения. Использование конечных точек и приложений, которые не защищены брандмауэром, а также учетных записей хранения с неограниченным доступом может привести к нежелательному доступу к данным, содержащимся в информационной системе.

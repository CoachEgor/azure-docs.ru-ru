# <a name="recommendation-mapping-of-the-cis-microsoft-azure-foundations-benchmark-blueprint-sample"></a>Сопоставление рекомендаций из примера проекта теста производительности CIS для платформ Microsoft Azure

В следующей статье подробно описано, как Azure Blueprints в примере схемы проекта теста производительности CIS для платформ Microsoft Azure сопоставляется с рекомендациями для теста производительности CIS для платформ Microsoft Azure. Дополнительные сведения о рекомендациях см. в разделе [Тест производительности CIS для платформ Microsoft Azure](https://www.cisecurity.org/benchmark/azure/).

Ниже приведены сопоставления соответствующие рекомендациям **Теста производительности CIS для платформ Microsoft Azure v1.1.0**. Используйте навигацию справа для перехода непосредственно к сопоставлению конкретной рекомендации.
Многие сопоставленные рекомендации реализуются с помощью инициативы [Политика Azure](../../../policy/overview.md). Чтобы просмотреть полную инициативу, откройте раздел **Политика** на портале Azure и перейдите на страницу **Определения**. Затем найдите и выберите встроенную инициативу политики **\[Предварительная версия\] Рекомендации по аудиту теста производительности CIS для платформ Microsoft Azure v1.1.0 и развертыванию определенных расширений виртуальной машины для поддержки требований аудита**.

> [!NOTE]
> Пример полной схемы ожидается в ближайшее время. Связанная инициатива политики Azure теперь доступна.

## <a name="11-ensure-that-multi-factor-authentication-is-enabled-for-all-privileged-users"></a>1.1 Убедитесь, что многофакторная аутентификация включена для всех привилегированных пользователей

Эта схема назначает определения [Политики Azure](../../../policy/overview.md), которые соответствуют этой рекомендации CIS.

- MFA должна быть включена для учетных записей с разрешениями владельца в вашей подписке.
- MFA должна быть включена для учетных записей с разрешениями на запись в вашей подписке.

## <a name="12-ensure-that-multi-factor-authentication-is-enabled-for-all-non-privileged-users"></a>1.2 Убедитесь, что многофакторная аутентификация включена для всех непривилегированных пользователей

Эта схема назначает определение [Политики Azure](../../../policy/overview.md), которое соответствует этой рекомендации CIS.

- MFA должна быть включена для учетных записей с разрешениями на чтение в вашей подписке.

## <a name="13-ensure-that-there-are-no-guest-users"></a>1.3 Убедитесь, что гостевых пользователей не существует

Эта схема назначает определения [Политики Azure](../../../policy/overview.md), которые соответствуют этой рекомендации CIS.

- Внешние учетные записи с разрешениями владельца следует удалять из подписки.
- Внешние учетные записи с разрешениями на чтение следует удалять из подписки.
- Внешние учетные записи с разрешениями на запись следует удалять из подписки.

## <a name="23-ensure-asc-default-policy-setting-monitor-system-updates-is-not-disabled"></a>2.3 Убедитесь, что параметр политики ASC "Мониторинг обновлений системы" по умолчанию не имеет значение "Отключен"

Эта схема назначает определение [Политики Azure](../../../policy/overview.md), которое соответствует этой рекомендации CIS.

- На ваших компьютерах должны быть установлены обновления системы

## <a name="24-ensure-asc-default-policy-setting-monitor-os-vulnerabilities-is-not-disabled"></a>2.4 Убедитесь, что параметр политики ASC "Мониторинг уязвимостей ОС" по умолчанию не имеет значение "Отключен"

## <a name="25-ensure-asc-default-policy-setting-monitor-endpoint-protection-is-not-disabled"></a>2.5 Убедитесь, что параметр политики ASC "Мониторинг Endpoint Protection" по умолчанию не имеет значение"Отключен"

Эта схема назначает определение [Политики Azure](../../../policy/overview.md), которое соответствует этой рекомендации CIS.

- Мониторинг отсутствия Endpoint Protection в Центре безопасности Azure

## <a name="26-ensure-asc-default-policy-setting-monitor-disk-encryption-is-not-disabled"></a>2.6 Убедитесь, что параметр политики ASC "Мониторинг шифрования дисков" по умолчанию не имеет значение"Отключен"

Эта схема назначает определение [Политики Azure](../../../policy/overview.md), которое соответствует этой рекомендации CIS.

- Шифрование дисков должно применяться к виртуальным машинам.

## <a name="28-ensure-asc-default-policy-setting-monitor-web-application-firewall-is-not-disabled"></a>2.8 Убедитесь, что параметр политики ASC "Мониторинг брандмауэра веб-приложений" по умолчанию не имеет значение "Отключен"

Эта схема назначает определение [Политики Azure](../../../policy/overview.md), которое соответствует этой рекомендации CIS.

- Строгость правил группы безопасности сети для веб-приложений в IaaS должна быть увеличена.

## <a name="210-ensure-asc-default-policy-setting-monitor-vulnerability-assessment-is-not-disabled"></a>2.10 Убедитесь, что параметр политики ASC "Мониторинг оценки уязвимостей" по умолчанию не имеет значение"Отключен"

## <a name="212-ensure-asc-default-policy-setting-monitor-jit-network-access-is-not-disabled"></a>2.12 Убедитесь, что параметр политики ASC "Мониторинг JIT-доступа к сети" по умолчанию не имеет значение"Отключен"

Эта схема назначает определение [Политики Azure](../../../policy/overview.md), которое соответствует этой рекомендации CIS.

- Управление JIT-доступом к сети должно применяться к виртуальным машинам.

## <a name="215-ensure-asc-default-policy-setting-monitor-sql-encryption-is-not-disabled"></a>2.15 Убедитесь, что параметр политики ASC "Мониторинг шифрования SQL" по умолчанию не имеет значение"Отключен"

Эта схема назначает определение [Политики Azure](../../../policy/overview.md), которое соответствует этой рекомендации CIS.

## <a name="31-ensure-that-secure-transfer-required-is-set-to-enabled"></a>3.1 Убедитесь, что для параметра "Требуется безопасная передача данных" задано значение "Включено"

Эта схема назначает определение [Политики Azure](../../../policy/overview.md), которое соответствует этой рекомендации CIS.

- Должно выполняться безопасное перемещение в учетные записи хранения.

## <a name="37-ensure-default-network-access-rule-for-storage-accounts-is-set-to-deny"></a>3.7 Убедитесь, что в правиле сетевого доступа по умолчанию для учетных записей хранения задано значение "Отклонить"

Эта схема назначает определение [Политики Azure](../../../policy/overview.md), которое соответствует этой рекомендации CIS.

- Аудит неограниченного сетевого доступа к учетным записям хранения

## <a name="41-ensure-that-auditing-is-set-to-on"></a>4.1 Убедитесь, что для параметра "Аудит" задано значение "Включить"

Эта схема назначает определение [Политики Azure](../../../policy/overview.md), которое соответствует этой рекомендации CIS.

- Аудит должен быть включен в настройках расширенной защиты данных на SQL Server.

## <a name="42-ensure-that-auditactiongroups-in-auditing-policy-for-a-sql-server-is-set-properly"></a>4.2 Убедитесь, что параметр "AuditActionGroups" в "политике аудита" для SQL-сервера установлен правильно

Эта схема назначает определение [Политики Azure](../../../policy/overview.md), которое соответствует этой рекомендации CIS.

- В настройках аудита SQL должны быть настроены группы действий для записи критических действий

## <a name="43-ensure-that-auditing-retention-is-greater-than-90-days"></a>4.3 Убедитесь, что для срока удержания параметра "Аудит" задано значение "более 90 дней"

Эта схема назначает определение [Политики Azure](../../../policy/overview.md), которое соответствует этой рекомендации CIS.

- Для серверов SQL заданное значение периода удержания должно быть более 90 дней.

## <a name="44-ensure-that-advanced-data-security-on-a-sql-server-is-set-to-on"></a>4.4 Убедитесь, что для параметра "Расширенная защита данных" на сервере SQL задано значение "Включено"

Эта схема назначает определение [Политики Azure](../../../policy/overview.md), которое соответствует этой рекомендации CIS.

- Расширенная защита данных должна быть включена на серверах SQL Server.

## <a name="45-ensure-that-threat-detection-types-is-set-to-all"></a>4.5 Убедитесь, что для параметра "Типы обнаружения угроз" задано значение "Все"

Эта схема назначает определения [Политики Azure](../../../policy/overview.md), которые соответствуют этой рекомендации CIS.

- Для типов "Расширенной защиты от угроз" должно быть задано значение "Все" в настройках "Расширенной защиты данных" сервера SQL
- Для типов "Расширенной защиты от угроз" должно быть задано значение "Все" в настройках "Расширенной защиты данных" управляемого экземпляра SQL

## <a name="46-ensure-that-send-alerts-to-is-set"></a>4.6 Убедитесь, что задано значение "Отправить сообщения в"

Эта схема назначает определение [Политики Azure](../../../policy/overview.md), которое соответствует этой рекомендации CIS.

- Дополнительные параметры безопасности данных для сервера SQL должны содержать адрес электронной почты для получения оповещений системы безопасности

## <a name="47-ensure-that-email-service-and-co-administrators-is-enabled"></a>4.7 Убедитесь, что для "Служба электронной почты и соадминистраторов" задано значение "Включено"

Эта схема назначает определение [Политики Azure](../../../policy/overview.md), которое соответствует этой рекомендации CIS.

- Дополнительные параметры безопасности данных для управляемого экземпляра SQL должны содержать адрес электронной почты для получения оповещений системы безопасности

## <a name="48-ensure-that-azure-active-directory-admin-is-configured"></a>4.8 Убедитесь, что администратор Azure Active Directory настроен

## <a name="49-ensure-that-data-encryption-is-set-to-on-on-a-sql-database"></a>4.9 Убедитесь, что для параметра "Шифрование данных" в базе данных SQL установлено значение "Включено".

## <a name="410-ensure-sql-servers-tde-protector-is-encrypted-with-byok-use-your-own-key"></a>4.10 Убедитесь, что предохранитель TDE сервера SQL зашифрован с помощью BYOK (используйте собственный ключ)

Эта схема назначает определения [Политики Azure](../../../policy/overview.md), которые соответствуют этой рекомендации CIS.

- На серверах SQL предохранитель TDE должен быть зашифрован с использованием вашего ключа
- Предохранитель TDE управляемого экземпляра SQL должен быть зашифрован с помощью вашего собственного ключа

## <a name="517-ensure-that-logging-for-azure-keyvault-is-enabled"></a>5.1.7 Убедитесь, что для параметра "Ведение журнала для Azure KeyVault" задано значение "Включено"

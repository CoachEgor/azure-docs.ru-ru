# <a name="create-management-groups-for-resource-organization-and-management"></a>Создание групп управления для упорядочения ресурсов и управления ими

Группы управления — это контейнеры, которые помогают управлять доступом, политикой и соответствием требованиям в нескольких подписках. Создание таких контейнеров позволяет построить эффективную и экономную иерархию, которую можно использовать с [политикой Azure](../policy/overview.md) и [элементами управления доступом на основе ролей Azure](../../role-based-access-control/overview.md). Дополнительные сведения о группах управления см. в статье [Упорядочение ресурсов с помощью групп управления Azure](overview.md).

Создание первой группы управления в каталоге может занять до 15 минут. Существуют процессы, выполняемые в первый раз при настройке службы групп управления в Azure для вашего каталога. По завершении процесса вы получите уведомление. Для получения дополнительной [информации см.](./overview.md#initial-setup-of-management-groups) 

## <a name="create-a-management-group"></a>Создание группы управления

Любой пользователь Azure AD в наемщике может создать группу управления без разрешения, назначенного этому пользователю.  Эта новая группа управления будет ребенком группы root Management Group, и создатель получит ролевую должность "Владелец". Служба группы управления позволяет эту способность, так что назначения ролей не нужны на корневом уровне. При создании пользователи не имеют доступа к Группе управления root.  Чтобы избежать препятствий на хотеть глобальных адских адских адских администраций Для начала использования групп управления, мы позволяем создавать первоначальные группы управления на корневом уровне.      

Вы можете создать группу управления с помощью портала, [шаблона менеджера ресурсов,](../../azure-resource-manager/templates/deploy-to-tenant.md#create-management-group)PowerShell или Azure CLI.

### <a name="create-in-portal"></a>Создание на портале

1. Войти на [портал Azure](https://portal.azure.com).

1. Выберите **Все службы** > **Управления и управления**.

1. Выберите **управление затратами и выставление счетов**

1. На странице Управления затратами и выставления счетов - группы управления, выберите **группы управления**

1. Выберите **группу управления .**

   ![Страница для работы с группами управления](./media/main.png)

1. Заполните поле идентификатора группы управления.

   - **Идентификатор группы управления** — уникальный идентификатор каталога, используемый для отправки команд в этой группе управления. Идентификатор не редактируется после ее создания, так как с его помощью эта группа идентифицируется во всей системе Azure. [Группа управления корнем](overview.md#root-management-group-for-each-directory) автоматически создается с идентификатором, который является идентификатором Active Directory Azure. Для всех других групп управления назначаем уникальный идентификатор.
   - Поле отображаемого имени — это имя, которое отображается во всех разделах портала Azure. Отдельное отображаемое имя можно указать в необязательном поле при создании группы управления и изменить в любой момент.  

   ![Панель опций для создания новой группы управления](./media/create_context_menu.png)  

1. Щелкните **Сохранить**.

### <a name="create-in-powershell"></a>Создание в PowerShell

Для PowerShell используйте [cmdlet New-AzManagementGroup](/powershell/module/az.resources/new-azmanagementgroup) для создания новой группы управления.

```azurepowershell-interactive
New-AzManagementGroup -GroupName 'Contoso'
```

**GroupName** — уникальный создаваемый идентификатор. Этот идентификатор используется другими командами для ссылки на эту группу, и он не может быть изменен позже.

Если вы хотите, чтобы группа управления отобразила другое имя на портале Azure, добавьте параметр **DisplayName.** Например, для создания группы управления с GroupName Of Contoso и названием дисплея "Contoso Group" используйте следующее cmdlet:

```azurepowershell-interactive
New-AzManagementGroup -GroupName 'Contoso' -DisplayName 'Contoso Group'
```

В предыдущих примерах новая группа управления создается под корневой группой управления. Чтобы указать другую группу управления в качестве родительской, используйте параметр **ParentId.**

```azurepowershell-interactive
$parentGroup = Get-AzManagementGroup -GroupName Contoso
New-AzManagementGroup -GroupName 'ContosoSubGroup' -ParentId $parentGroup.id
```

### <a name="create-in-azure-cli"></a>Создание в Azure CLI

Для Azure CLI используйте группу [управления учетными записями для создания](/cli/azure/account/management-group?view=azure-cli-latest#az-account-management-group-create) новой группы управления.

```azurecli-interactive
az account management-group create --name Contoso
```

**Название** является уникальным идентификатором создается. Этот идентификатор используется другими командами для ссылки на эту группу, и он не может быть изменен позже.

Если вы хотите, чтобы группа управления отобразила другое имя на портале Azure, добавьте параметр **имени дисплея.** Например, для создания группы управления с GroupName Of Contoso и названием дисплея "Contoso Group" используйте следующую команду:

```azurecli-interactive
az account management-group create --name Contoso --display-name 'Contoso Group'
```

В предыдущих примерах новая группа управления создается под корневой группой управления. Чтобы указать другую группу управления в качестве родителя, используйте **родительский** параметр и укажите название родительской группы.

```azurecli-interactive
az account management-group create --name ContosoSubGroup --parent Contoso
```

# <a name="troubleshoot-issues-related-to-azure-data-box-and-azure-data-box-heavy"></a>Устранение неполадок, связанных с Azure Data Box и Azure Data Box Heavy

В этой статье приводятся сведения об устранении неполадок, которые могут возникнуть при использовании Azure Data Box или Azure Data Box Heavy. В статье содержится список возможных ошибок, возникающих при копировании данных в Data Box или при передаче данных из Data Box.

## <a name="error-classes"></a>Классы ошибок

Ошибки в Data Box и Data Box Heavy обобщены следующим образом:

| Категория ошибки *        | Описание        | Рекомендуемое действие    |
|----------------------------------------------|---------|--------------------------------------|
| Имя контейнера или общего ресурса | Имена контейнера или общего ресурса не соответствуют правилам именования Azure.  |Скачайте списки ошибок. <br> Переименуйте контейнеры или общие папки. [Узнайте больше](#container-or-share-name-errors).  |
| Ограничение размера контейнера или общего ресурса | Общий объем данных в контейнерах или общих ресурсах превышает предел в Azure.   |Скачайте списки ошибок. <br> Сократите общие данные в контейнере или общей папке. [Узнайте больше](#container-or-share-size-limit-errors).|
| Предельный размер объекта или файла | Объект или файлы в контейнерах или общих папках превышают предел в Azure.|Скачайте списки ошибок. <br> Уменьшите размер файла в контейнере или в общей папке. [Узнайте больше](#object-or-file-size-limit-errors). |    
| Тип данных или файла | Формат данных или тип файла не поддерживаются. |Скачайте списки ошибок. <br> Для страничных BLOB-объектов или управляемых дисков убедитесь, что данные находятся в 512-байтах и скопированы в предварительно созданные папки. [Узнайте больше](#data-or-file-type-errors). |
| Некритические ошибки BLOB-объекта или файла  | Имена BLOB-объектов или файлов не соответствуют правилам именования Azure, или тип файла не поддерживается. | Эти большие двоичные объекты или файлы не могут быть скопированы, или имена могут быть изменены. [Узнайте, как устранить эти ошибки](#non-critical-blob-or-file-errors). |

\*Первые четыре категории ошибок являются критическими и должны быть исправлены, прежде чем можно будет приступить к подготовке к отправке.


## <a name="container-or-share-name-errors"></a>Ошибки имени контейнера или общего ресурса

Это ошибки, связанные с контейнером и общими именами.

### <a name="error_container_or_share_name_length"></a>ERROR_CONTAINER_OR_SHARE_NAME_LENGTH     

**Описание ошибки.** Имя контейнера или общей папки должно содержать от 3 до 63 знаков. 

**Предлагаемое решение:** Папка в папке Data Box или Data Box Heavy (SMB/NFS), в которую вы скопировали данные, станет контейнером Azure в вашей учетной записи хранения. 

- На странице **подключение и копирование** локального пользовательского веб-интерфейса устройства Скачайте и проверьте файлы ошибок, чтобы найти имена папок с проблемами.
- Измените имя папки в папке Data Box или Data Box Heavy, чтобы убедиться в том, что:

    - Имя содержит от 3 до 63 символов.
    - Имена могут содержать только буквы, цифры и дефисы.
    - Имена не могут начинаться или заканчиваться дефисами.
    - Имена не могут иметь последовательные дефисы.
    - Примеры допустимых имен: `my-folder-1`,`my-really-extra-long-folder-111`
    - Примеры недопустимых имен `my-folder_1`:, `my`, `--myfolder`, `myfolder--`,`myfolder!`

    Дополнительные сведения см. в разделе соглашения об именовании Azure для [имен контейнеров](https://docs.microsoft.com/rest/api/storageservices/naming-and-referencing-containers--blobs--and-metadata#container-names) и [общих имен](https://docs.microsoft.com/rest/api/storageservices/naming-and-referencing-shares--directories--files--and-metadata#share-names).

## <a name="container-or-share-size-limit-errors"></a>Ошибки ограничения размера контейнера или общего ресурса

Это ошибки, связанные с данными, превышающими размер данных, допустимых в контейнере или общей папке.

### <a name="error_container_or_share_capacity_exceeded"></a>ERROR_CONTAINER_OR_SHARE_CAPACITY_EXCEEDED

**Описание ошибки.** Файловый ресурс Azure ограничивает общую папку до 5 ТБ данных. Превышено ограничение для некоторых общих папок.

**Предлагаемое решение:** На странице **подключение и копирование** локального веб-интерфейса Скачайте и проверьте файлы ошибок.

Найдите папки с этой проблемой в журналах ошибок и убедитесь, что файлы в этой папке имеют значение менее 5 ТБ.


## <a name="object-or-file-size-limit-errors"></a>Ошибки предельного размера объектов или файлов

Это ошибки, связанные с данными, превышающими максимальный размер объекта или файла, разрешенного в Azure. 

### <a name="error_blob_or_file_size_limit"></a>ERROR_BLOB_OR_FILE_SIZE_LIMIT

**Описание ошибки.** Размер файла превышает максимальный размер файла для отправки.

**Предлагаемое решение:** Размер большого двоичного объекта или файлов превышает максимально допустимое значение для отправки.

- На странице **подключение и копирование** локального веб-интерфейса Скачайте и проверьте файлы ошибок.
- Убедитесь, что размеры больших двоичных объектов и файлов не превышают ограничения на размер объектов Azure.

## <a name="data-or-file-type-errors"></a>Ошибки данных или типа файлов

Это ошибки, связанные с неподдерживаемым типом файла или типом данных, обнаруженным в контейнере или общей папке. 

### <a name="error_blob_or_file_size_alignment"></a>ERROR_BLOB_OR_FILE_SIZE_ALIGNMENT

**Описание ошибки.** Большой двоичный объект или файл неправильно выровнен.

**Предлагаемое решение:** Общая папка BLOB-объектов на Data Box или Data Box Heavy поддерживает только файлы с согласованным 512 байт (например, VHD/VHDX). Все данные, скопированные в общую папку BLOB-объектов, передаются в Azure в виде страничных BLOB-объектов.

Удалите все данные, не связанные с VHD или VHDX, из общего ресурса страничного BLOB-объекта. Вы можете использовать общие ресурсы для блочных BLOB-объектов или файлов Azure для универсальных данных.

Дополнительные сведения см. в разделе Общие сведения о [страничных BLOB-](../storage/blobs/storage-blob-pageblob-overview.md)объектах.

### <a name="error_blob_or_file_type_unsupported"></a>ERROR_BLOB_OR_FILE_TYPE_UNSUPPORTED

**Описание ошибки.** В управляемом общем диске есть неподдерживаемый тип файла. Разрешены только фиксированные виртуальные жесткие диски.

**Предлагаемое решение:**

- Убедитесь, что вы отправляете только фиксированные виртуальные жесткие диски для создания управляемых дисков.
- VHDX-файлы, **динамические** и **разностные** виртуальные жесткие диски не поддерживаются.

### <a name="error_directory_disallowed_for_type"></a>ERROR_DIRECTORY_DISALLOWED_FOR_TYPE

**Описание ошибки.** Каталог не разрешен в существующих папках для управляемых дисков. В этих папках разрешены только фиксированные виртуальные жесткие диски.

**Предлагаемое решение:** Для управляемых дисков в каждой общей папке создаются следующие три папки, которые соответствуют контейнерам в вашей учетной записи хранения: SSD (цен. категория "Премиум"), HDD (цен. категория "Стандартный") и SSD (цен. категория "Стандартный"). Эти папки соответствуют уровню производительности для управляемого диска.

- Скопируйте данные страничного BLOB-объекта (VHD) в одну из существующих папок.
- Папка или каталог не разрешены в существующих папках. Удалите все папки, созданные в уже существующих папках.

Дополнительные сведения см. [в статье копирование на управляемые диски](data-box-deploy-copy-data-from-vhds.md#connect-to-data-box).

### <a name="reparse_point_error"></a>REPARSE_POINT_ERROR

**Описание ошибки.** Символические ссылки не разрешены в Linux. 

**Предлагаемое решение:** Символьные ссылки обычно являются ссылками, каналами и другими такими файлами. Либо удалите ссылки, либо разрешите ссылки и скопируйте данные.


## <a name="non-critical-blob-or-file-errors"></a>Некритические ошибки BLOB-объекта или файла

В следующем разделе приведены все некритические ошибки, связанные с именами больших двоичных объектов, файлов или контейнеров, которые отображаются во время копирования данных. Если эти ошибки существуют, имена будут изменены в соответствии с соглашениями об именовании Azure. Соответствующее состояние заказа для отправки данных будет **выполнено с предупреждениями**.  

### <a name="error_blob_or_file_name_character_control"></a>ERROR_BLOB_OR_FILE_NAME_CHARACTER_CONTROL

**Описание ошибки.** Имена BLOB-объектов или файлов содержат неподдерживаемые управляющие символы.

**Предлагаемое решение:** Большие двоичные объекты или файлы, которые вы скопировали, содержат имена с неподдерживаемыми символами.

На странице **подключение и копирование** локального веб-интерфейса Скачайте и проверьте файлы ошибок.
Удалите или переименуйте файлы, чтобы удалить неподдерживаемые символы.

Дополнительные сведения см. в разделе соглашения об именовании в Azure для [имен больших двоичных объектов](https://docs.microsoft.com/rest/api/storageservices/Naming-and-Referencing-Containers--Blobs--and-Metadata#blob-names) и [имен файлов](https://docs.microsoft.com/rest/api/storageservices/naming-and-referencing-shares--directories--files--and-metadata#directory-and-file-names).

### <a name="error_blob_or_file_name_character_illegal"></a>ERROR_BLOB_OR_FILE_NAME_CHARACTER_ILLEGAL

**Описание ошибки.** Имя большого двоичного объекта или файла содержит недопустимые символы.

**Предлагаемое решение:** Большие двоичные объекты или файлы, которые вы скопировали, содержат имена с неподдерживаемыми символами.

На странице **подключение и копирование** локального веб-интерфейса Скачайте и проверьте файлы ошибок.
Удалите или переименуйте файлы, чтобы удалить неподдерживаемые символы.

Дополнительные сведения см. в разделе соглашения об именовании в Azure для [имен больших двоичных объектов](https://docs.microsoft.com/rest/api/storageservices/Naming-and-Referencing-Containers--Blobs--and-Metadata#blob-names) и [имен файлов](https://docs.microsoft.com/rest/api/storageservices/naming-and-referencing-shares--directories--files--and-metadata#directory-and-file-names).


### <a name="error_blob_or_file_name_ending"></a>ERROR_BLOB_OR_FILE_NAME_ENDING

**Описание ошибки.** Имена больших двоичных объектов и файлов заканчиваются недопустимыми символами.

**Предлагаемое решение:** Большие двоичные объекты или файлы, которые вы скопировали, содержат имена с неподдерживаемыми символами.

На странице **подключение и копирование** локального веб-интерфейса Скачайте и проверьте файлы ошибок.
Удалите или переименуйте файлы, чтобы удалить неподдерживаемые символы.

Дополнительные сведения см. в разделе соглашения об именовании в Azure для [имен больших двоичных объектов](https://docs.microsoft.com/rest/api/storageservices/Naming-and-Referencing-Containers--Blobs--and-Metadata#blob-names) и [имен файлов](https://docs.microsoft.com/rest/api/storageservices/naming-and-referencing-shares--directories--files--and-metadata#directory-and-file-names).


### <a name="error_blob_or_file_name_segment_count"></a>ERROR_BLOB_OR_FILE_NAME_SEGMENT_COUNT

**Описание ошибки.** Имя большого двоичного объекта или файла содержит слишком много сегментов пути.

**Предлагаемое решение:** Большие двоичные объекты или файлы, которые вы скопировали, превышают максимальное число сегментов пути. Сегмент пути — это строка между последовательными символами-разделителями, например косой чертой/.

- На странице **подключение и копирование** локального веб-интерфейса Скачайте и проверьте файлы ошибок.
- Убедитесь, что [имена больших двоичных объектов](https://docs.microsoft.com/rest/api/storageservices/Naming-and-Referencing-Containers--Blobs--and-Metadata#blob-names) и [файлов](https://docs.microsoft.com/rest/api/storageservices/naming-and-referencing-shares--directories--files--and-metadata#directory-and-file-names) соответствуют соглашениям об именовании Azure.

### <a name="error_blob_or_file_name_aggregate_length"></a>ERROR_BLOB_OR_FILE_NAME_AGGREGATE_LENGTH

**Описание ошибки.** Имя большого двоичного объекта файла слишком длинное.

**Предлагаемое решение:** Длина большого двоичного объекта или имени файла превышает максимально допустимое значение.

- На странице **подключение и копирование** локального веб-интерфейса Скачайте и проверьте файлы ошибок.
- Длина имени большого двоичного объекта не должна превышать 1 024 символов.
- Удалите или переименуйте большой двоичный объект или файлы, чтобы имена не превышали 1024 символов.

Дополнительные сведения см. в разделе соглашения об именовании в Azure для имен больших двоичных объектов и имен файлов.

### <a name="error_blob_or_file_name_component_length"></a>ERROR_BLOB_OR_FILE_NAME_COMPONENT_LENGTH

**Описание ошибки.** Один из сегментов в большом двоичном объекте или имени файла слишком длинный.

**Предлагаемое решение:** Длина одного из сегментов пути в большом двоичном объекте или имени файла превышает максимальное число символов. Сегмент пути — это строка между последовательными символами-разделителями, например косой чертой/.

- На странице **подключение и копирование** локального веб-интерфейса Скачайте и проверьте файлы ошибок.
- Убедитесь, что [имена больших двоичных объектов](https://docs.microsoft.com/rest/api/storageservices/Naming-and-Referencing-Containers--Blobs--and-Metadata#blob-names) и [файлов](https://docs.microsoft.com/rest/api/storageservices/naming-and-referencing-shares--directories--files--and-metadata#directory-and-file-names) соответствуют соглашениям об именовании Azure.


### <a name="error_container_or_share_name_disallowed_for_type"></a>ERROR_CONTAINER_OR_SHARE_NAME_DISALLOWED_FOR_TYPE

**Описание ошибки.** Указаны неправильные имена контейнеров для управляемых дисковых ресурсов.

**Предлагаемое решение:** Для управляемых дисков в каждой общей папке создаются следующие папки, соответствующие контейнерам в вашей учетной записи хранения: SSD (цен. категория "Премиум"), HDD (цен. категория "Стандартный") и SSD (цен. категория "Стандартный"). Эти папки соответствуют уровню производительности для управляемого диска.

- Скопируйте данные страничного BLOB-объекта (VHD) в одну из существующих папок. В Azure отправляются только данные из этих существующих контейнеров.
- Любая другая папка, созданная на том же уровне, что и SSD (цен. категория "Премиум"), HDD (цен. категория "Стандартный") и SSD (цен. категория "Стандартный"), не соответствует допустимому уровню производительности и не может использоваться.
- Удаляет файлы или папки, созданные за пределами уровней производительности.

Дополнительные сведения см. [в статье копирование на управляемые диски](data-box-deploy-copy-data-from-vhds.md#connect-to-data-box).

# <a name="web-application-firewall-crs-rule-groups-and-rules"></a>Группы и правила веб-приложений Firewall CRS

Брандмауэр веб-приложения шлюза приложений (WAF) позволяет устранить уязвимости веб-приложений и защитить их от эксплойтов. Это делается с помощью правил, которые определяются на основе установленных основных правил OWASP 3.1, 3.0 или 2.2.9. Эти правила могут быть отключены на основе правил. Эта статья содержит предлагаемые текущие правила и наборы правил.

## <a name="core-rule-sets"></a>Основные наборы правил

Приложение шлюз WAF поставляется предварительно настроены с CRS 3.0 по умолчанию. Но вместо этого вы можете использовать CRS 3.1 или CRS 2.2.9. CRS 3.1 предлагает новые наборы правил защиты от Java-инфекций, первоначальный набор проверок загрузки файлов, фиксированные ложные срабатывания и многое другое. CRS 3.0 предлагает снижение ложных срабатываний по сравнению с CRS 2.2.9. Вы также можете [настроить правила в соответствии с вашими потребностями.](application-gateway-customize-waf-rules-portal.md)

> [!div class="mx-imgBorder"]
> ![Управление правилами](../media/application-gateway-crs-rulegroups-rules/managed-rules-01.png)

WAF защищает от следующих веб-уязвимостей:

- Атак на инъекционные инъекции
- Атаки с использованием межузловых сценариев
- Другие распространенные атаки, такие как впрыск команды, контрабанда запроса HTTP, разделение ответов HTTP и удаленное включение файлов
- Нарушения протокола HTTP
- Аномалии протокола HTTP, такие как отсутствие пользователя-агента хоста и прием заголовков
- Боты, программы-обходчики и сканеры
- Общие неправильные настройки приложений (например, Apache и IIS)

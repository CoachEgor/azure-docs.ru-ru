# <a name="frequently-asked-questions-for-azure-web-application-firewall-on-azure-front-door-service"></a>Часто задаваемые вопросы для брандмауэра веб-приложений Azure на службе передних дверей Azure

В этой статье ответы на общие вопросы о функциях и функциональности брандмауэра веб-приложений Azure (WAF). 

## <a name="what-is-azure-waf"></a>Что такое Azure WAF?

Azure WAF — это брандмауэр веб-приложений, который помогает защитить веб-приложения от распространенных угроз, таких как инъекция S'L, кросс-сайт скриптов и другие веб-подвиги. Вы можете определить политику WAF, состоящую из комбинации пользовательских и управляемых правил для контроля доступа к веб-приложениям.

Политика Azure WAF может быть применена к веб-приложениям, размещенным на шлюзе приложений или передних дверях Azure.

## <a name="what-is-waf-on-azure-front-door"></a>Что такое WAF на передней двери Azure? 

Azure Front Door — это высокомасштабируемая, глобально распределенная сеть доставки приложений и контента. Azure WAF, когда интегрирована с Front Door, останавливает отказ в обслуживании и целенаправленные атаки приложений на краю сети Azure, близко к источникам атак, прежде чем они входят в виртуальную сеть, предлагает защиту без ущерба для производительности.

## <a name="does-azure-waf-support-https"></a>Поддерживает ли Azure WAF HTTPS?

Передняя дверь предлагает SSL разгрузки. WAF интегрируется с Front Door и может проверить запрос после его расшифровки.

## <a name="does-azure-waf-support-ipv6"></a>Поддерживает ли Azure WAF IPv6?

Да. Можно настроить ограничение IP для IPv4 и IPv6.

## <a name="how-up-to-date-are-the-managed-rule-sets"></a>Насколько современны наборы управляемых правил?

Мы делаем все возможное, чтобы идти в ногу с меняющимся ландшафтом угроз. После обновления нового правила оно добавляется в набор правил по умолчанию с новым номером версии.

## <a name="what-is-the-propagation-time-if-i-make-a-change-to-my-waf-policy"></a>Каково время распространения, если я вносю изменения в свою политику WAF?

Развертывание политики WAF во всем мире обычно занимает около 5 минут и часто завершается раньше.

## <a name="can-waf-policies-be-different-for-different-regions"></a>Могут ли политики WAF быть разными для разных регионов?

В сочетании с Front Door, WAF является глобальным ресурсом. Та же конфигурация применяется во всех местах передних дверей.
 
## <a name="how-do-i-limit-access-to-my-back-end-to-be-from-front-door-only"></a>Как ограничить доступ к бэк-энду только от передней двери?

Вы можете настроить IP-доступ Control List в бэк-энд, чтобы только Front Door исходящих IP-адресов диапазонов и отказать в любом прямом доступе из Интернета. Теги обслуживания поддерживаются для использования в виртуальной сети. Кроме того, можно проверить, что поле заголовка X-Forwarded-Host HTTP действительно для вашего веб-приложения.

## <a name="which-azure-waf-options-should-i-choose"></a>Какие варианты Azure WAF я должен выбрать?

При применении политик и средств WAF в Azure существует два варианта. WAF с Azure Front Door — это глобально распределенное решение для обеспечения безопасности. WAF с Application Gateway является региональным, специализированным решением. Мы рекомендуем вам выбрать решение, основанное на общих требованиях к производительности и безопасности. Для получения дополнительной [информации](https://docs.microsoft.com/azure/frontdoor/front-door-lb-with-azure-app-delivery-suite)см.


## <a name="do-you-support-same-waf-features-in-all-integrated-platforms"></a>Поддерживаете ли вы одни и те же функции WAF на всех интегрированных платформах?

В настоящее время правила ModSec CRS 2.2.9, CRS 3.0 и CRS 3.1 поддерживаются только с помощью WAF на портале приложений. Ограничения скорости, геофильтрация и управляемые azure правила набора правил по умолчанию поддерживаются только с помощью WAF на двери Azure Front Door.

## <a name="is-ddos-protection-integrated-with-front-door"></a>Интегрирована ли DDoS-защита с передней дверью? 

Глобально распределенный по краям сети Azure, Azure Front Door может поглощать и географически изолировать атаки большого объема. Можно создать пользовательскую политику WAF для автоматической блокировки и ограничения скорости http(s) атак, которые известны подписи. Кроме того, вы можете включить DDoS-стандарт защиты на VNet, где ваши задние концы развернуты. Клиенты Azure DDoS Protection Standard получают дополнительные преимущества, включая защиту затрат, гарантию SLA и доступ к экспертам из группы быстрого реагирования DDoS для немедленной помощи во время атаки.

## <a name="why-do-additional-requests-above-the-threshold-configured-for-my-rate-limit-rule-get-passed-to-my-backend-server"></a>Почему дополнительные запросы, превышают порог, настроенный для правила ограничения ставок, передаются на сервер бэкэнда?

Правило ограничения скорости может ограничить аномально высокий трафик с любого IP-адреса клиента. Вы можете настроить порог на количество веб-запросов, разрешенных с IP-адреса клиента в течение одной минуты или пяти минут. Для детального контроля скорости ограничение скорости может быть объединено с дополнительными условиями соответствия параметра HTTP(S). 

Запросы от одного и того же клиента часто поступают на тот же сервер Front Door. В этом случае вы увидите, что дополнительные запросы выше порога будут немедленно заблокированы. 

Тем не менее, вполне возможно, что запросы от одного и того же клиента могут поступать на другой сервер Front Door, который еще не обновил счетчик лимита ставок. Например, клиент может открыть новое соединение для каждого запроса, и порог низкий. В этом случае первый запрос на новый сервер Front Door пройдет проверку лимита скорости. Порог ограничения скорости обычно устанавливается высоким для защиты от атак отказа в обслуживании с любого IP-адреса клиента. Для очень низкого порога, вы можете увидеть дополнительные запросы выше порога пройти.

# <a name="get-started-with-storage-explorer"></a>Начало работы с Обозревателем службы хранилища

## <a name="overview"></a>Обзор

Обозреватель службы хранилища Microsoft Azure — это автономное приложение, которое упрощает работу с данными службы хранилища Azure в Windows, macOS и Linux. В этой статье вы узнаете о нескольких способах подключения к учетным записям хранения Azure и управления ими.

![Обозреватель службы хранилища Microsoft Azure][0]

## <a name="prerequisites"></a>Предварительные требования

# <a name="windowstabwindows"></a>[Windows](#tab/windows)

Обозреватель службы хранилища поддерживается в следующих версиях Windows:

* Windows 10 (рекомендуется).
* Windows 8
* Windows 7

Для всех версий Windows требуется .NET Framework 4.6.2 или более поздней версии.

# <a name="macostabmacos"></a>[macOS](#tab/macos)

Обозреватель службы хранилища поддерживается в следующих версиях macOS:

* macOS 10.12 "Sierra" и более поздних версий.

# <a name="linuxtablinux"></a>[Linux](#tab/linux)

Обозреватель службы хранилища доступен в [хранилище привязкой](https://snapcraft.io/storage-explorer) для большинства распространенных дистрибутивов Linux и является рекомендуемым методом установки. При публикации новых версий в хранилище привязкой Обозреватель службы хранилища автоматически устанавливаются все ее зависимости и обновления.

Список поддерживаемых дистрибутивов см. на [странице "привязанная установка](https://snapcraft.io/docs/installing-snapd)".

Обозреватель службы хранилища требует использования диспетчера паролей, который может потребоваться установить вручную, прежде чем Обозреватель службы хранилища будет работать правильно. Вы можете подключить Обозреватель службы хранилища к диспетчеру паролей системы, выполнив следующую команду:

```bash
snap connect storage-explorer:password-manager-service :password-manager-service
```

Обозреватель службы хранилища также доступен в виде загрузки. tar. gz, но вам потребуется установить зависимости вручную. Установка tar. gz поддерживается в следующих дистрибутивах Linux:

* Ubuntu 18,04 x64
* Ubuntu 16,04 x64
* Ubuntu 14.04 x64.

Установка tar. gz может работать с другими дистрибутивами, но официально поддерживаются только перечисленные ниже.

Дополнительные сведения об установке Обозреватель службы хранилища в Linux см. в разделе [руководство по устранению неполадок](https://docs.microsoft.com/azure/storage/common/storage-explorer-troubleshooting#linux-dependencies).

## <a name="connect-to-a-storage-account-or-service"></a>Подключение к учетной записи хранения или к службе

Обозреватель службы хранилища предоставляет несколько способов подключения к учетным записям хранения. В целом можно выполнить одно из следующих действий.

* [Войдите в Azure, чтобы получить доступ к подпискам и их ресурсам](#sign-in-to-azure)
* [Подключение определенного ресурса хранилища или CosmosDB](#attach-a-specific-resource)

### <a name="sign-in-to-azure"></a>Вход в Azure

> [!NOTE]
> Для полного доступа к ресурсам после входа в систему Обозреватель службы хранилища требуются разрешения на управление (Azure Resource Manager) и на уровне данных. Это означает, что требуются разрешения Azure Active Directory (Azure AD), которые предоставляют доступ к учетной записи хранения, контейнерам в учетной записи и данным в контейнерах. Если у вас есть разрешения только на уровне данных, попробуйте [Добавить ресурс через Azure AD](#add-a-resource-via-azure-ad). Дополнительные сведения о конкретных разрешениях, Обозреватель службы хранилища требуется, см. в разделе [руководство по устранению неполадок](https://docs.microsoft.com/azure/storage/common/storage-explorer-troubleshooting#rbac-permissions-issues).

1. В Обозреватель службы хранилища выберите **Управление учетными записями** , чтобы открыть панель управления учетной записью.

    ![Управление учетными записями][1]

2. В области слева отображаются все учетные записи Azure, в которые вы вошли. Чтобы подключиться к другой учетной записи, выберите **Добавить учетную запись**.

3. Если вы хотите войти в Национальный облако или Azure Stack, выберите в раскрывающемся списке **Среда Azure** , чтобы выбрать облако Azure, которое вы хотите использовать. После выбора среды нажмите кнопку **Вход** . Дополнительные сведения см. [в статье Connect обозреватель службы хранилища to a Azure Stack Subscription](/azure-stack/user/azure-stack-storage-connect-se).

    ![Параметр входа][2]

4. Когда вы выполните вход с помощью учетной записи Azure, учетная запись и подписки Azure, связанные с ней, будут добавлены в область слева. Выберите подписки Azure, с которыми вы собираетесь работать, а затем щелкните **Применить**. (При выборе **всех подписок** вы выбираете выбор между всеми или ни одной из перечисленных подписок Azure.)

    ![Выбор подписок Azure][3]

    На панели слева отобразятся все учетные записи хранения, связанные с выбранными подписками Azure.

    ![Выбранные подписки Azure][4]

### <a name="attach-a-specific-resource"></a>Присоединение определенного ресурса

Существует несколько способов присоединения к ресурсу в Обозреватель службы хранилища.

* [Добавление ресурса с помощью Azure AD](#add-a-resource-via-azure-ad). Если у вас есть разрешения только на уровне данных, используйте этот параметр, чтобы добавить контейнер больших двоичных объектов или контейнер хранилища BLOB-объектов Azure Data Lake Storage 2-го поколения.
* [Используйте строку подключения](#use-a-connection-string). Используйте этот параметр, если у вас есть строка подключения к учетной записи хранения. Обозреватель службы хранилища поддерживает строки подключения ключей и [SAS](storage/common/storage-dotnet-shared-access-signature-part-1.md) .
* [Используйте URI SAS](#use-a-sas-uri). Если у вас есть [URI SAS](storage/common/storage-dotnet-shared-access-signature-part-1.md) для контейнера больших двоичных объектов, общей папки, очереди или таблицы, используйте его для подключения к ресурсу. Чтобы получить URI SAS, можно либо использовать [Обозреватель службы хранилища](#generate-a-sas-in-storage-explorer) , либо [портал Azure](https://portal.azure.com).
* [Используйте имя и ключ](#use-a-name-and-key). Если вы знакомы с любыми ключами учетной записи хранения, этот параметр можно использовать для быстрого подключения. Ключи для учетной записи хранения находятся на панели **ключи доступа к** учетной записи хранения в [портал Azure](https://portal.azure.com).
* [Присоединение к локальному эмулятору](#attach-to-a-local-emulator). Если вы используете один из доступных эмуляторов хранения Azure, используйте этот параметр, чтобы легко подключиться к эмулятору.
* [Подключитесь к учетной записи Azure Cosmos DB с помощью строки подключения](#connect-to-an-azure-cosmos-db-account-by-using-a-connection-string). Используйте этот параметр, если у вас есть строка подключения к экземпляру CosmosDB.
* [Подключение к Azure Data Lake Store по URI](#connect-to-azure-data-lake-store-by-uri). Используйте этот параметр, если у вас есть универсальный код ресурса (URI) для Azure Data Lake Store.

#### <a name="add-a-resource-via-azure-ad"></a>Добавление ресурса с помощью Azure AD

1. Откройте диалоговое окно **Подключение** , нажав кнопку **Подключиться** на вертикальной панели инструментов слева:

    ![Варианты подключения к службе хранилища Azure][9]

2. Если вы еще не сделали этого, используйте параметр **Добавить учетную запись Azure** для входа в учетную запись Azure, которая имеет доступ к ресурсу. После входа в систему вернитесь в диалоговое окно " **Подключение** ".

3. Выберите **Добавить ресурс с помощью Azure Active Directory (Azure AD)** , а затем нажмите кнопку **Далее**.

4. Выберите учетную запись Azure и клиента, которые имеют доступ к ресурсу хранилища, к которому вы хотите подключиться. Щелкните **Далее**.

5. Выберите тип ресурса, который нужно присоединить, а затем введите сведения, необходимые для подключения. Сведения, которые вы вводите на этой странице, зависят от типа добавляемого ресурса. Убедитесь, что выбран правильный тип ресурса. После введения необходимых сведений нажмите кнопку **Далее**.

6. Проверьте сводку подключения и убедитесь, что все сведения указаны правильно. Если это так, выберите **подключить**. В противном случае нажмите кнопку **назад** , чтобы вернуться к предыдущим страницам, чтобы исправить неправильные данные.

После успешного добавления соединения дерево ресурсов автоматически перейдет к узлу, который представляет подключение. Если он не переходит на этот узел, перейдите в раздел **локальные & подключенные** > **учетные записи** > хранения **(прикрепленные контейнеры)**  > **контейнеры больших двоичных объектов**. Если Обозреватель службы хранилища не удалось добавить подключение или вам не удается получить доступ к данным после успешного добавления подключения, см. [руководство по устранению неполадок](https://docs.microsoft.com/azure/storage/common/storage-explorer-troubleshooting).

#### <a name="attach-to-a-local-emulator"></a>Присоединение к локальному эмулятору

В настоящее время Обозреватель службы хранилища поддерживает два официальных эмулятора хранения:
* [Эмулятор хранения Azure](storage/common/storage-use-emulator.md) (Только для Windows)
* [Азурите](https://github.com/azure/azurite) (Windows, macOS или Linux)

Если эмулятор прослушивает порты по умолчанию, можно использовать узел **Эмуляторы (порты по умолчанию** ) (находится в разделе **локальные & подключенные** > **учетные записи хранения**) для быстрого доступа к эмулятору.

Если вы хотите использовать другое имя для подключения или если эмулятор не работает на портах по умолчанию, выполните следующие действия.

1. Запустите эмулятор. В этом случае запишите порты, которые эмулятор прослушивает для каждого типа службы.

   > [!IMPORTANT]
   > Обозреватель службы хранилища не запускает эмулятор автоматически. Его необходимо запустить вручную.

2. Откройте диалоговое окно **Подключение** , нажав кнопку **Подключиться** на вертикальной панели инструментов слева:

    ![Варианты подключения к службе хранилища Azure][9]

3. Выберите **присоединить к локальному эмулятору**, а затем нажмите кнопку **Далее**.

4. Выберите отображаемое имя для подключения и введите порты, прослушиваемые вашим эмулятором для каждого типа службы. Текстовые поля будут начинаться с значений порта по умолчанию для большинства эмуляторов. Поле **Port Files (порт файлов** ) остается пустым, поскольку ни один из официальных эмуляторов сейчас не поддерживает службу файлов. Если используемый эмулятор поддерживает файлы, можно ввести порт, который используется. Затем нажмите кнопку **Далее**.

5. Проверьте сводку подключения и убедитесь, что все сведения указаны правильно. Если это так, выберите **подключить**. В противном случае нажмите кнопку **назад** , чтобы вернуться к предыдущим страницам, чтобы исправить неправильные данные.

После успешного добавления соединения дерево ресурсов автоматически перейдет к узлу, который представляет подключение. Если он не переходит на этот узел, просмотрите раздел **локальные & подключенные** > **учетные записи хранения**. Если Обозреватель службы хранилища не удалось добавить подключение или вам не удается получить доступ к данным после успешного добавления подключения, см. [руководство по устранению неполадок](https://docs.microsoft.com/azure/storage/common/storage-explorer-troubleshooting).

#### <a name="connect-to-an-azure-cosmos-db-account-by-using-a-connection-string"></a>Подключение к учетной записи Azure Cosmos DB с помощью строки подключения

В качестве альтернативы управлению учетными записями Azure Cosmos DB с помощью подписки Azure можно также подключиться к Azure Cosmos DBу с помощью строки подключения. Для этого выполните следующие действия.

1. В левой части дерева ресурсов разверните узел **локальные и подключенные**, щелкните правой кнопкой мыши **Azure Cosmos DB учетные записи**и выберите **подключиться к Azure Cosmos DB**.

    ![Подключение к Azure Cosmos DB с помощью строки подключения][21]

2. Выберите Azure Cosmos DB API, введите данные **строки подключения** и нажмите кнопку **ОК** , чтобы подключить учетную запись Azure Cosmos DB. Сведения о том, как получить строку подключения, см. в разделе [Получение строки подключения](https://docs.microsoft.com/azure/cosmos-db/manage-account).

    ![Строка подключения][22]

#### <a name="connect-to-azure-data-lake-store-by-uri"></a>Подключение Azure Data Lake Store по URI

Если вы хотите получить доступ к ресурсу, который не находится в вашей подписке, вам потребуется кто-то, у кого есть доступ к этому ресурсу, чтобы предоставить вам URI ресурса. После входа вы можете подключиться к Data Lake Store с помощью URI. Для этого выполните следующие действия.

1. Откройте обозреватель хранилищ.
2. В левой области разверните узел **Local and Attached** (Локальные и присоединенные).
3. Щелкните правой кнопкой мыши **Data Lake Store**. В контекстном меню выберите **подключиться к Data Lake Store**.

    ![подключение к контекстному меню Data Lake Store](./media/vs-azure-tools-storage-manage-with-storage-explorer/storageexplorer-adls-uri-attach.png)

4. Введите URI. Средство переходит к расположению только что введенного URL-адреса.

## <a name="generate-a-sas-in-storage-explorer"></a>Создание SAS в Обозреватель службы хранилища

### <a name="account-level-sas"></a>SAS уровня учетной записи

1. Щелкните правой кнопкой мыши учетную запись хранения, которую хотите поделиться, и выберите **получить подпись общего доступа**.

    ![Вариант получения SAS в контекстном меню][14]

2. В диалоговом окне **Создание подписи общего доступа** укажите временные рамки и разрешения для учетной записи, а затем нажмите кнопку **создать**.

    ![Диалоговое окно получения SAS][15]

3. Теперь можно скопировать **строку подключения** или необработанную **строку запроса** в буфер обмена.

### <a name="service-level-sas"></a>SAS уровня обслуживания

[Как получить SAS для контейнера больших двоичных объектов в Обозреватель службы хранилища](vs-azure-tools-storage-explorer-blobs.md#get-the-sas-for-a-blob-container)

## <a name="search-for-storage-accounts"></a>Поиск учетных записей хранилищ

Если вам нужно найти ресурс хранилища и не знать его расположение, можно использовать поле поиска в верхней части левой панели для поиска ресурса.

По мере ввода в поле поиска в области слева будут отображаться все ресурсы, имена которых соответствуют условию поиска. Например, на следующем снимке экрана показан поиск по условию **endpoints** (конечные точки):

![Поиск учетной записи хранения][23]

> [!NOTE]
> Чтобы ускорить поиск, отмените выбор подписок, которые не содержат искомого элемента, с помощью панели управления учетными записями. Можно также щелкнуть правой кнопкой мыши узел и выбрать в **нем поиск** , чтобы начать поиск с определенного узла.

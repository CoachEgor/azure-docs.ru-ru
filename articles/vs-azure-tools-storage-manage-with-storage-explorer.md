# <a name="get-started-with-storage-explorer"></a>Начало работы с Обозревателем службы хранилища

## <a name="overview"></a>Обзор

Обозреватель службы хранилища Azure — это автономное приложение, позволяющее легко работать с данными из хранилища Azure на платформе Windows, macOS и Linux. В этой статье рассматриваются различные способы подключения к учетным записям хранения Azure и управления ими.

![Обозреватель службы хранилища Microsoft Azure][0]

## <a name="prerequisites"></a>Технические условия

# <a name="windowstabwindows"></a>[Windows](#tab/windows)

Обозреватель службы хранилища Azure поддерживается в следующих версиях Windows:

* Windows 10 (рекомендуется).
* Windows 8
* Windows 7

Обязательно для всех версий Windows и .NET Framework версии 4.6.2 или выше.

[Скачайте и установите Обозреватель службы хранилища](https://www.storageexplorer.com).

# <a name="macostabmacos"></a>[macOS](#tab/macos)

Обозреватель службы хранилища Azure поддерживается в следующих версиях macOS:

* macOS 10.12 "Sierra" и более поздних версий.

[Скачайте и установите Обозреватель службы хранилища](https://www.storageexplorer.com).

# <a name="linuxtablinux"></a>[Linux](#tab/linux)

Обозреватель службы хранилища Azure поддерживается в следующих дистрибутивах Linux:

* Ubuntu 18.04 x64
* Ubuntu 16.04 x64
* Ubuntu 14.04 x64.

Обозреватель службы хранилища Azure может работать на других дистрибутивах, но официальная поддержка предоставляется только для дистрибутивов, приведенных выше.

Помощь при установке обозревателя службы хранилища в Linux, см. в разделе [руководство по устранению неполадок](https://docs.microsoft.com/azure/storage/common/storage-explorer-troubleshooting#linux-dependencies).

В [заметках о выпуске](https://go.microsoft.com/fwlink/?LinkId=838275&clcid=0x409) для Обозревателя службы хранилища Azure содержатся конкретные инструкции для некоторых дистрибутивов.

[Скачайте и установите Обозреватель службы хранилища](https://www.storageexplorer.com).

## <a name="connect-to-a-storage-account-or-service"></a>Подключение к учетной записи хранения или к службе

Обозреватель службы хранилища предоставляет несколько способов подключения к учетным записям хранения. В целом вы можете сделать следующее.

* [Войдите в Azure для доступа к вашей подписки и свои ресурсы](#sign-in-to-azure)
* [Подключить конкретный ресурс хранилища или CosmosDB](#attach-a-specific-resource)

### <a name="sign-in-to-azure"></a>Вход в Azure

> [!NOTE]
> Для полного доступа к ресурсам после входа в систему, обозреватель службы хранилища требуется management (ARM) и разрешения уровня данных. Это означает, что необходимые разрешения Azure AD, которые обеспечивают доступ к вашей учетной записи хранения, контейнеры в учетной записи и данных в контейнерах. Если у вас только разрешения на уровне данных, рассмотрите возможность использования [присоединения с помощью Azure AD](#add-a-resource-via-azure-ad). Дополнительные сведения о обозреватель службы хранилища требуется именно тех разрешений, см. в разделе [руководство по устранению неполадок](https://docs.microsoft.com/azure/storage/common/storage-explorer-troubleshooting?tabs=1804#role-based-access-control-permission-issues).
>
>

1. В Обозревателе службы хранилища выберите **Управление учетными записями**, чтобы перейти к **панели управления учетными записями**.

    ![Управление учетными записями][1]

2. В области слева отображаются все учетные записи Azure, в которые вы вошли. Чтобы подключиться к другой учетной записи, выберите **Добавить учетную запись**.

3. Если вы хотите войти в учетную запись в национальном облаке или в Azure Stack, щелкните раскрывающийся список **Окружение Azure**, чтобы выбрать нужное облако Azure. После выбора окружения нажмите кнопку **входа**. Если вы входите в Azure Stack, см. дополнительные сведения в статье [Подключение обозревателя службы хранилища к подписке Azure Stack](/azure-stack/user/azure-stack-storage-connect-se).

    ![Вариант входа][2]

4. Когда вы выполните вход с помощью учетной записи Azure, учетная запись и подписки Azure, связанные с ней, будут добавлены в область слева. Выберите нужные подписки Azure, а затем нажмите кнопку **Применить** (с помощью флажка **Все подписки** можно выбрать все указанные подписки Azure или отменить выбор всех подписок).

    ![Выбор подписок Azure][3]

    На панели слева отобразятся все учетные записи хранения, связанные с выбранными подписками Azure.

    ![Выбранные подписки Azure][4]

### <a name="attach-a-specific-resource"></a>Подключить конкретный ресурс
    
Существуют различные параметры для присоединения ресурса обозреватель службы хранилища. Вы можете:

* [Добавить к ресурсу с помощью Azure AD](#add-a-resource-via-azure-ad): При наличии разрешений только на уровне данных можно использовать этот параметр, чтобы добавить контейнер больших двоичных объектов или контейнер больших двоичных объектов поколения 2 ADLS.
* [Использование строки подключения](#use-a-connection-string): Если у вас есть строки подключения к учетной записи хранения. Обозреватель службы хранилища поддерживает оба ключа и [SAS](storage/common/storage-dotnet-shared-access-signature-part-1.md) строки подключения.
* [Использовать SAS URI](#use-a-sas-uri): Если у вас есть [SAS](storage/common/storage-dotnet-shared-access-signature-part-1.md) URI-адрес контейнера больших двоичных объектов, файловый ресурс, очереди или таблицы. Чтобы получить SAS URI, вы можете использовать [обозреватель хранилищ](#generate-a-sas-in-storage-explorer) или [портала Azure](https://portal.azure.com).
* [Использовать имя и ключ](#use-a-name-and-key): Если известно, что один из ключей учетной записи для вашей учетной записи хранения, можно использовать этот параметр, чтобы быстро подключиться. Ключи для вашей учетной записи хранения находятся в учетной записи хранения **ключи доступа** колонке в [портала Azure](https://portal.azure.com).
* [Присоединение к локальному эмулятору](#attach-to-a-local-emulator): Если вы используете один из доступных эмуляторов хранилища Azure, используйте этот параметр, чтобы легко подключиться к эмулятору.
* [Подключение к учетной записи Azure Cosmos DB с помощью строки подключения](#connect-to-an-azure-cosmos-db-account-by-using-a-connection-string) — Если у вас есть строки подключения к экземпляру CosmosDB.
* [Подключение к Azure Data Lake Store по URI](#connect-to-azure-data-lake-store-by-uri): Если у вас есть URI для Azure Data Lake Store.

#### <a name="add-a-resource-via-azure-ad"></a>Добавить к ресурсу с помощью Azure AD

1. Откройте **диалоговое окно подключения** , щелкнув **кнопка "Подключить"** с левой стороны, вертикальной панели инструментов.

    ![Варианты подключения к службе хранилища Azure][9]

2. Если вы еще не сделано, используйте **добавить учетную запись Azure** параметр для входа учетную запись Azure, которая имеет доступ к ресурсу. После входа в систему **диалоговое окно подключения**.

3. Выберите **к ресурсу с помощью Azure Active Directory (Azure AD) добавить** и нажмите кнопку **Далее**.

4. Выберите учетную запись Azure, который имеет право на ресурс хранилища, который требуется вложить, а также подписку, ресурс находится в и нажмите кнопку **Далее**.

5. Выберите тип ресурса, который требуется вложить, а затем введите сведения, необходимые для подключения. В зависимости от типа ресурса, при добавлении изменится входные данные на этой странице. Не забудьте выбрать правильный тип ресурса. Если вы ввели необходимые сведения, щелкните **Далее**.

6. Просмотр сводки подключения и проверьте правильность всех данных. Если все сведения верны, нажмите кнопку **Connect**, в противном случае вернитесь к предыдущим страницам с **обратно** кнопку, чтобы исправить любые неверные данные.

После успешного добавления подключения дереве ресурсов остаются будет автоматически переходить на узел, представляющий соединение. Если для какой-либо причине выглядит в разделе **локальный & Attached** → **учетные записи хранения** → **(присоединенного контейнеров)** → **контейнеры больших двоичных объектов** . Если не удалось добавить подключение обозревателя службы хранилища, или если не удается получить доступ к данным после успешного добавления подключения, а затем обратитесь к [руководство по устранению неполадок](https://docs.microsoft.com/azure/storage/common/storage-explorer-troubleshooting) для справки.

#### <a name="attach-to-a-local-emulator"></a>Присоединение к локальному эмулятору

Обозреватель службы хранилища поддерживает эмуляторы на всех платформах. В настоящее время официальный доступных эмуляторов они:
* [Эмулятор хранения Azure](storage/common/storage-use-emulator.md) (только Windows)
* [Azurite](https://github.com/azure/azurite) (Windows, macOS или Linux)

Если эмулятор работает на порты по умолчанию можно использовать **эмулятора - порты по умолчанию** узла, который всегда можно найти в разделе **локальный & Attached** → **учетные записи хранения** , для быстрого доступа к эмулятору. Если вы хотите использовать другое имя для соединения или если симулятор не выполняется на порты по умолчанию, выполните следующие действия.

1. Запуск симулятора. Когда вы запомните или запишите какие порты эмулятор прослушивает для каждого типа службы. Вам потребуются в дальнейшем.

   > [!IMPORTANT]
   > Обозреватель службы хранилища не запускает эмулятор автоматически. Его нужно запустить самостоятельно.

2. Откройте **диалоговое окно подключения** , щелкнув **кнопка "Подключить"** с левой стороны, вертикальной панели инструментов.

    ![Варианты подключения к службе хранилища Azure][9]

3. Выберите **присоединение к локальному эмулятору** и нажмите кнопку **Далее**.

4. Выберите отображаемое имя для подключения и введите в порты, которые симулятора прослушивает для каждого типа службы. По умолчанию текстовые поля будет содержать значения порта по умолчанию для большинства эмуляторов. **Файлы порт** также оставлено пустым по умолчанию как ни один из эмуляторов официальный в настоящее время поддерживают службу файлов. Если эмулятор, который вы используете поддерживает его, однако, можно ввести в порт, который уже используется. Нажмите кнопку **Далее**.

5. Просмотр сводки подключения и проверьте правильность всех данных. Если все сведения верны, нажмите кнопку **Connect**, в противном случае вернитесь к предыдущим страницам с **обратно** кнопку, чтобы исправить любые неверные данные.

После успешного добавления подключения дереве ресурсов остаются будет автоматически переходить на узел, представляющий соединение. Если для какой-либо причине выглядит в разделе **локальный & Attached** → **учетные записи хранения**. Если не удалось добавить подключение обозревателя службы хранилища, или если не удается получить доступ к данным после успешного добавления подключения, а затем обратитесь к [руководство по устранению неполадок](https://docs.microsoft.com/azure/storage/common/storage-explorer-troubleshooting) для справки.

#### <a name="connect-to-an-azure-cosmos-db-account-by-using-a-connection-string"></a>Подключение к учетной записи Azure Cosmos DB с помощью строки подключения

Помимо управления учетными записями Azure Cosmos DB через подписку Azure другим способом подключения к Azure Cosmos DB является использование строки подключения. Для подключения с помощью строки подключения выполните следующие действия.

1. Найдите элемент **Локальные и присоединенные** в дереве слева, щелкните правой кнопкой мыши **Учетные записи Azure Cosmos DB** и выберите **Подключиться к Azure Cosmos DB...**

    ![Подключение к Azure Cosmos DB с помощью строки подключения][21]

2. Выберите API Azure Cosmos DB, вставьте **строку подключения** и нажмите кнопку **ОК**, чтобы подключиться к учетной записи Azure Cosmos DB. Сведения о том, как получить строку подключения, см. в разделе [Получение строки подключения](https://docs.microsoft.com/azure/cosmos-db/manage-account).

    ![Строка подключения][22]

#### <a name="connect-to-azure-data-lake-store-by-uri"></a>Подключение Azure Data Lake Store по URI

Если вы хотите получить доступ к ресурсам, которых нет в вашей подписке, но при этом у вас есть разрешение на получение URI для ресурсов, вы можете подключиться к Data Lake Store с помощью URI после входа в систему. Выполните действия ниже.

1. Откройте обозреватель хранилищ.
2. В левой области разверните узел **Local and Attached** (Локальные и присоединенные).
3. Щелкните правой кнопкой мыши **Хранилище озера данных** и в контекстном меню выберите команду **подключения к Data Lake Store**.

    ![команда контекстного меню: подключение к Data Lake Store](./media/vs-azure-tools-storage-manage-with-storage-explorer/storageexplorer-adls-uri-attach.png)

4. Введите URI, и инструмент перейдет в расположение введенного URL-адреса.

    ![диалоговое окно после выбора команды контекстного меню: подключение к Data Lake Store](./media/vs-azure-tools-storage-manage-with-storage-explorer/storageexplorer-adls-uri-attach-dialog.png)

    ![результат подключения к Data Lake Store](./media/vs-azure-tools-storage-manage-with-storage-explorer/storageexplorer-adls-attach-finish.png)


## <a name="generate-a-sas-in-storage-explorer"></a>Создать SAS в обозревателе службы хранилища

### <a name="account-level-sas"></a>SAS уровня учетной записи

1. Щелкните правой кнопкой мыши учетную запись хранения, вы хотите поделиться, а затем выберите **Get Shared Access Signature...** .

    ![Вариант получения SAS в контекстном меню][14]

2. В диалоговом окне **Generate Shared Access Signature** (Создание подписанного URL-адреса) укажите период времени и разрешения для учетной записи, а затем нажмите кнопку **Создать**.

    ![Диалоговое окно получения SAS][15]

3. Теперь вы можете либо копирования **строку подключения** или необработанный **строка запроса** в буфер обмена.

## <a name="search-for-storage-accounts"></a>Поиск учетных записей хранилищ

Если вам нужно найти ресурс хранилища, но вы не знаете, где он находится, можно воспользоваться полем поиска в верхней части области слева.

По мере ввода в поле поиска в области слева будут отображаться все ресурсы, имена которых соответствуют условию поиска. Например, на следующем снимке экрана показан поиск по условию **endpoints** (конечные точки):

![Поиск учетной записи хранения][23]

> [!NOTE]
> Чтобы отменить выбор подписок, которые не содержат искомых элементов, используйте **панель управления учетной записью**. Это позволит сократить время поиска. Можно также щелкнуть узел правой кнопкой мыши и выбрать **Искать от этого места**, чтобы начать поиск с конкретного узла.

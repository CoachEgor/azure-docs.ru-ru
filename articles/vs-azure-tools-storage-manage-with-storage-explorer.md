# <a name="get-started-with-storage-explorer"></a>Начало работы с Обозревателем службы хранилища

## <a name="overview"></a>Обзор

Обозреватель службы хранилища Microsoft Azure — это автономное приложение, которое позволяет легко работать с данными службы хранилища Azure в Windows, macOS и Linux. В этой статье рассматриваются различные способы подключения к учетным записям хранения Azure и управления ими.

![Обозреватель службы хранилища Microsoft Azure][0]

## <a name="prerequisites"></a>Предварительные требования

# <a name="windowstabwindows"></a>[Windows](#tab/windows)

Обозреватель службы хранилища поддерживается в следующих версиях Windows:

* Windows 10 (рекомендуется).
* Windows 8
* Windows 7

Обязательно для всех версий Windows и .NET Framework версии 4.6.2 или выше.

# <a name="macostabmacos"></a>[macOS](#tab/macos)

Обозреватель службы хранилища поддерживается в следующих версиях macOS:

* macOS 10.12 "Sierra" и более поздних версий.

# <a name="linuxtablinux"></a>[Linux](#tab/linux)

Обозреватель службы хранилища доступен в [хранилище привязкой](https://snapcraft.io/storage-explorer) для большинства распространенных дистрибутивов Linux и является рекомендуемым методом установки. Обозреватель службы хранилищаная привязка автоматически устанавливает все свои зависимости и обновления по мере публикации новых версий в хранилище привязкой.

Список поддерживаемых дистрибутивов см. на [странице с оснасткой установки](https://snapcraft.io/docs/installing-snapd).

Обозреватель службы хранилища требует использования диспетчера паролей, который может потребоваться подключить вручную, прежде чем Обозреватель службы хранилища будет работать правильно. Вы можете подключить Обозреватель службы хранилища к диспетчеру паролей системы, выполнив следующую команду:

```bash
snap connect storage-explorer:password-manager-service :password-manager-service
```

Обозреватель службы хранилища также доступен в виде загрузки. tar. gz, но вам потребуется установить зависимости вручную. Установка tar. gz поддерживается в следующих дистрибутивах Linux:

* Ubuntu 18,04 x64
* Ubuntu 16,04 x64
* Ubuntu 14.04 x64.

Установка tar. gz может работать с другими дистрибутивами, но официально поддерживаются только перечисленные выше.

Дополнительные сведения об установке Обозреватель службы хранилища в Linux см. в разделе [руководство по устранению неполадок](https://docs.microsoft.com/azure/storage/common/storage-explorer-troubleshooting#linux-dependencies).

## <a name="connect-to-a-storage-account-or-service"></a>Подключение к учетной записи хранения или к службе

Обозреватель службы хранилища предоставляет несколько способов подключения к учетным записям хранения. В целом можно выполнить одно из следующих действий.

* [Войдите в Azure, чтобы получить доступ к подпискам и их ресурсам](#sign-in-to-azure)
* [Подключение определенного ресурса хранилища или CosmosDB](#attach-a-specific-resource)

### <a name="sign-in-to-azure"></a>Войдите в Azure

> [!NOTE]
> Для полного доступа к ресурсам после входа в систему Обозреватель службы хранилища требуются разрешения на управление (ARM) и на уровне данных. Это означает, что вам нужны разрешения Azure AD, которые предоставляют доступ к учетной записи хранения, контейнерам в учетной записи и данным в контейнерах. Если у вас есть разрешения только на уровне данных, попробуйте использовать [присоединение с Azure AD](#add-a-resource-via-azure-ad). Дополнительные сведения о Обозреватель службы хранилища необходимых разрешений см. в разделе Руководство по [устранению неполадок](https://docs.microsoft.com/azure/storage/common/storage-explorer-troubleshooting?tabs=1804#role-based-access-control-permission-issues).

1. В Обозревателе службы хранилища выберите **Управление учетными записями**, чтобы перейти к **панели управления учетными записями**.

    ![Управление учетными записями][1]

2. В области слева отображаются все учетные записи Azure, в которые вы вошли. Чтобы подключиться к другой учетной записи, выберите **Добавить учетную запись**.

3. Если вы хотите войти в учетную запись в национальном облаке или в Azure Stack, щелкните раскрывающийся список **Окружение Azure**, чтобы выбрать нужное облако Azure. После выбора среды нажмите кнопку **Вход...** . Дополнительные сведения см. [в статье Connect обозреватель службы хранилища to a Azure Stack Subscription](/azure-stack/user/azure-stack-storage-connect-se).

    ![Параметр входа][2]

4. Когда вы выполните вход с помощью учетной записи Azure, учетная запись и подписки Azure, связанные с ней, будут добавлены в область слева. Выберите нужные подписки Azure, а затем нажмите кнопку **Применить** (с помощью флажка **Все подписки** можно выбрать все указанные подписки Azure или отменить выбор всех подписок).

    ![Выбор подписок Azure][3]

    На панели слева отобразятся все учетные записи хранения, связанные с выбранными подписками Azure.

    ![Выбранные подписки Azure][4]

### <a name="attach-a-specific-resource"></a>Присоединение определенного ресурса

К ресурсу можно присоединиться в Обозреватель службы хранилища с помощью различных параметров:

* [Добавление ресурса с помощью Azure AD](#add-a-resource-via-azure-ad): Если у вас есть разрешения только на уровне данных, этот параметр можно использовать для добавления контейнера больших двоичных объектов или ADLS 2-го поколения контейнера больших двоичных объектов.
* [Использовать строку подключения](#use-a-connection-string): Если у вас есть строка подключения к учетной записи хранения. Обозреватель службы хранилища поддерживает строки подключения ключей и [SAS](storage/common/storage-dotnet-shared-access-signature-part-1.md) .
* [Использовать URI SAS](#use-a-sas-uri): Если у вас есть URI [SAS](storage/common/storage-dotnet-shared-access-signature-part-1.md) для контейнера больших двоичных объектов, общей папки, очереди или таблицы. Чтобы получить URI SAS, можно либо использовать [Обозреватель службы хранилища](#generate-a-sas-in-storage-explorer) , либо [портал Azure](https://portal.azure.com).
* [Используйте имя и ключ](#use-a-name-and-key): Если вы знакомы с любыми ключами учетной записи хранения, этот параметр можно использовать для быстрого подключения. Ключи для учетной записи хранения находятся на панели **ключи доступа к** учетной записи хранения на [портал Azure](https://portal.azure.com).
* [Присоединение к локальному эмулятору](#attach-to-a-local-emulator): Если вы используете один из доступных эмуляторов хранения Azure, используйте этот параметр, чтобы легко подключиться к эмулятору.
* [Подключение к учетной записи Azure Cosmos DB с помощью строки подключения](#connect-to-an-azure-cosmos-db-account-by-using-a-connection-string) — При наличии строки подключения к экземпляру CosmosDB.
* [Подключение к Azure Data Lake Store по URI](#connect-to-azure-data-lake-store-by-uri): Если у вас есть универсальный код ресурса (URI) для Azure Data Lake Store.

#### <a name="add-a-resource-via-azure-ad"></a>Добавление ресурса с помощью Azure AD

1. Откройте **диалоговое окно Подключение** , нажав **кнопку подключиться** в левой части панели инструментов вертикальная.

    ![Варианты подключения к службе хранилища Azure][9]

2. Если вы еще не сделали этого, используйте параметр **Добавить учетную запись Azure** для входа в учетную запись Azure, которая имеет доступ к ресурсу. После входа вернитесь в **диалоговое окно подключения**.

3. Выберите параметр **Добавить ресурс с помощью Azure Active Directory (Azure AD)** и нажмите кнопку **Далее**.

4. Выберите учетную запись Azure и клиент, которые имеют доступ к ресурсу хранилища, к которому вы хотите подключиться. Нажмите кнопку **Далее**.

5. Выберите тип ресурса, который нужно присоединить, а затем введите сведения, необходимые для подключения. Входные данные на этой странице будут изменяться в зависимости от типа добавляемого ресурса. Убедитесь, что выбран правильный тип ресурса. Заполнив необходимые сведения, нажмите кнопку **Далее**.

6. Проверьте сводку подключения и убедитесь, что все сведения указаны правильно. Если все данные выглядят правильно, нажмите кнопку **подключить**. В противном случае вернитесь на предыдущие страницы с помощью кнопки **назад** , чтобы исправить неправильные данные.

После успешного добавления соединения дерево ресурсов автоматически перейдет к узлу, который представляет соединение. Вы также можете просмотреть **локальные & присоединен** → **учетные записи хранения** → **(подключенные контейнеры)** → **контейнеры больших двоичных объектов** , если по какой-либо причине это не так. Если Обозреватель службы хранилища не удалось добавить подключение или вы не можете получить доступ к данным после успешного добавления подключения, обратитесь к [руководству по устранению неполадок](https://docs.microsoft.com/azure/storage/common/storage-explorer-troubleshooting) .

#### <a name="attach-to-a-local-emulator"></a>Присоединение к локальному эмулятору

В настоящее время Обозреватель службы хранилища поддерживает два официальных эмулятора хранения:
* [Эмулятор хранения Azure](storage/common/storage-use-emulator.md) (Только для Windows)
* [Азурите](https://github.com/azure/azurite) (Windows, macOS или Linux)

Если ваш эмулятор прослушивает порты по умолчанию, вы можете использовать узел **Эмуляторы — порты по умолчанию** (в разделе **локальный & подключен** → **учетные записи хранения**), чтобы быстро получить доступ к эмулятору.

Если вы хотите использовать другое имя для подключения или если эмулятор не работает на портах по умолчанию:

1. Запустите эмулятор. В этом случае запишите, какие порты прослушивает эмулятор для каждого типа службы.

   > [!IMPORTANT]
   > Обозреватель службы хранилища не запускает эмулятор автоматически. Его нужно запустить самостоятельно.

2. Откройте **диалоговое окно Подключение** , нажав **кнопку подключиться** в левой части панели инструментов вертикальная.

    ![Варианты подключения к службе хранилища Azure][9]

3. Выберите параметр **присоединить к локальному эмулятору** и нажмите кнопку **Далее**.

4. Выберите отображаемое имя для подключения и введите порты, прослушиваемые вашим эмулятором для каждого типа службы. Текстовые поля будут начинаться с значений порта по умолчанию для большинства эмуляторов. **Порт Files** остается пустым, так как ни один из официальных эмуляторов сейчас не поддерживает службу файлов. Если используемый эмулятор поддерживает файлы, то можно ввести порт, который используется. Нажмите кнопку **Далее**.

5. Проверьте сводку подключения и убедитесь, что все сведения указаны правильно. Если все сведения верны, нажмите кнопку **Подключиться**. в противном случае вернитесь на предыдущие страницы с помощью кнопки **назад** , чтобы исправить неправильные данные.

После успешного добавления соединения дерево ресурсов автоматически перейдет к узлу, который представляет соединение. Можно также просмотреть **локальные & подключенные** → **учетные записи хранения** , если по какой бы то ни было причине это не так. Если Обозреватель службы хранилища не удалось добавить подключение или вы не можете получить доступ к данным после успешного добавления подключения, обратитесь к [руководству по устранению неполадок](https://docs.microsoft.com/azure/storage/common/storage-explorer-troubleshooting) .

#### <a name="connect-to-an-azure-cosmos-db-account-by-using-a-connection-string"></a>Подключение к учетной записи Azure Cosmos DB с помощью строки подключения

Помимо управления учетными записями Azure Cosmos DB через подписку Azure другим способом подключения к Azure Cosmos DB является использование строки подключения. Для подключения с помощью строки подключения выполните следующие действия.

1. Найдите элемент **Локальные и присоединенные** в дереве слева, щелкните правой кнопкой мыши **Учетные записи Azure Cosmos DB** и выберите **Подключиться к Azure Cosmos DB...**

    ![Подключение к Azure Cosmos DB с помощью строки подключения][21]

2. Выберите API Azure Cosmos DB, вставьте **строку подключения** и нажмите кнопку **ОК**, чтобы подключиться к учетной записи Azure Cosmos DB. Сведения о том, как получить строку подключения, см. в разделе [Получение строки подключения](https://docs.microsoft.com/azure/cosmos-db/manage-account).

    ![Строка подключения][22]

#### <a name="connect-to-azure-data-lake-store-by-uri"></a>Подключение Azure Data Lake Store по URI

Если вам нужен доступ к ресурсу, не подписке, вам потребуется кто-то с доступом, чтобы предоставить вам URI ресурса. После входа вы можете подключиться к Data Lake Store с помощью URI, выполнив следующие действия.

1. Откройте обозреватель хранилищ.
2. В левой области разверните узел **Local and Attached** (Локальные и присоединенные).
3. Щелкните правой кнопкой мыши **Хранилище озера данных** и в контекстном меню выберите команду **подключения к Data Lake Store**.

    ![команда контекстного меню: подключение к Data Lake Store](./media/vs-azure-tools-storage-manage-with-storage-explorer/storageexplorer-adls-uri-attach.png)

4. Введите URI, и инструмент перейдет в расположение введенного URL-адреса.

    ![диалоговое окно после выбора команды контекстного меню: подключение к Data Lake Store](./media/vs-azure-tools-storage-manage-with-storage-explorer/storageexplorer-adls-uri-attach-dialog.png)

    ![результат подключения к Data Lake Store](./media/vs-azure-tools-storage-manage-with-storage-explorer/storageexplorer-adls-attach-finish.png)


## <a name="generate-a-sas-in-storage-explorer"></a>Создание SAS в Обозреватель службы хранилища

### <a name="account-level-sas"></a>SAS уровня учетной записи

1. Щелкните правой кнопкой мыши учетную запись хранения, которую хотите поделиться, и выберите **получить подпись общего доступа...** .

    ![Вариант получения SAS в контекстном меню][14]

2. В диалоговом окне **Создание подписи общего доступа** укажите временные рамки и разрешения, необходимые для учетной записи. Нажмите кнопку **Создать**.

    ![Диалоговое окно получения SAS][15]

3. Теперь можно скопировать **строку подключения** или необработанную **строку запроса** в буфер обмена.

### <a name="service-level-sas"></a>SAS уровня обслуживания

[Как получить SAS для контейнера больших двоичных объектов в Обозреватель службы хранилища](vs-azure-tools-storage-explorer-blobs.md#get-the-sas-for-a-blob-container)

## <a name="search-for-storage-accounts"></a>Поиск учетных записей хранилищ

Если вам нужно найти ресурс хранилища и не знать его расположение, можно использовать поле поиска в верхней части левой панели для поиска ресурса.

По мере ввода в поле поиска в области слева будут отображаться все ресурсы, имена которых соответствуют условию поиска. Например, на следующем снимке экрана показан поиск по условию **endpoints** (конечные точки):

![Поиск учетной записи хранения][23]

> [!NOTE]
> Чтобы отменить выбор подписок, которые не содержат искомых элементов, используйте **панель управления учетной записью**. Это позволит сократить время поиска. Можно также щелкнуть узел правой кнопкой мыши и выбрать **Искать от этого места**, чтобы начать поиск с конкретного узла.

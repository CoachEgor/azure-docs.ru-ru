# <a name="create-and-schedule-your-first-job-by-using-azure-scheduler---azure-portal"></a>Создание и планирование первого задания с помощью планировщика Azure — портал Azure

> [!IMPORTANT]
> [Azure Logic Apps](../logic-apps/logic-apps-overview.md) заменяет планировщик Azure, который выводится из [эксплуатации](../scheduler/migrate-from-scheduler-to-logic-apps.md#retire-date). Чтобы продолжить работу с заданиями, настроенными в планировщике, выполните [миграцию на Azure Logic Apps](../scheduler/migrate-from-scheduler-to-logic-apps.md) как можно скорее.

Мы продемонстрируем, насколько легко можно создать и запланировать задание, а затем отслеживать его и управлять им.

Если у вас еще нет подписки Azure, [зарегистрируйтесь для получения бесплатной учетной записи Azure](https://azure.microsoft.com/free/).

## <a name="create-job"></a>Создание задания

1. Войдите на [портал Azure](https://portal.azure.com/).

1. В поле поиска Azure введите `scheduler` в качестве фильтра. В списке результатов выберите **коллекции заданий планировщика**и щелкните **создать**.

   ![Создать ресурса планировщика](./media/scheduler-get-started-portal/scheduler-v2-portal-marketplace-create.png)

   Теперь создайте задание, которое отправляет запрос GET на этот URL-адрес: `https://www.microsoft.com/` 

1. В разделе **Задание планировщика** введите следующие сведения:

   | Свойство | Пример значения | Description |
   |----------|---------------|-------------| 
   | **Название** | getMicrosoft | Имя задания. | 
   | **Коллекция заданий** | <*Имя коллекции заданий*> | Создайте коллекцию заданий или выберите существующую коллекцию. | 
   | **подписка** | <*Azure-subscription-name*> | Имя подписки Azure. | 
   |||| 

1. Выберите команду **Параметры действия — настроить**, укажите следующие сведения; закончив, нажмите кнопку **ОК**:

   | Свойство | Пример значения | Description |
   |----------|---------------|-------------| 
   | **Действие** | **Http** | Тип выполняемого действия. | 
   | **Метод** | **Get** | Вызываемый метод. | 
   | **URL-адрес** | **https://www.microsoft.com** | Конечный URL-адрес. | 
   |||| 
   
   ![Определите задание](./media/scheduler-get-started-portal/scheduler-v2-portal-action-settings.png)

1. Выберите пункт **Расписание — настроить**, определите расписание; закончив, нажмите кнопку **ОК**:

   Несмотря на то, что можно создать разовое задание, этот пример устанавливает расписание повторений.

   | Свойство | Пример значения | Description |
   |----------|---------------|-------------| 
   | **Периодичность** | **Повторяющееся** | Однократное или повторяющееся задание. | 
   | **Начинается** | <*Текущая дата*> | Дата начала задания | 
   | **Повторять через** | **1 ч** | Интервал и частота повторения | 
   | **Конец** | **Окончание** через два дня после текущей даты. | Дата окончания задания | 
   | **Смещение относительно UTC** | **UTC +08:00** | Разница во времени между временем в формате UTC и вашим локальным временем. | 
   |||| 

   ![Определите расписание](./media/scheduler-get-started-portal/scheduler-v2-portal-recurrence-schedule.png)

1. Когда вы будете готовы, нажмите кнопку **Создать**.

   Когда задание создано, Azure развертывает его; оно отображается в панели мониторинга Azure. 

1. Когда Azure отобразит уведомление о том, что развертывание прошло успешно, выберите команду **Закрепить на панели мониторинга**. Вы также можете выбрать значок **Уведомления** (колокольчик) на панели инструментов Azure, а затем выбрать команду **Закрепить на панели мониторинга**.

## <a name="monitor-and-manage-jobs"></a>Отслеживайте задания и управляйте ими

Для просмотра заданий, их мониторинга и управления ими выберите задание на панели мониторинга Azure. В разделе **Параметры** есть области сведений о задании, которые можно просматривать и которыми можно управлять:

![Параметры задания](./media/scheduler-get-started-portal/scheduler-v2-portal-job-overview-1.png)

Дополнительную информацию об этих областях см. в следующих разделах.

* [**Свойства**](#properties)
* [**Параметры действия**](#action-settings)
* [**Расписание**](#schedule)
* [**Журнал**](#history)
* [**Пользователи**](#users)

<a name="properties"></a>

### <a name="properties"></a>Свойства

Чтобы просмотреть свойства только для чтения, которые содержат метаданные управления для задания, выберите **Свойства**.

![Просмотр свойств задания](./media/scheduler-get-started-portal/scheduler-v2-portal-job-properties.png)

<a name="action-settings"></a>

### <a name="action-settings"></a>Параметры действия

Чтобы изменить дополнительные параметры задания, выберите **Параметры действия**. 

![Проверьте параметры действия](./media/scheduler-get-started-portal/scheduler-v2-portal-job-action-settings.png)

| Тип действия | Description | 
|-------------|-------------| 
| Все типы | Можно изменить **политику повторов** и **действие при возникновении ошибки**. | 
| HTTP и HTTPS | Вы можете изменить **Метод** на любой разрешенный метод. Кроме того, можно добавлять, удалять или изменять заголовки и данные для обычной проверки подлинности. | 
| Очередь службы хранилища| Можно изменить учетную запись хранения, имя очереди, маркер SAS и текст. | 
| Служебная шина | Можно изменить пространство имен, путь к разделу или очереди, параметры проверки подлинности, тип транспорта, свойства сообщения и текст сообщения. | 
||| 

<a name="schedule"></a>

### <a name="schedule"></a>Расписание

Если вы настроили расписание с помощью мастера задания, такие параметры этого расписания, как дата и время начала, расписание повторения и дату и время окончания, можно изменить.
Вы также можете [создавать более сложные расписания и расширенные схемы повторения](scheduler-advanced-complexity.md).

Чтобы изменить представление или расписание задания, выберите **Расписание**:

![Просмотр расписания заданий](./media/scheduler-get-started-portal/scheduler-v2-portal-job-schedule.png)

<a name="history"></a>

### <a name="history"></a>Журнал

Чтобы просмотреть метрики о каждом выполнении выбранного задания, выберите **Журнал**. Эти метрики предоставляют в реальном времени показатели работоспособности вашего задания, такие как состояние, число повторных попыток, число повторений, время начала и время окончания.

![Просмотр журнала и метрик задания](./media/scheduler-get-started-portal/scheduler-v2-portal-job-history.png)

Чтобы просмотреть в журнале данные об отдельном запуске, например полный ответ для каждого запуска, в разделе **Журнал** выберите соответствующий запуск. 

![Просмотр подробностей из журнала заданий](./media/scheduler-get-started-portal/scheduler-v2-portal-job-history-details.png)

<a name="users"></a>

### <a name="users"></a>Пользователи

Доступ к планировщику Azure для каждого пользователя на детальном уровне можно контролировать с помощью управления доступом на основе ролей (RBAC). Сведения о том, как настроить доступ на основе ролей, см. в разделе [Управление доступом с помощью RBAC](../role-based-access-control/role-assignments-portal.md).

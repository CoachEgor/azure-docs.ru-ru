# <a name="what-is-a-secured-virtual-hub"></a>Что такое защищенный виртуальный концентратор?

[!INCLUDE [Preview](../../includes/firewall-manager-preview-notice.md)]

Виртуальный концентратор — это виртуальная сеть, управляемая корпорацией Майкрософт, которая обеспечивает подключение с других ресурсов. Когда виртуальный концентратор создается из виртуального WAN на портале Azure, в качестве его компонентов создаются виртуальный концентратор VNet и шлюзы (необязательно).

*Защищенный* виртуальный концентратор — это [виртуальный wan-центр Azure](../virtual-wan/virtual-wan-about.md#resources) с связанными политиками безопасности и направления, настроенными менеджером брандмауэра Azure. Используйте защищенные виртуальные концентраторы для легкого создания узловидных и транзитных и переходных архитектур с помощью местных служб безопасности для управления трафиком и защиты. 

Защищенный виртуальный концентратор можно использовать в качестве управляемого центрального VNet без подключения на prem. Он заменяет центральную VNet, которая ранее требовалась для развертывания Azure Firewall. Поскольку защищенный виртуальный концентратор обеспечивает автоматизированную маршрутизирующую систему, нет необходимости настраивать собственные UDR (маршруты, определяемые пользователем) для маршрутизатирования трафика через брандмауэр.

Кроме того, можно использовать защищенные виртуальные концентраторы как часть полной виртуальной архитектуры WAN. Эта архитектура обеспечивает надежное, оптимизированное и автоматизированное подключение ветки к Azure и через нее. Вы можете выбрать службы для защиты и управления сетевым трафиком, включая Azure Firewall и других сторонних поставщиков безопасности в качестве поставщиков услуг (SECaaS).

## <a name="create-a-secured-virtual-hub"></a>Создание защищенного виртуального концентратора

Используя Firewall Manager на портале Azure, можно либо создать новый защищенный виртуальный концентратор, либо преобразовать существующий виртуальный концентратор, созданный ранее с помощью Azure Virtual WAN.

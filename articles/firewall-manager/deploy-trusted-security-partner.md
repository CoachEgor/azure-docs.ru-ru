# <a name="deploy-a-trusted-security-partner-preview"></a>Развертывание функции доверенного партнера по безопасности (предварительная версия)

[!INCLUDE [Preview](../../includes/firewall-manager-preview-notice.md)]

*Доверенные партнеры* по безопасности в Azure Firewall Manager позволяют использовать знакомые, лучшие в своем виде сторонние предложения по безопасности как услуги (SECaaS) для защиты доступа в Интернет для пользователей.

Чтобы узнать больше об поддерживаемых сценариях и рекомендациях по наилучшей практике, [см.](trusted-security-partners.md)

Для этого предварительного просмотра поддерживаемыми партнерами по безопасности являются **«Scaler»** и **iboss.** Поддерживаемые регионы: ЗападныйЦентральный, Северо-Центральный, Западный, Западный, Западный И Восточный.

## <a name="prerequisites"></a>Предварительные требования

> [!IMPORTANT]
> Предварительную версию Диспетчера брандмауэра Azure необходимо включить явным образом с помощью команды `Register-AzProviderFeature` в PowerShell.

В командной строке PowerShell выполните следующие команды:

```azure-powershell
connect-azaccount
Register-AzProviderFeature -FeatureName AllowCortexSecurity -ProviderNamespace Microsoft.Network
```
Регистрация функции занимает до 30 минут. Чтобы проверить состояние регистрации, выполните следующую команду:

`Get-AzProviderFeature -FeatureName AllowCortexSecurity -ProviderNamespace Microsoft.Network`

## <a name="deploy-a-third-party-security-provider-in-a-new-hub"></a>Развертывание стороннего поставщика безопасности в новом концентраторе

1. Войдите на портал Azure по адресу https://portal.azure.com.
2. В **поиске**введите **Firewall Manager** и выберите его в соответствии с **службами.**
3. Перейдите к **началу**работы . Выберите **Создать безопасный виртуальный концентратор.** 
4. Введите группу подписки и ресурсов, выберите поддерживаемый регион и добавьте концентратор и виртуальную информацию WAN. 
5. **Развертывание VPN шлюза** включено по умолчанию. ДЛЯ развертывания надежного партнера по безопасности в концентраторе требуется VPN-шлюз. 
6. Выберите **далее: Брандмауэр Azure**
   > [!NOTE]
   > Доверенные партнеры по безопасности подключаются к вашему концентратору с помощью туннелей VPN Gateway. При удалении VPN Шлюза, соединения с вашими доверенными партнерами по безопасности будут потеряны.
7. Если требуется развернуть Azure Firewall для фильтрации частного трафика вместе со сторонним поставщиком услуг для фильтрации интернет-трафика, выберите политику для Azure Firewall. Посмотреть [поддерживаемые сценарии](trusted-security-partners.md#key-scenarios).
8. Если требуется развернуть в концентраторе только стороннего поставщика безопасности, выберите **Azure Firewall: Enabled/Disabled,** чтобы настроить его **на отключенный.** 
9. Выберите **Далее: Доверенные партнеры по безопасности**.
10. Выберите **надежного партнера по безопасности,** чтобы настроить его на **Enabled.** Выберите партнера. 
11. Нажмите кнопку **Далее**. 
12. Просмотрите содержимое, а затем выберите **Создать**.

Развертывание VPN шлюза может занять более 30 минут.

Чтобы убедиться в создании концентратора, перейдите на azure Firewall Manager->Secured Hubs. Выберите страницу обзора >концентрата, чтобы показать имя партнера и статус **подключения к безопасности в ожидании.**

Как только концентратор создан и настроен партнер по безопасности, продолжайте подключать поставщика безопасности к концентратору.

## <a name="deploy-a-third-party-security-provider-in-an-existing-hub"></a>Развертывание стороннего поставщика безопасности в существующем концентраторе

Вы также можете выбрать существующий концентратор в виртуальном WAN и преобразовать его в *защищенный виртуальный концентратор.*

1. В **начале**работы выберите **Преобразование существующих концентраторов.**
2. Выберите подписку и существующий концентратор. Выполните остальные действия по развертыванию стороннего поставщика в новом концентраторе.

Помните, что VPN шлюз должен быть развернут для преобразования существующего концентратора в защищенный концентратор с сторонними поставщиками.

## <a name="configure-third-party-security-providers-to-connect-to-a-secured-hub"></a>Настройка сторонних поставщиков безопасности для подключения к защищенного концентратора

Для настройки туннелей для VPN Gateway вашего виртуального узла сторонним поставщикам необходимы права доступа к вашему концентратору. Для этого связывайте директора службы с вашей подпиской или группой ресурсов и предоставляете права доступа. Затем вы должны предоставить эти учетные данные третьей стороне с помощью их портала.

1. Создайте принцип службы Active Directory (AD): Вы можете пропустить URL-адрес перенаправления. 

   [Практическое руководство. Создание приложения Azure Active Directory и субъекта-службы с доступом к ресурсам с помощью портала](../active-directory/develop/howto-create-service-principal-portal.md#create-an-azure-active-directory-application)
2. Добавьте права доступа и область охвата для основного обслуживания.
   [Практическое руководство. Создание приложения Azure Active Directory и субъекта-службы с доступом к ресурсам с помощью портала](../active-directory/develop/howto-create-service-principal-portal.md#create-an-azure-active-directory-application)

   > [!NOTE]
   > Вы можете ограничить доступ только к группе ресурсов для более детального управления.
3. Следуйте [инструкциям по интеграции виртуальной WAN-интеграции Microsoft Azure](https://help.zscaler.com/zia/configuring-microsoft-azure-virtual-wan-integration) для:

   - Вопийте на портале-партнере и добавляйте свои учетные данные, чтобы предоставить доверенному партнеру доступ к защищенного концентратору.
   - Синхронизируйте виртуальные концентраторы в портале-партнере и навладийте туннель с виртуальным концентратором. Это можно сделать после проверки учетных данных проверки аутентификации Azure AD.
   
4. Вы можете посмотреть на состояние создания туннеля на портале Azure Virtual WAN в Azure. После того, как туннели **будут отображаться на** портале Azure и на портале-партнере, продолжайте дальнейшие шаги по настройке маршрутов, чтобы выбрать, какие ветви и VNets должны отправлять интернет-трафик партнеру.

## <a name="configure-route-settings"></a>Настройка настроек маршрута

1. Просмотрите доступ к менеджеру брандмауэра Azure -> защищенные концентраторы. 
2. Выберите концентратор. Статус концентратора должен теперь отображать **Provisioned** вместо **подключения к безопасности в ожидании**.

   Убедитесь, что сторонний поставщик может подключиться к концентратору. Туннели на VPN шлюзе должны быть в **подключенном** состоянии. Это состояние в большей степени отражает работоспособность соединения между концентратором и сторонним партнером, по сравнению с предыдущим статусом.
3. Выберите концентратор и перейдите к **настройкам маршрута.**

   При развертывании стороннего поставщика в концентратор он преобразует концентратор в *защищенный виртуальный концентратор.* Это гарантирует, что сторонний поставщик рекламирует 0.0.0.0/0 (по умолчанию) маршрут к концентратору. Однако соединения и сайты VNet, подключенные к концентратору, не получают этого маршрута, если только вы не выберете, на каком маршруте соединения должны быть по умолчанию.
4. Под **интернет-трафиком**выберите **VNet-to-Internet** или **branch-to-Internet** или оба маршрута настроены на отправку через третью сторону.

   Это только указывает, какой тип трафика должен быть направлен в концентратор, но это еще не влияет на маршруты на VNets или ветвях. Эти маршруты не распространяются на все VNets/ветви, прикрепленные к концентратору по умолчанию.
5. Необходимо выбрать **безопасные соединения** и выбрать соединения, на которых должны быть установлены эти маршруты. Это указывает на то, какие VNets/филиалы могут начать отправлять интернет-трафик стороннему поставщику.
6. Из **настроек маршрута**выберите **безопасные соединения** под интернет-трафиком, а затем выберите VNet или ветви *(сайты* в виртуальном WAN), которые будут защищены. Выберите **Безопасный интернет-трафик.**
   ![Безопасный интернет-трафик](media/deploy-trusted-security-partner/secure-internet-traffic.png)
7. Перейдите назад на страницу концентратов. Статус **надежного партнера** по безопасности концентратора теперь должен быть **защищен.**

## <a name="branch-or-vnet-internet-traffic-via-third-party-service"></a>Филиал или Интернет-трафик VNet через сторонний сервис

Далее можно проверить, можно ли получить доступ к Интернету виртуальным машинам VNet или на ветке, и подтвердить, что трафик перетекает в стороннюю службу.

После завершения этапов настройки маршрута виртуальные машины VNet, а также сайты филиалов отправляются по маршруту обслуживания 0/0 сторонней стороне. Вы не можете RDP или SSH в эти виртуальные машины. Для вху-на-е можно развернуть службу [Azure Bastion](../bastion/bastion-overview.md) в заглянувом VNet.

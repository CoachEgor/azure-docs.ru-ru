# <a name="what-are-the-azure-firewall-manager-architecture-options"></a>Каковы параметры архитектуры Azure Firewall Manager?

Менеджер брандмауэра Azure может обеспечить управление безопасностью для двух типов сетевой архитектуры:

- **защищенный виртуальный концентратор**

   [Виртуальный WAN Hub Azure](../virtual-wan/virtual-wan-about.md#resources) — это управляемый корпорацией Майкрософт ресурс, который позволяет легко создавать концентраторы и архитектуры. Когда политики безопасности и направления связаны с таким концентратором, он называется *[защищенным виртуальным концентратором.](secured-virtual-hub.md)* 
- **концентратор виртуальной сети**

   Это стандартная виртуальная сеть Azure, которую вы создаете и управляете собой. Когда политики безопасности связаны с таким концентратором, он называется *виртуальной сетью концентратора.* В настоящее время поддерживается только политика Брандмауэра Azure. Можно срабатывая виртуальные сети, содержащие серверы и службы рабочей нагрузки. Вы также можете управлять брандмауэрами в автономных виртуальных сетях, которые не заглядываются ни в одну из спиченок.

## <a name="comparison"></a>Сравнение

В следующей таблице сравниваются эти два варианта архитектуры и можно решить, какой из них подходит для требований безопасности организации:


|  |**виртуальная сеть концентратора;**|**Защищенный виртуальный концентратор**  |
|---------|---------|---------|
|**Базовый ресурс**     |Виртуальная сеть|Виртуальный wan Концентратор|
|**Пресс-& концентратор**     |Использует виртуальную сеть пиринга|Автоматизированное использование виртуального сетевого соединения концентратора|
|**Подключение на прем**     |VPN Gateway до 10 Гбит/с и 30 S2S-соединений; Expressroute|Более масштабируемый VPN шлюз до 20 Гбит/с и 1000 S2S-соединений; Экспресс-маршрут|
|**Автоматизированное подключение ветки с помощью SDWAN**      |Не поддерживается|Поддерживается|
|**Концентраторы в регионе**     |Несколько виртуальных сетей в регионе|Единый виртуальный концентратор в регионе. Несколько концентраторов возможно с несколькими виртуальными WANs|
|**Azure Firewall — несколько общедоступных IP-адресов**      |Предоставляемый пользователем|Автогенерируется. Чтобы быть доступны GA.|
|**Зоны доступности брандмауэра Azure**     |Поддерживается|Недоступен в предварительном просмотре. Чтобы быть доступны GA|
|**Расширенная безопасность Интернета с сторонними партнерами по безопасности**     |Клиент установил и управлял VPN-соединением с партнерским сервисом по выбору|Автоматизированный через Trusted Security Partner потока и опыт управления партнерами|
|**Централизованное управление маршрутами для направления трафика в концентратор**     |Управляемый клиентом пользовательский маршрут|Поддерживается с помощью BGP|
|**Брандмауэр веб-приложения в Шлюзе приложений** |Поддерживается в виртуальной сети|В настоящее время поддерживается в сектора сети|
|**Виртуальные сетевые модули**.|Поддерживается в виртуальной сети|В настоящее время поддерживается в сектора сети|

# <a name="virtual-network-service-tags"></a>Виртуальные сетевые теги обслуживания
<a name="network-service-tags"></a>

Тег службы представляет собой группу префиксов IP-адресов из данной службы Azure. Корпорация Майкрософт управляет адресными префиксами, охватываемыми тегом службы, и автоматически обновляет тег службы по мере изменения адресов, снижая сложность частых обновлений правил сетевой безопасности.

Теги служб можно использовать для определения элементов управления доступом к сети в  [группах сетевой безопасности](https://docs.microsoft.com/azure/virtual-network/security-overview#security-rules)или [Azure Firewall.](https://docs.microsoft.com/azure/firewall/service-tags) При создании правил безопасности используйте теги служб вместо конкретных IP-адресов. Указывая имя тега службы (например, **ApiManagement)** в соответствующем *поле источника* или *назначения* правила, можно разрешить или отклонить трафик для соответствующей службы.

Вы можете использовать теги служб для достижения сетевой изоляции и защиты ресурсов Azure от общего Интернета при доступе к службам Azure, которые имеют общедоступные конечные точки. Создавайте правила входящих/исходящих групп сетевой безопасности, чтобы отказать в трафике в/из **Интернета** и разрешить трафик в/из **AzureCloud** или другие [доступные теги служб](#available-service-tags) ы определенных служб Azure.

В следующую таблицу включены все теги служб, доступные для использования в правилах [группы сетевой безопасности.](https://docs.microsoft.com/azure/virtual-network/security-overview#security-rules)

Столбцы указывают, является ли тег:

- Подходит для правил, охватывающих входящий или исходящий трафик.
- Поддерживает [региональную](https://azure.microsoft.com/regions) область.
- Используется в правилах [Azure Firewall.](https://docs.microsoft.com/azure/firewall/service-tags)

## <a name="service-tags-on-premises"></a>Сервисные теги на месте  
Вы можете получить текущий тег обслуживания и информацию о диапазоне, чтобы включить в состав ваших конфигураций брандмауэра. Эта информация представляет собой текущий список диапазонов IP-адресов, соответствующих каждому тегу обслуживания. Вы можете получить информацию программно или через загрузку файла JSON, как описано в следующих разделах.

### <a name="use-the-service-tag-discovery-api-public-preview"></a>Используйте API обнаружения тегов службы (публичный предварительный просмотр)
Вы можете программно получить текущий список тегов услуг вместе с данными диапазона IP-адресов:

- [REST](https://docs.microsoft.com/rest/api/virtualnetwork/servicetags/list)
- [Azure PowerShell](https://docs.microsoft.com/powershell/module/az.network/Get-AzNetworkServiceTag?view=azps-2.8.0&viewFallbackFrom=azps-2.3.2)
- [Azure CLI](https://docs.microsoft.com/cli/azure/network?view=azure-cli-latest#az-network-list-service-tags)

> [!NOTE]
> В то время как он в общедоступном предварительном просмотре, API обнаружения может вернуть информацию, которая менее актуна, чем информация, возвращенная загрузками JSON. (См. следующий раздел).


### <a name="discover-service-tags-by-using-downloadable-json-files"></a>Откройте для себя теги служб, используя загружаемые файлы JSON 
Вы можете скачать файлы JSON, которые содержат текущий список сервисных тегов вместе с деталями диапазона IP-адресов. Эти списки обновляются и публикуются еженедельно. Места для каждого облака:

- [Azure Public](https://www.microsoft.com/download/details.aspx?id=56519)
- [Лазурное правительство США](https://www.microsoft.com/download/details.aspx?id=57063)  
- [Лазурный Китай](https://www.microsoft.com/download/details.aspx?id=57062) 
- [Azure для Германии](https://www.microsoft.com/download/details.aspx?id=57064)   

> [!NOTE]
>Подмножество этой информации опубликовано в файлах XML для [Azure Public,](https://www.microsoft.com/download/details.aspx?id=41653) [Azure China](https://www.microsoft.com/download/details.aspx?id=42064)и [Azure Germany.](https://www.microsoft.com/download/details.aspx?id=54770) Эти XML загрузки будут уволены до 30 июня 2020 года и больше не будут доступны после этой даты. Вы должны перейти на использование API-извне и файлов JSON, описанных в предыдущих разделах.

# <a name="what-is-ip-address-1686312916"></a>Что такое IP-адрес 168.63.129.16?

IP-адрес 168.63.129.16 является виртуальным общедоступным IP-адресом, который используется для содействия коммуникационному каналу, установленному с ресурсами платформы Azure. Клиенты могут определять любое адресное пространство для своей частной виртуальной сети в Azure. Таким образом, ресурсы платформы Azure должны быть представлены как уникальный общедоступный IP-адрес. Этот виртуальный общедоступный IP-адрес разрешает:

- агенту виртуальной машины взаимодействовать с платформой Azure, чтобы сигнализировать, что он "Готов";
- связь с виртуальным сервером DNS для предоставления разрешения отфильтрованных имен для ресурсов (например, для виртуальной машины), которые не имеют настраиваемого DNS-сервера. Такая фильтрация гарантирует, что клиенты могут разрешать только имена узлов своих ресурсов;
- Позволяет [зондам работоспособности с баланса нагрузки Azure](../load-balancer/load-balancer-custom-probe-overview.md) определять состояние работоспособности ВМ.
- Позволяет виртуальной машине получить динамический IP-адрес от службы DHCP в Azure.
- сообщения пульса гостевого агента для роли PaaS.

## <a name="scope-of-ip-address-1686312916"></a>Область действия IP-адреса 168.63.129.16

Общественный IP-адрес 168.63.129.16 используется во всех регионах и во всех национальных облаках. Этот специальный общедоступный IP-адрес принадлежит корпорации Майкрософт и не изменится. Мы рекомендуем разрешить этот IP-адрес в любых локальных (в VM) политиках брандмауэра (направление исходящего). Обмен данными между этим специальным IP-адресом и ресурсами безопасный, так как только внутренняя платформа Azure может быть источником сообщения с этого IP-адреса. Если этот адрес будет заблокирован, может возникнуть непредвиденная реакция в различных сценариях. 168.63.129.16 представляет собой [виртуальный IP-адрес узла хоста](../virtual-network/security-overview.md#azure-platform-considerations) и как таковой не подчиняется определенным маршрутам пользователя.

- Агент VM требует исходящих сообщений через порты 80, 443, 32526 с WireServer (168.63.129.16). Они должны быть открыты в локальном брандмауэре на VM. Связь на этих портах с 168.63.129.16 не подпадает под настроенные группы сетевой безопасности.
- 168.63.129.16 может предоставить Услуги DNS ВМ. Если это нежелательно, этот трафик может быть заблокирован в локальном брандмауэре на VM. По умолчанию dNS-связь не подпадает под действие настроенных групп безопасности сети, если только специально ориентирована на использование тега службы [AzurePlatformDNS.](../virtual-network/service-tags-overview.md#available-service-tags)
- Когда VM является частью пула балансов нагрузки, связь [зонда работы](../load-balancer/load-balancer-custom-probe-overview.md) должна быть позволена исходить от 168.63.129.16. Конфигурация группы сетевой безопасности по умолчанию имеет правило, которое позволяет это сообщение. Это правило использует тег службы [AzureLoadBalancer.](../virtual-network/service-tags-overview.md#available-service-tags) При желании этот трафик может быть заблокирован путем настройки группы сетевой безопасности, однако это приведет к тому, что зонды сбой.

В невиртуальном сетевом сценарии (Classic) зонд работоспособности получен из частного IP и 168.63.129.16 не используется.

# <a name="reserved-public-ipv6-address-prefix-preview"></a>Зарезервированная публичная приставка адреса IPv6 (Предварительный просмотр)

В Azure виртуальные сети с двойным стеком (IPv4-IPv6) и виртуальные машины (VM) защищены по умолчанию, так как у них нет подключения к Интернету. Вы можете легко добавить подключение К Интернету IPv6 к балансирурам и vMs-адресам Azure с общедоступными адресами IPv6, которые вы получаете от Azure.

Любые общедоступные ИП, которые вы резервируете, связаны с по вашему выбору регионом Azure и с подпиской Azure. Вы можете переместить зарезервированный (статический) IP-адрес IPv6 между любым из балансов azure Load или VM в вашей подписке. Вы можете полностью разъединить IP-адрес IPv6, и он будет проводиться для вашего использования, когда вы будете готовы.

> [!WARNING]
> Используйте осторожность, чтобы не удалять общедоступные IP-адреса случайно. Удаление публичного IP удаляет его из вашей подписки, и вы не сможете восстановить его (даже с помощью поддержки Azure).

В дополнение к резервированию отдельных адресов IPv6 можно зарезервировать смежные диапазоны адресов Azure IPv6 (известных как IP-префикс) для использования.  Как и в отношении отдельных IP-адресов, зарезервированные префиксы связаны с по вашему выбору регионом Azure и с подпиской Azure. Резервирование предсказуемого, смежного диапазона адресов имеет много применений. Например, вы можете значительно упростить *IP-белый список* приложений, размещенных в Azure, вашей компанией и клиентами, поскольку ваши статические диапазоны IP могут быть легко запрограммированы на брандмауэры.  Вы можете создавать отдельные общедоступные IP-адреса из приставки IP по мере необходимости, и при удалении этих отдельных публичных IP-адресов они *возвращаются* в зарезервированный диапазон, чтобы вы могли повторно использовать их позже. Все IP-адреса в ip-версии зарезервированы для эксклюзивного использования до тех пор, пока вы не удалите префикс.

> [!Important]
> IPv6 для виртуальной сети Azure в настоящее время находится в открытом доступе. Предварительная версия предоставляется без соглашения об уровне обслуживания. Не рекомендуем использовать ее в рабочей среде. Некоторые функции могут не поддерживаться или их возможности могут быть ограничены. См. [дополнительные условия использования для предварительных версий Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

## <a name="ipv6-prefix-sizes"></a>Размеры префикса IPv6
Доступны следующие общедоступные размеры префикса IP:

-  Минимальный размер префикса IPv6: /127 - 2 адреса
-  Максимальный размер префикса IPv6: /124 - 16 адресов

Размер префикса указан как размер маски Classless Inter-Domain Routing (CIDR). Например, маска /128 представляет собой индивидуальный адрес IPv6, поскольку адреса IPv6 состоят из 128 битов.

## <a name="limitations"></a>Ограничения
IPv6 поддерживается на базовых публичных i-адресах только при «динамическом» распределении, что означает, что адрес IPv6 изменится, если вы удалите и передислоцируете ваше приложение (балансеры VM или нагрузки) в Azure. Поддержка Standard IPv6 Public IP исключительно статичная (зарезервированная) распределение, хотя балансеры нагрузки Standard INTERNAL также могут поддерживать динамическое распределение из подсети, которой они назначены.  

В качестве наилучшей практики мы рекомендуем использовать стандартные общественные I-iPs и балансеры стандартной нагрузки для приложений IPv6.

# <a name="create-change-or-delete-service-endpoint-policy-using-the-azure-portal"></a>Создание, изменение или удаление политик конечной точки службы с помощью портала Azure

Политики конечной точки службы позволяют фильтровать трафик виртуальной сети, передаваемый к ресурсам Azure, с помощью конечных точек службы. Если вы не работали с политиками конечной точки службы, ознакомьтесь с [обзором политик конечной точки службы](virtual-network-service-endpoint-policies-overview.md), чтобы получить дополнительные сведения.

 Из этого руководства вы узнаете, как выполнять следующие задачи:

> [!div class="checklist"]
> * Создание политики конечной точки службы.
> * Создание определения политики конечной точки службы.
> * Создание виртуальной сети с подсетью.
> * Связывание политики конечной точки службы с подсетью.

Если у вас еще нет подписки Azure, [создайте бесплатную учетную запись Azure](https://azure.microsoft.com/free/?WT.mc_id=A261C142F), прежде чем начинать работу.

## <a name="sign-in-to-azure"></a>Вход в Azure 

Войдите на портал Azure по адресу https://portal.azure.com.

## <a name="create-a-service-endpoint-policy"></a>Создание политики конечной точки службы.

1. Выберите **+ Создать ресурс** в верхнем левом углу окна портала Azure.
2. В области поиска введите "service endpoint policy" и выберите **Service endpoint policy (Preview)** (Политика конечной точки службы (предварительная версия)), затем щелкните **Создать**.
3. В колонке **Основные данные** введите или выберите следующие сведения. 

   - "Подписка": выберите подписку для политики.    
   - "Группа ресурсов": Выберите **Создать новую**, а затем введите *myResourceGroup*.     
   - "Имя": myEndpointPolicy.
   - "Расположение": Западно-центральная часть США     
 
   ![Создание политики конечной точки службы в колонке "Основные данные"](./media/virtual-network-service-endpoint-policies-portal/virtual-network-endpoint-policies-create-startpane.PNG)
   
4. В колонке **Определения политики** введите или выберите следующие сведения.

   - Щелкните **+ Add a resource** (+Добавить ресурс) и введите или выберите приведенные ниже сведения, оставьте значения по умолчанию для остальных параметров и щелкните **Создать**.  
   - "Область": выберите **Одна учетная запись**, **Все учетные записи в подписке** или **Все учетные записи в группе ресурсов**.    
   - "Подписка": выберите подписку для учетной записи хранения. Политика и учетные записи хранения могут находиться в разных подписках.   
   - "Группа ресурсов": Выберите нужную группу ресурсов. Это обязательный параметр, если выбрана область "Одна учетная запись" или "All accounts in resource group" (Все учетные записи в группе ресурсов).  
   - "Ресурс": mystorageaccountportal.    
   - Щелкните **+ Add a resource** (+Добавить ресурс), чтобы добавить дополнительные ресурсы.
   
   ![Создание определений политики конечной точки службы](./media/virtual-network-service-endpoint-policies-portal/virtual-network-endpoint-policies-create-policydefinitionspane.PNG)
   
5. Необязательно: На вкладке **Теги** введите или выберите следующие сведения.
   
   - "Ключ": выберите ключ для политики. Например, Dept.     
   - "Значение": введите значение для ключа. Например, Finance.

6. Выберите **Review + Create** (Просмотреть и создать). Проверьте сведения и щелкните **Создать**. Чтобы внести дополнительные изменения, щелкните **Назад**. 

   ![Окончательные проверки при создании политики конечной точки службы](./media/virtual-network-service-endpoint-policies-portal/virtual-network-endpoint-policies-create-finalcreatereview.PNG)
  
 
## <a name="view-endpoint-policies"></a>Просмотр политик конечной точки службы 

1. В поле *Все службы* на портале начните вводить текст *service endpoint policies*. Выберите **Service Endpoint Policies (Preview)**.
2. В разделе **Подписки** выберите подписку и группу ресурсов, как показано на следующем рисунке.

   ![Отображение политики](./media/virtual-network-service-endpoint-policies-portal/virtual-network-endpoint-policies-viewpolicies.PNG)
       
3. Выберите политику и щелкните **Определения политики**, чтобы просмотреть или добавить дополнительные определения политики.

   ![Отображение определений политик](./media/virtual-network-service-endpoint-policies-portal/virtual-network-endpoint-policies-viewpolicy-adddefinitions.PNG)

4. Выберите **Связанные подсети**, чтобы просмотреть подсети, с которыми связана политика. Чтобы связать политику с подсетью, щелкните "Navigate to virtual network in the same region" (Перейти к виртуальной сети в том же регионе).

   ![Отображение связанных подсетей](./media/virtual-network-service-endpoint-policies-portal/virtual-network-endpoint-policies-view-associatedsubnets.PNG)
 
## <a name="associate-a-policy-to-a-subnet"></a>Связывание политики с подсетью

>[!WARNING] 
> Прежде чем связывать политику, убедитесь, что все ресурсы для выбранной службы, доступ к которым осуществляется из подсети, добавлены в эту политику. После связывания политики для регионов конечной точки службы будет разрешен доступ только к ресурсам, указанным в этой политике. 

Чтобы связать политику с подсетью, необходимо создать виртуальную сеть и подсеть, а затем связать политику с этой подсетью.

1. Выберите **+ Создать ресурс** в верхнем левом углу окна портала Azure.
2. Щелкните **Сеть** и выберите **Виртуальная сеть**.
3. В разделе **Создание виртуальной сети** введите или выберите следующие сведения, примите значения по умолчанию для остальных параметров и щелкните **Создать**:
   - "Имя": myVirtualNetwork.      
   - "Адресное пространство": 10.0.0.0/16      
   - "Подписка": Выберите свою подписку. Политика должна находиться в той же подписке, что и виртуальная сеть.     
   - "Группа ресурсов": установите флажок **Использовать существующий**, а затем выберите *myResourceGroup*.     
   - "Расположение": Западно-центральная часть США     
   - "Имя": "Private".     
   - "Диапазон адресов": 10.0.0.0/24
     
4. В поле **Search resources, services, and docs** (Поиск ресурсов, служб и документов) в верхней части портала введите *myVirtualNetwork*. Когда в результатах поиска появится пункт **myVirtualNetwork**, выберите его.
5. В разделе **Параметры** выберите **Подсети**, а затем — **Частный**.
6. Как показано на следующем рисунке, выберите **Конечные точки службы** > **Microsoft.Storage** > **Политики конечной точки службы** >  **myEndpointPolicy**, а затем щелкните **Сохранить**.

   ![Связывание политики](./media/virtual-network-service-endpoint-policies-portal/virtual-network-endpoint-policies-associatepolicies.PNG)

>[!WARNING] 
>Доступ из этой подсети к ресурсам службы в других регионах будет разрешен на основе групп безопасности сети (NSG). Чтобы ограничить доступ только регионами конечной точки, настройте группы безопасности сети для обслуживания только трафика в регионах конечной точки. Дополнительные сведения о том, как создавать группы безопасности сети с тегами служб по регионам, см. в разделе [Создание, изменение и удаление группы безопасности сети](manage-network-security-group.md?toc=%2fcreate-a-security-rule%2f.json).

В следующем примере группа безопасности сети настроена для предоставления доступа только к ресурсам службы хранилища Azure, размещенным в регионах WestCentralUS и WestUS2, с правилом "Запретить все", имеющим более низкий приоритет.

![Группа безопасности сети, запрещающая весь трафик](./media/virtual-network-service-endpoint-policies-portal/virtual-network-endpoint-policies-nsg-rules.PNG)

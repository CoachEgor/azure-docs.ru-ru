# <a name="what-is-ipv6-for-azure-virtual-network-preview"></a>Что такое IPv6 для виртуальной сети Azure? (предварительная версия)

IPv6 для Azure виртуальная сеть (VNET) позволяет размещать приложения в Azure с подключением к IPv6 и IPv4 в виртуальной сети и к и из Интернета. Из-за нехватки общедоступных адресов IPv4 новые сети для мобильных устройств и Интернет вещей (IoT) часто строятся на IPv6. Даже долго устанавливаются на IPv6 преобразуются поставщика услуг Интернета и мобильных сетей. Службы, поддерживающие только IPv4 можно найти сами на реальных недостаток существующих и развивающихся рынков. Двойной стек IPv4/IPv6-подключения позволяет служб, размещенных в Azure, для обхода пропуска технологии с помощью глобально доступной, двойным стеком служб, которые легко подключаются с существующей IPv4 и этих новых устройств IPv6 и сетей.

Исходный IPv6-подключения Azure позволяет легко обеспечить двойной стек подключения к Интернету (IPv4/IPv6) для приложений, размещенных в Azure. Он позволяет простое развертывание виртуальных машин с подключением по IPv6 с балансировкой нагрузки для входящего и исходящего инициируемых подключений. Эта функция по-прежнему доступна и Дополнительные сведения можно найти [здесь](../load-balancer/load-balancer-ipv6-overview.md).
IPv6 для виртуальной сети Azure является гораздо более полной избранные-Включение полной архитектуры решений IPv6 для развертывания в Azure.

> [!Important]
> IPv6 для виртуальной сети Azure в настоящее время находится в общедоступной предварительной версии. Предварительная версия предоставляется без соглашения об уровне обслуживания. Не рекомендуем использовать ее в рабочей среде. Некоторые функции могут не поддерживаться или их возможности могут быть ограничены. См. [дополнительные условия использования для предварительных версий Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

## <a name="benefits"></a>Преимущества

Преимущества Azure виртуальной сети IPv6:

- Помогает расширение области действия, размещенной в Azure приложений в возрастающую мобильных устройств и рынков Интернета вещей.
- Двух виртуальных машин с накоплением IPv4/IPv6 обеспечивают гибкость развертывания максимальное количество служб. Одного экземпляра службы можно подключиться с IPv4 и IPv6 с поддержкой Интернет-клиентами.
- Основан на давно сформулированных и стабильных Azure виртуальной Машины к Интернету IPv6-подключения.
- Обеспечение безопасности по умолчанию, так как IPv6-подключения к Интернету устанавливается только в том случае, когда вы явно запрашивает ее в развертывании.

## <a name="capabilities"></a>Возможности

Поддержка IPv6 для виртуальных машин включает в себя следующие возможности:

- Клиенты Azure можно определить свои собственные IPv6 виртуальной сети адресные пространства для удовлетворения потребностей своих приложений, клиентов, или легко интегрируйте их локальное пространство IP-адрес.
- Двойной стек (IPv4 и IPv6) виртуальные сети с подсетями двойной стек позволяют приложениям подключаться к ресурсам IPv4 и IPv6 в своей виртуальной сети или - Интернет.
- Защита ресурсов с правилами IPv6 для групп безопасности сети
- Настройте маршрутизацию трафика IPv6 в виртуальной сети с помощью определяемых пользователем маршрутов — особенно в том случае, при использовании виртуальных сетевых устройств для расширения вашего приложения.
- Поддержка подсистемы балансировки нагрузки IPv6 для создания надежных масштабируемых приложений, Azure DNS поддерживает записи AAAA для общедоступных IP-адресов IPv6.
- Легко добавьте IPv6-подключения для существующих развертываний только для IPv4, с помощью обновления на месте.
- Создавайте приложения двойной стек, автоматическое масштабирование в соответствии с нагрузкой с масштабируемыми наборами виртуальных машин.

## <a name="limitations"></a>Ограничения
Предварительная версия IPv6 для виртуальной сети Azure имеет следующие ограничения:
- Протокол IPv6 для виртуальной сети Azure (Предварительная версия) доступен во всех глобальных регионах Azure, но только в глобальной Azure - не облака для государственных организаций.   
- Поддержка портала для предварительного просмотра ограничено для просмотра только для многое, но не для всех IPv6-конфигурации, однако IPv6 для виртуальной сети имеет полную поддержку и документации (с примерами) для развертывания IPv6, с помощью Azure Powershell и интерфейса командной строки (CLI).
- Сети наблюдателя для предварительной версии поддерживается только для журналов потоков NSG и снимки сетевых пакетов.
- Поддержка балансировки нагрузки для предварительной версии изначально ограничено основные подсистемы балансировки нагрузки.
- Общедоступные IP-адреса на уровне экземпляра (общедоступные IP-адреса непосредственно на виртуальной Машине) не поддерживается в предварительной версии.  
- Пиринг виртуальных сетей (регионально и глобально) не поддерживается в предварительной версии.

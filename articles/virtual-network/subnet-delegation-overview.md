# <a name="what-is-subnet-delegation"></a>Что такое делегация подсети?

Делегация Subnet позволяет назначить определенную подсеть для службы Azure PaaS по вашему выбору, которая должна быть введена в виртуальную сеть. Делегация Subnet обеспечивает полный контроль перед клиентом по управлению интеграцией служб Azure в свои виртуальные сети.

Когда вы делегируете подсеть службе Azure, вы позволяете этой службе установить некоторые основные правила конфигурации сети для этой подсети, которые помогают службе Azure стабильно управлять их экземплярами. В результате служба Azure может установить некоторые условия развертывания до или после развертывания, такие как:
- развернуть службу в общей и специальной подсети.
- добавьте в службу набор пост-развертывания сетевых политик, который необходим для правильной работы службы.

##  <a name="advantages-of-subnet-delegation"></a>Преимущества делегации подсети

Делегирование подсети на конкретные услуги дает следующие преимущества:

- помогает обозначать подсеть для одного или нескольких служб Azure и управлять экземплярами в подсети в соответствии с требованиями. Например, владелец виртуальной сети может определить для делегированной подсети следующее, чтобы лучше управлять ресурсами и получить доступ следующим образом:
    - сетевой фильтрации политик трафика с группами сетевой безопасности.
    - политики маршрутизаний с определенными маршрутами.
    - интеграция услуг с конфигурациями конечных точек обслуживания.
- помогает впрыскиваемым службам лучше интегрироваться с виртуальной сетью, определяя их предварительные условия развертывания в виде политик и средств сетевых намерений. Это гарантирует, что любые действия, которые могут повлиять на функционирование инъекционной службы, могут быть заблокированы в PUT.


## <a name="who-can-delegate"></a>Кто может делегировать?
Делегация Subnet — это упражнение, которое владельцы виртуальных сетей должны выполнить для обозначения одной из подсетей для конкретной службы Azure. Служба Azure, в свою очередь, развертывает экземпляры в эту подсеть для потребления рабочими нагрузками клиента.

## <a name="impact-of-subnet-delegation-on-your-subnet"></a>Влияние делегирования подсети на вашу подсеть
Каждая служба Azure определяет свою собственную модель развертывания, где они могут определить, какие свойства они делают или не поддерживают в делегированной подсети для целей впрыска, например:
- общая субсеть с другими службами Azure или VM / виртуальной шкалой машины, установленной в той же подсети, или она поддерживает только специальную подсеть только с экземплярами этой службы в ней.
- поддерживает ассоциацию NSG с делегированной подсетью.
- поддержка NSG, связанная с делегированной подсетью, также может быть связана с любой другой подсетью.
- позволяет ассоциацию таблицы маршрутов с делегированной подсетью.
- позволяет ассоциировать таблицу маршрутов, связанную с делегированной подсетью, с любой другой подсетью.
- диктует минимальное количество IP-адресов в делегированной подсети.
- диктует пространство IP-адреса в делегированной подсети из пространства Private IP-адреса (10.0.0.0/8, 192.168.0.0/16, 172.16.0.0/12).
- диктует, что пользовательская конфигурация DNS имеет запись Azure DNS.

Инъекционные службы также могут добавлять свои собственные политики следующим образом:
- **Политики безопасности**: Сбор правил безопасности, необходимых для работы данной службы.
- **Политики маршрута**: Сбор маршрутов, необходимых для работы данной службы.

## <a name="what-subnet-delegation-does-not-do"></a>Что не делает делегация подсети

Службы Azure, вводимые в делегированную подсеть, по-прежнему имеют базовый набор свойств, доступных для неделенных подсетей, таких как:
-  Службы Azure могут вводить экземпляры в подсети клиентов, но не могут повлиять на существующие рабочие нагрузки.
-  Политики или маршруты, применяемые этими службами, являются гибкими и могут быть отменены клиентом.

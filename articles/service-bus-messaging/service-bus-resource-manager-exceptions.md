# <a name="service-bus-resource-manager-exceptions"></a>Исключения менеджера ресурсов сервисного автобуса

В этой статье перечислены исключения, создаваемые при взаимодействии с Azure Service Bus с помощью диспетчера ресурсов Azure - с помощью шаблонов или прямых вызовов.

> [!IMPORTANT]
> Этот документ часто обновляется. Пожалуйста, проверьте обновления.

Ниже приведены различные исключения/ошибки, которые всплывают через менеджер ресурсов Azure.

## <a name="error-bad-request"></a>Ошибка: Плохой запрос

"Плохой запрос" подразумевает, что запрос, полученный менеджером ресурсов, не прошел проверку.

| Код ошибки | Ошибка подкод | Сообщение об ошибке | Описание | Рекомендация |
| ---------- | ------------- | ------------- | ----------- | -------------- |
| Ошибка запроса | 40 000 | Подкод-код 40000. *"Имя свойства"* не может быть установлено при создании Очереди, поскольку в пространстве имен *используется* "Основной" уровень. Эта операция поддерживается только в уровне "Стандарт" или "Премиум". | На базовом уровне обслуживания службы Azure не могут быть установлены или обновлены ниже свойства - <ul> <li> RequiresDuplicateDetection </li> <li> AutoDeleteOnIdle </li> <li>RequiresSession</li> <li>DefaultMessageTimeToLive </li> <li> DuplicateDetectionHistoryTimeWindow </li> <li> EnableExpress </li> <li> Отправить </li> <li> Разделы </li> </ul> | Рассмотрите возможность перехода с базового на стандартный или премиум-уровня, чтобы использовать эту функциональность. |
| Ошибка запроса | 40 000 | Подкод-код 40000. Значение свойства «необходимоеобнаружение» существующей очереди (или темы) не может быть изменено. | Двойное обнаружение должно быть включено/отключено во время создания сущности. Параметр конфигурации дублирования обнаружения не может быть изменен после создания. | Для включения дублирования обнаружения в ранее созданной очереди/теме можно создать новую очередь/тему с дубликатом обнаружения, а затем перейти из исходной очереди в новую очередь/тему. |
| Ошибка запроса | 40 000 | Подкод-код 40000. Указанное значение 16384 является недействительным. Свойство 'MaxSizeInMegabytes', должно быть одним из следующих значений: 1024;2048;3072;4096;5120. | Значение MaxSizeInMegabytes является недействительным. | Убедитесь, что MaxSizeInMegabytes является одним из следующих - 1024, 2048, 3072, 4096, 5120. |
| Ошибка запроса | 40 000 | Подкод-код 40000. Раздел не может быть изменен для очереди/темы. | Раздел не может быть изменен для сущности. | Создайте новую сущность (очередь или тему) и включите разделы. | 
| Ошибка запроса | none | Пространство имен *«имя»* не существует. | Пространство имен не существует в подписке Azure. | Чтобы устранить эту ошибку, пожалуйста, попробуйте ниже <ul> <li> Убедитесь, что подписка Azure верна. </li> <li> Убедитесь, что пространство имен существует. </li> <li> Проверить имя пространства имен является правильным (без орфографических ошибок или нулевых строк). </li> </ul> | 
| Ошибка запроса | 40400 | Подкод-код No40400. Функция назначения автоматической пересылки не существует. | Назначение объекта назначения автофорвардинга не существует. | Сущность назначения (очередь или тема) должна существовать до создания источника. Повторная попытка после создания объекта назначения. |
| Ошибка запроса | 40 000 | Подкод-код 40000. Поставленное время блокировки превышает допустимый максимум '5' минут. | Время, в течение которого сообщение может быть заблокировано, должно соться от 1 минуты (минимум) и 5 минут (максимум). | Убедитесь, что время блокировки составляет от 1 мин до 5 минут. |

| Ошибка запроса | 40 000 | Подкод-код 40000. 'URI_PATH» содержит символ (ы), который не допускается Сервисным автобусом. Любые другие символы приводят к сбою запроса. | Убедитесь, что в Пути URI нет недействительных символов. |


## <a name="error-code-429"></a>Код ошибки: 429

Так же, как в HTTP, "Ошибка код 429" указывает на "слишком много запросов". Это означает, что конкретный ресурс (пространство имен) заглушен из-за слишком большого количества запросов (или из-за противоречивых операций) на этом ресурсе.

| Код ошибки | Ошибка подкод | Сообщение об ошибке | Описание | Рекомендация |
| ---------- | ------------- | ------------- | ----------- | -------------- |
| 429 | 50004 | Подкод-код No50004. Запрос был прекращен из-за того, что пространство имен задушило пространство *имен.* | Это условие ошибки попадает, когда количество входящих запросов превышает ограничение ресурса. | Подождите несколько секунд и попробуйте еще раз. <br/> <br/> Подробнее о [квотах](service-bus-quotas.md) и [лимитах запросов на ресурсы Azure](../azure-resource-manager/management/request-limits-and-throttling.md)|
| 429 | 40901 | Подкод-код No40901. В настоящее время проводится еще одна конфликтующая операция. | Еще одна конфликтующая операция выполняется на том же ресурсе/сущности | Подождите, пока текущая операция не завершена, прежде чем пытаться снова. |
| 429 | 40900 | Подкод-код No40900. Конфликт. Запрашиваемая операция не разрешена в текущем состоянии ресурса. | Это условие может быть поражено, когда делается несколько запросов для выполнения операций на одной и той же сущности (очередь, тема, подписка или правило) одновременно. | Подождите несколько секунд и попробуйте еще раз |
| 429 | 40901 | Запрос на *"имя сущности"* противоречил другому запросу | Еще одна конфликтующая операция выполняется на том же ресурсе/сущности | Подождите, пока предыдущая операция будет завершена, прежде чем пытаться снова |
| 429 | 40901 | Еще один запрос на обновление выполняется для *"имя сущности"* сущности. | Еще одна конфликтующая операция выполняется на том же ресурсе/сущности | Подождите, пока предыдущая операция будет завершена, прежде чем пытаться снова |
| 429 | none | Произошел ресурсный конфликт. Еще одна противоречивая операция может быть в процессе. Если это повторная попытка для неудачной операции, очистка фонового фона все еще не завершена. Повторите попытку позже. | Это условие может быть поражено, когда есть отложенная операция против одного и того же объекта. | Подождите, пока предыдущая операция будет завершена, прежде чем пытаться снова. |


## <a name="error-code-not-found"></a>Код ошибки: не найдено

Этот класс ошибок указывает на то, что ресурс не был найден.

| Код ошибки | Ошибка подкод | Сообщение об ошибке | Описание | Рекомендация |
| ---------- | ------------- | ------------- | ----------- | -------------- |
| Не найдено | none | *Сущность 'имя сущности'* не была найдена. | Сущность, в отношении которой операция не была найдена. | Проверьте, существует ли сущность, и повторите попытку операции. |
| Не найдено | none | Не найден. Операции не существует. | Операция, выполняемые по пыталась, не существует. | Проверьте операцию и попробуйте еще раз. |
| Не найдено | none | Входящий запрос не признается в качестве запроса на изменение политики пространства имен. | Входящий орган запроса является недействительным и, следовательно, не может быть выполнен в качестве пут-запроса. | Пожалуйста, проверьте орган запроса, чтобы убедиться, что он не является недействительным. | 
| Не найдено | none | Не удалось найти *"имя сущности"* мессенджера. | Сущность, против которую вы пытаетесь выполнить операцию, не найдена. | Пожалуйста, проверьте, существует ли сущность, и повторите попытку операции. |

| ---------- | ------------- | ------------- | ----------- | -------------- |
| Внутренняя ошибка сервера | 50 000 | Подкод-код 50000. Внутренняя ошибка сервера| Может случиться по разным причинам. Некоторые из симптомов - <ul> <li> Запрос клиента/тело поврежден и приводит к ошибке. </li> <li> Запрос клиента приурочен из-за проблем с обработкой на службе. </li> </ul> | Для решения этой проблемы <ul> <li> Убедитесь, что параметры запросов не являются нулевыми или недополученными. </li> <li> Повторите запрос. </li> </ul> |

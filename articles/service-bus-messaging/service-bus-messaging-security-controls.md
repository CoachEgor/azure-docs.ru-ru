# <a name="security-controls-for-azure-service-bus-messaging"></a>Элементы управления безопасностью для сообщений автобусов службы Azure

В этой статье документируются элементы управления безопасностью, встроенные в сообщения об иноре службы Azure.

[!INCLUDE [Security controls Header](../../includes/security-controls-header.md)]

## <a name="network"></a>Сеть

| Контроль безопасности | Да/нет | Примечания | Документация |
|---|---|--|--|
| Поддержка конечных точек обслуживания| Да (только премиум-уровня) | Конечные точки обслуживания VNet поддерживаются только для [уровня Service Bus Premium.](service-bus-premium-messaging.md) |  |
| Поддержка инъекций VNet| нет | |  |
| Поддержка сетевой изоляции и брандмауэра| Да (только премиум-уровня) |  |  |
| Принудительная поддержка туннелирования| нет |  |  |

## <a name="monitoring--logging"></a>Мониторинг & лесозаготовки

| Контроль безопасности | Да/нет | Примечания| Документация |
|---|---|--|--|
| Поддержка мониторинга Azure (аналитика журналов, информация о приложениях и т.д.)| Да | Поддерживается с помощью [azure Monitor и оповещения](service-bus-metrics-azure-monitor.md). |  |
| Управление и управление самолетом лесозаготовки и аудит| Да | Доступны журналы операций.  | [Журналы диагностики служебной шины](service-bus-diagnostic-logs.md) |
| Регистрация и аудит плоскости данных| нет |  |

## <a name="identity"></a>идентификации

| Контроль безопасности | Да/нет | Примечания| Документация |
|---|---|--|--|
| Проверка подлинности| Да | Управляется через [Azure Active Directory Managed Service Identity](service-bus-managed-service-identity.md).| [Проверка подлинности и авторизации автобуса](service-bus-authentication-and-authorization.md)сервиса. |
| Авторизация| Да | Поддерживает авторизацию через [токены RBAC](authenticate-application.md) и SAS. | [Проверка подлинности и авторизации автобуса](service-bus-authentication-and-authorization.md)сервиса. |

## <a name="data-protection"></a>Защита данных

| Контроль безопасности | Да/нет | Примечания | Документация |
|---|---|--|--|
| Шифрование на стороне сервера в состоянии покоя: клавиши, управляемые Корпорацией Майкрософт |  Да для шифрования на стороне сервера по умолчанию. |  |  |
| Шифрование на стороне сервера в состоянии покоя: ключи, управляемые клиентом (BYOK) | Да. | Ключ, управляемый клиентом в Azure KeyVault, может использоваться для шифрования данных в пространстве имен службы безопасности в состоянии покоя. | [Настройка ключей, управляемых клиентом, для шифрования данных Azure Service Bus в состоянии покоя с помощью портала Azure](configure-customer-managed-key.md)  |
| Шифрование уровня столбцов (Службы данных Azure)| Недоступно | |   |
| Шифрование в пути (например, шифрование ExpressRoute, шифрование VNet и шифрование VNet-VNet)| Да | Поддерживает стандартный механизм HTTPS/TLS. |   |
| Вызовы API в зашифрованном виде| Да | Вызовы API производятся через [Azure Resource Manager](../azure-resource-manager/index.yml) и HTTPS. |   |

## <a name="configuration-management"></a>Управление конфигурацией

| Контроль безопасности | Да/нет | Примечания| Документация |
|---|---|--|--|
| Поддержка управления конфигурацией (версия конфигурации и т.д.)| Да | Поддерживает версию поставщика ресурсов через [API-менеджера ресурсов Azure.](/rest/api/resources/)|   |

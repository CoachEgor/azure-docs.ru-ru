# <a name="enable-diagnostic-logs-for-service-bus"></a>Включение журналов диагностики для служебной шины

Когда вы начинаете использовать пространство имен служебной шины Azure, вам может потребоваться отслеживать, как и когда пространство имен создается, удаляется или к которому осуществляется доступ. В этой статье приводятся общие сведения о доступных журналах оперативных и диагностических сведений.

В настоящее время служебная шина Azure поддерживает журналы активности и эксплуатации, которые фиксируют **операции управления** , выполняемые в пространстве имен служебной шины Azure. В частности, в эти журналы записывается тип операции, включая создание очереди, используемые ресурсы и состояние операции.

## <a name="operational-logs-schema"></a>Схема операционных журналов

Все журналы хранятся в формате нотация объектов JavaScript (JSON) в следующих двух расположениях.

- **AzureActivity** — отображает журналы операций и действий, выполненных в пространстве имен на портале, или с помощью развертывания шаблонов Azure Resource Manager.
- **AzureDiagnostics** — отображает журналы операций и действий, выполненных в пространстве имен с помощью API, или клиентов управления на языке SDK.

Строки JSON операционного журнала содержат элементы, перечисленные в приведенной ниже таблице.

| Name | Описание |
| ------- | ------- |
| ActivityId | Внутренний идентификатор, используемый для идентификации указанного действия |
| EventName | Имя операции |
| ResourceId | Идентификатор ресурса Azure Resource Manager |
| SubscriptionId | Идентификатор подписки |
| EventTimeString | Время операции |
| EventProperties | Свойства операции |
| Status | Состояние операции |
| Вызывающий объект | Объект, вызвавший операцию (портал Azure или клиент управления) |
| Категория | OperationalLogs |

## <a name="what-eventsoperations-are-captured-in-operational-logs"></a>Какие события и операции фиксируются в операционных журналах?

Журналы операций записывают все операции управления, выполненные в пространстве имен служебной шины Azure. Операции с данными не фиксируются из-за большого объема операций с данными, проводимых в служебной шине Azure.

> [!NOTE]
> Для более эффективной отслеживания операций с данными рекомендуется использовать трассировку на стороне клиента.

Следующие операции управления фиксируются в операционных журналах. 

| Область действия | Операция|
|-------| -------- |
| Пространство имен | <ul> <li> Создание пространства имен</li> <li> Обновить пространство имен </li> <li> Удалить пространство имен </li>  </ul> | 
| Очереди | <ul> <li> Создать очередь</li> <li> Очередь обновления</li> <li> Удаление очереди. </li> </ul> | 
| Раздел | <ul> <li> Создать раздел </li> <li> Обновление раздела </li> <li> Удалить раздел </li> </ul> |
| Subscription | <ul> <li> Создание подписки </li> <li> Обновить подписку </li> <li> Удалить подписку </li> </ul> |

> [!NOTE]
> В настоящее время операции **чтения** не регистрируются в операционных журналах.

## <a name="how-to-enable-operational-logs"></a>Как включить Операционные журналы?

Операционные журналы по умолчанию отключены. Чтобы включить ведение журналов диагностики, выполните следующее.

1. В [портал Azure](https://portal.azure.com)перейдите к пространству имен служебной шины Azure и в разделе **мониторинг**щелкните **параметры диагностики**.

   ![Перемещение к колонке журналов диагностики](./media/service-bus-diagnostic-logs/image1.png)

2. Щелкните **Добавить параметр диагностики** , чтобы настроить параметры диагностики.  

   ![Включение журналов диагностики](./media/service-bus-diagnostic-logs/image2.png)

3. Настройка параметров диагностики
   1. Введите **имя** для поиска параметров диагностики.
   2. Выберите назначение для диагностики.
      - При выборе **учетной записи хранения**необходимо настроить учетную запись хранения, в которой будет храниться диагностика.
      - При выборе **концентраторов событий**необходимо настроить соответствующий концентратор событий, в который будут передаваться параметры диагностики.
      - При выборе **log Analytics**необходимо указать, какой экземпляр log Analytics будет отправляться диагностика.
    3. Проверьте **OperationalLogs**.

       ![Изменение состояния журналов диагностики](./media/service-bus-diagnostic-logs/image3.png)

4. В нижней части страницы нажмите кнопку **Save**.


Новые параметры вступят в силу в течение 10 минут. После этого журналы появятся в настроенной цели для архивирования на вкладке **Журналы диагностики**.

Дополнительные сведения о настройке системы диагностики доступны в [обзоре журналов диагностики Azure](../azure-monitor/platform/diagnostic-logs-overview.md).

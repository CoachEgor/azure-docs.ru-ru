# <a name="overview-of-integration-between-application-gateway-and-azure-security-center"></a>Общие сведения об интеграции шлюза приложений с центром безопасности Azure

В этой статье приведены сведения о том, как интеграция шлюза приложений с центром безопасности Azure позволяет обеспечить защиту ресурсов веб-приложений. Брандмауэр веб-приложения шлюза приложений (WAF) интегрируется с [центром безопасности](../security-center/security-center-intro.md) , чтобы обеспечить простое представление для предотвращения, обнаружения и реагирования угроз на незащищенные веб-приложения в вашей среде.

## <a name="overview"></a>Обзор

Брандмауэр веб-приложения шлюза приложений — это рекомендуемый компонент центра безопасности, обеспечивающий защиту веб-приложений от распространенных эксплойтов и уязвимостей. Веб-ресурсы, не защищенные WAF, имеют в центре безопасности рекомендации высокого уровня серьезности. Рекомендации в отношении брандмауэров веб-приложений приведены на странице **Обзор** в разделе **Приложения**.

![Интеграция с центром безопасности][1]

Если щелкнуть любую рекомендацию по использованию брандмауэра веб-приложения, откроется новая страница с подробными сведениями о ней.

## <a name="add-a-web-application-firewall-to-an-existing-resource"></a>Добавление брандмауэра веб-приложения в имеющийся шлюз приложений

Перейдите в **раздел все службы** > **безопасность и идентификация** > **центре безопасности** и в разделе **Обзор центра безопасности —** щелкните **приложения**. Таблица в области **Центр безопасности — Приложения** содержит список приложений, обнаруженных центром безопасности в вашей подписке.

![Веб-приложения][3]

Если щелкнуть веб-приложение с критической ошибкой, откроется страница **Состояние защиты приложения**. На рисунке ниже показан пример веб-приложение, незащищенного брандмауэром веб-приложения. 

![Незащищенные веб-ресурсы][2]

В разделе **Рекомендации** щелкните **Добавить брандмауэр веб-приложения**, чтобы открыть страницу **Добавление брандмауэра веб-приложения**.

Если у вас нет шлюза приложений или вы хотите создать другой, в области **Создание брандмауэра веб-приложения** нажмите кнопку **Создать** и выберите **Microsoft — Шлюз приложений**. Выполните шаги по созданию шлюза приложений. На этом этапе вы добавили веб-приложение как защищенный ресурс. Теперь этот ресурс помечен в центре безопасности как защищенный брандмауэром веб-приложения. Но он еще не добавлен как элемент внутреннего пула.

Имеющийся шлюз приложений можно выбрать в разделе **Использование существующего решения**.

![Страница добавления брандмауэра веб-приложения][4]

Если вы добавляете веб-приложение в шлюз приложений через центр безопасности, этот ресурс автоматически не добавляется как элемент внутреннего пула. Это необходимо сделать напрямую в ресурсе шлюза приложений.

## <a name="add-a-resource-to-an-existing-web-application-firewall"></a>Добавление ресурса в имеющийся брандмауэр веб-приложения

Перейдите в **раздел все службы** > **безопасность и идентификация** > **центре безопасности** и в разделе **Обзор центра безопасности —** щелкните **решения партнеров**. Имеющиеся шлюзы приложений, поддерживающие центр безопасности, отображаются на странице **Решения партнеров**.

![Решения партнеров][7]

Щелкните **Связать приложение**, чтобы открыть страницу **Связь приложений**. Здесь можно выбрать существующие приложения. Выберите приложения, которые необходимо защитить, и нажмите кнопку **ОК**. Эти действия не добавляют веб-приложение во внутренний пул. Ресурсы только помечаются как защищенные, за счет чего центр безопасности может отслеживать их. Чтобы добавить ресурс как элемент внутреннего пула, необходимо войти в шлюз приложений. Для этого на текущей странице щелкните **Консоль решений**, чтобы перейти к ресурсу шлюза приложений, где веб-приложение можно добавить во внутренний пул.

![Сторонние приложения][6]

## <a name="finalize-configuration"></a>Завершение конфигурации

Центр безопасности отслеживает приложения, добавленные в шлюз приложений как защищенный ресурс.  Он отслеживает работоспособность этого ресурса и обеспечивает его защиту в шлюзе приложений. Далее во внутренний пул шлюза приложений необходимо добавить частный IP-адрес, общедоступный IP-адрес или сетевой адаптер виртуальной машины. Во время выполнения этих действий отображаются дополнительные рекомендации по **завершению защиты приложения**.

![Страница добавления брандмауэра веб-приложения][5]

## <a name="security-alerts"></a>Оповещения системы безопасности

В центре безопасности перейдите к разделу **Обнаружение** > **оповещений системы безопасности**.  Здесь отображаются оповещения WAF для шлюзов приложений. Оповещения упорядочены по правилу WAF.

![Оповещения системы безопасности][8]

При выборе правила будет предоставлен список предупреждений для этого конкретного правила WAF. Каждое оповещение содержит дополнительные сведения, а также ссылку на шлюз приложений.
 
![Сведения в оповещении][9]

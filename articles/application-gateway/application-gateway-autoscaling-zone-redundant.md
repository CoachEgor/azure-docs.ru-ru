# <a name="autoscaling-and-zone-redundant-application-gateway"></a>Автоматическое масштабирование и шлюз приложений, избыточные в пределах зоны 

Шлюз приложений и брандмауэра веб-приложения (WAF) также доступны в разделе Standard_v2 и WAF_v2 SKU. V2 SKU обеспечивает повышение производительности и добавляет поддержку для важных новых функций, таких как автоматическое масштабирование, избыточность зон и поддержка статических виртуальных IP-адресов. Существующие функции в разделе "стандартный" и SKU для WAF по-прежнему будут поддерживаться в новый SKU v2, за некоторыми исключениями, перечисленные в [сравнения](#differences-with-v1-sku) раздел.

Новый номер SKU v2, включает следующие улучшения:

- **Автомасштабирование.** Шлюз приложений или развертывания WAF в SKU автоматического масштабирования могут изменять масштаб в соответствии с изменениями в шаблоне нагрузки трафика. Кроме того, автоматическое масштабирование позволяет не выбирать размер развертывания или количество экземпляров во время подготовки. Этот номер SKU предлагает true эластичности. В Standard_v2 и WAF_v2 SKU шлюз приложений может работать фиксированной емкости (отключить Автомасштабирование) и в режиме с поддержкой автомасштабирования. Режим фиксированной емкости подходит для сценариев с постоянной, предсказуемой рабочей нагрузкой. Режим автоматического масштабирования целесообразно использовать в приложениях, см. в разделе Вариативность в трафика приложения.
- **Избыточность в пределах зоны**. Шлюз приложений или развертывать WAF в может охватывать несколько зон доступности, избавляя от необходимости подготавливать отдельные экземпляры шлюза приложений в каждой зоне с помощью диспетчера трафика. Вы можете одной зоне или несколько зон, где развернуты экземпляры шлюза приложений, что делает его более устойчивым к сбоям зоны. Внутренний пул приложений можно аналогичным образом распределить по зонам доступности.

  Избыточность зон доступна только где доступны зоны Azure. В других регионах поддерживаются другие функции. Дополнительные сведения см. в разделе [что такое зоны доступности в Azure?](../availability-zones/az-overview.md#services-support-by-region)
- **Статический виртуальный IP-адрес**. Введите исключительно SKU v2 шлюз приложения поддерживает статический виртуальный IP-адрес. Это гарантирует, что виртуальный IP-адрес, связанный со шлюзом приложений не изменяется для жизненный цикл развертывания, даже после перезагрузки.
- **Перезапись заголовка**: Шлюз приложений позволяет добавить, удалить или обновить заголовки запроса и ответа HTTP с номером SKU v2. Дополнительные сведения см. в разделе [заголовки HTTP, перепишите со шлюзом приложений](rewrite-http-headers.md)
- **Интеграция Key Vault (Предварительная версия)**: Версия 2 шлюз приложения поддерживает интеграцию с хранилищем ключей (в общедоступной предварительной версии) для сертификатов сервера, подключенных к прослушиватели протокол HTTPS. Дополнительные сведения см. в разделе [завершение запросов SSL с сертификатами в Key Vault](key-vault-certs.md).
- **Контроллеру входящего трафика службы Azure Kubernetes (Предварительная версия)**: Контроллеру входящего трафика шлюза приложений версии 2 позволяет шлюз приложений Azure для использования в качестве входящих данных для Azure Kubernetes Service (AKS) называется кластера AKS. Дополнительные сведения см. в разделе [страницы документации](https://azure.github.io/application-gateway-kubernetes-ingress/).
- **Повышение производительности**. Версии 2, номер SKU предлагает до 5 X лучше SSL разгрузки производительности по сравнению с номера SKU Standard/WAF.
- **Сократить время развертывания и обновления** v2 SKU обеспечивает сократить время развертывания и обновления по сравнению с номера SKU Standard/WAF. Сюда также входят изменения конфигурации WAF.

## <a name="supported-regions"></a>Поддерживаемые регионы

Standard_v2 и WAF_v2 SKU доступна в следующих регионах: центрально-северной части США, центрально-южной части США, западной части США, западной части США 2, восточной части США, восточной части США 2, центральной части США, Северной Европе, Западной Европе, Юго-Восточной Азии, Центральной Франции, западной части Соединенного Королевства, Восточной Японии, Западной Японии. Дополнительные регионы будут добавляться в будущем.

## <a name="pricing"></a>Цены

С SKU v2 модель ценообразования зависит от потребления и более не подключен к количество экземпляров или размеров. Цены SKU v2 состоит из двух компонентов:

- **Фиксированная цена за** -это каждый час (или неполный час) стоимость подготовки Standard_v2 или WAF_v2 шлюза.
- **Цена за единицу емкости** -это связано с затратами на основе потребления, оплачиваются в дополнение к фиксированным издержкам. Плата за единицу мощности также вычисляется почасовой или частичное каждый час. Существуют три измерения единицу емкости — вычислительные единицы, постоянные подключения и пропускной способности. Потребленные единицы — это мера емкость процессора. Факторы, влияющие на единицу вычисления являются подключений TLS в секунду, переопределения URL-адресов вычислений и обработку правил для WAF. Постоянное подключение — это мера из установленных подключений TCP в шлюз приложений в заданном интервале выставления счетов. Пропускная способность — среднее мегабит в секунду обрабатываемых в системе в течение заданного интервала выставления счетов.

Каждая единица емкости представляет собой не более: 1 вычислительный подразделение, или постоянные подключения 2500 или 2.22 Мбит/с пропускной способности.

Единица рекомендации к вычислительным ресурсам:

- **Standard_v2** -каждой вычислительной единицы способен приблизительно 50 подключений с TLS сертификата ключа RSA 2048 бит в секунду.
- **WAF_v2** - каждого вычисления единицы может поддерживать приблизительно 10 одновременных запросов в секунду, для запросов набора 70 / 30% трафика с 70% меньше, чем 2 КБ GET/POST и оставшихся более поздней версии. WAF производительности не зависит от размера ответа в настоящее время.

> [!NOTE]
> Каждый экземпляр в настоящее время поддерживает примерно 10 единиц мощности.
> Число запросов, которые можно обрабатывать единицу вычисления зависит от различных критериев, такая как размер ключа сертификата TLS, алгоритм обмена ключами, заголовка перезаписи, а также в случае WAF размер входящего запроса. Мы рекомендуем выполнять тесты приложения для определения частоты запросов на единицу вычислений. Единицей мощности и единицу вычисления будут доступны как метрики перед выставлением счета начинается.

## <a name="scaling-application-gateway-and-waf-v2"></a>Масштабирование v2 шлюза приложений с WAF

Шлюз приложений и WAF можно настроить для масштабирования в двух режимах:

- **Автоматическое масштабирование** — и автоматически масштабируются, шлюз приложений и WAF v2 SKU вертикальное масштабирование в зависимости от требований трафика приложения. Этот режим обеспечивает лучшую эластичности в приложение и избавляет от необходимости угадать число размер или экземпляр шлюза приложений. Этот режим также позволяет сократить затраты, не требуя для выполнения подготовки пиковой нагрузки ожидаемого трафика максимальное достигнуто максимальное число шлюзов. Клиентам необходимо указать минимальное и при необходимости максимальных значений. Минимальная емкость гарантирует, что шлюз приложений и WAF v2 не упадет ниже минимальное число экземпляров указано, даже при отсутствии трафика. Плата взимается для этого минимальную емкость, даже при отсутствии любого трафика. Также при необходимости можно указать максимальное число экземпляров, что гарантирует, что шлюз приложений не масштабируется за указанное число экземпляров. Вы перейдете к выставляются объема трафика, переданного с помощью шлюза. Количество экземпляров находится в диапазоне от 0 до 125. Максимальное число экземпляров по умолчанию — 20 Если не указано.
- **Вручную** -можно также выбрать ручной режим, где Автомасштабирование не шлюза. В этом режиме при наличии большего объема трафика, чем что шлюз приложений или WAF можно обрабатывать, он может привести к потере трафика. С помощью ручной режим указав число экземпляров является обязательным. Число экземпляров может составлять от 1 до 125 экземпляров.

## <a name="what-is-the-azure-migrate-appliance"></a>Что такое прибор Azure Migrate?

Прибор Azure Migrate — это легкий прибор, который использует инструмент Azure Migrate: Server Assessment для обнаружения и оценки насерверах. Инструмент Azure Migrate: Server Migration также использует прибор для бесагентной миграции внутренних VM VMware.

Вот более подробная информация об приборе Azure Migrate:

- Прибор развернут на месте в качестве VM или физической машины.
- Прибор обнаруживает на месте машины и постоянно отправляет метаданные и данные о производительности машины в Azure Migrate.
- Обнаружение устройства осуществляется без агента. На обнаруженных машинах ничего не установлено.

[Узнайте больше](migrate-appliance.md) об приборе.

## <a name="how-does-the-appliance-connect-to-azure"></a>Как прибор подключается к Azure?

Прибор может подключиться через Интернет или с помощью Azure ExpressRoute с публичным/Microsoft.

## <a name="does-appliance-analysis-affect-performance"></a>Повлияет ли анализ приборов на производительность?

Профили приборов Azure Migrate постоянно работают в помещениях для измерения данных о производительности. Это профилирование почти не влияет на производительность профилированных машин.

## <a name="can-i-harden-the-appliance-vm"></a>Могу ли я затвердеть прибор VM?

При использовании загруженного шаблона для создания прибора VM можно добавить компоненты (антивирус, например) в шаблон, если вы оставите в силе правила связи и брандмауэра, необходимые для прибора Azure Migrate.

## <a name="how-is-data-stored"></a>Способ хранения данных

Данные, собранные прибором Azure Migrate, хранятся в месте, где был создан проект Azure Migrate.

Вот более подробная информация о том, как хранятся данные:

- Собранные данные надежно хранятся в CosmosDB в подписке Майкрософт. Данные удаляются при удалении проекта Azure Migrate. Хранение обрабатывается Azure Migrate. Вы не можете специально выбрать учетную запись хранения для собранных данных.
- При [использовании визуализации зависимостей](concepts-dependency-visualization.md)собранные данные хранятся в США в рабочем пространстве Analytics Azure, созданном в подписке Azure. Данные удаляются при удалении рабочего пространства Log Analytics в подписке.

## <a name="how-much-data-is-uploaded-during-continuous-profiling"></a>Сколько данных загружается во время непрерывного профилирования?

Объем данных, отправляемых в Azure Migrate, зависит от нескольких параметров. Например, проект Azure Migrate, который имеет 10 машин (каждый с одним диском и одним NIC), отправляет приблизительно 50 МБ данных в день. Это значение является приблизительным; фактическое значение варьируется в зависимости от количества точек данных для дисков и NICs. Если количество машин, дисков или NICs увеличивается, увеличение объема отправляемых данных является нелинейным.

## <a name="is-data-encrypted-at-rest-and-in-transit"></a>Зашифрованы ли данные в состоянии покоя и в пути?

Да, для обоих:

- Метаданные надежно отправляются в службу Azure Migrate через Интернет через HTTPS.
- Метаданные хранятся в базе данных [Azure Cosmos](../cosmos-db/database-encryption-at-rest.md) и в [хранилище Azure Blob](../storage/common/storage-service-encryption.md) в подписке Майкрософт. Метаданные шифруются в состоянии покоя для хранения.
- Данные для анализа зависимостей также шифруются в пути (по безопасному HTTPS). Он хранится в рабочей области журнала Analytics в вашей подписке. Данные шифруются в состоянии покоя для анализа зависимостей.

## <a name="how-does-the-appliance-connect-to-vcenter-server"></a>Как прибор подключается к vCenter Server?

Эти шаги описывают, как прибор подключается к VMware vCenter Server:

1. Прибор подключается к vCenter Server (порт 443) с помощью предоставленных учетных данных при настройке прибора.
2. Прибор использует VMware PowerCLI для запроса vCenter Server для сбора метаданных о вс-чах, управляемых vCenter Server.
3. Прибор собирает данные конфигурации о ВМ (ядра, память, диски, NICs) и истории производительности каждого VM за последний месяц.
4. Собранные метаданные отправляются в инструмент Azure Migrate: Server Assessment (через Интернет через HTTPS) для оценки.

## <a name="can-the-azure-migrate-appliance-connect-to-multiple-vcenter-servers"></a>Может ли прибор Azure Migrate подключиться к нескольким серверам vCenter?

Нет. Между прибором Azure Migrate и vCenter Server имеется отображение между [прибором Azure Migrate.](migrate-appliance.md) Чтобы обнаружить вневые визы в нескольких экземплярах vCenter Server, необходимо развернуть несколько приборов. 

## <a name="can-an-azure-migrate-project-have-multiple-appliances"></a>Может ли проект Azure Migrate иметь несколько приборов?
Проект может иметь несколько приборов прилагается к нему. Однако прибор может быть связан только с одним проектом. 


## <a name="how-many-vms-or-servers-can-i-discover-with-an-appliance"></a>Сколько вс-а-а-а-а,- я могу обнаружить с помощью прибора?

Вы можете обнаружить до 10 000 VMw, до 5000 V-V ИТ-м и до 250 физических серверов с одним прибором. Если у вас больше машин в среде, прочитайте о [масштабировании оценки Hyper-V,](scale-hyper-v-assessment.md) [масштабировании оценки VMware](scale-vmware-assessment.md)и [масштабировании оценки физического сервера.](scale-physical-assessment.md)

## <a name="can-i-delete-an-appliance"></a>Могу ли я удалить прибор?

В настоящее время исключение прибора из проекта не поддерживается.

Единственным способом удаления прибора является удаление группы ресурсов, содержащей проект Azure Migrate, связанный с прибором.

Однако удаление группы ресурсов также удаляет другие зарегистрированные приборы, обнаруженные запасы, оценки и все другие компоненты Azure в группе ресурсов, связанные с проектом.

## <a name="can-i-use-the-appliance-with-a-different-subscription-or-project"></a>Могу ли я использовать прибор с другой подпиской или проектом?

После использования прибора для инициирования обнаружения невозможно перенастроить прибор для использования с другой подпиской Azure, и вы не можете использовать его в другом проекте Azure Migrate. Вы также не можете обнаружить vMs на другом экземпляре vCenter Server. Настройка нового прибора для этих задач.

## <a name="can-i-set-up-the-appliance-on-an-azure-vm"></a>Можно ли настроить прибор на Azure VM?

Нет. В настоящее время эта опция не поддерживается. 

## <a name="can-i-discover-on-an-esxi-host"></a>Могу ли я обнаружить на хосте ESXi?

Нет. Чтобы обнаружить VMware VMs, вы должны иметь vCenter Server.

## <a name="how-do-i-update-the-appliance"></a>Как обновить прибор?

По умолчанию прибор и установленные агенты обновляются автоматически. Прибор проверяет обновления каждые 24 часа. Обновления, которые терпят неудачу, повторяются. 

Только прибор и приборы обновляются этими автоматическими обновлениями. Операционная система не обновляется автоматическими обновлениями Azure Migrate. Используйте обновления Windows, чтобы поддерживать операционную систему в актуальном состоянии.

## <a name="can-i-check-agent-health"></a>Могу ли я проверить здоровье агента?

Да. На портале перейдите на страницу **работы агента** для Azure Migrate: Оценка серверов или Azure Migrate: Инструмент миграции серверов. Там можно проверить состояние соединения между Azure и агентами обнаружения и оценки на приборе.

# <a name="azure-migrate-appliance-common-questions"></a>Устройство для миграции Azure: распространенные вопросы

В этой статье содержатся ответы на часто задаваемые вопросы об устройстве миграции Azure. Если у вас есть другие вопросы, ознакомьтесь со следующими статьями:

- [Общие вопросы](resources-faq.md) о службе "миграция Azure".
- [Вопросы](common-questions-discovery-assessment.md) об обнаружении, оценке и визуализации зависимостей.
- [Вопросы](common-questions-server-migration.md) о миграции сервера.
- Получите ответы на вопросы на [форуме по миграции Azure](https://aka.ms/AzureMigrateForum). 


## <a name="what-is-the-azure-migrate-appliance"></a>Что такое устройство миграции Azure?

Устройство "миграция Azure" — это упрощенное устройство, используемое средством "миграция Azure" для обнаружения и оценки локальных серверов. Это устройство также используется средством миграции Azure Migration Tool, предназначенным для миграции локальных виртуальных машин VMware без агента. 

- Устройство развертывается локально в качестве виртуальной машины или физического компьютера.
- Устройство обнаруживает локальные компьютеры и постоянно отправляет метаданные и данные о производительности компьютера в службу "миграция Azure".
- Обнаружение устройства осуществляется без агента. На обнаруженных компьютерах ничего не установлено.

Дополнительные [сведения](migrate-appliance.md) об устройстве.

## <a name="how-does-the-appliance-connect-to-azure"></a>Как устройство подключается к Azure?

Устройство может подключаться через Интернет или использовать Azure ExpressRoute с общедоступным или пирингом Майкрософт.

## <a name="does-appliance-analysis-impact-performance"></a>Влияет ли на производительность анализа устройств?

Azure выполняет миграцию локальных компьютеров в локальные профили для измерения данных производительности. Это профилирование почти не влияет на производительность на компьютерах.

### <a name="can-i-harden-the-appliance-vm"></a>Можно ли защитить виртуальную машину устройства?

При создании виртуальной машины устройства с помощью скачанного шаблона можно добавить в шаблон компоненты (например, антивирусную программу), если вы оставляете правила подключения и брандмауэра, необходимые для устройства "миграция Azure".


## <a name="what-network-connectivity-is-needed"></a>Какое сетевое подключение требуется?

Ознакомьтесь со сведениями о сетевом подключении:
- Оценка VMware с помощью устройства "миграция Azure": требования к [URL-адресу](migrate-appliance.md#url-access) и [портам](migrate-support-matrix-vmware.md#port-access) .
- Миграция без агента VMware с помощью устройства "миграция Azure": требования к [URL-адресу](migrate-appliance.md#url-access) и [портам](migrate-support-matrix-vmware-migration.md#agentless-ports) .
- Оценка Hyper-V с помощью устройства "миграция Azure": требования к [URL-адресу](migrate-appliance.md#url-access) и [портам](migrate-support-matrix-hyper-v.md#port-access) .


## <a name="what-data-does-the-appliance-collect"></a>Какие данные собираются устройством?

Проверьте собранные данные.

- Данные и [метаданные](migrate-appliance.md#collected-metadata-vmware) [производительности](migrate-appliance.md#collected-performance-data-vmware) виртуальной машины VMware.
- Данные и [метаданные](migrate-appliance.md#collected-metadata-hyper-v) [производительности](migrate-appliance.md#collected-performance-data-hyper-v) виртуальной машины Hyper-V.


## <a name="how-is-data-stored"></a>Способ хранения данных

Данные, собранные устройством "миграция Azure", хранятся в расположении Azure, в котором был создан проект службы "миграция Azure". 

- Данные безопасно хранятся в подписке Майкрософт и удаляются при удалении проекта службы "миграция Azure".
- Если используется [Визуализация зависимостей](concepts-dependency-visualization.md), собранные данные сохраняются в США в рабочей области log Analytics, созданной в подписке Azure. При удалении рабочей области Log Analytics из подписки эти данные удаляются.

## <a name="how-much-data-is-uploaded-in-continuous-profiling"></a>Какой объем данных отправляется в непрерывном профилировании?

Объем данных, отправляемых в службу "миграция Azure", зависит от ряда параметров. Чтобы дать вам смысл, проект службы "миграция Azure" с 10 компьютерами (каждый с одним диском и одной сетевой картой) отправляет около 50 МБ в день. Это значение приблизительно и изменяется в зависимости от количества точек данных для сетевых карт и дисков. Увеличение числа отправленных данных не является линейным, если количество компьютеров, сетевых карт или дисков увеличивается.

## <a name="is-data-encrypted-at-restin-transit"></a>Зашифрованы ли данные в незащищенном или транзитном месте?

Да, для обоих.

- Метаданные безопасно отправляются в службу "миграция Azure" через Интернет по протоколу HTTPS.
- Метаданные хранятся в базе данных [Cosmos Azure](../cosmos-db/database-encryption-at-rest.md) и в [хранилище BLOB-объектов Azure](../storage/common/storage-service-encryption.md)в подписке Майкрософт. Метаданные шифруются неактивных.
- Данные для анализа зависимостей также шифруются транзитным (защищенным HTTPS). Он хранится в Log Analytics рабочей области в подписке. Он также шифруется неактивных.

## <a name="how-does-the-appliance-connect-to-vcenter-server"></a>Как устройство подключается к vCenter Server?

1. Устройство подключается к vCenter Server (порт 443), используя учетные данные, указанные при настройке устройства.
2. Устройство использует VMware PowerCLI для запроса vCenter Server, чтобы получить метаданные о виртуальных машинах, управляемых vCenter Server.
3. Устройство собирает данные конфигурации о виртуальных машинах (ядрах, памяти, дисках, сетевых адаптерах) и журнале производительности каждой виртуальной машины за прошлый месяц.
4. Собранные метаданные отправляются в службу "миграция Azure": Оценка сервера (через Интернет через HTTPS) для оценки.

## <a name="can-i-connect-the-appliance-to-multiple-vcenter-servers"></a>Можно ли подключить устройство к нескольким серверам vCenter?

Нет. Между устройством и vCenter Serverом существует однозначное сопоставление. Для обнаружения виртуальных машин на нескольких экземплярах vCenter Server необходимо развернуть несколько устройств.

## <a name="how-many-vms-or-servers-can-i-discover-with-an-appliance"></a>Сколько виртуальных машин или серверов можно обнаружить с помощью устройства?

Вы можете обнаружить до 10 000 виртуальных машин VMware, до 5 000 виртуальных машин Hyper-V и до 250 физических серверов с одним устройством. Если в локальной среде имеется больше компьютеров, ознакомьтесь с разрешениями по масштабированию [Hyper-V](scale-hyper-v-assessment.md), [VMware](scale-vmware-assessment.md) и [физической](scale-physical-assessment.md) оценки.

## <a name="can-i-delete-an-appliance"></a>Можно ли удалить устройство?

В настоящее время удаление устройства из проекта не поддерживается.

- Единственный способ удалить устройство — удалить группу ресурсов, содержащую проект службы "миграция Azure", связанный с устройством.
- Однако при удалении группы ресурсов также будут удалены другие зарегистрированные устройства, обнаруженные инвентаризации, оценки и все другие компоненты Azure, связанные с проектом в группе ресурсов.


## <a name="can-i-use-the-appliance-with-a-different-subscriptionproject"></a>Можно ли использовать устройство с другой подпиской или проектом?

После использования устройства для запуска обнаружения его нельзя перенастроить с помощью другой подписки Azure или использовать в другом проекте службы "миграция Azure". Вы также не можете обнаружить виртуальные машины на разных vCenter Server. Настройте новое устройство для этих задач.

## <a name="can-i-set-up-the-appliance-on-an-azure-vm"></a>Можно ли настроить устройство на виртуальной машине Azure?
Нет. Сейчас это не поддерживается. 

## <a name="can-i-discover-on-an-esxi-host"></a>Могу ли я обнаружить на узле ESXi?
Нет. Для обнаружения виртуальных машин VMware требуется vCenter Server.

## <a name="how-do-i-update-the-appliance"></a>Разделы справки обновить устройство?

По умолчанию устройство и его установленные агенты обновляются автоматически. Устройство проверяет наличие обновлений каждые 24 часа. В случае сбоя процесса обновления происходит повторная попытка. Служба автоматического обновления обновляет только агенты устройств и устройств. Операционная система не обновлена. Используйте обновления Майкрософт, чтобы обновлять операционную систему в актуальном состоянии.

## <a name="can-i-check-agent-health"></a>Можно ли проверить работоспособность агента?

На портале перейдите на страницу **работоспособность агента** в средстве оценки серверов или миграции сервера. Там можно проверить состояние подключения между агентами обнаружения и оценки на устройстве и в Azure.

# <a name="best-practices-for-creating-assessments"></a>Рекомендации по созданию оценок

Служба [Миграция Azure](migrate-overview.md) объединяет в себе инструменты, которые используются для поиска, оценки и переноса приложений, инфраструктуры и рабочих нагрузок в Microsoft Azure. Она включает в себя инструменты Миграции Azure и предложения независимых поставщиков программного обеспечения.

В этой статье кратко излагаются рекомендации при создании оценок с использованием инструмента оценки Azure Migrate Server Assessment.

## <a name="about-assessments"></a>Об оценках

Оценки, которые вы создаете с помощью оценки Azure Migrate Server Assessment, являются моментальным моментом данных. В Azure Migrate существует два типа оценок.

**Тип оценки** | **Подробно** | **Данные**
--- | --- | ---
**На основе производительности** | Оценки, которые делают рекомендации на основе собранных данных о производительности | Рекомендация по размеру VM основана на данных по использованию процессора и памяти.<br/><br/> Рекомендация типа диска (стандартные HDD/SSD или диски премиум-класса) основана на IOPS и пропускной их пропускной.
**Как-это на-предпосылки** | Оценки, которые не используют данные о производительности для выполнения рекомендаций. | Рекомендация по размеру VM основана на<br/><br> Рекомендуемый тип диска основан на том, что вы выбираете в настройках типа хранилища для оценки.

### <a name="example"></a>Пример
Например, если у вас есть на месте VM с четырьмя ядрами на 20% использования, и память 8 ГБ с 10% использования, оценки будут следующими:

- **Оценка на основе производительности:**
    - Определяет эффективные ядра и память на основе ядра (4 х 0,20 и 0,8), а также памяти (8 ГБ х 0,10 и 0,8) использования.
    - Применяет коэффициент комфорта, указанный в свойствах оценки (скажем, 1.3x), чтобы получить значения, которые будут использоваться для размеров. 
    - Рекомендует ближайший размер VM в Azure, который может поддерживать ядра 1,04 евро (0,8 х 1,3) и 1,04 ГБ (0,8 х 1,3) памяти.

- **Как-это (как на месте) оценка:**
    -  Рекомендует VM с четырьмя ядрами; 8 ГБ памяти.

## <a name="best-practices-for-creating-assessments"></a>Рекомендации по созданию оценок

Прибор Azure Migrate непрерывно профилирует вашу среду и отправляет метаданные и данные о производительности в Azure. Следуйте этим рекомендациям для оценки серверов, обнаруженных с помощью прибора:

- **Создавайте оценки:** Вы можете создавать оценки как это сразу же после того, как ваши машины появляются на портале Azure Migrate.
- **Создание оценки на основе производительности:** После настройки обнаружения мы рекомендуем вам подождать по крайней мере за день до запуска оценки на основе производительности:
    - Сбор данных о производительности требует времени. Ожидание по крайней мере в день гарантирует наличие достаточного количества точек данных о производительности перед запуском оценки.
    - При выполнении оценок на основе производительности убедитесь, что вы профилируете среду на время оценки. Например, если вы создаете оценку с продолжительностью производительности, установленной до одной недели, необходимо подождать не менее недели после начала обнаружения, чтобы собрать все точки данных. Если вы этого не сможете, оценка не получит пятизвездочный рейтинг.
- **Перерасчет оценок:** Поскольку оценки являются моментальными моментальными снимками, они не обновляются автоматически с последними данными. Чтобы обновить оценку с последними данными, необходимо пересчитать ее.

Следуйте этим рекомендациям для оценки серверов, импортируемых в Azure Migrate через . Файл CSV:

- **Создавайте оценки:** Вы можете создавать оценки как это сразу же после того, как ваши машины появляются на портале Azure Migrate.
- **Создание оценки на основе производительности**: Это помогает получить более высокую оценку затрат, особенно если у вас есть перезадеченная емкость сервера на месте. Однако точность оценки на основе производительности зависит от данных о производительности, указанных вами для серверов. 
- **Перерасчет оценок:** Поскольку оценки являются моментальными моментальными снимками, они не обновляются автоматически с последними данными. Чтобы обновить оценку с последними импортированными данными, необходимо пересчитать ее.

## <a name="best-practices-for-confidence-ratings"></a>Лучшие практики для оценки доверия

При выполнении оценок на основе производительности оценка доверия от 1-звезды (самая низкая) до 5-звездочный (самый высокий) присуждается оценке. Эффективноиспользовать рейтинги доверия:
- Оценка перехода на сервер Azure Migrate Требует данных об использовании процессора VM/Memory.
- Для каждого диска, прикрепленного к предприимчивому VM, ему нужны данные чтения/записи IOPS/пропускной записи.
- Для каждого сетевого адаптера, прикрепленного к VM, ему нужна сеть в/высохае данные.

В зависимости от процентной доли точек данных, доступных для выбранного срока, рейтинг доверия к оценке приводится в соответствии со следующей таблицей.

   **Доступность точки данных** | **Оценка достоверности**
   --- | ---
   0–20 % | 1 звезда
   21–40 % | 2 звезды
   41–60 % | 3 звезды
   61–80 % | 4 звезды
   81–100 % | 5 звезд


## <a name="common-assessment-issues"></a>Общие вопросы оценки

Вот как решить некоторые общие проблемы среды, которые влияют на оценки.

###  <a name="out-of-sync-assessments"></a>Оценки вне синхронизации

Если вы добавите или удалите машины из группы после создания оценки, созданная оценка будет отмечена **вне синхронизации.** Повторите оценку **(Recalculate),** чтобы отразить изменения в группе.

### <a name="outdated-assessments"></a>Устаревшие оценки

Если есть предварительные изменения в ВМ, которые находятся в группе, которая была оценена, оценка помечена **как устаревшая.** Чтобы отразить изменения, запустите оценку еще раз.

### <a name="low-confidence-rating"></a>Низкий рейтинг доверия

Оценка может иметь не все точки данных по ряду причин:

- Среда не была профилирована на период времени, для которого создается оценка. Например, если вы создаете *оценку производительности* с продолжительностью производительности, установленной до одной недели, вам нужно подождать по крайней мере неделю после начала обнаружения для всех точек данных, чтобы получить собранные. Вы всегда можете нажать на **Recalculate,** чтобы увидеть последний применимый рейтинг доверия. Рейтинг доверия применим только при создании оценки *производительности.*

- В течение периода, для которого рассчитывается оценка, несколько виртуальных машин были отключены. Если какие-либо виртуальные машины были отключены в течение некоторого времени, функция оценки сервера не сможет собрать данные о производительности за этот период.

- Несколько виртуальных машин были созданы после запуска обнаружения в решении "Оценка сервера". Например, вы создаете оценку для журнала производительности за последний месяц, при этом несколько виртуальных машин были созданы всего неделю назад. В этом случает данные о производительности новых машин будут недоступны для всего периода анализа, а рейтинг достоверности будет низким.

# <a name="assess-large-numbers-of-hyper-v-vms-for-migration-to-azure"></a>Оценить большое число виртуальных машин Hyper-V для миграции в Azure

В этой статье описывается, как оценить большое (> 1000) из локальных виртуальных машин Hyper-V для миграции в Azure, используя средство оценки сервера миграции Azure.

[Служба "Миграция Azure"](migrate-services-overview.md) представляет собой центр для средств, которые помогают обнаруживать, оценивать и перенос приложений, инфраструктуры и рабочих нагрузок в Microsoft Azure. Концентратор включает "Миграция Azure" Инструменты и независимые поставщики независимых стороннего программного обеспечения. 


В этой статье раскрываются следующие темы:
> [!div class="checklist"]
> * План для оценки в нужном масштабе.
> * Настройте разрешения Azure и подготовка Hyper-V для оценки.
> * Создание проекта "Миграция Azure" и создайте оценку.
> * Просмотрите оценки при планировании миграции.


> [!NOTE]
> Если вы хотите опробовать-экспериментальную чтобы оценить несколько виртуальных машин перед оценкой в масштабе, выполните наши [серии руководств](tutorial-prepare-hyper-v.md)

## <a name="plan-for-assessment"></a>План для оценки

При планировании оценку большого количества виртуальных машин Hyper-V, существует несколько предметов для обдумывания:

- **Планирование проектов "Миграция Azure"** : Понять, как развернуть проекты "Миграция Azure". Например если в центрах обработки данных находятся в разных географических регионах или для хранения в другом географическом регионе обнаружения, оценки или метаданные, связанные с миграцией, может потребоваться несколько проектов.
- **План устройства**: Служба "Миграция Azure" использует "Миграция Azure" локального устройства, развернуть в качестве виртуальной Машины Hyper-V, постоянно обнаружить виртуальные машины для внутренней оценки и миграции. Модуль отслеживает изменения среды, таких как добавление виртуальных машин, дисков и сетевых адаптеров. Он также отправляет метаданные и данные производительности о них в Azure. Необходимо выяснить, сколько устройств для развертывания.


## <a name="planning-limits"></a>Планирование ограничения
 
Используйте ограничения, описано в этой таблице по планированию.

**Планирование** | **Ограничения**
--- | --- 
**Проекты Azure служба "Миграция"** | Оценка до 10 000 виртуальных машин в проекте.
**Устройство Azure служба "Миграция"** | Устройства можно обнаружить до 5000 виртуальные машины.<br/> Устройства могут подключаться к до 300 узлов Hyper-V.<br/> Устройство может быть связан только с одного проекта "Миграция Azure".<br/><br/> 
**Оценка Azure служба "Миграция"** | Можно оценить до 10 000 виртуальных машин в одной операции оценки.



## <a name="other-planning-considerations"></a>Другие рекомендации по планированию

- Для запуска обнаружения от устройства, необходимо выбрать в каждом узле Hyper-V. 
- Если вы работаете в среде с несколькими клиентами, вам не удается обнаружить в настоящее время только виртуальные машины, которые принадлежат конкретным клиентам. 

## <a name="prepare-for-assessment"></a>Подготовка к оценке

Подготовьте Azure и Hyper-V для оценки server. 

1. Проверьте [требования поддержки Hyper-V и ограничения](migrate-support-matrix-hyper-v.md).
2. Настройка разрешений для учетной записи Azure для взаимодействия с "Миграция Azure"
3. Подготовка узлов Hyper-V и виртуальных машин

Следуйте инструкциям в [учебником](tutorial-prepare-hyper-v.md) для настройки этих параметров.

## <a name="create-a-project"></a>Создание проекта

В соответствии с требованиями планирования сделайте следующее:

1. Создание проектов "Миграция Azure".
2. Добавьте средство оценки сервера миграции Azure в проекты.

[Подробнее](how-to-add-tool-first-time.md)

## <a name="create-and-review-an-assessment"></a>Создание и просмотр оценки

1. Создайте оценки для виртуальных машин Hyper-V.
1. Просмотрите оценки в процессе подготовки к планированию миграции.

[Дополнительные сведения](tutorial-assess-hyper-v.md) о создании и просмотре оценок.

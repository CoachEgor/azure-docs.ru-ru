# <a name="assess-large-numbers-of-hyper-v-vms-for-migration-to-azure"></a>Оценка большого количества ви-м с от визами hyper-V для миграции в Azure

В этой статье описывается, как оценить большое количество наемных микро-им. КрМ для миграции в Azure с помощью инструмента оценки azure Migrate Server Assessment.

Служба [Миграция Azure](migrate-services-overview.md) объединяет в себе инструменты, которые используются для поиска, оценки и переноса приложений, инфраструктуры и рабочих нагрузок в Microsoft Azure. Она включает в себя инструменты Миграции Azure и предложения независимых поставщиков программного обеспечения. 


Вы узнаете, как выполнять следующие задачи:
> [!div class="checklist"]
> * План оценки в масштабе.
> * Нанастройка разрешений Azure и подготовка Hyper-V к оценке.
> * Создайте проект Azure Migrate и создайте оценку.
> * Просмотрите оценку по мере планирования миграции.


> [!NOTE]
> Если вы хотите попробовать доказательство концепции для оценки нескольких ВМ, прежде чем оценивать в масштабе, следуйте нашей [серии учебников](tutorial-prepare-hyper-v.md)

## <a name="plan-for-assessment"></a>План оценки

При планировании оценки большого количества Hyper-V VM, Есть несколько вещей, чтобы думать о:

- **Планируйте проекты Azure Migrate:** Выясните, как развернуть проекты Azure Migrate. Например, если ваши центры обработки данных находятся в разных географических регионах или вам необходимо хранить метаданные обнаружения, оценки или миграции в другой географии, возможно, потребуется несколько проектов.
- **Приборы планирования:** Azure Migrate использует наместере прибор Azure Migrate, развернутый в качестве Hyper-V VM, чтобы постоянно открывать визы для оценки и миграции. Прибор отслеживает изменения среды, такие как добавление виз, дисков или сетевых адаптеров. Он также отправляет метаданные и данные о производительности о них в Azure. Вам нужно выяснить, сколько приборов для развертывания.


## <a name="planning-limits"></a>Ограничения планирования
 
Используйте ограничения, суммированные в этой таблице, для планирования.

**Планирования** | **Ограничения**
--- | --- 
**Проекты Azure Migrate** | Оцените до 35 000 вм в проекте.
**Устройство службы "Миграция Azure"** | Прибор может обнаружить до 5000 взл-Мм.<br/> Прибор может подключаться к 300 хостам Hyper-V.<br/> Прибор может быть связан только с одним проектом Azure Migrate.<br/> Любое количество приборов может быть связано с одним проектом Azure Migrate. <br/><br/> 
**Группы** | В одну группу можно добавить до 35 000 вм:
**Оценка миграции Azure** | Вы можете оценить до 35 000 вм в одной оценке.



## <a name="other-planning-considerations"></a>Другие соображения планирования

- Чтобы начать открытие с прибора, вы должны выбрать каждый хост Hyper-V. 
- Если вы работаете с мультитенантной средой, вы не можете в настоящее время обнаружить только VMs, которые принадлежат конкретному арендатору.

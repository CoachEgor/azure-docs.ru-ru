# <a name="add-migration-tools"></a>Добавление средств миграции

В этой статье описывается, как добавлять инструменты миграции в [Azure Migrate](migrate-overview.md).

Azure Migrate предоставляет центр инструментов для оценки и миграции в Azure. Она включает в себя нативные инструменты, инструменты, предоставляемые другими службами Azure, и сторонних независимых поставщиков программного обеспечения (ISV).

Если вы хотите добавить инструмент миграции и еще не создали проект Azure Migrate, следуйте этой [статье](how-to-add-tool-first-time.md):



## <a name="selecting-an-isv-tool"></a>Выбор инструментов независимого поставщика программного обеспечения

Если вы выберете [инструмент ISV](migrate-services-overview.md#isv-integration) для миграции, вы можете начать с получения лицензии или подписки на бесплатную пробную версию в соответствии с политикой ISV. В каждом инструменте есть возможность подключиться к службе "Миграция Azure". Развертывайте инструмент и следуйте инструкциям и документации инструмента для подключения рабочего пространства инструмента к Azure Migrate. 

## <a name="select-a-migration-scenario"></a>Выберите сценарий миграции

1. В проекте службы "Миграция Azure" щелкните **Обзор**.
2. Выберите сценарий миграции, который вы хотите использовать:

    - Чтобы перенести машины и рабочие нагрузки в Azure, выберите **Оценку и перенос серверов.**
    - Для переноса нахидных станков S'L выберите **Оценку и перенос баз данных.**
    - Для переноса на территории веб-приложений выберите **Оценку и миграцию веб-приложений.**
    - Чтобы перенести большие объемы данных в Azure в автономном режиме, выберите **«Окно данных».**

    ![Сценарий оценки](./media/how-to-migrate/assess-scenario.png)

## <a name="select-a-server-migration-tool"></a>Выберите инструмент миграции сервера

1. Нажмите **Оценка и миграция серверов**.
2. В **Azure Migrate - Серверы,** если вы еще не добавили инструменты миграции, под **инструментами миграции,** выберите **Нажмите здесь, чтобы добавить инструмент миграции.** Если вы уже добавили инструменты миграции, в **добавляете больше инструментов миграции,** выберите **Изменение**.

    > [!NOTE]
    > Если вам нужно перейти к другому проекту, в **Azure Migrate - Серверы**, рядом с **Просеяните детали для другого проекта мигрировать,** нажмите **Нажмите здесь**.

3. В **Azure Migrate**выберите инструмент миграции, который вы хотите использовать.
    - При использовании миграции серверов Azure Migrate можно настроить и запустить миграцию непосредственно в проекте Azure Migrate.
    - Если вы используете сторонний инструмент оценки, перейдите по ссылке, предоставленной для ISV, и запустите миграцию в соответствии с инструкциями, которые они предоставляют.

## <a name="select-a-database-migration-tool"></a>Выберите инструмент миграции базы данных

1. Нажмите **Оценка и миграция баз данных**
2. В **базах данных**нажмите **Добавить инструменты**.
3. В дополнение к инструменту > **Выберите инструмент миграции,** выберите инструмент, который вы хотите использовать для переноса базы данных.

## <a name="select-a-web-app-migration-tool"></a>Выберите инструмент миграции веб-приложений

1. Нажмите **Оценка и мигрировать веб-приложений**.
2. Перейдите по ссылке на инструмент «Миграция» для службы приложений Azure. Используйте инструмент миграции для:

    - **Оцените приложения онлайн:** Вы можете оценивать и мигрировать приложения с общедоступным URL-адресом в Интернете, используя помощник по миграции службы приложений Azure.
    - **.NET/PHP**: Для внутренних приложений .NET и PHP вы можете скачать и запустить помощник по миграции.

## <a name="order-an-azure-data-box"></a>Заказать ящик данных Azure

Для переноса больших объемов данных в Azure можно заказать поле Azure DAta для передачи данных в автономном режиме.

1. Нажмите **Заказать коробку данных**.
2. В **Select your Azure Data Box**укажите подписку. 
3. Передача будет импортом в Azure. Укажите источник данных и пункт назначения региона Azure для данных.

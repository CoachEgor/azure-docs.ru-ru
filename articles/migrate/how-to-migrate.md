# <a name="add-migration-tools"></a>Добавление средств миграции

В этой статье описывается, как добавить средства миграции в ["Миграция Azure"](migrate-overview.md).

Служба "Миграция Azure" предоставляют концентратор средств для оценки и миграции в Azure. Она также включает собственного средства, средства, предоставляемые других служб Azure и сторонних независимых Независимым производителям предложения.

Если вы хотите добавить средство миграции и еще не настроили проекта миграции Azure, выполните это [статье](how-to-add-tool-first-time.md).



## <a name="selecting-an-isv-tool"></a>Выбрав средство независимых поставщиков программного обеспечения

При выборе [независимых поставщиков программного обеспечения средство](migrate-services-overview.md#isv-integration) для миграции, можно запустить, получении лицензии или регистрации для получения бесплатной пробной версии, в соответствии с политикой независимых поставщиков программного обеспечения. В каждое средство имеется возможность подключиться к "Миграция Azure". Развертывание средства и следуйте инструкциям средства и документация к сообществу средства рабочей области "Миграция Azure". 

## <a name="select-a-migration-scenario"></a>Выберите сценарий миграции

1. В проекте "Миграция Azure", щелкните **Обзор**.
2. Выберите сценарий миграции, которую вы хотите использовать:

    - Чтобы перенести машины и рабочие нагрузки в Azure, выберите **оценки и миграции серверов**.
    - Чтобы перенести локальные компьютеры SQL, выберите **оценка и миграция баз данных**.
    - Чтобы перенести на локальном веб-приложений, выберите **оценка и перенос веб-приложений**.
    - Для переноса больших объемов локальных данных в Azure в автономном режиме, выберите **заказ Data Box**.

    ![Сценарий оценки](./media/how-to-migrate/assess-scenario.png)

## <a name="select-a-server-migration-tool"></a>Выберите средство миграции сервера

1. Нажмите кнопку **оценки и миграции серверов**.
2. В **"Миграция Azure" - серверы**, если вы еще не добавили средств миграции, в разделе **средства миграции**выберите **щелкните здесь, чтобы добавить средство миграции**. Если вы уже добавили средства миграции в **добавьте дополнительные средства миграции**выберите **изменение**.

    > [!NOTE]
    > Если необходимо перейти к другому проекту, в **"Миграция Azure" - серверы**рядом с элементом **см. в разделе сведений о проекте другой миграции**, нажмите кнопку **здесь**.

3. В **"Миграция Azure"** , выберите средство миграции, вы хотите использовать.
    - При использовании миграции сервера миграции Azure, можно установить и запустить миграций непосредственно в проекте "Миграция Azure".
    - Если вы используете это средство оценки независимых производителей, перейдите по ссылке для независимых поставщиков программного обеспечения и выполнение миграции в соответствии с инструкциями, которые они предоставляют.

## <a name="select-a-database-migration-tool"></a>Выберите средство миграции базы данных

1. Нажмите кнопку **оценка и миграция баз данных**
2. В **баз данных**, нажмите кнопку **добавить средства**.
3. В добавить средство > **выберите миграция**, выбрать инструмент, который вы хотите использовать для переноса базы данных.

## <a name="select-a-web-app-migration-tool"></a>Выберите веб-инструмент миграции приложений

1. Нажмите кнопку **оценка и перенос веб-приложений**.
2. Перейдите по ссылке на средства миграции для службы приложений Azure. Используйте средство миграции, чтобы:

    - **Оценка приложения в Интернете**: Можно оценить и перенести приложения с помощью общедоступного URL-адрес в Интернете, с помощью Azure App Service Migration Assistant.
    - **.NET/PHP**: Внутренние приложения .NET и PHP можно загрузить и запустить помощник по миграции.

## <a name="order-an-azure-data-box"></a>Заказ Azure Data Box

Для переноса больших объемов данных в Azure, вы можете заказать Azure DAta Box для передачи данных в автономном режиме.

1. Нажмите кнопку **заказ Data Box**.
2. В **выберите устройство Azure Data Box**, указать свою подписку. 
3. Передача будет импорта в Azure. Укажите источник данных и назначение регион Azure для данных.

# <a name="prepare-to-work-with-isv-toolsmovere"></a>Подготовка к работе со средствами независимых поставщиков программного обеспечения

Если вы добавили в проект службы "миграция Azure" [средство ISV](migrate-services-overview.md#isv-integration)или перенесите его, то перед компоновкой средства необходимо выполнить ряд действий и отправить данные в службу "миграция Azure". 

## <a name="check-azure-ad-permissions"></a>Проверка разрешений Azure AD

Для вашей учетной записи пользователя Azure требуется несколько разрешений:

- Разрешение на регистрацию приложения Azure AD в клиенте Azure.
- Разрешение на выделение роли для приложения Azure AD на уровне подписки.


## <a name="set-permissions-to-register-an-azure-ad-app"></a>Настройка разрешений для регистрации приложения Azure AD

1. В Azure Active Directory проверьте роль учетной записи. Если у вас есть роль пользователя, щелкните **Параметры пользователя** слева и проверьте, могут ли пользователи регистрировать приложения.
2. Если задано значение **Да**, все пользователи в клиенте Azure AD могут зарегистрировать приложение. Если это невозможно, только пользователи с правами администратора могут регистрировать приложения.
3. Если у вас нет разрешений, администратор может предоставить учетной записи пользователя роль [администратора приложения](../active-directory/users-groups-roles/directory-assign-admin-roles.md#application-administrator) , чтобы можно было зарегистрировать приложение.
4. После того как инструмент будет связан с миграцией Azure, администратор может удалить роль из вашей учетной записи.

## <a name="set-permissions-to-assign-a-role-to-an-azure-ad-app"></a>Настройка разрешений для назначения роли приложению Azure AD
 
В вашей подписке Azure учетной записи требуется **Microsoft. Authorization/*/Врите**, чтобы назначить роль для приложения Azure AD. 

1. Чтобы проверить разрешения подписки, в портал Azure откройте **подписки**.
2. Выберите нужную подписку. Если он не отображается, выберите **Фильтр глобальные подписки**. 
3. Выберите **Мои разрешения**. Затем выберите ссылку **Щелкните здесь, чтобы просмотреть полные сведения о доступе для этой подписки**.
4. В > **представлении** **назначений ролей**и проверьте разрешения.
5. Если у вашей учетной записи нет разрешений, попросите администратора подписки добавить вас в роль [администратора доступа пользователей](https://docs.microsoft.com/azure/role-based-access-control/built-in-roles#user-access-administrator) или роль [владельца](https://docs.microsoft.com/azure/role-based-access-control/built-in-roles#owner) .
 

## <a name="start-using-the-tool"></a>Начните использовать средство

1. Если у вас еще нет лицензии или бесплатной пробной версии средства, в записи средства в службе "миграция Azure" в поле " **Регистрация**" щелкните "дополнительные **сведения**".
2. В средстве следуйте инструкциям, чтобы создать ссылку из средства на проект службы "миграция Azure" и отправить данные в службу "миграция Azure".

## <a name="next-steps"></a>Дальнейшие действия

Чтобы отправить данные в службу "миграция Azure", следуйте инструкциям независимого поставщика программного обеспечения или перемещения.

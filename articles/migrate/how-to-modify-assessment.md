# <a name="customize-an-assessment"></a>Настройка оценки

В этой статье описывается, как настроить оценки, созданные Azure Migrate Server Assessment.

Служба [Миграция Azure](migrate-services-overview.md) объединяет в себе средства наблюдения за обнаружением, оценкой и миграцией локальных приложений и рабочих нагрузок, а также виртуальных машин частного или общедоступного облака в Azure. Эта служба предоставляет инструменты для оценки и миграции, а также предложения сторонних независимых поставщиков программного обеспечения (ISV).

В рамках подготовки к миграции в Azure Можно использовать инструмент оценки azure Migrate Server Assessment для создания оценок для vmware VM и Hyper-V VM.

## <a name="about-assessments"></a>Об оценках

Существует два типа оценки, которые можно запустить с помощью средства "Оценка сервера" службы "Миграция Azure".

**Оценка** | **Подробно** | **Данные**
--- | --- | ---
**На основе производительности** | Оценки на основе собранных данных производительности | **Рекомендуемый размер VM**: На основе данных по использованию процессора и памяти.<br/><br/> **Рекомендуемый тип диска (стандартный или премиальный управляемый диск)**: На основе IOPS и пропускной их пропускной полиеговы.
**Как в локальной среде** | Оценки на основе определения размера в локальной среде. | **Рекомендуемый размер VM**: в зависимости от размера помещения VM<br/><br> **Рекомендуемый тип диска**: На основе выбранной для оценки настройки типа хранилища.


## <a name="how-is-an-assessment-done"></a>Как проводится оценка?

Оценка, проведенная в Azure Migrate Server Assessment, имеет три этапа. Оценка начинается с анализа пригодности, затем следует оценка размера и, наконец, оценка ежемесячной стоимости. Компьютер переходит на следующий этап, только если проходит предыдущий. Например, если компьютер не прошел проверку пригодности для Azure, то он помечается как непригодный для Azure, и размер и стоимость для него вычисляться не будут. [См. дополнительные сведения.](https://docs.microsoft.com/azure/migrate/concepts-assessment-calculation)

## <a name="whats-in-an-assessment"></a>Что такое оценка?

**Свойство** | **Подробно**
--- | ---
**Целевое расположение** | Расположение Azure для миграции.<br/> Оценка серверов в настоящее время поддерживает эти целевые регионы: Австралия Восток, Австралия юго-восток, Бразилия на юге, Канада Центральной, Канада Восток, Центральная Индия, Центральная ЧАСТЬ США, Китай Восток, Китай Север, Восточная Азия, Восточная США, Восток US2, Германия Центральной, Германия Северо-Восток, Япония Восток, Япония Запад, Корея Центральной, Корея на юге, Северо-Центральной ЧАСТИ США, Северной Европы, южной части Центральной США, юго-восточной Азии, южной Индии, Великобритании на юге, Великобритании На западе, США Gov Аризона, США Gov Техас, США Gov Вирджиния, Западно-Центральный США, Западная Европа, Западная Индия, Западная Сша, и Запад US2.
**Тип хранилища** | Это свойство можно использовать для указания типа дисков, на которые вы хотите переехать, в Azure.<br/><br/> Для размера помещений можно указать тип целевого хранилища либо в виде дисков премиум-управления, дисков, управляемых Standard SSD, так и дисков, управляемых HdD. Для определения размеров на основе производительности можно указать тип целевого диска как автоматические, управляемые премиум-классами диски, диски с управлением Standard HDD или диски, управляемые Стандартным SSD.<br/><br/> При указании типа хранения в качестве автоматического рекомендация диска выполняется на основе данных производительности дисков (операции ввода-вывода в секунду и пропускная способность). Если вы укажете тип хранилища как премиум/стандарт, оценка порекомендует диск SKU в выбранном типе хранилища. Если вы хотите достичь одного экземпляра VM SLA 99,9%, вы можете указать тип хранилища как диски, управляемые премиум-управлением. Это гарантирует, что для всех дисков в результате оценки рекомендуется использовать управляемые диски категории "Премиум". Azure
**Зарезервированные инстанции (RI)** | Это свойство поможет вам указать, есть ли у вас [зарезервированные экземпляры](https://azure.microsoft.com/pricing/reserved-vm-instances/) в Azure, оценки затрат в оценке затем выполняются с учетом скидок RI. Зарезервированные экземпляры в настоящее время поддерживаются только для предложения с оплатой по мере использования в службе "Миграция Azure".
**Условия определения размера** | Критерий, который будет использоваться для использования в одноразмерных ВМ для Azure. Вы можете сделать размер *на основе производительности* или размер ВМ *как на месте,* без учета истории производительности.
**Журнал производительности** | Продолжительность рассмотрения для оценки данных о производительности машин. Это свойство применимо только тогда, когда критерий размеров *основан на производительности.*
**Использование процентиля** | Значение процентиля примера производительности, которое учитывается для выбора правильного размера. Это свойство применимо только в том случае, если размер ы *иссякает на основе производительности.*
**Серия VM** |     Вы можете указать серии VM, которые вы хотели бы рассмотреть для правильного размера. Например, если у вас есть рабочая среда, которую вы не планируете переносить на виртуальные машины Azure серии А, эту серию можно исключить из списка, и выбор правильного размера будет выполняться только для указанных серий.
**Фактор комфорта** | Оценка azure Migrate Server рассматривает сяочку (фактор комфорта) при оценке. Этот буфер применяется на основе данных об использовании компьютера для виртуальных машин (показателей ЦП, памяти, диска и сети). Фактор комфорта учитывается, например, для сезонного использования и малого количества записей в журнале с потенциальным повышением в будущем.<br/><br/> Например, если виртуальная машина с 10 ядрами загружена на 20 %, обычно в результате оценки определяется виртуальная машина с двумя ядрами. Но с фактором комфорта 2.0x в результате определяется машина с 4 ядрами.
**Предлагаем** | [Предложение Azure](https://azure.microsoft.com/support/legal/offer-details/), в котором вы зарегистрированы. Служба "Миграция Azure" оценивает стоимость соответствующим образом.
**Валюты** | Валюта для выставления счетов.
**Скидка (%)** | Любая относящаяся к подписке скидка, предоставляемая в рамках предложения Azure.<br/> Значение по умолчанию — 0 %.
**Время доступности виртуальной машины** | Если виртуальные машины не должны работать в Azure круглосуточно, вы можете указать период (число дней в месяц и число часов в день), на протяжении которого они будут работать в Azure. При этом будет выполнена соответствующая оценка расходов.<br/> Значение по умолчанию — 31 день в месяц и 24 часа в день.
**Преимущество гибридного использования Azure** | Укажите, есть ли у вас гарантии программного обеспечения и вы имеете право на [получение гибридной выгоды Azure.](https://azure.microsoft.com/pricing/hybrid-use-benefit/) Если установлено значение "Да", то для виртуальных машин Windows действуют цены, отличные от цен Microsoft Azure. Значение по умолчанию — "Да".


## <a name="edit-assessment-properties"></a>Изменение свойств оценки

Для отсечения свойств оценки после создания оценки сделайте следующее:

1. В проекте Azure Migrate нажмите **«Серверы**».
2. В **Azure Migrate: Оценка серверов**щелкните количество оценок.
3. В **оценке,** нажмите соответствующую оценку > **свойствам edit**.
5. Настройте свойства оценки в соответствии с таблицей выше.
6. Нажмите кнопку **Сохранить**, чтобы обновить оценку.


Вы также можете отсеивать свойства оценки при создании оценки.

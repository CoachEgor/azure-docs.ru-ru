# <a name="customize-an-assessment"></a>Настройка оценки

В этой статье описывается, как настроить внутренние оценки, созданные оценки сервера миграции Azure.

[Служба "Миграция Azure"](migrate-services-overview.md) предоставляют центральный концентратор для отслеживания, обнаружения, оценки и миграции локальных приложений и рабочих нагрузок и закрытого и открытого облака виртуальных машин в Azure. Концентратор средства "Миграция Azure" для оценки и миграции, а также сторонних независимых Независимым производителям предложений.

Средство оценки сервера миграции Azure можно использовать для создания оценок для локальных виртуальных машин VMware, а также виртуальных машин Hyper-V, в процессе подготовки к миграции в Azure. 

## <a name="about-assessments"></a>О оценок

Существует два типа тестов, которые можно запустить с помощью оценки сервера миграции Azure.

**Оценка** | **Сведения** | **Данные**
--- | --- | ---
**На основе производительности** | Оценку с учетом собранные данные о производительности | **Рекомендуемый размер виртуальной Машины**: На основе данных об использовании ЦП и памяти.<br/><br/> **Рекомендуется использовать тип диска (standard или premium управляемого диска)** : На основе операций ввода-ВЫВОДА и пропускную способность локальными дисками.
**Как для локальной** | Оценку с учетом локального изменения размера. | **Рекомендуемый размер виртуальной Машины**: Зависимости от размера виртуальной Машины на предприятии<br/><br> **Рекомендуется использовать тип диска**: На основании параметра тип хранения, выбранная для оценки.


## <a name="how-is-an-assessment-done"></a>Том, как выполняется оценка?

Оценка службы "Миграция Azure" состоит из трех этапов. Оценка начинается с анализа пригодности, затем следует оценка размера и, наконец, оценка ежемесячной стоимости. Компьютер переходит на следующий этап, только если проходит предыдущий. Например, если компьютер не прошел проверку пригодности для Azure, то он помечается как непригодный для Azure, и размер и стоимость для него вычисляться не будут.

## <a name="whats-in-an-assessment"></a>Что такое оценка?

**Property** | **Сведения**
--- | ---
**Целевое расположение** | Расположение Azure для миграции.<br/> В настоящее время служба "Миграция Azure" поддерживает эти целевые регионы: Восточная Австралия, Юго-Восточная Австралия, Южная Бразилия, Canada Central, Восточная Канада, центральная Индия, Центральная часть США, восточная часть Китая, Северная часть Китая, Восточная Азия, восточная часть США, восточной части США 2, центральная Германия Германия, Япония, восточная, Западная Япония, центральная Корея, Юг», «Север Центральная часть США, Северная Европа, Юго-центральный регион США, Юго-Восточная Азия, Южная Индия, Южная часть соединенного Королевства, Западная часть соединенного Королевства, Правительство США — Аризона для обслуживания государственных организаций правительственные учреждения, штат Техас, США штата Вирджиния, западно-Центральная часть США, Западная Европа, Западная Индия, Западная часть США и Западная часть США 2.<br/> По умолчанию в качестве целевого региона указана западная часть США 2.
**Тип хранилища** | Стандартный HHD дисков "или" стандартный SSD дисков и "премиум".<br/> При указании типа хранения в качестве автоматическая оценка, диск рекомендация основана на данных производительности дисков (операций ввода-ВЫВОДА и пропускной способности).<br/> Если указать тип хранения как Premium или Standard, выбрать рекомендует оценки используется диск SKU в качестве типа хранилища.<br/> Если вы хотите получить один экземпляр виртуальной Машины соглашение об уровне ОБСЛУЖИВАНИЯ 99,9%, можно задать тип хранения как уровня "премиум" управляемые диски. Затем все диски в процессе оценки, рекомендуется как уровня "премиум" управляемые диски. <br/> Служба "Миграция Azure" поддерживает только управляемые диски для оценки миграции.<br/> 
**Зарезервированные экземпляры (RI)** | Укажите это свойство, если содержат зарезервированные экземпляры в Azure. Оценки затрат в процессе оценки будет учтена скидки на использование зарезервированных Экземпляров. Зарезервированные экземпляры, в настоящее время поддерживается только для оплаты по мере использования предлагает в области "Миграция Azure".
**Условия определения размера** | Используется для выбора нужного размера виртуальных машин. Изменение размера может быть на основе производительности, или **как для локальной**, без учета журнала производительности.
**Журнал производительности** | Длительность, нужно учитывать для оценки производительности виртуальной Машины. Это свойство применимо только в случае, когда изменения размера на основе производительности.
**Использование процентиля** | Значение процентиля примера производительности, который используется для правильного размера виртуальных машин. Это свойство применимо только в случае, когда изменения размера на основе производительности.
**Серия виртуальных машин** | Серия виртуальной машины VM, используемая для оценки размера. Например, если у вас есть рабочая среда, которую вы не планируете переносить на виртуальные машины Azure серии А, эту серию можно исключить из списка серий. Изменение размера выполняется только для выбранных серий.
**Фактор комфорта** | Azure оценки миграции Server учитывается буфер (фактор комфорта) во время оценки. Этот буфер применяется на основе данных об использовании компьютера для виртуальных машин (показателей ЦП, памяти, диска и сети). Фактор комфорта учитывается, например, для сезонного использования и малого количества записей в журнале с потенциальным повышением в будущем.<br/><br/> Например, если виртуальная машина с 10 ядрами загружена на 20 %, обычно в результате оценки определяется виртуальная машина с двумя ядрами. Но с фактором комфорта 2.0x в результате определяется машина с 4 ядрами.
**ПРЕДЛОЖЕНИЕ** | [Предложение Azure](https://azure.microsoft.com/support/legal/offer-details/), в котором вы зарегистрированы. Служба "Миграция Azure" оценивает стоимость соответствующим образом.
**Валюта** | Валюта для выставления счетов. 
**Скидка (%)** | Любая относящаяся к подписке скидка, предоставляемая в рамках предложения Azure.<br/> Значение по умолчанию — 0 %.
**Время доступности виртуальной машины** | Если виртуальные машины не должны работать в Azure круглосуточно, вы можете указать период (число дней в месяц и число часов в день), на протяжении которого они будут работать в Azure. При этом будет выполнена соответствующая оценка расходов.<br/> Значение по умолчанию — 31 день в месяц и 24 часа в день.
**Преимущество гибридного использования Azure** | Указывает, ли имеет software assurance и распространяется [Преимущество гибридного использования Azure](https://azure.microsoft.com/pricing/hybrid-use-benefit/). Если установлено значение "Да", то для виртуальных машин Windows действуют цены, отличные от цен Microsoft Azure. | Значение по умолчанию — "Да".


## <a name="edit-assessment-properties"></a>Изменение свойств оценки

Чтобы изменить свойства оценки после создания оценки, сделайте следующее:

1. В проекте "Миграция Azure", щелкните **серверов**.
2. В **"Миграция Azure": Оценка Server**, щелкните число оценок.
3. В **оценки**, нажмите кнопку соответствующая оценка > **изменить свойства**.
5. Настройка свойств оценки в соответствии с приведенной выше таблице.
6. Нажмите кнопку **Сохранить**, чтобы обновить оценку.


При создании оценки можно также изменить свойства оценки.

# <a name="assess-large-numbers-of-vmware-vms-for-migration-to-azure"></a>Оцените большое количество VMware VMs для миграции в Azure


В этой статье описывается, как оценить большое количество (1000-35 000) внутренних VM VMw для миграции в Azure с помощью инструмента оценки серверов Azure Migrate Server.

Служба [Миграция Azure](migrate-services-overview.md) объединяет в себе инструменты, которые используются для поиска, оценки и переноса приложений, инфраструктуры и рабочих нагрузок в Microsoft Azure. Она включает в себя инструменты Миграции Azure и предложения независимых поставщиков программного обеспечения. 

Вы узнаете, как выполнять следующие задачи:
> [!div class="checklist"]
> * План оценки в масштабе.
> * Настроят разрешения Azure и подготовьте VMware к оценке.
> * Создайте проект Azure Migrate и создайте оценку.
> * Просмотрите оценку по мере планирования миграции.


> [!NOTE]
> Если вы хотите попробовать доказательство концепции для оценки нескольких ВМ, прежде чем оценивать в масштабе, следуйте нашей [серии учебников](tutorial-prepare-vmware.md)

## <a name="plan-for-assessment"></a>План оценки

При планировании оценки большого количества VMware VMs, Есть несколько вещей, чтобы думать о:

- **Планируйте проекты Azure Migrate:** Выясните, как развернуть проекты Azure Migrate. Например, если ваши центры обработки данных находятся в разных географических регионах или вам необходимо хранить метаданные обнаружения, оценки или миграции в другой географии, возможно, потребуется несколько проектов. 
- **Приборы планирования:** Azure Migrate использует наместере прибор Azure Migrate, развернутый в качестве VMware VM, для постоянного обнаружения vMs. Прибор отслеживает изменения среды, такие как добавление виз, дисков или сетевых адаптеров. Он также отправляет метаданные и данные о производительности о них в Azure. Вам нужно выяснить, сколько приборов необходимо развернуть.
- **Планируйте учетные записи для обнаружения:** Прибор Azure Migrate использует учетную запись с доступом к vCenter Server для обнаружения вс-кодов для оценки и миграции. Если вы обнаружили более 10 000 вс-случаев, навяжните несколько учетных записей.

## <a name="plan-discovery-in-a-multi-tenant-environment"></a>Планирование открытия в мультитенантной среде

Если вы планируете мультитенантную среду, можно развернуть обнаружение на сервере vCenter Server.

- Можно настроить область обнаружения приборов в центры обработки данных vCenter Server, кластеры или папку кластеров, узлов или папок хостов или отдельных ВМ.
- Если ваша среда распределена между арендаторами и вы хотите открыть для себя каждого клиента отдельно, можно получить доступ к учетной записи vCenter, которую прибор использует для обнаружения. 
    - Возможно, вы захотите получить область применения папками VM, если арендаторы разделяют хосты. Azure Migrate не может обнаружить VMs, если учетная запись vCenter имеет доступ на уровне папок vCenter VM. Если вы хотите определить область обнаружения по папкам виртуальных машин, это можно сделать, обеспечив для учетной записи vCenter доступ только для чтения, назначенный на уровне виртуальной машины. Ознакомьтесь с [дополнительными сведениями](set-discovery-scope.md).

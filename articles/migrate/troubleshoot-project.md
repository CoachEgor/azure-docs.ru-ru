# <a name="troubleshoot-azure-migrate-projects"></a>Устранение неполадок с проектами Миграции Azure

Эта статья помогает устранить проблемы при создании и управлении проектами [Azure Migrate.](migrate-services-overview.md)

## <a name="how-to-add-new-project"></a>Как добавить новый проект?

В подписке можно иметь несколько проектов Azure Migrate. [Узнайте, как](how-to-add-tool-first-time.md) создать проект в первый раз, или [добавить дополнительные](create-manage-projects.md#create-additional-projects) проекты.

## <a name="what-azure-permissions-are-needed"></a>Какие разрешения Azure необходимы?

Для создания проекта Azure Migrate в подписке необходимы разрешения для авторов или владельцев.

## <a name="cant-find-a-project"></a>Не умуно найти проект

Поиск существующего проекта Azure Migrate зависит от того, используете ли вы текущую или старую версию Azure Migrate. Следуйте за [новостями](create-manage-projects.md#find-a-project).


## <a name="cant-find-a-geography"></a>Не умудоли

Можно создать проект Azure Migrate в поддерживаемых географических регионах для [общедоступных](migrate-support-matrix.md#supported-geographies-public-cloud) и [правительственных облаков.](migrate-support-matrix.md#supported-geographies-azure-government)

## <a name="what-are-vm-limits"></a>Что такое лимиты VM?

Вы можете оценить до 35 000 VMware VMs или до 35 000 ВМ Hyper-V в одном проекте. Проект может включать в себя как VMware VMs и Hyper-V VMs, вплоть до пределов оценки.

## <a name="can-i-upgrade-old-project"></a>Могу ли я обновить старый проект?

Проекты из предыдущей версии Azure Migrate не могут быть обновлены. Вам нужно [создать новый проект,](how-to-add-tool-first-time.md)и добавить инструменты к нему.

## <a name="cant-create-a-project"></a>Не успомяно создать проект

Если вы попытаетесь создать проект и столкнулись с ошибкой развертывания:

- Попробуйте создать проект снова в случае, если это переходная ошибка. В **развертывании**нажмите на **повторное развертывание,** чтобы попробовать еще раз.
- Проверьте, что у вас есть разрешения на участие в подписке.
- Если вы развертываете в недавно добавленной географии, подождите некоторое время и повторите попытку.
- Если вы получили ошибку, "Запросы должны содержать заголовки идентификационных данных пользователя", это может означать, что у вас нет доступа к арендатору организации Azure Active Directory (Azure AD). В данном случае:
    - При первом добавлении в состав ad-назойливого клиента вы получаете приглашение по электронной почте присоединиться к арендатору.
    - Примите приглашение, чтобы быть добавленным к арендатору.
    - Если вы не видите электронную почту, свяжитесь с пользователем с доступом к арендатору и попросите его [отправить вам приглашение.](https://docs.microsoft.com/azure/active-directory/b2b/add-users-administrator#resend-invitations-to-guest-users)
    - После получения приглашения по электронной почте откройте его и выберите ссылку, чтобы принять приглашение. Затем выйти из портала Azure и войти снова. (Обновление браузера не будет работать.) Затем можно приступить к созданию проекта миграции.

## <a name="how-do-i-delete-a-project"></a>Как удалить проект

[Следуйте этим инструкциям,](create-manage-projects.md#delete-a-project) чтобы удалить проект. Обратите внимание, что при удалении проекта удаляются как проект, так и метаданные об обнаруженных машинах в проекте.

## <a name="added-tools-dont-show"></a>Добавленные инструменты не отображены

Убедитесь, что у вас есть правильный выбранный проект. В концентраторе Azure Migrate > **серверов** или в **базах данных**нажмите на **Изменение** рядом с **проектом «Мигрит» (Изменение)** в правом верхнем углу экрана. Выберите правильную подписку и название проекта > **OK**. Страница должна обновляться с помощью дополнительных инструментов выбранного проекта.

# <a name="set-up-an-appliance-for-hyper-v-vms"></a>Настройка устройства для виртуальных машин Hyper-V

В этой статье описывается настройка устройства "Миграция Azure", если вы оценке виртуальных машин Hyper-V с помощью средства оценки сервера миграции Azure, или миграции виртуальных машин VMware в Azure с помощью средства миграции сервера миграции Azure.

Модуль виртуальной Машины Hyper-V — это простой модуль, используемый с Azure перенести оценка/миграция сервера осуществлять следующее:

- Обнаружение локальных виртуальных машин Hyper-V.
- Отправьте метаданные и данные производительности для обнаруженных виртуальных машинах в Azure перенести оценка/миграция сервера.

[Дополнительные сведения](migrate-appliance.md) о "Миграция Azure".


## <a name="appliance-deployment-steps"></a>Шаги по развертыванию устройства

Чтобы настроить модуль вы:
- Скачайте сжатый виртуального жесткого диска Hyper-V с помощью портала Azure.
- Создать модуль и проверьте, что он может подключиться к оценки сервера миграции Azure. 
- Настройка устройства в первый раз и зарегистрируйте его с проектом "Миграция Azure".

## <a name="download-the-vhd"></a>Скачайте виртуальный жесткий ДИСК

Скачайте ZIP-шаблон виртуального жесткого диска для устройства.

1. В **цели миграции** > **серверы** >  **"Миграция Azure": Оценка Server**, нажмите кнопку **Discover**.
2. В **обнаружение компьютеров** > **ваши компьютеры виртуализированы??** , нажмите кнопку **Да, с помощью Hyper-V**.
3. Нажмите кнопку **загрузить** для загрузки VHD-файл.

    ![Загрузка виртуальной Машины](./media/how-to-set-up-appliance-hyper-v/download-appliance-hyperv.png)


### <a name="verify-security"></a>Проверка безопасности

Убедитесь, что ZIP-файл, перед его развертыванием.

1. На компьютере, на который был скачан файл, откройте командное окно с правами администратора.
2. Выполните следующую команду для создания хэша для виртуального жесткого диска
    - ```C:\>CertUtil -HashFile <file_location> [Hashing Algorithm]```
    - Пример использования: ```C:\>CertUtil -HashFile C:\AzureMigrate\AzureMigrate.ova SHA256```
3.  Для версии устройства 1.19.05.10 созданный хэш должен соответствовать этим параметрам.

  **Алгоритм** | **Значение хэша**
  --- | ---
  SHA256 | 598d2e286f9c972bb7f7382885e79e768eddedfe8a3d3460d6b8a775af7d7f79


  
## <a name="create-the-appliance-vm"></a>Создание устройства виртуальной Машины

Импортируйте скачанный файл и создать виртуальную Машину.

1. Извлеките ZIP-файл виртуального жесткого диска в папку на узле Hyper-V, где будет размещаться устройства виртуальной Машины. Извлекаются три папки.
2. Откройте диспетчер Hyper-V. В **действия**, нажмите кнопку **Импорт виртуальной машины**.

    ![Развертывание виртуального жесткого диска](./media/how-to-set-up-appliance-hyper-v/deploy-vhd.png)

2. В мастере импорта виртуальной машины > **перед началом**, нажмите кнопку **Далее**.
3. В **поиск папки**, укажите папку, содержащую извлеченного виртуального жесткого диска. Затем нажмите кнопку **Далее**.
1. В **Выбор виртуальной машины**, нажмите кнопку **Далее**.
2. В **Выбор типа импорта**, нажмите кнопку **копировать виртуальную машину (создать новый уникальный идентификатор)** . Затем нажмите кнопку **Далее**.
3. В **Выбор назначения**, оставьте параметр по умолчанию. Нажмите кнопку **Далее**.
4. В **папок хранения**, оставьте параметр по умолчанию. Нажмите кнопку **Далее**.
5. В **выберите сеть**, укажите виртуальный коммутатор, который будет использовать виртуальную Машину. Параметр требуется подключение к Интернету для отправки данных в Azure.
6. В **Сводка**, проверьте параметры. Нажмите кнопку **Готово**.
7. В диспетчере Hyper-V > **виртуальных машин**, запустите виртуальную Машину.


### <a name="verify-appliance-access-to-azure"></a>Проверка доступа устройства к Azure

Убедитесь, что устройство виртуальной Машины могут подключаться к [URL-адресам Azure](migrate-support-matrix-hyper-v.md#assessment-appliance-url-access).

## <a name="configure-the-appliance"></a>Настройка устройства

Настройка устройства в первый раз.

1. В диспетчере Hyper-V > **виртуальных машин**, щелкните правой кнопкой мыши виртуальную Машину > **Connect**.
2. Укажите язык, часовой пояс и пароль для устройства.
3. Откройте браузер на любом компьютере, который может подключиться к виртуальной Машине и откройте URL-адрес веб-приложения устройства: **https://*имя устройства или IP-адрес*: 44368**.

   Кроме того можно открыть приложение с рабочего стола устройства, щелкнув ярлык приложения.
1. В веб-приложения > **Настройка необходимых компонентов**, выполните следующие действия:
    - **Лицензии**: Примите условия лицензии и прочитайте информацию от сторонних производителей.
    - **Подключение**: Приложение проверяет, что виртуальная машина имеет доступ к Интернету. Если виртуальная машина использует прокси-сервер:
        - Нажмите кнопку **параметры прокси-сервера**и укажите адрес прокси-сервера и порт прослушивания, в виде http://ProxyIPAddress или http://ProxyFQDN.
        - Укажите учетные данные, если для прокси-сервера требуется аутентификация.
        - Поддерживается только прокси-сервер HTTP.
    - **Время синхронизации**: Время проверки. Время на устройстве должен быть синхронизирован с временем в Интернете для обнаружения виртуальных Машин, для правильной работы.
    - **Установка обновлений**: Azure оценки миграции Server проверяет, что модуль имеет установлены последние обновления.

### <a name="register-the-appliance-with-azure-migrate"></a>Регистрация устройства с "Миграция Azure"

1. Нажмите кнопку **вход**. Если он не отображается, убедитесь, что вы отключили блокирование всплывающих окон в браузере.
2. На новой вкладке вход с использованием учетных данных Azure. 
    - Войдите, используя имя пользователя и пароль.
    - Войдите с помощью ПИН-кода не поддерживается.
3. После успешного входа, вернитесь к веб-приложения.
4. Выберите подписку, в которой был создан проект "Миграция Azure". Затем выберите проект.
5. Укажите имя для устройства. Имя должно быть буквенно-цифровой с 14 символов или меньше.
6. Щелкните **Зарегистрировать**.


### <a name="delegate-credentials-for-smb-vhds"></a>Делегировать учетные данные для виртуальных жестких дисков SMB

Если вы используете виртуальные жесткие диски на малого и среднего бизнеса, необходимо включить делегирование учетных данных от устройства к узлам Hyper-V. Чтобы сделать это из устройства:

1. На устройстве, виртуальной Машины выполните следующую команду. HyperVHost1/HyperVHost2 приведены примеры имен узла.

    ```
    Enable-WSManCredSSP -Role Client -DelegateComputer HyperVHost1.contoso.com HyperVHost2.contoso.com -Force
    ```

2. Кроме того для этого в редакторе локальных групповых политик на устройстве:
    - В **политика локального компьютера** > **Конфигурация компьютера**, нажмите кнопку **административные шаблоны** > **системы**  >  **Учетные данные делегирования**.
    - Дважды щелкните **разрешить передачу новых учетных данных**и выберите **включено**.
    - В **параметры**, нажмите кнопку **Показать**, и добавьте каждый узел Hyper-V, нужно выполнить поиск в список с **wsman /** префиксом.
    - В **делегирование учетных данных**, дважды щелкните **разрешить передачу новых учетных данных с проверкой подлинности сервера только NTLM**. Опять же, добавьте каждый узел Hyper-V, нужно выполнить поиск в список с **wsman /** префиксом.

## <a name="start-continuous-discovery"></a>Запуск непрерывной обнаружения

Подключитесь из устройства, для узлов Hyper-V или кластеров и запустите обнаружение виртуальных Машин.

1. В **имя пользователя** и **пароль**, укажите учетные данные, которые устройство будет использовать для обнаружения виртуальных машин. Укажите понятное имя для учетных данных, нажмите кнопку **сохранить сведения о**.
2. Нажмите кнопку **добавить узел**и укажите описание узла или кластера Hyper-V.
3. Щелкните **Проверить**. После проверки отображается количество виртуальных машин, которые могут быть обнаружены для каждого узла или кластера.
    - Если проверка не пройдена для узла, просмотрите ошибку, наведя курсор мыши на значке в **состояние** столбца. Устранение проблем и повторно выполнить проверку.
    - Чтобы удалить узлы или кластеры, выберите > **удалить**.
    - Нельзя удалить конкретного узла из кластера. Удалить можно только весь кластер.
    - Можно добавить кластер, даже если существуют проблемы с конкретных узлов в кластере.
4. После проверки нажмите кнопку **сохранение и запуск обнаружения** для запуска процесса обнаружения.

При этом запускается обнаружение. Занимает около 15 минут для метаданных обнаруженных виртуальных машин отображаются на портале Azure. 

## <a name="verify-vms-in-the-portal"></a>Проверка виртуальных машин на портале

Когда обнаружение завершится, можно проверить, что виртуальные машины отображаются на портале.

1. Откройте панель мониторинга "Миграция Azure".
2. В **"Миграция Azure" - серверы** >  **"Миграция Azure": Оценка Server** щелкните значок, который выводит это количество для **обнаруженных серверов**.

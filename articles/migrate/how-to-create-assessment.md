# <a name="create-an-assessment"></a>Создание оценки

В этой статье описывается, как создать оценку для локальных виртуальных машин VMware или виртуальных машин Hyper-V с помощью службы "миграция Azure". Оценка сервера.

Служба " [Миграция Azure](migrate-services-overview.md) " помогает выполнить миграцию в Azure. Служба "миграция Azure" предоставляет централизованный центр для наблюдения за обнаружением, оценкой и миграцией локальной инфраструктуры, приложений и данных в Azure. Центр предоставляет средства Azure для оценки и миграции, а также предложения независимых поставщиков программного обеспечения (ISV) сторонних разработчиков. 

## <a name="before-you-start"></a>Перед началом работы

- Убедитесь, что вы [создали](how-to-add-tool-first-time.md) проект "миграция Azure".
- Если вы уже создали проект, убедитесь, что вы [добавили](how-to-assess.md) службу "миграция Azure": Средство оценки серверов.
- Чтобы создать оценку, необходимо настроить устройство "миграция Azure" для [VMware](how-to-set-up-appliance-vmware.md) или [Hyper-V](how-to-set-up-appliance-hyper-v.md). Устройство обнаруживает локальные компьютеры и отправляет метаданные и данные производительности в службу "миграция Azure": Оценка сервера. [Узнайте больше](migrate-appliance.md).


## <a name="assessment-overview"></a>Общие сведения об оценке
Существует два типа оценок, которые можно создать с помощью службы "миграция Azure". Оценка сервера.

**Оценка** | **Сведения** | **Данные**
--- | --- | ---
**На основе производительности** | Оценки на основе собранных данных о производительности | **Рекомендуемый размер виртуальной машины**: На основе данных об использовании ЦП и памяти.<br/><br/> **Рекомендуемый тип диска (управляемый диск уровня "Стандартный" или "Премиум")** : На основе операций ввода-вывода и пропускной способности локальных дисков.
**Как в локальной среде** | Оценки на основе размера в локальной среде. | **Рекомендуемый размер виртуальной машины**: На основе размера локальной виртуальной машины<br/><br> **Рекомендуемый тип диска**: На основе параметра типа хранилища, выбранного для оценки.

Дополнительные [сведения](concepts-assessment-calculation.md) об оценках.

## <a name="run-an-assessment"></a>Запуск оценки

Выполните оценку следующим образом:

1. Ознакомьтесь с [рекомендациями](best-practices-assessment.md) по созданию оценок.
2. На вкладке **серверы** в **службе "миграция Azure" выполните следующие действия. Плитки Оценка** сервера щелкните **Оценка**.

    ![Оценка](./media/how-to-create-assessment/assess.png)

2. В поле **Оценка серверов**укажите имя для оценки.
3. Щелкните **Просмотреть все**, чтобы просмотреть свойства оценки.

    ![Свойства оценки](./media/how-to-create-assessment//view-all.png)

3. В окне **Выбор или создание группы**выберите **создать**и укажите имя группы. Группа собирает одну или несколько виртуальных машин вместе для оценки.
4. В списке **Добавить компьютеры в группу**выберите виртуальные машины, которые нужно добавить в группу.
5. Щелкните **создать оценку** , чтобы создать группу, и запустите оценку.

    ![Создание оценки](./media/how-to-create-assessment//assessment-create.png)

6. После создания оценки просмотрите ее в списке **серверы** > **миграция Azure:** **Оценка**серверной оценки. > 
7. Щелкните **Экспорт оценки**, чтобы скачать оценку в виде файла Excel.



## <a name="review-an-assessment"></a>Проверка оценки

Оценка описывает:

- **Готовность Azure**: Подходят ли виртуальные машины для миграции в Azure.
- **Ежемесячная оценка затрат**: Расчетные ежемесячные затраты на вычисление и хранение для запуска виртуальных машин в Azure.
- **Ежемесячная оценка стоимости хранилища**: Предполагаемые затраты на дисковое хранилище после миграции.

### <a name="view-an-assessment"></a>Просмотр оценки

1. На**серверах** **цели** >  миграции щелкните **оценки** в **службе "миграция Azure": Оценка**сервера.
2. В окне **оценки**щелкните оценку, чтобы открыть ее.

    ![Сводка по оценке](./media/how-to-create-assessment/assessment-summary.png)

### <a name="review-azure-readiness"></a>Проверка готовности Azure

1. В окне готовность к работе с **Azure**проверьте, готовы ли виртуальные машины к миграции в Azure.
2. Проверьте состояние виртуальной машины.
    - **Все готово для Azure**: Служба "миграция Azure" рекомендует размер виртуальной машины и оценки затрат для виртуальных машин в оценке.
    - **Готовность к условиям**: Выводит проблемы и предлагает предлагаемое исправление.
    - **Не готово к работе в Azure**: Выводит проблемы и предлагает предлагаемое исправление.
    - **Неизвестная готовность**: Используется, когда служба "миграция Azure" не может оценить готовность из-за проблем с доступностью данных.

2. Щелкните состояние **готовности Azure** . Вы можете просмотреть сведения о готовности виртуальной машины и перейти к просмотру сведений о виртуальной машине, включая параметры вычислений, хранения и сети.



### <a name="review-cost-details"></a>Обзор сведений о затратах

В этом представлении отображаются приблизительные затраты на вычисление и хранение для работающих виртуальных машин в Azure.

1. Ознакомьтесь с ежемесячными затратами на вычисление и хранение. Затраты суммируются для всех виртуальных машин в группе оценки.

    - Оценка затрат зависит от рекомендаций по размеру компьютера, его дисков и свойств.
    - Показаны приблизительные месячные затраты на расчеты и хранение.
    - Оценка затрат предназначена для запуска локальных виртуальных машин в качестве виртуальных машин IaaS. При оценке серверов в службе "миграция Azure" не учитывается стоимость PaaS или SaaS.

2. Вы можете просматривать ежемесячные оценки затрат на хранение. В этом представлении отображаются Объединенные затраты на хранение для группы оценки, которые разбиваются на различные типы дисков хранилища.
3. Можно выполнить детализацию, чтобы просмотреть подробные сведения о конкретных виртуальных машинах.


### <a name="review-confidence-rating"></a>Просмотр рейтинга достоверности

При выполнении оценок на основе производительности для оценки назначается оценка достоверности.

![Оценка достоверности](./media/how-to-create-assessment/confidence-rating.png)

- Выделена Оценка от 1 звезды (самая низкая) до 5 звезд (самая высокая).
- Оценка достоверности помогает оценить надежность рекомендаций по размерам, обеспечиваемым оценкой.
- Оценка достоверности основана на доступности точек данных, необходимых для расчета оценки.

Ниже приведены оценки достоверности для оценки.

**Доступность точки данных** | **Оценка достоверности**
--- | ---
0–20 % | 1 звезда
21–40 % | 2 звезды
41–60 % | 3 звезды
61–80 % | 4 звезды
81–100 % | 5 звезд

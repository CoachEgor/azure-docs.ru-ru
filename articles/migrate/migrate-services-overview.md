# <a name="about-azure-migrate"></a>Сведения о службе "Миграция Azure"

В этой статье содержится общий обзор службы "Миграция Azure".

Служба "Миграция Azure" помогает выполнить миграцию из локальной среды в Azure. "Миграция Azure" — это центральный узел, с помощью которого можно отслеживать обнаружение, оценку и перенос локальной инфраструктуры, приложений и данных в Azure. Центр обеспечивает инструменты и службы Azure для оценки и миграции, а также предложения сторонних независимых поставщиков программного обеспечения (ISV). Служба "Миграция Azure" предоставляет следующее:

- **Единая платформа для миграции**: один портал для запуска, выполнения и отслеживания хода миграции в Azure.
- **Набор инструментов**. Собственные инструменты и интеграция с другими службами Azure, а также с инструментами ISV. Выберите соответствующие инструменты оценки и миграции, основанные на требованиях организации.
- **Рабочие нагрузки**: служба "Миграция Azure" предоставляет средства оценки и миграции для следующих объектов:
    - **Серверы**: используйте инструменты Microsoft или ISV для оценки и миграции серверов на виртуальные машины Azure.
    - **Базы данных**: используйте инструменты Microsoft и ISV для оценки и миграции локальных баз данных в Базу данных SQL Azure или Управляемый экземпляр SQL Azure.
    - **Веб-приложения**: воспользуйтесь помощником Службы приложений Azure, чтобы оценить и перенести локальные веб-приложения в Службу приложений Azure.
    - **Виртуальные рабочие столы**: используйте средства независимого поставщика программного обеспечения для оценки и миграции локальной инфраструктуры виртуальных рабочих столов (VDI) в Виртуальный рабочий стол Windows в Azure.
    - **Данные**. Используйте семейство продуктов Azure Data Box для быстрого и экономичного перемещения данных в Azure.

## <a name="azure-migrate-versions"></a>Версии службы "Миграция Azure"

Сейчас существует две версии службы "Миграция Azure":

- **Текущая версия**. С помощью этой версии службы можно создавать проекты службы "Миграция Azure", обнаруживать локальные компьютеры и выполнять оркестрацию оценок и миграций. [Дополнительные сведения](whats-new.md) о новых возможностях этой версии.
- **Предыдущая версия**. Если вы использовали предыдущую версию службы "Миграция Azure" (поддерживалась только оценка локальных виртуальных машин VMware), теперь следует использовать текущую версию. Вы больше не можете создавать проекты службы "Миграция Azure", используя предыдущие версии, поэтому мы рекомендуем не выполнять новые операции обнаружения. Чтобы получить доступ к существующим проектам, на портале Azure > **Все службы**, найдите службу **Миграция Azure**. На панели мониторинга службы "Миграция Azure" есть уведомление и ссылка для доступа к старым проектам службы.

### <a name="selecting-an-isv-tool"></a>Выбор инструментов независимого поставщика программного обеспечения

Вы определяете необходимый инструмент и добавляете его в проект службы "Миграция Azure".

- После добавления инструмента ISV начните работу с инструментом, получив лицензию или подписавшись на бесплатную пробную версию в соответствии с политикой ISV. Лицензирование инструментов ISV в соответствии с моделью лицензирования ISV.
- В каждом инструменте есть возможность подключиться к службе "Миграция Azure". Выполните приведенные инструкции и рекомендации в документации по инструменту, чтобы подключить инструмент с помощью службы "Миграция Azure".
- Вы можете централизованно отслеживать процесс миграции из проекта службы "Миграция Azure", используя инструменты Azure и независимых поставщиков программного обеспечения.


## <a name="azure-migrate-server-assessment-tool"></a>Средство оценки серверов службы "Миграция Azure"

Миграция Azure С помощью средства оценки серверов можно обнаруживать и оценивать локальные виртуальные машины VMware и Hyper-V, а также физические серверы на предмет миграции в Azure. Он поможет определить следующее:

- **Готовность к работе в Azure.** Оценивает готовность локальных компьютеров к миграции в Azure.
- **Изменение размера Azure.** Предполагаемый размер виртуальных машин Azure после миграции.
- **Оценка затрат в Azure.** . Предполагаемые затраты на работу локальных серверов в Azure.
- **Визуализация зависимостей.**  Межсерверные зависимости (если включена визуализация зависимостей) и оптимальные способы перемещения зависимых серверов в Azure.

При выполнении оценки сервера используется упрощенное устройство, которое развертывается локально и регистрируется с помощью оценки сервера.

- Устройство обнаруживает локальные компьютеры.
- Оно подключается к средству оценки серверов и непрерывно отправляет метаданные компьютеров и данные о производительности в службу "Миграция Azure".
- Обнаружение устройства осуществляется без агента. На обнаруженных компьютерах не нужно ничего устанавливать.
- После обнаружения необходимо объединить обнаруженные компьютеры в группы. Обычно нужно объединять компьютеры, которые необходимо перенести вместе.
- Создайте оценку для группы. Затем вы проанализируете оценку, чтобы узнать свою стратегию миграции.

## <a name="azure-migrate-server-migration-tool"></a>Средство миграции серверов Миграции Azure

Миграция Azure С помощью средства миграции серверов можно перенести локальные виртуальные машины VMware и Hyper-V в Azure, а также физические серверы, другие виртуальные машины и частные облачные виртуальные машины. Вы можете перенести машины после выполнения оценки или без нее.


## <a name="database-assessment"></a>Оценка базы данных

Служба "Миграция Azure" интегрируется с Помощником по миграции данных (Майкрософт) (DMA), чтобы оценивать локальные базы данных SQL Server на предмет миграции в Базу данных SQL Azure, Управляемый экземпляр SQL Azure или виртуальные машины Azure, работающие на SQL Server. DMA предоставляет информацию о потенциальных проблемах, связанных с блокировкой, для миграции. Он определяет неподдерживаемые и новые функции, которые можно использовать после миграции, а также помогает определить правильный путь для миграции базы данных. [Узнайте больше](https://docs.microsoft.com/sql/dma/dma-overview?view=sql-server-2017).

## <a name="database-migration"></a>Перенос баз данных

Служба "Миграция Azure" интегрируется со службой миграции баз данных Azure (DMS) для переноса локальных баз данных в Azure. Используйте DMS для переноса локальных баз данных на виртуальные машины Azure, на которых запущены SQL, БД SQL Azure и управляемые экземпляры SQL Azure. [Узнайте больше](https://docs.microsoft.com/azure/dms/dms-overview).

## <a name="web-app-migration"></a>Перенос веб-приложений

Служба "Миграция Azure" интегрируется с Помощником по миграции Службы приложений Azure. С помощью Центра миграции Azure можно оценивать и переносить локальные веб-приложения в Azure с использованием Помощника, как показано ниже.

- **Оценка подключенных веб-приложений**: Воспользуйтесь помощником по миграции службы приложений Azure, чтобы оценить локальные веб-сайты на предмет миграции в службу приложений Azure.
- **Миграция веб-приложений**: Перенос веб-приложений .NET и PHP в Azure с помощью помощника по миграции службы приложений Azure.

[Узнайте больше о Помощнике.](https://appmigration.microsoft.com/)

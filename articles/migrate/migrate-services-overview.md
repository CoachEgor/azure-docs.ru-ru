# <a name="about-azure-migrate"></a>Сведения о службе "Миграция Azure"

В этой статье содержится общий обзор службы "Миграция Azure".

Используйте службу "Миграция Azure" для миграции в Azure. Миграция Azure предоставляет центральный узел для оценки и переноса локальной инфраструктуры, приложений и данных в Azure с использованием следующих компонентов.

- **Унифицированная платформа миграции**: один портал для запуска, запуска и мониторинга перехода в Azure.
- **Диапазон средств**: набор средств для оценки и миграции. В состав центра входит служба "миграция Azure": Оценка серверов и миграция Azure: миграция сервера. Эта служба интегрируется с другими службами Azure, а также с инструментами независимых поставщиков программного обеспечения (ISV).
- **Оценка и миграция**. в центре миграции Azure можно оценивать и выполнять миграцию:
    - **Серверы**. Оцените и перенесите локальные серверы на виртуальные машины Azure.
    - **Базы данных**: Оценка и миграция локальных баз данных в базу данных SQL Azure или в azure SQL управляемый экземпляр.
    - **Веб-приложения**. Оцените и перенесите локальные веб-приложения в службу приложений Azure с помощью помощника по службе приложений Azure.
    - **Виртуальные рабочие столы**. Оцените и перенесите локальную инфраструктуру виртуальных рабочих столов (VDI) на виртуальный рабочий стол Windows в Azure.
    - **Данные**: вы можете быстро и экономично переносить большие объемы данных в Azure с помощью Azure Data Box продуктов.

## <a name="azure-migrate-server-assessment-tool"></a>Служба "миграция Azure": средство оценки серверов

Инструмент Azure Migration: средство оценки серверов обнаруживает и оценивает локальные виртуальные машины VMware, виртуальные машины Hyper-V и физические серверы для миграции в Azure. Он поможет определить следующее:

- **Готовность Azure:** Оцените, готовы ли локальные компьютеры к миграции в Azure.
- **Размер Azure:** Предполагаемый размер виртуальных машин Azure после миграции.
- **Оценка стоимости Azure:** Предполагаемые затраты на выполнение локальных серверов в Azure.
- **Визуализация зависимостей:** Если вы используете оценку серверов с визуализацией зависимостей, можно эффективно выработать межсерверные зависимости и оптимальные способы перемещения зависимых серверов в Azure.

При выполнении оценки сервера используется упрощенное устройство, которое развертывается локально и регистрируется с помощью оценки сервера.

- Это устройство работает на физическом сервере или виртуальной машине и легко устанавливается из скачиваемого шаблона.
- Устройство обнаруживает локальные компьютеры и непрерывно отправляет метаданные компьютера и сведения о производительности в Миграцию Azure.
- Обнаружение устройства осуществляется без агента. На обнаруженных компьютерах не нужно ничего устанавливать.
- После обнаружения необходимо объединить обнаруженные компьютеры в группы и оценить готовность этих групп к миграции.


## <a name="azure-migrate-server-migration-tool"></a>Миграция Azure: средство миграции сервера

Средство миграции Azure Migration Tool позволяет переносить локальные виртуальные машины VMware, виртуальные машины Hyper-V, физические серверы, другие виртуализированные компьютеры и виртуальные машины общедоступных облаков в Azure. Вы можете выполнить миграцию компьютеров после оценки или без нее.


## <a name="select-a-tool"></a>Выбор средства

В центре миграции Azure выберите инструмент, который вы хотите использовать для оценки, и добавьте его в проект Миграции Azure. Если вы добавляете инструмент независимого поставщика программного обеспечения или Movere, сделайте следующее.

- Для начала получите лицензию или бесплатную пробную версию в соответствии с приведенными инструкциями. Лицензирование средств контролируется независимым поставщиком программного обеспечения или самим средством. 
- В каждом инструменте есть возможность подключиться к службе "Миграция Azure". Следуйте инструкциям по подключению.
- Отслеживайте процесс миграции из проекта службы "Миграция Azure", единообразно для всех средств.


## <a name="movere"></a>Movere

Movere — это платформа SaaS, которая расширяет возможности бизнес-аналитики благодаря точному описанию всей ИТ-среды за один день. По мере роста, изменения и цифровой оптимизации организаций это решение обеспечит надежный визуальный контроль и управляемость локальной среды независимо от используемых платформ, приложений и расположений. Платформа Movere [была приобретена](https://azure.microsoft.com/blog/microsoft-acquires-movere-to-help-customers-unlock-cloud-innovation-with-seamless-migration-tools/) корпорацией Майкрософт и больше не предоставляется как отдельное предложение.  Платформа Movere доступна в рамках программ оценки решений Майкрософт и облачной экономики. [Дополнительные сведения](https://www.movere.io) о Movere. Если у вас есть вопросы, задайте их по адресу movereq@microsoft.com или обратитесь к представителю Майкрософт.

Кроме того, рекомендуем ознакомиться с нашей встроенной службой "Миграция Azure". "Миграция Azure" — это централизованное решение для упрощения миграции в облако. Решение предоставляет широкую поддержку разных рабочих нагрузок, включая физические и виртуальные серверы, базы данных и приложения. Полный визуальный контроль позволяет без труда отслеживать процессы обнаружения, оценки и миграции. Благодаря встроенным средствам Azure и партнерских независимых поставщиков программного обеспечения в службе "Миграция Azure" реализован широкий спектр возможностей. К ним относятся функции для обнаружения виртуальных и физических серверов, масштабирования на основе производительности, планирования затрат, оценки на основе импорта и анализа зависимостей приложений без агента. Если вам нужна поддержка специалиста, чтобы приступить к работе, корпорация Майкрософт рекомендует опытных [поставщиков управляемых служб Azure](https://azure.microsoft.com/partners), которые помогут вам. Ознакомьтесь с [веб-сайтом службы "Миграция Azure"](https://azure.microsoft.com/services/azure-migrate/). 
 

## <a name="azure-migrate-versions"></a>Версии службы "Миграция Azure"

Существует две версии службы "Миграция Azure".

- **Текущая версия**: Используйте эту версию для создания проектов службы "миграция Azure", обнаружения локальных компьютеров и управления оценками и миграциями. [Дополнительные сведения](whats-new.md) о новых возможностях этой версии.
- **Предыдущая версия**. Если вы использовали предыдущую версию службы "миграция Azure" (поддерживается только оценка локальных виртуальных машин VMware), теперь следует использовать текущую версию. Вы больше не можете создавать проекты службы "Миграция Azure", используя предыдущие версии, поэтому мы рекомендуем не выполнять новые операции обнаружения. Чтобы получить доступ к имеющимся проектам, на портале Azure найдите и выберите службу **Миграция Azure**. На панели мониторинга службы **Миграция Azure** есть уведомление и ссылка для доступа к старым проектам службы.

# <a name="about-azure-migrate"></a>Сведения о службе "Миграция Azure"

В этой статье содержится общий обзор службы "Миграция Azure".

Миграция Azure — это центральный узел для оценки и переноса серверов, инфраструктуры, приложений и данных из локальной среды в Azure. Миграция Azure предоставляет приведенные ниже возможности.

- **Единая платформа для миграции**: один портал для запуска, выполнения и отслеживания хода миграции в Azure.
- **Набор инструментов**. Набор инструментов для оценки и миграции. К этим инструментам относятся "Миграция Azure: оценка сервера" и "Миграция сервера" Миграции Azure. Миграция Azure интегрируется с другими службами Azure, а также с инструментами независимых поставщиков программного обеспечения.
- **Оценка и миграция**. С помощью Центра миграции Azure можно оценивать и переносить следующие компоненты.
    - **Серверы**: Оценка локальных серверов и миграция на виртуальные машины Azure.
    - **Базы данных**: Оценка локальных баз данных и миграция в Базу данных SQL Azure или на в Управляемый экземпляр Базы данных SQL Azure.
    - **Веб-приложения**: Оценка локальных веб-приложений и миграция в Службу приложений Azure с применением помощника Службы приложений Azure.
    - **Виртуальные рабочие столы**: оценка локальной инфраструктуры виртуальных рабочих столов (VDI) и ее перенос в Виртуальный рабочий стол Windows в Azure.
    - **Data**: Быстрая и экономичная миграция больших объемов данных в Azure с помощью продуктов Azure Data Box.

## <a name="azure-migrate-server-assessment-tool"></a>Средство оценки серверов службы "Миграция Azure"

С помощью средства оценки серверов службы "Миграция Azure" можно обнаруживать локальные виртуальные машины VMware и Hyper-V, а также физические серверы, и оценивать их готовность к миграции в Azure. Далее приводится назначение этого средства.

- **Готовность к работе в Azure.** Оценивает готовность локальных компьютеров к миграции в Azure.
- **Изменение размера Azure.** Оценивает размер виртуальных машин Azure после миграции.
- **Оценка затрат в Azure.** . Предполагаемые затраты на работу локальных серверов в Azure.
- **Анализ зависимостей**. Если вы используете средство "Оценка сервера" с [анализом зависимостей](concepts-dependency-visualization.md), вы сможете эффективно выявить межсерверные зависимости и оптимальные стратегии перемещения взаимозависимых серверов в Azure.


Средство оценки серверов использует упрощенный [модуль "Миграция Azure"](migrate-appliance.md), который развертывается в локальной среде.

- Этот модуль выполняется на виртуальной машине или на физическом сервере. Вы можете легко установить его с помощью скачиваемого шаблона.
- Модуль обнаруживает локальные компьютеры и непрерывно отправляет метаданные и сведения о производительности этих компьютеров в Миграцию Azure.
- Обнаружение устройства осуществляется без агента. На обнаруженных компьютерах не нужно ничего устанавливать.
- Когда модуль выполнит обнаружение, вы можете объединить обнаруженные компьютеры в группы и запустить оценку для любой группы.

## <a name="azure-migrate-server-migration-tool"></a>Средство миграции серверов Миграции Azure

С помощью средства миграции серверов службы "Миграция Azure" можно перенести в Azure локальные виртуальные машины VMware и Hyper-V, а также физические серверы, виртуальные машины на других платформах и в частных облаках. Вы можете выполнить миграцию компьютеров после оценки или без нее. 

Для миграции виртуальных машин VMware и Hyper-V без применения агентов средство миграции серверов применяет модуль службы "Миграция Azure", развернутый в локальной среде. Этот же модуль применяется для оценки серверов, как описано в предыдущем разделе.


## <a name="selecting-assessmentmigration-tools"></a>Выбор средств для оценки и миграции

В центре Миграции Azure выберите средство, которое вы хотите использовать для оценки или миграции, и добавьте его в проект Миграции Azure. Если вы добавляете инструмент независимого поставщика программного обеспечения или Movere, сделайте следующее.

- Для начала получите лицензию или зарегистрируйтесь для бесплатной пробной версии в соответствии с инструкциями к выбранному средству. Лицензирование средств контролируется независимым поставщиком программного обеспечения или самим средством. 
- В каждом инструменте есть возможность подключиться к службе "Миграция Azure". Следуйте инструкциям по подключению выбранного средства.
- Процесс миграции отслеживается для всех средств в проекте службы "Миграция Azure".


## <a name="movere"></a>Movere

Movere — это платформа SaaS, которая расширяет возможности бизнес-аналитики благодаря точному описанию всей ИТ-среды за один день. По мере роста, изменения и цифровой оптимизации организаций это решение обеспечит надежный визуальный контроль и управляемость локальной среды независимо от используемых платформ, приложений и расположений. Платформа Movere [была приобретена](https://azure.microsoft.com/blog/microsoft-acquires-movere-to-help-customers-unlock-cloud-innovation-with-seamless-migration-tools/) корпорацией Майкрософт и больше не предоставляется как отдельное предложение.  Платформа Movere доступна в рамках программ оценки решений Майкрософт и облачной экономики. [Дополнительные сведения](https://www.movere.io) о Movere. 

Кроме того, рекомендуем ознакомиться с нашей встроенной службой "Миграция Azure". "Миграция Azure" — это централизованное решение для упрощения миграции в облако. Решение предоставляет широкую поддержку разных рабочих нагрузок, включая физические и виртуальные серверы, базы данных и приложения. Полный визуальный контроль позволяет без труда отслеживать процессы обнаружения, оценки и миграции. Благодаря встроенным средствам Azure и партнерских независимых поставщиков программного обеспечения в службе "Миграция Azure" реализован широкий спектр возможностей. К ним относятся функции для обнаружения виртуальных и физических серверов, масштабирования на основе производительности, планирования затрат, оценки на основе импорта и анализа зависимостей приложений без агента. Если вам нужна поддержка специалиста, чтобы приступить к работе, корпорация Майкрософт рекомендует опытных [поставщиков управляемых служб Azure](https://azure.microsoft.com/partners), которые помогут вам. Ознакомьтесь с [веб-сайтом службы "Миграция Azure"](https://azure.microsoft.com/services/azure-migrate/). 
 

## <a name="azure-migrate-versions"></a>Версии службы "Миграция Azure"

Существует две версии службы "Миграция Azure".

- **Текущая версия**. С помощью этой версии службы можно создавать проекты службы "Миграция Azure", обнаруживать локальные компьютеры и выполнять оркестрацию оценок и миграций. [Дополнительные сведения](whats-new.md) о новых возможностях этой версии.
- **Предыдущая версия**. Если вы использовали предыдущую версию службы "Миграция Azure" (поддерживалась только оценка локальных виртуальных машин VMware), теперь следует использовать текущую версию. Вы больше не можете создавать проекты службы "Миграция Azure", используя предыдущие версии, поэтому мы рекомендуем не выполнять новые операции обнаружения. Чтобы получить доступ к имеющимся проектам, на портале Azure найдите и выберите службу **Миграция Azure**. На панели мониторинга службы **Миграция Azure** есть уведомление и ссылка для доступа к старым проектам службы.

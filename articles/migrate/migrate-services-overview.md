# <a name="about-azure-migrate"></a>Сведения о службе "Миграция Azure"

В этой статье содержится общий обзор службы "Миграция Azure".

Служба "Миграция Azure" помогает перенести рабочие нагрузки и данные предприятия из локальной среды в Azure. "Миграция Azure" — это центральный узел, с помощью которого можно отслеживать обнаружение, оценку и перенос локальной инфраструктуры, приложений и данных в Azure.  Служба "Миграция Azure" предоставляет следующее:

- **Единая платформа для миграции**: один портал для запуска, выполнения и отслеживания хода миграции в Azure.
- **Набор инструментов**. Этот центр предоставляет инструменты Миграции Azure для оценки и миграции, а также поддерживает интеграцию с другими службами Azure, средствами и предложениями независимых поставщиков программного обеспечения.
- **Рабочие нагрузки**: служба "Миграция Azure" выполняет оценку и миграцию для следующих объектов:
    - **Серверы**: С помощью функций оценки и миграции службы "Миграция Azure", а также других средств можно определить готовность серверов к переходу и перенести их виртуальные машины Azure.
    - **Базы данных**: С использованием инструментов Майкрософт и независимых поставщиков программного обеспечения можно оценивать базы данных и переносить их в Базу данных SQL Azure или Управляемый экземпляр SQL Azure.
    - **Веб-приложения**: воспользуйтесь помощником Службы приложений Azure, чтобы оценить и перенести локальные веб-приложения в Службу приложений Azure.
    - **Виртуальные рабочие столы**: используйте средства независимого поставщика программного обеспечения для оценки и миграции локальной инфраструктуры виртуальных рабочих столов (VDI) в Виртуальный рабочий стол Windows в Azure.
    - **Данные**. Семейство продуктов Azure Data Box позволяет быстро и экономично перемещать большие объемы данных в Azure.

## <a name="azure-migrate-versions"></a>Версии службы "Миграция Azure"

Сейчас существует две версии службы "Миграция Azure":

- **Текущая версия**. С помощью этой версии службы можно создавать проекты службы "Миграция Azure", обнаруживать локальные компьютеры и выполнять оркестрацию оценок и миграций. [Дополнительные сведения](whats-new.md) о новых возможностях этой версии.
- **Предыдущая версия**. Если вы использовали предыдущую версию службы "Миграция Azure" (поддерживалась только оценка локальных виртуальных машин VMware), теперь следует использовать текущую версию. Вы больше не можете создавать проекты службы "Миграция Azure", используя предыдущие версии, поэтому мы рекомендуем не выполнять новые операции обнаружения. Чтобы получить доступ к имеющимся проектам, на портале Azure найдите и выберите службу **Миграция Azure**. На панели мониторинга службы **Миграция Azure** есть уведомление и ссылка для доступа к старым проектам службы.

### <a name="selecting-a-tool"></a>Выбор инструмента

Выберите нужный инструмент и добавьте его в проект службы "Миграция Azure".

- Если вы добавляете инструмент независимого поставщика программного обеспечения или Movere, сделайте следующее:
    - Сначала получите лицензию или подписку на бесплатную пробную версию в соответствии с политикой использования инструмента. Лицензирование инструментов выполняется в соответствии с моделью лицензирования независимого поставщика программного обеспечения или конкретного инструмента.
    - В каждом инструменте есть возможность подключиться к службе "Миграция Azure". Выполните приведенные инструкции и документацию по инструменту, чтобы подключить инструмент с помощью службы "Миграция Azure".
- Вы можете централизованно отслеживать процесс миграции из проекта службы "Миграция Azure" для инструментов Azure и других инструментов.



## <a name="azure-migrate-server-assessment-tool"></a>Средство оценки серверов службы "Миграция Azure"

Миграция Azure С помощью средства оценки серверов можно обнаруживать и оценивать локальные виртуальные машины VMware и Hyper-V, а также физические серверы на предмет миграции в Azure. Он поможет определить следующее:

- **Готовность к работе в Azure.** Оценивает готовность локальных компьютеров к миграции в Azure.
- **Изменение размера Azure.** Предполагаемый размер виртуальных машин Azure после миграции.
- **Оценка затрат в Azure.** . Предполагаемые затраты на работу локальных серверов в Azure.
- **Визуализация зависимостей.**  Межсерверные зависимости (если включена визуализация зависимостей) и оптимальные способы перемещения зависимых серверов в Azure.

При выполнении оценки сервера используется упрощенное устройство, которое развертывается локально и регистрируется с помощью оценки сервера.

- Устройство обнаруживает локальные компьютеры.
- Оно подключается к средству оценки серверов и непрерывно отправляет метаданные компьютеров и данные о производительности в службу "Миграция Azure".
- Обнаружение устройства осуществляется без агента. На обнаруженных компьютерах не нужно ничего устанавливать.
- После обнаружения необходимо объединить обнаруженные компьютеры в группы. Обычно нужно объединять компьютеры, которые необходимо перенести вместе.
- Создайте оценку для группы. Затем вы проанализируете оценку, чтобы узнать свою стратегию миграции.

## <a name="azure-migrate-server-migration-tool"></a>Средство миграции серверов Миграции Azure

Миграция Azure С помощью средства миграции серверов можно перенести локальные виртуальные машины VMware и Hyper-V в Azure, а также физические серверы, другие виртуальные машины и частные облачные виртуальные машины. Вы можете перенести машины после выполнения оценки или без нее.


## <a name="database-migration-assistant"></a>Помощник по переносу баз данных

Служба "Миграция Azure" интегрируется с Помощником по миграции данных (Майкрософт) (DMA), чтобы оценивать локальные базы данных SQL Server на предмет миграции в Базу данных SQL Azure, Управляемый экземпляр SQL Azure или виртуальные машины Azure, работающие на SQL Server. DMA предоставляет информацию о потенциальных проблемах, связанных с блокировкой, для миграции. Он определяет неподдерживаемые и новые функции, которые можно использовать после миграции, а также помогает определить правильный путь для миграции базы данных. [Узнайте больше](https://docs.microsoft.com/sql/dma/dma-overview?view=sql-server-2017).

## <a name="database-migration-service"></a>Database Migration Service

Служба "Миграция Azure" интегрируется со службой миграции баз данных Azure (DMS) для переноса локальных баз данных в Azure. Используйте DMS для переноса локальных баз данных на виртуальные машины Azure, на которых запущены SQL, БД SQL Azure и управляемые экземпляры SQL Azure. [Узнайте больше](https://docs.microsoft.com/azure/dms/dms-overview).

## <a name="movere"></a>Movere

 
Movere — это платформа SaaS, которая расширяет возможности бизнес-аналитики благодаря точному описанию всей ИТ-среды за один день. По мере роста, изменения и цифровой оптимизации организаций это решение обеспечит надежный визуальный контроль и управляемость локальной среды независимо от используемых платформ, приложений и расположений. Платформа Movere [была приобретена](https://azure.microsoft.com/blog/microsoft-acquires-movere-to-help-customers-unlock-cloud-innovation-with-seamless-migration-tools/) корпорацией Майкрософт и больше не предоставляется как отдельное предложение.  Платформа Movere доступна в рамках программ оценки решений Майкрософт и облачной экономики. [Дополнительные сведения](https://www.movere.io) о Movere. Если у вас есть вопросы, задайте их по адресу movereq@microsoft.com или обратитесь к представителю Майкрософт.

Кроме того, рекомендуем ознакомиться с нашей встроенной службой "Миграция Azure". "Миграция Azure" — это централизованное решение для упрощения миграции в облако. Решение предоставляет широкую поддержку разных рабочих нагрузок, включая физические и виртуальные серверы, базы данных и приложения. Полный визуальный контроль позволяет без труда отслеживать процессы обнаружения, оценки и миграции. Благодаря встроенным средствам Azure и партнерских независимых поставщиков программного обеспечения в службе "Миграция Azure" реализован широкий спектр возможностей. К ним относятся функции для обнаружения виртуальных и физических серверов, масштабирования на основе производительности, планирования затрат, оценки на основе импорта и анализа зависимостей приложений без агента. Если вам нужна поддержка специалиста, чтобы приступить к работе, корпорация Майкрософт рекомендует опытных [поставщиков управляемых служб Azure](https://azure.microsoft.com/partners), которые помогут вам. Ознакомьтесь с [веб-сайтом службы "Миграция Azure"](https://azure.microsoft.com/services/azure-migrate/).

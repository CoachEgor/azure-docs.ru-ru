# <a name="support-matrix-for-hyper-v-assessment-and-migration"></a>Таблица поддержки для оценки и переноса виртуальных машин Hyper-V

Службу "миграция [Azure](migrate-overview.md) " можно использовать для оценки и миграции компьютеров в облако Microsoft Azure. В этой статье перечислены параметры поддержки и ограничения для оценки и миграции локальных виртуальных машин Hyper-V.



## <a name="hyper-v-scenarios"></a>Сценарии Hyper-V

В таблице приведены поддерживаемые сценарии для виртуальных машин Hyper-V.

**Развертывание** | **Дополнительно***
--- | ---
**Оценка локальных виртуальных машин Hyper-V** | [Настройте](tutorial-prepare-hyper-v.md) первую оценку.<br/><br/> [Выполните](scale-hyper-v-assessment.md) крупномасштабную оценку.
**Перенос виртуальных машин Hyper-V в Azure.** | [Попробуйте выполнить](tutorial-migrate-hyper-v.md) миграцию в Azure.

География | Проекты службы "миграция Azure" можно создавать в нескольких географических регионах. ## <a name="assessment-hyper-v-host-requirements"></a>Оценка — требования к узлу Hyper-V

| **Поддержка**                | **Сведения**               
| :-------------------       | :------------------- |
| **Развертывание узла**       | Узел Hyper-V может быть автономным или развернутым в кластере. |
| **Разрешения**           | Вам потребуются права администратора на узле Hyper-V. |
| **Операционная система узла** | Windows Server 2016 или Windows Server 2012 R2.<br/> Вы не можете оценивать виртуальные машины, расположенные на узлах Hyper-V под управлением Windows Server 2019. |
| **Удаленное взаимодействие PowerShell**   | Должен быть включен на каждом узле. |
| **Реплика Hyper-V**       | Если вы используете реплику Hyper-V (или несколько виртуальных машин с одинаковыми идентификаторами виртуальных машин) и обнаружите как исходные, так и реплицированные виртуальные машины с помощью службы "миграция Azure", оценка, созданная функцией "миграция Azure", может быть неточной. |


## <a name="assessment-hyper-v-vm-requirements"></a>Оценка — требования к виртуальным машинам Hyper-V

| **Поддержка**                  | **Сведения**               
| :----------------------------- | :------------------- |
| **Операционная система** | Все операционные системы [Windows](https://support.microsoft.com/help/2721672/microsoft-server-software-support-for-microsoft-azure-virtual-machines) и [Linux](https://docs.microsoft.com/azure/virtual-machines/linux/endorsed-distros) , поддерживаемые Azure. |
| **Разрешения**           | Для каждой виртуальной машины Hyper-V, которую вы хотите оценить, требуются права администратора. |
| **Integration Services**       | Для записи сведений об операционной системе на виртуальных машинах, которые вы оцениваете, необходимо запустить [Hyper-V Integration Services](https://docs.microsoft.com/virtualization/hyper-v-on-windows/reference/integration-services) . |
| **Загрузка UEFI**                  | Виртуальные машины с загрузкой UEFI не поддерживаются для миграции. |
| **Зашифрованные диски и тома**    | Виртуальные машины с зашифрованными дисками или томами не поддерживаются для миграции. |
| **RDM/транзитные диски**      | Если у виртуальных машин есть RDM или транзитные диски, эти диски не будут реплицированы в Azure. |
| **NFS**                        | Тома NFS, подключенные к виртуальным машинам как тома, не реплицируются. |
| **Целевой диск**                | Оценки службы "миграция Azure" рекомендуют выполнять миграцию на виртуальные машины Azure только с управляемыми дисками. |


## <a name="assessment-appliance-requirements"></a>Оценка — требования к устройству

Для оценки служба "миграция Azure" запускает упрощенное устройство для обнаружения виртуальных машин Hyper-V, а затем отправляет метаданные и данные производительности виртуальной машины в службу "миграция Azure". Устройство работает на виртуальной машине Hyper-V и настроено с помощью сжатого виртуального жесткого диска Hyper-V, скачанного из портал Azure. В следующей таблице перечислены требования к устройству.

| **Поддержка**                | **Сведения**               
| :-------------------       | :------------------- |
| **Проект службы "миграция Azure"**  |  Устройство может быть связано с одним проектом.<br/> Вы можете обнаружить до 5000 виртуальных машин Hyper-V с одним устройством.
| **Hyper-V**    |  Вы развертываете устройство как виртуальную машину Hyper-V.<br/> Указанная виртуальная машина устройства является виртуальной машиной Hyper-V версии 5,0.<br/> Узел виртуальной машины должен работать под Windows Server 2012 R2 или более поздней версии.<br/> Ему требуется достаточно места для выделения 16 ГБ ОЗУ, 8 виртуальных ЦП и 1 внешнего коммутатора для виртуальной машины устройства.<br/> Для устройства требуется статический или динамический IP-адрес, а также доступ к Интернету.


## <a name="assessment-appliance-url-access"></a>Оценка — доступ по URL-адресу устройства

Чтобы оценить виртуальные машины, устройству миграции Azure требуется подключение к Интернету.

- При развертывании устройства служба "миграция Azure" выполняет проверку подключения к URL-адресам, приведенным в таблице ниже.
- Если вы используете прокси-сервер на основе URL-адресов, разрешите доступ к URL-адресам в таблице, убедившись, что прокси-сервер разрешает все записи CNAME, полученные при поиске URL-адресов.
- Если у вас есть перехваченный прокси-сервер, может потребоваться импортировать сертификат сервера с прокси-сервера на устройство.


**URL-адрес** | **Сведения**  
--- | ---
*.portal.azure.com | Переход к портал Azure
*.windows.net | Войдите в подписку Azure
*.microsoftonline.com | Создание Azure Active Directory приложений для взаимодействия устройств между службами.
management.azure.com | Создание Azure Active Directory приложений для взаимодействия устройств между службами.
dc.services.visualstudio.com | Ведение журналов и мониторинг
*.vault.azure.net | Управление секретами в Azure Key Vault при обмене данными между устройством и службой.
aka.ms/* | Разрешение доступа к связям.
https://download.microsoft.com/download/* | Позволяет загружать файлы с веб-сайта центра загрузки Майкрософт.



## <a name="assessment-port-requirements"></a>Оценка — требования к портам

В следующей таблице перечислены требования к портам для оценки.

**Устройство** | **Connection**
--- | ---
**Appliance** | Входящие подключения через TCP-порт 3389 для разрешения подключения удаленного рабочего стола к устройству.<br/> Входящие подключения через порт 44368 для удаленного доступа к приложению управления устройством с помощью URL-адреса:``` https://<appliance-ip-or-name>:44368 ```<br/> Исходящие подключения через порт 443 для отправки метаданных обнаружения и производительности в службу "миграция Azure".
**Узел или кластер Hyper-V** | Входящие подключения через порты WinRM 5985 (HTTP) и 5986 (HTTPS) для извлечения метаданных конфигурации и производительности виртуальных машин Hyper-V с помощью сеанса модель CIM (CIM).

## <a name="migration-hyper-v-host-requirements"></a>Миграция — требования к узлу Hyper-V

| **Поддержка**                | **Сведения**               
| :-------------------       | :------------------- |
| **Развертывание узла**       | Узел Hyper-V может быть автономным или развернутым в кластере. |

## <a name="migration-hyper-v-vm-requirements"></a>Миграция — требования к виртуальным машинам Hyper-V

| **Поддержка**                  | **Сведения**               
| :----------------------------- | :------------------- |
| **Операционная система** | Все операционные системы [Windows](https://support.microsoft.com/help/2721672/microsoft-server-software-support-for-microsoft-azure-virtual-machines) и [Linux](https://docs.microsoft.com/azure/virtual-machines/linux/endorsed-distros) , поддерживаемые Azure. |
| **Разрешения**           | Для каждой виртуальной машины Hyper-V, которую вы хотите оценить, требуются права администратора. |
| **Integration Services**       | Для записи сведений об операционной системе на виртуальных машинах, которые вы оцениваете, необходимо запустить [Hyper-V Integration Services](https://docs.microsoft.com/virtualization/hyper-v-on-windows/reference/integration-services) . |
| **Необходимые изменения для Azure** | Некоторые виртуальные машины могут потребовать изменений, чтобы они могли работать в Azure. Служба "миграция Azure" автоматически вносит эти изменения в следующие операционные системы:<br/> -Red Hat Enterprise Linux 6.5 +, 7.0 +<br/> -CentOS 6.5 +, 7.0 +</br> -SUSE Linux Enterprise Server 12 SP1 +<br/> -Ubuntu 14.04 LTS, 16.04 LTS, 18.04 LTS<br/> -Debian 7, 8<br/><br/> Для других операционных систем необходимо внести корректировки вручную перед миграцией. В соответствующих статьях содержатся инструкции о том, как это сделать. |
| **Загрузка Linux**                 | Если/boot находится в выделенной секции, он должен находиться на диске ОС, а не распределяться по нескольким дискам.<br/> Если/boot является частью корневого раздела (/), то раздел "/" должен находиться на диске ОС и не охватывать другие диски. |
| **Загрузка UEFI**                  | Виртуальные машины с загрузкой UEFI не поддерживаются для миграции.  |
| **Размер диска**                  | 2 ТБ для диска ОС, 4 ТБ для дисков данных.
| **Номер диска** | Не более 16 дисков на виртуальную машину.
| **Зашифрованные диски и тома**    | Не поддерживается для миграции. |
| **RDM/транзитные диски**      | Не поддерживается для миграции. |
| **Общий диск** | Виртуальные машины, использующие общие диски, не поддерживаются для миграции.
| **NFS**                        | Тома NFS, подключенные к виртуальным машинам как тома, не реплицируются. |
| **ISCSI**                      | Виртуальные машины с целями iSCSI не поддерживаются для миграции.
| **Целевой диск**                | Вы можете выполнить миграцию на виртуальные машины Azure только с управляемыми дисками. |
| **Протокол** | Не поддерживается.
| **Объединение сетевых карт** | Не поддерживается.
| **Azure Site Recovery** | Невозможно выполнить репликацию с помощью миграции сервера Azure Migration, если для виртуальной машины включена репликация с Azure Site Recovery.

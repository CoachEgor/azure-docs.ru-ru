# <a name="support-matrix-for-hyper-v-assessment"></a>Матрица поддержки для оценки Hyper-V

В этой статье приведены общие сведения о параметрах и ограничениях для оценки виртуальных машин Hyper-V с помощью службы ["миграция Azure": Оценка серверов](migrate-services-overview.md#azure-migrate-server-assessment-tool) . Если вы ищете информацию о переносе виртуальных машин Hyper-V в Azure, ознакомьтесь с [матрицей поддержки миграции](migrate-support-matrix-hyper-v-migration.md).

## <a name="overview"></a>Обзор

Чтобы оценить локальные компьютеры для миграции в Azure с помощью этой статьи, добавьте средство "миграция Azure для серверов" в проект службы "миграция Azure". Вы развертываете [устройство "миграция Azure](migrate-appliance.md)". Устройство постоянно обнаруживает локальные компьютеры и отправляет данные о конфигурации и производительности в Azure. После обнаружения компьютеров вы собираете обнаруженные компьютеры в группы и выполняете оценку группы.


## <a name="limitations"></a>Ограничения

**Поддержка** | **Сведения**
--- | ---
**Ограничения оценки**| Обнаружение и оценка до 35 000 виртуальных машин Hyper-V в одном [проекте](migrate-support-matrix.md#azure-migrate-projects).
**Ограничения проекта** | В подписке Azure можно создать несколько проектов. Проект может включать виртуальные машины VMware, виртуальные машины Hyper-V и физические серверы, вплоть до ограничений оценки.
**Обнаружение** | Устройство для переноса Azure может обнаружить до 5000 виртуальных машин Hyper-V.<br/><br/> Устройство может подключаться к узлам Hyper-V до 300.
**Оценка** | В одну группу можно добавить до 35 000 компьютеров.<br/><br/> Вы можете оценить до 35 000 виртуальных машин за одну оценку.

Дополнительные [сведения](concepts-assessment-calculation.md) об оценках.



## <a name="hyper-v-host-requirements"></a>Требования к узлу Hyper-V

| **Поддержка**                | **Сведения**               
| :-------------------       | :------------------- |
| **Развертывание узла**       | Узел Hyper-V может быть автономным или развернутым в кластере. |
| **Разрешения**           | Вам потребуются права администратора на узле Hyper-V. <br/> Кроме того, если вы не хотите назначать разрешения администратора, создайте локальную или доменную учетную запись и добавьте пользователя в эти группы — Пользователи удаленного управления, администраторы Hyper-V и пользователи системного монитора. |
| **Операционная система узла** | Windows Server 2019, Windows Server 2016 или Windows Server 2012 R2.<br/> Вы не можете оценивать виртуальные машины, расположенные на узлах Hyper-V, запущенных на Windows Server 2012. |
| **Удаленное взаимодействие PowerShell**   | Должен быть включен на каждом узле. |
| **Реплика Hyper-V**       | Если вы используете реплику Hyper-V (или несколько виртуальных машин с одинаковыми идентификаторами виртуальных машин) и обнаружите как исходные, так и реплицированные виртуальные машины с помощью службы "миграция Azure", оценка, созданная функцией "миграция Azure", может быть неточной. |


## <a name="hyper-v-vm-requirements"></a>Требования к виртуальным машинам Hyper-V

| **Поддержка**                  | **Сведения**               
| :----------------------------- | :------------------- |
| **Операционная система** | Все операционные системы [Windows](https://support.microsoft.com/help/2721672/microsoft-server-software-support-for-microsoft-azure-virtual-machines) и [Linux](https://docs.microsoft.com/azure/virtual-machines/linux/endorsed-distros) , поддерживаемые Azure. |
| **Службы интеграции**       | Для записи сведений об операционной системе на виртуальных машинах, которые вы оцениваете, необходимо запустить [Hyper-V Integration Services](https://docs.microsoft.com/virtualization/hyper-v-on-windows/reference/integration-services) . |


## <a name="azure-migrate-appliance-requirements"></a>Требования к устройству для миграции в Azure

Служба "миграция Azure" использует [устройство "миграция Azure](migrate-appliance.md) " для обнаружения и оценки. Устройство для Hyper-V выполняется на виртуальной машине Hyper-V и развертывается с помощью сжатого VHD-диска Hyper/V, скачанного из портал Azure. 

- Сведения о [требованиях к устройству](migrate-appliance.md#appliance---hyper-v) для Hyper-V.
- Сведения об [URL-адресах](migrate-appliance.md#url-access) , к которым должно обращаться устройство.

## <a name="port-access"></a>Доступ к портам

В следующей таблице перечислены требования к портам для оценки.

**Устройство** | **Соединение**
--- | ---
**Appliance** | Входящие подключения через TCP-порт 3389 для разрешения подключения удаленного рабочего стола к устройству.<br/> Входящие подключения через порт 44368 для удаленного доступа к приложению управления устройством с помощью URL-адреса: ``` https://<appliance-ip-or-name>:44368 ```<br/> Исходящие подключения к портам 443, 5671 и 5672 для отправки метаданных обнаружения и производительности в службу "миграция Azure".
**Узел или кластер Hyper-V** | Входящие подключения через порты WinRM 5985 (HTTP) и 5986 (HTTPS) для извлечения метаданных конфигурации и производительности виртуальных машин Hyper-V с помощью сеанса модель CIM (CIM).

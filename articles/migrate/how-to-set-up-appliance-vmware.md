# <a name="set-up-an-appliance-for-vmware-vms"></a>Настройка устройства для виртуальных машин VMware

В этой статье описывается настройка устройства "Миграция Azure", если вы оценке виртуальных машин VMware с помощью средства оценки сервера миграции Azure, или перенос виртуальных машин VMware в Azure с помощью миграцию без агента с помощью средства миграции сервера миграции Azure.

Устройство виртуальной Машины VMware — это простой модуль, используемый с Azure перенести оценка/миграция сервера осуществлять следующее:

- Позволяет обнаружить локальные виртуальные машины VMware.
- Отправьте метаданные и данные производительности для обнаруженных виртуальных машинах в Azure перенести оценка/миграция сервера.

[Дополнительные сведения](migrate-appliance.md) о "Миграция Azure".


## <a name="appliance-deployment-steps"></a>Шаги по развертыванию устройства

Чтобы настроить модуль вы:
- Скачайте ova-файл шаблона и импортируйте его на сервер vCenter Server.
- Создать модуль и проверьте, что он может подключиться к оценки сервера миграции Azure. 
- Настройка устройства в первый раз и зарегистрируйте его с проектом "Миграция Azure".

## <a name="download-the-ova-template"></a>Скачивание шаблона OVA

1. В **цели миграции** > **серверы** >  **"Миграция Azure": Оценка Server**, нажмите кнопку **Discover**.
2. В **обнаружение компьютеров** > **ваши компьютеры виртуализированы??** , нажмите кнопку **Да, с помощью гипервизора VMWare vSphere**.
3. Нажмите кнопку **загрузить** для загрузки. Файл ova-файла шаблона.



### <a name="verify-security"></a>Проверка безопасности

Убедитесь, что ova-файл, перед его развертыванием.

1. На компьютере, на который был скачан файл, откройте командное окно с правами администратора.
2. Выполните следующую команду, чтобы создать хэш ova-файла:
    - ```C:\>CertUtil -HashFile <file_location> [Hashing Algorithm]```
    - Пример использования: ```C:\>CertUtil -HashFile C:\AzureMigrate\AzureMigrate.ova SHA256```
3. Для версии устройства 1.0.0.5 созданный хэш должен соответствовать этим параметрам. 

  **Алгоритм** | **Значение хэша**
  --- | ---
  MD5 | ddfdf21c64af02a222ed517ce300c977


## <a name="create-the-appliance-vm"></a>Создание устройства виртуальной Машины

Импортируйте скачанный файл и создать виртуальную Машину.

1. В консоли клиента vSphere щелкните **File** (Файл)  >  **Deploy OVF Template** (Развернуть шаблон OVF).
2. В мастере развертывания шаблона OVF выберите **Source** (Источник) и укажите расположение OVA-файла.
3. В **имя** и **расположение**, укажите понятное имя для виртуальной Машины. Выберите объект инвентаризации, в котором будет размещаться виртуальная машина.
5. В **узлом и кластером**укажите узел или кластер, в которой будет выполняться виртуальной Машины.
6. В **хранения**, укажите место хранения для виртуальной Машины.
7. В поле **Disk Format** (Формат диска) укажите тип и размер диска.
8. В **сетевое сопоставление**, указать сеть, к которой будет подключаться виртуальная машина. Этой сети требуется подключение к Интернету для отправки метаданных в оценки сервера миграции Azure.
9. Просмотрите и подтвердите параметры, а затем нажмите кнопку **Finish** (Готово).


### <a name="verify-appliance-access-to-azure"></a>Проверка доступа устройства к Azure

Убедитесь, что устройство виртуальной Машины могут подключаться к [URL-адресам Azure](migrate-support-matrix-vmware.md#assessment-url-access-requirements).


## <a name="configure-the-appliance"></a>Настройка устройства

Настройка устройства в первый раз.

1. В консоли клиента vSphere щелкните правой кнопкой мыши виртуальную машину и выберите **Open Console** (Открыть консоль).
2. Укажите язык, часовой пояс и пароль для устройства.
3. Откройте браузер на любом компьютере, который может подключиться к виртуальной Машине и откройте URL-адрес веб-приложения устройства: **https://*имя устройства или IP-адрес*: 44368**.

   Кроме того можно открыть приложение с рабочего стола устройства, щелкнув ярлык приложения.
4. В веб-приложения > **Настройка необходимых компонентов**, выполните следующие действия:
    - **Лицензии**: Примите условия лицензии и прочитайте информацию от сторонних производителей.
    - **Подключение**: Приложение проверяет, что виртуальная машина имеет доступ к Интернету. Если виртуальная машина использует прокси-сервер:
        - Нажмите кнопку **параметры прокси-сервера**и укажите адрес прокси-сервера и порт прослушивания, в виде http://ProxyIPAddress или http://ProxyFQDN.
        - Укажите учетные данные, если для прокси-сервера требуется аутентификация.
        - Поддерживается только прокси-сервер HTTP.
    - **Время синхронизации**: Время проверки. Время на устройстве должен быть синхронизирован с временем в Интернете для обнаружения для правильной работы.
    - **Установка обновлений**: Служба "Миграция Azure" проверяет, что установлены последние обновления устройства.
    - **Установка VDDK**: Служба "Миграция Azure" проверяет, что установлен VMWare vSphere пакета средств разработки виртуального диска (VDDK).
        - Azure Migrates использует VDDK реплицировать машины во время миграции в Azure.
        - Скачайте VDDK 6.7 в VMware и извлеките его содержимое скачанного ZIP-файла в указанное расположение на устройстве.

## <a name="register-the-appliance-with-azure-migrate"></a>Регистрация устройства с "Миграция Azure"

1. Нажмите кнопку **вход**. Если он не отображается, убедитесь, что вы отключили блокирование всплывающих окон в браузере.
2. На новой вкладке вход с использованием учетных данных Azure. 
    - Войдите, используя имя пользователя и пароль.
    - Войдите с помощью ПИН-кода не поддерживается.
3. После успешного входа, вернитесь к веб-приложения.
2. Выберите подписку, в которой был создан проект "Миграция Azure". Затем выберите проект.
3. Укажите имя для устройства. Имя должно быть буквенно-цифровой с 14 символов или меньше.
4. Щелкните **Зарегистрировать**.


## <a name="start-continuous-discovery"></a>Запуск непрерывной обнаружения

Теперь подключение устройства к vCenter Server и запустите обнаружение виртуальных Машин. 

1. В **укажите сведения о vCenter Server**, укажите имя (FQDN) или IP-адрес сервера vCenter. Можно оставить порт по умолчанию или указать другой порт, прослушиваемый vCenter Server.
2. В **имя пользователя** и **пароль**, укажите учетные данные учетной записи только для чтения, которые устройство будет использовать для обнаружения виртуальных машин на сервере vCenter. Убедитесь, что учетная запись имеет [разрешения, необходимые для обнаружения](migrate-support-matrix-vmware.md#assessment-vcenter-server-permissions).
3. Нажмите кнопку **Проверка подключения** чтобы убедиться в том, что устройство может подключиться к vCenter Server.
4. После того как соединение установлено, нажмите кнопку **сохранение и запуск обнаружения**.


При этом запускается обнаружение. Занимает около 15 минут для метаданных обнаруженных виртуальных машин отображаются на портале.

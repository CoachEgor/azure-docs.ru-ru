# <a name="delete-an-azure-migrate-project"></a>Удаление проекта службы "Миграция Azure"

В этой статье описывается, как удалить проект [Azure Migrate.](migrate-overview.md)


## <a name="before-you-start"></a>Перед началом работы

Перед удалением проекта:

- При удалении проекта удаляются метаданные и обнаруженные машины.
- Если вы прикрепили рабочее пространство Log Analytics к инструменту оценки сервера для анализа зависимостей, решите, хотите ли удалить рабочее пространство. 
    - Рабочее пространство не удаляется автоматически. Удалите его вручную.
    - Проверьте, для чего используется рабочее пространство, прежде чем удалить его. Одно и то же рабочее пространство Log Analytics можно использовать для нескольких сценариев.
    - Перед удалением проекта можно найти ссылку на рабочее пространство в **Azure Migrate - Серверы** > **Azure Migrate - Оценка серверов**, в **рамках рабочего пространства OMS**.
    - Чтобы удалить рабочее пространство после удаления проекта, найдите рабочее пространство в соответствующей группе ресурсов и следуйте [этим инструкциям.](../azure-monitor/platform/delete-workspace.md)


## <a name="delete-a-project"></a>Удаление проекта


1. На портале Azure откройте группу ресурсов, в которой был создан проект.
2. На странице группы ресурсов выберите **«Показать скрытые типы».**
3. Выберите проект и связанные с ним ресурсы, которые необходимо удалить.
    - Тип ресурса для проектов Azure Migrate — **это Microsoft.Migrate/migrateprojects.**
    - В следующем разделе просмотрите ресурсы, созданные для обнаружения, оценки и миграции в проекте Azure Migrate.
    - Если группа ресурсов содержит только проект Azure Migrate, можно удалить всю группу ресурсов.
    - Если требуется удалить проект из предыдущей версии Azure Migrate, то шаги будут одинаковыми. Тип ресурса для этих проектов — **проект «Миграция».**

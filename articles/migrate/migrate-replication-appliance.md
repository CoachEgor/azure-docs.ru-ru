# <a name="replication-appliance"></a>Модуль репликации

В этой статье описывается модуль репликации, используемый службой "Миграция Azure": Оценка Server при миграции виртуальных машин VMware, физических компьютеров и закрытого и открытого облачных виртуальных машин в Azure, используя миграцию на основе агентов. 

Средство доступно в ["Миграция Azure"](migrate-overview.md) концентратора. Концентратор предоставляет стандартные средства для оценки и миграции, а также средства из других служб Azure и от сторонних независимых поставщиков программного обеспечения (ISV).


## <a name="appliance-overview"></a>Общие сведения об устройстве

Модуль репликации развертывается как один локальный компьютер, либо как виртуальная машина VMware или физическом сервере. Он выполняется:
- **Модуль репликации**: Модуль репликации для управления обменом данными, а также репликацией данных, для локальных виртуальных машин VMware и физических серверов в Azure.
- **Сервер обработки** Сервер обработки, который устанавливается по умолчанию на устройстве репликации и выполняет указанные ниже:
    - **Шлюз репликации**: Он действует как шлюз репликации. Он получает данные репликации с компьютеров, включенные для репликации. Он оптимизирует данные репликации с помощью кэширования, сжатия и шифрования и отправляет его в Azure.
    - **Установка агента**: Выполняет принудительную установку службы Mobility Service. Эта служба устанавливается и установлена на каждом на локальном компьютере, который требуется выполнить репликацию для миграции.

## <a name="appliance-license"></a>Лицензия устройства
Устройство поставляется с лицензией Windows Server 2016 evaluation, который действителен в течение 180 дней. Если период оценки приближается срок действия, рекомендуется загрузить и развернуть новое устройство или активировать лицензию операционную систему устройства виртуальной Машины.

## <a name="replication-process"></a>Процесс репликации

1. При включении репликации для виртуальной машины начинается начальная репликация в службу хранилища Azure с помощью указанной политики репликации. 
2. Трафик реплицируется в общедоступные конечные точки службы хранилища Azure через Интернет. Репликация трафика через VPN типа "сеть — сеть" с локального сайта в Azure не поддерживается.
3. После завершения начальной репликации начинается разностная репликация. Регистрируются отслеживаемые изменения для компьютера.
4. Обмен данными происходит следующим образом.
    - Виртуальные машины обмениваются данными с устройством репликации через порт HTTPS 443 входящих подключений, для управления репликацией.
    - Модуль репликации управляет репликацией с Azure через порт HTTPS 443 для исходящих подключений.
    - Виртуальные машины отправляют данные репликации на сервер обработки (запущенного на устройстве репликации) через порт HTTPS 9443 входящего трафика. Этот порт можно изменить.
    - Сервер обработки получает данные репликации, оптимизирует и шифрует их, а затем отправляет в службу хранилища Azure через порт 443 для исходящих подключений.
5. Данные репликации первый land журналы в учетную запись хранения кэша в Azure. Эти журналы, обрабатываются и данные хранятся в Azure управляемого диска.

## <a name="appliance-upgrades"></a>Обновление устройства

Модуль обновляется вручную из центра "Миграция Azure". Мы рекомендуем всегда использовать последнюю версию.

1. В Azure миграция > серверы > "Миграция Azure": Оценка Server, серверы инфраструктуры, нажмите кнопку **серверы конфигурации**.
2. В **серверы конфигурации**, поместить ссылку **версия агента** когда доступна новая версия модуль репликации. 
3. Загрузите установщик на компьютере модуль репликации и установить обновления. Программа установки обнаруживает текущей установленной версии на устройстве.

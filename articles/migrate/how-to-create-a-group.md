# <a name="create-a-group-for-assessment"></a>Создание группы для оценки

В этой статье описывается, как создавать группы машин для оценки с помощью Azure Migrate: Server Assessment.

[Azure Migrate](migrate-services-overview.md) помогает вам мигрировать в Azure. "Миграция Azure" — это центральный узел, с помощью которого можно отслеживать обнаружение, оценку и перенос локальной инфраструктуры, приложений и данных в Azure. Центр обеспечивает инструменты Azure для оценки и миграции, а также предложения сторонних независимых поставщиков программного обеспечения (ISV). 

## <a name="grouping-machines"></a>Групповые машины

Вы собираете машины в группы, чтобы оценить, подходят ли они для миграции в Azure, и получить оценку размеров azure и оценки затрат для них. Существует несколько способов создания групп:

- Если вы знаете машины, которые необходимо перенести вместе, можно вручную создать группу в Azure Migrate.
- Если вы не уверены в машинах, которые должны быть сгруппированы вместе, вы можете использовать функциональность визуализации зависимостей в Azure Migrate для создания групп. 

> [!NOTE]
> Функция визуализации зависимостей недоступна в Azure для государственных организаций.

## <a name="create-a-group-manually"></a>Создание группы вручную

Вы можете создать группу одновременно с [созданием оценки.](how-to-create-assessment.md)

Если вы хотите создать группу вручную за пределами создания оценки, сделайте следующее:

1. В проекте Azure Migrate > **Обзор,** нажмите **Оценка и миграция серверов**. В **Azure Миграция: Оценка серверов**, нажмите **группы**
    - Если вы еще не добавили инструмент Azure Migrate: Server Assessment, нажмите, чтобы добавить его. Ознакомьтесь с [дополнительными сведениями](how-to-assess.md).
    - Если вы еще не создали проект Azure Migrate, [узнайте больше.](how-to-add-tool-first-time.md)

    ![Выбор групп](./media/how-to-create-a-group/select-groups.png)

2. Нажмите значок **группы.**
3. В **группе «Создание»** укажите имя группы, а в **названии прибора**выберите прибор Azure Migrate, который вы используете для обнаружения машины.
1. Из списка машин выберите машины, которые вы хотите добавить в группу > **Создать.**

    ![Создание группы](./media/how-to-create-a-group/create-group.png)

Теперь вы можете использовать эту группу при [создании оценки.](how-to-create-assessment.md)

## <a name="refine-a-group-with-dependency-mapping"></a>Уточните группу с картированием зависимостей

Отображение зависимостей помогает визуализировать зависимости в разных машинах. Обычно используется отображение зависимостей, когда требуется оценить группы машин с более высоким уровнем достоверности.
- Это поможет вам перепроверить зависимости машины, прежде чем запустить оценку. 
- Это также помогает эффективно планировать миграцию в Azure, гарантируя, что ничего не осталось позади, и тем самым избегая внезапных сбоев во время миграции.
- Можно обнаружить взаимозависимые системы, которым необходимо мигрировать вместе, и определить, является ли запущенная система по-прежнему обслуживает пользователей, или же она является кандидатом на вывод из эксплуатации вместо миграции.

Если вы уже [настроили отображение зависимостей](how-to-create-group-machine-dependencies.md)и хотите уточнить существующую группу, сделайте следующее:

1. Во вкладке **«Серверы»** в **Azure Migrate: Server Assessment** tile нажмите **«Группы»**
2. Нажмите на группу, которая требуется уточнить.
    - Если вы еще не настроили отображение зависимостей, столбец **зависимостей** покажет статус **установки «Требуется».** Для каждого VM, для которого вы хотите визуализировать зависимости, нажмите **Требуется установка.** Установите несколько агентов на каждый VM, прежде чем вы сможете сопоставить зависимости машины. Ознакомьтесь с [дополнительными сведениями](how-to-create-group-machine-dependencies.md).

        ![Добавление отображения зависимостей](./media/how-to-create-a-group/add-dependency-mapping.png)

    - Если вы уже настроили отображение зависимостей на странице группы, нажмите **на зависимые представления,** чтобы открыть карту групповых зависимостей.

3. После нажатия **зависимостей просмотра**на карте групповых зависимостей показано следующее:

    - Входящие (клиенты) и исходящие (серверы) соединения TCP к и из всех машин в группе, которые имеют агенты зависимости установлены.
    - Зависимые машины, в которых нет установленных агентов зависимости, сгруппированы по номерам портов.
    - Зависимые машины с установленными агентами зависимости отображаются как отдельные коробки.
    - Процессы, работающие внутри машины. Расширьте каждую машинную коробку для просмотра процессов.
    - Свойства машины (включая ФЗДН, операционную систему, адрес MAC). Нажмите на каждую машинную коробку, чтобы просмотреть детали.

4. Чтобы просмотреть зависимости в интервале времени по вашему выбору, измените временной диапазон (час по умолчанию), указав даты начала и окончания или продолжительность.

    > [!NOTE]
    > Диапазон времени может составить до часа. Если вам нужен более длительный диапазон, используйте [Azure Monitor для запроса зависимых данных](how-to-create-group-machine-dependencies.md) на более длительный период.

5. После того, как вы определили зависимости, которые вы хотели бы добавить или удалить из группы, можно изменить группу. Используйте Ctrl-Click, чтобы добавить или удалить машины из группы.

    - Добавить можно только компьютеры, которые были обнаружены.
    - Добавление и удаление машин делает недействительными прошлые оценки для группы.
    - При необходимости можно создать новую оценку при изменении группы.

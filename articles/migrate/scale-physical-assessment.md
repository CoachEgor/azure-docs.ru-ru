# <a name="assess-large-numbers-of-physical-servers-for-migration-to-azure"></a>Оценка большого количества физических серверов для миграции в Azure

В этой статье описывается, как оценить большое количество внутренних физических серверов для миграции в Azure с помощью инструмента оценки azure Migrate Server Assessment.

Служба [Миграция Azure](migrate-services-overview.md) объединяет в себе инструменты, которые используются для поиска, оценки и переноса приложений, инфраструктуры и рабочих нагрузок в Microsoft Azure. Она включает в себя инструменты Миграции Azure и предложения независимых поставщиков программного обеспечения. 


Вы узнаете, как выполнять следующие задачи:
> [!div class="checklist"]
> * План оценки в масштабе.
> * Нанастройка разрешений Azure и подготовка физических серверов к оценке.
> * Создайте проект Azure Migrate и создайте оценку.
> * Просмотрите оценку по мере планирования миграции.


> [!NOTE]
> Если вы хотите попробовать доказательство концепции для оценки нескольких серверов, прежде чем оценивать в масштабе, следуйте нашей [серии учебников](tutorial-prepare-physical.md).

## <a name="plan-for-assessment"></a>План оценки

При планировании оценки большого количества физических серверов, Есть несколько вещей, чтобы думать о:

- **Планируйте проекты Azure Migrate:** Выясните, как развернуть проекты Azure Migrate. Например, если ваши центры обработки данных находятся в разных географических регионах или вам необходимо хранить метаданные обнаружения, оценки или миграции в другой географии, возможно, потребуется несколько проектов.
- **Приборы плана:** Azure Migrate использует наместераренный прибор Azure Migrate, развернутый на компьютере Windows, для постоянного обнаружения серверов для оценки и миграции. Прибор отслеживает изменения среды, такие как добавление виз, дисков или сетевых адаптеров. Он также отправляет метаданные и данные о производительности о них в Azure. Вам нужно выяснить, сколько приборов для развертывания.


## <a name="planning-limits"></a>Ограничения планирования
 
Используйте ограничения, суммированные в этой таблице, для планирования.

**Планирования** | **Ограничения**
--- | --- 
**Проекты Azure Migrate** | Оцените до 35 000 серверов в проекте.
**Устройство службы "Миграция Azure"** | Прибор может обнаружить до 250 серверов.<br/> Прибор может быть связан только с одним проектом Azure Migrate.<br/> Любое количество приборов может быть связано с одним проектом Azure Migrate. <br/><br/> 
**Группы** | Вы можете добавить до 35 000 серверов в одну группу.
**Оценка миграции Azure** | Вы можете оценить до 35 000 серверов в одной оценке.


## <a name="other-planning-considerations"></a>Другие соображения планирования

- Чтобы начать открытие с прибора, вы должны выбрать каждый физический сервер. 

## <a name="prepare-for-assessment"></a>Подготовка к оценке

Подготовьте Azure и физические серверы для оценки серверов. 

1. Проверить [требования и ограничения поддержки физических серверов.](migrate-support-matrix-physical.md)
2. Настройка разрешений для учетной записи Azure для взаимодействия с Azure Migrate.
3. Подготовьте физические серверы.

Следуйте инструкциям в [этом учебнике,](tutorial-prepare-physical.md) чтобы настроить эти настройки.

## <a name="create-a-project"></a>Создание проекта

В соответствии с требованиями планирования, сделайте следующее:

1. Создайте проект службы "Миграция Azure".
2. Добавьте в проекты инструмент оценки миграционных серверов Azure Migrate.

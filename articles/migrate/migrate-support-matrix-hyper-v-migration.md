# <a name="support-matrix-for-hyper-v-migration"></a>Матрица поддержки для миграции Hyper-V

В этой статье кратко излагаются параметры поддержки и ограничения для миграции Hyper-V VMs с [Azure Migrate: Server Migration](migrate-services-overview.md#azure-migrate-server-migration-tool) . Если вы ищете информацию об оценке Hyper-V V MMs для миграции в Azure, просмотрите [матрицу поддержки оценки.](migrate-support-matrix-hyper-v.md)

## <a name="migration-limitations"></a>Ограничения миграции

Вы можете выбрать до 10 ВМ одновременно для репликации. Если вы хотите перенести больше машин, реплицируйте в группах по 10.


## <a name="hyper-v-hosts"></a>Узлы Hyper-V

| **Поддержка**                | **Сведения**               
| :-------------------       | :------------------- |
| **Развертывание**       | Хост Hyper-V может быть автономным или развернутым в кластере. <br/>Программное обеспечение репликации Azure Migrate (поставщик репликации Hyper-V) устанавливается на хостах Hyper-V.|
| **Разрешения**           | Вам нужны разрешения администратора на хосте Hyper-V. |
| **Операционная система хозяина** | Windows Server 2019, Windows Server 2016 или Windows Server 2012 R2. |
| **Доступ в порт** |  Исходящие соединения в порту HTTPS 443 для отправки данных репликации VM.

### <a name="url-access-public-cloud"></a>Доступ к URL (публичное облако)

Программному обеспечению поставщика репликации на хостах Hyper-V потребуется доступ к этим URL-адресам.

**URL-адрес** | **Сведения**
--- | ---
login.microsoftonline.com | Управление доступом и управление идентификаторами с помощью Active Directory.
backup.windowsazure.com | Передача и координация данных репликации.
*.hypervrecoverymanager.windowsazure.com | Используется для миграции.
*.blob.core.windows.net | Загрузка данных в учетные записи хранения. 
dc.services.visualstudio.com | Загружать журналы приложений, используемые для внутреннего мониторинга.
time.windows.com | Проверяет синхронизацию времени между системой и глобальным временем.

## <a name="hyper-v-vms"></a>Виртуальные машины Hyper-V

| **Поддержка**                  | **Сведения**               
| :----------------------------- | :------------------- |
| **Операционная система** | Все операционные системы [Windows](https://support.microsoft.com/help/2721672/microsoft-server-software-support-for-microsoft-azure-virtual-machines) и [Linux,](https://docs.microsoft.com/azure/virtual-machines/linux/endorsed-distros) поддерживаемые Azure. |
| **Необходимые изменения для Azure** | Для некоторых виртуальных машин могут потребоваться изменения, позволяющие им работать в Azure. Вносить коррективы вручную перед миграцией. Соответствующие статьи содержат инструкции о том, как это сделать. |
| **Linux загрузки**                 | Если /загрузка находится на выделенной разгородке, она должна находиться на диске ОС, а не распространяться на несколько дисков.<br/> Если /загрузка является частью корневой (/) раздела, то раздел '/' должен быть на диске ОС, а не охватывать другие диски. |
| **Загрузка UEFI**                  | Мигрированный VM в Azure будет автоматически преобразован в bioS boot VM. VM должен быть запущен Windows Server 2012 и только позже. Диск ОС должен иметь до пяти разделов или меньше, а размер диска ОС должен быть менее 300 ГБ.
  |
| **Размер диска**                  | 2 ТБ для диска ОС, 4 ТБ для дисков данных.
| **Номер диска** | Максимум 16 дисков на VM.
| **Зашифрованные диски/объемы**    | Не поддерживается для миграции. |
| **RDM/проходные диски**      | Не поддерживается для миграции. |
| **Общий диск** | VMs, использующие общие диски, не поддерживаются для миграции.
| **NFS**                        | Объемы NFS, установленные в виде объемов на ВМ, не будут реплицироваться. |
| **Iscsi**                      | VMs с целями iSCSI не поддерживаются для миграции.
| **Целевой диск**                | Вы можете мигрировать в Azure VMs только с управляемыми дисками. |
| **IPv6** | Не поддерживается.
| **Объединение NIC** | Не поддерживается.
| **Azure Site Recovery** | Вы не можете воспроизвести с помощью Azure Migrate Server Migration, если VM включен для репликации с восстановлением сайта Azure.
| **порты;** | Исходящие соединения в порту HTTPS 443 для отправки данных репликации VM.

## <a name="azure-vm-requirements"></a>Требования для виртуальных машин Azure

Все предприимчивые VM, реплицированные в Azure, должны соответствовать требованиям Azure VM, обобщенным в этой таблице.

**Компонент** | **Требования** | **Сведения**
--- | --- | ---
Размер диска операционной системы | До 2048 ГБ | Если не поддерживается, проверка завершается ошибкой.
Число дисков операционной системы | 1 | Если не поддерживается, проверка завершается ошибкой.
Число дисков с данными | 16 или меньше. | Если не поддерживается, проверка завершается ошибкой.
Размер диска данных | До 4095 ГБ | Если не поддерживается, проверка завершается ошибкой.
Сетевые адаптеры | Поддерживаются несколько адаптеров. |
Общий виртуальный жесткий диск | Не поддерживается. | Если не поддерживается, проверка завершается ошибкой.
Диск FC | Не поддерживается. | Если не поддерживается, проверка завершается ошибкой.
BitLocker | Не поддерживается. | Прежде чем включать репликацию для компьютера, необходимо отключить BitLocker.
имя виртуальной машины; | От 1 до 63 символов.<br/> при этом допустимы только буквы, цифры и дефисы<br/><br/> Имя компьютера должно начинаться и заканчиваться буквой или цифрой. |  Обновите значение в свойствах компьютера в службе Site Recovery.
Подключение после миграции-Windows | Для подключения к Пазур VMs под управлением Windows после миграции:<br/> - Перед миграцией позволяет RDP на месте VM. Убедитесь в том, что правила для протоколов TCP и UDP для **общедоступного** профиля добавлены, а протокол RDP разрешен в разделе **Брандмауэр Windows** > **Разрешенные программы** для всех профилей.<br/> Для доступа к VPN от сайта, включить RDP и позволяют RDP в **Windows Firewall** -> **Разрешенные приложения и функции** для **домена и частных** сетей. Кроме того, убедитесь, что политика SAN операционной системы установлена на **OnlineAll**. [Подробнее](prepare-for-migration.md). |
Подключение после миграции-Linux | Для подключения к MMs Azure после миграции с помощью SSH:<br/> Перед миграцией на предварительной машине проверьте, что служба Secure Shell настроена на запуск, и что правила брандмауэра позволяют подключение SSH.<br/> После сбоя в Azure VM разрешить входящие соединения в порт SSH для правил группы сетевой безопасности на сбой над VM, и для подсети Azure, к которой он подключен. Кроме того, добавьте общедоступный IP-адрес для VM. |

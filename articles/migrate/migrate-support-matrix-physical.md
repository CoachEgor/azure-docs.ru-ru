# <a name="support-matrix-for-physical-server-assessment"></a>Матрица поддержки для оценки физических серверов 

[Службу "миграция Azure](migrate-overview.md) " можно использовать для оценки и миграции компьютеров в облако Microsoft Azure. В этой статье перечислены параметры поддержки и ограничения для оценки и миграции локальных физических серверов.


## <a name="overview"></a>Обзор

Чтобы оценить локальные компьютеры для миграции в Azure с помощью этой статьи, добавьте средство "миграция Azure для серверов" в проект службы "миграция Azure". Вы развертываете [устройство "миграция Azure](migrate-appliance.md)". Устройство постоянно обнаруживает локальные компьютеры и отправляет данные о конфигурации и производительности в Azure. После обнаружения компьютеров вы собираете обнаруженные компьютеры в группы и выполняете оценку группы.

## <a name="limitations"></a>Ограничения

**Поддержка** | **Сведения**
--- | ---
**Ограничения оценки**| Обнаружение и оценка до 35 000 физических серверов в одном [проекте](migrate-support-matrix.md#azure-migrate-projects).
**Ограничения проекта** | В подписке Azure можно создать несколько проектов. Проект может включать виртуальные машины VMware, виртуальные машины Hyper-V и физические серверы, вплоть до ограничений оценки.
**Обнаружение** | Устройство для переноса Azure может обнаружить до 250 физических серверов.
**Оценка** | В одну группу можно добавить до 35 000 компьютеров.<br/><br/> Вы можете оценить до 35 000 компьютеров за одну оценку.

Дополнительные [сведения](concepts-assessment-calculation.md) об оценках.




## <a name="physical-server-requirements"></a>Требования к физическому серверу

| **Поддержка**                | **Сведения**               
| :-------------------       | :------------------- |
| **Развертывание физического сервера**       | Физический сервер может быть автономным или развернутым в кластере. |
| **Разрешения**           | **Windows:** Настройте локальную учетную запись пользователя на всех серверах Windows, которые необходимо включить в обнаружение. Учетная запись пользователя должна быть добавлена в эти группы — удаленный рабочий стол пользователи, монитор производительности и пользователи журнала производительности. <br/> **Linux:** Вам потребуется учетная запись root на серверах Linux, которые вы хотите обнаружить. |
| **Операционная система** | Поддерживаются все операционные системы [Windows](https://support.microsoft.com/help/2721672/microsoft-server-software-support-for-microsoft-azure-virtual-machines) и [Linux](https://docs.microsoft.com/azure/virtual-machines/linux/endorsed-distros) , за исключением следующих:<br/> Windows Server 2003 <br/> SUSE Linux|


## <a name="azure-migrate-appliance-requirements"></a>Требования к устройству для миграции в Azure

Служба "миграция Azure" использует [устройство "миграция Azure](migrate-appliance.md) " для обнаружения и оценки. Устройство для физических серверов может работать на виртуальной машине или на физическом компьютере. Его можно настроить с помощью скрипта PowerShell, скачанного из портал Azure.

- Сведения о [требованиях к устройству](migrate-appliance.md#appliance---physical) для физических серверов.
- Сведения об [URL-адресах](migrate-appliance.md#url-access) , к которым должно обращаться устройство.

## <a name="port-access"></a>Доступ к портам

В следующей таблице перечислены требования к портам для оценки.

**Устройство** | **Соединение**
--- | ---
**Appliance** | Входящие подключения через TCP-порт 3389 для разрешения подключения удаленного рабочего стола к устройству.<br/> Входящие подключения через порт 44368 для удаленного доступа к приложению управления устройством с помощью URL-адреса: ``` https://<appliance-ip-or-name>:44368 ```<br/> Исходящие подключения к портам 443, 5671 и 5672 для отправки метаданных обнаружения и производительности в службу "миграция Azure".
**Физические серверы** | **Windows:** Входящие подключения через порт 443, WinRM Ports 5985 (HTTP) и 5986 (HTTPS) для извлечения метаданных конфигурации и производительности с серверов Windows. <br/> **Linux:**  Входящие подключения через порт 22 (UDP) для извлечения метаданных конфигурации и производительности с серверов Linux. |

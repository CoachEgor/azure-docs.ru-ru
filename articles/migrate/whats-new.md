# <a name="whats-new-in-azure-migrate"></a>Новые возможности в службе "Миграция Azure"

С помощью [службы "Миграция Azure"](migrate-services-overview.md) можно обнаруживать, оценивать и переносить локальные серверы, приложения и данные в облако Microsoft Azure. В этой статье описаны новые функции в службе "Миграция Azure".



## <a name="update-november-2019"></a>Обновление (ноябрь 2019 г.)

В службу "Миграция Azure" добавлено несколько новых возможностей.

- **Оценка физического сервера**. Помимо миграции физических серверов, теперь также поддерживается оценка локальных физических серверов.
- **Оценка на основе импорта**. Теперь поддерживается оценка компьютеров с использованием метаданных и данных о производительности, указанных в CSV-файле.
- **Обнаружение приложений**. Служба "миграция Azure" теперь поддерживает обнаружение приложений, ролей и компонентов на уровне приложений с помощью устройства "миграция Azure". Сейчас эта возможность поддерживается только для виртуальных машин VMware и ограничена только обнаружением (в настоящее время оценка не поддерживается). [Дополнительные сведения](how-to-discover-applications.md)
- **Визуализация зависимостей без агента**. вам больше не нужно явно устанавливать агенты для визуализации зависимостей. Теперь поддерживается как безагентный вариант, так и вариант на основе агента.
- **Виртуальный рабочий стол**. Используйте средства ISV для оценки и миграции локальной инфраструктуры виртуальных рабочих столов (VDI) в виртуальные рабочие столы Windows в Azure.
- **Веб-приложение**. Помощник по миграции службы приложений Azure, которая используется для оценки и миграции веб-приложений, теперь интегрирована в службу "миграция Azure".

В службу "Миграция Azure" добавлены новые средства оценки и миграции.

- **Раккваре**: предложение миграции в облако.
- Перемещение **: Оценка**предложения.

[Узнайте больше](migrate-services-overview.md) об оценке и миграции в службе "Миграция Azure" с помощью средств и предложений от независимых поставщиков программного обеспечения.

## <a name="release-version-july-2019"></a>Версия выпуска (июль 2019 г.)

Текущая версия службы "Миграция Azure" была выпущена в июле 2019 г.

- **Текущая версия**: Используйте эту версию для создания проектов службы "миграция Azure", обнаружения локальных компьютеров и управления оценками и миграциями.
- **Предыдущая версия**: для клиента, использующего предыдущую версию службы "миграция Azure" (поддерживается только оценка локальных виртуальных машин VMware), теперь следует использовать текущую версию. В предыдущей версии вы больше не можете создавать новые проекты службы "Миграция Azure" или выполнять новые обнаружения. Вы по-прежнему можете получить доступ к существующим проектам. Перейдите на портал Azure, выберите **Все службы** и выполните поиск по запросу **Миграция Azure**. В уведомлении службы "Миграция Azure" есть уведомление и ссылка для доступа к старым проектам службы.


### <a name="azure-migrate-features"></a>Миграция Azure. Функции

Текущая версия службы "Миграция Azure" предоставляет ряд новых функций.


- **Унифицированная платформа миграции**. Теперь служба "миграция Azure" предоставляет единый портал для централизации, управления и мониторинга перехода в Azure с улучшенным потоком развертывания и интерфейсом портала.
- **Средства оценки и миграции**. Служба "миграция Azure" предоставляет собственные средства и интегрируется с другими службами Azure, а также с независимыми средствами поставщика программного обеспечения (ISV). [Дополнительные сведения](migrate-services-overview.md#isv-integration) об интеграции ISV.
- **Оценка службы "миграция Azure**". с помощью средства Azure Migration Server для миграции можно оценить виртуальные машины VMware и виртуальные машины Hyper-V для переноса в Azure. Также можно оценить миграцию, используя другие службы Azure, а также инструменты ISV.
- Миграция с **миграцией Azure**. с помощью средства миграции сервера Azure Migration Server можно перенести локальные виртуальные машины VMware и виртуальные машины Hyper-V в Azure, а также физические серверы, другие виртуализированные серверы и виртуальные машины частного или общедоступного облака. Кроме того, вы можете перенести их в Azure с помощью инструментов ISV.
- **Устройство для миграции Azure**. Служба "миграция Azure" развертывает упрощенное устройство для обнаружения и оценки локальных виртуальных машин VMware и виртуальных машин Hyper-V.
    - Это устройство используется при оценивании и миграции сервера службы "Миграция Azure" без агентов.
    - Метаданные сервера и данные производительности в рамках оценки и миграции можно обнаруживать с помощью непрерывной работы устройства.  
- **Миграция виртуальных машин VMware**. Миграция миграции сервера в Azure предоставляет несколько способов миграции локальных виртуальных машин VMware в Azure.  Миграция без агента с использованием устройства службы "Миграция Azure" и миграция с агентом, при которой используется устройство репликации и развертывается агент в каждой виртуальной машине, которую требуется перенести. [Дополнительные сведения](server-migrate-overview.md)
 - **Оценка и миграция базы данных**: из службы "миграция Azure" вы можете оценить локальные базы данных для миграции в Azure с помощью помощник по миграции базы данных Azure. Базы данных можно перенести с помощью сервера миграции базы данных Azure.
- **Перенос веб-приложений**. Вы можете оценивать веб-приложения с помощью URL-адреса общедоступной конечной точки в службе приложений Azure. Для переноса внутренних приложений .NET можно загрузить и запустить помощника по миграции службы приложений.
- **Data Box**: импорт больших объемов автономных данных в Azure с помощью Azure Data Box в службе "миграция Azure".

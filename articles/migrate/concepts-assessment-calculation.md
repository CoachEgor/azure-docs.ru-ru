# <a name="assessment-calculations"></a>Вычисление оценок

Служба " [Миграция Azure](migrate-services-overview.md) " предоставляет центральный концентратор для наблюдения за обнаружением, оценкой и миграцией локальных приложений и рабочих нагрузок, а также экземпляров частных и общедоступных облаков в Azure. Центр предоставляет средства миграции Azure для оценки и миграции, а также предложения независимых поставщиков программного обеспечения (ISV) сторонних разработчиков.

Оценка сервера — это средство в службе "миграция Azure", которое оценивает локальные серверы для миграции в Azure. В этой статье содержатся сведения о том, как эти оценки вычисляются.

## <a name="how-are-assessments-calculated"></a>Как вычисляется Оценка?

Оценка в службе "Миграция серверов Azure" вычисляется с использованием метаданных, собранных о локальных серверах. Вычисление оценки выполняется в три этапа; для каждого сервера расчет оценки начинается с анализа пригодности для Azure, за которым следует изменение размера и, наконец, оценка ежемесячных затрат. Сервер перемещается на более поздний этап, если он проходит предыдущий. Например, если сервер не прошел проверку на пригодность для Azure, он помечается как непригодный для Azure, а изменение размера и цены для этого не выполняется.

## <a name="azure-suitability-analysis"></a>Анализ пригодности для Azure

Не все компьютеры подходят для работы в Azure. Служба "миграция с сервера Azure" оценивает каждый локальный компьютер на соответствие требованиям для миграции в Azure и распределяет компьютеры по категориям в одной из следующих категорий пригодности:
- **Готово к работе в Azure**. Компьютер может быть перенесен в Azure без каких-либо изменений. Он загрузится в Azure и будет иметь полную поддержку Azure.
- **Условно готово к работе в Azure**. Компьютер может загружаться в Azure, но не будет иметь полную поддержку Azure. Например компьютеры с более старой версией ОС Windows Server не поддерживаются в Azure. Прежде чем переносить эти машины в Azure, необходимо внимательно следовать рекомендациям по исправлению, предложенным в процессе оценки, и устранить проблемы с готовностью.
- **Не готово к работе в Azure**. Компьютер не загрузится в Azure. Например, если к локальному компьютеру подключен диск размером более 64 ТБ, он не может быть размещен в Azure. Необходимо следовать рекомендациям по исправлению, предложенным в процессе оценки, и устранить проблемы с готовностью переноса в Azure. Не выполнена правильная оценка размера и стоимости для компьютера, отмеченных как не готовых к работе в Azure.
- **Неизвестная готовность** . службе "миграция Azure" не удалось найти готовность компьютера из-за недостатка метаданных, собранных из локальной среды.

Служба "миграция с сервера Azure" проверяет свойства компьютера и гостевой операционной системы, чтобы узнать готовность Azure к работе с локальным компьютером.

## <a name="sizing"></a>Изменения размера

После того как компьютер помечается как готовый для Azure, при оценке сервера выполняется его изменение, которое включает определение соответствующей ВИРТУАЛЬНОЙ машины Azure и номера SKU диска для локальной виртуальной машины. Рекомендации по размеру зависят от указанных свойств оценки.

- Если для оценки используется размер *на основе производительности*, то служба "миграция Azure" считает журнал производительности компьютера для определения размера виртуальной машины и типа диска в Azure. Этот метод особенно полезен, если вы превысили распределение локальной виртуальной машины, но при этом используется низкий уровень использования, и вам нужно щелкнуть правой кнопкой мыши размер виртуальной машины в Azure, чтобы сэкономить затраты. Этот метод поможет вам оптимизировать размеры во время миграции.
- Если вы не хотите учитывать данные о производительности для изменения размера виртуальной машины и хотите перевести локальные компьютеры в "как есть" в Azure, можно указать критерий изменения размера как *в локальной среде* , так и при оценке сервера, а затем выбрать размер виртуальных машин на основе локальной среды. конфигурация без учета данных об использовании. Размер диска в этом случае будет выполняться в зависимости от типа хранилища, указанного в свойствах оценки (HDD (цен. категория "Стандартный") диски, SSD (цен. категория "Стандартный") диски или диски уровня "Премиум").

### <a name="performance-based-sizing"></a>Выбор размера на основе производительности

Для изменения размера на основе производительности Служба "Миграция серверов Azure" начинается с дисков, подключенных к виртуальной машине, за которыми следуют сетевые адаптеры, а затем сопоставляет номера SKU виртуальной машины Azure на основе требований к вычислению локальной виртуальной машины. Устройство "миграция Azure" проводит профили в локальной среде для получения данных о производительности ЦП, памяти, дисков и сетевого адаптера.

**Сбор данных о производительности**

- Для виртуальных машин VMware устройство "миграция Azure" собирает образец точки в режиме реального времени каждые 20 секунд, для виртуальных машин Hyper-V в режиме реального времени каждый 30-секундный интервал собираются.
- Затем устройство выполняет сведение образцов точек, собранных на каждые 10 минут, и отправляет максимальное значение за последние 10 минут до оценки сервера в Azure.
- Служба "Миграция серверов Azure" хранит все 10-минутные точки выборки за последний месяц и в зависимости от свойств оценки, указанных для *журнала производительности* и *использования процентилей*, он определяет соответствующую точку данных. Этот параметр следует использовать для правильного изменения размера. Например, если для журнала производительности задано значение один день, а процентиль — 95 процентиль процентиль, то он использует 10-минутные точки выборки за последний день, сортирует их в возрастающем порядке и выбирает значение 95 процентиль процентиля для правильного изменения размера.
- Приведенное выше значение затем умножается на эффективный фактор, чтобы получить эффективные данные об использовании производительности для каждой метрики (использование ЦП, использование памяти, операции ввода-вывода на диск (чтение и запись), пропускная способность диска (чтение и запись), пропускная способность сети (in и out)), устройство собирает.
- После определения значения эффективного использования вычисление, хранение и изменение размера сети выполняются следующим образом.

**Размер хранилища**: Служба "Миграция Azure" пытается сопоставить каждый диск, подключенный к компьютеру, с диском в Azure.

> [!NOTE]
> Миграция Azure: Оценка серверов поддерживает оценку только управляемых дисков.

  - Для получения общего количества операций ввода-вывода в секунду добавляются операции чтения и записи диска, чтобы получить общую пропускную способность каждого диска.
  - Если вы указали автоматический тип хранилища на основе действующих значений операций ввода-вывода и пропускной способности, то служба "Миграция серверов Azure" выполняет оценку сервера, а затем определяет, следует ли сопоставить диск со стандартным HDD, стандартным SSD или диском уровня "Премиум" в Azure. Если для типа хранилища задано значение HDD (цен. категория "Стандартный")/SSD/Premium, он будет пытаться найти номер SKU диска в выбранном типе хранилища (HDD (цен. категория "Стандартный")/SSD/Premium).
  - Если серверной оценке не удается найти диск с требуемой пропускной способностью операций ввода-вывода в секунду &, он помечает компьютер как неподходящий для Azure.
  - Если он находит набор подходящих дисков, он выбирает те из них, которые поддерживают расположение, указанное в параметрах оценки.
  - Если доступно несколько соответствующих дисков, она выбирает тот, который обеспечивает наименьшие затраты.
  - Если данные о производительности для любого диска в недоступном расположении, данные конфигурации диска (размер диска) используются для поиска в Azure стандартного SSD-диска.

**Размер сети**: При оценке сервера "миграция Azure" выполняется попытка найти виртуальную машину Azure, которая может поддерживать количество сетевых адаптеров, подключенных к локальному компьютеру, и производительность, необходимую для этих сетевых адаптеров.
    - Чтобы получить эффективную производительность сети для локальной виртуальной машины, в ходе оценки сервера выполняется статистическая обработка данных, передаваемых в секунду (вне сети), между всеми сетевыми адаптерами и применением эффективного фактора. Это число используется для поиска виртуальной машины Azure, обеспечивающей нужную производительность сети.
    - Наряду с производительностью сети также учитывается, поддерживает ли виртуальная машина Azure необходимое количество сетевых адаптеров.
    - Если данные о производительности сети недоступны, то для выбора размера виртуальной машины учитывается только число сетевых адаптеров.

**Размер вычислений**: После вычисления требований к хранению и сети в ходе оценки серверов Azure Migrate считает требования к ЦП и памяти, чтобы найти подходящий размер виртуальной машины в Azure.
    - Служба "миграция Azure" рассматривает эффективные используемые ядра и память, чтобы найти подходящий размер виртуальной машины в Azure.
    - Если найти подходящий размер не удалось, компьютер помечается как неподходящий для Azure.
    - Если найден подходящий размер, служба "Миграция Azure" применяет результаты вычисления параметров хранилища и сети. Затем она применяет параметры расположения и ценовой категории, чтобы получить окончательный рекомендуемый размер виртуальной машины.
    - Если доступно несколько соответствующих размеров виртуальных машин, служба выбирает рекомендуемый, который обеспечивает наименьшие затраты.

### <a name="as-on-premises-sizing"></a>Определение размера как для локальной виртуальной машины

Если вы используете как *локальное изменение размера*, Оценка сервера не учитывает журнал производительности виртуальных машин и дисков и выделяет номер SKU виртуальной машины в Azure в зависимости от размера, выделенного в локальной среде. Точно так же, как и при определении размера диска, он рассматривает тип хранилища, указанный в свойствах оценки (HDD (цен. категория "Стандартный")/SSD/Premium), и рекомендует тип диска соответствующим образом. Типом хранилища по умолчанию являются диски уровня "Премиум".

## <a name="confidence-ratings"></a>Оценки достоверности
Каждая оценка на основе производительности в службе "миграция Azure" связана с рейтингом достоверности, который находится в диапазоне от одного (самого низкого) до пяти запусков (высший приоритет).
- Оценка достоверности назначается на основе доступности точек данных, необходимых для вычисления оценки.
- Оценка достоверности помогает определить надежность рекомендаций по выбору размера, предоставленных службой "Миграция Azure".
- Оценка достоверности неприменима для локальных оценок.
- Для изменения размера на основе производительности в Azure необходимо выполнить оценку сервера:
    - Данные об использовании ЦП и памяти виртуальной машины.
    - Кроме того, для каждого диска, присоединенного к виртуальной машине, необходимы сведения о количестве операций ввода-вывода в секунду и пропускной способности диска.
    - Аналогичным образом для каждого сетевого адаптера, подключенного к виртуальной машине, для оценки достоверности требуется сетевая операция ввода-вывода для определения размера на основе производительности.
    - Если любое из указанных выше номеров использования недоступно в vCenter Server, рекомендация по размеру может быть ненадежной.

В зависимости от процента доступных точек данных оценка достоверности может быть такой:

   **Уровень доступности точек данных** | **Оценка достоверности**
   --- | ---
   0–20 % | 1 звезда
   21–40 % | 2 звезды
   41–60 % | 3 звезды
   61–80 % | 4 звезды
   81–100 % | 5 звезд

### <a name="low-confidence-ratings"></a>Низкая оценка достоверности

Ниже приводится несколько причин, по которым оценка может получить низкую степень достоверности:

- Вы не проделали Профилирование среды в течение времени, в течение которого вы создаете оценку. Например, если вы создаете оценку с длительностью производительности в один день, необходимо подождать по крайней мере в день после начала обнаружения всех точек данных для сбора.
- Некоторые виртуальные машины были отключены в течение периода, для которого вычисляется оценка. Если какие бы то ни было виртуальные машины были отключены в течение некоторого времени, Оценка сервера "миграция Azure" не сможет собираются данные производительности за этот период.
- Некоторые виртуальные машины были созданы в промежутке между периодом, для которого вычисляется оценка. Например, если вы создаете оценку для журнала производительности за последний месяц, но некоторые виртуальные машины были созданы в среде только неделю назад, журнал производительности новых виртуальных машин не будет находиться в течение всей длительности.

> [!NOTE]
> Если оценка достоверности какой-либо оценки ниже пяти звезд, рекомендуется подождать по крайней мере один день, чтобы устройство было пропрофилировать среду, а затем повторно вычислить оценку. В противном случае размер на основе производительности может быть ненадежным, и мы рекомендуем переключить оценку на использование в качестве локального размера.

## <a name="monthly-cost-estimation"></a>Примерная ежемесячная стоимость

После выполнения рекомендаций по изменению размера служба "миграция Azure" вычисляет затраты на вычисления и хранение после миграции.

- **Стоимость вычислений**. Служба "Миграция Azure" использует API выставления счетов, чтобы на основании рекомендуемого размера виртуальной машины Azure вычислить ежемесячные затраты на виртуальную машину.
    - При этом учитывается операционная система, программа Software Assurance, зарезервированные экземпляры, время доступности виртуальных машин, расположение и параметры валюты.
    - Служба суммирует затраты на все компьютеры, чтобы рассчитать общие ежемесячные затраты на вычисления.
- **Стоимость хранения**. Ежемесячные затраты на хранение на компьютере рассчитываются путем суммирования ежемесячных затрат на все диски, подключенные к компьютеру.
    - Оценка сервера "миграция Azure" вычисляет общие ежемесячные затраты на хранение за счет статистической обработки затрат на хранение на всех компьютерах.
    - В настоящее время при вычислении не учитываются предложения, указанные в параметрах оценки.

Цены отображаются в валюте, заданной в настройках оценки.

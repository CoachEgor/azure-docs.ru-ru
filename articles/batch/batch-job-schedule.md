# <a name="schedule-jobs-for-efficiency"></a>Расписание заданий для повышения эффективности

Планирование заданий пакета позволяет определить приоритеты заданий, которые вы хотите выполнить в первую очередь, принимая во внимание задачи, которые зависят от других задач. Планируя работу, вы можете убедиться, что вы используете наименьшее количество ресурсов. Узлы могут быть выведены из эксплуатации, когда это не нужно, задачи, зависящие от других задач, закрутятся как раз во время оптимизации рабочих процессов. Работает только одна работа. Новый не начнется до тех пор, пока не завершится предыдущий. Вы можете настроить свою работу на автозаполнение. 

## <a name="benefit-of-job-scheduling"></a>Преимущество планирования заданий

Преимущество планирования заданий заключается в том, что можно указать график создания рабочих мест. Задачи, которые вы планируете выполнять с помощью задачи менеджера по работе, связаны с заданием. Задача менеджера по работе будет создавать задачи для задания. Для этого задача менеджера по работе с заданием должна быть проверена с помощью учетной записи Пакета. Используйте маркер доступа AZ_BATCH_AUTHENTICATION_TOKEN. Токен позволит получить доступ к остальной части задания. 

## <a name="use-the-portal-to-schedule-a-job"></a>Используйте портал для планирования задания

   1. Войти на [портал Azure](https://portal.azure.com/).

   2. Выберите учетную запись Пакета, в ней вы хотите запланировать вакансии.

   3. Выберите **Добавить,** чтобы создать новый график работы и заполнить **базовую форму.**



![Расписание работы][1]

**Идентификатор расписания работы**: Уникальный идентификатор для этого графика работы.

**Название дисплея**: Имя дисплея для задания не должно быть уникальным, но имеет максимальную длину 1024 символов.

**Не запускайте до :** Указывает самое раннее время выполнения задания. Если вы не установите это, график становится готовым к запуску заданий немедленно.

**Не запускайте после:** Нет рабочих мест работать после времени, установленное здесь. Если вы не указываете время, то вы создаете повторяющийся график работы, который остается активным до тех пор, пока вы явно прекратить его.

**Интервал повторения**: Вы можете указать количество времени между заданиями. Вы можете иметь только одно задание в назначенное время, так что если пришло время создать новое задание в соответствии с графиком работы, но предыдущее задание все еще работает, служба пакета не создаст новое задание до окончания предыдущей работы.  

**Стартовое окно**: Здесь вы указываете временной интервал, начиная с времени, указанного в расписании, задание должно быть создано, пока оно не должно быть завершено. Если текущее задание не выполняется во время его окна, следующее задание не начинается.

В нижней части базовой формы вы укажете пул, на котором вы хотите, чтобы задание было запущено. Чтобы найти идентификатор пула, выберите **обновление**. 

![Указать пул][2]


**Идентификатор пула:** Определите пул, на который вы запустите задание.

**Задача конфигурации**заданий: Выберите **обновление,** чтобы назвать задачу менеджера по работе, а также задачи подготовки и выпуска работы, если вы используете их.

**Приоритет**: Дайте задание приоритет.

**Max настенные часы :** Установите максимальное количество времени, задания может работать. Если он не выполняется в течение определенного периода времени, пакет завершает задание. Если вы не установите это, то нет никаких ограничений по времени для работы.

**Количество повторений задач —** Укажите количество повторений задачи максимум четыре раза. Это не то же самое, что число повторов вызова API может быть.

**Когда все задачи завершены**: По умолчанию нет действия.

**При сбою задачи:** По умолчанию не является действием. Задача завершается неудачей, если количество повторений исчерпано или при запуске задачи произошла ошибка. 

После выбора **Сохранить,** если вы идете в **графики работы** в левой навигации, вы можете отслеживать выполнение задания, выбрав **информацию о выполнении**.

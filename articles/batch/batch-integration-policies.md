# <a name="integration-with-azure-policy"></a>Интеграция с политикой Azure

Политика Azure — это служба в Azure, используемая для создания, присвоения и управления политиками, которые обеспечивают соблюдение правил ресурсов, чтобы гарантировать, что эти ресурсы будут соответствовать корпоративным стандартам и соглашениям об уровне обслуживания. Политика Azure оценивает ресурсы за несоблюдение присваиваемых политик. 

Azure Batch имеет два встроенных расширения, которые помогут вам управлять соответствием требованиям политики. 

|**Имя**...|   **Описание**|**Действие**|  **Версия**|    **Источник**
|----------------|----------|----------|----------------|---------------|
|Необходимо включить журналы диагностики в учетных записях пакетной службы|   Аудит активации журналов диагностики. Это позволит воссоздать действия для расследования в случае инцидентов безопасности или компрометации сети.|AuditIfNotExists, Disabled|  2.0.0|  GitHub|
|В учетных записях пакетной службы должны быть настроены правила генерации оповещений по метрикам| Аудит настройки правил оповещений по метрикам в учетной записи пакетной службы для активации нужной метрики|   AuditIfNotExists, Disabled| 1.0.0|  GitHub|

Определения политики описывают условия, которые необходимо выйти на себя. Условие сравнивает свойство ресурса с требуемым значением. Поля свойств ресурсов доступны с помощью заранее определенных псевдонимов. Для доступа к определенным свойствам для типа ресурса используются псевдонимы свойств. Псевдонимы позволяют ограничить значения или условия, разрешенные для свойства ресурса. Каждый псевдоним сопоставляется с путями в разных версиях API для заданного типа ресурса. Во время оценки политики модуль политики получает путь свойства для этой версии API.

Ресурсы, требуемые Batch, включают: учетную запись, вычислительный узлы, пул, задание и задачу. Таким образом, можно использовать псевдонимы свойств для доступа к определенным свойствам для этих ресурсов. Узнайте больше о [Aliases](https://docs.microsoft.com/azure/governance/policy/concepts/definition-structure#aliases)

Чтобы убедиться в том, что вы знаете текущие псевдонимы и просматриваете свои ресурсы и политики, используйте расширение политики Azure для Visual Studio Code. Он может быть установлен на всех платформах, которые поддерживаются Visual Studio Code. Эта поддержка включает windows, Linux и macOS. Смотрите [инструкции по установке](https://docs.microsoft.com/azure/governance/policy/how-to/extension-for-vscode).

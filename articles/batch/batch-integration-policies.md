# <a name="integration-with-azure-policy"></a>Интеграция со службой "политика Azure"

Политика Azure — это служба в Azure, которая используется для создания, назначения и управления политиками, которые применяют правила к ресурсам, чтобы обеспечить соответствие этих ресурсов корпоративным стандартам и соглашениям об уровне обслуживания. Политика Azure оценивает ресурсы на несоответствие политикам, которые вы назначаете. 

Пакетная служба Azure имеет два встроенных расширения, которые помогают управлять соответствием политике. 

|**Имя**...|   **Описание**|**Действие**|  **Версия**|    **Источник**
|----------------|----------|----------|----------------|---------------|
|Необходимо включить журналы диагностики в учетных записях пакетной службы|   Аудит активации журналов диагностики. Это позволит воссоздать действия для расследования в случае инцидентов безопасности или компрометации сети.|AuditIfNotExists, Disabled|  2.0.0|  GitHub|
|В учетных записях пакетной службы должны быть настроены правила генерации оповещений по метрикам| Аудит настройки правил оповещений по метрикам в учетной записи пакетной службы для активации нужной метрики|   AuditIfNotExists, Disabled| 1.0.0|  GitHub|

Определения политик описывают условия, которые должны быть удовлетворены. Условие сравнивает свойство ресурса с требуемым значением. Доступ к полям свойств ресурсов осуществляется с помощью предварительно определенных псевдонимов. Для доступа к конкретным свойствам типа ресурса используются псевдонимы свойств. Псевдонимы позволяют ограничить значения или условия, разрешенные для свойства ресурса. Каждый псевдоним сопоставляется с путями в разных версиях API для заданного типа ресурса. Во время оценки политики модуль политики получает путь свойства для этой версии API.

Ресурсы, необходимые для пакетной службы: учетная запись, расчетный узел, пул, задание и задача. Поэтому для доступа к конкретным свойствам этих ресурсов следует использовать псевдонимы свойств. Дополнительные сведения о [псевдонимах](https://docs.microsoft.com/azure/governance/policy/concepts/definition-structure#aliases)

Чтобы убедиться, что вы знаете текущие псевдонимы и просматриваете ресурсы и политики, используйте расширение политики Azure для Visual Studio Code. Его можно установить на всех платформах, которые поддерживаются Visual Studio Code. Эта поддержка включает в себя Windows, Linux и macOS. См. [руководство по установке](https://docs.microsoft.com/azure/governance/policy/how-to/extension-for-vscode).

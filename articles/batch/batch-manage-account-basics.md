# <a name="manage-your-batch-account"></a>Управление учетной записью пакета

Учетная запись Пакетной службы — это уникально идентифицируемая сущность в Пакетной службе. Вся обработка данных привязана к учетной записи пакетной службы.

Вы можете создать учетную запись пакетной службы Azure с помощью [портала](batch-account-create-portal.md) или программными средствами, например с помощью [библиотеки. NET управления пакетной службой](batch-management-dotnet.md). При создании учетной записи можно связать учетную запись хранения Azure для хранения связанных с заданием входных и выходных данных или приложений.

Можно запустить несколько рабочих нагрузок пакетной службы в одной учетной записи пакетной службы. Либо распределить рабочие нагрузки между учетными записями пакетной службы в рамках одной подписки, но в разных регионах Azure.

## <a name="components-of-the-batch-account"></a>Компоненты учетной записи пакета

Учетная запись Batch позволяет эффективно выполнять крупномасштабные параллельные и высокопроизводительные вычислительные (HPC) пакетные задания в Azure. В учетной записи вы управляете:

- Приложения, которые вы работаете

- Распределение бассейнов и узлов в бассейнах

- Количество и типы задач 

- Ввод и выход данных. Вам не нужно устанавливать дополнительное программное обеспечение для управления задачами.

- При создании учетной записи Пакета предлагается присвоить ей имя. Это имя является его идентификатором и после присвоения не может быть изменено.

- Чтобы изменить название учетной записи, необходимо удалить ее и создать новую учетную запись Batch.

- Учетная запись создается в подписке, которая вы хотите использовать.

- Используйте учетную запись для идентификации и извлечения первичных и вторичных ключей учетной записи из любой учетной записи Batch в вашей подписке.

- Счет поддерживает информацию о распределении пула и основных квотах.  

- Учетная запись содержит информацию о местоположении.

- Учетная запись идентифицирует учетную запись хранилища.

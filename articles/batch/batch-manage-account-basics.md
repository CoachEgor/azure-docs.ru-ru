# <a name="manage-your-batch-account"></a>Управление учетной записью пакетной службы

Учетная запись Пакетной службы — это уникально идентифицируемая сущность в Пакетной службе. Вся обработка данных привязана к учетной записи пакетной службы.

Вы можете создать учетную запись пакетной службы Azure с помощью [портала](batch-account-create-portal.md) или программными средствами, например с помощью [библиотеки. NET управления пакетной службой](batch-management-dotnet.md). При создании учетной записи можно связать учетную запись хранения Azure для хранения связанных с заданием входных и выходных данных или приложений.

Можно запустить несколько рабочих нагрузок пакетной службы в одной учетной записи пакетной службы. Либо распределить рабочие нагрузки между учетными записями пакетной службы в рамках одной подписки, но в разных регионах Azure.

## <a name="components-of-the-batch-account"></a>Компоненты учетной записи пакетной службы

Учетная запись пакетной службы позволяет эффективно выполнять в Azure пакетные задания для крупномасштабных параллельных и высокопроизводительных вычислений (HPC). В учетной записи вы можете управлять следующим.

- Приложения, которые вы запускаете.

- Распределение пулов и узлов в них.

- Число задач и их типы. 

- Ввод и вывод данных. Для управления задачами не требуется устанавливать дополнительное программное обеспечение.

- При создании учетной записи пакетной службы вам будет предложено назначить ей имя. Это имя является идентификатором, и после назначения его невозможно изменить.

- Чтобы изменить имя учетной записи, нужно удалить ее и создать новую учетную запись пакетной службы.

- Учетная запись создается в подписке, которую вы хотите использовать.

- Используйте эту учетную запись для определения и получения первичного и вторичного ключей из любой учетной записи пакетной службы в своей подписке.

- Учетная запись содержит сведения о выделении пулов и квотах ядер.  

- Учетная запись содержит сведения о расположении.

- Учетная запись идентифицирует вашу учетную запись хранения.

# <a name="build-mobile-back-end-components-with-compute-services"></a>Создание мобильных бэк-энд-компонентов с помощью вычислительных служб
Каждое мобильное приложение нуждается в задней части, которая отвечает за хранение данных, бизнес-логику и безопасность. Управление инфраструктурой для размещения и выполнения кода бэк-энда требует масштабирования, обеспечения и масштабирования нескольких серверов. Вы также должны управлять обновлениями ОС и аппаратного обеспечения и применять патчи безопасности. Затем необходимо отслеживать все эти компоненты инфраструктуры для производительности, доступности и допусков. 

Архитектура без серверов пригодится для такого типа сценария, потому что у вас нет серверов для управления и нет ОС или связанных с ними программных или аппаратных обновлений для управления. Архитектура без серверов экономит время и затраты разработчика, что означает более быстрое время на продажу и сосредоточенную энергию на создании приложений.

## <a name="benefits-of-compute"></a>Преимущества вычислений
- Абстракция серверов означает, что нет необходимости беспокоиться о хостинге, исправлении и безопасности, что позволяет сосредоточиться исключительно на коде.
- Мгновенное и эффективное масштабирование гарантирует автоматическое или по требованию ресурсы в любом масштабе.
- Высокая доступность и допуск неисправностей.
- Микро-биллинг гарантирует, что вы только счет, когда ваш код на самом деле работает.
- Код работает в облаке, написанном на языке по вашему выбору.

Используйте следующие службы для обеспечения возможностей бессерверных вычислений в мобильных приложениях.

## <a name="azure-functions"></a>Проверка
[Azure Functions](https://azure.microsoft.com/services/functions/) — это управляемый событиями вычислительный опыт, который можно использовать для выполнения кода, написанного на языке программирования по вашему выбору, не беспокоясь о серверах. Для его запуска не нужно управлять приложением или инфраструктурой. Масштаб функций масштабируется по требованию, и вы платите только за время запуска кода. Функции Azure являются отличным способом реализации API для мобильного приложения. Они просты в реализации и обслуживании и доступны через HTTP.

**Основные функции**
- Управляемые событиями и масштабируемые, где можно использовать триггеры и привязки для определения того, когда вызывается функция и к каким данным она подключается.
- Принесите свои собственные зависимости, потому что функции поддерживают NuGet и NPM, так что вы можете использовать ваши любимые библиотеки.
- Интегрированная безопасность, чтобы вы могли защитить функции HTTP-триггера с помощью таких поставщиков OAuth, как Azure Active Directory, Facebook, Google, Twitter и Учетная запись Майкрософт.
- Упрощенная интеграция с различными службами и программным обеспечением [Azure](/azure/azure-functions/functions-overview) в качестве услуг (SaaS).
- Гибкая разработка, чтобы вы могли кодировать свои функции прямо на портале Azure или настроить непрерывную интеграцию и развернуть свой код через GitHub, Azure DevOps Services и другие поддерживаемые средства разработки.
- Время выполнения функций с открытым исходным кодом и доступно на [GitHub.](https://github.com/azure/azure-webjobs-sdk-script)
- Улучшенный опыт разработки, где вы можете кодировать, тестировать и отладить локально, используя предпочитаемый редактор или простой в использовании веб-интерфейс с мониторингом с интегрированными инструментами и встроенными возможностями DevOps.
- Разнообразие языков программирования и вариантов хостинга для разработки, таких как C,, Node.js, Java, JavaScript или Python.
- Модель ценообразования с оплатой за использование означает, что вы платите только за время, затраченное на запуск кода.

## <a name="azure-app-service"></a>Служба приложений Azure
С [помощью Azure App Service](https://azure.microsoft.com/services/app-service/)вы можете создавать и размещать веб-приложения и RESTful AI на языке программирования по вашему выбору без управления инфраструктурой. Он предлагает автоматизацию и высокую доступность, поддерживает как Windows, так и Linux, а также позволяет автоматически развертывать gitHub, Azure DevOps или любое репо Git.

**Основные функции**
- Поддержка нескольких языков и рамок для ASP.NET, ASP.NET Core, Java, Ruby, Node.js, PHP или Python. Кроме того, вы можете запускать PowerShell и другие скрипты или исполняемые файлы в качестве фоновых служб.
- Оптимизация DevOps путем непрерывной интеграции и развертывания с помощью Azure DevOps, GitHub, BitBucket, Docker Hub или реестра контейнеров Azure. Управляйте приложениями в Службе приложений Azure с помощью оболочки Azure PowerShell или кроссплатформенного интерфейса командной строки (CLI).
- Глобальный масштаб с высокой доступностью для масштабирования или выхода вручную или автоматически.
- Подключение к платформам SaaS и данные на основе предварительных данных на выбор из более чем 50 разъемов для корпоративных систем, таких как SAP, SaaS услуг, таких как Salesforce, и интернет-сервисов, таких как Facebook. Доступ к данным с помощью гибридных подключений и виртуальных сетей Azure.
- Служба приложений Azure соответствует требованиям ISO, SOC и PCI. Аутентифицируйте пользователей с помощью Active Directory Azure или входящих в социальные сети, такие как Google, Facebook, Twitter и Microsoft. Создавайте ограничения IP-адресов и управляйте удостоверениями службы.
- Шаблоны приложений на выбор из обширного списка шаблонов приложений в Azure Marketplace, таких как WordPress, Joomla и Drupal.
- Интеграция Visual Studio с помощью специальных инструментов в Visual Studio упрощает работу по созданию, развертыванию и отладке.

## <a name="azure-kubernetes-service"></a>Служба Azure Kubernetes
[Служба Azure Kubernetes (AKS)](https://azure.microsoft.com/services/kubernetes-service/) управляет размещенной средой Kubernetes. AKS позволяет быстро и легко развернуть и администрировать контейнерные приложения даже без опыта в оркестрации контейнеров. Это также устраняет бремя текущих операций и технического обслуживания. Положения, обновления и масштабирование ресурсов по требованию AKS, не включив приложения в автономный режим.

**Основные функции**
- Легко переносите существующие приложения в контейнеры и запускаваемые в рамках AKS.
- Упростить развертывание и управление приложениями на основе микрослужб.
- Безопасные DevOps для AKS для достижения баланса между скоростью и безопасностью и доставки кода быстрее в масштабе.
- Масштабируйте с легкостью, используя AKS и Azure Container Instances для предоставления стручков внутри контейнерных инстанций, которые начинаются в считанные секунды.
- Развертывание и управление устройствами IoT по требованию.
- Обучить модели машинного обучения с использованием таких инструментов, как TensorFlow и KubeFlow.

## <a name="azure-container-instances"></a>Экземпляры контейнеров Azure
[Azure Container Instances](https://azure.microsoft.com/services/container-instances/) — это отличное решение для любого сценария, который может работать в изолированных контейнерах, таких как простые приложения, автоматизация задач и создание заданий. Быстро разрабатывайте приложения без управления vMs.

**Основные функции**
- Быстрые время запуска контейнеров, поскольку контейнеры могут запускаться в Azure за считанные секунды, без необходимости предоставления и управления ВМ.
- Публичное подключение IP и пользовательское имя DNS.
- Безопасность уровня Hypervisor, которая гарантирует ваше приложение, так же изолирована в контейнере, как и в VM.
- Пользовательские размеры для оптимального использования, позволяя точные спецификации ядер процессора и памяти. Оплата зависит от требуемых ресурсов и начисляется посекундно, позволяя вам оптимизировать расходы в соответствии с фактическими потребностями.
- Постоянное хранение для извлечения и сохранения состояния. Контейнерные инстанции предлагают прямое монтаж акций Azure Files.
- Контейнеры Linux и Windows запланированы с тем же API.

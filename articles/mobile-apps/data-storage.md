# <a name="store-sync-and-query-mobile-application-data-from-the-cloud"></a>Хранение, синхронизация и запрос данных мобильных приложений из облака
Независимо от типа создаваемого приложения, скорее всего, будут созданы и обработаны данные. У пользователей вашего приложения есть высокие ожидания. Они хотят, чтобы приложение работало быстро и без проблем во всех обстоятельствах. Большинство приложений также работают на нескольких устройствах. Вы можете получить доступ к приложению с настольного или мобильного устройства. Несколько пользователей могут одновременно использовать приложение и обмениваться данными с ожиданием мгновенного и доступного в режиме реального времени доступа к данным.

Пользователи приложения не всегда будут иметь подключение к Интернету. Приложения спроектированы и должны работать с Интернетом или без подключения к Интернету. Разработчики должны выбрать подходящее решение для хранения и синхронизации данных в облаке, чтобы обеспечить превосходный пользовательский интерфейс для своего приложения.

Корпорация Майкрософт предоставляет разнообразные службы, которые устраняют необходимость в выпуске серверов, выборе базы данных или изменении масштаба или безопасности, чтобы обеспечить максимально широкие возможности. Эти службы предоставляют прекрасную среду для разработчиков, которая позволяет хранить данные приложений в облаке с помощью API-интерфейсов SQL или NoSQL. Вы также можете синхронизировать данные на всех устройствах и позволить приложению работать с сетевым подключением или без него, чтобы помочь создавать масштабируемые и надежные приложения.

Используйте следующие службы для управления и хранения данных мобильных приложений в облаке.

## <a name="visual-studio-app-center"></a>Центр приложений Visual Studio
[Данные центра приложений](/appcenter/data/) — это служба управления данными. Это позволяет приложениям управлять, сохранять и синхронизировать данные приложений в облаке на различных устройствах и платформах в интерактивных и автономных сценариях. На основе данных, созданных на основе Azure Cosmos DB, данные центра приложений масштабируются по мере роста пользовательской базы и количества приложений. Это обеспечивает низкую задержку, высокую доступность и высокую масштабируемость для всех данных.

**Основные возможности**
- Простая подготовка к новой Azure Cosmos DB базе данных или существующей Azure Cosmos DB базе данных с портала центра приложений Visual Studio.
- Поддержка баз данных NoSQL для удобного хранения, синхронизации и запроса данных приложения.
- Эта служба, построенная на основе Azure Cosmos DB, наследует большинство основных функций, предлагаемых Azure Cosmos DB и может масштабироваться глобально в соответствии с потребностями бизнеса.
- Возможности оперативной и автономной синхронизации для синхронизации данных между устройствами.
- Пакеты SDK для мобильных клиентов, которые можно использовать для упрощения управления конфиденциальными данными приложений.
- Поддержка платформ для iOS, Android, Xamarin и реагирования на машинный код.

## <a name="azure-cosmos-db"></a>Azure Cosmos DB
[Azure Cosmos DB](https://azure.microsoft.com/services/cosmos-db/) — это многомодельная глобально распределенная служба баз данных. Его можно использовать для создания приложений в масштабе планеты. С Azure Cosmos DB можно гибко и независимо масштабировать пропускную способность и хранилище в любом количестве регионов Azure по всему миру. Вы можете воспользоваться преимуществами быстрого доступа к данным с одной цифрой миллисекунды с помощью предпочтительных поверхностей API. К этим областям относятся SQL, MongoDB, Cassandra, Tables или Gremlin. Azure Cosmos DB уникально предоставляет полные соглашения об уровне обслуживания (SLA) для пропускной способности, задержки, доступности и согласованности.

**Основные возможности**
- Поддерживает широкий спектр API-интерфейсов, включая API SQL (ядро), API Cassandra, API MongoDB, Gremlin API и API таблиц.
- При глобальном распределении данные реплицируются везде, где находятся ваши пользователи. Пользователи могут взаимодействовать с репликой данных, которые находятся ближе к ним.
- Нет управления схемой или индексом, так как ядро СУБД полностью не зависит от схемы.
- Общедоступное региональное присутствие, так как Azure Cosmos DB доступен во всех регионах Azure по всему миру, в том числе 54 и более.
- Точное определение нескольких вариантов согласованности, так как протокол репликации Azure Cosmos DB с несколькими хозяевами тщательно спроектирован с целью предоставления пяти четко определенных вариантов согласованности. Эти пять вариантов являются строгим, ограниченным устареваниями, сеансом, согласованным префиксом и, в конечном итоге,.
- 99,999% доступности для операций чтения и записи.
- Программно (или с помощью портал Azure) вызовите региональную отработку отказа учетной записи Azure Cosmos DB, чтобы убедиться в том, что ваше приложение предназначено для того, чтобы оно было недоступно
- Гарантируется низкая задержка на 99-м процентиль во всем мире.

## <a name="azure-sql-database"></a>База данных SQL Azure
 [База данных SQL Azure](https://azure.microsoft.com/services/sql-database/) — это управляемая служба реляционной базы данных общего назначения. Его можно использовать для создания высокодоступного и высокопроизводительного уровня хранилища данных для приложений и решений в облаке Azure.

**Основные возможности**
- **Модели и инструменты эластичной базы данных:** Благодаря эластичной базе данных разработчики могут распределять ресурсы по группам баз данных для масштабирования. Для административного управления этими ресурсами вы отправляете сценарий как задание. Затем база данных SQL выполняет скрипт в базах данных.
- **Высокая производительность.** Приложения с высокой пропускной способностью могут использовать преимущества последней версии. Она обеспечивает более 25% большей мощности расширенной базы данных.
- **Резервное копирование, репликация и высокий уровень доступности:** Встроенная репликация и соглашение об уровне обслуживания Майкрософт на уровне базы данных обеспечивают непрерывность работы приложений и защиту от разрушительных событий. Активная Георепликация позволяет настроить отработку отказа и самостоятельное восстановление, что обеспечивает полный контроль над восстановлением «ой» реплики. Восстановление данных доступно из резервных копий данных до 35 дней.
- **Почти ненулевое обслуживание:** Автоматическое программное обеспечение является частью службы. Встроенные системные реплики помогают обеспечить встроенную защиту данных, бесперебойную работу базы данных и стабильность системы. Системные реплики автоматически перемещаются на новые компьютеры. Они подготавливаются на лету, так как старые не завершаются сбоем.
- **Безопасность:** База данных SQL Azure предлагает портфель функций безопасности, отвечающих требованиям организации и отраслевым политикам соответствия.
    - Аудит предоставляет разработчикам возможность выполнять задачи, связанные с соответствием, и получать знания о действиях.
    - Разработчики и могут реализовать политики на уровне базы данных, чтобы ограничить доступ к конфиденциальным данным с помощью безопасности на уровне строк, динамического маскирования данных и прозрачного шифрования данных для базы данных SQL Azure.
    - База данных SQL Azure проверяется с помощью ключевых аудиторов облака в рамках основных сертификатов и утверждений Azure, таких как HIPAA BAA, ISO/IEC 27001:2005, FedRAMP и предложения модели ЕС.

# <a name="store-sync-and-query-mobile-application-data-from-the-cloud"></a>Хранение, синхронизация и запрос данных мобильных приложений из облака
Независимо от того, какое приложение вы создаете, вы, скорее всего, генерировать и обрабатывать данные. Пользователи вашего приложения возлагаем большие надежды. Они хотят, чтобы приложение работало быстро и беспрепятственно, при любых обстоятельствах. Большинство приложений также работают на нескольких устройствах. Вы можете получить доступ к приложению с рабочего или мобильного устройства. Несколько пользователей могут использовать приложение одновременно и обмениваться данными в расчете на получение мгновенного и в режиме реального времени доступа к данным.

Пользователи приложения не всегда будут иметь подключение к Интернету. Приложения разработаны и должны работать с или без подключения к Интернету. Разработчики должны выбрать правильное решение для хранения и синхронизации своих данных в облаке, чтобы обеспечить большой клиентский опыт для их приложения.

Корпорация Майкрософт предоставляет различные услуги, которые устраняют необходимость раскручивания серверов, выбора базы данных или беспокойства по поводу масштабирования или безопасности, чтобы обеспечить как можно более богатый опыт. Эти службы обеспечивают большой опыт разработчика, который позволяет хранить данные приложений в облаке с помощью AIS S'L или NoS'L. Вы также можете синхронизировать данные на всех устройствах и позволить приложению работать с подключением к сети или без нее, чтобы помочь создать масштабируемые и надежные приложения.

Используйте следующие службы для управления и хранения данных мобильных приложений в облаке.

## <a name="visual-studio-app-center"></a>Центр приложений Visual Studio
[App Center Data](/appcenter/data/) — это служба управления данными. Это позволяет приложениям управлять, упорствовать и синхронизировать данные приложений в облаке на различных устройствах и платформах в онлайновых и автономных сценариях. Построенный на вершине Azure Cosmos DB, App Center Data масштабируется по мере роста базы пользователей и количества приложений. Это обеспечивает низкую задержку, высокую доступность и высокую масштабируемость для всех ваших данных.

**Основные функции**
- Простое приподготовке к новой базе данных Azure Cosmos DB или существующей базе данных Azure Cosmos DB с портала Visual Studio App Center.
- Поддержка базы данных NoS'L для легкого хранения, синхронизации и обработки данных приложений запросов.
- Эта служба, созданная на базе Azure Cosmos DB, наследует большинство ключевых функций, предлагаемых Azure Cosmos DB, и может масштабироваться по всему миру для удовлетворения потребностей вашего бизнеса.
- Возможности синхронизации онлайн и оффлайн для синхронизации данных на устройствах.
- Мобильные клиенты SDKs, которые можно использовать для легкого управления данными частных приложений.
- Поддержка платформдлядля iOS, Android, Xamarin и React Native.

## <a name="azure-cosmos-db"></a>Azure Cosmos DB
[Azure Cosmos DB](https://azure.microsoft.com/services/cosmos-db/) — это многомодельная глобально распределенная служба баз данных. Вы можете использовать его для создания приложений масштаба планеты. С помощью Azure Cosmos DB вы можете упруго и независимо масштабировать пропускную связь и хранение в любых регионах Azure по всему миру. Вы можете воспользоваться быстрым, однозначным миллисекундным доступом к данным, используя ваши любимые поверхности API. Эти поверхности включают в себя СЗЛ, MongoDB, Кассандра, Таблицы, или Гремлин. Azure Cosmos DB однозначно предоставляет комплексные соглашения об уровне обслуживания (SL) для пропускной записи, задержки, доступности и согласованности.

**Основные функции**
- Поддерживает широкий спектр API, включающий API СЗЛ (Core), API Cassandra, API MongoDB, API Gremlin и Table API.
- Глобальное распределение под ключ воспроизводит ваши данные, где бы они ни находились. Пользователи могут взаимодействовать с репликой наиболее близких к ним данных.
- Нет схемы или управления индексом, потому что движок базы данных полностью агностик схемы.
- Повсеместное региональное присутствие, поскольку Azure Cosmos DB доступен во всех регионах Azure по всему миру, который включает в себя 54 региона в общедоступном облаке.
- Точно определены, несколько вариантов согласованности, потому что многомастерский протокол репликации Azure Cosmos DB тщательно разработан, чтобы предложить пять четко определенных вариантов согласованности. Эти пять вариантов являются сильными, ограниченные черствость, сессии, последовательной префикс, и в конечном итоге.
- 99,999% доступности для чтения и записи.
- Программно (или через портал Azure) ссылайтесь на региональный сбой вашей учетной записи Azure Cosmos DB, чтобы убедиться, что ваше приложение предназначено для того, чтобы выдерживать региональную катастрофу.
- Гарантированная низкая задержка на 99-м процентиле по всему миру.

## <a name="azure-sql-database"></a>База данных SQL Azure
 [База данных Azure S'L](https://azure.microsoft.com/services/sql-database/) — это служба общего реляционного управления базой данных общего назначения. Его можно использовать для создания высокодоступного и высокопроизводительного уровня хранения данных для приложений и решений в облаке Azure.

**Основные функции**
- **Упругие модели и инструменты баз данных:** С помощью эластичной базы данных разработчики могут объединять ресурсы между группой баз данных для масштабирования. Чтобы административно управлять этими ресурсами, вы отправляете скрипт в качестве задания. Затем база данных S'L выполняет скрипт по базам данных.
- **Высокая производительность:** Приложения с высокой пропускной стоимостью могут воспользоваться преимуществами последней версии. Он обеспечивает на 25% больше энергии базы данных премиум-класса.
- **Резервное копирование, репликация и высокая доступность:** Встроенная репликация и SLA, поддерживаемая корпорацией Майкрософт, на уровне базы данных обеспечивают непрерывность приложений и защиту от катастрофических событий. Активная георепликация позволяет настроить сбой и самообслуживание восстановления, которые обеспечивают полный контроль над "oops восстановления". Восстановление данных доступно из резервных данных до 35 дней.
- **Почти нулевое техническое обслуживание:** Автоматическое программное обеспечение является частью службы. Встроенные системные реплики помогают обеспечить неотъемлемую защиту данных, время простоя базы данных и стабильность системы. Системные реплики автоматически перемещаются на новые компьютеры. Они подготовлены на лету, как старые неудачу.
- **Безопасность:** База данных Azure S'L предлагает портфель функций безопасности для соответствия организационным или отраслевым политикам соответствия:
    - Аудит предоставляет разработчикам возможность выполнять задачи, связанные с соответствием требованиям, и получать знания о деятельности.
    - Разработчики и ИТ-специалисты могут реализовывать политики на уровне базы данных, чтобы ограничить доступ к конфиденциальным данным с помощью безопасности на уровне строки, динамической маскировки данных и прозрачного шифрования данных для базы данных Azure S'L.
    - База данных Azure S'L проверяется ключевыми аудиторами облачных технологий в рамках ключевых сертификатов и утверждений Azure, таких как HIPAA BAA, ISO/IEC 27001:2005, FedRAMP и Модельные статьи ЕС.

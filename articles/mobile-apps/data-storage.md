# <a name="store-sync-and-query-mobile-application-data-from-the-cloud"></a>Хранение, синхронизация и запрос данных мобильных приложений из облака
Независимо от типа создаваемого приложения, скорее всего, будут создаваться и обрабатываться данные. Ожидание пользователей велико, и они хотят, чтобы приложение работало быстро и без проблем во всех обстоятельствах. Большинство приложений работают на нескольких устройствах, поэтому при доступе к приложению доступ к нему можно получить с настольного или мобильного устройства. Несколько пользователей могут одновременно использовать приложение и обмениваться данными с ожиданием мгновенного и доступного в режиме реального времени доступа к данным.

Поверх этого, пользователи приложения не всегда будут иметь подключение к Интернету. приложения спроектированы и должны работать с Интернетом или без подключения к Интернету. Все эти растущие сложности позволяют разработчикам выбрать подходящее решение для хранения и синхронизации данных в облаке, чтобы обеспечить превосходную работу пользователей с их приложениями.

Корпорация Майкрософт предоставляет разнообразные службы, которые устраняют необходимость в выпуске серверов, выборе базы данных или изменении масштаба или безопасности, чтобы обеспечить максимально широкие возможности. Эти службы предоставляют прекрасную среду для разработчиков, которая позволяет хранить данные приложений в облаке с помощью API-интерфейсов SQL или NoSQL, синхронизировать данные на всех устройствах и обеспечить работу приложения с сетевым подключением или без него, чтобы обеспечить масштабируемость и надежность. приложений.

Используйте следующие службы для управления и хранения данных мобильных приложений в облаке.

## <a name="visual-studio-app-center"></a>Центр приложений Visual Studio.
[Данные центра приложений](/appcenter/data/) — это служба управления данными, которая позволяет приложениям управлять, сохранять и синхронизировать данные приложений в облаке на различных устройствах и платформах как в интерактивном, так и в автономном режиме. Эта служба, **построенная на** основе службы Cosmos DB, которая масштабируется по мере роста пользовательской базы и количества приложений, обеспечивает низкую задержку, высокую доступность и высокую масштабируемость всех данных.

**Основные возможности**
- Легко **подготавливайте новую базу данных Cosmos DB** или **Подключайтесь к существующей Cosmos DB базе данных** с портала центра приложений.
- **Поддержка баз данных NoSQL** для удобного хранения, синхронизации и запроса данных приложения.
- Эта служба, **построенная на основе Cosmos DB**, наследует большинство основных функций, предлагаемых Azure Cosmos DB и может **масштабироваться глобально** в соответствии с потребностями бизнеса.
- Возможности **оперативной и автономной синхронизации** для синхронизации данных между устройствами.
- **Пакеты SDK** для мобильных клиентов, позволяющие легко управлять данными частного приложения.
- **Поддержка платформ** — iOS, Android, Xamarin, реагирующая собственная.

## <a name="azure-cosmos-db"></a>Azure Cosmos DB;
[Azure Cosmos DB](https://azure.microsoft.com/services/cosmos-db/) — это глобально распределенная многомодельная служба базы данных Майкрософт, которая помогает создавать приложения для мирового масштаба. С помощью одного лишь нажатия кнопки Cosmos DB позволяет гибко и независимо масштабировать пропускную способность и ресурсы хранилища в любых регионах Azure по всему миру. Вы можете эластично масштабировать пропускную способность и хранилище и использовать преимущества быстрого доступа к данным с одной цифрой миллисекунды с помощью предпочтительных поверхностей API, включая SQL, MongoDB, Cassandra, таблицы или Gremlin. Cosmos DB уникально предоставляет полные соглашения об уровне обслуживания (SLA) для пропускной способности, задержки, доступности и согласованности.

**Основные возможности**
- Поддерживает **широкий спектр API** -интерфейсов, в том числе API SQL (ядро), API Cassandra, API MongoDB, Gremlin api и API таблиц.
- При **глобальном распределении** данные реплицируются везде, где находятся пользователи, поэтому пользователи могут взаимодействовать с репликой данных, которые находятся ближе к ним.
- **Отсутствие управления схемой или индексом** , так как ядро СУБД полностью не зависит от схемы.
- Общедоступное **региональное присутствие** в качестве Cosmos DB доступно во всех регионах Azure по всему миру, включая 54 + регионы в общедоступном облаке.
- **Точнее, несколько вариантов согласованности,** как и протокол репликации с несколькими хозяевами Cosmos DB, тщательно спроектированы с целью предоставления пяти четко определенных вариантов согласованности — strong, ограниченного устаревания, сеанса, согласованного префикса и, наконец, в конечном итоге.
- **99,999% доступности** для операций чтения и записи.
- **Программно (или с помощью портала) вызовите региональную отработку отказа** учетной записи Cosmos, чтобы убедиться в том, что ваше приложение предназначено для того, чтобы обеспечить устойчивость к
- **Гарантированная низкая задержка на 99-м процентиль, по**всему миру.

## <a name="azure-sql-database"></a>Базы данных SQL Azure
 [База данных SQL Azure](https://azure.microsoft.com/services/sql-database/) — это управляемая служба реляционной базы данных общего назначения, которая позволяет создавать высокодоступные и высокопроизводительные уровни хранилища данных для приложений и решений в облаке Azure.

**Основные возможности**
- **Модели и инструменты эластичной базы данных** . эластичная база данных позволяет разработчикам создавать пулы ресурсов для масштабирования, которые затем могут управляться административно, отправляя сценарий как задание, и база данных SQL выполняет Создание скриптов для баз данных.
- **Высокопроизводительные** приложения с высокой пропускной способностью могут воспользоваться преимуществами последней версии, что обеспечивает 25-процентную мощность базы данных Premium.
- **Резервное копирование, репликация и высокая доступность** . Встроенная репликация и соглашение об уровне обслуживания Майкрософт на уровне базы данных обеспечивают непрерывность приложений и защиту от разрушительных событий. Активная Георепликация позволяет настроить отработку отказа и самостоятельное восстановление, обеспечивающее полный контроль над "неудачной архивацией" (восстановление данных из доступных резервных копий данных до 35 дней).
- **Почти нулевое обслуживание** — автоматическое программное обеспечение является частью службы, а встроенные системные реплики помогают обеспечить безопасность данных, бесперебойную работу базы данных и стабильность системы. Системные реплики автоматически перемещаются на новые компьютеры, которые подготавливаются к работе в динамическом режиме по мере выхода старых машин из строя.
- **Безопасность** . база данных SQL предлагает портфель функций безопасности, отвечающих требованиям организации и отраслевым политикам соответствия.
    - Аудит предоставляет разработчикам возможность выполнять задачи, связанные с соответствием, и получать знания о действиях.
    - Разработчики и могут реализовать политики на уровне базы данных, чтобы ограничить доступ к конфиденциальным данным с помощью безопасности на уровне строк, динамического маскирования данных и прозрачного шифрования данных для базы данных SQL Azure.
    - База данных SQL проверяется с помощью ключевых аудиторов облака в рамках основных сертификатов и утверждений Azure, таких как HIPAA BAA, ISO/IEC 27001:2005, FedRAMP и предложения модели ЕС.

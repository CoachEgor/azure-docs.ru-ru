# <a name="different-stages-in-mobile-application-development"></a>Различные этапы разработки мобильных приложений
Создание мобильного приложения включает в себя несколько этапов. Вы можете легко воспользоваться услугами, инструментами и технологиями корпорации Майкрософт для создания высококлассных мобильных приложений.

## <a name="app-development"></a>Разработка приложений
Вы можете создавать нативные одноплатформенные клиентские приложения с такими языками, как Objective-C и Java. Вы можете создавать кросс-платформенные приложения с помощью Xamarin, React Native и Unity. Вы также можете создавать гибридные приложения с помощью Cordova. Для построения клиентских приложений используйте IDEs и редакторы кода, такие как Visual Studio, Visual Studio для Mac или Visual Studio Code на ПК и Mac.

## <a name="plan-and-design-with-back-end-services"></a>Планирование и проектирование с помощью бэк-энд-сервисов
 Корпорация Майкрософт предоставляет различные услуги по настройке безопасного, масштабируемого и надежного задней части для вашего мобильного приложения. Вы можете создавать без серверные AA и триггеры без необходимости поддерживать свои собственные серверы. Используйте push-уведомления, синхронизацию данных в автономном режиме, аутентификацию и службы передачи данных для формирования всеобъемлющего и обширного мобильного бэк-энда, который поддерживает различные сценарии построения приложений. 
 
 Создание бэк-энд-сервиса имеет много требований. Для этого необходимо:
   - Ограничьте хранение данных на устройстве.
   - Синхронизация данных на нескольких устройствах.
   - Убедитесь, что приложение работает без подключения к сети.
   - Отправка уведомлений пользователям.
   - Снижение расхода батареи.
   - Создайте масштабируемый, надежный и безопасный задний конец.
   - Автоматизировать обслуживание сервера.

## <a name="devops-and-continuous-monitoring"></a>DevOps и непрерывный мониторинг
Чтобы создать мобильные и облачные приложения, воспользуйтесь преимуществами непрерывной интеграции (CI), непрерывной доставки (CD) и непрерывного мониторинга для создания нативных сборок для всех платформ. Вы можете выполнить тесты автоматизации интерфейса на тысячах физических устройств, автоматизировать их выпуски через каналы бета-тестирования или развернуть непосредственно в магазинах приложений. Вы также можете собирать информацию об использовании аналитики, сбоях и исключениях для своих приложений.

## <a name="additional-services"></a>Дополнительные службы
Многие службы Azure могут быть использованы в зависимости от типа и характера строящихся приложений:
  - Azure Cognitive Search для поиска всех типов внутренних данных с быстрым запросом.
  - Azure Cognitive Services использует искусственный интеллект в приложении.
  - Azure SignalR для включения в приложение в режиме реального времени.

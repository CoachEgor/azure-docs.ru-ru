# <a name="different-stages-in-mobile-application-development"></a>Различные этапы разработки мобильных приложений
Создание мобильного приложения включает несколько этапов, и разработчики могут легко использовать преимущества служб, средств и технологий Майкрософт для создания высококлассных мобильных приложений.

## <a name="app-development"></a>Разработка приложений
Разработчики могут создавать собственные клиентские приложения с единой платформой с помощью таких языков, как цели-C и Java, кросс-платформенные приложения с использованием Xamarin, реагирование на собственные, Unity и гибридные с помощью Cordova. Они могут использовать IDE и редакторы кода, такие как Visual Studio, Visual Studio для Mac или Visual Studio Code на компьютерах и компьютерах Макинтош, для создания клиентских приложений.

## <a name="plan-and-design-with-back-end-services"></a>Планирование и проектирование с помощью серверных служб
 Корпорация Майкрософт предоставляет разнообразные службы для настройки надежной, масштабируемой и надежной серверной части мобильного приложения. Разработчики могут создавать бессерверные интерфейсы API и триггеры без необходимости обслуживания собственных серверов. Они могут использовать службы, такие как push-уведомления, автономную синхронизацию данных, проверку подлинности и данные, которые образуют комплексный и обширный мобильный сервер в качестве службы, поддерживающей разнообразные сценарии построения приложений. Создание серверной службы имеет множество требований:
   - Ограничение на хранение данных на устройстве.
   - Синхронизация данных на нескольких устройствах.
   - Убедитесь, что приложение выполняется без подключения к сети.
   - Отправка уведомлений пользователям.
   - Уменьшение потребления аккумулятора.
   - Создавайте масштабируемые, надежные и безопасные серверные части.
   - Автоматический обслуживания сервера.

## <a name="devops-and-continuous-monitoring"></a>DevOps и непрерывный мониторинг
Разработчики, создающие мобильные и облачные приложения, могут воспользоваться преимуществами непрерывной интеграции (CI), непрерывной поставки (CD) и непрерывного мониторинга, чтобы создавать собственные сборки для всех платформ, выполняющих тесты автоматизации пользовательского интерфейса на тысячах физические устройства, автоматизируйте свои выпуски с помощью бета-каналов тестирования или развертывайте их непосредственно в магазинах приложений. Кроме того, они могут собирать сведения об аналитике использования, сбоях и исключениях для своих приложений.

## <a name="additional-services"></a>Дополнительные службы
Существует множество служб Azure, которые можно использовать в зависимости от типа и природы создаваемого приложения.
  - Azure Когнитивный поиск для поиска всех типов внутренних данных с быстрыми запросами.
  - Azure Cognitive Services для использования искусственного интеллекта в приложении.
  - Azure SignalR для включения взаимодействия в режиме реального времени в приложении.

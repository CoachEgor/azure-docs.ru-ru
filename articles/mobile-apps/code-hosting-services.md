# <a name="cloud-hosted-source-code-management-services"></a>Облачные службы управления исходным кодом
Если у вас есть команды разработчиков с несколькими членами группы, которые работают на одной и той же базе кода, вам необходимо найти подходящее место для размещения кода. Хранение данных в облаке и наличие централизованного репозитория позволяет каждому загружать, отсылать и управлять файлами кода. Они также могут взаимодействовать с другими разработчиками по проектам. Код может быть легко доступен независимо от того, где вы находитесь, до тех пор, пока у вас есть подключение к Интернету.

Облачный хостинг намного проще, чем на месте. Это требует меньшеаппаратной конфигурации и позволяет организациям более гибко завершить процесс реализации.

## <a name="benefits-of-hosting-source-code-in-the-cloud"></a>Преимущества хостинга исходного кода в облаке
- **Централизованное облачное хранилище:** Просматривайте и управляйте данными из любого места.
- **Улучшенная совместная работа и более чистый код:** Следите за кодом в командах и управляйте проектами, чтобы обеспечить непрерывную доставку превосходного программного обеспечения.
- **Легче принять участие:** Легко вносить свой вклад в ваши проекты.
- **Быстрый цикл выпуска:** Работайте быстрее в своих командах и легко вносите свой вклад в ваши проекты.
- **Снижение стоимости:** Не беспокойтесь о поддержании собственного оборудования, серверов, VPN и так далее.

Используйте следующие службы для размещения данных приложения в облаке.

## <a name="github"></a>GitHub
[GitHub](https://github.com/) — это служба хостинга с открытым исходным кодом, в которой размещается проекты исходного кода на различных языках программирования. GitHub отслеживает различные изменения, внесенные в каждую итерацию.

**Основные функции**
- Используйте код-хостинг, чтобы сохранить весь код в одном месте. Частные, общедоступные и открытые репозитории оснащены инструментами, которые помогут вам разместить, версию и код выпуска.
- Просмотрите код и используйте встроенные инструменты обзора, чтобы сделать обзор кода неотъемлемой частью процесса любой команды:
    - Защита филиалов, предложение изменений и запрос отзывов.
    - Найдите различия, прокомментируйте в контексте и получите четкую обратную связь.
- Координируйте работу на ранних стадиях, оставайтесь на согласовании и спомощьи инструментамуправления проектами GitHub:
    - Посмотреть общую картину проекта.
    - Используйте доски задач, которые находятся рядом с кодом внутри GitHub.
    - Перетащите карты для присвоения проблем или запросов на вытягивание запросов членам группы.
    - Установите вехи для организации и отслеживания прогресса.
    - Напишите заметки для захвата идей, которые могут быть полезны, но не принадлежат к конкретной проблеме или запросу на вытягивание.
- Легко обнаружить и выбрать правильные инструменты для улучшения связи и автоматизации работы, покупая приложения от [GitHub Marketplace](https://github.com/marketplace).
- Управление и рост команд с помощью: 
    - Роли пользователей, помогая организовывать команды и получать разрешения.
    - Инструменты обсуждения потоков для поддержания разговоров по теме и сосредоточению внимания.
    - Руководящие принципы сообщества для быстрого создания новых членов команды с учетной записью.
- Просмотр и звезды популярных проектов, чтобы следовать за ними.
- Сеть и учиться у других в отрасли.

## <a name="azure-devops"></a>Azure DevOps
[Azure DevOps](https://azure.microsoft.com/services/devops/) поддерживает [Azure Repos](https://azure.microsoft.com/services/devops/repos/) и [управление версиями Team Foundation (TFVC)](https://docs.microsoft.com/azure/devops/repos/tfvc/index?view=azure-devops) в качестве вариантов управления исходными исходами. Он имеет неограниченное количество бесплатных частных репозиториев с совместными проверками кода, расширенным управлением файлами, поиском кода и политиками филиала для обеспечения высокого качества кода. Repos Azure отлично подходит для небольших проектов и крупных организаций, которым нужна поддержка активного каталога Azure и продвинутые политики.
    
**Основные функции**
- Используйте неограниченное хранилище исходного кода Git, размещенное в облаке, для публичного и частного репо:
    - Получите поддержку для любого клиента Git.
    - Используйте веб-крючки и интеграцию API.
- Kick off ваш следующий сборки от запроса репо тянуть:
    - Сотрудничайте для создания лучшего кода, используя обсуждение потоков и непрерывную интеграцию для каждого изменения.
    - Настройка непрерывной интеграции/непрерывной доставки (CI/CD) для автоматического запуска сборки, тестов и развертываний с каждым завершенным запросом на вытягивание. Вы можете использовать трубопроводы Azure или свои инструменты.
    - Защитите качество кода с помощью отраслевых политик.
- Поддерживайте централизованное управление версиями с помощью Управления версиями Team Foundation, который включает в себя обзор кода.
- Подключайтесь к коду с помощью Xcode, Eclipse, IntelliJ, Android Studio, Visual Studio, Visual Studio Code и многое другое.
- Используйте мощный семантический поиск кода.
- Получите выгоду от функций, готовых к работе. Azure DevOps имеет родную интеграцию с Active Directory Azure, что упрощает процесс управления доступом к репозиториям кода.
- Обеспечьте качество кода с помощью отраслевых политик, таких как минимальное количество обзоров кода, требования к успешным сборкам и выполнение стратегий слияния Git.
- Подключите любимую среду разработки для доступа к РЕПО и управления работой.

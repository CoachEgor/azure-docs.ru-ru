# <a name="choose-mobile-development-frameworks"></a>Выбор платформ разработки для мобильных устройств
Разработчики могут использовать клиентские технологии для создания мобильных приложений с помощью конкретных платформ и шаблонов для межплатформенного подхода. Основываясь на факторах принятия решений, разработчики могут создавать:
- Собственные одноплатформенные приложения с использованием таких языков, как цель C и Java
- Кросс-платформенные приложения с помощью Xamarin, .NET и C #
- Гибридные приложения с помощью Cordova и его разновидностей

## <a name="native-platforms"></a>Собственные платформы
Для создания собственного приложения требуются языки программирования для конкретных платформ, пакеты SDK, среды разработки и другие средства, предоставляемые поставщиками ОС.

### <a name="ios"></a>iOS
IOS, созданные и разработанные компанией Apple, используют для создания приложений на устройствах Apple, а именно iPhone и iPad.

- **Языки программирования**: цель-C, SWIFT
- **Интегрированная среда разработки**: Xcode
- **Пакет**SDK для iOS.

### <a name="android"></a>Android
Android, разработанный Google и наиболее популярная ОС в мире, используется для создания приложений, которые могут работать на различных смартфонах и планшетах.

- **Язык программирования**: Java, Котлин 
- **Интегрированная среда разработки**: средства разработчика для Android Studio и Android 
- **Пакет SDK**: пакет SDK для Android

### <a name="windows"></a>Windows
- **Язык программирования**: C #
- **Интегрированная среда разработки**: Visual Studio, Visual Studio Code
- **Пакет SDK**: Windows SDK

#### <a name="pros"></a>Плюсы
- Хороший пользовательский интерфейс
- Реагирование на приложения с высокой производительностью и возможность взаимодействия с собственными библиотеками
- Приложения с высоким уровнем безопасности

#### <a name="cons"></a>Минусы
- Приложение работает только на одной платформе
- Создание приложения с большим объемом ресурсов для разработчиков и затратами
- Более низкое повторное использование кода

## <a name="cross-platforms-and-hybrid-applications"></a>Кросс-платформенные и гибридные приложения
Кросс-платформенные приложения позволяют создавать собственные мобильные приложения один раз, совместно использовать код и запускать их в iOS, Android и Windows.

### <a name="xamarin"></a>Xamarin
[Xamarin](https://visualstudio.microsoft.com/xamarin/) , принадлежащий корпорации Майкрософт, используется для создания надежных, кросс-платформенных мобильных приложений на C#. Xamarin имеет библиотеку классов и среду выполнения, которая работает на многих платформах, таких как iOS, Android и Windows. Он также компилирует собственные приложения (не интерпретируемые), обеспечивающие высокую производительность. Xamarin сочетает все возможности собственных платформ и добавляет ряд мощных функций.

- **Язык программирования**: C #
- **Интегрированная среда разработки**: Visual Studio в Windows или Mac

### <a name="react-native"></a>React Native
В выпуске Facebook в 2015, [реагируя на машинный код](https://facebook.github.io/react-native/) — это платформа JavaScript с [открытым исходным кодом](https://github.com/facebook/react-native) , предназначенная для написания реальных, готовых к визуализации мобильных приложений для iOS и Android. Он основан на реагировании на библиотеку JavaScript Facebook для создания пользовательских интерфейсов. Вместо нацеливания на браузер, он предназначен для мобильных платформ. Реагирующий машинный код использует собственные компоненты вместо веб-компонентов в качестве стандартных блоков.
 
- **Язык программирования**: JavaScript
- **Интегрированная среда разработки**: Visual Studio Code

### <a name="unity"></a>Unity
 Unity — это подсистема, оптимизированная для создания игр. Его можно использовать для создания высококачественных 2D-или трехмерных приложений с помощью C# для платформ, таких как Windows, iOS, Android и Xbox.

### <a name="cordova"></a>Cordova
Cordova позволяет создавать гибридные приложения с помощью Инструменты Visual Studio для Apache Cordova или Visual Studio Code с расширениями для Cordova. Благодаря гибридному подходу вы можете совместно использовать компоненты с веб-сайтами и использовать приложения на основе веб-сервера с подходами размещенных веб-приложений на основе Cordova.

#### <a name="pros"></a>Плюсы
- Повышение удобства использования кода за счет создания одной базы кода для нескольких платформ
- Распределяться по более широкой аудитории на многих платформах
- Значительное сокращение времени разработки
- Простота запуска и обновления

#### <a name="cons"></a>Минусы
- Снижение производительности
- Отсутствие гибкости
- Каждая платформа имеет уникальный набор функций и функций, позволяющих сделать собственное приложение более творческим
- Увеличение времени разработки пользовательского интерфейса
- Ограничение инструмента

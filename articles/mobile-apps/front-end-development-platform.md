# <a name="choose-mobile-development-framework"></a>Выбор платформы разработки для мобильных устройств
Разработчики могут использовать клиентские технологии для создания собственных мобильных приложений, используя определенные платформы и шаблоны для межплатформенного подхода. Разработчики могут создавать собственные (собственные платформы) с использованием таких языков, как приложения уровня "цель-C" и Java), межплатформенные приложения с использованием C#Xamarin, .NET и), а также гибридные (с помощью Cordova) и их разновидности, в зависимости от их принятия. факторов.

## <a name="native-platforms"></a>Собственные платформы
Для создания собственного приложения требуются языки программирования для конкретных платформ, пакеты SDK, среда разработки и другие средства, предоставляемые поставщиками ОС.

### <a name="ios"></a>iOS
Создаваемые и разработанные компанией Apple приложения, созданные на iOS, работают на устройствах Apple, а именно iPhone и iPad.

- **Языки программирования** — цель-C, SWIFT
- **Интегрированная среда разработки** — Xcode
- **Пакет SDK** для iOS

### <a name="android"></a>Android
Разработанные Google и самые популярные ОС в мире, приложения, созданные на платформе Android, могут работать на различных смартфонах и планшетах.

- **Язык программирования** — Java, Котлин 
- **IDE** -Android Studio и Android средства для разработчиков 
- **Пакет SDK** — пакет SDK для Android

### <a name="windows"></a>Windows
- **Язык программирования** —C#
- **IDE** — Visual Studio, Visual Studio Code
- **Пакет SDK** — Windows SDK

#### <a name="pros"></a>Преимущества
- Хороший пользовательский интерфейс
- Реагирование на приложения с высокой производительностью и возможностью взаимодействия с собственными библиотеками
- Приложения с высоким уровнем безопасности

#### <a name="cons"></a>Недостатки
- Приложение запускается только на одной платформе
- Другие ресурсы для разработчиков и ресурсоемкие приложения для создания приложений
- Более низкое повторное использование кода

## <a name="cross-platforms-and-hybrid-applications"></a>Кросс-платформенные и гибридные приложения
Межплатформенные приложения дают возможность создавать собственные мобильные приложения один раз, совместно использовать код и запускать их в iOS, Android и Windows.

### <a name="xamarin"></a>Xamarin
Приложение [Xamarin](https://visualstudio.microsoft.com/xamarin/) , работающее корпорацией Майкрософт, позволяет создавать надежные и кросс-платформенные мобильные приложения в C#, используя библиотеку классов и среду выполнения, которая работает на многих платформах, включая iOS, Android и Windows, а также выполняет компиляцию машинного (не интерпретируемого) ) — это высокопроизводительные приложения. Xamarin сочетает все возможности собственных платформ и добавляет ряд мощных функций.

- **Язык программирования** —C#
- **Интегрированная среда разработки** Visual Studio в Windows или Mac

### <a name="react-native"></a>React Native
В выпуске Facebook в 2015, [реагируя на машинный код](https://facebook.github.io/react-native/) — это платформа JavaScript с [открытым исходным кодом](https://github.com/facebook/react-native) для написания реальных, встроенных в собственном коде мобильных приложений для iOS и Android. Он основан на использовании библиотеки JavaScript Facebook для создания пользовательских интерфейсов, но вместо ориентации на браузер она предназначена для мобильных платформ. Реагирующий машинный код использует собственные компоненты вместо веб-компонентов в качестве стандартных блоков.
 
- **Язык программирования** — JavaScript
- **Интегрированная среда разработки** — Visual Studio Code

### <a name="unity"></a>Unity
 Unity — это подсистема, оптимизированная для создания игр, которая позволяет разработчикам создавать высококачественные 2D C# -и трехмерные приложения для различных платформ, включая Windows, iOS, Android и Xbox.

### <a name="cordova"></a>Cordova
Cordova позволяет создавать гибридные приложения с помощью Инструменты Visual Studio для Apache Кодова или Visual Studio Code с расширениями для Cordova. Благодаря гибридному подходу вы можете совместно использовать компоненты с веб-сайтами и использовать приложения на основе веб-сервера с подходами размещенных веб-приложений на основе Cordova.

#### <a name="pros"></a>Преимущества
- Повышение удобства использования кода за счет создания одной базы кода для нескольких платформ
- Распределяться по более широкой аудитории на многих платформах
- Значительное сокращение времени разработки
- Простота запуска и обновления

#### <a name="cons"></a>Недостатки
- Снижение производительности
- Отсутствие гибкости
- Каждая платформа имеет уникальный набор функций и функций, позволяющих сделать собственное приложение более творческим
- Увеличение времени разработки пользовательского интерфейса
- Ограничение инструмента

# <a name="choose-mobile-development-frameworks"></a>Выберите мобильные платформы разработки
Разработчики могут использовать технологии клиентской стороны для создания мобильных приложений самостоятельно, используя определенные рамки и шаблоны для кросс-платформенного подхода. Основываясь на факторах принятия решений, разработчики могут создавать:
- Родные одноплатформенные приложения с использованием языков, таких как Objective C и Java
- Кросс-платформенные приложения с помощью Xamarin, .NET и C #
- Гибридные приложения с помощью Cordova и ее вариантов

## <a name="native-platforms"></a>Родные платформы
Для создания нативного приложения требуются языки программирования, SDK, среды разработки и другие инструменты, предоставляемые поставщиками ОС.

### <a name="ios"></a>iOS
Созданная и разработанная Apple, iOS используется для создания приложений на устройствах Apple, а именно iPhone и iPad.

- **Языки программирования**: Объектив-C, Swift
- **IDE**: Xcode
- **SDK**: iOS SDK

### <a name="android"></a>Android
Разработанный Google и самой популярной ОС в мире, Android используется для создания приложений, которые могут работать на ряде смартфонов и планшетов.

- **Язык программирования**: Java, Kotlin 
- **IDE**: Android Studio и Android инструменты для разработчиков 
- **SDK**: Android SDK

### <a name="windows"></a>Windows
- **Язык программирования**: C #
- **IDE**: Визуальная студия, Визуальный код студии
- **SDK**: Windows SDK

#### <a name="pros"></a>Плюсы
- Хороший пользовательский опыт
- Ответные приложения с высокой производительностью и возможностью взаимодействия с родными библиотеками
- Высокозащищенные приложения

#### <a name="cons"></a>Минусы
- Приложение работает только на одной платформе
- Более ресурсоемкие и дорогостоящие ресурсы разработчика для создания приложения
- Более низкое повторное использование кода

## <a name="cross-platforms-and-hybrid-applications"></a>Кросс-платформы и гибридные приложения
Кросс-платформенные приложения дают вам возможность писать нативные мобильные приложения один раз, обмениваться кодом и запускать их на iOS, Android и Windows.

### <a name="xamarin"></a>Xamarin
Компания [Xamarin,](https://visualstudio.microsoft.com/xamarin/) принадлежащая корпорации Майкрософт, используется для создания надежных кросс-платформенных мобильных приложений в СЗ. Xamarin имеет библиотеку классов и время выполнения, которая работает на многих платформах, таких как iOS, Android и Windows. Он также компилирует наивысшие (не интерпретируемые) приложения, обеспечивающие высокую производительность. Xamarin сочетает в себе все возможности отечественных платформ и добавляет ряд мощных особенностей своей собственной.

- **Язык программирования**: C #
- **IDE**: Визуальная студия на Windows или Mac

### <a name="react-native"></a>React Native
Выпущенная Facebook в 2015 году, [React Native](https://facebook.github.io/react-native/) является платформой JavaScript с [открытым исходным кодом](https://github.com/facebook/react-native) для написания реальных, роднивую рендеринг мобильных приложений для iOS и Android. Она основана на React, библиотеке JavaScript Facebook для создания пользовательских интерфейсов. Вместо того, чтобы таргетинг на браузер, он ориентирован на мобильные платформы. React Native использует нативные компоненты вместо веб-компонентов в качестве строительных блоков.
 
- **Язык программирования**: JavaScript
- **IDE**: Визуальный код студии

### <a name="unity"></a>Unity
 Unity — это движок, оптимизированный для создания игр. Вы можете использовать его для создания высококачественных 2D или 3D-приложений с помощью C-приложений для таких платформ, как Windows, iOS, Android и Xbox.

### <a name="cordova"></a>Cordova
Cordova позволяет создавать гибридные приложения с помощью Visual Studio Tools для Apache Cordova или Visual Studio Code с расширениями для Cordova. С помощью гибридного подхода можно обмениваться компонентами с веб-сайтами и повторно использовать веб-серверные приложения с помощью приложений, основанных на Cordova.

#### <a name="pros"></a>Плюсы
- Увеличение удобства использования кода путем создания одной базы кода для нескольких платформ
- Питание для более широкой аудитории на многих платформах
- Резкое сокращение времени разработки
- Легко запускать и обновлять

#### <a name="cons"></a>Минусы
- Более низкая производительность
- Отсутствие гибкости
- Каждая платформа имеет уникальный набор функций и функциональности, чтобы сделать родное приложение более творческим
- Увеличенное время проектирования uI
- Ограничение инструмента

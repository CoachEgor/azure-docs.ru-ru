# <a name="automate-the-deployment-and-release-of-your-mobile-applications-with-continuous-delivery-services"></a>Автоматизация развертывания и выпуска мобильных приложений с помощью непрерывного обслуживания доставки

Как разработчики, вы пишете код и проверяете его в репозитории кода, но коммиты, зарегистрированные в репо, не всегда могут быть последовательными. Когда несколько разработчиков работают над тем же проектом, могут возникать проблемы с интеграцией. Команды могут столкнуться с ситуациями, когда что-то не работает, ошибки накапливаются, а разработка проекта задерживается. Разработчикам приходится ждать, пока весь программный код будет построен и протестирован, чтобы проверить наличие ошибок, что делает процесс медленным и менее итеративным.

С непрерывной доставкой вы автоматизировать развертывание и выпуск мобильных приложений. Не имеет значения, распространяете ли вы приложение группе тестеров или сотрудников компании (для бета-тестирования) или в магазин приложений (для производства). Непрерывная доставка делает развертывание менее рискованным и стимулирует быструю итерацию. Вы также можете постоянно выпускать новые изменения для своих клиентов.

## <a name="distribute-application-binaries-to-beta-testers"></a>Распространение файловых файлов приложений для бета-тестеров
Бета-тестирование мобильного приложения является одним из важнейших этапов в процессе разработки приложения. Это помогает найти ошибки и проблемы в приложении на ранней стадии. Обратная связь улучшает качество приложения, когда вы готовите его к использованию.

Используйте следующие службы для обеспечения непрерывного конвейера доставки в мобильных приложениях.

### <a name="visual-studio-app-center"></a>Центр приложений Visual Studio
[App Center Distribute](/appcenter/distribution/) — это инструмент для разработчиков для быстрого выпуска сборок на устройства. Благодаря полному опыту установки портала, App Center Distribute является мощным решением для распространения бета-приложений тестера. Это также удобная альтернатива распространению через публичные магазины приложений. Разработчики могут еще больше автоматизировать свой рабочий процесс дистрибуции с помощью App Center Build и публичных интеграций в магазин приложений.

**Основные функции**
- Распределите приложение с бета-тестерами и пользователями и убедитесь, что все ваши тестеры находятся в последней версии приложения.
- Сообщите тестерам о новых релизах без тестеров, которые снова проходят через поток загрузки.
- Управление группами дистрибутива для различных версий приложения.
- Распространение в магазинах: 
    - [Apple](/appcenter/distribution/stores/apple)
    - [Google Play](/appcenter/distribution/stores/googleplay)
    - [Intune](/appcenter/distribution/stores/intune)
- Получите поддержку платформы для iOS, Android, macOS, tvOS, Xamarin, React Native, Unity и Cordova.
- Автоматически регистрируйте устройства iOS в профиле подготовки.

**Ссылки**

- [Начало работы с App Center Распространять](/appcenter/build/)

### <a name="azure-pipelines"></a>Azure Pipelines

[Azure Pipelines](https://azure.microsoft.com/services/devops/pipelines/) — это полнофункционная служба непрерывной интеграции (CI) и непрерывной доставки (CD), которая работает с вашим предпочтительным поставщиком Git. Конвейеры Azure могут развертываться в большинстве основных облачных служб, таких как службы Azure. Вы можете начать с кода на GitHub, GitHub Enterprise Server, GitLab, Биткветвет-Клаб или Azure Repos. Затем можно автоматизировать сборку, тестирование и развертывание кода в Microsoft Azure, облачную платформу Google или Amazon Web Services (AWS).

**Основные функции**
- **Упрощенный опыт работы на основе задач для настройки сервера CI:** Настройка сервера CI для мобильных приложений как для наивных (Android, iOS и Windows), так и для кросс-платформенных (Xamarin, Cordova и React Native).
- **Любой язык, платформа и облако:** Создавайте, тестируйте и развертывайте приложения Node.js, Python, Java, PHP, Ruby, Go, C/C, C, Android и iOS. Параллельно работаете на Linux, macOS и Windows. Развертывание облачных провайдеров, таких как Azure, AWS и облачная платформа Google. Распространение мобильных приложений через бета-каналы и магазины приложений.
- **Поддержка контейнеров коренных народов:** Создавайте новые контейнеры с легкостью и выталкивайте их в любой реестр. Развертывание контейнеров для независимых хостов или Kubernetes.
- **Расширенные рабочие процессы и функции:** Легко создавать цепочки сборки и многофазные сборки. Получите поддержку YAML, тестовой интеграции, выпускных ворот, отчетности и многое другое.
- **Расширяемый:** Используйте ряд задач сборки, тестирования и развертывания, созданных сообществом, который включает сотни расширений от Slack до SonarCloud. Вы даже можете развернуть из других систем CI, как Дженкинс. Веб-крючки и REST AIS могут помочь вам интегрироваться.
- **Бесплатные облачные сборки:** Эти сборки доступны для государственных и частных хранилищ.
- **Поддержка развертывания другим поставщикам облачных технологий:** Поставщики включают AWS и облачную платформу Google.

## <a name="distribute-your-application-directly-to-app-stores"></a>Распространяйте приложение непосредственно в App Stores
После того, как ваше приложение готово к использованию в производстве и вы хотите, чтобы оно использовалось публично, оно должно быть представлено в магазины приложений, где оно может быть загружено клиентами. Существует несколько способов распространения приложения непосредственно в магазинах приложений. 

### <a name="visual-studio-app-center"></a>Центр приложений Visual Studio
С [помощью App Center Distribute](/appcenter/distribution/stores/)вы можете публиковать свои мобильные приложения непосредственно в магазинах приложений. После того, как приложение будет готово к загрузке пользователями, вы можете опубликовать свои файлы приложения прямо с портала Visual Studio App Center. 

Вы можете напрямую распространять:
- [Apple App Store;](/appcenter/distribution/stores/apple)
- [Магазин Google Play](/appcenter/distribution/stores/googleplay)
- [Microsoft Intune](/appcenter/distribution/stores/intune)
    
### <a name="apple-app-store"></a>Apple App Store;
В магазине приложений, разработанном и поддерживаемом Apple, пользователи могут просматривать и загружать приложения, разработанные для устройств iOS, MacOS, WatchOS и tvOS. Разработчики должны отправить свои приложения для iOS в Apple App Store для публичного использования.

### <a name="google-play"></a>Google Play

Google Play является официальным магазином приложений для ОС Android, где пользователи могут просматривать и загружать приложения, разработанные для Android устройств, которые публикуются через Google.

### <a name="intune"></a>Intune

[Microsoft Intune](/intune/app-management) — это облачный сервис в пространстве управления корпоративной мобильностью, который помогает сотрудникам быть продуктивными, сохраняя при этом корпоративные данные защищенными. С помощью Intune можно выполнять следующие задачи:
- Управление мобильными устройствами и компьютерами, используемыми сотрудниками для доступа к данным организации.
- Управление мобильными приложениями, которые используют сотрудники вашей рабочей силы.
- Защитите информацию о компании, контролируя способ доступа сотрудников и их доступа.
- Убедитесь, что устройства и приложения соответствуют требованиям безопасности компании.
    
## <a name="deploy-updates-directly-to-users-devices"></a>Развертывание обновлений непосредственно на устройствах пользователей

### <a name="codepush"></a>CodePush
С [CodePush](/appcenter/distribution/codepush/) в App Center разработчики Apache Cordova и React Native могут развертывать обновления мобильных приложений непосредственно на устройствах своих пользователей. Он выступает в качестве центрального репозитория, в который разработчики могут публиковать определенные обновления, такие как JavaScript, HTML, CSS и изменения изображения. Затем приложения могут запросить обновления из репозитория с помощью предоставленных клиентских SDK. Таким образом, вы можете иметь более детерминированной и прямой модели взаимодействия с пользователями в то время как вы решаете ошибки или добавить небольшие функции. Вы не обязаны восстанавливать двоичный файл или распространять его через любые публичные магазины приложений.

**Основные функции**
- Разработчики Cordova и React Native могут развертывать обновления мобильных приложений непосредственно на устройствах своих пользователей, не выпуская их в магазине.
- Полезно для исправления ошибок или добавления и удаления небольших функций, которые не требуют от вас, чтобы восстановить двоичный и перераспределить его через соответствующие магазины.

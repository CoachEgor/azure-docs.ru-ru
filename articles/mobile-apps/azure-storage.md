# <a name="cloud-storage-for-highly-secure-durable-scalable-apps-with-azure-storage"></a>Облачное хранилище для высокозащищенных, прочных и масштабируемых приложений с Azure Storage
[Azure Storage](https://azure.microsoft.com/services/storage/) — это облачное решение Microsoft для современных приложений, которое предлагает массово масштабируемый объект для хранения объектов данных, службу файловой системы для облака, хранилище сообщений для надежного обмена сообщениями и хранилище НоСЗЛ. Служба хранилища Azure отличается следующими преимуществами:
- **Долговечные и высокодоступные:** Избыточность гарантирует безопасность данных в случае временных сбоев оборудования. Также для дополнительной защиты от локальной аварии или стихийного бедствия можно выполнить репликацию данных в другой центр обработки данных или другой географический регион. Таким образом можно обеспечить высокую доступность реплицированных данных в случае неожиданного сбоя.
- **Безопасный:** Все данные, записанные в Azure Storage, шифруются службой. В службе хранилища Azure предоставляется возможность точного управления доступом к данным.
- **Масштабируемый:** Службы предназначены для масштабирования для удовлетворения потребностей в хранении данных и производительности современных приложений.
- **Управляемые:** Azure обрабатывает техническое обслуживание оборудования, обновления и критические проблемы для вас.
- **Доступно:** Данные доступны из любой точки мира по адресу HTTP или HTTPS. Корпорация Майкрософт предоставляет клиентские библиотеки на различных языках, таких как .NET, Java, Node.js, Python, PHP, Ruby и Go, а также зрелый REST API. Сценарий поддерживается в Azure PowerShell или Azure CLI. Портал Azure и Azure Storage Explorer предлагают простые визуальные решения для работы с вашими данными.

Используйте следующие службы для обеспечения облачного хранилища в мобильных приложениях.

## <a name="azure-blob-storage"></a>Хранилище BLOB-объектов Azure
[Хранение Azure Blob](https://azure.microsoft.com/services/storage/blobs/) предлагает решение для хранения объектов для облака. Хранилище Blob оптимизировано для хранения огромных объемов неструктурированных данных, которые не соответствуют определенной модели данных или определению, таким как текст или двоичный файл. Он поддерживает различные языки, которые используют клиентские библиотеки. Хранилище Blob предназначено для:
- Подавать изображения или документы непосредственно в браузер.
- Храните файлы для распределенного доступа.
- Потоковое видео и аудио.
- Запись для входа файлов.
- Храните данные для резервного копирования и восстановления, аварийного восстановления и архивирования.
- Храните данные для анализа в службе, размещенной на территории или в Azure.

## <a name="azure-table-storage"></a>табличное хранилище Azure;
[Хранилище таблицы Azure](https://azure.microsoft.com/services/storage/tables/) — это служба, которая хранит структурированные данные NoS'L в облаке и обеспечивает магазин ключей или атрибутов с бессхеманым дизайном. В хранилище таблиц Azure содержатся большие объемы структурированных данных. Услуга представляет собой хранилище данных NoS'L, которое принимает аутентифицированные вызовы как внутри, так и за пределами облака Azure. Таблицы Azure идеально подходят для хранения структурированных, нереляционных данных. Таблица хранения, как правило, используется для:
- Храните терабайты структурированных данных, способных обслуживать приложения веб-масштабирования.
- Храните наборы данных, которые не требуют сложных соединений, иностранных ключей или сохраненных процедур и могут быть денормализованы для быстрого доступа.
- Быстрый запрос данных с помощью кластерного индекса.
- Доступ к данным с помощью протокола OData и запросов LIN' с windows Communication Foundation (WCF) Data Services .NET.

Вы можете использовать хранилище таблицы для хранения и запроса огромных наборов структурированных, нереляляционных данных. Ваши таблицы масштабируются по мере увеличения спроса.

## <a name="azure-files"></a>Файлы Azure
С [помощью Azure Files](https://azure.microsoft.com/services/storage/files/)можно настроить высокодоступные доли сетевых файлов, к которым можно получить доступ с помощью стандартного протокола Server Message Block (SMB). Несколько VMs могут обмениваться одинаковыми файлами как с доступом читать, так и писать. Вы также можете прочитать файлы с помощью интерфейса REST или библиотек и клиентов хранения. Вы можете получить доступ к файлам из любой точки мира, используя URL-адрес, который указывает на файл и включает маркер общей подписи доступа (SAS). Можно создавать токены SAS. Они позволяют получить определенный доступ к частному активу в течение определенного периода времени.

Файловые ресурсы Azure можно использовать следующим образом.
- **Заменить или дополнить файловых серверов:** Популярные операционные системы, такие как Windows, macOS и Linux, могут непосредственно монтировать файлы Azure, где бы они ни находились в мире. Общие ресурсы службы файлов Azure также можно реплицировать с помощью службы синхронизации файлов Azure на локальные или облачные серверы Windows Server для высокопроизводительного и распределенного кэширования данных в месте их использования.
- **Приложения подъема и смены:** Перенос приложений в облако, которые ожидают, что доля файлов будет хранить файлы или пользовательские данные.
- **Упрощение разработки облачных технологий:** Файлы Azure также могут использоваться во многих отношениях для упрощения новых проектов разработки облачных технологий. Пример:
    - **Настройки приложения:** Общим шаблоном для распределенных приложений является размещение файлов конфигураций в централизованном месте, где к ним можно получить доступ из многих экземпляров приложений. Экземпляры приложений могут загружать свою конфигурацию через API File REST. Пользователи могут получить к ним доступ по мере необходимости, устанавливая долю SMB локально.
    - **Диагностическая доля:** Совместное действие файлов Azure — это удобное место для записи облачных приложений для записи журналов, метрик и сбоев. Входы могут быть написаны экземплярами приложения через API File REST. Разработчики могут получить к ним доступ, устанавливая общую часть файлов на локальной машине. Эта возможность обеспечивает большую гибкость. Разработчики могут использовать разработку облачных технологий без необходимости отказываться от существующего инструмента, который они знают.

## <a name="azure-queue-storage"></a>Хранилище очередей Azure
[Хранение очереди Azure](https://azure.microsoft.com/services/storage/queues/) — это служба хранения большого количества сообщений. Вы получаете доступ к сообщениям из любой точки мира с помощью аутентифицированных звонков с помощью HTTP или HTTPS. Максимальный размер сообщения в очереди составляет 64 КБ. Очередь может содержать миллионы сообщений, вплоть до предела емкости учетной записи хранилища. Очереди обычно используются для создания списка невыполненных заданий для асинхронной обработки.

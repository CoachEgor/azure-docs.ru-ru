# <a name="cloud-storage-for-highly-secure-durable-scalable-apps-with-azure-storage"></a>Облачное хранилище для высокозащищенных, устойчивых и масштабируемых приложений с помощью службы хранилища Azure
Служба [хранилища Azure](https://azure.microsoft.com/services/storage/) — это облачное хранилище Майкрософт для современных приложений, которое предлагает масштабируемое хранилище объектов для объектов данных, службу файловой системы для облака, хранилище сообщений для надежного обмена сообщениями и NoSQL магазин. Служба хранилища Azure отличается следующими преимуществами:
- **Устойчивая и высокая доступность:** Избыточность гарантирует безопасность данных в случае временных сбоев оборудования. Также для дополнительной защиты от локальной аварии или стихийного бедствия можно выполнить репликацию данных в другой центр обработки данных или другой географический регион. Таким образом можно обеспечить высокую доступность реплицированных данных в случае неожиданного сбоя.
- **Безопасность:** Все данные, записываемые в службу хранилища Azure, шифруются службой. В службе хранилища Azure предоставляется возможность точного управления доступом к данным.
- **Масштабируемая:** Службы предназначены для массовой масштабируемости в соответствии с потребностями хранения и производительности современных приложений.
- **Управляемый:** Azure обрабатывает аппаратное обслуживание, обновления и критические проблемы.
- **Доступно:** Данные доступны из любой точки мира по протоколу HTTP или HTTPS. Корпорация Майкрософт предоставляет клиентские библиотеки на различных языках, таких как .NET, Java, Node. js, Python, PHP, Ruby и Go, а также на зрелых REST API. Скрипты поддерживаются в Azure PowerShell или в Azure CLI. Портал Azure и Обозреватель службы хранилища Azure предлагают простые визуальные решения для работы с данными.

Используйте следующие службы, чтобы включить облачное хранилище в мобильных приложениях.

## <a name="azure-blob-storage"></a>Хранилище BLOB-объектов Azure
[Хранилище BLOB-объектов Azure](https://azure.microsoft.com/services/storage/blobs/) предлагает решение для хранения объектов в облаке. Хранилище BLOB-объектов оптимизировано для хранения огромных объемов неструктурированных данных, которые не соответствуют определенной модели или определению данных, например текстовому или двоичному. Он поддерживает различные языки, используемые клиентскими библиотеками. Хранилище BLOB-объектов предназначено для:
- Обслуживание изображений или документов непосредственно в браузере.
- Хранение файлов для распределенного доступа.
- Потоковая передача видео и аудио.
- Запись в файлы журнала.
- Храните данные для резервного копирования и восстановления, аварийного восстановления и архивации.
- Храните данные для анализа в локальной или размещенной в Azure службе.

## <a name="azure-table-storage"></a>табличное хранилище Azure;
[Хранилище таблиц Azure](https://azure.microsoft.com/services/storage/tables/) — это служба, которая хранит структурированные данные NoSQL в облаке и предоставляет ключ или хранилище атрибутов со структурой без схемы. В хранилище таблиц Azure содержатся большие объемы структурированных данных. Служба — это хранилище данных NoSQL, которое принимает вызовы, прошедшие проверку подлинности, из облака Azure и за ее пределами. Таблицы Azure идеально подходят для хранения структурированных нереляционных данных. Хранилище таблиц обычно используется для:
- Хранение терабайтов структурированных данных, способных обслуживать веб-приложения.
- Храните наборы данных, не требующие сложных соединений, внешних ключей или хранимых процедур, и могут быть денормализованы для быстрого доступа.
- Быстрое выполнение запросов к данным с помощью кластеризованного индекса.
- Доступ к данным осуществляется с помощью протокола OData и запросов LINQ с помощью библиотек .NET служб данных Windows Communication Foundation (WCF).

Хранилище таблиц можно использовать для хранения и запроса огромных наборов структурированных нереляционных данных. Таблицы масштабируются по мере роста спроса.

## <a name="azure-files"></a>Файлы Azure
С помощью службы [файлов Azure](https://azure.microsoft.com/services/storage/files/)можно настроить высокодоступные сетевые файловые ресурсы, к которым можно получить доступ, используя стандартный протокол SMB. Несколько виртуальных машин могут совместно использовать одни и те же файлы с доступом на чтение и запись. Кроме того, файлы можно читать с помощью интерфейса RESTFUL или клиентских библиотек хранилища. Вы можете получить доступ к файлам из любой точки мира, используя URL-адрес, указывающий на файл и включающий маркер подписи общего доступа (SAS). Вы можете создавать маркеры SAS. Они обеспечивают конкретный доступ к частному ресурсу в течение определенного времени.

Файловые ресурсы Azure можно использовать следующим образом.
- **Замена или дополнение локальных файловых серверов:** В популярных операционных системах, таких как Windows, macOS и Linux, можно напрямую подключать файловые ресурсы Azure, где бы они ни находились в мире. Общие ресурсы службы файлов Azure также можно реплицировать с помощью службы синхронизации файлов Azure на локальные или облачные серверы Windows Server для высокопроизводительного и распределенного кэширования данных в месте их использования.
- **Приложения с нулификацией и сдвигом:** Перенос приложений в облако, в котором требуется файловый ресурс для хранения файлов приложения или данных пользователя.
- **Упростите разработку в облаке:** Службы файлов Azure также могут использоваться несколькими способами для упрощения новых проектов разработки в облаке. Пример.
    - **Параметры общего приложения:** Распространенным шаблоном для распределенных приложений является наличие файлов конфигурации в централизованном расположении, где доступ к ним можно получить из множества экземпляров приложения. Экземпляры приложения могут загружать свою конфигурацию через файл REST API. Пользователи могут обращаться к ним по мере необходимости, подключив общий ресурс SMB локально.
    - **Общая папка диагностики:** Файловый ресурс Azure представляет собой удобное место для создания журналов, метрик и аварийных дампов в облачных приложениях. Журналы могут записываться экземплярами приложения через файл REST API. Разработчики могут получить доступ к ним, подключив общую папку на своем локальном компьютере. Эта возможность обеспечивает большую гибкость. Разработчики могут принять разработку в облаке, не отменяя существующие средства, которые они узнают.

## <a name="azure-queue-storage"></a>Хранилище очередей Azure
[Хранилище очередей Azure](https://azure.microsoft.com/services/storage/queues/) — это служба для хранения большого количества сообщений. Доступ к сообщениям из любой точки мира осуществляется через вызовы с проверкой подлинности по протоколу HTTP или HTTPS. Максимальный размер сообщения в очереди составляет 64 КБ. Очередь может содержать миллионы сообщений вплоть до общего ограничения емкости учетной записи хранения. Очереди обычно используются для создания списка невыполненных заданий для асинхронной обработки.

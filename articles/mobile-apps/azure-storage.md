# <a name="cloud-storage-for-highly-secure-durable-scalable-apps-with-azure-storage"></a>Облачное хранилище для высокозащищенных, устойчивых и масштабируемых приложений с помощью службы хранилища Azure
Служба [хранилища Azure](https://azure.microsoft.com/services/storage/) — это облачное хранилище Майкрософт для современных приложений, которое предлагает масштабируемое хранилище объектов для объектов данных, службу файловой системы для облака, хранилище сообщений для надежного обмена сообщениями и NoSQL магазин. Служба хранилища Azure отличается следующими преимуществами:
- **Устойчивая и высокодоступная** избыточность обеспечивает безопасность данных в случае временных сбоев оборудования. Также для дополнительной защиты от локальной аварии или стихийного бедствия можно выполнить репликацию данных в другой центр обработки данных или другой географический регион. Таким образом можно обеспечить высокую доступность реплицированных данных в случае неожиданного сбоя.
- **Secure** — все данные, записываемые в службу хранилища Azure, шифруются службой. В службе хранилища Azure предоставляется возможность точного управления доступом к данным.
- **Масштабируемые** службы рассчитаны на массовую масштабируемость, чтобы удовлетворить потребности в хранении и производительности современных приложений.
- **Управляемый** — Azure обрабатывает аппаратное обслуживание, обновления и критические проблемы.
- Данные **доступны** из любой точки мира по протоколу HTTP или HTTPS. Корпорация Майкрософт предоставляет клиентские библиотеки на различных языках, включая .NET, Java, Node. js, Python, PHP, Ruby, Go и т. д., а также на зрелых REST API. Сценарии поддерживаются в Azure PowerShell или Azure CLI, а портал Azure и Обозреватель службы хранилища Azure предлагают простые визуальные решения для работы с данными.

Используйте следующие службы, чтобы включить облачное хранилище в мобильных приложениях.

## <a name="azure-blob-storage"></a>Хранилище BLOB-объектов Azure
[Хранилище BLOB-объектов Azure](https://azure.microsoft.com/services/storage/blobs/) предлагает решение для хранения объектов в облаке и оптимизировано для хранения огромных объемов неструктурированных данных, которые не соответствуют определенной модели данных или определению, например текстовому или двоичному. Он поддерживает клиентские библиотеки различных языков и предназначен для:
- Обслуживание изображений или документов непосредственно в браузере.
- Хранение файлов для распределенного доступа.
- Потоковая передача видео и аудио.
- Запись в файлы журнала.
- Хранение резервных копий и восстановление данных, аварийное восстановление и архивация.
- Хранение данных для анализа локальной службой или службой, размещенной в Azure.

## <a name="azure-table-storage"></a>Хранилище таблиц Azure
[Хранилище таблиц Azure](https://azure.microsoft.com/services/storage/tables/) — это служба, которая хранит структурированные данные NoSQL в облаке, предоставляя хранилище ключей или атрибутов с проектированием без схемы. В хранилище таблиц Azure содержатся большие объемы структурированных данных. Служба — это хранилище данных NoSQL, которое принимает вызовы, прошедшие проверку подлинности, из облака Azure и за ее пределами. Таблицы Azure идеально подходят для хранения нереляционных структурированных данных. Самые распространенные способы использования хранилища таблиц:
- Хранение TBs структурированных данных, способных обслуживать веб-приложения.
- Хранение наборов данных, которые не нуждаются в сложных объединениях, внешних ключах или хранимых процедурах и могут быть денормализованы для быстрого доступа.
- Быстрый запрос данных с помощью кластеризованного индекса.
- Доступ к данным с помощью протокола OData и запросов LINQ с библиотеками .NET службы данных WCF.

Хранилище таблиц можно использовать для хранения огромных наборов структурированных нереляционных данных и обращения к ним. Таблица масштабируется в соответствии с потребностями.

## <a name="azure-files"></a>Файлы Azure
[Служба файлов Azure](https://azure.microsoft.com/services/storage/files/) позволяет настроить сетевые файловые ресурсы с высоким уровнем доступности, доступ к которым можно получить с помощью стандартного протокола SMB. Несколько виртуальных машин могут совместно использовать одни и те же файлы с доступом на чтение и запись. Вы также можете считывать файлы с помощью интерфейса REST или клиентских библиотек хранилища. Вы можете получить доступ к файлам из любой точки мира, используя URL-адрес, указывающий на файл и содержащий маркер подписанного URL-адреса (SAS). Вы можете создавать маркеры SAS, которые предоставляют определенный доступ к частным ресурсам на конкретный срок.

Файловые ресурсы Azure можно использовать следующим образом.
- Замена или дополнение локальных файловых серверов: популярные операционные системы, такие как Windows, macOS и Linux, могут напрямую подключать файловые ресурсы Azure, где бы они ни находились в мире. Общие ресурсы службы файлов Azure также можно реплицировать с помощью службы синхронизации файлов Azure на локальные или облачные серверы Windows Server для высокопроизводительного и распределенного кэширования данных в месте их использования.
- Перенос **приложений** в облако, в котором требуется файловый ресурс для хранения файлов приложения или данных пользователя.
- **Упростите разработку в облаке**. файлы Azure также можно использовать множеством способов для упрощения новых проектов разработки в облаке. Пример.
    - Общие параметры приложения. распространенным шаблоном для распределенных приложений является наличие файлов конфигурации в централизованном расположении, где они доступны из множества экземпляров приложения. Экземпляры приложений могут загружать свою конфигурацию с помощью REST API службы файлов, а пользователи могут обращаться к ним, установив локально общий ресурс SMB.
    - Общая папка диагностики. файловый ресурс Azure — это удобное место для создания журналов, метрик и аварийных дампов в облачных приложениях. Экземпляры приложений могут записывать журналы через REST API службы файлов, а разработчики могут обращаться к ним через общий ресурс на своем локальном компьютере. Подобная гибкость позволяет разработчикам заниматься разработкой облачных приложений, используя привычные средства.

## <a name="azure-queues"></a>Очереди Azure
[Хранилище очередей Azure](https://azure.microsoft.com/services/storage/queues/) — это служба для хранения большого количества сообщений. Доступ к сообщениям возможен из любой точки мира с помощью вызовов с проверкой подлинности по протоколу HTTP или HTTPS. Сообщение очереди может иметь размер до 64 КБ, а очередь может содержать миллионы сообщений, вплоть до общего ограничения емкости учетной записи хранения. Очереди обычно используются для создания списка невыполненных заданий для асинхронной обработки.

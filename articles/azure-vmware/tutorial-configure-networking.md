# <a name="tutorial-configure-networking-for-your-vmware-private-cloud-in-azure"></a>Руководство по настройке сети для частного облака VMWare в Azure

Для частного облака решения Azure VMware (AVS) требуется наличие виртуальной сети. Так как AVS в предварительной версии не поддерживает локальный экземпляр vCenter, вам придется выполнить дополнительные шаги по интеграции с локальной средой. Также вам нужно настроить канал ExpressRoute и шлюз виртуальной сети, как описано далее в этом руководстве.

В этом руководстве описано следующее:

> [!div class="checklist"]
> * Создайте виртуальную сеть
> * Создание шлюза виртуальной сети.
> * Подключение канала ExpressRoute к шлюзу.
> * Поиск URL-адресов для vCenter и диспетчера NSX.

## <a name="sign-in-to-the-azure-portal"></a>Вход на портал Azure

Войдите на [портал Azure](https://portal.azure.com).

## <a name="create-a-virtual-network"></a>Создание виртуальной сети

Перейдите в группу ресурсов, которую вы создали по инструкциям из [предыдущего руководства](tutorial-create-private-cloud.md), и щелкните **Добавить** для создания нового ресурса.

В текстовое поле **Поиск по Marketplace** введите **Виртуальная сеть**. Найдите и выберите ресурс "Виртуальная сеть".

На странице "Виртуальная сеть" щелкните **Создать**, чтобы настроить виртуальную сеть для частного облака.

На странице **Создание виртуальной сети** введите необходимые сведения о виртуальной сети, которые описаны в приведенной ниже таблице.

> [!IMPORTANT]
> Укажите диапазон адресов, который **не пересекается** с диапазоном, указанным при создании частного облака при работе с предыдущим руководством.

На вкладке **Основные сведения** введите имя виртуальной сети, выберите подходящий регион и щелкните **Далее: IP-адреса**.

На вкладке **IP-адреса** введите в разделе **Диапазон IPv4-адресов** диапазон адресов, созданный при работе с предыдущим руководством.

Щелкните **+ Добавить подсеть**, чтобы открыть страницу **Добавление подсети**, где укажите имя и диапазон адресов для подсети. Когда все будет готово, щелкните **Добавить**.

Выберите **Просмотреть и создать**.

:::image type="content" source="./media/tutorial-configure-networking/create-virtual-network.png" alt-text="Создание виртуальной сети" border="true":::

Проверьте введенные сведения и щелкните **Создать**. Когда развертывание завершится, вы увидите новую виртуальную сеть в группе ресурсов.

## <a name="create-a-virtual-network-gateway"></a>Создание шлюза виртуальной сети

В предыдущем разделе вы создали виртуальную сеть, а теперь мы создадим шлюз виртуальной сети.

На странице группы ресурсов щелкните **Добавить**, чтобы добавить новый ресурс.

В текстовое поле **Поиск по Marketplace** введите **Шлюз виртуальной сети**. Найдите и выберите ресурс "Виртуальная сеть".

Щелкните **Создать** на странице **Шлюз виртуальной сети**.

На вкладке "Основные сведения" на странице **Создание шлюза виртуальной сети** укажите значения в полях, которые описаны в приведенной ниже таблице.

| Поле | Значение |
| --- | --- |
| **Подписка** | В этом поле уже указано значение подписки, к которой принадлежит группа ресурсов. |
| **Группа ресурсов** | В этом поле уже указаны сведения о текущей группе ресурсов. Это должна быть та группа ресурсов, которую вы создали в предыдущем тесте. |
| **имя**; | Введите уникальное имя для шлюза виртуальной сети. |
| **Регион** | Выберите географическое расположение для шлюза виртуальной сети. |
| **Тип шлюза** | Выберите **ExpressRoute**. |
| **Тип VPN** | Выберите **На основе маршрута**. |
| **SKU** | Оставьте значение по умолчанию: **Стандартный**. |
| **Виртуальная сеть** | Выберите созданную ранее виртуальную сеть. Если эта виртуальная сеть не отображается, убедитесь, что регион шлюза соответствует региону виртуальной сети. |
| **Диапазон адресов подсети шлюза** | Это значение подставляется, когда вы выбираете виртуальную сеть. Не изменяйте значение по умолчанию. |
| **Общедоступный IP-адрес** | Выберите **Создать**. |

:::image type="content" source="./media/tutorial-configure-networking/create-virtual-network-gateway.png" alt-text="Создание шлюза" border="true":::

Выберите **Проверка и создание**, затем на следующей странице проверьте правильность введенных данных и щелкните **Создать**, чтобы запустить развертывание шлюза виртуальной сети. Когда развертывание завершится, перейдите к следующему разделу этого руководства для подключения канала ExpressRoute к виртуальной сети, содержащей частное облако.

## <a name="connect-expressroute-to-the-virtual-network-gateway"></a>Подключение ExpressRoute к шлюзу виртуальной сети

В этом разделе содержится пошаговая инструкция по добавлению подключения между частным облаком AVS и созданным шлюзом виртуальной сети.

Перейдите к частному облаку, которое вы создали при работе с предыдущим руководством, и выберите **Подключение** в разделе **Управление**, а затем перейдите на вкладку **ExpressRoute**.

Скопируйте ключ авторизации. Если ключ авторизации отсутствует, его необходимо создать, выбрав действие **+ Request an authorization key** (+ Запрос ключа авторизации).

:::image type="content" source="./media/tutorial-configure-networking/request-auth-key.png" alt-text="Запрос ключа авторизации" border="true":::

Перейдите к шлюзу виртуальной сети, который вы создали на предыдущем шаге, и в разделе **Параметры** выберите **Подключения**. На странице **Подключения** выберите **+ Добавить**.

На странице **Добавление подключения** укажите значения в полях, которые перечислены в приведенной ниже таблице.

| Поле | Значение |
| --- | --- |
| **имя**;  | Введите имя для подключения.  |
| **Тип подключения:**  | Выберите **ExpressRoute**.  |
| **Активировать авторизацию**  | Обязательно установите этот флажок.  |
| **Шлюз виртуальной сети** | Это шлюз виртуальной сети, который вы создали ранее.  |
| **Ключ авторизации**  | Скопируйте и вставьте сюда ключ авторизации, полученный на вкладке ExpressRoute для группы ресурсов. |
| **URI одноранговой цепи**  | Скопируйте и вставьте сюда идентификатор ExpressRoute, полученный на вкладке ExpressRoute для группы ресурсов.  |

Щелкните **ОК**. В результате создается подключение между каналом ExpressRoute и виртуальной сетью.

:::image type="content" source="./media/tutorial-configure-networking/add-connection.png" alt-text="Добавление подключения" border="true":::

## <a name="locate-the-urls-for-vcenter-and-nsx-manager"></a>Поиск URL-адресов для vCenter и диспетчера NSX

Чтобы входить в vCenter и диспетчер NSX, вам нужны URL-адреса веб-клиента vCenter и веб-сайта диспетчера NSX-T. Чтобы найти эти URL-адреса, сделайте следующее:

Перейдите к частному облаку AVS, в разделе **Управление** выберите **Идентификатор**. Здесь вы найдете нужную информацию.

:::image type="content" source="./media/tutorial-configure-networking/locate-urls.png" alt-text="Поиск URL-адресов vCenter" border="true":::

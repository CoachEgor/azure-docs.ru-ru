# <a name="frequently-asked-questions-about-azure-vmware-solution-avs-preview"></a>Часто задаваемые вопросы о решении Azure VMware (AVS) Preview

Ответы на часто задаваемые вопросы о решении Azure VMware (AVS).

## <a name="general"></a>Общие сведения

**Что такое решение Azure VMware (AVS)**

Так как предприятия модернизации стратегии для повышения гибкости бизнеса, снижения затрат и ускорения инноваций, гибридные облачные платформы появилось в качестве ключевых созидателями цифровых преобразований клиентов. AVS объединяет программное обеспечение программного обеспечения центра обработки данных (SDDC) VMware с Microsoft Azure глобальной экосистемой облачных служб. Решение AVS управляется в соответствии с требованиями к производительности, доступности, безопасности и соответствию.

## <a name="avs-service"></a>Служба AVS

**Где доступна сегодня AVS?**

Во время предварительной версии она доступна в восточной части США в Северная Америка и в Амстердам в Западной Европе.

**Могут ли рабочие нагрузки, работающие в экземпляре решения Azure VMware (AVS), потребляться или интегрируются со службами Azure?**

Все службы Azure будут доступны клиентам с решениями AVS. Ограничения производительности и доступности для конкретных служб необходимо учитывать в каждом конкретном случае.

**Я использую те же средства, которые я использую для управления ресурсами частного облака?**

Да. Портал Azure используется для развертывания и ряда операций управления. v vCenter и НСКС Manager используются для управления ресурсами vSphere и НСКС-T.

**Можно ли управлять частным облаком с помощью локального vCenter?**

При запуске AVS не поддерживает единый интерфейс управления в локальных и частных облачных средах. Кластеры частного облака будут управляться с помощью vCenter и НСКС Manager local в частном облаке.

**Можно ли использовать Вреализе Suite, работающий локально?** 

Конкретные интеграции и варианты использования можно оценивать по отдельности.

**Можно ли перенести виртуальные машины vSphere из локальных сред в частные облака AVS?**

Да. Миграцию виртуальных машин и vMotion можно использовать для перемещения виртуальных машин в частное облако, если выполняются стандартные кросс-vhttps://kb.vmware.com/s/article/210695-vCenter [требования VMotion] [].

**Требуется ли определенная версия vSphere в локальных средах?**

Так как все облачные среды поставляются с ХККС, vSphere 5,5 или более поздней версии в локальных средах для vMotion.

**Как выглядит процесс управления изменениями?**

Обновления, вносимые в саму службу, будут следовать стандартному процессу управления изменениями Microsoft Azure. Клиенты отвечают за любые задачи администрирования рабочей нагрузки и связанные процессы управления изменениями.

**Как это отличается от решения Azure VMware с помощью Клаудсимпле?**

Благодаря новому решению Azure VMware Корпорация Майкрософт и VMware имеют прямую связь с поставщиком облачных служб. Новое решение полностью спроектировано, создано и поддерживается корпорацией Майкрософт и реализовано в VMware. В архитектуре решения являются единообразными, и в ней используется стек технологий VMware, работающий в выделенной инфраструктуре Azure.

**Если я уже имеющийся клиент решения VMware для Azure, что это означает Предварительная версия?**

Нет изменений в существующем решении Azure VMware с помощью Клаудсимпле. Мы продолжаем поддерживать решение в Azure. Решение VMware для Azure поддерживается нашим соглашением об уровне обслуживания [(SLA)](https://aka.ms/CSVMwareSLA). Клиенты должны продолжать использовать службу для рабочих нагрузок. Это доступное решение, которое регулируется [условиями службы корпорации Майкрософт](https://azure.microsoft.com/support/legal/).

**Можно ли выполнить миграцию из решения VMware для Azure, Клаудсимпле в это новое решение?**

Да, решение Azure VMware поддерживает миграцию с помощью привычных средств VMware, таких как ХККС. Для клиентов, заинтересованных в переходе к новому решению, обратитесь в службу учетная запись Майкрософт, чтобы узнать о доступных вариантах и поддержке.

## <a name="compute-network-and-storage"></a>Расчет, сеть и хранилище

**Доступно ли больше одного типа узла?**

Доступен только один тип узла.

**Каковы спецификации ЦП каждого типа узла?**

На серверах установлены два ЦП Intel с частотой от 18 до 2,3 ГГц.

**Какой объем памяти находится на каждом узле?**

На серверах установлено 576 ГБ ОЗУ.

**Какова емкость хранилища каждого узла?**

Каждый узел ESXi имеет два VSAN дискграупс с уровнем емкости 15,2 ТБ и уровнем кэша NVMe размером 3,2 ТБ (1,6 ТБ в каждом дискграуп).

**Какой объем пропускной способности сети доступен на каждом узле ESXi?**

Узлы ESXi поддерживают пропускную способность подключения до 25 Гбит/с.

**Хранятся ли данные в хранилищах данных VSAN, зашифрованных при хранении?**

Да, все данные VSAN шифруются по умолчанию с помощью ключей, хранящихся в Azure Key Vault.

## <a name="hosts-clusters-and-private-clouds"></a>Узлы, кластеры и частные облака

**Является ли базовая инфраструктура общей?**

Нет, узлы частного облака и кластеры выделены и безопасно удаляются до и после использования.

**Каково минимальное и максимальное число узлов на кластер?**

Кластеры могут масштабироваться между 3 и 16 узлами ESXi. Пробные кластеры ограничены тремя узлами.

**Можно ли масштабировать кластеры частного облака?**

Да, кластеры масштабируются между минимальным и максимальным числом узлов ESXi. Пробные кластеры ограничены тремя узлами.

**Что такое пробные кластеры?**

Пробные кластеры — это три кластера узлов, которые используются для оценки частных облаков AVS на один месяц.

**Можно ли использовать высокопроизводительные узлы для пробных кластеров?**

Нет. Узлы ESXi высокого уровня зарезервированы для использования в рабочих кластерах.

## <a name="avs-and-vmware-software"></a>Программное обеспечение AVS и VMware

**Какие версии программного обеспечения VMware используются в частных облаках?**

В частных облаках используется vSphere 6,7, vSAN 6,7, ХККС и версия 2,5 НСКС-T.  

**Используют ли частные облака VMware НСКС?**

Да, НСКС-T 2,5 используется для программно определяемой сети в частных облаках AVS.

**Можно ли использовать VMware НСКС-V в частном облаке?**

Нет. НСКС-T — это единственная поддерживаемая версия НСКС.

**НСКС требуется в локальных средах или сетях, которые подключаются к частному облаку.**

Нет, вам не обязательно использовать НСКС локально.

**Что такое расписание обновления и обновления для программного обеспечения VMware в частном облаке?**

Обновления пакетов для частного облака выполняются для обеспечения того, чтобы программное обеспечение сохранялось в одной версии последнего выпуска пакета программного обеспечения в виде VMware. Версии программного обеспечения частного облака могут отличаться от самых последних версий отдельных программных компонентов (ESXi, НСКС-T, vCenter, VSAN).

**Как часто будет обновляться программный стек частного облака?**

Программное обеспечение частного облака обновляется по расписанию, которое отслеживается в выпуске пакета программного обеспечения из VMware. Вашему частному облаку не требуется время простоя для обновления.

## <a name="connectivity"></a>Соединение

**Какие сетевые IP-адреса необходимы для внедрения частных облаков в локальные среды?**

Для развертывания частного облака AVS требуется частная сеть/22-адресное пространство. Это частное адресное пространство не должно пересекаться с другими виртуальными сетями в подписке или с локальными сетями.
 
**Разделы справки подключиться из локальных сред к частному облаку AVS?**

Подключиться к службе можно одним из двух способов: 

- С помощью виртуальной машины или шлюза приложений, развернутого в виртуальной сети Azure, которая подключена к частному облаку через ExpressRoute.
- С помощью ExpressRoute Global Reach из локального центра обработки данных в канал Azure ExpressRoute.

**Разделы справки подключить виртуальную машину рабочей нагрузки к Интернету или конечной точке службы Azure?**

В портал Azure включите подключение к Интернету для частного облака. С помощью диспетчера НСКС-T создайте маршрутизатор НСКС-T T1 и логический коммутатор. Затем с помощью vCenter разверните виртуальную машину в сегменте сети, определенном логическим коммутатором. Эта виртуальная машина будет иметь сетевой доступ к Интернету и службам Azure.

**Нужно ли ограничивать доступ из Интернета к виртуальным машинам в частных сетях в частном облаке?**

Нет. Сетевой трафик, входящий из Интернета напрямую в частные облака, не разрешен.

**Нужно ли ограничивать доступ через Интернет от виртуальных машин в логических сетях к Интернету?**

Да. Чтобы создать брандмауэр, который будет ограничивать доступ к виртуальной машине через Интернет, необходимо использовать диспетчер НСКС-T.

## <a name="accounts-and-privileges"></a>Учетные записи и привилегии

**Какие учетные записи и привилегии получит мое новое частное облако AVS?**

Вы указали учетные данные для пользователя cloudadmin в vCenter и административного доступа в диспетчере НСКС-T. Есть также группа CloudAdmin, которую можно использовать для включения Azure Active Directory. Дополнительные сведения см. в разделе [Основные понятия доступа и удостоверений](concepts-identity.md).

**Может иметь доступ администратора к узлам ESXi?**

Нет, доступ администратора к ESXi ограничен в соответствии с требованиями к безопасности решения.

**Какие привилегии и разрешения появятся в vCenter?**

У вас будут права доступа к группе CloudAdmin. Дополнительные сведения см. в разделе [Основные понятия доступа и удостоверений](concepts-identity.md).

**Какие привилегии и разрешения у меня есть у менеджера НСКС-T?**

У вас есть полные права администратора на НСКС-T, и управление доступом на основе ролей осуществляется так же, как и с НСКС-T Data Center в локальной среде. Дополнительные сведения см. в разделе [Основные понятия доступа и удостоверений](concepts-identity.md).

> [!NOTE]
> T0 маршрутизатор создается и настраивается в рамках развертывания частного облака. Любые изменения в этом логическом маршрутизаторе или виртуальных машинах НСКС-T граничных узлов могут повлиять на подключение к частному облаку.

Вы можете отправить запрос в службу поддержки по [этой ссылке](https://portal.azure.com/#blade/Microsoft_Azure_Support/HelpAndSupportBlade/newsupportrequest) .

Вам потребуется учетная запись Azure в подписке Azure.

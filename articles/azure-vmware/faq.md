# <a name="frequently-asked-questions-about-azure-vmware-solution-avs-preview"></a>Часто задаваемые вопросы о решении Azure VMware (предварительная версия)

В статье приведены ответы на часто задаваемые вопросы о решении Azure VMware (AVS).

## <a name="general"></a>Общие сведения

**Что такое решение Azure VMware (AVS)?**

Гибридные облачные платформы появились как важнейшие решения для поддержки цифровых преобразований клиентских систем на фоне развития стратегий модернизации, нацеленных на повышение гибкости бизнеса, снижение затрат и ускорение инноваций на предприятиях. Решение Azure VMware (AVS) сочетает программно-определяемый центр обработки данных VMware (SDDC) с глобальной облачной экосистемой Microsoft Azure. Решение AVS управляется так, чтобы удовлетворять все требования к производительности, доступности, безопасности и соответствию.

## <a name="avs-service"></a>Служба AVS

**Где сейчас доступно решение AVS?**

На период предварительной версии оно предоставляется в регионах "Восточная часть США" (Северная Америка) и "Амстердам" (Западная Европа).

**Могут ли рабочие нагрузки в экземпляре решения Azure VMware (AVS) использовать службы Azure или интегрироваться с ними?**

Клиентам решения AVS будут доступны все службы Azure. Ограничения производительности и доступности для конкретных служб следует решать на индивидуальной основе.

**Будут ли у меня те же средства, которые я использую сейчас, для управления ресурсами частного облака?**

Да. Для развертывания и многих операций управления используется портал Azure. Для управления ресурсами vSphere и NSX-T используются соответственно vCenter и NSX Manager.

**Можно ли управлять частным облаком через локальный vCenter?**

На момент запуска AVS не поддерживает единый интерфейс управления для локальных и частных облачных сред. Для управления кластерами частного облака будут применяться vCenter и NSX Manager, расположенные в частном облаке.

**Можно ли использовать vRealize Suite, работающий в локальной среде?** 

Конкретные сценарии интеграции и использования могут оцениваться индивидуально.

**Можно ли перенести виртуальные машины vSphere из локальных сред в частные облака AVS?**

Да. С помощью миграции виртуальных машин и vMotion вы можете переместить виртуальные машины в частное облако, если выполняются стандартные требования vMotion [https://kb.vmware.com/s/article/210695 ] к переносу между vCenter.

**Требуется ли определенная версия vSphere в локальных средах?**

Так как во всех облачных средах установлен HCX, в локальных средах для vMotion требуется vSphere 5.5 или более поздней версии.

**Как выглядит процесс управления изменениями?**

Обновления, вносимые в саму службу, будут соответствовать стандартному процессу управления изменениями Microsoft Azure. За любые задачи администрирования рабочей нагрузки и связанные процессы управления изменениями клиенты отвечают самостоятельно.

**Чем это решение отличается от Решения Azure VMware от CloudSimple?**

Новое решение Azure VMware — это результат прямого сотрудничества корпорации Майкрософт и VMware на уровне поставщиков облачных служб. Новое решение полностью спроектировано, создано и поддерживается корпорацией Майкрософт и рекомендуется VMware. Для решений используется согласованная архитектура, при этом стек технологий VMware выполняется в выделенной инфраструктуре Azure.

**Если я уже являюсь клиентом решения Azure VMware, как на меня повлияет эта предварительная версия?**

Она никак не повлияет на имеющееся Решение Azure VMware от CloudSimple. Мы продолжаем поддерживать это решение в Azure. Решение Azure VMware поддерживается нашим [соглашением об уровне обслуживания](https://aka.ms/CSVMwareSLA). Клиентам следует использовать имено эту службу для рабочих нагрузок. Это доступное решение, на которое распространяются [условия обслуживания Майкрософт](https://azure.microsoft.com/support/legal/).

**Могу ли я перейти с Решения Azure VMware от CloudSimple на новое решение?**

Да, решение Azure VMware поддерживает миграцию с использованием привычных средств VMware, таких как HCX. Клиентам, которые заинтересованы в переходе на новое решение, следует обратиться в службу поддержки учетных записей Майкрософт, чтобы узнать о доступных вариантах и поддержке.

<a name="how-to-request-a-quota-increase-for-existing-avs"></a>**Как запросить увеличение квоты узлов для существующего решения Azure VMware?**

Увеличить квоту можно, отправив [запрос на поддержку](..\azure-portal\supportability\how-to-create-azure-support-request.md). Команда управления квотами оценивает запрос и утверждает его в течение трех рабочих дней.  

> [!IMPORTANT]
> Прежде чем запросить увеличение квоты, обязательно [зарегистрируйте поставщик ресурсов **Microsoft. AVS**](tutorial-create-private-cloud.md) на портале Azure.  
> ```azurecli-interactive
> az provider register -n Microsoft.AVS --subscription <your subscription ID>
> ```
> В [списке поставщиков и типов ресурсов](https://review.docs.microsoft.com/en-us/azure/azure-resource-manager/management/resource-providers-and-types) описано, как еще можно зарегистрировать поставщик ресурсов.

1. На портале Azure в разделе **Справка и поддержка** создайте **новый запрос на поддержку** и предоставьте в этом запросе следующие сведения:
   - **Тип вопроса:** Технические требования
   - **Подписка**: идентификатор подписки;
   - **Служба**.  Решение Azure VMware 
   - **Описание.** Повышение квоты
   - **Тип проблемы:** "Проблемы с управлением емкостью".
   - **Подтип проблемы:** "Запрос клиента на дополнительную квоту или емкость узла".

1. В описании запроса в службу поддержки укажите на вкладке "Сведения" следующее:
   - Число дополнительных узлов.   
   - Ценовая категория узлов.
   - Регион

   > [!NOTE] 
   > По умолчанию будет предоставлено не менее четырех узлов.

1. Щелкните **Проверить и создать**, чтобы отправить запрос.

## <a name="compute-network-and-storage"></a>Вычисления, сеть и хранилище

**Доступно ли более одного типа узлов?**

Существует только один доступный тип узлов.

**Каковы спецификации ЦП для каждого типа узла?**

На серверах установлены по два 18-ядерных ЦП Intel с частотой 2,3 ГГц.

**Какой объем памяти на каждом узле?**

На серверах установлено по 576 ГБ ОЗУ.

**Какая емкость хранилища на каждом узле?**

Каждый узел ESXi имеет две группы дисков VSAN с уровнем емкости 15,2 ТБ и уровень кэша NVMe 3,2 ТБ (по 1,6 ТБ для каждой группы дисков).

**Какой объем пропускной способности сети доступен на каждом узле ESXi?**

Узлы ESXi поддерживают пропускную способность подключения до 25 Гбит/с.

**Шифруются ли данные при хранении в хранилищах данных VSAN?**

Да, все данные VSAN по умолчанию шифруются с использованием ключей, которые хранятся в Azure Key Vault.

## <a name="hosts-clusters-and-private-clouds"></a>Узлы, кластеры и частные облака

**Базовая инфраструктура используется совместно с другими?**

Нет, узлы и кластеры частного облака выделяются эксклюзивно и безопасно очищаются до и после использования.

**Какое минимальное и максимальное число узлов на один кластер?**

Кластеры могут масштабироваться в диапазоне от 3 до 16 узлов ESXi. Пробные кластеры ограничены тремя узлами.

**Можно ли масштабировать кластеры частного облака?**

Да, кластеры масштабируются в диапазоне от минимального до максимального числа узлов ESXi. Пробные кластеры ограничены тремя узлами.

**Что такое пробные кластеры?**

Пробные кластеры — это кластеры из трех узлов, которые используются в течение одного месяца для оценки возможностей частных облаков AVS.

**Можно ли использовать высокопроизводительные узлы для пробных кластеров?**

Нет. Высокопроизводительные узлы ESXi зарезервированы для использования в рабочих кластерах.

## <a name="avs-and-vmware-software"></a>Программное обеспечение AVS и VMware

**Какие версии программного обеспечения VMware используются в частных облаках?**

В частных облаках используется vSphere 6.7, vSAN 6.7, HCX и NSX-T версии 2.5.  

**Используется ли VMware NSX в частных облаках?**

Да, NSX-T 2.5 используется в частных облаках AVS для программно-определяемой сети.

**Можно ли использовать VMware NSX-V в частном облаке?**

Нет. В настоящее время поддерживается только версия NSX-T.

**Требуется ли наличие NSX в локальных средах или сетях, которые подключаются к частному облаку?**

Нет, вам не обязательно использовать NSX локально.

**Что такое расписание обновлений для программного обеспечения VMware в частном облаке?**

Пакетные обновления программного обеспечения частного облака позволяют поддерживать актуальность версии программного обеспечения (с разницей не более чем на одну версию в сравнении с последним выпуском пакета программного обеспечения VMware). Для отдельных программных компонентов (ESXi, NSX-T, vCenter, VSAN) версии программного обеспечения в частном облаке могут отличаться от самых последних версий.

**Как часто будет обновляться стек программного обеспечения частного облака?**

Программное обеспечение частного облака обновляется по графику, согласованному с выпуском пакетов программного обеспечения VMware. Для обновления частного облака не требуется время простоя.

## <a name="connectivity"></a>Соединение

**Какие действия по планированию IP-адресов в сети потребуются для внедрения частных облаков в локальные среды?**

Для развертывания частного облака AVS требуется частная сеть (с поддержкой около 4 млн адресов). Это частное адресное пространство не должно пересекаться с другими виртуальными сетями в подписке или с локальными сетями.
 
**Как подключаться из локальных сред к частному облаку AVS?**

Подключиться к службе можно одним из двух способов: 

- Через виртуальную машину или шлюз приложений, развернутые в виртуальной сети Azure, для которой настроен пиринг с частным облаком через ExpressRoute.
- Через канал ExpressRoute Global Reach от локального центра обработки данных к каналу Azure ExpressRoute.

**Как подключить виртуальную машину с рабочей нагрузкой к Интернету или к конечной точке службы Azure?**

Включите подключение к Интернету для частного облака на портале Azure. С помощью NSX-T Manager создайте маршрутизатор NSX-T T1 и логический коммутатор. Затем с помощью vCenter разверните виртуальную машину в том сегменте сети, который определен этим логическим коммутатором. Эта виртуальная машина получит сетевой доступ к Интернету и к службам Azure.

**Нужно ли ограничивать доступ из Интернета к виртуальным машинам в логических сетях частного облака?**

Нет. Сетевой трафик из Интернета напрямую в частные облака не пропускается.

**Нужно ли ограничивать доступ к Интернету из виртуальных машин в логических сетях?**

Да. Вам нужно с помощью NSX-T Manager создать брандмауэр, который ограничивает доступ к Интернету из виртуальных машин.

## <a name="accounts-and-privileges"></a>Учетные записи и привилегии

**Какие учетные записи и привилегии я получу после развертывания частного облака AVS?**

Вам предоставляются учетные данные для пользователя cloudadmin в vCenter и административный доступ к NSX-T Manager. Также доступна группа CloudAdmin, которую можно использовать для внедрения Azure Active Directory. Подробнее о концепциях доступа и удостоверений см. [здесь](concepts-identity.md).

**Если ли у меня доступ администратора к узлам ESXi?**

Нет, доступ администратора к узлам ESXi ограничен в соответствии с требованиями к безопасности этого решения.

**Какие привилегии и разрешения я получу в vCenter?**

У вас будут права группы CloudAdmin. Подробнее о концепциях доступа и удостоверений см. [здесь](concepts-identity.md).

**Какие привилегии и разрешения я получу в NSX-T Manager?**

У вас будут полные права администратора в NSX-T и возможность управления доступом на основе ролей (аналогично при использовании NSX-T Data Center в локальной среде). Подробнее о концепциях доступа и удостоверений см. [здесь](concepts-identity.md).

> [!NOTE]
> В рамках развертывания частного облака создается и настраивается маршрутизатор T0. Любые изменения в этом логическом маршрутизаторе или на виртуальных машинах граничных узлов NSX-T могут повлиять на возможность подключения к частному облаку.

## <a name="billing-and-support"></a>Выставление счетов и поддержка

**Как будет взиматься плата за использование предварительной версии AVS?**

Выставление счетов за AVS на период предварительной версии осуществляется ежемесячно с оплатой по мере использования. С выходом общедоступной версии появятся дополнительные варианты.

**Какая ценовая структура будет реализована для предварительной версии AVS?**

Общие вопросы по ценам см. на странице [цен на решение Azure VMware](https://azure.microsoft.com/pricing/details/azure-vmware). Сведения о ценах на предварительную версию доступны по запросу. Обратитесь в службу поддержки вашей учетной записи или перейдите по ссылке на страницу цен, чтобы связаться с отделом продаж.

**Кто осуществляет поддержку AVS?**

Поддержка для AVS предоставляется корпорацией Майкрософт. Обратите внимание, что мы предоставляем поддержку в рабочее время, с 9:00 до 17:00 по тихоокеанскому времени, с понедельника по пятницу (согласно нашим рекомендациям для предварительной версии). Чтобы создать запрос в службу поддержки, воспользуйтесь [этой ссылкой](https://portal.azure.com/#blade/Microsoft_Azure_Support/HelpAndSupportBlade/newsupportrequest).

**Какие учетные записи потребуются для создания частного облака AVS?**

Вам потребуется учетная запись Azure в подписке Azure.

# <a name="role-based-access-control-rbac-for-azure-vmware-solution"></a>Управление доступом на основе ролей (RBAC) для решения Azure VMware

В vCenter и ESXi локального развертывания администратор имеет доступ к administrator@vsphere.local учетной записи vCenter и может иметь дополнительные Active Directory (AD) пользователей и групп. Однако в развертывании решения Azure VMware администратор не имеет доступа к учетной записи администратора, но может назначать пользователей и группы AD роли CloudAdmin в vCenter.  Кроме того, пользователь частного облака решения Azure VMware не имеет разрешения на доступ или настройку конкретных компонентов управления, поддерживаемых и управляемых корпорацией Майкрософт, например кластерами, узлами, хранилищами данных и распределенными виртуальными коммутаторами.


В решении VMware для Azure vCenter имеется встроенный локальный пользователь с именем cloudadmin, назначенный встроенной роли CloudAdmin. Локальный пользователь cloudadmin используется для настройки дополнительных пользователей в AD. Роль CloudAdmin в целом имеет право создавать рабочие нагрузки в частном облаке (виртуальные машины, пулы ресурсов, хранилища данных и сети) и управлять ими. Роль CloudAdmin в решении VMware для Azure имеет специальный набор привилегий vCenter, отличающийся от других облачных решений VMware.   

> [!NOTE]
> Сейчас решение VMware для Azure не предлагает настраиваемые роли в vCenter или на портале решения VMware для Azure. 

## <a name="azure-vmware-solution-cloudadmin-role-on-vcenter"></a>Роль CloudAdmin решения Azure VMware в vCenter

Вы можете просмотреть привилегии, предоставленные роли CloudAdmin решения Azure VMware в частном облаке Azure для решения VMware.

1. Войдите в клиент vSphere SDDC и перейдите в **меню**  >  **Администрирование**.
1. В разделе **Управление доступом**выберите **роли**.
1. В списке ролей выберите **CloudAdmin** и щелкните **привилегии**.

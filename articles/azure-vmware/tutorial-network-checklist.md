# <a name="networking-checklist-for-azure-vmware-solution-avs"></a>Контрольный список для сети в решении Azure VMware (AVS)

Решение Azure VMware предоставляет среду частного облака VMware, доступную для пользователей и приложений из локальной среды и размещенных в Azure сред или ресурсов. Подключение предоставляется через такие сетевые службы, как Azure ExpressRoute и VPN-подключения, а для их работы потребуются определенные диапазоны адресов и порты брандмауэра. В этой статье представлены сведения, необходимые для правильной настройки сетевой среды для работы с AVS.

В этом руководстве описано следующее:

> [!div class="checklist"]
> * Требования к сетевому подключению.
> * DHCP в AVS.

## <a name="network-connectivity"></a>Сетевое подключение

Частное облако AVS подключается к виртуальной сети Azure через подключение Azure ExpressRoute. Его высокая пропускная способность и низкая задержка позволяют обращаться из среды частного облака к службам, работающим в подписке Azure.

Для частных облаков AVS требуется блок размером не менее `/22` сетевых адресов CIDR, в котором будут размещаться описанные ниже подсети. Эта сеть дополняет существующие локальные сети. Чтобы подключить ее к локальным средам и виртуальным сетям, их блоки сетевых адресов не должны перекрываться.

Пример блока из `/22` сетевых адресов CIDR: `10.10.0.0/22`.

Подсети.

| Назначение сети             | Подсеть | Пример        |
| ------------------------- | ------ | -------------- |
| Управление частным облаком            | `/24`    | `10.10.0.0/24`   |
| Сеть vMotion       | `/24`    | `10.10.1.0/24`   |
| Рабочие нагрузки виртуальных машин | `/24`   | `10.10.2.0/24`   |
| Пиринг ExpressRoute | `/24`    | `10.10.3.8/30`   |

## <a name="dhcp"></a>DHCP

Приложениям и рабочим нагрузкам, которые выполняются в среде частного облака, требуются разрешение имен и службы DHCP для назначения IP-адресов и поиска по ним. Для предоставления этих служб требуется определенная инфраструктура DHCP и DNS. Вы можете настроить виртуальную машину для предоставления этих служб в среде частного облака.  
Мы рекомендуем использовать встроенную в NSX службу DHCP или локальный DHCP-сервер в частном облаке вместо того, чтобы направлять широковещательный трафик DHCP через глобальную сеть обратно в локальную сеть.

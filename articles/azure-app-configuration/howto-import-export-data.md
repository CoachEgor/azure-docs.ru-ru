# <a name="import-or-export-configuration-data"></a>Импорт и экспорт данных конфигурации

Конфигурация приложений Azure поддерживает операции по импорту и экспорту данных. Используйте эти операции для работы с данными о настройках оптом и обмена данными между магазином конфигурации приложений и кодовым проектом. Например, можно настроить один магазин конфигурации приложений для тестирования, а другой для производства. Вы можете скопировать настройки приложения между ними, так что вам не придется вводить данные дважды.

В этой статье содержится руководство по импорту и экспорту данных с помощью конфигурации приложений. Если вы хотите настроить постоянную синхронизацию с репо GitHub, взгляните на наше [действие GitHub.](https://aka.ms/azconfig-gha1)

## <a name="import-data"></a>Импорт данных

Импорт приносит данные конфигурации в магазин конфигурации приложений из существующего источника. Используйте функцию импорта для переноса данных в хранилище конфигурации приложений или агрегирования данных из нескольких источников. Конфигурация приложений поддерживает импорт из файла JSON, YAML или свойств.

Импортируйте данные с помощью [портала Azure](https://portal.azure.com) или [Azure CLI.](./scripts/cli-import.md) На портале Azure сделайте следующее:

1. Просматривайте магазин конфигурации приложений и выберите **импорт/экспорт** из меню **операций.**

1. На вкладке **«Импорт»** выберите**файл конфигурации** **службы** > Исхода.

1. Выберите **для языка** и выберите желаемый тип ввода.

1. Выберите значок **Folder** и просмотрите файл для импорта.

    ![Файл импорта](./media/import-file.png)

1. Выберите **сепаратор**и по желанию введите **префикс** для использования для импортируемых ключевых имен.

1. Дополнительно выберите **этикетку**.

1. Выберите **применить,** чтобы закончить импорт.

    ![Импортный файл завершен](./media/import-file-complete.png)

## <a name="export-data"></a>Экспорт данных

Экспорт записывает данные конфигурации, хранящиеся в конфигурации приложения, в другой пункт назначения. Например, используйте функцию экспорта для сохранения данных в магазине конфигурации приложений в файл, встроенный в код приложения во время развертывания.

Экспортные данные с помощью [портала Azure](https://portal.azure.com) или [Azure CLI.](./scripts/cli-export.md) На портале Azure сделайте следующее:

1. Просмотрите в магазине конфигурации приложений и выберите **импорт/экспорт.**

1. На вкладке **«Экспорт»** выберите**файл конфигурации** **целевого обслуживания.** > 

1. Дополнительно введите **префикс** и выберите **этикетку** и точку во времени для экспорта ключей.

1. Выберите**сепаратор** **типа** > файла.

1. Выберите **применить,** чтобы закончить экспорт.

    ![Экспортный файл завершен](./media/export-file-complete.png)

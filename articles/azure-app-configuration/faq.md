# <a name="azure-app-configuration-faq"></a>Конфигурация Azure приложения часто задаваемые вопросы

Этой статье приведены ответы на часто задаваемые вопросы о паролях приложений Azure.

## <a name="how-is-app-configuration-different-from-key-vault"></a>Чем отличается конфигурацию приложения из Key Vault?

Конфигурация приложения предназначен для отдельного набора вариантов использования: она помогает разработчикам управлять параметрами приложения и контролировать доступность функций. Он стремится упростить многие задачи работы с данными сложной конфигурации. Он поддерживает иерархического пространства имен, добавления меток, сложных запросов, пакетное извлечение и операции специализированного управления и пользовательские интерфейсы. Конфигурация приложения представляет собой дополнение к Key Vault и двух следует использовать в большинстве развертываний приложения рядом друг с другом.

## <a name="should-i-store-secrets-in-app-configuration"></a>Следует ли мне хранить секреты в конфигурации приложения?

Конфигурация приложения предоставляет жесткой безопасности, хранилище ключей по-прежнему — лучшее место для хранения секретных данных приложения. Он обеспечивает шифрование на уровне оборудования, детализированные политики доступа и операции управления, такие как Смена сертификатов.

## <a name="does-app-configuration-encrypt-my-data"></a>Выполняет конфигурацию приложения шифрование Мои данные?

Да. Конфигурация приложения шифрует все значения ключей, он содержит и сетевых подключений. Имена ключей используются в качестве индексов для получения данных конфигурации и не шифруются.

## <a name="does-app-configuration-support-azure-virtual-network-vnet"></a>Поддерживает ли конфигурация приложения Azure виртуальная сеть (VNET)?

Пока нет. Общедоступной в будущем планируется поддержка виртуальной сети.

## <a name="how-should-i-store-configurations-for-multiple-environments-test-staging-production-and-so-on"></a>Как следует хранить конфигурации для нескольких сред (тестирования, промежуточного хранения, производства и т. д)?

В настоящее время можно управлять, кто имеет доступ к конфигурации приложений на уровне на хранилище. Следует использовать отдельное хранилище для каждой среды, требует разных разрешений. Такой подход обеспечивает лучшие изоляции и безопасности.

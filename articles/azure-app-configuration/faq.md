# <a name="azure-app-configuration-faq"></a>Конфигурация Azure приложения часто задаваемые вопросы

В этой статье рассматриваются часто задаваемые вопросы о паролях приложений Azure.

## <a name="how-is-app-configuration-different-from-azure-key-vault"></a>Чем отличается служба "Конфигурация приложений" от решения Azure Key Vault?

Конфигурация приложения предназначен для отдельного набора вариантов использования: она помогает разработчикам управлять параметрами приложения и контролировать доступность функций. Он стремится упростить многие задачи работы с данными сложной конфигурации.

Поддерживает конфигурации приложения:

- Иерархических пространств имен
- Добавление меток
- Сложных запросов
- Пакетное извлечение
- Операции управления специализированные
- Пользовательский интерфейс управления функциями

Конфигурация приложения представляет собой дополнение к Key Vault и двух следует использовать в большинстве развертываний приложения рядом друг с другом.

## <a name="should-i-store-secrets-in-app-configuration"></a>Следует ли мне хранить секреты в конфигурации приложения?

Несмотря на то, что конфигурация приложения обеспечивает безопасность усиленной защитой, Key Vault по-прежнему является оптимальным местом для хранения секретных данных приложения. Key Vault обеспечивает шифрование на уровне оборудования, детализированные политики доступа и операций управления, таких как Смена сертификатов.

## <a name="does-app-configuration-encrypt-my-data"></a>Выполняет конфигурацию приложения шифрование Мои данные?

Да. Конфигурация приложения шифрует все значения ключей, которые он удерживает, и он выполняет шифрование обмена данными. Имена ключей используются в качестве индексов для получения данных конфигурации и не шифруются.

## <a name="how-should-i-store-configurations-for-multiple-environments-test-staging-production-and-so-on"></a>Как следует хранить конфигурации для нескольких сред (тестирования, промежуточного хранения, производства и т. д)?

В настоящее время можно управлять, кто имеет доступ к конфигурации приложений на уровне на хранилище. Используйте отдельное хранилище для каждой среды, требует разных разрешений. Такой подход обеспечивает лучшие изоляции и безопасности.

# <a name="use-azure-data-factory-to-migrate-data-from-your-data-lake-or-data-warehouse-to-azure"></a>Перенос данных из хранилища данных в Azure с помощью фабрики данных Azure 

Фабрика данных Azure может быть средством переноса данных, если вы хотите перенести хранилище данных Data Lake или Enterprise (EDW) в Azure. Миграция Data Lake и перенос хранилища данных связаны со следующими сценариями. 

- Перенос рабочей нагрузки больших данных из AWS S3, локальной файловой системы Hadoop в Azure. 
- EDW миграция из Oracle Ексадата, Netezza, Teradata, AWS RedShift в Azure. 

Фабрика данных Azure может перемещать PBs данных для миграции Data Lake и десятки ТБ данных для переноса хранилища данных. 

## <a name="why-azure-data-factory-can-be-used-for-data-migration"></a>Почему фабрика данных Azure может использоваться для переноса данных 

- Фабрика данных Azure позволяет легко масштабировать мощность, чтобы переместить данные на бессерверный режим с высокой производительностью, устойчивостью и масштабируемостью, а также заплатить только за те ресурсы, которые используете.  
  - Фабрика данных Azure не имеет ограничений на объем данных и число файлов.
  - Фабрика данных Azure может 100% использовать пропускную способность сети и хранилища для достижения максимальной пропускной способности перемещения данных в вашей среде.   
  - Фабрика данных Azure соответствует стратегии оплаты по мере использования, поэтому необходимо оплатить только время, когда вы используете фабрику данных Azure для переноса данных в Azure.  
- Фабрика данных Azure может выполнять однократную загрузку, а также запланированную добавочную загрузку. 
- Фабрика данных Azure использует Azure IR для перемещения данных между общедоступными конечными точками Data Lake/Warehouse или с помощью самостоятельно размещенного IR для перемещения данных конечных точек Data Lake/Warehouse в виртуальной сети или за брандмауэром. 
- В фабрике данных Azure есть безопасность корпоративного уровня: используйте MSI или удостоверение службы для безопасной интеграции между службами или Azure Key Vault для управления учетными данными. 
- Фабрика данных Azure предоставляет возможности для разработки без поддержки кода и многофункциональной встроенной панели мониторинга.  

## <a name="online-vs-offline-data-migration"></a>Оперативная и автономная миграция данных

Фабрика данных Azure — это типичное средство переноса данных по сети (Интернет, ER или VPN), при котором автономная миграция данных позволяет физическим получателям передавать данные с устройств из вашей организации в центр обработки данных Azure.  

При выборе способа перехода в режим «в сети» и автономной миграции необходимо учитывать три основных фактора.  

- Размер данных для переноса. 
- Пропускная способность сети. 
- Окно миграции.   

Если вы хотите выполнить миграцию данных в течение двух недель (окно миграции), на рисунке ниже отображается вырезанная линия, в которой можно увидеть, когда лучше использовать средство оперативной миграции (фабрика данных Azure) с различными размерами данных и пропускной способностью сети.   

![в сети и вне сети](media/data-migration-guidance-overview/online-offline.png)

> [!NOTE]
> Преимущество интерактивной миграции заключается в том, что можно достичь как для загрузки исторических данных, так и для добавочных веб-каналов.  Таким образом, данные можно синхронизировать между существующим и новым магазинами во время всего окна миграции, чтобы можно было перестроить логику ETL в новом хранилище с обновленными данными.

# <a name="troubleshoot-data-flows-in-azure-data-factory"></a>Потоки данных о неполадок в Фабрике данных Azure

В этой статье рассматриваются общие методы устранения неполадок для потоков данных на фабрике данных Azure.

## <a name="common-errors-and-messages"></a>Общие ошибки и сообщения

### <a name="error-code-df-executor-sourceinvalidpayload"></a>Код ошибки: DF-Исполнитель-ИсточникInvalidPayload
- **Сообщение**: Предварительный просмотр данных, отладка и выполнение потока данных конвейера не удалось из-за того, что контейнер не существует
- **Причины**: Когда набор данных содержит контейнер, который не существует в хранилище
- **Рекомендация**: Убедитесь, что контейнер, на который ссылается в наборе данных, существует или доступен.

### <a name="error-code-df-executor-systemimplicitcartesian"></a>Код ошибки: DF-Исполнитель-СистемаПротекальныйКарпесиан

- **Сообщение**: Неявный декартовый продукт для соединения INNER не поддерживается, используйте CROSS JOIN вместо. Столбцы, используемые в соединении, должны создать уникальный ключ для строк.
- **Причины**: Неявный декартовый продукт для ВНУТРЕННЕГО присоединения между логическими планами не поддерживается. Если столбцы, используемые в соединении, создают уникальный ключ, требуется по крайней мере один столбец с обеих сторон отношений.
- **Рекомендация**: Для неравноправных присоединяется вы должны выбрать CUSTOM CROSS JOIN.

### <a name="error-code-df-executor-systeminvalidjson"></a>Код ошибки: DF-Исполнитель-SystemInvalidJson

- **Сообщение**: ошибка разбора JSON, неподдерживаемое кодирование или многолинейная
- **Причины**: Возможные проблемы с файлом JSON: неподдерживаемые кодирования, поврежденные байты, или использование источника JSON в качестве единого документа на многих вложенных линиях
- **Рекомендация**: Проверка кодирования файла JSON поддерживается. На преобразовании Источника, используюем набор данных JSON, расширьте параметры JSON и включите «Единый документ».
 
### <a name="error-code-df-executor-broadcasttimeout"></a>Код ошибки: DF-Исполнитель-ВещаниеTimeout

- **Сообщение**: Трансляция присоединиться к ошибке тайм-аута, убедитесь, что поток вещания производит данные в течение 60 секунд в отладке работает и 300 сек в работе работает
- **Причины**: Трансляция имеет тайм-аут по умолчанию 60 сек в отладке работает и 300 секунд в работе работает. Поток, выбранный для трансляции, кажется большим для получения данных в пределах этого предела.
- **Рекомендация**: Избегайте трансляции больших потоков данных, где обработка может занять более 60 секунд. Выберите меньший поток для трансляции вместо этого. Большие таблицы S'L/DW и исходные файлы, как правило, являются плохими кандидатами.

### <a name="error-code-df-executor-conversion"></a>Код ошибки: DF-Исполнитель-Конверсия

- **Сообщение**: Преобразование на дату или время не удалось из-за недействительного символа
- **Причины**: Данные не в ожидаемом формате
- **Рекомендация**: Используйте правильный тип данных

### <a name="error-code-df-executor-invalidcolumn"></a>Код ошибки: DF-Исполнитель-НедействительныйКолмерия

- **Сообщение**: Имя столбца должно быть указано в запросе, установить псевдоним при использовании функции S'L
- **Причины**: Имя столбца не было указано
- **Рекомендация**: Установите псевдоним при использовании функции S'L, такой как мин()/макс() и т.д.

## <a name="general-troubleshooting-guidance"></a>Общее руководство по устранению неполадок

1. Проверьте состояние подключений к набору данных. В каждой трансформации Источника и раковины посетите службу Linked Service для каждого набора данных, который вы используете, и тестируете соединения.
1. Проверьте состояние подключения файлов и таблицы от разработчика потока данных. Включите Debug и нажмите на просмотр данных на преобразованиях Исхода, чтобы убедиться, что вы сможете получить доступ к вашим данным.
1. Если все выглядит хорошо из предварительного просмотра данных, перейдите в конструктор конвейера и поместите поток данных в действие конвейера. Отогивать конвейер для сквозного тестирования.

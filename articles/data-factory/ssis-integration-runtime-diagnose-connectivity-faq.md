# <a name="troubleshoot-diagnose-connectivity-in-the-ssis-integration-runtime"></a>Устранение неполадок при диагностике подключений в среде выполнения интеграции SSIS

[!INCLUDE[appliesto-adf-xxx-md](includes/appliesto-adf-xxx-md.md)]

Если при выполнении пакетов служб SSIS в среде выполнения интеграции SSIS возникают проблемы с подключением, особенно если среда выполнения интеграции SSIS присоединена к виртуальной сети Azure. Вы можете попытаться самостоятельно диагностировать проблемы с помощью этой функции диагностики на странице мониторинг среды выполнения интеграции служб SSIS на портале фабрики данных Azure. 

 ![Страница "Мониторинг". Диагностика ](media/ssis-integration-runtime-diagnose-connectivity-faq/ssis-monitor-diagnose-connectivity.png) ![ страницы монитора подключения — проверка подключения](media/ssis-integration-runtime-diagnose-connectivity-faq/ssis-monitor-test-connection.png)
 
Эта статья содержит наиболее распространенные ошибки, которые могут возникнуть при тестировании соединения в среде выполнения интеграции SSIS. В нем описываются возможные причины и действия по устранению ошибок. 

## <a name="common-errors-potential-causes-and-recommendation-solutions"></a>Распространенные ошибки, возможные причины и рекомендации по решению рекомендаций

### <a name="error-code-invalidinput"></a>Код ошибки: Инвалидинпут.
* **Сообщение об ошибке**: Проверьте правильность входных данных.
* **Возможные причины:** Введенные данные неверны.
* **Рекомендации:** Проверьте входные данные.

### <a name="error-code-firewallornetworkissue"></a>Код ошибки: Фиреваллорнетворкиссуе.
* **Сообщение об ошибке**: Убедитесь, что этот порт открыт в брандмауэре, сервере или NSG, а сеть является стабильной.
* **Возможные причины:** 
  * Сервер не открывает этот порт.
  * Группе безопасности сети отказано в доступе к исходящему трафику через этот порт
  * Брандмауэр NVA/Azure Firewall/локальный брандмауэр не открывает этот порт.
* **Рекомендация.** 
  * Откройте этот порт на сервере.
  * Обновите группу безопасности сети, чтобы разрешить исходящий трафик через этот порт.
  * Откройте этот порт в брандмауэре NVA/Azure Firewall/локальный брандмауэр.

### <a name="error-code-misconfigureddnssettings"></a>Код ошибки: Мисконфигуредднссеттингс.
* **Сообщение об ошибке**. Если вы используете собственный DNS-сервер в виртуальной сети, присоединенной Azure-SSIS IR, убедитесь, что она может разрешить имя узла.
* **Возможные причины:** 
  *  Проблема с пользовательским DNS
  *  Для частного имени узла не используется полное доменное имя (FQDN)
* **Рекомендация.** 
  *  Исправьте пользовательскую проблему с DNS, чтобы убедиться, что она может разрешить имя узла.
  *  Используйте полное доменное имя (FQDN), например, используйте <your_private_server>. contoso.com вместо <your_private_server>, так как Azure-SSIS IR не будет автоматически добавлять собственный DNS-суффикс.

### <a name="error-code-servernotallowremoteconenction"></a>Код ошибки: Серверноталловремотеконенктион.
* **Сообщение об ошибке**: Убедитесь, что сервер разрешает удаленные подключения TCP через этот порт.
* **Возможные причины:** 
  *  Брандмауэр сервера не разрешает удаленные подключения TCP.
  *  Сервер не находится в режиме «в сети».
* **Рекомендация.** 
  *  Разрешить удаленные TCP-подключения в брандмауэре сервера.
  *  Запустите сервер.
   
### <a name="error-code-misconfigurednsgsettings"></a>Код ошибки: Мисконфигуреднсгсеттингс.
* **Сообщение об ошибке**. Убедитесь, что NSG виртуальной сети разрешает исходящий трафик через этот порт. Если вы используете Azure ExpressRoute или UDR, убедитесь, что этот порт открыт в брандмауэре или на сервере.
* **Возможные причины:** 
  *  Группе безопасности сети отказано в доступе к исходящему трафику через этот порт.
  *  Брандмауэр NVA/Azure Firewall/локальный брандмауэр не открывает этот порт.
* **Рекомендация.** 
  *  Обновите группу безопасности сети, чтобы разрешить исходящий трафик через этот порт.
  *  Откройте этот порт в брандмауэре NVA/Azure Firewall/локальный брандмауэр.

### <a name="error-code-genericissues"></a>Код ошибки: ЖенериЦиссуес.
* **Сообщение об ошибке**: не удалось проверить соединение из-за общих проблем.
* **Возможные причины:** При проверке подключения возникла общая временная проблема.
* **Рекомендации:** Повторите попытку проверки подключения позже. Если повторная попытка не поможет, обратитесь в службу поддержки фабрики данных Azure.


### <a name="error-code-pspingexecutiontimeout"></a>Код ошибки: Пспинжексекутионтимеаут.
* **Сообщение об ошибке**: время ожидания проверки соединения истекло, повторите попытку позже.
* **Возможные причины:** Истекло время ожидания для проверки подключения.
* **Рекомендации:** Повторите попытку проверки подключения позже. Если повторная попытка не поможет, обратитесь в службу поддержки фабрики данных Azure.

### <a name="error-code-networkinstable"></a>Код ошибки: Нетворкинстабле.
* **Сообщение об ошибке**: нерегулярное выполнение проверки подключения из-за нестабильности сети.
* **Возможные причины:** Временная проблема с сетью.
* **Рекомендации:** Убедитесь, что сеть сервера или брандмауэра является стабильной.

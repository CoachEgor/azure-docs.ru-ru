# <a name="visual-authoring-in-azure-data-factory"></a>Визуальная разработка в фабрике данных Azure

С пользовательским интерфейсом фабрики данных Azure пользователи могут визуально разрабатывать и развертывать ресурсы в своей фабрике, не написав не единой строки кода. Вы можете перетаскивать действия на панель холста конвейера, выполнять тестовые запуски, выполнять последовательную отладку, а также развертывать и выполнять мониторинг запуска конвейера.

В настоящее время Фабрика данных Azure UX поддерживается только в Microsoft Edge и Google Chrome.

## <a name="authoring-canvas"></a>Авторство холста

Чтобы открыть **авторское полотно,** нажмите на значок карандаша. 

![Авторство холста](media/author-visually/authoring-canvas.png)

Здесь вы будете авторить конвейеры, действия, наборы данных, связанные службы, потоки данных, триггеры и время интеграции, составляющее вашу фабрику. Для начала строительства конвейера с помощью авторского холста [см. Копию данных с помощью действия копии.](tutorial-copy-data-portal.md) 

Опыт визуального авторизации по умолчанию непосредственно работает со службой Data Factory. Интеграция Azure Repos Git или GitHub также поддерживается для обеспечения управления исходным источником и совместной работы в конвейерах фабрики данных. Чтобы узнать больше о различиях между [Source control in Azure Data Factory](source-control.md)этими авторами, см.

## <a name="expressions-and-functions"></a>Выражения и функции

Выражения и функции могут использоваться вместо статических значений для указания многих свойств в Azure Data Factory.

Чтобы указать выражение для значения свойства, выберите **Добавить динамическое содержимое** или нажмите **Alt и P,** сосредоточившись на поле.

![Добавление динамического содержимого](media/author-visually/dynamic-content-1.png)

Это открывает **систему выражения Data Factory Builder,** где можно создавать выражения из поддерживаемых системных переменных, выходных действий, функций и пользовательских переменных или параметров. 

![Построитель выражений](media/author-visually/dynamic-content-2.png)

Для получения информации о [Expressions and functions in Azure Data Factory](control-flow-expression-language-functions.md)языке выражения см.

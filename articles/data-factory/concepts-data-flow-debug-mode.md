# <a name="mapping-data-flow-debug-mode"></a>Режим отладки функции сопоставления потоков данных

[!INCLUDE [notes](../../includes/data-factory-data-flow-preview.md)]

Azure фабрики сопоставление данных поток данных в режиме отладки можно переключить с кнопкой «Данных потока Debug» в верхней части области конструктора. При разработке потоков данных, включение режима отладки позволяет интерактивно просматривать данные, преобразование фигуры при создании и отладке последовательностями данных. Сеанс отладки может использоваться как в занятиях по проектированию потока данных, а также во время выполнения отладки конвейера потоков данных.

![Отладка кнопку](media/data-flow/debugbutton.png "кнопку отладки")

## <a name="overview"></a>Обзор
Если включен режим отладки будет формировать в интерактивном режиме потока данных с кластером Spark active. Сеанс будет закрыт после отключения отладки в Фабрике данных Azure. Следует помнить о почасовой оплате, которая взимается в Azure Databricks за время, когда сеанс отладки был включен.

В большинстве случаев рекомендуется создавать ваши потоки данных в режиме отладки, чтобы можно было проверить бизнес-логики и просматривать ваши преобразования данных перед публикацией данные в фабрике данных Azure. Кнопка «Отладка» на панели «конвейер» для проверки потока данных в конвейере.

> [!NOTE]
> Во время отладки режим горит зеленым на панели инструментов фабрики данных, будет списана со скоростью потока данных отладки 8 ядер в час общих вычислительных ресурсов с 60-минутным time-to-live 

> [!NOTE]
>Если при работе с Потоком данных включен режим отладки, данные не записываются в преобразование приемника. Сеанс отладки предназначен для использования в качестве окружения теста для преобразования. Приемники не нужны во время отладки и игнорируются в потоке данных. Если вы хотите протестировать, записи данных в вашей приемника, выполнения потока данных из конвейера фабрики данных Azure и выполнения отладки из конвейера.

## <a name="debug-settings"></a>Параметры отладки
Отладка параметры можно изменить, нажав кнопку «Параметры отладки» на панели canvas потока данных. Вы можете выбрать, ограничения и/или исходный файл для каждого из ваши преобразования источника. Ограничения строк в этом параметре используются только для текущего сеанса отладки. Можно также выбрать связанной службой промежуточного хранения для исходного хранилища данных SQL. 

![Параметры отладки](media/data-flow/debug-settings.png "параметров отладки")

## <a name="cluster-status"></a>Состояние кластера
Отсутствует Признак состояния кластера в верхней части области конструктора, который станет зеленым, когда кластер будет готов для отладки. Если кластер уже готов, тогда зеленый индикатор появится практически мгновенно. Если кластер уже не был запущен перехода в режим отладки, вам придется подождать 5 – 7 минут для кластера запустить. Индикатор повернется до ее готовности.

Когда вы закончите отладку, отключите ключом Debug, чтобы кластером Azure Databricks можно завершить, и вы больше не взимается за действие отладки.

## <a name="data-preview"></a>Предварительный просмотр данных
В режиме отладки вкладка "Предварительный просмотр данных" засветится на нижней панели. Если режим отладки отключен, в Потоке данных отобразятся только текущие входящие и исходящие метаданные каждого из преобразований на вкладке "Проверка". Предварительный просмотр данных будет запрашивать только количество строк, которые были заданы как лимит в параметры отладки. Чтобы обновить предварительный просмотр данных, щелкните Fetch data (Получить данные).

![Предварительный просмотр данных](media/data-flow/datapreview.png "Предварительный просмотр данных")

## <a name="data-profiles"></a>Профили данных
Выбор отдельных столбцов на вкладке предварительного просмотра данных появится всплывающее окно диаграммы на правую сетки данных с помощью подробную статистику о каждом поле. Фабрика данных Azure будет принимать решения на основе выборки данных того типа диаграммы, который нужно отобразить. Хотя категорий, так и числовые данные с низкой кратностью будут отображаться гистограммы, показывающие частоту значение данных поля высокой мощности по умолчанию /NOT NULL диаграммы. Вы также увидите max/длина длина строковых полей, минимальных и максимальных значений числовых полей, стандартные отклонения, процентили, количество и среднее значение. 

![Статистика по столбцам](media/data-flow/stats.png "Статистика по столбцам")

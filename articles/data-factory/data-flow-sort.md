# <a name="azure-data-factory-data-flow-sort-transformations"></a>Преобразования сортировки потока данных в фабрике данных Azure



![Параметры сортировки](media/data-flow/sort.png "Сортировка")

Преобразование сортировки позволяет сортировать входящие строки в текущем потоке данных. После преобразования сортировки исходящие строки будут упорядочены в соответствии с заданными правилами. Вы можете выбрать отдельные столбцы и отсортировать их по возрастанию или по убыванию с помощью стрелки рядом с каждым полем. Если столбец необходимо изменить перед сортировкой, щелкните Computed Columns (Вычисляемые столбцы), чтобы запустить редактор выражений. Это позволит написать выражение для операции сортировки, а не просто выбрать столбец для сортировки.

## <a name="case-insensitive"></a>Без учета регистра
Вы можете включить параметр Case insensitive (Без учета регистра), чтобы не учитывать регистр при сортировке строковых или текстовых полей.

Параметр Sort Only Within Partitions (Сортировать только в пределах секции) позволяет применять секционирование данных в кластерах Spark. При сортировке входящих данных только в пределах каждой секции, можно отсортировать секционированные данные, а не весь поток данных.

Вы можете настроить порядок условий сортировки в преобразовании сортировки. Поэтому, если вам нужно переместить столбец выше в последовательности сортировки, захватите эту строку с помощью мыши и переместите ее вверх или вниз в списке сортировки.

Влияние секционирования на сортировку

Средство "Поток данных" Фабрики данных Azure работает в кластерах больших данных Spark, обрабатывая данные, распределенные между несколькими узлами и секциями. Это важно учитывать при разработке архитектуры потока данных, если в приложении используется преобразование сортировки для хранения данных в определенном порядке. Если вы решили повторно секционировать данные при последующим преобразовании, сортировка данных может нарушиться из-за перегруппировки данных.

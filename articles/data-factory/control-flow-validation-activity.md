# <a name="validation-activity-in-azure-data-factory"></a>Действия по проверке на фабрике данных Azure
Проверка может быть проведена в конвейере, чтобы гарантировать, что конвейер продолжает выполняться только после того, как он подтвердил наличие прилагаемого набора данных, что он соответствует указанным критериям или тайм-аут достигнут.


## <a name="syntax"></a>Синтаксис

```json

{
    "name": "Validation_Activity",
    "type": "Validation",
    "typeProperties": {
        "dataset": {
            "referenceName": "Storage_File",
            "type": "DatasetReference"
        },
        "timeout": "7.00:00:00",
        "sleep": 10,
        "minimumSize": 20
    }
},
{
    "name": "Validation_Activity_Folder",
    "type": "Validation",
    "typeProperties": {
        "dataset": {
            "referenceName": "Storage_Folder",
            "type": "DatasetReference"
        },
        "timeout": "7.00:00:00",
        "sleep": 10,
        "childItems": true
    }
}

```


## <a name="type-properties"></a>Свойства типа

Свойство | Описание | Допустимые значения | Обязательно
-------- | ----------- | -------------- | --------
name | Название действия «Проверка» | Строка | Да |
type | Должна быть установлена **валидация.** | Строка | Да |
набор данных | Действие блокирует выполнение до тех пор, пока не проверит существование этого набора данных и не соответствует указанным критериям или тайм-аут достигнут. Предоставляемый набор данных должен поддерживать свойство "MinimumSize" или "ChildItems". | Ссылка на набор данных | Да |
timeout | Указывает время ожидания для выполнения действия. Если значение не указано, значение по умолчанию составляет 7 дней ("7.00:00:00").. Формат d.hh:mm:ss | Строка | нет |
sleep | Задержка в секундах между попытками проверки. Если значение не указано, значение по умолчанию составляет 10 секунд. | Целое число | нет |
childItems | Проверяет, есть ли в папке элементы детского дотаза. Может быть установлен к истине : Проверить, что папка существует и что она имеет элементы. Блоки до тех пор, пока в папке не будет присутствовать хотя бы один элемент или значение тайм-аута.-ложь: Проверить, что папка существует и что она пуста. Блоки до тех пор, пока папка не пуста или пока значение тайм-аута не будет достигнуто. Если значение не указано, действие будет блокироваться до тех пор, пока папка не появится или пока тайм-аут не будет достигнут. | Логическое | нет |
minimumSize | Минимальный размер файла в байтах. Если значение не указано, значение по умолчанию составляет 0 байт | Целое число | нет |

# <a name="prepare-data-with-wrangling-data-flow"></a>Подготовка данных с помощью потока данных структурирование

## <a name="create-a-wrangling-data-flow"></a>Создание потока данных структурирование

Существует два способа создания потока данных структурирование в фабрике данных Azure. Один из способов — щелкнуть значок «плюс» и выбрать « **поток данных** » в области «ресурсы фабрики».

![Структурирование](media/wrangling-data-flow/tutorial7.png)

Другой метод находится на панели действия на холсте конвейера. Откройте элемент " **Перемещение и преобразование** " и перетащите действие **потока данных** на холст.

В обоих методах в боковой области щелкните **создать новый поток данных** и выберите **структурирование Data Flow (поток данных**). Нажмите кнопку ОК.

![Структурирование](media/wrangling-data-flow/tutorial1.png)

## <a name="author-a-wrangling-data-flow"></a>Создание потока данных структурирование

Добавьте **исходный набор данных** для потока данных структурирование. Можно либо выбрать существующий набор данных, либо создать новый. Можно также выбрать набор данных приемника. Можно выбрать один или несколько исходных наборов данных, но в настоящее время разрешен только один приемник. Выбор набора данных приемника является необязательным, но требуется хотя бы один исходный набор данных.

> [!NOTE]
> Для ограниченной предварительной версии поддерживается только текст с разделителями ADLS 2. 

![Структурирование](media/wrangling-data-flow/tutorial4.png)

Щелкните **создать** , чтобы открыть редактор гибридных веб-приложений Power Query.

![Структурирование](media/wrangling-data-flow/tutorial5.png)

Создайте поток данных структурирование, используя подготовку данных без кода. Список доступных функций см. в разделе [функции преобразования](wrangling-data-flow-functions.md)/

![Структурирование](media/wrangling-data-flow/tutorial6.png)

## <a name="running-and-monitoring-a-wrangling-data-flow"></a>Выполнение и мониторинг потока данных структурирование

Чтобы выполнить отладку потока данных структурирование в конвейере, щелкните **Отладка** на холсте конвейера. После публикации потока данных **триггер будет** выполнять запуск последнего опубликованного конвейера по требованию. Структурирование потоки данных могут планироваться с помощью всех существующих триггеров фабрики данных Azure.

![Структурирование](media/wrangling-data-flow/tutorial3.png)

Перейдите на вкладку " **монитор** ", чтобы визуализировать выходные данные запуска действия потока данных, активированного структурирование.

![Структурирование](media/wrangling-data-flow/tutorial2.png)

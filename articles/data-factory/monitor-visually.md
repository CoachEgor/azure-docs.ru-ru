# <a name="visually-monitor-azure-data-factory"></a>Визуальное наблюдение за фабрикой данных Azure

После создания и публикации конвейера на фабрике данных Azure можно связать его с триггером или вручную запустить специальный запуск. Вы можете отслеживать все провода, запускаемые на месте в пользовательском интерфейсе Azure Data Factory. Чтобы открыть мониторинг, выберите плитку **Monitor & Manage** в лезвии фабрики данных [портала Azure.](https://portal.azure.com/) Если вы уже находитесь в ADF UX, нажмите на значок **монитора** на левой боковой панели.

Все запуски фабрики данных отображаются в локальном часовом поясе браузера. Если вы измените часовой пояс, все поля даты/времени прикрепяются к выбранной.

## <a name="monitor-pipeline-runs"></a>Мониторинг выполнений конвейера

Представление мониторинга по умолчанию — это список запусков конвейера в выбранном периоде времени. Отображаются следующие столбцы.

| **Название колонки** | **Описание** |
| --- | --- |
| Pipeline Name | Имя конвейера. |
| Действия | Иконки, позволяющие просматривать сведения о действиях, отменять или повторно запускать конвейер |
| Run Start | Дата и время запуска трубопровода (MM/DD/YYYY, HH:MM:SS AM/PM) |
| Duration | Длительность выполнения (ЧЧ:ММ:СС) |
| Triggered By | Название триггера, который запустил конвейер |
| Состояние | **Не удалось**, **Удалось**, **В прогрессе**, **Отменено,** или **В очереди** |
| Заметки | Фильтруемые теги, связанные с конвейером  |
| Параметры | Параметры для выполнения конвейера (пары имен/значений) |
| Error | Если конвейер не сработал, ошибка выполнения |
| Run ID | Идентификатор выполнения конвейера. |

![Представление списка для мониторинга запусков трубопроводов](media/monitor-visually/pipeline-runs.png)

Для обновления списка проводок и выполняется действие. **Refresh** Autorefresh в настоящее время не поддерживается.

![Кнопка "Обновить"](media/monitor-visually/refresh.png)

## <a name="monitor-activity-runs"></a>Мониторинг выполнений действий

Для просмотра выполнения активности для каждого запуска конвейера выберите значок **значка «Обвина»** под столбом **«Действия».** Представление списка показывает запуски, соответствующие каждому запуску конвейера.

| **Название колонки** | **Описание** |
| --- | --- |
| Activity Name | Название деятельности внутри конвейера |
| Activity Type | Тип действия, **например, Copy,** **ExecuteDataFlow**или **AzureMLExecutePipeline** |
| Действия | Иконки, позволяющие видеть входной информацию JSON, информацию о выходе JSON или подробный опыт мониторинга, связанный с деятельностью | 
| Run Start | Дата и время начала выполнения действия (MM/DD/YYYY, HH:MM:SS AM/PM) |
| Duration | Длительность выполнения (ЧЧ:ММ:СС) |
| Состояние | **Не удалось,** **удалось**, **в прогрессе**, или **отменены** |
| Интеграция Runtime | На какой интеграции Runtime было запущено |
| Свойства пользователя | Определенные пользователем свойства действия |
| Error | Если действие не удалось, ошибка выполнения |
| Run ID | Идентификатор выполнения действия. |

![Представление списка для запусков мониторинга](media/monitor-visually/activity-runs.png)

### <a name="promote-user-properties-to-monitor"></a>Распространение свойств пользователя для отслеживания

Продвигайте любое свойство деятельности конвейера как свойство пользователя, чтобы оно стало сущностью, которую вы контролируете. Например, можно продвигать свойства **источника** и **назначения** действия копирования в конвейере в качестве свойств пользователя. Выберите **Auto Generate** для генерации свойств пользователя **Source** и **Destination** для копирования.

![Создание свойств пользователя](media/monitor-visually/monitor-user-properties-image1.png)

> [!NOTE]
> Вы можете продвигать только до пяти свойств активности конвейера в качестве свойств пользователя.

После создания свойств пользователей можно отслеживать их в представлениях списка мониторинга. Если источником действия копии является имя таблицы, можно отслеживать имя исходной таблицы в качестве столбца в представлении списка для запусков.

![Список выполнений действия без свойств пользователя](media/monitor-visually/monitor-user-properties-image2.png)

![Добавление столбцов для свойств пользователей в список запусков деятельности](media/monitor-visually/monitor-user-properties-image3.png)

![Список выполнений действия со столбцами для свойств пользователя](media/monitor-visually/monitor-user-properties-image4.png)

## <a name="configure-the-list-view"></a>Настройка представления списка

### <a name="order-and-filter"></a>Заказ и фильтр

Переключать ли трубопровод работает будет в спуске или восходящей в соответствии с временем запуска. Фильтр трубопровода работает с помощью следующих столбцов:

| **Название колонки** | **Описание** |
| --- | --- |
| Pipeline Name | Фильтр по названию конвейера. |
| Run Start |  Определите временной диапазон отображаемых запусков конвейера. Варианты включают быстрые фильтры за **последние 24 часа,** **на прошлой неделе**и **за последние 30 дней** или выбрать пользовательскую дату и время. |
| Run Status | Фильтр проходит по статусу: **Успешно,** Не **выполнено, в очереди,** **отменено**или **в процессе.** **Failed** |
| Заметки | Фильтр по тегам, применяемым к каждому конвейеру |
| Запуски | Фильтр, хотите ли вы видеть провода повторного ран |

![Варианты фильтрации](media/monitor-visually/filter.png)

### <a name="add-or-remove-columns"></a>Добавление и удаление столбцов
Нажмите правой кнопкой мыши заголовок представления списка и выберите столбцы, которые вы хотите отображить в представлении списка.

![Варианты для столбцов](media/monitor-visually/columns.png)

### <a name="adjust-column-widths"></a>Настройка ширины столбцов
Увеличьте и уменьшите ширину столбца в представлении списка, зависнув над заголовком столбца.

## <a name="rerun-activities-inside-a-pipeline"></a>Повторное выполнение действий в конвейере

Можно повторно запустить действия внутри конвейера. Выберите **запуски активности просмотра,** а затем выберите действие в конвейере, из какой точки требуется повторно запустить конвейер.

![Просмотр выполнений действий](media/monitor-visually/rerun-activities-image1.png)

![Выбор выполнения действия](media/monitor-visually/rerun-activities-image2.png)

### <a name="rerun-from-failed-activity"></a>Повтор из неудавшейся деятельности

Если действие не удается, время выбивания или отменено, можно повторно запустить конвейер из этого неудавшегося действия, выбрав **повтор из неудавшегося действия.**

![Повторное вылет действия](media/monitor-visually/rerun-failed-activity.png)

### <a name="view-rerun-history"></a>Просмотр журнала повторного выполнения

Вы можете просмотреть журнал повторного выполнения для всех выполнений конвейера в представлении списка.

![Просмотр истории](media/monitor-visually/rerun-history-image1.png)

Вы также можете просмотреть журнал повторного выполнения для запуска определенного конвейера.

![Просмотр журнала для выполнения конвейера](media/monitor-visually/rerun-history-image2.png)

## <a name="alerts"></a>видны узлы

Вы можете создавать оповещения на основе поддерживаемых метрик фабрики данных. Для начала выберите **monitor** > **Alerts & метрики** на странице мониторинга Data Factory.

![Страница монитора фабрики данных](media/monitor-visually/alerts01.png)

Уделите 7 минут своего времени, чтобы просмотреть следующее видео с кратким обзором и демонстрацией этой функции.

> [!VIDEO https://channel9.msdn.com/shows/azure-friday/Monitor-your-Azure-Data-Factory-pipelines-proactively-with-alerts/player]

### <a name="create-alerts"></a>Создание оповещений

1.  Выберите **новое правило оповещения** для создания нового оповещения.

    ![Кнопка «Новое правило оповещения»](media/monitor-visually/alerts02.png)

1.  Укажите имя правила и выберите серьезность оповещения.

    ![Коробки для названия правила и тяжести](media/monitor-visually/alerts03.png)

1.  Выберите критерии оповещения.

    ![Коробка для целевых критериев](media/monitor-visually/alerts04.png)

    ![Список критериев](media/monitor-visually/alerts05.png)

1.  Наверстойная логика оповещения. Можно создать оповещение для выбранной метрики для всех конвейеров и соответствующих действий. Можно также выбрать определенный тип действия, имя действия, имя конвейера или тип сбоя.

    ![Параметры настройки логики оповещения](media/monitor-visually/alerts06.png)

1.  Назначьте электронную почту, SMS, push и голосовые уведомления для оповещения. Создайте группу действий или выберите существующую группу для уведомлений о помнях.

    ![Варианты настройки уведомлений](media/monitor-visually/alerts07.png)

    ![Варианты добавления уведомления](media/monitor-visually/alerts08.png)

1.  Создайте правило генерации оповещений.

    ![Параметры для создания правила оповещения](media/monitor-visually/alerts09.png)

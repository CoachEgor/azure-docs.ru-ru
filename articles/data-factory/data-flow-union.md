# <a name="azure-data-factory-mapping-data-flow-union-transformation"></a>Преобразование соединения потока данных Azure Data Factory отображение

Операция объединения группирует несколько потоков данных в один, при этом результаты команды SQL Union для таких потоков используются в качестве новых выходных данных для преобразования с объединением. Вся схема из каждого входному потока будет объединена внутри потока данных, без необходимости иметь ключ соединения.

Вы можете объединить n-число потоков в таблице настроек, выбрав значок «К» рядом с каждой настроенной строкой, включая как исходные данные, так и потоки из существующих преобразований в потоке данных.

![Union transformation](media/data-flow/union.png "Union") (Преобразование "Объединение множеств")

В этом случае можно объединить разрозненные метаданные из нескольких источников (в данном примере три разных исходных файла) и объединить их в единый поток:

![Обзор трансформации союза](media/data-flow/union111.png "Союз 1")

Для достижения этой цели добавьте дополнительные строки в настройках соединения, включив весь источник, который вы хотите добавить. Нет необходимости в общем поиске или присоединении к ключу:

![Настройки преобразования соединения](media/data-flow/unionsettings.png "Настройки соединения")

Если вы установите преобразование Select после вашего Союза, вы сможете переименовать перекрывающиеся поля или поля, которые не были названы из бессетевых источников. Нажмите на кнопку "Проверка", чтобы увидеть метаданные комбайна с 132 общими столбцов в этом примере из трех различных источников:

![Финал трансформации союза](media/data-flow/union333.png "Союз 3")

## <a name="name-and-position"></a>Имя и позиция

При выборе "союз по имени" значение столбца будет попадать в соответствующую колонку из каждого источника с новой схемой стыковки метаданных.

Если вы выберете "союз по позициям", каждое значение столбца упадет в исходное положение из каждого соответствующего источника, в результате чего появится новый комбинированный поток данных, в котором данные из каждого источника добавляются в один и тот же поток:

![Выход союза](media/data-flow/unionoutput.png "Выход союза")

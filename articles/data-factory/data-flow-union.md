# <a name="mapping-data-flow-union-transformation"></a>Сопоставление объединений преобразование потока данных

[!INCLUDE [notes](../../includes/data-factory-data-flow-preview.md)]

Операция объединения группирует несколько потоков данных в один, при этом результаты команды SQL Union для таких потоков используются в качестве новых выходных данных для преобразования с объединением. Все схемы из каждого входного потока будет объединяться внутри потока данных, без необходимости иметь ключ соединения.

N — количество потоков в таблице «параметры» можно объединять, щелкнув значок «+» рядом с каждой настроенной строки, включая источник данных как потоки из существующих преобразований в потоке данных.

![Преобразование UNION](media/data-flow/union.png "объединения")

В этом случае можно объединять разрозненные метаданных из нескольких источников (в этом примере три различные исходные файлы) и объединить их в одном потоке:

![Общие сведения о преобразованиях объединения](media/data-flow/union111.png "объединения 1")

Для этого добавьте дополнительные строки в параметрах объединения, включая все исходные, которую вы хотите добавить. Нет необходимости для общего ключа поиска или соединения:

![Параметры объединения преобразования](media/data-flow/unionsettings.png "объединения параметров")

Если после объединение выберите преобразование, можно переименовать перекрывающиеся поля или поля, которые не были названы из headerless источников. Щелкните «Проверить» для просмотра метаданных объединение с 132 столбцы в этом примере из трех различных источников:

![Окончательный объединения преобразования](media/data-flow/union333.png "объединения 3")

## <a name="name-and-position"></a>Имя и расположение

При выборе «union по имени», каждое значение столбца будет удалена в соответствующий столбец из каждого источника, с новой схемой объединенных метаданных.

При выборе «union по позиции» в исходной позиции из каждого соответствующего источника, приводит к новый объединенный поток данных, где из каждого источника данных добавляется в тот же поток удалит каждого значения в столбце:

![Выходные данные UNION](media/data-flow/unionoutput.png "объединения выходных данных")

# <a name="mapping-data-flow-parameters"></a>Сопоставление параметров потока данных

[!INCLUDE [notes](../../includes/data-factory-data-flow-preview.md)]

Потоки в фабрике данных, сопоставление данных поддерживает использование параметров. Параметры можно определить внутри определения потока данных, который затем можно использовать на протяжении всего выражения. Затем можно задать параметры вызывающий конвейер с помощью действия выполнения потока данных. У вас есть три варианта для использования для задания значения в потоке данных действия выражений:

* Использовать язык выражений потока управления конвейера для задания динамического значения
* Использовать язык выражений потока данных для задания динамического значения
* Применять эти языки выражения статического значение литерала

Эту возможность можно используйте чтобы потоки данных общего назначения, гибкий и повторного использования. Позволяют задавать параметры для настройки потока данных и выражения со следующими параметрами.

> [!NOTE]
> Чтобы использовать выражения потока управления для конвейера, параметра потока данных должно иметь строковый тип.

* Добавление действия выполнения потока данных на холсте конвейера.
* Если поток данных имеет параметры, вы увидите список доступных параметров в параметры tab.* * Щелкните в текстовом поле рядом с каждым параметром, чтобы ввести значение параметра.
* Вы можете создать выражение параметра с помощью языка выражений потока управления конвейера или выражения потоков данных.

![Поток данных параметров 3](media/data-flow/params3.png "parameters 3 потока данных")

## <a name="create-parameters-in-data-flow"></a>Создание параметров в потоке данных

![Поток данных параметров 1](media/data-flow/params1.png "параметры 1 потока данных")

Чтобы добавить параметры к потоку данных, щелкните пустое место на холсте потока данных для просмотра общих свойств. В области "Параметры" вы увидите вкладку Параметры. Щелкните новую кнопку, чтобы создать новые параметры, которые затем можно задать из конвейера, передав значения в поток данных. Введите имя параметра и выберите тип данных для каждого параметра.

Внутри выражения потока данных можно использовать параметры, с помощью значения из конвейера. Параметры, начинающихся с $ и являются неизменяемыми. Вы также найдете список доступных параметров внутри построителя на вкладке параметров. Эти значения можно использовать в выражениях, несмотря на то, что вы не можете передавать новые значения для параметров.

![Поток данных параметров со 2](media/data-flow/params2.png "параметров со 2 потока данных")

## <a name="set-data-flow-parameters-from-pipeline"></a>Задание параметров потока данных из конвейера

После создания потока данных с параметрами, теперь можно выполнить этот поток данных из конвейера с действием выполнения потока данных. Когда вы добавите это действие на холст конструктора конвейера, откроется с доступными данными параметры потока в Настройка вкладку параметров действия.

![Язык выражений параметров потока данных](media/data-flow/params4.png "язык выражений параметров потока данных")

Если щелкнуть в текстовом поле введите значения параметров, появится построитель выражений потока данных. Здесь можно ввести любое выражение или литеральных значений, что вы хотите, соответствующие тип данных параметра. Ниже приведены примеры выражение потока данных и строковый литерал в построителе выражений.

* ```toInteger(Role)```
* ```'this is my static literal string'```

Если ваш тип данных параметра является строкой, можно ввести выражение потока данных или конвейера. Если выражение конвейера, вы вместо этого появится панель выражений конвейера. Убедитесь, что Конвейерные функции внутри синтаксиса интерполяции строки с помощью "@{<expression>}", например:

```'@{pipeline().RunId}'```

![Пример параметров потока данных](media/data-flow/params5.png "пример параметров потока данных")

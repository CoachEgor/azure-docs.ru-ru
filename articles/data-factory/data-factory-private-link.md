# <a name="azure-private-link-for-azure-data-factory"></a>Частная ссылка Azure для фабрики данных Azure

[!INCLUDE[appliesto-adf-asa-md](includes/appliesto-adf-xxx-md.md)]

Приватный канал позволяет подключаться к различным службам PaaS в Azure через частную конечную точку. Список служб PaaS, поддерживающих функцию закрытых ссылок, см. на [странице документации по частной ссылке](https://docs.microsoft.com/azure/private-link/). Частная конечная точка — это частный IP-адрес в определенной виртуальной сети и подсети.

## <a name="secure-communication-between-customer-network-and-azure-data-factory-service"></a>Защита обмена данными между сетью клиентов и службой фабрики данных Azure
Чтобы защитить ресурсы Azure от атак в общедоступной сети или обеспечить их безопасную связь друг с другом, можно настроить виртуальную сеть Azure как логическое представление сети в облаке. Вы также можете подключить локальную сеть к виртуальной сети, настроив VPN-подключение IPSec ("сеть — сеть") или ExpressRoute (частный пиринг). Локальная Integration Runtime может быть установлена на локальном компьютере или виртуальной машине в виртуальной сети для выполнения действий копирования между облачным хранилищем данных и хранилищем данных в частной сети или для отправки действий по преобразованию на ресурсы вычислений в локальной сети или в виртуальной сети Azure. 

Между фабрикой данных и виртуальной сетью клиента требуется несколько каналов связи.


| **Домен** | **порт**. | **Описание** |
| ---------- | -------- | --------------- |
| `adf.azure.com` | 443 | Плоскость управления. Требуется для создания и мониторинга фабрики данных. |
| `*.{region}.datafactory.azure.net` | 443 | Требуется для самостоятельно размещенного Integration Runtime для подключения к службе фабрики данных. |
| `*.servicebus.windows.net` | 443 | Требуется для автономного размещения Integration Runtime для интерактивной разработки. |
| `download.microsoft.com` | 443 | Требуется для самостоятельно размещенного Integration Runtime для загрузки обновлений. |


Благодаря поддержке частной связи Azure для фабрики данных Azure можно создать частную конечную точку (PE) в виртуальной сети и включить частное подключение к определенной фабрике данных Azure. Связь с службой фабрики данных Azure осуществляется через частную связь Azure, которая обеспечивает защищенное частное подключение. И вам не нужно настраивать домен и порт в виртуальной сети или брандмауэре, обеспечивающем более безопасный способ защиты ресурсов.  

![Архитектура частной связи фабрики данных Azure](./media/data-factory-private-link/private-link-architecture.png)

Ниже приведены преимущества включения службы Private Link для каждого из каналов связи, показанных выше:
- Поддерживается Вы можете создавать и отслеживать фабрику данных Azure в виртуальной сети, даже если вы блокируете все исходящие подключения.
- Поддерживается Связь между локальными Integration Runtime и службой фабрики данных Azure может быть безопасно выполнена в среде частной сети. Трафик между локальными Integration Runtime и службой фабрики данных Azure переходит по частной ссылке. 
- (Сейчас не поддерживается) Интерактивная разработка с использованием самостоятельно размещенного Integration Runtime переходить по частной ссылке, например проверить подключение, просмотреть список папок и список таблиц, получить схему и просмотреть данные.
- (Сейчас не поддерживается) Новая версия автономного Integration Runtime может быть автоматически загружена из центра загрузки, если включено автоматическое обновление.

> [!NOTE]
> Для функций, которые сейчас не поддерживаются, необходимо настроить выше домен и порт в виртуальной сети или корпоративном брандмауэре. 

> [!WARNING]
> При создании связанной службы убедитесь, что учетные данные хранятся в Azure Key Vault. В противном случае она не работает при включении службы Private Link в фабрике данных Azure.

## <a name="how-to-set-up-private-link-for-azure-data-factory"></a>Как настроить частную связь для фабрики данных Azure
Частные конечные точки можно создавать с помощью портала Azure, PowerShell или Azure CLI:

[Портал](https://docs.microsoft.com/azure/private-link/create-private-endpoint-portal)


Вы также можете переходить к фабрике данных Azure в портал Azure и создать закрытую конечную точку (PE):

![Создание частной конечной точки](./media/data-factory-private-link/create-private-endpoint.png)


Если вы хотите заблокировать общий доступ к этой фабрике данных Azure и разрешить доступ только через частную связь, вы можете отключить сетевой доступ фабрики данных Azure в портал Azure:

![Создание частной конечной точки](./media/data-factory-private-link/disable-network-access.png)

> [!NOTE]
> Отключение доступа к общедоступной сети применимо только к локальным Integration Runtime, а не Azure Integration Runtime и службам SSIS Integration Runtime.

> [!NOTE]
> Пользователи по-прежнему получают доступ к порталу фабрики данных Azure через общедоступную сеть после отключения доступа к общедоступной сети.

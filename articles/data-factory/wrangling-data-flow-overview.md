# <a name="what-are-wrangling-data-flows"></a>Что такое споры потоков данных?

Организациям необходимо проводить подготовку данных и бороться за точный анализ сложных данных, которые продолжают расти с каждым днем. Требуется подготовка данных, чтобы организации могли использовать данные в различных бизнес-процессах и сокращать время для оценки.

Потоки данных в Azure Data Factory позволяют осуществлять подготовку данных без кода в облачном масштабе. Потоки данных, разрушающиеся, интегрируются с [Power Query Online](https://docs.microsoft.com/power-query/) и делают функции Power Query M доступными для пользователей фабрики данных.

Поток данных wrangling переводит M, генерируемый редактором Power Query Online Mashup, в искровый код для выполнения облачного масштаба.

Потоки данных особенно полезны для инженеров данных или «интеграторов данных граждан».

## <a name="use-cases"></a>Варианты использования

### <a name="fast-interactive-data-exploration-and-preparation"></a>Быстрая интерактивная разведка и подготовка данных

Несколько инженеров по обработке данных и интеграторы данных граждан могут интерактивно исследовать и готовить наборы данных в облачном масштабе. С увеличением объема, разнообразия и скорости данных в озерах данных пользователям необходим эффективный способ изучения и подготовки наборов данных. Например, может потребоваться создать набор данных, который «имеет всю демографическую информацию о клиентах для новых клиентов с 2017 года». Вы не отображение к известной цели. Вы изучаете, спорите и преготовляете наборы данных, чтобы соответствовать требованию, прежде чем опубликовать их в озере. Потоки данных часто используются для менее формальных сценариев аналитики. Подготовленные наборы данных могут быть использованы для выполнения преобразований и операций машинного обучения вниз по течению.

### <a name="code-free-agile-data-preparation"></a>Подготовка гибкой обработки данных без кода

Интеграторы данных граждан тратят более 60% своего времени на поиск и подготовку данных. Они хотят сделать это в коде бесплатно для повышения производительности работы. Разрешение интеграторам данных граждан обогащать, формировать и публиковать данные с помощью известных инструментов, таких как Power Query Online, масштабируемым образом резко повышает их производительность. Поток данных на фабрике данных Azure позволяет знакомому редактору гибридного приложения Power Query Online позволить интеграторам данных граждан быстро устраивать ошибки, стандартизировать данные и создавать высококачественные данные для поддержки бизнес-решений.

### <a name="data-validation"></a>Проверка данных

Визуальное сканирование данных без кода, чтобы удалить любые выбросы, аномалии и соответствовать их форме для быстрой аналитики.

## <a name="the-mashup-editor"></a>Редактор Mashup

При создании потока данных споры все наборы исходных данных становятся запросами набора данных и помещаются в папку **ADFResource.** По умолчанию пользователь-кюкери указывает на первый запрос набора данных. Все преобразования должны быть сделаны на Userquery, так как изменения в запросах набора данных не поддерживаются и не будут упорствовать. Переименование, добавление и удаление запросов в настоящее время не поддерживается.

![Пререканий](media/wrangling-data-flow/editor.png)

В настоящее время не все функции Power Query M поддерживаются для обработки данных, несмотря на то, что они доступны во время авторизации. При построении потоков данных споры, вы будете предложено со следующим сообщением об ошибке, если функция не поддерживается:

`The wrangling data flow is invalid. Expression.Error: The transformation logic isn't supported. Please try a simpler expression`

Для получения дополнительной информации [wrangling data flow functions](wrangling-data-flow-functions.md)об поддерживаемых преобразованиях см.

В настоящее время спор поток данных поддерживает только запись в одну раковину.

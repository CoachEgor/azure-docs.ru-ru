# <a name="sink-transformation-in-mapping-data-flow"></a>Преобразование раковины в потоке данных отображения

После преобразования данных можно погрузить данные в набор данных назначения. Каждый поток данных требует, по крайней мере, одного преобразования раковины, но вы можете написать столько раковин, сколько необходимо для завершения процесса преобразования. Чтобы написать на дополнительные раковины, создавайте новые потоки через новые ветви и условные сплиты.

Каждая трансформация раковины связана с одним набором данных Data Factory. Набор данных определяет форму и местоположение данных, которые вы хотите написать.

## <a name="supported-sink-connectors-in-mapping-data-flow"></a>Поддерживаемые разъемы раковины в потоке данных отображения

В настоящее время при преобразовании раковины можно использовать следующие наборы данных:
    
* [Лазурный Blob хранение](connector-azure-blob-storage.md#mapping-data-flow-properties) (JSON, Avro, Текст, Паркет)
* [Лазурное хранилище озер данных Gen1](connector-azure-data-lake-store.md#mapping-data-flow-properties) (JSON, Avro, Текст, Паркет)
* [Лазурное хранилище данных озера Gen2](connector-azure-data-lake-storage.md#mapping-data-flow-properties) (JSON, Avro, Текст, Паркет)
* [Azure Synapse Analytics](connector-azure-sql-data-warehouse.md#mapping-data-flow-properties)
* [База данных SQL Azure](connector-azure-sql-database.md#mapping-data-flow-properties)
* [Azure КосмосDB](connector-azure-cosmos-db.md#mapping-data-flow-properties)

Настройки, характерные для этих разъемов, расположены во вкладке **"Настройки.Информация** об этих настройках находится в документации разъема". 

Фабрика данных Azure имеет доступ к более чем [90 нативным разъемым.](connector-overview.md) Чтобы написать данные в другие источники из потока данных, используйте действие Copy для загрузки этих данных из одной из поддерживаемых промежуточных областей после завершения потока данных.

## <a name="sink-settings"></a>Настройки раковины

После того как вы добавили раковину, настроить через **вкладку раковина.** Здесь вы можете выбрать или создать набор данных, который пишет раковина 

![Настройки раковины](media/data-flow/sink-settings.png "Настройки раковины")

**Дрифт Schema:** [Schema Drift](concepts-data-flow-schema-drift.md) — это способность фабрики данных самостоятельно обрабатывать гибкие схемы в потоках данных без необходимости четко определять изменения столба. Включить **Разрешить дрифт схемы,** чтобы написать дополнительные столбцы поверх того, что определено в схеме данных раковины.

**Схема проверки:** При выборе схемы проверки поток данных выйдет из строя, если в проекции исходного кода не будет обнаружен столбец входящие схемы исходного кода или если типы данных не совпадают. Используйте эту настройку для обеспечения того, чтобы исходные данные соответствовали контракту определенной проекции. Очень полезно в сценариях исходных данных сигнализировать об изменении имен или типов столбцов.

## <a name="field-mapping"></a>Сопоставление полей

Подобно преобразованию Select, в **вкладке отображения** раковины можно решить, какие входящие столбцы будут написаны. По умолчанию все вхотовые столбцы, включая дрейфуенные столбцы, отображаются. Это называется **автоматическое отображение**.

При выключении автоматического отображения у вас будет возможность добавлять либо фиксированные отображая на основе столбца, либо картографические отображения на основе правил. Карты на основе правил позволяют писать выражения с помощью шаблона, сопрягаемого, в то время как фиксированное отображение будет отображать логические и физические имена столбцов. Для получения дополнительной информации о [column patterns in mapping data flow](concepts-data-flow-column-pattern.md#rule-based-mapping-in-select-and-sink)картографии на основе правил см.

## <a name="custom-sink-ordering"></a>Пользовательский заказ раковины

По умолчанию данные записываются в несколько приемников в недетерминированном порядке. Двигатель выполнения будет записывать данные параллельно по мере завершения логики преобразования и упорядочения раковины может меняться каждый запуск. Чтобы указать и точно указать раковину, включите **пользовательский заказ раковины** в общей вкладке потока данных. При включении раковины будут написаны последовательно в увеличиваемом порядке.

![Пользовательский заказ раковины](media/data-flow/custom-sink-ordering.png "Пользовательский заказ раковины")

## <a name="data-preview-in-sink"></a>Предварительный просмотр данных в раковине

При получении предварительного просмотра данных в кластере отладки данные не будут записаны на раковину. Снимок того, как выглядят данные, будет возвращен, но ничего не будет написано к месту назначения. Чтобы проверить запись данных в раковину, запустите отладку конвейера из холста конвейера.

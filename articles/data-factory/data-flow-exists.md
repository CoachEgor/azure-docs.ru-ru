# <a name="mapping-data-flow-exists-transformation"></a>Сопоставление потока данных существует преобразование

[!INCLUDE [notes](../../includes/data-factory-data-flow-preview.md)]

Существующее преобразование является преобразованием фильтрации строк, которое останавливает или пропускает строки через поток данных. Существующее преобразование похоже на ```SQL WHERE EXISTS``` и ```SQL WHERE NOT EXISTS```. После преобразования существует результирующие строки из потока данных либо включить все строки, где значения столбцов из источника 1 существуют в источнике 2 или не существуют в источнике 2.

![Exists settings](Существующие параметры)(media/data-flow/exists.png "exists 1")

Чтобы выполнить сравнение значений для потока данных между 1 и 2 потоками существующего преобразования, необходимо выбрать второй источник.

Из 1 и 2 источника выберите столбец, значения которого необходимо проверить на существование или отсутствие.

## <a name="multiple-exists-conditions"></a>Несколько существует условия

Рядом с каждой строки в столбец условия для Exists, вы найдете + входа доступны при наведении указателя мыши над достичь строки. Это позволит добавить несколько строк для условий Exists. — Для каждого дополнительного условия «И».

## <a name="custom-expression"></a>Пользовательское выражение

![Существует пользовательские параметры](media/data-flow/exists1.png "существует пользовательский")

Можно щелкнуть «Пользовательское выражение», чтобы вместо этого создать выражение свободной формы как вашей существует или не существует условие. Установка этого флажка позволит для ввода в свое собственное выражение в качестве условия.

# <a name="frequently-asked-questions-about-the-azure-api-for-fhir"></a>Часто задаваемые вопросы об API Azure для FHIR

## <a name="azure-api-for-fhir"></a>Azure API для FHIR

### <a name="what-is-fhir"></a>Что такое FHIR?
Ресурсы по быстрому взаимодействию в сфере здравоохранения (FHIR — «пожар») — это стандарт взаимодействия, предназначенный для обмена данными здравоохранения между различными системами исправности. Этот стандарт был разработан HL7 Организацией и принят организациями здравоохранения по всему миру. Самой последней версией FHIR является R4 (выпуск 4). API Azure для FHIR поддерживает R4, а также предыдущую версию STU3 (Standard для пробного использования 3). Дополнительные сведения о FHIR см. в [HL7.org](http://hl7.org/fhir/summary.html).

### <a name="is-the-data-behind-the-fhir-apis-stored-in-azure"></a>Данные за API FHIR хранятся в Azure?

Да, данные хранятся в управляемых базах данных в Azure. API Azure для FHIR не предоставляет прямой доступ к базовому хранилищу данных.

### <a name="what-identity-provider-do-you-support"></a>Какой поставщик удостоверений поддерживается?

В настоящее время в качестве поставщика удостоверений поддерживается Microsoft Azure Active Directory.

### <a name="what-fhir-version-do-you-support"></a>Какая версия FHIR поддерживается?

Мы поддерживаем версии 4.0.0 и 3.0.1 как в API Azure для FHIR (PaaS), так и на сервере FHIR для Azure (с открытым кодом).

Дополнительные сведения см. в разделе [Поддерживаемые функции](fhir-features-supported.md). Узнайте о том, что изменилось между версиями в [журнале версий для HL7 FHIR](https://hl7.org/fhir/R4/history.html).

### <a name="whats-the-difference-between-the-open-source-microsoft-fhir-server-for-azure-and-the-azure-api-for-fhir"></a>В чем разница между сервером Microsoft FHIR с открытым исходным кодом для Azure и API Azure для FHIR?

API Azure для FHIR — это размещенная и управляемая версия сервера Microsoft FHIR с открытым исходным кодом для Azure. В управляемой службе Корпорация Майкрософт предоставляет все услуги по обслуживанию и обновлению. 

Если вы используете FHIR Server для Azure, у вас есть прямой доступ к базовым службам. Но вы также отвечаете за обслуживание и обновление сервера и все необходимые условия соответствия, если вы храните данные фи.

С точки зрения разработки все компоненты сначала развертываются на сервере Microsoft FHIR с открытым исходным кодом для Azure. После проверки в открытом источнике оно будет выпущено для решения PaaS Azure API for FHIR. Время между выпуском в с открытым исходным кодом и PaaS зависит от сложности функции и других приоритетов плана. 

### <a name="what-is-smart-on-fhir"></a>Что такое SMART on FHIR?

Интеллектуальные (подставляемые медицинские приложения и многократно используемые технологии) на FHIR — это набор открытых спецификаций для интеграции партнерских приложений с серверами FHIR и другими системными ИТ-системами, такими как электронные записи о работоспособности и обмен информацией о работоспособности. Создав ИНТЕЛЛЕКТУАЛЬНое приложение на FHIR, вы сможете обеспечить доступ к приложению и использовать его в различных системах.
Проверка подлинности и API Azure для FHIR. Чтобы получить дополнительные сведения о смарт-разработе, перейдите в [интеллектуальную Подправку](https://smarthealthit.org/).


## <a name="iot-connector-preview"></a>Соединитель IoT (Предварительная версия)

### <a name="what-is-iomt"></a>Что такое Иомт?
Иомт обозначает Интернет из медицинских вещей и это категория устройств Интернета вещей, которые фиксируют и обмениваются данными о работоспособности и здоровье с другими ИТ-системами в сфере ИТ по сети. К некоторым примерам устройств Иомт относится фитнес и клинической практике переносные устройства, датчики мониторинга, средства отслеживания активности, точки зрения работы в киосках или даже интеллектуальные пилюля.

### <a name="how-many-iot-connectors-do-i-need"></a>Сколько нужно соединителей IoT?
Для приема данных из большого числа различных типов устройств можно использовать один соединитель IoT. Вы по-прежнему можете использовать разные соединители по следующим причинам:
- **Масштаб**. для общедоступной предварительной версии емкость ресурсов соединителя IOT является фиксированной и ожидается для обеспечения пропускной способности около 200 сообщений в секунду. Вы можете добавить дополнительные соединители IoT, если требуется более высокая пропускная способность.
- **Тип устройства**. Вы можете настроить отдельный соединитель IOT для каждого типа устройств иомт, которые имеются у вас в целях управления устройствами.

### <a name="is-there-a-limit-on-number-of-iot-connectors-during-public-preview"></a>Существует ли ограничение на количество соединителей IoT во время общедоступной предварительной версии?
Да, вы можете создать только два соединителя IoT на подписку, пока эта функция находится в общедоступной предварительной версии. Это ограничение используется для предотвращения непредвиденных расходов, так как функция доступна бесплатно в течение предварительной версии. По запросу это ограничение может возникать до пяти соединителей Интернета вещей.

### <a name="what-azure-regions-iot-connector-feature-is-available-during-public-preview"></a>Какие регионы Azure доступны через соединитель IoT в общедоступной предварительной версии?
Соединитель IoT доступен во всех регионах Azure, где доступен API Azure для FHIR.

### <a name="can-i-configure-scaling-capacity-for-iot-connector"></a>Можно ли настроить масштабирование для соединителя IoT?
Так как соединитель IoT предоставляется бесплатно во время общедоступной предварительной версии, его масштабирование является фиксированным и ограниченным. Конфигурация соединителя IoT, доступная в общедоступной предварительной версии, должна обеспечивать пропускную способность около 200 сообщений в секунду. Некоторые формы конфигурации емкости ресурсов будут доступны в общедоступной версии.

### <a name="what-fhir-version-does-iot-connector-support"></a>Какая версия FHIR поддерживает соединитель IoT?
Сейчас соединитель IoT поддерживает только FHIR версии R4. Таким образом, эта функция видна только в экземплярах Azure API для FHIR, и корпорация Майкрософт не планирует поддерживать версию STU3 в настоящее время.

### <a name="why-cant-i-install-iot-connector-when-private-link-is-enabled-on-azure-api-for-fhir"></a>Почему не удается установить соединитель IoT, если частная ссылка включена в API Azure для FHIR?
В настоящее время соединитель IoT не поддерживает возможность использования частных ссылок. Таким образом, если в API Azure для FHIR включена частная ссылка, вы не сможете установить соединитель IoT и наоборот. Это ограничение должно исчезнуть, когда соединитель IoT доступен для общедоступной версии.

### <a name="whats-the-difference-between-the-open-source-iomt-fhir-connector-for-azure-and-iot-connector-feature-of-azure-api-for-fhir-service"></a>В чем разница между соединителем Иомт FHIR с открытым исходным кодом для Azure и соединителя IoT в службе API Azure для FHIR?
Соединитель IoT — это размещенная и управляемая версия соединителя FHIR Иомт с открытым исходным кодом для Azure. В управляемой службе Корпорация Майкрософт предоставляет все услуги по обслуживанию и обновлению.

Если вы используете соединитель Иомт FHIR для Azure, у вас есть прямой доступ к базовым ресурсам. Но вы также отвечаете за обслуживание и обновление сервера и все необходимые условия соответствия, если вы храните данные фи.

С точки зрения разработки каждый компонент развертывается в соединителе Иомт FHIR с открытым исходным кодом для Azure. После проверки в открытом источнике он будет включен в функцию "PaaS IoT" соединителя Azure API для службы FHIR. Время между выпуском в с открытым исходным кодом и PaaS зависит от сложности функции и других приоритетов в плане дороги.

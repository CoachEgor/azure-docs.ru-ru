# <a name="prepay-for-azure-cache-for-redis-compute-resources-with-reserved-capacity"></a>Предоплата за кэш Azure для redis вычислить ресурсы с зарезервированной емкостью

Azure Cache для Redis теперь помогает вам сэкономить деньги, предоплату за вычислительные ресурсы по сравнению с ценами с оплатой по мере работы. С зарезервированной емкостью Azure Cacдля Redis вы берете на себя авансовые обязательства по кэшу в течение одного или трех лет, чтобы получить значительную скидку на вычислительные затраты. Чтобы приобрести Кэш Azure для зарезервированной емкости Redis, необходимо указать область Azure, уровень обслуживания и термин.

Вам не нужно присваивать резервирование определенным экземплярам Azure Cache для экземпляров Redis. Уже запущенный кэш Azure для Redis или недавно развернутый автоматически получит выгоду от зарезервированных цен, вплоть до зарезервированного размера кэша. Приобретая резервирование, вы предварительно оплачиваете стоимость вычислений на один или три года. Как только вы покупаете бронирование, кэш Azure для Redis вычисляет сборы, которые соответствуют атрибутам бронирования, больше не взимается по тарифам с оплатой по мере покупки. Бронирование не распространяется на сетевые или хранилища, связанные с кэшом. В конце срока бронирования срок действия платежного пособия истекает, а кэш Azure cache для Redis выставляется по цене оплаты по мере того, как вы получаете плату. Резервирования не возобновляются автоматически. Для получения информации [Azure Cache for Redis reserved capacity offering](https://azure.microsoft.com/pricing/details/cache)о ценах см.

Вы можете купить Azure Cache для зарезервированной емкости Redis на [портале Azure.](https://portal.azure.com/) Чтобы купить зарезервированную емкость:

* Вы должны быть в роли владельца, по крайней мере один предприятие или индивидуальную подписку с оплатой по мере использования ставок.
* Для подписок с соглашением Enterprise параметр **Добавить зарезервированные экземпляры** следует включить на [портале EA](https://ea.azure.com/). Или, если этот параметр отключен, необходимо быть администратором подписки EA.
* В программе поставщика облачных решений (CSP) только агенты-админы или агенты по продажам могут приобрести Кэш Azure для зарезервированной емкости Redis.

Для получения подробной информации о том, как корпоративные клиенты и клиенты Pay-As-You-Go [understand Azure reservation usage for your Pay-As-You-Go subscription](https://docs.microsoft.com/azure/billing/billing-understand-reserved-instance-usage)взимаются за покупки бронирования, [см.](https://docs.microsoft.com/azure/billing/billing-understand-reserved-instance-usage-ea)


## <a name="determine-the-right-cache-size-before-purchase"></a>Определить правильный размер кэша перед покупкой

Размер резервирования должен основываться на общем количестве вычислений, используемых существующим или скоро развертываемым кэшом в определенном регионе и использующего тот же уровень обслуживания.

Например, предположим, что вы работаете с одной общей целью, Gen5 - 32 vCore кэш, и два оптимизированы памяти, Gen5 - 16 vCore кэшов. Кроме того, предположим, что вы планируете развернуть в течение следующего месяца дополнительную общую цель, Gen5 - 32 vCore сервер базы данных, и один оптимизированный, Gen5 - 16 vCore сервер базы данных. Предположим, что вы знаете, что вам понадобятся эти ресурсы, по крайней мере 1 год. В этом случае вы должны приобрести 64 (2x32) vCores, 1 год резервирования для одной базы данных общего назначения - Gen5 и 48 (2x16 16) vCore 1 год резервирования для одной базы данных памяти оптимизированы - Gen5


## <a name="buy-azure-cache-for-redis-reserved-capacity"></a>Купить Кэш Azure для зарезервированной емкости Redis

1. Войдите на [портал Azure](https://portal.azure.com/).
2. Выберите **Все услуги** > **Бронирование**.
3. Выберите **Добавить,** а затем в панели бронирования покупок, выберите **Кэш Azure для Redis,** чтобы приобрести новое бронирование для кэша.
4. Заполните требуемые поля. Существующие или новые базы данных, которые соответствуют выбранным атрибутам, чтобы получить скидку зарезервированной емкости. Фактическое число экземпляров Azure Cache для экземпляров Redis, получивших скидку, зависит от выбранной области и количества.


![Обзор зарезервированных цен](media/cache-reserved-pricing/cache-reserved-price.png)


В следующей таблице описаны необходимые поля.

| Поле | Описание |
| :------------ | :------- |
| Подписка   | Подписка, используемая для оплаты кэша Azure для резервирования резервных емок Redis. С способа оплаты по подписке взимается предоплата за резервирование зарезервированных емок Azure Cac. Тип подписки должен быть корпоративным соглашением (номерами предложений: MS-A'A'-0017P или MS-A'A'-0148P) или индивидуальным соглашением с ценами с оплатой по мере использования (номера предложений: MS-A'R-0003P или MS-A'A'-0023P). Для подписки с соглашением Enterprise плата вычитается из баланса денежных обязательств или относится к избыточным расходам. Для индивидуальной подписки с оплатой по мере использования, сборы выставляются на кредитную карту или метод оплаты счета-фактуры по подписке.
| Область | Область резервирования может охватывать одну или несколько подписок (общая область). Если выбрать: </br></br> **Общая**скидка на бронирование применяется к кэшу Azure Для экземпляров Redis, работающих в любых подписках в контексте выставления счетов. Для клиентов с соглашением Enterprise общая область действует в рамках регистрации и включает в себя все подписки, заданные при регистрации. Для клиентов с подпиской с оплатой по мере использования общая область включает в себя все подписки с оплатой по мере использования, созданные администратором учетной записи.</br></br> **Единая подписка,** скидка на бронирование применяется к экземплярам Azure Cacдля для экземпляров Redis в этой подписке. </br></br> **Единая группа ресурсов,** скидка на резервирование применяется к экземплярам Azure Cacдля для экземпляров Redis в выбранной подписке и выбранной группе ресурсов в рамках этой подписки.
| Регион | Область Azure, охватываемая кэшем Azure для резервирования резервных емок Redis.
| Ценовая категория | Уровень обслуживания для кэша Azure для серверов Redis.
| Термин | Один год или три года
| Количество | Объем вычислительных ресурсов, приобретаемых в кэше Azure для резервирования резервных емок Redis. Количество — это количество кэшов в выбранном регионе Azure и уровень обслуживания, которые зарезервированы и получат скидку на выставление счетов. Например, если вы работаете или планируете запустить кэш Azure для серверов Redis с общей емкостью кэша 26 ГБ в восточном регионе США, то в восточном регионе США будет указано количество в 26, чтобы максимизировать выгоду для всех кэшов.

## <a name="cache-size-flexibility"></a>Гибкость размера кэша

Гибкость размера кэша позволяет масштабировать или вниз в пределах уровня обслуживания и региона, не теряя зарезервированной выгоды от емкости.

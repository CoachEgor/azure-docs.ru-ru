# <a name="manage-access-power-and-connectivity-mode-for-your-azure-data-box-edge"></a>Управление режимом доступа, питания и подключения для Azure Data Box Edge

В этой статье описывается, как управлять режимом доступа, питания и подключения для Azure Data Box Edge. Эти операции выполняются с помощью локального веб-интерфейса или портала Azure.

Вы узнаете, как выполнять следующие задачи:

> [!div class="checklist"]
> * Управление доступом к устройству
> * Управление режимом подключения
> * Управление питанием


## <a name="manage-device-access"></a>Управление доступом к устройству

Доступ к устройству Data Box Edge контролируется с помощью пароля устройства. Вы можете изменить пароль через локальный веб-ui. Вы также можете сбросить пароль устройства на портале Azure.

### <a name="change-device-password"></a>Изменение пароля устройства

Выполните следующие действия в локальном файле обмена, чтобы изменить пароль устройства.

1. В локальном пользовательском веб-интерфейсе перейдите в раздел **Обслуживания > Изменение пароля**.
2. Введите текущий пароль, а затем — новый. Длина пароля должна быть не меньше 8 и не больше 16 знаков. Пароль должен содержать 3 знака из следующих категорий: прописные буквы, строчные буквы, цифры и специальные знаки. Подтвердите новый пароль.

    ![Изменить пароль](media/data-box-edge-manage-access-power-connectivity-mode/change-password-1.png)

3. Щелкните **Изменить пароль**.
 
### <a name="reset-device-password"></a>Пароль устройства сбросить

Сброс рабочего процесса не требует от пользователя ввода старого пароля и полезен при утере пароля. Этот рабочий процесс выполняется на портале Azure.

1. На портале Azure перейдите к **Обзор > Сброс пароля администратора**.

    ![Сброс пароля](media/data-box-edge-manage-access-power-connectivity-mode/reset-password-1.png)


2. Введите новый пароль и подтвердите его. Длина пароля должна быть не меньше 8 и не больше 16 знаков. Пароль должен содержать 3 знака из следующих категорий: прописные буквы, строчные буквы, цифры и специальные знаки. Выберите **Сброс**.

    ![Сброс пароля](media/data-box-edge-manage-access-power-connectivity-mode/reset-password-2.png)

## <a name="manage-resource-access"></a>Управление доступом к ресурсам

Для создания шлюза Box/Data Box, IoT Hub и ресурса хранения Azure необходимы разрешения в качестве участника или выше на уровне группы ресурсов. Для регистрации также необходимо зарегистрировать соответствующих поставщиков ресурсов. Для любых операций, связанных с ключом активации и учетных данных, также требуются разрешения на API Microsoft Graph. Они описаны в следующих разделах.

### <a name="manage-microsoft-graph-api-permissions"></a>Управление разрешениями Microsoft Graph API

При генерации ключа активации для устройства Data Box Edge или выполнения любых операций, требующих учетных данных, требуется разрешение на API Microsoft Graph. Операции, которым нужны учетные данные, могут быть:

-  Создание доли с связанной учетной записью хранения.
-  Создание пользователя, который может получить доступ к акциям на устройстве.

Вы должны `User` иметь доступ на Active Directory арендатора, как вы должны быть в `Read all directory objects`состоянии . Вы не можете быть гостем пользователя, поскольку `Read all directory objects`они не имеют разрешений на . Если вы гость, то такие операции, как генерация ключа активации, создание доли на устройстве Data Box Edge, создание пользователя, конфигурация роли вычислений Edge, пароль сбросить устройство, все это сбой.

Для получения дополнительной [информации](https://docs.microsoft.com/graph/permissions-reference)о том, как обеспечить доступ к пользователям Microsoft Graph API, см.

### <a name="register-resource-providers"></a>Регистрация поставщиков ресурсов

Для создания ресурса в Azure (в модели диспетчера ресурсов Azure) нужен поставщик ресурсов, который поддерживает создание этого ресурса. Например, для предоставления виртуальной машины в подписке должен быть поставщик ресурсов «Microsoft.Compute».
 
Поставщики ресурсов регистрируются на уровне подписки. По умолчанию любая новая подписка Azure предварительно регистрируется со списком часто используемых поставщиков ресурсов. Поставщик ресурсов для 'Microsoft.DataBoxEdge' не включен в этот список.

Вам не нужно предоставлять разрешения на доступ к уровню подписки для пользователей, чтобы иметь возможность создавать ресурсы, такие как 'Microsoft.DataBoxEdge' в своих ресурсных группах, на которые у них есть права владельца, до тех пор, пока поставщики ресурсов для этих ресурсов уже Зарегистрированных.

Прежде чем пытаться создать какой-либо ресурс, убедитесь, что поставщик ресурсов зарегистрирован в подписке. Если поставщик ресурсов не зарегистрирован, необходимо убедиться, что пользователь, создающий новый ресурс, имеет достаточно прав для регистрации необходимого поставщика ресурсов на уровне подписки. Если вы еще не сделали этого, а затем вы увидите следующую ошибку:

*Имя \<подписки> не имеет разрешений на регистрацию поставщика ресурсов (ы): Microsoft.DataBoxEdge.*


Чтобы получить список зарегистрированных поставщиков ресурсов в текущей подписке, запустите следующую команду:

```PowerShell
Get-AzResourceProvider -ListAvailable |where {$_.Registrationstate -eq "Registered"}
```

Для устройства Data `Microsoft.DataBoxEdge` Box Edge, должны быть зарегистрированы. Для `Microsoft.DataBoxEdge`регистрации, абонент подписки должен запустить следующую команду:

```PowerShell
Register-AzResourceProvider -ProviderNamespace Microsoft.DataBoxEdge
```

Для получения дополнительной информации о том, как зарегистрировать поставщика ресурсов, [см.](https://docs.microsoft.com/azure/azure-resource-manager/resource-manager-register-provider-errors)

## <a name="manage-connectivity-mode"></a>Управление режимом подключения

Помимо полностью подключенного к режиму по умолчанию, устройство также может работать в частично подключенном или полностью отключенном режиме. Ниже описан каждый из этих режимов:

- **Полностью подключен** - это нормальный режим по умолчанию, в котором устройство работает. В этом режиме включена загрузка и загрузка данных как в облаке. Для управления устройством можно использовать портал Azure или локальный веб-файл.

- **Частично отключен** - В этом режиме устройство не может загрузить или загрузить какие-либо данные общего обмена, однако можно управлять через портал Azure.

    Этот режим обычно используется в измеренной сети спутников. Целью является минимизация потребления пропускной способности сети. Минимальное потребление сети может по-прежнему требоваться для операций мониторинга устройств.

- **Отключен**. В этом режиме устройство полностью отключено от облака, а облачные отправки и скачивания отключены. Управлять устройством можно только через локальный пользовательский веб-интерфейс.

    Этот режим обычно используется, если вы хотите перевести устройство в автономный режим.

Чтобы изменить режим устройства, выполните следующие действия:

1. В локальном пользовательском веб-интерфейсе устройства перейдите к **Конфигурация > Параметры облака**.
2. Из списка выпадающих данных выберите режим, в который вы хотите управлять устройством. Вы можете выбрать из **полностью подключенных,** **частично подключенных**и **полностью отключенных.** Чтобы запустить устройство в частично автономном режиме, включите **управление порталом Azure**.

    ![Режим подключения](media/data-box-edge-manage-access-power-connectivity-mode/connectivity-mode.png)
 
## <a name="manage-power"></a>Управление питанием

Вы можете отключить или перезапустить физическое устройство с помощью локального веб-мина. Мы рекомендуем перед перезапуском перезапустить акции в автономном режиме на сервере данных, а затем на устройстве. Это действие уменьшает возможность повреждения данных.

1. В локальном пользовательском веб-интерфейсе последовательно выберите **Обслуживание > Настройки питания**.
2. Выберите **Shutdown** или **перезапустить** в зависимости от того, что вы собираетесь делать.

    ![Параметры питания](media/data-box-edge-manage-access-power-connectivity-mode/shut-down-restart-1.png)

3. Когда попросят подтверждение, выберите **Да,** чтобы продолжить.

> [!NOTE]
> Если вы выключите физическое устройство, вам нужно будет нажать кнопку питания на устройстве, чтобы включить его.

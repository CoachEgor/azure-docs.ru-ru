# <a name="deployment-checklist-for-your-azure-stack-edge-device"></a>Контрольный список развертывания для устройства Azure Stack пограничных устройств  

В этой статье описываются сведения, которые можно собрать перед фактическим развертыванием устройства Azure Stack пограничной Организации. 

Используйте следующий контрольный список, чтобы убедиться, что эти сведения имеются после размещения заказа для Azure Stack пограничной устройства и до получения устройства. 

## <a name="deployment-checklist"></a>Контрольный список для развертывания 

| Этап                             | Параметр                                                                                                                                                                                                                           | Сведения                                                                                                           |
|-----------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|-------------------------------------------------------------------------------------------------------------------|
| Управление устройствами               | <li>Подписка Microsoft Azure</li><li>API Graph Azure Active Directory</li><li>Учетная запись служба хранилища Microsoft Azure</li>|<li>Включено для Azure Stack граничных и Шлюз Data Box, разрешения участника</li><li>Обеспечение доступа администратора или пользователя</li><li>Требуются учетные данные для доступа</li> |
| Установка устройств               | Кабели питания в пакете. <br>Для нас SVE 18/3ный кабель с 125 V и 15 ампер с Нема 5-15P на C13 (вход в выход).                                                                                                                                                                                                          | Предоставляется вместе с устройством.<br>Дополнительные сведения см. в списке [поддерживаемых кабелей питания по странам](azure-stack-edge-technical-specifications-power-cords-regional.md) .                                                                                        |
|                                   | <li>По крайней мере один сетевой кабель RJ-45 с частотой 1 X 1-GbE для порта 1  </li><li> По крайней мере 1 X 25-GbE SFP + медный кабель для порта 3, порт 4, порт 5 или порт 6</li>| Клиенту необходимо приобрести эти кабели.<br>Полный список поддерживаемых сетевых кабелей, коммутаторов и приемопередатчиков для сетевых карт устройств см. в статье [матрица взаимодействия Кавиум фастлинк 41000](https://www.marvell.com/documents/xalflardzafh32cfvi0z/) и [два порта Mellanox, совместимые с сетевыми адаптерами](https://docs.mellanox.com/display/ConnectX4LxFirmwarev14271016/Firmware+Compatible+Products).| 
| Подключение устройства в первый раз      | Порт 1 имеет фиксированный IP-адрес (192.168.100.10/24) для первоначального подключения. <li>Требуется портативный компьютер для прямого подключения к порту 1 с IP-адресом в сети 192.168.100.0/24.</li><li> Используйте локальный пользовательский интерфейс устройства в: `https://192.168.100.10` для дальнейшей настройки.</li><li> После завершения начальной настройки необходимо использовать как минимум 1 коммутатора GbE для устройства. Локальный веб-интерфейс будет недоступен, если подключенный коммутатор не по крайней мере 1 GbE.</li>|                                                                                                                   |
| Вход в устройство                      | Пароль администратора устройства должен содержать от 8 до 16 символов. <br>Пароль должен содержать три знака из следующих категорий: прописные буквы, строчные буквы, цифры и специальные знаки.                                            | Пароль по умолчанию — *password1* , срок действия которого истекает при первом входе.                                                     |
| Параметры сети                  | Устройство поставляется с сетевыми портами 2 x 1 GbE, 4 x 25 GbE. <li>Порт 1 используется только для настройки параметров управления. Можно подключить и настроить один или несколько портов данных. </li><li> По крайней мере один сетевой интерфейс данных из порта 2 — порт 6 должен быть подключен к Интернету (с подключением к Azure).</li><li> Параметры IP-адресов поддерживают DHCP/статическую конфигурацию IPv4. | Для статической конфигурации IPv4 требуется IP-адрес, DNS-сервер и шлюз по умолчанию.                                                                                                                  |
| Параметры вычислений сети     | <li>Требовать 2 статических общедоступных IP-адресов для узлов Kubernetes и по крайней мере один статический IP для службы концентратора Azure Stack для доступа к модулям вычислений.</li><li>Требовать один IP-адрес для каждой дополнительной службы или контейнера, к которым требуется доступ извне извне кластера Kubernetes.</li>                                                                                                                       | Поддерживается только Статическая конфигурация IPv4.                                                                      |
| Используемых Параметры веб-прокси     | <li>IP-адрес веб-прокси/полное доменное имя, порт </li><li>Имя пользователя веб-прокси, пароль</li>                                                                                                                                                                                                    | В настоящее время не поддерживается при установке вычислений.                                                                     |
| Параметры брандмауэра и порта        | Используйте [указанные шаблоны URL-адресов и порты](azure-stack-edge-system-requirements.md#networking-port-requirements) , чтобы они были разрешены для адресов устройств.                                                                                                                                                  |                                                                                                                   |
| Используемых MAC-адрес            | Если MAC-адрес должен быть список разрешений, получите адрес подключенного порта из локального пользовательского интерфейса устройства. |                                                                                                                   |
| Используемых Порт сетевого коммутатора    | На устройстве размещаются виртуальные машины Hyper-V для вычислений. Некоторые конфигурации портов сетевого коммутатора не поддерживают эти настройки по умолчанию.                                                                                                        |                                                                                                                   |
| Такую Параметры времени       | Настройте часовой пояс, основной NTP-сервер, дополнительный NTP-сервер.                                                                                                                                                                    | Настройте основной и дополнительный NTP-серверы в локальной сети.<br>Если локальный сервер недоступен, можно настроить общедоступные NTP-серверы.                                                    |
| Используемых Обновить параметры сервера | <li>Требуется IP-адрес сервера обновлений в локальной сети, путь к серверу WSUS. </li> | По умолчанию используется общедоступный сервер центра обновления Windows.|
| Параметры устройства                   | <li>Имя устройства зарегистрировано в Организации DNS </li><li>Домен DNS</li> |                                                                                                                   |
| Используемых Сертификаты                      | Использовать сертификаты, созданные устройством <br><br> При переводе собственных сертификатов вам потребуется: <li>Доверенный корневой сертификат для подписи в формате DER с *. cer* </li><li>Сертификаты конечной точки в формате *PFX*</li>|Сертификаты конечных точек включают в себя Azure Resource Manager, хранилище BLOB-объектов, локальный Web UI.                                                                                                                   |
| Активация                        | Требовать ключ активации из ресурса Azure Stack ребра/Шлюз Data Box.                                                                                                                                                       | После создания ключ истекает через 3 дня.                                                                        |

# <a name="what-is-azure-data-box-edge"></a>Общие сведения об Azure Data Box Edge 

Azure Data Box Edge — это решение хранилища, с помощью которого можно обрабатывать данные и отправлять их по сети в Azure. Эта статья содержит общие сведения о решении Data Box Edge, его преимуществах, основных функциях и сценариях, в которых можно развернуть это устройство. 

Data Box Edge использует физическое устройство, предоставляемое корпорацией Майкрософт для ускорения безопасной передачи данных. Физическое устройство находится в вашей локальной среде, и вы можете записывать данные на него с использованием протоколов NFS и SMB. 

В решении Data Box Edge есть все возможности шлюза, присущие службе "Шлюз Data Box". Кроме того, решение Data Box оснащено вычислительными возможностями Edge с искусственным интеллектом, позволяющими анализировать, обрабатывать или фильтровать данные по мере их поступления в блочный или страничный BLOB-объект Azure либо в службу файлов Azure.  

## <a name="use-cases"></a>Варианты использования

Azure Data Box Edge — это вычислительное устройство Edge с искусственным интеллектом и функциями передачи данных по сети. Ниже перечислены примеры сценариев, в которых можно использовать решение Data Box Edge для передачи данных.

- **Предварительная обработка данных**. Анализ данных, поступающих из локальной среды или устройств категории "Интернет вещей". Для быстрого получения результатов устройство размещают вблизи источника данных. Решение Data Box Edge переносит полный набор данных в облако и выполняет расширенную обработку данных или их углубленный анализ.  Предварительную обработку данных можно использовать для указанных ниже целей. 

    - Агрегирование данных.
    - Изменение данных (например, для удаления личных сведений).
    - Разбиение данных, требующих углубленного анализа, на подмножества и передача их в облако.
    - Анализ и реагирование на события Интернета вещей. 

- **Машинное обучение Azure на основе выводов**. С помощью Data Box Edge вы можете запускать модели машинного обучения для быстрого получения результатов, на основании которых можно предпринимать действия, не дожидаясь отправки данных в облако. Система может передать полный набор данных для продолжения обучения и совершенствования ваших моделей машинного обучения. Дополнительные сведения о том, как использовать модели с аппаратным ускорением Azure ML на устройстве Databox Edge, см. в разделе [Развертывание моделей с аппаратным ускорением Azure ML в Databox Edge](https://docs.microsoft.com/azure/machine-learning/service/how-to-deploy-fpga-web-service#deploy-to-a-local-edge-server).

- **Передача данных по сети в Azure**. С помощью Data Box Edge вы можете быстро и просто передавать данные в Azure для последующих вычислений и анализа либо для их архивации. 

## <a name="benefits"></a>Преимущества

Решение Data Box Edge имеет указанные ниже преимущества.

- **Простая передача данных**. Перемещать данные в службу хранилища Azure и из нее так же просто, как и при работе с локальной сетевой папкой.  
- **Высокая производительность**. Высокоскоростная передача данных в Azure и обратно. 
- **Быстрый доступ**. Система кэширует самые последние файлы для быстрого доступа к файлам в локальной среде.  
- **Работа при ограниченной пропускной способности**. Вы можете записывать данные в Azure даже в сети, в которой действуют ограничения пропускной способности во время рабочих часов с наибольшей нагрузкой.  
- **Преобразование данных**. Анализ, обработка или фильтрация данных при их перемещении в Azure.

## <a name="components"></a>Компоненты

Решение Data Box Edge включает ресурс Data Box Edge, физическое устройство Data Box Edge и локальный пользовательский веб-интерфейс.

* **Физическое устройство Data Box Edge**. Сервер размера 1U для установки в стойке, поставляемый корпорацией Майкрософт, который можно настроить для передачи данных в Azure. 
    
* **Ресурс Data Box Edge**. Ресурс на портале Azure, с помощью которого можно управлять устройством Data Box Edge в веб-интерфейсе. Веб-интерфейс доступен из различных географических расположений. С помощью ресурса Data Box Edge вы можете создавать ресурсы и управлять ими, просматривать устройства и оповещения и управлять ими, а также управлять общими папками.  

    <!--![The Data Box Edge service in Azure portal](media/data-box-overview/data-box-Edge-service1.png)-->

    Дополнительные сведения см. в статье о [создании заказа для устройства Data Box Edge](data-box-edge-deploy-prep.md#create-a-new-resource).

* **Локальный пользовательский веб-интерфейс Data Box**. С его помощью можно выполнять диагностику, завершать работу устройства Data Box Edge и перезапускать его, просматривать журналы копирования и обращаться в службу поддержки Майкрософт для создания запросов на обслуживание.

    <!--![The Data Box Edge local web UI](media/data-box-Edge-overview/data-box-Edge-local-web-ui.png)-->

    См. дополнительные сведения о том, как [администрировать Data Box с помощью пользовательского веб-интерфейса](data-box-edge-manage-access-power-connectivity-mode.md).


## <a name="region-availability"></a>Доступность по регионам

Физическое устройство Data Box Edge, ресурс Azure и целевая учетная запись хранения, в которую вы передаете данные, необязательно должны находиться в одном и том же регионе.

- **Доступность ресурсов**. Для этого выпуска ресурс Data Box Edge доступен в указанных ниже регионах.
    - **США**: восточная часть США.
    - **Европейский союз**: Западная Европа.
    - **Азиатско-Тихоокеанский регион**: Юго-Восточная Азия.
    
    Data Box Edge также можно развернуть в облаке Azure для государственных организаций. Дополнительные сведения см. в обзорной статье об [Azure для государственных организаций](https://docs.microsoft.com/azure/azure-government/documentation-government-welcome).
    
- **Целевые учетные записи хранения**. Учетные записи хранения, в которых хранятся данные, доступны во всех регионах Azure. 

    Регионы, в которых учетные записи хранения сохраняют данные Data Box, должны размещаться рядом с местом расположения устройства, чтобы достичь оптимальной производительности. Если учетная запись хранения расположена далеко от устройства, возникают длительные задержки и снижение производительности.

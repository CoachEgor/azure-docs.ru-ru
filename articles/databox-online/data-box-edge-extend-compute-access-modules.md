# <a name="enable-compute-network-on-your-azure-data-box-edge"></a>Включить вычислительную сеть на вашем Edge Box данных Azure

В этой статье описывается, как модули, работающие на Azure Data Box Edge, могут получить доступ к вычислительной сети, включенной на устройстве.

Для настройки сети вы предпримете следующие шаги:

- Включить сетевой интерфейс на устройстве Data Box Edge для вычислений
- Добавление модуля для доступа к вычислительной сети на вашем краю коробки данных
- Проверка модуля может получить доступ к включенному сетевому интерфейсу

В этом уроке вы будете использовать модуль приложения веб-сервера, чтобы продемонстрировать сценарий.

## <a name="prerequisites"></a>Предварительные требования

Прежде чем начать, вам нужно:

- Завершено устройство Data Box Edge с настройкой устройства.
- Вы завершили этап **вычислений настройки** в соответствии [с учебным методом: преобразование данных с Azure Data Box Edge](data-box-edge-deploy-configure-compute-advanced.md#configure-compute) на вашем устройстве. Устройство должно иметь связанный с ним ресурс IoT Hub, ioT-устройство и устройство IoT Edge.

## <a name="enable-network-interface-for-compute"></a>Включить сетевой интерфейс для вычислений

Чтобы получить доступ к модулям, работающим на устройстве через внешнюю сеть, необходимо назначить IP-адрес сетевому интерфейсу на устройстве. Вы можете управлять этими параметрами вычислений из локального пользовательского веб-интерфейса.

Выполните следующие действия в локальном веб-интерфейсе, чтобы настроить параметры вычислений.

1. Выберите **Конфигурация > Параметры вычисления**.  

2. **Включите** сетевой интерфейс, который требуется использовать для подключения к вычислительному модулю, который вы будете работать на устройстве.

    - При использовании статических IP-адресов укажите IP-адрес сетевого интерфейса.
    - При использовании DHCP автоматически назначаются IP-адреса. В этом примере используется DHCP.

    ![Включить настройки вычислений 1](media/data-box-edge-extend-compute-access-modules/enable-compute-setting-1.png)

3. Выберите **Применить** для применения параметров. Обратите внимание на IP-адрес, присвоенный сетевому интерфейсу при использовании DHCP.

    ![Включение параметров вычислений](media/data-box-edge-extend-compute-access-modules/enable-compute-setting-2.png)

## <a name="add-webserver-app-module"></a>Добавление модуля приложения веб-сервера

Предпримите следующие шаги, чтобы добавить модуль приложения webserver на устройстве Data Box Edge.

1. Перейдите на ресурс IoT Hub, связанный с устройством Data Box Edge, а затем выберите **устройство IoT Edge.**
2. Выберите устройство IoT Edge, связанное с устройством Data Box Edge. На **деталях устройства,** выберите **набор модулей.** На **добавлении модулей**, выберите **и добавить,** а затем выбрать **IoT Edge модуль**.
3. В **лезвии пользовательских модулей IoT Edge:**

    1. Укажите **имя** для модуля приложения веб-сервера, который необходимо развернуть.
    2. Предоставьте **изображение URI** для изображения модуля. Извлекается модуль, соответствующий при условии имени и тегов. В этом `nginx:stable` случае, будет тянуть стабильное изображение nginx (помечены как стабильные) из публичного [репозитория Docker](https://hub.docker.com/_/nginx/).
    3. В **параметры создания контейнера**вставьте следующий пример кода:  

        ```
        {
            "HostConfig": {
                "PortBindings": {
                    "80/tcp": [
                        {
                            "HostPort": "8080"
                        }
                    ]
                }
            }
        }
        ```

        Эта конфигурация позволяет получить доступ к модулю с помощью IP-адреса вычислительной сети *над http* на порте TCP 8080 (с портом веб-сервера по умолчанию 80).

        ![Указать информацию о порневе в лопасти пользовательского модуля IoT Edge](media/data-box-edge-extend-compute-access-modules/module-information.png)

    4. Нажмите кнопку **Сохранить**.

## <a name="verify-module-access"></a>Проверка доступа к модулю

1. Проверка модуля успешно развернута и запущена. На странице **сведения об устройстве** на вкладке **Модули** должен быть **запущен**статус времени выполнения модуля.  
2. Подключитесь к модулю приложения для веб-сервера. Откройте окно браузера и введите:

    `http://<compute-network-IP-address>:8080`

    Вы должны видеть, что приложение веб-сервера работает.

    ![Проверка подключения к модулю над указанным портом](media/data-box-edge-extend-compute-access-modules/verify-connect-module-1.png)

# <a name="azure-data-box-edgeazure-data-box-gateway-general-availability-release-notes"></a>Заметки об общем выпуске системы доступа к базе данных Azure Data Box/Azure Data Box

## <a name="overview"></a>Обзор

Следующие примечания к выпуску определяют критические проблемы, открытые и решенные проблемы для выпуска общей доступности (GA) для Azure Data Box Edge и Azure Data Box Gateway.

Заметки о выпуске постоянно обновляются: обнаруживаются и добавляются критические проблемы, требующие временного решения. Перед развертыванием шлюза Data Box Edge/Data Box, внимательно просмотрите информацию, содержащуюся в примечаниях к выпуску.

Релиз GA соответствует версиям программного обеспечения:

- **Шлюз box 1903 (1.5.814.447)**
- **Дата Box Edge 1903 (1.5.814.447)**


## <a name="whats-new"></a>Новые возможности

- **Новые изображения виртуальных дисков** - Новые VHDX и VMDK теперь доступны на портале Azure. Загрузите эти изображения для предоставления, настройки и развертывания новых устройств Data Box Gateway GA. Устройства шлюза Data Box, созданные в более ранних предварительных выпусках, не удается обновить до этой версии. Дополнительные сведения см. в разделе [Подготовка к развертыванию шлюза Azure Data Box](data-box-gateway-deploy-prep.md).
- **Поддержка NFS** - Поддержка NFS в настоящее время находится в предварительном просмотре и доступна для клиентов v3.0 и v4.1, которые получают доступ к устройствам Data Box Edge и Data Box Gateway.
- **Устойчивость к хранению** - Устройство Data Box Edge может выдержать сбой одного диска данных с функцией устойчивости хранилища. Эта функция в настоящее время находится на стадии предварительной версии. Можно включить гибкость хранилища, выбрав **опцию Resilient** в **настройках хранения** в локальном веб-миге.


## <a name="known-issues-in-ga-release"></a>Известные проблемы в релизе GA

В следующей таблице приводится краткое изложение известных проблем для запуска релиза Data Box Gateway.

| Нет. | Компонент | Проблема | Временное решение и комментарии |
| --- | --- | --- | --- |
| **1.** |Типы файлов | Не поддерживаются следующие типы файлов: файлы символов, файлы блок-файлов, розетки, трубы, символические ссылки.  |Копирование этих файлов приведет к созданию пустых файлов в общей папке NFS. Эти файлы находятся в состоянии ошибки, а данные о них передаются в файл *error.xml*. <br> Символические ссылки на каталоги приводят к тому, что каталоги никогда не помечаются в автономном режиме. И как результат — вы можете не увидеть серый крест на каталогах, который указывает на то, что каталоги отключены и все связанное содержимое полностью отправлено в Azure. |
| **2.** |Удаление | Из-за ошибки в этом выпуске при удалении общего ресурса NFS, он может быть не удален. Состояние общего ресурса отобразится как *Удаление*.  |Это происходит только в том случае, если общий ресурс использует имя файла, которое не поддерживается. |
| **3.** |Копировать | Копия данных не справляется с ошибкой: Запрошенная операция не может быть завершена из-за ограничения файловой системы.  |Альтернативный поток данных (ADS), связанный с размером файла более 128 кБ, не поддерживается.   |

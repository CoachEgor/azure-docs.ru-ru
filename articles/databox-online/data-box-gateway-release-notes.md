# <a name="azure-data-box-edgeazure-data-box-gateway-general-availability-release-notes"></a>Заметки о выпуске шлюза данных поле Edge или Azure данных поле общедоступной версии Azure

## <a name="overview"></a>Обзор

Следующие заметки о выпуске определите критические открытые проблемы и устраненные проблемы Общая доступность (GA) выпуск для Edge поле данных Azure и Azure Data Box шлюза.

Заметки о выпуске постоянно обновляются: обнаруживаются и добавляются критические проблемы, требующие временного решения. Перед развертыванием ваш Data Gateway поле Edge/Data Box, внимательно просмотрите сведения, содержащиеся в заметках о выпуске.

Выпуск общедоступной версии соответствует версии программного обеспечения:

- **Шлюз поля данных 1903 (1.5.814.447)**
- **Край поля данных 1903 (1.5.814.447)**


## <a name="whats-new"></a>Новые возможности

- **Новые образы виртуальных дисков** - и VMDK новые VHDX-файлы теперь доступны на портале Azure. Скачайте эти изображения для подготовки, Настройка и развертывание новых устройств шлюза поля данных общедоступной версии. Устройства шлюза Data Box, созданные в более ранних предварительных выпусках, не удается обновить до этой версии. Дополнительные сведения см. в разделе [Подготовка к развертыванию шлюза Azure Data Box](data-box-gateway-deploy-prep.md).
- **Поддерживает NFS** -поддержку NFS сейчас находится на этапе предварительной версии и доступна для версии 3.0 и версии 4.1 клиентов, обращающихся к устройств Edge поле данных и шлюзу поле данных.
- **Устойчивость хранения** -устройство Your Edge поле данных могла выдерживать сбой диска данных с помощью функции исправления хранилища. Эта функция в настоящее время находится на стадии предварительной версии. Вы можете включить устойчивость хранилища, выбрав **отказоустойчивое** в диалоговом окне **параметры хранилища** в локальном веб-интерфейса.


## <a name="known-issues-in-ga-release"></a>Известные проблемы в выпуск общедоступной версии

Следующая таблица предоставляет список известных проблем для шлюза поле данных под управлением выпуска.

| № | Компонент | Проблема | Временное решение и комментарии |
| --- | --- | --- | --- |
| **1.** |Типы файлов | Не поддерживаются следующие типы файлов: символ файлы, блокировать файлы, сокеты, каналы, символические ссылки.  |Копирование этих файлов приведет к созданию пустых файлов в общей папке NFS. Эти файлы находятся в состоянии ошибки, а данные о них передаются в файл *error.xml*. <br> Символические ссылки на каталоги приводят к тому, что каталоги никогда не помечаются в автономном режиме. И как результат — вы можете не увидеть серый крест на каталогах, который указывает на то, что каталоги отключены и все связанное содержимое полностью отправлено в Azure. |
| **2.** |Удаление | Из-за ошибки в этом выпуске при удалении общего ресурса NFS, он может быть не удален. Состояние общего ресурса отобразится как *Удаление*.  |Это происходит только в том случае, если общий ресурс использует имя файла, которое не поддерживается. |
| **3.** |Копировать | Копирование данных завершается ошибкой.  Запрошенная операция не может быть завершена из-за ограничения файловой системы.  |Альтернативные данных Stream (ADS) связанный файл размером более 128 КБ, не поддерживается.   |

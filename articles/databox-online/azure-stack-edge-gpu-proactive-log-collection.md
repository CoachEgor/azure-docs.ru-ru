# <a name="proactive-log-collection-on-your-azure-stack-edge-device"></a>Упреждающий сбор журналов на Azure Stack пограничном устройстве

Вы можете включить упреждающее сбор журналов на Azure Stack пограничном устройстве на основе индикаторов работоспособности системы, чтобы помочь эффективно устранять проблемы с устройствами. В этой статье описывается, что такое упреждающий сбор журналов, как включить его и как обрабатываются данные при включении упреждающего сбора журналов.
   
Сведения, приведенные в этой статье, относятся к Azure Stackным графическим процессорам Pro, Azure Stackм пограничных Pro R и Azure Stack пограничных устройств R.

## <a name="about-proactive-log-collection"></a>Сведения о упреждающих сборах журналов

Группы поддержки пользователей Майкрософт и инженеров по разработке используют системные журналы из Azure Stack пограничных устройств для эффективного обнаружения и устранения проблем, которые могут возникнуть во время работы. Упреждающий сбор журналов — это метод, который предупреждает корпорации Майкрософт о том, что для отслеживаемых событий Azure Stack обнаружена ошибка или событие (см. раздел "Индикаторы сбора сведений об активном сборе журналов"). Журналы поддержки, относящиеся к этой проблемы, автоматически передаются в учетную запись хранения Azure, управляемую и управляемую корпорацией Майкрософт. Служба поддержки Майкрософт и инженеры Майкрософт просматривают эти журналы поддержки, чтобы определить оптимальный способ решения проблемы с клиентом.    

> [!NOTE]
> Эти журналы используются только в целях отладки и предоставляют пользователям поддержку в случае возникновения проблем. 


## <a name="enabling-proactive-log-collection"></a>Включение упреждающего сбора журналов

Вы можете включить сбор упреждающего журнала при попытке активировать устройство с помощью локального пользовательского интерфейса. 

1. В локальном веб-интерфейсе устройства перейдите на страницу **Начало работы**.
2. На плитке **Активация** выберите **Активировать**. 

    ![Локальный веб-интерфейс "сведения о облаке", страница 1](./media/azure-stack-edge-pro-r-deploy-activate/activate-1.png)
    
3. На панели **Активация** :
    1. Введите **ключ активации** , полученный в разделе [Получение ключа активации для Azure Stack ребра Pro R](azure-stack-edge-pro-r-deploy-prep.md#get-the-activation-key).

    1. Можно включить упреждающий сбор журналов, чтобы разрешить Майкрософт собирать журналы в зависимости от состояния работоспособности устройства. Журналы, собранные таким образом, передаются в учетную запись хранения Azure.
    
    1. Нажмите кнопку **Применить**.

    ![Страница "Сведения об облаке" в локальном пользовательском веб-интерфейсе 2](./media/azure-stack-edge-pro-r-deploy-activate/activate-2.png)



## <a name="proactive-log-collection-indicators"></a>Упреждающие индикаторы сбора журналов

После включения упреждающего сбора журналов журналы автоматически передаются при обнаружении одного из следующих событий на устройстве.  


|Предупреждение/ошибка/условие  |Описание  |
|---------|---------|
|аксунхеалсикондитион     |Неработоспособные службы Azure.         |
|иотеджеажентнотруннингкондитион      |Агент IoT Edge не работает.         |
|упдатеинсталлфаиледевент | Не удалось установить обновление.        |

 
Корпорация Майкрософт продолжит добавлять новые события в приведенный выше список. Для новых событий не требуется дополнительное согласие. Сведения о наиболее актуальных индикаторах сбора журналов см. на этой странице.    
 

## <a name="other-log-collection-methods"></a>Другие методы сбора журналов

Помимо упреждающего сбора журналов, который собирает конкретные журналы, относящиеся к определенной обнаруженной ошибке, существуют другие коллекции журналов, которые позволяют более глубоко понять работоспособность и поведение системы. Обычно эти другие коллекции журналов могут быть выполнены во время запроса на поддержку или запускаются корпорацией Майкрософт на основе данных телеметрии, предоставляемых устройством.  

## <a name="handling-data"></a>Обработка данных

Если клиент включает упреждающее сбор журналов, он соглашается корпорации Майкрософт собирать журналы с Azure Stack пограничных устройств, как описано здесь. Клиент также подтверждает и отправляет данные о передаче и хранении этих журналов в учетной записи хранения Azure, управляемой и управляемой корпорацией Майкрософт.

Корпорация Майкрософт использует данные для устранения неполадок с работоспособностью системы и проблемами. Данные не используются для маркетинга, рекламы или других коммерческих целей без согласия клиента. 

Данные, собираемые корпорацией Майкрософт, обрабатываются согласно стандартным рекомендациям по обеспечению конфиденциальности. Если клиент решил отозвать свое согласие на упреждающий сбор журналов, все данные, собранные с согласия до отзыва, не будут затронуты.

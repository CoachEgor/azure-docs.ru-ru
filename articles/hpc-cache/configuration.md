# <a name="configure-additional-azure-hpc-cache-settings"></a>Настройка дополнительных настроек кэша HPC Azure

Страница **Конфигурации** на портале Azure имеет варианты настройки нескольких настроек. Большинству пользователей не нужно изменять их из значений по умолчанию.

В этой статье также описывается, как использовать функцию моментального снимка для целей хранения Azure Blob. Функция моментального снимка не имеет настраиваемых настроек.

Чтобы увидеть настройки, откройте страницу **конфигурации** кэша на портале Azure.

![скриншот страницы конфигурации на портале Azure](media/configuration.png)

## <a name="adjust-mtu-value"></a>Отрегулируйте значение MTU
<!-- linked from troubleshoot-nas article -->

Вы можете выбрать максимальный размер единицы передачи для кэша, используя выпадающее меню с **пометкой размер MTU.**

Значение по умолчанию составляет 1500 байтов, но его можно изменить на 1400.

> [!NOTE]
> Если снизить размер MTU кэша, убедитесь, что клиенты и системы хранения данных, которые взаимодействуют с кэшем, имеют одинаковую настройку MTU или более низкое значение.

Снижение значения MTU кэша может помочь вам обойти ограничения размера пакета в остальной сети кэша. Например, некоторые VPN не могут успешно передавать полноразмерные пакеты байт 1500 байт. Уменьшение размера пакетов, отправленных через VPN, может устранить эту проблему. Однако обратите внимание, что более низкий параметр MTU кэша означает, что любой другой компонент, который общается с кэшем, включая клиентов и системы хранения данных, также должен иметь более низкую настройку, чтобы избежать проблем с связью с кэшем.

Если вы не хотите изменять настройки MTU на другие компоненты системы, не следует снижать настройки MTU кэша. Есть и другие решения для работы вокруг ограничений размера vpn пакетов. Прочитайте [ограничения размера VPN-пакетов](troubleshoot-nas.md#adjust-vpn-packet-size-restrictions) в статье NAS по устранению неполадок, чтобы узнать больше о диагностике и решении этой проблемы.

Узнайте больше о настройках MTU в виртуальных сетях Azure, [прочитав настройку производительности TCP/IP для MMs Azure.](../virtual-network/virtual-network-tcpip-performance-tuning.md)

## <a name="configure-root-squash"></a>Настройка корневой сквош
<!-- linked from troubleshoot -->

Настройка **root squash Enable** контролирует, как кэш Azure HPC позволяет доступ к корням. Корневой сквош помогает предотвратить доступ к неавторизованным клиентам на уровне корней.

Эта настройка позволяет пользователям контролировать доступ корней на уровне ``no_root_squash`` кэша, что может помочь компенсировать необходимые настройки для систем NAS, используемых в качестве целей хранения. (Подробнее о [целевых предпосылок nFS-хранилища.)](hpc-cache-prereqs.md#nfs-storage-requirements) Он также может повысить безопасность при использовании с целями хранения Azure Blob.

Параметр по умолчанию **Да**. (Кэши, созданные до апреля 2020 года, могут иметь настройку По умолчанию **No**.) При включении эта функция также предотвращает использование битов разрешения UID в запросах клиента в кэш.

## <a name="view-snapshots-for-blob-storage-targets"></a>Просмотр снимков для целей хранения кабликов

Кэш HPC Azure автоматически сохраняет моментальные снимки для хранения данных Azure Blob. Снимки обеспечивают быструю точку отсчета для содержимого контейнера для хранения бэк-энда. Снимки не являются заменой резервных данных и не содержат никакой информации о состоянии кэшированных данных.

> [!NOTE]
> Эта функция снимка отличается от функции снимка, включенной в программное обеспечение для хранения данных NetApp, Isilon или FS. Эти моментальные реализации замывают изменения от кэша к системе хранения бэк-энда перед моментальным снимком.
>
> Для повышения эффективности снимок Кэша HPC Azure сначала не закаливает изменения, а записывает только данные, записанные в контейнер Blob. Этот моментальный снимок не отражает состояние кэшированных данных, поэтому последние изменения могут быть исключены.

Эта функция доступна только для целей хранения данных Azure Blob, и ее конфигурация не может быть изменена.

Снимки делаются каждые восемь часов в UTC 0:00, 08:00 и 16:00.

Azure HPC Cache хранит ежедневные, еженедельные и ежемесячные снимки до тех пор, пока они не будут заменены новыми. Ограничения:

* до 20 ежедневных снимков
* до 8 еженедельных снимков
* до 3 ежемесячных снимков

Получите доступ к `.snapshot` снимкам из каталога в пространстве имени цели вашего хранилища каблов.

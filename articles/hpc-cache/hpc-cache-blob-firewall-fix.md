# <a name="work-around-blob-storage-account-firewall-settings"></a>Обход параметров брандмауэра для учетной записи хранения больших двоичных объектов

Определенные параметры, используемые в брандмауэрах учетной записи хранения данных, могут привести к сбою в создании цели хранения памяти Blob. Команда Azure HPC Cache работает над исправлением программного обеспечения для решения этой проблемы, но вы можете обойти ее, следуя инструкциям в этой статье.

Настройка брандмауэра, которая позволяет получить доступ только из "выбранных сетей", может предотвратить создание кэша целевой системы хранения Blob. Эта конфигурация находится на странице **брандмауэров** и настроек виртуальных сетей учетной записи.

Проблема в том, что служба кэша использует скрытую виртуальную сеть службы, которая отделена от среды клиента. Невозможно прямо разрешить этой сети доступ к учетной записи хранилища.

При создании цели хранения Blob служба кэша использует эту сеть, чтобы проверить, пуст контейнер или нет. Если брандмауэр не позволяет получить доступ из скрытой сети, проверка завершается сбой, а создание цели хранилища завершается сбой.

Чтобы обойти проблему, временно измените настройки брандмауэра при создании цели хранения:

1. Перейдите на учетную запись хранения **Firewalls и страницу виртуальных сетей** и измените настройку "Разрешить доступ от" для **всех сетей.**
1. Создайте цель хранения Blob в кэше Azure HPC.
1. После успешного создания цели хранения измените настройку брандмауэра учетной записи в **выбранные сети.**

Azure HPC Cache не использует виртуальную сеть службы для доступа к готовой цели хранения.

Для получения помощи в этом обходе [обратитесь в службу и поддержку корпорации Майкрософт.](hpc-cache-support-ticket.md)

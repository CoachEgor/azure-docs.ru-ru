# <a name="edit-storage-targets"></a>Изменение целевых расположений хранилищ

Можно удалить или изменить цель хранилища со страницы **целей хранения** кэша.

## <a name="remove-a-storage-target"></a>Удалить цель хранения

Чтобы удалить цель хранения, выберите ее в списке и нажмите кнопку **«Удалить».**

Это действие удаляет связуля целевой системы хранения с этой системой Кэша с накопительной частью Azure, но не изменяет систему хранения бэк-энда. Например, если вы использовали контейнер для хранения Azure Blob, контейнер и его содержимое все еще существуют после удаления его из кэша. Можно добавить контейнер в другой кэш Azure HPC, повторно добавить его в этот кэш или удалить с портала Azure.

Любые изменения файлов, хранящиеся в кэше, записываются в систему хранения бэк-энда до удаления цели хранения. Этот процесс может занять час или больше, если много измененных данных находится в кэше.

## <a name="update-storage-targets"></a>Обновление целевых показателей хранения

Можно отсеять цели хранения, чтобы изменить некоторые из их свойств. Различные свойства отодвиты для различных типов хранения:

* Для целей хранения Blob можно изменить траекторию пространства имен.

* Для целей хранения NFS можно изменить следующие свойства:

  * Путь пространства имен
  * Модель использования
  * Экспорт
  * Экспортная субдиректория

Вы не можете отсеить имя, тип или систему хранения на складе (контейнер Blob или хост-адрес NFS). Если вам нужно изменить эти свойства, удалите цель хранения и создайте замену с новым значением.

Чтобы изменить цель хранения, щелкните целевое имя хранилища, чтобы открыть страницу данных. Некоторые поля на странице можно отделать.

![скриншот страницы отсеивания для целевой системы хранения NFS](media/hpc-cache-edit-storage-nfs.png)

## <a name="update-an-nfs-storage-target"></a>Обновление целевого показателя хранения NFS

Для целевого хранилища NFS можно обновить несколько свойств. (Обратитесь к скриншоту выше для страницы для отобрачения примера.)

* **Модель использования** - Модель использования влияет на то, как кэш сохраняет данные. Прочитайте [выбрать модель использования,](hpc-cache-add-storage.md#choose-a-usage-model) чтобы узнать больше.
* **Виртуальный путь пространства имен** - путь, который клиенты используют для установки этой цели хранения. [Прочтите агрегированное пространство имен](hpc-cache-namespace.md) для получения подробной информации.
* **Путь экспорта NFS** - Экспорт системы хранения для использования для этого пути пространства имен.
* **Путь субдиректоров** - Субдиректор (под экспортом), чтобы связать с этим траекторией пространства имен. Оставьте это поле пустым, если вам не нужно указывать субдиректорию.

Каждый путь пространства имен нуждается в уникальном сочетании экспорта и субдиректоров. То есть, вы не можете сделать два разных клиента лицом пути к точно такой же каталог на бэк-энд системы хранения.

После внесения изменений нажмите **OK,** чтобы обновить цель хранения, или нажмите **Отмена,** чтобы сбросить изменения.

## <a name="update-an-azure-blob-storage-target"></a>Обновление целевого показателя хранения Azure Blob

Страница детализации для цели хранения Blob позволяет изменить виртуальный путь пространства имен.

![скриншот страницы для проверки для цели хранения капли](media/hpc-cache-edit-storage-blob.png)

После завершения, нажмите **OK,** чтобы обновить цель хранения, или нажмите **Отмена,** чтобы отказаться от изменений.

# <a name="working-remotely-using-azure-networking-services"></a>Работа удаленно с помощью сетевых служб Azure

>[!NOTE]
> В этой статье описывается, как можно использовать сетевые службы Azure, сеть Майкрософт и партнерскую экосистему Azure для удаленной работы и смягчения проблем сети, с которыми вы можете столкнуться из-за кризиса COVID-19.

В этой статье описаны варианты, доступные организациям для создания удаленного доступа для своих пользователей или для дополнения существующих решений с дополнительной емкостью в периоды пикового использования. Сетевые архитекторы сталкиваются со следующими проблемами:

- Решение проблемы увеличения использования сети.
- Обеспечить надежную и безопасную связь с большим количеством сотрудников своей компании и клиентов.
- Обеспечить подключение к удаленным местам по всему миру.

Не все сети (например, частные WAN и корпоративные сети ядра) испытывают перегрузку от пиковой удаленной нагрузки на работников. Узкие места обычно регистрируются только в домашних широкополосных сетях и VPN шлюзах корпоративных сетей.

Планировщики сети могут помочь облегчить узкие места и облегчить перегруженность сети, имея в виду, что различные типы трафика нуждаются в различных приоритетах сетевого лечения и некоторыми интеллектуальными перенаправлениеми/распределением нагрузки. Например, теле-медицинский трафик в режиме реального времени взаимодействия врач-пациент имеет большое значение и чувствительны к задержкам/нервным. Принимая во то время, что репликация одного и того же трафика между хранилищами не является задержкой чувствительной. Прежний трафик должен быть направлен по наиболее оптимальному сетевому пути с более высоким качеством обслуживания; в то время как это приемлемо для маршрута более позднего трафика по неоптимальному маршруту.



## <a name="sharing-our-best-practices---azure-network-is-designed-for-elasticity-and-high-availability"></a>Обмен нашими передовыми практиками - сеть Azure предназначена для эластичности и высокой доступности

Azure предназначен для того, чтобы выдерживать внезапные изменения в использовании ресурсов и может в значительной степени помочь в периоды пикового использования. Кроме того, Microsoft поддерживает и управляет одной из крупнейших сетей в мире. Сеть Microsoft была разработана для высокой доступности, которая может выдержать различные типы сбоев: от сбоя одного сетевого элемента до сбоя целого региона.

Сеть Майкрософт предназначена для удовлетворения требований и обеспечения оптимальной производительности для различных типов сетевого трафика, включая задержку конфиденциального мультимедийного трафика для Skype и Teams, CDN, анализа больших данных в реальном времени, хранения данных Azure, Bing и Xbox. Чтобы обеспечить оптимальную производительность для различных типов трафика, сеть Microsoft привлекает весь трафик, который предназначен для или желающих транзита через свои ресурсы как можно ближе к происхождению трафика.

>[!NOTE] 
>Использование сетевых функций Azure, описанных ниже, использует поведение притяжения трафика глобальной сети Майкрософт для обеспечения лучшего взаимодействия с клиентами. Поведение притяжения трафика сети Microsoft помогает поскорее загрузить трафик из сетей первой/последней мили, которые могут испытывать заторы в периоды пикового использования.
>

## <a name="enable-employees-to-work-remotely"></a>Позволяет сотрудникам работать удаленно

Vpn шлюз Azure поддерживает VPN-соединения от точки к сайту (P2S) и сайт-к-сайту (S2S). Используя VPN-шлюз Azure, вы можете масштабировать соединения сотрудников, чтобы обеспечить безопасный доступ как к развернутым ресурсам Azure, так и к ресурсам. Для получения дополнительной информации [см.](../vpn-gateway/work-remotely-support.md) 

При использовании протокола туннелирования безопасных розеток (SSTP) количество одновременных подключений ограничено 128. Чтобы получить большее количество подключений, мы предлагаем переход на OpenVPN или IKEv2. Для получения дополнительной информации смотрите [переход к протоколу OpenVPN или IKEv2 от SSTP](../vpn-gateway/ikev2-openvpn-from-sstp.md
).

Чтобы получить доступ к ресурсам, развернутым в Azure, удаленные разработчики могут использовать решение Azure Bastion вместо VPN-соединения, чтобы получить безопасный доступ к оболочке (RDP или SSH), не требуя доступа к vpn.ru. Для получения дополнительной информации [см.](../bastion/work-remotely-support.md)

Для агрегирования крупномасштабных VPN-соединений, для поддержки любых подключений между ресурсами в различных внутренних глобальных точках, в различных региональных центрах и виртуальных сетях, а также для оптимизации использования нескольких домашних широкополосных сетей можно использовать Azure Virtual WAN. Для получения дополнительной информации, см [Борьба удовлетворить работу из дома потребностей? Вот где Azure Виртуальный WAN может помочь](../virtual-wan/work-remotely-support.md).

Еще один способ поддержки удаленной рабочей силы — развертывание виртуальной инфраструктуры настольных компьютеров (VDI), размещенной в виртуальной сети Azure, защищенной брандмауэром Azure Firewall. Например, Windows Virtual Desktop (WVD) — это служба виртуализации настольных компьютеров и приложений, которая работает в Azure. С помощью Windows Virtual Desktop вы можете настроить масштабируемую и гибкую среду в подписке Azure без необходимости запуска дополнительных серверов шлюза. Вы несете ответственность только за виртуальные машины WVD в вашей виртуальной сети. Для получения дополнительной информации обратитесь в [службу удаленной работы Azure Firewall.](../firewall/remote-work-support.md) 

Azure также имеет богатый набор партнеров по экосистемам. Наши партнеры Сети Виртуальные приборы на Azure также может помочь масштабировать VPN подключения. Для получения дополнительной информации, см [Сеть Виртуальный прибор (NVA) соображения для удаленной работы](../vpn-gateway/nva-work-remotely-support.md).

## <a name="extend-employees-connection-to-access-globally-distributed-resources"></a>Расширить доступ сотрудников к глобально распределенным ресурсам

Следующие службы Azure могут помочь сотрудникам получить доступ к общемировой распределенным ресурсам. Ваши ресурсы могут быть в любом из регионов Azure, в закрытых сетях или даже в других общедоступных или частных облаках. 

- **Виртуальная сеть Azure:** Если вы развертываете свои ресурсы в нескольких регионах Azure и/или если вы агрегируете подключение удаленно работающих сотрудников с помощью нескольких виртуальных сетей, можно установить подключение между несколькими виртуальными сетями Azure с помощью виртуальной сети пиринга. Дополнительные сведения см. в статье [Пиринг между виртуальными сетями][VNet-peer].

- **Решение на основе VPN Azure:** Для удаленных сотрудников, подключенных к Azure через P2S или S2S VPN, вы можете включить доступ к предприимжениям, настраивая S2S VPN между вашими внепредки и vpn-шлюзом Azure. Для получения дополнительной информации [см.][S2S]

- **ExpressRoute**: С помощью частного пиринга ExpressRoute можно обеспечить частную связь между развертыванием Azure и инфраструктурой на местах или инфраструктурой в объекте совместного размещения. ExpressRoute через пиринг Майкрософт также позволяет получить доступ к общедоступным конечным точкам в корпорации Майкрософт из вашей предварительной сети. Подключения ExpressRoute не проходят через общедоступный Интернет. Они обеспечивают безопасную связь, надежность, более высокую пропускную способность, с более низкими и последовательными просрочками, чем обычные подключения через Интернет. См. дополнительные сведения об [ExpressRoute][ExR]. Использование существующего сетевого провайдера, который уже является частью нашей [партнерской экосистемы ExpressRoute,][ExR-eco] может помочь сократить время получения больших подключений к Microsoft.  С помощью [ExpressRoute Direct][ExR-D] вы можете напрямую подключить свою локтую сеть к магистрали Майкрософт. ExpressRoute Direct предлагает два различных линейных опция двойных 10 Гбит/с или 100 Гбит/с. 

- **Azure Virtual WAN**: Azure Virtual WAN обеспечивает бесшовную совместимость между VPN-соединениями и схемами ExpressRoute. Как упоминалось ранее, Azure Virtual WAN также поддерживает любые связи между ресурсами в различных глобальных местах на prem, в различных региональных центрах и говорил виртуальных сетей

## <a name="microsoft-support-for-multi-cloud-traffic"></a>Поддержка мультиоблачным трафиком майкрософт

Для развертывания в других общедоступных облаках корпорация Майкрософт может обеспечить глобальное подключение. В этом отношении могут помочь Azure Virtual WAN, VPN или ExpressRoute. Чтобы расширить подключение от Azure к другим облакам, можно настроить S2S VPN между двумя облаками. Можно также установить подключение из Azure в другие общедоступные облака с помощью ExpressRoute. Облако Oracle является частью партнерской экосистемы ExpressRoute. Можно [настроить прямую взаимосвязь между Azure и облачной инфраструктурой Oracle.][Az-OCI] Большинство поставщиков услуг, входят в партнерскую экосистему ExpressRoute, также предлагают частную связь с другими общедоступными облаками. Используя этих поставщиков услуг, можно установить частную связь между развертыванием в Azure и другими облаками через ExpressRoute.

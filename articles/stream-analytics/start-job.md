# <a name="how-to-start-an-azure-stream-analytics-job"></a>Как запустить задание Azure Stream Analytics

Можно запустить задания Azure Stream Analytics с помощью портала Azure, Visual Studio и PowerShell. При запуске задания выбрано время для начала создания выходных данных задания. Портал Azure, Visual Studio и PowerShell существуют различные методы для установки времени начала. Эти методы описаны ниже.

## <a name="start-options"></a>Параметры запуска
Три следующие параметры доступны для запуска задания. Обратите внимание, что все значения времени, указанный ниже значений, указанных в [TIMESTAMP BY](https://docs.microsoft.com/stream-analytics-query/timestamp-by-azure-stream-analytics). Если TIMESTAMP BY не указан, будет использоваться время поступления.
* **Теперь**: Упрощает начальную точку выходное событие потоковой передачи, так же, как при запуске задания. Если используется оператор темпоральной (например периода ЗАДЕРЖКИ или СОЕДИНЕНИЯ), Azure Stream Analytics будет автоматически взглянуть на данные в источнике входных данных. Например при запуске задания «Сейчас», и если в запросе используется "переворачивающегося" окна 5 минут, Azure Stream Analytics выполняет поиск данных от 5 минут назад во входных данных.
Первое событие вывода будет иметь метку времени меньше, чем текущее время и ASA гарантирует, что все события ввода, которые логически могут влиять на выход учтены. Например создаются не частичного агрегаты. Это всегда полный статистическое значение.

* **Настраиваемая**. Вы можете выбрать начальную точку выходные данные. Так же, как **теперь** параметр, Azure Stream Analytics будет автоматически считывать данные до этого времени, если используется оператор темпоральной 

* **Во время последней остановки**. Этот параметр доступен, если задание ранее был запущен, но была остановлена вручную или не удалось. При выборе этого параметра Azure Stream Analytics будет использовать время последнего вывода данных, чтобы перезапустить задание, поэтому данные не будут утеряны. Аналогичным образом для предыдущих способах Azure Stream Analytics будет автоматически считывать данные до этого времени при использовании темпоральной оператор. Поскольку несколько секций входных данных может иметь другое время, используется самое раннее время остановки всех разделов, в результате некоторые дубликаты возможны в выходных данных. Дополнительные сведения о точности-одну обработку, доступные на странице [гарантирует доставку событий](https://docs.microsoft.com/stream-analytics-query/event-delivery-guarantees-azure-stream-analytics).


## <a name="azure-portal"></a>Портал Azure

Перейдите к заданию в на портале Azure и выберите **запустить** на странице "Обзор". Выберите **время начала создания выходных данных задания** , а затем выберите **запустить**.

Выберите один из вариантов для **время начала создания выходных данных задания**. Возможные варианты: *теперь*, *Custom*, и, если задание запускалось ранее, *во время последней остановки*. Дополнительные сведения об этих параметрах см. выше.

## <a name="visual-studio"></a>Visual Studio

В представлении задания выберите кнопку с зеленой стрелкой, чтобы запустить задание. Задайте **режим запуска выходные данные задания** и выберите **запустить**. Состояние задания изменится на **под управлением**.

Существует три варианта для **режим запуска выходные данные задания**: *JobStartTime*, *CustomTime*, и *LastOutputEventTime*. Если это свойство отсутствует, по умолчанию используется *JobStartTime*. Дополнительные сведения об этих параметрах см. выше.


## <a name="powershell"></a>PowerShell

Для запуска задания с помощью PowerShell, используйте следующий командлет:

```powershell
Start-AzStreamAnalyticsJob `
  -ResourceGroupName $resourceGroup `
  -Name $jobName `
  -OutputStartMode 'JobStartTime'
```

Существует три варианта для **OutputStartMode**: *JobStartTime*, *CustomTime*, и *LastOutputEventTime*. Если это свойство отсутствует, по умолчанию используется *JobStartTime*. Дополнительные сведения об этих параметрах см. выше.

Дополнительные сведения о `Start-AzStreamAnalyitcsJob` командлета, представление [Справочник по Start-AzStreamAnalyticsJob](/powershell/module/az.streamanalytics/start-azstreamanalyticsjob).

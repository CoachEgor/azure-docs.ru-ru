# <a name="choose-a-real-time-analytics-and-streaming-processing-technology-on-azure"></a>Выберите аналитики в реальном времени и потоковой передачи технологии обработки в Azure

Существует несколько служб для аналитики в реальном времени и потоковой обработки в Azure. В этой статье сведения, вам нужно решить, какая технология лучше всего подходит для вашего приложения.

## <a name="when-to-use-azure-stream-analytics"></a>Когда следует использовать Azure Stream Analytics

Azure Stream Analytics является рекомендуемой службой для stream analytics в Azure. Она предназначена для широкого ряда сценариев, которые включают, но не ограничиваясь:

* Панели мониторинга для визуализации данных
* В режиме реального времени [оповещения](stream-analytics-set-up-alerts.md) из темпоральной и пространственных шаблонов или аномалии
* Извлечение, преобразование и загрузка.
* [Шаблон источников событий](/azure/architecture/patterns/event-sourcing)
* [IoT Edge](stream-analytics-edge.md)

Добавление Azure Stream Analytics задания в приложение — самый быстрый способ получить аналитики потоковой передачи, копирования и работающих в Azure с помощью языка SQL уже известно. Azure Stream Analytics — служба обработки заданий, поэтому вам не нужно тратить время управление кластеров, и вам не нужно беспокоиться о времени простоя, связанного с соглашением об уровне ОБСЛУЖИВАНИЯ 99,9% времени на уровне задания. Выставление счетов также выполняется на уровне задания, что делает запуска издержки низкой (один модуль потоковой передачи), но масштабируемые (до 192 единицы потоковой передачи). Это гораздо более выгодно для выполнения нескольких заданий Stream Analytics, а не для запуска и поддержки кластера.

Azure Stream Analytics имеет более удобной out of box. Можно немедленно воспользоваться преимуществами следующих возможностей без дополнительных настроек:

* Встроенные временных операторов, таких как [агрегаты](stream-analytics-window-functions.md), темпоральные соединения и темпоральные аналитические функции.
* Azure имеет встроенную систему [входной](stream-analytics-add-inputs.md) и [вывода](stream-analytics-define-outputs.md) адаптеров
* Поддержку для медленно изменяющихся [ссылочные данные](stream-analytics-use-reference-data.md) (также известный как таблицы подстановки), включая объединение с геопространственными данными ссылку для географическое зонирование.
* Интегрированные решения, такие как [обнаружение аномалий](stream-analytics-machine-learning-anomaly-detection.md)
* Несколько окон времени в одном запросе
* Возможность создания нескольких временных операторов в произвольной последовательности.
* В разделе 100 мс задержки end-to-end входных данных, поступающих в концентраторы событий, для вывода целевая в концентраторы событий, включая задержки сети и концентраторов событий из постоянной высокой пропускной способностью

## <a name="when-to-use-other-technologies"></a>Когда следует использовать другие технологии

### <a name="you-need-to-input-from-or-output-to-kafka"></a>Входные данные из или выходные данные в Kafka

Azure Stream Analytics не входных данных используется Apache Kafka или выходной адаптер. Если у вас события в целевой или нужно отправить в Kafka, и у вас нет необходимость выполнения собственный кластер Kafka, вы можете использовать Stream Analytics, отправка событий в концентраторы событий с помощью Kafka API концентраторов событий, не изменяя отправителя события. Если нужно запустить собственный кластер Kafka, можно использовать структурированную потоковую передачу Spark, который полностью поддерживается в [Azure Databricks](../azure-databricks/index.yml), или Storm на [Azure HDInsight](../hdinsight/storm/apache-storm-tutorial-get-started-linux.md).

### <a name="you-want-to-write-udfs-udas-and-custom-deserializers-in-a-language-other-than-javascript-or-c"></a>Вы хотите написать на языке, кроме JavaScript определяемых пользователем функций, определяемых пользователем агрегатных функций и пользовательских deserializers илиC#

Azure Stream Analytics поддерживает определяемые пользователем функции (UDF) или определяемые пользователем статистические функции (UDA) в JavaScript для заданий в облаке и C# для заданий Edge Интернета вещей. C#определяемые пользователем deserializers также поддерживаются. Если вы хотите реализовать десериализатора, определяемой пользователем функции или определяемой пользователем агрегатной функции в других языках, например Java или Python, можно использовать структурированную потоковую передачу Spark. Можно также запустить концентраторов событий **EventProcessorHost** на собственных виртуальных машинах произвольные обработки потоковой передачи.

### <a name="your-solution-is-in-a-multi-cloud-or-on-premises-environment"></a>Ваше решение — в несколько облачных, так и в локальной среде

Azure Stream Analytics — это технология собственности корпорации Майкрософт и доступна только в Azure. Если вам нужна для можно переносить на другие облака или локального решения, рассмотрим технологии открытым исходным кодом, например структурированной потоковой передачи Spark или Storm.

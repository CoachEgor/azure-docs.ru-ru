# <a name="choose-a-real-time-analytics-and-streaming-processing-technology-on-azure"></a>Выберите технологию аналитики и потоковой обработки в режиме реального времени на Azure

Есть несколько служб, доступных для аналитики в реальном времени и обработки потоковой передачи на Azure. В этой статье содержится информация, необходимая для решения о том, какая технология лучше всего подходит для вашего приложения.

## <a name="when-to-use-azure-stream-analytics"></a>Когда использовать аналитику потоков Azure

Azure Stream Analytics — рекомендуемая служба для анализа потоков в Azure. Он предназначен для широкого спектра сценариев, которые включают, но не ограничиваются:

* Панели мониторинга для визуализации данных
* [Предупреждения в режиме](stream-analytics-set-up-alerts.md) реального времени от временных и пространственных моделей или аномалий
* Извлечение, преобразование и загрузка.
* [Шаблон поиска событий](/azure/architecture/patterns/event-sourcing)
* [IoT Край](stream-analytics-edge.md)

Добавление задания Azure Stream Analytics в приложение — это самый быстрый способ запуска потоковой аналитики в Azure с помощью уже известного языка S'L. Azure Stream Analytics — это служба работы, поэтому вам не придется тратить время на управление кластерами, и вам не придется беспокоиться о простое с 99,9% SLA на уровне задания. Выставление счетов также осуществляется на уровне работы, что делает затраты на запуск низкими (одна потоковая единица), но масштабируемыми (до 192 единиц потоковой передачи). Гораздо более рентабельно выполнить несколько заданий Stream Analytics, чем запустить и поддерживать кластер.

Azure Stream Analytics обладает богатым нестандартным опытом работы. Вы можете сразу воспользоваться следующими функциями без каких-либо дополнительных настойок:

* Встроенные временные операторы, такие как [оконные агрегаты,](stream-analytics-window-functions.md)временные соединения и временные аналитические функции.
* Адаптеры [ввода](stream-analytics-add-inputs.md) и [вывода](stream-analytics-define-outputs.md) Native Azure
* Поддержка медленно меняющихся [справочных данных](stream-analytics-use-reference-data.md) (также известных как таблицы поиска), включая присоединение к геопространственным справочным данным для геофеченции.
* Интегрированные решения, такие как [обнаружение аномалий](stream-analytics-machine-learning-anomaly-detection.md)
* Несколько временных окон в одном и том же запросе
* Возможность сочинять несколько временных операторов в произвольных последовательностях.
* Под 100-мс сквозной задержкой от ввода, поступающей в центры событий, до вывода посадки в концентрах событий, включая задержку сети от и до концентраторов событий, при устойчивой высокой пропускной состоянии

## <a name="when-to-use-other-technologies"></a>Когда использовать другие технологии

### <a name="you-want-to-write-udfs-udas-and-custom-deserializers-in-a-language-other-than-javascript-or-c"></a>Вы хотите написать UDF, UDA и пользовательские десеризаторы на языке, отличном от JavaScript или C #

Azure Stream Analytics поддерживает функции, определяемые пользователями (UDF) или пользовательские агрегаты (UDA) в JavaScript для облачных заданий и c -S для заданий IoT Edge. Также поддерживаются десериализаторы, определяемые пользователями. Если вы хотите реализовать десеризатор, UDF или UDA на других языках, таких как Java или Python, вы можете использовать spark Structured Streaming. Вы также можете запустить Event Hubs **EventProcessorHost** на собственных виртуальных машинах, чтобы сделать произвольную обработку потоковой передачи.

### <a name="your-solution-is-in-a-multi-cloud-or-on-premises-environment"></a>Ваше решение находится в многооблачной или найме

Azure Stream Analytics — это запатентованная технология Корпорации Майкрософт, доступная только в Azure. Если вам нужно, чтобы ваше решение было переносимо в облаках или в помещениях, подумайте о технологиях с открытым исходным кодом, таких как Spark Structured Streaming или Storm.

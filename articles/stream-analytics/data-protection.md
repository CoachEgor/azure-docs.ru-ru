# <a name="data-protection-in-azure-stream-analytics"></a>Защита данных в аналитике потоков Azure 

Azure Stream Analytics — это полностью управляемая платформа как услуга, которая позволяет создавать конвейеры аналитики в режиме реального времени. Вся тяжелая работа, такая как кластерная подготовка, масштабирование узлов для размещения вашего использования и управление внутренними контрольно-пропускными пунктами, управляется за кулисами.

## <a name="encrypt-your-data"></a>Шифрование данных

Stream Analytics автоматически использует лучшие в своем классе стандарты шифрования по всей своей инфраструктуре для шифрования и защиты ваших данных. Вы можете просто доверять Stream Analytics надежно хранить все ваши данные, чтобы вам не пришлось беспокоиться об управлении инфраструктурой.

Если вы хотите использовать управляемые клиентом ключи (CMK) для шифрования данных, вы можете использовать свой собственный учет данных (общее назначение V1 или V2) для хранения любых частных ресурсов данных, которые требуются время выполнения Stream Analytics. Ваша учетная запись хранилища может быть зашифрована по мере необходимости. Ни один из ваших личных ресурсов данных не хранится постоянно инфраструктурой Stream Analytics. 

Этот параметр должен быть настроен во время создания рабочих мест Stream Analytics и не может быть изменен на протяжении всего жизненного цикла работы. Не рекомендуется модифицировать или узлать хранилище, используемое вашим Stream Analytics. Если вы удалите свою учетную запись хранения, вы навсегда удалите все личные ресурсы данных, что приведет к сбою в работе. 

Обновление или вращение ключей к учетной записи хранилища невозможно с помощью портала Stream Analytics. Вы можете обновить ключи с помощью APIs REST.


## <a name="configure-storage-account-for-private-data"></a>Настройка учетной записи хранения для личных данных 

Используйте следующие шаги для настройки учетной записи хранилища для личных ресурсов данных. Эта конфигурация производится из задания Stream Analytics, а не из учетной записи хранилища.

1. Войдите на [портал Azure](https://portal.azure.com/).

1. Щелкните **Создать ресурс** в верхнем левом углу окна портала Azure. 

1. Выберите **задание**  **Analytics** >Stream Analytics из списка результатов. 

1. Заполните страницу вакансий Stream Analytics с необходимыми сведениями, такими как имя, область и масштаб. 

1. Выберите флажок, в который *говорится, что защищайте все личные ресурсы данных, необходимые для этой работы, в моем аккаунте хранения.*

1. Выберите учетную запись хранилища из подписки. Обратите внимание, что этот параметр не может быть изменен на протяжении всего жизненного цикла задания. 

   ![Настройки учетной записи хранения личных данных](./media/data-protection/storage-account-create.png)

## <a name="private-data-assets-that-are-stored"></a>Сохраненные частные ресурсы данных

Любые личные данные, которые должны быть сохранены Stream Analytics, хранятся в вашей учетной записи хранения. Примеры частных активов данных: 

* Запросы, которые вы автор и связанные с ними конфигурации  

* Определяемые пользователем функции 

* Контрольно-пропускные пункты, необходимые для выполнения Stream Analytics

* Снимки справочных данных 

Также сохраняются сведения о подключении ваших ресурсов, которые используются в работе Stream Analytics. Шифрование учетной записи хранилища для защиты всех данных. 

Чтобы помочь вам выполнить свои обязательства по соблюдению нормативных требований в любой регулируемой отрасли или среде, вы можете узнать больше о [предложениях корпорации Майкрософт по обеспечению соответствия](https://gallery.technet.microsoft.com/Overview-of-Azure-c1be3942)требованиям.

# <a name="geofencing-and-geospatial-aggregation-scenarios-with-azure-stream-analytics"></a>Географическое зонирование и геопространственные сценарии статистической обработки с помощью Azure Stream Analytics

С встроенные геопространственные функции можно использовать для создания приложений для сценариев, таких как управление транспортным отделом и воспользуйтесь преимуществами нового совместного использования, подключенные автомобили и отслеживания ресурсов Azure Stream Analytics.

## <a name="geofencing"></a>Геозоны

Azure Stream Analytics поддерживает вычисления в режиме реального времени географическое зонирование низкой задержкой, в облаке и на среду выполнения IoT Edge.

### <a name="geofencing-scenario"></a>Географическое зонирование сценария

Производственная компания должна отслеживать ресурсы на их зданий. Они оборудованы каждого устройства с GPS и хотите получать уведомления при удалении устройства из определенной области.

Ссылочные данные, используемые в этом примере содержит сведения о геозоны здания и устройства, которые допускаются в каждом из здания. Помните, что ссылочных данных может быть статическим или медленно изменения. В этом сценарии используется статические эталонные данные. Поток данных постоянно отправляет идентификатор устройства и его текущее положение.

### <a name="define-geofences-in-reference-data"></a>Определение геозон в ссылочных данных

Геозоны можно определить с помощью объекта GeoJSON. Для заданий с совместимости версии 1.2 и более поздних версий, геозон также можно определить с помощью Well Known Text (WKT) как `NVARCHAR(MAX)`. WKT — это открытого геопространственного консорциума (OGC) standard, используемый для представления пространственных данных в текстовом формате.

Встроенные геопространственные функции позволяет проверить, является ли элемент в или из конкретного геозоны многоугольник определенных геозон.

### <a name="generate-alerts-with-geofence"></a>Формировать предупреждения с помощью геозоны

Устройства могут создавать свои идентификатор и расположение каждую минуту через поток с именем `DeviceStreamInput`. Следующая таблица представляет собой поток входных данных.

|идентификатор устройства|GeoPosition|
|--------|-----------|
|«A»|«POINT(-122.13292341559497 47.636318374032726)»|
|«B»|«POINT(-122.13338475554553 47.63743531308874)»|
|"C"|«POINT(-122.13354001095752 47.63627622505007)»|

Можно написать запрос, который соединяет потоке устройства со ссылочными данными геозоны и создает предупреждение каждый раз, чтобы устройство выходит за пределы допустимых стандартных.

```SQL
SELECT DeviceStreamInput.DeviceID, SiteReferenceInput.SiteID, SiteReferenceInput.SiteName 
INTO Output
FROM DeviceStreamInput 
JOIN SiteReferenceInput
ON st_within(DeviceStreamInput.GeoPosition, SiteReferenceInput.Geofence) = 0
WHERE DeviceStreamInput.DeviceID = SiteReferenceInput.AllowedDeviceID
```

На следующем рисунке показано геозон. Можно увидеть, где устройства находятся в соответствии с входных данных потока.

![Построение геозон](./media/geospatial-scenarios/building-geofences.png)

Устройство «C» расположен в здании 2 идентификатор, что не допускается в соответствии с ссылочных данных. Это устройство должны быть расположены внутри построение идентификатор 3. Выполняется запуск этого задания будет создавать оповещение для этого конкретного нарушения.

### <a name="site-with-multiple-allowed-devices"></a>Сайт с помощью нескольких разрешенных устройств

Если сайт позволяет нескольким устройствам, массив идентификаторов устройств могут быть определены в `AllowedDeviceID` и определяемые пользователем функции могут использоваться на `WHERE` предложение для проверки, если идентификатор потока соответствует любой идентификатор устройства в этом списке. Дополнительные сведения см. [определяемой пользователем функции Javascript](stream-analytics-javascript-user-defined-functions.md) учебника для заданий в облаке и [ C# определяемой пользователем функции](stream-analytics-edge-csharp-udf.md) учебника для заданий edge.

### <a name="geospatial-aggregation-scenario"></a>Сценарии геопространственного статистической обработки

CAB-файла необходимо создать приложение в режиме реального времени по использованию их драйверы cab, поиск по дороге к области городов, в настоящее время происходят более востребована.

Компания хранит логические области, города, в виде ссылочных данных. Каждый регион определяется RegionID, RegionName и Геозоны.

Следующий запрос соединяет потоке устройства со ссылочными данными геозоны и вычисляет количество запросов на регион в окне времени 15 минут каждую минуту.

```SQL
SELECT count(*) as NumberOfRequests, RegionsRefDataInput.RegionName 
FROM UserRequestStreamDataInput
JOIN RegionsRefDataInput 
ON st_within(UserRequestStreamDataInput.FromLocation, RegionsRefDataInput.Geofence) = 1
GROUP BY RegionsRefDataInput.RegionName, hoppingwindow(minute, 1, 15)
```

Этот запрос выводит число запросов, инициированных каждую минуту за последние 15 минут с каждого региона в городе. Эти сведения могут отображаться панель мониторинга Power BI легко или широковещательно на все драйверы как текстовые SMS-сообщения за счет интеграции со службами, например функции Azure.

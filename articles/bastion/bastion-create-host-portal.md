# <a name="create-an-azure-bastion-host-using-the-portal"></a>Создание хоста Azure Bastion с помощью портала

В этой статье показано, как создать хост Azure Bastion с помощью портала Azure. После предоставления службы Azure Bastion в виртуальной сети всем виртуальным сетям доступна бесшовная служба RDP/SSH. Бастион Azure развертывается на уровне виртуальной сети, а не подписки, учетной записи или виртуальной машины.

Вы можете создать новый ресурс узла бастиона на портале либо путем указания всех настроек вручную, либо с помощью настроек, которые соответствуют существующему VM. Чтобы создать унист бастиона с помощью настроек VM, см. [quickstart](quickstart-host-portal.md) Дополнительно можно использовать [Azure PowerShell](bastion-create-host-powershell.md) для создания узла Azure Bastion.

## <a name="create-a-bastion-host"></a><a name="createhost"></a>Создание бастиона

Этот раздел поможет создать новый ресурс Azure Bastion с портала Azure.

1. В меню [портала Azure](https://portal.azure.com) или на **главной** странице выберите **«Создайте ресурс».**

1. На **новой** странице, в поле *поиска Marketplace,* введите **Бастион**, затем нажмите **Enter,** чтобы добраться до результатов поиска.

1. Из результатов нажмите **Бастион**. Убедитесь, что издателем является *корпорация Майкрософт*, а категорией — *Сети*.

1. На странице **Бастион** нажмите **Создать,** чтобы открыть **страницу Создать бастион.**

1. На странице **«Создать бастион»** назначайте новый ресурс «Бастион». Укажите параметры конфигурации для ресурса Bastion.

    ![создать бастион](./media/bastion-create-host-portal/settings.png)

    * **Подписка**: Подписка Azure, которая требуется использовать для создания нового ресурса Bastion.
    * **Группа ресурсов**: Группа ресурсов Azure, в которой будет создан новый ресурс Bastion. Если у вас нет существующей группы ресурсов, можно создать новую.
    * **Название**: Название нового ресурса Бастион
    * **Регион**: Область общего пользования Azure, в которую будет создан ресурс.
    * **Виртуальная сеть**: Виртуальная сеть, в которой будет создан ресурс Bastion. Во время этого процесса можно создать новую виртуальную сеть на портале или использовать существующую виртуальную сеть. Если вы используете существующую виртуальную сеть, убедитесь, что существующая виртуальная сеть имеет достаточно свободного адресного пространства для удовлетворения требований подсети Bastion.
    * **Subnet**: Подсеть в виртуальной сети, в которой будет развернут новый ресурс bastion. Необходимо создать подсеть, используя значение имени **AzureBastionSubnet.** Это значение позволяет Azure знать, в какую подсеть развертывать ресурсы Bastion. Это отличается от подсети Gateway. Вы должны использовать подсеть, по крайней мере /27 или больше (/27, /26, и так далее).
    
       Создайте **AzureBastionSubnet** без каких-либо таблиц маршрутов или делегаций. Если вы используете группы сетевой безопасности в **AzureBastionSubnet,** обратитесь к статье [«Работа с НС».](bastion-nsg.md)
    * **Общественный IP-адрес**: Общественный IP ресурса Бастиона, на котором будет доступна RDP/SSH (над портом 443). Создайте новый общедоступный IP или используйте существующий IP.. Общественный IP-адрес должен находиться в том же регионе, что и создаваемый ресурсом Bastion.
    * **Имя публичного IP-адреса**: Название общедоступного ресурса IP-адреса.
    * **Общественный IP-адрес SKU**: Этот параметр предварительно заселен по умолчанию для **Стандарта.** Azure Bastion использует/поддерживает только стандартный общественный IP SKU.
    * **Назначение**: Этот параметр предварительно заселен по умолчанию в **Static**.

1. Когда вы закончите указывать настройки, нажмите **«Обзор» и создайте**. Это проверяет значения. Как только проверка пройдет, вы можете начать процесс создания.
1. На странице **«Создать бастион»** нажмите **«Создать».**
1. Вы увидите сообщение, в которое вы узнаете, что развертывание продолжается. Статус будет отображаться на этой странице по мере создания ресурсов. Создание и развертывание ресурса «Бастион» занимает около 5 минут.

# <a name="connect-to-a-windows-virtual-machine-using-azure-bastion-preview"></a>Подключение к виртуальной машине Windows, с помощью Бастиона Azure (Предварительная версия)

В этой статье рассказывается, как для безопасно и беспрепятственно протокола удаленного рабочего СТОЛА к виртуальным машинам Windows в Azure виртуальной сети с помощью Azure Бастиона. Подключиться к виртуальной машине можно непосредственно на портале Azure. При использовании службы "Бастион Azure" на виртуальной машине не нужно устанавливать клиенты, агенты или дополнительное программное обеспечение. Дополнительные сведения о Бастиона Azure, см. в разделе [Обзор](bastion-overview.md).

> [!IMPORTANT]
> Эта общедоступная предварительная версия предоставляется без соглашения об уровне обслуживания и не должна использоваться для производственных рабочих нагрузок. Некоторые функции могут не поддерживаться, иметь ограничения и быть доступными не во всех расположениях Azure. См. [дополнительные условия использования для предварительных версий Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).
>

## <a name="before-you-begin"></a>Перед началом работы

Убедитесь, что вы настроили Azure бастионом для виртуальной сети, в котором размещается виртуальная машина. Дополнительные сведения см. в разделе [создать узел-Бастион Azure](bastion-create-host-portal.md). Как только служба Бастиона подготовить и развернуть в виртуальной сети, его можно использовать для подключения к любой виртуальной Машины в этой виртуальной сети. В этой предварительной версии Бастиона предполагается, что вы используете RDP для подключения к виртуальной Машине Windows и SSH для подключения к виртуальным машинам Linux. Сведения о подключении к виртуальной Машине Linux, см. в разделе [подключение к виртуальной Машине - Linux](bastion-connect-vm-ssh.md).

Чтобы создать подключение, необходимы следующие роли:

* Роль читателя на виртуальной машине
* Роль читателя в сетевой КАРТЕ частный IP-адресом виртуальной машины
* Роль читателя в ресурсе Azure Бастиона

## <a name="rdp"></a>Подключение с использованием протокола удаленного рабочего СТОЛА

1. В [портала Azure](https://aka.ms/BastionHost) для предварительного просмотра Бастиона, перейдите к виртуальной машине, который вы хотите подключиться к, а затем нажмите кнопку **Connect**. При использовании RDP-подключение виртуальной Машины следует виртуальной машины Windows.

    ![Подключение виртуальной Машины](./media/bastion-connect-vm-rdp/connect.png)

1. После нажатия кнопки Connect боковой панели появится, содержит три вкладки — по протоколу RDP, SSH и Бастиона. Если Бастиона был подготовлен для виртуальной сети, по умолчанию активна вкладка Бастиона. Если вы не подготавливаете Бастиона для виртуальной сети, можно щелкнуть ссылку, чтобы настроить Бастиона. Инструкции по настройке, см. в разделе [Настройка Бастиона](bastion-create-host-portal.md). Если вы не видите **Бастиона** в списке, не открыты на портале предварительной версии. Откройте портал, с помощью этого [ссылка](https://aka.ms/BastionHost).

    ![Подключение виртуальной Машины](./media/bastion-connect-vm-rdp/bastion.png)

1. На вкладке Бастиона, имя пользователя и пароль для виртуальной машины, нажмите кнопку **Connect**. RDP-подключение к этой виртуальной машине с помощью Бастиона откроется непосредственно на портале Azure (через HTML5) с использованием порта 443 и службу Бастиона.

    ![Подключение виртуальной Машины](./media/bastion-connect-vm-rdp/443rdp.png)

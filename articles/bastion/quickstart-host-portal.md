# <a name="quickstart-connect-to-a-virtual-machine-using-a-private-ip-address-and-azure-bastion"></a>Краткое руководство. подключение к виртуальной машине с помощью частного IP-адреса и Azure бастиона

В этой статье краткого руководства показано, как подключиться к виртуальной машине с помощью частного IP-адреса. При подключении через бастиона виртуальным машинам не требуется общедоступный IP-адрес. Действия, описанные в этой статье, помогут вам развернуть бастиона в виртуальной сети с помощью виртуальной машины на портале. После подготовки службы интерфейс RDP/SSH доступен для всех виртуальных машин в той же виртуальной сети.

## <a name="prereq"></a>Предварительные требования

* Виртуальная сеть Azure.
* Виртуальная машина Azure, расположенная в виртуальной сети с портом 3389, открыта.

|**Название** | **Value** |
| --- | --- |
| Имя |  VNet1Bastion |
| Регион | eastus |
| Виртуальная сеть |  VNet1 |
| + Имя подсети | азуребастионсубнет |
| Азуребастионсубнет адреса |  10.1.254.0/27 |

| Номер SKU общедоступного IP-адреса |  Standard  |
| Назначение  | Static |

## <a name="createvmset"></a>Создание узла бастиона

При создании узла бастиона на портале с помощью существующей виртуальной машины различные параметры по умолчанию будут автоматически соответствовать вашей виртуальной машине и (или) виртуальной сети.

1. Откройте [портал Azure](https://portal.azure.com). Перейдите к виртуальной машине и нажмите кнопку **подключить**.

   ![параметры виртуальной машины](./media/quickstart-host-portal/vm-settings.png)
1. В раскрывающемся списке выберите **бастиона**.
1. На странице Подключение выберите **использовать бастиона**.

   ![Выбор бастиона](./media/quickstart-host-portal/select-bastion.png)

1. На странице бастиона заполните следующие поля параметров:

   * **Имя**: имя узла бастиона.
   * **Подсеть**. подсеть в виртуальной сети, в которую будет развернут ресурс бастиона. Подсеть должна быть создана с именем **азуребастионсубнет**. Имя позволяет Azure выяснить, в какой подсети развертывается ресурс бастиона. Это отличается от подсети шлюза. Используйте подсеть размером не менее/27 или больше (/27,/26,/25 и т. д.).
   
      * Выберите **Управление конфигурацией подсети**, а затем выберите **+ подсеть**.
      * На странице Добавление подсети введите **азуребастионсубнет**.
      * Укажите диапазон адресов в нотации CIDR. Например, 10.1.254.0/27.
      * Нажмите кнопку **ОК** , чтобы создать подсеть. В верхней части страницы перейдите назад к бастиона, чтобы завершить остальные параметры.

         ![Перейдите к параметрам бастиона](./media/quickstart-host-portal/navigate-bastion.png)
   * **Общедоступный IP-адрес**: это общедоступный IP-адреса ресурса бастиона, к которому будут обращаться протоколы RDP/SSH (через порт 443). Создайте новый общедоступный IP-адрес или используйте существующий. Общедоступный IP-адрес должен находиться в том же регионе, что и создаваемый ресурс бастиона.
   * **Имя общедоступного IP-адреса**: имя ресурса общедоступного IP-адреса.
1. На экране проверка нажмите кнопку **создать**. Подождите около 5 минут для ресурса бастиона создайте и разверните.

   ![Создание узла бастиона](./media/quickstart-host-portal/bastion-settings.png)

## <a name="connect"></a>Соединиться

После развертывания бастиона в виртуальной сети экран переходит на страницу подключения.

1. Введите имя пользователя и пароль для виртуальной машины. Затем щелкните **Connect** (Подключиться).

   ![подключение](./media/quickstart-host-portal/connect.png)
1. Подключение RDP к этой виртуальной машине через бастиона будет открываться непосредственно в портал Azure (через HTML5) с помощью порта 443 и службы бастиона.

   ![Подключение RDP](./media/quickstart-host-portal/443-rdp.png)

## <a name="clean-up-resources"></a>Очистка ресурсов

После завершения работы с виртуальной сетью и виртуальными машинами удалите группу ресурсов и все содержащиеся в ней ресурсы.

1. Введите *TestRG1* в поле **поиска** в верхней части портала и выберите **TestRG1** в результатах поиска.

2. Выберите **Удалить группу ресурсов**.

3. Введите *TestRG1* **в поле введите имя группы ресурсов** и выберите **Удалить**.

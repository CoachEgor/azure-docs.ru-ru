# <a name="quickstart-connect-to-a-virtual-machine-using-a-private-ip-address-and-azure-bastion"></a>Быстрый запуск: Подключение к виртуальной машине с помощью частного IP-адреса и Azure Bastion

Эта статья quickstart показывает вам, как подключиться к виртуальной машине с помощью частного IP-адреса. При подключении через Бастион, ваши виртуальные машины не нуждаются в общественном IP-адресе. Шаги в этой статье помогут вам развернуть Bastion в виртуальной сети через виртуальную машину на портале. После предоставления услуги опыт RDP/SSH доступен для всех виртуальных машин в одной виртуальной сети.

## <a name="prerequisites"></a><a name="prereq"></a>Предварительные требования

* Виртуальная сеть Azure.
* Виртуальная машина Azure, расположенная в виртуальной сети с открытым портом 3389.

## <a name="create-a-bastion-host"></a><a name="createvmset"></a>Создание узла-бастиона

При создании узла бастиона на портале с помощью существующей виртуальной машины различные настройки автоматически по умолчанию соответствуют вашей виртуальной машине и/или виртуальной сети.

1. Откройте [портал Azure](https://portal.azure.com). Перейти к вашей виртуальной машине, а затем нажмите **Connect**.

   ![виртуальные настройки машины](./media/quickstart-host-portal/vm-settings.png)
1. Из выпадения, выберите **Бастион**.
1. На странице Connect выберите **«Бастион».**

   ![выбрать Бастион](./media/quickstart-host-portal/select-bastion.png)

1. На странице Бастион заполните следующие поля настроек:

   * **Имя**: Назовите бастион хост
   * **Subnet**: Подсеть внутри вашей виртуальной сети, к которой будет развернут ресурс Bastion. Подсеть должна быть создана под названием **AzureBastionSubnet**. Имя позволяет Azure знать, в какую подсеть развертывать ресурс «Бастиона». Это отличается от подсети Gateway. Используйте подсеть не менее /27 или больше (/27, /26, /25 и так далее).
   
      * Выберите **конфигурацию подсети Manage,** затем выберите **подсеть.**
      * На странице подсети Добавить введите **AzureBastionSubnet**.
      * Укажите диапазон адресов в обозначении CIDR. Например, 10.1.254.0/27.
      * Выберите **OK** для создания подсети. В верхней части страницы перейдите обратно в Бастион, чтобы завершить остальные настройки.

         ![перейти к настройкам бастиона](./media/quickstart-host-portal/navigate-bastion.png)
   * **Общественный IP-адрес**: Это общедоступный IP ресурса Bastion, на котором будет доступна RDP/SSH (над портом 443). Создайте новый общедоступный IP или используйте существующий IP.. Общественный IP-адрес должен находиться в том же регионе, что и создаваемый ресурсом Bastion.
   * **Имя публичного IP-адреса**: Название общедоступного ресурса IP-адреса.
1. На экране проверки нажмите **«Создать».** Подождите около 5 минут, чтобы ресурс «Бастиона» создал и развернен.

   ![создать бастион хост](./media/quickstart-host-portal/bastion-settings.png)

## <a name="connect"></a><a name="connect"></a>Подключения

После развертывания Bastion в виртуальной сети экран изменяется на страницу подключения.

1. Введите имя пользователя и пароль для виртуальной машины. Затем щелкните **Connect** (Подключиться).

   ![подключение](./media/quickstart-host-portal/connect.png)
1. Подключение RDP к этой виртуальной машине через Бастион откроется непосредственно на портале Azure (над HTML5) с использованием порта 443 и службы Бастиона.

   ![Подключение RDP](./media/quickstart-host-portal/443-rdp.png)

## <a name="clean-up-resources"></a>Очистка ресурсов

Когда вы закончите использовать виртуальную сеть и виртуальные машины, удалите группу ресурсов и все ресурсы, которые она содержит:

1. Введите *TestRG1* в поле **поиска** в верхней части портала и выберите **TestRG1** из результатов поиска.

2. Выберите **Удалить группу ресурсов**.

3. Введите *TestRG1* для **TYPE THE RESOURCE GROUP NAME** и выберите **Удалить**.

# <a name="working-with-nsg-access-and-azure-bastion-preview"></a>Работа с NSG Access и Azure бастиона (Предварительная версия)

При работе с Azure бастиона можно использовать группы безопасности сети (группы безопасности сети). Дополнительные сведения см. в разделе [группы безопасности](../virtual-network/security-overview.md). 

> [!IMPORTANT]
> Эта общедоступная предварительная версия предоставляется без соглашения об уровне обслуживания и не должна использоваться для производственных рабочих нагрузок. Некоторые функции могут не поддерживаться, иметь ограничения и быть доступными не во всех расположениях Azure. См. [дополнительные условия использования для предварительных версий Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).
>

![Архитектура](./media/bastion-nsg/nsg_architecture.png)

На этой схеме:

* Узел Бастиона развернут в виртуальной сети.
* Пользователь подключается к порталу Azure с помощью любого браузера HTML5.
* Чтобы подключиться, пользователь выбирает виртуальную машину.
* Одним щелчком мыши в браузере открывается сеанс RDP или SSH.
* На виртуальной машине Azure не требуется общедоступный IP-адрес.

## <a name="nsg"></a>Группы безопасности сети

* **Азуребастионсубнет:** Azure бастиона развертывается в определенном Азуребастионсубнет.  
    * **Входящий трафик из общедоступного Интернета:** Бастиона Azure создаст общедоступный IP-адрес, для которого в общедоступном IP-адресе входящего трафика должен быть включен порт 443. Порт 3389/22 не требуется открывать на Азуребастионсубнет.
    * **Исходящий трафик к целевым виртуальным машинам:** Azure бастиона будет обращаться к целевым виртуальным машинам через частный IP-адрес. Группы безопасности сети должен разрешить исходящий трафик в другие целевые подсети виртуальных машин.
* **Целевая подсеть виртуальной машины:** Это подсеть, содержащая целевую виртуальную машину, к которой необходимо подключиться по протоколу RDP или SSH.
    * **Входящий трафик из Azure бастиона:** Azure бастиона будет обращаться к целевой виртуальной машине через частный IP-адрес. Порты RDP/SSH (порты 3389 и 22 соответственно) должны быть открыты на стороне целевой виртуальной машины через частный IP-адрес.

## <a name="apply"></a>Применение группы безопасности сети к Азуребастионсубнет

Если вы примените группы безопасности сети к **азуребастионсубнет**, разрешите следующие два тега службы для плоскости управления и инфраструктуры Azure:

* **Гатевайманажер (только диспетчер ресурсов)** : Этот тег определяет префиксы адресов для службы "Диспетчер шлюзов Azure". При указании значения Гатевайманажер для параметра трафик разрешается или отклоняется для Гатевайманажер.  Если вы создаете группы безопасности сети на Азуребастионсубнет, включите тег Гатевайманажер для входящего трафика.

* **AzureCloud (только диспетчер ресурсов)** : Этот тег обозначает пространство IP-адресов для Azure, включая все общедоступные IP-адреса центра обработки данных. Если указать значение AzureCloud, трафик разрешается или отклоняется для общедоступных IP-адресов Azure. Если вы хотите разрешить доступ только к AzureCloud в определенном регионе, можно указать регион. Например, если вы хотите разрешить доступ только к Azure AzureCloud в регионе "Восточная часть США", можно указать AzureCloud. EastUS в качестве тега службы. Если вы создаете группы безопасности сети на Азуребастионсубнет, включите тег AzureCloud для исходящего трафика. Если вы откроете порт 443 для входящего трафика в Интернет, вам не нужно включать тег AzureCloud для трафика.

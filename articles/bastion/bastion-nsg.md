# <a name="working-with-nsg-access-and-azure-bastion-preview"></a>Работа с доступом к группе безопасности сети и Бастиона Azure (Предварительная версия)

При работе с Azure Бастиона, можно использовать группы безопасности сети (Nsg). Дополнительные сведения см. в разделе [группы безопасности](../virtual-network/security-overview.md). 

> [!IMPORTANT]
> Эта общедоступная предварительная версия предоставляется без соглашения об уровне обслуживания и не должна использоваться для производственных рабочих нагрузок. Некоторые функции могут не поддерживаться, иметь ограничения и быть доступными не во всех расположениях Azure. См. [дополнительные условия использования для предварительных версий Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).
>

![Архитектура](./media/bastion-nsg/nsg_architecture.png)

На этой схеме:

* Узел-Бастион развертывается в виртуальной сети.
* Пользователь подключается к порталу Azure с помощью любого браузера HTML5.
* Пользователь выбирает для подключения к виртуальной машине.
* Одним щелчком в браузере откроется сеанс удаленного рабочего СТОЛА или SSH.
* На виртуальной Машине Azure требуется без общедоступного IP-адреса.

## <a name="nsg"></a>Группы безопасности сети

* **AzureBastionSubnet:** В определенных AzureBastionSubnet развертывается Azure Бастиона.  
    * **Входящий трафик через Интернет.** Бастионы Azure будет создан общедоступный IP-адрес, который должен порта 443 включен общедоступный IP-адрес для входящего трафика. Порт 3389 и 22 соответственно не обязаны быть открыты в AzureBastionSubnet.
    * **Исходящий трафик к виртуальным машинам целевой объект:** Azure Бастиона достигнет целевых виртуальных машин через частный IP-адрес. Группы безопасности сети должны разрешать исходящий трафик в другие подсети целевой виртуальной Машины.
* **Подсеть целевой виртуальной Машины:** Это подсети, содержащей целевой виртуальной машины, которую требуется RDP или SSH, чтобы.
    * **Входящий трафик из Azure Бастиона:** Azure Бастиона достигнет для целевой виртуальной Машине через частный IP-адрес. Порты протокола удаленного рабочего СТОЛА или SSH (порта 3389 и 22 соответственно) должны быть открыты на целевой стороне виртуальной Машины через частный IP-адрес.

## <a name="apply"></a>Применить группы безопасности сети для AzureBastionSubnet

Если применить группы безопасности сети для **AzureBastionSubnet**, разрешить следующими двумя тегами службы для управления Azure и инфраструктуры:

* **(Только для Resource Manager) GatewayManager**: Этот тег определяет префиксы адресов для службы "Диспетчер шлюзов Azure". Если указано значение GatewayManager, трафик разрешен или запрещен для GatewayManager.

* **AzureCloud (только для Resource Manager)** : Этот тег обозначает пространство IP-адресов для Azure, включая все общедоступные IP-адреса центра обработки данных. Если указано значение AzureCloud, трафик разрешен или запрещен в Azure общедоступные IP-адреса. Если вы только хотите разрешить доступ к AzureCloud в определенном регионе, можно указать этот регион. Например если вы хотите разрешить доступ только к AzureCloud Azure в восточной части США, можно указать AzureCloud.EastUS как тег службы.

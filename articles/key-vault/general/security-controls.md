# <a name="security-controls-for-azure-key-vault"></a>Элементы управления безопасностью для Хранилища ключей Azure

В этой статье документируются элементы управления безопасностью, встроенные в Хранилище ключей Azure. 

[!INCLUDE [Security controls Header](../../../includes/security-controls-header.md)]

## <a name="network"></a>Сеть

| Контроль безопасности | Да/нет | Примечания |
|---|---|--|
| Поддержка конечных точек обслуживания| Да | Использование конечных точек службы Virtual Network (VNet). |
| Поддержка инъекций VNet| нет |  |
| Поддержка сетевой изоляции и брандмауэра| Да | Использование правил брандмауэра VNet. |
| Принудительная поддержка туннелирования| нет |  |

## <a name="monitoring--logging"></a>Мониторинг & лесозаготовки

| Контроль безопасности | Да/нет | Примечания|
|---|---|--|
| Поддержка мониторинга Azure (аналитика журналов, информация о приложениях и т.д.)| Да | С использованием Log Analytics. |
| Ведение журнала и аудит плоскости управления и контроля| Да | С использованием Log Analytics. |
| Регистрация и аудит плоскости данных| Да | С использованием Log Analytics. |

## <a name="identity"></a>Идентификация

| Контроль безопасности | Да/нет | Примечания|
|---|---|--|
| Аутентификация| Да | Аутентификация выполняется с помощью Azure Active Directory. |
| Авторизация| Да | С использованием политики доступа к хранилищу Key Vault. |

## <a name="data-protection"></a>Защита данных

| Контроль безопасности | Да/нет | Примечания |
|---|---|--|
| Шифрование на стороне сервера в состоянии покоя: клавиши, управляемые Корпорацией Майкрософт | Да | Будут шифроваться все объекты. |
| Шифрование на стороне сервера в состоянии покоя: ключи, управляемые клиентом (BYOK) | Да | Клиент управляет всеми ключами в своем Ключе Vault. Когда указаны клавиши с поддержкой аппаратной безопасности (HSM), HSM уровня FIPS 2 защищает ключ, сертификат или секрет. |
| Шифрование уровня столбцов (Службы данных Azure)| Недоступно |  |
| Шифрование в пути (например, шифрование ExpressRoute, шифрование VNet и шифрование VNet-VNet)| Да | Вся связь осуществляется через зашифрованные вызовы API |
| Вызовы API в зашифрованном виде| Да | С использованием HTTPS. |

## <a name="access-controls"></a>Элементы управления доступом

| Контроль безопасности | Да/нет | Примечания|
|---|---|--|
| Элементы управления доступом в плоскости управления | Да | Управление доступом на основе ролей Azure Active Directory (RBAC) |
| Элементы управления доступом в плоскости данных (на каждом уровне обслуживания) | Да | Политика доступа к Key Vault |

# <a name="best-practices-to-use-key-vault"></a>Рекомендации по использованию Key Vault

## <a name="control-access-to-your-vault"></a>Контроль доступа к хранилищу

Azure Key Vault — это облачная служба, которая обеспечивает защиту ключей шифрования и секретов (например, сертификатов, строк подключения и паролей). Так как это критически важные для бизнеса конфиденциальные данные, следует обеспечить защиту доступа к хранилищу ключей, чтобы доступ могли получить только авторизованные приложения и пользователи. В [этой статье](secure-your-key-vault.md)представлен обзор модели доступа Key Vault. Здесь приводится описание процессов аутентификации и авторизации, а также содержатся сведения о том, как защитить доступ к вашему хранилищу ключей.

Предложения при контроле доступа к хранилищу следующие:
1. Блокировка доступа к подписке, группе ресурсов и ключевым убежищам (RBAC)
2. Создание политик доступа для каждого хранилища
3. Используйте наименьший принцип доступа к привилегиям для предоставления доступа
4. Включите брандмауэр и [конечные точки обслуживания VNET)](overview-vnet-service-endpoints.md)

## <a name="use-separate-key-vault"></a>Используйте отдельное убежище ключей

Наша рекомендация заключается в использовании хранилища для одной среды (разработка, предпроизводство и производство). Это поможет вам не делиться секретами в разных средах, а также уменьшает угрозу в случае нарушения.

## <a name="backup"></a>Резервное копирование

Убедитесь, что вы регулярно резервного копирования вашего [хранилища](https://blogs.technet.microsoft.com/kv/2018/07/20/announcing-backup-and-restore-of-keys-secrets-and-certificates/) на обновление / удаление / создание объектов в Убежище.

[Включите журнал )](logging.md) для вашего Убежища. Также настраиваем оповещения.

## <a name="turn-on-recovery-options"></a>Включить параметры восстановления

1. Включите [Soft Delete](overview-soft-delete.md)).
2. Включите защиту чистки, если вы хотите защититься от силового удаления секрета / свода даже после того, как мягкое удаление включено.

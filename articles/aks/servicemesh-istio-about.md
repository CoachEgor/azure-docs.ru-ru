# <a name="istio"></a>Истио

## <a name="overview"></a>Обзор

[Istio][istio] является полнофункциональным, настраиваемым и расширяемым сервисной сеткой.

## <a name="architecture"></a>Architecture

Istio предоставляет плоскость данных, которая состоит из наосновенных [в envoy][envoy-proxy]колясках. Эти интеллектуальные прокси контролируют весь сетевой трафик в и из ваших сетчатых приложений и рабочих нагрузок.

Плоскость управления управляет конфигурацией, политикой и телеметрией через следующие [компоненты:][what-is-istio]

- **Mixer** - Обеспечивает соблюдение политик управления доступом и использования. Собирает телеметрию от прокси, которые толкаемых в [Prometheus][prometheus].

- **Пилотный проект** - Обеспечивает поиск службы и политику управления трафиком/конфигурацию для прокси-сервисов.

- **Цитадель** - Обеспечивает возможности идентификации и безопасности, которые позволяют mTLS между службами.

- **Galley** - Аннотации и обеспечивает конфигурацию компонентов.

Следующая диаграмма архитектуры показывает, как взаимодействуют различные компоненты в плоскости данных и плоскости управления.

## <a name="selection-criteria"></a>Критерии отбора

При оценке Istio для рабочих нагрузок важно понимать и учитывать следующие области:

- [Цели проектирования](#design-goals)
- [Возможности](#capabilities)
- [Сценарии](#scenarios)


### <a name="design-goals"></a>Цели проектирования

Следующие цели проектирования [направляют][design-goals] проект Istio:

- **Максимальная прозрачность** - Разрешить принятие с минимальным объемом работы, чтобы получить реальную ценность от системы.

- **Расширяемость** - должна быть в состоянии расти и адаптироваться с меняющимися потребностями.

- **Портативность** - Запуск легко в различных видах сред - облако, на месте.

- **Единообразие политики** - последовательность в определении политики в различных ресурсах.


### <a name="capabilities"></a>Возможности

Istio предоставляет следующий набор возможностей:

- **Сетка** - шлюзы (многокластер), виртуальные машины (расширение сетки)

- **Управление трафиком** - маршрутирование, расщепление, тайм-ауты, выключатели, повторы, вход, выход

- **Политика** - контроль доступа, ограничение скорости, квота, адаптеры пользовательской политики

- **Безопасность** – аутентификация (jwt), авторизация, шифрование (mTLS), внешний CA (HashiCorp Vault)

- **Наблюдение** - золотые метрики, зеркало, трассировка, пользовательские адаптеры, прометей, графана

### <a name="scenarios"></a>Сценарии

Istio хорошо подходит и предлагается для следующих сценариев:

- Требуется расширяемость и богатый набор возможностей

- Расширение сетки с учетом рабочих нагрузок на основе VM

- Многокластерная сервисная сетка

# <a name="azure-kubernetes-service-diagnostics-preview-overview"></a>Обзор диагностики службы Azure Kubernetes (предварительный просмотр)

Проблемы, связанные с проблемой обслуживания Azure Kubernetes Service (AKS), являются важной частью поддержания кластера, особенно если кластер работает критически важные рабочие нагрузки. AKS Diagnostics — это интеллектуальный, самодиагностический опыт, который помогает выявлять и решать проблемы в кластере. AKS Diagnostics является облачным, и вы можете использовать ее без дополнительной конфигурации или стоимости выставления счетов.

Эта функция в настоящее время в открытом доступе.

## <a name="open-aks-diagnostics"></a>Открытая диагностика AKS

Чтобы получить доступ к диагностике AKS:

- Перейдите к кластеру Kubernetes на [портале Azure.](https://portal.azure.com)
- Нажмите на **диагностику и решить проблемы** в левой навигации, которая открывает AKS Диагностика.
- Выберите категорию, которая наилучшим образом описывает проблему кластера, используя ключевые слова в плитке главной страницы, или введите ключевое слово, которое лучше всего описывает вашу проблему в панели поиска, например _проблемы кластерного узла._

![Домашняя страница](./media/concepts-diagnostics/aks-diagnostics-homepage.png)

## <a name="view-a-diagnostic-report"></a>Просмотр диагностического отчета

После нажатия на категорию можно просмотреть диагностический отчет, специфичный для кластера. Диагностический отчет разумно вызывает, если есть какие-либо проблемы в кластере с иконками статуса. Вы можете просверлить вниз по каждой теме, нажав на **Подробную информацию,** чтобы увидеть подробное описание проблемы, рекомендуемые действия, ссылки на полезные документы, связанные метрики, и данные о регистрации. Диагностические отчеты разумно генерируются в зависимости от текущего состояния кластера после выполнения различных проверок. Диагностические отчеты могут быть полезным инструментом для определения проблемы кластера и поиска следующих шагов для решения проблемы.

## <a name="cluster-insights"></a>КластерНые исследования

Следующие диагностические **проверки**доступны в кластере Исследования .

### <a name="cluster-node-issues"></a>Проблемы кластерного узла

Проблемы кластерного узла проверяются на наличие проблем, связанных с узлами, которые могут привести к неожиданному сеточному заболеванию.

- Проблемы готовности узлов
- Сбои узлов
- Недостаточно ресурсов
- Отсутствует конфигурация IP-адреса
- Сбои узлов CNI
- Узел не найден
- Отключение узла
- Сбой проверки подлинности узлов
- Узла кубе-прокси устаревшей

### <a name="create-read-update--delete-operations"></a>Создание, чтение, обновление & удалению операций

Операции CRUD проверяют все операции CRUD, которые могут вызвать проблемы в кластере.

- Ошибка удаления операции в использовании
- Группа сетевой безопасности удаляет ошибку операции
- Ошибка операции в таблице маршрутов в использовании
- Референтная ошибка подготовки ресурсов
- Ошибка удаления операции IP-адреса
- Сбой развертывания из-за квоты развертывания
- Ошибка операции из-за политики организации
- Отсутствует регистрация подписки
- Ошибка подготовки расширения VM
- Емкость подсети
- Квота превышена погрешность

### <a name="identity-and-security-management"></a>Управление идентификаторами и безопасностью

Identity and Security Management обнаруживает ошибки проверки подлинности и авторизации, которые могут помешать связи с кластером.

- Сбои авторизации узлов
- 401 ошибка
- Ошибки 403

# <a name="status-of-migration-scenarios-supported-by-azure-database-migration-service"></a>Сценарии миграции, поддерживаемые миграционной службой базы данных Azure

Миграционная служба базы данных Azure предназначена для поддержки различных сценариев миграции (источников/целевых пар) как для автономных (одноразовых) и онлайновых (непрерывная синхронизация) миграций. Сценарий охвата, предоставляемый Службой миграции базы данных Azure, расширяется с течением времени. Регулярно добавляются новые сценарии. В этой статье определены сценарии миграции, поддерживаемые в настоящее время миграционной службой базы данных Azure, и статус (частный предварительный просмотр, публичный предварительный просмотр или общая доступность) для каждого сценария.

## <a name="offline-versus-online-migrations"></a>Автономная миграция и миграция с подключением к сети

С помощью службы миграции базы данных Azure вы можете выполнять в автономном режиме или миграцию в Интернете. При использовании *автономной* миграции простой приложения начинается в то же время, что и миграция. Чтобы ограничить время простоя временем, необходимым для переноса в новую среду по завершении миграции, используйте *онлайн-миграцию.* Рекомендуется протестировать миграцию в автономном режиме, чтобы определить, является ли время простоя приемлемым; если нет, сделайте онлайн-миграцию.

## <a name="migration-scenario-status"></a>Состояние сценария миграции

Статус сценариев миграции, поддерживаемых миграционной службой базы данных Azure, меняется с течением времени. Как правило, сценарии впервые публикуются в **закрытом предварительном просмотре.** Участие в частном предварительном просмотре требует от клиентов представить номинацию через [сайт DMS Preview.](https://aka.ms/dms-preview) После предварительного просмотра приватного сценария статус сценария изменяется в **общедоступный предварительный просмотр.** Пользователи службы миграции базы данных Azure могут опробовать сценарии миграции в общедоступном предварительном просмотре непосредственно из пользовательского интерфейса. Регистрация не требуется.  Однако сценарии миграции в общедоступном предварительном просмотре могут быть доступны не во всех регионах и могут претерпеть дополнительные изменения до окончательного выпуска. После публичного предварительного просмотра статус сценария изменяется в **целом в наличии**. Общая доступность (GA) является окончательным статусом выпуска, а функциональность является полной и доступной для всех пользователей.

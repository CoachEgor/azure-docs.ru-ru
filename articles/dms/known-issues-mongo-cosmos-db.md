# <a name="known-issuesmigration-limitations-with-migrations-from-mongodb-to-azure-cosmos-dbs-api-for-mongodb"></a>Известные проблемы/ограничения миграции с миграцией из MongoDB в API API Azure Cosmos DB для MongoDB

Известные проблемы и ограничения, связанные с миграцией из MongoDB в API Cosmos DB для MongoDB, описаны в следующих разделах.

## <a name="migration-fails-as-a-result-of-using-the-incorrect-ssl-cert"></a>Миграция не удается в результате использования неправильного SSL Cert

* **Симптом**: Эта проблема очевидна, когда пользователь не может подключиться к исходного сервера MongoDB. Несмотря на открытие всех портов брандмауэра, пользователь по-прежнему не может подключиться.

| Причина         | Решение |
| ------------- | ------------- |
| Использование самоподписанного сертификата в миграционной службе базы данных Azure может привести к сбою в миграции из-за неправильного SSL Cert. Сообщение об ошибке может включать в себя "Удаленный сертификат недействителен в соответствии с процедурой проверки". | Используйте подлинный сертификат от CA.  Самоподписанные сертификаты, как правило, используются только во внутренних тестах. При установке подлинного сертификата от органа CA можно использовать SSL в службе миграции базы данных Azure без проблем (соединения с Cosmos DB используют SSL над API Mongo).<br><br> |

## <a name="unable-to-get-the-list-of-databases-to-map-in-dms"></a>Невозможно получить список баз данных для картирования в DMS

* **Симптом**: Невозможно получить список DB на лезвии **настройки базы данных** при **использовании данных из** режима хранения Azure на лезвии **исходного кода Select.**

| Причина         | Решение |
| ------------- | ------------- |
| Строка подключения учетной записи хранилища отсутствует информация SAS и, таким образом, не может быть проверена. | Создайте SAS на контейнере с каплями в Storage Explorer и используйте URL-адрес с информацией о контейнере SAS в качестве строки подключения к деталям источника.<br><br> |

## <a name="using-an-unsupported-version-of-the-database"></a>Использование неподдерживаемой версии базы данных

* **Симптом**: Миграция терпит неудачу.

| Причина         | Решение |
| ------------- | ------------- |
| Попытка перехода в Azure Cosmos DB из неподдерживаемой версии MongoDB может быть перенесена на Azure Cosmos DB. | По мере выпуска новых версий MongoDB они тестируются на соответствие требованиям службы миграции базы данных Azure, и служба периодически обновляется для приема последней версии(ы). При найме срочной необходимости миграция может быть сделана в качестве обходного пути, вы можете экспортировать базы данных/коллекции в Хранилище Azure и точку источника в результирующую дампу. Создайте SAS на контейнере blob в Storage Explorer, а затем используйте URL-адрес с информацией о контейнере SAS в качестве строки подключения детали источника.<br><br> |

## <a name="azure-maps-supported-views"></a>Карты Azure поддерживали представления

> [!Note]
> 1 августа 2019 года Azure Maps была выпущена в следующих странах/регионах:
>  * Аргентина
>  * Индия
>  * Марокко
>  * Пакистан
>
> После 1 августа 2019 года параметр **View** определит возвращаемый контент карты для новых регионов/стран, перечисленных выше. Параметр Azure Maps **View** (также называемый «параметром региона пользователя») представляет собой двухбуквенное код страны ISO-3166, в котором будут отображаться правильные карты для этой страны/региона с указанием того, какой набор геополитически спорного содержимого возвращается через службы Azure Maps, включая границы и метки, отображаемые на карте. 

Убедитесь, что вы настроили параметр **Представления,** необходимый для APIs REST и SDKs, которые используются вашими службами.
>  
>
>  **АПИ отдыха:**
>  
>  Убедитесь, что вы настроили параметр представления по мере необходимости. Параметры представления определяют, какой набор геополитически спорного содержимого возвращается через службы Azure Maps. 
>
>  Пострадавшие услуги Azure Maps REST:
>    
>    * Получить карта плитки
>    * Получить изображение карты 
>    * Получить поиск нечетких
>    * Получить поиск POI
>    * Получить поиск POI Категория
>    * Получить поиск поблизости
>    * Получить адрес поиска
>    * Структурированный поисковый адрес
>    * Получение обратного адреса поиска
>    * Получить поиск адрес Обратный Крест-стрит
>    * Search — Post Search Inside Geometry (Поиск — запрос Post на поиск внутри геометрии)
>    * Опубликовать Поиск Адрес Пакет Предварительный
>    * Опубликовать Поиск Адрес Обратный пакет Предварительный просмотр
>    * Опубликовать Поиск по маршруту
>    * Опубликовать Поиск нечетких пакет Предварительный
>
>    
>  **Пакеты sdk:**
>
>  Убедитесь, что вы настроили параметр **View** по мере необходимости, и у вас есть последняя версия Web SDK и Android SDK. Пострадавшие SDK:
>
>    * Веб-пакет SDK Azure Maps
>    * Пакет SDK для Android в Azure Maps

По умолчанию параметр представления настроен на **unified,** даже если вы не определили его в запросе. Определите местоположение пользователей. Затем установите параметр **представления** правильно для этого местоположения. Кроме того, можно настроить "View-Auto", который вернет данные карты на основе IP-адреса запроса.  Параметр **представления** в Azure Maps должен использоваться в соответствии с применимым законодательством, включая законы о картографировании страны, где доступны карты, изображения и другие данные и сторонний контент, к которым вы имеете доступ через Azure Maps.

## <a name="azure-maps-supported-views"></a>Карты Azure поддерживается представления

> [!Note]
> Мы выпускаем "карты Azure" в следующих странах и регионах на 1 августа 2019 г.:
>  * Аргентина
>  * Индия
>  * Марокко
>  * Пакистан
>
> После 1 августа 2019 г. **представление** значения параметра будут определять содержимого карты, возвращенный для новых регионов или стран перечисленных выше. Мы рекомендуем вам, чтобы убедиться, что вы настроили параметр View, при необходимости для REST API и пакеты SDK, которые при использовании ваших служб.
>  
>
>  **REST API:**
>  
>  Убедитесь, что вы настроили параметр View, при необходимости. Параметр View указывает, какой набор геополитически спорных содержимое возвращается с помощью службы Azure Maps. 
>
>  Затронутые Azure Maps REST службы:
>    
>    * Получить плитку карты
>    * Получить изображение карты 
>    * Get поиска нечетких соответствий
>    * Получите поиск ДОСТОПРИМЕЧАТЕЛЬНОСТИ
>    * Получить категории POI поиска
>    * Получение близлежащих поиска
>    * Получить адрес поиска
>    * Получить адрес поиска структурированного
>    * Получение обратного поиска адреса
>    * Получить поиска адрес обратного пересекающей улицы
>    * POST поиска внутри геометрического объекта
>    * Предварительную версию пакета POST поиска адреса
>    * Опубликовать предварительную версию поиска адрес обратного пакетной службы
>    * POST поиска маршрута
>    * Предварительная версия пакета нечеткого поиска POST
>
>    
>  **Пакеты SDK:**
>
>  Убедитесь, что вы настроили параметр View, при необходимости, и у вас есть последнюю версию веб-пакета SDK и пакет SDK для Android. Затронутые пакеты SDK:
>
>    * Веб-пакет SDK Azure Maps
>    * Пакет SDK для Android в Azure Maps


Azure Maps **представление** параметр (называемый также «параметр области пользователя») является две буквы — код страны ISO 3166, отображающий правильный карты для этой страны или региона, указав, какой набор геополитически спорные содержимое возвращается с помощью службы Azure Maps, включая границы и меток, отображаемых на карте. 

По умолчанию является значение параметра View **единой**, даже если она еще не определена в запросе. Это необходимо определить местонахождение пользователей, а затем установите параметр View правильно к этому расположению. Кроме того, вы можете задать "представление = Auto", который возвращает данные карты на основании IP-адрес запроса.  Параметр View в Azure сопоставляет должен использоваться в соответствии с применимым законам, включая те, о сопоставление страны, где карты, изображения и другое содержимое данных и сторонних поставщиков, права на доступ к через Azure сопоставляет становится доступным.

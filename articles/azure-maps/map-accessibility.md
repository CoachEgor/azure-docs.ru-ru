# <a name="building-an-accessible-application"></a>Создание доступного приложения

Более 20% пользователей Интернета должны иметь доступ к доступным веб-приложениям. Поэтому важно убедиться, что приложение разработано таким образом, что любой пользователь может легко его использовать. Вместо того, чтобы думать о специальных возможностях в качестве набора задач, подумайте об этом как часть общего пользовательского интерфейса. Чем больше доступное приложение, тем больше пользователей оно сможет его использовать. 

Когда речь идет о форматированном интерактивном содержимом, например на карте, некоторые общие рекомендации по специальным возможностям:
- Поддержка средства чтения с экрана для пользователей, которые испытывают трудности при просмотре веб-приложения.
- Наличие нескольких методов для взаимодействия с веб-приложением, например мыши, сенсорного экрана и клавиатуры, и перехода к ним.
- Убедитесь, что цветовая контрастность не смешивается друг с другом и становится трудной для различения. 

Веб-пакет SDK для Azure Maps включает в себя множество специальных возможностей, таких как:
- Описание средства чтения с экрана, когда схема перемещается и когда пользователь фокусируется на элементе управления или всплывающем окне.
- Поддержка мыши, сенсорного ввода и клавиатуры.
- Доступная поддержка цветовой контрастности в стиле схемы дороги.

Полные сведения о согласованности специальных возможностей для всех продуктов Майкрософт можно найти [здесь](https://cloudblogs.microsoft.com/industry-blog/government/2018/09/11/accessibility-conformance-reports/). Выполните поиск по запросу "Azure Maps Web", чтобы найти документ специально для Azure Maps веб-пакета SDK. 

## <a name="navigating-the-map"></a>Навигация по карте
Существует несколько различных способов, с помощью которых можно увеличить, сдвигаемых, повернуть и наклонить карту. Ниже приведены все различные способы перемещения по карте.

**Изменить масштаб схемы**
- С помощью мыши дважды щелкните карту, чтобы увеличить масштаб на один уровень.
- С помощью мыши Прокрутите колесо колесика, чтобы увеличить карту.
- С помощью сенсорного экрана вы наведитее карту двумя пальцами и сжимаете вместе, чтобы уменьшить или распределить пальцы, чтобы увеличить масштаб.
- С помощью сенсорного экрана дважды коснитесь гиперкарты, чтобы увеличить масштаб на один уровень.
- Для масштабирования на один уровень используйте знак плюса (`+`) или знак равенства (`=`).
- Для масштабирования на один уровень используйте знак "минус", дефис (`-`) или символ подчеркивания (`_`).
- Использование элемента управления Zoom с мышью, клавишей TAB или вводом клавиш.
- Нажмите и удерживайте кнопку `Shift` и нажмите левую кнопку мыши на карте и перетащите, чтобы нарисовать область, чтобы изменить масштаб отображения.

**Панорамирование схемы**
- С помощью мыши нажмите левую кнопку мыши на карте и перетащите в любом направлении.
- С помощью сенсорного экрана нажимайте карту и перетаскивайте ее в любом направлении.
- С помощью кнопки со стрелками на карте переместите карту.

**Поворот схемы**
- С помощью мыши нажмите на карту правой кнопкой мыши и перетащите влево или вправо. 
- С помощью сенсорного экрана нажимайте на карту двумя пальцами и поверните.
- Используя карту, используйте клавишу Shift и клавиши со стрелками влево или вправо.
- Использование элемента управления вращением с помощью мыши, клавиши TAB или клавишей Ввод.

**Шаг на карте**
- С помощью мыши нажмите на карту правой кнопкой мыши и перетащите указатель вверх или вниз. 
- С помощью сенсорного экрана наведите указатель мыши на карту двумя пальцами и перетащите их вверх или вниз вместе.
- С помощью кнопки Shift, клавиши со стрелками вверх или вниз. 
- Использование элемента управления "шаг" с мышью, клавишей TAB или клавишей Ввод.

**Изменение стиля схемы** Не все разработчики хотят, чтобы все возможные стили карт были доступны в приложении. Разработчик может программно задавать и изменять стиль карт по своему усмотрению. Если разработчик отображает элемент управления Выбор стиля карт, пользователь сможет изменить стиль карт с помощью мыши, касания или клавиатуры, используя клавиши TAB и ввод. Разработчик может указать, какие стили карт нужно сделать доступными в элементе управления Выбор стиля карт. 

## <a name="keyboard-shortcuts"></a>Сочетания клавиш

На карте имеется несколько встроенных сочетаний клавиш, которые упрощают использование этой схемы. Эти сочетания клавиш работают, когда на карте установлен фокус.

| Ключ      | Действия                            |
|----------|-----------------------------------|
| `Tab` | Навигация по элементам управления и всплывающим окнам на карте. |
| `ESC` | Переместите фокус с любого элемента на карте в элемент карт верхнего уровня. |
| `Ctrl` + `Shift` + `D` | Переключить уровень детализации средства чтения с экрана.  |
| Клавиша со стрелкой влево | Сдвиг схемы влево на 100 пикселей |
| Клавиша со стрелкой вправо | Сдвиг карт справа 100 пикселей |
| Клавиша со стрелкой вниз | Панорамирование по карте вниз на 100 пикселей |
| Клавиша со стрелкой вверх | Панорамирование карт 100 пикселей |
| `Shift` + стрелка вверх | Увеличение высоты карт на 10 градусов |
| `Shift` + стрелка вниз | Уменьшить шаг на карте на 10 градусов |
| `Shift` + стрелка вправо | Поворот карт на 15 градусов по часовой стрелке |
| `Shift` + стрелка влево | Поворот карт на 15 градусов против часовой стрелки |
| Знак плюса (`+`) или <sup>*</sup>знак равенства (`=`) | Увеличить масштаб |
| Знак минус, дефис (`-`) или <sup>*</sup>символ подчеркивания (`_`) | Уменьшить масштаб | 
| `Shift` + перетаскивание на карте для рисования области | Увеличить область |

<sup>*</sup> Эти ключевые сочетания клавиш обычно используют один и тот же ключ на клавиатуре. Эти сочетания клавиш были добавлены для улучшения взаимодействия с пользователем. Это также не имеет значения, если пользователь использует клавишу Shift или не используется для этих сочетаний клавиш.

## <a name="screen-reader-support"></a>Поддержка средства чтения с экрана

Пользователи могут перемещаться по карте с помощью клавиатуры. Если запущено устройство чтения с экрана, карта будет уведомлять пользователя об изменениях в ее состоянии. Например, пользователи получают уведомления об изменениях карты при панорамировании или изменении масштаба. По умолчанию на карте представлены упрощенные описания, которые исключают уровень масштабирования и координаты центра на карте. Пользователь может переключить уровень детализации для этих описаний с помощью сочетания клавиш короткая вырезание `Ctrl` + `Shift` + `D`.

Дополнительные сведения, размещаемые на базовой карте, должны содержать соответствующую текстовую информацию для пользователей устройства чтения с экрана. При необходимости обязательно добавьте атрибуты ARIA, Alt и title [со специальными](https://www.w3.org/WAI/standards-guidelines/aria/)возможностями для Интернета. 

## <a name="make-popups-keyboard-accessible"></a>Сделать всплывающие окна доступными для клавиатуры

Маркер или символ часто используется для представления расположения на карте. Дополнительные сведения о расположении обычно отображаются во всплывающем окне, когда пользователь взаимодействует с маркером. В большинстве приложений всплывающие окна отображаются, когда пользователь щелкает или касанием маркера, однако для этого события требуется, чтобы пользователь использовал мышь или сенсорный экран. Рекомендуется сделать всплывающие окна доступными при использовании клавиатуры. Эта возможность может быть достигнута путем создания всплывающего окна для каждой точки данных и добавления их к карте. 

В следующем примере загружаются точки интересов на карте с помощью слоя символов и добавляется всплывающее окно на карту для каждой интересующей точки. Ссылка на каждое всплывающее окно хранится в свойствах каждой точки данных. Его также можно получить для маркера, например при щелчке маркера. Когда вы набираете карту, нажатие клавиши Tab позволит пользователю проходить через каждое всплывающее окно на карте.

## <a name="additional-accessibility-tips"></a>Дополнительные советы по специальным возможностям

Ниже приведены некоторые дополнительные советы по обеспечению доступности приложения веб-сопоставления.

- При отображении множества данных интерактивной точки на карте рекомендуется уменьшить помехи и использовать кластеризацию. 
- Убедитесь, что коэффициент контрастности между текстом и символами и фоновыми цветами равен 4.5:1 или более.
- Обеспечьте короткие, описательные и осмысленные сообщения для программы чтения с экрана (атрибуты ARIA, Alt и Title). Избегайте ненужных жаргоне и акронимов.
- Постарайтесь оптимизировать сообщения, отправляемые средству чтения с экрана, чтобы предоставить коротко информативную информацию, которая легко может быть дайджестом. Например, если вы хотите обновить средство чтения с экрана с высокой частотой, например при перемещении схемы, рассмотрите возможность выполнения следующих пунктов.
    - Дождитесь перемещения схемы для обновления средства чтения с экрана.
    - Регулирование обновлений каждые несколько секунд. 
    - Логическое объединение сообщений. 
- Избегайте использования цвета в качестве единственного средства передачи информации. Используйте текст, значки или узоры для дополнения или замены цвета. Некоторые замечания:
    - При использовании пузырькового слоя для отображения относительного значения точек данных рассмотрите возможность масштабирования радиуса для каждого пузырька в дополнение к или в качестве альтернативы их цвету. 
    - Рассмотрите возможность использования слоя символов с разными значками для различных категорий метрик, таких как треугольники, звезды и квадраты. Слой символов также поддерживает масштабирование размера значка. Также может отображаться текстовая метка.
    - При отображении данных линии ширина может быть использована для представления веса или размера. Для представления различных категорий линий можно использовать шаблон пунктирного массива. Слой символов можно использовать в сочетании со строкой для наложения значков вдоль линии. Значок со стрелкой удобно использовать для отображения последовательности или направления линии.
    - При отображении данных многоугольника шаблон, например полосы, можно использовать в качестве альтернативы цвету. 
- Некоторые визуализации, такие как тепловые карты, мозаичные слои и слои изображений, недоступны для пользователей с нарушениями зрения. Некоторые замечания:
    - Программа чтения с экрана описывает, что отображает слой при добавлении на карту. Например, если отображается мозаичный слой Weather, то средство чтения с экрана может сказать нечто вроде "данные прогноза погоды, наложенные на карту".
- Ограничьте количество функций, требующих наведения указателя мыши. Эти функции будут недоступны пользователям, использующим клавиатуру или сенсорное устройство для взаимодействия с приложением. Обратите внимание, что по-прежнему рекомендуется иметь стиль наведения для интерактивного содержимого, например, значки, ссылки и кнопки.
- Попробуйте перемещаться по приложению с помощью клавиатуры. Убедитесь, что порядок табуляции является логическим.
- При создании сочетаний клавиш попробуйте ограничить их двумя ключами или меньше.

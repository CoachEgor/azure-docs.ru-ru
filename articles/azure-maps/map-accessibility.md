# <a name="building-an-accessible-application"></a>Создание доступного приложения

Более 20% интернет-пользователей нуждаются в доступных веб-приложениях. Таким образом, важно, чтобы убедиться, что ваше приложение разработано таким образом, что любой пользователь может легко использовать его. Вместо того, чтобы думать о доступности как набор задач для завершения, думать о нем, как часть вашего общего пользовательского опыта. Чем доступнее ваше приложение, тем больше людей могут им пользоваться. 

Когда дело доходит до богатого интерактивного контента, как карта, некоторые общие соображения доступности являются:
- Поддержка чтения с экрана для пользователей, которые испытывают трудности с просмотром веб-приложения.
- Есть несколько методов для взаимодействия и навигации веб-приложений, таких как мышь, сенсорный и клавиатура.
- Убедитесь, что цветовой контраст таков, что цвета не смешиваются вместе и становятся трудно отличить друг от друга. 

Azure Maps Web SDK поставляется с множеством функций доступности, таких как:
- Описание чтения экрана, когда карта движется и когда пользователь фокусируется на элементе управления или всплывающем окне.
- Поддержка мыши, касания и клавиатуры.
- Доступная поддержка цветовой контрастной в стиле «дорожной карты».

Полную доступность соответствия детали для всех продуктов Microsoft можно найти [здесь](https://cloudblogs.microsoft.com/industry-blog/government/2018/09/11/accessibility-conformance-reports/). Поиск "Azure Maps веб", чтобы найти документ специально для Azure Maps Web SDK. 

## <a name="navigating-the-map"></a>Навигация по карте

Существует несколько различных способов масштабирования, панорамирования, вращения и разобрана карты. Ниже описаны все различные способы навигации по карте.

**Увеличить карту**

- С помощью мыши, дважды нажмите на карту, чтобы увеличить на один уровень.
- С помощью мыши прокрутите колесо, чтобы увеличить карту.
- Используя сенсорный экран, коснитесь карты двумя пальцами и прищипйте вместе, чтобы увеличить или распространить пальцы друг от друга, чтобы увеличить.
- Используя сенсорный экран, дважды коснитесь карты, чтобы увеличить на одном уровне.
- С помощью карты, ориентированной, используйте знак Плюс ()`+`или Знак Равные ()`=`для увеличения на одном уровне.
- С помощью карты сосредоточены, используйте`-`знак Минус,`_`Hyphen ( ), или Underscore ( ) увеличить один уровень.
- Использование управления масштабированием с помощью клавиш мыши, касания или клавиатуры/ ввода.
- Нажмите и `Shift` удерживайте кнопку и нажмите левую кнопку мыши вниз на карте и перетащите, чтобы вытянуть область, чтобы увеличить карту в.

**Панорамирование карты**

- С помощью мыши нажмите вниз с левой кнопкой мыши на карте и перетащите в любом направлении.
- Используя сенсорный экран, коснитесь карты и перетащите в любом направлении.
- С помощью карты сосредоточены, используйте клавиши стрелки для перемещения карты.

**Поверните карту**

- С помощью мыши нажмите вниз с правой кнопкой мыши на карте и перетащите влево или вправо. 
- Используя сенсорный экран, коснитесь карты двумя пальцами и поверните.
- С помощью карты, ориентированной, используйте клавишу смены и левые или правые клавиши стрелки.
- Использование управления вращением с помощью клавиш мыши, касания или клавиатуры/ ввода.

**Шаг карту**

- Используя мышь, нажмите вниз с правой кнопкой мыши на карте и перетащите вверх или вниз. 
- Используя сенсорный экран, коснитесь карты двумя пальцами и перетащите их вверх или вниз вместе.
- С помощью карты сосредоточены, используйте ключ смены плюс вверх или вниз клавиши стрелки. 
- Использование управления шагом с помощью клавиш и ключей мыши, касания или клавиатуры/ ввода.

## <a name="change-the-map-style"></a>Изменить стиль карты

Не все разработчики хотят, чтобы все возможные стили карт были доступны в их приложении. Если разработчик отображает управление картой на сборщике стилей, пользователь может изменить стиль карты с помощью мыши, касания или клавиатуры со вкладкой или введите ключ. Разработчик может указать, какие стили карты они хотят сделать доступными в управлении сборщиком стилей карты. Кроме того, разработчик может программно установить и изменить стиль карты.

**Использовать высокую контрастность**

- Когда управление картой загружается, он проверяет, включен ли высокий контраст и браузер поддерживает его.
- Управление картой не отслеживает режим высокой контрастности устройства. Если режим устройства изменяется, карта не изменится. Таким образом, пользователю необходимо будет перезагрузить карту, освежая страницу.
- При обнаружении высокого контраста стиль карты автоматически перейдет на высокий контраст, а все встроенные элементы управления будут использовать высокий контрастный стиль. Например, в качестве элемента управления будут использоваться, например, ЗумКонтроль, PitchControl, CompassControl, StyleControl и другие встроенные элементы управления.
- Есть два типа высокой контрастности, светлый и темный. Если тип высокого контраста может быть обнаружен с помощью элементов управления картой, то поведение карты будет соответствующим образом корректироваться. Если свет, то стиль grayscale_light карты будет загружен. Если тип не может быть обнаружен или темный, то high_contrast_dark стиль будет загружен.
- При создании пользовательских элементов управления полезно знать, используют ли встроенные элементы управления высокий контрастный стиль. Разработчики могут добавить css класс на карте контейнер div для проверки. Css классы, которые `high-contrast-dark` будут `high-contrast-light`добавлены и . Чтобы проверить с помощью JavaScript, используйте:

```javascript
map.getMapContainer().classList.contains("high-contrast-dark")
```

или, используйте:

```javascript
map.getMapContainer().classList.contains("high-contrast-light")
```

## <a name="keyboard-shortcuts"></a>сочетания клавиш

Карта имеет ряд клавиш, встроенных в которые делают его проще в использовании карты. Эти сочетания клавиш работают, когда карта имеет фокус.

| Ключ      | Действие                            |
|----------|-----------------------------------|
| `Tab` | Перейдите через элементы управления и всплывающие окно на карте. |
| `ESC` | Переместите фокус от любого элемента карты к элементу карты верхнего уровня. |
| `Ctrl` + `Shift` + `D` | Переключать уровень детализации чтения экрана.  |
| Клавиша со стрелкой влево | Панорамирование карты осталось 100 пикселей |
| Клавиша со стрелкой вправо | Панорамирование карты вправо 100 пикселей |
| Клавиша со стрелкой вниз | Перекать карту на 100 пикселей |
| Клавиша со стрелкой вверх | Панорамирование карты на 100 пикселей |
| `Shift`- вверх по стрелке | Увеличьте шаг карты на 10 градусов |
| `Shift`- вниз стрелка | Уменьшение шага карты на 10 градусов |
| `Shift`правая стрелка | Поверните карту на 15 градусов по часовой стрелке |
| `Shift`- левая стрелка | Поверните карту на 15 градусов против часовой стрелки |
| Плюс знак`+`() или <sup>*</sup>`=`Знак Равные () | Увеличить |
| Минус знак, дефис`-` <sup>*</sup>(),`_`или Underscore ( ) | Мельче | 
| `Shift`- мышь перетащить на карте, чтобы нарисовать области | Увеличить площадь |

<sup>*</sup>Эти ключевые ярлыки обычно имеют один и тот же ключ на клавиатуре. Эти ярлыки были добавлены для улучшения пользовательского опыта. Это также не имеет значения, если пользователь использует ключ смены или нет для этих ярлыков.

## <a name="screen-reader-support"></a>Поддержка чтения экрана

Пользователи могут перемещаться по карте с помощью клавиатуры. Если запущено устройство чтения с экрана, карта будет уведомлять пользователя об изменениях в ее состоянии. Например, пользователи получают уведомления об изменениях карты при панорамировании или изменении масштаба. По умолчанию карта предоставляет упрощенные описания, исключающие уровень масштабирования и координаты центра карты. Пользователь может переключить уровень детализации этих описаний `Ctrl`  +  `Shift`  +  `D`с помощью клавиатуры короткий разрез.

Дополнительные сведения, размещаемые на базовой карте, должны содержать соответствующую текстовую информацию для пользователей устройства чтения с экрана. Не забудьте добавить [доступные богатые интернет-приложения (ARIA)](https://www.w3.org/WAI/standards-guidelines/aria/), Alt, и название атрибуты, где это уместно. 

## <a name="make-popups-keyboard-accessible"></a>Сделать всплывающие клавиатуры доступны

Маркер или символ часто используется для представления местоположения на карте. Дополнительная информация о местоположении обычно отображается во всплывающем окне, когда пользователь взаимодействует с маркером. В большинстве приложений всплывающие данные появляются, когда пользователь нажимает или нажимает маркер. Однако нажатие и нажатие требуют, чтобы пользователь использовал мышь и сенсорный экран соответственно. Хорошая практика заключается в том, чтобы сделать всплывающие окно доступным при использовании клавиатуры. Эта функциональность может быть достигнута путем создания всплывающего пункта для каждой точки данных и добавления его на карту. 

Следующий пример загружает точки интересов на карте с помощью слоя символа и добавляет всплывающее окно на карту для каждой точки интереса. Ссылка на каждое всплывающее окно хранится в свойствах каждой точки данных. Он также может быть извлечен для маркера, например, при нажатии маркера. При фокусе на карте нажатие клавиши вкладки позволит пользователю пройти через каждое всплывающее окно на карте.

## <a name="additional-accessibility-tips"></a>Дополнительные советы по доступности

Вот несколько дополнительных советов, чтобы сделать ваше приложение для веб-карт более доступным.

- При отображении большого количества интерактивных точек на карте следует уменьшить беспорядок и использовать кластеризм. 
- Убедитесь, что соотношение контрастности цвета между текстом/символами и фоновыми цветами составляет 4,5:1 или более.
- Держите ваш читатель экрана (ARIA, alt, и атрибуты названия) сообщения короткие, описательные и значимые. Избегайте ненужных жаргонов и аббревиатур.
- Попробуйте оптимизировать сообщения, отправленные на экран, чтобы обеспечить короткую значимую информацию, которую пользователю легко переварить. Например, если вы хотите обновить считыватель экрана на высокой частоте, например, когда карта движется, подумайте о следующих точках:
    - Подождите, пока карта не завершит движение, чтобы обновить считыватель экрана.
    - Задушить обновления один раз в несколько секунд. 
    - Совмещайте сообщения логически. 
- Старайтесь не использовать цвета в качестве единственного способа передачи информации. Используйте текст, значки или узоры, чтобы дополнить или заменить цвет. Некоторые замечания:
    - При использовании слоя пузыря для отображания относительного значения между точками данных, рассмотрите возможность масштабирования радиуса каждого пузыря, окраски пузыря или и того, и другого. 
    - Рассмотрите возможность использования слоя символов с различными значками для различных категорий метрик, таких как треугольники, звезды и квадраты. Слой символа также поддерживает масштабирование размера значка. Также можно отображать текстовую метку.
    - При отображении данных строки ширина может быть использована для представления веса или размера. Шаблон тире-array может быть использован для представления различных категорий линий. Слой символа может быть использован в сочетании с линией для наложения значков вдоль линии. Использование значка стрелки полезно для отображения потока или направления линии.
    - При отображении данных полигона в качестве альтернативы цвету можно использовать такой шаблон, как полосы. 
- Некоторые визуализации, такие как тепловые карты, слои плитки и слои изображений, недоступны для пользователей с нарушениями зрения. Некоторые замечания:
    - Поимеется на рассеивание экрана, что отображаетслой слой при добавлении на карту. Например, если отображается слой плитки метеорологической плитки, то читатель экрана говорит: «На карте наложены данные радаров погоды».
- Ограничьте количество функциональных возможностей, требующих мыши нависшей. Эти функции будут недоступны для пользователей, которые используют клавиатуру или сенсорное устройство для взаимодействия с вашим приложением. Обратите внимание, что это все еще хорошая практика, чтобы иметь стиль нависка для интерактивного контента, таких как кликабельные значки, ссылки и кнопки.
- Попробуйте навигацию по приложению с помощью клавиатуры. Убедитесь, что заказ вкладок является логичным.
- При создании ярлыков клавиш, попробуйте ограничить его двумя клавишами или меньше.

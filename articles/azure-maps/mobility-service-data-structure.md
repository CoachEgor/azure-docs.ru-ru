# <a name="data-structures-in-azure-maps-mobility-service"></a>Структуры данных в службе мобильности Карт Azure

В этой статье представлена концепция Metro Area в [службе мобильности Azure Maps.](https://aka.ms/AzureMapsMobilityService) Мы обсуждаем некоторые из общих полей, которые возвращаются, когда эта услуга запрашивается для остановки общественного транспорта и линий. Мы рекомендуем прочитать эту статью, прежде чем разрабатывать с Помощью AA-сервиса мобильности.

## <a name="metro-area"></a>Район метро

Данные Службы мобильности сгруппированы в поддерживаемые районы метро. Районы метро не следуют границам города. Район метро может содержать несколько городов, густонаселенный город и окружающие города. В самом деле, страна / регион может быть один район метро. 

Это `metroID` идентификатор области метро, которые могут быть использованы для вызова [Get Metro Area Info API](https://aka.ms/AzureMapsMobilityMetroAreaInfo). Используйте API API Azure Maps "Get Metro", чтобы запросить типы транзита, транзитные агентства, активные оповещения и дополнительные сведения для выбранного метро. Вы также можете запросить поддерживаемые зоны метро и метро. Иди имитирует область метро.

## <a name="stop-ids"></a>Стоп идол

Остановки транзита могут быть переданы двумя типами идентификаторов: [идентификатором общей спецификации транзитных каналов (GFTS)](https://gtfs.org/) и идентификатором остановки Azure Maps. Идентификатор GFTS называется стоп-ключом, а идентификатор остановки Azure Maps — стопид. При частом соотвелении транзитных остановок рекомендуется использовать идентификатор остановки Azure Maps. stopID является более стабильным и, вероятно, останется неизменным до тех пор, пока существует физическая остановка. Идентификатор остановки GTFS обновляется чаще. Например, идентификатор остановки GTFS может быть обновлен по запросу поставщика GTFS или при выпуске новой версии GTFS. Несмотря на то, что физическая остановка не изменилась, идентификатор остановки GTFS может измениться.

Для начала можно запросить близлежащие транзитные остановки с помощью [Get Nearby Transit API.](https://aka.ms/AzureMapsMobilityNearbyTransit)

## <a name="line-groups-and-lines"></a>Линейные группы и линии

Служба мобильности использует параллельную модель данных для линейных и линейных групп. Эта модель используется для более эффективной обработки изменений, унаследованных от маршрутов [GTFS](https://gtfs.org/) и данных о поездках.


### <a name="line-groups"></a>Линейные группы

Линия группы — это сущность, которая объединяет все строки, которые логически являются частью одной группы. Обычно группа строк содержит две строки: одну из точки А в точку Б, а другую, возвращающуюся из точки Б в Точку А. Обе линии будут принадлежать одному и тому же агентству общественного транспорта и иметь один и тот же номер линии. Однако могут быть случаи, когда линейная группа имеет более двух строк или только одну строку внутри нее.


### <a name="lines"></a>Линии

Как уже говорилось выше, каждая группа строк состоит из набора строк. Каждая группа строк состоит из двух строк, и каждая строка описывает направление.  Однако есть случаи, когда больше строк составляют группу строк. Например, есть линия, которая иногда объезжает по определенному району, а иногда нет. В обоих случаях он работает под одинаковым номером строки. Также линейная группа может состоять из одной строки. Круговая линия с одним направлением представляет собой группу линг с одной строкой.

Для начала можно запросить линейные группы с помощью [API Get Transit Line.](https://aka.ms/AzureMapsMobilityTransitLine)

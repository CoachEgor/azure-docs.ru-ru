# <a name="data-structures-in-azure-maps-mobility-service"></a>Структуры данных в службе Mobility Service Azure Maps

В этой статье описывается понятие области Metro в [Azure Maps службы Mobility Service](https://aka.ms/AzureMapsMobilityService) и некоторые общие поля, возвращаемые через службы, когда запрашиваются запросы на общедоступное транзитное место и останавливается. Перед началом работы с API-интерфейсами службы Mobility Service рекомендуется ознакомиться с этой статьей. Ниже приведены общие поля.

## <a name="metro-area"></a>Область Metro

Данные службы мобильности делятся на поддерживаемые области метро. Области Metro не следуют границам городов, в области Metro может содержаться несколько городов, например заполнение города и окружающих городов. а страна или регион могут быть одной областью метро. 

`metroID` — это идентификатор области метро, который можно использовать для вызова [API получения сведений об областях](https://aka.ms/AzureMapsMobilityMetroAreaInfo) метро, чтобы запросить поддерживаемые транзитные типы и дополнительные сведения для области Metro, такие как транзитные агентства и активные оповещения. Вы можете использовать Azure Maps получить API Metro, чтобы запросить поддерживаемые области Metro и Метроидс. Идентификаторы областей Metro могут изменяться.

## <a name="stop-ids"></a>Идентификаторы завершения

На транзитные остановки могут ссылаться два типа идентификаторов: идентификатор [гфтс (общий канал транзитного канала)](https://gtfs.org/) (например, стопкэй) и идентификатор остановки Azure Maps (называется стопид). При ссылке на остановку с течением времени рекомендуется использовать идентификатор остановки Azure Maps, так как этот идентификатор является более стабильным и, скорее всего, будет изменяться при условии, что физическая остановка существует. Идентификатор ГТФС останавливает обновляться чаще, например, на случай, если поставщику ГТФС нужно изменить его или выпустилась новая версия ГТФС, хотя физическая точка не изменилась.

Для начала можно запросить расположенные поблизости Транзитные остановки с помощью [API Get поблизости](https://aka.ms/AzureMapsMobilityNearbyTransit).

## <a name="line-groups-and-lines"></a>Группы линий и линии

Служба Mobility Service использует параллельную модель данных для линий и групп строк, чтобы лучше справляться с изменениями, наследуемыми от модели данных [гтфс](https://gtfs.org/) Routes и поездками.


### <a name="line-groups"></a>Группы строк

Группа строк — это сущность, которая объединяет все строки, логически входящие в одну и ту же группу. Обычно группа линий будет содержать две строки: один переход от точки A к B, а другой возврат из точки B к A, как к одному открытому агентству транспорта, так и к одному и тому же номеру строки. Однако могут возникнуть ситуации, в которых группа строк содержит более двух строк или только одну строку внутри нее.


### <a name="lines"></a>Линии

Как упоминалось выше, каждая группа строк состоит из набора строк. Часто каждая строка описывает направление, а каждая группа строк состоит из двух строк. Однако бывают случаи, в которых больше строк состоит из группы строк, например, есть строка, которая иногда высматривает определенное окружение, а иногда не имеет и работает в обоих случаях с одинаковым номером строки, и существуют и другие случаи, в которых строка g ку состоит из одной строки, например круговой линии с одним направлением.

Чтобы начать, вы можете запросить группы строк с помощью [API получения транзитного](https://aka.ms/AzureMapsMobilityTransitLine) пакета и позже детализировать строки.

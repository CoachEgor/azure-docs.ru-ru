# <a name="data-structures-in-azure-maps-mobility-service"></a>Структуры данных в службу Mobility Service Azure Maps

В этой статье рассматриваются концепции в области «METRO» [службу Mobility Service Azure Maps](https://aka.ms/AzureMapsMobilityService) и некоторые из стандартных полей, возвращаемых через службы, ответ на запрос общественного останавливает и "строки". Рекомендуется выполнить в этой статье перед начинает работать с API-интерфейсы службы мобильности. Мы обсудим эти общие поля ниже.

## <a name="metro-area"></a>Область «METRO»

Данные службы мобильности разбивается на поддерживаемых зонах метро. Зонах метро несоблюдении границ города, область «METRO» может содержать несколько городов, например плотно заполненный Город и его окружающие городов; и страны или региона может быть одна из областей «Metro». 

`metroID` — Это идентификатор metro области, который может использоваться для вызова [Info API получения области Metro](https://aka.ms/AzureMapsMobilityMetroAreaInfo) типы транзитный запрос поддерживается и Дополнительные сведения о metro области, например пути учреждений и активные предупреждения. API получения Metro Azure Maps можно использовать для запроса, поддерживаемых зонах метро и metroIDs. Идентификаторы METRO области могут меняться.

## <a name="stop-ids"></a>Остановить идентификаторы

Останавливает пути можно ссылаться по два вида идентификаторов, [общие спецификации веб-канала передачи (GFTS)](https://gtfs.org/) идентификатор (называется stopKey) и Azure Maps остановить идентификатор (называется stopId). При ссылке на останавливается со временем, рекомендуется использовать идентификатор остановки службы карт Azure, как этот идентификатор Повышена стабильность работы и не меняется скорее всего, пока существует физических stop. Идентификатор остановки GTFS обновляется чаще, например, в случае GTFS поставщик должен изменить его или новые версии GTFS, несмотря на то, что физические stop бы без изменений.

Чтобы начать, вы можете запросить останавливает рядом пути с помощью [API получения рядом передаче](https://aka.ms/AzureMapsMobilityNearbyTransit).

## <a name="line-groups-and-lines"></a>Группы строк и строк

Служба Mobility Service использует модель параллельных данных для строк и группы строк, чтобы лучше работать с изменениями наследуется от [GTFS](https://gtfs.org/) маршруты и обработки модели данных.


### <a name="line-groups"></a>Группы строк

Группу строк — это сущность, в которой сгруппированы все строки, которые логически являются частью той же группе. Обычно группу строк будет содержать две строки, один переход от точки А, б и другие возврат из точки B до A, как принадлежащие к же агентство общественный транспорт и продолжение тот же номер. Однако возможны ситуации, в которых группу строк имеет более двух строк или только одной строке внутри него.


### <a name="lines"></a>Прямых линий

Как уже говорилось ранее, каждой группы строк представляет собой набор строк. Часто каждая строка описывает направление и каждой строки группы состоит из двух строк. Однако бывают случаи, в какие дополнительные строки образуют группу строк, например существует представляет собой строку, которая иногда обходит некоторые окружением и иногда — нет и эксплуатации в обоих случаях в тот же номер, а также других случаях, в котором строки g Группа состоит из одной строки, например окружностей с одним направлением.

Чтобы начать работу, вы можете запросить группы строк с помощью [API строки пути получения](https://aka.ms/AzureMapsMobilityTransitLine) и более поздней версии перейдите к разделу строк.

# <a name="render-custom-data-on-a-raster-map"></a>Обработка пользовательских данных на карты raster

В этой статье описываются способы использования [службы статическое изображение](https://docs.microsoft.com/rest/api/maps/render/getmapimage) с образа композиции функциональные возможности, обеспечивающие наложений поверх карты raster. Изображение композиции включает в себя возможность получить плитку растровых обратно, с дополнительными данными пользовательских кнопок, меток и наложений geometry.

Для подготовки к просмотру, пользовательских кнопок, меток и наложений geometry, можно использовать приложение Postman. Вы можете использовать карты Azure [API-интерфейсы службы данных](https://docs.microsoft.com/rest/api/maps/data) для хранения и отображения наложения.


## <a name="prerequisites"></a>Технические условия

### <a name="create-an-azure-maps-account"></a>создание учетной записи службы "Карты Azure";

Чтобы выполнить процедуры, описанные в этой статье, необходимо сначала [создать учетную запись Azure Maps](how-to-manage-account-keys.md) в ценовой категории S1.

## <a name="render-pushpins-with-labels-and-a-custom-image"></a>Визуализации кнопок с метками и пользовательского образа

> [!Note]
> Процедуру, описанную в этом разделе требуется учетная запись Azure Maps в ценовой категории, S0 или S1.

Карты Azure учетная запись поддерживает уровень S0 только один экземпляр `pins` параметра. Он позволяет отображать до пяти меток, указанный в запросе URL-адрес, с помощью пользовательского образа.

Для отображения кнопок с метками и пользовательский образ, выполните следующие действия:

1. Создайте коллекцию, в которой будут храниться запросы. В приложении Postman, выберите **New**. В **Create New** выберите **коллекции**. Имя коллекции и выберите **создать** кнопки. 

2. Чтобы создать запрос, выберите **New** еще раз. В **Create New** выберите **запроса**. Введите **имя запроса** для кнопок, выберите коллекцию, созданную на предыдущем шаге, как расположение, в котором следует сохранить запрос, а затем выберите **Сохранить**.
    
    ![Создайте запрос в Postman](./media/how-to-render-custom-data/postman-new.png)

3. Выберите метод GET HTTP на вкладку «Конструктор» и введите следующий URL-адрес, чтобы создать запрос GET.

    ```HTTP
    https://atlas.microsoft.com/map/static/png?subscription-key={subscription-key}&api-version=1.0&layer=basic&style=main&zoom=12&center=-73.98,%2040.77&pins=custom%7Cla15+50%7Cls12%7Clc003b61%7C%7C%27CentralPark%27-73.9657974+40.781971%7C%7Chttp%3A%2F%2Fazuremapscodesamples.azurewebsites.net%2FCommon%2Fimages%2Fpushpins%2Fylw-pushpin.png
    ```
    Ниже приведен результирующий образ.

    ![Пользовательский значок канцелярской кнопки с меткой](./media/how-to-render-custom-data/render-pins.png)


## <a name="get-data-from-azure-maps-data-storage"></a>Получение данных из хранилища данных Azure Maps

> [!Note]
> Процедуру, описанную в этом разделе требуется учетная запись Azure Maps в ценовой категории S1.

Можно также получить сведения о расположении пути и ПИН-код с помощью [API отправки данных](https://docs.microsoft.com/rest/api/maps/data/uploadpreview). Выполните следующие действия, чтобы отправить данные пути и кнопок.

1. В приложении Postman откройте новую вкладку в коллекции, созданной в предыдущем разделе. Выберите метод POST HTTP на вкладку «Конструктор» и введите следующий URL-адрес для выполнения запроса POST:

    ```HTTP
    https://atlas.microsoft.com/mapData/upload?subscription-key={subscription-key}&api-version=1.0&dataFormat=geojson
    ```

2. На **Params** введите следующие пары ключ/значение, которые используются для URL-АДРЕСЕ запроса POST. Замените `subscription-key` значение с ключ подписки для службы карт Azure.
    
    ![Params ключ/значение в Postman](./media/how-to-render-custom-data/postman-key-vals.png)

3. На **текст** выберите формат raw ввода и выберите JSON как формат входных данных из раскрывающегося списка. Ввести этот код JSON в качестве данных для загрузки:
    
    ```JSON
    {
      "type": "FeatureCollection",
      "features": [
        {
          "type": "Feature",
          "properties": {},
          "geometry": {
            "type": "Polygon",
            "coordinates": [
              [
                [
                  -73.98235,
                  40.76799
                ],
                [
                  -73.95785,
                  40.80044
                ],
                [
                  -73.94928,
                  40.7968
                ],
                [
                  -73.97317,
                  40.76437
                ],
                [
                  -73.98235,
                  40.76799
                ]
              ]
            ]
          }
        },
        {
          "type": "Feature",
          "properties": {},
          "geometry": {
            "type": "LineString",
            "coordinates": [
              [
                -73.97624731063843,
                40.76560773817073
              ],
              [
                -73.97914409637451,
                40.766826609362575
              ],
              [
                -73.98513078689575,
                40.7585866048861
              ]
            ]
          }
        }
      ]
    }
    ```

4. Выберите **отправки** и проверьте заголовок ответа. Заголовок location содержит URI, используемый для доступа к или загрузить данные для использования в будущем. Он также будет включать уникальное значение `udId` для отправленных данных.  

   ```HTTP
   https://atlas.microsoft.com/mapData/{udId}/status?api-version=1.0&subscription-key={Subscription-key}
   ```

5. Используйте `udId` значение, полученное из API отправки данных для подготовки к просмотру компоненты на карте. Чтобы сделать это, откройте новую вкладку в коллекции, созданной в предыдущем разделе. Выберите метод GET HTTP на вкладку «Конструктор» и введите этот URL-адрес для запроса GET:

    ```HTTP
    https://atlas.microsoft.com/map/static/png?subscription-key={subscription-key}&api-version=1.0&layer=basic&style=main&zoom=12&center=-73.96682739257812%2C40.78119135317995&pins=default|la-35+50|ls12|lc003C62|co9B2F15||'Times Square'-73.98516297340393 40.758781646381024|'Central Park'-73.96682739257812 40.78119135317995&path=lc0000FF|fc0000FF|lw3|la0.80|fa0.30||udid-{udId}
    ```

## <a name="render-a-circle-and-pushpins-with-custom-labels"></a>Отрисовывает круг и защелки с пользовательские метки

> [!Note]
> Процедуру, описанную в этом разделе требуется учетная запись Azure Maps в ценовой категории S1.


Внесения защелки и их метки увеличить или уменьшить с помощью `sc` модификатор стиля масштабирования. Этот модификатор принимает значение, которое больше нуля. Значение 1 — это стандартный масштаб. Значения, превышающие 1, будут увеличивать кнопки, а меньше 1 будут их уменьшать. Дополнительные сведения о стиле модификаторы, см. в разделе [параметры пути службы статическое изображение](https://docs.microsoft.com/rest/api/maps/render/getmapimage#uri-parameters).


Выполните следующие действия для подготовки к просмотру круг и кнопок с помощью пользовательских меток.

1. В приложении Postman откройте новую вкладку в коллекции, созданной ранее. Выберите метод GET HTTP на вкладку «Конструктор» и введите этот URL-адрес для запроса GET:

    ```HTTP
    https://atlas.microsoft.com/map/static/png?api-version=1.0&style=main&layer=basic&zoom=14&height=700&Width=700&center=-122.13230609893799,47.64599069048016&path=lcFF0000|lw2|la0.60|ra1000||-122.13230609893799 47.64599069048016&pins=default|la15+50|al0.66|lc003C62|co002D62||'Microsoft Corporate Headquarters'-122.14131832122801  47.64690503939462|'Microsoft Visitor Center'-122.136828 47.642224|'Microsoft Conference Center'-122.12552547454833 47.642940335653996|'Microsoft The Commons'-122.13687658309935  47.64452336193245&subscription-key={subscription-key}
    ```

    Вот изображение ответа:

    ![Визуализации круг с помощью настраиваемых кнопок](./media/how-to-render-custom-data/circle-custom-pins.png)

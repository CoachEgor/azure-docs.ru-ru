# <a name="choose-the-right-pricing-tier-in-azure-maps"></a>Выбор подходящей ценовой категории в Azure Maps

Azure Maps предлагает два ценовых уровня: S0 и S1. Цель этой статьи — помочь вам выбрать правильную ценовую категорию, которая соответствует вашим потребностям. Чтобы выбрать правильный уровень ценообразования, задайте себе следующие два вопроса.

## <a name="how-many-concurrent-users-do-i-plan-to-support"></a>Сколько одновременно работающих пользователей планируется поддерживать? 
Ценовые категории S0 и S1 обрабатывают различные объемы пропускной способности данных. Ценовая категория S0 обрабатывает до **50 запросов в секунду**. В то время как уровень S1 обрабатывает **более 50 запросов в секунду.**

## <a name="what-geospatial-capabilities-do-i-plan-to-use"></a>Какие геопространственные возможности планируется использовать?
Уровень ценообразования S0 подходит именно вам, если основные геопространственные AIS отвечают вашим требованиям к обслуживанию. Если вам нужны более продвинутые возможности для приложения, рассмотрите возможность выбора ценовой категории S1. Расширенные возможности включают в себя: Воздушные плюс гибридные изображения, получение диапазона маршрута, и пакет геокодирования. Просмотрите таблицу **возможностей уровня ценообразования,** чтобы выбрать наиболее подходящий для приложения уровень ценообразования.

### <a name="pricing-tier-targeted-customers"></a>Целевые клиенты ценовой категории

Ознакомьтесь с таблицей **ценовых категорий для целевых клиентов**, чтобы лучше понять смысл ценовых категорий S0 и S1. Дополнительные сведения см. на странице [Цены на Azure Maps](https://azure.microsoft.com/pricing/details/azure-maps/). 

| Ценовая категория  |     Целевые клиенты                                                                |
|-----------------|:-----------------------------------------------------------------------------------------|
| S0            |    <p>Уровень ценообразования S0 работает для приложений на всех этапах производства: от разработки концепции и ранней стадии тестирования до производства и развертывания приложений. Однако этот уровень предназначен для мелкомасштабной разработки, или клиентов с низкими одновременными пользователями, или обоих. <p>|
| S1            |    <p>Уровень ценообразования S1 предназначен для клиентов с крупномасштабными корпоративными приложениями, критически важными приложениями или большими объемами одновременных пользователей. Она также доступна для тех клиентов, которым необходимы дополнительные геопространственные службы.</p>|

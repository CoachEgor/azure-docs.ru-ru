# <a name="enable-browser-connection-on-lab-virtual-machines"></a>Включить подключение браузера на лабораторных виртуальных машинах 
DevTest Labs интегрируется с [Azure Bastion,](https://docs.microsoft.com/azure/bastion/)который позволяет подключаться к вашим виртуальным машинам через браузер. Сначала необходимо включить подключение браузера к лабораторным виртуальным машинам.

Как владелец лаборатории вы можете включить доступ ко всем лабораторным виртуальным машинам через браузер. Не требуется дополнительный клиент, агент или часть программного обеспечения. Azure Bastion обеспечивает безопасное и бесшовное подключение RDP/SSH к вашим виртуальным машинам непосредственно на портале Azure через TLS. При подключении через Azure Bastion вашим виртуальным машинам не нужен общедоступный IP-адрес. Для получения дополнительной [What is Azure Bastion?](../bastion/bastion-overview.md) информации см.


В этой статье показано, как включить подключение браузера на лабораторных виртуальных машинах.

## <a name="prerequisites"></a>Предварительные требования 
Либо развернуть хост Bastion в виртуальной сети **(ИЛИ)** существующей лаборатории, соединив лабораторию с виртуальной сетью Bastion. 

Чтобы узнать, как развернуть узел Bastion в виртуальной сети, [см.](../bastion/bastion-create-host-portal.md) При создании хоста Bastion выберите виртуальную сеть лаборатории. 

Во-первых, необходимо создать вторую подсеть в виртуальной сети Bastion, потому что AzureBastionSubnet не позволяет создавать в ней небастионные ресурсы. 

## <a name="create-a-second-sub-net-in-the-bastion-virtual-network"></a>Создание второй подсети в виртуальной сети «Бастион»
Вы не можете создавать лабораторные VMs в подсети Azure Bastion. Создайте еще одну подсеть в виртуальной сети Bastion, показанную на следующем изображении:

![Вторая подсеть в виртуальной сети Azure Bastion](./media/connect-virtual-machine-through-browser/second-subnet.png)

## <a name="enable-vm-creation-in-the-subnet"></a>Включить создание VM в подсети
Теперь, включите создание VMs в этой подсети, следуя следующим шагам: 

1. Войдите на [портал Azure](https://portal.azure.com).
1. Выберите **все службы** в левом навигационном меню. 
1. Выберите **DevTest Labs** из списка. 
1. Из списка лабораторий выберите *лабораторию.* 

    > [!NOTE]
    > Лазурный бастион теперь широко доступен в следующих регионах: Западная ЧАСТЬ США, Восточная ЧАСТЬ США, Западная Европа, южная центральная часть США, Австралия Восток и Япония Восток. Таким образом, создайте лабораторию в одном из этих регионов, если ваша лаборатория не в одном из них. 
    
1. Выберите **конфигурацию и политики** в разделе **Настройки** в левом меню. 
1. Выберите **виртуальные сети**.
1. Выберите **Добавить** из панели инструментов. 
1. Выберите **виртуальную сеть,** в которую развернут хост Bastion. 
1. Выберите подсеть для VMs, а не **AzureBastionSubnet**, другую, созданную ранее. Закройте страницу и откройте ее, если вы не увидите подсеть в списке внизу. 

    ![Включить создание VM в подсети](./media/connect-virtual-machine-through-browser/enable-vm-creation-subnet.png)
1. Выберите **Использование в варианте создания виртуальной машины.** 
1. На панели инструментов щелкните **Сохранить**. 
1. Если у вас есть старая виртуальная сеть для лаборатории, удалите ее, выбрав *...*  и **удалить**. 

## <a name="enable-browser-connection"></a>Включить подключение браузера 

Если у вас есть Bastion настроенвиртуальной виртуальной сети внутри лаборатории, как владелец лаборатории, вы можете включить браузер подключиться на лабораторных виртуальных машин.

Чтобы обеспечить подключение браузера на лабораторных виртуальных машинах, выполните следующие действия:

1. На портале Azure перейдите в *лабораторию.*
1. Выберите **Configuration and policies** (Конфигурация и политики).
1. В **настройках**выберите **подключение браузера.** Если вы не видите эту опцию, закройте страницу **политикконфигурации** и повторно откройте ее. 

    ![Включить подключение браузера](./media/enable-browser-connection-lab-virtual-machines/browser-connect.png)

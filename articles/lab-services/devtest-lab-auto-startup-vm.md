# <a name="auto-startup-lab-virtual-machines"></a>Авто запуск лаборатории виртуальных машин  
Azure DevTest Labs позволяет настроить виртуальные машины в лаборатории для автоматического запуска и выключения на основе графика. Для получения информации о настройке параметров автоотключения см. [Управление политиками автоматизации для лаборатории в Лабораториях Azure DevTest Labs.](devtest-lab-auto-shutdown.md) 

В отличие от автовыключения, где все ВМ включены при включении политики, политика автоматического запуска требует от лабораторного пользователя явно выбрать VM и выбрать в этом графике. Таким образом, вы не будете легко работать в ситуации, когда нежелательные VMs случайно автоматически запущен и вызвать неожиданные расходы.

В этой статье показано, как настроить политику автоматического запуска для лаборатории.

## <a name="configure-autostart-settings-for-a-lab"></a>Настройка настроек автоматического запуска для лаборатории 
1. Перейдите на главную страницу для вашей лаборатории. 
2. Выберите **конфигурацию и политики** в левом меню. 

    ![Меню конфигурации и политик](./media/devtest-lab-auto-startup-vm/configuration-policies-menu.png)
3. На странице **конфигурации и политики** сделайте следующие шаги:
    
    1. Выберите **On** для **Позволяет виртуальные машины, которые будут запланированы для автоматического запуска,** чтобы функция автоматического запуска для этой лаборатории. 
    2. Выберите время начала (например: 8:00:00) для стартового поля **Расписание.** 
    3. Выберите **часовой пояс** для использования. 
    4. Выберите **дни недели,** на которых вс-м всдолжны должны быть автоматически запущены. 
    5. Затем выберите **Сохранить** на панели инструментов, чтобы сохранить настройки. 

        ![Настройки автоматического запуска](./media/devtest-lab-auto-startup-vm/auto-start-configuration.png)

        > [!IMPORTANT]
        > Эта политика не применяется автоматически к любым виртуальным машинам в лаборатории. Чтобы **выбрать в** отдельных виртуальных машин, перейдите на страницу виртуальной машины и включить **автоматическое запуск** для этого VM.

## <a name="enable-autostart-for-a-vm-in-the-lab"></a>Включить автозапуск для ВМ в лаборатории
Следующая процедура предоставляет вам шаги для выбора в VM в политику автозапуска лаборатории. 

1. На главной странице вашей лаборатории выберите **VM** в списке **моих виртуальных машин.** 

    ![Меню конфигурации и политик](./media/devtest-lab-auto-startup-vm/select-vm.png)
2. На странице **Виртуальной машины** выберите **Autostart** в левом меню или в списке **Расписаний.** 

    ![Выберите меню автозапуска](./media/devtest-lab-auto-startup-vm/select-auto-start.png)
3. На странице **Autostart** выберите **On** для **Разрешить эту виртуальную машину, которая будет запланирована для автоматического запуска опции.**

    ![Включить автозапуск для VM](./media/devtest-lab-auto-startup-vm/auto-start-vm.png)
4. Затем выберите **Сохранить** на панели инструментов, чтобы сохранить настройки.

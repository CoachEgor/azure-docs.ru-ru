# <a name="get-started-and-create-a-lab-services-lab-from-teams"></a>Приступая к работе и создание лаборатории лабораторных служб из рабочих групп

В этой статье показано, как добавить приложение служб лабораторий Azure в группы. Затем создайте лабораторию из команд.

## <a name="add-a-lab-services-app-to-teams"></a>Добавление приложения служб лаборатории в команды

Вы можете добавить службы лаборатории непосредственно в каналы команд, после чего приложение будет доступно всем участникам группы. Для этого выполните приведенные ниже три шага.

1. Перейдите к каналу команд, в который нужно добавить приложение, и выберите, **+** чтобы добавить вкладку, нажав кнопку "...". в верхней части правого окна. 
1. Найдите **службы лаборатории Azure** в параметрах вкладки и добавьте это приложение. 

    > [!NOTE]
    > Только **владельцы** команды смогут создавать лаборатории для группы.
1. Выберите учетную запись служб лаборатории, которую вы хотите использовать для создания учебных занятий в этой команде. 

    Службы лаборатории Azure используют единый вход на [веб-сайт служб лаборатории Azure](https://labs.azure.com) и извлекают все учетные записи лаборатории, к которым у вас есть доступ. 

    Отображаются учетные записи, которые находятся в том же клиенте, что и команды, для которых у вас есть права **владельца**, **участника**или **создателя** . 

   ![Добро пожаловать в Кроме](./media/integrate-with-teams/welcome.png) 
1. Нажмите кнопку **сохранить** , и приложение добавится в команды, и вкладка будет добавлена в канал. 

    Теперь вы можете выбрать вкладку **службы лаборатории Azure** в своем канале и приступить к управлению лабораториями, как описано на следующем шаге.

    После того как один из участников команды добавит вкладку, она будет отображаться для всех пользователей в канале. Все пользователи, имеющие доступ к приложению, получают возможность единого входа с помощью учетных данных, которые они используют для доступа к Microsoft Teams. Для пользователей, у которых нет доступа к приложению, будет отображаться соответствующая вкладка в Teams, однако они будут заблокированы, пока вы не предоставите им разрешения на доступ к локальному приложению и его версии, опубликованной на портале Azure.

## <a name="create-a-classroom-lab"></a>Создание лаборатории для аудитории

После выбора учетной записи лаборатории владельцы групп смогут создавать лаборатории для группы. Весь процесс создания лаборатории и все задачи на уровне лаборатории можно выполнять в группах. Пользователи смогут создавать несколько лабораторий в рамках одной команды, а также владельца группы с соответствующим доступом на уровне учетной записи лаборатории, увидят только лабораторные работы, связанные с конкретной группой.

## <a name="giving-access-to-users-of-the-lab-account"></a>Предоставление доступа пользователям учетной записи лаборатории

Доступ к подготовке пользователей на уровне учетной записи лаборатории должен выполняться на портале [Azure](https://ms.portal.azure.com/) .

1. В портал Azure перейдите к учетной записи служб лаборатории Azure. 
1. На странице **Учетная запись лаборатории** выберите **Управление доступом (IAM)** и на панели инструментов щелкните **+ Добавить** и выберите **+ Добавить назначение ролей**. 

    !["Управление доступом" -> кнопка "Добавить назначение ролей"](./media/tutorial-setup-lab-account/add-role-assignment-button.png)
1. На странице **Добавление назначения ролей** для **роли** выберите **Создатель лаборатории**, выберите пользователя, которого нужно добавить к роли создателя лаборатории, а затем нажмите кнопку **Сохранить**. 

    ![Добавление создателя лаборатории](./media/tutorial-setup-lab-account/add-lab-creator.png)

### <a name="creating-classroom-labs"></a>Создание учебных заработок

Процесс создания занятий по созданию лабораторных занятий тот же, что и при создании лабораторий из команд или [веб-сайта служб лаборатории](https://labs.azure.com). 

Дополнительные сведения о настройке этой статьи см. в статье процесс создания лабораторных занятий: [Управление лабораториями в службах лаборатории Azure](how-to-manage-classroom-labs.md).

## <a name="deleting-classroom-labs"></a>Удаление учебных занятий

Лабораторные работы, созданные в командах, можно удалить на [веб-сайте служб лаборатории](https://labs.azure.com) , непосредственно удалив лабораторию, как описано в статье [Управление учебными лабораториями в службах лаборатории Azure](how-to-manage-classroom-labs.md). 

Удаление лаборатории также активируется при удалении команды. Если группа, в которой создана лаборатория, удаляется, лаборатория будет автоматически удалена через 24 часа после запуска автоматической синхронизации списка пользователей. 

Удаление вкладки или удаления приложения не приведет к удалению лаборатории. Если вкладка удалена, пользователи в списке членства в группе будут по-прежнему иметь доступ к виртуальным машинам на [веб-сайте служб лаборатории](https://labs.azure.com) , пока удаление лаборатории не будет явно активировано путем удаления лаборатории на веб-сайте или удаления команды. 

## <a name="next-steps"></a>Дальнейшие шаги

При создании лаборатории внутри групп список пользователей лаборатории автоматически заполняется и синхронизируется с членством в команде. Все участники группы, включая владельцев, участников и гостей, будут автоматически добавлены в список пользователей лаборатории. Службы лаборатории Azure будут поддерживать синхронизацию с членством в группе, и автоматическая синхронизация запускается каждые 24 часа. Подробная информация доступна в следующих статьях:

[Управление списками пользователей служб лаборатории с командами](how-to-manage-user-lists-within-teams.md)

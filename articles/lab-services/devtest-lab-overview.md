# <a name="about-azure-devtest-labs"></a>Сведения об Azure DevTest Labs
Azure DevTest Labs позволяет разработчикам в группах эффективно управлять виртуальными машинами (VM) и ресурсами PaaS, не дожидаясь одобрения.

DevTest Labs создает лаборатории, состоящие из предварительно настроенных баз или шаблонов Azure Resource Manager. Они имеют все необходимые инструменты и программное обеспечение, которые можно использовать для создания сред. Вы можете создавать среды в течение нескольких минут, в отличие от часов или дней.

Используя DevTest Labs, вы можете протестировать последние версии приложений, выполняя следующие задачи:

- Быстрое предоставление сред Windows и Linux с помощью многоразовых шаблонов и артефактов.
- Для подготовки сред по запросу интегрируйте с DevTest Labs конвейер развертывания.
- Масштабируйте тестирование нагрузки, подготовив несколько тестовых агентов и создайте предварительно подготовленные среды для обучения и демонстраций.

## <a name="capabilities"></a>Возможности
DevTest Labs предоставляет следующие возможности разработчикам, работающим с vMs:

- Создавайте всхоги быстро, выполняя менее пяти простых шагов.
- Выберите из кураторского списка баз VM, которые настроены, утверждены и авторизованы руководителем группы или центральным ИТ-центром.
- Создавайте vMs из заранее созданных пользовательских изображений, которые имеют все программное обеспечение и инструменты, уже установленные. 
- Создавайте vMs из формул, которые по существу являются пользовательскими изображениями в сочетании с последними сборками программного обеспечения, установленного при создании VM. 
- Установите артефакты, которые являются расширениями, развернутыми на VMs после их подготовки.
- Устанавливайте графики автоматического выключения и автоматического запуска на ВМ.
- Претензия заранее созданный VM, не проходя через процесс создания.

DevTest Labs предоставляет следующие возможности разработчикам, работающим с средами PaaS:

- Используйте ресурсный менеджер для быстрого создания сред PaaS, выполняя менее трех простых шагов.
- Выберите один из кураторского списка шаблонов менеджера ресурсов, которые настроены и авторизованы руководителем группы или центральным ИТ-центром.
- Раскрутите пустую группу ресурсов (песочницу) с помощью шаблона «Менеджер ресурсов» для изучения Azure в контексте лаборатории.

DevTest Labs также позволяет центральным ИТ-специалистам контролировать отходы, оптимизировать затраты на ресурсы и оставаться в рамках бюджетов, выполняя следующие задачи:  

- Настройка графиков автоматического выключения и автоматического запуска на ВМ.
- Настройка политик и количества вс-чаевых, которые могут создавать пользователи.
- Настройка политик на размеры ВМ и изображения галереи, которые выбирают пользователи.
- Отслеживание затрат и установление целевых показателей в лабораториях.
- Получение уведомления о высоких прогнозируемых затратах на лаборатории, чтобы вы могли принять необходимые меры.

DevTest Labs предоставляет следующие преимущества в создании, настройке и управлении средами в облаке.

## <a name="cost-control-and-governance"></a>Контроль затрат и управление
DevTest Labs упрощает управление затратами, позволяя выполнять следующие задачи:

- [Устанавливайте политики в лабораториях,](devtest-lab-get-started-with-lab-policies.md)например количество вдовых на одного пользователя или на одну лабораторию. 
- Создание [политик для автоматического выключения](devtest-lab-set-lab-policy.md) и запуска виз.
- Отслеживайте затраты на VMs и ресурсы PaaS, которые закружились внутри лабораторий, чтобы оставаться в рамках [вашего бюджета.](devtest-lab-configure-cost-management.md)
- Оставайтесь в контексте ваших лабораторий, чтобы не раскручивать ресурсы за их пределами.

## <a name="quickly-get-to-ready-to-test"></a>Быстрый переход в состояние "Готово к тестированию"
DevTest Labs позволяет создавать предварительно подготовленные среды, оснащенные всем необходимым для разработки и тестирования приложений. Просто [заявляйте о средах,](devtest-lab-add-claimable-vm.md) в которых установлена последняя хорошая сборка приложения, и начните работать. Или используйте контейнеры для еще более быстрого и компактного создания среды.

## <a name="use-iaas-and-paas-resources"></a>Использование ресурсов IaaS и PaaS 
Разработчики также могут использовать ресурсы PaaS, такие как кластеры Azure Service Fabric, функция Web Apps Службы приложений Azure и фермы SharePoint, используя шаблоны «Менеджер ресурсов». Чтобы начать работу на PaaS в лабораториях, используйте шаблоны из [общедоступного хранилища или](devtest-lab-configure-use-public-environments.md) [подключите лабораторию к своему репозиторию Git.](devtest-lab-create-environment-from-arm.md#configure-your-own-template-repositories) Вы также можете отслеживать затраты на эти ресурсы, чтобы оставаться в рамках вашего бюджета.

# <a name="allow-lab-creator-to-pick-location-for-the-lab-in-azure-lab-services"></a>Разрешить создателю лаборатории выбрать местоположение для лаборатории в Лабораторных службах Azure
В Лаборант-сервисе Azure Lab владелец учетной записи может разрешить создателям лабораторий (преподавателям) выбрать место для создаваемых лабораторий. Это местоположение может отличаться от расположения учетной записи лаборатории. Местоположение — это группа регионов Azure. Например, расположение Соединенных Штатов представляет собой группу регионов, таких как Восточная ЧАСТЬ США, Запад США и так далее. 

Вы, как владелец учетной записи лаборатории, можете выбрать **создателя лаборатории Allow для выбора** варианта определения местоположения в лаборатории при создании лабораторной учетной записи и после создания учетной записи лаборатории (или существующей учетной записи лаборатории). 

## <a name="at-the-time-of-lab-account-creation"></a>На момент создания лабораторного счета
При создании учетной записи лаборатории эта опция просматривается на первом экране **(вкладка Основы).** 

![Включить опцию во время создания лаборатории](../media/allow-lab-creator-pick-lab-location/create-lab-account.png)

Эта опция отключена, если вы выберете одноранговую виртуальную сеть для вашей учетной записи лаборатории во вкладке **Advanced.**  

![Когда включена одноранговая виртуальная сеть](../media/allow-lab-creator-pick-lab-location/peer-virtual-network.png)


## <a name="after-the-lab-account-is-created"></a>После создания учетной записи лаборатории
После создания учетной записи лаборатории можно включить или отключить эту опцию, выпустив следующие действия: 

1. На странице **учетной записи Lab** выберите **параметры Lab** в левом меню.
2. Выберите **создателя лаборатории Allow, чтобы выбрать** вариант определения местоположения в лаборатории, если вы хотите разрешить создателю лаборатории выбрать место для лаборатории. Если он отключен, лаборатории автоматически создаются в том же месте, в котором существует учетная запись лаборатории. 
    
    Это поле отключено при выборе виртуальной сети для **виртуального сетевого** поля Peer. Это потому, что лаборатории в лабораторной учетной записи должны быть в том же регионе, как учетная запись лаборатории для них для доступа к ресурсам в одноранговой виртуальной сети. 
1. На панели инструментов щелкните **Сохранить**. 

    ![Настройки лаборатории](../media/allow-lab-creator-pick-lab-location/lab-settings.png)

## <a name="no-virtual-network-and-location-selection-isnt-allowed"></a>Выбор виртуальной сети и местоположения не допускается
В этом сценарии создатель лаборатории Allow не позволил выбрать вариант **определения местоположения в лаборатории.** 

![Отсутствие местоположения лаборатории](../media/allow-lab-creator-pick-lab-location/lab-no-location.png)

Затем создатели лаборатории (преподаватели) не видят возможности выбрать место для лаборатории. Они будут видеть цену в час для каждого размера вариант, доступный для них. При создании лаборатории она будет создана в регионе Azure, находящемся в том же месте, что и область Azure, в которой находится их лабораторная учетная запись. Например, если учетная запись лаборатории находится в **Западной части США,** то лаборатория может быть создана в **южной части центральной части США,** но не будет создана в **Канаде Востоке**. Мы не гарантируем ничего о регионе, который мы выбираем, кроме того, что он находится в месте. Если размер в настоящее время ограничен, то создатель лаборатории увидит флажок, где они могут видеть размеры, которые мы обычно поддерживаем, но в настоящее время недоступны. 

## <a name="in-virtual-network-and-location-selection-isnt-allowed"></a>В виртуальной сети и выбор местоположения не допускается
В этом сценарии создатель лаборатории Allow для выбора варианта **определения местоположения лаборатории** отключен, поскольку вы выбрали одноранговую виртуальную сеть для учетной записи лаборатории. Затем создатели лаборатории увидят тот же экран, что и предыдущий вариант. Поскольку все виртуальные компании должны находиться в том же регионе Azure, что и виртуальная сеть, лаборатория будет создана в том же регионе Azure, в которых находится виртуальная сеть. Если этот конкретный регион ограничен для размера, размер будет отображаться как недоступный. 

## <a name="location-selection-is-enabled"></a>Выбор местоположения включен
При выборе **создателя лаборатории Разрешить выбрать местоположение лаборатории**создатели лаборатории (преподаватели) видят возможность выбора местоположения при создании лаборатории. 

![Выбрать место расположения лаборатории](../media/allow-lab-creator-pick-lab-location/location-selection.png)

Создатели лаборатории видят диапазон цен для всех мест, в которые находится размер, и могут выбрать местоположение. Лаборатория будет создана в любом регионе Azure, который отображает это место.

Если местоположение ограничено, оно не отображается в списке по умолчанию. Расширьте список выпадающих и выберите **недоступные места Show для этого размера.** 

![Показать недоступные места](../media/allow-lab-creator-pick-lab-location/show-unavailable-locations.png)

## <a name="cost"></a>Стоимость
Ранее, цены были основаны на размер VM, который вы выбираете для лаборатории. Теперь цена основана на сочетании операционной системы (ОС), размера и местоположения.

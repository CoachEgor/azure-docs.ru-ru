# <a name="enable-automatic-shutdown-of-vms-on-disconnect"></a>Включение автоматического завершения работы виртуальных машин при отключении
Из этой статьи вы узнаете, как включить или отключить автоматическое завершение работы виртуальных машин лаборатории **Windows 10** (шаблон или учащийся), после прерывания соединения с удаленным рабочим столом. Также можно указать, как долго виртуальные машины должны ожидать повторного подключения пользователя до автоматического завершения работы.

Администратор учетной записи лаборатории может настроить этот параметр для учетной записи лаборатории, в которой будут создаваться лаборатории. Дополнительные сведения см. в разделе [Автоматическое завершение работы виртуальных машин при отключении](how-to-configure-lab-accounts.md) Как владелец лаборатории вы можете переопределить этот параметр при создании лаборатории или после создания лаборатории. 

## <a name="configure-when-creating-a-lab"></a>Настройка при создании лаборатории
На странице "Шаг 3" мастера создания лабораторий можно включить или отключить эту функцию, а также указать, как долго виртуальная машина должна ожидать повторного подключения, прежде чем автоматически завершить работу. 

![Настройка во время создания лаборатории](../media/how-to-enable-shutdown-disconnect/configure-lab-creation.png)

## <a name="configure-after-the-lab-is-created"></a>Настройка после создания лаборатории
Этот параметр можно настроить на странице **Параметры**, как показано на следующем изображении. 

![Настройка после создания лаборатории](../media/how-to-enable-shutdown-disconnect/configure-lab-automatic-shutdown.png)

> [!WARNING]
> Если завершить работу операционной системы Windows (ОС) на виртуальной машине перед отключением сеанса RDP с виртуальной машиной, функция автоматического завершения работы не будет работать должным образом.

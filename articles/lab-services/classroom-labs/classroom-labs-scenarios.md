# <a name="use-classroom-labs-for-trainings"></a>Использование учебных занятий для обучения
Вы можете настроить лабораторию для обучения. Учебные лаборатории служб Azure Labs позволяют создать лабораторию для обучения, где каждый обучаемый использует идентичные и изолированные среды для обучения. Путем применения политик можно настроить лабораторию таким образом, чтобы каждый обучаемый мог использовать среды обучения только при необходимости и при достаточном количестве ресурсов, например виртуальных машин, требуемых для обучения. 

![Учебная лаборатория](../media/classroom-labs-scenarios/classroom.png)

Занятия в учебных лабораториях соответствуют следующим требованиям, необходимым для проведения обучения в любой виртуальной среде. 

- Обучаемые могут быстро подготовить среды для обучения.
- Все машины для обучения должны быть идентичными.
- Обучаемые не видят виртуальные машины, созданные другими обучаемыми.
- Обучаемые не могут получить больше виртуальных машин, чем требуется для обучения, и завершают работу виртуальных машин, которые не используются, что гарантирует контроль расходования ресурсов.
- Каждому обучаемому можно предоставить доступ к учебной лаборатории.
- При необходимости учебные лаборатории можно использовать повторно.

В этой статье вы узнаете о различных функциях служб лаборатории Azure, которые можно использовать для выполнения описанных выше требований к обучению, и подробное описание действий, которые можно выполнить, чтобы настроить лабораторию для обучения.  

## <a name="create-the-lab-account-as-a-lab-account-administrator"></a>Создание учетной записи лаборатории в качестве администратора учетной записи лаборатории
Первым шагом в использовании служб лабораторий Azure является создание учетной записи лаборатории на портал Azure. После того как администратор учетной записи лаборатории создаст учетную запись лаборатории, администратор добавляет пользователей, желающих создать лабораторию к роли **создателя лаборатории** . Преподаватели создают лаборатории с виртуальными машинами для учащихся, чтобы выполнять упражнения по курсу обучения. Дополнительные сведения см. в статье [Создание учетной записи лаборатории и управление ею](how-to-manage-lab-accounts.md).

## <a name="create-and-manage-classroom-labs"></a>Создание и администрирование лабораторий для аудитории
Преподаватель, который является членом роли "создатель лаборатории" в учетной записи лаборатории, может создать одну или несколько лабораторий в учетной записи лаборатории. Вы создадите и настроите шаблон виртуальной машины, используя все необходимое программное обеспечение для выполнения упражнений в вашем курсе. Вы можете выбрать готовый образ из доступных образов для создания лабораторной среды, а затем настроить его, установив программное обеспечение, необходимое для лаборатории. Дополнительные сведения см. в статье [Создание и управление учебными лабораториями](how-to-manage-classroom-labs.md).

## <a name="configure-usage-settings-and-policies"></a>Настройка параметров и политик использования
Создатель лаборатории может добавлять или удалять пользователей в лаборатории, получать ссылку для регистрации для отправки пользователям лаборатории, настраивать политики, такие как Настройка индивидуальных квот на пользователя, обновлять количество виртуальных машин, доступных в лаборатории, и многое другое. Дополнительные сведения см. в разделе [Настройка параметров использования и политик](how-to-configure-student-usage.md).

## <a name="create-and-manage-schedules"></a>Создание расписаний и управление ими
Расписания позволяют настроить лабораторию аудитории, например, виртуальные машины в лаборатории автоматически начинают и завершают работу в указанное время. Вы можете определить разовое расписание или повторяющееся расписание. Дополнительные сведения см. в статье [Создание расписаний для занятий и управление ими](how-to-create-schedules.md).

## <a name="set-up-and-publish-a-template-vm"></a>Настройка и публикация шаблона виртуальной машины
На основе шаблона, в роли которого выступает базовый образ виртуальной машины, в лаборатории создаются все виртуальные машины пользователей. Настройте виртуальную машину шаблона так, чтобы она была настроена точно так, как вы хотите предоставлять участникам обучения. Вы можете указать имя и описание шаблона, которое будет отображаться для пользователей лаборатории. После этого опубликуйте шаблон, чтобы сделать экземпляры шаблона виртуальной машины доступными для пользователей лаборатории. Когда вы опубликуете шаблон, Службы лабораторий Azure создадут виртуальные машины в лаборатории с помощью этого шаблона. Число созданных при этом процессе виртуальных машин соответствует максимальному числу разрешенных в лаборатории пользователей, которое можно настроить в политике использования лаборатории. Все виртуальные машины имеют ту же конфигурацию, что и шаблон. Дополнительные сведения см. в статье [Настройка и публикация шаблонов виртуальных машин](how-to-create-manage-template.md). 

## <a name="use-vms-in-the-classroom-lab"></a>Использование виртуальных машин в лаборатории занятий
Обучающий или обучающий участник регистрируется в лаборатории и подключается к виртуальной машине для выполнения упражнений в рамках курса. Дополнительные сведения см. в статье [как получить доступ к лабораторной работе](how-to-use-classroom-lab.md).

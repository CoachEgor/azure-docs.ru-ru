# <a name="use-classroom-labs-for-trainings"></a>Используйте классные лаборатории для тренировок
Вы можете создать лабораторию для тренировок. Классные лаборатории Лазурных Служб позволяют создать лабораторию для обучения, где каждый стажер использует идентичные и изолированные среды для обучения. Путем применения политик можно настроить лабораторию таким образом, чтобы каждый обучаемый мог использовать среды обучения только при необходимости и при достаточном количестве ресурсов, например виртуальных машин, требуемых для обучения. 

![Классная лаборатория](../media/classroom-labs-scenarios/classroom.png)

Классные лаборатории отвечают следующим требованиям, которые необходимы для проведения обучения в любой виртуальной среде: 

- Обучаемые могут быстро подготовить среды для обучения.
- Все машины для обучения должны быть идентичными.
- Обучаемые не видят виртуальные машины, созданные другими обучаемыми.
- Обучаемые не могут получить больше виртуальных машин, чем требуется для обучения, и завершают работу виртуальных машин, которые не используются, что гарантирует контроль расходования ресурсов.
- Каждому обучаемому можно предоставить доступ к учебной лаборатории.
- При необходимости учебные лаборатории можно использовать повторно.

В этой статье вы узнаете о различных функциях Azure Lab Services, которые могут быть использованы для удовлетворения ранее описанных требований к обучению и подробных шагов, которые можно извлечь для создания лаборатории для обучения.  

## <a name="create-the-lab-account-as-a-lab-account-administrator"></a>Создание учетной записи лаборатории в качестве администратора лабораторных учетных записей
Первым шагом в использовании служб Лаборатории Azure является создание лабораторной учетной записи на портале Azure. После того, как администратор лабораторной учетной записи создает учетную запись лаборатории, администратор добавляет пользователей, которые хотят создать лаборатории, к роли **создателя лаборатории.** Тренеры создают лаборатории с виртуальными машинами для студентов, чтобы делать упражнения для курса, который они преподают. Для получения подробной информации [см.](how-to-manage-lab-accounts.md)

## <a name="create-and-manage-classroom-labs"></a>Создание и администрирование лабораторий для аудитории
Тренер, который является членом роли создателя лаборатории в лабораторной учетной записи, может создать одну или несколько лабораторий в учетной записи лаборатории. Вы создаете и настраиваете шаблон VM со всем необходимым программным обеспечением для выполнения упражнений в вашем курсе. Вы выбираете готовое изображение из доступных изображений для создания классной лаборатории, а затем настраиваете его, установив программное обеспечение, необходимое для лаборатории. Для получения подробной информации [см.](how-to-manage-classroom-labs.md)

## <a name="configure-usage-settings-and-policies"></a>Настройка параметров и политик использования
Создатель лаборатории может добавлять или удалять пользователей в лабораторию, получать регистрационную ссылку для отправки пользователям лаборатории, настраивать политики, такие как установление индивидуальных квот на одного пользователя, обновлять количество вс-м в лаборатории и многое другое. Для получения подробной информации [см.](how-to-configure-student-usage.md)

## <a name="create-and-manage-schedules"></a>Создание расписаний и управление ими
Расписания позволяют настроить лабораторию аудитории, например, виртуальные машины в лаборатории автоматически начинают и завершают работу в указанное время. Вы можете определить разовое расписание или повторяющееся расписание. Для получения подробной информации [см.](how-to-create-schedules.md)

## <a name="set-up-and-publish-a-template-vm"></a>Настройка и публикация шаблона VM
На основе шаблона, в роли которого выступает базовый образ виртуальной машины, в лаборатории создаются все виртуальные машины пользователей. Навлажьте шаблон VM таким образом, чтобы он был настроен с тем, что именно вы хотите предоставить участникам тренинга. Вы можете указать имя и описание шаблона, которое будет отображаться для пользователей лаборатории. После этого опубликуйте шаблон, чтобы сделать экземпляры шаблона виртуальной машины доступными для пользователей лаборатории. Когда вы опубликуете шаблон, Службы лабораторий Azure создадут виртуальные машины в лаборатории с помощью этого шаблона. Число созданных при этом процессе виртуальных машин соответствует максимальному числу разрешенных в лаборатории пользователей, которое можно настроить в политике использования лаборатории. Все виртуальные машины имеют ту же конфигурацию, что и шаблон. Для получения подробной информации смотрите [Настройка и публикация шаблона виртуальных машин.](how-to-create-manage-template.md) 

## <a name="use-vms-in-the-classroom-lab"></a>Использование VMs в классной лаборатории
Студент или участник тренинга регистрируется в лаборатории и подключается к ВМ, чтобы сделать упражнения для курса. Для получения подробной информации, см [Как получить доступ к классной лаборатории](how-to-use-classroom-lab.md).

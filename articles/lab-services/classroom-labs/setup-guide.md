# <a name="classroom-lab-setup-guide"></a>Руководство по настройке лаборатории в классе

Процесс публикации лаборатории для ваших студентов может занять до нескольких часов, в зависимости от количества виртуальных машин (VM), которые будут созданы в вашей лаборатории. Разрешить по крайней мере день, чтобы создать лабораторию, чтобы убедиться, что он работает должным образом и дать достаточно времени для публикации студентов VMs.

## <a name="understand-the-lab-requirements-of-your-class"></a>Понимание лабораторных требований вашего класса

Перед созданием новой лаборатории следует рассмотреть следующие вопросы.

### <a name="what-software-requirements-does-the-class-have"></a>Какие требования к программному обеспечению предъявляется к классу?

Основываясь на целях обучения вашего класса, решите, какая операционная система, приложения и инструменты должны быть установлены на визажах в лабораторию. Для настройки лабораторных ВМ-ит-видео у вас есть три варианта:

- **Используйте изображение Azure Marketplace:** Azure Marketplace предоставляет сотни изображений, которые можно использовать при создании лаборатории. Для некоторых классов одно из этих изображений может уже содержать все, что вам нужно для вашего класса.

- **Создайте новое пользовательское изображение:** Вы можете создать собственное изображение, используя изображение Azure Marketplace в качестве отправной точки, и настроить его путем установки дополнительного программного обеспечения и внесения изменений конфигурации.

- **Используйте существующее пользовательское изображение:** Вы можете повторно использовать существующие пользовательские изображения, которые вы ранее создали, или которые были созданы другими администраторами или преподавателями в вашей школе. Это требует, чтобы администраторы настроили общую галерею изображений, которая является репозиторием для сохранения пользовательских изображений.

> [!NOTE]
> Администраторы отвечают за включение изображений и пользовательских изображений ВАшего базара, чтобы вы могли их использовать. Координируйте свои действия с ИТ-отделом, чтобы гарантировать, что изображения, которые вам нужны, включены. Пользовательские изображения, которые вы создаете, автоматически включены для использования в лабораториях, которыми вы владеете.

### <a name="what-hardware-requirements-does-the-class-have"></a>Какие требования к оборудованию предъявляется к классу?

Есть множество размеров вычислений, которые вы можете выбрать из:

- Вложенные размеры виртуализации, так что вы можете дать доступ к студентам к VM, который способен хостинг несколько, вложенных VMs. Например, этот размер вычисления можно использовать для сетевых курсов.

- Размеры gPU, так что ваши студенты могут использовать компьютерные типы приложений. Например, этот выбор может быть подходящим для искусственного интеллекта и машинного обучения.

Обратитесь к руководству по [размеру VM,](https://docs.microsoft.com/azure/lab-services/classroom-labs/administrator-guide#vm-sizing) чтобы увидеть полный список доступных размеров вычислений.

> [!NOTE]
> В зависимости от области лаборатории, вы можете увидеть меньше вычислений размеров доступны, потому что это зависит от региона. Как правило, следует выбрать наименьший размер вычислений, который ближе всего к вашим потребностям. С помощью служб Лаборатории Лазурного Изл. вы можете создать новую лабораторию с другой вычислительной мощностью позже, если это необходимо.

### <a name="what-dependencies-does-the-class-have-on-external-azure-or-network-resources"></a>Какие зависимости класс имеет от внешних ресурсов Azure или сети?

Если вашим вдовым управляющим лаборатории необходимо использовать внешние ресурсы, такие как база данных, доля файлов или сервер лицензирования, координируйте свои действия с администраторами, чтобы обеспечить доступ к этим ресурсам.

Для доступа к ресурсам Azure, *которые не* защищены виртуальной сетью, администраторам не нужно искать дополнительную конфигурацию. Вы можете получить доступ к этим ресурсам через общественный интернет.

> [!NOTE]
> Следует подумать о том, можно ли уменьшить зависимость лаборатории от внешних ресурсов, предоставляя ресурс непосредственно на VM. Например, чтобы исключить необходимость считывать данные из внешней базы данных, можно установить базу данных непосредственно на VM.  

### <a name="how-will-costs-be-controlled"></a>Как будут контролироваться затраты?

Lab Services использует модель ценообразования с оплатой по мере использования, что означает, что вы платите только за время работы лаборатории VM. Для управления затратами у вас есть три варианта, которые обычно используются в сочетании друг с другом:

- **Расписание**: Расписание позволяет автоматически контролировать, когда ввашихся и выключаются в ваших лабораториях.
- **Квота**: Квота контролирует количество часов, которые студенты будут иметь доступ к VM за пределами запланированных часов. Если квота достигается во время использования учащимся, VM автоматически выключается. Студент не может перезапустить VM, если квота не будет увеличена.
- **Автоматическое выключение**: При включении настройки автоматического выключения приводит к автоматическому отключению компьютеров Windows VMs по определенному времени после того, как студент отключился от сеанса удаленного протокола рабочего стола (RDP). Эта функция отключена по умолчанию.  

    > [!NOTE]
    > Эта настройка в настоящее время существует только для Windows.

### <a name="how-will-students-save-their-work"></a>Как студенты сохранят свою работу?

Каждому учащимся присваивается свой собственный VM, который назначается им на всю жизнь в лаборатории. Они могут выбрать:

- Сохранить непосредственно на VM.
- Сохранить на внешнем месте, например OneDrive или GitHub.

Можно настроить OneDrive автоматически для студентов на их лабораторных VMs.

> [!NOTE]
> Чтобы обеспечить, чтобы ваши студенты продолжали получать доступ к сохраненной работе за пределами лаборатории, и после окончания класса мы рекомендуем учащимся сохранить свою работу во внешнем репозитории.

### <a name="how-will-students-connect-to-their-vm"></a>Как студенты будут подключаться к своим VM?

Для RDP для Windows VMs мы рекомендуем учащимся использовать [клиент Microsoft Remote Desktop.](https://docs.microsoft.com/windows-server/remote/remote-desktop-services/clients/remote-desktop-clients) Удаленный настольный клиент поддерживает компьютеры Mac, Chromebooks и Windows.

Для Linux VMs студенты могут использовать sSH или RDP. Чтобы студенты подключились с помощью RDP, необходимо установить и настроить необходимые пакеты RDP и GUI.

## <a name="set-up-your-lab"></a>Настройка лаборатории

После того как вы поймете требования к лаборатории вашего класса, вы будете готовы настроить его. Следите за ссылками в этом разделе, чтобы узнать, как настроить лабораторию.

1. **Создайте лабораторию.** Обратитесь к учебнику по [созданию классной лаборатории](https://docs.microsoft.com/azure/lab-services/classroom-labs/tutorial-setup-classroom-lab#create-a-classroom-lab) для получения инструкций.

    > [!NOTE]
    > Если вашему классу требуется вложенная виртуализация, [см.](https://docs.microsoft.com/azure/lab-services/classroom-labs/how-to-enable-nested-virtualization-template-vm)

1. **Настраивайте изображения и публикуйте лабораторные VMs.** Подключайтесь к специальному VM под названием шаблон VM. Смотрите шаги в следующих руководствах:
    - [Создание и управление шаблоном VM](https://docs.microsoft.com/azure/lab-services/classroom-labs/tutorial-setup-classroom-lab#publish-the-template-vm)
    - [Использование коллекции общих образов](https://docs.microsoft.com/azure/lab-services/classroom-labs/how-to-use-shared-image-gallery)

    > [!NOTE]
    > Если вы используете Windows, вы также должны увидеть инструкции при [подготовке шаблона Windows VM.](https://docs.microsoft.com/azure/lab-services/classroom-labs/how-to-prepare-windows-template) Эти инструкции включают в себя шаги по настройке OneDrive и Office для ваших студентов.

1. **Управление пулом ИВ и емкостью.** Вы можете легко масштабировать емкость VM вверх или вниз, по мере необходимости вашим классом. Имейте в виду, что увеличение емкости VM может занять несколько часов, потому что это предполагает создание новых VM. Ознакомьтесь с шагами по [настройке и управлению пулом VM.](https://docs.microsoft.com/azure/lab-services/classroom-labs/how-to-set-virtual-machine-passwords)

1. **Добавляйте и управляйте пользователями лабораторий.** Чтобы добавить пользователей в лабораторию, обратитесь к шагам в следующих учебниках:
   - [Добавление пользователей в лабораторию](https://docs.microsoft.com/azure/lab-services/classroom-labs/tutorial-setup-classroom-lab#add-users-to-the-lab)
   - [Отправка приглашений пользователям](https://docs.microsoft.com/azure/lab-services/classroom-labs/tutorial-setup-classroom-lab#send-invitation-emails-to-users)

    Для получения информации о типах учетных записей, которые могут использовать студенты, [см.](https://docs.microsoft.com/azure/lab-services/classroom-labs/how-to-configure-student-usage#student-accounts)
  
1. **Установите контроль затрат.** Чтобы контролировать затраты лаборатории, установите графики, квоты и автоматическое выключение. Ознакомьтесь со следующими руководствами:

   - [Установить расписание](https://docs.microsoft.com/azure/lab-services/classroom-labs/tutorial-setup-classroom-lab#set-a-schedule-for-the-lab)
        > [!NOTE]
        > В зависимости от типа установленной операционной системы запуск VM может занять несколько минут. Чтобы убедиться, что лаборатория VM готова к использованию в запланированные часы, мы рекомендуем вам запустить VMs за 30 минут.

   - [Установите квоты для пользователей](https://docs.microsoft.com/azure/lab-services/classroom-labs/how-to-configure-student-usage#set-quotas-for-users) и [установите дополнительную квоту для конкретного пользователя](https://docs.microsoft.com/azure/lab-services/classroom-labs/how-to-configure-student-usage#set-additional-quotas-for-specific-users)
  
   - [Включение автоматического завершения работы при отключении](https://docs.microsoft.com/azure/lab-services/classroom-labs/how-to-enable-shutdown-disconnect)

        > [!NOTE]
        > Расписание, квоты и автоматическое отключение не применяются к шаблону VM. В результате необходимо убедиться, что вы выключите шаблон VM, когда он не используется. В противном случае он продолжает нести расходы. Кроме того, по умолчанию при создании лаборатории автоматически запускается шаблон VM. Убедитесь, что вы немедленно закончите настройку лаборатории и выключите шаблон VM.

1. **Используйте панель мониторинга.** Для получения [инструкций см.](https://docs.microsoft.com/azure/lab-services/classroom-labs/use-dashboard)

    > [!NOTE]
    > Ориентировочная стоимость, указанная в панели мониторинга, является максимальной стоимостью, которую можно ожидать для использования учащимися лаборатории. Например, ваши студенты *не* будут платить за неиспользованные квотные часы. Сметные затраты *не* отражают никаких расходов на использование шаблона VM или общей галереи изображений.

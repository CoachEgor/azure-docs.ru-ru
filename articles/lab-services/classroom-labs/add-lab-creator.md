# <a name="add-lab-creators-to-a-lab-account-in-azure-lab-services"></a>Добавление создателей лаборатории в учетную запись лаборатории в Лабораторных службах Azure
В этой статье показано, как добавить пользователей в качестве создателей лаборатории в учетную запись лаборатории в Лабораторной службе Azure. Эти виды использования могут создавать классные лаборатории в лабораторном учете. 

## <a name="add-microsoft-user-account-to-lab-creator-role"></a>Добавление учетной записи пользователя Майкрософт в роль создателя лаборатории
Чтобы настроить лабораторию для аудитории в учетной записи лаборатории, вы должны быть членом роли **Создатель лаборатории** в учетной записи лаборатории. Учетная запись, использованная для создания учетной записи лаборатории, автоматически добавляется в эту роль. Этот шаг можно пропустить, если вы планируете использовать ту же учетную запись для создания лаборатории для аудитории. Чтобы использовать другую учетную запись для создания лаборатории для аудитории, сделайте следующее: 

Чтобы предоставить преподавателям разрешение на создание лаборатории для классов, добавьте их в роль **Создатель лаборатории**.

1. На странице **Учетная запись лаборатории** выберите **Управление доступом (IAM)** и на панели инструментов щелкните **+ Добавить назначение ролей**. 

    !["Управление доступом" -> кнопка "Добавить назначение ролей"](../media/tutorial-setup-lab-account/add-role-assignment-button.png)
1. На странице **Добавление назначения ролей** для **роли** выберите **Создатель лаборатории**, выберите пользователя, которого нужно добавить к роли создателя лаборатории, а затем нажмите кнопку **Сохранить**. 

    ![Добавление создателя лаборатории](../media/tutorial-setup-lab-account/add-lab-creator.png)

    > [!NOTE]
    > Если вы добавляете пользователя учетной записи, не являйвасэрого, в качестве создателя лаборатории, обратитесь к [пользователю учетной записи, не относясь к Microsoft, в разделе "Создатель лаборатории".](#add-a-non-microsoft-account-user-as-a-lab-creator) 

## <a name="add-a-non-microsoft-account-user-as-a-lab-creator"></a>Добавление пользователя учетной записи, не являвасебяь корпорации Майкрософт, в качестве создателя лаборатории
Чтобы добавить пользователя в качестве создателя лаборатории, вы используете его учетные записи электронной почты. Следующие типы учетных записей электронной почты могут быть использованы:

- Учетная запись электронной почты, предоставленная офисом 365 Azure Active Directory (AAD) вашего университета. 
- Учетная запись электронной `@outlook.com` `@hotmail.com`почты `@live.com`Майкрософт, например, `@msn.com`,
- Учетная запись электронной почты, не относясь к Microsoft, например учетная запись, предоставленная Yahoo или Google. Однако эти типы учетных записей должны быть связаны с учетной записью Майкрософт.
- Учетная запись GitHub. Эта учетная запись должна быть связана с учетной записью Майкрософт.

### <a name="using-a-non-microsoft-email-account"></a>Использование учетной записи электронной почты, не относящегося к Microsoft
Создатели/инструкторы лаборатории могут использовать учетные записи электронной почты, не относясь к Microsoft, для регистрации и входиный в классную лабораторию.  Однако при входе на портал Lab Services требует, чтобы преподаватели сначала создали учетную запись Майкрософт, связанную с их адресом электронной почты, не относясь к Microsoft.

Многие преподаватели, возможно, уже имеют учетную запись Майкрософт, связанную с их адресами электронной почты, не относящимися к Microsoft. Например, у инструкторов уже есть учетная запись Майкрософт, если они использовали свой адрес электронной почты с другими продуктами или службами Майкрософт, такими как Office, Skype, OneDrive или Windows.  

Когда инструкторы входят на портал Lab Services, им предлагается адрес электронной почты и пароль. Если инструктор попытается войти в учетную запись, не связанную с учетной записью Майкрософт, он получит следующее сообщение об ошибке: 

![Сообщение об ошибке](../media/how-to-configure-student-usage/cant-find-account.png)

Чтобы подписаться на учетную запись Майкрософт, инструкторы должны перейти к [http://signup.live.com](http://signup.live.com).  


### <a name="using-a-github-account"></a>Использование учетной записи GitHub
Преподаватели также могут использовать существующую учетную запись GitHub для регистрации и вхоложена в классную лабораторию. Если у инструктора уже есть учетная запись Майкрософт, связанная с их учетной записью GitHub, он может войти в систему и предоставить свой пароль, как показано в предыдущем разделе. Если они еще не связали свою учетную запись GitHub с учетной записью Майкрософт, они должны выбрать **параметры ввоза:**

![Ссылка на опции ввески](../media/how-to-configure-student-usage/signin-options.png)

На странице **опций ввески** выберите **регистрацию с помощью GitHub.**

![Вопиющий с помощью ссылки GitHub](../media/how-to-configure-student-usage/signin-github.png)

Наконец, им предлагается создать учетную запись Майкрософт, связанную с их учетной записью GitHub. Это происходит автоматически, когда инструктор выбирает **Next.**  Инструктор затем сразу же вписываются и подключены к классной лаборатории.

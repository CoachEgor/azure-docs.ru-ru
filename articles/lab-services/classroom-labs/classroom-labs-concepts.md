# <a name="classroom-labs-concepts"></a>Основные понятия, используемые при работе с лабораториями для аудитории

Следующий список содержит ключевые концепции и определения Лабораторных Служб:

## <a name="quota"></a>Quota

Квота — это ограничение по времени (в часах), которое учитель может установить для того, чтобы ученик использовал лабораторию VM. Он может быть установлен на 0, или определенное количество часов. Если квота установлена до 0, студент может использовать виртуальную машину только при запуске расписания или при ручном входе виртуальной машины для студента.  

Квотные часы учитываются, когда студент сам запускает лабораторию VM.  Если учитель вручную запускает лабораторию VM для студента, квотные часы не используются для этого студента.

## <a name="schedules"></a>Расписания

Расписание – это временные интервалы, которые учитель может создать для класса, чтобы студенческие VMs были доступны для занятий.  Расписание может быть одноразовым или повторяющимся.  Квотные часы не используются при запуске расписания.

Есть три типа расписаний: Стандартный, Только начать и остановить только.

- **Стандартный**.  Это расписание запустит все студенческие VMs в указанное время начала и отключение всех студенческих ВМ в указанное время остановки.
- **Только начните**только .   Это расписание начнется все студентV-м в указанное время.  Студенческие ВМ не будут останавливаться до тех пор, пока студент не остановится свой VM через портал Лабораторных служб Azure или не произойдет только расписание остановок.
- **Только остановитесь.**  Это расписание остановит все студенческие ВМ в указанное время.  

## <a name="template-virtual-machine"></a>Шаблон виртуальной машины

Шаблон виртуальной машины в лаборатории представляет собой базовое виртуальное изображение машины, из которого создаются виртуальные машины всех пользователей. Создатели trainers/lab создали виртуальную машину шаблона и настроили ее с помощью программного обеспечения, которое они хотят предоставить участникам обучения делать лаборатории. При публикации шаблона VM службы Лаборатории Послужбы Лазурного Средства создают или обновляли vMs-лаборатории на основе шаблона VM.

## <a name="user-profiles"></a>Профили пользователей

В этой статье описываются различные профили пользователей в Службах лабораторий Azure.

### <a name="lab-account-owner"></a>Владелец учетной записи лаборатории

Как правило, ИТ-администратор облачных ресурсов организации, которому принадлежит подписка Azure, действует как владелец учетной записи лаборатории и выполняет следующие задачи:

- Настраивает учетную запись лаборатории для вашей организации.
- Настраивает и контролирует политики всех лабораторий.
- Предоставляет другим пользователям в организации разрешения на создание лабораторий в учетной записи лаборатории.

### <a name="professor"></a>Профессор

Как правило, пользователи, такие как учитель или онлайн инструктор, создают классные лаборатории в учетной записи лаборатории. Преподаватель лаборатории выполняет следующие задачи:

- Создает лаборатории для аудитории.
- Создает виртуальные машины в лаборатории.
- Устанавливает соответствующее программное обеспечение на виртуальных машинах.
- Указывает, кто может получить доступ к лаборатории.
- Предоставляет ссылки для регистрации в лабораторию для учащихся.

### <a name="student"></a>Студент

Учащийся выполняет следующие задачи:

- Использует ссылку для регистрации в лаборатории, которую он получает от создателя.
- Подключается к виртуальной машине в лаборатории и использует ее для выполнения работы в классе, заданий и проектов.

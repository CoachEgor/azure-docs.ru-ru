# <a name="classroom-labs-in-azure-lab-services--frequently-asked-questions-faq"></a>Классные лаборатории в Лаборантах Azure — часто задаваемые вопросы (Часто)
Получите ответы на некоторые из наиболее распространенных вопросов о классных лабораториях в лабораториях Azure Lab Services. 

## <a name="quotas"></a>Квоты

### <a name="is-the-quota-per-user-or-per-week-or-per-entire-duration-of-the-lab"></a>Является ли квота на одного пользователя или в неделю или на весь срок действия лаборатории? 
Квота, установленная для лаборатории, предназначена для каждого студента в течение всего срока действия лаборатории. И [запланированное время работы ВМ](how-to-create-schedules.md) не учитывается в отношении квоты, выделенной пользователю. Квота рассчитывается на время, которое выходит за пределы расписания и которое учащийся тратит на виртуальные машины.  Для получения дополнительной информации [Set quotas for users](how-to-configure-student-usage.md#set-quotas-for-users)о квотах см.

### <a name="if-professor-turns-on-a-student-vm-does-that-affect-the-student-quota"></a>Если профессор включает студента VM, это влияет на квоту студента? 
Нет. Это не так. Когда профессор включает студента VM, это не влияет на квоту, выделенную студенту. 

## <a name="schedules"></a>Расписания

### <a name="do-all-vms-in-the-lab-start-automatically-when-a-schedule-is-set"></a>Все ли вмв: в лаборатории начинаются автоматически при установлении расписания? 
Нет. Не все вдыхать. Только VMs, которые назначаются пользователям по расписанию. Чаты автоматически не запускались на чаты ми-мы, не назначенные пользователю. Это по замыслу. 

## <a name="lab-accounts"></a>Учетные записи лаборатории

### <a name="why-am-i-not-able-to-create-a-lab-because-of-unavailability-of-the-address-range"></a>Почему я не могу создать лабораторию из-за отсутствия диапазона адресов? 
Классные лаборатории могут создавать лабораторные VMs в диапазоне IP-адресов, указанных при создании учетной записи лаборатории на портале Azure. При предоставлении диапазона адресов каждая лаборатория, созданная после выделения 512 IP-адресов для лабораторных VMs. Диапазон адресов для лабораторного счета должен быть достаточно большим, чтобы вместить все лаборатории, которые вы собираетесь создать под лабораторным счетом. 

Например, если у вас есть блок /19 - 10.0.0.0/19, этот диапазон адресов вмещает 8192 IP-адреса и 16 лабораторий (8192/512 и 16 лабораторий). В этом случае создание лаборатории не удается на 17-й лаборатории создания.

### <a name="what-port-ranges-should-i-open-on-my-organizations-firewall-setting-to-connect-to-lab-virtual-machines-via-rdpssh"></a>Какие диапазоны портов я должен открыть на настройки брандмауэра моей организации для подключения к виртуальным машинам Lab через RDP/SSH?

Порты: 49152-65535. Классные лаборатории сидят за балансером нагрузки. Каждая лаборатория имеет один публичный IP-адрес, и каждая виртуальная машина в лаборатории имеет уникальный порт. 

Вы также можете увидеть личный IP-адрес каждой виртуальной машины на вкладке **виртуального пула машины** на главной странице для лаборатории на портале Azure. Если переиздать лабораторию, общественный IP-адрес лаборатории не изменится, но частный IP и номер порта каждой виртуальной машины в лаборатории может измениться. Вы можете узнать больше в статье: [Настройки брандмауэра для служб лаборатории Azure.](how-to-configure-firewall-settings.md)

### <a name="what-public-ip-address-range-should-i-open-on-my-organizations-firewall-settings-to-connect-to-lab-virtual-machines-via-rdpssh"></a>Какой общедоступный диапазон IP-адресов я должен открыть на настройках брандмауэра моей организации для подключения к виртуальным машинам Lab через RDP/SSH?
Смотрите [теги IP-адресов Azure IP - Public Cloud,](https://www.microsoft.com/download/details.aspx?id=56519)которое предоставляет общедоступный диапазон IP-адресов для центров обработки данных в Azure. Вы можете открыть IP-адреса для регионов, где находятся ваши лабораторные учетные записи.

## <a name="virtual-machine-images"></a>Образы виртуальных машин

### <a name="as-a-lab-creator-why-cant-i-enable-additional-image-options-in-the-virtual-machine-images-dropdown-when-creating-a-new-lab"></a>Как создатель лаборатории, почему я не могу включить дополнительные параметры изображения в виртуальной машине изображения выпадения при создании новой лаборатории?

Когда администратор добавляет вас как создателя лаборатории в учетную запись лаборатории, вам дается разрешение на создание лабораторий. Но у вас нет разрешений на отослали ные настройки внутри учетной записи лаборатории, включая список включенных изображений виртуальной машины. Чтобы включить дополнительные изображения, свяжитесь с администратором учетной записи лаборатории, чтобы сделать это за вас, или попросите администратора добавить вас в качестве роли вкладчика в учетную запись лаборатории. Роль вкладчика даст вам разрешение на отображение списка изображений виртуальной машины в учетной записи лаборатории.

## <a name="users"></a>Пользователи

### <a name="how-many-users-can-be-in-a-classroom-lab"></a>Сколько пользователей может быть в классной лаборатории?
Вы можете добавить до 400 пользователей в классную лабораторию.

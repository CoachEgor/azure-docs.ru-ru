# <a name="connect-your-labs-network-with-a-peer-virtual-network-in-azure-lab-services"></a>Подключение сети в лаборатории с одноранговой виртуальной сети в службы лабораторий Azure 
В этой статье сведения о пиринге лабораторные занятия по сети с другой сетью. 

## <a name="overview"></a>Обзор
Пиринг виртуальных сетей позволяет легко подключить виртуальные сети Azure. После создания пиринговой связи две виртуальные сети выглядят как одна сеть при подключении. Трафик между виртуальными машинами в пиринговых виртуальных сетях направляется через магистральную инфраструктуру Майкрософт, точно так же как трафик направляется между виртуальными машинами в той же виртуальной сети, через только частные IP-адреса. Дополнительные сведения см. в статье [Пиринг между виртуальными сетями](../../virtual-network/virtual-network-peering-overview.md).

Может потребоваться подключение сети в лаборатории с одноранговой виртуальной сети в некоторых сценариях, включая следующие:

- Программное обеспечение, которое подключается к локальным серверам лицензии для получения лицензии у виртуальных машин в лаборатории
- Виртуальные машины в лаборатории требуется доступ к наборам данных (или любых других файлов) на общих сетевых ресурсах университета. 

Определенные в локальной сети подключены к виртуальной сети Azure, либо через [ExpressRoute](../../expressroute/expressroute-introduction.md) или [шлюза виртуальной сети](../../vpn-gateway/vpn-gateway-about-vpngateways.md). Эти службы должны настраиваться за пределами службы лабораторий Azure. Дополнительные сведения о подключении к локальной сети к Azure с помощью ExpressRoute, см. в разделе [Обзор ExpressRoute]) (.. /expressroute/expressroute-Introduction.MD). Для подключения к локальным ресурсам с помощью шлюза виртуальной сети, шлюза и указан виртуальной сети и учетной записи лаборатории должны находиться в том же регионе.

## <a name="configure-at-the-time-of-lab-account-creation"></a>Настроить во время создания учетной записи лаборатории
При создании новой учетной записи лаборатории, вы можете выбрать существующую виртуальную сеть, в котором показано **одноранговой виртуальной сети** раскрывающегося списка. Выбранной виртуальной сети — connected(peered) для лабораторий, созданных в учетной записи лаборатории. Все виртуальные машины в лабораториях, созданные после внесения этого изменения будет иметь доступ к ресурсам в пиринговой виртуальной сети. 

![Выберите виртуальную сеть для узла](../media/how-to-connect-peer-virtual-network/select-vnet-to-peer.png)

> [!NOTE]
> Подробные пошаговые инструкции по созданию учетной записи лаборатории, см. в разделе [Настройка учетной записи лаборатории](tutorial-setup-lab-account.md)


## <a name="configure-after-the-lab-is-created"></a>Настроить после создания лаборатории
То же свойство можно включить из **конфигурация Labs** вкладке **учетной записи лаборатории** странице, если вы не настроили одноранговой сети во время создания учетной записи лаборатории. Изменения, внесенные в этот параметр применяется только к labs, создаваемых после изменения. Как показано на рисунке, можно включить или отключить **одноранговой виртуальной сети** для лабораторных работ в учетной записи лаборатории. 

![Включение или отключение Пиринговую связь после создания лаборатории](../media/how-to-connect-peer-virtual-network/select-vnet-to-peer-existing-lab.png) 

При выборе виртуальной сети для **одноранговой виртуальной сети** поле **разрешить создатель лаборатории для выбора расположения лаборатории** может быть отключена. Это так, как лаборатории в рамках учетной записи лаборатории должны находиться в том же регионе, что учетной записи лаборатории, чтобы подключаться к ресурсам в одноранговой виртуальной сети. 

> [!IMPORTANT]
> Изменение этого параметра применяется только к labs, созданные после внесения этих изменений, не к существующей лаборатории.

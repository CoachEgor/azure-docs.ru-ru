# <a name="connect-your-labs-network-with-a-peer-virtual-network-in-azure-lab-services"></a>Подключение сети лаборатории с помощью одноранговой виртуальной сети в службах Лабораторной лаборатории Azure 
В этой статье содержится информация о том, как сеть лабораторий внедряет сеть с другой сетью. 

## <a name="overview"></a>Обзор
Виртуальная сеть пиринга позволяет легко подключить виртуальные сети Azure. После создания пиринговой связи две виртуальные сети выглядят как одна сеть при подключении. Трафик между виртуальными машинами в заглянувных виртуальных сетях направляется через магистральной инфраструктуры Microsoft, так же, как трафик направляется между виртуальными машинами в той же виртуальной сети, только через частные IP-адреса. Дополнительные сведения см. в статье [Пиринг между виртуальными сетями](../../virtual-network/virtual-network-peering-overview.md).

Возможно, вам придется подключить сеть лаборатории к одноранговой виртуальной сети в некоторых сценариях, включая следующие:

- Виртуальные машины в лаборатории имеют программное обеспечение, которое подключается к наемным лицензионным серверам для получения лицензии
- Виртуальные машины в лаборатории нуждаются в доступе к наборам данных (или любым другим файлам) в сетевых акциях университета. 

Некоторые сети подключены к виртуальной сети Azure либо через [ExpressRoute,](../../expressroute/expressroute-introduction.md) либо через [Virtual Network Gateway.](../../vpn-gateway/vpn-gateway-about-vpngateways.md) Эти службы должны быть настроены за пределами служб лаборатории Azure. Чтобы узнать больше о подключении к сети в Azure с помощью ExpressRoute, смотрите [обзор ExpressRoute](../../expressroute/expressroute-introduction.md). Для подключения с использованием виртуального сетевого шлюза, шлюза, указанной виртуальной сети и лабораторной учетной записи все должно быть в одном регионе.

> [!NOTE]
> При создании виртуальной сети Azure, которая будет всматриваться с лабораторной учетной записью, важно понимать, как регион виртуальной сети влияет на создание классных лабораторий.  Для получения дополнительной информации обратитесь в раздел руководства администратора по [регионам.](https://docs.microsoft.com/azure/lab-services/classroom-labs/administrator-guide#regions-or-locations)

## <a name="configure-at-the-time-of-lab-account-creation"></a>Настройка во время создания лабораторного счета
Во время создания нового лабораторного счета можно выбрать существующую виртуальную сеть, которая отображается в списке выпадающих сетей **Peer** на вкладке **Advanced.** Выбранная виртуальная сеть подключена (внесебя) к лабораториям, созданным под лабораторным счетом. Все виртуальные машины в лабораториях, которые создаются после внесения этого изменения, будут иметь доступ к ресурсам в виртуальной сети. 

![Выберите VNet для одноранговой](../media/how-to-connect-peer-virtual-network/select-vnet-to-peer.png)

> [!NOTE]
> Подробные пошаговые инструкции по созданию лабораторной учетной записи можно [узнать](tutorial-setup-lab-account.md)


## <a name="configure-after-the-lab-is-created"></a>Настройка после создания лаборатории
То же свойство может быть включено из вкладки **конфигурации Labs** страницы **учетной записи Lab,** если вы не создали одноранговую сеть во время создания учетной записи лаборатории. Изменение, внесенное в этот параметр, относится только к лабораториям, которые создаются после изменения. Как вы можете видеть на изображении, вы можете включить или отключить **виртуальную сеть Peer** для лабораторий в учетной записи лаборатории. 

![Включить или отключить вглядывание VNet после создания лаборатории](../media/how-to-connect-peer-virtual-network/select-vnet-to-peer-existing-lab.png) 

При выборе виртуальной сети для **поля виртуальной сети Peer** создатель лаборатории, чтобы выбрать вариант определения **местоположения лаборатории,** отключается. Это потому, что лаборатории в лабораторной учетной записи должны быть в том же регионе, как учетная запись лаборатории для них, чтобы соединиться с ресурсами в одноранговой виртуальной сети. 

> [!IMPORTANT]
> Это изменение настройки применяется только к лабораториям, которые создаются после изменения, а не к существующим лабораториям.

# <a name="tutorial-set-up-a-classroom-lab"></a>Руководство по Настройка лаборатории для аудитории 
В этом руководстве вы настроите аудиторную лабораторию с виртуальными машинами, используемыми учащимися в аудитории.  

Вот какие действия выполняются в этом руководстве:

> [!div class="checklist"]
> * Создание лаборатории для аудитории
> * Добавление пользователей в лабораторию
> * Отправка ссылки для регистрации учащимся

## <a name="prerequisites"></a>Предварительные требования
Чтобы настроить лабораторию аудитории в учетной записи лаборатории, вы должны быть членом одной из ролей в учетной записи лаборатории: "Владелец", "Создатель лаборатории" или "Участник". Учетная запись, которую вы использовали для создания учетной записи лаборатории, автоматически добавляется в роль владельца.

Владелец лаборатории может назначать роль **создателя лаборатории** другим пользователям. Например, владелец лаборатории может назначить роль создателя лаборатории профессорам. Затем профессора создают лаборатории с виртуальными машинами для своих классов. Учащиеся используют ссылки для регистрации, полученные от профессоров, для регистрации в лаборатории. После регистрации они могут использовать виртуальные машины в лабораториях для работы в классе и дома. См. дополнительные сведения о [назначении пользователям роли создателя лаборатории](tutorial-setup-lab-account.md#add-a-user-to-the-lab-creator-role).


## <a name="create-a-classroom-lab"></a>Создание лаборатории для аудитории

1. Перейдите на [веб-сайт Служб лабораторий Azure](https://labs.azure.com). Обратите внимание на то, что Internet Explorer 11 еще не поддерживается. 
2. Выберите **Вход** и введите свои учетные данные. Службы лабораторий Azure поддерживают учетные записи организации и учетные записи Майкрософт. 
3. В окне **New Lab** (Создание лаборатории) выполните следующие действия. 
    1. Укажите **имя** своей лаборатории. 
    2. Укажите максимальное **число виртуальных машин** в лаборатории. Количество виртуальных машин можно всегда увеличить или уменьшить после создания лаборатории. Дополнительные сведения см. в разделе [Обновление количества виртуальных машин в лаборатории](how-to-configure-student-usage.md#update-number-of-virtual-machines-in-lab)/
    6. Щелкните **Сохранить**.

        ![Создание лаборатории для аудитории](../media/tutorial-setup-classroom-lab/new-lab-window.png)
4. На странице **выбора характеристик виртуальных машин** выполните следующие действия:
    1. Выберите **размер** для виртуальных машин, созданных в лаборатории. В настоящее время допустимы размеры: **малый**, **средний**, **средний (виртуализация)** и **большой** и **GPU**.
    3. Выберите **образ виртуальной машины**, который будет использоваться для создания виртуальных машин в лаборатории. Если вы выберете образ Linux, вы увидите возможность включить для него подключение к удаленному рабочему столу. Дополнительные сведения см. в статье [Enable and use remote desktop for Linux virtual machines in a lab in Azure Lab Services](how-to-enable-remote-desktop-linux.md) (Включение и использование удаленного рабочего стола для виртуальных машин Linux в лаборатории в Службах лабораторий Azure).
    4. Щелкните **Далее**.

        ![Указание характеристик виртуальной машины](../media/tutorial-setup-classroom-lab/select-vm-specifications.png)    
5. На странице **Задание учетных данных** укажите учетные данные по умолчанию для всех виртуальных машин в лаборатории. 
    1. Укажите **имя пользователя** для всех виртуальных машин в лаборатории.
    2. Укажите **пароль** для пользователя. 

        > [!IMPORTANT]
        > Запишите имя пользователя и пароль. Они больше не будут отображаться.
    3. Нажмите кнопку **Создать**. 

        ![Настройка учетных данных](../media/tutorial-setup-classroom-lab/set-credentials.png)
6. На странице **настройки шаблона** отображаются сведения о состоянии создания лаборатории. Создание шаблона лаборатории занимает до 20 минут. 

    ![Настройка шаблона](../media/tutorial-setup-classroom-lab/configure-template.png)
7. По завершении настройки шаблона отобразится следующая страница: 

    ![Страница настройки шаблона по завершении процесса](../media/tutorial-setup-classroom-lab/configure-template-after-complete.png)
8. На странице **Configure template** (Настройка шаблона) выполните следующие действия: Эти действия **необязательно** выполнять для этого руководства.
    2. Подключитесь к шаблону виртуальной машины. Для этого выберите **Подключиться**. Если это шаблон виртуальной машины Linux, выберите протокол, с помощью которого нужно подключиться: SSH или протокол удаленного рабочего стола (если он включен).
    1. Щелкните **Сброс пароля**, чтобы сбросить пароль для виртуальной машины. 
    1. Установите и настройте программное обеспечение в шаблоне виртуальной машины. 
    1. **Остановите** работу виртуальной машины.  
    1. Введите **описание** шаблона.
9. Выберите **Далее** на странице шаблона. 
10. На странице **публикации шаблона** выполните описанные ниже действия. 
    1. Чтобы сразу опубликовать шаблон, выберите **Опубликовать**.  

        > [!WARNING]
        > После публикации отменить это действие будет нельзя. 
    2. Чтобы опубликовать шаблон позже, выберите **Сохранить для использования в будущем**. Вы можете опубликовать шаблон виртуальной машины по завершении работы мастера. Дополнительные сведения о том, как настроить и опубликовать шаблон после завершения работы мастера, см. в [этом разделе](how-to-create-manage-template.md#publish-the-template-vm) статьи [Управление лабораторией для аудитории в решении "Службы лабораторий Azure"](how-to-manage-classroom-labs.md).

        ![Публикация шаблона](../media/tutorial-setup-classroom-lab/publish-template.png)
11. Отобразятся сведения о **ходе публикации** шаблона. Этот процесс может занять до одного часа. 

    ![Публикация шаблона — ход выполнения](../media/tutorial-setup-classroom-lab/publish-template-progress.png)
12. Если шаблон опубликован успешно, отобразится показанная ниже страница. Нажмите кнопку **Готово**.

    ![Публикация шаблона — успешное выполнение](../media/tutorial-setup-classroom-lab/publish-success.png)
1. Откроется **панель мониторинга** лаборатории. 
    
    ![Панель мониторинга лаборатории для аудитории](../media/tutorial-setup-classroom-lab/classroom-lab-home-page.png)
4. Перейдите на страницу **Виртуальные машины**, выбрав соответствующий пункт в меню слева или плитку "Виртуальные машины". Убедитесь, что на странице отображаются виртуальные машины в состоянии **Не назначено**. Эти виртуальные машины еще не назначены учащимся. Они должны находиться в состоянии **Остановлено**. На этой странице вы можете запустить виртуальную машину для учащегося, подключиться к ней, а также остановить или удалить ее. Виртуальные машины на этой странице можете запускать как вы, так и ваши учащиеся. 

    ![Виртуальные машины в остановленном состоянии](../media/tutorial-setup-classroom-lab/virtual-machines-stopped.png)

## <a name="add-users-to-the-lab"></a>Добавление пользователей в лабораторию

1. В меню слева выберите **Пользователи**. По умолчанию включен параметр **Ограничить доступ**. Когда этот параметр включен, даже если пользователь имеет ссылку на регистрацию, он не сможет зарегистрироваться в лаборатории, пока не будет включен в список пользователей. С помощью отправляемой ссылки для регистрации могут зарегистрироваться только пользователи из списка. Ниже описано, как добавить пользователей в список. Параметр **Ограничить доступ** можно отключить, что позволит пользователям регистрироваться в лаборатории, используя ссылку для регистрации. 
2. На панели инструментов выберите **Добавить пользователей**. 

    ![Кнопка "Добавить пользователей"](../media/how-to-configure-student-usage/add-users-button.png)
1. На странице **Добавление пользователей** введите адреса электронной почты пользователей в отдельных строках или одной строке, разделяя их точкой с запятой. 

    ![Добавление адресов электронной почты пользователей](../media/how-to-configure-student-usage/add-users-email-addresses.png)
4. Щелкните **Сохранить**. В списке отобразятся адреса электронной почты и состояние пользователей (зарегистрирован или нет). 

    ![Список пользователей](../media/how-to-configure-student-usage/users-list-new.png)

## <a name="set-quotas-for-users"></a>Настройка квот для пользователей
Можно установить квоты на пользователя, выполнив следующие действия: 

1. Щелкните **Пользователи** в меню слева, если страница еще не активна. 
2. На панели инструментов выберите **Quota per user: 10 hours** (Квота на пользователя: 10 часов). 
3. На странице **Quota per user** (Квота на пользователя) укажите количество часов, которое вы хотите предоставить каждому пользователю (учащемуся): 
    1. **Total number of lab hours per user** (Общее количество часов в лаборатории на пользователя). Пользователи могут использовать свои виртуальные машины в течение заданного времени (указано в этом поле) в **дополнение к расписанию**. При выборе этого параметра введите **количество часов** в текстовом поле. 

        ![Количество часов на одного пользователя](../media/how-to-configure-student-usage/number-of-hours-per-user.png). 
    1. **0 hours (schedule only)** (0 часов (только в запланированное время)). Пользователи могут использовать свои виртуальные машины только в запланированное время или когда вы, как владелец лаборатории, включаете для них виртуальные машины.

        ![0 часов (только в запланированное время)](../media/how-to-configure-student-usage/zero-hours.png)
    4. Щелкните **Сохранить**. 
5. Измененные значения теперь отображаются на панели инструментов. **Quota per user: &lt;number of hours&gt;** . (Квота на пользователя: <количество часов>) 

    ![Квота для пользователя](../media/how-to-configure-student-usage/quota-per-user.png)

## <a name="set-a-schedule-for-the-lab"></a>Настройка расписания для лаборатории
Если для параметра квоты вы выбрали **0 hours (schedule only)** (0 часов (только в запланированное время)), вы должны настроить расписание для лаборатории. В этом учебнике настройте регулярное еженедельное расписание.

1. Перейдите на страницу **Расписания** и на панели инструментов выберите **Добавить расписание**. 

    ![Добавление кнопки расписания на страницу "Расписания"](../media/how-to-create-schedules/add-schedule-button.png)
2. На странице **Добавить расписание** переключитесь на **Еженедельно** вверху. 
3. В разделе **Schedule days (required)** (Запланированные дни (обязательно)) выберите дни, необходимые, чтобы расписание вступило в силу. В следующем примере выбран период "Понедельник — пятница". 
4. В поле **Из** введите **дату начала расписания** или выберите ее, нажав кнопку **Календарь**. Это поле обязательно к заполнению. 
5. В поле **Дата окончания плана** введите или выберите дату окончания, когда виртуальные машины должны прекратить работу. 
6. В разделе **Время начала** выберите время запуска виртуальных машин. Время начала требуется, если не указано время остановки. Если необходимо указать только время остановки, выберите **Remove start event** (Удалить событие начала). Если **Время начала** отключено, то, чтобы включить его, выберите **Add start event** (Добавить событие начала) рядом с раскрывающимся списком. 
7. В разделе **Время остановки** выберите время завершения работы виртуальных машин. Время остановки необходимо, когда время начала не указано. Если необходимо указать только время начала, выберите **Remove stop event** (Удалить событие остановки). Если **Время остановки** отключено, то, чтобы его включить, выберите **Add stop event** (Добавить событие остановки) рядом с раскрывающимся списком.
8. В поле **Time zone (required)** (Часовой пояс (обязательно)) выберите часовой пояс для указанного времени начала и окончания.  
9. В разделе **Примечания** введите описание или заметки, которые соответствуют расписанию. 
10. Щелкните **Сохранить**. 

    ![Еженедельное расписание](../media/how-to-create-schedules/add-schedule-page-weekly.png)

## <a name="send-an-email-with-the-registration-link"></a>Отправка сообщения электронной почты со ссылкой для регистрации

1. Перейдите на страницу **Пользователи**. 
2. Выберите конкретного пользователя или всех пользователей в списке. Чтобы выбрать конкретных пользователей, установите флажки в первом столбце списка. Чтобы выбрать всех пользователей, установите флажок перед заголовком первого столбца (**Имя**) или установите все флажки для всех пользователей в списке. В списке вы увидите **состояние приглашения**.  На следующем изображении состояние приглашения для всех учащихся имеет значение **Invitation not sent** (Приглашение не отправлено). 

    ![Выбор учащихся](../media/tutorial-setup-classroom-lab/select-students.png)
1. Щелкните значок **электронной почты (конверт)** в одной из строк (или) щелкните **Отправить приглашение** на панели инструментов. Вы также можете навести указатель мыши на имя учащегося в списке и увидеть значок электронной почты. 

    ![Отправка ссылки для регистрации по электронной почте](../media/tutorial-setup-classroom-lab/send-email.png)
4. На странице **Отправка ссылки для регистрации по электронной почте** выполните следующие действия. 
    1. Введите **необязательное сообщение**, которое вы хотите отправить учащимся. Письмо автоматически включает ссылку для регистрации. 
    2. На странице **Отправка ссылки для регистрации по электронной почте** выберите **Отправить**. Вы видите, что состояние приглашения меняется на **Отправляется приглашение**, а затем на **Приглашение отправлено**. 
        
        ![Отправленные приглашения](../media/tutorial-setup-classroom-lab/invitations-sent.png)

# <a name="azure-lab-services---administrator-guide"></a>Услуги Лаборатории Лазурного АСВ - Руководство по администраторам
Администраторы информационных технологий (ИТ), управляющие облачными ресурсами университета, как правило, отвечают за создание учетной записи лаборатории для своей школы. После настройки лабораторной учетной записи администраторы или преподаватели создают классные лаборатории, содержащиеся в учетной записи лаборатории. В этой статье содержится обзор ресурсов Azure высокого уровня и рекомендации по их созданию.

![Высокое представление ресурсов Azure на высоком уровне в лабораторной учетной записи](../media/administrator-guide/high-level-view.png)

- Классные лаборатории размещаются в рамках подписки Azure, принадлежащей Azure Lab Services.
- Лабораторные учетные записи, общая галерея изображений и версии изображений размещаются в рамках подписки.
- В одной и той же группе ресурсов можно иметь свою лабораторную учетную запись и общую галерею изображений. На этой диаграмме они находятся в разных группах ресурсов. 

## <a name="subscription"></a>Подписка
Ваш университет имеет одну или несколько подписок На Azure. Подписка используется для управления выставлением счетов и безопасности для всех служб ресурсов Azure, которые используются в ней, включая лабораторные учетные записи.

Взаимосвязь между лабораторной учетной записью и ее подпиской важна, потому что:

- О выставлении счетов сообщается через подписку, содержащую лабораторную учетную запись.
- Вы можете предоставить пользователям в каталоге Active Directory (AD) подписки доступ к службам Лаборатории Azure. Вы можете добавить пользователя в качестве владельца учетной записи лаборатории, создателя лаборатории или владельца лаборатории в классе.

Классные лаборатории и их виртуальные машины (VM) управляются и размещаются для вас в рамках подписки, принадлежащей Azure Lab Services.

## <a name="resource-group"></a>Группа ресурсов
Подписка содержит одну или несколько групп ресурсов. Группы ресурсов используются для создания логических групп ресурсов Azure, которые используются вместе в одном и том же решении.  

При создании учетной записи лаборатории необходимо настроить группу ресурсов, содержащую учетную запись лаборатории. 

При создании [общей галереи изображений](#shared-image-gallery)также требуется ресурсная группа. Вы можете поместить свою лабораторную учетную запись и общую галерею изображений в две отдельные группы ресурсов, что характерно для того, чтобы вы планировали поделиться галереей изображений в различных решениях. Или вы можете поместить их в одну и ту же группу ресурсов.

При создании лабораторной учетной записи можно автоматически создавать и прикреплять общую галерею изображений одновременно.  Этот параметр приводит к созданию лабораторной учетной записи и общей галереи изображений в отдельных группах ресурсов. Вы увидите это поведение при использовании шагов, описанных в этом учебнике: [Настроить общую галерею изображений во время создания учетной записи лаборатории.](how-to-attach-detach-shared-image-gallery.md#configure-at-the-time-of-lab-account-creation) Изображение в верхней части этой статьи также использует эту конфигурацию. 

Мы рекомендуем инвестировать время вперед, чтобы спланировать структуру *not* групп ресурсов, так как после ее создания невозможно изменить группу ресурсов лабораторной учетной записи или общей галереи изображений. Если необходимо изменить группу ресурсов для этих ресурсов, необходимо удалить и воссоздать свою лабораторную учетную запись и галерею изображений.

## <a name="lab-account"></a>Лабораторный счет
Лабораторный счет служит контейнером для одной или нескольких классных лабораторий. При работе с Azure Lab Services обычно имеется только одна учетная запись лаборатории. По мере масштабирования использования лаборатории можно создать больше лабораторных учетных записей.

В следующем списке освещаются сценарии, в которых более одного лабораторного счета могут быть полезными:

- **Управление различными требованиями политики в различных классных лабораториях** 
    
    При настройке учетной записи лаборатории вы устанавливаете политики, которые применяются ко *всем* классным лабораториям в лабораторных работах, таким как:
    - Виртуальная сеть Azure с общими ресурсами, к которым может получить доступ классная лаборатория. Например, в виртуальной сети может быть набор классных лабораторий, которым необходим доступ к общему набору данных.
    - Изображения виртуальной машины (VM), которые классные лаборатории могут использовать для создания виртуальных машин. Например, у вас может быть набор классных лабораторий, которые нуждаются в доступе к [Изображению Data Science VM для](https://azuremarketplace.microsoft.com/marketplace/apps/microsoft-dsvm.linux-data-science-vm-ubuntu) изображения Linux Marketplace. 
    
    Если у вас есть классные лаборатории, которые имеют уникальные требования политики друг от друга, может быть полезно создать отдельные лабораторные учетные записи для управления этими классными лабораториями по отдельности.

- **Отдельный бюджет по лабораторным счетам**
  
    Вместо того, чтобы сообщать обо всех расходах в лаборатории через одну учетную запись лаборатории, вам может понадобиться более четко разделенный бюджет. Например, можно создать лабораторные учетные записи для факультета математики вашего университета, факультета компьютерных наук и т.д., чтобы разделить бюджет между департаментами.  Затем можно просмотреть стоимость каждого отдельного учетной записи лаборатории с помощью [Управления затратами Azure.](https://docs.microsoft.com/azure/cost-management-billing/cost-management-billing-overview)
    
- **Изолировать пилотные лаборатории из лабораторий активного производства**
  
    Возможно, у вас есть случаи, когда вы хотите пилотировать изменения политики для лабораторной учетной записи без потенциального влияния на лаборатории активного производства. В этом типе сценария создание отдельной учетной записи лаборатории для целей пилотирования позволяет изолировать изменения. 

## <a name="classroom-lab"></a>Классная лаборатория
Классная лаборатория содержит виртуальные машины (VM), каждый из которых назначен одному студенту. В общем, вы можете ожидать:

- Есть одна классная лаборатория для каждого класса.
- Создавайте новый набор классных лабораторий каждый семестр (или для каждого таймфрейма, который предлагается вашему классу). Обычно для классов, которые имеют одинаковые потребности изображения, следует использовать [общую галерею изображений](#shared-image-gallery) для повторного использования изображений в лабораториях и семестрах.

Рассмотрим следующие моменты при определении того, как структурировать классные лаборатории:

- **Все ВМ в классной лаборатории развернуты с тем же изображением, которое опубликовано**

    В результате, если у вас есть класс, который требует, чтобы различные лабораторные изображения были опубликованы в то же время, отдельные классные лаборатории должны быть созданы для каждого из них.
  
- **Квота использования устанавливается на лабораторном уровне и применяется ко всем пользователям в лаборатории**
    
    Чтобы установить различные квоты для пользователей, необходимо создать отдельные классные лаборатории. Тем не менее, можно добавить больше часов для конкретного пользователя после того, как вы установили квоту.
  
- **Расписание запуска или выключения устанавливается на лабораторном уровне и применяется ко всем вм-то в лаборатории**

    Как и в предыдущем пункте, если вам нужно установить различные графики для пользователей, необходимо создать отдельные классные лаборатории. 

## <a name="shared-image-gallery"></a>Общая галерея изображений
Общая галерея изображений прилагается к лабораторной учетной записи и служит центральным хранилищем для хранения изображений. Изображение сохраняется в галерее, когда педагог решает экспортировать из шаблона виртуальной машины лаборатории (VM). Каждый раз, когда педагог вносит изменения в шаблон VM и экспорт, новые версии изображения сохраняются при сохранении предыдущих версий.

Преподаватели могут публиковать версию изображений из общей галереи изображений при создании новой классной лаборатории. Хотя галерея может хранить несколько версий изображения, преподаватели могут выбрать только последнюю версию во время создания лаборатории.

Общая галерея изображений является дополнительным ресурсом, который может не понадобиться сразу, когда вы начинаете только с нескольких классных лабораторий. Тем не менее, использование общей галереи изображений имеет много преимуществ, которые полезны, как вы масштабируете, чтобы иметь больше классных лабораторий:

- **Позволяет сохранять и управлять версиями шаблона VM-изображения**

    Полезно создать пользовательское изображение или внести изменения (программное обеспечение, конфигурация и т.д.) в изображение из общедоступной галереи Marketplace.  Например, для преподавателей обычно требуется установка другого программного обеспечения. Вместо того, чтобы требовать от студентов ручной установки этих предварительных реквизитов самостоятельно, различные версии изображения шаблона VM могут быть экспортированы в общую галерею изображений. Эти версии изображений могут быть использованы при создании новых классных лабораторий.
- **Позволяет совместноиспользовать шаблонные Изображения VM в классных лабораториях**

    Вы можете сохранить и повторно использовать изображение так, что вам не придется настраивать изображение с нуля каждый раз, когда вы создаете новую классную лабораторию. Например, если предлагается несколько классов, которые нуждаются в одном и том же изображении, это изображение должно быть создано только один раз и экспортировано в общую галерею изображений, чтобы его можно было совместно делить в классных лабораториях.
- **Обеспечивает доступность изображения посредством репликации**

    При сохранении общей галереи изображений из классной лаборатории изображение автоматически реплицируется в другие [регионы в пределах той же географии.](https://azure.microsoft.com/global-infrastructure/regions/) В случае сбоя в регионе публикация изображения в классную лабораторию не затрагивается, так как можно использовать реплику изображения из другого региона.  Публикация vMs из нескольких реплик также может помочь с производительностью.

Чтобы логически сгруппировать общие изображения, у вас есть несколько вариантов:

- Создание нескольких общих галерей изображений. Каждая учетная запись лаборатории может подключаться только к одной общей галерее изображений, поэтому эта опция также потребует создания нескольких лабораторных учетных записей.
- Или можно использовать единую общую галерею изображений, которая используется несколькими лабораторными учетными записями. В этом случае каждая учетная запись лаборатории может включать только те изображения, которые применимы к классным лабораториям, которые она содержит.

## <a name="regionslocations"></a>Регионы-локации

При настройке ресурсов Службы lab Azure необходимо установить регион (или местоположение) центра обработки данных, в который будет размещен ресурс. Ниже приведены более подробные сведения о том, как регион влияет на каждый из ресурсов, связанных с созданием лаборатории.

### <a name="resource-group"></a>Группа ресурсов

Регион определяет центр обработки данных, где хранится информация о группе ресурсов. Ресурсы Azure, содержащиеся в группе ресурсов, могут находиться в разных регионах от родительских.

### <a name="lab-account"></a>Лабораторный счет

Местоположение учетной записи лаборатории указывает область, в которую существует этот ресурс.  

### <a name="classroom-lab"></a>Классная лаборатория
    
Расположение, в которое существует классная лаборатория, варьируется в зависимости от следующих факторов:

  - **Учетная запись лаборатории присматривается к виртуальной сети (VNet)**
  
    Лабораторная учетная запись может быть [заглянула в VNet,](https://docs.microsoft.com/azure/lab-services/classroom-labs/how-to-connect-peer-virtual-network) когда она находятся в одном регионе.  При загляну в лабораторию с помощью VNet классные лаборатории автоматически создаются в том же регионе, что и лабораторная учетная запись и VNet.

    > [!NOTE]
    > Когда учетная запись лаборатории заглядывается с помощью VNet, настройка, **позволяющая создателю лаборатории выбирать местоположение лаборатории,** отключается. Дополнительную информацию об этом параметре можно найти в статье: [Разрешить создателю лаборатории выбрать место для лаборатории.](https://docs.microsoft.com/azure/lab-services/classroom-labs/allow-lab-creator-pick-lab-location)
    
  - **Нет VNet заглянул ***и*** лаборатории создатели не могут выбрать местоположение лаборатории**
  
    Когда **нет** VNet заглянул с учетной записью лаборатории *и* [создатели лаборатории **не** могут выбрать расположение лаборатории,](https://docs.microsoft.com/azure/lab-services/classroom-labs/allow-lab-creator-pick-lab-location)классные классные лаборатории автоматически создаются в регионе, который имеет доступную емкость VM.  В частности, Службы лаборатории Azure ищут доступность в [регионах, которые находятся в той же географии, что и учетная запись лаборатории.](https://azure.microsoft.com/global-infrastructure/regions)

  - **Нет VNet заглянул ***и*** лаборатории создатели имеют право выбрать расположение лаборатории**
       
    Когда **нет** VNet заглянул и [лаборатории создатели могут выбрать расположение лаборатории](https://docs.microsoft.com/azure/lab-services/classroom-labs/allow-lab-creator-pick-lab-location), места, которые могут быть выбраны создателем лаборатории основаны на доступной емкости.

> [!NOTE]
> Чтобы обеспечить достаточную емкость ВМ для региона, важно сначала запросить емкость через лабораторный счет или при создании лаборатории.

Общее правило заключается в установлении региона ресурса к региону, который ближе всего к его пользователям. Для классных лабораторий это означает создание классной лаборатории, наиболее близкой к вашим ученикам. Для онлайн-курсов, где студенты расположены по всему миру, вам нужно использовать ваши лучшие суждения, чтобы создать классную лабораторию, которая находится в центре. Или разделите класс на несколько классных лабораторий, основанных на регионе ваших студентов.

### <a name="shared-image-gallery"></a>Общая галерея изображений

## <a name="manage-identity"></a>Управление личностью
Используя [элементы управления доступом на основе роли Azure,](https://docs.microsoft.com/azure/role-based-access-control/overview)можно назначить следующие роли для предоставления доступа к лабораторным счетам и классным лабораториям:

- **Владелец учетной записи лаборатории**

    Администратор, создающий учетную запись лаборатории, автоматически добавляется к роли **владельца** учетной записи лаборатории.  Администратор, назначенный роли **владельца,** может:
     - Измените настройки учетной записи лаборатории.
     - Предоставь другим администраторам доступ к учетной записи лаборатории в качестве владельцев или участников. 
     - Предоставь преподавателям доступ к классным лабораториям в качестве создателей, владельцев или участников.
     - Создавайте и управляйте всеми классными лабораториями в лабораторных счетах.

- **Вкладчик учетной записи лаборатории**

    Администратор, назначенный ролью **вкладчика,** может:
    - Измените настройки учетной записи лаборатории.
    - Создавайте и управляйте всеми классными лабораториями в лабораторных счетах.
    
    Однако они *не могут* предоставить другим пользователям доступ ни к лабораторным счетам, ни к классным лабораториям.

- **Создатель лаборатории класса**

    Для создания классных лабораторий в лабораторных счетах преподаватель должен быть членом роли **создателя лаборатории.**  Когда педагог создает классную лабораторию, они автоматически добавляются как владелец лаборатории.  Обратитесь к учебнику о том, как [добавить пользователя в роль **создателя лаборатории.** ](https://docs.microsoft.com/azure/lab-services/classroom-labs/tutorial-setup-lab-account#add-a-user-to-the-lab-creator-role) 

- **Владелец лаборатории в классе**
  
    Педагог может просматривать и изменять настройки классной лаборатории, когда они являются членом роли **владельца** лаборатории или **автора;** они также должны быть членом роли **читателя** лабораторного счета.

    Ключевое различие между ролями **владельца** лаборатории и **участников** заключается в том, что участник *не может* предоставить другим пользователям доступ к управлению лабораторией - только владельцы могут предоставить другим пользователям доступ к управлению лабораторией.
    
    Кроме того, педагог *не может* создавать новые классные лаборатории, если они также не являются членами роли **создателя лаборатории.**

- **Общая галерея изображений**
    
    При присоединении общей галереи изображений к учетной записи лаборатории владельцам лабораторных учетных записей и создателям лабораторий автоматически предоставляется доступ к просмотру и сохранению изображений в галерее. 

Вот несколько советов, которые помогут при назначении ролей:
   - Как правило, только администраторы должны быть членами ролей **владельца** или **вкладчика** учетной записи лаборатории; у вас может быть более одного владельца.

   - Дать преподавателю возможность создавать новые классные лаборатории и управлять лабораториями, которые они создают; вам нужно только назначить доступ к роли **создателя лаборатории.**
   
   - Дать преподавателю возможность управлять конкретными классными лабораториями, но *не* возможность создавать новые лаборатории; вы должны назначить доступ к роли **владельца** или **вкладчика** для каждой из классных лабораторий, которые они будут управлять.  Например, можно разрешить как профессору, так и ассистенту преподавателя совместно владеть классной лабораторией.  Обратитесь к руководству о том, как [добавить пользователя в качестве владельца в классную лабораторию.](https://docs.microsoft.com/azure/lab-services/classroom-labs/how-to-add-user-lab-owner)

## <a name="pricing"></a>Цены

### <a name="azure-lab-services"></a>Службы лабораторий Azure
Цены на лабораторные службы Azure описаны в следующей статье: [Цены на лабораторные услуги Azure](https://azure.microsoft.com/pricing/details/lab-services/)Lab .

Вы также должны рассмотреть цены на общую галерею изображений, если вы планируете использовать его для хранения и управления версиями изображений. 

### <a name="shared-image-gallery"></a>Общая галерея изображений
Создание общей галереи изображений и присоединение ее к вашей учетной записи лаборатории бесплатно. Затраты не понесены, пока вы не сохраните версию изображения в галерее. Как правило, цены на использование общей галереи изображений довольно ничтожны, но важно понимать, как она рассчитывается, поскольку она не включена в цены для служб Лаборатории Azure.  

#### <a name="storage-charges"></a>Плата за хранение
Для хранения версий изображений общая галерея изображений использует стандартные диски с HDD. Размер используемого диска с HDD зависит от размера хранимой версии изображения. Смотрите следующую статью для просмотра цен: [Управляемые диски ценообразования](https://azure.microsoft.com/pricing/details/managed-disks/).


#### <a name="replication-and-network-egress-charges"></a>Репликация и сборы за выход из сети
При сохранении версии изображения с помощью виртуальной машины (VM) лаборатории класса сначала хранит ее в исходном регионе, а затем автоматически воспроизводит исходную версию изображения в один или несколько целевых регионов. Важно отметить, что Службы Лаборатории Азов автоматически воспроизводит исходную версию изображения во все [целевые регионы в географическом](https://azure.microsoft.com/global-infrastructure/regions/) классе, где расположена классная лаборатория. Например, если ваша классная лаборатория находится в географии США, версия изображения реплицируется в каждом из восьми регионов, которые существуют в США.

Заряд выведения сети возникает при реплицировании версии изображения из исходного региона в дополнительные целевые регионы. Сумма, взимаемая, зависит от размера версии изображения при первоначальной передаче данных изображения, исходящих из исходного региона.  Для получения подробной информации о ценах, обратитесь к следующей статье: [Bandwidth цен ы детали](https://azure.microsoft.com/pricing/details/bandwidth/).

[Образовательные решения клиенты](https://www.microsoft.com/licensing/licensing-programs/licensing-for-industries?rtc=1&activetab=licensing-for-industries-pivot:primaryr3) могут быть отменены от уплаты сборов за выход. Поговорите со своим менеджером, чтобы узнать больше.  Для получения дополнительной информации, **см. обратитесь к разделу часто задаваемых вопросов** в связанном документе, в частности, вопрос "Какие программы передачи данных существуют для академических клиентов и как я могу претендовать?".

#### <a name="pricing-example"></a>Пример ценообразования
Чтобы резюмировать цены, описанные выше, давайте рассмотрим пример сохранения нашего шаблона VM изображения в общей галерее изображений. Предположим следующие сценарии:

- У вас есть одно пользовательское изображение VM.
- Вы сохраняете две версии изображения.
- Ваша лаборатория находится в США, где в общей сложности восемь регионов.
- Каждая версия изображения имеет размер 32 ГБ; в результате, HDD-управляемый диск цена составляет $ 1,54 в месяц.

Общая стоимость оценивается как:

Количество изображений , количество версий , количество реплик и управляемая цена диска

В этом примере стоимость:

1 пользовательское изображение (32 ГБ) x 2 версии x 8 регионов США x $1.54 и $24.64 в месяц

#### <a name="cost-management"></a>управления затратами;
Администратору учетной записи лаборатории важно управлять затратами, регулярно удаляя ненужные версии изображений из галереи. 

Не следует удалять репликацию в определенные регионы для снижения затрат (эта опция существует в общей галерее изображений). Изменения репликации могут негативно сказаться на способности службы Azure Lab публиковать видеоизображения на изображениях, сохраненных в общей галерее изображений.

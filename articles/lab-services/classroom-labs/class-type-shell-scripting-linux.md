# <a name="set-up-a-lab-to-teach-shell-scripting-on-linux"></a>Настройка лаборатории для обучения сценарию оболочки на Linux
В этой статье показано, как создать лабораторию для обучения сценарию оболочки на Linux. Скрипты — это полезная часть системного администрирования, которая позволяет администраторам избежать повторяющихся задач. В этом примере сценария класс охватывает традиционные bash-скрипты и расширенные скрипты. Расширенные скрипты — это скрипты, сочетающие bash-команды и Ruby. Этот подход позволяет Ruby передавать данные и bash-команды для взаимодействия с оболочкой. 

Студенты, посещающие эти занятия по написанию сценариев, получают виртуальную машину Linux для изучения основ Linux, а также знакомятся со скриптами оболочки bash. Виртуальная машина Linux поставляется с включенным доступом к удаленному рабочему столу и установленными текстовыми редакторами [gedit](https://help.gnome.org/users/gedit/stable/) и [ Visual Studio Code](https://code.visualstudio.com/).

## <a name="lab-configuration"></a>Настройка лаборатории
Чтобы настроить эту лабораторию, для начала работы требуется подписка Azure. Если у вас еще нет подписки Azure, [создайте бесплатную учетную запись](https://azure.microsoft.com/free/), прежде чем начинать работу. Если у вас есть подписка Azure, можно либо создать новую учетную запись лаборатории в Лабораторных службах Azure, либо использовать существующую учетную запись лаборатории. Смотрите следующий учебник для создания новой учетной записи лаборатории: [Учебник по настройке учетной записи лаборатории](tutorial-setup-lab-account.md).

Для получения дополнительной информации смотрите [изображения Specify Marketplace, доступные создателям лабораторий.](specify-marketplace-images.md) |

## <a name="install-ruby"></a>Установка Ruby
Ruby — это динамический язык с открытым исходным кодом, который можно сочетать со скриптами bash. В этом разделе `apt-get` показано, как использовать для установки последней версии [Ruby](https://www.ruby-lang.org/).

1. Установите обновления, запустив следующие команды:

    ```bash
    sudo apt-get update 
    sudo apt-get upgrade 
    ```
2.  Установка [Ruby](https://www.ruby-lang.org/).  Ruby — это динамический язык с открытым исходным кодом, который можно сочетать со скриптами bash. 
    
    ```bash
    sudo apt-get install ruby-full
    ```

## <a name="install-development-tools"></a>Установка средств разработки
В этом разделе показано, как установить несколько текстовых редакторов. Gedit — это текстовый редактор по умолчанию для среды рабочего стола gnome. Он разработан в качестве редактора текста общего назначения. Visual Studio Code — это текстовый редактор, который включает поддержку отладки и интеграции управления исходным кодом.

> [!NOTE]
> Есть несколько различных текстовых редакторов доступны. Визуальный код студии и gedit являются лишь двумя примерами.

1. Установка [gedit](https://help.gnome.org/users/gedit/stable/).

    ```bash
    sudo apt-get install gedit
    ```
1. Установка [визуального кода студии](https://code.visualstudio.com/).  Визуальный код студии может быть установлен с помощью Snap Store.  Для альтернативных вариантов установки, см [Visual Studio Код альтернативные загрузки](https://code.visualstudio.com/#alt-downloads).

    ```bash
    sudo snap install vscode --classic 
    ```

    Шаблон теперь обновляется и имеет как язык программирования, так и инструменты разработки, необходимые для завершения лаборатории. Изображение шаблона теперь может быть опубликовано в лаборатории. Выберите кнопку **«Публикация»** на странице шаблона, чтобы опубликовать шаблон в лаборатории.

# <a name="set-up-and-manage-virtual-machine-pool"></a>Настройка и администрирование пула виртуальных машин 
В этой статье показано, как выполнять следующие задачи:

- Увеличьте количество виртуальных машин (VM) в лаборатории
- Запуск всех VMs или выбранных VMs 
- Сбросить VMs

## <a name="update-the-lab-capacity"></a>Обновление емкости лаборатории
Чтобы увеличить или уменьшить емкость лаборатории (количество виртуальных машин в лаборатории), сделайте следующие шаги:

1. На странице **виртуального пула машин** выберите **емкость &lt;лаборатории: номер&gt; машины.**
2. Введите новое **число виз,** которые вы хотите в лаборатории. Это число должно быть больше или равно числу пользователей, зарегистрированных в лаборатории. 
3. Затем выберите **Сохранить**. 

    ![Запустите все кнопки](../media/how-to-set-virtual-machine-passwords/number-of-vms-in-lab.png)
4. При увеличении емкости можно увидеть создаваемые VM или VM. Если вы не видите новый VM в списке, обновите страницу. 

    ![Создается ВМ](../media/how-to-set-virtual-machine-passwords/vm-being-created.png)

## <a name="start-vms"></a>Запуск виртуальных машин

### <a name="start-ot-stop-all-vms"></a>Начните ot остановить все VMs
1. Переключитесь на страницу **виртуального пула машин.** 
2. Выберите **«Начать все** из панели инструментов». 

    ![Запустите все кнопки](../media/how-to-set-virtual-machine-passwords/start-all-vms-button.png)
3. После запуска всех вм- информации можно остановить все ввоза, выбрав кнопку **Stop all** на панели инструментов. 

    ![Остановить все кнопки](../media/how-to-set-virtual-machine-passwords/stop-all-vms-button.png)

### <a name="start-selected-vms"></a>Запуск выбранных VMs
Существует два способа запуска выбранных ВМ (один или более). Первый способ состоит в том, чтобы выбрать VM или VMs в списке, а затем выбрать **Начало** на панели инструментов. 

Второй способ заключается в выборе одного или нескольких вс-да в списке и переключении кнопки в столбце **Состояния.** 

![Запуск выбранных VMs](../media/how-to-set-virtual-machine-passwords/start-selected-vms.png)

Аналогичным образом, можно остановить один или несколько вс-ка, перерезая кнопку в столбце **состояния** или выбрав **Стоп** на панели инструментов. 

> [!NOTE]
> Когда педагог включает студента VM, квота для студента не влияет. Квота для пользователя определяет количество лабораторных часов, доступных пользователю за пределами запланированного времени занятий. Для получения дополнительной информации [Set quotas for users](how-to-configure-student-usage.md?#set-quotas-for-users)о квотах см.

## <a name="reset-vms"></a>Сбросить VMs
Чтобы сбросить один или несколько виз, выберите их в списке, а затем выберите **сбросить** на панели инструментов. 

![Сбросить выбранные VM](../media/how-to-set-virtual-machine-passwords/reset-vm-button.png)

На поле диалога **виртуальной машины Reset (s)** выберите **Reset.** 

![Сбросить диалоговую коробку VM](../media/how-to-set-virtual-machine-passwords/reset-vms-dialog.png)



## <a name="set-password-for-vms"></a>Установка пароля для vMs
Владелец лаборатории (преподаватель) может установить/сбросить пароль для vMs во время создания лаборатории (мастер создания лаборатории) или после создания лаборатории на странице **шаблона.** 

### <a name="set-password-at-the-time-of-lab-creation"></a>Установите пароль во время создания лаборатории
Владелец лаборатории (преподаватель) может установить пароль для виртуальных внештатных компьютеров в лаборатории на странице **учетных данных виртуальной машины** мастера создания лаборатории.

![Окно создания лаборатории](../media/tutorial-setup-classroom-lab/virtual-machine-credentials.png)

Включив/отключив **один и тот же пароль для всех виртуальных машин** на этой странице, учитель может использовать один и тот же пароль для всех виртуальных машин в лаборатории или разрешить учащимся устанавливать пароли для своих виртуальных машин. По умолчанию эта настройка включена для всех изображений операционной системы Windows и Linux, за исключением Ubuntu. Когда эта настройка отключена, студентам будет предложено установить пароль, когда они пытаются подключиться к VM в первый раз. 

### <a name="reset-password-later"></a>Сбросить пароль позже

1. На странице **шаблона** лаборатории выберите **пароль сбросить на** панели инструментов. 
1. В поле диалога **с отбрасывания введите** пароль и выберите **пароль сбросить.**
    
    ![Установка окна диалога пароля](../media/how-to-set-virtual-machine-passwords/set-password.png)

## <a name="connect-to-student-vms"></a>Подключение к студенческим VMs
Создатель лаборатории (преподаватель/профессор) может подключиться к студенту VM, если будут выполнены следующие условия: 

- При создании лаборатории был выбран **один и тот же пароль для всех виртуальных машин**
- ВМ работает 

 Чтобы подключиться к студенту VM, нависните мышь на VM в списке и выберите кнопку компьютера.  

![Подключение к кнопке студенческого VM](../media/how-to-set-virtual-machine-passwords/connect-student-vm.png)

> [!NOTE]
> Когда профессор запускает VM и подключается к нему, студенческая квота не влияет. 

## <a name="export-list-of-virtual-machines-to-a-csv-file"></a>Экспортный список виртуальных машин в файл CSV

1. Переключитесь на вкладку **виртуального пула машин.**
2. Выберите **...** (эллипсис) на панели инструментов, а затем выберите **Экспорт CSV.** 

    ![Экспортный список виртуальных машин](../media/how-to-export-users-virtual-machines-csv/virtual-machines-export-csv.png)

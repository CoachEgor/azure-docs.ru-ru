# <a name="enable-remote-desktop-for-linux-virtual-machines-in-a-lab-in-azure-lab-services"></a>Включить удаленный рабочий стол для виртуальных машин Linux в лаборатории в Лабораторной службе Azure
В этой статье показано, как выполнять следующие задачи:

- Включить удаленный рабочий стол для Linux VM
- Как учитель может подключиться к шаблону VM через удаленное подключение к рабочему столу (RDP).

## <a name="enable-remote-desktop-for-linux-vm"></a>Включить удаленный рабочий стол для Linux VM
Во время создания лаборатории преподаватели могут включить **удаленное подключение к рабочему столу** для **изображений Linux.** Опция **«Включить удаленное подключение к рабочему столу»** отображается при выборе изображения Linux для шаблона. Когда эта опция включена, учителя могут подключиться к шаблону VM и студенческим VMs через RDP (Remote Desktop). 

![Включить удаленное подключение к рабочему столу для изображения Linux](../media/how-to-enable-remote-desktop-linux/enable-rdp-option.png)

На **включенном удаленном** устройстве соединения рабочего стола выберите **Продолжить с помощью удаленного рабочего стола.** 

![Включить удаленное подключение к рабочему столу для изображения Linux](../media/how-to-enable-remote-desktop-linux/enabling-remote-desktop-connection-dialog.png)

> [!IMPORTANT] 
> Включение **удаленного соединения рабочего стола** открывает порт **RDP** только на машинах Linux. Если RDP уже установлен и настроен на изображение виртуальной машины (например: Ubuntu Data Science Virtual Machine image), вы/студенты можете подключиться к виртуальным изображениям через RDP, не выполняя никаких дополнительных шагов.
> 
> Если изображение VM не установлено и настроено RDP, необходимо впервые подключиться к машине Linux с помощью SSH и установить пакеты RDP и GUI, чтобы вы/студенты могли подключиться к машине Linux с помощью RDP позже. Для получения дополнительной информации [см. Установить и настроить удаленный рабочий стол для подключения к Linux VM в Azure.](../../virtual-machines/linux/use-remote-desktop.md) Затем вы публикуете изображение так, чтобы студенты могли RDP в студенческих Linux VMs. 

## <a name="supported-operating-systems"></a>Поддерживаемые операционные системы
В настоящее время удаленное подключение к рабочему столу поддерживается для следующих операционных систем:

- openSUSE Leap 42.3
- Версия 7.5 на основе CentOS
- Debian 9 "Stretch"
- Ubuntu Server 16.04 LTS

## <a name="connect-to-the-template-vm"></a>Подключение к шаблону виртуальной машины 
Учителя должны подключиться к шаблону VM, сначала используя SSH, и установить на него пакеты RDP и GUI. Затем учителя могут использовать RDP для подключения к шаблону VM: 

1. Если вы **видите шаблон Customize** на панели инструментов, выберите его. Затем выберите **Продолжить** на **настраиваемый** шаблон диалоговом поле. Это действие запускает шаблон VM.  

    ![Настройка шаблона](../media/how-to-enable-remote-desktop-linux/customize-template.png)
2. После запуска шаблона VM можно выбрать **шаблон Connect,** а затем **подключиться через SSH** на панели инструментов. 

    ![Подключение к шаблону через RDP после создания лаборатории](../media/how-to-enable-remote-desktop-linux/rdp-after-lab-creation.png) 
3. Вы видите следующее **Подключение к вашей виртуальной машине** диалоговая коробка. Выберите кнопку **Copy** рядом с текстовым полем, чтобы скопировать ее на буфер обмена. Сохранить строку соединения SSH. Используйте эту строку подключения из терминала SSH (например [PuTTY](https://www.putty.org/)) для подключения к виртуальной машине.
 
    ![Строка соединения SSH](../media/how-to-enable-remote-desktop-linux/ssh-connection-string.png)
4. Установите пакеты RDP и GUI, чтобы вы/студенты могли подключиться к машине Linux с помощью RDP позже. Для получения дополнительной информации [см. Установить и настроить удаленный рабочий стол для подключения к Linux VM в Azure.](../../virtual-machines/linux/use-remote-desktop.md) Затем вы публикуете изображение так, чтобы студенты могли RDP в студенческих Linux VMs.
5. После установки этих пакетов можно использовать **шаблон Connect to template** на панели инструментов, а затем выбрать Connect через **RDP** для подключения к шаблону VM через RDP. Сохранить файл RDP и использовать его для подключения к шаблону VM через RDP.

# <a name="configure-lab-accounts-in-azure-lab-services"></a>Настройка учетных записей лаборатории в службах лаборатории Azure 
В службах лаборатории Azure лабораторная учетная запись — это контейнер для управляемых лабораторных типов, таких как учебные лаборатории. Администратор задает учетную запись лаборатории с помощью Служб лабораторий Azure и предоставляет доступ владельцам лабораторий, которые могут создать лаборатории в учетной записи. В этой статье содержатся подробные сведения о том, как создавать, просматривать и удалять учетные записи лабораторий.

## <a name="connect-with-a-peer-virtual-network"></a>Подключение с помощью одноранговой виртуальной сети
Чтобы подключить виртуальную сеть в качестве одноранговой сети к виртуальной сети лаборатории, выполните следующие действия.

1. На странице **учетная запись лаборатории** в меню слева выберите **Конфигурация Labs** .

    ![Страница конфигурации Labs](../media/how-to-manage-lab-accounts/labs-configuration-page.png) 
1. Для **одноранговой виртуальной сети**выберите **включено** или **отключено**. Значение по умолчанию — **disabled**. Чтобы включить одноранговую виртуальную сеть, выполните следующие действия. 
    1. Выберите **Включено**.
    2. Выберите **виртуальную сеть** из раскрывающегося списка. 
3. На панели инструментов щелкните **Сохранить**. 

Лабораторные работы, созданные в этой учетной записи, подключены к выбранной виртуальной сети. Они могут получить доступ к ресурсам в выбранной виртуальной сети. Дополнительные сведения см. [в статье подключение сети лаборатории к одноранговой виртуальной сети в службах лаборатории Azure](how-to-connect-peer-virtual-network.md).

Если выбрана виртуальная сеть для поля **одноранговая виртуальная сеть** , параметр **Разрешить создателю лаборатории выбрать расположение лаборатории** будет отключен. Это связано с тем, что лаборатории в учетной записи лаборатории должны находиться в том же регионе, что и учетная запись лаборатории для подключения к ресурсам в одноранговой виртуальной сети. 

## <a name="specify-an-address-range-for-vms-in-the-lab"></a>Укажите диапазон адресов для виртуальных машин в лаборатории
Ниже описана процедура указания диапазона адресов для виртуальных машин в лаборатории. Если вы обновляете ранее указанный диапазон, измененный диапазон адресов применяется только к виртуальным машинам, созданным после внесения изменений. 

Ниже приведены некоторые ограничения, которые следует учитывать при указании диапазона адресов. 

- Префикс должен быть меньше или равен 23. 
- Если виртуальная сеть является одноранговой для учетной записи лаборатории, указанный диапазон адресов не может перекрывать диапазон адресов от одноранговой виртуальной сети.

1. На странице **учетная запись лаборатории** в меню слева выберите **Конфигурация Labs** .
2. В поле **диапазон адресов** укажите диапазон адресов для виртуальных машин, которые будут созданы в лаборатории. Диапазон адресов должен находиться в нотации с независимым междоменным маршрутизацией (CIDR) (например: 10.20.0.0/23). В этом диапазоне адресов будут создаваться виртуальные машины в лаборатории.
3. На панели инструментов щелкните **Сохранить**. 

    ![Настройка диапазона адресов](../media/how-to-manage-lab-accounts/labs-configuration-page-address-range.png)

## <a name="add-a-user-to-the-lab-creator-role"></a>Добавление пользователя к роли создателя лаборатории
Чтобы настроить лабораторию для аудитории в учетной записи лаборатории, вы должны быть членом роли **Создатель лаборатории** в учетной записи лаборатории. Учетная запись, использованная для создания учетной записи лаборатории, автоматически добавляется в эту роль. Этот шаг можно пропустить, если вы планируете использовать ту же учетную запись для создания лаборатории для аудитории. Чтобы использовать другую учетную запись для создания лаборатории для аудитории, сделайте следующее: 

Чтобы предоставить преподавателям разрешение на создание лаборатории для классов, добавьте их в роль **Создатель лаборатории**.

1. На странице **Учетная запись лаборатории** выберите **Управление доступом (IAM)** и на панели инструментов щелкните **+ Добавить назначение ролей**. 

    !["Управление доступом" -> кнопка "Добавить назначение ролей"](../media/tutorial-setup-lab-account/add-role-assignment-button.png)
1. На странице **Добавление назначения ролей** для **роли** выберите **Создатель лаборатории**, выберите пользователя, которого нужно добавить к роли создателя лаборатории, а затем нажмите кнопку **Сохранить**. 

    ![Добавление создателя лаборатории](../media/tutorial-setup-lab-account/add-lab-creator.png)

    > [!NOTE]
    > Если вы добавляете пользователя, не являющегося учетная запись Майкрософт, в качестве создателя лаборатории, см. раздел [Добавление пользователя, не являющегося учетная запись Майкрософтным в качестве создателя лаборатории](#add-a-non-microsoft-account-user-as-a-lab-creator) . 

## <a name="specify-marketplace-images-available-to-lab-creators"></a>Выбор образов Marketplace, доступных для создателей лаборатории
Являясь владельцем учетной записи лаборатории, вы можете указать образы Marketplace, которые можно использовать при создании лабораторий в учетной записи лаборатории. 

1. В меню слева выберите **Образы Marketplace**. По умолчанию отображается полный список образов (включенных и отключенных). В раскрывающемся списке вверху можно отфильтровать список, чтобы увидеть только включенные или отключенные образы, выбрав **Только включенные**/**Только отключенные**. 
    
    ![Страница с образами Marketplace](../media/tutorial-setup-lab-account/marketplace-images-page.png)

    В списке отображаются только образы Marketplace, которые соответствуют следующим условиям:
        
    - позволяют создать одну виртуальную машину;
    - используют диспетчер ресурсов Azure для подготовки виртуальных машин;
    - не требуют приобретения дополнительного плана лицензирования.
2. Чтобы **отключить** включенный образ Marketplace, выполните одно из следующих действий: 
    1. Щелкните **... (многоточие)** в последнем столбце и выберите **Отключить образ**. 

        ![Отключение одного образа](../media/tutorial-setup-lab-account/disable-one-image.png) 
    2. Выберите один или несколько образов в списке, установив флажки перед их именами, и щелкните **Отключить выбранные образы**. 

        ![Отключение нескольких образов](../media/tutorial-setup-lab-account/disable-multiple-images.png) 
1. Аналогично, чтобы **включить** отключенный образ Marketplace, выполните одно из следующих действий: 
    1. Щелкните **... (многоточие)** в последнем столбце и выберите **Включить образ**. 
    2. Выберите один или несколько образов в списке, установив флажки перед их именами, и щелкните **Включить выбранные образы**. 

## <a name="add-a-non-microsoft-account-user-as-a-lab-creator"></a>Добавление пользователя, не являющегося учетная запись Майкрософт, в качестве создателя лаборатории
Чтобы добавить пользователя в качестве создателя лаборатории, используйте учетные записи электронной почты. Можно использовать следующие типы учетных записей электронной почты:

- Учетная запись электронной почты, предоставляемая Office 365 Azure Active Directory (AAD) университета. 
- Учетная запись электронной почты Майкрософт, например `@outlook.com`, `@hotmail.com`, `@msn.com`или `@live.com`.
- Учетная запись электронной почты, не относящаяся к Майкрософт, например Yahoo или Google. Однако эти типы учетных записей должны быть связаны с учетная запись Майкрософт.
- Учетная запись GitHub. Эта учетная запись должна быть связана с учетная запись Майкрософт.

### <a name="using-a-non-microsoft-email-account"></a>Использование учетной записи электронной почты, отличной от Майкрософт
Создатели и преподаватели лаборатории могут использовать сторонние учетные записи электронной почты для регистрации и входа в лабораторную среду.  Однако для входа на портал служб лаборатории необходимо сначала создать учетная запись Майкрософт, связанный с адресом электронной почты не от Майкрософт.

У многих инструкторов уже может быть учетная запись Майкрософт, связанных с адресами электронной почты, не являющимися Майкрософт. Например, преподаватели уже имеют учетная запись Майкрософт, если они использовали свой адрес электронной почты с другими продуктами или службами корпорации Майкрософт, такими как Office, Skype, OneDrive или Windows.  

Когда преподаватели входят на портал служб лаборатории, им запрашиваются адрес электронной почты и пароль. Если преподавателю предпринимается попытка входа с помощью неучетная запись Майкрософт, не имеющего учетная запись Майкрософт со ссылкой, инструктор получит следующее сообщение об ошибке: 

![Сообщение об ошибке](../media/how-to-configure-student-usage/cant-find-account.png)

Чтобы зарегистрироваться в учетная запись Майкрософт, преподаватели должны переключиться на [http://signup.live.com](http://signup.live.com).  


### <a name="using-a-github-account"></a>Использование учетной записи GitHub
Преподаватели также могут использовать существующую учетную запись GitHub для регистрации и входа в лабораторную среду. Если у преподавателя уже есть учетная запись Майкрософт, связанный с учетной записью GitHub, они смогут войти в систему и ввести пароль, как показано в предыдущем разделе. Если учетная запись GitHub еще не связана с учетная запись Майкрософт, они должны выбрать **Параметры входа**:

![Ссылка на параметры входа](../media/how-to-configure-student-usage/signin-options.png)

На странице **Параметры входа** выберите **Вход с помощью GitHub**.

![Вход с помощью ссылки GitHub](../media/how-to-configure-student-usage/signin-github.png)

Наконец, им предлагается создать учетная запись Майкрософт, связанный с учетной записью GitHub. Это происходит автоматически, когда инструктор выбирает **Next**.  Затем инструктор сразу же вошел в среду и подключился к лабораторной работе.

## <a name="automatic-shutdown-of-vms-on-disconnect"></a>Автоматическое завершение работы виртуальных машин при отключении
После отключения подключения к удаленному рабочему столу можно включить или отключить автоматическое завершение работы виртуальных машин Windows Lab (шаблон или учащийся). Вы также можете указать, как долго виртуальные машины должны ожидать повторного подключения пользователя перед автоматическим завершением работы.

![Параметр автоматического завершения работы в учетной записи лаборатории](../media/how-to-configure-lab-accounts/automatic-shutdown-vm-disconnect.png)

Этот параметр применяется ко всем лабораторным занятиям, созданным в учетной записи лаборатории. Владелец лаборатории может переопределить этот параметр на уровне лаборатории. Изменение этого параметра в учетной записи лаборатории будет влиять только на лаборатории, созданные после внесения изменений.

Сведения о том, как владелец лаборатории может настроить этот параметр на уровне лаборатории, см. в [этой статье](how-to-enable-shutdown-disconnect.md) .

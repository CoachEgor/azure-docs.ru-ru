# <a name="configure-lab-accounts-in-azure-lab-services"></a>Настройка лабораторных учетных записей в лабораторных службах Azure 
В Лаборант-сервисе Azure Lab Account — это контейнер для управляемых типов лабораторий, таких как классные лаборатории. Администратор задает учетную запись лаборатории с помощью Служб лабораторий Azure и предоставляет доступ владельцам лабораторий, которые могут создать лаборатории в учетной записи. 

В этой статье показано, как выполнять следующие задачи: 

- Укажите диапазон адресов для ВМ в лаборатории
- Настройка автоматического выключения ВМ при отключении

## <a name="specify-an-address-range-for-vms-in-the-lab"></a>Укажите диапазон адресов для ВМ в лаборатории
Следующая процедура имеет шаги, чтобы указать диапазон адресов для VMs в лаборатории. При обновлении диапазона, указанного ранее, измененный диапазон адресов применяется только к вм: высоздаваемые после изменения. 

Вот некоторые ограничения при указании диапазона адресов, которые вы должны иметь в виду. 

- Префикс должен быть меньше или равен 23. 
- Если виртуальная сеть заглянула в лабораторную учетную запись, предоставленный диапазон адресов не может перекрываться с диапазоном адресов от загляной виртуальной сети.

1. На странице **учетной записи Lab** выберите **настройки Labs** в левом меню.
2. В поле **диапазона адресов** укажите диапазон адресов для VMs, которые будут созданы в лаборатории. Диапазон адресов должен находиться в бесклассовой междоменном обозначении (CIDR) (пример: 10.20.0.0.0/23). В этом диапазоне адресов будут создаваться виртуальные машины в лаборатории.
3. На панели инструментов щелкните **Сохранить**. 

    ![Настройка диапазона адресов](../media/how-to-manage-lab-accounts/labs-configuration-page-address-range.png)


## <a name="automatic-shutdown-of-vms-on-disconnect"></a>Автоматическое отключение ВМ при отключении
Можно включить или отключить автоматическое отключение VMs-технологий для лаборатории Windows (шаблон или студент) после отключения удаленного соединения рабочего стола. Вы также можете указать, как долго вспыхнуть в ожидании повторного подключения пользователя, прежде чем автоматически выключиться.

![Автоматическая настройка выключения в лабораторной учетной записи](../media/how-to-configure-lab-accounts/automatic-shutdown-vm-disconnect.png)

Эта настройка применяется ко всем лабораториям, созданным в учетной записи лаборатории. Владелец лаборатории может переопределить эту настройку на уровне лаборатории. Изменение этого параметра в учетной записи лаборатории будет влиять только на лаборатории, которые создаются после изменения.

Чтобы узнать о том, как владелец лаборатории может настроить этот параметр на уровне лаборатории, смотрите [эту статью](how-to-enable-shutdown-disconnect.md)

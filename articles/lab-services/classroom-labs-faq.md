# <a name="classroom-labs-in-azure-lab-services--frequently-asked-questions-faq"></a>Лаборатории для аудитории в Службах лабораторий Azure — вопросы и ответы
В этой статье содержатся ответы на некоторые распространенные вопросы о лабораториях для аудитории в Службах лабораторий Azure. 

## <a name="quotas"></a>Квоты

### <a name="is-the-quota-per-user-or-per-week-or-per-entire-duration-of-the-lab"></a>Применяется ли квота на одного пользователя, на одну неделю или на все время использования лаборатории? 
Заданная для лаборатории квота применяется к каждому учащемуся на протяжении всего времени использования лаборатории. Кроме того, [запланированное время запуска виртуальных машин](how-to-create-schedules.md) не входит в квоту, выделенную пользователю. Квота рассчитывается на время, которое выходит за пределы расписания и которое учащийся тратит на виртуальные машины.  Дополнительные сведения см. в разделе об [установке квот для пользователей](how-to-configure-student-usage.md#set-quotas-for-users).

### <a name="if-educator-turns-on-a-student-vm-does-that-affect-the-student-quota"></a>Если виртуальную машину учащегося включает преподаватель, влияет ли это на квоту учащегося? 
Нет. Это не учитывается. Когда преподаватель включает виртуальную машину учащегося, выделенная для учащегося квота не затрагивается. 

## <a name="schedules"></a>Расписания

### <a name="do-all-vms-in-the-lab-start-automatically-when-a-schedule-is-set"></a>Запускаются ли автоматически все виртуальные машины в лаборатории, если установлено расписание? 
Нет. Нет, запускаются не все виртуальные машины. Это применяется только к тем виртуальным машинам, которые назначены пользователям по расписанию. Виртуальные машины, не назначенные пользователям, не запускаются автоматически. Это сделано специально. 

## <a name="lab-accounts"></a>Учетные записи лаборатории

### <a name="why-am-i-not-able-to-create-a-lab-because-of-unavailability-of-the-address-range"></a>Что делать, если не удается создать лабораторию из-за недоступности диапазона адресов? 
Лаборатории для аудитории могут создавать виртуальные машины лаборатории в том диапазоне IP-адресов, который вы указываете при создании учетной записи лаборатории на портале Azure. Каждой лаборатории, которая создается после предоставления диапазона адресов, выделяется 512 IP-адресов для виртуальных машин этой лаборатории. Диапазон адресов для учетной записи лаборатории должен быть достаточно большим, чтобы разместить все лаборатории, которые вы намерены создать в этой учетной записи лаборатории. 

Например, блока размером /19 (10.0.0.0/19) будет достаточно для размещения 8192 IP-адресов, то есть 16 лабораторий (8192/512 = 16). Это означает, что создание 17 лаборатории завершится сбоем.

### <a name="what-port-ranges-should-i-open-on-my-organizations-firewall-setting-to-connect-to-lab-virtual-machines-via-rdpssh"></a>Какие диапазоны портов нужно открыть в настройках корпоративного брандмауэра, чтобы подключаться к виртуальным машинам лаборатории через RDP или SSH?

Используются следующие порты: 49152–65535 Лаборатории для аудитории размещаются за подсистемой балансировки нагрузки. Каждая лаборатория имеет один общедоступный IP-адрес, а каждая виртуальная машина в лаборатории использует уникальный порт. 

Частные IP-адреса виртуальных машин вы можете узнать на вкладке **Пул виртуальных машин** на домашней странице лаборатории на портале Azure. Если вы повторно публикуете лабораторию, ее общедоступный IP-адрес останется тем же, но частные IP-адреса и номера портов каждой виртуальной машины в лаборатории могут измениться. См. сведения в следующей статье: [Параметры брандмауэра для Служб лабораторий Azure](how-to-configure-firewall-settings.md).

### <a name="what-public-ip-address-range-should-i-open-on-my-organizations-firewall-settings-to-connect-to-lab-virtual-machines-via-rdpssh"></a>Какой диапазон общедоступных IP-адресов нужно открыть в настройках корпоративного брандмауэра, чтобы подключаться к виртуальным машинам лаборатории через RDP или SSH?
В статье о [диапазонах IP-адресов и тегах служб Azure в общедоступном облаке](https://www.microsoft.com/download/details.aspx?id=56519) указаны диапазоны общедоступных IP-адресов для центров обработки данных Azure. Вы можете открыть IP-адреса для тех регионов, в которых находятся ваши учетные записи лабораторий.

## <a name="virtual-machine-images"></a>Образы виртуальных машин

### <a name="as-a-lab-creator-why-cant-i-enable-additional-image-options-in-the-virtual-machine-images-dropdown-when-creating-a-new-lab"></a>Почему я как создатель лаборатории не могу при ее создании выбрать дополнительные варианты образов в раскрывающемся списке образов для виртуальных машин?

Когда администратор добавляет вас в роль создателя лабораторий для учетной записи лабораторий, вы получаете разрешения на создание лабораторий. Но это действие не дает вам прав на изменение параметров учетной записи лаборатории, включая список разрешенных образов виртуальных машин. Чтобы включить дополнительные образы, обратитесь к администратору учетных записей лаборатории, который может сделать это сам или назначить вам роль участника для учетной записи лаборатории. Роль участника предоставит вам разрешения на изменение списка образов виртуальных машин в учетной записи лаборатории.

### <a name="can-i-attach-additional-disks-to-a-virtual-machine"></a>Можно ли подключить дополнительные диски к виртуальной машине?
Нет. Сейчас подключить дополнительные диски к виртуальной машине в лаборатории для аудитории нельзя. 

## <a name="users"></a>Пользователи

### <a name="how-many-users-can-be-in-a-classroom-lab"></a>Сколько пользователей может существовать в лаборатории для аудитории?
Вы можете добавить в лабораторию для аудитории до 400 пользователей.

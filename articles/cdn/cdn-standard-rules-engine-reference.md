# <a name="standard-rules-engine-reference-for-azure-cdn"></a>Справочник по стандартному обработчику правил для Azure CDN

В [движке стандартных правил](cdn-standard-rules-engine.md) для сети доставки содержимого Azure (Azure CDN) правило состоит из одного или нескольких условий соответствия и действия. В этой статье приводится подробное описание условий соответствия и функций, доступных в движке стандартных правил для Azure CDN.

Механизм правил предназначен для окончательного органа по обработке конкретных типов запросов Standard Azure CDN.

**Общие применения для правил:**

- определять или переопределять пользовательские политики кэширования;
- перенаправлять запросы;
- Измените заголовки запросов и ответов HTTP.

## <a name="terminology"></a>Терминология

Определить правило в движке правил, установить [условия соответствия](cdn-standard-rules-engine-match-conditions.md) и [действия:](cdn-standard-rules-engine-actions.md)

 ![Структура правил Azure CDN](./media/cdn-standard-rules-engine-reference/cdn-rules-structure.png)

Каждое правило может иметь до четырех условий матча и три действия. Каждая конечная точка Azure CDN может иметь до пяти правил. 

Включено в текущее ограничение пяти правил для точки зрения Azure CDN является *глобальным правилом*по умолчанию. Глобальное правило не имеет условий соответствия, и действия, которые определяются в глобальном правиле, всегда срабатывают.

## <a name="syntax"></a>Синтаксис

Способ обращения с особыми символами в правиле зависит от того, как различные условия и действия соответствуют текстовым значениям. Условие соответствия или действие может интерпретировать текст одним из следующих способов:

- [Литературные значения](#literal-values)
- [Значения Wildcard](#wildcard-values)


### <a name="literal-values"></a>Литеральные значения

Текст, интерпретируемый как буквальное значение, рассматривает все специальные символы, *за исключением символа %,* как часть значения, которое должно быть сопоставлено в правиле. Например, буквальное условие `'*'` соответствия, установленное, `'*'` удовлетворяется только тогда, когда будет найдено точное значение.

Знак процента используется для обозначения кодирования URL `%20`(например,).

### <a name="wildcard-values"></a>Значения с подстановочными знаками

Текст, интерпретируемый как значение подстановочного знака, присваивает дополнительный смысл специальным символам. В следующей таблице описывается, как в движке стандартных правил интерпретируются конкретные специальные символы:

Знак | Описание
----------|------------
\ | Обратная косая черта экранирует любой из символов, указанных в этой таблице. Обратная косая черта должна располагаться непосредственно перед тем специальным символом, который нужно экранировать. Например, в этом примере экранируется символ звездочки: `\*`
% | Знак процента используется для обозначения кодирования URL `%20`(например,).
\* | Звездочка является подстановочным знаком, которому соответствуют один или несколько символов.
space | Космический символ указывает, что условие соответствия может быть удовлетворено либо из указанных значений или шаблонов.
одиночные метки цитаты | Один знак цитаты не имеет особого значения. Однако набор одиночных кавычек указывает на то, что значение следует рассматривать как буквальное значение. Единые кавычки могут быть использованы следующим образом:<ul><li>Для того, чтобы условие соответствия удовлетворялось всякий раз, когда указанное значение совпадает с любой частью значения сравнения.  Например, условию `'ma'` будет соответствовать любая из следующих строк: <ul><li>/business/**ma**rathon/asset.htm</li><li>**ma**p.gif</li><li>/business/template.**ma**p</li></ul><li>Для того, чтобы специальный символ был указан как буквальный символ. Например, можно указать буквальное космическое символ, привязав космического `'<sample value>'`символа в набор одиночных кавычек (или`' '` ).</li><li>Для уточнения пустого значения. Укажите пустое значение, указав набор отдельных кавычек **(''**).</li></ul>**Важно**!<br /><ul><li>Если указанное значение не содержит подстановочный знак, значение автоматически считается буквальным значением. Вам не нужно указывать набор одиночных кавычек для буквального значения.</li><li>Если backslash не используется для избежания другого символа в этой таблице, backslash игнорируется, когда он указан в наборе одиночных кавычек.</li><li>Еще один способ указать специальный символ в качестве буквального символа, чтобы избежать его с помощью backslash (`\`).</li></ul>

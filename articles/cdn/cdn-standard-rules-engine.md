# <a name="azure-cdn-standard-rules-engine"></a>Azure CDN стандартного обработчика правил

> [!NOTE]
> Обработчик стандартных правил доступен только для Azure CDN корпорации Майкрософт. 

Обработчик Azure CDN стандартных правил позволяет настроить обработку HTTP-запросов. Например, принудительная доставка содержимого по конкретным протоколам, определение политики кэширования или изменение заголовка HTTP. В этом учебнике показано, как создать правило, которое будет автоматически перенаправлять пользователей в HTTPS. 


## <a name="tutorial"></a>Учебник

1. На странице **профиль CDN** на Azure CDN из профилей Майкрософт выберите конечную точку, для которой нужно настроить правила.
  
2. Выберите вкладку **обработчик правил** слева.
   
    Отобразится колонка обработчика правил с глобальным правилом. 
   
    [страница новых правил ![CDN](./media/cdn-standard-rules-engine/cdn-new-rule.png)](./media/cdn-standard-rules-engine/cdn-new-rule.png#lightbox)
   
   > [!IMPORTANT]
   > Порядок перечисления правил влияет на способ их обработки. Последующее правило может переопределить действия, заданные предыдущим правилом.
   >

3. Нажмите кнопку **Добавить правило** и укажите имя правила. Имена правил должны начинаться с буквы и содержать только цифры и буквы.

4. Укажите тип запросов, к которым применяется правило. Используйте раскрывающийся список, чтобы выбрать условие соответствия **протоколу запроса** , и используйте значение, **равное** **http**.
   
   [условие соответствия "правило ![CDN"](./media/cdn-standard-rules-engine/cdn-match-condition.png)](./media/cdn-standard-rules-engine/cdn-match-condition.png#lightbox)
   
   > [!NOTE]
   > В раскрывающемся списке доступны несколько условий соответствия. Подробный список условий соответствия см. в статье [Условия соответствия обработчика правил](cdn-standard-rules-engine-match-conditions.md).
   
5. Выберите действие, которое будет применено к идентифицированным запросам. Используйте раскрывающееся меню, чтобы выбрать действие **перенаправления URL-адреса** и использовать **найденное значение (302)** для типа и **HTTPS** для протокола. Оставьте все остальные поля пустыми, чтобы использовать входящие значения.
   
   [действие правила ![CDN](./media/cdn-standard-rules-engine/cdn-action.png)](./media/cdn-standard-rules-engine/cdn-action.png#lightbox)
   
   > [!NOTE]
   > В раскрывающемся списке доступно несколько действий. Подробный список действий см. в разделе [действия обработчика правил](cdn-standard-rules-engine-actions.md).

6. Нажмите кнопку **сохранить** , чтобы сохранить новое правило.  Теперь новое правило будет развернуто.
   
   > [!IMPORTANT]
   > Изменение правил может занять до 15 минут, чтобы распространяться по Azure CDN.
   >

# <a name="security-attributes-for-azure-app-service"></a>Атрибуты безопасности для службы приложений Azure

В этой статье описываются наиболее распространенные атрибуты безопасности, встроенные в службе приложений Azure.

[!INCLUDE [Security attributes header](../../includes/security-attributes-header.md)]

## <a name="preventative"></a>профилактическая;

| Атрибут безопасности | Да или нет | Примечания |
|---|---|--|
| Шифрование неактивных данных (например, шифрование на стороне сервера, шифрование на стороне сервера с управляемыми клиентом ключами и другие функции шифрования) | Да | Содержимое файла веб-сайта хранится в службе хранилища Azure, которая автоматически шифрует содержимое при хранении. См. в разделе [шифрованием службы хранилища Azure для неактивных данных](../storage/common/storage-service-encryption.md).<br><br>Секреты предоставленный клиента шифруются в неактивном состоянии. Секретные данные шифруются в неактивном состоянии, во время хранения в базах данных конфигурации службы приложений.<br><br>Локально подключенные диски, при необходимости может использоваться как временное хранилище, веб-сайтов (D:\local и % TMP %). Локально подключенные диски, не шифруются при хранении. |
| Шифрование при обмене данными (например, шифрование ExpressRoute в виртуальной сети шифрования и шифрование между сетями)| Да | Клиенты могут настроить веб-сайтов для запроса и использовать протокол HTTPS для входящего трафика. См. в записи блога [как осуществлять только HTTPS службы приложений Azure](https://blogs.msdn.microsoft.com/benjaminperkins/2017/11/30/how-to-make-an-azure-app-service-https-only/). |
| Обработка ключа шифрования (CMK, BYOK и т. д.)| Да | Клиенты могут выбрать для хранения секретов приложения в хранилище ключей и получения их во время выполнения. См. в разделе [ссылается на использование хранилища ключей Azure для службы приложений и функций Azure (Предварительная версия)](app-service-key-vault-references.md).|
| Шифрование на уровне столбцов (службы данных Azure)| Н/Д | |
| Вызовы API в зашифрованном виде| Да | Происходят запросы управления настроить службу приложений с помощью [Azure Resource Manager](../azure-resource-manager/index.yml) вызовы по протоколу HTTPS. |

## <a name="network-segmentation"></a>сегментация сети;

| Атрибут безопасности | Да или нет | Примечания |
|---|---|--|
| Поддержка конечной точки службы| Да | В настоящее время доступна в предварительной версии для службы приложений. См. в разделе [ограничения доступа к службе приложений Azure](app-service-ip-restrictions.md). |
| Поддержка внедрения виртуальной сети| Да | Среды службы приложений — частные реализации службы приложений, выделенном единственному заказчику, вставленных в виртуальной сети клиента. См. в разделе [введение в среду службы приложений](environment/intro.md). |
| Поддержка сетевой изоляции и Firewalling| Да | Для общедоступной версии мультитенантное приложение службы клиенты могут настроить сетевые списки управления доступом (ограничения IP-адресов) для блокировки разрешенного входящего трафика.  См. в разделе [ограничения доступа к службе приложений Azure](app-service-ip-restrictions.md).  Среды службы приложений развертываются непосредственно в виртуальных сетях, следовательно, могут быть защищены с помощью групп безопасности сети. |
| Поддержка принудительного туннелирования | Да | Среды службы приложений можно развернуть в виртуальной сети клиента, где настраивается Принудительное туннелирование. Клиенты должны следовать указаниям, приведенным в [Настройка среды службы приложений с помощью принудительного туннелирования](environment/forced-tunnel-support.md). |

## <a name="detection"></a>Обнаружение

| Атрибут безопасности | Да или нет | Примечания|
|---|---|--|
| Azure Monitor поддержки (Log analytics, Application insights, и т.д.)| Да | Служба приложений интегрируется с помощью Application Insights для языков, поддерживающих Application Insights (полной версии .NET Framework, .NET Core, Java и Node.JS).  См. в разделе [службе приложений Azure монитор производительности](../azure-monitor/app/azure-web-apps.md). Служба приложений также отправляет метрики приложения в Azure Monitor. См. в разделе [мониторинг приложений в службе приложений Azure](web-sites-monitor.md). |

## <a name="identity-and-access-management"></a>Управление удостоверениями и доступом

| Атрибут безопасности | Да или нет | Примечания|
|---|---|--|
| Authentication| Да | Клиенты могут создавать приложения в службе приложений, которые автоматически интегрируются с [Azure Active Directory (Azure AD)](../active-directory/index.md) а также других поставщиков совместимых удостоверений OAuth; см. в разделе [проверки подлинности и авторизации в Служба приложений Azure](overview-authentication-authorization.md). Для управления доступа к ресурсам службы приложений все доступ контролируется с помощью сочетания основной проверки подлинности Azure AD и ролей RBAC Azure Resource Manager. |
| Авторизация| Да | Для управления доступа к ресурсам службы приложений все доступ контролируется с помощью сочетания основной проверки подлинности Azure AD и ролей RBAC Azure Resource Manager.  |


## <a name="audit-trail"></a>журнал аудита;

| Атрибут безопасности | Да или нет | Примечания|
|---|---|--|
| Ведение журнала и аудит плана управления и контроля| Да | Все операции управления, над объектами службы приложений, выполняются с помощью [Azure Resource Manager](../azure-resource-manager/index.yml). Журналы этих операций доступны как на портале, так и с помощью интерфейса командной строки; см. в разделе [операции поставщиков ресурсов Azure Resource Manager](../role-based-access-control/resource-provider-operations.md#microsoftweb) и [az monitor-журнал действий](/cli/azure/monitor/activity-log). |
| Ведение журнала и аудит плоскости данных | Нет | Плоскость данных для службы приложений является удаленной общей папке, содержимым развернутого веб-сайта клиента.  Нет, отключить аудит удаленной общей папке. |

## <a name="configuration-management"></a>Управление конфигурацией

| Атрибут безопасности | Да или нет | Примечания|
|---|---|--|
| Поддержка конфигурации management (Управление версиями конфигурации и т. д.)| Да | Для операций управления состояние конфигурации службы приложений можно экспортировать как шаблон Azure Resource Manager и управлять их версиями со временем. Для операций среды выполнения пользователи могут поддерживают несколько различных версий динамические приложения с помощью компонента слоты развертывания службы приложений. |

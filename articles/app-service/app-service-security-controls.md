# <a name="security-controls-for-azure-app-service"></a>Элементы управления безопасностью для службы приложений Azure

В этой статье документируются элементы управления безопасностью, встроенные в службу приложений Azure.

[!INCLUDE [Security controls header](../../includes/security-controls-header.md)]

## <a name="network"></a>Сеть

| Контроль безопасности | Да/нет | Примечания | Документация
|---|---|--|
| Поддержка конечных точек обслуживания| Да | Доступно для службы приложений.| [Ограничения доступа к службам приложений Azure](app-service-ip-restrictions.md)
| Поддержка инъекций VNet| Да | Среды службы приложений являются частными реализациями Службы приложений, предназначенными для одного клиента, вводимого в виртуальную сеть клиента. | [Общие сведения о средах службы приложений](environment/intro.md)
| Поддержка сетевой изоляции и брандмауэра| Да | Для общедоступных вариантов мультитенантов Службы приложений клиенты могут настроить сетевые ACL (Ip Restrictions) для блокировки разрешенного входящего трафика.  Среды службы приложений развертываются непосредственно в виртуальных сетях и, следовательно, могут быть защищены с помощью НСК. | [Ограничения доступа к службам приложений Azure](app-service-ip-restrictions.md)
| Принудительная поддержка туннелирования| Да | Среды службы приложений могут быть развернуты в виртуальной сети клиента, где настроен принудительный туннель. | [Настройка принудительного туннелирования в среде службы приложений](environment/forced-tunnel-support.md)

## <a name="monitoring--logging"></a>Мониторинг & лесозаготовки

| Контроль безопасности | Да/нет | Примечания | Документация
|---|---|--|
| Поддержка мониторинга Azure (аналитика журналов, информация о приложениях и т.д.)| Да | Служба приложений интегрируется с приложениями Insights для языков, поддерживающих Application Insights (Full .NET Framework, .NET Core, Java и Node.JS).  Смотрите [производительность службы приложений Monitor Azure](../azure-monitor/app/azure-web-apps.md). Служба приложений также отправляет метрики приложений в Azure Monitor. | [Мониторинг приложений в Службе приложений Azure](web-sites-monitor.md)
| Управление и управление самолетом лесозаготовки и аудит| Да | Все операции управления, выполняемые на объектах службы приложений, происходят через [Azure Resource Manager.](../azure-resource-manager/index.yml) Исторические журналы этих операций доступны как на портале, так и через CLI. | [Операции поставщика ресурсов Ресурсов менеджера ресурсов Azure,](../role-based-access-control/resource-provider-operations.md#microsoftweb) [az мониторинг активности-журнал](/cli/azure/monitor/activity-log)
| Регистрация и аудит плоскости данных | нет | Плоскость данных для Службы приложений — это удаленная доля файлов, содержащая развернутый веб-сайт клиента.  Отсутствует аудит доли удаленного файла. |

## <a name="identity"></a>идентификации

| Контроль безопасности | Да/нет | Примечания |  Документация
|---|---|--|
| Проверка подлинности| Да | Клиенты могут создавать приложения в Службе приложений, которые автоматически интегрируются с [Azure Active Directory (Azure AD),](../active-directory/index.yml) а также с другими совместимыми поставщиками идентификационных данных OAuth для управления доступом к ресурсам Службы приложений, весь доступ контролируется комбинацией аутентифицированных основных функций Azure AD и ролей менеджера ресурсов Azure RBAC. | [Проверка подлинности и авторизация в службе приложений Azure](overview-authentication-authorization.md)
| Авторизация| Да | Для доступа к ресурсам службы управления весь доступ контролируется комбинацией аутентифицированных основных функций Azure AD и rBAC-менеджером ресурсов Azure.  | [Проверка подлинности и авторизация в службе приложений Azure](overview-authentication-authorization.md)

# <a name="configure-a-custom-domain-name-in-azure-app-service-with-traffic-manager-integration"></a>Настройка пользовательского доменного имени в службе приложений Azure с интеграцией менеджера трафика

[!INCLUDE [web-selector](../../includes/websites-custom-domain-selector.md)]

> [!NOTE]
> Сведения об облачных службах см. в статье [Настройка пользовательского доменного имени для облачной службы Azure](../cloud-services/cloud-services-custom-domain-name.md).

При использовании [Azure Traffic Manager](/azure/traffic-manager/) для загрузки трафика баланса в [службу приложений Azure](overview.md)приложение можно получить доступ с помощью ** \<трафика менеджера-конце>.trafficmanager.net.** Вы можете назначить пользовательское доменное\.имя, например www contoso.com, с вашим приложением Службы приложений, чтобы обеспечить более узнаваемое доменное имя для ваших пользователей.

В этой статье показано, как настроить пользовательское доменное имя с приложением App Service, интегрированным с [диспетчером трафика.](../traffic-manager/traffic-manager-overview.md)

> [!NOTE]
> Поддерживаются только записи [CNAME](https://en.wikipedia.org/wiki/CNAME_record) при настройке доменного имени с помощью конечной точки Traffic Manager. Поскольку записи не поддерживаются, отображение корневого домена, например contoso.com также не поддерживается.
> 

## <a name="prepare-the-app"></a>Подготовка приложения

Чтобы сопоставить пользовательское имя DNS с приложением, интегрированным с Azure Traffic Manager, [план службы приложений](https://azure.microsoft.com/pricing/details/app-service/) веб-приложения должен быть в **стандартном** уровне или выше. На этом шаге следует убедиться, что приложение службы приложений находится в поддерживаемой ценовой категории.

### <a name="check-the-pricing-tier"></a>Проверка ценовой категории

На [портале Azure](https://portal.azure.com)ищите **службы приложений**и выберите их.

На странице **Службы приложений** выберите имя приложения Azure.

![Переход к приложению Azure на портале](./media/app-service-web-tutorial-custom-domain/select-app.png)

В левой навигации страницы приложения выберите **Масштабирование (план службы приложений).**

![Меню увеличения масштаба](./media/app-service-web-tutorial-custom-domain/scale-up-menu.png)

Текущий уровень приложения выделен синей рамкой. Убедитесь, что приложение находится в **стандартном** уровне или выше (любой уровень в **категории Производство** или **Изолированная).** Если да, закройте страницу **Масштабирования** и перейдите к [созданию отображения CNAME.](#create-the-cname-mapping)

![Проверка ценовой категории](./media/app-service-web-tutorial-custom-domain/check-pricing-tier.png)

### <a name="scale-up-the-app-service-plan"></a>Изменение уровня плана службы приложений

Если вам необходимо расширить приложение, выберите любой из уровней ценообразования в категории **Production.** Чтобы просмотреть дополнительные параметры, щелкните **См. дополнительные параметры**.

Щелкните **Применить**.

## <a name="create-traffic-manager-endpoint"></a>Создание конечная точка менеджера трафика

После выполнения последующих шагов в Add [или Delete Endpoints](../traffic-manager/traffic-manager-endpoints.md)добавьте приложение Службы приложения в качестве конечной точки в профиль менеджера трафика.

После того, как приложение App Service находится в поддерживаемом ценовом уровне, оно отображается в списке доступных целей Службы поддержки приложений при добавлении конечной точки. Если приложение не включено в список, [проверьте ценовой уровень вашего приложения.](#prepare-the-app)

## <a name="create-the-cname-mapping"></a>Создание отображения CNAME
> [!NOTE]
> Чтобы настроить [домен Службы Приложений, который вы приобрели,](manage-custom-dns-buy-domain.md)пропустите этот раздел и перейдите к [пользовательскому домену Enable.](#enable-custom-domain)
> 

[!INCLUDE [Access DNS records with domain provider](../../includes/app-service-web-access-dns-records-no-h.md)]

В то время как специфика каждого поставщика доменов варьируется, вы отображаете *от* пользовательского доменного имени **(например, contoso.com)** *до* доменного имени traffic Manager **(contoso.trafficmanager.net),** которое интегрировано с вашим приложением.

> [!NOTE]
> Если запись уже используется и вам нужно заблаговременно привязать к ней свое приложение, создайте дополнительную запись CNAME. Например, чтобы упреждающе привязать **contoso.com\.к** приложению, создайте запись CNAME с **awverify.www** до **contoso.trafficmanager.net.** Затем вы можете\.добавить "www contoso.com" в приложение без необходимости изменения записи "www" CNAME. Для получения дополнительной информации [см.](manage-custom-dns-migrate-domain.md)

По завершении добавления или изменения записей DNS сохраните эти изменения в своем поставщике домена.

## <a name="enable-custom-domain"></a>Включить пользовательский домен
После того, как записи для доменного имени будут распространены, используйте браузер, чтобы убедиться, что пользовательское доменное имя разрешается с приложением Службы поддержки приложений.

> [!NOTE]
> Распространение CNAME через систему DNS может занять некоторое время. Вы можете использовать такую <a href="https://www.digwebinterface.com/">https://www.digwebinterface.com/</a> услугу, как проверить, что CNAME доступен.
> 
> 

1. Как только разрешение домена будет успешным, вернуться на страницу приложения на [портале Azure](https://portal.azure.com)
2. Из левой навигации выберите **пользовательские домены** > **Добавить хостимя.**
4. Введите пользовательское доменное имя, которое вы отображали ранее, и выберите **Validate.**
5. Убедитесь, что **тип записи Hostname** установлен на **\.CNAME (www example.com или любой поддомен).**

6. Поскольку приложение App Service теперь интегрировано с конечным пунктом менеджера трафика, вы должны увидеть доменное имя traffic Manager под **конфигурацией CNAME.** Выберите его и нажмите **Добавить пользовательский домен**.

    ![Добавление DNS-имени в приложение](./media/configure-domain-traffic-manager/enable-traffic-manager-domain.png)

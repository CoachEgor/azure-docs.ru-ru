# <a name="overview-of-security-features-in-azure-backup"></a>Общие сведения о функциях безопасности в Azure Backup

Одним из наиболее важных действий, которые можно предпринять для защиты данных, является наличие надежной инфраструктуры резервного копирования. Но очень важно убедиться, что резервное копирование данных выполняется безопасным образом, и что резервные копии защищены в любое время. Azure Backup обеспечивает безопасность для среды резервного копирования — как при передаче, так и при хранении данных. В этой статье перечислены возможности обеспечения безопасности в Azure Backup, которые помогут защитить данные резервных копий и удовлетворить потребности системы безопасности вашего бизнеса.

## <a name="management-and-control-of-identity-and-user-access"></a>Управление удостоверениями и доступом пользователей и управление ими

Azure Backup позволяет управлять детальным доступом с помощью [управления доступом на основе ролей (RBAC) Azure](https://docs.microsoft.com/azure/role-based-access-control/built-in-roles). RBAC позволяет разделить обязанности внутри группы и предоставить пользователям доступ только для пользователей, необходимых для выполнения своих задач.

Azure Backup предоставляет три встроенных роли для управления операциями управления резервным копированием.

* Участник резервного копирования — создание резервных копий и управление ими, за исключением удаления хранилища служб восстановления и предоставления доступа другим пользователям.
* Оператор BACKUP — все, что делает участник, за исключением удаления резервных копий и управления политиками архивации.
* Читатель резервных копий — разрешения на просмотр всех операций управления архивацией

Дополнительные сведения об управлении [доступом на основе ролей для управления Azure Backup](https://docs.microsoft.com/azure/backup/backup-rbac-rs-vault).

В Azure Backup имеется несколько средств управления безопасностью, которые позволяют предотвращать, обнаруживать уязвимости системы безопасности и реагировать на них. Дополнительные сведения об [элементах управления безопасностью для Azure Backup](https://docs.microsoft.com/azure/backup/backup-security-controls).

## <a name="separation-between-guest-and-azure-storage"></a>Разделение между гостевой службой и хранилищем Azure

При использовании Azure Backup, которая включает в себя резервное копирование виртуальных машин и SQL и SAP HANA в резервной копии виртуальных машин, данные резервного копирования хранятся в службе хранилища Azure, а у гостя нет прямого доступа к хранилищу службы архивации или его содержимому.  При использовании резервного копирования виртуальных машин создание моментального снимка резервной копии и хранилище выполняется структурой Azure, в которой гостевая система не имеет участия, кроме замораживания рабочей нагрузки для создания резервных копий с учетом приложений.  При использовании SQL и SAP HANA расширение резервного копирования получает временный доступ для записи в определенные большие двоичные объекты.  Таким образом, даже в скомпрометированной среде существующие резервные копии не могут быть изменены или удалены гостевыми системами.

## <a name="internet-connectivity-not-required-for-azure-vm-backup"></a>Подключение к Интернету не требуется для резервного копирования виртуальных машин Azure

Для резервного копирования виртуальных машин Azure требуется перемещение данных с диска виртуальной машины в хранилище служб восстановления. Однако все необходимые подключения и передача данных выполняются только в магистральной сети Azure без необходимости доступа к виртуальной сети. Поэтому для резервного копирования виртуальных машин Azure, размещенных в защищенных сетях, не требуется предоставлять доступ к IP-адресам или полным доменным именам.

## <a name="private-endpoints-for-azure-backup"></a>Частные конечные точки для службы архивации Azure

Теперь можно использовать [частные конечные точки](https://docs.microsoft.com/azure/private-link/private-endpoint-overview) для безопасного резервного копирования данных с серверов внутри виртуальной сети в хранилище служб восстановления. Частная конечная точка использует IP-адрес из пространства адресов виртуальной сети для вашего хранилища, поэтому вам не нужно предоставлять виртуальные сети для общедоступных IP. Частные конечные точки можно использовать для резервного копирования и восстановления баз данных SQL и SAP HANA, которые выполняются внутри виртуальных машин Azure. Его также можно использовать для локальных серверов с помощью агента MARS.

## <a name="encryption-of-data-in-transit-and-at-rest"></a>Шифрование передаваемых и неактивных данных

Шифрование защищает ваши данные и помогает удовлетворить ваши обязательства по обеспечению безопасности и соответствия требованиям Организации. В Azure данные, передаваемые между службой хранилища Azure и хранилищем, защищаются по протоколу HTTPS. Эти данные остаются в магистральной сети Azure.

* Данные архивации автоматически шифруются с помощью ключей, управляемых корпорацией Майкрософт. Вы также можете зашифровать резервные копии виртуальных машин с управляемыми дисками в хранилище служб восстановления с помощью [управляемых клиентом ключей](https://docs.microsoft.com/azure/backup/backup-azure-security-feature-cloud#encryption-of-backup-data-using-customer-managed-keys) , хранящихся в Azure Key Vault. Для включения этого шифрования не нужно предпринимать никаких явных действий. Она применяется ко всем рабочим нагрузкам, которые архивируются в хранилище служб восстановления.

* Azure Backup поддерживает резервное копирование и восстановление виртуальных машин Azure с дисками ОС и данных, зашифрованными с помощью шифрования дисков Azure (ADE). [Узнайте больше о зашифрованных виртуальных машинах Azure и Azure Backup](https://docs.microsoft.com/azure/backup/backup-azure-vms-encryption).

## <a name="protection-of-backup-data-from-unintentional-deletes"></a>Защита резервных копий данных от непреднамеренных удалений

Azure Backup предоставляет функции безопасности, помогающие защитить данные резервных копий даже после удаления. При обратимом удалении, если пользователь удаляет резервную копию виртуальной машины, данные резервной копии сосохранены в течение 14 дополнительных дней, что позволяет восстановить этот элемент без потери данных. Дополнительные 14 дней при хранении данных резервных копий в состоянии "обратимого" удаления не взимается никаких затрат на клиент. Дополнительные [сведения об обратимом удалении](https://docs.microsoft.com/azure/backup/backup-azure-security-feature-cloud#soft-delete).

## <a name="monitoring-and-alerts-of-suspicious-activity"></a>Мониторинг подозрительных действий и оповещения о них

Azure Backup предоставляет [встроенные возможности мониторинга и оповещения](https://docs.microsoft.com/azure/backup/backup-azure-monitoring-built-in-monitor) для просмотра и настройки действий для событий, связанных с Azure Backup. [Отчеты о резервном копировании](https://docs.microsoft.com/azure/backup/configure-reports) служат в качестве места назначения для отслеживания использования, аудита резервных копий и восстановления, а также выявления ключевых тенденций на различных уровнях детализации. Использование средств мониторинга и создания отчетов Azure Backup может оповещать вас о любых несанкционированных, подозрительных или вредоносных действиях сразу же после их появления.

## <a name="security-features-to-help-protect-hybrid-backups"></a>Функции безопасности, помогающие защитить гибридные резервные копии

Azure Backup служба использует агент Службы восстановления Microsoft Azure (MARS) для резервного копирования и восстановления файлов, папок, тома или состояния системы с локального компьютера в Azure. Теперь режим MARS предоставляет функции безопасности, помогающие защитить гибридные резервные копии. Эти функции включают в себя следующие:

* При выполнении критически важных операций, таких как изменение парольной фразы, добавляется дополнительный уровень аутентификации. Это гарантирует, что такие операции могут выполнять только пользователи с действительными учетными данными Azure. Дополнительные [сведения о функциях, предотвращающих атаки](https://docs.microsoft.com/azure/backup/backup-azure-security-feature#prevent-attacks).

* Удаленные резервные копии данных сохранены в течение еще 14 дней с момента удаления. Это обеспечивает возможность восстановления данных в течение заданного периода времени, поэтому потери данных даже при атаках не происходит. Кроме того, повышение минимального числа точек восстановления позволяет защититься от повреждения данных. Дополнительные [сведения о восстановлении удаленных резервных копий данных](https://docs.microsoft.com/azure/backup/backup-azure-security-feature#recover-deleted-backup-data).

* Для данных, резервное копирование которых осуществляется с помощью агента Службы восстановления Microsoft Azure (MARS), используется парольная фраза, чтобы гарантировать, что данные шифруются перед отправкой в Azure Backup и расшифровываться только после скачивания из Azure Backup. Сведения о парольной фразе доступны только пользователю, создавшему парольную фразу и настроенную с ней агент. Передача данных или общий доступ к службе не осуществляется. Это гарантирует полную безопасность данных, так как любые данные, непреднамеренно такие как атака "злоумышленник в середине" в сети, не смогут использоваться без парольной фразы, а парольная фраза не отправляется в сеть.

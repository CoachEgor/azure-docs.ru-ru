# <a name="support-matrix-for-backup-with-microsoft-azure-backup-server-or-system-center-dpm"></a>Матрица поддержки для резервного копирования с сервером резервного копирования Microsoft Azure или DPM System Center

Службу [резервного копирования Azure](backup-overview.md) можно использовать для резервного копирования на месте машины и рабочие нагрузки, а также виртуальные машины Azure (VM). В этой статье кратко излагаются параметры поддержки и ограничения для резервного копирования машин с помощью сервера резервного копирования Microsoft Azure (MABS) или менеджера по защите данных System Center (DPM) и резервного копирования Azure.

## <a name="about-dpmmabs"></a>О DPM/MABS

[System Center DPM](https://docs.microsoft.com/system-center/dpm/dpm-overview?view=sc-dpm-1807) — это корпоративное решение, которое настраивает, облегчает и управляет резервным копированием и восстановлением корпоративных машин и данных. Эта служба входит в набор продуктов [System Center](https://www.microsoft.com/cloud-platform/system-center-pricing).

MABS — это серверный продукт, который можно использовать для резервного копирования на внутренних физических серверах, вдовых мыслящих и приложений, работающих на них.

MABS основан на System Center DPM и предоставляет аналогичную функциональность с некоторыми отличиями:

- Для запуска MABS не требуется лицензия System Center.
- Как для MABS, так и для DPM Azure обеспечивает долгосрочное хранение резервного копирования. Кроме того, DPM позволяет выполнять резервное копирование данных для долгосрочного хранения на ленте. В MABS такая возможность не предоставляется.
- Вы можете создать резервную перевесом основной сервер DPM со вторичным сервером DPM. Сервер-получатель будет защищать базы данных сервера-источника и реплики источников данных, хранящиеся на сервере-источнике. В случае сбоя сервера-источника сервер-получатель может продолжать защищать рабочие нагрузки, защищаемые сервером-источником, до тех пор, пока сервер-источник снова не станет доступным.  В MABS такая возможность не предоставляется.

Вы загружаете MABS из [Центра загрузки Microsoft](https://www.microsoft.com/download/details.aspx?id=57520). Он может работать на месте или на Azure VM.

В DPM и MABS поддерживается резервное копирование самых разнообразных приложений, а также серверных и клиентских операционных систем. Они поддерживают несколько сценариев резервного копирования:

- резервное копирование на уровне компьютера путем создания резервной копии состояния системы или исходного состояния системы;
- резервное копирование определенных томов, общих папок, папок и файлов;
- Вы можете создать резервную резервную информацию о конкретных приложениях с помощью оптимизированных настроек, осведомленных о приложениях.

## <a name="dpmmabs-backup"></a>Резервное копирование DPM/MABS

Резервное копирование с помощью DPM/MABS и резервного копирования Azure работает следующим образом:

1. На каждой машине, которая будет резервной копией, устанавливается защитный агент DPM/MABS.
1. Машины и приложения резервное копирование до локального хранения на DPM/MABS.
1. Выполняется установка агента Служб восстановления Microsoft Azure (MARS) на серверы DPM и MABS.
1. Агент MARS резервирует диски DPM/MABS в хранилище служб восстановления резервного копирования в Azure с помощью резервного копирования Azure.

### <a name="azure-expressroute-support"></a>Поддержка Azure ExpressRoute

Вы можете создать резервную связь с данными azure ExpressRoute с помощью общедоступных пирингов (доступных для старых схем) и спиринга Майкрософт. Резервное копирование над приватным пирингом не поддерживается.

С публичным пирингом: Обеспечить доступ к следующим доменам/адресам:

* `http://www.msftncsi.com/ncsi.txt`
* `microsoft.com`
* `.WindowsAzure.com`
* `.microsoftonline.com`
* `.windows.net`

С помощью пиринга Майкрософт, пожалуйста, выберите следующие службы/регионы и соответствующие значения сообщества:

* Активный каталог Azure (12076:5060)
* Регион Microsoft Azure (в зависимости от местоположения хранилища служб восстановления)
* Хранилище Azure (в зависимости от местоположения хранилища служб восстановления)

## <a name="dpmmabs-storage-support"></a>Поддержка хранилища DPM или MABS

Данные, резервные для DPM/MABS, хранятся на локальном дисковом хранилище.

**Хранилище** | **Подробно**
--- | ---
**MBS** | Современное резервное хранилище (MBS) поддерживается от DPM 2016/MABS v2 и позже. MBS недоступно для MABS версии 1.
**Хранилище MABS на виртуальной машине Azure** | Данные хранятся на дисках Azure, которые прикрепляются к DPM/MABS VM и управляются в DPM/MABS. Количество дисков, которые могут быть использованы для пула хранения DPM/MABS, ограничено размером VM.<br/><br/> A2 VM: 4 диска; A3 VM: 8 дисков; A4 VM: 16 дисков, с максимальным размером 1 ТБ для каждого диска. Это определяет общий доступный пул хранения резервного копирования.<br/><br/> Объем данных, для которых можно выполнить резервное копирование, зависит от количества и размера присоединенных дисков.
**Хранение данных MABS на виртуальной машине Azure** | Мы рекомендуем хранить данные в течение одного дня на диске DPM/MABS Azure и резервное копирование с DPM/MABS в хранилище для более длительного хранения. Таким образом, вы можете защитить большой объем данных за счет их выгрузки в службу Azure Backup.

### <a name="modern-backup-storage-mbs"></a>Современное хранилище резервных копий (MBS)

Из DPM 2016/MABS v2 (работает на Windows Server 2016) и позже, вы можете воспользоваться современным резервным хранилищем (MBS).

- Резервные копии MBS хранятся на диске Resilient File System (ReFS).
- MBS использует клонирование блоков ReFS для более быстрого резервного копирования и более эффективного использования складских помещений.
- При добавлении томов в локальный пул хранения DPM/MABS вы настраиваете их с помощью дисковых букв. Затем можно настроить хранилище рабочих нагрузок на разных томах.
- При создании групп защиты для резервного копирования данных в DPM или MABS нужно выбрать диск для использования. Например, можно хранить резервные копии для S'L или других высокопроизводительных рабочих нагрузок IOPS на высокопроизводительном диске, а также хранить рабочие нагрузки, которые резервируются реже на диске с более низкой производительностью.

## <a name="supported-backups-to-dpm"></a>Поддерживаемые типы резервных копий, передаваемые в DPM

Для получения информации о различных серверах и рабочих нагрузок, которые можно защитить с менеджером по защите данных, обратитесь к статье [Что может DPM резервное копирование?](https://docs.microsoft.com/system-center/dpm/dpm-protection-matrix?view=sc-dpm-2019).

- Кластерные рабочие нагрузки, поддерживаемые DPM/MABS, должны находиться в том же домене, что и DPM/MABS, или в домене ребенка/доверенного.
- Для резервного копирования данных из недоверенных доменов или рабочих групп можно использовать NTLM или аутентификацию на основе сертификата.

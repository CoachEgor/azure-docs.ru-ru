# <a name="support-matrix-for-backup-with-microsoft-azure-backup-server-or-system-center-dpm"></a>Таблица поддержки для резервного копирования с помощью Microsoft Azure Backup Server или System Center DPM

Можно использовать [службы архивации Azure](backup-overview.md) для резервного копирования локальных компьютеров и рабочих нагрузок и виртуальных машин (ВМ) Azure. В этой статье перечислены параметры поддержки и ограничения для архивации машин с помощью службы Microsoft Azure Backup Server (MABS) или System Center Data Protection Manager (DPM) и Azure Backup.

## <a name="about-dpmmabs"></a>О DPM или MABS

[System Center DPM](https://docs.microsoft.com/system-center/dpm/dpm-overview?view=sc-dpm-1807) — это корпоративное решение, который настраивает, упрощает и управляет резервного копирования и восстановления машины enterprise и данных. Эта служба входит в набор продуктов [System Center](https://www.microsoft.com/cloud-platform/system-center-pricing).

MABS — это продукт сервера, который может использоваться для резервного копирования локальных физических серверов, виртуальных машин и приложений, выполняющихся на них.

MABS основана на System Center DPM и предоставляет такие же функции с несколькими различия:
- Для запуска MABS не требуется лицензия System Center.
- MABS, и DPM Azure обеспечивают Долгосрочное хранение резервных копий. Кроме того, DPM позволяет выполнять резервное копирование данных для долгосрочного хранения на ленте. В MABS такая возможность не предоставляется.

Скачайте MABS из [центра загрузки Майкрософт](https://www.microsoft.com/en-us/download/details.aspx?id=57520). Она может выполняться на локальном или на виртуальной Машине Azure.

В DPM и MABS поддерживается резервное копирование самых разнообразных приложений, а также серверных и клиентских операционных систем. Они поддерживают несколько сценариев резервного копирования:

- резервное копирование на уровне компьютера путем создания резервной копии состояния системы или исходного состояния системы;
- резервное копирование определенных томов, общих папок, папок и файлов;
- Можно выполнять архивацию конкретных приложений с помощью оптимизированного приложения с поддержкой параметров.

## <a name="dpmmabs-backup"></a>Резервное копирование DPM или MABS

Резервное копирование с помощью DPM или MABS и Azure Backup работает следующим образом:

1. Агент защиты DPM или MABS устанавливается на каждом компьютере, который будут создаваться резервные копии.
1. Машины и приложения архивируются в локальное хранилище на сервере DPM или MABS.
1. Выполняется установка агента Служб восстановления Microsoft Azure (MARS) на серверы DPM и MABS.
1. Агент служб восстановления Microsoft AZURE создает резервные копии дисков DPM или MABS в резервное хранилище служб восстановления в Azure с помощью Azure Backup.

Дополнительные сведения

- [Дополнительные сведения](backup-architecture.md#architecture-back-up-to-dpmmabs) об архитектуре MABS.
- [Просмотрите поддерживаемые](backup-support-matrix-mars-agent.md) для агента MARS.

### <a name="modern-backup-storage-mbs"></a>Современное хранилище резервных копий (MBS)
С DPM 2016 или MABS v2 (под управлением Windows Server 2016) и более поздней версии, можно воспользоваться преимуществами современного хранилища резервных копий (MBS).

- Резервные копии MBS хранятся на диске Resilient File System (ReFS).
- MBS использует клонирование быстрее резервного копирования и более эффективного использования пространства для хранения блоков ReFS.
- При добавлении тома в локальный пул носителей DPM или MABS, их настройка с буквами дисков. Затем можно настроить хранилище рабочих нагрузок на разных томах.
- При создании групп защиты для резервного копирования данных в DPM или MABS нужно выбрать диск для использования. Например может хранение резервных копий SQL или другого высокой диска рабочих нагрузок высокопроизводительных операций ввода-ВЫВОДА и хранения рабочих нагрузок, резервное копирование реже на диске ниже производительности.

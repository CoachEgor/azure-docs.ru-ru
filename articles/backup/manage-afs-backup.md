# <a name="manage-azure-file-share-backups"></a>Управление резервными копиями файловых ресурсов Azure

В этой статье описаны общие задачи по управлению и мониторингу файлов Azure, которые поддерживаются [резервным копированием Azure.](https://docs.microsoft.com/azure/backup/backup-overview) Вы узнаете, как выполнять задачи управления в хранилище служб восстановления.

## <a name="monitor-jobs"></a>Мониторинг заданий

При запуске резервного копирования или восстановлении операции служба резервного копирования создает задание для отслеживания. Вы можете отслеживать ход выполнения всех заданий на странице **Задания резервного копирования**.

Чтобы открыть страницу **Задания резервного копирования**, выполните следующее:

1. Откройте хранилище служб восстановления, используемое для настройки резервной копирования для файлов. В панели **обзора** выберите **вакансии резервного копирования** в разделе **Мониторинг.**

   ![Вакансии резервного копирования в разделе Мониторинг](./media/manage-afs-backup/backup-jobs.png)

1. После выбора **OK**панель **резервных заданий** перечисляет состояние всех заданий. Выберите имя рабочей нагрузки, которое соответствует разделу файла, который вы хотите контролировать.

   ![Имя рабочей нагрузки](./media/manage-afs-backup/workload-name.png)

## <a name="create-a-new-policy"></a>Создание новой политики

Можно создать новую политику для резервного копирования файлов Azure из **раздела политики резервного копирования** в хранилище служб восстановления. Все политики, созданные при настройке резервного копирования для файлов, отображаются с **типом политики** как **Azure File Share.**

Для просмотра существующих политик резервного копирования:

1. Откройте хранилище служб восстановления, используемое для настройки резервной системы для общего файла. В меню хранилища служб восстановления выберите **политики резервного копирования** в разделе **«Управление».** Отображаются все политики резервного копирования, настроенные в хранилище.

   ![Все политики резервного копирования](./media/manage-afs-backup/all-backup-policies.png)

1. Для просмотра политик, специфиченных **для раздела файлов Azure,** выберите **раздел файлов Azure** из списка выпадающих в правом верхнем углу.

   ![Выберите раздел файла Azure](./media/manage-afs-backup/azure-file-share.png)

Для создания новой политики резервного копирования:

1. В панели **политик резервного копирования** выберите **и добавьте**.

   ![Новая политика резервного копирования](./media/manage-afs-backup/new-backup-policy.png)

1. В панели **добавления** выберите **раздел «Общий файл Azure»** в качестве **типа политики.** Открывается панель **политики резервного копирования** для **совместного файла Azure.** Укажите имя политики, частоту резервного копирования и диапазон удержания для точек восстановления. После определения политики выберите **OK**.

   ![Определение политики резервного копирования](./media/manage-afs-backup/define-backup-policy.png)

## <a name="modify-policy"></a>Изменение политики

Можно изменить политику резервного копирования, чтобы изменить частоту или диапазон хранения резервного копирования.

Чтобы изменить политику:

1. Откройте хранилище служб восстановления, используемое для настройки резервной системы для общего файла. В меню хранилища служб восстановления выберите **политики резервного копирования** в разделе **«Управление».** Отображаются все политики резервного копирования, настроенные в хранилище.

   ![Все политики резервного копирования в хранилище](./media/manage-afs-backup/all-backup-policies-modify.png)

1. Для просмотра политик, специфиченных для раздела файла Azure, выберите **раздел файлов Azure** из списка выпадающих в правом верхнем углу. Выберите политику резервного копирования, изменяемую.

   ![Совместное представление файлов Azure для изменения](./media/manage-afs-backup/azure-file-share-modify.png)

1. Открывается панель **расписания.** Изредьте **расписание резервного копирования** и **диапазон удержания** по мере необходимости и выберите **Сохранить.** Вы увидите сообщение "Обновление в прогрессе" в панели. После успешного обновления изменений политики вы увидите сообщение «Успешное обновление политики резервного копирования».

   ![Сохранить измененную политику](./media/manage-afs-backup/save-policy.png)

## <a name="stop-protection-on-a-file-share"></a>Остановка защиты файлового ресурса

Есть два способа отключить защиту файловых ресурсов Azure:

* Остановите все будущие задания резервного копирования и *удалите все точки восстановления.*
* Остановите все будущие задания резервного копирования, но *оставьте точки восстановления.*

Возможно, это может привести к удорожанию точек восстановления в хранилище, поскольку основные моменты, созданные резервным копированием Azure, будут сохранены. Преимущество выхода из точек восстановления заключается в том, что вы можете восстановить общую часть файла позже. Подробную информацию о стоимости выхода из пунктов восстановления можно узнать в [деталях ценообразования.](https://azure.microsoft.com/pricing/details/backup/) Если вы решите удалить все точки восстановления, вы не сможете восстановить общий доступ файла.

Чтобы отключить защиту файлового ресурса Azure, сделайте вот что:

1. Откройте хранилище служб восстановления, содержащее точки восстановления доли файла. Выберите **элементы резервного копирования** в разделе **Защищенные элементы.** Отображается список типов элементов резервного копирования.

   ![Резервные элементы](./media/manage-afs-backup/backup-items.png)

1. В списке **Backup Management Type** (Тип управления резервным копированием) выберите **Azure Storage (Azure Files)** (Служба хранилища Azure (файлы Azure)). Отображается список **элементов резервного копирования (Azure Storage (Azure Files).**

   ![Выберите хранилище Azure (Файлы Azure)](./media/manage-afs-backup/azure-storage-azure-files.png)

1. В списке **элементов резервного копирования (Azure Storage (Azure Files))** выберите элемент резервного копирования, для которого необходимо остановить защиту.

1. Выберите опцию **резервного копирования Stop.**

   ![Выбор "Остановить архивацию"](./media/manage-afs-backup/stop-backup.png)

1. В панели **резервного копирования Stop** выберите **Сохранить данные резервного копирования** или **удалить данные резервного копирования.** Затем выберите **Stop backup.**

    ![Выберите Сохранить данные резервного копирования или удалить данные резервного копирования](./media/manage-afs-backup/retain-or-delete-backup-data.png)

## <a name="resume-protection-on-a-file-share"></a>Возобновление защиты файлового ресурса

Если параметр **Retain Backup Data** был выбран при прекращении защиты для общего использования файла, можно возобновить защиту. Если выбран анотицизм **данных удаленного** отстава, защита для общего использования файла не может возобновиться.

Чтобы возобновить защиту для совместного файла Azure:

1. Откройте хранилище служб восстановления, содержащее точки восстановления доли файла. Выберите **элементы резервного копирования** в разделе **Защищенные элементы.** Отображается список типов элементов резервного копирования.

   ![Элементы резервного копирования для резюме](./media/manage-afs-backup/backup-items-resume.png)

1. В списке **Backup Management Type** (Тип управления резервным копированием) выберите **Azure Storage (Azure Files)** (Служба хранилища Azure (файлы Azure)). Отображается список **элементов резервного копирования (Azure Storage (Azure Files).**

   ![Список хранилища Azure (Файлы Azure)](./media/manage-afs-backup/azure-storage-azure-files.png)

1. В списке **элементов резервного копирования (Azure Storage (Azure Files))** выберите элемент резервного копирования, для которого требуется возобновить защиту.

1. Выберите опцию **резервного копирования Резюме.**

   ![Выберите резервное копирование резюме](./media/manage-afs-backup/resume-backup.png)

1. Открывается панель **политики резервного копирования.** Выберите выбранную по своему выбору политику для возобновления резервного копирования.

1. После выбора политики резервного копирования выберите **Сохранить**. Вы увидите сообщение "Обновление в прогрессе" на портале. После успешного возобновления резервного копирования будет видеться сообщение "Успешно обновленная политика резервного копирования для защищенного раздела файлов Azure".

   ![Успешно обновленная политика резервного копирования](./media/manage-afs-backup/successfully-updated.png)

## <a name="delete-backup-data"></a>удаление резервных копий;

Вы можете удалить резервную часть файла во время выполнения задания **резервного копирования Stop** или в любое время после остановки защиты. Это может быть полезно ждать дней или даже недель, прежде чем удалить точки восстановления. При удалении данных резервного копирования нельзя выбрать для удаления конкретные точки восстановления. Если вы решите удалить данные резервного копирования, вы удалите все точки восстановления, связанные с доле файла.

Следующая процедура предполагает, что защита была остановлена для доли файла.

Чтобы удалить данные резервного копирования для общего использования файла Azure:

1. После остановки задания резервного копирования параметры **резервного копирования** и **удаления данных по резервному** копированию доступны в панели мониторинга **элемента резервного копирования.** Выберите опцию **данных резервного копирования Delete.**

   ![удаление резервных копий;](./media/manage-afs-backup/delete-backup-data.png)

1. Открывается панель **данных резервного копирования Delete.** Введите имя раздела файла для подтверждения удаления. Дополнительно предоставьте дополнительную информацию в коробках **«Причина»** или **«Комментарии».** После того как вы уверены в удалении данных резервного копирования, выберите **Удалить**.

   ![Подтвердить удаление данных](./media/manage-afs-backup/confirm-delete-data.png)

## <a name="unregister-a-storage-account"></a>Отменить регистрацию учетной записи хранилища

Для защиты ваших файлов в определенном хранилище с помощью другого хранилища служб восстановления сначала [защиту остановки для всех файлов в](#stop-protection-on-a-file-share) этом счете хранения. Затем отменить учет из текущего хранилища служб восстановления, используемого для защиты.

Следующая процедура предполагает, что защита была остановлена для всех файлов акций на хранении счета, который вы хотите отменить.

Чтобы отменить регистрацию учетной записи хранилища:

1. Откройте хранилище служб восстановления, где зарегистрирована учетная запись хранилища.
1. На панели **Обзора** выберите опцию **инфраструктуры резервного копирования** в разделе **Управление.**

   ![Выбор "Инфраструктура резервного копирования"](./media/manage-afs-backup/backup-infrastructure.png)

1. Открывается панель **инфраструктуры резервного копирования.** Выберите **учетные записи хранения** в разделе **Учетные записи хранения Azure.**

   ![Выберите учетные записи хранения данных](./media/manage-afs-backup/storage-accounts.png)

1. После выбора **учетных записей хранения**появляется список учетных записей хранения, зарегистрированных в хранилище.
1. Справа щелкните учетную запись хранилища, который вы хотите отменить, и выберите **Unregister.**

   ![Выберите нерегистрировать](./media/manage-afs-backup/select-unregister.png)

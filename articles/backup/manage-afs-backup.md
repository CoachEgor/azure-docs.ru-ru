# <a name="manage-azure-file-share-backups"></a>Управление резервными копиями файловых ресурсов Azure

В этой статье описаны распространенные задачи по управлению и мониторингу файловых ресурсов Azure, которые архивируются службой [Azure Backup](https://docs.microsoft.com/azure/backup/backup-overview) . Вы узнаете, как выполнять следующие задачи управления в хранилище служб восстановления.

* [Мониторинг заданий](#monitor-jobs)
* [Создание политики](#create-a-new-policy).
* [Изменение политики](#modify-policy)
* [Остановка защиты файлового ресурса](#stop-protection-on-a-file-share).
* [Возобновление защиты файлового ресурса](#resume-protection-on-a-file-share).
* [Удаление данных резервных копий](#delete-backup-data).
* [Отмена регистрации учетной записи хранения](#unregister-storage-account)

## <a name="monitor-jobs"></a>Мониторинг заданий

При запуске операции резервного копирования или восстановления служба резервного копирования создает задание для отслеживания. Вы можете отслеживать ход выполнения всех заданий на странице **Задания резервного копирования**.

Чтобы открыть страницу **Задания резервного копирования**, выполните следующее:

1. Откройте хранилище служб восстановления, которое использовалось для настройки резервного копирования файловых ресурсов. В колонке **Обзор** щелкните **задания резервного копирования** в разделе **мониторинг** .

   ![Задания резервного копирования в разделе "Мониторинг"](./media/manage-afs-backup/backup-jobs.png)

2. После нажатия кнопки ОК отобразится колонка **задания резервного копирования** со списком состояния всех заданий. Можно щелкнуть имя рабочей нагрузки, соответствующую общей папке, которую необходимо отслеживать.

   ![Имя рабочей нагрузки](./media/manage-afs-backup/workload-name.png)

## <a name="create-a-new-policy"></a>Создание новой политики

Вы можете создать новую политику для резервного копирования файловых ресурсов Azure из раздела **политики архивации** в хранилище служб восстановления. Все политики, созданные при настройке резервного копирования для файловых ресурсов, отображаются с типом политики "файловый ресурс Azure".

Чтобы просмотреть имеющиеся политики резервного копирования, выполните следующее:

1. Откройте хранилище служб восстановления, которое использовалось для настройки резервного копирования для общей папки, и в меню хранилища служб восстановления щелкните **политики резервного копирования** в разделе Управление. Будут перечислены все политики резервного копирования, настроенные в хранилище.

   ![Все политики архивации](./media/manage-afs-backup/all-backup-policies.png)

2. Чтобы просмотреть политики, относящиеся к файловому ресурсу Azure, выберите **файловый ресурс Azure** в раскрывающемся списке в правом верхнем углу.

   ![Выберите файловый ресурс Azure](./media/manage-afs-backup/azure-file-share.png)

Чтобы создать политику резервного копирования, выполните следующее:

1. Щелкните **+ Добавить** в колонке политики архивации.

   ![Новая политика архивации](./media/manage-afs-backup/new-backup-policy.png)

2. Выберите **файловый ресурс Azure** в качестве **типа политики** в колонке **добавить** . Откроется колонка политика архивации для файлового ресурса Azure. Укажите имя политики, периодичность резервного копирования и диапазон хранения для точек восстановления. После определения политики нажмите кнопку **ОК** .

   ![Определение политики архивации](./media/manage-afs-backup/define-backup-policy.png)

## <a name="modify-policy"></a>Изменение политики

Вы можете изменить политику резервного копирования, чтобы изменить частоту резервного копирования или диапазон хранения.

Чтобы изменить политику, выполните следующие действия.

1. Откройте хранилище служб восстановления, которое использовалось для настройки резервного копирования для общей папки, и в меню хранилища служб восстановления щелкните **политики резервного копирования** в разделе "Управление". Будут перечислены все политики резервного копирования, настроенные в хранилище.

   ![Все политики архивации в хранилище](./media/manage-afs-backup/all-backup-policies-modify.png)

2. Чтобы просмотреть политики, относящиеся к общему файловому ресурсу Azure, выберите **файловый ресурс Azure** в раскрывающемся списке в правом верхнем углу. Выберите политику архивации, которую необходимо изменить.

   ![Файловый ресурс Azure для изменения](./media/manage-afs-backup/azure-file-share-modify.png)

3. Откроется колонка **Расписание** . Измените расписание или диапазон хранения резервных копий, как требуется, и нажмите кнопку **сохранить**. В колонке появится сообщение "выполняется обновление". После успешного обновления политики вы увидите сообщение "успешно обновили политику архивации".

   ![Сохранение измененной политики](./media/manage-afs-backup/save-policy.png)

## <a name="stop-protection-on-a-file-share"></a>Остановка защиты файлового ресурса

Есть два способа отключить защиту файловых ресурсов Azure:

* Завершите все будущие задания резервного копирования и *удалите все точки восстановления* .
* Останавливать все будущие задания резервного копирования *, но оставить точки восстановления*

При хранении точек восстановления в хранилище могут возникать затраты, поскольку базовые моментальные снимки, созданные Azure Backup, будут сохранены. Однако позже с помощью этих точек при необходимости можно восстановить файловый ресурс. Сведения о затратах на уход точек восстановления см. в [сведениях о ценах](https://azure.microsoft.com/pricing/details/backup/). Если вы удалите все точки восстановления, вы не сможете восстановить файловый ресурс.

Чтобы отключить защиту файлового ресурса Azure, сделайте вот что:

1. Откройте хранилище служб восстановления, содержащее точки восстановления общей папки, и щелкните **элемент резервного копирования** в разделе защищенные элементы. Откроется список типов элементов архивации.

   ![Архивные элементы](./media/manage-afs-backup/backup-items.png)

2. В списке **Backup Management Type** (Тип управления резервным копированием) выберите **Azure Storage (Azure Files)** (Служба хранилища Azure (файлы Azure)). Отобразится список **элементов архивации для (служба хранилища Azure (службы файлов Azure))** .

   ![Выбор службы хранилища Azure (службы файлов Azure)](./media/manage-afs-backup/azure-storage-azure-files.png)

3. В списке **элементов архивации (служба хранилища Azure (службы файлов Azure))** выберите элемент резервного копирования, для которого требуется отключить защиту.

4. Выберите параметр " **прерывать резервное копирование** " в меню колонки **архивации элемента** .

   ![Выбор "Остановить архивацию"](./media/manage-afs-backup/stop-backup.png)

5. В колонке " **Завершение архивации** " выберите вариант " **хранить данные архивации** " или " **удалить данные архивации** " и нажмите кнопку " **Отменить архивацию**"

    ![Выберите пункт "хранить или удалить данные архивации"](./media/manage-afs-backup/retain-or-delete-backup-data.png)

## <a name="resume-protection-on-a-file-share"></a>Возобновление защиты файлового ресурса

Если параметр « **хранить данные резервной копии** » был выбран при остановке защиты для файлового ресурса, то можно возобновить защиту. Если **данные резервного копирования были удалены**, возобновить защиту файлового ресурса нельзя.

Чтобы возобновить защиту для файлового ресурса Azure, сделайте следующее:

1. Откройте хранилище служб восстановления, содержащее точки восстановления общей папки, и щелкните **элемент резервного копирования** в разделе защищенные элементы. Откроется список типов элементов архивации.

   ![Элементы резервного копирования для возобновления](./media/manage-afs-backup/backup-items-resume.png)

2. В списке **Backup Management Type** (Тип управления резервным копированием) выберите **Azure Storage (Azure Files)** (Служба хранилища Azure (файлы Azure)). Отобразится список **элементов архивации для (служба хранилища Azure (службы файлов Azure))** .

   ![Список службы хранилища Azure (службы файлов Azure)](./media/manage-afs-backup/azure-storage-azure-files.png)

3. В списке **элементов архивации (служба хранилища Azure (службы файлов Azure))** выберите элемент резервного копирования, для которого требуется возобновить защиту.

4. Выберите параметр **возобновить резервное копирование** в меню колонки **элемента архивации** .

   ![Выбор возобновления резервного копирования](./media/manage-afs-backup/resume-backup.png)

5. Откроется колонка **Политика архивации** , и вы сможете выбрать политику для возобновления резервного копирования.

6. После выбора нужной **политики архивации**нажмите кнопку **сохранить** . На портале появится сообщение "выполняется обновление", а после успешного восстановления резервной копии появится сообщение "успешно обновлена политика резервного копирования для защищенного файлового ресурса Azure".

   ![Политика архивации успешно обновлена](./media/manage-afs-backup/successfully-updated.png)

## <a name="delete-backup-data"></a>удаление резервных копий;

Вы можете удалить резервную копию общего файлового ресурса во время задания **остановки резервного копирования** или в любое время после остановки защиты. Возможно, стоит подождать несколько дней или недель, прежде чем удалять точки восстановления. При удалении данных резервного копирования нельзя выбрать конкретные точки восстановления для удаления. Если вы решили удалить резервные копии данных, удалите все точки восстановления, связанные с общей папкой.

В следующей процедуре предполагается, что защита для общей папки остановлена.

Чтобы удалить данные архивации для файлового ресурса Azure, сделайте следующее:

1. После остановки задания резервного копирования на панели мониторинга **элемента архивации** доступны параметры **возобновить** резервное копирование и **удаление резервных копий данных** . Выберите параметр **удалить данные резервной копии** в меню колонки **резервной копии** .

   ![удаление резервных копий;](./media/manage-afs-backup/delete-backup-data.png)

2. Откроется колонка **Удаление данных архивации** . Введите имя общего файлового ресурса, чтобы подтвердить удаление. При необходимости укажите **причину** удаления или **комментария**. По завершении удаления данных архивации нажмите кнопку **Удалить** .

   ![Подтверждение удаления данных](./media/manage-afs-backup/confirm-delete-data.png)

## <a name="unregister-storage-account"></a>Отмена регистрации учетной записи хранения

Если вы хотите защитить файловые ресурсы в определенной учетной записи хранения с помощью другого хранилища служб восстановления, сначала [Отключите защиту для всех файловых ресурсов](#stop-protection-on-a-file-share) в этой учетной записи хранения. Затем отмените регистрацию учетной записи из текущего хранилища служб восстановления, используемого для защиты.

В следующей процедуре предполагается, что защита была остановлена для всех файловых ресурсов в учетной записи хранения, регистрация которых будет отменена.

Чтобы отменить регистрацию учетной записи хранения, выполните следующие действия.

1. Откройте хранилище служб восстановления, в котором зарегистрирована ваша учетная запись хранения.
2. Выберите вариант **инфраструктура резервного копирования** в разделе **Управление** в колонке **Обзор** .

   ![Щелкните инфраструктура резервного копирования](./media/manage-afs-backup/backup-infrastructure.png)

3. Откроется колонка **инфраструктура резервного копирования** . Щелкните **учетные записи хранения** в разделе **учетные записи хранения Azure** в этой колонке.

   ![Щелкните Учетные записи хранения.](./media/manage-afs-backup/storage-accounts.png)

4. Когда вы щелкните **учетные записи хранения**, отобразится список учетных записей хранения, зарегистрированных в хранилище.
5. Щелкните правой кнопкой мыши учетную запись хранения, для которой нужно отменить регистрацию, и выберите команду **отменить регистрацию**.

   ![Выбрать отмену регистрации](./media/manage-afs-backup/select-unregister.png)

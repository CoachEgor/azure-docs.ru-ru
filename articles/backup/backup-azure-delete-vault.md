# <a name="delete-an-azure-backup-recovery-services-vault"></a>Удаление хранилища служб восстановления резервного копирования Azure

В этой статье описывается, как удалить хранилище служб восстановления [резервного копирования Azure.](backup-overview.md) Он содержит инструкции по удалению зависимостей, а затем удалению хранилища.

## <a name="before-you-start"></a>Перед началом работы

Нельзя удалять хранилище служб восстановления, в которое есть зависимости, такие как защищенные серверы или серверы управления резервным копированием, связанные с ним.

- Убежища, содержащие данные резервного копирования, не могут быть удалены (даже если вы прекратили защиту, но сохранили данные резервного копирования).

- Если вы удалите хранилище, содержащее зависимости, появится следующее сообщение:

  ![Удалите ошибку хранилища.](./media/backup-azure-delete-vault/error.png)

- Если вы удалите защищенный элемент из предварительного помещения с портала, содержащего зависимости, появляется предупреждающее сообщение:

  ![Удалите ошибку защищенного сервера.](./media/backup-azure-delete-vault/error-message.jpg)

- Если элементы резервного копирования находятся в мягком удаленном состоянии, появляется предупреждение, и вам придется подождать, пока они навсегда не будут удалены. Для получения дополнительной информации смотрите эту [статью](https://docs.microsoft.com/azure/backup/backup-azure-security-feature-cloud).

   ![Удалите ошибку хранилища.](./media/backup-azure-delete-vault/error-message-soft-delete.png)

- Убежища, зарегистрированные учетные записи хранения, не могут быть удалены. Чтобы узнать, как отменить учетную запись, [см.](manage-afs-backup.md#unregister-a-storage-account)
  
Чтобы удалить хранилище, выберите сценарий, который соответствует вашей настройке, и выполните рекомендуемые действия:

Сценарий | Шаги для удаления зависимостей для удаления хранилища |

У меня есть внутренние машины, которые защищены с помощью MABS (Microsoft Azure Backup Server) или DPM (менеджер защиты данных System Center) в Azure | Выполните действия в [элементах резервного копирования Delete с консоли управления MABS](#delete-backup-items-from-the-mabs-management-console)

## <a name="delete-protected-items-in-the-cloud"></a>Удаление защищенных элементов в облаке

Во-первых, прочитайте раздел **[«Прежде чем начать»](#before-you-start)** для понимания зависимостей и процесса удаления хранилища.

Чтобы остановить защиту и удалить данные резервного копирования, выполните следующие действия:

1. С портала перейдите в **хранилище служб восстановления,** а затем перейдите к **элементам резервного копирования.** Затем выберите защищенные элементы в облаке (например, виртуальные машины Azure, хранилище Azure (служба Azure Files) или сервер S'L на виртуальных машинах Azure).

    ![Выберите тип резервного копирования.](./media/backup-azure-delete-vault/azure-storage-selected.png)

2. Нажмите правой кнопкой мыши, чтобы выбрать элемент резервного копирования. В зависимости от того, защищен элемент резервного копирования или нет, в меню отображается либо панель **резервного копирования Stop,** либо панель **данных резервного копирования Delete.**

    - При попаской панели **резервного копирования Stop** выберите **данные резервного копирования из** меню выпадения. Введите имя элемента резервного копирования (это поле является чувствительным к случаям), а затем выберите причину из меню выпадения. Введите свои комментарии, если у вас есть. Затем выберите **резервную пристани**.

        ![Панель резервного копирования stop.](./media/backup-azure-delete-vault/stop-backup-item.png)

    - При попавании панели **данных резервного копирования** , введите имя элемента резервного копирования (это поле является чувствительным к случаям), а затем выберите причину из меню выпадения. Введите свои комментарии, если у вас есть. Затем нажмите кнопку **Удалить**.

         ![Панель данных резервного копирования.](./media/backup-azure-delete-vault/stop-backup-blade-delete-backup-data.png)

3. Проверьте значок ![ **уведомления:** значок уведомления.](./media/backup-azure-delete-vault/messages.png) После завершения процесса служба отображает следующее сообщение: *Остановка резервного копирования и удаливание данных резервного копирования для «Элемента*резервного копирования *».* *Успешно завершена операция*.
4. Выберите **Обновление** в меню **элементов резервного копирования,** чтобы убедиться, что элемент резервного копирования был удален.

      ![Страница Элементы резервного копирования.](./media/backup-azure-delete-vault/empty-items-list.png)

## <a name="delete-protected-items-on-premises"></a>Удаление защищенных товаров в помещениях

Во-первых, прочитайте раздел **[«Прежде чем начать»](#before-you-start)** для понимания зависимостей и процесса удаления хранилища.

1. Из меню панели мониторинга хранилища выберите **инфраструктуру резервного копирования.**
2. В зависимости от вашего предварительного сценария, выберите один из следующих вариантов:

      - Для MARS выберите **защищенные серверы,** а затем **агент резервного копирования Azure.** Затем выберите сервер, который вы хотите удалить.

        ![Для MARS выберите хранилище, чтобы открыть свою панель мониторинга.](./media/backup-azure-delete-vault/identify-protected-servers.png)

      - Для MABS или DPM выберите **серверы управления резервным копированием.** Затем выберите сервер, который вы хотите удалить.

          ![Для MABS выберите хранилище, чтобы открыть панель мониторинга.](./media/backup-azure-delete-vault/delete-backup-management-servers.png)

3. Панель **Удаления** отображается с предупреждением.

     ![Удалять панель.](./media/backup-azure-delete-vault/delete-protected-server.png)

     Просмотрите предупреждающее сообщение и инструкции в флажке согласия.
    > [!NOTE]
    >- Если защищенный сервер синхронизирован с службами Azure и резервными элементами, в контрольном поле согласия будет отображаться количество зависимых элементов резервного копирования и ссылка для просмотра элементов резервного копирования.
    >- Если защищенный сервер не синхронизирован с службами Azure и элементы резервного копирования существуют, в контрольном поле согласия будет отображаться только количество элементов резервного копирования.
    >- Если нет элементов резервной копирования, флажок согласия будет просить об удалении.

4. Выберите флажок согласия, а затем выберите **Удалить**.

5. Проверьте **Notification** значок ![уведомления](./media/backup-azure-delete-vault/messages.png)удалить данные резервного копирования. После завершения операции служба отображает сообщение: *Остановка резервного копирования и удаливание данных резервного копирования для "Элемент резервного копирования".* *Успешно завершена операция*.
6. Выберите **Обновление** в меню **элементов резервного копирования,** чтобы убедиться, что элемент резервного копирования удален.

После завершения этого процесса можно удалить элементы резервного копирования из консоли управления:

- [Удаление элементов резервного копирования с консоли управления MARS](#delete-backup-items-from-the-mars-management-console)
- [Удаление элементов резервного копирования с консоли управления MABS](#delete-backup-items-from-the-mabs-management-console)

### <a name="delete-backup-items-from-the-mars-management-console"></a>Удаление элементов резервного копирования с консоли управления MARS

1. Откройте консоль управления MARS, перейдите на панель **действий** и выберите **Резервное копирование расписания.**
2. На странице **«Изменить или остановить запланированную страницу резервного копирования»** выберите **«Остановить использование этого графика резервного копирования» и удалить все сохраненные резервные upups.** Затем нажмите кнопку **Далее**.

    ![Измените или остановите запланированное резервное копирование.](./media/backup-azure-delete-vault/modify-schedule-backup.png)

3. На странице **«Остановить запланированное резервное копирование»** выберите **«Завершение».**

    ![Остановите запланированную резервную пристани.](./media/backup-azure-delete-vault/stop-schedule-backup.png)
4. Вам предлагается ввести PIN-код безопасности (личный идентификационный номер), который вы должны создать вручную. Для этого сначала вопием на портал Azure.
5. Перейдите к**свойствам** > **настройки** > хранилища служб **восстановления.**
6. Под **PIN-кодом безопасности**выберите **Generate**. Копируйте этот PIN-код. PIN-код действителен только в течение пяти минут.
7. В консоли управления вставьте PIN-код, а затем выберите **OK**.

    ![Создайте PIN-код безопасности.](./media/backup-azure-delete-vault/security-pin.png)

8. На странице **«Прогресс резервного копирования»** отображается следующее сообщение: *Удаленные данные резервного копирования будут храниться в течение 14 дней. По истечением времени данные резервного копирования будут удалены навсегда.*  

    ![Удалите инфраструктуру резервного копирования.](./media/backup-azure-delete-vault/deleted-backup-data.png)

После удаления элементов резервного копирования, следуйте следующим шагам с портала.

### <a name="delete-backup-items-from-the-mabs-management-console"></a>Удаление элементов резервного копирования с консоли управления MABS

Существует два метода, которые можно использовать для удаления элементов резервного копирования с консоли управления MABS.

#### <a name="method-1"></a>Метод 1

Чтобы остановить защиту и удалить данные резервного копирования, сделайте следующие действия:

1. Откройте консоль администратора DPM, а затем выберите **Защиту** на панели навигации.
2. На панели отображения выберите удаляемый член группы защиты. Нажмите правой кнопкой мыши, чтобы выбрать опцию **Stop Protection of Group.**
3. Из диалогового окна **Stop Protection** выберите **Удалить защищенные данные,** а затем выберите **онлайн-флажок для хранения delete.** Затем выберите **Stop Protection.**

    ![Выберите Удалить защищенные данные из панели Stop Protection.](./media/backup-azure-delete-vault/delete-storage-online.png)

    Защищенный статус участника изменяется на *неактивную реплику.*

4. Нажмите правой кнопкой неактивной группы защиты и выберите **Удалить неактивную защиту.**

    ![Удалите неактивную защиту.](./media/backup-azure-delete-vault/remove-inactive-protection.png)

5. Из окна **Delete Inactive Protection** выберите **флажок для хранения данных Delete,** а затем выберите **OK.**

    ![Удалить онлайн-хранилище.](./media/backup-azure-delete-vault/remove-replica-on-disk-and-online.png)

#### <a name="method-2"></a>Метод 2

Откройте консоль **управления MABS.** В **рамках метода защиты данных Select,** очистить **Я хочу онлайн** защитный флажок.

  ![Выберите способ защиты данных.](./media/backup-azure-delete-vault/data-protection-method.png)

После удаления элементов резервного копирования, следуйте следующим шагам с портала.

## <a name="delete-the-recovery-services-vault"></a>Удаление хранилища служб восстановления

1. Когда все зависимости удалены, прокрутите в меню Essentials панель **Essentials.**
2. Убедитесь, что в списке нет элементов резервного копирования, серверов управления резервным копированием или реплицированных элементов. Если элементы по-прежнему отображаются в хранилище, обратитесь к разделу [«Перед запуском».](#before-you-start)

3. Когда в хранилище больше нет элементов, выберите **Удалить** на панели мониторинга хранилища.

    ![Выберите Удалить на панели мониторинга хранилища.](./media/backup-azure-delete-vault/vault-ready-to-delete.png)

4. Выберите **Да,** чтобы убедиться, что вы хотите удалить хранилище. Хранилище удалено. Портал возвращается в **новое** меню сервиса.

## <a name="delete-the-recovery-services-vault-by-using-powershell"></a>Удалить хранилище служб восстановления с помощью PowerShell

Во-первых, прочитайте раздел **[«Прежде чем начать»](#before-you-start)** для понимания зависимостей и процесса удаления хранилища.

Чтобы остановить защиту и удалить данные резервного копирования:

- Если вы используете резервное копирование vMs-серверов Azure и обеспечивается автоматической защитой для экземпляров S'L, сначала отогнать автоматическую защиту.

    ```PowerShell
        Disable-AzRecoveryServicesBackupAutoProtection
           [-InputItem] <ProtectableItemBase>
           [-BackupManagementType] <BackupManagementType>
           [-WorkloadType] <WorkloadType>
           [-PassThru]
           [-VaultId <String>]
           [-DefaultProfile <IAzureContextContainer>]
           [-WhatIf]
           [-Confirm]
           [<CommonParameters>]
    ```

  [Узнайте больше](https://docs.microsoft.com/powershell/module/az.recoveryservices/disable-azrecoveryservicesbackupautoprotection?view=azps-2.6.0) о том, как отключить защиту для элемента, защищенного от резервного копирования Azure.

- Остановить защиту и удалить данные для всех защищенных от резервного копирования элементов в облаке (например. laaS VM, Azure File Share и т.д.):

    ```PowerShell
       Disable-AzRecoveryServicesBackupProtection
       [-Item] <ItemBase>
       [-RemoveRecoveryPoints]
       [-Force]
       [-VaultId <String>]
       [-DefaultProfile <IAzureContextContainer>]
       [-WhatIf]
       [-Confirm]
       [<CommonParameters>]
    ```

    [Подробнее](https://docs.microsoft.com/powershell/module/az.recoveryservices/disable-azrecoveryservicesbackupprotection?view=azps-2.6.0&viewFallbackFrom=azps-2.5.0) о защите отвозможности для элемента, защищенного от резервного копирования.

- Для защиты файлами и складками с помощью резервного агента Azure (MARS) для резервного копирования Azure используйте следующую команду PowerShell для удаления резервных данных из каждого модуля MARS PowerShell:

    ```powershell
    Get-OBPolicy | Remove-OBPolicy -DeleteBackup -SecurityPIN <Security Pin>
    ```

    Опубликовать, где появится следующий запрос:

    *Резервное копирование Microsoft Azure Вы уверены, что хотите удалить эту политику резервного копирования? Удаленные данные резервного копирования будут храниться в течение 14 дней. По истечением времени данные резервного копирования будут удалены навсегда. <br/> Да, да всем (N) Нет (L) Нет всем (S) Приостановить (S)? Справка (по умолчанию является "Y"):*

- Для внутренних машин, защищенных с помощью MABS (Microsoft Azure Backup Server) или DPM к Azure (менеджер системы центра защиты данных), используйте следующую команду для удаления резервных данных в Azure.

    ```powershell
    Get-OBPolicy | Remove-OBPolicy -DeleteBackup -SecurityPIN <Security Pin>
    ```

    Опубликовать, где появится следующий запрос:

   *Резервное копирование Microsoft Azure* Вы уверены, что хотите удалить эту политику резервного копирования? Удаленные данные резервного копирования будут храниться в течение 14 дней. По истечением времени данные резервного копирования будут удалены навсегда. <br/>
   Да, да всем (N) Нет (L) Нет всем (S) Приостановить (S)? Справка (по умолчанию является "Y") :*

После удаляния резервных данных оторегистрируйте любые контейнеры и серверы управления.

- Для защищенных файлов и складок с помощью резервного агента Azure (MARS) резервного копирования в Azure:

    ```PowerShell
    Unregister-AzRecoveryServicesBackupContainer
              [-Container] <ContainerBase>
              [-PassThru]
              [-VaultId <String>]
              [-DefaultProfile <IAzureContextContainer>]
              [-WhatIf]
              [-Confirm]
              [<CommonParameters>]
    ```

    [Подробнее](https://docs.microsoft.com/powershell/module/az.recoveryservices/unregister-azrecoveryservicesbackupcontainer?view=azps-2.6.0) об нерегистрации Windows-сервера или другого контейнера из хранилища.

- Для внутренних машин, защищенных с помощью MABS (Microsoft Azure Backup Server) или DPM в Azure (System Center Data Protection Manage:

    ```PowerShell
        Unregister-AzRecoveryServicesBackupManagementServer
          [-AzureRmBackupManagementServer] <BackupEngineBase>
          [-PassThru]
          [-VaultId <String>]
          [-DefaultProfile <IAzureContextContainer>]
          [-WhatIf]
          [-Confirm]
          [<CommonParameters>]
    ```

    [Подробнее](https://docs.microsoft.com/powershell/module/az.recoveryservices/unregister-azrecoveryservicesbackupcontainer?view=azps-2.6.0) об нерегистрации контейнера управления резервным копированием из хранилища.

После постоянного удаления резервных данных и снятия регистрации всех контейнеров приступим к удалению хранилища.

Удаление хранилища служб восстановления:

   ```PowerShell
       Remove-AzRecoveryServicesVault
      -Vault <ARSVault>
      [-DefaultProfile <IAzureContextContainer>]
      [-WhatIf]
      [-Confirm]
      [<CommonParameters>]
   ```

[Подробнее](https://docs.microsoft.com/powershell/module/az.recoveryservices/remove-azrecoveryservicesvault) об удалянии хранилища служб восстановления.

## <a name="delete-the-recovery-services-vault-by-using-cli"></a>Удалить хранилище служб восстановления с помощью CLI

Во-первых, прочитайте раздел **[«Прежде чем начать»](#before-you-start)** для понимания зависимостей и процесса удаления хранилища.

> [!NOTE]
> В настоящее время Azure Backup CLI поддерживает управление только резервными upups Azure VM, поэтому следующая команда удалить хранилище работает только в том случае, если хранилище содержит резервные ожидания Azure VM. Невозможно удалить хранилище с помощью CLI резервного копирования Azure, если хранилище содержит какой-либо элемент резервного копирования типа, кроме MM Azure.

Чтобы удалить существующее хранилище служб восстановления, выполните ниже:

- Чтобы остановить защиту и удалить данные резервного копирования

    ```azurecli
    az backup protection disable --container-name
                             --item-name
                             [--delete-backup-data {false, true}]
                             [--ids]
                             [--resource-group]
                             [--subscription]
                             [--vault-name]
                             [--yes]
    ```

    Для получения дополнительной информации смотрите эту [статью](/cli/azure/backup/protection#az-backup-protection-disable).

- Удалите существующее хранилище служб восстановления:

    ```azurecli
    az backup vault delete [--force]
                       [--ids]
                       [--name]
                       [--resource-group]
                       [--subscription]
                       [--yes]
    ```

    Для получения дополнительной информации смотрите эту [статью](https://docs.microsoft.com/cli/azure/backup/vault?view=azure-cli-latest)

## <a name="delete-the-recovery-services-vault-by-using-azure-resource-manager"></a>Удаление хранилища служб восстановления с помощью менеджера ресурсов Azure

Эта опция для удаления хранилища служб восстановления рекомендуется только в том случае, если все зависимости удалены и вы все еще получаете *ошибку удаления Хранилища.* Попробуйте любые или все следующие советы:

- Из панели Essentials в меню хранилища убедитесь, что нет никаких элементов резервного копирования, серверов управления **резервными копиями** или реплицированных элементов, перечисленных. Если есть элементы резервного копирования, направьтесь в раздел [«Перед запуском».](#before-you-start)
- Попробуйте [снова удалять хранилище с портала.](#delete-the-recovery-services-vault)
- Если все зависимости удалены и вы все еще получаете *ошибку удаления Vault,* используйте инструмент ARMClient для выполнения следующих шагов (после записи).

1. Перейти к [chocolatey.org](https://chocolatey.org/) скачать и установить Chocolatey. Затем установите ARMClient, запустив следующую команду:

   `choco install armclient --source=https://chocolatey.org/api/v2/`
2. Вопийте в учетную запись Azure, а затем запустите следующую команду:

    `ARMClient.exe login [environment name]`

3. На портале Azure соберите идентификатор подписки и название группы ресурсов для хранилища, которое вы хотите удалить.

Для получения дополнительной информации о [ARMClient README](https://github.com/projectkudu/ARMClient/blob/master/README.md)команде ARMClient см.

### <a name="use-the-azure-resource-manager-client-to-delete-a-recovery-services-vault"></a>Используйте клиент-менеджер ресурсов Azure для удаления хранилища служб восстановления

1. Выполнить следующую команду, используя идентификатор подписки, имя группы ресурсов и имя хранилища. Если у вас нет зависимостей, хранилище удаляется при запуске следующей команды:

   ```azurepowershell
   ARMClient.exe delete /subscriptions/<subscriptionID>/resourceGroups/<resourcegroupname>/providers/Microsoft.RecoveryServices/vaults/<recovery services vault name>?api-version=2015-03-15
   ```

2. Если хранилище не пусто, вы получите следующее сообщение об ошибке: *Убежище не может быть удалено, так как в этом хранилище есть существующие ресурсы.* Чтобы удалить защищенный элемент или контейнер в хранилище, запустите следующую команду:

   ```azurepowershell
   ARMClient.exe delete /subscriptions/<subscriptionID>/resourceGroups/<resourcegroupname>/providers/Microsoft.RecoveryServices/vaults/<recovery services vault name>/registeredIdentities/<container name>?api-version=2016-06-01
   ```

3. На портале Azure убедитесь, что хранилище удалено.

# <a name="security-features-to-help-protect-cloud-workloads-that-use-azure-backup"></a>Функции безопасности для защиты рабочих нагрузок облака, которые используют резервное копирование Azure

Среди пользователей растет обеспокоенность вопросами безопасности, связанными с вредоносными программами, программами-шантажистами и вторжениями в системы. Эти угрозы могут привести к значительным расходам и потере данных. Чтобы защититься от таких атак, Резервное копирование Azure теперь предоставляет функции безопасности, чтобы защитить данные резервного копирования даже после удаления.

Одной из таких функций является мягкое удаление. При мягком удалении, даже если вредоносный актер удаляет резервную часть VM (или данные резервного копирования случайно удаляются), данные резервного копирования сохраняются в течение 14 дополнительных дней, что позволяет восстановить этот элемент резервного копирования без потери данных. Дополнительные 14 дней хранения данных резервного копирования в состоянии "мягкого удаления" не несут никаких затрат для клиента. Azure также шифрует все резервные данные в состоянии покоя с помощью [шифрования служб хранения](https://docs.microsoft.com/azure/storage/common/storage-service-encryption) для дальнейшей защиты ваших данных.

Мягкая защита от удаления для виртуальных машин Azure, как правило, доступна.

>[!NOTE]
>Мягкое удаление для сервера S'L в Azure VM и мягкое удаление для SAP HANA в рабочих нагрузках Azure VM теперь доступно в предварительном просмотре.<br>
>Чтобы подписаться на предварительный просмотр, напишите нам по адресуAskAzureBackupTeam@microsoft.com

## <a name="soft-delete"></a>Мягкое удаление

### <a name="soft-delete-for-vms"></a>Мягкое удаление для VMs

Мягкое удаление для VM защищает резервные отпор ы вашего ввоза от непреднамеренного удаления. Даже после удаления резервных отставных, они сохраняются в состоянии мягкого удаления в течение 14 дополнительных дней.

> [!NOTE]
> Мягкое удаление защищает только удаленные данные резервного копирования. Если VM удаляется без резервной копирования, функция «мягкого удаления» не сохраняет данные. Все ресурсы должны быть защищены резервным копированием Azure, чтобы обеспечить полную устойчивость.
>

### <a name="supported-regions"></a>Поддерживаемые регионы

Мягкое удаление в настоящее время поддерживается в Западной Центральной Части США, Восточной Азии, Канаде Центральной, Канада Восток, Франция Центральной, Франция Южная, Корея Центральной, Корея South, Великобритания Запад, Австралия Восток, Австралия юго-восток, Северная Европа, Запад США, Запад US2, Центральный США, Восточная Азия, Северо-Центральная часть США, южная центральная часть США, Япония Восток, Япония Запад, Индия на юге, Индия Центральная, Индия Запад, Восток США 2, Швейцария Север, Швейцария Запад, и все национальные регионы.

### <a name="soft-delete-for-vms-using-azure-portal"></a>Мягкое удаление для VMs с помощью портала Azure

1. Чтобы удалить данные резервного копирования VM, резервное копирование должно быть остановлено. На портале Azure перейдите в хранилище служб восстановления, нажмите на элемент резервного копирования и выберите **резервное копирование Stop.**

   ![Скриншот элементов резервного копирования портала Azure](./media/backup-azure-security-feature-cloud/backup-stopped.png)

2. В следующем окне вам будет предоставлен выбор в пользу удаления или сохранения данных резервного копирования. Если вы выберете **данные резервного копирования,** а затем **остановите резервную,** резервное копирование VM не будет удалено навсегда. Скорее, данные резервного копирования будут храниться в течение 14 дней в мягком удаленном состоянии. При выборе **данных резервного копирования** Delete будет выбрано оповещение об удалении электронной почты, информирующее пользователя о том, что для резервного копирования остается 14 дней расширенного хранения данных резервного копирования. Кроме того, на 12-й день отправляется оповещение по электронной почте, информирующее о том, что для восстановления удаленных данных осталось еще два дня. Удаление откладывается до 15-го дня, когда произойдет постоянное удаление и отправляется окончательное оповещение по электронной почте, информирующее о постоянном удалении данных.

   ![Скриншот портала Azure, экран резервного копирования Stop](./media/backup-azure-security-feature-cloud/delete-backup-data.png)

3. В течение этих 14 дней в Хранилище Служб восстановления мягкий удаленный VM будет отображаться с красным значком «мягко-удалить» рядом с ним.

   ![Скриншот портала Azure, VM в мягком состоянии удаления](./media/backup-azure-security-feature-cloud/vm-soft-delete.png)

   > [!NOTE]
   > Если в хранилище присутствуют элементы резервного копирования с мягким удалением, хранилище не может быть удалено в это время. Пожалуйста, попробуйте удалить хранилище после того, как элементы резервного копирования навсегда удалены, и в хранилище не осталось элемента в мягком удаленном состоянии.

4. Чтобы восстановить мягкий удаленный VM, он должен быть сначала удален. Чтобы удалить, выберите мягкий удаленный VM, а затем выберите опцию **Undelete.**

   ![Скриншот портала Azure, Undelete VM](./media/backup-azure-security-feature-cloud/choose-undelete.png)

   Окно будет отображаться предупреждение о том, что при выборе неудаляемого, все точки восстановления для VM будут удалены и доступны для выполнения операции восстановления. VM будет сохранен в состоянии "стоп-защиты с сохранением данных" с приостановленными резервными ожиданиями и резервными данными, сохраненными навсегда без эффективной политики резервного копирования.

   ![Скриншот портала Azure, Подтвердите неудаление VM](./media/backup-azure-security-feature-cloud/undelete-vm.png)

   На этом этапе можно также восстановить VM, выбрав **Restore VM** из выбранной точки восстановления.  

   ![Скриншот портала Azure, опция Восстановление VM](./media/backup-azure-security-feature-cloud/restore-vm.png)

   > [!NOTE]
   > Сборщик мусора будет работать и очищать истекаемые точки восстановления только после того, как пользователь выполнит операцию **резервного копирования Резюме.**

5. После завершения процесса удаления статус аверии вернется в "Остановка резервного копирования с сохранением данных", а затем вы можете выбрать **резервное копирование резюме.** Операция **резервного копирования Резюме** возвращает элемент резервного копирования в активном состоянии, связанный с политикой резервного копирования, выбранной пользователем, определяющим графики резервного копирования и удержания.

   ![Скриншот портала Azure, опция резервного копирования резюме](./media/backup-azure-security-feature-cloud/resume-backup.png)

На этой диаграмме потока отображаются различные шаги и состояния элемента резервного копирования при включении Soft Delete:

![Срок службы элемента резервного копирования с мягким удалением](./media/backup-azure-security-feature-cloud/lifecycle.png)

### <a name="soft-delete-for-vms-using-azure-powershell"></a>Мягкое удаление для VMs с помощью Azure PowerShell

> [!IMPORTANT]
> Версия Az.RecoveryServices, необходимая для использования мягкого удаления с помощью Azure PS, составляет 2.2.0. Используйте ```Install-Module -Name Az.RecoveryServices -Force``` для получения последней версии.

Как указано выше для портала Azure, последовательность шагов одинакова и при использовании Azure PowerShell.

#### <a name="delete-the-backup-item-using-azure-powershell"></a>Удаление элемента резервного копирования с помощью Azure PowerShell

Удалите элемент резервного копирования с помощью [Disable-AzRecoveryServicesBackUpProtection](https://docs.microsoft.com/powershell/module/az.recoveryservices/Disable-AzRecoveryServicesBackupProtection?view=azps-1.5.0) PS cmdlet.

```powershell
Disable-AzRecoveryServicesBackupProtection -Item $myBkpItem -RemoveRecoveryPoints -VaultId $myVaultID -Force

WorkloadName     Operation            Status               StartTime                 EndTime                   JobID
------------     ---------            ------               ---------                 -------                   -----
AppVM1           DeleteBackupData     Completed            12/5/2019 12:44:15 PM     12/5/2019 12:44:50 PM     0488c3c2-accc-4a91-a1e0-fba09a67d2fb
```

Элемент резервного копирования 'DeleteState' будет изменяться с "NotDeleted" на "ToBeDeleted". Данные резервного копирования будут храниться в течение 14 дней. Если вы хотите вернуть операцию удаления, то следует отменить удаление.

#### <a name="undoing-the-deletion-operation-using-azure-powershell"></a>Отмена операции удаления с помощью Azure PowerShell

Во-первых, возьмите соответствующий элемент резервного копирования, который находится в состоянии мягкого удаления (т.е. вот-вот будет удален).

```powershell

Get-AzRecoveryServicesBackupItem -BackupManagementType AzureVM -WorkloadType AzureVM -VaultId $myVaultID | Where-Object {$_.DeleteState -eq "ToBeDeleted"}

Name                                     ContainerType        ContainerUniqueName                      WorkloadType         ProtectionStatus     HealthStatus         DeleteState
----                                     -------------        -------------------                      ------------         ----------------     ------------         -----------
VM;iaasvmcontainerv2;selfhostrg;AppVM1    AzureVM             iaasvmcontainerv2;selfhostrg;AppVM1       AzureVM              Healthy              Passed               ToBeDeleted

$myBkpItem = Get-AzRecoveryServicesBackupItem -BackupManagementType AzureVM -WorkloadType AzureVM -VaultId $myVaultID -Name AppVM1
```

Затем выполните операцию отсвагивания с помощью [undo-AzRecoveryRecoveryBackUpItemDeletion](https://docs.microsoft.com/powershell/module/az.recoveryservices/undo-azrecoveryservicesbackupitemdeletion?view=azps-3.1.0) PS cmdlet.

```powershell
Undo-AzRecoveryServicesBackupItemDeletion -Item $myBKpItem -VaultId $myVaultID -Force

WorkloadName     Operation            Status               StartTime                 EndTime                   JobID
------------     ---------            ------               ---------                 -------                   -----
AppVM1           Undelete             Completed            12/5/2019 12:47:28 PM     12/5/2019 12:47:40 PM     65311982-3755-46b5-8e53-c82ea4f0d2a2
```

Элемент резервного копирования 'DeleteState' вернется к 'NotDeleted'. Но защита по-прежнему остановлена. [Возобновить резервную работу](https://docs.microsoft.com/azure/backup/backup-azure-vms-automation#change-policy-for-backup-items) для повторного включения защиты.

## <a name="disabling-soft-delete"></a>Отключение мягкого удаления

Мягкое удаление по умолчанию включено в недавно созданные хранилища для защиты данных резервного копирования от случайных или вредоносных удалений.  Отключение этой функции не рекомендуется. Единственное обстоятельство, при котором вы должны рассмотреть возможность отключения мягкого удаления, это если вы планируете перенести защищенные элементы в новое хранилище, и не можете ждать 14 дней, необходимых для удаления и повторной защиты (например, в тестовой среде). Только владелец хранилища может отключить эту функцию. Если отключить эту функцию, все будущие удаления защищенных элементов приведут к немедленному удалению без возможности восстановления. Резервные данные, которые существуют в мягком удаленном состоянии до отключения этой функции, будут оставаться в мягком удаленном состоянии в течение 14 дней. Если вы хотите навсегда удалить их немедленно, то вам нужно удалить и удалить их снова, чтобы получить постоянно удалены.

### <a name="disabling-soft-delete-using-azure-portal"></a>Отключение мягкого удаления с помощью портала Azure

Чтобы отключить мягкое удаление, выполните следующие действия:

1. На портале Azure перейдите в хранилище, а затем перейдите в **Settings** -> **Properties**.
2. В панели свойств выберите**обновление** **настроек безопасности.** ->   
3. В настройках безопасности панели, под **Soft Delete**, выберите **отключить.**

![Отключить мягкое удаление](./media/backup-azure-security-feature-cloud/disable-soft-delete.png)

### <a name="disabling-soft-delete-using-azure-powershell"></a>Отключение мягкого удаления с помощью Azure PowerShell

Чтобы отключить, используйте [Set-AzRecoveryRecoveryVaultBackUpPropertyPROPERTY](https://docs.microsoft.com/powershell/module/az.recoveryservices/set-azrecoveryservicesbackupproperty?view=azps-3.1.0) PS cmdlet.

## <a name="permanently-deleting-soft-deleted-backup-items"></a>Постоянное удаление мягких удаленных элементов резервного копирования

Резервное копирование данных в мягком удаленном состоянии, предварительно отключенном от этой функции, останется в мягком удаленном состоянии. Если вы хотите навсегда удалить их немедленно, а затем удалить и удалить их снова, чтобы получить навсегда удалены.

### <a name="using-azure-portal"></a>Использование портала Azure

Выполните следующие действия.

1. Следуйте шагам, чтобы [отключить мягкое удаление.](#disabling-soft-delete)
2. На портале Azure перейдите в хранилище, перейдите на **резервные элементы**и выберите мягкий удаленный VM.

   ![Выберите мягкий удаленный VM](./media/backup-azure-security-feature-cloud/vm-soft-delete.png)

3. Выберите опцию **Undelete**.

   ![Выберите Undelete](./media/backup-azure-security-feature-cloud/choose-undelete.png)

4. Появится окно. Выберите **Undelete**.

   ![Выберите Неудализм](./media/backup-azure-security-feature-cloud/undelete-vm.png)

5. Выберите **Удалить данные резервного копирования,** чтобы навсегда удалить данные резервного копирования.

   ![Выберите данные резервного копирования Удалить](https://docs.microsoft.com/azure/backup/media/backup-azure-manage-vms/delete-backup-buttom.png)

6. Введите имя элемента резервного копирования, чтобы подтвердить, что вы хотите удалить точки восстановления.

   ![Введите имя элемента резервного копирования](https://docs.microsoft.com/azure/backup/media/backup-azure-manage-vms/delete-backup-data1.png)

7. Чтобы удалить данные резервного копирования для элемента, выберите **Удалить**. Сообщение об уведомлении позволяет узнать, что данные резервного копирования были удалены.

### <a name="using-azure-powershell"></a>Использование Azure PowerShell

Если элементы были удалены до того, как было отключено немягкое удаление, то они будут находиться в мягком удаленном состоянии. Чтобы немедленно удалить их, операция удаления должна быть отменена, а затем выполнена снова.

Определите элементы, которые находятся в состоянии с мягким удалением.

```powershell

Get-AzRecoveryServicesBackupItem -BackupManagementType AzureVM -WorkloadType AzureVM -VaultId $myVaultID | Where-Object {$_.DeleteState -eq "ToBeDeleted"}

Name                                     ContainerType        ContainerUniqueName                      WorkloadType         ProtectionStatus     HealthStatus         DeleteState
----                                     -------------        -------------------                      ------------         ----------------     ------------         -----------
VM;iaasvmcontainerv2;selfhostrg;AppVM1    AzureVM             iaasvmcontainerv2;selfhostrg;AppVM1       AzureVM              Healthy              Passed               ToBeDeleted

$myBkpItem = Get-AzRecoveryServicesBackupItem -BackupManagementType AzureVM -WorkloadType AzureVM -VaultId $myVaultID -Name AppVM1
```

Затем отмените операцию удаления, которая была выполнена при включении мягкого удаления.

```powershell
Undo-AzRecoveryServicesBackupItemDeletion -Item $myBKpItem -VaultId $myVaultID -Force

WorkloadName     Operation            Status               StartTime                 EndTime                   JobID
------------     ---------            ------               ---------                 -------                   -----
AppVM1           Undelete             Completed            12/5/2019 12:47:28 PM     12/5/2019 12:47:40 PM     65311982-3755-46b5-8e53-c82ea4f0d2a2
```

Поскольку "мягкое удаление" отключено, операция удаления приведет к немедленному удалению данных резервного копирования.

```powershell
Disable-AzRecoveryServicesBackupProtection -Item $myBkpItem -RemoveRecoveryPoints -VaultId $myVaultID -Force

WorkloadName     Operation            Status               StartTime                 EndTime                   JobID
------------     ---------            ------               ---------                 -------                   -----
AppVM1           DeleteBackupData     Completed            12/5/2019 12:44:15 PM     12/5/2019 12:44:50 PM     0488c3c2-accc-4a91-a1e0-fba09a67d2fb
```

3. Затем удалите резервные отпор ы с помощью REST API, как упоминалось [здесь.](backup-azure-arm-userestapi-backupazurevms.md#stop-protection-and-delete-data)

## <a name="encryption"></a>Шифрование

Все резервные данные автоматически шифруются при хранении в облаке с помощью шифрования Azure Storage, что помогает выполнять свои обязательства по обеспечению безопасности и соответствию требованиям. Эти данные в состоянии покоя шифруются с помощью 256-битного шифрования AES, одного из самых сильных доступных шифров блоков, и соответствуют требованиям FIPS 140-2.

В дополнение к шифрованию в состоянии покоя, все ваши данные резервного копирования в пути передается через HTTPS. Он всегда остается в сети Azure.

Для получения дополнительной [Azure Storage encryption for data at rest](https://docs.microsoft.com/azure/storage/common/storage-service-encryption)информации см. Обратитесь к [часто задаваемым вопросам резервного копирования Azure,](https://docs.microsoft.com/azure/backup/backup-azure-backup-faq#encryption) чтобы ответить на любые вопросы, которые могут возникнуть у вас о шифровании.

### <a name="encryption-of-backup-data-using-platform-managed-keys"></a>Шифрование данных резервного копирования с помощью ключей, управляемых платформой

По умолчанию все ваши данные шифруются с помощью управляемых платформой ключей. Вам не нужно предпринимать никаких явных действий с конца, чтобы включить это шифрование, и это относится ко всем рабочим нагрузкам, резервным в хранилище служб восстановления.

### <a name="encryption-of-backup-data-using-customer-managed-keys"></a>Шифрование данных резервного копирования с помощью ключей, управляемых клиентом

При резервном копировании виртуальных машин Azure теперь можно шифровать данные с помощью ключей, принадлежащих и управляемых вами. Резервное копирование Azure позволяет использовать клавиши RSA, хранящиеся в Хранилище ключей Azure, для шифрования резервных данных. Ключ шифрования, используемый для шифрования резервных данных, может отличаться от ключа, используемого для источника. Данные защищены с помощью ключа шифрования данных AES 256 (DEK), который, в свою очередь, защищен с помощью ключей. Это дает вам полный контроль над данными и ключами. Для обеспечения шифрования необходимо предоставить хранилище служб восстановления доступ к ключу шифрования в Хранилище ключей Azure. Вы можете отключить ключ или отозвать доступ, когда это необходимо. Тем не менее, вы должны включить шифрование с помощью ключей, прежде чем пытаться защитить любые элементы в хранилище.

>[!NOTE]
>Эта функция в настоящее время в ограниченной доступности. Пожалуйста, заполните AskAzureBackupTeam@microsoft.com этот [опрос](https://forms.microsoft.com/Pages/ResponsePage.aspx?id=v4j5cvGGr0GRqy180BHbR0H3_nezt2RNkpBCUTbWEapURE9TTDRIUEUyNFhNT1lZS1BNVDdZVllHWi4u) и напишите нам, если вы хотите шифровать данные резервного копирования с помощью ключей, управляемых клиентом. Обратите внимание, что возможность использования этой функции подлежит утверждению службой резервного копирования Azure.

### <a name="backup-of-managed-disk-vms-encrypted-using-customer-managed-keys"></a>Резервное копирование управляемых дисковых вм, зашифрованных с помощью ключей, управляемых клиентами

Резервное копирование Azure также позволяет создать резервную связь с VMs-серверами Azure, которые используют ключ для шифрования на стороне сервера. Ключ, используемый для шифрования дисков, хранится в Хранилище ключей Azure и управляется вами. Шифрование на стороне сервера с использованием управляемых клиентом ключей отличается от шифрования Azure Disk, так как ADE использует BitLocker (для Windows) и DM-Crypt (для Linux) для выполнения шифрования в госте, SSE шифрует данные в службе хранения, что позволяет использовать любую ОС или изображения для вашего VMs. Обратитесь к [шифрованию управляемых дисков с управляемыми ключами клиента](https://docs.microsoft.com/azure/virtual-machines/windows/disk-encryption#customer-managed-keys) для получения более подробной информации.

### <a name="backup-of-vms-encrypted-using-ade"></a>Резервное копирование vMs, зашифрованных с помощью ADE

С помощью резервного копирования Azure вы также можете создать резервную связь с виртуальными машинами Azure, которые имеют шифрование Своих ОС или дисков данных с помощью шифрования Azure Disk. ADE использует BitLocker для Windows VMs и DM-Crypt для Linux VMs для выполнения шифрования в гостях. Для получения подробной информации просмотрите [резервное копирование и восстановите зашифрованные виртуальные машины с помощью резервного копирования Azure.](https://docs.microsoft.com/azure/backup/backup-azure-vms-encryption)

## <a name="frequently-asked-questions"></a>Часто задаваемые вопросы

### <a name="for-soft-delete"></a>Для мягкого удаления

#### <a name="do-i-need-to-enable-the-soft-delete-feature-on-every-vault"></a>Нужно ли включать функцию «мягкого удаления» в каждом хранилище?

Нет, он построен и включен по умолчанию для всех хранилищ служб восстановления.

#### <a name="can-i-configure-the-number-of-days-for-which-my-data-will-be-retained-in-soft-deleted-state-after-delete-operation-is-complete"></a>Могу ли я настроить количество дней, в течение которых мои данные будут храниться в состоянии с мягким удалением после завершения операции по удалению?

Нет, это зафиксировано на 14 дней дополнительного удержания после операции удаления.

#### <a name="do-i-need-to-pay-the-cost-for-this-additional-14-day-retention"></a>Нужно ли мне оплачивать расходы на это дополнительное 14-дневное удержание?

Нет, это 14-дневное дополнительное удержание поставляется бесплатно как часть функции мягкого удаления.

#### <a name="can-i-perform-a-restore-operation-when-my-data-is-in-soft-delete-state"></a>Могу ли я выполнить операцию восстановления, когда мои данные в мягком состоянии удаления?

Нет, для восстановления необходимо удалить мягкий удаленный ресурс. Операция неудаления вернет ресурс обратно в **защиту Stop с состоянием хранения данных,** где можно восстановить его в любой момент времени. Сборщик мусора остается приостановленным в этом состоянии.

#### <a name="will-my-snapshots-follow-the-same-lifecycle-as-my-recovery-points-in-the-vault"></a>Будут ли мои снимки следовать тому же жизненному циклу, что и точки восстановления в хранилище?

Да.

#### <a name="how-can-i-trigger-the-scheduled-backups-again-for-a-soft-deleted-resource"></a>Как запустить запланированные резервные ожидания снова для слабоудаленного ресурса?

Неудаление с последующим возобновлением операции снова защитит ресурс. Возобновление операции связывает политику резервного копирования для запуска запланированных резервных upups с выбранным периодом удержания. Кроме того, сборщик мусора запускается, как только операция возобновления завершается. Если вы хотите выполнить восстановление из точки восстановления, которая истекает, рекомендуется сделать это перед запуском операции возобновления.

#### <a name="can-i-delete-my-vault-if-there-are-soft-deleted-items-in-the-vault"></a>Могу ли я удалить хранилище, если в хранилище есть мягкие удаленные элементы?

Хранилище служб восстановления не может быть удалено, если в хранилище есть элементы резервного копирования в состоянии с мягким удалением. Элементы с мягким удалением навсегда удаляются через 14 дней после операции по удалению. Если вы не можете ждать 14 дней, а затем [отключить мягкое удаление,](#disabling-soft-delete)удалить мягкие удаленные элементы, и удалить их снова, чтобы навсегда получить удалены. После обеспечения того, чтобы не было защищенных элементов и мягких удаленных элементов, хранилище может быть удалено.  

#### <a name="can-i-delete-the-data-earlier-than-the-14-days-soft-delete-period-after-deletion"></a>Могу ли я удалить данные раньше 14-дневного периода мягкого удаления после удаления?

Нет. Вы не можете заставить удалить элементы с мягким удалением, они автоматически удаляются через 14 дней. Эта функция безопасности позволяет защитить резервные данные от случайных или вредоносных удалений.  Вы должны ждать 14 дней, прежде чем выполнять любые другие действия на VM.  Мягкие удаленные элементы не будут взиматься.  Если вам необходимо повторно защитить ВМ, отмеченные для мягкого удаления в течение 14 дней, в новое хранилище, обратитесь в службу поддержки Майкрософт.

#### <a name="can-soft-delete-operations-be-performed-in-powershell-or-cli"></a>Можно ли выполнять операции мягкого удаления в PowerShell или CLI?

Операции мягкого удаления могут выполняться с помощью [PowerShell.](#soft-delete-for-vms-using-azure-powershell) В настоящее время CLI не поддерживается.

#### <a name="is-soft-delete-supported-for-other-cloud-workloads-like-sql-server-in-azure-vms-and-sap-hana-in-azure-vms"></a>Поддерживается ли мягкое удаление для других рабочих нагрузок облака, таких как сервер S'L в Azure VMs и SAP HANA в Azure VMs?

Нет. В настоящее время мягкое удаление поддерживается только для виртуальных машин Azure.

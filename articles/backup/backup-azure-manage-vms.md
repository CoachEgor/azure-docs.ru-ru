# <a name="manage-azure-vm-backups"></a>Управление резервными копиями виртуальных машин Azure

В этой статье описывается управление виртуальными машинами Azure (ВМ), резервное копирование с помощью [службы архивации Azure](backup-overview.md). Статья также содержит сведения о резервной копии сведения, которые можно найти на панели мониторинга хранилища.


На портале Azure, панели мониторинга хранилища служб восстановления предоставляет доступ к хранилище сведения, включая:

* последняя резервная копия, которая одновременно является последней точкой восстановления;
* Политику резервного копирования.
* Общий размер всех моментальных снимков резервных копий.
* Число виртуальных машин, которые включены для резервных копий.

Резервные копии можно управлять с помощью панели мониторинга и детализации до отдельных виртуальных машин. Чтобы начать резервное копирование компьютера, откройте хранилище на панели мониторинга.

![Полноценная информационная панель представление с ползунком](./media/backup-azure-manage-vms/bottom-slider.png)

## <a name="view-vms-on-the-dashboard"></a>Просмотр виртуальных машин на панели мониторинга

Чтобы просмотреть виртуальные машины на панели мониторинга хранилища:

1. Войдите на [портале Azure](https://portal.azure.com/).
2. В главном меню выберите **Обзор**. В списке ресурсов введите **Службы восстановления**. По мере ввода список фильтрация выполняется на основе входных данных. Выберите **Хранилища служб восстановления**.

    ![Создание хранилища служб восстановления](./media/backup-azure-manage-vms/browse-to-rs-vaults.png)

3. Для удобства использования, щелкните правой кнопкой мыши хранилище и выберите **закрепить на панели мониторинга**.
4. Откройте панель мониторинга хранилища.

    ![Откройте панель мониторинга хранилища и в колонке "Параметры"](./media/backup-azure-manage-vms/full-view-rs-vault.png)

5. На **архивные элементы** плитки, выберите **виртуальных машинах**.

    ![Откройте плитку архивные элементы](./media/backup-azure-manage-vms/contoso-vault-1606.png)

6. На **архивные элементы** колонке вы можете просмотреть список защищенных виртуальных машин. В этом примере хранилище обеспечивает защиту одной виртуальной машины: demobackup.  

    ![Просмотрите колонку архивные элементы](./media/backup-azure-manage-vms/backup-items-blade-select-item.png)

7. Из панели мониторинга элемента хранилища изменение политики резервного копирования, запуск по требованию резервного копирования, Стоп или возобновить защиту виртуальных машин, удалить данные архивации, просматривать точки восстановления и запуска задания восстановления.

    ![Архивные элементы панели мониторинга и колонка "Параметры"](./media/backup-azure-manage-vms/item-dashboard-settings.png)

## <a name="manage-backup-policy-for-a-vm"></a>Управление политикой резервного копирования для виртуальной Машины

Для управления политику резервного копирования:

1. Войдите на [портале Azure](https://portal.azure.com/). Откройте панель мониторинга хранилища.
2. На **архивные элементы** плитки, выберите **виртуальных машинах**.

    ![Откройте плитку архивные элементы](./media/backup-azure-manage-vms/contoso-vault-1606.png)

3. На **архивных элементов** колонке, можно просмотреть список защищенных виртуальных машин и состояние последней архивации с последней время точки восстановления.

    ![Просмотрите колонку архивные элементы](./media/backup-azure-manage-vms/backup-items-blade-select-item.png)

4. На панели мониторинга элемента хранилища можно выбрать политику резервного копирования.

   * Для переключения политики, выберите другую политику и затем выберите **Сохранить**. Новая политика будет немедленно применена к хранилищу.

     ![Выберите политику резервного копирования](./media/backup-azure-manage-vms/backup-policy-create-new.png)

## <a name="run-an-on-demand-backup"></a>Выполнение резервного копирования по требованию
После настройки защиты, можно запустить из резервной копии по запросу виртуальной Машины. Эти сведения не забывайте:

- Если начальное резервное копирование остается незавершенным, резервное копирование по запросу создает полную копию виртуальной машины в хранилище служб восстановления.
- Если начальное резервное копирование завершено, резервное копирование по требованию будет отправлять только изменения после предыдущего снимка в хранилище служб восстановления. То есть последующие резервные копии являются добавочными.
- Диапазон хранения для резервного копирования по запросу — это значение срока хранения, укажите, когда вы инициируете резервное копирование.

Чтобы запустить резервное копирование по запросу, сделайте следующее:

1. На [панели мониторинга элемента хранилища](#view-vms-on-the-dashboard)в разделе **защищенного элемента**выберите **архивируемый элемент**.

    ![Теперь параметр резервного копирования](./media/backup-azure-manage-vms/backup-now-button.png)

2. Из **типом управления архивацией**выберите **виртуальной машине Azure**. **Элементов архивации (виртуальная машина Azure)** появится колонка.
3. Выберите виртуальную Машину и выберите **создать резервную копию сейчас** создать резервную копию по запросу. **Моментальная Архивация** появится колонка.
4. В **хранить архивные копии до** укажите дату сохранения архивной копии.

    ![Календарь моментальная Архивация](./media/backup-azure-manage-vms/backup-now-check.png)

5. Выберите **ОК** для запуска задания резервного копирования.

Для отслеживания хода выполнения задания, на панели мониторинга хранилища выберите **задания резервного копирования** плитку.

## <a name="stop-protecting-a-vm"></a>Отключение защиты для виртуальной машины

Существует два способа остановить защиту виртуальной Машины:

* **Остановите защиту и сохранить данные архивации**. Выбор этого варианта остановит все будущие операции резервного копирования от защиты виртуальной Машины; Тем не менее служба архивации Azure будет сохранять точки восстановления, которые будут заархивированы.  Необходимо будет платить за сохранить точки восстановления в хранилище (см. в разделе [цены на Azure Backup](https://azure.microsoft.com/pricing/details/backup/) сведения). Вы сможете восстановить виртуальную Машину, при необходимости. Если принято решение возобновить защиту виртуальных Машин, то можно использовать *возобновить резервное копирование* параметр.
* **Остановка защиты и удаление данных резервных копий**. Этот параметр будет остановить все будущие операции резервного копирования от защиты виртуальной Машины и удалите все точки восстановления. Вы не сможете восстановить виртуальную Машину и использует *возобновить резервное копирование* параметр.

>[!NOTE]
>При удалении источника данных без остановки резервного копирования, новое резервное копирование завершится ошибкой. Согласно политике срока действия старых точек восстановления, но один из последней точки восстановления всегда храниться, пока резервные копии остановки и удаления данных.
>

### <a name="stop-protection-and-retain-backup-data"></a>Остановите защиту и сохранить данные архивации

Для остановки защиты и хранения данных виртуальной машины:

1. На [мониторинга элемента хранилища](#view-vms-on-the-dashboard)выберите **остановить архивацию**.
2. Выберите **данные резервного копирования**и подтвердите свой выбор, при необходимости. Если нужно добавьте комментарий. Если вы не уверены в правильности имени элемента, наведите указатель мыши восклицательный знак, чтобы просмотреть имя.

    ![Хранить данные резервного копирования](./media/backup-azure-manage-vms/retain-backup-data.png)

Уведомление позволяет узнать, что задания резервного копирования были остановлены.

### <a name="stop-protection-and-delete-backup-data"></a>Остановка защиты и удаление данных резервных копий

Остановите защиту и удалить данные виртуальной машины:

1. На [мониторинга элемента хранилища](#view-vms-on-the-dashboard)выберите **остановить архивацию**.
2. Выберите **удаление данных архивации**и подтвердите свой выбор, при необходимости. Введите имя архивируемого элемента и при необходимости добавить комментарий.

    ![удаление резервных копий;](./media/backup-azure-manage-vms/delete-backup-data1.png)

## <a name="resume-protection-of-a-vm"></a>Возобновление защиты виртуальной машины

При выборе [остановите защиту и сохранить данные архивации](#stop-protection-and-retain-backup-data) параметр во время остановки защиты виртуальной Машины, то можно использовать **возобновить резервное копирование**. Этот параметр недоступен при выборе [остановка защиты и удаление данных резервных копий](#stop-protection-and-delete-backup-data) параметр или [удалить данные архивации](#delete-backup-data).

Чтобы возобновить защиту для виртуальной Машины:

1. На [мониторинга элемента хранилища](#view-vms-on-the-dashboard)выберите **возобновить резервное копирование**.

2. Выполните действия, описанные в [Управление политиками архивации](#manage-backup-policy-for-a-vm) Чтобы назначить политику для виртуальной Машины. Щелкните политику начальной защиты виртуальной Машины не нужно.
3. После применения политики резервного копирования к виртуальной Машине, может появиться следующее сообщение:

    ![Сообщение о том, успешно защищенной виртуальной Машины](./media/backup-azure-manage-vms/success-message.png)

## <a name="delete-backup-data"></a>удаление резервных копий;

Существует два способа для удаления архивируемых данных виртуальной Машины:

- На панели мониторинга элемента хранилища выберите остановить архивацию и следуйте инструкциям по [остановка защиты и удаление данных резервных копий](#stop-protection-and-delete-backup-data) параметр.

  ![Выбор "Остановить архивацию"](./media/backup-azure-manage-vms/stop-backup-buttom.png)

- На панели мониторинга элемента хранилища выберите Удалить данные архивации. Этот параметр включен, если был выбран для [остановите защиту и сохранить данные архивации](#stop-protection-and-retain-backup-data) во время остановки защиты виртуальной Машины

  ![Выберите удаление архива](./media/backup-azure-manage-vms/delete-backup-buttom.png)

  - На [панели мониторинга элемента хранилища](#view-vms-on-the-dashboard)выберите **удалить данные архивации**.
  - Введите имя архивируемого элемента, чтобы убедиться, что Вы действительно хотите удалить точки восстановления.

    ![удаление резервных копий;](./media/backup-azure-manage-vms/delete-backup-data1.png)

  - Чтобы удалить данные архивации для элемента, выберите **удалить**. Сообщение уведомления позволяет узнать, что резервные копии данных была удалена.

  > [!NOTE]
  > При удалении данных резервного копирования удаляются все связанными точками восстановления. Нельзя выбрать определенные точки для удаления.

# <a name="manage-azure-vm-backups-with-azure-backup-service"></a>Управление резервными отстаями ВМ Azure с помощью службы резервного копирования Azure

В этой статье описывается, как управлять виртуальными машинами Azure (VMs), которые поддерживаются с помощью [службы резервного копирования Azure.](backup-overview.md) В статье также кратко излагается информация о резервной копировании, которая можно найти на панели мониторинга хранилища.

На портале Azure панель мониторинга хранилища служб восстановления обеспечивает доступ к информации о хранилище, в том числе:

* последняя резервная копия, которая одновременно является последней точкой восстановления;
* Политика резервного копирования.
* Общий размер всех снимков резервного копирования.
* Количество вс-да, включенных для резервного копирования.

Вы можете управлять резервными копированиями, используя приборную панель и сверля до отдельных VMs. Чтобы начать резервное копирование машины, откройте хранилище на приборной панели.

![Полный вид приборной панели с ползунок](./media/backup-azure-manage-vms/bottom-slider.png)

## <a name="view-vms-on-the-dashboard"></a>Просмотр вышина на приборной панели

Для просмотра VMs на панели мониторинга хранилища:

1. Войдите на [портал Azure](https://portal.azure.com/).
2. В меню концентратора выберите **Обзор**. В списке ресурсов введите **Службы восстановления**. При вводе список фильтруется на основе ввода. Выберите **Хранилища служб восстановления**.

    ![Создание хранилища Служб восстановления](./media/backup-azure-manage-vms/browse-to-rs-vaults.png)

3. Для удобства использования нажмите правой кнопкой мыши хранилище и выберите **Pin на приборной панели.**
4. Откройте панель мониторинга хранилища.

    ![Откройте панель мониторинга хранилища и панель настроек](./media/backup-azure-manage-vms/full-view-rs-vault.png)

5. На плитке **элементов резервного копирования** выберите **виртуальные машины Azure.**

    ![Откройте плитку элементов резервного копирования](./media/backup-azure-manage-vms/contoso-vault-1606.png)

6. На панели **элементов резервного копирования** можно просмотреть список защищенных vMs. В этом примере хранилище защищает одну виртуальную машину: демобэк.  

    ![Просмотр панели элементов резервного копирования](./media/backup-azure-manage-vms/backup-items-blade-select-item.png)

7. Со приборной панели элемента хранилища измените политики резервного копирования, запустите резервное копирование по требованию, остановите или возобновите защиту VMs, удалите данные резервного копирования, просмотр точек восстановления и запустите восстановление.

    ![Панель мониторинга элементов резервного копирования и панель настроек](./media/backup-azure-manage-vms/item-dashboard-settings.png)

## <a name="manage-backup-policy-for-a-vm"></a>Управление политикой резервного копирования для VM

Для управления политикой резервного копирования:

1. Войдите на [портал Azure](https://portal.azure.com/). Откройте панель мониторинга хранилища.
2. На плитке **элементов резервного копирования** выберите **виртуальные машины Azure.**

    ![Откройте плитку элементов резервного копирования](./media/backup-azure-manage-vms/contoso-vault-1606.png)

3. На панели **элементов резервного копирования** можно просматривать список защищенных вс-баров и последний статус резервного копирования с последним временем восстановления.

    ![Просмотр панели элементов резервного копирования](./media/backup-azure-manage-vms/backup-items-blade-select-item.png)

4. Из панели мониторинга элемента хранилища можно выбрать политику резервного копирования.

   * Для переключения политики выберите другую политику, а затем выберите **Сохранить.** Новая политика будет немедленно применена к хранилищу.

     ![Выберите политику архивации](./media/backup-azure-manage-vms/backup-policy-create-new.png)

## <a name="run-an-on-demand-backup"></a>Выполнение резервного копирования по запросу

Вы можете запустить резервное копирование VM по требованию после настройки его защиты. Имейте в виду эти детали:

* Если начальная резервная копия находится на рассмотрении, резервное копирование по требованию создает полную копию VM в хранилище служб восстановления.
* Если начальная резервная запись завершена, резервное копирование по требованию будет отправлять изменения из предыдущего снимка только в хранилище служб восстановления. То есть, более поздние резервные копирования всегда являются постепенными.
* Диапазон удержания для резервного копирования по требованию — это значение удержания, указанное при запуске резервного копирования.

Чтобы запустить резервное копирование по запросу, сделайте следующее:

1. На [панели мониторинга элемента хранилища](#view-vms-on-the-dashboard)под **защищенным элементом**выберите **элемент резервного копирования.**

    ![Параметр резервного копирования теперь](./media/backup-azure-manage-vms/backup-now-button.png)

2. Из **типа управления резервным копированием**выберите **виртуальную машину Azure.** Появляется панель **резервного копирования (Azure Virtual Machine).**
3. Выберите VM и выберите **резервное копирование сейчас** для создания резервного копирования по требованию. Отображается панель **резервного копирования now.**
4. В поле **Retain Backup Till** укажите дату сохранения резервного копирования.

    ![Календарь резервного копирования сейчас](./media/backup-azure-manage-vms/backup-now-check.png)

5. Выберите **OK** для выполнения задания резервного копирования.

Чтобы отслеживать ход выполнения задания на панели мониторинга хранилища, выберите плитку **Backup Jobs.**

## <a name="stop-protecting-a-vm"></a>Отключение защиты для виртуальной машины

Существует два способа прекратить защиту VM:

* **Остановите защиту и сохраните данные резервного копирования.** Эта опция остановит все будущие задания резервного копирования от защиты вашего VM; Однако служба резервного копирования Azure сохранит резервные точки восстановления.  Вам нужно будет заплатить, чтобы сохранить точки восстановления в хранилище (см. [цены на резервное копирование Azure](https://azure.microsoft.com/pricing/details/backup/) для деталей). Вы сможете восстановить VM при необходимости. Если вы решите возобновить защиту VM, то вы можете использовать опцию *резервного копирования Resume.*
* **Остановить защиту и удалить данные резервного копирования.** Эта опция остановит все будущие задания резервного копирования от защиты вашего VM и удалит все точки восстановления. Вы не сможете восстановить VM или использовать опцию *резервного копирования Resume.*

>[!NOTE]
>если вы удалите источник данных без остановки резервного копирования, новые операции резервного копирования будут завершаться сбоем. Старые точки восстановления истекают в соответствии с политикой, но последняя точка восстановления всегда будет сохранена до тех пор, пока вы не остановите резервные копирования и не удалите данные.
>

### <a name="stop-protection-and-retain-backup-data"></a>Остановка защиты и сохранение данных резервного копирования

Чтобы остановить защиту и сохранить данные VM:

1. На [приборной панели элемента хранилища](#view-vms-on-the-dashboard)выберите **резервное копирование.**
2. Выберите **Сохранить данные резервного копирования**и подтвердить свой выбор по мере необходимости. Добавить комментарий, если вы хотите. Если вы не уверены в названии элемента, нависните над восклицательным знаком, чтобы просмотреть имя.

    ![Сохранение данных о резервном копировании](./media/backup-azure-manage-vms/retain-backup-data.png)

Уведомление позволяет узнать, что задания резервного копирования были остановлены.

### <a name="stop-protection-and-delete-backup-data"></a>Остановить защиту и удалить данные резервного копирования

Чтобы остановить защиту и удалить данные VM:

1. На [приборной панели элемента хранилища](#view-vms-on-the-dashboard)выберите **резервное копирование.**
2. Выберите **удалить данные резервного копирования**и подтвердите свой выбор по мере необходимости. Введите имя элемента резервного копирования и добавьте комментарий, если вы хотите.

    ![удаление резервных копий;](./media/backup-azure-manage-vms/delete-backup-data1.png)

## <a name="resume-protection-of-a-vm"></a>Возобновление защиты виртуальной машины

Если вы выбрали защиту Stop и сохранили опцию [резервного копирования данных](#stop-protection-and-retain-backup-data) во время защиты VM- системы stop, то можно использовать **резервное копирование Резюме.** Эта опция недоступна, если вы выберете опцию Stop protection и удалите опцию [резервного копирования данных](#stop-protection-and-delete-backup-data) или [удалить данные резервного копирования.](#delete-backup-data)

Чтобы возобновить защиту для VM:

1. На [панели мониторинга элемента хранилища](#view-vms-on-the-dashboard)выберите **резервное копирование резюме.**

2. Выполните последующие действия в [политике резервного копирования,](#manage-backup-policy-for-a-vm) чтобы назначить политику для VM. Вам не нужно выбирать первоначальную политику защиты VM.
3. После применения политики резервного копирования к VM вы увидите следующее сообщение:

    ![Сообщение с указанием успешно защищенного VM](./media/backup-azure-manage-vms/success-message.png)

## <a name="delete-backup-data"></a>удаление резервных копий;

Существует два способа удаления резервных данных VM:

* На панели мониторинга элемента хранилища выберите резервную копию stop и следуйте инструкциям по [защите стопа и удалите](#stop-protection-and-delete-backup-data) опцию данных резервного копирования.

  ![Выбор "Остановить архивацию"](./media/backup-azure-manage-vms/stop-backup-buttom.png)

* На панели мониторинга элемента хранилища выберите данные резервного копирования. Эта опция включена, если вы решили остановить защиту и сохранить опцию [резервного копирования данных](#stop-protection-and-retain-backup-data) во время защиты от Stop VM

  ![Выберите Резервное копирование для удалений](./media/backup-azure-manage-vms/delete-backup-buttom.png)

  * На [панели мониторинга элемента хранилища](#view-vms-on-the-dashboard)выберите **Удалить данные резервного копирования.**
  * Введите имя элемента резервного копирования, чтобы подтвердить, что вы хотите удалить точки восстановления.

    ![удаление резервных копий;](./media/backup-azure-manage-vms/delete-backup-data1.png)

  * Чтобы удалить данные резервного копирования для элемента, выберите **Удалить**. Сообщение об уведомлении позволяет узнать, что данные резервного копирования были удалены.

Для защиты данных резервное копирование Azure включает функцию мягкого удаления. При мягком удалении, даже после удаления резервного копирования (все точки восстановления) VM, данные резервного копирования сохраняются в течение 14 дополнительных дней. Для получения дополнительной информации [см.](https://docs.microsoft.com/azure/backup/backup-azure-security-feature-cloud)

  > [!NOTE]
  > При удалении данных резервного копирования вы удаляете все связанные точки восстановления. Вы не можете выбрать конкретные точки восстановления для удаления.

### <a name="backup-item-where-primary-data-source-no-longer-exists"></a>Элемент резервного копирования, в котором больше не существует основного источника данных

* Если vMs Azure, настроенные для резервного копирования Azure, либо удаляются, либо перемещаются без остановки защиты, то как запланированные задания резервного копирования, так и по требованию (ad-hoc) задания резервного копирования потерпят неудачу с ошибкой UserErrorVmNotFoundV2. Предпроверка резервного копирования будет отображаться как критическая только для неудачных заданий резервного копирования по требованию (не отображаются незапланированные задания).
* Эти элементы резервного копирования остаются активными в системе, придерживающейся политики резервного копирования и удержания, установленной пользователем. Резервные данные для этих VMs Azure будут сохранены в соответствии с политикой удержания. Срок действия моментов восстановления (кроме последней точки восстановления) очищается в соответствии с диапазоном удержания, установленным в политике резервного копирования.
* Пользователям рекомендуется удалять элементы резервного копирования, в которых основной источник данных больше не существует, чтобы избежать каких-либо дополнительных затрат, если элемент резервного копирования/данные для ресурсов удаления больше не требуется, поскольку последняя точка восстановления сохраняется навсегда и пользователь взимается в соответствии с применимым ценамрезервно.

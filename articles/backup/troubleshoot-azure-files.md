# <a name="troubleshoot-problems-backing-up-azure-file-shares"></a>Устранение неполадок с резервным копированием файловых ресурсов Azure
Для устранения неполадок и ошибок, обнаруженных в ходе резервного копирования файловых ресурсов Azure, можно использовать информацию в таблицах ниже.

## <a name="limitations-for-azure-file-share-backup-during-preview"></a>Ограничения резервного копирования файловых ресурсов Azure в предварительной версии
Функция резервного копирования файловых ресурсов Azure находится в стадии предварительной версии. Файловые ресурсы Azure поддерживаются в учетных записях хранения общего назначения версий 1 и 2. Следующие сценарии резервного копирования не поддерживаются для файловых ресурсов Azure:
- Нельзя обеспечить защиту файловых ресурсов Azure в учетных записях хранения с включенными виртуальными сетями или брандмауэром.
- Нельзя использовать CLI для применения защиты Файлов Azure с помощью Azure Backup.
- Максимальное число запланированных резервных копий в день — одна.
- Максимальное число резервных копий по запросу в день — четыре.
- Чтобы предотвратить случайное удаление резервных копий в хранилище служб восстановления, используйте [блокировку ресурсов](https://docs.microsoft.com/cli/azure/resource/lock?view=azure-cli-latest) в учетной записи хранения.
- Не удаляйте моментальные снимки, созданные службой Azure Backup. Удаление моментальных снимков может привести к потере точек восстановления и (или) сбоям при выполнении восстановления.
- Не удаляйте файловые ресурсы, защищенные службой Azure Backup. После удаления файлового ресурса текущее решение удалит все моментальные снимки, сделанные службой Azure Backup, поэтому все точки восстановления будут утеряны.

Резервное копирование файловых ресурсов Azure в учетные записи хранения с помощью репликации [хранилища, избыточного в пределах зоны](../storage/common/storage-redundancy-zrs.md) (ZRS), сейчас доступно только в центральной части США (CUS), восточной части США (EUS), восточной части США 2 (EUS2), Северной Европе (NE), Юго-Восточной Азии (SEA), Западной Европе (WE) и Западной Европе 2 (WUS2).

## <a name="configuring-backup"></a>Настройка резервного копирования
Следующая таблица содержит сведения о настройке резервного копирования.

| Сообщения об ошибках | Возможное решение или советы по решению проблем |
| ------------------ | ----------------------------- |
| Не удается найти учетную запись хранения для настройки резервного копирования файловых ресурсов Azure. | <ul><li>Подождите, пока процесс обнаружения завершится. <li>Проверьте, не защищен ли какой-нибудь файловый ресурс из учетной записи хранения другим хранилищем служб восстановления. **Примечание**. Все файловые ресурсы в учетной записи хранения могут быть защищены только одним хранилищем Служб восстановления. <li>Убедитесь, что файловый ресурс отсутствует в неподдерживаемых учетных записях хранения.|
| На портале возникает ошибка с сообщением о том, что не удалось обнаружить состояние учетных записей хранения. | Если у вас подписка партнера (с поддержкой CSP), игнорируйте ошибку. Если ваша подписка не поддерживает CSP и ваши учетные записи хранения не обнаруживаются, обратитесь в службу поддержки.|
| Сбой проверки или регистрации выбранной учетной записи хранения.| Повторите операцию. Если проблема повторяется, обратитесь в службу поддержки.|
| Не удалось перечислить или найти файловые ресурсы в выбранной учетной записи хранения. | <ul><li> Убедитесь, что учетная запись хранения находится в группе ресурсов (что она не была удалена или перемещена в хранилище после последней проверки или регистрации).<li>Убедитесь, что файловый ресурс, который необходимо защитить, не удален. <li>Убедитесь, что учетная запись хранения поддерживает резервное копирование файловых ресурсов.<li>Проверьте, не защищен ли файловый ресурс в том же хранилище служб восстановления.|
| Сбой настройки резервного копирования файлового ресурса (или настройки политики защиты). | <ul><li>Повторите операцию, чтобы проверить, повторяется ли проблема. <li> Убедитесь, что файловый ресурс, который вы хотите защитить, не удален. <li> Если вы пытаетесь защитить сразу несколько файловых ресурсов и произошел сбой защиты некоторых ресурсов, повторно настройте для них резервное копирование. |
| Не удалось удалить хранилище служб восстановления после снятия защиты с файлового ресурса. | На портале Azure последовательно откройте Хранилище > **Инфраструктура резервного копирования** > **Учетные записи хранения** и щелкните **Отменить регистрацию**, чтобы удалить учетную запись хранения из хранилища служб восстановления.|


## <a name="error-messages-for-backup-or-restore-job-failures"></a>Сообщения об ошибках заданий резервного копирования или восстановления

| Сообщения об ошибках | Возможное решение или советы по решению проблем |
| -------------- | ----------------------------- |
| Не удалось выполнить операцию, так как общая папка не найдена. | Убедитесь, что файловый ресурс, который необходимо защитить, не удален.|
| Учетная запись хранения не найдена или не поддерживается. | <ul><li>Убедитесь, что учетная запись хранения находится в группе ресурсов и не была удалена или перемещена из группы ресурсов после последней проверки. <li> Убедитесь, что учетная запись хранения поддерживает резервное копирование файловых ресурсов.|
| Достигнуто предельное число моментальных снимков для этой общей папки. Вы сможете сделать новые снимки, когда истечет срок действия старых. | <ul><li> Эта ошибка может возникнуть при создании нескольких резервных копий для файла по запросу. <li> Существует ограничение на 200 моментальных снимков для одного файлового ресурса, включая те, которые были сделаны службой Azure Backup. Старые запланированные резервные копии (или моментальные снимки) очищаются автоматически. Если достигнут максимальный предел, необходимо удалить резервные копии по запросу (или моментальные снимки).<li> Удалите резервные копии по запросу (моментальные снимки файловых ресурсов Azure) с портала решения "Файлы Azure". **Примечание**. Если удалить моментальные снимки, созданные службой Azure Backup, точки восстановления будут утеряны. |
| Не удалось выполнить резервное копирование или восстановление файловых ресурсов по причине регулирования службы хранилища. Это может быть связано с тем, что служба хранения занята обработкой других запросов для данной учетной записи хранения.| Повторите операцию через некоторое время. |
| Сбой восстановления с сообщением: "Конечный файловый ресурс не найден". | <ul><li>Убедитесь, что выбранная учетная запись хранения существует, а конечный файловый ресурс не удален. <li> Убедитесь, что учетная запись хранения поддерживает резервное копирование файловых ресурсов. |
| Сейчас резервное копирование не поддерживается для файловых ресурсов Azure в учетных записях хранения, имеющих включенные виртуальные сети. | Отключите виртуальные сети в учетной записи хранения, чтобы обеспечить успешное выполнение операций резервного копирования или восстановления файлов. |
| Произошел сбой операции резервного копирования или восстановления, так как учетная запись хранения заблокирована. | Разблокируйте учетную запись хранения или вместо блокировки на чтение используйте блокировку на удаление и повторите операцию. |
| Операция восстановления завершилась сбоем, так как количество файлов с ошибками превышает пороговое значение. | <ul><li> Причины сбоя операции восстановления перечислены в файле (путь указан в сведениях о задании). Перечитайте информацию об ошибках и повторите операцию восстановления только для файлов с ошибками. <li> Можно выделить следующие распространенные причины сбоев операции восстановления файлов: <br/> — файлы с ошибками в настоящее время используются; <br/> — в родительском каталоге находится каталог с тем же именем, что и файл с ошибкой. |
| Операция восстановления завершилась сбоем, так как ни один файл невозможно восстановить. | <ul><li> Причины сбоя операции восстановления перечислены в файле (путь указан в сведениях о задании). Перечитайте информацию об ошибках и повторите операцию восстановления только для файлов с ошибками. <li> Можно выделить следующие распространенные причины сбоев операции восстановления файлов: <br/> — файлы, которые не удалось восстановить, в настоящее время используются; <br/> — в родительском каталоге находится каталог с тем же именем, что и файл с ошибкой. |
| Произошел сбой операции восстановления по причине отсутствия файлов в источнике. | <ul><li> Выбранные элементы отсутствуют в данных точки восстановления. Чтобы восстановить файлы, укажите правильный список файлов. <li> Моментальный снимок файлового ресурса, соответствующий точке восстановления, удален вручную. Выберите другую точку восстановления и повторите операцию. |
| В этом же месте назначения уже выполняется задание восстановления. | <ul><li>Резервное копирование файлового ресурса не поддерживает параллельное восстановление для одного и того же конечного ресурса. <li>Подождите, пока процесс восстановления завершится, и повторите попытку. Если вы не нашли задание восстановления в хранилище служб восстановления, проверьте другие хранилища в той же подписке. |
| Операция восстановления завершилась сбоем, так как конечный файловый ресурс заполнен. | Увеличьте квоту размера конечного файлового ресурса для размещения данных восстановления и повторите операцию. |
| Операция восстановления завершилась сбоем, так как при подготовке к восстановлению в ресурсах службы "Синхронизация файлов", связанных с целевым файловым ресурсом, произошла ошибка. | Повторите попытку через некоторое время. Если проблема возникнет снова, обратитесь в службу поддержки Майкрософт. |
| Восстановить некоторые файлы не удалось. Дополнительные сведения см. в списке файлов по указанному выше пути. | <ul> <li> Причины сбоя восстановления приводятся в файле (путь указан в описании задания). Перечитайте сведения о причинах и повторите операцию восстановления только для файлов с ошибками. <li> Можно выделить следующие распространенные причины сбоев операции восстановления файлов: <br/> — сейчас используются файлы с ошибками; <br/> — в родительском каталоге находится каталог с тем же именем, что и файл с ошибкой. |

# <a name="about-the-microsoft-azure-recovery-services-mars-agent"></a>О агенте служб восстановления Microsoft Azure (MARS)

В этой статье описывается, как служба резервного копирования Azure использует агент служб ы восстановления Microsoft Azure (MARS) для резервного копирования и восстановления файлов, папок и состояния громкости или системы с предприимчивого компьютера в Azure.

Агент MARS поддерживает следующие сценарии резервного копирования:

![Сценарии резервного копирования MARS](./media/backup-try-azure-backup-in-10-mins/backup-scenarios.png)

- **Файлы и складки**: выборочно защищают файлы и папки Windows.
- **Уровень громкости**: Защитите весь объем Windows вашей машины.
- **Уровень системы**: Защитите все состояние системы Windows.

Агент MARS поддерживает следующие сценарии восстановления:

![Сценарии восстановления MARS](./media/backup-try-azure-backup-in-10-mins/restore-scenarios.png)

- **Тот же сервер**: Сервер, на котором первоначально была создана резервная часть.
  - **Файлы и складки**: Выберите отдельные файлы и папки, которые вы хотите восстановить.
  - **Уровень громкости**: Выберите точку рекуперации громкости и восстановления, которую вы хотите восстановить, а затем восстановить его в том же месте или в другом месте на той же машине.  Создайте копию существующих файлов, перезапишите существующие файлы или пропустите восстановление существующих файлов.
  - **Уровень системы**: Выберите состояние системы и точку восстановления для восстановления той же машины в указанном месте.

- **Альтернативный сервер**: Сервер, кроме сервера, на котором была сделана резервная часть.
  - **Файлы и складки**: Выберите отдельные файлы и папки, пункт восстановления которых вы хотите восстановить в целевой машине.
  - **Уровень объема**: Выберите точку громкости и восстановления, которую вы хотите восстановить в другом месте. Создайте копию существующих файлов, перезапишите существующие файлы или пропустите восстановление существующих файлов.
  - **Уровень системы**: Выберите состояние системы и точку восстановления для восстановления в качестве файла системного состояния в альтернативную машину.

## <a name="backup-process"></a>Процесс резервного копирования

1. На портале Azure создайте [хранилище служб восстановления](install-mars-agent.md#create-a-recovery-services-vault)и выберите файлы, папки и состояние системы из целей резервного копирования.
2. [Загрузите учетные данные хранилища и установщика агентов Recovery Services](https://docs.microsoft.com/azure/backup/install-mars-agent#download-the-mars-agent) на автомат.

    Чтобы защитить локта-компьютер, выбрав опцию резервного копирования, выберите файлы, папки и состояние системы, а затем загрузите агент MARS.

3. Подготовьте инфраструктуру:

    а. Выполнить установщик для [установки агента.](https://docs.microsoft.com/azure/backup/install-mars-agent#install-and-register-the-agent)

    b. Используйте загруженные учетные данные хранилища для регистрации машины в хранилище служб восстановления.
4. С консоли агента на клиенте [наверсните резервную ставку.](https://docs.microsoft.com/azure/backup/backup-windows-with-mars-agent#create-a-backup-policy) Укажите политику хранения данных резервного копирования, чтобы начать защищать их.

![Диаграмма резервного агента Azure](./media/backup-try-azure-backup-in-10-mins/backup-process.png)

### <a name="additional-scenarios"></a>Дополнительные сценарии

- **Резервное копирование определенных файлов и папок в виртуальных машинах Azure:** Основным методом резервного копирования виртуальных машин Azure (VM) является использование расширения резервного копирования Azure на VM. Расширение резервное копирование всего VM. Если требуется резервное копирование определенных файлов и папок в VM, можно установить агент MARS в VMs Azure. Для получения дополнительной информации [см. Архитектура: Встроенный резервное копирование Azure VM.](https://docs.microsoft.com/azure/backup/backup-architecture#architecture-built-in-azure-vm-backup)

- **Посев в автономном режиме**: Первоначальные полные резервные копирования данных в Azure обычно передают большие объемы данных и требуют большей пропускной способности сети. Последующие резервные копирования передают только дельта, или дополнительный, объем данных. Следовательно, для первой архивации требуется более высокая пропускная способность по сравнению с последующими операциями архивации, в ходе которых передаются только изменения или добавочные резервные копии. В процессе *автономного посева*Azure Backup может использовать диски для загрузки сжатых исходных данных резервного копирования в автономном режиме в Azure. Для получения дополнительной информации смотрите в [автономном режиме резервного копирования Azure с помощью Azure Data Box.](offline-backup-azure-data-box.md)

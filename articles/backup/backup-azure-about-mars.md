# <a name="about-the-microsoft-azure-recovery-services-mars-agent"></a>Сведения об агенте Службы восстановления Microsoft Azure (MARS)

В этой статье описывается, как служба Azure Backup использует агент Службы восстановления Microsoft Azure (MARS) для резервного копирования и восстановления файлов и папок, тома или состояния системы с локального компьютера в Azure.

Агент MARS поддерживает следующие сценарии резервного копирования:

![Панель мониторинга хранилища служб восстановления](./media/backup-try-azure-backup-in-10-mins/backup-scenarios.png)

- **Файлы и папки**. Выборочная защита файлов и папок Windows.
- **Уровень громкости**: защита всего тома Windows компьютера.
- **Уровень системы**: защита всего состояния системы Windows.

Агент MARS поддерживает следующие сценарии восстановления:

![Панель мониторинга хранилища служб восстановления](./media/backup-try-azure-backup-in-10-mins/restore-scenarios.png)

-   **Тот же сервер**: тот же сервер, на котором изначально была создана резервная копия.
    -    **Файлы и папки**. Вы можете просматривать и выбирать отдельные файлы и папки для восстановления.
    -    **Уровень громкости**. Вы можете выбрать том и точку восстановления, которые необходимо восстановить, и восстановить ее в том же расположении или в другом расположении на том же компьютере.  Можно либо создать копию существующих файлов, перезаписать существующие файлы, либо пропустить восстановление существующих файлов.
    -    **Уровень системы**. Вы можете выбрать состояние системы и точку восстановления для восстановления на том же компьютере в указанном расположении.


-   **Альтернативный сервер**: другой сервер, т. е. не тот же сервер, на котором была создана резервная копия.
    -    **Файлы и папки**. Вы можете просматривать и выбирать отдельные файлы и папки, на которые вы хотите восстановить точку восстановления на целевой компьютер.
    -    **Уровень громкости**. можно выбрать том и точку восстановления, которые требуется восстановить в альтернативное расположение, либо создать копию существующих файлов, перезаписать существующие файлы или пропустить восстановление существующих файлов.
    -    **Уровень системы**. Вы можете выбрать состояние системы и точку восстановления для восстановления в качестве файла состояния системы на альтернативный компьютер.

## <a name="backup-process"></a>Процесс резервного копирования

1.  В портал Azure создайте [хранилище службы восстановления](https://docs.microsoft.com/azure/backup/backup-configure-vault#create-a-recovery-services-vault) и выберите файлы и папки, а также состояние системы из целей резервного копирования.
2.  [Скачайте](https://docs.microsoft.com/azure/backup/backup-configure-vault#download-the-mars-agent) учетные данные хранилища службы восстановления и установщик агента на локальный компьютер. Чтобы защитить локальный компьютер, выбрав вариант резервного копирования, выберите файлы и папки и состояние системы и скачайте агент MARS.
3.  Подготовка инфраструктуры:

    а)    Запустите установщик, чтобы [установить](https://docs.microsoft.com/azure/backup/backup-configure-vault#install-and-register-the-agent) агент.

    б)  Используйте Скачанные учетные данные хранилища для регистрации компьютера в хранилище служб восстановления.
4.  В консоли агента на клиенте используйте [Расписание архивации](https://docs.microsoft.com/azure/backup/backup-configure-vault#create-a-backup-policy) , чтобы настроить резервное копирование. Укажите политику хранения данных резервных копий и приступите к защите.

![Панель мониторинга хранилища служб восстановления](./media/backup-try-azure-backup-in-10-mins/backup-process.png)


### <a name="additional-scenarios"></a>Дополнительные сценарии
-   **Резервное копирование конкретных файлов и папок в виртуальной машине Azure** . основной метод резервного копирования виртуальных машин Azure — это использование расширения Azure Backup на виртуальной машине. В этом случае резервное копирование будет выполняться для всей виртуальной машины. Если вы хотите создать резервную копию конкретных файлов и папок в виртуальной машине, можно установить агент MARS на виртуальных машинах Azure. [Узнайте больше](https://docs.microsoft.com/azure/backup/backup-architecture#architecture-built-in-azure-vm-backup).

-   **Автономное заполнение** — начальные полные резервные копии данных в Azure. обычно они передают большие объемы данных и требуют дополнительной пропускной способности сети при сравнении с последующими резервными копиями, которые переносят только Дельта Следовательно, для первой архивации требуется более высокая пропускная способность по сравнению с последующими операциями архивации, в ходе которых передаются только изменения или добавочные резервные копии. Служба архивации Azure сжимает начальные резервные копии, а также предоставляет механизм передачи сжатых копий данных в Azure с помощью дисков за счет процесса автономного заполнения. [Узнайте больше](https://docs.microsoft.com/azure/backup/backup-azure-backup-server-import-export-).

# <a name="back-up-and-restore-encrypted-azure-vm"></a>Резервное копирование и восстановление зашифрованного Azure VM

В этой статье описывается, как создать резервную связь и восстановить виртуальные машины Windows или Linux Azure (VMs) с зашифрованными дисками с помощью службы [резервного копирования Azure.](backup-overview.md)

Если вы хотите узнать больше о том, как резервное копирование Azure взаимодействует с VMs-ми ресурсами Azure перед началом, просмотрите следующие ресурсы:

- [Просмотрите](backup-architecture.md#architecture-built-in-azure-vm-backup) архитектуру резервного копирования Azure VM.
- [Узнать о](backup-azure-vms-introduction.md) Резервное копирование Azure VM и расширение резервного копирования Azure.

## <a name="encryption-support"></a>Поддержка шифрования

Резервное копирование Azure поддерживает резервное копирование VMs Azure, которые имеют свои диски ОС/данных, зашифрованные с помощью шифрования дисков Azure (ADE). ADE использует BitLocker для шифрования Windows VMs, а также функцию dm-crypt для Linux VMs. ADE интегрируется с Azure Key Vault для управления ключами и секретами шифрования дисков. Ключи шифрования Key Vault (KEKs) можно использовать для добавления дополнительного уровня безопасности, шифрования секретов шифрования перед их написанием в Key Vault.

Резервное копирование Azure может резервное копирование и восстановление VMs Azure с помощью ADE с приложением Azure AD и без нее, как обобщено в следующей таблице.

**Тип диска VM** | **ADE (BEK/dm-crypt)** | **ADE и KEK**
--- | --- | ---
**Неуправляемые** | Да | Да
**Управляемых**  | Да | Да

- Узнайте больше о [ADE,](../security/azure-security-disk-encryption-overview.md) [Key Vault](../key-vault/key-vault-overview.md)и [KEK.](https://docs.microsoft.com/azure/virtual-machine-scale-sets/disk-encryption-key-vault#set-up-a-key-encryption-key-kek)
- Прочитайте [часто задаваемые вопросы](../security/azure-security-disk-encryption-faq.md) для шифрования vM-диска Azure.

### <a name="limitations"></a>Ограничения

- Вы можете создать резервную резервную и восстановленную зашифрованную ВМ в пределах одной подписки и региона.
- Резервное копирование Azure поддерживает VMs, зашифрованные с помощью автономных ключей. Любой ключ, который является частью сертификата, используемого для шифрования VM, в настоящее время не поддерживается.
- Вы можете создать резервную резервную и восстановленную зашифрованную вво-банку в пределах той же подписки и региона, что и хранилище резервного копирования служб восстановления.
- Зашифрованные виртуальные машины нельзя восстановить на уровне файла или папки. Для восстановления файлов и папок необходимо восстановить весь VM.
- При восстановлении VM нельзя использовать существующую опцию [VM-замены](backup-azure-arm-restore-vms.md#restore-options) для зашифрованных VM. Эта опция поддерживается только для незашифрованных управляемых дисков.

## <a name="before-you-start"></a>Перед началом работы

Прежде чем начать, сделайте следующее:

1. Убедитесь, что у вас есть один или несколько [Windows](../security/azure-security-disk-encryption-windows.md) или [Linux](../virtual-machines/linux/disk-encryption-overview.md) VMs с ADE включен.
2. [Просмотрите матрицу поддержки](backup-support-matrix-iaas.md) для резервного копирования Azure VM
3. [Создайте](backup-azure-arm-vms-prepare.md#create-a-vault) хранилище резервного копирования служб восстановления, если его нет.
4. Если вы включили шифрование для вс-кодов, которые уже включены для резервного копирования, вам просто необходимо предоставить резервное копирование с разрешениями на доступ к Key Vault, чтобы резервные отработки могли продолжаться без сбоев. [Подробнее](#provide-permissions) о назначении этих разрешений.

Кроме того, есть несколько вещей, которые вы, возможно, потребуется сделать в некоторых обстоятельствах:

- **Установка агента VM на VM:** Резервное копирование Azure резервное копирование Azure VMs, установив расширение для агента Azure VM, работая на машине. Если vM был создан на базе изображения рынка Azure, агент устанавливается и работает. Если вы создаете пользовательский VM или мигрируете в помещение, возможно, потребуется [установить агент вручную.](backup-azure-arm-vms-prepare.md#install-the-vm-agent)

## <a name="configure-a-backup-policy"></a>Настройка политики резервного копирования

1. Если вы еще не создали хранилище резервного копирования Служб восстановления, следуйте [этим инструкциям](backup-azure-arm-vms-prepare.md#create-a-vault)
2. Откройте хранилище на портале и выберите **резервное копирование** в разделе **«Начало работы».**

    ![Колонка резервного копирования](./media/backup-azure-vms-encryption/select-backup.png)

3. В **Azure**цели >  **резервного копирования****Где работает рабочая нагрузка?**
4. В **Что вы хотите создать резервную систему?** выберите **Виртуальную машину** > **OK**.

      ![Колонка сценариев](./media/backup-azure-vms-encryption/select-backup-goal-one.png)

5. В >  **политике резервного копирования****Выберите политику резервного копирования,** выберите политику, которую вы хотите связать с хранилищем. После этого щелкните **OK**.
    - Политика резервного копирования определяет, когда используются резервные отсутсвия и как долго они хранятся.
    - Подробные сведения о политике по умолчанию указаны в раскрывающемся меню.

    ![Открытие колонки сценария](./media/backup-azure-vms-encryption/select-backup-goal-two.png)

6. Если вы не хотите использовать политику по умолчанию, выберите **Создать новую**и [создать пользовательскую политику.](backup-azure-arm-vms-prepare.md#create-a-custom-policy)

7. Выберите зашифрованные ВМ, которые вы хотите создать резервную резервную часть с помощью политики выбора, и выберите **OK.**

      ![Выбор зашифрованных виртуальных машин](./media/backup-azure-vms-encryption/selected-encrypted-vms.png)

8. Если вы используете Azure Key Vault на странице хранилища, вы увидите сообщение о том, что резервному копированию Azure необходим доступ только к ключам и секретам в Хранилище ключей.

    - Если вы получили это сообщение, никаких действий не требуется.

        ![OK доступа](./media/backup-azure-vms-encryption/access-ok.png)

    - Если вы получили это сообщение, вам необходимо установить разрешения, как описано в [процедуре ниже.](#provide-permissions)

        ![Предупреждение о доступе](./media/backup-azure-vms-encryption/access-warning.png)

9. Нажмите **кнопку «Включить резервное копирование»,** чтобы развернуть политику резервного копирования в хранилище и включить резервное копирование для выбранных VMs.

## <a name="trigger-a-backup-job"></a>активация задания архивации;

Начальная резервная часть будет работать в соответствии с расписанием, но вы можете запустить его сразу же следующим образом:

1. В меню хранилища щелкните **Элементы архивации**.
2. В **элементах резервного копирования**нажмите **виртуальная машина Azure.**
3. В списке **элементов резервного копирования** щелкните эллипсы (...).
4. Щелкните **Создать резервную копию**.
5. В **Backup Now**используйте элемент управления календаря, чтобы выбрать последний день сохранения точки восстановления. После этого щелкните **OK**.
6. Отслеживайте уведомления на портале. Вы можете отслеживать ход выполнения заданий на панели мониторинга хранилища >**в процессе выполнения**заданий **резервного** > копирования. В зависимости от размера виртуальной машины создание начального архива может занять некоторое время.

## <a name="provide-permissions"></a>Предоставление разрешений

Azure VM нуждается в доступном доступе только для чтения для резервного копирования ключей и секретов, а также связанных с ними VM.

- Ключевое хранилище связано с арендатором Azure AD подписки Azure. Если вы **являетесь пользователем-участником,** Резервное копирование Azure получает доступ к Key Vault без дальнейших действий.
- Если вы **являетесь пользователем Guest,** необходимо предоставить разрешения для резервного копирования Azure для доступа к хранилищу ключей.

Для установки разрешений:

1. На портале Azure выберите **все службы**и ищите **хранилища ключей.**
2. Выберите хранилище ключей, связанное с зашифрованным VM, который вы резервируете.
3. Выберите **политики** > доступа**Добавление новых**.
4. Выберите **основной**принцип, а затем введите **управление резервным копированием.**
5. Выберите >  **службу управления резервным копированием****Выберите**.

    ![Выбор службы резервного копирования](./media/backup-azure-vms-encryption/select-backup-service.png)

6. В политике **добавления доступа** > **Настройте шаблон (необязательно)** выберите **резервное копирование Azure.**
    - Необходимые разрешения автоматически заполняются в раскрывающихся списках **Разрешения ключей** и **Разрешения секретов**.
    - Если ваш VM зашифрован только с помощью **BEK,** удалите выбор для **ключевых разрешений,** так как вам нужны только разрешения на секреты.

    ![Выбор службы Azure Backup](./media/backup-azure-vms-encryption/select-backup-template.png)

7. Нажмите кнопку **ОК**. **Служба управления резервным копированием** добавляется к **политикам доступа.**

    ![Политики доступа](./media/backup-azure-vms-encryption/backup-service-access-policy.png)

8. Нажмите **Сохранить,** чтобы предоставить резервное копирование Azure с разрешениями.

## <a name="restore-an-encrypted-vm"></a>Восстановление зашифрованной виртуальной машины

Вы восстанавливаете зашифрованные VMs следующим образом:

1. [Восстановление VM-диска](backup-azure-arm-restore-vms.md#restore-disks).
2. Воссоздайте экземпляр виртуальной машины, сделав один из следующих:
    1. Используйте шаблон, созданный во время операции восстановления, чтобы настроить настройки VM и вызвать развертывание VM. Ознакомьтесь с [дополнительными сведениями](backup-azure-arm-restore-vms.md#use-templates-to-customize-a-restored-vm).
    2. Создайте новый VM из восстановленных дисков с помощью PowerShell. Ознакомьтесь с [дополнительными сведениями](backup-azure-vms-automation.md#create-a-vm-from-restored-disks).
3. Для Linux VMs переустановите расширение ADE, чтобы диски данных были открыты и установлены.

# <a name="overview-of-offline-backup"></a>Обзор резервного копирования в автономном режиме

В этой статье приводится обзор резервного копирования в автономном режиме.

Первоначальные полные резервные копирования в Azure обычно передают большие объемы данных в Интернете и требуют большей пропускной способности сети по сравнению с последующими резервными копированиями, которые передают только постепенные изменения. Удаленные офисы или центры обработки данных в определенных географических регионах не всегда имеют достаточную пропускную способность сети. По этой причине эти первоначальные резервные копирования занимают несколько дней. В течение этого времени резервные копирования постоянно используют ту же сеть, которая была подготовлена для приложений, работающих в центре обработки данных.

Резервное копирование Azure поддерживает автономное резервное копирование, которое передает исходные данные резервного копирования в автономном режиме без использования пропускной способности сети. Он обеспечивает механизм копирования данных резервного копирования на физические устройства хранения. Затем устройства отправляются в ближайший центр обработки данных Azure и загружаются в хранилище служб восстановления. Этот процесс обеспечивает надежную передачу данных резервного копирования без использования какой-либо пропускной способности сети.

## <a name="offline-backup-options"></a>Параметры резервного копирования в автономном режиме

Резервное копирование в автономном режиме предлагается в двух режимах в зависимости от права собственности на устройства хранения:

- Резервное копирование в автономном режиме на основе ящика данных Azure (предварительный просмотр)
- Резервное копирование в автономном режиме на основе службы импорта/экспорта Azure

## <a name="offline-backup-based-on-azure-data-box-preview"></a>Резервное копирование в автономном режиме на основе ящика данных Azure (предварительный просмотр)

В настоящее время этот режим поддерживается агентом Microsoft Azure Recovery Services (MARS) в предварительном просмотре. Эта опция использует [Azure Data Box](https://azure.microsoft.com/services/databox/) для отправки защищенных, защищенных и устойчивых к несанкционированным устройствам передачи устройств с USB-разъемами в ваш центр обработки данных или удаленный офис. Данные резервного копирования непосредственно записываются на эти устройства. Эта опция экономит усилия, необходимые для приобретения собственных дисков и разъемов, совместимых с Azure, или для обеспечения временного хранения в качестве промежуточного места. Корпорация Майкрософт также занимается сквозной логистикой передачи, которую можно отслеживать через портал Azure. 

Архитектура, описывающая движение данных резервного копирования с помощью этой опции, показана здесь.

![Архитектура коробки данных резервного копирования Azure](./media/offline-backup-overview/azure-backup-databox-architecture.png)

Вот краткое изложение архитектуры:

1. Резервное копирование Azure непосредственно копирует данные резервного копирования на эти предварительно настроенные устройства.
2. Затем эти устройства можно отправить обратно в центр обработки данных Azure.
3. Azure Data Box копирует данные на принадлежащем клиенту учетную запись хранения.
4. Резервное копирование Azure автоматически копирует данные резервного копирования из учетной записи хранилища в назначенное хранилище служб восстановления. Запланированы дополнительные резервные копирования в Режиме онлайн.

Для использования резервного копирования в автономном режиме на основе Azure Data Box см. Резервное копирование в [автономном режиме с помощью Azure Data Box.](offline-backup-azure-data-box.md)

## <a name="offline-backup-based-on-the-azure-importexport-service"></a>Резервное копирование в автономном режиме на основе службы импорта/экспорта Azure

Эта опция поддерживается сервером резервного копирования Microsoft Azure (MABS), диспетчером системной защиты данных (DPM) DPM-A и агентом MARS. Он использует [сервис Azure Import/Export.](https://docs.microsoft.com/azure/storage/common/storage-import-export-service) Исходные данные резервного копирования можно перенести в Azure, используя собственные диски и разъемы, совместимые с Azure. Этот подход требует, чтобы вы предоставили временное хранилище, известное как место презавочного, и использовали заранее построенные утилиты для форматирования и копирования резервных данных на принадлежащих клиенту дисках. 

Архитектура, описывающая движение данных резервного копирования с помощью этой опции, показана здесь.

![Архитектура службы резервного копирования Azure](./media/offline-backup-overview/azure-backup-import-export.png)

Вот краткое изложение архитектуры:

1. Вместо отправки данных резервного копирования по сети резервное копирование Azure записывает данные резервного копирования в место постановки.
2. Данные в месте постановки записываются на один или несколько дисков SATA с помощью пользовательской утилиты.
3. В рамках подготовительной работы утилита создает работу по импорту Azure. Диски SATA отправляются в ближайший центр обработки данных Azure и ссылаются на работу импорта для подключения действий.
4. В центре обработки данных Azure данные с дисков копируются в учетную запись хранилища Azure.
5. Резервное копирование Azure копирует данные резервного копирования из учетной записи хранилища в хранилище служб восстановления. Запланированы дополнительные резервные копирования.

Для использования резервного копирования в автономном режиме на основе [Offline backup workflow in Azure Backup](https://docs.microsoft.com/azure/backup/backup-azure-backup-import-export)службы импорта/экспорта Azure с агентом MARS см.

Чтобы использовать то же самое вместе с MABS или DPM-A, [см.](https://docs.microsoft.com/azure/backup/backup-azure-backup-server-import-export-)

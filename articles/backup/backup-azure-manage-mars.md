# <a name="manage-microsoft-azure-recovery-services-mars-agent-backups-by-using-the-azure-backup-service"></a>Управление резервными отработками служб восстановления Microsoft Azure (MARS) с помощью службы резервного копирования Azure

В этой статье описывается, как управлять файлами и папками, которые поддерживаются агентом служб восстановления Microsoft Azure.

## <a name="modify-a-backup-policy"></a>Изменение политики архивации

При изменении политики резервного копирования можно добавлять новые элементы, удалять существующие элементы из резервного копирования или исключать файлы из резервного копирования с помощью настроек исключения.

- **Добавить Элементы** использовать эту опцию только для добавления новых элементов для резервного копирования. Чтобы удалить существующие элементы, используйте **параметры «Удалить элементы»** или **«Настройки исключения».**  
- **Удалить элементы** использовать эту опцию для удаления элементов из резервного копирования.
  - Используйте **параметры исключения** для удаления всех элементов в объеме вместо **удаления элементов.**
  - Очистка всех выделений в томе приводит к тому, что старые резервные копирования элементов сохраняются в настройках удержания во время последнего резервного копирования без возможности для модификации.
  - Перевыбор этих элементов приводит к первому полному резервному копированию, и новые изменения политики не применяются к старым резервным копированиям.
  - Невыбрать весь объем сохраняет за собой резервную кавер-резервную часть без каких-либо возможностей для изменения политики удержания.
- **Параметры исключения** используют эту опцию, чтобы исключить резервную копию определенных элементов.

### <a name="add-new-items-to-existing-policy"></a>Добавление новых элементов в существующую политику

1. В **действиях**, нажмите **Резервное копирование расписания**.

    ![Планирование архивации Windows Server](./media/backup-configure-vault/schedule-first-backup.png)

2. Во вкладке **«Выберите элемент политики»** и выберите **изменить расписание резервного копирования для файлов и папок** и нажмите **далее.**

    ![Выберите элементы политики](./media/backup-azure-manage-mars/select-policy-items.png)

3. В **модифицировать или остановить резервное копирование** вкладке, выберите Внести изменения в **элементы резервного копирования или раз** и нажмите **Далее.**

    ![Изменение или график резервного копирования](./media/backup-azure-manage-mars/modify-schedule-backup.png)

4. В **«Выберите элементы для вкладки резервного копирования»** нажмите **«Добавить элементы»,** чтобы добавить элементы, которые вы хотите создать резервную.

    ![Изменение или график резервного копирования добавлять элементы](./media/backup-azure-manage-mars/modify-schedule-backup-add-items.png)

5. В окне **Select Items** выберите мухили или папки, которые вы хотите добавить, и нажмите **OK.**

    ![Выберите элементы](./media/backup-azure-manage-mars/select-item.png)

6. Выполните последующие шаги и нажмите **Finish,** чтобы завершить операцию.

### <a name="add-exclusion-rules-to-existing-policy"></a>Добавление правил исключения в существующую политику

Можно добавить правила исключения, чтобы пропустить файлы и папки, которые вы не хотите, чтобы их резервная копила. Это можно сделать во время определения новой политики или изменения существующей политики.

1. Из панели действий нажмите **Кнопка Резервного копирования.** Перейдите на выбор элементов в настройках **резервного копирования** и нажмите **кнопку «Настройки исключения».**

    ![Выберите элементы](./media/backup-azure-manage-mars/select-exclusion-settings.png)

2. В **настройках исключения**нажмите **Добавить исключение**.

    ![Выберите элементы](./media/backup-azure-manage-mars/add-exclusion.png)

3. От **выбора элементов до исключения**, просмотрите файлы и папки и выберите элементы, которые вы хотите исключить, и нажмите **OK.**

    ![Выберите элементы](./media/backup-azure-manage-mars/select-items-exclude.png)

4. По умолчанию все **Subfolders** в выбранных папках исключены. Вы можете изменить это, выбрав **Да** или **Нет**. Можно отсеить и указать типы файлов, чтобы исключить, как показано ниже:

    ![Выберите элементы](./media/backup-azure-manage-mars/subfolders-type.png)

5. Выполните последующие шаги и нажмите **Finish,** чтобы завершить операцию.

### <a name="remove-items-from-existing-policy"></a>Удаление элементов из существующей политики

1. Из панели действий нажмите **Кнопка Резервного копирования.** Перейдите к **выбору элементов в резервное копирование.** Из списка выберите файлы и папки, которые вы хотите удалить из резервного расписания, и нажмите **Удалить элементы.**

    ![Выберите элементы](./media/backup-azure-manage-mars/select-items-remove.png)

> [!NOTE]
> Будьте осторожны, когда вы полностью удалите том из политики.  Если вам нужно добавить его снова, то он будет рассматриваться как новый объем. Следующее запланированное резервное копирование будет выполнять начальную резервную проверку (полное резервное копирование) вместо инкрементного резервного копирования. Если вам нужно временно удалить и добавить элементы позже, то рекомендуется использовать **настройки исключений** вместо **удаления элементов,** чтобы обеспечить дополнительную резервную отставку вместо полного резервного копирования.

2. Выполните последующие шаги и нажмите **Finish,** чтобы завершить операцию.

## <a name="stop-protecting-files-and-folder-backup"></a>Прекратить защиту файлов и резервного копирования Folder

Существует два способа прекратить защиту резервных файлов и складок:

- **Остановите защиту и сохраните данные резервного копирования.**
  - Этот параметр остановит все будущие задания резервного копирования от защиты.
  - Служба резервного копирования Azure сохранит резервные точки восстановления, которые были сохранены на основе политики удержания.
  - Вы сможете восстановить резервные данные для неиспользованных точек восстановления.
  - Если вы решите возобновить защиту, то вы можете использовать опцию *резервного копирования Re-enable.* После этого данные будут сохранены на основе новой политики удержания.
- **Остановить защиту и удалить данные резервного копирования.**
  - Эта опция остановит все будущие задания резервного копирования от защиты ваших данных и удалит все точки восстановления.
  - Вы получите письмо с предупреждением о резервном копировании с сообщением, в каком *ваши данные для этого элемента резервного копирования были удалены. Эти данные будут временно доступны в течение 14 дней, после чего они будут навсегда удалены* и рекомендованы действия *Reprotect элемент резервного копирования в течение 14 дней для восстановления данных.*
  - Чтобы возобновить защиту, защитите в течение 14 дней от операции удаления.

### <a name="stop-protection-and-retain-backup-data"></a>Остановка защиты и сохранение данных резервного копирования

1. Откройте консоль управления MARS, перейдите на **панель действий**и **выберите Резервное копирование расписания.**

    ![Измените или остановите запланированное резервное копирование.](./media/backup-azure-manage-mars/mars-actions.png)
1. На странице **Select Policy Item** выберите изменить **расписание резервного копирования для файлов и папок** и нажмите **далее.**

    ![Измените или остановите запланированное резервное копирование.](./media/backup-azure-manage-mars/select-policy-item-retain-data.png)
1. На странице **«Изменить или остановить запланированную страницу резервного копирования»** выберите **«Остановить использование этого графика резервного копирования», но сохраните сохраненные резервные upups до тех пор, пока расписание не будет активировано снова.** Затем выберите **Следующий**.

    ![Измените или остановите запланированное резервное копирование.](./media/backup-azure-manage-mars/stop-schedule-backup.png)
1. В **Пауза Запланированное резервное копирование** просмотрите информацию и нажмите **"Завершение"**.

    ![Измените или остановите запланированное резервное копирование.](./media/backup-azure-manage-mars/pause-schedule-backup.png)
1. При **изменении процесса резервного копирования** проверьте, что пауза резервного копирования графика находится в состоянии успеха и нажмите **ближе** к финишу.

### <a name="stop-protection-and-delete-backup-data"></a>Остановить защиту и удалить данные резервного копирования

1. Откройте консоль управления MARS, перейдите на панель **действий** и выберите **Резервное копирование расписания.**
2. На странице **«Изменить или остановить запланированную страницу резервного копирования»** выберите **«Остановить использование этого графика резервного копирования» и удалить все сохраненные резервные upups.** Затем выберите **Следующий**.

    ![Измените или остановите запланированное резервное копирование.](./media/backup-azure-delete-vault/modify-schedule-backup.png)

3. На странице **«Остановить запланированное резервное копирование»** выберите **«Завершение».**

    ![Остановите запланированную резервную пристани.](./media/backup-azure-delete-vault/stop-schedule-backup.png)
4. Вам предлагается ввести PIN-код безопасности (личный идентификационный номер), который вы должны создать вручную. Для этого сначала вопием на портал Azure.
5. Перейдите к**свойствам** > **настройки** > хранилища служб **восстановления.**
6. Под **PIN-кодом безопасности**выберите **Generate**. Копируйте этот PIN-код. PIN-код действителен только в течение пяти минут.
7. В консоли управления вставьте PIN-код, а затем выберите **OK**.

    ![Создайте PIN-код безопасности.](./media/backup-azure-delete-vault/security-pin.png)

8. На странице **«Прогресс резервного копирования»** отображается следующее сообщение: *Удаленные данные резервного копирования будут храниться в течение 14 дней. По истечением времени данные резервного копирования будут удалены навсегда.*  

    ![Удалите инфраструктуру резервного копирования.](./media/backup-azure-delete-vault/deleted-backup-data.png)

После удаления элементов резервного копирования, следуйте следующим шагам с портала.

## <a name="re-enable-protection"></a>Защита от повторного включения

Если вы прекратили защиту при сохранении данных и решили возобновить защиту, то можно повторно включить график резервного копирования с помощью изменения политики резервного копирования.

1. На **действия** выберите **Резервное расписание**.
1. Выберите **график резервного копирования повторного включения. Вы также можете изменить элементы резервного копирования или время** и нажмите **Далее.**<br>

    ![Удалите инфраструктуру резервного копирования.](./media/backup-azure-manage-mars/re-enable-policy-next.png)
1. В **выберите элементы для резервного копирования,** нажмите **далее**.

    ![Удалите инфраструктуру резервного копирования.](./media/backup-azure-manage-mars/re-enable-next.png)
1. В **расписании резервного копирования**укажите расписание резервного копирования и нажмите **далее.**
1. В **политике удержания Select**укажите продолжительность удержания и нажмите **далее.**
1. Наконец, на экране **Подтверждения,** просмотрите детали политики и нажмите **Finish**.

## <a name="re-generate-passphrase"></a>Перегенерировать пароль

Пароль используется для шифрования и расшифровки данных при резервном копировании или восстановлении локальной или локальной машины с помощью агента MARS в Azure или из нее. Если вы потеряли или забыли пароль, то вы можете регенерировать пароль (при условии, что ваша машина по-прежнему зарегистрирована в Хранилище служб восстановления и настроена резервная система) путем следующих следующих шагов:

- С консоли агента MARS перейдите к**свойствам изменения** **действий** > >. Затем перейдите на **вкладку Шифрование**.<br>
- Выберите флажок **Change Passphrase.**<br>
- Введите новую фразу или нажмите **Generate Passphrase.**
- Нажмите **Просмотр,** чтобы сохранить новую фразу.

    ![Генерировать фразу.](./media/backup-azure-manage-mars/passphrase.png)
- Нажмите кнопку **ОК**, чтобы применить изменения.  Если [функция безопасности](https://docs.microsoft.com/azure/backup/backup-azure-security-feature#enable-security-features) включена на портале Azure для Убежища служб восстановления, вам будет предложено ввести PIN-код безопасности. Чтобы получить PIN-код, выполните действия, перечисленные в этой [статье](https://docs.microsoft.com/azure/backup/backup-azure-security-feature#authentication-to-perform-critical-operations).<br>
- Вставьте PIN-код безопасности с портала и нажмите **OK,** чтобы применить изменения.<br>

    ![Генерировать фразу.](./media/backup-azure-manage-mars/passphrase2.png)
- Убедитесь, что пароль надежно сохраняется в другом месте (кроме исходной машины), предпочтительно в Хранилище ключей Azure. Следите за всеми фразами, если у вас есть несколько машин, резервных копирования с агентами MARS.

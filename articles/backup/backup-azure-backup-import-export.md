# <a name="offline-backup-workflow-in-azure-backup"></a>Рабочий процесс резервного копирования в автономном режиме в резервном копировании Azure

В службу архивации Azure встроено несколько эффективных методов, которые позволяют экономить затраты на сеть и хранилище во время передачи начальных полных резервных копий данных в Azure. Обычно при этом выполняется передача больших объемов данных. Служба Azure Backup может использовать диски для передачи автономных резервных копий данных в Azure в рамках автономного заполнения.

Процесс автономного посева Резервного копирования Azure тесно интегрирован с [сервисом Azure Import/Export.](../storage/common/storage-import-export-service.md) Эту службу можно использовать для передачи исходных данных резервного копирования в Azure с помощью дисков. Если у вас есть терабайты (TBs) исходных данных резервного копирования, которые должны быть переданы через сеть с высокой задержкой и низкой пропускной способностью, вы можете использовать рабочий процесс автономного посева для отправки исходной копии резервного копирования на один или несколько жестких дисков в центр обработки данных Azure. На следующем рисунке показана схема рабочего процесса.

  ![Обзор процесса рабочего процесса импорта в автономном режиме](./media/backup-azure-backup-import-export/offlinebackupworkflowoverview.png)

Процесс резервного копирования в автономном режиме включает в себя следующие действия:

1. Вместо отправки по сети данные резервной копии записываются в промежуточное расположение.
1. Используйте утилиту *AzureOfflineBackupDiskPrep* для записи данных в месте постановки на один или несколько дисков SATA.
1. В рамках подготовительной работы утилита *AzureOfflineBackupDiskPrep* создает работу по импорту Azure. Отправьте диски SATA в ближайший центр обработки данных Azure и создайте ссылку на задание импорта для привязки к действиям.
1. В центре обработки данных Azure данные с дисков копируются в учетную запись хранилища Azure.
1. Служба Azure Backup копирует данные резервной копии из учетной записи хранилища в хранилище служб восстановления, при этом создается расписание для добавочных резервных копий.

## <a name="supported-configurations"></a>Поддерживаемые конфигурации

Следующие функции резервного копирования Azure или рабочие нагрузки поддерживают использование резервного копирования в автономном режиме для:

> [!div class="checklist"]
>
> * Резервное копирование файлов и папок с агентом Службы восстановления Microsoft Azure (MARS), также именуемым агентом резервного копирования Azure.
> * Резервное копирование всех рабочих нагрузок и файлов с помощью менеджера по защите данных System Center (DPM).
> * Резервное копирование всех рабочих нагрузок и файлов с сервером резервного копирования Microsoft Azure.
 
   > [!NOTE]
   > Резервное копирование в автономном режиме не поддерживается для резервного копирования состояния системы, выполненного с помощью резервного агента Azure.

## <a name="prerequisites"></a>Предварительные требования

  > [!NOTE]
  > Следующие предпосылки и рабочий процесс применяются только к автономному резервному копированию файлов и папок с использованием [последнего агента служб восстановления Azure.](https://aka.ms/azurebackup_agent) Для выполнения автономных резервных upups для рабочих нагрузок с помощью System Center DPM или Azure Backup Server смотрите рабочий процесс резервного копирования в [автономном режиме для DPM и Сервера резервного копирования Azure.](backup-azure-backup-server-import-export-.md)

Перед запуском рабочего процесса резервного копирования в автономном режиме выполните следующие предпосылки:

* Создайте [хранилище служб восстановления.](backup-azure-recovery-services-vault-overview.md) Чтобы создать хранилище, выполните последующие действия в [хранилище служб восстановления.](tutorial-backup-windows-server-to-azure.md#create-a-recovery-services-vault)
* Убедитесь, что только [последняя версия агента резервного копирования Azure](https://aka.ms/azurebackup_agent) установлена на клиенте Windows Server или Windows, если это применимо, и компьютер зарегистрирован в хранилище служб восстановления.
* На компьютере под управлением агента резервного копирования Azure требуется Azure PowerShell 3.7.0. Скачать и [установить версию 3.7.0 Azure PowerShell](https://github.com/Azure/azure-powershell/releases/tag/v3.7.0-March2017).
* На компьютере под управлением агента резервного копирования Azure убедитесь, что Microsoft Edge или Internet Explorer 11 установлен и включен JavaScript.
* Создайте учетную запись хранения Azure в той же подписке, что и хранилище служб восстановления.
* Убедитесь, что у вас есть [необходимые разрешения](../active-directory/develop/howto-create-service-principal-portal.md) на создание приложения Azure Active Directory. Рабочий процесс автономного резервного копирования создает приложение Active Directory Azure в подписке, связанной с учетной записью хранения Azure. Цель приложения состоит в том, чтобы предоставить резервному копированию Azure безопасный и объемный доступ к службе импорта/экспорта Azure, которая необходима для рабочего процесса резервного копирования в автономном режиме.
* Зарегистрируйте поставщика ресурсов *Microsoft.ImportExport* с подпиской, содержащей учетную запись хранения Azure. Чтобы зарегистрировать поставщик ресурсов, сделайте следующее:
    1. В основном меню выберите **Подписку.**
    1. Если вы подписались на несколько подписок, выберите подписку, которая планируется использовать для резервного копирования в автономном режиме. Если используется одна подписка, она будет отображена.
    1. В меню подписки выберите **поставщиков ресурсов** для просмотра списка поставщиков.
    1. В списке поставщиков прокрутите вниз на *Microsoft.ImportExport*. Если **статус** **незарегистрирован,** выберите **Регистрацию.**

        ![Регистрация поставщика ресурсов](./media/backup-azure-backup-import-export/registerimportexport.png)

* Создано промежуточное расположение. Это может быть сетевая папка или дополнительный диск на сервере, внутренний или внешний, на котором доступно достаточно пространства для хранения начальной копии. Например, если требуется резервное копирование файлового сервера на 500 ГБ, убедитесь, что промежуточная область составляет не менее 500 ГБ. (Хотя фактически использованный объем будет меньшим.)
* При отправке дисков в Azure используйте только 2,5-дюймовый SSD или 2,5-дюймовый или 3,5-дюймовый SATA II/III внутренние жесткие диски SATA II/III. Можно использовать жесткие диски емкостью до 10 ТБ. Список поддерживаемых дисков см. в [документации по службе импорта и экспорта Azure](../storage/common/storage-import-export-requirements.md#supported-hardware).
* Диски SATA должны быть подключены к компьютеру (*компьютер копирования*), на котором выполняется копирование данных резервной копии из промежуточного расположения на диски SATA. Убедитесь, что BitLocker включен на копировальной компьютере.

## <a name="workflow"></a>Рабочий процесс

В этом разделе описывается рабочий процесс резервного копирования в автономном режиме, чтобы ваши данные могли быть доставлены в центр обработки данных Azure и загружены в хранилище Azure. Если у вас есть вопросы об услуге импорта или любом аспекте процесса, обратитесь в [обзорную документацию службы azure Import/Export.](../storage/common/storage-import-export-service.md)

## <a name="initiate-offline-backup"></a>Запуск автономного резервного копирования

1. При составлении резервного копирования агент службы восстановления вы видите эту страницу.

    ![Страница импорта](./media/backup-azure-backup-import-export/offlinebackup_inputs.png)

1. Выберите опцию **Transfer с помощью собственных дисков.**

    > [!NOTE]
    > Используйте опцию Azure Data Box для перевода исходных данных резервного копирования в автономный режим. Эта опция экономит усилия, необходимые для приобретения собственных дисков, совместимых с Azure. Он обеспечивает Microsoft-собственные, безопасные и tamperproofProof Данные Box устройств, на которые резервные данные могут быть непосредственно записаны на агента служб восстановления.

1. Выберите **Следующий**, и заполнить в ящиках тщательно.

    ![Введите данные диска](./media/backup-azure-backup-import-export/your-disk-details.png)

   Коробки, которые вы заполняете:

    * **Промежуточное расположение**— временное хранилище, куда записывается начальная резервная копия. Место для постановки может находиться на сетевой акции или локальном компьютере. Если компьютер копии и исходный компьютер отличаются, укажите полный сетевой путь промежуточного местоположения.
    * **Учетная запись управления ресурсами Azure:** имя учетной записи типа ресурсного менеджера (общее назначение v1 или v2 общего назначения) в любой подписке Azure.
    * **Контейнер хранения Azure**: Имя капли хранилища назначения в учетной записи хранения Azure, где данные резервного копирования импортируются перед копированием в хранилище служб восстановления.
    * **Идентификатор подписки Azure:** Идентификатор для подписки Azure, где создается учетная запись хранения Azure.
    * **Название работы Azure Import:** Уникальное название, под которым служба Azure Import/Export и резервное копирование Azure отслеживают передачу данных, отправляемых на дисках, в Azure.
  
   После заполнения коробок выберите **следующий**. Сохранить **месторасположение постановки** и информацию об **имени импорта Azure.** Это необходимо для подготовки дисков.

1. По запросу вопийте подписку На Azure. Необходимо войти в систему, чтобы резервное копирование Azure можно создать приложение Active Directory Azure. Введите необходимые разрешения на доступ к службе импорта/экспорта Azure.

    ![Страница регистрации подписки Azure](./media/backup-azure-backup-import-export/azure-login.png)

1. Завершите рабочий процесс. На консоли Резервного агента Azure выберите **Back Up Now.**

    ![Резервное копирование сейчас](./media/backup-azure-backup-import-export/backupnow.png)

1. На странице **Подтверждения** мастера выберите **Back Up**. Начальная резервная копия записывается в промежуточное расположение в процессе установки.

   ![Подтвердите, что вы готовы к резервному копированию сейчас](./media/backup-azure-backup-import-export/backupnow-confirmation.png)

    После завершения операции промежуточное расположение можно использовать для подготовки дисков.

   ![Резервное копирование теперь мастер страницы](./media/backup-azure-backup-import-export/opbackupnow.png)

## <a name="prepare-sata-drives-and-ship-to-azure"></a>Подготовка дисков SATA и их отправка в Azure

Служебная программа *AzureOfflineBackupDiskPrep* выполняет подготовку дисков SATA, которые отправляются в ближайший центр обработки данных Azure. Эта утилита доступна в каталоге установки резервного агента Azure в следующем пути:

```*\Microsoft Azure Recovery Services Agent\Utils\\*```

1. Перейдите в каталог и скопируйте каталог *AzureOfflineBackupDiskPrep* на другой компьютер, где подключены диски SATA. На компьютере с подключенными дисками SATA убедитесь, что:

   * Копировальная компьютер может получить доступ к местоположению постановки для автономного посева рабочего процесса, используя тот же сетевой путь, который был предоставлен в рабочем процессе в разделе "Инициировать автономное резервное копирование".
   * На компьютере копирования включен инструмент BitLocker.
   * Установите Azure PowerShell 3.7.0.
   * Установлены последние совместимые браузеры (Microsoft Edge или Internet Explorer 11), и JavaScript включен.
   * Компьютер копирования имеет доступ к порталу Azure. При необходимости целевым компьютером может быть исходный компьютер.

     > [!IMPORTANT]
     > Если исходный компьютер — это виртуальная машина, то в качестве компьютера копирования нужно использовать другой физический сервер или клиентский компьютер, т. е. не исходный.

1. Откройте запрос повышенной команды на копировальном компьютере с каталогом утилиты *AzureOfflineBackupDiskPrep* в качестве текущего каталога. Выполните следующую команду:

    ```.\AzureOfflineBackupDiskPrep.exe s:<Staging Location Path>```

    | Параметр | Описание |
    | --- | --- |
    | s:&lt;*Путь расположения постановки*&gt; |Этот обязательный вход используется для обеспечения пути к местоположению постановки, всводноваемым в рабочий процесс в разделе "Инициировать автономное резервное копирование". |
    | p:&lt;*Путь к публикациямНастройкиФайл*&gt; |Этот дополнительный вхотворный данный используется для обеспечения пути к файлу параметров публикации Azure, который вы ввели в рабочий процесс в разделе "Инициировать автономное резервное копирование". |

    При запуске команды утилита запрашивает выбор задания импорта Azure, которое соответствует дискам, которые должны быть подготовлены. Если только одно задание импорта связано с предоставленным местом постановки, вы видите страницу, подобную этой.

    ![Ввод инструмента подготовки дисков Azure](./media/backup-azure-backup-import-export/diskprepconsole0_1.png) <br/>

1. Введите букву подключенного диска (без двоеточия), который нужно подготовить для передачи в Azure.
1. При появлении запроса подтвердите форматирование диска.
1. Вам предлагается войти в подписку Azure. Введите свои учетные данные.

    ![Регистрация подписки Azure](./media/backup-azure-backup-import-export/signindiskprep.png) <br/>

    Затем инструмент начинает готовить диск и копировать данные резервного копирования. Возможно, потребуется прикрепить дополнительные диски по запросу инструмента в случае, если предоставленный диск не имеет достаточного места для данных резервного копирования. <br/>

    При успешном выполнении средства командная строка отображает следующие сведения:

   1. Один или несколько предоставленных вами дисков подготовлены для доставки в Azure.
   1. Вы получаете подтверждение того, что ваша работа импорта была создана. В задании импорта используется предоставленное вами имя.
   1. Средство отобразит адрес доставки для центра обработки данных Azure.

      ![Подготовка лазурного диска завершена](./media/backup-azure-backup-import-export/console2.png)<br/>

1. Когда команда будет выполнена, можно обновить сведения о доставке.

1. Отправь диски по адресу, предоставленному инструментом. Храните номер отслеживания для будущей ссылки.

   > [!IMPORTANT]
   > Нет двух заданий импорта Azure может иметь один и тот же номер отслеживания. Убедитесь, что диски, подготовленные утилитой под единым заданиям по импорту Azure, отправляются вместе в один пакет и что для пакета есть единый уникальный номер отслеживания. Не комбинируйте диски, подготовленные в рамках отдельных заданий импорта Azure, в единый пакет.

## <a name="update-shipping-details-on-the-azure-import-job"></a>Обновление данных о доставке о работе по импорту Azure

Следующая процедура обновляет сведения о доставке рабочих мест Azure. Эти данные включают следующие сведения:

* Имя оператора, который доставляет диски в Azure.
* Возврат данных о доставке для дисков.

1. Войдите в свою подписку Azure.
1. В основном меню выберите **все услуги.** В диалоговом окне **All services** введите **импорт.** Когда вы видите **задания импорта/экспорта,** выберите его.

    ![Введите информацию о доставке](./media/backup-azure-backup-import-export/search-import-job.png)<br/>

    Открывается меню **заданий «Импорт/экспорт»,** и отображается список всех заданий импорта/экспорта в выбранной подписке.

1. Если у вас несколько подписок, выберите подписку, используемую для импорта данных резервного копирования. Затем выберите вновь созданное задание импорта, чтобы открыть его детали.

    ![Просмотр информации о доставке](./media/backup-azure-backup-import-export/import-job-found.png)<br/>

1. В меню **«Настройки»** для задания импорта выберите **информацию об управлении доставкой.** Введите данные обратной доставки.

    ![Информация о доставке в магазин](./media/backup-azure-backup-import-export/shipping-info.png)<br/>

1. Если у вас есть номер отслеживания от вашего перевозчика, выберите баннер на странице обзора работы импорта Azure и введите следующие сведения.

   > [!IMPORTANT]
   > Сведения о перевозчике и номер для отслеживания должны быть обновлены в течение двух недель с момента создания задания импорта Azure. Несоблюдение этой информации в течение двух недель может привести к удалению задания и необработке дисков.

   ![Предупреждение об обновлении информации](./media/backup-azure-backup-import-export/joboverview.png)<br/>

   ![Информация и номер отслеживания перевозчика](./media/backup-azure-backup-import-export/tracking-info.png)

### <a name="time-to-process-the-drives"></a>Время обработки дисков

Время, необходимое для обработки задания импорта Azure, варьируется. Время процесса зависит от таких факторов, как время доставки, тип задания, тип и размер копируется данных, а также размер предоставленных дисков. Служба импорта/экспорта Azure не имеет SLA. После получения дисков служба стремится заполнить копию данных резервного копирования на учетную запись хранения Azure в течение 7-10 дней.

### <a name="monitor-azure-import-job-status"></a>Мониторинг состояния задания на импорт Azure

Вы можете следить за состоянием работы импорта с портала Azure. Перейдите на страницу **рабочих мест «Импорт/Экспорт»** и выберите работу. Для получения дополнительной информации о состоянии рабочих мест импорта [см.](../storage/common/storage-import-export-service.md)

### <a name="finish-the-workflow"></a>Завершить рабочий процесс

Когда задание импорта будет выполнено, данные начальной резервной копии появятся в вашей учетной записи хранения. Во время следующего запланированного резервного копирования резервное копирование Azure копирует содержимое данных из учетной записи хранилища в хранилище служб восстановления.

   ![Копирование данных в хранилище служб восстановления](./media/backup-azure-backup-import-export/copyingfromstorageaccounttoazurebackup.png)<br/>

При следующем запланированном резервном копировании служба Azure Backup выполнит добавочное резервное копирование.

### <a name="clean-up-resources"></a>Очистка ресурсов

После завершения первоначального резервного копирования можно безопасно удалить данные, импортируемые в контейнер Хранения Azure, и данные резервного копирования в месте постановки.

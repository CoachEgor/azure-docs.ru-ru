# <a name="restore-azure-file-shares"></a>Восстановление файловых ресурсов Azure

В этой статье объясняется, как использовать портал Azure для восстановления всей общей папки или отдельных файлов из точки восстановления, созданной службой [Azure Backup](https://docs.microsoft.com/azure/backup/backup-overview) .

В этом руководство вы узнаете, как выполнять следующие задачи:

* Восстановление полной общей папки Azure
* Восстановление отдельных файлов или папок
* Отслеживание состояния операции восстановления

## <a name="steps-to-perform-restore"></a>Шаги для выполнения восстановления

### <a name="select-the-file-share-to-restore"></a>Выберите общую папку для восстановления

1. В [портал Azure](https://portal.azure.com/)откройте хранилище служб восстановления, которое использовалось для настройки резервного копирования для общей папки.

2. Щелкните **элементы архивации** в разделе **защищенные элементы** в колонке **Обзор** .

    ![Щелкните элемент архивации](./media/restore-afs/backup-items.png)

3. После выбора **элемента архивации**в колонке " **Обзор** " появится новая колонка со списком всех типов управления архивацией, как показано ниже:

    ![Типы управления резервным копированием](./media/restore-afs/backup-management.png)

4. В разделе **элементы архивации**в поле **тип управления архивацией**выберите Служба **хранилища Azure (служба файлов Azure)** . Вы увидите список всех файловых ресурсов и соответствующую учетную запись хранения, резервная копия которых использовалась в этом хранилище.

    ![Список всех общих файловых ресурсов](./media/restore-afs/file-shares.png)

5. В списке файловых ресурсов Azure выберите нужную общую папку, для которой требуется выполнить операцию восстановления.

### <a name="full-share-recovery"></a>Полное восстановление общего ресурса

С помощью этого параметра можно восстановить полную общую папку в исходном или альтернативном расположении.

1. Выберите параметр **восстановить общий доступ** в колонке **элемент резервного копирования** , который появится после выбора нужного общего файлового ресурса для восстановления на шаге 5 раздела [Выбор общей папки для восстановления](#select-the-file-share-to-restore) .

   ![Выбор восстановления общего ресурса](./media/restore-afs/restore-share.png)

2. После нажатия кнопки **восстановить общий ресурс**в колонке **Восстановление** откроется меню **точка восстановления** с отображением списка точек восстановления, доступных для выбранного файлового ресурса.

3. Выберите точку восстановления, которую необходимо использовать для выполнения операции восстановления, и нажмите кнопку **ОК**.

    ![Выбор точки восстановления](./media/restore-afs/restore-point.png)

4. После нажатия кнопки ОК в меню колонка восстановления переключается на пункт **восстановить расположение**. В поле **расположение восстановления**укажите место (или как) для восстановления данных. Можно выбрать один из двух вариантов:

    * **Исходное расположение**. Восстановите полную общую папку в том же расположении, что и исходный источник.
    * **Альтернативное расположение**. Восстановите полный файловый ресурс в альтернативное расположение и обеспечьте, чтобы исходная общая папка **была как есть**.

#### <a name="restore-to-original-location"></a>Восстановить в исходное расположение

1. Выберите **исходное расположение** в качестве **места назначения восстановления** и укажите, следует ли пропустить или перезаписать при наличии конфликтов. После выбора соответствующего варианта нажмите кнопку **ОК** .

    ![Выбор исходного расположения](./media/restore-afs/original-location.png)

2. Нажмите кнопку **восстановить** , чтобы начать операцию восстановления.

    ![Нажмите кнопку восстановить для запуска.](./media/restore-afs/click-restore.png)

#### <a name="restore-to-an-alternate-location"></a>Восстановление в альтернативное расположение

1. Выберите **альтернативное расположение** в качестве места назначения восстановления.
2. Выберите целевую учетную запись хранения, в которой вы хотите восстановить резервную копию содержимого, из раскрывающегося меню в поле **учетная запись хранения** .
3. На основе учетной записи хранения, выбранной на шаге 2, в раскрывающемся меню **Выбор общей папки** отобразится список общих папок, имеющихся в выбранной учетной записи хранения. Выберите общую папку, в которую вы хотите восстановить резервную копию содержимого.
4. В поле **имя папки** укажите имя папки, которую необходимо создать в целевой общей папке с восстановленным содержимым.
5. Выберите, следует ли пропустить или перезаписать при наличии конфликтов.
6. После ввода соответствующих значений во все поля нажмите кнопку **ОК** .

    ![Выбор альтернативного расположения](./media/restore-afs/alternate-location.png)

7. Нажмите кнопку восстановить, чтобы начать операцию восстановления.

    ![Нажмите кнопку восстановить для запуска.](./media/restore-afs/click-restore.png)

### <a name="item-level-recovery"></a>Восстановление на уровне элементов

С помощью этого параметра можно восстановить отдельные файлы или папки в исходном или альтернативном расположении.

1. Выберите параметр **восстановления файлов** в колонке **элемент архивации** , который появится после выбора требуемой общей папки для восстановления на шаге 5 раздела [Выбор общей папки для восстановления](#select-the-file-share-to-restore) .

    ![Выберите восстановление файлов](./media/restore-afs/file-recovery.png)

2. После нажатия кнопки **Восстановление файлов**в колонке **Восстановление** откроется меню **точка восстановления** с отображением списка точек восстановления, доступных для выбранного файлового ресурса.

3. Выберите точку восстановления, которую необходимо использовать для выполнения операции восстановления, и нажмите кнопку **ОК**.

    ![Выбор точки восстановления](./media/restore-afs/restore-point.png)

4. После нажатия кнопки ОК в меню колонка восстановления переключается на пункт **восстановить расположение**. В поле **расположение восстановления**укажите место (или как) для восстановления данных. Можно выбрать один из двух вариантов:

    * **Исходное расположение**. Восстановите выбранные файлы или папки в тот же общий файловый ресурс, что и исходный источник.
    * **Альтернативное расположение**. Восстановите выбранные файлы или папки в альтернативное расположение и **изменяйте**исходное содержимое общей папки как есть.

#### <a name="restore-to-original-location"></a>Восстановить в исходном месте

1. Выберите **исходное расположение** в качестве **места назначения восстановления** и укажите, следует ли пропустить или перезаписать при наличии конфликтов.

    ![Исходное расположение для восстановления на уровне элементов](./media/restore-afs/original-location-item-level.png)

2. Щелкните **выбрать файл** , чтобы выбрать файлы и папки, которые требуется восстановить.

    ![Щелкните выбрать файл.](./media/restore-afs/select-file.png)

3. При нажатии кнопки **выбрать файл**отображается колонка общей папки с содержимым точки восстановления общей папки, выбранной для восстановления.

4. Установите флажок, соответствующий файлу или папке, которые необходимо восстановить, и нажмите кнопку **выбрать**.

    ![Выберите файл или папку](./media/restore-afs/select-file-folder.png)

5. Повторите шаги 2-4, чтобы выбрать несколько файлов или папок для восстановления.
6. Выбрав все элементы, которые требуется восстановить, нажмите кнопку **ОК**.

    ![После выбора всех восстанавливаемых элементов нажмите кнопку ОК.](./media/restore-afs/after-selecting-items.png)

7. Нажмите кнопку восстановить, чтобы начать операцию восстановления.

    ![Нажмите кнопку восстановить для запуска.](./media/restore-afs/click-restore.png)

#### <a name="restore-to-an-alternate-location"></a>Восстановление в альтернативном расположении

1. Выберите **альтернативное расположение** в качестве места назначения восстановления.
2. Выберите целевую учетную запись хранения, в которой вы хотите восстановить резервную копию содержимого, из раскрывающегося меню в поле **учетная запись хранения** .
3. На основе учетной записи хранения, выбранной на шаге 2, в раскрывающемся меню **Выбор общей папки** отобразится список общих папок, имеющихся в выбранной учетной записи хранения. Выберите общую папку, в которую вы хотите восстановить резервную копию содержимого.
4. В поле **имя папки** укажите имя папки, которую необходимо создать в целевой общей папке с восстановленным содержимым.
5. Выберите, следует ли пропустить или перезаписать при наличии конфликтов.
6. Щелкните **выбрать файл** , чтобы выбрать файлы и папки, которые требуется восстановить.

    ![Выберите элементы для восстановления в другом расположении](./media/restore-afs/restore-to-alternate-location.png)

7. При нажатии кнопки **выбрать файл**отображается колонка общей папки с содержимым точки восстановления общей папки, выбранной для восстановления.
8. Установите флажок, соответствующий файлу или папке, которые необходимо восстановить, и нажмите кнопку **выбрать**.

    ![Выбор назначения для восстановления](./media/restore-afs/recovery-destination.png)

9. Повторите шаги 6-8, чтобы выбрать несколько файлов или папок для восстановления.
10. Выбрав все элементы, которые требуется восстановить, нажмите кнопку **ОК**.

    [Нажмите кнопку ОК после выбора всех файлов](./media/restore-afs/after-selecting-all-items.png)

11. Нажмите кнопку **восстановить** , чтобы начать операцию восстановления.

## <a name="track-restore-operation"></a>Отслеживание операции восстановления

Когда вы запускаете операцию восстановления, служба Backup создает задание для отслеживания операции. Azure Backup отображает уведомления о задании на портале. Чтобы просмотреть операции для задания, щелкните гиперссылку уведомлений.

![Щелкните гиперссылку уведомления](./media/restore-afs/notifications-link.png)

Вы также можете отслеживать ход восстановления из хранилища служб восстановления. Ниже приведены шаги для проверки состояния операции восстановления.

1. Откройте хранилище служб восстановления, в котором вы активировали операцию восстановления.
2. Щелкните **задания резервного копирования** в **разделе "мониторинг** " в колонке " **Обзор** ", чтобы просмотреть состояние операций, выполняемых с разными рабочими нагрузками.

    ![Щелкните задания резервного копирования.](./media/restore-afs/backup-jobs.png)

3. Щелкните имя рабочей нагрузки, соответствующее общей папке, чтобы просмотреть дополнительные сведения об операции восстановления, такие как передаваемые данные, количество восстановленных файлов и т. д.

    ![См. восстановленные сведения](./media/restore-afs/restore-details.png)

# <a name="manage-and-monitor-backed-up-sap-hana-databases"></a>Мониторинг резервных копий баз данных SAP HANA и управление ими

В этой статье описаны общие задачи для управления и мониторинга баз данных SAP HANA, которые работают на виртуальном компьютере Azure (VM) и которые резервируются в хранилище служб ы восстановления резервного копирования Azure службой [резервного копирования.](https://docs.microsoft.com/azure/backup/backup-overview) Вы узнаете, как отслеживать задания и оповещения, запускать резервное копирование по требованию, отстранять политики, останавливать и обновлять защиту базы данных и отменить учет VM из резервных ups.

Если вы еще не настроили резервные ожидания для баз данных SAP HANA, см. [Резервное копирование баз данных SAP HANA на M M Azure VMs.](https://docs.microsoft.com/azure/backup/backup-azure-sap-hana-database)

## <a name="monitor-manual-backup-jobs-in-the-portal"></a>Мониторинг заданий ручного резервного копирования на портале

Резервное копирование Azure отображает все задания с вручную в разделе **задания резервного копирования** на портале Azure.

![Раздел задания резервного копирования](./media/sap-hana-db-manage/backup-jobs.png)

Задания, которые вы видите на этом портале, включают обнаружение и регистрацию базы данных, а также резервное копирование и восстановление операций. Запланированные задания, включая резервные копии журнала, не отображаются в этом разделе. Ручно срабатывая резервные копирования из SAP HANA местных клиентов (Студия / Кокпит / DBA Cockpit) также не отображаются здесь.

![Список заданий резервного копирования](./media/sap-hana-db-manage/backup-jobs-list.png)

Чтобы узнать больше о мониторинге, перейдите на [портал Azure](https://docs.microsoft.com/azure/backup/backup-azure-monitoring-built-in-monitor) и [мониторинг с помощью Azure Monitor.](https://docs.microsoft.com/azure/backup/backup-azure-monitoring-use-azuremonitor)

## <a name="view-backup-alerts"></a>Просмотр оповещений о резервном копировании

Оповещения являются простым средством мониторинга резервных данных баз данных SAP HANA. Оповещения помогут вам сосредоточиться на событиях, которые вы заботитесь о наиболее, не заблудившись во множестве событий, которые генерирует резервное копирование. Резервное копирование Azure позволяет устанавливать оповещения, и они могут контролироваться следующим образом:

* Войдите на [портал Azure](https://portal.azure.com/).
* На панели мониторинга хранилища выберите **резервные оповещения.**

  ![Резервное копирование оповещений на панели мониторинга хранилища](./media/sap-hana-db-manage/backup-alerts-dashboard.png)

* Вы сможете увидеть оповещения:

  ![Список резервных оповещений](./media/sap-hana-db-manage/backup-alerts-list.png)

* Нажмите на оповещения, чтобы увидеть более подробную информацию:

  ![Сведения об оповещении](./media/sap-hana-db-manage/alert-details.png)

Сегодня резервное копирование Azure позволяет отправлять оповещения по электронной почте. Эти предупреждения:

* Срабатывает для всех сбоев резервного копирования.
* Консолидированна на уровне базы данных по коду ошибки.
* Отправлено только для первого сбоя резервного копирования базы данных.

Чтобы узнать больше о мониторинге, перейдите на [портал Azure](https://docs.microsoft.com/azure/backup/backup-azure-monitoring-built-in-monitor) и [мониторинг с помощью Azure Monitor.](https://docs.microsoft.com/azure/backup/backup-azure-monitoring-use-azuremonitor)

## <a name="management-operations"></a>Операции управления

Резервное копирование Azure упрощает управление резервной базой данных SAP HANA с большим количеством поддерживаемых операций управления. Эти операции более подробно обсуждаются в следующих разделах.

### <a name="run-an-on-demand-backup"></a>Выполнение резервного копирования по запросу

Резервное копирование работает в соответствии с графиком политики. Вы можете запустить резервную по требованию следующим образом:

1. В меню хранилища щелкните **Элементы архивации**.
2. В **элементах резервного копирования**выберите VM, подпустив базу данных SAP HANA, а затем нажмите **кнопку «Теперь— резервная кнопка».**
3. В **Backup Now**используйте элемент управления календаря, чтобы выбрать последний день сохранения точки восстановления. После этого щелкните **OK**.
4. Отслеживайте уведомления на портале. Вы можете отслеживать ход выполнения заданий на панели мониторинга хранилища >**в процессе выполнения**заданий **резервного** > копирования. В зависимости от размера базы данных создание первоначальной резервной системы может занять некоторое время.

### <a name="hana-native-client-integration"></a>Интеграция клиентов HANA

Теперь по требованию полные резервные копирования, срабатывающие с любого из нативных клиентов HANA, будут отображаться в качестве полного резервного копирования на странице **элементов резервного копирования.**

![Последний запуск резервных копирований](./media/sap-hana-db-manage/last-backups.png)

Эти специальные полные резервные копии также будут отображаться в списке точек восстановления для восстановления.

![Список точек восстановления](./media/sap-hana-db-manage/list-restore-points.png)

### <a name="run-sap-hana-native-client-backup-on-a-database-with-azure-backup-enabled"></a>Выполнить резервное копирование родного клиента SAP HANA в базе данных с включенным резервным копированием Azure

Если вы хотите взять локальную резервную копию (с помощью HANA Studio / Cockpit) базы данных, которая резервируется с помощью резервного копирования Azure, сделайте следующее:

1. Подождите, пока будет завершена резервная копии полного или регистра для завершения базы данных. Проверьте состояние в SAP HANA Studio/ Cockpit.
2. Отменяем резервные копии журналов и установите каталог резервного копирования в файловую систему для соответствующей базы данных.
3. Для этого дважды щелкните **systemdb** > **Configuration Select** > **Database** > **Filter (Log)**.
4. Установить **enable_auto_log_backup** **нет**.
5. Установить **log_backup_using_backint** **ложным**.
6. Возьмите полную резервную резервную часть базы данных по требованию.
7. Подождите, пока будет завершена полная резервная и каталогная резервная часть.
8. Верните предыдущие настройки обратно в настройки Для Azure:
   * Установите **enable_auto_log_backup** **Да**.
   * Установить **log_backup_using_backint** **True**.

### <a name="change-policy"></a>Изменение политики

Можно изменить базовую политику для элемента резервного копирования SAP HANA.

* В панели мониторинга хранилища перейдите к **элементам резервного копирования:**

  ![Выберите элементы резервного копирования](./media/sap-hana-db-manage/backup-items.png)

* Выберите **SAP HANA в Azure VM**

  ![Выберите SAP HANA в Azure VM](./media/sap-hana-db-manage/sap-hana-in-azure-vm.png)

* Выберите элемент резервного копирования, базовую политику которого вы хотите изменить
* Нажмите на существующую политику резервного копирования

  ![Выберите существующую политику резервного копирования](./media/sap-hana-db-manage/existing-backup-policy.png)

* Измените политику, выбрав из списка. [При необходимости создайте новую политику резервного копирования.](https://docs.microsoft.com/azure/backup/backup-azure-sap-hana-database#create-a-backup-policy)

  ![Выберите политику из списка выпадающих](./media/sap-hana-db-manage/choose-backup-policy.png)

* Сохраните изменения

  ![Сохраните изменения](./media/sap-hana-db-manage/save-changes.png)

* Изменение политики повлияет на все связанные элементы резервного копирования и запустит соответствующие задания **защиты настройки.**

>[!NOTE]
> Любое изменение периода удержания будет ретроспективно применено ко всем старым точкам восстановления, кроме новых.
>
> Дополнительные политики резервного копирования не могут использоваться для баз данных SAP HANA. Инкрементное резервное копирование в настоящее время не поддерживается для этих баз данных.

### <a name="modify-policy"></a>Изменение политики

Измените политику для изменения типов резервного копирования, частот и диапазона удержания.

>[!NOTE]
>Любое изменение периода удержания будет применяться задним числом ко всем старым точкам восстановления, в дополнение к новым.

1. На панели мониторинга хранилища перейдите на **управление > политики резервного копирования** и выберите политику, которая требуется отсылку к отсвадеванию.

   ![Выберите политику для отосвобки для отодвиной](./media/sap-hana-db-manage/manage-backup-policies.png)

1. Выберите **Изменить**.

   ![Выберите изменить](./media/sap-hana-db-manage/modify-policy.png)

1. Выберите частоту для типов резервного копирования.

   ![Выберите частоту резервного копирования](./media/sap-hana-db-manage/choose-frequency.png)

Изменение политики повлияет на все связанные элементы резервного копирования и запустит соответствующие задания **защиты настройки.**

### <a name="inconsistent-policy"></a>Непоследовательная политика

Иногда изменение операции политики может привести к **несогласованной** версии политики для некоторых элементов резервного копирования. Это происходит, когда соответствующее задание **защиты настройки** выходит из строя для элемента резервного копирования после запуска операции изменения политики. В представлении элемента резервного копирования отображается следующее:

![Непоследовательная политика](./media/sap-hana-db-manage/inconsistent-policy.png)

Вы можете исправить версию политики для всех затронутых элементов одним щелчком мыши:

![Исправление версии политики](./media/sap-hana-db-manage/fix-policy-version.png)

### <a name="stop-protection-for-an-sap-hana-database"></a>Остановка защиты для базы данных SAP HANA

Вы можете остановить создание резервных копий базы данных SQL Server несколькими способами:

* можно остановить все будущие задания резервного копирования и удалить все точки восстановления;
* остановить все будущие задания резервного копирования, но сохранить точки восстановления без изменений.

Если вы решили сохранить точки восстановления, учитывайте следующие факторы:

* все точки восстановления навсегда остаются неизменными, любая очистка прекращается одновременно с отключением защиты;
* С вас будет взиматься плата за защищенный экземпляр и потребленохранилище. Для получения дополнительной [Azure Backup pricing](https://azure.microsoft.com/pricing/details/backup/)информации см.
* если вы удалите источник данных без остановки резервного копирования, новые операции резервного копирования будут завершаться сбоем.

Чтобы остановить защиту базы данных, выполните следующие действия.

* На панели мониторинга хранилища выберите **элементы резервного копирования.**
* Под **типом управления резервным копированием**выберите **SAP HANA в Azure VM**

  ![Выберите SAP HANA в Azure VM](./media/sap-hana-db-manage/sap-hana-azure-vm.png)

* Выберите базу данных, для которой вы хотите остановить защиту на:

  ![Выберите базу данных, чтобы остановить защиту](./media/sap-hana-db-manage/select-database.png)

* В меню базы данных выберите **Остановить архивацию**.

  ![Выберите резервное копирование стопа](./media/sap-hana-db-manage/stop-backup.png)

* В меню **Stop Backup** выберите, следует ли сохранять или удалять данные. Если вы хотите, предоставьте причину и комментарий.

  ![Выберите сохранить или удалить данные](./media/sap-hana-db-manage/retain-backup-data.png)

* Выберите **остановить резервное копирование.**

### <a name="resume-protection-for-an-sap-hana-database"></a>Возобновление защиты базы данных SAP HANA

При остановке защиты базы данных SAP HANA, если вы выберете опцию **Сохранить резервные данные,** вы можете возобновить защиту. Если вы не сохраните резервные данные, вы не сможете возобновить защиту.

Чтобы возобновить защиту базы данных SAP HANA:

* Откройте элемент резервного копирования и выберите **резервное копирование резюме.**

   ![Выберите резервное копирование резюме](./media/sap-hana-db-manage/resume-backup.png)

* В меню **Политика архивации** выберите политику и нажмите кнопку **Сохранить**.

### <a name="unregister-an-sap-hana-instance"></a>Отменить регистрацию экземпляра SAP HANA

Отменить регистрацию экземпляра SAP HANA после того, как вы отключаете защиту, но прежде чем удалить хранилище:

* На панели мониторинга хранилища, под **управлением,** выберите **инфраструктуру резервного копирования.**

   ![Выберите незарегистрированную](./media/sap-hana-db-manage/unregister.png)

### <a name="re-register-extension-on-the-sap-hana-server-vm"></a>Расширение перерегистрации на сервере SAP HANA VM

Иногда расширение рабочей нагрузки на VM может повешно по той или иной причине. В таких случаях все операции, срабатывающие на ВМ, начнут сбой. Возможно, вам потребуется перерегистрировать расширение на VM. Операция перерегистрации переустанавливает расширение резервного копирования рабочей нагрузки на VM для продолжения операций.

Используйте эту опцию с осторожностью: при срабатывании на VM с уже здоровым расширением, эта операция приведет к перезапуску расширения. Это может привести к сбою всех незавершенных заданий. Проверьте один или несколько [симптомов](backup-azure-sap-hana-database-troubleshoot.md#re-registration-failures) перед запуском операции перерегистрации.

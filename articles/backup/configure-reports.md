# <a name="configure-azure-backup-reports"></a>Настройка отчетов службы Azure Backup

Общим требованием для резервных админинов является получение информации о резервных копированиях на основе данных, охватывающих длительный период времени. Для такого решения может быть несколько вариантов использования - распределение и прогнозирование потребляемого облачного хранилища, аудит резервных копий и восстановления, а также выявление ключевых тенденций на различных уровнях детализации.

Сегодня Резервное копирование Azure предоставляет решение для отчетов, которое использует [журналы Azure Monitor и](https://docs.microsoft.com/azure/azure-monitor/log-query/get-started-portal) [рабочие книги Azure,](https://docs.microsoft.com/azure/azure-monitor/app/usage-workbooks)помогая получить информацию о резервных копиях во всей области резервного копирования. В этой статье объясняется, как настроить и просмотреть отчеты о резервном копировании.

## <a name="supported-scenarios"></a>Поддерживаемые сценарии

* Отчеты о резервном копировании поддерживаются для Azure VMs, СЗЛ в Azure VMs, SAP HANA/ASE в Azure VMs, агента резервного копирования Azure (MARS), сервера резервного копирования Azure (MABS) и Системного центра DPM.
* Для рабочих нагрузок DPM отчеты резервного копирования поддерживаются для DPM-версии 5.1.363.0 и выше, а также версии Agent 2.0.9127.0 и выше.
* Для рабочих нагрузок MABS отчеты резервного копирования поддерживаются для MABS Version 13.0.415.0 и выше, и Agent Version 2.0.9170.0 и выше.
* Отчеты о резервном копировании можно просматривать во всех элементах резервного копирования, хранилищах, подписках и регионах, пока их данные отправляются в рабочее пространство Log Analytics (LA), к которое пользователь имеет доступ. 
* Если вы являетесь пользователем [Azure Lighthouse](https://docs.microsoft.com/azure/lighthouse/) с делегированным доступом к подпискам ваших клиентов, вы можете использовать эти отчеты с Azure Lighthouse для просмотра отчетов среди всех ваших клиентов.
* Данные для заданий резервного копирования журнала в настоящее время не отображаются в отчетах.

## <a name="getting-started"></a>Начало работы

Чтобы начать с использования отчетов, выполните три шага, описанные ниже:

1. **Создайте рабочее пространство Для анализа журналов (LA) (или используйте существующее):**

Для хранения данных резервной отчетности необходимо настроить одно или несколько рабочих пространств LA Workspaces. Местоположение и подписка, где это рабочее пространство Лос-Анджелеса может быть создано, не зависит от местоположения и подписки, где существуют хранилища. 

Обратитесь к следующей статье: [Создайте рабочее пространство для анализа журналов на портале Azure](https://docs.microsoft.com/azure/azure-monitor/learn/quick-create-workspace) для создания рабочего пространства Лос-Анджелеса.

По умолчанию данные в рабочем пространстве Лос-Анджелеса сохраняются в течение 30 дней. Чтобы увидеть данные в течение более длительного периода времени, измените период хранения рабочего пространства LA. Чтобы изменить период хранения, обратитесь к следующей статье: [Управление использованием и затратами с помощью журналов Azure Monitor.](https://docs.microsoft.com/azure/azure-monitor/platform/manage-cost-storage)

2. **Настройка параметров диагностики для хранилищ:**

Ресурсы Менеджера ресурсов Azure, такие как хранилища служб восстановления, записывая информацию о запланированных операциях и пользовательских операциях в качестве диагностических данных. 

В разделе мониторинга хранилища служб восстановления выберите **параметры диагностики** и укажите цель диагностических данных хранилища восстановления. [Подробнее об использовании диагностических событий.](https://docs.microsoft.com/azure/backup/backup-azure-diagnostic-events)

![Диагностика Настройки Клинок](./media/backup-azure-configure-backup-reports/resource-specific-blade.png)

Резервное копирование Azure также предоставляет встроенную политику Azure, которая автоматизирует конфигурацию диагностических настроек для всех хранилищ в данной области. Обратитесь к следующей статье, чтобы узнать, как использовать эту политику: [Настройка Настройки диагностики Убежища в масштабе](https://docs.microsoft.com/azure/backup/azure-policy-configure-diagnostics)

> [!NOTE]
> После настройки диагностики может потребоваться до 24 часов, чтобы первоначальный толчок данных. Как только данные начинают поступать в рабочее пространство Лос-Анджелеса, вы не сможете сразу увидеть данные в отчетах, так как данные за текущий частичный день не отображаются в отчетах (подробнее [здесь).](https://docs.microsoft.com/azure/backup/configure-reports#conventions-used-in-backup-reports) Поэтому рекомендуется начать просмотр отчетов через 2 дня после настройки хранилищ для отправки данных в журнал Analytics.

3. **Просмотр отчетов на портале Azure:**

После настройки хранилищ для отправки данных в Лос-Анджелес просматривайте отчеты резервного копирования, перемещаясь по лезвию любого хранилища и нажимайте на элемент меню **резервных отчетов.** 

![Vault Dashboard](./media/backup-azure-configure-backup-reports/vault-dashboard.png)

Нажав на эту ссылку открывает резервное копирование отчет Workbook.

> [!NOTE]
> * В настоящее время начальная загрузка отчета может занять до 1 минуты.
> * Хранилище служб восстановления является лишь отправной точкой для отчетов о резервных копированиях. Как только справочная книга отчетов о резервном копировании откроется из лезвия хранилища, вы сможете увидеть данные, агрегированные во всех хранилищах (выбрав соответствующий набор рабочих пространств LA).

Ниже приведено описание различных вкладок, которые содержит отчет:

* **Резюме** - Вкладка Резюме предоставляет обзор на высоком уровне вашего резервного поместья. Под вкладкой Резюме можно получить быстрый взгляд на общее количество элементов резервного копирования, общее количество потребляемых облачных хранилищ, количество защищенных экземпляров и скорость успешности выполнения заданий на тип рабочей нагрузки. Для получения более подробной информации о конкретном типе артефакта резервного копирования перейдите на соответствующие вкладки.

![Вкладка Summary (Сводка)](./media/backup-azure-configure-backup-reports/summary.png)

* **Элементы резервного копирования** - вкладка Элементы резервного копирования позволяет просматривать информацию и тенденции в облачном хранилище, потребляемом на уровне элемента резервного копирования. Например, если вы используете резервное копирование ВМ ВМ, вы можете увидеть резервное копирование облачного хранилища, потребляемого для каждой базы данных S'L. Вы также можете просмотреть данные для элементов резервного копирования определенного статуса защиты. Например, нажав на плитку **Protection Stopped** в верхней части вкладки, фильтруются все нижевые виджеты, чтобы показывать данные только для элементов резервного копирования в состоянии защиты остановлено.

![Вкладка элементов резервного копирования](./media/backup-azure-configure-backup-reports/backup-items.png)

* **Использование** - Вкладка «Использование» помогает просматривать ключевые параметры выставления счетов для резервных копирования. Информация, показанная на этой вкладке, находится на уровне биллинговой сущности (защищенный контейнер). Например, в случае резервного копирования сервера DPM в Azure можно просмотреть тенденцию защищенных экземпляров и облачного хранилища, потребляемого для сервера DPM. Аналогичным образом, если вы используете S'L в резервном копировании Azure или SAP HANA в Azure Backup, эта вкладка предоставляет информацию, связанную с использованием, на уровне виртуальной машины, в которой содержатся эти базы данных.

![Вкладка использования](./media/backup-azure-configure-backup-reports/usage.png)

* **Вкладка Jobs** - The Jobs позволяет просматривать длительные тенденции заданий, такие как количество неудачных заданий в день и основные причины сбоя в работе. Эту информацию можно просматривать как на агрегированном уровне, так и на уровне элемента резервного копирования. Нажатие на определенный элемент резервного копирования в сетке позволяет просматривать подробную информацию о каждом заданиях, которые были срабатываны на этом элементе резервного копирования в выбранном временном диапазоне.

![Вкладка "Задания"](./media/backup-azure-configure-backup-reports/jobs.png)

* **Политики** - Вкладка Политики позволяет просматривать информацию обо всех активных политиках, таких как количество связанных элементов и общее хранилище облачных технологий, потребляемых элементами, резервными по тем данным политиками. Нажатие на конкретную политику позволяет просматривать информацию о каждом связанном с ней элементе резервного копирования.

![Вкладка политики](./media/backup-azure-configure-backup-reports/policies.png)

## <a name="exporting-to-excel"></a>Экспорт в Excel

Нажав на кнопку вниз стрелка в правом верхнем виде любого виджета (таблица / диаграмма) экспортирует содержимое этого виджета, как лист Excel, как и с существующими фильтрами применяется. Для экспорта большего количества строк таблицы в Excel можно увеличить количество строк, отображаемых на странице, используя выпадение **строк на странице** в верхней части каждой сетки.

## <a name="pinning-to-dashboard"></a>Закрепление на панели мониторинга

Нажмите значок Pin в верхней части каждого виджета, чтобы прикрепить виджет к панели мониторинга портала Azure. Это поможет создать индивидуальные панели мониторинга, адаптированные для отображения наиболее важной информации, которая вам нужна.

## <a name="cross-tenant-reports"></a>Отчеты о кросс-арендаторах

Если вы являетесь пользователем [Azure Lighthouse](https://docs.microsoft.com/azure/lighthouse/) с делегированным доступом к подпискам в нескольких средах для арендаторов, вы можете использовать фильтр подписки по умолчанию (нажав на значок фильтра в правом верхнем правом доступе к порталу Azure), чтобы выбрать все подписки, для просмотра данных. Это позволит вам выбрать рабочие места LA в ваших клиентах для просмотра многоквартирных отчетов.

## <a name="conventions-used-in-backup-reports"></a>Конвенции, используемые в отчетах о резервном копировании

* Фильтры работают слева направо и сверху вниз на каждой вкладке, то есть любой фильтр применяется только ко всем тем виджетам, которые расположены либо справа от этого фильтра, либо ниже этого фильтра. 
* Нажав на цветную плитку фильтры виджеты ниже плитки для записей, относящихся к стоимости этой плитки. Например, нажатие на плитку *Protection Stopped* в вкладке Элементы резервного копирования фильтрует сетки и диаграммы ниже, чтобы показать данные для элементов резервного копирования в состоянии 'Protection Stopped'.
* Плитки, которые не окрашены, не являются интерактивными.
* Данные по части текущего дня в отчетах не отображаются. Таким образом, при выбранном значении **Диапазона Времени,** ***за последние 7 дней,*** отчет показывает записи за последние 7 завершенных дней (который не включает текущий день).
* Отчет показывает детали заданий (кроме заданий журнала), которые были **срабатываны** в выбранном временном диапазоне. 
* Значения, отображаемые для облачных хранилищ и защищенных экземпляров, находятся в **конце** выбранного диапазона времени.
* Элементы резервного копирования, отображаемые в отчетах, являются теми элементами, которые существуют в **конце** выбранного диапазона времени. Элементы резервного копирования, удаленные в середине выбранного диапазона времени, не отображаются. Та же конвенция применяется и для политик резервного копирования.

## <a name="query-load-times"></a>Время загрузки запроса

Виджеты в отчете о резервном копировании питаются от запросов Kusto, которые работают на рабочих пространствах пользователя LA Workspaces. Поскольку эти запросы обычно включают обработку больших объемов данных, с несколькими соединениями, позволяющими более богатым исчисляться информацией, виджеты могут не загружаться мгновенно, когда пользователь просматривает отчеты в большом резервном поместья. | **Источники данных**                         | **Горизонт времени** | **Время нагрузки (приблизительно.)**                                              |
| --------------------------------- | ------------- | ------------------------------------------------------------ |
| No 5 K                       | 1 месяц          | Плитки: 5-10 сек <br> Сетки: 5-10 сек <br> Диаграммы: 5-10 сек <br> Фильтры уровня отчета: 5-10 сек|
| No 5 K                       | 3 месяца          | Плитки: 5-10 сек <br> Сетки: 5-10 сек <br> Диаграммы: 5-10 сек <br> Фильтры уровня отчета: 5-10 сек|
| 10 K                       | 3 месяца          | Плитки: 15-20 сек <br> Сетки: 15-20 сек <br> Графики: 1-2 мин. <br> Фильтры уровня отчета: 25-30 сек|
| No 15 K                       | 1 месяц          | Плитки: 15-20 сек <br> Сетки: 15-20 сек <br> Диаграммы: 50-60 сек <br> Фильтры уровня отчета: 20-25 сек|
| No 15 K                       | 3 месяца          | Плитки: 20-30 сек <br> Сетки: 20-30 сек <br> Графики: 2-3 мин. <br> Фильтры уровня отчета: 50-60 сек |

## <a name="what-happened-to-the-power-bi-reports"></a>Что случилось с Power BI Отчеты?
* Наше предыдущее приложение Power BI для отчетности (которое исходило от учетной записи хранения Azure) находится на пути амортизации. Рекомендуется начать отправку диагностических данных хранилища в Log Analytics, как описано выше, для просмотра отчетов.

* Кроме того, схема V1 отправки диагностических данных на учетную запись хранения или в LA Workspace также находится на пути амортизации. Это означает, что если вы написали какие-либо пользовательские запросы или автоматизацию на основе схемы V1, рекомендуется обновить эти запросы, чтобы использовать поддерживаемую в настоящее время схему V2.

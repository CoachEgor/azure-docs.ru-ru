# <a name="back-up-azure-file-shares-in-a-recovery-services-vault"></a>Резервное копирование акций файлов Azure в хранилище служб восстановления

В этой статье объясняется, как использовать портал Azure для резервного копирования [акций файлов Azure.](https://docs.microsoft.com/azure/storage/files/storage-files-introduction)

В этой статье вы узнаете, как выполнять следующие задачи.

* Создайте хранилище служб восстановления,
* Откройте для себя файлы и настроить резервные копирования.
* Запустите задания резервного копирования по требованию для создания точки восстановления.

## <a name="prerequisites"></a>Предварительные требования

* Определите или создайте [хранилище служб восстановления](#create-a-recovery-services-vault) в том же регионе, что и учетная запись хранилища, хусируя файла.
* Убедитесь, что доля файла присутствует в одном из [поддерживаемых типов учетной записи хранилища.](#limitations-for-azure-file-share-backup-during-preview)

## <a name="limitations-for-azure-file-share-backup-during-preview"></a>Ограничения резервного копирования общих папок Azure в предварительной версии

Функция резервного копирования файловых ресурсов Azure находится в стадии предварительной версии. Файловые ресурсы Azure поддерживаются в учетных записях хранения общего назначения версий 1 и 2. Вот ограничения для резервного копирования акций файлов Azure:

* Поддержка резервного копирования файлов Azure в учетных записях хранения с репродукцией из [зоны избыточного хранилища](https://docs.microsoft.com/azure/storage/common/storage-redundancy-zrs) (ЗРС) в настоящее время ограничена [этими регионами.](https://docs.microsoft.com/azure/backup/backup-azure-files-faq#in-which-geos-can-i-back-up-azure-file-shares)
* В настоящее время Резервное копирование Azure поддерживает настройку запланированных раз в день резервных копирований файлов Azure.
* Максимальное число запланированных резервных копий в день — одна.
* Максимальное число резервных копий по запросу в день — четыре.
* Чтобы предотвратить случайное удаление резервных копий в хранилище служб восстановления, используйте [блокировку ресурсов](https://docs.microsoft.com/cli/azure/resource/lock?view=azure-cli-latest) в учетной записи хранения.
* Не удаляйте снимки, созданные резервным копированием Azure. Обнаружение моментальных снимков может привести к потере точек восстановления или восстановлению сбоев.
* Не удаляйте файлы, защищенные резервным копированием Azure. Текущее решение удаляет все моменты, сделанные резервным копированием Azure после удаления раздела файла, поэтому все точки восстановления будут потеряны.

## <a name="modify-storage-replication"></a>Изменение репликации хранилища

По умолчанию хранилища используют [гео-избыточное хранилище (GRS).](https://docs.microsoft.com/azure/storage/common/storage-redundancy-grs)

* Если хранилище является основным механизмом резервного копирования, мы рекомендуем использовать GRS.
* В качестве недорогого варианта можно использовать [локально избыточное хранилище (LRS).](https://docs.microsoft.com/azure/storage/common/storage-redundancy-lrs?toc=%2fazure%2fstorage%2fblobs%2ftoc.json)

Чтобы изменить тип репликации хранилища:

1. В новом хранилище выберите **Свойства** в разделе **Настройки.**

1. На странице **Свойства,** под **конфигурацией резервного** **копирования,** выберите обновление .

1. Выберите тип репликации хранилища и выберите **Сохранить.**

    ![Обновление конфигурации резервного копирования](./media/backup-afs/backup-configuration.png)

> [!NOTE]
> Вы не можете изменить тип репликации хранилища после настройки хранилища и содержит элементы резервного копирования. Если вы хотите сделать это, вам нужно воссоздать хранилище.
>

## <a name="discover-file-shares-and-configure-backup"></a>Откройте для себя файлы и настроить резервную систему

1. На [портале Azure](https://portal.azure.com/)откройте хранилище служб восстановления, в использовании чего можно использовать для резервного копирования файла.

1. В панели мониторинга **хранилища «Восстановление служб»** выберите **«Backup**».

   ![Хранилище Служб восстановления](./media/backup-afs/recovery-services-vault.png)

    а. В **цели резервного копирования**, установить **Где ваша рабочая нагрузка работает?** к **Azure**.

    ![Выберите раздел файлов Azure в качестве цели резервного копирования](./media/backup-afs/backup-goal.png)

    b.  В Каком списке резервного **Azure File Share** **копирования выберите резервное копирование?**

    c.  Выберите **резервное копирование** для регистрации расширения общего обмена файлами Azure в хранилище.

    ![Выберите резервное копирование, чтобы связать совместное хранилище файлов Azure с хранилищем](./media/backup-afs/register-extension.png)

1. После выбора **резервного копирования**панель **резервного копирования** открывается и предлагает выбрать учетную запись хранилища из списка обнаруженных учетных записей хранения. Они либо связаны с этим хранилищем, либо присутствуют в том же регионе, что и хранилище, но еще не связаны с хранилищем служб восстановления.

   ![Выбор учетной записи хранения](./media/backup-afs/select-storage-account.png)

1. Из списка обнаруженных учетных записей хранилища выберите учетную запись и выберите **OK**. Azure выполняет поиск в учетной записи хранилища для создания копий файлов. Если недавно вы добавили свои файлы и не видите их в списке, дайте некоторое время для отображаются акции файлов.

    ![Обнаружение файлов](./media/backup-afs/discovering-file-shares.png)

1. Из списка **файлов-акций** выберите одну или несколько файлов, которые вы хотите создать для резервного копирования. Нажмите кнопку **ОК**.

1. После выбора файлов меню **резервного копирования** переходит в **политику резервного копирования.** Из этого меню выберите существующую политику резервного копирования или создайте новую. Затем выберите **Резервное копирование включить.**

    ![Выбор политики резервного копирования](./media/backup-afs/select-backup-policy.png)

После настройки политики резервного копирования моментальный снимок акций файла делается в запланированное время. Точка восстановления также сохраняется за выбранный период.

## <a name="create-an-on-demand-backup"></a>Создание резервной копии по запросу

Иногда может потребоваться создать моментальный снимок резервного копирования или точку восстановления за пределами времени, запланированного в политике резервного копирования. Распространенной причиной создания резервного копирования по требованию является сразу после настройки политики резервного копирования. В зависимости от расписания в политике резервного копирования может быть несколько часов или дней до момента снимок. Чтобы защитить свои данные до резервного копирования по расписанию, инициируйте резервное копирование по запросу. Для внесения запланированных изменений в файлов часто требуется создание резервного копирования по требованию.

### <a name="create-a-backup-job-on-demand"></a>Создание задания резервного копирования по требованию

1. Откройте хранилище служб восстановления, используемое для резервного копирования файла. На панели **обзора** выберите **элементы резервного копирования** в разделе **Защищенные элементы.**

   ![Выберите элементы резервного копирования](./media/backup-afs/backup-items.png)

1. После выбора **элементов резервного копирования**рядом с панелью **«Обзор»** отображается новая панель, в которых перечислены все **типы управления резервным** копированием.

   ![Список типов управления резервного копирования](./media/backup-afs/backup-management-types.png)

1. Из списка **типа управления резервным копированием** выберите **хранилище Azure (Azure Files)**— — Вы увидите список всех файлов и соответствующие учетные записи хранения, подкрепленные с помощью этого хранилища.

   ![Элементы резервного копирования Хранилища Azure (Azure Files)](./media/backup-afs/azure-files-backup-items.png)

1. Из списка разделов файлов Azure выберите нужное совместное файла. Отображаются детали **элемента резервного копирования.** В меню **элемента резервного копирования** **выберите сейчас резервное копирование.** Поскольку это задание резервного копирования по требованию, нет политики удержания, связанной с точкой восстановления.

   ![Выберите резервное копирование сейчас](./media/backup-afs/backup-now.png)

1. Открывается панель **резервного копирования.** Укажите срок хранения точки восстановления (последний день). Максимальное удержание может событь 10 лет для резервного копирования по требованию.

   ![Выберите дату удержания](./media/backup-afs/retention-date.png)

1. Выберите **OK,** чтобы подтвердить работу резервного копирования по требованию, которая выполняется.

1. Мониторинг уведомлений портала, чтобы отслеживать завершение выполнения задания резервного копирования. Вы можете следить за ходом выполнения работ в панели мониторинга хранилища. Выберите >  **вакансии резервного копирования****в процессе.**

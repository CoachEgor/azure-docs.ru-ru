# <a name="install-the-azure-backup-mars-agent"></a>Установка агента резервного копирования Azure MARS

В этой статье объясняется, как установить агент Службы восстановления Microsoft Azure (MARS). MARS также известен как агент резервного копирования Azure.

## <a name="about-the-mars-agent"></a>Общие сведения об агенте MARS

Резервное копирование Azure использует агент MARS для резервного копирования файлов, папок и системного состояния с непредпомочных машин и VM Azure. Эти резервные копирования хранятся в хранилище служб восстановления в Azure. Вы можете запустить агента:

* Непосредственно на припредх Windows-машинах. Эти машины могут резервное копирование непосредственно в хранилище служб восстановления в Azure.
* На вазовных вс-чатах, которые работают над Windows бок о бок с расширением резервного копирования Azure VM. Агент резервное копирование конкретных файлов и папок на VM.
* На экземпляре сервера резервного копирования Microsoft Azure (MABS) или на сервере System Center Data Protection Manager (DPM). В этом случае машины и рабочие нагрузки возвращаются в MABS или менеджер по защите данных. Затем MABS или менеджер по защите данных используют агент MARS для резервного копирования в хранилище в Azure.

Данные, доступные для резервного копирования, зависят от того, где установлен агент.

> [!NOTE]
> Как правило, резервное копирование VM Azure с помощью расширения резервного копирования Azure на VM. Этот метод резервное копирование всего VM. Если вы хотите создать резервную папку и папки на VM, установите и используйте агент MARS вместе с расширением. Для получения дополнительной [информации см. Архитектура встроенного резервного копирования Azure VM.](backup-architecture.md#architecture-built-in-azure-vm-backup)

## <a name="modify-storage-replication"></a>Изменение репликации хранилища

По умолчанию хранилища используют [гео-избыточное хранилище (GRS).](https://docs.microsoft.com/azure/storage/common/storage-redundancy-grs)

* Если хранилище является основным механизмом резервного копирования, мы рекомендуем использовать GRS.
* Для снижения затрат на хранение Azure можно использовать [локально избыточное хранилище (LRS).](https://docs.microsoft.com/azure/storage/common/storage-redundancy-lrs?toc=%2fazure%2fstorage%2fblobs%2ftoc.json)

Чтобы изменить тип репликации хранилища:

1. В новом хранилище выберите **Свойства** в разделе **Настройки.**

1. На странице **Свойства,** под **конфигурацией резервного** **копирования,** выберите обновление .

1. Выберите тип репликации хранилища и выберите **Сохранить.**

    ![Обновление конфигурации резервного копирования](./media/backup-afs/backup-configuration.png)

> [!NOTE]
> Вы не можете изменить тип репликации хранилища после настройки хранилища и содержит элементы резервного копирования. Если вы хотите сделать это, вам нужно воссоздать хранилище.
>

### <a name="verify-internet-access"></a>Проверка доступа к Интернету

Если ваша машина имеет ограниченный доступ в Интернет, убедитесь, что настройки брандмауэра на машине или прокси позволяют следующие URL-адреса и IP-адреса:

* URL-адреса
  * `www\.msftncsi.com`
  * `*.Microsoft.com`
  * `*.WindowsAzure.com`
  * `*.microsoftonline.com`
  * `*.windows.net`
* IP-адреса
  * 20.190.128.0/18
  * 40.126.0.0/18

### <a name="use-azure-expressroute"></a>Используйте Azure ExpressRoute

Вы можете создать резервную связь с данными Azure ExpressRoute с помощью общедоступных пирингов (доступных для старых схем) и пиринга Майкрософт. Резервное копирование над приватным пирингом не поддерживается.

Чтобы использовать публичные пиринг, сначала обеспечить доступ к следующим доменам и адресам:

* `http://www.msftncsi.com/ncsi.txt`
* `microsoft.com`
* `.WindowsAzure.com`
* `.microsoftonline.com`
* `.windows.net`

Чтобы использовать пиринг Майкрософт, выберите следующие службы, регионы и соответствующие значения сообщества:

* Активный каталог Azure (12076:5060)
* Область Azure, в зависимости от местоположения хранилища служб восстановления
* Хранение Azure в зависимости от местоположения хранилища служб восстановления

Для получения дополнительной информации [см.](https://docs.microsoft.com/azure/expressroute/expressroute-routing)

> [!NOTE]
> Публичное пиринг унижается для новых схем.

Все предыдущие URL-адреса и IP-адреса используют протокол HTTPS в порту 443.

## <a name="download-the-mars-agent"></a>Скачать агент MARS

Скачать агент MARS, чтобы вы могли установить его на машины, которые вы хотите создать резервную.

Если вы уже установили агент на любых машинах, убедитесь, что вы работаете последнюю версию агента. Найти последнюю версию на портале, или перейти непосредственно к [загрузке](https://aka.ms/azurebackup_agent).

1. В хранилище, под **Getting Started**, выберите **резервное копирование.**

    ![Откройте цель резервного копирования](./media/backup-try-azure-backup-in-10-mins/open-backup-settings.png)

1. **Где работает ваша рабочая нагрузка?**, выберите **On-premises**. Выберите эту опцию, даже если вы хотите установить агент MARS на Azure VM.
1. Под **Что вы хотите создать резервную резервную часть?**, выберите **файлы и папки.** Вы также можете выбрать **Системное состояние.** Доступно много других опций, но эти параметры поддерживаются только в том случае, если вы работаете на вторичном сервере резервного копирования. Выберите **инфраструктуру подготовки.**

    ![Настройка файлов и папок](./media/backup-try-azure-backup-in-10-mins/set-file-folder.png)

1. Для **подготовки инфраструктуры,** под **агентом Установить Восстановление услуг**, скачать агент MARS.

    ![Подготовка инфраструктуры](./media/backup-try-azure-backup-in-10-mins/choose-agent-for-server-client.png)

1. В меню загрузки выберите **Сохранить**. По умолчанию файл *MARSagentinstaller.exe* сохраняется в папке для скачивания.

1. Выберите **уже скачать или с помощью последних услуг восстановления агента**, а затем скачать хранилище учетных данных.

    ![Скачивание учетных данных хранилища](./media/backup-try-azure-backup-in-10-mins/download-vault-credentials.png)

1. Нажмите кнопку **Сохранить**. Файл загружается в папку Загрузки. Вы не можете открыть файл учетных данных хранилища.

## <a name="install-and-register-the-agent"></a>Установка и регистрация агента

1. Запустите файл *MARSagentinstaller.exe* на машинах, которые вы хотите создать резервную систему.
1. В мастере настройки агента MARS выберите **настройки установки.** Там выберите, где установить агент, и выберите место для кэша. Затем выберите **Следующий**.
   * Резервное копирование Azure использует кэш для хранения моментальных снимков данных перед отправкой в Azure.
   * Местоположение кэша должно иметь свободное пространство, равное не менее 5 процентам размера данных, которые вы будете резервировать.

    ![Выберите настройки установки в мастере установки агента MARS](./media/backup-configure-vault/mars1.png)

1. Для **конфигурации прокси**укажите, как агент, который работает на машине Windows, будет подключаться к Интернету. Затем выберите **Следующий**.

   * Если вы используете пользовательский прокси-
   * Помните, что агенту необходим доступ к [определенным URL-адресам.](#before-you-start)

    ![Настройка доступа в Интернет в мастере MARS](./media/backup-configure-vault/mars2.png)

1. Для **установки**просмотрите предпосылки и выберите **Install.**
1. После установки агента выберите **Приступить к регистрации.**
1. В **Регистрационном Сервере Волшебник** > **Убежище идентификации**, просматривать и выбрать файл учетных данных, который вы загрузили. Затем выберите **Следующий**.

    ![Добавление учетных данных хранилища с помощью мастера сервера регистров](./media/backup-configure-vault/register1.png)

1. На странице **настройки шифрования** укажите пароль, который будет использоваться для шифрования и расшифровки резервных данных для машины.

    * Сохраните пароль в безопасном месте. Он необходим для восстановления резервного копирования.
    * Если вы потеряете или забудете пароль, корпорация Майкрософт не сможет помочь вам восстановить данные резервного копирования.

1. Нажмите кнопку **Готово**. Агент теперь установлен, и ваша машина зарегистрирована в хранилище. вы можете настроить параметры резервного копирования, в том числе расписание.

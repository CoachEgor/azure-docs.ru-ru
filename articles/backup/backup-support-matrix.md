# <a name="azure-backup-support-matrix"></a>Таблица поддержки Azure Backup

Можно использовать [Azure Backup](backup-overview.md) для резервного копирования данных на облачной платформе Microsoft Azure. В этой статье перечислены параметры общей поддержки и ограничения для сценариев архивации Azure и развертываний.

Доступны другие таблицы поддержки:

- Таблица поддержки для [архивации виртуальных машин (ВМ)](backup-support-matrix-iaas.md)
- Матрица поддержки для резервного копирования с помощью [System Center Data Protection Manager (DPM) и резервного копирования сервера Microsoft Azure (MABS)](backup-support-matrix-mabs-dpm.md)
- Матрица поддержки для резервного копирования с помощью [агент служб восстановления Microsoft Azure (MARS)](backup-support-matrix-mars-agent.md)

## <a name="vault-support"></a>Поддержка хранилища

Служба архивации Azure использует хранилищ служб восстановления для координации и управления резервными копиями. Он также использует хранилища для хранения резервных копий данных.

В следующей таблице описаны возможности хранилищ служб восстановления:

**Возможность** | **Сведения**
--- | ---
**Количество хранилищ в подписке** | До 500 хранилищ Служб восстановления в одной подписке.
**Количество компьютеров в хранилище** | До 1000 виртуальных машин Azure в одном хранилище.<br/><br/> В одном хранилище можно зарегистрировать не более 50 серверов MABS.
**Источник данных в хранилище** | Максимальное 54 400 ГБ. Резервные копии виртуальной машины Azure без ограничений.
**Резервные копии в хранилище** | **Виртуальные машины Azure:** Один раз в день.<br/><br/>**Машины, защищенные DPM или MABS:** Два раза в день.<br/><br/> **Резервное копирование машин непосредственно с помощью агента служб восстановления Microsoft AZURE:** Три раза в день.
**Резервное копирование данных между хранилищами** | Резервное копирование в пределах региона.<br/><br/> Вам потребуется хранилище в каждом регионе Azure, в котором есть виртуальные машины для резервного копирования. Вы не сможете создать резервные копии в другом регионе.
**Перемещение хранилищ** | Вы можете [перемещение хранилищ](https://review.docs.microsoft.com/azure/backup/backup-azure-move-recovery-services-vault) нескольких подписок или между группами ресурсов в той же подписке.
**Перемещение данных между хранилищами** | Перемещение резервных копий данных между хранилищами не поддерживается.
**Изменение типа хранилища** | Можно изменить тип репликации хранилища (геоизбыточного хранилища или локально избыточное хранилище) для хранилища, прежде чем резервные копии хранятся. После начала резервного копирования в хранилище тип репликации невозможно изменить.

## <a name="daylight-saving-time-support"></a>Поддержка летнее время

Служба архивации Azure не поддерживает часов автоматической коррекции для летнего времени для резервных копий виртуальных Машин Azure. Измените политики резервного копирования вручную при необходимости.

## <a name="disk-deduplication-support"></a>Поддержка дедупликации дисков

Поддержка дедупликации диска выглядит следующим образом:

- Дедупликация диска — поддерживаемому локальному при использовании DPM или MABs для резервного копирования виртуальных машин Hyper-V, работающих под управлением Windows. Windows Server выполняет дедупликацию данных (на уровне узла) для виртуальных жестких дисков (VHD), подключенных к виртуальной Машине хранилище для резервных копий.
- Дедупликация не поддерживается в Azure ни для одного из компонентов службы Backup. Когда DPM и MABS развертываются в Azure, нельзя выполнить дедупликацию дисков, подключенных к виртуальной Машине.

## <a name="security-and-encryption-support"></a>Поддержка безопасности и шифрования

Служба архивации Azure поддерживает шифрование передаваемых и хранящихся данных.

### <a name="network-traffic-to-azure"></a>Сетевой трафик в Azure

- Данные трафика резервного копирования с серверов в хранилище служб восстановления шифруется с помощью AES 256.
- Данные файла резервной копии передаются через защищенное соединение HTTPS.
- Данные архивации хранятся в хранилище служб восстановления в зашифрованном виде.
- Только у вас есть парольная фраза для расшифровки этих данных. Корпорация Майкрософт не может расшифровывать данные резервных копий.

    > [!WARNING]
    > После настройки хранилища только у вас есть доступ к ключу шифрования. Корпорация Майкрософт никогда не управляет копированием и не имеет доступа к ключу. При утрате ключа Майкрософт не сможет восстановить данные резервных копий.

### <a name="data-security"></a>Защита данных

- При резервном копировании виртуальных машин Azure, необходимо настроить шифрование *в* виртуальной машины.
- Служба Azure Backup поддерживает шифрование дисков Azure, которое использует BitLocker на виртуальных машинах Windows и **dm-crypt** на виртуальных машинах Linux.
- На внутреннем сервере, служба архивации Azure использует [шифрование службы хранилища Azure](../storage/common/storage-service-encryption.md), что обеспечивает защиту неактивных данных.

**Локальные компьютеры Windows без DPM или MABS** | ![Да][green] | ![Да][green]
**Виртуальные машины Azure** | ![Да][green] | ![Да][green]
**Windows локальных компьютеров или виртуальных машин Azure с помощью DPM** | ![Да][green] | ![Да][green]

## <a name="compression-support"></a>Поддержка сжатия

Резервное копирование поддерживает сжатие трафика резервного копирования, как описано в следующей таблице.

- Для виртуальных машин Azure расширение виртуальной Машины считывает данные непосредственно из учетной записи хранения Azure по сети хранения данных, поэтому нет необходимости сжатие этого трафика.
- Если вы используете DPM или MABS, можно сэкономить пропускную способность за счет сжатия данных, прежде чем его резервная копия.

**Компьютер** | **Сжатие в MABS или DPM (протокол TCP)** | **Сжать хранилище (HTTPS)**
--- | --- | ---
**Прямое резервное копирование локальных компьютеров Windows** | Нет данных | ![Да][green]
**Резервное копирование виртуальных машин Azure с помощью расширения виртуальной Машины** | Нет данных | Нет данных
**Резервное копирование на машинах в локальной среде и Azure с помощью MABS/DPM** | ![Да][green] | ![Да][green]

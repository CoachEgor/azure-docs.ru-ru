# <a name="support-matrix-for-azure-backup"></a>Матрица поддержки для резервного копирования Azure

Для резервного копирования данных на облачную платформу Microsoft Azure можно использовать [резервное копирование](backup-overview.md) данных. В этой статье кратко излагаются общие параметры и ограничения для сценариев и развертываний резервного копирования Azure.

Доступны другие таблицы поддержки:

- Матрица поддержки для [резервного копирования виртуальной машины Azure (VM)](backup-support-matrix-iaas.md)
- Матрица поддержки для резервного копирования с помощью [системного менеджера по защите данных (DPM)/Microsoft Azure Backup Server (MABS)](backup-support-matrix-mabs-dpm.md)
- Матрица поддержки для резервного копирования с помощью [агента Служб восстановления Microsoft Azure (MARS)](backup-support-matrix-mars-agent.md)

## <a name="vault-support"></a>Поддержка хранилища

Резервное копирование Azure использует хранилища служб восстановления для организации и управления резервными копированиями. Он также использует хранилища для хранения резервных данных.

В следующей таблице описаны особенности хранилищ служб восстановления:

**Компонент** | **Сведения**
--- | ---
**Количество хранилищ в подписке** | До 500 хранилищ Служб восстановления в одной подписке.
**Компьютеры в хранилище** | До 1000 VMs Azure в одном хранилище.<br/><br/> В одном хранилище можно зарегистрировать не более 50 серверов MABS.
**Источники данных** | Максимальный размер отдельного [источника данных](https://docs.microsoft.com/azure/backup/backup-azure-backup-faq#how-is-the-data-source-size-determined) составляет 54 400 ГБ. Это ограничение не распространяется на резервные ожидания Azure VM. К общему объему данных, которые можно перепрозаться в хранилище, не распространяются никаких ограничений.
**Резервные копии в хранилище** | **Azure VMs:** Раз в день.<br/><br/>**Машины, защищенные DPM/MABS:** Два раза в день.<br/><br/> **Машины резервного копирования непосредственно с помощью агента MARS:** Три раза в день.
**Резервное копирование данных между хранилищами** | Резервное копирование в пределах региона.<br/><br/> Вам потребуется хранилище в каждом регионе Azure, в котором есть виртуальные машины для резервного копирования. Вы не сможете создать резервные копии в другом регионе.
**Перемещение хранилищ** | Вы можете [перемещать хранилища](https://docs.microsoft.com/azure/backup/backup-azure-move-recovery-services-vault) по подписке или между группами ресурсов в одной подписке. Однако перемещение хранилищ по регионам не поддерживается.
**Перемещение данных между хранилищами** | Перемещение резервных данных между хранилищами не поддерживается.
**Изменение типа хранилища** | Можно изменить тип репликации хранилища (либо гео-избыточное хранилище, либо локально избыточное хранилище) для хранилища до хранения резервных копий. После начала резервного копирования в хранилище тип репликации невозможно изменить.

## <a name="daylight-saving-time-support"></a>Поддержка летнего времени

Резервное копирование Azure не поддерживает автоматическую регулировку часов для летнего времени для резервного копирования Azure VM. Он не сдвигает час резервного копирования вперед или назад. Чтобы обеспечить запуск резервного копирования в нужное время, изменяй политики резервного копирования вручную по мере необходимости.

## <a name="disk-deduplication-support"></a>Поддержка дедупликации дисков

Поддержка дедупликации диска выглядит следующим образом:

- Дисковое deduplication поддерживается на месте при использовании DPM или MABs для резервного копирования Hyper-V VMs, которые работают под управлением Windows. Windows Server выполняет deduplication данных (на уровне хоста) на виртуальных жестких дисках (VHD), которые прикрепляются к VM в качестве резервного хранилища.
- Дедупликация не поддерживается в Azure ни для одного из компонентов службы Backup. При развертывании DPM и MABS в Azure диски хранения, прикрепленные к VM, не могут быть разложены.

## <a name="security-and-encryption-support"></a>Поддержка безопасности и шифрования

Резервное копирование Azure поддерживает шифрование данных в пути и на отдыхе.

### <a name="network-traffic-to-azure"></a>Сетевой трафик в Azure

- Резервное копирование трафика с серверов в хранилище служб восстановления шифруется с помощью Advanced Encryption Standard 256.
- Данные файла резервной копии передаются через защищенное соединение HTTPS.
- Данные резервного копирования хранятся в хранилище служб восстановления в зашифрованном виде.
- Только у вас есть парольная фраза для расшифровки этих данных. Корпорация Майкрософт не может расшифровывать данные резервных копий.

    > [!WARNING]
    > После настройки хранилища только у вас есть доступ к ключу шифрования. Корпорация Майкрософт никогда не управляет копированием и не имеет доступа к ключу. При утрате ключа Майкрософт не сможет восстановить данные резервных копий.

### <a name="data-security"></a>Безопасность данных

- При резервном копировании виртуальных технологий Azure необходимо настроить шифрование *в* виртуальной машине.
- Служба Azure Backup поддерживает шифрование дисков Azure, которое использует BitLocker на виртуальных машинах Windows и **dm-crypt** на виртуальных машинах Linux.
- На задней части резервное копирование Azure использует [шифрование службы хранения Azure,](../storage/common/storage-service-encryption.md)которое защищает данные в состоянии покоя.

**Компьютер** | **При передаче** | **При хранении**
--- | --- | ---
**Локальные компьютеры Windows без DPM или MABS** | ![Да][green] | ![Да][green]
**Виртуальные машины Azure** | ![Да][green] | ![Да][green]
**На-предприимченные windows-машины или VMs Azure с DPM** | ![Да][green] | ![Да][green]
**На-предприимченные windows-машины или VMs Azure с MABS** | ![Да][green] | ![Да][green]

## <a name="compression-support"></a>Поддержка сжатия

Резервное копирование поддерживает сжатие трафика резервного копирования, как обобщено в следующей таблице.

- Для VMs-м всуза расширение VM считывает данные непосредственно из учетной записи хранения Данных Azure по сети хранения данных, поэтому нет необходимости сжимать этот трафик.
- Если вы используете DPM или MABS, вы можете сохранить пропускную способность, сжав данные до их резервного копирования.

**Компьютер** | **Сжатие в MABS или DPM (протокол TCP)** | **Сжатие к хранилищу (HTTPS)**
--- | --- | ---

**Резервное копирование на наемных/лазурных машинах с помощью MABS/DPM** | ![Да][green] | ![Да][green]

## <a name="cross-region-restore"></a>Восстановление Крестовского района

| ---------------------- | ------------------------------------------------------------ | ----------------- |
| Azure               | Да.   Поддерживается для зашифрованных VM и VMs с менее чем 4-ТБ дисков | Все общественные регионы Azure.  |
| Агент MARS/На территории | нет                                                           | Недоступно               |

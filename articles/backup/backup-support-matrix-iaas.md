# <a name="support-matrix-for-azure-vm-backup"></a>Матрица поддержки для резервного копирования виртуальных машин Azure

Вы можете использовать [службу Azure Backup](backup-overview.md) для резервного копирования локальных компьютеров и рабочих нагрузок, а также виртуальных машин Azure. В этой статье перечислены параметры поддержки и ограничения при резервном копировании виртуальных машин Azure с помощью Azure Backup.

Другие матрицы поддержки:

- [Общая таблица поддержки](backup-support-matrix.md) для Azure Backup
- [Матрица поддержки](backup-support-matrix-mabs-dpm.md) для резервного копирования Azure Backup Server/System Center Data Protection Manager (DPM)
- [Таблица поддержки](backup-support-matrix-mars-agent.md) для резервного копирования с помощью агента службы восстановления Microsoft Azure (MARS)

## <a name="supported-scenarios"></a>Поддерживаемые сценарии использования.

Вот как вы можете выполнить резервное копирование и восстановление виртуальных машин с помощью службы Azure Backup.

**Сценарий** | **Архивация** | **Agent** |**Восстановление**
--- | --- | --- | ---
Прямое резервное копирование виртуальных машин Azure  | Создайте резервную копию всей виртуальной машины.  | Агент на виртуальной машине Azure не требуется. Azure Backup устанавливает и использует расширение [агента виртуальной машины Azure](https://docs.microsoft.com/azure/virtual-machines/extensions/agent-windows) , работающего на виртуальной машине. | Восстановление состоит из следующих шагов.<br/><br/> - Создание базовой виртуальной машины **. Это полезно, если виртуальная машина не имеет специальной конфигурации, такой как несколько IP-адресов.<br/><br/> - Восстановление диска виртуальной машины **. Восстановите диск. Затем подключите его к существующей виртуальной машине или создайте новую виртуальную машину с диска с помощью PowerShell.<br/><br/> - Замена диска виртуальной машины **. Если виртуальная машина существует и использует управляемые диски (без шифрования), вы можете восстановить диск и заменить ним существующий диск на виртуальной машине.<br/><br/> - Восстановление определенных файлов или папок **. Непосредственная Архивация виртуальных машин Azure (только для Windows)  | Резервное копирование конкретных файлов, папок или томов. | Установите [агент служб восстановления Azure](backup-azure-file-folder-backup-faq.md).<br/><br/> Агент MARS можно запустить вместе с расширением резервного копирования для агента виртуальной машины Azure, чтобы выполнить резервное копирование виртуальной машины на уровне файла или папки. | Восстановление определенных папок или файлов.
Резервное копирование виртуальной машины Azure на резервный сервер  | Резервное копирование файлов, папок или томов; файлы состояния системы или исходное состояние; Данные приложения в System Center DPM или на сервер Microsoft Azure Backup (MABS).<br/><br/> Затем DPM/MABS создает резервную копию в хранилище службы архивации. | Установите агент защиты DPM/MABS на виртуальной машине. Агент MARS установлен на DPM или MABS.| Восстановление файлов, папок и томов; файлов состояния системы и исходного состояния; данных приложений.

Узнайте больше о резервном копировании [с помощью сервера архивации](backup-architecture.md#architecture-back-up-to-dpmmabs) и о [требованиях к поддержке](backup-support-matrix-mabs-dpm.md).

## <a name="supported-backup-actions"></a>Поддерживаемые действия резервного копирования

**Действие** | **Поддержка**

Включение резервного копирования при создании виртуальной машины Linux | Поддерживается для:<br/><br/> -Ubuntu Server: 18,04, 17,10, 17,04, 16,04 (LTS), 14,04 (LTS)<br/><br/> -Red Hat: RHEL 6,7, 6,8, 6,9, 7,2, 7,3, 7,4<br/><br/> -SUSE Linux Enterprise Server: 11 SP4, 12 SP2, 12 SP3, 15 <br/><br/> -Debian: 8, 9<br/><br/> -CentOS: 6,9, 7,3<br/><br/> -Oracle Linux: 6,7, 6,8, 6,9, 7,2, 7,3
Резервное копирование ВИРТУАЛЬНОЙ машины, которая находится в состоянии завершения работы или автономного режима | Поддерживается.<br/><br/> Можно создать только отказоустойчивый моментальный снимок, но не согласованный на уровне приложений.
Резервное копирование дисков после миграции на управляемые диски | Поддерживается.<br/><br/> Резервное копирование продолжит работу. Никаких действий не требуется.
Резервное копирование управляемых дисков после включения блокировки группы ресурсов | Не поддерживается.<br/><br/> Azure Backup не удается удалить более старые точки восстановления, поэтому резервное копирование начнется неудачей при достижении максимального предела точек восстановления.
Изменение политики резервного копирования для виртуальной машины | Поддерживается.<br/><br/> Для виртуальной машины будет создана резервная копия с использованием параметров расписание и хранение в новой политике. Если срок хранения увеличен, существующие точки восстановления будут помечены и сохранены. Если они уменьшаются, существующие точки восстановления будут удалены в следующем задании очистки и в конечном итоге удаляются.
Отмена задания резервного копирования| Поддерживается во время процесса создания моментального снимка.<br/><br/> Не поддерживается, если моментальный снимок переносится в хранилище.
Резервное копирование виртуальной машины в другой регион или подписку |Не поддерживается.
Операций резервного копирования в день (с использованием расширения виртуальных машин Azure) | Одно запланированное резервное копирование в день.<br/><br/> Служба позволяет использовать до девяти резервных копий по запросу в день, но корпорация Майкрософт рекомендует и поддерживает до четырех ежедневных резервных копий по запросу.
Операций резервного копирования в день (с использованием агента MARS) | Три запланированных операции резервного копирования в день.
Операций резервного копирования в день (с использованием DPM или MABS) | Две запланированных операции резервного копирования в день.
Ежемесячное или ежегодное резервное копирование| Не поддерживается при создании резервных копий с использованием расширения виртуальной машины Azure. Поддерживается только ежедневное и еженедельное.<br/><br/> Вы можете настроить политику, чтобы хранить ежедневные или еженедельные резервные копии в течение месяца или года.
Автоматическая настройка часов | Не поддерживается.<br/><br/> Azure Backup не выполняет автоматическое изменение перехода на летнее время при резервном копировании виртуальной машины.<br/><br/>  Измените политику вручную, если это необходимо.
[Функции безопасности для гибридных резервных копий](https://docs.microsoft.com/azure/backup/backup-azure-security-feature) |Отключение функций безопасности не поддерживается.
Создание резервной копии виртуальной машины, время машины которой изменилось | Не поддерживается.<br/><br/> Если время компьютера изменилось на будущее, после включения резервного копирования для этой виртуальной машины; Однако даже если изменение времени отменено, успешная архивация не гарантируется.

## <a name="operating-system-support-linux"></a>Поддержка операционной системы (Linux)

Для резервного копирования виртуальных машин Azure под управлением Linux служба Azure Backup поддерживает весь [список дистрибутивов Linux, рекомендуемых для использования в Azure](https://docs.microsoft.com/azure/virtual-machines/linux/endorsed-distros). Обратите внимание на следующее:

- Azure Backup не поддерживает CoreOS Linux.
- Azure Backup не поддерживает 32-разрядные операционные системы.
- Другие собственные дистрибутивы Linux могут работать при условии, что [Агент виртуальной машины Azure для Linux](https://docs.microsoft.com/azure/virtual-machines/extensions/agent-linux) доступен на виртуальной машине, пока поддерживается Python.
- Azure Backup не поддерживает настроенную прокси-сервер Linux, если на нем не установлена версия Python 2,7.

## <a name="backup-frequency-and-retention"></a>Частота резервного копирования и период хранения

**Параметр** | **Ограничения**
--- | ---
Максимальное количество точек восстановления на защищенный экземпляр (компьютер или рабочая нагрузка) | 9999.
Максимальное время окончания срока действия для точки восстановления | Без ограничений.
Максимальная частота резервного копирования в хранилище (расширение виртуальной машины Azure) | Раз в день.
Максимальная частота резервного копирования в хранилище (агент MARS) | Три операции резервного копирования в день.
Максимальная частота резервного копирования в DPM или MABS | Каждые 15 минут для SQL Server.<br/><br/> Один раз в час для других рабочих нагрузок.
Период удержания точки восстановления | Ежедневно, еженедельно, ежемесячно и ежегодно.
Максимальный срок хранения | Зависит от частоты резервного копирования.
Точки восстановления на диске DPM или MABS | 64 для файловых серверов и 448 для серверов приложений.<br/><br/> Точки восстановления ленты не имеют ограничений для локального DPM.

## <a name="supported-restore-methods"></a>Поддерживаемые методы восстановления

**Метод восстановления** | **Дополнительные сведения**
--- | ---
Создание виртуальной машины | Виртуальную машину можно создать во время восстановления. <br/><br/> Выбрав этот вариант, вы получите базовую виртуальную машину, запущенную и готовую к работе. Вы можете указать имя виртуальной машины, группу ресурсов, виртуальную сеть, подсеть и хранилище.  
Восстановление диска | Вы можете восстановить диск и использовать его для создания виртуальной машины.<br/><br/> Если вы выберите этот вариант, Azure Backup копирует данные из хранилища в выбранную учетную запись хранения. Задание восстановления создает шаблон. Вы можете скачать этот шаблон, использовать его для указания пользовательских параметров виртуальной машины и создать виртуальную машину.<br/><br/> Этот вариант позволяет указать больше параметров для создаваемой виртуальной машины, чем предыдущий вариант.<br/><br/>
Замена существующего диска | Вы можете восстановить диск, а затем использовать его для замены диска, который в настоящий момент используется на виртуальной машине.
Восстановить файлы | Вы можете восстановить файлы из выбранной точки восстановления. Вы скачаете сценарий для подключения диска виртуальной машины из точки восстановления. Затем просмотрите тома дисков, чтобы найти файлы и папки, которые необходимо восстановить, и отключите диск по завершении.

См. [таблицу совместимой ОС](backup-azure-restore-files-from-vm.md#system-requirements).

Восстановление файлов со специальными сетевыми параметрами | Восстановление на той же виртуальной машине не поддерживается. <br/><br/> Выполните восстановление на совместимой виртуальной машине.

## <a name="support-for-vm-management"></a>Поддержка управления виртуальными машинами

В следующей таблице приведены сведения о поддержке резервного копирования во время выполнения задач управления виртуальной машиной, таких как добавление или замена дисков виртуальной машины.

**Восстановление** | **Поддерживаются**
--- | ---
Восстановление между подписками, регионами или зонами | Не поддерживается.
Восстановление на существующей виртуальной машине | Используйте вариант замены диска.
Восстановление диска с помощью учетной записи хранения, в которой включено Шифрование службы хранилища Azure | Не поддерживается.<br/><br/> Выполните восстановление в учетной записи, в которой отключено SSE.
Восстановление в смешанных учетных записях хранения |Не поддерживается.<br/><br/> В зависимости от типа учетной записи хранения все восстановленные диски будут класса Premium или Standard, но не смешанными.
Восстановление виртуальной машины напрямую в группе доступности | Для управляемых дисков можно восстановить диск и использовать параметр группы доступности в шаблоне.<br/><br/> Эта возможность недоступна для неуправляемых дисков. Что касается неуправляемых дисков, восстановите диск, а затем создайте виртуальную машину в группе доступности.
Восстановление резервной копии неуправляемых виртуальных машин после обновления до управляемой виртуальной машины| Поддерживается.<br/><br/> Вы можете восстановить диски, а затем создать управляемую виртуальную машину.
Восстановление виртуальной машины до точки восстановления до переноса виртуальной машины на управляемые диски | Поддерживается.<br/><br/> Нужно выполнить восстановление на неуправляемых дисках (по умолчанию), преобразовать восстановленные диски в управляемые диски и создать виртуальную машину с управляемыми дисками.
Восстановление удаленной виртуальной машины | Поддерживается.<br/><br/> Вы можете восстановить виртуальную машину из точки восстановления.
Восстановление виртуальной машины контроллера домена, которая входит в конфигурацию с несколькими контроллерами домена, с помощью портала | Поддерживается, если вы восстанавливаете диск и создаете виртуальную машину с помощью PowerShell.
Восстановление виртуальной машины в другой виртуальной сети |Поддерживается.<br/><br/> Виртуальная сеть должна находиться в той же подписке и регионе.

## <a name="vm-compute-support"></a>Поддержка вычислений виртуальной машины

## <a name="vm-security-and-encryption-support"></a>Поддержка безопасности и шифрования виртуальных машин

Azure Backup поддерживает шифрование передаваемых и хранящихся данных.

Сетевой трафик в Azure:

- Трафик резервного копирования с серверов в хранилище служб восстановления шифруется с помощью AES 256.
- Данные файла резервной копии передаются через защищенное соединение HTTPS.
- Данные файла резервной копии также хранятся в зашифрованном виде в хранилище Служб восстановления.
- Только у вас есть парольная фраза для расшифровки этих данных. Корпорация Майкрософт не может расшифровывать данные резервных копий.

  > [!WARNING]
  > После настройки хранилища вы получите доступ только к ключу шифрования. Корпорация Майкрософт никогда не управляет копированием и не имеет доступа к ключу. При утрате ключа Майкрософт не сможет восстановить данные резервных копий.

Защита данных:

- При резервном копировании виртуальных машин Azure необходимо настроить шифрование *в* виртуальной машине.
- Azure Backup поддерживает шифрование дисков Azure, которое использует BitLocker на виртуальных машинах Windows и **DM-** Encryption на виртуальных машинах Linux.
- В серверной части Azure Backup использует [шифрование службы хранилища Azure](../storage/common/storage-service-encryption.md), что обеспечивает защиту неактивных данных.

# <a name="support-matrix-for-azure-vm-backup"></a>Матрица поддержки для резервного копирования виртуальных машин Azure

Службу [резервного копирования Azure](backup-overview.md) можно использовать для резервного копирования на месте машины и рабочие нагрузки, а также виртуальные машины Azure (VM). В этой статье кратко излагаются параметры и ограничения поддержки при резервном копировании VMs Azure с помощью резервного копирования Azure.

Другие матрицы поддержки:

- [Общая матрица поддержки](backup-support-matrix.md) резервного копирования Azure
- [Матрица поддержки](backup-support-matrix-mabs-dpm.md) для резервного копирования сервера Azure/System Center Data Protection Manager (DPM) резервного копирования
- [Матрица поддержки](backup-support-matrix-mars-agent.md) для резервного копирования с агентом Microsoft Azure Recovery Services (MARS)

Вот как вы можете выполнить резервное копирование и восстановление виртуальных машин с помощью службы Azure Backup.

**Сценарий** | **Резервного копирования** | **Агент** |**Восстановление**

Резервное копирование Azure устанавливает и использует расширение для [агента Azure VM,](https://docs.microsoft.com/azure/virtual-machines/extensions/agent-windows) работая на VM. | Восстановление состоит из следующих шагов.<br/><br/> - **Создайте базовый VM**. Это полезно, если VM не имеет специальной конфигурации, такой как несколько IP-адресов.<br/><br/> **Восстановление диска виртуальной машины**. Восстановите диск. Затем прикрепите его к существующему VM или создайте новый VM с диска с помощью PowerShell.<br/><br/> **Замена диска виртуальной машины**. Если виртуальная машина существует и использует управляемые диски (без шифрования), вы можете восстановить диск и заменить ним существующий диск на виртуальной машине.<br/><br/> **Восстановление определенных файлов или папок**. Подробнее о резервном копировании [с помощью сервера резервного копирования](backup-architecture.md#architecture-back-up-to-dpmmabs) и [о требованиях к поддержке.](backup-support-matrix-mabs-dpm.md)

>[!NOTE]
> **Резервное копирование Azure теперь поддерживает выборочное резервное копирование диска и восстановление с помощью решения резервного копирования виртуальной машины Azure.**
>
>Сегодня Резервное копирование Azure поддерживает резервное копирование всех дисков (операционная система и данные) в VM вместе с помощью решения резервного копирования виртуальной машины. С функцией исключения диска вы получаете возможность резервного копирования одного или нескольких из множества дисков данных в VM. Это обеспечивает эффективное и экономное решение для резервного копирования и восстановления потребностей. Каждая точка восстановления содержит данные дисков, включенных в операцию резервного копирования, что позволяет дополнительно восстановить подмножество дисков из данной точки восстановления во время операции восстановления. Это относится к восстановлению как из снимка, так и из хранилища.
>

## <a name="supported-backup-actions"></a>Поддерживаемые действия резервного копирования

**Действие** | **Поддержка**
--- | ---
Резервное копирование VM, который выключения / автономной VM | Поддерживается.<br/><br/> Можно создать только отказоустойчивый моментальный снимок, но не согласованный на уровне приложений.
Резервное копирование дисков после перехода на управляемые диски | Поддерживается.<br/><br/> Резервное копирование продолжит работу. Никаких действий не требуется.
Резервное копирование управляемых дисков после включения блокировки группы ресурсов | Не поддерживается.<br/><br/> Резервное копирование Azure не может удалить старые точки восстановления, и резервные ожидания начнут выходить из строя при максимальном лимите точек восстановления.
Изменение политики резервного копирования для виртуальной машины | Поддерживается.<br/><br/> VM будет подкреплен с помощью настройки расписания и удержания в новой политике. Если срок хранения увеличен, существующие точки восстановления будут помечены и сохранены. Если они будут сокращены, существующие точки восстановления будут сокращены в следующем задания очистки и в конечном итоге удалены.
Отмена задания резервного копирования| Поддерживается во время процесса создания моментального снимка.<br/><br/> Не поддерживается, если моментальный снимок переносится в хранилище.
Резервное копирование виртуальной машины в другой регион или подписку |Не поддерживается.
Операций резервного копирования в день (с использованием расширения виртуальных машин Azure) | Одно запланированное резервное копирование в день.<br/><br/>Служба резервного копирования Azure поддерживает до девяти резервных копирований по требованию в день, но корпорация Майкрософт рекомендует не более четырех ежедневных резервных отработок по требованию, чтобы обеспечить лучшую производительность.
Операций резервного копирования в день (с использованием агента MARS) | Три запланированных операции резервного копирования в день.
Операций резервного копирования в день (с использованием DPM или MABS) | Две запланированных операции резервного копирования в день.
Ежемесячное или ежегодное резервное копирование| Не поддерживается при создании резервных копий с использованием расширения виртуальной машины Azure. Поддерживается только ежедневное и еженедельное.<br/><br/> Вы можете настроить политику, чтобы хранить ежедневные или еженедельные резервные копии в течение месяца или года.
Автоматическая настройка часов | Не поддерживается.<br/><br/> Резервное копирование Azure не настраивается автоматически на изменение времени летнего времени при резервном копировании VM.<br/><br/>  Измените политику вручную, если это необходимо.
[Функции безопасности для гибридных резервных копий](https://docs.microsoft.com/azure/backup/backup-azure-security-feature) |Отключение функций безопасности не поддерживается.
Резервное копирование VM, время машины которого изменено | Не поддерживается.<br/><br/> Если время машины изменено на будущее время даты после включения резервного копирования для этого VM; Однако, даже если изменение времени возвращается, успешное резервное копирование не гарантируется.

## <a name="operating-system-support-linux"></a>Поддержка операционной системы (Linux)

Для резервного копирования виртуальных машин Azure под управлением Linux служба Azure Backup поддерживает весь [список дистрибутивов Linux, рекомендуемых для использования в Azure](https://docs.microsoft.com/azure/virtual-machines/linux/endorsed-distros). Следует отметить следующее.

- Azure Backup не поддерживает CoreOS Linux.
- Azure Backup не поддерживает 32-разрядные операционные системы.
- Другие собственные дистрибутивы Linux могут работать до тех пор, пока на VM будет доступен [агент Azure VM для Linux,](https://docs.microsoft.com/azure/virtual-machines/extensions/agent-linux) и до тех пор, пока поддерживается Python.
- Резервное копирование Azure не поддерживает настроенный на прокси Linux VM, если у него нет установленной версии Python 2.7.

Максимальное время окончания срока действия для точки восстановления | Нет предела.
Максимальная частота резервного копирования в хранилище (расширение виртуальной машины Azure) | Раз в день.
Максимальная частота резервного копирования в хранилище (агент MARS) | Три операции резервного копирования в день.
Максимальная частота резервного копирования в DPM или MABS | Каждые 15 минут для сервера S'L.<br/><br/> Один раз в час для других рабочих нагрузок.
Период удержания точки восстановления | Ежедневно, еженедельно, ежемесячно и ежегодно.
Максимальный срок хранения | Зависит от частоты резервного копирования.
Точки восстановления на диске DPM или MABS | 64 для файловых серверов и 448 для серверов приложений.<br/><br/> Точки восстановления ленты не имеют ограничений для локального DPM.

## <a name="supported-restore-methods"></a>Поддерживаемые методы восстановления

**Вариант восстановления** | **Подробно**
--- | ---
**Создание виртуальной машины** | Быстрое создание и запуск базовой виртуальной машины из точки восстановления.<br/><br/> Вы можете указать имя для VM, выбрать группу ресурсов и виртуальную сеть (VNet), в которой он будет размещен, и указать учетную запись хранения для восстановленного VM. Новый VM должен быть создан в том же регионе, что и источник VM.
**Восстановление диска** | Восстанавливает VM-диск, который затем может быть использован для создания нового VM.<br/><br/> Azure Backup предоставляет шаблон для настройки и создания виртуальной машины. <br/><br> Задание восстановления генерирует шаблон, который можно скачать и использовать, чтобы указать пользовательские параметры и создать виртуальную машину.<br/><br/> Диски копируются в группу ресурсов, указанную вами.<br/><br/> Кроме того, можно подключить диск к существующей виртуальной машине или создать новую виртуальную машину с помощью PowerShell.<br/><br/> Этот вариант полезен, если требуется настроить виртуальную машину, добавить параметры конфигурации, которых не было во время резервного копирования, или добавить параметры, которые необходимо настроить с помощью шаблона или PowerShell.
**Замена существующей** | Можно восстановить диск и заменить им диск на существующей виртуальной машине.<br/><br/> Текущая виртуальная машина должна существовать. Если он удален, эта опция не может быть использована.<br/><br/> Перед заменой диска резервное копирование Azure делает снимок существующего VM и хранит его в указанном вами месте постановки. Существующие диски, подключенные к VM, заменяются выбранной точкой восстановления.<br/><br/> Снимок скопирован в хранилище и сохраняется в соответствии с политикой удержания. <br/><br/> После операции замены диска исходный диск сохраняется в группе ресурсов. Вы можете удалить исходные диски вручную, если они не нужны. <br/><br/>Замена существующей виртуальной машины поддерживается для незашифрованных управляемых виртуальных машин. Она не поддерживается для неуправляемых дисков, [общих виртуальных машин](https://docs.microsoft.com/azure/virtual-machines/windows/capture-image-resource) или для виртуальных машин, [созданных с помощью пользовательских образов](https://azure.microsoft.com/resources/videos/create-a-custom-virtual-machine-image-in-azure-resource-manager-with-powershell/).<br/><br/> Если точка восстановления содержит больше или меньше дисков, чем текущая виртуальная машина, то в конфигурации виртуальной машины будет отражено только количество дисков в точке восстановления.<br><br> Замена существующих не поддерживается для VMs с связанными ресурсами [(например, назначенных пользователем управляемой идентификацией](https://docs.microsoft.com/azure/active-directory/managed-identities-azure-resources/overview) или [Key Vault),](https://docs.microsoft.com/azure/key-vault/key-vault-overview)поскольку клиент-приложение резервного копирования не имеет разрешений на эти ресурсы при выполнении восстановления.
**Крестный регион (вторичный регион)** | Восстановление Cross Region может быть использовано для восстановления VMs Azure во вторичном регионе, который является [парным регионом Azure.](https://docs.microsoft.com/azure/best-practices-availability-paired-regions#what-are-paired-regions)<br><br> Можно восстановить все ми-м носы Azure для выбранной точки восстановления, если резервное копирование выполнено во вторичном регионе.<br><br> Эта функция доступна для вариантов ниже:<br> * [Создание VM](https://docs.microsoft.com/azure/backup/backup-azure-arm-restore-vms#create-a-vm) <br> * [Восстановление дисков](https://docs.microsoft.com/azure/backup/backup-azure-arm-restore-vms#restore-disks) <br><br> В настоящее время мы не поддерживаем опцию [«Заменить существующие диски».](https://docs.microsoft.com/azure/backup/backup-azure-arm-restore-vms#replace-existing-disks)<br><br> Разрешения<br> Операция восстановления на вторичном регионе может выполняться резервными админами и админами приложений.

## <a name="support-for-file-level-restore"></a>Поддержка восстановления на уровне файлов

**Восстановление** | **Поддерживается**
--- | ---
Восстановление файлов во всех операционных системах | Вы можете восстановить файлы на любом компьютере, операционная система на котором совпадает или совместима с операционной системой виртуальной машины, для которой выполнено резервное копирование. Посмотреть [таблицу совместимой ОС](backup-azure-restore-files-from-vm.md#system-requirements).
Восстановление файлов на классических виртуальных машинах | Не поддерживается.
Восстановление файлов из зашифрованных виртуальных машин | Не поддерживается.
Восстановление файлов из учетных записей хранения с ограниченным доступом из сети | Не поддерживается.
Восстановление файлов на виртуальных машинах с помощью дисковых пространств Windows | Восстановление на той же виртуальной машине не поддерживается.<br/><br/> Вместо этого нужно восстановить файлы на совместимой виртуальной машине.

## <a name="vm-security-and-encryption-support"></a>Поддержка безопасности и шифрования VM

Azure Backup поддерживает шифрование передаваемых и хранящихся данных.

Сетевой трафик в Azure:

- Резервное копирование трафика с серверов в хранилище служб восстановления шифруется с помощью Advanced Encryption Standard 256.
- Данные файла резервной копии передаются через защищенное соединение HTTPS.
- Данные файла резервной копии также хранятся в зашифрованном виде в хранилище Служб восстановления.
- Только у вас есть парольная фраза для расшифровки этих данных. Корпорация Майкрософт не может расшифровывать данные резервных копий.

  > [!WARNING]
  > После настройки хранилища только у вас есть доступ к ключу шифрования. Корпорация Майкрософт никогда не управляет копированием и не имеет доступа к ключу. При утрате ключа Майкрософт не сможет восстановить данные резервных копий.

Защита данных:

- При резервном копировании Виртуальных Виртуальных ММ необходимо настроить шифрование *в* виртуальной машине.
- Azure Backup поддерживает шифрование Azure Disk, которое использует BitLocker на виртуальных компьютерах Windows и **dm-crypt** на виртуальных машинах Linux.
- В серверной части Azure Backup использует [шифрование службы хранилища Azure](../storage/common/storage-service-encryption.md), что обеспечивает защиту неактивных данных.

Локальные компьютеры Windows без DPM или MABS | ![Да][green] | ![Да][green]
Виртуальные машины Azure | ![Да][green] | ![Да][green]

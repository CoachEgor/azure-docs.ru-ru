# <a name="support-matrix-for-azure-vm-backup"></a>Матрица поддержки для резервного копирования виртуальных машин Azure
Можно использовать [службы архивации Azure](backup-overview.md) для резервного копирования локальных компьютеров и рабочих нагрузок и виртуальных машин (ВМ) Azure. В этой статье перечислены параметры поддержки и ограничения при резервном копировании виртуальных машин Azure с помощью службы архивации Azure.

Другие матрицы поддержки:

- [Матрица поддержки Общие](backup-support-matrix.md) для резервного копирования Azure
- [Матрица поддержки](backup-support-matrix-mabs-dpm.md) для Azure Backup server и System Center Data Protection Manager (DPM) создать резервную копию
- [Матрица поддержки](backup-support-matrix-mars-agent.md) для резервного копирования с агентом служб восстановления Microsoft Azure (MARS)

Вот как вы можете выполнить резервное копирование и восстановление виртуальных машин с помощью службы Azure Backup.

--- | --- | --- | ---

Это полезно, если никакой специальной настройки, такие как несколько IP-адресов к виртуальной Машине.<br/><br/> - **Восстановление диска виртуальной машины**. Восстановите диск. Затем присоединить его к существующей виртуальной Машины или создать новую виртуальную Машину с диска с помощью PowerShell.<br/><br/> - **Замена диска виртуальной машины**. ## <a name="supported-backup-actions"></a>Поддерживаемые действия резервного копирования

**Действие** | **Поддержка**
--- | ---
Включение резервного копирования при создании виртуальной машины Azure Windows | Поддерживается для:  Windows Server 2019 (ядро центра обработки данных или Datacenter), Windows Server 2016 (ядро центра обработки данных или Datacenter); Windows Server 2012 R2 Datacenter; Windows Server 2008 R2 (RTM и SP1)
Включение резервного копирования при создании виртуальной машины Linux | Поддерживается для:<br/><br/> — Ubuntu Server: 1710, 1704, 1604 (LTS), 1404 (LTS);<br/><br/> — Red Hat: RHEL 6.7, 6.8, 6.9, 7.2, 7.3, 7.4;<br/><br/> — SUSE Linux Enterprise Server: 11 SP4, 12 SP2, 12 SP3, 15 <br/><br/> — Debian: 8, 9<br/><br/> — CentOS: 6.9, 7.3;<br/><br/> -Oracle Linux: 6.7, 6.8, 6.9, 7.2, 7.3.
Резервное копирование виртуальной Машины shutdown/автономной виртуальной Машины |  Поддерживается.<br/><br/> Можно создать только отказоустойчивый моментальный снимок, но не согласованный на уровне приложений.
Резервное копирование дисков после перехода на управляемые диски |  Поддерживается.<br/><br/> Резервное копирование продолжит работу. Никаких действий не требуется.
Резервное копирование управляемых дисков после включения блокировки группы ресурсов | Не поддерживается.<br/><br/> Служба архивации Azure не удается удалить устаревшие точки ресурсов и резервных копий начинается переход на другой при достижении предельного числа точек восстановления.
Изменение политики резервного копирования для виртуальной машины |  Поддерживается.<br/><br/> Виртуальная машина будет резервного копирования, используя параметры расписания и хранения в новую политику. Если срок хранения увеличен, существующие точки восстановления будут помечены и сохранены. Если они длительному, существующие точки восстановления удаляются при следующем задании очистки и удаляется.
Отмена задания резервного копирования | Поддерживается во время процесса создания моментального снимка.<br/><br/> Не поддерживается, если моментальный снимок переносится в хранилище.
Резервное копирование виртуальной машины в другой регион или подписку |  Не поддерживается.
Операций резервного копирования в день (с использованием расширения виртуальных машин Azure) | Одно запланированное резервное копирование в день.<br/><br/> Можно сделать до четырех резервных копий по запросу в день.
Операций резервного копирования в день (с использованием агента MARS) | Три запланированных операции резервного копирования в день.
Операций резервного копирования в день (с использованием DPM или MABS) | Две запланированных операции резервного копирования в день.
Ежемесячное или ежегодное резервное копирование   | Не поддерживается при создании резервных копий с использованием расширения виртуальной машины Azure. Поддерживается только ежедневное и еженедельное.<br/><br/> Вы можете настроить политику, чтобы хранить ежедневные или еженедельные резервные копии в течение месяца или года.
Автоматическая настройка часов | Не поддерживается.<br/><br/> Служба архивации Azure не изменяются автоматически перехода на летнее время, при резервном копировании виртуальной Машины.<br/><br/>  Измените политику вручную, если это необходимо.
[Функции безопасности для гибридных резервных копий](https://docs.microsoft.com/azure/backup/backup-azure-security-feature) |  Отключение функции безопасности не поддерживается.

## <a name="operating-system-support-linux"></a>Поддержка операционной системы (Linux)

Для резервного копирования виртуальных машин Azure под управлением Linux служба Azure Backup поддерживает весь [список дистрибутивов Linux, рекомендуемых для использования в Azure](https://docs.microsoft.com/azure/virtual-machines/linux/endorsed-distros). Обратите внимание на следующее.

- Azure Backup не поддерживает CoreOS Linux.
- Azure Backup не поддерживает 32-разрядные операционные системы.
- Другие дистрибутивы Linux перевести собственной может работать, пока [агент виртуальной Машины Azure для Linux](https://docs.microsoft.com/azure/virtual-machines/extensions/agent-linux) доступен на виртуальной Машине, и, если поддерживается Python.
- Azure Backup не поддерживает виртуальной Машины Linux настроен прокси-сервера, если он не поддерживает Python версии 2.7 установлен.


## <a name="backup-frequency-and-retention"></a>Частота резервного копирования и период хранения

**Параметр** | **Ограничения**
--- | ---
Максимальное количество точек восстановления на защищенный экземпляр (или рабочей нагрузке машины) | 9999.
Максимальное время окончания срока действия для точки восстановления | Нет ограничений.
Максимальная частота резервного копирования в хранилище (расширение виртуальной машины Azure) | Один раз в день.
Максимальная частота резервного копирования в хранилище (агент MARS) | Три операции резервного копирования в день.
Максимальная частота резервного копирования в DPM или MABS | Каждые 15 минут для SQL Server.<br/><br/> Один раз в час для других рабочих нагрузок.
Период удержания точки восстановления | Ежедневно, еженедельно, ежемесячно, "и" ежегодно.
Максимальный срок хранения | Зависит от частоты резервного копирования.
Точки восстановления на диске DPM или MABS | 64 для файловых серверов и 448 для серверов приложений.<br/><br/> Точки восстановления ленты не имеют ограничений для локального DPM.

## <a name="supported-restore-methods"></a>Поддерживаемые методы восстановления

**Метод восстановления** | **Дополнительные сведения**
--- | ---
Создание виртуальной машины | Виртуальную машину можно создать во время восстановления. <br/><br/> Выбрав этот вариант, вы получите базовую виртуальную машину, запущенную и готовую к работе. Вы можете указать имя виртуальной машины, группу ресурсов, виртуальную сеть, подсеть и хранилище.  
Восстановление диска | Можно восстановить диск и использовать его для создания виртуальной Машины.<br/><br/> Если вы выберите этот вариант, Azure Backup копирует данные из хранилища в выбранную учетную запись хранения. Задание восстановления создает шаблон. Можно загрузить этот шаблон, он используется для указания пользовательских параметров виртуальной Машины и создание виртуальной Машины.<br/><br/> Этот вариант позволяет указать больше параметров для создаваемой виртуальной машины, чем предыдущий вариант.<br/><br/>
Замените существующий диск | Вы можете восстановить диск, а затем использовать его для замены диска, который в настоящий момент используется на виртуальной машине.
Восстановление файлов | Вы можете восстановить файлы из выбранной точки восстановления. Вы скачаете сценарий для подключения диска виртуальной машины из точки восстановления. Затем вы просмотрите томов диска, чтобы найти файлы и папки, которые вы хотите восстановить и отключите диск, когда все будет готово.

В следующей таблице перечислены поддержка резервного копирования во время выполнения задач управления виртуальной Машины, такие как добавление или замена дисков виртуальной Машины.

**Restore** | **Поддерживаются**
--- | ---
Восстановление между подписками, регионами или зонами | Не поддерживается.

Восстановление диска с помощью учетной записи хранения, в которой включено Шифрование службы хранилища Azure | Не поддерживается.<br/><br/> Выполните восстановление в учетной записи, в которой отключено SSE.
Восстановление в смешанных учетных записях хранения | Не поддерживается.<br/><br/> В зависимости от типа учетной записи хранения все восстановленные диски будут класса Premium или Standard, но не смешанными.
Восстановить учетную запись хранения с помощью хранилище, избыточное в пределах зоны (ZRS) | Поддерживается (для виртуальной Машины, которая сохраненный в резервной копии после января 2019 и где [зона доступности](https://azure.microsoft.com/global-infrastructure/availability-zones/) доступны)
Восстановление виртуальной машины напрямую в группе доступности | Для управляемых дисков можно восстановить диск и использовать параметр set доступности в шаблоне.<br/><br/> Эта возможность недоступна для неуправляемых дисков. Что касается неуправляемых дисков, восстановите диск, а затем создайте виртуальную машину в группе доступности.
Восстановите резервную копию неуправляемые виртуальные машины, после обновления до управляемых виртуальных Машин|  Поддерживается.<br/><br/> Вы можете восстановить диски, а затем создать управляемую виртуальную машину.
Восстановление виртуальной машины до точки восстановления до переноса виртуальной машины на управляемые диски |  Поддерживается.<br/><br/> Нужно выполнить восстановление на неуправляемых дисках (по умолчанию), преобразовать восстановленные диски в управляемые диски и создать виртуальную машину с управляемыми дисками.
Восстановление удаленной виртуальной машины |  Поддерживается.<br/><br/> Вы можете восстановить виртуальную машину из точки восстановления.
Восстановление виртуальной машины контроллера домена, которая входит в конфигурацию с несколькими контроллерами домена, с помощью портала | Поддерживается, если восстановить диск и создать виртуальную Машину с помощью PowerShell.
Восстановить виртуальную Машину в другую виртуальную сеть |    Поддерживается.<br/><br/> Виртуальная сеть должны находиться в той же подписке и регионе.

## <a name="vm-security-and-encryption-support"></a>Поддержка безопасности и шифрования виртуальной Машины

Azure Backup поддерживает шифрование передаваемых и хранящихся данных.

Сетевой трафик в Azure:

- Данные трафика резервного копирования с серверов в хранилище служб восстановления шифруется с помощью AES 256.
- Данные файла резервной копии передаются через защищенное соединение HTTPS.
- Данные файла резервной копии также хранятся в зашифрованном виде в хранилище Служб восстановления.
- Только у вас есть парольная фраза для расшифровки этих данных. Корпорация Майкрософт не может расшифровывать данные резервных копий.

  > [!WARNING]
  > После настройки хранилища только имеется доступ к ключу шифрования. Корпорация Майкрософт никогда не управляет копированием и не имеет доступа к ключу. При утрате ключа Майкрософт не сможет восстановить данные резервных копий.

Защита данных:

- При резервном копировании виртуальных машин Azure, необходимо настроить шифрование *в* виртуальной машины.
- Служба архивации Azure поддерживает шифрование дисков Azure, которая использует BitLocker на виртуальных машинах Windows и нам **dm-crypt** на виртуальных машинах Linux.
- В серверной части Azure Backup использует [шифрование службы хранилища Azure](../storage/common/storage-service-encryption.md), что обеспечивает защиту неактивных данных.

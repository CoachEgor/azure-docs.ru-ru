## <a name="azure-expressroute-support"></a>Поддержка Azure ExpressRoute

Вы можете создать резервную связь с данными azure ExpressRoute с помощью общедоступных пирингов (доступных для старых схем) и спиринга Майкрософт. Резервное копирование над приватным пирингом не поддерживается.

С публичным пирингом: Обеспечить доступ к следующим доменам/адресам:

* `http://www.msftncsi.com/ncsi.txt`
* `microsoft.com`
* `.WindowsAzure.com`
* `.microsoftonline.com`
* `.windows.net`

С помощью пиринга Майкрософт, пожалуйста, выберите следующие службы/регионы и соответствующие значения сообщества:

* Активный каталог Azure (12076:5060)
* Регион Microsoft Azure (в зависимости от местоположения хранилища служб восстановления)
* Хранилище Azure (в зависимости от местоположения хранилища служб восстановления)

Для получения более [подробной](https://docs.microsoft.com/azure/expressroute/expressroute-routing)информации см.

>[!NOTE]
>Общественное Пиринг утихает для новых схем.

## <a name="cluster-support"></a>Поддержка кластера

Azure Backup Server может защищать данные в следующих кластерных приложениях:

* Файловые серверы

* SQL Server

* Hyper-V - Если вы защищаете кластер Hyper-V с помощью уменьшенного средства защиты MABS, вы не можете добавить вторичную защиту для защищенных рабочих нагрузок Hyper-V.

    Если Hyper-V работает в Windows Server 2008 R2, необходимо установить обновление, описываемое в статье базы знаний [975354](https://support.microsoft.com/kb/975354).
    Если Hyper-V работает в Windows Server 2008 R2 в кластерной конфигурации, необходимо установить пакет обновления 2 (SP2) и обновление, описанное в статье базы знаний [971394](https://support.microsoft.com/kb/971394).

* Exchange Server — Azure Backup Server может защищать кластеры дисков, не являющиеся общими, для поддерживаемых версий Exchange Server (непрерывная репликация в кластере), а также может защищать сервер Exchange Server, для которого настроена локальная непрерывная репликация.

* SQL Server — Azure Backup Server не поддерживает резервное копирование баз данных SQL Server, размещенных в общих томах кластера (CSV).

Сервер резервного копирования Azure может защитить рабочие нагрузки кластеров, расположенные в том же домене, что и сервер MABS, а также в детском или доверительном домене. Если необходимо защитить источники данных в недоверенных доменах или рабочих группах, используйте проверку подлинности NTLM или на основе сертификата для отдельного сервера либо проверку подлинности на основе сертификата только для кластера.

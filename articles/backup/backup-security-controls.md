# <a name="security-controls-for-azure-backup"></a>Элементы управления безопасностью для резервного копирования Azure

В этой статье документируются элементы управления безопасностью, встроенные в резервное копирование Azure.

[!INCLUDE [Security controls Header](../../includes/security-controls-header.md)]

## <a name="network"></a>Сеть

| Контроль безопасности | Да/нет | Примечания | Документация
|---|---|--|
| Поддержка конечных точек обслуживания| нет |  |  |
| Поддержка инъекций VNet| нет |  |  |
| Поддержка сетевой изоляции и брандмауэра| Да | Принудительное туннелирование поддерживается для резервного копирования виртуальных машин. Принудительное туннелирование не поддерживается для рабочих нагрузок, выполняющихся на виртуальных машинах. |  |
| Принудительная поддержка туннелирования| нет |  |  |

## <a name="monitoring--logging"></a>Мониторинг & лесозаготовки

| Контроль безопасности | Да/нет | Примечания| | Документация
|---|---|--|
| Поддержка мониторинга Azure (аналитика журналов, информация о приложениях и т.д.)| Да | Аналитика журналов поддерживается с помощью журналов ресурсов. Для получения дополнительной информации см. [Мониторинг резервного копирования защищены рабочими нагрузками с помощью журнала Analytics](https://azure.microsoft.com/blog/monitor-all-azure-backup-protected-workloads-using-log-analytics/). |  |
| Управление и управление самолетом лесозаготовки и аудит| Да | Все действия, активированные клиентом на портале Azure, регистрируются в журналах действий. |  |
| Регистрация и аудит плоскости данных| нет | К плоскости данных Azure Backup невозможно обратиться напрямую.  |  |

## <a name="identity"></a>идентификации

| Контроль безопасности | Да/нет | Примечания| | Документация
|---|---|--|
| Проверка подлинности| Да | Аутентификация выполняется с помощью Azure Active Directory. |  |
| Авторизация| Да | Используются встроенные роли RBAC и роли, созданные клиентом. Для получения дополнительной информации [см.](/azure/backup/backup-rbac-rs-vault) |  |

## <a name="data-protection"></a>Защита данных

| Контроль безопасности | Да/нет | Примечания | | Документация
|---|---|--|
| Шифрование на стороне сервера в состоянии покоя: клавиши, управляемые Корпорацией Майкрософт | Да | Использование шифрования службы хранилища для учетных записей хранения. |  |
| Шифрование на стороне сервера в состоянии покоя: ключи, управляемые клиентом (BYOK) | нет |  |  |
| Шифрование уровня столбцов (Службы данных Azure)| нет |  |  |
| Шифрование в пути (например, шифрование ExpressRoute, шифрование VNet и шифрование VNet-VNet)| нет | С использованием HTTPS. |  |
| Вызовы API в зашифрованном виде| Да |  |  |

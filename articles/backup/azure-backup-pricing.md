# <a name="azure-backup-pricing"></a>Цены на Azure Backup

Дополнительные сведения о ценах на Azure Backup см. на [странице цен на Azure Backup](https://azure.microsoft.com/pricing/details/backup/).

## <a name="download-detailed-estimates-for-azure-backup-pricing"></a>Скачать подробные оценки для Azure Backup цен

Если вы хотите оценить затраты для бюджетирования или целей сравнения затрат, скачайте подробный [Azure Backupный оценщик цен](https://aka.ms/AzureBackupCostEstimates).  

### <a name="what-does-the-estimator-contain"></a>Что содержит оценщик?

На листе оценки затрат Azure Backup имеется возможность оценить все возможные рабочие нагрузки, для которых требуется создать резервную копию с помощью Azure Backup. К таким рабочим нагрузкам относятся:

- Виртуальные машины Azure
- Локальные серверы
- SQL на виртуальных машинах Azure
- SAP HANA на виртуальных машинах Azure
- Общие папки службы файлов Azure

## <a name="estimate-costs-for-backing-up-azure-vms-or-on-premises-servers"></a>Оценка затрат на резервное копирование виртуальных машин Azure или локальных серверов

Чтобы оценить затраты на резервное копирование виртуальных машин Azure или локальных серверов с помощью Azure Backup, вам потребуются следующие параметры:

- Размер виртуальных машин или локальных серверов, для которых выполняется резервное копирование
  - Введите "используемый размер" дисков или серверов, для которых необходимо создать резервную копию

- Количество серверов с таким размером

- Каков ожидаемый объем обработки данных на этих серверах?<br>
  «Обработка» — это объем изменений в данных. Например, если у вас есть виртуальная машина с 200 ГБ данных для резервного копирования и каждые 10 ГБ изменений каждый день, ежедневное обновление составляет 5%.

  - Более высокие значения будут означать, что вы создаете резервные копии дополнительных данных.

  - Выберите **низкий** или **средний** для файловых серверов и **высокий** , если вы используете базы данных

  - Если вы уверены в проценте изменения **%**, можно использовать параметр **ввести собственный%**

- Выбор политики архивации

  - Сколько времени предполагается хранить ежедневное резервное копирование? (в днях)

  - Как долго предполагается хранить еженедельные резервные копии? (в неделях)

  - Как долго предполагается хранить ежемесячные резервные копии? (в месяцах)

  - Как долго предполагается хранить ежегодное резервное копирование? (в годах)

  - Как долго вы планируете хранить моментальные снимки мгновенного восстановления? (1-5 дней)

    - Этот параметр позволяет быстро выполнить восстановление до семи дней с помощью моментальных снимков, хранящихся на дисках.

- **Необязательно** — Выборочная архивация диска

  - Если при резервном копировании виртуальных машин Azure используется параметр **Выборочная архивация диска** , выберите параметр **исключить диск** и введите процент дисков, исключаемых из резервной копии в терминах размера. Например, если у вас есть виртуальная машина, подключенная к трем дискам с 200 ГБ на каждом диске, и если требуется исключить два из них из резервной копии, введите 66,7%.

- **Необязательно** — избыточность хранилища резервных копий

  - Это указывает на избыточность учетной записи хранения, в которую передаются данные резервных копий. Мы рекомендуем использовать **GRS** для обеспечения наивысшей доступности. Так как это гарантирует, что копия данных резервных копий хранится в другом регионе, она поможет вам удовлетворить несколько стандартов соответствия. Измените значение избыточности на **LRS** , если выполняется резервное копирование сред разработки или тестирования, для которых не требуется резервное копирование корпоративного уровня. Выберите параметр **RAGRS** на листе, если необходимо оценить затраты при включенном восстановлении резервных копий в [нескольких регионах](backup-azure-arm-restore-vms.md#cross-region-restore) .

- **Необязательно** — изменение региональных цен или применение тарифов со скидкой

  - Если вы хотите проверить оценки для другого региона или тарифов со скидкой, выберите **Да** для параметра **опробовать оценки для другого региона?** и введите ставки, с помощью которых требуется выполнить оценки.

## <a name="estimate-costs-for-backing-up-sql-servers-in-azure-vms"></a>Оценка затрат на резервное копирование серверов SQL Server на виртуальных машинах Azure

Чтобы оценить затраты на резервное копирование серверов SQL Server, работающих на виртуальных машинах Azure, с помощью Azure Backup, вам потребуются следующие параметры:

- Размер серверов SQL Server, для которых выполняется резервное копирование

- Количество серверов SQL с указанным выше размером

- Каково ожидаемое сжатие данных резервного копирования серверов SQL Server?

  - Большинство Azure Backup клиентов видят, что данные резервного копирования имеют уровень сжатия 80% по сравнению с размером SQL Server при **включенном**сжатии SQL.

  - Если вы хотите увидеть другое сжатие, введите номер в это поле.

- Каков ожидаемый размер резервных копий журналов?

  - % Указывает ежедневный размер журнала в виде% от размера SQL Server

- Каков ожидаемый объем ежедневных обновлений данных на этих серверах?

  - Обычно базы данных имеют "высокий" уровень

  - Если вы уверены в проценте изменения **%**, можно использовать параметр **ввести собственный%**

- Выбор политики архивации

  - Тип резервного копирования

    - Самая эффективная политика, которую можно выбрать, — это **ежедневные разностные** резервные копии с еженедельными или ежемесячными полными резервными копиями. Azure Backup можно восстановить из разностных резервных копий с помощью одного и того же щелчка.

    - Кроме того, можно выбрать политику с ежедневными, еженедельными, ежемесячными или ежегодно полными резервными копиями. Этот параметр будет потреблять немного больше места, чем первый вариант.

  - Как долго предполагается хранить резервные копии журнала? (в днях) [7-35]

  - Сколько времени предполагается хранить ежедневное резервное копирование? (в днях)

  - Как долго предполагается хранить еженедельные резервные копии? (в неделях)

  - Как долго предполагается хранить ежемесячные резервные копии? (в месяцах)

  - Как долго предполагается хранить ежегодное резервное копирование? (в годах)

- **Необязательно** — избыточность хранилища резервных копий

  - Это указывает на избыточность учетной записи хранения, в которую передаются данные резервных копий. Мы рекомендуем использовать **GRS** для обеспечения наивысшей доступности. Так как это гарантирует, что копия данных резервных копий хранится в другом регионе, она поможет вам удовлетворить несколько стандартов соответствия. Измените значение избыточности на **LRS** , если выполняется резервное копирование сред разработки или тестирования, для которых не требуется резервное копирование корпоративного уровня.

- **Необязательно** — изменение региональных цен или применение тарифов со скидкой

  - Если вы хотите проверить оценки для другого региона или тарифов со скидкой, выберите **Да** для параметра **опробовать оценки для другого региона?** и введите ставки, с помощью которых требуется выполнить оценки.

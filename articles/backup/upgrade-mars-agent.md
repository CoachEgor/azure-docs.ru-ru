# <a name="upgrade-the-microsoft-azure-recovery-services-mars-agent"></a>Обновление агента служб восстановления Microsoft Azure (MARS)

Из этой статьи вы узнаете о следующем.

* Определение серверов с более ранними версиями агента MARS
* Обновление установок MARS на этих серверах

## <a name="identify-servers-with-earlier-versions-of-the-mars-agent"></a>Определение серверов с более ранними версиями агента MARS

Для установки резервного агента Azure и сервера резервного копирования Azure:

1. Перейдите в Хранилище служб восстановления, где вы зарегистрировали серверы, которые потенциально подкреплены более старыми версиями агента. В обновлении обновления резервного копирования Azure из Azure можно найти репрезентативный список сводов со старыми агентами резервного копирования Azure.
1. В разделе **«Настройки левого** борта» Хранилища служб восстановления выберите **инфраструктуру резервного копирования** в разделе **«Управление».**
1. Чтобы обнаружить резервные агенты Azure, установленные в рамках установок серверов резервного копирования Azure, перейдите на **серверы управления резервным** копированием под **серверами управления.** В этом списке будут указаны серверы с установками серверов резервного копирования Azure, а также номер версии для связанного агента резервного копирования Azure.

    ![Список агентов MARS, установленных в рамках установок серверов резервного копирования Azure](./media/upgrade-mars-agent/backup-management-servers.png)

1. Чтобы проверить версии агентов для установок агентов Microsoft Azure Recovery Services (MARS) или агента резервного копирования Azure, перейдите на **защищенные серверы** под **серверами управления.** Затем выберите **агент резервного копирования Azure** в типе управления резервным копированием. В этом списке будут указаны серверы с установками агента резервного копирования Azure, а также номер версии для установки.

    ![Список серверов с установленным агентом MARS](./media/upgrade-mars-agent/protected-servers.png)

1. Сортируйте колонку «Агент резервного копирования» Azure, нажав на столбец **«Агент версии»** для установок «Агент ОВС» или на колонку **Azure Backup Agent Version** для установок сервера резервного копирования Azure.

1. Предыдущий шаг даст вам список серверов с агентами резервного копирования Azure, которые имеют версии ниже 2.0.9083.0 или версии агента, перечисленные как пробелы. Это серверы, на которых необходимо обновить агенты резервного копирования Azure.

## <a name="update-the-mars-agent-installation-on-the-server"></a>Обновление установки агента MARS на сервере

После того как вы определили серверы, которые нуждаются в обновлении агента резервного копирования Azure, выполните следующие шаги для каждого идентифицированного сервера (с помощью сервера резервного копирования Azure или агента MARS). [Загрузите последнюю версию резервного агента Azure,](https://aka.ms/azurebackup_agent) прежде чем выступить ниже.

1. Нажмите на строку с агентом резервного копирования Azure ниже 2.0.9083.0 или пустой. Это откроет экран детали сервера.

    ![Защищенные серверы с устаревшими версиями агентов](./media/upgrade-mars-agent/old-agent-version.png)

    ![Серверы резервного копирования Azure с устаревшими версиями агентов](./media/upgrade-mars-agent/backup-management-servers-old-versions.png)

1. Нажмите на **Connect,** чтобы получить удаленный файл соединения рабочего стола для подключения к серверу или непосредственного подключения к серверу через удаленное соединение на рабочем столе на сервере.

    ![Подключение к серверу через удаленное подключение к рабочему столу](./media/upgrade-mars-agent/connect-to-server.png)

    >[!NOTE]
    > Если перечисленный сервер не существует или был списан, можно проигнорировать оставшиеся ниже шаги и перейти к следующему серверу.

1. Введите свои административные данные входа и войдите в систему.

1. Если прокси-сервер сервера или сервера имеет ограниченный доступ в Интернет, убедитесь, что настройки брандмауэра на сервере/прокси настроены таким образом, чтобы URL-адрес соответствовал облачности Azure, который вы используете:

    Облако Azure | URL-адрес
    -- | ---
    Облако azure (публично) |   `https://login.windows.net`
    Облако Azure China 21Vianet   | `https://login.chinacloudapi.cn`
    Облако правительства США Azure |   `https://login.microsoftonline.us`
    Облако Azure для Германии.  |  `https://login.microsoftonline.de`

1. Копирование обновления агента резервного копирования Azure на сервер.

    ![Копирование установки резервного агента Azure для сервера](./media/upgrade-mars-agent/copy-agent-installer.png)

1. Запустите установщик. Открывается мастер обновления служб восстановления Microsoft Azure.

    ![Мастер обновления служб восстановления Microsoft Azure](./media/upgrade-mars-agent/agent-upgrade-wizard.png)

1. Нажмите кнопку **Далее**.

1. Нажмите кнопку **Обновить**.

    ![Установка агента служб восстановления Microsoft Azure](./media/upgrade-mars-agent/upgrade-installation.png)

1. Окончательный экран подтверждения указывает на то, что агент резервного копирования Azure был успешно обновлен.

## <a name="for-system-center-data-protection-manager-sc-dpm-customers"></a>Для клиентов Системного центра защиты данных (SC DPM)

Если вы установили резервные агенты Azure на серверах System Center Data Protection Manager (SC DPM), вам необходимо выполнить ниже приведенные ниже действия, чтобы определить, нужно ли вашим серверам DPM обновление резервного агента Azure:

1. Войти на сервер SC DPM в качестве администратора.
2. Откройте консоль DPM.
3. Нажмите на **управление** в левом нижнем нижнем списке консоли.
4. В информации, которая отображается в левой навигации, ищите информацию об версии агента резервного копирования Azure.
5. Если версия ниже 2.0.9083.0, загрузите последний установщик резервного агента Azure и запустите установщик на сервере DPM для обновления агента резервного копирования Azure.

Повторите вышеприведенные шаги для всех серверов DPM в вашей среде.

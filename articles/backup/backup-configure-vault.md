# <a name="back-up-windows-machines-with-the-azure-backup-mars-agent"></a>Резервное копирование компьютеров Windows с помощью агента MARS Azure Backup

В этой статье объясняется, как выполнять резервное копирование компьютеров Windows с помощью службы [Azure Backup](backup-overview.md) и агента службы восстановления Microsoft Azure (MARS), также известного как агент Azure Backup.

В этой статье раскрываются следующие темы:

> [!div class="checklist"]
>
> * Проверьте предварительные требования и создайте хранилище служб восстановления.
> * Скачивание и настройка агента служб восстановления Microsoft Azure.
> * Создайте политику и расписание архивации.
> * Выполните резервное копирование по запросу.

## <a name="about-the-mars-agent"></a>Общие сведения об агенте MARS

Агент MARS используется Azure Backup для архивации файлов, папок и состояния системы с локальных компьютеров и виртуальных машин Azure в хранилище резервных копий служб восстановления в Azure. Агент можно запустить следующим образом:

* Запустите агент непосредственно на локальных компьютерах Windows, чтобы они могли выполнять резервное копирование непосредственно в хранилище служб восстановления резервных копий в Azure.
* Запустите агент на виртуальных машинах Azure под управлением Windows (параллельно с расширением резервного копирования виртуальных машин Azure), чтобы создать резервную копию конкретных файлов и папок на виртуальной машине.
* Запустить агент на сервере Microsoft Azure Backup Server (MABS) или на сервере System Center Data Protection Manager (DPM). В этом сценарии виртуальные машины и рабочие нагрузки архивируются в MABS/DPM, а затем MABS/DPM архивируются в хранилище в Azure с помощью агента MARS.
Целевой объект резервного копирования зависит от того, где установлен агент.

> [!NOTE]
> Основным методом резервного копирования виртуальных машин Azure является использование расширения Azure Backup на виртуальной машине. В этом случае резервное копирование будет выполняться для всей виртуальной машины. Если вы хотите создать резервную копию конкретных файлов и папок на виртуальной машине, может потребоваться установить и использовать агент MARS вместе с расширением. [Подробнее](backup-architecture.md#architecture-built-in-azure-vm-backup).

#### <a name="urls"></a>URL-адреса

* веб-\.msftncsi.com

Для доступа ко всем указанным выше URL-адресам и IP-адреса используется протокол HTTPS через порт 443.

## <a name="create-a-recovery-services-vault"></a>Создание хранилища Служб восстановления

Хранилище служб восстановления хранит все созданные резервные копии и точки восстановления и содержит политику резервного копирования, применяемую к резервным копиям компьютеров. Создайте хранилище следующим образом.

1. Войдите на [портал Azure](https://portal.azure.com/), используя свою подписку Azure.

2. Выполните поиск по строке **Хранилища служб восстановления** и выберите соответствующий элемент.

    ![Создание хранилища служб восстановления — шаг 1](./media/backup-configure-vault/open-recovery-services-vaults.png)

3. В меню **хранилища служб восстановления** щелкните **+ Добавить**.

    ![Создание хранилища служб восстановления — шаг 2](./media/backup-try-azure-backup-in-10-mins/rs-vault-menu.png)

4. В поле **Имя**введите понятное имя хранилища.

   * Имя должно быть уникальным в пределах подписки Azure.
   * Оно может содержать от 2 до 50 символов.
   * Имя должно начинаться с буквы, оно может содержать только буквы, цифры и дефисы.

5. Выберите подписку Azure, группу ресурсов и географический регион, в которых должно быть создано хранилище. Данные архивации отправляются в хранилище. Затем нажмите кнопку **Создать**.

    ![Создание хранилища служб восстановления — шаг 3](./media/backup-try-azure-backup-in-10-mins/rs-vault-step-3.png)

   * Для создания хранилища может потребоваться некоторое время.
   * Следите за уведомлениями о состоянии на портале в верхней правой области. Если через несколько минут хранилище не будет отображаться, нажмите кнопку **Обновить**.

     ![Кнопка "Обновить"](./media/backup-try-azure-backup-in-10-mins/refresh-button.png)

### <a name="set-storage-redundancy"></a>Установка избыточности хранилища

Azure Backup автоматически обрабатывает хранилище для хранилища. Необходимо указать способ репликации хранилища.

1. В колонке **Хранилища служб восстановления** щелкните новое хранилище. В разделе **Параметры** щелкните **свойства**.
2. В окне **Свойства**в разделе **Конфигурация архивации**щелкните **Обновить**.

3. Выберите тип репликации хранилища и нажмите кнопку **сохранить**.

      ![Настройка конфигурации для нового хранилища](./media/backup-try-azure-backup-in-10-mins/recovery-services-vault-backup-configuration.png)

* Если вы используете Azure в качестве конечной точки основного хранилища резервных копий, продолжайте использовать **геоизбыточное** значение по умолчанию.
* а если нет, — **локально избыточное** (это позволит снизить плату за хранилище Azure).
* Дополнительные сведения о [географической](../storage/common/storage-redundancy-grs.md) и [локальной](../storage/common/storage-redundancy-lrs.md) избыточности.

3. В **поле что вы хотите архивировать?** выберите **файлы и папки** и/или **состояние системы**. Доступно несколько других вариантов, но они поддерживаются только при использовании дополнительного резервного сервера. Щелкните **Подготовка инфраструктуры**.

## <a name="install-and-register-the-agent"></a>Установка и регистрация агента

1. Запустите файл **файл marsagentinstaller. exe** на компьютерах, для которых требуется создать резервную копию.
2. В мастере установки агента MARS > **Параметры установки**, укажите, где вы хотите установить агент, и расположение, используемое для кэша. Затем нажмите кнопку **Далее**.
   * Azure Backup использует кэш для хранения моментальных снимков данных перед их отправкой в Azure.
   * В расположении кэша должно быть свободное пространство, равное по крайней мере 5% размера данных, которые будут архивироваться.

    ![Параметры установки мастера MARS](./media/backup-configure-vault/mars1.png)

3. В поле **Конфигурация прокси-сервера**укажите, как агент, работающий на компьютере Windows, будет подключаться к Интернету. Затем нажмите кнопку **Далее**.

   * Если вы используете настраиваемый прокси-сервер, укажите параметры прокси-сервера и учетные данные, если это необходимо.
   * Помните, что агенту требуется доступ к [этим URL-адресам](#verify-internet-access).

     ![Доступ к Интернету через мастер MARS](./media/backup-configure-vault/mars2.png)

4. На странице **Установка** проверьте предварительные требования и нажмите кнопку **установить**.
5. После установки агента нажмите кнопку " **Перейти к регистрации**".
6. В **мастере регистрации сервера** > **идентификации хранилища**найдите и выберите скачанный файл учетных данных. Затем нажмите кнопку **Далее**.

    ![Учетные данные хранилища Register](./media/backup-configure-vault/register1.png)

7. В **параметре шифрование**укажите парольную фразу, которая будет использоваться для шифрования и расшифровки резервных копий на компьютере.

    * Сохранить парольную фразу шифрования в безопасном месте.
    * Если вы потеряли или забудете парольную фразу, корпорация Майкрософт не сможет восстановить резервные копии данных. Сохраните файл в безопасном месте, Он понадобится для восстановления резервной копии.

8. Нажмите кнопку **Готово**. Теперь, когда агент установлен и компьютер зарегистрирован в хранилище, вы можете настроить параметры резервного копирования, в том числе расписание.

## <a name="create-a-backup-policy"></a>создание политики архивации;

Политика архивации определяет, когда следует создавать моментальные снимки данных для создания точек восстановления и как долго хранить точки восстановления.

* Политика архивации настраивается с помощью агента MARS.
* Azure Backup не будет автоматически принимать летнее время (DST). Это может привести к несоответствию между реальным временем и временем запланированного резервного копирования.

Создайте политику следующим образом.
1. После загрузки и регистрации агента MARS запустите консоль агента. Его можно найти, выполнив поиск строки **Microsoft Azure Backup** на компьютере.  
2. В меню **действия**выберите пункт **Расписание архивации**.

    ![Планирование архивации Windows Server](./media/backup-configure-vault/schedule-first-backup.png)
3. В мастере планирования архивации > **Приступая к работе**, нажмите кнопку **Далее**.
4. В списке **выберите элементы для архивации**нажмите кнопку **Добавить элементы**.

    ![Выберите элементы для резервного копирования](./media/backup-azure-manage-mars/select-item-to-backup.png)

5. В окне **Выбор элементов**выберите объекты, для которых требуется создать резервную копию, и нажмите кнопку **ОК**.

    ![Выбранные элементы для резервного копирования](./media/backup-azure-manage-mars/selected-items-to-backup.png)

6. На странице **Выбор элементов для резервного копирования** нажмите кнопку **Далее**.
7. На странице « **Выбор расписания резервного копирования** » укажите, когда нужно создавать ежедневное или еженедельное резервное копирование. Затем нажмите кнопку **Далее**.

    - Точка восстановления создается при создании резервной копии.
    - Количество точек восстановления, созданных в вашей среде, зависит от расписания архивации.


8. Можно запланировать ежедневное резервное копирование до трех раз в день. Например, на снимке экрана показана две ежедневные операции резервного копирования: одна в полночь, а другая — 6:00 PM.

    ![Ежедневное расписание](./media/backup-configure-vault/day-schedule.png)


9. Вы также можете запускать еженедельные резервные копии. Например, на снимке экрана показаны резервные копии, созданные каждую альтернативное воскресенье & среду в 9:30 AM и 1:00 AM.

    ![Еженедельное расписание](./media/backup-configure-vault/week-schedule.png)


10. На странице **Выбор политики хранения** укажите, как будут храниться исторические копии данных. Затем нажмите кнопку **Далее**.

    - Параметры хранения указывают, какие точки восстановления должны храниться и как долго они должны храниться.
    - Например, при настройке параметра ежедневного хранения указывается, что во время, указанное для ежедневного сохранения, последняя точка восстановления будет храниться в течение указанного числа дней. Кроме того, можно указать ежемесячную политику хранения, чтобы указать, что точка восстановления, созданная в 30-го числа каждого месяца, должна храниться в течение 12 месяцев.
    - Хранение точек восстановления по дням и неделям обычно совпадает с расписанием резервного копирования. Это означает, что при запуске резервного копирования в соответствии с расписанием точка восстановления, созданная резервной копией, сохраняется в течение продолжительности, указанной в политике хранения ежедневно или еженедельно.
    - Например, на следующем снимке экрана:

        -   Ежедневные резервные копии в полночь и 6:00 PM хранятся в течение семи дней.
        -   Резервные копии, создаваемые в субботу в полночь и 6:00 PM, хранятся в течение четырех недель.
        -   Резервные копии, созданные в субботу на прошлой неделе месяца в полночь и 6:00 PM, хранятся в течение 12 месяцев.
        -   Резервные копии, созданные в субботу за последнюю неделю марта, хранятся в течение 10 лет.

        ![Пример хранения](./media/backup-configure-vault/retention-example.png)


11. В поле **Выбор типа начальной резервной копии** решите, следует ли выполнять начальную архивацию по сети или использовать автономное резервное копирование (Дополнительные сведения о автономной архивации см. в этой [статье](offline-backup-azure-data-box.md)). Чтобы выполнить начальную архивацию по сети, установите флажок **автоматически по сети** и нажмите кнопку **Далее**.

    ![Тип начальной архивации](./media/backup-azure-manage-mars/choose-initial-backup-type.png)


12. Просмотрите сведения на странице **Подтверждение**и нажмите кнопку **Готово**.

    ![Подтверждение типа резервной копии](./media/backup-azure-manage-mars/confirm-backup-type.png)


13. Когда мастер завершит создание расписания архивации, нажмите кнопку **Закрыть**.

    ![Подтверждение изменения процесса резервного копирования](./media/backup-azure-manage-mars/confirm-modify-backup-process.png)

Необходимо создать политику на каждом компьютере, на котором установлен агент.

### <a name="perform-the-initial-backup-offline"></a>Выполнение начальной архивации в автономном режиме

Начальное резервное копирование можно выполнять автоматически по сети или автономно. Автономное заполнение для начальной резервной копии полезно при наличии больших объемов данных, требующих передачи большого количества пропускной способности сети. Перенос в автономный режим выполняется следующим образом.

1. Данные резервной копии записываются в промежуточное расположение.
2. Используйте средство AzureOfflineBackupDiskPrep для копирования данных из промежуточного расположения на один или несколько дисков SATA.
3. Средство создает задание импорта Azure. Дополнительные [сведения](https://docs.microsoft.com/azure/storage/common/storage-import-export-service) об импорте и экспорте Azure.
4. Диски SATA отправляются в центр обработки данных Azure.
5. В центре обработки данных данные диска копируются в учетную запись хранения Azure.
6. Azure Backup копирует данные из учетной записи хранения в хранилище, а также планируется добавочное резервное копирование.

Дополнительные [сведения](offline-backup-azure-data-box.md) о заполнении в автономном режиме.

### <a name="enable-network-throttling"></a>Включение регулирования сети

С помощью функции регулирования сети можно управлять использованием пропускной способности сети агентом MARS. Регулирование полезно в том случае, если необходимо выполнять резервное копирование данных в рабочее время, но контролировать, сколько пропускной способности используется для резервного копирования и восстановления.

* Azure Backup регулирования сети использует [качество обслуживания (QoS)](https://docs.microsoft.com/windows-server/networking/technologies/qos/qos-policy-top) в локальной операционной системе.
* Регулирование сети для резервного копирования доступно в Windows Server 2012 и более поздних версий, а Windows 8 — в порядке. В операционных системах должны быть установлены последние пакеты обновления.

Включите регулирование сети следующим образом.

1. В агенте MARS щелкните **изменить свойства**.
2. На вкладке **регулирование** установите флажок **включить регулирование использования полосы пропускания Интернета для операций резервного копирования**.

    ![Регулирование сети](./media/backup-configure-vault/throttling-dialog.png)
3. Укажите допустимую пропускную способность во время работы и вне рабочих часов. Значения пропускной способности начинаются с 512 кбит/с и переходят до 1 023 Мбит/с. Нажмите кнопку **ОК**.

## <a name="run-an-on-demand-backup"></a>Выполнение резервного копирования по запросу

1. В агенте MARS щелкните **создать резервную копию сейчас**.

    ![Моментальная архивация Windows Server](./media/backup-configure-vault/backup-now.png)

2. Если версия агента MARS — 2.0.9169.0 или более поздняя, можно задать пользовательское хранение. В разделе **хранить резервную копию до** выберите дату из представленного календаря:

   ![Хранить календарь резервного копирования](./media/backup-configure-vault/mars-ondemand.png)

3. На странице **Подтверждение**проверьте параметры и нажмите кнопку **создать резервную копию**.
4. Нажмите кнопку **Закрыть** , чтобы закрыть мастер. Если сделать это до завершения резервного копирования, мастер продолжит работу в фоновом режиме.
5. После завершения начальной архивации в консоли службы архивации отобразится состояние **Задание выполнено**.

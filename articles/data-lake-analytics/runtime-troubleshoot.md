# <a name="learn-how-to-troubleshoot-u-sql-runtime-failures-due-to-runtime-changes"></a>Узнайте, как устранить неполадки в расписании выполнения u-S'L из-за изменений времени выполнения

Время выполнения программы Azure Data Lake U-S'L, включая компилятор, оптимизатор и менеджер по работе, — это то, что обрабатывает код U-S'L.

## <a name="choosing-your-u-sql-runtime-version"></a>Выбор версии выполнения U-S'L

When you submit U-SQL jobs from either Visual Studio, the ADL SDK or the Azure Data Lake Analytics portal, your job will use the currently available default runtime. Новые версии runtime U-S'L выпускаются на регулярной основе и включают как незначительные обновления, так и исправления безопасности.

Вы также можете выбрать пользовательскую версию времени выполнения; либо потому, что вы хотите попробовать новое обновление, нужно остаться на более старой версии времени выполнения, или были предоставлены с hotfix для сообщили проблемы, где вы не можете ждать регулярного нового обновления.

> [!CAUTION]
> Выбор времени выполнения, отличающегося от значения по умолчанию, может привести к разрыву заданий u-S'S. Используйте эти другие версии только для тестирования.

В редких случаях поддержка Майкрософт может приурочить другую версию времени выполнения по умолчанию для вашей учетной записи. Пожалуйста, убедитесь, что вы вернете этот штифт как можно скорее. Если вы остаетесь прижатыми к этой версии, срок ее действия истекает позже.

### <a name="monitoring-your-jobs-u-sql-runtime-version"></a>Мониторинг вашей работы U-S'L версии выполнения

Вы можете увидеть историю, которую ваши прошлые задания использовали в истории работы вашей учетной записи, через браузер работы Visual Studio или историю работы портала Azure.

1. На портале Azure выберите свою учетную запись Data Lake Analytics.
2. Выберите **просмотр всех заданий**. Появляется список всех активных и недавно готовых заданий в учетной записи.
3. При необходимости щелкните **Фильтр** для поиска заданий по значениям **Диапазон времени**, **Имя задания** и **Автор**.
4. Вы можете увидеть время выполнения, используемое в завершенных заданиях.

![Отображение версии времени выполнения прошлой работы](./media/runtime-troubleshoot/prior-job-usql-runtime-version-.png)

Доступные версии времени выполнения меняются с течением времени. Время выполнения по умолчанию всегда называется "по умолчанию", и мы сохраняем по крайней мере предыдущее время выполнения в течение некоторого времени, а также сделать специальные время выполнения доступны по целому ряду причин. Явно названные время выполнения обычно следуют следующему формату (курсив используется для переменных частей и указывает на факультативные части):

release_YYYYMMDD_adl_buildno (_modifier)

Например, release_20190318_adl_3394512_2 означает вторую версию сборки 3394512 запуска 18 марта 2019 года и release_20190318_adl_3394512_private означает частную сборку того же релиза. Примечание: Дата связана с тем, когда была проведена последняя регистрация для этого выпуска, а не обязательно официальная дата релиза.

Ниже приведены доступные в настоящее время версии времени выполнения.

- release_20190318_adl_3394512
- release_20190318_adl_5832669 текущего дефолта
- release_20190703_adl_4713356

## <a name="troubleshooting-u-sql-runtime-version-issues"></a>Проблемы устранения проблем в версии для выполнения U-S'L

Есть две возможные проблемы версии выполнения, с которыми вы можете столкнуться:

1. Скрипт или какой-либо пользовательский код изменяет поведение от одного выпуска к другому. Такие изменения, как правило, сообщаются заранее с публикацией примечаний к выпуску. Если вы столкнулись с таким нарушением, пожалуйста, свяжитесь с службой поддержки Майкрософт, чтобы сообщить об этом нарушении поведения (в случае, если он еще не был задокументирован) и представить свои рабочие места в отношении старой версии времени выполнения.

2. Вы используете время выполнения без дефолта либо явно, либо косвенно, когда оно было закреплено в вашей учетной записи, и время выполнения было удалено через некоторое время. Если вы столкнулись с отсутствующими выполнением, пожалуйста, обновите свои сценарии для выполнения текущего времени выполнения по умолчанию. Если вам нужно дополнительное время, пожалуйста, свяжитесь с службой поддержки Майкрософт

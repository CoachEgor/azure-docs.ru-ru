# <a name="entities"></a>Сущности

*Сущность* представляет движимый объект в пространстве и является основным строительным блоком удаленно отображаемого содержимого.

## <a name="entity-properties"></a>Свойства сущности

Сущности имеют преобразование, определяемое положением, вращением и масштабом. Сами по себе объекты не имеют никакой наблюдаемой функциональности. Вместо этого поведение добавляется через компоненты, которые прикрепляются к сущностям. Например, при подключении [CutPlaneComponent](../overview/features/cut-planes.md) создаст плоскость разреза в положении сущности.

Наиболее важным аспектом самой сущности является иерархия и образующееся в результате этого иерархическое преобразование. Например, когда несколько сущностей прикрепляются как дети к общей родительской сущности, все эти сущности могут быть перемещены, перемещены и масштабированы в унисон, изменяя преобразование родительской сущности.

Сущность является уникальным принадлежащим его родителем, а `Entity.Destroy()`это означает, что, когда родитель уничтожается с , так что его дети и все связанные [компоненты.](components.md) Таким образом, удаление модели со сцены `Destroy` осуществляется путем вызова на корневой узел модели, возвращенной `AzureSession.Actions.LoadModelAsync()` или ее вариантом `AzureSession.Actions.LoadModelFromSASAsync()`SAS.

Сущности создаются, когда сервер загружает содержимое или когда пользователь хочет добавить объект в сцену. Например, если пользователь хочет добавить плоскость разреза для визуализации интерьера сетки, пользователь может создать сущность, в которой должна существовать плоскость, а затем добавить к нему компонент плоскости разреза.

## <a name="query-functions"></a>Функции запроса

Существует два типа функций запроса на сущностях: синхронные и асинхронные вызовы. Синхронные запросы могут использоваться только для данных, которые присутствуют на клиенте и не предполагают больших вычислений. Примерами являются запрос компонентов, относительных преобразований объектов или отношений родителей/детей. Асинхронные запросы используются для данных, которые находятся только на сервере или включают дополнительные вычисления, которые были бы слишком дорогими для выполнения на клиенте. Примерами являются пространственные запросы границ или метаданные запросы.

### <a name="querying-components"></a>Компоненты запроса

Чтобы найти компонент определенного типа, используйте: `FindComponentOfType`

```cs
CutPlaneComponent cutplane = (CutPlaneComponent)entity.FindComponentOfType(ObjectType.CutPlaneComponent);

// or alternatively:
CutPlaneComponent cutplane = entity.FindComponentOfType<CutPlaneComponent>();
```

### <a name="querying-transforms"></a>Преобразования запросов

Запросы преобразования являются синхронными вызовами на объекте. Важно отметить, что преобразования, запрашиваемые через API, являются локальными преобразованиями пространства по отношению к родительскому объекту. Исключениясоставляют сяровые предметы, для которых местное пространство и космос мира идентичны.

> [!NOTE]
> Не существует специального API для запроса преобразования пространства мира произвольных объектов.

```cs
// local space transform of the entity
Double3 translation = entity.Position;
Quaternion rotation = entity.Rotation;
```

### <a name="querying-spatial-bounds"></a>Запрос пространственных границ

Запросы границ — это асинхронные вызовы, которые работают на полной иерархии объектов, используя одну сущность в качестве корня. Смотрите выделенную главу о [границах объектов.](object-bounds.md)

### <a name="querying-metadata"></a>Запрос метаданных

Метаданные — это дополнительные данные, хранящиеся на объектах, которые игнорируются сервером. Метаданные объектов по существу — это набор (имя, значение) пар, где _значение_ может быть численного, булеанового или типа строки. Метаданные могут экспортироваться с помощью модели.

Метаданные запросы являются асинхронными вызовами на определенную сущность. Запрос возвращает только метаданные одной сущности, а не объединенную информацию подграфика.

```cs
MetadataQueryAsync metaDataQuery = entity.QueryMetaDataAsync();
metaDataQuery.Completed += (MetadataQueryAsync query) =>
{
    if (query.IsRanToCompletion)
    {
        ObjectMetaData metaData = query.Result;
        ObjectMetaDataEntry entry = metaData.GetMetadataByName("MyInt64Value");
        System.Int64 intValue = entry.AsInt64;

        // ...
    }
};
```

Запрос будет успешным, даже если объект не имеет никаких метаданных.

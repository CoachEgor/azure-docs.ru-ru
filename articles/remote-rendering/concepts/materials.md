# <a name="materials"></a>Материалы

Материалы — это [Общие ресурсы](../concepts/lifetime.md) , которые определяют, как дополняются [сетки](meshes.md) . Материалы используются для указания того, какие [текстуры](textures.md) следует применить, будь то объекты прозрачны и как рассчитывается освещение.

Материалы автоматически создаются во время [преобразования модели](../how-tos/conversion/model-conversion.md) и доступны во время выполнения. Вы также можете создавать пользовательские материалы из кода и заменять существующие. Этот сценарий особенно важен, если вы хотите использовать одни и те же материалы во многих сетках. Так как изменения материала видны в каждой сетке, ссылающейся на него, этот метод можно использовать для простого применения изменений.

> [!NOTE]
> Некоторые варианты использования, например выделение подобранного объекта, можно выполнить путем изменения материалов, но гораздо проще в [хиерарчикалстатеоверридекомпонент](../overview/features/override-hierarchical-state.md).

## <a name="material-types"></a>Типы материалов

Удаленная визуализация Azure имеет два разных типа материалов:

* [Материалы PBR](../overview/features/pbr-materials.md) используются для поверхностей, которые должны отображаться как можно физически. Реалистичное освещение вычисляются для этих материалов с помощью функции *отрисовки на основе физической* среды (PBR). Чтобы максимально эффективно воспользоваться этим типом материала, важно предоставить высококачественные входные данные, такие как приблизительные и обычные карты.

* [Цветовые материалы](../overview/features/color-materials.md) используются в случаях, когда не требуется дополнительное освещение. Эти материалы всегда полностью освещены и проще в настройке. Цветовые материалы используются для данных, которые не должны иметь освещения или уже содержат статическое освещение, например модели, полученные через [фотограмметри](https://en.wikipedia.org/wiki/Photogrammetry).

## <a name="mesh-vs-meshcomponent-material-assignment"></a>Сетка и назначение материалов Мешкомпонент

В [сетках](meshes.md) есть одна или несколько вложенных сеток. Каждая подсеть ссылается на один материал. Вы можете изменить материал, который будет использоваться непосредственно в сетке, или переопределить материал, который будет использоваться для подсети в [мешкомпонент](meshes.md#meshcomponent).

При изменении материала непосредственно в сетчатом ресурсе это изменение затрагивает все экземпляры этой сети. Однако изменение его в Мешкомпонент влияет только на один экземпляр сетки. Более подходящий метод зависит от требуемого поведения, но изменение Мешкомпонент является более распространенным подходом.

## <a name="material-classes"></a>Классы материалов

Все материалы, предоставляемые API, являются производными от базового `Material`класса. Их тип можно запрашивать через `Material.MaterialSubType` или путем приведения напрямую:

``` cs
void SetMaterialColorToGreen(Material material)
{
    if (material.MaterialSubType == MaterialType.Color)
    {
        ColorMaterial colorMaterial = material as ColorMaterial;
        colorMaterial.AlbedoColor = new Color4(0, 1, 0, 1);
        return;
    }

    PbrMaterial pbrMat = material as PbrMaterial;
    if( pbrMat!= null )
    {
        PbrMaterial pbrMaterial = material.PbrMaterial.Value;
        pbrMaterial.AlbedoColor = new Color4(0, 1, 0, 1);
        return;
    }
}
```

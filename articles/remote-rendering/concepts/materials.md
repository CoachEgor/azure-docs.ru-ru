# <a name="materials"></a>Материалы

Материалы являются [общими ресурсами,](../concepts/lifetime.md) определяющими способ отображаемого [сетки.](meshes.md) Материалы используются для определения того, какие [текстуры](textures.md) применять, следует ли сделать объекты прозрачными и как будет рассчитываться освещение.

Материалы автоматически создаются во время [преобразования модели](../how-tos/conversion/model-conversion.md) и доступны во время выполнения. Вы также можете создавать пользовательские материалы из кода и заменять существующие. Этот сценарий имеет особое значение, если вы хотите поделиться тем же материалом во многих сетках. Поскольку изменения материала видны на каждой сетке, которая ссылается на него, этот метод может быть использован для легкого применения изменений.

> [!NOTE]
> Некоторые случаи использования, такие как выделение выбранного объекта, могут быть сделаны путем изменения материалов, но гораздо проще достигается с помощью [HierarchicalStateOverrideComponent.](../overview/features/override-hierarchical-state.md)

## <a name="material-types"></a>Типы материалов

Удаленное рендерирование Azure имеет два различных типа материала:

* [Материалы PBR](../overview/features/pbr-materials.md) используются для поверхностей, которые должны быть отрисованы как можно более физически правильно. Реалистичное освещение вычисляется для этих материалов с помощью *физически основанной визуализации* (PBR). Чтобы получить максимальную отдачу от этого типа материала, важно предоставить высококачественные входные данные, такие как шероховатости и обычные карты.

* [Цветовые материалы](../overview/features/color-materials.md) используются для случаев, когда не желательно дополнительное освещение. Эти материалы всегда полный яркий и легче настроить. Цветовые материалы используются для данных, которые должны либо не иметь освещения вообще, либо уже включает статическое освещение, например, модели, полученные с помощью [фотограмметрии.](https://en.wikipedia.org/wiki/Photogrammetry)

## <a name="mesh-vs-meshcomponent-material-assignment"></a>Сетка против задания материала MeshComponent

[У сетки](meshes.md) есть один или несколько подмешок. Каждый подмеш ссылается на один материал. Вы можете изменить материал для использования либо непосредственно на сетке, или вы можете переопределить, какой материал использовать для подсетки на [MeshComponent.](meshes.md#meshcomponent)

При изменении материала непосредственно на ресурсе сетки это изменение влияет на все экземпляры этой сетки. Изменение его на MeshComponent, однако, влияет только на один экземпляр сетки. Какой метод является более подходящим, зависит от желаемого поведения, но изменение MeshComponent является более распространенным подходом.

## <a name="material-classes"></a>Классы материалов

Все материалы, предоставляемые API, `Material`происходят из базового класса. Их тип можно запрашивать через `Material.MaterialSubType` или путем литья их непосредственно:

``` cs
void SetMaterialColorToGreen(Material material)
{
    if (material.MaterialSubType == MaterialType.Color)
    {
        ColorMaterial colorMaterial = material as ColorMaterial;
        colorMaterial.AlbedoColor = new Color4(0, 1, 0, 1);
        return;
    }

    PbrMaterial pbrMat = material as PbrMaterial;
    if( pbrMat!= null )
    {
        PbrMaterial pbrMaterial = material.PbrMaterial.Value;
        pbrMaterial.AlbedoColor = new Color4(0, 1, 0, 1);
        return;
    }
}
```

# <a name="object-and-resource-lifetime"></a>Срок службы объектов и ресурсов

Удаленное рендерирование Azure различает два типа: **объекты** и **ресурсы.**

## <a name="object-lifetime"></a>Срок службы объекта

*Объекты* считаются вещами, которые пользователь может создавать, изменять и уничтожать по своему усмотрению. Объекты могут быть дублированы свободно, и каждый экземпляр может мутировать с течением времени. Следовательно, [сущности](entities.md) и [компоненты](components.md) являются объектами.

Срок службы объектов полностью находится под контролем пользователя. Это не связано с сроком службы представительства на стороне клиента. Классы, `Component` как `Destroy` `Entity` и функция, которая должна быть вызвана для дераспределения объекта на удаленном узла рендеринга. Кроме того, `Entity.Destroy()` уничтожит сущность, ее детей и все компоненты в этой иерархии.

## <a name="resource-lifetime"></a>Срок службы ресурсов

*Ресурсы* — это вещи, срок службы которых полностью управляется удаленным узлам визуализации. Ресурсы учитываются внутренне. Они получают deallocated, когда никто не ссылается на них больше.

Большинство ресурсов может быть создано только косвенно, как правило, путем загрузки их из файла. При многократной загрузке одного и того же файла удаленная рендеринг Azure возвращает одну и ту же ссылку и не загружает данные снова.

Многие ресурсы неизменяемы, например [сетки](meshes.md) и [текстуры.](textures.md) Некоторые ресурсы являются изменяемыми, хотя, [например, материалы](materials.md). Поскольку ресурсы часто используются совместно, изменение ресурса может повлиять на несколько объектов. Например, изменение цвета материала изменит цвет всех объектов, которые используют сетки, которые, в свою очередь, ссылаются на этот материал.

### <a name="built-in-resources"></a>Встроенные ресурсы

Удаленная рендеринг Azure содержит некоторые встроенные ресурсы, которые могут `builtin://` быть загружены, предусмотрев их соответствующий идентификатор во время `AzureSession.Actions.LoadXYZAsync()`вызова. Доступные встроенные ресурсы перечислены в документации для каждой соответствующей функции. Например, в [главе «Небо»](../overview/features/sky.md) перечислены встроенные текстуры неба.

## <a name="general-lifetime"></a>Общий срок службы

Срок службы всех объектов и ресурсов привязан к соединению. При отключении все отбрасывается. При повторном подключении к тому же сеансу график сцены будет пуст, а все ресурсы удалены.

На практике загрузка одного и того же ресурса в сеанс после отключения обычно происходит быстрее, чем в первый раз. Это так, потому что большинство ресурсов должны быть загружены из Azure Storage в первый раз, что не является необходимым во второй раз, экономя значительное количество времени.

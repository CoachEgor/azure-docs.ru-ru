# <a name="create-an-azure-remote-rendering-account"></a>Создание учетной записи удаленного отсчета Azure

В этой главе вы проведете шаги по созданию учетной записи для службы **удаленного отжимания Azure.** Действительная учетная запись является обязательной для завершения любого из quickstarts или учебники.

## <a name="create-an-account"></a>Создание учетной записи

Для создания учетной записи для службы удаленного рендеринга Azure необходимы следующие шаги:

1. Перейти на [страницу предварительного просмотра смешанной реальности](https://aka.ms/MixedRealityPrivatePreview)
1. Нажмите кнопку «Создать ресурс»
1. В поле поиска ("Поиск рынка"), введите в "Удаленное рендеринг" и нажмите "введите".
1. В списке результатов нажмите на плитку "Удаленное рендеринг"
1. На следующем экране нажмите кнопку "Создать". Открывается форма для создания новой учетной записи удаленного рендеринга:
    1. Установите "Имя ресурса" на имя учетной записи
    1. Обновление 'Подписка' при необходимости
    1. Установите «Группу ресурсов» в группу ресурсов по вашему выбору
1. Как только учетная запись создана, перейдите к нему и:
    1. Во вкладке *«Обзор»* обратите внимание на «Идентификатор счета»
    1. В *вкладке «Настройки > ключей доступа»* обратите внимание на «Основной ключ» - это секретный ключ учетной записи

### <a name="retrieve-the-account-information"></a>Извлечение информации об учетной записи

Образцы и учебники требуют, чтобы вы предоставили идентификатор учетной записи и ключ. Например, в файле **arrconfig.json,** который используется для скриптов образца PowerShell:

```json
    "accountSettings": {
        "arrAccountId": "<fill in the account ID from the Azure portal>",
        "arrAccountKey": "<fill in the account key from the Azure portal>",
        "region": "<select from available regions>"
    },
```

Ознакомьтесь со [списком доступных регионов](../reference/regions.md) для заполнения варианта *региона.*

Значения и **`arrAccountKey`** **`arrAccountId`** можно найти на портале, как описано в следующих шагах:

* Перейдите на [портал Azure](https://www.portal.azure.com)
* Найти свой **"Удаленный счет рендеринга"** - он должен быть в списке **"Последние ресурсы".** Вы также можете искать его в панели поиска в верхней части. В этом случае убедитесь, что подписка, которую вы хотите использовать, выбрана в фильтре подписки по умолчанию (значок фильтра рядом с панелью поиска):

![Фильтр подписки](./media/azure-subscription-filter.png)

Нажатие на вашу учетную запись приводит вас к этому экрану, который показывает **идентификатор учетной записи** сразу:

![Идентификатор учетной записи Azure](./media/azure-account-id.png)

Для ключа выберите **ключи доступа** в панели слева. На следующей странице отображается основной и вторичный ключ:

![Ключи доступа Azure](./media/azure-account-primary-key.png)

Значение для **`arrAccountKey`** может быть как первичным, так и вторичным ключом.

## <a name="link-storage-accounts"></a>Учетные записи хранилища ссылки

В этом пункте объясняется, как связать учетные записи хранилища с учетной записью удаленного рендеринга. При подключении учетной записи хранилища нет необходимости создавать SAS URI каждый раз, когда вы хотите взаимодействовать с данными в вашей учетной записи, например, при загрузке модели. Вместо этого можно использовать имена учетных записей хранилища непосредственно, как описано в [разделе загрузки модели.](../concepts/models.md#loading-models)

Шаги в этом пункте должны быть выполнены для каждой учетной записи хранилища, которая должна использовать этот альтернативный метод доступа. Если вы еще не создали учетные записи хранения, можно пройти соответствующий шаг в [преобразовании модели для рендеринга.](../quickstarts/convert-model.md#storage-account-creation)

Теперь предполагается, что у вас есть учетная запись для хранения. Перейдите к учетной записи хранилища на портале и перейдите на вкладку **Access Control (IAM)** для этой учетной записи хранения:

![Учетная запись хранения IAM](./media/azure-storage-account.png)

 Убедитесь, что у вас есть разрешения владельца на эту учетную запись хранения, чтобы убедиться, что вы можете добавить назначения ролей. Если у вас нет доступа, опция **назначения роли** будет отключена.

 Необходимо добавить три различных ролей, описанных в следующих шагах. Если вы не предоставите все три уровня доступа, при попытке получить доступ к учетной записи хранилища возникнут проблемы с разрешением.

 Нажмите на кнопку **Добавить** в "Добавить роль назначения" плитка, чтобы добавить первую роль:

![Учетная запись хранения IAM](./media/azure-add-role-assignment.png)

* Первая роль, которую следует назначить, — это **Владелец,** показанный на скриншоте выше. 
* Выберите **учетную запись удаленного рендеринга** из**доступа к** отсеву.
* Выберите учетную запись подписки и удаленного рендеринга в последних выпадающих пунктах.

Повторите добавление новых ролей еще два раза для соответствующих выделений из отсева **ролей:**
* **Участник учетной записи хранения**
* **участник данных BLOB-объектов хранилища**;

Другие выпадающие результаты выбираются в качестве первого шага.

Если вы добавили все три роли, ваша учетная запись удаленного рендеринга Azure имеет доступ к учетной записи хранилища с помощью системы, назначенной идентификаторами управляемых служб.

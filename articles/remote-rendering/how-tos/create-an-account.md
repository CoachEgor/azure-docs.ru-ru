# <a name="create-an-azure-remote-rendering-account"></a>Создание учетной записи для Удаленной отрисовки Azure

В этой главе описано, как создать учетную запись для службы **удаленной визуализации Azure** . Допустимая учетная запись является обязательной для выполнения любого из кратких руководств или учебников.

## <a name="create-an-account"></a>Создание учетной записи

Чтобы создать учетную запись для службы удаленной визуализации Azure, необходимо выполнить следующие действия.

1. Переход на [страницу предварительного просмотра смешанной реальности](https://aka.ms/MixedRealityPrivatePreview)
1. Нажмите кнопку "создать ресурс"
1. В поле поиска ("Поиск в Marketplace") введите "Удаленная подготовка к просмотру" и нажмите клавишу "Ввод".
1. В списке результатов щелкните плитку "Удаленная подготовка к просмотру".
1. На следующем экране нажмите кнопку "создать". Откроется форма для создания новой учетной записи удаленной подготовки к просмотру:
    1. В поле "имя ресурса" укажите имя учетной записи.
    1. Обновить "Subscription" при необходимости
    1. Задайте для параметра "Группа ресурсов" выбранную группу ресурсов
1. После создания учетной записи перейдите к ней и:
    1. На вкладке *Обзор* Обратите внимание на значение "идентификатор учетной записи".
    1. На вкладке " *параметры > доступа* " Обратите внимание на раздел "первичный ключ" — это ключ секретной учетной записи.

### <a name="retrieve-the-account-information"></a>Получение сведений об учетной записи

Для примеров и учебников требуется указать идентификатор учетной записи и ключ. Например, в файле **аррконфиг. JSON** , который используется для примеров сценариев PowerShell:

```json
    "accountSettings": {
        "arrAccountId": "<fill in the account ID from the Azure portal>",
        "arrAccountKey": "<fill in the account key from the Azure portal>",
        "region": "<select from available regions>"
    },
```

См. [список доступных регионов](../reference/regions.md) для заполнения параметра *Region* .

Значения для **`arrAccountId`** и **`arrAccountKey`** можно найти на портале, как описано в следующих шагах:

* Перейдите на [портал Azure](https://www.portal.azure.com).
* Найдите **"учетная запись удаленной подготовки к просмотру"** — она должна находиться в списке **"Недавние ресурсы"** . Вы также можете найти его на панели поиска вверху. В этом случае убедитесь, что в фильтре подписки по умолчанию выбрана нужная подписка (значок фильтра рядом с панелью поиска):

![Фильтр подписки](./media/azure-subscription-filter.png)

Щелкнув свою учетную запись, вы перейдете на этот экран, который сразу же покажет **идентификатор учетной записи** :

![Идентификатор учетной записи Azure](./media/azure-account-id.png)

Для ключа выберите **ключи доступа** на панели слева. На следующей странице показан первичный и вторичный ключи:

![Ключи доступа Azure](./media/azure-account-primary-key.png)

Значение для **`arrAccountKey`** может быть либо первичным, либо вторичным ключом.

## <a name="link-storage-accounts"></a>Связывание учетных записей хранения

В этом абзаце объясняется, как связать учетные записи хранения с учетной записью удаленной подготовки. При связывании учетной записи хранения не требуется создавать URI SAS каждый раз, когда требуется взаимодействовать с данными в учетной записи, например при загрузке модели. Вместо этого можно использовать имена учетных записей хранения напрямую, как описано в [разделе Загрузка модели](../concepts/models.md#loading-models).

Действия, описанные в этом абзаце, необходимо выполнять для каждой учетной записи хранения, которая должна использовать этот альтернативный метод доступа. Если вы еще не создали учетные записи хранения, можно проанализировать соответствующий шаг в [кратком руководстве преобразование модели для подготовки к просмотру](../quickstarts/convert-model.md#storage-account-creation).

Теперь предполагается, что у вас есть учетная запись хранения. Перейдите к учетной записи хранения на портале и перейдите на вкладку **Управление доступом (IAM)** для этой учетной записи хранения.

![IAM учетной записи хранения](./media/azure-storage-account.png)

 Убедитесь, что у вас есть разрешения владельца этой учетной записи хранения, чтобы обеспечить возможность добавления назначений ролей. Если у вас нет доступа, параметр **добавить назначение роли** будет отключен.

 Необходимо добавить три отдельные роли, как описано в следующих шагах. Если не предоставить все три уровня доступа, при попытке доступа к учетной записи хранения будут выдаваться проблемы с разрешениями.

 Нажмите кнопку **Добавить** на плитке "добавить назначение роли", чтобы добавить первую роль:

![IAM учетной записи хранения](./media/azure-add-role-assignment.png)

* Первая роль для назначения — **owner** , как показано на снимке экрана выше. 
* Выберите **учетную запись удаленной отрисовки** в раскрывающемся списке ***назначить доступ** .
* Выберите подписку и учетную запись удаленной отрисовки в последних раскрывающихся списках.

Повторите Добавление новых ролей еще раз для соответствующих вариантов выбора из раскрывающегося списка **ролей** .
* **Участник учетной записи хранения**
* **участник данных BLOB-объектов хранилища**;

В качестве первого шага выбираются другие раскрывающиеся списки.

Если вы добавили все три роли, учетная запись удаленной отрисовки Azure имеет доступ к вашей учетной записи хранения, используя назначенные системой удостоверения управляемой службы.

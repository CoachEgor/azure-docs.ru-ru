# <a name="convert-models"></a>Преобразование моделей

Дистанционное рендерирование Azure позволяет отобразить очень сложные модели. Для достижения максимальной производительности данные должны быть предварительно обработаны, чтобы быть в оптимальном формате. В зависимости от объема данных этот шаг может занять некоторое время. Было бы непрактично, если бы это время было потрачено во время загрузки модели. Кроме того, было бы расточительно повторять этот процесс в течение нескольких сессий. По этим причинам сервис ARR предоставляет специальную *услугу конверсии,* которую вы можете запустить заранее.
После преобразования модель может быть загружена из учетной записи хранения Azure.

## <a name="supported-source-formats"></a>Поддерживаемые форматы источников

Служба конверсии поддерживает следующие форматы:

- **FBX** (версия 2011 и выше)
- **GLTF** (версия 2.x)
- **GLB** (версия 2.x)

Существуют незначительные различия между форматами в отношении преобразования материального свойства, как указано в материалах главы [отображения для форматов моделей.](../../reference/material-mapping.md)

## <a name="the-conversion-process"></a>Процесс преобразования

1. [Подготовьте два контейнера для хранения Azure Blob:](blob-storage.md)один для ввода, один для вывода
1. Загрузите модель в входиный контейнер (по желанию под подпаской)
1. Триггер процесс преобразования через [модель преобразования REST API](conversion-rest-api.md)
1. Опрос службы для прогресса преобразования
1. После завершения, загрузить модель
    - из учетной записи хранилища (см. шаги "Ссылка на учетные записи хранения" на [Создание учетной записи](../create-an-account.md#link-storage-accounts) для ссылки учетной записи хранилища)
    - или путем предоставления *общей подписи доступа (SAS).*

Все данные модели (вход и выход) хранятся в хранилище Azure blob при условии использования Azure blob. Дистанционное рендерирование Azure дает полный контроль над управлением активами.

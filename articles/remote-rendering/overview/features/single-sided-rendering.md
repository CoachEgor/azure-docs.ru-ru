# <a name="single-sided-rendering"></a>Одностороннее рендеринг

Большинство рендереров используют [закулисный выбраковку](https://en.wikipedia.org/wiki/Back-face_culling) на заднем лице для повышения производительности. Однако, когда сетки разрезаются с [разрезами плоскостей,](cut-planes.md)пользователи часто смотрят на заднюю сторону треугольников. Если эти треугольники отбрасываются, результат не выглядит убедительным.

Способ надежно предотвратить эту проблему, заключается в том, чтобы сделать треугольники *двусторонними.* Поскольку неиспользование закулисного выбраковки имеет последствия для производительности, по умолчанию Azure Remote Rendering переключается только на двустороннее рендеринг для сетки, пересекающиеся с плоскостью разреза.

Односторонний *параметр визуализации* позволяет настроить это поведение.

> [!CAUTION]
> Одностороннее настройка рендеринга является экспериментальной особенностью. Он может быть удален снова в будущем. Пожалуйста, не изменяйте настройки по умолчанию, если она действительно не решает критическую проблему в приложении.

## <a name="prerequisites"></a>Предварительные требования

## <a name="single-sided-rendering-setting"></a>Одностороннее настройка рендеринга

Существует три различных режима:

**Нормальный:** В этом режиме сетки всегда отображаются при их преобразовании. Это означает, что `opaqueMaterialDefaultSidedness` сетки `SingleSided` преобразуется с набором всегда будет получать оказанные с задней лицом выбраковки включен, даже если они пересекаются разрез плоскости.

**ДинамическийДублвейдинг:** В этом режиме, когда разрезательная плоскость пересекает сетку, она автоматически переключается на двусторонню визуализацию. Этот режим является режимом по умолчанию.

**ВсегдаDoubleSided:** Принуждает всю одностороннюю геометрию быть оказанной двусторонней во все времена. Этот режим в основном подвергается, так что вы можете легко сравнить влияние производительности между односторонним и двусторонним рендеринга.

Изменение односторонней настройки рендеринга может быть сделано следующим образом:

``` cs
void ChangeSingleSidedRendering(AzureSession session)
{
    SingleSidedSettings settings = session.Actions.SingleSidedSettings;

    // Single-sided geometry is rendered as is
    settings.Mode = SingleSidedMode.Normal;

    // Single-sided geometry is always rendered double-sided
    settings.Mode = SingleSidedMode.AlwaysDoubleSided;
}
```

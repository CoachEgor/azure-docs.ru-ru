# <a name="debug-rendering"></a>Отладка отрисовки

API отладки для подготовки к просмотру предоставляет ряд глобальных параметров для изменения отрисовки на стороне сервера с различными эффектами отладки.

## <a name="available-debug-rendering-effects"></a>Доступные эффекты отрисовки отладки

|Параметр                          | Действие                               |
|---------------------------------|:-------------------------------------|
|Счетчик кадров                    | Отображает наложение текста в левом верхнем углу фрейма. Текст показывает текущий идентификатор кадра на стороне сервера, который постоянно увеличивается по мере выполнения подготовки к просмотру. |
|Число многоугольников                    | Отображает наложение текста в левом верхнем углу фрейма. Текст показывает текущее количество многоугольников, то же самое значение, запрашиваемое [запросами производительности на стороне сервера](performance-queries.md)| 
|Каркасной модели                        | Если параметр включен, все геометрические объекты, загруженные на сервере, будут подготавливаться к просмотру в режиме каркаса. В этом режиме выполняется растрирование только краев многоугольников. |

Следующий код включает следующие эффекты отладки:

``` cs
void EnableDebugRenderingEffects(AzureSession session, bool highlight)
{
    DebugRenderingSettings settings = session.Actions.DebugRenderingSettings;

    // Enable frame counter text overlay on the server side rendering
    settings.RenderFrameCount = true;

    // Enable polygon count text overlay on the server side rendering
    settings.RenderPolygonCount = true;

    // Enable wireframe rendering of object geometry on the server
    settings.RenderWireframe = true;
}
```

> [!NOTE]
> Все эффекты рендеринга отладки — это глобальные параметры, влияющие на весь кадр.

## <a name="use-cases"></a>Варианты использования

API отладки для подготовки к просмотру предназначен для выполнения простых задач отладки, таких как проверка правильности работы подключения службы. Параметры отрисовки текста непосредственно влияют на видеокадровки с пониженным потоком. При их включении проверяется, правильно ли получены и перекодированы новые кадры.

Однако предоставленные эффекты не дают никакого подробного самоанализ в работоспособность службы. Для этого варианта использования рекомендуется использовать [запросы производительности на стороне сервера](performance-queries.md) .

## <a name="performance-considerations"></a>Вопросы производительности

* Включение наложения текста незначительно снижает производительность.
* Включение каркасного режима приводит к нетривиальным затратам на производительность, хотя он может меняться в зависимости от сцены. Для сложных сцен этот режим может привести к сбросу частоты кадров ниже целевого объекта 60 Гц.

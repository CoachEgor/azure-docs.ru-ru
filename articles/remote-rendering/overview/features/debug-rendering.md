# <a name="debug-rendering"></a>Отображение рендеринга

Отладка отображения API предоставляет ряд глобальных вариантов для изменения отправления сервера с различными эффектами отладки.

## <a name="available-debug-rendering-effects"></a>Доступные эффекты отладки рендеринга

|Параметр                          | Действие                               |
|---------------------------------|:-------------------------------------|
|Счетчик рамы                    | Рендерс текстовая накладка в верхний левый угол кадра. Текст показывает текущий идентификатор кадра на стороне сервера, который постоянно приращается как рендеринг. |
|Количество полигонов                    | Рендерс текстовая накладка в верхний левый угол кадра. Текст показывает в настоящее время отображено количество полигонов, то же значение, что и запрашиваемые [запросами на стороне сервера производительности](performance-queries.md)| 
|Каркаса                        | Если включена, вся геометрия объекта, загруженная на сервер, будет отображаться в режиме wireframe. Только края полигонов будут rasterized n этот режим. |

Следующий код позволяет эти эффекты отладки:

``` cs
void EnableDebugRenderingEffects(AzureSession session, bool highlight)
{
    DebugRenderingSettings settings = session.Actions.DebugRenderingSettings;

    // Enable frame counter text overlay on the server side rendering
    settings.RenderFrameCount = true;

    // Enable polygon count text overlay on the server side rendering
    settings.RenderPolygonCount = true;

    // Enable wireframe rendering of object geometry on the server
    settings.RenderWireframe = true;
}
```

![Отображение](./media/debug-rendering.png)

> [!NOTE]
> Все эффекты отладки рендеринга являются глобальными настройками, которые влияют на весь кадр.

## <a name="use-cases"></a>Варианты использования

Отладка отображения API предназначена для простых задач отладки, например, для проверки подключения службы, фактически работающего правильно. Параметры визуализации текста непосредственно влияют на вниз поток видео кадры. Включение их проверяет, если новые кадры получены и видео-расшифровывается правильно.

Тем не менее, при условии, эффекты не дают каких-либо подробных самоанализа в службе здоровья. Для этого случая рекомендуется [запрашивать производительность на стороне сервера.](performance-queries.md)

## <a name="performance-considerations"></a>Вопросы производительности

* Включение текстовых накладок практически не ложится накладные расходы на производительность.
* Дополнительно включение наложения действительно несет нетривиальные накладные расходы производительности, хотя она может варьироваться в зависимости от сцены. Для сложных сцен этот режим может привести к тому, что частота кадров опустится ниже цели 60 Гц.

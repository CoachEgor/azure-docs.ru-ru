# <a name="system-requirements"></a>Требования к системе

> [!IMPORTANT]
> В настоящее время **удаленная рендеринг Azure** находится в открытом доступе.
> Эта предварительная версия предоставляется без соглашения об уровне обслуживания и не рекомендована для использования рабочей среде. Некоторые функции могут не поддерживаться или их возможности могут быть ограничены. Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

В этой главе перечислены минимальные системные требования для работы с *удаленным отжимацией Azure* (ARR).

## <a name="development-pc"></a>Разработка ПК

* Версия для Windows 10 1903 или выше.
* Современные графические драйверы.
* Дополнительно: Декодер аппаратного видеоH265, если вы хотите использовать локальный предварительный просмотр удаленно отображаемого содержимого (например, в Unity).

> [!IMPORTANT]
> Обновление Windows не всегда обеспечивает самые последние драйверы GPU, проверьте веб-сайт производителя графического процессора для последних драйверов:
>
> * [Водители AMD](https://www.amd.com/en/support)
> * [Драйверы Intel](https://www.intel.com/content/www/us/en/support/detect.html)
> * [Водители NVIDIA](https://www.nvidia.com/Download/index.aspx)

В таблице ниже перечислены графические процессоры, которые поддерживают декодирование аппаратного видео H265.

| Производитель GPU | Поддерживаемые модели |
|-----------|:-----------|
| NVIDIA | Проверьте **матрицу поддержки NVDEC** [в нижней части этой страницы.](https://developer.nvidia.com/video-encode-decode-gpu-support-matrix) Ваш графический процессор нуждается в ДА в **8-битной колонке H.265 4:2:0.** |
| AMD | Графические процессоры с по крайней мере версией 6 [единого видеодедера](https://en.wikipedia.org/wiki/Unified_Video_Decoder#UVD_6)AMD. |
| Intel | Скайлейк и новые процессоры |

Несмотря на то, что может быть установлен правильный кодек H265, свойства безопасности на dLL кодек могут привести к сбоям инициализации кодека. Руководство [по устранению неполадок](../resources/troubleshoot.md#h265-codec-not-available) описывает шаги, как решить эту проблему. Проблема DLL может возникнуть только при использовании службы в настольном приложении, например в Unity.

## <a name="devices"></a>Устройства

В настоящее время Azure Remote Rendering поддерживает только **HoloLens 2** и Windows desktop UWP в качестве целевого устройства.

Важно использовать новейший кодек HEVC, так как новые версии имеют значительные улучшения в задержке. Чтобы проверить, какая версия установлена на вашем устройстве:

1. Запустите **магазин Майкрософт**.
1. Нажмите кнопку **"..."** в правом верхнем правом.
1. Выберите **Загрузки и Обновления**.
1. Поиск по списку **для HEVC видео расширения от производителя устройств**.
1. Убедитесь, что указанный кодек имеет по крайней мере версию **1.0.21821.0**.
1. Нажмите кнопку **Get Updates** и дождитесь ее установки.

## <a name="software"></a>Программное обеспечение

Необходимо установить следующее программное обеспечение:

* Последняя версия **Visual Studio 2019** [(загрузка)](https://visualstudio.microsoft.com/vs/older-downloads/)
* **Windows SDK 10.0.18362.0** [(загрузка)](https://developer.microsoft.com/windows/downloads/windows-10-sdk)
* **ГИТ** [(загрузка)](https://git-scm.com/downloads)
* Дополнительно: Для просмотра видеопотока с сервера на настольном ПК необходимо расширение **видео HEVC** [(ссылка Microsoft Store).](https://www.microsoft.com/p/hevc-video-extensions/9nmzlz57r3t7)

# <a name="what-is-a-virtual-network-link"></a>Что такое виртуальная сетевая ссылка?

После создания частной зоны DNS в Azure она не сразу доступна из любой виртуальной сети. Вы должны связать его с виртуальной сетью, прежде чем VM, размещенный в этой сети, сможет получить доступ к частной зоне DNS.
Чтобы связать частную зону DNS с виртуальной сетью, необходимо создать виртуальную сетевую ссылку в частной зоне DNS. Каждая частная зона DNS имеет коллекцию виртуальных сетевых ресурсов, связывающих детские ресурсы. Каждый из этих ресурсов представляет собой подключение к виртуальной сети.

Вы можете связать виртуальную сеть с частной зоной DNS как регистрационную виртуальную сеть или как виртуальную сеть разрешения.

## <a name="registration-virtual-network"></a>Регистрация виртуальной сети

При создании связи между [частной](https://docs.microsoft.com/azure/dns/private-dns-getstarted-portal#link-the-virtual-network) зоной DNS и виртуальной сетью у вас есть возможность включить [авторегистрацию](./private-dns-autoregistration.md) записей DNS для виртуальных машин. Если вы выберете этот вариант, виртуальная сеть становится регистрационной виртуальной сетью для частной зоны DNS. Запись DNS автоматически создается для виртуальных машин, которые развертываются в сети. Записи DNS создаются для виртуальных машин, которые вы уже развернули в виртуальной сети. С точки зрения виртуальной сети частная зона DNS становится зоной регистрации этой виртуальной сети.
Одна частная зона DNS может иметь несколько виртуальных сетей регистрации, однако каждая виртуальная сеть может иметь именно одну зону регистрации, связанную с ней.

## <a name="resolution-virtual-network"></a>Разрешение виртуальной сети

Когда вы создаете виртуальную сетевую ссылку в частной зоне DNS и выбираете не включать авторегистрацию DNS, виртуальная сеть рассматривается как разрешение только виртуальной сети. Записи DNS для виртуальных машин, развернутых в таких сетях, не будут автоматически создаваться в связанной частной зоне DNS. Однако виртуальные машины, развернутые в такой сети, могут успешно задавить dNS-записи из частной зоны DNS. Эти записи могут быть созданы вами вручную или могут быть заселены из других виртуальных сетей, которые были связаны как регистрационные сети с частной зоной DNS.
Одна частная зона DNS может иметь виртуальные сети с несколькими разрешениями, а виртуальная сеть может иметь несколько зон разрешения, связанных с ней.

## <a name="limits"></a>Ограничения

Чтобы понять, сколько сетей регистрации и разрешения, можно связать с частными зонами DNS см. [ограничения Azure DNS](https://docs.microsoft.com/azure/azure-resource-manager/management/azure-subscription-service-limits#azure-dns-limits)

## <a name="other-considerations"></a>Другие замечания

* Виртуальные сети, развернутые с использованием классической модели развертывания, не поддерживаются.

* Можно создать только одну связь между частной зоной DNS и виртуальной сетью.

* Каждая виртуальная сетевая ссылка в частной зоне DNS должна иметь уникальное имя в контексте частной зоны DNS. Вы можете иметь ссылки с тем же именем в различных частных зонах DNS.

* После создания виртуальной сетевой ссылки проверьте поле "Статус ссылки" виртуального сетевого ресурса ссылки. В зависимости от размера виртуальной сети, это может занять несколько минут, прежде чем ссылка работает и статус ссылки изменения *завершена*.

* При удалении виртуальной сети все виртуальные сетевые ссылки и автоматически зарегистрированные DNS-записи, связанные с ней в различных частных зонах DNS, автоматически удаляются.

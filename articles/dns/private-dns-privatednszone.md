# <a name="what-is-a-private-azure-dns-zone"></a>Что такое частная зона Azure DNS

Azure DNS предоставляет надежную и защищенную службу DNS для разрешения доменных имен в виртуальных сетях, позволяя обойтись без поддержки собственной системы DNS. Частные зоны DNS позволяют использовать пользовательские доменные имена, а не только предоставленные Azure имена, как ранее. 

Записи, содержащиеся в частной зоне DNS, не разрешимы из Интернета. Разрешение DNS против частной зоны DNS работает только из виртуальных сетей, которые к ней подключены.

Вы можете связать частную зону DNS с одной или более виртуальными сетями, создав [виртуальные сетевые ссылки.](./private-dns-virtual-network-links.md)
Вы также можете включить функцию [автоматической регистрации](./private-dns-autoregistration.md) для автоматического управления жизненным циклом Записей DNS для виртуальных машин, развернутых в виртуальной сети.

## <a name="limits"></a>Ограничения

Чтобы понять, сколько частных зон DNS можно создать в подписке и сколько наборов записей поддерживается в частной зоне DNS, сможете [ограничить Azure DNS](https://docs.microsoft.com/azure/azure-resource-manager/management/azure-subscription-service-limits#azure-dns-limits)

## <a name="restrictions"></a>Ограничения

* Отдельные помеченные частные зоны DNS не поддерживаются. Ваша частная зона DNS должна иметь две или более меток. Например, contoso.com имеет две метки, разделенные точкой. Частная зона DNS может иметь максимум 34 метки.
* Нельзя создавать делегации зон (записи NS) в частной зоне DNS. Если вы собираетесь использовать домен ребенка, вы можете непосредственно создать домен в качестве частной зоны DNS и связать его с виртуальной сетью без настройки делегации сервера имен из родительской зоны.

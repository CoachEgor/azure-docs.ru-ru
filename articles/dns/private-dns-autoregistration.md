# <a name="what-is-the-autoregistration-feature-of-azure-dns-private-zones"></a>Какова функция авторегистрации частных зон Azure DNS

Функция автоматической регистрации частных зон Azure DNS избавляет от управления записью DNS для виртуальных машин, развернутых в виртуальной сети. При [свяжении виртуальной сети](./private-dns-virtual-network-links.md) с частной зоной DNS и автоматической регистрации для всех виртуальных машин записи DNS для виртуальных машин, развернутых в виртуальной сети, автоматически создаются в частной зоне DNS. В дополнение к записям с переднего взгляда (записи, записи обратного поиска (записи PTR) также автоматически создаются для виртуальных машин.
Если вы добавите больше виртуальных машин в виртуальную сеть, DNS-записи для этих виртуальных машин также автоматически создаются в связанной частной dNS-зоне.

При удалении виртуальной машины записи DNS для виртуальной машины автоматически удаляются из частной зоны DNS.

Можно включить авторегистрацию, выбрав опцию "Включить авторегистрацию" при создании виртуальной сетевой ссылки.

![Включить авторегистрацию](./media/privatedns-concepts/enable-autoregistration.png)

## <a name="restrictions"></a>Ограничения

* Авторегистрация работает только для виртуальных машин. Для всех других ресурсов, таких как балансеры внутренней нагрузки и т.д., вы можете создавать записи DNS вручную в частной зоне DNS, связанной с виртуальной сетью.
* Записи DNS создаются автоматически только для первичной виртуальной машины NIC. Если ваши виртуальные машины имеют более одного NIC, вы можете вручную создавать записи DNS для других сетевых интерфейсов.
* авторегистрация для IPv6 (записи AAAA) не поддерживается.

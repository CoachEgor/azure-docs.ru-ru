# <a name="how-azure-dns-works-with-other-azure-services"></a>Как Azure DNS работает с другими службами Azure

Azure DNS является размещенной службой управления и разрешения имен (DNS). Вы можете использовать ее для создания общедоступных DNS-имен для других приложений и служб, развертываемых в Azure. Создание имени службы Azure в личном домене не представляет никаких трудностей. Нужно просто добавить для этой службы запись нужного типа.

* Для динамически выделенных IP-адресов можно создать запись CNAME DNS, которая сопоставляет DNS-имя, созданное Azure, с вашей службой. Стандарты DNS не позволяют использовать запись CNAME для вершины зоны. Вместо нее можно использовать запись псевдонима. Дополнительные сведения см. в статье [Руководство Настройка записи псевдонима для ссылки на Azure общедоступным IP-адресом](tutorial-alias-pip.md).
* Для статически выделенных IP-адресов можно создать DNS-запись A с любым именем, что подразумевает, что вы можете указать даже *голый домен*.

В следующей таблице приведены поддерживаемые типы записей, которые можно использовать для различных служб Azure. Как видно из этой таблицы, Azure DNS поддерживает записи DNS только для сетевых ресурсов с выходом в Интернет. Azure DNS невозможно использовать для разрешения имен внутренних, частных адресов.

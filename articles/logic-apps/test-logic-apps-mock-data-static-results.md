# <a name="test-logic-apps-with-mock-data-by-setting-up-static-results"></a>Тест логических приложений с макет данных путем настройки статических результатов

При тестировании логических приложений вы можете быть не готовы вызывать или получать доступ к приложениям, службам и системам по разным причинам. Обычно в этих сценариях, возможно, придется запускать различные пути состояния, ошибки силы, предоставлять определенные органы реагирования сообщений, или даже попытаться пропустить некоторые шаги. Настройка статических результатов для действия в приложении логики позволяет высмеивать выходные данные из этого действия. Включение статических результатов в действие не приводит к запуску действия, а возвращает данные макета.

Например, если вы настроили статические результаты для Outlook 365, движок Logic Apps просто возвращает указанные вами макетные данные в качестве статических результатов, а не звонит Outlook и отправляет электронное письмо.

## <a name="prerequisites"></a>Предварительные требования

* Подписка Azure. Если у вас еще нет подписки Azure, <a href="https://azure.microsoft.com/free/" target="_blank">зарегистрируйтесь для получения бесплатной учетной записи Azure</a>.

* Основные знания о [том, как создавать логические приложения](../logic-apps/quickstart-create-first-logic-app-workflow.md)

* Приложение логики, где требуется настроить статические результаты

<a name="set-up-static-results"></a>

## <a name="set-up-static-results"></a>Настройка статических результатов

1. Если вы еще не сделали этого, на [портале Azure](https://portal.azure.com)откройте свое логическое приложение в Logic Apps Designer.

1. В действии, где требуется настроить статические результаты, выполните следующие действия: 

   1. В верхнем правом углу действия выберите эллипсы *(...*) кнопку и выберите **Статический результат,** например:

      ![Выберите "Статический результат" > "Включить статический результат"](./media/test-logic-apps-mock-data-static-results/select-static-result.png)

   1. Выберите **Enable Static Result.** Для требуемых свойств укажите значения вывода макета, которые необходимо вернуть для ответа действия.

      Например, вот необходимые свойства для действия HTTP:

      | Свойство | Описание |
      |----------|-------------|
      | **Состояние** | Статус действия для возврата |
      | **Код статуса** | Конкретный код состояния для возврата |
      | **Заголовки** | Содержимое заголовка для возврата |
      |||

      ![Выберите "Включить статический результат"](./media/test-logic-apps-mock-data-static-results/enable-static-result.png)

      Для ввода макетных данных в формате JavaScript Object Notation![(JSON) выберите](./media/test-logic-apps-mock-data-static-results/switch-to-json-mode-button.png)«Переключатель в **режимjJ»** (Выберите «Переключиться в режим JSON»).

   1. Для дополнительных свойств откройте список **дополнительных полей Select** и выберите свойства, над которыми вы хотите издеваться.

      ![Выберите дополнительные свойства](./media/test-logic-apps-mock-data-static-results/optional-properties.png)

1. Когда вы будете готовы к экономии, выберите **Done.**

   В верхнем правом углу заглавной панели теперь отображается значок![тестового](./media/test-logic-apps-mock-data-static-results/static-results-test-beaker-icon.png)стакана (Icon для статических результатов), который указывает на то, что вы включили статические результаты.

   ![Значок, показывающий включенные статические результаты](./media/test-logic-apps-mock-data-static-results/static-results-enabled.png)

   Чтобы найти предыдущие запуски, в которые используются данные макета, см. [Поиск запусков, которые используют статические результаты](#find-runs-mock-data) позже в этой теме.

<a name="reuse-sample-outputs"></a>

## <a name="reuse-previous-outputs"></a>Повторное использование предыдущих выходов

Если приложение логики имеет предыдущий запуск с выходами, которые можно повторно использовать в качестве макетных выходов, можно скопировать и вставить выводы из этого запуска.

1. Если вы еще не сделали этого, на [портале Azure](https://portal.azure.com)откройте свое логическое приложение в Logic Apps Designer.

1. В главном меню приложения логики выберите **Обзор**.

1. В разделе **История «Бега»** выберите забега приложения логики, который вы хотите.

1. В рабочем процессе приложения логики найдите и расширьте действие с нужными выводами.

1. Выберите ссылку **на выпусксырья Show.**

1. Копируйте либо полный объект JavaScript Object Notation (JSON), либо конкретный подраздел, который вы хотите использовать, например, раздел выводов, или даже просто раздел заголовков.

1. Выполните последующие действия по открытию окна **результатов Static** для ваших действий в [Настройке статических результатов.](#set-up-static-results)

1. После открытия окна **Static,** выберите любой из шагов:

   * Чтобы вставить полный объект JSON, выберите![ **Переключатель в режимj JSON** ( Выберите "Переключиться в режим JSON"):](./media/test-logic-apps-mock-data-static-results/switch-to-json-mode-button.png)

     ![Выберите "Переключиться в режим JSON" для полного объекта](./media/test-logic-apps-mock-data-static-results/switch-to-json-mode-button-complete.png)

   * Чтобы вставить только раздел JSON, рядом с этикеткой этого раздела, выберите **Режим Переключения** для этого раздела, например:

     ![Выберите "Переключиться в режим JSON" для выходов](./media/test-logic-apps-mock-data-static-results/switch-to-json-mode-button-outputs.png)

1. В редакторе JSON вставьте ранее скопированный JSON.

   ![Режим JSON](./media/test-logic-apps-mock-data-static-results/json-editing-mode.png)

1. По завершении нажмите кнопку **Готово**. Или, чтобы вернуться к дизайнеру,![выберите режим](./media/test-logic-apps-mock-data-static-results/switch-editor-mode-button.png) **редактора switch** (выберите "Режим редактора переключателя").

<a name="find-runs-mock-data"></a>

## <a name="find-runs-that-use-static-results"></a>Поиск запусков, которые используют статические результаты

История запусков приложения логики определяет запуски, в которых действия используют статические результаты. Чтобы найти эти трассы, выполните следующие действия:

1. В главном меню приложения логики выберите **Обзор**. 

1. В правом стеле, под **историей запусков,** найдите столбец **Static Results.** 

   Например, при любом запуске, включающий действия с результатами, столбец **Static Results** настроен на **enabled:**

   ![История выполнения - столбец статических результатов](./media/test-logic-apps-mock-data-static-results/run-history.png)

1. Для просмотра действий, которые используют статические результаты, выберите запуск, который вы хотите, где настроен столбец **Static Results** **enabled.**

   Действия, применяющие статические результаты, показывают значок тестового стакана (Icon![для статических результатов)](./media/test-logic-apps-mock-data-static-results/static-results-test-beaker-icon.png)значок, например:

   ![История выполнения - действия, которые используют статические результаты](./media/test-logic-apps-mock-data-static-results/static-results-enabled-run-details.png)

## <a name="disable-static-results"></a>Отключить статические результаты

Отключение статических результатов не выбрасывает значения из последней установки. Таким образом, при следующем входе статических результатов можно продолжить использование прежних значений.

1. Найдите действие, в котором вы хотите отключить статические выходы. В верхнем правом углу действия выберите значок тестового стакана (Icon![для статических результатов).](./media/test-logic-apps-mock-data-static-results/static-results-test-beaker-icon.png)

   ![Отключить статические результаты](./media/test-logic-apps-mock-data-static-results/disable-static-results.png)

1. Выберите **отключить статический результат** > **сделано.**

   ![Отключить статические результаты](./media/test-logic-apps-mock-data-static-results/disable-static-results-button.png)

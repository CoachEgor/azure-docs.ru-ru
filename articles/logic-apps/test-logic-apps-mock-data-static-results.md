# <a name="test-logic-apps-with-mock-data-by-setting-up-static-results"></a>Тестирование приложений логики с помощью фиктивных данных, настроив статический результатов

При тестировании приложения логики, может оказаться готовы фактического вызова или доступа к приложениями, службами и системами по различным причинам. Обычно в этих сценариях может потребоваться запустить другое условие пути, принудительно реализующие ошибки, укажите текст ответа определенных сообщений или даже Попробуйте пропустить некоторые шаги. Настроить статический результаты для действия в приложении логики, вы можете создать макет выходных данных из этого действия. Возможности статического результаты в действие не работает действие, но вместо этого возвращает фиктивных данных.

Например если вы настроили статический результаты для Outlook 365 mail действие отправки, обработчик Logic Apps просто возвращает макет данных, указанной в качестве статического результатов, а не вызывать Outlook и отправить сообщение электронной почты.

## <a name="prerequisites"></a>Технические условия

* Подписка Azure. Если у вас еще нет подписки Azure, <a href="https://azure.microsoft.com/free/" target="_blank">зарегистрируйтесь для получения бесплатной учетной записи Azure</a>.

* Базовые знания [создания приложений логики](../logic-apps/quickstart-create-first-logic-app-workflow.md).

* Приложение логики, где вы хотите настроить статический результатов

<a name="set-up-static-results"></a>

## <a name="set-up-static-results"></a>Настройка статического результатов

1. Если у вас ее еще в [портала Azure](https://portal.azure.com), откройте приложение логики в конструкторе Logic Apps.

1. Действие, где вы хотите настроить статический результаты выполните следующие действия. 

   1. В правом верхнем углу действия, нажмите кнопку с многоточием (*...* ) и выберите **статический результат**, например:

      ![Выберите «Статический результат» > «Включить статический результат»](./media/test-logic-apps-mock-data-static-results/select-static-result.png)

   1. Выберите **включить статический результат**. Для свойства требуется (*) укажите макет выходных значений, которые необходимо вернуть для действия ответа.

      Например ниже приведены обязательные свойства для действия HTTP.

      | Свойство | Описание |
      |----------|-------------|
      | **Состояние** | Состояние действия для возврата |
      | **Код состояния** | Код состояния, который возвращает |
      | **Заголовки** | Содержимое заголовка для возврата |
      |||

      ![Выберите «Включить статический результат»](./media/test-logic-apps-mock-data-static-results/enable-static-result.png)

      Чтобы ввести макетов данные в формате JavaScript Object Notation (JSON), выберите **переключиться в режим JSON** (![Выбор «Переключиться в режим JSON»](./media/test-logic-apps-mock-data-static-results/switch-to-json-mode-button.png)).

   1. Необязательные свойства, откройте **выберите необязательные поля** и выберите свойства, которые требуется имитировать.

      ![Выберите дополнительные свойства](./media/test-logic-apps-mock-data-static-results/optional-properties.png)

1. Когда вы будете готовы для сохранения, выберите **сделать**.

   В правом верхнем углу действия, заголовок теперь отображается значок с изображением мензурки теста (![значок для статических результатов](./media/test-logic-apps-mock-data-static-results/static-results-test-beaker-icon.png)), что означает, что вы включили статический результаты.

   ![Отображение значка включить статический результатов](./media/test-logic-apps-mock-data-static-results/static-results-enabled.png)

   Для поиска предыдущих запусков, использующих фиктивных данных, см. в разделе [найти, что используют статический результаты](#find-runs-mock-data) далее в этом разделе.

<a name="reuse-sample-outputs"></a>

## <a name="reuse-previous-outputs"></a>Выходные данные предыдущих повторного использования

Если приложение логики имеет предыдущего запуска с выходные данные, которые можно повторно использовать как макетов выходные данные, можно скопировать и вставить выходные данные выполнения.

1. Если у вас ее еще в [портала Azure](https://portal.azure.com), откройте приложение логики в конструкторе Logic Apps.

1. В главном меню приложения логики, выберите **Обзор**.

1. В **журнал запусков** раздела, выберите приложение логики, вы запустите.

1. В рабочем процессе приложения логики найдите и разверните действие, которое имеет выходные данные, которые вы хотите.

1. Выберите **Показать необработанные выходные данные** ссылку.

1. Скопируйте полный объект JavaScript Object Notation (JSON) или определенного подраздела, которые вы хотите использовать, например, раздел выходных данных или даже просто разделе заголовки.

1. Выполните действия для открытия **статический результат** поле для действий в [настроить статический результаты](#set-up-static-results).

1. После **статический результат** откроется, выберите этапа:

   * Чтобы вставить весь объект JSON, выберите **переключиться в режим JSON** (![Выбор «Переключиться в режим JSON»](./media/test-logic-apps-mock-data-static-results/switch-to-json-mode-button.png)):

     ![Выберите «Переключиться в режим JSON» для полного объекта](./media/test-logic-apps-mock-data-static-results/switch-to-json-mode-button-complete.png)

   * Чтобы вставить только один раздел JSON, рядом с меткой этот раздел, выберите **переключиться в режим JSON** этого раздела, например:

     ![Выберите «Переключиться в режим JSON» для выходных данных](./media/test-logic-apps-mock-data-static-results/switch-to-json-mode-button-outputs.png)

1. Вставьте скопированные ранее JSON в редакторе JSON.

   ![Режим JSON](./media/test-logic-apps-mock-data-static-results/json-editing-mode.png)

1. По завершении нажмите кнопку **Готово**. Чтобы вернуться в конструктор, выберите **режим Переключить редактор** (![выберите «Переключить режим редактора»](./media/test-logic-apps-mock-data-static-results/switch-editor-mode-button.png)).

<a name="find-runs-mock-data"></a>

## <a name="find-runs-that-use-static-results"></a>Найти, что используют статический результатов

Журнал запусков приложения логики идентифицирует выполняется, где действия использовать статический результаты. Чтобы найти эти запусков, выполните следующие действия.

1. В главном меню приложения логики, выберите **Обзор**. 

1. В области справа в разделе **журнал запусков**, найти **статический результаты** столбца. 

   Любое выполнение, позволяет выполнять действия с результатами имеет **статический результаты** столбец значение **включено**, например:

   ![Журнал - выполнения результаты статического столбца](./media/test-logic-apps-mock-data-static-results/run-history.png)

1. Чтобы просмотреть действия, использующие статические результаты, выберите выполнение, требуется where **статический результаты** столбцу присваивается **включено**.

   Действия, использующие статические результаты Показать о теста (![значок для статических результатов](./media/test-logic-apps-mock-data-static-results/static-results-test-beaker-icon.png)) значок, например:

   ![Журнал - выполнения действий, которые использовать статический результатов](./media/test-logic-apps-mock-data-static-results/static-results-enabled-run-details.png)

## <a name="disable-static-results"></a>Отключить статический результатов

Отключение статического результаты не отказываться значения из последнего пакета установки. Таким образом при включении статический результаты следующий раз, можно продолжить использование предыдущего значения.

1. Найдите действие, где вы хотите отключить статические выходные данные. В правом верхнем углу действия, выберите значок с изображением мензурки теста (![значок для статических результатов](./media/test-logic-apps-mock-data-static-results/static-results-test-beaker-icon.png)).

   ![Отключить статический результатов](./media/test-logic-apps-mock-data-static-results/disable-static-results.png)

1. Выберите **отключения статический результирующих** > **сделать**.

   ![Отключить статический результатов](./media/test-logic-apps-mock-data-static-results/disable-static-results-button.png)

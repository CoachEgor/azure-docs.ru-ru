# <a name="get-more-data-items-or-records-by-using-pagination-in-azure-logic-apps"></a>Получить дополнительные данные, элементов или записей с помощью разбиения на страницы в Azure Logic Apps

При извлечении данных, элементов и записей с помощью действия соединителя в [Azure Logic Apps](../logic-apps/logic-apps-overview.md), может появиться результирующих наборов настолько большим, что действие не возвращает все результаты, в то же время. С некоторых действий количество результатов может превысить размер страницы по умолчанию. В этом случае действие возвращает только первую страницу результатов. Например, размер страницы по умолчанию для соединителя SQL Server **получить строки** действие — 2048, но могут отличаться в зависимости от других параметров.

Некоторые действия позволяют включить *разбиение на страницы* параметр, чтобы приложение логики получения других результатов, вплоть до предела разбиение на страницы, но возвращают эти результаты в одном сообщении, по завершении действия. При использовании разбиения на страницы, необходимо указать *пороговое значение* значение, равное целевое количество результатов, чтобы вернуть действие. Действие извлекает результаты до достижения вашей указанное пороговое значение. Когда общее число элементов меньше указанного порогового значения, действие извлекает все результаты.

Включение извлекает страницы параметров разбиения на страницы результатов на основе размера страницы соединителя. Это означает, что в некоторых случаях может появиться больше результатов, чем на указанное пороговое значение. Например, при использовании SQL Server **получить строки** действие, которое поддерживает параметр разбиение на страницы:

* Размер страницы по умолчанию действия равен 2048 записей на одну страницу.
* Предположим, что у вас 10 000 записей и указать в качестве минимального 5000 записей.
* Разбиение на страницы получает страницы записей, поэтому для получения по крайней мере минимального, это действие возвращает 6144 записи (3 страницы x 2048 записи), не 5000 записей.

Ниже приведен список с лишь некоторые из которых может превышать размер страницы по умолчанию для выполнения конкретных действий соединителей:

## <a name="turn-on-pagination"></a>Включить разбиение на страницы

Чтобы определить, поддерживает ли действие разбиения на страницы в конструкторе приложений логики, проверьте параметры действия для **разбиение на страницы** параметр. В этом примере показано, как включить разбиение на страницы в SQL Server **получить строки** действие.

1. В правом верхнем углу действия, нажмите кнопку с многоточием (**...** ) и выберите **параметры**.

   ![Откройте параметры действия](./media/logic-apps-exceed-default-page-size-with-pagination/sql-action-settings.png)

   Если действие поддерживает разбиение на страницы, действие показано **разбиение на страницы** параметр.

1. Изменение **разбиение на страницы** Azure с помощью **Off** для **на**. В **пороговое значение** свойство, укажите целочисленное значение для количества результаты, которые нужно действие для возврата.

   ![Укажите минимальное количество возвращаемых результатов](./media/logic-apps-exceed-default-page-size-with-pagination/sql-action-settings-pagination.png)

1. Когда вы будете готовы, нажмите кнопку **Готово**.

## <a name="workflow-definition---pagination"></a>Определение рабочего процесса — разбиение на страницы

При включении разбиения на страницы для действия, которое поддерживает такую возможность, определение рабочего процесса приложения логики включает в себя `"paginationPolicy"` свойство вместе с `"minimumItemCount"` свойства в этом действии `"runtimeConfiguration"` свойство, например:

```json
"actions": {
   "HTTP": {
      "inputs": {
         "method": "GET",
         "uri": "https://www.testuri.com"
      },
      "runAfter": {},
      "runtimeConfiguration": {
         "paginationPolicy": {
            "minimumItemCount": 1000
         }
      },
      "type": "Http"
   }
},
```

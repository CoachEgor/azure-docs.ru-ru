# <a name="add-trading-partners-to-integration-accounts-for-azure-logic-apps"></a>Добавление торговых партнеров в учетные записи интеграции для приложений логики Azure

В [приложениях Для логики Azure](../logic-apps/logic-apps-overview.md)можно создавать автоматизированные рабочие процессы интеграции между бизнесом (B2B) с помощью [учетной записи интеграции](../logic-apps/logic-apps-enterprise-integration-create-integration-account.md) с логическими приложениями. Чтобы представлять свою организацию и других лиц, вы создаете и добавляете торговых партнеров в качестве артефактов в свой интеграционный счет. Партнеры являются организациями, которые участвуют в B2B-транзакциях и обмениваются сообщениями друг с другом.

Прежде чем создавать этих партнеров, убедитесь, что обсудите и поделитесь информацией с партнерами о том, как идентифицировать и проверять сообщения, которые отправляют другие. После согласования этих сведений вы будете готовы создать партнеров в вашей учетной записи интеграции.

## <a name="partner-roles-in-integration-accounts"></a>Роли партнеров в интеграционных счетах

Чтобы определить сведения о сообщениях, обмениваемых с вашими партнерами, вы создаете и добавляете [соглашения](../logic-apps/logic-apps-enterprise-integration-agreements.md) в качестве артефактов в свою учетную запись интеграции. Соглашения требуют, по крайней мере, двух партнеров в вашей интеграционной учетной записи. Ваша организация всегда является *принимающим партнером* в соглашении. Организация, которая обменивается сообщениями с вашей организацией, является *гостем.* Гостем может быть другая компания или даже отдел в вашей собственной организации. Добавив этих партнеров, можно приступать к созданию соглашения.

В соглашении указаны детали обработки входящих и исходящих сообщений с точки зрения партнера-хозяина. Для входящих сообщений в **настройках receive** указывается, как партнер-хозяин получает сообщения от гостевого партнера в соглашении. Для исходящих сообщений **в настройках отправки** указывается, как партнер-хост отправляет сообщения гостевому партнеру.

## <a name="prerequisites"></a>Предварительные требования

* Подписка Azure. Если у вас еще нет подписки Azure, [получите бесплатную учетную запись Azure](https://azure.microsoft.com/free/).

* [Интеграционный учет](../logic-apps/logic-apps-enterprise-integration-create-integration-account.md) для хранения ваших партнеров, соглашений и других артефактов B2B. Эта учетная запись интеграции должна быть связана с подпиской Azure.

## <a name="create-partner"></a>Создание партнера

1. Войдите на [портал Azure](https://portal.azure.com).

1. В главном меню Azure выберите **Все службы**. В поле поиска введите "интеграцию" и выберите **учетные записи интеграции.**

   ![Выберите "Интеграционные счета"](./media/logic-apps-enterprise-integration-partners/find-integration-accounts.png)

1. В разделе **Учетные записи интеграции** выберите учетную запись интеграции, в которую нужно добавить партнеров.

   ![Выбор учетной записи интеграции](./media/logic-apps-enterprise-integration-partners/select-integration-account.png)

1. Выберите плитку **Партнеры**.

   ![Выбор плитки "Партнеры"](./media/logic-apps-enterprise-integration-partners/choose-partners.png)

1. В разделе **Партнеры** выберите **Добавить**. В Add **Partner**предоставьте сведения о партнере, описанные в таблице ниже.

   ![Выберите "Добавить" и предоставить информацию о партнере](./media/logic-apps-enterprise-integration-partners/add-partners.png)

   | Свойство | Обязательно | Описание |
   |----------|----------|-------------|
   | **Название** | Да | Имя партнера |
   | **Квалификатор** | Да | Идентификационный орган, предоставляющий уникальные бизнес-идентификационные данные организациям, например **D-U-N-S (Dun & Bradstreet).** <p>Партнеры могут выбрать взаимоопределенную бизнес-идентичность. Для этих сценариев выберите **взаимно определяемый** для EDIFACT или взаимно **определенный (X12)** для X12. <p>Для RosettaNet выберите только **DUNS**, который является стандартом. |
   | **Значение** | Да | Значение, идентифицирует документы, получаемые вашими логическими приложениями. <p>Для RosettaNet это значение должно быть девятизначным числом, которое соответствует номеру DUNS. |
   ||||

   > [!NOTE]
   > Для партнеров, которые используют RosettaNet, вы можете указать дополнительную информацию, сначала создав этих партнеров, а затем [отредактировав их впоследствии.](#edit-partner)

1. Когда все будет готово, нажмите кнопку **ОК**.

   Ваш новый партнер теперь фигурирует в списке **Партнеров.** Кроме того, плитка **Партнеров** обновляет текущее число партнеров.

   ![Новый партнер](./media/logic-apps-enterprise-integration-partners/new-partner.png)

## <a name="edit-partner"></a>Изменение данных партнера

1. На [портале Azure](https://portal.azure.com) найдите и выберите свою учетную запись интеграции.
Выберите плитку **Партнеры**.

   ![Выбор плитки "Партнеры"](./media/logic-apps-enterprise-integration-partners/edit.png)

1. Под **партнерами**выберите партнера, который вы хотите отсеить, и выберите **Edit.** Под **edit,** внести изменения.

   ![Внесение и сохранение изменений](./media/logic-apps-enterprise-integration-partners/edit-partner.png)

   Для RosettaNet, в **RosettaNet Partner Properties,** вы можете указать эту дополнительную информацию:

   | Свойство | Обязательно | Описание |
   |----------|----------|-------------|
   | **Классификация партнеров** | нет | Тип организации партнера |
   | **Код цепочки поставок** | нет | Код цепочки поставок партнера, например, "Информационные технологии" или "Электронные компоненты" |
   | **Имя контакта** | нет | Контактное имя партнера |
   | **Отправить по электронной почте** | нет | Адрес электронной почты партнера |
   | **Факс** | нет | Номер факса партнера |
   | **Телефон** | нет | Номер телефона партнера |
   ||||

1. Когда вы закончите, выберите **OK,** чтобы сохранить изменения.

## <a name="delete-partner"></a>Удаление партнера

1. На [портале Azure](https://portal.azure.com) найдите и выберите свою учетную запись интеграции. Выберите плитку **Партнеры**.

   ![Выбор плитки "Партнеры"](./media/logic-apps-enterprise-integration-partners/choose-partners-to-delete.png)

1. В разделе **Партнеры** выберите партнера, которого нужно удалить. Щелкните **Удалить**.

   ![Удаление партнера](./media/logic-apps-enterprise-integration-partners/delete-partner.png)

# <a name="access-to-azure-virtual-network-resources-from-azure-logic-apps-by-using-integration-service-environments-ises"></a>Доступ к ресурсам виртуальных сетей Azure из Azure Logic Apps с использованием сред службы интеграции (ISE)

В некоторых случаях logic apps и учетные записи интеграции требуется доступ к защищенным ресурсам, таким как виртуальные машины (VM) и других систем или служб, которые находятся внутри [виртуальной сети Azure](../virtual-network/virtual-networks-overview.md). Чтобы настроить этот доступ, вы можете [создать *среда службы интеграции* (ISE)](../logic-apps/connect-virtual-network-vnet-isolated-environment.md) где можно запускать приложения логики и создать интеграции учетных записей.

При создании интегрированную среду Сценариев Azure развертывает частных и изолированного экземпляра службы Logic Apps в виртуальной сети Azure. Частный экземпляр использует выделенные ресурсы, такие как служба хранилища, и выполняется отдельно от общедоступной глобальной службы Logic Apps. Отделение изолированного экземпляра закрытый и открытый глобальный экземпляр также помогает уменьшить влияние других клиентов Azure на производительность вашего приложения, который также называется [влияния «шумных соседей»](https://en.wikipedia.org/wiki/Cloud_computing_issues#Performance_interference_and_noisy_neighbors).

После создания вашей интегрированной среды Сценариев, когда вы собираетесь создать логику приложения или интеграция учетной записи, можно выбрать вашей интегрированной среды Сценариев в расположение логику приложения или интеграция учетной записи:

![Выбор среды службы интеграции](./media/connect-virtual-network-vnet-isolated-environment-overview/select-logic-app-integration-service-environment.png)

Приложение логики может теперь прямой доступ к системам, которые находятся внутри или подключенных к виртуальной сети с помощью любого из этих элементов:

* **Интегрированной среды Сценариев**-с меткой соединитель для этой системы, таких как SQL Server
* Объект **Core**-встроенные триггер или действие, например триггер HTTP или действие
* Настраиваемый соединитель

В этом обзоре приводится более подробное описание как интегрированную среду Сценариев с помощью приложений логики и интеграции учетных записей прямой доступ к вашей виртуальной сети Azure и сравнивает различия между интегрированную среду Сценариев и глобальной службы Logic Apps.

> [!NOTE]
> Приложения логики, триггеры, встроенные, встроенных действий и соединителей, которые выполняются в интегрированной среде Сценариев использования ценовой план отличается от тарифный план с оплатой. Дополнительные сведения см. на странице с [ценами на Logic Apps](../logic-apps/logic-apps-pricing.md).

## <a name="isolated-versus-global"></a>Изолированная и глобальная среды

При создании среды службы интеграции (ISE) в Azure, можно выбрать виртуальной сети Azure, которой вы хотите *внедрить* вашей интегрированной среды Сценариев. Azure затем внедряет или развертывает закрытого экземпляра служба Logic Apps в виртуальной сети. Это действие создает изолированную среду, в которой можно создавать и выполнять приложения логики на выделенных ресурсах. При создании приложения логики, как расположение для вашего приложения, которая предоставляет приложению логики прямой доступ к виртуальной сети и ресурсы в этой сети выбирается вашей интегрированной среды Сценариев.

Приложения логики в среде ISE имеют тот же пользовательский интерфейс и аналогичные возможности, что и глобальная служба Logic Apps. Не только можно использовать же встроенные триггеры, встроенные действия и соединителей из глобальной службы Logic Apps, но можно также использовать соединители интегрированной среды сценариев. Например Вот некоторые стандартные соединители, которые предлагают версий, которые выполняются в интегрированную среду Сценариев:

* Хранилище BLOB-объектов, хранилище файлов и хранилище таблиц Azure
* Очереди Azure, Служебная шина Azure, Центры событий Azure и IBM MQ
* FTP и SFTP-SSH
* SQL Server, Хранилище данных SQL, Azure Cosmos DB
* AS2, X12 и EDIFACT

Различие между соединителями среды ISE и другими соединителями заключается в местах, где запускаются триггеры и действия.

* В интегрированной среды Сценариев, встроенных триггеров и действий, таких как HTTP, всегда выполняются в одной интегрированной среды Сценариев, как приложение логики и отображения **Core** метки.

  ![Выберите «Core» встроенных триггеров и действий](./media/connect-virtual-network-vnet-isolated-environment-overview/select-core-built-in-actions-triggers.png)

* Соединители, которые выполняются в интегрированную среду Сценариев публично размещенные находятся версии, доступные в глобальной службе Logic Apps. Для соединителей, которые предлагают две версии соединителей с помощью **интегрированной среды Сценариев** метки, всегда выполняются в одной интегрированной среде Сценариев ваше приложение логики. Соединители без надписи **ISE** выполняются в глобальной службе Logic Apps.

  ![Выбор соединителей ISE](./media/connect-virtual-network-vnet-isolated-environment-overview/select-ise-connectors.png)

### <a name="access-to-on-premises-data-sources"></a>Доступ к локальным источникам данных

Для локальных систем, которые подключены к виртуальной сети Azure внедрить интегрированную среду Сценариев в этой сети, поэтому приложения логики можно непосредственно обращаться к системам с помощью любого из этих элементов:

* Соединитель версии интегрированной среды Сценариев для этой системы, например, SQL Server
  
* Действие HTTP
  
* Настраиваемый соединитель

  * Если у вас есть настраиваемые соединители, которые требуется локальный шлюз данных, и вы создали эти соединители за пределами интегрированную среду Сценариев, эти соединители можно также использовать в приложениях логики в интегрированную среду Сценариев.
  
  * Настраиваемые соединители, созданные в интегрированную среду Сценариев не работают с локальным шлюзом данных. Тем не менее эти соединители можно непосредственно получить доступ к локальным источникам данных, подключенных к виртуальной сети, размещение в интегрированную среду Сценариев. Таким образом приложение логики в интегрированную среду Сценариев скорее не нуждается шлюз данных при взаимодействии с этими ресурсами.

Для локальных систем, которые не подключены к виртуальной сети или нет соединителей версии интегрированной среды Сценариев, необходимо сначала [настроить локальный шлюз данных](../logic-apps/logic-apps-gateway-install.md) перед ваша логика приложения может подключиться к этим системам.

## <a name="integration-accounts-with-ise"></a>Учетные записи интеграции в ISE

Учетные записи интеграции с приложениями логики можно использовать в среде службы интеграции. Тем не менее эти учетные записи интеграции должны использовать *ту же ISE*, что и связанные приложения логики. Приложения логики в среде ISE могут ссылаться только на те учетные записи интеграции, которые находятся в одной с ними среде ISE. При создании учетной записи интеграции можно выбрать среду ISE в качестве расположения учетной записи интеграции.

# <a name="create-and-manage-trading-partner-agreements-in-azure-logic-apps"></a>Создавайте и управляйте соглашениями о торговых партнерах в приложениях Azure Logic Apps

*Соглашение о* [торговом партнере](../logic-apps/logic-apps-enterprise-integration-partners.md) 
помогает организациям и компаниям беспрепятственно общаться друг с другом, определяя конкретный отраслевой протокол для использования при обмене сообщениями между предприятиями (B2B). Соглашения обеспечивают общие преимущества, например:

* Позволяет организациям обмениваться информацией с помощью хорошо известного формата.
* Повышение эффективности при проведении B2B-транзакций.
* Легко создавать, управлять и использовать для создания корпоративных интеграционных решений.

В этой статье показано, как создать соглашение AS2, EDIFACT или X12, которое можно использовать при создании корпоративных интеграционных решений для сценариев B2B с помощью [пакета корпоративных интеграций](../logic-apps/logic-apps-enterprise-integration-overview.md) и [приложений логики Azure.](../logic-apps/logic-apps-overview.md) После создания соглашения можно использовать разъемы AS2, EDIFACT или X12 для обмена B2B-сообщениями.

Для создания соглашений об обмене сообщениями RosettaNet [см.](../logic-apps/logic-apps-enterprise-integration-rosettanet.md)

## <a name="prerequisites"></a>Предварительные требования

* Подписка Azure. Если у вас еще нет подписки Azure, [получите бесплатную учетную запись Azure](https://azure.microsoft.com/free/).

* Учетная [запись интеграции](../logic-apps/logic-apps-enterprise-integration-create-integration-account.md) для хранения вашего соглашения и других артефактов B2B. Эта учетная запись интеграции должна быть связана с подпиской Azure.

* По крайней мере, два [торговых партнера,](../logic-apps/logic-apps-enterprise-integration-partners.md) которые вы уже создали в вашем интеграционном счете. Для соглашения требуется как партнер-хозяин, так и гостевой партнер. Оба партнера должны использовать тот же квалификатор "бизнес-идентичности", как и соглашение, которое вы хотите создать, такие как AS2, X12 или EDIFACT.

* Дополнительно: приложение логики, где вы хотите использовать ваше соглашение и триггер, который запускает рабочий процесс приложения логики. Чтобы просто создать учетную запись интеграции и артефакты B2B, вам не нужно приложение логики. Однако, прежде чем приложение логики сможет использовать артефакты B2B в учетной записи интеграции, необходимо связать учетную запись интеграции с приложением логики. Если вы не работали с приложениями логики, см. руководства по [Azure Logic Apps](../logic-apps/logic-apps-overview.md) и [созданию первого приложения логики](../logic-apps/quickstart-create-first-logic-app-workflow.md).

## <a name="create-agreements"></a>Создание соглашений

1. Войдите на [портал Azure](https://portal.azure.com).
В главном меню Azure выберите **Все службы**. В поле поиска введите "интеграцию" в качестве фильтра. Из результатов выберите этот ресурс: **Интеграционные счета**

   ![Поиск учетной записи интеграции](./media/logic-apps-enterprise-integration-agreements/find-integration-accounts.png)

1. В **учетных записях интеграции**выберите учетную запись интеграции, где вы хотите создать соглашение.

   ![Выбор учетной записи интеграции для создания соглашения](./media/logic-apps-enterprise-integration-agreements/select-integration-account.png)

1. В правой панели, под **компонентами,** выберите плитку **Соглашений.**

   ![Выберите "Соглашения"](./media/logic-apps-enterprise-integration-agreements/agreement-1.png)

1. В разделе **Соглашения** щелкните **Добавить**. В панели **Add** предоставьте информацию о своем соглашении, например:

   ![Кнопка "Добавить"](./media/logic-apps-enterprise-integration-agreements/agreement-2.png)

   | Свойство | Обязательно | Значение | Описание |
   |----------|----------|-------|-------------|
   | **Название** | Да | <*соглашение-имя*> | Название вашего соглашения |
   | **Тип соглашения** | Да | **AS2**, **X12**, или **EDIFACT** | Тип протокола для вашего соглашения. При создании файла соглашения содержимое этого файла должно соответствовать типу соглашения. | |  
   | **"Host Partner" (Главный партнер)** | Да | <*хост-партнер-имя*> | Партнер-хозяин представляет организацию, которая определяет соглашение |
   | **"Host Identity" (Идентификатор главного партнера)** | Да | <*хост-партнер-идентификатор*> | Идентификатор партнера-хозяина |
   | **"Guest Partner" (Партнер-гость)** | Да | <*гость-партнер-имя*> | Гостевой партнер представляет организацию, у которой есть деловые отношения с главным партнером. |
   | **Guest Identity** | Да | <*гость-партнер-идентификатор*> | Идентификатор гостевого партнера |
   | **"Receive Settings" (Параметры получения)** | Различается | Различается | Эти свойства указывают, как партнер-хозяин получает все входящие сообщения от гостевого партнера в соглашении. Для получения дополнительной информации см. <p>- [Настройки сообщений AS2](../logic-apps/logic-apps-enterprise-integration-as2-message-settings.md) <br>- [Настройки сообщений EDIFACT](logic-apps-enterprise-integration-edifact.md) <br>- [Настройки сообщений X12](logic-apps-enterprise-integration-x12.md) |
   | **"Send Settings" (Параметры отправки)** | Различается | Различается | Эти свойства указывают, как партнер-хозяин отправляет все исходящие сообщения гостевому партнеру в соглашении. Для получения дополнительной информации см. <p>- [Настройки сообщений AS2](../logic-apps/logic-apps-enterprise-integration-as2-message-settings.md) <br>- [Настройки сообщений EDIFACT](logic-apps-enterprise-integration-edifact.md) <br>- [Настройки сообщений X12](logic-apps-enterprise-integration-x12.md) |
   |||||

1. Когда вы закончите создание соглашения, на странице **Добавить,** выберите **OK**и вернитесь к учетной записи интеграции.

   В списке **Соглашений** теперь отображается ваше новое соглашение.

## <a name="edit-agreements"></a>Отстеченные соглашения

1. На [портале Azure](https://portal.azure.com)в главном меню Azure выберите **все сервисы.**

1. В поле поиска введите "интеграцию" в качестве фильтра. Из результатов выберите этот ресурс: **Интеграционные счета**

1. В **учетных записях интеграции**выберите учетную запись интеграции, которая имеет соглашение, которое вы хотите отсеить.

1. В правой панели, под **компонентами,** выберите плитку **Соглашений.**

1. В соответствии с **соглашениями,** выберите соглашение и **отображайте.**

1. Сделайте, а затем сохраните ваши изменения.

## <a name="delete-agreements"></a>Удаление соглашений

1. На [портале Azure](https://portal.azure.com)в главном меню Azure выберите **все сервисы.**

1. В поле поиска введите "интеграцию" в качестве фильтра. Из результатов выберите этот ресурс: **Интеграционные счета**

1. В **учетных записях интеграции**выберите учетную запись интеграции, которая имеет соглашение, которое вы хотите удалить.

1. В правой панели, под **компонентами,** выберите плитку **Соглашений.**

1. В соответствии с **соглашениями,** выберите соглашение и выберите **Удалить**.

1. Подтвердите удаление соглашения.

# <a name="validate-xml-for-b2b-enterprise-integration-in-azure-logic-apps-with-enterprise-integration-pack"></a>Проверка XML для интеграции с предприятием B2B в Azure Logic Apps с помощью Пакета интеграции Enterprise

Часто в сценариях B2B торговые партнеры в соглашении должны убедиться, что сообщения, которыми они обмениваются, действительны, прежде чем начать обработку данных. Вы можете проверить документы на предопределенной схеме, используя действие проверки XML, которое доступно в пакете корпоративной интеграции.

## <a name="prerequisites"></a>Предварительные требования

* Подписка Azure. Если у вас еще нет подписки, [зарегистрируйтесь на бесплатную учетную запись Azure.](https://azure.microsoft.com/free/)

* Пустое или существующее приложение логики, где вы хотите использовать действие проверки XML. Если вы не работали с приложениями логики, см. руководства по [Azure Logic Apps](../logic-apps/logic-apps-overview.md) и [созданию первого приложения логики](../logic-apps/quickstart-create-first-logic-app-workflow.md).

* [Интеграционная учетная запись,](../logic-apps/logic-apps-enterprise-integration-create-integration-account.md) связанная с подпиской Azure, связана с логическим приложением, в котором планируется использовать действие проверки XML, и содержит схему, которую вы хотите использовать для проверки содержимого XML. Приложение логики и учетная запись интеграции должны существовать в одном месте или в регионе Azure.

## <a name="add-xml-validation-action"></a>Добавить действие проверки XML

1. На [портале Azure](https://portal.azure.com)откройте логическое приложение в Logic App Designer.

1. Если у вас есть пустое приложение логики, в Logic `HTTP request` App Designer, в поле поиска, введите в качестве фильтра, и выберите **Когда запрос HTTP получил** триггер. В противном случае перейдите к следующему шагу.

1. Под последним шагом в рабочем процессе выберите **новый шаг.**

   Чтобы добавить действие между существующими шагами, переместите указатель на**+** стрелку, которая соединяет эти шаги так, чтобы появится знак плюс () Выберите этот знак плюс, а затем выберите **Добавить действие.**

1. В разделе **Выберите действие** выберите **Встроенный**. В поле поиска введите `xml validation` в качестве фильтра. Из списка действий выберите **проверку XML.**

   ![Найти и выбрать действие "Проверка XML"](./media/logic-apps-enterprise-integration-xml-validation/select-xml-validation-action.png)

1. Чтобы указать содержимое XML, которое необходимо проверить, щелкните внутри окна **Содержимого,** чтобы создать список динамических содержимого.

   ![Открыть список динамического контента](./media/logic-apps-enterprise-integration-xml-validation/open-dynamic-content-list.png)

   В списке динамических содержимого отображают маркеры свойств, отображающие выходы из предыдущих этапов рабочего процесса. Если список не показывает ожидаемое свойство, проверьте триггер или направление действия, можно ли выбрать **Подробнее.**

1. Из списка динамических содержимого выберите свойство, которое имеет содержимое, которое вы хотите проверить.

   Этот пример выбирает выход **тела** из триггера.

   ![Выберите содержимое для проверки](./media/logic-apps-enterprise-integration-xml-validation/select-content-to-validate.png)

1. Чтобы указать схему, которую необходимо использовать для проверки, откройте список **имен Schema** и выберите схему проверки, добавленную в связанную учетную запись интеграции.

   ![Выберите схему для проверки](./media/logic-apps-enterprise-integration-xml-validation/select-validation-schema.png)

1. Сохраните приложение логики.

   Теперь вы закончили настройку проверки. В приложении в реальном мире можно хранить проверенные данные в приложении line-of-business (LOB), например SalesForce. Чтобы отправить проверенные выходные данные в Salesforce, добавьте действие.

1. Чтобы проверить действие проверки, можно отправить запрос на запуск рабочего процесса приложения логики.

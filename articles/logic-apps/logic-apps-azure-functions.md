# <a name="call-azure-functions-from-azure-logic-apps"></a>Вызов функций Azure из приложений логики Azure

При запуске кода, выполняемого определенной задачей в логических приложениях, можно создать собственную функцию с помощью [Azure Functions.](../azure-functions/functions-overview.md) Эта служба помогает создавать функции Node.js, C и F-функции, чтобы не создавать полное приложение или инфраструктуру для запуска кода. Также можно [вызывать приложения логики из функций](#call-logic-app). Служба "Функции Azure" обеспечивает бессерверные вычисления в облаке и является полезной при выполнении задач, приведенных ниже.

* Расширение режимов работы приложений логики с помощью функций в Node.js или C#.
* Выполнение вычислений в рабочем процессе приложения логики.
* Применение расширенных функции форматирования или вычисления полей в приложениях логики.

Чтобы запустить фрагменты кода без создания функций Azure, узнайте, как [добавлять и запускать встроенный код.](../logic-apps/logic-apps-add-run-inline-code.md)

> [!NOTE]
> Интеграция между логическими приложениями и функциями Azure в настоящее время не работает с включенными слотами.

## <a name="prerequisites"></a>Предварительные требования

* Подписка Azure. Если у вас еще нет подписки Azure, [зарегистрируйтесь для получения бесплатной учетной записи Azure](https://azure.microsoft.com/free/).

* Функциональное приложение Azure, которое является контейнером для функций Azure, наряду с функцией Azure. При отсутствии приложения-функции [создайте его](../azure-functions/functions-create-first-azure-function.md). Затем можно создать функцию либо за пределами приложения логики на портале Azure, либо [из приложения логики](#create-function-designer) в Logic App Designer.

* При работе с логическими приложениями те же требования применяются к функциональным приложениям и функциям, независимо от того, являются ли они существующими или новыми:

  * Приложение функции и приложение логики должны использовать ту же подписку Azure.

  * Новые функциональные приложения должны использовать как .NET, либо JavaScript в качестве стека времени выполнения. При добавлении новой функции в существующие функциональные приложения можно выбрать либо C-е, либо JavaScript.

  * Функция использует шаблон **триггера HTTP.**

    Этот шаблон триггера HTTP поддерживает содержимое приложения логики с типом `application/json`. При добавлении функции Azure в логическое приложение Конструктор приложения Logic показывает пользовательские функции, созданные из этого шаблона в подписке Azure.

  * Функция не использует пользовательские маршруты, если только вы не определили [определение OpenAPI](../azure-functions/functions-openapi-definition.md) (ранее известное как [файл Swagger).](https://swagger.io/)

  * Если у вас есть определение OpenAPI для вашей функции, Дизайнер логических приложений дает вам более богатый опыт, когда ваша работа с параметрами функции. Прежде чем приложение логики сможет найти и получить доступ к функции с определениями OpenAPI, необходимо [настроить приложение-функцию, выполнив следующие действия](#function-swagger).

* Приложение логики, в которое необходимо добавить функцию, включая [триггер](../logic-apps/logic-apps-overview.md#logic-app-concepts) в качестве первого шага.

  Прежде чем добавить действия, которые работают, приложение логики должно начать с триггера. Если вы не работали с приложениями логики, см. руководства по [Azure Logic Apps](../logic-apps/logic-apps-overview.md) и [созданию первого приложения логики](../logic-apps/quickstart-create-first-logic-app-workflow.md).

## <a name="access-property-values-inside-http-requests"></a>Доступ к значениям свойств внутри HTTP-запросов

Функции веб-перехватчика могут принимать HTTP-запросы в качестве входных и передавать эти запросы другим функциям. Например, несмотря на то, что служба Logic Apps имеет [функции преобразования значений DateTime](../logic-apps/workflow-definition-language-functions-reference.md), в этом базовом примере функции JavaScript отображается доступ к свойству внутри запрашиваемого объекта, который передается функции и выполняет операции над значением свойства. Чтобы получить доступ к свойствам, которые находятся внутри объекта, в этом примере используется [оператор точка (.)](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Operators/Property_accessors).

```javascript
function convertToDateString(request, response){
   var data = request.body;
   response = {
      body: data.date.ToDateString();
   }
}
```

Ниже приведены процессы, которые выполняются внутри функции.

1. Функция создает переменную `data` и присваивает ей объект `body`, который находится внутри объекта `request`. Чтобы ссылаться на объект `body` внутри объекта `request`, функция использует оператор точку (.).

   ```javascript
   var data = request.body;
   ```

1. Теперь функция может получить доступ к свойству `date` через переменную `data` и изменить значение свойства с типа DateTime на тип DateString с помощью вызова функции `ToDateString()`. Также в ответе функции она возвращает результат через свойство `body`.

   ```javascript
   body: data.date.ToDateString();
   ```

После создания функции Azure можно выполнить шаги из раздела [Добавление существующей функций в приложения логики](#add-function-logic-app).

<a name="create-function-designer"></a>

## <a name="create-functions-inside-logic-apps"></a>Создание встроенных функций приложений логики

Прежде чем создать функцию Azure, начиная с приложения логики с помощью Logic App Designer, сначала необходимо создать приложение функции Azure, которое является контейнером для ваших функций. При отсутствии приложения-функции создайте его. См. статью [Создание первой функции на портале Azure](../azure-functions/functions-create-first-azure-function.md).

1. На [портале Azure](https://portal.azure.com)откройте логическое приложение в Logic App Designer.

1. Чтобы создать и добавить функцию, выполните шаг, который применим к вашему сценарию.

   * Под последним шагом в рабочем процессе приложения логики выберите **новый шаг.**

   * Между существующими шагами в рабочем процессе приложения логики переместите мышь по стрелке, выберите знак «плюс» (кв), а затем выберите **«Добавить действие».**

1. В поле поиска введите "Функции Azure" в качестве условия фильтра. Из списка действий выберите действие **функции Azure,** например:

   ![Поиск "Функции Azure"](./media/logic-apps-azure-functions/find-azure-functions-action.png)

1. Выберите приложение-функцию из списка приложений-функций. После открытия списка действий выберите это действие: **Создайте новую функцию**

   ![Выбор приложения-функции](./media/logic-apps-azure-functions/select-function-app-create-function.png)

1. Определите функцию в редакторе определения функции.

   1. Введите имя функции в поле **Имя функции**.

   1. В окне **кода** добавьте код в шаблон функции, включая ответ и полезную нагрузку, которые вы хотите вернуть в логическое приложение после завершения работы функции. Когда все будет готово, выберите **Создать**.

   Пример:

   ![Определение функции](./media/logic-apps-azure-functions/add-code-function-definition.png)

   В коде шаблона * `context` объект* ссылается на сообщение, которое приложение логики отправляет через поле **Органа запроса** на более позднем этапе. Чтобы получить доступ к свойствам объекта `context` изнутри вашей функции, используйте этот синтаксис:

   `context.body.<property-name>`

   Например, чтобы указать ссылку на свойство `content` внутри объекта `context`, используйте этот синтаксис:

   `context.body.content`

   Код шаблона также включает переменную `input`, которая сохраняет значение из параметра `data`, чтобы ваша функция могла выполнять с ним операции. Внутри функций JavaScript переменная `data` также является ярлыком для `context.body`.

   > [!NOTE]
   > Свойство `body` здесь применяется к объекту `context` и не совпадает с токеном **Текст** из выходных данных действия, который вы также можете передать своей функции.

1. В поле **Текст запроса** укажите входные данные функции, которые должны быть отформатированы как объект JSON (нотация объектов JavaScript).

   Эти входные данные представляют собой *объект контекста* или сообщение, которое приложение логики отправляет в функцию. Если щелкнуть поле **Текст запроса**, откроется список динамического содержимого, что позволит выбрать токены для выходных данных с предыдущих шагов. В этом примере указывается, что полезная `content` нагрузка контекста содержит свойство, названное значение **от** маркера от триггера электронной почты.

   ![Пример "Текста запроса": полезные данные объекта контекста](./media/logic-apps-azure-functions/function-request-body-example.png)

   Здесь объект контекста не приводится как строка, что позволяет добавить содержимое объекта напрямую в полезные данные JSON. Тем не менее, если объект контекста не является токеном JSON, который передает строку, либо объектом или массивом JSON, появится сообщение об ошибке. Таким образом, если в этом примере вместо этого используется маркер **Полученного Времени,** можно отбросить объект контекста в строку, добавив двойные отметки.

   ![Приведение объекта контекста к состоянию строки](./media/logic-apps-azure-functions/function-request-body-string-cast-example.png)

1. Чтобы указать другие детали, такие как метод использования, заголовки запросов или параметры запроса или проверку подлинности, откройте новый список **параметров Идможное** и выберите параметры, которые вы хотите. Для проверки подлинности ваши параметры различаются в зависимости от выбранной функции. Смотрите [Включить аутентификацию для функций Azure](#enable-authentication-functions).

В этом примере указывается, что полезная `content` нагрузка контекста содержит свойство, названное значение **от** маркера от триггера электронной почты.

Тем не менее, если объект контекста не является токеном JSON, который передает строку, либо объектом или массивом JSON, появится сообщение об ошибке. Таким образом, если в этом примере использовался токен **Время получения**, вы можете привести объект контекста к состоянию строки, добавив двойные кавычки:

## <a name="call-logic-apps-from-azure-functions"></a>Вызов логических приложений из функций Azure

Если вы хотите вызвать приложение логики изнутри функции Azure, оно должно запускаться триггером, который предоставляет вызываемую конечную точку. Например, вы можете запустить приложение логики с помощью триггера **HTTP**, **Запрос**, **Очереди Azure** или **Сетка событий**. Внутри функции отправьте HTTP-запрос POST на URL-адрес триггера и включите полезные данные, необходимые для обработки приложения логики. Дополнительные сведения см. в разделе [Вызов, активация и вложение приложений логики](../logic-apps/logic-apps-http-endpoint.md).

<a name="enable-authentication-functions"></a>

## <a name="enable-authentication-for-azure-functions"></a>Включить аутентификацию для функций Azure

Для проверки подлинности доступа к ресурсам в других клиентах Active Directory (Azure AD) без входиныи и предоставления учетных данных или секретов приложение логики может использовать [управляемую идентификацию](../active-directory/managed-identities-azure-resources/overview.md) (ранее известную как Идентификатор управляемой службы или MSI). Azure управляет этим удостоверением за вас и помогает защитить учетные данные, потому что вам не нужно предоставлять или сменять секреты. Узнайте больше об [службах Azure, которые поддерживают управляемые идентификаторы для проверки подлинности Azure AD.](../active-directory/managed-identities-azure-resources/services-support-managed-identities.md#azure-services-that-support-azure-ad-authentication)

Если вы настроили приложение логики для использования установленного системой удостоверения личности или созданного вручную удостоверения пользователя, функции Azure в приложении логики также могут использовать ту же итог для проверки подлинности. Для получения дополнительной информации о поддержке аутентификации функций Azure в логических приложениях [см.](../logic-apps/logic-apps-securing-a-logic-app.md#add-authentication-outbound)

Чтобы настроить и использовать управляемую идентификацию с помощью функции, выполните следующие действия:

1. Включить управляемое удостоверение в приложении логики и настроить доступ к целевому ресурсу. Просмотрите [доступ к ресурсам Azure, используя управляемые идентификаторы в приложениях Azure Logic Apps.](../logic-apps/create-managed-service-identity.md)

1. Включите аутентификацию в приложении функции и функции Azure, выдвинув следующие действия:

   * [Настройка анонимной аутентификации в вашей функции](#set-authentication-function-app)
   * [Настройка аутентификации Azure AD в приложении функции](#set-azure-ad-authentication)

<a name="set-authentication-function-app"></a>

### <a name="set-up-anonymous-authentication-in-your-function"></a>Настройка анонимной аутентификации в вашей функции

Чтобы использовать управляемую идентификацию приложения логики в функции Azure, вы установили уровень аутентификации вашей функции на анонимный. В противном случае приложение логики бросает ошибку "BadRequest".

1. На [портале Azure](https://portal.azure.com)найдите и выберите приложение функции. Эти шаги используют "FabrikamFunctionApp" в качестве примера функции приложения.

1. На панели приложения функции выберите **функции Платформы.** Под **инструментами разработки**выберите **Расширенные инструменты (Kudu).**

   ![Откройте передовые инструменты для Kudu](./media/logic-apps-azure-functions/open-advanced-tools-kudu.png)

1. На заглавной панели веб-сайта Kudu, из меню **Приконсоль Debug,** выберите **CMD**.

   ![Из меню консоли отладки выберите опцию "CMD"](./media/logic-apps-azure-functions/open-debug-console-kudu.png)

1. После того, как следующая страница появится, из списка папок, выберите **сайт** > **wwwroot** > *вашей функции*. Эти шаги используют функцию «FabrikamAzureFunction» в качестве примера.

   ![Выберите "сайт" > "wwwroot" > вашей функции](./media/logic-apps-azure-functions/select-site-wwwroot-function-folder.png)

1. Откройте `function.json` файл для редактирования.

   ![Нажмите на кнопку "функция.json" файл](./media/logic-apps-azure-functions/edit-function-json-file.png)

1. В `bindings` объекте проверьте, существует ли свойство. `authLevel` Если свойство существует, установите `anonymous`значение свойства. В противном случае добавьте это свойство и установите значение.

   ![Добавить свойство "authLevel" и установить на "анонимный"](./media/logic-apps-azure-functions/set-authentication-level-function-app.png)

1. Когда вы закончите, сохраните настройки, а затем перейти к следующему разделу.

<a name="set-azure-ad-authentication"></a>

### <a name="set-up-azure-ad-authentication-for-your-function-app"></a>Настройка аутентификации Azure AD для приложения функции

Перед тем, как приступить к выполнению этой задачи, найдите и отложите эти значения для последующего использования:

* Идентификатор объекта, сгенерированный для установленного системой идентификатора, представляющего приложение логики

  * Чтобы создать этот идентификатор объекта, [включите системную идентификацию приложения логики.](../logic-apps/create-managed-service-identity.md#azure-portal-system-logic-app)

  * В противном случае, чтобы найти этот идентификатор объекта, откройте приложение логики в Logic App Designer. В меню приложения логики в **настройках**выберите**назначенную систему** **идентификации.** > 

* Идентификатор каталога для вашего клиента в Active Directory Azure (Azure AD)

  Чтобы получить идентификатор каталога вашего клиента, вы можете запустить команду [`Get-AzureAccount`](https://docs.microsoft.com/powershell/module/servicemanagement/azure/get-azureaccount) Powershell. Или, на портале Azure, выполните следующие действия:

  1. На [портале Azure](https://portal.azure.com)найдите и выберите приложение функции.

  1. Найдите и выберите арендатора Azure AD. Эти шаги используют "Fabrikam" в качестве примера арендатора.

  1. В меню арендатора, под **управлением,** выберите **Свойства**.

  1. Например, копируйте идентификатор каталога и сохраните этот идентификатор для последующего использования.

     ![Найти и скопировать идентификатор каталога AD-идентификатора Aure AD](./media/logic-apps-azure-functions/azure-active-directory-tenant-id.png)

* Идентификатор ресурса для целевого ресурса, к который требуется получить доступ

  * Чтобы найти эти иные иные данные ресурсов, просмотрите [службы Azure, поддерживающие Azure AD.](../active-directory/managed-identities-azure-resources/services-support-managed-identities.md#azure-services-that-support-azure-ad-authentication)

  > [!IMPORTANT]
  > Этот идентификатор ресурсов должен точно соответствовать значению, которое ожидает Azure AD, включая любые необходимые слэши.

  Этот идентификатор ресурса также является тем же значением, которое вы позже используете в свойстве **аудитории** при [настройке действия функции для использования установленного системой удостоверения.](../logic-apps/create-managed-service-identity.md#authenticate-access-with-identity)

Теперь вы готовы настроить аутентификацию Azure AD для приложения функции.

1. На [портале Azure](https://portal.azure.com)найдите и выберите приложение функции.

1. На панели приложения функции выберите **функции Платформы.** В **сети**, выберите **аутентификация / Авторизация**.

   ![Просмотр настроек проверки подлинности и авторизации](./media/logic-apps-azure-functions/view-authentication-authorization-settings.png)

1. Измените настройки **аутентификации службы приложений** **на On**. Из **действия, которое необходимо принять, когда запрос не является проверенным списком,** выберите **войти в систему с помощью Active Directory Azure.** В разделе **Поставщики проверки подлинности** выберите **Azure Active Directory**.

   ![Включите аутентификацию с помощью Azure AD](./media/logic-apps-azure-functions/turn-on-authentication-azure-active-directory.png)

1. На панели **настроек active Directory Настройки Azure** выполните следующие действия:

   1. Установить **режим управления** для **Расширенный**.

   1. В свойстве **Идентификаторклиента** введите идентификатор объекта для системного удостоверения приложения логики.

   1. В свойстве **Url-адреса Эмитента** введите `https://sts.windows.net/` URL-адрес и приведите идентификатор каталога AD-идентификатора AD..

      `https://sts.windows.net/<Azure-AD-tenant-directory-ID>`

   1. В свойстве **«Разрешенные маркеры»** введите идентификатор ресурса для целевого ресурса, к который необходимо получить доступ.

      Этот идентификатор ресурса — это то же значение, которое вы позже используете в свойстве **аудитории** при [настройке действия функции для использования установленного системой удостоверения.](../logic-apps/create-managed-service-identity.md#authenticate-access-with-identity)

   На данный момент ваша версия похожа на этот пример:

   ![Настройки проверки подлинности активных каталогов Azure](./media/logic-apps-azure-functions/azure-active-directory-authentication-settings.png)

1. Закончив, нажмите кнопку **OK**.

1. Вернитесь к Logic App Designer и следуйте [за шагами по аутентификации доступа с управляемой идентификацией.](../logic-apps/create-managed-service-identity.md#authenticate-access-with-identity)

# <a name="azure-serverless-overview-for-building-cloud-based-apps-and-solutions-with-azure-logic-apps-and-azure-functions"></a>Azure Serverless: Обзор для создания облачных приложений и решений с функциями логики Azure и функциями Azure

[Приложения без серверов](https://azure.microsoft.com/solutions/serverless/) предлагают такие преимущества, как увеличение скорости разработки, снижение кода, простота и масштабирование. В этой статье рассматриваются различные атрибуты решений без серверов и безсерверных предложений Azure.

## <a name="what-is-serverless"></a>Что значит "без сервера"?

Сервер не означает, что нет серверов, а разработчики не должны беспокоиться о серверах. Традиционная разработка приложений по большей части связана с решением задач масштабирования, размещения и мониторинга решений, которые бы соответствовали требованиям приложения. С сервером, эти вопросы рассматриваются как часть решения. Кроме того, приложения без серверов выставляются по плану, основанному на потреблении. Если приложение никогда не используется, плата не взимается. Эти функции помогают разработчикам сосредоточиться исключительно на бизнес-логике решения.

Основными службами Azure для серверов являются [приложения Azure Logic Apps](https://azure.microsoft.com/services/logic-apps/) и [функции Azure.](https://azure.microsoft.com/services/functions/) Оба решения следуют ранее описанным принципам и помогают разработчикам создавать надежные облачные приложения с минимальным кодом.

## <a name="what-is-azure-logic-apps"></a>Что такое Azure Logic Apps

[Приложения Azure Logic Apps](logic-apps-overview.md) предоставляют способ упрощения и реализации масштабируемых интеграций и рабочих процессов в облаке. Эта услуга предоставляет визуальный дизайнер для моделирования и автоматизации процесса в виде ряда шагов, называемых рабочим процессом. Есть много [разъемов в облачных сервисах](../connectors/apis-list.md) и в предприимчивых системах, которые быстро подключают приложение без серверов к другим AA. Каждое приложение логики начинается с триггера, например"Когда учетная запись добавляется в Dynamics CRM". После срабатывания триггера в рабочем процессе могут работать комбинации действий, преобразований и условной логики. Логические приложения — это отличный выбор при организации различных функций Azure в процессе, особенно когда процесс требует взаимодействия с внешней системой или API.

Чтобы приступить к работе с Logic Apps, начните с [создания первого приложения логики](quickstart-create-first-logic-app-workflow.md). Для получения дополнительной технической информации о Logic Apps см. [developer reference](logic-apps-workflow-definition-language.md)

## <a name="what-is-azure-functions"></a>Что такое Функции Azure?

Azure Functions — это служба для легкого запуска фрагментов кода или «функций» в облаке. Вы можете написать только код, необходимый для текущей проблемы, не беспокоясь о целом приложении или необходимой инфраструктуре. Функции делают разработку более продуктивной. Кроме того, вы можете использовать предпочтительный язык программирования, например C#, F#, Node.js, Python или PHP. Вы платите только за время запуска кода и масштабов Azure по мере необходимости.

Чтобы начать работу с функциями Azure, начните с [создания первой функции Azure.](../azure-functions/functions-create-first-azure-function.md) Для получения дополнительной технической [developer reference](../azure-functions/functions-reference.md)информации о функциях см.

## <a name="how-can-i-build-and-deploy-serverless-apps-in-azure"></a>Как построить и развернуть приложения без серверов в Azure?

Azure предоставляет богатые инструменты для разработки, развертывания и управления приложениями без серверов. Вы можете создавать приложения непосредственно на портале Azure, с [инструментами в Visual Studio](logic-apps-serverless-get-started-vs.md)или [Visual Studio Code.](quickstart-create-logic-apps-visual-studio-code.md) После создания приложения можно [быстро развернуть это приложение с помощью шаблонов Azure Resource Manager.](logic-apps-deploy-azure-resource-manager-templates.md) Azure также обеспечивает мониторинг, к которому можно получить доступ через портал Azure, через API или SDK, или с интегрированным инструментом для журналов Azure Monitor и Application Insight.

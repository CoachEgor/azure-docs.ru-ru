# <a name="reference-for-as2-message-settings-in-azure-logic-apps-with-enterprise-integration-pack"></a>Справочник по параметрам сообщения AS2 в Azure Logic Apps с помощью пакета интеграции Enterprise

Этот справочник описывает свойства, которые можно задать для указания того, как соглашение AS2 обрабатывает сообщения отправленных и полученных между торговыми партнерами. Настройте эти свойства в соответствии с вашим соглашением с партнером, которая обменивается сообщениями с вами.

<a name="AS2-incoming-messages"></a>

## <a name="as2-receive-settings"></a>Параметры получения AS2

![Выберите «Параметры получения»](./media/logic-apps-enterprise-integration-as2-message-settings/receive-settings.png)

| Свойство | Обязательно для заполнения | Описание |
|----------|----------|-------------|
| **Переопределить свойства сообщения** | Нет | Переопределяет свойства входящих сообщений с параметры свойств. |
| **Сообщение должно быть подписано** | Нет | Указывает, является ли все входящие сообщения должны иметь цифровую подпись. Если требуется подписывание, из **сертификат** выберите существующий общий сертификат гостевого партнера для проверки подписи сообщений. Если у вас нет сертификата, Дополнительные сведения о [Добавление сертификатов](../logic-apps/logic-apps-enterprise-integration-certificates.md). |
| **Сообщение должно быть зашифровано** | Нет | Указывает, необходимо ли шифровать все входящие сообщения. Незашифрованные сообщения будут отклонены. Если требуется шифрование, из **сертификат** выберите существующий закрытый сертификат главного партнера для расшифровки входящих сообщений. Если у вас нет сертификата, Дополнительные сведения о [Добавление сертификатов](../logic-apps/logic-apps-enterprise-integration-certificates.md). |
| **Сообщение должно быть сжато** | Нет | Указывает, должны быть сжаты все входящие сообщения. В противном случае сообщения будут отклонены. |
| **Запретить дублирование Идентификаторов сообщений** | Нет | Указывает, следует ли разрешить сообщения с повторяющиеся идентификаторы. Если запретить повторяющиеся идентификаторы, выберите число дней между проверками. Также можно выбрать, следует ли приостановить дубликаты. |
| **Текст MDN** | Нет | Указывает значение по умолчанию уведомление о состоянии сообщения (MDN), вы должны отправитель сообщения. |
| **Отправить MDN** | Нет | Указывает, следует ли отправлять синхронные MDN для полученных сообщений.  |
| **Отправить подписанное MDN** | Нет | Указывает, следует ли отправлять подписанные MDN для полученных сообщений. Если требуется подписывание, из **алгоритм MIC** выберите алгоритм, используемый для подписания сообщений. |
| **Отправить асинхронное MDN** | Нет | Указывает, следует ли отправлять уведомления о состоянии сообщения в асинхронном режиме. Если вы выбираете асинхронные уведомления о состоянии в **URL-адрес** поле, укажите URL-адрес, на который будет отправлять уведомления о состоянии. |
||||

<a name="AS2-outgoing-messages"></a>

## <a name="as2-send-settings"></a>Параметры отправки AS2

![Выберите «Параметры отправки»](./media/logic-apps-enterprise-integration-as2-message-settings/send-settings.png)

| Свойство | Обязательно для заполнения | Описание |
|----------|----------|-------------|
| **Включить подпись сообщений** | Нет | Указывает, является ли все исходящие сообщения должны иметь цифровую подпись. Если требуется подписывание, выберите следующие значения: <p>-Из **алгоритм подписывания** выберите алгоритм, используемый для подписания сообщений. <br>-Из **сертификат** выберите существующий закрытый сертификат главного партнера для подписания сообщений. Если у вас нет сертификата, Дополнительные сведения о [Добавление сертификатов](../logic-apps/logic-apps-enterprise-integration-certificates.md). |
| **Включить шифрование сообщений** | Нет | Указывает, необходимо ли шифровать все исходящие сообщения. Если требуется шифрование, выберите следующие значения: <p>-Из **алгоритм шифрования** выберите алгоритм открытого сертификата гостевого партнера, используемый для шифрования сообщений. <br>-Из **сертификат** выберите существующий гостевой партнер закрытый сертификат для шифрования исходящих сообщений. Если у вас нет сертификата, Дополнительные сведения о [Добавление сертификатов](../logic-apps/logic-apps-enterprise-integration-certificates.md). |
| **Включить сжатие сообщений** | Нет | Указывает, должны быть сжаты все исходящие сообщения. |
| **Раскрыть заголовки HTTP** | Нет | Помещает HTTP `content-type` заголовка в одну строку. |
| **Передача имени файла в заголовок MIME** | Нет | Указывает, следует ли включать имя файла в заголовке MIME. |
| **Запросить MDN** | Нет | Указывает, следует ли получать уведомления о расположении сообщения (MDN) для всех исходящих сообщений. |
| **Запросить подписанное MDN** | Нет | Указывает, следует ли получать подписанные уведомления о состоянии для всех исходящих сообщений. Если требуется подписывание, из **алгоритм MIC** выберите алгоритм, используемый для подписания сообщений. |
| **Запросить асинхронное MDN** | Нет | Указывает, следует ли получать уведомления о состоянии сообщения асинхронно. Если вы выбираете асинхронные уведомления о состоянии в **URL-адрес** поле, укажите URL-адрес, на который будет отправлять уведомления о состоянии. |
| **Включить NRR** | Нет | Указывает, следует ли требовать неотказуемость получения (NRR). Этот атрибут связи предоставляет свидетельство, решает, получении данных. |
| **Формат алгоритма SHA2** | Нет | Указывает формат алгоритма MIC, используемый для входа в заголовки исходящих сообщений AS2 или MDN |
||||

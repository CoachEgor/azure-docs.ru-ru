# <a name="reference-for-as2-message-settings-in-azure-logic-apps-with-enterprise-integration-pack"></a>Ссылка на настройки сообщений AS2 в приложениях логики Azure с пакетом корпоративной интеграции

Эта ссылка описывает свойства, которые можно установить для указания того, как соглашение AS2 обрабатывает сообщения, отправленные и полученные между торговыми партнерами. Настройка этих свойств на основе соглашения с партнером, который обменивается сообщениями с вами.

<a name="AS2-incoming-messages"></a>

## <a name="as2-receive-settings"></a>Параметры получения AS2

![Выберите "Настройки получения"](./media/logic-apps-enterprise-integration-as2-message-settings/receive-settings.png)

| Свойство | Обязательно | Описание |
|----------|----------|-------------|
| **Override message properties** | нет | Переопределяет свойства входящих сообщений с настройками свойств. |
| **Message should be signed** | нет | Уточняется, должны ли все входящие сообщения быть подписаны в цифровом виде. Если вам требуется подписание из списка **сертификатов,** выберите существующий государственный сертификат гостя для проверки подписи на сообщениях. Если у вас нет сертификата, узнайте больше о [добавлении сертификатов.](../logic-apps/logic-apps-enterprise-integration-certificates.md) |
| **Message should be encrypted** | нет | Уточняется, должны ли быть зашифрованы все входящие сообщения. Незашифрованные сообщения будут отклонены. Если вам требуется шифрование из списка **сертификатов,** выберите существующий частный сертификат партнера-хозяина для расшифровки входящих сообщений. Если у вас нет сертификата, узнайте больше о [добавлении сертификатов.](../logic-apps/logic-apps-enterprise-integration-certificates.md) |
| **Message should be compressed** | нет | Определяется, должны ли все входящие сообщения быть сжаты. В противном случае сообщения будут отклонены. |
| **Дубликаты идентификатора сообщения** | нет | Уточняется, разрешать ли сообщения с дублирующими итоговыми иданными данными. Если вы отдавили дубликаты итогов, выберите количество дней между чеками. Вы также можете выбрать, следует ли приостанавливать дубликаты. |
| **"MDN Text" (Текст MDN)** | нет | Уотрежено уведомление о расположении сообщений по умолчанию (MDN), которое необходимо отправить отправителю сообщения. |
| **"Send MDN" (Отправлять MDN)** | нет | Определяет, отправлять ли синхронные MDN для полученных сообщений.  |
| **"Send signed MDN" (Отправлять MDN с подписью)** | нет | Определяет, отправлять ли подписанные MDN для полученных сообщений. Если требуется подписание из списка **алгоритмов MIC,** выберите алгоритм для подписания сообщений. |
| **"Send asynchronous MDN" (Отправлять асинхронные MDN)** | нет | Определяет, следует ли отправлять MDNs асинхронно. Если вы выберете асинхронные MDN, в поле **URL,** укажите URL, куда отправить MDNs. |
||||

<a name="AS2-outgoing-messages"></a>

## <a name="as2-send-settings"></a>Параметры отправки AS2

![Выберите "Настройки отправки"](./media/logic-apps-enterprise-integration-as2-message-settings/send-settings.png)

| Свойство | Обязательно | Описание |
|----------|----------|-------------|
| **"Enable message signing" (Включить подписание сообщений)** | нет | Уточняется, должны ли все исходящие сообщения быть подписаны в цифровом виде. Если требуется подписание, выберите следующие значения: <p>- Из списка **алгоритмов подписи** выберите алгоритм для подписи для подписи сообщений. <br>- Из списка **сертификатов** выберите существующий частный сертификат партнера-хозяина для подписания сообщений. Если у вас нет сертификата, узнайте больше о [добавлении сертификатов.](../logic-apps/logic-apps-enterprise-integration-certificates.md) |
| **"Enable message encryption" (Включить шифрование сообщений)** | нет | Уточняется, должны ли быть зашифрованы все исходящие сообщения. Если требуется шифрование, выберите следующие значения: <p>- Из списка **алгоритмов шифрования** выберите алгоритм публичного сертификата гостевого партнера для использования для шифрования сообщений. <br>- Из списка **сертификатов** выберите существующий личный сертификат гостевого партнера для шифрования исходящих сообщений. Если у вас нет сертификата, узнайте больше о [добавлении сертификатов.](../logic-apps/logic-apps-enterprise-integration-certificates.md) |
| **Включить сжатие сообщений** | нет | Определяется, должны ли быть сжаты все исходящие сообщения. |
| **"Unfold HTTP headers" (Развернуть заголовки HTTP)** | нет | Помещает заголовок HTTP `content-type` на одну строку. |
| **Передача имени файла в заголовке MIME** | нет | Определяется, следует ли включать имя файла в заголовок MIME. |
| **"Request MDN" (Запрашивать MDN)** | нет | Определяет, следует ли получать уведомления о расположении сообщений (MDN) для всех исходящих сообщений. |
| **"Request signed MDN" (Запрашивать подписанные MDN)** | нет | Определяет, следует ли получать подписанные MDN для всех исходящих сообщений. Если требуется подписание из списка **алгоритмов MIC,** выберите алгоритм для подписания сообщений. |
| **"Request asynchronous MDN" (Запрашивать асинхронные MDN)** | нет | Определяет, следует ли получать MDNs асинхронно. Если вы выберете асинхронные MDN, в поле **URL,** укажите URL, куда отправить MDNs. |
| **"Enable NRR" (Включить NRR)** | нет | Уточняется, следует ли требовать получения неотказных данных (NRR). Этот атрибут связи предоставляет доказательства того, что данные были получены в качестве адресных. |
| **Параметр "Формат алгоритма SHA2"** | нет | Определяет формат алгоритма MIC для использования для подписи в заголовках для исходящих сообщений AS2 или MDN |
||||

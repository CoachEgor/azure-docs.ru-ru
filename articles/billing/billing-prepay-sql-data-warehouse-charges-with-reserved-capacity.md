# <a name="prepay-for-sql-data-warehouse-charges-with-reserved-capacity"></a>Оплатите плата за хранилище данных SQL с зарезервированным объемом ресурсов

Можно сократить затраты с хранилищем данных SQL Azure, Предоплата за использование в cDWU за длительностью один или три года. Чтобы приобрести емкость зарезервировано хранилище данных SQL, необходимо выбрать Azure регион и срок действия. Затем добавьте номер SKU хранилища данных SQL в корзину и укажите количество единиц cDWU, которые вы хотите приобрести.

Когда вы покупаете резервирование, в хранилище данных SQL, соответствует атрибутам резервирования для использования больше не взимается оплата как то перейдите тарифы.

Резервирование не охватывает хранилища или сети накладные расходы, связанные с использованием хранилища данных SQL.

По истечении срока действия зарезервированной емкости хранилища данных SQL экземпляры продолжают работать, но выставляются по перейдите ставке оплаты как то. Резервирования не продлевает автоматически.

Сведения о ценах см. в разделе [хранилище данных SQL защищены предложения емкость](https://azure.microsoft.com/pricing/details/sql-data-warehouse/gen2/).

Вы можете приобрести емкость хранилища данных SQL защищены [портала Azure](https://portal.azure.com/#blade/Microsoft_Azure_Reservations/ReservationsBrowseBlade). Требования для приобретения резервной мощности.

- Необходимо иметь роль владельца для по крайней мере один enterprise или оплатой по мере использования.
- Для подписок Enterprise **Добавление зарезервированных экземпляров** параметр должен быть включен в [портал EA](https://ea.azure.com/). Если этот параметр отключен, необходимо быть администратором EA
- Для программы поставщика облачных решений (CSP) только агенты администрирования или агентов по продажам может приобрести емкость зарезервировано хранилище данных SQL.

Дополнительные сведения о корпоративных клиентов и клиентов по мере использования выставление счета за покупки резервирования, см. в разделе [Общие сведения об использовании Azure резервирования о регистрации Enterprise](billing-understand-reserved-instance-usage-ea.md) и [Общие сведения о Azure данные об использовании резервирования для подписки с оплатой по мере использования](billing-understand-reserved-instance-usage.md).

## <a name="choose-the-right-size-before-purchase"></a>Выбор правильного размера перед покупкой

Хранилище данных SQL, размер резервирования должны быть основаны на общей вычислительные единицы использования хранилища данных (cDWU), которые можно использовать. Покупки сделаны шагом в 100 cDWU.

Например предположим, что общее потребление ресурсов хранилища данных SQL является DW3000c. Вы хотите приобрести зарезервированный объем ресурсов для всех его. Таким образом вы должны приобрести 30 единиц cDWU зарезервировано емкости.

## <a name="buy-sql-data-warehouse-reserved-capacity"></a>Хранилище данных SQL купить зарезервированная емкость

1. Войдите на [портал Azure](https://portal.azure.com/).
2. Выберите **Все службы** > **Резервирование**.
3. Выберите подписку. Используйте список подписок выберите подписку, используемой для оплаты зарезервированный объем ресурсов. Метод оплаты подписки будет взиматься плата первоначальные затраты для зарезервированной емкости. Нужно выбрать подписку с соглашением Enterprise (номера предложений: MS-AZR-0017P или MS-AZR-0148P) или подписку с оплатой по мере использования (номера предложений: MS-AZR-0003P или MS-AZR 0023P).
  - Для подписки с соглашением Enterprise плата вычитается из баланса денежных обязательств или относится к избыточным расходам.
  - Для подписки с оплатой по мере использования плата взимается с помощью метода оплаты (кредитной карты или счета), указанного для подписки.
4. Выберите область. Используйте список области, чтобы выбрать область подписки.
  - С помощью **единый** , скидка на резервирование применяется к хранилищу данных SQL, развернутых в выбранной подписке.
  - С помощью **Shared** , скидка на резервирование применяется к экземплярам, работающих в любых подписках в текущем контексте выставления счетов.
    - Для корпоративных клиентов контекст выставления счетов является соглашением EA.
    - Для клиентов с подпиской с оплатой по мере использования общая область включает в себя все подписки с оплатой по мере использования, созданные администратором учетной записи.
5. Выберите регион, чтобы выбрать регион Azure, охватываемой зарезервированный объем ресурсов.
6. Выберите количество. Введите количество 100 единиц хранилища данных (cDWU), которые вы хотите приобрести.    
  Например количество 30 получатся 3 000 cDWU за счет зарезервированного объема каждый час.
7. Обзор хранилища данных SQL защищены стоимость резервирования емкости в **затраты** раздел.
8. Щелкните **Приобрести**.
9. Выберите **просмотреть резервирование** Чтобы просмотреть состояние покупки.

## <a name="cancellations-and-exchanges"></a>Отмена и обмен

Если требуется отменить хранилище данных SQL зарезервированной емкости, может возникнуть 12% досрочное завершение. Сумма возмещенных средств основывается на наименьшей стоимости, например на стоимости покупки или текущей стоимости резервирования. Возмещения ограничены 50,000.00 долл. США в год. Возврат, появляется является пропорционально оставшемуся балансу минус 12% досрочное завершение. Если нужно запросить отмену, перейдите к резервированию на портале Azure и выберите **Возместить**, чтобы создать запрос в службу поддержки.

Если вам нужно изменить емкости хранилища данных SQL, которые защищены в другой регион или термин, его можно обмена на другой резервирование, которое имеет значение равно или больше. При обмене дата начала использования для нового резервирования не переносится из текущего резервирования. Один или три года начинается при создании нового резервирования. Чтобы запросить exchange, откройте резервирование на портале Azure и выберите **Exchange** создать запрос в службу поддержки.

Дополнительные сведения о том, как exchange или возмещение резервирования см. в разделе [обмен резервирования и возмещения](billing-azure-reservations-self-service-exchange-and-refund.md).

Скидка на резервирование автоматически применяется к число экземпляров хранилища данных SQL, которые соответствуют область определения емкости хранилища данных SQL, которые защищены и регион. Можно обновить область емкости хранилища данных SQL, которые защищены с помощью [портала Azure](https://portal.azure.com/), PowerShell, CLI или API.

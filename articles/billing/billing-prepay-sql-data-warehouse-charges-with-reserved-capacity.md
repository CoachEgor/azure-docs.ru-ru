# <a name="prepay-for-sql-data-warehouse-charges-with-reserved-capacity"></a>Оплата за использование хранилища данных SQL с зарезервированной емкостью

Вы можете сэкономить деньги с помощью хранилища данных SQL Azure, предоплата за использование cDWU в течение одного или трех лет. Чтобы приобрести зарезервированную емкость хранилища данных SQL, необходимо выбрать регион Azure и термин. Затем добавьте номер SKU хранилища данных SQL в корзину и выберите количество единиц cDWU, которые вы хотите приобрести.

При покупке резервирования использование хранилища данных SQL, которое соответствует атрибутам резервирования, больше не оплачивается по тарифам с оплатой по мере использования.

Резервирование не охватывает затраты на хранение и сетевые подключения, связанные с использованием хранилища данных SQL.

По истечении зарезервированной емкости экземпляры хранилища данных SQL продолжают работать, но выставляются по тарифам с оплатой по мере использования. Резервирования не обновляются автоматически.

Сведения о ценах см. в [предложении зарезервированных ресурсов хранилища данных SQL](https://azure.microsoft.com/pricing/details/sql-data-warehouse/gen2/).

Вы можете приобрести зарезервированную емкость хранилища данных SQL Azure в [портал Azure](https://portal.azure.com/#blade/Microsoft_Azure_Reservations/ReservationsBrowseBlade). Требования для приобретения резервной мощности.

- У вас должна быть роль владельца по крайней мере для одной подписки предприятия или с оплатой по мере использования.
- Для корпоративных подписок параметр **Добавить зарезервированные экземпляры** должен быть включен на [портале EA](https://ea.azure.com/). Если параметр отключен, необходимо быть администратором EA.
- Для программы поставщика облачных решений (CSP) можно приобрести зарезервированную емкость хранилища данных SQL только агентами администратора или агентами по продажам.

Дополнительные сведения о том, как корпоративные клиенты и клиенты с оплатой по мере использования оплачиваются по покупкам резервирования, см. в статье [Использование резервирования Azure для регистрации на предприятии](billing-understand-reserved-instance-usage-ea.md) и [сведения об использовании резервирования Azure для вашего Подписка с оплатой по мере](billing-understand-reserved-instance-usage.md)использования.

## <a name="choose-the-right-size-before-purchase"></a>Выбор правильного размера перед покупкой

Размер резервирования хранилища данных SQL должен основываться на общем количестве используемых единиц вычислений для хранилища данных (cDWU). Покупки создаются с шагом 100 cDWU.

Например, предположим, что общий объем потребления хранилища данных SQL — DW3000c. Вы хотите приобрести зарезервированную емкость для всех ИТ. Поэтому следует приобрести 30 единиц cDWU зарезервированной емкости.

## <a name="buy-sql-data-warehouse-reserved-capacity"></a>Приобретение зарезервированной емкости хранилища данных SQL

1. Войдите на [портале Azure](https://portal.azure.com/).
2. Выберите **Все службы** > **Резервирование**.
3. Выберите подписку. Используйте список подписка, чтобы выбрать подписку, которая будет использоваться для оплаты зарезервированной емкости. В методе оплаты подписки взимается предварительные затраты на зарезервированную емкость. Нужно выбрать подписку с соглашением Enterprise (номера предложений: MS-AZR-0017P или MS-AZR-0148P) или подписку с оплатой по мере использования (номера предложений: MS-AZR-0003P или MS-AZR 0023P).
   - Для подписки с соглашением Enterprise плата вычитается из баланса денежных обязательств или относится к избыточным расходам.
   - Для подписки с оплатой по мере использования плата взимается с помощью метода оплаты (кредитной карты или счета), указанного для подписки.
4. Выберите область. Используйте список область для выбора области подписки.
   - **Единственная область группы ресурсов** — применяет скидку резервирования к соответствующим ресурсам в выбранной группе ресурсов.
   - **Область действия одной подписки** — применяет скидку резервирования к соответствующим ресурсам в выбранной подписке.
   - **Общая область** — применяет скидку резервирования для сопоставления ресурсов в подходящих подписках, которые находятся в контексте выставления счетов. Для клиентов Соглашение Enterprise в качестве контекста выставления счетов используется регистрация. Для отдельных подписок с оплатой по мере использования в качестве области выставления счетов используются все подходящие подписки, созданные администратором учетной записи.
   - Для корпоративных клиентов в качестве контекста выставления счетов используется регистрация EA.
   - Для клиентов с подпиской с оплатой по мере использования общая область включает в себя все подписки с оплатой по мере использования, созданные администратором учетной записи.
5. Выберите регион для выбора региона Azure, охватываемого зарезервированной емкостью.
6. Выберите количество. Введите количество единиц использования хранилища данных 100 (cDWU), которые вы хотите приобрести.    
   Например, количество 30 выдаст 3 000 cDWU зарезервированной емкости каждый час.
7. Ознакомьтесь с зарезервированной емкостью резервирования ресурсов в хранилище данных SQL в разделе **затраты** .
8. Щелкните **Приобрести**.
9. Выберите **просмотреть это резервирование** , чтобы просмотреть состояние покупки.

## <a name="cancel-exchange-or-refund-reservations"></a>Отмена, Обмен или возмещение в резервированиях

Вы можете отменять, обмениваться и возмещать деньги на резервирования с определенными ограничениями. Дополнительные сведения см. в статье [самостоятельная служба Exchange и возмещение для резервирования Azure](billing-azure-reservations-self-service-exchange-and-refund.md).

Скидка на резервирование автоматически применяется к числу экземпляров хранилища данных SQL, которые соответствуют области и региону зарезервированной емкости хранилища данных SQL. Вы можете обновить область зарезервированной емкости хранилища данных SQL с помощью [портал Azure](https://portal.azure.com/), POWERSHELL, CLI или API.

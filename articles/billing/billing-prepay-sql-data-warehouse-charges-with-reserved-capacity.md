# <a name="save-costs-for-sql-data-warehouse-charges-with-reserved-capacity"></a>Экономия затрат на Хранилище данных SQL с помощью резервной мощности

Вы можете сократить расходы в Хранилище данных SQL, предварительно зафиксировав резервирование на потребление cDWU на один год, или три года. Чтобы приобрести резервную мощность Хранилища данных SQL, необходимо выбрать регион Azure и термин. Затем добавьте номер SKU Хранилища данных SQL в корзину и выберите количество единиц cDWU, которые хотите приобрести.

При приобретении резервирования плата за использование Хранилища данных SQL, которое соответствует атрибутам резервирования, больше не взимается в соответствии с расценками по мере использования.

Резервирование не покрывает затраты на хранилище и сетевые ресурсы, связанные с использованием Хранилища данных SQL.

Когда срок действия резервной мощности истекает, экземпляры Хранилища данных SQL продолжают работать, но оплата за них взимается в соответствии с расценками по мере использования. Резервирования не продлеваются автоматически.

Информацию о ценах см. в разделе [Резервная мощность Хранилища данных SQL](https://azure.microsoft.com/pricing/details/sql-data-warehouse/gen2/).

Вы можете приобрести резервную мощность Хранилища данных SQL Azure на [портале Azure](https://portal.azure.com/#blade/Microsoft_Azure_Reservations/ReservationsBrowseBlade). Платите за резервирование [наперед или ежемесячными платежами](billing-monthly-payments-reservations.md). Требования для приобретения резервной мощности.

- Необходимо иметь роль "Владелец" по крайней мере для одной подписки с соглашением Enterprise или подписки с оплатой по мере использования.
- Для подписок с соглашением Enterprise параметр **Добавить зарезервированные экземпляры** следует включить на [портале EA](https://ea.azure.com/). Если этот параметр отключен, необходимо быть администратором EA.
- В рамках программы для поставщиков облачных решений (CSP) только агенты администрирования или агенты продаж могут приобрести резервную мощность Хранилища данных SQL.

Дополнительные сведения о выставлении счетов за приобретение резервирования для корпоративных клиентов и клиентов с оплатой по мере использования см. в статьях [Общие сведения об использовании зарезервированных экземпляров Azure с Соглашением о регистрации Enterprise](billing-understand-reserved-instance-usage-ea.md) и [Общие сведения об использовании резервирования Azure для подписки с оплатой по мере использования](billing-understand-reserved-instance-usage.md).

## <a name="choose-the-right-size-before-purchase"></a>Выберите правильный размер перед покупкой

Размер резервирования Хранилища данных SQL должен основываться на общем количестве используемых единиц вычислений для хранилища данных (cDWU). Покупки осуществляются с шагами приращения в 100 cDWU.

Например, предположим, что общий объем потребления Хранилища данных SQL — DW3000c. Вы хотите приобрести резервную мощность для всего. Поэтому следует приобрести 30 единиц резервной мощности cDWU.

## <a name="buy-sql-data-warehouse-reserved-capacity"></a>Покупка резервной мощности Хранилища данных SQL

1. Войдите на [портале Azure](https://portal.azure.com/).
2. Выберите **Все службы** > **Резервирование**.
3. Выберите подписку. Из списка "Подписка" выберите подписку, которая используется для оплаты резервной мощности. Резервная мощность оплачивается с помощью метода оплаты, указанного для подписки. Нужно выбрать подписку с соглашением Enterprise (номера предложений: MS-AZR-0017P или MS-AZR-0148P) или подписку с оплатой по мере использования (номера предложений: MS-AZR-0003P или MS-AZR 0023P).
   - Для подписки с соглашением Enterprise плата вычитается из баланса денежных обязательств или относится к избыточным расходам.
   - Для подписки с оплатой по мере использования плата взимается с помощью метода оплаты (кредитной карты или счета), указанного для подписки.
4. Выберите область. Используйте список "Область" для выбора диапазона подписки.
   - **Одна группа ресурсов** — скидка по резервированию применяется к подходящим ресурсам только в выбранной группе ресурсов.
   - **Одна подписка** — скидка по резервированию применяется к подходящим ресурсам только в выбранной подписке.
   - **Общая область резервирования** — скидка по резервированию применяется к подходящим ресурсам во всех допустимых подписках в контексте выставления счетов. Для клиентов Соглашения Enterprise контекстом выставления счетов считается регистрация. Для индивидуальных клиентов с оплатой по мере использования областью резервирования считаются все допустимые подписки, созданные администратором учетной записи.
   - Для клиентов Соглашения Enterprise контекстом выставления счетов считается соглашение о регистрации EA.
   - Для клиентов с подпиской с оплатой по мере использования общая область включает в себя все подписки с оплатой по мере использования, созданные администратором учетной записи.
5. Выберите регион Azure, который охватывает резервная мощность.
6. Выберите количество. Введите количество в 100 единиц Хранилища данных (cDWU), которые вы хотите приобрести.    
   Например, количество 30 будет выдавать 3 000 cDWU резервной мощности каждый час.
7. Просмотрите затраты на резервирование резервной мщности Хранилища данных SQL в разделе **Затраты**.
8. Щелкните **Приобрести**.
9. Выберите **Просмотреть резервирование**, чтобы просмотреть состояние покупки.

## <a name="cancel-exchange-or-refund-reservations"></a>Отмена, обмен или возврат резервирования

Вы можете отменить и обменять резервирования, а также вернуть вложенные в резервирование средства, но при этом применяются определенные ограничения. Дополнительные сведения см. в статье [Self-service exchanges and refunds for Azure Reservations](billing-azure-reservations-self-service-exchange-and-refund.md) (Самостоятельное выполнение обмена и возмещения средств для резервирований Azure).

Скидка на резервирование автоматически применяется к количеству экземпляров Хранилища данных SQL, которые соответствуют объему резервной мощности Хранилища данных SQL и региону. Область резервной мощности Хранилища данных SQL можно обновить с помощью [портала Azure](https://portal.azure.com/), PowerShell, CLI или API.

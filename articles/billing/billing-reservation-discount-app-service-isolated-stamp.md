# <a name="how-reservation-discounts-apply-to-azure-app-service-isolated-stamps"></a>Как скидки на резервирование применяются к меткам Службы приложений Azure (цен. категория "Изолированный")

После покупки резервной мощности с Ежемесячным гербовым сбором изолированной службы приложений скидка на резервирование автоматически применяется к базовой плате в регионе. Скидка на резервирование применяется к использованию, предусмотренному единицей измерения гербового сбора (изолированный). Рабочие роли, дополнительные роли, связанные с внешним интерфейсом, и любые другие ресурсы, связанные с меткой, продолжают получать счета по обычному тарифу.

## <a name="reservation-discount-application"></a>Применение скидки на резервирование

Скидка на Ежемесячный гербовый сбор изолированной службы приложений применяется к действующим изолированным меткам на почасовой оплате. Если вы не установили метку в течение часа, то резервная мощность будет потрачена впустую на этот час. Она не переносится.

После покупки приобретенное резервирование сопоставляется с изолированной меткой, работающей в определенном регионе. Если вы закроете эту метку, скидки на резервирование будут автоматически применены к любым другим меткам, работающим в регионе. Если метки не существуют, резервирование применяется к следующей метке, созданной в регионе.

Если метки не работают в течение полного часа, резервирование автоматически применяется к другим соответствующим меткам в том же регионе в течение того же часа.

## <a name="choose-a-stamp-type---windows-or-linux"></a>Выбор типа метки — Windows или Linux

Пустая изолированная метка по умолчанию создает единицу измерения меток Windows. Например, когда не развернуты рабочие роли. Она продолжает создавать единицу измерения при развертывании рабочих ролей Windows. Единица измерения изменится на единицу измерения метки Linux, если вы развернете рабочую роль Linux. Метка создает единицу измерения Windows, когда развернуты рабочие роли Linux и Windows.

В результате единица измерения метки может меняться между Windows и Linux в течение срока использования метки. В то же время резервирование зависит от операционной системы. Вам нужно будет купить резервирование, поддерживающее рабочие роли, которые вы планируете развернуть в метке. Для меток только для Windows и смешанных меток используется резервирование Windows. Метки только с рабочими ролями Linux используют резервирование Linux.

Единственный раз, когда вам необходимо будет приобрести резервирование Linux, это если вы планируете иметь _только_ рабочие роли Linux в метке.

## <a name="discount-examples"></a>Примеры скидок

В следующих примерах показано, как применяется скидка на зарезервированный экземпляр изолированного гербового сбора в зависимости от развертываний.

- **Пример 1**. Вы приобретаете один экземпляр мощности изолированных зарезервированных меток в регионе без меток Службы приложений (цен. категория "Изолированный"). Вы развертываете новую метку в регионе и платите по зарезервированным тарифам для этой метки.
- **Пример 2**. Вы приобретаете один экземпляр изолированной зарезервированной метки в регионе, в котором уже развернута метка Службы приложений (цен. категория "Изолированный"). Вы начинаете получать зарезервированный тариф для развернутой метки.
- **Пример 3**. Вы приобретаете один экземпляр мощности изолированной зарезервированной метки в регионе с уже развернутой меткой Службы приложений (цен. категория "Изолированный"). Вы начинаете получать зарезервированный тариф для развернутой метки. Позже вы удалите метку и развернете новую. Вы получаете зарезервированный тариф для новой метки. Скидки не переносятся на время без развернутых меток.
- **Пример 4**. Вы приобретаете один экземпляр мощности изолированной зарезервированной метки Linux в регионе, а затем развертываете новую метку в регионе. Когда метка изначально развертывается без рабочих ролей, она создает единицу измерения меток Windows. Скидка не получена. Когда первая рабочая роль Linux развертывает метку, она создает единицу измерения меток Linux и применяется скидка на резервирование. Если рабочая роль Windows позже развернется в метке, единица измерения меток вернется к Windows. Вы больше не получаете скидку на резервирование изолированной зарезервированной метки Linux.

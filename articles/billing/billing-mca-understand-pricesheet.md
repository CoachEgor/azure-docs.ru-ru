# <a name="terms-in-your-microsoft-customer-agreement-price-sheet"></a>Условия вашего соглашения для Microsoft Customer прайс-листа

Эта статья относится к учетной записи Azure выставления счетов для клиента соглашение Microsoft. [Проверьте наличие доступа к соглашению клиента Microsoft](#check-access-to-a-microsoft-customer-agreement).

Если профиль выставления счетов, владелец, участник, чтения или Управление счетами прайс-лист вашей организации можно загрузить на портале Azure. См. в разделе [представление и загрузки ценообразования организации](billing-ea-pricing.md).

## <a name="terms-and-descriptions-in-your-price-sheet"></a>Термины и описания в прайс-листа

В следующем разделе описаны важные термины, встречающиеся в прайс-лист в соглашении клиента Microsoft.

| **Имя поля**   | **Описание**   |
| --- | --- |
| billingAccountId  | Уникальный идентификатор для учетной записи выставления счетов.   |
| billingAccountName  | Имя учетной записи выставления счетов.  |
| billingProfileId  | Уникальный идентификатор для профиля выставления счетов.   |
| billingProfileName  | Имя профиля выставления счетов, которая настроена для получения счета. Цены в прайс-лист связаны с этим профилем выставления счетов. |
| productOrderName  | Имя плана приобретаемой продукции. |
| serviceFamily  | Тип службы Azure. Пример: Вычисления, аналитика и безопасности |
| Продукт  | Имя продукта, накапливаемый накладные расходы. Пример: Основные SQL DB vs базы данных SQL Standard  |
| productId  | Уникальный идентификатор продукта, потребляется которого счетчик. |
| unitOfMeasure  | Определяет единицы измерения для выставления счетов для службы. Например службы вычислений действует почасовая оплата. |
| meterId  | Уникальный идентификатор для единицы измерения. |
| meterName  | Имя индикатора. Выражают ресурс службы Azure. |
| meterCategory  | Имя категории, классификации для единицы измерения. Например _облачные службы_, _сети_и т. д. |
| meterType  |  Имя типа средства измерения. |
| meterSubCategory  | Имя категории счетчика дочернюю классификацию.  |
| meterRegion  | Имя региона, где доступны метрики для службы. В этом столбце указывается расположение центра обработки данных для определенных служб. От этого расположения может зависеть стоимость некоторых услуг.    |
| tierId  | Определяет ценовую категорию, если это применимо. Эта процедура работает совместно с tierMinimumUnits для задания многоуровневых цены когда цены изменяются в зависимости от числа единиц, используемых.    |
| tierMinimumUnits  | Определяет нижнюю границу диапазона уровня, для которого определены цены. Например если диапазон — от 0 до 100, tierMinimumUnits будет равно 0.  |
| effectiveStartDate  | Начальная дата, когда цена вступает в силу. |
| effectiveEndDate  | Дата окончания низкую цену. |
| unitPrice  | Цена за единицу во время выставления счетов (не действующие переходом цены) как относящиеся к имени порядок индикатора и продукта.  Примечание. Цена за единицу отличается по мере загрузки действующих цена в сведения об использовании в случае службы, имеющие разностные цены по уровням.  В случае службы с многоуровневой моделью ценообразования действующих цен представляет собой смесь интенсивность всех уровнях и не показывает цену единицы относящихся к уровню. Объединенный или действующие цены — это чистая цена за использованное количество связующих на нескольких уровнях (где каждый уровень имеет определенную цену единицы). |
| basePrice  | Рыночная стоимость во время выходе клиента или рыночная стоимость во время индикаторе служба запускает в случае после входа.   |

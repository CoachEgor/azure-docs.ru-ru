# <a name="understand-how-the-reservation-discount-is-applied-to-azure-storage"></a>Общие сведения о применении скидки на резервирование в службе хранилища Azure

После приобретения зарезервированной емкости службы хранилища Azure скидка на резервирование автоматически применяется к блочным BLOB-объектам и ресурсам Azure Data Lake Storage 2-го поколения, которые соответствуют условиям резервирования. Скидка на резервирование применяется только к емкости хранилища. Плата за пропускную способность и частоту запросов взимается по тарифам с оплатой по мере использования.

Дополнительные сведения о зарезервированной емкости службы хранилища Azure см. в статье об [оптимизации затрат на хранилище BLOB-объектов за счет использования зарезервированной емкости](../storage/blobs/storage-blob-reserved-capacity.md).

Сведения о ценах на резервирование службы хранилища Azure см. в статьях [Цены на хранение блочных BLOB-объектов](https://azure.microsoft.com/pricing/details/storage/blobs/) и [Цены на Azure Data Lake Storage 2-го поколения](https://azure.microsoft.com/pricing/details/storage/data-lake/).

## <a name="how-the-reservation-discount-is-applied"></a>Применение скидки на резервирование

Скидка на зарезервированную емкость службы хранилища Azure применяется к блочным BLOB-объектам и ресурсам Azure Data Lake Storage 2-го поколения на почасовой основе.

Скидка на зарезервированную емкость службы хранилища Azure сгорает, если ее не использовать. При отсутствии блочных BLOB-объектов или ресурсов Azure Data Lake Storage 2-го поколения, которые соответствуют условиям резервирования в течение заданного часа, количество резервирования за этот час будет потеряно. Вы не можете переносить неиспользуемые зарезервированные часы.

При удалении ресурса скидка на резервирование автоматически применяется к другому подходящему ресурсу в указанной области резервирования. Если в указанной области не найдены подходящие ресурсы, зарезервированные часы теряются.

## <a name="discount-examples"></a>Примеры скидок

В следующих примерах показано, как применяется скидка на зарезервированную емкость службы хранилища Azure в зависимости от развертываний.

Предположим, что вы приобрели 100 ТБ зарезервированной емкости в регионе "Западная часть США 2" на один год. Резервирование предназначено для локально избыточного хранилища на горячем уровне доступа.

Предположим, что стоимость этого примера резервирования — 18 540 долл. США. Вы можете оплатить всю сумму заранее или оплачивать ежемесячные взносы в размере 1545 долл. США в месяц в течение следующих двенадцати месяцев.

В этих примерах предполагается, что вы зарегистрировались для использования плана на резервирование с ежемесячной оплатой. В следующих сценариях описывается, что происходит при неполном или чрезмерном использовании зарезервированной емкости.

### <a name="underusing-your-capacity"></a>Неполное использование емкости

Предположим, что в течение заданного часа в период резервирования использовано только 80 ТБ из 100 ТБ зарезервированной емкости. Оставшиеся 20 ТБ не применяются к этому часу и не переносятся.

### <a name="overusing-your-capacity"></a>Чрезмерное использование емкости

Предположим, что в течение заданного часа в период резервирования использовано 101 ТБ емкости хранилища. Скидка на резервирование применяется к 100 ТБ данных, а оставшийся 1 ТБ оплачиваются по тарифам с оплатой по мере использования за этот час. Если в течение следующего часа ваш объем использования изменится на 100 ТБ, то все данные о потреблении покрываются резервированием.

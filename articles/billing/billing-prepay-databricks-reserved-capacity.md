# <a name="optimize-azure-databricks-costs-with-a-pre-purchase"></a>Оптимизируйте затраты на Azure Databricks с помощью предварительного приобретения

Можно сохранить в ваш Azure Databricks (DBU) Себест при покупке Azure Databricks фиксацию единиц (DBCU) для один или три года. Можно использовать предварительно приобретенных DBCUs в любое время в течение срока покупки. В отличие от виртуальных машин, предварительно приобретенные единицы измерения имеют неограниченный срок жизни на почасовой основе, и использовать их в любое время в течение срока действия покупки.

Любое использование Azure Databricks предусматривает удержание из предварительно приобретенных Dbu автоматически. Не требуется повторное развертывание или назначать рабочие области Azure Databricks для использования DBU для получения скидки предварительной покупки предварительно приобретенных плана.

Скидка предварительной покупки применяется только к DBU использования. Другие расходы, такие как вычисления, хранения и сети, взимается отдельно.

## <a name="determine-the-right-size-to-buy"></a>Определение правильного размера купить

Databricks предварительной покупки применяется для всех рабочих нагрузок Databricks и уровней. Можно представить предварительной покупки, как пул предоплаченных единиц фиксации Databricks. Использование вычитается из пула, независимо от рабочей нагрузки или уровня. В следующем соотношении вычитается использования:

| **Рабочая нагрузка** | **Отношение приложения DBU — уровень "стандартный"** | **Отношение приложения DBU — уровень "премиум"** |
| --- | --- | --- |
| Аналитика данных | 0,4 | 0.55 |
| Инжиниринг данных | 0,15 | 0.30 |
| Инжиниринг данных с облегченной нагрузкой | 0,07 | 0.22 |

Например когда количество анализа данных — используются в уровня "стандартный", предварительно приобретенные единицы Databricks фиксации вычитается 0,4 приращения.

Перед покупкой, вычислите общее количество DBU, использованных для разных рабочих нагрузок и уровней. Используйте предыдущий коэффициенты для нормализации для DBCU, а затем запустите проекцию общее использование за следующие один или три года.

## <a name="purchase-databricks-commit-units"></a>Приобрести единицы Databricks фиксации

Вы можете приобрести планы Databricks [портала Azure](https://portal.azure.com/#blade/Microsoft_Azure_Reservations/CreateBlade/referrer/documentation/filters/%7B%22reservedResourceType%22%3A%22Databricks%22%7D). Чтобы купить зарезервированный объем ресурсов, необходимо иметь роль владельца для подписки по крайней мере один enterprise.

- В настоящее время предварительно приобрести доступен только для клиентов с соглашением Enterprise.
- Будьте в роли "Владелец" для по крайней мере одна подписка Enterprise.
- Для подписок с соглашением Enterprise параметр **Добавить зарезервированные экземпляры** следует включить на [портале EA](https://ea.azure.com/). Или, если этот параметр отключен, необходимо быть администратором подписки EA.

**Для приобретения:**

1. Перейдите на [портал Azure](https://portal.azure.com/#blade/Microsoft_Azure_Reservations/CreateBlade/referrer/documentation/filters/%7B%22reservedResourceType%22%3A%22Databricks%22%7D).
1. Выберите подписку. Используйте **подписки** список, чтобы выбрать подписку, которая используется для оплаты для зарезервированной емкости. Метод оплаты подписки будет взиматься плата первоначальные затраты для зарезервированной емкости. Плата вычитается из баланса денежных обязательств регистрации или избыточным расходам.
1. Выберите область. Используйте **область** список, чтобы выбрать область подписки:
    - **Единый области действия группы ресурсов** — применяется скидка на резервирование в соответствующие ресурсы в выбранной группе ресурсов только.
    - **Единый область подписки** — применяется скидка на резервирование в соответствующие ресурсы в выбранной подписке.
    - **Общая область** — применяется скидка на резервирование к соответствующим ресурсам в соответствующих подписках, которые находятся в контексте выставления счетов. Для клиентов с соглашением Enterprise контекст выставления счетов является регистрации.
1. Выберите количество единиц фиксации Azure Databricks, необходимо приобрести и завершите покупку.

## <a name="change-scope-and-ownership"></a>Изменение области и владения

После приобретения резервирования можно внести следующие изменения:

- Обновить область резервирования
- Доступ на основе ролей

Нельзя разделить или объединить единица Databricks фиксации предварительной покупки. Дополнительные сведения об управлении резервирования см. в разделе [управлению резервированиями после приобретения](billing-manage-reserved-vm-instance.md).

## <a name="cancellations-and-exchanges"></a>Отмена и обмен

Отмена и exchange не поддерживается для Databricks предварительного приобретения планов. Все покупки являются окончательными.

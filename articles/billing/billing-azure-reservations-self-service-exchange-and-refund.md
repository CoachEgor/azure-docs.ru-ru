# <a name="self-service-exchanges-and-refunds-for-azure-reservations"></a>Обмен самообслуживания и возмещения для резервирования Azure

Резервирование Azure обеспечивают необходимую гибкость для удовлетворения изменяющихся потребностей. Зарезервированный экземпляр можно обменять на другой зарезервированный экземпляр того же типа. Также можно вернуть средства за резервирование (до 50 000 долл. США в год), если вы больше не используете его.

Самостоятельный обмен и возможность отмены недоступны для клиентов с соглашением Enterprise для государственных организаций США. Поддерживаются другие типы подписки для государственных организаций США, включая оплату по мере использования и CSP.

Необходимо иметь доступ владельца на заказ на резервирование для обмена или возмещение существующее резервирование.

## <a name="exchange-an-existing-reserved-instance"></a>Exchange существующий зарезервированный экземпляр

Можно организовать обмен резервировании с три простых шага в [портала Azure](https://portal.azure.com/#blade/Microsoft_Azure_Reservations/ReservationsBrowseBlade).

1. Выберите резервирование, которые вы хотите возмещение и нажмите кнопку **Exchange**.  
    ![Пример изображение резервирования для возврата](./media/billing-azure-reservations-self-service-exchange-and-refund/exchange-refund-return.png)
2. Выберите продукт в категории виртуальных машин, который вы хотите приобрести, и введите количество. Убедитесь, что новая сумма покупки больше, чем возвращаемого значения общего количества. [Определить подходящий размер, до приобретения](../virtual-machines/windows/prepay-reserved-vm-instances.md#determine-the-right-vm-size-before-you-buy).  
    ![Пример изображение продукта виртуальной Машины приобретается в комплекте с exchange](./media/billing-azure-reservations-self-service-exchange-and-refund/exchange-refund-select-purchase.png)
3. Просмотрите и завершения транзакции.  
    ![Пример изображение продукта приобретается в комплекте с exchange, выполнив возврат виртуальной Машины](./media/billing-azure-reservations-self-service-exchange-and-refund/exchange-refund-confirm-exchange.png)

Чтобы возмещение резервирование, перейдите в раздел **сведения о резервировании** и нажмите кнопку **возмещение**.

## <a name="how-transactions-are-processed"></a>Обработка транзакций

Во-первых Корпорация Майкрософт отменяет существующее резервирование и возмещения сумму пропорционально для этого резервирования. Если обмен, обрабатывается новой покупкой. Корпорация Майкрософт обрабатывает возмещения, с помощью одного из следующих методов, в зависимости от типа учетной записи и метод оплаты:

### <a name="enterprise-agreement-customers"></a>Клиентов с соглашением Enterprise

Money добавляется денежных обязательств для обмена и возмещения, если первоначальной покупки была выполнена с помощью одного. Используется ни одного избыточные счета, поскольку исходный покупки, открыть повторно и rerated, чтобы убедиться, что денежных обязательств. Если денежные термин, используя резервирование было приобретено больше не активна, кредит добавляется текущего срока денежные обязательства соглашения enterprise.

Если исходная Покупка была выполнена как за, корпорация Microsoft выпускает кредит-ноты.

### <a name="pay-as-you-go-invoice-payments-and-csp-program"></a>Счета на оплату по мере использования платежей и программы CSP

Исходного счета покупки резервирования отменяется, а затем создается новый счет для возмещения. Для обмена с нового счета показывает возмещения и новой покупкой. Сумма возврата корректируется для покупки. Если вы только компенсируются резервирование, сумму пропорционально остается с Microsoft, и он настраивается для будущих оплаченное.

### <a name="pay-as-you-go-credit-card-customers"></a>Оплата по мере использования кредитной карты клиентов

Исходного счета отменяется, и создается новый счет. Деньги возвращается с кредитной карты, которая использовалась для первоначальной покупки. Если вы изменили вашей карты [обратитесь в службу поддержки](https://portal.azure.com/#blade/Microsoft_Azure_Support/HelpAndSupportBlade/newsupportrequest).

## <a name="exchange-policies"></a>Политики Exchange

- Может возвращать несколько существующих резервирований, чтобы купить новое резервирование того же типа. Не удается обмениваться резервирования одного типа другим. Например нельзя вернуть резервирование виртуальных Машин для покупки резервирования SQL.
- Только владельцы резервирования может обрабатывать обмен. [Узнайте, как добавить или изменить пользователей, которые могут управлять резервированием](https://docs.microsoft.com/azure/billing/billing-manage-reserved-vm-instance#add-or-change-users-who-can-manage-a-reservation).
- Обмен обрабатывается как возмещения и repurchase – различные транзакции создаются для отмены и новой покупкой. Для резервирования возвращается сумму пропорционально резервирования, вы по лицензиям. Вы оплачиваете полностью новой покупкой. Сумму пропорционально резервирования значение по ежедневного пропорционально остаточная возвращаемых резервирования.
- Можно обменять или возмещение резервирования даже в том случае, если использовать соглашение enterprise для приобретения резервирования истек срок действия и продлен как новое соглашение.
- Можно изменить любое свойство резервирование, такие как размер, регион, количество и термин с exchange.
- Новая сумма покупки следует равно или быть больше суммы возвращенного.
- Новое резервирование, приобретенных в составе сервера exchange имеет новый термин, начиная со времени exchange.
- Нет, снижается или годовой ограничения для обмена.

## <a name="refund-policies"></a>Политики возврата

- Сумма вашего общее возврата не может превышать 50 000 Долларов США в 12-месячного скользящее окно.
- Только владельцы резервирования может обрабатывать возмещение. [Узнайте, как добавить или изменить пользователей, которые могут управлять резервированием](billing-manage-reserved-vm-instance.md#add-or-change-users-who-can-manage-a-reservation).
- Майкрософт оставляет за собой право взимать плату штраф 12% на любые символы возврата, несмотря на то, что в настоящее время не взиматься штрафа.

## <a name="exchange-non-premium-storage-for-premium-storage"></a>Exchange без поддержки расширенной хранилища для хранилища класса premium

Можно организовать обмен приобретенных для размера виртуальной Машины, который не поддерживает хранилище уровня "премиум" для соответствующего размера виртуальной Машины, который выполняет резервирование. Например _F1_ для _F1s_. Для этого обмена, перейдите к сведения о резервировании и щелкните **Exchange**. Exchange не сбросить термин зарезервированного экземпляра или создать новую транзакцию.

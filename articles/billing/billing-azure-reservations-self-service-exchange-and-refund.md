# <a name="self-service-exchanges-and-refunds-for-azure-reservations"></a>Самостоятельный обмен и возмещение для резервирования Azure

Резервирование Azure обеспечивает гибкость, позволяющую удовлетворить ваши потребности в развитии. Зарезервированный экземпляр можно обменять на другой зарезервированный экземпляр того же типа. Также можно вернуть средства за резервирование (до 50 000 долл. США в год), если вы больше не используете его.

Самостоятельный обмен и возможность отмены недоступны для клиентов с соглашением Enterprise для государственных организаций США. Поддерживаются другие типы подписок для государственных организаций США, включая оплату по мере использования и CSP.

Для обмена или возврата существующего резервирования необходимо иметь права владельца в заказе на резервирование.

## <a name="exchange-an-existing-reserved-instance"></a>Обмен существующим зарезервированным экземпляром

Вы можете обмениваться резервированием с помощью трех быстрых действий в [портал Azure](https://portal.azure.com/#blade/Microsoft_Azure_Reservations/ReservationsBrowseBlade).

1. Выберите резервирования, которые требуется возвратить, и нажмите кнопку **Exchange**.  
    ![Пример изображения, демонстрирующий возвращаемые резервирования](./media/billing-azure-reservations-self-service-exchange-and-refund/exchange-refund-return.png)
2. Выберите продукт в категории виртуальных машин, который вы хотите приобрести, и введите количество. Убедитесь, что сумма нового приобретения больше суммы, возвращенной в итоге. [Определите правильный размер перед покупкой](../virtual-machines/windows/prepay-reserved-vm-instances.md#determine-the-right-vm-size-before-you-buy).  
    ![Пример изображения, показывающего продукт виртуальной машины для приобретения с помощью Exchange](./media/billing-azure-reservations-self-service-exchange-and-refund/exchange-refund-select-purchase.png)
3. Проверьте и завершите транзакцию.  
    ![Пример изображения, показывающего продукт виртуальной машины для покупки с помощью Exchange, завершение возврата](./media/billing-azure-reservations-self-service-exchange-and-refund/exchange-refund-confirm-exchange.png)

Чтобы возместить резервирование, перейдите в раздел **сведения о резервировании** и щелкните **Возврат**.

## <a name="how-transactions-are-processed"></a>Обработка транзакций

Во-первых, корпорация Майкрософт отменяет существующую зарезервированную сумму и возмещение для этого резервирования. При наличии Exchange будет обработана Новая покупка. Корпорация Майкрософт обрабатывает средства возмещения, используя один из следующих методов, в зависимости от типа учетной записи и способа оплаты:

### <a name="enterprise-agreement-customers"></a>Клиенты с соглашением Enterprise

Деньги добавляются в денежные обязательства по обмену и возмещению, если исходная покупка была выполнена с помощью одной из них. Все промежуточные счета, так как первоначальные покупки повторно открыты и перечисляются, чтобы гарантировать использование денежных обязательств. Если срок действия денежного обязательства, использованного в резервировании, уже не активен, кредит будет добавлен к текущему выражению денежных обязательств в рамках соглашения Enterprise.

Если первоначальная покупка была сделана как избыточная, корпорация Майкрософт выдает кредитовое авизо.

### <a name="pay-as-you-go-invoice-payments-and-csp-program"></a>Оплата счетов с оплатой по мере использования и программа CSP

Исходный счет на покупку резервирования отменяется, а затем для возмещения создается новый счет. Для Exchange в новом счете отображается возмещение и Новая покупка. Сумма возмещения корректируется по покупкам. Если вы только что изменяете резервирование, то пропорциональное количество остается в корпорации Майкрософт и корректируется на будущее приобретение резервирования.

### <a name="pay-as-you-go-credit-card-customers"></a>Клиенты кредитных карт с оплатой по мере использования

Исходный счет отменяется, и создается новый счет. Деньги возмещаются на кредитную карту, которая использовалась для первоначальной покупки. Если вы изменили карточку, [обратитесь в службу поддержки](https://portal.azure.com/#blade/Microsoft_Azure_Support/HelpAndSupportBlade/newsupportrequest).

## <a name="cancel-exchange-and-refund-policies"></a>Политики отмены, обмена и возврата денежных средств

В Azure имеются следующие политики для отмены, обмена и возмещения.

**Политики Exchange**

- Вы можете вернуть несколько существующих резервирований, чтобы приобрести новое резервирование того же типа. Нельзя обмениваться резервированиями одного типа с другим. Например, нельзя вернуть резервирование виртуальной машины для приобретения резервирования SQL.
- Только владельцы резервирования могут обрабатывать Exchange. [Узнайте, как добавлять или изменять пользователей, которые могут управлять резервированием](billing-manage-reserved-vm-instance.md#add-or-change-users-who-can-manage-a-reservation).
- Обмен обрабатывается как возврат и повторное приобретение — разные транзакции создаются для отмены и новой покупки. Сумма распределенного резервирования возобновляется для резервируемых вами резервирований. Вы полностью заряжены на новую покупку. Пропорциональное количество резервирований — это остаточная сумма пропорционально распределенного резервирования.
- Вы можете обмениваться резервированиями, даже если Соглашение Enterprise, используемое для приобретения резервирования, просрочено и продлено как новое соглашение.
- Вы можете изменить любое свойство резервирования, например размер, регион, количество и срок, с помощью Exchange.
- Новый итог покупки должен равняться или быть больше возвращенной суммы.
- Новое резервирование, приобретенное в рамках Exchange, имеет новый термин, начиная с момента обмена.
- Нет штрафов или годовых ограничений для Exchange.

**Политики возврата денег**
- При отмене резервирования может взиматься 12% оплаты за ранний увольнение.
- Возмещение, которое вы получаете для отмены, — это оставшийся баланс с учетом Pro за вычетом 12%. Чтобы отменить операцию, перейдите к зарезервированию в портал Azure и выберите **Возврат**.
- Общий объем возмещения не может превышать $50 000 долларов США в течение 12-месячного скользящего окна.
- Возмещения рассчитываются на основе наименьшей цены на цену покупки или текущую цену резервирования.
- Только владельцы резервирования могут обработать возмещение. [Узнайте, как добавлять или изменять пользователей, которые могут управлять резервированием](billing-manage-reserved-vm-instance.md#add-or-change-users-who-can-manage-a-reservation).
- Корпорация Майкрософт оставляет за собой право взимать 12% штрафа за любые возвраты. Штраф в настоящее время не оплачен, но будет выплачен в будущем.

## <a name="exchange-non-premium-storage-for-premium-storage"></a>Хранилище непривилегированных хранилищ Exchange для хранилища класса Premium

Вы можете обмениваться зарезервированными данными для размера виртуальной машины, который не поддерживает хранилище класса Premium, до соответствующего размера виртуальной машины. Например, _F1_ для _F1s_. Чтобы выполнить обмен, перейдите в раздел сведения о резервировании и нажмите кнопку **Exchange**. Exchange не сбрасывает термин зарезервированного экземпляра или создает новую транзакцию.

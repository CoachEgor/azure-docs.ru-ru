# <a name="how-azure-databricks-pre-purchase-discount-is-applied"></a>О применении скидки предварительной покупки Azure Databricks

Предварительно приобретенных единиц фиксации Azure Databricks (DBCU) можно использовать в любое время в течение срока покупки. Любой Azure Databricks — использование предусматривает удержание из предварительно приобретенных DBCUs автоматически.

В отличие от виртуальных машин, предварительно приобретенных единиц имеют неограниченный срок жизни на почасовой основе. Их можно использовать в любое время в течение срока действия покупки. Чтобы получить скидки предварительной покупки, не нужно повторно развернуть или назначить рабочие области Azure Databricks для использования предварительно приобретенных плана.

Скидка предварительной покупки применяется только к Azure Databricks (DBU) единица использования. Другие расходы, такие как вычисления, хранения и сети, взимается отдельно.

## <a name="pre-purchase-discount-application"></a>Правила применения скидок предварительной покупки

Databricks предварительной покупки применяется для всех рабочих нагрузок Databricks и уровней. Можно представить предварительной покупки, как пул предоплаченных единиц фиксации Databricks. Использование вычитается из пула, независимо от рабочей нагрузки или уровня. В следующем соотношении вычитается использования:

| **Рабочая нагрузка** | **Отношение приложения DBU — уровень "стандартный"** | **Отношение приложения DBU — уровень "премиум"** |
| --- | --- | --- |
| Аналитика данных | 0,4 | 0.55 |
| Инжиниринг данных | 0,15 | 0.30 |
| Инжиниринг данных с облегченной нагрузкой | 0,07 | 0.22 |

Например при использовании количество Data Analytics — уровень "стандартный", предварительно приобретенные единицы Databricks фиксации вычитается 0,4 приращения. Когда количество данных Engineering Light — используется уровень "стандартный", предварительно приобретенные единицы Databricks фиксации вычитается 0,07 приращения

## <a name="determine-plan-use"></a>Определить использование плана

Чтобы определить план использования DBCU, перейдите на портал Azure > **резервирования** и выберите план, приобретенных Databricks. На использование до сегодняшней даты отображается с любые оставшиеся единицы. Дополнительные сведения об определении резервировании использовать, см. в разделе [см. в разделе об использовании резервирования](billing-reservation-apis.md#see-reservation-usage) статьи.

## <a name="how-discount-application-shows-in-usage-data"></a>Как показывает правила применения скидок в данных об использовании

При предварительной покупки скидка применяется к потребление Databricks, расходы по запросу отображаются как ноль в данных об использовании. Дополнительные сведения о затратах резервирования и об использовании см. в разделе [получить соглашением Enterprise резервирование затраты и использование](billing-understand-reserved-instance-usage-ea.md).

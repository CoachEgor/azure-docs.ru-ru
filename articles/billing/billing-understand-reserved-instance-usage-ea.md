# <a name="get-enterprise-agreement-reservation-costs-and-usage"></a>Получение Соглашение Enterprise затраты и использование резервирования

Данные о затратах на резервирование и сведения об использовании доступны для Соглашение Enterprise клиентов в портал Azure и API-интерфейсах RESTFUL. Эта статья поможет вам:

- Получение данных о покупках резервирования
- Сведения о подписке, группе ресурсов или ресурсе, где используется резервирование
- Расход ресурсов на использование резервирования
- Вычисление экономии резервирования
- Получение данных об использовании резервирования
- Амортизация затрат на резервирование

Плата за использование Marketplace консолидируются в данные об использовании. Вы просматриваете расходы на использование первой стороны, использование Marketplace и покупку из одного источника данных.

## <a name="reservation-charges-in-azure-usage-data"></a>Расходы на резервирование в данных об использовании Azure

Данные делятся на два отдельных набора данных: _Фактическая стоимость_ и _стоимость амортизации_. Чем отличаются эти два набора данных:

**Фактические затраты** . предоставляет данные для сверки с месячным счетом. Эти данные зарезервированы стоимость приобретения и сведения о приложении резервирования. С помощью этих данных можно указать, какая подписка, Группа ресурсов или ресурс получали скидку резервирования в определенный день. Еффективеприце для использования, который получает скидку на резервирование, равен нулю.

**Стоимость амортизации** — этот набор данных аналогичен набору данных "фактические затраты", за исключением того, что еффективеприце для использования, который получает скидку резервирования, является пропорциональной стоимостью резервирования (вместо нулевой). Это поможет вам определить денежное значение потребления резервирования по подписке, группе ресурсов или ресурсу и может помочь в издержки на использование резервирования внутренним образом. Набор данных также имеет неиспользованные часы резервирования. В наборе данных нет записей о резервированиях покупок. 

Сравнение двух наборов данных:

| Data | Набор данных фактической стоимости | Набор данных по амортизации |
| --- | --- | --- |
| Резервирование покупок | Доступно в этом представлении.<br><br>  Чтобы получить этот фильтр данных в чаржетипе = &quot;Purchase&quot;. <br><br> Обратитесь к ReservationID или Ресерватионнаме, чтобы узнать, за какое резервирование взимается плата.  | Неприменимо к этому представлению. <br><br> Затраты на покупку не предоставляются в незаполненных данных. |
| еффективеприце | Для использования, которое получает скидку резервирования, значение равно нулю. | Это значение относится к почасовому распределению по стоимости резервирования для использования с скидкой резервирования. |
| Неиспользуемое резервирование (предоставляет количество часов, в течение которых резервирование не использовалось в день и денежное значение отхода) | Неприменимо в этом представлении. | Доступно в этом представлении.<br><br> Чтобы получить эти данные, отфильтрусь по &quot;чаржетипе&quot;= унуседресерватион.<br><br>  Чтобы узнать, какое резервирование было использовано, обратитесь к ReservationID или Ресерватионнаме. В течение этого дня объем резервирования был недоступен.  |
| Цена (Цена ресурса из прайс-листа) | Доступно | Доступно |

Другие сведения, доступные в данных об использовании Azure, изменились:

- Сведения о продукте и показателях. Azure не заменяет изначально потребляемый счетчик на ReservationId и Ресерватионнаме, как было раньше.
- ReservationId и Ресерватионнаме — они являются собственными полями в данных. Ранее он использовался для доступа только в AdditionalInfo.
- Продуктордерид — идентификатор заказа на резервирование, который добавляется в качестве собственного поля.
- Продуктордернаме — название продукта приобретенного резервирования.
- Срок — 12 месяцев или 36 месяцев.
- Ринормализатионратио — доступно в разделе AdditionalInfo. Это отношение, при котором резервирование применяется к записи об использовании. Если для резервирования включена гибкость размера экземпляра, она может применяться к другим размерам. Значение показывает соотношение, к которому было применено резервирование, для записи об использовании.

## <a name="get-azure-consumption-and-reservation-usage-data-using-api"></a>Получение данных об использовании и резервировании Azure с помощью API

Данные можно получить с помощью API или скачать из портал Azure.

Для получения новых данных вы вызываете [API сведений об использовании](/rest/api/consumption/usagedetails/list) с&quot; помощью API версии &quot;2019-04-01-Preview. Дополнительные сведения о терминологии см. в разделе [условия использования](billing-understand-your-usage.md). Вызывающая сторона должна быть администратором предприятия для соглашения Enterprise с помощью [портала EA](https://ea.azure.com). Администраторы предприятия только для чтения могут также получать данные.

Данные недоступны в [API-интерфейсах отчетов для корпоративных клиентов. сведения об использовании](/rest/api/billing/enterprise/billing-enterprise-api-usage-detail).

Ниже приведен пример вызова API:

```
https://management.azure.com/providers/Microsoft.Billing/billingAccounts/{enrollmentId}/providers/Microsoft.Billing/billingPeriods/{billingPeriodId}/providers/Microsoft.Consumption/usagedetails?metric={metric}&amp;api-version=2019-04-01-preview&amp;$filter={filter}
```

Дополнительные сведения о {и {Биллингпериодид} см. в статье [сведения об использовании — API списка](https://docs.microsoft.com/rest/api/consumption/usagedetails/list) .

Сведения в следующей таблице о метрике и фильтре могут помочь в решении распространенных проблем резервирования.

| **Тип данных API** | Действие вызова API |
| --- | --- |
| **Все расходы (использование и покупки)** | Замена {Metric} на Актуалкост |
| **Использование, полученное в качестве скидки на резервирование** | Замена {Metric} на Актуалкост<br><br>Замените {Filter} на: Properties/reservationId% 20Ne% 20 |
| **Использование, которое не получило скидку на резервирование** | Замена {Metric} на Актуалкост<br><br>Замените {Filter} на: Properties/reservationId% 20eq% 20 |
| **Амортизация (использование и покупки)** | Замена {Metric} на Амортизедкост |
| **Неиспользованный отчет о резервировании** | Замена {Metric} на Амортизедкост<br><br>Замените {Filter} на: Properties/Чаржетипе% 20eq% 20 ' Унуседресерватион ' |
| **Резервирование покупок** | Замена {Metric} на Актуалкост<br><br>Замените {Filter} на: Properties/Чаржетипе% 20eq% 20 "Purchase"  |
| **Возмещения** | Замена {Metric} на Актуалкост<br><br>Замените {Filter} на: Properties/Чаржетипе% 20eq% 20 "возмещение" |

## <a name="download-the-usage-csv-file-with-new-data"></a>Скачивание CSV-файла использования с новыми данными

Если вы являетесь администратором EA, вы можете скачать CSV-файл, содержащий новые данные об использовании, из портал Azure. Эти данные недоступны на портале [EA](https://ea.azure.com).

В портал Azure перейдите к пункту [Управление затратами + выставление счетов](https://portal.azure.com/#blade/Microsoft_Azure_Billing/ModernBillingMenuBlade/BillingAccounts).

1. Выберите счет выставления счетов.
2. Щелкните **Использование + расходы**.
3. Щелкните элемент **Загрузить**.  
![Пример загрузки файла данных об использовании CSV в портал Azure](./media/billing-understand-reserved-instance-usage-ea/portal-download-csv.png)
4. В разделе **Загрузка сведений об** использовании и платах в области **сведения об использовании версии 2** выберите **все расходы (использование и покупки)** и нажмите кнопку Скачать. Повторите эту операцию для платных **выплат (использование и покупки)** .

Скачанные CSV-файлы содержат фактические затраты и стоимость амортизации.

## <a name="common-cost-and-usage-tasks"></a>Общие задачи по затратам и использованию

В следующих разделах приведены распространенные задачи, которые большинство пользователей используют для просмотра данных о затратах на резервирование и использовании.

### <a name="get-reservation-purchase-costs"></a>Получение затрат на покупку резервирования

Стоимость приобретения резервирования доступна в данных фактических затрат. Фильтр для _чаржетипе = Покупка_. Чтобы определить заказ на резервирование, для которого предназначена покупка, обратитесь к Продуктордерид.

### <a name="get-underutilized-reservation-quantity-and-costs"></a>Получить недостаточно загруженное количество и затраты на резервирование

Получите данные о стоимости амортизации и фильтр для _чаржетипе_ _= унуседресерватион_. Вы получаете ежедневное неиспользуемое количество резервирования и стоимость. Можно отфильтровать данные для резервирования или заказа на резервирование с помощью полей _ReservationId_ и _продуктордерид_ соответственно. Если резервирование было использовано 100%, запись имеет количество 0.

### <a name="amortize-reservation-costs"></a>Амортизация затрат на резервирование

Получите данные об амортизации затрат и фильтр для заказа на резервирование с помощью _продуктордерид_ , чтобы получить ежедневные затраты на резервирование.

### <a name="chargeback-for-a-reservation"></a>Гибкое резервирование для резервирования

Вы можете использовать резервирование для других организаций по подписке, группам ресурсов или тегам. Данные о стоимости амортизации обеспечивают денежное значение использования резервирования в следующих типах данных:

- Ресурсы (например, виртуальная машина)
- Группа ресурсов
- Tags
- Подписка

### <a name="get-the-blended-rate-for-chargeback"></a>Получите норму смешения для гибкого управления

Чтобы определить частоту смешения, получите данные по амортизации и совокупные затраты. Для виртуальных машин можно использовать данные Метернаме или ServiceType из данных AdditionalInfo JSON. Разделите общую стоимость на количество, используемое для получения ставки смешения.

### <a name="audit-optimum-reservation-use-for-instance-size-flexibility"></a>Аудит оптимального использования резервирования для размера экземпляра

Кратное количеству с помощью _ринормализатионратио_из AdditionalInfo. Результаты указывают, сколько часов использования резервирования было применено к записи об использовании.

### <a name="determine-reservation-savings"></a>Определение экономии для резервирования

Получите данные о стоимости амортизации и отфильтруйте данные для зарезервированного экземпляра. Затем сделайте следующее:

1. Получите приблизительные затраты на оплату по мере использования. Умножьте значение _UnitPrice_ на значения _количества_ , чтобы получить оценочную стоимость оплаты по мере использования, если скидка на резервирование не была применена к использованию.
2. Получите затраты на резервирование. Вычислите значения _стоимости_ , чтобы получить денежное значение для зарезервированного экземпляра. Он включает в себя использованные и неиспользованные затраты на резервирование.
3. Вычтите затраты на резервирование из оценочной стоимости с оплатой по мере использования, чтобы получить предполагаемую экономию.

## <a name="reservation-purchases-and-amortization-in-cost-analysis"></a>Резервирование покупок и амортизация в анализе затрат

Затраты на резервирование доступны в [анализе затрат](https://aka.ms/costanalysis). По умолчанию в анализе затрат отображаются **фактические затраты**, а именно — расходы на ваш счет. Чтобы просмотреть незавершенные покупки и связать с ресурсами, которые использовали преимущество, переключитесь на **стоимость амортизации**:

![Пример, демонстрирующий выбор амортизации затрат в анализе затрат](./media/billing-understand-reserved-instance-usage-ea/portal-cost-analysis-amortized-view.png)

Группировка по типу оплаты, чтобы увидеть перерыв в использовании, покупки и возврат денег; или с помощью резервирования для распределения затрат на резервирование и по запросу. Помните только те затраты на резервирование, которые вы увидите при просмотре фактической стоимости покупки, но затраты будут выделяться для отдельных ресурсов, которые использовали бенфит при просмотре платных затрат. При просмотре платных затрат вы увидите также новый тип оплаты **унуседресерватион** .

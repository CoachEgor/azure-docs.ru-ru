# <a name="get-billing-ownership-of-azure-subscriptions-from-other-users"></a>Получить права для подписок Azure, у других пользователей на выставление счетов

Вы можете стать владельцем подписки Azure, если выходит из существующего владельца выставления счетов вашей организации, или вы хотите заплатить за подписки с помощью учетной записи выставления счетов.

Можно отправить запрос на получение владения подписки Azure от существующих владельцев в других выставления счетов. Назначение владельцем передает выставления счетов обязанности подписок в раздел счета.

Для запроса владения на выставление счетов, необходимо быть **счета владелец разделе** или **участник раздел счета**. Дополнительные сведения см. в разделе [выставляется счет разделе роли](billing-understand-mca-roles.md#invoice-section-roles-and-tasks).

Эта статья относится к учетной записи выставления счетов для клиента соглашение Microsoft. [Проверьте наличие доступа к соглашению клиента Microsoft](#check-access-to-a-microsoft-customer-agreement).

## <a name="request-billing-ownership-in-the-azure-portal"></a>Запросить владения на выставление счетов на портале Azure

1. Войдите на [портале Azure](https://portal.azure.com).

2. Выполните поиск по **Управление затратами + выставление счетов**.

   ![Снимок экрана, показывающий поиска портала Azure для управления затратами + выставление счетов](./media/billing-mca-request-billing-ownership/billing-search-cost-management-billing.png)

3. Перейдите к разделу счета. В зависимости от вашего доступа необходимо выбрать учетную запись выставления счетов или профиль выставления счетов. Выставления счетов учетной записи или профиля выберите **выставляется счет разделы** и затем раздел счета.
   <!-- Todo - Add a screenshot -->

4. Выберите **запросов на перенос** в нижнем левом углу.

5. В верхней части страницы выберите **Добавить**.

6. Введите адрес электронной почты пользователя, для которого запрашивается из владения на выставление счетов. Пользователь должен быть администратором учетной записи на учетную запись Microsoft Online служебной программы для выставления счетов или владелец учетной записи в рамках соглашения Enterprise. Дополнительные сведения см. в разделе [просмотреть учетные записи выставления счетов на портале Azure](billing-view-all-accounts.md).

   ![Снимок экрана, показывающий, добавив новый запрос на перемещение](./media/billing-mca-request-billing-ownership/mca-new-transfer-request.png)

7. Выберите **отправки запроса на передачу**.

8. Пользователь получает сообщение электронной почты с инструкциями по просмотрите свой запрос передачи.

   ![Снимок экрана передачи запроса показан просмотрите сообщение электронной почты](./media/billing-mca-request-billing-ownership/mca-review-transfer-request-email.png)

9. Чтобы утвердить запрос на перемещение, пользователь выбирает ссылку в сообщении электронной почты и следует инструкциям.

    ![Снимок экрана передачи запроса показан просмотрите сообщение электронной почты](./media/billing-mca-request-billing-ownership/mca-review-transfer-request.png)

## <a name="check-the-status-of-your-transfer-request-in-the-azure-portal"></a>Проверить состояние передачи запроса на портале Azure

1. Войдите на [портале Azure](https://portal.azure.com).

2. Выполните поиск по **Управление затратами + выставление счетов**.

   ![Снимок экрана, показывающий поиска портала Azure для управления затратами + выставление счетов](./media/billing-mca-request-billing-ownership/billing-search-cost-management-billing.png)

3. Перейдите к разделу счета. В зависимости от вашего доступа необходимо выбрать учетную запись выставления счетов или профиль выставления счетов. Выставления счетов учетной записи или профиля выберите **выставляется счет разделы** и затем раздел счета.
   <!-- Todo - Add a screenshot -->

4. Выберите **запросов на перенос** в нижнем левом углу.

5. На странице запросов передачи отображаются следующие сведения.

    ![Снимок экрана, показывающий список запросов на передачу](./media/billing-mca-request-billing-ownership/mca-view-transfer-requests.png)

   |столбец|Определение|
   |---------|---------|
   |Дата запроса|Дата, когда был отправлен запрос на перемещение|
   |Recipient|Адрес электронной почты пользователя, что вы отправили запрос на передачу владения на выставление счетов|
   |Срок действия|Дата, когда истекает срок действия запроса|
   |Статус|Состояние запроса на передачу|

    Запрос на перемещение может принимать одно из следующих состояний:

   |Статус|Определение|
   |---------|---------|
   |выполняется;|Пользователь не принял запрос на перемещение|
   |Обработка|Пользователь утвердил запрос на перемещение. Выставление счетов для подписок, выбранные пользователем передаются в раздел счета|
   |Завершено| Выставление счетов за подписки, выбранные пользователем передаются в раздел счета|
   |Завершено с ошибками|Не удалось передать запрос выполнен, но выставление счетов для некоторых подписок, выбранные пользователем|
   |Срок действия истек|Пользователь не приняли запрос на время, и срок его действия истек|
   |Canceled|Пользователь с доступом к запрос на перемещение отменен запроса|
   |Отклонено|Пользователь отклонил запрос на перемещение|

6. Выберите запрос на передачу, чтобы просмотреть сведения. На странице сведений о передачи отображаются следующие сведения:
   <!-- Todo - Add a screenshot -->

   |столбец  |Определение|
   |---------|---------|
   |Идентификатор запроса передачи|Уникальный идентификатор для запроса на передачу. Если отправить запрос в службу поддержки, совместно использовать код в службу поддержки с просьбой ускорить обработку запроса в службу поддержки Azure|
   |Перемещение запрошено|Дата, когда был отправлен запрос на перемещение|
   |Передача по запросу|Адрес электронной почты пользователя, отправившего запрос на перемещение|
   |Запрос на перемещение истекает| Дата, когда истекает срок действия запроса на передачу|
   |Адрес электронной почты получателя|Адрес электронной почты пользователя, что вы отправили запрос на передачу владения на выставление счетов|
   |Отправлена ссылка для перемещения|URL-адрес, который был отправлен пользователю для просмотра запроса передачи|

## <a name="additional-information"></a>Дополнительная информация

Дополнительные сведения о переносе подписки ниже.

### <a name="no-service-downtime"></a>Нет простоя служб

Службы Azure в подписке продолжит работу без прерывания. Переходить только отношения оплаты для подписок Azure, которые пользователь выбирает для передачи.

### <a name="disabled-subscriptions"></a>Отключенные подписки

Отключенные подписки не переносятся. Подписки должны быть в активном состоянии для передачи их владельца для выставления счетов.

### <a name="azure-resources-transfer"></a>Перемещение ресурсов Azure

Все ресурсы из подписки, такие как виртуальные машины, диски и передачи веб-сайтов.

### <a name="azure-marketplace-products-transfer"></a>Перенос продуктов Azure Marketplace

Продукты Azure Marketplace передачи ее вместе с их соответствующих подписок.

### <a name="azure-reservations-transfer"></a>Передача резервирования в Azure

Azure резервирования не перемещается автоматически вместе с подписками. [Обратитесь в службу поддержки Azure](https://portal.azure.com/?#blade/Microsoft_Azure_Support/HelpAndSupportBlade) для перемещения резервирования.

### <a name="access-to-azure-services"></a>Доступ к службам Azure

Доступ к ресурсам Azure, который был задан с помощью Azure RBAC (управления доступом на основе ролей) не затрагивается во время перехода.

### <a name="azure-support-plan"></a>План поддержки Azure

Поддержка Azure не переносятся с подписками. Если пользователь передает все подписки на Azure, попросите его отменить их план поддержки.

### <a name="charges-for-transferred-subscription"></a>Плата за передаваемые подписки

Владелец исходного выставления счетов подписки несет ответственность за все расходы, которые были выведены вплоть до завершения передачи. Раздел счета несет ответственность за накладных расходов, полученные от времени передачи и более поздних версий. Возможно, некоторые платежи, которые были использованы до передачи, но было сообщено после нее. Эти расходы отображаются в разделе вашего счета.

### <a name="supported-offers"></a>Поддерживаемые предложения

Можно передать подписку на все типы или предложений, за исключением того, CSP предоставляет.

### <a name="cancel-a-transfer-request"></a>Отменить запрос на передачу

Вы можете отменить запрос на перемещение, пока не будет утверждено или отклонено запроса. Чтобы отменить запрос на перемещение, перейдите на страницу сведений о передачи и выберите "Отмена" в нижней части страницы.

### <a name="software-as-a-service-saas-transfer"></a>Программное обеспечение как услуга (SaaS) перемещение

Продукты SaaS не будут перенесены с подписками. Попросите пользователя [обратиться в службу поддержки](https://portal.azure.com/?#blade/Microsoft_Azure_Support/HelpAndSupportBlade) для передачи прав владения выставлением счетов продуктов SaaS. Вместе с прав владения выставлением счетов пользователь может также передать права владения ресурсов. Права владения ресурсов позволяет выполнять операции управления, такие как удаление и просмотр сведений о продукте. Пользователь должен быть владельцем ресурса на продукт SaaS чтобы передать права владельца ресурсов.

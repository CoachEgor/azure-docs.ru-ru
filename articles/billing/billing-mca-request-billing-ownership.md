# <a name="get-billing-ownership-of-azure-subscriptions-from-other-accounts"></a>Получение счета за владение подписками Azure от других учетных записей

Вам может потребоваться владение подписками Azure, если существующий владелец выставления счетов выходит из организации или вы хотите платить за подписки через счет выставления счетов. Принимая владение, вы несете ответственность за выставление счетов по подпискам на вашу учетную запись.

Эта статья относится к учетной записи выставления счетов для соглашения клиента Майкрософт. [Проверьте, есть ли у вас доступ к соглашению клиента Майкрософт](#check-for-access).

Чтобы запросить права собственности на выставление счетов, необходимо быть участником раздела счета в разделе " **владелец** " или " **счет**". Дополнительные сведения см. в [статье о ролях раздела счета](billing-understand-mca-roles.md#invoice-section-roles-and-tasks).

## <a name="request-billing-ownership"></a>Запрос права владения счетами

1. Войдите в [портал Azure](https://portal.azure.com) как владелец раздела счета или участник для счета выставления счетов для соглашения клиента Майкрософт.

2. Выполните поиск по запросу **Управление затратами и выставление счетов**.

   ![Снимок экрана, показывающий портал Azure Поиск по управлению затратами + выставление счетов](./media/billing-mca-request-billing-ownership/billing-search-cost-management-billing.png)

3. Выберите **раздел счета** в левой части. В зависимости от доступа может потребоваться выбрать счет выставления счетов или профиль выставления счетов. Из учетной записи выставления счетов или профиля выберите **раздел счета**.
   
   ![Снимок экрана, показывающий выбор разделов счета](./media/billing-mca-request-billing-ownership/mca-select-invoice-sections.png)        

4. Выберите в списке раздел счета. После того как вы перейдете за владение подпиской, счет будет выставлен в этом разделе.

5. Выберите **запросы на перемещение** в нижней левой части и нажмите кнопку **Добавить**.
 
   ![Снимок экрана, показывающий выбор запросов на перемещение](./media/billing-mca-request-billing-ownership/mca-select-transfer-requests.png)

6. Введите адрес электронной почты пользователя, от которого запрашивается владение выставлением счетов. Пользователь должен быть администратором учетной записи в учетной записи выставления счетов программы Microsoft Online Service или владельцем учетной записи на Соглашение Enterprise. Дополнительные сведения см. [в разделе Просмотр учетных записей выставления счетов в портал Azure](billing-view-all-accounts.md). Выберите **Отправить запрос на передачу**.

   ![Снимок экрана, показывающий отправку запроса на передачу](./media/billing-mca-request-billing-ownership/mca-send-transfer-requests.png)

7. Пользователь получает сообщение электронной почты с инструкциями по проверке запроса на перемещение.

   ![Снимок экрана, на котором показан адрес электронной почты для запроса на перемещение проверки](./media/billing-mca-request-billing-ownership/mca-review-transfer-request-email.png)

8. Чтобы утвердить запрос на перемещение, пользователь выбирает ссылку в сообщении электронной почты и выполняет инструкции.

    ![Снимок экрана, на котором показан адрес электронной почты для запроса на перемещение проверки](./media/billing-mca-request-billing-ownership/mca-review-transfer-request.png)

## <a name="check-the-transfer-request-status"></a>Проверка состояния запроса на перемещение

1. Войдите на [портале Azure](https://portal.azure.com).

2. Выполните поиск по запросу **Управление затратами и выставление счетов**.

   ![Снимок экрана, показывающий портал Azure Поиск по управлению затратами + выставление счетов](./media/billing-mca-request-billing-ownership/billing-search-cost-management-billing.png)


3. Выберите **раздел счета** в левой части. В зависимости от доступа может потребоваться выбрать счет выставления счетов или профиль выставления счетов. Из учетной записи выставления счетов или профиля выберите **раздел счета**.
   
   ![Снимок экрана, показывающий выбор разделов счета](./media/billing-mca-request-billing-ownership/mca-select-invoice-sections.png)        

4. Выберите в списке раздел счет, для которого был отправлен запрос на перемещение.

5. Выберите **запросы на перемещение** в нижней левой части. На странице запросы на перемещение отображаются следующие сведения.

    ![Снимок экрана, на котором показан список запросов на перемещение](./media/billing-mca-request-billing-ownership/mca-select-transfer-requests-for-status.png)

   |Столбец|Определение|
   |---------|---------|
   |Дата запроса|Дата отправки запроса на перемещение|
   |Получатель|Адрес электронной почты пользователя, отправившего запрос на перемещение владельца выставления счетов|
   |Дата окончания срока действия|Дата истечения срока действия запроса|
   |Сообщение о состоянии|Состояние запроса на перемещение|

    Запрос на перемещение может иметь одно из следующих состояний:

   |Сообщение о состоянии|Определение|
   |---------|---------|
   |Выполняется|Пользователь не принял запрос на перемещение|
   |Обработка|Пользователь утвердил запрос на перемещение. Выставление счетов за подписки, выбранные пользователем, передается в раздел "счет"|
   |Завершено| Выставление счетов за подписки, выбранные пользователем, передается в раздел "счет"|
   |Завершено с ошибками|Запрос выполнен, но выставление счетов для некоторых подписок, выбранных пользователем, не удалось передать|
   |Истек срок|Пользователь не принял запрос вовремя, и срок его действия истек.|
   |Отменено|Пользователь с доступом к запросу на перемещение отменил запрос|
   |Отклонено|Пользователь отклонил запрос на перемещение|

7. Выберите запрос на перемещение, чтобы просмотреть подробные сведения. На странице сведения о переносе отображаются следующие сведения.
   
   ![Снимок экрана, показывающий список перенесенных подписок](./media/billing-mca-request-billing-ownership/mca-transfer-completed.png)
    
   |Столбец  |Определение|
   |---------|---------|
   |Идентификатор запроса на перемещение|Уникальный идентификатор запроса на перемещение. Если вы отправите запрос в службу поддержки, поделитесь ИДЕНТИФИКАТОРом со службой поддержки Azure, чтобы ускорить запрос в службу поддержки.|
   |Запрошено перемещение|Дата отправки запроса на перемещение|
   |Запрос на перемещение, запрошенный|Адрес электронной почты пользователя, отправившего запрос на перемещение|
   |Срок действия запроса на перемещение истекает| Дата истечения срока действия запроса на перемещение|
   |Адрес электронной почты получателя|Адрес электронной почты пользователя, отправившего запрос на перемещение владельца выставления счетов|
   |Отправить ссылку на перемещение получателю|URL-адрес, который был отправлен пользователю для просмотра запроса на перемещение|

## <a name="additional-information"></a>Дополнительные сведения

В следующем разделе приведены дополнительные сведения о передаче подписок.

### <a name="no-service-downtime"></a>Нет простоя служб

Службы Azure в подписке продолжают работать без перерывов. Мы переносим только отношение выставления счетов для подписок Azure, которые пользователь выбирает для передачи.

### <a name="disabled-subscriptions"></a>Отключенные подписки

Невозможно передать отключенные подписки. Подписки должны находиться в активном состоянии для перемещения владения выставлением счетов.

### <a name="azure-resources-transfer"></a>Перенос ресурсов Azure

Все ресурсы из подписок, таких как виртуальные машины, диски и веб-сайты, переносятся.

### <a name="azure-marketplace-products-transfer"></a>Перенос продуктов Azure Marketplace

Продукты Azure Marketplace переносятся вместе с соответствующими подписками.

### <a name="azure-reservations-transfer"></a>Перенос резервирования Azure

Резервирование Azure не переносится автоматически с подписками. Чтобы переместить резервирования, [обратитесь в службу поддержки Azure](https://portal.azure.com/?#blade/Microsoft_Azure_Support/HelpAndSupportBlade) .

### <a name="access-to-azure-services"></a>Доступ к службам Azure

Доступ для существующих пользователей, групп или субъектов-служб, которые были назначены с помощью (Azure RBAC (Управление доступом на основе ролей)) [.. /роле-басед-акцесс-контрол/овервиев.МД] не затрагивается во время перехода.

### <a name="azure-support-plan"></a>План поддержки Azure

Поддержка Azure не передается с подписками. Если пользователь перенесет все подписки Azure, попросите его отменить план поддержки.

### <a name="charges-for-transferred-subscription"></a>Плата за перенесенную подписку

Первоначальный владелец выставления счетов за подписки несет ответственность за любые расходы, которые были переданы до момента завершения перемещения. В разделе счета несется плата за накладные расходы с момента перемещения. Возможны некоторые платежи, которые были выполнены до перемещения, но были зарегистрированы позже. Эти расходы отображаются в разделе счета.

### <a name="cancel-a-transfer-request"></a>Отмена запроса на перемещение

Вы можете отменить запрос на перемещение, пока запрос не будет утвержден или отклонен. Чтобы отменить запрос на перемещение, перейдите на [страницу сведения о переносе](#check-the-transfer-request-status) и нажмите кнопку Отмена в нижней части страницы.

### <a name="software-as-a-service-saas-transfer"></a>Перенос программного обеспечения как услуги (SaaS)

Продукты SaaS не передаются вместе с подписками. Попросите пользователя [обратиться в службу поддержки Azure](https://portal.azure.com/?#blade/Microsoft_Azure_Support/HelpAndSupportBlade) , чтобы передавать права собственности на выставление счетов за продукты SaaS. Вместе с владельцем выставления счетов пользователь может также передавать владение ресурсами. Владение ресурсами позволяет выполнять операции управления, такие как удаление и Просмотр сведений о продукте. Для перемещения владения ресурсами пользователь должен быть владельцем ресурса в продукте SaaS.

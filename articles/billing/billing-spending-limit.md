# <a name="azure-spending-limit"></a>Предельная сумма расходов Azure

Предельная сумма расходов в Azure не дописывает расходов на сумму кредита. Все новые клиенты, которые подписываются на пробную версию Azure или предложения, включающие кредиты за несколько месяцев, имеют установленную предельную сумму расходов по умолчанию. Предельная сумма расходов — $0, и ее нельзя изменить. Например, нельзя изменить предельную сумму расходов на $100. Однако вы можете удалить предельную сумму расходов. Таким образом, у вас нет ограничения, или у вас есть ограничение в ноль, которое не позволит вам использовать большинство видов расходов. Предельная сумма расходов недоступна для подписок в некоторых планах, таких как планы и планы с оплатой по мере использования. Список всех доступных предложений Azure и сведения о доступности предельной суммы расходов см. [здесь](https://azure.microsoft.com/support/legal/offer-details/).

## <a name="reaching-a-spending-limit"></a>Достижение предельной суммы расходов

Если в результате использования взимается плата за использование ежемесячных сумм, включенных в подписку Azure, развернутые службы будут отключены в оставшейся части этого расчетного периода.

Например, если вы протратите весь кредит, входящий в подписку, развернутые ресурсы Azure будут удалены из рабочей среды, а виртуальные машины Azure будут остановлены и освобождены. Данные в учетных записях хранения доступны только для чтения.

В начале следующего расчетного периода, если в предложении подписки содержатся кредиты за несколько месяцев, ваша подписка снова включается автоматически. Затем вы можете повторно развернуть ресурсы Azure и получить полный доступ к учетным записям хранения и базам данных.

Azure отправляет уведомления по электронной почте при достижении предельной суммы расходов для подписки. Войдите в [центр управления учетной записью](https://account.windowsazure.com/Subscriptions) , чтобы просмотреть уведомления о подписках, которые достигли предельной суммы расходов.

Если у вас бесплатная пробная подписка и достигнута предельная сумма расходов, можно выполнить обновление до плана с [оплатой по мере](billing-upgrade-azure-subscription.md) использования, чтобы удалить предельную сумму расходов и автоматически включить подписку.

## <a name="remove-the-spending-limit-in-account-center"></a>Снятие ограничения на предельную сумму расходов в Центре управления учетной записью

Вы можете удалить предельную сумму расходов в любое время, пока есть допустимый метод оплаты, связанный с подпиской Azure. Для предложений, которые кредитуются на несколько месяцев, можно также включить предельную сумму расходов в начале следующего расчетного периода.

Чтобы снять ограничение, выполните следующие действия:

1. Войдите в [Центр управления учетной записью](https://account.windowsazure.com/Subscriptions).
1. Выберите подписку. Если подписка отключена из-за достижения предельной суммы расходов, щелкните уведомление: **Подписка достигла предельной суммы расходов и была отключена, чтобы предотвратить оплату.** В противном случае в области **Состояние подписки** щелкните **Снять предельную сумму расходов**.
1. Выберите нужный вам вариант.

![Выбор параметра для удаления предельной суммы расходов](./media/billing-spending-limit/remove-spending-limit.PNG)

| Параметр | Эффект |
| --- | --- |
| Удалить предельную сумму расходов на неопределенный срок | Удаление предельной суммы расходов без ее автоматического включения в начале следующего расчетного периода. |
| Удалить предельную сумму расходов для текущего расчетного периода | Удаление предельной суммы расходов с ее автоматическим включением в начале следующего расчетного периода. |

## <a name="why-you-might-want-to-remove-the-spending-limit"></a>Почему может потребоваться удалить предельную сумму расходов

Из-за установленной предельной суммы расходов у вас могут возникнуть проблемы с развертыванием или использованием некоторых сторонних служб, а также служб Майкрософт. Ниже приведены ситуации, в которых следует удалить предельную сумму расходов для подписки.

-  Вы планируете развернуть образы от таких поставщиков, как Oracle, или таких служб, как Azure DevOps Services. Эта ситуация приводит к превышению предельной суммы расходов, которая приводит к отключению подписки.
- У вас есть службы, которые вы не хотите прерывать.
- У вас есть службы или ресурсы с определенными параметрами (например, виртуальные IP-адреса), которые вы не хотите терять. При освобождении ресурсов и служб такие параметры сбрасываются.

## <a name="turn-on-the-spending-limit-after-removing"></a>Включить предельную сумму расходов после удаления

Эта функция доступна, только если предельная сумма расходов удалена без возможности восстановления. Настройте автоматическое включение этой функции в начале следующего периода выставления счетов.

1. Войдите в [Центр управления учетной записью](https://account.windowsazure.com/Subscriptions).
1. Щелкните желтый баннер, чтобы изменить настройку для предельной суммы расходов.
1. Выберите **Включить функцию предельной суммы расходов для следующего периода выставления счетов \<дата начала расчетного периода\>** .

## <a name="custom-spending-limit"></a>Настраиваемая предельная сумма расходов

Пользовательские предельные суммы расходов недоступны.

## <a name="a-spending-limit-doesnt-prevent-all-charges"></a>Предельная сумма расходов не мешает всем расходам

[Некоторые внешние службы, опубликованные в Azure Marketplace](billing-understand-your-azure-marketplace-charges.md) , нельзя использовать с кредитовыми обязательствами, и при этом может взиматься отдельная плата, даже если установлена предельная сумма расходов. Примером таких служб могут быть лицензии на Visual Studio, служба Azure Active Directory Premium, планы поддержки и большинство сторонних служб. При подготовке новой внешней службы или ресурса отображается предупреждение о дополнительной тарификации:

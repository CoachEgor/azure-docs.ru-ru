# <a name="azure-spending-limit"></a>Предельная сумма расходов Azure

Лимит оплаты в Azure не позволяет превышения суммы по кредиту. Для всех новых клиентов, которые зарегистрировали бесплатную пробную версию Azure или предложений, включающих деньги на счете на несколько месяцев установлена предельная включены по умолчанию. Она составляет 0 долл. США. Ее невозможно изменить. Лимита оплаты недоступна для планов подписки в некоторые такие обязательства и планов с оплатой по. Список всех доступных предложений Azure и сведения о доступности предельной суммы расходов см. [здесь](https://azure.microsoft.com/support/legal/offer-details/).

## <a name="reaching-a-spending-limit"></a>Достижение лимита оплаты

Если результаты использования сумму, предусмотренную для ежемесячных с вашей подпиской Azure, вы развернутые службы отключаются до конца расчетного периода.

Например если вы использовали все кредиты, включенные в подписку, ресурсы Azure, которые вы развернули удаляются из рабочей среды и виртуальные машины Azure останавливаются и отменить выделение. Данные в учетных записях хранения доступны только для чтения.

В начале следующего расчетного периода, если вашей подписки предусмотрены деньги на счете на несколько месяцев подписки повторно включается автоматически. Затем можно повторно развернуть ресурсы Azure и имеют полный доступ к учетным записям хранения и баз данных.

Azure отправляет сообщение электронной почты, который ограничивает уведомления, когда достигнете расходов для вашей подписки. Для входа на [Центр управления учетной записью](https://account.windowsazure.com/Subscriptions) для уведомления о подписках, которые достигли лимита оплаты см. в разделе.

Если вы используете бесплатную пробную подписку и достигли предельной, можно обновить до плана с [оплаты по мере использования](billing-upgrade-azure-subscription.md) цены для удаления расходов ограничить и автоматически включить подписку.

## <a name="remove-the-spending-limit-in-account-center"></a>Снятие ограничения на предельную сумму расходов в Центре управления учетной записью

Можно удалить предельную сумму расходов в любое время, до тех пор, пока имеется действительный метод оплаты с вашей подпиской Azure. Для предложений с деньгами на несколько месяцев можно также включить предельную сумму расходов в начале следующего периода выставления счетов.

Чтобы снять ограничение, выполните следующие действия:

1. Для входа на [Центр управления учетной записью](https://account.windowsazure.com/Subscriptions).
1. Выберите подписку. Если подписка отключена из-за достижения лимита оплаты, щелкните уведомление: **Подписки достигнут Расходный лимит и была отключена для предотвращения дополнительных затрат.** В противном случае в области **Состояние подписки** щелкните **Снять предельную сумму расходов**.
1. Выберите нужный вам вариант.

![Выбор параметра для удаления предельной суммы расходов](./media/billing-spending-limit/remove-spending-limit.PNG)

| Параметр | Эффект |
| --- | --- |
| Удалить предельную сумму расходов на неопределенный срок | Удаление предельной суммы расходов без ее автоматического включения в начале следующего расчетного периода. |
| Удалить предельную сумму расходов для текущего расчетного периода | Удаление предельной суммы расходов с ее автоматическим включением в начале следующего расчетного периода. |

## <a name="why-you-might-want-to-remove-the-spending-limit"></a>Почему может потребоваться удалить предельную сумму расходов

Из-за установленной предельной суммы расходов у вас могут возникнуть проблемы с развертыванием или использованием некоторых сторонних служб, а также служб Майкрософт. Ниже приведены ситуации, где следует удалить предельную сумму расходов в своей подписке.

-  Вы планируете развернуть образы от таких поставщиков, как Oracle, или таких служб, как Azure DevOps Services. Эта ситуация будет превышена предельная почти мгновенно и ваша подписка отключена.
- У вас есть службы, которые вы не хотите прерваны.
- У вас есть службы или ресурсы с определенными параметрами (например, виртуальные IP-адреса), которые вы не хотите терять. При освобождении ресурсов и служб такие параметры сбрасываются.

## <a name="turn-on-the-spending-limit-after-removing"></a>Включение лимита оплаты после удаления

Эта функция доступна, только если предельная сумма расходов удалена без возможности восстановления. Настройте автоматическое включение этой функции в начале следующего периода выставления счетов.

1. Для входа на [Центр управления учетной записью](https://account.windowsazure.com/Subscriptions).
1. Щелкните желтый баннер, чтобы изменить настройку для предельной суммы расходов.
1. Выберите **Включить функцию предельной суммы расходов для следующего периода выставления счетов \<дата начала расчетного периода\>** .

## <a name="custom-spending-limit"></a>Custom предельная сумма расходов

Пользовательские предельные суммы расходов недоступны.

## <a name="a-spending-limit-doesnt-prevent-all-charges"></a>Предельная сумма расходов не предотвращает все расходы

[Некоторые внешние службы, опубликованные в Azure Marketplace](billing-understand-your-azure-marketplace-charges.md) нельзя использовать с кредитами подписок и могут повлечь за собой отдельная плата, даже в том случае, если ваша предельная имеет значение. Примером таких служб могут быть лицензии на Visual Studio, служба Azure Active Directory Premium, планы поддержки и большинство сторонних служб. При подготовке новой внешней службы или ресурса отображается предупреждение о дополнительной тарификации:

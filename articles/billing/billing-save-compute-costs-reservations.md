# <a name="what-are-azure-reservations"></a>Общие сведения о резервированиях в Azure

Резервирование Azure помогает сэкономить деньги путем внесения предоплаты за использование виртуальной машины, вычислительной мощности Базы данных SQL, пропускной способности Azure Cosmos DB или других ресурсов Azure за один или три года. Предоплата позволяет получить скидку на использование ресурсов. Резервирование может сократить ваши расходы на виртуальную машину, вычислительную мощность Базы данных SQL, Azure Cosmos DB или другие ресурсы до 72 % с оплатой по мере использования. Резервирования предоставляют скидку при выставлении счетов и не влияют на состояние выполнения ваших ресурсов.

Резервирование можно приобрести на [портале Azure](https://ms.portal.azure.com/?microsoft_azure_marketplace_ItemHideKey=Reservations&Microsoft_Azure_Reservations=true#blade/Microsoft_Azure_Reservations/ReservationsBrowseBlade).

## <a name="why-buy-a-reservation"></a>Зачем покупать резервирования?

Если у вас есть виртуальные машины, Azure Cosmos DB или баз данных SQL, которые работают длительное время, покупки резервирования дает наиболее экономичное решение. Например если вы постоянно работают четыре экземпляра службы без резервирования, плата взимается по ставкам повременной оплаты. Если вы покупаете резервирование для этих ресурсов, вы немедленно получаете скидка на резервирование. К ресурсам больше не будут применяться ставки оплаты по мере использования.

## <a name="charges-covered-by-reservation"></a>Расходы, связанная с резервированием

Планы обслуживания:

- **Зарезервированного экземпляра виртуальной машины** -резервирование относится только стоимость вычислений виртуальной машины. Оно не включает оплату дополнительного программного обеспечения, сетей и хранилищ.
- **Azure Cosmos DB зарезервированная емкость** -резервирование покрывает пропускную способность для ваших ресурсов. Но она не распространяется на расходы хранилища и сети.
- **База данных SQL защищены виртуальное** — только вычислительные затраты включены с резервированием. Лицензия оплачивается отдельно.

Если вы используете виртуальные машины Windows и Базу данных SQL, расходы на лицензирование вам поможет сократить [Преимущество гибридного использования Azure](https://azure.microsoft.com/pricing/hybrid-benefit/).

## <a name="whos-eligible-to-purchase-a-reservation"></a>Кто может приобрести резервирование?

Чтобы купить план, необходимо иметь роль владельца подписки в подписку с оплатой по мере использования (MS-AZR - 003P или MS-AZR - 0023P) или Enterprise (MS-AZR - 0017P или MS-AZR - 0148P). Поставщиков облачных решений можно использовать портал Azure или [центра партнеров](/partner-center/azure-reservations) для приобретения Azure резервирования.

Клиенты с соглашением Enterprise можно ограничить покупки администраторам EA, отключив **Добавление зарезервированных экземпляров** параметр на портале EA. EA "Администраторы" должен быть владельцем подписки для по крайней мере одна подписка EA для покупки резервирования. Параметр полезен для предприятий, желающих централизованной группе приобрести резервирование для влияния различных центров. После приобретения централизованное команд можно добавить владельцев center стоимость резервирования. Владельцы затем можно ограничить область резервирования в свои подписки. Центральная группа не имеют доступа владельца подписки, на котором приобретено резервирования.

Скидка резервирования применяется только к ресурсам, связанным с подписками, приобретенными по программе Enterprise, CSP и отдельные планы с оплатой.

## <a name="scope-reservations"></a>Резервирование областей

Вы можете задать область резервирования группам ресурсов или подписку. Настройка области для резервирования выбирает применяемыми экономия резервирования. При настройке резервирования в группу ресурсов, скидки резервирования применяются только к группе ресурсов, не всю подписку.

### <a name="reservation-scoping-options"></a>Параметры области для резервирования

С ресурсом группы области вы существует три варианта для определения области резервирования, в зависимости от потребностей.

- **Единый области действия группы ресурсов** — применяется скидка на резервирование в соответствующие ресурсы в выбранной группе ресурсов только.
- **Единый область подписки** — применяется скидка на резервирование в соответствующие ресурсы в выбранной подписке.
- **Общая область** — применяется скидка на резервирование к соответствующим ресурсам в соответствующих подписках, которые находятся в контексте выставления счетов. Для клиентов с соглашением Enterprise контекст выставления счетов является регистрации. Для отдельных подписок с оплатой выставления счетов область имеет право все подписки, созданные администратором учетной записи.

Во время применения скидки резервирования от использования, Azure обрабатывает резервирования в следующем порядке:

1. Резервирования, которые относятся к группе ресурсов
2. Одна область резервирования
3. Общая область резервирования

В одной группе ресурсов можно получить скидки резервирования из нескольких резервирования, в зависимости от того, как определить область бронирования.

### <a name="scope-a-reservation-to-a-resource-group"></a>Область резервирования для группы ресурсов

Вы можете задать область резервирования в группу ресурсов, при покупке резервирования или задать область обнаружения после покупки. Необходимо быть владельцем подписки, чтобы определить область резервирования для группы ресурсов.

Чтобы задать область, перейдите к статье [приобрести резервирование](https://ms.portal.azure.com/#blade/Microsoft\_Azure\_Reservations/CreateBlade/referrer/Browse\_AddCommand) страницы на портале Azure. Выберите тип резервирования, который вы хотите приобрести. На **выберите продукт, который вы хотите приобрести** форма выбора, изменение **область** значение **одной группе ресурсов** и выберите группу ресурсов.

![Пример, демонстрирующий Выбор покупки резервирования виртуальной Машины](./media/billing-save-compute-costs-reservations/select-product-to-purchase.png)

Отображаются покупки рекомендаций для группы ресурсов в резервирование виртуальных машин. Рекомендации вычисляется путем анализа вашего использования за последние 30 дней. Создается покупки рекомендация в том случае, если запущены ресурсы с помощью зарезервированных экземпляров обходится дешевле, чем стоимость выполнения ресурсы с оплатой. Дополнительные сведения о рекомендациях по покупки резервирования см. в разделе [рекомендации покупку получить зарезервированный экземпляр на основе использования шаблона](https://azure.microsoft.com/blog/get-usage-based-reserved-instance-recommendations) записи блога.

Вы всегда можете обновить область после покупки резервирования. Чтобы сделать это, перейдите к резервированию, щелкните **конфигурации** и rescope резервирование. Изменение области действия резервирование не коммерческих транзакций. Срока резервирования не изменяется. Дополнительные сведения об обновлении области см. в разделе [обновить область, вы приобретете резервирование](billing-manage-reserved-vm-instance.md#change-the-reservation-scope).

![Пример, показывающий изменения области резервирования](./media/billing-save-compute-costs-reservations/rescope-reservation-resource-group.png)

### <a name="monitor-and-optimize-reservation-usage"></a>Отслеживать и оптимизировать использование резервирования

Вы можете отслеживать потребление резервирования различными способами — через портал Azure, через API-интерфейсы или данные об использовании. Чтобы просмотреть все резервирования, которые у вас есть доступ к, перейдите к **резервирования** на портале Azure. Сетка резервирования отображает последние записанные использование в процентах для резервирования. Выберите резервирование, чтобы увидеть использование длительный срок резервирования.

Также можно получить при помощи использования резервирования [API-интерфейсы](billing-reservation-apis.md#see-reservation-usage) и из вашей [данные об использовании](billing-understand-reserved-instance-usage-ea.md#common-cost-and-usage-tasks) Если вы являетесь клиентом соглашения enterprise.

Если вы заметили, что использование группы ресурсов с заданной областью резервирования не хватает, можно обновить область резервирования одна подписка или совместно использовать ее в контексте выставления счетов. Также можно разделить резервирование и применить итоговый резервирования в разных группах ресурсов.

### <a name="other-considerations"></a>Дополнительные рекомендации

Если у вас соответствующие ресурсы в группе ресурсов, резервирование будет недостаточно. Резервирование не применяется автоматически в другую группу ресурсов или подписку при наличии низкой загрузкой.

Область резервирования не обновится автоматически, если переместить группу ресурсов из одной подписки в другую. Необходимо будет rescope резервирование. В противном случае будет редко используемые резервирования.

## <a name="discounted-subscription-and-offer-types"></a>Со скидкой подписки и типов предложений

Скидки резервирования применяются для следующих соответствующими подписками и предложить типы.

- Соглашение Enterprise (предложение номера: MS-AZR-0017P или MS-AZR - 0148 P)
- Отдельные планы с оплатой (предлагают номера: MS-AZR-0003P или MS-AZR - 0023 P)
- Подписки CSP

Ресурсы, работающие в подписке с любым другим типом, не могут воспользоваться скидкой резервирования.

## <a name="how-is-a-reservation-billed"></a>Как оплачивается резервирование?

Для покупки резервирования используется метод оплаты, привязанный к подписке. Если вы используете подписку Enterprise Agreement, стоимость резервирования вычитается из баланса денежных обязательств. Если у вас недостаточно баланса денежных обязательств для покрытия полной стоимости резервирования, вам выставляется счет на соответствующую разницу. При наличии подписки из отдельных плана с оплатой кредитной карты, которые возникают в вашей учетной записи взимается немедленно. Если вы используете оплату по накладным, эта сумма будет включена в ближайшую накладную.

## <a name="how-reservation-discount-is-applied"></a>Применение скидки резервирования

Скидка на резервирование применяется к использование ресурсов, соответствующих атрибутам, выбранным при покупке резервирования. Атрибуты включают в себя область, в которой выполняются виртуальные машины, базы данных SQL, Azure Cosmos DB или другие ресурсы. К примеру Если требуется скидки резервирования для четырех виртуальных машин Standard D2 в регионе Западная часть США, выберите подписку, в котором работают виртуальные машины.

Скидка резервирования "*использования ИТ или потерять ИТ-* «. Если у вас соответствующие ресурсы для данного часа, тогда у вас не количество резервирований для этого часа. Не удается выполнить пересылать неиспользуемые зарезервированные часов.

При завершении работы ресурса скидка на резервирование автоматически применяется к другим соответствующий ресурс в указанной области. Если нет соответствия ресурсов находятся в указанной области, а затем зарезервированный часы: *потеряны*.

Например может позднее создать ресурс и иметь сопоставления резервирование, которое используется недостаточно. В этом примере скидка на резервирование автоматически применяется к новой соответствующий ресурс.

Если виртуальные машины работают в разных подписках в одной учетной записи, выберите вариант совместного использования. Совместное использование позволяет применить скидки резервирования сразу в нескольких подписках. Область резервирования можно изменить и после покупки. Дополнительные сведения см. в статье [Управление резервированиями в Azure](billing-manage-reserved-vm-instance.md).

Скидка резервирования применяется только к ресурсам, связанным с Enterprise, CSP, или подписки с оплатой как то узле курсы. Ресурсы, работающие в подписке с любым другим типом, не могут воспользоваться скидкой резервирования.

## <a name="when-the-reservation-term-expires"></a>Когда срок резервирования

При завершении срока резервирования скидка на оплату отменяется, и для виртуальной машины, базы данных SQL, Azure Cosmos DB или другого ресурса снова применяется оплата по мере использования. Резервирования Azure не возобновляются автоматически. Чтобы сохранить скидку на оплату, необходимо приобрести новое резервирование для соответствующих служб и программного обеспечения.

## <a name="discount-applies-to-different-sizes"></a>Скидка применяется к разных размеров

Когда вы покупаете резервирование, скидку можно применить к другим экземплярам с атрибутами, которые находятся в группе того же размера. Эта функция называется гибкость размер экземпляра. Гибкость покрытия скидки зависит от типа резервирования и атрибутов, которые вы выбираете при покупке резервирования.

Планы обслуживания:

- Зарезервированные экземпляры виртуальных машин. При покупке резервирования и выберите **оптимизировано для**: **экземпляра гибкость размер**, покрытие скидка зависит от размера виртуальной Машины, выбран. Резервирование можно применить к размерам в группе виртуальных машин серии того же размера. Дополнительные сведения см. в статье [Гибкость размеров виртуальных машин при использовании зарезервированных экземпляров виртуальных машин](../virtual-machines/windows/reserved-vm-instance-size-flexibility.md).
- Резервная мощность Базы данных SQL. Покрытие скидки зависит от выбранного уровня производительности. Дополнительные сведения см. в статье [Сведения о применении скидки на резервирование к Базам данных SQL](billing-understand-reservation-charges.md).
- Резервная мощность Azure Cosmos DB. Покрытие скидки зависит от подготовленной пропускной способности. Дополнительные сведения см. в статье [Общие сведения о применении скидки на резервирование в Azure Cosmos DB](billing-understand-cosmosdb-reservation-charges.md).

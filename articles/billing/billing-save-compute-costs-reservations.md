# <a name="what-are-azure-reservations"></a>Общие сведения о резервированиях в Azure

Резервирование Azure помогает сэкономить деньги путем внесения предоплаты за использование виртуальной машины, вычислительной мощности Базы данных SQL, пропускной способности Azure Cosmos DB или других ресурсов Azure за один или три года. Предоплата позволяет получить скидку на использование ресурсов. Резервирование может сократить ваши расходы на виртуальную машину, вычислительную мощность Базы данных SQL, Azure Cosmos DB или другие ресурсы до 72 % с оплатой по мере использования. Резервирования предоставляют скидку при выставлении счетов и не влияют на состояние выполнения ваших ресурсов.

Резервирование можно приобрести на [портале Azure](https://ms.portal.azure.com/?microsoft_azure_marketplace_ItemHideKey=Reservations&Microsoft_Azure_Reservations=true#blade/Microsoft_Azure_Reservations/ReservationsBrowseBlade).

## <a name="why-buy-a-reservation"></a>Зачем покупать резервирование?

Если у вас есть виртуальные машины, Azure Cosmos DB или базы данных SQL, работающие в течение длительного периода времени, приобретение резервирования дает наиболее экономичный вариант. Например, при постоянном запуске четырех экземпляров службы без резервирования плата взимается по тарифам с оплатой по мере использования. При покупке резервирования для этих ресурсов вы немедленно получаете скидку на резервирование. К ресурсам больше не будут применяться ставки оплаты по мере использования.

## <a name="charges-covered-by-reservation"></a>Расходы, охваченные резервированием

Планы обслуживания:

- **Зарезервированный экземпляр виртуальной машины** . резервирование охватывает только затраты на вычислительные ресурсы виртуальных машин. Оно не включает оплату дополнительного программного обеспечения, сетей и хранилищ.
- **Azure Cosmos DB зарезервированная емкость** . резервирование охватывает пропускную способность, подготовленную для ваших ресурсов. Но она не распространяется на расходы хранилища и сети.
- **База данных SQL** , зарезервированная Виртуальное ядро — в резервирование включены только затраты на вычисление. Лицензия оплачивается отдельно.

Если вы используете виртуальные машины Windows и Базу данных SQL, расходы на лицензирование вам поможет сократить [Преимущество гибридного использования Azure](https://azure.microsoft.com/pricing/hybrid-benefit/).

## <a name="whos-eligible-to-purchase-a-reservation"></a>Кто может приобрести резервирование?

Чтобы приобрести план, необходимо иметь роль владельца подписки в корпоративной подписке (MS-AZR-0017P или MS-AZR-0148P) или с подпиской с оплатой по мере использования (MS-AZR-003P или MS-AZR-0023P). Поставщики облачных решений могут использовать портал Azure или [центр](/partner-center/azure-reservations) партнеров для приобретения резервирований Azure.

Клиенты Соглашение Enterprise (EA) могут ограничить покупки администраторами EA. Это можно сделать, отключив параметр **Добавить зарезервированные экземпляры** на портале EA. Для приобретения резервирования администраторы EA должны быть владельцами подписки по крайней мере одной подписки EA. Этот вариант удобен для предприятий, которые хотят, чтобы централизованная группа приобрела резервирования для различных центров затрат. После приобретения централизованные группы могут добавить к резервированиям владельцев центра затрат. Владельцы могут зарезервировать резервирование для своих подписок. Центральной команде не требуется доступ владельца подписки на место приобретения резервирования.

Скидка на резервирование применяется только к ресурсам, связанным с подписками, приобретенными в рамках предприятия, поставщика облачных решений (CSP) и отдельных планов с оплатой по мере использования.

## <a name="scope-reservations"></a>Резервирование области

Вы можете ограничить резервирование для подписки или групп ресурсов. Задание области для резервирования позволяет выбрать, где будет применяться экономия резервирования. При создании резервирования для группы ресурсов скидки на резервирование применяются только к группе ресурсов, а не ко всей подписке.

### <a name="reservation-scoping-options"></a>Параметры области резервирования

В зависимости от потребностей в области группы ресурсов есть три варианта резервирования.

- **Единственная область группы ресурсов** — применяет скидку резервирования к соответствующим ресурсам в выбранной группе ресурсов.
- **Область действия одной подписки** — применяет скидку резервирования к соответствующим ресурсам в выбранной подписке.
- **Общая область** — применяет скидку резервирования для сопоставления ресурсов в подходящих подписках, которые находятся в контексте выставления счетов. Для клиентов Соглашение Enterprise в качестве контекста выставления счетов используется регистрация. Для отдельных подписок с оплатой по мере использования в качестве области выставления счетов используются все подходящие подписки, созданные администратором учетной записи.

При применении скидок на использование резервирование Azure обрабатывает резервирование в следующем порядке:

1. Резервирования, ограниченные группой ресурсов
2. Резервирование одной области
3. Резервирование общей области

Одна группа ресурсов может получать скидки на резервирование из нескольких резервирований в зависимости от того, как вы ограничиваете область резервирования.

### <a name="scope-a-reservation-to-a-resource-group"></a>Область резервирования для группы ресурсов

Вы можете ограничить резервирование группой ресурсов при покупке резервирования или задать область после покупки. Вы должны быть владельцем подписки, чтобы ограничить область резервирования группой ресурсов.

Чтобы задать область, перейдите на страницу [резервирования покупки](https://ms.portal.azure.com/#blade/Microsoft\_Azure\_Reservations/CreateBlade/referrer/Browse\_AddCommand) в портал Azure. Выберите тип резервирования, который вы хотите купить. На странице **выберите продукт, который требуется приобрести** в форме выбора измените значение области на одну группу ресурсов. Затем выберите группу ресурсов.

![Пример, демонстрирующий выбор покупки для резервирования виртуальной машины](./media/billing-save-compute-costs-reservations/select-product-to-purchase.png)

Отобразятся рекомендации по приобретению группы ресурсов в резервировании виртуальной машины. Рекомендации рассчитываются путем анализа использования за последние 30 дней. Если затраты на выполнение ресурсов с зарезервированными экземплярами дешевле, чем затраты на выполнение ресурсов с оплатой по мере использования, устанавливается рекомендация по покупке. Дополнительные сведения о рекомендациях по приобретению резервирования см. в статье получение зарезервированных данных о [приобретении экземпляра на основе шаблона использования](https://azure.microsoft.com/blog/get-usage-based-reserved-instance-recommendations).

Вы всегда можете обновить область после приобретения резервирования. Для этого перейдите к резервированию, щелкните **Конфигурация**и переустановите область резервирования. Переопределение области резервирования не является коммерческой транзакцией. Срок резервирования не изменился. Дополнительные сведения об обновлении области см. в разделе [обновление области после приобретения резервирования](billing-manage-reserved-vm-instance.md#change-the-reservation-scope).

![Пример, показывающий изменение области резервирования](./media/billing-save-compute-costs-reservations/rescope-reservation-resource-group.png)

### <a name="monitor-and-optimize-reservation-usage"></a>Мониторинг и оптимизация использования резервирования

Вы можете отслеживать использование резервирования несколькими способами: с помощью портал Azure, через API или с помощью данных об использовании. Чтобы просмотреть все резервирования, к которым у вас есть доступ, перейдите к разделу **резервирования** в портал Azure. В сетке резервирования отображается последний записанный процент использования для резервирования. Щелкните резервирование, чтобы просмотреть долгосрочное использование резервирования.

Вы также можете получить сведения об использовании резервирования с помощью [API](billing-reservation-apis.md#see-reservation-usage) -интерфейсов и [данных использования](billing-understand-reserved-instance-usage-ea.md#common-cost-and-usage-tasks) , если вы являетесь клиентом соглашения Enterprise.

Если вы заметили, что использование резервируемой области группы ресурсов мало, можно обновить область резервирования до одной подписки или поделиться ею в контексте выставления счетов. Можно также разделить резервирование и применить итоговые резервирования к разным группам ресурсов.

### <a name="other-considerations"></a>Дополнительные рекомендации

Если у вас нет соответствующих ресурсов в группе ресурсов, резервирование будет считаться недостаточным. Резервирование не применяется автоматически к другой группе ресурсов или подписке с низким уровнем использования.

Область резервирования не обновляется автоматически при перемещении группы ресурсов из одной подписки в другую. Область не обновляется при удалении группы ресурсов. Вам потребуется переопределение [области резервирования](billing-manage-reserved-vm-instance.md#change-the-reservation-scope). В противном случае резервирование будет считаться недостаточным.

## <a name="discounted-subscription-and-offer-types"></a>Подписки и типы предложений со скидкой

Скидки на резервирование применяются к следующим подходящим подпискам и типам предложений.

- Соглашение Enterprise (номера предложений: MS-AZR-0017P или MS-AZR-0148P)
- Отдельные планы с оплатой по мере использования (номера предложений: MS-AZR-0003P или MS-AZR-0023P)
- Подписки CSP

Ресурсы, работающие в подписке с любым другим типом, не могут воспользоваться скидкой резервирования.

## <a name="how-is-a-reservation-billed"></a>Как оплачивается резервирование?

Для покупки резервирования используется метод оплаты, привязанный к подписке. Если вы используете подписку Enterprise Agreement, стоимость резервирования вычитается из баланса денежных обязательств. Если баланс денежных обязательств не охватывает затраты на резервирование, плата за нее взимается. Если у вас есть подписка из отдельного плана с тарифами с оплатой по мере использования, плата за кредитную карту, имеющуюся в вашей учетной записи, будет взиматься сразу. Когда счет выставляется по счету, вы получаете оплату за ваш следующий счет.

## <a name="how-reservation-discount-is-applied"></a>Как применяется скидка на резервирование

Скидка на резервирование применяется к использованию ресурсов, соответствующих атрибутам, выбранным при покупке резервирования. Атрибуты включают в себя область, в которой выполняются соответствующие виртуальные машины, базы данных SQL, Azure Cosmos DB или другие ресурсы. Например, если требуется скидка на резервирование для четырех стандартных виртуальных машин D2 в регионе "Западная часть США", выберите подписку, в которой выполняются виртуальные машины.

Скидка на резервирование — "*использовать или потерять*ее". Если у вас нет соответствующих ресурсов в течение какого-либо часа, вы потеряли количество резервирования в течение этого часа. Вы не можете переносить неиспользуемые зарезервированные часы.

При завершении работы ресурса скидка на резервирование автоматически применяется к другому соответствующему ресурсу в указанной области. Если в указанной области не найдены соответствующие ресурсы, зарезервированные часы теряются.

Например, вы можете позднее создать ресурс и использовать подходящее резервирование. Скидка на резервирование автоматически применяется к новому соответствующему ресурсу.

Если виртуальные машины работают в разных подписках в одной учетной записи, выберите вариант совместного использования. Совместное использование позволяет применить скидки резервирования сразу в нескольких подписках. Область резервирования можно изменить и после покупки. Дополнительные сведения см. в статье [Управление резервированиями в Azure](billing-manage-reserved-vm-instance.md).

Скидка на резервирование применяется только к ресурсам, связанным с корпоративными, CSP или подписками с оплатой по мере использования. Ресурсы, работающие в подписке с любым другим типом, не могут воспользоваться скидкой резервирования.

## <a name="when-the-reservation-term-expires"></a>По истечении срока действия условия резервирования

По окончании срока резервирования срок оплаты истекает. Плата за виртуальную машину, базу данных SQL, Azure Cosmos DB или другой ресурс оплачивается по цене оплаты по мере использования. Резервирование Azure не продлится автоматически. Чтобы сохранить скидку на оплату, необходимо приобрести новое резервирование для соответствующих служб и программного обеспечения.

## <a name="discount-applies-to-different-sizes"></a>Скидка применяется к различным размерам

Когда вы покупаете резервирование, скидку можно применить к другим экземплярам с атрибутами, которые находятся в группе того же размера. Эта функция называется гибкостью размера экземпляра. Гибкость покрытия скидки зависит от типа резервирования и атрибутов, которые вы выбираете при покупке резервирования.

Планы обслуживания:

- Зарезервированные экземпляры виртуальных машин. При покупке резервирования и выборе оптимизирована **для гибкости размера экземпляра**покрытие скидки зависит от выбранного размера виртуальной машины. Резервирование можно применить к размерам в группе виртуальных машин серии того же размера. Дополнительные сведения см. в статье [Гибкость размеров виртуальных машин при использовании зарезервированных экземпляров виртуальных машин](../virtual-machines/windows/reserved-vm-instance-size-flexibility.md).
- Резервная мощность Базы данных SQL. Покрытие скидки зависит от выбранного уровня производительности. Дополнительные сведения см. в статье [Сведения о применении скидки на резервирование к Базам данных SQL](billing-understand-reservation-charges.md).
- Резервная мощность Azure Cosmos DB. Покрытие скидки зависит от подготовленной пропускной способности. Дополнительные сведения см. в статье [Общие сведения о применении скидки на резервирование в Azure Cosmos DB](billing-understand-cosmosdb-reservation-charges.md).

## <a name="reservation-notifications"></a>Уведомления о резервировании

В зависимости от того, как вы платите за подписку Azure, мы отправляем уведомления о резервировании следующим пользователям в вашей организации. Уведомления отправляются для различных событий, включая:

- Купить
- Предстоящие сроки действия резервирования
- Срок действия
- Обновления
- Отмена
- Изменение области

Для клиентов с подписками EA:
- Уведомление о покупке отправляется покупателям и контактным данным уведомления EA.
- Другие уведомления жизненного цикла резервирования отправляются только контактам по уведомлениям EA.
- Пользователи, добавленные в резервирование с помощью разрешения RBAC (IAM), не получают уведомлений по электронной почте.

Для клиентов с отдельными подписками:
- Покупатель получает уведомление о покупке.
- Во время покупки владелец учетной записи выставления счетов по подписке получает уведомление о покупке.
- Владелец учетной записи получает все остальные уведомления.

# <a name="transfer-billing-ownership-of-an-azure-subscription-to-another-account"></a>Перенос владельца выставления счетов для подписки Azure на другую учетную запись

Вам может потребоваться переместить владение выставлением счетов для подписки Azure, если вы выходите из организации или хотите, чтобы подписка была выставлена на другую учетную запись. Передача владения счетами на другую учетную запись предоставляет администраторам в новой учетной записи разрешения на выполнение задач выставления счетов, таких как изменение способа оплаты, просмотр расходов и Отмена подписки.

Если вы хотите обеспечить владение выставлением счетов, но изменить тип подписки, см. статью [Переключение подписки Azure на другое предложение](billing-how-to-switch-azure-offer.md). Если вы хотите контролировать, кто может управлять ресурсами в подписке, см. статью [встроенные роли для ресурсов Azure](https://docs.microsoft.com/azure/role-based-access-control/built-in-roles).

Если вы являетесь клиентом Соглашение Enterprise (EA), Администраторы предприятия могут передавать права собственности на выставление счетов за ваши подписки между учетными записями. Дополнительные сведения см. в статье о [переносе владения выставлением счетов для подписок Соглашение Enterprise (EA)](#transfer-billing-ownership-of-enterprise-agreement-ea-subscriptions).

## <a name="transfer-billing-ownership-of-an-azure-subscription"></a>Перенос владения выставлением счетов для подписки Azure

1. Войдите в [портал Azure](https://portal.azure.com) в качестве администратора учетной записи выставления счетов с подпиской, которую требуется переместить. Чтобы узнать, являетесь ли вы администратором, ознакомьтесь с [часто задаваемыми вопросами](#faq).

1. Выполните поиск по **Управление затратами + выставление счетов**.

   ![Снимок экрана, показывающий поиск на портале Azure](./media/billing-subscription-transfer/billing-search-cost-management-billing.png)

1. На левой панели выберите **подписки** . В зависимости от доступа может потребоваться выбрать область выставления счетов и выбрать **подписки** или **подписки Azure**.

1. Выберите параметр **передавать владение выставлением счетов** для подписки, которую требуется переместить. 

   ![Выберите подписку для перемещения](./media/billing-subscription-transfer/billing-select-subscription-to-transfer.png)

1. Введите адрес электронной почты пользователя, который является администратором выставления счетов учетной записи, которая будет новым владельцем подписки.

1. Если вы переносите подписку на учетную запись в другом клиенте Azure AD, выберите, если хотите переместить подписку в клиент новой учетной записи. Дополнительные сведения см. [в статье передача подписки в учетную запись в другом клиенте Azure AD](#transferring-subscription-to-an-account-in-another-azure-ad-tenant) .

    > [!IMPORTANT]
    >
    > Если вы решили переместить подписку на клиент Azure AD новой учетной записи, все назначения [управления доступом на основе ролей (RBAC)](../role-based-access-control/overview.md) для управления ресурсами в подписке будут удалены без возможности восстановления. Только пользователь в новой учетной записи, который принимает ваш запрос на перемещение, будет иметь доступ к управлению ресурсами в подписке. Дополнительные сведения см. [в статье передача подписки пользователю в другом клиенте Azure AD](../active-directory/managed-identities-azure-resources/known-issues.md). Кроме того, можно снять флажок подписки для клиента Azure AD, чтобы передавать владение выставлением счетов без перемещения подписки на клиент новой учетной записи. В этом случае существующие разрешения RBAC для управления ресурсами Azure будут поддерживаться.
  
    ![Страница отправки передачи](./media/billing-subscription-transfer/billing-send-transfer-request.PNG)

1. Выберите **Отправить запрос на передачу**.

1. Пользователь получит письмо с инструкциями по рассмотрению вашего запроса на передачу прав.

   ![Отправленное получателю сообщение электронной почты с передачей подписки](./media/billing-subscription-transfer/billing-receiver-email.png)

1. Чтобы утвердить запрос на перемещение, пользователь выбирает ссылку в сообщении электронной почты и выполняет инструкции. Пользователю нужно будет выбрать метод оплаты, который будет использоваться для оплаты подписки. Кроме того, если у пользователя нет учетной записи Azure, ему пришлось бы зарегистрироваться для получения новой учетной записи. 

   ![Первая страница процесса передачи подписки](./media/billing-subscription-transfer/billing-accept-ownership-step1.png)

   ![Вторая страница процесса передачи подписки](./media/billing-subscription-transfer/billing-accept-ownership-step2.png)

   ![Вторая страница процесса передачи подписки](./media/billing-subscription-transfer/billing-accept-ownership-step3.png)

1. Готово! Теперь подписка передана.

## <a name="transferring-subscription-to-an-account-in-another-azure-ad-tenant"></a>Передача подписки на учетную запись в другом клиенте Azure AD

При регистрации в Azure создается клиент Azure Active Directory (AD). Клиент представляет вашу учетную запись. Клиент используется для управления доступом к подпискам и ресурсам.

При создании новой подписки она размещается в клиенте Azure AD вашей учетной записи. Если вы хотите предоставить другим пользователям доступ к подписке или ее ресурсам, необходимо пригласить их для приподключения к клиенту. Это помогает управлять доступом к подпискам и ресурсам.

При переносе права владения подпиской на учетную запись в другом клиенте Azure AD можно переместить подписку в клиент новой учетной записи. При этом все пользователи, группы или субъекты-службы, у которых есть [доступ на основе ролей (RBAC)](../role-based-access-control/role-assignments-portal.md) для управления подписками и их ресурсами, теряют доступ к ним. Только пользователь в новой учетной записи, который принимает ваш запрос на перемещение, будет иметь доступ к управлению ресурсами. Чтобы предоставить доступ пользователям, у которых изначально есть доступ, новый владелец должен [вручную добавить этих пользователей в подписку](https://docs.microsoft.com/azure/role-based-access-control/role-assignments-portal).


## <a name="transferring-visual-studio-mpn-and-pay-as-you-go-devtest-subscriptions"></a>Передача подписок Visual Studio, MPN и оплаты по мере использования подписки для разработки и тестирования

Для подписок Visual Studio и Microsoft Partner Network используются ежемесячные кредитные платежи Azure. При переносе этих подписок ваш кредит недоступен в учетной записи выставления счетов. В подписке используется кредит в целевой учетной записи выставления счетов. Например, если Боб передает Visual Studio Enterpriseную подписку на учетную запись Джейн в 9 сентября, а Мария принимает его. После завершения перемещения подписка начнет использовать кредит в учетной записи Джейн. Кредит будет сбрасываться каждый месяц в 9-й день.

## <a name="next-steps-after-accepting-billing-ownership"></a>Дальнейшие действия после принятия прав собственности на выставление счетов

Если вы приняли владение выставлением счета за подписку Azure, мы рекомендуем ознакомиться со следующими шагами:

1. Просмотрите и обновите роли администратора служб, соадминистраторов и другие роли RBAC. Дополнительные сведения см. в статьях [Добавление или изменение администраторов подписки Azure](billing-add-change-azure-subscription-administrator.md) и [Управление доступом с помощью RBAC и портала Azure](../role-based-access-control/role-assignments-portal.md).
1. Обновите учетные данные, связанные со службами этой подписки, включая следующие:
   1. Сертификаты управления, которые предоставляют пользователю доступ к ресурсам подписки с правами администратора. Дополнительные сведения см. в статье [Создание и передача сертификата управления для Azure](../cloud-services/cloud-services-certs-create.md).
   1. Ключи доступа для служб, таких как служба хранилища. Дополнительные сведения см. в статье [Об учетных записях хранения Azure](../storage/common/storage-create-storage-account.md).
   1. Учетные данные удаленного доступа для служб, таких как Виртуальные машины Azure.
1. Если вы работаете с партнером, мы рекомендуем изменить идентификатор партнера для этой подписки. Вы можете обновить идентификатор партнера на [портале Azure](https://portal.azure.com). Дополнительные сведения см. в статье [связывание идентификатора партнера с учетными записями Azure](billing-partner-admin-link-started.md) .

## <a name="frequently-asked-questions-faq-for-senders"></a>Часто задаваемые вопросы о отправителях

Эти часто задаваемые вопросы относятся к пользователям, которые перемещают владение выставлением счетов для подписки Azure другой учетной записи.

### <a name="whoisaa"></a>Кто является администратором выставления счетов для учетной записи?

Администратор выставления счетов — это пользователь, имеющий разрешение на Управление выставлением счетов для учетной записи. Они имеют право на доступ к выставлению счетов на [портал Azure](https://portal.azure.com) и выполняют различные задачи выставления счетов, такие как создание подписок, просмотр и оплата счетов, а также обновление методов оплаты.

Чтобы указать учетные записи, для которых вы являетесь администратором выставления счетов, выполните следующие действия.

1. Посетите [страницу "Управление затратами + выставление счетов" в портал Azure](https://portal.azure.com/#blade/Microsoft_Azure_Billing/ModernBillingMenuBlade/Overview).
1. Выберите **все области выставления счетов** в области слева. 
1. На странице подписки перечислены все подписки, для которых вы являетесь администратором выставления счетов.

### <a name="does-everything-transfer-including-resource-groups-vms-disks-and-other-running-services"></a>Можно ли передать все группы ресурсов, виртуальные машины, диски и другие работающие службы?

Все ресурсы, такие как виртуальные машины, диски и веб-сайты, переносятся в новую учетную запись. Однако при переносе подписки на учетную запись в другом клиенте Azure AD любые [роли администратора](billing-add-change-azure-subscription-administrator.md) и назначения [управления доступом на основе ролей (RBAC)](../role-based-access-control/role-assignments-portal.md) в подписке [не переносятся](#transferring-subscription-to-an-account-in-another-azure-ad-tenant). Кроме того, [Регистрация приложений](../active-directory/develop/quickstart-v1-integrate-apps-with-azure-ad.md) и другие службы для конкретного клиента не передаются вместе с подпиской.

### <a name="can-i-transfer-ownership-to-an-account-in-another-country"></a>Можно ли передавать владение учетной записи в другой стране?
К сожалению, переносы в разных странах не могут быть выполнены в портал Azure. Чтобы переместить подписку в разных странах, [обратитесь в службу поддержки](https://portal.azure.com/?#blade/Microsoft_Azure_Support/HelpAndSupportBlade).

### <a name="i-am-an-administrator-on-two-accounts-can-i-transfer-a-subscription-from-one-of-my-accounts-to-another"></a>Я являюсь администратором двух учетных записей. Можно ли перенести подписку из одной из моих учетных записей в другую?
Да, можно передавать подписку между вашими учетными записями. Учетные записи концептуально считаются учетными записями двух разных пользователей, поэтому вы можете использовать описанные выше действия для передачи подписок между вашими учетными записями.

### <a name="does-a-subscription-transfer-result-in-any-service-downtime"></a>Возникают ли простои в работе службы при передаче подписки?

Если вы передаете подписку в учетную запись в том же клиенте Azure AD, то не влияет на ресурсы, работающие в подписке. Однако при переносе подписки в учетную запись другого клиента и принятии решения о перемещении подписки на клиент все пользователи, группы и субъекты-службы, которым назначен [доступ на основе ролей (RBAC)](../role-based-access-control/overview.md) для управления ресурсами в подписке, теряют доступ к ним. . Это может привести к простою службы.

### <a name="do-users-in-new-account-have-access-to-usage-and-billing-history"></a>Есть ли у пользователей в новой учетной записи доступ к журналу использования и выставлению счетов?

Пользователям в новой учетной записи будут доступны лишь сведения о стоимости подписки за последний месяц. Остальная часть журнала использования и выставления счетов не передается вместе с подпиской.

### <a name="how-do-i-migrate-data-and-services-for-my-azure-subscription-to-new-subscription"></a>Как перенести данные и службы из подписки Azure в новую подписку?

Если не удается передать права владения подпиской, то можно перенести ресурсы вручную. Ознакомьтесь со статьей [Move resources to new resource group or subscription](../azure-resource-manager/resource-group-move-resources.md) (Перемещение ресурсов в новую группу ресурсов или подписку).

### <a name="if-i-transfer-a-visual-studio-or-microsoft-partner-network-subscription-does-my-credit-carry-forward-with-the-subscription-in-the-new-account"></a>Если я переберу подписку Visual Studio или Microsoft Partner Network, будет ли кредит переноситься с подпиской в новой учетной записи?

Нет, ваш кредит недоступен в новой учетной записи. Пользователь, который принимает запрос на перемещение, должен иметь лицензию Visual Studio для принятия запроса на перемещение. В подписке используется кредит Visual Studio, доступный в учетной записи пользователя. Дополнительные сведения см. в статьях [Передача Visual Studio, Microsoft Partner Network (MPN) и оплата по мере использования для разработки и тестирования подписок](#transferring-visual-studio-mpn-and-pay-as-you-go-devtest-subscriptions).


## <a name="frequently-asked-questions-faq-for-recipients"></a>Часто задаваемые вопросы о получателях

Эти часто задаваемые вопросы относятся к пользователям, которые принимают владение выставлением счетов для подписки Azure из другой учетной записи.

### <a name="if-i-take-over-billing-ownership-of-a-subscription-from-another-account-do-users-in-that-account-continue-to-have-access-to-my-resources"></a>Если я занимаю владение выставлением счета для подписки из другой учетной записи, у пользователей этой учетной записи будет по-прежнему доступ к моим ресурсам?

Да. Однако если ваша учетная запись находится в клиенте Azure AD, который отличается от клиента подписки, и пользователь, отправивший запрос на передачу, перемещает подписку в клиент вашей учетной записи, любые [роли администратора](billing-add-change-azure-subscription-administrator.md) и [Управление доступом на основе ролей (RBAC). ](../role-based-access-control/role-assignments-portal.md)назначения удаляются. Чтобы просмотреть пользователей, имеющих доступ [на основе ролей (RBAC)](../role-based-access-control/overview.md) для управления ресурсами в подписке, выполните следующие действия.

1. Посетите [страницу подписки в портал Azure](https://portal.azure.com/#blade/Microsoft_Azure_Billing/SubscriptionsBlade).
1. Выберите подписку, которую необходимо проверить, а затем выберите **Управление доступом (IAM)** в левой области.
1. Выберите **назначения ролей** в верхней части страницы. На странице назначения ролей перечислены все пользователи, имеющие доступ RBAC к подписке.

Даже если назначения [управления доступом на основе ролей (RBAC)](../role-based-access-control/role-assignments-portal.md) удаляются во время перемещения, пользователи в исходной учетной записи владельца могут по-прежнему иметь доступ к подписке с помощью некоторых механизмов безопасности, включая следующие.

* Сертификаты управления, которые предоставляют пользователю доступ к ресурсам подписки с правами администратора. Дополнительные сведения см. в статье о [создании и передаче сертификата управления для Azure](../cloud-services/cloud-services-certs-create.md).
* Ключи доступа для служб, таких как служба хранилища. Дополнительные сведения см. в статье [Об учетных записях хранения Azure](../storage/common/storage-create-storage-account.md).
* Учетные данные удаленного доступа для служб, таких как Виртуальные машины Azure.

Если получателю необходимо ограничить доступ к своим ресурсам, рекомендуется обновить все секреты, связанные со службой. Большинство ресурсов можно обновить с помощью следующих действий:

  1. Войдите на [портале Azure](https://portal.azure.com).
  2. В главном меню выберите **Все ресурсы**.
  3. Выберите ресурс.
  4. На странице ресурсов щелкните **Параметры**. Здесь можно просматривать и обновлять существующие секретные данные.

### <a name="if-i-take-over-the-billing-ownership-of-a-subscription-in-the-middle-of-the-billing-cycle-do-i-have-to-pay-for-the-entire-billing-cycle"></a>Если я занимаю владение выставлением счета за подписку в середине цикла выставления счетов, нужно ли платить за весь цикл выставления счетов?

Ваша учетная запись отвечает за оплату за использование, которое сообщается с момента перемещения. Возможно, некоторые ресурсы были использованы до передачи, но об этом было сообщено после нее. Использование будет включено в счет для вашей учетной записи.

### <a name="can-i-use-a-different-payment-method"></a>Можно ли использовать другой способ оплаты?

Да. Принимая запрос на перенос, вы можете выбрать существующий метод оплаты, связанный с вашей учетной записью, или добавить новый метод оплаты.

## <a name="troubleshooting"></a>Устранение неполадок

### <a id="no-button"></a>Почему не отображается кнопка "Перенос подписки"?

Перемещение по подписке самообслуживания недоступно для вашей учетной записи выставления счетов. Сейчас мы не поддерживаем передачу прав собственности на выставление счетов по подпискам в учетных записях Соглашение Enterprise (EA) в портал Azure. Кроме того, учетные записи соглашения клиента Майкрософт, созданные при работе с представителями корпорации Майкрософт, не поддерживают передачу прав собственности на выставление счетов. 

### <a id="no-button"></a>Почему тип подписки не поддерживает перенос? 

Не все типы подписок поддерживают перенос владения выставлением счетов. Чтобы просмотреть список типов подписок, поддерживающих передачи, см. раздел [Поддерживаемые типы подписок](#supported-subscription-types) .

### <a id="no-button"></a>Почему я получаю сообщение об отказе в доступе при попытке переместить владение выставлением счета за подписку? 

Эта ошибка возникает, если вы пытаетесь переместить подписку на план Microsoft Azure и у вас нет необходимых разрешений. Чтобы переместить подписку на план Microsoft Azure, необходимо быть владельцем или участником в разделе счета, к которому взимается подписка. Дополнительные сведения см. в [разделе Управление подписками для счета](billing-understand-mca-roles.md#manage-subscriptions-for-invoice-section).

# <a name="transfer-billing-ownership-of-an-azure-subscription-to-another-account"></a>Передача владения подпиской Azure другой учетной записи

Может потребоваться передачи прав владения выставлением счетов подписки Azure, если вы покидаете вашей организации, или подписки, выставление счетов в другую учетную запись. Передача прав владения выставлением счетов в другую учетную запись предоставляет администраторам в новой учетной записи разрешение для выполнения задач выставления счетов, таких как изменить метод оплаты, Просмотр расходов и отменить подписку.

Если вы хотите сохранить прав владения выставлением счетов, но изменение типа вашей подписки, см. в разделе [переключить свою подписку Azure на другое предложение](billing-how-to-switch-azure-offer.md). Если вы хотите контролировать, кто может управлять ресурсами в подписке, см. в разделе [встроенных ролей ресурсов Azure](https://docs.microsoft.com/azure/role-based-access-control/built-in-roles).

Если вы являетесь клиентом действия соглашения Enterprise, администраторов предприятия можно передачи прав владения выставлением счетов подписки между учетными записями. Дополнительные сведения см. в разделе [передачи прав владения выставлением счетов подписки Enterprise Agreement (EA)](#transfer-billing-ownership-of-enterprise-agreement-ea-subscriptions).

1. Выберите **подписок** в левой области. В зависимости от доступа, может потребоваться выбрать область, выставления счетов, а затем выберите **подписок** или **подписок Azure**.

1. Выберите **передачи прав владения выставлением счетов** для подписки, которую вы хотите передать. 

   ![Выберите подписку для передачи](./media/billing-subscription-transfer/billing-select-subscription-to-transfer.png)

1. Введите адрес электронной почты пользователя, являющегося администратором выставления счетов, учетной записи, необходимо быть владельцем новой подписки и выберите пункт **отправки запроса на передачу**.

    > [!IMPORTANT]

Только новый владелец будет иметь доступ к управлению ресурсами в подписке. ## <a name="transferring-subscription-to-an-account-in-another-azure-ad-tenant"></a>Передача подписки в учетную запись в другом клиенте Azure AD

Клиент Azure Active Directory (AD) создается автоматически при регистрации в Azure. Клиент представляет вашу учетную запись. Использовать клиент для управления доступом к подпискам и ресурсам.

При создании новой подписки, она размещается в клиенте Azure AD вашей учетной записи. Если вы хотите предоставить другим пользователям доступ к подписке или ресурсы, необходимо пригласить его в клиенте. Это помогает управлять доступом к подпискам и ресурсам.

При передаче прав владения выставлением счетов вашей подписки в учетную запись в другом клиенте Azure AD, подписки перемещается на новую учетную запись клиента. Все пользователи, группы или субъекты-службы, которые имели право [на основе ролей (RBAC) доступа](../role-based-access-control/overview.md) для управления ресурсами в подписке утратят его. Только пользователь, в новой учетной записи, который принимает запрос передачи будут доступны для управления ресурсами. Чтобы предоставить доступ пользователям, которые изначально имели право доступа, новый владелец бы [вручную добавить этих пользователей к подписке](https://docs.microsoft.com/azure/role-based-access-control/role-assignments-portal).


## <a name="transferring-visual-studio-microsoft-partner-network-mpn-and-pay-as-you-go-devtest-subscriptions"></a>Передача Visual Studio, Microsoft Partner Network (MPN) и подписки с оплатой по мере разработки и тестирования

Подписки Visual Studio и Microsoft Partner Network имеют ежемесячный повторяющихся кредит Azure, связанных с ними. При передаче этих подписок, деньги на счете в целевой учетной записи выставления счетов недоступна. Подписка использует кредита в целевой учетной записи выставления счетов. Например если Алексей передает подписку Visual Studio Enterprise для учетной записи на 9 сентября и Мария Джейн принимает передачу. После завершения передачи подписки начнет использовать кредит в учетной записи Джейн. Кредит приведет к сбросу каждый месяц на 9-е число.

## <a name="next-steps-after-accepting-billing-ownership"></a>Дальнейшие действия после принятия владения на выставление счетов

Если вы приняли выставления счетов владельца подписки Azure, мы рекомендуем просмотреть следующие шаги:

1. Просмотрите и обновите роли администратора служб, соадминистраторов и другие роли RBAC. Дополнительные сведения см. в статьях [Добавление или изменение администраторов подписки Azure](billing-add-change-azure-subscription-administrator.md) и [Управление доступом с помощью RBAC и портала Azure](../role-based-access-control/role-assignments-portal.md).
1. Обновите учетные данные, связанные со службами этой подписки, включая следующие:
   1. Сертификаты управления, которые предоставляют пользователю доступ к ресурсам подписки с правами администратора. Дополнительные сведения см. в статье [Создание и передача сертификата управления для Azure](../cloud-services/cloud-services-certs-create.md).
   1. Ключи доступа для служб, таких как служба хранилища. Дополнительные сведения см. в статье [Об учетных записях хранения Azure](../storage/common/storage-create-storage-account.md).
   1. Учетные данные удаленного доступа для служб, таких как Виртуальные машины Azure.
1. Если вы работаете с партнером, мы рекомендуем изменить идентификатор партнера для этой подписки. Вы можете обновить идентификатор партнера на [портале Azure](https://portal.azure.com). Дополнительные сведения см. в разделе [ссылка на Партнера к учетной записи Azure](billing-partner-admin-link-started.md)

## <a name="frequently-asked-questions-faq-for-senders"></a>Часто задаваемые вопросы (FAQ) для отправителей

Эти вопросы и ответы по применяются к пользователям, которые передачи выставления счетов владения подпиской Azure другой учетной записи.

### <a name="whoisaa"></a> Кто является администратором выставления счетов для учетной записи?

Администратор выставления счетов — это лицо, имеет разрешение на управление выставлением счетов для учетной записи. Они права на доступ к выставления счетов на [портала Azure](https://portal.azure.com) и выполнять различные задачи выставления счетов, как создавать подписки, представление и оплаты счета-фактуры и обновления способов оплаты.

Для идентификации подписки, для которых вы являетесь администратором выставления счетов, сделайте следующее:

1. Посетите [управление затратами + выставление счетов страница на портале Azure](https://portal.azure.com/#blade/Microsoft_Azure_Billing/ModernBillingMenuBlade/Overview).
1. Выберите **подписок** в левой области. В зависимости от доступа, может потребоваться выбрать область, выставления счетов, а затем выберите **подписок** или **подписок Azure**
1. Страницы «подписки» перечислены все подписки, для которых вы являетесь администратором выставления счетов.

### <a name="does-everything-transfer-including-resource-groups-vms-disks-and-other-running-services"></a>Можно ли передать все группы ресурсов, виртуальные машины, диски и другие работающие службы?

Все ресурсы, такие как виртуальные машины, диски и передачи веб-сайтов для новой учетной записи. Тем не менее, если при передаче подписки в учетной записи в Azure AD, клиента, любой [ролей администратора](billing-add-change-azure-subscription-administrator.md) и [управления доступом на основе ролей (RBAC)](../role-based-access-control/role-assignments-portal.md) назначения в подписке [— нет Передача](#transferring-subscription-to-an-account-in-another-azure-ad-tenant). Кроме того [регистрация приложений](../active-directory/develop/quickstart-v1-integrate-apps-with-azure-ad.md) и другие службы для конкретного клиента не будут перенесены вместе с подпиской.

### <a name="can-i-transfer-ownership-to-an-account-in-another-country"></a>Как передавать право собственности на учетную запись в другой стране?
К сожалению кросс-страны невозможно выполнить передачу на портале Azure. На передачу вашей подписки в странах, [обратитесь в службу поддержки](https://portal.azure.com/?#blade/Microsoft_Azure_Support/HelpAndSupportBlade).

### <a name="i-am-an-administrator-on-two-accounts-can-i-transfer-a-subscription-from-one-of-my-accounts-to-another"></a>Я являюсь администратором на две учетные записи. Можно ли передать подписку из одной из моих учетных записей в другую?
Да, вы можете передать подписки между учетными записями. Учетные записи по существу считаются учетные записи двух разных пользователей, поэтому приведенные выше действия можно использовать для передачи подписки между учетными записями.

### <a name="does-a-subscription-transfer-result-in-any-service-downtime"></a>Возникают ли простои в работе службы при передаче подписки?

Если передать подписку пользователю в том же клиенте Azure AD, нет никакого влияния на ресурсы, работающие в подписке.  Тем не менее, если передача подписки пользователю в другом клиенте, всех пользователей, групп и субъектов-служб, которые имели право [на основе ролей (RBAC) доступа](../role-based-access-control/overview.md) для управления ресурсами в подписке утратят его. 

### <a name="does-the-recipient-have-access-to-usage-and-billing-history"></a>Имеет ли получатель доступ к журналу использования и выставленным счетам?

Единственная информация, доступная для получателя — стоимость за прошлый месяц для своей подписки. Остальная часть использовании и выставленных ранее не передается с подпиской

### <a name="how-do-i-migrate-data-and-services-for-my-azure-subscription-to-new-subscription"></a>Как перенести данные и службы из подписки Azure в новую подписку?

Если не удается передать права владения подпиской, то можно перенести ресурсы вручную. Ознакомьтесь со статьей [Move resources to new resource group or subscription](../azure-resource-manager/resource-group-move-resources.md) (Перемещение ресурсов в новую группу ресурсов или подписку).

### <a name="if-i-transfer-a-visual-studio-or-microsoft-partner-network-subscription-does-my-credit-carry-forward-with-the-subscription-in-the-new-account"></a>Если передать подписку Visual Studio или Microsoft Partner Network, закончится переноса с подпиской в новой учетной записи?

Нет, деньги на счете недоступна в новой учетной записи. Пользователь, который принимает запрос на перемещение должен иметь лицензию Visual Studio, чтобы принять запрос на перемещение. Подписка использует кредит Visual Studio, который доступен в учетной записи пользователя. Дополнительные сведения см. в разделе [подписки передачи Visual Studio, Microsoft Partner Network (MPN) и оплатой по мере разработки и тестирования](#transferring-visual-studio-microsoft-partner-network-mpn-and-pay-as-you-go-devtest-subscriptions)


## <a name="frequently-asked-questions-faq-for-recipients"></a>Часто задаваемые вопросы (FAQ) для получателей

Эти вопросы и ответы по применяются к пользователям, которые делаются счетами подписки Azure из другой учетной записи.

### <a name="if-i-take-over-billing-ownership-of-a-subscription-from-another-account-do-users-in-that-account-continue-to-have-access-to-my-resources"></a>Если я принимаю Управление счетами подписки от другой учетной записи, у пользователей, в том, что учетная запись по-прежнему иметь доступ к моим ресурсам?

Если подписка передается в учетную запись, в том же клиенте Azure AD, всех пользователей, групп и субъектов-служб, которые имели право [на основе ролей (RBAC) доступа](../role-based-access-control/overview.md) для управления ресурсами в подписке сохранить свой доступ. Чтобы просмотреть пользователей, имеющих доступ RBAC к подписке, сделайте следующее:

1. Посетите [странице подписки на портале Azure](https://portal.azure.com/#blade/Microsoft_Azure_Billing/SubscriptionsBlade).
1. Выберите подписку, которую нужно проверить, а затем щелкните **управление доступом (IAM)** в левой области.
1. Выберите **назначения ролей** в верхней части страницы. Страница назначения роли список всех пользователей, имеющих доступ RBAC для подписки.

Если подписка передается в учетную запись в другой клиент Azure AD, все пользователи, группы и субъекты-службы, которые имели право [на основе ролей (RBAC) доступа](../role-based-access-control/overview.md) для управления ресурсами в подписке утратят его. Тем не менее даже если они не имеют доступа RBAC больше, они по-прежнему имеют доступ к подписке через некоторые механизмы безопасности, включая:

* Сертификаты управления, которые предоставляют пользователю доступ к ресурсам подписки с правами администратора. Дополнительные сведения см. в статье о [создании и передаче сертификата управления для Azure](../cloud-services/cloud-services-certs-create.md).
* Ключи доступа для служб, таких как служба хранилища. Дополнительные сведения см. в статье [Об учетных записях хранения Azure](../storage/common/storage-create-storage-account.md).
* Учетные данные удаленного доступа для служб, таких как Виртуальные машины Azure.

Если получателю следует ограничить, доступ к их ресурсам, они подумайте над переработкой таких все секреты, связанные со службой. Большинство ресурсов можно обновить с помощью следующих действий:

  1. Войдите на [портале Azure](https://portal.azure.com).
  2. В главном меню выберите **Все ресурсы**.
  3. Выберите ресурс.
  4. На странице ресурсов щелкните **параметры**. Здесь можно просматривать и обновлять существующие секретные данные.

### <a name="if-i-take-over-the-billing-ownership-of-a-subscription-in-the-middle-of-the-billing-cycle-do-i-have-to-pay-for-the-entire-billing-cycle"></a>Если я принимаю Управление счетами подписки в середине цикла выставления счетов, нужно ли платить за весь цикл выставления счетов?

Учетной записи должен оплатить все использованные ресурсы, полученные от времени передачи и более поздних версий. Возможно, некоторые ресурсы были использованы до передачи, но об этом было сообщено после нее. Эти ресурсы будут включены в счет вашей учетной записи.

### <a name="can-i-use-a-different-payment-method"></a>Можно ли использовать другой метод оплаты?

Да. При принятии условий запроса на передачу, можно выбрать существующий метод оплаты, связанную с вашей учетной записи или добавьте новый метод оплаты.

## <a name="troubleshooting"></a>Устранение неполадок

### <a id="no-button"></a>Почему не отображается кнопка "Перенос подписки"?

К сожалению самостоятельная передача подписки недоступен для учетной записи выставления счетов. В настоящее время мы не поддерживаем перенос out выставления счетов владение учетной записи Enterprise Agreement (EA) на портале Azure. Кроме того соглашении клиента Microsoft учетные записи, созданные при работе с Microsoft sales не поддерживают передача выставления счетов подписки. 

### <a id="no-button"></a> Почему Моя подписка введите поддержки передачи? 

К сожалению не все типы подписок поддерживают выставления счетов передачи права владения. Чтобы просмотреть список типов подписки, которые поддерживают передачу, см. в разделе [поддерживаемые типы подписки](#supported-subscription-types)

### <a id="no-button"></a> Почему я получаю отказ в доступе при попытке передачи прав владения выставлением счетов подписки? 

Вы увидите эту ошибку, если вы пытаетесь передать подписку Microsoft Azure план, и у вас нет необходимых разрешений. Чтобы перенести план подписки Microsoft Azure, необходимо быть владельцем или участником, в разделе счета, к которому выставляются в подписке. Дополнительные сведения см. в разделе [управление подписками для раздела счета](billing-understand-mca-roles.md#manage-subscriptions-for-invoice-section).

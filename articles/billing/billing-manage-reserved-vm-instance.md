# <a name="manage-reservations-for-azure-resources"></a>Управление резервированиями для ресурсов Azure

После приобретения резервирования Azure может потребоваться применить резервирование к другой подписке, изменить, кто может управлять резервированием, или изменить область резервирования. Вы также можете разделить резервирование на два, чтобы применить некоторые купленные экземпляры к другой подписке.

Если вы приобрели Azure Reserved Virtual Machine Instances, вы можете изменить параметр оптимизации для резервирования. Скидка на резервирование может применяться к виртуальным машинам в одной и той же серии. Кроме того, можно зарезервировать ресурсы центра обработки данных для определенного размера виртуальной машины. И следует попытаться оптимизировать резервирование, чтобы они были полностью использованы.

## <a name="reservation-order-and-reservation"></a>Заказ на резервирование и резервирование

При покупке резервирования создаются два объекта: **Заказ** на резервирование и **резервирование**.

На момент покупки в качестве заказа на резервирование будет зарезервировано по одному. Такие действия, как разбиение, слияние, частичное возмещение или Exchange, создают новые резервирования в **порядке резервирования**.

Чтобы просмотреть заказ на резервирование, перейдите в раздел **резервирования** > выберите резервирование, а затем щелкните **идентификатор заказа**на резервирование.

![Пример сведений о заказе на резервирование с ИДЕНТИФИКАТОРом заказа на резервирование ](./media/billing-manage-reserved-vm-instance/reservation-order-details.png)

Резервирование наследует разрешения от его порядка резервирования.

## <a name="change-the-reservation-scope"></a>Изменение области резервирования

 Скидка на резервирование применяется к виртуальным машинам, базам данных SQL, Azure Cosmos DB или другим ресурсам, которые соответствуют резервированию и запускаются в области резервирования. Контекст выставления счетов зависит с подписки, использованной для покупки резервирования.

Вот как можно обновить область резервирования.

1. Войдите на [портале Azure](https://portal.azure.com).
2. Выберите **Все службы** > **Резервирование**.
3. Выберите резервирование.
4. Выберите **Параметры** > **Конфигурация**.
5. Изменение область.

Если вы переменяете с Shared на одну область, вы можете выбрать только те подписки, в которых вы являетесь владельцем. Можно выбрать только подписки в том же контексте выставления счетов, что и резервирование.

Область применима только к отдельным подпискам с оплатой по мере использования (предложения MS-AZR-0003P или MS-AZR-0023P), корпоративные предложения MS-AZR-0017P или MS-AZR-0148P или CSP.

## <a name="add-or-change-users-who-can-manage-a-reservation"></a>Добавление или изменение пользователей, которые могут управлять резервированием

Вы можете делегировать управление резервированием, добавив людей в роли в заказе на резервирование или резервировании. По умолчанию пользователь, который помещает заказ на резервирование и Администратор учетной записи, имеет роль владельца в заказе на резервирование и резервировании.

Вы можете управлять доступом к заказам и резервированиям резервирования независимо от подписок, которые получают скидку резервирования. При предоставлении пользователям разрешений на Управление заказом на резервирование или резервированием он не предоставляет им разрешения на управление подпиской. Аналогично, если вы предоставляете пользователям разрешения на управление подпиской в области резервирования, она не предоставляет ей права на Управление заказом на резервирование или резервированием.

Чтобы выполнить обмен или возврат денег, пользователь должен иметь доступ к заказу на резервирование. При предоставлении пользователю разрешений лучше предоставлять разрешения для заказа на резервирование, а не резервирования.


Чтобы делегировать управление доступом к резервированию, сделайте следующее.

1. Войдите на [портале Azure](https://portal.azure.com).
2. Выберите **Все службы** > **Резервирование**, чтобы отобразить список доступных вам резервирований.
3. Выберите резервирование, доступ к которому требуется делегировать другим пользователям.
4. Выберите **Управление доступом (IAM)** .
5. Выберите **Добавить назначение ролей** > **Роль** > **Владелец**. Если вы хотите предоставить ограниченный доступ, выберите другую роль.
6. Укажите адрес электронной почты пользователя, для которого хотите добавить роль "Владелец".
7. Выберите пользователя, а затем щелкните **Сохранить**.

## <a name="split-a-single-reservation-into-two-reservations"></a>Разделение отдельного резервирования на два резервирования

 После приобретения нескольких экземпляров ресурсов в резервировании может потребоваться назначить экземпляры в резервировании разным подпискам. По умолчанию все экземпляры имеют одну область. Это либо одна подписка, либо общая область. Например, вы приобрели 10 экземпляров резервирования и указали область действия подписки A. Теперь может потребоваться изменить область для семи резервирований на подписку A, а остальные три — на подписку б. разделение резервирования позволяет распределять экземпляры для детального управления областью. Можно упростить выделение для подписок, выбрав общую область. Но с целью управления затратами или бюджетирования можно распределить количество между определенными подписками.

 Можно разделить резервирование на два резервирования с помощью PowerShell, интерфейса командной строки или API.

### <a name="split-a-reservation-by-using-powershell"></a>Разделение резервирования с помощью PowerShell

1. Получите идентификатор резервирования, выполнив следующую команду.

    ```powershell
    # Get the reservation orders you have access to
    Get-AzReservationOrder
    ```

2. Получите сведения о резервировании.

    ```powershell
    Get-AzReservation -ReservationOrderId a08160d4-ce6b-4295-bf52-b90a5d4c96a0 -ReservationId b8be062a-fb0a-46c1-808a-5a844714965a
    ```

3. Разделите резервирование на две части и распределите экземпляры.

    ```powershell
    # Split the reservation. The sum of the reservations, the quantity, must equal the total number of instances in the reservation that you're splitting.
    Split-AzReservation -ReservationOrderId a08160d4-ce6b-4295-bf52-b90a5d4c96a0 -ReservationId b8be062a-fb0a-46c1-808a-5a844714965a -Quantity 3,2
    ```
4. Чтобы изменить область, выполните следующую команду.

    ```powershell
    Update-AzReservation -ReservationOrderId a08160d4-ce6b-4295-bf52-b90a5d4c96a0 -ReservationId 5257501b-d3e8-449d-a1ab-4879b1863aca -AppliedScopeType Single -AppliedScope /subscriptions/15bb3be0-76d5-491c-8078-61fe3468d414
    ```

## <a name="cancel-exchange-or-refund-reservations"></a>Отмена, Обмен или возмещение в резервированиях

Вы можете отменять, обмениваться и возмещать деньги на резервирования с определенными ограничениями. Дополнительные сведения см. в статье [самостоятельная служба Exchange и возмещение для резервирования Azure](billing-azure-reservations-self-service-exchange-and-refund.md).

## <a name="change-optimize-setting-for-reserved-vm-instances"></a>Изменение параметра оптимизации для зарезервированных экземпляров виртуальной машины

 После приобретения зарезервированного экземпляра виртуальной машины выберите гибкость размеров для экземпляров или приоритет емкости. Гибкость размеров для экземпляров применяет скидку на резервирование для других виртуальных машин в той же [группе размеров виртуальных машин](https://aka.ms/RIVMGroups). Приоритет емкости назначает приоритет емкости центра обработки данных для развертываний. Этот параметр обеспечивает дополнительную уверенность в возможности запуска экземпляров виртуальных машин, когда они понадобятся.

Для общей области резервирования по умолчанию обеспечивается гибкость размера экземпляра. Емкость центра обработки данных не является приоритетной для развертываний виртуальной машины.

Вы можете оптимизировать резервирование, где указана одна область, по приоритету производительности вместо гибкости размера экземпляра виртуальной машины.

Чтобы обновить параметр оптимизации для резервирований:

1. Войдите на [портале Azure](https://portal.azure.com).
2. Выберите **Все службы** > **Резервирования**.
3. Выберите резервирование.
4. Выберите **Параметры** > **Конфигурация**.
5. Измените параметр **Оптимизировать для**.

## <a name="optimize-reservation-use"></a>Оптимизация использования резервирования

Служба "резервирование Azure" является результатом только использования устойчивых ресурсов. Когда вы выполняете покупку резервирования, вы платите за то, что по сути на 100% возможных ресурсов в течение одного или трех лет. Попробуйте максимально увеличить резервирование, чтобы получить максимально эффективное использование и экономию. В следующих разделах объясняется, как отслеживать резервирование и оптимизировать его использование.

### <a name="view-reservation-use-in-the-azure-portal"></a>Просмотр использования резервирования в портал Azure

Одним из способов просмотра использования резервирования является портал Azure.

1. Войдите на [портале Azure](https://portal.azure.com/).
2. Выберите **все службы** > [**резервирования**](https://portal.azure.com/#blade/Microsoft_Azure_Reservations/ReservationsBrowseBlade) и запишите **использование (%)** для резервирования.  
  ![Изображение, показывающее список резервирований](./media/billing-manage-reserved-vm-instance/reservation-list.png)
3. Выберите резервирование.
4. Изучите тренд использования резервирования с течением времени.  
  ![Изображение, показывающее использование резервирования ](./media/billing-manage-reserved-vm-instance/reservation-utilization-trend.png)

### <a name="view-reservation-use-with-api"></a>Просмотр использования резервирования с помощью API

Если вы являетесь клиентом Соглашение Enterprise (EA), вы можете программным способом просмотреть, как используются резервирования в вашей организации. Вы получаете неиспользуемое резервирование через данные об использовании. При проверке расходов на резервирование следует помнить, что данные делятся между фактической ценой и затратами. Фактические затраты предоставляют данные для сверки ежемесячных выставляемых счетов. Он также содержит сведения о стоимости приобретения и резервировании для резервирования. Стоимость амортизации аналогична фактической стоимости, за исключением того, что эффективная цена за использование резервирования распределяется пропорционально. Неиспользованные часы резервирования отображаются в фиксированных данных о затратах. Дополнительные сведения о данных об использовании для клиентов EA см. в статье [получение Соглашение Enterprise резервирование затрат и использование](billing-understand-reserved-instance-usage-ea.md).

Для других типов подписок используйте сводки по [резервированию API — список по заказам на резервирование и резервированию](/rest/api/consumption/reservationssummaries/listbyreservationorderandreservation).

### <a name="optimize-your-reservation"></a>Оптимизация резервирования

Если вы обнаружите, что резервирования вашей организации используются не так:

- Убедитесь, что виртуальные машины, создаваемые вашей организацией, соответствуют размеру виртуальной машины для резервирования.
- Убедитесь, что включена гибкость размера экземпляра. Дополнительные сведения см. в разделе [Изменение области для резервирования](#change-optimize-setting-for-reserved-vm-instances).
- Измените область резервирования на _Общая_ , чтобы она применялась более широко. Дополнительные сведения см. [в разделе Изменение области для резервирования](#change-the-reservation-scope).
- Рассмотрите возможность обмена неиспользуемым количеством. Дополнительные сведения см. в разделе [отмены и обмен](#cancel-exchange-or-refund-reservations).

# <a name="create-an-additional-azure-subscription-for-microsoft-customer-agreement"></a>Создание дополнительной подписки Azure для Клиентского соглашения Майкрософт

Создайте дополнительные подписки для учетной записи выставления счетов, чтобы настроить отдельные среды для разработки и тестирования, обеспечения безопасности или для изоляции данных по соображениям соответствия.

В этой статье рассматривается учетная запись выставления счетов для Клиентского соглашения Майкрософт. [Проверьте наличие доступа к Клиентскому соглашению Майкрософт](#check-access). Если вы хотите создать подписки для других типов учетных записей выставления счетов см. статью [Создание дополнительной подписки на портале Azure](billing-create-subscription.md).

Для создания подписки необходимо быть **владельцем раздела счетов**, **участником раздела счетов** или **автором подписки Azure**. Дополнительные сведения см. в статье [Роли и задачи выставления счетов в подписке](billing-understand-mca-roles.md#subscription-billing-roles-and-tasks). Для предоставления другим пользователям разрешения на создание подписок Azure для вашей учетной записи выставления счетов, см. раздел [Предоставление другим пользователям разрешения на создание подписок Azure](#give-others-permission).

## <a name="create-a-subscription"></a>Создание подписки

1. Войдите на [портале Azure](https://portal.azure.com).

2. Выполните поиск по запросу **Подписки**.

   ![Снимок экрана: поиск подписок на портале](./media/billing-mca-create-subscription/billing-search-subscriptions.png)

3. Выберите **Добавить**.

4. Если у вас есть доступ к нескольким учетным записям выставления счетов, выберите учетную запись выставления счетов для Клиентского соглашения Майкрософт.

   ![Снимок экрана: страница создания подписки](./media/billing-mca-create-subscription/billing-mca-create-azure-subscription.png)

5. Выберите профиль выставления счетов. Плата за подписку будет взиматься по выбранному профилю выставления счетов. Если у вас есть доступ только к одному профилю выставления счетов, выбор будет недоступен.

6. Выберите раздел счета. Плата за подписку будет взиматься по этому разделу профиля выставления счетов. Если у вас есть доступ только к одному разделу счета, выбор будет недоступен.

7. Выберите план для подписки. Выберите **План Azure для разработки и тестирования**, если вы планируете использовать эту подписку для рабочих нагрузок разработки или тестирования, иначе используйте **План Microsoft Azure**. Если у вас есть доступ только к одному плану, выбор будет недоступен.

8. Введите имя подписки. Имя помогает легко найти подписку на портале Azure.

9. Нажмите кнопку **Создать**.

## <a name="give-others-permission"></a>Предоставление разрешения другим пользователям

Добавьте пользователей в качестве авторов подписки Azure в раздел счета, чтобы предоставить им разрешение на создание подписок Azure.

1. Войдите на [портале Azure](https://portal.azure.com).

2. Выполните поиск по фразе **Управление затратами + выставление счетов**.

   ![Снимок экрана: поиск подписок на портале](./media/billing-mca-create-subscription/billing-search-cost-management-billing.png)

3. Перейдите к разделу счета. В зависимости от уровня вашего доступа вам, возможно, потребуется выбрать учетную запись выставления счетов или профиль выставления счетов. В учетной записи выставления счетов или в профиле выставления счетов выберите **Разделы счета** и затем нужный раздел счета из списка. Ко всем подпискам, созданным пользователями, будут выставляться счета в этом разделе счетов.
   
   ![Снимок экрана: выбор разделов счетов](./media/billing-mca-create-subscription/mca-select-invoice-sections.png)        

4. Выберите**Управление доступом (IAM)** в верхней левой части.

5. В верхней части страницы выберите **Добавить**.

6. Выберите роль **Создатель подписки Azure**.

7. Введите адрес электронной почты пользователя, которому необходимо предоставить доступ.

8. Щелкните **Сохранить**.

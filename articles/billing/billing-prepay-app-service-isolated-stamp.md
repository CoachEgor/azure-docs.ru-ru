# <a name="save-costs-for-azure-app-service-isolated-stamp-fee-with-reserved-capacity"></a>Экономия расходов на отработку денежных отметок Служба приложений (цен. категория "Изолированный") Azure с зарезервированной емкостью

Вы можете сэкономить деньги на тарифных отметках Azure Служба приложений (цен. категория "Изолированный"), применив резервирование для использования отметки в течение трех лет. Чтобы приобрести зарезервированную емкость для оплаты изолированной отметки, необходимо выбрать регион Azure, в котором будет развернута отметка, и количество отметок для приобретения.

При приобретении резервирования использование оплаты изолированной отметки, которое соответствует атрибутам резервирования, больше не взимается по тарифам с оплатой по мере использования. Резервирование автоматически применяется к числу изолированных меток, соответствующих зарезервированной области и региону. Не нужно назначать резервирование изолированной метке. Резервирование не применяется к работникам, поэтому все прочие ресурсы, связанные с этим штампом, начисляются отдельно.

После истечения срока действия зарезервированной емкости изолированные метки продолжают работать, но выставляются по тарифам с оплатой по мере использования. Резервирования не продлеваются автоматически.

## <a name="determine-the-right-reservation-to-purchase"></a>Определение правильного резервирования для приобретения

Приобретая резервирование, вы фиксируете использование зарезервированных количеств в течение следующих трех лет. Проверьте данные об использовании, чтобы определить, сколько Служба приложений (цен. категория "Изолированный")ых меток вы будете постоянно использовать и может использоваться в будущем.

Кроме того, убедитесь, что вы понимаете, как изолированная метка выдает показания счетчиков Linux или Windows.

- По умолчанию пустая изолированная метка выдает счетчик штампов Windows. Например, без развернутых рабочих ролей. Он продолжит создавать этот счетчик, если в отметке развернуты рабочие роли Windows.
- При развертывании рабочей роли Linux индикатор изменяется на индикатор отметки Linux.
- В случаях, когда развертываются рабочие процессы Linux и Windows, отметка выдает счетчик Windows.

Таким образом, индикатор отметки может измениться между Windows и Linux в течение срока.

Приобретите резервирование Windows, если в отметке есть один или несколько рабочих ролей Windows. Если вы планируете использовать _только_ рабочие роли Linux в отметке, вы можете приобрести резервирование отметки Linux.

## <a name="buy-isolated-stamp-reserved-capacity"></a>Купить зарезервированную емкость изолированной метки

Вы можете приобрести зарезервированную емкость изолированной отметки в [портал Azure](https://portal.azure.com/#blade/Microsoft_Azure_Reservations/CreateBlade/referrer/documentation/filters/%7B%22reservedResourceType%22%3A%22AppService%22%7D). Платите за резервирование на [передний план или с ежемесячными платежами](billing-monthly-payments-reservations.md). Чтобы приобрести зарезервированную емкость, необходимо иметь роль владельца по крайней мере для одной корпоративной подписки или отдельную подписку с оплатой по мере использования.

- Для корпоративных подписок параметр **Добавить зарезервированные экземпляры** должен быть включен на [портале EA](https://ea.azure.com/). Если параметр отключен, необходимо быть администратором EA.
- Для программы поставщика облачных решений (CSP) можно приобрести зарезервированную емкость хранилища данных SQL только агентами администратора или агентами по продажам.

**Для приобретения:**

1. Перейдите к [портал Azure](https://portal.azure.com/#blade/Microsoft_Azure_Reservations/CreateBlade/referrer/documentation/filters/%7B%22reservedResourceType%22%3A%22AppService%22%7D).
1. Выберите подписку. Используйте список **Подписка** , чтобы выбрать подписку, которая будет использоваться для оплаты зарезервированной емкости. Для метода оплаты подписки взимается плата за зарезервированную емкость. Нужно выбрать подписку с соглашением Enterprise (номера предложений: MS-AZR-0017P или MS-AZR-0148P) или подписку с оплатой по мере использования (номера предложений: MS-AZR-0003P или MS-AZR-0023P) или подписка CSP.
    - Для подписки с соглашением Enterprise плата вычитается из баланса денежных обязательств или относится к избыточным расходам.
    - Для подписки с оплатой по мере использования плата взимается с помощью метода оплаты (кредитной карты или счета), указанного для подписки.
1. Выберите **область** , чтобы выбрать область подписки.
    - **Единственная область группы ресурсов** — применяет скидку резервирования к соответствующим ресурсам в выбранной группе ресурсов.
    - **Область действия одной подписки** — применяет скидку резервирования к соответствующим ресурсам в выбранной подписке.
    - **Общая область** — применяет скидку резервирования для сопоставления ресурсов в подходящих подписках, которые находятся в контексте выставления счетов. Для клиентов Соглашение Enterprise в качестве контекста выставления счетов используется регистрация. Для отдельных подписок с оплатой по мере использования в качестве области выставления счетов используются все подходящие подписки, созданные администратором учетной записи.
1. Выберите **регион** , чтобы выбрать регион Azure, на который распространяется зарезервированная емкость, и добавьте резервирование в корзину.
1. Выберите тип изолированного плана и нажмите кнопку **выбрать**.  
    ![Например](./media/billing-prepay-app-service-isolated-stamp/app-service-isolated-stamp-select.png)
1. Введите количество Служба приложений (цен. категория "Изолированный") штампов для резервирования. Например, количество, равное трем, даст вам три зарезервированных штампа региона. Щелкните **Далее: Проверьте и купите**.
1. Проверьте и нажмите кнопку **Купить сейчас**.

После приобретения перейдите в раздел [резервирования](https://portal.azure.com/#blade/Microsoft_Azure_Reservations/ReservationsBrowseBlade) , чтобы просмотреть состояние покупки и отслеживать его в любое время.

Вы можете отменять, обмениваться и возмещать деньги на резервирования с определенными ограничениями. Дополнительные сведения см. в статье [самостоятельная служба Exchange и возмещение для резервирования Azure](billing-azure-reservations-self-service-exchange-and-refund.md).

## <a name="discount-application-shown-in-usage-data"></a>Приложение для скидок, отображаемое в данных об использовании

Данные об использовании имеют эффективную цену, равную нулю, для использования, которая получает скидку резервирования. Данные об использовании показывают скидку резервирования для каждого экземпляра отметки в каждом резервировании.

Дополнительные сведения о том, как отображается скидка резервирования в данных об использовании, см. в статье [получение Соглашение Enterprise резервирование затрат и использование](billing-understand-reserved-instance-usage-ea.md) , если вы являетесь клиентом соглашение Enterprise (EA). В противном случае см. сведения [об использовании резервирования Azure для индивидуальной подписки с тарифами с оплатой по мере использования](billing-understand-reserved-instance-usage.md).

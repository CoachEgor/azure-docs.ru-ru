# <a name="how-the-azure-reservation-discount-is-applied-to-virtual-machines"></a>Как скидка на резервирование Azure применяется к виртуальным машинам

После того, как вы купили зарезервированный экземпляр виртуальной машины Azure, скидка резервирования автоматически применяется к виртуальным машинам, атрибуты и количество которых совпадают с резервированием. Резервирование покрывает затраты на вычислительные ресурсы виртуальных машин.

Скидка резервирования применяется к базовых виртуальных машин, которые вы приобрели в магазине Azure Marketplace.

Сведения о резервной мощности базы данных SQL см. в статье [Сведения о применении к базам данных SQL скидки на резервирование](billing-understand-reservation-charges.md).

В следующей таблице описаны затраты на виртуальную машину после приобретения зарезервированного экземпляра виртуальной машины. Во всех случаях с вас взимается плата за хранение и сеть по обычным тарифам.

| Тип виртуальной машины  | Расходы на зарезервированный экземпляр виртуальной машины |
|-----------------------|--------------------------------------------|
|Виртуальные машины Linux без дополнительного программного обеспечения | Резервирование покрывает затраты на инфраструктуру виртуальной машины.|
|Виртуальные машины Linux с платой за программное обеспечение (например, Red Hat) | Резервирование покрывает затраты на инфраструктуру. Взимается плата за дополнительное программное обеспечение.|
|Виртуальные машины Windows без дополнительного программного обеспечения |Резервирование покрывает затраты на инфраструктуру. Взимается плата за дополнительное программное обеспечение Windows.|
|Виртуальные машины Windows с дополнительным программным обеспечением (например, SQL Server) | Резервирование покрывает затраты на инфраструктуру. Взимается плата за программное обеспечение Windows, а также за дополнительное программное обеспечение.|
|Виртуальные машины Windows с [преимуществом гибридного использования Azure](../virtual-machines/windows/hybrid-use-benefit-licensing.md) | Резервирование покрывает затраты на инфраструктуру. Затраты на программное обеспечение Windows покрываются преимуществом гибридного использования Azure. Все дополнительное программное обеспечение оплачивается отдельно.|

## <a name="how-reservation-discount-is-applied"></a>Применение скидки резервирования

Скидка резервирования "*использования ИТ или потерять ИТ-*«. Таким образом Если у вас нет сопоставления ресурсов для данного часа, тогда у вас не количество резервирований для этого часа. Не удается выполнить пересылать неиспользуемые зарезервированные часов.

При завершении работы ресурса скидка на резервирование автоматически применяется к другим соответствующий ресурс в указанной области. Если нет соответствия ресурсов находятся в указанной области, а затем зарезервированный часы: *потеряны*.

## <a name="reservation-discount-for-non-windows-vms"></a>Скидка на резервирование для не - Windows виртуальных машин

 Скидка резервирования Azure применяется к работающим экземплярам виртуальных машин на почасовой основе. Для применения скидки приобретенное резервирование должно соответствовать потреблению ресурсов работающими виртуальными машинами. Для виртуальных машин, использующихся меньше часа, резервирование будет заполнено из других виртуальных машин, которые его не используют, включая одновременно работающие виртуальные машины. По окончании часа приложение резервирования для виртуальных машин блокируется. Если виртуальная машина работает неполный час или одновременно работающие виртуальные машины в течение часа не занимают весь час резервирования, резервирование становится недостаточным за этот час. Приведенная ниже диаграмма иллюстрирует резервирование экземпляров виртуальной машины с оплачиваемым использованием. На диаграмме представлено одно оплаченное резервирование и два соответствующих экземпляра виртуальных машин.

![Снимок экрана одного примененного резервирования и двух соответствующих экземпляров виртуальной машины](./media/billing-reserved-vm-instance-application/billing-reserved-vm-instance-application.png)

1. Потребление, проходит над линией резервирования, оплачиваются по обычным тарифам оплаты по мере использования. Если вы используете меньше ресурсов, чем предоставляет резервированние, дополнительная плата не взимается.
2. В течение первого часа экземпляр 1 работает 0,75 часа, а экземпляр 2 работает 0,5 часа. Общее использование в течение 1 часа — 1,25 часа. За оставшиеся 0,25 часа с вас взимается оплата по мере использования.
3. В течение второго и третьего часов оба экземпляра работали 1 час. Работа одного экземпляра покрывается резервированием, а за второй взимается оплата по мере использования.
4. В течение четвертого часа экземпляр 1 работает 0,5 часа, а экземпляр 2 работает 1 час. Использование экземпляра 1 полностью покрываться резервированием. 0,5 часа работы экземпляра 2 также покрываются. За оставшиеся 0,5 часа с вас взимается оплата по мере использования.

Чтобы больше узнать о применении резервирования Azure и просмотреть отчеты о начислении платы за использование, см. статью об [использовании резервирования](billing-understand-reserved-instance-usage-ea.md).

## <a name="reservation-discount-for-windows-vms"></a>Скидки резервирования для виртуальных машин Windows

Если используются экземпляры виртуальных машин Windows, резервирование покрывает затраты на инфраструктуру. Резервирование применяется к затратам на инфраструктуру виртуальных машин Windows так же, как и для виртуальных машин с другими ОС. За программное обеспечение Windows плата взимается отдельно для каждого виртуального ЦП. См. статью о [затратах на программное обеспечение Windows с резервированием](billing-reserved-Instance-windows-software-costs.md). Вы можете покрыть расходы на лицензии Windows, используя [Преимущество гибридного использования Azure для Windows Server](../virtual-machines/windows/hybrid-use-benefit-licensing.md).

## <a name="discount-can-apply-to-different-sizes"></a>Скидки можно применить для различных размеров

Если при покупке зарезервированного экземпляра виртуальной машины вы выбираете **Оптимизировано для**: **гибкость размера экземпляра**, покрытие скидки зависит от выбранного размера виртуальной машины. Резервирование можно применить к размерам в группе виртуальных машин серии того же размера. Дополнительные сведения см. в статье [Гибкость размеров виртуальных машин при использовании зарезервированных экземпляров виртуальных машин](../virtual-machines/windows/reserved-vm-instance-size-flexibility.md).

## <a name="discount-applies-to-matching-servicetype-only"></a>Скидка применяется к соответствующим ServiceType только

Скидка резервирования применяется только к использование виртуальных Машин где `ServiceType` значение `AdditionalInfo` соответствует резервирование, которое приобретается. Правила применения скидок на резервирование игнорирует единицы, используемые для виртуальных машин и предназначена только для оценки `ServiceType`. Знаете, какой службе типа, что вы приобрели для виртуальной Машины. Можно организовать обмен без поддержки расширенной хранилища виртуальной Машины резервирования для резервирования хранилища уровня "премиум", или в обратном порядке.

## <a name="classic-vms-and-cloud-services"></a>Классических виртуальных машин и облачных служб

Экземпляры виртуальных Машин, которые защищены автоматически применяются к обоих классических виртуальных машин и облачных служб при включении гибкость размер экземпляра. Для облачных служб скидка резервирования применяется только к затраты на вычисление. Когда скидка на резервирование применяется к облачным службам, плата за использование разбиваются на плата за вычислительные ресурсы (индикатор Linux) и облачными службами (индикатор управления облачных служб) плата. Дополнительные сведения см. в разделе [как скидка на резервирование применяется к облачным службам](billing-reserved-instance-windows-software-costs.md#cloud-services-software-meters-not-included-in-reservation-cost).

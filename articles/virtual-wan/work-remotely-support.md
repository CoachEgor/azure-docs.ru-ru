# <a name="azure-virtual-wan-and-supporting-remote-work"></a>Azure Virtual WAN и поддержка удаленной работы

>[!NOTE]
>В этой статье описывается, как можно использовать Azure Virtual WAN, Azure, сеть Майкрософт и партнерскую экосистему Azure для удаленной работы и устранения проблем сети, с которыми вы сталкиваетесь из-за кризиса COVID-19.
>

Вы скремблирования для обеспечения подключения для удаленных пользователей?
Вы вдруг видите необходимость поддерживать всплеск пользователей за то, что вы планировали?
Вам нужно, чтобы пользователь подключался из дома и не только к доступу к облаку, но и мог добраться до помещения?
Вам нужны удаленные пользователи, чтобы иметь возможность охватить ресурсы за частным WAN?
Есть ли у пользователей потребность в доступе к внутриоблачным ресурсам без необходимости настраивать подключение между регионами?
Поскольку эта глобальная пандемия создает беспрецедентные изменения вокруг нас, команда Azure Virtual WAN здесь для вас, чтобы удовлетворить ваши потребности в подключении.

Azure Virtual WAN — это сетевая служба, которая объединяет множество функций сетей, безопасности и реукторов, чтобы обеспечить единый рабочий интерфейс. Эти функции включают подключение филиала (через автоматизацию подключения с виртуальных устройств WAN Partner, таких как SD-WAN или VPN CPE), подключение VPN от сайта к сайту, удаленное подключение VPN пользователя (точка к сайту), частное (ExpressRoute) подключение, Внутриоблачное подключение (транзитное подключение для виртуальных сетей), VPN ExpressRoute Interconnectivity, маршрут, брандмауэр Azure, шифрование для личного подключения и т.д. Вам не нужно иметь все эти случаи использования, чтобы начать использовать Виртуальный WAN. Вы можете начать работу только с одного случая использования и настроить свою сеть по мере ее развития.

![Схема Виртуальной глобальной сети](./media/virtual-wan-about/virtualwan1.png)

Теперь, говоря об удаленных пользователях, давайте посмотрим, что вам нужно, чтобы ваша сеть и работает:

## <a name="set-up-remote-user-connectivity"></a><a name="connectivity"></a>Настройка удаленного подключения пользователей

Вы можете подключаться к своим ресурсам в Azure, используя подключение IPsec/IKE (IKEv2) или OpenVPN. Этот тип соединения требует настройки VPN-клиента для удаленного пользователя. Этим клиентом может быть [клиент Azure VPN](https://go.microsoft.com/fwlink/?linkid=2117554) или OpenVPN Client или любой клиент, поддерживающий IKEv2. Дополнительные сведения см. в разделе [Создание подключения типа "точка — сеть"](virtual-wan-point-to-site-portal.md).

## <a name="connectivity-from-the-remote-user-to-on-premises"></a><a name="remote user connectivity"></a>Подключение от удаленного пользователя к наместера

У вас есть два варианта здесь:

* Настройте подключение от сайта к сайту с любым существующим VPN-устройством. При подключении VPN-устройства IPsec к виртуальному wan-узле Azure, подключение между VPN пользователя Point-to-site (удаленный пользователь) и VPN от сайта к сайту происходит автоматически. Для получения дополнительной информации о том, как настроить VPN от сайта к сайту с вашего на-предпосылки VPN устройства для Azure Виртуальный WAN, [см.](virtual-wan-site-to-site-portal.md)

* Подключите схему ExpressRoute к виртуальному концентратору WAN. Подключение схемы ExpressRoute требует развертывания шлюза ExpressRoute в виртуальном WAN. Как только вы развернули один, взаимосвязь между точки к сайту пользователь VPN и ExpressRoute пользователя происходит автоматически. Для создания подключения ExpressRoute [см. Создание подключения ExpressRoute с помощью Virtual WAN.](virtual-wan-expressroute-portal.md) Для подключения к Azure Virtual WAN можно использовать существующую схему ExpressRoute.

## <a name="existing-basic-virtual-wan-customer"></a><a name="basic vWAN"></a>Существующий базовый виртуальный клиент WAN

Базовый виртуальный WAN предоставляет только VPN от сайта. Для того, чтобы удаленные пользователи могли подключиться, необходимо обновить виртуальный WAN до стандартного виртуального WAN. Для шагов по обновлению виртуального WAN [см.](upgrade-virtual-wan.md)

## <a name="additional-information"></a><a name="other considerations"></a>Дополнительная информация

Виртуальный WAN поддерживает один концентратор в регионе/местоположении. Сведения о расположении см. в статье [Virtual WAN partners and virtual hub locations](virtual-wan-locations-partners.md) (Партнеры Виртуальной глобальной сети и расположения виртуальных концентраторов). Каждый концентратор поддерживает до 10 000 удаленных пользовательских подключений, 1000 ветвей, четыре схемы ExpressRoute и до 500 виртуальных сетевых подключений. По мере расширения удаленных пользователей, если у вас есть какие-либо вопросы, не стесняйтесь обратиться за помощью, отправив электронное azurevirtualwan@microsoft.comписьмо. Если вам требуется техническая поддержка, обязательно откройте билет поддержки с портала Azure и помощь будет в пути.

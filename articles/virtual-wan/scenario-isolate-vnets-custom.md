# <a name="scenario-custom-isolation-for-vnets"></a>Сценарий: Пользовательская Изоляция для виртуальных сетей

При работе с маршрутизацией виртуальных сетей виртуальной глобальной сети существует несколько доступных сценариев. В пользовательском сценарии изоляции для виртуальных сетей цель заключается в предотвращении доступа определенного набора виртуальных сетей к другому конкретному набору виртуальных сетей. Однако виртуальных сетей должны обращаться ко всем ветвям (VPN/ER/VPN-подключение пользователей).

В этом сценарии VPN, ExpressRoute и VPN-подключения пользователей (совместно называемые ветви) связаны с одной и той же таблицей маршрутов (таблицей маршрутов по умолчанию). Все VPN-подключения ExpressRoute и User VPN распространяют маршруты на один и тот же набор таблиц маршрутов. Дополнительные сведения о маршрутизации виртуальных концентраторов см. в статье [о маршрутизации виртуальных концентраторов](about-virtual-hub-routing.md).

## <a name="scenario-workflow"></a><a name="architecture"></a>Рабочий процесс сценария

На **рис. 1**есть синие и красные подключения к виртуальной сети.

* Синяя подключенная виртуальных сетей может соединяться друг с другом, а также достигать всех ветвей (VPN/ER/P2S).
* Красный виртуальных сетей может подключиться друг к другу, а также к подключениям всех ветвей (VPN/ER/P2S).

При настройке маршрутизации примите во внимание следующие шаги.

1. Создайте две пользовательские таблицы маршрутов в портал Azure, **RT_BLUE** и **RT_RED**.
2. Для таблицы маршрутов **RT_BLUE**в разделе:
   * **Ассоциация**: выберите все синие виртуальных сетей
   * **Распространение**: для ветвей выберите вариант для ветвей, что означает, что подключения Branch (VPN/ER/P2S) будут распространять маршруты в эту таблицу маршрутов.
3. Повторите те же действия для таблицы маршрутов **RT_RED** для Red виртуальных сетей и ветвей (VPN/ER/P2S).

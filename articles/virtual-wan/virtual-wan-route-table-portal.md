# <a name="create-a-virtual-wan-hub-route-table-for-nvas-azure-portal"></a>Создайте таблицу маршрутов виртуальной глобальной сети центра для NVA: Портал Azure

В этой статье показано, как для направления трафика с помощью центра для сетевой виртуальный модуль (NVA).

![Схема Виртуальной глобальной сети](./media/virtual-wan-route-table/vwanroute.png)

## <a name="before-you-begin"></a>Перед началом работы

Убедитесь, что вы выполнили следующие критерии:

*  У вас есть сетевой виртуальный модуль (NVA). Виртуальный сетевой модуль представляет собой независимых производителей по своему усмотрению, которая обычно подготавливается из Azure Marketplace в виртуальной сети.

    * Частный IP-адрес должны назначаться сетевому интерфейсу виртуального сетевого устройства.

    * Виртуальное сетевое устройство не развертывается в виртуального концентратора. Он должен быть развернут в отдельной виртуальной сети (VNet).

    *  В виртуальной сети виртуальное сетевое устройство может иметь одну или несколько виртуальных сетей, подключенных к нему. В этой статье мы называем виртуальной сети виртуальное сетевое устройство «косвенных периферийной виртуальной сети». Эти виртуальные сети можно подключить к виртуальной сети виртуальное сетевое устройство с помощью пиринга.
*  Вы создали двух виртуальных сетей. Они будут использоваться в качестве виртуальной сети периферийных зон.

    * В этом упражнении, адресные пространства виртуальной сети периферийных зон: VNet1: 10.0.2.0/24 и VNet2. 10.0.3.0/24. Если вам нужна информация о том, как создать виртуальную сеть, см. в разделе [Создание виртуальной сети](../virtual-network/quick-create-portal.md).

    * Убедитесь, что нет шлюзов виртуальной сети в любой из виртуальных сетей.
    * Для этой конфигурации эти виртуальные сети не требуется подсеть шлюза.

## <a name="signin"></a>1. Вход

В браузере откройте [портал Azure](https://portal.azure.com) и выполните вход с помощью учетной записи Azure.

* **Имя виртуальной глобальной сети:** myVirtualWAN
* **Группа ресурсов:** testRG
* **Расположение**. Запад США

## <a name="route"></a>4. Создание и применение таблицы маршрутов концентратора

Обновите концентратор с таблицей маршрутов концентратора. В рамках этого упражнения можно использовать следующие значения:

* **Адресные пространства виртуальной сети косвенных периферийной зоны:** (VNet1 и VNet2) 10.0.2.0/24 и 10.0.3.0/24
* **Сеть Периметра NVA сетевой интерфейс частный IP-адрес:** 10.0.4.5

1. Перейдите к виртуальной глобальной сети.
2. Щелкните концентратор, для которого необходимо создать таблицу маршрутов.
3. Нажмите кнопку **...** , а затем нажмите кнопку **виртуального концентратора редактирования**.
4. На **виртуального концентратора редактирования** странице, прокрутите вниз и установите флажок **использовать таблицу для маршрутизации**.
5. В **Если префикс назначения** столбца, добавить адресные пространства. В **отправки функции определения следующего прыжка** столбца, добавить виртуальное сетевое устройство в сети Периметра сети интерфейс частный IP-адрес.
6. Нажмите кнопку **Подтверждение** обновить ресурс концентратора с параметрами таблицы маршрутов.

## <a name="connections"></a>5. Создание подключений виртуальной сети

Создайте подключение из каждой периферийной зоне косвенных виртуальной сети (VNet1 и VNet2) к концентратору. Затем создайте подключение из виртуальной сети виртуальное сетевое устройство к центру.

 Для выполнения этого шага можно использовать следующие значения:

| Имя виртуальной сети| Имя подключения|
| --- | --- |
| VNet1 | testconnection1 |
| VNet2 | testconnection2 |
| NVAVNet | testconnection3 |

Повторите следующую процедуру для каждой виртуальной сети, который вы хотите подключиться.

1. На странице своей глобальной сети щелкните **Подключения к виртуальной сети**.
2. На странице подключения к виртуальной сети щелкните **+Добавить подключение**.
3. На странице **Добавление подключения** заполните следующие поля.

    * **Имя подключения**. Укажите имя своего подключения.
    * **Концентраторы**. Выберите концентратор, который нужно связать с этим подключением.
    * **Подписка**. Проверьте подписку.
    * **Виртуальная сеть**. Выберите виртуальную сеть, которую вы хотите подключить к этому концентратору. В виртуальной сети не должно быть шлюза виртуальной сети.
4. Нажмите кнопку **ОК**, чтобы создать подключение.

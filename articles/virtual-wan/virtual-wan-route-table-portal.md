# <a name="create-a-virtual-wan-hub-route-table-for-nvas-azure-portal"></a>Создание виртуальной таблицы маршрутов WAN для NVA: Портал Azure

В этой статье показано, как управлять трафиком из филиала (на месте, подключенного к виртуальному концентратору WAN, к виртуальной сети Spoke (VNet) через сетевой виртуальный прибор (NVA).

![Схема Виртуальной глобальной сети](./media/virtual-wan-route-table/vwanroute.png)

## <a name="before-you-begin"></a>Перед началом

Убедитесь, что вы выполнили следующие критерии:

*  У вас есть сетевой виртуальный прибор (NVA). Сетевой виртуальный прибор — это стороннее программное обеспечение по вашему выбору, которое обычно готовится с Azure Marketplace в виртуальной сети.

    * Частный IP-адрес должен быть приписан к сетевому интерфейсу NVA.

    * NVA не развернут в виртуальном центре. Он должен быть развернут в отдельной виртуальной сети.

    *  Виртуальная сеть NVA может иметь одну или несколько виртуальных сетей, подключенных к ней. В этой статье мы называем виртуальную сеть NVA «косвенной vNet». Эти виртуальные сети могут быть подключены к NVA VNet с помощью вглядывания VNet. Ссылки VNet Peering изображены черными стрелками на приведенной выше фигуре между VNet 1, VNet 2 и NVA VNet.
*  Вы создали две виртуальные сети. Они будут использоваться в качестве говорил VNets.

    * VNet говорил адрес пространства: VNet1: 10.0.0/24 и VNet2: 10.0.3.0/24. Если вам нужна информация о том, как создать виртуальную сеть, [см.](../virtual-network/quick-create-portal.md)

    * Убедитесь, что в vNets нет виртуальных сетевых шлюзов.

    * VNets не требуют подсети шлюза.

## <a name="1-sign-in"></a><a name="signin"></a>1. Вопивай в

Из браузера перейдите на [портал Azure](https://portal.azure.com) и войдите в систему с помощью учетной записи Azure.

## <a name="2-create-a-virtual-wan"></a><a name="vwan"></a>2. Создание виртуального WAN

Создание виртуальной глобальной сети. Используйте следующие значения примера:

* **Виртуальное имя WAN:** myVirtualWAN
* **Ресурсная группа:** testRG
* **Местонахождение:** Западная часть США

## <a name="3-create-a-hub"></a><a name="hub"></a>3. Создание концентрата

Создайте концентратор. Используйте следующие значения примера:

* **Местонахождение:** Западная часть США
* **Имя:** westushub
* **Пространство для частного адреса концентратора:** 10.0.0.0/24

## <a name="4-create-and-apply-a-hub-route-table"></a><a name="route"></a>4. Создание и применение таблицы маршрутов концентратора

Обновление концентратора с таблицей маршрутов концентратора. Используйте следующие значения примера:

* **Адресные пространства Погосцы VNet:** (VNet1 и VNet2) 10.0.0/24 и 10.0.3.0/24
* **Интерфейс сетевого интерфейса ДМЗ NVA:** 10.0.4.5

1. Перейдите к виртуальному WAN.
2. Нажмите на концентратор, для которого требуется создать таблицу маршрутов.
3. Нажмите **на ...,** а затем нажмите **Edit виртуальный концентратор**.
4. На странице **виртуального концентратора Edit** прокрутите вниз и выберите таблицу использования флажка **для разгрома.**
5. В **префиксе "Если" является** столбец, добавьте адресные пробелы. В столбце **«Отправить в следующий прыжок»** добавьте частный IP-адрес сетевого интерфейса DM' NVA.
6. Нажмите **Confirm,** чтобы обновить ресурс концентратора с настройками таблицы маршрутов.

## <a name="5-create-the-vnet-connections"></a><a name="connections"></a>5. Создание соединений VNet

Создание виртуального сетевого соединения от каждого косвенного говорил VNet (VNet1 и VNet2) к концентратору. Эти виртуальные сетевые соединения изображены синими стрелками на вышеуказанной фигуре. Затем создайте соединение VNet из NVA VNet в концентратор (черная стрелка на рисунке).

 Для этого шага можно использовать следующие значения:

| Имя виртуальной сети| Имя соединения|
| --- | --- |
| VNet1 | testconnection1 |
| VNet2 | testconnection2 |
| NVAVNet | testconnection3 |

Повторите следующую процедуру для каждой виртуальной сети, которую вы хотите подключить.

1. На странице своей глобальной сети щелкните **Подключения к виртуальной сети**.
2. На странице подключения к виртуальной сети щелкните **+Добавить подключение**.
3. На странице **Добавление подключения** заполните следующие поля.

    * **Имя подключения**. Укажите имя своего подключения.
    * **Концентраторы**. Выберите концентратор, который нужно связать с этим подключением.
    * **Подписка**. Проверьте подписку.
    * **Виртуальная сеть**. Выберите виртуальную сеть, которую вы хотите подключить к этому концентратору. В виртуальной сети не должно быть шлюза виртуальной сети.
4. Нажмите кнопку **ОК**, чтобы создать подключение.

# <a name="view-effective-routes-of-a-virtual-hub"></a>Просмотр эффективных маршрутов виртуального концентратора

Вы можете просмотреть все маршруты вашего виртуального концентратора WAN на портале Azure. Для просмотра маршрутов перейдите к виртуальному узлу, а затем выберите **Маршрут - > Просмотр эффективных маршрутов.**

## <a name="understanding-routes"></a><a name="understand"></a>Понимание маршрутов

Следующий пример может помочь вам лучше понять, как отображается виртуальная routing WAN.

В этом примере у нас есть виртуальный WAN с тремя концентраторами. Первый хаб находится в восточном регионе США, второй хаб находится в регионе Западной Европы, а третий хаб находится в регионе Западной ЧАСТИ США. В виртуальном WAN все концентраторы взаимосвязаны. В этом примере мы предположим, что центры Восточной ИС и Западной Европы имеют связи с автономными филиалами (спицы) и виртуальными сетями Azure (спицы).

Azure VNet смеется (10.4.0.0.0/16) с виртуальным прибором сети (10.4.0.6) далее заглядывается в VNet (10.5.0.0/16). Дополнительная [информация](#abouthubroute) в этой статье смотрите позже, чтобы узнать больше о таблице маршрутов концентратора.

В этом примере мы также предполагаем, что Западно-Европейский филиал 1 связан с центром Восточной США, а также с центром Западной Европы. Схема ExpressRoute в восточной части США соединяет филиал 2 с восточным концентратором США.

## <a name="view-effective-routes"></a><a name="view"></a>Просмотр эффективных маршрутов

При выборе «Просмотр эффективных маршрутов» на портале он производит выход, отображаемый в [таблице маршрутов концентратора](#routetable) для Восточного концентратора США.

Для того чтобы положить это в перспективу, первая линия подразумевает что концентратор Восточного США выучил трассу 10.20.1.0/24 (Branch 1) из-за соединения *типа хмеля* VPN следующего перехода ('Next hop' VPN Gateway Instance0 IP 10.1.0.6, Instance1 IP 10.1.0.7). *Происхождение маршрута* указывает на идентификатор ресурса. *AS Path* указывает as Путь для ветви 1.

## <a name="additional-information"></a><a name="additional"></a>Дополнительная информация

### <a name="about-the-hub-route-table"></a><a name="abouthubroute"></a>О таблице маршрутов концентратора

Вы можете создать маршрут виртуального концентратора и добавить его в таблицу маршрутов виртуального концентратора. В эту таблицу можно добавить несколько маршрутов. Это позволяет установить маршрут назначения VNet через IP-адрес (обычно сетевой виртуальный прибор (NVA) в спицкой VNet). Для получения дополнительной информации о NVA, см [трафик маршрута от виртуального концентратора до NVA](virtual-wan-route-table-portal.md).

### <a name="about-default-route-00000"></a><a name="aboutdefaultroute"></a>О маршруте по умолчанию (0.0.0.0/0)

Виртуальный концентратор имеет возможность распространять изученный маршрут по умолчанию в виртуальную сеть, VPN от сайта к сайту и соединение ExpressRoute, если флаг «включен» на соединение. Этот флаг виден при правлении виртуального сетевого соединения, VPN-соединения или подключения ExpressRoute. 'EnableInternetSecurity' всегда ложно по умолчанию на соединениях Hub VNet, ExpressRoute и VPN.

Маршрут по умолчанию не происходит в виртуальном концентраторе WAN. Маршрут по умолчанию распространяется, если он уже изучен виртуальным концентратором WAN в результате развертывания брандмауэра в концентраторе, или если другой подключенный участок заставил включенный туннель.

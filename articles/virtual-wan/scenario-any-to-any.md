# <a name="scenario-any-to-any"></a>Сценарий: "любой к любому"

При работе с маршрутизацией виртуальных сетей виртуальной глобальной сети существует несколько доступных сценариев. В любом сценарии любой периферийный сервер может достигать другой. При наличии нескольких концентраторов маршрутизация между концентраторами (также называемая межцентральным концентратором) включается по умолчанию в стандартную виртуальную сеть WAN. 

В этом сценарии VPN, ExpressRoute и VPN-подключения пользователей связаны с одной и той же таблицей маршрутов. Все VPN-подключения ExpressRoute и User VPN распространяют маршруты на один и тот же набор таблиц маршрутов. Дополнительные сведения о маршрутизации виртуальных концентраторов см. в статье [о маршрутизации виртуальных концентраторов](about-virtual-hub-routing.md).

## <a name="scenario-architecture"></a><a name="architecture"></a>Архитектура сценария

На **рис. 1**все виртуальных сетей и ветви (VPN, EXPRESSROUTE, P2S) могут быть доступны друг другу. В виртуальном концентраторе подключения работают следующим образом:

* Подключение VPN подключает VPN-сайт к VPN-шлюзу.
* Подключение виртуальной сети подключает виртуальную сеть к виртуальному концентратору. Маршрутизатор виртуального концентратора обеспечивает транзитную функциональность между виртуальных сетей.
* Подключение ExpressRoute подключает канал ExpressRoute к шлюзу ExpressRoute.

Эти соединения (по умолчанию при создании) связаны с таблицей маршрутов по умолчанию, если конфигурация маршрутизации не настроена для подключения ни к **одному**, ни к пользовательской таблице маршрутов. Эти соединения также распространяют маршруты по умолчанию в таблицу маршрутов по умолчанию. Это позволяет выполнять любые сценарии, где любой периферийный сервер (виртуальная сеть, VPN, ER, P2S) может быть доступен друг за другом.

## <a name="scenario-workflow"></a><a name="workflow"></a>Рабочий процесс сценария

Этот сценарий включен по умолчанию для стандартной виртуальной глобальной сети. Если параметр ветвления-to-Branch отключен в конфигурации глобальной сети, то он запрещает подключение между периферийными сетями. VPN-подключения ExpressRoute и пользователей в виртуальной глобальной сети считаются периферийными.

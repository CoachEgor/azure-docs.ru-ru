# <a name="azure-security-baseline-for-virtual-wan"></a>Базовый уровень безопасности Azure для виртуальной глобальной сети

Этот базовый план безопасности применяет рекомендации из [контрольного показателя безопасности Azure версии 2,0](../security/benchmarks/overview.md) к Microsoft Azure виртуальной глобальной сети. Azure Security Benchmark содержит рекомендации по обеспечению безопасности облачных решений в Azure. Содержимое группируются по **элементам управления безопасностью** , определенным в ходе тестирования системы безопасности Azure, и связанным рекомендациям, применимым к виртуальной глобальной сети. **Элементы управления** , неприменимые к виртуальной глобальной сети, были исключены.

Сведения о том, как виртуальная Глобальная сеть полностью сопоставляется с тестовым уровнем безопасности Azure, см. в [полном файле сопоставления базовых показателей безопасности виртуальной глобальной сети](https://github.com/MicrosoftDocs/SecurityBenchmarks/tree/master/Azure%20Offer%20Security%20Baselines).

### <a name="ns-1-implement-security-for-internal-traffic"></a>NS-1: реализация безопасности для внутреннего трафика

**Руководство**. Microsoft Azure Виртуальная глобальная сеть предоставляет возможности настраиваемой маршрутизации и обеспечивает шифрование для трафика ExpressRoute. Все управление маршрутами осуществляется маршрутизатором виртуального концентратора, который также обеспечивает транзитное подключение между виртуальными сетями. Шифрование трафика ExpressRoute с помощью виртуальной глобальной сети обеспечивает зашифрованное переработку между локальными сетями и виртуальными сетями Azure через ExpressRoute, не переключаясь в общедоступный Интернет и не используя общедоступные IP-адреса. 

- [Шифрование трафика ExpressRoute](virtual-wan-about.md#encryption)

- [Использование частной ссылки в виртуальной глобальной сети](howto-private-link.md)

- [Сценарии настраиваемой маршрутизации](scenario-any-to-any.md)

- [Документация по пользовательской таблице маршрутов](how-to-virtual-hub-routing.md)

### <a name="ns-2-connect-private-networks-together"></a>NS-2. Совместное подключение частных сетей 

**Руководство**. Microsoft Azure ExpressRoute предлагает частное подключение к виртуальной глобальной сети Azure. Так как подключения ExpressRoute не проходят через общедоступный Интернет, ExpressRoute обеспечивает повышенную надежность, более быстрые и низкие задержки, чем обычные подключения к Интернету. Виртуальную частную сеть можно также использовать для подключения к Azure через VPN-подключение типа "сеть — сеть" (S2S) или "точка — сеть" (P2S) VPN.

### <a name="ns-4-protect-applications-and-services-from-external-network-attacks"></a>NS-4: Защита приложений и служб от внешних сетевых атак

**Руководство**. Виртуальная глобальная сеть не предоставляет конечные точки внешним сетям, которые нуждаются в защите с помощью обычной защиты сети. Вы можете свободно защищать ресурсы в периферийных виртуальных сетях (любой виртуальной сети, подключенной к виртуальному концентратору) с помощью служб защиты виртуальной сети. 

Используйте брандмауэр Azure для защиты приложений и служб от потенциально вредоносного трафика из Интернета и других внешних расположений. 

Выберите Azure от атак DDoS Protection для защиты ресурсов от атак в виртуальных сетях Azure. Используйте центр безопасности Azure для обнаружения рисков, связанных с сетевыми ресурсами.

- [Документация по брандмауэру Azure](/azure/firewall)

- [Рекомендации Центра безопасности Azure](../security-center/recommendations-reference.md#recs-network)

Вы можете создать политику брандмауэра Azure и связать ее с виртуальным концентратором глобальной сети, чтобы разрешить функционирование существующего виртуального концентратора глобальной сети в качестве защищенного виртуального концентратора с развернутыми необходимыми ресурсами брандмауэра Azure.

### <a name="im-1-standardize-azure-active-directory-as-the-central-identity-and-authentication-system"></a>IM-1: стандартизация Azure Active Directory в качестве централизованной системы идентификации и проверки подлинности

**Руководство**. Azure Active Directory (Azure AD) — это служба управления удостоверениями и доступом по умолчанию для служб Azure. включая виртуальную глобальную сеть. Стандартизация Azure AD для управления удостоверениями и доступом в вашей организации в:

- Ресурсы Организации, такие как приложения в Azure или ресурсы корпоративной сети;

Защитите Azure AD как высокий приоритет в целях безопасности в облаке вашей организации. Оцените свои удостоверения и безопасность с помощью функции оценки безопасности в центре безопасности, чтобы оценить, насколько близко ваша конфигурация соответствует рекомендациям Майкрософт. При необходимости реализуйте рекомендации Майкрософт для улучшения безопасности.

Azure AD также поддерживает внешние удостоверения, которые позволяют пользователям без учетная запись Майкрософт входить в свои приложения и ресурсы с внешними удостоверениями.

### <a name="im-4-use-strong-authentication-controls-for-all-azure-active-directory-based-access"></a>IM-4. использование строгих средств проверки подлинности для всех Azure Active Directory доступом

**Руководство**. в настоящее время проверка подлинности Azure Active Directory (Azure AD) обеспечивается за счет интеграции с виртуальной глобальной сетью VPN типа "точка — сеть".

Azure Active Directory (Azure AD) — это служба управления удостоверениями и доступом по умолчанию для служб Azure. Azure AD поддерживает надежные элементы управления аутентификацией с многофакторной проверкой подлинности и надежными методами без пароля.

Azure AD рекомендует использовать следующие средства для управления надежными проверками подлинности:

- Многофакторная проверка подлинности. Включите многофакторную проверку подлинности Azure AD и следуйте рекомендациям по управлению удостоверениями и доступом в центре безопасности Azure для получения рекомендаций по обеспечению безопасности. Применяйте многофакторную проверку подлинности для всех пользователей или на уровне отдельных пользователей на основе условий входа и факторов риска.

- Проверка подлинности с паролем — доступны три варианта проверки подлинности с паролем. К ним относятся, Windows Hello для бизнеса, Microsoft Authenticator приложение и локальные методы проверки подлинности, такие как смарт-карты.

Убедитесь, что для администраторов и привилегированных пользователей используется высший уровень надежной проверки подлинности, а затем развертывание политики строгой проверки подлинности для других пользователей.

Обратите внимание, что проверка подлинности Azure AD доступна только для шлюзов, использующих протокол Опенвпн и клиенты под управлением Windows.

### <a name="am-1-ensure-security-team-has-visibility-into-risks-for-assets"></a>AM-1: обеспечение контроля над рисками для ресурсов в группе безопасности

**Руководство**. Убедитесь, что группам безопасности предоставлены разрешения на чтение безопасности в клиенте и подписках Azure, чтобы они могли отслеживать риски безопасности с помощью центра безопасности Azure.

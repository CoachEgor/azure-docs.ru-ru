# <a name="connect-a-vpn-gateway-virtual-network-gateway-to-virtual-wan"></a>Подключение VPN шлюза (виртуальный сетевой шлюз) к Виртуальному WAN

Эта статья поможет настроить подключение от Azure VPN Gateway (виртуальный сетевой шлюз) к виртуальному WAN (VPN шлюз). Создание соединения от VPN Gateway (виртуальный сетевой шлюз) к виртуальному WAN (VPN шлюз) похоже на настройку подключения к виртуальному WAN с ветвей VPN-сайтов.

Для того, чтобы свести к минимуму возможную путаницу между двумя функциями, мы предисловим к шлюзу с названием функции, о которой мы говорим. Например, виртуальный сетевой шлюз VPN Gateway и виртуальный wan VPN-шлюз.

## <a name="before-you-begin"></a>Перед началом

Прежде чем начать, создайте следующие ресурсы:

Виртуальная глобальная сеть Azure (WAN)

* [Создайте виртуальный WAN](virtual-wan-site-to-site-portal.md#openvwan).
* [Создайте концентратор](virtual-wan-site-to-site-portal.md#hub). Виртуальный концентратор содержит виртуальный wan VPN шлюз.

Виртуальная сеть Azure

* Создайте виртуальную сеть без виртуальных сетевых шлюзов. Ни одна из подсетей локальной сети не должна перекрывать виртуальные сети, к которым вы хотите подключиться. Сведения о создании виртуальной сети на портале Azure см. в [этой статье](../virtual-network/quick-create-portal.md).

## <a name="1-create-an-azure-virtual-network-gateway"></a><a name="vnetgw"></a>1. Создание виртуального сетевого шлюза Azure

Создайте виртуальный сетевой шлюз VPN Gateway для виртуальной сети в активном режиме для виртуальной сети. При создании шлюза можно использовать существующие общедоступные IP-адреса для двух экземпляров шлюза, либо создавать новые общедоступные IP-адреса. Вы используете эти общедоступные ИС при настройке виртуальных wan сайтов. Для получения дополнительной информации об активно-активном режиме [см.](../vpn-gateway/vpn-gateway-activeactive-rm-powershell.md#aagateway)

### <a name="active-active-mode-setting"></a><a name="active-active"></a>Установка активно-активного режима

![активно-активный](./media/connect-virtual-network-gateway-vwan/active.png "Активное-активное")

### <a name="bgp-setting"></a><a name="BGP"></a>Настройка BGP

BGP ASN не может быть 65515. 66515 будет использоваться Azure Virtual WAN.

![BGP](./media/connect-virtual-network-gateway-vwan/bgp.png "Bgp")

### <a name="public-ip-addresses"></a><a name="pip"></a>Публичные IP-адреса

При создании шлюза перейдите на страницу **Свойств.** Свойства и настройки конфигурации будут аналогичны следующему примеру. Обратите внимание на два общедоступных IP-адреса, которые используются для шлюза.

![Вариантов размещения](./media/connect-virtual-network-gateway-vwan/publicip.png "properties")

## <a name="2-create-virtual-wan-vpn-sites"></a><a name="vwansite"></a>2. Создание виртуальных сайтов WAN VPN

Чтобы создать виртуальные WAN VPN сайты, перейдите на ваш виртуальный WAN и, под **connectivity**, выберите **САЙТы VPN.** В этом разделе вы создадите два виртуальных WAN VPN-сайта, которые соответствуют созданным вами в предыдущем разделе шлюзам виртуальной сети.

1. Выберите **сайт «Создать».**
2. На странице **веб-сайтов Create VPN** введите следующие значения:

   * **Регион** - (Тот же регион, что и виртуальный сетевой шлюз Azure VPN Gateway)
   * **Поставщик устройств** - Введите поставщика устройства (любое имя)
   * **Частное адресное пространство** - (Введите значение, или оставьте пустым, когда BGP включен)
   * **Протокол пограничного шлюза** - (Установить **для включения,** если Azure VPN Gateway виртуальный шлюз сети имеет BGP включен)
   * **Подключение к концентратам** (Выберите концентратор, созданный в предпосылках от выпадения)
3. В **ссылках**введите следующие значения:

   * **Имя поставщика** - Введите имя ссылки и имя поставщика (любое имя)
   * **Скорость** - Скорость (любое число)
   * **IP-адрес** - Введите IP-адрес (так же, как первый публичный IP-адрес, показанный под (VPN Gateway) виртуальные свойства сети шлюза)
   * **BGP Адрес** и **ASN** - адрес BGP и ASN. Они должны быть такими же, как один из одноранговых IP-адресов BGP, и ASN из виртуального сетевого шлюза VPN Gateway, который вы настроили в [шаге 1.](#vnetgw)
4. Просмотрите и выберите **Подтвердите** для создания сайта.
5. Повторите предыдущие шаги по созданию второго сайта в соответствии со вторым экземпляром виртуального сетевого шлюза VPN Gateway. Вы сохраните те же настройки, за исключением использования второго публичного IP-адреса и второго однорангового IP-адреса BGP из конфигурации VPN Gateway.
6. Теперь у вас есть два сайта успешно подготовлены и может перейти к следующему разделу для загрузки файлов конфигурации.

## <a name="3-download-the-vpn-configuration-files"></a><a name="downloadconfig"></a>3. Скачать файлы конфигурации VPN

В этом разделе вы загружаете файл конфигурации VPN для каждого из сайтов, созданных в предыдущем разделе.

1. В верхней части страницы виртуальных **WAN VPN сайтов,** выберите **сайт**, а затем выберите **Скачать сайт на сайт VPN конфигурации.** Azure создает файл конфигурации с настройками.

   ![скачать файл конфигурации](./media/connect-virtual-network-gateway-vwan/download.png "скачиваемого файла")
2. Скачать и открыть файл конфигурации.
3. Повторите эти шаги для второго сайта. После открытия обоих файлов конфигурации можно перейти к следующему разделу.

## <a name="4-create-the-local-network-gateways"></a><a name="createlocalgateways"></a>4. Создание локальных сетевых шлюзов

В этом разделе вы создаете два локальных сетевых шлюза Azure VPN Gateway. Файлы конфигурации предыдущего шага содержат настройки конфигурации шлюза. Используйте эти настройки для создания и настройки локальных сетевых шлюзов Azure VPN Gateway.

1. Создайте локальный сетевой шлюз, используя эти настройки. Для получения информации о том, как создать VPN [Create a local network gateway](../vpn-gateway/vpn-gateway-howto-site-to-site-resource-manager-portal.md#LocalNetworkGateway)Gateway локальной сети шлюз, см.

   * **IP-адрес** - Используйте IP-адрес Instance0, показанный для *настройки шлюза* из файла конфигурации.
   * **BGP** - Если соединение находится над BGP, выберите **настройки BGP** и введите ASN '65515'. Введите одноранговой IP-адрес BGP. Используйте 'Instance0 BgpPeeringAddresses' для *настройки шлюза* из файла конфигурации.
   * **Подписка, ресурсная группа и расположение** такие же, как и для виртуального концентратора WAN.
2. Просмотрите и создайте локальный сетевой шлюз. Локальный сетевой шлюз должен выглядеть так же, как и в этом примере.

   ![скачать файл конфигурации](./media/connect-virtual-network-gateway-vwan/lng1.png "экземпляр0")
3. Повторите эти шаги, чтобы создать другой локальный сетевой шлюз, но на этот раз используйте значения 'Instance1' вместо значений 'Instance0' из файла конфигурации.

   ![скачать файл конфигурации](./media/connect-virtual-network-gateway-vwan/lng2.png "instance1")

## <a name="5-create-connections"></a><a name="createlocalgateways"></a>5. Создание соединений

В этом разделе вы создаете соединение между локальными сетевыми шлюзами VPN Gateway и виртуальными сетевыми шлюзами. Для нас есть шаги по созданию VPN-шлюза, [см.](../vpn-gateway/vpn-gateway-howto-site-to-site-resource-manager-portal.md#CreateConnection)

1. На портале перейдите к виртуальному сетевому шлюзу и нажмите **На connections.** В верхней части страницы "Подключения" нажмите кнопку **Добавить**, чтобы открыть страницу **Добавление подключения**.
2. На странице **подключения Добавить** настройте следующие значения для соединения:

   * **Имя** — имя подключения.
   * **Тип соединения:** Выберите **сайт на сайт (IPSec)**
   * **Шлюз виртуальной сети** — значение является фиксированным, так как вы подключаетесь из этого шлюза.
   * **Локальный сетевой шлюз:** Это соединение соединит виртуальный сетевой шлюз с локальным сетевым шлюзом. Выберите один из локальных сетевых шлюзов, созданных ранее.
   * **Общий ключ:** Введите общий ключ.
   * **Протокол IKE:** Выберите протокол IKE.
   * **BGP:** Выберите **включить BGP,** если соединение находится за BGP.
3. Щелкните **ОК** для создания подключения.
4. Вы можете просмотреть соединение на странице **соединения** виртуального сетевого шлюза.

   ![Подключения](./media/connect-virtual-network-gateway-vwan/connect.png "соединение")
5. Повторите предыдущие шаги, чтобы создать второе соединение. Для второго соединения выберите другой созданный вами локальный сетевой шлюз.

## <a name="6-test-connections"></a><a name="test"></a>6. Тестовые соединения

Вы можете проверить подключение, создав две виртуальные машины, один на стороне VPN Gateway виртуальный сетевой шлюз, и один в виртуальной сети для виртуального WAN, а затем пинг двух виртуальных машин.

1. Создайте виртуальную машину в виртуальной сети (Test1-VNet) для Azure VPN Gateway (Test1-VNG). Не создавайте виртуальную машину в GatewaySubnet.
2. Создайте другую виртуальную сеть для подключения к виртуальному WAN. Создайте виртуальную машину в подсети этой виртуальной сети. Эта виртуальная сеть не может содержать виртуальные сетевые шлюзы. Вы можете быстро создать виртуальную сеть, используя шаги PowerShell в статье [о подключении к сайту.](virtual-wan-site-to-site-portal.md#vnet) Не забудьте изменить значения перед запуском cmdlets.
3. Подключите VNet к виртуальному концентратору WAN. На странице для вашего виртуального WAN, выберите **Виртуальные сетевые соединения**, а затем **"Добавить соединение**. На странице **Добавление подключения** заполните следующие поля.

    * **Имя подключения**. Укажите имя своего подключения.
    * **Концентраторы**. Выберите концентратор, который нужно связать с этим подключением.
    * **Подписка**. Проверьте подписку.
    * **Виртуальная сеть**. Выберите виртуальную сеть, которую вы хотите подключить к этому концентратору. В виртуальной сети не должно быть шлюза виртуальной сети.
4. Нажмите кнопку **ОК**, чтобы создать подключение виртуальной сети.
5. Связь теперь устанавливается между VMs. Вы должны иметь возможность пинговать один VM от другого, если Есть какие-либо брандмауэры или другие политики, блокирующие связь.

# <a name="scenario-azure-firewall---interhub-preview"></a>Сценарий: брандмауэр Azure — Интерхуб (Предварительная версия)

При работе с маршрутизацией виртуальных сетей виртуальной глобальной сети существует несколько доступных сценариев. В этом сценарии целью является маршрутизация между несколькими концентраторами, которые содержат брандмауэр Azure. Дополнительные сведения о маршрутизации виртуальных концентраторов см. в статье [о маршрутизации виртуальных концентраторов](about-virtual-hub-routing.md).

## <a name="scenario-architecture"></a><a name="architecture"></a>Архитектура сценария

В этом сценарии предполагается следующая конфигурация:

* В каждом концентраторе имеется несколько концентраторов с брандмауэром Azure.
* Вы используете защищенный виртуальный концентратор.
* Были настроены соответствующие политики для трафика частного трафика (VNet), Интернета и филиала.
* Между виртуальными сетями (V2I), "VNet-to-Branch" (V2B), Branch-to-VNet (B2V), проходят через брандмауэр Azure в каждом концентраторе.

Дополнительные рекомендации

* Для межцентрового сценария с брандмауэром Azure предполагается, что лучевая виртуальных сетей не связана с отдельными таблицами маршрутов. (например, **Виртуальная сеть 1** , связанная с **таблицей маршрутизации A**, и **Виртуальная сеть 2** , связанная с **таблицей маршрутизации B**). Все виртуальных сетей связаны с той же таблицей маршрутов, в которой находятся маршруты для брандмауэра Azure.
* Безопасность через брандмауэр Azure в настоящее время ограничена только **Подразделением < > виртуальной сети** и **Интернет** -трафиком. Поток ветвей между ветвями через брандмауэр Azure в настоящее время не поддерживается.

## <a name="scenario-workflow"></a><a name="workflow"></a>Рабочий процесс сценария

Чтобы настроить этот сценарий, примите во внимание следующие шаги:

### <a name="step-1"></a><a name="step-1"></a>Шаг 1

Если вы уже защитили подключения с помощью диспетчера брандмауэра Azure, сначала необходимо убедиться, что все подключения филиалов и виртуальных сетей распространяют **None**. Это необходимо для обеспечения настройки трафика через брандмауэр Azure.

1. Выберите центр и измените таблицу маршрутов **None** .
1. **Распространение** обновлений для выбора всех ветвей и всех виртуальных сетей.

### <a name="step-2"></a><a name="step-2"></a>Шаг 2

Настройка пользовательской таблицы маршрутов для каждого концентратора.

1. Для **концентратора 1**создайте таблицу маршрутов **RT_Hub1**.

    * Если вы использовали диспетчер брандмауэра Azure, он автоматически создает маршрут для 0.0.0.0/0 со следующим прыжком **AZFW1** в таблице маршрутов по умолчанию для концентратора. Этот параметр будет изменен на шаге 3. Для **RT_Hub1**явно создайте запись для 0.0.0.0/0 с помощью следующего прыжка **AZFW1**. Этот параметр активирует V2V, B2V и V2I через AZFW1.
    * Так как вы хотите, чтобы ветви **Hub** и виртуальных сетей были достигнуты через **AZFW2** из **концентратора 1** (вместо AZFW1), вам нужно добавить еще два агрегированных маршрута для ветвей **2 концентратора** (10.5.0.0/16->AZFW2) и виртуальных сетей (10.2.0.0/16->AZFW2).

1. В **концентраторе 2**создайте таблицу маршрутов **RT_Hub2**.

    * Если вы использовали диспетчер брандмауэра Azure, он автоматически создает маршрут для 0.0.0.0/0 со следующим прыжком **AZFW2** в таблице маршрутов по умолчанию для концентратора. Этот параметр будет изменен на шаге 3. Для **RT_Hub2**явно создайте запись для 0.0.0.0/0 с помощью следующего прыжка **AZFW2**. Этот параметр активирует V2V, B2V и V2I через **AZFW2**.
    * Так как вы хотите, чтобы трафик между концентраторами был защищен **AZFW2ом**Hub 2, нет необходимости явно добавлять шаг, аналогичный второму, на предыдущем шаге концентратора 1.

### <a name="step-3"></a><a name="step-3"></a>Шаг 3

Измените **таблицу маршрутов по умолчанию** в каждом концентраторе, чтобы добавить статический маршрут для **ветвей Branch to VNet** (B2V) и **branch to Internet** (B2I) через брандмауэр Azure. Ветвь в ветви сейчас не поддерживается через брандмауэр Azure.

1. Для **таблицы маршрутов по умолчанию для концентратора 1**:

    * **Ветвление к ветви Hub 2 через AZFW2**: 10.5.0.0/16->AZFW2. В этой конфигурации настраивается маршрут к брандмауэру концентратора 2.
    * **Переход к концентратору 2 виртуальных сетей через AZFW2**: 10.2.0.0/16->AZFW2. В этой конфигурации настраивается маршрут к брандмауэру концентратора 2.
    * **Ветвь к ветви (локальная), ветвь — виртуальная сеть (локальная) или ветвь в Интернет**: 0.0.0.0/0->AZFW1.

2. Для **таблицы маршрутов по умолчанию для концентратора 2**:

    * Ветвь к ветви (локальная и удаленная)/ветвь к виртуальной сети (локальная и удаленная)/ветвь в Интернет: 0.0.0.0/0->AZFW2.

Это приведет к изменению конфигурации маршрутизации, как показано на рисунке ниже.

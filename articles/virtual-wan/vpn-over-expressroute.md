# <a name="expressroute-encryption-ipsec-over-expressroute-for-virtual-wan"></a>Шифрование ExpressRoute: IPsec над ExpressRoute для виртуального WAN

В этой статье показано, как использовать Azure Virtual WAN для создания VPN-соединения IPsec/IKE из вашей предварительной сети в Azure через приватное пиринг цепи Azure ExpressRoute. Этот метод может обеспечить зашифрованный транзит между находящимися сетями и виртуальными сетями Azure через ExpressRoute, не переходя через общедоступный интернет или не используя общедоступные IP-адреса.

## <a name="topology-and-routing"></a>Топология и разгром

На следующей диаграмме показан пример подключения VPN по сравнению с частным пирингом ExpressRoute:

![VPN над ExpressRoute](./media/vpn-over-expressroute/vwan-vpn-over-er.png)

На диаграмме показана сеть в рамках предварительной сети, подключенной к VPN-шлюзу Azure hub, через приватное пиринг ExpressRoute. Установка подключения проста:

1. Установите подключение ExpressRoute с помощью схемы ExpressRoute и частного пиринга.
2. Установите подключение VPN, как описано в этой статье.

Важным аспектом этой конфигурации является маршрутирование между прилттыми сетями и Azure по путям ExpressRoute и VPN.

### <a name="traffic-from-on-premises-networks-to-azure"></a>Трафик от нахлок до Azure

Для трафика из закрытых сетей в Azure префиксы Azure (включая виртуальный концентратор и все виртуальные сети, подключенные к концентратору) рекламируются как через частный вглядывающийся BGP ExpressRoute, так и VPN BGP. Это приводит к подвезовке двух сетевых маршрутов (путей) к Azure из предприимческих сетей:

- Один над iPsec-защищенным путем
- Один прямо над ExpressRoute *без* защиты IPsec 

Чтобы применить шифрование к сообщению, необходимо убедиться, что для сети, подключенной к VPN, на диаграмме маршруты Azure через предприимчивые VPN шлюзпредпочтительны по прямому пути ExpressRoute.

### <a name="traffic-from-azure-to-on-premises-networks"></a>Трафик от Azure до сетей

То же самое требование применяется к трафику от Azure к электрическим сетям. Для обеспечения того, чтобы путь IPsec был предпочтительным по прямому маршруту ExpressRoute (без IPsec), у вас есть два варианта:

- Рекламируйте более конкретные префиксы на сессии VPN BGP для сети, подключенной к VPN. Вы можете рекламировать больший диапазон, который включает в себя VPN-подключенную сеть через ExpressRoute частных пиринг, а затем более конкретные диапазоны в сессии VPN BGP. Например, рекламировать 10.0.0.0/16 через ExpressRoute и 10.0.0.0/24 по VPN.

- Реклама разрозненных префиксов для VPN и ExpressRoute. Если диапазоны сети, подключенные к VPN, несвязаны с другими подключенными сетями ExpressRoute, вы можете рекламировать префиксы на сессиях VPN и ExpressRoute BGP соответственно. Например, рекламировать 10.0.0.0/24 по ExpressRoute и 10.0.0.0/24 по VPN.

В обоих примерах Azure будет отправлять трафик на 10.0.1.0/24 по VPN-соединению, а не непосредственно через ExpressRoute без защиты VPN.

> [!WARNING]
> Если вы рекламируете одни и те *же* префиксы как над ExpressRoute, так и vpn-соединениями, Azure будет использовать путь ExpressRoute непосредственно без защиты VPN.
>

## <a name="2-create-a-site-for-the-on-premises-network"></a><a name="site"></a>2. Создание сайта для находинной сети

Ресурс сайта такой же, как не-ExpressRoute VPN сайтов для виртуального WAN. IP-адресом натерритории VPN-устройства теперь может быть либо частный IP-адрес, либо публичный IP-адрес в предприимчивой сети, доступный через частный пиринг ExpressRoute, созданный в шаге 1.

> [!NOTE]
> IP-адрес для собственного VPN-устройства *должен* быть частью адресных префиксов, рекламируемых в виртуальном концентраторе WAN через частный пиринг Azure ExpressRoute.
>

1. Перейдите на портал Azure в браузере. 
1. Выберите созданный WAN. На странице WAN, под **Связь**, выберите **сайты VPN**.
1. На странице **VPN-сайтов** выберите **сайт «Создайте».**
1. На странице **создания сайта** заполните следующие поля:
   * **Подписка**: Проверьте подписку.
   * **Группа ресурсов**: Выберите или создайте группу ресурсов, которую вы хотите использовать.
   * **Регион**: Введите область Azure для ресурса VPN-сайта.
   * **Имя**: Введите имя, под которым вы хотите обратиться к своему месту.
   * **Поставщик устройств**: Введите поставщика на месте VPN устройства.
   * **Протокол пограничного шлюза**: Выберите "Включить", если в вашей сети используется BGP.
   * **Пространство для частных адресов**: Введите пространство IP-адреса, расположенное на вашем сайте. Трафик, предназначенный для этого адресного пространства, направляется в локальная сеть через VPN шлюз.
   * **Концентраторы**: Выберите один или несколько концентраторов для подключения этого сайта VPN. Выбранные концентраторы должны иметь VPN шлюзы уже созданы.
1. Выберите **далее: Ссылки >** для настроек VPN-ссылки:
   * **Имя ссылки**: Имя, под которым вы хотите сослаться на эту связь.
   * **Имя поставщика**: Имя интернет-провайдера для этого сайта. Для сети ExpressRoute, это имя поставщика услуг ExpressRoute.
   * **Скорость**: Скорость ссылки на интернет-услуги или схему ExpressRoute.
   * **IP-адрес**: публичный IP-адрес УСТРОЙСТВА VPN, который находится на вашем сайте. Или, для ExpressRoute на месте, это частный IP-адрес устройства VPN через ExpressRoute.

   Если BGP включен, он будет применяться ко всем соединениям, созданным для этого сайта в Azure. Настройка BGP на виртуальном WAN эквивалентна настройке BGP на VPN-шлюзе Azure. 
   
   Ваш адрес электронного ПЭГП *не должен* быть таким же, как IP-адрес вашего VPN на устройстве или виртуальном сетевом адресном пространстве сайта VPN. Используйте в качестве IP-адреса узла BGP другой IP-адрес VPN-устройства. Можно использовать адрес интерфейса замыкания на себя. Тем не менее, это *не может* быть APIPA (169.254.* x*. *x)* адрес. Укажите этот адрес в соответствующем локальном сетевом шлюзе, представляющем местоположение. Предварительные требования BGP см. статье [About BGP with Azure VPN Gateway](../vpn-gateway/vpn-gateway-bgp-overview.md) (Сведения об использовании BGP с VPN-шлюзом Azure).

1. Выберите **далее: Просмотрите и создайте >,** чтобы проверить значения настройки и создать сайт VPN. Если вы выбрали **концентраторы** для подключения, соединение будет установлено между предварительной сетью и шлюзом концентратора VPN.

## <a name="3-update-the-vpn-connection-setting-to-use-expressroute"></a><a name="hub"></a>3. Обновление настройки VPN соединения для использования ExpressRoute

После создания сайта VPN и подключения к концентратору используйте следующие шаги для настройки соединения для использования приватного пиринга ExpressRoute:

1. Вернитесь на виртуальную страницу ресурсов WAN и выберите ресурс концентратора. Или перейдите с сайта VPN на подключенный концентратор.
1. Под **connectivity**, выберите **VPN (Сайт-к-сайт)**.
1. Выберите эллипсис **(...**) на сайте VPN через ExpressRoute, и выберите **подключение Edit VPN к этому узлу.**
1. Для **использования частного IP-адреса Azure**выберите **Yes**. Настройка настраивает шлюз концентратора VPN для использования частных IP-адресов в диапазоне адресов концентратора на шлюзе для этого соединения, а не общедоступных IP-адресов. Это гарантирует, что трафик из предварительной сети пересекает частные пути пиринга ExpressRoute, а не с помощью общедоступного интернета для этого VPN-соединения. На следующем скриншоте показана параметр.

   ![Настройка для использования частного IP-адреса для VPN-соединения](./media/vpn-over-expressroute/vpn-link-configuration.png)
   
1. Нажмите кнопку **Сохранить**.

После того как вы сохраните ваши изменения, шлюз концентратор VPN будет использовать частные IP-адреса на шлюзе VPN для установления iPsec / IKE соединения с на-прилф-устройство VPN через ExpressRoute.

## <a name="4-get-the-private-ip-addresses-for-the-hub-vpn-gateway"></a><a name="associate"></a>4. Получить частные IP-адреса для шлюза концентратор VPN

Скачать конфигурацию VPN устройства, чтобы получить частные IP-адреса концентратора VPN шлюза. Эти адреса необходимы для настройки на территории VPN-устройства.

1. На странице для вашего концентратора выберите **VPN (Site-to-Site)** под **Connectivity.**
1. В верхней части страницы **Обзор,** выберите **Скачать VPN Config**. 

   Azure создает учетную запись хранилища в группе ресурсов "Microsoft-network-'location", где *местоположение* является местоположением WAN. После того, как вы примените конфигурацию к vpn-устройствам, вы можете удалить эту учетную запись хранения.
1. После создания файла выберите ссылку, чтобы загрузить его.
1. Примените конфигурацию к VPN-устройству.

### <a name="vpn-device-configuration-file"></a>Файл конфигурации VPN-устройства

Файл конфигурации устройства содержит настройки для использования при настройке вашего на фотоустройства VPN. При просмотре этого файла обратите внимание на следующие сведения:

* **vpnSiteConfiguration**: Этот раздел обозначает детали устройства, настроенные как сайт, который подключается к виртуальному WAN. Она включает в себя имя и публичный IP-адрес ветвях устройства.
* **vpnSiteConnections**: В этом разделе содержится информация о следующих настройках:

    * Адресное пространство виртуальной сети виртуального центра виртуального центра.<br/>Пример
           ```
           "AddressSpace":"10.51.230.0/24"
           ```
    * Адресное пространство виртуальных сетей, подключенных к концентратору.<br>Пример
           ```
           "ConnectedSubnets":["10.51.231.0/24"]
            ```
    * IP-адреса VPN шлюза виртуального центра. Поскольку каждое соединение VPN шлюза состоит из двух туннелей в активной активной конфигурации, вы увидите оба IP-адреса, перечисленные в этом файле. В этом примере `Instance0` `Instance1` вы видите и для каждого сайта, и они частные IP-адреса, а не общедоступные IP-адреса.<br>Пример
           ``` 
           "Instance0":"10.51.230.4"
           "Instance1":"10.51.230.5"
           ```
    * Детали конфигурации для подключения VPN-шлюза, такие как BGP и предварительно общий ключ. Предварительно общий ключ автоматически генерируется для вас. Вы всегда можете отсеить соединение на странице **Обзор** для пользовательского предварительно голого ключа.

### <a name="configuring-your-vpn-device"></a>Настройка VPN-устройства

Если вам нужны инструкции по настройке устройства, см. раздел [Загрузка сценариев конфигурации VPN-устройства из Azure](~/articles/vpn-gateway/vpn-gateway-about-vpn-devices.md#configscripts). При этом учтите следующее:

* Инструкции на странице VPN-устройства не написаны для виртуального WAN. Но вы можете использовать виртуальные значения WAN из файла конфигурации для ручной настройки вашего VPN устройства. 
* Загружаемые скрипты конфигурации устройств, которые предназначены для VPN шлюза, не работают для виртуального WAN, потому что конфигурация отличается.
* Новый виртуальный WAN может поддерживать как IKEv1, так и IKEv2.
* Виртуальный WAN может использовать только маршрутные VPN-устройства и инструкции для устройств.

Когда эти ресурсы больше не нужны, можно использовать [Remove-AzResourceGroup](/powershell/module/az.resources/remove-azresourcegroup) для удаления группы ресурсов и всех ресурсов, которые она содержит. Выполнить следующую команду PowerShell и заменить `myResourceGroup` название группы ресурсов:

```azurepowershell-interactive
Remove-AzResourceGroup -Name myResourceGroup -Force
```

# <a name="manage-users"></a>Управление пользователями

Azure FarmBeats включает в себя управление пользователями для пользователей, которые входят в ваш экземпляр Azure Active Directory (Azure AD). Вы можете добавить пользователей в экземпляр Azure FarmBeats для доступа к API, просмотра генерируемых карт и телеметрии датчиков доступа с фермы.

## <a name="prerequisites"></a>Предварительные требования

- Требуется установка Azure FarmBeats. Для получения дополнительной информации [см.](install-azure-farmbeats.md)
- Идоверки электронной почты пользователей, которые вы хотите добавить или удалить из экземпляра Azure FarmBeats.

## <a name="manage-azure-farmbeats-users"></a>Управляйте пользователями Azure FarmBeats

Azure FarmBeats использует Azure AD для проверки подлинности, управления доступом и ролей. Вы можете добавить пользователей в арендатор Azure AD в качестве пользователей в Azure FarmBeats.

> [!NOTE]
> Если пользователь не является пользователем-арендатором Azure AD, следуйте инструкциям в разделе **Add Azure AD для** завершения настройки.

Azure FarmBeats поддерживает два типа ролей пользователей:

 - **Админ**: Полный доступ к ApIs Datahub Azure FarmBeats Datahub. Пользователи в этой роли могут зазировать все объекты Azure FarmBeats Datahub и выполнять все операции с ускорителя FarmBeats.
 - **Читать только**: Читать только доступ к FarmBeats Datahub AIS. Пользователи могут просматривать API Datahub, панели мониторинга ускорителей и карты. Пользователи с доступом только для чтения не могут выполнять такие операции, как генерация карт, связывание устройств или создание ферм.

## <a name="add-users-to-azure-farmbeats"></a>Добавление пользователей в Azure FarmBeats

Чтобы добавить пользователей в Azure FarmBeats:

1. Вопийте на Accelerator, а затем выберите значок **настроек.**
2. Выберите **элемент управления доступом**.

    ![Панель настроек Фермы](./media/create-farms-in-azure-farmbeats/settings-users-1.png)

3. Введите идентификатор электронной почты пользователя, к который вы хотите предоставить доступ.
4. Выберите нужную роль, **Админ** или **Только для чтения.**
5. Выберите **Роль добавления**.

Добавленный пользователь теперь может получить доступ к Azure FarmBeats (как Datahub, так и accelerator).

## <a name="delete-users-from-azure-farmbeats"></a>Удаление пользователей из Azure FarmBeats

Чтобы удалить пользователей из системы Azure FarmBeats:

1. Вопийте на Accelerator, а затем выберите значок **настроек.**
2. Выберите **элемент управления доступом**.
3. Выберите **Удалить**.

   Пользователь удаляется из системы. Вы получите следующее сообщение подтверждения:

   ![Сообщение о подтверждении Azure FarmBeats](./media/create-farms-in-azure-farmbeats/manage-users-2.png)

## <a name="add-azure-ad-users"></a>Добавление пользователей Azure AD

> [!NOTE]
> Пользователям Azure FarmBeats необходимо существовать в наемщике Azure AD, прежде чем присвоить их приложениям и ролям. Если пользователь не существует в разделе Azure AD, следуйте инструкциям в этом разделе. Пропустить инструкции, если пользователь уже существует в агонеля Azure AD.

Выполните последующие действия по добавлению пользователей в Azure AD:

1. Войдите на [портал Azure](https://portal.azure.com/).
2. В правом верхнем ряде выберите учетную запись, а затем переключитесь на арендатора Azure AD, связанного с FarmBeats.
3. Выберите > **активных пользователей** **каталога Azure**.

    Отображается список пользователей Azure AD.

4. Чтобы добавить пользователя в каталог, выберите **нового пользователя.** Чтобы добавить внешнего пользователя, выберите **нового гостя**пользователя.

    ![Панель "Все пользователи"](./media/create-farms-in-azure-farmbeats/manage-users-3.png)

5. Выберите имя нового пользователя, а затем заполните необходимые поля для этого пользователя.
6. Выберите **Создать**.

Для получения информации об управлении пользователями Azure AD можно [добавить или удалить пользователей в Azure AD.](https://docs.microsoft.com/azure/active-directory/fundamentals/add-users-azure-active-directory/)

# <a name="generate-maps"></a>Создание карт

Используя Azure FarmBeats, вы можете создавать следующие карты с помощью спутниковых снимков и входов данных датчиков. Карты помогут вам увидеть географическое местоположение фермы и определить соответствующее местоположение для ваших устройств.

  - **Карта размещения датчиков**: Содержит рекомендации о том, сколько датчиков использовать и где разместить их на ферме.
  - **Карта спутниковых индексов**: Отображает индекс растительности и индекс воды для фермы.
  - **Тепловая карта влажности почвы**: Показывает распределение влажности почвы путем сплавлять спутниковые данные и данные датчика.

## <a name="sensor-placement-map"></a>Карта размещения датчиков

Карта размещения датчиков FarmBeats помогает вам с размещением датчиков влажности почвы. Выход карты состоит из списка координат для развертывания датчиков. Входы от этих датчиков используются вместе со спутниковыми снимками для создания тепловой карты почвы.

Эта карта получена путем сегментирования навеса, как видно на несколько дат в течение года. Даже голая почва и здания являются частью навеса. Вы можете удалить датчики, которые не требуются на месте. Эта карта предназначена для руководства, и вы можете изменить положение и цифры немного на основе ваших пользовательских знаний. Добавление датчиков не будет регрессировать почвенной влаги heatmap результаты, но есть вероятность ухудшения точности тепловой карты, если количество датчика уменьшается.

## <a name="before-you-begin"></a>Перед началом

Совырусь со следующими предпосылками перед попыткой создать карту размещения датчиков:

- Размер фермы должен быть более одного акра.
- Количество безоблачных сцен Sentinel должно быть больше шести для выбранного диапазона дат.
- По крайней мере, шесть безоблачных сцен Sentinel должны иметь нормализованный индекс растительности разницы (NDVI), превышающей или равный 0,3.

    > [!NOTE]
    > Sentinel позволяет только два одновременных потока на пользователя. В результате задания попадают в очередь и могут занять больше времени.

### <a name="dependencies-on-sentinel"></a>Зависимости от Стража

Следующие зависимости относятся к Sentinel:

- Мы зависим от производительности Sentinel для загрузки спутниковых изображений. Проверьте состояние производительности и [эксплуатацию](https://scihub.copernicus.eu/twiki/do/view/SciHubNews/WebHome)Sentinel.
- Sentinel позволяет только два [одновременных потоков загрузки](https://sentinels.copernicus.eu/web/sentinel/sentinel-data-access/typologies-and-services) на пользователя.
- Точность генерации карты зависит от [покрытия Sentinel и пересмотреть частоту.]( https://sentinel.esa.int/web/sentinel/user-guides/sentinel-2-msi/revisit-coverage)

## <a name="create-a-sensor-placement-map"></a>Создание карты размещения датчиков
В этом разделе подробно описаны процедуры создания карт размещения датчиков.

> [!NOTE]
> Вы можете инициировать карту размещения датчиков со **страницы «Карты»** или из меню **«Карты точного получения»** на странице **«Детали фермы».**

Выполните следующие действия.

1. На главной странице перейдите на **карты** из меню левой навигации.
2. Выберите **Создать Карты**и выбрать **размещение датчиков** из выпадающего меню.

    ![Выбрать размещение датчиков](./media/get-sensor-data-from-sensor-partner/create-maps-drop-down-1.png)

3. После выбора **размещения датчиков**появляется окно **размещения датчиков.**

    ![Окно размещения датчиков](./media/get-sensor-data-from-sensor-partner/sensor-placement-1.png)

4. Выберите ферму из меню **«Выпадение фермы».**
   Для поиска и выбора фермы можно прокрутить в списке выпадающих или ввести название фермы в текстовом поле.
5. Чтобы создать карту за последний год, выберите **Рекомендуемые**.
6. Чтобы создать карту для пользовательского диапазона дат, выберите **опцию Выберите Диапазон Дат.** Введите дату начала и окончания, для которой требуется создать карту размещения датчиков.
7. Выберите **Карты создания**.
 Появляется сообщение о подтверждении с данными о работе.

  Для получения информации о **View Jobs**состоянии задания см. Если статус задания *отображается неудавшимся,* на инструменте —наборе *статуса Failed* отображается подробное сообщение об ошибке. В этом случае повторите предыдущие шаги и повторите попытку.

  Если проблема сохраняется, см. раздел [Troubleshoot](troubleshoot-azure-farmbeats.md) или обратитесь в [форум Azure FarmBeats для поддержки](https://aka.ms/FarmBeatsMSDN) с соответствующими журналами.

### <a name="view-and-download-a-sensor-placement-map"></a>Просмотр и загрузка карты размещения датчиков

Выполните следующие действия.

1. На главной странице перейдите на **карты** из меню левой навигации.

    ![Выберите Карты из меню левой навигации](./media/get-sensor-data-from-sensor-partner/view-download-sensor-placement-map-1.png)

2. Выберите **фильтр** из левой навигации окна.
  Окно **фильтра** отображает критерии поиска.

    ![Окно фильтра](./media/get-sensor-data-from-sensor-partner/view-download-filter-1.png)

3. Выберите **тип,** **Дата**и значения **имен** из выпадающих меню. Затем выберите **Apply** для поиска карты, которая требуется для просмотра.
  Дата создания задания отображается в формате type_farmname_YYYY-Mm-DD.
4. Прокрутите список карт, доступных с помощью навигационных баров в конце страницы.
5. Выберите карту, которая вы хотите просмотреть. Всплывающее окно отображает предварительный просмотр для выбранной карты.
6. Выберите **Скачать**, и скачать файл GeoJSON датчиккопесков.

    ![Предварительный просмотр карты размещения датчиков](./media/get-sensor-data-from-sensor-partner/download-sensor-placement-map-1.png)

## <a name="satellite-indices-map"></a>Карта спутниковых индексов

В следующих разделах разъясняется процедура, связанная с созданием и просмотром карты спутниковых индексов.

> [!NOTE]
> Вы можете инициировать карту спутниковых индексов со страницы **Карты** или из меню выпадающих карт **Generate Precision Maps** на странице **«Подробности фермы».**

FarmBeats предоставляет вам возможность генерировать NDVI, Расширенный индекс растительности (EVI), и нормализованный индекс воды разница (NDWI) карты для ферм. Эти индексы помогают определить, как урожай в настоящее время растет, или вырос в прошлом, и репрезентативные уровни воды в земле.


> [!NOTE]
> Для дней, в течение которых создается карта, требуется изображение Sentinel.

## <a name="soil-moisture-heatmap"></a>Тепловая карта влажности почвы

Почва влаги воды, которая находится в пространствах между частицами почвы.Тепловая карта Soil Moisture поможет вам понять данные о влажности почвы на любой глубине, на высоком разрешении в пределах вашей фермы. Для создания точной и применяемой в лагаемой тепловой карты Soil Moisture требуется единообразное развертывание датчиков. Все датчики должны быть от одного и того же поставщика. Различные поставщики имеют различия в том, как влажность почвы измеряется вместе с различиями в калибровке. Тепловая карта генерируется на определенную глубину с помощью датчиков, развернутых на этой глубине.

### <a name="before-you-begin"></a>Перед началом

Совысьте со следующими предпосылками, прежде чем пытаться создать тепловую карту влажности почвы:

- Необходимо развернуть не менее трех датчиков влажности почвы. Не пытайтесь создать тепловую карту влажности почвы до того, как датчики будут развернуты и связаны с фермой.
- Создание тепловой карты Soil Moisture зависит от покрытия траектории Sentinel, облачности и тени облаков. По крайней мере, одна безоблачная сцена Sentinel должна быть доступна в течение последних 120 дней, начиная со дня, в течение которого была запрошена тепловая карта Soil Moisture.
- По крайней мере половина датчиков, развернутых на ферме, должны быть в режиме онлайн и иметь поток данных в datahub.
- Тепловая карта должна быть создана с помощью датчиков от того же поставщика.


## <a name="create-a-soil-moisture-heatmap"></a>Создание тепловой карты влажности почвы

Выполните следующие действия.

1. На главной странице перейдите на **карты** из меню левой навигации, чтобы просмотреть страницу **Карты.**
2. Выберите **Создать Карты**и выбрать **почву влаги** из выпадающих меню.

    ![Выберите влажность почвы из выпадающего меню](./media/get-sensor-data-from-sensor-partner/create-maps-drop-down-soil-moisture-1.png)

3. После выбора **почвы влаги,** окно **почвы влаги** появляется.

    ![Окно влажности почвы](./media/get-sensor-data-from-sensor-partner/soil-moisture-1.png)

4. Выберите ферму из меню **«Выпадение фермы».**
   Для поиска и выбора фермы можно прокрутить из списка выпадающих или ввести название фермы в меню выпадения **фермы Select.**
5. В меню **Select Soil Moisture Sensor Measure drop-down,** выберите датчик влажности почвы (глубина), для которого вы хотите создать карту.
Чтобы найти меру датчика, перейдите к **датчикам,** и выберите любой датчик влажности почвы. Затем, под разделом **Sensor Properties,** используйте значение в **названии измерения.**
6. Для создания карты на **сегодня** или **на этой неделе**, выберите один из вариантов.
7. Чтобы создать карту для пользовательского диапазона дат, выберите **опцию Выберите Диапазон Дат.** Введите дату начала и окончания, для которой вы хотите создать тепловую карту почвы влаги.
8. Выберите **Карты создания**.
 Появляется сообщение о подтверждении с данными о работе.

   ![Сообщение о подтверждении карты влажности почвы](./media/get-sensor-data-from-sensor-partner/successful-soil-moisture-1.png)

    Для получения информации о **View Jobs**состоянии задания см. Если статус задания *отображается неудавшимся,* на инструменте —наборе *статуса Failed* отображается подробное сообщение об ошибке. В этом случае повторите предыдущие шаги и повторите попытку.

    Если проблема сохраняется, см. раздел [Troubleshoot](troubleshoot-azure-farmbeats.md) или обратитесь в [форум Azure FarmBeats для поддержки](https://aka.ms/FarmBeatsMSDN) с соответствующими журналами.

Выполните следующие действия.

1. На главной странице перейдите на **карты** из меню левой навигации.

2. Выберите **фильтр** из левой навигации окна. Окно **фильтра** отображает, откуда можно искать карты.

    ![Выберите фильтр из левой навигации](./media/get-sensor-data-from-sensor-partner/view-download-filter-1.png)

3.  Выберите **тип,** **Дата**и значения **имен** из выпадающих меню. Затем выберите **Apply** для поиска карты, которая требуется для просмотра. Дата создания задания отображается в формате type_farmname_YYYY-Mm-DD.

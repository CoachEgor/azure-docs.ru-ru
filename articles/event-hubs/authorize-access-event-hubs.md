# <a name="authorize-access-to-azure-event-hubs"></a>Авторизация доступа к службе "Центры событий Azure"
Каждый раз, когда вы публикуете или потребляете события/данные из концентратора событий, клиент пытается получить доступ к ресурсам Event Hubs. Каждый запрос на безопасный ресурс должен быть авторизован, чтобы служба была уверена, что клиент имеет необходимые разрешения на публикацию/потребление данных. 

Azure Event Hubs предлагает следующие варианты авторизации доступа к защищенным ресурсам:

- Azure Active Directory
- Подписанный URL-адрес

> [!NOTE]
> Эта статья относится как к сценариям Event Hubs, так и к сценариям [Apache Kafka.](event-hubs-for-kafka-ecosystem-overview.md) 

## <a name="azure-active-directory"></a>Azure Active Directory
Интеграция Azure Active Directory (Azure AD) для ресурсов Event Hubs обеспечивает элементанный контроль доступа (RBAC) для мелкого контроля над доступом клиента к ресурсам. Можно использовать элементуправления доступа на основе ролей (RBAC) для предоставления разрешений директору по безопасности, который может быть пользователем, группой или директором службы приложений. Директор безопасности проверяется Azure AD для возврата токена OAuth 2.0. Токен может быть использован для авторизации запроса на доступ к ресурсу Концентраторов событий.

Для получения дополнительной информации о проверке подлинности с помощью Azure AD см.

- [Аутентификации запросов в концентраторы событий Azure с помощью активного каталога Azure](authenticate-application.md)
- [Разрешить доступ к ресурсам концентраторов событий с помощью active-каталога Azure.](authorize-access-azure-active-directory.md)

## <a name="shared-access-signatures"></a>Подписанные URL-адреса 
Общие подписи доступа (SAS) для ресурсов Event Hubs обеспечивают ограниченный делегированный доступ к ресурсам Event Hubs. Добавление ограничений на временной интервал, для которого подпись действительна, или на разрешения, которые она предоставляет, обеспечивает гибкость в управлении ресурсами. Для получения дополнительной информации [см. Authenticate с помощью общих подписей доступа (SAS).](authenticate-shared-access-signature.md) 

Разрешение пользователей или приложений с помощью токена OAuth 2.0, возвращенного Azure AD, обеспечивает превосходную безопасность и простоту использования над общими подписями доступа (SAS). С Azure AD нет необходимости хранить маркеры доступа с вашим кодом и рисковать потенциальными уязвимостями безопасности. В то время как вы можете продолжать использовать подписи общего доступа (SAS) для предоставления мелкозернистых доступа к ресурсам Event Hubs, Azure AD предлагает аналогичные возможности без необходимости управлять токенами SAS или беспокоиться об отзыве скомпрометированного SAS. 

По умолчанию все ресурсы Концентраторов событий защищены и доступны только владельцу учетной записи. Хотя вы можете использовать любую из стратегий авторизации, описанных выше, чтобы предоставить клиентам доступ к ресурсам Event Hub. Корпорация Майкрософт рекомендует использовать Azure AD, когда это возможно, для максимальной безопасности и простоты использования.

Для получения дополнительной информации о авторизации с помощью SAS см. [Авторизующий доступ к ресурсам концентраторов событий с использованием общих подписей доступа.](authorize-access-shared-access-signature.md)

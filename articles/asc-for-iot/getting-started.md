# <a name="get-started-with-azure-security-center-for-iot"></a>Начало работы с центром безопасности Azure для Интернета вещей 

> [!IMPORTANT]
> Центр безопасности Azure для Интернета вещей сейчас предоставляется в общедоступной предварительной версии.
> Эта предварительная версия предоставляется без соглашения об уровне обслуживания и не рекомендована для использования рабочей среде. Некоторые функции могут не поддерживаться или их возможности могут быть ограничены. Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

В этой статье приводится объяснение различных стандартных блоков центр безопасности Azure (ASC) для службы Интернета вещей и объясняется, как приступить к работе с [Включение службы](quickstart-onboard-iot-hub.md). 

ASC для Интернета вещей можно легко интегрируется в центр Интернета вещей для анализа безопасности конфигурации концентратора, удостоверения устройства и устройств для центра Интернета вещей шаблоны взаимодействия.
Для расширенные возможности обеспечения безопасности ASC для Интернета вещей предоставляет коллекции на основе агента безопасности данных с устройств Интернета вещей.

## <a name="asc-for-iot-seamless-iot-hub-integration"></a>ASC для Интернета вещей интеграция центра Интернета вещей

При попытке защитить отдельных устройств Интернета вещей, необходима возможность собирать данные напрямую из устройств или из своей сети. Для поддержки этих усилий, ASC для Интернета вещей предлагает широкий ряд небольшого безопасности агентов для обеспечения мониторинга устройств и усиления защиты.

В ASC для Интернета вещей preview Эталонная архитектура для безопасности агентов Linux и Windows, как в C# и предоставляются C.
Агенты обрабатывать сбора необработанных событий из операционной системы устройства, статистической обработки событий для сокращения затрат и конфигурацию с помощью двойника модуля устройства.
Безопасность сообщения отправляются через Центр Интернета вещей в ASC для служб аналитики для Интернета вещей.

## <a name="asc-for-iot-basics"></a>ASC для основы Интернета вещей

Выберите сценарий рабочего процесса, который наилучшим образом соответствующего требованиям вашей Интернета вещей устройства и среды:

### <a name="get-started-with-asc-for-iot-seamless-iot-hub-integration"></a>Начало работы с ASC для Интернета вещей интеграция центра Интернета вещей 

>[!Note]
>Этот рабочий процесс позволяет использовать службу без использования ASC для агентов безопасности Интернета вещей. 

Чтобы включить мониторинг устройства управления удостоверениями, устройства в облако и облако для шаблонов взаимодействия устройств, используйте приведенный ниже основной рабочий процесс для тестирования и запустить службу: 

1. [Включить ASC для службы Интернета вещей в центре Интернета вещей](quickstart-onboard-iot-hub.md)
1. Если нет зарегистрированных устройств центра Интернета вещей [Зарегистрируйте новое устройство](https://docs.microsoft.com/azure/iot-accelerators/quickstart-device-simulation-deploy).
1. [Создание модуля безопасности azureiotsecurity устройств](quickstart-create-security-twin.md) для устройств. 
1. Определить нормальное поведение устройства и системы через [настраиваемые оповещения](quickstart-create-custom-alerts.md). 
1. Выполните тестирование для проверки состояния службы и устройства системы. 
1. Изучите [оповещения](concept-security-alerts.md), [рекомендации](concept-recommendations.md), и [глубокое погружение в обработку с помощью Log Analytics](how-to-security-data-access.md) с помощью центра Интернета вещей. 


### <a name="get-started-with-asc-for-iot-security-agents"></a>Начало работы с ASC для агентов безопасности Интернета вещей

Воспользоваться ASC для возможности обеспечения безопасности усиленной Интернета вещей, такие как мониторинг удаленные подключения, активные приложения, события входа в систему и рекомендации по конфигурации операционной системы, используя следующий базовый рабочего процесса для тестирования и включения службы: 

1. [Включить ASC для службы Интернета вещей к центру Интернета вещей](quickstart-onboard-iot-hub.md)
1. Если нет зарегистрированных устройств центра Интернета вещей [Зарегистрируйте новое устройство](https://docs.microsoft.com/azure/iot-accelerators/quickstart-device-simulation-deploy).
1. [Создание модуля безопасности azureiotsecurity](quickstart-create-security-twin.md) для устройств.
1. Чтобы установить агент на устройстве Azure имитации вместо установки на настоящем устройстве, [spin up новой виртуальной машины (ВМ) Azure](https://docs.microsoft.com/azure/virtual-machines/linux/quick-create-portal) в зоне доступны. 
1. [Развертывание ASC для агента безопасности Интернета вещей](how-to-deploy-linux-cs.md) на устройстве Интернета вещей, или в новую виртуальную Машину.
1. Следуйте инструкциям по [trigger_events](https://aka.ms/iot-security-github-trigger-events) запуска моделирования безвредным атаки.
1. Проверка ASC для Интернета вещей оповещения в ответ для имитации атак на предыдущем шаге. Начните проверку подлинности через пять минут после выполнения скрипта.
1. Изучите [оповещения](concept-security-alerts.md), [рекомендации](concept-recommendations.md), и [глубокое погружение в обработку с помощью Log Analytics](how-to-security-data-access.md) с помощью центра Интернета вещей.

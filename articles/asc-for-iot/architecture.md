# <a name="azure-security-center-for-iot-architecture"></a>Центр безопасности Azure для архитектуры Интернета вещей

В этой статье описывается архитектура работы системы из центра безопасности Azure (ASC) решения Интернета вещей. 

> [!IMPORTANT]
> Центр безопасности Azure для Интернета вещей сейчас предоставляется в общедоступной предварительной версии.
> Предварительная версия предоставляется без соглашения об уровне обслуживания. Мы не рекомендуем использовать ее в рабочей среде. Некоторые функции могут не поддерживаться или их возможности могут быть ограничены. Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

## <a name="asc-for-iot-components"></a>ASC для компонентов Интернета вещей

ASC для Интернета вещей состоит из следующих компонентов:
- Агенты устройств
- Отправить сообщение о безопасности пакета SDK
- Интеграция центра Интернета вещей
- Конвейер аналитики
 
### <a name="asc-for-iot-workflow"></a>ASC для Интернета вещей рабочего процесса

ASC для агентов устройства Интернета вещей позволяет легко собирать события необработанной безопасности с устройств. События необработанной безопасности может включать IP-адрес подключения, создания процесса, имена входа и других связанных с безопасностью сведения. ASC для агентов устройства Интернета вещей также обрабатывать статистической обработки событий, что помогает избежать высокой нагрузки на сеть пропускной способности. Агенты являются широкие возможности настройки, позволяя использовать их для выполнения конкретных задач, например, отправку только важную информацию в наиболее быстрый соглашение об уровне ОБСЛУЖИВАНИЯ, или для объединения в большие сегменты, сведения о безопасности и контекст Предотвращение больше затрат на службу.
 
Агенты устройств и используются в других приложениях **Azure ASC отправки сообщения безопасности SDK** для отправки информации о безопасности в центре Интернета вещей Azure. Центр Интернета вещей извлекает эти сведения и перенаправляет его ASC для службы Интернета вещей.

После включения ASC для службы Интернета вещей, помимо пересылки данных, центр Интернета вещей также отправляет все внутренние данные для анализа с помощью ASC для Интернета вещей. Эти данные включают журналы операций устройства облако удостоверений устройств и конфигурация концентратора. Вся эта информация помогает в создании ASC для конвейера аналитики Интернета вещей.
 
ASC для конвейера аналитики IoT также получает дополнительных угроз аналитики потоки из различных источников в корпорации Microsoft и партнеров. ASC для конвейера всего аналитика Интернета вещей работает с каждой клиентской конфигурации, внесенные в службе (например, пользовательские оповещения и использование пакета SDK для отправки безопасности сообщений).
 
С помощью конвейера аналитики, ASC для Интернета вещей сочетает в себе все потоки информации для формирования практические рекомендации и оповещения. Конвейер содержит оба настраиваемые правила, созданные исследователей систем безопасности и экспертов, а также поиск отклонение от анализа поведения и риска стандартное устройство моделей машинного обучения.
 
ASC для Интернета вещей рекомендации и предупреждения, (выходные данные конвейера аналитики) записывается в рабочей области Log Analytics для каждого клиента. Включая необработанных событий в рабочей области, а также оповещений, а также рекомендации позволяет исследования глубокое погружение в обработку и запросы, используя точные сведения о подозрительных действий, обнаруженных.

# <a name="azure-security-center-for-iot-architecture"></a>Архитектура Центра безопасности Azure для Интернета вещей

В этой статье объясняется архитектура функциональной системы решения Azure Security Center for IoT. 

## <a name="azure-security-center-for-iot-components"></a>Центр безопасности Azure для компонентов IoT

Центр безопасности Azure для IoT состоит из следующих компонентов:
- Интеграция Концентратора IoT
- Агенты устройства (необязательно)
- Отправка сообщения безопасности SDK
- Аналитический конвейер
 
### <a name="azure-security-center-for-iot-workflows"></a>Центр безопасности Azure для рабочих процессов IoT

Центр безопасности Azure для IoT работает в одном из двух рабочих процессов функциональности: Встроенный и расширенный  

### <a name="built-in"></a>Встроено
В **режиме встроенного** в себя центр безопасности Azure для IoT включен при выборе опции **безопасности** в концентраторе IoT. Предлагая мониторинг, рекомендации и оповещения в режиме реального времени, встроенный режим предлагает одноступенчатую видимость устройства и непревзойденную безопасность. Режим сборки не требует установки агента на любых устройствах и использует расширенную аналитику на зарегистрированных действиях для анализа и защиты вашего полевого устройства. 

### <a name="enhanced"></a>Расширенная архитектура 
В **расширенном** режиме после включения **опции безопасности** в концентраторе IoT и установки Центра безопасности Azure для агентов устройств IoT на ваших устройствах агенты собирают, агрегируют и анализируют необработанные события безопасности с ваших устройств. Необработанные события безопасности могут включать IP-соединения, создание процессов, логины пользователей и другую информацию, соответствующую безопасности. Центр безопасности Azure для агентов устройств IoT также занимается агрегацией событий, чтобы избежать высокой пропускной записи сети. Агенты очень настраиваемые, что позволяет использовать их для конкретных задач, таких как отправка только важной информации в самом быстром SLA, или для агрегирования обширной информации безопасности и контекста в более крупных сегментах, избегая более высоких затрат на обслуживание.

![Архитектура Центра безопасности Azure для Интернета вещей](./media/architecture/azure-iot-security-architecture.png)
 
Агенты устройств и другие приложения используют Сообщение безопасности SDK для отправки **sDK сообщения безопасности** для отправки информации о безопасности в концентратор Azure IoT. Концентратор IoT забирает эту информацию и направляет ее в Центр безопасности Azure для службы IoT.

После включения службы безопасности Azure Для IoT, в дополнение к переадресованным данным, IoT Концентр также отправляет все свои внутренние данные для анализа Центром безопасности Azure для IoT. Эти данные включают журналы операций устройства и облака, идентификаторы устройств и конфигурацию концентратора. Вся эта информация помогает создать конвейер Azure Security Center для анализа IoT.
 
Центр безопасности Azure Security Center для конвейера ioT-аналитики также получает дополнительные потоки информации об угрозах из различных источников в рамках партнеров Майкрософт и Майкрософт. Центр безопасности Azure для всего конвейера аналитики IoT работает с каждой конфигурацией клиента, сделанной в службе (например, пользовательские оповещения и использование sDK сообщения безопасности отправки).
 
Используя конвейер аналитики, Центр безопасности Azure для IoT объединяет все потоки информации для создания практических рекомендаций и оповещений. Конвейер содержит как пользовательские правила, созданные исследователями безопасности и экспертами, так и модели машинного обучения, ищущие отклонения от стандартного поведения устройства и анализа рисков.
 
Центр безопасности Azure для рекомендаций и оповещений IoT (аналитический выход конвейера) записывается в рабочее пространство журнала Analytics каждого клиента. Включение необработанных событий в рабочее пространство, а также оповещений и рекомендаций позволяет проводить глубокие исследования и запросы с использованием точных деталей обнаруженной подозрительной деятельности.

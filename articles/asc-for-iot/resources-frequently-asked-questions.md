# <a name="azure-security-center-for-iot-frequently-asked-questions"></a>Центр безопасности Azure для Интернета вещей часто задаваемые вопросы  

> [!IMPORTANT]
> Центр безопасности Azure для Интернета вещей сейчас предоставляется в общедоступной предварительной версии.
> Предварительная версия предоставляется без соглашения об уровне обслуживания. Мы не рекомендуем использовать ее в рабочей среде. Некоторые функции могут не поддерживаться или их возможности могут быть ограничены. Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

Статья содержит список часто задаваемых вопросов и ответов о центре безопасности Azure (ASC) для Интернета вещей. 

## <a name="does-azure-provide-support-for-iot-security"></a>Azure предоставляет поддержку безопасности Интернета вещей?

Azure предоставляет интегрированное представление для мониторинга и управления ими безопасности Интернета вещей в рамках общего решения безопасности в центре безопасности Azure. Если вы являетесь разработчиком приложения, можно использовать представление вещей для управления безопасности приложений Интернета вещей.

## <a name="what-is-azures-unique-value-proposition-for-iot-security"></a>Что такое предложение Azure уникальное значение для безопасности Интернета вещей?

ASC для Интернета вещей позволяет организациям расширить существующие кибер безопасности представление и все свое решение Интернета вещей. Azure обеспечивает комплексное представление бизнес-решения, позволяя предпринимать действий, связанных с бизнесом и решения, исходя из уровня безопасности предприятия и собранных данных. В сочетании с помощью Azure IoT Edge Интернета вещей Azure, Azure Sphere, Azure центра безопасности и центр безопасности Azure позволяют создавать решения, которые нужно с обеспечением необходимой безопасности.

## <a name="who-is-asc-for-iot-made-for"></a>Кто является ASC Интернета вещей, внесенных? 

ASC для Интернета вещей интегрирована в безопасности в центре Интернета вещей Azure и обеспечивает управление для решения повседневных бизнес-операций безопасности. ASC для Интернета вещей также интегрируется в возможности центра безопасности Azure и создать интегрированное представление для мониторинга и управления ими безопасности Интернета вещей как часть решения безопасности в целом.

## <a name="how-does-asc-for-iot-compare-to-the-competition"></a>ASC для Интернета вещей сравнение конкурентов?

Хотя другие решения предоставляют ряд возможностей, которые позволяют пользователям создавать собственные решения, ASC для Интернета вещей предоставляет уникальное решение безопасности Интернета вещей end-to-end, который предоставляет широкий обзор безопасности всех связанных ресурсов Azure. Azure обеспечивает быстрое развертывание и полная интеграция с двойниками модулей центра Интернета вещей для облегчения интеграции с существующими средствами управления устройства.

## <a name="do-i-have-to-be-an-azure-security-center-asc-customer"></a>Нужно ли быть являетесь клиентом центра безопасности Azure (ASC)?

Нет, но рекомендуется. Без ASC ASC для Интернета вещей получает ограниченный ресурс подключенных данных и предоставляет ограниченный анализа потенциальных атак, угроз и потенциальных атак. 

## <a name="do-i-have-to-be-an-azure-iot-customer"></a>Нужно ли быть клиента из Интернета вещей Azure?

Да. ASC для Интернета вещей зависит от подключения к Интернету вещей Azure и инфраструктурой.

## <a name="do-i-have-to-install-an-agent"></a>Нужно ли установить агент?

Установка агента на устройствах Интернета вещей, не является обязательным для включения ASC Майкрософт для Интернета вещей. Можно выбрать один из следующих трех вариантов получения возможности мониторинга и управления в зависимости от выбора различных уровней безопасности:

1. Установите ASC для агента безопасности Интернета вещей, с изменениями или без. Этот вариант обеспечивает самый высокий уровень усиленной безопасности анализ поведения устройства и доступа.

2. Создайте свой собственный агент и реализуйте Microsoft ASC для схемы сообщений безопасности Интернета вещей. Этот параметр включает использование Microsoft ASC для Интернета вещей средства анализа на основе агент безопасности вашего устройства.

3. Установка агента не безопасности на устройствах Интернета вещей. Этот параметр позволяет центра Интернета вещей обмен данными мониторинга, с помощью возможности мониторинга и управления снижению уровня безопасности. 

## <a name="what-does-the-asc-for-iot-agent-do"></a>Что делает ASC для Интернета вещей агент?

ASC для Интернета вещей агент обеспечивает полного покрытия устройств угроз на уровне конфигурации устройства, поведение и доступ (путем сканирования конфигурации), процесс и подключения. ASC для агента безопасности Интернета вещей не проверяли, бизнес-данные или действия.

## <a name="where-i-can-get-the-asc-for-iot-security-agent"></a>Где можно получить ASC для агента безопасности Интернета вещей?

ASC для агента безопасности Интернета вещей имеет открытый исходный код и доступны на сайте GitHub в 32-разрядные и 64-разрядных версий Windows и Linux: https://github.com/Azure/Azure-Security-IoT-Preview

## <a name="where-does-the-asc-for-iot-agent-get-installed"></a>Где получить установлен ASC для Интернета вещей агент? 

Подробные сведения об установке и агент развертывания можно найти в GitHub: https://github.com/Azure/Azure-Security-IoT-Preview

## <a name="what-are-the-dependencies-and-prerequisites-of-the-agent"></a>Каковы зависимости и необходимые компоненты агента?

ASC для Интернета вещей поддерживает широкий спектр платформ. См. в разделе [платформ устройств поддерживается](how-to-deploy-agent.md) о поддержке для конкретных устройств. 

## <a name="which-data-is-collected-by-the-agent"></a>Какие данные собираются агентом?

Подключения, доступа, конфигурация брандмауэра, список процессов и базовой ОС собираются агентом.

## <a name="how-much-data-will-the-agent-generate"></a>Какой объем данных создаст агент?

Создание агента данных зависит от устройства, приложения, тип подключения и конфигурация агента клиента. Из-за высокой вариативностью между устройствами и решений Интернета вещей мы рекомендуем предварительного развертывания агента в лаборатории или параметр тестирования для наблюдать, Дополнительные сведения и установите конкретной конфигурации, который соответствует вашим потребностям, при измерении объем созданных данных. После запуска службы, ASC для агента Интернета вещей предоставляет рекомендации по эксплуатации для оптимизации пропускной способности агента для помощи в процессе конфигурации и настройки.

## <a name="how-can-i-control-my-billing"></a>Как контролировать счета?

ASC для Интернета вещей предоставляет можно настроить агент, сканирование, буферов данных и возможность создавать пользовательские оповещения, увеличить или уменьшить объем данных, создаваемые агентом.

## <a name="do-agent-messages-use-up-quota-from-iot-hub"></a>Следует ли использовать сообщения агента вверх квоты из центра Интернета вещей?

Да. Агент передачи данных считается квоты центра Интернета вещей. 

## <a name="what-next-ive-installed-an-agent-and-dont-see-any-activities-or-logs"></a>Что делать дальше? Я установлен агент и не будете видеть действия или журналы

1. Проверьте [тип агента соответствует указанной платформы ОС устройства](how-to-deploy-agent.md)

1. Подтвердите [агент запущен на устройстве](how-to-agent-configuration.md).

2. Проверьте [успешно включена служба](quickstart-onboard-iot-hub.md) для **безопасности** в центре Интернета вещей. 

3. Убедитесь, что устройство не соответствует [настроен в центре Интернета вещей с ASC для модуля IoT](quickstart-create-security-twin.md).  

Если действия или журналы по-прежнему недоступны, обратитесь к вашей ASC для Интернета вещей партнера для получения дополнительной поддержки.

## <a name="what-happens-when-the-internet-connection-stops-working"></a>Что произойдет, если подключение к Интернету перестает работать?

Агент продолжает работать и хранения данных, до тех пор, пока устройство работает под управлением. Данные хранятся в кэше сообщения безопасности в соответствии с конфигурацией размер. Когда устройство восстанавливает подключение, безопасность сообщений возобновить отправку. 

## <a name="if-the-device-is-restarted-will-the-security-agent-self-recover"></a>Если устройство перезагружается, будет агент безопасности самостоятельное восстановление?

Агент безопасности предназначен для повторного выполнения автоматически при каждой перезагрузке компьютера устройства.

## <a name="can-the-agent-affect-the-performance-of-the-device-or-other-installed-software"></a>Агент, которую может повлиять на производительность устройства или другого установленного программного обеспечения?

Агент потребляет ресурсы компьютера, как любые другие приложения или процесса и не следует нарушить действий обычного устройства. Потребление ресурсов на устройства, на котором работает агент, связана с его установкой и настройкой. Мы рекомендуем, тестирование конфигурации агента в автономной среде, а также взаимодействие с другими приложениями Интернета вещей и функциональные возможности, прежде чем приступать к развертыванию в рабочей среде.

## <a name="im-making-some-maintenance-on-the-device-can-i-turn-off-the-agent"></a>Когда я установлю некоторые обслуживания на устройстве. Можно ли отключить агент?

Агент не может быть отключен.

## <a name="is-there-a-way-to-test-if-the-agent-is-working-correctly"></a>Есть ли способ проверить, если агент работает правильно? 

Если агент прекращает взаимодействие или не удается отправить сообщения о безопасности, **устройства молчаливы** предупреждения.

## <a name="can-i-create-my-own-alerts"></a>Можно создать собственные оповещения?

Да. Можно задать настраиваемые оповещения на предварительно определенный набор поведений, такие как IP-адрес и открытые порты. См. в разделе [создать настраиваемые оповещения](quickstart-create-custom-alerts.md) Дополнительные сведения о настраиваемых оповещений и их оптимизации. 

## <a name="where-can-i-see-logs-can-i-customize-logs"></a>Где можно просмотреть журналы? Как настроить журналы?

- Просмотр оповещений и рекомендаций с помощью подключенной рабочей области Log Analytics. Настройте размер хранилища и длительности в рабочей области.

- Необработанные данные от агента безопасности также могут храниться в учетную запись Log Analytics. Перед изменением конфигурации этого параметра рекомендуется размер, длительность, требования к хранилищу и сопутствующие затраты. 

## <a name="why-should-i-add-asc-for-iot-to-the-module-identity-what-is-it-used-for"></a>Почему следует добавить ASC для Интернета вещей на идентификатор модуля? Для чего он используется?

ASC для Интернета вещей модуля используется для настройки агента и управления.

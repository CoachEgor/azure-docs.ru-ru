# <a name="azure-security-center-for-iot-frequently-asked-questions"></a>Azure Security Center for IoT frequently asked questions (Центр безопасности Azure для Интернета вещей: часто задаваемые вопросы)  

В этой статье приводится список часто задаваемых вопросов и ответов о Центре безопасности Azure для IoT. 

## <a name="does-azure-provide-support-for-iot-security"></a>Предоставляет ли Azure поддержку безопасности IoT?

Azure предоставляет интегрированное представление для мониторинга и управления безопасностью IoT в рамках общего решения безопасности через Центр безопасности Azure. Если вы являетесь разработчиком приложений, вы можете использовать концентратор IoT для управления безопасностью приложений IoT.

## <a name="what-is-azures-unique-value-proposition-for-iot-security"></a>Каково уникальное ценностное предложение Azure для безопасности IoT?

Центр безопасности Azure для IoT позволяет предприятиям расширить существующее представление о кибербезопасности на все решение IoT. Azure обеспечивает конец конечного представления о бизнес-решении, позволяя вам принимать бизнес-действия и решения на основе вашей позиции безопасности предприятия и собранных данных. Комбинированная безопасность с помощью Azure IoT, Azure IoT Edge и Центра безопасности Azure позволяет создать необходимое решение с необходимой безопасностью.

## <a name="who-is-azure-security-center-for-iot-made-for"></a>Для кого предназначен Центр безопасности Azure для IoT? 

Центр безопасности Azure для IoT интегрирован в систему Azure IoT Hub Security и обеспечивает управление повседневных операций по обеспечению безопасности бизнес-решений. Центр безопасности Azure для IoT также интегрирован в возможности Центра безопасности Azure и обеспечивает интегрированное представление для мониторинга и управления безопасностью IoT как часть общего решения безопасности.

## <a name="how-does-azure-security-center-for-iot-compare-to-the-competition"></a>Как Центр безопасности Azure для IoT сравнивается с конкурентами?

В то время как другие решения предоставляют набор возможностей, которые позволяют клиентам создавать свои собственные решения, Azure Security Center for IoT предоставляет уникальное комплексное решение безопасности IoT, которое обеспечивает широкий обзор безопасности всех связанных с вами Azure Ресурсы. Azure обеспечивает быструю развертывание и полную интеграцию с близнецами модулей IoT Hub для легкой интеграции с существующими инструментами управления устройствами.

## <a name="do-i-have-to-be-an-azure-security-center-customer-to-use-this-service"></a>Должен ли я быть клиентом Центра безопасности Azure, чтобы воспользоваться этой услугой?

Нет, но рекомендуется. Центр безопасности Azure, Azure Security Center для IoT получает ограниченные данные о подключенных ресурсах и предоставляет ограниченный анализ потенциальной поверхности атаки, угроз и потенциальных атак. 

## <a name="do-i-have-to-be-an-azure-iot-customer"></a>Должен ли я быть клиентом Azure IoT?

Да. Центр безопасности Azure для IoT опирается на подключение и инфраструктуру Azure IoT.

## <a name="do-i-have-to-install-an-agent"></a>Должен ли я установить агент?

Установка агента на устройствах IoT не является обязательной для того, чтобы включить Центр безопасности Microsoft Azure для IoT. Вы можете выбрать один из трех вариантов, получив различные уровни мониторинга безопасности и возможностей управления в зависимости от вашего выбора:

1. Установите Центр безопасности Azure для агента безопасности IoT с изменениями или без нее. Эта опция обеспечивает наивысший уровень расширенной безопасности поведения и доступа к устройствам.

2. Создайте свой собственный агент и создайте центр безопасности Microsoft Azure для схемы создания сообщений безопасности IoT. Эта опция позволяет использовать центр безопасности Microsoft Azure для инструментов анализа IoT поверх агента безопасности устройства.

3. Установка агента безопасности на устройствах IoT не возможна. Эта опция обеспечивает мониторинг связи IoT Hub с меньшими возможностями мониторинга безопасности и управления. 

## <a name="what-does-the-azure-security-center-for-iot-agent-do"></a>Что делает Центр безопасности Azure для агента IoT?

Центр безопасности Azure для агента IoT обеспечивает покрытие уровня угрозы устройства для конфигурации, поведения и доступа (путем сканирования конфигурации), обработки & подключения. Центр безопасности Azure для агента безопасности IoT не сканирует данные или действия, связанные с бизнесом.

## <a name="where-can-i-get-the-azure-security-center-for-iot-security-agent"></a>Где я могу получить Центр безопасности Azure для агента безопасности IoT?

Центр безопасности Azure для агента безопасности IoT с открытым исходным кодом и доступен https://github.com/Azure/Azure-Security-IoTна GitHub в 32 битных и 64-битных версиях Windows и Linux: .

## <a name="where-does-the-azure-security-center-for-iot-agent-get-installed"></a>Где устанавливается Центр безопасности Azure для ioT-агента? 

Подробная информация об установке и развертывании агента можно найти в GitHub: https://github.com/Azure/Azure-IoT-Security.

## <a name="what-are-the-dependencies-and-prerequisites-of-the-agent"></a>Каковы зависимости и предпосылки агента?

Центр безопасности Azure для IoT поддерживает широкий спектр платформ. Смотрите [платформы поддержки устройств](how-to-deploy-agent.md) для проверки поддержки для ваших конкретных устройств. 

## <a name="which-data-is-collected-by-the-agent"></a>Какие данные собираются агентом?

Связь, доступ, конфигурация брандмауэра, список процессов & базовый список ОС собираются агентом.

## <a name="how-much-data-will-the-agent-generate"></a>Сколько данных будет генерировать агент?

Генерация данных агента определяется устройством, приложением, типом подключения и конфигурацией агента клиента. В связи с высокой изменчивостью между устройствами и решениями IoT мы рекомендуем сначала развернуть агента в лаборатории или тестовой настройки для наблюдения, изучения и настройки конкретной конфигурации, которая соответствует вашим потребностям, при измерении объема генерируемых данных. После запуска службы центр безопасности Azure для ioT-агента предоставляет оперативные рекомендации по оптимизации пропускной записи агента, чтобы помочь вам с процессом настройки и настройки.

## <a name="how-can-i-control-my-billing"></a>Как я могу контролировать выставление счетов?

Центр безопасности Azure для IoT предоставляет настраиваемое сканирование агентов, буферы данных и возможность создавать пользовательские оповещения, которые увеличивают или уменьшают объем данных, генерируемых агентом.

## <a name="do-agent-messages-use-up-quota-from-iot-hub"></a>Используют ли сообщения агента квоту от IoT Hub?

Да. Передаваемые агентом данные учитываются в квоте IoT Hub. 

## <a name="what-next-ive-installed-an-agent-and-dont-see-any-activities-or-logs"></a>Что делать дальше? Я установил агента и не вижу никаких действий или журналов ...

1. Проверьте [тип агента, который соответствует назначенной платформе ОС вашего устройства](how-to-deploy-agent.md)

1. Подтвердите, [что агент работает на устройстве.](how-to-agent-configuration.md)

2. Проверьте, что [служба была успешно включена](quickstart-onboard-iot-hub.md) в **систему безопасности** в вашем концентраторе IoT. 

3. Убедитесь, что устройство [настроено в концентраторе IoT с помощью модуля Azure Security Center для IoT.](quickstart-create-security-twin.md)  

Если действия или журналы по-прежнему недоступны, обратитесь в Центр безопасности Azure для партнера IoT для получения дополнительной помощи.

## <a name="what-happens-when-the-internet-connection-stops-working"></a>Что происходит, когда подключение к Интернету перестает работать?

Агент продолжает запускать и хранить данные до тех пор, пока устройство работает. Данные хранятся в кэше сообщений безопасности в зависимости от конфигурации размера. Когда устройство восстанавливает подключение, сообщения безопасности возобновляют отправку. 

## <a name="if-the-device-is-restarted-will-the-security-agent-self-recover"></a>Если устройство перезапущено, будет ли агент безопасности самовосстанавливаться?

Агент безопасности предназначен для автоматического запуска с каждым перезапуском устройства.

## <a name="can-the-agent-affect-the-performance-of-the-device-or-other-installed-software"></a>Может ли агент повлиять на производительность устройства или другого установленного программного обеспечения?

Агент потребляет ресурсы машины, как и любое другое приложение/процесс, и не должен нарушать нормальную активность устройства. Потребление ресурсов на устройстве, на которое работает агент, в сочетании с его настройкой и конфигурацией. Мы рекомендуем проверить конфигурацию агента в содержащейся среде, а также совместимость с другими приложениями и функциональностью IoT, прежде чем пытаться развернуть в производственной среде.

## <a name="im-making-some-maintenance-on-the-device-can-i-turn-off-the-agent"></a>Я делаю кое-какие обслуживания на устройстве. Могу я выключить агента?

Агент не может быть выключен.

## <a name="is-there-a-way-to-test-if-the-agent-is-working-correctly"></a>Есть ли способ проверить, работает ли агент правильно? 

Если агент перестает общаться или не отправляет сообщения безопасности, генерируется **оповещение о бесшумном** устройстве.

## <a name="can-i-create-my-own-alerts"></a>Могу ли я создать свои собственные оповещения?

Да. Можно настроить настраиваемый оповещение на заранее определенный набор поведения, таких как IP-адрес и открытые порты. Смотрите [Создать пользовательские оповещения,](quickstart-create-custom-alerts.md) чтобы узнать больше о пользовательских оповещений и как их сделать. 

## <a name="where-can-i-see-logs-can-i-customize-logs"></a>Где можно увидеть журналы? Могу ли я настроить журналы?

- Просматривайте оповещения и рекомендации, используя подключенное рабочее пространство Log Analytics. Настройка размера и продолжительности хранилища в рабочей области.

- Необработанные данные от вашего агента по безопасности также могут храниться в вашей учетной записи Log Analytics. Рассмотрите размер, продолжительность, требования к хранению и связанные с ними затраты, прежде чем менять конфигурацию этой опции. 

## <a name="why-should-i-add-azure-security-center-for-iot-to-the-module-identity-what-is-it-used-for"></a>Почему следует добавить Центр безопасности Azure для IoT в идентификационный номер модуля? Для чего он используется?

Для настройки и управления агентом используется центр безопасности Azure для IoT.

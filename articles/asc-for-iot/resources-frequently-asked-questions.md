# <a name="azure-security-center-for-iot-frequently-asked-questions"></a>Часто задаваемые вопросы о центре безопасности Azure для Интернета вещей  

В этой статье содержится список часто задаваемых вопросов и ответов на вопросы о центре безопасности Azure для Интернета вещей. 

## <a name="does-azure-provide-support-for-iot-security"></a>Предоставляет ли Azure поддержку безопасности IoT?

Azure предоставляет интегрированное представление для мониторинга и управления безопасностью Интернета вещей в рамках общего решения по обеспечению безопасности с помощью центра безопасности Azure. Если вы являетесь разработчиком приложений, вы можете использовать представление центра Интернета вещей для управления безопасностью приложений Интернета вещей.

## <a name="what-is-azures-unique-value-proposition-for-iot-security"></a>Что такое уникальное ценное предложение Azure для безопасности IoT?

Центр безопасности Azure для Интернета вещей позволяет предприятиям расширить имеющееся представление безопасности кибератак до всего решения IoT. Azure предоставляет комплексное представление вашего бизнес-решения, позволяя принимать бизнес-действия и решения, основанные на уровне корпоративной безопасности и собранных данных. Объединенная безопасность с помощью Azure IoT, Azure IoT Edge и центра безопасности Azure позволяет создать необходимое решение с учетом необходимых средств обеспечения безопасности.

## <a name="who-is-azure-security-center-for-iot-made-for"></a>Кто входит в центр безопасности Azure для Интернета вещей? 

Центр безопасности Azure для Интернета вещей интегрирован в систему безопасности центра Интернета вещей Azure и обеспечивает управление для выполнения повседневных операций безопасности бизнес-решений. Центр безопасности Azure для Интернета вещей также интегрирован в возможности центра безопасности Azure и предоставляет интегрированное представление для мониторинга и управления безопасностью Интернета вещей в рамках общего решения по обеспечению безопасности.

## <a name="how-does-azure-security-center-for-iot-compare-to-the-competition"></a>Как центр безопасности Azure для Интернета вещей сравнивается с конкурентами?

В то время как другие решения предоставляют набор возможностей, позволяющих клиентам создавать собственные решения, центр безопасности Azure для Интернета вещей предоставляет уникальное комплексное решение для обеспечения безопасности IoT, предоставляющее широкие возможности по обеспечению безопасности всех связанных Azure. источников. Azure обеспечивает быстрое развертывание и полную интеграцию с модулем центра Интернета вещей двойников для простоты интеграции с существующими средствами управления устройствами.

## <a name="do-i-have-to-be-an-azure-security-center-customer-to-use-this-service"></a>Нужно ли быть клиентом центра безопасности Azure для использования этой службы?

Нет, но рекомендуется. Без центра безопасности Azure центр безопасности Azure для Интернета вещей получает ограниченные данные ресурсов и обеспечивает ограниченный анализ потенциальных уязвимых областей, угроз и потенциальных атак. 

## <a name="do-i-have-to-be-an-azure-iot-customer"></a>Нужно ли быть клиентом Интернета вещей Azure?

Да. Центр безопасности Azure для Интернета вещей полагается на возможности подключения и инфраструктуры Azure IoT.

## <a name="do-i-have-to-install-an-agent"></a>Нужно ли устанавливать агент?

Установка агента на устройствах IoT не является обязательной для включения центра безопасности Microsoft Azure для Интернета вещей. Вы можете выбрать один из следующих трех вариантов, чтобы получить различные уровни мониторинга безопасности и возможности управления в соответствии с выбранными параметрами.

1. Установите центр безопасности Azure для агента безопасности IoT с изменениями или без них. Этот вариант обеспечивает наивысший уровень усиления безопасности в поведении и доступе к устройствам.

2. Создайте собственный агент и реализуйте Microsoft Azure центре безопасности для схемы сообщений безопасности IoT. Этот параметр включает использование центра безопасности Microsoft Azure для средств анализа IoT на основе агента безопасности устройства.

3. На устройствах IoT не установлена установка агента безопасности. Этот параметр включает наблюдение за связью в центре Интернета вещей с помощью сниженных возможностей мониторинга и управления безопасностью. 

## <a name="what-does-the-azure-security-center-for-iot-agent-do"></a>Что делает центр безопасности Azure для агента IoT?

Центр безопасности Azure для агента IoT обеспечивает покрытие угроз на уровне устройства для настройки, поведения и доступа устройства (путем сканирования конфигурации), обработки & подключения. В центре безопасности Azure для агента безопасности IoT не выполняется проверка данных или действий, связанных с бизнес-операциями.

## <a name="where-can-i-get-the-azure-security-center-for-iot-security-agent"></a>Где можно получить центр безопасности Azure для агента безопасности IoT?

Центр безопасности Azure для агента безопасности IoT — это открытый исходный код, который доступен на сайте GitHub в 32 и 64-разрядных версиях Windows и Linux: https://github.com/Azure/Azure-Security-IoT.

## <a name="where-does-the-azure-security-center-for-iot-agent-get-installed"></a>Где устанавливается центр безопасности Azure для агента IoT? 

Подробные сведения об установке и развертывании агента можно найти в GitHub: https://github.com/Azure/Azure-IoT-Security.

## <a name="what-are-the-dependencies-and-prerequisites-of-the-agent"></a>Каковы зависимости и предварительные условия агента?

Центр безопасности Azure для Интернета вещей поддерживает широкий спектр платформ. См. раздел [Поддерживаемые платформы устройств](how-to-deploy-agent.md) для проверки поддержки конкретных устройств. 

## <a name="which-data-is-collected-by-the-agent"></a>Какие данные собираются агентом?

Подключение, доступ, конфигурация брандмауэра, список процессов & базовую базу ОС собирается агентом.

## <a name="how-much-data-will-the-agent-generate"></a>Какой объем данных будет создан агентом?

Создание данных агента зависит от устройства, приложения, типа подключения и конфигурации агента клиента. Из-за высокой вариативности между устройствами и решениями IoT рекомендуется сначала развернуть агент в лабораторной среде или параметрах тестирования для наблюдения, изучения и настройки конкретной конфигурации, удовлетворяющей вашим потребностям, при измерении объема созданных данных. После запуска службы центр безопасности Azure для агента IoT предоставляет оперативные рекомендации для оптимизации пропускной способности агента, помогающих в настройке и настройке.

## <a name="how-can-i-control-my-billing"></a>Как можно управлять выставлением счетов?

Центр безопасности Azure для Интернета вещей предоставляет настраиваемые средства просмотра агентов, буферы данных и возможность создания пользовательских оповещений, увеличивающих или уменьшающих объем данных, создаваемых агентом.

## <a name="do-agent-messages-use-up-quota-from-iot-hub"></a>Используют ли сообщения агента квоты из центра Интернета вещей?

Да. Передаваемые агентом данные учитываются в квоте центра Интернета вещей. 

## <a name="what-next-ive-installed-an-agent-and-dont-see-any-activities-or-logs"></a>Что делать дальше? Я установил агент и не вижу действий или журналов...

1. Убедитесь, что [Тип агента соответствует назначенной платформе ОС устройства](how-to-deploy-agent.md) .

1. Убедитесь, что [агент работает на устройстве](how-to-agent-configuration.md).

2. Убедитесь, что [Служба успешно включена](quickstart-onboard-iot-hub.md) для **обеспечения безопасности** в центре Интернета вещей. 

3. Убедитесь, что устройство [настроено в центре Интернета вещей с помощью центра безопасности Azure для модуля IOT](quickstart-create-security-twin.md).  

Если действия или журналы по-прежнему недоступны, обратитесь за дополнительной помощью к центру безопасности Azure для партнера IoT.

## <a name="what-happens-when-the-internet-connection-stops-working"></a>Что происходит при прекращении работы подключения к Интернету?

Агент продолжит работу и сохранит данные при условии, что устройство работает. Данные хранятся в кэше сообщений безопасности в соответствии с конфигурацией размера. Когда устройство восстанавливает подключение, сообщения безопасности возобновляют отправку. 

## <a name="if-the-device-is-restarted-will-the-security-agent-self-recover"></a>Если устройство перезагружено, будет ли агент безопасности самостоятельно восстанавливаться?

Агент безопасности предназначен для автоматического повторного запуска при каждом перезапуске устройства.

## <a name="can-the-agent-affect-the-performance-of-the-device-or-other-installed-software"></a>Может ли агент влиять на производительность устройства или другого установленного программного обеспечения?

Агент использует ресурсы компьютера в качестве любого другого приложения или процесса и не должен нарушать нормальную активность устройств. Потребление ресурсов на устройстве, на котором запущен агент, связан с его установкой и конфигурацией. Перед развертыванием в рабочей среде рекомендуется протестировать конфигурацию агента в автономной среде, а также взаимодействовать с другими приложениями и функциями Интернета вещей.

## <a name="im-making-some-maintenance-on-the-device-can-i-turn-off-the-agent"></a>Я делаю некоторое обслуживание на устройстве. Можно ли отключить агент?

Агент не может быть отключен.

## <a name="is-there-a-way-to-test-if-the-agent-is-working-correctly"></a>Существует ли способ проверки правильности работы агента? 

Если агент перестает взаимодействовать или не отправляет сообщения безопасности, создается оповещение о сбое **устройства** .

## <a name="can-i-create-my-own-alerts"></a>Можно ли создавать собственные оповещения?

Да. Можно настроить настраиваемое оповещение по заранее определенному набору поведений, такому как IP-адрес и открытые порты. Дополнительные сведения о пользовательских оповещениях и способах их выполнения см. в разделе [Создание настраиваемых оповещений](quickstart-create-custom-alerts.md) . 

## <a name="where-can-i-see-logs-can-i-customize-logs"></a>Где можно просмотреть журналы? Можно ли настраивать журналы?

- Просмотрите оповещения и рекомендации с помощью подключенной рабочей области Log Analytics. Настройка размера и длительности хранения в рабочей области.

- Необработанные данные агента безопасности также могут храниться в учетной записи Log Analytics. Прежде чем изменять конфигурацию этого параметра, учитывайте размер, длительность, требования к хранению и связанные затраты. 

## <a name="why-should-i-add-azure-security-center-for-iot-to-the-module-identity-what-is-it-used-for"></a>Зачем добавлять центр безопасности Azure для Интернета вещей в удостоверение модуля? Для чего он используется?

Центр безопасности Azure для модуля IoT используется для настройки агента и управления им.

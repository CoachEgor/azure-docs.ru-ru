# <a name="security-recommendations"></a>Рекомендации по безопасности

Центр безопасности Azure для Интернета вещей сканирует ресурсы Azure и устройства IoT и предоставляет рекомендации по безопасности для сокращения зоны атак. Рекомендации по безопасности являются практичными и предназначены для поддержки клиентов в соответствии с рекомендациями по обеспечению безопасности.

В этой статье вы найдете список рекомендаций, которые можно активировать в центре Интернета вещей и на устройствах IoT.

## <a name="recommendations-for-iot-devices"></a>Рекомендации для устройств IoT

Рекомендации по устройствам содержат подробные сведения и рекомендации по улучшению безопасности устройств. 

| severity | Название                                                      | источника данных | Описание                                                                                                                                                                                           |
|----------|-----------------------------------------------------------|-------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Среднее   | Открыть порты на устройстве                                      | Агент       | На устройстве обнаружена конечная точка прослушивания.                                                                                                                                                        |
| Среднее   | В одной из цепочек обнаружена разрешительное политика брандмауэра. | Агент       | Обнаружена разрешенная политика брандмауэра (ВХОДные и выходные данные). Политика брандмауэра должна запрещать весь трафик по умолчанию и определять правила, обеспечивающие необходимое взаимодействие с устройством.                               |
| Среднее   | Обнаружено разрешительное правило брандмауэра во входной цепочке     | Агент       | Обнаружено правило в брандмауэре, содержащее шаблон разрешительного типа для широкого диапазона IP-адресов или портов.                                                                                    |
| Среднее   | Обнаружено разрешительное правило брандмауэра в цепочке выходных данных    | Агент       | Обнаружено правило в брандмауэре, содержащее шаблон разрешительного типа для широкого диапазона IP-адресов или портов.                                                                                   |
| Среднее   | Сбой проверки базовых показателей системы для операции           | Агент       | Устройство не соответствует [результатам тестирования CIS Linux](https://www.cisecurity.org/cis-benchmarks/).                                                                                                        |

### <a name="operational-recommendations-for-iot-devices"></a>Рекомендации по эксплуатации для устройств IoT

Рекомендации по эксплуатации предоставляют аналитические сведения и рекомендации по улучшению конфигурации агента безопасности.

| severity | Имя                                    | источника данных | Описание                                                                       |
|----------|-----------------------------------------|-------------|-----------------------------------------------------------------------------------|
| Низкий      | Агент отправляет неиспользуемые сообщения          | Агент       | 10% или больше сообщений безопасности были меньше 4 КБ за последние 24 часа.  |
| Низкий      | Конфигурация двойника безопасности не оптимальна | Агент       | Конфигурация двойника безопасности не является оптимальной.                                        |
| Низкий      | Конфликт конфигурации двойника безопасности    | Агент       | В конфигурации двойника безопасности обнаружены конфликты. |                          |


## <a name="recommendations-for-iot-hub"></a>Рекомендации для центра Интернета вещей

Оповещения о рекомендациях предоставляют подробные сведения и рекомендации по улучшению безопасности среды.  

| severity | Название                                                     | источника данных | Описание                                                                                                                                                                                                             |
|----------|----------------------------------------------------------|-------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Высокий     | Идентичные учетные данные для проверки подлинности, используемые несколькими устройствами | Центр Интернета вещей     | Учетные данные для проверки подлинности центра Интернета вещей используются несколькими устройствами. Это может означать, что устройство столкновении, выполняющее олицетворение допустимого устройства. Использование дублирующихся учетных данных увеличивает риск олицетворения устройства вредоносным субъектом. |
| Среднее   | Политика фильтрации IP-адресов по умолчанию должна быть запрещена                  | Центр Интернета вещей     | Конфигурация IP-фильтра должна иметь правила, определенные для разрешенного трафика, и по умолчанию запретить весь остальной трафик по умолчанию.                                                                                                     |
| Среднее   | Правило фильтрации IP-адресов включает диапазон больших IP-адресов                   | Центр Интернета вещей     | Диапазон IP-адресов источника правила "разрешить IP-фильтры" слишком велик. Более строгие правила могут предоставлять центр Интернета вещей вредоносным субъектам.                                                                                       |
| Низкий      | Включите журналы диагностики в Центре Интернета вещей                       | Центр Интернета вещей     | Включите журналы и сохраняйте до года. Хранение журналов позволяет воссоздать журнал действий для изучения при возникновении инцидента безопасности или компрометации сети.                                       |
|

# <a name="connect-your-data-from-azure-security-center-for-iot-to-azure-sentinel-preview"></a>Подключение данных из центра безопасности Azure для Azure для Интернета вещей с помощью Sentinel (Предварительная версия)

> [!IMPORTANT]
> Центр безопасности Azure для соединителя данных IoT в Azure в настоящее время находится в общедоступной предварительной версии.
> Эта функция предоставляется без соглашения об уровне обслуживания и не рекомендуется для рабочих нагрузок. Некоторые функции могут не поддерживаться или их возможности могут быть ограничены.
> Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

В этом руководство вы узнаете, как подключить центр безопасности Azure к данным IoT в Azure Sentinel.

> [!div class="checklist"]
> * Предварительные условия
> * Параметры подключения
> * Log Analytics представление предупреждений

Подключение оповещений из центра безопасности Azure для Интернета вещей и их потоковая передача непосредственно в Azure Sentinel.

## <a name="prerequisites"></a>Предварительные условия

- Необходимо иметь разрешения на **Чтение** и **запись** рабочей области.
- **Центр безопасности Azure для Интернета вещей** должен быть **включен** в соответствующих центрах Интернета вещей.
- У вас должны быть разрешения на **Чтение** и **запись** в **центре Интернета вещей Azure** , к которому вы хотите подключиться.
- У вас также должны быть разрешения на **Чтение** и **запись** в **группе ресурсов центра Интернета вещей Azure**.

> [!NOTE]
> Для отправки общих оповещений о ресурсах Azure необходимо, чтобы в вашей подписке была запущена лицензия "Стандартный уровень" центра безопасности Azure. Если требуется лицензирование уровня "бесплатный" для центра безопасности Azure для Интернета вещей, только оповещения центра безопасности Azure для Интернета вещей, связанные с IoT, будут перенаправлены в Azure Sentinel.

## <a name="connect-to-azure-security-center-for-iot"></a>Подключение к центру безопасности Azure для Интернета вещей

1. В поле Sentinel Azure выберите **соединители данных** , а затем щелкните элемент **Центр безопасности Azure для IOT** .
1. В нижней части правой панели щелкните **открыть страницу соединителя**.
1. Нажмите кнопку **подключить**рядом с каждой подпиской центра Интернета вещей, предупреждениями и оповещениями устройств, которые вы хотите передавать в метку Azure.
    - Если центр безопасности Azure для Интернета вещей не включен в этом центре, вы увидите сообщение Enable Warning (включить предупреждение). Щелкните ссылку **включить** , чтобы запустить и включить службу.
1. Вы можете решить, должны ли оповещения в центре безопасности Azure для Интернета вещей автоматически создавать инциденты в Azure Sentinel. В разделе **Создание инцидентов**выберите **включить** , чтобы включить правило автоматического создания инцидентов из созданных оповещений.  Это правило можно изменить или изменить в разделе **аналитика** > **Активные** правила.

> [!NOTE]
>Обновление списка концентраторов после внесения изменений в подключение может занять 10 или более секунд.

## <a name="log-analytics-alert-display"></a>Отображение предупреждений Log Analytics

Чтобы использовать соответствующую схему в Log Analytics, чтобы отобразить центр безопасности Azure для оповещений IoT, сделайте следующее:

1. Откройте **журналы** > **секуритинсигхтс** > **секуритялерт**или выполните поиск по запросу **секуритялерт**.
1. Фильтр для просмотра оповещений, созданных центром безопасности Azure для Интернета вещей с помощью следующего фильтра ККЛ:

```kusto
SecurityAlert | where ProductName == "Azure Security Center for IoT"
```

### <a name="service-notes"></a>Заметки о службе

После подключения центра Интернета вещей данные концентратора доступны в Sentinel в течение приблизительно 15 минут позже.

# <a name="azure-security-center-for-iot-baseline-and-custom-checks"></a>Центр безопасности Azure для базовых и пользовательских проверок IoT

В этой статье объясняется базовый базовый унизел Azure Security Center для IoT и обобщены все связанные с этим свойства базовых пользовательских проверок.

## <a name="baseline"></a>Базовый

Базовый уровень устанавливает стандартное поведение для каждого устройства и упрощает установление необычного поведения или отклонения от ожидаемых норм.  

## <a name="baseline-custom-checks"></a>Базовые пользовательские проверки

Базовые пользовательские проверки устанавливают пользовательский список проверок для каждого базового устройства с помощью **двойника идентификации модуля** устройства. 

## <a name="setting-baseline-properties"></a>Установка базовых свойств

1. В концентраторе IoT найдите и выберите устройство, которое вы хотите изменить.
1. Нажмите на устройство, а затем нажмите на модуль **azureiotsecurity.**
1. Нажмите **Модуль Идентичность Twin**.
1. Загрузите **базовый файл пользовательских проверок** на устройство.
1. Добавьте базовые свойства в модуль безопасности и нажмите **Сохранить**.

### <a name="baseline-custom-check-file-example"></a>Пример файла проверки пользовательского кода базового

Для настройки базовых пользовательских проверок:

   ```json
    "desired": {
      "ms_iotn:urn_azureiot_Security_SecurityAgentConfiguration": {
        "baselineCustomChecksEnabled": {
          "value" : true
        },
        "baselineCustomChecksFilePath": {
          "value" : "/home/user/full_path.xml"
        },
        "baselineCustomChecksFileHash": {
          "value" : "#hashexample!"
        }
      }
    },
   ```

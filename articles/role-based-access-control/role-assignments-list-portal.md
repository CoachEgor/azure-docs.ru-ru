# <a name="list-role-assignments-using-azure-rbac-and-the-azure-portal"></a>Список ролевых заданий с помощью Azure RBAC и портала Azure

[!INCLUDE [Azure RBAC definition list access](../../includes/role-based-access-control-definition-list.md)]В этой статье описывается, как перечислить назначения ролей с помощью портала Azure.

> [!NOTE]
> Если организация передала функции управления на внешний подряд поставщику услуг, который использует [управление делегированными ресурсами Azure,](../lighthouse/concepts/azure-delegated-resource-management.md)назначения ролей, авторизованные этим поставщиком услуг, не будут отображаться здесь.

## <a name="list-role-assignments-for-a-user-or-group"></a>Список ролевых заданий для пользователя или группы

Самый простой способ увидеть роли, назначенные пользователю или группе в подписке, — это использовать панель **ресурсов Azure.**

1. На портале Azure щелкните **все службы,** а затем выберите **пользователей** или **групп.**

1. Нажмите на пользователя или группу, для которого вы хотите перечислить назначения ролей.

1. Щелкните **Ресурсы Azure**.

    Список ролей, назначенных выбранному пользователю или группе, можно увидеть в различных областях, таких как группа управления, подписка, группа ресурсов или ресурс. Этот список включает в себя все назначения ролей, которые у вас есть разрешение на чтение.

    ![Назначения ролей для пользователя](./media/role-assignments-list-portal/azure-resources-user.png)    

1. Чтобы изменить подписку, щелкните список **подписок.**

## <a name="list-owners-of-a-subscription"></a>Список владельцев подписки

Пользователи, которым была назначена роль [Владельца](built-in-roles.md#owner) для подписки, могут управлять всем в подписке. Выполните следующие действия, чтобы перечислить владельцев подписки.

1. На портале Azure щелкните **Все службы**, а затем **Подписки**.

1. Нажмите на подписку, в какую вы хотите перечислить владельцев.

1. Нажмите **на элемент управления доступом (IAM)**.

1. Щелкните вкладку **Назначения ролей**, чтобы просмотреть все назначения ролей для этой подписки.

1. Прокрутите в разделе **Владельцы,** чтобы увидеть всех пользователей, которым была назначена роль Владельца для этой подписки.

   ![Контроль доступа к подписке - Вкладка назначений ролей](./media/role-assignments-list-portal/access-control-role-assignments-subscription.png)

## <a name="list-role-assignments-at-a-scope"></a>Список назначений ролей в области

1. На портале Azure щелкните **все службы,** а затем выберите область охвата. Например, можно выбрать **Группы управления**, **Подписки**, **Группы ресурсов** или конкретный ресурс.

1. Нажмите на конкретный ресурс.

1. Нажмите **на элемент управления доступом (IAM)**.

1. Щелкните вкладку **Назначения ролей**, чтобы просмотреть все назначения ролей в этой области.

   ![Управление доступом — вкладка "Назначение ролей"](./media/role-assignments-list-portal/access-control-role-assignments.png)

   На вкладке "Назначение ролей" вы увидите назначения прав доступа в этой области. Обратите внимание, что одни роли привязаны к **этому ресурсу**, а другие **унаследованы** из другой области. Доступ назначается конкретному ресурсу либо наследуется от назначения в родительской области.

## <a name="list-role-assignments-for-a-user-at-a-scope"></a>Список ролевых заданий для пользователя в области

Чтобы перечислить доступ к пользователю, группе, главному обслуживанию или управляемому удостоверению, вы перечисляете их ролевые назначения. Выполните следующие действия, чтобы перечислить назначения ролей для одного пользователя, группы, директора службы или управляемого удостоверения в определенной области.

1. На портале Azure щелкните **все службы,** а затем выберите область охвата. Например, можно выбрать **Группы управления**, **Подписки**, **Группы ресурсов** или конкретный ресурс.

1. Нажмите на конкретный ресурс.

1. Нажмите **на элемент управления доступом (IAM)**.

1. Щелкните вкладку **Проверить доступ**.

    ![Управление доступом — вкладка "Проверить доступ"](./media/role-assignments-list-portal/access-control-check-access.png)

1. В списке **Найти** выберите тип субъекта безопасности, для которого нужно проверить доступ.

1. В поле поиска введите строку для поиска в каталоге по отображаемым именам, адресам электронной почты или идентификаторам объектов.

    ![Проверка доступа — выбор списка](./media/role-assignments-list-portal/check-access-select.png)

1. Выберите субъект безопасности, чтобы открыть область **Назначения**.

    ![Область назначений](./media/role-assignments-list-portal/check-access-assignments.png)

    В этой области можно увидеть роли, которые назначены выбранному субъекту безопасности для выбранной области. Здесь будут перечислены все запрещающие назначения, которые определены или наследуются в выбранной области.

## <a name="list-role-assignments-for-a-system-assigned-managed-identity"></a>Список назначений ролей для системного управляемого удостоверения

1. На портале Azure откройте управляемую идентификацию, назначенную системой.

1. В левом меню нажмите **Identity**.

    ![Управляемое удостоверение, назначаемое системой](./media/role-assignments-list-portal/identity-system-assigned.png)

1. В **рамках заданий ролей**нажмите **«Показать роли Azure RBAC, назначенные этому управляемому итевии.**

    Список ролей, назначенных выбранной системе, назначенных управляемым удостоверениями в различных областях, таких как группа управления, подписка, группа ресурсов или ресурс. Этот список включает в себя все назначения ролей, которые у вас есть разрешение на чтение.

    ![Назначения ролей для системного управляемого удостоверения](./media/role-assignments-list-portal/azure-resources-system-assigned.png)

## <a name="list-role-assignments-for-a-user-assigned-managed-identity"></a>Список назначений ролей для управляемого изначера, назначенного пользователем

1. На портале Azure откройте управляемое удостоверение, назначенное пользователем.

1. Щелкните **Ресурсы Azure**.

    Список ролей, назначенных выбранному пользователем, управляемым удостоверениями в различных областях, таких как группа управления, подписка, группа ресурсов или ресурс. Этот список включает в себя все назначения ролей, которые у вас есть разрешение на чтение.

    ![Назначения ролей для системного управляемого удостоверения](./media/role-assignments-list-portal/azure-resources-user-assigned.png)

1. Чтобы изменить подписку, щелкните список **подписок.**

## <a name="list-number-of-role-assignments"></a>Список ролевых заданий

В каждой подписке может быть до **2000** ролевых заданий. Чтобы помочь вам отслеживать этот лимит, вкладка **«Назначения ролей»** включает диаграмму, в которой перечислено количество назначений ролей для текущей подписки.

![Контроль доступа - Количество диаграмм ывыполняющих заданий](./media/role-assignments-list-portal/access-control-role-assignments-chart.png)

Если вы приближаетесь к максимальному числу и пытаетесь добавить больше назначений ролей, вы увидите предупреждение в панели **назначения роли Add.** Для способов сокращения числа ролевых заданий можно найти примеры [Troubleshoot Azure RBAC.](troubleshooting.md#azure-role-assignments-limit)

![Элемент управления доступом - Добавление предупреждения о назначении ролей](./media/role-assignments-list-portal/add-role-assignment-warning.png)

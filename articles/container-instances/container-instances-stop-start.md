# <a name="manually-stop-or-start-containers-in-azure-container-instances"></a>Вручную остановить или запустить контейнеры в Azure контейнерных инстанций

Настройка [политики перезагрузки](container-instances-restart-policy.md) группы контейнеров определяет, как экземпляры контейнера начинаются или останавливаются по умолчанию. Вы можете переопределить настройки по умолчанию вручную, остановив или запустив группу контейнеров.

## <a name="stop"></a>Остановить

Вручную остановите бегущей группу контейнеров - например, с помощью команды [остановки контейнера az][az-container-stop] или портала Azure. Для определенных контейнерных нагрузок может потребоваться остановить длительную группу контейнеров после определенного периода, чтобы сэкономить на затратах. 

*Когда группа контейнеров входит в состояние Остановлено, она завершает и перерабатывает все контейнеры в группе. Он не сохраняет состояние контейнера.*

Когда контейнеры перерабатываются, [ресурсы](container-instances-container-groups.md#resource-allocation) распоряжаются и платежные остановки для группы контейнеров.

Действие остановки не действует, если группа контейнеров уже завершена (находится в состоянии успешного или невыполненного). Например, в состоянии успешного выполнения контейнерной группы с задачами запуска емкости, которая успешно завершается в состоянии Успешного. Попытки остановить группу в таком состоянии не меняют состояние. 

## <a name="start"></a>Запуск

Когда группа контейнеров остановлена - либо потому, что контейнеры прекращены сами по себе, либо вы вручную остановили группу - вы можете запустить контейнеры. Например, используйте команду [запуска контейнеров az][az-container-start] или портал Azure для ручного запуска контейнеров в группе. Если образ какого-либо контейнера обновлен, извлекается новый образ. 

Запуск группы контейнеров начинает новое развертывание с той же конфигурацией контейнера. Это действие поможет вам повторно использовать известную конфигурацию группы контейнеров, которая работает необходимым образом. Вам не нужно будет создавать новую группу контейнеров для выполнения той же рабочей нагрузки.

Все контейнеры в группе контейнеров запускаются этим действием. Вы не можете запустить определенный контейнер в группе.

После того как вы вручную запустите или перезапустите группу контейнеров, она будет работать в соответствии с настроенной политикой перезапуска.
  
## <a name="restart"></a>Перезагрузить

Вы можете перезапустить группу контейнеров во время ее работы - например, с помощью команды [перезагрузки контейнера az.][az-container-restart] Это действие перезагружает все контейнеры в группе контейнеров. Если образ какого-либо контейнера обновлен, извлекается новый образ. 

Перезапуск группы контейнеров полезен при устранении проблемы с развертыванием. Например, если временное ограничение ресурса препятствует успешной работе ваших контейнеров, перезапуск группы может решить проблему.

Все контейнеры в группе контейнеров перезагружаются этим действием. Вы не можете перезапустить определенный контейнер в группе.

После ручной перезагрузки группы контейнеров группа контейнеров работает в соответствии с настроенной политикой перезагрузки.

## <a name="next-steps"></a>Дальнейшие действия

# <a name="frequently-asked-questions-about-azure-container-instances"></a>Часто задаваемые вопросы о инстанциях контейнеров Azure

В этой статье часто рассматриваются часто задаваемые вопросы о инстанциях контейнеров Azure.

## <a name="deployment"></a>Развертывание

### <a name="how-large-can-my-container-image-be"></a>Насколько большим может быть изображение контейнера?

Максимальный размер изображения для развертываемого контейнера в экземплярах контейнеров Azure составляет 15 ГБ. Возможно, вы сможете развертывать большие изображения в зависимости от точной доступности на момент развертывания, но это не гарантируется.

Размер изображения контейнера влияет на то, сколько времени требуется для развертывания, поэтому обычно требуется, чтобы изображения контейнера были как можно меньше.

### <a name="how-can-i-speed-up-the-deployment-of-my-container"></a>Как ускорить развертывание контейнера?

Поскольку одним из основных факторов времени развертывания является размер изображения, ищите способы уменьшения размера. Удалите слои, которые вам не нужны, или уменьшите размер слоев на изображении (выбрав более легкое базовое изображение ОС). Например, если вы используете контейнеры Linux, рассмотрите возможность использования Alpine в качестве базового изображения, а не полноценного Ubuntu Server. Аналогичным образом, для контейнеров Windows используйте базовое изображение Nano Server, если это возможно. 

Следует также проверить список предварительно кэшированных изображений в изображениях контейнеров Azure, доступных через API [Кэшированных изображений списка.](/rest/api/container-instances/listcachedimages) Возможно, вы сможете выключить слой изображения для одного из предварительно кэшированных изображений. 

Более [подробные рекомендации](container-instances-troubleshooting.md#container-takes-a-long-time-to-start) по сокращению времени запуска контейнеров.

### <a name="what-net-or-net-core-image-layer-should-i-use-in-my-container"></a>Какой уровень изображения .NET или .NET Core следует использовать в контейнере? 

Используйте наименьшее изображение, которое удовлетворяет вашим требованиям. Для Linux можно использовать изображение *.NET* Core, которое поддерживается с момента выпуска .NET Core 2.1. Для Windows, если вы используете полную рамку .NET, то вам нужно использовать изображение Windows Server Core (изображение только для выполнения, например *4.7.2-windowsservercore-ltsc2016).* Изображения только для выполнения меньше, но не поддерживают рабочие нагрузки, которые требуют .NET SDK.

## <a name="availability-and-quotas"></a>Наличие и квоты

### <a name="how-many-cores-and-memory-should-i-allocate-for-my-containers-or-the-container-group"></a>Сколько ядер и памяти я должен выделить для контейнеров или группы контейнеров?

Это действительно зависит от вашей рабочей нагрузки. Начните с малого и проверьте производительность, чтобы увидеть, как ваши контейнеры делать. [Монитор процессора и использования ресурсов памяти,](container-instances-monitor.md)а затем добавить ядра или память на основе вида процессов, которые вы развертываете в контейнере. 

Убедитесь также, что необходимо проверить [доступность ресурсов](container-instances-region-availability.md#availability---general) для региона, в который вы развертываете для верхних границ ядер процессора и памяти, доступных для одной контейнерной группы. 

### <a name="what-underlying-infrastructure-does-aci-run-on"></a>На какой базовой инфраструктуре работает ACI?

Azure Container Instances стремится быть сервисом без серверных контейнеров по требованию, поэтому мы хотим, чтобы вы были сосредоточены на разработке контейнеров и не беспокоились об инфраструктуре! Для тех, кто любопытны или хочет сделать сравнения по производительности, ACI работает на наборы Azure VMs различных SKUs, в основном из серии F и D. Мы ожидаем, что это изменится в будущем, как мы продолжаем развивать и оптимизировать сервис. 

### <a name="i-want-to-deploy-thousand-of-cores-on-aci---can-i-get-my-quota-increased"></a>Я хочу развернуть тысячи ядер на ACI - могу ли я увеличить свою квоту?
 
Да (иногда). Ознакомьтесь с [статьей о квотах и ограничениях](container-instances-quotas.md) для текущих квот, а также о том, какие ограничения могут быть увеличены по запросу.

### <a name="can-i-deploy-with-more-than-4-cores-and-16-gb-of-ram"></a>Могу ли я развернуть с более чем 4 ядрами и 16 ГБ оперативной памяти?

Пока нет. В настоящее время это максимумы для контейнерной группы. Свяжитесь с службой поддержки Azure с конкретными требованиями или запросами. 

### <a name="when-will-aci-be-in-a-specific-region"></a>Когда ACI будет находиться в определенном регионе?

Текущая доступность региона опубликована [здесь](container-instances-region-availability.md#availability---general). Если у вас есть требование к определенному региону, обратитесь в службу поддержки Azure.

## <a name="features-and-scenarios"></a>Функции и сценарии

### <a name="how-do-i-scale-a-container-group"></a>Как масштабировать группу контейнеров?

В настоящее время масштабирование недоступно для контейнеров или контейнерных групп. Если вам нужно запустить больше экземпляров, используйте наш API для автоматизации и создания дополнительных запросов на создание контейнерной группы для службы. 

### <a name="what-features-are-available-to-instances-running-in-a-custom-vnet"></a>Какие функции доступны для экземпляров, работающих в пользовательском VNet?

Вы можете [развернуть контейнерные группы в виртуальной сети Azure](container-instances-vnet.md) по вашему выбору и делегировать частные I-изображения контейнерным группам для направления трафика в Сети через ресурсы Azure. Развертывание контейнерной группы в виртуальную сеть в настоящее время доступно для производственных нагрузок в подмножестве регионов Azure.

## <a name="pricing"></a>Цены

### <a name="when-does-the-meter-start-running"></a>Когда начинает работать счетчик?

Длительность группы контейнеров рассчитывается с момента, когда мы начинаем вытягивать изображение первого контейнера (для нового развертывания) или перезапуска группы контейнеров (если она уже развернута), до тех пор, пока группа контейнеров не будет остановлена. Подробнее о ценах на [контейнерные инстанции.](https://azure.microsoft.com/pricing/details/container-instances/)

### <a name="do-i-stop-being-charged-when-my-containers-are-stopped"></a>Перестану ли я взимать плату при остановке контейнеров?

Метры перестают работать после того, как вся группа контейнеров остановлена. До тех пор, пока работает контейнер в вашей группе контейнеров, мы храствуем ресурсы в случае, если вы хотите запустить контейнеры снова.

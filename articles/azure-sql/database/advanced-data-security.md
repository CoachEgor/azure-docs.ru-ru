# <a name="advanced-data-security"></a>Расширенная защита данных
[!INCLUDE[appliesto-sqldb-sqlmi-asa](../includes/appliesto-sqldb-sqlmi-asa.md)]


Расширенная защита данных (ADS) — это единый пакет для расширенных возможностей обеспечения безопасности SQL. Реклама доступна для базы данных SQL Azure, Azure SQL Управляемый экземпляр и Azure синапсе Analytics. Он включает в себя функции для обнаружения и классификации конфиденциальных данных, обнаружения и устранения потенциальных уязвимостей базы данных и обнаружения аномальных действий, которые могут указывать на угрозу для вашей базы данных. Эта служба предоставляет единый центр для включения этих возможностей и управления ими.

## <a name="overview"></a>Обзор

ADS предоставляет набор расширенных возможностей безопасности SQL, включая классификацию данных & классификации, оценку уязвимостей SQL и расширенную защиту от угроз.
- [& классификация обнаружения данных](data-discovery-and-classification-overview.md) предоставляет возможности, встроенные в базу данных SQL azure, управляемый экземпляр SQL Azure и Azure синапсе для обнаружения, классификации, добавления меток и создания отчетов о конфиденциальных данных в базах данных. Она может использоваться для просмотра состояния классификации базы данных, а также отслеживания доступа к конфиденциальным данным в базе данных и за ее пределами.
- [Оценка уязвимостей](sql-vulnerability-assessment.md) — это простая в настройке служба, которая может обнаруживать, записывать и помогать устранять потенциальные уязвимости базы данных. Он предоставляет сведения о состоянии безопасности и содержит действия по устранению проблем безопасности и улучшению фортификатионс базы данных.
- [Расширенная защита от угроз](threat-detection-overview.md) позволяет выявить подозрительную активность, указывающую на необычные и потенциально опасные попытки получить доступ к базам данных или воспользоваться ими. Она постоянно отслеживает подозрительные действия в базе данных и обеспечивает немедленное оповещение системы безопасности о потенциальных уязвимостях, атаках путем внедрения кода SQL Azure и аномальных шаблонах доступа к базам данных. Оповещения Расширенной защиты от угроз содержат сведения о подозрительных операциях и рекомендации о том, как исследовать причину угрозы и устранить ее.

Включите расширенную безопасность данных один раз, чтобы включить все эти включенные функции. Одним щелчком можно включить рекламу для всех баз данных на [сервере](logical-servers.md) в Azure или в управляемый экземпляр SQL. Для включения или управления параметрами БАННЕРа требуется принадлежность к роли [диспетчера безопасности SQL](https://docs.microsoft.com/azure/role-based-access-control/built-in-roles#sql-security-manager) или одной из ролей администратора базы данных или сервера.

Цены на рекламу по уровням "Стандартный" центра безопасности Azure, где каждый защищенный сервер или управляемый экземпляр считаются одним узлом. Созданные защищенные ресурсы в Центре безопасности Azure ценовой категории "Стандартный" предоставляются бесплатно в течение пробного периода. Дополнительные сведения см. на странице [цен на центр безопасности Azure](https://azure.microsoft.com/pricing/details/security-center/).

## <a name="getting-started-with-ads"></a>Начало работы с рекламными объявлениями

Ниже указаны шаги по началу работы с ADS.

## <a name="1-enable-ads"></a>1. Включение рекламы

Включите рекламу, перейдя к **расширенной защите данных** под заголовком **Безопасность** сервера или управляемого экземпляра.

> [!NOTE]
> Учетная запись хранения автоматически создается и настраивается для хранения результатов проверки **уязвимостей** . Если вы уже включили рекламу для другого сервера в той же группе ресурсов и регионе, используется существующая учетная запись хранения.

![Включение ADS](./media/advanced-data-security/enable_ads.png)

> [!NOTE]
> Стоимость рекламы соответствует ценам на стандартный уровень центра безопасности Azure на каждый узел, где узлом является весь сервер или управляемый экземпляр. Поэтому вы платите только один раз для защиты всех баз данных на сервере или управляемом экземпляре с рекламными объявлениями. Вы можете для начала воспользоваться бесплатной пробной версией ADS.

## <a name="2-start-classifying-data-tracking-vulnerabilities-and-investigating-threat-alerts"></a>2. приступите к классификации данных, отслеживанию уязвимостей и исследованию оповещений об угрозах

Щелкните карту **Обнаружение и классификация данных**, чтобы просмотреть рекомендуемые для классификации конфиденциальные столбцы и классифицировать данные с помощью постоянных меток конфиденциальности. Щелкните карту **Оценка уязвимостей**, чтобы просмотреть отчеты о поиске уязвимостей, управлять проверками и отслеживать состояние безопасности. Если оповещения системы безопасности получены, щелкните карту **Advanced Threat protection** , чтобы просмотреть подробные сведения о предупреждениях, а также чтобы просмотреть объединенный отчет по всем оповещениям в подписке Azure на странице оповещения безопасности центра безопасности Azure.

## <a name="3-manage-ads-settings"></a>3. Управление параметрами БАННЕРов

Чтобы просмотреть параметры БАННЕРов и управлять ими, перейдите к **расширенной защите данных** под заголовком **Безопасность** для сервера или управляемого экземпляра. На этой странице можно включить или отключить рекламу, а также изменить средства оценки уязвимостей и дополнительные параметры защиты от угроз для всего сервера или управляемого экземпляра.

![Параметры сервера](./media/advanced-data-security/server_settings.png)

## <a name="4-manage-ads-settings-for-a-database"></a>4. Управление параметрами БАННЕРов для базы данных

Чтобы переопределить параметры ADS для конкретной базы данных, установите флажок, позволяющий **включить расширенную защиту данных на уровне базы данных**. Используйте этот параметр только в том случае, если у вас есть определенное требование для получения отдельных оповещений о повышенной защите от угроз или результатов оценки уязвимостей для отдельной базы данных вместо предупреждений и результатов, полученных для всех баз данных на сервере или управляемом экземпляре.

После выбора флажка можно настроить соответствующие параметры для этой базы данных.

![Параметры базы данных и дополнительные средства защиты от угроз](./media/advanced-data-security/database_threat_detection_settings.png)

Дополнительные параметры безопасности данных для сервера или управляемого экземпляра можно также получить из области базы данных ADS. Щелкните **Параметры** в основной области ADS, а затем выберите **Просмотреть параметры сервера Расширенной защиты данных**.

![Параметры базы данных](./media/advanced-data-security/database_settings.png)

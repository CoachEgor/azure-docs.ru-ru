> [!NOTE]
> Клиенты, использующие SQL Server 2019 версии CU8 и выше в Windows Server 2016 и более поздних версий, могут использовать [прослушиватель имени распределенной сети (DNN)](../virtual-machines/windows/availability-group-distributed-network-name-dnn-listener-configure.md) вместо стандартного прослушивателя VNN и Azure Load Balancer. Пропустите остальные инструкции, приведенные в этой статье, по созданию прослушивателя и подсистемы балансировки нагрузки.

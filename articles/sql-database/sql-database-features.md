# <a name="azure-sql-database-features"></a>Возможности Базы данных SQL Azure

Служба баз данных Azure S'L имеет общую кодовую базу с последней стабильной версией сервера S'L. Большинство стандартных языков, обработки запросов и управления базами данных идентичны в базе данных S'L Server и Azure S'L. Функции, которые являются общими в сервере S'L, и все вкусы базы данных Azure S'L:
  - Языковые функции - [Управление ключевыми словами языка потока](https://docs.microsoft.com/sql/t-sql/language-elements/control-of-flow), [Курсоры](https://docs.microsoft.com/sql/t-sql/language-elements/cursors-transact-sql), [Типы данных](https://docs.microsoft.com/sql/t-sql/data-types/data-types-transact-sql), [DML заявления](https://docs.microsoft.com/sql/t-sql/queries/queries), [Предикаты](https://docs.microsoft.com/sql/t-sql/queries/predicates), [Последовательность номера](https://docs.microsoft.com/sql/relational-databases/sequence-numbers/sequence-numbers), [Сохраненные процедуры](https://docs.microsoft.com/sql/relational-databases/stored-procedures/stored-procedures-database-engine), и [переменные](https://docs.microsoft.com/sql/t-sql/language-elements/variables-transact-sql).
  - Особенности базы данных - [Автоматическая настройка (планирование принуждения)](https://docs.microsoft.com/sql/relational-databases/automatic-tuning/automatic-tuning), [Отслеживание изменений,](https://docs.microsoft.com/sql/relational-databases/track-changes/about-change-tracking-sql-server) [сопоставление баз данных](https://docs.microsoft.com/sql/relational-databases/collations/set-or-change-the-database-collation), [Содержащиеся базы данных](https://docs.microsoft.com/sql/relational-databases/databases/contained-databases), [Содержащиеся пользователи](https://docs.microsoft.com/sql/relational-databases/security/contained-database-users-making-your-database-portable), [Сжатие данных](https://docs.microsoft.com/sql/relational-databases/data-compression/data-compression), [настройки конфигурации базы данных](https://docs.microsoft.com/sql/t-sql/statements/alter-database-scoped-configuration-transact-sql), [онлайн-операции индекса,](https://docs.microsoft.com/sql/relational-databases/indexes/perform-index-operations-online) [раздел](https://docs.microsoft.com/sql/relational-databases/partitions/partitioned-tables-and-indexes), и [временные таблицы](https://docs.microsoft.com/sql/relational-databases/tables/temporal-tables) [(см. начало руководства).](sql-database-temporal-tables.md)
  - Функции безопасности - [Роли приложений,](https://docs.microsoft.com/sql/relational-databases/security/authentication-access/application-roles) [Динамическая маскировка данных](https://docs.microsoft.com/sql/relational-databases/security/dynamic-data-masking) [(см. начало руководства),](sql-database-dynamic-data-masking-get-started.md) [Безопасность уровня строки](https://docs.microsoft.com/sql/relational-databases/security/row-level-security)и обнаружение угроз - см. начало руководства для [единой базы данных и эластичных пулов](sql-database-threat-detection.md) и [управляемых экземпляров.](sql-database-managed-instance-threat-detection.md)
  - Мультимодельные возможности - [Обработка графиков,](https://docs.microsoft.com/sql/relational-databases/graphs/sql-graph-overview) [данные JSON](https://docs.microsoft.com/sql/relational-databases/json/json-data-sql-server) [(см. начало руководства),](sql-database-json-features.md) [OPENXML,](https://docs.microsoft.com/sql/t-sql/functions/openxml-transact-sql) [Пространственный,](https://docs.microsoft.com/sql/relational-databases/spatial/spatial-data-sql-server) [OPENJSON](https://docs.microsoft.com/sql/t-sql/functions/openjson-transact-sql)и [индексы XML.](https://docs.microsoft.com/sql/t-sql/statements/create-xml-index-transact-sql)

База данных Azure S'L управляет базами данных и гарантирует их высокую доступность. Некоторые функции, которые могут повлиять на доступность или не могут быть использованы в мире PaaS, имеют ограниченные функциональные возможности в базе данных Azure S'L. Кроме того, некоторые функции базы данных зависят от типа создаваемых баз данных Azure S'L. Эти особенности описаны в таблицах ниже. С помощью Базы данных SQL Azure вы можете создать базу данных в качестве части [управляемого экземпляра](sql-database-managed-instance.md), отдельной базы данных или эластичного пула. Если вам нужны более подробная информация о различиях, вы можете найти их на отдельных страницах для [Единой базы данных и упругих пулов](sql-database-transact-sql-information.md) или [управляемых экземпляров.](sql-database-managed-instance-transact-sql-information.md)

## <a name="migration-methods"></a>Способы миграции

Некоторые методы **являются онлайн** и собирание всех изменений, которые вносятся на источник во время работы миграции, в то время как в **автономных** методах необходимо остановить рабочую нагрузку, которая изменяет данные об источнике во время миграции.

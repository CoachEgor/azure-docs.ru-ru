# <a name="azure-sql-database-traffic-migration-to-newer-gateways"></a>Миграции трафика базы данных SQL Azure на более новые шлюзы

Так как улучшает инфраструктуры Azure, Майкрософт будет регулярно обновлять оборудование, чтобы убедиться, что мы предоставляем максимально возможный результат. В ближайшие месяцы мы планируем добавить что шлюзы построен на новые поколения оборудования и списать шлюзы на старое оборудование в некоторых регионах.  

Клиенты получат уведомление по электронной почте и на портале Azure, задолго до любых изменений шлюзы в каждом регионе. Наиболее актуальные сведения сохраняются в [IP-адреса шлюза базы данных SQL Azure](sql-database-connectivity-architecture.md#azure-sql-database-gateway-ip-addresses) таблицы.

## <a name="impact-of-this-change"></a>Влияние этого изменения

Первого раунда шлюза вывода из эксплуатации 1 сентября 2019 г. планируется в следующих регионах:

- Запад США
- Западная Европа
- East US
- Центральный регион США
- Юго-Восточная Азия
- Центрально-южная часть США
- Северная Европа
- Центрально-северная часть США
- Западная часть Японии
- Восточная часть Японии
- Восток США 2
- Восточная Азия

IP-адрес из эксплуатации перестанут принимать трафик, и все новые попытки подключения будут направляться на один из шлюзов в регионе.

Где вы не увидите влияние этого изменения:

- Клиенты, использующие перенаправления, как их политика подключения не будет видеть какого-либо влияния.
- Подключения к базе данных SQL из внутри Azure и с помощью тегов службы это не повлияет.
- Соединения с использованием поддерживаемых версиях драйвера JDBC для SQL Server будет см. в разделе не влияет. Поддерживаемые версии JDBC, см. в разделе [скачайте Microsoft JDBC Driver для SQL Server](/sql/connect/jdbc/download-microsoft-jdbc-driver-for-sql-server).

## <a name="what-to-do-you-do-if-youre-affected"></a>Что делать, можно делать, если вы являетесь влияет

Рекомендуется разрешить исходящий трафик в IP-адреса для всех [IP-адреса шлюза базы данных SQL Azure](sql-database-connectivity-architecture.md#azure-sql-database-gateway-ip-addresses) в регионе, через TCP-порт 1433 и порт 11000 – 11999 диапазона в устройство брандмауэра. Дополнительные сведения о диапазоны портов, см. в разделе [политику подключения](sql-database-connectivity-architecture.md#connection-policy).

Подключения из приложений с помощью драйвера Microsoft JDBC версии 4.0 ниже может произойти сбой проверки сертификата. Более ранние версии Microsoft JDBC полагаются на обычное имя (CN) в поле субъекта сертификата. В этом случае нужно убедиться, что свойство hostNameInCertificate имеет значение *. database.windows.net. Дополнительные сведения о том, как задать свойство hostNameInCertificate, см. в разделе [соединение с помощью шифрования SSL](/sql/connect/jdbc/connecting-with-ssl-encryption).

Если выше устранение рисков не сработает, файл запрос в службу поддержки для базы данных SQL, используя следующий URL-адрес: https://aka.ms/getazuresupport

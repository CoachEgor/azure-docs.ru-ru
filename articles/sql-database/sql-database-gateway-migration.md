# <a name="azure-sql-database-traffic-migration-to-newer-gateways"></a>Миграция трафика базы данных Лазурного Средства данных S'L на новые шлюзы

По мере совершенствования инфраструктуры Azure корпорация Майкрософт будет периодически обновлять оборудование, чтобы обеспечить наилучшее взаимодействие с клиентами. В ближайшие месяцы мы планируем добавить шлюзы, построенные на новых поколениях оборудования, перенести трафик на них и в конечном итоге вывести из эксплуатации шлюзы, построенные на старом оборудовании в некоторых регионах.  

Клиенты будут уведомлены по электронной почте и на портале Azure заблаговременно до любых изменений в шлюзах, доступных в каждом регионе. Самая актуальная информация будет сохранена в таблице [IP-адресов IP-адресов шлюза базы данных Azure S'L.](sql-database-connectivity-architecture.md#azure-sql-database-gateway-ip-addresses)

## <a name="impact-of-this-change"></a>Влияние этого изменения

Первый раунд миграции транспорта в новые шлюзы запланирован на **14 октября 2019** года в следующих регионах:
- Южная Бразилия
- западная часть США
- Западная Европа
- Восточная часть США
- Центральная часть США
- Юго-Восточная Азия
- Центрально-южная часть США
- Северная Европа
- Центрально-северная часть США
- Западная Япония
- Восточная Япония
- восточная часть США 2
- Восточная Азия

Миграция трафика изменит общедоступный IP-адрес, который DNS разрешает для вашей базы данных S'L.
Вы будете затронуты, если у вас есть:
- Твердый закодированный IP-адрес для любого конкретного шлюза в вашем предварительном брандмауэре
- Любые подсети, использующие Microsoft.S'L в качестве точки обслуживания, но не могут общаться с IP-адресами Шлюза

Вы не будете затронуты, если у вас есть:
- Перенаправление как политика подключения
- Подключение к базе данных S'L изнутри Azure и использование тегов служб
- Соединения, сделанные с использованием поддерживаемых версий Драйвера JDBC для сервера S'L, не будут оказывать никакого влияния. Для поддерживаемых версий JDBC [см. См. Скачать драйвер Microsoft JDBC для сервера S'L.](/sql/connect/jdbc/download-microsoft-jdbc-driver-for-sql-server)

## <a name="what-to-do-you-do-if-youre-affected"></a>Что делать, если вы пострадали

Мы рекомендуем разрешить исходящий трафик на IP-адреса для всех [IP-адресов шлюза базы данных Azure S'L](sql-database-connectivity-architecture.md#azure-sql-database-gateway-ip-addresses) database в регионе на порте TCP 1433 и портового диапазона 11000-11999. Эта рекомендация применима к клиентам, подключенным из помещений, а также тем, кто подключается через Услуги Endpoints. Для получения дополнительной информации [Connection policy](sql-database-connectivity-architecture.md#connection-policy)о диапазонах портов см.

Соединения, сделанные из приложений, использующих Microsoft JDBC Driver ниже версии 4.0, могут привести к отказу в проверке сертификата. Нижние версии Microsoft JDBC опираются на общее имя (CN) в поле предмета сертификата. Смягчение должно обеспечить, чтобы свойство hostNameInCertificate было установлено на database.windows.net. Для получения дополнительной информации о том, как настроить свойство hostNameInCertificate, [см. Подключение с шифрованием](/sql/connect/jdbc/connecting-with-ssl-encryption).

Если приведенное выше смягчение не работает, подайте запрос на поддержку базы данных S'L, используя следующий URL:https://aka.ms/getazuresupport

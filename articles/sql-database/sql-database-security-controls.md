# <a name="security-controls-for-azure-sql-database"></a>Элементы управления безопасностью для базы данных Azure S'L

В этой статье документируются элементы управления безопасностью, встроенные в базу данных Azure S'L.

[!INCLUDE [Security controls Header](../../includes/security-controls-header.md)]

База данных S'L включает в себя как [единую базу данных,](sql-database-single-index.yml) так и [управляемый экземпляр.](sql-database-managed-instance.md) Следующие записи применяются к обоим предложениям, за исключением случаев, когда иное отмечается.

## <a name="network"></a>Сеть

| Контроль безопасности | Да/нет | Примечания |
|---|---|--|
| Поддержка конечных точек обслуживания| Да | Применяется только к [единой базе данных.](sql-database-single-index.yml) |
| Поддержка инъекций виртуальной сети Azure| Да | Применяется только к [управляемому экземпляру.](sql-database-managed-instance.md) |
| Изоляция сети и поддержка брандмауэра| Да | Брандмауэр как на уровне базы данных, так и на уровне сервера. Изоляция сети предназначена только для [управляемого экземпляра.](sql-database-managed-instance.md) |
| Принудительная поддержка туннелирования| Да | [Управляемый экземпляр](sql-database-managed-instance.md) через [ExpressRoute](../expressroute/index.yml) VPN. |

## <a name="monitoring--logging"></a>Мониторинг & лесозаготовки

| Контроль безопасности | Да/нет | Примечания|
|---|---|--|
| Поддержка мониторинга Azure, например, Log Analytics или Application Insights| Да | SecureSphere, решение SIEM от Imperva, также поддерживается через интеграцию [концентратов событий Azure](../event-hubs/index.yml) с [помощью аудита S'L.](sql-database-auditing.md) |
| Управление самолетом и управлением самолетом лесозаготовки и аудит| Да | Да только для некоторых событий |
| Регистрация и аудит данных и аудит данных | Да | Через [аудит СЗЛ](sql-database-auditing.md) |

## <a name="data-protection"></a>Защита данных

| Контроль безопасности | Да/нет | Примечания |
|---|---|--|
| Шифрование на стороне сервера в состоянии покоя: клавиши, управляемые Корпорацией Майкрософт | Да | Называется "шифрование в использовании", как описано в статье [Всегда зашифровано](sql-database-always-encrypted.md). Шифрование на стороне сервера использует [прозрачное шифрование данных.](transparent-data-encryption-azure-sql.md)|
| Шифрование в пути:<ul><li>Шифрование Azure ExpressRoute</li><li>Шифрование в виртуальной сети</li><li>Шифрование между виртуальными сетями</ul>| Да | С использованием HTTPS. |
| Обработка ключей шифрования, например CMK или BYOK| Да | Предлагаются как управляемое обслуживанием, так и управляемое клиентом управление ключевым и вылавливой обработкой. Последний предлагается через [Azure Key Vault](../key-vault/index.yml). |
| Шифрование на уровне столбцов, предоставляемое службами данных Azure| Да | Через [всегда зашифровано](sql-database-always-encrypted.md). |
| Зашифрованные вызовы API| Да | Использование HTTPS/TLS. |

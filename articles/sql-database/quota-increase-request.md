# <a name="request-quota-increases-for-azure-sql-database"></a>Увеличивается квота запросов для базы данных Azure S'L

В этой статье объясняется, как запросить увеличение квот для базы данных Azure S'L для отдельных баз данных, серверов и управляемых экземпляров. В нем также объясняется, как обеспечить доступ к подписке в регионе.

## <a name="create-a-new-support-request"></a><a id="newquota"></a>Создание нового запроса на поддержку

Используйте следующие шаги для создания нового запроса на поддержку с портала Azure для базы данных S'L.

1. В меню [портала Azure](https://portal.azure.com) выберите **поддержку Справки .**

   ![Ссылка на справку и поддержку](./media/quota-increase-request/help-plus-support.png)

1. В **справке и поддержке**выберите **новый запрос поддержки.**

    ![Создание нового запроса на поддержку](./media/quota-increase-request/new-support-request.png)

1. Для поля **Тип проблемы** выберите **Ограничения службы и подписки (квоты)**.

   ![Выберите тип проблемы](./media/quota-increase-request/select-quota-issue-type.png)

1. Для **подписки**выберите подписку, квоту которой вы хотите увеличить.

   ![Выберите подписку для увеличенной квоты](./media/quota-increase-request/select-subscription-support-request.png)

1. Для **типа квоты**выберите один из следующих типов квот:

   - **База данных S'L** для единой базы данных и эластичных квот пула.
   - **Управляемый экземпляр базы данных для** управляемых экземпляров.

   Затем выберите **Следующий: Решения >>**.

   ![Выберите тип квоты](./media/quota-increase-request/select-quota-type.png)

1. В окне **подробностей** выберите **Предоставить подробную информацию** для ввода дополнительной информации.

   ![Ссылка "Предоставить подробную информацию"](./media/quota-increase-request/provide-details-link.png)

Нажатие **На кнопку Подробная** информация Provide отображает окно **детали квоты,** что позволяет добавлять дополнительную информацию. В следующих разделах описывались различные варианты типов квот **данных s'L и** управляемых экземпляров базы **данных.**

## <a name="sql-database-quota-types"></a><a id="sqldbquota"></a>Типы квот базы данных S'L

В следующих разделах описывают три варианта увеличения квот для типов квот **базы данных S'L:**

- Единицы транзакций баз данных (DTUs) на сервер
- Серверы за подписку
- Включить доступ к подписке в регион

### <a name="database-transaction-units-dtus-per-server"></a>Единицы транзакций баз данных (DTUs) на сервер

Используйте следующие шаги, чтобы запросить увеличение DTUs на сервер.

1. Выберите **единицы транзакций базы данных (DTUs) на тип квоты сервера.**

1. В списке **ресурсов** выберите целевой ресурс.

1. В **новом** поле квот введите новый лимит DTU, который вы запрашиваете.

   ![Детали квоты DTU](./media/quota-increase-request/quota-details-dtus.png)

Для получения дополнительной информации см. [Лимиты ресурсов для отдельных баз данных с использованием модели закупок DTU](sql-database-dtu-resource-limits-single-databases.md) и [лимиты ресурсов для эластичных пулов с использованием модели закупок DTU.](sql-database-dtu-resource-limits-elastic-pools.md)

### <a name="servers-per-subscription"></a>Серверы за подписку

Используйте следующие шаги, чтобы запросить увеличение числа серверов на одну подписку.

1. Выберите серверы по типу **квоты подписки.**

1. В списке **местоположений** выберите область Azure для использования. Квота на подписку в каждом регионе.

1. В **поле новых квот** введите запрос на максимальное количество серверов в этом регионе.

   ![Детали квоты серверов](./media/quota-increase-request/quota-details-servers.png)

Для получения дополнительной информации ознакомьтесь с [ограничениями ресурсов базы данных и управлением ресурсами.](sql-database-resource-limits-database-server.md)

### <a name="enable-subscription-access-to-a-region"></a><a id="other"></a>Включить доступ к подписке в регион

Некоторые типы предложений доступны не в каждом регионе. Вы можете увидеть ошибку, такую как:

`This location is not available for subscription`

Если вашей подписке необходим доступ в определенном регионе, используйте опцию **«Другой запрос квот»,** чтобы запросить доступ. В своем запросе укажите информацию о предложении и sKU, которые вы хотите включить для региона. Чтобы изучить варианты предложения [Azure SQL Database pricing](https://azure.microsoft.com/pricing/details/sql-database/single/)и SKU, см.

![Другие детали квоты](./media/quota-increase-request/quota-details-whitelisting.png)

## <a name="managed-instance-quota-type"></a><a id="sqlmiquota"></a>Тип квоты управляемых экземпляров

Для типа квоты управляемых экземпляров **сервера S'L** используйте следующие действия:

1. В списке **регионов** выберите область Azure для целей.

1. Введите новые лимиты, которые вы запрашиваете для **Subnet** и **vCore.**

   ![Детали квоты управляемых экземпляров](./media/quota-increase-request/quota-details-managed-instance.png)

Для получения дополнительной [Overview Azure SQL Database managed instance resource limits](sql-database-managed-instance-resource-limits.md)информации см.

## <a name="submit-your-request"></a>Отправка запроса

Последним шагом является заполнение оставшихся деталей запроса на квоту базы данных S'L. Затем выберите **Следующий: Обзор и создайте>>, **и после рассмотрения сведений о запросе нажмите **Создать,** чтобы отправить запрос.

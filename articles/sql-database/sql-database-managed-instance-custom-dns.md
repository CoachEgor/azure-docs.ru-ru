# <a name="configuring-a-custom-dns-for-azure-sql-database-managed-instance"></a>Настройка пользовательской службы DNS для Управляемого экземпляра Базы данных SQL Azure

Управляемый экземпляр Базы данных SQL Azure должен быть развернут в [виртуальной сети (VNet)](../virtual-network/virtual-networks-overview.md). Существует несколько сценариев (например, Database Mail, связанные серверы с другими экземплярами SQL в облачной или гибридной среде), для которых требуется, чтобы имена частных узлов разрешались из Управляемого экземпляра. В таком случае в Azure необходимо настроить пользовательскую службу DNS. 

Поскольку Управляемый экземпляр использует одну и ту же службу DNS для своих внутренних задач, необходимо настроить пользовательский DNS-сервер, чтобы он мог разрешать общедоступные доменные имена.

   > [!IMPORTANT]
   > Всегда используйте полные доменные имена (FQDN) для почтовых серверов, серверов SQL Server и других служб, даже если они находятся в частной зоне DNS. Например, используйте доменное имя `smtp.contoso.com` для почтового сервера, так как простое имя `smtp` не будет правильно разрешено.

   > [!IMPORTANT]
   > Обновление DNS-серверов виртуальной сети не повлияет на Управляемый экземпляр немедленно. Управляемый экземпляр конфигурация DNS будет обновлена после истечения срока аренды DHCP или после упгараде платформы, что происходит в первую очередь. **Прежде чем создавать первые Управляемый экземпляр, пользователям рекомендуется задать конфигурацию DNS виртуальной сети.**

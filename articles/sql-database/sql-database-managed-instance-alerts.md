# <a name="create-alerts-for-azure-sql-managed-instance-using-the-azure-portal"></a>Создавайте оповещения для управляемых экземпляров Azure S'L с помощью портала Azure

## <a name="overview"></a>Обзор

В этой статье показано, как настроить оповещения для баз данных в базе данных управляемых экземпляров Azure с помощью портала Azure. Оповещения могут отправить вам электронное письмо или вызвать веб-крюк, когда некоторая метрика (например, размер хранилища или использование процессора) достигает порога. В этой статье также приведены рекомендации по настройке периодов оповещений.

Вы можете получать оповещения на основе отслеживания метрик или событий в службах Azure.

* **Значения метрик**. Оповещение активируется, когда значение указанной метрики выходит за рамки заданного порогового значения. То есть сначала оно активируется, когда условие выполняется, а затем — когда условие перестает выполняться.
* **События журнала действий**. Оповещение может активироваться при *каждом* событии или только тогда, когда выполняется определенное число событий.

Для оповещения можно настроить действие, выполняемое при активации оповещения:

* отправка уведомлений по электронной почте администратору службы и соадминистраторам;
* отправка уведомления на указанные дополнительные электронные адреса;
* Вызов webhook.

Для настройки правил генерации оповещений и получении сведений о них можно использовать:

* [Портал Azure](../monitoring-and-diagnostics/insights-alerts-portal.md)
* [PowerShell](../azure-monitor/platform/alerts-classic-portal.md)
* [интерфейс командной строки (CLI)](../azure-monitor/platform/alerts-classic-portal.md)
* [Azure Monitor REST API](https://msdn.microsoft.com/library/azure/dn931945.aspx)

## <a name="create-an-alert-rule-on-a-metric-with-the-azure-portal"></a>Создание правила генерации оповещений на основе метрики с помощью портала Azure

1. На [портале](https://portal.azure.com/)найдите ресурс, который нужно отслеживать, и выберите его.
2. Выберите **оповещения** в разделе Мониторинг. Текст и значок для разных ресурсов могут незначительно отличаться.  

   ![Наблюдение](media/sql-database-insights-alerts-portal/Alerts.png)
  
3. Выберите кнопку **«Новое правило оповещения»,** чтобы открыть страницу **правила «Создание».**
   ![Создание правила](media/sql-database-insights-alerts-portal/create-rule.png)

4. В разделе **Состояние** нажмите **Добавить**.
   ![Определение состояния](media/sql-database-insights-alerts-portal/create-rule.png)
5. На странице **логики настройки сигнала** выберите сигнал.
   ![Выберите сигнал](media/sql-database-insights-alerts-portal/select-signal.png)
6. После выбора сигнала, **например, процент ыt0 CPU,** появляется страница **логики сигнала Настройка.**
   ![Настройка логики сигналов](media/sql-database-insights-alerts-portal/configure-signal-logic.png)
7. На этой странице назначаем этот тип порога, оператора, тип агрегации, пороговое значение, детализацию агрегации и частоту оценки. Затем нажмите **Готово**.
8. В **правиле «Создание»** выберите существующую **группу действий** или создайте новую группу. Группа действий позволяет определить действие, которое должно быть принято при возникновении состояния оповещения.
  ![Определение группы действий](media/sql-database-insights-alerts-portal/action-group.png)

9. Определите имя для правила, предоставьте дополнительное описание, выберите уровень серьезности для правила, выберите, включить ли правило при создании правила, а затем нажмите **«Создать правило оповещения** для создания предупреждения о правилах».

В течение 10 минут оповещение активируется и срабатывает, как описано ранее.

# <a name="delete-subnet-after-deleting-azure-sql-database-managed-instance"></a>Удалить подсеть после удаления базы данных SQL управляемого экземпляра

Эта статья содержит рекомендации о том, как вручную удалить подсеть после удалении последней базы данных SQL управляемого экземпляра, находящихся в нем.

[Виртуальный кластер](sql-database-managed-instance-connectivity-architecture.md#virtual-cluster-connectivity-architecture) , содержал удаленный управляемый экземпляр хранится в течение 12 часов от удаления экземпляра. Виртуальный кластер поддерживаться в рабочем состоянии намеренно, чтобы ускорить создание управляемых экземпляров в той же подсети. В течение этого периода не удается удалить подсеть, связанная с виртуальный кластер.

Немедленное освобождение подсети, используемой с пустой виртуальный кластер можно реализовать с помощью вручную удалены виртуальный кластер. Удаление виртуального кластера может осуществляться через портал Azure или виртуальные кластеры API.

> [!NOTE]
> Виртуальный кластер должен содержать управляемые экземпляры, не для удаления было успешным.

## <a name="delete-virtual-cluster-from-azure-portal"></a>Удаление виртуального кластера с портала Azure

Чтобы удалить виртуальный кластер, с помощью портала Azure, поиск ресурсов виртуальный кластер, используя встроенный поиск.

![Найдите виртуальный кластер.](./media/sql-database-managed-instance-delete-virtual-cluster/virtual-clusters-search.png)

После нахождения виртуальный кластер, который вы хотите удалить, выберите этот ресурс и выберите «Удалить». Вам будет предложено подтвердить удаление виртуальный кластер.

![Удалите виртуальный кластер.](./media/sql-database-managed-instance-delete-virtual-cluster/virtual-clusters-delete.png)

Подтверждение удаления виртуальный кластер предоставляется в уведомления на портале Azure. Виртуальный кластер успешного удаления немедленно освобождает подсети для повторного использования.

## <a name="delete-virtual-cluster-using-api"></a>Удалить виртуальный кластер, с помощью API

Для удаления виртуального кластера посредством использования API параметров URI, указанных в [метод delete виртуальные кластеры](https://docs.microsoft.com/rest/api/sql/virtualclusters/delete).

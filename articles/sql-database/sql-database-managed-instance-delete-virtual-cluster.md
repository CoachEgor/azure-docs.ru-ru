# <a name="delete-a-subnet-after-deleting-an-azure-sql-database-managed-instance"></a>Удаление подсети после удаления управляемого экземпляра базы данных Azure S'L

В этой статье содержатся рекомендации по удалению подсети вручную после удаления последнего экземпляра управляемой базы данных Azure S'L, проживающего в ней.

Управляемые экземпляры развертываются в [виртуальные кластеры.](sql-database-managed-instance-connectivity-architecture.md#virtual-cluster-connectivity-architecture) Каждый виртуальный кластер связан с подсетью. Виртуальный кластер сохраняется по дизайну в течение 12 часов после удаления последнего экземпляра, чтобы позволить вам быстрее создавать управляемые экземпляры в той же подсети. За хранение пустого виртуального кластера плата не взимается. В течение этого периода нельзя удалить подсеть, связанную с виртуальным кластером.

Если вы не хотите ждать 12 часов и предпочитаете удалять виртуальный кластер и его подсеть раньше, вы можете сделать это вручную. Удалите виртуальный кластер вручную, используя портал Azure или API виртуальных кластеров.

> [!IMPORTANT]
> - Виртуальный кластер не должен содержать управляемых экземпляров для успешного удаления. 
> - Удаление виртуального кластера — это длительная операция продолжительностью около 1,5 часов (см. [Управляемые операции управления экземплярами](https://docs.microsoft.com/azure/sql-database/sql-database-managed-instance#managed-instance-management-operations) для современного виртуального времени удаления кластера), в течение которого виртуальный кластер будет по-прежнему виден на портале до завершения этого процесса.

## <a name="delete-virtual-cluster-from-the-azure-portal"></a>Удаление виртуального кластера с портала Azure

Чтобы удалить виртуальный кластер с помощью портала Azure, ищите ресурсы виртуального кластера.

![Скриншот портала Azure с выделенным полем поиска](./media/sql-database-managed-instance-delete-virtual-cluster/virtual-clusters-search.png)

После того, как вы обнаружите виртуальный кластер, который необходимо удалить, выберите этот ресурс и **выберите Удалить**. Вам предлагается подтвердить удаление виртуального кластера.

![Скриншот панели виртуальных кластеров портала Azure с выделенной опцией Delete](./media/sql-database-managed-instance-delete-virtual-cluster/virtual-clusters-delete.png)

Уведомления портала Azure покажут вам подтверждение успешного отправки запроса на удаление виртуального кластера. Сама операция удаления продлится около 1,5 часов, в течение которых виртуальный кластер все еще будет виден на портале. Как только процесс будет завершен, виртуальный кластер больше не будет виден, и связанная с ним подсеть будет выпущена для повторного использования.

> [!TIP]
> Если в виртуальном кластере нет управляемых экземпляров, и вы не можете удалить виртуальный кластер, убедитесь, что у вас нет текущего развертывания экземпляра. Это включает в себя запущенные и отмененные развертывания, которые все еще находятся в процессе выполнения. Это происходит потому, что эти операции будут по-прежнему использовать виртуальный кластер, блокируя его от удаления. Вкладка развертывания ресурсов группы ресурсов, развернутой для указания любых развертываний. В этом случае подождите завершения развертывания, удалите управляемый экземпляр, а затем виртуальный кластер.

## <a name="delete-virtual-cluster-by-using-the-api"></a>Удаление виртуального кластера с помощью API

Чтобы удалить виртуальный кластер через API, используйте параметры URI, указанные в [методе удаления виртуальных кластеров.](https://docs.microsoft.com/rest/api/sql/virtualclusters/delete)

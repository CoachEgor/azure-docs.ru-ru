# <a name="delete-a-subnet-after-deleting-an-azure-sql-database-managed-instance"></a>Удаление подсети после удаления управляемого экземпляра базы данных SQL Azure

В этой статье приводятся рекомендации по удалению подсети вручную после удаления последнего управляемого экземпляра базы данных SQL Azure, находящегося в нем.

Управляемые экземпляры развертываются в [виртуальных кластерах](sql-database-managed-instance-connectivity-architecture.md#virtual-cluster-connectivity-architecture). Каждый виртуальный кластер связан с подсетью. Виртуальный кластер сохраняется в течение 12 часов после последнего удаления экземпляра, чтобы вы могли быстрее создавать управляемые экземпляры в одной подсети. Плата за поддержание пустого виртуального кластера не взимается. В течение этого периода нельзя удалить подсеть, связанную с виртуальным кластером.

Если вы не хотите ждать 12 часов и предпочитаете удалить виртуальный кластер и его подсеть, это можно сделать вручную. Удалите виртуальный кластер вручную с помощью портал Azure или API виртуальных кластеров.

> [!IMPORTANT]
> - Виртуальный кластер не должен содержать управляемых экземпляров, чтобы удаление было успешным. 
> - Удаление виртуального кластера — это длительная операция, выполняемая в течение примерно 1,5 часов (см. раздел [операции управления управляемыми экземплярами](https://docs.microsoft.com/azure/sql-database/sql-database-managed-instance#managed-instance-management-operations) для обновления времени удаления виртуального кластера), в течение которого виртуальный кластер по-прежнему будет отображаться на портале, пока этот процесс завершен.

## <a name="delete-virtual-cluster-from-the-azure-portal"></a>Удаление виртуального кластера из портал Azure

Чтобы удалить виртуальный кластер с помощью портал Azure, выполните поиск ресурсов виртуального кластера.

![Снимок экрана портал Azure с выделенным полем поиска](./media/sql-database-managed-instance-delete-virtual-cluster/virtual-clusters-search.png)

Найдя виртуальный кластер, который необходимо удалить, выберите этот ресурс и нажмите кнопку **Удалить**. Вам будет предложено подтвердить удаление виртуального кластера.

![Снимок экрана панели мониторинга "портал Azure виртуальные кластеры" с выделенным параметром "Удалить"](./media/sql-database-managed-instance-delete-virtual-cluster/virtual-clusters-delete.png)

В портал Azure уведомлениях будет показано подтверждение успешной отправки запроса на удаление виртуального кластера. Операция удаления будет выполнена в течение примерно 1,5 часов, в течение которых виртуальный кластер по-прежнему будет отображаться на портале. После завершения процесса виртуальный кластер станет невидимым, и связанная с ним подсеть будет выпущена для повторного использования.

> [!TIP]
> Если в виртуальном кластере нет управляемых экземпляров и вы не можете удалить виртуальный кластер, убедитесь в отсутствии выполняющегося развертывания экземпляра. Сюда входят запущенные и отмененные развертывания, которые еще выполняются. Это связано с тем, что эти операции будут по-прежнему использовать виртуальный кластер, блокирующий его удаление. Вкладка Просмотр развертываний группы ресурсов, в которой развернут экземпляр, будет указывать на все выполняющиеся развертывания. В этом случае дождитесь завершения развертывания, удалите управляемый экземпляр, а затем виртуальный кластер.

## <a name="delete-virtual-cluster-by-using-the-api"></a>Удаление виртуального кластера с помощью API

Чтобы удалить виртуальный кластер с помощью API, используйте параметры URI, указанные в [методе Delete виртуальных кластеров](https://docs.microsoft.com/rest/api/sql/virtualclusters/delete).

# <a name="set-azure-roles-for-administrative-access-to-azure-cognitive-search"></a>Настройка ролей Azure для административного доступа к Azure Когнитивный поиск

Azure реализует [глобальную модель авторизации на основе ролей](../role-based-access-control/role-assignments-portal.md) для всех служб, управляемых через портал или API Resource Manager. Роли "владелец", "участник" и "читатель" определяют уровень *администрирования служб* для Active Directory пользователей, групп и субъектов безопасности, назначенных каждой роли. 

> [!Note]
> Для защиты содержимого службы не существует управления доступом на основе ролей (RBAC). Вы будете использовать ключ API администратора или ключ API запроса для проверки подлинности запросов к самой службе. Чтобы применить доступ к результатам поиска на основе удостоверений, вы можете создать фильтры безопасности по идентификаторам, удаляя из результатов те документы, к которым запрашивающая сторона не должна иметь доступ. Дополнительные сведения см. в разделе [фильтры безопасности](search-security-trimming-for-azure-search.md).

## <a name="management-tasks-by-role"></a>Задачи управления с использованием разных ролей

Для Когнитивный поиск Azure роли связаны с уровнями разрешений, которые поддерживают следующие задачи управления:

| Роль | Задача |
| --- | --- |
| Владелец |Создание или удаление службы или любого объекта в службе, включая ключи API, индексы, индексаторы, источники данных индексаторов и расписания индексаторов.<p>Просмотр состояния службы, в том числе счетчиков и размера хранилища.<p>Добавление или удаление ролевой принадлежности (только владелец может управлять ролевой принадлежностью).<p>Администраторам подписки и владельцам службы автоматически присваивается роль владельца. |
| Участник | Такой же уровень доступа, как у владельца, за вычетом управления ролями Azure. Например, участник может создавать, удалять или просматривать и повторно создавать ключи [api-keys](search-security-api-keys.md), но не может изменять членство в ролях.<br><br>[Служба поиска участник](../role-based-access-control/built-in-roles.md#search-service-contributor) эквивалентен встроенной роли «универсальный участник». |
| Читатель |Просмотр основных компонентов службы, таких как конечная точка службы, подписка, Группа ресурсов, регион, уровень и емкость. На вкладке мониторинг можно также просмотреть метрики службы, например среднее число запросов в секунду. Члены этой роли не могут просматривать сведения об индексе, индексаторе, источнике данных или наборе навыков. Сюда входят данные об использовании для этих объектов, например количество индексов, существующих в службе. |

Роли не гарантируют права доступа к конечной точке службы. Операции службы поиска, такие как управление индексами, совокупностями индексов и поисковые запросы к данным, управляются посредством ключей API, а не через роли. Дополнительные сведения см. в разделе [Управление ключами API](search-security-api-keys.md).

## <a name="permissions-table"></a>Таблица разрешений

В следующей таблице перечислены операции, разрешенные в Azure Когнитивный поиск, а также ключ, разблокирующий доступ к определенной операции.

Разрешения RBAC применяются к операциям портала и управлению службами (создание, удаление или изменение службы или ее ключей API). Ключи API создаются после существования службы и применяются к операциям содержимого в службе. Кроме того, для операций с содержимым на портале, таких как создание или удаление объектов, владелец или участник RBAC взаимодействуют со службой с помощью неявного ключа API администратора.

| Операция | Управляется |
|-----------|-------------------------|
| Создание службы | Разрешения RBAC: владелец или участник |
| Масштабирование службы | Разрешения RBAC: владелец или участник|
| удаление службы; | Разрешения RBAC: владелец или участник |
| Управление ключами администратора или запроса | Разрешения RBAC: владелец или участник|
| Просмотр сведений о службе на портале или API управления | Разрешения RBAC: владелец, участник или читатель  |
| Просмотр сведений об объекте и метрик на портале или API управления | Разрешения RBAC: владелец или участник |
| Создание, изменение, удаление объектов в службе: <br>Индексы и компоненты (включая определения анализаторов, профили оценки, параметры CORS), индексаторы, источники данных, синонимы, средства подбора | Ключ администратора при использовании API, владельца или участника RBAC при использовании портала |
| Запрос индекса | Администратор или ключ запроса при использовании API, владельца или участника RBAC при использовании портала |
| Запрос системных сведений об объектах, таких как возврат статистики, количество и списки объектов | Ключ администратора при использовании API, владельца или участника RBAC при использовании портала |

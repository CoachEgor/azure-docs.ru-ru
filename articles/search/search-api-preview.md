# <a name="preview-features-in-azure-cognitive-search"></a>Предварительные версии функций в Azure Когнитивный поиск

В этой статье перечислены функции, которые сейчас доступны в предварительной версии. В этом списке удаляются функции, которые переходят из режима предварительной версии в общую доступность. Вы можете проверить [обновления службы](https://azure.microsoft.com/updates/?product=search) или [новые](whats-new.md) возможности для объявлений о доступности в целом.

Хотя некоторые функции предварительной версии могут быть доступны на портале и пакете SDK для .NET, REST API всегда имеет предварительные версии функций. 

+ Для операций поиска Текущая версия API предварительной версии [`2019-05-06-Preview`](https://docs.microsoft.com/rest/api/searchservice/index-2019-05-06-preview)
+ Для операций управления текущая Предварительная версия [`2019-10-01-Preview`](https://docs.microsoft.com/rest/api/searchmanagement/index-2019-10-01-preview)

> [!IMPORTANT]
> Для предварительной версии функции соглашение об уровне обслуживания не предусмотрено. Мы не рекомендуем использовать ее в рабочей среде. Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

## <a name="ai-enrichment-features"></a>Возможности обогащения искусственного интеллекта

Изучите новейшие усовершенствования искусственного интеллекта в [предварительной версии API поиска](https://docs.microsoft.com/rest/api/searchservice/index-2019-05-06-preview).

+ [Пользовательский навык поиска сущности (Предварительная версия)](cognitive-search-skill-custom-entity-lookup.md ) — это нестандартный навык, который ищет текст из пользовательского списка слов и фраз, определяемых пользователем. Используя этот список, он помечает все документы с любыми совпадающими сущностями. Навык также поддерживает степень нечеткого соответствия, которую можно применить для поиска совпадений, которые похожи, но не совсем точны. 

+ [Навык определения PII (Предварительная версия)](cognitive-search-skill-pii-detection.md) — это непонятный навык, используемый во время индексирования, который извлекает личные сведения из входного текста и дает возможность маскировать его из этого текста различными способами.

+ [Добавочное дополнение (Предварительная версия)](cognitive-search-incremental-indexing-conceptual.md) добавляет кэширование в конвейер обогащения, что позволяет повторно использовать существующие выходные данные, если целевые изменения, такие как обновление набора навыков или другого объекта, не изменяют содержимое. Кэширование применяется только к обогащенным документам, созданным набором навыков.

+ [Хранилище знаний (Предварительная версия)](knowledge-store-concept-intro.md) — это новое назначение для конвейера обогащения на основе искусственного интеллекта. Физическая структура данных существует в хранилище BLOB-объектов Azure и хранилище таблиц Azure, а также создается и заполняется при запуске индексатора, имеющего вложенный набор квалификационных навыков. Определение самого хранилища знаний указывается в определении набора навыков. В определении хранилища знаний вы управляете физическими структурами данных с помощью элементов *проекции* , которые определяют способ формирования данных, хранятся ли данные в хранилище таблиц или хранилище BLOB-объектов, и существует ли несколько представлений.

## <a name="indexing-and-query-features"></a>Функции индексирования и запросов

Функции предварительной версии индексатора доступны в API предварительной версии для поиска. 

+ [Cosmos DB индексатор](search-howto-index-cosmosdb.md) поддерживает API MongoDB (Предварительная версия), API Gremlin (Предварительная версия) и API Cassandra (Предварительная версия).

+ [Индексатор Azure Data Lake Storage 2-го поколения (Предварительная версия)](search-howto-index-azure-data-lake-storage.md) может индексировать содержимое и метаданные из Data Lake Storage 2-го поколения.

+ [параметр запроса MoreLikeThis (Предварительная версия)](search-more-like-this.md) находит документы, относящиеся к определенному документу. Эта компонент был доступен в более ранних предварительных версиях. 

## <a name="management-features"></a>Функции управления

+ [Поддержка частных конечных точек](service-create-private-endpoint.md) с помощью [`api-version=2019-10-01-Preview`](https://docs.microsoft.com/rest/api/searchmanagement/index-2019-10-01-preview) REST API управления. Вы можете создать службу с ограничениями на доступ к ее конечной точке.

## <a name="earlier-preview-features"></a>Функции более ранней предварительной версии

Функции, объявленные в более ранних предварительных версиях, если они не перешли на общедоступную версию, по-прежнему доступны в режиме предварительного просмотра. При вызове API с более ранней предварительной версией API вы можете продолжать использовать эту версию или переключиться на `2019-05-06-Preview` без изменений ожидаемого поведения.

## <a name="how-to-call-a-preview-api"></a>Вызов API предварительной версии

Более старые предварительные версии по-прежнему работают, но со временем устаревают. Если код вызывает `api-version=2016-09-01-Preview` или `api-version=2017-11-11-Preview`, эти вызовы остаются действительными. Однако только в последнюю версию предварительного просмотра будут добавлены усовершенствования. 

Следующий пример синтаксиса демонстрирует вызов предварительной версии API.

    GET https://[service name].search.windows.net/indexes/[index name]/docs?search=*&api-version=2019-05-06-Preview

Служба Когнитивный поиск Azure доступна в нескольких версиях. Дополнительные сведения см. в разделе [Версии API](search-api-versions.md).

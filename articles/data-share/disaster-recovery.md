# <a name="disaster-recovery-for-azure-data-share"></a>Восстановление аварийности для совместного использования данных Azure

В этой статье мы посмотрим, как настроить среду аварийного восстановления для совместного использования данных Azure. Перебои в работе центра обработки данных Azure случаются редко, но могут длиться от нескольких минут до нескольких часов. Перебои в работе центра обработки данных могут привести к нарушению работы сред, которые полагаются на данные, которыми делится поставщик данных. Следуя шагам, описанным в этой статье, поставщики данных могут продолжать обмениваться данными со своими потребителями данных в случае сбоя в центре обработки данных для основного региона, где размещается ваш обмен данными. 

## <a name="achieving-business-continuity-for-azure-data-share"></a>Достижение непрерывности бизнеса для совместного использования данных Azure

Чтобы быть готовым к отключению центра обработки данных, поставщик данных может иметь среду обмена данными, подготовленную во вторичном регионе. Есть меры, которые могут быть приняты, которые обеспечат плавное сбой в случае, если происходит отключение центра обработки данных. 

Поставщики данных могут предоставлять вторичные ресурсы обмена данными Azure В дополнительном регионе. Эти ресурсы «Общий доступ к данным» могут быть настроены таким образом, чтобы включить наборы данных, которые существуют в среде совместного использования данных. Потребители данных могут быть добавлены в обмен на обмен данными при настройке среды DR или добавлены в более поздний момент времени (т.е. как часть ручных шагов неудачи).

Если потребители данных имеют активную подписку общего обмена во вторичной среде, подготовленную для целей DR, они могут включить график моментальных снимков как часть сбоя. Если потребители данных не хотят подписываться на вторичный регион для целей DR, они могут быть приглашены в вторичный обмен данными на более поздний момент времени. 

Потребители данных могут либо иметь активную подписку на совместное использование, которая простаивает для целей DR, либо поставщики данных могут добавить их в более поздний момент времени в рамках ручных процедур сбоя.

# <a name="monitor-azure-data-share"></a>Мониторинг в Azure Data Share  

В этой статье объясняется, как можно контролировать обмен данными с помощью Azure Data Share. Как поставщик данных, вы можете контролировать различные аспекты ваших отношений обмена данными. Такие сведения, как, приняли ли ваши потребители данных приглашение на обмен данными, а также о том, создали ли они подписку на обмен и начали использовать ваши данные, доступны для мониторинга. 

Как потребитель данных, вы можете отслеживать моментальные снимки, которые были срабатываны в подписке Azure. 

## <a name="monitor-invitation-status"></a>Мониторинг статуса приглашения

Просмотр состояния приглашений обмена данными, перемещаясь по sent shares - > приглашения. 

![Статус приглашения](./media/invitation-status.png "Статус приглашения") 

Есть три состояния, которые ваше приглашение может быть в:

* Ожидание - Получатель обмена данными еще не принял приглашение.
* Принято - получатель обмена данными принял приглашение.
* Отклонено - получатель доли данных отклонил приглашение.

> [!IMPORTANT]
> Если вы удалили приглашение после того, как оно уже было принято, оно не эквивалентно отзыву доступа. Если вы хотите, чтобы будущие снимки не были скопированы в учетную запись хранения данных потребителей, необходимо отозвать доступ через вкладку *подписки «Общий доступ».* 

## <a name="monitor-share-subscriptions"></a>Мониторинг обмена подписями

Просмотр состояния подписки на акции, перемещаясь по отправленным долям - > подписки Share. Это даст вам подробную информацию об активных подписках, созданных потребителями данных после принятия приглашения. Вы можете остановить будущие обновления для потребителя данных, выбрав подписку на акции и выбрав *отзыв.* 

## <a name="snapshot-history"></a>История моментальных снимков 

В вкладке «История» можно просматривать снимки, которые были скопированы арендатору потребителя данных. Вы можете контролировать частоту и продолжительность каждого интервала снимка. 

![История моментальных снимков](./media/sent-shares.png "История моментальных снимков") 

Более подробную информацию о каждом снимке можно просмотреть, нажав на дату начала запуска. 

История моментального снимка до 30 дней отображается по умолчанию. Если вам нужно увидеть более 30 дней истории, перейдите к мониторингу-> Диагностические настройки и выберите **Добавить диагностические настройки.** Вам необходимо будет выбрать учетную запись для хранения этих журналов. 

![История моментальных снимков](./media/diagnostic-settings.png "Параметры диагностики")

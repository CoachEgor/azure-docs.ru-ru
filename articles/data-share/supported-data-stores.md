# <a name="supported-data-stores-in-azure-data-share"></a>Поддерживаемые хранилища данных в акции данных Azure

Azure Data Share обеспечивает открытый и гибкий обмен данными, включая возможность совместного использования из различных хранилищ данных и в них. Поставщики данных могут обмениваться данными из одного типа хранилища данных, а их потребители данных могут выбирать, в какой хранилище данных получать данные. 

В этой статье вы узнаете о богатом наборе хранилищ данных Azure, которые поддерживаются в акции данных Azure. Вы также можете найти информацию о сочетаниях хранилителей данных, которые могут быть использованы поставщиками данных и потребителями данных.

## <a name="share-from-a-storage-account"></a>Поделиться со счета хранения
Azure Data Share поддерживает совместное использование файлов, папок и файловых систем из Azure Data Lake Gen1 и Azure Data Lake Gen2. Он также поддерживает совместное использование капли, папки и контейнеров из Azure Blob Storage. Только блок капли в настоящее время поддерживается. Когда папки совместно используются в совместном использовании моментальных снимков, потребитель данных может выбрать полную копию данных общего доступа или использовать дополнительные возможности снимка для копирования только новых или обновленных файлов. Существующие файлы с тем же именем будут перезаписаны.

## <a name="share-from-a-sql-based-source"></a>Поделиться из источника, основанного на S'L
Azure Data Share поддерживает совместное использование таблиц или представлений из базы данных Azure S'L и аналитики Azure Synapse (ранее Azure S'L DW). Потребители данных могут принять данные в Azure Data Lake Store Gen2 или Azure Blob Storage в качестве файла csv или паркета. Обратите внимание, что по умолчанию форматы файлов являются csv. Потребитель данных может при желании выбрать для получения данных в формате паркета. Это можно сделать в настройках отображения набора данных при получении данных. 

При приеме данных в Azure Data Lake Store Gen2 или Azure Blob Storage полные снимки перезаписывают содержимое целевого файла. 

Потребитель данных может выбрать для получения данных в таблице по своему выбору. В этом случае, если целевая таблица еще не существует, Azure Data Share создает таблицу S'L с исходной схемой. Если таблица назначения уже существует с тем же именем, она будет удалена и перезаписана с последним полным моментальным снимком. При отображении таблицы назначения можно указать альтернативную схему и название таблицы. Дополнительные снимки в настоящее время не поддерживаются. 

Обмен информацией из источников, основанных на S'L, имеет предварительные требования, связанные с правилами и разрешениями брандмауэра. Пожалуйста, обратитесь к разделу предварительного запроса [в разделе поделиться своим](share-your-data.md) учебником данных для получения подробной информации.

## <a name="share-from-azure-data-explorer"></a>Предоставление общего доступа к данным из Azure Data Explorer
Совместное использование данных Azure поддерживает возможность совместного использования баз данных на местах из кластеров Azure Data Explorer. Поставщик данных может обмениваться на уровне базы данных или кластера. При совместном доступе на уровне базы данных потребитель данных сможет получить доступ только к определенной базе данных, которой является поставщик данных. При совместном доступе на уровне кластеров потребитель данных может получить доступ ко всем базам данных из кластера поставщика, включая любые будущие базы данных, созданные поставщиком данных.

Чтобы получить доступ к общим базам данных, потребителю данных необходимо иметь свой собственный кластер Azure Data Explorer. Кластер Azure Data Explorer для потребителей данных необходимо найти в том же центре обработки данных Azure, что и кластер Azure Data Explorer. При установлении совместного использования azure Data Share создает символическую связь между кластерами azure Data Explorer.

Azure Data Explorer поддерживает два способа приема данных: пакет и потоковое воспроизведение. Данные, полученные из пакета в общей базе данных, будут отображаться от нескольких секунд до нескольких минут на стороне потребителя данных. Данные, полученные от потоковой передачи, могут отображаться на стороне потребителя данных до 24 часов.

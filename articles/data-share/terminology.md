# <a name="azure-data-share-preview-concepts"></a>Основные понятия предварительной версии Azure данных общей папки 

Azure предварительной версии общей папки данных вводятся новые термины, связанные с совместным использованием данных. Здесь объясняются некоторые часто используемые терминов, которые может появиться, используемых в службе. 

## <a name="data-provider"></a>Поставщик данных

Поставщик данных — организация, совместное использование данных с их пользователями. Обычно поставщик данных может быть владельцем или Куратор данных. Поставщики данных должны совместно использовать данные различных типов. Некоторые примеры данных, поставщик данных может потребоваться совместное использование необработанных данных, таких как продажи или данные временных рядов. Поставщик данных может потребоваться совместное использование данных предварительно обработанного, проверенный, уже содержащий функциям аналитики. 

## <a name="data-consumer"></a>Потребитель данных 

Потребитель данных — организация, которая получает данные от поставщика данных. Потребителя данных может хочет присоединиться к общим данным с свои собственные данные для получения полезных сведений. В некоторых случаях данных-получатель может получать данные, которые уже была обработана. 

## <a name="data-share"></a>Совместное использование данных

Совместное использование данных — это группа наборов данных, которые совместно как единая сущность. Наборы данных можно из нескольких источников данных Azure, поддерживаемых совместное использование данных Azure. В настоящее время совместное использование данных Azure поддерживает хранилище BLOB-объектов Azure и Azure Data Lake Store. 

## <a name="share-subscription"></a>Подписки общей папки 

Совместное использование подписки создается в том случае, когда потребитель данных принимает приглашение общего доступа данных от поставщика данных. Поставщики данных можно просмотреть активные моментальные подписки, перейдя по адресу **общих папок, отправленных** в Azure данных совместно используют учетную запись, а затем выбрав **совместного использования подписки**.

Потребитель данных можно проверить, имеют ли они подписка активные моментальные, перейдя по адресу **общих ресурсов, полученных** и просмотреть состояние своих полученных общих ресурсов. 

## <a name="snapshot"></a>Снимок

Моментальный снимок могут создаваться объектом-получателем данных, при принятии данных приглашение общего доступа. При принятии приглашения, они могут активировать полный моментальный снимок данных, им предоставлен доступ. Моментальный снимок — это копия данных на тот момент времени, что потребитель данных создается моментальный снимок. 

Существует два типа моментальных снимков - полные и добавочные. Создается полный снимок содержит все данные в общей папке данных. Добавочный моментальный снимок содержит все данные, которые обновлены или добавлен с момента последнего моментального снимка было активировано. 

## <a name="snapshot-settings-in-azure-data-share"></a>Параметры моментальных снимков в папке данных Azure
 
Поставщик данных можно включить параметр моментальных снимков для общей папки данных. Этот параметр включает потребителей данных получать добавочные обновления по мере их появления. Этот параметр следует использовать, если поставщик данных их потребителей данных, чтобы получать обновления для данных, который предоставлен общий доступ. 

Если поставщик данных включает этот параметр, можно выбрать интервал повторения. Интервал повторения может быть часам или дням. 

Потребитель данных имеет параметр, чтобы согласиться на это расписание моментальных снимков получать добавочные обновления, включающий все данные, изменившиеся с момента их первоначального создания нового моментального снимка. 

## <a name="invitation"></a>Приглашение

Поставщик данных можно пригласить несколько получателей для их совместное использование данных. Их можно сделать, добавив получателей к общей папке данных. Приглашения можно также добавить после создания общей папки данных. 

Поставщик данных можно удалить приглашение, после его отправки. Обратите внимание, что если поставщик данных удаляет приглашение, после его принятия, потребитель данных можно по-прежнему нет подписки активные моментальные. Если поставщик данных удаляет приглашение, он не был принят потребителя данных будет невозможно принять их. 

## <a name="recipient"></a>Recipient

Получатель – человек, который получает приглашение на совместное использование данных. Как правило поставщик данных будет добавление получателей к общей папке, они создают. Когда получатель приглашение примет приглашение, они становятся потребителя данных.

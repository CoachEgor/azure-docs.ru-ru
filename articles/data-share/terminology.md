# <a name="azure-data-share-concepts"></a>Концепции совместного использования данных Azure 

Azure Data Share вводит некоторые новые термины, связанные с обменом данными. В этой статье объясняется некоторые часто используемые термины, которые можно увидеть, используемые во всей службе. 

## <a name="data-provider"></a>Поставщик данных

Поставщик данных — это организация, которая обменивается данными со своими потребителями. Обычно поставщиком данных может быть владелец или куратор данных. Поставщики данных хотят обмениваться данными различных типов. Некоторые примеры данных, которыми может захотеть поделиться поставщиком данных, включают необработанные данные, такие как данные о точках продаж или временных рядах. Поставщик данных может также захотеть поделиться предварительно обработанными, кураторами данными, которые уже содержат аналитику и аналитические данные. 

## <a name="data-consumer"></a>Потребитель данных 

Потребитель данных — это организация, которая получает данные от поставщика данных. Потребитель данных может захотеть присоединиться к общим данным со своими собственными данными для получения информации. В некоторых случаях потребитель данных может получать уже обработанные данные. 

## <a name="data-share"></a>Data Share

Общий доступ к данным — это группа наборов данных, которые совместно используются как единое целое. Наборы данных могут быть получены из ряда источников данных Azure, которые поддерживаются разделом данных Azure. В настоящее время Azure Data Share поддерживает хранилище данных Azure Blob и магазин озер данных Azure. 

## <a name="share-subscription"></a>Поделиться подпиской 

Подписка на обмен данными создается, когда потребитель данных принимает приглашение на обмен данными от поставщика данных. Поставщики данных могут просматривать активные подписки общего обмена, перемещаясь по **отправленной учетной** записи «Поделиться данными» и выбирая **подписки на акции.**

Потребитель данных может проверить, есть ли у него активная подписка на акции, перенаправившись в **полученные акции** и ознакомившись со статусом полученных акций. 

## <a name="snapshot"></a>Моментальный снимок

Снимок может быть создан потребителем данных, когда он принимает приглашение на совместное использование данных. Принимая приглашение, они могут вызвать полный снимок данных, которыми они делятся. Снимок — это копия данных в момент, когда потребитель данных генерировал моментальный снимок. 

Существует два типа снимков - полный и постепенный. Полный снимок содержит все данные в рамках общего обмена данными. Дополнительный снимок содержит все данные, которые были обновлены/добавлены с момента запуска последнего снимка. 

## <a name="snapshot-settings-in-azure-data-share"></a>Настройки моментального снимка в доступе к данным Azure
 
Поставщик данных может включить настройку моментального снимка для обмена данными. Эта настройка позволяет потребителям данных получать дополнительные обновления по мере их возникновения. Эта настройка должна быть включена, если поставщик данных хотел бы, чтобы их потребители данных получали обновления для данных, которые были общими. 

Если поставщик данных позволяет эту настройку, может быть выбран интервал повторения. Интервал повторения может быть ежечасно или ежедневно. 

Потребитель данных имеет возможность зарегистрироваться в этом графике моментального снимка для получения дополнительных обновлений, который включает в себя любые данные, которые изменились с момента первого порожденного нового снимка. 

## <a name="invitation"></a>Приглашение

Поставщик данных может пригласить нескольких получателей в свой общий доступ к данным. Они могут сделать это, добавив получателей в обмен данными. Приглашения также могут быть добавлены после создания совместного использования данных. 

Поставщик данных может удалить приглашение после его отправки, если оно не было принято. Если поставщик данных удаляет приглашение и оно еще не принято, потребитель данных не сможет принять его. 

Приглашения могут быть возмущаться до пяти раз в день. 

## <a name="recipient"></a>Recipient

Получатель — это тот, кто получает приглашение на обмен данными. Как правило, поставщик данных добавляет получателей в создаваемые ими данные. Как только получатель приглашения принимает приглашение, он становится потребителем данных.

# <a name="how-to-configure-a-dataset-mapping-for-a-received-share-in-azure-data-share-preview"></a>Настройка сопоставления набора данных для полученного общего ресурса в предварительной версии общего ресурса Azure Data Share

В этой статье объясняется, как настроить сопоставление набора данных для полученного общего ресурса с помощью предварительной версии общего ресурса Azure Data Share. Это необходимо сделать, если вы приняли приглашение на обмен данными, но выбрали "принять и настроить позже". В противном случае вы можете просто изменить целевую учетную запись хранения для полученных данных. 

## <a name="navigate-to-a-received-data-share"></a>Переход к полученной общей папке данных

В службе общего ресурса данных Azure перейдите к полученному общему ресурсу и перейдите на вкладку **сведения** . 

![](./media/dataset-mapping.png "Сопоставление набора") данных сопоставления 

Установите флажок рядом с набором данных, которому вы хотите назначить назначение, и щелкните **+ Map to Target (+ сопоставить с целевым объектом**). Если вы уже настроили целевую учетную запись хранения и хотите изменить сопоставление на другую учетную запись хранения, вам может потребоваться отменить сопоставление. 

![Соотнести с целевой](./media/dataset-map-target.png "картой с целевым объектом") 

## <a name="select-a-new-storage-account"></a>Выберите новую учетную запись хранения 

Выберите учетную запись хранения, в которой должны находиться данные. Обратите внимание, что все данные, которые уже существуют в ранее сопоставленных учетных записях хранения, не будут автоматически перемещены в новую учетную запись хранения.

Целевое(./media/map-target.png "хранилище") целевой ![учетной записи хранения]

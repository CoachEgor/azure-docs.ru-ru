# <a name="how-to-configure-a-dataset-mapping-for-a-received-share-in-azure-data-share"></a>Настройка сопоставления набора данных для полученного общего ресурса в общей папке данных Azure

В этой статье объясняется, как настроить сопоставление набора данных для полученного общего ресурса с помощью общего ресурса данных Azure. Это необходимо сделать, если вы приняли приглашение на обмен данными, но выбрали "принять и настроить позже". Вы можете настроить сопоставление набора данных, если необходимо изменить назначение для данных, к которым вам предоставлен доступ, или если вы хотите получить данные в SQL Server. 

## <a name="navigate-to-a-received-data-share"></a>Переход к полученной общей папке данных

В службе общего ресурса данных Azure перейдите к полученному общему ресурсу и перейдите на вкладку **сведения** . 

![Сопоставление набора данных](./media/dataset-mapping.png "Сопоставление набора данных") 

Установите флажок рядом с набором данных, которому вы хотите назначить назначение. Нажмите кнопку **отменить** , чтобы отменить сопоставление существующего сопоставления. Выберите **+ соотнести с целевым объектом** , чтобы выбрать новое целевое хранилище. 

![Связать с целевым объектом](./media/dataset-map-target.png "Связать с целевым объектом") 

## <a name="select-a-new-destination-store"></a>Выберите новое целевое хранилище

Выберите целевой тип данных, в котором должны находиться данные. Обратите внимание, что все данные, которые уже существуют в ранее сопоставленных учетных записях хранения, не будут автоматически перемещены в новое место назначения.

![Целевая учетная запись хранения](./media/dataset-map-target-sql.png "Целевое хранилище") 

## <a name="select-a-file-format-sql-sources-only"></a>Выберите формат файла (только для источников SQL)

Если исходные данные находятся в источнике на основе SQL, можно выбрать формат, в котором они получены. 

![Выберите формат](./media/sql-file-formats.png "Форматы файлов SQL")

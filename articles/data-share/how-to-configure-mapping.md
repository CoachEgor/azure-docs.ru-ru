# <a name="how-to-configure-a-dataset-mapping-for-a-received-share-in-azure-data-share-preview"></a>Как настроить сопоставление набора данных для общего ресурса, полученного в предварительной версии совместного использования данных Azure

В этой статье объясняется, как можно настроить сопоставление набора данных для получения общей папки, с помощью предварительной версии Azure данных общей папки. Вы захотите это сделать, если принял приглашение общего ресурса данных, но указан параметр «принять и настроить позже». В противном случае просто можно изменить целевую учетную запись хранения полученных данных. 

## <a name="navigate-to-a-received-data-share"></a>Перейдите в общую папку полученных данных

В службе совместного использования данных Azure, перейдите к папке полученных и выберите **сведения** вкладки. 

![Сопоставление набора данных](./media/dataset-mapping.png "сопоставление набора данных") 

Установите флажок рядом с набором данных вы хотите назначить места назначения для и нажмите кнопку **+ карты к целевому объекту**. Может потребоваться отменить сопоставление первый, если уже была настроена в целевую учетную запись хранения, и вы хотите изменить сопоставление другую учетную запись хранения. 

![Карты к целевому объекту](./media/dataset-map-target.png "карты к целевому объекту") 

## <a name="select-a-new-storage-account"></a>Выберите учетную запись хранения 

Выберите учетную запись хранения, которые получит данные поступали в. Обратите внимание, что любые данные, которые уже существует в любом ранее сопоставить учетные записи хранения не перемещается автоматически новую учетную запись хранения.

![Конечная учетная запись хранения](./media/map-target.png "целевое хранилище") 

## <a name="next-steps"></a>Следующие шаги

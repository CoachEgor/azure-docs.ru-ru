# <a name="roles-and-requirements-for-azure-data-share-preview"></a>Роли и требования для предварительной версии Azure данных общей папки

В этой статье описываются роли, необходимые для обмена данными с помощью предварительной версии Azure данных совместного использования, также принимают и получать данные с помощью предварительной версии совместно использовать данные Azure. 

## <a name="roles-and-requirements"></a>Роли и требования

Для совместного использования или получения данных с помощью совместного использования данных Azure, учетной записи пользователя, используемой для входа Azure должен иметь возможность предоставить разрешения на совместное использование данных в учетную запись хранения, что совместное использование данных из или получение данных в. Обычно это разрешение, которое существует в **владельца** роли или пользовательскую роль с разрешением Microsoft.Authorization/role назначения и записи. 

Чтобы совместно использовать или получать данные в учетную запись хранилища Azure или из него, необходимо быть владельцем учетной записи хранения. Даже если вы создали учетную запись хранения, это не дает автоматически владение учетной записи хранения. Чтобы добавить себя в роли владельца вашей учетной записи хранения Azure, выполните следующие действия.

1. Перейдите к учетной записи хранения на портале Azure
1. Выберите **управление доступом (IAM)**
1. Щелкните **Добавить**.
1. Добавить себя в качестве владельца

Чтобы просмотреть имеющиеся разрешения в подписке, на портале Azure выберите имя пользователя в правом верхнем углу и щелкните **Мои разрешения**. Если у вас есть доступ к нескольким подпискам, выберите соответствующую подписку. 

## <a name="resource-provider-registration"></a>Регистрация поставщика ресурсов 

При получении приглашения совместное использование данных Azure, необходимо вручную зарегистрировать поставщика ресурсов Microsoft.DataShare в свою подписку. Выполните следующие действия, чтобы зарегистрировать поставщик ресурсов Microsoft.DataShare в подписке Azure. 

1. На портале Azure перейдите к **подписок**
1. Выберите подписку, которую вы используете для общей папки данных Azure
1. Щелкните **поставщиков ресурсов**
1. Поиск Microsoft.DataShare
1. Щелкните **Зарегистрировать**.

# <a name="about-expressroute-fastpath"></a>Сведения об ExpressRoute FastPath

Виртуальный сетевой шлюз ExpressRoute предназначен для обмена сетевыми маршрутами и маршрутного сетевого трафика. FastPath предназначен для повышения производительности пути передачи данных между вашей предварительной сетью и виртуальной сетью. При включении FastPath отправляет сетевой трафик непосредственно на виртуальные машины в виртуальной сети, минуя шлюз.

## <a name="requirements"></a>Требования

### <a name="circuits"></a>Цепи

FastPath доступен на всех схемах ExpressRoute.

### <a name="gateways"></a>Шлюзы

FastPath по-прежнему требует создания виртуального сетевого шлюза для обмена маршрутами между виртуальной сетью и сетью. Для получения дополнительной информации о виртуальных сетевых шлюзах и ExpressRoute, включая информацию о производительности и шлюзах SRU, [см.](expressroute-about-virtual-network-gateways.md)

Для настройки FastPath виртуальный сетевой шлюз должен быть либо:

* Ультра производительность
* ErGw3AZ

## <a name="supported-features"></a>Поддерживаемые функции

Хотя FastPath поддерживает большинство конфигураций, он не поддерживает следующие функции:

* UDR в подсети шлюза: Если вы применяете UDR к подсети шлюза вашей виртуальной сети, сетевой трафик из вашей сети будет по-прежнему отправляться в виртуальный сетевой шлюз.

* VNet Peering: Если у вас есть другие виртуальные сети, заглянули с тем, что подключено к ExpressRoute, сетевой трафик из вашей сети в другие виртуальные сети (т.е. так называемые "Spoke" VNets) будет по-прежнему отправляться в виртуальную сеть Шлюза. Решение заключается в подключении всех виртуальных сетей к схеме ExpressRoute напрямую.

* Базовый балансосфер нагрузки: Если вы развернете балансизатор основной внутренней нагрузки в виртуальной сети или развертывание службы Azure PaaS в виртуальной сети, используется баланселер basic internal load, сетевой трафик из вашей внутренней сети в виртуальные i-пор, размещенные на виртуальных i-пордляенах Базовый балансосиватор нагрузки будет отправлен на виртуальный сетевой шлюз. Решение заключается в обновлении балансо-баланса базовой нагрузки до [балансивизатора стандартной нагрузки.](https://docs.microsoft.com/azure/load-balancer/load-balancer-overview)

* Частная ссылка: Если вы подключитесь к [частной конечной точке](../private-link/private-link-overview.md) в виртуальной сети из вашей предварительной сети, соединение будет проходить через виртуальный сетевой шлюз.

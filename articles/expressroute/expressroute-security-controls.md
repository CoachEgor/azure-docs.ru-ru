# <a name="security-controls-for-azure-expressroute"></a>Элементы управления безопасностью для Azure ExpressRoute

В этой статье документируются элементы управления безопасностью, встроенные в Azure ExpressRoute.

[!INCLUDE [Security controls Header](../../includes/security-controls-header.md)]

## <a name="network"></a>Сеть

| Контроль безопасности | Да/нет | Примечания |
|---|---|--|
| Поддержка конечных точек обслуживания| Недоступно |  |
| Поддержка инъекций VNet| Недоступно | |
| Поддержка сетевой изоляции и брандмауэра| Да | Каждый клиент содержится в своем домене для разгрома и туннелируется в своем собственном VNet |
| Принудительная поддержка туннелирования| Недоступно | Через протокол пограничного шлюза (BGP). |

## <a name="monitoring--logging"></a>Мониторинг & лесозаготовки

| Контроль безопасности | Да/нет | Примечания|
|---|---|--|
| Поддержка мониторинга Azure (аналитика журналов, информация о приложениях и т.д.)| Да | Смотрите [мониторинг, метрики и оповещения ExpressRoute.](expressroute-monitoring-metrics-alerts.md)|
| Управление и управление самолетом лесозаготовки и аудит| Да |  |
| Регистрация и аудит плоскости данных| нет |   |

## <a name="identity"></a>идентификации

| Контроль безопасности | Да/нет | Примечания|
|---|---|--|
| Проверка подлинности| Да | Учетная запись службы для шлюза для Microsoft (GWM) (контроллер); Как раз во времени (JIT) доступ для Dev и OP. |
| Авторизация|  Да |Учетная запись службы для шлюза для Microsoft (GWM) (контроллер); Как раз во времени (JIT) доступ для Dev и OP. |

## <a name="data-protection"></a>Защита данных

| Контроль безопасности | Да/нет | Примечания |
|---|---|--|
| Шифрование на стороне сервера в состоянии покоя: клавиши, управляемые Корпорацией Майкрософт |  Недоступно | ExpressRoute не хранит данные о клиентах. |
| Шифрование на стороне сервера в состоянии покоя: ключи, управляемые клиентом (BYOK) | Недоступно |  |
| Шифрование уровня столбцов (Службы данных Azure)| Недоступно | |
| Шифрование в пути (например, шифрование ExpressRoute, шифрование VNet и шифрование VNet-VNet)| нет | |
| Вызовы API в зашифрованном виде| Да | Через [менеджер ресурсов Azure](../azure-resource-manager/index.yml) и HTTPS. |


## <a name="configuration-management"></a>Управление конфигурацией

| Контроль безопасности | Да/нет | Примечания|
|---|---|--|
| Поддержка управления конфигурацией (версия конфигурации и т.д.)| Да | Через поставщика сетевых ресурсов (NRP). |

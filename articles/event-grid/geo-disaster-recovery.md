# <a name="server-side-geo-disaster-recovery-in-azure-event-grid"></a>Аварийного восстановления с георепликацией на стороне сервера в сетке событий Azure
"Сетка событий" теперь имеет автоматическую географическую аварийного восстановления (GeoDR) метаданных не только для новых, но все имеющиеся домены, разделы и подписки на события. Если весь регион Azure выходит из строя, "Сетка событий" уже есть все метаданные вашей инфраструктуры, связанные с событиями, синхронизированные в дополнительный регион. Ваши новые события начинается снова потока без вмешательства вами. 

Аварийное восстановление основана на числах две метрики:

- [Целевая точка восстановления (RPO)](https://en.wikipedia.org/wiki/Disaster_recovery#Recovery_Point_Objective): минут или часов данных, которые могут быть потеряны.
- [Время восстановления (RTO)](https://en.wikipedia.org/wiki/Disaster_recovery#Recovery_time_objective): минуты часа, возможно, служба вниз.

Автоматическая отработка отказа сетки событий имеет разные RPO и RTO для метаданных (подписки на события и т.д.), а также данные (события). Если требуется другой спецификацией из приведенные ниже запросы, можно реализовать собственный [клиентские отработку отказа с помощью раздела интерфейсы API работоспособности](custom-disaster-recovery.md).

## <a name="recovery-point-objective-rpo"></a>Целевая точка восстановления (RPO)
- **Метаданные RPO**: ноль минут. Каждый раз, когда ресурс будет создан в "Сетка событий", мгновенно реплицируются между регионами. При отработке отказа, теряется без метаданных.
- **Данные RPO**: если ваша система работоспособна и испытывает сильную нагрузку на трафик во время региональной отработки отказа, RPO для событий составляет около 5 минут.

## <a name="recovery-time-objective-rto"></a>Целевое время восстановления (RTO)
- **Метаданные RTO**: Хотя обычно это происходит гораздо быстрее, в течение 60 минут "Сетка событий" начнет принимать вызовы создания, обновления и удаления для разделов и подписок.
- **Данные RTO**: Как метаданные это обычно происходит гораздо быстрее, однако в течение 60 минут "Сетка событий" начнет принимать нового трафика после региональной отработки отказа.

> [!NOTE]
> Затраты для метаданных GeoDR на "Сетка событий": 0 долл. США.

# <a name="auth0-partner-topics"></a>Партнерские разделы Auth0
![Логотип Auth0](./media/auth0-overview/auth0-logo.png)

Auth0 — это платформа удостоверений для создателей приложений, которая предоставляет разработчикам и предприятиям необходимые компоненты для защиты их приложений.

В партнерском разделе Auth0 можно использовать создаваемые системой Auth0 события для выполнения ряда задач. Обеспечьте для пользователей удобное взаимодействие с вашей системой после проверки подлинности или автоматизируйте задачи обеспечения безопасности и поддержания инфраструктуры.

Интеграция позволяет выполнять потоковую передачу событий журнала Auth0 с высокой надежностью в Azure, где можно использовать события с избранными ресурсами Azure. Такая интеграция позволяет реагировать на события, получать аналитические сведения, отслеживать проблемы с безопасностью и взаимодействовать с другими мощными конвейерами данных.

В организациях, использующих Auth0 и Azure, эта интеграция позволит легко интегрировать данные в пределах всего стека.

## <a name="use-cases"></a>Варианты использования

### <a name="engage-with-your-users"></a>Взаимодействие с пользователями
Обеспечение эффективного взаимодействия с пользователем — это действенное средство, которое помогает снизить отток клиентов. Создавайте приложения, которые адаптированы к требованиям пользователей, используя события Auth0 с Функциями Azure и Logic Apps. 

### <a name="understand-user-behavior"></a>Понимание особенностей использования приложения пользователями
Определите, когда пользователи используют ваш продукт, а также где они выполняют вход и на каких устройствах. Проанализируйте эту информацию и определите наиболее важные аспекты, связанные с продуктом. Так вы сможете определить, какие браузеры и устройства нужно поддерживать, на какие языки локализовать приложение и в какое время ожидать пикового трафика. 

### <a name="manage-user-data"></a>Управление данными пользователей
Для обеспечения безопасности и соблюдения отраслевых требований необходимо хранить и выполнять аудит действий пользователей. Возможность изменять, удалять и экспортировать данные пользователей становится все более важной для соблюдения законов о конфиденциальности, например Общего регламента по защите данных Европейского Союза (GDPR).

### <a name="secure-your-application"></a>Обеспечение безопасности приложения
Объединение процедур мониторинга безопасности и реагирования на инциденты требуется при защите распределенной системы. По этой причине важно размещать все данные в одном месте и отслеживать весь стек.

# <a name="system-topics-in-azure-event-grid"></a>Разделы о системе в службе "Сетка событий Azure"
Служба "Сетка событий Azure" создает разделы системы при создании первой подписки на события для источника событий Azure. В настоящее время сетка событий не создает системные разделы для источников разделов, созданных до Mar, 15, 2020. Для всех источников разделов, созданных в или после этой даты, сетка событий автоматически создает системные разделы. В этой статье описываются **системные разделы** в службе "Сетка событий Azure".

> [!NOTE]
> В настоящее время эта функция не включена для облака Azure для государственных организаций. 

## <a name="overview"></a>Обзор
При создании первой подписки на события для источника событий Azure, например учетной записи хранения Azure, процесс подготовки для подписки создает дополнительный ресурс типа **Microsoft. EventGrid/системтопикс**. При удалении последней подписки на события в источнике событий Azure системный раздел автоматически удаляется.

Системный раздел неприменим к сценариям с пользовательскими разделами, таким как разделы сетки событий и домены сетки событий. 

## <a name="location"></a>Расположение
Для источников событий Azure, которые находятся в определенном регионе или расположении, системный раздел создается в том же расположении, что и источник событий Azure. Например, если вы создаете подписку на события для хранилища BLOB-объектов Azure в восточной части США, системный раздел создается в восточной части США. Для глобальных источников событий Azure, таких как подписки Azure, группы ресурсов или Azure Maps, служба "Сетка событий" создает системный раздел в **глобальном** расположении. 

## <a name="resource-group"></a>Группа ресурсов 
Как правило, системный раздел создается в той же группе ресурсов, в которой находится источник событий Azure. Для подписок на события, созданных в области подписки Azure, системный раздел создается в группе ресурсов **по умолчанию — EventGrid**. Если группа ресурсов не существует, служба "Сетка событий Azure" создаст ее перед созданием системного раздела. 

При попытке удалить группу ресурсов с помощью учетной записи хранения вы увидите раздел System в списке затронутых ресурсов.

# <a name="system-topics-in-azure-event-grid"></a>Темы системы в Azure Event Grid
Служба Azure Event Grid создает системные темы при создании первой подписки на события для источника событий Azure. В настоящее время Event Grid не создает системные темы для тематических источников, которые были созданы до 15 марта 2020 года. Для всех источников темы, созданных в течение этой даты или после нее, Event Grid автоматически создает системные темы. В этой статье **описаны темы системы** в Azure Event Grid.

> [!NOTE]
> В настоящее время эта функция не включена для облака управления Azure. 

## <a name="overview"></a>Обзор
При создании первой подписки на события для источника событий Azure, такого как учетная запись Azure Storage, процесс подготовки подписки создает дополнительный ресурс типа **Microsoft.EventGrid/systemTopics.** При удалении последней подписки на событие Azure тема системы автоматически удаляется.

Тема системы не применима к пользовательским сценариям тем, т.е. темам Event Grid и доменам Event Grid. 

## <a name="location"></a>Расположение
Для источников событий Azure, которые находятся в определенном регионе/местоположении, тема системы создается в том же месте, что и источник события Azure. Например, при создании подписки на события для хранилища Azure blob в Восточной части США тема системы создается в Восточной части США. Для глобальных источников событий Azure, таких как подписки Azure, группы ресурсов или Карты Azure, Event Grid создает тему системы в **глобальном** местоположении. 

## <a name="resource-group"></a>Группа ресурсов 
В целом тема системы создается в той же группе ресурсов, в которую находится источник событий Azure. Для подписки на события, созданные в области подписки Azure, тема системы создается в группе ресурсов **Default-EventGrid.** Если группа ресурсов не существует, Azure Event Grid создает ее перед созданием темы системы. 

При попытке удалить группу ресурсов с учетной записью хранилища вы увидите тему системы в списке затронутых ресурсов.

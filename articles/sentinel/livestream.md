# <a name="use-hunting-livestream-in-azure-sentinel-to-detect-threats"></a>Используйте охотничий прямой эфир в Azure Sentinel для обнаружения угроз

> [!IMPORTANT]
> Охота в прямом эфире в Azure Sentinel в настоящее время находится в открытом доступе и постепенно развертывается для арендаторов.
> Эта функция предоставляется без соглашения об уровне обслуживания, и она не рекомендуется для производственных нагрузок. Некоторые функции могут не поддерживаться или их возможности могут быть ограничены. Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).


Используйте охотничий живой поток для создания интерактивных сеансов, которые позволят вам проверить вновь созданные запросы по мере возникновения событий, получать уведомления от сеансов, когда найден матч, и при необходимости запускать расследования. Можно быстро создать сеанс в прямом эфире с помощью любого запроса Log Analytics.

- **Тестирование вновь созданных запросов по мере возникновения событий**
    
    Вы можете тестировать и корректировать запросы без каких-либо противоречий с текущими правилами, которые активно применяются к событиям. После подтверждения работы этих новых запросов, как и ожидалось, легко продвигать их к правилам пользовательского оповещения, выбрав опцию, которая повышает сеанс до оповещения.

- **Получайте уведомления о возникновении угроз**
    
    Можно сравнить каналы данных угроз с агрегированными данными журнала и уведомлять о совпадении. Каналы данных об угрозах — это постоянные потоки данных, связанные с потенциальными или текущими угрозами, поэтому уведомление может указывать на потенциальную угрозу для организации. Создайте сеанс в прямом эфире вместо правила пользовательского оповещения, когда вы хотите получать уведомления о потенциальной проблеме без накладных расходов на поддержание правила пользовательского оповещения.

- **Запуск расследований**
    
    При активном расследовании, в рамках которого имеется такой актив, как унист или пользователь, можно просмотреть конкретную (или любую) активность в данных журнала, как это происходит на этом активе. Вы можете быть уведомлены, когда это действие происходит.


## <a name="create-a-livestream-session"></a>Создание сеанса в прямом эфире

Можно создать сеанс в прямом эфире из существующего охотничьего запроса или создать сеанс с нуля.

1. На портале Azure перейдите к**управлению** >  **sentinel** > Threat**Hunting**.

2. Для создания сеанса в прямом эфире из охотничьего запроса:
    
    1. Из **вкладки Запросы** найдите запрос на охоту для использования.
    2. Нажмите правой кнопкой мыши запроса и выберите **Добавить в прямом эфире.** Пример:
    
    > [!div class="mx-imgBorder"]
    > ![создать сеанс Livestream из охотничьего запроса Azure Sentinel](./media/livestream/livestream-from-query.png)

3. Чтобы создать сеанс в прямом эфире с нуля: 
    
    1. Выберите вкладку **Livestream**
    2. Выберите **Перейти в прямом эфире.**
    
4. На панели **Livestream:**
    
    - Если вы начали прямую трансляцию с запроса, просмотрите запрос и внедрите любые изменения, которые вы хотите внести.
    - Если вы начали прямую трансляцию с нуля, создайте запрос. 

5. Выберите **play** из панели команд.
    
    Панель статуса в панели команд указывает, работает ли сеанс livestream или приостанавливается. В следующем примере сеанс выполняется:
    
    > [!div class="mx-imgBorder"]
    > ![создать сеанс в прямом эфире с охоты Azure Sentinel](./media/livestream/livestream-session.png)

6. Выберите **Сохранить** из панели команд.
    
    Если вы не выберете **Паузу,** сеанс продолжается до тех пор, пока вы не выйдете из портала Azure.

## <a name="view-your-livestream-sessions"></a>Просмотр сеансов в прямом эфире

1. На портале Azure перейдите на вкладку **Sentinel** > **Threat** > **Hunting** > **Livestream.**

2. Выберите сеанс в прямом эфире, который вы хотите просмотреть или отменить. Пример:
    
    > [!div class="mx-imgBorder"]
    > ![создать сеанс прямого эфира из охотничьего запроса Azure Sentinel](./media/livestream/livestream-tab.png)
    
    Выбранный сеанс в прямом эфире открывается для воспроизведения, паузы, отработки и так далее.

## <a name="receive-notifications-when-new-events-occur"></a>Получение уведомлений при возникновении новых событий

Поскольку уведомления в прямом эфире для новых событий используют уведомления портала Azure, эти уведомления можно увидеть при использовании портала Azure. Пример:

![Уведомление портала Azure для прямого эфира](./media/livestream/notification.png)

Выберите уведомление, чтобы открыть панель **Livestream.**
 
## <a name="elevate-a-livestream-session-to-an-alert"></a>Повысить сеанс в прямом эфире до оповещения

Вы можете продвинуть сеанс в прямом эфире к новому оповещению, выбрав **Elevate для оповещения** из панели команд на соответствующей сессии livestream:

> [!div class="mx-imgBorder"]
> ![Повысить сеанс в прямом эфире до оповещения](./media/livestream/elevate-to-alert.png)

Это действие открывает мастер создания правил, который заселен запросом, связанным с сеансом livestream.

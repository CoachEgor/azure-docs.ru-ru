# <a name="connect-your-beyond-security-besecure-to-azure-sentinel"></a>Подключение к Azure Sentinel с защитой безопасности

> [!IMPORTANT]
> В настоящее время в Azure Sentinel используется общедоступная Предварительная версия соединителя ЗАЩИЩЕНных данных безопасности. Эта функция предоставляется без соглашения об уровне обслуживания. Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

Помимо безопасного соединителя безопасности, позволяет легко подключать все журналы решений безопасности с помощью метки Azure, просматривать панели мониторинга, создавать пользовательские оповещения и улучшать исследование. Интеграция между безопасностью и Sentinel Azure позволяет использовать REST API.

> [!NOTE]
> Данные будут храниться в географическом расположении рабочей области, на которой вы используете метку Azure.

## <a name="configure-and-connect-besecure"></a>Настройка и подключение в безопасном месте

БЕЗОПАСНОЕ взаимодействие может интегрироваться с и экспортировать журналы непосредственно в Azure Sentinel.

1. На портале Sentinel Azure щелкните **соединители данных** и выберите пункт **больше безопасности защищенный (Предварительная версия)** , а затем **откройте страницу соединителя** .

1. Выполните следующие действия, чтобы настроить безопасное решение для отправки результатов проверки, проверки состояния и журналов аудита в Azure Sentinel.

    **Доступ к меню интеграции:**
    1. Щелкните пункт "Дополнительно".

    1. Выбор сервера

    1. Выбор интеграции

    1. Включение Azure Sentinel 

    **Обеспечьте безопасность с помощью параметров Sentinel Azure.**
      - Скопируйте *идентификатор рабочей области* и значения *первичного ключа* со страницы соединителя Azure Sentinel, вставьте их в конфигурацию «незащищенный» и нажмите кнопку **изменить** .

## <a name="find-your-data"></a>Поиск данных

После установки успешного подключения данные отображаются в **журналах** в разделе **CustomLogs** в одной или нескольких следующих таблицах:
  - `beSECURE_ScanResults_CL`
  - `beSECURE_ScanEvents_CL`
  - `beSECURE_Audit_CL`

Чтобы запросить безопасный журнал в Log Analytics, введите одно из приведенных выше имен таблиц в верхней части окна запроса.

## <a name="validate-connectivity"></a>Проверка подключения
После того, как журналы начнут отображаться в Log Analytics, может пройти до 20 минут.

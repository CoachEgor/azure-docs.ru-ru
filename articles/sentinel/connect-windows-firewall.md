# <a name="connect-windows-firewall"></a>Подключение брандмауэра Windows



Разъем брандмауэра Windows позволяет легко подключать журналы брандмауэров Windows, если они подключены к рабочему пространству Azure Sentinel. Это соединение позволяет просматривать панели мониторинга, создавать пользовательские оповещения и улучшать исследование. Это дает вам более глубокое представление о сети вашей организации и улучшает возможности работы службы безопасности. Решение собирает события брандмауэра Windows из компьютеров Windows, на которых установлен агент Log Analytics. 


> [!NOTE]
> - Данные будут храниться в географическом местоположении рабочего пространства, на котором работает Azure Sentinel.
> - Если Центр безопасности Azure Sentinel и Azure собран ы в одно и то же рабочее пространство, нет необходимости включать решение Windows Firewall через этот разъем. Если вы включили его в любом случае, это не вызовет дублированных данных. 

## <a name="enable-the-connector"></a>Включить разъем 

1. На портале Azure Sentinel выберите **разъемы данных,** а затем нажмите на плитку **брандмауэра Windows.** 
1.  Если компьютеры Windows находятся в Azure:
    1. Нажмите **Установка агента на Azure Windows виртуальной машины**.
    1. В списке **виртуальных машин** выберите машину Windows, которую вы хотите передать в Azure Sentinel. Убедитесь, что это Windows VM.
    1. В окне, которое открывается для этого VM, нажмите **Connect**.  
    1. Нажмите **Включить** в окне **разъема разъема брандмауэра Windows.** 

2. Если ваша машина Windows не является VM Azure:
    1. Нажмите **На ссылку на агент Установки на машинах, не относясь от Azure.**
    1. В окне **прямого агента** выберите либо загрузить агента **Windows (64 бит)** или **загрузить агента Windows (32 бит)**.
    1. Установите агент на компьютер Windows. Копируйте **идентификатор рабочего пространства,** **основной ключ**и **вторичный ключ** и используйте их при установке.

4. Выберите типы данных, которые вы хотите поток.
5. Нажмите **Установить решение**.
6. Чтобы использовать соответствующую схему в журнале Analytics для брандмауэра Windows, выйдите в **SecurityEvent.**

## <a name="validate-connectivity"></a>Проверка подключения

Это может занять более 20 минут, пока ваши журналы начинают появляться в журнале Analytics.

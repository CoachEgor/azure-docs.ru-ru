# <a name="keep-track-of-data-during-hunting"></a>Хранить список данных во время поиск работы

> [!IMPORTANT]
> Сейчас Azure Sentinel предоставляется в общедоступной предварительной версии.
> Эта предварительная версия предоставляется без соглашения об уровне обслуживания и не рекомендована для использования рабочей среде. Некоторые функции могут не поддерживаться или их возможности могут быть ограничены. Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).
 
Поиск угроз обычно требует проверки горы данных журнала, ищете свидетельства вредоносного поведения. В ходе этого процесса исследователей найти события, которые они необходимо запомнить, вернитесь и проанализировать как часть проверки потенциальных гипотез и понимания статью полностью нарушения.
С помощью закладок поможет при покупке можно это сделать, с сохранением запросы выполнялись в Log Analytics, вместе с результатами запроса, для достижения соответствующих. Можно также записать наблюдаемой контекстные и ссылаться на результаты, добавление заметок и теги. Закладкой данных является видимым для вы и ваши коллеги для совместную работу.   

Вернитесь к отмеченному закладкой данных в любое время на **закладки** вкладке **поможет при покупке** страницы. Можно использовать фильтрацию и параметры для быстрого поиска нужных данных для анализа текущего поиска. Кроме того, можно просмотреть закладками данных непосредственно в **HuntingBookmark** таблицы в Log Analytics. Это позволяет фильтровать, обобщения и Присоединяйтесь к отмеченному закладкой данных с другими источниками данных, что упрощает поиск corroborating свидетельства.

Вы также можете визуализировать данные закладкой, щелкнув **исследовать**. Это откроет возможности анализа, в котором можно просматривать, анализа и визуального представления результатов с помощью интерактивных графа сущностей схемы и временная шкала.

1. Чтобы выполнить поиск кампании, выберите один поиск работы запросов и на левой, просмотрите результаты.

## <a name="view-and-update-bookmarks"></a>Просмотрите и обновите закладки 

1. На портале Azure Sentinel щелкните **поможет при покупке**. 
2. Нажмите кнопку **закладки** tab в середине страницы, чтобы просмотреть список закладок.
3. Используйте поле или фильтр функции поиска для поиска определенной закладке.
4. Выберите отдельные закладки в таблице ниже для получения сведений закладки в области сведений правой руки.
5. Чтобы обновить теги и заметки, щелкните на полях ввода и редактирования текста и нажмите кнопку **Сохранить** для сохранения изменений.

   ![Показать результаты](./media/bookmarks/view-update-bookmarks.png)

## <a name="view-bookmarked-data-in-log-analytics"></a>С закладками Просмотр данных в Log Analytics 

Существует несколько вариантов, чтобы просмотреть данные закладками в Log Analytics. 

— Самый простой способ просмотра закладками запросы, результаты или журнала, выбрав требуемую закладку в **закладки** таблице и использовать ссылки, указанные в области сведений. Возможны следующие значения. 
- Щелкните **запрос представления** для просмотра в исходном запросе в Log Analytics.  
- Щелкните **просмотреть журнал закладки** для просмотра все закладки, в том числе метаданные: кто вносил обновления, обновленные значения и время обновления. 

- Можно также просматривать данные необработанных закладку для всех закладок, щелкнув **закладки журналы** над сеткой закладки. Этом представлении будут показаны все ваши закладки в таблице поможет при покупке закладки с соответствующими метаданными. KQL запросов можно использовать для фильтрации до последней версии определенная закладка, которую вы ищете.  


> [!NOTE]
> Может существовать значительная задержка (измеряется в минутах) между Создание закладки и при отображении в **HuntingBookmark** таблицы. Рекомендуется сначала создать закладок, а затем проанализировать их после данные принимаются. 

## <a name="delete-a-bookmark"></a>Удалить закладку
Если вы хотите удалить закладку, выполните следующее: 
1.  Откройте th **закладки поможет при покупке** вкладки. 
2.  Выберите закладку целевой объект.
3.  Нажмите кнопку с многоточием (...) в конце строки и выберите **удалить закладку**.
    
Удаление закладки удаляет закладку из списка в **закладки** вкладки.  Log Analytics «HuntingBookmark» таблица по-прежнему использоваться предыдущей закладки, но приведет к изменению последней записи **SoftDelete** значение true, что позволяет легко отфильтровать старых закладок.  При удалении закладки не удаляет все сущности из возможности анализа, которые связаны с другими закладки или предупреждения.

# <a name="connect-your-symantec-icdx-appliance"></a>Подключение устройства Symantec ICDx 

> [!IMPORTANT]
> Сейчас Azure Sentinel предоставляется в общедоступной предварительной версии.
> Эта предварительная версия предоставляется без соглашения об уровне обслуживания и не рекомендована для использования рабочей среде. Некоторые функции могут не поддерживаться или их возможности могут быть ограничены. Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

Соединитель Symantec ICDx позволяет легко подключить все журналы решений безопасности Symantec с вашей Azure Sentinel, для просмотра панелей мониторинга, создать настраиваемые оповещения и улучшит исследование. Это дает больше возможностей регулировать сети вашей организации и улучшает возможности безопасности операции. Интеграция между Symantec ICDx и Azure Sentinel использует REST API.


> [!NOTE]
> Данные будут храниться в географическое расположение рабочей области, на котором выполняется Azure Sentinel.

## <a name="configure-and-connect-symantec-icdx"></a>Настройка и подключение Symantec ICDx 

Symantec ICDx можно интегрировать и экспортировать журналы непосредственно в Azure Sentinel.

1. Откройте консоль управления ICDx для добавления серверов пересылки, Microsoft Azure Sentinel (Log Analytics).
2. На панели навигации ICDx щелкните **конфигурации**. 
3. В верхней части **конфигурации** щелкните **серверов пересылки**.
4. В разделе **серверов пересылки**, рядом с Microsoft Azure Sentinel (Log Analytics), щелкните **добавить**. 
4. В **Microsoft Azure Sentinel (Log Analytics)** окно, нажмите кнопку **Показать дополнительные**. 
5. В верхней части развернутое окно Microsoft Azure Sentinel (Log Analytics), сделайте следующее:
    -   **Имя.** Введите имя для данного сервера пересылки, который имеет не более 30 символов. Выберите уникальное и информативное имя. Это имя отображается в списке серверов пересылки на **конфигурации** экрана и на панелях мониторинга на **панели мониторинга** экрана. Пример: Microsoft Azure Log Analytics вправо. Это поле обязательно к заполнению.
    -   **Описание** Введите описание для данного сервера пересылки. Это описание также отображается в списке серверов пересылки на **конфигурации** экрана. Включают сведения, такие как перенаправление типа события и группы, которым нужен для проверки данных.
    -   **Тип запуска**: Выберите метод запуска для конфигурации сервера пересылки. Имеются следующие варианты: ручная и автоматическая.<br>Значение по умолчанию — "автоматически". 
6. В разделе **события**, выполните следующие действия: 
    - **Источник**. Выберите один или несколько архивов, из которого нужно перенаправить события. Можно выбрать active сборщика архивов (включая общие архив), связь с учетной записью сборщика архивов (т. е. архивы для сборщиков, которые вы удалили), ICDx получателя архивы или архива системы. <br>По умолчанию используется общий архив.
      > [!NOTE]
      > Архивы ICDx получателя, перечислены отдельно, по имени. 
 
    - **Фильтр**. Добавьте фильтр, который определяет подмножество событий для переадресации. Выполните одно из следующих действий.
        - Чтобы выбрать условие фильтра, щелкните тип, атрибут, оператор и значение. 
        - В поле фильтра просмотрите условия фильтра. Можно изменить его непосредственно в поле или удалите его при необходимости.
        - Нажмите кнопку и или или добавить к условиям фильтра.
        - Можно также щелкнуть сохраненные запросы, чтобы применить сохраненный запрос.
    - **Включенные атрибуты**: Введите разделенный запятыми список атрибутов для включения в перенаправленных данные. Включенные атрибуты имеют приоритет над исключенных атрибутах.
    - **Исключенные атрибуты**: Введите разделенный запятыми список атрибутов для исключения из пересылаемых данных.
    - **Размер пакета**: Выберите количество событий для отправки в пакете. Вам доступны 10, 50, 100, 500 и 1000.<br>Количество по умолчанию — 100. 
    - **Ограничение скорости**: Выберите скорость, при которой события переадресуются, выраженное как событий в секунду. Имеются следующие варианты: без ограничений, 500 и 1000, 5000, 10000. <br> Значение по умолчанию — 5000. 
7. В разделе **назначение Azure**, выполните следующие действия: 
    - **Идентификатор рабочей области**: Вставьте идентификатор рабочей области из приведенных ниже. Это поле обязательно к заполнению.
    - **Первичный ключ**: Вставьте скопированный первичный ключ из приведенных ниже. Это поле обязательно к заполнению.
    - **Имя пользовательского журнала**: Введите имя пользовательского журнала в Microsoft Azure портала рабочей области Log Analytics к которому нужно перенаправить события. По умолчанию используется SymantecICDx. Это поле обязательно к заполнению.
8. Нажмите кнопку *Сохранить* чтобы завершить настройку сервера пересылки. 
9. Чтобы запустить сервер пересылки, в разделе **параметры**, нажмите кнопку **дополнительные** и затем **запустить**.
10. Для соответствующей схемы в Log Analytics можно использовать для событий Symantec ICDx, поиск **SymantecICDx_CL**.


## <a name="validate-connectivity"></a>Проверка подключения

Может потребоваться до 20 минут, пока не журналов в Log Analytics.

# <a name="connect-your-symantec-icdx-appliance"></a>Подключите прибор Symantec ICDx 



Разъем Symantec ICDx позволяет легко подключить все журналы решений безопасности Symantec с помощью Azure Sentinel, просматривать панели мониторинга, создавать пользовательские оповещения и улучшать исследования. Это дает вам более глубокое представление о сети вашей организации и улучшает возможности работы службы безопасности. Интеграция между Symantec ICDx и Azure Sentinel использует REST API.


> [!NOTE]
> Данные будут храниться в географическом местоположении рабочего пространства, на котором работает Azure Sentinel.

## <a name="configure-and-connect-symantec-icdx"></a>Налаживание и подключение Symantec ICDx 

Symantec ICDx может интегрировать и экспортировать журналы непосредственно в Azure Sentinel.

1. Откройте консоль управления ICDx для добавления экспедиторов Microsoft Azure Sentinel (Log Analytics).
2. На панели навигации ICDx нажмите **Конфигурация**. 
3. В верхней части экрана **конфигурации** нажмите **На передние .**
4. Под **forwarders**, рядом с Microsoft Azure Sentinel (Log Analytics), нажмите **Добавить**. 
4. В окне **Microsoft Azure Sentinel (Log Analytics)** нажмите **Show Advanced**. 
5. В верхней части расширенного окна Microsoft Azure Sentinel (Log Analytics) сделайте следующее:
    -   **Имя**: Введите имя для форварда, который имеет не более 30 символов. Выберите уникальное, значимое имя. Это имя отображается в списке форвардов на экране **конфигурации** и в панелях мониторинга на экране **панели мониторинга.** Например: Аналитика журналов Microsoft Azure На востоке. Это поле обязательно к заполнению.
    -   **Описание**: Введите описание для форварда. Это описание также отображается в списке форвардов на экране **конфигурации.** Включите такие сведения, как переадресованный тип события и группа, которая должна проверить данные.
    -   **Тип запуска**: Выберите метод запуска для конфигурации форварда. Ваши опции являются ручными и автоматическими.<br>По умолчанию автоматический. 
6. В **рамках событий,** сделать следующее: 
    - **Источник**: Выберите один или несколько архивов, из которых можно пересылать события. Вы можете выбрать архивы активных коллекторов (включая Общий архив), архивы осиротевших коллекционеров (т.е. архивы для коллекционеров, которые вы удалили), архивы приемников ICDx или Системный архив. <br>По умолчанию является Общий архив.
      > [!NOTE]
      > Архивы приемника ICDx перечислены отдельно, по имени. 
 
    - **Фильтр**: Добавить фильтр, который определяет подмножество событий для переадресов. Выполните одно из следующих действий.
        - Чтобы выбрать условие фильтра, нажмите «Тип», «Атрибут», «Оператор» и «Значение». 
        - В поле фильтра просмотрите состояние фильтра. Вы можете отсвазать его непосредственно в поле или удалить по мере необходимости.
        - Нажмите и или или или, чтобы добавить к вашему состоянию фильтра.
        - Вы также можете нажать Сохраненные запросы, чтобы применить сохраненный запрос.
    - **Включены атрибуты**: Введите список атрибутов, делимитированных запятой, чтобы включить их в переадресованные данные. Включенные атрибуты имеют приоритет над исключенными атрибутами.
    - **Исключенные атрибуты**: Введите список атрибутов, делимитированных запятой, чтобы исключить из переадресованных данных.
    - **Размер пакета**: Выберите количество событий для отправки на каждую партию. Ваши варианты 10, 50, 100, 500 и 1000.<br>Значение по умолчанию — 100. 
    - **Ограничение скорости**: Выберите скорость, по которой перенаправляются события, выраженные как события в секунду. Ваши варианты Неограниченные, 500, 1000, 5000, 10000. <br> По умолчанию 5000. 
7. В **azure Destination**сделайте следующее: 
    - **Идентификатор рабочего пространства:** вставьте идентификатор рабочего пространства снизу. Это поле обязательно к заполнению.
    - **Первичный ключ**: Вставьте основной ключ снизу. Это поле обязательно к заполнению.
    - **Пользовательское имя журнала**: Введите пользовательское имя журнала в рабочей области журнала Microsoft Azure Log Analytics, на которую вы собираетесь пересылать события. По умолчанию symantecICDx. Это поле обязательно к заполнению.
8. Нажмите *Сохранить,* чтобы закончить конфигурацию форварда. 
9. Для начала нападающий, под **опционами,** нажмите **Больше,** а затем **начать**.
10. Чтобы использовать соответствующую схему в журнале Analytics для событий Symantec ICDx, ищите **SymantecICDx_CL**.

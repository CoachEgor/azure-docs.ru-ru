# <a name="tutorial-visualize-and-monitor-your-data"></a>Учебник: Визуализация и мониторинг данных



После [подключения источников](quickstart-onboard.md) данных к Azure Sentinel можно визуализировать и отслеживать данные с помощью данных Azure Sentinel, использующих рабочие книги Azure Monitor, которые обеспечивают универсальность в создании пользовательских панелей мониторинга. Хотя трудовые книжки отображаются по-разному в Azure Sentinel, возможно, вам будет полезно увидеть, как [создавать интерактивные отчеты с помощью учебников Azure Monitor.](../azure-monitor/app/usage-workbooks.md) Azure Sentinel позволяет создавать пользовательские книги для данных, а также встроенные шаблоны книг, позволяющие быстро получать ценные сведения по данным сразу после подключения к источнику данных.


Этот учебник поможет вам визуализировать данные в Azure Sentinel.
> [!div class="checklist"]
> * Использование встроенных книг
> * Создание новых книг

## <a name="prerequisites"></a>Предварительные требования

- Вы должны иметь по крайней мере разрешения для чтения в «Workbook» или разрешения для участников Workbook в группе ресурсов рабочего пространства Azure Sentinel.

> [!NOTE]
> Рабочие книги, которые можно увидеть в Azure Sentinel, сохраняются в группе ресурсов рабочего пространства Azure Sentinel и помечены рабочим пространством, в котором они были созданы.

## <a name="use-built-in-workbooks"></a>Использование встроенных книг

1. Перейдите в **трудовые книжки,** а затем выберите **шаблоны,** чтобы увидеть полный список встроенных рабочих книг Azure Sentinel. Чтобы узнать, какие из них имеют отношение к типам данных, которые вы подключили, в поле **требуемых типов данных** в каждой рабочей книге будет указан тип данных рядом с зеленой контрольной отметкой, если вы уже передаете релевантные данные в Azure Sentinel.
  ![перейти к трудовым книжки](./media/tutorial-monitor-data/access-workbooks.png)
1. Нажмите **Посмотреть рабочую книгу,** чтобы увидеть шаблон, населенный с вашими данными.
  
1. Чтобы отсеивать трудовую книжку, выберите **Сохранить,** а затем выберите место, где вы хотите сохранить файл json для шаблона. 

   > [!NOTE]
   > Это создает ресурс Azure на основе соответствующего шаблона и сохраняет сам шаблон Json, а не данные.


1. Выберите **посмотреть рабочую книгу**. Затем нажмите кнопку **«Оторит»** вверху. Теперь вы можете отойти от работы и настроить ее в соответствии с вашими потребностями. Для получения дополнительной информации о том, как [Create interactive reports with Azure Monitor Workbooks](../azure-monitor/app/usage-workbooks.md)настроить учебник, см.
![просмотр трудовых книжек](./media/tutorial-monitor-data/workbook-graph.png)
1. После внесения изменений можно сохранить трудовую книжку. 

1. Вы также можете клонировать трудовую книжку: Выберите **edit** и после этого **Сохранить как,** убеждаясь сохранить его с другим именем, под такой же подпиской и группой ресурсов. Эти трудовые книжки отображаются под вкладкой **"Мои трудовые книжки".**


## <a name="create-new-workbook"></a>Создание новой трудовой книжки

1. Перейти к **трудовым книжки,** а затем выбрать **Добавить трудовую книжку,** чтобы создать новую книгу с нуля.
  ![перейти к трудовым книжки](./media/tutorial-monitor-data/create-workbook.png)

1. Чтобы отсеять трудовую книжку, выберите **edit,** а затем добавьте текст, запросы и параметры по мере необходимости. Для получения дополнительной информации о том, как [Create interactive reports with Azure Monitor Workbooks](../azure-monitor/app/usage-workbooks.md)настроить учебник, см. 

1. При создании запроса установите, что **источник данных** устанавливается в **журналы,** **тип ресурса** устанавливается в **журнал Analytics,** а затем выбирает соответствующее рабочее пространство (ы). 

1. После создания трудовой книжки сохраните рабочую книгу, чтобы сохранить ее под подпиской и группой ресурсов рабочего пространства Azure Sentinel.

1. Если вы хотите, чтобы другие в вашей организации использовали рабочую книгу, под **сохранением выберите** **общие отчеты.** Если вы хотите, чтобы эта трудовая книга была доступна только вам, выберите **Мои отчеты**.

1. Чтобы переключаться между трудовыми книжками в](./media/tutorial-monitor-data/switch.png)рабочей области, вы можете выбрать **рабочие** ![книги Open Switch в верхней панели любой рабочей книги. На окне, которое открывается вправо, переключаться между трудовыми книжеками.

   ![Переключить трудовые книжки](./media/tutorial-monitor-data/switch-workbooks.png)


## <a name="how-to-delete-workbooks"></a>Удаление книг

Можно удалить рабочие книги, созданные из шаблона Azure Sentinel. 

Чтобы удалить настраиваемую трудовую книжку на странице Трудовых книг, выберите сохраненную трудовую книгу, которую вы хотите удалить, и выберите **Удалить**. Это позволит удалить сохраненную трудовую книжку.

> [!NOTE]
> Это удаляет ресурс, а также любые изменения, внесенные в шаблон. Оригинальный шаблон останется доступным.

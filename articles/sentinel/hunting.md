# <a name="hunt-for-threats-with-in-azure-sentinel-preview"></a>Слежения для угроз в предварительной версии Sentinel Azure

> [!IMPORTANT]
> Сейчас Azure Sentinel предоставляется в общедоступной предварительной версии.
> Эта предварительная версия предоставляется без соглашения об уровне обслуживания и не рекомендована для использования рабочей среде. Некоторые функции могут не поддерживаться или их возможности могут быть ограничены. Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

Если вы человек, кто желает упреждением выполнять поиск угроз безопасности, Azure Sentinel мощные охота средства поиска и запросов для слежения наличие угроз безопасности в источниках данных вашей организации. Однако системы и устройства безопасности создавать горы данных, который может быть трудно проанализировать и фильтрации в значимые события. Для безопасности аналитики найдите заранее новые аномалии, которые не были обнаружены при безопасности приложений, Azure Sentinel "поможет при покупке встроенных запросов помогут в правильной формулировки вопросов поиск ошибок в данных, у вас уже есть в вашей сети. 

Например, один встроенный запрос предоставляет данные о наиболее часто процессов, работающих в инфраструктуре — стоит оповещение о каждом запуске, они могут быть полностью безобидно, но взгляните на запрос, в некоторых случаях, чтобы увидеть, если может потребоваться th где и что-то необычное. 



С помощью Azure Sentinel поиск, можно воспользоваться преимуществами следующие возможности:

- Встроенных запросов: Чтобы приступить к работе, начальная страница содержит примеры предварительно загруженному запросу позволяет приступить к работе и узнать о таблицах и язык запросов. Эти запросы встроенный поиск работы разработаны Майкрософт исследователей систем безопасности на постоянной основе, добавление новых запросов, и точной существующие запросы, чтобы предоставить вам искать новые обнаружения и выяснить, с которого следует начать поиск работы для точки входа об основах новых атак. 

- Эффективный язык запросов с использованием IntelliSense: Построена на базе язык запросов, который обеспечивает гибкость, которые необходимо выполнить поиск на следующий уровень.

- Создайте собственные закладки: Во время процесса поможет при покупке вы можете столкнуться совпадений или результаты, панели мониторинга и действия, которые выглядят необычные или подозрительные. Чтобы пометить эти элементы, поэтому вы сможете вернуться к ним в будущем, используйте функцию закладок. Закладки позволяют сохранять элементы для использования в будущем, использовать для создания оболочка для расследования. Дополнительные сведения о закладках см. в разделе Использование [закладки в поиск работы].

- Использование записных книжек для автоматизации расследования: Записные книжки похожи на пошаговые сборники тренировочных заданий, которые можно выполнить пошаговое руководство по исследования и слежения.  Записные книжки инкапсулируют все этапы поможет при покупке многократно используемых сборника, который можно использовать совместно с другими пользователями в вашей организации. 
- Запрос сохраненных данных: Данные можно получить в таблицах для запроса. Например можно запросить создание процесса, события DNS-сервера и многих других типов событий.

- Содержит ссылки на сообщества. Воспользуйтесь преимуществами более сообщества для поиска дополнительных запросов и источники данных.
 
## <a name="get-started-hunting"></a>Начало работы охота

1. На портале Azure Sentinel щелкните **поможет при покупке**.
  ![Azure Sentinel запускает поиск работы](media/tutorial-hunting/hunting-start.png)

2. При открытии **поможет при покупке** страницы, все запросы поможет при покупке отображаются в одной таблице. В таблице перечислены все запросы, созданного группой разработчиков корпорации Майкрософт, аналитиков в сфере безопасности, а также любой дополнительный запрос, созданные или измененные. Каждый запрос содержит описание его проверок для, и какие данные он выполняется. Эти шаблоны группируются по их различные приемы - значки справа классификации типа угрозы, такие как первоначальный доступ, сохраняемости и утечки данных. Вы можете отфильтровать эти шаблоны запросов поможет при покупке, с помощью любого из поля. Любой запрос можно сохранить в Избранное. Сохранив запрос в Избранное, запрос автоматически выполняется при каждом **поможет при покупке** обращении к странице. Можно создавать собственные запросы поможет при покупке или клонирования и настройка существующего шаблона запроса поможет при покупке. 
 
2. Нажмите кнопку **выполните запрос** поможет при покупке сведения о запросе страницы для любых запросов без выхода со страницы поможет при покупке.  Количество совпадений, которое отображается в таблице. Просмотрите список поможет при покупке запросов и их совпадения. Ознакомьтесь с какой этап в цепочке продвинутой, соответствие связан с.

3. Выполнить краткий обзор того, базовый запрос в области сведений запрос или нажмите кнопку **просмотра результата запроса** открыть запрос в Log Analytics. В нижней и просмотрите совпадения для запроса.

4.  Щелкните строку и выберите **добавить закладку** для добавления строк изучения - это можно сделать для любого элемента, подозрительные. 

5. Затем вернитесь к главному **поможет при покупке** и нажмите кнопку **закладки** вкладку, чтобы увидеть все подозрительные действия. 

6. Выберите закладку и нажмите кнопку **исследовать** открыть возможности анализа. Вы можете отфильтровать закладки. Например если вы изучаете кампании, можно создать тег для кампании, а затем отфильтровать все закладки, в зависимости от кампании.

1. После обнаружения, какой запрос поможет при покупке обеспечивает высокое значение анализ возможных атак, также можно создать определения правил на основе вашего запроса и аналитически предупреждений о вашей ответчики инцидентов безопасности.

## <a name="useful-operators"></a>Полезные операторы

Язык запросов, обладает широкими возможностями и имеет много доступных операторов, здесь перечислены некоторые полезные операторы:

**где** — отфильтровать таблицу для подмножества строк, которые удовлетворяют предикату.

**суммировать** -сформировать таблицу, которая объединяет содержимое входной таблицы.

**соединения** -слияние строк двух таблиц для формирования новой таблицы, сопоставляя значения указанные столбцы из каждой таблицы.

**число** -возвращают количество записей во входном наборе записей.

**верхней** -Return первые N записей, отсортированных по указанным столбцам.

**ограничение** -возвращать не больше указанного числа строк.

**проект** — выберите столбцы для включения, переименования или удаления и вставьте новые вычисляемые столбцы.

**расширить** — Создание вычисляемых столбцов и добавляет их к результирующему набору.

**makeset** -возвращать динамический массив данных набор уникальных значений, который принимает Expr (JSON) в группе

**найти** -поиск строк, соответствующих предикату в набор таблиц.

## <a name="save-a-query"></a>Сохранение запроса

Можно создать или изменить запрос и сохраните его как собственный запрос или поделиться им с пользователями, которые находятся в одном клиенте.

   ![Сохранение запроса](./media/tutorial-hunting/save-query.png)

Создайте новый запрос поможет при покупке:

1. Нажмите кнопку **новый запрос** и выберите **Сохранить**.
2. Заполните все пустые поля и выберите **Сохранить**.

   ![Создать запрос](./media/tutorial-hunting/new-query.png)

Клонируйте и изменить существующий запрос поможет при покупке:

1. Выберите запрос, поможет при покупке в таблицу, которую требуется изменить.
2. Нажмите кнопку с многоточием (...) в строке запроса, нужно изменить и выберите **клонировать запрос**.

   ![клонировать запрос](./media/tutorial-hunting/clone-query.png)
 

3. Измените запрос и щелкните **создать**.

   ![пользовательский запрос](./media/tutorial-hunting/custom-query.png)

# <a name="tutorial-investigate-cases-with-azure-sentinel-preview"></a>Руководство по Изучите варианты с предварительной версией Sentinel Azure

> [!IMPORTANT]
> Сейчас Azure Sentinel предоставляется в общедоступной предварительной версии.
> Эта предварительная версия предоставляется без соглашения об уровне обслуживания и не рекомендована для использования рабочей среде. Некоторые функции могут не поддерживаться или их возможности могут быть ограничены. Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

Это руководство поможет вам обнаруживать угрозы с помощью Azure Sentinel.

После того как вы [подключенных источников данных](quickstart-onboard.md) Azure Sentinel, чтобы получать уведомления, когда происходит что-то подозрительные. Чтобы позволяют это сделать, Azure Sentinel возможность создать дополнительные правила генерации оповещений, которые создают случаи, которые можно назначить и глубоко изучить аномалии и угроз в вашей среде с помощью. 

> [!div class="checklist"]
> * Создание случаев
> * Расследование инцидентов
> * Реагирование на угрозы

## <a name="investigate-cases"></a>Расследование инцидентов

Случай может включать несколько предупреждений. Он считается совокупность всех соответствующих свидетельство для конкретного расследования. Создан на основе предупреждения, определенные в случае **Analytics** страницы. Свойства, связанные с оповещениями, таких как серьезность и состояние задаются на уровне вариантов. После позволить Azure Sentinel знать, какие виды угроз, которые вы ищете, а также способы их найти, вы можете отслеживать угрозы, которые определяются путем изучения вариантов. 

1. Выберите **случаях**. **Случаях** страницы позволяет узнать, сколько вариантов, у вас есть, сколько — открыть, сколько вы задали **выполняется**, и сколько закрываются. Для каждого варианта вы увидите, время, когда оно произошло и состояние случая. Рассмотрим серьезности, чтобы решить, что для обработки первого. В **случаях** щелкните **оповещения** вкладку, чтобы увидеть все предупреждения, связанные с вариантом. Сущности, которые были ранее частью так можно просмотреть в **сущностей** вкладки.  Вы можете отфильтровать вариантов, при необходимости, например с состоянием или серьезностью. Если взглянуть на **случаях** вкладку, вы увидите откройте вариантов, содержащих оповещения, активируемые ваши правила обнаружения, определенные в **Analytics**. В верхней части вы увидите свои активных обращений, новые случаи и в случаях хода выполнения. Также вы увидите Обзор всех примеров по уровню серьезности.

   ![Панель мониторинга оповещений](./media/tutorial-investigate-cases/cases.png)

2. Чтобы начать расследование, щелкните это частный случай достижения. В правой части вы увидите подробные сведения для случая, в том числе уровень серьезности, сводка количество субъектов (основываясь на сопоставлении). Каждый случай имеет уникальный идентификатор. Степень серьезности обращения определяется в соответствии с наиболее серьезные предупреждения, включенные в случае.  

1. Чтобы просмотреть дополнительные сведения о предупреждениях и сущности в случае, щелкните **просмотреть полные сведения** в случае страницы и просмотрите соответствующие вкладки, отображающих сведения об обращении.  Представление полного вариантов объединяет все собранные доказательства в предупреждения, связанные предупреждения и сущностей.

1. В **оповещения** вкладке, просмотрите самого - оповещения, когда оно было включено и сколько превышения пороговых значений, можно задать. Вы увидите все важные сведения о оповещения — запрос, вызвавшие предупреждение, количество результатов, возвращаемых в запросе, а также возможность запустить модули playbook для оповещения. Для перемещения вниз еще дальше в случае, щелкните число попаданий. При этом откроется запрос, который создал результаты и результаты, которые активируют оповещения в Log Analytics.

3. В **сущностей** вкладку, вы увидите все сущности, которые можно сопоставить как часть определения правила генерации оповещений. 

4. Если вы изучаете активно обращения, рекомендуется присвоить статус **выполняется** до его закрытия. Можно также закрыть так, где **закрыто Разрешить** состояние для случаев, которые указывают, что было обработано инцидента, хотя **закрытия отклоненных** состояние для случаев, которые не требуют обработки. Объяснения являются обязательными, объясняя вашей обоснование для закрытия обращения.

5. Варианты могут быть назначены конкретным пользователем. Для каждого случая можно назначить владельца, задав в случае **владельца** поля. Все случаи запускается как неназначенные. Можно перейти в случаях и фильтровать по свое имя, чтобы просмотреть все случаи, которыми вы владеете. 

5. Нажмите кнопку **исследовать** для просмотра схема анализа и области нарушения с действия по устранению проблемы. 



## <a name="respond-to-threats"></a>Реагирование на угрозы

Azure Sentinel предоставляет два основных варианта за реагирование на угрозы с помощью наборов инструкций playbook. Можно задать playbook для автоматического запуска при активации оповещения, или вы можете вручную запустить playbook в ответ на оповещение.

- Можно задать playbook для автоматического запуска при активации оповещения при настройке сборника. 

- Вы можете вручную запустить сборник тренировочных заданий из в предупреждении, щелкнув **просмотреть модули playbook** и выбрав playbook для запуска.

# <a name="tutorial-investigate-incidents-with-azure-sentinel-preview"></a>Учебник. Изучение инцидентов с предварительной версией Azure Sentinel

> [!IMPORTANT]
> Сейчас Azure Sentinel предоставляется в общедоступной предварительной версии.
> Эта предварительная версия предоставляется без соглашения об уровне обслуживания и не рекомендована для использования рабочей среде. Некоторые функции могут не поддерживаться или их возможности могут быть ограничены. Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

Этот учебник поможет обнаружить угрозы с помощью Azure Sentinel.

После [подключения источников данных](quickstart-onboard.md) к Sentinel Azure вы будете получать уведомления о подозрительных событиях. Чтобы сделать это, Azure Sentinel позволяет создавать расширенные правила генерации оповещений, которые создают инциденты, которые можно назначить и использовать для глубокого анализа аномалий и угроз в вашей среде. 

> [!div class="checklist"]
> * Создание инцидентов
> * Изучение инцидентов
> * Реагирование на угрозы

## <a name="investigate-incidents"></a>Изучение инцидентов

инцидент может включать несколько предупреждений. Это совокупность всех соответствующих доказательств для конкретного расследования. инцидент создается на основе предупреждений, определенных на странице **аналитики** . Свойства, связанные с предупреждениями, такие как серьезность и состояние, задаются на уровне инцидента. После того как вы разрешите Azure Sentinel узнать, какие типы угроз вы ищете, и как их найти, можно отслеживать угрозы, обнаруженные с помощью исследования инцидентов. 

1. Выберите **инциденты**. На странице инциденты вы узнаете, сколько инцидентов имеется, сколько открыто, сколько настроено на **выполнение**и сколько закрыто. Для каждого инцидента можно увидеть время его возникновения и состояние инцидента. Взгляните на серьезность, чтобы решить, что следует обменять первыми. На странице **инциденты** щелкните вкладку **предупреждения** , чтобы просмотреть все оповещения, связанные с инцидентом. Сущности, сопоставленные ранее как часть инцидента, можно просмотреть на вкладке **сущности** .  Можно отфильтровать инциденты по мере необходимости, например по состоянию или серьезности. При просмотре вкладки **инциденты** вы увидите открытые инциденты, которые содержат предупреждения, активируемые правилами обнаружения, определенными в **аналитике**. В верхней части вы увидите активные инциденты, новые инциденты и выполняемые инциденты. Вы также можете просмотреть общие сведения о всех инцидентах по серьезности.

   ![Панель мониторинга оповещений](./media/tutorial-investigate-cases/cases.png)

2. Чтобы начать исследование, щелкните конкретный инцидент. Справа можно просмотреть подробные сведения об инциденте, включая его серьезность, сводку по количеству участвующих сущностей (в зависимости от сопоставления). Каждый инцидент имеет уникальный идентификатор. Серьезность инцидента определяется в соответствии с наиболее серьезным предупреждением, включенным в инцидент.  

1. Чтобы просмотреть дополнительные сведения об оповещениях и сущностях в инциденте, щелкните **просмотреть полные сведения** на странице инцидент и просмотрите соответствующие вкладки, которые обобщаются сведения об инциденте.  Полное представление инцидента объединяет все доказательства в предупреждении, связанные предупреждения и сущности.

1. На вкладке **оповещения** проверьте само предупреждение — когда оно было запущено и сколько было превышено установленное пороговое значение. Вы можете просмотреть все соответствующие сведения о предупреждении — запрос, который активировал предупреждение, число возвращенных результатов на запрос, а также возможность запуска модули PlayBook для оповещений. Чтобы продолжить детализацию в инциденте, щелкните число попаданий. Откроется запрос, создавший результаты, и результаты, которые активировали предупреждение в Log Analytics.

3. На вкладке **сущности** можно просмотреть все сущности, сопоставленные как часть определения правила генерации оповещений. 

4. Если вы активно изучаете инцидент, рекомендуется установить состояние инцидента **в значение выполняется** до тех пор, пока вы не закроете его. Можно также закрыть инцидент, где закрыто **разрешено** — состояние инцидентов, указывающих на то, что инцидент был обработан, а **закрытое отклонение** — состояние инцидентов, которые не нуждаются в обработке. Пояснения необходимы для объяснения причины закрытия инцидента.

5. Инциденты могут назначаться конкретному пользователю. Для каждого инцидента можно назначить владельца, задав поле **владелец** инцидента. Все инциденты запускаются как неназначенные. Вы можете войти в инциденты и выполнить фильтрацию по имени, чтобы просмотреть все собственные инциденты. 

5. Нажмите кнопку **исследовать** , чтобы просмотреть карту и область расследования нарушений в действиях по исправлению. 



## <a name="respond-to-threats"></a>Реагирование на угрозы

Azure Sentinel предоставляет два основных варианта реагирования на угрозы с помощью модули PlayBook. Можно настроить сборник тренировочных заданий для автоматического запуска при срабатывании оповещения или вручную запустить сборник тренировочных заданий в ответ на предупреждение.

- Вы можете настроить сборник тренировочных заданий для автоматического запуска при активации оповещения при настройке сборник тренировочных заданий. 

- Вы можете вручную запустить сборник тренировочных заданий из оповещения, нажав кнопку **View модули PlayBook (Просмотр** ), а затем выбрав сборник тренировочных заданий для запуска.

# <a name="connect-data-from-azure-web-application-firewall-waf"></a>Подключение данных из брандмауэра веб-приложения Azure (WAF)

Веб-приложения все чаще подвергаются вредоносным атакам, использующим общеизвестные уязвимости. Брандмауэр веб-приложения Azure (WAF) обеспечивает централизованную защиту веб-приложений от распространенных эксплойтов и угроз, таких как внедрение кода и межузловые сценарии. Azure WAF можно развернуть в службе [шлюза приложений Azure](https://docs.microsoft.com/azure/web-application-firewall/ag/ag-overview) , службе " [Передняя дверца Azure](https://docs.microsoft.com/azure/web-application-firewall/afds/afds-overview) " и с помощью политики WAF [сети доставки содержимого (CDN)](https://docs.microsoft.com/azure/web-application-firewall/cdn/cdn-overview) Azure (последняя версия в общедоступной предварительной версии).
Вы можете подключить журналы WAF для Azure к Azure Sentinel, что позволит просматривать данные журналов в книгах, использовать их для создания пользовательских оповещений и внедрять их для улучшения расследования.

## <a name="prerequisites"></a>Предварительные требования

- У вас должны быть разрешения на чтение и запись в рабочей области Sentinel Azure.

## <a name="connect-to-azure-waf"></a>Подключение к Azure WAF

### <a name="instructions-tab"></a>Вкладка "инструкции"

1. В меню навигации меток Azure выберите **соединители данных**.

1. В коллекции соединителей данных выберите **брандмауэр веб-приложения Azure (WAF)** , а затем на панели предварительного просмотра выберите **открыть страницу соединителя** .

1. Выберите ссылку для типа ресурса WAF, к которому нужно подключиться, — **откройте ресурс шлюза приложений >**, **откройте >ресурсов передней дверцы **или **откройте политику WAF "сеть доставки содержимого" >** — и один раз на экране списка ресурсов, выберите ресурс WAF из списка.

    1. В меню навигации ресурса WAF выберите **параметры диагностики**.

    1. Выберите **+ Добавить параметр диагностики** в нижней части списка.

    1. На экране **параметры диагностики** введите имя в поле **имя параметров диагностики** .

    1. Установите флажок **отправить на log Analytics** . Под ним будут отображаться два новых поля. Выберите подходящую **подписку** и **log Analytics рабочую область** (где находится метка Azure).

    1. Установите флажки для типов правил, журналы которых вы хотите принять. Рекомендации для каждого типа ресурсов:

        | Ресурс | Журналы, выбранные для приема |
        |----------|------------------------------|
        | Шлюз приложений | ApplicationGatewayAccessLog<br>ApplicationGatewayFirewallLog |
        | Front Door          | FrontdoorAccessLog<br>FrontdoorWebApplicationFirewallLog |
        | Политика WAF CDN      | вебаппликатионфиревалллогс |
        |

    1. Щелкните **Сохранить**.

### <a name="next-steps-tab"></a>Вкладка "дальнейшие действия"

- Ознакомьтесь с рекомендуемыми книгами, примерами запросов и шаблонами правил аналитики, объединенными с соединителем данных **брандмауэра веб-приложения Azure** , чтобы получить представление о данных журнала WAF для Azure.

- Чтобы запросить данные Azure WAF в **журналах**, введите **AzureDiagnostics** в окне запроса.

> [!NOTE]
>
> С этим конкретным соединителем данных индикаторы состояния подключения (цветовая полоса в коллекции соединителей данных и значки подключения рядом с именами типов данных) отображаются как *подключенные* (зеленые), только если данные были приняты в течение последних двух недель. По прошествии двух недель без приема данных соединитель будет отображаться как отключенный. В то время, когда поступает больше данных, состояние *Connected* будет возвращено.

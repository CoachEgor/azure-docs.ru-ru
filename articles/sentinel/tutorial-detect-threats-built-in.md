# <a name="tutorial-detect-threats-out-of-the-box"></a>Учебник: Обнаружение угроз из коробки


> [!IMPORTANT]
> В настоящее время в открытом доступе к обнаружению угроз ведется публичный предварительный просмотр.
> Эта функция предоставляется без соглашения об уровне обслуживания, и она не рекомендуется для производственных нагрузок.
> Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

После [подключения источников](quickstart-onboard.md) данных к Azure Sentinel необходимо получить уведомление о подозрительном случае. Чтобы сделать это, Azure Sentinel предоставляет вам встроенные шаблоны. Эти шаблоны были разработаны командой экспертов и аналитиков Microsoft по безопасности на основе известных угроз, общих векторов атак и цепочек эскалации подозрительной активности. После включения этих шаблонов они будут автоматически искать любые действия, которые выглядят подозрительно по всей среде. Многие шаблоны могут быть настроены для поиска или фильтрации действий в соответствии с вашими потребностями. Оповещения, генерируемые этими шаблонами, создадут инциденты, которые можно назначить и исследовать в вашей среде.

Этот учебник поможет вам обнаружить угрозы с помощью Azure Sentinel:

> [!div class="checklist"]
> * Использование нестандартных обнаружений
> * Автоматизация реагирования на угрозы

## <a name="about-out-of-the-box-detections"></a>Сведения об обнаружении угроз с помощью встроенных средств

Чтобы просмотреть все готовые обнаружения, перейдите в раздел **Аналитика** а затем **Шаблоны правил**. На этой вкладке содержатся все встроенные правила Azure Sentinel.

   ![Использование встроенных обнаружений для поиска угроз с помощью Azure Sentinel](media/tutorial-detect-built-in/view-oob-detections.png)

Доступны следующие типы шаблонов:

- **Шаблоны безопасности Майкрософт** - шаблоны безопасности Майкрософт автоматически создают инциденты Azure Sentinel из оповещений, генерируемых в других решениях безопасности Майкрософт, в режиме реального времени. Правила безопасности Майкрософт можно использовать в качестве шаблона для создания новых правил с аналогичной логикой. Для получения дополнительной информации о правилах безопасности [см. Автоматически создавайте инциденты из предупреждений майкрософт о безопасности.](create-incidents-from-alerts.md)
- **Fusion** - На основе технологии Fusion, передовые многоступенчатые обнаружения атак в Azure Sentinel использует масштабируемые алгоритмы машинного обучения, которые могут соотнести многие предупреждения с низкой точностью и события через несколько продуктов в высокой точности и действия инцидентов. Fusion включен по умолчанию. Поскольку логика скрыта, вы не можете использовать ее в качестве шаблона для создания более одного правила.
- **Поведенческая аналитика машинного обучения** - Эти шаблоны основаны на запатентованных алгоритмах машинного обучения Майкрософт, поэтому вы не можете увидеть внутреннюю логику того, как они работают и когда они работают. Поскольку логика скрыта, вы не можете использовать ее в качестве шаблона для создания более одного правила.
-   **Запланировано** - Запланированные аналитические правила — это запланированные запросы, написанные экспертами по безопасности Майкрософт. Вы можете увидеть логику запроса и внести в него изменения. Запланированные правила можно использовать в качестве шаблона для создания новых правил с аналогичной логикой.

## <a name="use-out-of-the-box-detections"></a>Использование нестандартных обнаружений

1. Чтобы использовать встроенный шаблон, нажмите на **правило «Создать»** для создания нового активного правила, основанного на этом шаблоне. Каждая запись имеет список необходимых источников данных, которые автоматически проверяются, и это может привести к отключению **правила Create.**
  
   ![Использование встроенных обнаружений для поиска угроз с помощью Azure Sentinel](media/tutorial-detect-built-in/use-built-in-template.png)
 
1. Это открывает мастер создания правил, основанный на выбранном шаблоне. Все детали заполнились автоматически, а для **правил, запланированных** или **правил безопасности Майкрософт,** вы можете настроить логику, чтобы лучше соответствовать вашей организации, или создать дополнительные правила на основе встроенного шаблона. После выполнения шагов в мастере создания правил и завершения создания правила, основанного на шаблоне, новое правило появляется во вкладке **Активные правила.**

Для получения дополнительной информации о [Tutorial: Create custom analytic rules to detect suspicious threats](tutorial-detect-threats-custom.md)полях в мастере см.

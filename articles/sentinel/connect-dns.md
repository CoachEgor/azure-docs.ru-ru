# <a name="connect-your-domain-name-server"></a>Подключите сервер доменных имен

> [!IMPORTANT]
> Разъем данных DNS в Azure Sentinel в настоящее время находится в открытом доступе.
> Эта функция предоставляется без соглашения об уровне обслуживания, и она не рекомендуется для производственных нагрузок. Некоторые функции могут не поддерживаться или их возможности могут быть ограничены. Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

Вы можете подключить любой сервер доменных имен (DNS), работающий на Windows, к Azure Sentinel. Это делается путем установки агента на машине DNS. Используя DNS-журналы, можно получить информацию о безопасности, производительности и операциях в инфраструктуре DNS вашей организации, собирая, анализируя и сопоставляя аналитические и аудиторские журналы и другие связанные с этим данные с серверов DNS.

При включании соединения dNS-журнала можно:
- определить клиенты, которые пытаются разрешить вредоносные доменные имена;
- определить устаревшие записи ресурсов;
- определить часто запрашиваемые доменные имена и "разговорчивые" DNS-клиенты;
- просмотреть нагрузку запросов на DNS-серверы;
- просмотреть ошибки при регистрации динамических DNS.

## <a name="connected-sources"></a>Подключенные источники

В следующей таблице описаны подключенные источники, поддерживаемые этим решением.

| **Подключенный источник** | **Поддержка** | **Описание** |
| --- | --- | --- |
| [Агенты Windows](../azure-monitor/platform/agent-windows.md) | Да | Решение собирает сведения о DNS из агентов Windows. |
| [Агенты Linux](../azure-monitor/learn/quick-collect-linux-computer.md) | нет | Решение не собирает сведения о DNS из прямых агентов Linux. |
| [Группа управления System Center Operations Manager](../azure-monitor/platform/om-agents.md) | Да | Решение собирает сведения о DNS из агентов в подключенной группе управления Operations Manager. Прямое подключение агента Operations Manager к Azure Monitor не требуется. Данные пересылаются из группы управления в рабочую область Log Analytics. |
| [Учетная запись хранения Azure](../azure-monitor/platform/collect-azure-metrics-logs.md) | нет | Решение не использует службу хранилища Azure. |

### <a name="data-collection-details"></a>Сведения о сборе данных

Решение собирает данные, связанные с инвентаризацией и событиями DNS, с DNS-серверов, на которых установлен агент Log Analytics. Данные, связанные с инвентаризацией, такие как количество DNS-серверов, зон и записей ресурсов, собираются с помощью командлетов PowerShell для DNS. Эти данные обновляются каждые два дня. Данные, связанные с событиями, собираются практически в реальном времени из [журналов аналитики и аудита](https://technet.microsoft.com/library/dn800669.aspx#enhanc), которые обеспечиваются расширенными функциями диагностики и ведения журнала DNS в Windows Server 2012 R2.


## <a name="connect-your-dns-appliance"></a>Подключите прибор DNS

1. На портале Azure Sentinel выберите **разъемы данных** и выберите плитку **DNS (Preview).**
1. Если ваши DNS-машины находятся в Azure:
    1. Нажмите **Установка агента на Azure Windows виртуальной машины**.
    1. В списке **виртуальных машин** выберите машину DNS, которую вы хотите перейти в Azure Sentinel. Убедитесь, что это Windows VM.
    1. В окне, которое открывается для этого VM, нажмите **Connect**.  
    1. Нажмите **Включить** в окне **разъема DNS.** 

2. Если машина DNS не является VM Azure:
    1. Нажмите **На ссылку на агент Установки на машинах, не относясь от Azure.**
    1. В окне **прямого агента** выберите либо загрузить агента **Windows (64 бит)** или **загрузить агента Windows (32 бит)**.
    1. Установите агент на машину DNS. Копируйте **идентификатор рабочего пространства,** **основной ключ**и **вторичный ключ** и используйте их при установке.

3. Чтобы использовать соответствующую схему в журнале Analytics для журналов DNS, ищите **DnsEvents**.

## <a name="validate"></a>Проверить 

В журнале Analytics ищите схему **DnsEvents** и убедитесь, что есть события.

## <a name="troubleshooting"></a>Устранение неполадок

Если запросы поиска не отображаются в Azure Sentinel, выполните следующие действия, чтобы запросы отображались должным образом:
1. Включите [журналы DNS Analytics на серверах.](https://docs.microsoft.com/previous-versions/windows/it-pro/windows-server-2012-R2-and-2012/dn800669(v=ws.11))
2. Убедитесь, что DNSEvents отображается в списке коллекций Log Analytics.
3. Включите [on Azure DNS Analytics](../azure-monitor/insights/dns-analytics.md).
4. В Azure DNS Analytics в **настройках**изменяйте любые настройки, сохраняйте их, затем изменяйте обратно, если это необходимо, а затем сохраняйте его снова.
5. Проверьте аналитику Azure DNS, чтобы убедиться, что запросы теперь отображаются.

# <a name="connect-windows-security-events"></a>Подключение событий безопасности Windows 



Вы можете выполнять потоковую передачу всех событий безопасности с серверов Windows, подключенных к рабочей области Sentinel Azure. Это подключение позволяет просматривать панели мониторинга, создавать пользовательские предупреждения и улучшать исследование. Это позволяет получить более подробные сведения о сети организации и улучшить возможности обеспечения безопасности.  Вы можете выбрать события для потоковой передачи:

- **Все события** — все события безопасности Windows и AppLocker.
- **Common** — стандартный набор событий для целей аудита. В этот набор включен полный журнал аудита пользователя. Например, этот набор содержит как вход пользователя, так и события выхода пользователя (идентификатор события 4634). Мы добавили действия аудита, такие как изменение группы безопасности, операции основного контроллера домена Kerberos и другие события, рекомендованные отраслевыми организациями.

События небольших объемов были включены в набор "Общий". Основной причиной для его выбора среди всех событий является возможность уменьшить объем и оставить определенные события нефильтрованными.
- **Минимальный** набор событий, которые могут указывать на потенциальные угрозы. Если включить этот параметр, вы не сможете получить полный журнал аудита.  Этот набор охватывает только события, которые могут указывать на успешное нарушение и важные события с очень низким объемом. Например, этот набор содержит успешные и неудачные попытки входа (идентификаторы событий 4624, 4625), но не содержит сведений о выходе, которые важны для аудита, но не имеют смысла для обнаружения и имеют относительно большой объем. Большая часть объема данных этого набора — события входа и создание процесса (идентификатор события 4688).
- **Нет** — события безопасности и AppLocker отсутствуют.

> [!NOTE]
> 
> - Данные будут храниться в географическом расположении рабочей области, на которой вы используете метку Azure.
> - Если центр безопасности Azure и метка Azure выполняются в одной рабочей области, соединитель событий безопасности можно подключить только из центра безопасности Azure или с помощью Sentinel Azure. Чтобы управлять этими событиями из Azure Sentinel, рекомендуется отключить его от центра безопасности Azure и подключить его только к Azure Sentinel.

## <a name="set-up-the-windows-security-events-connector"></a>Настройка соединителя событий безопасности Windows

Чтобы полностью интегрировать события безопасности Windows с помощью Sentinel Azure, сделайте следующее:

1. На портале Sentinel Azure выберите **соединители данных** , а затем щелкните плитку **события безопасности Windows** . 
1. Выберите типы данных, которые необходимо передать в поток.
1. Нажмите кнопку **Обновить**.
6. Чтобы использовать соответствующую схему в Log Analytics для событий безопасности Windows, выполните поиск по запросу **SecurityEvent**.

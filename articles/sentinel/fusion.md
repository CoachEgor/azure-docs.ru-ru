# <a name="advanced-multistage-attack-detection-in-azure-sentinel"></a>Расширенное обнаружение атак многоступенчатых в Azure Sentinel

С помощью технологии Fusion, основанной на машинном обучении, Azure Sentinel может автоматически обнаруживать атаки многоступенчатых, объединяя аномальные поведения и подозрительные действия, наблюдаемые на различных этапах цепочки уничтожения. Затем Sentinel Azure создает инциденты, которые в противном случае было бы трудно перехватить. Эти инциденты добавлен два или более оповещений или действий. Эти инциденты являются низким объемом, высокой точностью и высокой серьезностью.

Это обнаружение, настроенное для вашей среды, не только сокращает ложные положительные результаты, но и может обнаруживать атаки с ограниченной или отсутствующей информацией.

## <a name="configuration-for-advanced-multistage-attack-detection"></a>Конфигурация для расширенного обнаружения многоступенчатыхных атак

Это обнаружение включено по умолчанию в Azure Sentinel. Чтобы проверить состояние или отключить его, возможно, это связано с тем, что вы используете альтернативное решение для создания инцидентов на основе нескольких оповещений, выполните следующие действия.

1. Перейдите на [портал Azure](https://portal.azure.com), если вы еще этого не сделали.

2. Перейдите к **Azure Sentinel** > **Configuration** > **Analytics**

3. Выберите **активные правила** и в столбце **имя** перейдите к **расширенному обнаружению многоступенчатых атак** . Проверьте столбец **состояние** , чтобы убедиться, что это обнаружение включено или отключено.

4. Чтобы изменить состояние, выберите эту запись и в колонке **Advanced многоступенчатых атаковать Detection (Расширенный поиск атак** ) выберите **изменить**.

5. В колонке **Мастер создания правил** изменится состояние автоматически, поэтому выберите **Далее: Проверка**, а затем **Сохраните**. 

Шаблоны правил неприменимы для расширенного обнаружения многоступенчатыхных атак.

    Это предупреждение указывает на событие входа, \<*имя учетной записи*> из невозможности перемещения в \<*Расположение*>, нетипичное расположение.
    
    Затем учетная запись \<*имя учетной записи*> выполняется через \<*число*> административных действий в одном сеансе.

- **Невозможно переместиться в нетипичные расположения, ведущие к массовому удалению файлов**
    
    Это предупреждение указывает на событие входа, \<*имя учетной записи*> \<*Расположение*>, нетипичное расположение. 
    
    Затем учетная запись \<*имя учетной записи*> удалена \<*число*> уникальных файлов в одном сеансе.

    Затем *имя*учетной записи \<> загружено через \<*число*> уникальных файлов в одном сеансе.

- **Невозможно переместиться в нетипичные расположения, приводящие к олицетворению Office 365**
    
    Это предупреждение указывает на событие входа, \<*имя учетной записи*> из невозможности перемещения в \<*Расположение*>, нетипичное расположение. 
    
    Затем учетная запись \<*имя учетной записи*> выводила необычное количество действий олицетворения в одном сеансе (\<*число действий,* >).

- **Невозможно переместиться в нетипичные расположения, ведущие к массовому обмену файлами**
    
    Это предупреждение указывает на событие входа, \<*имя учетной записи*> из невозможности перемещения в \<*Расположение*>, нетипичное расположение.

- **Невозможно переместиться в нетипичные расположения, ведущие к отдельному для атаки в облачном приложении**
    
    Это предупреждение указывает на событие входа, \<*имя учетной записи*> из невозможности перемещения в \<*Расположение*>, нетипичное расположение. 
    
    Затем учетная запись \<*имя учетной записи*> отправлена \<*число*файлов > и удалено \<*число*>ных файлов. 
    
    Эта схема действия указывает на потенциальную атаку на взломщик.

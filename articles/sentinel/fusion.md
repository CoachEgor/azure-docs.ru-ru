# <a name="advanced-multistage-attack-detection-in-azure-sentinel"></a>Расширенное многоступенчатое обнаружение атак в Azure Sentinel


> [!IMPORTANT]
> Некоторые функции Fusion в Azure Sentinel в настоящее время находятся в открытом доступе.
> Эти функции предоставляются без соглашения об уровне обслуживания, и это не рекомендуется для производственных нагрузок. Некоторые функции могут не поддерживаться или их возможности могут быть ограничены. Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).



Используя технологию Fusion, основанную на машинном обучении, Azure Sentinel может автоматически обнаруживать многоступенчатые атаки, комбинируя аномальное поведение и подозрительные действия, которые наблюдаются на различных стадиях цепочки убийства. Затем Azure Sentinel генерирует инциденты, которые в противном случае было бы очень трудно поймать. Эти инциденты угожат два или более предупреждений или действия. По замыслу, эти инциденты имеют низкий объем, высокую точность и высокую степень тяжести.

Это обнаружение, настроенное для вашей среды, не только снижает ложноположительные показатели, но и позволяет обнаруживать атаки с ограниченной или отсутствующими сведениями.

## <a name="configuration-for-advanced-multistage-attack-detection"></a>Настройка расширенного обнаружения многоэтапных атак

Это обнаружение включено по умолчанию в Azure Sentinel. Чтобы проверить состояние или отключить его, возможно, потому, что вы используете альтернативное решение для создания инцидентов на основе нескольких предупреждений, используйте следующие инструкции:

1. Если вы еще не сделали этого, вопийте на [порталaz.](https://portal.azure.com)

2. Перейдите к**аналитике** **конфигурации** >  **Azure Sentinel** > 

3. Выберите **активные правила** и найдите **расширенное многоступенчатое обнаружение атак** в столбце **NAME.** Проверьте столбец **STATUS,** чтобы подтвердить, включено ли это обнаружение или отключено.

4. Чтобы изменить статус, выберите эту запись и на **расширенном многоступенчатом** лезвии обнаружения атак, выберите **Edit.**

5. На лезвии **мастера создания правил** изменение статуса автоматически выбирается для вас, поэтому выберите **Следующий: Просмотр,** а затем **сохранить.** 

Шаблоны правил не применимы для расширенного многоступенчатого обнаружения атак.

> [!NOTE]
> В настоящее время Azure Sentinel использует исторические данные за 30 дней для обучения систем машинного обучения. Эти данные всегда шифруются с помощью ключей Майкрософт при прохождении через конвейер машинного обучения. Однако обучающие данные не шифруются с помощью [управляемых ключей клиента (CMK),](customer-managed-keys.md) если вы включили CMK в рабочей области Azure Sentinel. Чтобы отказаться от Fusion, перейдите на **Azure Sentinel** \> **Configuration** \> **Analytics \> \> Active правила Расширенного многоступенчатого обнаружения атак** и в столбце **состояния,** выберите **отключить.**

# <a name="connect-one-identity-safeguard"></a>Подключение одной гарантии идентификации
Разъем данных One Identity Safeguard улучшает стандартный формат общего события (CEF) с панелями мониторинга Safeguard for Privileged Sessions. Используйте этот разъем, чтобы легко начать использовать события, которые генерируются устройством для визуализации, оповещения, исследований и многое другое.


## <a name="prerequisites"></a>Предварительные требования

- Прочитайте и напишите разрешения в рабочее пространство Azure Sentinel.

## <a name="connect-azure-sentinel-to-one-identity-safeguard"></a>Подключение Azure Sentinel к одной защите идентификации

1. На портале Azure перейдите на**разъемы данных** **Azure Sentinel** > Data, а затем выберите разъем **One Identity Safeguard.**

2. Выберите **страницу Open connector**.

3. Следуйте инструкциям на странице **One Identity Safeguard.**

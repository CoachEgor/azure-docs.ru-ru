# <a name="connect-your-data-from-azure-defender-formerly-azure-security-center-for-iot-to-azure-sentinel"></a>Подключение данных из Azure Defender (прежнее название — центр безопасности Azure) для Azure IoT 


> [!IMPORTANT]
> Azure Defender для соединителя данных IoT сейчас находится в общедоступной предварительной версии. Эта функция предоставляется без соглашения об уровне обслуживания и не рекомендуется для рабочих нагрузок. Некоторые функции могут не поддерживаться или их возможности могут быть ограничены. Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

Используйте соединитель защитника Azure для Интернета вещей для потоковой передачи всех событий защитника Azure для IoT в Azure Sentinel. 

## <a name="prerequisites"></a>Предварительные требования

- Разрешения на **Чтение** и **запись** в рабочей области, на которой развернута Sentinel Azure
- На соответствующих центрах Интернета вещей должна быть **включена** служба **"защитник Azure для Интернета вещей"**
- Разрешения на **Чтение** и **запись** в **центре Интернета вещей Azure** , к которому вы хотите подключиться.
- Разрешения на **Чтение** и **запись** в **группе ресурсов центра Интернета вещей Azure**

## <a name="connect-to-azure-defender-for-iot"></a>Подключение к службе "защитник Azure для Интернета вещей"

1. В качестве метки Azure выберите **соединители данных** , а затем в галерее щелкните **"защитник Azure для Интернета вещей"** (также может называться "Центр безопасности Azure для Интернета вещей").
1. В нижней правой области щелкните **открыть страницу соединителя**. 
1. Нажмите кнопку **подключить**рядом с каждой подпиской центра Интернета вещей, предупреждениями и оповещениями устройств, которые вы хотите передавать в метку Azure. 
    - Если служба "защитник Azure для Интернета вещей" не включена в этом центре, вы увидите сообщение Enable Warning ( **включить** предупреждение). Щелкните ссылку **Enable (включить** ), чтобы запустить службу. 
1. Вы можете решить, должны ли оповещения из защитника Azure для IoT автоматически создавать инциденты в Azure Sentinel. В разделе **Создание инцидентов**выберите **включить** , чтобы включить стандартное правило аналитики для автоматического создания инцидентов на основе предупреждений, созданных в подключенной службе безопасности. Это правило можно изменить или изменить в разделе **аналитика**  >  **Активные** правила.

> [!NOTE]
> Обновление списка концентраторов может занять некоторое время после внесения изменений в соединение. 

## <a name="log-analytics-alert-display"></a>Отображение предупреждений Log Analytics

Чтобы использовать соответствующую схему в Log Analytics для вывода оповещений о защитнике Azure для IoT:

1. Откройте **журналы**  >  **секуритинсигхтс**  >  **секуритялерт**или выполните поиск по запросу **секуритялерт**. 
2. Фильтр для просмотра только тех оповещений, которые созданы защитником Azure для Интернета вещей, с помощью следующего фильтра ККЛ:

```kusto
SecurityAlert | where ProductName == "Azure Defender for IoT"
``` 

### <a name="service-notes"></a>Заметки о службе

После подключения центра Интернета вещей данные концентратора доступны в Sentinel в течение приблизительно 15 минут позже.

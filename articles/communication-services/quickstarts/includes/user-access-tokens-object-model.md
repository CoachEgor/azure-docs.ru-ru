## <a name="object-model"></a>Объектная модель

Маркеры доступа пользователей позволяют авторизовать пользователей приложения для непосредственного подключения к ресурсу Служб коммуникации Azure. Эти токены создаются на сервере, передаются обратно в клиентское приложение, а затем используются для инициализации клиентских библиотек Служб коммуникации Azure. Для реализации этой схемы следует создать доверенную конечную точку службы, которая может проверять подлинность пользователей и выдавать им маркеры доступа. Эта служба должна обеспечивать постоянное сопоставление удостоверений пользователя приложения и пользователей Служб коммуникации Azure.

Класс CommunicationIdentityClient предоставляет все функциональные возможности, связанные с пользователями и маркерами доступа. С помощью строки подключения можно создать его экземпляр, а затем использовать его для управления пользователями и выдачи маркеров доступа.

> [!WARNING]
> Маркеры считаются конфиденциальными данными, поскольку они предоставляют доступ к ресурсам пользователя. Поэтому крайне важно защитить их от компрометации. Следует создать доверенную конечную точку службы, которая будет выдавать маркеры доступа только полномочным пользователям приложения. При кэшировании маркеров доступа пользователей в резервное хранилище настоятельно рекомендуется использовать шифрование.

### <a name="access-token-scopes"></a>Области маркеров доступа

Области позволяют указать точный набор функциональных возможностей Служб коммуникации Azure, использование которых можно авторизовать с помощью маркера доступа пользователя. Области применяются к маркерам доступа отдельных пользователей. Службы коммуникации Azure поддерживают перечисленные ниже области для маркеров доступа пользователей.

| Имя   | Описание                                                                         |
| ------ | ----------------------------------------------------------------------------------- |
| `Chat` | Предоставляет возможность участвовать в чате                                         |
| `VoIP` | Предоставляет возможность отправки и приема вызовов VOIP с помощью клиентской библиотеки вызовов* |
| `Pstn` | Предоставляет возможность отправки вызовов ТСОП с помощью клиентской библиотеки вызовов*           |

\* Функция пока не поддерживается, но скоро станет доступна

Если вы хотите запретить пользователю доступ к определенной функциональной возможности, сначала [отзовите все существующие маркеры доступа](#revoke-user-access-tokens), которые могут включать ненужные области, а затем выдайте новый маркер с более ограниченным набором областей.

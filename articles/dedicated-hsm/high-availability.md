# <a name="azure-dedicated-hsm-high-availability"></a>Высокая доступность выделенного устройства HSM Azure

Выделенное устройство HSM Azure поддерживается высокодоступными центрами обработки данных Майкрософт. Тем не менее любой высокодоступный центр обработки данных уязвим к локальным сбоям, а в экстренных ситуациях — к сбоям на уровне региона. Майкрософт развертывает устройства HSM в разных центрах обработки данных в пределах региона, чтобы гарантировать, что подготовка нескольких устройств не приведет к использованию одной стойки для этих устройств. Следующий уровень определения степени высокий уровень доступности достигается путем связывания этих аппаратные модули безопасности в центрах обработки данных в регионе, используя функцию Gemalto высокого уровня ДОСТУПНОСТИ группы. Кроме того, возможно связать устройства между регионами для выполнения отработки отказа в регионе в случае аварийного восстановления. С помощью такой многоуровневой конфигурации с высоким уровнем доступности любой сбой в работе устройства будет автоматически разрешен для обеспечения работоспособности приложения. Все центры обработки данных также включают запасные устройства и компоненты на месте, поэтому можно своевременно заменить любое неисправное устройство.

## <a name="high-availability-example"></a>Пример высокого уровня доступности

Со сведениями о том, как настроить устройства HSM для обеспечения высокого уровня доступности на уровне программного обеспечения, можно ознакомиться в руководстве по администрированию устройств HSM в сети Gemalto Luna. Этот документ доступен на [портале поддержки клиентов Gemalto](https://supportportal.gemalto.com/csm/).

На следующей схеме показана архитектура с высоким уровнем доступности. В ней используются несколько устройств в регионе и несколько устройств, связанных в отдельном регионе. В этой архитектуре используются как минимум четыре устройства HSM и компоненты виртуальной сети.

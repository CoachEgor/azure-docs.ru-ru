## <a name="data-protection"></a>Защита данных

| Контроль безопасности | Да/нет | Примечания | Документация |
|---|---|--|--|
| Шифрование на стороне сервера в состоянии покоя: клавиши, управляемые Корпорацией Майкрософт | Да | Чувствительные данные, такие как сертификаты, ключи и значения с секретными именами, шифруются с помощью управляемых сервисом ключей для экземпляров службы. |  |
| Шифрование на стороне сервера в состоянии покоя: ключи, управляемые клиентом (BYOK) | нет | Все ключи шифрования находятся в экземпляре службы и управляются службой. |  |
| Шифрование уровня столбцов (Службы данных Azure)| Недоступно | |  |
| Шифрование в пути (например, шифрование ExpressRoute, шифрование VNet и шифрование VNet-VNet)| Да | [Шифрование Express Route](../expressroute/index.yml) и VNet обеспечивается [сетью Azure.](../virtual-network/index.yml) |  |
| Вызовы API в зашифрованном виде| Да | Вызовы на плоскости управления производятся через [менеджер ресурсов Azure](../azure-resource-manager/index.yml) через TLS. Требуется допустимый веб-маркер JSON (JWT).  Вызовы плоскости данных могут быть защищены tLS и одним из поддерживаемых механизмов аутентификации (например, сертификат клиента или JWT). |   |

## <a name="vulnerability-scans-false-positives"></a>Уязвимость сканирует ложные срабатывания

В этом разделе документы учитывают общие уязвимости, которые не влияют на управление API Azure.

| Уязвимость               | Описание                                                                                                                                                                                                                                                                                                               |
|-----------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Билетный блик (CVE-2016-9244) | Ticketbleed — это уязвимость в реализации расширения TLS SessionTicket, найденного в некоторых продуктах F5. Это позволяет утечки ("кровотечение") до 31 байт данных из непервоначальной памяти. Это вызвано укладкой TLS, набиваюя идентификатором сеанса, передаваемым от клиента, с данными, чтобы сделать его длиной 32 бита. |

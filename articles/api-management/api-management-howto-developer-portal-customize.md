# <a name="access-and-customize-developer-portal"></a>Доступ и настройка портала разработчиков

Портал разработчика — это автоматически созданный, полностью настраиваемый веб-сайт с документацией ваших AA. Это место, где потребители API могут открыть для себя ваши API, узнать, как их использовать, и запросить доступ.

В этом руководстве описано следующее:

> [!div class="checklist"]
> * Доступ к управляемой версии портала разработчика
> * Навигация по административному интерфейсу
> * Настройка содержимого
> * Опубликовать изменения
> * Посмотреть опубликованный портал

Более подробную информацию можно найти на портале разработчиков в [обзоре портала разработчиков Azure API Management.](api-management-howto-developer-portal.md)

![Портал разработчика API Management - режим админ](media/api-management-howto-developer-portal-customize/cover.png)

## <a name="prerequisites"></a>Предварительные требования

- Завершите следующий быстрый запуск: [Создайте экземпляр управления API Azure](get-started-create-service-instance.md)
- Импортируйте и опубликуйте экземпляр службы управления API Azure. Для получения дополнительной информации [см.](import-and-publish.md)

## <a name="access-the-portal-as-an-administrator"></a>Доступ к порталу в качестве администратора

Следуйте ниже, чтобы получить доступ к управляемой версии портала.

1. Перейдите на пример службы управления API на портале Azure.
1. Нажмите на кнопку **портала разработчика** в верхней панели навигации. Откроется новая вкладка браузера с административной версией портала.

## <a name="understand-the-portals-administrative-interface"></a>Понимание административного интерфейса портала

### <a name="default-content"></a>Содержимое по умолчанию 

Если вы получаете доступ к порталу в первый раз, содержимое по умолчанию будет автоматически подготовлено в фоновом режиме. Содержимое по умолчанию было разработано для демонстрации возможностей портала и минимизации количества настроек, необходимых для персонализации портала. Вы можете узнать больше о том, что включено в содержание портала, в [обзоре портала разработчиков Azure API Management.](api-management-howto-developer-portal.md)

### <a name="visual-editor"></a>Визуальный редактор

Вы можете настроить содержимое портала с помощью визуального редактора. Разделы меню слева позволяют создавать или изменять страницы, носители, макеты, меню, стили или настройки веб-сайта. Элементы меню внизу позволяют переключаться между портами просмотра (например, мобильными или настольными, просматривать элементы портала, видимые для аутентифицированных или анонимных пользователей, или сохранять или отменять действия.

Вы можете добавить строки на страницу, нажав на синий значок с знаком плюс. Виджеты (например, список текста, изображений или AIS) могут быть добавлены, нажав серый значок с знаком плюс. Вы можете переставить элементы на странице с перетаскиванием и падением взаимодействия. 

### <a name="layouts-and-pages"></a>Планы и страницы

![Страницы и макеты](media/api-management-howto-developer-portal-customize/pages-layouts.png)

Планы определяют, как отображаются страницы. Например, в содержимом по умолчанию есть два макета: один относится к главной странице, а другой на все остальные страницы.

Макет наносится на страницу, сопоставляя шаблон URL-адреса с URL-адресом страницы. Например, макет с `/wiki/*` URL-шаблоном будет применяться `/wiki/` к каждой `/wiki/getting-started`странице с сегментом в URL: `/wiki/styles`и т.д.

На рисунке выше содержимое, относящееся к макету, обозначено синим цветом, в то время как страница отмечена красным цветом. Разделы меню отмечены соответственно.

### <a name="styling-guide"></a>Руководство по укладке

![Руководство по укладке](media/api-management-howto-developer-portal-customize/styling-guide.png)

Стиль руководство представляет собой панель, созданная с дизайнерами в виду. Это позволяет осуществлять надзор и укладку всех визуальных элементов в вашем портале. Укладка иерархична - многие элементы наследуют свойства от других элементов. Например, элементы кнопки используют цвета для текста и фона. Чтобы изменить цвет кнопки, необходимо изменить исходный вариант цвета.

Чтобы отсеять вариант, нажмите на него и выберите значок карандаша, который появляется на нем. Как только вы внесете изменения во всплывающее окно, закройте его.

### <a name="save-button"></a>Кнопка "Сохранить"

![Кнопка "Сохранить"](media/api-management-howto-developer-portal-customize/save-button.png)

Всякий раз, когда вы вносить изменения в портал, вы должны сохранить его вручную, нажав кнопку **Сохранить** в меню в нижней части. При сохранении изменений измененное содержимое автоматически загружается в службу управления API.

## <a name="customize-the-portals-content"></a>Настройка содержимого портала

Прежде чем сделать свой портал доступным для посетителей, необходимо персонализировать автоматически созданный контент. Рекомендуемые изменения включают макеты, стили и содержимое главной страницы.

> [!NOTE]
> Из-за соображений интеграции, следующие страницы не могут `/404`быть `/500` `/captcha`удалены или перемещены под `/signin-sso` `/signup`другим URL: , `/change-password` `/config.json` `/confirm/invitation` `/confirm-v2/identities/basic/signup` `/confirm-v2/password` `/internal-status-0123456789abcdef` `/publish` `/signin`, , , , , , .

### <a name="home-page"></a>Главная страница

Главная **страница** по умолчанию заполнена фиктивным содержимым. Вы можете удалить все разделы с содержанием или сохранить структуру и настроить элементы по одному. Замените созданный текст и изображения своими собственными и убедитесь, что ссылки указывают на желаемые места.

### <a name="layouts"></a>Макеты

Замените автоматически генерируемый логотип в панели навигации собственным изображением.

### <a name="styling"></a>Задание стиля

Хотя вам не нужно корректировать какие-либо стили, вы можете рассмотреть возможность корректировки определенных элементов. Например, измените основной цвет, чтобы соответствовать цвету вашего бренда.

## <a name="publish-the-portal"></a><a name="publish"> </a>Публикация портала

Чтобы сделать ваш портал и его последние изменения доступными для посетителей, необходимо опубликовать его.

1. Убедитесь, что вы сохранили свои изменения, нажав на значок **Сохранить.**
1. Нажмите на **веб-сайт Publish** в разделе **Операции** меню. Это может занять несколько минут.  
    ![Публикация портала](media/api-management-howto-developer-portal-customize/publish-portal.png)

> [!NOTE]
> Портал должен быть переиздан после изменения конфигурации службы API Management, таких как назначение пользовательского домена, обновление поставщиков идентификационных данных, настройка делегирования, указание условий ввоза и продукта и многое другое.

## <a name="visit-the-published-portal"></a>Посетите опубликованный портал

После публикации портала вы можете получить к нему доступ по `https://contoso-api.developer.azure-api.net`тому же URL, что и административная панель, например. Просмотр его в отдельном сеансе браузера (инкогнито / режим частного просмотра) в качестве внешнего посетителя.

## <a name="apply-the-cors-policy-on-apis"></a>Применение политики CORS на AIS

Необходимо включить CORS (совместное использование ресурсов кросс-происхождения) на aI-классах, чтобы посетители портала могли протестировать AI через встроенную интерактивную консоль. Для более подробной информации обратитесь к [этой статье документации.](api-management-howto-developer-portal.md#cors)

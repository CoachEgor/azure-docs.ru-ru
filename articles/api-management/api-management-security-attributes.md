# <a name="common-security-attributes-for-api-management"></a>Общие атрибуты безопасности для службы управления API

Средства безопасности интегрированы в каждом аспекте службы Azure. В этой статье описываются наиболее распространенные атрибуты безопасности, встроенные в службе управления API.

[!INCLUDE [Security Attributes Header](../../includes/security-attributes-header.md)]

## <a name="preventative"></a>профилактическая;

| Атрибут безопасности | Да/нет | Примечания |
|---|---|--|
| Шифрование при хранении:<ul><li>Шифрование на стороне сервера</li><li>Шифрование на стороне сервера с использованием управляемых клиентом ключей</li><li>Другие возможности шифрования (например, на стороне клиента, постоянное шифрование и т. д.)</ul>| Да (только шифрование на стороне службы) | Зашифрованные конфиденциальные данные, например сертификаты, ключи и значения с именем секрета с помощью службы управляемое за ключей экземпляра службы. |
| Шифрование при передаче:<ul><li>Шифрование ExpressRoute</li><li>При шифровании виртуальной сети</li><li>Шифрование между виртуальными сетями</ul>| Yes | [Каналы Expressroute](../expressroute/index.yml) и виртуальной сети Шифрование обеспечивается корпорацией [сети Azure](../virtual-network/index.yml). |
| Управление ключами шифрования (CMK, BYOK, и т.д.)| Нет  | Все ключи шифрования на экземпляр службы и управление службой. |
| Шифрование на уровне столбцов (службы данных Azure)| Н/Д | |
| Вызовы API в зашифрованном виде| Yes | Вызовы плоскости управления выполняются через [Azure Resource Manager](../azure-resource-manager/index.yml) через TLS. Требуется допустимый веб-маркер JSON (JWT).  Вызовы плоскости данных могут быть защищены с помощью TLS и из механизмов проверки подлинности (например, сертификат клиента или JWT).
 |

## <a name="network-segmentation"></a>сегментация сети;

| Атрибут безопасности | Да/нет | Примечания |
|---|---|--|
| Поддержка конечной точки службы| Нет  | |
| Внедрение поддержки виртуальной сети| Yes | |
| Сетевая изоляция и множество поддержки| Yes | С помощью сетевых групп безопасности (NSG) и шлюз приложений Azure (или другие устройства программного обеспечения) соответственно. |
| Принудительного туннелирования поддержки| Yes | Сеть Azure предоставляет принудительное туннелирование. |

## <a name="vulnerability-scans-false-positives"></a>Уязвимость сканирует ложных положительных результатов

В этом разделе перечислены распространенные уязвимости, которые не влияют на управление API.

| Уязвимость               | ОПИСАНИЕ                                                                                                                                                                                                                                                                                                               |
|-----------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Ticketbleed (CVE-2016-9244) | Ticketbleed — уязвимость в реализации расширения TLS SessionTicket найден в некоторых СУБД F5. Она позволяет утечки данных («самые») до 31 байтов данных из неинициализированной памяти. Это вызвано стек TLS, padding идентификатор сеанса, переданные от клиента, с данными, чтобы сделать его 32-битное значение long. |

# <a name="configure-a-custom-domain-name"></a>Настройка имени личного домена

При предоставлении [самостоятельного шлюза Azure API Management](self-hosted-gateway-overview.md) ему не присваивается имя хоста и на него должен ссылаться его IP-адрес. В этой статье показано, как сопоставить существующее пользовательское имя DNS (также называемое хостименой) как самоходном шлюзе.

> [!NOTE]
> Функция самостоятельного входной шлюза находится в предварительном просмотре. Во время предварительного просмотра самоходной шлюз доступен только в уровнях разработчика и Premium без дополнительной платы. Уровень разработчика ограничен одним самоходном развертыванием шлюза.

## <a name="prerequisites"></a>Предварительные требования

Чтобы выполнить действия, описанные в этой статье, необходимо следующее.

-   Активная подписка Azure.

    [!INCLUDE [quickstarts-free-trial-note](../../includes/quickstarts-free-trial-note.md)]

-   Экземпляр управления API. Дополнительные сведения см. в статье о [создании экземпляра управления API Azure](get-started-create-service-instance.md).
- Самоуправляемый шлюз. Для получения дополнительной информации [см.](api-management-howto-provision-self-hosted-gateway.md)
-   Пользовательское доменное имя, принадлежащее вам или вашей организации. Эта тема не дает инструкций о том, как приобрести пользовательское доменное имя.
-   Запись DNS, размещенная на сервере DNS, который отображает пользовательское доменное имя на IP-адрес самостоятельного шлюза. Эта тема не дает инструкций о том, как разместить запись DNS.
-   Необходимо иметь действующий сертификат с открытым и закрытым ключами (PFX-файл). Альтернативное имя субъекта или субъекта (SAN) должно соответствовать доменному имени (это позволяет экземпляру Управления API безопасно разоблачать URL-адреса по TLS).

## <a name="add-custom-domain-certificate-to-your-api-management-service"></a>Добавление пользовательского сертификата домена в службу управления API

1. Выберите **Сертификаты** из **системы безопасности.**
2. Выберите **+ Добавить**.
3. Введите имя ресурса для сертификата в поле **Id.**
4. Выберите файл, содержащий сертификат (. PFX) путем выбора поля **сертификата** или значка папки, примыкающей к нему.
5. Введите пароль для сертификата в поле **Password.**
6. Выберите **Создать,** чтобы добавить сертификат в службу управления API.

## <a name="use-the-azure-portal-to-set-a-custom-domain-name-for-your-self-hosted-gateway"></a>Используйте портал Azure, чтобы установить пользовательское доменное имя для самостоятельного шлюза

1. Выберите **шлюзы** **из-под настроек.**
2. Выберите самохлаженный шлюз, для которого вы хотите настроить доменное имя.
3. Выберите **имена хост в** **настройках.**
4. Выберите **и добавить**
5. Введите имя ресурса для имени хоста в поле **«Имя».**
6. Введите доменное имя в поле **Hostname.**
7. Выберите сертификат **Certificate** из сертификата.
8. Выберите чекбокс **сертификата клиента Negotiate,** если какой-либо из AA, выставленных через этот шлюз, использует аутентификацию сертификата клиента.
    > [!WARNING]
    > Эта настройка используется всеми именами доменов, настроенными для шлюза.
9. Выберите **Добавить,** чтобы назначить пользовательское доменное имя выбранному самостоятельно размещенному шлюзу.

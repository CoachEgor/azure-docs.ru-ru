# <a name="azure-defender-for-iot-prerequisites"></a>Предварительные требования для защитника Azure для IoT

В этой статье приводится описание различных компонентов защитника для службы IoT, необходимые для начала действия и объясняются основные понятия, помогающие понять службу.

## <a name="minimum-requirements"></a>Минимальные требования

- Безагентный мониторинг для Интернета вещей и устройств (на основе технологии Циберкс)
    - Сетевые коммутаторы, поддерживающие мониторинг трафика через порт SPAN
    - Аппаратные устройства для датчика НТА. Дополнительные сведения см. в разделе [сертифицированное оборудование](https://aka.ms/AzureDefenderforIoTBareMetalAppliance) .
    - Роль **участника** подписки Azure (требуется только во время адаптации для определения зафиксированных устройств)
    - Роль **участника** центра Интернета вещей (уровень "бесплатный" или "Стандартный") (для управления с подключением к облаку)
- Безопасность управляемых устройств IoT, управляемых через центр Интернета вещей Azure
    - Роль **участника** центра Интернета вещей (уровня "Стандартный")
    - Центр Интернета вещей: переключение функции **"защитник Azure для Интернета вещей"** должно быть включено
    - Для поддержки модуля безопасности на уровне устройства  
        - Защитник для агентов IoT поддерживает растущей список устройств и платформ. см. [список поддерживаемых платформ](how-to-deploy-agent.md) .


## <a name="supported-service-regions"></a>Поддерживаемые регионы для службы

Дополнительные сведения см. в разделе [Поддерживаемые регионы центра Интернета вещей](https://azure.microsoft.com/global-infrastructure/services/?products=iot-hub) . 

Защитник для Интернета вещей направляет весь трафик из всех стран Европейского региона в региональный центр обработки данных Западной Европы и все остальные регионы в Центральный Региональный центр обработки данных США.

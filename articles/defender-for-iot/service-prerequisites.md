# <a name="azure-defender-for-iot-prerequisites"></a>Предварительные требования для защитника Azure для IoT

В этой статье приводится описание различных компонентов защитника для службы IoT, необходимые для начала действия и объясняются основные понятия, помогающие понять службу.

## <a name="minimum-requirements"></a>Минимальные требования

- Уровень "Стандартный" центра Интернета вещей
  - Привилегии уровня **владельца** роли Azure
- [Рабочая область Log Analytics](https://docs.microsoft.com/azure/azure-monitor/learn/quick-create-workspace)
- Центр безопасности Azure (рекомендуется)
  - Использование центра безопасности Azure является рекомендацией, а не требованием. Без центра безопасности Azure вы не сможете просматривать другие ресурсы Azure в центре Интернета вещей.

## <a name="working-with-defender-for-iot-service"></a>Работа с защитником для службы IoT

Защитник для Azure IoT Insights и отчеты доступны с помощью центра Интернета вещей и Azure Security Center. Чтобы включить защитник для Интернета вещей в центре Интернета вещей Azure, требуется учетная запись с правами уровня **владельца** . После включения ASC для IoT в центре Интернета вещей защитник для Azure IoT Insights отображается как функция **безопасности** в центре Интернета вещей и в качестве центра  **Интернета** вещей Azure.

## <a name="supported-service-regions"></a>Поддерживаемые регионы для службы

Сейчас защитник для IoT поддерживает центры Интернета вещей в следующих регионах Azure:

- Центральная часть США
- Восточная часть США
- восточная часть США 2
- центрально-западная часть США
- западная часть США
- Западная часть США 2
- Центрально-южная часть США
- Центрально-северная часть США
- Центральная Канада
- Восточная Канада
- Северная Европа
- Южная Бразилия
- Центральная Франция
- западная часть Соединенного Королевства
- южная часть Соединенного Королевства
- Западная Европа
- Северная Европа
- Западная Япония
- Japan East
- Australia Southeast
- Восточная Австралия
- Восточная Азия
- Юго-Восточная Азия
- Республика Корея, центральный регион
- Республика Корея, южный регион
- Центральная Индия
- Южная Индия

Защитник для Интернета вещей направляет весь трафик из всех стран Европейского региона в региональный центр обработки данных Западной Европы и все остальные регионы в Центральный Региональный центр обработки данных США.

## <a name="wheres-my-iot-hub"></a>Где находится центр Интернета вещей?

Перед началом работы проверьте расположение центра Интернета вещей, чтобы проверить доступность службы.

1. Откройте Центр Интернета вещей.
1. Нажмите **Обзор**.
1. Убедитесь, что указанное расположение соответствует одному из [поддерживаемых регионов службы](#supported-service-regions).

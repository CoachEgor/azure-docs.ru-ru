# <a name="getting-started-with-azure-defender-for-iot"></a>Приступая к работе с защитником Azure для IoT

В этой статье описываются процессы развертывания и адаптации, необходимые для запуска службы "защитник Azure для Интернета вещей". Также требуются дополнительные действия. Рекомендуется разобраться с этими шагами и ознакомиться с информацией в сопроводительных документах.

После выполнения всех шагов защитник Azure для датчиков Интернета вещей будет наблюдать за сетью. В зависимости от настройки решения обнаружения можно также отправлять в локальную консоль управления или в центр Интернета вещей.

Выполните следующие действия, чтобы скачать и запустить защитник Azure для IoT.

## <a name="1-set-up-azure"></a>1. Настройка Azure

- Настройте учетную запись Azure. Дополнительные сведения см. в статье [Создание учетной записи Azure](/learn/modules/create-an-azure-account/).

- Брандмауэр или прокси-сервер. Если у вас есть брандмауэр или аналогичное промежуточное сетевое устройство, которое настроено для разрешения конкретных подключений, проверьте, что в брандмауэре или прокси-сервере открыт параметр *. azure-devices.net:443. Если подстановочные знаки не поддерживаются или требуется более полный контроль, необходимо открыть конкретное полное доменное имя центра Интернета вещей в вашем FW или прокси-сервере. Дополнительные сведения см. в разделе [Reference-Endpoints центра Интернета вещей](../iot-hub/iot-hub-devguide-endpoints.md).

## <a name="2-deploy-hardware-software-and-onboard-to-sensor"></a>2. развертывание оборудования, программного обеспечения и подключения к датчику

- Приобретите оборудование датчика и установите программное обеспечение. Выполните действия, описанные здесь, а дополнительные сведения см. в этой статье и руководстве по [оборудованию "защитник для Интернета вещей"](https://aka.ms/AzureDefenderforIoTBareMetalAppliance) и [руководстве по установке](https://aka.ms/AzureDefenderforIoTInstallSensorISO).

  - После установки датчика безопасно запишите учетные данные для входа с датчика. Для передачи файла активации в датчик потребуются учетные данные.

  - При работе с датчиками, управляемыми локально, безопасно запишите IP-адрес датчика или имя датчика, определенное в установке. Его можно использовать при создании имени датчика во время регистрации датчика в защитнике для портала IoT. Их можно использовать позже, чтобы обеспечить более простое отслеживание и согласованное именование имен регистрации в службе "защитник Azure для Интернета вещей" и IP-адрес развернутого датчика, отображаемого в консоли датчика.

- Зарегистрируйте датчик с помощью защитника для портала IoT и скачайте файл активации датчика.

- Отправьте файл активации на датчик.

## <a name="3-perform-network-setup-for-sensor-monitoring-and-management"></a>3. Выполните настройку сети для мониторинга и управления датчиками.

- Подключите датчик к сети. Описано [в этом разделе](https://aka.ms/AzureDefenderForIoTNetworkSetup).

## <a name="4-start-discovering-your-network"></a>4. Начните обнаружение сети

- Настройте параметры системы в консоли датчика.

- Подключите датчики к локальной консоли управления.

Дополнительные сведения см. в разделе [Руководство пользователя "защитник Azure для IOT](https://aka.ms/AzureDefenderforIoTUserGuide) " и [Руководство пользователя консоли управления IOT для локальной среды](https://aka.ms/DefenderForIoTManagementConsole).

## <a name="5-populate-azure-sentinel-with-alert-information"></a>5. Заполнение Sentinel Azure сведениями о предупреждении

- Чтобы отправить сведения об оповещении в Azure Sentinel, настройте метку Azure: [подключение данных из защитника для Интернета вещей к Azure Sentinel](how-to-configure-with-sentinel.md).

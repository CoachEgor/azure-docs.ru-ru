# <a name="connect-your-data-from-defender-for-iot-to-azure-sentinel-preview"></a>Подключение данных из защитника для Интернета вещей к Azure Sentinel (Предварительная версия)

Azure Defender для соединителя данных IoT в Azure в настоящее время доступна в общедоступной предварительной версии. Эта функция предоставляется без соглашения об уровне обслуживания и не рекомендуется для рабочих нагрузок. Некоторые функции могут не поддерживаться или их возможности могут быть ограничены. Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/en-us/support/legal/preview-supplemental-terms/).

В этом руководство описано, как подключить защитник для данных IoT к Azure Sentinel.

> [!div class="checklist"]
> * Предварительные требования
> * Параметры подключения
> * Log Analytics представление предупреждений

Подключение оповещений из защитника для Интернета вещей и их потоковая передача непосредственно в Azure Sentinel.

Благодаря более тесной интеграции защитника Azure для IoT с Azure Sentinel, первой облачной SIEM и первой SIEM с собственными центрами Интернета вещей и безопасностью Корпорация Майкрософт предоставляет более простой подход к объединению безопасности в ИТ-и промышленных сетях. В сочетании с машинным обучением Sentinel в Azure эта интеграция позволяет организациям быстро обнаруживать многоступенчатыхные атаки, которые часто пересекают ее и границы. Кроме того, Azure Defender для интеграции IoT с возможностями оркестрации в отношении безопасности, автоматизации и реагирования Azure Sentinel (ВЗЛЕТЕЛ) позволяет автоматизировать реагирование и предотвращение с помощью встроенных оптимизированных модули PlayBook. 

## <a name="prerequisites"></a>Предварительные требования

- Необходимо иметь разрешения на **Чтение** и **запись** рабочей области.
- **Защитник для Интернета вещей** должен быть **включен** в соответствующих центрах Интернета вещей.
- У вас должны быть разрешения на **Чтение** и **запись** в **центре Интернета вещей Azure** , к которому вы хотите подключиться.
- У вас также должны быть разрешения на **Чтение** и **запись** в **группе ресурсов центра Интернета вещей Azure**.


## <a name="connect-to-defender-for-iot"></a>Подключение к защитнику для Интернета вещей

1. В поле Sentinel Azure выберите **соединители данных** , а затем щелкните плитку **защитник для Интернета вещей** .
1. В нижней части правой панели щелкните **открыть страницу соединителя**.
1. Нажмите кнопку **подключить**рядом с каждой подпиской центра Интернета вещей, предупреждениями и оповещениями устройств, которые вы хотите передавать в метку Azure.
    - Если защитник для IoT не включен в этом концентраторе, вы увидите сообщение Enable Warning (включить предупреждение). Щелкните ссылку **включить** , чтобы запустить и включить службу.
1. Вы можете решить, должны ли оповещения из защитника для Интернета вещей автоматически создавать инциденты в Azure Sentinel. В разделе **Создание инцидентов**выберите **включить** , чтобы включить правило автоматического создания инцидентов из созданных оповещений.  Это правило можно изменить или изменить в разделе **аналитика**  >  **Активные** правила.

> [!NOTE]
>Обновление списка концентраторов после внесения изменений в подключение может занять 10 или более секунд.

## <a name="using-log-analytics-for-alert-display"></a>Использование Log Analytics для вывода предупреждений

Чтобы использовать соответствующую схему в Log Analytics, чтобы отобразить защитник для оповещений IoT, выполните следующие действия.

1. Откройте **журналы**  >  **секуритинсигхтс**  >  **секуритялерт**или выполните поиск по запросу **секуритялерт**.
1. Фильтр, чтобы просмотреть оповещения только для защитника, созданные IoT, с помощью следующего фильтра ККЛ:

```kusto
SecurityAlert | where ProductName == "Defender for IoT"
```

### <a name="service-notes"></a>Заметки о службе

После подключения центра Интернета вещей данные концентратора доступны в Sentinel в течение приблизительно 15 минут позже.

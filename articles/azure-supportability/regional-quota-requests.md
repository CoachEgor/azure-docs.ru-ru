# <a name="total-regional-vcpu-limit-increase"></a>Общее региональное увеличение лимита виртуальных ЦП 

Диспетчер ресурсов квоты виртуальных ЦП для виртуальных машин и масштабируемых наборов виртуальных машин применяются на двух уровнях для каждой подписки в каждом регионе. 

Первый уровень — это **Общее виртуальных ЦП ограничение региональных ограничений** (для всех виртуальных машин), а второй уровень — **ограничение виртуальных ЦП для серии виртуальных машин** (например, виртуальных ЦП серии D). При каждом развертывании новой виртуальной машины сумма новых и существующих виртуальных ЦП использования для этого ряда виртуальных машин не должна превышать квоту виртуальных ЦП, утвержденную для этой конкретной серии виртуальных машин. Кроме того, общее число новых и существующих виртуальных ЦП, развернутых во всех рядах виртуальных машин, не должно превышать общее количество региональных виртуальных ЦП квоты, утвержденных для подписки. При превышении любой из этих квот развертывание виртуальной машины будет запрещено.
Вы можете запросить увеличение предельной квоты виртуальных ЦП для серии виртуальных машин из портал Azure. Увеличение квоты для серии виртуальных машин автоматически увеличивает общее значение региональных виртуальных ЦП на один и тот же объем. 

При создании подписки по умолчанию общий региональный виртуальных ЦП не может быть равен сумме квот виртуальных ЦП по умолчанию для всех отдельных рядов виртуальных машин. Это может привести к тому, что в подписке будет достаточно квоты для каждой отдельной серии виртуальных машин, которую требуется развернуть, но недостаточно квот для общего региона виртуальных ЦП для всех развертываний. В этом случае вам потребуется отправить запрос на увеличение общего регионального предела виртуальных ЦП явным образом. Общее ограничение региональных виртуальных ЦП не может превышать сумму утвержденной квоты во всех рядах виртуальных машин для региона.

Дополнительные сведения о квотах см. на странице [квоты виртуальных ЦП виртуальной машины](https://docs.microsoft.com/azure/virtual-machines/windows/quotas) , а также на странице [подписки Azure и ограничения службы](https://aka.ms/quotalimits) . 

Теперь вы можете запросить увеличение через колонку **Справка и поддержка** или колонку **использование и квота** на портале. 

## <a name="request-total-regional-vcpus-quota-increase-at-subscription-level-using-the-help--support-blade"></a>Запрос общего количества региональных виртуальных ЦП квоты на уровне подписки с помощью колонки " **Справка и поддержка** "

Следуйте приведенным ниже инструкциям, чтобы создать запрос на поддержку через колонку "Справка и поддержка" Azure, доступную в портал Azure. 

1. В https://portal.azure.com выберите **Справка и поддержка**.

![Справка и поддержка](./media/resource-manager-core-quotas-request/helpsupport.png)
 
2.  Выберите **Новый запрос в службу поддержки**. 

![Новый запрос на техническую поддержку](./media/resource-manager-core-quotas-request/newsupportrequest.png)

3. В раскрывающемся списке Тип проблемы выберите **пределы службы и подписки (квоты)** .

![Раскрывающийся список типов проблем](./media/resource-manager-core-quotas-request/issuetypedropdown.png)

4. Выберите подписку, которая требует увеличенную квоту.

![Выберите Новости подписки](./media/resource-manager-core-quotas-request/select-subscription-sr.png)
   
5. Выберите **другие запросы** в раскрывающемся списке **тип квоты** .

![куотатипе](./media/resource-manager-core-quotas-request/regional-quotatype.png)

6. В области **сведений** укажите дополнительные сведения в соответствии с приведенным ниже примером, чтобы упростить обработку запроса и продолжить создание вариантов. 
    1.  **Модель развертывания** — укажите "Диспетчер ресурсов"
    2.  **Запрошенный регион** — укажите требуемый регион, например Восточная часть США 2
    3.  **Новое предельное значение** — укажите новый предел для региона. Это не должно превышать сумму утвержденной квоты для отдельных семейств SKU для этой подписки

![куотадетаилс](./media/resource-manager-core-quotas-request/regional-details.png)

![Запросить увеличение](./media/resource-manager-core-quotas-request/request-increase.png)

5. Выберите **другие запросы** в раскрывающемся списке **тип квоты** .

# <a name="classic-deployment-model"></a>Классической модели развертывания

Классической модели развертывания является более старого поколения land режим развертывания Azure применяет квоту глобальных виртуальных процессоров для виртуальных машин и масштабируемых наборов виртуальных машин. Классической модели развертывания больше не рекомендуется и теперь заменены модели Resource Manager. Дополнительные сведения об этих двух развертывания моделей и преимущества Resource Manager см. страницу модели развертывания Resource Manager. При создании новой подписки, Квота по умолчанию виртуальных процессоров будет ей назначен. Любое время, когда новая виртуальная машина развертывается с помощью классической модели развертывания, сумма использования новых и существующих виртуальных ЦП во всех регионах, не должен превышать квоту на виртуальные ЦП, утвержденные для классической модели развертывания. Дополнительные сведения о квотах на подписку Azure и ограничения службы.

Вы можете запросить увеличение предела виртуальных ЦП для классической модели развертывания с помощью справки + колонке поддержки или использование + Квота на портале в колонке.

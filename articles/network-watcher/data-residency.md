# <a name="data-residency-for-the-azure-network-watcher"></a>Местонахождение данных для наблюдателя за сетями Azure
Наблюдатель за сетями Azure не хранит данные клиента, за исключением службы монитора подключений (Предварительная версия).


## <a name="connection-monitor-preview-data-residency"></a>Монитор подключения (Предварительная версия) местонахождение данных
В службе *монитора подключений (Предварительная версия)* хранятся данные клиентов. Эти данные автоматически сохраняются службой "наблюдатель за сетями" в одном регионе. Итак, *монитор подключений (Предварительная версия)* автоматически удовлетворяет требованиям местонахождение данных в регионе, включая, как указано в [центре управления безопасностью](https://azuredatacentermap.azurewebsites.net/).

## <a name="singapore-data-residency"></a>Местонахождение данных Сингапур

В Azure функция, позволяющая хранить данные клиентов в одном регионе, в настоящее время доступна только в регионе Юго-Восточной Азии (Сингапур) Азиатско-Тихоокеанский регион Geo. Для всех других регионов данные клиента хранятся в географическом виде. Дополнительные сведения см. в разделе [центр управления безопасностью](https://azuredatacentermap.azurewebsites.net/).

> [!Note]
> **Прошлая репликация** Клиенты имеют возможность хранить IP-адреса конечных пользователей с помощью экземпляра наблюдателя за сетями, чтобы наблюдатель за сетями мог отслеживать достижимость, задержку и изменения топологии сети, связанные с IP-адресами конечных пользователей. Эти IP-адреса конечных пользователей могут классифицироваться как данные клиента. По состоянию на 15 июля 2020, наблюдатель за сетями хранит эти данные в одном регионе. (Данные клиента больше не реплицируются в HK.) Эти данные больше не реплицируются в HK. Это данные клиента, которые были зашифрованы неактивных данных.
> 
> Если данные клиента были доступны сторонним лицом, это позволит сторонним производителям получить IP-адрес, но не предоставит сторонним пользователям доступ к компьютеру или устройству, связанным с этим IP-адресом. Наблюдатель за сетями считает, что третьим сторонам не обращались к данным клиентов в HK.

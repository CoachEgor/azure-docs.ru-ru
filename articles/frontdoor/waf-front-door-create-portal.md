# <a name="create-a-waf-policy-for-azure-front-door-by-using-the-azure-portal"></a>Создание политики WAF для передней дверцы Azure с помощью портал Azure

В этой статье описывается, как создать базовую политику брандмауэра веб-приложения Azure (WAF) и применить ее к ведущему узлу на передней дверце Azure.

## <a name="prerequisites"></a>Предварительные требования

Создайте профиль Front Door, следуя инструкциям, описанным в статье [Краткое руководство. Создание профиля Front Door для высокодоступного глобального веб-приложения](quickstart-create-front-door.md). 

## <a name="create-a-waf-policy"></a>Создание политики WAF

Сначала создайте базовую политику WAF с управляемым набором правил по умолчанию (DRS) с помощью портала. 

1. В верхней левой части экрана выберите **создать ресурс**> поиска для **WAF**> выберите **брандмауэр веб-приложения (Предварительная версия)** > выберите **создать**.
2. На вкладке **основы** на странице **Создание политики WAF** введите или выберите следующие сведения, примите значения по умолчанию для остальных параметров, а затем выберите **проверить и создать**:

    | Параметр                 | Значение                                              |
    | ---                     | ---                                                |
    | Subscription            |Выберите имя подписки на переднюю дверцу.|
    | Resource group          |Выберите имя группы ресурсов для передней дверцы.|
    | Имя политики             |Введите уникальное имя для политики WAF.|

   ![Создание политики WAF](./media/waf-front-door-create-portal/basic.png)

3. На вкладке **взаимосвязь** на странице **Создание политики WAF** выберите **добавить интерфейсный узел**, введите следующие параметры и нажмите кнопку **Добавить**:

    | Параметр                 | Значение                                              |
    | ---                     | ---                                                |
    | Передняя дверь              | Выберите имя профиля передней дверцы.|
    | Узел внешнего интерфейса           | Выберите имя узла передней дверцы, а затем щелкните **Добавить**.|
    
    > [!NOTE]
    > Если интерфейсный узел связан с политикой WAF, он отображается серым цветом. Сначала необходимо удалить интерфейсный узел из связанной политики, а затем повторно связать интерфейсный узел с новой политикой WAF.
1. Выберите **проверить и создать**, а затем щелкните **создать**.

## <a name="configure-waf-rules-optional"></a>Настройка правил WAF (необязательно)

### <a name="change-mode"></a>Изменить режим

При создании политики WAF политика WAF по умолчанию находится в режиме **обнаружения** . В режиме **обнаружения** WAF не блокирует запросы, вместо этого запросы, соответствующие правилам WAF, регистрируются в журналах WAF.
Чтобы увидеть WAF в действии, можно изменить параметры режима с " **Обнаружение** " на " **предотвращение**". В режиме **предотвращения** запросы, соответствующие правилам, которые определены в наборе правил по умолчанию (DRS), блокируются и регистрируются в журналах WAF.

 ![Изменить режим политики WAF](./media/waf-front-door-create-portal/policy.png)

### <a name="custom-rules"></a>Настраиваемые правила

Пользовательское правило можно создать, выбрав **Добавить настраиваемое правило** в разделе **Настраиваемые правила** . Откроется страница конфигурации настраиваемого правила. Ниже приведен пример настройки настраиваемого правила для блокировки запроса, если строка запроса содержит **блоккме**.

![Изменить режим политики WAF](./media/waf-front-door-create-portal/customquerystring2.png)

### <a name="default-rule-set-drs"></a>Набор правил по умолчанию (DRS)

Набор правил по умолчанию, управляемый Azure, включен по умолчанию. Чтобы отключить отдельное правило в группе правил, разверните правила в этой группе правил, установите **флажок** перед номером правила и нажмите кнопку **Отключить** на вкладке выше. Чтобы изменить типы действий для отдельных правил в наборе правил, установите флажок перед номером правила, а затем выберите вкладку **действие изменения** выше.

 ![Изменить набор правил WAF](./media/waf-front-door-create-portal/managed2.png)

### <a name="preventative"></a>профилактическая;

| Атрибут безопасности | Да/нет | Примечания |
|---|---|--|
| Шифрование при хранении:<ul><li>Шифрование на стороне сервера</li><li>Шифрование на стороне сервера с использованием управляемых клиентом ключей</li><li>Другие возможности шифрования (например, на стороне клиента, постоянное шифрование и т. д.)</ul>| Да (только шифрование на стороне службы) | Зашифрованные конфиденциальные данные, например сертификаты, ключи и значения с именем секрета с помощью службы управляемое за ключей экземпляра службы. |
| Шифрование при передаче:<ul><li>Шифрование ExpressRoute</li><li>При шифровании виртуальной сети</li><li>Шифрование между виртуальными сетями</ul>| Да | [Каналы Expressroute](../expressroute/index.yml) и виртуальной сети Шифрование обеспечивается корпорацией [сети Azure](../virtual-network/index.yml). |
| Управление ключами шифрования (CMK, BYOK, и т.д.)| Нет | Все ключи шифрования на экземпляр службы и управление службой. |
| Шифрование на уровне столбцов (службы данных Azure)| Н/Д | |
| Вызовы API в зашифрованном виде| Да | Вызовы плоскости управления выполняются через [Azure Resource Manager](../azure-resource-manager/index.yml) через TLS. Требуется допустимый веб-маркер JSON (JWT).  Вызовы плоскости данных могут быть защищены с помощью TLS и из механизмов проверки подлинности (например, сертификат клиента или JWT).
 |

См. в разделе [шифрованием службы хранилища Azure для неактивных данных](../storage/common/storage-service-encryption.md).<br><br>Секреты предоставленный клиента шифруются в неактивном состоянии. Секретные данные шифруются в неактивном состоянии, во время хранения в базах данных конфигурации службы приложений.<br><br>Локально подключенные диски, при необходимости может использоваться как временное хранилище, веб-сайтов (D:\local и % TMP %). Локально подключенные диски, не шифруются при хранении. |
| Шифрование при обмене данными (например, шифрование ExpressRoute в виртуальной сети шифрования и шифрование между сетями)| Да | Клиенты могут настроить веб-сайтов для запроса и использовать протокол HTTPS для входящего трафика. См. в записи блога [как осуществлять только HTTPS службы приложений Azure](https://blogs.msdn.microsoft.com/benjaminperkins/2017/11/30/how-to-make-an-azure-app-service-https-only/). |
| Управление ключами шифрования (CMK, BYOK, и т.д.)| Да | Клиенты могут выбрать для хранения секретов приложения в хранилище ключей и получения их во время выполнения. См. в разделе [ссылается на использование хранилища ключей Azure для службы приложений и функций Azure (Предварительная версия)](../app-service/app-service-key-vault-references.md).|
| Шифрование на уровне столбцов (службы данных Azure)| Н/Д | |
| Вызовы API в зашифрованном виде| Да | Происходят запросы управления настроить службу приложений с помощью [Azure Resource Manager](../azure-resource-manager/index.yml) вызовы по протоколу HTTPS. |

### <a name="network-segmentation"></a>сегментация сети;

| Атрибут безопасности | Да/нет | Примечания |
|---|---|--|
| Поддержка конечной точки службы| Да | В настоящее время доступна в предварительной версии для службы приложений. См. в разделе [ограничения доступа к службе приложений Azure](../app-service/app-service-ip-restrictions.md). |
| Внедрение поддержки виртуальной сети| Да | Среды службы приложений — частные реализации службы приложений, выделенном единственному заказчику, вставленных в виртуальной сети клиента. См. в разделе [введение в среду службы приложений](../app-service/environment/intro.md). |
| Поддержка сетевой изоляции и Firewalling| Да | Для общедоступной версии мультитенантное приложение службы клиенты могут настроить сетевые списки управления доступом (ограничения IP-адресов) для блокировки разрешенного входящего трафика.  См. в разделе [ограничения доступа к службе приложений Azure](../app-service/app-service-ip-restrictions.md).  Среды службы приложений развертываются непосредственно в виртуальных сетях, следовательно, могут быть защищены с помощью групп безопасности сети. |
| Принудительного туннелирования поддержки| Да | Среды службы приложений можно развернуть в виртуальной сети клиента, где настраивается Принудительное туннелирование. Клиенты должны следовать указаниям, приведенным в [Настройка среды службы приложений с помощью принудительного туннелирования](../app-service/environment/forced-tunnel-support.md). |

См. в разделе [службе приложений Azure монитор производительности](../azure-monitor/app/azure-web-apps.md). Служба приложений также отправляет метрики приложения в Azure Monitor. См. в разделе [мониторинг приложений в службе приложений Azure](../app-service/web-sites-monitor.md). |

### <a name="identity-and-access-management"></a>Управление удостоверениями и доступом

| Атрибут безопасности | Да/нет | Примечания|
|---|---|--|
| Authentication| Да | Клиенты могут создавать приложения в службе приложений, которые автоматически интегрируются с [Azure Active Directory (Azure AD)](../active-directory/index.yml) а также других поставщиков совместимых удостоверений OAuth; см. в разделе [проверки подлинности и авторизации в Служба приложений Azure](../app-service/overview-authentication-authorization.md). Для управления доступа к ресурсам службы приложений все доступ контролируется с помощью сочетания основной проверки подлинности Azure AD и ролей RBAC Azure Resource Manager. |
| Авторизация| Да | Для управления доступа к ресурсам службы приложений все доступ контролируется с помощью сочетания основной проверки подлинности Azure AD и ролей RBAC Azure Resource Manager.  |

|---|---|--|

Журналы этих операций доступны как на портале, так и с помощью интерфейса командной строки; см. в разделе [операции поставщиков ресурсов Azure Resource Manager](../role-based-access-control/resource-provider-operations.md#microsoftweb) и [az monitor-журнал действий](/cli/azure/monitor/activity-log). |

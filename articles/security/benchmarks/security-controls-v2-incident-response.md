# <a name="security-control-v2-incident-response"></a>Управление безопасностью v2: реагирование на инциденты

Реагирование на инциденты охватывает элементы управления в жизненном цикле реагирования на инциденты — подготовка, обнаружение и анализ, вложения и действия после инцидента. Сюда входит использование служб Azure, таких как центр безопасности Azure и Sentinel, для автоматизации процесса реагирования на инциденты.

## <a name="ir-1-preparation--update-incident-response-process-for-azure"></a>IR-1: подготовка — процесс обновления реагирования на инциденты для Azure

|--|--|--|--|
| IR-1 | 19 | IR — 4, IR-8 |

Убедитесь, что в Организации есть процессы для реагирования на инциденты безопасности, обновили эти процессы для Azure и регулярно исмерены их применение для обеспечения готовности.

- [Реализация безопасности в среде предприятия](https://aka.ms/AzSec4)

## <a name="ir-3-detection-and-analysis--create-incidents-based-on-high-quality-alerts"></a>IR-3. обнаружение и анализ — создание инцидентов на основе высококачественных оповещений

| Идентификатор Azure | ИДЕНТИФИКАТОРы элементов управления CIS v 7.1 | ИДЕНТИФИКАТОРы для директивы NIST SP800-53 |
|--|--|--|--|
| IR-3 | 19,6 | IR-4, IR-5 |

Убедитесь, что у вас есть процесс создания высококачественных оповещений и измерения качества оповещений. Это позволяет изучать уроки прошлых инцидентов и определять приоритеты для аналитиков, чтобы они не тратили время на ложные срабатывания. 

Высококачественные оповещения можно создавать на основе опыта прошлых инцидентов, проверенных источников сообщества и средств, предназначенных для создания и очистки оповещений путем отказа и корреляции различных источников сигналов. 

Центр безопасности Azure обеспечивает высококачественные оповещения во многих ресурсах Azure. Для потоковой передачи оповещений в Azure Sentinel можно использовать соединитель ASC Data Connector. Azure Sentinel позволяет создавать дополнительные правила генерации оповещений для автоматического создания инцидентов для расследования. 

Экспортируйте оповещения и рекомендации центра безопасности Azure с помощью функции экспорта, чтобы определить риски для ресурсов Azure. Экспортируйте оповещения и рекомендации либо вручную, либо непрерывно.

- [Настройка экспорта](../../security-center/continuous-export.md)

- [Как выполнить потоковую передачу оповещений в Azure Sentinel](../../sentinel/connect-azure-security-center.md)

## <a name="ir-4-detection-and-analysis--investigate-an-incident"></a>IR-4. обнаружение и анализ — исследование инцидента

| Идентификатор Azure | ИДЕНТИФИКАТОРы элементов управления CIS v 7.1 | ИДЕНТИФИКАТОРы для директивы NIST SP800-53 |
|--|--|--|--|
| IR-4 | 19 | IR-4 |

Убедитесь, что аналитики могут запрашивать и использовать различные источники данных при исследовании потенциальных инцидентов, чтобы создать полное представление о том, что произошло. Для наблюдения за действиями потенциального злоумышленника в цепочке аннулирования необходимо собирать различные журналы, чтобы избежать слепых точек.  Кроме того, следует убедиться, что аналитические данные и сведения о них записываются для других аналитиков и для будущих исторических справочников.  

Источники данных для исследования включают в себя централизованные источники журналов, которые уже собираются из служб, выполняемых в области, и работающих систем, но могут также включать:

- Сетевые данные — используйте журналы потоков для групп безопасности сети, наблюдателя за сетями Azure и Azure Monitor для записи журналов потоков сети и других данных аналитики. 

- Моментальные снимки выполняющихся систем: 

    - Используйте возможности моментального снимка виртуальной машины Azure для создания моментального снимка диска работающей системы. 

    - Используйте встроенную функцию дампа памяти операционной системы для создания моментального снимка памяти работающей системы.

    - Используйте функцию моментального снимка служб Azure или собственную возможность создания моментальных снимков работающих систем.

Azure Sentinel предоставляет широкие возможности анализа данных практически на любом источнике журнала и на портале управления вариантами для управления полным жизненным циклом инцидентов. Данные аналитики во время расследования можно связать с инцидентом для отслеживания и отчетности. 

- [Моментальный снимок диска компьютера Windows](../../virtual-machines/windows/snapshot-copy-managed-disk.md)

- [Моментальный снимок диска компьютера Linux](../../virtual-machines/linux/snapshot-copy-managed-disk.md)

- [Microsoft Azure поддержка сбора диагностических данных и дампа памяти](https://azure.microsoft.com/support/legal/support-diagnostic-information-collection/) 

- [Изучение инцидентов с помощью Azure Sentinel](../../sentinel/tutorial-investigate-cases.md)

## <a name="ir-5-detection-and-analysis--prioritize-incidents"></a>IR-5: обнаружение и анализ — определение приоритета инцидентов

| Идентификатор Azure | ИДЕНТИФИКАТОРы элементов управления CIS v 7.1 | ИДЕНТИФИКАТОРы для директивы NIST SP800-53 |
|--|--|--|--|
| IR-5 | 19,8 | ЦС-2, IR-4 |

Укажите контекст для аналитиков о том, на каких инцидентах следует сосредоточиться в первую очередь, исходя из серьезности предупреждения и чувствительности к ресурсу. 

Центр безопасности Azure назначает серьезность каждому оповещению, чтобы помочь вам определить, какие предупреждения следует изучить первыми. Серьезность основывается на том, насколько надежно центр безопасности находится в поиске или аналитике, используемой для выдаче оповещения, а также на уровне достоверности, который был вредоносной задачей, вызвавшей оповещение.

Кроме того, пометьте ресурсы с помощью тегов и создайте систему именования для обнаружения и классификации ресурсов Azure, особенно для обработки конфиденциальных данных.  Вы несете ответственность за назначение приоритета оповещениям, требующим действий по исправлению, в зависимости от важности ресурсов Azure и среды, в которой произошел инцидент.

## <a name="ir-6-containment-eradication-and-recovery--automate-the-incident-handling"></a>IR-6: вложение, удаление и восстановление — Автоматизируйте обработку инцидентов.

| Идентификатор Azure | ИДЕНТИФИКАТОРы элементов управления CIS v 7.1 | ИДЕНТИФИКАТОРы для директивы NIST SP800-53 |
|--|--|--|--|
| IR-6 | 19 | IR-4, IR-5, IR-6 |

Автоматизируйте повторяющиеся задачи вручную, чтобы ускорить время отклика и снизить нагрузку на аналитики. Ручные задачи выполняются дольше, замедление каждого инцидента и уменьшается количество инцидентов, которые может обработать аналитик. Задачи, выполняемые вручную, также расширяют аналитику выносливости, что повышает риск возникновения ошибок, вызванных человеческими ошибками, и снижает способность аналитиков эффективно сосредоточиться на сложных задачах. Используйте функции автоматизации рабочих процессов в центре безопасности Azure и метку Azure, чтобы автоматически активировать действия или запустить сборник тренировочных заданий для реагирования на входящие оповещения системы безопасности. Сборник тренировочных заданий выполняет такие действия, как отправка уведомлений, отключение учетных записей и изоляция проблемных сетей.

- [Настройка автоматизированных ответов угроз в центре безопасности Azure](../../security-center/tutorial-security-incident.md#triage-security-alerts)

# <a name="azure-security-benchmark-introduction"></a>Внедрение теста безопасности Azure

Вы можете иметь несколько лет или даже десятилетий опыта работы с корпоративными вычислениями. Вы знаете, как обеспечить эти развертывания; но облако отличается. Как узнать, безопасны ли развертывание облачных технологий? В чем разница между практиками безопасности для систем и облачных развертываний?

Существует большая коллекция информационных документов, передовой практики, эталонных архитектур, веб-руководства, инструментов с открытым исходным кодом, коммерческих решений, интеллектуальных каналов и многое другое, которые могут быть использованы для защиты облака. Какой вариант следует использовать? Что вы можете сделать, чтобы получить приемлемый уровень безопасности в облаке? 

Одним из лучших способов обеспечения развертывания в облаке является сосредоточение внимания на рекомендациях по тестовой облачной безопасности. Рекомендации по бенчмарку для обеспечения безопасности любого сервиса начинаются с фундаментального понимания риска кибербезопасности и того, как с ним управлять. Затем вы можете использовать это понимание, приняв рекомендации по безопасности от поставщика облачных услуг, чтобы помочь выбрать определенные параметры конфигурации безопасности в вашей среде. 

Контрольный ориентир безопасности Azure включает в себя набор рекомендаций по безопасности с высокой отдачей, которые можно использовать для обеспечения безопасности большинства служб, которые вы используете в Azure. Эти рекомендации можно считать «общими» или «организационными», поскольку они применимы к большинству служб Azure. Рекомендации Azure Security Benchmark затем настраиваются для каждой службы Azure, и это индивидуальное руководство содержится в статьях рекомендаций по обслуживанию. 

В документации Azure Security Benchmark указаны рекомендации по обеспечению безопасности и рекомендации службы.

- **Элементы управления безопасностью**: Рекомендации Azure Security Benchmark классифицируются по элементам управления безопасностью. Элементы управления безопасностью представляют собой требования безопасности высокого уровня, такие как сетевая безопасность и защита данных. Каждый контроль безопасности имеет набор рекомендаций и инструкций по безопасности, которые помогут вам реализовать эти рекомендации. 
- **Рекомендации по обслуживанию:** При наличии базовых рекомендаций для служб Azure будут содержаться рекомендации Azure Security Benchmark, специально разработанные специально для этой службы. 

Термины «Контроль», «Бенчмарк» и «Базовый» часто используются в документации Azure Security Benchmark, и важно понимать, как Azure использует эти термины. 

| Термин | Описание | Пример |
|--|--|--|
| Control | **Контроль** — это описание объекта или действия высокого уровня, которые необходимо решать и которые не специфичны для технологии или реализации. | Защита данных является одним из элементов управления безопасностью. Этот элемент управления содержит конкретные действия, которые необходимо адресовать, чтобы обеспечить защиту данных. |
| Тест производительности | **Ориентир** содержит рекомендации по безопасности для конкретной технологии, такой как Azure. Рекомендации классифицируются по элементу управления, к которому они принадлежат. | В бенчмаркaz Azure Security содержатся рекомендации по безопасности, характерные для платформы Azure  |
| Базовый | Базовым **уровнем** являются требования к безопасности для организации. Требования к безопасности основаны на рекомендациях, выбываемых на основе базовых рекомендаций. Каждая организация решает, какие рекомендации следует включить в свои базовые показатели. | Компания Contoso создает свою базу безопасности, выбирая для этого конкретные рекомендации в тесте безопасности Azure. |

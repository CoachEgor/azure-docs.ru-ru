# <a name="azure-security-benchmarks-introduction"></a>Общие сведения о производительности системы безопасности Azure

У вас может быть несколько лет или даже десятилетий опыта работы с локальными вычислениями. Вы умеете защищать эти развертывания. Но облако отличается. Как узнать, безопасны ли облачные развертывания? Каковы различия между методиками безопасности для локальных систем и методов обеспечения безопасности для облачных развертываний?

Существует обширный набор технических документов, рекомендаций, эталонных архитектур, веб-руководств, средств с открытым кодом, коммерческих решений, веб-каналов аналитики и т. д., которые можно использовать для защиты облака. Какой вариант следует использовать? Что можно сделать для получения приемлемого уровня безопасности в облаке? 

Один из лучших способов защиты облачных развертываний — сосредоточиться на рекомендациях по повышению производительности Cloud Security. Рекомендации по тестированию для защиты любой службы начинаются с фундаментального понимания кибербезопасности риска и способов управления им. Затем это понимание можно использовать, применив рекомендации по обеспечению безопасности от поставщика облачных служб, чтобы помочь выбрать определенные параметры конфигурации безопасности в вашей среде. 

В состав теста безопасности Azure входит набор рекомендаций по обеспечению безопасности, которые можно использовать для защиты большинства служб, используемых в Azure. Эти рекомендации можно рассматривать как "Общие" или "организационные", так как они применимы к большинству служб Azure. Затем рекомендации по производительности системы безопасности Azure настраиваются для каждой службы Azure, и эти настраиваемые инструкции содержатся в статьях с рекомендациями по службам. 

В документации по производительности системы безопасности Azure указаны средства управления безопасностью и рекомендации по службам.

- Средства **управления безопасностью**. рекомендации по производительности системы безопасности Azure подразделяются на категории элементов управления безопасностью. Средства управления безопасностью представляют высокоуровневые требования к безопасности, независимые от поставщиков, такие как безопасность сети и защита данных. Каждый контроль безопасности имеет набор рекомендаций по обеспечению безопасности и инструкций, помогающих включить эти рекомендации. 
- **Рекомендации по службам**. при наличии рекомендаций по тестированию производительности служб Azure будут включены рекомендации по производительности системы безопасности Azure, предназначенные для службы, а также дополнительные рекомендации, уникальные для конкретной службы. 

Термины "Управление", "тест производительности" и "базовый" часто используются в документации по производительности системы безопасности Azure. Важно понимать, как Azure использует эти термины. 

| Срок действия | Description | Пример |
|--|--|--|
| Control | **Элемент управления** — это высокоуровневое описание функции или действия, которое необходимо решить, и не относится к технологии или реализации. | Защита данных — один из средств управления безопасностью. Этот элемент управления содержит конкретные действия, которые необходимо решить, чтобы обеспечить защиту данных. |
| Тест производительности | **Тест производительности** содержит рекомендации по безопасности для конкретной технологии, например Azure. Рекомендации классифицируются по элементу управления, к которому они относятся. | Производительность системы безопасности Azure состоит из рекомендаций по обеспечению безопасности, характерных для платформы Azure.  |
| Базовый показатель | **Базовые показатели** — требования к безопасности для Организации. Требования к безопасности основаны на рекомендациях по производительности. Каждая организация решает, какие рекомендации по производительности следует включить в их базовые показатели. | Компания Contoso создает базовый уровень безопасности, выбирая конкретные рекомендации в тестовом плане безопасности Azure. |

# <a name="overview-of-azure-security-baselines"></a>Обзор базовых показателей безопасности Azure

Базовые показатели безопасности Azure помогают усилить безопасность наших продуктов за счет улучшенных средств, отслеживания и обеспечения безопасности, а также обеспечивают единообразную работу при защите среды.

Базовые показатели безопасности службы Azure ориентированы на облачные области управления. Эти элементы управления согласуются с хорошо известными тестами безопасности, например, описанными в центре безопасности Интернета (CIS). Наши базовые планы предоставляют рекомендации по управлению областями, перечисленными в статье о [производительности системы безопасности Azure](overview.md).

Каждая рекомендация включает следующие сведения:
- **Идентификатор Azure**: идентификатор теста безопасности Azure, соответствующий рекомендациям.
- **Рекомендация**: непосредственно после идентификатора Azure в рекомендации представлено подробное описание элемента управления.
- **Руководство**. обоснование для рекомендации и ссылки на рекомендации по ее внедрению. Если рекомендация поддерживается центром безопасности Azure, эти сведения также будут перечислены в списке.
- **Ответственность**: кто отвечает за реализацию элемента управления. Возможными сценариями являются ответственность за клиента, ответственность за корпорацию Майкрософт или коллективная ответственность.
- **Мониторинг центра безопасности Azure**: управляет ли элемент управления центром безопасности Azure со ссылкой на ссылку.

Все рекомендации, включая рекомендации, которые не применяются к данной конкретной службе, включены в базовую базу, чтобы получить полную картину того, как тест производительности системы безопасности Azure связан с каждой службой. Иногда могут быть элементы управления, неприменимые по различным причинам. Например, элементы управления IaaS или COMPUTE (такие как элементы управления конфигурацией ОС) могут быть неприменимы к службам PaaS.

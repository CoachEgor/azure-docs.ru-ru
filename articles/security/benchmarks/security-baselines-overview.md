# <a name="overview-of-azure-security-baselines"></a>Обзор базовых уровней безопасности Azure

Базовые системы безопасности Azure помогают повысить безопасность наших продуктов за счет улучшения инструментария, отслеживания и функций безопасности и обеспечивают вам согласованный опыт при обеспечении безопасности среды.

Базовые направления безопасности службы Azure сосредоточены на облачных областях управления. Эти элементы управления соответствуют хорошо известным критериям безопасности, таким, как те, которые описаны Центром интернет-безопасности (CIS). Наши базовые стандарты содержат рекомендации для областей управления, перечисленных в [тесте безопасности Azure.](overview.md)

Каждая рекомендация включает в себя следующую информацию:
- **Идентификатор Azure:** Идентификатор теста безопасности Azure, соответствующий рекомендации.
- **Рекомендация**: После идентификатора Azure рекомендация содержит высокоуровневое описание элемента управления.
- **Руководство**: Обоснование рекомендации и ссылки на руководство о том, как ее реализовать. Если рекомендация поддерживается Центром безопасности Azure, эта информация также будет указана.
- **Ответственность**: Кто отвечает за реализацию элемента управления. Возможными сценариями являются ответственность клиента, ответственность корпорации Майкрософт или общая ответственность.
- **Мониторинг Центра безопасности Azure:** контролируется ли управление Центром безопасности Azure со ссылкой на ссылку.

Все рекомендации, включая рекомендации, не применимые к данной службе, включены в базовый уряд, чтобы предоставить вам полную картину того, как контрольный показатель безопасности Azure относится к каждой службе. Иногда могут быть элементы управления, которые не применимы по различным причинам, например, элементы управления IaaS/compute-centric (например, элементы управления, характерные для управления конфигурацией ОС), могут быть неприменимы к службам PaaS.

# <a name="container-security-in-microsoft-azure"></a>Защита контейнеров в Microsoft Azure
## <a name="abstract"></a>Аннотация

Технология контейнеров приводит к структурным изменениям в среде облачных вычислений. Контейнеры позволяют запускать несколько экземпляров приложения в одном экземпляре операционной системы, тем самым используя ресурсы более эффективно. Контейнеры обеспечивают для организаций согласованность и гибкость. Они обеспечивают непрерывное развертывание, так как приложение может быть разработано на настольном ПК, протестировано на виртуальной машине, а затем развернуто для производства в облаке. Контейнеры обеспечивают гибкость, ускоренные операции, масштабируемость и снижение затрат за счет оптимизации ресурсов.

Так как технология контейнеров является относительно новой, у многих ИТ-специалистов возникают опасения касательно безопасности ввиду отсутствия видимости и использования в рабочей среде. Команды разработчиков часто не знают о рекомендациях по обеспечению безопасности. Этот технический документ поможет командам по безопасности и разработчикам выбрать варианты защиты разработки и развертывания контейнеров на платформе Microsoft Azure.

В этом документе описываются контейнеры, их развертывание и управление ими, собственные службы платформы, а также рассматриваются проблемы безопасности среды выполнения, возникающие при использовании контейнеров на платформе Azure. На рисунках и в примерах этого документа Docker рассматривается как модель контейнеров, а Kubernetes — как оркестратор контейнеров. Большинство рекомендаций по обеспечению безопасности также применимы к другим моделям контейнеров от партнеров Майкрософт на платформе Azure.

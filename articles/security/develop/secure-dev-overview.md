# <a name="secure-development-best-practices-on-azure"></a>Разработка безопасных рекомендации по Azure
Эта серия статей представляется действия безопасности и элементы управления, которые следует учитывать при разработке приложений для облака. Рассматриваются этапы Microsoft Security Development Lifecycle (SDL) и контрольные вопросы, а также основные понятия, которые следует учитывать во время каждой фазы жизненного цикла. Призван помочь вам определить действия и служб Azure, которые можно использовать на каждом этапе жизненного цикла для проектирования, разработки и развертывания более безопасных приложений.

Рекомендации в статьях основаны на нашем опыте безопасности Azure и из опыта наших клиентов. Эти статьи можно использовать как ссылку для что следует учитывать при определенном этапе проекта разработки, но мы рекомендуем также ознакомиться с через все статьи от начала до конца по крайней мере один раз. Чтение всех статей рассказывается о концепции, которые вы не решены на ранних этапах проекта. Реализация этих концепций перед выпуском продукта может помочь построения безопасного программного обеспечения, обеспечить соответствие требованиям безопасности и снизить затраты на разработку.

Эти статьи предназначены для ресурсов для разработчиков программного обеспечения, разработчиков и инженеров-испытателей на всех уровнях, создания и развертывания безопасных приложений Azure.

## <a name="overview"></a>Обзор

Безопасность — один из самых важных аспектов любое приложение, и это не простой операции трудной. К счастью Azure предоставляет множество служб, которые помогут вам обеспечить безопасность приложения в облаке. В этих статьях адресов действий и служб Azure, можно реализовать на каждом этапе жизненного цикла разработки программного обеспечения для разработки более безопасного кода и развертыванию более безопасных приложений в облаке.

## <a name="security-development-lifecycle"></a>Жизненный цикл разработки безопасности

Следуя рекомендациям для разработки безопасного программного обеспечения требуется интеграция безопасности в каждом этапе жизненного цикла разработки программного обеспечения, от проанализировать требования к обслуживанию, независимо от того, проектную методологию ([каскадная](https://en.wikipedia.org/wiki/Waterfall_model), [agile](https://en.wikipedia.org/wiki/Agile_software_development), или [DevOps](https://en.wikipedia.org/wiki/DevOps)). В пробуждения подобного важные данные и эксплуатация операционной безопасности недостатки большему числу разработчиков понимание того, что безопасность должна быть рассмотрена в процессе разработки.

Позже решить проблему в жизненном цикле разработки, чем больше, исправление будет стоить вам. Проблемы безопасности не являются исключением. Если не обращать внимания на проблемы безопасности на ранних этапах разработки программного обеспечения, каждый этап, следующий за может наследовать уязвимостей на предыдущем этапе. Окончательная версия продукта будет накопиться несколько проблем безопасности и вероятность возникновения нарушения безопасности. Внедрение безопасности в каждом этапе жизненного цикла разработки помогает выявление проблем на ранней стадии, и помогает снизить затраты на разработку.

Мы выполните этапы Microsoft [Security Development Lifecycle (SDL)](https://msdn.microsoft.com/library/windows/desktop/84aed186-1d75-4366-8e61-8d258746bopq.aspx) обсуждение действий и служб Azure, которые можно использовать для выполнения на каждом этапе жизненного цикла разработки безопасного программного обеспечения.

Они перечислены SDL:

  - Обучение
  - Требования
  - Проектирование
  - Реализация
  - Проверка
  - Release
  - Ответ

![Жизненный цикл разработки безопасности](./media/secure-dev-overview/01-sdl-phase.png)

В следующих статьях мы группирования SDL этапов проектирования, разработки и развертывания.

## <a name="engage-your-organizations-security-team"></a>Обратиться в службу безопасности вашей организации

Формальное приложение программы безопасности, которая поможет вам с действия безопасности от начала до конца во время жизненного цикла разработки может сталкиваться ваша организация. Если ваша организация использует групп безопасности и соответствия, не забудьте подключить их, прежде чем начать разработку вашего приложения. Попросите каждого этапа SDL существуют ли все задачи, которые вы пропустили.

Мы понимаем, что многие читатели могут отсутствовать группой безопасности и соответствия требованиям для взаимодействия. Эти статьи могут быть полезными в вопросы безопасности и решения, которые необходимо учитывать на каждом этапе SDL.

## <a name="resources"></a>Ресурсы

Дополнительные сведения о разработке безопасных приложений и защиты приложений в Azure, используйте следующие ресурсы:

[Microsoft Security Development Lifecycle (SDL)](https://msdn.microsoft.com/library/windows/desktop/84aed186-1d75-4366-8e61-8d258746bopq.aspx) — SDL — процесс разработки программного обеспечения от корпорации Майкрософт, позволяющее разработчикам встраивать более безопасного программного обеспечения. Это поможет вам обеспечить соответствие требованиям безопасности, одновременно снижая затраты на разработку.

[Откройте Web Application Security Project (OWASP)](https://www.owasp.org/index.php/Main_Page) — OWASP — это Интернет-сообщество, создающее свободно статей, методологии, документацию, средства и технологии в области безопасности веб-приложений.

[Помещает слева, например начальника](https://code.likeagirl.io/pushing-left-like-a-boss-part-1-80f1f007da95?WT.mc_id=docs-blog-tajanca) — это ряд статьи в Интернете, представлены различные типы действий безопасности приложения, которые разработчики должны выполнить для создания более безопасного программного кода.

[Платформы удостоверений Microsoft](https://docs.microsoft.com/azure/active-directory/develop/) — платформой Microsoft identity является результатом эволюционного развития платформы службы и разработчик удостоверений Azure AD. Это полнофункциональная платформа, состоящая из службы проверки подлинности, библиотек с открытым кодом, Регистрация приложения и конфигурации, весь документов для разработчиков, примеры кода и другое содержимое разработчика. Платформе Microsoft identity поддерживает стандартные отраслевые протоколы OAuth 2.0 и OpenID Connect.

[Рекомендации по безопасности для решений Azure](https://azure.microsoft.com/resources/security-best-practices-for-azure-solutions/) — коллекция рекомендации по обеспечению безопасности для использования при проектировании, развертывании и управлять облачных решений с помощью Azure. Этот документ предназначен ресурс для ИТ-специалистов. К ним могут относиться проектировщики, архитекторы, разработчики и тестировщики, занимающиеся созданием и развертыванием безопасных решений в Azure.

[Безопасность и соответствие проекты в Azure](https://servicetrust.microsoft.com/ViewPage/BlueprintOverview) — Azure безопасность и соответствие проекты — это ресурсы, которые помогут вам построение и запуск облачных приложений, которые соответствуют строгим нормативным требованиям и стандартам.

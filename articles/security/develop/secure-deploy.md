# <a name="deploy-secure-applications-on-azure"></a>Развертывание безопасных приложений в Azure
В этой статье мы представляем действия и элементы управления безопасности, чтобы учесть, когда развертываются приложения для облака. Рассматриваются вопросы безопасности и концепции, которые необходимо учитывать на этапах выпуска и ответа [на жизненном цикле разработки microsoft Security Development (SDL).](https://msdn.microsoft.com/library/windows/desktop/84aed186-1d75-4366-8e61-8d258746bopq.aspx) Цель состоит в том, чтобы помочь определить действия и службы Azure, которые можно использовать для развертывания более безопасного приложения.

Следующие фазы SDL описаны в этой статье:

- Release
- Ответ

## <a name="release"></a>Release
Основное внимание на этапе выпуска является подготовка проекта для публичного выпуска.
Это включает в себя планирование способов эффективного выполнения задач после выпуска обслуживания и устранения уязвимостей в системе безопасности, которые могут возникнуть позже.

### <a name="check-your-applications-performance-before-you-launch"></a>Проверьте производительность приложения перед запуском

Проверьте производительность приложения перед запуском или развернуть обновления в производство. Запустите облачные [тесты нагрузки](https://www.visualstudio.com/docs/test/performance-testing/getting-started/getting-started-with-performance-testing) с помощью Visual Studio, чтобы найти проблемы с производительностью в приложении, улучшить качество развертывания, убедиться, что ваше приложение всегда работает или доступно, и что ваше приложение может обрабатывать трафик для вашего запуска.

### <a name="install-a-web-application-firewall"></a>Установка брандмауэра веб-приложения

Веб-приложения все чаще подвергаются вредоносным атакам, использующим общеизвестные уязвимости. Среди этих эксплойтов являются атаки на инъекционные удары и атаки на кросс-сайт ыгостевых сценариев. Предотвращение этих атак в коде приложения может быть сложной задачей. Это может потребовать строгого обслуживания, исправления и мониторинга на многих уровнях топологии приложений. Централизованный WAF помогает упростить управление безопасностью. Решение WAF также может реагировать на угрозу безопасности, исправляя известную уязвимость в центральном месте по сравнению с обеспечением безопасности каждого отдельного веб-приложения.

[WAF приложений Azure](../../application-gateway/waf-overview.md) обеспечивает централизованную защиту веб-приложений от общих эксплойтов и уязвимостей. WAF основан на правилах из [основных правил OWASP 3.0](https://www.owasp.org/index.php/Category:OWASP_ModSecurity_Core_Rule_Set_Project) или 2.2.9.

### <a name="create-an-incident-response-plan"></a>Составление плана реагирования на инциденты

Подготовка плана реагирования на инциденты имеет решающее значение для решения новых угроз, которые могут возникнуть с течением времени. Подготовка плана реагирования на инциденты включает в себя выявление соответствующих контактов по чрезвычайным ситуациям безопасности и разработку планов обслуживания безопасности для кода, унаследованного от других групп организации, и лицензированного стороннего кода.

### <a name="conduct-a-final-security-review"></a>Проведение окончательного обзора безопасности

Утомленный просмотр всех действий по обеспечению безопасности, которые были выполнены, помогает обеспечить готовность к выпуску или приложению программного обеспечения. Окончательный обзор безопасности (FSR) обычно включает изучение моделей угроз, выводов инструментов и производительности в отношении стробов качества и баров ошибок, которые были определены на этапе требований.

### <a name="certify-release-and-archive"></a>Сертификация выпуска и архива

Сертификация программного обеспечения перед выпуском помогает обеспечить соответствие требованиям безопасности и конфиденциальности. Архивирование всех соответствующих данных имеет важное значение для выполнения задач после освобождения обслуживания. Архивирование также помогает снизить долгосрочные затраты, связанные с устойчивой разработкой программного обеспечения.

## <a name="response"></a>Ответ
Фаза ответа после релиза сосредоточена на том, что команда разработчиков может и доступна для адекватного реагирования на любые сообщения о возникающих угрозах и уязвимостях программного обеспечения.

### <a name="execute-the-incident-response-plan"></a>Выполнение плана реагирования на инциденты

Возможность реализации плана реагирования на инциденты, разработанного на этапе выпуска, имеет важное значение для защиты клиентов от возникающих уязвимостей программного обеспечения или уязвимостей конфиденциальности.

### <a name="monitor-application-performance"></a>Мониторинг производительности приложений

Постоянный мониторинг приложения после его развертывания потенциально помогает обнаруживать проблемы с производительностью, а также уязвимости безопасности.
Службы Azure, помогающие в мониторинге приложений:

  - Azure Application Insights
  - Центр безопасности Azure

#### <a name="application-insights"></a>Application Insights

[Application Insights](../../azure-monitor/app/app-insights-overview.md) — это расширяемый сервис управления производительностью приложений (APM) для веб-разработчиков на нескольких платформах. Используйте ее для мониторинга вашего работающего веб-приложения. Application Insights автоматически обнаруживает аномалии производительности. Она включает в себя мощные инструменты аналитики, которые помогут вам диагностировать проблемы и понять, что пользователи на самом деле делают с вашим приложением. Эта служба помогает постоянно улучшать производительность и удобство использования.

#### <a name="azure-security-center"></a>Центр безопасности Azure

[Центр безопасности Azure](../../security-center/security-center-intro.md) помогает предотвращать, обнаруживать и реагировать на угрозы с повышенной видимостью (и контролем) безопасности ресурсов Azure, включая веб-приложения. Центр безопасности Azure помогает обнаруживать угрозы, которые в противном случае могли бы остаться незамеченными. Он работает с различными решениями безопасности.

Центр безопасности ценовой категории "Бесплатный" предлагает ограниченные возможности безопасности только для ваших ресурсов Azure. [Стандартный уровень Центра безопасности](../../security-center/security-center-onboarding.md) расширяет эти возможности на ресурсы и другие облака.
Стандарт Центра безопасности поможет вам:

  - Найдите и исправьте уязвимости безопасности.
  - Применяйте элементы доступа и приложения для блокирования вредоносных действий.
  - Обнаружение угроз с помощью аналитики и разведки.
  - Быстро реагируйте при атаке.

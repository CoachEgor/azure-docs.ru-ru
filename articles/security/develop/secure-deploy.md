# <a name="deploy-secure-applications-on-azure"></a>Развертывайте безопасные приложения в Azure
В этой статье рассматриваются действия безопасности и элементы управления, которые следует учитывать при развертывании приложений для облака. Вопросы безопасности и основные понятия, которые следует учитывать на этапах выпуска и ответ Microsoft [Security Development Lifecycle (SDL)](https://msdn.microsoft.com/library/windows/desktop/84aed186-1d75-4366-8e61-8d258746bopq.aspx) рассматриваются. Призван помочь вам определить действия и служб Azure, которые можно использовать для развертывания более безопасных приложений.

В этой статье рассматриваются следующие этапы SDL:

- Release
- Ответ

## <a name="release"></a>Release
Цель этапа выпуска готовит проект для общедоступного выпуска.
Это включает в себя Планирование способов для эффективного выполнения задач обслуживания после выпуска и устранения уязвимостей безопасности, которые могут возникнуть более поздней версии.

### <a name="check-your-applications-performance-before-you-launch"></a>Проверка производительности приложения перед запуском

Проверьте производительность приложения, прежде чем запустить его или развернуть обновления в рабочей среде. Запустите облачную [нагрузочные тесты](https://www.visualstudio.com/docs/test/performance-testing/getting-started/getting-started-with-performance-testing) с помощью Visual Studio для поиска проблем с производительностью в приложении, повысить качество развертывания, убедитесь, что приложение всегда остается работу или доступность, и что приложение может обрабатывать трафик к выпуску.

### <a name="install-a-web-application-firewall"></a>Установка брандмауэра веб-приложения

Веб-приложения все чаще подвергаются вредоносным атакам, использующим общеизвестные уязвимости. Атаки путем внедрения кода SQL и атак с использованием межузловых сценариев являются общими для этих уязвимостей. Предотвращение таких атак в коде приложения может оказаться сложной задачей. Оно требует неукоснительного обслуживания, установки исправлений и мониторинга на нескольких уровнях топологии приложения. Централизованный Брандмауэр помогает упростить управление безопасностью. Решение WAF может реагировать на угрозы безопасности по установке исправлений известных уязвимостей в центральном расположении сравнению с защитой каждого отдельного веб-приложения.

[WAF шлюза приложений Azure](https://docs.microsoft.com/azure/application-gateway/waf-overview) обеспечивающий централизованную защиту веб-приложений от распространенных эксплойтов и уязвимостей. WAF на основе правил из [основных наборов правил OWASP](https://www.owasp.org/index.php/Category:OWASP_ModSecurity_Core_Rule_Set_Project) версии 3.0 или 2.2.9.

### <a name="create-an-incident-response-plan"></a>Создание плана реагирования на происшествия

Подготовка плана реагирования на происшествия крайне важны для решения новых угроз, которые могут появляются со временем. Подготовка плана реагирования на происшествия включающего определение родственников безопасности и установления обслуживания планы для кода, который наследуется от других групп в организации, а также для лицензированных сторонний код безопасности.

### <a name="conduct-a-final-security-review"></a>Поведения к окончательной проверке безопасности

Намеренно проверки все действия безопасности, которые были выполнены помогает гарантировать готовность выпуска программного обеспечения или приложения. Окончательная проверка безопасности (FSR) обычно включает в себя анализа моделей угроз, выходные данные средства и производительности от системы контроля качества и полос ошибки, которые были определены на этапе требования.

### <a name="certify-release-and-archive"></a>Сертификация выпуска и архивация

После прохождения сертификации программного обеспечения до выпуска помогает обеспечить соблюдение требований к безопасности и конфиденциальности. Архивация всех данных крайне важна для выполнения задач обслуживания после выпуска. Архивирование также помогает ниже долгосрочные затраты, связанные с постоянной разработки программного обеспечения.

## <a name="response"></a>Ответ
На этапе после выпуска ответа ориентирована на разработчиков могут и доступна соответствующим образом реагировать на любые отчеты новых программного обеспечения угроз и уязвимостей.

### <a name="execute-the-incident-response-plan"></a>Выполнить план реагирования на инциденты

Для помощи в защите клиентов от уязвимости безопасность или конфиденциальность программного обеспечения, которые появляются важно возможность реализации плана реагирования на инциденты, предотвращающих на этапе выпуска.

### <a name="monitor-application-performance"></a>Мониторинг производительности приложений

Постоянное наблюдение за приложения после его развертывания потенциально помогает выявить проблемы с производительностью, а также проблемы безопасности.
Службы Azure, с помощью приложения мониторинга которых являются:

  - Azure Application Insights
  - Центр безопасности Azure

#### <a name="application-insights"></a>Application Insights

[Application Insights](https://docs.microsoft.com/azure/application-insights/app-insights-overview) — это расширяемая служба управления производительностью приложений (APM) для веб-разработчиков на нескольких платформах. Используйте ее для мониторинга вашего работающего веб-приложения. Application Insights автоматически обнаруживает аномалии производительности. Он включает мощные аналитические средства, помогающие диагностировать проблемы и понять, что пользователи фактически делают с вашим приложением. Эта служба помогает постоянно улучшать производительность и удобство использования.

#### <a name="azure-security-center"></a>Центр безопасности Azure

[Центр безопасности Azure](https://docs.microsoft.com/azure/security-center/security-center-intro) помогает выявлять и реагировать на угрозы благодаря более полной информации в (и контроль над) безопасность ресурсов Azure, включая веб-приложения. Центр безопасности Azure помогает выявлять угрозы, которые в противном случае оказаться незамеченными. Он работает с различных решений по обеспечению безопасности.

Центр безопасности ценовой категории "Бесплатный" предлагает ограниченные возможности безопасности только для ваших ресурсов Azure. [Безопасности центр "стандартный"](https://docs.microsoft.com/azure/security-center/security-center-onboarding) расширяет эти возможности на локальных ресурсов и в других облаках.
Стандартный центр безопасности помогает:

  - Находите и исправляйте уязвимости системы безопасности.
  - Применение элементов управления доступом и приложениями для блокировки вредоносных действий.
  - Обнаруживать угрозы с помощью аналитики и аналитики.
  - Быстро реагировать на атаки.

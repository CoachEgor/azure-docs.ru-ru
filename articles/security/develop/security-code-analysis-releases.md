# <a name="microsoft-security-code-analysis-releases-and-roadmap"></a>Выпуски и дорожная карта анализа кода безопасности Майкрософт

Команда Microsoft Security Code Analysis в партнерстве с поддержкой разработчиков с гордостью объявляет о недавних и предстоящих усовершенствованиях нашего расширения MSCA. Пожалуйста, смотрите дорожную карту ниже.

![Выпуски](./media/security-code-analysis-releases/releases.png)

## <a name="credential-scanner-v20-released-on-april-1-2020"></a>Сканер учетных данных v2.0: Выпущен 1 апреля 2020 г.

### <a name="innovations--improvements"></a>Инновации & улучшения

- **Основной двигатель**

   - Среднее обновление производительности 25% с почти линейным временем выполнения
   - Контекст/доказательства на основе поиска и ранжирования для повышения точности
   - Улучшения общего обнаружения паролей и сопоставления логики для очевидных заполнителей (например, fakePassword)

- **Покрытие** - Поддержка для 25 "секретных типов, включая следующие топ просил:

   - Сертификат сертификата ткани Passphrase
   - Клиент секрет / API ключ
   - Заголовок авторизации HTTP
   - Amazon S3 Клиент Секретный ключ доступа
   - Токен доступа к клиенту Active Directory
   - Мастер функции Azure/aPI Key
   - Ключ доступа Power BI
   - Шаблон пароля шаблона управления ресурсами Azure

- **Выходные данные**

   - Поддержка форматов файлов SARIF 2.1 и CSV

## <a name="binskim-v160-to-be-released-on-april-2020"></a>BinSkim v1.6.0: Будет выпущен в апреле 2020 года

### <a name="improvements"></a>Улучшения

- FEATURE: Обновление до окончательного SARIF v2 (версия 2.1.16). Это позволяет кэшировать результаты при прохождении - хэши на командной строке, значительное улучшение производительности при повторном анализе каталогов с несколькими копиями целей сканирования.
- BUG FIX: Исправить опечатку в BA2021. DoNotMarkWritableSectionsAsExecutable выход.
- PERFORMANCE: Устранить загрузку PDB для всех несмешанных режимов для управляемых сборок, включая IL Library (досрочно составленные) бинарные файлы.
- FALSE NEGATIVE FIX: Проверить, что PDB размещены рядом с двоичным на самом деле соответствует двоичный под анализом
- FEATURE: Обеспечить --местный символ-каталоги аргумент, чтобы указать дополнительные (местные, не-символ-сервер) PDB поиска места
- FALSE POSITIVE FIX: Пропустить PDB-управляемый анализ для генерируемого родного bootstrap exe .NET ядра (который не является управляемым пользователем кодом).

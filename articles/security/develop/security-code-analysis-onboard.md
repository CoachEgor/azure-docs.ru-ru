# <a name="how-to-onboarding-and-installing"></a>Как Подключение и установка

Предварительные требования для начала работы с анализом кода безопасности Майкрософт
  - Предложение соответствующих служб Единая поддержка Майкрософт (подробные сведения см. ниже)
  - Организация Azure DevOps
  - Разрешения для установки расширений в Организации Azure DevOps
  - Исходный код, который можно синхронизировать с конвейером Azure DevOps, размещенным в облаке


## <a name="onboarding-microsoft-security-code-analysis-extension"></a>Адаптация расширения анализа кода безопасности Майкрософт

- Если у вас уже есть одно из следующих предложений поддержки, просто обратитесь к менеджеру по технической поддержке и пурчасе\свап существующие часы, чтобы получить доступ к расширению.
   - Единая поддержка — продвинутые и уровни производительности, поддержка Premier для разработчиков, поддержка Premier для партнеров или поддержка Premier для предприятий.
- Если у вас есть одна из следующих служб поддержки или у вас нет плана поддержки корпорации Майкрософт, вам потребуется выполнить обновление до соответствующего предложения поддержки:
   - Поддержка Azure для партнеров, Azure Basic, Azure Developer, Azure Standard, Azure Professional Direct или унифицированная поддержка — уровень ядра
- Чтобы приобрести соответствующее предложение поддержки, посетите [домашнюю страницу службы поддержки](https://www.microsoft.com/enterprise/services/support) .
- После получения контракта на поддержку обратитесь к менеджеру по технической поддержке, который поможет вам приступить к работе и соберет все необходимые сведения.
 
>[!NOTE]
> Только зарегистрированные партнеры в сети партнеров Майкрософт могут покупать поддержка Premier для партнеров, в противном случае приобрести одно из соответствующих предложений поддержки, упомянутых ранее.

## <a name="installing-microsoft-security-code-analysis-extension"></a>Установка расширения анализа кода безопасности Майкрософт

1. Когда расширение будет предоставлено для вашей организации Azure DevOps, перейдите на страницу Организации Azure DevOps (например, http://dev.azure.com/contoso)
2. Щелкните значок "набор покупок" в правом верхнем углу рядом с именем, а затем выберите Управление расширениями. 
3. Щелкните расширение "анализ кода безопасности Microsoft" и запустите мастер пользовательского интерфейса Azure DevOps, чтобы начать установку.
4. Выберите организацию Azure DevOps для установки расширения из раскрывающегося списка
5. Нажмите кнопку установить. По завершении можно приступить к использованию расширения.

>[!NOTE]
> Даже если у вас нет доступа, продолжайте процедуру установки, и вы сможете запросить доступ у администратора организации Azure DevOps во время этого процесса.
>
После установки расширения задачи сборки безопасной разработки будут видимы и доступны для добавления в Azure Pipelines.

## <a name="adding-specific-build-tasks-to-your-devops-pipeline"></a>Добавление конкретных задач сборки в конвейер DevOps

1. Откройте командный проект в Организации Azure DevOps.
2. Переход на вкладку " **сборки** " в разделе " **конвейеры** " 
3. Выберите конвейер, в который требуется добавить задачи сборки расширения. 
   - Создать — щелкните **создать** и выполните действия, описанные в разделе Создание нового конвейера.
   - Изменить — выберите конвейер и нажмите кнопку **изменить** , чтобы начать редактирование существующего конвейера.
4. Щелкните +, чтобы открыть панель **добавить задачи** .
5. Найдите задачу сборки, которую нужно добавить, из списка или с помощью поля поиска, а затем нажмите кнопку **Добавить**. 
6. Теперь можно продолжить указывать параметры, необходимые для задачи.
>[!NOTE]
>Пути к файлам или каталогам зависят от корня исходного репозитория, и параметры, указывающие выходную папку или файлы, будут заменены на общее расположение, которое мы определили в агенте сборки.

7. Поставьте новую сборку в очередь.
> [!TIP]
>  - Чтобы запустить анализ после построения, поместите задачи сборки "анализ кода безопасности Microsoft" после этапа сборки "опубликовать артефакты сборки". Таким образом, сборка может завершиться и публиковать результаты перед выполнением статических средств анализа.
>  - Всегда установите флажок **"продолжать при ошибке"** в задачах безопасной сборки разработки. Даже в случае сбоя одного из инструментов другие могут запуститься. Взаимозависимости нет.
>  - Задачи сборки анализа кода безопасности Майкрософт завершатся сбоем, только если средство не запускается успешно, но они не будут завершаться ошибкой, если и средство выявляет проблемы в коде. Можно настроить прерывание сборки, если средство выявляет проблемы в коде с помощью задачи построения **после анализа** .
>  - Некоторые задачи сборки Azure DevOps не поддерживаются при выполнении через конвейер "Release". Это ограничение Azure DevOps. Они не поддерживают задачи, публикующие артефакты в конвейере выпуска.
>  - Список стандартных переменных в Azure DevOps Team Build, которые можно указать в качестве параметров, см. в статье [переменные сборки DevOps Azure](https://docs.microsoft.com/azure/devops/pipelines/build/variables?tabs=batch&view=vsts) .

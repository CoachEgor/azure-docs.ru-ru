# <a name="onboarding-and-installing"></a>Посадка и установка

Предпосылки для начала работы с анализом кода безопасности Майкрософт:

- Подходящее предложение объединенной поддержки Майкрософт, подробно описанное в следующем разделе.
- Организация Azure DevOps.
- Разрешение на установку расширений в организацию Azure DevOps.
- Исходный код, который можно синхронизировать с проводом Azure DevOps, на котором будет облачно.

## <a name="onboarding-the-microsoft-security-code-analysis-extension"></a>Расширение анализа кода безопасности Майкрософт

### <a name="interested-in-purchasing-the-microsoft-security-code-analysis-extension"></a>Заинтересованы в приобретении расширения анализа кода безопасности Майкрософт?

Если у вас есть одно из следующих предложений поддержки, свяжитесь с вашим менеджером технического счета, чтобы приобрести или поменять существующие часы, чтобы получить доступ к расширению:

- Единый расширенный уровень поддержки
- Унифицированный уровень производительности поддержки
- Премьер-поддержка для разработчиков
- Премьер-поддержка партнеров
- Премьер-поддержка предпринимательства

Если у вас нет одного из упомянутых выше соглашений о поддержке, вы можете приобрести продление у одного из наших Партнеров.

**Следующие шаги:**

Свяжитесь с Партнером из списка ниже и попросите приобрести расширение анализа кода безопасности Майкрософт.

>**Партнеров:**

## <a name="installing-the-microsoft-security-code-analysis-extension"></a>Установка расширения анализа кода безопасности Майкрософт

1. После того, как расширение будет передано в организацию Azure DevOps, перейдите на страницу организации Azure DevOps. Пример URL для такой `https://dev.azure.com/contoso`страницы .
1. Выберите значок сумки для покупок в правом верхнем углу рядом с вашим именем, а затем выберите **расширения Управления.**
1. Выберите **общий .**
1. Выберите расширение анализа кода безопасности Майкрософт, выберите **установку.**
1. Из списка выпадающих списков выберите организацию Azure DevOps для установки расширения.
1. Выберите пункт **Установить**. После завершения установки можно начать использовать расширение.

>[!NOTE]
> Даже если у вас нет доступа для установки расширения, продолжайте выполнять этапы установки. Вы можете запросить доступ у вашего админа организации Azure DevOps во время процесса установки.

После установки расширения задачи безопасной сборки разработки становятся видимыми и доступными для добавления в конвейеры Azure.

## <a name="adding-specific-build-tasks-to-your-azure-devops-pipeline"></a>Добавление конкретных задач сборки в конвейер Azure DevOps

1. От организации Azure DevOps откройте свой командный проект.
1. Выберите **сборки трубопроводов** > **.**
1. Выберите конвейер, в который требуется добавить задачи сборки расширений:
   - Новый конвейер: Выберите **новый** и выполните детали, описанные для создания нового конвейера.
   - Редактирование конвейера: Выберите существующий конвейер, а затем выберите **редактировать,** чтобы начать редактирование конвейера.
1. Выберите **+** и перейдите на панель **дополнительных задач.**
1. Из списка или с помощью окна поиска найдите задачу сборки, которую вы хотите добавить. Выберите **Добавить**.
1. Укажите параметры, необходимые для выполнения задачи.
1. Поставьте новую сборку в очередь.
   >[!NOTE]
   >Пути файлов и папок относительно корня исходного репозитория. Если в качестве параметров указаны выходные файлы и папки, они заменяются общим местоположением, которое мы определили в агенте сборки.

> [!TIP]
>
> - Чтобы выполнить анализ после сборки, поместите задачи по сборке кода безопасности Майкрософт после шага сборки «Опубликовать артефакты сборки». Таким образом, сборка может закончить и опубликовать результаты перед запуском инструментов статического анализа.
> - Всегда выбирайте **«Продолжить ошибку»** для задач безопасной разработки. Даже если один инструмент выходит из строя, другие могут работать. Между инструментами нет взаимозависимостей.
> - Работа по сборке кода безопасности Майкрософт завершается только в том случае, если инструмент успешно выполняется успешно. Но они удаляются, даже если инструмент определяет проблемы в коде. Используя задачу сборки post-Analysis, можно настроить сборку на сбой, когда инструмент определяет проблемы в коде.
> - Некоторые задачи сборки Azure DevOps не поддерживаются при запуске через конвейер выпуска. В частности, Azure DevOps не поддерживает задачи, публикующие артефакты из конвейера выпуска.
> - Для списка предопределенных переменных в сборке команды Azure DevOps, [Azure DevOps Build Variables](https://docs.microsoft.com/azure/devops/pipelines/build/variables?tabs=batch&view=vsts)которые можно указать в качестве параметров, см.

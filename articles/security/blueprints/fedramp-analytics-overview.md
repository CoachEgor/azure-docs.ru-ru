# <a name="azure-security-and-compliance-blueprint-analytics-for-fedramp"></a>Схема безопасности и соответствия требованиям Azure. Аналитика для FedRAMP

## <a name="overview"></a>Обзор

[Федеральная программа аккредитации облачных служб (FedRAMP)](https://www.fedramp.gov/) — государственная программа США, которая определяет стандартизированный подход к оценке безопасности, сертификации и постоянному мониторингу облачных продуктов и услуг. Схема безопасности и соответствия требованиям Azure предоставляет руководство по поставке архитектуры аналитики Microsoft Azure, которая помогает реализовывать ряд элементов управления FedRAMP уровня High. Это решение предоставляет руководство по развертыванию и конфигурации ресурсов Azure для общей эталонной архитектуры, демонстрируя, как клиенты могут обеспечить соответствие требованиям к безопасности и соответствию. Оно также служит основой для создания и настройки собственных решений аналитики в Azure.

Эта эталонная архитектура, соответствующие руководства по реализации элементов управления и модели рисков считаются основой, которую клиенты должны адаптировать для конкретных требований, а не использовать "как есть" в рабочей среде. Развертывания приложения в этой среде без изменений недостаточно для полного соответствия базовым требованиям FedRAMP уровня High. Обратите внимание на следующее:
- Архитектура является своего рода эталоном, который помогает клиентам развертывать рабочие нагрузки Azure в соответствии с требованиями FedRAMP.
- Клиенты несут ответственность за выполнение надлежащей оценки безопасности и соответствия любого решения, созданного с использованием этой архитектуры, так как требования могут варьироваться в зависимости от специфики реализации каждого клиента.

## <a name="architecture-diagram-and-components"></a>Схема и компоненты архитектуры

Это решение предоставляет платформу аналитики, на которой клиенты могут создавать собственные средства аналитики. Эталонная архитектура описывает универсальный вариант использования, в котором клиенты вводят данные с помощью массового импорта (администратор SQL и данных) или с помощью обновлений операционных данных (операционный пользователь). Оба этих рабочих потока используют [Функции Azure](https://docs.microsoft.com/azure/azure-functions/functions-overview) для импорта данных в Базу данных SQL. Клиент должен самостоятельно настроить Функции Azure на портале Azure, чтобы выполнять уникальные задания импорта согласно конкретным требованиям клиента к аналитике.

Microsoft Azure предоставляет клиентам множество служб для отчетов и аналитики. Описываемое решение включает в себя Azure Analysis Services и Базу данных SQL Azure, что позволяет оперативно просматривать данные и быстрее получать результаты благодаря оптимизации моделирования клиентских данных. Службы аналитики Azure основаны на технологиях машинного обучения и предназначены повысить скорость запросов за счет обнаружения новых связей между наборами данных. Обучив модель данных с помощью нескольких статистических функций, вы сможете синхронизировать дополнительные пулы запросов (до 7 или 8, если считать сервер клиента) с одним набором табличных моделей, чтобы равномерно распределить рабочую нагрузку и сократить время отклика.

Чтобы получить расширенные возможности аналитики и создания отчетов, для Базы данных SQL можно настроить индексы columnstore. Как службы аналитики Azure, так и Базу данных SQL можно масштабировать, а также полностью отключить в соответствии с текущей активностью клиентов. Весь трафик SQL шифруется с помощью SSL и самозаверяющих сертификатов. Azure рекомендует использовать доверенный центр сертификации для повышенной безопасности.

После загрузки данных в базу данных SQL Azure и обучения через Azure Analysis Services, эти данные обрабатываются в потоках операционного пользователя и администратора SQL и данных с использованием Power BI. Power BI отображает данные в интуитивно понятном виде и собирает воедино информацию из нескольких наборов данных, чтобы предоставить дополнительные аналитические сведения. Высокая степень адаптивности и простота интеграции с Базой данных SQL Azure позволяют клиентам настроить решение для широкого набора сценариев в соответствии с их бизнес-требованиями.

Все решение основывается на службе хранилища Azure, которую пользователи настраивают через портал Azure. Служба хранилища Azure шифрует все данные с помощью шифрования службы хранилища, чтобы гарантировать конфиденциальность неактивных данных.  Геоизбыточное хранилище (GRS) гарантирует, что неполадки в основном клиентском центре обработки данных не приведут к потере данных, так как вторая копия хранится в другом расположении за сотни километров от основного.

Чтобы обеспечить усиленную безопасность, эта архитектура применяет для управления ресурсами службы Azure Active Directory и Azure Key Vault. Мониторинг работоспособности системы осуществляется через Azure Monitor. Клиенты могут настроить обе службы мониторинга для ведения журналов и отображения работоспособности системы на единой панели мониторинга с удобной навигацией.

Как правило, управление базой данных SQL Azure выполняется с помощью средств SQL Server Management Studio (SSMS), запущенных на локальном компьютере с доступом к базе данных SQL Azure через безопасное соединение VPN или ExpressRoute. **Azure рекомендует настроить подключение VPN или Azure ExpressRoute для управления и импорта данных в группу ресурсов эталонной архитектуры.**

### <a name="roles"></a>Роли
Схема аналитики описывает сценарий с тремя типами пользователей: операционный пользователь, администратор SQL или данных и системный инженер. Управление доступом на основе ролей (RBAC) в Azure позволяет реализовывать детальное управление доступом через встроенные пользовательские роли. Доступны ресурсы для настройки [управления доступом на основе ролей](../../role-based-access-control/role-assignments-portal.md), а также структурирования и реализации [предопределенных ролей](../../role-based-access-control/built-in-roles.md).

#### <a name="systems-engineer"></a>Системный инженер
Системный инженер является владельцем клиентской подписки Azure и настраивает развертывание решения через портал Azure.

#### <a name="sqldata-administrator"></a>Администратор SQL и данных
Администратор SQL и данных настраивает функции массового импорта данных и операционного обновления данных для их передачи в базу данных SQL Azure. Администратор SQL и данных не отвечает за операционное обновления данных в базе данных, но может просматривать данные с помощью Power BI.

#### <a name="operational-user"></a>Операционный пользователь
Операционный пользователь регулярно обновляет данные и отвечает за рутинные задачи создания данных. Операционный пользователь также занимается оценкой результатов с помощью Power BI.

В следующем разделе подробно описываются компоненты разработки и реализации.

**Функции Azure**. [Функции Azure](https://docs.microsoft.com/azure/azure-functions/functions-overview) — это решения для выполнения небольших фрагментов кода в облаке с помощью большинства языков программирования. В этом решении функции интегрируются со службой хранилища Azure автоматически передают клиентские данные в облако, что упрощает интеграцию с другими службами Azure. Функции легко масштабируются и оплачиваются только за время фактического выполнения.

**Служба Azure Analysis Service**: [Служба Azure Analysis Service](https://docs.microsoft.com/azure/analysis-services/analysis-services-overview) обеспечивает моделирование корпоративных данных и интеграцию со службами платформы данных Azure. Служба Azure Analysis Service ускоряет просмотр больших объемов данных, позволяя объединять данные из нескольких источников в единую модель данных.

**Power BI**. [Power BI](https://docs.microsoft.com/power-bi/service-azure-and-power-bi) предоставляет функции аналитики и создания отчетов для клиентов, пытающихся получить более подробные сведения о своих усилиях по обработке данных.

### <a name="networking"></a>Сети
**Группы безопасности сети**. [Группы безопасности сети](../../virtual-network/virtual-network-vnet-plan-design-arm.md) настраиваются для управления трафиком, направленным на развернутые ресурсы и службы. Группы NSG используют схему запрета по умолчанию, то есть пропускают только тот трафик, который соответствует предварительно настроенному списку управления доступом.

В каждой NSG открыты определенные порты и протоколы, чтобы решение могло работать безопасно и правильно. Кроме того, для каждой NSG настроены следующие конфигурации:
  - включены [журналы диагностики и события](https://docs.microsoft.com/azure/virtual-network/virtual-network-nsg-manage-log), которые хранятся в учетной записи хранения;
  - [Журналы Azure Monitor](../../azure-monitor/insights/azure-networking-analytics.md) подключаются к журналам диагностики NSG.

### <a name="data-at-rest"></a>Неактивные данные
Архитектура защищает неактивные данные, используя шифрование, аудит базы данных и другие меры.

**Репликация данных**. Azure для государственных организаций предлагает два метода [репликации данных](https://docs.microsoft.com/azure/storage/common/storage-redundancy).
 - По умолчанию для репликации применяется **геоизбыточное хранилище**, которое асинхронно сохраняет клиентские данные в отдельный центр обработки данных, размещенный за пределами основного региона. Это позволяет восстановить данные даже при полной потере информации в основном центре обработки данных.
 - Также можно настроить **локально избыточное хранилище (LRS)** с помощью учетной записи хранения Azure. LRS реплицирует данные в единице масштабирования хранилища, размещенной в том же регионе, где клиент создал учетную запись. Все данные реплицируются параллельно, а значит они не будут потеряны в случае сбоя основной единицы масштабирования хранилища.

**Служба хранилища Azure**. Для выполнения требований к шифрованию неактивных данных во всей [службе хранилища Azure](https://azure.microsoft.com/services/storage/) используется [шифрование службы хранилища](../../storage/common/storage-service-encryption.md).

**Шифрование дисков Azure**.
[Шифрование дисков Azure](../azure-security-disk-encryption-overview.md) использует возможность BitLocker Windows, чтобы обеспечить шифрование тома для ОС и дисков данных. Решение интегрируется с Azure Key Vault, что помогает управлять ключами шифрования дисков.

**База данных SQL Azure**. В экземпляре Базы данных SQL Azure используются следующие меры безопасности базы данных:
-   [Аутентификация и авторизация AD](https://docs.microsoft.com/azure/sql-database/sql-database-aad-authentication) обеспечивает централизованное управление удостоверениями для пользователей баз данных и других служб Майкрософт.
-   [Аудит базы данных SQL](../../sql-database/sql-database-auditing.md) позволяет отслеживать события базы данных и записывать их в журнал аудита в учетной записи хранения Azure.
-   База данных SQL настроена для использования [прозрачного шифрования данных (TDE)](https://docs.microsoft.com/sql/relational-databases/security/encryption/transparent-data-encryption-azure-sql), которое обеспечивает шифрование и расшифровку файлов данных и журналов в реальном времени для защиты неактивных данных. TDE позволяет защитить сохраненные данные от несанкционированного доступа.
-   [Правила брандмауэра](https://docs.microsoft.com/azure/sql-database/sql-database-firewall-configure) предотвращают любой доступ к серверам баз данных без надлежащих разрешений. Брандмауэр предоставляет доступ к базам данным на основе исходного IP-адреса каждого запроса.
-   [Обнаружение угроз SQL](../../sql-database/sql-database-threat-detection.md) позволяет немедленно выявлять потенциальные угрозы и реагировать на них. При этом поступают оповещения системы безопасности о подозрительных действиях с базами данных, потенциальных уязвимостях, атаках путем внедрения кода SQL и аномальных шаблонах доступа к базам данных.
-   [Столбцы Always Encrypted](https://docs.microsoft.com/azure/sql-database/sql-database-always-encrypted-azure-key-vault) гарантируют, что конфиденциальные данные никогда не отобразятся как открытый текст внутри системы баз данных. После включения шифрования данных получить доступ к данным в виде открытого текста смогут только клиентские приложения и серверы приложений, у которых есть доступ к ключам.
-   После развертывания эталонной архитектуры можно выполнить [динамическое маскирование данных в базе данных SQL](https://docs.microsoft.com/azure/sql-database/sql-database-dynamic-data-masking-get-started). Клиентам потребуется настроить параметры динамического маскирования данных в соответствии со своей схемой базы данных.

### <a name="logging-and-audit"></a>Ведение журнала и аудит
[Azure Monitor](../../azure-monitor/overview.md) формирует полное отображение данных мониторинга, включая журналы действий, метрики и диагностические данные, что позволяет клиентам получить полное представление о работоспособности системы.  
[Журналы Azure Monitor](../azure-security-disk-encryption-overview.md) предоставляют широкие возможности ведения журнала системных и пользовательских действий, а также работоспособность системы. Это решение собирает и анализирует данные, создаваемые ресурсами в Azure и локальных средах.
- **Журналы действий**. [Журналы действий](../../azure-monitor/platform/activity-logs-overview.md) предоставляют информацию об операциях, которые выполнялись с ресурсами в подписке.
- **Журналы диагностики**. [Журналы диагностики](../../azure-monitor/platform/resource-logs-overview.md) включают в себя все журналы, создаваемые ресурсами. Эти журналы включают в себя системные журналы событий Windows, хранилища BLOB-объектов, таблиц и очередей.
- **Журналы брандмауэра**. Шлюз приложений предоставляет все журналы доступа и диагностики. Журналы брандмауэра доступны для ресурсов шлюза приложений с включенным WAF.
- **Архивация журналов**. Все журналы диагностики записываются в централизованную зашифрованную учетную запись Azure для архивирования на определенный период хранения (2 дня). Эти журналы подключаются к журналам Azure Monitor для обработки, хранения и создания отчетов на панели мониторинга.

Кроме того, в составе этой архитектуры устанавливаются указанные ниже решения мониторинга:
-   [Служба автоматизации Azure](https://docs.microsoft.com/azure/automation/automation-hybrid-runbook-worker). Решение "Служба автоматизации" хранит модули runbook, выполняет их и управляет ими.
-   [Безопасность и аудит](../../security-center/security-center-intro.md). Панель мониторинга "Безопасность и аудит" создает высокоуровневое представление для состояния безопасности ресурсов, которое содержит метрики по доменам безопасности, важным проблемам, обнаружениям и исследованию угроз, а также по стандартным запросам безопасности.
-   [Оценка SQL](../../azure-monitor/insights/sql-assessment.md). Решение "Проверка работоспособности SQL" регулярно оценивает риски и работоспособность серверных сред и предоставляет клиентам список рекомендаций, относящихся к развернутой серверной инфраструктуре и перечисленных в порядке приоритета.
-   [Журналы действий Azure](../../azure-monitor/platform/collect-activity-logs.md). Решение "Аналитика журнала действий" помогает анализировать журналы действий Azure во всех подписках Azure для клиента.

### <a name="identity-management"></a>Управление удостоверениями
-   Аутентификация приложения выполняется с использованием Azure AD. Дополнительные сведения см. в статье [Интеграция приложений с Azure Active Directory](../../active-directory/develop/quickstart-v1-integrate-apps-with-azure-ad.md). Кроме того, при шифровании столбцов базы данных используется Azure AD для аутентификации приложения в Базе данных Azure SQL. Дополнительные сведения см. в статье о [защите конфиденциальных данных в Базе данных SQL](https://docs.microsoft.com/azure/sql-database/sql-database-always-encrypted-azure-key-vault).
-   [Защита идентификации Azure Active Directory](../../active-directory/identity-protection/overview.md) выявляет потенциальные уязвимости, влияющие на удостоверения организации, настраивает автоматическое реагирование на обнаруженные подозрительные действия, связанные с удостоверениями организации и исследует подозрительные инциденты для принятия соответствующих мер по их устранению.
-   [Управление доступом на основе ролей (RBAC) Azure](../../role-based-access-control/role-assignments-portal.md) обеспечивает детальный контроль доступа для Azure. Доступ к подписке есть только у администратора подписки.

Подробнее об использовании функций безопасности Базы данных SQL Azure см. в примере [демонстрационного приложения Contoso Clinic](https://github.com/Microsoft/azure-sql-security-sample).

### <a name="security"></a>Группа безопасности
**Управление секретами**. Для управления ключами и секретами в решении используется [Azure Key Vault](https://azure.microsoft.com/services/key-vault/). Хранилище ключей Azure помогает защитить криптографические ключи и секреты, используемые облачными приложениями и службами.

## <a name="guidance-and-recommendations"></a>Инструкции и рекомендации

### <a name="expressroute-and-vpn"></a>ExpressRoute и VPN
Необходимо настроить защищенный туннель VPN или [ExpressRoute](https://docs.microsoft.com/azure/expressroute/expressroute-introduction), чтобы установить защищенное подключение к ресурсам, развертываемым в рамках этой эталонной архитектуры аналитики данных. Подключения ExpressRoute не осуществляются через Интернет и обеспечивают повышенный уровень безопасности, надежности и быстродействия подключений с низким уровнем задержки по сравнению со стандартными подключениями через Интернет. Правильно настроив ExpressRoute или VPN-подключение, клиенты могут добавить уровень защиты передаваемых данных.

### <a name="azure-active-directory-setup"></a>Настройка Azure Active Directory
Служба [Azure Active Directory](../../active-directory/fundamentals/active-directory-whatis.md) играет важную роль в управлении развертыванием и подготовкой доступа для персонала, взаимодействующего со средой. Windows Server Active Directory можно интегрировать с AAD [с помощью четырех простых действий](../../active-directory/hybrid/how-to-connect-install-express.md). Клиенты также могут привязать развернутую инфраструктуру Active Directory (контроллеры домена) к существующей службе AAD, сделав развернутую инфраструктуру Active Directory дочерним доменом леса AAD.

### <a name="additional-services"></a>Дополнительные службы
#### <a name="iaas---vm-considerations"></a>IaaS — рекомендации по виртуальным машинам
Это решение PaaS не включает виртуальные машины IaaS Azure. Клиент может самостоятельно создать виртуальные машины Azure для запуска многих из этих служб PaaS. В этом случае можно использовать определенные функции и службы для обеспечения непрерывности бизнес-процессов и журналов Azure Monitor.

##### <a name="business-continuity"></a>Непрерывность бизнес-процессов
- **Высокий уровень доступности**. Рабочие нагрузки сервера группируются в группе [доступности](https://docs.microsoft.com/azure/virtual-machines/virtual-machines-windows-manage-availability?toc=%2fazure%2fvirtual-machines%2fwindows%2ftoc.json) для обеспечения высокой доступности виртуальных машин в Azure. Во время планового или внепланового технического обслуживания доступна как минимум одна виртуальная машина, что соответствует соглашению об уровне обслуживания Azure, гарантирующему доступность в течение 99,95 % времени.

- **Хранилище служб восстановления**. [Хранилище Служб восстановления](https://docs.microsoft.com/azure/backup/backup-azure-recovery-services-vault-overview) содержит резервные копии данных и защищает все конфигурации виртуальных машин Azure в этой архитектуре. С помощью хранилища служб восстановления клиенты могут восстановить отдельные файлы и папки виртуальной машины IaaS, а не всю виртуальную машину. Это сокращает время восстановления.

##### <a name="monitoring-solutions"></a>Решения для мониторинга
-   [Оценка AD](../../azure-monitor/insights/ad-assessment.md). Решение "Проверка работоспособности Active Directory" регулярно оценивает риски и работоспособность серверных сред и предоставляет список рекомендаций, относящихся к развернутой серверной инфраструктуре, в порядке приоритета.
-   [Оценка защиты от вредоносных программ](../../security-center/security-center-install-endpoint-protection.md). Решение для защиты от вредоносных программ уведомляет о вредоносных программах, угрозах и состоянии защиты.
-   [Управление обновлениями](../../automation/automation-update-management.md). Решение "Управление обновлениями" позволяет клиенту управлять обновлениями безопасности операционной системы, включая состояние доступных обновлений, а также процесс установки необходимых обновлений.
-   [Работоспособность агента](../../monitoring/monitoring-solution-agenthealth.md). Решение "Работоспособность агентов" сообщает количество развернутых агентов, их географическое распределение, а также количество агентов, которые перестают отвечать на запросы, и агентов, которые отправляют операционные данные.
-   [Отслеживание изменений](../../automation/change-tracking.md). Решение "Отслеживание изменений" позволяет клиентам легко определять изменения в среде.

##### <a name="security"></a>Группа безопасности
- **Защита от вредоносных программ**. [Microsoft Antimalware](https://docs.microsoft.com/azure/security/fundamentals/antimalware) для виртуальных машин Azure предоставляет возможности защиты в реальном времени для обнаружения и удаления вирусов, шпионских и других вредоносных программ. Вы можете настроить оповещения о попытках установки или выполнения вредоносных или нежелательных программ на защищенных виртуальных машинах.
- **Управление исправлениями**. Виртуальные машины Windows, развернутые как часть этой эталонной архитектуры, настраиваются по умолчанию для получения автоматических обновлений из службы Центра обновления Windows. Это решение также включает службу [автоматизации Azure](https://docs.microsoft.com/azure/automation/automation-intro), с помощью которой можно создавать обновленные развертывания для исправления виртуальных машин при необходимости.

#### <a name="azure-commercial"></a>Azure для коммерческих организаций
Хотя эта архитектура аналитики данных не предназначена для развертывания в среде [Azure для коммерческих организаций](https://azure.microsoft.com/overview/what-is-azure/), аналогичных целей можно достичь с помощью служб, описанных в этой эталонной архитектуре, а также дополнительных служб, доступных только в среде Azure для коммерческих организаций. Обратите внимание, что среда Azure для коммерческих организаций поддерживает FedRAMP JAB P-ATO на среднем уровень влияния, позволяя государственным учреждениям и партнерам развертывать в облаке умеренно конфиденциальные сведения, используя эту среду.

Azure для коммерческих организаций предлагает разнообразные службы аналитики для извлечения сведений из больших объемов данных.
-   [Студия машинного обучения Azure](https://docs.microsoft.com/azure/machine-learning/studio/what-is-ml-studio), которая входит в состав [Cortana Intelligence Suite](https://azure.microsoft.com/overview/ai-platform/), позволяет разрабатывать модели прогнозной аналитики на основе одного или нескольких источников данных. Несколько этапов итерации статистических функций позволяют создать эффективную модель для применения в приложениях, например в Power BI.
-   [Azure Data Lake Store](https://docs.microsoft.com/azure/data-lake-store/data-lake-store-overview) позволяет централизованно сохранять данные любого размера, типа и с любой скоростью приема для эксплуатационной и исследовательской аналитики. Служба Azure Data Lake Store совместима с большинством компонентов с открытым кодом в экосистеме Hadoop и хорошо интегрируется с другими службами Azure.

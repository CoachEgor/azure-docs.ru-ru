# <a name="get-started-with-your-microsoft-partner-agreement-billing-account"></a>Начало работы с учетной записью выставления счетов Соглашения с партнером Майкрософт

Учетная запись выставления счетов создается при регистрации в Azure. Учетная запись выставления счетов используется для управления счетами, платежами и отслеживания расходов. Вы можете иметь доступ к нескольким учетным записям выставления счетов. Например, вы могли зарегистрироваться в Azure для своих личных проектов. Вы также могли получить доступ к Azure через Соглашение Enterprise вашей организации, Клиентское соглашение Майкрософт или Соглашение с партнером Майкрософт. Для каждого из этих сценариев у вас будет отдельная учетная запись выставления счетов.

Эта статья касается учетных записей выставления счетов по Соглашению с партнером Майкрософт. Эти учетные записи создаются для поставщиков облачных решений (CSP), чтобы они могли управлять выставлением счетов клиентам в новой коммерческой программе. Новая программа доступна только тем партнерам, у которых есть по меньшей мере один клиент, принявший условия клиентского соглашения Майкрософт и использующий план Azure. [Проверьте наличие доступа к Соглашению с партнером Майкрософт](#check-access-to-a-microsoft-partner-agreement). [План Azure](https://azure.microsoft.com/pricing/purchase-options/microsoft-customer-agreement/) предоставляет клиентам доступ к службам Azure с оплатой по мере использования в рамках клиентского соглашения Майкрософт.

## <a name="your-billing-account"></a>Ваша учетная запись выставления счетов

Учетная запись выставления счетов по Соглашению с партнером Майкрософт содержит профиль выставления счетов для каждой валюты, в которой ведется бизнес. Профиль выставления счетов позволяет управлять счетами для вашей валюты. При установлении отношений с клиентами, в зависимости от их валют, подписки Azure и другие покупки оплачиваются в соответствии с профилями выставления счетов.

На схеме ниже показана связь между учетной записью выставления счетов, профилями выставления счетов, клиентами и торговыми посредниками.

![Схема, отображающая иерархию выставления счетов по Соглашению с партнером Майкрософт](./media/mpa-overview/mpa-hierarchy.svg)

Пользователи вашей организации с ролью **глобального администратора** и **агентов администратора** могут управлять учетными записями выставления счетов, профилями выставления счетов и клиентами. Дополнительные сведения см. в статье [Назначение пользователям ролей и разрешений](https://docs.microsoft.com/partner-center/permissions-overview).

## <a name="billing-profiles"></a>Профили выставления счетов

Профиль выставления счетов используется для управления счетами для вашей валюты. В начале месяца для каждого профиля выставления счетов в вашей учетной записи создается счет за месяц. Счет содержит соответствующие расходы в валюте профиля выставления счетов по всем подпискам Azure и другим покупкам за предыдущий месяц.

Вы можете просмотреть счет и скачать связанные документы, например файл использования и прайс лист, на портале Azure. Дополнительные сведения см. в разделе [Просмотр и скачивание счета Microsoft Azure](download-azure-invoice.md).

> [!IMPORTANT]
>
> Счета для профилей выставления счетов содержат расходы для клиентов с планами Azure, а также для SaaS, Azure Marketplace и резервирования покупок для клиентов, которые не приняли Клиентское соглашение Майкрософт и не имеют планов Azure.

## <a name="customers"></a>Клиенты

Вы можете просматривать клиентов, которые приняли Клиентское соглашение Майкрософт и имеют план Azure, и управлять ими на портале Azure. Вы можете просматривать расходы и транзакции, а также создавать подписки Azure для этих клиентов и управлять ими.

### <a name="enable-policy-to-give-visibility-into-cost"></a>Включение политики для предоставления видимости затрат

Примените политику, чтобы контролировать, могут ли пользователи в организации клиентов просматривать и анализировать расходы по тарифам с оплатой по мере использования для своего потребления Azure. По умолчанию политика отключена, и пользователи не могут просматривать затраты. После включения, пользователи, имеющие соответствующий доступ [Azure RBAC](https://docs.microsoft.com/azure/role-based-access-control/overview) к подписке, могут просматривать и анализировать затраты по подписке.

Чтобы включить политику, выполните приведенные ниже действия.

1. Войдите на [портал Azure](https://portal.azure.com).

1. Выполните поиск по **Управление затратами + выставление счетов**.

   ![Снимок экрана, показывающий поиск на портале Azure](./media/mpa-overview/search-cmb.png)

1. В меню слева щелкните **Клиенты**, а затем выберите клиента из списка.

   ![Снимок экрана: выбор клиента](./media/mpa-overview/mpa-customers.png)

1. Выберите **Политики** в левой части.

   ![Снимок экрана: политики](./media/mpa-overview/mpa-change-policy.png)

1. Выберите **Да**.

## <a name="resellers"></a>Торговые посредники

Непрямые поставщики в [двухуровневой модели](https://docs.microsoft.com/partner-center) CSP могут выбирать торгового посредника при создании подписок для клиентов на портале Azure. После создания они могут просматривать список подписок, отфильтрованных торговым посредником, и анализировать затраты для клиента торговыми посредниками в анализе затрат Azure.

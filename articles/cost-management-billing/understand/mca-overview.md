# <a name="get-started-with-your-microsoft-customer-agreement-billing-account"></a>Начало работы с учетной записью выставления счетов клиентского соглашения Майкрософт

Учетная запись выставления счетов создается при регистрации в Azure. Учетная запись выставления счетов используется для управления счетами, платежами и отслеживания расходов. Вы можете иметь доступ к нескольким учетным записям выставления счетов. Например, вы могли зарегистрироваться в Azure для своих личных проектов. Вы также могли получить доступ к Azure через Соглашение Enterprise вашей организации или клиентское соглашение Майкрософт. Для каждого из этих сценариев у вас будет отдельная учетная запись выставления счетов.

В этой статье рассматривается учетная запись выставления счетов для Клиентского соглашения Майкрософт. [Проверьте наличие доступа к Клиентскому соглашению Майкрософт](#check-access-to-a-microsoft-customer-agreement).

## <a name="your-billing-account"></a>Ваша учетная запись выставления счетов

Ваша учетная запись выставления счетов по клиентскому соглашению Майкрософт содержит один или несколько профилей выставления счетов, которые позволяют управлять счетами и методами оплаты. Каждый профиль выставления счетов содержит один или несколько разделов счетов, которые позволяют упорядочить расходы в счете профиля выставления счетов.

На схеме ниже показана связь между учетной записью выставления счетов, профилями выставления счетов и разделами счетов.

![Схема, показывающая иерархию выставления счетов по клиентскому соглашению Майкрософт](./media/mca-overview/mca-billing-hierarchy.png)

Роли в учетной записи выставления счетов имеют самый высокий уровень разрешений. По умолчанию доступ к учетной записи выставления счетов получает только пользователь, выполнивший вход в Azure. Эти роли должны быть назначены пользователям, которым необходимо просматривать счета и отслеживать затраты для всей организации, например финансовым или ИТ-руководителям. Дополнительные сведения см. в разделе о [ролях и задачах учетной записи выставления счетов](../manage/understand-mca-roles.md#billing-account-roles-and-tasks).

## <a name="billing-profiles"></a>Профили выставления счетов

Профиль выставления счетов используется для управления счетами и методами оплаты. В начале месяца для каждого профиля выставления счетов в вашей учетной записи создается счет за месяц. Счет содержит соответствующие расходы на все подписки Azure и другие покупки за предыдущий месяц.

Профиль выставления счетов автоматически создается для учетной записи выставления счетов. Он содержит один раздел счета. Вы можете создать дополнительные разделы, чтобы легко отслеживать и упорядочивать расходы в соответствии с потребностями, например для отдельных проектов, подразделений или сред разработки. Эти разделы отображаются в счете профиля выставления счетов и содержат информацию об использовании для назначенных подписок и приобретенных продуктов.

Роли в профилях выставления счетов имеют разрешения на просмотр счетов и методов оплаты и управление ими. Назначьте эти роли пользователям, которые оплачивают счета, например сотрудникам из отдела бухгалтерского учета в организации. Дополнительные сведения см. в разделе о [ролях и задачах профиля выставления счетов](../manage/understand-mca-roles.md#billing-profile-roles-and-tasks).

### <a name="each-billing-profile-gets-a-monthly-invoice"></a>Каждый профиль выставления счетов получает счет за месяц

В начале месяца для каждого профиля выставления счетов создается счет за месяц. Счет содержит все расходы за предыдущий месяц.

На портале Azure можно просмотреть счет, скачать документы и изменить параметр, чтобы получать будущие счета по электронной почте. Дополнительные сведения см. в разделе о [скачивании счетов для клиентского соглашения Майкрософт](../manage/download-azure-invoice-daily-usage-date.md#download-invoices-for-a-microsoft-customer-agreement).

### <a name="invoice-payment-methods"></a>Методы оплаты счетов

В каждом профиле выставления счетов используется свой метод оплаты счетов. Поддерживаются следующие методы оплаты:

| Тип             | Определение  |
|------------------|-------------|
|Деньги на счете в Azure    |  Эти средства автоматически применяются к соответствующим расходам, включенным в ваш счет, что уменьшает сумму, которую необходимо оплатить. Дополнительные сведения см. в статье об [отслеживании остатка на счете в Azure в профиле выставления счетов](../manage/mca-check-azure-credits-balance.md). |
|Чек или банковский перевод | Если для вашей учетной записи утверждена оплата по чеку или банковскому переводу, вы можете внести сумму оплаты по счету с помощью чека или банковского перевода. Инструкции по оплате указаны в счете. |
|Кредитная карта | Клиенты, которые регистрируются для работы с Azure на веб-сайте Azure, могут совершать оплату кредитной картой. |

### <a name="apply-policies-to-control-purchases"></a>Применение политик для управления покупками

Примените политики для управления покупками Azure Marketplace и покупками резервирования с помощью профиля выставления счетов. Вы можете настроить политики для отключения покупки Azure Reserved Virtual Machine Instances и продуктов Azure Marketplace. При применении политик для совершения этих покупок нельзя использовать подписки, счета по которым выставляются в профиле выставления счетов.

### <a name="azure-plans-determine-pricing-and-service-level-agreement-for-subscriptions"></a>Определение цен и соглашений об уровне обслуживания для подписок в планах Azure

Планы Azure определяют цены и соглашения об уровне обслуживания для подписок. Они автоматически включаются при создании профиля выставления счетов. Все разделы счетов, связанные с профилем выставления счетов, могут использовать эти планы. Пользователи с доступом к разделу счета используют планы для создания подписок Azure. В учетных записях выставления счетов по клиентскому соглашению Майкрософт поддерживаются следующие планы Azure:

| План             | Определение  |
|------------------|-------------|
|План Microsoft Azure   | Позволяет пользователям создавать подписки, в которых могут выполняться любые рабочие нагрузки.  |
|План Microsoft Azure для разработки и тестирования | Позволяет подписчикам Visual Studio создавать подписки, предназначенные только для рабочих нагрузок разработки или тестирования. Эти подписки получают такие преимущества, как более низкие тарифы и доступ к эксклюзивным образам виртуальных машин на портале Azure. |

## <a name="invoice-sections"></a>Разделы счетов

Создайте раздел счета, чтобы упорядочить затраты по счету. Например, вам может понадобиться один счет для организации, но вам будет необходимо упорядочить затраты по отделу, команде или проекту. В этом сценарии у вас есть один профиль выставления счетов, в котором создается раздел счета для каждого отдела, команды или проекта.

При создании раздела счета можно предоставить другим пользователям разрешение на создание подписок Azure, счета по которым выставляются в разделе. Счета по всем расходам на использование и покупки для подписок будут выставляться в разделе.

Роли в разделе счета имеют разрешения на контроль создания подписок Azure. Назначьте эти роли пользователям, которые настроили среду Azure для команд в нашей организации, например руководителям разработки и техническим архитекторам. Дополнительные сведения см. в разделе о [ролях и задачах раздела счета](../manage/understand-mca-roles.md#invoice-section-roles-and-tasks).

# <a name="what-are-azure-reservations"></a>Общие сведения о резервированиях в Azure

Резервирования Azure помогают сэкономить средства за счет приобретения планов сроком на один или три года для разных продуктов. Это позволяет получить скидку на использование ресурсов. Резервирование может сократить ваши расходы на ресурсы до 72 % с оплатой по мере использования. Резервирования предоставляют скидку при выставлении счетов и не влияют на состояние выполнения ваших ресурсов. Когда вы приобретете резервирование, соответствующая скидка автоматически применяется ко всем подходящим ресурсам.

Вы можете оплачивать резервирование заранее или ежемесячно. Общие затраты на резервирование с предварительными или ежемесячными платежами одинаковы, и вы не будете переплачивать, если выберете ежемесячную оплату. Вариант помесячной оплаты доступен для резервирований Azure, но не для сторонних продуктов.

Резервирование можно приобрести на портале Azure на вкладке [https://portal.azure.com/#blade/Microsoft_Azure_Reservations/ReservationsBrowseBlade](https://portal.azure.com/#blade/Microsoft_Azure_Reservations/ReservationsBrowseBlade).

## <a name="why-buy-a-reservation"></a>Зачем приобретать резервирование?

Если вы стабильно используете ресурс, для которого поддерживаются резервирования, приобретение такого резервирования позволит уменьшить затраты. Например, если у вас постоянно работают несколько экземпляров службы без резервирования, за них взимается обычная оплата по мере использования. Если вы приобретете резервирование, вы сразу же получите на них скидку. К ресурсам больше не будут применяться ставки оплаты по мере использования.

## <a name="how-reservation-discount-is-applied"></a>Применение скидки на резервирование

После приобретения скидка по резервированию автоматически применяется к используемым ресурсам, которые соответствуют атрибутам, выбранным при покупке резервирования. К таким атрибутам относятся номер SKU, регионы (если применимо) и область резервирования. Область резервирования позволяет выбрать, где будет применяться скидка по резервированию.

Дополнительные сведения о применении скидки на зарезервированные экземпляры см. [в этой статье](reservation-discount-application.md).

Дополнительные сведения о том, как работают области резервирования, см. в [этой статье](prepare-buy-reservation.md#scope-reservations).

## <a name="determine-what-to-purchase"></a>Выбор приобретаемых ресурсов 

Все резервирования, кроме Azure Databricks, применяются на почасовой основе. Рассмотрите возможность приобретения резервирования, основываясь на базовом стабильном объеме потребления. Вы можете определить, какое резервирование приобрести, проанализировав данные об использовании или воспользовавшись рекомендациями по резервированию. Чтобы просмотреть рекомендации, воспользуйтесь указанными ниже источниками:

- помощник по Azure (только для виртуальных машин);
- интерфейс покупки резервирования на портале Azure;
- приложение Power BI Azure "Управление затратами";
- Программные интерфейсы 

Дополнительные сведения см. в статье  [Выбор резервирования для приобретения](determine-reservation-purchase.md).

Перейдите на портал Azure, чтобы приобрести резервирование (https://ms.portal.azure.com/#blade/Microsoft_Azure_Reservations/CreateBlade/referrer/Docs) 

Дополнительные сведения см. в статье  [Подготовка к приобретению резервирования](prepare-buy-reservation.md). 

## <a name="how-is-a-reservation-billed"></a>Как оплачивается резервирование? 

Для покупки резервирования используется метод оплаты, привязанный к подписке. Стоимость резервирования вычитается из баланса денежных обязательств (при наличии). Если у вас недостаточно баланса денежных обязательств для покрытия полной стоимости резервирования, вам выставляется счет на соответствующую разницу. Если вы используете подписку из индивидуального плана с оплатой по мере использования, все покупки с предварительной оплатой немедленно оплачиваются с той кредитной карты, которая подключена к учетной записи. Ежемесячные платежи отображаются в счете и ежемесячно взимаются с той же кредитной карты. Если вы используете оплату по счетам, эти расходы будут включены в следующий счет. 

## <a name="permissions-to-view-and-manage-reservations"></a>Разрешения на просмотр резервирований и управление ими 

Пользователь, который приобретает резервирование, а также администратор учетной записи подписки, используемой для выставления счета за резервирование, получают роль владельца в заказе на резервирование и резервировании.

Вы можете делегировать управление резервированием, добавив пользователей в роли в заказе на резервирование или резервировании. Вы назначаете роли на портале Azure или с помощью API и PowerShell. 

Дополнительные сведения см. в разделе  [Добавление или изменение пользователей, которые могут управлять резервированием](manage-reserved-vm-instance.md#add-or-change-users-who-can-manage-a-reservation). 

## <a name="get-reservation-details-and-utilization-after-purchase"></a>Получение сведений о резервировании и использовании после покупки

Если у вас есть разрешение на просмотр резервирования, вы можете просмотреть его, а также сведения о его использовании на портале Azure. Вы также можете получить данные с помощью API. 

Дополнительные сведения о том, как просматривать резервирования на портале Azure, см.  [здесь](view-reservations.md). 

## <a name="manage-reservations-after-purchase"></a>Управление резервированиями после покупки 

После приобретения резервирования Azure вы можете обновить область, чтобы применить резервирование к другой подписке, изменить управляющего, разделить резервирование на более мелкие части или изменить гибкость размера экземпляра. 

Дополнительные сведения см. в статье  [Управление резервированиями для ресурсов Azure](manage-reserved-vm-instance.md). 

## <a name="flexibility-with-azure-reservations"></a>Гибкое использование резервирований в Azure

Azure Reserved Virtual Machine Instances обеспечивают гибкость, позволяющую удовлетворить ваши требования. Зарезервированный экземпляр можно обменять на другой зарезервированный экземпляр того же типа. Также можно вернуть средства за резервирование (до 50 000 долл. США в течение 12 месяцев), если вы больше не используете его. Максимальное ограничение возмещения применяется ко всем резервированиям в области действия соглашения с корпорацией Майкрософт.

Дополнительные сведения см. в статье [Самостоятельный обмен и возмещение средств для резервирований Azure](exchange-and-refund-azure-reservations.md).


## <a name="charges-covered-by-reservation"></a>Расходы, на которые распространяется резервирование

- **Зарезервированный экземпляр виртуальной машины** — резервирование покрывает только расходы на вычислительные ресурсы виртуальных машин и облачных служб. Оно не включает оплату дополнительного программного обеспечения, Windows, сетей и (или) хранилищ.
- **Зарезервированная емкость службы хранилища Azure** — резервирование охватывает емкость хранилища BLOB-объектов или Azure Data Lake 2-го поколения для стандартных учетных записей хранения. Резервирование не охватывает пропускную способность или скорость выполнения транзакций.
- **Резервная мощность Azure Cosmos DB** — резервирование покрывает расходы на пропускную способность, подготовленную для ресурсов. Но она не распространяется на расходы хранилища и сети.
- **Зарезервированные виртуальные ядра Базы данных SQL** — резервирование покрывает расходы только на вычислительные ресурсы. Лицензия SQL оплачивается отдельно.
- **Azure Synapse Analytics** — резервирование покрывает потребление cDWU. Оно не покрывает затраты на хранилище и сетевые ресурсы, связанные с использованием Azure Synapse Analytics.
- **Azure Databricks** — резервирование охватывает только использование DBU. Другие расходы, такие как вычисления, хранение и сетевое подключение, взимаются отдельно.
- **Базовая плата для Службы приложений** — резервирование покрывает базовую плату. Оно не применяется к рабочим ролям, поэтому все прочие ресурсы, связанные с базовой платой, оплачиваются отдельно.
- **База данных Azure для MySQL** — в резервирование включены только затраты на вычисление. Резервирование не покрывает затраты на программное обеспечение, сети или хранилище, связанные с сервером базы данных MySQL.
- **База данных Azure для PostgreSQL** — в резервирование включены только затраты на вычисление. Резервирование не покрывает затраты на программное обеспечение, сети или хранилище, связанные с экземпляром базы данных PostgreSQL.
- **База данных Azure для MariaDB** — в резервирование включены только затраты на вычисление. Резервирование не покрывает затраты на программное обеспечение, сети или хранилище, связанные с экземпляром базы данных MariaDB.
- **Azure Data Explorer** — резервирование покрывает расходы на разметку. Резервирование не применяется к расходам на вычисления, сетевые подключения и хранение, связанные с кластерами.
- **Кэш Azure для Redis** — в резервирование включены только затраты на вычисление. Резервирование не покрывает расходы на сетевые подключения и хранилище, связанные с экземплярами кэша Redis.
- **Выделенный узел Azure** — в резервирование Выделенного узла включены только затраты на вычисление.
- **Резервирование Хранилища дисков Azure** доступно только для SSD (цен. категория "Премиум") размером P30 или больше. Оно не распространяется на диски других типов или размеров меньше, чем P30.

Планы программного обеспечения:

- **SUSE Linux** — резервирование покрывает расходы на план программного обеспечения. Скидки применяются только для единиц измерения SUSE, а не для виртуальных машин.
- **Планы Red Hat** — резервирование покрывает расходы на план программного обеспечения. Скидки применяются только для единиц измерения RedHat, а не для виртуальных машин.
- **Решение VMware для Azure от CloudSimple** — резервирование покрывает узлы VMWare CloudSimple. По-прежнему применяются расходы на дополнительное программное обеспечение.
- **Azure Red Hat OpenShift** — резервирование покрывает расходы на OpenShift, а не расходы на инфраструктуру Azure.

Для виртуальных машин Windows и Базы данных SQL скидка по резервированию не применяется к стоимости программного обеспечения. Вы можете покрыть расходы на лицензирование с помощью программы [Преимущество гибридного использования Azure](https://azure.microsoft.com/pricing/hybrid-benefit/).

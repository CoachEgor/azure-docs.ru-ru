# <a name="how-reservation-recommendations-are-created"></a>Создание рекомендаций по резервированию

Рекомендации по приобретению зарезервированных экземпляров (RI) Azure создаются [API рекомендаций по резервированию Azure](/rest/api/consumption/reservationrecommendations). Рекомендации от API также используются в [Помощнике по Azure](../..//advisor/advisor-cost-recommendations.md#buy-reserved-virtual-machine-instances-to-save-money-over-pay-as-you-go-costs). Помощник отображает рекомендации на портале Azure.

Если у вас есть ВМ, работающие в Azure, вы можете воспользоваться скидкой на RI и предоплатой за свои ВМ. API рекомендаций по потреблению от Майкрософт оценивает использование в течение 7, 30 и 60 дней и рекомендует оптимальные конфигурации для RI. Он рассчитывает затраты, которые вы бы понесли, если бы у вас не было RI, в сравнении с затратами, которые вы бы понесли при использовании RI, чтобы оптимизировать экономию.

Помощник по Azure отображает рекомендации на основе 30-дневного периода.

Для простоты в следующем примере показаны вычисления, которые происходят в течение семи-дневных рекомендаций. Этот же метод применяется при вычислении рекомендаций на 30 или 60 дней.

## <a name="calculation-method-example"></a>Пример метода вычисления

Предположим, что каждый час использования виртуальной машины Windows для конкретного SKU и региона может варьироваться в течение семи дней (168 часов). Минимальное использование — 65 единиц, а максимальное — 127 единиц за семь дней. В этом примере при часе 79 использовались 80 ВМ, и вы приобрели 75 RI.

Если вы приобрели 75 зарезервированных экземпляров, вы оплачиваете следующие затраты на час 79.

- 75 зарезервированных экземпляров. Стоимость предоплаты при покупке RI.
- Зарезервированные экземпляры охватывают затраты на оборудование работающих виртуальных машин, поэтому вы платите за 75 часов по цене только по программному обеспечению.
- Расход за час 79 составляет 80, поэтому вы заплатите за пять часов работы Windows плюс стоимость комбинированного счетчика аппаратного обеспечения. Комбинированная цена зависит от Соглашения Enterprise (EA) или оплаты по мере использования.

При покупке 75 RI вы можете рассчитать общую стоимость, добавив предшествующие почасовые затраты. Вы также можете рассчитать текущие затраты, используя свою ставку. Разница между двумя суммами в этом примере — это ваши сбережения на семь дней.

API выполняет вычисления для каждой конкретной точки использования. Затем возвращается рекомендуемое количество, при котором экономия увеличивается до максимума. В следующем примере диаграмма показывает, что пик экономии — 68. После него экономия сокращается, и поэтому API рекомендует 68.

## <a name="other-expected-api-behavior"></a>Другое ожидаемое поведение API

- API показывает возможную экономию с помощью [Преимущества гибридного использования Azure](https://azure.microsoft.com/pricing/hybrid-benefit/) для Windows, когда используется преимущество. Если преимущество не используется, API рекомендации основываются на основной стоимости Windows. Если это возможно, рассмотрите возможность использования Преимущества гибридного использования Azure, чтобы увеличить экономию.
- При использовании периода наблюдения в семь дней вы можете не получать рекомендации, когда виртуальные машины завершают работу больше, чем на день.

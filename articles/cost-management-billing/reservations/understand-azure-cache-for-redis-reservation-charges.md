# <a name="how-the-reservation-discount-is-applied-to-azure-cache-for-redis"></a>Как применять скидку на резервирование в Кэше Azure для Redis

После того как вы купили зарезервированную емкость Кэша Azure для Redis, скидка на резервирование автоматически применяется к экземплярам кэша, атрибуты и количество которых совпадают с резервированием. Резервирование охватывает только затраты на вычисление для Кэша Azure для Redis. С вас взимается плата за хранение и сеть по обычным тарифам. Функция зарезервированной емкости доступна только для кэшей [категории "Премиум"](/azure/azure-cache-for-redis/quickstart-create-redis).

## <a name="how-reservation-discount-is-applied"></a>Применение скидки на резервирование

Скидка на резервирование * **сгорает, если не используется** _. Поэтому, если у вас не существует подходящих ресурсов в течение некоторого часа, вы потеряете количество ресурсов, зарезервированных на этот час. Вы не можете переносить неиспользуемые зарезервированные часы.

При завершении работы ресурса скидка на резервирование автоматически переносится на другой подходящий ресурс в указанной области резервирования. Если в указанной области не найдены подходящие ресурсы, зарезервированные часы теряются.

## <a name="discount-applied-to-azure-cache-for-redis"></a>Скидка, применяемая к Кэшу Azure для Redis

Скидка для функции зарезервированной емкости Кэша Azure для Redis применяется к вашим кэшам каждый час. Приобретенное резервирование должно соответствовать потреблению вычислительных ресурсов работающими кэшами. Если кэши работают менее часа, резервирование автоматически применяется к другим кэшам, соответствующим атрибутам резервирования. Скидка может применяться к одновременно работающим кэшам. Если у вас нет кэшей, работающих в течение целого часа и соответствующих атрибутам резервирования, вы не получите полную скидку на резервирование за этот час.

В следующих примерах показано применение скидки на зарезервированную емкость Кэша Azure для Redis в зависимости от количества приобретенных вами кэшей и времени их работы.

_ **Пример 1.** вы приобретаете зарезервированную емкость Кэша Azure для Redis для кэша размером 6 ГБ. Если вы используете кэш размером 13 ГБ, который соответствует остальным атрибутам резервирования, вы оплачиваете цену по мере использования за 7 ГБ потребленных вычислительных ресурсов Кэша Azure для Redis и получаете скидку на резервирование за один час использования вычислений кэша размером 6 ГБ.

В остальных примерах предположим, что резервная мощность Кэша Azure для Redis, которую вы приобрели, предназначена для кэша размером 26 ГБ, а остальные атрибуты резервирования соответствуют работающему кэшу.

* **Пример 2**. Вы запускаете 2 кэша размером 13 ГБ в течение часа. Скидка на резервирование 26 ГБ применяется к использованию вычислений обоих кэшей.

* **Пример 3** : вы запускаете 1 кэш размером 26 ГБ с 13:00 до 13:30. Вы запускаете другой кэш размером 26 ГБ с 13:30 до 14:00. Для обеих Баз данных будет применяться скидка на резервирование.

* **Пример 4**. Вы запускаете 1 кэш размером 26 ГБ с 13:00 до 13:45. Вы запускаете другой кэш размером 26 ГБ с 13:30 до 14:00. За превышение работы на 15 минут с вас взимается оплата по мере использования. Скидка на резервирование применяется к использованию вычислительных ресурсов за остальное время.

Чтобы больше узнать о применении резервирований Azure и просмотреть отчеты о начислении платы за использование, ознакомьтесь со статьей [Общие сведения об использовании зарезервированных экземпляров Azure с Соглашением о регистрации Enterprise](./understand-reserved-instance-usage-ea.md).

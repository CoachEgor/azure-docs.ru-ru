# <a name="troubleshoot-azure-reservation-recommendations"></a>Устранение неполадок с рекомендациями по резервированию в Azure

Эта статья поможет вам понять и устранить неполадки с рекомендациями по резервированию Azure, которые отображаются на портале Azure. Вы можете не увидеть рекомендации или увидеть те, которые не соответствуют вашим ожиданиям. Например, у вас уже может быть зарезервированный экземпляр для конкретной конфигурации виртуальной машины. Вы можете увидеть рекомендацию для подобной конфигурации виртуальной машины.

## <a name="symptoms"></a>Симптомы

1. Войдите на [портал Azure](https://portal.azure.com/) и перейдите к разделу **Резервирования**.
2. Щелкните **+ Добавить** , а затем выберите продукт.
3. На вкладке **Рекомендуется** вы можете не увидеть рекомендации или увидеть те, которые не соответствуют вашим ожиданиям.

## <a name="cause"></a>Причина

Список рекомендуемых продуктов создается исходя из количества использованных вами ресурсов для области (объединенной или отдельной) по сравнению с затратами на резервирование для продукта. Если подсистема рекомендаций обнаруживает экономию, она рекомендует продукт для приобретения. Однако если подсистема не выявляет экономии, то не рекомендует продукт.

Если вы используете ресурс с заданной конфигурацией, нет никакой гарантии, что вы сэкономите деньги, приобретя резервирование для этой конфигурации. Например, у вас могут быть единичные случаи использования. В этом случае общая стоимость резервирования может не привести к экономии денег во время существования периода резервирования.

Кроме того, важно понимать, как выбор области влияет на рекомендации. Если для области задано значение **Общая** , в списке рекомендаций отображаются зарезервированные экземпляры, в которых Azure находит экономию для всей регистрации, связанной с подпиской для выставления счетов. Если для области задано значение **Отдельная** , рекомендации в списке применяются только к ресурсам, которые выполняются в подписке. Возможно, некоторые размеры виртуальных машин рекомендуются для одной области, но не для другой. Например, у вас уже может быть объединенное использование **Standard_B1ls** в рамках регистрации, что достаточно объемно, чтобы оправдать затраты на приобретение резервирования в области регистрации. Однако одна подписка в регистрации может не иметь достаточного объема использования, чтобы оправдать затраты на приобретение резервирования в области. Изменение области между **общей** и **отдельной** может привести к различным рекомендациям.

Azure может рекомендовать приобрести резервирование для одних сроков, но не рекомендовать для других, в зависимости от обнаруженной экономии. В частности, сроки на три года имеют более высокие скидки, чем на один год. Вероятнее, что Azure обнаружит экономию в течение срока на три года, чем на один.

Если вы хотите понять, почему Azure рекомендует конкретный размер и количество ресурсов, выберите **&lt;Количество&gt; См. подробности** для получения подробных сведений о возможностях экономии за период времени.

## <a name="solution"></a>Решение

Вы можете купить любое количество резервирований на нужный срок. Приобретение резервирования с количеством и сроком, которые отличаются от рекомендации, скорее всего, приведет к снижению экономии и оптимального использования.

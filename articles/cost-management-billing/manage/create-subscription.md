# <a name="create-an-additional-azure-subscription"></a>Создание дополнительной подписки Azure

Вы можете создать дополнительную подписку для учетной записи выставления счетов для вашего [Соглашения Enterprise (EA)](https://azure.microsoft.com/pricing/enterprise-agreement/), [Клиентского соглашения Майкрософт](https://azure.microsoft.com/pricing/purchase-options/microsoft-customer-agreement/) или [Соглашения с партнером Майкрософт](https://www.microsoft.com/licensing/news/introducing-microsoft-partner-agreement) на портале Azure. Дополнительную подписку можно использовать, чтобы избежать превышения ограничений на подписки и создать отдельные среды для обеспечения безопасности, а также чтобы изолировать данные для обеспечения соответствия.

Если у вас есть учетная запись выставления счетов для программы Microsoft Online Service (MOSP), вы можете создать дополнительные подписки на [портале регистрации Azure](https://account.azure.com/signup?offer=ms-azr-0003p).

Дополнительные сведения об учетных записях выставления счетов и определении типа своей учетной записи выставления счетов, см. раздел [Просмотр учетных записей выставления счетов на портале Azure](view-all-accounts.md).

## <a name="permission-required-to-create-azure-subscriptions"></a>Разрешение, необходимое для создания подписок Azure

Для создания подписки вам необходимы следующие разрешения:

|учетная запись выставления счетов;  |Разрешение  |
|---------|---------|
|Соглашение Enterprise (EA) |  Роль владельца учетной записи при регистрации на соглашение Enterprise. Дополнительные сведения см. в описании [административных ролей в Azure при использовании Соглашения Azure Enterprise](understand-ea-roles.md).    |
|Клиентское соглашение Майкрософт (MCA) |  Роль владельца или участника в разделе счетов, профиле выставления счетов или учетной записи выставления счетов. Или роль создателя подписки Azure в разделе счета.  Дополнительные сведения см. в статье [Роли и задачи выставления счетов в подписке](understand-mca-roles.md#subscription-billing-roles-and-tasks).    |
|Соглашение с партнером Майкрософт (ПСМ) |   Роль глобального администратора и агента администратора в партнерской организации CSP. Дополнительные сведения см. в статье [Назначение пользователям ролей и разрешений](/partner-center/permissions-overview).  Пользователь должен войти в клиент партнера для создания подписок Azure.   |

## <a name="create-a-subscription-in-the-azure-portal"></a>Создание подписки на портале Azure

1. Войдите на [портал Azure](https://portal.azure.com).
1. Выполните поиск по запросу **Подписки**.

   ![Снимок экрана: поиск подписок на портале](./media/create-subscription/billing-search-subscription-portal.png)

1. Выберите **Добавить**.

   ![Снимок экрана: кнопка "Добавить" в представлении "Подписки"](./media/create-subscription/subscription-add.png)

1. Если у вас есть доступ к нескольким учетным записям, выберите учетную запись выставления счетов, для которой хотите создать подписку.

1. Заполните форму и щелкните **Создать**. В таблицах ниже перечислены поля в форме для каждого типа учетной записи выставления счетов.

**Соглашение Enterprise**

|Поле  |Определение  |
|---------|---------|
|Имя     | Отображаемое имя поможет легко найти подписку на портале Azure.  |
|ПРЕДЛОЖЕНИЕ     | Выберите "Разработка и тестирование EA", если планируете использовать эту подписку для разработки или тестирования рабочих нагрузок, иначе используйте Microsoft Azure Enterprise. Предложение "Разработка и тестирование" должно быть включено для вашей учетной записи регистрации для создания подписок для "Разработки и тестирования EA".|

**Клиентское соглашение Майкрософт**

|Поле  |Определение  |
|---------|---------|
|Профиль выставления счетов     | Плата за подписку будет взиматься по выбранному профилю выставления счетов. Если у вас есть доступ только к одному профилю выставления счетов, выбор будет недоступен.     |
|Раздел счета     | Плата за подписку будет отображаться в этом разделе счета профиля выставления счетов. Если у вас есть доступ только к одному разделу счета, выбор будет недоступен.  |
|План     | Выберите План Microsoft Azure для разработки и тестирования, если планируете использовать эту подписку для рабочих нагрузок разработки или тестирования, иначе используйте План Microsoft Azure. Если для профиля выставления счетов включен только один план, выбор будет недоступен.  |
|Имя     | Отображаемое имя поможет легко найти подписку на портале Azure.  |

**Соглашение с партнером Майкрософт**

|Поле  |Определение  |
|---------|---------|
|Customer    | Подписка создается для выбранного вами клиента. Если у вас есть доступ только к одному клиенту, выбор будет недоступен.  |
|Reseller    | Торговый посредник, который будет предоставлять службы клиенту. Это необязательное поле, применимое только к непрямым поставщикам в двухуровневой модели CSP. |
|Имя     | Отображаемое имя поможет легко найти подписку на портале Azure.  |

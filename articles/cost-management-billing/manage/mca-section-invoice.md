# <a name="organize-costs-by-customizing-your-billing-account"></a>Организация затрат путем настройки счета для выставления счетов

Учетная запись выставления счетов для соглашения клиента Майкрософт предоставляет гибкие возможности для Организации затрат в зависимости от потребностей Отдела, проекта или среды разработки. 

В этой статье описывается, как можно использовать портал Azure для Организации затрат. Он применяется к учетной записи выставления счетов для соглашения клиента Майкрософт. [Проверьте наличие доступа к Клиентскому соглашению Майкрософт](#check-access-to-a-microsoft-customer-agreement).

## <a name="structure-your-account-with-billing-profiles-and-invoice-sections"></a>Структурирование учетной записи с помощью профилей выставления счетов и разделов накладных

В учетной записи выставления счетов для соглашения клиента Майкрософт вы используете профили выставления счетов и разделы счета для Организации затрат.

![Снимок экрана, на котором показана иерархия выставления счетов для соглашения клиента Майкрософт](./media/mca-section-invoice/mca-hierarchy.png)

### <a name="billing-profile"></a>Профиль выставления счетов

Профиль выставления счетов представляет счет и связанные с ним сведения, такие как способы оплаты и адрес для выставления счетов. В начале месяца для каждого профиля выставления счетов в вашей учетной записи создается счет за месяц. Счет содержит оплату за использование Azure и другие покупки за предыдущий месяц.

Профиль выставления счетов автоматически создается вместе с учетной записью выставления счетов при регистрации в Azure. Вы можете создать дополнительные профили выставления счетов для Организации затрат в нескольких ежемесячных счетах. 

> [!IMPORTANT]
>
> Создание дополнительных профилей выставления счетов может повлиять на общую стоимость. Дополнительные сведения см. [в разделе факторы, которые следует учитывать при добавлении новых профилей выставления счетов](#things-to-consider-when-adding-new-billing-profiles).

### <a name="invoice-section"></a>Раздел счета

Раздел счета представляет группирование затрат в счете. Раздел счета создается автоматически для каждого профиля выставления счетов в вашей учетной записи. Вы можете создать дополнительные разделы для Организации затрат в соответствии с вашими потребностями. Каждый раздел счета отображается в счете с оплатой за этот месяц. 

На рисунке ниже показан счет с двумя разделами счета-инженеров и маркетингом. В счете отображается сводка и подробные сведения по каждому разделу. Цены, показанные на изображении, предназначены только для примера и не представляют фактические цены на службы Azure. 

![Изображение, показывающее счет с разделами](./media/mca-section-invoice/mca-invoice-with-sections.png)

## <a name="create-a-new-invoice-section"></a>Создание нового раздела счета

Чтобы создать раздел счета, вы должны быть **владельцем учетной записи выставления счетов** или **поставщиком учетной записи выставления счетов**. Дополнительные сведения см. в разделе [Управление разделом счета для профиля выставления счетов](understand-mca-roles.md#manage-invoice-sections-for-billing-profile).

1. Войдите на [портал Azure](https://portal.azure.com).

2. Выполните поиск по фразе **Управление затратами + выставление счетов**.

   ![Снимок экрана с изображением поиска на портале по фразе "Управление затратами + выставление счетов"](./media/mca-section-invoice/search-cmb.png)

3. В области слева выберите **Профили выставления счетов** . В списке выберите профиль выставления счетов. Новый раздел будет отображаться для выбранного счета профиля выставления счетов. 

   [Снимок экрана ![, отображающий список профилей выставления счетов](./media/mca-section-invoice/mca-select-profile.png)](./media/mca-section-invoice/mca-select-profile-zoomed-in.png#lightbox)

4. Выберите **раздел счета** в левой области, а затем выберите **Добавить** в верхней части страницы.

   [Снимок экрана ![, на котором показано добавление разделов счета](./media/mca-section-invoice/mca-list-invoice-sections.png)](./media/mca-section-invoice/mca-list-invoice-sections-zoomed-in.png#lightbox)

5. Введите имя раздела счета. 

   [Снимок экрана ![, на котором показана страница создания раздела счета](./media/mca-section-invoice/mca-create-invoice-section.png)](./media/mca-section-invoice/mca-create-invoice-section-zoomed-in.png#lightbox)

6. Нажмите кнопку **Создать**.

## <a name="create-a-new-billing-profile"></a>Создание нового профиля выставления счетов

Чтобы создать профиль выставления счетов, необходимо быть **владельцем учетной записи выставления** счетов или **участником счета выставления счетов**. Дополнительные сведения см. в разделе [Управление профилями выставления счетов для счета выставления счетов](understand-mca-roles.md#manage-billing-profiles-for-billing-account).

> [!IMPORTANT]
>
> Создание дополнительных профилей выставления счетов может повлиять на общую стоимость. Дополнительные сведения см. [в разделе факторы, которые следует учитывать при добавлении новых профилей выставления счетов](#things-to-consider-when-adding-new-billing-profiles).

1. Войдите на [портал Azure](https://portal.azure.com).

2. Выполните поиск по фразе **Управление затратами + выставление счетов**.

   ![Снимок экрана с изображением поиска на портале по фразе "Управление затратами + выставление счетов"](./media/mca-section-invoice/search-cmb.png)

3. Выберите **Профили выставления счетов** в левой области, а затем выберите **Добавить** в верхней части страницы.

   [Снимок экрана ![, отображающий список профилей выставления счетов](./media/mca-section-invoice/mca-list-profiles.png)](./media/mca-section-invoice/mca-list-profiles-zoomed-in.png#lightbox)

    > [!Note]
    >
    > Если кнопка добавить не отображается на странице Профиль выставления счетов, эта функция недоступна для вашей учетной записи. Сейчас она доступна только для учетных записей, настроенных при работе с представителями корпорации Майкрософт.

4. Заполните форму и щелкните **Создать**.

   [Снимок экрана ![, отображающий страницу создания профиля выставления счетов](./media/mca-section-invoice/mca-add-profile.png)](./media/mca-section-invoice/mca-add-profile-zoomed-in.png#lightbox)

    |Поле  |Определение  |
    |---------|---------|
    |Имя     | Отображаемое имя, помогающее легко найти профиль выставления счетов в портал Azure.  |
    |Номер заказа    | Необязательный номер заказа на покупку. Номер заказа на покупку будет отображаться в счетах, созданных для профиля выставления счетов. |
    |Адрес выставления счетов   | Адрес для выставления счетов будет отображаться в счетах, созданных для профиля выставления счетов. |
    |Счет по электронной почте   | Установите флажок "счет по электронной почте", чтобы получить счета для этого профиля выставления счетов по электронной почте. Если вы не наберете согласие, вы можете просмотреть и скачать счета в портал Azure.|

5. Нажмите кнопку **Создать**.

## <a name="link-charges-to-invoice-sections-and-billing-profiles"></a>Связь взимается с разделами счета и профилями выставления счетов

После настройки учетной записи для выставления счетов на основе ваших потребностей можно связать подписки и другие продукты с нужным разделом счета и профилем выставления счетов.

### <a name="link-a-new-subscription"></a>Связать новую подписку

1. Войдите на [портал Azure](https://portal.azure.com).

2. Выполните поиск по запросу **Подписки**.

   [Снимок экрана ![, отображающий Поиск на портале для подписки](./media/mca-section-invoice/search-subscriptions.png)](./media/mca-section-invoice/search-subscriptions.png#lightbox)

3. В верхней части страницы выберите **Добавить**.

   ![Снимок экрана: кнопка "Добавить" в представлении "Подписки"](./media/mca-section-invoice/subscription-add.png)

4. Если у вас есть доступ к нескольким счетам на выставление счетов, выберите счет для выставления счетов в рамках соглашения с клиентами Майкрософт.

   ![Снимок экрана: кнопка "Добавить" в представлении "Подписки"](./media/mca-section-invoice/mca-create-azure-subscription.png)

5. Выберите профиль выставления счетов, для которого будет выставлен счет за использование подписки. Плата за использование Azure и другие покупки для этой подписки будет взиматься по выбранному счету профиля выставления счетов.

6. Выберите раздел счет, чтобы связать расходы по подписке. Плата будет отображаться в этом разделе в счете профиля выставления счетов.

7. Выберите план Azure и введите понятное имя для своей подписки. 

9. Нажмите кнопку **Создать**.  

### <a name="link-existing-subscriptions-and-products"></a>Связывание существующих подписок и продуктов

Если у вас есть подписки Azure или другие продукты, такие как Azure Marketplace и исходные ресурсы приложения, их можно переместить из раздела "существующие счета" в другой раздел "счет" для реорганизации затрат. 

> [!IMPORTANT]
>
> Подписки и другие продукты можно перемещать только между разделами накладных, которые принадлежат одному профилю выставления счетов. Перемещение подписок и продуктов между разделами накладных в разных профилях выставления счетов не поддерживается.

1. Войдите на [портал Azure](https://portal.azure.com).

2. Выполните поиск по фразе **Управление затратами + выставление счетов**.

   ![Снимок экрана: поиск подписок на портале](./media/mca-section-invoice/search-cmb.png)

3. Чтобы связать подписку с новым разделом счета, выберите **подписки Azure** в левой части экрана. Для других продуктов, таких как Azure Marketplace и исходные ресурсы приложений, выберите **повторяющиеся платежи**.

   [![снимок экрана, на котором показан параметр изменения раздела счета](./media/mca-section-invoice/mca-select-change-invoice-section.png)](./media/mca-section-invoice/mca-select-change-invoice-section.png#lightbox)

4. На странице щелкните многоточие (три точки) для подписки или продукта, которые нужно связать с новым разделом счета. Выберите **изменить счет в разделе**.

5. В раскрывающемся списке выберите раздел новый счет. В раскрывающемся списке отображаются только разделы счета, связанные с тем же профилем выставления счетов, что и у существующего раздела счета.

    [Снимок экрана ![, на котором показано, как выбрать новый раздел счета](./media/mca-section-invoice/mca-select-new-invoice-section.png)](./media/mca-section-invoice/mca-select-new-invoice-section-zoomed-in.png#lightbox)

6. Щелкните **Сохранить**.

## <a name="things-to-consider-when-adding-new-billing-profiles"></a>Факторы, которые следует учитывать при добавлении новых профилей выставления счетов

### <a name="azure-usage-charges-may-be-impacted"></a>Могут возникать расходы на использование Azure

В учетной записи выставления счетов для соглашения клиента Майкрософт Использование Azure вычислено ежемесячно для каждого профиля выставления счетов. Цены на ресурсы Azure с многоуровневыми ценами определяются на основе использования каждого профиля выставления счетов отдельно. При вычислении цены использование не объединяется по профилям выставления счетов. Это может повлиять на общую стоимость использования Azure для учетных записей с несколькими профилями выставления счетов.

Рассмотрим пример того, как издержки отличаются в двух сценариях. Цены, используемые в сценариях, предназначены только для примера и не представляют фактические цены на службы Azure.

#### <a name="you-only-have-one-billing-profile"></a>У вас есть только один профиль выставления счетов.

Предположим, что вы используете хранилище блочных BLOB-объектов Azure, стоимость которых USD. 00184 за ГБ для первых 50 терабайт (ТБ), а затем 00177 за ГБ за следующие 450 терабайты (ТБ). Вы использовали 100 ТБ в подписках, которые выставляются в профиле выставления счетов, Вот сколько будет взиматься плата.

|  Цены на уровень (долл. США) |Количество | Сумма (долл. США)|
|---------|---------|---------|
|1,84 за ТБ за первые 50 ТБ в месяц    | 50 ТБ        | 92,0   |
|1,77 за ТБ за следующие 450 ТБ в месяц    |  50 ТБ         | 88,5   |
|Итого     |     100 ТБ  | 180,5

Общая плата за использование 100 ТБ данных в этом сценарии составляет **180,5**

#### <a name="you-have-multiple-billing-profiles"></a>У вас есть несколько профилей выставления счетов.

Теперь предположим, что вы создали еще один профиль выставления счетов и использовали 50 Гб через подписки, счета которых выставляются в первый профиль выставления счетов, а 50 Гб — через подписки, которые выставляются во второй профиль выставления счетов.

`Charges for the first billing profile`

|  Цены на уровень (долл. США) |Количество | Сумма (долл. США)|
|---------|---------|---------|
|1,84 за ТБ за первые 50 ТБ в месяц    | 50 ТБ        | 92,0  |
|1,77 за ТБ за следующие 450 ТБ в месяц    |  0 TБ         | 0,0  |
|Итого     |     50 ТБ  | 92,0 

`Charges for the second billing profile`

|  Цены на уровень (долл. США) |Количество | Сумма (долл. США)|
|---------|---------|---------|
|1,84 за ТБ за первые 50 ТБ в месяц    | 50 ТБ        | 92,0  |
|1,77 за ТБ за следующие 450 ТБ в месяц    |  0 TБ         | 0,0  |
|Итого     |     50 ТБ  | 92,0 

Общая плата за использование 100 ТБ данных в этом сценарии составляет **184,0** (92,0 * 2).

### <a name="azure-reservation-benefits-might-not-apply-to-all-subscriptions"></a>Преимущества резервирования Azure могут не применяться ко всем подпискам.

Резервирования Azure с общей областью применения применяются к подпискам в одном профиле выставления счетов и не являются общими для профилей выставления счетов. 

![Изображение сведений о приложении резервирования для разных структур счетов](./media/mca-section-invoice/mca-reservations-benefits-by-bg.png)

На приведенном выше рисунке компания Contoso имеет две подписки. Преимущество резервирования Azure применяется по-разному в зависимости от структуры счета выставления счетов. В сценарии, расположенном слева, преимущество резервирования применяется к обеим подпискам, выставленным в профиле выставления счетов. В сценарии, расположенном справа, преимущество резервирования будет применено только к подписке 1, так как это единственная подписка, выставленная для профиля выставления счетов.

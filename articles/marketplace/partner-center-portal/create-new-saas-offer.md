# <a name="create-a-new-saas-offer"></a>Create a new SaaS offer (Создание предложения SaaS)

Чтобы начать создавать предложения Software as a Service (SaaS), убедитесь, что сначала создадите [учетную запись «Центр партнеров»](./create-account.md) и откроете [панель мониторинга Commercial Marketplace](https://partner.microsoft.com/dashboard/commercial-marketplace/offers)с выбранной вкладкой **«Обзор».**

![Панель мониторинга коммерческого рынка в партнерской центре](./media/new-offer-overview.png)

>[!Note]
> После публикации предложения, внеочередное предложение, сделанное в Partner Center, будет обновляться только в системе и витринах магазинов после повторного опубликования. Убедитесь, что вы отправляете предложение для публикации после внесения изменений.

Выберите **новое предложение...** кнопку, а затем выберите **Программное обеспечение в качестве элемента меню службы.**

Если вы выберете другой тип предложения, вы можете быть перенаправлены на старый [портал Cloud Partner.](https://cloudpartner.azure.com/) В настоящее время на портале Коммерческого маркетплейса в Партнерском центре доступны только предложения SaaS и Dynamics 365.

![Создание окна предложения в партнерской центре](./media/new-offer-click.png)

Отображается диалоговая коробка **Нового предложения.**

![Новое предложение диалоговая коробка](./media/new-offer-popup.png)

## <a name="offer-id-and-alias"></a>Идентификатор предложения и псевдоним

- **Идентификатор предложения**: Уникальный идентификатор для каждого предложения в вашей учетной записи. Этот идентификатор будет виден клиентам в URL-адресе для предложений на рынке и шаблонах Azure Resource Manager (если это применимо). Идентификатор предложения должен быть нижним регистром, алфавитом (включая дефисы и подчеркнутые, но не имеет пробела). **Идентификатор предложения** ограничен 50 символами и не может быть изменен после выбора *Create.*  
Пример: тест-предложение-1
<br>Результат в URL:`https://azuremarketplace.microsoft.com/marketplace/../test-offer-1`

- **Предложение псевдоним**: Имя, используемое для обозначения предложения в партнерской центр портала. Это имя не будет использоваться на рынке и отличается от *названия предложения* и других значений, которые будут показаны клиентам. Это значение не может быть изменено после выбора *Create.*

<br>Пример: Тест-предложение 1&#8482;

Выберите **Создать**.  Для этого предложения создается страница **обзора предложения.**

## <a name="offer-overview"></a>Обзор предложения

Страница **обзора предложения** включает в себя:

- **Статус Публикации** отображает визуальное представление шагов, необходимых для публикации этого предложения, и того, сколько времени потребуется каждому шагу для завершения. Неполное издание значков шага будет серым.

- **Меню обзора предложения** содержит список ссылок для выполнения операций по этому предложению. Этот список операций будет меняться в зависимости от выбора, который вы сделаете для вашего предложения.  
    - Если предложение является проект - Удалить проект

    - Если вы еще не завершили издателя выйти - Отменить публикацию

Продажа через microsoft обеспечивает лучшее обнаружение и приобретение клиентов, позволяет корпорации Майкрософт размещать транзакции на рынке от вашего имени и использует возможности Microsoft, доступные в глобальном масштабе.

#### <a name="saas-offer-requirements"></a>Требования к предложению SaaS

Для того, чтобы перечислить программное обеспечение как услугу (SaaS) предложения с коммерческим marketplace на партнерский центр, следующие критерии должны быть выполнены:

- Ваше предложение должно использовать [Active Directory Azure (Azure AD)](https://azure.microsoft.com/services/active-directory/) для управления ими и проверки подлинности.
- Ваше предложение должно использовать [AAP SaaS Fulfillment Для](https://docs.microsoft.com/azure/marketplace/partner-center-portal/pc-saas-fulfillment-api-v2) интеграции в Azure Marketplace.
- Для получения более обширных требований, см [SaaS Предложение Издательский путеводитель](https://docs.microsoft.com/azure/marketplace/marketplace-saas-applications-technical-publishing-guide).

#### <a name="saas-pricing-and-billing-options"></a>SaaS ценообразования и опционов на выставление счетов
С помощью решений SaaS, работающих в подписке на Azure издателя, лицензионные сборы, уплачиваемые клиентами, включают стоимость инфраструктуры, на которой развернуто программное обеспечение. Использование инфраструктуры Azure управляется и выставляет счет непосредственно вам, партнеру. Фактическая стоимость использования инфраструктуры не видна пользователю. Издатели должны увязывать сборы за использование инфраструктуры Azure в цены лицензий на программное обеспечение. 

SaaS предлагает поддержку ежемесячного или ежегодного выставления счетов на основе фиксированной платы, на одного пользователя или платы за потребление с помощью дозированной платежной службы. Коммерческий рынок Майкрософт работает по агентской модели, в которой издатели устанавливают цены, Microsoft выставляет счета клиентам, а корпорация Майкрософт выплачивает доход издателю, удерживая при этом агентскую плату.

В следующей таблице приводится пример разбивки затрат и выплат, чтобы продемонстрировать модель агентства.

|**Стоимость лицензии**|**$100 в месяц**|
|:---|:---|
|Стоимость использования Azure (D1 с 1 ядром)|Счета выставляются непосредственно издателю, а не клиенту.|
|Счет выставляется корпорацией Майкрософт|$100.00 в месяц (Издатель должен учитывать любые понесенные или сквозные расходы на инфраструктуру в лицензионном сборе)|

|**Майкрософт выставляет счет**|**$100 в месяц**|
|:---|:---|
|Корпорация Майкрософт оплачивает 80 % стоимости лицензии <br>**Для квалифицированных приложений SaaS корпорация Майкрософт оплачивает 90% стоимости лицензии*|80,00 долл. США в месяц <br>*$* 90.00 в месяц|

- В этом примере корпорация Майкрософт выставляет клиенту $100.00 за лицензию на программное обеспечение и выплачивает издателю $80,00.
- Партнеры, которые получили право на снижение платы за **услуги Marketplace,** увидят сниженную плату за транзакцию по предложениям SaaS с мая 2019 года по июнь 2020 года. В этом случае, Microsoft счета $ 100.00 для лицензии на программное обеспечение и выплачивает $ 90,00 для издателя.

> [!NOTE]
> **Сниженная плата за обслуживание Marketplace:** Для некоторых предложений SaaS, которые вы опубликовали на нашем коммерческом рынке, корпорация Майкрософт снизит плату за обслуживание Marketplace с 20% (как описано в соглашении Microsoft Publisher) до 10%. Для того, чтобы ваше предложение, чтобы претендовать, по крайней мере одно из ваших предложений должны быть назначены Microsoft как либо IP совместно продать готовы или IP совместно продать приоритеты.  Для получения этой сниженной платы за обслуживание на рынке необходимо выдать не менее пяти (5) рабочих дней до конца каждого календарного месяца.  Сниженная плата за обслуживание marketplace не распространяется на vMs, управляемые приложения или любые другие продукты, доступные через наш коммерческий рынок.  Сниженная плата за обслуживание marketplace будет доступна только для квалифицированных предложений по лицензионным сборам, собранным корпорацией Майкрософт в период с 1 мая 2019 года по 30 июня 2020 года.  По истечение этого времени плата за обслуживание Marketplace вернется к нормальной сумме.

##### <a name="get-it-now-free"></a>Получить его сейчас (бесплатно)
Перечислите свое предложение клиентам бесплатно, предоставив действительный URL (начиная с *http* или *https),* где они могут получить пробную версию с [помощью проверки подлинности одним щелчком мыши с помощью Azure Active Directory (Azure AD).](https://docs.microsoft.com/azure/marketplace/marketplace-saas-applications-technical-publishing-guide#using-azure-active-directory-to-enable-trials)  Например: `https://contoso.com/saas-app`

##### <a name="free-trial-listing"></a>Бесплатная пробная версия (листинг)
Перечислите свое предложение клиентам со ссылкой на бесплатную пробную версию, предоставив действительный URL (начиная с *http* или *https),* где они могут получить пробную версию с [помощью проверки подлинности одним щелчком мыши с помощью Azure Active Directory (Azure AD).](https://docs.microsoft.com/azure/marketplace/marketplace-saas-applications-technical-publishing-guide#using-azure-active-directory-to-enable-trials)  Например: `https://contoso.com/trial/saas-app`. Бесплатные пробные версии предложения создаются, управляются и настраиваются вашей службой и не имеют подписок, управляемых корпорацией Майкрософт.

> [!NOTE]
> Токены, которые ваше приложение получит через пробную ссылку, могут использоваться только для получения пользовательской информации через Azure AD для автоматизации создания учетной записи в приложении. Учетные записи Майкрософт (MSA) не поддерживаются для проверки подлинности с помощью этого маркера.

### <a name="standard-contract-for-the-microsoft-commercial-marketplace"></a>Стандартный контракт для коммерческого рынка Майкрософт

Корпорация Майкрософт предоставляет шаблон Standard Contract.

- **Используете стандартный контракт для коммерческого рынка Майкрософт?**

Чтобы упростить процесс закупок для клиентов и снизить юридическую сложность для поставщиков программного обеспечения, корпорация Майкрософт предлагает стандартный контракт для коммерческого рынка Майкрософт, чтобы облегчить транзакции на рынке. Вместо того, чтобы создавать пользовательские условия, коммерческие издатели рынка могут выбрать, чтобы предложить свое программное обеспечение в соответствии со Стандартным контрактом, который клиенты должны только проверить и принять один раз. Стандартный контракт можно найти https://go.microsoft.com/fwlink/?linkid=2041178здесь: .

Вы можете выбрать для использования стандартного контракта вместо предоставления собственных пользовательских условий, выбрав "Использовать стандартный контракт для коммерческого рынка" флажок.

![Использование флажка Standard Contract](./media/use-standard-contract.png)

> [!NOTE]
> Как только вы публикуете предложение, используя стандартный контракт для коммерческого рынка Майкрософт, вы не сможете использовать свои собственные пользовательские условия. Это "или" сценарий. Вы либо предлагаете свое решение по стандартному контракту, **либо** на своих условиях. Если вы хотите изменить условия Стандартного контракта, вы можете сделать это с помощью стандартных поправок к контракту.

#### <a name="standard-contract-amendments"></a>Стандартные поправки к контракту

Стандартные поправки к контракту позволяют издателям выбирать условия стандартного контракта для простоты и настраивать условия для своего продукта или бизнеса. Клиентам необходимо ознакомиться с поправками к договору только в том случае, если они уже рассмотрели и приняли стандартный контракт Майкрософт.

Есть два вида поправок, доступных для издателей коммерческого рынка:

- Всеобщие поправки: Эти поправки применяются повсеместно к Стандартному контракту для всех клиентов. Универсальные поправки показаны каждому клиенту предложения в потоке покупки. Клиенты должны принять условия Стандартного контракта и поправки, прежде чем они смогут использовать ваше предложение.
- Пользовательские поправки: Эти поправки являются специальными поправками к Стандартному договору, которые предназначены для конкретных клиентов только через иные иутмы арендаторов Azure. Издатели могут выбрать арендатора, на который они хотят нацелиться. Только клиенты от арендатора будут представлены пользовательские условия изменения в потоке покупки предложения.  Клиенты должны принять условия Стандартного контракта и поправки(ы), прежде чем они смогут использовать ваше предложение.

>[!NOTE]
> Эти два типа поправок стек друг на друга. Клиенты, ориентированные на пользовательские поправки, также получат универсальную поправку к Стандартному контракту во время покупки.

**Универсальные условия поправки к Стандартному контракту для коммерческого рынка Microsoft:** Введите универсальные условия поправки в этом поле. Вы можете предоставить единую погоду на предложение. Вы можете ввести неограниченное количество символов в этой коробке. ## <a name="technical-configuration"></a>Техническая конфигурация

Вкладка **Техническая конфигурация** определяет технические детали (путь URL, веб-крюк, идентификатор клиента и идентификатор приложения), используемые для подключения к вашему предложению. Это соединение позволяет нам предоставить ваше предложение для конечном клиента, если они решат приобрести его. Диаграммы, описывающие использование собранных полей, доступны в документации для [AAS выполнения AIS.](https://docs.microsoft.com/azure/marketplace/partner-center-portal/pc-saas-fulfillment-api-v2)

- **URL-адрес страницы посадки** (требуется): Определите URL-адрес сайта, на который клиенты приземлятся после получения вашего предложения с рынка. Этот URL-адрес будет конечным пунктом, который получает маркер при направлении клиента на страницу. Этот токен можно обменять на детали подготовки, используя разрешение в AIS выполнения. Эти сведения и любые другие, которые вы собираете, могут быть использованы как часть интерактивной веб-страницы, построенной в вашем опыте для завершения регистрации и активации их покупки.

- **Подключение webhook** (требуется): Для всех асинхронных событий, которые Microsoft должна отправить вам от имени клиента (например: Подписка SaaS стала недействительной), мы требуем, чтобы вы предоставили веб-крюк соединения. Если у вас еще нет системы webhook на месте, простейшая конфигурация заключается в том, чтобы иметь HTTP Endpoint Logic App,\/который будет слушать любые события, размещенные на него, а затем обрабатывать их надлежащим образом (например, https: / prod-1westus.logic.azure.com:443/work). Подробнее см. статью [Вызовы, срабатывания триггеров и создание вложенных рабочих процессов в приложениях логики](https://docs.microsoft.com/azure/logic-apps/logic-apps-http-endpoint).

- **Идентификатор арендатора Azure AD** (требуется): Внутри портала Azure мы требуем, чтобы вы [создали приложение Active Directory (AD),](https://docs.microsoft.com/azure/active-directory/develop/howto-create-service-principal-portal) чтобы мы могли проверить связь между нашими двумя службами, которая стоит за аутентифицированной связью. Чтобы найти [идентификатор клиента,](https://docs.microsoft.com/azure/active-directory/develop/howto-create-service-principal-portal#get-values-for-signing-in)перейдите в свой каталог Azure Active Directory и выберите **свойства,** затем ищите номер **идентификатора каталога** (например, 50c464d3-4930-494c-963c-1e951d15360e).

- **Идентификатор приложения Azure AD** (требуется): Вам также нужен [идентификатор приложения](https://docs.microsoft.com/azure/active-directory/develop/howto-create-service-principal-portal#get-values-for-signing-in) и ключ проверки подлинности. Чтобы получить эти значения, перейдите в свой каталог Azure Active Directory и выберите **регистрацию приложений,** затем ищите номер **идентификатора приложения** (например, 50c464d3-4930-494c-963c-1e951d15360e). Чтобы найти ключ проверки подлинности, перейдите в **Настройки** и выберите **ключи.** Вам нужно будет предоставить описание и продолжительность, а затем будет предоставлено значение числа.

>[!Note]
>Идентификатор приложения Azure связан с идентификатором издателя, поэтому убедитесь, что один идентификатор приложения используется во всех ваших предложениях.

## <a name="plan-overview"></a>Обзор плана

Вкладка **обзора плана** позволяет предоставить различные варианты плана в рамках одного и того же предложения. Эти планы (иногда называемые СКУ) могут отличаться с точки зрения версии, монетизации или уровней обслуживания. Вы должны настроить по крайней мере один план для того, чтобы продать свое предложение на рынке.

После создания вы увидите имена планов, идентионные данные, модели ценообразования, доступность (государственную или частную), текущее состояние публикации и любые доступные действия.

**Действия,** доступные в **обзоре Плана,** варьируются в зависимости от текущего состояния плана и могут включать:

- Если статус плана **является проектом** - Удалить проект
- Если статус плана **Live** - Остановить продать план или синхронизации частной аудитории

**Создание нового плана** (минимум одного плана для тех, кто решил продать через Microsoft)

- **Идентификатор плана:** Создайте уникальный идентификатор плана для каждого плана в этом предложении. Этот идентификатор будет виден клиентам в URL-адресе продукта и шаблонах Azure Resource Manager (если это применимо). Используйте только нижний регистр, буквенно-цифровые символы, тире или подчеркивает. Для идентификатора этого плана допускается не более 50 символов. Идентификатор не может быть изменен после выбора создания.
- **Название плана:** Клиенты увидят это имя при принятии решения о том, какой план выбрать в рамках вашего предложения. Создайте уникальное имя предложения для каждого плана в этом предложении. Название плана используется для дифференциации программных планов, которые могут быть частью одного и того же предложения (например, название предложения: Windows Server; планы: Windows Server 2016, Windows Server 2019).

### <a name="plan-listing"></a>Список планов

Вкладка **списка плана** отображает языки (и рынки), где ваш план доступен, в настоящее время английский (Соединенные Штаты) является единственным доступным местом. Кроме того, на этой странице отображается статус конкретного списка и дата/время его добавления. Вам нужно будет определить данные рынка (имя предложения, описание, условия поиска и т.д.) для каждого языка / рынка.

#### <a name="plan-listing-details"></a>Подробная информация о плане

При выборе одного из языков плана будет отображаться информация о **списке плана,** включая **имя** и **описание.**

- **Имя**: Предварительно заполнено на основе вашего предварительного просмотра **Новый план** вступления и будет отображаться как название вашего предложения "Программный план", отображается на рынке.
- **Описание:** Это описание дает возможность объяснить, что делает этот программный план уникальным, и любые отличия от других программных планов в вашем предложении. Может содержать до 500 символов.

Выберите **Сохранить** после завершения этих полей.

#### <a name="plan-pricing-and-availability"></a>Планирование ценообразования и доступности

Вкладка **Ценообразования и доступности** позволяет настроить рынки, на которые будет доступен этот план, желаемую модель монетизации, цену и срок выставления счетов. Кроме того, можно указать, следует ли сделать план видимым для всех или только для конкретных клиентов (частной аудитории).

##### <a name="enabling-free-trials"></a>Включение бесплатных испытаний

SaaS предложения через коммерческий рынок позволяют предоставить один месяц бесплатной пробной версии при продаже через Microsoft. Для всех моделей выставления счетов и терминов, за исключением дозированных планов, поддерживаются бесплатные пробные версии. Эта опция позволяет клиентам иметь низкий барьер для входа через один месяц свободного доступа.  Если вы решите включить бесплатную пробную версию планов в рамках вашего предложения, клиент не сможет перейти на платную подписку до конца первоначального месячного периода.  В течение этого времени, клиенты, покупающие ваше предложение, могут опробовать любой из поддерживаемых планов, которые имеют бесплатную пробную версию и конвертируют между ними.  Преобразование в платную подписку осуществляется автоматически в конце семестра.

>[!Note]
>Если клиент решит перейти на план без бесплатных пробных версий, конверсия произойдет, но бесплатная пробная версия будет немедленно потеряна.  Кроме того, как только клиент начинает платить за план, они больше не могут получить бесплатную пробную версию по той же подписке снова, даже если они преобразуют в SKU, который поддерживает бесплатные пробные версии.

Возможность настройки бесплатной пробной версии доступна для каждого плана в вашем предложении. Просто перейдите к ценообразованию и доступности для каждого предложения и проверьте поле, чтобы один месяц пробной версии.

![Один месяц бесплатной пробной флажк](./media/free-trial-enable.png)

>[!Note]
>После того, как ваше трансформатируемое предложение было опубликовано с бесплатной пробной версией, оно не может быть отключено для этого плана. Убедитесь, что эта настройка верна для первого публикации, чтобы избежать необходимости повторного создания плана.

Для получения информации о подписках клиентов, участвующих в `isFreeTrial`настоящее время в бесплатной пробной версии, используйте новое свойство API, которое будет помечено как истинное или ложное. Для получения дополнительной [информации](https://docs.microsoft.com/azure/marketplace/partner-center-portal/pc-saas-fulfillment-api-v2#get-subscription)см.

>[!Note]
>Бесплатные пробные версии не поддерживаются для планов, которые используют услугу учета рынка.

#### <a name="markets"></a>Рынки

- **Рынок edit** (необязательно)

Каждый план должен быть доступен по крайней мере на одном рынке. Выберите флажок для любого местоположения рынка, где вы хотели бы сделать этот план доступным. Для помощи включены окно поиска и кнопка для выбора стран ,Remitted Tax), в которых корпорация Майкрософт перечисляет налог на продажу и использование от вашего имени.

Если вы уже установили цены на свой план в долларах США (USD) и добавили другое место на рынке, цена на новом рынке будет рассчитываться в соответствии с текущими обменными курсами. Вы всегда должны пересмотреть цену для каждого рынка перед публикацией. Цены могут быть пересмотрены с помощью ссылки "Экспортные цены (xlsx)" после сохранения изменений.

#### <a name="pricing"></a>Цены

- **Модель ценообразования**: Плоская ставка или место на основе

**Плоская ставка:** Включите доступ к вашему предложению с одной ежемесячной или годовой ценой фиксированной цены. Иногда это называют ценами на основе сайтов. С помощью этой модели ценообразования можно дополнительно определить дозированные планы, которые используют API-сервис учета рынка для зарядки клиентов в соответствии с нестандартными единицами.  Для получения дополнительной информации о дозированных биллинга, см [дозированных биллинга с помощью рынка измерительных услуг](./saas-metered-billing.md).

**На пользователя:** Включите доступ к вашему предложению с ценой, основанной на количестве пользователей, имеющих доступ к предложению или занимающих места. Эта пользовательская модель позволяет установить минимальное и максимальное количество пользователей, разрешенных в зависимости от цены. Таким образом, различные ценовые точки могут быть настроены на основе количества пользователей путем настройки нескольких планов.  Эти поля необязательные. Если не выбрать, количество пользователей будет интерпретироваться как не имеющий предела (мин 1 и максимум столько, сколько система может поддерживать). Эти поля могут быть отредактированы как часть обновления плана.

После публикации выбор модели ценообразования выставления счетов не может быть изменен. Кроме того, все планы на одно и то же предложение должны иметь одну и ту же модель ценообразования.

- **Срок выставления счетов**: ежемесячный или ежегодный

Выберите частоту, которую клиенты должны оплатить указанную цену. Необходимо предоставить по крайней мере одну ежемесячную или годовую цену, или оба варианта могут быть доступны клиентам.

- **Цена**: USD в месяц или USD в год

Цены, установленные в местной валюте (USD и Доллар США), конвертируются в местную валюту всех выбранных рынков с использованием текущих обменных курсов, доступных во время установки. Проверить эти цены перед публикацией, экспортируя таблицу ценообразования и обзор цены на каждом рынке. Если вы хотите установить пользовательские цены на индивидуальном рынке, измените и импортируйте таблицу ценообразования. Вы несете ответственность за проверку этих цен и владеете этими настройками.
*\*Сначала необходимо сохранить изменения в ценообразовании, чтобы обеспечить экспорт ценовых данных.*

Внимательно просмотрите цены перед публикацией, так как существуют некоторые ограничения на то, что может измениться после публикации плана:

- После публикации плана модель ценообразования не может быть изменена.
- После публикации платежного термина для плана он не может быть удален позже.
- Как только цена для рынка в вашем плане опубликована, она не может быть изменена позже.

### <a name="plan-audience"></a>Планирование аудитории

У вас есть возможность настроить каждый план, чтобы быть видимым для всех или только для конкретной аудитории по вашему выбору. Вы можете назначить членство в этой ограниченной аудитории с помощью идонов для арендаторов Azure AD.

#### <a name="privacy"></a>Конфиденциальность

- **Это частный план** (Необязательный флажок)

Проверьте это поле, чтобы сделать ваш план закрытым и видимым только для ограниченной аудитории по вашему выбору. После публикации в качестве частного плана, вы можете обновить аудиторию или выбрать, чтобы сделать план доступным для всех. Как только план публикуется как видимый для всех, он должен оставаться видимым для всех. (План не может быть настроен как частный план снова).

- **Ограниченная аудитория (Ит-иутдляна арендаторов)**

Назначаем аудиторию, которая будет иметь доступ к этому частному плану. Доступ назначается с использованием идентификаторов арендатора с возможностью включения описания каждого идентификатора клиента. При импорте файла электронной таблицы .csv можно добавить не более 10 идонов клиентов или 20 000 идовинителей клиентов.

Арендатор представляет собой представление организации, с идентификатором, представленным как GUID (Globally Unique Identifier, 128-битный целый номер, используемый для идентификации ресурсов). Это выделенный экземпляр службы Azure AD, который организация или разработчик приложения получает после начала сотрудничества с корпорацией Майкрософт, то есть после регистрации в Azure, Microsoft Intune или Microsoft 365. Каждый клиент Azure AD отделен от остальных клиентов Azure AD. Чтобы проверить клиента, войдите на портал Azure под учетной записью, которую вы хотите использовать для управления приложением. Если у вас есть клиент, вы автоматически войдете в него и сможете увидеть имя клиента непосредственно под своей учетной записью. Наведите указатель мыши на имя учетной записи в верхней правой части портала Azure, чтобы увидеть свое имя, адрес электронной почты, идентификатор каталога или клиента (GUID), а также домен. Если ваша учетная запись связана с несколькими клиентами, выберите имя учетной записи, чтобы открыть меню, где можно переключаться между клиентами. У каждого клиента есть свой идентификатор. Вы также можете найти идентификатор арендатора вашей организации, используя URL-адрес доменного имени по адресу: [https://www.whatismytenantid.com](https://www.whatismytenantid.com).

В то время как SaaS предлагает использовать идолы для клиентов для определения частной аудитории, другие типы предложений могут использовать иные идя подписки Azure (которые также представлены как GUI).

> [!NOTE]
> Частная аудитория (или ограниченная аудитория) отличается от предварительной аудитории. Во вкладке **[Предварительный просмотр](#preview)** можно определить аудиторию предварительного просмотра. Предварительная аудитория имеет доступ к вашему предложению *до* публикации предложения в прямом эфире на рынке. В то время как обозначение частной аудитории применяется только к определенному плану, аудитория предварительного просмотра может просматривать все планы (частные или нет), но только в течение ограниченного периода предварительного просмотра, пока план тестируется и проверяется.

## <a name="publish"></a>Публикация

После того как вы завершили все необходимые разделы предложения, выберите **опубликовать** в правом верхнем углу портала. Вы будете перенаправлены на **обзор и опубликовать** страницу.

#### <a name="submit-offer-to-preview"></a>Отправить предложение для предварительного просмотра

Если это ваш первый раз публикации этого предложения, вы можете:

- Ознакомьтесь со статусом завершения для каждого раздела предложения.
    - *Не запущен* - означает, что раздел не был затронут и должен быть завершен.
    - *Неполный* - означает, что раздел имеет ошибки, которые должны быть исправлены или требует больше информации, которая будет предоставлена. Вам нужно вернуться в раздел и обновить его.
    - *Полный* - означает, что раздел завершен, все необходимые данные предоставлены и нет никаких ошибок. Все разделы предложения должны быть в полном состоянии, прежде чем вы сможете отправить предложение.
- Предоставьте инструкции по тестированию команде по сертификации, чтобы убедиться, что ваше приложение тестируется правильно, в дополнение к любым дополнительным заметкам, полезным для понимания вашего приложения.
- Отправить предложение для публикации, выбрав **Отправить**. Мы вышлем вам электронное письмо, чтобы сообщить вам, когда предварительный вариант предложения доступен для вас, чтобы рассмотреть и утвердить. Вы должны вернуться в Партнерский центр и выбрать **Go-live** для предложения опубликовать ваше предложение для общественности (или, если частное предложение, для частной аудитории).

Корпорация Майкрософт предоставляет стандартный шаблон контракта для коммерческого рынка.

- **Используете Стандартный контракт для коммерческой платформы Майкрософт?**

Чтобы упростить процесс приобретения для клиентов и уменьшить юридические сложности для разработчиков программного обеспечения, корпорация Майкрософт предлагает Стандартный контракт для коммерческой платформы Майкрософт, который помогает упростить транзакции на платформе. Вместо того чтобы создавать собственные условия, как издатель на коммерческой платформе вы можете предложить свое программное обеспечение на основе Стандартного контракта, который клиентам надо просмотреть и принять лишь один раз. Стандартный контракт можно найти здесь: https://go.microsoft.com/fwlink/?linkid=2041178 .

Вы можете использовать Стандартный контракт вместо предоставления собственных условий, установив флажок "Использовать стандартный контракт для коммерческой платформы корпорации Майкрософт?".

![Установка флажка "Использовать стандартный контракт для коммерческой платформы корпорации Майкрософт?"](./media/use-standard-contract.png)

> [!NOTE]
> После публикации предложения с использованием Стандартного контракта для коммерческой платформы Майкрософт вы уже не сможете использовать собственные условия. Это сценарий "или". Или вы предлагаете свое решение по Стандартному контракту, **или** на собственных условиях. Если вы хотите изменить условия Стандартного контракта, это можно сделать с помощью поправок к Стандартному контракту.

**Поправки к Стандартному контракту**

Поправки к Стандартному контракту позволяют издателю выбрать стандартные условия контракта для простоты и создать условия для своего продукта или бизнеса на его основе. Клиентам необходимо будет изучить только поправки к стандартному контракту, если они уже изучили и приняли Стандартный контракт Майкрософт.

Для издателей на коммерческой платформе доступно два вида поправок:

- Универсальные поправки: применяются к стандартному контракту для всех клиентов. Универсальные поправки демонстрируются каждому клиенту в процессе покупки предложения. Чтобы использовать ваше предложение, клиенты должны принять условия Стандартного контракта и поправки.
- Индивидуальные поправки: это особые поправки к Стандартному контракту, ориентированные на конкретных клиентов по их идентификаторам клиентов Azure. Издатели могут выбирать, к каким клиентам будут применяться такие поправки. В процессе покупки условия с индивидуальными поправками будут предлагаться только клиентам с соответствующими идентификаторами.  Чтобы использовать ваше предложение, клиенты должны принять условия стандартного контракта и поправки (-ок).

>[!NOTE]
> Эти два типа поправок накладываются друг на друга. Клиенты, для которых предназначены индивидуальные поправки, в процессе покупки также сталкиваются с универсальными поправками.

**Условия универсальной переправки в стандартный контракт для коммерческого рынка корпорации Майкрософт**: Введите термины универсальной переправки в этом поле. Вы можете указать одну универсальную поправку для всех предложений. В этом поле можно ввести неограниченное количество символов. Данные условия демонстрируются клиентам в AppSource, Azure Marketplace и на портале Azure во время поиска и в процессе покупки.

**Условия настраиваемых изменений для стандартного контракта для коммерческого рынка корпорации Майкрософт**: Начните с выбора параметра **Добавить пользовательские условия**переправки. Вы можете указать до десяти индивидуальных поправок к условиям для каждого предложения.

- **Условия пользовательской**перенастройки. Введите условия пользовательской переправки в поле условия пользовательской перенастройки. В этом поле можно ввести неограниченное количество символов. В процессе покупки предложения на портале Azure эти индивидуальные поправки к условиям будут предлагаться только клиентам с соответствующими идентификаторами.  
- **Идентификаторы клиентов** (обязательные). Каждая пользовательская поправка может быть нацелена на 20 идентификаторов клиентов. При добавлении индивидуальной поправки необходимо указать хотя бы один идентификатор клиента для ваших клиентов в Azure. Вы можете запросить у клиента этот идентификатор. Чтобы его найти, необходимо перейти по адресу portal.azure.com и выбрать Active Directory Azure > Свойства. Значение идентификатора каталога — это идентификатор клиента (например, 50c464d3-4930-494c-963c-1e951d15360e). Вы также можете узнать идентификатор клиента организации вашего клиента по URL-адресу его доменного имени на странице [Что такое идентификатор клиента в Microsoft Azure и Office 365?](https://www.whatismytenantid.com)
- **Описание** (необязательно). при необходимости введите понятное описание идентификатора клиента, помогающего найти клиента, на который нацелена эта поправка.

**Условия**

Если вы хотите предоставить собственные условия, вы можете ввести их в поле условий. В этом поле можно ввести до 10 000 символов. Если для ваших условий требуется более длинное описание, введите в это поле ссылку с URL-адресом, по которой можно найти условия. Она будет отображаться для клиентов как активная ссылка.

Клиентам будет предложено принять эти условия, прежде чем использовать это предложение.

Чтобы продолжить, выберите **Сохранить черновик**.

Корпорация Майкрософт предоставляет стандартный шаблон контракта для коммерческого рынка.

- **Используете стандартный контракт для коммерческого рынка Майкрософт?**

Чтобы упростить процесс приобретения клиентов и снизить юридическую сложность для поставщиков программного обеспечения, корпорация Майкрософт предлагает стандартный контракт для коммерческого рынка Майкрософт, помогающий упростить транзакции в Marketplace. Вместо того чтобы создавать собственные условия, издатели коммерческих магазинов могут предлагать программное обеспечение в соответствии со стандартным контрактом, который клиенты должны венесуэльскому и принимать только один раз. Стандартный контракт можно найти здесь: https://go.microsoft.com/fwlink/?linkid=2041178.

Вы можете выбрать использование стандартного контракта вместо предоставления собственных условий, выбрав параметр "использовать стандартный контракт для коммерческого рынка".

![Использование стандартного контракта](./media/use-standard-contract.png)

> [!NOTE]
> После публикации предложения с помощью стандартного контракта для коммерческого рынка Майкрософт вы не сможете использовать собственные условия. Это сценарий «или». Вы либо предлагаете свое решение в соответствии со стандартным контрактом, **либо** ваши собственные условия. Если вы хотите изменить условия стандартного контракта, это можно сделать с помощью стандартных поправок контракта.

**Поправки стандартного контракта**

Изменения в стандартном контракте позволяют издателям выбрать стандартные условия контракта для простоты и настроить условия для их продукта или бизнеса. Клиентам необходимо только просмотреть внесенные в контракте поправки, если они уже проверили и приняли Соглашение Microsoft Standard.

Для коммерческих издателей Marketplace доступно два вида поправности:

- Универсальные поправки. Эти действия применяются к стандартному контракту для всех клиентов. Универсальные поправки отображаются для каждого клиента предложения в потоке покупки. Клиенты должны принять условия стандартного контракта и поправки, прежде чем они смогут использовать ваше предложение.
- Пользовательские поправки. Эти действия являются специальными для стандартных контрактов, которые предназначены для конкретных клиентов только через идентификаторы клиентов Azure. Издатели могут выбрать целевой клиент. Только клиенты из клиента будут представлены с условиями пользовательской переправки в потоке покупки предложения.  Клиенты должны принять условия стандартного контракта и поправки, прежде чем они смогут использовать ваше предложение.

>[!NOTE]
> Эти два типа стека поправки поверх других. Клиенты, нацеленные на пользовательскую поправку, также получат универсальную поправку на стандартный контракт во время покупки.

**Условия универсальной переправки в стандартный контракт для коммерческого рынка корпорации Майкрософт**: Введите термины универсальной переправки в этом поле. Вы можете предоставить единую универсальную поправку для каждого предложения. В этом поле можно ввести неограниченное число символов. Эти термины отображаются для клиентов в AppSource, Azure Marketplace и/или портал Azure во время обнаружения и покупки.

**Условия настраиваемых изменений для стандартного контракта для коммерческого рынка корпорации Майкрософт**: Начните с выбора параметра **Добавить пользовательские условия**переправки. Вы можете предоставить до 10 условий пользовательской перенастройки для каждого предложения.

- **Условия пользовательской**перенастройки. Введите условия пользовательской переправки в поле условия пользовательской перенастройки. В этом поле можно ввести неограниченное число символов. Только клиенты из идентификаторов клиента, указанные для этих пользовательских терминов, будут представлены с помощью настраиваемых условий переправки в потоке покупки предложения в портал Azure.  
- **Идентификаторы клиентов** (обязательные). Каждая пользовательская поправка может быть нацелена на 20 идентификаторов клиентов. При добавлении пользовательской поправки необходимо указать по крайней мере один идентификатор клиента. Идентификатор клиента идентифицирует клиента в Azure. Вы можете попросить клиента получить этот идентификатор и найти его, перейдя по адресу portal.azure.com > Azure Active Directory > свойства. Значение идентификатора каталога — идентификатор клиента (например, 50c464d3-4930-494c-963c-1e951d15360e). Вы также можете найти идентификатор клиента организации, используя URL-адрес доменного имени, в [котором находится мой Microsoft Azure и идентификатор клиента Office 365](https://www.whatismytenantid.com).
- **Описание** (необязательно). при необходимости введите понятное описание идентификатора клиента, помогающего найти клиента, на который нацелена эта поправка.

**Условия**

Если вы хотите предоставить собственные пользовательские условия, вы можете ввести их в поле условия. В этом поле можно ввести до 10 000 символов текста. Если для ваших условий требуется более длинное описание, введите ссылку на один URL-адрес в это поле, где можно найти условия. Он будет отображаться для клиентов как активная ссылка.

Клиенты должны принять эти условия, прежде чем они смогут попробовать ваше предложение.

Прежде чем продолжить, выберите **Сохранить черновик** .

# <a name="company-work-accounts-and-partner-center"></a>Рабочие учетные записи организации и центр партнеров

Центр партнеров используется рабочих учетных записей организации, также называется клиентов Azure Active Directory (AD), чтобы контролировать доступ к учетной записи для нескольких пользователей, разрешения на доступ, групп узлов и приложений и обслуживать данные профиля. Связав домен учетной записи электронной почты работы вашей компании к учетной записи центра партнеров, сотрудников вашей компании может зарегистрироваться центр партнеров для управления предложения marketplace, используя свои собственные имена пользователей работы учетной записи и пароли.

## <a name="check-whether-your-company-already-has-a-work-account"></a>Проверить, имеет ли ваша компания уже рабочей учетной записи

Если ваша компания, подписанного облачной службе Майкрософт, например Azure, Microsoft Intune или Office 365, то у вас уже есть работы электронной почты учетной записи домена (также известные как клиент Azure Active Directory "), может использоваться с центром партнеров.

Выполните следующие действия для проверки.
1. Войдите на портал администрирования Azure по адресу https://portal.azure.com.
2. Выберите **Azure Active Directory** меню навигации слева, а затем выберите **имена пользовательских доменов**.
3. Если вас уже есть рабочая учетная запись, отображаются имя домена.

Если вашей организации еще нет рабочей учетной записи, один создается автоматически во время регистрации центра партнеров.

## <a name="set-up-multiple-work-accounts"></a>Настройка нескольких рабочих учетных записей

Перед принятие решения об использовании существующей рабочей учетной записи, определите, сколько пользователей рабочей учетной записи потребуется доступ к центра партнеров. Если пользователи находятся в рабочей учетной записи, которым не нужен доступ к центру партнеров, можно рассмотреть возможность создания нескольких рабочих учетных записей, только пользователям, которым необходимо получить доступ к центру партнеров представлены на определенной учетной записи.

## <a name="create-a-new-work-account"></a>Создание новой рабочей учетной записи

Для создания новой рабочей учетной записи для вашей компании, выполните следующие действия. Может потребоваться запрашивать удаленную помощь у выдается тому, кто имеет права администратора на учетной записи Microsoft Azure.

1. Войдите на [портал Microsoft Azure](https://portal.azure.com).
2. В меню навигации слева выберите **Azure Active Directory** -> **пользователей**.
3. Выберите **нового пользователя** и создать учетную запись Azure работы, введя имя и адрес электронной почты. Убедитесь, что **роль каталога** присваивается **пользователя** и выберите **Показать пароль** флажок внизу, чтобы просмотреть и запомните или запишите автоматически сгенерированный пароль.
4. Выберите **создать** для сохранения нового пользователя.

Адрес электронной почты для учетной записи пользователя должно быть имя проверенного домена в каталоге. Можно вывести список всех проверенных доменов в каталоге, выбрав **Azure Active Directory** -> **имена пользовательских доменов** в меню навигации слева.

Дополнительные сведения о добавлении пользовательских доменов в Azure Active Directory, см. в разделе [Add или связать домен в Azure AD](https://docs.microsoft.com/azure/active-directory/active-directory-add-domain).

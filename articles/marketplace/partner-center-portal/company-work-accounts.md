# <a name="company-work-accounts-and-partner-center"></a>Рабочие счета компании и Партнерский центр

Центр партнеров использует рабочие учетные записи компании, также известные как арендаторы Active Directory (AD), для управления доступом к учетным записям для нескольких пользователей, управления разрешениями, группами и приложениями, а также для поддержания данных профиля. Связав домен рабочей учетной записи вашей компании с учетной записью Partner Center, сотрудники вашей компании могут войти в Центр Партнеров для управления предложениями на рынке, используя свои собственные имена пользователей и пароли рабочих учетных записей.

## <a name="check-whether-your-company-already-has-a-work-account"></a>Проверьте, есть ли у вашей компании уже рабочий счет

Если ваша компания подписалась на облачные службы Майкрософт, такие как Azure, Microsoft Intune или Office 365, то у вас уже есть домен учетной записи рабочей почты (также называемый арендатором Active Directory Azure), который можно использовать в Partner Center.

Выполните следующие действия, чтобы проверить:
1. Войти на портал Admin https://portal.azure.comAzure по адресу .
2. Выберите **Active Directory Azure** из меню левой навигации, а затем выберите **пользовательские доменные имена.**
3. Если у вас уже есть рабочая учетная запись, отобразится имя вашего домена.

Если у вашей компании еще нет рабочего счета, он будет создан для вас в процессе регистрации ВАшего центра партнеров.

## <a name="set-up-multiple-work-accounts"></a>Настройка нескольких рабочих учетных записей

Прежде чем принять решение об использовании существующей рабочей учетной записи, подумайте, сколько пользователей в рабочей учетной записи должны будут получить доступ к Партнерс-центру. Если в рабочей учетной записи есть пользователи, которым не нужно будет получать доступ к Партнерс-центру, вы можете рассмотреть возможность создания нескольких рабочих учетных записей, так что только те пользователи, которым потребуется доступ к Партнерс-центру, будут представлены на определенной учетной записи.

## <a name="create-a-new-work-account"></a>Создание новой учетной записи

Чтобы создать новую учетную запись работы для вашей компании, выполните приведенные ниже действия. Возможно, вам придется запросить помощь у того, у кого есть административные разрешения на учетную запись Microsoft Azure вашей компании.

1. Вопием на [портале Microsoft Azure](https://portal.azure.com).
2. Из меню левой навигации выберите -> **пользователей** **Active Directory Azure.**
3. Выберите **нового пользователя** и создайте новую рабочую учетную запись Azure, введя имя и адрес электронной почты. Убедитесь, что **роль каталога** настроена на **пользователя** и выберите флажок **Show Password** внизу для просмотра и сделать примечание автогенерируемого пароля.
4. Выберите **Создать** для сохранения нового пользователя.

Адрес электронной почты учетной записи пользователя должен быть проверенным доменным именем в вашем каталоге. Вы можете перечислить все проверенные домены в каталоге, выбрав пользовательские**доменные имена** **Azure Active Directory** -> Custom в меню левой навигации.

Чтобы узнать больше о добавлении пользовательских доменов в Active Directory Azure, [см.](https://docs.microsoft.com/azure/active-directory/active-directory-add-domain)

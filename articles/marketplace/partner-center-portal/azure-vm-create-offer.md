# <a name="create-an-azure-virtual-machine-offer"></a>Создайте предложение виртуальной машины Azure

> [!IMPORTANT]
> Мы перемещаем управление вашими предложениями Azure VM с облачного партнерского портала в партнерский центр. До тех пор, пока ваши предложения не будут перенесены, пожалуйста, продолжайте следовать инструкциям в [предложении Create Virtual Machine](https://docs.microsoft.com/azure/marketplace/cloud-partner-portal/virtual-machine/cpp-create-offer) в Облачном партнерской портале для управления вашими предложениями.

В этой статье описывается, как создать и опубликовать предложение виртуальной машины Azure для [Azure Marketplace.](https://azuremarketplace.microsoft.com/) Он предназначен как для Windows, так и для Linux, которые содержат операционную систему, виртуальный жесткий диск (VHD) и до 16 дисков данных.

## <a name="create-an-azure-virtual-machine-offer"></a>Создайте предложение виртуальной машины Azure

Прежде чем создать предложение виртуальной машины Azure, вы должны иметь коммерческий счет в Партнерс-центре. Если вы еще не создали его, [см.](https://docs.microsoft.com/azure/marketplace/partner-center-portal/create-account)

1. Войдите в [Центр Партнеров,](https://partner.microsoft.com/dashboard/home)а затем из верхнего меню выберите **панель мониторинга.**
2. В баре left-nav выберите **Коммерческий рынок**, затем **Обзор**.
3. На странице **Обзор** выберите **новое предложение,** затем **Виртуальная машина Azure.** Появляется диалоговая коробка **Нового предложения.**

![Иллюстрирует страницу Обзор в Партнерс-центре с новой кнопкой предложения и Azure Виртуальная машина предлагает выбран.](media/avm-create7.png)

## <a name="offer-id-and-alias"></a>Идентификатор предложения и псевдоним

Введите **идентификатор предложения**. Это уникальный идентификатор для каждого предложения в вашем аккаунте.

- Этот идентификатор отображается клиентам в веб-адресе для предложения на рынке и в Azure PowerShell и Azure CLI, если это применимо.
- Вы можете использовать только строчные буквы и цифры. Он может включать дефисы и подчеркнутые, но никаких пробелов, и ограничивается 50 символов. Например, если вы введете **тест-предложение-1** здесь, веб-адрес предложения будет `https://azuremarketplace.microsoft.com/marketplace/../test-offer-1`.
- Идентификатор предложения не может быть изменен после выбора **Create.**

Введите **псевдоним Offer**. Это имя используется для предложения в Partner Center. Это имя не используется на рынке и отличается от названия предложения и других значений, показанных клиентам.

Выберите **Создать** для создания предложения и продолжить.

## <a name="offer-setup"></a>Настройка предложения

### <a name="test-drive"></a>Тестовый выпуск

Настройка демонстрации (тест-драйв), которая позволяет клиентам попробовать ваше предложение перед покупкой. Чтобы создать демонстрационную среду, которая позволяет клиентам попробовать ваше предложение в течение определенного периода времени, смотрите [тест-драйв ваше предложение на коммерческом рынке.](https://docs.microsoft.com/azure/marketplace/partner-center-portal/test-drive)

Для включения тест-драйва выберите флажок **для тест-драйва.** Чтобы удалить тест-драйв из вашего предложения, очистите этот флажок.

Дополнительные ресурсы тест-драйва:

- [Технические рекомендации](https://github.com/Azure/AzureTestDrive/wiki/Test-Drive-Best-Practices)
- [Рекомендации по маркетингу](https://aka.ms/TestDriveMarketingBestPractices)
- [Обзор тест-драйвов](https://assetsprod.microsoft.com/mpn/azure-marketplace-appsource-test-drives.pdf) PDF (убедитесь, что ваш всплывающий блокировщик выключен).

### <a name="lead-management"></a>Управление потенциальными клиентами

При публикации вашего предложения на коммерческом рынке с Партнерским центром подключите его к системе управления взаимоотношениями с клиентами (CRM). Это позволяет получать контактную информацию клиента, как только кто-то проявляет интерес или использует ваш продукт. Подключение к CRM необходимо, если вы включите **тест-драйв** (см. предыдущий раздел), в противном случае это необязательно.

1. Выберите назначение интереса, в которое будут отправляться данные о потенциальных клиентах. Партнерский центр поддерживает следующие CRM-системы:
    - [Динамика 365](https://aka.ms/Dyn365LeadMgmt) для привлечения клиентов
    - [Marketo](https://aka.ms/LeadMgmtMarketo)
    - [Salesforce](https://aka.ms/LeadMgmtSalesforce)

    > [!NOTE]
    > Если система CRM не указана выше, используйте [таблицу Azure](https://aka.ms/AzureTableLeadMgmt) или [конечную точку Https](https://aka.ms/LeadMgmtHTTPS) для хранения данных о свинце клиента. Затем экспортируйте данные в crM-систему.

2. Подключите ваше предложение к месту назначения при публикации в Partner Center.
3. Подтвердите, что подключение к месту назначения должно быть настроено правильно. После публикации в Partner Center мы проверяем подключение и отправим вам тестовый свинец. При просмотре предложения до его начала в прямом эфире можно также протестировать соединение с ведущими, пытаясь развернуть предложение самостоятельно в среде предварительного просмотра.
4. Убедитесь, что подключение к месту назначения остается обновленным, чтобы не потерять ни одной зацепки.

Выберите **проект Сохранить** перед продолжением.

## <a name="properties"></a>Свойства

Эта страница позволяет определить категории и отрасли, используемые для группы вашего предложения на рынке, версию приложения и юридические контракты, подтверждающие ваше предложение.

### <a name="category"></a>Категория

Выберите минимум одну и максимум три категории. Эти категории используются для размещения вашего предложения в соответствующих областях поиска рынка. В описании предложения объясните, как ваше предложение поддерживает эти категории. Предложения виртуальных машин отображаются в категории **Compute** в Azure Marketplace.

### <a name="legal"></a>Юридическая информация

Вы должны предоставить условия для предложения. Имеются две возможности.

- Используйте свои собственные условия
- Используйте стандартный контракт для коммерческого рынка Майкрософт

#### <a name="use-your-own-terms-and-conditions"></a>Используйте свои собственные условия

Чтобы предоставить свои собственные пользовательские условия, введите до 10000 символов текста в поле **Условия и условия.** Если вам требуется более подробное описание, введите единый веб-адрес, который указывает на то, где могут быть найдены ваши условия. Он будет отображаться для клиентов в качестве активной ссылки.

Клиенты должны принять эти условия, прежде чем они смогут попробовать ваше предложение.

#### <a name="use-the-standard-contract-for-the-microsoft-commercial-marketplace"></a>Используйте стандартный контракт для коммерческого рынка Майкрософт

Чтобы упростить процесс закупок для клиентов и снизить юридическую сложность для поставщиков программного обеспечения, корпорация Майкрософт предлагает стандартный контракт для коммерческого рынка. Когда вы предлагаете свое программное обеспечение в соответствии со Стандартным контрактом, клиентам нужно только читать и принимать его один раз, и вам не нужно создавать пользовательские условия.

Используйте стандартный контракт, выбрав **стандартный контракт для торгового флажка Microsoft** на рынке, а затем **примите** на всплывающем окне (возможно, вам придется прокрутить, чтобы увидеть его).

![Иллюстрирует страницу Обзор в Партнерс-центре с новой кнопкой предложения и предложение консалтинга выбран.](media/use-standard-contract.png)

> [!NOTE]
> После публикации предложения, используя Стандартный контракт для коммерческого рынка, вы не можете использовать свои собственные пользовательские условия. Либо предложите свое решение по стандартному контракту **или** на ваших собственных условиях.

Чтобы узнать больше о Стандартном контракте, см. [commercial marketplace](https://docs.microsoft.com/azure/marketplace/standard-contract) Вы можете скачать [стандартный контракт](https://go.microsoft.com/fwlink/?linkid=2041178) в качестве PDF (убедитесь, что ваш всплывающий блокировщик выключен).

##### <a name="standard-contract-amendments"></a>Стандартные поправки к контракту

Стандартные поправки к контракту позволяют выбрать условия стандартного контракта для простоты и создать условия для вашего продукта или бизнеса. Клиентам необходимо ознакомиться с поправками к договору только в том случае, если они уже рассмотрели и приняли стандартный контракт Майкрософт. Доступны два вида поправок: универсальный и обычай.

**Универсальные поправки** - Они применяются повсеместно к стандартному контракту для всех клиентов. Они показаны каждому клиенту предложения в потоке покупки. Клиенты должны принять условия Стандартного контракта и поправки(ы), прежде чем они смогут использовать ваше предложение. Вы можете предоставить единую погоду на предложение. Вы можете ввести неограниченное количество символов в этой коробке. Эти термины отображаются клиентам на портале AppSource, Azure Marketplace и/или портале Azure во время потока обнаружения и покупки.

**Пользовательские поправки** - Это специальные поправки к Стандартному контракту, которые предназначены для конкретных клиентов через идоты арендаторов Azure. Вы можете выбрать арендатора, на который вы хотите нацелиться. Только клиенты от арендатора будут представлены пользовательские условия изменения в потоке покупки предложения. Клиенты должны принять условия Стандартного контракта и поправки(ы), прежде чем они смогут использовать ваше предложение.

Начните с выбора **пользовательских условий поправки (Max 10).** Вы можете предоставить до десяти пользовательских условия хитроев за предложение.

- **Пользовательские условия поправки** - Введите свои собственные условия поправки в пользовательских условиях поправки поле. Вы можете ввести неограниченное количество символов. Только клиенты из итогов клиента, указанные для этих пользовательских условий, увидят их в потоке покупки предложения на портале Azure.
- **Идентимативные идентиматы** (требуется) - Каждая пользовательская поправка может быть направлена на до 20 идентипов цевосных подок. При добавлении пользовательской поправки необходимо предоставить хотя бы один идентификатор клиента, который идентифицирует вашего клиента в Azure. Ваш клиент может найти для вас в Azure под, а затем Свойства. Значение идентификатора каталога является идентификатором арендатора (например, 50c464d3-4930-494c-963c-1e951d15360e). Вы также можете найти идентификатор клиента-арендатора организации, используя их веб-адрес доменного имени в [Идентификаторе Microsoft Azure и Идентификаторе клиента Office 365?.](https://www.whatismytenantid.com/)
- **Описание** (необязательно) - Предоставьте дружественное описание идентификатора клиента, которое поможет вам определить клиента, на которого вы ориентируетесь с поправкой.

> [!NOTE]
> Эти два типа поправок стек друг на друга. Клиенты, ориентированные на пользовательские поправки, также получат универсальную поправку к Стандартному контракту во время покупки.

Выберите **проект Сохранить** перед продолжением.

## <a name="offer-listing"></a>Объявление о предложениях

Эта страница позволяет определить детали предложения, такие как имя предложения, описание, ссылки и контакты.

> [!NOTE]
> Содержание предложения (например, описание, документы, скриншоты и условия использования) не обязательно будет на английском языке, пока описание предложения начинается с фразы: «Это приложение доступно только на «неанглийском языке».» Вы также можете предоставить _веб-адрес Useful Link,_ чтобы предложить контент на языке, отличном от того, который используется в содержимом листинга Предложения.

### <a name="marketplace-details"></a>Сведения о рынке

#### <a name="name"></a>Имя

Имя, которое вы вводите здесь, отображается клиентам как название вашего предложения. Это поле предварительно заполнено текстом, который вы ввели в поле **псевдонима Offer** при создании предложения. Это имя можно изменить.

Название:

- Может быть товарным знаком (и вы можете включить товарный знак и символы авторского права)
- Не может быть больше, чем 50 символов в длину
- Не утихает эмодзи.

#### <a name="search-results-summary"></a>Резюме результатов поиска

Краткое описание вашего предложения. Это может быть до 100 символов в длину и используется в результатах поиска на рынке.

#### <a name="long-summary"></a>Длинное резюме

Предоставьте более подробное описание вашего предложения. Это может быть до 256 символов долго и используется в результатах поиска на рынке.

#### <a name="description"></a>Описание

Предоставьте подробное описание вашего предложения, до 3000 символов. Это отображается для клиентов в обзоре коммерческого листинга рынка.

Включите один или несколько из следующих в вашем описании:

- Ценность и ключевые преимущества вашего предложения
- Категории или отраслевые ассоциации, или оба
- Возможности покупки в приложении
- Любые необходимые раскрытия информации

Вот несколько советов для написания описания:

- Четко укажите преимущества, предоставляемые вашим предложением, в начале описания. Включите следующие элементы:
  - Описание вашего предложения.
  - Тип пользователя, который выигрывает от вашего предложения.
  - Потребности или проблемы клиентов, которые адресут предложение.
- Помните, что первые несколько предложений могут отображаться в результатах поиска.
- Не полагайтесь на функции и функциональность, чтобы продать свое предложение. Вместо этого сосредоточьтесь на ценности, которая предоставляется ввашем предложение.
- Используйте отраслевые или благотворные слова.

Чтобы сделать описание предложения более привлекательным, используйте богатый текстовый редактор для формата описания. Богатый текстовый редактор позволяет добавлять цифры, пули, смелые, курсивные и отступы, чтобы сделать ваше описание более читаемым.

![Иллюстрирует страницу Обзор в Партнерс-центре с новой кнопкой предложения и предложение консалтинга выбран.](media/power-bi-rich-text-editor.png)

#### <a name="privacy-policy-link"></a>Ссылка политики конфиденциальности

Введите веб-адрес (URL) в политику конфиденциальности вашей организации. Убедитесь, что ваше предложение соответствует законам и правилам конфиденциальности. Вы также должны разместить действительную политику конфиденциальности на вашем сайте.

### <a name="useful-links"></a>Полезные ссылки

Предоставьте дополнительные онлайн-документы о вашем предложении. Чтобы добавить ссылку, выберите **ссылку,** а затем заполните следующие поля:

- **Имя** - Клиенты увидят имя на странице деталей.
- **Ссылка (URL)** - Введите ссылку для клиентов, чтобы просмотреть ваш онлайн документ.

### <a name="customer-support-links"></a>Ссылки на поддержку клиентов

Предоставьте веб-сайт поддержки, где клиенты могут связаться с вашей группой поддержки.

- Веб-сайт глобальной поддержки Azure
- Веб-сайт поддержки правительства Azure

## <a name="plan-overview"></a>Обзор плана

Вы можете предоставить различные варианты плана в рамках одного и того же предложения в Partner Center. Эти планы ранее назывались СКУ. Предложение требует по крайней мере одного плана, который может отличаться с точки зрения аудитории монетизации, облаков Azure, функций или VM-изображений.

После создания планов вкладка **«Обзор плана»** показывает:

- Имена планов
- Лицензионные модели
- Аудитория (публичная или частная)
- Текущее состояние публикации
- Доступные действия

Действия, доступные в обзоре Плана, варьируются в зависимости от текущего состояния плана. в том числе:

- **Удаление проекта** - Если статус плана является проектом
- **Остановить план продажи** или **синхронизации частной аудитории** - Если статус плана опубликован в прямом эфире

### <a name="create-new-plan"></a>Создание плана

Выберите **и создайте новый план** в верхней части. Появляется диалоговая коробка **Нового плана.**

В окне **идентификатора плана** создайте уникальный идентификатор плана для каждого плана в этом предложении. Этот идентификатор будет виден клиентам в веб-адресе продукта. Используйте только буквы и цифры в нижнем регистре, тире или подчеркивания, и максимум 50 символов.

> [!NOTE]
> Идентификатор плана не может быть изменен после выбора **Create.**

В поле **имени Плана** введите имя для этого плана. Клиенты видят это имя при принятии решения о том, какой план выбрать в рамках вашего предложения. Создайте уникальное имя, которое четко указывает на различия каждого плана. Например, можно использовать **Windows Server** с планами **Pay-as-you-go,** **BYOL,** **Advanced**и **Enterprise.**

Нажмите кнопку **создания**.

#### <a name="reuse-technical-configuration"></a>Повторное использование технической конфигурации

Если у вас есть несколько раз одного типа, и пакеты идентичны между ними, вы можете выбрать **этот план повторно использует техническую конфигурацию из другого плана.** Эта опция позволяет выбрать один из других планов того же типа для этого предложения и повторно использовать его техническую конфигурацию.

> [!NOTE]

**Общественность** - Ваш план может быть просмотрено всеми.

**Частная аудитория** – Сделайте свой план видимым только для заранее выбранной аудитории. После публикации в качестве частного плана можно обновить аудиторию или изменить ее на общедоступный. Предоставьте дисковую версию и SAS URI для виртуальных машинных изображений. Добавьте до 16 дисков данных для каждого изображения VM. Предоставьте только одну новую версию изображения в плане в данной представлении. После публикации изображения вы не можете отобразить его, но вы можете удалить его. Удаление версии не позволит пользователям развернуть новый экземпляр удаленной версии.

- **Дисковая версия** — это версия изображения, который вы предоставляете.
- **SAS URI** — это место в Хранилище Azure, где вы хранили операционную систему VHD.
- Изображения дисков данных также являются URIs VHD SAS, хранящимися в их хранилище Azure.
- Добавьте в план только одно изображение на отправку.

Независимо от того, какую операционную систему вы используете, добавьте только минимальное количество дисков данных, необходимое для решения. Клиенты не могут удалить диски, которые являются частью изображения во время развертывания, но они всегда могут добавлять диски во время или после развертывания.

## <a name="test-drive"></a>Тестовый выпуск

Настройка демонстрации (тест-драйв), которая позволяет клиентам попробовать ваше предложение перед покупкой. Чтобы создать демонстрационную среду, которая позволяет клиентам попробовать ваше предложение в течение определенного периода времени, смотрите [тест-драйв ваше предложение на коммерческом рынке.](https://docs.microsoft.com/azure/marketplace/partner-center-portal/test-drive)

Для включения тест-драйва выберите флажок проверки тест-драйва на вкладке [настройки Offer.](#test-drive) Чтобы удалить тест-драйв из вашего предложения, очистите этот флажок.

Дополнительные ресурсы тест-драйва:

- Технические рекомендации
- Рекомендации по маркетингу
- Обзор тест-драйвов (PDF; убедитесь, что ваш всплывающий блокировщик выключен).

Выберите **проект Сохранить** перед продолжением.

## <a name="review-and-publish"></a>Рецензия и публикация

После того как вы завершили все необходимые разделы предложения, вы можете отправить его на рассмотрение и публикацию.

В правом верхнем углу портала выберите **обзор и опубликуйте.**

На этой странице доступны следующие возможности:

- Ознакомьтесь со статусом завершения для каждого раздела предложения.
  - **Не запущен** - раздел не был запущен и должен быть завершен.
  - **Неполный** - В разделе есть ошибки, которые должны быть исправлены или требует, чтобы вы предоставили больше информации. Смотрите разделы, которые были ранее в этом документе, для получения рекомендаций о том, как обновить этот раздел.
  - **Завершено** - Раздел завершен и нет ошибок. Все разделы предложения должны быть завершены, прежде чем вы сможете отправить предложение.
- **Заметки для сертификации** - Предоставьте инструкции по тестированию команде по сертификации, чтобы убедиться, что ваше приложение протестировано правильно. Предоставьте любые дополнительные заметки, полезные для понимания вашего приложения.

Чтобы отправить предложение для публикации, выберите **Обзор и опубликовать**.

Мы вышлем вам электронное письмо, чтобы сообщить вам, когда предварительный вариант предложения доступен для рассмотрения и утверждения. Чтобы опубликовать свое предложение для общественности (или если частное предложение, для частной аудитории), перейдите в Партнерский центр, найти страницу **Обзор** вашего предложения, и выберите **Go-Live**. Статус вашего предложения будет отображаться в верхней части страницы, когда публикация продолжается.

### <a name="errors-and-review-feedback"></a>Ошибки и отзывы в результате проверки

**Шаг проверки руководства** в процессе публикации представляет собой обширный обзор вашего предложения и связанных с ним технических активов. Если этот процесс выяснит ошибки с вашим предложением, вы получите отчет о сертификации, в который подробно описаны проблемы. Просто внести необходимые коррективы и переиздать свое предложение.

## <a name="offer-overview"></a>Обзор предложения

Страница **обзора предложения** показывает визуальное представление шагов, необходимых для публикации этого предложения (как завершено, так и в процессе) и сколько времени должен предпринять каждый шаг.

Эта страница содержит ссылки для выполнения операций по этому предложению на основе выбора вы делаете. Пример:

- Если предложение является проект - [Удалить проект предложения](https://docs.microsoft.com/azure/marketplace/partner-center-portal/update-existing-offer#delete-a-draft-offer)
- Если предложение живое - [Прекратите продавать предложение](https://docs.microsoft.com/azure/marketplace/partner-center-portal/update-existing-offer#stop-selling-an-offer-or-plan)
- Если предложение находится в предварительном просмотре - [Go-live](https://docs.microsoft.com/azure/marketplace/partner-center-portal/publishing-status#publisher-approval)
- Если вы еще не завершили регистрацию издателя - [Отмена публикации](https://docs.microsoft.com/azure/marketplace/partner-center-portal/update-existing-offer#cancel-publishing)

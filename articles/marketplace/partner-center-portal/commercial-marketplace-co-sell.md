# <a name="co-sell-option-in-partner-center"></a>Вариант совместной продажи в партнерской центре

Предоставление информации о вкладке Co-sell является совершенно необязательным. Он должен достичь статуса Co-sell Ready и IP Co-sell Ready. Предоставляемая вами информация будет использована отделами продаж корпорации Майкрософт для получения дополнительной информации о вашем решении при оценке его пригодности для нужд клиентов. Он не доступен непосредственно для клиентов. Для получения дополнительной информации, [увидеть продать через Microsoft](https://docs.microsoft.com/azure/marketplace/partner-center-portal/create-new-saas-offer#sell-through-microsoft).

Опция совместной продажи в Partner Center в настоящее время доступна для некоторых типов предложений. Поддержка дополнительных типов предложений будет предоставляться с течением времени.

| **Тип предложения** | **Страница совместной продажи доступна в партнерской сети** |
|    -----------------------    |    -----------------------------  |
| Программное обеспечение как услуга (SaaS) |     Общая доступность около февраля 2020      |
| Динамика 365 для взаимодействия с клиентами & PowerApps      |     Уже доступна      |
| Dynamics 365 for Operations     |     Уже доступна       |
| Dynamics 365 Business Central      |  Общая доступность около февраля 2020     |
| Приложение Azure     |     Общая доступность около февраля 2020     |
| Виртуальная машина Azure      |     Будет объявлено позднее       |
| Контейнер Azure     |     Будет объявлено позднее       |
| Консультационная услуга      |     Будет объявлено позднее       |
| Модуль IoT Edge      |     Будет объявлено позднее       |
| Приложение службы Power BI     |     Будет объявлено позднее       |
|  |

## <a name="listing"></a>Вывод списка

Co-sell листинги помогут командам по продажам Майкрософт продать ваше решение более широкой аудитории.

- Выберите платформы Майкрософт, с которыми ваше предложение создано, расширяется или интегрируется с ними.
- Выберите сегменты рынка, на которые ориентировано ваше предложение.
- Выберите [тип решения](#solution-type) для вашего предложения.
- Выберите до трех [областей решения](#solution-areas) и подрайонов.

### <a name="solution-type"></a>Тип решения

Типы решений помогают определить сценарии, которые ваше предложение предназначено для решения.

| **Тип решения**    | **Описание**  |
| :------------------- | :-------------------|
| Устройство (аппаратное обеспечение) | Предложение, которое включает в себя создание или продажу оборудования от производителя устройств. |
| IP (приложение) | Приложения или другие материалы, защищенные авторским правом, лицензированные для использования клиентом. Например: crM-программа, которая может быть лицензирована и установлена на территории. |
| Служба | Практический опыт для конкретного одноразового проекта, часто поставляемый через консультантов. Например: настройка базы данных клиентов (с клиентом, накоторыйная на управление базой данных после доставки). |
| Управляемое обслуживание | Практический опыт для облачного проекта, как правило, на постоянной основе. Например: Предоставление платформы и инструментов для запуска онлайновой базы данных с постоянным управлением, обеспечиваемым управляемым поставщиком услуг. |

### <a name="solution-areas"></a>Области решения

Области решений помогают дополнительно определить ваше решение.

Вы можете добавить до трех областей решения для вашего предложения. Для каждой области решения можно выбрать до двух подобластов решения. Чтобы выбрать несколько подкатегорий, используйте `Ctrl` ключ (на Windows) или ключ команды (на Mac OS).

## <a name="contacts"></a>Контакты

Ваша контактная информация позволяет командам по продажам майкрософт и реселлерам каналов запрашивать дополнительную информацию с соответствующего ресурса в вашей организации. Контактная информация доступна всем отделам продаж Майкрософт. Если вы решите сделать свое предложение доступным в программе CSP, эта контактная информация также доступна для реселлеров канала.

В этом документе содержатся рекомендации по различным ролям для партнеров и как выбирать между ними. Вы можете предоставить свои контакты, загрузив шаблон CSV из Partner Center, заполнив каждую строку информацией о контакте и импортировав этот файл CSV. Импорт файла CSV перезавержает существующие контакты.

Вы также можете экспортировать существующий список контактов предложения, а затем внести изменения в этот файл.

* *Имя (обязательно)*: Имя контакта.
* *Адрес электронной почты (требуется)*: адрес электронной почты контакта.
* *Заголовок вакансий (требуется)*: Заголовок вакансий.
* *Роль (требуется)*: Используйте одну из следующих ролей в этой области в шаблоне.

    | **Role**    | **Описание**  |
    | :------------------- | :-------------------|
    | *:::no-loc text="Partner marketing":::* | Эта роль направлена на маркетинг вашего решения и сотрудничество на маркетинговые усилия с microsoft отделов продаж и канал посредников. Основной контакт для маркетинговых обязательств и предложения листинг амсодержание, такие как описания продукта, изображения и видео. |
    | *:::no-loc text="Partner sales":::* | Эта роль сосредоточена на продаже решения и сотрудничестве по продажам с командами продаж Майкрософт и реселлерами каналов. Для совместного продажи решений укажите по крайней мере один контакт партнера по продажам в регионе, в котором вы хотите быть готовым к продаже. Один и тот же партнер по продажам может охватывать несколько регионов. |
    | *:::no-loc text="Partner technical sales":::* | Поддерживает техническую архитектуру и соображения развертывания в течение цикла продаж, послепродажной интеграции и периодов развертывания. |
    | *:::no-loc text="Partner customer success manager":::* | Обычно поддерживает клиентов после развертывания, чтобы помочь им получить максимальную отдачу от вашего решения и увеличить его присутствие в организации клиента. |

* *Страны/регионы (обязательно):*  При заполнении шаблона используйте [в списке два кода страны/региона.](./commercial-marketplace-co-sell-countries.md) Если контакт охватывает все страны/регионы, используйте три буквенные коды "OOO". Если контакт охватывает более одной страны/региона, введите каждый из двух буквенных кодов, разделенных запятой (например, введите "US, CA, FR" без кавычек в шаблон).

    Страны/регионы должны отражать территорию каждого контакта. Команды продаж майкрософт и реселлеры каналов будут использовать эту информацию для запроса информации или совместной работы по продажам в конкретной стране/регионе.

* *Государства/провинции (необязательно):*  При заполнении шаблона используйте формат XX-XX, указанный в [таблице Штатов/Провинций.](./commercial-marketplace-co-sell-states.md)

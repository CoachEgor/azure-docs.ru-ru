# <a name="saas-fulfillment-apis---faq"></a>Вопросы и ответы об API-интерфейсах исполнения SaaS

В списке указаны требования к интеграции с Azure Marketplace, позволяющие клиентам Azure подписаться на предложение SaaS.

## <a name="discovery-experience"></a>Опыт открытия

После публикации предложения пользователи Azure смогут открыть для себя предложение SaaS в Azure Marketplace. Ваши клиенты смогут фильтровать предложения в зависимости от типа продукта (SaaS) и открывать интересующие их услуги SaaS.

## <a name="purchase-experience"></a>Опыт покупки

Как только пользователь заинтересовался определенным сервисом SaaS, пользователь может подписаться на него в Azure Marketplace.

### <a name="what-does-it-mean-for-an-azure-user-to-subscribe-to-a-saas-offer-in-azure-marketplace"></a>Что означает для пользователя Azure подписаться на предложение SaaS в Azure Marketplace?

Это означает, что пользователь может просматривать условия использования и выписки о конфиденциальности, связанные с службой SaaS, и соглашаться платить за это в соответствии с условиями выставления счетов, установленными вами, издателем предложения SaaS, на счете-фактуре Корпорации Майкрософт. Пользователи могут использовать свой существующий платежный профиль в Azure для оплаты потребления службы SaaS.

Это выгодно по многим причинам. Теперь клиенты могут обнаружить и подписаться в одном месте, используя облачную платформу Майкрософт в качестве надежного источника, без необходимости проверять каждое программное обеспечение ISV, которым она намерена пользоваться. Кроме того, клиенты могут использовать свой существующий платежный профиль без необходимости явно оплачивать каждое программное обеспечение ISV самостоятельно.

### <a name="is-the-user-charged-automatically-when-the-offer-is-subscribed"></a>Заряжается ли пользователь автоматически при подписке?

Подписавшись на предложение SaaS, пользователь согласился заплатить за потребление сервиса SaaS через платформу Microsoft. Однако сборы начнутся только тогда, когда предложение будет использовано. Пользователь должен перейти к вашему предложению SaaS и подтвердить создание учетной записи, чтобы начать потребление предложения. Затем вы уведомите корпорацию Майкрософт о начале выставления счетов за подписку SaaS.

### <a name="how-are-you-notified-when-a-user-subscribes-to-your-saas-offer"></a>Как вы уведомляете, когда пользователь подписывается на ваше предложение SaaS?

Подписавшись на предложение, пользователь Azure может обнаружить и управлять всеми своими предложениями в Azure. По умолчанию состояние недавно подписанного предложения SaaS является **"Обеспечение, выполнение в ожидании"**. В этом состоянии пользователю Azure будет предложено сделать действие **«Настройка учетной записи»,** чтобы просмотреть их опыт управления подпиской SaaS на портале Azure.

Когда пользователь нажмет на **"Настройка счета",** они будут перенаправлены на веб-сайт службы SaaS. URL-адрес, по которому они перемещаются, предоставляется издателем на момент публикации предложения. Эта страница называется целевой страницей издателя. Пользователи Azure должны иметь возможность войти на целевую страницу SaaS на основе существующих учетных данных AAD в Azure.

При перенаправленности пользователя Azure на целевую страницу в URL-адрес запроса добавляется маркер. Этот маркер недолговечен и действителен в течение 24 часов. Затем можно обнаружить наличие этого маркера и вызвать API Корпорации Майкрософт, чтобы получить больше контекста, связанного с маркером.

### <a name="how-do-you-know-the-saas-offer-to-which-the-user-subscribes-in-azure"></a>Как вы знаете предложение SaaS, на которое пользователь подписался в Azure?

Ответ на `Resolve` API включает в себя информацию о предложении и планировании, связанную с подпиской SaaS.

### <a name="how-can-the-azure-user-change-the-plan-associated-with-this-azure-subscription"></a>Как пользователь Azure может изменить план, связанный с этой подпиской Azure?

* Пользователь Azure может изменить план, связанный с подпиской SaaS, непосредственно в опыте SaaS или через платформу Майкрософт.

* Преобразования могут быть сделаны в любое время в цикле выставления счетов. Вы должны признать любое обращение, которое вступит в силу после признания.

* Предоплаченный план **(ежемесячные** или **годовые)** ставки проражаются. Любой излиневенный излиней до момента преобразования будет взиматься в следующем счете-фактуре. Новые перепланировки будут излучаться на основе нового плана.

>[!Note]
>Вы можете заблокировать понижение, если не хотите поддерживать определенные пути преобразования.

### <a name="how-can-the-azure-user-unsubscribe-from-the-plan-associated-with-azure-subscription"></a>Как пользователь Azure может отписаться от плана, связанного с подпиской Azure?

Пользователь Azure может отказаться от купленного предложения SaaS либо непосредственно в опыте SaaS, либо через платформу Майкрософт. После того, как пользователь отпишется, с них больше не будет взиматься плата со следующего цикла выставления счетов.

# <a name="saas-offer-creation-checklist"></a>Контрольный список для создания предложений SaaS

Процесс создания предложения SaaS повлечет за собой несколько страниц. Ниже приведены сведения, которые можно указать на каждой странице, а также ссылки для получения дополнительных сведений о каждом элементе.

Элементы, которые требуется указать или указать, указаны ниже. Некоторые области являются необязательными или имеют значения по умолчанию, которые можно изменить при необходимости. Вам не нужно работать с этими разделами в порядке, указанном здесь.

| **Элемент**    | **Назначение**  |
| :---------- | :-------------------|
| [**Модальное предложение New**](#new-offer-modal) | Собирает сведения об удостоверении предложения.  |
| [Страница «Настройка предложения»](#offer-setup-page) | Позволяет использовать ключевые функции и выбирать способ продажи вашего предложения в корпорации Майкрософт.  |
| [Страница «Свойства»](#properties-page) | Определите категории и отрасли, используемые для группировки предложения в Marketplace, юридические контракты, поддерживающие Ваше предложение, и версию вашего приложения. |
| [Страница списка предложений](#offer-listing-page) | Определите сведения о предложении, которые будут отображаться в Marketplace, включая описания ваших предложений и маркетинговых активов. |
| [Страница предварительного просмотра](#preview-page) | Определите ограниченную аудиторию предварительного просмотра для выпуска предложения перед публикацией предложения в более широкой аудитории Marketplace. |
| [Страница технической настройки предложения](#technical-configuration-page)  | Доступно только при продаже предложения через корпорацию Майкрософт. Укажите технические сведения (URL-путь, веб перехватчик, идентификатор клиента и идентификатор приложения), используемые для подключения к вашему предложению. |
| [**Модальный новый план**](#plan-identity-modal) | Собирает сведения об удостоверении плана.  |
| [Страница «плановый список»](#plan-listing-page)  | Доступно только при продаже предложения через корпорацию Майкрософт. Укажите сведения, используемые для перечисления плана в Marketplace.  |
| [Страница "Планирование цен &"](#plan-pricing--availability-page)  | Доступно только при продаже предложения через корпорацию Майкрософт.  Собирает сведения о бизнес-характеристиках (модели ценообразования), аудитории и доступности рынка для каждого плана (версии) вашего предложения.  |
| [Страница списка тестовых дисков](#test-drive-listing-page)  | Доступно, только если вы выбрали для предложения тестовый выпуск. Укажите сведения, используемые для вывода списка тестового диска в Marketplace.  |
| Страница технической настройки тестового диска  | Доступно, только если вы выбрали для предложения тестовый выпуск. Определите технические подробности для демонстрации (или "тестового диска"), что позволит клиентам опробовать предложение, прежде чем приобретать его.  |
| [Страница "Проверка и публикация"](#review-and-publish-page)  | Выберите изменения, которые необходимо опубликовать, просмотрите состояние каждой страницы и предоставьте сведения о команде сертификации.  |


## <a name="new-offer-modal"></a>Модальное предложение New 

Первая часть информации, которую вы запрашиваете, — это идентификатор и псевдоним для вашего предложения. 

| **Имя поля**    | **Примечания**   |  
| :---------------- | :-----------| 
| Offer ID (Идентификатор предложения)  | Обязательно, невозможно изменить после создания. Максимальная длина 50 символов и должна состоять только из строчных, буквенно-цифровых символов, дефисов или знаков подчеркивания. |
| Псевдоним предложения  | обязательный параметр. |

## <a name="offer-setup-page"></a>Страница «Настройка предложения»

На странице «Настройка предложения» можно выбрать различные каналы и продавать движение, а также объявить использование ключевых функций, таких как тестовый выпуск и интересы клиентов. 

| **Имя поля**    | **Примечания**   | 
| :---------------- | :-----------|  
| Вы хотите продать через корпорацию Майкрософт?  | обязательный параметр. Значение по умолчанию: Да |
| Как вы хотите, чтобы потенциальные клиенты могли взаимодействовать с перечнем предложений? (Вызов действия)  | Требуется, если не продает через Майкрософт. По умолчанию: Бесплатная пробная версия, варианты: "получить сейчас", "Бесплатная пробная версия", "связаться со мной". |
| URL-адрес пробной версии  | Требуется, если выбран вариант "Бесплатная пробная версия", так как клиенты должны взаимодействовать с перечнем предложений. |
| URL-адрес предложения  | Требуется, если выбран параметр "получить сейчас", так как клиенты должны взаимодействовать с перечнем предложения |
| Каналы  | необязательный параметр. По умолчанию: не следует приучастию в канале CSP (Торговый посредник).  |
| Тестовый выпуск | необязательный параметр. По умолчанию: нет включенного тестового диска.  |
| Тип тестового выпуска | Требуется, если включен тестовый диск. По умолчанию: нет выбранных. Варианты: Azure Resource Manager, Dynamics 365 для бизнеса Central, Dynamics 365 для взаимодействия с клиентами, Dynamics 365 для операций, приложение логики, Power BI.  |
| Управление интересами — подключение к системе CRM | Требуется при продаже через корпорацию Майкрософт или при перечислении предложений "связаться со мной". По умолчанию: нет подключенной системы CRM. Параметры CRM: Таблица Azure, большой двоичный объект Azure, Dynamics CRM Online, конечная точка HTTPs, Marketo, Salesforce  |

## <a name="properties-page"></a>Страница свойств

На странице свойств можно определить категории и отрасли, используемые для группировки предложения в Marketplace, юридические контракты, поддерживающие Ваше предложение, и версию вашего приложения. Не забудьте предоставить полные и точные сведения о предложении на этой странице, чтобы они отображались правильно и предложены для правильного набора клиентов. 

| **Имя поля**    | **Примечания**   | 
| :---------------- | :-----------|  
| Категория и Подкатегория | Требуется 1 и Макс. 3. По умолчанию: нет выбранных. |
| Отрасли и подотрасли | необязательный параметр. Максимальное число отраслей L1 и максимальное 2 подотрасли в каждой отрасли L1, по умолчанию — нет выбранных |
| версия приложения.  | необязательный параметр. По умолчанию: нет. |
| Использовать стандартный контракт  | необязательный параметр. По умолчанию: не выбрано.  | |
| Условия использования  | Требуется, если не выбран стандартный контракт.  |

## <a name="offer-listing-page"></a>Страница списка предложений

На странице со списком можно указать текст и изображения, которые видят клиенты при просмотре списка вашего предложения в Marketplace. 

| **Имя поля**    | **Примечания**   |
| :---------------- | :-----------| 
| Имя  | Обязательное, максимальное 50 знаков. |
| Сводка  | Обязательное, максимальное 100 знаков. | 
| Description (Описание)  | Обязательное, максимальное 3000 знаков. |
| Инструкции по началу работы  | Обязательное, максимальное 3000 знаков. |
| Инструкции по началу работы  | Обязательное, максимальное 3000 знаков. |
| Ключевые слова для поиска  | Необязательный, рекомендуемый, Макс. 3 Ключевые слова. |
| URL-адрес политики конфиденциальности  | обязательный параметр. |
| URL-адрес маркетинговых материалов программы CSP  | необязательный параметр. |
| Полезные ссылки заголовок + URL-адрес  | необязательный параметр. |
| Сопроводительные документы Title + File  | Обязательно, min 1 и Max 3. Должен быть формат PDF-файла. |
| Снимки экрана  | Обязательный, минимум 1 снимок экрана и максимум 5; рекомендуется четыре или более. Должно быть 1280 X 720 в формате PNG. |
| Логотипы в магазине (небольшие, средние, крупные, широкие, Hero)  | Малый (48 X 48) и крупный (216 X 216); другие размеры необязательно, но рекомендуется: средний (90 x 90), Wide (255 x 115), Hero (815 x 290). Должен быть в формате PNG. |
| Имя видео + URL + эскиз  | Необязательно, рекомендуется, максимум 4 видео. Эскиз должен иметь размер 1280 x 720 в формате PNG. Видео должно размещаться на YouTube или Vimeo. |
| Контакты (программа CSP, проектирование, поддержка)  | Требуется Контактное лицо по проектированию и поддержке (имя, адрес электронной почты и номер телефона); Связаться с программой CSP необязательно, но рекомендуется. |
| URL-адрес службы поддержки  | обязательный параметр. |

## <a name="preview-page"></a>Страница предварительного просмотра

На странице предварительный просмотр можно указать аудиторию, чтобы получить доступ к предварительной версии предложения, чтобы убедиться, что предложение соответствует всем требованиям, прежде чем оно станет активным. 

| **Имя поля**    | **Примечания**   | 
| :---------------- | :-----------| 
| Электронная почта AAD/MSA + описание | Обязательно, min 1 и Max 10, если они указаны вручную, или до 20 при отправке CSV-файла. |

## <a name="technical-configuration-page"></a>Страница технической настройки 

На странице Technical Configuration (техническая конфигурация) можно указать технические сведения, используемые корпорацией Майкрософт для подключения к вашему предложению. Эта страница не видна вам, если вы решили не продавать в Майкрософт.

| **Имя поля**    | **Примечания**   |  
| :---------------- | :-----------| 
| URL-адрес целевой страницы | Требуется при продаже через Майкрософт. |
| Веб-перехватчик подключения | Требуется при продаже через Майкрософт. |
| Идентификатор клиента Azure AD | Требуется при продаже через Майкрософт. |
| Идентификатор приложения Azure AD | Требуется при продаже через Майкрософт. |

## <a name="plan-identity-modal"></a>Модальное удостоверение плана

Первая часть информации, которую необходимо предоставить, — это имя и идентификатор для плана. Эта страница не видна вам, если вы решили не продавать в корпорации Майкрософт.

| **Имя поля**    | **Примечания**   |  
| :---------------- | :-----------| 
| ИДЕНТИФИКАТОР плана  | Требуется при продаже через Майкрософт. Его нельзя изменить после создания. Максимальная длина 50 символов и должна состоять только из строчных, буквенно-цифровых символов, дефисов или знаков подчеркивания. |
| Имя плана  | Требуется при продаже через Майкрософт. Должно быть уникальным для всех планов в предложении. Максимальная длина 50 символов. |

## <a name="plan-listing-page"></a>Страница «плановый список»

На странице список планов можно указать текст для клиентов, который будет отображаться при просмотре плана в Marketplace. Эта страница не видна вам, если вы решили не продавать в Майкрософт.

| **Имя поля**    | **Примечания**   |  
| :---------------- | :-----------| 
| Описание плана   | Требуется при продаже через Майкрософт. Максимальная длина 500 символов. | |

## <a name="plan-pricing--availability-page"></a>Страница "Планирование цен &"

На странице цены и доступности плана можно определить бизнес-характеристики, аудиторию и доступность рынка для каждого плана (версии) вашего предложения. Эта страница не видна вам, если вы решили не продавать в Майкрософт.

| **Имя поля**    | **Примечания**   | 
| :---------------- | :-----------| 
| Доступность рынка  | Required, min 1 и Max 141. |
| Модель ценообразования  | обязательный параметр. По умолчанию: фиксированная скорость. Параметры: фиксированная ставка на пользователя. |
| Минимальное и максимальное число рабочих мест  | Необязательно, доступно только в том случае, если выбрана модель ценообразования на основе места. |
| Срок выставления счетов  | обязательный параметр. По умолчанию: ежемесячно. Параметры: ежемесячно, ежегодно. |
| Цена  | Обязательный USD в месяц, если выбран месячный срок выставления счетов; или USD в год, если выбран ежегодный срок выставления счетов. |
| Аудитория плана  | необязательный параметр. По умолчанию: открытый план. Параметры: общедоступный, частный по ИДЕНТИФИКАТОРу клиента |
| Аудитория ограниченного плана (идентификатор клиента + описание)  | Требуется, если выбран частный план. Минимальный 1 и Макс. 10 идентификаторов клиентов, если они указаны вручную. Макс. 20000 при импорте CSV-файла. |

## <a name="test-drive-listing-page"></a>Страница списка тестовых дисков

Доступно, только если вы выбрали для предложения тестовый выпуск. Укажите сведения, используемые для вывода списка тестового диска в Marketplace.

| **Имя поля**    | **Примечания**   | 
| :---------------- | :-----------| 
| Description (Описание)  | обязательный параметр. |
| Имя пользователя и файл вручную  | Обязательно. максимальный размер документа: 1. формат должен быть в формате PDF. |
| Имя видео, URL-адрес + эскиз  | Необязательно, рекомендуется. Эскиз должен иметь размер 533 x 324 в формате ЖПГП или PNG. Видео должно размещаться на YouTube или Vimeo. |

# <a name="offer-creation-checklist"></a>Контрольный список для создания предложений

Процесс создания предложения вы перейдете через несколько страниц. Ниже приведены сведения, которые можно указать на каждой странице, со ссылками на дополнительные сведения о каждом элементе.

Ниже указаны элементы, которые вы должны указать или задать. Некоторые разделы являются необязательными или имеют значения по умолчанию указано, что можно изменить при необходимости. Не нужно работать в этих разделах в указанном порядке.

| **Элемент**    | **Назначение**  |
| :---------- | :-------------------|
| [**Новые предложения модальное окно**](#new-offer-modal) | Сбор данных о количестве предоставляют сведения об удостоверении.  |
| [Страница установки предложения](#offer-setup-page) | Можно согласиться использовать основные функции и Выбор способа продавать предложения через Microsoft.  |
| [Страница «Свойства»](#properties-page) | Определите категории и отраслей, используемое для группировки предложение на рынки, контрактов, поддержкой вашего предложения и версии приложения. |
| [Листинг страница предложения](#offer-listing-page) | Определите сведения о предложении для отображения на рынке, включая описание вашего предложения и маркетинговым актива. |
| [Предварительный просмотр страницы](#preview-page) | Определение ограниченного круга пользователей предварительной версии за освобождение вашего предложения до публикации вашего предложения динамической широкой аудитория marketplace. |
| [Страница технической конфигурации предложения](#technical-configuration-page)  | Доступно только при выборе продавать предложения через Microsoft. Определите технические сведения (URL-адрес веб-перехватчика, идентификатор клиента и идентификатор приложения), используемый для подключения к вашему предложению. |
| [**Новый план-Modal**](#plan-identity-modal) | Сбор данных о количестве планирование сведения об удостоверении.  |
| [Планирование страницу с описанием](#plan-listing-page)  | Доступно только при выборе продавать предложения через Microsoft. Укажите сведения о плане в marketplace создает список.  |
| [Ценовая категория плана & странице доступности](#plan-pricing--availability-page)  | Доступно только при выборе продавать предложения через Microsoft.  Собирает характеристики бизнеса (модель ценообразования), аудитории и на рынке доступности для каждого плана (версия) вашего предложения.  |
| [Список дисков тестовой страницы](#test-drive-listing-page)  | Доступно только при выборе предложить тестовый выпуск вашего предложения. Определение сведений о, используемый к списку, протестируйте в marketplace.  |
| Тестовая страница технической конфигурации диска  | Доступно только при выборе предложить тестовый выпуск вашего предложения. Определите технические сведения демонстрации (или «тестовый выпуск»), которые клиенты смогут попробовать вашего предложения до фиксации для его приобретения.  |
| [Просмотрите и страница публикации.](#review-and-publish-page)  | Выберите нужные для публикации, см. в разделе состояние каждой страницы и укажите сведения команде сертификации для изменения.  |


## <a name="new-offer-modal"></a>Новое предложение модального 

Первый, вам будет предложено предоставить являются имя и идентификатор для вашего предложения. 

| **Имя поля**    | **Примечания**   |  
| :---------------- | :-----------| 
| Offer ID (Идентификатор предложения)  | Требуется, не может быть изменено после создания. Более 50 символов и может включать только строчные буквы, буквенно-цифровые символы, дефисы и символы подчеркивания. |
| Название предложения  | Обязательный элемент. |

## <a name="offer-setup-page"></a>Страница установки предложения

На этой странице программы установки —, где вы может согласиться на различные каналы и Продажа движения, а также объявить потенциальных клиентов использование основных возможностей, например тестовый выпуск и клиента. 

| **Имя поля**    | **Примечания**   | 
| :---------------- | :-----------|  
| Вы действительно хотите продавать через Microsoft?  | Обязательный элемент. Значение по умолчанию: Да |
| Как вы бы хотели потенциальных клиентов для взаимодействия с предложением в список? (Призыв к действию)  | Обязательно, если не продаж по Майкрософт. Значение по умолчанию: Бесплатная пробная версия, параметры: «Компания», «Бесплатная пробная версия», «Свяжитесь со мной.» |
| URL-адрес пробной версии  | Требуется, если выбран «Бесплатная пробная версия», как должна взаимодействовать как со списком предложения. |
| URL-адрес предложения  | Требуется, если «Получить сейчас» выбран, как должна взаимодействовать как со списком предложения |
| Каналы  | Необязательный элемент. Значение по умолчанию: Не участвует в канал CSP (торгового посредника).  |
| Тестовый выпуск | Необязательный элемент. Значение по умолчанию: Тестовый выпуск не включена.  |
| Тип тестового выпуска | Обязательно, если включена тестовый выпуск. Значение по умолчанию: Не выбрано. Параметры: Диспетчер ресурсов Azure, Dynamics 365 для центральной части бизнеса, Dynamics 365 для взаимодействия с клиентами, Dynamics 365 для операций, приложение логики, Power BI.  |
| Управление интересами — подключение к системе CRM | Требуется, если продажа через Microsoft или список предлагает как «Свяжитесь со мной.» Значение по умолчанию: нет подключенных системы CRM. Параметры CRM: Таблица Azure, BLOB-объектов Azure, Dynamics CRM online, конечная точка HTTPs, Marketo, Salesforce  |

## <a name="properties-page"></a>Страница свойств

На странице свойств является определение категорий и отраслей, используемое для группировки предложение на рынки, контрактов, поддержкой вашего предложения и версии приложения. Не забудьте предоставить полные и точные сведения о вашем предложении на этой странице, так что он правильное отображение и предлагается правильный набор клиентов. 

| **Имя поля**    | **Примечания**   | 
| :---------------- | :-----------|  
| Категории и подкатегории | Требуется 3 1 до максимального размера. Значение по умолчанию: Не выбрано. |
| Сфер деятельности и subindustries | Необязательный элемент. Макс. 2 L1 отраслей и Макс. 2 subindustries в каждой отрасли L1, по умолчанию: Ничего не выбрано |
| версия приложения.  | Необязательный элемент. Значение по умолчанию: Отсутствует. |
| Использовать стандартный контракт  | Необязательный элемент. Значение по умолчанию — не выбран.  | |
| Условия использования  | Требуется, если не выбран стандартный контракт.  |

## <a name="offer-listing-page"></a>Страницу

На странице списка, где дается текста и изображений, которые клиенты будут видеть при просмотре списка вашего предложения в marketplace. 

| **Имя поля**    | **Примечания**   |
| :---------------- | :-----------| 
| ИМЯ  | Требуется, максимальная 50 знаков. |
| Сводка  | Требуется, максимальная 100 знаков. | 
| Описание  | Требуется, максимальная 3000 знаков. |
| Инструкции по началу работы  | Требуется, максимальная 3000 знаков. |
| Инструкции по началу работы  | Требуется, максимальная 3000 знаков. |
| Ключевые слова для поиска  | Не обязательно, рекомендуется, max три ключевые слова. |
| URL-адрес политики конфиденциальности  | Обязательный элемент. |
| URL-адрес материалы маркетинговые программы CSP  | Необязательный элемент. |
| Полезные ссылки заголовка + URL-адрес  | Необязательный элемент. |
| Вспомогательные документы Title + файл  | Требуется минимум 1 и Макс. 3. Должен быть в формате PDF. |
| Снимки экрана  | Требуется снимок экрана min 1 и max 5; четыре или больше. Должен быть в формате PNG 1280 X 720. |
| Store логотипы (малый, средний, крупный, Wide, Hero)  | Малый (48 X 48) и большой (216 X 216) обязательным компонентом; другие размеры, необязательно, но рекомендуется: Средняя (90 x 90), Hero (815 x 290) "или" Wide (255 x 115). Должен быть в формате PNG. |
| Имя видео + URL-адрес эскиза  | Не обязательно, рекомендуется, max видео 4. Эскиз должен быть равен 1280 x 720, в формате PNG. Должна быть размещена видео в YouTube или Vimeo. |
| Контакты (программа CSP, отдел проектирования, поддержки)  | Проектирование и контактные данные для поддержки необходимых (имя, адрес электронной почты и номер телефона); Обратитесь к программе CSP необязательно, но рекомендуется. |
| URL-адрес службы поддержки  | Обязательный элемент. |

## <a name="preview-page"></a>Предварительный просмотр страницы

На странице предварительного просмотра здесь указывается audience, чтобы иметь доступ к предварительной версии предложения, чтобы убедиться, что предложение соответствует всем требованиям, перед ее запуском. 

| **Имя поля**    | **Примечания**   | 
| :---------------- | :-----------| 
| Адрес электронной почты AAD/MSA и описание | Требуется min 1 и max 10, если введенные вручную, или до 20, если отправка CSV-файл. |

| **Имя поля**    | **Примечания**   |  
| :---------------- | :-----------| 
| URL-адрес целевой страницы | Обязательно, если продажа через Microsoft. |
| Подключение веб-перехватчика | Обязательно, если продажа через Microsoft. |
| Идентификатор клиента Azure AD | Обязательно, если продажа через Microsoft. |
| Идентификатор приложения Azure AD | Обязательно, если продажа через Microsoft. |

## <a name="plan-identity-modal"></a>Планирование modal удостоверений

Первый, вам будет предложено указать являются имя и идентификатор для плана. Эта страница не отображается для вас, если вы решили не продаются через Microsoft.

| **Имя поля**    | **Примечания**   |  
| :---------------- | :-----------| 
| Идентификатор плана  | Обязательно, если продажа через Microsoft. Его нельзя изменить после создания. Более 50 символов и может включать только строчные буквы, буквенно-цифровые символы, дефисы и символы подчеркивания. |
| Имя плана  | Обязательно, если продажа через Microsoft. Должно быть уникальным для всех планов предложения. Максимум 50 символов. |

## <a name="plan-listing-page"></a>Странице сведений плана

План, страницы —, которое необходимо ввести текст для клиентов, для просмотра плана в marketplace. Эта страница не отображается для вас, если вы решили не продаются через Microsoft.

| **Имя поля**    | **Примечания**   |  
| :---------------- | :-----------| 
| Описание плана   | Обязательно, если продажа через Microsoft. Максимум 500 символов. | |

## <a name="plan-pricing--availability-page"></a>Страница цены и доступность плана

План цены и доступность страницы является определение характеристик бизнеса, аудитории и доступности рынка для каждого плана (версия) вашего предложения. Эта страница не отображается для вас, если вы решили не продаются через Microsoft.

| **Имя поля**    | **Примечания**   | 
| :---------------- | :-----------| 
| Рынок доступности  | Требуется min 1 и max 141. |
| Модель ценообразования  | Обязательный элемент. Значение по умолчанию: По фиксированной ставке. Параметры: Фиксированной ставке на пользователя. |
| Минимальное и максимальное рабочих мест  | Необязательно, доступна, только если выбрана модель ценообразования на основе рабочих мест. |
| Термин выставления счетов  | Обязательный элемент. Значение по умолчанию: Ежемесячно. Параметры: Ежемесячно, ежегодно. |
| Цена  | Требуется долл. США в месяц, если сумма ежемесячных счетов термин выбран; или долл. США в год, если годовой выставления счетов выбранного термина. |
| Планирование аудитории  | Необязательный элемент. Значение по умолчанию: Общедоступный план. Параметры: Общедоступные, частные по Идентификатору клиента |
| Ограниченным доступом аудитории план (идентификатор клиента + описание)  | Требуется, если выбрана частный план. Min 1 и max 10 идентификаторов клиентов, если ввести вручную. Max 20000, если CSV-файл импорта. |

## <a name="test-drive-listing-page"></a>Список дисков тестовой страницы

Доступно только при выборе предложить тестовый выпуск вашего предложения. Определение сведений о, используемый к списку, протестируйте в marketplace.

| **Имя поля**    | **Примечания**   | 
| :---------------- | :-----------| 
| Описание  | Обязательный элемент. |
| Вручную имя пользователя и файл  | Требуется, максимальная 1 документа. Должны быть в формате PDF. |
| Название видео, URL-адрес + эскиза  | Необязательно, рекомендуется. Эскиз должен быть 533 × 324 в формате JPGP или PNG. Должна быть размещена видео в YouTube или Vimeo. |

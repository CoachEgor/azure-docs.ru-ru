# <a name="saas-offer-creation-checklist"></a>Контрольный список для создания предложений SaaS

Процесс создания предложения SaaS перейдя через несколько страниц. Вот подробности, которые вы можете предоставить на каждой странице, со ссылками, чтобы узнать больше о каждом пункте.

Элементы, которые вы должны предоставить или указать, приведены ниже. Некоторые области являются необязательными или имеют значения по умолчанию при условии, что вы можете изменить по желанию. Вы не должны работать над этими разделами в порядке, указанном здесь.

| **Item**    | **Назначение**  |
| :---------- | :-------------------|
| [**Новое предложение Модаль**](#new-offer-modal) | Сборы предлагают идентификационную информацию.  |
| [Страница настройки предложения](#offer-setup-page) | Позволяет использовать ключевые функции и выбирать, как продавать свое предложение через корпорацию Майкрософт.  |
| [Страница свойств](#properties-page) | Определите категории и отрасли, используемые для группирования вашего предложения на рынках, юридические контракты, поддерживающие ваше предложение, и версию приложения. |
| [Страница листинга предложения](#offer-listing-page) | Определите детали предложения, которые будут отображаться на рынке, включая описания вашего предложения и маркетинговых активов. |
| [Страница предварительного просмотра](#preview-page) | Определите ограниченную аудиторию предварительного просмотра для выпуска вашего предложения до публикации вашего предложения жить для более широкой аудитории рынка (ы). |
| [Предложить страницу технической конфигурации](#technical-configuration-page)  | Доступно только в том случае, если вы решили продать предложение через корпорацию Майкрософт. Определите технические детали (путь URL, веб-крюк, идентификатор клиента и идентификатор приложения), используемые для подключения к вашему предложению. |
| [**Новый план Модаль**](#plan-identity-modal) | Собирает информацию о компании,план итог.  |
| [Страница «План листинг»](#plan-listing-page)  | Доступно только в том случае, если вы решили продать предложение через корпорацию Майкрософт. Определите детали, используемые для списка Плана на рынке.  |
| [Страница планирования ценообразования & доступности](#plan-pricing--availability-page)  | Доступно только в том случае, если вы решили продать предложение через корпорацию Майкрософт.  Собирает бизнес-характеристики (модель ценообразования), аудиторию и доступность рынка для каждого плана (версии) вашего предложения.  |
| [Страница листинга тест-драйва](#test-drive-listing-page)  | Только если вы решили предложить тест-драйв для вашего предложения. Определите детали, используемые для списка тест-драйва на рынке.  |
| Страница технической конфигурации тест-драйва  | Только если вы решили предложить тест-драйв для вашего предложения. Определите технические детали для демонстрации (или "тест-драйв"), которые позволят клиентам попробовать ваше предложение, прежде чем совершить его покупку.  |
| [Обзор и публикация страницы](#review-and-publish-page)  | Выберите изменения, которые вы хотите опубликовать, просмотрите состояние каждой страницы и предоставьте заметки группе по сертификации.  |


## <a name="new-offer-modal"></a>Новое предложение модальный 

Первые части информации, которые вам будет предложено предоставить являются ID и псевдоним для вашего предложения. 

| **Имя поля**    | **Примечания**   |  
| :---------------- | :-----------| 
| Offer ID (Идентификатор предложения)  | Требуется, не может быть изменен после создания. Макс 50 символов и должен состоять только из нижнего регистра, алфавитных символов, тире, или подчеркивает. |
| Предложение псевдоним  | Обязательный элемент. |

## <a name="offer-setup-page"></a>Страница настройки предложения

Страница настройки предложения, где вы можете выбрать в различных каналах и продажи движений, а также объявить об использовании ключевых функций, таких как тест-драйв и клиентов приводит. 

| **Имя поля**    | **Примечания**   | 
| :---------------- | :-----------|  
| Хотите продавать через Microsoft?  | Обязательный элемент. Значение по умолчанию: "Да" |
| Как вы хотите, чтобы потенциальные клиенты взаимодействовали с объявлением о предложении? (Призыв к действию)  | Требуется, если не продавать через Microsoft. По умолчанию: Бесплатная пробная версия, опции: "Получить его сейчас", "Бесплатная пробная версия", "Свяжитесь со мной". |
| URL-адрес пробной версии  | Требуется, если "Бесплатная пробная версия" выбрана, как способ клиенты должны взаимодействовать с предложением листинг. |
| URL-адрес предложения  | Требуется, если "Получить это сейчас" выбран, как способ клиенты должны взаимодействовать с предложением листинг |
| Каналы  | Необязательный параметр. По умолчанию: Не выбран в cSP (реселлер) канал.  |
| Тестовый выпуск | Необязательный параметр. По умолчанию: тест-драйв не включен.  |
| Тип тестового выпуска | Требуется при включении тест-драйва. По умолчанию: Не выбран. Варианты: менеджер ресурсов Azure, Dynamics 365 для Business Central, Dynamics 365 для привлечения клиентов, Dynamics 365 для операций, Logic app, Power BI.  |
| Руководство - подключение к crM-системе | Требуется, если продажа через Microsoft, или если листинг предлагает как "Свяжитесь со мной". По умолчанию: crM-система не подключена. Параметры CRM: таблица Azure, Azure blob, Dynamics CRM онлайн, конечная точка HTTPs, Marketo, Salesforce  |

## <a name="properties-page"></a>Страница свойств

Страница свойств — это место, где определяются категории и отрасли, используемые для группирования предложения на рынках, юридические контракты, поддерживающие ваше предложение, и версия приложения. Не забудьте предоставить полную и точную информацию о вашем предложении на этой странице, так что он отображается надлежащим образом и предложил правильный набор клиентов. 

| **Имя поля**    | **Примечания**   | 
| :---------------- | :-----------|  
| Категория и подкатегория | Требуется 1 и максимум 3. По умолчанию: Не выбран. |
| Отрасли и суботрасли | Необязательный параметр. max 2 L1 Industries и max 2 субиндустрии в каждой отрасли L1, по умолчанию: Нет выбранного |
| версия приложения.  | Необязательный параметр. По умолчанию: Нет. |
| Использование стандартного контракта  | Необязательный параметр. По умолчанию: не выбран.  | |
| Условия использования  | Требуется, если стандартный контракт не выбран.  |

## <a name="offer-listing-page"></a>Страница объявления предложения

Страница листинга, где вы предоставляете текст и изображения, которые клиенты видят при просмотре вашего предложения листинг на рынке. 

| **Имя поля**    | **Примечания**   |
| :---------------- | :-----------| 
| name  | Требуется, максимум 50 chars. |
| Сводка  | Требуется, максимум 100 chars. | 
| Описание  | Требуется, максимум 3000 chars. |
| Инструкции по началу работы  | Требуется, максимум 3000 chars. |
| Инструкции по началу работы  | Требуется, максимум 3000 chars. |
| Ключевые слова для поиска  | Необязательно, рекомендуется, максимум 3 ключевых слова. |
| URL-адрес с политикой конфиденциальности  | Обязательный элемент. |
| CSP Программа Маркетинговые материалы URL  | Необязательный параметр. |
| Полезные ссылки Название и URL  | Необязательный параметр. |
| Название вспомогательных документов и файл  | Требуется, мин 1 и максимум 3. Должно быть формат файла PDF. |
| Снимки экрана  | Требуется, мин 1 скриншот и максимум 5; четыре или более рекомендуется. Должно быть 1280 X 720 в формате PNG. |
| Логотипы магазинов (малые, средние, большие, широкие, герой)  | Малый (48 X 48) и большой (216 X 216) требуется; другие размеры необязательные, но рекомендуемые: Средний (90 x 90), Широкий (255 x 115), Герой (815 x 290). Должно быть в формате PNG. |
| Название видео - URL-адрес и эскиз  | Необязательно, рекомендуется, максимум 4 видео. Thumbnail должен быть 1280 x 720 в формате PNG. Видео должно быть размещено на YouTube или Vimeo. |
| Контакты (Программа CSP, инженерия, поддержка)  | Требуется инженерный и вспомогательный контакт (имя, электронная почта и номер телефона); CSP Программа контакт необязательный, но рекомендуется. |
| URL-адрес службы поддержки  | Обязательный элемент. |

## <a name="preview-page"></a>Страница "Предварительный просмотр"

Страница предварительного просмотра, где вы указать аудиторию, чтобы иметь доступ к вашему предложению предварительного просмотра, чтобы убедиться, что предложение отвечает всем вашим требованиям, прежде чем он идет жить. 

| **Имя поля**    | **Примечания**   | 
| :---------------- | :-----------| 
| AAD/MSA электронная почта - описание | Требуется, мин 1 и максимум 10, если введен вручную, или до 20 при загрузке файла CSV. |

| **Имя поля**    | **Примечания**   |  
| :---------------- | :-----------| 
| URL-адрес целевой страницы | Требуется при продаже через корпорацию Майкрософт. |
| Подключение веб-крюк | Требуется при продаже через корпорацию Майкрософт. |
| Идентификатор клиента Azure AD | Требуется при продаже через корпорацию Майкрософт. |
| Идентификатор приложения Azure AD | Требуется при продаже через корпорацию Майкрософт. |

## <a name="plan-identity-modal"></a>План идентификационный модал

Первыми сведениями, которые вас просят предоставить, являются имя и идентификатор для вашего плана. Эта страница не видна вам, если вы решили не продавать через Microsoft.

| **Имя поля**    | **Примечания**   |  
| :---------------- | :-----------| 
| Идентификатор плана  | Требуется при продаже через корпорацию Майкрософт. Она не может быть изменена после создания. Макс 50 символов и должен состоять только из нижнего регистра, алфавитных символов, тире, или подчеркивает. |
| Имя плана  | Требуется при продаже через корпорацию Майкрософт. Должно быть уникальным во всех планах в предложении. Макс 50 символов. |

Страница списка плана, где вы предоставляете текст для клиентов, чтобы увидеть при просмотре плана на рынке. Эта страница не видна вам, если вы решили не продавать через Microsoft.

| **Имя поля**    | **Примечания**   |  
| :---------------- | :-----------| 
| Описание плана   | Требуется при продаже через корпорацию Майкрософт. Макс 500 символов. | |

## <a name="plan-pricing--availability-page"></a>Страница планирования ценообразования & доступности

Страница ценообразования и доступности плана — это место, где определяются бизнес-характеристики, аудитория и доступность рынка для каждого плана (версии) вашего предложения. Эта страница не видна вам, если вы решили не продавать через Microsoft.

| **Имя поля**    | **Примечания**   | 
| :---------------- | :-----------| 
| Доступность рынка  | Требуется, мин 1 и максимум 141. |
| Модель ценообразования  | Обязательный элемент. По умолчанию: Плоская ставка. Варианты: Плоская ставка, на пользователя. |
| Минимальные и максимальные места  | Необязательная, доступна только в том случае, если выбрана модель ценообразования на основе сидений. |
| Срок выставления счетов  | Обязательный элемент. По умолчанию: ежемесячно. Варианты: Ежемесячный, Ежегодный. |
| Price  | Требуемый USD в месяц, если выбран месячный срок выставления счетов; или USD в год, если выбран годовой срок выставления счетов. |
| Планирование аудитории  | Необязательный параметр. По умолчанию: Общественный план. Варианты: Публичный, Частный по идентификатору арендатора |
| Аудитория с ограниченным планом (идентификатор арендатора и описание)  | Требуется, если выбран частный план. Мин 1 и максимум 10 идентификаций арендатора, если введены вручную. Макс 20000 если импорт архива CSV. |

# <a name="analytics-for-the-commercial-marketplace-in-partner-center"></a>Аналитика для коммерческих решений партнеров в центре партнеров

Узнайте, как получить доступ к аналитическим отчетам в центре партнеров Майкрософт, чтобы отслеживать продажи, оценки производительности и оптимизировать ваши предложения в marketplace. Как партнер можно отслеживать вашей с предложениями с помощью визуализации данных и представление диаграмм, поддерживаемых центра партнеров и узнайте, как максимально увеличить объем продаж. Analytics улучшенные средства позволяют реагировать на результаты производительности и поддерживать Улучшайте отношения с клиентами и торговых посредников. 

Для доступа к средствам аналитики центра партнеров откройте **[анализ](https://partner.microsoft.com/dashboard/commercial-marketplace/analytics/summary)** панели мониторинга в разделе коммерческих Marketplace.

|**Панель мониторинга**|**Отображаемое содержимое**|
|:---|:---|
|[Сводка](#summary-dashboard)|Графы, тенденции и значения статистических данных, используемых для объединения активность для ваших предложений в marketplace|
|[Заказы](#orders-dashboard)|Сведения о заказах в виде графического интерфейса и загрузки|
|[Клиенты](#customer-dashboard)|Сведения о клиентах, включая роста трендов, представлены в формате графического интерфейса и загрузки|
|[Загрузки](#downloads-dashboard)|Список запросов загрузки за последние 30 дней|

**Сводка** панель мониторинга представляет обзор на основе каждого типа предложения. **Insights** представляют критически важной информации с одного взгляда и выводят широкое представление продаж действия ваших предложений. Вы можете визуализировать эти отчеты с помощью **Сводка** панели мониторинга:

- [Общее число заказов](#totals)
- [Всего клиентов](#totals)
- [Географическое расположение клиентов](#customers-by-geography)

- [Количество заказов, организованные по название предложения](#offers-by-orders)

### <a name="date-range"></a>диапазон дат;

Выбор диапазона дат можно найти в правом верхнем углу каждой страницы. Выходные данные **Сводка** странице диаграмм можно настроить, выбрав диапазон дат, в зависимости от ранее 3, 6 или 12 месяцев, или выбрав диапазон дат с Максимальная продолжительность 12 месяцев. Диапазон дат по умолчанию составляет шесть месяцев.

### <a name="totals"></a>Итоги

**Итоги** разделе отображается количество всех заказов, созданных или клиенты, которые получены в течение выбранного диапазона дат. 

- Процентное значение в поле **всего заказов** и **общее клиентов** представляет объем рост по сравнению с предыдущего месяца. 
- Зеленый треугольник, указывающий вверх указывает положительное экспоненциальной. Красный треугольник, указывающий вниз указывает отрицательное экспоненциальной относительно предыдущего месяца. 
- Порядок и тенденций роста клиента представлены гистограмм и будет отображаться значение для каждого месяца, наведя указатель столбцов диаграммы.

- Можно найти и выбрать страну в сетке, чтобы увеличить место на карте. Вернуться к исходному виду, нажав клавишу **Главная** кнопки на карте.
- Объект **новый** клиент приобрел один из ваших предложений в первый раз в месяц в рамках выбранного диапазона дат.

Можно далее проанализировать эти тенденции, выбрав ссылки под диаграммой, которые перейдите на соответствующие **порядок** или **клиента** страниц.

Пользователи с **участник** роли не сможете просматривать эти сведения. [Дополнительные сведения о роли пользователей и разрешения](./manage-account.md#define-user-roles-and-permissions).

### <a name="trends-for-active-and-canceled-orders"></a>Тенденции активных и отмененных заказов

**Активный упорядочивает по предложения** кольцевой график единое активных заказов в соответствии с их имена предложения.

- Top 4 предложения отображаются в графе и до конца предложения группируются как «Rest-All».
- Можно выбрать конкретные предложения в условных обозначениях, чтобы отобразить только эти предложения на графике. 
- При наведении курсора на срез в графе отображает количество заказов и процент предложения по сравнению с вашей общее количество заказов по всем предложениям.
- **Заказы по тренда предложения** показывает тенденции роста по месяцам. Столбца «месяц» представляет количество заказов по имя предложения. График отображает роста в процентах по оси z.
- Можно использовать ползунок в верхней части диаграммы для прокрутки вправо и влево вдоль оси x и сосредоточиться на определенных точках данных.
- Диаграммы трендов можно отобразить, выбрав конкретного элемента в условных обозначениях.
- Также можно отобразить тенденции и данные для **отмененных заказов**. Граф будет функционировать в так же, как активных заказов.

### <a name="orders-by-customer-type"></a>Заказы по типам клиентов

**Заказы по типам клиентов** линейчатая диаграмма отображает количество заказов, разделены между **новых клиентов** и **существующих клиентов**. 

- Объект **нового клиента** приобрела один или несколько ваших предложений в первый раз в тот же месяц календаря (ось y). **Существующий клиент** ранее получил предложение от вас до календарного месяца, учитывающее (ось y). 
- Дополнительные круговая диаграмма представляет все заказы, созданные нового или существующего клиента для выбранного диапазона дат.
- В обе диаграммы можно просмотреть только новые или только существующие клиенты, выбрав соответствующие условные обозначения.

![Введите центра партнеров анализ заказы по клиентам](./media/analyze-order-by-customer.png)

### <a name="order-details-table"></a>Таблицу сведений о заказах

**Таблицу сведений о заказах** отображается нумерованный список заказов top 1000, отсортированных по дате приобретения.

- Каждый столбец в сетке поддерживает сортировку.
- Данные могут быть извлечены TSV-файл, если число записей меньше 1000.
- Если количество записей по 1000, экспортировать данные асинхронно помещаются на странице загрузки для следующих 30 дней.
- Фильтры могут применяться к **таблицу сведений о заказах** для отображения только данные, которые вас интересуют. Данные могут быть отфильтрованы по странам, типам лицензий Azure, типу лицензии Marketplace, тип предложения, состояние заказа, бесплатные след, Marketplace подписки идентификатор, Customer ID, и название компании.

### <a name="customer-totals"></a>Итоги по клиента

**Итоги клиента** разделе отображается количество всех клиентов, включая новые, существующих и находящиеся, во время выбранного [диапазон дат](#date-range).

- Клиенты процент роста по сравнению с прошлого месяца обозначается количество и вверх индикатора в индикатор зеленый или вниз красным цветом.
- Тренде представлены гистограмм и будет отображаться значение для каждого месяца, наведя указатель столбцов диаграммы.

#### <a name="customer-types"></a>Типы клиентов

Существует три типа клиента: новых, существующих и находящиеся. 

- Новый клиент получил один или несколько ваших предложений в первый раз в выбранном месяце.
- Существующий клиент получил один или несколько ваших предложений до месяца.
- Все предложения, которые ранее приобретенной отменил churned клиента.

### <a name="customer-trends"></a>Тенденции клиентов

**Клиентов тенденции** диаграмме отображается количество всех клиентов, включая новые, существующих и находящиеся с трендом роста по месяцам.

- График представляет общую процентные показатели роста клиента. 
- Столбца «месяц» представляет число клиентов с накоплением новых, существующих и находящиеся клиентами.
- Число клиентов, находящиеся отображается отрицательное направление оси Y.
- Можно выбрать конкретными условными обозначениями, элементы отображаются более подробные представления. Например выберите новые клиенты из условных обозначений для отображения только новых клиентов.
- Можно использовать ползунок в верхней части диаграммы для прокрутки вправо и влево по оси x и сосредоточиться на определенных точках данных для просмотра более подробно.
- При наведении курсора на столбец диаграммы будут отображаться сведения о только в этом месяце.

![Тенденции клиентов анализ центра партнеров](./media/analyze-customer-trends.png)

### <a name="customers-by-orders"></a>Клиенты, заказы

Оси z (график) отображается суммарный процент от общего числа заказов. Можно просмотреть сведения, наведя указатель точек на графике.

Например можно узнать из данных заказа клиента, первые 30% клиентов, вносящих вклад 83% заказов, равным 2,130 заказов.

# <a name="check-the-publishing-status-of-your-commercial-marketplace-offer"></a>Проверьте состояние публикации вашего предложения Коммерческого рынка

Вы можете просмотреть свой **текущий статус Публикации** на вкладке **Обзор предложения** [портала Коммерческого рынка](https://partner.microsoft.com/dashboard/commercial-marketplace/offers) в Партнерском центре.

Для каждого предложения должен отображаться один из следующих индикаторов состояния.

| **Состояние**    | **Описание**  |
| :---------- | :-------------------|
| **Draft** | Предложение было создано, но оно не публикуется. |
| **Publish in progress** (Публикация выполняется) | Предложение/План проходит через этапы процесса публикации. |
| **Требуется внимание** | Критическая проблема была обнаружена во время сертификации корпорацией Майкрософт или любым из этапов публикации. |
| **Предварительный просмотр** | Предложение было сертифицировано корпорацией Майкрософт, и теперь ожидает окончательной проверки издателем. Выберите идти жить, чтобы сделать предложение жить. |
| **Опубликовано** | Предложение живут на рынке и могут быть замечены и приобретены клиентами. |
| **В ожидании остановки продать** | Издатель выбрал "остановить продажу" на предложение или план, но действие еще не завершено. |
| **Недоступно на рынке** | Ранее опубликованное предложение/план на рынке было удалено. |

## <a name="automated-validation"></a>Автоматизированная проверка

Первым шагом в процессе публикации является набор автоматизированных валидий. Каждый шаг проверки соответствует функции, которую вы выбрали для создания вашего предложения. Если эта функция не была включена, проверка переходят к следующему этапу публикации. Каждая проверка проверки должна быть завершена до утверждения статуса публикации.

- **Установка потока предложения (<10 мин)**

На этом этапе мы гарантируем, что ваше предложение может быть выполнено при покупке клиентами через портал Azure. Этот шаг применим только для предложений, продаваемых через корпорацию Майкрософт.

- **Проверка данных тест-драйва (5 мин)**

На этом этапе мы проверяем данные, предоставленные вами в разделе технической конфигурации тест-драйва предложения. Функциональность тест-драйва протестирована и одобрена. Этот шаг применим только для предложений с включенным тест-драйвом.

- **Подготовка тест-драйва (30 мин)**

На этом этапе, после проверки данных и функциональности вашего тест-драйва на предыдущем этапе, мы развертываем и реплицируем экземпляры вашего тест-драйва, чтобы они были готовы к использованию клиентами.  Этот шаг применим только для предложений с включенным тест-драйвом.

- **Проверка и регистрация руководства (<15 мин)**

На этом этапе мы подтверждаем, что ваша ведущая система управления может получать интересы клиентов на основе сведений, представленных в настройке Предложения. Этот шаг применим только для предложений с включенным управлением Lead.

## <a name="certification"></a>Сертификация

Перед публикацией предложения, представленные в Коммерческий рынок в партнерской центре, должны быть сертифицированы. Представленные предложения проходят тщательное тестирование, некоторые автоматизированные и другие руководства, в том числе проверки против [политики сертификации Azure Marketplace.](https://docs.microsoft.com/legal/marketplace/general-policies) Предложения должны быть помечены как имеющие право на сертификацию, прежде чем они переходят к следующему шагу в потоке публикации.

### <a name="types-of-validation-that-take-place-during-certification"></a>Виды проверки, которые проводятся во время сертификации

Для каждого представленного предложения в процесс сертификации включены три уровня проверки.

- Право на участие в издательстве
- Проверка содержимого
- Техническая проверка

#### <a name="publisher-business-eligibility"></a>Право на участие в издательстве

Каждый тип предложения проверяет набор базовых критериев отбора, которым должен соответствовать издатель. Критерии отбора могут включать статус MPN издателя, компетенции, уровни компетентности и т.д.

#### <a name="content-validation"></a>Проверка содержимого

Во время проверки содержимого информация, введенная при создании предложения, проверяется на качество и релевантность. Эти проверки будут рассматривать ваши записи на рынке список деталей, цены, доступность, связанные планы и т.д. Чтобы соответствовать критериям azure Marketplace и/или AppSource, мы проверяем, что ваше предложение включает в себя:

- название, которое точно описывает предложение;
- хорошо написанные описания, которые обеспечивают тщательный обзор и ценностное предложение;
- качественные скриншоты и сопровождающие видео; И
- объяснение того, как предложение использует платформы и инструменты Microsoft.

Узнайте больше о критериях проверки содержимого, прочитав [общие правила листинга.](https://docs.microsoft.com/legal/marketplace/certification-policies#100-general)

#### <a name="technical-validation"></a>Техническая проверка

Во время технической проверки предложение (пакет или двоичный) проходит следующие проверки.
- Сканируется на вредоносные программы
- Сетевые звонки отслеживаются
- Анализ пакета
- Тщательное сканирование фактической функциональности предложения

Предложение тестируется на различных платформах и версиях, чтобы убедиться, что оно надежное.

Просмотрите конкретные детали конфигурации, необходимые для вашего предложения в разделе Технической конфигурации этого документа.

### <a name="certification-failure-report"></a>Отчет о сбое сертификации

По завершении обзора, если ваше предложение прошло сертификацию, то оно переходит к следующему шагу в процессе публикации. Если ваше предложение не удалось любой из листинга, технических или политики проверки, или если вы не имеете права представить предложение такого типа, отчет о сбое сертификации генерируется и по электронной почте к вам.

В этом отчете содержатся описания любых неудавшихся политик, а также заметки о рецензии. Просмотрите этот отчет по электронной почте, устраните любые проблемы, внося обновления в ваше предложение, где это необходимо, и повторно отправьте предложение с помощью [портала Commercial Marketplace](https://partner.microsoft.com/dashboard/commercial-marketplace/offers) в Партнерс-центре. (Вы можете повторно отправить предложение столько раз, сколько необходимо до прохождения сертификации).

## <a name="preview-creation"></a>Создание предварительного просмотра

На этапе **создания Preview** мы создаем версию вашего предложения, доступную только той аудитории, которую вы указали в разделе Preview вашего предложения.

>[!Note]
> Не используйте этот шаг, чтобы дать людям за пределами вашей организации видимость в предложение. Вместо этого используйте опцию **«Частное предложение».** На данный момент ваше предложение не было полностью протестировано и проверено и не готово к внешнему распределению.

## <a name="publisher-approval"></a>Утверждение издателя

На этом этапе вам будет отправлен запрос на просмотр и утверждение предварительного просмотра предложения до последнего шага публикации.

Если вы решили продать свое предложение через корпорацию Майкрософт, вы сможете протестировать приобретение и развертывание вашего предложения, чтобы убедиться, что оно соответствует вашим требованиям на этом этапе предварительного просмотра. Ваше предложение пока не будет доступно на публичном рынке. После тестирования и утверждения предварительного просмотра необходимо выбрать **Go-Live** на панели мониторинга [**Offer Overview.**](https://partner.microsoft.com/dashboard/commercial-marketplace/overview)

Если вы хотите внести изменения в предложение на этом этапе предварительного просмотра, вы можете отойти и отправить повторно, чтобы опубликовать новый предварительный просмотр. Смотрите статью [Обновление существующих рыночных предложений](#update-existing-marketplace-offers) для получения подробной информации о более изменений.

Если ваше предложение уже доступно и доступно для публики на рынке, любые обновления, которые вы делаете, не будут жить, пока вы не выберете **Go-live** на панели мониторинга [**Offer Overview.**](https://partner.microsoft.com/dashboard/commercial-marketplace/overview)

### <a name="publish-offer-to-the-public"></a>Опубликовать предложение для общественности

Вопийте в Партнерс-центре и получите доступ к предложению. Вы будете перенаправлены на страницу **обзора предложения.** В верхней части этой страницы вы увидите опцию **Go Live.** Выберите **Go Live,** и после подтверждения, предложение начнет получать опубликованы для общественности. Вы получите уведомление по электронной почте, когда предложение будет в прямом эфире.

## <a name="publish"></a>Публикация

Теперь, когда вы выбрали **Go Live** с вашим предложением, что делает его доступным на рынке, Есть ряд окончательных проверок проверок, которые будут активизированы, чтобы убедиться, что живое предложение настроено так же, как предварительный вариант предложения.

- **Установка потока предложения (>10 мин)**

На этом этапе мы гарантируем, что ваше предложение может быть выполнено при покупке клиентами через портал Azure. Этот шаг применим только для предложений, продаваемых через корпорацию Майкрософт.

- **Проверка данных тест-драйва (5 мин)**

На этом этапе мы проверяем данные, предоставленные вами в разделе технической конфигурации тест-драйва предложения. Функциональность тест-драйва протестирована и одобрена. Этот шаг применим только для предложений с включенным тест-драйвом.

- **Подготовка тест-драйва (30 мин)**

На этом этапе мы развертываем и воспроизводит экземпляры вашего тест-драйва, чтобы они были готовы к использованию клиентами.  Этот шаг применим только для предложений с включенным тест-драйвом.

- **Проверка и регистрация руководства (>15 мин)**

На этом этапе мы подтверждаем, что ваша ведущая система управления может получать интересы клиентов на основе сведений, представленных в настройке Предложения. Этот шаг применим только для предложений с включенным управлением Lead.

- **Окончательное издание (>30 минут)**

На этом этапе мы гарантируем, что ваше предложение станет общедоступным на рынке.

## <a name="update-existing-marketplace-offers"></a>Обновление существующих предложений на рынке

Если вы хотите внести изменения в предложение, опубликовавеееее, вам необходимо сначала обновить существующее предложение, а затем опубликовать его снова.

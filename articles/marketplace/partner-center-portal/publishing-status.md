# <a name="check-the-publishing-status-of-your-commercial-marketplace-offer"></a>Проверьте статус публикации вашего предложения для коммерческих Marketplace

Можно просмотреть текущие **состояние публикации** на **предлагают общие сведения о** вкладке [коммерческих Marketplace портала](https://partner.microsoft.com/dashboard/commercial-marketplace/offers) в центре партнеров.

## <a name="automated-validation"></a>Автоматические проверочные

Первым шагом в процессе публикации — это набор автоматических проверок. Каждый шаг проверки соответствует функция, которую вы решили включить при создании вашего предложения. Если этот компонент не был включен, проверка пропускает к следующему шагу публикации. Каждая проверка должны быть завершены перед утверждением состояние публикации.

- **Предложить установки поток приобрести (> 10 мин)**

На этом шаге мы гарантируем, что ваше предложение может быть выполнено при покупке клиентами на портале Azure. Этот шаг применяется только в предложениях распространяемые через Microsoft.

- **Тестирование проверки диска данных (приблизительно на 5 мин)**

На этом шаге мы проверяем данные, которые вы указали в тестовый выпуск раздел технической конфигурации предложения. Функции тестирования диска тестирования и утверждения. Этот шаг применяется только для предложений с поддержкой тестовый выпуск.

- **Проверить диск подготовки (30 мин)**

На этом шаге, после проверки данных и функциональных возможностей в тестовый выпуск на предыдущем шаге развертывание и реплицировать экземпляры ваш тестовый запуск, чтобы они готовы для пользователей.  Этот шаг применяется только для предложений с поддержкой тестовый выпуск.

- **Привести управления проверки и регистрации (> 15 мин.)**

На этом шаге убедитесь, что системе интереса управления может получать интересы на основе сведений, предоставленный в настройках предложения. Этот шаг применяется только для предложений с поддержкой управления интереса.

## <a name="certification"></a>Сертификация

Перед публикацией, должны быть сертифицированы предложения, отправленные коммерческих партнеров в центре партнеров. Отправить предложения проходят тщательное тестирование, некоторые автоматизированные и другим пользователям вручную, включая проверку по [политики сертификации Azure Marketplace](https://docs.microsoft.com/legal/marketplace/general-policies). Предложите отправленные данные должны быть помечены для сертификации продолжения к следующему шагу в процесс публикации.

### <a name="types-of-validation-that-take-place-during-certification"></a>Виды проверок, выполняемых во время сертификации

Существует три уровня проверки, включаемых в процесс сертификации для каждого предложения отправлен.

- Соответствия требованиям бизнеса издателя
- Проверка содержимого
- Техническая проверка

#### <a name="publisher-business-eligibility"></a>Соответствия требованиям бизнеса издателя

Каждый тип предложения проверяет набор критериев допустимости базовый, которым должны соответствовать издателю. Условия для допуска может включать состояние издателя MPN, удерживаемых компетенции, уровней компетенции т. д.

#### <a name="content-validation"></a>Проверка содержимого

Во время проверки содержимого сведения, введенные при создании вашего предложения проверяются на качество. Эти проверки будут просмотрите записи для рынка, сведения, цены, доступности, соответствующих планов т. д. В соответствии с Azure Marketplace или AppSource, список критериев, мы проверим, что ваше предложение включает в себя:

- Заголовок, который точно описывает предложение;
- правильно составленный описания, которые обеспечивают тщательное Обзор и ценность;
- снимки экрана качества и сопутствующие видео; и
- Объяснение, как предложение использует платформ и средств Майкрософт.

Дополнительные сведения о критерии проверки содержимого можно найти [общие список политик](https://docs.microsoft.com/legal/marketplace/general-policies#10-general-listing-policies).

#### <a name="technical-validation"></a>Техническая проверка

Во время проверки технические предложение (пакета или двоичный) проходит следующие проверки.
- Проверка на наличие вредоносных программ
- Мониторинг сетевых вызовов
- Анализ пакетов
- Тщательное сканирование фактическую функциональность предложения

Предложение тестируется в различных платформ и версий, чтобы убедиться, что служба является устойчивой.

Просмотрите сведения о конкретной конфигурации, необходимые для вашего предложения в разделе технических конфигурации этого документа.

### <a name="certification-failure-report"></a>Отчет о сбоях при сертификации

После завершения проверки Если ваше предложение достиг сертификации передает к следующему шагу в процесс публикации. Если вашего предложения произошел сбой проверки листинг, технического характера или политики или не имеете права Отправить предложение этого типа, создается отчет об ошибках сертификации и вам по электронной почте.

Этот отчет содержит описания всех политик, которые не удалось, а также примечания к проверке. Просмотрите этот отчет по электронной почте, устраните все проблемы, внесение обновлений к вашему предложению там, где требуется и повторно отправьте предложение с помощью [коммерческих Marketplace портала](https://partner.microsoft.com/dashboard/commercial-marketplace/offers) в центре партнеров. (Вы можете повторно отправить предложение столько раз, при необходимости до передачи сертификации).

## <a name="preview-creation"></a>Создание предварительного просмотра

Во время **предварительного просмотра создания** шаге мы создать версию предложения доступной для аудитории, который указан в разделе предварительного просмотра вашего предложения.

## <a name="publisher-approval"></a>Издатель утверждения

На этом шаге вы будет отправлять по электронной почте с помощью запроса можно просмотреть и утвердить предварительной версии предложения перед последним шагом публикации.

Если вы решили продавать предложения через Microsoft, можно протестировать приобретения и развертывания вашего предложения, чтобы обеспечить соответствие требованиям во время этого этапа утверждения предварительной версии. Ваше предложение будет еще недоступен в marketplace общедоступного. После тестирования и утверждения этой предварительной версии, необходимо выбрать **Go-Live** на [ **предлагают общие сведения о** ](https://partner.microsoft.com/dashboard/commercial-marketplace/overview) панели мониторинга.

Если вы хотите внести изменения в предложение, на этом этапе предварительной версии, можно изменить и повторно отправить для публикации новой предварительной версии. См. в статье [обновления существующих marketplace предлагает](#update-existing-marketplace-offers) Дополнительные сведения о дополнительных изменений.

Если ваше предложение уже динамической и общедоступным в marketplace, все обновления, внесенные не будут активированы только после выбора **Go-live** на [ **предлагают общие сведения о** ](https://partner.microsoft.com/dashboard/commercial-marketplace/overview) панель мониторинга.

### <a name="publish-offer-to-the-public"></a>Опубликовать предложение общедоступным

Войдите в центр партнеров и доступ к предложению. Вы будете перенаправлены к **предлагают общие сведения о** страницы. В верхней части этой страницы, вы увидите параметр для **функционирование**. Выберите **активированы,** и после подтверждения, предложение будет запущена, опубликованным общедоступным. Вы получите уведомление по электронной почте, когда предложение станет активным.

- **Публикация final (> 30 минут)**

На этом шаге мы гарантируем, что ваше предложение станет общедоступным в marketplace.

# <a name="usage-dashboard-in-microsoft-commercial-marketplace-analytics"></a>Панель мониторинга использования в аналитике коммерческой платформы Майкрософт

Эта статья содержит сведения о панели мониторинга использования в Центре партнеров. На этой панели отображаются все метрики использования виртуальных машин и измеренные метрики выставления счетов на двух отдельных вкладках: "Использование виртуальной машины" и "Использование с выставлением счетов по тарифу".

Чтобы перейти к панели мониторинга использования, откройте панель мониторинга **[Анализ](https://partner.microsoft.com/dashboard/commercial-marketplace/analytics/summary)** в разделе **Коммерческий Marketplace**.

>[!NOTE]
> Подробные определения терминов аналитики см. в статье [Часто задаваемые вопросы и терминология по аналитике коммерческой платформы](./faq-terminology.md).

## <a name="usage-dashboard"></a>Панель мониторинга использования

Панель мониторинга использования представляет метрики для всех предложений использования виртуальной машины (ВМ) и использования с выставлением счетов по тарифу. Эти метрики находятся на двух отдельных вкладках: "Использование виртуальной машины" и "Использование с выставлением счетов по тарифу".

На вкладке "Использование виртуальной машины" можно просмотреть приведенные ниже графические представления.

- [Сводка использования](#usage-summary)
- [Использование по географическому расположению](#usage-by-geography)
- [Использование по предложениям](#usage-by-offers)
- [Тренд использования по предложениям и планам](#usage-trend-by-offers-and-plans)
- [Использование по типам предложений](#usage-by-offer-type)
- [Использование по размеру виртуальной машины](#usage-by-vm-size)
- [Использование по каналам продаж](#usage-by-sales-channel)
- [Подробные данные об использовании](#detailed-usage-data)

Максимальная задержка между созданием событий использования и отчетами в центре партнеров составляет 48 часов.

### <a name="usage-summary"></a>Сводка использования

В таблице сводки использования отображаются часы использования ресурсов клиентами для всех предложений, которые они приобрели.

- Часы нормализованного использования определяются как часы использования, нормализованные с учетом количества ядер виртуальных машин: [количество ядер виртуальных машин] x [количество часов фактического использования]. Для виртуальных машин, используемых в режиме общего ядра (SHAREDCORE), в качестве множителя [количество ядер виртуальных машин] используется значение 1/6 (или 0,1666).
- Часы фактического использования определяются как количество времени работы виртуальных машин в часах.
- Значение в процентах представляет изменение роста показателей использования для выбранного диапазона дат ([использование в прошлом месяце – использование в первом месяце]) / использование в первом месяце).
- Зеленые треугольники, указывающие вверх, обозначают изменение роста.
- Красный треугольник, указывающий вниз, обозначает отрицательное изменение роста относительно предыдущего месяца.
- Миниатюрные линейные диаграммы представляют месячные значения. Если навести указатель мыши на столбцы этой диаграммы, отображается значение для каждого месяца.

### <a name="usage-by-geography"></a>Использование по географическому расположению

**Нормализованная** тепловая схема использования отображает часы использования, сопоставленные в соответствии со страной или регионом клиента. Изменение цвета страны или региона представляет собой концентрацию нормализованного использования. Чтобы вернуться к исходному представлению, нажмите кнопку **Домой** на карте.

### <a name="usage-by-offers"></a>Использование по предложениям

- На круговой диаграмме **нормализованного использования по предложениям** показана разбивка часов нормализованного использования по предложениям в соответствии с выбранным диапазоном дат. Пять лучших предложений отображаются на графике, а остальные сгруппированы в категорию **Все остальное**.
- На линейчатой диаграмме отображается тренд роста по месяцам для выбранного диапазона дат. Столбцы "Месяц" представляют часы использования из предложений с наибольшим количеством часов использования для соответствующего месяца. На графике отображается тренд роста в процентах, построенный по вторичной оси Y.
- Используйте ползунок в верхней части диаграммы, чтобы выполнить прокрутку вправо и влево вдоль оси x и (или) сосредоточиться на конкретных точках данных.

### <a name="usage-trend-by-offers-and-plans"></a>Тренд использования по предложениям и планам

На этой диаграмме показана тенденция нормализованного использования для выбранных планов (ранее именуемых номерами SKU) предложения. В списке лидеров предложений отображаются 50 основных предложений с максимальным уровнем использования, отсортированные по часам использования. План список лидеров отображает первые 50 планов с наибольшим использованием для выбранного предложения.

### <a name="usage-by-offer-type"></a>Использование по типам предложений

- Круговая диаграмма **Использование по типу предложения** организует использование в зависимости от типа предложения.
- Основные предложения отображаются на диаграмме, а остальные предложения группируются как "Все остальное".
- На диаграмме **Тренд** отображаются тренды роста по месяцам. Столбец "Месяц" представляет использование по основным типам предложений в этом месяце.

### <a name="usage-by-vm-size"></a>Использование по размеру виртуальной машины

Эта диаграмма представляет тренд использования для выбранных размеров виртуальных машин (максимум 5) всех ваших предложений и планов. Гистограмма помещается в стек с учетом часов использования выбранных размеров виртуальных машин.

В списке лидеров отображаются 50 основных размеров виртуальных машин с максимальным уровнем использования и сортировкой по часам использования.

### <a name="usage-by-sales-channel"></a>Использование по каналам продаж

- Круговая диаграмма использования по каналам продаж организует использование в соответствии с каналом продаж.
- Верхний канал продаж с наибольшим уровнем использования отображается на графике, а остальные каналы продаж сгруппированы как "Все остальное".
- Столбец "Месяц" представляет использование по основным каналам продаж в этом месяце.
- Возможности этой диаграммы такие же, как и у диаграммы "Использование по предложениям".

### <a name="detailed-usage-data"></a>Подробные данные об использовании

В **таблице сведений об использовании** отображается нумерованный список из первых 1000 записей об использовании, отсортированных по использованию.

- Каждый столбец в сетке доступен для сортировки.
- Данные можно извлечь в CSV-файл, если число записей меньше 1000.
- Если число записей превышает 1000, экспортированные данные будут асинхронно помещены на страницу загрузки, которая будет доступна в течение следующих 30 дней.
- Примените фильтры к **подробным данным об использовании** , чтобы отобразить только интересующие вас данные. Фильтрация данных по странам/регионам, каналу продаж, типу лицензии Marketplace, типу использования, имени предложения, типу предложения, бесплатным пробным, ИДЕНТИФИКАТОРу подписки Marketplace, ИДЕНТИФИКАТОРу клиента и названию компании.

> [!NOTE]
> В фильтре страницы выберите **Тип использования**, чтобы просмотреть диаграммы на странице в "нормализованном представлении" или "базовом представлении". По умолчанию для этих диаграмм используется "нормализованное представление".

**Фильтры страницы использования** применяются на уровне страницы. Вы можете выбрать несколько фильтров, чтобы отобразить диаграмму с выбранными критериями для просмотра и данными, которые должны отображаться в сетке/экспорте "Подробные данные об использовании". Фильтры применяются к данным, извлеченным для диапазона данных, выбранного в правом верхнем углу страницы заказов.

- **Типы предложений** и **названия предложений** указаны только для предложений, полученных в течение выбранного диапазона дат. Названия предложений в списке отображаются для типов предложений, выбранных в списке.
- По умолчанию для каждого из параметров фильтра, кроме **типа использования**, выбрано значение "Все". Выбором по умолчанию для **типа использования** является нормализованное использование. Чтобы отобразить фактическое использование в диаграммах, выберите "фактическое использование".
- Примененные фильтры показывают количество выбранных фильтров. Примененные фильтры не отображаются для выборов по умолчанию.

> [!NOTE]
> Подробное определение каждого из полей в сетке "подробные данные о заказе", фильтры страниц и все возможные варианты выбора определены в разделе словаря данных статьи [Часто задаваемые вопросы и терминология](link needed).

На вкладке **Использование с выставлением счетов по тарифу** представлена информация об использовании для типов предложений, где использование измеряется по единице измерения. В настоящее время предоставляется превышение типа предложения SaaS. На вкладке находятся графические представления трендов превышения для использования контроля потребления с помощью SaaS:

- **Тренд превышения по измерению**: отображает месячный тренд превышения для выбранного измерения предложения. Ось X представляет месяц, а ось Y — объем использования. Единица измерения настраиваемой единицы также отображается на оси Y.
- **Чрезмерная тенденция по плану**: представляет тенденцию количества использований выбранного измерения измерительных единиц по планам. Отображаемые планы будут представлять пять первых планов с наибольшим объемом использования для выбранного предложения.
- **Тренд превышения по 50 основным клиентам**: 50 основных предложений с наибольшим количеством часов использования отображаются в ***списке лидеров*** и сортируются по наибольшему уровню использования в настраиваемых единицах измерения. Выберите клиента в списке лидеров, чтобы просмотреть тренд использования выбранного измерения.
- **Тренд превышения по основным клиентам**: представляет процентили основных клиентов, которые влияют на процент общего использования. Процентиль основного клиента отображается вдоль оси X и определяется объемом использования клиента. Ось Y отображает объем использования. Вы можете отобразить сведения, наведя указатель мыши на точки вдоль графика.

> [!NOTE]
> Сведения об использовании и все диаграммы на этой странице отображаются для любого измерения, выбранного для фильтра страницы.

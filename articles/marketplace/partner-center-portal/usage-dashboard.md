# <a name="usage-dashboard-in-commercial-marketplace-analytics"></a>Использование dashboard в коммерческих аналитики рынка

В этой статье содержится информация о панели мониторинга использования в Partner Center. Эта панель мониторинга отображает все VM предлагает использование и дозированные метрики выставления счетов в двух отдельных вкладках: использование VM и дозированные выставления счетов.

Чтобы получить доступ к панели мониторинга Usage, откройте панель мониторинга **[Analyse](https://partner.microsoft.com/dashboard/commercial-marketplace/analytics/summary)** под коммерческим рынком.

>[!NOTE]
> Для подробных определений терминологии аналитики, [см. Часто задаваемые вопросы и терминологию для коммерческой аналитики рынка](./faq-terminology.md).

## <a name="usage-dashboard"></a>Панель мониторинга использования

Панель мониторинга использования представляет метрики для всех виртуальных машин (VM), предлагает использование и дозированные использование биллинга. Они содержатся в двух отдельных вкладок: использование VM и дозированных выставления счетов.

Во вкладке использования VM есть графические изображения следующих элементов:

- [Сводка использования](#usage-summary)
- [Использование по географии](#usage-by-geography)
- [Использование по предложениям](#usage-by-offers)
- [Тенденция использования по предложениям и SKUs](#usage-trend-by-offers-and-skus)
- [Использование по типу предложения](#usage-by-offer-type)
- [Использование по размеру VM](#usage-by-vm-size)
- [Использование по каналу продаж](#usage-by-sales-channel)
- [Подробные данные об использовании](#detailed-usage-data)

> [!NOTE]
> Отчеты analytics по-разному отображаются в облачном партнерской портале (CPP) и Партнерской центре. **Продавца идеи** в CPP имеет заказы и использование вкладке, которая отображает данные как для использования на основе предложений и не-использования на основе предложений. В Partner Center метрики использования отображаются на отдельной странице.

### <a name="usage-summary"></a>Сводка использования

Сводка использования отображает часы использования клиентом для всех предложений, которые они приобрели.

- Нормализованные часы использования определяются как часы использования, нормализованные для учета количества ядер VM (количество ядер VM) x (часы сырого использования). ВМ, обозначенные как "SHAREDCORE", используют 1/6 (или 0.1666) в качестве множителя «числа ядер VM».
- Часы использования raw определяются как количество времени, время, время, время были запущены с точки зрения часов.
- Процентное значение представляет собой изменение роста использования для выбранного диапазона дат («использование в прошлом месяце – использование первого месяца»)/использование первого месяца).
- Зеленые треугольники, указывающие вверх, указывают на изменение роста.
- Красный треугольник, указывающий вниз, указывает на отрицательное изменение роста по отношению к предыдущему месяцу.
- Диаграммы микро-баров представляют ежемесячные значения. Значение можно отображать за каждый месяц, паря над столбцовами в диаграмме.

### <a name="usage-by-geography"></a>Использование по географии

**Нормализованное использование географической** тепловой карты отображает часы использования, отображаемые в зависимости от страны клиента. Изменение цвета страны представляет собой нормализованную концентрацию использования. Вернуться к исходному представлению, нажав **кнопку «Домой»** на карте.

### <a name="usage-by-offers"></a>Использование по предложениям

- **Нормализованное использование предложениями** круговой диаграммы отображает разбивку нормализованных часов использования предложениями в соответствии с выбранным диапазоном дат. Топ 5 предложений отображаются в графике, в то время как остальные сгруппированы в категории "все остальные".
- Диаграмма бара показывает месячный тренд роста выбранного диапазона дат. Столбцы месяца представляют часы использования предложений с самыми высокими часами использования за соответствующий месяц. Диаграмма линии изображает тренд процентного роста, начертанный на вторичной оси Y.
- Используйте ползунок в верхней части диаграммы, чтобы прокрутить справа налево вдоль x-оси и/или сосредоточиться на конкретных точках данных.

### <a name="usage-trend-by-offers-and-skus"></a>Тенденция использования по предложениям и SKUs

Эта диаграмма отображает тенденцию нормализованного использования для выбранных SUS предложения. В таблице лидеров предложения отображаются 50 лучших предложений с самым высоким уровнем использования и отсортированы по часам использования. В таблице лидеров SKU представлены 50 лучших SkKU с самым высоким использованием выбранного предложения.

### <a name="usage-by-offer-type"></a>Использование по типу предложения

- **Диаграмма типа предложения** организует использование в соответствии с типом предложения.
- Лучшие предложения отображаются на графике, а остальные предложения сгруппированы как "Rest All".
- Диаграмма **тренда** отображает месячные тенденции роста. Столбец месяца представляет использование верхними типами предложений в этом месяце.

### <a name="usage-by-vm-size"></a>Использование по размеру VM

Эта диаграмма представляет тенденцию использования для выбранных размеров VM (максимум 5) всех ваших предложений/SkUs. Диаграмма столбца укладывается с часами использования выбранных размеров VM.

В таблице лидеров отображаются 50 лучших размеров VM с максимальным использованием и отсортированы по часам использования.

### <a name="usage-by-sales-channel"></a>Использование по каналу продаж

- Использование продаж канал круговая диаграмма организует использование в соответствии с каналом продаж
- Верхний канал продаж с самым высоким использованием отображается на графике, а остальная часть канала продаж сгруппирована как 'Rest All'.
- Столбец месяца представляет использование ведущим каналом продаж в этом месяце.
- Особенности этой диаграммы такие же, как диаграмма «Использование предложения»

### <a name="detailed-usage-data"></a>Подробные данные об использовании

Таблица **сведения об использовании** отображает пронумерованный список 1000 записей об использовании, отсортированных по использованию.

- Каждая колонка в сетке сортируется.
- Данные могут быть извлечены в файл CSV, если количество записей меньше 1000.
- Если количество записей составляет более 1000, экспортные данные будут асинхронно размещены на странице загрузки, которая будет доступна в течение следующих 30 дней.
- Фильтры могут быть применены к **подробным данным об использовании** для отображения только интересуемых вами данных. Данные могут быть отфильтрованы по странам, каналу продаж, типу лицензии Marketplace, типу использования, названию, типу предложения, бесплатным пробным версиям, идентификатору подписки Marketplace, идентификатору клиента и названию компании.

> [!NOTE]
> Выберите **тип использования** в фильтре страницы для просмотра диаграмм на странице в "Нормализованном представлении" или "Сырое представление". Представление по умолчанию для этих диаграмм — «Нормализация представления».

**Фильтры страницы «Использование»** применяются на уровне страницы. Можно выбрать несколько фильтров для визуализации диаграммы для выбранных критериев и данных, которые вы хотите отобразить в сетке/экспорте данных детального использования. Фильтры применяются на данных, извлеченных для диапазона данных, выбранных в правом верхнем углу страницы заказов.

- **Типы предложений** и **названия предложений** перечислены только для предложений, которые вы приобрели в выбранном диапазоне дат. Имена предложений в списке отображаются для типов предложений, выбранных из списка.
- Выбор по умолчанию является "Все" для каждого из вариантов **фильтра,** за исключением типа использования. Выбор по умолчанию для **типа «Использование»** нормализованное использование. Для отображения необработанного использования в диаграммах выберите "сырое использование".
- Прикладные фильтры отображали выбор ы (ы) для сделанных фильтров. Прикладные фильтры не отображаются для выбора по умолчанию.

> [!NOTE]
> Подробное определение каждого из полей в сетке «данных детального заказа», фильтрах страниц и всех возможных выборах определено в разделе словаря данных в статье [часто задаваемых вопросов и терминологии.](link needed)

На вкладке **Metered billing use** inпредставляет информацию об использовании для типов предложений, где использование измеряется измерением метра. SaaS предложение типа избытка представлена в настоящее время. Вкладка представляет графические представления о тенденциях избытка для SaaS дозированных биллинга использования:

- **Тенденция перекорма по размеру метра**: Отображает ежемесячный тренд перекорма для выбранного измерения метра предложения. X-Axis представляет месяц, а Y-Axis — количество использования. Единица измерения пользовательского счетчика также отображается на Y-Axis.
- **Тенденция перекорма от SKU**: Представляет тенденцию использования количества выбранного метра размером с SKUs. Отображается SKUs будет представлять топ 5 SKUs с наибольшим объемом использования для выбранного предложения.
- **Тенденция к избытку по Топ-50 Клиентов**: Топ 50 предложений с самым высоким уровнем использования часов отображаются на ***доске лидеров*** и ранжируются по самому высокому использованию пользовательского счетчика. Выберите клиента в таблице лидеров, чтобы просмотреть тенденцию использования выбранного измерения метра.
- **Тенденция к переизбытку со стороны ведущих клиентов:** Представляет топ-процентиль клиента (ы), которые способствуют % от общего использования. Верхний процентиль клиента отображается вдоль X-оси и определяется количеством использования клиента. Оси Y отображает количество использования. Вы можете отображать детали, паря над точками вдоль диаграммы линии.

> [!NOTE]
> Детали использования и все диаграммы на этой странице отображаются для того, какой метр измерения выбран для фильтра страницы.

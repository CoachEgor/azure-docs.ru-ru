# <a name="create-a-commercial-marketplace-account-in-partner-center"></a>Создание коммерческой учетной записи Marketplace в центре партнеров

Чтобы опубликовать свои предложения в [Azure Marketplace](https://azuremarketplace.microsoft.com/) или [Microsoft AppSource](https://appsource.microsoft.com/), необходимо создать учетную запись в программе коммерческого рынка в центре партнеров. В этой статье описывается, как создать учетную запись центра партнеров и как добавить новых издателей в программу коммерческого рынка.

>[!IMPORTANT]
>Если у вас есть учетная запись в [портал Cloud Partner](https://cloudpartner.azure.com) , которая была перемещена в центр партнеров, вам не нужно создавать новую учетную запись. Дополнительные сведения см. в разделе [для издателей, которые перемещаются из портал Cloud Partner](#for-publishers-who-are-moving-from-the-cloud-partner-portal).

## <a name="before-you-begin"></a>Перед началом

Чтобы создать учетную запись в центре партнеров, убедитесь, что у вас есть:

- Право на подписание юридических соглашений от имени компании.
- Юридическое имя, адрес и основное контактное лицо вашей компании. Это может быть пользователь.

Мы проверим эту информацию в процессе создания учетной записи.

Существует два способа создания учетной записи.

- Если вы не знакомы с центром партнеров и у вас нет учетной записи Microsoft Partner Network (MPN), перейдите к статье [Создание учетной записи с помощью страницы регистрации центра партнеров](#create-an-account-by-using-the-partner-center-enrollment-page).
- Если вы уже зарегистрировались в Microsoft Partner Network, создайте учетную запись непосредственно в центре партнеров, перейдя к [созданию учетной записи, используя существующие регистрации в центре партнеров](#create-an-account-by-using-existing-microsoft-partner-center-enrollments).

## <a name="create-an-account-by-using-the-partner-center-enrollment-page"></a>Создание учетной записи с помощью страницы "регистрация в центре партнеров"

Используйте этот метод, если вы не знакомы с центром партнеров и не зарегистрированы в Microsoft Partner Network. В этом разделе описано, как создать новую учетную запись центра партнеров, в том числе:

- [Регистрация на странице регистрации в центре партнеров](#register-on-the-partner-center-enrollment-page)
- [Вход с рабочей учетной записью](#sign-in-with-a-work-account)
- [Принять условия](#agree-to-the-terms-and-conditions)
- [Укажите профиль издателя](#provide-your-publisher-profile)

>[!IMPORTANT]
>Если у вас есть учетная запись [портал Cloud Partner](https://cloudpartner.azure.com) , которая была перемещена в центр партнеров, вам не нужно создавать новую учетную запись. Дополнительные сведения см. в разделе [для издателей, которые перемещаются из портал Cloud Partner](#for-publishers-who-are-moving-from-the-cloud-partner-portal).

### <a name="register-on-the-partner-center-enrollment-page"></a>Регистрация на странице регистрации в центре партнеров

Просмотрите сведения на странице регистрации в [**центре партнеров Майкрософт**](https://partner.microsoft.com/dashboard/account/v3/enrollment/introduction/partnership) , а затем зарегистрируйтесь для получения учетной записи.

### <a name="sign-in-with-a-work-account"></a>Вход с рабочей учетной записью

Войдите с помощью рабочей учетной записи, чтобы связать домен учетной записи электронной почты организации с новой учетной записью центра партнеров. После того как вы собрались с этими двумя учетными записями, сотрудники компании могут войти в центр партнеров, используя свои имена пользователей и пароли своей рабочей учетной записи.

>[!NOTE]
>Чтобы узнать, есть ли у вашей компании Рабочая учетная запись, Узнайте, как создать новую рабочую учетную запись или настроить несколько рабочих учетных записей для использования с центром партнеров, перейдите [в раздел Рабочая учетная запись организации и центр партнеров](./company-work-accounts.md).

### <a name="agree-to-the-terms-and-conditions"></a>Принять условия

В рамках процесса регистрации необходимо согласиться с условиями [соглашения Microsoft Publisher](https://go.microsoft.com/fwlink/?LinkID=699560).

### <a name="provide-your-publisher-profile"></a>Укажите профиль издателя

Ваш профиль издателя включает название вашей компании и идентификатор MPN. Если вы еще не сделали этого, присоединитесь к [Microsoft Partner Network](https://partner.microsoft.com/commercial). После приподключения к Microsoft Partner Network вам будет предоставлен идентификатор MPN.

1. Создайте идентификатор издателя. Идентификатор издателя однозначно определяет вашу компанию и ваши предложения в Azure Marketplace и AppSource.

1. После подтверждения сведений о профиле издателя примите условия, а затем создайте учетную запись центра партнеров, выбрав **принять и продолжить**.

    > [!IMPORTANT]
    > Чтобы принять эти условия, необходимо иметь права на действия от имени компании.

    После регистрации вы перейдете на страницу обзора коммерческого рынка. На левой панели отобразится учетная запись коммерческого рынка. 
    
1. Чтобы убедиться, что учетная запись коммерческого магазина указана как зарегистрированная, выберите **Параметры**настройки  >  **партнеров**  >  **программы**.

Теперь вы создали коммерческую учетную запись Marketplace в центре партнеров. Дополнительные сведения см. в статье [Обзор коммерческого рынка](./commercial-marketplace-overview.md). Дополнительные сведения о добавлении издателей см. в статье [Добавление новых издателей в коммерческий](#add-new-publishers-to-the-commercial-marketplace-program)магазин.

## <a name="create-an-account-by-using-existing-microsoft-partner-center-enrollments"></a>Создание учетной записи с помощью существующих регистраций центра партнеров Майкрософт

Используйте этот раздел для создания коммерческой учетной записи Marketplace, если у вас уже есть регистрация в центре партнеров Майкрософт.

Существует два типа существующих регистраций, которые можно использовать для настройки учетной записи коммерческого магазина. Выберите ту, которая применяется к вам:

- Если у вас уже есть Microsoft Partner Network регистрации, создайте учетную запись с помощью функции [регистрации в Microsoft Partner Network](#what-if-im-already-enrolled-in-the-microsoft-partner-network).
- Если вы зарегистрировали существующую программу для разработчиков, создайте свою учетную запись с помощью команды [что, если я уже зарегистрировался в программе разработчика?](#what-if-im-already-enrolled-in-a-developer-program).

Для обоих типов регистрации вы будете:

- Войдите в центр партнеров с имеющимися учетными данными.
- Введите сведения о профиле публикации.

Убедитесь, что информация вашей учетной записи удобна.

>[!IMPORTANT]
>Если у вас есть учетная запись в [портал Cloud Partner](https://cloudpartner.azure.com/) , которая была перемещена в центр партнеров, вам не нужно создавать новую учетную запись. Дополнительные сведения см. в разделе [для издателей, которые перемещаются из портал Cloud Partner](#for-publishers-who-are-moving-from-the-cloud-partner-portal).

Если вы уже зарегистрировались в Microsoft Partner Network, используйте этот раздел для создания учетной записи непосредственно в центре партнеров.

Войдите в [Центр партнеров](https://partner.microsoft.com/dashboard/account/v3/enrollment/introduction/partnership) , используя учетную запись Microsoft Partner Network, чтобы связать домен учетной записи электронной почты вашей компании с новой учетной записью центра партнеров. После того как вы собрались с этими двумя учетными записями, сотрудники компании могут войти в центр партнеров, используя свои имена пользователей и пароли своей рабочей учетной записи.

>[!NOTE]
>Чтобы узнать, есть ли у вашей компании Рабочая учетная запись, Узнайте, как создать новую рабочую учетную запись или настроить несколько рабочих учетных записей для использования с центром партнеров, перейдите [в раздел Рабочая учетная запись организации и центр партнеров](./company-work-accounts.md).

После входа в систему завершите настройку учетной записи, выполнив следующие действия.

>[!NOTE]

1. В [центре партнеров](https://partner.microsoft.com/)выберите **Параметры**настройки  >  **партнеров**  >  **программы**  >  **Приступая к работе**.

   Microsoft Partner Network обнаружит подписку и отобразит панель **профиль издателя** .
   
1. Введите название компании и идентификатор MPN.
1. Ознакомьтесь с условиями [соглашения Microsoft Publisher](https://go.microsoft.com/fwlink/?LinkID=699560), а затем выберите **принять и продолжить** , чтобы завершить регистрацию.

    > [!IMPORTANT]
    > Чтобы принять эти условия, необходимо иметь права на действия от имени компании.

    После регистрации вы перейдете на страницу обзора коммерческого рынка. На левой панели отобразится учетная запись коммерческого рынка. 
    
1. Чтобы убедиться, что учетная запись коммерческого магазина указана как зарегистрированная, выберите **Параметры**настройки  >  **партнеров**  >  **программы**.

Теперь вы создали коммерческую учетную запись Marketplace в центре партнеров. Дополнительные сведения см. в статье [Обзор коммерческого рынка](./commercial-marketplace-overview.md). Дополнительные сведения о добавлении издателей см. в статье [Добавление новых издателей в коммерческий](#add-new-publishers-to-the-commercial-marketplace-program)магазин.

### <a name="what-if-im-already-enrolled-in-a-developer-program"></a>Что делать, если я уже зарегистрировался в программе разработчика?

Если вы уже зарегистрировались в программе для разработчиков центра партнеров, используйте этот раздел для создания учетной записи непосредственно в центре партнеров.

>[!NOTE]
>Чтобы зарегистрироваться для коммерческого рынка в той же учетной записи, в которой вы выполнили вход, необходимо зарегистрироваться в центре партнеров с учетной записью компании. Если вы зарегистрировались в отдельной учетной записи, ваша заявка на коммерческой рынок будет отображаться под новой учетной записью.
>
>Отдельные учетные записи предназначены для разработчиков, работающих над ними. Учетные записи компании предназначены для организаций и предприятий. Учетные записи компаний предоставляют доступ к отправке приложений с дополнительными функциональными возможностями. 
>
> Чтобы убедиться, что вы имеете право на настройку учетной записи компании, после регистрации вам потребуется дополнительная проверка учетных записей компании. Эта проверка может занять от нескольких дней до нескольких недель, и она часто включает телефонный звонок в компанию. Оба типа учетных записей позволяют отправлять приложения, надстройки и службы. Дополнительные сведения см. в разделе [типы счетов, местоположения и сборы](https://docs.microsoft.com/windows/uwp/publish/account-types-locations-and-fees).

Войдите в [Центр партнеров](https://partner.microsoft.com/dashboard/account/v3/enrollment/introduction/partnership) с помощью учетной записи центра партнеров Майкрософт, чтобы зарегистрироваться в программе коммерческого рынка под той же учетной записью. Зарегистрировавшись на коммерческом рынке с той же учетной записью, вы можете увидеть как существующие регистрации программы, так и торговую регистрацию Marketplace в левой области центра партнеров.

После входа в центр партнеров завершите настройку учетной записи, выполнив следующие действия.

1. В центре партнеров выберите **Параметры**  >  **Параметры разработчика**  >  **программы**начало  >  **работы**и введите рабочий адрес электронной почты.
1. Заполните сведения на странице **профиля издателя** .
1. Ознакомьтесь с условиями [соглашения Microsoft Publisher](https://go.microsoft.com/fwlink/?LinkID=699560), а затем выберите **принять и продолжить** , чтобы завершить регистрацию.

    >[!Important]
    > Чтобы принять эти условия, необходимо иметь права на действия от имени компании.

    После регистрации вы перейдете на страницу обзора коммерческого рынка. На левой панели отобразится учетная запись коммерческого рынка. 
    
1. Чтобы убедиться, что учетная запись коммерческого магазина указана как зарегистрированная, выберите **Параметры**настройки  >  **партнеров**  >  **программы**.

Теперь вы создали коммерческую учетную запись Marketplace в центре партнеров. Дополнительные сведения см. в статье [Обзор коммерческого рынка](./commercial-marketplace-overview.md). Дополнительные сведения о добавлении издателей см. в статье [Добавление новых издателей в коммерческий](#add-new-publishers-to-the-commercial-marketplace-program)магазин.

## <a name="for-publishers-who-are-moving-from-the-cloud-partner-portal"></a>Для издателей, которые перемещаются из Портал Cloud Partner

Если ваша учетная запись была перенесена из [портал Cloud Partner](https://cloudpartner.azure.com), вам не нужно создавать новую учетную запись центра партнеров. Вы должны получить настроенную ссылку на новую учетную запись центра партнеров как в сообщении электронной почты, так и в уведомлении в виде баннера после входа в существующую учетную запись Портал Cloud Partner.

После выбора настроенной ссылки и включения новой учетной записи центра партнеров вы можете вернуться к своей учетной записи, перейдя на [панель мониторинга коммерческого рынка](https://partner.microsoft.com/dashboard/commercial-marketplace/overview) в центре партнеров.

Сведения о соглашении о публикации и профиле компании переносятся в новую учетную запись центра партнеров, а также все ранее настроенные сведения профиля о выплатах учетных записей, учетные записи пользователей и разрешения, а также активные предложения, связанные с вашей учетной записью Портал Cloud Partner.

После перемещения сведений об учетной записи из Портал Cloud Partner в центр партнеров вы больше не будете использовать Портал Cloud Partner для выполнения обновления учетной записи, а также для управления пользователями, разрешениями и выставлением счетов. В течение ограниченного времени все обновления учетной записи, внесенные в центре партнеров, автоматически обновляются в учетной записи Портал Cloud Partner только для чтения, пока она не будет считаться устаревшей.

## <a name="add-new-publishers-to-the-commercial-marketplace-program"></a>Добавление новых издателей в программу коммерческого рынка

Организация может иметь несколько издателей, связанных с коммерческой учетной записью Marketplace. Существующий пользователь может добавить дополнительных издателей после входа в центр партнеров, выбрав **Параметры учетной записи**  >  **Издатели**  >  **Добавить издатель**.

>[!NOTE]
>Перед добавлением нового издателя проверьте список существующих издателей, войдя в центр партнеров и выбрав **Параметры учетной записи**  >  **Издатели**.

Дополнительные пользователи из того же клиента Azure Active Directory могут добавить нового издателя, выполнив следующие действия.

1. Запустите поток регистрации в [центре партнеров Майкрософт](https://partner.microsoft.com/dashboard/account/v3/enrollment/introduction/partnership).
2. Выберите **войти с рабочей учетной записью**и введите рабочий адрес электронной почты.
3. Нажмите кнопку **добавить издателя** .
4. Выберите идентификатор MPN, который нужно связать с издателем.
5. Обновите **сведения об издателе** в форме.

   * **Имя издателя**: имя, которое отображается в коммерческом магазине с предложением.  
   * **PublisherID**— идентификатор, используемый центром партнеров для уникальной идентификации издателя. Значение по умолчанию для этого поля соответствует существующему и уникальному ИДЕНТИФИКАТОРу издателя в системе. Поскольку идентификатор издателя нельзя использовать повторно, это поле необходимо обновить.  
   * **Контактные данные**. при необходимости обновите контактные данные.

После завершения этого процесса вы сможете управлять только что созданным издателем, перейдя к учетной записи коммерческого магазина, указанной на левой панели. Если вы не видите учетную запись коммерческого магазина, обновите страницу. Новый издатель появится в списке **издателей** .

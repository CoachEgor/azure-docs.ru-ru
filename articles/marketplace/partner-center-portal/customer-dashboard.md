# <a name="customer-dashboard-in-commercial-marketplace-analytics"></a>Панель мониторинга клиентов в аналитике коммерческого рынка

В этой статье содержится информация о **панели мониторинга Клиента** в Центре Партнеров. Эта панель мониторинга отображает информацию о ваших клиентах, включая тенденции роста, представленные в графическом и загружаемом формате.

Чтобы получить доступ к **панели мониторинга Клиента,** откройте панель мониторинга **[Analyse](https://partner.microsoft.com/dashboard/commercial-marketplace/analytics/summary)** под коммерческим рынком.

>[!NOTE]
> Для подробных определений терминологии аналитики, [см. Часто задаваемые вопросы и терминологии для аналитики коммерческого рынка](./faq-terminology.md).

## <a name="customer-dashboard"></a>Панель мониторинга клиентов

Панель **мониторинга клиента** меню **Analyse** отображает данные для клиентов, которые приобрели ваши предложения. Вы можете просмотреть графические представления следующих элементов:

- [Резюме клиентов](#customer-summary)
- [Клиент по географии](#customer-by-geography)
- [Тенденции клиентов](#customer-trends)
- [Клиенты по заказам и использованию](#customers-by-orders-and-usage)
- [Клиенты по СКУ](#customers-by-skus)
- [Заказы и использование по типу клиента](#orders-and-usage-by-customer-type)
- [Таблица сведений о клиентах](#customer-details-table)
- [Фильтры страниц клиента](#customer-page-filters)

### <a name="customer-summary"></a>Резюме клиентов

В разделе "Сводка клиента" отображается количество всех клиентов, включая новых, существующих и вспененных, в выбранном диапазоне дат.

- Общее количество клиентов определяется как количество всех клиентов, которые приобрели ваше предложение и имеют по крайней мере один заказ, который не был отменен.
- Процент роста клиентов по сравнению с предыдущим месяцем указывается на число и восходящий индикатор в зеленом или нисходящей индикаторе красным цветом.
- Тенденции роста представлены барными графиками и будут отображать значение для каждого месяца, паря над столбцовами диаграммы.

Существует три **типа клиентов:** новые, существующие и вспененные.

- Новый клиент приобрел один или несколько ваших предложений впервые в течение выбранного месяца.
- Существующий клиент приобрел одно или несколько ваших предложений до выбранного месяца.
- Вспененный клиент отменил все ранее приобретенные предложения.

### <a name="customer-by-geography"></a>Клиент по географии

**График Клиента по географии** показывает количество всех клиентов и клиентов, приобретенных в выбранном диапазоне дат, и отображается на основе страны клиента. Свет к темный цвет на карте представляет собой низкое и высокое значение количества клиентов. Нажмите запись в таблице, чтобы увеличить на страну.

Тепловая карта отображает количество клиентов и % по стране клиента. Карту можно переместить, чтобы просмотреть точное местоположение и увеличить в определенное место. Эта карта имеет дополнительную сетку, которая позволяет просматривать % клиентов по местоположению, а также клиентов, недавно добавленных к этому местоположению.

### <a name="customer-trends"></a>Тенденции клиентов

Диаграмма **тенденций клиентов** отображает количество всех клиентов, включая новых, существующих и вспененных, с месячным трендом роста.

- Диаграмма строки представляет общие проценты роста клиентов.
- Столбец месяца представляет количество клиентов, уложенных новыми, существующими и вспененные клиенты.
- Вспененный подсчет клиентов отображается на отрицательном направлении Y Axis.
- Можно выбрать определенные элементы легенды для отображения более подробных представлений. Например, выберите новых клиентов из легенды, чтобы отображать только новых клиентов.
- Слайдер можно использовать в верхней части диаграммы для прокрутки правой и левой на оси x и сосредоточиться на конкретных точках данных для более подробного просмотра.
- Нависая над столбцом диаграммы будет отображаться детали только за этот месяц.

### <a name="customers-by-orders-and-usage"></a>Клиенты по заказам и использованию

**Диаграмма Клиентов по заказам/использованию** имеет три вкладки: "заказы", "нормализованное использование" и "сырое использование". **Верхний процентиль клиента** отображается вдоль оси x, определяемый их количеством заказов. Оси y отображает количество заказов клиента. Вторичная y-оси (линейный график) отображает совокупный процент от общего числа ордеров. Вы можете отображать детали, паря над точками вдоль диаграммы линии.

В качестве примера приведена диаграмма ниже для нормализованного использования: 30-й процентиль клиентов вносит свой вклад в 87% нормализованного использования в совокупности. 30-й процентиль клиентов только способствует 1,57 миллиона часов использования.

### <a name="customers-by-skus"></a>Клиенты по СКУ

Ниже описаны **графики клиентов по SKUs/использованию.**

1. Совет лидеров представляет подробную информацию о 50 лучших клиентах, ранжированных по количеству заказов. После выбора клиента, детали клиента представлены в разделах 2, 3 и 4 этой доски лидеров.
2. Сведения о профиле клиента отображаются в этом пространстве, когда издатели входят в систему с ролью владельца. Если издатели вошли в систему с ролью участника, детали в этом разделе будут недоступны.
3. Заказы SKUs donut диаграммы отображает разбивку заказов, приобретенных для SKUs. Отображаются топ-5 SkUS с самым высоким количеством заказов, в то время как остальные заказы сгруппированы под "все остальные".
4. Сиденья по SKUs пончик диаграммы отображает разбивку мест, заказанных для SKUs. Отображаются топ-5 SkUs с самыми высокими сиденьями, в то время как остальные заказы сгруппированы под остальными.

### <a name="orders-and-usage-by-customer-type"></a>Заказы и использование по типу клиента

Диаграмма **заказов/использования по типу клиента** отображает количество заказов, нормализованное использование и необработанные часы использования, разделенные между новыми клиентами и существующими клиентами; на основе выбора ордеров, нормализованного и сырого использования соответственно на графике.

- Новый клиент приобрел одно или несколько ваших предложений или впервые сообщил об использовании в течение того же календарного месяца (y-axis).
- Существующий клиент ранее приобрел предложение от вас или сообщил об использовании до календарного месяца сообщили (на y-оси).

### <a name="customer-details-table"></a>Таблица сведений о клиентах

Таблица **сведений о клиентах** отображает номерной список 1000 лучших клиентов, отсортированных по дате, когда они впервые приобрели одно из ваших предложений.

- Личная информация клиента будет доступна только в том случае, если клиент дал согласие. Просмотреть эту информацию можно только в том случае, если вы вошли в систему с уровнем ролей владельца разрешений. Подробнее о роли пользователя и разрешениях.
- Каждая колонка в сетке сортируется.
- Данные могут быть извлечены в файл TSV, если количество записей меньше 1000.
- Если число записей превышает 1000, экспортированные данные будут асинхронно размещены на странице загрузки в течение следующих 30 дней.
- Фильтры могут быть применены к таблице для отображения только данных, которые вас интересуют. Данные могут быть отфильтрогнуты по имени компании, идентификатору клиента, идентификатору подписки Marketplace, типу лицензии Azure, Дате, потерянной дате, электронной почте клиента, стране клиента/государству/зипу, языку клиентов и т.д.

### <a name="customer-page-filters"></a>Фильтры страниц клиента

Фильтры **страницы Клиентов** применяются на уровне страницы Заказчиков. Можно выбрать несколько фильтров, чтобы отобразить диаграмму для критериев, которые вы хотите просмотреть, и данных, которые вы хотите видеть в сетке /экспорте данных подробных заказов'. Фильтры применяются на данных, извлеченных для диапазона данных, выбранных на верхнем правом углу страницы заказов.

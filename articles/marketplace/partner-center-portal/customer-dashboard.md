# <a name="customer-dashboard-in-commercial-marketplace-analytics"></a>Пользовательская панель мониторинга в коммерческой аналитике Marketplace

Эта статья содержит сведения о **панели мониторинга клиентов** в Центре партнеров. На этой панели мониторинга отображаются сведения о клиентах, включая тенденции роста, представленные в графическом и загружаемом формате.

Чтобы перейти к **панели мониторинга клиентов**, откройте панель мониторинга **[Анализ](https://partner.microsoft.com/dashboard/commercial-marketplace/analytics/summary)** в разделе "Коммерческий Marketplace".

>[!NOTE]
> Подробные определения терминов аналитики см. в статье [Часто задаваемые вопросы и терминология по аналитике коммерческой платформы](./faq-terminology.md).

## <a name="customer-dashboard"></a>Панель мониторинга клиентов

На **панели мониторинга клиентов** (меню **Анализ**) отображаются данные для клиентов, получивших ваши предложения. Можно просматривать следующие графические представления:

- [Сводка по клиенту](#customer-summary)
- [Клиенты по географическим регионам](#customer-by-geography)
- [Тенденции клиентов](#customer-trends)
- [Клиенты по заказам и использованию](#customers-by-orders-and-usage)
- [Клиенты по планам](#customers-by-plans)
- [Заказы и использование по типу клиента](#orders-and-usage-by-customer-type)
- [Таблица сведений о клиенте](#customer-details-table)
- [Фильтры на странице клиента](#customer-page-filters)

Максимальная задержка между получением клиентов и отчетами в центре партнеров составляет 48 часов.

### <a name="customer-summary"></a>Сводка по клиенту

В разделе "Сводка по клиенту" отображается количество всех клиентов, включая новых, существующих и ушедших, в течение выбранного диапазона дат.

- Общее количество клиентов определяется как количество всех клиентов, которые приобрели предложение и имеют, по крайней мере, один заказ, который не был отменен.
- Процент роста по сравнению с предыдущим месяцм определяется числом и перечисленностью, отображаемым красным или вертикальным индикатором красного цвета.
- Тенденции роста представлены гистограммами. Чтобы просмотреть значения для каждого месяца, наведите указатель мыши на столбцы диаграммы.

Существует три **типа клиентов**: новые, существующие и ушедшие.

- Новый клиент — это клиент, который приобрел одно или несколько предложений в первый раз в выбранном месяце.
- Существующий клиент — клиент, который приобрел одно или несколько предложений до выбранного месяца.
- Ушедший клиент — это клиент, отменивший все предложения, приобретенные ранее.

### <a name="customer-by-geography"></a>Клиенты по географическим регионам

На диаграмме **клиент по географии** отображается количество клиентов и клиентов, приобретенных в течение выбранного диапазона дат, которые сопоставляются по стране или региону клиента. Количество клиентов обозначается цветом. Чем больше клиентов, тем темнее цвет. Выберите запись в таблице, чтобы увеличить страну или регион.

На тепловой карте отображается количество клиентов и процент по стране или региону клиента. Карту можно переместить для просмотра точного расположения и увеличения масштаба в определенном месте. На этой карте имеется вспомогательная сетка, которая позволяет просматривать % клиентов по расположению, а также только что добавленных в это расположение.

### <a name="customer-trends"></a>Тенденции клиентов

На диаграмме **Тенденций клиентов** отображается количество всех клиентов, включая новых, существующих и ушедших, с тенденциями роста за месяц.

- На графике представлен общий процент роста числа клиентов.
- В столбце месяца указывается число клиентов с разбивкой на новых, существующих и ушедших.
- Число ушедших клиентов указывается в отрицательном направлении оси Y.
- Можно выбрать определенные элементы условных обозначений для отображения более подробных представлений. Например, в условных обозначениях выберите новых клиентов, чтобы отобразить только новых клиентов.
- С помощью ползунка в верхней части диаграммы можно выполнять прокрутку вправо и влево вдоль оси x и сосредоточиться на конкретных точках данных для просмотра более подробных сведений.
- При наведении указателя мыши на столбец диаграммы отображаются сведения только за выбранный месяц.

### <a name="customers-by-orders-and-usage"></a>Клиенты по заказам и использованию

На диаграмме **Клиенты по заказам и использованию** имеется три вкладки: "Заказы", "Нормализованное использование" и "Непосредственное использование". **Процентиль основного клиента** отображается вдоль оси X и определяется объемом количества его заказов. На оси Y отображается количество заказов клиента. На вспомогательной оси Y (график) отображается суммарный процент общего количества заказов. Вы можете отобразить сведения, наведя указатель мыши на точки вдоль графика.

В качестве примера см. приведенную ниже диаграмму для нормализованного использования: Первый 30-й процентиль клиентов составляют 87 % от нормализованного использования. 30-й процентиль клиентов составляет только 1 570 000 часов использования.

### <a name="customers-by-plans"></a>Клиенты по планам

Ниже описаны **клиенты по планам и** диаграммам использования.

1. В таблице лидеров представлены сведения о 50 самых активных заказчиках с сортировкой по количеству заказов. После выбора клиента сведения о нем отображаются в разделах 2, 3 и 4 этой таблицы.
2. Сведения о профиле клиента отображаются в этом пространстве, когда издатели входят в систему с ролью владельца. Если издатели вошли в систему с ролью участника, сведения в этом разделе будут недоступны.
3. На кольцевой диаграмме **ORDER BY Plans (заказы по планам** ) отображается разбивка заказов, приобретенных для планов. Отображаются пять первых планов с наибольшим числом заказов, а остальные — в разделе "все остальные".
4. На кольцевой диаграмме места **по планам** отображается распределение мест, упорядоченных по планам. Отображаются пять ведущих планов с наибольшим количеством рабочих мест, а остальные заказы группируются в разделе все остальные.

### <a name="orders-and-usage-by-customer-type"></a>Заказы и использование по типу клиента

На диаграмме **Заказы и использование по типу клиента** отображается число заказов, нормализованное использование и количество часов непосредственного использования по новым клиентам и существующим клиентам на основе выбора заказов, нормализованного и необработанного использования соответственно.

- Новый клиент — это клиент, который приобрел одно или несколько предложений в первый раз в одном и том же месяце (ось Y) или сообщил о первом использовании предложения.
- Существующий клиент — это клиент, который ранее получал предложение от вас или сообщил об использовании до того, как был сформирован отчет по календарному месяцу (на оси Y).

### <a name="customer-details-table"></a>Таблица сведений о клиенте

В таблице **Сведения о клиенте** представлен нумерованный список первых 1000 клиентов, отсортированных по дате первого приобретения одного из ваших предложений.

- Персональные данные клиента отображаются только в том случае, если клиент дал согласие на это. Вы можете просмотреть эти сведения, только если вошли с уровнем разрешений, соответствующим роли "Владелец". Узнайте подробнее о ролях пользователей и разрешениях.
- Каждый столбец в сетке доступен для сортировки.
- Данные можно извлечь в TSV-файл, если число записей меньше 1000.
- Если число записей превышает 1000, экспортированные данные будут асинхронно помещены на страницу загрузки в течение следующих 30 дней.
- Примените фильтры к таблице, чтобы отобразить только интересующие вас данные. Фильтрация данных по названию компании, ИДЕНТИФИКАТОРу клиента, ИДЕНТИФИКАТОРу подписки Marketplace, типу лицензии Azure, дате получения, дате потери, адресу электронной почты клиента, стране/региону клиента, штату, городу/ZIP, языку клиента и т. д.
- Когда предложение приобретено защищенным клиентом, информация в поле **Подробные сведения о клиенте** будет скрыта (************).
- Сведения об измерении клиента, такие как название компании, имя клиента и адрес электронной почты клиента, находятся на уровне идентификатора организации, а не на уровне транзакций Azure Marketplace или AppSource.

### <a name="customer-page-filters"></a>Фильтры на странице клиента

Фильтры на **странице клиента** применяются на уровне этой страницы. Можно выбрать несколько фильтров, чтобы отобразить диаграмму с выбранными критериями для просмотра и данными, которые должны отображаться в сетке/экспорте подробных сведений о заказах. Фильтры применяются к данным, извлеченным для диапазона данных, выбранного в правом верхнем углу страницы заказов.

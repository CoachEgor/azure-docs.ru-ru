# <a name="customer-dashboard-in-commercial-marketplace-analytics"></a>Панель мониторинга клиентов в аналитике коммерческого рынка

В этой статье содержатся сведения о **панели мониторинга клиентов** в центре партнеров. На этой панели мониторинга отображаются сведения о клиентах, включая тенденции роста, представленные в графическом и загружаемом формате.

Для доступа к **панели мониторинга клиента**откройте панель мониторинга **[анализ](https://partner.microsoft.com/dashboard/commercial-marketplace/analytics/summary)** в разделе коммерческая Marketplace.

>[!NOTE]
> Подробные определения терминологии аналитики см. в разделе [часто задаваемые вопросы и терминология для коммерческого анализа](./faq-terminology.md).

## <a name="customer-dashboard"></a>Панель мониторинга клиентов

На **панели мониторинга клиента** в меню **анализ** отображаются данные для клиентов, получивших ваши предложения. Можно просматривать графические представления следующих элементов:

- [Сводка клиента](#customer-summary)
- [Клиент по географическому](#customer-by-geography)
- [Тенденции клиентов](#customer-trends)
- [Клиенты по заказам и использованию](#customers-by-orders-and-usage)
- [Клиенты по номерам SKU](#customers-by-skus)
- [Заказы и использование по типу клиента](#orders-and-usage-by-customer-type)
- [Таблица сведений о клиенте](#customer-details-table)
- [Фильтры страниц клиента](#customer-page-filters)

### <a name="customer-summary"></a>Сводка клиента

В разделе Сводка клиента отображается количество всех клиентов, включая новые, существующие и обработанные, в течение выбранного диапазона дат.

- Общее количество клиентов определяется как количество всех клиентов, которые приобрели предложение и имеют по крайней мере один заказ, который не был отменен.
- Процент роста числа клиентов по сравнению с предыдущим месяцм определяется с помощью индикатора число и стрелка вверх в красном или вертикальном индикаторе красным цветом.
- Тенденции роста представлены линейчатыми диаграммами и отображают значения для каждого месяца путем наведения указателя мыши на столбцы диаграммы.

Существует три **типа клиентов**: новый, существующий и обработанный.

- Новый клиент приобрел одно или несколько предложений в первый раз в выбранном месяце.
- Существующий клиент приобрел одно или несколько предложений до выбранного месяца.
- Обработанный Клиент отменил все предложения, приобретенные ранее.

### <a name="customer-by-geography"></a>Клиент по географическому

На диаграмме **клиент по географии** отображаются сведения обо всех клиентах и клиентах, приобретенных в течение выбранного диапазона дат, и сопоставляются на основе страны клиента. Светло-светлый цвет на карте представляет самое низкое значение числа клиентов. Щелкните запись в таблице, чтобы увеличить масштаб страны.

Тепловой карты отображает число клиентов и% от страны клиента. Карту можно переместить для просмотра точного расположения и масштабирования в определенном месте. Эта схема содержит дополнительную сетку, которая позволяет просматривать% клиентов по расположению, а также только что добавленные в это место.

### <a name="customer-trends"></a>Тенденции клиентов

На диаграмме **тенденций клиентов** отображается количество всех клиентов, включая новые, существующие и обработанные, с тенденциями роста за месяц.

- График представляет общие проценты роста клиента.
- В столбце month (месяц) представлено число клиентов, разделенных на новые, существующие и обработанные заказчики.
- Число обработанных клиентов отображается в отрицательном направлении оси Y.
- Можно выбрать определенные элементы условных обозначений для отображения более подробных представлений. Например, в условных обозначениях выберите новые клиенты, чтобы отобразить только новых клиентов.
- Можно использовать ползунок в верхней части диаграммы для прокрутки вправо и влево по оси x и сосредоточиться на конкретных точках данных для более подробного представления.
- При наведении указателя мыши на столбец диаграммы отображаются сведения только за этот месяц.

### <a name="customers-by-orders-and-usage"></a>Клиенты по заказам и использованию

Диаграмма **клиенты по заказам и использованию** имеет три вкладки: «заказы», «нормализованное использование» и «необработанное использование». **Верхний процент клиента** отображается вдоль оси x, как определено по количеству заказов. На оси y отображается количество заказов клиента. На вспомогательной оси y (график) отображается суммарный процент общего количества заказов. Вы можете отобразить подробные сведения, наведя указатель мыши на точки вдоль графика.

В качестве примера см. приведенную ниже диаграмму для нормализованного использования: первые 30% клиентов участвуют в 87% общего использования. 30 – й процентиль клиентов составляет только 1 570 000 часов использования.

### <a name="customers-by-skus"></a>Клиенты по номерам SKU

Сведения о **клиентах по SKU и** диаграммам использования описаны ниже.

1. На лидерской доске представлены сведения о первых 50 заказчиках, ранжированных по количеству заказов. После выбора клиента сведения о клиенте представлены в разделах 2, 3 и 4 этой доски заполнения.
2. Сведения о профиле клиента отображаются в этом пространстве, когда издатели входят в систему с ролью владельца. Если издатели вошли в систему с ролью участника, сведения в этом разделе будут недоступны.
3. На кольцевой диаграмме заказы по номерам SKU отображается разбивка заказов, приобретенных для номеров SKU. Отображаются 5 основных номеров SKU с наибольшим количеством заказов, а остальные — в разделе "все остальные".
4. На кольцевой диаграмме «места по номерам SKU» отображается распределение мест, заказанных по номерам SKU. Отображаются пять основных номеров SKU с самыми высокими местами, а остальные заказы группируются в разделе все остальные.

### <a name="orders-and-usage-by-customer-type"></a>Заказы и использование по типу клиента

Диаграмма " **заказы/использование по типам клиентов** " отображает число заказов, нормализованное использование и количество часов необработанного использования, разделенных между новыми клиентами и существующими клиентами. на основе выбора заказов, нормализованного и необработанного использования соответственно в диаграмме.

- Новый клиент приобрел одно или несколько предложений или сообщил об использовании в первый раз в том же календарном месяце (ось y).
- Существующий клиент ранее получал предложение от вас или сообщил об использовании до того, как отчетный месяц выводится (на оси y).

### <a name="customer-details-table"></a>Таблица сведений о клиенте

В таблице " **сведения о клиенте** " отображается нумерованный список лучших 1000 клиентов, отсортированных по дате первого приобретения одного из ваших предложений.

- Персональные данные клиента будут доступны только в том случае, если клиент предоставил согласие. Эти сведения можно просмотреть только в том случае, если вы выполнили вход с правами владельца на уровне роли. Дополнительные сведения о ролях и разрешениях пользователей.
- Каждый столбец в сетке доступен для сортировки.
- Данные можно извлечь в TSV-файл, если число записей меньше 1000.
- Если номер записи превышает 1000, экспортированные данные будут асинхронно помещены на страницу загрузки в течение следующих 30 дней.
- К таблице можно применить фильтры для вывода только интересующих вас данных. Данные могут быть отфильтрованы по названию компании, ИДЕНТИФИКАТОРу клиента, ИДЕНТИФИКАТОРу подписки Marketplace, типу лицензии Azure, дате получения, дате потери, электронной почте клиента, стране клиента/штату/городу/ZIP, языку клиента и т. д.

### <a name="customer-page-filters"></a>Фильтры страниц клиента

Фильтры **страниц клиентов** применяются на уровне страницы клиентов. Можно выбрать несколько фильтров, чтобы отобразить диаграмму с выбранными критериями, а также данные, которые необходимо просмотреть в сетке или экспорте подробных данных о заказах. Фильтры применяются к данным, извлеченным для диапазона данных, выбранного в правом верхнем углу страницы «заказы».

>[!NOTE]

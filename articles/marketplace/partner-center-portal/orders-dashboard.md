# <a name="orders-dashboard-in-commercial-marketplace-analytics"></a>Панель мониторинга заказов в коммерческой аналитике Marketplace

Эта статья содержит сведения о **панели мониторинга "Заказы"** в Центре партнеров. Эта панель мониторинга отображает сведения о заказах в графическом и загружаемом формате.

Чтобы открыть **панель мониторинга заказов** в средствах аналитики Центра партнеров, откройте **[панель мониторинга анализа](https://partner.microsoft.com/dashboard/commercial-marketplace/analytics/summary)** на коммерческой платформе.

>[!NOTE]
> Подробные определения терминов аналитики см. в статье [Часто задаваемые вопросы и терминология по аналитике коммерческой платформы](./faq-terminology.md).

## <a name="orders-dashboard"></a>Панель мониторинга заказов

**Панель мониторинга заказов** в меню **Анализ** отображает текущие заказы для всех предложений SaaS. Можно просматривать следующие графические представления:

- [Сводка заказов](#order-summary)
- [Заказы по географическим регионам](#orders-by-geography)
- [Заказы по предложениям](#orders-by-offers)
- [Тенденция числа заказов на сайт и на рабочее место](#orders-trend-per-site-versus-per-seat)
- [Заказы по планам](#orders-by-plans)
- [Тенденция заказов и рабочих мест](#orders-and-seats-trend)
- [Таблица сведений о заказах](#order-details-table)

Максимальная задержка между созданием заказов и отчетами в центре партнеров составляет 48 часов.

## <a name="order-dashboard-details"></a>Сведения о панели заказов

В этом разделе более подробно описаны аналитические отчеты.

### <a name="order-summary"></a>Сводка заказов

В разделе "Сводка заказов" отображается количество всех приобретенных заказов (за исключением отмененных заказов), отмененных заказов и рабочих мест.

Процентное значение рядом с пунктом "Общее количество заказов" представляет величину роста в выбранном диапазоне дат.

![Сводка анализа заказов в Центре партнеров](./media/order-summary.png)

- Зеленый треугольник, указывающий вверх, указывает на положительную тенденцию роста.
- Красный треугольник, указывающий вниз, обозначает отрицательную тенденцию роста относительно предыдущего месяца.
- Тенденции роста представлены в виде миниатюрных линейных диаграмм. Если навести указатель мыши на столбцы этой диаграммы, отображается значение для каждого месяца.
- Отмененные заказы — это количество заказов, которые были ранее приобретены, а затем отменены в течение выбранного диапазона дат.
- Рабочие места — это количество рабочих мест, созданных в течение выбранного диапазона дат.

### <a name="orders-by-geography"></a>Заказы по географическим регионам

На карте " **заказы по географическому** расположению" отображается количество заказов на схеме мира и отображаются места, сопоставленные с учетом страны или региона клиента. Эта тепловая карта функционирует так же, как **[Тепловая карта клиентов по географическому положению](./customer-dashboard.md#customer-by-geography)** .

![Анализ заказов в Центре партнеров по географическому положению](./media/orders-by-geography.png)

### <a name="orders-by-offers"></a>Заказы по предложениям

Кольцевая диаграмма **Заказы по предложениям** упорядочивает заказы (включая отмененные заказы) в соответствии с названиями их предложений.

- Основные предложения отображаются на диаграмме, а остальные предложения группируются как "Все остальные".
- Вы можете выбрать определенные предложения в условных обозначениях, чтобы отобразить только эти предложения на диаграмме.
- При наведении указателя мыши на срез на диаграмме отображается количество заказов и процент этого предложения по сравнению с общим количеством заказов по всем предложениям.
- **Тенденция заказов по предложениям** отображает тенденции роста по месяцам. Столбец "месяц" представляет число заказов по названию предложения. На линейном графике отображается тенденция роста в процентах, построенная по оси z.
- С помощью ползунка в верхней части диаграммы можно выполнять прокрутку вправо и влево вдоль оси x и сосредоточиться на конкретных точках данных.
- Диаграмму тренда можно отобразить, выбрав конкретный элемент в условных обозначениях.
- Можно также выбрать отображение тенденций и данных для **отмененных заказов**. График будет работать точно так же, как график **заказы по предложениям**.

### <a name="orders-trend-per-site-versus-per-seat"></a>Тенденция числа заказов на сайт и на рабочее место

**Односайтовая и одноместная** кольцевая диаграмма представляет декомпозицию SaaS по каждому сайту и заказов на SaaS на уровне отдельных рабочих мест, приобретенных клиентами (Эта диаграмма включает отмененные заказы). Гистограмма представляет тренд числа заказов SaaS на сайт и на рабочее место, приобретенных клиентами (эта диаграмма включает отмененные заказы).

### <a name="orders-by-plans"></a>Заказы по планам

Диаграмма **заказы по планам** представляет тенденцию заказов на уровне плана для всех ваших предложений (включая отмененные заказы). Кольцевая диаграмма представляет собой разбивку пяти заказов на план, а гистограмма представляет тенденцию заказов для пяти ведущих планов.

### <a name="orders-and-seats-trend"></a>Тенденция заказов и рабочих мест

В диаграмме **тенденции заказов и рабочих мест** представлены первые 50 предложений с наибольшим количеством заказов. Они отображаются на доске заполнения и сортируются по максимальному количеству заказов и процентам заказов.

- **Заказы по планам** . Выберите предложение, чтобы просмотреть разбивку количества заказов для пяти ведущих планов в диаграмме.
- Рабочие места **по планам** — месячный тренд рабочих мест для пяти ведущих планов. Если выбранное предложение не является предложением для каждого места, данные здесь отображаться не будут.

### <a name="canceled-orders-by-offers"></a>Отмененные заказы по предложениям

Круговая диаграмма **Отмененные заказы по предложениям** организует все отмененные заказы согласно их названиям. Основные предложения отображаются на диаграмме, а остальные предложения группируются как "Все остальные". Вы можете выбрать определенные предложения в условных обозначениях, чтобы отобразить их на диаграмме.

- При наведении указателя мыши на срез на диаграмме отображается число заказов и процент выбранного предложения по сравнению с общим количеством заказов по всем предложениям.
- На гистограмме отображаются тенденции по месяцам. Столбцы представляют число отмененных заказов по имени предложения. С помощью ползунка в верхней части диаграммы можно выполнять прокрутку вправо и влево вдоль оси x и сосредоточиться на конкретных точках данных. Диаграмму тренда можно отобразить, выбрав конкретный элемент в условных обозначениях.

### <a name="order-details-table"></a>Таблица сведений о заказах

В таблице "Сведения о заказах" представлен нумерованный список 1000 основных заказов, отсортированных по дате приобретения.

- Каждый столбец в сетке доступен для сортировки.
- Данные можно извлечь в TSV-файл, если число записей меньше 1000.
- Если число записей превышает 1000, экспортированные данные будут асинхронно помещены на страницу загрузки в течение следующих 30 дней.
- Примените фильтры к **таблице Order Details** , чтобы отобразить только интересующие вас данные. Фильтрация по стране/региону, типу лицензии Azure, типу лицензии коммерческого магазина, типу предложения, статусу заказа, свободному следу, ИДЕНТИФИКАТОРу подписки коммерческого магазина, идентификатору клиента и названию компании.
- Так как предложения SaaS приобретены в Azure Marketplace или AppSource, не требуется подписка Azure, идентификатор подписки Marketplace будет отображаться как 00000000-0000-0000-0000-000000000000 в разделе **подробных данных о заказах** .

#### <a name="orders-page-filters"></a>Фильтры страниц заказов

Эти фильтры применяются на уровне страницы.

Можно выбрать несколько фильтров, чтобы отобразить диаграмму с выбранными критериями для просмотра и данными, которые должны отображаться в сетке/экспорте **Подробных данных заказа**. Фильтры применяются к данным, извлеченным для диапазона данных, выбранного в правом верхнем углу страницы «заказы».

- Типы предложений и названия предложений указаны только для предложений, для которых в выбранном диапазоне дат есть заказы. Имена предложений в списке отображаются для типов предложений, выбранных в списке.
- Примененные фильтры показывают общее количество метрик в каждой выборке для каждого выбранного фильтра. Примененные фильтры не отображаются при выборе по умолчанию.
- Если для одного из раскрывающихся списков выбрано **Все**, то будут агрегированы все метрики на выбранной странице. Пример: Значение "Все" в параметре фильтра "Типы предложений" означает, что выбраны все типы предложений. Это выбор по умолчанию для раскрывающихся списков. Отображение примененных фильтров будет пустым, если выбрано **Все**.
- **Выбор нескольких значений**. Все метрики на странице будут агрегированы для всех параметров, выбранных в раскрывающемся списке. Если выбрано несколько, в примененном фильтре будет показано общее количество выбранных элементов. Дополнительные сведения см. на рисунке ниже.

    ![Анализ заказов в Центре партнеров с несколькими значениями, примененными к фильтру](./media/filters-applied.png)

- **Выбор одного значения**. Если выбрано одно значение, то в примененном фильтре будет показано количество выбранных фильтров. См. ниже изображение для справки.

     ![Анализ заказов в Центре партнеров с одним значением, примененным к фильтру](./media/filters-applied-single.png)

# <a name="orders-dashboard-in-commercial-marketplace-analytics"></a>Панель мониторинга заказов в аналитике коммерческого рынка

В этой статье содержится информация о **панели мониторинга заказов** в Центре Партнеров. Эта панель мониторинга отображает информацию о ваших заказах в графическом и загружаемом формате.

Чтобы получить доступ к **панели мониторинга заказов** в инструментах аналитики Partner Center, откройте **[панель мониторинга Analyse](https://partner.microsoft.com/dashboard/commercial-marketplace/analytics/summary)** под коммерческим рынком.

>[!NOTE]
> Для подробных определений терминологии аналитики, [см. Часто задаваемые вопросы и терминологию для коммерческой аналитики рынка](./faq-terminology.md).

## <a name="orders-dashboard"></a>Панель мониторинга заказов

Панель **мониторинга заказов** меню **Analyse** отображает текущие заказы для всех ваших предложений SaaS. Вы можете просмотреть графические представления следующих элементов:

- [Сводка заказов](#order-summary)
- [Заказы по географии](#orders-by-geography)
- [Заказы по предложениям](#orders-by-offers)
- [Тенденция заказов на сайт по сравнению с местом](#orders-trend-per-site-versus-per-seat)
- [Заказы СКУ](#orders-by-skus)
- [Тенденция заказов и мест](#orders-and-seats-trend)
- [Таблица деталей заказа](#order-details-table)

> [!NOTE]
> Существуют различия между тем, как аналитические отчеты отображаются в облачном портале партнеров (CPP) и новой программой коммерческого рынка в Partner Center. Одним из конкретных способов является то, что **Seller Insights** в CPP имеет вкладку **Заказы & использование,** которая отображает данные для предложений на основе использования и неиспользования на основе предложений. В Партнерс-центре страница **Заказов** имеет отдельную вкладку для предложений SaaS.

## <a name="order-dashboard-details"></a>Детали панели заказа

В этом разделе более подробно описаны аналитические отчеты.

### <a name="order-summary"></a>Описание Резюме

В разделе "Резюме заказов" отображается подсчет всех приобретенных заказов (за исключением отмененных заказов), отмененных заказов и мест.

Процентное значение рядом с общими ордерами представляет собой объем роста выбранного диапазона дат.

![Сводка заказа центра партнера анализирует](./media/order-summary.png)

- Зеленый треугольник, указывающий вверх, указывает на положительную тенденцию роста.
- Красный треугольник, указывающий вниз, указывает на отрицательную тенденцию роста по отношению к предыдущему месяцу.
- Тенденции роста представлены микро-бар графиками. Значение можно отображать за каждый месяц, паря над столбцовами в диаграмме.
- Отмененные заказы — это подсчет заказов, которые были ранее приобретены и затем отменены во время выбранного диапазона дат.
- Места - это количество мест, созданных во время выбранного диапазона дат.

### <a name="orders-by-geography"></a>Заказы по географии

**Заказы по географии** тепловой карты отображает подсчет ваших заказов на карте мира и отображает места, отображаемые на основе страны клиента. Эта тепловая карта функционирует так же, как **[клиент по географии тепловой карты](./customer-dashboard.md#customer-by-geography)**.

![Партнерский центр анализирует заказы по географии](./media/orders-by-geography.png)

### <a name="orders-by-offers"></a>Заказы по предложениям

**Заказы по предложениям** пончик граф организует заказы (в том числе отмененных заказов) в соответствии с их названия миноносцев.

- Лучшие предложения отображаются на графике, а остальные предложения сгруппированы как 'Rest All'.
- Вы можете выбрать конкретные предложения в легенде, чтобы отображать только те предложения в графике.
- На графике будет отображаться количество ордеров и процент от этого предложения по сравнению с общим количеством заказов во всех предложениях.
- **Заказы по предложениям тренда** отображают месячные тенденции роста. Столбец месяца представляет количество ордеров по названию предложения. Диаграмма линии отображает тренд процентного роста, начертанный на оси z.
- Слайдер можно использовать в верхней части диаграммы для прокрутки правой и левой вдоль оси х и сосредоточиться на конкретных точках данных.
- Вы можете отобразить диаграмму тренда, выбрав конкретный элемент на легенде.
- Вы также можете выбрать для отображения тенденций и данных для **отмененных заказов.** График будет функционировать так же, как **и заказы по предложениям** графика.

### <a name="orders-trend-per-site-versus-per-seat"></a>Заказы Тренд на сайт по сравнению с местом

Диаграмма **на один сайт по сравнению с** наодного пончика представляет собой разбивку на сайте SaaS и на одно место SaaS заказов, приобретенных клиентами (эта диаграмма включает в себя отмененные заказы). Диаграмма столбцов представляет тенденцию на сайте SaaS и на одно место SaaS заказов, приобретенных клиентами (эта диаграмма включает в себя отмененные заказы).

### <a name="orders-by-skus"></a>Заказы СКУ

График **заказов по SKUs** представляет тенденцию заказов на уровне Stock Keeping Unit (SKU) для всех ваших предложений (это включает отмененные ордера). Диаграмма пончика представляет собой разбивку пяти лучших ордеров SKU, а диаграмма столбцов представляет тренд заказов для пяти крупнейших SKUs.

### <a name="orders-and-seats-trend"></a>Тенденции заказов и мест

Диаграмма **тренда заказов и мест** представляет топ-50 предложений с наибольшим количеством заказов. Они отображаются на доске лидеров и сортируются по наивысшему проценту заказов и заказов.

- **Заказы от SKUs**: Выберите предложение для просмотра разбивки количества заказов для пяти крупнейших SkUS на графике.
- **Места от SKUs**: Ежемесячный тренд мест для пяти лучших СКУ. Если выбранное вами предложение не является предложением на одно место, вы не увидите никаких данных на этой диаграмме области.

### <a name="canceled-orders-by-offers"></a>Отмененные заказы по предложениям

**Отмененные заказы по предложениям** круговой диаграммы организует все отмененные заказы в соответствии с их именами предложений. Лучшие предложения отображаются на графике, а остальные предложения сгруппированы как "Rest All". Вы можете выбрать конкретные предложения в легенде для отображения на графике.

- На графике будет отображаться количество ордеров и процент выбранного предложения по сравнению с общим количеством заказов во всех предложениях.
- Диаграмма столбцов отображает месячные тенденции. Столбцы представляют количество отмененных ордеров по имени предложения. Слайдер можно использовать в верхней части диаграммы для прокрутки правой и левой вдоль оси х и сосредоточиться на конкретных точках данных. Вы можете отобразить диаграмму тренда, выбрав конкретный элемент на легенде.

### <a name="order-details-table"></a>Таблица деталей заказа

В таблице с подробным описанием Заказа отображается номерной список 1000 лучших заказов, отсортированных по дате приобретения.

- Каждая колонка в сетке сортируется.
- Данные могут быть извлечены в файл TSV, если количество записей меньше 1000.
- Если число записей превышает 1000, экспортированные данные будут асинхронно размещены на странице загрузки в течение следующих 30 дней.
- Фильтры могут быть применены к **таблице деталей Заказа** для отображения только интересуемых вами данных. Данные могут быть отфильтрованы по странам, типу лицензии Azure, типу лицензии Marketplace, типу предложения, статусу заказа, бесплатным трассам, идентификатору подписки Marketplace, идентификатору клиента и названию компании.

#### <a name="orders-page-filters"></a>Заказы фильтров Страницы

Эти фильтры применяются на уровне страницы.

Можно выбрать несколько фильтров для визуализации диаграммы для просмотра критериев и данных, которые вы хотите отобразить в сетке/экспорте **данных детального заказа.** Фильтры применяются на данных, извлеченных для диапазона данных, выбранных в правом верхнем углу страницы заказов.

- Типы предложений и наименований предложений перечислены только для предложений, на которые у вас есть заказы в выбранном диапазоне дат. Имена предложений в списке отображаются для выбранных вами типов предложений в списке.
- Прикладные фильтры отображаются общие метрики в каждом выбранном выбранном фильтре(ы) для каждого выбранного фильтра. При выборе выбора прикладных фильтрах не отображаются привыборе по умолчанию.
- Если **все** выбрано для одного из списков выпадающих, то все метрики на выбранной странице будут агрегированы. Например: опция фильтра типов предложений означает, что все типы предложений были выбраны. Это выбор по умолчанию для списков выпадающих. При выбранных фильтров дисплеи **All** ничего не отображают.
- **Выбор нескольких значений**: Все метрики на странице будут агрегированы для всех выборов, сделанных в списке выпадающих данных. При нескольких выделениях применяемый фильтр покажет количество всех сделанных выделений. Смотрите изображение ниже для справки.

    ![Заказ Partner Center анализирует сятвоие с несколькими значениями, применяемыми для фильтрации](./media/filters-applied.png)

- **Выбор одного значения:** При выборе одного значения применяемый фильтр будет отображать количество выбранного фильтра. Смотрите ниже изображение для справки.

     ![Заказ Partner Center анализирует заказ с единым значением, применяемым к фильтрации](./media/filters-applied-single.png)

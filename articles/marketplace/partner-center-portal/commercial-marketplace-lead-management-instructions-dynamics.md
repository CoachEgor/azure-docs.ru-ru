# <a name="configure-lead-management-for-dynamics-365-for-customer-engagement"></a>Настройка руководства для Dynamics 365 для привлечения клиентов

В этой статье описывается, как настроить Dynamics 365 для взаимодействия с клиентами (ранее Dynamics CRM Online), подробнее об изменении [здесь](https://docs.microsoft.com/dynamics365/customerengagement/on-premises/admin/on-prem-server-based-sharepoint-online) для обработки продаж приводит от вашего предложения рынка. 

>[!Note]
>Эти инструкции специфичны для облачной динамики Microsoft Dynamics 365 для среды взаимодействия с клиентами. Подключение непосредственно к среде Dynamics on-prem в настоящее время не поддерживается, есть другие варианты для получения интересов, таких как настройка [конечная точка https](./commercial-marketplace-lead-management-instructions-https.md) или [таблица Azure](./commercial-marketplace-lead-management-instructions-azure-table.md) для получения интересов.

## <a name="prerequisites"></a>Предварительные требования

Для выполнения шагов в этой статье необходимы следующие разрешения пользователя:

* Вы должны быть админ на динамике 365 для обслуживания клиентов экземпляр, чтобы иметь возможность установить решение и следовать этим инструкциям.
* Для создания новой учетной записи службы для службы, используемой для отправки клиентов из предложений на рынке, необходимо создать новую учетную запись службы обслуживания, используемую для отправки клиентов.
* Вы должны иметь доступ к порталу Office 365 admin.
* Вам необходимо иметь доступ к порталу Azure.

## <a name="install-the-solution"></a>Установка решения

1.  Загрузите [решение Microsoft Marketplace Lead Writer](https://mpsapiprodwus.blob.core.windows.net/documentation/MicrosoftMarketplacesLeadIntegrationSolution_1_0_0_0_target_CRM_6.1_managed.zip) и сохраните его локально на компьютере.

2.  Open Dynamics 365 для привлечения клиентов, перемещаясь по URL-адресу для экземпляра Dynamics `https://tenant.crm.dynamics.com`(например, ).

3.  Настройки доступа, выбрав значок передач и **расширенные настройки** на верхней панели навигации.
 
    ![Динамика - Расширенные настройки](./media/commercial-marketplace-lead-management-instructions-dynamics/dynamics-advanced-settings.png)

4.  После того, как на странице Настройки, доступ к меню настройки из верхней панели навигации и выберите **решения**.

    >[!Note]
    >Если вы не видите параметры в следующем захвате экрана, то у вас нет разрешений, необходимых для продолжения работы. Обратитесь к админ-менеджеру в динамической динамике 365 для примера взаимодействия с клиентами.

    ![Динамика 365 - Решения](./media/commercial-marketplace-lead-management-instructions-dynamics/dynamics-solutions.png)

5. Оказавшись на странице Решения, выберите **«Импорт»** и перейдите туда, где вы сохранили решение *Microsoft Marketplace Lead Writer,* которое вы загрузили в шаге 1.

    ![Динамика 365 для привлечения клиентов - Импорт](./media/commercial-marketplace-lead-management-instructions-dynamics/dynamics-crm-import.png)

6. Полный импорт решения, следуя мастеру решения Импорта.

## <a name="configure-user-permissions"></a>Настройка разрешений пользователя

Чтобы написать интересы в свой экземпляр Dynamics 365 для обслуживания клиентов, необходимо поделиться с нами учетной записью службы и настроить разрешения на учетную запись.

Чтобы создать учетную запись службы и назначить ей разрешения, выполните описанные ниже действия. Можно использовать **Azure Active Directory** или **Office 365**.

>[!Note]
>На основе выбранной вами опции аутентификации вы можете перейти к соответствующим инструкциям, основанным на вашем выборе. Смотрите [Активный каталог Azure](https://docs.microsoft.com/azure/marketplace/partner-center-portal/commercial-marketplace-lead-management-instructions-dynamics#azure-active-directory) или [Office 365](https://docs.microsoft.com/azure/marketplace/partner-center-portal/commercial-marketplace-lead-management-instructions-dynamics#office-365).

### <a name="azure-active-directory"></a>Azure Active Directory

Мы рекомендуем эту опцию, потому что вы получите дополнительное преимущество никогда не нужно обновлять имя пользователя / пароль, чтобы продолжать получать приводит. Чтобы использовать опцию Active Directory Azure, вы предоставляете идентификатор приложения, ключ приложения и идентификатор каталога из приложения Active Directory.

Используйте следующие шаги для настройки Активного каталога Azure для Dynamics 365 для взаимодействия с клиентами.

1. Вопийте на [портале Azure,](https://portal.azure.com/)а затем выберите службу active Directory Azure из левой навигации.

2. Выберите **свойства** из оставленного навигации Azure Active Directory и скопируйте значение **идентификатора каталога** на этой странице. Сохраните это значение, так как это значение *идентификатора каталога,* которое необходимо предоставить на портале публикации для получения интересов для вашего предложения на рынке.

    ![Активный каталог Azure - Свойства](./media/commercial-marketplace-lead-management-instructions-dynamics/aad-properties.png)

3. Выберите **регистрацию приложений** из каталога Azure Active Directory левой навигации, а затем выберите **новую регистрацию** на этой странице.
4. Введите имя приложения. Укажите значимое имя приложения.
5. Под типами поддерживаемых учетных записей выберите **учетные записи в любом каталоге организационных элементов.**
6. Под перенаправлением URI выберите **Веб** и `https://contosoapp1/auth`предоставьте URI (например,). 
7. Выберите **Зарегистрировать**.

    ![Регистрация приложения](./media/commercial-marketplace-lead-management-instructions-dynamics/register-an-application.png)

8. Теперь, когда ваше приложение зарегистрировано, получите доступ к странице обзора приложения и скопируйте значение **идентификатора приложения (клиента)** на этой странице. Сохраните это значение, так как это значение *идентификатора приложения (клиента),* которое необходимо предоставить на портале публикации и в Dynamics для получения интересов для вашего предложения на рынке.

    ![Идентификатор приложения (клиента)](./media/commercial-marketplace-lead-management-instructions-dynamics/application-id.png)

9. Выберите **Сертификаты и секреты** из левой навигации приложения и выберите **новый секрет клиента** на этой странице. Введите значимое описание для секрета клиента и выберите опцию **Never** под expires. Выберите **Добавить,** чтобы создать секрет клиента.

    ![Приложение - Сертификация и секреты](./media/commercial-marketplace-lead-management-instructions-dynamics/aad-certificates-secrets.png)

10. Как только секрет клиента успешно создан, **Копировать клиента секретное значение**. Вы не сможете получить значение после навигации по странице. Сохраните это значение, так как это секретное значение *клиента,* которую необходимо предоставить на портале публикации для получения интересов для вашего предложения на рынке. 
11. Выберите **разрешения API** из левой навигации приложений, а затем выберите **Добавление разрешения.**
12. Выберите API Microsoft, а затем выберите **Dynamics CRM** в качестве API.
13. В соответствии с *типами разрешений требуется ваше приложение,* убедитесь, что **делегированные разрешения выбраны.** Проверьте разрешение на **user_impersonation** *Службы доступа к общим данным в качестве пользователей организации.* Выберите **Добавить разрешения**.

    ![Добавление разрешений](./media/commercial-marketplace-lead-management-instructions-dynamics/api-permissions.png)

14. После выполнения шагов 1-13 на портале Azure перейдите на динамические 365 для экземпляра `https://tenant.crm.dynamics.com`взаимодействия с клиентами, перейдя на URL (например, ).
15. Настройки доступа, выбрав значок передачи и **расширенные настройки** на верхней панели навигации.
16. После того, как на странице Настройки, доступ к меню настройки из верхней панели навигации и выберите **безопасности.**
17. Оказавшись на странице безопасности, выберите **пользователей.**  На странице пользователей выберите отсев "Включенные пользователи", чтобы переключиться на **пользователей приложений.**
18. Нажмите **Создать**, чтобы создать пользователя. 

    ![Создание пользователя](./media/commercial-marketplace-lead-management-instructions-dynamics/application-users.png)

19. В **новом пользователе**убедитесь, что USER: APPLICATION USER выбран. Предоставьте пользователю имя пользователя, полное имя и адрес электронной почты пользователю, который вы хотите использовать с этим соединением. Кроме того, вставьте в **идентификатор приложения** для приложения, созданного на портале Azure, с 8-го шага. Выберите **Сохранить и закрыть** для полного добавления пользователя.

    ![Новый пользователь](./media/commercial-marketplace-lead-management-instructions-dynamics/new-user-info.png)

20. Перейдите в раздел "Параметры безопасности" этой статьи и завершите настройку подключения для этого пользователя.

### <a name="office-365"></a>Office 365

Если вы не хотите использовать Active Directory Azure, вы можете зарегистрировать нового пользователя в *центре админы Microsoft 365.* Вам нужно будет обновлять свое имя пользователя и пароль каждые 90 дней, чтобы продолжать получать данные о потенциальных клиентах.

Используйте следующие шаги для настройки Office 365 для Dynamics 365 для взаимодействия с клиентами.

1. Выполните вход в [центр администрирования Microsoft 365](https://admin.microsoft.com).

2. Нажмите **Добавить пользователя**.

    ![Microsoft 365 Центр админ - добавить пользователя](./media/commercial-marketplace-lead-management-instructions-dynamics/ms-365-add-user.png)

4. Создайте пользователя для службы модуля записи интересов. Настройте следующие параметры:

    * Укажите имя пользователя
    * Предоставьте пароль и отменить опцию "Заставьте этого пользователя изменить пароль при первом входе в систему".
    * Укажите для пользователя роль "Пользователь (без прав администратора)".
    * Выберите "Dynamics 365 План взаимодействия с клиентами" в качестве лицензии продукта, показанной в следующем захвате экрана. За выбранную лицензию будет начисляться плата. 

Сохраните эти значения, поскольку они являются значениями *имени пользователя и пароля,* которые необходимо предоставить на портале публикации для получения интересов для вашего предложения на рынке.

![Microsoft 365 Центр админ - новый пользователь](./media/commercial-marketplace-lead-management-instructions-dynamics/ms-365-new-user.png)

## <a name="security-settings"></a>Параметры безопасности

На последнем шаге нужно разрешить созданному пользователю записывать данные о потенциальных клиентах.

1. Open Dynamics 365 для привлечения клиентов, перемещаясь по URL-адресу для экземпляра Dynamics `https://tenant.crm.dynamics.com`(например, ).
2. Настройки доступа, выбрав значок передач и **расширенные настройки** на верхней панели навигации.
3. После того, как на странице Настройки, доступ к меню настройки из верхней панели навигации и выберите **безопасности.**
4. После того, как на странице безопасности, выберите **пользователей** и выберите пользователя, который вы создали в разделе Разрешения пользователя этого документа, а затем выберите **Управление ролями.** 

    ![Управление ролями](./media/commercial-marketplace-lead-management-instructions-dynamics/security-manage-roles.png)

5. Поиск ролевой имени "Microsoft Marketplace Ведущий писатель" и выберите его, чтобы назначить пользователю роль.

    ![Создание ролей пользователей](./media/commercial-marketplace-lead-management-instructions-dynamics/security-manage-user-roles.png)

    >[!Note]
    >Эту роль создало импортированное решение, и ей предоставлены разрешения только на запись данных о потенциальных клиентах и на отслеживание версии решения для обеспечения совместимости.

6. Перейдите на страницу безопасности и выберите **роли безопасности.** Поиск роли "Ведущий писатель microsoft Marketplace" и выберите ее.

    ![Роли безопасности](./media/commercial-marketplace-lead-management-instructions-dynamics/security-roles.png)

7. После того, как в роли безопасности, выберите **вкладку Core Records.** Поиск "Настройки пользовательского интерфейса" и включить создание, чтение и запись разрешений для пользователя (1/4 желтый круг) для этой сущности, нажав один раз в каждый из соответствующих кругов.

    ![Ведущий автор microsoft Marketplace - Основные отчеты](./media/commercial-marketplace-lead-management-instructions-dynamics/marketplace-lead-writer.png)

8. Теперь перейдите на вкладку **Настройка.** Поиск tor "Система Работа" лица и позволяет читать, писать и AppendTo разрешений организации (твердый зеленый) для этой организации, нажав четыре раза в каждом из соответствующих кругов.

    ![Ведущий автор Microsoft Marketplace - настройка](./media/commercial-marketplace-lead-management-instructions-dynamics/marketplace-lead-writer-customization.png)

9. **Сохранить и закрыть**.

## <a name="configure-your-offer-to-send-leads-to-dynamics-365-for-customer-engagement"></a>Налажить ваше предложение отправить приводит к динамике 365 Для привлечения клиентов

Когда вы будете готовы настроить информацию о руководстве для вашего предложения на портале публикации, выполните ниже:

1. Перейдите на страницу **настройки предложения** для вашего предложения.
2. Выберите **Подключение** в разделе «Управление ведущим иссяка».

    ![Подключение к руководству свинцом](./media/commercial-marketplace-lead-management-instructions-dynamics/connect-lead-management.png)

3. На всплывающем окне подключения выберите **Dynamics 365 для привлечения клиентов** для ведущего назначения

    ![Детали подключения - направление свинца](./media/commercial-marketplace-lead-management-instructions-dynamics/connection-details-lead-destination.png)

4. Предоставьте **динамические 365 Instance URL,** такие как `https://contoso.crm4.dynamics.com`.
5. Выберите метод **аутентификации,** активный каталог Azure или Office 365. 
6. Если вы выбрали Active Directory Azure, предоставьте **идентификатор приложения (клиент):),** `23456052-aaaa-bbbb-8662-1234df56788f` **идентификатор каталога** (пример:) `12345678-8af1-4asf-1234-12234d01db47`и **секрет клиента** (пример: `1234ABCDEDFRZ/G/FdY0aUABCEDcqhbLn/ST122345nBc=`).

    ![Детали подключения - Активный каталог Azure](./media/commercial-marketplace-lead-management-instructions-dynamics/connection-details-application-id.png)

7. Если вы выбрали Office 365, укажите **имя пользователя** (пример: `contoso@contoso.onmicrosoft.com`) и пароль (пример: `P@ssw0rd`).

    ![Детали подключения - имя пользователя](./media/commercial-marketplace-lead-management-instructions-dynamics/connection-details-authentication.png)

>[!Note]
>Вы должны закончить настройку остальной части предложения и опубликовать его, прежде чем вы сможете получить приводит для предложения.

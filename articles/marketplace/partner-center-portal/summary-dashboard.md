# <a name="summary-dashboard-in-commercial-marketplace-analytics"></a>Сводная панель мониторинга в службе "аналитика коммерческих рынков"

Эта статья содержит сведения о сводной панели мониторинга в центре партнеров. На этой панели мониторинга отображаются графики, тенденции и значения статистических данных, которые обобщаются действия Marketplace для ваших предложений.

Чтобы получить доступ к панели мониторинга "Сводка", откройте **[панель мониторинга анализ](https://partner.microsoft.com/dashboard/commercial-marketplace/analytics/summary)** в разделе коммерческий Marketplace.

>[!NOTE]
> Подробные определения терминологии аналитики см. в разделе [часто задаваемые вопросы и терминология для коммерческого анализа](./faq-terminology.md).

## <a name="summary-dashboard"></a>Панель мониторинга "Сводка"

**Сводная** панель мониторинга представляет собой обзор на основе каждого типа предложения. **Insights** наглядно отображает важную информацию и предоставляет общее представление о действиях по продажам ваших предложений. Эти отчеты можно визуализировать с помощью панели мониторинга " **Сводка** ". В этой статье приводятся дополнительные сведения о каждом из следующих элементов.

- [Диапазон дат](#date-range)
- [Раздел сводки](#summary-section)
- [Клиенты по географическому расположению](#customers-by-geography)
- [Диаграммы трендов роста](#growth-trend-charts)
- [Список лидеров клиента](#customer-leaderboard)
- [Тренд подсчета рабочих мест](#seat-count-trend)
- [Бесплатная пробная тенденция заказов SaaS](#free-trials-saas-orders-trend)

## <a name="elements-of-the-summary-dashboard"></a>Элементы панели мониторинга "Сводка"

В следующих разделах описывается, как использовать панель мониторинга сводки и как считывать данные.

### <a name="date-range"></a>Диапазон даты

Выбор диапазона дат можно найти в правом верхнем углу каждой страницы. Настройте выходные данные на диаграммах **сводных** данных, выбрав диапазон дат на основе последних 3, 6 или 12 месяцев или выбрав Настраиваемый диапазон дат с максимальной длительностью 12 месяцев. Диапазон дат по умолчанию — шесть месяцев.

![Панель мониторинга для анализа центра партнеров](./media/analyze-dashboard.png)

### <a name="summary-section"></a>Раздел сводки

В разделе Сводка отображается количество всех созданных заказов, приобретенных клиентов и сведений об использовании за выбранный диапазон дат. Частичный текущий месяц будет исключен из вычисления этих метрик. Например, если выбран кадр времени 6 мин, часы использования будут вычисляться за шесть месяцев до текущего месяца. Если выбран настраиваемый диапазон дат, частичная сумма из текущего месяца будет исключена из вычисления.

![Тенденции роста на информационной панели сводки](./media/summary-summary-section.png)

#### <a name="reading-the-summary-section"></a>Чтение раздела сводки

- **Заказы**: количество приобретенных заказов (за исключением отмененных заказов) для предложений, опубликованных на данный момент.
- **Клиенты**: количество клиентов, которые приобрели предложения и имеют по крайней мере один неотмененный заказ.
- **Нормализованные часы использования**: определяются как часы использования, нормализованные для учета количества ядер виртуальных машин ([число ядер виртуальных машин] x [часов использования необработанного кода]). Виртуальные машины, обозначенные как "ШАРЕДКОРЕ", используют 1/6 (или 0,1666) в качестве множителя [число ядер ВМ].
- **Часы использования необработанных**данных: время, в течение которого виртуальные машины работают в часах. Процентное значение для **общих заказов**, **Общее количество клиентов**, **нормализованные часы**использования, **необработанные часы использования**, **посещения страниц**и **вызов действий** представляют объем роста использования для выбранного диапазона дат ([использование последнего месяца — первый месяц])/использование первого месяца). Как описано выше, часть текущего месяца будет исключена из этой метрики.
- **Тенденции роста**: при наведении указателя мыши на столбцы диаграммы на линейчатых диаграммах отображается значение для каждого месяца.
- **Зеленый треугольник, указывающий вверх**: обозначает положительный тренд роста.
- **Красный треугольник, указывающий вниз**: обозначает отрицательную тенденцию роста относительно предыдущего месяца.

### <a name="customers-by-geography"></a>Клиенты по географическому расположению

**Клиенты по географическому** тепловой карты отображает число клиентов на карте мира.

![Клиенты по географическому отчету на информационной панели](./media/summary-customers-by-geography.png)

- Чтобы просмотреть точное расположение, можно переместить карту.
- Можно увеличить конкретное расположение.
- Тепловой карты имеет дополнительную сетку для просмотра сведений о количестве клиентов, количестве заказов, нормализованных часах использования в определенном месте.
- Вы можете выполнить поиск и выбрать страну в сетке, чтобы увеличить расположение на карте. Вернитесь к исходному представлению, нажав кнопку **Главная** на карте.
- **Новый** клиент приобрел одно из предложений в первый раз в течение месяца в выбранном диапазоне дат.

### <a name="growth-trend-charts"></a>Диаграммы трендов роста

Вы можете просматривать тенденции на основе роста **приобретенных заказов** (включая отмененные заказы), **приобретенных клиентов** (включая потерянных клиентов) и отчетов **об использовании** , которые отображаются в месяц по месяцам в соответствии с выбранным диапазоном дат. Вы можете дополнительно проанализировать эти тенденции, выбрав ссылки под диаграммой, чтобы переходить к соответствующим страницам **заказа**, **использования**, **клиента**или **Marketplace Insights** .

**Посещения страницы предложения Marketplace и обращения к** диаграммам тенденций действий отображаются для Azure Marketplace и AppSource на двух вкладках.

![Посещения страниц и обращение к действиям диаграммы тенденций в сводных панелях мониторинга](./media/summary-page-visits-and-cta.png)

Диаграмма **заказы по предложениям** упорядочивает заказы в соответствии с названием предложения.

На круговой диаграмме « **заказы по продажам** » упорядочиваются заказы (включая заказы, отмененные клиентами) в течение выбранного диапазона дат по каналу продаж. Канал продаж — это тип лицензионного соглашения, используемого клиентами для приобретения Azure, который является поставщиком облачных решений (CSP), предприятием, предприятием через торгового посредника, GTM и оплатой по мере использования.

**Использование предложений** и использования на круговых диаграммах **канала продаж** представляет разделение использования по лучшим предложениям и каналам продаж соответственно. Внутренняя круговая диаграмма представляет необработанное использование, а внешнюю круговую диаграмму — нормализованное использование.

Тип лицензии **ORDER BY Marketplace** и **Использование по типам лицензий Marketplace тип лицензии** на круговой диаграмме отображают разбивку заказов и использования по соответствующему типу лицензии. К типам лицензий относятся:

- **Выставление счетов за использование Azure**: выставляются счета корпорации Майкрософт от вашего имени, если вы решили продать свое предложение в корпорации Майкрософт с этим типом лицензии. Типы оплаты включают оплату по мере использования с помощью кредитных карт или корпоративного выставления счетов.
- Получите **собственную лицензию**. Корпорация Майкрософт не оплачивает счета клиентов за их использование с этим типом предложения Marketplace. Эти сведения об использовании можно найти в Marketplace **бесплатно** .
- **Бесплатно**. Корпорация Майкрософт не оплачивает счета клиентов за их использование с этим типом предложения Marketplace. Это использование будет отображаться как **Бесплатная пробная версия** в Marketplace.
- **Корпорация Майкрософт в качестве торгового посредника**— представляет предложения, продаваемые торговыми посредниками Майкрософт в составе **программы поставщика облачных решений (CSP)**.

### <a name="customer-leaderboard"></a>Список лидеров клиента

Первые 50 клиентов с наибольшим количеством заказов отображаются на *лидерской доске*, отсортированном по максимальному количеству заказов и процентам заказов.

- Выберите клиента, чтобы просмотреть сведения о профиле, заказы, упорядоченные по предложениям, или заказы, упорядоченные по типу лицензии Azure и каналу ценообразования.
- Кольцевая диаграмма **предложения предложений по** заказам представляет четыре первых предложения (по количеству заказов), а оставшиеся предложения группируются в виде "все остальные".
- В **нормализованной** кольцевой диаграмме предложения представлены пять предложений по использованию.

> [!NOTE]
> Персональные данные клиента будут представлены только в том случае, если клиент предоставил согласие. Эти сведения можно просмотреть, если вы выполнили вход с правами **владельца** роль-уровень. Пользователи с ролью **участник** не смогут просматривать эти сведения. Дополнительные [сведения о ролях и разрешениях пользователей](./manage-account.md#define-user-roles-and-permissions).

### <a name="seat-count-trend"></a>Тренд подсчета рабочих мест

На диаграмме **заказы по каждому местному размещению/на сайт** представлена декомпозиция всех заказов, приобретенных в соответствии с моделью ценообразования. На диаграмме **тренда "количество** рабочих мест" представлены места и заказы, приобретенные для всех предложений по каждому местному программному обеспечению как услуги (SaaS).

### <a name="free-trials-saas-orders-trend"></a>Бесплатная пробная тенденция заказов SaaS

На диаграмме **тренда бесплатных пробных заказов SaaS** представлены тенденции заказов на бесплатные пробные версии предложений SaaS с 30-дневным пробным периодом.

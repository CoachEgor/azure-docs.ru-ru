# <a name="summary-dashboard-in-commercial-marketplace-analytics"></a>Сводка приборной панели в аналитике коммерческого рынка

В этой статье содержится информация о панели мониторинга Резюме в Партнерс-центре. Эта панель мониторинга отображает графики, тенденции и значения агрегированных данных, которые обобщают активность рынка для ваших предложений.

Чтобы получить доступ к панели мониторинга Summary, откройте **[панель мониторинга Analyse](https://partner.microsoft.com/dashboard/commercial-marketplace/analytics/summary)** под коммерческим рынком.

>[!NOTE]
> Для подробных определений терминологии аналитики, [см. Часто задаваемые вопросы и терминологии для аналитики коммерческого рынка](./faq-terminology.md).

## <a name="summary-dashboard"></a>Панель мониторинга "Сводка"

Панель мониторинга **Резюме** представляет обзор, основанный на каждом типе предложения. **Исследования** показывают важную информацию с первого взгляда и предоставляют широкое представление о деятельности продаж ваших предложений. Эти отчеты можно визуализировать с помощью панели мониторинга **Резюме.** В этой статье подробно рассказывается о каждом из следующих элементов:

- [Диапазон дат](#date-range)
- [Раздел Резюме](#summary-section)
- [Клиенты по географии](#customers-by-geography)
- [Диаграммы тенденций роста](#growth-trend-charts)
- [Таблица лидеров клиентов](#customer-leaderboard)
- [Тенденция отсчета мест](#seat-count-trend)
- [Бесплатные пробные версии SaaS заказы тенденция](#free-trials-saas-orders-trend)

## <a name="elements-of-the-summary-dashboard"></a>Элементы панели мониторинга Резюме

В следующих разделах описывается, как использовать панель мониторинга резюме и как читать данные.

### <a name="date-range"></a>Диапазон даты

Вы можете найти выбор диапазона дат в правом верхнем углу каждой страницы. Настройте выход графиков страниц **Резюме,** выбрав диапазон дат на основе последних 3, 6 или 12 месяцев, или выбрав пользовательский диапазон дат с максимальной продолжительностью 12 месяцев. Диапазон дат по умолчанию составляет шесть месяцев.

![Аналитическая панель центра партнеров](./media/analyze-dashboard.png)

### <a name="summary-section"></a>Раздел Резюме

В разделе Резюме отображается подсчет всех созданных заказов, приобретенных клиентов и использования, сообщенных в выбранном диапазоне дат. Частичный текущий месяц будет исключен из расчета этих метрик. Например: Если вы выбрали временный промежуток 6M, часы использования вычисляются за шесть месяцев до текущего месяца. При выборе пользовательского диапазона дат из вычислений будет исключена частичная сумма из текущего месяца.

![Тенденции роста в панель мониторинга Резюме](./media/summary-summary-section.png)

#### <a name="reading-the-summary-section"></a>Чтение раздела резюме

- **Заказы**: Подсчет всех заказов, приобретенных (исключает отмененные заказы) для предложений, которые вы опубликовали до сих пор.
- **Клиенты**: Граф всех клиентов, которые приобрели ваши предложения и имеют по крайней мере один не отмененный заказ.
- **Нормализованные часы использования**: Определяется как часы использования нормализованы для учета количества ядер VM (количество ядер VM) x «часы сырого использования»). ВМ, обозначенные как "SHAREDCORE", используют 1/6 (или 0.1666) в качестве множителя «числа ядер VM».
- **Часы использования raw**: Количество времени VMs были идущими в терминах часов. Процентное значение рядом с **общим числом заказов,** **общим числом клиентов,** **нормализованными часами использования,** **необработанными часами использования,** **посещениями страниц**и **призывом к действиям,** представляют собой объем роста использования для выбранного диапазона дат («использование в прошлом месяце – использование первого месяца»)/использование первого месяца). Как описано выше, частичная сумма текущего месяца будет исключена из этой метрики.
- **Тенденции роста**: Если вы нависнете над столбцов диаграммы, бар графики отображают значение для каждого месяца.
- **Зеленый треугольник, указывающий вверх**: Указывает на положительную тенденцию роста.
- **Красный треугольник, указывающий вниз**: Указывает на отрицательную тенденцию роста по отношению к предыдущему месяцу.

### <a name="customers-by-geography"></a>Клиенты по географии

**Клиенты по географии** тепловой карты отображает клиент рассчитывать на карте мира.

![Клиенты по географии в панель мониторинга Резюме](./media/summary-customers-by-geography.png)

- Вы можете переместить карту, чтобы просмотреть точное местоположение.
- Вы можете увеличить в определенном месте.
- Тепловая карта имеет дополнительную сетку для просмотра деталей подсчета клиентов, количества заказов, нормализованных часов использования в конкретном месте.
- Вы можете искать и выбирать страну в сетке, чтобы увеличить местоположение на карте. Вернуться к исходному представлению, нажав кнопку **«Домой»** на карте.
- **Новый** клиент приобрел одно из ваших предложений впервые в течение месяца в пределах выбранного диапазона дат.

### <a name="growth-trend-charts"></a>Диаграммы тенденций роста

Вы можете просматривать тенденции, основанные на росте **приобретенных заказов** (включает отмененные заказы), **приобретенных клиентов** (включая потерянных клиентов) и **сообщений** об использовании, которые отображаются месяц за месяцем в зависимости от выбранного диапазона дат. Вы можете дополнительно проанализировать эти тенденции, выбрав ссылки ниже диаграммы, которые перемещаются на соответствующие **страницы заказа,** **использования,** **клиента**или **Marketplace Insights.**

Marketplace предлагает **посещения Страницы и** диаграммы трендов для действий отображаются для рынка Azure и AppSource на двух вкладок.

![Посещения страниц и вызов к диаграммам тенденций действий в панели мониторинга Резюме](./media/summary-page-visits-and-cta.png)

**Заказы по предложениям** диаграммы организуют ваши заказы в соответствии с названием предложения.

Заказы по круговой диаграмме **канала продаж** организует ваши заказы (включая заказы, которые клиенты отменили) во время выбранного диапазона дат, по каналу Sales. Канал продаж — это тип лицензионного соглашения, используемого клиентами для покупки Azure, которые являются поставщиком облачных решений (CSP), Enterprise, Enterprise через реселлера, GTM и Pay As You Go.

**Использование предложениями** и использование продаж **канал** пирог диаграммы представляют разбивку использования по топ-предложений и каналов продаж, соответственно. Внутренняя диаграмма пирога представляет сырое использование, а внешняя диаграмма кругов представляет собой нормализованное использование.

**Заказы по типу лицензии на рынке** и использованию по диаграммам **круговтипа типа лицензии рынка** отображают разбивку заказов и использования по их соответствующим типу лицензии. Типы лицензий включают:

- **Счет через Azure:** Microsoft счета клиентов от вашего имени, когда вы решите продать свое предложение через Microsoft с этим типом лицензии. Типы платежей включают оплату по мере входинисты с помощью кредитной карты или счет-счет предприятия.
- **Принесите свою собственную лицензию:** Microsoft не выставляет счета клиентам за их использование с этим типом предложения рынка. Это использование перечислено как **Получить его сейчас (бесплатно)** на рынке.
- **Бесплатно**: Microsoft не выставляет счета клиентам за их использование с этим типом предложения рынка. Это использование перечислено как **бесплатная пробная версия** на рынке.
- **Microsoft в качестве реселлера**: Представляет предложения, продаваемые реселлерами Майкрософт в рамках **программы Поставщика облачных решений (CSP).**

### <a name="customer-leaderboard"></a>Таблица лидеров клиентов

50 лучших клиентов с наибольшим количеством заказов отображаются на *доске лидеров,* отсортированные по наивысшему проценту заказов и заказов.

- Выберите клиента для просмотра данных профиля, заказов, организованных по предложению, или заказов, организованных типом лицензии Azure и каналом ценообразования.
- **Предложения по заказу** пончик диаграммы представляет четыре лучших предложения (по количеству заказов) и остальные предложения сгруппированы в качестве "Отдых Все".
- **Нормализованное использование на графике пончиков предложений** представляет пятерку предложений по использованию.

> [!NOTE]
> Личная информация клиента будет представлена только в том случае, если клиент дал согласие. Вы можете просмотреть эту информацию, если вы вошли в систему с уровнем разрешений на роль **владельца.** Пользователи с ролью **автора** не смогут просматривать эту информацию. [Подробнее о роли пользователя и разрешениях.](./manage-account.md#define-user-roles-and-permissions)

### <a name="seat-count-trend"></a>Тенденция от счета мест

**Заказы на место/ на** график сайта представляет собой разбивку всех заказов, приобретенных в соответствии с моделью ценообразования. Диаграмма **тренда отсчета мест** представляет места против заказов закупленных для всех ваших вне программного обеспечения места как обслуживание (SaaS) предложения.

### <a name="free-trials-saas-orders-trend"></a>Бесплатные пробные версии SaaS заказы тенденция

**Бесплатная пробная версия SaaS заказов трендовая** диаграмма представляет тенденцию заказов на бесплатные пробные версии SaaS предлагает с 30-дневным пробным периодом.

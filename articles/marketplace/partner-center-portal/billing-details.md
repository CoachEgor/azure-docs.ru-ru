# <a name="commercial-marketplace-billing"></a>Коммерческий рынок биллинга

В этой статье рассматриваются темы, связанные с коммерцией для коммерческого рынка:

- [Варианты публикации в Marketplace](#marketplace-publishing-options)
- [Общие сведения о транзакциях](#transact-general-overview)
- [Модели выставления счетов по транзакциям](#transact-billing-models)

## <a name="marketplace-publishing-options"></a>Варианты публикации в Marketplace

Коммерческий рынок предлагает несколько вариантов публикации для издателей.

### <a name="list-and-trial-publishing-options"></a>Варианты списка и пробной публикации

Издатели могут использовать список, пробную версию и использовать собственные варианты публикации лицензии (BYOL) для рекламных целей и приобретения пользователей. С помощью этих опций корпорация Майкрософт не участвует непосредственно в транзакциях лицензий на программное обеспечение издателя, и плата за транзакции не взимается. Издатели несут ответственность за поддержку всех аспектов транзакции с лицензией на программу, в том числе заказа, исполнения, измерения, выставления счетов, оплаты и сбора. При публикации описаний и пробных версий издатели оставляют себе 100 % платы за лицензирование программного обеспечения, которая взимается с пользователя.

### <a name="transact-publishing-option"></a>Публикация транзакций

В дополнение к списку и варианты пробной публикации, вариант публикации сделки доступен для издателей. Эта опция использует возможности Microsoft, доступные в глобальном масштабе, и позволяет корпорации Майкрософт размещать транзакции на облачном рынке от имени издателя.

## <a name="transact-general-overview"></a>Общие сведения о транзакциях

При использовании опции публикации сделки корпорация Майкрософт разрешает продажу стороннего программного обеспечения и развертывание некоторых типов предложений в подписке клиента Azure. Издатель должен учитывать выставление счетов за инфраструктуру и сборы издателя за лицензирование программного обеспечения при выборе модели выставления счетов и типа предложения.

Опция публикации Transact в настоящее время поддерживается для следующих типов предложений: виртуальные машины, приложения Azure и приложения SaaS.

### <a name="billing-infrastructure-costs"></a>Выставление счетов за использование инфраструктуры

#### <a name="for-virtual-machines-and-azure-applications"></a>Для виртуальных машин и приложений Azure

Для виртуальных машин и приложений Azure плата за использование инфраструктуры Azure взимается за подписку клиента Azure. Плата за пользование инфраструктурой взимается и предоставляется отдельно от лицензионных сборов поставщика программного обеспечения по счету клиента.

#### <a name="for-saas-apps"></a>Для приложений SaaS

В случае приложений SaaS издателю нужно рассматривать тарифы на использование инфраструктуры Azure и лицензирование программного обеспечения как одну позицию расходов. Она представлена в виде фиксированной платы для клиента. Использование инфраструктуры Azure контролируется, а счета выставляются непосредственно партнеру. Фактическая стоимость использования инфраструктуры не видна пользователю. Как правило, издатели добавляют плату за использование инфраструктуры Azure в счета за лицензирование программного обеспечения. Плата за лицензирование программного обеспечения не измеряется и не взимается на основе потребления.

## <a name="transact-billing-models"></a>Модели выставления счетов по транзакциям

В зависимости от используемого опциона транзакции, лицензионные сборы издателя могут быть представлены следующим образом:

- *Бесплатно*: Бесплатно для лицензий на программное обеспечение.
- *Принесите свою собственную лицензию (BYOL)*: Любые применимые сборы за лицензии на программное обеспечение управляются непосредственно между издателем и клиентом. Корпорация Майкрософт обрабатывает только плату за использование инфраструктуры Azure. (Только для виртуальных машин и приложений Azure.)
- *Оплата по мере использования*: Лицензионные сборы за программное обеспечение представлены как почасовая ставка ценообразования по тарифам на один ядр (vCPU) на основе используемой инфраструктуры Azure. Эта модель применяется только к виртуальным машинам и приложениям Azure.
- *Цены на подписку*: Лицензионные сборы за программное обеспечение представлены как ежемесячная или ежегодная, повторяющаяся плата, выставляемый в виде фиксированной ставки или на место. Эта модель применяется только к приложениям SaaS и приложениям Azure - Управляемым приложениям.
- *Бесплатная пробная версия программного обеспечения*: Не взимается плата за лицензии на программное обеспечение в течение 30 дней или 90 дней.

### <a name="free-and-bring-your-own-license-byol-pricing"></a>Предоставление решений бесплатно или с использованием собственной лицензии (BYOL)

При публикации предложений бесплатно или с использованием собственной лицензии корпорация Майкрософт не принимает участия в совершении транзакций по оплате лицензий на программное обеспечение. Как и при публикации описаний и пробных версий, издатель оставляет себе 100 % стоимости лицензий.

### <a name="pay-as-you-go-and-subscription-site-based-pricing"></a>Оплата по мере использования и тарифы на подписки (на основе сайтов)

При публикации предложения о транзакциях с оплатой по мере использования или подписке корпорация Майкрософт предоставляет технологии и услуги для обработки покупок, возвратов и возврата лицензий на программное обеспечение. В этом случае издатель разрешает корпорации Майкрософт действовать в качестве агента для этих целей. Издатель разрешает корпорации Майкрософт использовать транзакции по лицензированию программного обеспечения, в то же время сохранив свою роль продавца, поставщика, распространителя и лицензиара.

Корпорация Майкрософт позволяет клиентам заказывать, лицензировать и использовать программное обеспечение издателя, подчиняясь условиям как коммерческого рынка Майкрософт, так и лицензионного соглашения издателя. Издатели должны предоставить свое лицензионное соглашение с конечным пользователем или выбрать [Стандартный контракт](https://docs.microsoft.com/azure/marketplace/standard-contract) при создании предложения.

### <a name="free-software-trials"></a>Бесплатные пробные версии

При публикации транзакций издатель может бесплатно предоставить лицензию на программное обеспечение на 30 или 90 дней. Это предложение не распространяется на стоимость использования инфраструктуры Azure, связанную с использованием партнерского решения.

### <a name="private-offers"></a>Частные предложения

В дополнение к использованию типов предложений и моделей выставления счетов для монетизации предложения, издатели могут заключать частное предложение, в комплекте с согласованными и конкретными ценами, или пользовательскими конфигурациями. Частные предложения поддерживаются во всех 3 вариантах публикации.

Эта опция позволяет более высокую или более низкую цену, чем общедоступное предложение. Частные предложения могут быть использованы для скидки или добавить премию за предложение. Частные предложения можно предоставлять одному или нескольким клиентам, добавляя их подписки Azure на уровне предложения.

**Сниженная плата за обслуживание рынка:** Для некоторых продуктов SaaS, которые вы публикуете на нашем коммерческом рынке, корпорация Майкрософт снизит плату за обслуживание на рынке с 20% (как описано в соглашении Microsoft Publisher) до 10%.  Для того, чтобы ваш продукт, чтобы претендовать, по крайней мере один из ваших продуктов должны быть назначены Microsoft либо IP совместно продать готовы или IP совместно продать приоритеты. Чтобы получить эту сниженную плату за обслуживание Marketplace за месяц, право должно быть выполнено по крайней мере за пять (5) рабочих дней до конца предыдущего календарного месяца. Сниженная плата за обслуживание Marketplace не будет распространяться на vMs, управляемые приложения или любые другие продукты, доступные через наш коммерческий рынок.  Эта сниженная плата за услуги Marketplace будет доступна для квалифицированных предложений, с лицензионными сборами, собранными корпорацией Майкрософт в период с 1 мая 2019 года по 30 июня 2020 года.  По истечение этого времени плата за обслуживание Marketplace вернется к нормальной сумме.

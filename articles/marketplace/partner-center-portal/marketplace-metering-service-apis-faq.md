# <a name="marketplace-metering-service-apis---faq"></a>Вопросы и ответы об API-интерфейсах службы из Marketplace для контроля потребления

Как только пользователь Azure подпишется на сервис SaaS, включающий дозированные счета, вы будете отслеживать потребление для каждого аспекта выставления счетов, используемого клиентом. Если потребление превышает включенные количества, установленные для термина, выбранного клиентом, служба будет излучать события использования в корпорацию Майкрософт.

## <a name="emit-usage-events"></a>Мероприятия по использованию Эмита

>[!Note]
>Этот раздел применим только для предложений SaaS, где по крайней мере один из планов имеет измерения обслуживания, определенные на момент публикации предложения.

![Мероприятия по использованию Эмита](media/isv-emits-usage-event.png)

Ознакомьтесь с [API использования пакетов SaaS](./marketplace-metering-service-apis.md#batch-usage-event) для получения информации о контракте API на события использования.

### <a name="how-often-is-it-expected-to-emit-usage"></a>Как часто ожидается использование?

В идеале, вы должны излучать использование каждый час в течение последнего часа, только если есть использование в предыдущем часе.

### <a name="what-is-the-maximum-delay-between-the-time-an-event-occurs-and-the-time-a-usage-event-is-emitted-to-microsoft"></a>Какова максимальная задержка между моментом события и временем, когда событие использования испускается в корпорацию Майкрософт?

В идеале событие использования излучается каждый час для событий, произошедших за последний час. Однако ожидаются задержки. Разрешенная максимальная задержка составляет 24 часа, после чего события использования не принимаются.

Например, если событие использования происходит в 13:00 в день, у вас есть время до 1 3:00 на следующий день, чтобы излучать событие использования, связанное с этим событием. Это означает, что в случае использования системы, испускающей, есть время простоя, он может восстановиться, а затем отправить событие использования на часовой интервал, в котором произошло использование, без потери верности.

### <a name="what-happens-when-you-send-more-than-one-usage-event-on-the-same-hour"></a>Что происходит при отправке нескольких событий использования в один и тот же час?

Для часового интервала принимается только одно событие использования. Часовой интервал начинается с минуты 0 и заканчивается в минуту 59.  Если за один и тот же часовой интервал излучается несколько событий использования, любые последующие события использования отогнаны в качестве дубликатов.

### <a name="what-happens-when-you-emit-usage-for-a-saas-subscription-that-has-been-unsubscribed-already"></a>Что происходит, когда вы испускаете использование для подписки SaaS, которая уже отподписана?

Любое событие использования, излучаемое на платформе marketplace, не будет принято после удаления подписки SaaS.

### <a name="can-you-get-a-list-of-all-saas-subscriptions-including-active-and-unsubscribed-subscriptions"></a>Можете ли вы получить список всех saaS подписки, в том числе активных и неподписанных подписок?

Да, при вызове `GET /saas/subscriptions` API он включает в себя список всех saaS подписки. Поле состояния в ответе для каждой подписки SaaS фиксирует, является ли подписка активной или отписана. Вызов для списка подписок возвращает максимум 100 подписок в то время.

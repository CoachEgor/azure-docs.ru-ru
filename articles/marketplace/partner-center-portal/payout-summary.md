# <a name="payout-summaries"></a>Сводки по выплатам

В сводке по выплатам отображаются сведения о денежных средствах, полученных вами с помощью корпорации Майкрософт. Кроме того, с ее помощью можно узнать, когда вы будете получать платежи и сколько вы получите.

Если вы продаете свои продукты через Azure Marketplace, также отобразятся сведения об успешных выплатах в сводке. Подробные сведения см. в [политиках участия в Microsoft Azure Marketplace](https://docs.microsoft.com/legal/marketplace/participation-policy) и [Соглашении с издателем Microsoft Azure Marketplace](https://query.prod.cms.rt.microsoft.com/cms/api/am/binary/RE3ypvt).

> [!NOTE]
> Чтобы получить выплату, необходимо достигнуть [порогового значения](payment-thresholds-methods-timeframes.md) в 50 долларов США. Подробные сведения о пороговом значении платежа см. на этой странице и в соглашении с разработчиком приложений.

> [!NOTE]
> Если вам нужна поддержка по поводу выплат, включая настройку счетов для выплат, отсутствующие выплаты, переведение выплат на удержание и т. д., обратитесь в службу поддержки [здесь](https://developer.microsoft.com/windows/support).

## <a name="access-the-payout-summary-pages"></a>Доступ к страницам сводки по выплатам

Чтобы открыть одну из страниц сводки по выплатам, выполните указанные ниже действия.

1. Щелкните значок "Выплата" в правом верхнем углу.
2. Выберите элемент "Журнал транзакций", "Платежи" или "Экспорт данных".

## <a name="transaction-history-page"></a>Страница "Журнал транзакций"

На этой странице приводятся все доходы, включая дату, тип и размер. Вы можете выбрать период времени для просмотра, а также выполнить фильтрацию по идентификатору регистрации, программе, идентификатору выплаты, типу прибыли, рычагу и состоянию. Данные доступны за текущий финансовый год (с 1 июля по 30 июня) и два предыдущих финансовых года.

Чтобы просмотреть дополнительные сведения о прибыли, щелкните стрелку вниз в правой части страницы. Отобразятся рычаг, сумма дохода и продукт. Если какие-либо из этих данных недоступны, но нужны вам, обратитесь в [службу поддержки](https://developer.microsoft.com/windows/support). Если прибыль является результатом корректировки, а не транзакции, поля продукта не отображаются.

Чтобы экспортировать какие-либо данные транзакции с этой страницы, используйте страницу **Экспорт данных**.

## <a name="payments-page"></a>Страница "Платежи"

Суммарные значения на этой странице отражают все программы, в которых вы участвуете. Вы можете выполнить фильтрацию по идентификатору участника, программе, идентификатору выплаты и типу прибыли. Суммы указываются в долларах США. Выплаченная сумма также указывается в валюте оплаты.

| Область                   | Описание                                                                                |
|------------------------|---------------------------------------------------------------------------------------------|
| Всего выплачено в этом году   | Общая сумма полученных вами выплат за этот год в долларах США в рамках всех программ       |
| Следующий предполагаемый платеж | Следующая ожидаемая выплата (только одна, даже если в ближайшее время их будет несколько) в долларах США. |
| Последняя выплата           | Сумма (в долларах США), название программы и программа, в рамках которой была получена последняя выплата           |
| Платежи по источнику     | Сумма выплат (в долларах США) по программам за последние 12 месяцев           |
| Платежи               | Выберите **Выплачено** или **Ожидание**, а затем выполните требуемую сортировку. Для получения дополнительных сведений об определенной выплате выберите **Просмотреть**. Чтобы скачать копию извещения о переводе оплаты, выберите **Скачать**. Так как на отображение данных журнала транзакций может потребоваться до 24 часов, возможно, вы не сразу увидите связанные доходы. |
|||

Чтобы экспортировать какие-либо данные с этой страницы, выберите команду **Экспорт** и следуйте указаниям на странице "Экспорт данных".

Вы можете выбрать период времени для просмотра, а также выполнить фильтрацию по идентификатору регистрации, программе, идентификатору выплаты, типу прибыли, рычагу и состоянию. Если какие-либо из этих данных недоступны, но нужны вам, обратитесь в [службу поддержки](https://developer.microsoft.com/windows/support). ## <a name="export-data-page"></a>Страница "Экспорт данных"

Чтобы экспортировать данные, следуйте приведенным ниже инструкциям.

Страница "Экспорт данных" не обновляется самостоятельно. Для отображения последних данных может потребоваться обновить страницу вручную.

Фильтр может вызвать ошибку **Данные недоступны**. Это может означать, что вы оставили период времени по умолчанию, равный трем месяцам, а затем выбрали идентификатор выплаты для дохода, не попадающего в этот период. Расширьте период времени и повторите попытку.

## <a name="historical-statements"></a>Ретроспективная отчетность

![Экспорт ретроспективной отчетности](./media/pc-export-statements.png)

Журнал транзакций, дата начала которого раньше 1 июля 2019 г., обрабатывается отдельно. Вместо текущих полей в отчетности будут использоваться указанные ниже поля.

> [!NOTE]
> В устаревшем журнале транзакций есть столбец "Зарезервировано", который соответствует столбцу "Прибыль" в современном журнале, за исключением того, что в нем исключены все доходы с состоянием "Платеж отправлен".

> [!NOTE]
> Фильтры, такие как 3 месяца, 6 месяцев или 12 месяцев, не будут применяться к разделу **Ретроспективная отчетность**.

| Имя поля              | Описание                                                                                                                                                             |
|-------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Источник дохода          | Источник дохода в зависимости от того, где была выполнена транзакция, например Microsoft Store, Магазине Windows Phone, Магазин Windows 8 или реклама                  |
| Идентификатор заказа                | Уникальный идентификатор заказа. Этот идентификатор позволяет идентифицировать транзакции покупки вместе с соответствующими транзакциями, не связанными с покупкой, такими как возмещение или возвратный платеж. И те, и другие будут иметь одинаковые идентификаторы заказов. Кроме того, в случае использования нескольких методов оплаты для одной покупки, можно установить связь с транзакциями покупки. |
| Transaction ID          | Уникальный идентификатор транзакции.                                                                                                                                          |
| Дата и время транзакции   | Дата и время выполнения транзакции (в формате UTC).                                                                                                                       |
| Идентификатор родительского продукта       | Уникальный идентификатор родительского продукта. Если для транзакции не указан родительский продукт, то в качестве идентификатора родительского продукта будет использоваться идентификатор продукта.                                |
| Product ID              | Уникальный идентификатор продукта                                                                                                                                              |
| Название родительского продукта     | Название родительского продукта. Если для транзакции не указан родительский продукт, то в качестве названия родительского продукта будет использоваться название продукта.                                  |
| Название продукта            | Название продукта                                                                                                                                                    |
| Тип продукта            | Тип продукта, например приложение, надстройка или игра                                                                                                                       |
| Количество                | Если источником дохода является Microsoft Store для бизнеса, это значение представляет количество приобретенных лицензий. Для остальных источников дохода количество всегда равно 1. Даже если одна транзакция разбивается на две позиции из-за использования двух разных методов оплаты, каждая позиция будет иметь количество 1. |
| Тип транзакции        | Тип транзакции, например покупка, возмещение, реверсивный платеж или возвратный платеж                                                                                              |
| Метод оплаты          | Платежное средство клиента, используемое для транзакции, например платежная карта, оплата через оператора мобильной сети или PayPal                                                               |
| Страна или регион        | Страна или регион, где была произведена транзакция.                                                                                                                          |

| Сумма транзакции      | Сумма транзакции                                                                                                                                              |
| Перечисленные налоги            | Сумма налогов, от уплаты которых освобождена сторона (налог с продаж, налог за использование или НДС/GST)                                                                                                                  |

# <a name="common-questions-about-migrating-from-the-cloud-partner-portal-to-partner-center"></a>Распространенные вопросы о миграции из Портала Cloud Partner в Центр партнеров

В этой статье рассмотрены часто задаваемые вопросы о миграции предложений из Портала Cloud Partner в Центр партнеров.

## <a name="what-does-offer-migration-mean"></a>Что означает миграция предложений?

Мы переносим данные ваших предложений из Портала Cloud Partner в Центр партнеров с изменениями в интерфейсе публикации предложений и управления ими.

| Область  | Изменения  |
|-------|----------|
| **Возможности публикации предложений и управления ими** | Благодаря интуитивному интерфейсу Центр партнеров удобнее использовать. Дополнительные сведения о различиях между Центром партнеров и Порталом Cloud Partner см. в [этой статье](#what-are-the-differences-between-partner-center-and-the-cloud-partner-portal). |
| **Доступность ваших предложений в Marketplace** | Без изменений. Если ваше предложение доступно в Marketplace, оно сохранится во время и после завершения миграции. |
| **Новые покупки и развертывания** | Без изменений. Ваши клиенты и дальше смогут покупать и развертывать предложения без каких-либо перебоев. |
| **выплаты;** | Средства за все покупки и развертывания, происходящие во время и после миграции, будут поступать вам в обычном режиме. |
|**Интеграция API с существующими [API Портала Cloud Partner](https://docs.microsoft.com/azure/marketplace/cloud-partner-portal-orig/cloud-partner-portal-api-overview)** | Поддержка существующих API Портала Cloud Partner сохранится после миграции, а существующие интеграции будут продолжать работать. Дополнительные сведения см. в разделе [Будут ли поддерживаться REST API Портала Cloud Partner после миграции?](#will-the-cloud-partner-portal-rest-apis-be-supported-post-migration) |
| | |

## <a name="can-i-still-access-the-cloud-partner-portal-and-manage-my-offers-during-migration"></a>Смогу ли я пользоваться возможностями Портала Cloud Partner и управлять моими предложениями во время миграции?

Все предложения перенесены в Центр партнеров. Период миграции для каждого предложения, как правило, составляет менее 24 часов. После миграции предложения вам будет отправлено уведомление по электронной почте.

После миграции предложений они будут находиться на Портале Cloud Partner в режиме только для чтения **в течение ограниченного периода времени**. Перенесенное предложение будет иметь состояние Moved to Partner Center (Перемещено в Центр партнеров). Также будет приведена ссылка на предложение в Центре партнеров, как показано на следующих снимках экрана. С этого момента вы будете управлять обновлениями всех ваших предложений или создавать новые предложения исключительно через Центр партнеров.

:::image type="content" source="media/migration-faq/all-offers-2.png" alt-text="Иллюстрация сообщения, отображаемого для предложений, которые были перенесены в Центр партнеров.":::

## <a name="do-i-need-to-create-a-new-account-to-manage-offers-in-partner-center"></a>Нужно ли создавать новую учетную запись для управления предложениями в Центре партнеров?

Нет. Ваша базовая учетная запись будет сохранена, и вы уже должны управлять ею в Центре партнеров. Это означает, что если вы уже являетесь партнером, вы можете использовать существующие учетные данные учетной записи Портала Cloud Partner для входа в Центр партнеров после миграции. Из Портала Cloud Partner в Центр партнеров перемещаются только предложения и связанные с ними возможности управления. Мы просим вас не создавать новые учетные записи, так как ваши предложения не будут связаны с новой учетной записью.

## <a name="i-see-a-message-in-the-cloud-partner-portal-to-activate-my-account-what-does-this-mean"></a>Я вижу сообщение на Портале Cloud Partner об активации учетной записи. Что это значит?

Чтобы учетная запись была правильно перенесена в Центр партнеров, предоставьте еще некоторые сведения. Это позволит вам управлять предложениями в Центре партнеров после завершения миграции предложения. Дополнительные сведения об активации учетной записи см. в статье о [миграции учетной записи из Портала Cloud Partner в Центр партнеров](https://docs.microsoft.com/azure/marketplace/partner-center-portal/account-migration-from-cpp-to-pc).

Шаги, необходимые для активации учетной записи, зависят от роли вашей учетной записи.

| Роль учетной записи | Шаги активации |
|--------------|----------------|
|Владелец | Перейдите на страницу [Publisher Profile](https://cloudpartner.azure.com/#profile) (Профиль издателя) на Портале Cloud Partner, а затем щелкните ссылку в баннере, чтобы активировать учетную запись. Вы будете перенаправлены в Центр партнеров, чтобы завершить активацию. |
| Участник | Активировать учетную запись может только пользователь в учетной записи с ролью владельца. Свяжитесь с владельцами учетной записи, чтобы завершить активацию учетной записи. Владельцы учетной записи должны быть указаны в баннере. |
| | |

### <a name="what-are-the-differences-between-partner-center-and-the-cloud-partner-portal"></a>В чем заключаются различия между Центром партнеров и Порталом Cloud Partner?

Вы можете заметить следующие различия между Порталом Cloud Partner и Центром партнеров.

### <a name="modular-publishing-capabilities"></a>Возможности модульной публикации

Центр партнеров предоставляет возможность модульной публикации, позволяющую выбрать изменения, которые необходимо опубликовать, а не публиковать все обновления одновременно. Например, на следующем снимке экрана показано, что единственными изменениями, выбранными для публикации, являются изменения на страницах **Properties** (Свойства) и **Offer Listing** (Список предложений).

:::image type="content" source="media/migration-faq/review-and-publish-migration.png" alt-text="Иллюстрация страницы Review and Publish (Предварительный просмотр и публикация).":::

Обновления, которые вы не публикуете, сохраняются в виде черновиков. Используйте возможности предварительного просмотра предложения, чтобы проверить предложение, прежде чем сделать его общедоступным.

### <a name="rich-text-format"></a>Формат RTF

Сделайте описание предложения и плана более привлекательным, воспользовавшись редактором форматированного текста на странице Offer Listing (Список предложений) и Plan Listing (Список планов).

:::image type="content" source="media/migration-faq/rich-text-editor-migration.png" alt-text="Иллюстрация редактора форматированного текста.":::

### <a name="enhanced-preview-options"></a>Расширенные параметры предварительного просмотра

Центр партнеров включает [функцию сравнения](https://docs.microsoft.com/azure/marketplace/partner-center-portal/update-existing-offer#compare-changes-to-marketplace-offers) с усовершенствованными параметрами фильтрации. Это дает возможность сравнивать версию предложения на этапе предварительного просмотра с опубликованной версией.

:::image type="content" source="media/migration-faq/enhanced-preview.png" alt-text="Иллюстрация функции сравнения.":::

### <a name="branding-and-navigation-changes"></a>Изменение наименований и навигации

Вы заметите некоторые изменения наименований. Например, в Центре партнеров номера SKU теперь называются планами.

:::image type="content" source="media/migration-faq/plan-overview-migration.png" alt-text="Иллюстрация обзора планов.":::

Кроме того, сбор сведений, которые вы ранее предоставляли на страницах **Marketplace** или **сведений об онлайн-магазине** (консультационная услуга, приложение Power BI) на Портале Cloud Partner, теперь выполняется на странице **Offer listing** (Список предложений) в Центре партнеров.

:::image type="content" source="media/migration-faq/offer-listing-migration.png" alt-text="Иллюстрация страницы со списком предложений.":::

Сбор сведений, которые вы ранее предоставляли для номеров SKU на одной странице Портала Cloud Partner, теперь может выполняться на нескольких страницах в Центре партнеров:

* на странице настройки планов;
* на странице списка планов;
* на странице доступности планов;
* на странице технической конфигурации планов, показанной на снимке экрана.

:::image type="content" source="media/migration-faq/tech-config-migration.png" alt-text="Иллюстрация страницы технической конфигурации плана.":::

Ваш идентификатор предложения теперь отображается на левой панели навигации предложения.

:::image type="content" source="media/migration-faq/offer-id-offer-overview.png" alt-text="Иллюстрация левого меню навигации с идентификатором предложения.":::

Эта функция доступна на странице **Offer overview** (Обзор предложения).

## <a name="will-the-cloud-partner-portal-rest-apis-be-supported-post-migration"></a>Будут ли поддерживаться REST API Портала Cloud Partner после миграции?

Программные интерфейсы Портала Cloud Partner интегрированы с Центром партнеров и продолжат работать после миграции предложений в Центр партнеров. Интеграция включает небольшие изменения. Ознакомьтесь с изменениями, приведенными в таблице ниже, чтобы код продолжал работать после миграции в Центр партнеров.

| **API** | **Описание изменения** | **Влияние** |
| ------- | ---------------------- | ---------- |
| POST Publish, GoLive, Cancel | У заголовка ответа перенесенных предложений будет другой формат, но он будет продолжать работать таким же образом, обозначая относительный путь для получения состояния операции. | При отправке любого из соответствующих запросов POST для предложения заголовок Location будет иметь один из двух форматов в зависимости от состояния миграции предложения:<ul><li>Предложения, которые не были перенесены<br>`/api/operations/{PublisherId}${offerId}$2$preview?api-version=2017-10-31`</li><li>Перенесенные предложения<br>`/api/publishers/{PublisherId}/offers/{offereId}/operations/408a4835-0000-1000-0000-000000000000?api-version=2017-10-31`</li> |
| Операция GET | Для предложений, которые ранее поддерживали в ответе поле notification-email, это поле станет нерекомендуемым и больше не будет возвращаться для перенесенных предложений. | Для перенесенных предложений мы больше не будем отправлять уведомления на списки адресов электронной почты, указанные в запросах. Вместо этого служба API будет отправлять сообщения электронной почты посредством процесса уведомления по электронной почте в Центре партнеров. В частности, уведомления о ходе выполнения операции будут отправляться по адресу электронной почты, указанному в разделе "Контактные данные продавца" параметров учетной записи в Центре партнеров.<br><br>Проверьте в разделе "Контактные данные продавца" [параметров учетной записи](https://partner.microsoft.com/dashboard/account/management) в Центре партнеров правильность адреса электронной почты, который будет использоваться для уведомлений.  |

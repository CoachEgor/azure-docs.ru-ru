# <a name="commercial-marketplace-transact-capabilities"></a>Возможности Transact Marketplace для коммерческого рынка

В этой статье рассматриваются следующие вопросы, связанные с коммерческими решениями для коммерческого рынка Майкрософт.

* Ответственность за транзакции для различных вариантов публикации
* Общие сведения о транзакциях
* Модели выставления счетов по транзакциям
* Требования к транзакциям

## <a name="transactions-by-publishing-option"></a>Транзакции по параметрам публикации

Издатель или корпорация Майкрософт несут ответственность за управление транзакциями лицензий на программное обеспечение для предложений в коммерческом магазине. Вариант публикации, выбранный для предложения, определяет, кто управляет транзакцией. См. раздел [Определение параметров публикации](./determine-your-listing-type.md#choose-a-publishing-option) для обеспечения доступности и объяснение каждого варианта публикации.

### <a name="list-trial-and-byol-publishing-options"></a>Параметры публикации списков, пробных и BYOL

Издатели с существующими возможностями торговли могут выбирать списки, пробные версии и использовать собственные лицензии (BYOL) для получения рекламных и пользовательских целей. Благодаря этим параметрам Корпорация Майкрософт не участвует непосредственно в транзакциях программного обеспечения издателя и не взимается за транзакцию. Издатели несут ответственность за поддержку всех аспектов транзакции с лицензией на программу, в том числе заказа, исполнения, измерения, выставления счетов, оплаты и сбора. При публикации описаний и пробных версий издатели оставляют себе 100 % платы за лицензирование программного обеспечения, которая взимается с пользователя.

### <a name="transact-publishing-option"></a>Публикация транзакций

Вариант публикации на языке Transact использует преимущества Microsoft Commerce и обеспечивает комплексный опыт обнаружения и оценки для приобретения и реализации. Счета на языке Transact выставляются по существующей подписке Майкрософт или кредитной карте, что позволяет корпорации Майкрософт размещать транзакции в облаке Marketplace от имени издателя.

При создании нового предложения в центре партнеров вы выбираете параметр Transact. На странице **Настройка предложения** в разделе **сведения о настройке**выберите "Да, я хочу продавать через корпорацию Майкрософт и получит от моего имени транзакции Microsoft Host". Этот параметр будет отображаться, только если для вашего типа предложения доступен язык Transact.

## <a name="transact-general-overview"></a>Общие сведения о транзакциях

При использовании параметра публикации на языке Transact Корпорация Майкрософт включает продажу сторонних программ и развертывание некоторых типов предложений в подписку Azure клиента. При выборе модели выставления счетов и типа предложения издатель должен учитывать выставление счетов за инфраструктуру и лицензии на лицензирование программного обеспечения издателя.

В настоящее время параметр публикации Transact поддерживает следующие типы предложений: виртуальные машины, приложения Azure и приложения SaaS.

### <a name="billing-infrastructure-costs"></a>Выставление счетов за использование инфраструктуры

**Для виртуальных машин и приложений Azure**

Для виртуальных машин и приложений Azure плата за использование инфраструктуры Azure оплачивается по подписке Azure клиента. Плата за использование инфраструктуры взимается и предоставляется отдельно от оплаты поставщика программного обеспечения по счету клиента.

**Для приложений SaaS**

В случае приложений SaaS издателю нужно рассматривать тарифы на использование инфраструктуры Azure и лицензирование программного обеспечения как одну позицию расходов.  Он представляется как фиксированная плата клиенту. Использование инфраструктуры Azure контролируется, а счета выставляются непосредственно партнеру. Фактическая стоимость использования инфраструктуры не видна пользователю. Как правило, издатели добавляют плату за использование инфраструктуры Azure в счета за лицензирование программного обеспечения. Плата за лицензирование программного обеспечения не измеряется и не взимается на основе потребления.

## <a name="transact-billing-models"></a>Модели выставления счетов по транзакциям

В зависимости от используемого параметра транзакции плата за лицензии на программное обеспечение издателя может быть представлена следующим образом:  

* Бесплатно: плата за программные лицензии не взимается.

* С использованием собственной лицензии (BYOL): управление всеми применимыми выплатами за лицензии на программное обеспечение осуществляется непосредственно между издателем и пользователем. Корпорация Майкрософт обрабатывает только плату за использование инфраструктуры Azure. Это относится только к виртуальным машинам и приложениям Azure.

* С оплатой по мере использования: плата за лицензии на программное обеспечение взимается согласно почасовому тарифу с учетом количества ядер (виртуальных ЦП) и использования инфраструктуры Azure. Это относится только к виртуальным машинам и приложениям Azure.

* Цены на подписку. плата за лицензирование программного обеспечения представлена в виде ежемесячной или ежегодной платы с фиксированным тарифом или на рабочее место. Это относится только к приложениям SaaS и приложениям Azure, управляемым только для приложений.

* Бесплатная пробная версия программного обеспечения: плата за 30- или 90-дневные лицензии на программы не взимается.

### <a name="free-and-bring-your-own-license-byol-pricing"></a>Предоставление решений бесплатно или с использованием собственной лицензии (BYOL)

При публикации предложений бесплатно или с использованием собственной лицензии корпорация Майкрософт не принимает участия в совершении транзакций по оплате лицензий на программное обеспечение. Как и при публикации описаний и пробных версий, издатель оставляет себе 100 % стоимости лицензий.

### <a name="pay-as-you-go-and-subscription-site-based-pricing"></a>Оплата по мере использования и тарифы на подписки (на основе сайтов)

При публикации предложений с оплатой по мере использования или по количеству подписок корпорация Майкрософт предоставляет технологии и услуги для обработки покупок и возвратов лицензий на программное обеспечение, а также возвратных платежей. В этом случае издатель разрешает корпорации Майкрософт действовать в качестве агента для этих целей. Издатель разрешает корпорации Майкрософт использовать транзакции по лицензированию программного обеспечения, в то же время сохранив свою роль продавца, поставщика, распространителя и лицензиара.

Корпорация Майкрософт позволяет клиентам заказать, лицензировать и использовать программное обеспечение издателя, в соответствии с условиями и условиями коммерческого рынка Майкрософт и лицензионным соглашением с конечным пользователем. Издатели должны предоставить свое соглашение о лицензировании конечных пользователей или выбрать [стандартный контракт](./standard-contract.md) при создании предложения.

### <a name="free-software-trials"></a>Бесплатные пробные версии

При публикации транзакций издатель может бесплатно предоставить лицензию на программное обеспечение на 30 или 90 дней. Это предложение не распространяется на стоимость использования инфраструктуры Azure, связанную с использованием партнерского решения.

### <a name="private-offers"></a>Частные предложения

В дополнение к использованию типов предложений и моделей выставления счетов для монетизации предложения, издатели могут выполнить инструкцию по выпуску частного предложения с согласованными, зависящими от него ценами или пользовательскими конфигурациями. Частные предложения поддерживаются во всех 3 вариантах публикации.

Этот параметр позволяет использовать более высокие или более низкие цены, чем общедоступное предложение. Частные предложения могут содержать как скидки, так и премиум-содержимое. Частные предложения можно предоставлять одному или нескольким клиентам, добавляя их подписки Azure на уровне предложения.

### <a name="customer-invoicing-payment-billing-and-collections"></a>Выставление счетов, оплата и сбор средств у клиента

**Выставление счетов и оплата**

Издатель может использовать предпочтительный метод выставления счетов клиента, чтобы доставлять подписки или лицензии на программное обеспечение ПАЙГО.

**Соглашение Enterprise** 

Если предпочтительным методом выставления счетов является Microsoft Соглашение Enterprise, плата за лицензии на программное обеспечение будет взиматься с помощью этого метода выставления счетов в виде детализированной стоимости, отдельно от любых затрат на использование, связанных с Azure.

**Кредитные карты и ежемесячные накладные** 

Клиенты также могут оплачивать решения с помощью кредитных карт и ежемесячных накладных. В этом случае стоимость лицензий на программное обеспечение взимается точно так же, как и при использовании Соглашения Enterprise, в виде постатейной суммы отдельно от платы за использование Azure.

Например, если клиент приобретает решение с помощью кредитной карты:

|Описание    |    Дата  |
|----------|----------|

|Дата выставления счета | 1 сентября 2018 г. |

**Бесплатные кредиты и денежные обязательства** 

Некоторые клиенты предварительно оплачивают Azure с помощью денежного обязательства в Соглашении Enterprise или получают бесплатные кредиты для использования с Azure. Эти кредиты можно использовать для оплаты использования Azure, но они не подходят для оплаты лицензий на программное обеспечение издателя.

**Выставление счетов и сборы** 

Счета за лицензии на программное обеспечение издателя предоставляются при помощи выбранного клиентом способа в соответствии с расписанием выставления счетов. Клиенты без Соглашения Enterprise ежемесячно получают счета за лицензии на программное обеспечение из Marketplace. Клиенты с Соглашением Enterprise ежемесячно оплачивают счета через накладную, представляемую раз в квартал.

Если выбрана ценовая модель на основе подписок или с оплатой по мере использования, то корпорация Майкрософт выступает в качестве агента издателя и отвечает за все аспекты выставления счетов, оплаты и сбора.

### <a name="publisher-payout-and-reporting"></a>Отчеты и выплаты издателя

* Из платы за лицензирование программного обеспечения, которую собирает корпорация Майкрософт, вычитается операционный сбор в размере 20 % (если не указано иное) во время выплаты издателю.

* Как правило, клиенты покупают решения при помощи Соглашения Enterprise или соглашения об оплате по мере использования с поддержкой кредитных карт. Тип соглашения определяет расписание выставления счетов, сбора и выплат.

## <a name="transact-requirements"></a>Требования к транзакциям

В этом разделе описываются требования к транзакциям для предложений различных типов.

### <a name="requirements-for-all-offer-types"></a>Требования для предложений всех типов

* Для варианта публикации на языке Transact требуется учетная запись Майкрософт и финансовая информация, независимо от модели ценообразования предложения.
* Обязательные финансовые сведения включают счет выплата и налоговый профиль.

Дополнительные сведения о настройке этих учетных записей см. [в статье Управление учетной записью коммерческого магазина в центре партнеров](https://docs.microsoft.com/azure/marketplace/partner-center-portal/manage-account).

### <a name="requirements-for-specific-offer-types"></a>Требования для предложений определенных типов

Публикация транзакций доступна для использования только с предложениями Marketplace следующих типов:

**Виртуальная машина** 

Выберите ценовую модель с бесплатным использованием, с использованием собственной лицензии или с оплатой по мере использования и представьте номера SKU, определенные на уровне предложения. В счете клиента Azure Корпорация Майкрософт предоставляет лицензии на программное обеспечение издателя отдельно от базовой платы Azure. Стоимость использования инфраструктуры Azure зависит от использования программного обеспечения издателя.

**Приложения Azure: шаблон решения или управляемое приложение** 

Необходимо подготовить одну или несколько виртуальных машин и посчитать сумму тарифов за их использование. Для управляемых приложений в одном плане можно выбрать ежемесячную подписку с фиксированным тарифом вместо оплаты по количеству виртуальных машин. В некоторых случаях плата за использование инфраструктуры Azure передается клиенту отдельно от платных лицензий на программное обеспечение, но в рамках одной и той же инструкции выставления счетов. Однако если вы настроили предложение управляемого приложения для оплаты инфраструктуры ISV, ресурсы Azure выставляются на издатель, а клиент получает фиксированный тариф, включающий в себя стоимость инфраструктуры, лицензий на программное обеспечение и служб управления.

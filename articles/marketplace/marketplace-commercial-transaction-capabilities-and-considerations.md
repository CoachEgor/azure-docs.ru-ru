# <a name="commercial-marketplace-transact-capabilities"></a>Возможности Transact Marketplace для коммерческого рынка

## <a name="transactions-by-publishing-option"></a>Транзакции по параметрам публикации

Издатель или корпорация Майкрософт несут ответственность за управление транзакциями лицензий на программное обеспечение для предложений в коммерческом магазине. Вариант публикации, выбранный для предложения, определяет, кто управляет транзакцией. См. раздел [Определение параметров публикации](./determine-your-listing-type.md#choose-a-publishing-option) для обеспечения доступности и объяснение каждого варианта публикации.

### <a name="list-trial-and-byol-publishing-options"></a>Параметры публикации списков, пробных и BYOL

Издатели с существующими возможностями торговли могут выбирать списки, пробные версии и использовать собственные лицензии (BYOL) для получения рекламных и пользовательских целей. Благодаря этим параметрам Корпорация Майкрософт не принимает участие в лицензиях на программное обеспечение издателя напрямую и не взимается за транзакцию. Издатели отвечают за поддержку всех аспектов транзакции лицензии на программное обеспечение, включая, но не ограничиваясь заказами, исполнением, контролем, выставлением счетов, оплатой и сбором. При публикации описаний и пробных версий издатели оставляют себе 100 % платы за лицензирование программного обеспечения, которая взимается с пользователя.

### <a name="transact-publishing-option"></a>Публикация транзакций

Вариант публикации в режиме Transact-The использует преимущества Microsoft Commerce и предоставляет комплексный опыт от обнаружения и оценки для приобретения и реализации. Счета на языке Transact выставляются по существующей подписке Майкрософт или кредитной карте, что позволяет корпорации Майкрософт размещать транзакции в облаке Marketplace от имени издателя.

При создании нового предложения в центре партнеров вы выбираете параметр Transact. На странице **Настройка предложения** в разделе **сведения о настройке**выберите "Да, я хочу продавать через корпорацию Майкрософт и получит от моего имени транзакции Microsoft Host". Этот параметр будет отображаться, только если для вашего типа предложения доступен язык Transact.

## <a name="transact-overview"></a>Общие сведения о языке Transact

При использовании параметра публикации на языке Transact Корпорация Майкрософт включает продажи сторонних программного обеспечения и развертывание некоторых типов предложений в подписку Azure клиента. При выборе модели выставления счетов и типа предложения издатель должен учитывать выставление счетов за инфраструктуру и плату лицензирования программного обеспечения.

В настоящее время параметр публикации Transact поддерживается для следующих типов предложений:

- Виртуальные машины
- Приложения Azure
- Приложения SaaS

### <a name="billing-infrastructure-costs"></a>Выставление счетов за использование инфраструктуры

Для **виртуальных машин и приложений Azure**плата за использование инфраструктуры Azure оплачивается по подписке Azure клиента. Плата за использование инфраструктуры взимается и предоставляется отдельно от оплаты поставщика программного обеспечения по счету клиента.

Для **приложений SaaS**издатель должен учитывать сборы за использование инфраструктуры Azure и лицензирование программного обеспечения в виде единой номенклатуры затрат.  Он представляется как фиксированная плата клиенту. Использование инфраструктуры Azure контролируется, а счета выставляются непосредственно партнеру. Фактическая стоимость использования инфраструктуры не видна пользователю. Как правило, издатели добавляют плату за использование инфраструктуры Azure в счета за лицензирование программного обеспечения. Плата за лицензирование программного обеспечения не измеряется и не взимается на основе потребления.

## <a name="transact-billing-models"></a>Модели выставления счетов по транзакциям

В зависимости от используемого параметра транзакции плата за лицензирование программного обеспечения выглядит следующим образом:

- **Бесплатно** — плата за лицензии на программное обеспечение не взимается.
- Использование **собственной лицензии** (BYOL). любые применимые платежи для лицензий на программное обеспечение управляются напрямую между издателем и клиентом. Корпорация Майкрософт обрабатывает только плату за использование инфраструктуры Azure. Это относится только к виртуальным машинам и приложениям Azure.
- **Оплата по мере** использования. плата за лицензии на программное обеспечение представлена в виде тарифа за каждый час, в зависимости от используемой инфраструктуры Azure. Это относится только к виртуальным машинам и приложениям Azure.
- **Цены на подписку** . плата за лицензии на программное обеспечение представлена в виде ежемесячной или ежегодной платы с фиксированным тарифом или на рабочее место. Это относится только к приложениям SaaS и приложениям Azure, управляемым только для приложений.
- **Бесплатная пробная версия программного обеспечения** — плата за лицензии на программное обеспечение не взимается в течение 30 или 90 дней.

### <a name="free-and-bring-your-own-license-byol-pricing"></a>Предоставление решений бесплатно или с использованием собственной лицензии (BYOL)

При публикации предложений бесплатно или с использованием собственной лицензии корпорация Майкрософт не принимает участия в совершении транзакций по оплате лицензий на программное обеспечение. Как и при публикации описаний и пробных версий, издатель оставляет себе 100 % стоимости лицензий.

### <a name="pay-as-you-go-and-subscription-site-based-pricing"></a>Оплата по мере использования и тарифы на подписки (на основе сайтов)

При публикации предложений с оплатой по мере использования или по количеству подписок корпорация Майкрософт предоставляет технологии и услуги для обработки покупок и возвратов лицензий на программное обеспечение, а также возвратных платежей. В этом случае издатель разрешает корпорации Майкрософт действовать в качестве агента для этих целей. Издатель разрешает корпорации Майкрософт использовать транзакции по лицензированию программного обеспечения, в то же время сохранив свою роль продавца, поставщика, распространителя и лицензиара.

Корпорация Майкрософт дает клиентам возможность заказать, лицензировать и использовать ваше программное обеспечение в соответствии с условиями коммерческого магазина Майкрософт и лицензионным соглашением конечных пользователей. Необходимо указать собственное соглашение о лицензировании конечных пользователей или выбрать [стандартный контракт](./standard-contract.md) при создании предложения.

### <a name="free-software-trials"></a>Бесплатные пробные версии

В сценариях публикации на языке Transact можно сделать лицензию на программное обеспечение доступной бесплатно в течение 30 или 90 дней. Эта возможность не учитывает стоимость использования инфраструктуры Azure, управляемой с помощью партнерского решения.

### <a name="private-offers"></a>Частные предложения

Помимо использования типов предложений и моделей выставления счетов для монетизации предложения, вы можете выполнить инструкцию по выполнению частного предложения с согласованными, зависящими от конкретной цены или пользовательскими конфигурациями. Частные предложения поддерживаются всеми тремя вариантами публикации Transact.

Этот параметр позволяет использовать более высокие или более низкие цены, чем общедоступное предложение. Частные предложения можно использовать для скидки или добавления уровня "Премиум" в предложение. Частные предложения можно сделать доступными для одного или нескольких клиентов с помощью белого списка подписки Azure на уровне предложения.

### <a name="customer-invoicing-payment-billing-and-collections"></a>Выставление счетов, оплата и сбор средств у клиента

**Выставление счетов и оплата** — вы можете использовать предпочтительный метод выставления счетов клиента, чтобы доставлять подписки или лицензии на программное обеспечение пайго.

**Соглашение Enterprise** — если предпочтительным методом выставления счетов клиента является Microsoft Соглашение Enterprise, плата за лицензии на программное обеспечение будет взиматься с использованием этого метода выставления счетов в виде детализированной стоимости, отдельно от любых затрат на использование в Azure.

**Кредитные карты и ежемесячный счет** — клиенты также могут платить за использование кредитной карты и ежемесячного счета. В этом случае стоимость лицензий на программное обеспечение взимается точно так же, как и при использовании Соглашения Enterprise, в виде постатейной суммы отдельно от платы за использование Azure.

**Бесплатные кредиты и денежные обязательства** . Некоторые клиенты заставляют Azure с денежным обязательством в Соглашение Enterprise или предоставлены бесплатные кредиты для использования с Azure. Эти кредиты можно использовать для оплаты использования Azure, но они не подходят для оплаты лицензий на программное обеспечение издателя.

**Выставление счетов и сбор** данных — выставление счетов по лицензиям издателя предоставляется с помощью выбранного пользователем метода выставления счетов и соответствует временной шкале выставления счетов. Клиенты без Соглашения Enterprise ежемесячно получают счета за лицензии на программное обеспечение из Marketplace. Клиенты с Соглашением Enterprise ежемесячно оплачивают счета через накладную, представляемую раз в квартал.

Если выбрана ценовая модель на основе подписок или с оплатой по мере использования, то корпорация Майкрософт выступает в качестве агента издателя и отвечает за все аспекты выставления счетов, оплаты и сбора.

### <a name="publisher-payout-and-reporting"></a>Отчеты и выплаты издателя

Из платы за лицензирование программного обеспечения, которую собирает корпорация Майкрософт, вычитается операционный сбор в размере 20 % (если не указано иное) во время выплаты издателю.

Как правило, клиенты покупают решения при помощи Соглашения Enterprise или соглашения об оплате по мере использования с поддержкой кредитных карт. Тип соглашения определяет расписание выставления счетов, сбора и выплат.

## <a name="transact-requirements"></a>Требования к транзакциям

В этом разделе рассматриваются требования к языку Transact для различных типов предложений.

### <a name="requirements-for-all-offer-types"></a>Требования для предложений всех типов

- Для варианта публикации на языке Transact требуется учетная запись Майкрософт и финансовая информация, независимо от модели ценообразования предложения.
- Обязательные финансовые сведения включают счет выплата и налоговый профиль.

Дополнительные сведения о настройке этих учетных записей см. [в статье Управление учетной записью коммерческого магазина в центре партнеров](https://docs.microsoft.com/azure/marketplace/partner-center-portal/manage-account).

### <a name="requirements-for-specific-offer-types"></a>Требования для предложений определенных типов

Публикация транзакций доступна для использования только с предложениями Marketplace следующих типов:

- **Виртуальная машина** — выберите модели "бесплатный", "присвоить собственные лицензии" или "Оплата по мере использования" и приводятся как номера SKU, определенные на уровне предложения. В счете клиента Azure Корпорация Майкрософт предоставляет лицензии на программное обеспечение издателя отдельно от базовой платы Azure. Стоимость использования инфраструктуры Azure зависит от использования программного обеспечения издателя.
- **Приложения Azure: шаблон решения или управляемое приложение** — необходимо подготавливать одну или несколько виртуальных машин и запрашивать сумму цен на виртуальную машину. Для управляемых приложений в одном плане можно выбрать ежемесячную подписку с фиксированным тарифом вместо оплаты по количеству виртуальных машин. В некоторых случаях плата за использование инфраструктуры Azure передается клиенту отдельно от платных лицензий на программное обеспечение, но в рамках одной и той же инструкции выставления счетов. Однако если вы настроили предложение управляемого приложения для оплаты инфраструктуры ISV, ресурсы Azure выставляются на издатель, а клиент получает фиксированный тариф, включающий в себя стоимость инфраструктуры, лицензий на программное обеспечение и служб управления.

# <a name="commercial-marketplace-transact-capabilities"></a>Коммерческие возможности сделки на рынке

В этой статье рассматриваются следующие темы, связанные с коммерцией для коммерческого рынка Майкрософт:

* Ответственность за транзакции для различных вариантов публикации
* Общие сведения о транзакциях
* Модели выставления счетов по транзакциям
* Требования к транзакциям

## <a name="transactions-by-publishing-option"></a>Транзакции путем публикации опциона

Либо издатель, либо корпорация Майкрософт отвечает за управление транзакциями по лицензиям на программное обеспечение для предложений на коммерческом рынке. Опция публикации, выбранная для вашего предложения, определит, кто управляет транзакцией. Просчитайте [сьм, чтобы определить сяокопию](./determine-your-listing-type.md#choose-a-publishing-option) и объяснения каждого варианта публикации.

### <a name="list-trial-and-byol-publishing-options"></a>Варианты публикации списка, пробной версии и публикации BYOL

Издатели с существующими возможностями коммерции могут выбрать список, пробную версию и принести варианты публикации собственной лицензии (BYOL) для рекламных целей и целей приобретения пользователей. С помощью этих опций корпорация Майкрософт не участвует непосредственно в транзакциях лицензий на программное обеспечение издателя, и плата за транзакции не взимается. Издатели несут ответственность за поддержку всех аспектов транзакции с лицензией на программу, в том числе заказа, исполнения, измерения, выставления счетов, оплаты и сбора. При публикации описаний и пробных версий издатели оставляют себе 100 % платы за лицензирование программного обеспечения, которая взимается с пользователя.

### <a name="transact-publishing-option"></a>Публикация транзакций

Опция публикации сделать сделку использует возможности коммерции Майкрософт и обеспечивает сквозной опыт от обнаружения и оценки до покупки и реализации. Предложения о транзакциях выставляются в отношении существующей подписки Майкрософт или кредитной карты, что позволяет корпорации Майкрософт размещать транзакции на облачном рынке от имени издателя.

При создании нового предложения в Partner Center вы выбираете опцию сделки. На странице **настройки предложения** под **сведениями о настройках**выберите :Да, я хотел бы продавать через корпорацию Майкрософт и иметь транзакции microsoft host от моего имени». Эта опция будет отображаться только в том случае, если для вашего предложения доступна сделать сделку.

## <a name="transact-general-overview"></a>Общие сведения о транзакциях

При использовании опции публикации сделки корпорация Майкрософт разрешает продажу стороннего программного обеспечения и развертывание некоторых типов предложений в подписке клиента Azure. Издатель должен учитывать выставление счетов за инфраструктуру и сборы издателя за лицензирование программного обеспечения при выборе модели выставления счетов и типа предложения.

Опция публикации Transact в настоящее время поддерживается для следующих типов предложений: Виртуальные машины, приложения Azure и приложения SaaS.

### <a name="billing-infrastructure-costs"></a>Выставление счетов за использование инфраструктуры

**Для виртуальных машин и приложений Azure**

Для виртуальных машин и приложений Azure плата за использование инфраструктуры Azure взимается за подписку клиента Azure. Плата за пользование инфраструктурой взимается и предоставляется отдельно от лицензионных сборов поставщика программного обеспечения по счету клиента.

**Для приложений SaaS**

В случае приложений SaaS издателю нужно рассматривать тарифы на использование инфраструктуры Azure и лицензирование программного обеспечения как одну позицию расходов.  Она представлена в виде фиксированной платы для клиента. Использование инфраструктуры Azure контролируется, а счета выставляются непосредственно партнеру. Фактическая стоимость использования инфраструктуры не видна пользователю. Как правило, издатели добавляют плату за использование инфраструктуры Azure в счета за лицензирование программного обеспечения. Плата за лицензирование программного обеспечения не измеряется и не взимается на основе потребления.

## <a name="transact-billing-models"></a>Модели выставления счетов по транзакциям

В зависимости от используемого опциона транзакции, лицензионные сборы издателя могут быть представлены следующим образом:  

* Бесплатно: плата за программные лицензии не взимается.

* С использованием собственной лицензии (BYOL): управление всеми применимыми выплатами за лицензии на программное обеспечение осуществляется непосредственно между издателем и пользователем. Корпорация Майкрософт обрабатывает только плату за использование инфраструктуры Azure. Это относится только к виртуальным машинам и приложениям Azure.

* С оплатой по мере использования: плата за лицензии на программное обеспечение взимается согласно почасовому тарифу с учетом количества ядер (виртуальных ЦП) и использования инфраструктуры Azure. Это относится только к виртуальным машинам и приложениям Azure.

* Цены на подписку: Лицензионные сборы за программное обеспечение представляются как ежемесячная или ежегодная, повторяющаяся плата, выставляемый в виде фиксированной ставки или за место. Это относится только к приложениям SaaS и приложениям Azure - Управляемым приложениям.

* Бесплатная пробная версия программного обеспечения: плата за 30- или 90-дневные лицензии на программы не взимается.

### <a name="free-and-bring-your-own-license-byol-pricing"></a>Предоставление решений бесплатно или с использованием собственной лицензии (BYOL)

При публикации предложений бесплатно или с использованием собственной лицензии корпорация Майкрософт не принимает участия в совершении транзакций по оплате лицензий на программное обеспечение. Как и при публикации описаний и пробных версий, издатель оставляет себе 100 % стоимости лицензий.

### <a name="pay-as-you-go-and-subscription-site-based-pricing"></a>Оплата по мере использования и тарифы на подписки (на основе сайтов)

При публикации предложений с оплатой по мере использования или по количеству подписок корпорация Майкрософт предоставляет технологии и услуги для обработки покупок и возвратов лицензий на программное обеспечение, а также возвратных платежей. В этом случае издатель разрешает корпорации Майкрософт действовать в качестве агента для этих целей. Издатель разрешает корпорации Майкрософт использовать транзакции по лицензированию программного обеспечения, в то же время сохранив свою роль продавца, поставщика, распространителя и лицензиара.

Корпорация Майкрософт позволяет клиентам заказывать, лицензировать и использовать программное обеспечение издателя, подчиняясь условиям как коммерческого рынка Майкрософт, так и лицензионного соглашения издателя. Издатели должны предоставить свое лицензионное соглашение с конечными пользователями или выбрать [Стандартный контракт](./standard-contract.md) при создании предложения.

### <a name="free-software-trials"></a>Бесплатные пробные версии

При публикации транзакций издатель может бесплатно предоставить лицензию на программное обеспечение на 30 или 90 дней. Это предложение не распространяется на стоимость использования инфраструктуры Azure, связанную с использованием партнерского решения.

### <a name="private-offers"></a>Частные предложения

В дополнение к использованию типов предложений и моделей выставления счетов для монетизации предложения, издатели могут заключать частное предложение, в комплекте с согласованными, конкретными ценами или пользовательскими конфигурациями. Частные предложения поддерживаются во всех 3 вариантах публикации.

Эта опция позволяет более высокую или более низкую цену, чем общедоступное предложение. Частные предложения могут содержать как скидки, так и премиум-содержимое. Частные предложения можно предоставлять одному или нескольким клиентам, добавляя их подписки Azure на уровне предложения.

### <a name="customer-invoicing-payment-billing-and-collections"></a>Выставление счетов, оплата и сбор средств у клиента

**Выставление счетов и оплата**

Издатель может использовать предпочтительный метод оценивания платежей клиента для доставки подписок или лицензионных сборов за программное обеспечение PAYGO.

**Корпоративное соглашение** 

Если предпочтительным методом выставления счетов-счетов клиента является корпоративное соглашение майкрософт, плата за лицензию на программное обеспечение будет выставлена с использованием этого метода выставления счетов в качестве детализированной стоимости, отдельно от любых затрат на использование, связанных с Azure.

**Кредитные карты и ежемесячные накладные** 

Клиенты также могут оплачивать решения с помощью кредитных карт и ежемесячных накладных. В этом случае стоимость лицензий на программное обеспечение взимается точно так же, как и при использовании Соглашения Enterprise, в виде постатейной суммы отдельно от платы за использование Azure.

**Бесплатные кредиты и денежные обязательства** 

Некоторые клиенты предварительно оплачивают Azure с помощью денежного обязательства в Соглашении Enterprise или получают бесплатные кредиты для использования с Azure. Эти кредиты можно использовать для оплаты использования Azure, но они не подходят для оплаты лицензий на программное обеспечение издателя.

**Выставление счетов и сборы** 

Счета за лицензии на программное обеспечение издателя предоставляются при помощи выбранного клиентом способа в соответствии с расписанием выставления счетов. Клиенты без Соглашения Enterprise ежемесячно получают счета за лицензии на программное обеспечение из Marketplace. Клиенты с Соглашением Enterprise ежемесячно оплачивают счета через накладную, представляемую раз в квартал.

Если выбрана ценовая модель на основе подписок или с оплатой по мере использования, то корпорация Майкрософт выступает в качестве агента издателя и отвечает за все аспекты выставления счетов, оплаты и сбора.

### <a name="publisher-payout-and-reporting"></a>Отчеты и выплаты издателя

* Из платы за лицензирование программного обеспечения, которую собирает корпорация Майкрософт, вычитается операционный сбор в размере 20 % (если не указано иное) во время выплаты издателю.

* Как правило, клиенты покупают решения при помощи Соглашения Enterprise или соглашения об оплате по мере использования с поддержкой кредитных карт. Тип соглашения определяет расписание выставления счетов, сбора и выплат.

## <a name="transact-requirements"></a>Требования к транзакциям

В этом разделе описываются требования к транзакциям для предложений различных типов.

### <a name="requirements-for-all-offer-types"></a>Требования для предложений всех типов

* Учетная запись майкрософт и финансовая информация необходимы для сделки публикации вариант, независимо от модели ценообразования предложения.
* Обязательная финансовая информация включает в себя счет выплат и налоговый профиль.

Для получения дополнительной информации о настройке этих учетных записей, [см. Управление вашей коммерческой учетной записи рынка в Partner Center](https://docs.microsoft.com/azure/marketplace/partner-center-portal/manage-account)).

### <a name="requirements-for-specific-offer-types"></a>Требования для предложений определенных типов

Публикация транзакций доступна для использования только с предложениями Marketplace следующих типов:

**Виртуальная машина** 

Выберите ценовую модель с бесплатным использованием, с использованием собственной лицензии или с оплатой по мере использования и представьте номера SKU, определенные на уровне предложения. В законопроекте клиента Azure корпорация Майкрософт представляет лицензионные сборы издателя программного обеспечения отдельно от базовых сборов за инфраструктуру Azure. Стоимость использования инфраструктуры Azure зависит от использования программного обеспечения издателя.

**Приложения Azure: шаблон решения или управляемое приложение** 

Необходимо подготовить одну или несколько виртуальных машин и посчитать сумму тарифов за их использование. Для управляемых приложений в одном плане можно выбрать ежемесячную подписку с фиксированным тарифом вместо оплаты по количеству виртуальных машин. В некоторых случаях плата за использование инфраструктуры Azure передается клиенту отдельно от лицензионных сборов программного обеспечения, но на той же выписке по биллингу. Однако, если настроить управляемое приложение, предлагающее для инфраструктуры ISV, ресурсы Azure выставляются издателю, и клиент получает фиксированную плату, которая включает стоимость инфраструктуры, лицензий на программное обеспечение и услуги по управлению.

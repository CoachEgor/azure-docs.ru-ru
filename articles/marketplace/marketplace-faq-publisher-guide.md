# <a name="microsoft-commercial-marketplace-faqs"></a>Вопросы и ответы о коммерческой платформе Майкрософт

Ответы на общие вопросы о коммерческом рынке.

## <a name="faq-for-customers"></a>Вопросы и ответы для клиентов

### <a name="what-you-need-to-know-about-the-commercial-marketplace"></a>Что нужно знать о коммерческом рынке

**Что такое Azure Marketplace?**

[Azure Marketplace](https://azuremarketplace.microsoft.com/marketplace) предоставляет доступ и информацию о доступных решениях и службах от корпорации Майкрософт и наших партнеров. Клиенты могут обнаружить, попробовать или купить облачные программные решения, встроенные в Azure или для нее. Наш каталог, состоящий из более чем 8000 записей, предоставляет стандартные блоки Azure, такие как виртуальные машины, API, приложения Azure, шаблоны решений и управляемые приложения, приложения SaaS, контейнеры и консультационные услуги.

**Кем являются клиенты Azure Marketplace?**

Azure Marketplace предназначен для ИТ-специалистов и облачных разработчиков, заинтересованных в коммерческом ИТ-программном обеспечении и услугах.

**Какие типы продуктов доступны в Azure Marketplace сейчас?**

Azure Marketplace предлагает технические решения и службы корпорации Майкрософт и ее партнеров, созданные для расширения номенклатуры продуктов и служб Azure. Каталог решений охватывает несколько категорий, в ключая:

* базовая операционная система;
* базы данных
* security
* identity
* networking
* блокчейн;
* средства разработчика

### <a name="azure-marketplace-for-customers"></a>Azure Marketplace для клиентов

**Как начать работу в Azure Marketplace?**

Вы найдете самые разнообразные корпоративные приложения и решения, сертифицированные и оптимизированные для работы в Azure, посетив [Azure Marketplace](https://azuremarketplace.microsoft.com/marketplace/apps). Доступ к Azure Marketplace также можно получить через портал «Azure» при [**создании ресурса.**](https://portal.azure.com/#create/hub)

**Каковы основные преимущества использования Azure Marketplace?**

Благодаря Azure Marketplace клиенты могут обнаруживать технические приложения, созданные для платформы Azure или на ее основе. Там рынок решений и служб Microsoft Azure объединен в единую унифицированную платформу для изучения, пробного использования, приобретения или развертывания решений всего несколькими щелчками мыши.

**Как можно купить продукты, предлагаемые в Azure Marketplace?**

Предложения Azure Marketplace можно приобрести с помощью:

* [Веб-магазин](https://azuremarketplace.microsoft.com/marketplace/apps)
* (Портал Azure) (Портал Azure)
* [Интерфейс командной строки Azure (CLI)](/cli/azure/?view=azure-cli-latest)

>[!Note]
>Предоплаченные деньги на счете и другие формы денежных обязательств невозможно использовать для оплаты стоимости лицензий на программное обеспечение, хотя их можно использовать для оплаты услуг Azure. Исключения перечислены в [денежных обязательствах Azure](https://azure.microsoft.com/updates/azure-marketplace-third-party-reseller-services-now-use-azure-monetary-commitment/).

**Могу ли я выбрать, в каком регионе Azure (ы) развернуть покупку Azure Marketplace?**

Клиент имеет возможность развертывания в любом регионе центра обработки данных Azure, который они включат. Мы рекомендуем выбрать ближайшее к службе расположение центра обработки данных для оптимизации производительности и управления бюджетом.

**Если я случайно удалю покупку Azure Marketplace, могу ли я *отменить* действие?**

Нет, удаление отменить нельзя. Если подписка случайно удалена, ее можно купить повторно. Любые неиспользованные функции или предоплаченные службы будут утеряны.

**Получу ли я предупреждение при попытке удалить покупку из Azure Marketplace, которая используется одним из моих приложений?**

Нет, Azure не предоставляет предупреждений при удалении покупки, даже если она используется в настоящее время или от него зависит приложение.

**Если у приложения из магазина Azure Marketplace имеются зависимости от других платформ, таких как веб-сайты Azure, нужно ли будет управлять ими?**

Зависимости для предложений магазина Azure Marketplace не управляются автоматически. Тщательно изучите описание покупки в магазине Azure Marketplace и узнайте о необходимых зависимостях до развертывания решения.

**Можно ли приобрести решения Azure Marketplace у поставщика облачных решений Azure?**

Если издатель настроил свое предложение, чтобы быть доступным через канал Cloud Solution Provider (CSP), партнеры-поставщики облачных решений имеют возможность перепродать решение.

**Какие страны/регионы получают поддержку при покупке приложений и услуг, продаваемых/предоставляемых через Azure Marketplace?**

Azure Marketplace доступен клиентам Azure в странах/регионах, перечисленных в [политике участия.](/legal/marketplace/participation-policy)

**Какие валюты поддерживает Azure Marketplace?**

Сделки могут проводиться в следующих 17 валютах: AUD, BRL, CAD, CHF, DKK, EUR, GBP, INR, JPY, KRW, NOK, НЗД, RUB, SEK, TWD, USD, юаня.

### <a name="deploying-a-solution-from-azure-marketplace"></a>Развертывание решения Azure Marketplace

**Я развернул виртуальную машину Azure Marketplace (VM) в подписку, и теперь я хочу перенести подписку с одной учетной записи Azure на другую. Поддерживается ли это в настоящее время?**

Для переноса подписки Azure, включая виртуальные машины Azure Marketplace и службы, удалите или отмените любые предыдущие подписки Azure прежде, чем связываться с новой учетной записью Azure. После завершения миграции взимается плата за использование с использованием метода оплаты нового зарегистрированного счета.

**Я хочу перенести подписку на виртуальную машину Azure Marketplace (VM) в мое корпоративное соглашение. Поддерживается ли это в настоящее время?**

Чтобы перенести подписку виртуальной машины Azure Marketplace в Соглашение Enterprise Agreement, остановите или отмените любую предыдущую подписку перед миграцией. После завершения миграции учетной записи Azure и связанных с ней подписок можно выкупить VM или сервис Azure Marketplace.  Плата за пользование выставляется ежеквартально, в рамках Соглашения Enterprise.

### <a name="pricing-and-payment"></a>Цены и варианты оплаты

**Каким образом формируется цена на подписки Azure Marketplace?**

Ценовая политика меняется в зависимости от типов продуктов и характеристик, заявленных издателями. Отчисления по лицензии на программное обеспечение и стоимость услуг Azure взимаются раздельно согласно подписке Azure.

*Отдельно.*

+ *Принесите-ваш-собственный-лицензии (BYOL) модель*: При получении лицензии на программное обеспечение непосредственно от издателя или реселлера, Есть никаких дополнительных связанных с программным обеспечением сборов или сборов.

*Объединено.*

Подписка Azure включена в систему независимых поставщиков программного обеспечения (ISV) издателя.

*Взимается:*

+ *Бесплатно:* Бесплатный SKU. Плата за лицензию на программное обеспечение или использование предложения не взимается.

+ *Пробная версия свободного программного обеспечения:* Предложение, которое является бесплатным в течение ограниченного периода времени. Плата за использование лицензии на программное обеспечение издателя не взимается в течение испытательного периода. После истечения срока действия оно автоматически превращается в платное приложение, основанное на стандартных тарифах издателя.

+ *Использование на основе:* Тарифы взимаются или выставляются в зависимости от степени использования предложения. Для образов виртуальных машин взимается почасовая плата. Для служб разработчиков и API с вас взимается плата за каждую единицу измерения в соответствии с предложением.

+ *Плоская плата:* SaaS подписки могут быть оценены как фиксированная плата, которая выставляется ежемесячно или ежегодно. Это также может включать дополнительные размеры биллинга, которые взимают плату в зависимости от потребления (например, пропускная способность, электронная почта или билеты). 

+ *На пользователя:* SaaS подписки могут быть оценены на основе на одного пользователя, который выставлен счет ежемесячно или ежегодно. 

Детали цен, оцениваемые в [https://azure.microsoft.com/pricing/](https://azure.microsoft.com/pricing/) конкретных предложениях, можно найти на странице подробной информации о решении в «портале Azure».

> [!Note]
> Помимо ежемесячных отчислений, плата за использование Azure касается также других моделей ценообразования, если иное не оговорено отдельно.

**Как я должен предоставить свой ключ лицензии программного обеспечения для решений marketplace с использованием собственной лицензии и какую роль играет Azure Marketplace?**

За получение и применение учетных данных лицензии для решений с использованием собственной лицензии отвечает издатель. Для предложений Virtual Machine приобретение ключа лицензии обычно происходит в приложении издателя после запуска приложения. При использовании предложения Virtual Machine, развернутого с помощью шаблона решений Azure Application, шаблон ресурсного менеджера может быть настроен для того, чтобы предложить пользователю ряд входов, включая учетные данные лицензий.

Это наиболее распространенные параметры каждого типа предложения:

*Предложение виртуальной машины:*

+ *Вариант 1:* Приобретение ключа лицензии обычно происходит в приложении издателя после начала приложения.

+ *Вариант 2:* Ключ лицензии вводится конечным пользователем (через командную линию / веб-интерфейс, предоставляемый предложением) после развертывания предложения VM в выбранной подписке. Лицензия может быть ключом и/или файлом, определенным издателем.

*Приложение Azure (шаблон приложения и управляемые приложения).*

+ *Вариант 1:* Шаблон менеджера ресурсов можно настроить, чтобы подсказать вам ряд входных данных, включая учетные данные лицензий. Это можно сделать в виде файла лицензии (отправка файла) или ключа (входные данные текстового поля) перед развертыванием предложения в подписке пользователя.

+ *Вариант 2:* Вы можете ввести ключ лицензии через командную линию / веб-интерфейс, предоставляемый предложением.  Это делается после развертывания предложения приложения Azure в выбранной подписке. Лицензия может быть ключом и/или файлом, определенным издателем.

**Какие испытания поддерживаются?**

Издатели могут добавить один бесплатный месяц для платных предложений SaaS и один или три свободных месяца потребления для VM изображений. Бесплатные пробные предложения - это объявления с призывом к действию для инициирования пробной версии. Они приводят клиента к веб-сайту, определяемому издателем для настройки пробного опыта. Испытания также могут быть добавлены к платным предложениям, где первый месяц бесплатный. 

**Нужно ли зарегистрированное платежное средство (например, кредитная карта) для развертывания бесплатных приложений или предложений с использованием собственной лицензии?**

Нет. Для развертывания предложений уровня "Бесплатный" или предложений с использованием собственной лицензии платежное средство не требуется. Однако для развертывания предложений бесплатной пробной версии платежное средство необходимо указать. Списки, включающие кнопки **Получить сейчас** или **Бесплатная пробная версия**, развертываются в выбранную подписку Azure.  Эти списки выставляются с использованием зарегистрированного метода оплаты выбранного счета. Плата за использование Azure взимается отдельно от платы за использование лицензии на ПО.

**С кем может связаться заказчик с непрямым Соглашением Enterprise, если у него есть вопросы о ценах на предложения, реализуемые через магазин Azure Marketplace?**

Заказчики с непрямыми Соглашениями Enterprise должны связаться со своим поставщиком решений по лицензированию (LSP) по всем вопросам, связанным с ценами магазина Azure Marketplace.

**Можно ли управлять доступом сотрудников в Azure Marketplace и правами на приобретение?**

Да, для клиентов Соглашения Enterprise администратор регистрации может отключить привилегии приобретения для всех учетных записей и включить их обратно на достаточно долгий срок для выполнения приобретения.

**Какие способы оплаты поддерживаются при покупке на коммерческом рынке?**

Клиенты могут приобрести предложения с коммерческого рынка с помощью кредитных карт. Если у вас есть существующая подписка Azure, покупки в Azure Marketplace будут использовать метод оплаты, настроенный на учетную запись, и отображаться на том же счете-фактуре, что и отдельный элемент строки. Некоторые предложения потребляют денежные обязательства Azure, но большинство коммерческих покупок на рынке не стягивают обязательства по корпоративному соглашению, хотя потребление инфраструктуры Azure будет.

**Можно ли использовать кредиты подписки Azure или денежные средства в моей учетной записи для предложений в Azure Marketplace?**

Для определенных предложений Azure Marketplace предусмотрено использование средств на счете в рамках подписок Azure или обязательств по оплате. Полный список продуктов, охватываемых этой программой, приведен в [этой статье](https://azure.microsoft.com/updates/azure-marketplace-third-party-reseller-services-now-use-azure-monetary-commitment/). Эти предложения не включают варианты с использованием собственной лицензии и BYOS. Все остальные предложения Azure Marketplace не могут использовать кредиты на подписку Azure или денежные обязательства: например, бесплатный месячный пробный кредит, ежемесячные кредиты MSDN, кредиты от акции Azure, остатки денежных обязательств и любые другие бесплатные кредиты, предоставляемые Azure.

**Применимы ли скидки, действующие для корпоративного лицензирования, к покупкам в магазине Azure Marketplace?**

Нет. Издатель, которому принадлежит решение в Azure Marketplace, может задать цены.  Стандартные скидки на корпоративные лицензии Майкрософт не применяются к предложениям магазина Azure Marketplace.

**Где найти подробные сведения о подписке Azure Marketplace и информацию о выставлении счетов?**

КЛИЕНТы MOSP - [Клиенты программы подписки на веб-страницы Майкрософт](https://azure.microsoft.com/support/legal/subscription-agreement/?country=us&language=en) (веб-прямые) могут просматривать сведения о покупке Marketplace во вкладке "Управление затратами" и разделом "Портал Azure".

Клиенты Enterprise Agreement (EA) могут просматривать сведения о покупке Marketplace во вкладке «Управление затратами» и разделе «Портал затрат».

Партнеры-поставщики облачных решений (CSP) могут просматривать сведения о покупке Marketplace для каждого клиента в Центре-партнере в рамках вкладки «История заказов» после выбора компании в представлении Клиентов.

**Как мне отменить надстройку Azure Marketplace к виртуальной машине Azure?**

Так как эта надстройка связана с виртуальной машиной Azure, чтобы отменить покупку в Azure Marketplace, необходимо остановить виртуальную машину, удалив ее. Таким образом прекращается использование всех подписок и сборов при покупке Azure Marketplace.

**Как часто выставляется счет за покупки в магазине Azure Marketplace?**

Для всех предложений Azure Marketplace, для которых не предусмотрены обязательства по оплате, счета будут выставляться ежемесячно по итогам использования. Ежегодные подписки SaaS выставляются один раз за услуги за полный год.

MOSP - [Клиенты программы подписки Microsoft Online](https://azure.microsoft.com/support/legal/subscription-agreement/) (веб-прямые) ежемесячно списываются с той же кредитной карты, которая находится в файле для их профиля подписки Azure. Ежегодные подписки SaaS выставляются один раз за услуги за полный год.

**Как можно перенести мои покупки в Azure Marketplace из подписки MOSP в подписку с прямым Соглашением Enterprise (EA)?**

Несмотря на то что большинство подписок Майкрософт можно легко включить в Соглашение Enterprise, с покупками в Azure Marketplace этого сделать нельзя.

Для переноса других служб, приобретенных в Azure Marketplace для подписки EA, сначала следует удалить из имеющейся подписки MOSP и заново приобрести эти предложения в рамках этой подписки. После этого можно отправить запрос на возмещение в течение потенциального месяца перекрытия между подписками служб Marketplace. Для этого создайте [запрос в службу поддержки](https://portal.azure.com/#blade/Microsoft_Azure_Support/HelpAndSupportBlade/overview).

**В чем разница между *ценой,* *ценой программного обеспечения*и *общей ценой* в структуре затрат на предложения виртуальной машины в Azure Marketplace?**

*Цена* относится к стоимости Виртуальной машины Azure для запуска программного обеспечения. *Цена программного обеспечения* относится к стоимости программного обеспечения издателя Marketplace, работаемого на виртуальной машине Azure. *Общая цена* относится к общей общей стоимости виртуальной машины Azure и программного обеспечения издателя Marketplace, работаемого на виртуальной машине Azure.

**Как выяснить, какая часть покупок в Azure Marketplace была мной использована?**

Предполагаемую информацию об использовании можно найти на портале «Azure». Такие сведения могут не включать последнее действие и основываться на предполагаемых показателях, полученных во время предыдущего использования. В течение действия предварительной версии этот вариант может не распространяться на покупку всех продуктов и зависеть от типа продукта.

### <a name="customer-support"></a>Поддержка клиентов

**С кем можно связаться по общим вопросам поддержки в Azure Marketplace?**

По вопросам общей поддержки приложения в процессе использования или устранения неполадок обращайтесь непосредственно к издателю.

**Если я не удовлетворен, могу ли я вернуть покупку?**

Покупки, сделанные в [Azure Marketplace,](https://azuremarketplace.microsoft.com/) не могут быть возвращены, но могут быть отменены/удалены. Предложения на основе потребления выставляются в соответствии с использованием, поэтому, когда он останавливается, сборы также прекращаются. Подписка отменяется и не будет выставленсчет авотировки за текущий период выставления счетов. Если подписка отменяется вскоре после покупки (24 часа в месяц и 14 дней в год), предоставляется полный возврат средств.

Клиенты должны непосредственно связываться с издателем по любым техническим вопросам относительно их службы Marketplace или покупки. Контактные данные издателя или ссылку на страницу поддержки можно найти на странице с подробными сведениями о решении в Azure Marketplace.

**Как обрабатываются дополнения лицензии при увеличении среднесрочных полномочий?**

Лицензии, добавленные к существующей подписке, проражаются на оставшуюся часть срока действия подписки.

**Как обрабатываются удаления лицензий в среднесрочной перспективе?**

Лицензии, которые будут аннулированы, будут подлежать политике возврата средств, найденной в этом часто задаваемых вопросах.  Все аннулированные лицензии немедленно удаляются из вашей учетной записи и больше не будут доступны для использования.

**Поддерживаются ли возвраты предложений на основе потребления?**

Любые сборы, основанные на потреблении - будь то почасовые ВМ или пользовательские счетчики - не подлежат возврату путем отмены. Как только потребление произошло, сборы обрабатываются торговой платформой Майкрософт, и любые споры о качестве обслуживания, когда запрашивается возврат средств, будут обрабатываться за пределами систем Майкрософт между издателем и клиентом напрямую. Предложения, которые поддерживают фиксированную ставку плюс дозированные счета следует стандартной политике возврата для фиксированной ставки сборов.

**Можно ли изменить планы в среднесрочной перспективе?**

Нет, переход между месячным и годовым переходом невозможен.

**Может ли клиент приобрести два плана из одного и того же предложения?**

Да, клиент может владеть двумя планами из одного и того же предложения одновременно.

**Различаются ли политики возврата и изменения плана в зависимости от магазина?**

Нет, бизнес-политики согласованы на коммерческом рынке. При перепродаже партнером программы Cloud Solution Provide партнер может применять другую политику для своих клиентов.

Azure Marketplace предназначен для ИТ-специалистов и разработчиков облачных решений, заинтересованных в коммерческом программном обеспечении для ИТ и службах.

**Зачем публиковать приложение в Azure Marketplace и какую пользу это принесет мне?**

Azure Marketplace — это магазин, в котором партнеры корпорации Майкрософт могут продвигать и продавать продукцию и службы клиентам Azure. Издатели мгновенно получают доступ к 140 глобальным рынкам, нашим 300 000 партнерам и сети корпоративных клиентов Azure.  Рынок включает в себя более 90% компаний из списка Fortune 500 и многих ведущих мировых разработчиков. **В чем разница между онлайн-магазинами Azure Marketplace и AppSource?**

Партнеры корпорации Майкрософт могут выбрать место публикации в зависимости от целевой аудитории.

Корпорация Майкрософт предоставляет два различных витрины облачных рынков - Azure Marketplace и AppSource. В этих магазинах клиенты могут подобрать, опробовать и приобрести облачные приложения и службы. Каждый онлайн-магазин удовлетворяет уникальные потребности клиентов и позволяет партнерам корпорации Майкрософт ориентировать свои решения или службы на целевую аудиторию.

Выберите [Azure Marketplace](https://azuremarketplace.microsoft.com/marketplace/apps), если ваша целевая аудитория — ИТ-специалисты и разработчики или пользователи с профессиональной технической подготовкой.

Выберите [AppSource](https://appsource.microsoft.com/), если ваша целевая аудитория — лица, принимающие решения относительно бизнес-предложений, и владельцы предприятий.

Подробные сведения и преимущества использования Marketplace и AppSource см. в [руководстве издателя](marketplace-publishers-guide.md).

### <a name="billing-and-payments"></a>Тарификация и оплата

**Как часто я буду получать выплаты за продажи магазина Azure Marketplace?**

Все выплаты от корпорации Майкрософт будут осуществляться ежемесячно через PayPal или систему электронных платежей (EFT). Оплата производится в течение двух месяцев после предоставления услуги клиенту, но конкретные сроки зависят еще и от того, каким финансовым инструментом пользуется клиент. Клиентам, пользующимся кредитными картами, для оплаты услуг предоставляется отсрочка в 45 дней.

**Будет ли цена на лицензию на использование программного обеспечения изменяться, если в решении используется виртуальная машина, приобретаемая на условиях оплаты по факту использования, при увеличении или уменьшении размера виртуальной машины?**

Да, новая цена будет немедленно тарифицирована.  Изменения в ценообразовании происходят, если клиент изменяет размер виртуальной машины и указывает в таблице с ценами различные цены, которые основаны на размере виртуальных машин.

**Доступно ли выставление счетов за каждый узел в Azure Marketplace?**

Azure Marketplace сейчас не поддерживает поузловую тарификацию для виртуальных машин. Издатели могут по-прежнему определять тарифы для каждого узла по тарифам виртуальных машин корпорации Майкрософт.  Расчет заключается в определении количества виртуальных машин по количеству использованных часов и скорости в час.

**К кому можно обратиться по вопросам тарификации с предложениями или административным вопросом?**

Отправьте запрос в [службу поддержки Майкрософт](https://support.microsoft.com/getsupport?oaspworkflow=start_1.0.0.0&wf=0&wfName=productselection&prid=15635).

**Как определить географическую доступность, чтобы обеспечить продажи в разных странах/регионах?**

1.  В Partner Center перейдите к SKU, к которому вы хотите добавить новые страны/регионы.  В **sKU Подробная информация**, перейдите к **стране / региону доступности** и нажмите **Выберите регионы**.

    ![Выбор регионов](media/marketplace-publishers-guide/FAQ-choose-geo.png)

1.  Список будет всплывающее со всеми доступными странами / регионами, чтобы продать.  Нажмите на флажок рядом с каждой страной/регионом, который вы хотите сделать доступным для этого SKU. Нажмите кнопку **ОК**.

    ![Выберите список стран/регионов](media/marketplace-publishers-guide/FAQ-select-countries.png)

1.  Наконец, для изменений, которые будут применяться к вашему живому предложению, нажмите **Опубликовать**.

**Как издатель может изменить географическую доступность существующего предложения?**

Издатель может отредактировать существующее предложение, выбрать новые страны/регионы и использовать функцию загрузки/загрузки электронных таблиц для установления цен.

**В каких странах/регионах клиенты могут приобрести предложения Azure Marketplace?**

Azure Marketplace поддерживает 141 географию покупки, как это определено по платежному адресу клиента. Ознакомьтесь с [политикой участия](/legal/marketplace/participation-policy) в списке стран/регионов.

### <a name="pricing-and-payment"></a>Цены и варианты оплаты

**Чем отличается бесплатный уровень и бесплатная пробная версия программного обеспечения?**

Предложение подписки уровня "Бесплатный" предоставляется бесплатно.  Предложение "Бесплатная пробная версия программного обеспечения (Попробуйте его сейчас)" является платной подпиской и бесплатно только в течение ограниченного периода времени.

**Каков процесс проверки потока до конца покупки и подготовки?**

Во время процесса публикации вам будет предоставлен доступ к предварительному просмотру вашего предложения. Доступ ограничен пользователями, указанными во вкладке Preview, и это живое предложение, которое не видно никому другому. Вы можете приобрести это и протестировать процесс; однако с вас будет взиматься полная сумма в соответствии с конфигурацией вашего предложения.

Чтобы завершить покупку по очень низкой цене, корпорация Майкрософт предлагает опубликовать частный план для себя, установленный по цене, которую вы можете принять в качестве стоимости тестирования. Нулевой поддерживается, но не будет отражать полный опыт, как платная подписка.

**Будет ли корпорация Майкрософт предоставлять возмещение вне стандартных политик?**

Да, по вашему запросу через поддержку билет, Microsoft будет обрабатывать кредиты для вашего клиента, если вы сочтете это целесообразным.

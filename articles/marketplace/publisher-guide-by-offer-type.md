# <a name="publishing-guide-by-offer-type"></a>Руководство по публикации разных типов предложений

Выбрав [вариант публикации](https://docs.microsoft.com/azure/marketplace/determine-your-listing-type), вы можете выбрать тип предложения, который будет использоваться для его представления.

 *Тип предложения* определяет структуру предложения, включая метаданные, артефакты и другое содержимое, используемое для представления предложения в Marketplace.

Перед созданием предложения необходимо выбрать тип предложения. Тип предложения будет соответствовать типу предложения решения, приложения или службы, которое вы собираетесь опубликовать, а также соответствующим продуктам и службам Майкрософт.

Один тип предложения может быть настроен по-разному, чтобы включить различные варианты публикации, вызовы к действию, подготовку или ценообразование. Конфигурация и вариант публикации типа предложения также определяются техническими требованиями и соответствием предложения.

Перед созданием предложения обязательно ознакомьтесь с требованиями к соответствию онлайн-магазина и типа предложения, а также техническими требованиями к публикации.

## <a name="list-of-offer-types"></a>Список типов предложений

В следующей таблице перечислены типы предложений Azure Marketplace.

| **Тип предложения**    | **Описание**  |
| :------------------- | :-------------------|
| [**Виртуальные машины**](./marketplace-virtual-machines.md) | Тип предложения "Виртуальная машина" используется при развертывании виртуального модуля в подписке, связанной с пользователем. |
| [**Шаблоны решений**](./marketplace-solution-templates.md) | Используйте шаблон решения (также известный как приложение Azure) типа, когда ваше решение требует дополнительного развертывания и автоматизации конфигурации за пределами одного VM. Шаблоны решений могут использовать множество различных видов ресурсов Azure, включая, но не ограничиваемые ими.  |
| [**Управляемые приложения**](./marketplace-managed-apps.md) | Тип предложения "Приложение Azure: управляемое приложение" используется в описанных ниже условиях. <br> <ul> <li> Вы развертываете решение на основе подписки для пользователя, применяя либо виртуальную машину, либо целое решение на основе IaaS. </li> <li>Вам или вашему клиенту требуется, чтобы решением управлял партнер. </li> <ul> |
| [**Приложения SaaS**](./marketplace-saas-applications-technical-publishing-guide.md) | Используйте тип предложения "Приложение SaaS", чтобы предоставить пользователю возможность приобрести ваше техническое решение на основе SaaS в виде подписки. |
| [**Предложения контейнеров**](./marketplace-containers.md) | Используйте тип предложения "Контейнер", если ваше решение представляет собой образ контейнера Docker, подготовленный в виде службы контейнеров Azure на основе Kubernetes. |
| [**Модули Azure IoT Edge**](./iot-edge-module.md) | Модули Azure IoT Edge — это наименьшая единица вычислительных ресурсов, которыми управляет IoT Edge. Эти модули могут содержать службы Майкрософт (например, Azure Stream Analytics), сторонние службы или пользовательский код для конкретного решения. |
| [**Консультационные услуги**](./consulting-services.md) | Консультационные услуги помогают подключить клиентов к услугам поддержки и расширения использования сервисов Azure, Dynamics 365 или Power Suite.|
| [**Office 365, Dynamics 365 и Power BI**](./appsource-offer-publishing-guide.md) | Вы можете публиковать предложения AppSource, которые основывается или расширяют Dynamics 365, Office 365, Power BI и Power Apps.|
| [**Интегрированные решения**](./integrated-solutions-for-publishers.md) | Вы можете публиковать интегрированные, отраслевые решения, которые объединяют технологии и услуги в качестве единого предложения.|

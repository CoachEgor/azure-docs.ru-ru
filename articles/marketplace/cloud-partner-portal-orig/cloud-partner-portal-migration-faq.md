# <a name="frequently-asked-questions-for-migrating-from-the-cloud-partner-portal-to-partner-center"></a>Часто задаваемые вопросы о переходе с Портал Cloud Partner в центр партнеров

В этой статье рассматриваются часто задаваемые вопросы о миграции предложений с Портал Cloud Partner в центр партнеров.

## <a name="what-does-offer-migration-mean"></a>Что означает миграция предложений?

Мы переносите данные предложения из Портал Cloud Partner в центр партнеров с изменениями в интерфейсе публикации и управления предложениями.

| Область  | Изменения  |
|-------|----------|
| **Возможности управления публикациями и предложениями** | Благодаря интуитивному интерфейсу в центре партнеров вы сможете улучшить взаимодействие с пользователем. Дополнительные сведения см. в разделе [что такое различия между центром партнеров и портал Cloud Partner?](#what-are-the-differences-between-partner-center-and-the-cloud-partner-portal) |
| **Доступность ваших предложений в Marketplace** | Без изменений. Если ваше предложение действительно в Marketplace, оно будет продолжать работать во время и после завершения миграции. |
| **Новые покупки и развертывания** | Без изменений. Ваши клиенты продолжат покупать и развертывать предложения без перерывов. |
| **выплаты;** | Все покупки и развертывания, происходящие во время или после миграции, будут по своему счету выставляться в нормальном режиме. |
|**Интеграция API с существующими [портал Cloud Partner API](https://docs.microsoft.com/azure/marketplace/cloud-partner-portal-orig/cloud-partner-portal-api-overview)** | Существующие API Портал Cloud Partner продолжат поддерживаться после миграции, и существующие интеграции будут продолжать работать. Дополнительные сведения см [. в разделе портал Cloud Partner API-интерфейсы RESTful, которые будут поддерживаться после миграции?](#will-the-cloud-partner-portal-rest-apis-be-supported-post-migration) |
| | |

## <a name="can-i-still-access-the-cloud-partner-portal-and-manage-my-offers-during-migration"></a>Можно ли по-прежнему получать доступ к Портал Cloud Partner и управлять моими предложениями во время миграции?

Предложения будут перенесены в центр партнеров, начиная с 13 апреля по всему месяцу. В течение этого периода времени вы сможете получить доступ к Портал Cloud Partner как обычно, за исключением того момента, когда будет запланирована миграция.
При переносе ваших предложений они будут иметь состояние "заблокировано — перемещено в центр партнеров", как показано на следующем снимке экрана. Этот период миграции должен быть менее 24 часов. В течение этого времени вы не сможете вносить какие бы то ни было изменения в ваши предложения. После завершения переноса ваших предложений вы получите уведомление по электронной почте.

:::image type="content" source="media/migration-faq/all-offers-1.png" alt-text="Показывает состояние перенесенных предложений.":::

После переноса предложений они будут находиться в режиме только для чтения в **течение ограниченного периода времени** в портал Cloud Partner. Их состояние будет показывать "перемещено в центр партнеров" и содержать ссылку на Ваше предложение в центре партнеров, как показано на следующих снимках экрана. С этого момента вы будете управлять обновлениями всех ваших предложений или создавать новые предложения исключительно через центр партнеров,

:::image type="content" source="media/migration-faq/all-offers-2.png" alt-text="Показывает сообщение, предоставленное для предложений, которые были перенесены в центр партнеров":::

:::image type="content" source="media/migration-faq/offer-has-moved.png" alt-text="Иллюстрирует Портал Cloud Partner страницу для перенесенного предложения.":::

## <a name="do-i-need-to-create-a-new-account-to-manage-offers-in-partner-center"></a>Нужно ли создавать новую учетную запись для управления предложениями в центре партнеров?

Нет. Ваша Базовая учетная запись будет сохранена, и вы уже должны управлять ею в центре партнеров. Это означает, что если вы являетесь партнером, вы можете использовать существующие учетные данные Портал Cloud Partner учетной записи для входа в центр партнеров после миграции. Только предложения и связанные с ними возможности управления перемещаются из Портал Cloud Partner в центр партнеров. Мы просим вас не создавать новые учетные записи, так как ваши предложения не будут связаны с новой учетной записью.

## <a name="i-see-a-message-in-the-cloud-partner-portal-to-activate-my-account-what-does-this-mean"></a>Я вижу сообщение в Портал Cloud Partner для активации учетной записи, что это значит?

Для правильного переноса вашей учетной записи в центр партнеров требуется еще несколько сведений, чтобы вы могли управлять предложениями в центре партнеров после завершения миграции предложения. Дополнительные сведения об активации учетной записи см. в статье [Миграция учетных записей из портал Cloud Partner в центр партнеров](https://docs.microsoft.com/azure/marketplace/partner-center-portal/account-migration-from-cpp-to-pc).

Шаги, необходимые для выполнения активации учетной записи, зависят от роли вашей учетной записи.

| Роль учетной записи | Процедура активации |
|--------------|----------------|
|Владелец | Перейдите на страницу [профиля издателя](https://cloudpartner.azure.com/#profile) в портал Cloud Partner и щелкните ссылку в баннере, чтобы активировать ее. Вы будете перенаправлены в центр партнеров, чтобы завершить активацию учетной записи. |
| Участник | Только пользователь в учетной записи с ролью владельца может активировать учетную запись. Свяжитесь с владельцами учетной записи, чтобы завершить активацию учетной записи. Владельцы учетной записи должны быть указаны в рекламном сообщении. |
| | |

### <a name="what-are-the-differences-between-partner-center-and-the-cloud-partner-portal"></a>Каковы различия между центром партнеров и Портал Cloud Partner?

Вы можете заметить следующие различия между Портал Cloud Partner и центром партнеров.

### <a name="modular-publishing-capabilities"></a>Функции модульной публикации

Центр партнеров предоставляет параметр модульной публикации, позволяющий выбрать изменения, которые необходимо опубликовать, а не всегда публиковать все обновления одновременно. Например, на следующем снимке экрана показано, что единственными изменениями, выбранными для публикации, являются изменения **свойств** и **списка предложений**.

:::image type="content" source="media/migration-faq/review-and-publish-migration.png" alt-text="Демонстрирует страницу просмотра и публикации":::

Обновления, которые вы не публикуете, сохраняются в виде черновиков. Продолжайте использовать предварительную версию предложения, чтобы проверить предложение, прежде чем сделать его общедоступным.

### <a name="rich-text-format"></a>Формат форматированного текста

Улучшите описание предложения и плана с помощью редактора форматированного текста на странице предложения и списка планов.

:::image type="content" source="media/migration-faq/rich-text-editor-migration.png" alt-text="Показывает редактор форматированного текста":::

### <a name="enhanced-preview-options"></a>Расширенные параметры предварительного просмотра

Центр партнеров включает [функцию сравнения](https://docs.microsoft.com/azure/marketplace/partner-center-portal/update-existing-offer#compare-changes-to-marketplace-offers) с улучшенными параметрами фильтрации. Это дает возможность сравнивать версии предложения с предварительной версией и в реальном времени.

:::image type="content" source="media/migration-faq/enhanced-preview.png" alt-text="Демонстрирует функцию сравнения":::

### <a name="branding-and-navigation-changes"></a>Изменение фирменной символики и навигации

Вы заметите некоторые изменения фирменной символики. Например, "номера SKU" обозначены как "планы" в центре партнеров.

:::image type="content" source="media/migration-faq/plan-overview-migration.png" alt-text="Иллюстрируется обзор плана.":::

Кроме того, сведения, которые вы использовали для предоставления**сведений о** **Marketplace** или Торефронт (консультационная служба, Power BI приложение) в портал Cloud Partner, собираются на странице **со списком предложений** в центре партнеров.

:::image type="content" source="media/migration-faq/offer-listing-migration.png" alt-text="Показывает страницу со списком предложений.":::

Сведения, используемые для номеров SKU на одной странице в Портал Cloud Partner, теперь можно собирать на нескольких страницах в центре партнеров:

* Страница настройки плана
* Страница «плановый список»
* Страница "доступность плана"
* Спланируйте страницу технической конфигурации, как показано на снимке экрана.

:::image type="content" source="media/migration-faq/tech-config-migration.png" alt-text="Демонстрирует страницу технической настройки плана.":::

Ваш идентификатор предложения теперь отображается на левой навигационной панели предложения.

:::image type="content" source="media/migration-faq/offer-id-offer-overview.png" alt-text="Показывает левое меню навигации с ИДЕНТИФИКАТОРом предложения.":::

### <a name="stop-selling-an-offer"></a>Отмена продажи предложения

Вы можете запросить, чтобы вы не могли [продавать предложения](https://docs.microsoft.com/azure/marketplace/partner-center-portal/update-existing-offer#stop-selling-an-offer-or-plan) в Marketplace непосредственно на портале центра партнеров. Этот параметр доступен на странице **Обзор предложения** для вашего предложения.

:::image type="content" source="media/migration-faq/stop-selling-migration.png" alt-text="Иллюстрирует страницу обзор предложения с параметром "закончить продажу".":::

## <a name="will-the-cloud-partner-portal-rest-apis-be-supported-post-migration"></a>Поддерживаются ли Портал Cloud Partner API-интерфейсы RESTFUL после миграции?

Портал Cloud Partner API интегрированы с центром партнеров и продолжат работать после переноса ваших предложений в центр партнеров. Интеграция содержит небольшие изменения. Ознакомьтесь с изменениями, приведенными в таблице ниже, чтобы код продолжал работать после перехода в центр партнеров.

| **API** | **Описание изменений** | **Влияние** |
| ------- | ---------------------- | ---------- |
| После публикации, GoLive, отмена | Для перенесенных предложений заголовок ответа будет иметь другой формат, но будет продолжать работать таким же образом, обозначая относительный путь для получения состояния операции. | При отправке любого из соответствующих запросов POST для предложения заголовок Location будет иметь один из двух форматов в зависимости от состояния переноса предложения:<ul><li>Неперенесенные предложения<br>`/api/operations/{PublisherId}${offerId}$2$preview?api-version=2017-10-31`</li><li>Перенесенные предложения<br>`/api/publishers/{PublisherId}/offers/{offereId}/operations/408a4835-0000-1000-0000-000000000000?api-version=2017-10-31`</li> |
| Операция получения | Для предложений, которые ранее поддерживали в ответе поле "уведомление — адрес электронной почты", это поле будет нерекомендуемым и больше не вернется для перенесенных предложений. | Для перенесенных предложений мы больше не будем отправлять уведомления в список сообщений электронной почты, указанных в запросах. Вместо этого служба API будет согласована с процессом уведомления по электронной почте в центре партнеров для отправки сообщений электронной почты. В частности, уведомления будут отправляться по адресу электронной почты, указанному в разделе "контактные данные продавца" в параметрах учетной записи в центре партнеров, чтобы уведомить о ходе выполнения операции.<br><br>Проверьте адрес электронной почты в разделе "контактные данные продавца" в [параметрах учетной записи](https://partner.microsoft.com/dashboard/account/management) в центре партнеров, чтобы убедиться, что для уведомлений указан правильный адрес электронной почты.  |

# <a name="how-to-create-dynamics-365-for-operations-offer-via-cloud-partner-portal"></a>Создание предложения Dynamics 365 for Operations с помощью портала Cloud Partner

Доступ к порталу публикации предоставляется на основе ролей, что позволяет нескольким пользователям совместно работать над публикацией предложения. Дополнительные сведения см. в статье [Cloud Portal Manage Users](./cloud-partner-portal-manage-users.md) (Управление пользователями на портале Cloud Partner).

Прежде чем опубликовать предложение от имени учетной записи издателя, одному из пользователей с ролью \"владельца\" необходимо принять [условия использования](https://azure.microsoft.com/support/legal/website-terms-of-use/), [заявление о конфиденциальности Майкрософт](https://www.microsoft.com/privacystatement/default.aspx) и [соглашение по программе сертификации Microsoft Azure](https://azure.microsoft.com/support/legal/marketplace/certified-program-agreement/).

## <a name="how-to-create-a-new-dynamics-365-for-operations-offer"></a>Создание предложения Dynamics 365 for Operations

Выполнив все предварительные требования, вы можете приступить к созданию предложения Dynamics 365 for Operations.

1. Войдите на [облачный портал для партнеров](https://cloudpartner.azure.com/).
2. В левой панели навигации щелкните \"+ New offer\" (Создать предложение) и выберите \"Dynamics 365 for Operations\".
3. Откроется представление \"Editor\" (Редактор) для нового предложения, после чего можно приступить к разработке.
4. \"Формы\", которые нужно заполнить, отображаются в левой части представления \"Редактор\". Каждая \"форма\" состоит из набора полей, которые нужно заполнить. Обязательные для заполнения поля отмечены красной звездочкой (\*).

Существует четыре основных формы для создания предложения Dynamics 365 for Operations:

- Параметры предложения
- Технические сведения
- Подробные сведения об онлайн-магазинах
- Контакты

## <a name="how-to-fill-out-the-offer-settings-form"></a>Инструкции по заполнению формы параметров предложения

Форма параметров предложения является базовой для указания параметров предложения. Ниже описаны ее поля.

### <a name="offer-id"></a>Offer ID (Идентификатор предложения)

Это уникальный идентификатор для предложения в профиле издателя. Этот идентификатор будет отображаться в URL-адресах продукта. Идентификатор может содержать только строчные буквы, цифры или дефисы (-). Он не может заканчиваться дефисом, а его длина не должна превышать 50 символов. После активации предложения это поле блокируется.

Например, если издатель издателем contoso создает предложение с предлагать идентификатор *пример dynamics365 для операций*, оно будет отображаться в AppSource как `https://appsource.microsoft.com/marketplace/apps/**contoso**.*sample-dynamics365 for operations*?tab=Overview\`.

### <a name="publisher-id"></a>Publisher ID (Идентификатор издателя)

Этот раскрывающийся список позволяет выбрать профиль издателя, используемый для публикации этого предложения. После активации предложения это поле блокируется.

Name

Это отображаемое имя для предложения. Это же имя будет отображаться в [AppSource](https://appsource.microsoft.com). Его длина не должна превышать 50 символов.

Нажмите кнопку \"Сохранить\", чтобы сохранить настройки. На следующем шаге мы укажем технические сведения о предложении.

## <a name="how-to-fill-out-the-technical-info-form"></a>Инструкции по заполнению формы с техническими сведениями

Эта форма содержит технические сведения, которые будут отображаться на странице предложения. Инструкции по заполнению различных полей приводятся ниже.

![Экран для создания предложения](./media/publish_d365_new_offer/Technical_info.png)

### <a name="solution-identifier"></a>Solution Identifier (Идентификатор решения)

В первом поле указывается идентификатор решения.

1. Чтобы найти этот идентификатор, перейдите к службам Lifecycle Services и выберите "Управление решениями".
2. Когда вы выберете нужное решение, в разделе "Обзор пакета" появится идентификатор решения. \*\*Если поле идентификатора пустое, выберите "Изменить" и повторно опубликуйте пакет, чтобы отобразился этот идентификатор решения.

### <a name="validation-assets"></a>Validation Assets (Файлы с результатами проверки)

Здесь нужно отправить отчеты по анализу настройки.

### <a name="does-solution-enable-translations"></a>Does solution enable translation(s) (Включен ли перевод в решении?)

Выберите \'Yes\' (Да) или \'No\' (Нет).

### <a name="does-solution-include-localizations"></a>Does solution include Localization(s)? (Существуют ли локализованные версии для решения?)

Выберите \'Yes\' (Да) или \'No\' (Нет).

### <a name="product-version"></a>Версия продукта

Выберите "New AX" (Новая версия AX). Наконец, нажмите кнопку "Save" (Сохранить).

## <a name="how-to-fill-out-storefront-details-form"></a>Инструкции по заполнению формы для сведений об онлайн-магазине

Сначала указываются сведения о предложении.

1. **Offer Summary** (Краткие сведения о предложении)

    \- Введите краткие сведения о вашем решении (не более 100 знаков).

2. **Offer Description** (Описание предложения)

    \- Введите краткое описание вашего решения. В описании нужно представить функциональные возможности решения, которые должны соответствовать указанным в библиотеке BPM. Например, если в маркетинговых материалах вы указываете, что решение имеет определенные функции, во время окончательного анализа мы проверим, что все они описаны в библиотеке BPM в LCS.

![Экран со сведениями об онлайн-магазине](./media/publish_d365_new_offer/offer_details.png)

### <a name="listing-details"></a>Указание сведений

![Экран со сведениями об онлайн-магазине](./media/publish_d365_new_offer/storefront_details.png)

1. **Industries** (Отрасли) — выберите не более двух отраслей из предложенных вариантов.
2. **Categories** (Категории) — выберите не более трех категорий из предложенных вариантов.
3. **App Type** (Тип приложения) — выберите тип из предложенных вариантов.
4. **Help link for your app** (Ссылка на справку для приложения) — введите ссылку на справку для приложения.
5. **Supported countries/regions** (Поддерживаемые страны и регионы) — выберите регионы из предложенных вариантов.
6. **Supported Languages** (Поддерживаемые языки) — выберите языки из предложенных вариантов.
7. **App Version** (Версия приложения) — укажите выпускаемую версию решения, например 1.0.0.0.
8. **App Release Date** (Дата выпуска приложения) — введите дату выпуска вашего решения (мм/дд/гггг).
9. **Products your app works with** (Продукты, с которыми работает ваше приложение) — укажите определенные продукты, с которыми работает ваше приложение. Можно указать не более трех продуктов. Чтобы указать продукт, щелкните знак "плюс" рядом со словом New (Новое), после чего откроется текстовое поле для ввода имени продукта, с которым работает ваше приложение.
10. **Search Keywords** (Ключевые слова для поиска) — введите распространенные термины, которые пользователи могут указывать при поиске вашего решения. \*\*Эти ключевые слова не будут отображаться в Marketplace.
11. **Hide Key** (Ключ скрытия) — это ключ, объединенный с URL-адресом предварительной версии предложения и позволяющий скрыть предложение из общего доступа. Это не пароль. Здесь можно ввести любую строку.

### <a name="marketing-artifacts"></a>Маркетинговые артефакты

1. Теперь нужно отправить **логотипы** и **снимки экрана**. \*\*Обратите внимание на размер каждого отправляемого файла и на то, что все изображения должны быть в формате PNG.
2. **Demo videos** (Демонстрационные видео) — щелкните \"+ New\" (Новое). Отправьте демонстрационное видео с вашим решением (только ссылки на YouTube или Vimeo) \*\*. Обязательно отправьте эскиз определенного размера, чтобы видео отображалось на промежуточном этапе.
3. **Documents** (Документы) — отправьте все документы, относящиеся к решению, и не забудьте ввести имя для документа.

### <a name="legal"></a>Юридическая информация

В этом разделе указываются ссылки на политику конфиденциальности и условия использования. Введите URL-адрес политики конфиденциальности, а также условий использования для решения. \*\*Клиент сможет просмотреть эти политики на портале.

### <a name="customer-support"></a>Поддержка клиентов

URL-адрес службы поддержки будет отображаться на портале только для ваших пользователей.

### <a name="leads-management"></a>Управление потенциальными клиентами

Выберите систему CRM, где будут храниться сведения о потенциальных клиентах. Выберите \"таблиц Azure\" здесь, если у вас один из следующих систем CRM: Salesforce, Marketo, Microsoft Dynamics CRM. В выбранной вами системе CRM мы будем записывать сведения о пользователях, которые тестируют ваше приложение в AppSource (потенциальные клиенты). В зависимости от выбранной системы CRM щелкните соответствующий URL-адрес ниже, чтобы узнать, как заполнить следующий набор полей.

![Сведения об управлении потенциальными клиентами](./media/publish_d365_new_offer/leads.png)

1. Инструкции для Таблиц Azure см. [здесь](https://aka.ms/leadsettingforazuretable).
2. Инструкции для Dynamics CRM Online см. [здесь](https://aka.ms/leadsettingfordynamicscrm).
3. Инструкции для Marketo см. [здесь](https://aka.ms/leadsettingformarketo).

## <a name="how-to-fill-out-the-contacts-form"></a>Инструкции по заполнению формы контактов

Эта информация будет использоваться для службы поддержки корпорации Майкрософт и службы поддержки клиентов. Введите контакты сотрудников технического отдела и отдела поддержки клиентов вашей компании, а также URL-адрес службы поддержки для вашего решения. Эта информация будет использоваться непосредственно для связи, если у специалистов корпорации Майкрософт возникнут вопросы о вашем решении, а также для поддержки пользователей.

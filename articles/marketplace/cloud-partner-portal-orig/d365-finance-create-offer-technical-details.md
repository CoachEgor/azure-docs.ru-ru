<a name="how-to-fill-out-the-technical-info-form"></a>Инструкции по заполнению формы с техническими сведениями
===========================================

1.  В разделе **Choose Application Type** (Выберите тип приложения) отправьте ваш файл пакета расширения (APP) и все файлы, от которых зависит ваше расширение.

    ![Сведения о пакете приложения](./media/d365-financials/image015.png)

-   **Extensions Package File** (Файл пакета расширения). APP-файл пакета расширения (требуется).

-   **Dependency package file** (Файл пакета, от которого зависит текущий пакет). Требуется, если приложение зависимо от другого приложения, опубликованного в AppSource. Это файл APP уже опубликованного в AppSource расширения, от которого зависит текущее приложение. 

-   **Library Package File** (Файл пакета библиотеки). Требуется, если приложение зависимо от другого приложения, которое *не* опубликовано в AppSource. Это файл APP существующего приложения, которое не было и не будет опубликовано в AppSource.

-   **App Test Automation** (Автоматизация тестирования приложений). Пакет теста, закодированного в VS, который необходимо создать для автоматического тестирования расширений (требуется).

1. В разделе **Additional Information for the extension** (Дополнительные сведения для расширения) отправьте дополнительные сведения для расширения. Эти сведения используются во время проверки.

   ![Дополнительные сведения для формы расширения приложения](./media/d365-financials/image016.png)


-   **URL to Product Documentation** (URL-адрес документации по продукту). URL-адрес документации для расширения (требуется).

-   **Key Usage Scenarios** (Сценарии использования ключа). Документ, в котором приведены пошаговые инструкции по установке и использованию расширения (требуется). Пример можно найти в [документации по пользовательскому сценарию](https://docs.microsoft.com/dynamics-nav/compliance/apptest-userscenario/).

-   **Target Release** (Целевой выпуск). Выберите выпуск, где будет развернуто приложение (требуется). Выберите **current** (текущий) для развертывания в текущей версии. Выберите **next minor** (следующий дополнительный) для развертывания в следующем дополнительном выпуске. Выберите **next major** (следующий основной) для развертывания в следующем основном выпуске.

-   **Requires Premium SKU** (Требуется SKU "Премиум"). Нажмите кнопку "Премиум", если приложению требуется SKU "Премиум" (необязательно). Управление службой и производство доступны только для уровня "Премиум". Подробные сведения о планах "Основной" и "Премиум" можно найти в статье [Изменение набора отображаемых функций](https://docs.microsoft.com/dynamics365/financials/ui-experiences).

-   **Explanation for Code Analysis Errors** (Объяснение ошибок анализа кода). Документ, где содержится код, который не соответствует требованиям, вместе с объяснением (необязательно).

-   **Explanation of Impacted Core Functionality** (Объяснение влияния на основные функциональные возможности). Документ, в котором перечислены и описаны основные функциональные возможности, которые ограничиваются расширением (необязательно).

-   **Test Accounts** (Тестовые учетные записи). Учетные записи для удаленных служб, веб-сайтов и т. д, которые потребуются для выполнения сквозного теста использования (необязательно).

-   **UX requirements exceptions** (Исключения требований пользовательского интерфейса). Документ, где содержатся требования к пользовательскому интерфейсу, которые не выполняет расширение, вместе с объяснением (необязательно).

Следующим шагом является добавление сведений об онлайн-магазине для вашего предложения.

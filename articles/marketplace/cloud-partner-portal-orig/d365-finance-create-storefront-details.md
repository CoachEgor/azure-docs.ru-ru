<a name="how-to-fill-out-the-storefront-details-form"></a>Инструкции по заполнению формы для сведений об онлайн-магазине
===========================================

Простое перечисление функций и возможностей приложения не превратит перспективных клиентов в покупателей. Дополнительные сведения о том, как лучше всего рекламировать приложения в AppSource Marketplace, см. в техническом документе по [разработке приложений для Dynamics 365 for Finance and Operations](https://go.microsoft.com/fwlink/?linkid=841518).  


### <a name="offer-summary"></a>Offer Summary (Сводка предложения)

Эта сводка появится на AppSource в виде сводки в обзоре приложений, а также на странице сведений о приложении под именем приложения и именем издателя. Мы рекомендуем создавать чувство любопытства и положительный настрой, задавая перспективным клиентам вопрос, на который они либо не знают ответ, либо могут соотнести с собой. Вопрос должен стимулировать сильную эмоциональную реакцию. Обратите внимание на значительное неудобство, которое они, скорее всего, испытывают и которое оказывает значительное отрицательное влияние. Длина не должна превышать 100 символов.

![Сводка предложения Dynamics 365 Business Central](./media/d365-financials/image018.png)


### <a name="offer-description"></a>Описание предложения

Это описание появится на AppSource сразу под **сводкой предложения**. При описании решения не фокусируйте внимание на основной технологии или функциональных возможностях. Перспективные клиенты могут получить подробные сведения позже. Основная цель — преобразование, которое означает, что необходимо побудить перспективных клиентов к действию. Оптимальный подход — напомнить им о неудобстве, которое они, скорее всего, испытывают, и указать преимущества, которые они получат, приступив к использованию решения. Четко сформулируйте привлекательный желаемый результат. Мы рекомендуем добавить ссылку на целевую страницу своего приложения в конце описания. Максимальная длина — 1300 символов.

### <a name="industries"></a>Отрасли

Выберите отрасли, к которым лучше всего подходит ваше приложение. Если приложение относится к нескольким отраслям, оставьте это поле пустым.

### <a name="categories"></a>Categories

Выберите категории, относящиеся к вашему приложению. Выберите хотя бы одну категорию, но не более трех. Выбирайте тщательно, так как клиенты могут проводить поиск в AppSource на основе категорий.

### <a name="app-type"></a>Тип решения

Выберите тип пробной версии, которая будет предлагаться для приложения в AppSource.
**Free** (Бесплатная) означает, что приложение доступно бесплатно. **Пробная версия** означает, что клиенты могут бесплатно работать с приложением в AppSource в течение короткого периода времени. **Request for trial** (Запрос пробной версии) означает, что клиенты могут запросить пробную версию приложения с AppSource.

### <a name="help-link-for-your-app"></a>Ссылка на справку для приложения

Должна существовать активная ссылка на справку, которая направляет клиентов к интерактивной справке о продукте, относящейся к приложению. Рекомендуется добавить инструкции по настройке, ссылки на существующие видео и документацию, а также на часто задаваемые вопросы.

### <a name="supported-countriesregions"></a>Поддерживаемые страны или регионы

Это поле определяет страны и регионы, в которых ваше предложение будет доступно на период пробной версии. Можно выбрать только те страны, в которых Microsoft Dynamics 365 Business Central уже доступен или находится в предварительной версии. Текущий список см. в разделе [Countries and Translations Supported](https://docs.microsoft.com/dynamics-nav/compliance/apptest-countries-and-translations) (Поддерживаемые страны и переводы).

![Выбор поддерживаемых стран и переводов](./media/d365-financials/image008.png)


### <a name="supported-languages"></a>Поддерживаемые языки

Выберите языки, которые поддерживает приложение. Если приложение поддерживает языки, отсутствующие в этом списке, опубликуйте предложение и напишите нам по адресу <appsource@microsoft.com>, чтобы сообщить о поддержке других языков.

### <a name="app-version"></a>версия приложения.

Введите номер версии приложения.

### <a name="products-your-app-works-with-max-3"></a>Продукты, с которыми работает ваше приложение (не более 3)

Укажите конкретные продукты, с которыми работает ваше приложение. Можно указать не более трех продуктов. Чтобы указать продукт, щелкните знак "плюс" рядом со словом New (Новое), после чего откроется текстовое поле для ввода имени продукта, с которым работает ваше приложение.

### <a name="hide-key"></a>Hide key (Ключ скрытия)

Укажите ключ скрытия, который можно использовать для просмотра предложения в промежуточной среде в AppSource. Здесь можно ввести любую строку

### <a name="offer-logo-png-format-48x48"></a>Логотип предложения (формат PNG, 48x48)

Этот логотип будет отображаться на AppSource в обзоре приложения или в результатах приложения при выполнении поиска. *Допускается только формат png.*  Отправьте изображение в формате .png с разрешением 48\*48 пикселей

### <a name="offer-logo-png-format-216x216"></a>Логотип предложения (формат PNG, 216x216)

Отобразится на странице сведений о приложении на AppSource. *Допускается только формат png.* Отправьте изображение в формате .png с разрешением 216\*216 пикселей

### <a name="video"></a>Видео

Рекомендуется отправить хотя бы одно видео (но необязательно). Разрешено не более четырех видео. Для каждого переданного видео нужно указать имя видео, URL-адрес (поддерживаются только YouTube и Vimeo) и эскиз, связанный с этим видео. Эскиз должен быть в формате .png с размером 1280\*720 пикселей. Чтобы добавить новое видео, щелкните знак "плюс" (см. снимок экрана ниже). Эскизы видео появятся на AppSource.

![Отправка видео](./media/d365-financials/image009.png)


### <a name="document"></a>Документ

Документы, отправляемые сюда, появятся на AppSource в разделе \"Learn more\" (Подробнее).
Можно отправить не более трех документов в формате PDF. Для каждого передаваемого документа нужно указать имя документа и выбрать действие "Отправить документ". Документ должен быть в формате .pdf. Чтобы добавить новые документы, щелкните знак "плюс" (см. снимок экрана ниже).

![Отправка документов](./media/d365-financials/image010.png)


### <a name="screenshots"></a>Снимки экрана

Требуется как минимум три снимка экрана. Разрешено не более пяти снимков. Сделайте снимки экрана в песочнице Financials и выберите снимки экрана, которые включают реалистичные демонстрационные данные, а также передают привлекательную завлекающую информацию.

### <a name="privacy-policy"></a>Privacy Policy (Политика конфиденциальности)

Введите URL-адрес документа "Политика конфиденциальности" для приложения.

### <a name="terms-of-use"></a>Условия использования

Введите условия использования приложения. Клиенты AppSource будут обязаны принять эти условия, прежде чем смогут использовать приложение.

### <a name="customer-support"></a>Поддержка клиентов

Должна существовать определенная страница поддержки, которая предоставляет клиентам несколько вариантов связи. Включите в нее поддержку по телефону, через сообщение электронной почты и в идеале интерактивный чат (если это возможно).

### <a name="lead-destination"></a>Назначения потенциального клиента

Выберите систему CRM, где будут храниться сведения о потенциальных клиентах. Выберите \"таблиц Azure\" здесь, если у вас один из следующих систем CRM: Salesforce, Marketo, Microsoft Dynamics CRM. В выбранной вами системе CRM мы будем записывать сведения о пользователях, которые тестируют ваше приложение в AppSource (потенциальные клиенты). В зависимости от выбранной системы CRM щелкните соответствующий URL-адрес ниже, чтобы узнать, как заполнить следующий набор полей.

-   [Таблица Azure](./cloud-partner-portal-lead-management-instructions-azure-table.md)
-   [Marketo](./cloud-partner-portal-lead-management-instructions-marketo.md)
-   [Microsoft Dynamics CRM](./cloud-partner-portal-lead-management-instructions-dynamics.md)
-   [Salesforce](./cloud-partner-portal-lead-management-instructions-salesforce.md)

Следующий шаг — продолжить публикацию приложения.

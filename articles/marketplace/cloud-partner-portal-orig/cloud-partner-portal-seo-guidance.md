# <a name="azure-marketplace-seo-guidance"></a>Руководство по Azure Marketplace оптимизации поисковой системы

В этой статье объясняется, как максимально увеличить возможности обнаружения своего предложения, функциональные возможности поиска в [Azure Marketplace](https://azuremarketplace.microsoft.com) и [AppSource](https://appsource.microsoft.com). 


## <a name="general-explanation-of-algorithm"></a>Общие сведения об алгоритме

Магазины Майкрософт использовать поиск Azure для повышения эффективности возможности поиска веб-узла. Алгоритм основывается на частоте условия или инверсной частота в документе ([TF/IDF](https://en.wikipedia.org/wiki/Tf–idf)). Используется стандартный [анализатор Lucene](https://lucene.apache.org/core/).

В общем, все текстовые поля, категории и отрасли включены в релевантность весомости связей. Чтобы получить более высокие результаты поиска можно использовать определенные термины, редко используемые для приложения, но часто используемые самим приложением. Поэтому, включение таких терминов, как виртуальная машина, позволит получить небольшое преимущество, в то время, как использование поиска Azure является более определенным.
Далее приведены наиболее важные поля, которые следует рассмотреть.

 
|  Поле                   | Важность | Руководство                                                                                            |
|  --------------------    | ----------                   | ---------------                                                                   |
| Название предложения               |  Высокая      | Точное или близкое совпадение с результатами поиска приведет к получению более высокой оценки.                       |
| Имя издателя           |  Высокая      | Точное или близкое совпадение с результатами поиска приведет к получению более высокой оценки.                       |
| Краткое описание        |  Средние    | Учитывая, что имена приложений и имена издателей почти гарантируют высокий результат, это может быть не самым подходящим. В таком случае краткое описание является критическим. Текст должен быть кратким и точным. Для улучшения результатов необходимо включить ключевые слова и ожидаемые условия поиска.  Например, фраза "Это лучший Retail POS, который был создан поверх Dynamics 365" является менее эффективной нежели "Retail POS (пункт обслуживания клиентов) для Dynamics 365".  | 
| Длинное описание         |  Низкая       | С помощью описания можно получить дополнительную информацию. В наиболее эффективных описаниях используются ключевые слова. Также они должны быть "разумного" размера.  Точное описание, в котором используются ключевые слова, будет более полезным нежели описание с длинным текстом. Убедитесь, что в описании присутствуют такие ключевые слова, как Интернет вещей.  |
| Категории продуктов       | Средние     |  Категории продуктов определяются с помощью сочетания выборов издателя и корпорации Майкрософт. Данные категории необходимо выбирать соответствующим образом. Чтобы пользователи могли легко находить требуемые приложения в соответствующей категории. |
|  |  |  |


## <a name="other-tips"></a>Другие советы

-   Поиск предложений является тяжелой пользовательской деятельностью. В нем определяются совпадения с именем приложения или именем издателя. Когда условие поиска не совпадает с именем издателя или приложения, краткое описание становится ключевым.
-   В поиск весомых связей не включены документы для загрузки.
-   Также на результаты поиска будет влиять фактическое использование и приобретение приложения. Например, если существует два идентичных приложения, то более высокою оценку получит то, у которого больше пользователей.

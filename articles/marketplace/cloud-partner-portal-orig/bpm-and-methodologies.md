# <a name="business-process-libraries"></a>Библиотеки бизнес-процессов

Библиотеки бизнес-процессов — руководство по задачам комплексных сценариев. Библиотеки бизнес-процессов непосредственно должны быть на ровне с содержимым маркетинга. Каждый бизнес-процесс в библиотеке должен иметь [запись задачи](https://docs.microsoft.com/dynamics365/operations/dev-itpro/user-interface/task-recorder) связанную с ней. Если некоторые процессы находятся за пределами Dynamics 365 for Operations, вложите Visio в библиотеку BPM.

Дополнительные сведения см. в разделе [Настройка библиотек средства моделирования бизнес-процессов для финансовых и операционных решений](https://docs.microsoft.com/dynamics365/operations/dev-itpro/lcs-solutions/business-process-modeler-libraries-lcs-solutions).

## <a name="methodologies"></a>Методология

Lifecycle Services (LCS) для Microsoft Dynamics предоставляет методологию, с помощью которой можно обеспечить более предсказуемую и повторяемую реализацию проекта. Можно использовать одну из предоставленных методологий или создать собственную. С помощью методологии можно легко отслеживать процесс или создавать о нем отчеты.

Методология состоит из этапов, задач и вех. Каждый этап может иметь любое количество задач, некоторые из которых являются обязательными. По завершении всех задач в ходе этапа, этап будет помечен как завершенный. Можно также создать веху, в случае критического завершения этапа. В проекте LCS включены следующие методики.

- Реализация
- Уверенный шаг.
- Разработка обучения.
- Миграция и создание решений.
- Использование решений.

Дополнительные сведения см. в разделе [Создание или обновление методологии](https://docs.microsoft.com/dynamics365/operations/dev-itpro/lifecycle-services/create-methodology).

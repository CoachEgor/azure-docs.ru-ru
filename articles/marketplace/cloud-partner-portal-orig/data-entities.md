# <a name="data-entities"></a>Сущности данных

В этой статье определены и предоставлены общие сведения о сущности данных. Она содержит сведения о возможности сущностей данных, о поддерживаемых ими сценариях, категориях, которые используются для них, а также методах их создания.

## <a name="overview"></a>Обзор

Сущность данных — это абстракция аспектов физической реализации таблиц базы данных. Например, в нормализованных таблицах, значительная часть данных каждого пользователя может храниться в таблице а остальные могут быть распределены между небольшим набором связанных таблиц. В этом случае сущность данных этой концепции пользователя будет представлена как одно денормализованное представление, в котором каждая строка содержит все данные из таблицы пользователя и связанные с ним таблицы. Сущность данных инкапсулирует бизнес-концепция в формат, который упрощает разработку и интеграцию. Разработку приложений и их настройку можно упростить благодаря абстрактному характеру сущности данных. Позже абстракция также изолирует код приложения между версиями обязательных требований физических таблиц.

Итог: сущность данных предоставляет концептуальную абстракцию и инкапсуляцию (денормализованное представление) схемы базовой таблицы для представления данных ключевых концепций и функций.

## <a name="capabilities"></a>Возможности

Сущность данных имеет указанные ниже функции.

- Заменяет расходящиеся и фрагментированные концепции AXD, сущности платформы импорта и экспорта данных (DIXF), и вычисляет запросы с единой концепцией.
- Предоставляет единственный комплект для определения бизнес-логики и для реализации сценариев, например, импорт и экспорт, интеграция и программирование.
- Становится основным механизмом для экспорта и импорта пакетов данных для управления жизненным циклом приложений (ALM) и сценариев данных (демоверсия).
- Ее можно предоставить в качестве службы OData и затем использовать в табличном виде сценариев синхронной интеграции и интеграции Microsoft Office.

Дополнительные сведения см. в разделе [Сущности данных](https://docs.microsoft.com/dynamics365/operations/dev-itpro/data-entities/data-entities).

# <a name="consulting-services-for-azure-marketplace-and-appsource"></a>Консультационные услуги для Azure Marketplace и AppSource
Консультационные услуги в Azure Marketplace и AppSource позволяют партнерам корпорации Майкрософт организовать подключение клиентов к службам, поддерживающим и расширяющим взаимодействие с Azure и связанные с ним бизнес-задачи. 

Azure Marketplace и AppSource поддерживают список предложений консультационных услуг от партнеров корпорации Майкрософт по системному интегратору (SI), целью которого является предоставление консультационных услуг, позволяющих клиентам подключаться к необходимым им службам. Они помогают клиентам оценить, изучить и развернуть правильные решения и реализации, помогающие реализовать бизнес-цели и выполнить поставленные задачи.  

Эти предложения консультационных услуг строго привязаны к конкретным заказчикам. Издатели имеют возможность выбрать тип приложения, определить область действия и продолжительность, стоимость (фиксированная или бесплатно), а также описать программу передачи клиенту результатов работы. Обращения потенциальных клиентов поступают в виде уведомлений в системе CRM партнера. На запросы клиентов следует реагировать не позднее 48 часов. 

## <a name="consulting-services-in-azure-marketplace-vs-appsource-storefront"></a>Консультационные услуги в Azure Marketplace и AppSource Storefront
Онлайн-магазины Marketplace связаны с целевой аудиторией и облачными продуктами Майкрософт, которые помогают клиентам найти то, что им нужно. Каждый онлайн-магазин предлагает специализированные варианты публикации, чтобы вы могли извлечь максимальную выгоду из инвестиций в публикации. Эти варианты обобщены в следующей таблице.

|         |Azure Marketplace  |Microsoft AppSource |
|---------|---------|---------|
|**Аудитория**   |**ИТ-специалисты и разработчики облачных решений**|**Бизнес-пользователи и лица, принимающие решения в сфере бизнес-приложений**|
|Соответствие решений     |Технические приложения, созданные для (или в) Azure|Бизнес-приложения, которые используют или расширяют D365, Power BI, O365, Azure. (Надстройка или дополнение готового приложения)|
|Параметры публикации     |Связь со мной, пробная версия, Transact или службы для Azure|Свяжитесь со мной, пробной версией или службами для Dynamics 365 и Power BI|
|В продукте      |Портал управления Azure|Dynamics 365, Office 365, клиентские приложения Office| 
|Прочее      |Локальная публикация с помощью Azure Stack, консультационные службы для Azure, управляемые службы для Azure|Консультационные услуги для Dynamics и Power BI|

Предложения консультационных услуг, основанные на Microsoft Dynamics 365, Power Apps и Power BI, могут быть перечислены в Microsoft AppSource. Дополнительные сведения см. в документе [с рекомендациями по размещению консультационных услуг в Microsoft AppSource](https://go.microsoft.com/fwlink/?LinkId=828734&clcid=0x409).

Предложения консультационных услуг и управляемых служб, основанные на Microsoft Azure, могут быть указаны в Azure Marketplace. Дополнительные сведения см. в статьях о [публикации службы консультационных услуг Azure](https://docs.microsoft.com/azure/marketplace/cloud-partner-portal/consulting-services/cloud-partner-portal-consulting-services-publishing-offer) и [публикации предложения управляемых служб в Azure Marketplace](https://docs.microsoft.com/azure/lighthouse/how-to/publish-managed-services-offers).

> [!Note]
> Предложения консультационных услуг в Azure Marketplace должны предназначаться для **технических специалистов, которые работают с *техническими решениями и службами Azure*. ** Публикуемые в AppSource консультационные услуги должны основываться на опыте в разработке качественных решений для Dynamics 365, Power BI и (или) PowerApps
> 
> 
> [!TIP]
> Для публикации в онлайн-магазине Azure Marketplace на портале Cloud Partner в раскрывающемся меню основного продукта нужно выбрать "Azure". Для публикации в AppSource на портале Cloud Partner в раскрывающемся меню основного продукта нужно выбрать "Dynamics 365", "Power BI" или "Power Apps".

## <a name="how-to-register-for-consulting-services-in-azure-marketplace-and-appsource"></a>Регистрация на предоставление консультационных услуг в Azure Marketplace и AppSource
Прежде всего следует [зарегистрировать](https://partner.microsoft.com/dashboard/account/v3/enrollment/introduction/azureisv) сведения о компании в Marketplace. Полный контрольный список требований к регистрации издатели можно найти в статье [Become a Publisher in the Marketplace](https://docs.microsoft.com/azure/marketplace/become-publisher) (Как стать издателем в Marketplace). 

**Чего ожидать после регистрации**. Команда подключения в течение 1–3 рабочих дней отправит вам по электронной почте приветственное сообщение с данными для входа на портал Cloud Partner. [Портал Cloud Partner](https://cloudpartner.azure.com) используется издателями для предоставления информации и заявок по своим предложениям. 

>[!Note]
>Партнеру достаточно зарегистрироваться однократно, чтобы стать издателем. Получив по электронной почте приветственное сообщение с информацией для входа на портал Cloud Partner, издателю больше не нужно регистрироваться повторно для публикации предложений других типов.

## <a name="publishing-process"></a>Процесс публикации
Процесс публикации предложения консультационных услуг в Azure Marketplace и AppSource включает следующие этапы.

1. **Предложение**. Чтобы зарегистрировать новое предложение, войдите в учетную запись на [Портале Cloud Partner](https://cloudpartner.azure.com). Выберите в раскрывающемся меню "Новое предложение", а затем "Consulting Services". Введите артефакты для предложения и опубликуйте его.
2. **Проверка**. Ваш запрос будет проверен на соответствие требованиям к публикации. Затем мы отправим вам инструкции по размещению предложения консультационных услуг. 
3. **Отправка**. Предоставьте необходимую информацию в соответствии с инструкциями. Все содержимое и вспомогательные материалы должны быть на английском языке. 
4. **Просмотр**. Наша группа маркетологов и редакторов изучит предоставленные материалы и скорректирует их для повышения разборчивости, грамотности и эффективности. 
5. **Проверка**. Заявка возвращается вам на проверку и утверждение. 
6. **Публикация**. Когда все обсуждаемые материалы будут полностью приемлемы для вас и корпорации Майкрософт, предложение будет считаться готовым для публикации в Marketplace. Дата вашего списка зависит от страны или региона, указанных в вашем предложении. Если каталог для выбранной страны или региона уже открыт в Azure Marketplace или AppSource, ваш список будет отображаться в течение нескольких часов. В противном случае предложение станет доступным с момента публикации каталога.

## <a name="offer-listings-and-content-review-criteria"></a>Перечисление предложений и критерии проверки содержимого
Принимаются к рассмотрению предложения услуг следующих типов (вы можете разместить сразу несколько типов). 

* **Оценка**: Оценка среды клиента для определения применимости решения и предоставления оценки затрат и времени. 
* **Брифинг**. Ознакомление с решением или консультационной услугой для привлечения интереса клиента с использованием схем, демонстрационных материалов и историй клиентов. 
* **Реализация**. Полная установка, предоставляющая клиенту полностью рабочее решение. Мы рекомендуем использовать только такие решения, которые реализуются не дольше двух недель. 
* **Подтверждение концепции**. Ограниченная реализация, позволяющая определить применимость решения к требованиям клиента. 
* **Семинар**. интерактивное сотрудничество, проводимое в локальной среде клиента. Она может содержать обучающие, краткие или демонстрационные версии, основанные на данных или среде клиента. 

**Примечание.** За исключением типа службы " *Краткая* служба" консультационные услуги могут быть предоставлены пользователю (т. е. в расположении клиента или партнерской компании) или виртуально (т. е. с помощью телеконференций, веб-конференций или удаленной реализации). Брифинги должны проводиться в личном порядке.

После того как вы настроите необходимые элементы, перечисленные выше, команда проверки содержимого проверит, чтобы ваше описание предложения было информативным и исчерпывающим для целевой аудитории.

Примеры:


|Пример заголовка  |Верный формат  |
|---------|---------|
|Приступая к работе с Центром Интернета вещей Azure в сфере производства    |   Интернет вещей в сфере производства: 2-дневная проверка      |
|Семинар по интеллектуальным тостерам        |  Интеллектуальные тостеры: 1-недельный семинар       |

На портале Cloud Partner в ваше предложение можно добавить форматирование, используя [Markdown](/contribute/markdown-reference). Это простой язык проектирования, который поможет применить форматирование к странице вашего предложения. 

**Примечания по разметке Markdown**
* Добавьте символ * в начале и конце блока текста (не отделяя их пробелами), чтобы выделить этот текст курсивом (*пример текста*).
* Символ двоеточия (:) создает разрыв строки
* Символ * с пробелом создает маркеры ненумерованного списка (*(пробел)Элемент списка)
* ### <a name="space-for-heading-ex-spaceagendadeliverables"></a>Пробел для заголовка (###(пробел)Расписание/Результаты)
* Разрыв строки после заголовка
* ** = полужирный 
* \* Выделите курсивом обзорное описание предложения

# <a name="consulting-services-for-azure-marketplace-and-appsource"></a>Консультационные услуги для Azure Marketplace и AppSource
Консультационные услуги в Azure Marketplace и AppSource позволяют партнерам корпорации Майкрософт организовать подключение клиентов к службам, поддерживающим и расширяющим взаимодействие с Azure и связанные с ним бизнес-задачи. 

Azure Marketplace и AppSource поддерживают перечень предложений консалтинговых услуг от партнеров System Integrator (SI) корпорации Майкрософт - цель заключается в предоставлении качественных консалтинговых услуг, позволяющих клиентам связываться с экспертными услугами, в которых они нуждаются. Они помогают клиентам оценить, изучить и развернуть правильные решения и реализации, помогающие реализовать бизнес-цели и выполнить поставленные задачи.  

Эти предложения консультационных услуг строго привязаны к конкретным заказчикам. Издатели имеют возможность выбрать тип приложения, определить область действия и продолжительность, стоимость (фиксированная или бесплатно), а также описать программу передачи клиенту результатов работы. Обращения потенциальных клиентов поступают в виде уведомлений в системе CRM партнера. На запросы клиентов следует реагировать не позднее 48 часов. 

## <a name="consulting-services-in-azure-marketplace-vs-appsource-storefront"></a>Консультационные услуги в Azure Marketplace и AppSource Storefront

Каждый онлайн-магазин предлагает специализированные варианты публикации, чтобы вы могли извлечь максимальную выгоду из инвестиций в публикации. Эти варианты обобщены в следующей таблице.

|---------|---------|---------|
|**Аудитории**      |**ИТ-специалисты и разработчики облачных решений**|**Бизнес-пользователи и лица, принимающие решения в сфере бизнес-приложений**|
|Соответствие решений      |Технические приложения, созданные для (или в) Azure|Бизнес-приложения, которые используют или расширяют D365, Power BI, O365, Azure. (Надстройка или дополнение готового приложения)|
|Варианты публикации      |Свяжитесь со мной, Судебная практика, сделки или услуги для Azure|Свяжитесь со мной, пробой или службами для динамики 365 и Power BI|

Предложения консалтинговых услуг, основанные на Microsoft Dynamics 365, Power Apps и Power BI, могут быть перечислены в Microsoft AppSource. Дополнительные сведения см. в документе [с рекомендациями по размещению консультационных услуг в Microsoft AppSource](https://go.microsoft.com/fwlink/?LinkId=828734&clcid=0x409).

Консультационные услуги и предложения управляемых сервисов, основанные на Microsoft Azure, могут быть перечислены на Azure Marketplace. Для получения дополнительной информации смотрите статьи о [публикации консалтинговой службы Azure](./partner-center-portal/create-consulting-service-offer.md) и [публикуйте управляемое предложение услуг для Azure Marketplace.](https://docs.microsoft.com/azure/lighthouse/how-to/publish-managed-services-offers)

Консультационные услуги в AppSource должны использовать предложения, ориентированные на их опыт в предоставлении качественных решений для Dynamics 365, Power BI или Power Apps, чтобы перечислить свои предложения консалтинговых услуг на AppSource

## <a name="publishing-process"></a>Процесс публикации
Процесс публикации предложения консультационных услуг в Azure Marketplace и AppSource включает следующие этапы.

1. **Листинг**: Начните составление нового предложения листинга, войдя в свой аккаунт [Партнерского центра.](https://partner.microsoft.com/dashboard/account/v3/enrollment/introduction/partnership) Выберите в раскрывающемся меню "Новое предложение", а затем "Consulting Services". Введите артефакты для предложения и опубликуйте его.
2. **Проверка**. Ваш запрос будет проверен на соответствие требованиям к публикации. Затем мы отправим вам инструкции по размещению предложения консультационных услуг. 
3. **Отправка**. Предоставьте необходимую информацию в соответствии с инструкциями. Все содержимое и вспомогательные материалы должны быть на английском языке. 
4. **Просмотр**. Наша группа маркетологов и редакторов изучит предоставленные материалы и скорректирует их для повышения разборчивости, грамотности и эффективности. 
5. **Проверка**. Заявка возвращается вам на проверку и утверждение. 
6. **Публикация**. Когда все обсуждаемые материалы будут полностью приемлемы для вас и корпорации Майкрософт, предложение будет считаться готовым для публикации в Marketplace. Дата объявления зависит от страны/региона, указанного в вашем предложении. Если каталог выбранной страны/региона уже находится в Azure Marketplace или AppSource, ваше объявление появится в течение нескольких часов. В противном случае предложение станет доступным с момента публикации каталога.

## <a name="offer-listings-and-content-review-criteria"></a>Перечисление предложений и критерии проверки содержимого
Принимаются к рассмотрению предложения услуг следующих типов (вы можете разместить сразу несколько типов). 

* **Оценка**: Оценка среды клиента для определения применимости решения и предоставления оценки затрат и сроков. 
* **Брифинг**. Ознакомление с решением или консультационной услугой для привлечения интереса клиента с использованием схем, демонстрационных материалов и историй клиентов. 
* **Реализация**. Полная установка, предоставляющая клиенту полностью рабочее решение. Мы рекомендуем использовать только такие решения, которые реализуются не дольше двух недель. 
* **Подтверждение концепции**. Ограниченная реализация, позволяющая определить применимость решения к требованиям клиента. 
* **Семинар**: Интерактивное взаимодействие, которое проводится в помещениях клиента. Она может включать обучение, инструктаж, оценки или демонстрации, построенные на данных или среде клиента. 

**Примечание:** За исключением типа услуг *Briefing,* консультационные услуги могут быть предложены лично (т.е. на месте в месте нахождения клиента или объекта партнера) или виртуально (то есть с помощью телеконференций, веб-конференций или удаленной реализации). Брифинги должны проводиться в личном порядке.

После того как вы настроите необходимые элементы, перечисленные выше, команда проверки содержимого проверит, чтобы ваше описание предложения было информативным и исчерпывающим для целевой аудитории. 

#### <a name="listing-title"></a>Заголовок списка

Поле заголовка имеет ограничение в 50 символов и должно содержать срок действия и тип службы предложения для оптимизации для поисковых систем. Используйте такой формат: "НАЗВАНИЕ: ТИП ПРОДОЛЖИТЕЛЬНОСТИ".

Примеры:


|Пример заголовка  |Верный формат  |
|---------|---------|
|Приступая к работе с Центром Интернета вещей Azure в сфере производства    |   Интернет вещей в сфере производства: 2-дневная проверка      |
|Семинар по интеллектуальным тостерам        |  Интеллектуальные тостеры: 1-недельный семинар       |
|  Подтверждение концепции миграции SQL Server    | Миграция SQL: 3-недельное подтверждение концепции        |

В Partner Center вы можете добавить форматирование в свой список с помощью [Markdown.](/contribute/markdown-reference) Это простой язык проектирования, который поможет применить форматирование к странице вашего предложения. 

**Примечания по разметке Markdown**
* Добавьте символ * в начале и конце блока текста (не отделяя их пробелами), чтобы выделить этот текст курсивом (*пример текста*).
* Символ двоеточия (:) создает разрыв строки
* Символ * с пробелом создает маркеры ненумерованного списка (*(пробел)Элемент списка)
* ### <a name="space-for-heading-ex-spaceagendadeliverables"></a>Пробел для заголовка (###(пробел)Расписание/Результаты)
* Разрыв строки после заголовка
* ** = полужирный 
* * Выделите курсивом обзорное описание предложения

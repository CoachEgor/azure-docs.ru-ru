# <a name="consulting-services-for-azure-marketplace-and-appsource"></a>Консультационные услуги для Azure Marketplace и AppSource
Службы Azure Marketplace и AppSource консультационные услуги предлагают партнерам по системам Майкрософт преимущества подключения клиентов к службам, которые поддерживают и расширяют возможности использования Azure. Эти консультационные услуги помогают клиентам оценивать, оценивать и развертывать бизнес-решения, которые более полно применяют свои бизнес-задачи и цели.  

Предложения консультационных услуг являются взаимоотношениями с клиентами. Издатели имеют возможность определить тип перечисления, область, длительность и структуру ценообразования (фиксированная цена или бесплатная). Предложения имеют определенную повестку конечных результатов клиенту. Интересы получаются через уведомление в системах CRM партнеров, а партнеры — для реагирования на клиентов в течение 48 часов. 

## <a name="compare-consulting-services-in-azure-marketplace-and-appsource-storefronts"></a>Сравнение консультационных услуг в Azure Marketplace и витринах AppSource
Витрины Marketplace выводятся в соответствие с аудиторией и облачными продуктами, чтобы помочь клиентам найти необходимые объекты. Каждый онлайн-магазин предлагает специализированные варианты публикации, чтобы вы могли извлечь максимальную выгоду из инвестиций в публикации. Эти варианты обобщены в следующей таблице.

|         |Azure Marketplace  |Microsoft AppSource |
|---------|---------|---------|
|Аудитория      |ИТ-специалисты и разработчики облачных решений|Бизнес-пользователи и бизнес-лица, ответственные за принятие решений|
|Релевантность решения      |Технические приложения, созданные для (или в) Azure|Бизнес-приложения, использующие или расширяющие возможности Dynamics 365, Power BI, Office 365, Azure (надстройки или надстройки готовых приложений)|
|Варианты публикации      |Связь со мной, пробная версия, Transact или службы для Azure|Свяжитесь со мной, пробной версией или службами для Dynamics 365 и Power BI|
|Работа в продукте      |Портал Azure|Dynamics 365, Office 365, клиентские приложения Office| 
|Другой      |Локальная публикация с помощью Azure Stack, консультационные службы для Azure, управляемые службы для Azure|Консультационные услуги для Dynamics 365 и Power BI|

Предложения консультационных услуг, основанные на Dynamics 365, Power Apps и Power BI, могут быть перечислены в AppSource. Чтобы получить дополнительные сведения, скачайте PDF-файл [рекомендаций по Microsoft AppSource консультационных услуг](https://go.microsoft.com/fwlink/?LinkId=828734&clcid=0x409) .

Предложения консультационных услуг и управляемых служб, основанные на Microsoft Azure, могут быть указаны в Azure Marketplace. Дополнительные сведения см. в статье [Публикация предложения службы консультационных услуг Azure](./partner-center-portal/create-consulting-service-offer.md) и [Публикация управляемых служб в Azure Marketplace](https://docs.microsoft.com/azure/lighthouse/how-to/publish-managed-services-offers).

> [!Note]
> Консультационные услуги в Azure Marketplace должны сосредоточиться на предложениях для технической аудитории и подчеркивать технические решения и службы Azure.
>
> Консультационные услуги в AppSource должны сосредоточиться на предложениях пользователей с опытом предоставления качественных решений для Dynamics 365, Power BI или Power Apps.
 
> [!TIP]
> Для публикации в витрине Azure Marketplace в раскрывающемся списке **основной продукт** центра партнеров выберите **Azure**.
>
> Чтобы опубликовать в AppSource, в раскрывающемся списке **основной продукт** центра партнеров выберите **Dynamics 365**, **Power BI**или **Power Apps**.

## <a name="the-publishing-process"></a>Процесс публикации
Процесс публикации предложения консультационных услуг в Azure Marketplace и AppSource включает следующие этапы.

1. **Листинг**: Начало создания списка новых предложений путем входа в учетную запись [центра партнеров](https://partner.microsoft.com/dashboard/account/v3/enrollment/introduction/partnership) . В раскрывающемся списке выберите пункт **новое предложение**, а затем выберите **консультационные услуги**. Введите артефакты для предложения и опубликуйте его.
2. **Проверка**. Ваш запрос будет проверен на соответствие требованиям к публикации. Затем мы отправим вам инструкции по размещению предложения консультационных услуг. 
3. **Отправка**. Предоставьте необходимую информацию в соответствии с инструкциями. Все содержимое и вспомогательные материалы должны быть на английском языке. 
4. **Просмотр**. Наша группа маркетологов и редакторов изучит предоставленные материалы и скорректирует их для повышения разборчивости, грамотности и эффективности. 
5. **Проверка**. Заявка возвращается вам на проверку и утверждение. 
6. **Публикация**. Когда все обсуждаемые материалы будут полностью приемлемы для вас и корпорации Майкрософт, предложение будет считаться готовым для публикации в Marketplace. Дата вашего списка зависит от страны или региона, указанных в вашем предложении. Если каталог для выбранной страны или региона уже открыт в Azure Marketplace или AppSource, ваш список появится в течение нескольких часов. В противном случае список появится сразу же после того, как каталог станет активным.



Предложения должны иметь по крайней мере один из следующих пяти типов служб: 

* **Оценка**: Оценка среды клиента для определения применимости решения и предоставления оценки затрат и времени. 
* **Брифинг**. Ознакомление с решением или консультационной услугой для привлечения интереса клиента с использованием схем, демонстрационных материалов и историй клиентов. 
* **Реализация**. Полная установка, предоставляющая клиенту полностью рабочее решение. Мы рекомендуем использовать только такие решения, которые реализуются не дольше двух недель. 
* **Подтверждение концепции**. Ограниченная реализация, позволяющая определить применимость решения к требованиям клиента. 
* **Семинар**. интерактивное сотрудничество, проводимое в локальной среде клиента. Она может содержать обучающие, краткие или демонстрационные версии, основанные на данных или среде клиента. 

> [!NOTE]
> Тип службы " *информационный бюллетень* " должен быть предложен *пользователем*. Вы можете предложить другие четыре консультационные услуги одним из двух способов:
> * **В личном**порядке: в расположении клиента или в средстве партнера.
> * **Виртуально**: через телеконференцию, веб-конференции или удаленную реализацию.

После того как вы настроили ранее указанные элементы, Группа проверки содержимого проверит, что ваше описание предложения является информативным и полным для целевой аудитории. 

#### <a name="the-listing-title"></a>Заголовок списка

Поле Title имеет ограничение в 50 символов и должно передавать длительность и тип службы предложения, чтобы обеспечить максимальную оптимизацию поисковой системы (SEO). Требуемый формат — " *Имя: тип длительности*".

Примеры:


|Пример заголовка  |Правильный формат  |
|---------|---------|
|Приступая к работе с Центром Интернета вещей Azure в сфере производства    |   Интернет вещей в сфере производства: 2-дневная проверка      |
|Семинар по интеллектуальным тостерам        |  Интеллектуальные тостеры: 1-недельный семинар       |
|  Подтверждение концепции миграции SQL Server    | Миграция SQL: 3-недельное подтверждение концепции        |

В центре партнеров можно добавить форматирование в список с помощью [Markdown](/contribute/markdown-reference), упрощенного языка проектирования, который поможет добавить в список польского. 

> [!TIP]
> Вот несколько примеров Markdown советов:
> * Для форматирования *курсивом* добавьте символ звездочки\*(), без пробела, в начале и в конце слова или ряда слов (например, введите \*Content *, чтобы форматировать слово как *содержимое*).
> * Вставить разрыв строки после двоеточия (:) , чтобы ввести маркированный или нумерованный список.
> * Чтобы создать маркированный список, введите звездочку (\*), за которой следует пробел, для каждого маркированного элемента (например, \*[пробел] Первое слово содержимого).
> * Чтобы создать заголовки, введите один или несколько знаков фунта (\#), за которыми следует пробел (например, # # # [Space] повестка или конечные результаты).
> * Добавить разрыв строки после каждого заголовка.
> * Для **форматирования полужирным шрифтом** добавьте двойную\*\*звездочку (), без пробела, в начале и в конце слова или ряда слов (например, \* \*введите содержимое\* \* , чтобы форматировать слово как **содержимое**). 
> * Отформатируйте *сводку предложения* курсивом.

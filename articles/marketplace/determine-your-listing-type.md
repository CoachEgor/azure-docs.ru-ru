# <a name="determine-your-publishing-option"></a>Выбор варианта публикации
Вариант публикации, выбранный для вашего предложения, определяется требованиями соответствия и преимуществами GTM в Marketplace. Что более важно, выбор варианта публикации и типа предложения определяет, как пользователи будут взаимодействовать с вашим предложением Marketplace.

Чтобы настроить предложение, необходимо ознакомиться со следующими основными концепциями Marketplace: вариантами публикации, типами и конфигурацией, а также вызовами действий, которые будут определять, как и где ваше предложение представлено в витринах Marketplace.

![](./media/marketplace-publishers-guide/storefronts_options_table.png)


Прочитав эту статью, вы научитесь...
<ul><li>    определять подходящий онлайн-магазин для своего решения; </ul></li>
<ul><li>    определять варианты публикации и призывы к действию, доступные в каждом онлайн-магазине; </ul></li>
<ul><li>    определять типы предложений, доступные для каждого варианта публикации. </ul></li>


## <a name="selecting-a-storefront-publishing-option-and-offer-type-for-your-solution"></a>Выбор онлайн-магазина, варианта публикации и типа предложения для решения

Прежде чем выбрать вариант публикации, важно понимать требования к допустимости в витрине для решений, приложений и служб Marketplace:

Приложения **Azure Marketplace** — это технические решения, встроенные или встроенные в Azure и предназначенные для аудитории ИТ или разработчиков. Консультационные услуги в Azure Marketplace — предложения профессиональных услуг, которые помогают клиентам начать работу с Azure или повысить ее эффективность.

**AppSource** приложения — это бизнес-решения, которые могут быть встроенными в Azure или встроенными в составе: Dynamics 365, Office 365, Power BI или Power Apps. Консультационные услуги в AppSource — предложения профессиональных услуг, которые помогают клиентам начать работу с Dynamics 365 и Power BI или повысить ее эффективность.


## <a name="understand-storefront-selection"></a>Выбор правильного онлайн-магазина

Каждый витринный магазин обслуживает уникальные клиентские требования. Магазин, в котором будет опубликовано ваше предложение, будет определяться по типу предложения, предоставлять возможности transactа и категории и (или) отраслевый выбор для конкретной аудитории. 

>[!Note]
>"Перекрестное перечисление" (только для приложений SaaS): Если список или пробное предложение удовлетворяют критериям как для технического, так и для бизнес-аудитории, ваше предложение будет указано в обоих витринах. Дополнительные сведения о вариантах публикации см. ниже.

## <a name="choose-a-publishing-option"></a>Выбор варианта публикации

Доступные варианты публикации предлагают дифференцированное взаимодействие с клиентами, предоставляя вам доступ к преимуществам совместного использования и [коммерческой Marketplace](https://docs.microsoft.com/azure/marketplace/gtm-your-marketplace-benefits).  Обратите внимание на призывы к действию, которые соответствуют разным вариантам публикации.

| **Вариант публикации**    | **Описание**  |
| :------------------- | :-------------------|
| **Список** | Простое описание приложения или службы, которое позволяет пользователю Marketplace попросить вас связаться с клиентом с помощью призыва к действию **Свяжитесь со мной**. |
| **Пробная версия** | Используйте Marketplace для улучшения возможности обнаружения и автоматизации подготовки пробной версии вашего решения, позволяя потенциальным пользователям использовать возможности SaaS, IaaS или Майкрософт в течение ограниченного времени до их приобретения. Призывы к действию для варианта публикации "Пробная версия" — **Бесплатная пробная версия** или **Тестовый выпуск**. |
|**BYOL**  |Воспользуйтесь Marketplace, чтобы улучшить обнаружение и автоматизировать подготовку решения, а также выполнить отдельную финансовую транзакцию. Типы предложений BYOL идеально подходят для миграций между локальными средами и облачными средами. Вызов действия — **теперь получить его**.
| **Транзакция** | Приложение выполняется в Azure и может быть подготовлено в качестве ресурса непосредственно в подписке Azure клиента, когда клиент выбирает действие **получить сейчас** . Плата за лицензии на программное обеспечение может дополнительно быть приобретена и взиматься через клиентский выбор платежного средства и условий, и вы можете предоставить доступ к **бесплатной пробной версии по** с ограниченным сроком действия (доступно только для Azure Marketplace). |

>[!Note]
>Если вы используете вариант публикации Transact, перед выбором типа предложения и его созданием важно составить представление о ценах, выставлении счетов и оплате. Дополнительные сведения см. в статье о [выставлении счетов в Marketplace и основных коммерческих аспектах](./marketplace-commercial-transaction-capabilities-and-considerations.md).

# <a name="determine-your-publishing-option"></a>Выбор варианта публикации

Выбор варианта публикации для вашего предложения напрямую связан как с требованиями, так и с преимуществами коммерческой платформы GTM. Что еще важнее, выбор варианта публикации и типа предложения определяет то, каким образом пользователи будут взаимодействовать с вашим предложением на коммерческой платформе.

Чтобы настроить предложение, необходимо ознакомиться со следующими основными понятиями коммерческого рынка: варианты публикации, типы предложений и конфигурации, а также параметры перечисления, которые будут определять, как и где ваше предложение представлено в Интернет-магазинах коммерческого рынка.

В этой статье вы узнаете следующее:

- Определение соответствующего Интернет-магазина для решения.
- Параметры публикации и листинга доступны в каждом сетевом магазине.
- Какие типы предложений доступны для каждого варианта публикации.

## <a name="choose-a-listing-option"></a>Выбор параметра перечисления

Доступные варианты перечисления предлагают дифференцированное взаимодействие с клиентами, предоставляя вам доступ к преимуществам совместного использования и [коммерческого рынка](https://docs.microsoft.com/azure/marketplace/gtm-your-marketplace-benefits). Обратите внимание на параметры листинга, соответствующие параметру публикации:

| **Вариант публикации**    | **Описание**  |
| :------------------- | :-------------------|
| **Список** | Простой список вашего приложения или службы, который позволяет пользователю коммерческого магазина попросить вас связаться с клиентом с помощью параметров список **контактов** . |
| **Пробная версия** | Коммерческая платформа расширяет возможности поиска и автоматической подготовки пробной версии решения, позволяя потенциальным пользователям бесплатно работать с приложением SaaS, IaaS или Майкрософт в течение ограниченного времени перед покупкой. Параметры листинга, используемые для пробной публикации, — это **Бесплатная пробная версия** или **тестовый**выпуск. |
| **BYOL** | Коммерческая платформа расширяет возможности поиска и автоматической подготовки решения, а также полностью берет на себя финансовые транзакции. Типы предложений с использованием собственной лицензии идеально подходят для переноса локальных данных в облако. Параметр List **теперь получает**.
| **Транзакция** | Предложения с типом "Транзакция" продаются через коммерческую платформу. Выставление счетов и сбор оплаты берет на себя корпорация Майкрософт. Параметр List **теперь получает**.|

> [!Note]
> Если вы используете вариант публикации Transact, перед выбором типа предложения и его созданием важно составить представление о ценах, выставлении счетов и оплате. Дополнительные сведения см. в статье [Управление транзакциями на коммерческой платформе Майкрософт](./marketplace-commercial-transaction-capabilities-and-considerations.md).

## <a name="selecting-an-online-store"></a>Выбор Интернет-магазина

Каждый Интернет-магазин выполняет различные требования клиентов для бизнеса и ИТ-решений. Тип предложения, возможности transactа и Категория определяют, где будет опубликовано предложение. Категории и подкатегории сопоставляются с каждым из Интернет-магазинов в зависимости от типа публикуемого решения:

**Microsoft AppSource** предлагает бизнес-решения, такие как отраслевые решения и консультационные услуги, для Dynamics 365, Microsoft 365 и Power Platform.

**Azure Marketplace** предлагает ИТ Solutions, созданные для Azure, а также службы консультационных услуг, которые ускоряют использование Azure клиентами.

Выберите категорию и подкатегорию, которые лучше согласованы с типом решения. Например, брандмауэр веб-приложения — это решение ИТ, которое должно быть опубликовано в Azure Marketplace в категории безопасности. Приложение управления контрактами — это бизнес-решение, которое должно быть опубликовано в AppSource в категории "продажи". Выбор неверной категории или подкатегории может привести к публикации предложения в неправильном Интернет-магазине.

### <a name="publishing-to-both-online-stores-saas-offers-only"></a>Публикация в обоих Интернет-магазинах (только предложения SaaS)

Предложения SaaS можно опубликовать в Azure Marketplace или AppSource. Если ваше предложение *SaaS является ИТ-решением* (Azure Marketplace) и бизнес-решением (AppSource), выберите категорию и/или подкатегорию, применимую к каждому из Интернет-магазинов. Предложения, опубликованные как в Интернет-магазинах, должны иметь преимущества в качестве ИТ-решений *и* решения для бизнеса.

> [!IMPORTANT]
> Предложения SaaS с [лимитным выставленным счетом](partner-center-portal/saas-metered-billing.md) доступны в Azure Marketplace и портал Azure. Предложения SaaS с частными планами доступны через портал Azure.

| Выставление счетов | Общедоступный план | Частный план | Доступно в: |
|---|---|---|---|
| Да             | Да         | Нет           | Azure Marketplace и портал Azure |
| Да             | Да         | Да          | Azure Marketplace и портал Azure * |
| Да             | Нет          | Да          | Только портал Azure |
| Нет              | Нет          | Да          | Только портал Azure |

&#42; частный план предложения будет доступен только через портал Azure

Например, предложение с Лимитным счетом и только частный план (без общего плана) будет приобретено клиентами в портал Azure. Дополнительные сведения о [частных предложениях в коммерческом магазине Майкрософт](private-offers.md).

### <a name="categories"></a>Категории

Категории и подкатегории сопоставляются с каждым из Интернет-магазинов на основе типа решения. Выберите категории и подкатегории, которые лучше согласованы с вашим решением. Можно выбрать:

- По крайней мере одна и две категории. Можно выбрать первичную и вторичную категории.
- До двух подкатегорий для каждой первичной и (или) вторичной категории. Если не выбрать подкатегорию, предложение будет по-прежнему обнаруживаться в выбранной категории.

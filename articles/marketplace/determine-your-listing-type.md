# <a name="determine-your-publishing-option"></a>Выбор варианта публикации
Вариант публикации, выбранный для вашего предложения, определяется требованиями соответствия и преимуществами GTM в Marketplace. Что более важно выбор публикации, введите предложение определяет, как пользователи будут взаимодействовать с предложением marketplace.

Чтобы настроить свое предложение, необходимо понимать следующие концепции ключа marketplace: параметры публикации, Предложить типы и конфигурации и вызовы к действию, управляющий как и где ваше предложение представлено в онлайн-магазина marketplace.

![](./media/marketplace-publishers-guide/storefronts_options_table.png)


Прочитав эту статью, вы научитесь...
<ul><li>    определять подходящий онлайн-магазин для своего решения; </ul></li>
<ul><li>    определять варианты публикации и призывы к действию, доступные в каждом онлайн-магазине; </ul></li>
<ul><li>    определять типы предложений, доступные для каждого варианта публикации. </ul></li>


## <a name="selecting-a-storefront-publishing-option-and-offer-type-for-your-solution"></a>Выбор онлайн-магазина, варианта публикации и типа предложения для решения

Прежде чем выбрать вариант публикации, важно понять требования онлайн-магазина к соответствию для решений, приложений и служб Marketplace.

**Azure Marketplace** приложения, техническая «стандартных блоков» решения на основе или созданных для Azure и предназначенный для ИТ или аудитория разработчиков. Консультационные услуги в Azure Marketplace — предложения профессиональных услуг, которые помогают клиентам начать работу с Azure или повысить ее эффективность.

Приложения **AppSource** — это бизнес-решения, которые могут быть созданы на основе Azure либо для Dynamics 365, Office 365, Power BI или Power Apps. Консультационные услуги в AppSource — предложения профессиональных услуг, которые помогают клиентам начать работу с Dynamics 365 и Power BI или повысить ее эффективность.


## <a name="understand-storefront-selection"></a>Выбор правильного онлайн-магазина

Storefront, где будут представлены предложения, Azure Marketplace или AppSource, автоматически определяется вашей подробные сведения о предложении и целевая аудитория, а также категорий и отраслей, выбранном вами при создании предложения. 

>[!Note]
>"Перекрестный список" (только для приложений SaaS): если предложение в списке или предложение на основе пробной версии соответствует критериям для технической и бизнес-аудитории, оно появится в обоих онлайн-магазинах. Дополнительные сведения о вариантах публикации см. ниже.

## <a name="choose-a-publishing-option"></a>Выбор варианта публикации

Предлагаемые варианты публикации обеспечивают дифференцированное привлечение клиентов, кроме того, они дают вам доступ к единой базе потенциальных клиентов и [преимуществам выхода на рынок](https://partner.microsoft.com/reach-customers/gtm). Обратите внимание на призывы к действию, которые соответствуют разным вариантам публикации.

| **Вариант публикации**    | **Описание**  |
| :------------------- | :-------------------|
| **Список** | Простое описание приложения или службы, которое позволяет пользователю Marketplace попросить вас связаться с клиентом с помощью призыва к действию **Свяжитесь со мной**. |
| **Пробная версия** | Используйте marketplace для улучшения поиска и автоматизировать подготовку решения пробной версии, теперь потенциальным пользователи могут использовать процесс в приложении SaaS, IaaS или Microsoft бесплатно в течение ограниченного времени перед покупкой. Призывы к действию для варианта публикации "Пробная версия" — **Бесплатная пробная версия** или **Тестовый выпуск**. |
|**BYOL**  |Используйте marketplace для улучшения поиска и автоматизация подготовки решения и завершения финансовые транзакции отдельно. Типы предложений BYOL идеально подходят для локальных облаками. Призыв к действию выражен **получить сейчас**.
| **Транзакция** | Приложение работает в Azure и может предоставляться как ресурс непосредственно в подписке Azure клиента, когда клиент выбирает **получить сейчас** призыв к действию. Лицензионные сборы за ваше ПО будут оплачиваться клиентом на основе выбранного им платежного средства и условий, а вы можете предложить ограниченный по времени доступ к **бесплатной пробной версии программного обеспечения** (доступно только для Azure Marketplace). |

>[!Note]
>Если вы используете вариант публикации Transact, перед выбором типа предложения и его созданием важно составить представление о ценах, выставлении счетов и оплате. Дополнительные сведения см. в статье о [выставлении счетов в Marketplace и основных коммерческих аспектах](./marketplace-commercial-transaction-capabilities-and-considerations.md).

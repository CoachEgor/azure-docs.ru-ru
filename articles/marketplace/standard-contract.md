# <a name="standard-contract"></a>Стандартный контракт

Чтобы упростить процесс приобретения клиентов и снизить юридическую сложность для поставщиков программного обеспечения, корпорация Майкрософт предлагает шаблон стандартного контракта для упрощения транзакции в Marketplace. Вместо создания пользовательских условий, издатели Azure Marketplace могут предлагать программное обеспечение в соответствии со стандартным контрактом, который клиенты должны венесуэльскому и принимать только один раз. Стандартный контракт можно найти здесь: [https://go.microsoft.com/fwlink/?linkid=2041178](https://go.microsoft.com/fwlink/?linkid=2041178). 

Положения и условия предложения определяются на вкладке Marketplace при создании предложения в Портал Cloud Partner. Стандартный параметр контракта включается путем изменения параметра на Да.

![Включение стандартного параметра контракта](media/marketplace-publishers-guide/standard-contract.png)

>[!Note] 
>Если вы решили использовать стандартный контракт, то для канала [поставщика облачных решений](./cloud-solution-providers.md) будут по-прежнему применяться отдельные условия.

## <a name="standard-contract-amendments"></a>Поправки стандартного контракта

Поправка стандартного контракта позволяет издателям выбрать стандартный контракт для простоты и настроить условия для их продукта или бизнеса.  Клиентам необходимо только просмотреть внесенные в контракте поправки, если они уже проверили и приняли Соглашение Microsoft Standard.

Для издателей Azure Marketplace доступно два типа поправности:

* Универсальные поправки. Эти действия применяются к стандартному контракту для всех клиентов. Универсальные поправки отображаются для каждого клиента продукта в потоке покупки.

![Универсальные поправки](media/marketplace-publishers-guide/universal-amendaments.png)

* Настраиваемые поправки. в Azure Marketplace также предусмотрена возможность предоставления пользовательских поправок, нацеленных на клиентов. Они представляют собой особое значение для стандартного контракта, предназначенного только для определенных клиентов. Издатели могут выбрать целевой клиент. Клиенты из этого клиента будут покупать продукт в соответствии со стандартным контрактом и целевыми разправками.

![Пользовательская поправка](media/marketplace-publishers-guide/custom-amendaments.png)

>[!Note] 
>Клиенты, нацеленные на пользовательскую поправку, также получат универсальную поправку на стандартные условия во время приобретения.

>[!Note]
>Следующие типы предложений поддерживают стандартные действия с контрактом: приложения Azure (шаблоны решений и управляемые приложения), виртуальные машины, контейнеры, приложения-контейнеры.

### <a name="customer-experience"></a>Взаимодействие с пользователем

В ходе процесса покупки в портал Azure клиенты смогут увидеть термины, связанные с продуктом, как стандартный контракт Майкрософт и его поправку.

![Портал Azure взаимодействие с пользователем.](media/marketplace-publishers-guide/ibiza-customer-experience.png)

### <a name="api"></a>API

Клиенты могут использовать `Get-AzureRmMarketplaceTerms`, чтобы получить условия предложения и принять его. Стандартный контракт и связанные с ним поправки будут возвращены в выходных данных командлета.

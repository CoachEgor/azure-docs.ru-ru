# <a name="standard-contract-for-the-microsoft-commercial-marketplace"></a>Стандартный контракт для коммерческого рынка Майкрософт

Чтобы упростить процесс закупок для клиентов и снизить юридическую сложность для поставщиков программного обеспечения, корпорация Майкрософт предлагает стандартный контракт для коммерческого рынка Майкрософт, чтобы облегчить транзакции на рынке. Вместо того, чтобы создавать пользовательские условия, коммерческие издатели рынка могут выбрать, чтобы предложить свое программное обеспечение в соответствии со Стандартным контрактом, который клиенты должны только проверить и принять один раз. Стандартный контракт можно найти [https://go.microsoft.com/fwlink/?linkid=2041178](https://go.microsoft.com/fwlink/?linkid=2041178)здесь: .

Условия оферты определяются при создании предложения в Partner Center. Вы можете использовать Стандартный контракт для коммерческого рынка Майкрософт вместо предоставления собственных пользовательских условий.

>[!Note]
>Как только вы публикуете предложение, используя стандартный контракт для коммерческого рынка Майкрософт, вы не сможете использовать свои собственные пользовательские условия. Это "или" сценарий. Вы либо предлагаете свое решение по стандартному контракту, *либо* на своих условиях. Если вы хотите изменить условия Стандартного контракта, вы можете сделать это с помощью стандартных поправок к контракту.

## <a name="standard-contract-amendments"></a>Стандартные поправки к контракту

Стандартные поправки к контракту позволяют издателям выбирать Стандартный контракт для простоты, а также с индивидуальными условиями для своего продукта или бизнеса. Клиентам необходимо ознакомиться с поправками к договору только в том случае, если они уже рассмотрели и приняли стандартный контракт Майкрософт.

Есть два вида поправок, доступных для издателей коммерческого рынка:

* Всеобщие поправки: Эти поправки применяются повсеместно к Стандартному контракту для всех клиентов. Универсальные поправки показаны каждому клиенту предложения в потоке покупки. Клиенты должны принять условия Стандартного контракта и поправки, прежде чем они смогут использовать ваше предложение.

* Пользовательские поправки: Эти поправки являются специальными поправками к Стандартному договору, которые предназначены для конкретных клиентов только через иные иутмы арендаторов Azure. Издатели могут выбрать арендатора, на который они хотят нацелиться. Только клиенты от арендатора будут представлены пользовательские условия изменения в потоке покупки предложения.  Клиенты должны принять условия Стандартного контракта и поправки(ы), прежде чем они смогут использовать ваше предложение.

>[!Note]
>Эти два типа поправок стек друг на друга. Клиенты, ориентированные на пользовательские поправки, также получат универсальную поправку к Стандартному контракту во время покупки.

Вы можете использовать Стандартный контракт для коммерческого рынка Майкрософт для следующих типов предложений: Azure Applications (шаблоны решений и управляемые приложения), виртуальные машины, контейнеры, контейнерные приложения, модули IoT Edge и SaaS.

## <a name="api"></a>API

Клиенты могут использовать Get-AzureRmMarketplaceTerms для получения условий предложения и его принятия. Стандартный контракт и связанные с ним поправки будут возвращены в выходе cmdlet.

# <a name="virtual-machine-test-drive-tab"></a>Вкладка тестового выпуска виртуальной машины

Вкладка **Тестовый выпуск** на странице **Новое предложение** позволяет предоставлять потенциальным потребителям практическую, самостоятельную демонстрацию ключевых возможностей и преимуществ продукта в стандартном сценарии.  Тестовый выпуск является дополнительной возможностью для типов предложений, которые его поддерживают.  Для правильной реализации тестовому выпуску требуются вспомогательные ресурсы.  Дополнительные сведения о тестовом выпуске Azure Marketplace см. в [этой статье](https://azure.microsoft.com/blog/azure-marketplace-test-drive/).  <!--TD: Replace with migrated version of Test Drive article! -->

Чтобы включить эту функцию, на вкладке **Тестовый выпуск** выберите для **Enable a Test Drive** (Включить тестовый выпуск) значение **Да**.  На вкладке **Тестовый выпуск** появятся поля, доступные для редактирования.  Символ звездочки (*) возле имени поля означает, что оно является обязательным.

![Вкладка "Тестовый выпуск" в окне "Новое предложение" для виртуальных машин](./media/publishvm_007.png)


## <a name="field-values"></a>Полевые значения

В следующей таблице описаны назначение и содержимое этих полей.  Обязательные поля помечены звездочкой (*).


|    Поле                  |       Описание                                                            |
|  ---------                |     ---------------                                                          |
|  *Подробно*   |  |
| **Описание\***           | Предоставьте общие сведения о сценарии тестового выпуска. Этот текст будет отображаться для пользователя во время подготовки тестового выпуска. Это поле поддерживает базовый HTML, поэтому вы можете предоставить форматированное содержимое.  |
| **Руководство пользователя\***           | Отправьте подробное руководство пользователя (PDF-файл), из которого пользователи тестового выпуска смогут узнать, как использовать ваше решение.  |
| **Test Drive Demo Video** (Демонстрационное видео о тестовом выпуске) | Отправьте видео, демонстрирующее решение.  Если вы выбрали этот параметр, необходимо указать имя, URL-адрес видео (размещенное на YouTube или Vimeo) и эскиз (533 x 324 пикселей) для него. |
| *Техническая конфигурация* |  |
| **Экземпляры\***             | Укажите доступность по регионам и относительную доступность экземпляра виртуальной машины (щелкните значок информации для получения дополнительных сведений).  <br/>Потенциальные одновременные сеансы тестового выпуска не должны превышать квоту для подписки.  Она вычисляется следующим образом: [количество выбранных регионов x количество "горячих" экземпляров] + [количество выбранных регионов x количество "теплых" экземпляров] + [количество выбранных регионов x количество "холодных" экземпляров] |
| **Длительность тест-драйва\***   | Максимальная длительность сеанса (в часах). Сеанс тестового выпуска автоматически завершит работу после превышения этого периода времени.  |
|**Шаблон тест-драйва ARM\***| Отправьте шаблон Azure Resource Manager, связанный с этим тестовым выпуском. Дополнительные сведения см. в статье [Transforming Virtual Machine Deployment Template for Test Drive](https://github.com/Azure/AzureTestDrive/wiki/Transforming-Virtual-Machine-Deployment-Template-for-Test-Drive) (Преобразование шаблона развертывания виртуальной машины для тестового выпуска). |
| **Информация о доступе\***    | Информация о доступе к Azure Resource Manager и учетные данные пробной версии, записанные в виде обычного текста или простого кода HTML. |
| *Подробная информация о размещении тест-драйва* |  |
| **Идентификатор подписки Azure\*** | Его можно получить, выполнив вход на [портал Microsoft Azure](https://ms.portal.azure.com) и щелкнув **Подписки** в строке меню слева. (Пример: "a83645ac-1234-5ab6-6789-1h234g764ghty")    Этот идентификатор должен быть `a83645ac-1234-5ab6-6789-1h234g764ghty`GUID формы.|
| **Идентификатор арендатора AD Azure\***    | Идентификатор клиента Azure Active Directory.  Его можно получить, выполнив вход на [портал Microsoft Azure](https://ms.portal.azure.com) и щелкнув **Azure Active Directory** в строке меню слева, а затем выбрав **Свойства** в средней строке меню и скопировав **идентификатор каталога** из формы.  Этот идентификатор также должен быть глобальным уникальным идентификатором.  Если не указано, необходимо создать идентификатор клиента для организации. |
| **Идентификатор приложения Azure AD\***       | Идентификатор зарегистрированного решения виртуальной машины Azure  |
| **Ключ приложения Azure AD\***      | Ключ проверки подлинности для зарегистрированного решения |
|   |   |

# <a name="azure-application-marketplace-tab"></a>Вкладка Marketplace для приложения Azure

Вкладка Marketplace позволяет ввести описание приложения Azure и предоставить маркетинговые ресурсы. Эта вкладка содержит следующие формы. Overview (Обзор), Marketing Artifacts (Маркетинговые артефакты), Lead Management (Управление потенциальными клиентами), Legal (Юридические сведения).

## <a name="overview-form"></a>Форма Overview (Обзор)

Форма Overview (Обзор) включает обязательные и необязательные поля, которые представлены на следующем снимке экрана. Обязательные поля помечены звездочкой (*).

![Форма Overview (Обзор)](./media/azureapp-marketplace-overview.png)

В следующей таблице перечислены параметры, которые позволяют создать онлайн-магазин для вашего предложения.   Поля, дополненный звездочки являются обязательными.

|      Поле         |    Описание    |
|  ---------------   |  ---------------  |
| **Заголовок\***        | Заголовок предложения. Он будет отображаться на видном месте в Marketplace. Максимальная длина составляет 50 символов. |
| **Сводка\***      | Краткое описание предложения. Максимальная длина составляет 100 символов.           |
| **Развернутая Сводка\*** | Более подробное описание предложения (но вы можете сохранить то же описание, что и в поле Summary (Сводка)). Максимальная длина составляет 256 символов.           |
| **Описание\***  | Описание предложения. Максимальная длина составляет 3000 символов. Допускается простое форматирование HTML, например теги &lt;p&gt;, &lt;em&gt;, &lt;ul&gt;, &lt;li&gt;, &lt;ol&gt; и теги заголовков.  |
| **Идентификатор маркетинг\*** | Уникальный URL-адрес, который будет связан с этим предложением. Обычно включает имя организации и имя решения. Максимальная длина составляет 50 символов. Выберите понятный маркетинговый идентификатор для вашей службы. Он будет использоваться в URL-адресах Marketplace для этого предложения. Если идентификатор издателя имеет значение contoso, и вы назначили маркетинговый идентификатор sampleApp, для вашего предложения в Azure Marketplace будет создан URL-адрес https://azuremarketplace.microsoft.com/en-us/marketplace/apps/contoso.sampleApp.  
| **Идентификаторы подписок для предварительного просмотра\*** | Добавьте от 1 до 100 идентификаторов подписок пользователей, которые смогут использовать приложение в режиме предварительной версии. Эти разрешенные подписки получат доступ к предложению в предварительной версии сразу после его публикации (еще до официального выпуска).          |
| **Полезные ссылки**    | При необходимости можно предоставить ссылки на различные ресурсы для пользователей вашего предложения, например поддержка, документации, форумы и т. д.  Рекомендуется добавить по крайней мере одну ссылку к документации.            |
| **Предлагаемые категории (макс. 5)\*** | Выберите одного до пяти категорий. Выбранные категории используются для сопоставления вашего предложения с категориями продуктов, доступными в Azure Marketplace и на портале Azure. Они будут отображаться на страницах просмотра и на странице сведений о продукте. |
|  |  |


## <a name="marketing-artifacts"></a>Marketing Artifacts (Маркетинговые артефакты)

Форма Marketing Artifacts (Маркетинговые артефакты) включает обязательные и необязательные поля, которые представлены на следующем снимке экрана. Обязательные поля помечены звездочкой (*).

![Форма Marketing Artifacts (Маркетинговые артефакты)](./media/azureapp-marketplace-artifacts.png)

В следующей таблице описаны поля раздела маркетинговых артефактов.

|      Поле         |    Описание    |
|  ---------------   |  ---------------  |
| **Малый\***        | Эмблемы: Формат PNG, размер 40x40 пикселей.     |
| **Средний\***       | Среднего логотипа: 90 x 90 пикселов в формате PNG    |
| **Крупный\***        | Большого логотипа: пиксели 115 × 115 в формате PNG   |
| **Расширенный\***         | Широкий логотип: пиксели 255 × 115 в формате PNG    |
| **Имиджевый**           | Необязательный имиджевого логотипа: 815 x 290 пикселей в формате PNG. **Примечание.** После передачи имиджевого значка его нельзя удалить. |
| **Снимки экрана (максимум 5)** |        Снимки экрана отображаются на странице сведений о продукте. Это хороший способ наглядно пояснить цели и принципы работы приложения. Например, здесь можно опубликовать схемы архитектуры или примеры использования. Снимки экрана не являются обязательными, и допускается не более 5 снимков на номер SKU. Чтобы отправить снимок экрана, сделайте следующее.<ul><li>Щелкните **+ Добавить снимок экрана**, чтобы открыть окно "Снимок экрана".</li><li>В поле **Имя** введите имя или заголовок длиной не более 100 символов.</li><li>Кнопка **Отправить** позволяет передать изображение. Поддерживается только формат PNG и размер 533×324 пикселей.</li></ul>           |
| **Добавление видео**      | Необязательно, видео, отображаются на вашей странице сведений о продукте. Это хороший способ наглядно пояснить цели и принципы работы приложения. Чтобы добавить видеофайл, сделайте следующее. <ul><li>Выберите **+ Добавить видео**, чтобы открыть окно "Видео".</li><li>В поле **Имя** введите имя или заголовок длиной не более 100 символов.</li><li>В поле **Ссылка** введите URL-адрес сайта, на котором размещается видео (YouTube или Vimeo).</li><li>Поле **Эскиз** позволяет передать эскиз видео. Поддерживается только формат PNG и размер 533×324 пикселей.</li></ul>          |
|  |  |

### <a name="logo-guidelines"></a>Рекомендации по логотипам

Все логотипы, передаваемые на Портале Cloud Partner, должны соответствовать следующим рекомендациям:

- Дизайн Azure отличается простой цветовой палитрой. Используйте в логотипах небольшое число основных и дополнительных цветов.
- Цвета темы портала Azure — белый и черный. Старайтесь не использовать эти цвета для фона логотипов. Используйте цвета, которые сделают логотипы четко видимыми на портале Azure. Мы рекомендуем простые основные цвета. Если вы используете прозрачный фон, логотип и текст не должны быть белого, черного или синего цвета.
- Не используйте в логотипе градиентный фон.
- Не размещайте в логотипе текст, включая название вашей компании или торговую марку. Логотип должен быть "плоским". Также избегайте градиентов.
- Не растягивайте логотип.


#### <a name="hero-logo"></a>Имиджевый логотип

Имиджевый логотип использовать необязательно.

>[!IMPORTANT]
>После отправки имиджевый логотип нельзя удалить.

Придерживайтесь следующих рекомендаций по созданию имиджевого логотипа:

- Фон не может быть черным, белым или прозрачным.
- Не используйте светлые цвета для фона логотипа. Отображаемое имя издателя, название плана и развернутая сводка предложения отображаются в белом цвете и должны выделяться относительно фона.
- Избегайте любых текстовых элементов при разработке логотипа. Когда предложение публикуется, в логотип программным образом вставляются отображаемое имя издателя, название плана, развернутая сводка предложения и кнопка "Создать".
- Включите незанятое пространство прямоугольный формы справа от имиджевого логотипа. Это пространство должно иметь размер 415x100 пикселей и отступ в 370 пикселей от левого края.


## <a name="lead-management"></a>Управление интересами

Форма управления потенциальными клиентами имеет необязательное поле для настройки. Чтобы настроить управление потенциальными клиентами, выберите назначение потенциального клиента из раскрывающегося списка. На следующем снимке экрана показаны доступные назначения.

![Выбор назначения для управления потенциальными клиентами](./media/azureapp-marketplace-leadmgmt.png)

>[!TIP]
>Щелкните значок сведений, чтобы увидеть такое сообщение: "Select the system where your leads will be stored. Learn how to connect to your CRM system [here](https://docs.microsoft.com/azure/marketplace/cloud-partner-portal-orig/cloud-partner-portal-get-customer-leads)." (Выберите систему, в которой будут храниться сведения о потенциальных клиентах. Здесь вы можете узнать, как подключить систему CRM.).

Дополнительные сведения см. в статье [о настройке потенциальных клиентов](https://docs.microsoft.com/azure/marketplace/cloud-partner-portal-orig/cloud-partner-portal-get-customer-leads).


## <a name="legal"></a>Юридическая информация

Форма Legal (Юридическая информация) позволяет указать юридические документы, обязательные для каждого предложения.

Введите следующие сведения:

- **URL-адрес политики конфиденциальности\***  — введите ссылку на политику конфиденциальности приложения.
- **Условия использования\***  — введите условия использования для вашего приложения. Клиентам будет предложено принять эти условия, прежде чем использовать это приложение.

![Форма "Legal" (Юридическая информация)](./media/azureapp-marketplace-legal.png)

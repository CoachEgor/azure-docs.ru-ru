# <a name="azure-application-prerequisites"></a>Необходимые условия для приложения Azure

В этой статье описаны обязательные технические и бизнес требования для публикации управляемых предложений в Azure Marketplace.  Если вы еще этого не сделали, просмотрите следующие источники информации:
- В зависимости от типа SKU, либо [Azure Applications: Руководство по публикации предложений solution Template Или](../../marketplace-solution-templates.md) [приложения Azure: Руководство по публикации предложений управляемых приложений: Руководство по публикации предложений приложений](../../marketplace-managed-apps.md)
- [Шаблоны решений и управляемые приложения для](https://channel9.msdn.com/Events/Build/2018/BRK3603) видео Azure Marketplace


## <a name="technical-requirements"></a>Технические требования

Технические требования включают следующие пункты.

*   Дополнительные сведения о шаблонах Azure Resource Manager см. в статье [Описание структуры и синтаксиса шаблонов Azure Resource Manager](https://docs.microsoft.com/azure/azure-resource-manager/resource-group-authoring-templates). В этой статье описана структура шаблона Azure Resource Manager. Статья содержит информацию о разных разделах шаблона и свойствах, которые доступны в этих разделах. Шаблон состоит из JSON и выражений, на основе которых можно создавать значения для развертывания. 
* Шаблоны быстрого запуска Azure.<br> Дополнительные сведения см. в разделе:

  * [Лазурные шаблоны быстрого запуска](https://azure.microsoft.com/documentation/templates/). Развертывайте ресурсы Azure из Azure Resource Manager с помощью созданных сообществом шаблонов, которые позволяют выполнять больший объем работы. Диспетчер ресурсов Azure позволяет подготавливать приложения с помощью декларативного шаблона. В одном шаблоне можно развернуть несколько служб и их зависимые компоненты. Один шаблон используется для развертывания приложения на каждом этапе его жизненного цикла.
  * [GitHub: Шаблоны ресурсов ресурсного отдела Azure.](https://github.com/azure/azure-quickstart-templates) Этот репозиторий содержит все доступные на данный момент шаблоны Azure Resource Manager, предоставленные сообществом. Индекс для поиска шаблона поддерживается на https://azure.microsoft.com/documentation/templates/.
* Создание определения пользовательского интерфейса<br>
Для получения дополнительной информации смотрите [пользовательский интерфейс портала Azure для управляемого приложения.](https://docs.microsoft.com/azure/azure-resource-manager/managed-application-createuidefinition-overview) В этой статье описывается файл createUiDefinition.json. На основе этого файла портал Azure создает пользовательский интерфейс для создания управляемого приложения.


## <a name="business-requirements"></a>Бизнес-требования

Бизнес-требования включают следующие процедурные, контрактные и юридические обязательства.

* Необходимо зарегистрироваться в качестве издателя в облаке Marketplace. Если вы не зарегистрированы, выполните последующие действия в статье [Станьте издателем облачного рынка](https://docs.microsoft.com/azure/marketplace/become-publisher
).

>[!NOTE]
>Следует использовать одну и ту же учетную запись регистрации Центра разработки для Майкрософт для входа на портал Cloud Partner. У вас должна быть только одна учетная запись Майкрософт для публикации предложений в Azure Marketplace. Эта учетная запись не должна быть специфична для отдельных услуг или предложений.

* Ваша компания (или ее дочерняя компания) должна находиться в зоне продаж из страны/региона, поддерживаемом Azure Marketplace. Для текущего списка этих стран/регионов [см.](https://azure.microsoft.com/support/legal/marketplace/participation-policies/)
* Ваш продукт должен быть лицензирован таким образом, чтобы он был совместим с платежными моделями, поддерживаемыми Azure Marketplace. Подробнее см. статью [Варианты выставления счетов за использование Azure Marketplace](https://docs.microsoft.com/azure/marketplace/marketplace-commercial-transaction-capabilities-and-considerations).
* Вы должны предоставить клиентам техническую поддержку экономически обоснованным образом. Поддержка может быть платной, бесплатной или обеспечиваться посредством поддержки от сообщества.
* Вы несете ответственность за лицензирование программного обеспечения, а также любых зависимостей от стороннего программного обеспечения.
* Необходимо предоставить маркетинговые материалы, отвечающие критериям вашего предложения, соблюдение которых требуется для добавления приложения в Azure Marketplace и на портале Azure.
* Вам нужно принять условия политики участия в Microsoft Azure Marketplace и соглашения для издателей.
* Вам необходимо соответствовать условиям использования веб-сайта Microsoft Azure, заявлению о конфиденциальности корпорации Майкрософт и соглашению по программе сертификации Microsoft Azure.


## <a name="publishing-requirements"></a>Требования к публикации

Чтобы опубликовать новое предложение приложения Azure, необходимо выполнить следующие условия.

* Ваши метаданные должны быть готовы к использованию. Следующий список (не исчерпывающий) содержит примеры этих метаданных:
  * название;
  * описание (в формате HTML);
  * Изображение логотипа (в формате PNG) и в этих фиксированных размерах изображения: 40 x 40 пикселей, 90 x 90 пикселей, 115 x 115 пикселей и 255 x 115 пикселей.
* *Условия использования* и документы *политики конфиденциальности*
* Заяводокументов
* контактные данные службы поддержки.

# <a name="hosted-test-drive"></a>Размещенный тестовый выпуск

Размещенный тестовый выпуск устраняет сложности, связанные с настройкой, за счет размещения в корпорации Майкрософт и обслуживания службы, выполняющей подготовку и отзыв пользователей тестового выпуска. Эта статья предназначена для издателей, которые имеют предложения в AppSource или создают новое и хотят предложить размещенный тестовый выпуск, который подключается к экземпляру Dynamics 365 for Customer Engagement, Dynamics 365 for Finance and Operations или Dynamics 365 Business Central.

## <a name="how-to-publish-a-test-drive"></a>Публикация тестового выпуска

Создайте предложение или перейдите к имеющемуся.

Выберите параметр "Тестовый выпуск" в боковом меню.

Выберите Да для параметра Enable a Test Drive (Включить тестовый выпуск).

Укажите значения для следующих полей в разделе Сведения.

- **Описание** Укажите Общие сведения о ваш тестовый запуск. Этот текст будет отображаться для пользователя во время подготовки тестового выпуска. Это поле поддерживает HTML, поэтому вы можете предоставить форматированное содержимое.
- **Руководство пользователя**: Отправьте подробные пользователь вручную (файл типа .pdf), которая помогает понять, как использовать ваше приложение пользователям тестовый выпуск.
- **Видео о тестовом выпуске демонстрации**: При необходимости отправьте видео, демонстрирующее приложения.

Предоставляйте разрешение AppSource на подготовку и отзыв пользователей тестового выпуска в клиенте с помощью инструкций, расположенных [здесь](https://github.com/Microsoft/AppSource/blob/patch-1/Microsoft%20Hosted%20Test%20Drive/Setup-your-Azure-subscription-for-Dynamics365-Microsoft-Hosted-Test-Drives.md).

На этом шаге вы создадите значения идентификатора Azure AD App и ключа Azure AD App, приведенные ниже.

Укажите значения для следующих полей в разделе Technical Configuration\' (Техническая конфигурация):

- **Тип тестового выпуска**: Выберите \'размещенная Майкрософт (например, Dynamics 365 для взаимодействия с клиентами) "параметр. Это означает, что в корпорации Майкрософт будет размещаться и обслуживаться служба, выполняющая подготовку и отзыв пользователя тестового выпуска.
- **Максимальное количество одновременных тестовые выпуски**: Задайте в этом поле число одновременных пользователей, которые могут иметь active тестовый выпуск в любой момент времени. Каждый пользователь будет использовать лицензию Dynamics, пока активен его тестовый выпуск, поэтому вам необходимо убедиться в наличии по крайней мере такого количества доступных лицензий Dynamics для пользователей тестового выпуска. Рекомендуемое значение: 3–5.
- **Диск продолжительность теста (в часах)** : Выберите значение количества часов, тестовый выпуск будут активными для пользователей. По истечении указанного количества часов пользователь будет отозван из вашего клиента. Рекомендуемое значение — от 2 до 24 часов в зависимости от сложности приложения. Пользователь всегда может запросить еще один тестовый выпуск, если ему не хватило времени и требуется снова получить доступ к тестовому выпуску.
- **URL-адрес экземпляра**: Укажите URL-адрес, который пользователь тестовый выпуск будет изначально открыт при запуске тестовый выпуск. Как правило, это URL-адрес экземпляра Dynamics 365, в котором установлено приложение и демонстрационные данные. Пример значения: https:\//testdrive.crm.dynamics.com
- **Идентификатор клиента Azure AD**: Укажите идентификатор клиента Azure для экземпляра Dynamics 365. Чтобы получить это значение, войдите на портал Azure, перейдите к Azure Active Directory, выберите "Свойства" из колонки меню и скопируйте идентификатор каталога. Пример значения: 72f988bf-86f1-41af-91ab-2d7cd0111234
- **Идентификатор приложения Azure AD**: Идентификатор приложения Azure AD, созданный на шаге 7. \ Пример значения: 53852862-a2ae-4e43-9461-faa49650a096
- **Ключ приложения Azure AD**: Секрета для приложения Azure AD, созданного на шаге 7. \ Пример значения: IJUgaIOfq9b9LbUjeQmzNBW4VGn6grr1l/n3aMrnfdk=
- **Имя клиента Azure AD**: Укажите имя клиента Azure для экземпляра Dynamics 365. Используйте формат \<имя_клиента.\>onmicrosoft.com. Пример значения: testdrive.onmicrosoft.com.
- **Экземпляр URL-адрес веб-API**: Укажите URL-адреса Web API для экземпляра Dynamics 365. Чтобы получить это значение, войдите в экземпляр Microsoft Dynamics 365 и выберите "Параметры -\> Настройка -\> Ресурсы разработчика -\> Веб-API экземпляра" (скопируйте этот URL-адрес). Пример значения: https:\//testdrive.crm.dynamics.com/api/data/v9.0 
- **Имя роли**: Укажите имя настраиваемой роли безопасности Dynamics 365, вы создали для тестового выпуска. Эта роль будет назначаться пользователям во время запуска их тестового выпуска. Пример значения: testdriverole

## <a name="next-steps"></a>Дальнейшие действия

По окончании **публикации** вашего предложения, после приложение сертификации, вы получите **предварительной версии** вашего предложения. Запустите тестовый выпуск в пользовательском интерфейсе и убедитесь, что он работает надлежащим образом. Как только вы считаете себя комфортно с вашим предложением предварительной версии, теперь пора **go live!**

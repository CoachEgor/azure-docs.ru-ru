# <a name="saas-application-test-drive-tab"></a>Вкладка "Тестовый выпуск" приложения SaaS

С помощью вкладки "Тестовый выпуск" вы можете предоставить пробную версию своим клиентам.

## <a name="test-drive-benefits"></a>Преимущества тестового выпуска

Создание пробной версии для клиентов — оптимальный способ обеспечить их уверенностью при покупке. Из доступных вариантов пробной версии тестовый выпуск наиболее эффективен для генерирования высококачественных потенциальных клиентов и их конверсии в реальных.

При использовании тестового выпуска клиенты самостоятельно протестируют пробную версию продукта, ознакомятся с его ключевыми функциями и преимуществами на практике.


## <a name="how-a-test-drive-works"></a>Как работает тестовый выпуск

Потенциальный клиент выполняет поиск и обнаруживает ваше приложение в Marketplace. Клиент выполняет вход и принимает условия использования. На этом этапе клиент получает предварительно настроенную среду, где он может работать с вашим предложением в течение фиксированного количества часов, а вы получаете сведения о высококачественном потенциальном клиенте для дальнейших действий. Дополнительные сведения см. в статье о [тестовом выпуске](https://docs.microsoft.com/azure/marketplace/cloud-partner-portal/test-drive/what-is-test-drive).


## <a name="publishing-steps"></a>Этапы публикации

Основные этапы публикации для добавления тестового выпуска:

1. Определение сценария тестового выпуска.
2. Создание и (или) изменение шаблона Resource Manager.
3. Создание пошагового руководства для тестового выпуска.
4. Повторная публикация предложения.

### <a name="power-bi-test-drive"></a>Тестовый выпуск Power BI

Следуйте инструкциям ниже, чтобы настроить тестовый выпуск.

1. В разделе "Новое предложение" выберите вкладку **Тестовый выпуск**.
2. На вкладке "Тестовый выпуск" выберите **Да**.

   ![Включение тестового выпуска](./media/saas-enable-test-drive.png)

   При включении тестового выпуска отображаются формы сведений и технической конфигурации, которые показаны на следующем снимке экрана.

   ![Форма конфигурации тестового выпуска](./media/saas-test-drive-yes.png)

3. В разделе **Details** (Сведения) укажите сведения для следующих полей:
  
   - Сведения (Описание) — описание тестового выпуска и возможностей его использования. Для форматирования описания можно использовать базовые теги HTML.
   - User Manual (Руководство пользователя) — отправка документа с руководством пользователя, которое клиенты могут использовать при работе с тестовым выпуском. Данное руководство должно быть сохранено в виде файла PDF.
   - Test Drive Demo Video (Демонстрационное видео о тестовом выпуске) [необязательно] — вы можете предоставить видео (на YouTube или Vimeo), которое клиенты могут просмотреть перед началом использования тестового выпуска. Укажите URL-адрес видео.

4. В разделе **Technical Configuration** (Техническая конфигурация) укажите сведения для следующих полей:

   - Type of Test Drive (Тип тестового выпуска) — выберите **Power BI** из раскрывающегося списка.
   - Link to Power BI Dashboard (Ссылка на панель мониторинга Power BI) — укажите ссылку на панели мониторинга.

5. После завершения настройки тестового выпуска нажмите кнопку **Save** (Сохранить).

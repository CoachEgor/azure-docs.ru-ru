# <a name="offer-settings-tab"></a>Вкладка "Параметры предложения"

На экране **нового предложения** первым шагом является создание идентификации предложения. Идентификатор предложения состоит из трех частей: **Идентификатор предложения**, **Идентификатор издателя** и **Имя**. Все они описываются в следующих разделах.

![Создание нового предложения консультационных услуг. Вкладка "Параметры предложения"](media/consultingoffer-settings-tab.png)


### <a name="offer-id"></a>Идентификатор предложения

Этот идентификатор является уникальным именем, которое создается при первой отправке предложения. Оно должно содержать только строчные буквы, цифры, дефисы и символы подчеркивания. **Идентификатор предложения** будет отображаться в URL-адресе и влиять на результаты поисковой системы. Например, *yourcompanyname_exampleservice*.

Как показано в примере, **идентификатор предложения** добавляется к идентификатору издателя, чтобы создать уникальный идентификатор. Он предоставляется как постоянная ссылка, которую можно добавить в закладки, и индексируется поисковыми системами.

>[!Note]
>Как только предложение станет активным, его идентификатор невозможно будет обновить.


### <a name="publisher-id"></a>Идентификатор издателя

Этот идентификатор связан с вашей учетной записью. Когда вы войдете в учетную запись организации, ваш **идентификатор издателя** отобразится в раскрывающемся меню.


### <a name="name"></a>Имя

Эта строка отображается в качестве имени предложения в AppSource или Azure Marketplace. Поле **Имя** имеет ограничение на 50 символов. Рецензенту может потребоваться изменить название, чтобы добавить в название предложения длительность и тип предложения.

В следующем примере показано, как собирается имя предложения. 

![Создание нового предложения консультационных служб](media/cppsampleconsultingoffer.png)

Имя предложения состоит из четырех частей.

-   **Продолжительность:** Определено на вкладке **Storefront Подробности** редактора. Длительность может быть выражена в часах, днях или неделях.
-   **Тип обслуживания:** Определено на вкладке **Storefront Подробности** редактора. Типы услуги бывают `Assessment`, `Briefing`, `Implementation`, `Proof of concept` и `Workshop`.
-   **Предположение:** Вставленрецензент рецензентом.
-   **Имя:** Определяется на странице **Настройки Предложений.**

>[!Note]
>Поле **Имя** имеет ограничение на 50 символов. Рецензенту может потребоваться изменить название, чтобы добавить в название предложения длительность и тип предложения.

В следующем списке приведены несколько понятных имен предложений:

-   Необходимое для профессиональных услуг: 1-часовой брифинг.
-   Платформа для миграции в облако: 1-часовой брифинг.
-   PowerApps и Microsoft Flow: 1-дневный семинар.
-   Лазурное машинное обучение: 3-Wk PoC
-   Решение для розничной торговли Brick and Click: 1-часовой брифинг.
-   Принесите свои собственные данные: 1-Wk семинар
-   Облачная аналитика: 3-дневный семинар.
-   Обучение Power BI: 3-дневный семинар.
-   Решение для управления продажами: реализация за 1 неделю.
-   CRM Быстрый старт: 1-дневный семинар
-   Dynamics 365 for Sales: оценка за 2 дня.

После заполнения вкладки **Offer Settings** (Параметры предложения) сохраните вашу отправку. Теперь имя предложения будет отображаться над областью редактора, и его можно будет найти в разделе **Все предложения**.

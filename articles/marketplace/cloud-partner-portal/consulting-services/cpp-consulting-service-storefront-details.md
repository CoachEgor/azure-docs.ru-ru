# <a name="storefront-details-tab"></a>Вкладка "Сведения об онлайн-магазине"

>[!Important]
>Начиная с 13 апреля 2020 г. Мы начинаем перемещаться по управлению предложениями консультационной службы в центр партнеров. После миграции вы создадите предложения в центре партнеров и будете управлять ими. Следуйте инструкциям в статье [Обзор создания службы консалтинга](https://docs.microsoft.com/azure/marketplace/partner-center-portal/create-consulting-service-offer) , чтобы управлять перенесенными предложениями.

В этой статье описывается, как ввести сведения о вашем онлайн-магазине. Вкладка **сведений об онлайн-магазине** состоит из следующих разделов:

-   **Offer Details (Сведения о предложении);**
-   **Сведения об издателе**
-   **Сведения о выводе**
-   **Маркетинговые артефакты**

![Создание нового предложения консультационных услуг — вкладка со сведениями об онлайн-магазине](media/consultingoffer-storefront-details.png)


## <a name="offer-details"></a>Offer details (Сведения о предложении)

В разделе **Offer details** (Сведения о предложении) содержатся следующие поля:

-   **Offer Summary (Сводка предложения);\***
-   **Описание предложения\***


### <a name="offer-summary"></a>Offer Summary (Сводка предложения);\*

В области кратких сведений о предложении приводится краткое описание вашего предложения, которое отображается сразу под его именем. Введите краткие сведения о предложении в виде обычного текста без разрывов строк. Ниже приведены удачные примеры краткого описания предложений с соответствующими им названиями.

*Пример 1*

-   **Имя предложения**: облачная аналитика: 3-дневный семинар
-   **Сводка предложений**: обзор Microsoft Azure и Power BI, оценки текущей среды и мини-проверки концепции.

*Пример 2*

-   **Название предложения**: промышленный Интернет вещей Azure: срок действия 30-дневного подтверждения концепции
-   **Сводка предложения**. Создайте пилотный проект с промышленным подключением для безопасного подключения оборудования в поле к решению центра Интернета вещей Azure с панелями мониторинга, отчетами и уведомлениями.

*Пример 3*

-   **Название предложения**: профессиональные услуги: 1 — информационный отдел
-   **Сводка по предложениям**: обзор и демонстрация предварительно настроенного решения Dynamics 365 для операций, которое обеспечивает улучшенное управление проектами, выставлением счетов и ресурсами для профессиональных служб.

*Пример 4*

-   **Название предложения**: Power BI в вашем мире: 4 — семинар по персоналу
-   **Сводка предложений**: приступить к работе с первой панелью мониторинга и ознакомьтесь с рекомендациями. Для группы из 12 учащихся и меньше проводится на месте.

*Пример 5*

-   **Название предложения**: Dynamics и Projects: 3-дневная оценка
-   **Сводка по предложениям**: сбор и оценка требований для решения ERP, предназначенного для профессиональных служб и компаний, основанных на проектах.


### <a name="offer-description"></a>Описание предложения\*

Введите описание предложения консультационной услуги в поле **Offer description** (Описание предложения). Хорошее описание предложения должно содержать точные сведения о том, как выглядит взаимодействие и какой конечный результат для клиента. Очевидно, что оно должно помочь клиенту понять, что он получит. Включите сведения о том, как это предложение связано с продуктами корпорации Майкрософт, для которых предлагаются консультационные услуги.

Не вносите в описание предложения свой адрес электронной почты или номер телефона. В предложение включена кнопка **Свяжитесь со мной**, передающая сведения о потенциальных клиентах в целевой объект управления потенциальными клиентами, который вы определяете для предложения.

Введите описание предложения в формате Markdown. Если вы не знакомы с форматом Markdown или HTML, ознакомьтесь со статьей [Использование языка разметки Markdown для написания документации](/contribute/markdown-reference).

Используйте эти форматы, чтобы клиенты могли с легкостью прочесть ваше предложение.

Стремитесь сделать описание предложения кратким и придерживаться ограничения на количество символов, так как пользователям не нравится читать длинный текст. Вы также можете отправлять маркетинговые брошюры, справочные материалы и другие документы с более подробным описанием вашего предложения.

В следующем примере показано хорошо составленное описание предложения, а также связанные с ним имя и краткие сведения.

**Имя предложения**: облачная аналитика: 3-дневный семинар

**Сводка предложений**: обзор Microsoft Azure и Power BI, оценки текущей среды и мини-проверки концепции.

**Описание предложения**. Этот 3-дневный семинар предназначен для технических и бизнес-лидеров и хранится на сайте в клиентской сфере.

Расписание

День 1

-   Основное внимание уделяется тому, как защитить, масштабировать и упорядочить данные в облаке Майкрософт с помощью Azure Data Lake, Azure HDInsight или Хранилища данных SQL Azure.

День 2

-   Описывается настройка и развертывание решений расширенной аналитики с помощью Microsoft R и службы "Машинное обучение Azure".

День 3

-   Описывается, как получать ценные аналитические сведения и операционную аналитику с помощью Power BI, включая сеанс совместной работы по созданию панели мониторинга Power BI.

Конечные результаты

К концу семинара клиент может самостоятельно составить общий план и график реализации решений аналитики и обработки данных в облачной среде Майкрософт.

Следующий пример файла Markdown приведен для предыдущего предложения:

    This 3-day workshop is for technical and business leaders and is held on-site at the client's facility.

      ### Agenda

      **Day 1**

      * Focuses on how to secure, scale, and organize data within the Microsoft cloud by using Azure Data Lake, Azure HDInsight, or Azure SQL Data Warehouse.

      **Day 2**

      * Covers how to configure and deploy advanced analytics solutions with Microsoft R and Azure Machine Learning.

      **Day 3**

      * Covers how to draw actionable insights and operationalize analytics with Power BI and includes a collaborative session to cobuild a Power BI dashboard.


      ### Deliverables
      By the end of the workshop, the client will be able to define a high-level plan and an implementation roadmap for data and analytics solutions in the Microsoft cloud.


## <a name="publisher-information"></a>Publisher Information (Сведения об издателе)

**Идентификатор MPN**

Введите свой девятизначный идентификатор в сети Microsoft Partner Network (MPN). Если у вас нет идентификатора MPN, его можно получить в Центре партнеров Майкрософт.

**Partner Center ID (Идентификатор Центра партнеров)**

Введите новый идентификатор Центра партнеров, если у вас имеется таковой.

**Идентификатор MPN**

Введите секретный ключ для предварительного просмотра вашего предложения в AppSource перед тем, как оно будет активировано.
Этот идентификатор не является паролем.


## <a name="listing-details"></a>Listing Details (Сведения списка)

**Consulting service type (Тип консультационной услуги)**

Корпорация Майкрософт сосредотачивается исключительно на фиксированном объеме, фиксированной продолжительности, оценке или фиксированной цене (или бесплатном предложении) и в первую очередь на предложениях консультационных услуг, ориентированных на предпродажи, для отдельного клиента. Имеются следующие типы услуг — предложение оценки, брифинга, реализации, подтверждения концепции и семинара. Они предоставляются либо на месте, либо виртуально. Магазин консультационных услуг AppSource не поддерживает объявления для управляемых служб и служб подписки.

>[!Note]
>Он не подходит для подписок или обучения по запросу.

Предусмотрены следующие пять типов предложений:

-   **Оценка**: Оценка среды клиента для определения применимости решения и предоставления оценки затрат и времени.
-   **Брифинг**. Ознакомление с решением или консультационной услугой для привлечения интереса клиента с использованием схем, демонстрационных материалов и историй клиентов. Брифинги должны проводиться на месте.
-   **Реализация**. Полная установка, предоставляющая клиенту полностью рабочее решение. Для этой пилотной реализации Майкрософт рекомендует ограничиваться только теми решениями, которые могут быть реализованы за одну неделю или быстрее.
-   **Подтверждение концепции**. Реализация ограниченного диапазона, которая определяет, соответствует ли решение требованиям клиента.
-   **Семинар**. интерактивное сотрудничество, проводимое на предприятии клиента, которое может включать учебные курсы, обзоры, оценки или демонстрационные версии, созданные на основе данных или среды клиента.

**Доступность в различных регионах и странах**

Выберите страну и регион, где доступно это предложение консультационных услуг. Одно предложение нельзя опубликовать в нескольких странах или регионах. Для каждой страны или региона необходимо создать новое предложение.

>[!Note]
>Консультационные услуги AppSource в настоящее время предоставляются в США, Великобритании и Канаде. Вы можете отправить предложение для страны или региона, которые пока еще не активны, и будут рассмотрены и подготовлены к переходу в интерактивный формат. Для открытия новой страны или региона необходимо минимальное число предложений, которые готовы к использованию, поэтому предлагаются страны и регионы, которые не включены в Live.

**Промышленных**

Выберите отрасли, к которым более всего применимо ваше предложение консультационных услуг.

**Длитель**

Выберите число (например, 3 или 4) в поле **Продолжительность**, а затем укажите меру измерения: **Час**, **День** или **Неделя**.

**Primary products** (Основные продукты)

Чтобы опубликовать в Azure Marketplace, выберите **Azure** в качестве основного продукта. Затем выберите соответствующие **области решения**.

Для публикации в AppSource выберите **Dynamics 365**, **Power BI** или **PowerApps** в качестве основного продукта. Можно также выбрать другие соответствующие **применимые продукты**. Затем предложение консультационных услуг отображается в объявлениях, связанных с каждым из этих продуктов в AppSource.

**Релевантные компетенции**

Выберите компетенции, соответствующие этому предложению, чтобы они отображались вместе со сведениями о предложении.

## <a name="marketing-artifacts"></a>Marketing Artifacts (Маркетинговые артефакты)

**Логотип компании (формат PNG, 48 x 48 пикселей)**

Отправьте изображение, которое отображается на плитке вашего предложения на странице просмотра коллекции предложений. Изображение должно быть файлом в формате PNG с разрешением 48 x 48 пикселей.

**Логотип компании (формат PNG, 216 x 216 пикселей)**

Отправьте изображение, которое отображается на странице сведений о вашем предложении. Изображение должно быть файлом в формате PNG с разрешением 216 x 216 пикселей.

**Видеозаписи (ограничено четырьмя)**

Отправьте до четырех видеороликов с практическими примерами клиентов или со ссылками на них. Если у вас их нет, отправьте видеоролик, который описывает, как связан опыт вашей компании с этим предложением. Если есть демонстрационное видео в Power BI или PowerApps, отправьте его также. Ссылки на видео должны вести на YouTube или Vimeo.

**Документы (ограничено тремя)**

Отправьте маркетинговую брошюру с подробным описанием вашего предложения консультационных услуг. Вы также можете отправить обзорный документ о компании, справочные материалы и практические примеры. Убедитесь, что в документе используются текущие наименования рекомендуемых продуктов и не упоминаются продукты конкурентов корпорации Майкрософт.

**Снимки экрана (ограничено пятью)**

Отправьте до пяти изображений, которые содержат дополнительные сведения о предложении, о конечных результатах предложения или о вашей компании. Например, фрагмент вашей маркетинговой брошюры, соответствующий слайд из презентации или рисунок, изображающий темп развития компании или опыт ее работы.

# <a name="update-a-power-bi-app-offer"></a>Обновить предложение приложение Power BI

В этой статье показано, как обновить приложение Microsoft Power BI предложение [облачный портал для партнеров](https://cloudpartner.azure.com/) и затем повторно опубликовать предложение. Ниже приведены некоторые распространенные причины для обновления предложения.

- Чтобы обновить содержимое приложения в Power BI и получить URL-адрес установки из заново упакованные приложения
- Для обновления предложения Azure Marketplace метаданных (продаж, маркетинга, или сведения о поддержке и ресурсами)
 
С помощью портала **сравнения** и **журнал** функции для отслеживания изменений, такие как следующие.

## <a name="common-update-operations"></a>Стандартные операции обновления

Некоторые атрибуты предложения приложение Power BI невозможно изменить после публикации предложения в AppSource. Недопустимые изменения включают в себя **идентификатор предложения** и **идентификатор издателя**. Но по-прежнему можно изменить широкий спектр характеристики. Ниже приведены некоторые распространенные виды изменений.

### <a name="update-app-content-in-power-bi"></a>Обновление содержимого приложения в Power BI

Приложения Power BI часто обновляются периодически с новым содержимым, исправлений системы безопасности, функции и т. д. При изменении приложения обновите его URL-адрес установки, выполнив следующие действия:

1.  Войдите на [облачный портал для партнеров](https://cloudpartner.azure.com/).
2.  В разделе **Все предложения** найдите предложение, которое нужно обновить.
3.  На **технические сведения о** введите новый URL-адрес установки.
4.  Выберите **публикации** запускается рабочий процесс для публикации новой версии приложения в AppSource.


### <a name="update-the-offers-marketplace-metadata"></a>Обновление метаданных для предложения в Marketplace

Метаданные своего предложения Marketplace включает в себя название вашей компании, эмблемы и другие сведения. Чтобы обновить метаданные, выполните следующие действия.

1.  Войдите на [облачный портал для партнеров](https://cloudpartner.azure.com/).
2.  В разделе **Все предложения** найдите предложение, которое нужно обновить.
3.  На **сведения Storefront** вкладке, изменения метаданных, следуя инструкциям в статье [вкладку подробностей Storefront приложение Power BI](./cpp-storefront-details-tab.md).
4.  Щелкните **Опубликовать**, чтобы запустить рабочий процесс публикации изменений.


## <a name="the-compare-feature"></a>Функция сравнения

Поскольку обновить опубликованные предложения, вы можете проверить изменения, внесенные. Чтобы использовать **сравнения** функции:

1.  В любой момент в процессе редактирования, выберите предложение **сравнения** кнопки.

    ![Сравнение обычной кнопки](./media/compare-feature-button.png)

2.  Вы увидите сравнительную таблицу маркетинговых ресурсов и метаданных, указанных в обеих версиях.


## <a name="history-of-publishing-actions"></a>Журнал действий, связанных с публикацией

Для просмотра вашего предложения при публикации на журнал, в левой части на портале Cloud Partner, откройте **журнал** вкладки. Здесь вы увидите предложения находится журнал действий с отметкой времени на AppSource.

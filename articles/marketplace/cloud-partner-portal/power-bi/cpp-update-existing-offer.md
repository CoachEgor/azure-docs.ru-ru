# <a name="update-a-power-bi-app-offer"></a>Обновление предложения Power BI приложения

В этой статье показано, как обновить предложение приложения Microsoft Power BI в [портал Cloud Partner](https://cloudpartner.azure.com/) , а затем повторно опубликовать предложение. Ниже приведены некоторые распространенные причины обновления предложения.

- Обновление содержимого приложения в Power BI и получение URL-адреса установки из нового упакованного приложения
- Чтобы обновить метаданные предложения Azure Marketplace (продажи, маркетинг или сведения о поддержке и ресурсы)
 
Используйте функции **сравнения** и **ведения журнала** на портале, чтобы контролировать подобные изменения.

## <a name="common-update-operations"></a>Стандартные операции обновления

Некоторые атрибуты предложения приложения Power BI нельзя изменить после публикации предложения в AppSource. Недопустимые изменения включают **идентификатор предложения** и **идентификатор издателя**. Но вы все равно можете изменить широкий спектр характеристик. Ниже приведены некоторые распространенные изменения.

### <a name="update-app-content-in-power-bi"></a>Обновление содержимого приложения в Power BI

Power BI приложения часто обновляются с помощью нового содержимого, исправлений для системы безопасности, функций и т. д. При изменении приложения обновите его URL-адрес установки, выполнив следующие действия.

1.  Войдите на [облачный портал для партнеров](https://cloudpartner.azure.com/).
2.  В разделе **Все предложения** найдите предложение, которое нужно обновить.
3.  На вкладке **технические сведения** введите новый URL-адрес установки.
4.  Выберите **опубликовать** , чтобы запустить рабочий процесс, чтобы опубликовать новую версию приложения в AppSource.


### <a name="update-the-offers-marketplace-metadata"></a>Обновление метаданных Marketplace в предложении

Метаданные Marketplace вашего предложения включают название компании, логотипы и другую информацию. Чтобы обновить метаданные, выполните следующие действия.

1.  Войдите на [облачный портал для партнеров](https://cloudpartner.azure.com/).
2.  В разделе **Все предложения** найдите предложение, которое нужно обновить.
3.  На вкладке **сведения о витрине** измените метаданные, следуя инструкциям в статье [Power BI сведения о витрине приложения](./cpp-storefront-details-tab.md).
4.  Щелкните **Опубликовать**, чтобы запустить рабочий процесс публикации изменений.


## <a name="the-compare-feature"></a>Функция сравнения

После обновления опубликованного предложения вы можете проверить внесенные изменения. Чтобы использовать функцию **сравнения** , выполните следующие действия.

1.  В любом месте процесса редактирования нажмите кнопку **сравнения** предложения.

    ![Кнопка "сравнить"](./media/compare-feature-button.png)

2.  Вы увидите сравнительную таблицу маркетинговых ресурсов и метаданных, указанных в обеих версиях.


## <a name="history-of-publishing-actions"></a>Журнал действий, связанных с публикацией

Чтобы просмотреть историю публикации Вашего предложения, в левой части Портал Cloud Partner откройте вкладку **Журнал** . Здесь вы увидите историю действий с отметкой времени в предложениях AppSource.

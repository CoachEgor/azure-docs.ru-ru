# <a name="power-bi-app-storefront-details-tab"></a>Вкладка сведения о Power BI

На странице **«Новое предложение»** используйте вкладку **Storefront Подробности** для предоставления маркетинговой, продаж и правовой информации потенциальным клиентам. На этой вкладке можно также настроить управление для версий, генерируются Azure Marketplace. Эта длинная форма разделена на шесть разделов: **Подробная информация о предложении**, **Подробная информация о листинге**, **Маркетинговые артефакты**, **юридические**, **поддержка клиентов**, и **руководство.**  Звездочка (кв) в конце метки поля означает, что поле требуется.


## <a name="offer-details-section"></a>Раздел "Сведения о предложении"

В разделе **Подробная информация о предложении** введите общую информацию о предложении AppSource.

![Раздел сведений о предложении на вкладке подробностей об онлайн-магазине](./media/offer-details-section.png)

В следующей таблице содержится более подробная информация об этих полях. Обязательные поля помечены звездочкой (*).  

|   Поле               |   Описание                                                                           |
|-----------------------|-----------------------------------------------------------------------------------------|
| **Offer Summary (Сводка предложения);\***     | Краткое назначение приложения, используя до 100 символов.                             |
| **Описание предложения\*** | Описание приложения, в использовании до 3000 символов. Это поле поддерживает простой форматирование HTML. |
|   |    |


## <a name="listing-details-section"></a>Раздел "Сведения списка"

Раздел **«Детали списка»** собирает информацию о контексте приложения: отрасли, в которых оно обычно используется, категория приложения, совместимые продукты и связанные с ними условия поиска.

![Раздел сведений списка на вкладке подробностей об онлайн-магазине](./media/listing-details-section.png)

Для завершения полей в этом разделе используйте следующую таблицу.  Обязательные поля помечены звездочкой (*).
 
|   Поле                                  |   Описание                                                        |
| --------------                           | ---------------------                                                |
| **Отраслей**                           | Выберите отрасль, в которой ваше приложение подходит лучше всего. Если приложение относится к нескольким отраслям, оставьте это поле пустым.      |
| **Категории\***                           | Выберите до трех категорий, относясь к приложению.     |
| **Ссылка на справку для приложения\***               | Предоставьте URL-адрес страницы, которая предлагает онлайн-справку для вашего приложения.           |
| **Продукты, с помощью работы приложений (Max 3)** | Выберите **новый** знак плюс для создания текстового поля. В поле введите название продукта, с которым работает ваше приложение. Перечислите до трех продуктов.       |
| **Поиск ключевых слов (Max 3)**              | Введите до трех ключевых слов, которые пользователи, скорее всего, будут использовать для поиска вашего приложения в AppSource. Например, если приложение называется "Мое приложение для отправки электронной почты", ваши ключевые слова могут быть **электронными письмами,** **рассылкой**и **почтовым приложением.** |
|  |  |


## <a name="marketing-artifacts-section"></a>Раздел Marketing Artifacts (Маркетинговые артефакты)

В разделе **Маркетинг Артефакты,** загрузить брендинг и маркетинговые материалы для отображения на AppSource.  Этот раздел разделен на четыре подраздела: **Логотипы,** **Видео,** **Документы**и **Скриншоты.** Логотипы и скриншоты являются единственными необходимыми артефактами маркетинга. Но для лучшей привлекательности клиента, мы рекомендуем добавить видео и документы, а также.

![Раздел маркетинговых артефактов на вкладке подробностей об онлайн-магазине](./media/marketing-artifacts-section.png)

Для завершения полей в этом разделе используйте следующую таблицу. Обязательные поля помечены звездочкой (*).
 
|    Поле                             |    Описание                                                    |
|   -----------                        |    -------------                                                  |
| *Логотипы*                              |                                                                   |
| **Логотип предложения (формат png, 48x48)\***   | Загрузите логотип для отображения в обзоре приложения или в результатах поиска приложения. AppSource поддерживает только формат PNG, с разрешением 48 x 48 px.  |
| **Логотип предложения (формат PNG, 216x216)\*** | Загрузите логотип для отображения на странице детализации приложения.  AppSource поддерживает только формат PNG, с разрешением 216 x 216 px.  |
| *Видео*                             |                                                                   |
| **Название**                             | Введите имя или название приложения.                                          |
| **URL-адрес**                              | Введите URL-адрес видео, размещенного на YouTube или Vimeo.                              |
| **Эскиз**                        | Добавьте миниатюрное изображение приложения.  AppSource поддерживает только формат PNG, с разрешением 1280 x 720 px.   |
| *Документы*                          | Добавьте до трех документов для отображения на AppSource, под заголовком **«Узнайте больше».**  |
| **Название**                             | Введите имя или название подтверждающего документа.                              |
| **Файл**                             | Загрузите файл PDF.                             |
| *Скриншоты\**                      | Добавьте до пяти скриншотов.                        |
| **Название**                             | Введите имя или название скриншота.                                       |
| **ОС контейнера**                            | Загрузите скриншот PNG. Его разрешение должно быть 1280 x 720 px.  | 
|   |   |

Логотипы, которые вы загружаете на [портал Cloud Partner,](https://cloudpartner.azure.com) *не*должны:

- Используйте градиент. Внешний вид вашего логотипа должен быть плоским.
- Включите название вашей компании, фирменное наименование или другой текст. 
- Смотри растянутым.

## <a name="legal-section"></a>Юридический раздел

В разделе **«Право»** предоставьте два юридических документа, необходимых для каждого предложения: вашу политику конфиденциальности и условия использования.

![Раздел юридических сведений на вкладке сведений об онлайн-магазине](./media/legal-section.png)

Для завершения полей в этом разделе используйте следующую таблицу:

|   Поле                |   Описание                           |
|------------------------|--------------------------------------   |
| **URL-адрес политики конфиденциальности\*** | URL-адрес политики конфиденциальности, размещенной       |
| **Условия использования\***       | Ваша политика использования, отформатированная как простой текст или простой HTML     |
|  |  |


## <a name="customer-support-section"></a>Раздел "Поддержка клиентов"

В разделе **Поддержка клиентов** предоставьте **URL-адрес службы поддержки** для страницы поддержки в Интернете.  Эта страница должна предоставлять несколько вариантов контакта, таких как телефон, электронная почта и чат. 


## <a name="lead-management-section"></a>Раздел Lead Management (Управление потенциальными клиентами)

В разделе **«Ведущий менеджмент»** наведите систему на сбор интересов клиентов, которые предлагает сяритоге AppSource. Чтобы выбрать параметры хранения интересов, используйте следующую таблицу:

|    Поле               |   Назначение потенциального клиента                               |
|------------------------|--------------------------------------            |
|  **Нет**              | Не собирайте никаких зацепок. Это параметр по умолчанию.  |
| **Большой двоичный объект Azure (не рекомендуется)** | Укажите [хранилище Azure Blob,](https://docs.microsoft.com/azure/storage/blobs/storage-blobs-overview) используя имя контейнера и строку соединения.  Этот выбор является deprecated. Вместо этого используйте **таблицу Azure.**  |
| **Лазурный стол**        | Укажите [хранилище таблицы Azure](https://docs.microsoft.com/azure/cosmos-db/table-storage-overview) с помощью строки соединения.  |
| **Динамика CRM Онлайн** | Укажите [Dynamics 365](https://dynamics.microsoft.com/) с помощью URL-адресов и учетных данных проверки подлинности. |
| **Конечная точка HTTPS**     | Укажите конечную точку HTTPS с помощью полезной нагрузки JSON.   |
| **Marketo**            | Укажите экземпляр Marketo, используя идентификатор [сервера,](https://www.marketo.com/) идентификатор munchkin и идентификатор формы.   |
| **Salesforce**         | Укажите [Salesforce](https://www.salesforce.com/) с помощью идентификатора объекта. |
|  |  |

После публикации предложения происходит проверка подключения для передачи сведений о потенциальных клиентах, а затем в указанное назначение автоматически отправляется тестовый потенциальный клиент. Постоянное управление информацией о свинце и оперативнообновляе обновление настроек, чтобы отразить текущую архитектуру управления клиентами.

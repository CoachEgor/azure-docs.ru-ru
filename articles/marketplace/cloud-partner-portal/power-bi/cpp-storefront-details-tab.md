# <a name="power-bi-app-storefront-details-tab"></a>Вкладка Storefront сведения приложения Power BI

На **новое предложение** странице **сведения Storefront** tab, чтобы предоставить маркетинга, продаж и юридические сведения для ваших потенциальных клиентов. На этой вкладке можно также настроить управление для потенциальных клиентов, которые создает Azure Marketplace. Эта длинная форма состоит из шести разделов: **Сведения о предложении**, **Сведения списка**, **Маркетинговые артефакты**, **Юридическая информация**, **Поддержка клиентов** и **Управление потенциальными клиентами**.  Символ звездочки (*) в конце метку поля означает, что поле является обязательным.


## <a name="offer-details-section"></a>Раздел "Сведения о предложении"

В **сведения о предложении** введите общие сведения о вашем предложении AppSource.

![Раздел сведений о предложении на вкладке подробностей об онлайн-магазине](./media/offer-details-section.png)

Следующая таблица предоставляет дополнительные сведения об этих полях. Обязательные поля помечены звездочкой (*).  

|   Поле               |   ОПИСАНИЕ                                                                           |
|-----------------------|-----------------------------------------------------------------------------------------|
| **Сводка предложения\***     | Brief: цель приложения, с помощью до 100 символов.                             |
| **Описание предложения\*** | Описание приложения, с помощью до 3000 символов. Это поле поддерживает простой HTML-форматировании. |
|   |    |


## <a name="listing-details-section"></a>Раздел "Сведения списка"

**Листинг сведений** раздел собирает сведения о контексте приложения: отраслей, где он обычно используется, категория приложений, совместимые продукты и связанные поисковый запрос.

![Раздел сведений списка на вкладке подробностей об онлайн-магазине](./media/listing-details-section.png)

Чтобы заполнить поля в этом разделе, используйте следующую таблицу.  Обязательные поля помечены звездочкой (*).
 
|   Поле                                  |   ОПИСАНИЕ                                                        |
| --------------                           | ---------------------                                                |
| **Отрасли**                           | Выбор отрасли, где лучше ваше приложение. Если отношение вашего приложения в нескольких отраслях, оставьте это поле пустым.      |
| **Категории\***                           | Выберите до трех категорий, которые относятся к приложению.     |
| **Ссылка на справку для вашего приложения\***               | Укажите URL-адрес страницы, которая справку в Интернете для вашего приложения.           |
| **Продукты, с которыми работает ваше приложение (не более 3)** | Выберите **New** плюса, чтобы создать текстовое поле. В поле введите имя продукта, работу приложения. Отображение списка до трех продуктов.       |
| **Ключевые слова для поиска (не более 3)**              | Введите до трех ключевых слов, которые пользователи скорее всего будет использовать для поиска приложения в AppSource. Например, если приложение именуется «Мое приложение отправка», ключевые слова можно **сообщений электронной почты**, **рассылки**, и **приложение "Почта"**. |
|  |  |


## <a name="marketing-artifacts-section"></a>Раздел Marketing Artifacts (Маркетинговые артефакты)

В **маркетинговые артефакты** раздел, фирменная символика передачи и маркетинговые материалы для отображения в AppSource.  Этот раздел состоит из четырех подразделов: **Логотипы**, **Видео**, **Документы** и **Снимки экрана**. Логотипы и снимки экрана — это единственная необходимые маркетинговые артефакты. Но для рекомендации пересмотра клиента, мы рекомендуем добавить видео и документов.

![Раздел маркетинговых артефактов на вкладке подробностей об онлайн-магазине](./media/marketing-artifacts-section.png)

Чтобы заполнить поля в этом разделе, используйте следующую таблицу. Обязательные поля помечены звездочкой (*).
 
|    Поле                             |    ОПИСАНИЕ                                                    |
|   -----------                        |    -------------                                                  |
| *Логотипы*                              |                                                                   |
| **Логотип предложения (формат png, 48 x 48)\***   | Отправьте логотип для отображения в обзоре приложения или в результатах поиска приложения. AppSource поддерживает только формат PNG, с разрешением 48 x 48 пикселей.  |
| **Логотип предложения (формат png, 216 x 216)\*** | Отправьте логотип для отображения на странице сведений о приложении.  AppSource поддерживает только формат PNG, с разрешением 216 x 216 пикселей.  |
| *Видеоролики*                             |                                                                   |
| **Имя**                             | Введите имя или название вашего приложения.                                          |
| **URL-адрес**                              | Введите URL-адрес видео, размещенное на YouTube или Vimeo.                              |
| **Эскиз**                        | Добавьте эскиз приложения.  AppSource поддерживает только формат PNG и с разрешением 1280 x 720 пикселей.   |
| *Документы*                          | Добавить до трех документы для отображения в AppSource, в разделе **Подробнее** заголовок.  |
| **Имя**                             | Введите имя или заголовок вспомогательные документа.                              |
| **File**                             | Отправьте PDF-файла.                             |
| *Снимки экрана\**                      | Добавьте до пяти снимков экрана.                        |
| **Имя**                             | Введите имя или название снимок экрана.                                       |
| **Образ —**                            | Отправьте снимок экрана PNG. Разрешение должно быть разрешение 1280 x 720 пикселей.  | 
|   |   |

Логотипы, загруженный в [облачный портал для партнеров](https://cloudpartner.azure.com) следует *не*:

- Использование градиента. Внешний вид логотип должны быть плоскими.
- Включить имя компании, название торговой марки или другой текст. 
- Выглядеть растянутым.

## <a name="legal-section"></a>Юридический раздел

В **юридические** укажите два юридических документов, необходимых для каждого предложения: Ваша политика конфиденциальности и ваши условия использования.

![Раздел юридических сведений на вкладке сведений об онлайн-магазине](./media/legal-section.png)

Чтобы заполнить поля в этом разделе, используйте следующую таблицу:

|   Поле                |   ОПИСАНИЕ                           |
|------------------------|--------------------------------------   |
| **URL-адрес политики конфиденциальности\*** | URL-адрес политики отправленной конфиденциальности       |
| **Условия использования\***       | Используйте политики, в формате обычного текста или простой HTML     |
|  |  |


## <a name="customer-support-section"></a>Раздел "Поддержка клиентов"

В **поддержки** укажите **URL-адрес поддержки** страницы службу поддержки.  Эта страница должна предоставляют несколько способов связи, например номер телефона, адрес электронной почты и чата в реальном времени. 


## <a name="lead-management-section"></a>Раздел Lead Management (Управление потенциальными клиентами)

В **управления интересами** разделе настройки системы для сбора интересов, которые создают своих предложений в AppSource. Чтобы выбрать параметры хранения для потенциальных клиентов, используйте следующую таблицу:

|    Поле               |   Назначение потенциального клиента                               |
|------------------------|--------------------------------------            |
|  **None**              | Сбор не потенциальных клиентов. Это вариант по умолчанию.  |
| **Большой двоичный объект Azure (не рекомендуется)** | Укажите [хранилище BLOB-объектов Azure](https://docs.microsoft.com/azure/storage/blobs/storage-blobs-overview) с помощью имени контейнера и строку подключения.  Этот выбор является устаревшим. Используйте **таблиц Azure** вместо этого.  |
| **Таблица Azure**        | Укажите [хранилища таблиц Azure](https://docs.microsoft.com/azure/cosmos-db/table-storage-overview) с помощью строки подключения.  |
| **Dynamics CRM Online** | Укажите [Dynamics 365](https://dynamics.microsoft.com/) с помощью URL-адрес и проверку подлинности. |
| **Конечная точка HTTPS**     | Укажите конечную точку HTTPS с помощью полезных данных JSON.   |
| **Marketo**            | Укажите [Marketo](https://www.marketo.com/) экземпляра с помощью сервера идентификатор, идентификатор munchkin и идентификатора формы.   |
| **Salesforce**         | Укажите [Salesforce](https://www.salesforce.com/) с помощью идентификатора объекта. |
|  |  |

После публикации предложения происходит проверка подключения для передачи сведений о потенциальных клиентах, а затем в указанное назначение автоматически отправляется тестовый потенциальный клиент. Постоянно управление сведения об идеях и обновите параметры в соответствии с вашей текущей архитектуре управления клиента.

## <a name="publishing-steps"></a>Этапы публикации

Ниже приведены общие инструкции по публикации предложения приложение Power BI.

![Приложение Power BI предлагают этапы публикации](media/publishing-steps.png)

Ниже приведен процесс публикации предложения приложение Power BI.

1. Создание шаблона приложения в Power BI. Это действие создает пакет установки URL-адрес, который представляет основной технические средства для предложения. Также в настоящее время, повысить уровень пакета тестирования в предварительной рабочей среде. Дополнительные сведения см. в разделе [что такое приложения Power BI шаблона?](https://docs.microsoft.com/power-bi/service-template-apps-overview). 
2. Собирать или создать предложения маркетинговых материалов, включающих: официальное название, описание, логотипы, и т.д. 
3. Собирать или создание предложения юридическим отделом и поддержки документов: *условия использования*, *политику конфиденциальности*, *политика поддержки*, Справка по пользовательскому и т. д.
4. Создайте предложение: позволяет настроить сведения предложения, включая описание предложения, маркетинговые материалы, юридические сведения, сведения о поддержке и спецификации активов на портале Cloud Partner.  После предложения указано полностью, отправьте его для публикации.
5. Отслеживайте процесс публикации на портале Cloud Partner.  На этом этапе команда адаптации AppSource проверяет, проверяет и выполняет сертификацию приложения. 
6. После успешной сертификации приложения, просмотрите его в тестовую среду и освободите его. 
7. Приложение Power BI отображается на AppSource (it «переходит в реальном времени»).
8. В Power BI повысить уровень подготовительных пакет в рабочую среду. Дополнительные сведения см. в разделе [управлять версиями приложения шаблона](https://docs.microsoft.com/power-bi/service-template-apps-create#manage-the-template-app-release).

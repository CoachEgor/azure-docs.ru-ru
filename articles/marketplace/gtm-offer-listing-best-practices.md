# <a name="offer-listing-best-practices"></a>Рекомендации по размещению предложений

В таблицах в этом разделе приведены рекомендации по созданию привлекательных предложений в marketplace. Для записи и настройки описываемых маркетинговых материалов вы будете использовать [Портал Cloud Partner](https://cloudpartner.azure.com/#insights). 

## <a name="storefront-offer-details"></a>Сведения о предложении в онлайн-магазине
| Параметр в онлайн-магазине | Рекомендации |
|:--- |:--- |  
| Название предложения | Введите эффективное название.<br> <ul> <li> Название вашего приложения должно сообщать основные преимущества, которые получат клиенты. </li> <li> Включите ключевые слова поиска в заголовок приложения, чтобы получить более высокий рейтинг в результатах поиска. </li> </ul> Для описаний консультационных служб нужно соблюдать специальный формат заголовка:<br>[Имя предложения]: [длительность] [тип предложения] (например, *Contoso: 2-недельная реализация*) <br><br> Следуйте рекомендациям по торговой марке.<br>Когда вы ссылаетесь на товарные знаки и названия программного обеспечения, продуктов и услуг корпорации Майкрософт, следуйте [рекомендациям по использованию товарных знаков и фирменной символики Майкрософт](https://www.microsoft.com/en-us/legal/intellectualproperty/trademarks/usage/general.aspx), а также соответствующим рекомендациям по конкретным продуктам. |
| Описание предложения | Создайте четкое описание предложения.<br> <ul> <li> Четко укажите преимущества, предоставляемые вашим предложением, в начале описания. </li> <li> Имейте в виду, что первые несколько предложений могут отображаться в результатах поисковой системы. </li> <li> Не полагайтесь на то, что функции и возможности помогут продавать ваш продукт. Вместо этого сосредоточьтесь на его преимуществах. </li> <li> Используйте как можно больше отраслевого лексикона или формулировок на основе преимуществ.</li> <li> Кроме того, в описании консультационной службы должна явно указываться профессиональная услуга, предоставляемая вами. </li></ul> Основные компоненты предлагаемых вами преимуществ должны включать следующие сведения:<ul><li>Описание продукта.</li><li>Тип пользователя, которому будет полезен продукт.</li><li>Потребность клиента, которую удовлетворяет продукт, или трудность, которую он устраняет.</li></ul> |

## <a name="storefront-listing-details"></a>Сведения об описании в онлайн-магазине
| Параметр в онлайн-магазине | Рекомендации |
|:--- |:--- |  
| Отрасли: только приложения и консультационные службы AppSource | Выберите отрасли, если ваше предложение соответствует отраслевым требованиям: <ul><li> Укажите отраслевые возможности в описании предложения.</li> </ul> **Примечание**. Максимальное число доступных параметров зависит от типа предложения и указывается на [Портале Cloud Partner](https://cloudpartner.azure.com/#insights). |
| Categories | Выберите категории, которые наилучшим образом соответствуют вашему предложению.<br> <ul> <li> Выбирайте эти категории тщательно, так как клиенты используют их для поиска соответствующих приложений. </li> <li> Не забудьте указать в описании предложения, как оно обеспечивает специфические для конкретной категории потребности. </li> </ul> **Примечание**. Максимальное число доступных параметров зависит от типа предложения и указывается на [Портале Cloud Partner](https://cloudpartner.azure.com/#insights). |
| Продукты, с которыми работает ваше приложение (до 3): только приложения AppSource | Перечислите дополнительные продукты и технологии, которые ваше решение использует или расширяет. Например, Центр Интернета вещей Azure или Машинное обучение Azure. |
| Ключевые слова для поиска (до 3) | Ключевые слова для поиска могут помочь бизнес-пользователям найти ваше приложение при поиске: <ul> <li> в Azure Marketplace; </li> <li> в AppSource; </li> <li> с помощью поисковых систем. </li> </ul> Чтобы ваше объявление появилось в этих поисковых запросах, вам необходимо: <ul> <li> определить три ключевых слова для поиска для вашего предложения; </li> <li> включить ключевые слова для поиска в сводку предложения и описание; </li> <li> перечислить здесь ключевые слова поиска. </li> </ul> |

## <a name="storefront-marketing-details"></a>Сведения о маркетинге в онлайн-магазине
| Параметр в онлайн-магазине | Рекомендации |
|:--- |:--- |  
| Логотип предложения (формат PNG, 48&nbsp;&times;&nbsp;48): страница поиска | Разработайте и оптимизируйте логотип для цифрового носителя.<br>Отправьте логотип в формате PNG на страницу поиска предложения. |
| Логотип предложения (формат PNG, 216&nbsp;&times;&nbsp;216): страница сведений о приложении | Разработайте и оптимизируйте логотип для цифрового носителя.<br>Отправьте логотип в формате PNG на страницу сведений о предложении. |
| Документы с дополнительными сведениями | Добавьте вспомогательные ресурсы по продажам и маркетингу в разделе "Подробнее", например: <ul> <li> технические документы; </li> <li> брошюры; </li> <li> контрольные списки; </li> <li> презентации PowerPoint.</li> </ul>Сохраните все файлы в формате PDF. Цель этих документов — информировать клиентов, а не продавать им что-либо. <br><br>Добавьте ссылку на целевую страницу вашего приложения во все свои документы и добавьте параметры URL-адреса, чтобы отслеживать посетителей и использование пробных версий. |
| Видеоролики: только приложения AppSource, консультационные службы и предложения SaaS | Самые убедительные видео передают ценность вашего предложения в описательной форме.<ul> <li> Удостоверьтесь, что в нем акцент делается на клиенте, а не на вашей компании. </li> <li> Видео должно продемонстрировать, как ваше решение позволяет справиться с основными сложностями и задачами целевого клиента. </li> <li> Рекомендуемая длительность: 60–90 секунд.</li> <li> Включите ключевые слова для поиска в названия видео. </li> <li> Добавьте дополнительные видео, например видеоинструкции, обучающее видео по началу работы или видео с отзывами клиентов. </li> </ul> |
| Снимки экрана (1280&nbsp;&times;&nbsp;720) | Добавьте до пяти снимков экрана.<br>Добавьте ключевые слова для поиска в названия файлов. |

## <a name="link-to-your-offer-page-from-your-website"></a>Ссылка на страницу предложения с веб-сайта

Связывая эмблему AppSource или Azure Marketplace на своем сайте с описанием в marketplace, включите следующие параметры запроса в конец URL-адреса, чтобы обеспечить возможность тщательного анализа и создания отчетов.
* **src** — включите источник, из которого трафик перенаправляется в AppSource (например, веб-сайт, LinkedIn или Facebook).
* **mktcmpid**: идентификатор вашей маркетинговой кампании, который может включать до 16 символов и содержать любое сочетание букв, цифр, знаков подчеркивания и дефисов (например, *blogpost_12*).

Следующий пример URL-адреса содержит оба предыдущих параметра запроса: `https://appsource.microsoft.com/product/dynamics-365/mscrm.04931187-431c-415d-8777-f7f482ba8095?src=website&mktcmpid=blogpost_12`

Добавив эти параметры в ваш URL-адрес в AppSource, вы можете проанализировать эффективность кампании на панели мониторинга аналитики на [Портале Cloud Partner](https://cloudpartner.azure.com/#insights).

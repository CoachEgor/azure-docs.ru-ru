# <a name="publishing-integrated-solutions"></a>Интегрированные решения для публикации

Интегрированное решение сочетает в себе несколько технологий решения "точка поставщика", а также развертывание и службы системного интегратора (SI) в единое Объединенное предложение, представленное в Marketplace в качестве [предложения консультационных услуг](https://docs.microsoft.com/azure/marketplace/consulting-services).

Эти решения разработаны с учетом конкретных нужд или сценариев клиента, которые начинаются с понимания отраслевых задач или проблемных точек. После этого две или более лучшие технологии интегрируются с помощью SI, чтобы создать единый и повторяемый процесс развертывания для клиентов, чтобы они быстро запускались и выполнялись. Системный интегратор также предоставляет единую точку контакта для клиента, предоставляя соответствующие настройки и консультации, а также поддерживает решение. Результатом является интегрированное решение, включая технологии и службы.

Общие сведения о [программе интегрированных решений](https://aka.ms/AA5qos4)см. в этом видео.

## <a name="key-benefits-to-integrated-solutions-in-the-marketplace"></a>Основные преимущества интегрированных решений в Marketplace

* Клиенты могут найти Упакованные решения, которые решают комплексные бизнес-задачи с помощью Marketplace, а также ускорить работу с помощью технологии, интегрированной в переднюю среду. Клиенты избегают необходимости отдельно привлекать несколько приложений ISV, развернутых в фрагментированном виде.
* У независимых поставщиков программного обеспечения есть IP-адрес, а также другие решения и службы, которые улучшают ценность их технологий, предоставляемых экспертным хранилищем единственных версий.
* Хранилище единственных копий создает предварительно определенные предварительно интегрированные решения с лучшими технологиями масштабирования, земли и расширения дополнительных учетных записей клиентов.
* Корпорация Майкрософт может использовать эти предварительно интегрированные отраслевые решения для более легкой совместной продажи с клиентами, с ценами и соглашениями, отсортированными заранее. 

## <a name="eligibility-criteria"></a>Критерии приемлемости

* Все участвующие партнеры должны иметь идентификатор MPN
* Участвующие в сопродаже удостоверения должны быть готовыми к совместной продаже
* Участвующие независимые поставщики программного обеспечения должны быть сопродажными IP-адресами 
* По крайней мере один участвующий партнер должен иметь компетенцию Gold в одной из следующих областей:

    * Разработка приложений
    * Интеграция приложений
    * Управление жизненным циклом приложений
    * Облачная платформа
    * Анализ данных
    * Центр обработки данных
    * Платформа данных
    * DevOps

* Решение должно включать по крайней мере две партнерские организации, объединяя IP-адреса и службы для полноценного решения и взаимодействия с клиентами.
* IP-адрес партнера должен быть создан на платформе Azure, Dynamics 365 и/или Power Platform
* IP-адрес интегрированного решения должен отличаться от простого IP-адреса служб независимых поставщиков программного обеспечения (VBA)
* Цель всех партнеров — создать повторяемое предложение, которое может быть перечислено в Azure Marketplace и (или) AppSource в конечном итоге.

## <a name="nomination-process"></a>Процесс предварительного утверждения

Партнеры, желающие участвовать в программе, должны найти других партнеров, с которыми они хотят работать, и заполнить форму [встроенных решений предварительного утверждения](https://aka.ms/AA5qicu) . Затем группа разработчиков интегрированных решений просматривает предварительного утверждения и поддерживайтесь в течение 10 рабочих дней с помощью утверждения и дальнейших действий. 

## <a name="developing-your-integrated-solution"></a>Разработка интегрированного решения 

После утверждения предварительного утверждения выполните следующие действия, используя ресурсы связанной программы, чтобы разработать интегрированное решение: 

1. Заполните основные деловые и технические обсуждения в формате семинара, чтобы определить решение (область, целевые клиенты, выгодное предложение, цены, лицензирование, условия, подход к поддержке, маркетинговые действия, процесс продаж и действия). подход к интеграции (точки интеграции, архитектура решения, проверка подлинности, Пользовательский интерфейс, мониторинг, механизм развертывания, API-интерфейсы и унифицированный репозиторий данных). 

    Следующие ресурсы помогут партнерам по этому процессу:

    1. [Пошаговое описание обсуждения деловых решений](https://aka.ms/AA5qicx)
    1. [Техническое описание семинара по решению технических решений](https://aka.ms/AA5qid1)
    1. [Видео быстрого запуска — деловые и технические семинары](https://aka.ms/AA5qos9)

1. Определите кросс-Партнерский Диспетчер проектов для управления планом проекта и временной шкалой, полученной из бизнес-и технических семинаров.

1. Полная техническая интеграция решения.

1. Окончательно выведите цены на решение и выберите единую ценовую точку для отображения в Azure Marketplace или AppSource.

1. Полный маркетинговый материал по Azure Marketplace или AppSource, в том числе:

    1. Имя объединенного решения
    2. Список описания интегрированного решения (рекомендации по предоставлению[списка предложений](https://docs.microsoft.com/azure/marketplace/gtm-offer-listing-best-practices))
    1. Пейджер клиента 1 ([шаблон](https://aka.ms/AA5s08a))
    1. Пользовательская презентация ([шаблон](https://aka.ms/AA5s7ql))
    1. Логотип для Объединенного решения 

## <a name="publishing-your-integrated-solution"></a>Публикация интегрированного решения 

После завершения технической интеграции и готовности маркетинговых материалов ознакомьтесь с руководством по издателю [консультационных услуг для Azure Marketplace и AppSource](https://docs.microsoft.com/azure/marketplace/consulting-services). С помощью этого ресурса определите, следует ли публиковать в Azure Marketplace или AppSource в зависимости от аудитории, к которой вы хотите получить доступ, подготовить артефакты публикации и завершить процесс публикации.

Хотя существует пять типов служб, доступных для предложений консультационных услуг, интегрированное решение должно быть либо доказательством концепции, либо полной реализацией.

## <a name="going-to-market-with-your-integrated-solution"></a>Переход на рынке с помощью интегрированного решения 

Когда ваше решение будет активно в Azure Marketplace или AppSource, вы захотите обеспечить осведомленность и создание спроса с помощью маркетинговых действий. Ознакомьтесь с действиями, перечисленными в [плане проекта и временной шкалой](https://aka.ms/AA5qiuc), включая действия, которые партнеры могут делать в своих группах и друг друга. Затем вы получите доступ к [преимуществам перехода на рынок](https://docs.microsoft.com/azure/marketplace/gtm-your-marketplace-benefits#how-to-access-microsoft-resources) , предоставленным партнерам, опубликованным в Marketplace. Все предложения интегрированных решений могут использовать преимущества пробного уровня.

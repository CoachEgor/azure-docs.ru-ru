# <a name="publish-an-integrated-solution"></a>Публикация интегрированного решения

В одном комплексном предложении интегрированное решение сочетает в себе технологии точечного решения от поставщиков программного обеспечения с развертыванием и поддержкой партнера по услугам. Партнер службы перечисляет предложение в Microsoft AppSource или Azure Marketplace в качестве [консалтинговой услуги.](https://docs.microsoft.com/azure/marketplace/consulting-services)

Интегрированные решения направлены на удовлетворение конкретных потребностей или сценария клиента. Поставщики начинают с понимания проблемы отрасли или болевые точки. Затем они интегрируют две или более лучших в породе технологии для создания единого и повторяемого опыта развертывания для клиентов. Поставщики получают клиентов и работает быстро. Они предлагают клиенту единую точку контакта, обеспечивающую соответствующую настройку, консультацию и поддержку решений. Результатом является комплексное решение, включая технологии и услуги.

Для обзора программы комплексных решений, смотреть [это видео](https://aka.ms/AA5qos4).

## <a name="benefits-of-integrated-solutions-in-the-marketplace"></a>Преимущества интегрированных решений на рынке

* Клиенты могут найти упакованные решения, которые решают сквозные бизнес-задачи через рынок. Они получают ускоренное время для оценки, потому что технология интегрирована заранее. Клиенты избегают необходимости взаимодействовать отдельно с несколькими приложениями, развернутыми фрагментированным способом.
* IP упаковывается вместе с другими решениями и услугами, которые повышают ценность технологии, поставляемой партнерами-экспертами.
* Партнеры по услугам создают предопределенные, заранее интегрированные решения с передовой технологией для масштабирования, земли и расширения в большем количестве учетных записей клиентов.
* Корпорация Майкрософт может использовать эти прединтегрированные, отраслевые решения для более легкой совместной продажи клиентам. Цены и условия сортируются заранее.

## <a name="eligibility-criteria"></a>Условия для допуска

* Партнеры-участники должны иметь MPN ID.
* Участвующие поставщики услуг должны быть готовы к совместной продаже.
* Участвующие поставщики программного обеспечения должны быть IP совместно продавать готовы.
* По крайней мере один участник должен иметь Золотую компетенцию в одной из следующих областей:

    * Разработка приложений
    * Интеграция приложений
    * Управление жизненным циклом приложения
    * Облачная платформа
    * Аналитика данных
    * Центр обработки данных
    * Платформа данных
    * DevOps

* Решение должно включать по крайней мере две организации-партнеры, объединяющие ИС и услуги для полного клиентского решения и опыта.
* ИС-партнера должен быть построен на Azure, Dynamics 365 или Power Platform.
* ИС интегрированного решения должен быть дифференцирован просто с ИС поставщика программного обеспечения и базовых предложений поставщиков услуг.
* Все партнеры разделяют цель создания повторяемого предложения, которое может быть перечислено и в конечном итоге осуществляться через Microsoft AppSource, Azure Marketplace или и то, и другое.

## <a name="submit-a-nomination"></a>Подайте заявку.

Партнеры, желающие принять участие в программе, должны определить других партнеров, с которыми они хотят работать. Они должны заполнить [форму номинации Интегрированные решения](https://aka.ms/AA5qicu). Команда программы «Интегрированные решения» рассматривает номинацию и следит за происходящим в течение 10 рабочих дней. Команда предоставляет одобрение и дальнейшие шаги.

## <a name="develop-an-integrated-solution"></a>Разработка интегрированного решения

После утверждения кандидатуры используйте связанные ресурсы программы для разработки интегрированного решения. Выполните следующие действия.

1. Завершите ключевые деловые и технические обсуждения в формате семинара для определения решения и интеграционного подхода.

    Ваше обсуждение решения должно включать в себя следующие моменты:
    * Область
    * Целевые клиенты
    * Предлагаемые преимущества
    * Цены
    * Лицензирование
    * Условия
    * Подход к поддержке
    * Маркетинговая деятельность
    * Процесс продажи
    * Действия

    Ваше обсуждение интеграционного подхода должно включать следующие моменты:
    * Точки интеграции
    * Архитектура решения
    * Проверка подлинности
    * Пользовательский интерфейс
    * Наблюдение
    * Механизм развертывания
    * Программные интерфейсы
    * Единое репозиторий данных

    Используйте эти ресурсы для процесса разработки:

    * [Руководство по обсуждению бизнес-решений](https://aka.ms/AA5qicx)
    * [Руководство по обсуждению технических решений](https://aka.ms/AA5qid1)
    * [Быстрый запуск видео: Бизнес и технические семинары](https://aka.ms/AA5qos9)

1. Определите менеджера проекта, который будет управлять планом проекта и графиком, разработанным на бизнес- и технических семинарах.

1. Разработайте полную техническую интеграцию решения.

1. Определите ценообразование на решение и единую ценовую точку на Microsoft AppSource или Azure Marketplace.

1. Заполните маркетинговое обеспечение для листинга Microsoft AppSource или Azure Marketplace, включая:

    * Комбинированное имя решения.
    * Описание списка интегрированного решения. Следуйте [предлагаемым рекомендациям.](https://docs.microsoft.com/azure/marketplace/gtm-offer-listing-best-practices)
    * Одностраничный шаг для клиентов. Используйте [шаблон одного пейджера.](https://aka.ms/AA5s08a)
    * Презентация клиента. Используйте [шаблон презентации](https://aka.ms/AA5s7ql).
    * Логотип для комбинированного решения.

## <a name="publish-your-integrated-solution"></a>Опубликовать интегрированное решение

После завершения технической интеграции и маркетингового обеспечения обратитесь в руководство по [консультированию microsoft AppSource и Azure Marketplace](https://docs.microsoft.com/azure/marketplace/consulting-services)в руководстве по издателям. Используйте этот ресурс, чтобы определить, будет ли ваше решение опубликовано в Microsoft AppSource или Azure Marketplace. Также используйте руководство для подготовки публикации артефактов и завершения процесса публикации.

Хотя для предложений консалтинговых услуг доступны пять типов услуг, интегрированное решение должно быть либо доказательством концепции, либо полной реализацией.

## <a name="go-to-market-with-your-integrated-solution"></a>Выход на рынок с интегрированным решением

Когда ваше решение находится в Microsoft AppSource или Azure Marketplace, вы хотите повысить осведомленность и спрос с помощью маркетинговых мероприятий. Просмотрите действия, перечисленные в [плане проекта и графике.](https://aka.ms/AA5qiuc) Этот ресурс включает в себя действия, которые партнеры могут управлять в своих командах и друг с другом. Затем получите доступ [к преимуществам выхода](https://docs.microsoft.com/azure/marketplace/gtm-your-marketplace-benefits#list-trial-and-consulting-benefits) на рынок для партнеров, которые публикуют на рынке. Все предложения комплексного решения имеют право на получение преимуществ на пробном уровне.

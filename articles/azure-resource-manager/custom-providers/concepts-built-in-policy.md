# <a name="deploy-associations-for-a-custom-provider-using-azure-policy"></a>Развертывание ассоциаций для пользовательского поставщика с использованием политики Azure

Политики Azure могут использоваться для развертывания ассоциаций для ассоциировать ресурсы с пользовательским поставщиком. В этой статье мы описываем встроенную политику, которая развертывает ассоциации и как вы можете использовать эту политику.

## <a name="built-in-policy-to-deploy-associations"></a>Встроенная политика для развертывания ассоциаций

Развертывание ассоциаций для пользовательского поставщика — это встроенная политика, которая может использоваться для развертывания ассоциации для ассоциации ресурса с пользовательским поставщиком. Политика принимает три параметра:

- Идентификатор пользовательского поставщика - этот идентификатор является идентификатором ресурсов пользовательского поставщика, к которому необходимо ассоциировать ресурсы.
- Типы ресурсов для ассоциировать - Эти типы ресурсов представляют собой список типов ресурсов, которые будут связаны с пользовательским поставщиком. Можно связать несколько типов ресурсов с пользовательским поставщиком, используя одну и ту же политику.
- Приставка названия ассоциации - Эта строка является приставкой, которая будет добавлена к названию создаваемого ресурса ассоциации. Значение по умолчанию — «РазвертываниеПолитики».

Политика использует оценку DeployIfNotExistsExists. Он работает после того, как поставщик ресурсов обрабатывает запрос на создание или обновление ресурса, а оценка возвращает код состояния успеха. После этого ресурс ассоциации развертывается с помощью развертывания шаблонов.
Для получения дополнительной информации об ассоциациях [см.](./concepts-resource-onboarding.md)

## <a name="how-to-use-the-deploy-associations-built-in-policy"></a>Как использовать встроенную политику развертываемых ассоциаций 

### <a name="prerequisites"></a>Предварительные требования
Если пользовательскому поставщику требуется разрешение на область действия, развертывание ресурсов ассоциации не будет работать без предоставления разрешений.

### <a name="policy-assignment"></a>Назначение политики
Чтобы использовать встроенную политику, создайте назначение политики и назначайте объединения развертываний для политики пользовательского поставщика. Затем политика будет выявлять несоответствующие ресурсы и развертывать ассоциации для этих ресурсов.

# <a name="troubleshoot-moving-azure-resources-to-new-resource-group-or-subscription"></a>Устранение неполадок с перемещением ресурсов в новую группу ресурсов или подписку

В этой статье содержатся предложения, помогающих решить проблемы при перемещении ресурсов.

## <a name="upgrade-a-subscription"></a>Обновление подписки

Если вы действительно хотите обновить подписку Azure (например, переключиться с бесплатного уровня до уровня с оплатой по мере использования), необходимо преобразовать вашу подписку.

* Сведения о том, как перейти с бесплатной пробной версии или подписки Azure Microsoft Imagine на подписку с оплатой по мере использования, см. [здесь](../../billing/billing-upgrade-azure-subscription.md).
* Сведения о том, как перейти с подписки с оплатой по мере использования на другое предложение, см. [здесь](../../billing/billing-how-to-switch-azure-offer.md).

Если вам не удается перейти на другую подписку, [создайте запрос на поддержку Azure](../../azure-portal/supportability/how-to-create-azure-support-request.md). Выберите тип проблемы **Управление подпиской**.

## <a name="service-limitations"></a>Ограничения на обслуживание

Некоторые службы требуют дополнительных соображений при перемещении ресурсов. Если вы перемещаете следующие службы, убедитесь, что вы проверяете руководство и ограничения.

* [Службы приложений](./move-limitations/app-service-move-limitations.md)
* [Услуги Azure DevOps](/azure/devops/organizations/billing/change-azure-subscription?toc=/azure/azure-resource-manager/toc.json)
* [Классическая модель развертывания](./move-limitations/classic-model-move-limitations.md)
* [Сеть](./move-limitations/networking-move-limitations.md)
* [Службы восстановления](../../backup/backup-azure-move-recovery-services-vault.md?toc=/azure/azure-resource-manager/toc.json)
* [Виртуальные машины](./move-limitations/virtual-machines-move-limitations.md)

## <a name="large-requests"></a>Большие запросы

Если это возможно, разделите большие перемещения на несколько операций перемещения. При попытке перемещения более чем 800 ресурсов за одну операцию Resource Manager немедленно возвращает ошибку. Однако перемещение менее чем 800 ресурсов может также завершиться со сбоем из-за превышения времени ожидания.

## <a name="resource-not-in-succeeded-state"></a>Ресурс не в состоянии успеха

Когда вы получаете сообщение об ошибке, указывающее на то, что ресурс не может быть перемещен, поскольку он не находится в состоянии успешного, он может быть зависимым ресурсом, блокирующим перемещение. Как правило, код ошибки **MoveCannotProceedWithРесурсЫНеНеудалосьгосударство**.

Если группа ресурсов источника или целевого ресурса содержит виртуальную сеть, во время перемещения проверяются состояния всех зависимых ресурсов виртуальной сети. Проверка включает в себя эти ресурсы, прямо или косвенно зависящие от виртуальной сети. Если какой-либо из этих ресурсов находится в неудавом состоянии, ход блокируется. Например, если виртуальная машина, используюая виртуальную сеть, сработала, ход блокируется. Перемещение блокируется даже тогда, когда виртуальная машина не является одним из перемещаемых ресурсов и не находится в одной из групп ресурсов для перемещения.

Когда вы получаете эту ошибку, у вас есть два варианта. Либо перемещайте ресурсы в группу ресурсов, у которых нет виртуальной сети, либо [свяжитесь с службой поддержки.](../../azure-portal/supportability/how-to-create-azure-support-request.md)

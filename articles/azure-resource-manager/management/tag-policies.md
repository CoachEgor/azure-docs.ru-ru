# <a name="assign-policies-for-tag-compliance"></a>Назначать политики для соответствия тегов

Политика [Azure](../../governance/policy/overview.md) используется для обеспечения соблюдения правил и конвенций по метки. Создавая политику, вы избегаете сценария развертывания ресурсов в подписке, которые не имеют ожидаемых тегов для организации. Вместо того, чтобы вручную применять теги или искать ресурсы, которые не соответствуют требованиям, вы создаете политику, которая автоматически применяет необходимые теги во время развертывания. Теги также теперь могут быть применены к существующим ресурсам с новым эффектом [модификации](../../governance/policy/concepts/effects.md#modify) и [задачей исправления.](../../governance/policy/how-to/remediate-resources.md) В следующем разделе приведены примеры политик для тегов.

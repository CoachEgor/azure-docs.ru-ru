# <a name="deploy-associations-for-a-managed-application-using-azure-policy"></a>Развертывание ассоциаций для управляемого приложения с использованием политики Azure

Политики Azure могут использоваться для развертывания объединений для объединения ресурсов с управляемым приложением. В этой статье мы описываем встроенную политику, которая развертывает ассоциации и как вы можете использовать эту политику.

## <a name="built-in-policy-to-deploy-associations"></a>Встроенная политика для развертывания ассоциаций

Развертывание ассоциаций для управляемого приложения — это встроенная политика, которая может использоваться для развертывания ассоциации для объединения ресурса с управляемым приложением. Политика принимает три параметра:

- Управляемый идентификатор приложения - этот идентификатор является идентификатором ресурсов управляемого приложения, к которому необходимо ассоциировать ресурсы.
- Типы ресурсов для ассоциировать - Эти типы ресурсов представляют собой список типов ресурсов, которые будут связаны с управляемым приложением. Можно связать несколько типов ресурсов с управляемым приложением с помощью одной и той же политики.
- Приставка названия ассоциации - Эта строка является приставкой, которая будет добавлена к названию создаваемого ресурса ассоциации. Значение по умолчанию — «РазвертываниеПолитики».

Политика использует оценку DeployIfNotExistsExists. Он выполняется после того, как Поставщик ресурсов обрабатывает запрос на создание или обновление ресурса выбранного типа ресурса (ы) и оценка возвращает код состояния успеха. После этого ресурс ассоциации развертывается с помощью развертывания шаблонов.
Для получения дополнительной информации об ассоциациях [см.](../custom-providers/concepts-resource-onboarding.md)

## <a name="how-to-use-the-deploy-associations-built-in-policy"></a>Как использовать встроенную политику развертываемых ассоциаций 

### <a name="prerequisites"></a>Предварительные требования
Если управляемому приложению требуется разрешение на выполнение действия подписки, развертывание ресурсов ассоциации не будет работать без предоставления разрешений.

### <a name="policy-assignment"></a>Назначение политики
Чтобы использовать встроенную политику, создайте назначение политики и назначайте объединения развертываний для политики управляемых приложений. После успешного присвоения политики политика определит несовместимые ресурсы и развернет ассоциацию для этих ресурсов.

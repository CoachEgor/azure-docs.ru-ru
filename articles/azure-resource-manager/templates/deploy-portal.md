# <a name="deploy-resources-with-arm-templates-and-azure-portal"></a>Развертывание ресурсов с помощью шаблонов ARM и портала Azure

Узнайте, как использовать [портал Azure](https://portal.azure.com) с [шаблонами Azure Resource Manager (ARM)](overview.md) для развертывания ресурсов Azure. Чтобы узнать об управлении ресурсами, [см. Управление ресурсами Azure с помощью портала Azure.](../management/manage-resources-portal.md)

Развертывание ресурсов Azure с помощью портала Azure обычно включает в себя два этапа:

- Создайте группу ресурсов.
- Развертывание ресурсов в группе ресурсов.

Кроме того, можно также развернуть шаблон ARM для создания ресурсов Azure.

В этой статье описаны оба метода.

## <a name="create-a-resource-group"></a>Создание группы ресурсов

1. Чтобы создать новую группу ресурсов, выберите **группы ресурсов** на [портале Azure.](https://portal.azure.com)

   ![Выбор групп ресурсов](./media/deploy-portal/select-resource-groups.png)

1. Под группами ресурсов выберите **Добавить**.

   ![Добавление группы ресурсов](./media/deploy-portal/add-resource-group.png)

1. Выберите или введите следующие значения свойств:

    - **Подписка**— выберите подписку Azure.
    - **Группа ресурсов**: Назовите группе ресурсов имя.
    - **Область**: Укажите местоположение Azure. Здесь группа ресурсов хранит метаданные о ресурсах. Для соответствия требованиям вам, возможно, нужно указать, где хранятся эти метаданные. Обычно мы рекомендуем указывать расположение, в котором будет храниться большинство ресурсов. Используя одно и то же расположение, можно упростить шаблон.

   ![Настройка значений группы](./media/deploy-portal/set-group-properties.png)

1. Выберите **Review + create** (Просмотреть и создать).
1. просмотреть значения, а затем выберите **Создать**.
1. Выберите **Обновление,** прежде чем вы сможете увидеть новую группу ресурсов в списке.

## <a name="deploy-resources-to-a-resource-group"></a>Развертывание ресурсов в группе ресурсов

После создания группы ресурсов можно развернуть ресурсы в группу из Marketplace. Marketplace предоставляет предварительно определенные решения для распространенных сценариев использования.

1. Чтобы начать развертывание, выберите **Создать ресурс** на [портале Azure.](https://portal.azure.com)

   ![Новый ресурс](./media/deploy-portal/new-resources.png)

1. Найдите тип ресурса, который нужно развернуть. Ресурсы организованы по категориям. Если конкретное решение, которое вы хотите развернуть, отсутствует, его можно поискать в Marketplace. На следующем скриншоте показано, что ubuntu Server выбран.

   ![Выбор типа ресурсов](./media/deploy-portal/select-resource-type.png)

1. В зависимости от выбранного типа ресурса вам потребуется задать набор соответствующих свойств, прежде чем начинать развертывание. Для всех типов необходимо выбрать целевую группу ресурсов. На следующем изображении показано, как создать виртуальную машину Linux и развернуть ее в созданной группой ресурсов.

   ![Создать группу ресурсов](./media/deploy-portal/select-existing-group.png)

   Кроме того, можно создать новую группу ресурсов непосредственно при развертывании ресурсов. Щелкните **Создать** и укажите имя группы ресурсов.

1. Начнется развертывание. Развертывание может занять несколько минут. Некоторые ресурсы занимают больше времени, чем другие ресурсы. По окончании развертывания появится уведомление. Выберите **Перейти к ресурсу,** чтобы открыть

   ![Просмотр уведомления](./media/deploy-portal/view-notification.png)

1. После развертывания ресурсов можно добавить дополнительные ресурсы в группу ресурсов, выбрав **Добавить**.

   ![Добавление ресурса](./media/deploy-portal/add-resource.png)

## <a name="deploy-resources-from-custom-template"></a>Развертывание ресурсов с помощью настраиваемого шаблона

Если вы хотите выполнить развертывание без использования шаблонов из Marketplace, создайте настраиваемый шаблон, определяющий инфраструктуру для вашего решения. Чтобы узнать о создании шаблонов, [см.](template-syntax.md)

> [!NOTE]
> Интерфейс портала не поддерживает использование ссылок на [секрет из Key Vault](key-vault-parameter.md). Вместо этого используйте [PowerShell](deploy-powershell.md) или [Azure CLI](deploy-cli.md), чтобы развернуть шаблон локально или на основе внешнего URI.

1. Чтобы развернуть настраиваемый шаблон через портал, выберите **Создать ресурс,** поиск **шаблона.** а затем выберите **развертывание шаблонов.**

   ![Поиск развертывания шаблона](./media/deploy-portal/search-template.png)

1. Выберите **Создать**.
1. Вы видите несколько вариантов создания шаблона:

    - **Создайте свой собственный шаблон в редакторе:** создайте шаблон с помощью редактора шаблона портала.  Редактор способен добавить схему шаблона ресурсов.
    - **Общие шаблоны**: Есть четыре общих шаблона для создания виртуальной машины Linux, виртуальной машины Windows, веб-приложения и базы данных Azure S'L.
    - **Загрузите шаблон быстрого запуска GitHub:** используйте существующие [шаблоны быстрого запуска.](https://azure.microsoft.com/resources/templates/)

   ![Параметры представления](./media/deploy-portal/see-options.png)

    Этот учебник предоставляет инструкцию для загрузки шаблона быстрого запуска.

1. Под **загрузить шаблон quickstart GitHub,** введите или выберите **101-хранилище-аккаунт-создать.**

    Имеются две возможности.

    - **Выберите шаблон:** развернуть шаблон.
    - **Отобразите шаблон:** отодвить шаблон quickstart перед его развертыванием.

1. Выберите **шаблон Edit** для изучения редактора шаблонов портала. Шаблон загружается в редактор. Обратите внимание, что есть два параметра: **storageAccountType** и **местоположение**.

   ![Создание шаблона](./media/deploy-portal/show-json.png)

1. Внести незначительные изменения в шаблон. Например, обновите переменную **storageAccountName** до:

    ```json
    "storageAccountName": "[concat('azstore', uniquestring(resourceGroup().id))]"
    ```

1. Нажмите кнопку **Сохранить**. Теперь вы видите интерфейс развертывания шаблона портала. Обратите внимание на два параметра, которые вы определили в шаблоне.
1. Введите или выберите значения свойств:

    - **Подписка**— выберите подписку Azure.
    - **Группа ресурсов**: Выберите **Создать новое** и дать имя.
    - **Местоположение**: Выберите местоположение Azure.
    - **Тип учетной записи хранилища**: Используйте значение по умолчанию.
    - **Расположение**: Используйте значение по умолчанию.
    - Выберите **Я принимаю указанные выше условия**.

1. Нажмите кнопку **Купить**.

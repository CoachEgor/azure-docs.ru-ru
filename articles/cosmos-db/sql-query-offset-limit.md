# <a name="offset-limit-clause-in-azure-cosmos-db"></a>Пункт OFFSET LIMIT в Azure Cosmos DB

Оговорка OFFSET LIMIT является факультативным положением, которое можно пропустить, а затем взять некоторое количество значений из запроса. Подсчет OFFSET и количество LIMIT необходимы в оговорке OFFSET LIMIT.

Когда OFFSET LIMIT используется в сочетании с оговоркой ORDER BY, набор результатов производится путем пропуска и выполнения заказанных значений. Если не используется оговорка ORDER BY, это приведет к детерминированному порядку значений.

## <a name="syntax"></a>Синтаксис
  
```sql  
OFFSET <offset_amount> LIMIT <limit_amount>
```  
  
## <a name="arguments"></a>Аргументы

- `<offset_amount>`

   Устраняет целый ряд элементов, которые должны пропустить результаты запроса.

- `<limit_amount>`
  
   Определяет целый ряд элементов, которые должны включать результаты запроса

## <a name="remarks"></a>Remarks
  
  В `OFFSET` `OFFSET LIMIT` оговорке `LIMIT` требуется как подсчет, так и подсчет. При использовании факультативного `ORDER BY` положения набор результатов производится путем пропуска заказанных значений. В противном случае запрос вернет фиксированный порядок значений.

  Ru заряд запроса с `OFFSET LIMIT` будет увеличиваться по мере увеличения числа терминов, компенсируемых увеличивается. Для запросов, которые имеют несколько страниц результатов, мы обычно рекомендуем использовать токены продолжения. Токены продолжения — это «закладка» для места, где запрос может позже возобновиться. Если вы `OFFSET LIMIT`используете, нет "закладки". Если вы хотите вернуть следующую страницу запроса, вам придется начать с самого начала.
  
  Вы должны `OFFSET LIMIT` использовать для случаев, когда вы хотели бы пропустить документы полностью и сохранить ресурсы клиентов. Например, следует `OFFSET LIMIT` использовать, если вы хотите перейти к результату 1000-го запроса и не должны просматривать результаты с 1 по 999. На бэкэнде, `OFFSET LIMIT` по-прежнему загружает каждый документ, в том числе те, которые пропущены. Преимуществом производительности является экономия ресурсов клиентов, избегая обработки документов, которые не нужны.

## <a name="examples"></a>Примеры

Например, вот запрос, который пропускает первое значение и возвращает второе значение (в порядке названия города-резидента):

```sql
    SELECT f.id, f.address.city
    FROM Families f
    ORDER BY f.address.city
    OFFSET 1 LIMIT 1
```

Результаты:

```json
    [
      {
        "id": "AndersenFamily",
        "city": "Seattle"
      }
    ]
```

Вот запрос, который пропускает первое значение и возвращает второе значение (без заказа):

```sql
   SELECT f.id, f.address.city
    FROM Families f
    OFFSET 1 LIMIT 1
```

Результаты:

```json
    [
      {
        "id": "WakefieldFamily",
        "city": "Seattle"
      }
    ]
```

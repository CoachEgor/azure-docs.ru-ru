## <a name="-operator"></a>* - оператор

Специальный оператор - проектирует весь товар как есть. При его использовании должно быть единственное отображаемое поле. Запрос как `SELECT * FROM Families f` действителен, `SELECT VALUE * FROM Families f` но `SELECT *, f.id FROM Families f` не действителен.

## <a name="-and--operators"></a>? И?? операторы

Операторы Ternary (?) и Coalesce (??) можно использовать для создания условных выражений, как в языках программирования, таких как C и JavaScript.

Вы можете использовать ? оператора для строительства новых свойств JSON на лету. Например, следующий запрос классифицирует `elementary` `other`уровни классов в или:

```sql
     SELECT (c.grade < 5)? "elementary": "other" AS gradeLevel
     FROM Families.children[0] c
```

Вы также можете гнездо звонки в ? оператора, как и в следующем запросе: 

```sql
    SELECT (c.grade < 5)? "elementary": ((c.grade < 9)? "junior": "high") AS gradeLevel
    FROM Families.children[0] c
```

Как и в других операторах запросов, ? оператор исключает элементы, если ссылки свойства отсутствуют или типы, сравниваемые отличаются.

Использовать ?? оператора для эффективной проверки свойства в элементе при запросе на полуструктурированные или смешанные данные. Например, следующий запрос `lastName` возвращается `surname` при `lastName` наличии или нет.

```sql
    SELECT f.lastName ?? f.surname AS familyName
    FROM Families f
```

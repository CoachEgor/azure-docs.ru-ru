# <a name="optimize-cost-with-reserved-capacity-in-azure-cosmos-db"></a>Оптимизация затрат на резервную мощностью в Azure Cosmos DB

Зарезервированная емкость Azure Cosmos DB позволяет экономить деньги благодаря фиксированной оплате ресурсов Azure Cosmos DB на один или три года вперед. С помощью резервной мощности Azure Cosmos DB можно получить скидку на пропускную способность, подготовленную для ресурсов Cosmos DB. Примерами ресурсов являются базы данных и контейнеры (таблицы, коллекции и графы).

Однолетние и трехлетние контракты на резервную мощность Azure Cosmos DB с предварительной оплатой позволят сэкономить существенную сумму &mdash;до 65 % по сравнению с обычной платой. Резервная мощность предоставляет скидку на выставление счетов и не влияет на состояние среды выполнения ресурсов Azure Cosmos DB.

Резервная мощность Azure Cosmos DB покрывает расходы на пропускную способность, подготовленную для ресурсов. Но она не распространяется на расходы хранилища и сети. После того как вы купите резервирование, при начислении платы за пропускную способность, соответствующей атрибутам резервирования, не будут применяться ставки оплаты по мере использования. Дополнительные сведения о резервировании см. в статье [Общие сведения об Azure Reserved VM Instances](../cost-management-billing/reservations/save-compute-costs-reservations.md).

Вы можете приобрести резервную мощность Azure Cosmos DB на [портале Azure](https://portal.azure.com). Платите за резервирование [наперед или ежемесячными платежами](../cost-management-billing/reservations/monthly-payments-reservations.md). Требования для приобретения резервной мощности.

* Вы должны быть в роли владельца, по крайней мере, для одного предприятия или индивидуальной подписки с оплатой по мере использования ставок.  
* Для подписок с соглашением Enterprise параметр **Добавить зарезервированные экземпляры** следует включить на [портале EA](https://ea.azure.com). Или, если этот параметр отключен, необходимо быть администратором подписки EA.
* В рамках программы для поставщиков облачных решений (CSP) только агенты администрирования или агенты продаж могут приобрести резервную мощность Azure Cosmos DB.

## <a name="determine-the-required-throughput-before-purchase"></a>Определение требуемой пропускной способности перед приобретением

Размер зарезервированной покупки емкости должен основываться на общем объеме пропускной способности, который существующие или скоро развернутые ресурсы Azure Cosmos DB будут использоваться на почасовой основе. Например: Покупка зарезервированной емкости 30 000 RU/s, если это ваш последовательный шаблон почасового использования. В этом примере любая прокладка свыше 30 000 руб./с будет выставлена с помощью ставки "Оплата по мере использования". Если прослеживаемая пропускная способность ниже 30 000 руб./с в час, то дополнительная зарезервированная емкость за этот час будет потрачена впустую.

Мы вычисляем рекомендации по покупке на основе вашей почасовой модели использования. Анализируется использование за последние 7, 30 и 60 дней, и рекомендуется обеспечить покупку зарезервированной емкости, которая максимизирует ваши сбережения. Рекомендуемые размеры бронирования можно просмотреть на портале Azure, используя следующие действия:

1. Войдите на [портал Azure](https://portal.azure.com).  

2. Выберите **Все услуги** > **Reservations** > Бронирование**Добавить**.

3. Из панели **бронирования покупок** выберите **Azure Cosmos DB.**

4. Выберите **Рекомендуемую** вкладку для просмотра рекомендуемых оговорок:

Рекомендации можно отфильтровать по следующим атрибутам:

- **Срок** (1 год или 3 года)
- **Частота выставления счетов** (ежемесячно или авансом)
- **Тип пропускной прихожей** (RU против Multi-master RU's)

Кроме того, можно присвоить рекомендации, которые будут находиться в пределах одной группы ресурсов, одной подписки или всей регистрации Azure. 

Вот пример рекомендации:

![Рекомендации по зарезервированной емкости](./media/cosmos-db-reserved-capacity/reserved-capacity-recommendation.png)

Эта рекомендация о покупке бронирования за 30 000 руб./с указывает на то, что из трехлетнего бронирования размер бронирования в размере 30 000 руб./с позволит максимально сэкономить. В этом случае рекомендация рассчитывается на основе последних 30 дней использования Azure Cosmos DB. Если этот клиент ожидает, что последние 30 дней использования Azure Cosmos DB являются репрезентативными для использования в будущем, они позволят максимально сэкономить, купив резервирование за 30 000 руб./с.

## <a name="buy-azure-cosmos-db-reserved-capacity"></a>Приобретение резервной мощности Azure Cosmos DB

1. Войдите на [портал Azure](https://portal.azure.com).  

2. Выберите **Все услуги** > **Reservations** > Бронирование**Добавить**.  

3. Из панели **бронирования покупок** выберите **Azure Cosmos DB,** чтобы купить новое бронирование.  

4. Заполните обязательные поля, как описано в следующей таблице.

   ![Заполнение формы резервной мощности](./media/cosmos-db-reserved-capacity/fill-reserved-capacity-form.png)

   |Поле  |Описание  |
   |---------|---------|
   |Область   |   Параметр, который определяет, сколько подписок может использовать преимущество на выставление счетов, связанное с резервированием. Он также контролирует, как резервирование применяется к определенным подпискам. <br/><br/>  Если выбрать **Любая подписка**, скидка на резервирование применяется к экземплярам Azure Cosmos DB, работающим в любых подписках в текущем контексте выставления счетов. Контекст выставления счетов зависит от способа регистрации в Azure. Для клиентов с соглашением Enterprise общая область действует в рамках регистрации и включает в себя все подписки, заданные при регистрации. Для клиентов с оплатой по мере использования общая область — это все индивидуальные подписки с тарифами с оплатой по мере использования, созданными администратором учетной записи.  <br/><br/>  Если выбрать **Одна подписка**, скидка на резервирование применяется к экземплярам Azure Cosmos DB в выбранной подписке. <br/><br/> При выборе **единой группы ресурсов**скидка на бронирование применяется к экземплярам Azure Cosmos DB в выбранной подписке и выбранной группе ресурсов в рамках этой подписки. <br/><br/> После приобретения резервной мощности можно изменить область резервирования.  |
   |Подписка  |   Подписка, используемая для оплаты резервной мощности Azure Cosmos DB. Метод оплаты по выбранной подписке используется для взимания расходов. Подписка должна быть одним из следующих типов: <br/><br/>  Корпоративное соглашение (номера предложений: MS-A'A'-0017P или MS-A'A'A'-0148P): Для подписки на предприятие плата вычитается из остатка денежных обязательств регистрации или взимается как избанов. <br/><br/> Индивидуальная подписка с тарифами с оплатой по мере использования (предложения номеров: MS-A'R-0003P или MS-A'A'R-0023P): Для индивидуальной подписки с оплатой по мере использования ставки, сборы выставляются на кредитную карту или метод оплаты счета-фактуры по подписке.    |
   | Группа ресурсов | Группа ресурсов, к которой применяется скидка зарезервированной емкости. |
   |Термин  |   Один или три года.   |
   |Тип пропускной присяжек   |  Пропускная часть предоставляется в качестве единиц запроса. Вы можете купить резервирование для предусмотренной пропускной записи для обеих населеннок - пишет один регион, а также несколько записей региона. Тип пропускной перевалки имеет два значения на выбор: 100 RU/s в час и 100 Multi-master RU/s в час.|
   | Зарезервированные единицы емкости| Объем пропускной способности, который необходимо зарезервировать. Вы можете вычислить это значение путем определения пропускной способности, необходимой для всех ресурсов Cosmos DB (например, баз данных или контейнеров) в регионе. Затем умножить его на количество регионов, которые вы будете ассоциировать с вашей базой данных Космос. Например, если у вас есть пять регионов с 1 млн ЕЗ/с в каждом, выберите 5 миллионов ЕЗ/с, чтобы приобрести емкость резервирования. |


5. После заполнения формы рассчитывается цена, необходимая для покупки зарезервированной мощности. Выход также показывает процент скидки, который вы получаете с выбранными вариантами. Следующий клик **Выберите**

6. В панели **бронирования покупки** просмотрите скидку и цену бронирования. Цена за резервирование применяется к ресурсам Azure Cosmos DB с подготовленной пропускной способностью во всех регионах.  

   ![Резюме зарезервированных емкости](./media/cosmos-db-reserved-capacity/reserved-capacity-summary.png)

7. Выберите **Обзор и купить,** а затем **купить сейчас**. После успешной покупки откроется следующая страница.

После приобретения резервирования оно немедленно применится ко всем имеющимся ресурсам Azure Cosmos DB, которые соответствуют условиям резервирования. Если у вас нет ресурсов Azure Cosmos DB, резервирование будет применяться при развертывании нового экземпляра Cosmos DB, который соответствует условиям резервирования. В обоих случаях период резервирования начинается сразу после успешной операции приобретения.

По истечении срока действия резервирования экземпляры Azure Cosmos DB будут продолжать работу, и плата за них будет взиматься по обычным тарифам оплаты по мере использования.

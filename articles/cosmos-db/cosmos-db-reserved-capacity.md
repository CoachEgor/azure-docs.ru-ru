# <a name="optimize-cost-with-reserved-capacity-in-azure-cosmos-db"></a>Оптимизация затрат на резервную мощностью в Azure Cosmos DB

Резервная мощность Azure Cosmos DB позволяет экономить деньги за счет предварительной оплаты ресурсов Azure Cosmos DB в течение одного года или трех лет. С помощью резервной мощности Azure Cosmos DB можно получить скидку на пропускную способность, подготовленную для ресурсов Cosmos DB. Примерами ресурсов являются базы данных и контейнеры (таблицы, коллекции и графы).

Однолетние и трехлетние контракты на резервную мощность Azure Cosmos DB с предварительной оплатой позволят сэкономить существенную сумму &mdash;до 65 % по сравнению с обычной платой. Резервная мощность предоставляет скидку на выставление счетов и не влияет на состояние среды выполнения ресурсов Azure Cosmos DB.

Резервная мощность Azure Cosmos DB покрывает расходы на пропускную способность, подготовленную для ресурсов. Но она не распространяется на расходы хранилища и сети. После того как вы купите резервирование, при начислении платы за пропускную способность, соответствующей атрибутам резервирования, не будут применяться ставки оплаты по мере использования. Дополнительные сведения о резервировании см. в статье [Общие сведения об Azure Reserved VM Instances](../billing/billing-save-compute-costs-reservations.md).

Вы можете приобрести резервную мощность Azure Cosmos DB на [портале Azure](https://portal.azure.com). Требования для приобретения резервной мощности.

* Необходимо быть назначена роль владельца для по крайней мере один Enterprise или отдельные подписки с оплатой.  
* Для подписок с соглашением Enterprise параметр **Добавить зарезервированные экземпляры** следует включить на [портале EA](https://ea.azure.com). Или, если этот параметр отключен, необходимо быть администратором подписки EA.
* В рамках программы для поставщиков облачных решений (CSP) только агенты администрирования или агенты продаж могут приобрести резервную мощность Azure Cosmos DB.

## <a name="determine-the-required-throughput-before-purchase"></a>Определение требуемой пропускной способности перед приобретением

Размер резервирования должен основываться на общем объеме пропускной способности, используемой имеющимися или будущими ресурсами Azure Cosmos DB. Требуемую пропускную способность можно определить одним из следующих способов:

* Получите данные журнала для общей подготовленной пропускной способности в ваших учетных записях Azure Cosmos DB, базах данных и коллекциях во всех регионах. Например, вы можете оценить ежедневную подготовленную пропускную способность, загрузив свой ежедневной отчет по использованию из `https://account.azure.com`.

* Если вы являетесь клиентом с соглашением Enterprise (EA), то для получения сведений о пропускной способности Azure Cosmos DB можно загрузить файл своего потребления. См. значение **Тип службы** в разделе **Дополнительная информация** файла потребления.

* Вы также можете суммировать среднюю пропускную способность для всех рабочих нагрузок в учетных записях Azure Cosmos DB, которая планируется на следующие один или три года. Затем это количество можно использовать для резервирования.

## <a name="buy-azure-cosmos-db-reserved-capacity"></a>Приобретение резервной мощности Azure Cosmos DB

1. Войдите на [портале Azure](https://portal.azure.com).  

2. Выберите **Все службы** > **Резервирования** > **Добавить**.  

3. Из **приобрести резервирование** панели выберите **Azure Cosmos DB** купить новое резервирование.  

4. Заполните обязательные поля, как описано в следующей таблице.

   ![Заполнение формы резервной мощности](./media/cosmos-db-reserved-capacity/fill-reserved-capacity-form.png)

   |Поле  |Описание  |
   |---------|---------|
   |`Scope`   |   Параметр, который определяет, сколько подписок может использовать преимущество на выставление счетов, связанное с резервированием. Он также контролирует, как резервирование применяется к определенным подпискам. <br/><br/>  Если выбрать **Любая подписка**, скидка на резервирование применяется к экземплярам Azure Cosmos DB, работающим в любых подписках в текущем контексте выставления счетов. Контекст выставления счетов зависит от способа регистрации в Azure. Для клиентов с соглашением Enterprise общая область действует в рамках регистрации и включает в себя все подписки, заданные при регистрации. Для клиентов, по мере использования общая область включает все отдельные подписки с оплатой, созданные администратором учетной записи.  <br/><br/>  Если выбрать **Одна подписка**, скидка на резервирование применяется к экземплярам Azure Cosmos DB в выбранной подписке. <br/><br/> При выборе **одной группе ресурсов**, скидка на резервирование применяется к экземплярам Azure Cosmos DB в выбранной подписке и выбранной группе ресурсов в рамках этой подписки. <br/><br/> После приобретения резервной мощности можно изменить область резервирования.  |
   |Подписка  |   Подписка, используемая для оплаты резервной мощности Azure Cosmos DB. Метод оплаты для выбранной подписки используется для взимания первоначальных затрат. Необходимо наличие одного из следующих типов подписки: <br/><br/>  Соглашение Enterprise (номера предложений: MS-AZR-0017P или MS-AZR-0148P): для подписки Enterprise плата вычитается из баланса денежных обязательств или относится к избыточным расходам. <br/><br/> Отдельные подписки с оплатой (предлагают номера: MS-AZR-0003P или MS-AZR 0023P). Для отдельной подписки с оплатой плата взимается способ оплаты кредитной карты или счета для подписки.    |
   | Группа ресурсов | Группа ресурсов, к которому применяется скидка на зарезервированный объем ресурсов. |
   |Термин  |   Один или три года.   |
   |Тип пропускной способности   |  Пропускная способность предоставляется в качестве единиц запросов. Вы можете приобрести резервирование для подготовленной пропускной способности для настройки обоих параметров — один регион записывает записи также несколькими регионами. Пропускная способность тип имеет два значения: 100 ЕЗ/с в час или 100 ЕЗ/с несколькими хозяевами в час.|
   | Зарезервированные единицы емкости| Объем пропускной способности, который необходимо зарезервировать. Вы можете вычислить это значение путем определения пропускной способности, необходимой для всех ресурсов Cosmos DB (например, баз данных или контейнеров) в регионе. Затем умножьте его на количество регионов, которые необходимо связать с базой данных Cosmos DB. Пример: Если у вас есть пять регионов с 1 млн ЕЗ/с в каждом, выберите 5 миллионов ЕЗ/с, чтобы приобрести емкость резервирования. |
 

5. После заполнения формы, цена, необходимо приобрести зарезервированный объем ресурсов рассчитывается. Выходные данные также показывает процент скидки, которые можно получить с выбранными параметрами. Далее щелкните **выберите**

6. В **приобрести резервирование** области просмотрите скидки и стоимость резервирования. Цена за резервирование применяется к ресурсам Azure Cosmos DB с подготовленной пропускной способностью во всех регионах.  

   ![Зарезервированная емкость сводки](./media/cosmos-db-reserved-capacity/reserved-capacity-summary.png)

7. Выберите **проверки + купить** и затем **купить сейчас**. После успешной покупки откроется следующая страница.

После приобретения резервирования оно немедленно применится ко всем имеющимся ресурсам Azure Cosmos DB, которые соответствуют условиям резервирования. Если у вас нет ресурсов Azure Cosmos DB, резервирование будет применяться при развертывании нового экземпляра Cosmos DB, который соответствует условиям резервирования. В обоих случаях период резервирования начинается сразу после успешной операции приобретения.

По истечении срока действия резервирования экземпляры Azure Cosmos DB будут продолжать работу, и плата за них будет взиматься по обычным тарифам оплаты по мере использования.

# <a name="security-attributes-azure-cosmos-db"></a>Атрибуты безопасности, Azure Cosmos DB

В этой статье описываются наиболее распространенные атрибуты безопасности, встроенных в Azure Cosmos DB.

[!INCLUDE [Security Attributes Header](../../includes/security-attributes-header.md)]

## <a name="preventative"></a>профилактическая;

| Атрибут безопасности | Да/Нет | Примечания |
|---|---|--|
| Шифрование неактивных данных (например, шифрование на стороне сервера, шифрование на стороне сервера с управляемыми клиентом ключами и другие функции шифрования) | Да | Все базы данных Cosmos DB и резервные копии шифруются по умолчанию; см. в разделе [шифрование данных в Azure Cosmos DB](database-encryption-at-rest.md). Шифрование на стороне сервера с управляемыми клиентом ключами не поддерживается. |
| Шифрование при обмене данными (например, шифрование ExpressRoute в виртуальной сети шифрования и шифрование между сетями)| Да | Все данные Azure Cosmos DB шифруются при передаче. |
| Обработка ключа шифрования (CMK, BYOK и т. д.)| Нет |  |
| Шифрование на уровне столбцов (службы данных Azure)| Да | Только в таблицах API категории "премиум". Не все API-интерфейсы поддерживают эту функцию. См. в разделе [Знакомство с Azure Cosmos DB: API таблицы](table-introduction.md). |
| Вызовы API в зашифрованном виде| Да | Все подключения к Azure Cosmos DB поддерживает протокол HTTPS. Azure Cosmos DB также поддерживает подключения TLS 1.2, но это еще не задается. Если клиенты отключить низкого уровня TLS с их стороны, они могут обеспечить для подключения к Cosmos DB.  |

## <a name="network-segmentation"></a>сегментация сети;

| Атрибут безопасности | Да/Нет | Примечания |
|---|---|--|
| Поддержка конечной точки службы| Да |  |
| Поддержка внедрения виртуальной сети| Да | С конечной точкой службы виртуальной сети можно настроить учетную запись Azure Cosmos DB, чтобы разрешить доступ только из определенной подсети виртуальной сети (VNet). Можно также объединять доступ виртуальной сети с помощью правил брандмауэра.  См. в разделе [доступа к Azure Cosmos DB из виртуальных сетей](vnet-service-endpoint.md). |
| Поддержка сетевой изоляции и Firewalling| Да | Благодаря поддержке брандмауэра можно настроить учетную запись Azure Cosmos, чтобы разрешить доступ только из утвержденных IP-адресов, диапазон IP-адресов и (или) облачных служб. См. в разделе [настроить IP-адрес брандмауэра в Azure Cosmos DB](how-to-configure-firewall.md).|
| Поддержка принудительного туннелирования | Да | Можно настроить на стороне клиента в виртуальной сети, где расположены виртуальные машины.   |

## <a name="detection"></a>Обнаружение

| Атрибут безопасности | Да/Нет | Примечания|
|---|---|--|
| Azure Monitor поддержки (Log analytics, Application insights, и т.д.)| Да | Регистрируются все запросы, которые отправляются в Azure Cosmos DB. [Мониторинг Azure](../azure-monitor/overview.md), поддерживаемых метрик Azure, журналов аудита Azure.  Вы можете войти сведения, относящиеся к запросы плоскости данных, статистики времени выполнения запросов, текст запроса, запрашивает MongoDB. Можно также настроить предупреждения. |

## <a name="identity-and-access-management"></a>Управление удостоверениями и доступом

| Атрибут безопасности | Да/Нет | Примечания|
|---|---|--|
| Authentication| Да | Да, на уровне учетной записи базы данных; на уровне плоскости данных Cosmos DB использует маркеры ресурсов и доступ к ключу. |
| Авторизация| Да | В учетной записи Azure Cosmos с помощью главных ключей (первичных и вторичных) и маркерах ресурсов поддержки. Вы можете получить чтение и запись или доступ только к данным с помощью главных ключей для чтения. Маркеры ресурсов позволяют ограниченного времени доступ к ресурсам, таких как документы и контейнеры. |

|---|---|--|

| Ведение журнала и аудит плоскости данных | Да | Диагностика, мониторинг, такие как ведение журнала для операций уровня контейнера создайте контейнер, Подготовка пропускной способности, индексирование, политики и операции CRUD с документами. |

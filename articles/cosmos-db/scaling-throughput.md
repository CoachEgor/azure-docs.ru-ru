# <a name="globally-scale-provisioned-throughput"></a>Глобальное масштабирование подготовленной пропускной способности 

В Azure Cosmos DB прокладываемая пропускная часть представлена как единицы запроса/секунда (RU/s или rUs формы множественного числа). Эти единицы запросов измеряют стоимость операций чтения и записи в контейнере Cosmos, как показано на рисунке ниже.

![Единицы запросов](./media/scaling-throughput/request-unit-charge-of-read-and-write-operations.png)

Вы можете подготовить ЕЗ в контейнере Cosmos или базе данных Cosmos. RUs, подготовленные на контейнере, доступны исключительно для операций, выполняемых на этом контейнере. ЕЗ, подготовленные в базе данных, распределяются между всеми контейнерами в этой базе данных (за исключением контейнеров с отдельно назначенными ЕЗ).

Для упругого масштабирования прокладки можно увеличить или уменьшить предусмотрительную RU/s в любое время. Для получения дополнительной информации см. [Как обеспечить пропускную емкость](set-throughput.md) и упруго масштабировать контейнеры и базы данных Cosmos. Для глобального масштабирования пропускной записи можно в любое время добавить или удалить регионы из учетной записи Cosmos. Дополнительные сведения см. в разделе [Добавление и удаление регионов из учетной записи базы данных](how-to-manage-database-account.md#addremove-regions-from-your-database-account). Связывание нескольких регионов с учетной записью Космос имеет важное значение во многих сценариях - для достижения низкой задержки и [высокой доступности](high-availability.md) по всему миру.

## <a name="how-provisioned-throughput-is-distributed-across-regions"></a>Распределение подготовленной пропускной способности по регионам

Если вы предоставляете *R'* RUs в контейнере (или базе данных Космоса), Cosmos DB гарантирует, что *'R'* RUs доступны в *каждом* регионе, связанном с вашей учетной записью Cosmos. Каждый раз, когда вы добавляете новый регион в свой аккаунт, Cosmos DB автоматически проветрывает *R'* RUs в недавно добавленном регионе. Операции, выполняемые против контейнера Cosmos, гарантированно получают *R'* RUs в каждом регионе. Нельзя выборочно назначать ЕЗ для определенного региона. RUs, подготовленные на контейнере Космоса (или базе данных), подготовлены во всех регионах, связанных с вашей учетной записью Космос.

Предполагая, что контейнер Космос настроен с *r'* R и есть *"N"* регионов, связанных с счетом Космос, то:

- Если учетная запись Cosmos настроена с одной областью записи, то общая сумма рС, доступная во всем мире, на контейнере *R* x *N.*

- Если учетная запись Cosmos настроена с несколькими регионами записи, то общая сумма, доступная rUs глобально в контейнере *r* x *(N*Q1). Дополнительные *R-R* автоматически готовятся к обработке конфликтов обновления и антиэнтропного трафика в регионах.

Выбор [модели согласованности](consistency-levels.md) также влияет на пропускную стоимость. Вы можете получить примерно 2x считывательной пропускной стол для более расслабленной консистенции уровней (например, *сессии*, *последовательной префикс* и *в конечном итоге* последовательность) по сравнению с более сильными уровнями согласованности (например, *ограниченной черствости* или *сильной* последовательности).

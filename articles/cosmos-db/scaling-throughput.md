# <a name="globally-scale-provisioned-throughput"></a>Глобальное масштабирование подготовленной пропускной способности 

В Azure Cosmos DB, подготовленной пропускной способности представлены в виде запроса единиц в секунду (ЕЗ/с или множественном RUs). Эти единицы запросов измеряют стоимость операций чтения и записи в контейнере Cosmos, как показано на рисунке ниже.

![Единицы запросов](./media/scaling-throughput/request-unit-charge-of-read-and-write-operations.png)

Вы можете подготовить ЕЗ в контейнере Cosmos или базе данных Cosmos. Количество единиц в контейнере доступны исключительно для операций, выполняемых в этом контейнере. ЕЗ, подготовленные в базе данных, распределяются между всеми контейнерами в этой базе данных (за исключением контейнеров с отдельно назначенными ЕЗ).

Используйте возможность гибкого масштабирования подготовленной пропускной способности, можно увеличить или уменьшить подготовленных ЕЗ/с в любое время. Дополнительные сведения см. в разделе [руководства Подготовка пропускной способности](set-throughput.md) и гибко масштабировать Cosmos контейнеры и баз данных. Для глобально Масштабирование пропускной способности, можно добавить или удалить регионы из учетной записи Cosmos в любое время. Дополнительные сведения см. в разделе [Добавление и удаление регионов из учетной записи базы данных](how-to-manage-database-account.md#addremove-regions-from-your-database-account). Связывание нескольких регионов с учетной записью Cosmos важно во многих сценариях — чтобы обеспечить низкую задержку и [высокий уровень доступности](high-availability.md) по всему миру.

## <a name="how-provisioned-throughput-is-distributed-across-regions"></a>Распределение подготовленной пропускной способности по регионам

Если вы подготавливаете *'R'* RUs Cosmos контейнера (или базы данных), Cosmos DB гарантирует, что *'R'* RUs доступны в *каждого* регион, связанный с вашей учетной записи Cosmos. Каждый раз при добавлении новой области в учетную запись Cosmos DB автоматически подготавливает *'R'* RUs в вновь добавленный регион. Операции, выполняемые для вашего контейнера Cosmos гарантированно получат *'R'* RUs в каждом регионе. Нельзя выборочно назначать ЕЗ для определенного региона. Количество единиц в контейнере Cosmos (или базы данных), подготавливаются во всех регионах, связанных с учетной записью Cosmos.

При условии, что контейнер Cosmos настраивается с помощью *'R'* RUs и *"n"* регионов, связанных с учетной записью Cosmos, затем:

- Если учетная запись Cosmos настроена с областью одной операции записи, общее RUs глобально в контейнере = *R* x *N*.

- Если учетная запись Cosmos настроена с использованием нескольких регионов записи, общее RUs глобально в контейнере = *R* x (*N*+ 1). Дополнительный *R* RUs будут автоматически добавляться конфликты процесс обновления и защиты от энтропии трафика во всех регионах.

Выбор [модель согласованности](consistency-levels.md) также влияет на пропускную способность. Вы можете получить примерно 2 x пропускная способность при чтении для уровней более нестрогой согласованности (например, *сеанса*, *согласованность префиксов* и *итоговой* согласованности) по сравнению с строже уровень согласованности (например, *ограниченное устаревание* или *строгого* согласованности).

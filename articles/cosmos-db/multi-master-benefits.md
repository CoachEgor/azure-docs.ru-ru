# <a name="understand-multi-master-benefits-in-azure-cosmos-db"></a>Узнайте о преимуществах использования нескольких основных возможностей в Azure Cosmos DB

Cosmos DB операторы учетных записей должны выбрать соответствующую глобальную конфигурацию распространения, чтобы обеспечить требования к задержке, доступности и RTO для своих приложений. Учетные записи Cosmos для Azure, настроенные с несколькими расположениями для записи, имеют значительные преимущества по сравнению с учетными записями с единственным местом записи, включая 99,999% уровня обслуживания для записи, < 10 мс соглашение об уровне обслуживания для задержки записи в 99-м процентиль и RTO = 0 при региональной аварии.

## <a name="comparison-of-features"></a>Сравнение функций

|Требование к приложению|Несколько расположений для записи|Место для одной записи|Примечание|
|---|---|---|---|
|Соглашение об уровне обслуживания для задержки записи < 10 мс в P99|**Да**|Нет|Учетные записи с одним расположением записи вызывают дополнительную задержку сети между регионами для каждой записи.|
|Соглашение об уровне обслуживания для задержки чтения < 10 мс по адресу P99|**Да**|ДА| |
|Написание соглашения об уровне обслуживания 99,999%|**Да**|Нет|Учетные записи с одним местом записи гарантируют соглашение об уровне обслуживания 99,99%|
|RTO = 0|**Да**|Нет|Обнуление времени записи в случае региональной аварии. Учетные записи с одним расположением записи имеют значение RTO, равное 15 минутам.|

# <a name="understand-multi-master-benefits-in-azure-cosmos-db"></a>Понять преимущества мультимастеров в Azure Cosmos DB

Операторы учетных записей Cosmos DB должны выбрать соответствующую глобальную конфигурацию дистрибуции, чтобы обеспечить задержку, доступность и требования RTO для своих приложений. Учетные записи Azure Cosmos, настроенные с несколькими местоположениями записей, предлагают значительные преимущества по сравнению с учетными записями с одним местоположением записи, включая, 99,999% записывает доступность SLA, <10 мс написать задержку SLA на 99-м процентиле и RTO 0 в региональной катастрофе.

## <a name="comparison-of-features"></a>Сравнение функций

|Требование к приложению|Несколько Местоположений записи|Одноместный Написать Местоположение|Примечание|
|---|---|---|---|
|Написать задержку SLA <10ms на P99|**Да**|нет|Учетные записи с одноразовым местоположением записи повлекли за собой дополнительную задержку сети по межрегионутам для каждой записи.|
|Читать задержки SLA <10ms на P99|**Да**|Да| |
|Написать SLA 99.999%|**Да**|нет|Счета с однозначным местоположением записи гарантируют SLA 99.99%|
|RTO No 0|**Да**|нет|Нулевое время простоя для записей в случае региональных катастроф. Учетные записи с одним местоположением записи имеют RTO 15 мин.|

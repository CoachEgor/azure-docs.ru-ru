# <a name="regional-presence-with-azure-cosmos-db"></a>Географическое присутствие службы Azure Cosmos DB

Azure Cosmos DB является фундаментальной службой в Azure и, по умолчанию, всегда доступен во всех регионах, где доступна Azure. Сейчас Azure доступна в [54 регионах](https://azure.microsoft.com/global-infrastructure/regions/) по всему миру. 

[![Регионы, где доступна DB Azure Cosmos](./media/regional-presence/regional-presence.png)](./media/regional-presence/regional-presence.png#lightbox)

Cosmos DB доступна для клиентов в пяти различных облачных средах Azure:

* **Общедоступный пиринг Azure** — облако, которое доступно по всему миру.

* **Azure China 21Vianet** доступен благодаря уникальному партнерству между Microsoft и 21Vianet, одним из крупнейших интернет-провайдеров страны в Китае.

* **Azure для Германии** — предоставляет службы по модели доверенного лица по вопросам данных. Это означает, что данные клиента хранятся в Германии в T-Systems International GmbH, дочернем подразделении компании Deutsche Telecom, выступающем в роли доверенного лица по вопросам данных в Германии.

* **Azure для государственных организаций** — доступна в четырех регионах в Соединенных Штатах для государственных организаций США и их партнеров. 

* **Azure правительства для Министерства обороны (DOD)** доступна в двух регионах в Соединенных Штатах для Министерства обороны США.

## <a name="regional-presence-with-global-distribution"></a>Географическое присутствие с глобальным распределением

Все AA-иное, выставленные Azure Cosmos DB (включая S'L, MongoDB, Cassandra, Gremlin и Table) доступны во всех регионах Azure по умолчанию. Например, apIs MongoDB и Cassandra могут быть выставлены Azure Cosmos DB не только во всех регионах Лазурного региона, но и в суверенных облаках, таких как Китай, Германия, правительство и Министерство обороны (МО).

Azure Cosmos DB — это [глобально распределенная](distribute-data-globally.md) служба баз данных. Вы можете связать любое количество регионов Azure с учетной записью Azure Cosmos, и ваши данные будут автоматически и прозрачно реплицированы. Добавить или удалить регион в учетной записи Azure Cosmos можно в любое время. Благодаря комплексной возможности глобального распределения и протоколу репликации с несколькими источниками Azure Cosmos DB обеспечивает задержку операций чтения и записи менее 10 мс на уровне 99-го процентиля, возможность чтения и записи на уровне 99,999, а также возможность гибко масштабировать подготовленную пропускную способность для операций чтения и записи во всех регионах, связанных с учетной записью Azure Cosmos. Azure Cosmos DB также предлагает пять четко определенных моделей согласованности, и вы можете применить к своим данным конкретную модель. Наконец, Azure Cosmos DB является единственной службой баз данных в отрасли, которая обеспечивает всеобъемлющее [соглашение об уровне обслуживания (SLA),](https://azure.microsoft.com/support/legal/sla/cosmos-db/v1_2/) охватывающее просрочку пропускной записи, задержку на 99-м процентили, высокую доступность и согласованность. Вышеуказанные возможности доступны во всех облаках Azure.

# <a name="configure-and-approve-just-in-time-access-for-azure-managed-applications"></a>Настройка и утвердить доступ just-in-time для управляемых приложений Azure

Как пользователь управляемого приложения, может быть слишком сложными, предоставляя издателя постоянный доступ к управляемой группе ресурсов. Чтобы получить больший контроль над предоставлением доступа к управляемых ресурсов, управляемых приложений Azure предусмотрена возможность доступа just-in-time (JIT), которая в настоящее время в предварительной версии. Он позволяет утвердить, когда и как долго издателя имеет доступ к группе ресурсов. Издатели могут предлагать необходимых обновлений в течение этого времени, но при на это время, срок действия истекает доступ издателя.

— Рабочий процесс предоставления доступа:

1. Publisher добавляет управляемого приложения в Marketplace и указывает, что JIT-доступ к доступен.

1. Во время развертывания как разрешить JIT-доступ к свой экземпляр управляемого приложения.

1. После развертывания можно изменить параметры для JIT-доступ.

1. Издатель посылает запрос на доступ.

1. Утверждения запроса.

В этой статье рассматриваются действия, которые потребителей предпринять, чтобы включить JIT-доступ и утверждать запросы. Дополнительные сведения о публикации управляемого приложения с помощью JIT-доступ, см. в разделе [запросить доступ just-in-time в управляемых приложений Azure](request-just-in-time-access.md).

> [!NOTE]
> Для использования доступа just-in-time, необходимо иметь [лицензии Azure Active Directory P2](../active-directory/privileged-identity-management/subscription-requirements.md).

## <a name="enable-during-deployment"></a>Включить во время развертывания

1. Войдите на [портале Azure](https://portal.azure.com).

1. Найти запись marketplace для управляемого приложения с поддержкой JIT-компилятора. Нажмите кнопку **Создать**.

1. То же время предоставляя значения для нового управляемого приложения **JIT-конфигурация** шаг позволяет включить или отключить JIT-доступ для управляемого приложения. Выберите **Да** для **включить JIT-доступ к**. Этот параметр выбран по умолчанию для управляемых приложений, которые определены с помощью JIT-компилятора, включена в marketplace.

   ![Настройка доступа](./media/approve-just-in-time-access/configure-jit-access.png)

   JIT-доступ можно включить только во время развертывания. При выборе **нет**, издатели получают постоянный доступ к управляемой группе ресурсов. Невозможно включить JIT-доступ к более поздней версии.

1. Чтобы изменить параметры утверждения по умолчанию, выберите **Настройка JIT-конфигурация**.

   ![Настройка доступа](./media/approve-just-in-time-access/customize-jit-access.png)

   По умолчанию управляемого приложения с поддержкой JIT-имеет следующие параметры:

   * Режим одобрения — автоматическое
   * Длительность максимальный доступ — 8 часов
   * Утверждающие — none

   Если режим одобрения присвоено **автоматического**, утверждающих получают уведомление для каждого запроса, но запрос утверждается автоматически. Если задано значение **вручную**утверждающих получают уведомление для каждого запроса и один из них необходимо утвердить.

   Максимальная длительность активации указывает максимальное количество времени, которые могут запрашивать издателя для доступа к управляемой группе ресурсов.

   В списке утверждающих является Azure Active Directory пользователей, которые могут утверждать запросов JIT-доступа. Добавление утверждающего, выберите **Добавление утверждающего** и выполните поиск пользователя.

   После обновления параметр, выберите **Сохранить**.

## <a name="update-after-deployment"></a>Обновить после развертывания

Можно изменить значения как запросы утверждаются. Тем не менее если вы не включали JIT-доступ во время развертывания, его нельзя включить позже.

Чтобы изменить параметры для развернутого управляемого приложения:

1. На портале выберите управляемого приложения.

1. Выберите **JIT-конфигурация** и при необходимости измените параметры.

   ![Изменение параметров доступа](./media/approve-just-in-time-access/change-settings.png)

1. Затем нажмите кнопку **Сохранить**.

## <a name="approve-requests"></a>Утверждение запросов

Если издатель подает запрос доступа, вы получите уведомление об запроса. Вы можете утвердить запросы на доступ JIT напрямую с помощью управляемого приложения или для всех управляемых приложений через службу Azure AD Privileged Identity Management. Для использования доступа just-in-time, необходимо иметь [лицензии Azure Active Directory P2](../active-directory/privileged-identity-management/subscription-requirements.md).

Утверждение запросов с помощью управляемого приложения:

1. Выберите **JIT-доступ к** для управляемого приложения и выберите **утверждение запросов**.

   ![Утверждение запросов](./media/approve-just-in-time-access/approve-requests.png)
 
1. Выберите запрос, который требуется утвердить.

   ![Выберите запрос](./media/approve-just-in-time-access/select-request.png)

1. В форме, укажите причину утверждения и выберите **утвердить**.

Чтобы утвердить запросы через Azure AD Privileged Identity Management:

1. Выберите **все службы** и начинать поиск **Azure AD Privileged Identity Management**. Выберите его из списка доступных вариантов.

   ![Поиск службы](./media/approve-just-in-time-access/search.png)

1. Выберите **утверждать запросы**.

   ![Выберите утверждение запросов](./media/approve-just-in-time-access/select-approve-requests.png)

1. Выберите **управляемые приложения Azure**и выберите запрос на утверждение.

   ![Выберите запросы](./media/approve-just-in-time-access/view-requests.png)

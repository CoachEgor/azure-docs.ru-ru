# <a name="orchestration-mode-preview"></a>Режим оркестровки (предварительный просмотр)

Наборы масштабов виртуальных машин обеспечивают логическую группировку управляемых платформами виртуальных машин. С наборами масштабов вы создаете виртуальную модель конфигурации машины, автоматически добавляете или удаляете дополнительные экземпляры на основе нагрузки процессора или памяти, а автоматически обновляетесь до последней версии ОС. Традиционно наборы масштабов позволяют создавать виртуальные машины с помощью модели конфигурации VM, предоставленной на момент создания набора масштаба, а набор масштабов может управлять только виртуальными машинами, которые неявно создаются на основе модели конфигурации.

С режимом оркестровки набора масштабов (предварительный просмотр) теперь можно выбрать, должен ли набор масштабов организовывать виртуальные машины, которые создаются явно вне модели конфигурации набора масштабов, или виртуальные экземпляры машины, созданные неявно на основе конфигурации модели. Режим организации установки масштаба также помогает вам проектировать инфраструктуру VM для высокой доступности, развернув эти VMs в доменах неисправностей и зонах доступности.


Виртуальные наборы масштабов машин будут поддерживать 2 различных режима оркестровки:

- ScaleSetVM - Виртуальные экземпляры машины, добавленные в набор масштабов, основаны на модели конфигурации набора масштабов. Виртуальный жизненный цикл экземпляра машины - создание, обновление, удаление - управляется набором масштаба.
- VM (виртуальные машины) - Виртуальные машины, созданные вне набора масштабов, могут быть явно добавлены в шкалу. 
 

> [!IMPORTANT]
> Режим оркестровки определяется при создании набора масштаба и не может быть изменен или обновлен позже. 
> 
> Эта функция виртуальных наборов масштабов машины в настоящее время в открытом доступе.
> Эта предварительная версия предоставляется без соглашения об уровне обслуживания и не рекомендована для использования рабочей среде. Некоторые функции могут не поддерживаться или их возможности могут быть ограничены. 
> Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).


## <a name="orchestration-modes"></a>Режимы оркестрации

|                             | "оркестровка": "VM" (VirtualMachine) | "Оркестрия": "ScalesetVM" (VirtualMachineScaleSetVM) |
|-----------------------------|--------------------------------------------|--------------------------------------------------------------|
| Модель конфигурации VM      | None                                       | Обязательно |
| Добавление нового VM в набор масштабов  | VM явно добавляются к шкале, установленной при создании VM. | ВМ неявно создаются и добавляются в набор масштабов, основанный на модели конфигурации VM, подсчете экземпляров и правилах autoScaling | |
| Удаление виртуальной машины                   | VMs должны быть удалены индивидуально, набор масштабов не будет удален, если он имеет какие-либо VMs в нем. | VMs могут быть удалены по отдельности, удаление набора масштабов удалит все экземпляры VM.  |
| Прикрепить/отсоединить ВМ           | Не поддерживается                              | Не поддерживается |
| Жизненный цикл инстанции (Создание путем удаления) | ВМ и их артефакты (например, диски и NICs) можно управлять самостоятельно. | Инстанции и их артефакты (например, диски и NICs) подразумеваются для наборов масштаба экземпляров, которые их создают. Они не могут быть отделены или управляются отдельно за пределами набора масштаба |
| Домены сбоя               | Можно определить домены неисправностей. 2 или 3 на основе региональной поддержки и 5 для зоны доступности. | Может определить домены неисправностей, идущие от 1 до 5 |
| Домены обновления              | Обновления доменов автоматически отображаются на доменах неисправностей | Обновления доменов автоматически отображаются на доменах неисправностей |
| зоны доступности;          | Поддержка регионального развертывания или ВМ в одной зоне доступности | Поддерживает региональное развертывание или несколько зон доступности; Может определить стратегию балансировки зон |
| Автомасштабирование                   | Не поддерживается                              | Поддерживается |
| Обновление ОС                  | Не поддерживается                              | Поддерживается |
| Обновления моделей               | Не поддерживается                              | Поддерживается |
| Контроль экземпляров            | Полный vM-контроль. VMs имеют полную квалификацию URI, которая поддерживает весь спектр возможностей управления Azure VM (например, политика Azure, резервное копирование Azure и восстановление сайта Azure) | ВМ зависят ресурсы набора масштабов. Инстанции могут быть доступны для управления только через набор масштаба. |
| Модель инстанции              | Определение модели Microsoft.Compute/VirtualMachines. | Microsoft.Compute/VirtualMachineScaleSets/VirtualMachines определение модели. |
| Capacity                    | Можно создать пустой набор масштабов; до 200 ВМ могут быть добавлены в набор масштабов | Наборы шкалы можно определить с числом экземпляров 0 - 1000 |
| Переместить                        | Поддерживается                                  | Поддерживается |
| Одноразовая группа размещения ложной | Не поддерживается                          | Поддерживается |

# <a name="view-azure-automation-update-management-update-assessments"></a>Просмотр оценок обновления управления обновлением автоматизации Azure

В учетной записи Azure Automation выберите **управление обновлением** для просмотра состояния ваших машин.

В этом представлении содержатся сведения о ваших компьютерах, отсутствующих обновлениях, развертываниях обновлений и запланированных развертываниях обновлений. В колонке **COMPLIANCE** можно увидеть последний раз, когда машина была оценена. В колонке **UPDATE AGENT READINESS** вы можете увидеть работоспособность агента обновления. Если возникла проблема, выберите ссылку, чтобы перейти к документации по устранению неполадок, которая поможет вам исправить проблему.

Для выполнения поиска журналов, возвращающем информацию о машине, обновлении или развертывании, выберите соответствующий элемент в списке. Панель поиска журнала открывается запросом для выбранного элемента.

![Представление по умолчанию решения "Управление обновлениями"](media/automation-update-management/update-management-view.png)

## <a name="view-missing-updates"></a>Просмотр отсутствующих обновлений

Выберите **Отсутствующие обновления**, чтобы просмотреть список обновлений, отсутствующих на ваших компьютерах. Каждое обновление перечислено и может быть выбрано. Отображается информация о количестве машин, требующих обновления, детали операционной системы и ссылка на дополнительную информацию. Панель поиска журнала также показывает более подробную информацию об обновлениях.

![отсутствующие обновления.](./media/automation-view-update-assessments/automation-view-update-assessments-missing-updates.png)

## <a name="update-classifications"></a>Классификации обновлений

В следующих таблицах перечисляются поддерживаемые классификации обновлений в области управления обновлениями с определением для каждой классификации.

### <a name="windows"></a>Windows

|Классификация  |Описание  |
|---------|---------|
|критические обновления;     | Обновления для конкретных проблем, которые устраняются критические ошибки, не связанные с безопасностью.        |
|Обновления для системы безопасности     | Обновления для конкретных продуктов, связанных с безопасностью вопросов.        |
|накопительные пакеты обновления;     | Наборы горячих фиксов, упакованных вместе для легкого развертывания.        |
|пакеты дополнительных компонентов;     | Новые функции продукта, которые распространяются вне выпуска продукта.        |
|пакеты обновления;     | Наборы горячих фиксов, которые применяются к приложению.        |
|обновления определений;     | Обновления для вирусных или других файлов определения.        |
|Инструменты     | Утилиты или функции, которые помогают выполнить одну или несколько задач.        |
|Обновления     | Обновления для приложений или файлов, которые установлены в настоящее время.        |

### <a name="linux"></a><a name="linux-2"></a>Linux

|Классификация  |Описание  |
|---------|---------|
|критические обновления и обновления для системы безопасности;     | Обновления для конкретной проблемы или проблемы, связанной с безопасностью продукта.         |
|Другие обновления     | Все другие обновления, которые не являются критическими по своему характеру или которые не являются обновлениями безопасности.        |

Для Linux управление обновлениями может проводить различие между критическими обновлениями и обновлениями безопасности в облаке при отображении данных оценки. (Эта детализация возможна благодаря обогащению данных в облаке.) Для исправления update Management опирается на классификационные данные, доступные на машине. В отличие от других дистрибутивов, CentOS не имеет этой информации, доступной в версиях продукта RTM. Если у вас есть машины CentOS, настроенные для возврата данных безопасности для следующей команды, управление обновлением может патч на основе классификаций:

```bash
sudo yum -q --security check-update
```

В настоящее время нет поддерживаемого метода, позволяющего обеспечить наличие нативных классификационных данных на CentOS. В настоящее время клиентам, которые самостоятельно включили эту функцию, предоставляется только поддержка с наилучшими усилиями.

Чтобы классифицировать обновления на версии Red Hat Enterprise 6, необходимо установить плагин для безопасности yum. На Red Hat Enterprise Linux 7, плагин уже является частью сам yum, нет необходимости устанавливать что-либо. Для получения дополнительной информации, [knowledge article](https://access.redhat.com/solutions/10021)см.

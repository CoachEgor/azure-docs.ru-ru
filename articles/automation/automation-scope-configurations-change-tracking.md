# <a name="work-with-scope-configurations-for-change-tracking-and-inventory"></a>Работа с конфигурациями области для Отслеживания изменений и инвентаризации

В этой статье рассказывается, как работать с конфигурациями областей при включенной возможности [Управление обновлениями](automation-update-management.md) для виртуальных машин. 

## <a name="sign-in-to-azure"></a>Вход в Azure

Войдите на портал Azure по адресу https://portal.azure.com.

## <a name="check-the-scope-configuration"></a><a name="scope-configuration"></a>Проверка конфигурации области

В Управлении обновлениями конфигурация области используется в пределах рабочей области Log Analytics для определения целевых компьютеров, для которых будет применяться Управление обновлениями. Конфигурация области — это группа из одного или нескольких сохраненных поисковых запросов, которая используется для ограничения области применения функции к конкретным компьютерам. Для получения доступа к конфигурациям области сделайте следующее:

1. В учетной записи службы автоматизации в разделе **Связанные ресурсы** выберите **Рабочая область**. 

2. В разделе **Источники данных рабочей области** выберите рабочую область, а затем щелкните **Конфигурации области**.

3. Если в выбранной рабочей области Управление обновлениями еще не включено, создается конфигурация области `MicrosoftDefaultScopeConfig-ChangeTracking`. 

4. Если в выбранной рабочей области это решение включено, оно не развертывается повторно и конфигурация области не добавляется. 

5. Нажмите кнопку с многоточием для любой из конфигураций областей и выберите **Изменить**. 

6. В области изменения щелкните **Выбор группы компьютеров**. В области "Группы компьютеров" отображаются сохраненные поисковые запросы, которые используются для создания конфигурации области.

## <a name="view-a-saved-search"></a>Просмотр сохраненного поиска

При добавлении компьютера в Отслеживание изменений и инвентаризацию он также добавляется в сохраненный поисковый запрос в рабочей области. Сохраненный поисковый запрос — это запрос, который содержит целевые компьютеры.

1. Перейдите к рабочей области Log Analytics и в разделе **Общие** выберите **Сохраненные поисковые запросы**. Сохраненный поисковый запрос, используемый Управлением обновлениями, выглядит так:

    |Имя     |Категория  |Псевдоним  |
    |---------|---------|---------|
    |MicrosoftDefaultComputerGroup     |  ChangeTracking       | ChangeTracking__MicrosoftDefaultComputerGroup        |

2. Выберите сохраненный поисковый запрос, чтобы просмотреть запрос, используемый для заполнения группы. На следующем рисунке показан запрос и его результаты:

    ![Сохраненные поисковые запросы](media/automation-scope-configurations-change-tracking/logsearch.png)

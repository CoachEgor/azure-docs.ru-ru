# <a name="my-first-graphical-runbook"></a>Первый графический Runbook

> [!div class="op_single_selector"]
> * [Графический](automation-first-runbook-graphical.md)
> * [PowerShell](automation-first-runbook-textual-powershell.md)

В данном учебнике описана процедура создания [графического модуля Runbook](automation-runbook-types.md#graphical-runbooks) в службе автоматизации Azure. Начните с простой книги, которую вы можете протестировать и опубликовать, изучая, как отслеживать состояние задания runbook. Затем измените руникум, чтобы фактически управлять ресурсами Azure, в данном случае запуская виртуальную машину Azure. Заполните учебник, чтобы сделать runbook более надежным, добавив параметры Runbook и условные ссылки.

>[!NOTE]
>Эта статья была изменена и теперь содержит сведения о новом модуле Az для Azure PowerShell. Вы по-прежнему можете использовать модуль AzureRM, исправления ошибок для которого будут продолжать выпускаться как минимум до декабря 2020 г. Дополнительные сведения о совместимости модуля Az с AzureRM см. в статье [Introducing the new Azure PowerShell Az module](https://docs.microsoft.com/powershell/azure/new-azureps-module-az?view=azps-3.5.0) (Знакомство с новым модулем Az для Azure PowerShell). Для инструкций по установке модуля Az на гибридном Runbook Worker [см.](https://docs.microsoft.com/powershell/azure/install-az-ps?view=azps-3.5.0) Для учетной записи Автоматизация вы можете обновить свои модули до последней версии, используя [как обновить модули Azure PowerShell в Azure Automation.](automation-update-azure-modules.md)

## <a name="prerequisites"></a>Предварительные требования

Для работы с этим учебником требуется:

* Подписка Azure. Если у вас ее нет, [активируйте преимущества для подписчиков MSDN](https://azure.microsoft.com/pricing/member-offers/msdn-benefits-details/) или [зарегистрируйте бесплатную учетную запись](https://azure.microsoft.com/free/?WT.mc_id=A261C142F).
* [Учетная запись службы автоматизации](automation-offering-get-started.md) , чтобы хранить модуль Runbook и выполнять проверку подлинности ресурсов Azure. Эта учетная запись должна иметь разрешение на запуск и остановку виртуальной машины.
* Виртуальная машина Azure. Так как вы остановитесь и запустите эту машину, она не должна быть производством VM.

## <a name="step-1---create-runbook"></a>Шаг 1. Создание нового модуля Runbook

Начните с создания простого runbook, который выводит текст. `Hello World`

1. На портале Azure выберите свою учетную запись службы автоматизации. 

    Страница учетной записи в службе автоматизации позволяет быстро получить представление о ресурсах, доступных в этой учетной записи. Некоторые ресурсы уже должны быть доступны. Большинство из этих активов являются модулями, автоматически включенными в новую учетную запись Автоматизация. Актив credential также должен быть связан с подпиской.
2. Выберите **Runbooks** под **автоматизацией процессов,** чтобы открыть список runbooks.
3. Создайте новый runbook, выбрав **Создать runbook**.
4. Присвойте модулю Runbook имя **MyFirstRunbook-Graphical**.
5. В этом случае вы собираетесь создать [графический runbook](automation-graphical-authoring-intro.md). Выберите **графический** для **типа Runbook**.<br> ![Новый Runbook](media/automation-first-runbook-graphical/create-new-runbook.png)<br>
6. Щелкните **Создать** , чтобы создать модуль Runbook и открыть графический редактор.

## <a name="step-2---add-activities"></a>Шаг 2 - Добавить действия

Элемент управления "Библиотека" в левой части редактора позволяет добавлять действия в модуль Runbook. Вы собираетесь добавить `Write-Output` cmdlet к выходу текста из runbook.

1. В управлении библиотекой нажмите в `write-output`поле поиска и введите . Результаты поиска отображаются на следующем изображении. <br> ![Microsoft.PowerShell.Utility](media/automation-first-runbook-graphical/search-powershell-cmdlet-writeoutput.png)
1. Прокрутите список до конца. Правый щелчок **Запись-выход** и выберите **Добавить в холст**. Кроме того, вы можете нажать эллипсис (...) рядом с названием cmdlet, а затем выберите **Добавить в холст.**
1. Щелкните **Write-Output** на холсте. Откроется страница управления конфигурацией, где можно настроить выбранное действие.
1. Поле **Label** по умолчанию влияет на название cmdlet, но вы можете изменить его на что-то более дружелюбное. Измените `Write Hello World to output`его на .
1. Щелкните **Параметры**, чтобы указать значения параметров командлета.

   Некоторые cmdlets имеют несколько наборов параметров, и вам нужно выбрать, какой из них использовать. В этом `Write-Output` случае, имеет только один набор параметров.

1. Выберите `InputObject` параметр. Это параметр, который используется для указания текста для отправки в поток вывода.
1. Меню выпадения **исходного кода данных** предоставляет источники, которые можно использовать для заполнения значения параметра. В этом меню выберите **выражение PowerShell.** 

   Выход из таких источников можно использовать в качестве другого действия, актива автоматизации или выражения PowerShell. В этом случае выход `Hello World`просто . Для этого воспользуйтесь выражением PowerShell и укажите строку.<br>

1. В поле **выражения** `Hello World` введите, а затем щелкните **OK** дважды, чтобы вернуться на холст.
1. Сохраните модуль Runbook, щелкнув **Сохранить**.

## <a name="step-3---test-the-runbook"></a>Шаг 3. Тестирование модуля Runbook

Перед публикацией runbook, чтобы сделать его доступным в производстве, вы должны проверить его, чтобы убедиться, что он работает должным образом. Тестирование runbook запускает свою версию проекта и позволяет просматривать его вывод в интерактивном режиме.

1. Выберите **тестовое стекло,** чтобы открыть тестовое стекло.
1. Щелкните **Пуск** , чтобы начать тестирование. Активным должен быть только этот параметр.
1. Обратите внимание, что [создается задание runbook](automation-runbook-execution.md) и отображается его статус в панели.

   Статус задания `Queued`начинается как, указывая, что задание ждет сотрудника runbook в облаке, чтобы стать доступным. Статус меняется `Starting` на время выполнения требования работника. Наконец, статус `Running` становится, когда запуск фактически начинает работать.

1. Когда задание runbook завершается, панель test отображает его выход. В этом случае, `Hello World`вы видите .

    ![Hello World](media/automation-first-runbook-graphical/runbook-test-results.png)
1. Закройте область тестирования, чтобы вернуться на холст.

## <a name="step-4---publish-and-start-the-runbook"></a>Шаг 4. Публикация и запуск модуля Runbook

Созданная книга по разработке по-прежнему находится в режиме Draft. Прежде чем запустить модуль в рабочей среде, его нужно опубликовать. При публикации модуля Runbook существующая опубликованная версия перезаписывается черновой версией. В нашем случае опубликованной версии не существует, поскольку модуль runbook создан только что.

1. Выберите **Опубликовать**, чтобы опубликовать runbook, и нажмите кнопку **Да** в появившемся запросе.
1. Прокрутите влево, чтобы просмотреть книгу на странице Runbooks, и обратите внимание, что значение **статуса автора** настроено на **опубликовано.**
1. Прокрутите экран вправо до страницы **MyFirstRunbook-Graphical**.

   Параметры в верхней части позволяют начать runbook сейчас, запланировать будущее время начала, или создать [веб-крюк,](automation-webhooks.md) так что runbook может быть запущен через http вызова.

1. Чтобы запустить runbook, выберите **Запуск** и нажмите кнопку **Да** в появившемся запросе.
1. Панель вакансий вакансии открыта для созданного задания runbook. Проверить, что поле **состояния вакансии** отображает **завершено.**
1. Нажмите **на выходе,** чтобы открыть `Hello World` страницу Вывода, где вы можете увидеть отображение.
1. Закройте страницу выходных данных.
1. Щелкните **Все журналы** , чтобы открыть область "Потоки" для задания Runbook. Вы должны `Hello World` видеть только в потоке вывода. 

    Обратите внимание, что панель потоков может отображать другие потоки для выполнения задания, такие как Verbose и Ошибка потоков, если runbook пишет им.
1. Закройте панель потоков и панель вакансий, чтобы вернуться на страницу MyFirstRunbook-Graphical.
1. Чтобы просмотреть все задания для runbook, выберите **Вакансии** под **ресурсами.** Страница Вакансии перечисляет все задания, созданные в вашем runbook. Вы должны увидеть только одну работу в списке, так как вы только запустить работу один раз.
1. Нажмите на имя задания, чтобы открыть ту же панель вакансий, что вы просматривали при запуске. Используйте эту панель для просмотра деталей любого задания, созданного для runbook.

## <a name="step-5---create-variable-assets"></a>Шаг 5. Создание ресурсов для переменных

Вы протестировали и опубликовали свою книгу запуска, но пока он не делает ничего полезного для управления ресурсами Azure. Перед настройкой запуска для проверки подлинности необходимо создать переменную для удержания идентификатора подписки, настроить действие для проверки подлинности, а затем ссылку на переменную. Включение ссылки на контекст подписки позволяет легко работать с несколькими подписками.

1. Копируйте идентификатор подписки из **опции Подписки** на панели навигации.
1. На странице учетных **записей** автоматизации выберите переменные под **общими ресурсами.**
1. Выберите **Добавить переменную**.
1. На новой переменной страницы сделайте следующие параметры в предоставленных полях.

    * **Имя** - `AzureSubscriptionId`введите .
    * **Значение** - введите идентификатор подписки. 
    * **Тип** - держите строку выбранной.
    * **Шифрование** - используйте значение по умолчанию.
1. Щелкните **Создать** , чтобы создать переменную.

## <a name="step-6---add-authentication"></a>Шаг 6 - Добавить аутентификацию

Теперь, когда у вас есть переменная для хранения идентификатора подписки, вы можете настроить runbook для проверки подлинности с помощью учетных данных Run As для вашей подписки. Сделайте это, добавив подключение Azure Run As как актив. Вы также должны добавить [cmdlet Connect-AzAccount](https://docs.microsoft.com/powershell/module/az.accounts/Connect-AzAccount?view=azps-3.5.0) и cmdlet [Set-AzContext](https://docs.microsoft.com/powershell/module/az.accounts/Set-AzContext?view=azps-3.5.0) к холсту.

>[!NOTE]
>Для PowerShell runbooks, `Add-AzAccount` и `Add-AzureRMAccount` псевдонимы для `Connect-AzAccount`. Обратите внимание, что эти псевдонимы не доступны для графических runbooks. Графический runbook может `Connect-AzAccount`использовать только себя.

1. Перейдите к своему runbook и выберите **Edit** на странице MyFirstRunbook-Graphical.
1. Вам больше не `Write Hello World to output` нужна запись. Просто нажмите эллипсис и выберите **Удалить**.
1. В управлении библиотекой расширьте **ASSETS**, затем **соединения**. Добавьте `AzureRunAsConnection` к холсту, выбрав **Добавить в холст.**
1. Переименуйте `AzureRunAsConnection` в `Get Run As Connection`.
1. В управлении библиотекой введите `Connect-AzAccount` в поле поиска.
1. Добавить `Connect-AzAccount` в холст.
1. Наведите `Get Run As Connection` круг до тех пор, пока круг не появится в нижней части формы. Нажмите на круг и `Connect-AzAccount` перетащите стрелку, чтобы сформировать ссылку. Runbook начинается `Get Run As Connection` с, `Connect-AzAccount`а затем работает .<br> ![Создание связи между действиями](media/automation-first-runbook-graphical/runbook-link-auth-activities.png)
1. На холсте `Connect-AzAccount`выберите . В панели управления конфигурацией введите **Login to Azure** в поле **меток.**
1. Нажмите **Параметры,** и появляется страница конфигурации параметра активности.
1. Cmdlet `Connect-AzAccount` имеет несколько наборов параметров, и вам нужно выбрать один, прежде чем предоставить значения параметров. Нажмите **Параметр Набор,** а затем выберите **ServicePrincipalCertificateWithSubscriptionId**.
1. Параметры этого набора параметров отображаются на странице конфигурации параметров активности. Щелкните **APPLICATIONID**.<br> ![Добавление параметров учетной записи Azure](media/automation-first-runbook-graphical/Add-AzureRmAccount-params.png)
1. На странице значение параметра сделайте следующие настройки, а затем нажмите **OK**.

   * **Источник данных** - выберите **выход активности**.
   * Список источников данных - выберите **Get Automation Connection.**
   * **Полевая дорожка** - тип `ApplicationId`. Вы указываете имя свойства для пути поля, поскольку действие выводит объект с несколькими свойствами.

1. Нажмите **CERTIFICATETHUMBPRINT**, и на странице значение параметра, сделать следующие настройки, а затем нажмите **OK**.

    * **Источник данных** - выберите **выход активности**.
    * Список источников данных - выберите **Get Automation Connection.**
    * **Полевая дорожка** - тип `CertificateThumbprint`.
1. Нажмите **SERVICEPRINCIPAL**, и на странице значения параметра, выберите **ConstantValue** для поля **исходного кода данных;** нажмите на опцию **Правда**; , а затем нажмите **OK**.
1. Нажмите **TENANTID**и сделайте следующие настройки на странице значения параметра. После завершения, нажмите **OK** дважды.

    * **Источник данных** - выберите **выход активности**. 
    * Список источников данных - выберите **Get Automation Connection.**
    * **Полевая дорожка** - тип `TenantId`. 
1. В управлении библиотекой введите `Set-AzContext` в поле поиска.
1. Добавить `Set-AzContext` в холст.
1. Выберите `Set-AzContext` на холсте. В панели управления конфигурацией `Specify Subscription Id` введите поле **Label.**
1. Щелкните **Параметры**, чтобы открыть страницу "Конфигурация параметров действия".
1. Cmdlet `Set-AzContext` имеет несколько наборов параметров, и вам нужно выбрать один, прежде чем предоставить значения параметров. Нажмите **Параметр Набор,** а затем выберите **SubscriptionId**.
1. Параметры этого набора параметров отображаются на странице конфигурации параметров активности. Нажмите **Подписка**ID .
1. На странице значения параметра выберите **переменный актив** для поля **исходного кода данных** и выберите **AzureSubscriptionId** из списка исходных источников. После завершения, нажмите **OK** дважды.
1. Наведите `Login to Azure` круг до тех пор, пока круг не появится в нижней части формы. Нажмите на круг и `Specify Subscription Id`перетащите стрелку. Ваш runbook должен выглядеть следующим на данный момент.

## <a name="step-7---add-activity-to-start-a-virtual-machine"></a>Шаг 7. Добавление действия для запуска виртуальной машины

Теперь вы `Start-AzVM` должны добавить действие, чтобы запустить виртуальную машину. Вы можете выбрать любой VM в подписке Azure, и сейчас вы жестко кодируете его имя в [cmdlet Start-AzVM.](https://docs.microsoft.com/powershell/module/az.compute/start-azvm?view=azps-3.5.0)

1. В управлении библиотекой введите `Start-Az` в поле поиска.
2. Добавить `Start-AzVM` в холст, а затем `Specify Subscription Id`нажмите и перетащите его под .
1. Наведите `Specify Subscription Id` круг до тех пор, пока круг не появится в нижней части формы. Нажмите на круг и `Start-AzVM`перетащите стрелку.
1. Выберите `Start-AzVM`. Нажмите **Параметры,** а затем **параметр Установить** для просмотра наборов для деятельности.
1. Выберите **ResourceGroupNameParameterSetName** для набора параметров. Поля **ResourceGroupName** и **Name** имеют рядом с ними восклицательные знаки, указывающие на то, что они являются необходимыми параметрами. Обратите внимание, что оба поля ожидают значения строки.
1. Выберите **имя**. Выберите **выражение PowerShell** для поля **исхода данных.** Для VM, который используется для запуска этого runbook, введите имя машины, окруженное двойными цитатами. Нажмите кнопку **ОК**.
1. Выберите **ResourceGroupName**. Используйте выражение значения **PowerShell** для поля **исходного кода данных** и введите имя группы ресурсов, окруженной двойными котировками. Нажмите кнопку **ОК**.
1. Щелкните **Тестовая область**, чтобы проверить модуль runbook.
1. Нажмите **Начало,** чтобы начать тест. Как только он завершается, убедитесь, что VM начал. Ваш runbook должен выглядеть следующим на данный момент.

    ![Настройка проверки подлинности Runbook](media/automation-first-runbook-graphical/runbook-startvm.png)

## <a name="step-8---add-additional-input-parameters"></a>Шаг 8 - Добавить дополнительные параметры ввода

В настоящее время в вашем runbook запускается `Start-AzVM` VM в группе ресурсов, указанной для cmdlet. Runbook будет более полезным, если вы укажете как имя, так и группу ресурсов при запуске. Давайте добавим параметры ввода в книгу, чтобы обеспечить эту функциональность.

1. Откройте графический редактор, нажав на **страницу** MyFirstRunbook-Graphical.
1. Выберите **Входные и выходные данные**, а затем выберите **Добавить входные данные**, чтобы открыть область входных параметров для runbook.
1. Сделайте следующие настройки в предоставленных полях, а затем нажмите **OK**.
   * **Имя** -- `VMName`укажите .
   * **Тип** - сохранить настройки строки.
   * **Обязательно** - изменить значение **Да**.
1. Создайте второй обязательный `ResourceGroupName` параметр ввода, а затем нажмите **OK,** чтобы закрыть входное и выходное стекло.<br> ![Входные параметры Runbook](media/automation-first-runbook-graphical/start-azurermvm-params-outputs.png)
1. Выберите `Start-AzVM` действие, а затем нажмите **Параметры**.
1. Измените поле **исходного кода данных** для ввода **имен** **в Runbook.** Затем выберите **VMName**.
1. Измените поле **исходного кода данных** для **Ввода ResourceGroupName** на **ввод Runbook,** а затем выберите **ResourceGroupName.**<br> ![Параметры Start-AzVM](media/automation-first-runbook-graphical/start-azurermvm-params-runbookinput.png)
1. Сохраните модуль Runbook и откройте область тестирования. Теперь вы можете указать значения двух входных переменных, которые будут использоваться в тесте.
1. Закройте область тестирования.
1. Щелкните **Опубликовать** , чтобы опубликовать новую версию модуля Runbook.
1. Остановите VM, который вы начали ранее.
1. Щелкните **Пуск** , чтобы запустить модуль Runbook. Введите значения `VMName` для `ResourceGroupName` Идля ВМ, которые вы собираетесь начать.
1. Когда запуск завершается, убедитесь, что VM был запущен.

## <a name="step-9---create-a-conditional-link"></a>Шаг 9. Создание условной связи

Теперь можно изменить runbook так, чтобы он только пытается запустить VM, если он еще не запущен. Сделайте это, добавив cmdlet [Get-AzVM,](https://docs.microsoft.com/powershell/module/Az.Compute/Get-AzVM?view=azps-3.5.0) который получает статус уровня экземпляра VM. Затем можно добавить модуль кода PowerShell Workflow, вызванный `Get Status` фрагментом кода PowerShell, чтобы определить, работает или останавливается состояние VM. Условная ссылка из модуля `Get Status` выполняется `Start-AzVM` только при прекращении текущего состояния. В конце этой процедуры, ваш `Write-Output` runbook использует cmdlet для вывода сообщения, чтобы сообщить вам, если VM был успешно запущен.

1. Откройте **MyFirstRunbook-Graphical** в графическом редакторе.
1. Удалите `Specify Subscription Id` связь `Start-AzVM` между и, нажав на него, а затем нажав **Удалить**.
1. В управлении библиотекой введите `Get-Az` в поле поиска.
1. Добавить `Get-AzVM` в холст.
1. Выберите, `Get-AzVM` а затем нажмите **Параметр Набор** для просмотра наборов для cmdlet. 
1. Выберите набор параметров **GetVirtualMachineInResourceGroupNameParamSet** . Поля **ResourceGroupName** и **Name** имеют рядом с ними восклицательные знаки, указывающие на то, что они указывают необходимые параметры. Обратите внимание, что оба поля ожидают значения строки.
1. Под **источником данных** для **имени**, выберите **ввод Runbook,** затем **VMName**. Нажмите кнопку **ОК**.
1. Под **источником данных** для **ResourceGroupName**, выберите **ввод Runbook**, затем **ResourceGroupName**. Нажмите кнопку **ОК**.
1. Под **источником данных** для **состояния**, выберите **значение Постоянного**, затем **True**. Нажмите кнопку **ОК**.
1. Создайте ссылку от `Specify Subscription Id` `Get-AzVM`.
1. В управлении библиотекой расширьте **управление Runbook** и добавьте **Код** к холсту.  
1. Создайте ссылку от `Get-AzVM` `Code`.  
1. Нажмите `Code` и, в панели конфигурации, изменить метку, чтобы **получить статус**.
1. Выберите `Code` и появится страница редактора кода.  
1. Вставьте следующий фрагмент кода на страницу редактора.

    ```powershell-interactive
     $StatusesJson = $ActivityOutput['Get-AzVM'].StatusesText
     $Statuses = ConvertFrom-Json $StatusesJson
     $StatusOut =""
     foreach ($Status in $Statuses){
     if($Status.Code -eq "Powerstate/running"){$StatusOut = "running"}
     elseif ($Status.Code -eq "Powerstate/deallocated") {$StatusOut = "stopped"}
     }
     $StatusOut
     ```

1. Создайте ссылку от `Get Status` `Start-AzVM`.

    ![Runbook с модулем кода](media/automation-first-runbook-graphical/runbook-startvm-get-status.png)  
1. Выберите ссылку и в панели конфигурации измените **условие применить** к **Да**. Обратите внимание, что ссылка становится пунктирной строкой, указывая, что целевое действие выполняется только в том случае, если условие соответствует действительности.  
1. Для **выражения состояния,** введите `$ActivityOutput['Get Status'] -eq "Stopped"`. `Start-AzVM`теперь работает только в том случае, если VM остановлен.
1. В элементе управления "Библиотека" разверните узел **Командлеты** и выберите **Microsoft.PowerShell.Utility**.
1. Добавить `Write-Output` в холст дважды.
1. Для `Write-Output` первого элемента управления щелкните **параметры** и измените значение **меток,** чтобы **уведомить VM Started.**
1. Для **InputObject**измените **источник данных** на выражение `$VMName successfully started.` **PowerShell**и введите выражение.
1. На втором `Write-Output` элементе управления щелкните **параметры** и измените значение **меток,** чтобы **уведомить VM Start Failed.**
1. Для **InputObject**измените **источник данных** на выражение `$VMName could not start` **PowerShell**и введите выражение.
1. Создание ссылок `Notify VM Started` `Notify VM Start Failed`от `Start-AzVM` к и .
1. Выберите ссылку `Notify VM Started` и измените **условие Применить** к истине.
1. Для **выражения условия,** введите `$ActivityOutput['Start-AzVM'].IsSuccessStatusCode -eq $true`. Этот `Write-Output` элемент управления теперь работает только в том случае, если VM успешно запускается.
1. Выберите ссылку `Notify VM Start Failed` и измените **условие Применить** к истине.
1. Для поля **выражения** состояния `$ActivityOutput['Start-AzVM'].IsSuccessStatusCode -ne $true`введите. Этот `Write-Output` элемент управления теперь работает только в том случае, если VM не запущен успешно. Ваша книга должна выглядеть следующим изображением.

    ![Runbook с командлетом Write-Output](media/automation-first-runbook-graphical/runbook-startazurermvm-complete.png)
1. Сохраните модуль Runbook и откройте область тестирования.
1. Запустите runbook с VM остановлен, и машина должна начаться.

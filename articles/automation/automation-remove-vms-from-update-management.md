# <a name="remove-vms-from-update-management"></a>Исключение виртуальных машин из Управления обновлениями

После завершения развертывания обновлений на виртуальных машинах в вашей среде их можно удалить из компонента [Управление обновлениями](automation-update-management.md).

## <a name="to-remove-your-vms"></a>Удаление виртуальных машин

1. В учетной записи службы автоматизации в области **Управление обновлениями** выберите **Управление обновлениями**.

2. Используйте следующую команду, чтобы указать UUID компьютера, который требуется удалить из управления.

    ```azurecli
    az vm show -g MyResourceGroup -n MyVm -d
    ```

3. В рабочей области Log Analytics в разделе **Общие**обратитесь к сохраненным поисковым запросам конфигурации области `MicrosoftDefaultScopeConfig-Updates` .

4. Для сохраненного поискового запроса `MicrosoftDefaultComputerGroup` нажмите кнопку с многоточием справа и выберите **Изменить**.

5. Удалите данный UUID для виртуальной машины.

6. Повторите эти шаги для всех остальных виртуальных машин, которые нужно удалить.

7. Завершив редактирование сохраненного поискового запроса, снова сохраните его.

>[!NOTE]
>Компьютеры по-прежнему отображаются после отмены регистрации, так как мы получаем отчет по всем компьютерам, оцененным за последние 24 часа. После отключения компьютера необходимо подождать 24 часа, прежде чем они перестанут отображаться в списке.

# <a name="use-dynamic-groups-with-update-management"></a>Использование динамических групп с Управлением обновлениями

Управление обновлениями позволяет ориентироваться на динамическую группу Azure или виртуальных машин без Azure для развертываний обновлений. Эти группы, определяемые запросами, оцениваются во время развертывания, поэтому вам не нужно изменять развертывание для добавления компьютеров.

## <a name="azure-machines"></a>Компьютеры Azure

Динамические группы не работают с классическими виртуальными машинами. При определении запроса следующие элементы можно использовать вместе для заполнения динамической группы:

* Подписка
* Группы ресурсов
* Расположения
* Теги

![Выбор групп](./media/automation-update-management/select-groups.png)

Чтобы просмотреть результаты динамической группы, нажмите кнопку **Предварительный просмотр**. В предварительной версии отображается членство в группе в текущее время. В этом примере выполняется поиск компьютеров, у которых есть тег `Role` для группы **баккендсервер**. Если добавлен этот тег для большего числа компьютеров, они будут добавлены в любые будущие развертывания этой группы.

![просмотр групп](./media/automation-update-management/preview-groups.png)

## <a name="non-azure-machines"></a>Компьютеры, не относящиеся к Azure

Для компьютеров, не относящихся к Azure, сохраненные поисковые запросы, также называемые группами компьютеров, используются для создания динамической группы. Сведения о создании сохраненного поиска см. в разделе [Создание группы компьютеров](../azure-monitor/platform/computer-groups.md#creating-a-computer-group). После создания группы ее можно выбрать в списке сохраненных поисков. Щелкните **Предварительный просмотр** , чтобы просмотреть компьютеры в сохраненном поиске в это время.

![Выбор групп](./media/automation-update-management/select-groups-2.png)

# <a name="use-dynamic-groups-with-update-management"></a>Использование динамических групп с Управлением обновлениями

Управление обновлениями позволяет настроить таргетинг на динамическую группу VMs Azure или не Azure для развертывания. Эти группы, определяемые запросами, оцениваются во время развертывания, так что вам не придется отсеивать развертывание для добавления машин.

## <a name="azure-machines"></a>Лазурные машины

Динамические группы не работают с классическими VM. При определении запроса для заполнения динамической группы можно использовать следующие элементы:

* Подписка
* Группы ресурсов
* Расположения
* Теги

![Выбор групп](./media/automation-update-management/select-groups.png)

Чтобы просмотреть результаты динамической группы, нажмите **Предварительный просмотр**. Предварительный просмотр показывает членство в группе в текущем времени. В этом примере мы ищем машины, имеющие тег `Role` для группы **BackendServer.** Если к этой группе добавлено больше машин, они добавляются к любым будущим развертываниям против этой группы.

![просмотр групп](./media/automation-update-management/preview-groups.png)

## <a name="non-azure-machines"></a>Машины, не являемые лазурными средствами

Для машин, не относясь от Azure, для создания динамической группы используются сохраненные поисковые системы, также называемые компьютерными группами. Чтобы узнать, как создать сохраненный поиск, [см.](../azure-monitor/platform/computer-groups.md#creating-a-computer-group) После создания группы можно выбрать ее из списка сохраненных поисков. Нажмите **Предварительный просмотр** компьютеров в сохраненном поиске в это время.

![Выбор групп](./media/automation-update-management/select-groups-2.png)

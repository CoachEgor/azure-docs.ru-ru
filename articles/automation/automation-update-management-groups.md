# <a name="use-dynamic-groups-with-update-management"></a>Использование динамических групп с Управлением обновлениями

Управление обновлениями позволяет настроить таргетинг на динамическую группу VMs Azure или не Azure для развертывания. Эти группы оцениваются во время развертывания, поэтому вам не придется отсеивать развертывание для добавления машин.

## <a name="azure-machines"></a>Лазурные машины

Эти группы определяются по запросу. Когда начинается развертывание обновлений, оцениваются участники этой группы. Динамические группы не работают с классическими VM. При определении запроса для заполнения динамической группы можно использовать следующие элементы:

* Подписка
* Группы ресурсов
* Расположения
* Теги

![Выбор групп](./media/automation-update-management/select-groups.png)

Чтобы просмотреть результаты в динамической группе, нажмите кнопку **Preview** (Просмотр). В этом представлении для предварительного просмотра показан состав группы на текущий момент. В этом примере мы выполняем поиск компьютеров с тегом **Роль** со значением **BackendServer**. Если добавить этот тег большему количеству компьютеров, они будут добавлены во все будущие развертывания данной группы.

![просмотр групп](./media/automation-update-management/preview-groups.png)

## <a name="non-azure-machines"></a>Машины, не являемые лазурными средствами

Для машин, не относясь от Azure, сохраненные поиски, также называемые компьютерными группами, используются для создания динамической группы. Чтобы узнать, как создать сохраненный поиск, [см.](../azure-monitor/platform/computer-groups.md#creating-a-computer-group) После создания группы вы можете выбрать ее из списка сохраненных поисков. Нажмите **Предварительный просмотр** компьютеров в сохраненном поиске в это время.

![Выбор групп](./media/automation-update-management/select-groups-2.png)

# <a name="work-with-scope-configurations-for-update-management"></a>Работа с конфигурациями областей для Управления обновлениями

В этой статье рассказывается о работе с конфигурациями областей при использовании компонента [Управление обновлениями](automation-update-management.md) для виртуальных машин. 

## <a name="sign-in-to-azure"></a>Вход в Azure

Войдите на портал Azure по адресу https://portal.azure.com.

## <a name="check-the-scope-configuration"></a><a name="scope-configuration"></a>Проверка конфигурации области

В Управлении обновлениями используется конфигурация области в пределах рабочей области Log Analytics для определения целевых компьютеров, на которых будет включено управление. Конфигурация области представляет собой группу из одного или нескольких сохраненных поисковых запросов, которая используется для ограничения области применения функции к конкретным компьютерам. Для доступа к конфигурациям областей выполните указанные ниже действия.

1. В учетной записи службы автоматизации в разделе **Связанные ресурсы** выберите **Рабочая область**. 

2. В разделе **Источники данных рабочей области** выберите рабочую область, а затем выберите **Конфигурации области**.

3. Если в выбранной рабочей области еще нет решения "Управление обновлениями", создается конфигурация области `MicrosoftDefaultScopeConfig-Updates`. 

4. Если в выбранной рабочей области уже включено это решение, то оно не развертывается повторно и конфигурация области не добавляется. 

5. Нажмите кнопку с многоточием для любой из конфигураций областей и выберите **Изменить**. 

6. В области изменения щелкните **Выбор группы компьютеров**. В области "Группы компьютеров" отображаются сохраненные поисковые запросы, которые используются для создания конфигурации области.

## <a name="view-a-saved-search"></a>Просмотр сохраненного поиска

При добавлении компьютера в Управление обновлениями он также добавляется в сохраненный поисковый запрос в рабочей области. Сохраненный поисковый запрос — это запрос, который содержит конечные компьютеры.

1. Перейдите к своей рабочей области Log Analytics и в разделе **Общие** выберите **Сохраненные поисковые запросы**. Сохраненный поисковый запрос, используемый Управлением обновлениями:

|Имя     |Категория  |Псевдоним  |
|---------|---------|---------|
|MicrosoftDefaultComputerGroup     | Обновления        | Updates__MicrosoftDefaultComputerGroup         |

2. Выберите сохраненный поисковый запрос, чтобы просмотреть запрос, используемый для заполнения группы. На следующем рисунке показан запрос и его результаты:

# <a name="memory-and-concurrency-limits-for-azure-sql-data-warehouse"></a>Ограничения параллелизма и памяти для хранилища данных SQL Azure
Из этой статьи вы узнаете об ограничениях памяти и параллелизма, существующих для различных уровней производительности и классов ресурсов в хранилище данных SQL Azure. Дополнительные сведения об этих ограничениях, а также сведения о применении этих возможностей в вашем плане управления рабочими нагрузками см. в статье [Классы ресурсов для управления рабочими нагрузками](resource-classes-for-workload-management.md). 

В настоящее время доступны два поколения хранилища данных SQL — Gen1 и Gen2. Для обеспечения высокой производительности рабочей нагрузки хранилища данных рекомендуется использовать уровень Gen2 хранилища данных SQL. На уровне Gen2 появился новый кэш NVMe SSD для хранения наиболее часто используемых данных рядом с ЦП. Это позволяет отказаться от удаленных операций ввода-вывода для самых ресурсоемких и требовательных к производительности рабочих нагрузок. Наряду с ростом производительности Gen2 предлагает высочайший уровень масштабирования — до 30 000 единиц хранилища данных — и предоставляет хранилище столбцов неограниченного размера. Мы по-прежнему будем поддерживать предыдущее поколение (Gen1) хранилища данных SQL и сохраним те же функции. Однако мы рекомендуем как можно скорее выполнить [обновление до Gen2](upgrade-to-latest-generation.md).

## <a name="concurrency-maximums"></a>Максимальные ограничения параллелизма
Чтобы каждый запрос имел достаточно ресурсов для эффективной работы, хранилище данных SQL отслеживает использование ресурсов путем присвоения каждому запросу определенного количества слотов параллельности. Система помещает запросы в очередь на основе важности и слотов выдачи. Запросы ожидать в очереди, пока достаточное число слотов доступны. [Важность](https://docs.microsoft.com/azure/sql-data-warehouse/sql-data-warehouse-workload-importance) и слотов определяют приоритеты ЦП. Дополнительные сведения см. в статье [Анализ рабочей нагрузки](analyze-your-workload.md).

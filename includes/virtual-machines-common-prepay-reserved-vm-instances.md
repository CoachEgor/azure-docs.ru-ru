При фиксации зарезервированного экземпляра VM Azure вы можете сэкономить деньги. Скидка на резервирование автоматически применяется к количеству работающих виртуальных машин, которые соответствуют области и атрибутам резервирования. Вам не нужно присваивать бронирование виртуальной машине, чтобы получить скидки. Зарезервированная покупка экземпляра охватывает только вычислительную часть вашего использования VM. Для Windows VMs счетчик использования делится на два отдельных метра. Есть счетчик вычислений, который такой же, как измеритель Linux, и IP-метр Windows. Расходы, которые вы видите, когда вы делаете покупку только для вычислений. Сборы не включают расходы на программное обеспечение Windows. Для получения дополнительной информации о стоимости [программного обеспечения](../articles/cost-management-billing/reservations/reserved-instance-windows-software-costs.md)см.

## <a name="determine-the-right-vm-size-before-you-buy"></a>Определение нужного размера виртуальной машины перед покупкой

Прежде чем купить бронирование, вы должны определить размер VM, что вам нужно. Следующие разделы помогут вам определить правильный размер VM.

### <a name="use-reservation-recommendations"></a>Использование рекомендаций по бронированию

Вы можете использовать рекомендации по бронированию, чтобы определить бронирование, которые вы должны приобрести.

- Рекомендации по покупке и рекомендуемое количество отображаются при покупке зарезервированного экземпляра VM на портале Azure.
- Azure Advisor предоставляет рекомендации по закупкам для отдельных подписок.  
- Можно использовать AA, чтобы получить рекомендации по покупке как для общей сферы, так и для единой сферы подписки. Для получения дополнительной [Reserved instance purchase recommendation APIs for enterprise customers](/rest/api/billing/enterprise/billing-enterprise-api-reserved-instance-recommendation)информации см.
- Для клиентов Enterprise Agreement (EA) и Microsoft Customer Agreement (MCA) рекомендации по закупкам для общих и одной сфер подписки доступны с [пакетом контента Azure Consumption Insights Power BI.](/power-bi/service-connect-to-azure-consumption-insights)

### <a name="services-that-get-vm-reservation-discounts"></a>Службы, которые получают скидки на резервирование виртуальных машин

Резервирование виртуальных машин может применяться к использованию виртуальной машины, созданной из нескольких служб, а не только для развертываний виртуальных машин. Ресурсы, которые получают скидки на резервирование, изменяются в зависимости от параметра гибкости размера экземпляра.

#### <a name="instance-size-flexibility-setting"></a>Параметр гибкости размера экземпляра

Параметр гибкость размера экземпляра определяет, какие службы получают скидки на зарезервированные экземпляры.

Независимо от того, включен ли параметр или выключен, скидки на резервирование автоматически применяются к любому соответствующему использованию виртуальной машины, если параметр *ConsumedService* равен `Microsoft.Compute`. Поэтому проверьте данные об использовании для значения *ConsumedService*. Некоторые примеры:

- Виртуальные машины
- Масштабируемые наборы виртуальных машин
- Служба контейнеров
- Развертывания пакетной службы Azure (в режиме пользовательских подписок)
- Служба Azure Kubernetes (AKS)
- Service Fabric

Если эта настройка включена, скидки на резервирование автоматически применяются к соответствующему использованию виртуальных машин, когда *ConsumedService* является одним из следующих элементов:

- Microsoft.Compute;
- Microsoft.ClassicCompute
- Microsoft.Batch
- Microsoft.MachineLearningServices
- Microsoft.Kusto

Проверьте значение *ConsumedService* в данных об использовании, чтобы определить, подходит ли использование для скидки на резервирование.

Дополнительные сведения о гибкости размера экземпляров см. в статье [Гибкость размеров виртуальных машин при использовании зарезервированных экземпляров виртуальных машин](../articles/virtual-machines/windows/reserved-vm-instance-size-flexibility.md).

### <a name="analyze-your-usage-information"></a>Проанализируйте информацию об использовании

Проанализируйте информацию об использовании, чтобы определить, какие бронирования вы должны приобрести. Данные об использовании доступны в файле использования и AIS. Используйте их вместе, чтобы определить, какое бронирование купить. Проверьте для VM экземпляров, которые имеют высокое использование на ежедневной основе, чтобы определить количество бронирований на покупку. Избегайте `Meter` подкатегории и `Product` полей в данных об использовании. Они не различают размеры VM, которые используют премиум-хранилище. Если вы используете эти поля для определения размера VM для покупки бронирования, вы можете купить неправильный размер. Тогда вы не получите скидку на бронирование, который вы ожидаете. Вместо этого обратитесь к `AdditionalInfo` области в файле использования или API использования, чтобы определить правильный размер VM.

Файл использования показывает ваши расходы по периоду выставления счетов и ежедневному использованию. Для получения информации о загрузке файла использования, [просмотреть и скачать использование Azure и сборов](../articles/cost-management-billing/understand/download-azure-daily-usage.md). Затем, используя информацию об использовании файла, вы можете [определить, какое бронирование купить.](../articles/cost-management-billing/reservations/determine-reservation-purchase.md)

### <a name="purchase-restriction-considerations"></a>Соображения ограничения на покупку

Зарезервированные VM Instances доступны для большинства размеров VM, за некоторыми исключениями. Скидки на бронирование не распространяются на следующие ВМ:

- **Серия VM** - A-серия, Av2-серия, или G-серии.

- **Предварительный просмотр или промо-вм -** Любой VM-серии или размер, который находится в предварительном просмотре или использует рекламный счетчик.

- **Облака** - Бронирование не доступно для покупки в Германии или Китае регионов.

- **Недостаточная квота** - резервирование, которое приковано к одной подписке, должно иметь квоту vCPU, доступную в подписке на новый RI. Например, если целевая подписка имеет квоту на 10 виртуальных ЦП для серии D, вы не сможете приобрести резервирование для 11 экземпляров Standard_D1. При проверке квоты для резервирования учитываются все виртуальные машины, уже размещенные в подписке. Например, если подписка имеет квоту на 10 виртуальных ЦП для серии D и два развернутых экземпляра Standard_D1, в этой подписке можно приобрести резервирование для 10 экземпляров Standard_D1. Для решения этой проблемы можно [создать запрос на увеличение котировок.](../articles/azure-portal/supportability/resource-manager-core-quotas-request.md)

- **Ограничения емкости** - В редких случаях Azure ограничивает покупку новых резерваций для поднабора размеров VM из-за низкой емкости в регионе.

## <a name="buy-a-reserved-vm-instance"></a>Приобретение Reserved VM Instance

Зарезервированный экземпляр VM можно купить на [портале Azure.](https://portal.azure.com/#blade/Microsoft_Azure_Reservations/CreateBlade/referrer/documentation/filters/%7B%22reservedResourceType%22%3A%22VirtualMachines%22%7D) Платите за резервирование [наперед или ежемесячными платежами](../articles/cost-management-billing/reservations/monthly-payments-reservations.md).
Эти требования применяются к покупке зарезервированного экземпляра VM:

- Вы должны быть в роли владельца, по крайней мере, одна подписка EA или подписка с оплатой по мере использования.
- Для подписок EA опция **Add Reserved Instances** должна быть включена на [портале EA.](https://ea.azure.com/) Если этот параметр отключен, необходимо быть администратором подписки EA.
- В рамках программы для поставщиков облачных решений (CSP) только агенты администрирования или агенты продаж могут приобрести резервирования.

Требования для приобретения экземпляра.

1. Войдите на [портал Azure](https://portal.azure.com).
1. Выберите **Все услуги** > **Бронирование**.
1. Выберите **Добавить,** чтобы купить новое бронирование, а затем нажмите **Виртуальная машина.**
1. Заполните обязательные поля. Работающие экземпляры виртуальной машины, соответствующие выбранным атрибутам, подходят для получения скидки на резервирование. Фактическое число экземпляров виртуальной машины, получающих скидку, зависит от области и выбранного количества.

Если у вас есть соглашение EA, можно использовать **опцию Добавить больше** для быстрого добавления дополнительных экземпляров. Опция недоступна для других типов подписки.

## <a name="change-a-reservation-after-purchase"></a>Изменение бронирования после покупки

После приобретения в резервирование можно внести следующие типы изменений:

- Обновить область резервирования
- Гибкость размера экземпляра (если это применимо)
- владельца;

Можно также разделить резервирование на более мелкие куски и объединить уже разделенные резервии. Ни одно из изменений не приводит к новой коммерческой транзакции или изменению даты окончания резервирования.

Вы не можете внести следующие типы изменений после покупки, непосредственно:

- Существующий регион резервации
- номер SKU
- Количество
- Duration

Тем не менее, вы можете *обменять* бронирование, если вы хотите внести изменения.

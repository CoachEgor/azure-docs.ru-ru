# <a name="prepay-for-virtual-machines-with-azure-reserved-vm-instances-ri"></a>Предоплата виртуальных машин Azure зарезервированных экземпляров виртуальных Машин (RI)

Вы можете воспользоваться предоплатой виртуальных машин и сэкономить при использовании Azure Reserved Virtual Machine Instances. Дополнительные сведения см. на странице [Azure Reserved VM Instances](https://azure.microsoft.com/pricing/reserved-vm-instances/).

Reserved VM Instance можно приобрести на [портале Azure](https://portal.azure.com). Требования для приобретения экземпляра.

- Необходимо быть участником роли "Владелец" по крайней мере для одной подписки с соглашением Enterprise или подписки с оплатой по мере использования.
- Для подписок с соглашением Enterprise параметр **Добавить зарезервированные экземпляры** следует включить на [портале EA](https://ea.azure.com). Или, если этот параметр отключен, необходимо быть администратором подписки EA.
- В рамках программы для поставщиков облачных решений (CSP) только агенты администрирования или агенты продаж могут приобрести резервирования.

Скидка на резервирование автоматически применяется к количеству работающих виртуальных машин, которые соответствуют области и атрибутам резервирования. Можно обновить область резервирования с помощью [портала Azure](https://portal.azure.com), PowerShell, CLI или API.

## <a name="determine-the-right-vm-size-before-you-buy"></a>Определение нужного размера виртуальной машины перед покупкой

Прежде чем вы покупаете резервирование, следует определить размер виртуальной Машины, которая может потребоваться. Следующие разделы помогут вам определить правильный размер виртуальной Машины.

### <a name="use-reservation-recommendations"></a>Рекомендации по использованию резервирования

Резервирование рекомендации можно использовать для определения, следует приобрести резервирование.

- Рекомендации для покупки и рекомендуемое количество показано, при покупке зарезервированного экземпляра виртуальной Машины на портале Azure.
- Помощник по Azure предоставляет рекомендации покупки для отдельных подписок.  
- API-интерфейсы можно использовать для получения рекомендаций покупки для общей области и областью одной подписки. Дополнительные сведения см. в разделе [зарезервировано рекомендация покупки экземпляр API-интерфейсы для корпоративных клиентов](/rest/api/billing/enterprise/billing-enterprise-api-reserved-instance-recommendation).
- Клиенты с соглашением Enterprise приобрести рекомендации для общих и одной подписке областей доступны с [пакет содержимого Azure Consumption Insights Power BI](/power-bi/service-connect-to-azure-consumption-insights).

### <a name="classic-vms-and-cloud-services"></a>Классических виртуальных машин и облачных служб

Зарезервированные экземпляры виртуальных машин автоматически применяются к обоих классических виртуальных машин и облачных служб при включении гибкость размер экземпляра. Отсутствуют специальные номера SKU для классических виртуальных машин или облачных служб. Номера SKU виртуальных Машин применимо также к их.

Например для виртуальных машин на основе Azure Resource Manager может преобразовать классических виртуальных машин или облачных служб. В этом примере скидка на резервирование автоматически применяется к сопоставления виртуальные машины. Нет необходимости в *exchange* зарезервированный экземпляр — он автоматически применяется.

### <a name="analyze-your-usage-information"></a>Анализ сведений об использовании
Следует проанализировать сведений об использовании можно определить, какие резервирования, следует приобрести.

Данные об использовании доступна в файле использования и API-интерфейсы. Их использование для определить, какие резервирования для приобретения. Необходимо проверить для экземпляров виртуальных Машин, имеющих высокий уровень использования на ежедневной основе, чтобы определить количество резервирований для приобретения.

Избегайте `Meter` подкатегории и `Product` поля в данных об использовании. Они не различают размеры виртуальных Машин, использующие хранилище уровня "премиум". Если вы используете эти поля для определения размера виртуальной Машины для покупки резервирования, можно приобрести неправильный размер. Затем вы не получите скидки на резервирование, в который вы ожидаете получить уведомление. Вместо этого обратитесь к `AdditionalInfo` поле в файле использования или использования API, чтобы определить правильный размер виртуальной Машины.

### <a name="purchase-restriction-considerations"></a>Приобрести вопросы ограниченного использования программ

Зарезервированные экземпляры виртуальных Машин доступны для большинства размеров виртуальных Машин за некоторыми исключениями. Скидки на резервирование не применяются для следующих виртуальных машин:

- **Виртуальные Машины серии** -серии A, серии Av2 или серии G.

- **Виртуальные машины на этапе предварительной версии** -все серии виртуальных Машин или размер, который находится в предварительной версии.

- **Облака** -резервирования недоступны для приобретения в регионах, Германии и Китая.

- **Недостаточная Квота** -резервирование, относящемся к одной подписке должна иметь свободную квоту, доступных в подписке для размещения этого зарезервированного Экземпляра. Например, если целевая подписка имеет квоту на 10 виртуальных ЦП для серии D, вы не сможете приобрести резервирование для 11 экземпляров Standard_D1. При проверке квоты для резервирования учитываются все виртуальные машины, уже размещенные в подписке. Например, если подписка имеет квоту на 10 виртуальных ЦП для серии D и два развернутых экземпляра Standard_D1, в этой подписке можно приобрести резервирование для 10 экземпляров Standard_D1. Вы можете [предложение создать запрос на увеличение](../articles/azure-supportability/resource-manager-core-quotas-request.md) для устранения этой проблемы.

- **Ограничения емкости** — в редких случаях Azure ограничения размеров приобретение новых резервирований для некоторых виртуальных Машин, из-за нехватки емкости в регионе.

## <a name="buy-a-reserved-vm-instance"></a>Приобретение Reserved VM Instance

1. Войдите на [портале Azure](https://portal.azure.com).
2. Выберите **Все службы** > **Резервирование**.
3. Выберите **Добавить**, чтобы купить новое резервирование.
4. Заполните обязательные поля. Работающие экземпляры виртуальной машины, соответствующие выбранным атрибутам, подходят для получения скидки на резервирование. Фактическое число экземпляров виртуальной машины, получающих скидку, зависит от области и выбранного количества.

    | Поле      | ОПИСАНИЕ|
    |------------|--------------|
    |ИМЯ        |Имя данного резервирования.|
    |Подписка|Подписка, используемая для оплаты резервирования. Резервирование оплачивается предварительно с помощью метода оплаты, указанного для подписки. Нужно выбрать подписку с соглашением Enterprise (номера предложений: MS-AZR-0017P или MS-AZR-0148P) или подписку с оплатой по мере использования (номера предложений: MS-AZR-0003P или MS-AZR 0023P). Для подписки с соглашением Enterprise плата вычитается из баланса денежных обязательств или относится к избыточным расходам. Для подписки с оплатой по мере использования плата взимается с помощью метода оплаты (кредитной карты или счета), указанного для подписки.|    
    |`Scope`       |Область резервирования может охватывать одну или несколько подписок (общая область). Если выбрать: <ul><li>"Для одной подписки" — скидка на резервирование применяется к виртуальным машинам в этой подписке. </li><li>"Общая" — скидка на резервирование применяется к виртуальным машинам, работающих в любых подписках в текущем контексте выставления счетов. Для клиентов с соглашением Enterprise общая область действует в рамках регистрации и включает в себя все подписки, заданные при регистрации. Для клиентов с подпиской с оплатой по мере использования общая область включает в себя все подписки с оплатой по мере использования, созданные администратором учетной записи.</li></ul>|
    |Регион    |Регион Azure, охватываемый резервированием.|    
    |Размер виртуальной машины     |Размер экземпляров виртуальной машины.|
    |Оптимизировать для     |Гибкость размеров для экземпляров виртуальных машин применяет скидку на резервирование для других виртуальных машин в той же [группе размеров виртуальных машин](https://aka.ms/RIVMGroups). Приоритет емкости назначает приоритет емкости центра обработки данных для развертываний. Это обеспечивает дополнительную уверенность в возможности запуска экземпляров виртуальных машин, когда они понадобятся. Приоритет емкости доступен только в том случае, если область резервирования является одиночной подпиской. |
    |Термин        |Один или три года.|
    |Количество    |Число экземпляров, приобретаемых в рамках резервирования. Это количество работающих экземпляров виртуальной машины, которые могут получить скидку. Например, если у вас работают 10 виртуальных машин Standard_D2 в регионе "Восточная часть США", следует указать количество, равное 10, чтобы получить максимальную выгоду для всех работающих виртуальных машин. |

## <a name="change-a-reservation-after-purchase"></a>Изменить резервирование после покупки

После приобретения резервирования можно внести следующие изменения:

- Обновить область резервирования
- Гибкость размер экземпляра (если применимо)
- Владение

Также можно разделить резервирование на небольших блоки и слияния уже разделено резервирования. Ни одно из изменений вызвать новую транзакцию коммерческих или изменить дату окончания резервирования.

После приобретения, непосредственно не может вносить следующие изменения:

- Существующее резервирование области
- SKU
- Количество
- Продолжительность

Тем не менее, вы можете *exchange* резервирования, если вы хотите внести изменения.

## <a name="cancellations-and-exchanges"></a>Отмена и обмен

Если необходимо отменить резервирование, возможно, с вас будет взиматься плата за досрочное аннулирование (12 %). Сумма возмещенных средств основывается на наименьшей стоимости, например на стоимости покупки или текущей стоимости резервирования. Максимальная сумма возмещенных средств — 50 000 долл. США в год. Полученные возмещенные средства включают оставшийся пропорциональный баланс за вычетом 12 % за досрочное аннулирование. Если нужно запросить отмену, перейдите к резервированию на портале Azure и выберите **Возместить**, чтобы создать запрос в службу поддержки.

Если в зарезервированных экземплярах виртуальных машин нужно изменить регион, группу размеров виртуальных машин или срок, можете обменять это резервирование на другое с такими же или большими значениями. При обмене дата начала использования для нового резервирования не переносится из текущего резервирования. Срок использования (1 или 3 года) начинается с момента создания резервирования. Если нужно запросить обмен, перейдите в резервирование на портале Azure и выберите **Обмен**, чтобы создать запрос в службу поддержки.

Дополнительные сведения о том, как exchange или возмещение резервирования см. в разделе [обмен резервирования и возмещения](../articles/billing/billing-azure-reservations-self-service-exchange-and-refund.md).

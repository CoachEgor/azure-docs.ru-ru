В следующей таблице описаны целевые показатели емкости, масштабируемости и производительности для хранения таблицы.

| Ресурс | Назначение |
|----------|---------------|
| Количество таблиц в учетной записи хранения Azure | Ограничено только емкостью учетной записи хранения |
| Количество разделов в таблице | Ограничено только емкостью учетной записи хранения |
| Количество сущностей в разделе | Ограничено только емкостью учетной записи хранения |
| Максимальный размер одной таблицы | 500 ТиБ |
| Максимальный размер одного объекта, включая все значения свойств | 1 МиБ |
| Максимальное количество свойств в объекте таблицы | 255 (включая три системных свойства, **PartitionKey**, **RowKey**и **Timestamp)** |
| Максимальный общий размер индивидуального имущества в юридическом лице | Изменяется по типу свойства. Для получения дополнительной информации **см. Типы свойств** в [понимании модели данных службы таблицы](/rest/api/storageservices/understanding-the-table-service-data-model). |
| Размер свойства **PartitionKey** | Строка размером до 1 киБ |
| Размер свойства **RowKey** | Строка размером до 1 киБ |
| Размер транзакции группы сущности | Транзакция может включать не более 100 сущностей, а полезная нагрузка должна быть меньше 4 МБ. Транзакция группы сущности может включать обновление сущности только один раз. |
| Максимальное количество сохраненных политик доступа в таблице | 5 |
| Максимальная частота запросов на учетную запись хранения | 20 000 транзакций в секунду, что предполагает размер 1-КИБ |
| Целевая пропускная система для раздела одной таблицы (1 КИБ-сущности) | До 2000 организаций в секунду |

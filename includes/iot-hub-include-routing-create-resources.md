## <a name="create-base-resources"></a>Создание базовых ресурсов

Перед настройкой маршрутизации сообщений необходимо создать центр Интернета вещей, учетную запись хранения и очередь служебной шины. Эти ресурсы можно создать с помощью одного из четырех средств, описанных в первой части этого руководства: портала Azure, шаблона Azure Resource Manager, Azure CLI или Azure PowerShell.

Используйте те же группу ресурсов и расположение для всех ресурсов. Затем в конце можно удалить все ресурсы за один шаг, удалив группу ресурсов.

Ниже перечислены действия, которые необходимо выполнить: 

1. Создайте [группу ресурсов](../articles/azure-resource-manager/resource-group-overview.md).

2. Создаете Центр Интернета вещей на уровне S1. Добавьте группы потребителей в Центр Интернета вещей. Azure Stream Analytics использует группу потребителей при получении данных.

   > [!NOTE]
   > Для работы с этим руководством необходимо использовать Центр Интернета вещей на платном уровне. Бесплатный уровень позволяет настраивать только одну конечную точку, а в этом руководстве требуется настроить множество.
   > 

3. Создайте стандартную учетная запись хранения V1 с репликацией Standard_LRS.

4. Создайте пространство имен и очередь служебной шины.

5. Создайте идентификатор устройства для имитированного устройства, которое отправляет сообщения концентратору. Сохраните ключ для этапа тестирования. (Если вы создаете шаблон Resource Manager, этот шаг нужно выполнить после развертывания шаблона.)

<sup>1</sup>конвейера, набора данных и связанной службы представляют логическую группировку вашей рабочей нагрузки. Ограничения для этих объектов не связаны с объемом данных можно перемещать и обрабатывать с помощью фабрики данных Azure. Фабрика данных позволяет масштабировать для обработки петабайтов данных.

<sup>2</sup> Ядра HDInsight по запросу выделяются из подписки, которая содержит фабрику данных. Таким образом предыдущее ограничение — это ограничение принудительно фабрики данных ядер для ядер HDInsight по запросу. Оно отличается от основного ограничения, который связан с вашей подпиской Azure.

<sup>3</sup>в ходе операции копирования в облако используется блок данных интеграции (DIU) для версии 2 или облачная единица перемещения данных (DMU) для версии 1. Это мера, представляющая производительность одной единицы в фабрике данных. Оно объединяет в себе Процессора, памяти и выделения ресурсов сети. Вы можете достичь более высокой пропускной способности при копировании, используя дополнительные единицы DMU в некоторых сценариях. Дополнительные сведения см. в разделе [единицы интеграции данных (версия 2)](../articles/data-factory/copy-activity-performance.md#data-integration-units) и [облачные единицы перемещения данных (версия 1)](../articles/data-factory/v1/data-factory-copy-activity-performance.md#cloud-data-movement-units). Сведения о выставлении счетов см. в разделе [цен фабрики данных Azure](https://azure.microsoft.com/pricing/details/data-factory/).

<sup>4</sup>среды выполнения интеграции (IR) — это инфраструктура вычислений, используемые фабрикой данных Azure для обеспечения интеграции данных в различных сетевых средах, таких как перемещение данных, диспетчеризации действий для служб, вычислений и выполнение пакетов служб SSIS. Дополнительные сведения см. в разделе [среды выполнения интеграции](../articles/data-factory/concepts-integration-runtime.md).

## <a name="business-disaster-recovery"></a>Непрерывность бизнес-процессов и аварийное восстановление

В этом разделе описаны особенности Azure Time Series Insights, которые поддерживают работу приложений и служб, даже если происходит сбой (известный как *восстановление бизнес-катастрофы).*

### <a name="high-availability"></a>Высокий уровень доступности

Как служба Azure, Time Series Insights предоставляет определенные *функции высокой доступности,* используя избыточности на уровне региона Azure. Например, Azure поддерживает возможности *аварийного восстановления через функцию доступности перерегиона* Azure.

Дополнительные функции высокой доступности, предоставляемые через Azure (а также доступны для любого экземпляра Time Series Insights) включают:

- **Failover**: Azure обеспечивает [георепликацию и балансировку нагрузки.](https://docs.microsoft.com/azure/architecture/resiliency/recovery-loss-azure-region)
- Восстановление и **восстановление хранилища** **данных:** Azure предоставляет [несколько вариантов сохранения и восстановления данных.](https://docs.microsoft.com/azure/architecture/resiliency/recovery-data-corruption)
- **Восстановление сайта Azure**: Azure предоставляет функции восстановления сайта через [восстановление сайта Azure.](https://docs.microsoft.com/azure/site-recovery/)
- **Резервное копирование Azure:** [Резервное копирование Azure](https://docs.microsoft.com/azure/backup/backup-architecture) поддерживает как резервное копирование в предварительном, так и в облаке для облачных вычислений.

Убедитесь, что соответствующие функции Azure обеспечивают глобальную, кросс-регионам высокую доступность для ваших устройств и пользователей.

> [!NOTE]
> Если Azure настроен для обеспечения доступности кросс-региона, в Azure Time Series Insights не требуется дополнительная конфигурация доступности для перекрестных регионов.

### <a name="iot-and-event-hubs"></a>IoT и центры событий

Некоторые службы Azure IoT также включают встроенные функции восстановления бизнес-аварий:

- [Высокое аварийное восстановление azure IoT Hub,](https://docs.microsoft.com/azure/iot-hub/iot-hub-ha-dr)которое включает в себя избыточность внутрирегиона
- [Политики концентратов событий Azure](https://docs.microsoft.com/azure/event-hubs/event-hubs-geo-dr)
- [Избыточность хранилища Azure](https://docs.microsoft.com/azure/storage/common/storage-redundancy)

Интеграция Time Series Insights с другими службами предоставляет дополнительные возможности аварийного восстановления. Например, телеметрия, отправленная в концентратор событий, может быть упорна в резервную базу данных хранения Azure Blob.

### <a name="time-series-insights"></a>Аналитика временных рядов

Существует несколько способов обеспечения работы данных, приложений и служб Time Series Insights, даже если они нарушены. 

Однако можно определить, что для следующих целей также требуется полная копия резервной копии среды Серии времени Azure:

- В качестве *примера неудачи* специально для Time Series Insights перенаправить данные и трафик на
- Для сохранения данных и аудита информации

Как правило, лучший способ дублировать среду Time Series Insights — это создать вторую среду Time Series Insights в области резервного копирования Azure. События также отправляются в эту вторичную среду из исходного источника событий. Убедитесь, что вы используете вторую выделенную группу потребителей. Следуйте руководящим принципам восстановления бизнес-катастроф, описанным ранее.

Для создания дубликатной среды:

1. Создайте среду во втором регионе. Для получения дополнительной информации прочитайте [Создать новую среду исследования временных рядов на портале Azure](https://docs.microsoft.com/azure/time-series-insights/time-series-insights-get-started).
1. Создайте для вашего источника событий вторую выделенную группу потребителей.
1. Подключите этот источник событий к новой среде. Убедитесь, что вы обозначите вторую специальную группу потребителей.
1. Просмотрите документацию Time Series Insights [IoT hub](https://docs.microsoft.com/azure/time-series-insights/time-series-insights-how-to-add-an-event-source-iothub) и [документацию Event Hubs.](https://docs.microsoft.com/azure/time-series-insights/time-series-insights-data-access)

При возникновении события:

1. Если во время инцидента ваш основной регион дал сбой, переключите операции на резервную среду службы "Аналитика временных рядов".
1. Используйте второй регион для резервного копирования и восстановления всех телеметрических данных Time Series Insights и запросов.

> [!IMPORTANT]
> При сбою происходит:
> 
> * Также может произойти задержка.
> * По мере перенаправления операций может произойти сиюминутный всплеск обработки сообщений.
> 
> Для получения дополнительной информации, [прочитайте Mitigate задержки в Time Series Исследования](https://docs.microsoft.com/azure/time-series-insights/time-series-insights-environment-mitigate-latency).

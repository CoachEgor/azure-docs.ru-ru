Возможности хранилища ограничиваются дисковым пространством или аппаратным ограничением *максимального количества* индексов, документов или высокоуровневых ресурсов (в зависимости от того, какое из ограничений достигается первым). В приведенной ниже таблице указаны размеры хранилища. Максимальные ограничения на индексы, документы и другие объекты см. [в разделе ограничения по ресурсам](../articles/search/search-limits-quotas-capacity.md#index-limits).

| Resource | Бесплатный | Базовый<sup>1</sup> | S1 | S2 | S3 | S3&nbsp;HD<sup>2</sup> | L1 | L2 |
| -------- | --- | --- | --- | --- | --- | --- | --- | --- |
| Соглашение об уровне обслуживания (SLA)<sup>3</sup>  |Нет |Да |Да |Да |Да |Да |Да |Да |
| Объем хранилища на секцию |50 MB |2 ГБ |25 ГБ |100 ГБ |200 ГБ |200 ГБ |1 TБ |2 ТБ |
| Количество секций для службы |Н/Д |1 |12 |12 |12 |3 |12 |12 |
| Размер раздела |Н/Д |2 ГБ |25 ГБ |100 ГБ |200 ГБ |200 ГБ |1 TБ |2 ТБ |
| Реплики |Н/Д |3 |12 |12 |12 |12 |12 |12 |

<sup>1</sup> На уровне "Базовый" есть одна фиксированная секция. На этом уровне дополнительные единицы поиска используются для выделения дополнительных реплик для повышения нагрузки запросов.

<sup>2</sup> S3 HD имеет жесткое ограничение в трех разделах, что меньше, чем ограничение на число секций для S3. Нижняя граница ограничения на секционирование применяется, так как количество индексов для S3 HD значительно выше. Учитывая, что ограничения службы действуют для вычислительных ресурсов (хранение и обработка) и содержимого (индексы и документы), сначала достигаются ограничения для содержимого.

<sup>3</sup> соглашения об уровне обслуживания предлагаются для оплачиваемых служб на выделенных ресурсах. Для бесплатных служб и функций в предварительной версии соглашения об уровне обслуживания не предоставляются. Для оплачиваемых служб соглашения об уровне обслуживания вступают в силу, если для службы обеспечена достаточная избыточность. Для запроса соглашений об уровне обслуживания (Read) требуются две или более реплики. Для запросов и индексирования (для чтения и записи) требуются три или более реплики. Количество секций не учитывает соглашение об уровне обслуживания.

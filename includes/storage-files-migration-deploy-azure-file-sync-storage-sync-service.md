На этом этапе вам понадобятся учетные данные подписки Azure.

Основной ресурс для настройки синхронизации файлов Azure называется "Служба синхронизации хранения".
Мы рекомендуем вам развернуть только один для всех серверов в компании, которые синхронизируют один и тот же набор файлов сейчас или в будущем. Создавайте несколько служб синхронизации хранения данных только в том случае, если у вас есть различные наборы серверов, которые никогда не должны обмениваться данными. (например: синхронизация той же доли файла Azure). В противном случае наилучшей практикой является единая служба синхронизации хранилища.

Выберите регион Azure для службы синхронизации данных, которая находится близко к расположению офиса. Все остальные облачные ресурсы должны быть развернуты в одном регионе.
Чтобы упростить управление, создайте новую группу ресурсов в подписке, в которой размещаются ресурсы синхронизации и хранения.

В следующей статье описывается, как развернуть службу синхронизации данных. Только следуйте этой части документа. В более поздних шагах будут установлены ссылки на другие подразделы этого документа.

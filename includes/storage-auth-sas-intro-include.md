Общая подпись доступа (SAS) позволяет предоставить ограниченный доступ к контейнерам и каплям в вашей учетной записи хранения. При создании SAS указывается его ограничения, в том числе, к каким ресурсам Azure Storage, к которым клиент унего имеет доступ, какие разрешения у них есть на этих ресурсах и как долго SAS действителен.

Каждый SAS подписывается ключом. Вы можете подписать SAS одним из двух способов:

- С помощью ключа, созданного с помощью учетных данных Azure Active Directory (Azure AD). SAS, подписанный с учетными данными Azure AD, является *делегацией пользователей* SAS.
- С ключом учетной записи хранилища. Как *служба SAS,* так и *учетная запись SAS* подписаны ключом учетной записи хранилища.

Делегация пользователя SAS обеспечивает превосходную безопасность SAS, которая подписывается ключом учетной записи хранилища. Корпорация Майкрософт рекомендует использовать делегацию пользователей SAS, когда это возможно. Для получения дополнительной информации см. [Грант ограниченный доступ к данным с общими подписями доступа (SAS)](../articles/storage/common/storage-sas-overview.md).

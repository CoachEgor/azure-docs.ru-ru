С помощью шаблона проекта функций Azure в Visual Studio можно создать проект, который затем можно опубликовать в приложении-функции в Azure. Приложение-функция позволяет группировать функции в логические единицы и упростить развертывание, масштабирование и совместное использование ресурсов, а также управление ими.

1. В Visual Studio в меню **Файл** выберите пункты **Создать** > **Проект**.

1. В диалоговом окне **Создание проекта** выполните поиск по строке `functions`, выберите шаблон **Функции Azure** и нажмите кнопку **Далее**.

1. Введите имя проекта и нажмите кнопку **Создать**. Имя приложения-функции должно быть допустимым в качестве пространства имен C#, поэтому не используйте символы подчеркивания, дефисы и другие символы, не являющиеся буквенно-цифровыми.

1. В диалоговом окне **Создание нового приложения Функций Azure** используйте следующие варианты:

    + **Azure Functions v2 (.NET Core)** (Функции Azure версии 2 (.NET Core));
    + **триггером HTTP**
    + **Учетная запись хранения**. **Эмулятор хранения**;
    + **Уровень авторизации**: **Анонимный**. 

    | Параметр      | Рекомендуемое значение  | Описание                      |
    | ------------ |  ------- |----------------------------------------- |
    | **Functions runtime** (Среда выполнения функций) | **Функции Azure 2.x <br />(.NET Core)** | Этот параметр создает проект функции, использующий среду выполнения Функций Azure версии 2.x, которая поддерживает .NET Core. Функции Azure 1.x поддерживают .NET Framework. Дополнительные сведения см. в статье [Azure Functions runtime versions overview](../articles/azure-functions/functions-versions.md) (Общие сведения о версиях среды выполнения Функций Azure).   |
    | **Function template** (Шаблон функции) | **триггером HTTP** | Этот параметр создает функцию, которая активируется с помощью HTTP-запроса. |
    | **Учетная запись хранения**  | **Эмулятор хранения** | Триггер HTTP не использует подключение к учетной записи службы хранилища Azure. Для всех триггеров других типов требуется допустимая строка подключения к учетной записи хранения. Так как для Функций требуется учетная запись хранения, она назначается или создается при публикации проекта в Azure. |
    | **Уровень авторизации** | **Анонимный** | Создаваемую функцию может активировать любой клиент без указания ключа. Этот параметр авторизации позволяет легко тестировать новые функции. Дополнительные сведения о ключах и авторизации см. в разделе [Ключи авторизации](../articles/azure-functions/functions-bindings-http-webhook-trigger.md#authorization-keys) статьи [Привязки HTTP и webhook в функциях Azure](../articles/azure-functions/functions-bindings-http-webhook.md). |
    
    > [!NOTE]
    > Убедитесь, что для параметра **Уровень авторизации** задано значение `Anonymous`. Если вы выберете для `Function` уровень по умолчанию, вы должны указать [функциональную клавишу](../articles/azure-functions/functions-bindings-http-webhook-trigger.md#authorization-keys) в запросах на доступ к конечной точке функции.
    
4. Нажмите кнопку **Создать**, чтобы создать проект функции и функцию, активируемую с помощью HTTP.

Azure Storage предлагает несколько типов учетных записей хранения данных. Каждый тип поддерживает различные возможности, и у каждого из них своя модель ценообразования. Прежде чем создавать учетную запись хранения, рассмотрите эти различия, чтобы определить подходящий для ваших приложений тип учетной записи. Типы учетных записей хранения:

- **Учетные записи v2 общего назначения**: Базовый тип учетной записи хранилища для капли, файлы, очереди и таблицы. Рекомендуется для большинства сценариев, в которых используется служба хранилища Azure.
- **Учетные записи v1**общего назначения : Тип учетной записи Legacy для капли, файлы, очереди и таблицы. Вместо этих учетных записей используйте учетные записи общего назначения версии 2, если возможно.
- **Учетные записи BlockBlobStorage:** Учетные записи хранения с премиум-характеристиками производительности для блоков и капли приложений. Рекомендуется для сценариев с высокими ставками транзакций или сценариев, в которых используются более мелкие объекты или которые требуют постоянно низкой задержки хранилища.
- **Учетные записи FileStorage**: Учетные записи только для хранения файлов с высокопроизводительными характеристиками. Рекомендуется для корпоративных или высокопроизводительных приложений.
- **Учетные записи BlobStorage**: Учетные записи только для хранения данных Legacy Blob. Вместо этих учетных записей используйте учетные записи общего назначения версии 2, если возможно.

<div id="deployment-model"><sup>1</sup> Рекомендуется использовать модель развертывания с помощью Azure Resource Manager. Учетные записи хранения, в которых используется классическая модель развертывания, по-прежнему можно создавать в некоторых расположениях, а имеющиеся классические учетные записи по-прежнему поддерживаются. Дополнительные сведения см. в статье <a href="https://docs.microsoft.com/azure/azure-resource-manager/resource-manager-deployment-model">Развертывание с помощью Azure Resource Manager и классическое развертывание: сведения о моделях развертывания и состоянии ресурсов</a>.</div><br/>

<div id="encryption"><sup>2</sup> Все учетные записи хранения шифруются с помощью Шифрования службы хранилища (SSE) для неактивных данных. Дополнительные сведения см. в статье <a href="https://docs.microsoft.com/azure/storage/common/storage-service-encryption">Шифрование службы хранилища Azure для неактивных данных (предварительная версия)</a>.</div><br/>

<div id="archive"><sup>3</sup> Архив хранения и капли уровня многоуровневые только поддержка блок капли. Уровень Архива доступен только на уровне отдельной капли, а не на уровне учетной записи хранилища. Дополнительные сведения см. в статье <a href="https://docs.microsoft.com/azure/storage/blobs/storage-blob-storage-tiers">Хранилище BLOB-объектов Azure: горячий, холодный и архивный уровни</a>.</div><br/>

<div id="zone-redundant-storage"><sup>4</sup> Излишние хранилища зоны (ЗРС) и геозоны-излишние хранилища (ГЗРС/РА-ГЗРС) (предварительный просмотр) доступны только для стандартных учетных записей V2, BlockBlobStorage и FileStorage в определенных регионах. Для получения дополнительной информации о вариантах <a href="https://docs.microsoft.com/azure/storage/common/storage-redundancy">Azure Storage redundancy</a>избыточности хранилища Azure см.</div><br/>

<div id="premium-performance"><sup>5</sup> Премиум-производительность для учетных записей v2 общего назначения и общего назначения доступна только для дисковых и страницных кабб. Премиум-производительность для блок-или приложения blobs доступна только на учетных записях BlockBlobStorage. Премиум-производительность для файлов доступна только в учетных записях FileStorage.</div><br/>

<div id="data-lake-gen2"><sup>6</sup> Azure Data Lake Storage 2 — это набор возможностей, предназначенных для анализа больших данных, построенных на хранилище Azure Blob. Data Lake Storage Gen2 поддерживается только в учетных записях хранения V2 общего назначения с включенным иерархическим пространством имен. Для получения дополнительной информации о хранении данных озера Gen2, смотрите <a href="https://docs.microsoft.com/azure/storage/blobs/data-lake-storage-introduction">Введение в Azure Data Lake Storage Gen2</a>.</div>

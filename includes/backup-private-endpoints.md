Теперь можно использовать [частные конечные точки](https://docs.microsoft.com/azure/private-link/private-endpoint-overview) для безопасного резервного копирования данных с серверов внутри виртуальной сети в хранилище служб восстановления. Частная конечная точка использует IP-адрес из пространства адресов виртуальной сети для вашего хранилища. Сетевой трафик между ресурсами в виртуальной сети и хранилищем передается по виртуальной сети и частной связи в магистральной сети Майкрософт. Это устраняет уязвимость в общедоступном Интернете. Частные конечные точки можно использовать для резервного копирования и восстановления баз данных SQL и SAP HANA, которые выполняются внутри виртуальных машин Azure. Его также можно использовать для локальных серверов с помощью агента MARS.

Для резервного копирования виртуальных машин Azure не требуется подключение к Интернету, поэтому не требуются частные конечные точки, чтобы разрешить сетевую изоляцию.

>[!NOTE]
> В настоящее время эта функция находится в ограниченном доступе и доступна в западной центральной части США, юго-центральном регионе США, Западная часть США 2 и на Восточная часть США (доступность в других регионах Azure для отслеживания). Заполните [этот опрос](https://forms.microsoft.com/Pages/ResponsePage.aspx?id=v4j5cvGGr0GRqy180BHbR0H3_nezt2RNkpBCUTbWEapUQk5EQ1QxRzVOWDNDS1Y1Q0xLTkdLQ0U0RC4u) и напишите нам по azbackupnetsec@microsoft.com адресу, если вы хотите использовать частные конечные точки для Azure Backup. Возможность использовать эту функцию может быть утверждена службой Azure Backup.

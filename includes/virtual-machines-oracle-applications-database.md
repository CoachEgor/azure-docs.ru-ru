### <a name="database-tier"></a>Уровень базы данных

Уровень базы данных содержит экземпляры базы данных для приложения. База данных может быть как Oracle DB, Oracle RAC, либо Oracle Exadata Database system. 

Если выбор заключается в использовании Oracle DB, экземпляр базы данных может быть развернут в Azure через изображения Oracle DB, доступные на Azure Marketplace. Кроме того, можно использовать взаимосвязь между Azure и OCI для развертывания Oracle DB в модели PaaS на OCI.

Для Oracle RAC вы можете развернуть Oracle RAC в Azure CloudSimple в модели IaaS или в модели OCI в модели PaaS. Рекомендуется использовать двухузловую систему RAC. 

Наконец, для систем Exadata используйте соединение OCI и развертывание системы Exadata в OCI. Предыдущая выше схема архитектуры показывает систему Exadata, развернутую в OCI в двух подсетях.

Для сценариев производства разместите несколько экземпляров базы данных в двух зонах доступности (при развертывании в Azure) или двух доменах доступности (в OCI). Используйте Oracle Active Data Guard для синхронизации первичных и резервных баз данных.

Уровень базы данных получает запросы только от среднего уровня. Рекомендуется настроить группу сетевой безопасности (список безопасности при развертывании базы данных в OCI), чтобы разрешить запросы только на порте 1521 от среднего уровня и в порту 22 с сервера бастиона по административным причинам.

Для баз данных, развернутых в OCI, должна быть настроена отдельная виртуальная облачная сеть с динамическим шлюзом маршрутизатора (DRG), который подключен к контуру FastConnect.

Следуйте шагам для ваших конкретных обстоятельств.

### <a name="unregister-a-connected-process-server"></a>Отменить регистрацию подключенного сервера процесса

1. Создание удаленного подключения к серверу процесса в качестве администратора.
2. В **панели управления,** **открытые программы > Установить программу**.
3. Удалите программу **Службы восстановления мобильности сайта Microsoft Azure/Master Target Server.**
4. Удалите программу **Microsoft Azure Настройка восстановления сайта / Процесс сервера**.
5. После того, как программы в шагах 3 и 4 не установлены, удалите **Microsoft Azure настройки восстановления сайта / процесс сервера зависимостей**.

### <a name="unregister-a-disconnected-process-server"></a>Отменить отключенный сервер процесса

Используйте эти действия только в том случае, если нет возможности оживить машину, на которой установлен сервер процесса.

1. Войти в настройку сервера в качестве администратора.
2. Откройте запрос административной команды `%ProgramData%\ASR\home\svsystems\bin`и просмотрите.
3. Запустите эту команду, чтобы получить список одного или нескольких серверов процесса.

    ```
    perl Unregister-ASRComponent.pl -IPAddress <IP_of_Process_Server> -Component PS
    ```
    - Х. Нет: серийный номер сервера процесса.
    - IP/Name: IP-адрес и имя машины, работая на сервере процесса.
    - Сердцебиение: Последнее сердцебиение от машины сервера процесса.
    ![Команда отмены регистрации](media/site-recovery-vmware-unregister-process-server/Unregister-cmd.PNG)

4. Укажите серийный номер сервера процесса, который вы хотите отменить.
5. Нерегистрирующее сервер процесса удалит все его детали из системы и отображает сообщение: **Успешно незарегистрированное имя сервера> (сервер-IP-адрес)**

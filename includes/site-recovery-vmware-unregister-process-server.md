Выполните действия для конкретных обстоятельств.

### <a name="unregister-a-connected-process-server"></a>Отмена регистрации сервера подключенного процесса

1. Установите удаленное подключение к серверу обработки с правами администратора.
2. В **панели управления**откройте **программы > Удаление программы**.
3. Удалите программу **Microsoft Azure сайт Recovery Mobility Service/главный целевой сервер**.
4. Удалите программу **сервер конфигурации или обработки Microsoft Azure Site Recovery**.
5. После удаления программы в шагах 3 и 4 удалить **зависимости сервера конфигурации или обработки Microsoft Azure Site Recovery**.

### <a name="unregister-a-disconnected-process-server"></a>Отмена регистрации сервера в отключенный процесс

Используйте эти действия только в том случае, если невозможно восстановить компьютер, на котором установлен сервер обработки.

1. Войдите в сервер конфигурации с правами администратора.
2. Откройте командную строку администратора и перейдите к `%ProgramData%\ASR\home\svsystems\bin`.
3. Выполните следующую команду, чтобы получить список один или несколько серверов обработки.

    ```
    perl Unregister-ASRComponent.pl -IPAddress <IP_of_Process_Server> -Component PS
    ```
    - S. Нет: серийный номер процесса сервера.
    - Имя или IP-адрес: IP-адрес и имя компьютера сервера обработки.
    - Периодический сигнал: Последний пульс от компьютера сервера обработки.
    ![Команда отмены регистрации](media/site-recovery-vmware-unregister-process-server/Unregister-cmd.PNG)

4. Укажите серийный номер сервер обработки, который вы хотите отменить.
5. Отмена регистрации сервера обработки, удалить все сведения о нем из системы и отображает сообщение: **Успешно отменена имя_сервера > (-IP-адрес сервера)**

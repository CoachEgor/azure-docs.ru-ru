1. Щелкните правой кнопкой мыши проект в **обозревателе решений** и выберите пункт **Опубликовать**.

1. В **публикации** диалоговое окно, выберите **служба приложений Microsoft Azure**, выберите **Create New**, а затем выберите **публикации**.

   ![Выбор целевого объекта публикации](./media/webjobs-publish-netcore/pick-publish-target.png)

1. В **создать службу приложений** диалоговое окно, используйте параметры размещения, указано в таблице под изображением:

    ![Диалоговое окно "Создание службы приложений"](./media/webjobs-publish-netcore/app-service-dialog.png)

    | Параметр      | Предлагаемое значение  | Описание                                |
    | ------------ |  ------- | -------------------------------------------------- |
    | **Имя приложения** | Глобально уникальное имя | Имя, которое однозначно идентифицирует новое приложение-функцию. |
    | **Подписка** | Выберите свою подписку | Подписка Azure, которую нужно использовать. |
    | **[Группа ресурсов](../articles/azure-resource-manager/resource-group-overview.md)** | myResourceGroup |  Имя группы ресурсов, в которой создается приложение-функция. Чтобы создать группу ресурсов, выберите **Создать**.|
    | **[План размещения](../articles/app-service/overview-hosting-plans.md)** | План службы приложений | [План службы приложений](../articles/app-service/overview-hosting-plans.md) указывает расположение, размер и функции фермы веб-серверов, в которой размещается приложение. Вы можете сэкономить средства при размещении нескольких приложений, настроив совместное использование одного плана службы приложений в веб-приложениях. Планы службы приложений определяют область, размер экземпляра, число масштабируемых элементов и SKU (Free, Shared, Basic, Standard или Premium). Выберите **New** создать новый план службы приложений. |

1. Нажмите кнопку **создать** для создания веб-задания и связанные ресурсы в Azure с использованием этих параметров и развернуть код проекта.

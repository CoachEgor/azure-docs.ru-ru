1. Щелкните правой кнопкой мыши проект в **обозревателе решений** и выберите пункт **Опубликовать**.

1. В диалоге **Publish** выберите **службу приложений Microsoft Azure,** выберите **«Создайте новое»,** а затем выберите **Publish**.

   ![Выберите цель публикации](./media/webjobs-publish-netcore/pick-publish-target.png)

1. В диалоге **службы создания приложений** используйте настройки хостинга, указанные в таблице ниже изображения:

    ![Диалоговое окно "Создание службы приложений"](./media/webjobs-publish-netcore/app-service-dialog.png)

    | Параметр      | Рекомендуемое значение  | Описание                                |
    | ------------ |  ------- | -------------------------------------------------- |
    | **Название приложения** | Глобально уникальное имя | Имя, которое однозначно идентифицирует новое приложение-функцию. |
    | **Подписка** | Выберите свою подписку | Подписка Azure, которую нужно использовать. |
    | **[Ресурсная группа](../articles/azure-resource-manager/management/overview.md)** | myResourceGroup |  Имя группы ресурсов, в которой создается приложение-функция. Чтобы создать группу ресурсов, выберите **Создать**.|
    | **[План хостинга](../articles/app-service/overview-hosting-plans.md)** | План службы приложений | [План Службы приложений](../articles/app-service/overview-hosting-plans.md) определяет местоположение, размер и особенности фермы веб-серверов, на вещах которого находится ваше приложение. Вы можете сэкономить средства при размещении нескольких приложений, настроив совместное использование одного плана службы приложений в веб-приложениях. Планы службы приложений определяют область, размер экземпляра, количество масштабов и SKU (Free, Shared, Basic, Standard или Premium). Выберите **новый** для создания нового плана службы приложений. |

1. Нажмите **«Создать»** для создания WebJob и связанных с ними ресурсов в Azure с этими настройками и развернуть код проекта.

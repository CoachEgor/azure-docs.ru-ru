1. Щелкните правой кнопкой мыши проект в **обозревателе решений** и выберите пункт **Опубликовать**.

1. В диалоговом окне **Публикация** выберите **Microsoft Azure служба приложений**, щелкните **создать**и выберите **опубликовать**.

   ![Выбор целевого объекта публикации](./media/webjobs-publish-netcore/pick-publish-target.png)

1. В диалоговом окне **Создание службы приложений** используйте параметры размещения, как указано в таблице под изображением.

    ![Диалоговое окно "Создание службы приложений"](./media/webjobs-publish-netcore/app-service-dialog.png)

    | Параметр      | Рекомендуемое значение  | Описание                                |
    | ------------ |  ------- | -------------------------------------------------- |
    | **Имя приложения** | Глобально уникальное имя | Имя, которое однозначно идентифицирует новое приложение-функцию. |
    | **Подписка** | Выберите свою подписку | Подписка Azure, которую нужно использовать. |
    | **[Группа ресурсов](../articles/azure-resource-manager/management/overview.md)** | myResourceGroup |  Имя группы ресурсов, в которой будет создано приложение-функция. Чтобы создать группу ресурсов, выберите **Создать**.|
    | **[План размещения](../articles/app-service/overview-hosting-plans.md)** | План службы приложений | [План службы приложений](../articles/app-service/overview-hosting-plans.md) указывает расположение, размер и функции фермы веб-серверов, в которой размещается приложение. Вы можете сэкономить средства при размещении нескольких приложений, настроив совместное использование одного плана службы приложений в веб-приложениях. Планы службы приложений определяют регион, размер экземпляра, число масштабируемых и SKU (Free, Shared, Basic, Standard или Premium). Выберите **создать, чтобы** создать новый план службы приложений. |

1. Щелкните **создать** , чтобы создать веб-задание и связанные ресурсы в Azure с этими параметрами и развернуть код проекта.

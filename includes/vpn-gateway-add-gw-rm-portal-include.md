1. Из меню [портала Azure](https://portal.azure.com) выберите **Создать ресурс.** 

   ![Создание ресурса на портале Azure](./media/vpn-gateway-add-gw-rm-portal-include/azure-portal-create-resource.png)
2. В поле **поиска Marketplace** введите «Виртуальный сетевой шлюз». Найдите **виртуальный сетевой шлюз** в поисковой верете и выберите запись. На странице **виртуального сетевого шлюза** выберите **Create**. Откроется страница **Создание шлюза виртуальной сети**.
3. На вкладке **Basics** заполните значения для виртуального сетевого шлюза.

   ![Создание полей страниц виртуального сетевого шлюза](./media/vpn-gateway-add-gw-rm-portal-include/gateway1.png "Создание полей страниц виртуального сетевого шлюза")

   ![Создание полей страниц виртуального сетевого шлюза](./media/vpn-gateway-add-gw-rm-portal-include/gateway2.png "Создание полей страниц виртуального сетевого шлюза")

   **Детали проекта**

   - **Подписка**: Выберите подписку, которая вы хотите использовать из выпадения.
   - **Группа ресурсов**: Эта настройка автоматически заполняется при выборе виртуальной сети на этой странице.

   **Детали инстанции**

   - **Имя**. Назовите свой шлюз. Наименование шлюза не то же самое, что наименование подсети шлюза. Это имя объекта шлюза, который создается.
   - **Регион**: Выберите регион, в котором вы хотите создать этот ресурс. Область шлюза должна быть такой же, как и виртуальная сеть.
   - **Тип шлюза**. Выберите **VPN**. VPN-шлюзы используют тип шлюза виртуальной сети **VPN**.
   - **Тип VPN**. Выберите тип VPN, который указан для конфигурации. Для большинства конфигураций требуется тип VPN на основе маршрута.
   - **SKU**. Выберите номер SKU шлюза из раскрывающегося списка. Номера SKU, перечисленные в раскрывающемся списке, зависят от выбранного типа VPN. Дополнительные сведения о номерах SKU шлюзов см. в разделе [SKU шлюзов](../articles/vpn-gateway/vpn-gateway-about-vpn-gateway-settings.md#gwsku).
   - **Поколение**: Для получения информации о генерации VPN Шлюза, [см.](../articles/vpn-gateway/vpn-gateway-about-vpngateways.md#gwsku)
   - **Виртуальная сеть**: От выпадения, выберите виртуальную сеть, к которой вы хотите добавить этот шлюз.
   - **Диапазон адресов подсети шлюза:** Это поле появляется только в том случае, если у вашего VNet нет подсети шлюза. Если возможно, сделайте диапазон /27 или больше (/26,/25 и т.д.). Мы не рекомендуем создавать диапазон меньше, чем /28. Если у вас уже есть подсеть шлюза, вы можете просматривать данные GatewaySubnet, перемещаясь по виртуальной сети. Нажмите **на подсети,** чтобы просмотреть диапазон. Если вы хотите изменить диапазон, вы можете удалить и воссоздать GatewaySubnet.

   **Общедоступный IP-адрес**. Этот параметр задает объект общедоступного IP-адреса, который связывается с VPN-шлюзом. Общедоступный IP-адрес динамически назначается этому объекту при создании VPN-шлюза. Общедоступный IP-адрес изменяется только после удаления и повторного создания шлюза. При изменении размера, сбросе или других внутренних операциях обслуживания или обновления IP-адрес VPN-шлюза не изменяется.

     - **Общественный IP-адрес**: Оставьте **Создать новый** выбранный.
     - **Имя публичного IP-адреса**: В текстовом поле введите имя для экземпляра общедоступного IP-адреса.
     - **Назначение**: VPN шлюз поддерживает только Dynamic.

   **Режим Active-Active:** Выберите активный **режим Включить** только при создании активной конфигурации шлюза. В противном случае не выбирайте этот параметр.

   Не выбирайте параметр **Настроить ASN BGP**, кроме случаев, когда его выбор обусловлен используемой конфигурацией. Если этот параметр необходим, по умолчанию для ASN устанавливается значение 65515, которое при необходимости можно изменить.
4. Выберите **Обзор и создайте** для выполнения проверки. После того, как проверка пройдет, выберите **Создать** для развертывания ШЛюза VPN. Полностью создание и развертывание шлюза может занять до 45 минут. Статус развертывания можно увидеть на странице «Обзор» для шлюза.

После создания шлюза вы можете просмотреть назначенный ему IP-адрес в разделе сведений о виртуальной сети на портале. Шлюз будет отображаться как подключенное устройство.

## <a name="register-extensions"></a>Регистрация расширений

За исключением триггеров HTTP и таймера, привязки функций в версии 2.x и выше времени выполнения реализованы в виде пакетов расширений. В версии 2.x и за пределами времени выполнения функций Azure необходимо явно зарегистрировать расширения для связывающих типов, используемых в функциях. Исключение составляют привязки HTTP и триггеры таймер, которые не требуют расширения.

Вы можете установить обязательные расширения по отдельности, или вы можете добавить ссылку пакета расширения на файл проекта host.json. Пакеты расширения устраняют вероятность проблем совместимости пакетов при использовании нескольких типов связывания. Это рекомендуемый подход к регистрации обязательных расширений. Пакеты расширения также удаляют требование об установке .NET Core 2.x SDK.

### <a name="register-individual-extensions"></a>Регистрация индивидуальных расширений

Если вам нужно установить расширения, которые не в комплекте, вы можете вручную зарегистрировать отдельные пакеты расширения для конкретных привязок. 

> [!NOTE]
> Чтобы вручную зарегистрировать расширения `func extensions install`с помощью, необходимо установить ядро .NET Core 2.x SDK.

После обновления файла *function.json* для включения всех привязок, необходимых для функции, выполните следующую команду в папке проекта.

```bash
func extensions install
```

Эта команда считывает файл *function.json*, чтобы определить, какие пакеты требуются, а затем устанавливает их и перестраивает проект расширений. Затем добавляются все новые привязки в текущей версии, но существующие привязки не обновляются. Используйте параметр `--force`, чтобы обновить существующие привязки до последней версии при установке новых пакетов.

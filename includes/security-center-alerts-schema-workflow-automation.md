### <a name="the-data-model-of-the-schema"></a>Модель данных схемы

|Поле|Тип данных|Описание|
|----|----|----|
|**AlertDisplayName**|Строка|Имя отображения оповещения.|
|**AlertType**|Строка|Тип оповещения. Оповещения одного и того же типа должны иметь одинаковое значение. Это поле представляет собой строку с ключом, представляющую тип оповещения, а не экземпляр апогея. Все экземпляры оповещения из одной и той же логики обнаружения/аналитика должны иметь одинаковое значение для типа оповещения.|
|**Компромиссentity**|Строка|Имя отображения ресурса, наиболее связанного с этим предупреждением.|
|**Описание**|Строка|Описание оповещения.|
|**EndTimeUtc**|Дата и время|Время последнего события или действия, включенного в оповещение.  Поле должно быть строкой, соответствующей формату ISO8601, включая информацию о часовом поясе UTC.|
|**Entities**|IEnumerable (IEntity)|Список сущностей, связанных с предупреждением. Этот список может содержать смесь сущностей различных типов. Тип сущностей может быть любым из типов, определенных в разделе Entities. Сущности, не вкоторых которых ниже приведены ниже, также могут быть отправлены, однако не гарантируется их обработка (оповещение не завершит проверку с новыми типами сущностей).|
|**РасширенныйСвойства**|Словарь (Струна, струна)|Поставщики могут (по желанию) включать пользовательские поля здесь.|
|**Намерения**|Перечисление.|Цепочка убийства, связанная с намерением, лежащим в состоянии предупреждения. Для списка поддерживаемых значений и пояснений поддерживаемых намерений Azure Security Center по цепочке убийств [см.](../articles/security-center/alerts-reference.md#intentions)<br/>Это поле может иметь несколько значений (разделенных запятой).|
|**IsIncident**|Bool|Это поле определяет, является ли предупреждение инцидентом (составная группировка нескольких предупреждений) или одним предупреждением. Значение по умолчанию для поля является «ложным» (имеется в виду, что это одно оповещение).|
|**ProcessingEndTime**|Дата и время|Время оповещения было доступно конечному пользователю в исходном продукте, удерживая оповещение.|
|**ProductName**|Строка|Название продукта, опубликовававого этим оповещением (Azure Security Center, Azure ATP, Microsoft Defender ATP, O365 ATP, MCAS и так далее).|
|**Восстановительные шаги**|Список<String>|Ручные элементы действия для исправления оповещения.|
|**Идентификаторы ресурсов**|Список (Идентификаторы ресурсов)|Идентификаторы ресурсов для этого оповещения, которые могут быть использованы для направления оповещения в правильную группу экспозиции продукта (арендатор, рабочее пространство, подписка и т.д.). На оповещение может быть несколько идентификаторов разного типа.|
|**Серьезность**|Перечисление.|Тяжесть предупреждения, как сообщил поставщик. Возможные значения: информационные, низкие, средние и высокие.|
|**StartTimeUtc**|Дата и время|Время первого события или действия, включенных в оповещение. Поле должно быть строкой, соответствующей формату ISO8601, включая информацию о часовом поясе UTC.|
|**Состояние**|Перечисление.|Состояние жизненного цикла оповещения.<br/>Поддерживаемые статусы: Новые, Разрешенные, Уволенные, Неизвестные.<br/>Оповещению, означаваемому значение, не относящееся к поддерживаемым опционам, присваивается статус "Неизвестный".<br/>Оповещению, в которое не указывается значение, присваивается статус "Новое".|
|**SystemAlertId**|Строка|Идентификатор оповещения.|
|**TimeGenerated**|Дата и время|Время оповещения было создано поставщиком оповещения. Если не сообщается внутренними поставщиками оповещения, продукт может назначить время, полученное для обработки продуктом.  Поле должно быть строкой, соответствующей формату ISO8601, включая информацию о часовом поясе UTC.|
|**Имя поставщика**|Строка|Имя поставщика, который поднимает оповещение.|

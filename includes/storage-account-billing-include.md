Счет за хранилище Azure основан на использовании учетной записи хранилища. Все объекты в учетной записи хранения при выставлении счета учитываются вместе как группа. 

Затраты на хранение рассчитываются по следующим факторам: 

* **Регион** означает географический регион, в котором расположена учетная запись.
* **Тип учетной записи** относится к типу учетной записи хранения, который вы используете. 
* **Уровень доступа** относится к шаблону использования данных, указанному для учетной записи хранения v2 общего назначения или Blob.
* **Емкость** хранилища относится к тому, сколько выделения учетной записи хранилища вы используете для хранения данных.
* **Репликация** определяет, какое количество копий ваших данных следует хранить и где.
* **Транзакции** относятся ко всем операциям чтения и записи в Хранилище Azure.
* **Передача данных** означает любые данные, переданные из региона Azure. Когда данные в учетной записи хранилища доступны в приложении, которое не работает в том же регионе, с вас взимается плата за выход данных. Сведения об использовании групп ресурсов для группирования данных и служб в одном регионе, чтобы ограничить плату за исходящий трафик, см. в разделе [Что такое группа ресурсов Azure?](https://docs.microsoft.com/azure/cloud-adoption-framework/govern/resource-consistency/resource-access-management#what-is-an-azure-resource-group) 

На странице с [ценами на хранилище Azure](https://azure.microsoft.com/pricing/details/storage/) представлена подробная информация о ценах в зависимости от типа учетной записи, емкости хранилища, репликации и транзакций. Подробные сведения о ценах на исходящие данные см. на странице с [ценами на передачу данных](https://azure.microsoft.com/pricing/details/data-transfers/). Используйте [калькулятор цен на хранилище Azure](https://azure.microsoft.com/pricing/calculator/?scenario=data-management) для оценки затрат.

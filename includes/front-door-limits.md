### <a name="timeout-values"></a>Значения времени ожидания
#### <a name="client-to-front-door"></a>Клиент — Front Door
* Front Door использует время ожидания простоя TCP-подключения, равное 61 секунде.

#### <a name="front-door-to-application-back-end"></a>Передняя дверь для применения бэк-энда
* Если ответ представляет собой фрагментированный ответ, возврат 200 возвращается, если или когда получен первый кусок.
* После того, как запрос HTTP перенаправляется на заднюю часть, Front Door ждет 30 секунд для первого пакета из задней части. Затем он возвращает клиенту ошибку 503. Это значение настраивается через поле sendRecvTimeoutSeconds в API.
    * Для сценариев кэширования этот тайм-аут не настраивается, и поэтому, если запрос кэшируется и он занимает более 30 секунд для первого пакета от передней двери или из бэкэнда, то ошибка 504 возвращается клиенту. 
* После того, как первый пакет получен из задней части, Front Door ждет 30 секунд в простое тайм-аут. Затем он возвращает клиенту ошибку 503. Это значение тайм-аута не настраивается.
* Передняя дверь к задней части TCP сессии тайм-аут составляет 90 секунд.

### <a name="upload-and-download-data-limit"></a>Ограничение загрузки и загрузки данных

|  | С кородингом передачи (CTE) | Без HTTP chunking |
| ---- | ------- | ------- |
| **Скачать** | Размер загрузки не ограничен. | Размер загрузки не ограничен. |
| **Отправка** |    Там нет предела до тех пор, как каждый CTE загрузить меньше, чем 2 ГБ. | Размер не может быть больше, чем 2 ГБ. |

### <a name="other-limits"></a>Другие ограничения
* Максимальный размер URL - 8192 байта - Указывает максимальную длину необработанного URL (схема - хост-имя , порт - путь - строка запроса URL)
* Максимальный размер строки запроса - 4096 байтов - определяет максимальную длину строки запроса в байтах.
* Максимальный размер заголовка ответа HTTP от URL-адреса зонда работы - 4096 байтов - Указана максимальная длина всех заголовков ответов зондов здоровья.

### <a name="timeout-values"></a>Значения времени ожидания
#### <a name="client-to-front-door"></a>Клиент — Front Door
* Front Door использует время ожидания простоя TCP-подключения, равное 61 секунде.

#### <a name="front-door-to-application-back-end"></a>Передняя дверь к серверной части приложения
* Если ответ является фрагментированным, то возвращается значение 200, если или когда получен первый фрагмент.
* После перенаправления HTTP-запроса на серверную часть первая дверца ждет 30 секунд для первого пакета от серверной части. Затем он возвращает клиенту ошибку 503. Это значение можно настроить с помощью поля Сендреквтимеаутсекондс в API.
    * В сценариях кэширования это время ожидания не настраивается, поэтому, если запрос кэшируется и занимает более 30 секунд для первого пакета с передней дверцы или из серверной части, то клиенту возвращается ошибка 504. 
* После получения первого пакета от серверной части фронтальная дверца ждет 30 секунд в течение времени ожидания простоя. Затем он возвращает клиенту ошибку 503. Это значение времени ожидания не настраивается.
* Время ожидания для внутренней дверцы TCP-сеанса серверной части составляет 90 секунд.

### <a name="upload-and-download-data-limit"></a>Загрузка и скачивание ограничения данных

|  | С поблочным кодированием данных (CTE) | Без фрагментации HTTP |
| ---- | ------- | ------- |
| **Загрузить** | Размер загружаемого файла не ограничен. | Размер загружаемого файла не ограничен. |
| **Отправка** |    Ограничение не ограничено, если размер каждого ОТВ превышает 2 ГБ. | Размер не может быть больше 2 ГБ. |

### <a name="other-limits"></a>Другие ограничения
* Максимальный размер URL-адреса — 8 192 байт — указывает максимальную длину необработанного URL-адреса (схема + имя узла + порт + путь + строка запроса URL-адреса).
* Максимальный размер строки запроса — 4 096 байт — указывает максимальную длину строки запроса в байтах.
* Максимальный размер заголовка ответа HTTP от URL-адреса пробы работоспособности — 4 096 байта — указывает максимальную длину всех заголовков ответа проверок работоспособности.

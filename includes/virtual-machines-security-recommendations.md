Эта статья содержит рекомендации по безопасности для виртуальных машин Azure. Реализация этих рекомендаций поможет вам выполнить обязательства по обеспечению безопасности, как описано в нашей модели общей ответственности, и повысит общую безопасность решений веб-приложений. Дополнительные сведения о том, что делает корпорация Майкрософт для выполнения обязанностей поставщиков услуг, см. в статье [Общие обязанности для облачных вычислений](https://gallery.technet.microsoft.com/Shared-Responsibilities-81d0ff91/file/153019/1/Shared%20responsibilities%20for%20cloud%20computing.pdf).

Некоторые из рекомендаций, описанных в этой статье, можно автоматически отслеживать с помощью центра безопасности Azure. Центр безопасности Azure — это первая строка защиты ресурсов в Azure. Он периодически анализирует состояние безопасности ресурсов Azure и выявляет потенциальные уязвимости системы безопасности. Затем он предоставляет рекомендации по их устранению.

- Дополнительные сведения о рекомендациях центра безопасности Azure см. [в статье рекомендации по безопасности в центре безопасности Azure](../articles/security-center/security-center-recommendations.md).
- Сведения о центре безопасности Azure см. в статье [что такое центр безопасности Azure?](../articles/security-center/security-center-intro.md)

## <a name="recommendations"></a>Рекомендации

| Category | Рекомендация | Комментарии | Центр безопасности |
|-|-|----|--|
| Общее | При создании пользовательских образов виртуальных машин используйте последние обновления | Перед созданием образов убедитесь, что установлены все последние обновления для операционной системы и все приложения, которые будут частью образа.  | - |
| Общее | Обеспечение актуальности виртуальных машин | Вы можете использовать решение [Управление обновлениями](../articles/automation/automation-update-management.md) в службе автоматизации Azure для управления обновлениями операционной системы для компьютеров под управлением Windows и Linux в Azure. | [Да](../articles/security-center/security-center-apply-system-updates.md) |
| Общее | Резервное копирование виртуальных машин | [Azure Backup](../articles/backup/backup-overview.md) помогает защитить данные приложения с минимальными эксплуатационными издержками. Ошибки приложений могут повредить данные, а ошибки пользователей — привести к ошибкам в коде приложений. Служба архивации Azure защитит виртуальные машины Windows и Linux. | - |
| Общее | Использование нескольких виртуальных машин для повышения устойчивости и доступности | Если на виртуальной машине выполняются приложения, которым требуется высокий уровень доступности, следует использовать несколько виртуальных машин или [групп доступности](../articles/virtual-machines/windows/manage-availability.md) . | - |
| Общее | Внедрение стратегии непрерывности бизнес-процессов и аварийного восстановления (BCDR) | Azure Site Recovery позволяет выбирать из различных параметров, предназначенных для обеспечения непрерывности бизнес-процессов. Она поддерживает различные задачи репликации и отработки отказа. Дополнительные сведения см. в разделе [About Site Recovery](../articles/site-recovery/site-recovery-overview.md) | - |
| Управление удостоверениями и доступом | Централизованная проверка подлинности виртуальной машины | Вы можете централизовать проверку подлинности виртуальных машин Windows и Linux с помощью [аутентификации Azure AD](../articles/active-directory/develop/authentication-scenarios.md). | - |
| Безопасность данных | Шифрование дисков операционной системы | [Служба шифрования дисков Azure](../articles/security/azure-security-disk-encryption-overview.md) помогает шифровать диски виртуальной машины IaaS под управлением Windows и Linux. Шифрование дисков делает содержимое нечитаемым без необходимых ключей. Шифрование дисков защищает хранимые данные от несанкционированного доступа, которые в противном случае могли бы быть доступны при копировании диска.| [Да](../articles/security-center/security-center-apply-disk-encryption.md) |
| Безопасность данных | Шифрование дисков данных | [Служба шифрования дисков Azure](../articles/security/azure-security-disk-encryption-overview.md) помогает шифровать диски виртуальной машины IaaS под управлением Windows и Linux. Шифрование дисков делает содержимое нечитаемым без необходимых ключей. Шифрование дисков защищает хранимые данные от несанкционированного доступа, которые в противном случае могли бы быть доступны при копировании диска.| -  |
| Безопасность данных | Ограничение установленного программного обеспечения | Ограничение установленного программного обеспечения на то, что необходимо для успешной реализации решения, помогает снизить уязвимость вашего решения | - |
| Безопасность данных | Использование антивирусных и антивредоносных программ | В Azure можно использовать антивредоносное по от поставщиков безопасности, таких как Microsoft, Symantec, Trend Micro и Kaspersky. Это ПО помогает защитить ваши виртуальные машины от вредоносных файлов, программ для показа рекламы и других угроз. Вы можете развернуть защиту от вредоносных программ Майкрософт на основе рабочих нагрузок приложения, используя базовую, защищенную по умолчанию или расширенную настраиваемую конфигурацию. Дополнительные сведения о антивредоносном по Майкрософт для Azure см. в [статье антивредоносное по Майкрософт для облачных служб и виртуальных машин Azure](../articles/security/azure-security-antimalware.md) . | - |
| Безопасность данных | Безопасное хранение ключей и секретов | Упрощение управления секретами и ключами, предоставляя владельцам приложений безопасный централизованный режим, позволяет снизить риск случайной компрометации или утечки. Azure Key Vault можете безопасно хранить ключи в аппаратных модулях безопасности (HSM), сертифицированных на уровне 2 FIPS 140-2. Если вам нужно хранить ключи и секреты с помощью FIPs 140,2 уровня 3, можно использовать [выделенный модуль HSM Azure](../articles/dedicated-hsm/overview.md) . | - |
| Сети | Ограничение доступа к портам управления | Злоумышленники проверяют диапазоны IP-адресов общедоступного облака на наличие открытых портов управления и попытаются выполнить простые атаки, такие как общие пароли и известные неисправленные уязвимости. [Доступ к виртуальной машине по требованию (JIT](../articles/security-center/security-center-just-in-time.md) ) можно использовать для блокировки входящего трафика к виртуальным машинам Azure, что снижает вероятность атак и обеспечивает простой доступ к виртуальным машинам при необходимости. | - |
| Сети | Ограничение доступа к сети | Группы безопасности сети позволяют ограничить доступ к сети и управлять количеством видимых конечных точек. Дополнительные сведения см. [в разделе Создание, изменение или удаление группы безопасности сети](../articles/virtual-network/manage-network-security-group.md) . | - |
| Отслеживание | Мониторинг виртуальных машин | [Azure Monitor можно использовать для виртуальных машин](../articles/azure-monitor/insights/vminsights-overview.md) , чтобы отслеживать состояние виртуальных машин Azure и масштабируемых наборов виртуальных машин. Снижение производительности виртуальной машины может привести к прерыванию работы служб, что в свою очередь нарушает доступность, являющуюся одним из основных принципов безопасности. | - |

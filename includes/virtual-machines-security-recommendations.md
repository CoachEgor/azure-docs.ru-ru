Эта статья содержит рекомендации по безопасности для виртуальных машин Azure. Следуйте этим рекомендациям, чтобы помочь выполнить обязательства по обеспечению безопасности, описанные в нашей модели совместной ответственности. Рекомендации также помогут вам повысить общую безопасность решений для веб-приложений. Для получения дополнительной информации о том, что корпорация Майкрософт делает для выполнения обязанностей поставщика услуг, [см.](https://gallery.technet.microsoft.com/Shared-Responsibilities-81d0ff91)

Некоторые рекомендации этой статьи могут быть автоматически рассмотрены Центром безопасности Azure. Центр безопасности Azure — первая линия защиты ресурсов ВАшего Ресурса в Azure. Он периодически анализирует состояние безопасности ресурсов Azure для выявления потенциальных уязвимостей безопасности. Затем он рекомендует, как устранить уязвимости. Для получения дополнительной информации смотрите [рекомендации по безопасности в Центре безопасности Azure](../articles/security-center/security-center-recommendations.md).

| Рекомендация | Комментарии | Центр безопасности |
|-|----|--|
| Шифрование дисков операционной системы. | [Шифрование azure Disk](../articles/security/azure-security-disk-encryption-overview.md) помогает шифровать диски Windows и Linux IaaS VM. Без необходимых ключей содержимое зашифрованных дисков нечитаемо. Шифрование диска защищает сохраненные данные от несанкционированного доступа, что в противном случае было бы возможно, если диск был скопирован.| [Да](../articles/security-center/security-center-apply-disk-encryption.md) |
| Шифрование дисков данных. | [Шифрование azure Disk](../articles/security/azure-security-disk-encryption-overview.md) помогает шифровать диски Windows и Linux IaaS VM. Без необходимых ключей содержимое зашифрованных дисков нечитаемо. Шифрование диска защищает сохраненные данные от несанкционированного доступа, что в противном случае было бы возможно, если диск был скопирован.| -  |
| Ограничьте установленное программное обеспечение. | Ограничьте установленное программное обеспечение тем, что требуется для успешного применения решения. Это руководство помогает уменьшить атаку поверхности вашего решения. | - |
| Используйте антивирус или антивредоносные программы. | В Azure вы можете использовать антивредоносное программное обеспечение от поставщиков безопасности, таких как Microsoft, Symantec, Trend Micro и Kaspersky. Это программное обеспечение помогает защитить ваши vMs от вредоносных файлов, рекламных программ и других угроз. Развертывание Microsoft Antimalware можно на основе рабочих нагрузок приложений. Используйте либо базовую безопасную по умолчанию, либо расширенную пользовательскую конфигурацию. Для получения дополнительной информации [см.](../articles/security/azure-security-antimalware.md) | - |
| Безопасно хранить ключи и секреты. | Упростите управление своими секретами и ключами, предоставив владельцам приложений безопасный, централизованно управляемый вариант. Такое управление снижает риск случайного компромисса или утечки. Azure Key Vault может надежно хранить ключи в модулях безопасности оборудования (HSMs), сертифицированных на уровне FIPS 140-2. Если вам нужно использовать FIPs 140.2 Уровень 3 для хранения ключей и секретов, вы можете использовать [Azure Dedicated HSM.](../articles/dedicated-hsm/overview.md) | - |

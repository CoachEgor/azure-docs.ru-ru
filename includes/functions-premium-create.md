1. Откройте портал Azure от[https://portal.azure.com](https://portal.azure.com)

1. Нажмите кнопку **Создать ресурс**.

    ![Создайте ресурс](./media/functions-create-function-app-portal/function-app-create-resource.png)

1. Выберите приложение **вычислить** > **функции**.

    ![Создание приложения-функции на портале Azure](./media/functions-premium-create/function-app-create-start.png)

1. Используйте настройки приложения-функции, указанные в таблице под рисунком.

    ![Основы](./media/functions-premium-create/function-app-create-basics.png)

    | Параметр      | Рекомендуемое значение  | Описание |
    | ------------ | ---------------- | ----------- |
    | **Подписка** | Ваша подписка | Подписка, в которой создано приложение-функция. |
    | **[Ресурсная группа](../articles/azure-resource-manager/management/overview.md)** |  *myResourceGroup* | Имя новой группы ресурсов, в которой создается приложение-функция. |
    | **Имя приложения-функции** | Глобально уникальное имя | Имя, которое идентифицирует ваше новое приложение-функцию. Допустимые символы: `a-z` (без учета регистра), `0-9`и `-`.  |
    |**Публикация**| Код | Параметр для публикации файлов кода или контейнера Docker. |
    | **Стек среды выполнения** | Предпочитаемый язык | Выберите среду выполнения, которая поддерживает нужный функциональный язык программирования. Выберите **.NET** для функций СИ и ФЗ. |
    |**Регионе**| Предпочтительный регион | Выберите [область](https://azure.microsoft.com/regions/) рядом с вами или рядом с другими службами, к ним доступ к функциям. |

    Выберите **следующую кнопку хостинга >.**

1. Введите следующие настройки хостинга.

    ![Размещение](./media/functions-premium-create/function-app-premium-create-hosting.png)

    | Параметр      | Рекомендуемое значение  | Описание |
    | ------------ | ---------------- | ----------- |
    | **[Учетная запись хранилища](../articles/storage/common/storage-account-create.md)** |  Глобально уникальное имя |  Создайте учетную запись хранения для использования приложением-функцией. Имя учетной записи хранения должно содержать от 3 до 24 символов и состоять только из цифр и строчных букв. Можно также использовать существующую учетную запись при условии, что она соответствует [требованиям учетной записи хранилища](../articles/azure-functions/functions-scale.md#storage-account-requirements). |
    |**Операционная система**| Предпочтительная операционная система | Операционная система предварительно выбирается с учетом выбранного стека среды выполнения, но при необходимости ее можно изменить. |
    | **[План](../articles/azure-functions/functions-scale.md)** | Premium | Для типа плана выберите **Premium (Preview)** и выберите значения по умолчанию для *Плана Windows* и Sku и выбора *размера.* |

    Выберите следующую кнопку **мониторинга >.**

1. Введите следующие настройки мониторинга.

    ![Наблюдение](./media/functions-create-function-app-portal/function-app-create-monitoring.png)

    | Параметр      | Рекомендуемое значение  | Описание |
    | ------------ | ---------------- | ----------- |
    | **[Application Insights](../articles/azure-functions/functions-monitoring.md)** | Значение по умолчанию | Создает ресурс Application Insights с таким же *именем приложения* в ближайшем поддерживаемом регионе. Развернув этот параметр, можно задать **новое имя ресурса** или выбрать другое **расположение** в [географическом регионе Azure](https://azure.microsoft.com/global-infrastructure/geographies/), где будут хранится данные. |

    Выберите **Просмотр и создание **, чтобы просмотреть выбранные параметры конфигурации приложения.

1. Выберите **Создать**, чтобы подготовить и развернуть приложение-функцию.

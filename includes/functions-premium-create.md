1. Перейдите на [портал Azure](https://portal.azure.com).

1. Выберите **+ создать ресурс** с левой стороны, а затем выберите приложение- **функция**.

1. В качестве **плана размещения**выберите **план службы приложений**, а затем выберите **план службы приложений или расположение**.

    ![Создание приложения-функции](./media/functions-premium-create/create-function-app-resource.png)

1. Выберите **создать**, введите имя **плана службы приложений** , выберите **Расположение** в расположении [рядом с](https://azure.microsoft.com/regions/) вами или рядом с другими службами, к которым ваши функции обращаются, а затем выберите **ценовая**Категория.

    ![Создание плана службы приложений](./media/functions-premium-create/new-app-service-plan.png)

1. Выберите план **EP1** (эластичный Premium) и нажмите кнопку **Применить**.

    ![Выбор плана Premium](./media/functions-premium-create/hosting-plan.png) 

1. Нажмите кнопку **ОК** , чтобы создать план, а затем используйте оставшиеся параметры приложения функции, как указано в таблице под изображением. 

    ![Завершенный план службы приложений](./media/functions-premium-create/create-function-app.png)  

    | Параметр      | Предлагаемое значение  | Описание                                        |
    | ------------ |  ------- | -------------------------------------------------- |
    | **Имя приложения** | Глобально уникальное имя | Имя, которое идентифицирует ваше новое приложение-функцию. Допустимые символы: `a-z`, `0-9` и `-`.  | 
    | **Подписка** | Ваша подписка | Подписка, в которой создано приложение-функция. |
    | **[Группа ресурсов](../articles/azure-resource-manager/resource-group-overview.md)** |  myResourceGroup | Имя новой группы ресурсов, в которой создается приложение-функция. Можно также использовать предлагаемое значение. |
    | **ОС** | Предпочтительная ОС | В плане Premium поддерживаются Linux и Windows. |
    | **Стек среды выполнения** | Предпочитаемый язык | Выберите среду выполнения, которая поддерживает нужный функциональный язык программирования. Выберите **.NET** для функций C# и F#. Отображаются только языки, поддерживаемые выбранной **ОС** . |
    | **[Хранилище](../articles/storage/common/storage-quickstart-create-account.md)** |  Глобально уникальное имя |  Создайте учетную запись хранения для использования приложением-функцией. Имя учетной записи хранения должно содержать от 3 до 24 символов и состоять только из цифр и строчных букв. Можно также использовать существующую учетную запись при условии, что она соответствует [требованиям учетной записи хранилища](../articles/azure-functions/functions-scale.md#storage-account-requirements). |
    | **[Application Insights](../articles/azure-functions/functions-monitoring.md)** | Значение по умолчанию | Создает ресурс Application Insights с таким же *именем приложения* в ближайшем поддерживаемом регионе. Развернув этот параметр, можно задать **новое имя ресурса** или выбрать другое **расположение** в [географическом регионе Azure](https://azure.microsoft.com/global-infrastructure/geographies/), где будут хранится данные. |

1. После проверки параметров нажмите кнопку **создать**.

1. Выберите значок уведомления в правом верхнем углу портала. Вы должны увидеть сообщение **Развертывание выполнено**.

    ![Определение параметров нового приложения-функции](./media/functions-premium-create/function-app-create-notification.png)

1. Выберите **Перейти к ресурсу** для просмотра нового приложения-функции. Можно также установить флажок **Закрепить на панели мониторинга**. Это упростит возвращение к этому ресурсу приложения-функции из панели мониторинга.

Общий галерей изображений позволяют изображений общего ресурса, с помощью RBAC. RBAC можно использовать для совместного использования образов в вашем клиенте и даже для отдельных пользователей за пределами вашего клиента. Но если вы хотите поделиться образов за пределами клиента Azure, в масштабе, необходимо создать регистрацию приложения для облегчения совместного использования.  Использование регистрацию приложения позволяет реализовать более сложные сценарии общего доступа, например: 

* Управление общими изображений, когда одна компания приобретает другую и инфраструктуры Azure, распределяются между раздельные клиенты. 
* Партнеры Azure управлять инфраструктурой Azure от имени своих клиентов. Настройка образов осуществляется в рамках клиента партнеров, но будет происходить развертывания инфраструктуры в клиенте. 


## <a name="create-the-app-registration"></a>Создание регистрации приложения

Создайте регистрацию приложения, который будет использоваться в обоих клиентов для совместного использования ресурсов образа из коллекции.
1. Откройте [регистрация приложений (Предварительная версия) на портале Azure](https://ms.portal.azure.com/#blade/Microsoft_AAD_RegisteredApps/ApplicationsListBlade/quickStartType//sourceType/).    
1. Выберите **Регистрация нового** меню в верхней части страницы.
1. В **имя**, тип *myGalleryApp*.
1. В **поддерживаемые типы учетных записей**выберите **учетные записи в любой каталог организации и личных учетных записей Майкрософт**.
1. В **URI перенаправления**, тип *https://www.microsoft.com* , а затем выберите **зарегистрировать**. После регистрации приложения будет создан, откроется на странице обзора.
1. На странице "Обзор", скопируйте **идентификатор приложения (клиент)** и сохранить для использования позже.   
1. Выберите **сертификаты и секреты**, а затем выберите **новый секрет клиента**.
1. В **описание**, тип *общий секрет приложения между клиентами коллекции изображений*.
1. В **Expires**, оставьте значение по умолчанию **через один год** , а затем выберите **добавить**.
1. Скопируйте значение секрета и сохраните его в безопасном месте. Его нельзя будет восстановить после ухода со страницы.


Предоставление разрешения регистрации приложения на использование общего образа коллекции.
1. На портале Azure выберите коллекции образов Shared, который вы хотите поделиться с другого клиента.
1. Выберите **выберите элемент управления доступом (IAM)**, а затем в разделе **добавить назначение роли** выберите *добавить*. 
1. В разделе **роли**выберите **чтения**.
1. В разделе **назначать доступ к:**, оставьте это как **пользователя, группы или субъекта-службы Azure AD**.
1. В разделе **выберите**, тип *myGalleryApp* и выберите его, когда оно появится в списке. Когда все будет готово, выберите **Сохранить**.


## <a name="give-tenant-2-access"></a>Предоставить доступ клиента 2

Предоставьте доступ клиент 2 к приложению, запрашивая вход в систему с помощью браузера. Замените *<Tenant2 ID>* для клиента, который вы хотите поделиться коллекции образов с соответствующим идентификатором клиента. Замените *< идентификатор приложения (клиент) >* с Идентификатором зарегистрированного приложения, созданного приложения. После завершения замены, вставьте URL-адрес в адресную строку браузера и следуйте инструкциям на экране входа в систему для входа в клиент 2.

```
https://login.microsoftonline.com/<Tenant 2 ID>/oauth2/authorize?client_id=<Application (client) ID>&response_type=code&redirect_uri=https%3A%2F%2Fwww.microsoft.com%2F 
```

В [портала Azure](https://portal.azure.com) войдите в систему как клиент 2 и предоставить доступ регистрации приложения в группу ресурсов, в которой вы хотите создать виртуальную Машину.

1. Выберите группу ресурсов, а затем выберите **управление доступом (IAM)**. В разделе **добавить назначение роли** выберите **добавить**. 
1. В разделе **роли**, тип **участник**.
1. В разделе **назначать доступ к:**, оставьте это как **пользователя, группы или субъекта-службы Azure AD**.
1. В разделе **выберите** тип *myGalleryApp* затем выберите его, когда оно появится в списке. Когда все будет готово, выберите **Сохранить**.

> [!NOTE]
> Необходимо дождаться версию образа, чтобы полностью завершить сборки и репликацию перед один и тот же управляемый образ можно использовать для создания другой версии образа.

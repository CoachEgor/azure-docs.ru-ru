Но, если вы хотите поделиться изображениями за пределами арендатора Azure, в масштабе, вы должны создать регистрацию приложения для облегчения совместного использования.  Использование регистрации приложений может включать более сложные сценарии совместного использования, такие как: 

* Управление общими изображениями, когда одна компания приобретает другую, а инфраструктура Azure распределяется между отдельными арендаторами. 
* Партнеры Azure управляют инфраструктурой Azure от имени своих клиентов. Настройка изображений осуществляется в составе арендатора партнеров, но развертывание инфраструктуры будет происходить в клиенте. 


## <a name="create-the-app-registration"></a>Создание регистрации приложения

Создайте регистрацию приложений, которая будет использоваться обоими арендаторами для совместного использования ресурсов галереи изображений.
1. Откройте [регистрацию приложений (предварительный просмотр) на портале Azure](https://ms.portal.azure.com/#blade/Microsoft_AAD_RegisteredApps/ApplicationsListBlade/quickStartType//sourceType/).    
1. Выберите **новую регистрацию** из меню в верхней части страницы.
1. В **имя**, введите *myGalleryApp*.
1. В **поддерживаемых типах учетных записей**выберите **учетные записи в любом каталоге организаций и личных учетных записей Майкрософт.**
1. В **перенаправить URI,** введите, *https://www.microsoft.com* а затем выберите **Регистр.** После создания регистрации приложения страница обзора откроется.
1. На странице обзора скопируйте **идентификатор приложения (клиента)** и сохраните его для использования позже.   
1. Выберите **Сертификаты & секреты,** а затем выберите **новый секрет клиента.**
1. В **Описании**, *введите Общие изображения галерея кросс-арендатор приложение секрет*.
1. В **expires,** оставьте по умолчанию **в 1 год,** а затем выберите **Добавить**.
1. Копируйте значение секрета и сохраните его в безопасное место. Вы не можете получить его после того, как вы покидаете страницу.


Дайте приложению разрешение на регистрацию приложения, чтобы использовать общую галерею изображений.
1. На портале Azure выберите общую галерею изображений, которой вы хотите поделиться с другим арендатором.
1. Выберите **элемент управления доступом (IAM)** и под **назначением роли Добавить** *выберите Добавить.* 
1. Под **ролью**, выберите **Reader**.
1. При **назначайте доступ к:**, оставьте это в качестве **пользователя Azure AD, группы или основного сервиса.**
1. Под **Выберите**, введите *myGalleryApp* и выберите его, когда он появляется в списке. По завершении нажмите кнопку **Сохранить**.


## <a name="give-tenant-2-access"></a>Предоставь арендатору 2 доступ

Предоставь Арендатору 2 доступ к приложению, запросив вход с помощью браузера. Замените * \<Tenant2 ID>* идентификатором арендатора для арендатора, с которым вы хотели бы поделиться своей картинной галереей. Замените * \<ID->приложения (клиента)* идентификатором приложения, который вы создали. После выполнения замены, вставьте URL в браузер и следуйте подсказкам для вхаски для вхастывки в Tenant 2.

```
https://login.microsoftonline.com/<Tenant 2 ID>/oauth2/authorize?client_id=<Application (client) ID>&response_type=code&redirect_uri=https%3A%2F%2Fwww.microsoft.com%2F 
```

На [портале Azure](https://portal.azure.com) вопийте как Tenant 2 и предоставим регистрации приложения группе ресурсов, где вы хотите создать VM.

1. Выберите группу ресурсов, а затем выберите **элемент управления доступом (IAM).** При **добавлении назначения роли** выберите **Добавить**. 
1. Под **ролью**, **введите Вкладчик**.
1. При **назначайте доступ к:**, оставьте это в качестве **пользователя Azure AD, группы или основного сервиса.**
1. Под **выберите** тип *myGalleryApp* затем выберите его, когда он появляется в списке. По завершении нажмите кнопку **Сохранить**.

> [!NOTE]
> Вам нужно ждать, пока версия изображения полностью закончит строиться и реплицироваться, прежде чем вы сможете использовать то же управляемое изображение для создания другой версии изображения.

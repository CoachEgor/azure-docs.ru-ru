## <a name="sign-in-to-azure"></a>Вход в Azure 

Войдите на портал Azure по адресу https://portal.azure.com.

> [!NOTE]
> Если вы зарегистрировались для использования общих галерей изображений `Microsoft.Compute` во время предварительного просмотра, возможно, потребуется перерегистрировать поставщика. Откройте [облачную оболочку](https://shell.azure.com/bash) и введите:`az provider register -n Microsoft.Compute`

## <a name="create-an-image-gallery"></a>Создание коллекции образов

Коллекция образов является основным ресурсом, который позволяет обмен изображениями. Допустимыми знаками для имени коллекции являются прописные или строчные буквы, цифры и точки. Название галереи не может содержать тире.  Имена коллекций должны быть уникальным в пределах вашей подписки. 

В следующем примере показано, как создать коллекцию с именем *myGallery* в группе ресурсов *myGalleryRG*.

1. Щелкните **Создать ресурс** в верхнем левом углу окна портала Azure.
1. Используйте **галерею изображений типа «Общий»** в поле поиска и выберите **общую галерею изображений** в результатах.
1. На странице **галереи общих изображений** нажмите **«Создать».**
1. Выберите правильную подписку.
1. В **группе ресурсов**выберите **Создать новый** и введите *myGalleryRG* для имени.
1. В **названии**, введите *myGallery* для названия галереи.
1. Оставьте значение по умолчанию для **региона.**
1. Вы можете ввести краткое описание галереи, как *моя галерея изображений для тестирования.* а затем нажмите **Обзор и создать**.
1. После проверки проходит, выберите **Создать**.
1. После завершения развертывания выберите **Перейти к ресурсу**.
   
## <a name="create-an-image-definition"></a>Создание определения образа 

Определения изображений создают логическую группировку изображений. Они используются для управления информацией о версиях изображений, которые создаются в них. Имена определения изображения могут состоять из верхних или нижних букв, цифр, точек, тире и периодов. Для получения дополнительной информации о значениях, которые вы можете указать для определения изображения, [см.](https://docs.microsoft.com/azure/virtual-machines/windows/shared-image-galleries#image-definitions)

Создайте определение изображения галереи внутри галереи. В этом примере изображение галереи называется *myImageDefinition*.

1. На странице для новой галереи изображений выберите **Добавить новое определение изображения** из верхней части страницы. 
1. Для **названия определения изображения,** введите *myImageDefinition*.
1. Для **операционной системы**выберите правильный вариант на основе вашего источника VM.
1. Для **поколения VM**выберите опцию на основе вашего источника VM. В большинстве случаев это будет *Gen 1*. Для получения дополнительной информации [см.](https://docs.microsoft.com/azure/virtual-machines/windows/generation-2)
1. Для **состояния операционной системы**выберите опцию на основе исходного источника VM. Для получения дополнительной информации [см.](../articles/virtual-machines/linux/shared-image-galleries.md#generalized-and-specialized-images)
1. Для **издателя**, введите *myPublisher*. 
1. Для **предложения,** введите *myOffer*.
1. Для **SKU**, введите *mySKU*.
1. После завершения, выберите **Обзор и создать**.
1. После того, как определение изображения пройдет проверку, выберите **Create**.
1. После завершения развертывания выберите **Перейти к ресурсу**.


## <a name="create-an-image-version"></a>Создание версии образа

Создайте версию изображения из управляемого изображения. В этом примере используется версия образа *1.0.0*, которая реплицируется в центры обработки данных в регионах *центрально-западная часть США* и *центрально-южная часть США*. При выборе целевых областей для репликации помните, что в качестве цели для репликации необходимо также включить область *исходного кода.*

Допустимыми знаками для имени версии образа являются цифры и точки. Числа должны быть в диапазоне 32-битного целого числа. Формат: *MajorVersion*. *MinorVersion*. *Патч*.

Шаги для создания версии изображения немного отличаются, в зависимости от того, является ли источник обобщенным изображением или снимком специализированного VM. 

### <a name="option-generalized"></a>Вариант: Обобщенный

1. На странице для определения изображения выберите **Добавление версии** из верхней части страницы.
1. В **регионе**выберите регион, в котором хранится управляемое изображение. Версии изображений должны создаваться в том же регионе, что и управляемое изображение, из которых они создаются.
1. Для **имени**, тип *1.0.0*. Имя версии изображения должно следовать *основным*. *несовершеннолетний*. *формат патча* с использованием многоразовых. 
1. На **исходном изображении**выберите управляемое исходное изображение из выпадения.
1. В **Исключении из последнего,** оставьте значение по умолчанию *No*.
1. Для **даты окончания жизни**выберите дату из календаря, которая состоится на пару месяцев в будущем.
1. В **репликации,** оставьте **значение реплики по умолчанию как** 1. Вам нужно воспроизвести к зоне источника, поэтому выйдьте первую реплику как значение значения по умолчанию, а затем выберите второй регион реплики, чтобы быть *Восточной США.*
1. Когда вы закончите, выберите **Обзор и создать**. Azure проверит конфигурацию.
1. Когда версия изображения проходит проверку, выберите **Create**.
1. После завершения развертывания выберите **Перейти к ресурсу**.

Это может занять некоторое время, чтобы воспроизвести изображение во всех целевых регионах.

### <a name="option-specialized"></a>Вариант: Специализированный

1. На странице для определения изображения выберите **Добавление версии** из верхней части страницы.
1. В **регионе**выберите регион, где хранится снимок. Версии изображений должны создаваться в том же регионе, из которых они создаются.
1. Для **имени**, тип *1.0.0*. Имя версии изображения должно следовать *основным*. *несовершеннолетний*. *формат патча* с использованием многоразовых. 
1. В **моментальном снимке диска ОС**выберите снимок из исходного VM из выпадения вниз. Если у вашего источника VM был диск данных, который вы хотели бы включить, выберите правильный номер **LUN** из выпадения вниз, а затем выберите снимок диска данных для **снимка диска данных.** 
1. В **Исключении из последнего,** оставьте значение по умолчанию *No*.
1. Для **даты окончания жизни**выберите дату из календаря, которая состоится на пару месяцев в будущем.
1. В **репликации,** оставьте **значение реплики по умолчанию как** 1. Вам нужно воспроизвести к зоне источника, поэтому выйдьте первую реплику как значение значения по умолчанию, а затем выберите второй регион реплики, чтобы быть *Восточной США.*
1. Когда вы закончите, выберите **Обзор и создать**. Azure проверит конфигурацию.
1. Когда версия изображения проходит проверку, выберите **Create**.
1. После завершения развертывания выберите **Перейти к ресурсу**.

## <a name="share-the-gallery"></a>Поделитесь галереей

Мы рекомендуем вам поделиться доступом на уровне галереи изображений. Следующие прогулки вас через обмен галерея, что вы только что создали.

1. Откройте [портал Azure](https://portal.azure.com).
1. В меню слева выберите **группы ресурсов.** 
1. В списке групп ресурсов выберите **myGalleryRG.** Лопасти для вашей группы ресурсов откроется.
1. В меню слева от страницы **myGalleryRG** выберите **элемент управления доступом (IAM).** 
1. При **добавлении назначения роли**выберите **Добавить**. Откроется панель **назначения ролей.** 
1. Под **ролью**, выберите **Reader**.
1. При **присвоении доступа к,** оставьте по умолчанию **пользователя Azure AD, группы или основного сервиса.**
1. В **соответствии**с Select введите адрес электронной почты человека, которого вы хотели бы пригласить.
1. Если пользователь находится за пределами организации, вы увидите сообщение, которое пользователю **будет отправлено по электронной почте, которое позволит ему сотрудничать с корпорацией Майкрософт.** Выберите пользователя с адресом электронной почты, а затем нажмите **Сохранить**.

Если пользователь находится за пределами вашей организации, он получит приглашение по электронной почте присоединиться к организации. Пользователь должен принять приглашение, после чего он сможет увидеть галерею и все определения изображения и версии в своем списке ресурсов.

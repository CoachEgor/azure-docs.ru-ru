#### <a name="applications"></a>[Приложения](#tab/applications/)

1. На странице **обзора зарегистрированного приложения** выберите **Настройки.**
1. В соответствии **с доступом API**выберите **требуемые разрешения.**
1. Выберите **Microsoft Graph**.
1. В соответствии с разрешениями на **приложение,** выберите флажок разрешения на предоставление вашего приложения управления. Пример:
    * **Прочитайте все данные журнала аудита:** Выберите это разрешение для чтения журналов аудита каталога.
    * **Прочитайте и напишите данные каталога:** Выберите это разрешение для сценариев миграции пользователей или управления пользователем.
    * **Прочитайте и напишите политики рамочной базы доверия организации:** Выберите это разрешение для сценариев непрерывной интеграции/непрерывной доставки (CI/CD). Например, развертывание пользовательских политик с Azure Pipelines.
1. Нажмите кнопку **Сохранить**.
1. Щелкните **Предоставить разрешения** и затем выберите **Да**. Полностью распространение разрешений может занять несколько минут.

#### <a name="app-registrations-preview"></a>[Регистрация приложений (предварительная версия)](#tab/app-reg-preview/)

1. В разделе **Управление** выберите **Разрешения API**.
1. В разделе **Настроенные разрешения** выберите **Добавить разрешение**.
1. Выберите вкладку **Microsoft AA,** а затем выберите **Microsoft Graph.**
1. Выберите **Разрешения приложений**.
1. Расширьте соответствующую группу разрешений и выберите флажок разрешения на предоставление вашего приложения управления. Пример:
    * **AuditLog** > **AuditLog.Read.All**: Для чтения журналов аудита каталога.
    * **Каталог** > **Каталог.ReadWrite.All**: Для миграции пользователей или сценариев управления пользователем.
    * **Политика** > **Policy.ReadWrite.TrustFramework:** Для непрерывной интеграции / непрерывной доставки (CI/CD) сценариев. Например, развертывание пользовательских политик с Azure Pipelines.
1. Выберите **Добавить разрешения**. В соответствии с инструкциями подождите несколько минут, прежде чем перейти к следующему шагу.
1. Выберите **Предоставить согласие администратора для (имя арендатора)**.
1. Выберите учетную запись администратора, с которой выполнен вход, или выполните вход с учетной записью в арендаторе Azure AD B2C, которой назначена по крайней мере роль *Администратор облачных приложений*.
1. Нажмите кнопку **Принять**.
1. Выберите **Обновить**, а затем проверить, что "Предоставлено для ..." отображается в соответствии со **статусом**. Распространение разрешений может занять несколько минут.

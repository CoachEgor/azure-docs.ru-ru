При создании классических виртуальных сетей на портале Azure имя, которое вы просматриваете, не является полным именем, используемым для PowerShell. Например, если виртуальная сеть отображается на портале с именем **TestVNet1**, то в файле конфигурации сети ее имя может быть гораздо длиннее. Это имя может выглядеть примерно следующим образом: **Group ClassicRG TestVNet1**. При создании подключений важно использовать значения, приведенные в файле конфигурации сети.

Выполняя следующие действия, вы подключитесь к учетной записи Azure, а также скачаете и просмотрите файл конфигурации сети для получения значений, необходимых для подключений.

1. Скачайте и установите последнюю версию командлетов PowerShell для управления службами Azure. Большинство пользователей имеют диспетчер ресурсовные модули, установленные локально, но не имеют модулей управления службами. Модули управления службами являются устаревшими и должны устанавливаться отдельно. Дополнительные сведения см. в разделе [Install Service Management командлеты](/powershell/azure/servicemanagement/install-azure-ps).

1. Откройте консоль PowerShell с повышенными правами и подключитесь к своей учетной записи. Используйте следующие примеры для подключения. Эти команды необходимо выполнять локально с помощью модуля управления службами PowerShell. Подключитесь к учетной записи. Для подключения используйте следующий пример кода:

   ```powershell
   Add-AzureAccount
   ```
1. Просмотрите подписки учетной записи.

   ```powershell
   Get-AzureSubscription
   ```
1. При наличии нескольких подписок выберите подписку, которую вы хотите использовать.

   ```powershell
   Select-AzureSubscription -SubscriptionId "Replace_with_your_subscription_ID"
   ```
1. Экспортируйте и просмотрите файл конфигурации сети. Создайте каталог на компьютере, а затем экспортируйте в него файл конфигурации сети. В этом примере файл конфигурации сети экспортируется в каталог **C:\AzureNet**.

   ```powershell
   Get-AzureVNetConfig -ExportToFile C:\AzureNet\NetworkConfig.xml
   ```
1. Откройте файл в текстовом редакторе и просмотрите имена для виртуальных сетей и сайтов. Эти имена будут именами, используемыми при создании подключений.<br>Имена виртуальных сетей перечислены как **VirtualNetworkSite name =**.<br>Имена сайтов перечислены как **LocalNetworkSiteRef name =**.

Чтобы максимизировать производительность, используйте отдельный учет хранения для каждого приложения функции. Это особенно важно, когда у вас есть функции «Долговечные» или функции Event Hub, которые генерируют большой объем транзакций хранения данных. Когда логика приложения взаимодействует с Хранилищем Azure, либо непосредственно (с помощью SDK хранилища), либо через одну из привязок хранилища, следует использовать специальную учетную запись хранения. Например, если у вас есть функция Event Hub-triggered, пишущая&mdash;некоторые данные для хранения кабы, используйте две учетные записи хранилища, одна для приложения функции, а другая для капли, хранящиеся в функции.

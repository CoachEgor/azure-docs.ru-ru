Ниже перечислены ограничения по умолчанию для Azure общего назначения версии 1, 2 и учетные записи хранения BLOB-объектов. *Ingress* ограничение относится ко всем данным из запросов, которые отправляются в учетную запись хранения. *Исходящий трафик* ограничение относится ко всем данным из ответов, полученных от учетной записи хранения.

| Ресурс | Ограничение по умолчанию |
| --- | --- |
| Количество учетных записей хранения на регион или подписку, включая учетные записи ценовых категорий "Стандартный" и "Премиум". | 250 |
| Максимальная емкость учетной записи | 2 PB для жителей США и Европе, 500 ТБ для всех других регионов, включая Великобритании |
| Максимальное число контейнеров больших двоичных объектов, большие двоичные объекты, файловые ресурсы, таблицы, очереди, сущностей или сообщений в учетную запись хранения | Без ограничений |
| Максимальная частота запросов<sup>1</sup> на учетную запись хранения | 20 000 запросов в секунду |
| Входящие<sup>1</sup> на учетную запись хранения (регионы США) | 10 Гбит/с, если включено RA-GRS/GRS, 20 Гбит/с для хранилищ LRS и ZRS<sup>2</sup> |
| Входящие<sup>1</sup> на учетную запись хранения (регионы за пределами США) | 5 Гбит/с, если включено RA-GRS/GRS, 10 Гбит/с для хранилищ LRS и ZRS<sup>2</sup> |
| Исходящего трафика для общего назначения версии 2 и учетные записи хранения BLOB-объектов (все регионы) | 50 Гбит/с |
| Исходящего трафика для учетных записей хранения общего назначения версии 1 (регионы США) | 20 Гбит/с, если включено RA-GRS/GRS, 30 Гбит/с для хранилищ LRS и ZRS<sup>2</sup> |
| Исходящего трафика для учетных записей хранения общего назначения версии 1 (регионы за пределами США) | 10 Гбит/с, если включено RA-GRS/GRS, 15 Гбит/с для хранилищ LRS и ZRS<sup>2</sup> |

<sup>1</sup>учетных записей хранилища azure класса Standard, которые поддерживаются более высокие пределы для обработки входящего запроса. Чтобы подать запрос на увеличение скорости передачи входящих данных для учетной записи, обратитесь в [службу поддержки Azure](https://azure.microsoft.com/support/faq/).

<sup>2</sup> Параметры [репликации службы хранилища Azure](https://docs.microsoft.com/azure/storage/common/storage-redundancy):
* **RA-GRS** — геоизбыточное хранилище с доступом на чтение. Если параметр RA-GRS включен, входящие целевые объекты для вторичного расположения идентичны целевым объектам для первичного расположения.
* **GRS** — геоизбыточное хранилище. 
* **ZRS** — хранилище, избыточное между зонами.
* **LRS** — Локально избыточное хранилище. 

> [!NOTE]
> Мы рекомендуем использовать учетную запись хранения общего назначения версии 2 для большинства сценариев. Можно легко обновить общего назначения версии 1 или учетную запись хранилища BLOB-объектов Azure в учетную запись общего назначения версии 2 без простоя и без необходимости копирования данных.
>
> Дополнительные сведения об учетных записях хранения Azure см. в разделе [учетной записи хранения](../articles/storage/common/storage-account-overview.md). 

Если предельно достижимых показателей масштабируемости для одной учетной записи хранения недостаточно для работы приложения, то при разработке такого приложения его можно настроить на использование нескольких учетных записей и распределить объекты данных между ними. Сведения о ценах см. в разделе [цены на хранилища Azure](https://azure.microsoft.com/pricing/details/storage/).

Все учетные записи хранения функционируют в плоской топологии сети и поддерживают описанные ниже целевые показатели масштабируемости и производительности независимо от того, когда они были созданы. Дополнительные сведения о неструктурированной сетевой архитектуре службы хранилища Azure и масштабируемости см. в статье [SOSP Paper – Windows Azure Storage: A Highly Available Cloud Storage Service with Strong Consistency](http://blogs.msdn.com/b/windowsazurestorage/archive/2011/11/20/windows-azure-storage-a-highly-available-cloud-storage-service-with-strong-consistency.aspx) (Хранилище Microsoft Azure: высокодоступная служба облачного хранения со строгой согласованностью).

Конвейер выполняется с целевым объектом вычислений, который является вычислительным ресурсом, подключенным к вашей рабочей области. После создания целевого объекта вычислений его можно повторно использовать при последующих запусках.

1. Выберите **Запуск** в верхней части холста, чтобы запустить конвейер.

1. В появившейся области **Параметры** щелкните **Select compute target** (Выбор целевого объекта вычислений).

    Если у вас уже есть доступный целевой объект вычислений, его можно выбрать для запуска этого конвейера.

    > [!NOTE]
    > Конструктор может выполнять эксперименты только с целевыми объектами вычислений для машинного обучения. Другие целевые объекты вычислений не отображаются.

1. Укажите имя для вычислительного ресурса.

1. Щелкните **Сохранить**.

    ![Настройка целевого объекта вычислений](./media/aml-ui-create-training-compute/set-compute.png)

1. Выберите **Запуск**.

1. В диалоговом окне **Set up pipeline run** (Настройка выполнения конвейера) для параметра **Эксперимент** выберите **+ New experiment** (Новый эксперимент).

    > [!NOTE]
    > Эксперименты группы аналогичных конвейеров выполняются вместе. Если вы запускаете конвейер несколько раз, то можете выбрать один и тот же эксперимент для последовательных запусков.

    * Введите описательное имя в поле **Имя эксперимента**.

    * Выберите **Выполнить**.
    
    Состояние выполнения можно просмотреть в правой верхней части холста.

    > [!NOTE]
    > Создание вычислительного ресурса занимает порядка 5 минут. После создания ресурса его можно повторно использовать, экономя время при последующих запусках.
    >
    > Вычислительный ресурс автомасштабируется до 0 узлов во время простоя для экономии затрат.  При повторном его использовании после простоя придется снова ждать около 5 минут, пока выполняется масштабирование ресурса.

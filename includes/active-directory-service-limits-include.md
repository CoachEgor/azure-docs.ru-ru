Ниже перечислены ограничения на использование и другие ограничения для службы Azure Active Directory (Azure AD).

| Категория | Ограничение |
| --- | --- |
| Каталоги | Один пользователь может принадлежать максимум к 500 каталогам Azure AD как член или гость.<br/>Один пользователь может создать не более 200 каталогов. |
| Домены | Можно добавить не более 900 управляемых доменных имен. Если вы настраиваете все домены для федерации с помощью локальной службы Active Directory, в каждый каталог можно добавить до 450 доменных имен. |
|Ресурсы |<ul><li>Можно создать не более 50 000 ресурсов Azure AD в одном каталоге, если пользователи бесплатного выпуска Azure Active Directory по умолчанию. Если у вас есть по крайней мере один проверенный домен, квота службы Azure AD по умолчанию для вашей организации расширяется до 300 000 ресурсов Azure AD. Это ограничение службы не связано с ограничением ценовой категории 500 000 ресурсов на странице цен на Azure AD. Чтобы не выходить за пределы квоты по умолчанию, необходимо обратиться в служба поддержки Майкрософт.</li><li>Пользователь без прав администратора может создавать не более 250 ресурсов Azure AD. Как активные ресурсы, так и удаленные ресурсы, доступные для восстановления в отношении этой квоты. Для восстановления доступны только удаленные ресурсы Azure AD, которые были удалены менее чем за 30 дней назад. Удаленные ресурсы Azure AD, которые больше не доступны для восстановления, преобразуются в эту квоту со значением, равным 1 кварталу, в течение 30 дней. Если у вас есть разработчики, которые, скорее всего, будут постоянно превысить эту квоту в рамках их обычных обязанностей, вы можете [создать и назначить пользовательскую роль](../articles/active-directory/users-groups-roles/roles-quickstart-app-registration-limits.md) с разрешением на создание бесконечного количества регистраций приложений.</li></ul> |
| Расширения схемы |<ul><li>Расширения типа String не должны превышать 256 символов. </li><li>Расширения типа Binary не должны превышать 256 байт.</li><li>В один ресурс Azure AD можно записывать только значения расширения 100, для *всех* типов и *всех* приложений.</li><li>Дополнить однозначными атрибутами типов String или Binary можно только сущности User, Group, TenantDetail, Device, Application и ServicePrincipal.</li><li>Расширения схемы доступны только в предварительной версии API Graph 1.21. Приложению должен быть предоставлен доступ на запись для регистрации расширения.</li></ul> |
| Приложения |Владельцами одного приложения могут быть не более 100 пользователей. |
|Манифест приложения |В манифест приложения можно добавить не более 1200 записей. |
| Группы |<ul><li>Пользователь может создать максимум 250 групп в Организации Azure AD.</li><li>В Организации Azure AD может быть не более 5000 динамических групп.<li>Владельцами одной группы могут быть не более 100 пользователей.</li><li>Любое количество ресурсов Azure AD может быть членом одной группы.</li><li>Пользователь может участником любого количества групп.</li><li>Число участников в группе, которых можно синхронизировать из локальной службы Active Directory в Azure Active Directory с помощью Azure AD Connect, ограничено 50 000.</li><li>Вложенные группы в Azure AD не поддерживаются во всех сценариях.</li></ul><br/> В настоящее время ниже перечислены поддерживаемые сценарии с вложенными группами.<ul><li> Одну группу можно добавить в качестве члена другой группы, чтобы обеспечить вложенность группы.</li><li> Утверждения членства в группах (когда приложение настроено для получения утверждений о членстве в группах в маркере) вложенные группы, в которые входит вошедший пользователь, входят в состав.</li><li>Условный доступ (при области применения политики условного доступа к группе)</li><li>Ограниченный доступ для самостоятельного сброса пароля</li><li>Ограничьте круг пользователей, которые могут присоединяться к Azure AD и регистрацию устройств</li></ul><br/>Следующие сценарии не поддерживают вложенные группы:<ul><li> Назначение ролей приложения (назначение групп для приложения поддерживается, но группы, вложенные в непосредственно назначенную группу, не будут иметь доступа) и для доступа, и для подготовки.</li><li>Лицензирование на основе групп (автоматическое назначение лицензии всем участникам группы)</li><li>Группы Office 365.</li></ul> |
| Прокси приложения | <ul><li>Не более 500 транзакций в секунду на приложение прокси приложения</li><li>Не более 750 транзакций в секунду для Организации Azure AD</li></ul><br/>Транзакция определяется как один HTTP-запрос и ответ для уникального ресурса. При регулировании клиенты получат ответ 429 (слишком много запросов). |
| Панель доступа |<ul><li>Число приложений, которые могут отображаться в Панели доступа для каждого пользователя, не ограничено. Это также относится и к пользователям с назначенными лицензиями Azure AD Premium или Enterprise Mobility Suite.</li><li>Для каждого пользователя в Панели доступа могут отображаться не более 10 плиток приложений. Это ограничение применяется к пользователям, которым назначены лицензии Azure AD Free плане лицензии. Примеры таких плиток приложений: Box, Salesforce или Dropbox. Это ограничение не действует для учетных записей администраторов.</li></ul> |
| Отчеты | В любом отчете можно просматривать и загружать не более 1000 строк. Любые дополнительные данные будут усечены. |
| Административные единицы | Ресурс Azure AD может быть членом не более 30 единиц администрирования. |
| Роли и разрешения администратора | <ul><li>Группу нельзя добавить в качестве [владельца](https://docs.microsoft.com/azure/active-directory/fundamentals/users-default-permissions?context=azure/active-directory/users-groups-roles/context/ugr-context#object-ownership).</li><li>Невозможно назначить группу [роли](https://docs.microsoft.com/azure/active-directory/users-groups-roles/directory-assign-admin-roles).</li><li>Возможность чтения сведений о каталоге других пользователей не может быть ограничена за пределами коммутатора всей Организации Azure AD, чтобы отключить доступ всех пользователей без прав администратора ко всем данным каталога (не рекомендуется). Дополнительные сведения о разрешениях по умолчанию см. [здесь](https://docs.microsoft.com/azure/active-directory/fundamentals/users-default-permissions?context=azure/active-directory/users-groups-roles/context/ugr-context#to-restrict-the-default-permissions-for-member-users).</li><li>Для вступления в силу членства в роли администратора и отзыва пользователей может потребоваться до 15 минут или для входа и входа.</li></ul> |

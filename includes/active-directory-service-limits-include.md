Ниже перечислены ограничения на использование и другие ограничения для службы Azure Active Directory (Azure AD).

| Категория | Ограничение |
| --- | --- |
| Каталоги | Один пользователь может принадлежать максимум к 500 каталогам Azure AD как член или гость.<br/>Один пользователь может создать максимум 20 каталогов. |
| Домены | Можно добавить не более 900 управляемых доменных имен. Если вы настраиваете все домены для федерации с помощью локальной службы Active Directory, в каждый каталог можно добавить до 450 доменных имен. |
|Ресурсы |<ul><li>Не более 50 000 ресурсов Azure AD могут быть созданы в одном каталоге пользователями бесплатного издания Active Directory Azure по умолчанию. Если у вас есть хотя бы один проверенный домен, квота службы AD по умолчанию Azure для вашей организации будет расширена до 300 000 ресурсов Azure AD. Этот лимит услуг не связан с ограничением уровня цен в 500 000 ресурсов на странице ценообразования Azure AD. Чтобы выйти за рамки квоты по умолчанию, необходимо обратиться в службу поддержки Майкрософт.</li><li>Пользователь, не являйся аминтом, может создать не более 250 ресурсов Azure AD. Как активные ресурсы, так и удаленные ресурсы, доступные для восстановления счета к этой квоте. Для восстановления доступны только удаленные ресурсы Azure AD, которые были удалены менее 30 дней назад. Удаленные ресурсы Azure AD, которые больше не доступны для восстановления счета к этой квоте, стоимостью одну четверть в течение 30 дней. Если у вас есть разработчики, которые, вероятно, неоднократно превышают эту квоту в ходе своих обычных обязанностей, вы можете [создать и назначить пользовательскую роль](../articles/active-directory/users-groups-roles/roles-quickstart-app-registration-limits.md) с разрешением на создание безграничного количества регистраций приложений.</li></ul> |
| Расширения схемы |<ul><li>Расширения типа String не должны превышать 256 символов. </li><li>Расширения типа Binary не должны превышать 256 байт.</li><li>Только 100 значений расширения во *всех* типах и *приложениях* могут быть записаны на любой отдельный ресурс Azure AD.</li><li>Дополнить однозначными атрибутами типов String или Binary можно только сущности User, Group, TenantDetail, Device, Application и ServicePrincipal.</li><li>Расширения схемы доступны только в предварительной версии API Graph 1.21. Приложению должен быть предоставлен доступ на запись для регистрации расширения.</li></ul> |
| Приложения |Владельцами одного приложения могут быть не более 100 пользователей. |
|Манифест приложения |Максимум 1200 записей могут быть добавлены в Манифест приложения. |
| Группы |<ul><li>Пользователь может создать не более 250 групп в организации Azure AD.</li><li>Организация Azure AD может иметь не более 5000 динамических групп.<li>Владельцами одной группы могут быть не более 100 пользователей.</li><li>Любое количество ресурсов Azure AD может быть членом одной группы.</li><li>Пользователь может участником любого количества групп.</li><li>Число участников в группе, которых можно синхронизировать из локальной службы Active Directory в Azure Active Directory с помощью Azure AD Connect, ограничено 50 000.</li><li>Nested groups в Azure AD не поддерживаются во всех сценариях</li></ul><br/> В настоящее время ниже приведены поддерживаемые сценарии с вложенными группами.<ul><li> Одна группа может быть добавлена в качестве члена другой группы, и вы можете достичь группового гнездования.</li><li> Претензии членства в группе (когда приложение настроено для получения требований о членстве в группе в токене, вложенные группы, в которые включен пользователь, входящий в систему), включены)</li><li>Условный доступ (при отслеживании политики условного доступа к группе)</li><li>Ограничение доступа к сбросить паролем самообслуживания</li><li>Ограничение того, какие пользователи могут выполнять регистрацию Azure AD и регистрацию устройств</li></ul><br/>Следующие сценарии НЕ поддерживают вложенные группы:<ul><li> Назначение роли приложения (назначение групп в приложение поддерживается, но группы, вложенные в непосредственно назначенную группу, не будут иметь доступа), как для доступа, так и для подготовки</li><li>Групповое лицензирование (автоматическое назначение лицензии всем членам группы)</li><li>Офис 365 групп.</li></ul> |
| Прокси приложения | <ul><li>Максимум 500 транзакций в секунду в приложении App Proxy</li><li>Максимум 750 транзакций в секунду для организации Azure AD</li></ul><br/>Транзакция определяется как единый запрос http и ответ на уникальный ресурс. При задушении клиенты получат 429 ответов (слишком много запросов). |
| Панель доступа |<ul><li>Число приложений, которые могут отображаться в Панели доступа для каждого пользователя, не ограничено. Это также относится и к пользователям с назначенными лицензиями Azure AD Premium или Enterprise Mobility Suite.</li><li>Для каждого пользователя в Панели доступа могут отображаться не более 10 плиток приложений. Это ограничение распространяется на пользователей, которым присваиваются лицензии для программы Azure AD Free. Примеры таких плиток приложений: Box, Salesforce или Dropbox. Это ограничение не действует для учетных записей администраторов.</li></ul> |
| Отчеты | В любом отчете можно просматривать и загружать не более 1000 строк. Любые дополнительные данные будут усечены. |
| Административные единицы | Ресурс Azure AD может быть членом не более 30 административных единиц. |
| Роли и разрешения администратора | <ul><li>Группа не может быть добавлена в качестве [владельца.](https://docs.microsoft.com/azure/active-directory/fundamentals/users-default-permissions?context=azure/active-directory/users-groups-roles/context/ugr-context#object-ownership)</li><li>Группа не может быть назначена на [роль.](https://docs.microsoft.com/azure/active-directory/users-groups-roles/directory-assign-admin-roles)</li><li>Возможность пользователей читать информацию каталогов других пользователей не может быть ограничена за пределами общеорганизационного переключателя Azure AD, чтобы отключить доступ всех пользователей, не являющихся админами, ко всей информации каталога (не рекомендуется). Более подробная информация о разрешениях по умолчанию [здесь](https://docs.microsoft.com/azure/active-directory/fundamentals/users-default-permissions?context=azure/active-directory/users-groups-roles/context/ugr-context#to-restrict-the-default-permissions-for-member-users).</li><li>Это может занять до 15 минут или вступления / вступления в до админ роль членства дополнений и аннулирования вступит в силу.</li></ul> |

В размерах виртуальных машин, оптимизированных для операций в памяти, предоставляется высокое соотношение ресурсов памяти и ядра, что отлично подходит для серверов реляционной базы данных, кэша среднего и большого объема, а также выполняющейся в памяти аналитики. Эта статья содержит сведения о количестве виртуальных ЦП, дисков данных и сетевых адаптеров, а также о пропускной способности хранилища и сети для каждого размера ВМ этой группы.

* В серии Ev3 используется процессор E5-2673 версии 4 с тактовой частотой 2,3 ГГц (Broadwell) и поддержкой технологии Hyper Threading. Серия Ev3 подходит для большей части стандартных рабочих нагрузок и соответствует виртуальным машинам общего назначения большинства других облачных служб.  Объем памяти увеличен с 7 ГиБ до 8 ГиБ на виртуальный ЦП, а дисковые и сетевые ограничения в связи с переходом на технологию Hyper Threading подстроены под каждое ядро.  Ev3 — это следующий уровень виртуальных машин с большим объемом памяти, к которым также относятся виртуальные машины семейств D и Dv2.

* Серии Eav3 и Easv3 используют процессор AMD с частотой 2.35 ГГц ЕПИК<sup>TM</sup> 7452V в многопоточной конфигурации с кэшем объемом до 256 ГБ третьего уровня, повышая уровень возможностей для выполнения большинства оптимизированных для памяти рабочих нагрузок.  Серии Eav3 и Easv3 имеют те же конфигурации памяти и дисков, что и Ev3 & Esv3 Series.

* Серия Mv2 предлагает наибольшее число виртуальных ЦП (до 208 виртуальных ЦП) и максимальный объем памяти (до 5,7 Тиб) любой виртуальной машины в облаке. Это идеальный вариант для очень крупных баз данных или других приложений, которым требуются высокие показатели виртуальной ЦП и памяти.

* Серия M предлагает большое количество виртуальных ЦП (до 128 виртуальных ЦП) и большой объем памяти (до 3,8 Тиб). Он также идеально подходит для очень больших баз данных или других приложений, которые используют высокие показатели виртуальных ЦП и большие объемы памяти.

* Аналоги серии Dv2, G-Series и DSv2/GS идеально подходят для приложений, которые требуют более быстрого виртуальных ЦП, более производительного временного хранилища или имеют более высокие требования к памяти. Они предлагают мощную комбинацию для многих приложений корпоративного уровня.

* Серия Dv2, продолжение оригинальной серии D, отличается более мощным ЦП. Процессор серии Dv2 примерно на 35 % быстрее, чем процессор серии D. Он основан на последних поколениях процессоров Intel Xeon® 2673 с тактовой частотой 2,4 ГГц 2,4 ГГц (Haswell) или 3 v4 2,3 ГГц (Broadwell), а также с технологией Intel Turbo Boost 2,0, может пройти до 3,1 ГГц. Серия Dv2 имеет такие же конфигурации памяти и диска, как и серия D.

* Служба вычислений Azure предлагает размеры виртуальных машин, которые используют оборудование определенного типа и выделяются отдельному клиенту.  Эти размеры лучше всего подходят для рабочих нагрузок, требующих высокого уровня изоляции от других клиентов. Это могут быть рабочие нагрузки, включающие в себя такие элементы, как соблюдение соответствия и нормативных требований.  Клиенты также могут разделить ресурсы этих изолированных виртуальных машин благодаря [поддержке вложенных виртуальных машин в Azure](https://azure.microsoft.com/blog/nested-virtualization-in-azure/).  Ознакомьтесь с таблицами семейств виртуальных машине, чтобы узнать, какие варианты изолированных виртуальных машин доступны.

#### <a name="find-a-suse-image"></a>Поиск образа SUSE

Чтобы выбрать соответствующий образ SUSE Linux в портал Azure: 

1. В портал Azure выберите **создать ресурс** . 
1. Поиск по запросу "SUSE SAP" 
1. SLES для образов SAP поколения 2 доступны как с оплатой по мере использования, так и с помощью собственной подписки (BYOS). В результатах поиска разверните нужную категорию изображений:

    * SUSE Linux Enterprise Server (SLES) для SAP
    * SUSE Linux Enterprise Server (SLES) для SAP (BYOS)
    
1. Для образов SUSE, совместимых с серией Mv2, используется префикс с именем `GEN2:`. Для виртуальных машин серии Mv2 доступны следующие образы SUSE:

    * GEN2 SUSE Linux Enterprise Server (SLES) 12 SP4 для приложений SAP
    * GEN2 SUSE Linux Enterprise Server (SLES) 15 для приложений SAP
    * GEN2 SUSE Linux Enterprise Server (SLES) 12 SP4 для приложений SAP (BYOS)
    * GEN2 SUSE Linux Enterprise Server (SLES) 15 для приложений SAP (BYOS)

#### <a name="select-a-suse-image-via-azure-cli"></a>Выберите образ SUSE с помощью Azure CLI

Чтобы просмотреть список доступных в настоящее время SLES для образа SAP для виртуальных машин серии Mv2, используйте следующую [`az vm image list`](https://docs.microsoft.com/cli/azure/vm/image?view=azure-cli-latest#az-vm-image-list) команду:

```azurecli
az vm image list --output table --publisher SUSE --sku gen2 --all
```

Команда выводит доступные на данный момент виртуальные машины версии 2, доступные в SUSE для виртуальных машин серии Mv2. 

Выходные данные примера:

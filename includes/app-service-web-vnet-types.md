* Мультитенантные системы, которые поддерживают весь спектр ценовых планов, за исключением изолированных.
* Среда службы приложений, которая развертывается в вашей виртуальной сети и поддерживает изолированные приложения плана ценообразования.

Функция интеграции VNet используется в мультитенантных приложениях. Если ваше приложение находится в [среде службы приложений,][ASEintro]то оно уже находится в виртуальной сети и не требует использования функции интеграции VNet для охвата ресурсов в той же виртуальной сети. Для получения дополнительной информации обо всех сетевых функциях, [см.][Networkingfeatures]

Интеграция VNet предоставляет вашему приложению доступ к ресурсам в виртуальной сети, но не предоставляет входявой частный доступ к приложению из виртуальной сети. Частный доступ к сайту означает сделать приложение доступным только из частной сети, например, из виртуальной сети Azure. Интеграция VNet используется только для того, чтобы совершать исходящие звонки из приложения в виртуальную сеть. Функция интеграции VNet ведет себя по-разному, когда она используется с виртуальными сетями в том же регионе и с виртуальными сетями в других регионах. Функция интеграции VNet имеет два варианта:

* **Региональная интеграция VNet:** При подключении к виртуальным сетям Azure Resource Manager в одном регионе в виртуальной сети, с ней интегрируется, необходимо иметь специальную подсеть.
* **Интеграция VNet, необходимая для шлюза:** При подключении к виртуальным сетям в других регионах или к классической виртуальной сети в том же регионе необходим шлюз виртуальной сети Azure, подготовленный в целевой виртуальной сети.

Особенности интеграции VNet:

* Требуется стандартный, премиум, PremiumV2 или упругий премиум-план ценообразования.
* Поддержка TCP и UDP.
* Работайте с приложениями и функциональными приложениями Azure App Service.

Есть некоторые вещи, которые Интеграция VNet не поддерживает, например:

* подключение диска;
* Активная интеграция каталога.
* Netbios.

Интеграция VNet, требующая шлюза, обеспечивает доступ к ресурсам только в целевой виртуальной сети или в сетях, подключенных к целевой виртуальной сети с пирингом или VPN. Интеграция VNet, необходимая для шлюза, не позволяет получить доступ к ресурсам, доступным в соединениях Azure ExpressRoute, или работать с конечными точками обслуживания.

Независимо от используемой версии, VNet Integration предоставляет вашему приложению доступ к ресурсам в виртуальной сети, но не предоставляет входящее частное приложение из виртуальной сети. Частный доступ к сайту означает сделать ваше приложение доступным только из частной сети, например, из виртуальной сети Azure. Интеграция VNet предназначена только для совершения исходящих звонков из приложения в виртуальную сеть.

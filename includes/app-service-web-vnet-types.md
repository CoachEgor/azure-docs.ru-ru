* Мультитенантные системы, которые поддерживают весь спектр ценовых планов, за исключением изолированных.
* Среда службы приложений, которая развертывается в vNet и поддерживает изолированные приложения плана ценообразования.

Функция интеграции VNet используется в мультитенантных приложениях. Если приложение размещено в [среде службы приложений][ASEintro], то оно уже находится в виртуальной сети, а для доступа к ресурсам из этой сети не требуется использовать функцию интеграции. Для получения дополнительной информации обо всех сетевых функциях, [см.][Networkingfeatures]

Интеграция VNet предоставляет вашему приложению доступ к ресурсам в VNet, но не предоставляет входявой частный доступ к вашему приложению из VNet. Частный доступ к сайту означает сделать приложение доступным только из частной сети, например, из виртуальной сети Azure. Интеграция VNet используется только для того, чтобы совершать исходящие звонки из вашего приложения в Ваш VNet. Функция интеграции VNet ведет себя по-разному, когда она используется с VNet в том же регионе и с VNet в других регионах. Функция интеграции VNet имеет два варианта:

* **Региональная интеграция VNet:** При подключении к виртуальным сетям Azure Resource Manager в одном регионе в VNet должна быть специальная подсеть, с помощью которых вы интегрируете.
* **Интеграция VNet, необходимая для шлюза:** При подключении к VNet в других регионах или к классической виртуальной сети в том же регионе, вам нужен шлюз виртуальной сети Azure, подготовленный в целевом VNet.

Особенности интеграции VNet:

* Требуется стандартный, премиум, PremiumV2 или упругий премиум-план ценообразования.
* Поддержка TCP и UDP.
* Работайте с приложениями и функциональными приложениями Azure App Service.

Есть некоторые вещи, которые Интеграция VNet не поддерживает, например:

* подключение диска;
* Активная интеграция каталога.
* Netbios.

Интеграция VNet, требующая шлюза, обеспечивает доступ к ресурсам только в целевом VNet или в сетях, подключенных к целевому VNet с пирингом или VPN. Интеграция VNet, необходимая для шлюза, не позволяет получить доступ к ресурсам, доступным в соединениях Azure ExpressRoute, или работать с конечными точками обслуживания.

Независимо от используемой версии, VNet Integration предоставляет вашему приложению доступ к ресурсам в VNet, но не предоставляет входящий частный доступ к приложению из VNet. Частный доступ к сайту означает сделать ваше приложение доступным только из частной сети, например, из Сети Azure VNet. Интеграция VNet предназначена только для совершения исходящих звонков из вашего приложения в Ваш VNet.

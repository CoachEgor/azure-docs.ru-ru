> [!TIP]
> Служба хранилища Azure поддерживает авторизацию запросов к хранилищу BLOB-объектов и очередей с помощью Azure Active Directory (Azure AD). Авторизация пользователей или приложений с помощью токена OAuth 2,0, возвращенного Azure AD, обеспечивает более высокую безопасность и простоту использования общих ключей. В Azure AD нет необходимости хранить ключ доступа к учетной записи в коде и угрожать потенциальным уязвимостям безопасности.
>
> Кроме того, служба хранилища Azure поддерживает подписанный URL-адрес для делегирования пользователей для хранилища больших двоичных объектов. SAS делегирования пользователя подписан с помощью учетных данных Azure AD. Если для разработки приложения требуются подписанные URL-адрес для доступа к хранилищу BLOB-объектов, используйте учетные данные Azure AD, чтобы создать SAS для делегирования пользователей для обеспечения высокого уровня безопасности.
>
> Корпорация Майкрософт рекомендует использовать Azure AD вместе с вашими приложениями службы хранилища Azure, когда это возможно. Дополнительные сведения см. [в статье авторизация доступа к BLOB-объектам и очередям Azure с помощью Azure Active Directory](../articles/storage/common/storage-auth-aad.md).

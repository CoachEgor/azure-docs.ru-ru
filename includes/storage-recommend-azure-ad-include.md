> [!TIP]
> Хранилище Azure поддерживает авторизации доступа к ресурсам хранилища BLOB-объектов и очередей с помощью Azure Active Directory (Azure AD). Авторизация пользователей или приложений с помощью маркера OAuth 2.0, возвращенный Azure AD обеспечивает превосходную безопасности и простоты использования общий ключ авторизации и подписей общего доступа (SAS). С помощью Azure AD нет необходимости хранить ключ доступа учетной записи с вашего кода и риск потенциальных уязвимостей системы безопасности.
>
> Корпорация Майкрософт рекомендует использование Azure AD с помощью ваших приложений службы хранилища Azure, если это возможно. Дополнительные сведения см. в разделе [авторизовать доступ к Azure, большие двоичные объекты и очереди, используя Azure Active Directory](../articles/storage/common/storage-auth-aad.md).

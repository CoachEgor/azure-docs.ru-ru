Все данные, хранящиеся в Azure Files, шифруются в состоянии покоя с помощью шифрования служб хранения Azure (SSE). Шифрование службы хранения данных работает так же, как BitLocker на Windows: данные шифруются под уровнем файловой системы. Поскольку данные шифруются под файловой системой раздела Azure, так как они закодированы на диске, вам не нужно иметь доступ к базовому ключу клиента для чтения или записи в раздел файла Azure.

По умолчанию данные, хранящиеся в Azure Files, шифруются с помощью ключей, управляемых корпорацией Майкрософт. С помощью ключей, управляемых корпорацией Майкрософт, корпорация Майкрософт хранит ключи для шифрования/расшифровки данных и отвечает за их регулярную ротацию. Вы также можете управлять своими ключами, что дает вам контроль над процессом вращения. Если вы решите зашифровать свои файлы с помощью ключей, управляемых клиентом, Azure Files имеет право получить доступ к вашим ключам для выполнения запросов о чтении и записи ваших клиентов. С помощью ключей, управляемых клиентом, вы можете отозвать эту авторизацию в любое время, но это означает, что ваша доля файлов Azure больше не будет доступна через SMB или FileREST API.

Azure Files использует ту же схему шифрования, что и другие службы хранения Azure, такие как хранилище Azure Blob.

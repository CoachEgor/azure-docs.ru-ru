Все данные, хранящиеся в службе файлов Azure, шифруются при помощи шифрования службы хранилища Azure (SSE). Шифрование службы хранилища работает аналогично BitLocker в Windows: данные шифруются под уровнем файловой системы. Так как данные шифруются под файловой системой файлового ресурса Azure по мере их кодирования на диск, вам не нужно иметь доступ к базовому ключу на клиенте для чтения и записи в файловый ресурс Azure.

По умолчанию данные, хранящиеся в службе файлов Azure, шифруются с помощью ключей, управляемых корпорацией Майкрософт. С ключами, управляемыми корпорацией Майкрософт, корпорация Майкрософт удерживает ключи для шифрования и расшифровки данных и отвечает за их регулярное вращение. Вы также можете управлять собственными ключами, что позволяет управлять процессом вращения. Если вы решили зашифровать файловые ресурсы с помощью управляемых клиентом ключей, служба файлов Azure имеет право доступа к ключам для выполнения запросов на чтение и запись от клиентов. С помощью управляемых клиентом ключей эту авторизацию можно отозвать в любое время, но это означает, что файловый ресурс Azure больше не будет доступен через SMB или API Филерест.

Служба файлов Azure использует ту же схему шифрования, что и другие службы хранилища Azure, такие как хранилище BLOB-объектов Azure. Дополнительные сведения о шифровании службы хранилища Azure (SSE) см. в статье [Шифрование хранилища Azure для неактивных данных](../articles/storage/common/storage-service-encryption.md?toc=%2fazure%2fstorage%2ffiles%2ftoc.json).

## <a name="about-the-user-delegation-sas"></a>О делегации пользователей SAS

Токен SAS для доступа к контейнеру или кабелю может быть защищен с помощью либо учетных данных Azure AD, либо ключа учетной записи. SAS, защищенный учетными данными Azure AD, называется SAS-делегацией пользователей, поскольку токен OAuth 2.0, используемый для подписания SAS, запрашивается от имени пользователя.

Корпорация Майкрософт рекомендует использовать по возможности учетные данные Azure AD в качестве наилучшей практики безопасности, а не использовать ключ учетной записи, который может быть более легко скомпрометирован. Когда для разработки приложения требуется общее количество подписей доступа, используйте учетные данные Azure AD для создания SAS-делегации пользователя для обеспечения повышенной безопасности. Для получения дополнительной информации о делегации пользователя SAS [см.](/rest/api/storageservices/create-user-delegation-sas)

> [!CAUTION]
> Любой клиент, обладающий действительным SAS, может получить доступ к данным в вашей учетной записи хранения, как это разрешено этим SAS. Важно защитить SAS от злонамеренного или непреднамеренного использования. Используйте дискреционные полномочия при распространении SAS, и есть план для отзыва скомпрометированного SAS.

Для получения дополнительной информации об подписях общего доступа см. [Грант ограниченный доступ к ресурсам хранения Azure с использованием общих подписей доступа (SAS).](../articles/storage/common/storage-sas-overview.md)

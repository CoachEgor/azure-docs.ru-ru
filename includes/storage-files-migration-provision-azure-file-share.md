Доля файлов Azure хранится в облаке в учетной записи хранения Azure.
Существует еще один уровень производительности соображений здесь.

Если у вас есть очень активные акции - акции, используемые многими пользователями и/или приложениями, то две файлы Azure могут достичь предела производительности учетной записи хранилища.

Наилучшей практикой является развертывание учетных записей хранения с одной долей файла каждый.
Вы можете объединить несколько файлов Azure в одну и ту же учетную запись хранения, если у вас есть архивные акции или вы ожидаете в них низкую повседневную активность.

Эти соображения в большей степени применимы к прямому доступу к облачным технологиям (через VM Azure), чем к Синхронизации файлов Azure. Если вы планируете использовать синхронизацию файлов Azure только на этих долях, то группировка нескольких в единую учетную запись хранения Azure будет хорошо.

Если вы составили список своих акций, то вы должны сопоставить каждую акцию со счетом хранения, в которой они будут находиться.

На предыдущем этапе вы определили соответствующее количество акций. На этом этапе создается отображение учетных записей хранилища для файла акций. Развертывайте теперь соответствующее количество учетных записей хранения Azure с соответствующим числом файлов Azure.

Убедитесь, что область каждой из учетных записей хранилища одинакова и соответствует области ресурса службы синхронизации хранения, который вы уже развернули.

> [!CAUTION]
> При создании раздела файла Azure с ограничением 100 TiB эта доля может использовать только локально избыточные параметры хранения или резервные параметры резервирования хранилища. Рассмотрим ваши потребности в избыточности хранилища перед использованием 100 файлов TiB.

Акции файлов Azure по-прежнему создаются с лимитом 5 TiB по умолчанию. Так как вы создаете новые учетные записи хранения, убедитесь, что следуйте [инструкциям по созданию учетных записей хранения, которые позволяют Azure файл акций с 100 TiB пределы.](../articles/storage/files/storage-files-how-to-create-large-file-share.md)

Другим фактором при развертывании учетной записи хранилища является избыточность хранилища Azure. См.: [Параметры избыточности хранилища Azure](../articles/storage/common/storage-redundancy.md).

Названия ваших ресурсов также важны. Например, если вы группируете несколько акций отдела управления персоналом в учетную запись хранения Azure, следует соответствующим образом назвать учетную запись хранилища. Аналогичным образом при наименовании акций файла Azure следует использовать имена, аналогичные тем, которые используются для их собственных аналогов.

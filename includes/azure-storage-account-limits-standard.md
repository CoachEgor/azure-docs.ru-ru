В следующей таблице описаны лимиты по умолчанию для общих v1, v2, хранилищ Blob, хранилища блоков blob и учетных записей хранения данных Lake Storage Gen2 с включенными данными. Ограничение *входа* относится ко всем данным, которые отправляются на учетную запись хранения. Лимит *на выход* относится ко всем данным, полученным от учетной записи хранилища.

| Ресурс | Ограничение |
| --- | --- |
| Количество учетных записей хранилища в регионе в регионе на подписку, включая стандартные, премиум-аккаунты и учетные записи Data Lake Storage Gen2. <sup>3</sup> | 250 |
| Максимальная емкость счета хранения | 5 ПиБ <sup>1</sup>|
| Максимальное количество контейнеров с каплями, капли, файловых акций, таблиц, очередей, сущностей или сообщений на счет хранения | Без ограничений |
| Максимальная частота запросов<sup>1</sup> на учетную запись хранения | 20 000 запросов в секунду |
| Максимальный вход<sup>1</sup> на счет хранения (США, регионы Европы) | 25 Гбит/с |
| Максимальный вход<sup>1</sup> на счет хранения (регионы, кроме США и Европы) | 5 Гбит/с, если включен RA-GRS/GRS, 10 Гбит/с для LRS/ЗРС<sup>2</sup> |
| Максимальный выход для общего назначения v2 и Blob хранения счетов (все регионы) | 50 Гбит/с |
| Максимальный выход для учетных записей хранения v1 общего назначения (регионы США) | 20 Гбит/с, если включен ас-GRS/GRS, 30 Гбит/с для LRS/ЗРС<sup>2</sup> |
| Максимальный выход для учетных записей хранения v1 общего назначения (не американские регионы) | 10 Гбит/с, если включен ас-GRS/GRS, 15 Гбит/с для LRS/ЗРС<sup>2</sup> |
| Максимальное количество виртуальных сетевых правил на счет хранения | 200 |
| Максимальное количество правил IP-адреса на счет хранения | 200 |

<sup>1</sup> Стандартные учетные записи Azure Storage поддерживают более высокие пределы емкости и более высокие пределы для входа по запросу. Чтобы подать запрос на увеличение ограничений для учетной записи, обратитесь в [службу поддержки Azure](https://azure.microsoft.com/support/faq/).

<sup>2</sup> Если учетная запись хранилища имеет доступ к чтению с гео-излишним хранилищем (RA-GRS) или геозон-излишним хранилищем (RA-G'S), то цели выхода для вторичного местоположения идентичны целям основного местоположения.

>[!NOTE]
>Эти операции должны выполняться пользователем с ролью *владельца* в подписке Azure. Несмотря на то, что некоторые части могут быть завершены без этого разрешения, взаимодействие владельца потребуется для полной настройки пригодного к использованию экземпляра. Дополнительные сведения см. в разделе необходимые [*разрешения*](#prerequisites-permission-requirements) ниже.

Полная настройка нового экземпляра Digital двойников для Azure состоит из трех частей:
1. **Создание экземпляра**
2. **Настройка разрешений для доступа пользователей**. Пользователи Azure должны иметь роль *владельца Azure Digital двойников (Предварительная версия)* в экземпляре Azure Digital двойников, чтобы иметь возможность управлять им и данными. На этом шаге вы как владелец в подписке Azure назначите эту роль пользователю, который будет управлять своим экземпляром Azure Digital двойников. Это может быть пользователь или кто-то другой в вашей организации.
3. **Настройка разрешений на доступ к клиентским приложениям**. часто создается клиентское приложение, которое будет использоваться для взаимодействия с экземпляром. Чтобы это клиентское приложение могли получить доступ к цифровым двойников Azure, необходимо настроить *регистрацию приложения* в [Azure Active Directory](../articles/active-directory/fundamentals/active-directory-whatis.md) , которую клиентское приложение будет использовать для проверки подлинности в экземпляре.

Для продолжения Вам потребуется подписка Azure. Вы можете задать одно из них бесплатно [.](https://azure.microsoft.com/free/?WT.mc_id=A261C142F)

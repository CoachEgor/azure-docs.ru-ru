Чтобы защитить данные в файловых ресурсах Azure от потери или повреждения данных, все файловые ресурсы Azure хранят несколько копий каждого файла по мере их написания. В зависимости от требований рабочей нагрузки можно выбрать дополнительные степени избыточности. Сейчас служба "файлы Azure" поддерживает следующие параметры обеспечения избыточности данных:

- **Локально избыточное**: локально избыточное хранилище, которое часто называют LRS, означает, что каждый файл хранится в кластере службы хранилища Azure в три раза. Это защищает от потери данных из-за сбоев оборудования, таких как неисправный диск.
- **Избыточность**в пределах зоны. хранилище, избыточное в пределах зоны, часто называется ZRS, означает, что каждый файл хранится в три раза в трех разных кластерах службы хранилища Azure. Три отдельных кластера службы хранилища Azure сохраняют файл три раза, как и локально избыточное хранилище, и физически изолированы в разных *зонах доступности*Azure. Зоны доступности являются уникальными физическими расположениями в пределах региона Azure. Каждая зона состоит из одного или нескольких центров обработки данных, оснащенных независимыми системами электроснабжения, охлаждения и сетевого взаимодействия. Запись в хранилище не принимается, пока она не будет записана в кластеры хранения во всех трех зонах доступности. 
- **Геоизбыточное**геоизбыточное хранилище, которое часто называют GRS, похоже на локально избыточное хранилище. в этом случае файл хранится в три раза в кластере службы хранилища Azure в основном регионе. Затем все операции записи асинхронно реплицируются в определенный корпорацией Майкрософт дополнительный регион. Геоизбыточное хранилище предоставляет 6 копий распределенных данных между двумя регионами Azure. В случае серьезной аварии, такой как постоянная утрата региона Azure из-за стихийного сбоя или другого аналогичного события, корпорация Майкрософт выполнит отработку отказа, чтобы база данных-получатель стала основной, обслуживая все операции. Так как репликация между основным и дополнительным регионами является асинхронной, в случае серьезной аварии данные, еще не реплицированные в дополнительный регион, будут потеряны. Можно также вручную выполнить отработку отказа геоизбыточной учетной записи хранения.
- **Геозона с избыточностью**. хранилище, избыточное в геопоясе, часто называется гзрс, является хранилищем, избыточным в пределах зоны. в этом случае файл хранится в девять раз в трех разных кластерах хранилища в основном регионе. Затем все операции записи асинхронно реплицируются в определенный корпорацией Майкрософт дополнительный регион. Процесс отработки отказа для хранилища, избыточного в пределах зоны, работает так же, как и для геоизбыточного хранилища.

Стандартные общие файловые ресурсы Azure поддерживают все четыре типа избыточности, а файловые ресурсы Azure уровня "Премиум" поддерживают только локально избыточное хранилище и избыточную зону.

Учетные записи хранения общего назначения версии 2 (GPv2) предоставляют два дополнительных параметра обеспечения избыточности, которые не поддерживаются службой файлов Azure: доступное для чтения геоизбыточное хранилище, часто называемое RA-GRS, и доступное для чтения хранилище геозоны с избыточными зонами, часто называется RA-ГЗРС. Вы можете подготавливать общие файловые ресурсы Azure в учетных записях хранения, если эти параметры заданы, но файлы Azure не поддерживают чтение из дополнительного региона. Для файловых ресурсов Azure, развернутых в учетных записях хранения с избыточной географической зоной или с геозонами, будет выставлен счет в виде геоизбыточного хранилища или геоизбыточной зоны.

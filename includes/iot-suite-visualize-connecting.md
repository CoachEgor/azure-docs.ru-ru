## <a name="view-device-telemetry"></a>Просмотр телеметрии устройства

Данные телеметрии, отправленные с устройства, можно просмотреть на **Device Explorer** странице решения.

1. Выберите устройство, подготовленное в списке устройств на **Device Explorer** странице. На панели отображаются сведения об устройстве, включая график телеметрии устройства:

    ![Просмотр сведений об устройстве](media/iot-suite-visualize-connecting/devicesdetail.png)

1. Выберите **Pressure** (Давление) для изменения порядка отображения телеметрии:

    ![Просмотр телеметрии по давлению](media/iot-suite-visualize-connecting/devicespressure.png)

1. Чтобы просмотреть диагностические сведения об устройстве, прокрутите вниз до раздела **Diagnostics** (Диагностика):

    ![Просмотр диагностики устройства](media/iot-suite-visualize-connecting/devicesdiagnostics.png)

## <a name="act-on-your-device"></a>Действия на устройстве

Для вызова методов на устройствах используйте страницу **Device Explorer** в решении для удаленного мониторинга. Например, в решении для удаленного мониторинга **Chiller** устройства реализуют метод **перезагрузки** .

1. Выберите **устройства** для перехода на страницу **Device Explorer** в решении.

1. Выберите устройство, подготовленное в списке устройств на **Device Explorer** странице:

    ![Выбор реального устройства](media/iot-suite-visualize-connecting/devicesselect.png)

1. Чтобы отобразить список методов, которые можно вызвать на устройстве, выберите **задания**, а затем — **методы**. Чтобы запланировать выполнение задания на нескольких устройствах, можно выбрать несколько устройств в списке. На панели **Jobs** (Задания) отображаются типы метода, который является общим для всех выбранных устройств.

1. Выберите **Перезагрузка**, задайте имя задания **ребутфисикалчиллер** и нажмите кнопку **Применить**.

    ![Планирование обновления встроенного ПО](media/iot-suite-visualize-connecting/deviceschedule.png)

1. Когда имитированное устройство обрабатывает метод, в консоли отображается последовательность сообщений о выполнении кода устройства.

> [!NOTE]
> Чтобы отвести состояние задания в решении, выберите **Просмотреть состояние задания**.

Запросы к контейнеру оплачиваются по ценовой категории ресурсов Azure, используемый для `<ApiKey>`.

Cognitive Services контейнеры не лицензируются для запуска без подключения к конечной точке выставления счетов для отслеживания использования. Клиенты должны включить контейнеров, взаимодействующих выставления счетов с конечной точкой выставления счетов все время. Контейнеры Cognitive Services не отправляют в корпорацию Майкрософт данные клиента (например анализируемые изображения или тексты). 

### <a name="connecting-to-azure"></a>Подключение к Azure

Контейнер должен расчетного значения аргументов для запуска. Эти значения позволяют контейнера для подключения к конечной точке выставления счетов. Отчеты об использовании контейнера примерно каждые 10—15 минут. Если контейнер не сможет подключиться в течение времени, разрешенных в Azure, контейнер будет продолжать работать, но будет не обрабатывать запросы до выставления счетов конечной точки восстановления. Подключение попытки 10 раз в тот же интервал времени, 10 – 15 минут. Если не удается подключиться к конечной точке выставления счетов в пределах 10 попыток, контейнер перестанет работать. 

### <a name="billing-arguments"></a>Аргументы выставления счетов

Все три из следующих параметров должен быть указан с допустимыми значениями выполнился `docker run` команду, чтобы запустить контейнер:

| Параметр | ОПИСАНИЕ |
|--------|-------------|
| `ApiKey` | Ключ API ресурса Cognitive Services, используемый для отслеживания информации о выставлении счетов.<br/>Этому параметру следует присвоить значение ключа API для подготовленного ресурса, который можно получить в `Billing`. |
| `Billing` | Конечная точка ресурса Cognitive Services, используемый для отслеживания информации о выставлении счетов.<br/>Значение этого параметра должно быть присвоено URI ресурса Azure, подготовленные для конечной точки.|
| `Eula` | Указывает, что вы приняли условия лицензии для контейнера.<br/>Для этого параметра следует задать значение `accept`. |

Дополнительные снимки — это резервные моменты во времени для управляемых дисков, которые, при их взятии, состоят только из всех изменений, начиная с последнего снимка. При попытке загрузки или иным образом использовать дополнительный снимок используется полный VHD. Эта новая возможность для управляемых снимков диска потенциально позволяет им быть более рентабельными, так как, если вы не решите, вам не придется хранить весь диск с каждым отдельным моментальным снимком. Так же, как обычные снимки, дополнительные снимки могут быть использованы либо для создания полного управляемого диска или сделать регулярный снимок.

Есть несколько различий между инкрементным моментальным снимком и обычным моментальным снимком. Инкрементные снимки всегда будут использовать стандартные HDD-хранилища, независимо от типа хранилища диска, в то время как обычные снимки могут использовать премиальные SSD. Если вы используете обычные снимки на Premium Storage для расширения развертывания VM, мы рекомендуем использовать пользовательские изображения на стандартном хранении в [общей галерее изображений.](../articles/virtual-machines/linux/shared-image-galleries.md) Это поможет вам достичь более масштабного масштаба с меньшими затратами. Кроме того, дополнительные снимки потенциально обеспечивают лучшую надежность при [переизбытке зон](../articles/storage/common/storage-redundancy-zrs.md) (ЗРС). Если в выбранном регионе доступна возможность использования RS, то дополнительный снимок будет автоматически использовать сяр. Если в регионе нет данных о недоступен, моментальный снимок будет по умолчанию по умолчанию локально [избыточным хранилищем](../articles/storage/common/storage-redundancy-lrs.md) (LRS). Вы можете переопределить это поведение и выбрать его вручную, но мы не рекомендуем этого.

Инкрементные снимки также предлагают дифференциальные возможности, доступные только для управляемых дисков. Они позволяют получить изменения между двумя дополнительными снимками одних и тех же управляемых дисков, вплоть до уровня блока. Эту возможность можно использовать для уменьшения объема данных при копировании снимков в разных регионах.  Например, можно загрузить первый дополнительный снимок в качестве базовой капли в другом регионе. Для последующих дополнительных снимков можно скопировать только изменения, начиная с последнего снимка в базовый капля. После копирования изменений можно делать снимки на базовой каплей, которые представляют вашу точку резервного копирования диска в другом регионе. Вы можете восстановить свой диск либо из базовой капли, либо из снимка на базовой капли в другом регионе.

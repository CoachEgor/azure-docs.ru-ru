Для учетной записи хранения доступны такие варианты репликации:

* [Локально избыточное хранилище (LRS)](../articles/storage/common/storage-redundancy-lrs.md): Простая и экономичная стратегия репликации. Данные реплицируются синхронно три раза в основном регионе.
* [Хранилище, избыточное между зонами (ZRS)](../articles/storage/common/storage-redundancy-zrs.md). Репликация для сценариев, требующих высокой доступности. Данные реплицируются синхронно по трем зонам доступности Azure в основном регионе.
* [Геоизбыточное хранилище (GRS).](../articles/storage/common/storage-redundancy-grs.md) Репликация между регионами для защиты от региональных простоев. Данные реплицируются синхронно три раза в основном регионе, а затем реплицируются асинхронно в дополнительный регион. Для доступа на чтение к данным в дополнительном регионе включите геоизбыточное хранилище с доступом на чтение (RA-GRS).
* Хранилище, избыточное в геопоясе [(гзрс) (Предварительная версия)](../articles/storage/common/storage-redundancy-gzrs.md): Репликация для сценариев, требующих высокой доступности и максимальной устойчивости. Данные реплицируются синхронно по трем зонам доступности Azure в основном регионе, а затем реплицируются асинхронно в дополнительный регион. Для доступа на чтение к данным в дополнительном регионе включите хранилище, избыточное в геозону с доступом на чтение (RA-ГЗРС).

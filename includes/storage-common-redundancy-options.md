Варианты избыточности для учетной записи хранилища включают:

* Локально избыточное хранилище (LRS): простая, недорогая стратегия избыточности. Данные копируются синхронно три раза в пределах первичной области.
* Резервная хранилища зон (ЗРС): Избыточность сценариев, требующих высокой доступности. Данные синхронно копируются в трех зонах доступности Azure в основном регионе.
* Геоизбыточное хранилище (GRS): межрегиональная избыточность для защиты от региональных отключений. Данные скопированы синхронно три раза в первичной области, а затем копируются асинхронно во вторичную область. Для чтения доступа к данным во вторичном регионе включите геоизмовую хранилище (RA-GRS).
* Гео-зонно-излишним хранилищем (ГЗРС) (предварительный просмотр): Избыточность сценариев, требующих как высокой доступности, так и максимальной долговечности. Данные синхронно копируются в трех зонах доступности Azure в основном регионе, а затем синхронно копируются во вторичную область. Для чтения доступа к данным во вторичном регионе, включите считывающий доступ к геозоне-избыточного хранения (RA-G'S).

Для получения дополнительной информации о вариантах [Azure Storage redundancy](../articles/storage/common/storage-redundancy.md)избыточности в хранилище Azure см.

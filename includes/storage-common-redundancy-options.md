Ниже приведены варианты избыточности учетной записи хранения.

* Локально избыточное хранилище (LRS) — простая и экономичная стратегия избыточности. Данные копируются синхронно три раза в основном регионе.
* Хранилище, избыточное в виде зоны (ZRS): избыточность для сценариев, требующих высокого уровня доступности. Данные копируются синхронно по трем зонам доступности Azure в основном регионе.
* Геоизбыточное хранилище (GRS): избыточность в разных регионах для защиты от региональных простоев. Данные копируются три раза в основном регионе, а затем асинхронно копируются в дополнительный регион. Для доступа на чтение к данным в дополнительном регионе включите геоизбыточное хранилище с доступом на чтение (RA-GRS).
* Хранилище, избыточное в геопоясе (ГЗРС) (Предварительная версия): избыточность для сценариев, требующих высокой доступности и максимальной устойчивости. Данные копируются синхронно в трех зонах доступности Azure в основном регионе, а затем асинхронно копируются в дополнительный регион. Для доступа на чтение к данным в дополнительном регионе включите хранилище, избыточное в геозону с доступом на чтение (RA-ГЗРС).

Дополнительные сведения о параметрах избыточности в службе хранилища Azure см. в статье [избыточность службы хранилища Azure](../articles/storage/common/storage-redundancy.md).

1. Войдите на [портале Azure](https://portal.azure.com).
1. Выберите фильтр **каталог и подписка** в верхнем меню, а затем выберите каталог, содержащий клиент Azure AD B2C.
1. В меню слева выберите **Azure AD B2C**. Или выберите **все службы** , найдите и выберите **Azure AD B2C**.
1. Щелкните **Приложения**, а затем выберите **Добавить**.
1. Введите имя приложения. Например, *nativeapp1*.
1. Для **собственного клиента**выберите **Да**.
1. Введите **пользовательский URI перенаправления** с уникальной схемой. Например, `com.onmicrosoft.contosob2c.exampleapp://oauth/redirect`. При выборе URI перенаправления нужно учитывать два важных момента:
    1. **Уникальность.** Схема URI перенаправления должна быть уникальной для каждого приложения. В примере `com.onmicrosoft.contosob2c.exampleapp://oauth/redirect` `com.onmicrosoft.contosob2c.exampleapp` представляет собой схему. Рекомендуется придерживаться этого шаблона. Если два приложения используют ту же схему, пользователю предоставляется возможность выбрать приложение. Если пользователь выберет неверно, произойдет сбой входа.
    1. **Полнота.** URI перенаправления должен содержать как схему, так и путь. Путь должен содержать по крайней мере одну косую черту после имени домена. Например, `//oauth/` работает при сбое `//oauth`. Не включайте в URI специальные символы, например символы подчеркивания.
1. Нажмите кнопку **Создать**.

В этом разделе вы устанавливаете агент Azure File Sync на серверЕ Windows.
[Руководство по развертыванию](../articles/storage/files/storage-sync-files-deployment-guide.md) иллюстрирует, что необходимо отключить **повышенную безопасность IE.** Повышенная безопасность IE — это мера безопасности, которая не применима к синхронизации файлов Azure, и отключение ее позволяет проверить подлинность в Azure без каких-либо проблем.

Откройте PowerShell и установите необходимые модули PowerShell со следующими командами. Убедитесь в том, чтобы установить полный модуль и NuGet поставщика, когда предложено:

```powershell
Install-Module -Name Az -AllowClobber
Install-Module -Name Az.StorageSync
```

Если у вас есть какие-либо проблемы, достигающие Интернета с вашего сервера, сейчас самое время их решить. Azure File Sync использует любое доступное сетевое подключение к Интернету.
Также поддерживается требование прокси-сервера для охвата Интернета. Вы можете настроить общемашинный прокси сейчас, либо указать прокси, который просто синхронизация файлов будет использоваться во время установки агента.

Если это означает, что вам нужно открыть брандмауэры для этого сервера, то это может быть приемлемым подходом к вам. В конце установки сервера, после завершения регистрации сервера, будет отчет о подключении сети, показывающий точные URL-адреса конечной точки в Azure, с которым синхронизация файлов должна быть связана с выбранным вами регионом. В отчете также говорится, почему необходимо общение. Отчет можно использовать для блокировки брандмауэров вокруг этого сервера на определенные URL-адреса.

Вы также можете следовать более консервативному подходу, при котором вы не открываете брандмауэры широко, а ограничиваете сервер для общения с более высоким уровнем DNS-пространствами имен - в статье по прокси-серверу [Azure File Sync и настройках брандмауэра](../articles/storage/files/storage-sync-files-firewall-and-proxy.md) имеется больше документации и деталей. Следуйте своим собственным рекомендациям сети.

В конце мастера *установки* сервера появится мастер *регистрации* серверов.
Зарегистрируйте сервер на ресурсе службы синхронизации хранения Azure ранее.

Эти шаги описаны более подробно в руководстве по развертыванию, включая приведенные выше модули PowerShell, которые следует установить в первую очередь: [установка агента Azure File Sync.](../articles/storage/files/storage-sync-files-deployment-guide.md)

Последний агент должен быть использован и может быть загружен из Microsoft Download Center: [Azure File Sync - агент](https://aka.ms/AFS/agent "Загрузка агента Azure File Sync").

После успешной установки и регистрации сервера вы можете проверить, успешно ли вы выполнили этот шаг: перейдите на ресурс службы синхронизации хранения на портале Azure, а затем следуйте левому меню на "Зарегистрированные серверы". Вы увидите ваш сервер, перечисленных там сразу.

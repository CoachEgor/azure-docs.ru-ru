Сохранить использование Azure Disk Storage с зарезервированной емкостью. Бронирование данных по хранению дисков Azure в сочетании с Azure Reserved Virtual Machine Instances позволяет снизить общие затраты на виртуальные машины (VM). Скидка на резервирование автоматически применяется к соответствующим дискам в выбранной области бронирования. Из-за этого автоматического приложения, вам не нужно присваивать резервирование управляемого диска, чтобы получить скидки.

Скидки применяются почасово в зависимости от использования диска. Неиспользованные зарезервированные емкости не переносятся. Скидки на резервирование azure Disk Storage не распространяются на неуправляемые диски, ультрадиски или потребление капли страницы.

## <a name="determine-your-storage-needs"></a>Определите потребности в хранении

Перед покупкой бронирования определите свои потребности в хранении. В настоящее время резервирование данных Azure Disk Storage доступно только для некоторых премиальных SSD SSD SKUs премиум-класса Azure. Размер и производительность диска определяет Сяог премиум-класса.

При определении потребностей в хранении не думайте о дисках, основанных только на емкости. Например, вы не можете иметь резервирование для диска P40 и использовать его для оплаты двух меньших дисков P30. При покупке бронирования вы покупаете только бронирование на общее количество дисков на SKU.

Резервирование диска производится на диск SKU. В результате потребление резервирования основано на единице диска SKUs вместо предоставленного размера.

Например, предположим, что вы зарезервируете один диск P40 с 2 TiB предусмотренной емкости для хранения. Также предположим, что вы выделяете только два диска P30. Резервирование P40 в этом случае не учитывает потребление P30, и вы платите по ставке оплаты по мере отсчета на дисках P30.

## <a name="purchase-considerations"></a>Вопросы, связанные с приобретением

Мы рекомендуем следующие практики при рассмотрении покупки бронирования диска:

- Проанализируйте информацию об использовании, чтобы определить, какие бронирования вы должны приобрести. Убедитесь, что вы отслеживаете использование в дисковых SKUs вместо предусмотренных или используемых дисковых емкостей.
- Изучите бронирование диска вместе с вашим бронированием VM. Мы настоятельно рекомендуем сделать бронирование как для использования VM, так и для использования диска для максимальной экономии. Вы можете начать с определения правильного резервирования VM, а затем оценить бронирование диска. Как правило, для каждой рабочей нагрузки у вас будет стандартная конфигурация. Например, сервер сервера Сервера S'L может иметь два диска данных P40 и один диск операционной системы P30.
  
  Этот шаблон может помочь вам определить зарезервированную сумму, которую вы можете приобрести. Такой подход позволяет упростить процесс оценки и гарантировать, что у вас есть согласованный план как для вашего VM, так и для дисков. План содержит такие соображения, как подписка или регионы.

## <a name="purchase-restrictions"></a>Ограничения на покупку

Скидки на бронирование в настоящее время недоступны для следующих:

- Неуправляемые диски или капли страниц.
- Стандартные SSD-накопители или стандартные жесткие диски (HDD).
- Премиум SSD SKUs меньше, чем P30: P1, P2, P3, P4, P6, P10, P15, и P20 SSD SKUs.
- Диски в регионах Azure Government, Azure ВГермании или В Китае.

В редких случаях Azure ограничивает покупку новых резерваций подмножеством дисковых SKUS из-за низкой емкости в регионе.

## <a name="buy-a-disk-reservation"></a>Купить диск бронирования

Бронирование azure Disk Storage можно приобрести через [портал Azure.](https://portal.azure.com/) Вы можете оплатить бронирование либо авансом, либо ежемесячными платежами. Для получения дополнительной информации о покупке с ежемесячными [платежами, см.](../articles/cost-management-billing/reservations/monthly-payments-reservations.md)

Следуйте этим шагам, чтобы приобрести зарезервированную емкость:

1. Перейдите на панель [бронирования покупок](https://portal.azure.com/#blade/Microsoft_Azure_Reservations/CreateBlade/referrer/Browse_AddCommand) на портале Azure.

1. Выберите **управляемые диски Azure** для покупки бронирования.

    ![Пан для покупки бронирования](media/disks-reserved-capacity/disks-reserved-purchase-reservation.png) 

1. Укажите требуемые значения, описанные в следующей таблице:

   |Элемент  |Описание  |
   |---------|---------|
   |**Область**   |  Сколько подписок может использовать платежное преимущество, связанное с бронированием. Это значение также определяет, как бронирование применяется к конкретным подпискам. <br/><br/> При выборе **общего элемента**скидка на бронирование применяется к емкостью хранилища Azure в каждой подписке в контексте выставления счетов. Контекст выставления счетов зависит от способа регистрации в Azure. Для клиентов с соглашением Enterprise общая область действует в рамках регистрации и включает в себя все подписки, заданные при регистрации. Для клиентов с оплатой по мере использования общий объем включает все индивидуальные подписки с тарифами с оплатой по мере использования, созданными администратором учетной записи.  <br/><br/>  При выборе **Единой подписки**скидка на бронирование применяется к емкости хранения Azure в выбранной подписке. <br/><br/> При выборе **единой группы ресурсов**скидка на резервирование применяется к емкости хранения Azure в выбранной подписке и в выбранной группе ресурсов этой подписки. <br/><br/> Вы можете изменить область бронирования после покупки бронирования.  |
   |**Подписка**  | Подписка, используемая для оплаты бронирования хранилища Azure. Метод оплаты по выбранной подписке используется для взимания расходов. Подписка должна быть одним из следующих типов:<br/><ul><li> Корпоративное соглашение (предложения номеров МС-АЗ-0017P и МС-АЗ-0148P). для подписки Enterprise плата вычитается из баланса денежных обязательств или относится к избыточным расходам.</li><br/><li>Индивидуальная подписка с тарифами с оплатой по мере использования (предлагают номера MS-A'R-0003P и MS-A'A-0023P). Для индивидуальной подписки с оплатой по мере использования ставки, сборы выставляются на кредитную карту или метод оплаты счета-фактуры по подписке.</li></ul>    |
   | **Дисков** | SKU, который вы хотите создать. |
   | **Регионе** | Регион, где действует резервация. |
   | **Частота выставления счетов** | Как часто счет выставляется за бронирование. Варианты включают **ежемесячно** и **upfront**. |

    ![Пандля выбора продукта, который вы хотите приобрести.](media/disks-reserved-capacity/premium-ssd-reserved-purchase-selection.png)

1. После указания значений для бронирования портал Azure отображает стоимость. Портал также показывает процент скидок по сравнению с оплатой по мере оплаты. Выберите **Далее,** чтобы продолжить панель **бронирования покупок.**

1. На панели **бронирования покупки** вы можете назвать свое бронирование и выбрать общее количество бронирований, которые вы хотите сделать. Количество карт оговорок на количество дисков. Например, если вы хотите зарезервировать сто дисков, введите значение **количества** **100**.

1. Просмотрите общую стоимость бронирования.

    ![Панель бронирования покупки](media/disks-reserved-capacity/premium-ssd-reserved-selecting-sku-total-purchase.png)

После покупки бронирования оно автоматически применяется к любым существующим ресурсам disk Storage, которые соответствуют условиям бронирования. Если вы еще не создали ресурсы Disk Storage, бронирование применяется всякий раз, когда вы создаете ресурс, который соответствует условиям бронирования. В любом случае срок бронирования начинается сразу после успешной покупки.

## <a name="cancel-exchange-or-refund-reservations"></a>Отмена, обмен резервирования, возмещение средств за резервирование

Вы можете отменить, обменять или вернуть бронирование в определенных ограничениях. Дополнительные сведения см. в статье [Self-service exchanges and refunds for Azure Reservations](https://docs.microsoft.com/azure/cost-management-billing/reservations/exchange-and-refund-azure-reservations) (Самостоятельное выполнение обмена и возмещения средств для резервирований Azure).

## <a name="expiration-of-a-reservation"></a>Срок действия бронирования

По истечении срока действия резервирования любая емкость хранилища Azure Disk, используемая в рамках этого резервирования, оплачивается по тарифу «оплата по мере использования». Резервирования не продлеваются автоматически.

Вы получите уведомление по электронной почте за 30 дней до истечения срока бронирования и снова на дату истечения срока действия. Чтобы продолжить использование экономии средств, предоставляемой бронированием, продлите ее не позднее срока действия.

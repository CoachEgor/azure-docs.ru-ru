Сэкономьте на использовании твердотельного накопителя уровня "Премиум" с зарезервированной емкостью в сочетании с зарезервированными экземплярами виртуальных машин, вы можете снизить общую стоимость виртуальных машин. Скидка на резервирование автоматически применяется к соответствующим дискам в выбранной области резервирования. для получения скидок не нужно назначать резервирование управляемому диску. Скидки применяются каждый час на использование диска, а все неиспользованные зарезервированные ресурсы не переносятся. Скидка на резервирование управляемого диска неприменима к неуправляемым дискам, Ultra Disks и потреблению страничных BLOB-объектов.

## <a name="determine-your-storage-needs"></a>Определение потребностей в хранении

Прежде чем приобретать резервирование, необходимо определить, что именно требуется для хранения. В настоящее время резервирование диска доступно только для номеров SKU SSD уровня "Премиум". SKU твердотельного накопителя уровня "Премиум" определяет размер и производительность диска. При определении потребностей в хранении не рекомендуется думать о дисках так, как общая емкость. Вы не можете использовать резервирование для большего диска (например, P40) и использовать его для оплаты двух небольших дисков (P30). При покупке резервирования вы приобретаете только общее число дисков на номер SKU.

Резервирование диска устанавливается на номер SKU диска, поэтому потребление резервирования зависит от единицы номеров SKU дисков, а не от указанного размера. Например, если вы зарезервировано 1 P40 из 2 Тиб емкости, но только выделили 2 диска P30, то эти два P30 потребления не будут учтены для P40 резервирования, и вы будете платить по тарифу по мере использования.

## <a name="purchase-considerations"></a>Вопросы, связанные с приобретением

При рассмотрении приобретения резервирования дисков рекомендуется принять во внимание следующие рекомендации.

- Проанализируйте сведения об использовании, чтобы определить, какие резервирования следует приобрести. Убедитесь, что вы отслеживаете использование в номерах дисков, а не подготовленных или используемых дисковых мощностях. 
- Изучите резервирование дисков и резервирование виртуальных машин. Мы настоятельно рекомендуем зарезервировать использование виртуальной машины и диска для максимального сохранения. Вы можете начать с определения правильного резервирования виртуальной машины, а затем соответствующим образом оценить резервирование диска. Как правило, у вас есть Стандартная конфигурация для каждой рабочей нагрузки, например, SQL Server может иметь два P40 диска данных и один диск ОС P30. Этот тип шаблона может помочь определить объем резервирования, который вы можете приобрести. Этот подход позволяет упростить процесс оценки, а также обеспечить согласованный план для виртуальных машин и дисков с точки зрения подписок, регионов и других. 

## <a name="purchase-restrictions"></a>Ограничения на приобретение

Скидки на резервирование в настоящее время недоступны для следующих дисков:
- Неуправляемые диски и страничные BLOB-объекты
- SSD (цен. категория "Стандартный") или стандартный HDD
- SSD (цен. категория "Премиум") номера SKU меньше P30 — резервирования недоступны для номеров SKU P1/P2/P3/P4/P6/P10/p15/P20 SSD (цен. категория "Премиум").
- Облака. резервирования не доступны для приобретения в регионах Azure gov, Германии или Китая.
- Ограничения емкости. в редких обстоятельствах Azure ограничивает покупку новых резервирований для подмножества номеров SKU дисков из-за нехватки емкости в регионе.

## <a name="buy-a-disk-reservation"></a>Приобретение резервирования диска

Вы можете приобрести резервирование дисков Azure с помощью [портал Azure](https://portal.azure.com/). Вы можете платить за резервирование заранее или с ежемесячными платежами. Дополнительные сведения о приобретении с ежемесячными платежами см. [в разделе покупка резервирования с ежемесячными платежами](../articles/cost-management-billing/reservations/monthly-payments-reservations.md).

Чтобы приобрести зарезервированную емкость, выполните следующие действия.

1. Перейдите в колонку [резервирования покупки](https://portal.azure.com/#blade/Microsoft_Azure_Reservations/CreateBlade/referrer/Browse_AddCommand) в портал Azure.
1. Выберите **управляемые диски Azure** , чтобы приобрести резервирование.

1. Заполните обязательные поля, как описано в следующей таблице.

   |Поле  |Description  |
   |---------|---------|
   |**Область действия**   |  Указывает, сколько подписок может использовать преимущества выставления счетов, связанные с резервированием. Он также контролирует, как резервирование применяется к определенным подпискам. <br/><br/> Если выбран **общий доступ**, скидка резервирования применяется к емкости хранилища Azure в любой подписке в контексте выставления счетов. Контекст выставления счетов зависит от способа регистрации в Azure. Для клиентов с соглашением Enterprise общая область действует в рамках регистрации и включает в себя все подписки, заданные при регистрации. Для клиентов с оплатой по мере использования общая область включает все индивидуальные подписки с тарифами оплаты по мере использования, созданными администратором учетной записи.  <br/><br/>  При выборе **одной подписки**скидка на резервирование применяется к емкости хранилища Azure в выбранной подписке. <br/><br/> Если выбрать **одну группу ресурсов**, скидка резервирования будет применена к емкости хранилища Azure в выбранной подписке и выбранной группе ресурсов в этой подписке. <br/><br/> Вы можете изменить область резервирования после приобретения резервирования.  |
   |**Подписка**  | Подписка, используемая для оплаты резервирования службы хранилища Azure. Метод оплаты в выбранной подписке используется для зарядки затрат. Подписка должна иметь один из следующих типов: <br/><br/>  Соглашение Enterprise (номера предложений: MS-AZR-0017P или MS-AZR-0148P). в рамках подписки Enterprise плата вычитается из баланса денежных обязательств по регистрации или оплачивается как дополнительное. <br/><br/> Отдельная подписка с оплатой по мере использования (номера предложений: MS-AZR-0003P или MS-AZR-0023P): для отдельной подписки с тарифами с оплатой по мере использования плата оплачивается по методу оплаты кредитной карты или счета-фактуры в подписке.    |
   | **диски;** | Номер SKU, который вы хотите создать. |
   | **Регион** | Регион, в котором действует резервирование. |
   | **Периодичность выставления счетов** | Указывает, как часто оплачивается счет за резервирование. Варианты: *ежемесячно* или *заранее*. |

    ![Премиум-ССД-ресервед-пурчасе-селектион. png](media/disks-reserved-capacity/premium-ssd-reserved-purchase-selection.png)

1. После выбора параметров резервирования портал Azure отображает затраты. На портале также отображается процент скидки по выставлению счетов с оплатой по мере использования. Нажмите кнопку **Далее** , чтобы перейти в колонку **резервирования покупки** .

1. На панели **резервирования покупки** можно присвоить имя резервированию и выбрать общее количество резервирований, которое вы хотите сделать. Число резервирований, сопоставленное с количеством дисков. Например, если вы хотите зарезервировать сотни дисков, вы измените **количество** на 100.
1. Ознакомьтесь с общей стоимостью резервирования.

    ![Премиум-ССД-ресервед-селектинг-СКУ-тотал-пурчасе. png](media/disks-reserved-capacity/premium-ssd-reserved-selecting-sku-total-purchase.png)

После приобретения резервирования оно автоматически применяется к любым существующим ресурсам дискового хранилища Azure, которые соответствуют условиям резервирования. Если вы еще не создали ресурсы хранилища дисков Azure, резервирование будет применяться при каждом создании ресурса, удовлетворяющего условиям резервирования. В любом случае срок резервирования начинается сразу после успешной покупки.

## <a name="exchange-or-refund-a-reservation"></a>Exchange или возмещение резервирования

С некоторыми ограничениями вы можете обмениваться резервированиями или возместить их.

Для обмена или возмещения резервирования перейдите к сведениям о резервировании в портал Azure. Выберите вариант **Exchange или возмещение**и следуйте инструкциям, чтобы отправить запрос в службу поддержки. Когда запрос будет обработан, корпорация Майкрософт отправит вам сообщение электронной почты, чтобы подтвердить завершение запроса.

Дополнительные сведения о политиках резервирования Azure см. в разделе [самостоятельная служба Exchange и возмещение для резервирования Azure](../articles/cost-management-billing/reservations/exchange-and-refund-azure-reservations.md).

### <a name="exchange-a-reservation"></a>Обмен резервированиями

Обмен резервированием позволяет получить пропорциональное возмещение на основе неиспользуемой части резервирования. Затем можно применить возмещение к цене приобретения нового резервирования дисков Azure.
Количество проводимых обменов не ограничено. Кроме того, плата за обмен не взымается. Новое зарезервированное значение должно быть равно или больше значения, превышающего сумму кредита от первоначального резервирования. Резервирование дисков Azure можно использовать только для другого резервирования дисков Azure, а не для резервирования любой другой службы Azure.

### <a name="refund-a-reservation"></a>Возмещение резервирования

Резервирование дисков Azure можно отменить в любое время. Если вы отмените операцию, вы получите пропорциональное возмещение на основе оставшегося срока резервирования, за вычетом 12% от раннего увольнения. Максимальное значение возмещения в год — $50 000.
Отмена резервирования немедленно прерывает резервирование и возвращает оставшиеся месяцы в корпорацию Майкрософт. Остаток пропорционального баланса, за вычетом платы, будет взиматься с вашей первоначальной формы покупки.

## <a name="expiration-of-a-reservation"></a>Срок действия резервирования

По истечении срока действия резервирования для любой емкости диска Azure, используемой при резервировании, взимается плата по мере использования. Резервирования не продлеваются автоматически.
Вы получите по электронной почте уведомление через 30 дней до истечения срока действия резервирования и снова на дату окончания срока действия. Чтобы продолжить использовать преимущества, которые предоставляет резервирование, обновите его позже даты окончания срока действия.

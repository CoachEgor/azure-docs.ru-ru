Файловые ресурсы Azure развертываются в *учетных записях хранения*, которые представляют собой объекты верхнего уровня, представляющие общий пул хранилища. Этот пул хранилища можно использовать для развертывания нескольких файловых ресурсов, а также других ресурсов хранения, таких как контейнеры больших двоичных объектов, очереди или таблицы. Все ресурсы хранилища, развернутые в учетной записи хранения, совместно используют ограничения, которые применяются к этой учетной записи хранения. Сведения о текущих ограничениях для учетной записи хранения см. в статье [целевые показатели масштабируемости и производительности службы файлов Azure](../articles/storage/files/storage-files-scale-targets.md).

Существует два основных типа учетных записей хранения, которые будут использоваться для развертывания службы файлов Azure. 
- **Учетные записи хранения общего назначения версии 2 (GPv2)**. учетные записи хранения GPv2 позволяют развертывать файловые ресурсы Azure на оборудовании на основе стандартного или жесткого диска (на основе жестких дисков). Помимо хранения файловых ресурсов Azure, учетные записи хранения GPv2 могут хранить другие ресурсы хранилища, такие как контейнеры больших двоичных объектов, очереди или таблицы. 
- **Учетные записи хранения филестораже**. учетные записи хранения филестораже позволяют развертывать файловые ресурсы Azure на оборудовании уровня "Премиум" или твердотельного накопителя (на основе SSD). Учетные записи Филестораже можно использовать только для хранения файловых ресурсов Azure. другие ресурсы хранилища (контейнеры больших двоичных объектов, очереди, таблицы и т. д.) не могут быть развернуты в учетной записи Филестораже.

Существует несколько других типов учетных записей хранения, которые вы можете поступить в портал Azure, PowerShell или CLI. Учетные записи хранения двух типов: Блоккблобстораже и Блобстораже не могут содержать файловые ресурсы Azure. Другие типы учетных записей хранения, которые вы можете увидеть, — это общие назначения версии 1 (GPv1) и классические учетные записи хранения, которые могут содержать файловые ресурсы Azure. Хотя GPv1 и классические учетные записи хранения могут содержать файловые ресурсы Azure, большинство новых функций службы файлов Azure доступны только в учетных записях хранения GPv2 и Филестораже. Поэтому рекомендуется использовать учетные записи хранения GPv2 и Филестораже только для новых развертываний, а также для обновления GPv1 и классических учетных записей хранения, если они уже существуют в вашей среде.

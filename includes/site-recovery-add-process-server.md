1. Запуск файла единой установки Azure Site Recovery (UnifiedSetup.exe)
2. В окне **перед началом**нажмите кнопку **добавить дополнительные серверы обработки для масштабирования развертывания**.

   ![Добавление сервера обработки](./media/site-recovery-add-process-server/ps-page-1.png)

3. На странице **сведения о сервере конфигурации**укажите IP-адрес сервера конфигурации и парольную фразу.

   ![Добавление сервера обработки 2](./media/site-recovery-add-process-server/ps-page-2.png)
4. В окне **Параметры Интернета**укажите, как поставщик, выполняющийся на сервере конфигурации, подключается к Azure Site Recovery через Интернет.

   ![Добавление сервера обработки 3](./media/site-recovery-add-process-server/ps-page-3.png)

   * Если вы хотите подключиться к прокси-серверу, который сейчас настроен на компьютере, выберите **подключиться с существующими параметрами прокси-сервера**.
   * Если вы хотите, чтобы поставщик подключается напрямую, выберите **подключиться напрямую без прокси-сервера**.
   * Если для существующего прокси-сервера требуется проверка подлинности или вы хотите использовать настраиваемый прокси-сервер для подключения поставщика, выберите **подключиться с настраиваемыми параметрами прокси-сервера**.

     * При использовании пользовательского прокси-сервера необходимо указать адрес, порт и учетные данные.
     * Если используется прокси-сервер, скорее всего, вы уже разрешили использовать эти URL-адреса служб.

5. В окне **Проверка предварительных требований**программа установки выполняет проверку, чтобы убедиться, что установка может быть выполнена. Если появится предупреждение о **проверке глобальной синхронизации времени**, убедитесь, что время на системных часах (параметры**даты и времени** ) совпадает с часовым поясом.

     ![Добавление сервера обработки 4](./media/site-recovery-add-process-server/ps-page-4.png)

6. На странице **Сведения о среде** укажите, нужно ли выполнять репликацию виртуальных машин VMware. Если нужно, программа установки проверяет, установлен ли компонент PowerCLI 6.0.

     ![Добавление сервера обработки 5](./media/site-recovery-add-process-server/ps-page-5.png)

7. На странице **Расположение установки** выберите место для установки двоичных файлов и хранения кэша. Выбранный вами диск кэша должен иметь не менее 5 ГБ доступной памяти. Однако рекомендуемый объем — не менее 600 ГБ.
     ![Добавление сервера обработки 5](./media/site-recovery-add-process-server/ps-page-6.png)

8. В поле **Выбор сети**укажите прослушиватель (сетевой адаптер и порт SSL), на котором сервер конфигурации будет отправлять и получать данные репликации. Порт 9443 — это стандартный порт для отправки и приема трафика репликации, но вы можете изменить номер порта в соответствии с требованиями среды. Наряду с портом 9443 мы также открываем порт 443, который используется веб-сервером для оркестрации операций репликации. Не следует использовать порт 443 для отправки и приема трафика репликации.

     ![Добавление сервера обработки 6](./media/site-recovery-add-process-server/ps-page-7.png)
9. Просмотрите информацию на странице **Сводка** и нажмите кнопку **Установить**. Когда установка завершится, будет создана парольная фраза. Они потребуются при включении репликации, поэтому ее необходимо скопировать и сохранить в безопасном месте.

     ![Добавление сервера обработки 7](./media/site-recovery-add-process-server/ps-page-8.png)

Следующие предостережения относятся к данным во время перемещения в Azure.

- Мы советуем избегать ситуации, когда в один и тот же контейнер запись производит более одного устройства.
- При наличии объекта Azure (например, BLOB-объекта или файла) в облаке с тем же именем, что и у копируемого объекта, устройство перезапишет файл в облаке.
- Пустая иерархия каталогов (без файлов), созданная в общих папках, не передается в контейнеры больших двоичных объектов.
- Данные можно скопировать с помощью перетаскивания в проводнике или с помощью командной строки. Если размер совокупного размера копируемых файлов превышает 10 ГБ, рекомендуется использовать программу для выполнения операций копирования, например Robocopy или rsync. Средства быстрого копирования повторируют операцию копирования для периодических ошибок и обеспечивают дополнительную устойчивость.
- Если в общую папку, связанную с контейнером хранилища Azure, загружаются BLOB-объекты, которые не соответствуют типу определенному для общей папки в момент создания, то такие BLOB-объекты обновляться не будут. Например, создаете общую папку блочных BLOB-объектов на устройстве. Свяжите общую папку с существующим контейнером облака, который имеет страничные BLOB-объекты. Обновите ее, чтобы скачать файлы. Изменяете некоторые обновленные файлы, которые уже хранятся в виде страничных BLOB-объектов в облаке. Появятся сбои отправки.
- После создания файла в общих папках его невозможно переименовать.
- Удаление файла из общей папки не приводит к удалению записи в учетной записи хранения.
- Если для копирования данных используется rsync, то `rsync -a` параметр не поддерживается.

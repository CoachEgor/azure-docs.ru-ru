## <a name="limitations"></a>Ограничения

- Виртуальные наборы масштабов машин в настоящее время не поддерживаются на выделенных хостах.
- Размеры и типы оборудования, доступные для выделенных узлов, различаются в зависимости от региона. Чтобы узнать больше, обратитесь к [странице ценообразования](https://aka.ms/ADHPricing) хоста.

## <a name="create-a-host-group"></a>Создание группы узлов

Группа **узлов** — это новый ресурс, представляющий коллекцию выделенных узлов. Вы создаете группу узлов в регионе и зоне доступности и добавляете к ней хосты. При планировании высокой доступности, Есть дополнительные варианты. Вы можете использовать один или оба из следующих вариантов с выделенными хостами: 
- Пролет через несколько зон доступности. В этом случае необходимо иметь группу хоста в каждой из зон, которые вы хотите использовать.
- Пролетчерез несколько доменов неисправностей, которые отображаются на физических стойках. 
 
В любом случае необходимо учесть количество доменов неисправностей для принимающей группы. Если вы не хотите охватывать домены неисправностей в группе, используйте количество доменов неисправностей 1. 

Вы также можете решить использовать как зоны доступности, так и домены неисправностей. 

В этом примере мы создадим группу хоста, используя 1 зону доступности и 2 домена неисправностей. 


1. Откройте [портал](https://portal.azure.com)Azure .
1. Выберите **Создать ресурс** в левом верхнем углу.
1. Поиск **группы хоста,** а затем выберите **группы хоста** из результатов.
1. На странице **групп хоста** выберите **Создать**.
1. Выберите подписку, которая вы хотели бы использовать, а затем выберите **Создать новую** для создания новой группы ресурсов.
1. Введите *myDedicatedHostsRG* как **имя,** а затем выберите **OK**.
1. Для **имени группы host,** введите *myHostGroup.*
1. В поле **Расположение** выберите **Восточная часть США**.
1. Для **зоны доступности**выберите **1**.
1. Для **количества доменов неисправностей,** выберите **2**.
1. Выберите **Обзор и создайте,** а затем подождите проверки.
1. Как только вы увидите пройденое сообщение **Проверки,** выберите **Создать** для создания группы узла.

Создание группы хозяев займет всего несколько минут.

## <a name="create-a-dedicated-host"></a>Создание выделенного узла

Теперь создайте специальный хост в группе хоста. В дополнение к имени хоста, вы должны предоставить SKU для хозяина. Host SKU фиксирует поддерживаемую серию VM, а также аппаратное поколение для выделенного хоста.

Для получения дополнительной информации о хост-ску и ценах [см.](https://aka.ms/ADHPricing)

Если вы установите счет домена неисправностей для вашей хост-группы, вам будет предложено указать домен неисправности для вашего хоста.  

1. Выберите **Создать ресурс** в левом верхнем углу.
1. Поиск **выделенных узлов,** а затем выберите **выделенные хосты** из результатов.
1. На странице **Выделенных Хостов** выберите **Создать**.
1. Выберите идентификатор подписки, которую вы хотите использовать.
1. Выберите *myDedicatedHostsRG* в качестве **группы ресурсов.**
1. В **деталях instanceа,** введите *myHost* для **имени** и выберите *Восток США* для положения.
1. В **профиле оборудования**, выберите *Стандартное семейство Es3 - Тип 1* для **семейства размеров**, выберите *myHostGroup* для **группы host** и после этого выберите *1* для **домена разлома.** Оставьте по умолчанию для остальных полей.
1. Когда вы закончите, выберите **Обзор и создайте** и дождитесь проверки.
1. Как только вы увидите пройденое сообщение **Проверки,** выберите **Создать** для создания узла.

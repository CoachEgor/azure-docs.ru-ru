|----------|---------------|
| Максимальный размер контейнера с одной каплей | То же самое, что и максимальная емкость учетной записи хранилища |
| Максимальное количество блоков в блоке капли или приложение капли | 50 000 блоков |
| Максимальный размер блока в блоке капли | 100 МиБ |
| Максимальный размер блочного кабы | 50 000 X 100 MiB (примерно 4,75 TiB) |
| Максимальный размер блока в каплях приложения | 4 МиБ |
| Максимальный размер капли приложения | 50 000 x 4 МБ (примерно 195 GiB) |
| Максимальный размер капли страницы | 8 ТиБ |
| Максимальное количество сохраненных политик доступа на контейнер с blob | 5 |
|Целевая ставка запроса для одной капли | До 500 запросов в секунду |
|Целевая пропускная выливка для одной страницы капли | До 60 МиБ в секунду |
|Целевая пропускная выливка для одного блока капли |До вхинга учетной записи хранилища/накопителя лимитов<sup>1</sup> |

<sup>1</sup> Пропускная способность для одной капли зависит от нескольких факторов, включая, но не ограничиваемый: параллелизм, размер запроса, уровень производительности, скорость исхода для загрузки и назначение для загрузок. Чтобы воспользоваться преимуществами повышения производительности [высокой пропускной способности блока капли](https://azure.microsoft.com/blog/high-throughput-with-azure-blob-storage/), загрузить большие капли или блоки. В частности, позвоните в [операцию Put Blob](/rest/api/storageservices/put-blob) или [Put Block](/rest/api/storageservices/put-block) с blob или размером блока, который превышает 4 MiB для стандартных учетных записей хранения. Для премиум-блока капли или для data Lake Storage Gen2 хранения счетов, используйте блок или blob размер, который больше, чем 256 КиБ.

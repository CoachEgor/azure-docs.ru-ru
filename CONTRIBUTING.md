# <a name="contributing-to-microsoft-azure-documentation"></a>Дополнение к документации по Microsoft Azure

Спасибо, что уделили время и участия в разработке документации Microsoft Azure.

В этом руководстве рассматриваются некоторые общие темы, вокруг вклад и ссылается наших [руководством для участников](https://docs.microsoft.com/contribute) более подробные объяснения, где применимо.

## <a name="code-of-conduct"></a>Правила поведения

Microsoft Open Source Code of Conduct [Microsoft Open Source Code of Conduct](https://opensource.microsoft.com/codeofconduct/).
Дополнительные сведения см. в разделе [код из поведения часто задаваемые вопросы о](https://opensource.microsoft.com/codeofconduct/faq/), или обратитесь в службу [ opencode@microsoft.com ](mailto:opencode@microsoft.com) с вопросы или комментарии.

## <a name="how-can-i-contribute"></a>Показывает, как можно влияет?

Существует множество способов участия в разработке документации, ознакомьтесь с разделами ниже, чтобы узнать, какой из них подходит вам.

### <a name="reporting-bugs-and-suggesting-enhancements"></a>Сообщения об ошибках и предложением усовершенствования

Чтобы отправлять ошибки и предложения используйте средство обратной связи в нижней части статьи.

![Средство обратной связи](media/feedback-tool.png)

### <a name="editing-in-github"></a>Редактирование в GitHub

Следуйте указаниям для [Быстрое редактирование существующих документов](https://docs.microsoft.com/contribute/#quick-edits-to-existing-documents) в нашем руководстве для участников.

### <a name="pull-request"></a>Запрос Pull

Просмотрите руководство по [запросов на Вытягивание](https://docs.microsoft.com/contribute/how-to-write-workflows-major#pull-request-processing) в нашем руководстве для участников.
